Preconditions: user has multiple plot NFTs; attacker controls daemon. Call sequence: `getPoolState` returns duplicate `launcherId` across two NFTs → user opens login link dialog for NFT-B → cache returns NFT-A's link → user authenticates to wrong pool. Invariant: pool login links must be bound to the correct NFT and pool. Scoped impact: approval hijack, wrong-pool authentication. Proof idea: integration test with two NFTs sharing a `launcherId` and asserting each gets a distinct login link.
