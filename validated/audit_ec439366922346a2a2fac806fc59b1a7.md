### Title
Unprivileged Caller Can Advance Global `last_reward_block` Without Distributing Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

Any non-zero address can call `update_rewards` with `disable_rewards: true`, which advances the **global** `last_reward_block` storage variable to the current block without distributing any consensus rewards. Because `last_reward_block` is a single global value shared across all stakers, this blocks every staker from receiving block rewards in that block. Repeated across consecutive blocks, this permanently freezes all unclaimed consensus-phase yield.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

When called with `disable_rewards: true`, the function:

1. Validates the target staker exists and has non-zero balance (lines 1460–1482).
2. **Writes the current block number into the global `last_reward_block`** (line 1485).
3. Returns immediately without distributing any rewards (lines 1487–1489). [2](#0-1) 

`last_reward_block` is declared as a single global `BlockNumber` — not a per-staker map: [3](#0-2) 

The guard at the top of `update_rewards` enforces that only one call per block can succeed: [4](#0-3) 

Because `last_reward_block` is global, a single call with `disable_rewards: true` for **any** valid staker exhausts the per-block reward slot for **all** stakers. No legitimate reward update can succeed in the same block.

This is structurally identical to the reported pattern: a state-advancing operation (timer/block-counter reset) that bypasses the economic action (reward distribution) it is supposed to accompany, callable without any privileged role.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block prevents every staker from ever accumulating consensus-phase block rewards. The `last_reward_block` is advanced each time, so the per-block guard permanently blocks legitimate reward calls. All unclaimed yield in the consensus rewards phase is frozen indefinitely.

---

### Likelihood Explanation

**High.** The function is fully public (only `general_prerequisites()` guard). The attacker needs only a valid staker address (readable from on-chain events) and enough gas to submit one transaction per block. There is no economic barrier; the attacker does not need to hold any stake.

---

### Recommendation

Restrict the `disable_rewards` parameter to a privileged caller (e.g., the attestation contract or a governance role), or remove the parameter entirely and handle the pre-consensus early-return internally without advancing `last_reward_block`. Alternatively, make `last_reward_block` a per-staker mapping so that one call cannot block all other stakers.

---

### Proof of Concept

1. Attacker observes any active staker address `S` from on-chain `NewStaker` events.
2. At the start of every block (or front-running legitimate reward calls), attacker submits:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `last_reward_block` is set to the current block number.
4. Any subsequent call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. No staker receives block rewards for that block.
6. Repeated every block, all consensus-phase unclaimed yield is permanently frozen. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
