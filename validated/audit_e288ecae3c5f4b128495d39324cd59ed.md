### Title
Staker's Own Balance Used as Total Stake Denominator in `update_rewards`, Granting 100% of Block Rewards to Caller - (File: src/staking/staking.cairo)

### Summary

In `StakingRewardsManagerImpl::update_rewards`, the individual staker's balance (`staker_total_strk_balance`) is passed as the `strk_total_stake` argument to `_update_rewards`, instead of the global total staking power. This mirrors the H-01 pattern: an attacker-controlled or incorrectly-scoped value is substituted into a reward-distribution denominator, inflating the caller's reward share to 100% of all block rewards per block.

### Finding Description

`update_rewards` (callable by any unprivileged address) computes the staker's individual balance and then passes it as the network-wide total stake to `_update_rewards`:

```cairo
// src/staking/staking.cairo lines 1474-1506
let (staker_total_strk_balance, staker_total_btc_balance) = self
    .get_staker_total_strk_btc_balance_at_epoch(
        :staker_address, :staker_pool_info, epoch_id: curr_epoch,
    );
...
let (strk_block_rewards, btc_block_rewards) = self
    .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
self
    ._update_rewards(
        :staker_address,
        strk_total_rewards: strk_block_rewards,
        btc_total_rewards: btc_block_rewards,
        strk_total_stake: staker_total_strk_balance,   // ← staker's own balance
        btc_total_stake:  staker_total_btc_balance,    // ← staker's own balance
        ...
    );
```

Contrast this with `update_rewards_from_attestation_contract`, which correctly passes the **global** total staking power:

```cairo
// src/staking/staking.cairo lines 1408-1422
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
self
    ._update_rewards(
        :staker_address,
        strk_total_rewards: strk_epoch_rewards,
        btc_total_rewards:  btc_epoch_rewards,
        :strk_total_stake,   // ← global total
        :btc_total_stake,    // ← global total
        ...
    );
```

`_update_rewards` computes each staker's proportional share as:

```
staker_reward = total_rewards × staker_balance / total_stake
```

When `total_stake = staker_balance` (as passed in `update_rewards`):

```
staker_reward = total_rewards × staker_balance / staker_balance = total_rewards
```

The calling staker receives **100% of the epoch's block rewards** for that block, regardless of their actual network share.

### Impact Explanation

`update_rewards` is gated only by `last_reward_block` (one successful call per block) and `general_prerequisites` (unpaused, non-zero caller). Any registered staker can call it. By front-running every block, an attacker staker claims the entire block reward budget for themselves, permanently denying other stakers their proportional yield. Over a full epoch this drains the entire consensus-reward pool allocated to all other participants.

**Impact class**: Theft of unclaimed yield — **High**.

### Likelihood Explanation

- `update_rewards` is a public, permissionless function with no role check.
- The only race condition is being first in a block; on Starknet, a staker running their own sequencer node or submitting with a high fee can reliably win this race every block.
- The attacker only needs to be a registered staker (minimum stake threshold), which is a low barrier.

**Likelihood**: High.

### Recommendation

Replace the staker-scoped balance with the network-wide total staking power, matching the pattern used in `update_rewards_from_attestation_contract`:

```cairo
// Replace:
strk_total_stake: staker_total_strk_balance,
btc_total_stake:  staker_total_btc_balance,

// With:
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
// then pass:
:strk_total_stake,
:btc_total_stake,
```

### Proof of Concept

1. Attacker registers as a staker with the minimum stake (e.g. 1 STRK).
2. Each block, attacker submits `update_rewards(attacker_address, false)` as the first transaction.
3. Inside `update_rewards`:
   - `staker_total_strk_balance` = attacker's 1 STRK (normalized).
   - `strk_block_rewards` = full network block reward (e.g. 1000 STRK equivalent).
   - `_update_rewards` is called with `strk_total_stake = 1 STRK`.
   - Reward computed: `1000 STRK × 1 / 1 = 1000 STRK` → attacker credited 1000 STRK.
4. All other stakers receive 0 STRK for that block.
5. Repeated every block across an epoch, the attacker drains the entire block-reward budget.

**Access gate (permissionless)**: [3](#0-2)
