### Title
Unprivileged Caller Can Permanently Freeze Block Rewards for All Stakers via `update_rewards(disable_rewards: true)` — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract accepts a caller-controlled `disable_rewards` boolean parameter with no access control. Because `last_reward_block` is a **global** state variable updated on every call, any unprivileged address can front-run legitimate reward updates at each block with `disable_rewards: true`, causing all stakers to permanently lose their consensus block rewards.

---

### Finding Description

`update_rewards` is part of the public `IStakingRewardsManager` interface. Its only gate is `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role check exists. [1](#0-0) 

The function writes `last_reward_block = current_block_number` unconditionally, then branches on `disable_rewards`: [2](#0-1) 

Because `last_reward_block` is a single global storage slot: [3](#0-2) 

…once it is set to block N, the guard `current_block_number > last_reward_block` prevents any second call at block N: [4](#0-3) 

An attacker who calls `update_rewards(any_valid_active_staker, disable_rewards: true)` at block N therefore:
1. Advances `last_reward_block` to N.
2. Distributes zero rewards (early return).
3. Blocks every legitimate `update_rewards` call for block N with `REWARDS_ALREADY_UPDATED`.

The attacker only needs a publicly observable valid staker address (emitted in `NewStaker` events) and enough gas to front-run one transaction per block.

By contrast, the pre-consensus reward path (`update_rewards_from_attestation_contract`) is correctly restricted to the trusted attestation contract: [5](#0-4) 

The consensus-rewards path has no equivalent restriction.

---

### Impact Explanation

Every block where the attacker front-runs, **all** stakers lose their consensus block rewards permanently — those rewards are never re-queued. This constitutes permanent freezing of unclaimed yield across the entire protocol. The attacker has no profit motive; the damage is pure griefing of stakers and the protocol.

Matches allowed impact: **High — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

- Entry point is fully public; no role, no signature, no stake required.
- The attacker only needs to know one valid active staker address (trivially obtained from on-chain events).
- On Starknet, gas costs per transaction are low, making continuous per-block front-running economically viable.
- The attack is silent — it looks like a normal (if early) `update_rewards` call.

Likelihood: **High.**

---

### Recommendation

Remove the `disable_rewards` parameter from the public interface entirely, or restrict it to a privileged role (e.g., `only_security_agent` or `only_app_governor`). If the parameter is needed for migration purposes, gate it behind an appropriate role check analogous to how `update_rewards_from_attestation_contract` gates its caller:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   if disable_rewards {
+       self.roles.only_security_agent(); // or appropriate role
+   }
    ...
}
```

Alternatively, split the function into two: one public function (always distributes rewards) and one privileged function (can disable rewards).

---

### Proof of Concept

```
// Attacker contract (no special privileges needed)
// Called once per block, front-running legitimate staker calls.

fn attack(staking: IStakingRewardsManagerDispatcher, valid_staker: ContractAddress) {
    // disable_rewards = true → last_reward_block advances, zero rewards distributed
    staking.update_rewards(staker_address: valid_staker, disable_rewards: true);
    // All subsequent update_rewards calls at this block now revert with REWARDS_ALREADY_UPDATED
}
```

Step-by-step:
1. Consensus rewards become active (`consensus_rewards_first_epoch` is set).
2. At block N, attacker calls `update_rewards(valid_staker, true)`.
3. `last_reward_block` is written to N; no rewards distributed.
4. Staker's own `update_rewards(staker, false)` at block N reverts: `REWARDS_ALREADY_UPDATED`.
5. Staker permanently loses block-N rewards.
6. Repeat every block.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
