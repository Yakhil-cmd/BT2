### Title
`TurnBridge` Is a No-Op Stub: Bridge Circuit-Breaker Permanently Broken and Callable by Any Unprivileged User — (`File: x/cronos/keeper/msg_server.go`)

### Summary

The `MsgTurnBridge` handler in `x/cronos/keeper/msg_server.go` is an unimplemented stub that returns `nil, nil` unconditionally. It performs no permission check and changes no state. As a result: (1) any unprivileged user can submit `MsgTurnBridge` and receive a success response, and (2) the bridge can never actually be disabled — the emergency circuit-breaker is permanently non-functional.

### Finding Description

ADR-009 explicitly designates `MsgTurnBridge` as a permission-restricted message requiring `CanTurnBridge`:

> "For now, there are only two messages that require permission: `MsgUpdateTokenMapping` and `MsgTurnBridge`."

`UpdateTokenMapping` correctly enforces this:

```go
// x/cronos/keeper/msg_server.go:69-81
func (k msgServer) UpdateTokenMapping(...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, ...)
    }
    ...
}
```

But `TurnBridge` is a dead stub:

```go
// x/cronos/keeper/msg_server.go:84-87
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

No `HasPermission(ctx, ..., CanTurnBridge)` check is present, and no state is written. The `CanTurnBridge` permission constant is defined and the permission system is wired up: [2](#0-1) 

But the handler that is supposed to use it is empty.

The proto definition and the `MsgServer` interface both declare `TurnBridge` as a live RPC endpoint: [3](#0-2) [4](#0-3) 

The CLI command `CmdTurnBridge` is also fully wired and broadcasts the message to the chain: [5](#0-4) 

### Impact Explanation

**High — Bypass of bridge authorization/control mechanism and permanent inability to disable bridge flows.**

Two distinct impacts compound:

1. **Missing permission check (unprivileged access):** Any address can submit `MsgTurnBridge` and receive a `200 OK` response. The `CanTurnBridge` permission gate that ADR-009 mandates is never evaluated.

2. **Bridge circuit-breaker is permanently broken:** The bridge disable mechanism (`bridge_active`) can never be set to `false` through the intended path. If a security incident requires an emergency halt of bridge operations, the admin's `MsgTurnBridge(enable=false)` call silently succeeds but changes nothing. Bridge-related flows (EVM hook handlers, IBC transfers triggered by `__CronosSendToIbc`, `__CronosSendToEvmChain`) continue to execute unimpeded.

This directly matches the allowed High impact: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks"* and *"Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows."*

### Likelihood Explanation

- The message is fully registered, has a CLI command, and is reachable by any unprivileged user on-chain.
- The missing permission check means no barrier to calling it.
- The no-op body means the bridge state is immutably stuck at its genesis value — the circuit-breaker can never be activated.
- The integration test `test_gravity_turn_bridge` expects `turn_bridge("false", from_="validator")` to succeed and then block `send_to_evm_chain`, but the keeper implementation guarantees this test would fail against the actual keeper logic. [1](#0-0) 

### Recommendation

Implement `TurnBridge` with the required permission check and state update, consistent with how `UpdateTokenMapping` is implemented:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)

    senderAddr, err := sdk.AccAddressFromBech32(msg.Sender)
    if err != nil {
        return nil, err
    }
    if !k.HasPermission(ctx, []sdk.AccAddress{senderAddr}, CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }

    // Update the bridge_active parameter in the gravity/cronos params
    params := k.GetParams(ctx)
    params.BridgeActive = msg.Enable  // or equivalent field
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

The exact field name for `bridge_active` should match the gravity bridge params structure. All EVM hook handlers that process bridge-related events (`SendToEvmChain`, `SendToIbc`) should also gate on this parameter.

### Proof of Concept

1. Any unprivileged address submits `MsgTurnBridge{Sender: attacker, Enable: false}`.
2. The handler returns `nil, nil` — transaction succeeds with no error.
3. No state is written; bridge remains active.
4. Admin submits the same message — also returns success, also changes nothing.
5. Bridge flows (`__CronosSendToEvmChain`, `__CronosSendToIbc` EVM hooks, `IbcTransferCoins`) continue to execute indefinitely regardless of any `TurnBridge` calls.
6. The emergency circuit-breaker is permanently inoperable. [1](#0-0) [6](#0-5) [7](#0-6)

func CmdTurnBridge() *cobra.Command {

		Short: "Turn Bridge",
		Args:  cobra.ExactArgs(1),

				return err

			if err != nil {
				return err
			}

				return err
			}
			return tx.GenerateOrBroadcastTxCLI(clientCtx, cmd.Flags(), msg)
		},
	}
