### Title
Unrestricted `update_rewards` Allows Anyone to Permanently Suppress Block Rewards for All Stakers - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `StakingRewardsManagerImpl` has no caller restriction. Any unprivileged address can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` to the current block without distributing any rewards. Because `last_reward_block` is a single contract-wide storage slot, one such call per block permanently forfeits that block's rewards for every staker in the system.

### Finding Description
`update_rewards` is exposed as a public entry point with no role check: [1](#0-0) 

The function first writes the current block number to the global `last_reward_block` storage slot, then branches on `disable_rewards`: [2](#0-1) 

The storage field is declared as a single (non-per-staker) value: [3](#0-2) 

Because `last_reward_block` is updated **before** the early-return guard, any caller who passes a valid active staker address and sets `disable_rewards: true` will:
1. Consume the block's reward slot (`last_reward_block = current_block`).
2. Skip all reward distribution.
3. Cause every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

The only prerequisite is supplying a valid, active staker address with non-zero STRK balance — information that is fully public from on-chain events (`NewStaker`, `StakeOwnBalanceChanged`).

Compare with the properly guarded setter functions in the same contract, all of which call a role-check before touching state: [4](#0-3) [5](#0-4) 

`update_rewards` has no equivalent guard.

### Impact Explanation
Every block in which the attacker fires `update_rewards(valid_staker, true)`:
- `last_reward_block` is stamped with that block number.
- No STRK block rewards are computed or credited to any staker.
- The block's reward window is permanently closed; it cannot be reclaimed.

Repeated across every block, this permanently freezes all consensus-era block rewards for the entire staking system. This maps directly to **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- The function is callable by any EOA or contract with no special role.
- The only input needed is any active staker address, which is trivially obtained from emitted `NewStaker` events.
- The cost is a single cheap transaction per block.
- There is no economic barrier or time-lock preventing continuous execution.

Likelihood is **High**.

### Recommendation
Add a caller restriction to `update_rewards` so that only the attestation contract (or another designated privileged caller) may invoke it. For example:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.assert_caller_is_attestation_contract(); // add this guard
    self.general_prerequisites();
    ...
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "no-reward" path internally, gated behind the attestation contract caller check already used by `update_rewards_from_attestation_contract`. [6](#0-5) 

### Proof of Concept
1. Deploy or identify any active staker (address available from `NewStaker` events).
2. Each block, call:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(known_staker, disable_rewards=true)
   ```
3. Observe that `last_reward_block` advances to the current block with no rewards distributed.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers receive zero block rewards for every block the attacker executes step 2.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
