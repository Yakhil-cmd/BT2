### Title
`update_rewards` Uses a Global `last_reward_block` With No Access Control, Allowing Any Caller to Monopolize All Consensus Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable with no role restriction. It uses a single **global** `last_reward_block` variable (not per-staker) as its only guard against double-distribution. Any unprivileged staker can call `update_rewards` for their own staker address every block, monopolizing all consensus block rewards and starving every other staker of their yield.

---

### Finding Description

`update_rewards` is exposed as part of `IStakingRewardsManager` and is callable by any non-zero address:

```cairo
// src/staking/staking.cairo:1448-1507
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only checks: not paused, caller != zero
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        self.last_reward_block.write(current_block_number);
        ...
        // distributes strk_block_rewards to staker_address
    }
}
``` [1](#0-0) 

`general_prerequisites` imposes no role check:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [2](#0-1) 

`last_reward_block` is a **single global value** in storage, not a per-staker map:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [3](#0-2) 

Because the guard is `current_block_number > last_reward_block` (global), only **one staker** can receive block rewards per block across the entire protocol. The first caller in each block wins; all subsequent callers for other stakers revert with `REWARDS_ALREADY_UPDATED`.

The block rewards distributed per call are computed by `calculate_block_rewards`, which returns a fixed per-block STRK/BTC amount for the current epoch:

```cairo
fn calculate_block_rewards(
    ref self: ContractState,
    reward_supplier_dispatcher: IRewardSupplierDispatcher,
    curr_epoch: Epoch,
) -> (Amount, Amount) {
    if curr_epoch > self.last_calculated_epoch.read() {
        ...
        let block_rewards = reward_supplier_dispatcher.update_current_epoch_block_rewards();
        self.block_rewards.write(block_rewards);
        block_rewards
    } else {
        self.block_rewards.read()
    }
}
``` [4](#0-3) 

An attacker who calls `update_rewards(attacker_staker_address, false)` at the start of every block:
1. Receives their proportional share of every block's rewards.
2. Sets `last_reward_block` to the current block, preventing any other staker from receiving rewards for that block.
3. Repeats this across all blocks in an epoch, accumulating rewards that should have been distributed to other stakers.

Additionally, the `disable_rewards: bool` parameter allows the attacker to call `update_rewards(any_staker, true)` every block to burn the global slot without distributing rewards to anyone — a pure griefing vector.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

In consensus mode (post `consensus_rewards_first_epoch`), block rewards are the primary yield mechanism. By monopolizing `last_reward_block` every block, the attacker captures their proportional share of every block's rewards while all other stakers receive zero yield for those blocks. The stolen yield is real STRK transferred from the reward supplier to the attacker's reward address. Other stakers' unclaimed yield is permanently lost for every block the attacker front-runs.

---

### Likelihood Explanation

**High.** The entry point is a public, permissionless function. No special role, key, or privilege is required. Any registered staker with a non-zero balance can execute this attack by submitting a transaction at the start of each block. The cost is only gas per block, and the attacker profits proportionally to their stake share on every block.

---

### Recommendation

Replace the single global `last_reward_block` with a **per-staker** last-rewarded-block map, or add an access-control check so that only the consensus layer or the staker's own operational address can call `update_rewards`. The minimal fix:

```cairo
// Replace:
last_reward_block: BlockNumber,

// With:
last_reward_block: Map<ContractAddress, BlockNumber>,
```

And in `update_rewards`, guard against the per-staker block:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(staker_address),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
...
self.last_reward_block.write(staker_address, current_block_number);
```

Alternatively, restrict the caller to the staker's registered operational address or a privileged consensus role.

---

### Proof of Concept

1. Alice and Bob are both registered stakers with equal stake.
2. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
3. Every block, Alice calls `update_rewards(alice_staker_address, false)` before any other transaction.
4. `last_reward_block` is updated to the current block; Alice receives her proportional block reward.
5. Bob (or anyone acting on Bob's behalf) calls `update_rewards(bob_staker_address, false)` in the same block → reverts with `REWARDS_ALREADY_UPDATED`.
6. Bob receives zero rewards for that block.
7. After one full epoch of N blocks, Alice has received N × (her proportional block reward); Bob has received 0.
8. Alice has stolen Bob's entire epoch yield.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;

                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
