### Title
Unprivileged Caller Can Permanently Freeze All Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable by any non-zero address and accepts a `disable_rewards: bool` parameter. When called with `disable_rewards: true`, the function advances the global `last_reward_block` checkpoint to the current block without distributing any rewards. Because the contract enforces exactly one reward update per block via this checkpoint, an attacker can call this every block to permanently prevent all stakers from receiving consensus rewards.

---

### Finding Description

`update_rewards` is exposed through the `IStakingRewardsManager` interface with no caller access control beyond the generic `general_prerequisites()` check, which only asserts the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function enforces a single-call-per-block invariant by comparing `current_block_number` against the global `last_reward_block` storage variable: [2](#0-1) 

After validating the staker, it unconditionally writes `current_block_number` to `last_reward_block`, then branches on `disable_rewards`: [3](#0-2) 

When `disable_rewards` is `true`, the function returns immediately — no rewards are calculated or distributed — but the block slot is consumed. Any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

The `general_prerequisites` helper confirms there is no role check: [4](#0-3) 

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at every block permanently destroys that block's consensus rewards for **all** stakers. The rewards are not deferred or accumulated elsewhere; they are simply never computed or transferred. Sustained over time this constitutes **permanent freezing of unclaimed yield** for the entire protocol.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield or unclaimed royalties**.

---

### Likelihood Explanation

- No privileged role, leaked key, or external dependency is required.
- The only precondition is supplying any currently-active staker address (publicly readable from the `stakers` vector).
- The attack is fully automatable: a bot monitors each new block and submits the griefing transaction.
- Gas cost on Starknet is low relative to the value of rewards destroyed across all stakers.

Likelihood: **High**.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus layer contract or a dedicated role). Alternatively, remove the `disable_rewards` parameter from the public ABI and handle the "no-reward" case internally based on protocol state, so that the block slot cannot be consumed without distributing rewards.

---

### Proof of Concept

1. Attacker identifies any active staker address `S` (readable from the public `stakers` vector in storage).
2. At every new Starknet block `N`, attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. Inside the call:
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - Staker `S` is validated as active with non-zero balance.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → early return, zero rewards distributed.
4. Any legitimate call to `update_rewards` at block `N` now reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated each block, **no consensus rewards are ever distributed** to any staker in the protocol. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
