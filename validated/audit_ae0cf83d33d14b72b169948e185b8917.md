### Title
Unauthorized Transfer and Burn of Native `evm/*` Tokens via Unguarded `sender` Parameter in Bank Precompile â€” (File: `x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The bank precompile's `transfer` and `burn` methods accept an arbitrary `sender`/`recipient` address from calldata without verifying that the calling contract is authorized to act on behalf of that address. Any unprivileged contract can transfer or burn `evm/<contract_address>` native tokens from any victim address that holds them, without the victim's consent.

---

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, the `Run` function handles three state-mutating operations: `mint`, `burn`, and `transfer`. The denom for all three is derived from `contract.Caller()` â€” the address of the EVM contract calling the precompile â€” producing `evm/<caller_address>`.

**`transfer` (lines 167â€“200):**

```go
sender := args[0].(common.Address)   // fully attacker-controlled calldata
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
to := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())  // evm/<attacker_contract>
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

There is no check that `sender == contract.Caller()`. The precompile unconditionally calls `SendCoins` from the attacker-supplied `from` address.

**`burn` (lines 113â€“156):**

```go

...

...

bc.bankKeeper.BurnCoins(ctx, types.ModuleName, ...)
```

Again, no check that `recipient == contract.Caller()`. The precompile burns tokens from any address supplied in calldata.

---

### Impact Explanation

**Critical â€” Unauthorized transfer and burn of `evm/*` native tokens.**

Any holder of `evm/<attacker_contract>` tokens is fully exposed. The attacker controls the denom (by controlling the calling contract address) and can freely move or destroy any victim's balance of that denom. Since `evm/*` tokens are first-class Cosmos SDK native coins tracked by the bank module, this constitutes a direct unauthorized balance change for precompile-controlled assets.

---

### Likelihood Explanation

High. The attack requires only:
1. Deploying a contract (unprivileged, costs gas only).
2. Distributing `evm/<attacker_contract>` tokens to victims â€” trivially done via the `mint` method of the same precompile, which has no access control and lets any contract mint its own denom to any address.
3. Calling `transfer(victim, attacker, amount)` or `burn(victim, amount)` from the attacker contract.

No admin keys, governance, or privileged access is required.

---

### Recommendation

In the `transfer` case, enforce that the `sender` argument equals `contract.Caller()`:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be the calling contract")
}
```

In the `burn` case, enforce that the address being burned from equals `contract.Caller()`:

```go
if recipient != contract.Caller() {
    return nil, errors.New("can only burn from the calling contract's own account")
}
```

This mirrors the authorization model used by `mint_by_cronos_module` in the CRC20/CRC21 contracts, which gate all privileged operations behind `require(msg.sender == module_address)`.
