### Title
`protocolUnpausePool` Always Transitions to Admin-Paused State (Level 1) Instead of Active (Level 0), Leaving Pool Permanently Paused Without Additional Admin Action — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.protocolUnpausePool` unconditionally sets the pool's pause level to `1` (admin-paused) rather than `0` (active). Because `_checkNotPaused()` reverts for any non-zero pause level, swaps remain blocked after the protocol owner calls `protocolUnpausePool`. The pool admin must then separately call `unpausePool` to restore full functionality — an action they may be unable or slow to perform, leaving the pool stuck in a paused state the admin never set.

---

### Finding Description

The pause-level state machine in `MetricOmmPool` uses three levels:

| Level | Meaning |
|-------|---------|
| `0` | Active — swaps allowed |
| `1` | Admin-paused — swaps blocked |
| `2` | Protocol-paused — swaps blocked |

`protocolPausePool` can pause from **either** level `0` (active) or level `1` (admin-paused):

```solidity
// MetricOmmPoolFactory.sol L393-L396
function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}
```

`protocolUnpausePool` **always** transitions to level `1`, regardless of what the original state was before the protocol pause:

```solidity
// MetricOmmPoolFactory.sol L399-L403
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1);   // ← always 1, never 0
}
```

`_checkNotPaused()` in `MetricOmmPool` blocks swaps for **any** non-zero level:

```solidity
// MetricOmmPool.sol L643-L645
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
```

The `whenNotPaused` modifier (which calls `_checkNotPaused`) is applied to `swap`:

```solidity
// MetricOmmPool.sol L224
) external whenNotPaused nonReentrant(PoolActions.SWAP) returns (int128, int128) {
```

**Concrete scenario:**

1. Pool is at level `0` (active, swaps working normally).
2. Protocol owner calls `protocolPausePool` → pool moves to level `2`.
3. Protocol owner calls `protocolUnpausePool` → pool moves to level `1` (admin-paused), **not** `0`.
4. Swaps still revert with `PoolPaused()`.
5. Pool admin must now call `unpausePool` (level `1` → `0`) to restore the pool — an action the admin never initiated and may not be able to perform promptly.

The factory stores no record of the pre-pause level, so it cannot restore the original state. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

After `protocolUnpausePool` executes, the pool is still paused at level `1`. All swap calls revert with `PoolPaused()`. Liquidity providers cannot trade through the pool; any router or aggregator routing through it will fail. The pool is functionally insolvent from a swap perspective until the pool admin separately calls `unpausePool`. If the pool admin is a slow-moving multisig, DAO, or an unavailable EOA, the pool can remain stuck indefinitely in a state the admin never set — a broken core-functionality invariant. [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

The trigger is a normal, expected operational sequence: the protocol owner pauses a pool (e.g., during an incident) and later unpauses it. This is a semi-trusted, valid action path. No malicious setup or non-standard tokens are required. The bug fires every time `protocolUnpausePool` is called on a pool that was active (level `0`) before the protocol pause. [6](#0-5) 

---

### Recommendation

Track the pre-pause level in factory storage when `protocolPausePool` is called, and restore it in `protocolUnpausePool`. Alternatively, if the intent is always to restore to fully active, change the target in `protocolUnpausePool` from `1` to `0`:

```solidity
// Fix: restore to active (0) instead of admin-paused (1)
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
}
```

If the two-step design (protocol unpause → admin-paused → admin unpause) is intentional, it must be clearly documented and the pool admin must be guaranteed to be reachable, otherwise the pool can be permanently stuck. [1](#0-0) 

---

### Proof of Concept

```
State: pool at pauseLevel = 0 (active)

1. owner calls protocolPausePool(pool)
   → setPause(2) → pauseLevel = 2

2. owner calls protocolUnpausePool(pool)
   → setPause(1) → pauseLevel = 1   ← BUG: should be 0

3. user calls pool.swap(...)
   → whenNotPaused → _checkNotPaused()
   → pauseLevel (1) != 0 → revert PoolPaused()   ← swap still blocked

4. Only fix: poolAdmin calls unpausePool(pool)
   → checks cur == 1 ✓ → setPause(0) → pauseLevel = 0
   (admin never paused the pool, but must now unpause it)
``` [6](#0-5) [7](#0-6)
