### Title
Gateway `skip_stateful_validations` Admits Invoke Transactions with Arbitrary Signatures When Any Account Transaction Exists in Mempool - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function in the gateway bypasses the `__validate__` entry-point call (and therefore all signature verification) for any invoke transaction with `nonce=1` sent to an account whose on-chain nonce is still `0`, provided `account_tx_in_pool_or_recent_block` returns `true` for that address. Because `account_tx_in_pool_or_recent_block` returns `true` for **any** transaction type in the pool — not specifically a `deploy_account` — an unprivileged attacker can inject an invoke with a completely fabricated signature into the mempool for any account that has a pending `deploy_account`, bypassing the gateway's only signature-verification gate.

---

### Finding Description

**Broken invariant:** Every transaction admitted to the mempool must have passed account-contract signature verification (`__validate__`).

**Root cause — `skip_stateful_validations`:**

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  lines 429-461
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())   // ← any tx type
                .await
                ...
        }
    }
    Ok(false)
}
```

When `skip_validate` is `true`, `run_validate_entry_point` sets `execution_flags.validate = false`:

```rust
// lines 311-312
let execution_flags =
    ExecutionFlags { only_query, charge_fee, validate: !skip_validate, strict_nonce_check };
```

And `validate_tx` in the blockifier short-circuits immediately:

```rust
// crates/blockifier/src/transaction/account_transaction.rs  lines 999-1001
if !self.execution_flags.validate {
    return Ok(None);
}
```

**The mempool check is not type-specific:**

```rust
// crates/apollo_mempool/src/mempool.rs  lines 697-700
pub fn account_tx_in_pool_or_recent_block(&self, account_address: ContractAddress) -> bool {
    self.state.contains_account(account_address)
        || self.tx_pool.contains_account(account_address)
}
```

`tx_pool.contains_account` returns `true` for **any** transaction type (invoke, declare, deploy_account) belonging to that address. The code comment claims this is sufficient because "it means that either it has a deploy_account transaction or transactions with future nonces that passed validations," but the second branch is circular: those future-nonce transactions may themselves have been admitted with `skip_validate = true`.

**Attack path:**

1. Alice submits a valid `deploy_account` for address `X` (nonce=0). It enters the mempool.
2. Attacker calls `add_tx` with an `InvokeV3` for address `X`, nonce=1, **empty/garbage signature**.
   - `account_nonce` from state = 0 ✓
   - `tx.nonce()` = 1 ✓
   - `account_tx_in_pool_or_recent_block(X)` = `true` (Alice's deploy_account is in the pool) ✓
   - `skip_validate` = `true` → `__validate__` is **never called**
   - Transaction is accepted into the mempool with no signature check.
3. The mempool now holds both Alice's `deploy_account` and the attacker's unsigned invoke. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid (unsigned) transactions before sequencing.**

The gateway is the sole point where `__validate__` is run before a transaction enters the mempool. Once `skip_validate = true`, no signature check occurs at any earlier stage. The batcher will later run `__validate__` with `ExecutionFlags::default()` (`validate: true`), so the fake invoke will be rejected at execution time and not land on-chain. However:

- The gateway's admission invariant is broken: an unsigned invoke occupies a mempool slot.
- The attacker can do this for every account that has a pending `deploy_account`, flooding the mempool with unsigned transactions at negligible cost (no fee is charged for rejected transactions).
- If the mempool enforces per-account nonce uniqueness, Alice's legitimate nonce=1 invoke may be blocked or delayed until the batcher processes and discards the attacker's fake transaction. [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

**Likelihood: Medium.**

- The window is open for every account that has submitted a `deploy_account` but whose transaction has not yet been included in a block.
- The attacker needs no special privilege — only the ability to call the public gateway `add_tx` endpoint.
- The attacker must know (or guess) the target account address, which is deterministic from `class_hash`, `salt`, and `constructor_calldata` and is publicly visible in the mempool.
- The attack is cheap: no fee is required because the transaction is never executed on-chain.

---

### Recommendation

Replace the generic `account_tx_in_pool_or_recent_block` check with a check that specifically confirms a **`deploy_account`** transaction exists for the address. The mempool should expose a dedicated query such as `deploy_account_in_pool(address) -> bool` that only returns `true` when the pending transaction is of type `DeployAccount`. This mirrors the `PyValidator` approach in `native_blockifier`, which requires an explicit `deploy_account_tx_hash` to be passed by the caller before skipping validation:

```rust
// crates/native_blockifier/src/py_validator.rs  lines 109-110
let deploy_account_not_processed =
    deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
``` [6](#0-5)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
