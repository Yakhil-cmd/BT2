### Title
`old_threshold = 0` Bypasses Intersection Enforcement in Reshare, Allowing Key Substitution — (`src/dkg.rs`)

### Summary

`assert_reshare_keys_invariants` validates the **new** threshold through `assert_key_invariants` (enforcing `>= 2`) but applies **no lower-bound validation** to `old_threshold`. When a caller passes `old_threshold = 0`, the intersection-size guard `intersection.len() < old_threshold` becomes a `usize < 0` comparison that is always `false`, silently bypassing the check. A reshare then proceeds with zero old-participant contributions, producing a key entirely unrelated to the original shared secret.

---

### Finding Description

In `src/dkg.rs`, `assert_reshare_keys_invariants` validates the new threshold via `assert_key_invariants` (which rejects any value `< 2`), but the `old_threshold` argument is only converted to `usize` and used directly in a single comparison: [1](#0-0) 

```rust
let threshold = usize::from(threshold.into());
let old_threshold = usize::from(old_threshold.into());

let participants = assert_key_invariants(participants, me, threshold)?;   // ← validates new threshold ≥ 2
// ...
if old_participants.intersection(&participants).len() < old_threshold {   // ← old_threshold unchecked
    return Err(InitializationError::NotEnoughParticipantsForNewThreshold { … });
}
```

`ReconstructionLowerBound` is a plain `usize` newtype with no invariant: [2](#0-1) 

When `old_threshold = 0`, the guard `intersection.len() < 0usize` is vacuously `false` for every possible intersection size, including an empty one. The function returns `Ok`, and `do_reshare` is entered.

Inside `do_reshare`, participants not present in the intersection contribute `secret = 0`: [3](#0-2) 

If the intersection is empty (no old participant is in the new set), every participant's secret is `0`, and `do_keyshare` generates a completely fresh key that is cryptographically unrelated to the old shared secret, while still receiving the `old_public_key` as a supposed consistency anchor.

The public entry point `reshare` in `src/lib.rs` passes `old_threshold` directly from the caller without any prior sanitisation: [4](#0-3) 

By contrast, `refresh` passes `old_threshold` as both the old and new threshold, so `assert_key_invariants` catches a zero value there. The gap exists only in `reshare`.

---

### Impact Explanation

A malicious coordinator who orchestrates a reshare can supply `old_threshold = 0` together with a new participant set that has no overlap with the old set. Every participant's contribution is `0`, so `do_keyshare` produces a fresh random key. Honest participants complete the protocol and store a `KeygenOutput` whose `public_key` is unrelated to the original key. Any subsequent signing sessions using the new shares will produce signatures that verify under the new (attacker-chosen) public key, not the original one — a full key substitution. This matches the allowed impact: **Corruption of reshare outputs so honest parties accept an inconsistent public key**.

---

### Likelihood Explanation

The `reshare` function is a public library entry point. `old_threshold` is a plain caller-supplied integer with no type-level constraint. Any party that initiates or coordinates a reshare (e.g., a malicious new participant who also acts as coordinator) can trivially pass `0`. No cryptographic capability or key material is required beyond the ability to call the function.

---

### Recommendation

Add the same lower-bound guard that `assert_key_invariants` applies to the new threshold, applied to `old_threshold` inside `assert_reshare_keys_invariants`:

```rust
if old_threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: old_threshold,
        min: 2,
    });
}
```

This mirrors the existing pattern: [5](#0-4)

pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],

    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

    let participants = assert_key_invariants(participants, me, threshold)?;

pub fn reshare<C: Ciphersuite>(
    old_participants: &[Participant],
    old_threshold: impl Into<ReconstructionLowerBound> + Send + 'static,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    new_participants: &[Participant],
    new_threshold: impl Into<ReconstructionLowerBound> + Copy + Send + 'static,
    me: Participant,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = KeygenOutput<C>>, InitializationError>
where
    Element<C>: Send,
    Scalar<C>: Send,
{
    let comms = Comms::new();
    let threshold = new_threshold;
    let (participants, old_participants) = assert_reshare_keys_invariants::<C>(
        new_participants,
        me,
        threshold,
        old_signing_key,
        old_threshold,
        old_participants,
    )?;
    let fut = do_reshare(
        comms.shared_channel(),
        participants,
        me,
        threshold,
        old_signing_key,
        old_public_key,
        old_participants,
        rng,
    );
    Ok(make_protocol(comms, fut))
}
