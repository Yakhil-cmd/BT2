### Title
Production Sequencer Node Uses Hardcoded Testing Private Key for Consensus Precommit Vote Signing - (File: crates/apollo_signature_manager/src/lib.rs)

### Summary

The production node component factory calls `create_signature_manager()`, which unconditionally constructs a `LocalKeyStoreSignatureManager` backed by `LocalKeyStore::new_for_testing()`. That function hard-codes a well-known private key (`0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133`) that is publicly visible in the repository. Every sequencer node therefore signs consensus precommit votes and peer-identity challenges with the same, publicly-known key. An explicit TODO in the same file acknowledges that the production key-store design is unresolved.

### Finding Description

`create_signature_manager()` is the sole factory used by the production node to instantiate the `SignatureManager` component:

```rust
// crates/apollo_signature_manager/src/lib.rs
// TODO(Elin): understand how key store would look in production and better define the way the
// signature manager is created.
pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()   // calls LocalKeyStoreSignatureManager::new()
}
```

`LocalKeyStoreSignatureManager::new()` calls `LocalKeyStore::new_for_testing()`:

```rust
impl LocalKeyStoreSignatureManager {
    pub fn new() -> Self {
        Self(GenericSignatureManager::new(LocalKeyStore::new_for_testing()))
    }
}
```

`LocalKeyStore::new_for_testing()` embeds a fixed private key:

```rust
pub(crate) const fn new_for_testing() -> Self {
    const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    ));
    const PUBLIC_KEY: PublicKey = PublicKey(Felt::from_hex_unchecked(
        "0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a",
    ));
    Self { private_key: PRIVATE_KEY, public_key: PUBLIC_KEY }
}
```

The `SignatureManager` is invoked for two operations:

1. `sign_precommit_vote(block_hash)` — produces the ECDSA signature that a validator attaches to its Tendermint PRECOMMIT message, binding a specific `BlockHash` to the validator's identity.
2. `sign_identification(peer_id, challenge)` — authenticates the node's P2P identity.

`create_signature_manager()` is called unconditionally in the production component-creation path:

```rust
// crates/apollo_node/src/components.rs
let signature_manager = match config.components.signature_manager.execution_mode {
    ReactiveComponentExecutionMode::LocalExecutionWithRemoteDisabled
    | ReactiveComponentExecutionMode::LocalExecutionWithRemoteEnabled => {
        Some(create_signature_manager())
    }
    ...
};
```

There is no code path that loads a per-node private key from configuration, environment, or a secrets store. The only constructor that accepts an external key (`fn _new(private_key: PrivateKey)`) is private, prefixed with an underscore, and never called.

### Impact Explanation

Because every node uses the same publicly-known private key:

1. **Forged precommit votes**: Any party that knows the private key (it is in the public repository) can produce a valid ECDSA signature over an arbitrary `BlockHash` that will pass `verify_precommit_vote_signature`. This allows forging consensus votes for blocks that were never actually proposed or executed, binding the wrong block hash to a validator's identity.
2. **Indistinguishable validators**: All honest validators produce identical signatures for the same block hash, making it impossible to attribute votes to individual validators or detect equivocation.
3. **Forged peer identity**: The same key is used for `sign_identification`, so any attacker can impersonate any sequencer node in the P2P layer.

This matches the allowed impact: **"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."** The precommit-vote signature binds the wrong signer (a hardcoded test key instead of the actual validator's key) to every block hash committed by the sequencer.

### Likelihood Explanation

The trigger requires no special privilege. Any operator who deploys the node binary as-is, or any external party who submits a crafted precommit message signed with the known key, can exercise this path. The private key is permanently embedded in the source code and is therefore universally known.

### Recommendation

1. Remove the `new_for_testing()` call from the production `LocalKeyStoreSignatureManager::new()` constructor.
2. Implement a production `KeyStore` that loads the private key from a secrets manager, HSM, or environment variable at startup — analogous to how `OhttpGateway::from_env()` loads its key from `OHTTP_KEY`.
3. Resolve the outstanding TODO: `// TODO(Elin): understand how key store would look in production and better define the way the signature manager is created.`
4. Gate `LocalKeyStore::new_for_testing()` behind `#[cfg(test)]` so it cannot be referenced from non-test code.

### Proof of Concept

```
# 1. Extract the known private key from the public repository:
PRIVATE_KEY=0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133

# 2. For any target block hash H, compute the ECDSA signature using the known key.
#    The message digest is: blake2s("PRECOMMIT_VOTE" || H.to_bytes_be())
#    (see build_precommit_vote_message_digest in signature_manager.rs)

# 3. Submit the forged precommit vote to the consensus layer.
#    verify_precommit_vote_signature(H, forged_sig, PUBLIC_KEY) returns true,
#    because PUBLIC_KEY = 0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a
#    is also publicly known and matches the hardcoded private key.
```

The forged signature is indistinguishable from a legitimate validator precommit, because every deployed node uses the same key pair.

    async fn sign(&self, message_digest: MessageDigest) -> SignatureManagerResult<RawSignature> {
        let private_key = self.keystore.get_key().await?;
        let signature = ecdsa_sign(&private_key, &message_digest)
            .map_err(|e| SignatureManagerError::Sign(e.to_string()))?;

        Ok(signature.into())
    }
