### Title
Missing Authorization Check in `update_rewards` Allows Any Caller to Consume Block Reward Slots Without Distributing Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `Staking` contract is callable by any unprivileged address. It unconditionally writes `last_reward_block` to the current block number **before** checking the `disable_rewards` flag. An attacker can call `update_rewards(valid_staker, disable_rewards: true)` every block, consuming each block's single reward slot without distributing any rewards, permanently preventing all stakers from accumulating yield.

### Finding Description
`update_rewards` is a public function with no role-based access control — `general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function writes `last_reward_block` to the current block number unconditionally, before the `disable_rewards` branch: [2](#0-1) 

After this write, the guard at the top of the function: [3](#0-2) 

…prevents any other call in the same block. Because `last_reward_block` is a single global value shared across all stakers, one call per block is the protocol-wide limit. If that call carries `disable_rewards: true`, the function returns early and no rewards are ever distributed: [4](#0-3) 

The attacker only needs to supply any valid, active staker address with non-zero STRK balance. The full list of staker addresses is publicly readable from the `stakers` storage vector: [5](#0-4) 

The missing check is the analog of the external report's missing `collatValue >= USGToRepay` guard: just as the liquidation function processes a liquidation without verifying the collateral covers the debt, `update_rewards` consumes the block's reward slot without verifying the caller is authorized to suppress rewards.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

By calling `update_rewards(valid_staker, disable_rewards: true)` once per block, an attacker prevents every staker and every pool member from ever accumulating rewards. The `unclaimed_rewards_own` field on staker info is never incremented, and `update_rewards_from_staking_contract` is never called on pool contracts, so pool members' cumulative reward traces are never updated. All yield is permanently frozen at zero. [6](#0-5) 

### Likelihood Explanation
**Medium.**

The attacker must submit one transaction per block. On Starknet, transaction fees are low, making sustained griefing economically feasible for a motivated adversary (e.g., a competing protocol or a large token holder who benefits from suppressing staking rewards). No privileged access, leaked key, or external dependency is required — only a valid staker address, which is publicly enumerable.

### Recommendation
1. **Move `last_reward_block` write after the `disable_rewards` check**, so the slot is only consumed when rewards are actually distributed (or when an authorized party legitimately disables them).
2. **Add an access-control check** (e.g., `only_app_governor` or a dedicated `REWARDS_MANAGER` role) so that only the consensus mechanism or a trusted operator can call `update_rewards`.

```cairo
// Proposed fix: only update last_reward_block when rewards are distributed
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // moved here
// ... rest of reward distribution
```

### Proof of Concept
1. Attacker reads any active staker address from `stakers` storage (e.g., index 0).
2. Every block, attacker submits: `staking.update_rewards(staker_address, disable_rewards: true)`.
3. `last_reward_block` is set to the current block; the function returns early.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. No staker ever accumulates `unclaimed_rewards_own`; no pool contract ever receives a `update_rewards_from_staking_contract` call.
6. All staker and pool-member yield is permanently frozen. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
