### Title
Integer Truncation in `validate_tx_l2_gas_price_within_threshold` Collapses Threshold to Zero for Small Previous-Block Gas Prices - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price using `Ratio::to_integer()`, which performs **floor (truncating) integer division**. When `min_gas_price_percentage * previous_block_l2_gas_price < 100`, the computed threshold collapses to `0`, and the check `tx_l2_gas_price.0 < threshold` becomes `tx_l2_gas_price.0 < 0`, which is always false for a `u128`. Any transaction — including one with `max_price_per_unit = 0` — passes the stateful admission gate.

### Finding Description

The threshold is computed as:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold { … reject … }
```

`Ratio::to_integer()` returns `⌊numerator / denominator⌋`. Therefore:

```
threshold = ⌊(min_gas_price_percentage × previous_block_l2_gas_price) / 100⌋
```

Whenever `min_gas_price_percentage × previous_block_l2_gas_price < 100`, the result is `0`. Concrete examples:

| `min_gas_price_percentage` | `previous_block_l2_gas_price` | Intended threshold | Actual threshold |
|---|---|---|---|
| 99 | 1 | 1 (ceiling of 0.99) | **0** |
| 50 | 1 | 1 (ceiling of 0.50) | **0** |
| 1  | 99 | 1 (ceiling of 0.99) | **0** |
| 80 | 1 | 1 (ceiling of 0.80) | **0** |

Even for larger prices the floor is always strictly below the mathematically correct ceiling, so a transaction whose gas price is exactly `⌊percentage × price / 100⌋` passes even though it is below the intended `percentage%` of the previous block price.

The stateless validator has a separate static `min_gas_price` check (default 8 billion), but:
1. That check is skipped entirely when `validate_resource_bounds = false` in `StatelessTransactionValidatorConfig`.
2. The stateful check is the *dynamic* guard that tracks the live market price; collapsing it to 0 defeats its purpose regardless of the static floor. [1](#0-0) 

The stateless guard that could backstop this: [2](#0-1) 

The config field controlling the percentage: [3](#0-2) 

### Impact Explanation

When the threshold collapses to 0, the gateway's stateful L2 gas price admission check is completely bypassed. A transaction with `l2_gas.max_price_per_unit = 0` (or any value below the intended dynamic floor) is admitted to the mempool. This matches the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

Downstream consequences:
- Transactions with a gas price of 0 enter the mempool and are forwarded to the batcher.
- The blockifier's `check_fee_bounds` will then reject them at execution time (since the block's actual L2 gas price is non-zero), but only after they have consumed mempool and batcher resources.
- If `charge_fee = false` (e.g., during gateway validation with `skip_validate`), such a transaction can execute without paying any fee.

### Likelihood Explanation

The condition `min_gas_price_percentage × previous_block_l2_gas_price < 100` requires either:
- A very small previous-block L2 gas price (< 100 / percentage), which is possible during system bootstrap or with a low configured minimum; or
- A small `min_gas_price_percentage` (e.g., 1–10), which is a valid operator configuration.

The default production configuration uses `min_gas_price_percentage = 100`, for which `floor(100 × price / 100) = price` — no truncation. However, operators running with any percentage below 100 and a low gas price environment are exposed. The test suite itself demonstrates the zero-threshold case is reachable: [4](#0-3) 

### Recommendation

Replace `to_integer()` (floor) with ceiling division so the threshold is never lower than the intended percentage:

```rust
// Instead of:
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0)
    .to_integer();

// Use ceiling:
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

`Ratio::ceil()` returns `⌈numerator / denominator⌉`, ensuring the threshold is always at least 1 whenever `min_gas_price_percentage > 0` and `previous_block_l2_gas_price > 0`. This mirrors the pattern already used in `convert_l1_to_l2_gas_price_round_up` and `sierra_gas_to_l1_gas_amount_round_up`: [5](#0-4) 

### Proof of Concept

```rust
use num_rational::Ratio;

fn threshold_floor(percentage: u8, prev_price: u128) -> u128 {
    (Ratio::new(percentage as u128, 100_u128) * prev_price).to_integer()
}

fn threshold_ceil(percentage: u8, prev_price: u128) -> u128 {
    (Ratio::new(percentage as u128, 100_u128) * prev_price).ceil().to_integer()
}

fn main() {
    // min_gas_price_percentage=99, previous_block_l2_gas_price=1
    assert_eq!(threshold_floor(99, 1), 0);  // BUG: threshold is 0, any tx passes
    assert_eq!(threshold_ceil(99, 1),  1);  // FIX: threshold is 1

    // min_gas_price_percentage=1, previous_block_l2_gas_price=99
    assert_eq!(threshold_floor(1, 99), 0);  // BUG
    assert_eq!(threshold_ceil(1, 99),  1);  // FIX

    // With threshold=0, a tx with gas_price=0 passes:
    // tx_l2_gas_price.0 < threshold  =>  0 < 0  =>  false  =>  accepted
}
```

A transaction submitted with `AllResourceBounds { l2_gas: ResourceBounds { max_price_per_unit: GasPrice(0), .. }, .. }` against a gateway whose previous block has `l2_gas_price = 1` and `min_gas_price_percentage = 99` will pass `validate_tx_l2_gas_price_within_threshold` and proceed to mempool admission.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,

pub struct StatefulTransactionValidatorConfig {
    // If true, ensures the max L2 gas price exceeds (a configurable percentage of) the base gas
    // price of the previous block.
    pub validate_resource_bounds: bool,
    pub max_allowed_nonce_gap: u32,
    pub reject_future_declare_txs: bool,
    pub max_nonce_for_validation_skip: Nonce,
    pub versioned_constants_overrides: Option<VersionedConstantsOverrides>,
    // Minimum gas price as percentage of threshold to accept transactions.
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}

impl Default for StatefulTransactionValidatorConfig {
    fn default() -> Self {
        StatefulTransactionValidatorConfig {
            validate_resource_bounds: true,
            max_allowed_nonce_gap: 200,
            reject_future_declare_txs: true,
            max_nonce_for_validation_skip: Nonce(Felt::ONE),
            min_gas_price_percentage: 100,
            versioned_constants_overrides: None,
        }
    }
}
