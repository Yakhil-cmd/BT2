### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged caller can invoke it with `disable_rewards: true` to consume the per-block reward slot without distributing rewards, permanently denying yield to any targeted staker.

### Finding Description
The specification at `docs/spec.md` lines 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `update_rewards` in `src/staking/staking.cairo` only calls `general_prerequisites()`: [1](#0-0) 

`general_prerequisites()` only checks that the contract is not paused and the caller is not the zero address: [2](#0-1) 

There is no check that the caller is the Starkware sequencer. The function uses a single global `last_reward_block` storage slot to prevent double-calling within the same block: [3](#0-2) 

When `disable_rewards = true`, the function writes `last_reward_block` to the current block and returns early without distributing any rewards: [4](#0-3) 

Because `last_reward_block` is a **global** value (not per-staker), a single call to `update_rewards(any_staker_address, disable_rewards: true)` in block N sets `last_reward_block = N`, causing every subsequent call to `update_rewards` in block N to revert with `REWARDS_ALREADY_UPDATED`. This blocks reward distribution for **all** stakers in that block.

The spec's access control requirement is defined here: [5](#0-4) 

The interface definition confirms the function is public with no caller restriction: [6](#0-5) 

### Impact Explanation
An attacker calling `update_rewards(any_staker, disable_rewards: true)` every block:

1. Consumes the single per-block reward slot (`last_reward_block = current_block`).
2. Prevents the legitimate sequencer call from succeeding in the same block (`REWARDS_ALREADY_UPDATED`).
3. No staker's `unclaimed_rewards_own` is ever incremented.
4. No pool rewards are ever transferred to delegation pools.

This constitutes **permanent freezing of unclaimed yield** for all stakers — a High-severity impact per the allowed scope.

### Likelihood Explanation
On Starknet, the sequencer controls intra-block transaction ordering, which means the sequencer can place its own `update_rewards` call first in each block. This limits but does not eliminate the attack surface:

- In any block where the sequencer does not call `update_rewards` (missed block, sequencer downtime, or blocks where no staker is targeted), the attacker can call it with `disable_rewards: true` to consume the slot.
- The attacker needs no privileged access — only the ability to submit a standard transaction.
- The attack is cheap (one transaction per block) and can be sustained indefinitely.

### Recommendation
Add an explicit caller check in `update_rewards` to enforce the spec's "Only starkware sequencer" requirement. For example, store the authorized sequencer address in contract storage and assert `get_caller_address() == sequencer_address` at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` enforces its caller: [7](#0-6) 

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns false).
2. Attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` in block N.
3. `last_reward_block` is written to N; function returns without distributing rewards.
4. The legitimate sequencer call `update_rewards(victim_staker, disable_rewards: false)` in block N reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number > last_reward_block` is false.
5. Victim staker's `unclaimed_rewards_own` is never incremented for block N.
6. Attacker repeats step 2 in every subsequent block — victim staker permanently receives zero yield. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
