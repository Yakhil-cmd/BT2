### Title
Gateway Stateful Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` for Resource-Bound Threshold — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bound check compares a transaction's `max_price_per_unit` (L2 gas) against the **current block's** `l2_gas_price`, but the transaction will be executed in the **next** block at `next_l2_gas_price`. The block header stores both values as distinct fields. The code itself contains a `TODO` acknowledging the wrong field is being read. This creates a systematic mismatch between the admission decision and the execution decision, causing valid transactions to be rejected (when price is falling) or invalid transactions to be admitted (when price is rising).

### Finding Description

`StorageBlockHeader` stores two semantically different L2 gas price fields:

- `l2_gas_price: GasPricePerToken` — the price used **inside** the current block
- `next_l2_gas_price: GasPrice` — the EIP-1559-derived price that will be used in the **next** block [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` constructs a `BlockInfo` from the block header, populating `strk_gas_prices.l2_gas_price` from `block_header.l2_gas_price.price_in_fri` — the current block's price — and **never exposing** `next_l2_gas_price`. [2](#0-1) 

`StatefulTransactionValidator::validate_resource_bounds` then reads `strk_gas_prices.l2_gas_price` from that `BlockInfo` and uses it as the threshold reference. The inline `TODO` comment explicitly acknowledges the wrong field is being used:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [3](#0-2) 

The threshold is then computed as `(min_gas_price_percentage / 100) * l2_gas_price` (default: 100%, so `threshold == l2_gas_price`): [4](#0-3) 

Meanwhile, blockifier's `perform_pre_validation_stage` checks `resource_bounds.max_price_per_unit < actual_gas_price.get()` using the **actual block's** gas prices — which will be `next_l2_gas_price`: [5](#0-4) 

The `next_l2_gas_price` is computed by `calculate_next_l2_gas_price_for_fin` and stored in the block header's `next_l2_gas_price` field, which is the price the next block will actually execute at: [6](#0-5) 

### Impact Explanation

**When price is rising** (`next_l2_gas_price > l2_gas_price`): The gateway admits transactions with `l2_gas_price <= tx_price < next_l2_gas_price`. These transactions pass gateway admission but will be rejected by blockifier's `InsufficientResourceBounds` check at execution time. The mempool fills with transactions that cannot be included in any block until the price falls back below the user's bid.

**When price is falling** (`next_l2_gas_price < l2_gas_price`): The gateway rejects transactions with `next_l2_gas_price <= tx_price < l2_gas_price`. These transactions would succeed at execution but are incorrectly rejected at the gateway, denying service to users who priced their transactions correctly for the next block.

This matches: **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The EIP-1559 L2 gas price adjusts every block based on utilization. Any block with utilization above or below the gas target causes `next_l2_gas_price != l2_gas_price`. Under normal network load this condition is nearly always true, making the mismatch continuously active. The discrepancy per block is bounded by `1/gas_price_max_change_denominator`, but it is nonzero on every non-perfectly-utilized block.

### Recommendation

In `validate_resource_bounds`, read `next_l2_gas_price` from the block header rather than `l2_gas_price`. The `GatewayFixedBlockStateReader` trait and its `GatewayFixedBlockSyncStateClient` implementation should expose `next_l2_gas_price` from `block_header_without_hash.next_l2_gas_price` and pass it to `validate_tx_l2_gas_price_within_threshold` as the reference price. This resolves the acknowledged `TODO` at line 229.

### Proof of Concept

1. Observe block N with `l2_gas_price = 10_000` fri and `next_l2_gas_price = 10_030` fri (price rising due to high utilization).
2. Submit an invoke V3 transaction with `l2_gas.max_price_per_unit = 10_015` fri.
3. Gateway `validate_resource_bounds` computes threshold = `100% * 10_000 = 10_000`. Since `10_015 >= 10_000`, the transaction is **admitted** to the mempool.
4. When the batcher attempts to include the transaction in block N+1, blockifier's `perform_pre_validation_stage` checks `10_015 < 10_030` and returns `InsufficientResourceBounds { resource: L2Gas, max_gas_price: 10_015, actual_gas_price: 10_030 }`.
5. The transaction is rejected at execution, wasting mempool capacity and causing the user's transaction to silently fail admission to any block until the price drops. [7](#0-6) [1](#0-0) [8](#0-7)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
