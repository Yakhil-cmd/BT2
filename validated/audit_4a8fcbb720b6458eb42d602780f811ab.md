### Title
Permissionless `update_rewards` with Global `last_reward_block` Allows Any Caller to Grief All Stakers' Block Reward Distribution - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `staking.cairo` has no caller restriction and uses a **global** (not per-staker) `last_reward_block` storage variable. Any address can call `update_rewards(any_valid_staker, disable_rewards: true)` to consume the per-block reward slot without distributing rewards, permanently blocking all stakers from receiving consensus block rewards in that block. The attacker can repeat this every block.

---

### Finding Description

The `IStakingRewardsManager` interface exposes `update_rewards` as a fully public, permissionless function:

```cairo
// src/staking/interface.cairo L308-L310
fn update_rewards(
    ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
);
```

The implementation in `staking.cairo` performs a global block-number guard:

```cairo
// src/staking/staking.cairo L1453-L1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After passing the guard, it unconditionally writes the current block number to the **single global** `last_reward_block` storage slot:

```cairo
// src/staking/staking.cairo L1484-L1485
// Update last block rewards.
self.last_reward_block.write(current_block_number);
```

Then, if `disable_rewards` is `true`, it returns immediately without distributing any rewards:

```cairo
// src/staking/staking.cairo L1487-L1489
if disable_rewards || self.is_pre_consensus() {
    return;
}
```

There is **no caller check** anywhere in this function. `general_prerequisites()` only checks whether the contract is paused. Any externally-owned account can call `update_rewards(any_valid_staker_address, disable_rewards: true)` and:

1. Pass the block-number guard (consuming the slot for the current block).
2. Write `current_block_number` into the global `last_reward_block`.
3. Return without distributing any rewards to any staker.

Because `last_reward_block` is global (not keyed per staker), **all** subsequent calls to `update_rewards` for **any** staker in the same block will revert with `REWARDS_ALREADY_UPDATED`. The attacker can repeat this in every block, indefinitely preventing reward distribution.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

`update_rewards` is the sole mechanism by which consensus block rewards are credited to stakers and their pools. If an attacker calls it with `disable_rewards: true` in every block, no staker ever accumulates consensus rewards. The yield is not merely delayed; it is permanently lost for each griefed block because the per-block reward calculation is tied to the specific block at which `update_rewards` is called. Stakers and delegators lose all consensus-era block rewards for as long as the attack is sustained.

---

### Likelihood Explanation

**High.** The attack requires only a single cheap, permissionless transaction per block. The attacker needs no stake, no privileged role, and no special setup — only a valid `staker_address` to pass the `STAKER_NOT_EXISTS` / `INVALID_STAKER` checks. On Starknet, while there is no traditional public mempool, the attacker can simply submit `update_rewards(any_staker, disable_rewards: true)` as the first transaction in every block. The cost is minimal (one contract call per block) and the damage is maximal (all stakers lose all block rewards for every griefed block).

---

### Recommendation

Restrict `update_rewards` to authorized callers only. For example, require the caller to be the staker themselves, their registered reward address, or a designated keeper/operator role. A minimal fix:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let caller = get_caller_address();
    let staker_info = self.internal_staker_info(:staker_address);
    assert!(
        caller == staker_address || caller == staker_info.reward_address,
        "{}",
        Error::CALLER_CANNOT_UPDATE_REWARDS,
    );
    // ... rest of function
}
```

Alternatively, if `update_rewards` is intended to be callable by anyone (e.g., for keeper bots), the `disable_rewards` flag should be removed from the public interface and handled internally, so that no external caller can suppress reward distribution.

---

### Proof of Concept

1. Staker `S` is active and eligible for consensus block rewards in block `N`.
2. Attacker `A` (any address) calls `update_rewards(S, disable_rewards: true)` in block `N`.
   - `current_block_number (N) > last_reward_block` → guard passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → function returns early, **no rewards distributed**.
3. Staker `S` (or any keeper) calls `update_rewards(S, disable_rewards: false)` in block `N`.
   - `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
4. Attacker repeats step 2 in block `N+1`, `N+2`, … indefinitely.
5. No staker ever receives consensus block rewards.
