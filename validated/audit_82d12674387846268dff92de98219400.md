### Title
Authorized Precompile Pausers Can Temporarily Freeze User ERC-20 Token Exits Without Ability to Unfreeze - (File: engine-precompiles/src/lib.rs)

### Summary
The `pause_precompiles` function allows any account in the `EngineAuthorizer` ACL to pause the `EXIT_TO_NEAR` and `EXIT_TO_ETHEREUM` precompiles, blocking users from exiting their ERC-20 tokens and ETH from Aurora. However, `resume_precompiles` is restricted to owner-only access. This asymmetry — analogous to toggle/eligibility delegates in the original report — means authorized pausers can unilaterally freeze user fund exits with no ability to unfreeze them, leaving users dependent on the owner to intervene.

### Finding Description
In `engine-precompiles/src/lib.rs`, the `execute` function of the `Precompiles` set checks `self.is_paused(&address)` before dispatching any precompile call. If the precompile is flagged as paused, it immediately returns a fatal `ERR_PAUSED` error, aborting the EVM transaction:

```rust
if self.is_paused(&address) {
    return Some(Err(PrecompileFailure::Fatal {
        exit_status: ExitFatal::Other(prelude::Cow::Borrowed("ERR_PAUSED")),
    }));
}
```

The pause flags are set by `pause_precompiles` in `engine/src/contract_methods/admin.rs`, which checks `authorizer.is_authorized()` — any account in the `EngineAuthorizer` ACL can call this:

```rust
pub fn pause_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    ...
    let authorizer: EngineAuthorizer = engine::get_authorizer(&io);
    if !authorizer.is_authorized(&env.predecessor_account_id()) {
        return Err(b"ERR_UNAUTHORIZED".into());
    }
    ...
    pauser.pause_precompiles(flags);
```

However, `resume_precompiles` in the same file requires `require_owner_only`:

```rust
pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    ...
    require_owner_only(&state, &predecessor_account_id)?;
    ...
    pauser.resume_precompiles(flags);
```

The `EXIT_TO_NEAR` and `EXIT_TO_ETHEREUM` precompiles are the **only** EVM-level exit paths for ERC-20 tokens bridged onto Aurora. When both are paused, users holding ERC-20 tokens on Aurora have no EVM-accessible route to exit their funds. The NEAR-level `withdraw` function is gated on `require_running` (full engine pause) and only handles NEP-141 ETH, not arbitrary ERC-20 tokens. There is no alternative exit path for ERC-20 tokens.

The `EnginePrecompilesPauser` stores pause flags in contract storage under `PAUSE_FLAGS_KEY`, so the freeze persists across blocks until the owner explicitly calls `resume_precompiles`.

### Impact Explanation
Any account in the `EngineAuthorizer` ACL — a role distinct from the owner — can call `pause_precompiles` with `EXIT_TO_NEAR | EXIT_TO_ETHEREUM` flags set. This immediately and persistently blocks all EVM-level exits for ERC-20 tokens. Users with bridged ERC-20 balances on Aurora cannot exit them until the owner calls `resume_precompiles`. The freeze duration is entirely at the owner's discretion and responsiveness. This constitutes **High — Temporary freezing of funds**.

### Likelihood Explanation
The `EngineAuthorizer` ACL is a separate privilege tier from the owner. An authorized pauser acting maliciously or whose account is compromised can trigger the freeze with a single transaction. The asymmetry — pause requires ACL membership, resume requires owner — means the freeze cannot be self-corrected by the pauser and persists until owner intervention. This mirrors the original report's pattern: a delegate-tier actor (authorized pauser) can block a user action (exit), and only a higher-tier actor (owner) can restore it.

### Recommendation
Align the access control for `pause_precompiles` and `resume_precompiles` symmetrically. Either:
1. Restrict `pause_precompiles` to owner-only, matching `resume_precompiles`.
2. Allow authorized pausers to also call `resume_precompiles` for flags they themselves set.
3. Introduce a maximum pause duration enforced on-chain (e.g., auto-resume after N blocks), so no single actor can freeze exits indefinitely.

### Proof of Concept
1. `authorized_pauser.near` is a member of the `EngineAuthorizer` ACL (a role distinct from the owner).
2. `authorized_pauser.near` calls `pause_precompiles` with `paused_mask = EXIT_TO_NEAR_FLAG | EXIT_TO_ETHEREUM_FLAG`.
3. The `EnginePrecompilesPauser` writes the combined flags to storage under `PAUSE_FLAGS_KEY`.
4. A user holding bridged ERC-20 tokens on Aurora calls `submit` with an EVM transaction that invokes the `EXIT_TO_NEAR` precompile at address `0xe9217bc70b7ed1f598ddd3199e80b093fa71124f`.
5. The `Precompiles::execute` function reads the pause flags, finds `EXIT_TO_NEAR` set, and returns `ExitFatal::Other("ERR_PAUSED")`.
6. The EVM transaction reverts; no tokens are transferred; the user's ERC-20 balance remains locked on Aurora.
7. The same outcome occurs for `EXIT_TO_ETHEREUM` at `0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab`.
8. `authorized_pauser.near` cannot call `resume_precompiles` — it fails `require_owner_only`.

impl<I: IO> EnginePrecompilesPauser<I> {
    /// Key for storing [`PrecompileFlags`].
    const PAUSE_FLAGS_KEY: &'static [u8; 11] = b"PAUSE_FLAGS";

    /// Creates new [`EnginePrecompilesPauser`] instance that reads from and writes into storage accessed using `io`.
    pub const fn from_io(io: I) -> Self {
        Self { io }
    }

    fn read_flags_from_storage(&self) -> PrecompileFlags {
        self.io
            .read_storage(&Self::storage_key())
            .map_or_else(PrecompileFlags::empty, |bytes| {
                const U32_SIZE: usize = size_of::<u32>();
                assert_eq!(bytes.len(), U32_SIZE, "PrecompileFlags value is corrupted");
                let mut buffer = [0u8; U32_SIZE];
                bytes.copy_to_slice(&mut buffer);
                PrecompileFlags::from_bits_truncate(u32::from_le_bytes(buffer))
            })
    }

    fn write_flags_into_storage(&mut self, pause_flags: PrecompileFlags) {
        self.io
            .write_storage(&Self::storage_key(), &pause_flags.bits().to_le_bytes());
    }

    fn storage_key() -> Vec<u8> {
        bytes_to_key(KeyPrefix::Config, Self::PAUSE_FLAGS_KEY)
    }
}
