### Title
Gateway Stateful Validator Checks L2 Gas Price Against Stale Current-Block Price Instead of Next-Block Price — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`validate_resource_bounds` in the gateway's stateful validator gates admission on the **current (already-finalized) block's** L2 gas price, while the transaction will actually be executed in the **next block** at a price computed by the EIP-1559 / SNIP-35 mechanism. The code itself carries a `TODO` acknowledging the wrong field is used. The mismatch causes two symmetric admission errors: (a) valid transactions are rejected when the next-block price is lower than the current-block price, and (b) transactions that will fail in the blockifier are admitted when the next-block price is higher.

### Finding Description

In `validate_resource_bounds` the gateway reads the L2 gas price from the latest finalized block:

```rust

    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold applied is `min_gas_price_percentage (default 100%) × current_block_l2_gas_price`: [2](#0-1) 

The correct reference price is `next_l2_gas_price`, which is already stored in `BlockHeaderWithoutHash` and is the price the batcher will actually use when it builds the next block: [3](#0-2) 

The batcher's blockifier execution calls `check_fee_bounds`, which compares `resource_bounds.max_price_per_unit` against the **actual next-block** L2 gas price in the block context: [4](#0-3) 

The gateway's own blockifier validation in `run_validate_entry_point` also uses the current block's gas prices (only the block number is bumped), so it is consistent with the pre-validation check but both are anchored to the wrong price: [5](#0-4) 

The next-block L2 gas price is computed by `calculate_next_l2_gas_price_for_fin`, which applies EIP-1559 adjustment and the SNIP-35 `fee_actual` floor — both of which can move the price materially away from the current block's price: [6](#0-5) 

### Impact Explanation

**Case A — gas price decreasing (next < current):** A transaction with `current_block_price > max_price_per_unit ≥ next_block_price` is rejected by the gateway with `GAS_PRICE_TOO_LOW` even though it would pass `check_fee_bounds` in the batcher. Valid transactions are denied sequencing.

**Case B — gas price increasing (next > current):** A transaction with `current_block_price ≤ max_price_per_unit < next_block_price` passes the gateway check and enters the mempool, but will fail `check_fee_bounds` in the batcher and be reverted. The gateway admits transactions that are invalid for the block they will land in.

Both cases match: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price every block based on gas usage relative to `gas_target`. The SNIP-35 `fee_actual` floor can cause larger step changes. Price divergence between consecutive blocks is a normal, frequent, unprivileged condition — no special role or attack is required to trigger it.

### Recommendation

Replace the `l2_gas_price` field read with `next_l2_gas_price` from the block header. This requires either adding `next_l2_gas_price` to the `BlockInfo` struct returned by `GatewayFixedBlockStateReader::get_block_info`, or adding a dedicated accessor to the trait. The TODO comment at line 229 already identifies this exact fix.

### Proof of Concept

**Case A (valid transaction rejected):**
1. Block N finalizes with `l2_gas_price = 100 FRI`; gas usage was below target, so `next_l2_gas_price = 90 FRI`.
2. User submits a transaction with `max_price_per_unit = 95 FRI`.
3. Gateway threshold = `100% × 100 = 100`. Check: `95 < 100` → rejected with `GAS_PRICE_TOO_LOW`.
4. The transaction would have passed blockifier's `check_fee_bounds` (`95 ≥ 90`) and executed successfully.

**Case B (invalid transaction admitted):**
1. Block N finalizes with `l2_gas_price = 100 FRI`; gas usage was above target, so `next_l2_gas_price = 110 FRI`.
2. User submits a transaction with `max_price_per_unit = 105 FRI`.
3. Gateway threshold = `100`. Check: `105 ≥ 100` → admitted to mempool.
4. Batcher executes in block N+1 at price 110 FRI. Blockifier check: `105 < 110` → `MaxGasPriceTooLow`, transaction reverted.

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
