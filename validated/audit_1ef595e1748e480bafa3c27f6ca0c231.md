### Title
Missing Upper-Bound Invariant on Signing Participant Set Enables Presignature Reuse Across Non-Overlapping Quorums — (`src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

The OT-based ECDSA `sign()` function enforces only a lower-bound check (`participants.len() >= threshold`) on the signing participant set. It does not enforce the upper-bound invariant `N <= 2t - 1` documented as mandatory in the orchestration specification. When `N >= 2t`, two fully non-overlapping groups of size `t` can both be assembled from the same presigning participant set. A malicious coordinator can exploit this to run two concurrent signing sessions — each with a disjoint subset of honest participants — both consuming shares derived from the same presignature. This constitutes ECDSA nonce reuse across two signatures, enabling full private key extraction.

---

### Finding Description

The orchestration documentation explicitly states the invariant:

> "If the threshold $t_i$ is such that $N_i \leq 2t - 1$, then it's impossible to have two non-overlapping quorums… Otherwise, you might have two independent groups of parties trying to use the same output, which is bad." [1](#0-0) 

The `sign()` function in the OT-based ECDSA implementation enforces only:

```rust
// ensure number of participants during the signing phase is >= threshold
if participants.len() < threshold {
    return Err(InitializationError::NotEnoughParticipantsForThreshold { ... });
}
``` [2](#0-1) 

There is no check that `participants.len() <= 2 * threshold - 1`. This means the library accepts any signing group of size `>= threshold`, including groups large enough to permit two fully disjoint signing quorums from the same presigning output.

By contrast, the robust ECDSA variant explicitly enforces `participants.len() == 2 * max_malicious + 1` to prevent exactly this class of split-view / presignature-reuse attack:

```rust
if participants.len() != robust_ecdsa_threshold {
    return Err(InitializationError::BadParameters(
        "the number of participants during signing must be exactly 2*max_malicious+1 to avoid split view attacks".to_string(),
    ));
}
``` [3](#0-2) 

No equivalent guard exists in the OT-based ECDSA `sign()`. [4](#0-3) 

---

### Impact Explanation

A presignature encodes a secret nonce `k` and `sigma = k * x` (where `x` is the private key), distributed as threshold shares `(k_i, sigma_i)` across `N` participants. [5](#0-4) 

During signing, each participant linearizes their share using Lagrange coefficients over the signing participant set and sends a signature share `s_i = h * lambda_i * k_i + Rx * lambda_i * sigma_i` to the coordinator. [6](#0-5) 

If two non-overlapping groups `G_A = {P_1, ..., P_t}` and `G_B = {P_{t+1}, ..., P_{2t}}` each independently run `sign()` with different message hashes `h_1` and `h_2` using shares from the same presignature, the coordinator obtains:

```
s_A = h_1 * k + Rx * sigma   (mod q)
s_B = h_2 * k + Rx * sigma   (mod q)
```

Subtracting: `s_A - s_B = (h_1 - h_2) * k`, so `k = (s_A - s_B) / (h_1 - h_2)`. With `k` known, `x = sigma / k` recovers the full private key.

**Impact: Critical** — full private key extraction via ECDSA nonce reuse, enabled by a missing library-level invariant.

---

### Likelihood Explanation

The attacker is a **malicious coordinator**, an explicitly in-scope role. The coordinator controls which participant subsets are assembled for each signing session and can initiate two concurrent sessions with disjoint participant sets. Each honest participant only uses their presignature share once (so they detect no anomaly locally). The condition `N >= 2t` is a common and natural deployment configuration (e.g., `N=4, t=2`). No cryptographic break, leaked key, or external dependency failure is required.

---

### Recommendation

Add an upper-bound check in `sign()` mirroring the invariant stated in the orchestration documentation:

```rust
// Prevent two non-overlapping quorums from consuming the same presignature
if participants.len() > 2 * threshold - 1 {
    return Err(InitializationError::BadParameters(
        "participants.len() must be <= 2*threshold - 1 to prevent presignature reuse across non-overlapping quorums".to_string(),
    ));
}
```

This should be applied at `src/ecdsa/ot_based_ecdsa/sign.rs` in the `sign()` function, immediately after the existing lower-bound check at line 58. The same invariant should be enforced in `presign()` at `src/ecdsa/ot_based_ecdsa/presign.rs` line 31, since the presigning participant set `N_1` determines which signing subsets are possible. [2](#0-1) [7](#0-6)

### Proof of Concept

**Setup:** `N=4` participants `{P1, P2, P3, P4}`, threshold `t=2`. Since `N=4 >= 2t=4`, two non-overlapping groups `G_A={P1,P2}` and `G_B={P3,P4}` each of size `t=2` exist.

1. All four participants run `presign()` together, producing shares `(k_i, sigma_i)` for `i ∈ {1,2,3,4}`.
2. Malicious coordinator initiates **Session A**: calls `sign(participants=[P1,P2], ..., msg_hash=h1)`. The library accepts this because `2 >= threshold=2`. Session A completes, producing signature `(R, s_A)`.
3. Malicious coordinator initiates **Session B**: calls `sign(participants=[P3,P4], ..., msg_hash=h2)` with the **same presignature shares**. The library accepts this because `2 >= threshold=2`. Session B completes, producing signature `(R, s_B)`.
4. Both sessions share the same `R` (same presignature nonce `k`). The coordinator computes:
   - `k = (s_A - s_B) * (h1 - h2)^{-1} mod q`
   - `x = sigma * k^{-1} mod q`

Full private key `x` is recovered. The `sign()` function at `src/ecdsa/ot_based_ecdsa/sign.rs` line 58 permits both sessions because it only checks `participants.len() >= threshold`, not `participants.len() <= 2*threshold - 1`. [2](#0-1) [1](#0-0)

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
