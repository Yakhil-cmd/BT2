### Title
Missing Protocol-Level Enforcement of Signing Context Binding in Robust ECDSA Rerandomization — (`File: src/ecdsa/robust_ecdsa/sign.rs`, `src/ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

The `RerandomizedPresignOutput` struct carries no cryptographic binding to the `(msg_hash, tweak, participants, entropy)` used during rerandomization. The `sign()` function accepts a `RerandomizedPresignOutput` and a `msg_hash` as independent, unverified inputs. There is no protocol-level mechanism to ensure all participants rerandomized with the same context. A malicious coordinator can present different `(msg_hash, tweak, entropy)` values to different participants during the rerandomization step, causing them to compute signature shares over multiplicatively related nonces from the same presignature — enabling full secret key extraction via standard ECDSA nonce-reuse techniques.

---

### Finding Description

`RerandomizationArguments::derive_randomness()` in `src/ecdsa/mod.rs` computes a binding scalar `delta` via:

```
HKDF(salt=SALT, ikm=entropy, info=counter||pk||tweak||msg_hash||big_r||participants)
``` [1](#0-0) 

This `delta` is used to rerandomize the presignature nonce: `R' = delta·R`, `alpha' = alpha·delta⁻¹`, `beta' = (beta + c·tweak)·delta⁻¹`. [2](#0-1) 

However, the resulting `RerandomizedPresignOutput` struct stores only `(big_r, e, alpha, beta)` — **no record of which `(msg_hash, tweak, participants, entropy)` produced this rerandomization**: [3](#0-2) 

The `sign()` entry point then accepts `presignature: RerandomizedPresignOutput` and `msg_hash: Scalar` as completely independent parameters with no cross-verification: [4](#0-3) 

The only enforced invariants are `N = 2t+1` and `msg_hash ≠ 0`. There is no check that the `presignature` was rerandomized with the same `msg_hash`, `tweak`, `participants`, or `entropy` that is being used in the current signing session.

The security documentation explicitly acknowledges this requirement:

> **Ensure all participants agree on (h, ε) and the signing set. The coordinator must not be able to present different message hashes, tweaks, or participant lists to different signers.** [5](#0-4) 

But the code provides **no protocol mechanism to enforce this**. Each participant independently calls `rerandomize_presign()` with their own `RerandomizationArguments` before the signing session begins. The coordinator is the natural entity that distributes `(msg_hash, tweak, entropy)` to participants, and nothing prevents it from sending different values to different participants.

The `entropy` field is described as "Fresh, Unpredictable, and Public" but is not broadcast through any authenticated channel in the library: [6](#0-5) 

The `pk` field is explicitly marked "non-binding": [7](#0-6) 

---

### Impact Explanation

If a malicious coordinator sends participant A `(h₁, ε₁, entropy₁)` and participant B `(h₂, ε₂, entropy₂)` for rerandomization of the same presignature, they compute different `delta_A ≠ delta_B`. Their signature shares are then:

- `s_A = lambda_A · (alpha·delta_A⁻¹·h₁ + (beta + c·ε₁)·delta_A⁻¹·Rx + e)`
- `s_B = lambda_B · (alpha·delta_B⁻¹·h₂ + (beta + c·ε₂)·delta_B⁻¹·Rx + e)`

The resulting nonces are `R_A = delta_A·R` and `R_B = delta_B·R`, which are multiplicatively related: `R_A = (delta_A/delta_B)·R_B`. This is exactly the nonce-reuse condition documented in the security considerations: [8](#0-7) 

Standard ECDSA nonce-reuse algebra then recovers the secret key `x` from two such signatures. This is a **Critical** impact: unauthorized extraction of the aggregate private signing key.

---

### Likelihood Explanation

The coordinator is a participant in the signing protocol — a role reachable without privileged assumptions (any participant can be coordinator). The attack requires only that the coordinator distribute different `(msg_hash, tweak, entropy)` tuples to different participants before the signing round, which is entirely within the coordinator's control since the library provides no authenticated broadcast of these values. The novel split-view variant documented for this specific scheme requires as few as `2t+2` presigning participants and two signing sessions: [9](#0-8) 

Likelihood is **Medium**: the attack requires a malicious coordinator but no leaked keys, no cryptographic breaks, and no external dependencies.

---

### Recommendation

1. **Bind the signing context into `RerandomizedPresignOutput`**: Store a commitment (e.g., a hash) of `(msg_hash, tweak, participants, entropy)` inside `RerandomizedPresignOutput` at rerandomization time.

2. **Verify context consistency in `sign()`**: At the start of `sign()`, verify that the `msg_hash` and `tweak` passed match the binding stored in the `RerandomizedPresignOutput`. Reject mismatches.

3. **Broadcast `entropy` via authenticated channel**: Require the coordinator to broadcast `entropy` through the existing reliable broadcast mechanism so all participants can verify they received the same value before rerandomizing.

---

### Proof of Concept

**Step 1** — Malicious coordinator sends participant A `entropy_A` and participant B `entropy_B` (different values) along with the same presignature `big_r`.

**Step 2** — Each participant independently calls:
```rust
RerandomizedPresignOutput::rerandomize_presign(&presig, &args_X)
```
with their respective `args_A` / `args_B`, producing different `delta_A`, `delta_B` and thus different rerandomized shares. [10](#0-9) 

**Step 3** — Both participants call `sign()` with the same `msg_hash` (as instructed by the coordinator). Their `s_i` shares are computed over multiplicatively related nonces. [11](#0-10) 

**Step 4** — The coordinator collects both shares. The coordinator's final `verify()` call may pass (if the coordinator selects the right combination), yielding a valid signature. Repeating with a second presignature gives two signatures with related nonces, from which the secret key is recovered algebraically. [12](#0-11)

    pub fn derive_randomness(&self) -> Result<Scalar, ProtocolError> {
        // create a string containing (pk, msg_hash, big_r, sorted(participants))
        let pk_encoded_point = self.pk.to_encoded_point(true);
        let encoded_pk: &[u8] = pk_encoded_point.as_bytes();
        let encoded_tweak: &[u8] = &<Secp256K1ScalarField as Field>::serialize(&self.tweak.value());
        let encoded_msg_hash: &[u8] = &self.msg_hash;
        let big_r_encoded_point = self.big_r.to_encoded_point(true);
        let encoded_big_r: &[u8] = big_r_encoded_point.as_bytes();

        // concatenate all the bytes
        let mut concatenation = Vec::new();
        // 1 byte counter, used in the unlikely case that the hash result is 0
        concatenation.extend_from_slice(&[0u8, 1]);
        concatenation.extend_from_slice(encoded_pk);
        concatenation.extend_from_slice(encoded_tweak);
        concatenation.extend_from_slice(encoded_msg_hash);
        concatenation.extend_from_slice(encoded_big_r);
        // Append each ParticipantId's
        for participant in self.participants.participants() {
            concatenation.extend_from_slice(&participant.bytes());
        }

        // initiate hkdf with the salt and with some `good' entropy
        let hk = Hkdf::<sha3::Sha3_256>::new(Some(&Self::SALT), &self.entropy);

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta * R
        let rerandomized_big_r = presignature.big_r * delta;

        // alpha * delta^{-1}
        let rerandomized_alpha = presignature.alpha * inv_delta;

        // (beta + c*tweak) * delta^{-1}
        let rerandomized_beta =
            (presignature.beta + presignature.c * args.tweak.value()) * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            alpha: rerandomized_alpha,
            beta: rerandomized_beta,
            e: presignature.e,
        })
    }

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    max_malicious: impl Into<MaxMalicious>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= 2 * max_malicious + 1
    let robust_ecdsa_threshold = max_malicious
        .into()
        .value()
        .checked_mul(2)
        .and_then(|v| v.checked_add(1))
        .ok_or_else(|| {
            InitializationError::BadParameters(
                "2*threshold+1 must be less than usize::MAX".to_string(),
            )
        })?;
    if robust_ecdsa_threshold > participants.len() {
        return Err(InitializationError::BadParameters(
            "2*max_malicious+1 must be less than or equals to participant count".to_string(),
        ));
    }

    // The next two conditions prevent split-view attacks
    // documented in docs/ecdsa/robust_ecdsa/signing.md
    if participants.len() != robust_ecdsa_threshold {
        return Err(InitializationError::BadParameters(
            "the number of participants during signing must be exactly 2*max_malicious+1 to avoid split view attacks".to_string(),
        ));
    }
    if bool::from(msg_hash.is_zero()) {
        return Err(InitializationError::BadParameters(
            "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
        ));
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))

async fn do_sign_coordinator(
    mut chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    public_key: AffinePoint,

    let wait_round = chan.next_waitpoint();

    for (_, s_i) in
        recv_from_others::<SerializableScalar<C>>(&chan, wait_round, &participants, me).await?
    {
        // Sum the linearized shares
        s += s_i.0;
    }

    // raise error if s is zero
    if s.is_zero().into() {
        return Err(ProtocolError::AssertionFailed(
            "signature part s cannot be zero".to_string(),
        ));
    }
    // Normalize s
    s.conditional_assign(&(-s), s.is_high());

    let sig = Signature {
        big_r: presignature.big_r,
        s,
    };

    if !sig.verify(&public_key, &msg_hash) {
        return Err(ProtocolError::AssertionFailed(
            "signature failed to verify".to_string(),
        ));
    }

    Ok(Some(sig))
