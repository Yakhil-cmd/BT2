### Title
`MsgTurnBridge` Handler Is a No-Op: Missing Permission Check and Missing State Update — (`x/cronos/keeper/msg_server.go`)

### Summary

The `TurnBridge` message handler in the Cronos keeper is a stub that unconditionally returns `nil, nil` without performing any permission check or writing any state. This is a direct analog to the external bug: just as `migration_token_allocation` was never applied to the `Global` struct, the `enable` field of `MsgTurnBridge` is never applied to any on-chain state. The result is a permanently broken bridge circuit breaker and a complete bypass of the `CanTurnBridge` authorization check.

### Finding Description

`MsgTurnBridge` is a restricted message explicitly designed to allow authorized addresses (those with the `CanTurnBridge` permission) to enable or disable the Gravity bridge as an emergency circuit breaker. ADR-009 states:

> "For now, there are only two messages that require permission: `MsgUpdateTokenMapping` and `MsgTurnBridge`."

The `UpdateTokenMapping` handler correctly enforces this:

```go
// x/cronos/keeper/msg_server.go
func (k msgServer) UpdateTokenMapping(...) (...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, ...)
    }
    if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil { ... }
    return &types.MsgUpdateTokenMappingResponse{}, nil
}
```

The `TurnBridge` handler, however, is a complete stub:

```go
// x/cronos/keeper/msg_server.go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

Two things are missing:
1. **Permission check** — `HasPermission(ctx, ..., CanTurnBridge)` is never called. Any unprivileged address can submit `MsgTurnBridge` and receive a success response.
2. **State update** — The `msg.Enable` value is never written to the keeper store. The bridge enabled/disabled state is never changed regardless of who calls the handler.

The `CanTurnBridge` permission constant is defined and the `HasPermission` infrastructure is fully implemented, but it is simply never invoked in this handler.

### Impact Explanation

**High — Bypass of bridge authorization check + permanent inability to control bridge flows.**

- Any unprivileged user can submit `MsgTurnBridge` and receive a success (`code == 0`) response, bypassing the `CanTurnBridge` permission gate entirely.
- More critically, no caller — including the admin or a permissioned address — can ever actually disable the bridge. The `enable` field is silently discarded. The emergency circuit breaker is permanently non-functional.
- The integration test `test_gravity_turn_bridge` asserts that `cli.turn_bridge("false", from_="community")` must fail with a non-zero code, and that `cli.turn_bridge("false", from_="validator")` must succeed and actually stop bridge operations. Both assertions would fail against this implementation.

### Likelihood Explanation

The `MsgTurnBridge` transaction is a live, registered gRPC message with a working CLI command (`CmdTurnBridge`). Any user on the network can submit it. The handler is reachable by any unprivileged sender with no preconditions.

### Recommendation

The `TurnBridge` handler must be implemented to:
1. Check `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `ErrUnauthorized` if the check fails.
2. Persist the `msg.Enable` value to the keeper store (e.g., as a dedicated key in the KV store or as a field in `Params`), so that downstream bridge logic can read and enforce the enabled/disabled state.

func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}
