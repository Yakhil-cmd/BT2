### Title
Unauthenticated `LedgerService.Set` gRPC Endpoint Allows Any Network Peer to Overwrite Arbitrary Execution State - (`cmd/ledger/main.go`, `ledger/remote/service.go`)

---

### Summary

The standalone `flow-ledger-service` binary exposes a `LedgerService.Set` gRPC RPC that writes arbitrary key-value pairs directly into the execution state trie. The gRPC server is created with no transport credentials and no authentication interceptor. When the service is bound to a TCP address (the documented and supported deployment mode), any network-reachable attacker can call `Set` to overwrite any register in the execution state, enabling direct theft of FLOW tokens or corruption of any account's on-chain storage.

---

### Finding Description

**Root cause — no authentication on the gRPC server:**

`cmd/ledger/main.go` creates the gRPC server as:

```go
grpcServer := grpc.NewServer(
    grpc.MaxRecvMsgSize(int(*maxRequestSize)),
    grpc.MaxSendMsgSize(int(*maxResponseSize)),
)
```

No `grpc.Creds(...)` option, no server-side TLS, no authentication interceptor is passed. The service is then registered and served on a TCP listener:

```go
lis, err := net.Listen("tcp", *ledgerServiceTCP)
...
grpcServer.Serve(info.listener)
```

The README documents and encourages this deployment mode with `0.0.0.0:9000`.

**Root cause — `Set` handler performs no caller check:**

`ledger/remote/service.go` `Set()` validates only the shape of the request (non-nil state, non-empty keys, matching lengths) and then unconditionally calls `s.ledger.Set(update)`:

```go
func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
    // ... input shape validation only ...
    newState, trieUpdate, err := s.ledger.Set(update)
    ...
}
```

There is no check on who the caller is. The `ctx` carries no verified identity because the transport layer enforces none.

**Contrast with the node admin server:** `cmd/scaffold.go` `EnqueueAdminServerInit()` supports optional mutual TLS (`tls.RequireAndVerifyClientCert`). The ledger service has no equivalent option at all — not even an optional flag.

**Attacker-controlled entry path:**

1. Attacker discovers the ledger service TCP port (documented default `0.0.0.0:9000`).
2. Attacker connects with any gRPC client using `insecure.NewCredentials()` (the same credentials the legitimate `remote.Client` uses).
3. Attacker calls `LedgerService.Set` with a crafted `SetRequest` containing target register keys and attacker-chosen values.
4. `service.go:Set()` processes the request, calls `s.ledger.Set(update)`, and the MTrie is mutated.
5. The new state root is returned to the attacker, confirming success.

---

### Impact Explanation

The execution state trie is the authoritative source of all on-chain account balances, contract code, and storage. Writing arbitrary values to arbitrary keys allows:

- **Direct theft of FLOW tokens**: overwrite the `balance` register of any account to drain it or inflate the attacker's balance.
- **Smart contract state corruption**: overwrite any contract's storage registers to manipulate DeFi logic, bypass access controls, or destroy contract invariants.
- **Account takeover**: overwrite account public keys to replace them with attacker-controlled keys.

The impact is equivalent to or worse than the original report's "any ERC20 token left in the Diamond can be stolen" — here the entire execution state of the Flow blockchain is at risk.

**Severity: Critical.**

---

### Likelihood Explanation

The README explicitly documents and provides copy-paste examples for TCP deployment (`0.0.0.0:9000`). The `remote.Client` connects with `insecure.NewCredentials()`, confirming the protocol is designed to work without TLS. Any attacker with network access to the execution node's infrastructure (e.g., a co-located cloud tenant, a misconfigured firewall, or an internal network attacker) can reach the port and exploit this with a standard gRPC client. No special privileges, no leaked keys, and no staked node role are required.

---

### Recommendation

Add mandatory mutual TLS to the ledger service gRPC server, mirroring the pattern already used by the node admin server in `cmd/scaffold.go`:

```go
// Require TLS credentials on the gRPC server
grpcServer := grpc.NewServer(
    grpc.Creds(credentials.NewTLS(tlsConfig)), // require client cert
    grpc.MaxRecvMsgSize(int(*maxRequestSize)),
    grpc.MaxSendMsgSize(int(*maxResponseSize)),
)
```

Add corresponding CLI flags (`-tls-cert`, `-tls-key`, `-client-ca`) and make TLS mandatory (not optional) when the service is bound to a TCP address. Unix socket deployments may rely on filesystem permissions, but TCP deployments must enforce transport-level authentication.

conn, _ := grpc.NewClient(

```go
// Set updates keys with new values at a specific state and returns the new state
func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
	if req.State == nil || len(req.State.Hash) != len(ledger.State{}) {
		return nil, status.Error(codes.InvalidArgument, "invalid state")
	}

	if len(req.Keys) == 0 {
		return nil, status.Error(codes.InvalidArgument, "keys cannot be empty")
	}

	if len(req.Keys) != len(req.Values) {
		return nil, status.Error(codes.InvalidArgument, "keys and values length mismatch")
	}

	var state ledger.State
	copy(state[:], req.State.Hash)

	keys := make([]ledger.Key, len(req.Keys))
	for i, protoKey := range req.Keys {
		key, err := protoKeyToLedgerKey(protoKey)
		if err != nil {
			return nil, err // protoKeyToLedgerKey already returns status.Error
		}
		keys[i] = key
	}

	values := make([]ledger.Value, len(req.Values))
	for i, protoValue := range req.Values {
		var value ledger.Value
		// Reconstruct the original value type using is_nil flag
		// This preserves the distinction between nil and []byte{} that protobuf loses
		if len(protoValue.Data) == 0 {
			if protoValue.IsNil {
				// Original value was nil
				value = nil
			} else {
				// Original value was []byte{} (empty slice)
				value = ledger.Value([]byte{})
			}
		} else {
			// Non-empty value, use data as-is
			value = ledger.Value(protoValue.Data)
		}
		values[i] = value
	}

	update, err := ledger.NewUpdate(state, keys, values)
	if err != nil {
		return nil, status.Error(codes.InvalidArgument, err.Error())
	}

	newState, trieUpdate, err := s.ledger.Set(update)
	if err != nil {
		return nil, status.Error(codes.Internal, err.Error())
	}

	// Encode trie update using centralized encoding function to ensure
	// client and server use the same encoding method
	trieUpdateBytes := encodeTrieUpdateForTransport(trieUpdate)

	return &ledgerpb.SetResponse{
		NewState: &ledgerpb.State{
			Hash: newState[:],
		},
		TrieUpdate: trieUpdateBytes,
	}, nil
}
```
