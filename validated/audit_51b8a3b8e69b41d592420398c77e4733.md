### Title
Unprivileged Caller Can Permanently Freeze Per-Block Staker Rewards via `update_rewards` with `disable_rewards: true` — (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` unconditionally writes `last_reward_block` to the current block number **before** checking the `disable_rewards` flag. Because the function has no access control beyond "contract not paused" and "caller not zero", any unprivileged caller can invoke it with `disable_rewards: true`, consuming the block's reward slot without distributing any rewards. Subsequent legitimate calls for the same block revert with `REWARDS_ALREADY_UPDATED`, permanently destroying that block's rewards for all stakers.

---

### Finding Description

In `StakingRewardsManagerImpl::update_rewards`: [1](#0-0) 

The only gate is `general_prerequisites()`: [2](#0-1) 

which only asserts the contract is not paused and the caller is non-zero — no role check, no whitelist.

The critical ordering flaw: [3](#0-2) 

`last_reward_block` is written at line 1485 **before** the early-return guard at line 1487. When `disable_rewards = true` the function returns immediately after the write, distributing nothing. The per-block uniqueness check: [4](#0-3) 

then prevents any further call for that block number, making the reward loss permanent.

`last_reward_block` is a single contract-wide variable: [5](#0-4) 

so one attacker call per block silences rewards for **every** staker in that block.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block prevents the entire protocol from distributing consensus-phase block rewards for that block. Repeated across consecutive blocks this amounts to a complete, permanent denial of staker and delegator yield with no recovery path (the block number can never be revisited).

---

### Likelihood Explanation

**High.** The function is fully public, costs only gas, requires no privileged role, no token balance, and no prior setup. A single EOA can execute the attack on every block indefinitely.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so the block slot is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only mark the block as processed when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... rest of reward calculation
```

Alternatively, add an access-control assertion (e.g., `only_reward_distributor`) before the function body so that `disable_rewards: true` can only be supplied by a trusted caller such as the consensus contract.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. At block `N`, attacker calls `update_rewards(valid_staker, disable_rewards: true)`.
3. All validation passes (staker exists, is active, has non-zero balance at current epoch).
4. `self.last_reward_block.write(N)` executes — state is mutated.
5. `if disable_rewards { return; }` fires — no rewards are calculated or transferred.
6. Any legitimate call to `update_rewards` at block `N` now reverts: `current_block_number > self.last_reward_block.read()` is `false`.
7. All stakers permanently lose their block-`N` rewards.
8. Attacker repeats at block `N+1`, `N+2`, … to drain all future yield. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
