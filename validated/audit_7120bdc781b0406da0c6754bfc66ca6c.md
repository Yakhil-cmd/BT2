### Title
Unprivileged Caller Can Permanently Freeze Block Rewards for All Stakers via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the consensus rewards phase is callable by any unprivileged address. Because it unconditionally writes `last_reward_block` before checking the `disable_rewards` flag, a malicious caller can permanently prevent all stakers from receiving rewards for any given block by front-running the legitimate call with `disable_rewards: true`.

### Finding Description
`update_rewards` is a public ABI function gated only by `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

The function writes `last_reward_block` to the current block number **before** branching on `disable_rewards`: [2](#0-1) 

`last_reward_block` is a single contract-wide value: [3](#0-2) 

The guard at the top of the function enforces that only one call per block is accepted: [4](#0-3) 

Consequently, if an attacker calls `update_rewards(any_staker_address, disable_rewards: true)` for block N:
1. `last_reward_block` is set to N.
2. The function returns immediately without distributing any rewards.
3. Every subsequent call to `update_rewards` for block N — for **any** staker — reverts with `REWARDS_ALREADY_UPDATED`.
4. Block N's rewards are permanently lost; they are never accumulated anywhere else.

The contrast with `update_rewards_from_attestation_contract` (the pre-consensus path) is instructive: that function enforces `assert_caller_is_attestation_contract()`: [5](#0-4) 

No equivalent guard exists on `update_rewards`.

### Impact Explanation
In consensus rewards mode (`is_pre_consensus()` returns false), each block produces a fixed STRK and BTC reward amount calculated by `calculate_block_rewards`. Once `last_reward_block` is advanced for a block without distributing rewards, those block rewards are permanently unclaimable — they are never credited to `staker_info.unclaimed_rewards_own` and never forwarded to delegation pools. This constitutes **permanent freezing of unclaimed yield** for all active stakers, matching the High-severity impact tier. [6](#0-5) 

### Likelihood Explanation
The attack requires no special role, no capital, and no protocol knowledge beyond the public ABI. The attacker pays only gas. By calling `update_rewards(any_valid_staker, disable_rewards: true)` once per block, the attacker can continuously suppress all consensus rewards indefinitely. The attack is trivially scriptable and economically rational for any party that benefits from weakening the staking protocol (e.g., a competing L2 or a short-seller of STRK).

### Recommendation
Add an access-control check to `update_rewards` so that only an authorized contract (e.g., the consensus/block-proposer contract, analogous to how `update_rewards_from_attestation_contract` restricts to the attestation contract) may call it. Alternatively, if `disable_rewards` is intended to be set by an external caller, gate the `disable_rewards: true` path behind a privileged role check, and only advance `last_reward_block` when rewards are actually distributed or when the caller is authorized.

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus()` returns false).
2. At block N, before any legitimate `update_rewards` call, the attacker submits:
   ```
   staking_contract.update_rewards(
       staker_address: <any_active_staker>,
       disable_rewards: true
   )
   ```
3. The function passes `general_prerequisites()` (contract unpaused, caller non-zero).
4. `last_reward_block` is written to N.
5. The `disable_rewards` branch fires; the function returns with no rewards distributed.
6. Any legitimate call for block N now reverts: `current_block_number > last_reward_block` is false.
7. Block N's STRK and BTC block rewards are permanently lost for all stakers.
8. Repeating this every block completely freezes all consensus-phase yield. [7](#0-6) [8](#0-7)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
