### Title
Unprivileged Caller Can Permanently Freeze All Staker Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control. It accepts a `disable_rewards` boolean parameter and unconditionally writes the current block number to the global `last_reward_block` storage slot **before** checking `disable_rewards`. Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` every block, consuming the single per-block reward slot without distributing any rewards, permanently freezing consensus yield for all stakers.

---

### Finding Description

`update_rewards` is exposed via `#[abi(embed_v0)]` under `StakingRewardsManagerImpl` with no caller restriction beyond `general_prerequisites()` (which only checks the contract is unpaused and the caller is non-zero). [1](#0-0) 

The function enforces a global, single-slot per-block guard: [2](#0-1) 

It then **unconditionally** writes `current_block_number` to `last_reward_block` before the `disable_rewards` branch: [3](#0-2) 

If `disable_rewards` is `true`, the function returns immediately without distributing any rewards: [4](#0-3) 

Because `last_reward_block` is a single global value shared across all stakers, any subsequent call to `update_rewards` in the same block — by the legitimate staker or anyone else — will revert with `REWARDS_ALREADY_UPDATED`. The attacker only needs to supply any currently-active staker address with non-zero balance to pass the validity checks: [5](#0-4) 

---

### Impact Explanation

An attacker who calls `update_rewards(valid_staker, disable_rewards: true)` on every block permanently prevents all stakers from accumulating consensus rewards. No staker can ever satisfy `current_block_number > last_reward_block` because the attacker always consumes the slot first. This constitutes **permanent freezing of unclaimed yield** for every staker and delegation pool member in the protocol — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

The entry path requires only a non-zero caller address and a valid (active, non-zero-balance) staker address, both of which are trivially obtainable from on-chain state (e.g., `get_stakers`). On Starknet, transaction fees are low relative to Ethereum, making sustained per-block griefing economically feasible for a motivated adversary (e.g., a competing validator set). No privileged access, leaked key, or external dependency is required.

---

### Recommendation

Restrict who may call `update_rewards` with `disable_rewards: true`. The simplest fix is to gate the function on a trusted role (e.g., `only_operator` or the attestation contract), or to split the function into two: a public `update_rewards` that always distributes rewards, and a privileged `skip_rewards` callable only by the consensus/attestation layer. At minimum, move the `last_reward_block.write` **after** the `disable_rewards` guard so that a no-op call does not consume the per-block slot.

---

### Proof of Concept

1. Attacker reads any active staker address `S` with non-zero STRK balance from `get_stakers()`.
2. At every new block `N`, attacker submits: `update_rewards(S, disable_rewards: true)`.
3. Inside the call: `current_block_number (N) > last_reward_block` passes → `last_reward_block` is written to `N` → function returns early, zero rewards distributed.
4. Any legitimate call to `update_rewards` at block `N` (by the staker, a delegator, or the consensus layer) reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block: all stakers accumulate zero consensus rewards indefinitely. Pool members calling `claim_rewards` receive nothing beyond what was already accrued before the attack began. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
