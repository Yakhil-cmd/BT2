### Title
Missing Empty `AccountId` Validation in `set_owner()` Allows Permanent Loss of Owner Role - (File: `engine/src/contract_methods/admin.rs`)

### Summary
`set_owner()` accepts a Borsh-encoded `AccountId` as the new owner without rejecting an empty string. The `AccountId` Borsh deserializer explicitly allows empty strings for backward compatibility, silently returning `AccountId::default()`. If the owner accidentally passes an empty string, `owner_id` is permanently set to an uncontrollable value, locking out every `require_owner_only`-gated function forever — including `upgrade()`, `pause_contract()`, `resume_contract()`, and `set_key_manager()`.

### Finding Description

`set_owner()` reads the new owner via `io.read_input_borsh()`: [1](#0-0) 

The deserialized type is `SetOwnerArgs { new_owner: AccountId }`: [2](#0-1) 

The `AccountId` Borsh deserializer contains an explicit backward-compatibility bypass: if the raw string is empty, it returns `Self::default()` (an empty `AccountId`) **without error**: [3](#0-2) 

`AccountId::default()` produces an empty string. The only guard in `set_owner()` against a bad new owner is the same-owner check: [4](#0-3) 

There is no check that `args.new_owner` is non-empty or otherwise valid before writing it to state. After the write, `owner_id` in `EngineState` is the empty default: [5](#0-4) 

`require_owner_only` then compares `&state.owner_id` (empty string) against `predecessor_account_id`. Because NEAR enforces a minimum account-ID length of 2 characters, no real account can ever match the empty string: [6](#0-5) [7](#0-6) 

Every function gated by `require_owner_only` becomes permanently uncallable:

- `set_owner` — owner can never be corrected
- `upgrade` / `stage_upgrade` — contract is permanently unupgradeable
- `pause_contract` / `resume_contract` — pause state is permanently frozen
- `set_upgrade_delay_blocks`, `set_key_manager`, `attach_full_access_key`, `factory_update`, `factory_set_wnear_address`, `mirror_erc20_token`, `set_eth_connector_contract_account` [8](#0-7) 

### Impact Explanation

**Permanent freezing of funds (Critical path):** If the contract is in a paused state (`is_paused = true`) at the moment `owner_id` is zeroed, `resume_contract()` — which requires `require_owner_only` — can never be called. All ETH and ERC-20 balances held by the Aurora Engine contract are permanently frozen with no recovery path.

**Permanent administrative lockout (High path):** Even when the contract is running, the permanent loss of `upgrade()` means no future critical bug can be patched. Any subsequently discovered vulnerability that requires a contract upgrade to fix becomes permanently exploitable, leading to potential fund loss.

### Likelihood Explanation

The owner must call `set_owner` with a Borsh-encoded empty string as `new_owner`. This is an accidental-input scenario: the backward-compatibility bypass in `AccountId::BorshDeserialize` silently accepts the empty string rather than returning a deserialization error, so the caller receives no warning. The likelihood is low but non-zero — the same class of mistake (accidentally passing a zero/empty privileged address) is the exact scenario the original report describes.

### Recommendation

Add an explicit non-empty check inside `set_owner()` before writing the new owner to state:

```rust
let args: SetOwnerArgs = io.read_input_borsh()?;
if args.new_owner.as_bytes().is_empty() {
    return Err(b"ERR_INVALID_OWNER".into());
}
if state.owner_id == args.new_owner {
    return Err(errors::ERR_SAME_OWNER.into());
}
```

Additionally, consider removing or restricting the empty-string bypass in `AccountId::BorshDeserialize` so that it returns an error instead of silently producing a default value, eliminating this class of issue at the type level. [9](#0-8) 

### Proof of Concept

1. Owner account calls `set_owner` with Borsh-encoded `SetOwnerArgs { new_owner: "" }` (a Borsh string of length 0).
2. `AccountId::deserialize_reader` sees `account.is_empty()` → returns `AccountId::default()` (empty string) with no error.
3. `set_owner` passes the same-owner check (current owner is non-empty), writes `owner_id = ""` to `EngineState`, and returns `Ok(())`.
4. Any subsequent call to `set_owner`, `upgrade`, `pause_contract`, `resume_contract`, or any other `require_owner_only`-gated function fails with `ERR_NOT_ALLOWED` because no real NEAR account ID can equal the empty string.
5. If the contract was paused before step 1, all user funds are permanently frozen with no recovery path.

#[named]
pub fn set_owner<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;

        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;

        let args: SetOwnerArgs = io.read_input_borsh()?;
        if state.owner_id == args.new_owner {
            return Err(errors::ERR_SAME_OWNER.into());
        }

        state.owner_id = args.new_owner;
        state::set_state(&mut io, &state)?;

        Ok(())
    })
}

    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas

        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility

    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![

            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

impl BorshDeserialize for AccountId {
    fn deserialize_reader<R: io::Read>(reader: &mut R) -> io::Result<Self> {
        let account: String = BorshDeserialize::deserialize_reader(reader)?;

        // It's for saving backward compatibility.
        if account.is_empty() {
            return Ok(Self::default());
        }

        Self::try_from(account)
            .map_err(|e| io::Error::new(io::ErrorKind::InvalidData, e.to_string()))
    }
}

pub fn require_owner_only(
    state: &state::EngineState,
