### Title
Unspent `msg.value` permanently locked in `RewardSupplier` when `tick()` is called with no pending L2 mint requests — (`L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

`tick()` is declared `external payable` but only forwards `msg.value` inside the `if (amountToMint > 0)` branch. When no L2 mint-request messages are pending, the branch is skipped entirely and any ETH sent by the caller is silently retained by the contract with no refund path and no withdrawal mechanism.

---

### Finding Description

`tick()` is an unrestricted, payable function: [1](#0-0) 

`msg.value` is consumed only inside the `if (amountToMint > 0)` block at lines 125–141: [2](#0-1) 

When `requiredMinting()` returns `(0, 0)` — i.e., `messagingContract().l2ToL1Messages(msgHash) == 0` — the entire block is bypassed: [3](#0-2) 

The contract inherits from `RewardSupplierStorage` and `ProxySupportImpl`. Neither defines a `receive()`, `fallback()`, or any ETH-recovery function: [4](#0-3) [5](#0-4) 

A grep across all L1 Solidity files confirms zero occurrences of `receive()`, `fallback()`, or `withdraw` anywhere in the L1 codebase.

---

### Impact Explanation

Any ETH sent to `tick()` when no mint-request messages are pending is permanently trapped in the contract. There is no code path — in any inherited contract — that can recover it. The loss is irreversible.

Impact: **High — Permanent freezing of funds (caller ETH).**

---

### Likelihood Explanation

- `tick()` carries no access control; any EOA or contract can call it.
- The spec diagram explicitly labels the caller `anyAccount`: [6](#0-5) 

- A caller who races with another caller that already consumed the last pending message, or who simply misjudges the queue state, will lose their ETH with no warning.
- The window is always open: between epochs there are routinely zero pending messages.

---

### Recommendation

Add an explicit refund of unused ETH at the end of `tick()`, or revert when `msg.value > 0` and `amountToMint == 0`:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ... existing logic ...
    } else {
        require(msg.value == 0, "NO_MINT_REQUIRED_NO_FEE_NEEDED");
    }
}
```

Alternatively, refund any leftover ETH after the branch executes (to handle rounding from `msg.value / 2`).

---

```text
    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
```
