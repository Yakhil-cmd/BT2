### Title
`protocolUnpausePool` Always Sets Pause Level to 1 (Admin-Paused) Instead of 0 (Active), Leaving Pool Swap-Blocked After Protocol Unpause — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

The `protocolUnpausePool` function in `MetricOmmPoolFactory` unconditionally sets the pool's pause level to `1` (admin-paused) instead of `0` (active). When the protocol pauses an active pool (`0 → 2`) and then calls `protocolUnpausePool` (`2 → 1`), the pool remains in admin-paused state with all swaps blocked. The pool admin must separately call `unpausePool` to restore swap functionality — and if the admin is unresponsive, the pool is permanently swap-blocked.

---

### Finding Description

The pool's pause semantics are:

- `0` = active
- `1` = paused by admin
- `2` = paused by protocol

`protocolPausePool` accepts pools at either level `0` or `1` as valid pre-conditions: [1](#0-0) 

```solidity
function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}
```

`protocolUnpausePool`, however, always restores to level `1` regardless of what the pre-pause state was: [2](#0-1) 

```solidity
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1);   // ← always admin-paused, never active
}
```

The `swap` function enforces `whenNotPaused`, which reverts on **any** non-zero pause level: [3](#0-2) 

```solidity
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
``` [4](#0-3) 

So after `protocolUnpausePool` executes, `pauseLevel == 1`, and every call to `swap` still reverts with `PoolPaused`. The pool admin's `unpausePool` is the only path back to level `0`: [5](#0-4) 

```solidity
function unpausePool(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 1) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
}
```

The protocol has no mechanism to call `unpausePool` — it is gated to `onlyPoolAdmin`. The protocol owner cannot complete the unpause unilaterally.

---

### Impact Explanation

Every time the protocol pauses an **active** pool (level `0`) and subsequently calls `protocolUnpausePool`, the pool is left in admin-paused state (level `1`). All swaps remain blocked. `addLiquidity` and `removeLiquidity` are not gated by `whenNotPaused`, so LP positions are accessible, but the core swap functionality — the primary revenue and utility path — is completely unusable until the pool admin intervenes. If the admin key is lost, rotated, or the admin is unresponsive, the pool is permanently swap-blocked with no protocol-level recovery path.

---

### Likelihood Explanation

The trigger is the ordinary emergency-pause-then-unpause lifecycle: protocol pauses an active pool for an emergency, resolves it, and calls `protocolUnpausePool`. This is the expected operational use of the function. Any pool that was at level `0` when the protocol paused it will be left at level `1` after the protocol unpauses it. No special attacker setup is required — the protocol owner acting in good faith is sufficient to trigger the bug.

---

### Recommendation

`protocolUnpausePool` should restore the pool to fully active state (`0`), not to admin-paused state (`1`). The simplest correct fix:

```solidity
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
}
```

If the design intent is to preserve the admin-paused state when the pool was at level `1` before the protocol pause, the factory must record the pre-pause level in `protocolPausePool` and restore it in `protocolUnpausePool`.

---

### Proof of Concept

1. Pool is active: `pauseLevel = 0`.
2. Protocol owner calls `protocolPausePool(pool)` → `pauseLevel = 2`.
3. Emergency resolved; protocol owner calls `protocolUnpausePool(pool)` → `pauseLevel = 1`.
4. Any call to `pool.swap(...)` reverts with `PoolPaused` because `pauseLevel != 0`.
5. Protocol owner cannot call `unpausePool` (gated to `onlyPoolAdmin`).
6. Pool admin must call `unpausePool(pool)` → `pauseLevel = 0` to restore swaps.
7. If admin is unresponsive, the pool is permanently swap-blocked with no protocol-level recovery.
