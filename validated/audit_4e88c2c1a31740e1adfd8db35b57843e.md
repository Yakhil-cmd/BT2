### Title
Native Resource Unconditionally Charged for No-Op SSTORE Writes, Overstating `native_used` and `gas_used` - (File: `basic_system/src/system_implementation/system/mod.rs`)

---

### Summary

`EthereumLikeStorageAccessCostModel::charge_storage_write_extra` charges native (proving) resources unconditionally for every SSTORE, including no-op writes where `new_value == current_value`. The EVM ergs branch correctly returns 0 for no-op writes (matching the EVM spec), but the native branch has no such guard. This creates a specification mismatch that inflates `native_used` and `computational_native_used` per transaction, and—via the `deltaGas` adjustment—inflates the `gas_used` reported and charged to the user.

---

### Finding Description

In `EthereumLikeStorageAccessCostModel::charge_storage_write_extra`, the EVM ergs cost is correctly conditioned on whether the write is a no-op:

```rust
let total_cost = if new_value == current_value {
    0   // ← no extra ergs for a no-op write
} else if current_value == initial_value {
    if initial_value.is_zero() { SSTORE_SET_EXTRA } else { SSTORE_RESET_EXTRA }
} else { 0 };
``` [1](#0-0) 

However, the native cost is computed and charged unconditionally, with no check for `new_value == current_value`:

```rust
let native = if is_new_slot {
    R::Native::from_computational(COLD_NEW_STORAGE_WRITE_EXTRA_NATIVE_COST)
} else {
    R::Native::from_computational(COLD_EXISTING_STORAGE_WRITE_EXTRA_NATIVE_COST)
};
resources.charge(&R::from_ergs_and_native(ergs, native))
``` [2](#0-1) 

`COLD_EXISTING_STORAGE_WRITE_EXTRA_NATIVE_COST` is approximately 40,000 native units: [3](#0-2) 

This function is invoked from `apply_write_impl` for every storage write, including no-op SSTOREs: [4](#0-3) 

The result is that `native_used` is inflated by ~40,000 native units per no-op SSTORE, even though no state change occurs and no pubdata is generated.

---

### Impact Explanation

The `deltaGas` mechanism in `compute_gas_refund` converts excess native consumption into additional EVM gas charged to the user:

```rust
let delta_gas = (native_used / native_per_gas) as i64 - (gas_used as i64);
if delta_gas > 0 {
    gas_used += delta_gas as u64;
}
``` [5](#0-4) 

Because `native_used` is inflated by no-op SSTOREs, `delta_gas` is inflated, and `gas_used` reported to the block accumulator is overstated. Concretely, for a transaction with `nativePerGas = 10`, each no-op SSTORE adds ~4,000 phantom gas units to `gas_used`. A contract executing 100 no-op SSTOREs inflates `gas_used` by ~400,000 gas beyond what the EVM gas schedule specifies.

This overstated `gas_used` feeds directly into `block_computational_native_used`: [6](#0-5) 

Consequences:
1. **User overpayment**: Users are charged more gas than EVM gas estimation predicts, causing unexpected transaction failures when gas limits are set based on standard `eth_estimateGas`.
2. **Block native inflation**: `block_computational_native_used` is inflated, causing the block to hit its native limit sooner and reducing block capacity for other transactions.
3. **Divergence between forward and proving modes**: The native overcharge is real in both modes, but the `deltaGas` adjustment means the forward-mode gas receipt diverges from what a pure EVM would report.

---

### Likelihood Explanation

Any unprivileged EVM transaction can trigger this by executing `SSTORE(slot, current_value)` — writing the same value already stored in a slot. This is a standard EVM operation reachable by any contract caller. No special role, governance access, or oracle manipulation is required. The attacker controls the calldata and the contract bytecode, making this fully externally reachable.

---

### Recommendation

Condition the native write-extra charge on whether the write is actually a no-op, mirroring the existing ergs logic:

```rust
let native = if new_value == current_value {
    // No-op write: no extra native beyond the warm-read already charged
    R::Native::from_computational(0)
} else if is_new_slot {
    R::Native::from_computational(COLD_NEW_STORAGE_WRITE_EXTRA_NATIVE_COST)
} else {
    R::Native::from_computational(COLD_EXISTING_STORAGE_WRITE_EXTRA_NATIVE_COST)
};
```

This aligns the native resource model with the EVM ergs model and eliminates the specification mismatch.

---

### Proof of Concept

1. Deploy an EVM contract containing bytecode that executes `SSTORE(0, 1)` ten times after an initial `SSTORE(0, 1)` that sets the slot (so all subsequent writes are no-ops).
2. Submit a transaction calling this contract with a realistic `gasPrice` and `nativePrice` (e.g., `nativePerGas = 10`).
3. Observe `native_used` in the transaction result: it is inflated by `10 × COLD_EXISTING_STORAGE_WRITE_EXTRA_NATIVE_COST ≈ 400,000` native units beyond what the read-only path would charge.
4. Observe `gas_used`: via `deltaGas`, it is inflated by `400,000 / 10 = 40,000` gas units beyond what `eth_estimateGas` (which uses EVM gas only) would predict.
5. Set `gasLimit` to the `eth_estimateGas` value and resubmit: the transaction reverts with out-of-gas despite the estimate appearing sufficient, demonstrating the divergence. [7](#0-6) [8](#0-7)

    fn charge_storage_write_extra(
        &self,
        ee_type: ExecutionEnvironmentType,
        initial_value: &Bytes32,
        current_value: &Bytes32,
        new_value: &Bytes32,
        resources: &mut R,
        is_warm_write: bool,
        is_new_slot: bool,
    ) -> Result<(), SystemError> {
        let ergs = match ee_type {
            ExecutionEnvironmentType::NoEE => Ergs::empty(),
            ExecutionEnvironmentType::EVM => {
                let total_cost = if new_value == current_value {
                    0
                } else if current_value == initial_value {
                    if initial_value.is_zero() {
                        // we do not purge slots, so we use another indicator here
                        SSTORE_SET_EXTRA
                    } else {
                        SSTORE_RESET_EXTRA
                    }
                } else {
                    0
                };

                let total_cost =
                    // In EVM spec there's a discrepancy for cold read and cold write costs. Cold
                    // writes add another 100 from thin air.
                    if is_warm_write == false { total_cost + 100 }
                    else { total_cost };

                Ergs(total_cost * ERGS_PER_GAS)
            }
        };
        let native = if is_new_slot {
            R::Native::from_computational(
                crate::system_implementation::flat_storage_model::cost_constants::COLD_NEW_STORAGE_WRITE_EXTRA_NATIVE_COST,
            )
        } else {
            R::Native::from_computational(
          crate::system_implementation::flat_storage_model::cost_constants::COLD_EXISTING_STORAGE_WRITE_EXTRA_NATIVE_COST,)
        };
        resources.charge(&R::from_ergs_and_native(ergs, native))
    }
