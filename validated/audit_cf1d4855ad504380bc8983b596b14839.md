### Title
EVM Bridge Escrow Mis-Accounting via Deposit to NativeTokenBridgeAddress - (File: `fvm/evm/emulator/emulator.go`)

### Summary
Any unprivileged user can construct an `EVM.EVMAddress` whose bytes equal the `NativeTokenBridgeAddress` (a deterministic precompile address) and call `EVMAddress.deposit(from: vault)`. Because `mintTo` in the EVM emulator adds balance to the bridge account before executing the EVM transfer, and the EVM transfer becomes a self-transfer (bridge → bridge, a no-op), the bridge account balance is permanently inflated by the deposited amount. The depositor's Cadence `FlowToken` vault is consumed (kept alive in EVM contract storage), but no user EVM account ever receives the tokens. The result is a permanent cross-VM asset loss for the depositor and an inflated EVM total supply.

### Finding Description

**Root cause — `fvm/evm/emulator/emulator.go`, `mintTo`:**

```go
func (proc *procedure) mintTo(call *types.DirectCall) (*types.Result, error) {
    // ...
    bridge := call.From.ToCommon()          // always NativeTokenBridgeAddress
    if !proc.state.Exist(bridge) {
        proc.state.CreateAccount(bridge)
    }
    // (1) unconditionally inflate bridge balance
    proc.state.AddBalance(bridge, value, gethTracing.BalanceIncreaseWithdrawal)
    // (2) EVM transfer FROM bridge TO call.To
    res, err := proc.run(call.Message(), call.Hash(), types.DirectCallTxType)
    // ...
    res.StateChangeCommitment, err = proc.commit(true)
    return res, err
}
```

`NewDepositCall` sets `From = bridge` and `To = recipient`. When `recipient == bridge`, step (2) is a self-transfer (EVM no-op). Step (1) has already added `value` to the bridge balance, so the bridge balance is permanently inflated by `value` with no corresponding user account receiving the tokens. [1](#0-0) 

**No guard in the Cadence entry point — `fvm/evm/stdlib/contract.cdc`, `EVMAddress.deposit`:**

```cadence
access(all)
fun deposit(from: @FlowToken.Vault) {
    pre {
        !EVM.isPaused(): "EVM operations are temporarily paused"
    }
    let amount = from.balance
    if amount == 0.0 { destroy from; return }
    let depositedUUID = from.uuid
    InternalEVM.deposit(from: <-from, to: self.bytes)   // self.bytes can be bridge address
    emit FLOWTokensDeposited(...)
}
```

There is no check that `self.bytes` is not the `NativeTokenBridgeAddress`. Any user can construct `EVM.EVMAddress(bytes: bridgeAddressBytes)` and call `deposit`. [2](#0-1) 

**No guard in the Go host function — `fvm/evm/impl/impl.go`, `newInternalEVMTypeDepositFunction`:**

```go
toAddress, err := AddressBytesArrayValueToEVMAddress(context, toAddressValue)
// ...
const isAuthorized = false
account := handler.AccountByAddress(toAddress, isAuthorized)
account.Deposit(types.NewFlowTokenVault(amount))
```

`toAddress` is fully attacker-controlled; no validation against the bridge address is performed. [3](#0-2) 

**Bridge address is deterministic and public — `fvm/evm/handler/addressAllocator.go`:**

```go
func (aa *AddressAllocator) NativeTokenBridgeAddress() types.Address {
    return MakePrecompileAddress(0)
}
```

The address is derived from a known prefix and index 0, so any user can compute it off-chain. [4](#0-3) 

### Impact Explanation

When a user deposits to the `NativeTokenBridgeAddress`:

1. The Cadence `FlowToken` vault is consumed (kept alive in EVM contract storage) — the Cadence-side accounting records the tokens as locked in EVM.
2. `AddBalance(bridge, value)` inflates the bridge precompile's EVM balance.
3. The EVM self-transfer (bridge → bridge) is a no-op; no user EVM account receives the tokens.
4. The bridge precompile is not a COA and cannot be the subject of a `Withdraw` call, so the balance is permanently inaccessible.

Result: the depositor permanently loses their Cadence tokens (cross-VM asset loss), and the EVM total supply is inflated relative to what users can actually access (bridge escrow mis-accounting). The invariant that `EVM total supply == sum of all user-accessible EVM balances` is broken.

### Likelihood Explanation

The `NativeTokenBridgeAddress` is deterministic and computable by anyone. The `EVMAddress.deposit` function is `access(all)` — callable by any unprivileged transaction sender. No special role, key, or node compromise is required. A user could trigger this accidentally (e.g., by copy-pasting the wrong address) or intentionally to grief the protocol's accounting invariants.

### Recommendation

Add a guard in `EVMAddress.deposit` (Cadence) and/or in `newInternalEVMTypeDepositFunction` (Go) rejecting deposits whose target address equals the `NativeTokenBridgeAddress`:

**Cadence (`fvm/evm/stdlib/contract.cdc`):**
```cadence
fun deposit(from: @FlowToken.Vault) {
    pre {
        !EVM.isPaused(): "EVM operations are temporarily paused"
        self.bytes != InternalEVM.nativeTokenBridgeAddress():
            "EVM.EVMAddress.deposit(): Cannot deposit directly to the native token bridge address"
    }
    ...
}
```

**Go (`fvm/evm/emulator/emulator.go`, `mintTo`):**
```go
// Guard: recipient must not be the bridge itself
if call.To.ToCommon() == bridge {
    return types.NewInvalidResult(call.Type, call.Hash(),
        errors.New("deposit recipient must not be the native token bridge address")), nil
}
```

### Proof of Concept

```cadence
import EVM from <EVMContractAddress>
import FungibleToken from <FungibleTokenAddress>
import FlowToken from <FlowTokenAddress>

transaction(bridgeAddressBytes: [UInt8; 20], amount: UFix64) {
    prepare(signer: auth(BorrowValue) &Account) {
        // Withdraw Cadence tokens
        let vault = signer.storage
            .borrow<auth(FungibleToken.Withdraw) &FlowToken.Vault>(from: /storage/flowTokenVault)!
            .withdraw(amount: amount) as! @FlowToken.Vault

        // Construct EVMAddress pointing at NativeTokenBridgeAddress
        let bridgeEVMAddress = EVM.EVMAddress(bytes: bridgeAddressBytes)

        // Deposit — no guard prevents this
        bridgeEVMAddress.deposit(from: <-vault)

        // Result: Cadence vault consumed, EVM bridge balance inflated,
        // tokens permanently inaccessible to any user.
    }
}
```

`bridgeAddressBytes` is computed as `MakePrecompileAddress(0)` — a publicly known constant derived from `FlowEVMExtendedPrecompileAddressPrefix` with index `0`. [5](#0-4) [6](#0-5)

```go
func (proc *procedure) mintTo(
	call *types.DirectCall,
) (*types.Result, error) {
	// check and convert value
	value, isValid := checkAndConvertValue(call.Value)
	if !isValid {
		return types.NewInvalidResult(
			call.Type,
			call.Hash(),
			types.ErrInvalidBalance,
		), nil
	}

	// create bridge account if not exist
	bridge := call.From.ToCommon()
	if !proc.state.Exist(bridge) {
		proc.state.CreateAccount(bridge)
	}

	// add balance to the bridge account before transfer
	proc.state.AddBalance(bridge, value, gethTracing.BalanceIncreaseWithdrawal)
	// check state errors
	if err := proc.state.Error(); err != nil {
		return nil, err
	}

	// withdraw the amount and move it to the bridge account
	res, err := proc.run(call.Message(), call.Hash(), types.DirectCallTxType)
	if err != nil {
		return res, err
	}

	// if any error (invalid or vm) on the internal call, revert and don't commit any change
	// this prevents having cases that we add balance to the bridge but the transfer
	// fails due to gas, etc.
	if res.Invalid() || res.Failed() {
		// reset the state to revert the add balances
		proc.state.Reset()
		return res, nil
	}

	// commit and finalize the state and return any stateDB error
	res.StateChangeCommitment, err = proc.commit(true)
	return res, err
}
```
