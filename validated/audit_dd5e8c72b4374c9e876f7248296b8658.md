### Title
Integer Division Truncation in WalletConnect Offer Royalty Display Understates User's Total Spend - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
The `formatAmountWithRoyalties` function, used exclusively in the WalletConnect approval dialog for `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` commands, performs two successive BigInt integer divisions that truncate (floor) the royalty calculation. The result is that `amountWithRoyalties` — the total cost shown to the user before they approve — is systematically understated. A user approves a WalletConnect offer believing they are spending less than the transaction will actually deduct.

### Finding Description

In `packages/gui/src/electron/commands/parseCommandDisplay.ts`, the function `formatAmountWithRoyalties` computes the total fungible amount (base + royalties) to display in the WalletConnect signing dialog:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // truncates
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,  // truncates
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

Two truncations occur on every call:

1. **Split truncation** (`amount / BigInt(royaltyPercentages.length)`): When the mojo amount is not evenly divisible by the number of NFTs, the remainder is silently dropped before royalties are computed. The royalty basis is smaller than the actual per-NFT allocation.

2. **Royalty truncation** (`(splitAmount * BigInt(royaltyPercentage)) / 10_000n`): For each NFT, the per-NFT royalty is floored. The discarded remainder can be up to 9 999 mojos per NFT per fungible asset.

The result `amountWithRoyalties` is then surfaced directly in the WalletConnect confirmation dialog via `walletDeltaToDisplay` → `withRoyaltyTotals`: [2](#0-1) 

This display value is the only cost signal the user sees before approving the WalletConnect request for `chia_wallet.take_offer`: [3](#0-2) 

### Impact Explanation

The `amountWithRoyalties` field is the total-spend figure shown in the WalletConnect approval dialog. Because both BigInt divisions floor their results, the displayed total is always ≤ the actual on-chain cost. The user reads an understated number and approves.

For XCH (12 decimal places, 1 XCH = 10¹² mojos) the per-NFT truncation error is at most 9 999 mojos ≈ 0.00000001 XCH — negligible. For CATs (3 decimal places, 1 CAT = 1 000 mojos) the per-NFT truncation error is up to 9 999 mojos ≈ 9.999 CAT per NFT. With multiple NFTs in a single offer the errors accumulate: 10 NFTs → up to ~99.99 CAT understated in the approval dialog.

This fits the allowed High impact: *"WalletConnect state that causes a user to approve… the wrong… amount."*

### Likelihood Explanation

Any multi-NFT WalletConnect offer where the mojo amount is not a multiple of `(10 000 × number_of_NFTs)` will trigger at least one truncation. A malicious dApp can freely choose the offer amount and the NFTs included, making it straightforward to craft values that maximise the discrepancy. No special privileges are required; the attacker only needs to send a WalletConnect `chia_wallet.take_offer` request.

### Recommendation

Replace the two flooring BigInt divisions with exact arithmetic that accumulates the full royalty for each NFT against the unmodified `amount`:

```typescript
// Correct: each NFT's royalty is computed on the full amount, then summed
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
```

This eliminates the split-truncation entirely and reduces the per-NFT royalty truncation to at most 9 999 mojos (unavoidable with integer arithmetic), matching the behaviour of the Chia node.

### Proof of Concept

**Setup:** A dApp sends a WalletConnect `chia_wallet.take_offer` request for an offer that:
- Requests 3 CAT tokens (3 000 mojos) from the user
- Offers 2 NFTs in return, with royalty percentages 9 999 and 9 999 (99.99% each, the maximum)

**Current GUI calculation (`formatAmountWithRoyalties`):**
```
amount              = 3_000n mojos
royaltyPercentages  = [9999, 9999]
splitAmount         = 3_000n / 2n = 1_500n          ← truncation 1 (exact here)
royalty NFT1        = (1_500n * 9_999n) / 10_000n
                    = 14_998_500n / 10_000n = 1_499n  ← truncation 2 (drops 8_500)
royalty NFT2        = 1_499n                          ← same
royaltyAmount       = 2_998n
amountWithRoyalties = 5_998n mojos → displayed as "5.998 CAT"
```

**Correct total (what the node charges):**
```
royalty NFT1 = (3_000n * 9_999n) / 10_000n = 2_999n
royalty NFT2 = 2_999n
royaltyAmount = 5_998n
actual total  = 8_998n mojos → "8.998 CAT"
```

The user sees **5.998 CAT** in the approval dialog but the transaction deducts **8.998 CAT** — a 50 % understatement. The user approves based on the incorrect figure. [4](#0-3)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}
