### Title
Missing Maximum Commission Validation in Pool Entry Functions Allows Staker to Drain Delegator Yield - (File: `src/pool/pool.cairo`)

### Summary
The pool contract's `enter_delegation_pool()`, `add_to_delegation_pool()`, and `switch_delegation_pool()` functions accept no `max_commission` parameter. A staker who holds an active `commission_commitment` can increase commission to `max_commission` (up to 100%) at any point after a delegator commits funds. Delegators have no on-chain slippage guard, resulting in theft of their unclaimed yield.

### Finding Description
`enter_delegation_pool()` at `src/pool/pool.cairo` line 182 accepts only `reward_address` and `amount`:

```rust
fn enter_delegation_pool(
    ref self: ContractState, reward_address: ContractAddress, amount: Amount,
)
```

`add_to_delegation_pool()` (line 221) and `switch_delegation_pool()` (line 379) are identically unguarded.

The enabling mechanism is `set_commission_commitment()` in `src/staking/staking.cairo` (line 748). A staker calls it to register a `CommissionCommitment { max_commission, expiration_epoch }` where `max_commission` may be up to `COMMISSION_DENOMINATOR` (10 000 = 100%) and the window may span up to one year. Once the commitment is active, `update_commission()` (line 1573) permits the staker to raise commission to any value ≤ `max_commission`:

```rust
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
}
```

The developers themselves flag this gap in a NatSpec comment directly above `set_commission_commitment()` at line 745–747:

```
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
```

Because `enter_delegation_pool()` carries no `max_commission` guard, a delegator who observes a 2 % commission and submits a delegation transaction has no protection if the staker raises commission to 100 % in the same block or any subsequent block within the commitment window. The delegator's principal is safe, but every unit of yield accrued until they complete the exit-wait-window is redirected entirely to the staker as commission.

### Impact Explanation
A staker with an active commitment can raise commission to 100 % immediately after a delegator enters the pool. All rewards generated during the exit-wait-window flow to the staker as commission; the delegator receives zero yield. This constitutes **theft of unclaimed yield** (High impact per the allowed scope). The loss is bounded by the exit-wait-window duration and the delegator's proportional share of pool rewards, but can be material for large delegations or long exit windows.

### Likelihood Explanation
Medium. The precondition is that the staker has called `set_commission_commitment()` with a `max_commission` above the current rate. This is a public on-chain action, but delegators may not monitor it, may trust the staker's reputation, or may delegate in the same block as a commitment-enabled commission hike. The staker has a clear profit motive (capturing delegator yield) and the action requires no privileged access beyond owning the staker key.

### Recommendation
Add a `max_commission: Commission` parameter to `enter_delegation_pool()`, `add_to_delegation_pool()`, and `switch_delegation_pool()`. At the start of each function, read the current commission from the staking contract and assert it does not exceed the caller-supplied `max_commission`, reverting with a descriptive error if it does. This mirrors the `minOut` pattern recommended in the reference report.

### Proof of Concept
1. Staker calls `set_commission(200)` (2 %) and `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 50)`.
2. Delegator observes 2 % commission and calls `enter_delegation_pool(reward_address, amount)` — no `max_commission` guard exists.
3. Staker immediately calls `set_commission(10000)` (100 %). `update_commission()` at `src/staking/staking.cairo:1580–1597` permits this because `10000 <= commitment.max_commission`.
4. All subsequent reward distributions assign 100 % of pool rewards to the staker as commission; the delegator's `unclaimed_rewards` accumulates zero.
5. Delegator calls `exit_delegation_pool_intent()` and waits out the exit-wait-window. Every epoch of yield during that window is lost to the staker's commission.
6. Delegator recovers principal via `exit_delegation_pool_action()` but receives no yield for the entire period.

                        pool_member, old_delegated_stake, new_delegated_stake,

        /// **Note**: Current commission increase safeguards still allow for sudden commission
        /// changes.
        /// **Note**: Updating epoch info can impact the commission commitment expiration date.
        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
