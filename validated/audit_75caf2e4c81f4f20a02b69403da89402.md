### Title
Blocking Extension Architecture With No Bypass Mechanism Permanently Traps LP Funds When a `beforeRemoveLiquidity` or `afterRemoveLiquidity` Extension Fails - (File: metric-core/contracts/ExtensionCalling.sol)

---

### Summary

Extensions in `MetricOmmPool` are stored as **immutable** contract variables and called in a **fully blocking** manner. If any extension registered for `beforeRemoveLiquidity` or `afterRemoveLiquidity` permanently reverts — due to a bug, self-destruct, or external dependency failure — `removeLiquidity` is permanently bricked with no admin or protocol escape hatch. LP principal is irretrievably locked.

---

### Finding Description

**Immutable, blocking extension wiring:**

`ExtensionCalling.sol` stores up to seven extension addresses as Solidity `immutable` variables set at construction time: [1](#0-0) 

These are set once in the constructor and can never be changed: [2](#0-1) 

**Blocking call semantics — reverts bubble up unconditionally:**

`CallExtension.callExtension` propagates any extension revert directly to the caller with no try/catch or skip path: [3](#0-2) 

`_callExtensionsInOrder` calls each extension in the configured order through this blocking path: [4](#0-3) 

**`removeLiquidity` is gated by both before and after extension hooks:** [5](#0-4) 

Note that `removeLiquidity` has **no** `whenNotPaused` modifier — pausing the pool does not bypass extensions and does not restore LP withdrawal access.

**No admin or protocol escape hatch exists:**

The factory's pool-admin interface exposes fee changes, pause transitions, and price-provider rotation — but **zero** mechanism to disable, replace, or bypass a broken extension: [6](#0-5) 

By contrast, the price provider — a similarly critical dependency — has a full `proposePoolPriceProvider` / `executePoolPriceProviderUpdate` rotation path: [7](#0-6) 

Extensions have no equivalent recovery path.

**`ValidateExtensionsConfig` validates structure at creation only — not liveness:** [8](#0-7) 

The factory validates that extension addresses are non-zero and non-duplicate at pool creation, but performs no liveness check and provides no post-deployment recourse.

---

### Impact Explanation

If any extension registered for `BEFORE_REMOVE_LIQUIDITY_ORDER` or `AFTER_REMOVE_LIQUIDITY_ORDER` permanently reverts, every call to `removeLiquidity` reverts. All LP shares become permanently non-redeemable. The underlying token balances remain locked in the pool contract with no withdrawal path for any LP. This is a direct, permanent loss of user principal with no protocol-level recovery.

---

### Likelihood Explanation

The trigger does not require malicious setup. Legitimate failure modes include:

- An extension that depends on an external oracle or registry that becomes permanently unavailable (e.g., `OracleValueStopLossExtension` calling into a deprecated oracle)
- A bug in a first-party extension (e.g., arithmetic overflow, storage corruption) that causes it to always revert after a state transition
- An extension contract that is self-destructed (possible on chains where `SELFDESTRUCT` is still available), causing `callExtension` to receive empty return data and revert with `InvalidExtensionResponse`

The `OracleValueStopLossExtension` is a first-party extension shipped in `metric-periphery` that reads external oracle state in `afterSwap`. Any permanent oracle failure would permanently block swaps on pools using it. A similar extension registered for `beforeRemoveLiquidity` would permanently block withdrawals.

---

### Recommendation

1. **Add a non-blocking extension call mode** (try/catch pattern) so that a failed extension emits an event and is skipped rather than reverting the entire operation — analogous to LayerZero's nonblocking receiver pattern.
2. **Add an emergency extension bypass** callable by the pool admin or protocol owner that allows a specific extension slot to be disabled (set to a no-op passthrough) after a timelock, analogous to `forceResumeReceive`.
3. At minimum, **document clearly** that extensions registered for `beforeRemoveLiquidity` / `afterRemoveLiquidity` must be treated as critical infrastructure with the same reliability requirements as the price provider, and that no recovery path exists if they fail.

---

### Proof of Concept

```
1. Pool is deployed with ExtensionA registered for beforeRemoveLiquidity.
2. ExtensionA has a dependency on ExternalContract.
3. ExternalContract is deprecated/self-destructed; ExtensionA now always reverts.
4. LP calls removeLiquidity(owner, salt, deltas, "").
5. MetricOmmPool._beforeRemoveLiquidity() calls _callExtensionsInOrder(BEFORE_REMOVE_LIQUIDITY_ORDER, ...).
6. CallExtension.callExtension(ExtensionA, data) → ExtensionA reverts.
7. Revert propagates unconditionally through callExtension → _callExtensionsInOrder → removeLiquidity.
8. LP's call reverts. LP cannot withdraw.
9. Pool admin calls pausePool / unpausePool — no effect on removeLiquidity (no whenNotPaused guard).
10. Protocol owner has no function to disable ExtensionA.
11. LP funds are permanently locked.
```

  function _callExtensionsInOrder(uint256 order, bytes memory data) private {
    if (order == 0) return;

    while (true) {
      uint256 extensionIndex = order & 0x7;
      if (extensionIndex == 0) break;
      address extension = _extensionAddress(extensionIndex);
      if (extension == address(0)) revert PanicEmptyExtension();
      CallExtension.callExtension(extension, data);
      order >>= 3;
    }
  }

  function callExtension(address extension, bytes memory data) internal {
    (bool success, bytes memory result) = extension.call(data);
    if (!success) {
      if (result.length > 0) {
        assembly ("memory-safe") {
          revert(add(result, 32), mload(result))
        }
      }
      revert ExtensionCallFailed();
    }
    if (result.length < 32) {
      revert InvalidExtensionResponse();
    }
    bytes4 returnedSelector;
    assembly ("memory-safe") {
      returnedSelector := mload(add(result, 32))
    }
    bytes4 expectedSelector;
    assembly ("memory-safe") {
      expectedSelector := mload(add(data, 32))
    }
    if (returnedSelector != expectedSelector) {
      revert InvalidExtensionResponse();
    }
  }

  function proposePoolPriceProvider(address pool, address newPriceProvider)
    external
    override
    nonReentrant
    onlyPoolAdmin(pool)
  {
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    uint256 timelock = priceProviderTimelock[pool];
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, newPriceProvider);

    address mutableProvider = PoolStateLibrary._slot3(pool);
    address current = mutableProvider != address(0) ? mutableProvider : p.immutablePriceProvider;
    uint256 executeAfter = block.timestamp + timelock;
    pendingPriceProvider[pool] = newPriceProvider;
    pendingPriceProviderExecuteAfter[pool] = executeAfter;
    emit PoolPriceProviderChangeProposed(pool, current, newPriceProvider, executeAfter);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function executePoolPriceProviderUpdate(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPriceProvider[pool];
    if (pending == address(0)) revert NoPriceProviderChangeProposed();
    uint256 execAfter = pendingPriceProviderExecuteAfter[pool];
    // forge-lint: disable-next-line(block-timestamp) -- timelock enforcement legitimately relies on `block.timestamp`.
    if (block.timestamp < execAfter) revert PriceProviderTimelockNotElapsed(execAfter, block.timestamp);
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, pending);
    IMetricOmmPoolFactoryActions(pool).setPriceProvider(pending);
    delete pendingPriceProvider[pool];
    delete pendingPriceProviderExecuteAfter[pool];
    emit PoolPriceProviderUpdated(pool, pending);
  }

  function validateExtensionsConfig(
    address[] calldata extensions,
    ExtensionOrders calldata orders,
    bytes[] calldata extensionInitData
  ) internal pure {
    if (extensions.length > MAX_EXTENSIONS) revert InvalidExtensionsConfig();
    if (extensionInitData.length != extensions.length) revert InvalidExtensionsConfig();

    if (extensions.length == 0) {
      if (!_ordersAreZero(orders)) revert InvalidExtensionsConfig();
      return;
    }

    for (uint256 i = 0; i < extensions.length; i++) {
      if (extensions[i] == address(0)) revert InvalidExtensionsConfig();
      for (uint256 j = 0; j < i; j++) {
        if (extensions[i] == extensions[j]) revert InvalidExtensionsConfig();
      }
    }

    if (_ordersAreZero(orders)) revert InvalidExtensionsConfig();

    _validateExtensionOrder(orders.beforeAddLiquidity, extensions.length);
    _validateExtensionOrder(orders.afterAddLiquidity, extensions.length);
    _validateExtensionOrder(orders.beforeRemoveLiquidity, extensions.length);
    _validateExtensionOrder(orders.afterRemoveLiquidity, extensions.length);
    _validateExtensionOrder(orders.beforeSwap, extensions.length);
    _validateExtensionOrder(orders.afterSwap, extensions.length);
  }
