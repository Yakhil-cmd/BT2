### Title
Unauthenticated gRPC `Set()` on TCP Listener Allows Any Network Peer to Permanently Mutate Execution-Node Ledger WAL and MTrie — (`cmd/ledger/main.go`, `ledger/remote/service.go`)

### Summary
The standalone ledger service binary (`cmd/ledger/main.go`) starts a gRPC server with no authentication interceptors and no TLS. When deployed with `--ledger-service-tcp 0.0.0.0:9000`, any process that can reach the TCP port — including an unstaked Access or Observer node — can call `LedgerServiceClient.Set()` with an arbitrary state root and arbitrary register keys/values, causing the execution node's WAL and in-memory MTrie forest to be permanently mutated with attacker-supplied data.

### Finding Description
`grpc.NewServer()` in `cmd/ledger/main.go` (line 121) registers no `grpc.UnaryInterceptor` or `grpc.StreamInterceptor` for authentication or authorization. The `remote.Service.Set()` handler in `ledger/remote/service.go` (line 133) performs only structural input validation before forwarding the caller-supplied `(state, keys, values)` tuple directly to `ledger.Ledger.Set()`. That call path unconditionally:
1. Converts the update to a `TrieUpdate` via `pathfinder.UpdateToTrieUpdate`.
2. Sends it to the compactor goroutine, which calls `DiskWAL.RecordUpdate()` and writes the record to the on-disk WAL segment.
3. Creates a new MTrie node via `forest.NewTrie(trieUpdate)` and adds it to the in-memory forest via `forest.AddTrie(newTrie)`.

No guard exists at any layer between the TCP socket and the WAL write.

### Impact Explanation
A successful call permanently appends an attacker-controlled `TrieUpdate` record to the WAL and inserts the resulting trie into the MTrie forest. On the next execution block, the execution node will read the injected state as authoritative, compute a divergent execution state root, and include it in execution receipts. Verification nodes will reject those receipts, halting finalization for blocks processed by this execution node. The WAL record survives process restart (it is replayed on startup), making the corruption persistent.

### Likelihood Explanation
The TCP flag is a documented, first-class production option (`--ledger-service-tcp 0.0.0.0:9000`). Any network peer that can route to the port — including an unstaked Access or Observer node co-located in the same data center or reachable via the public internet if the port is not firewalled — can exploit this with a single standard gRPC call using `insecure.NewCredentials()`. No special privileges, no leaked keys, and no staked node role are required.

### Recommendation
1. **Mutual TLS**: Require client certificates on the gRPC server; only the execution node's certificate should be trusted.
2. **gRPC authentication interceptor**: Register a `grpc.UnaryInterceptor` that validates a shared secret or client certificate before dispatching any RPC.
3. **Restrict write RPCs**: At minimum, gate `Set()` and `Prove()` behind an authorization check; read-only RPCs (`Get`, `GetSingleValue`, `HasState`, `InitialState`) have lower risk.
4. **Prefer Unix sockets**: The `--ledger-service-socket` path relies on OS file-system permissions (mode `0660`) and is inherently not network-reachable; prefer it over TCP for same-host deployments.
5. **Document network exposure**: If TCP is required for cross-host deployment, document that the port must be firewalled to the execution node's IP only.

func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
	if req.State == nil || len(req.State.Hash) != len(ledger.State{}) {
		return nil, status.Error(codes.InvalidArgument, "invalid state")
	}

	if len(req.Keys) == 0 {
		return nil, status.Error(codes.InvalidArgument, "keys cannot be empty")
	}

	if len(req.Keys) != len(req.Values) {
		return nil, status.Error(codes.InvalidArgument, "keys and values length mismatch")
	}

	var state ledger.State
	copy(state[:], req.State.Hash)

	keys := make([]ledger.Key, len(req.Keys))
	for i, protoKey := range req.Keys {
		key, err := protoKeyToLedgerKey(protoKey)
		if err != nil {
			return nil, err // protoKeyToLedgerKey already returns status.Error
		}
		keys[i] = key
	}

	values := make([]ledger.Value, len(req.Values))
	for i, protoValue := range req.Values {
		var value ledger.Value
		// Reconstruct the original value type using is_nil flag
		// This preserves the distinction between nil and []byte{} that protobuf loses
		if len(protoValue.Data) == 0 {
			if protoValue.IsNil {
				// Original value was nil
				value = nil
			} else {
				// Original value was []byte{} (empty slice)
				value = ledger.Value([]byte{})
			}
		} else {
			// Non-empty value, use data as-is
			value = ledger.Value(protoValue.Data)
		}
		values[i] = value
	}

	update, err := ledger.NewUpdate(state, keys, values)
	if err != nil {
		return nil, status.Error(codes.InvalidArgument, err.Error())
	}

	newState, trieUpdate, err := s.ledger.Set(update)
	if err != nil {
		return nil, status.Error(codes.Internal, err.Error())
	}

func (l *Ledger) set(trieUpdate *ledger.TrieUpdate) (newState ledger.State, err error) {

	// resultCh is a buffered channel to receive WAL update result.
	resultCh := make(chan error, 1)

	// trieCh is a buffered channel to send updated trie.
	// trieCh can be closed without sending updated trie to indicate failure to update trie.
	trieCh := make(chan *trie.MTrie, 1)
	defer close(trieCh)

	// There are two goroutines:
	// 1. writing the trie update to WAL (in Compactor goroutine)
	// 2. creating a new trie from the trie update (in this goroutine)
	// Since writing to WAL is running concurrently, we use resultCh
	// to receive WAL update result from Compactor.
	// Compactor also needs new trie created here because Compactor
	// caches new trie to minimize memory foot-print while checkpointing.
	// `trieCh` is used to send created trie to Compactor.
	l.trieUpdateCh <- &WALTrieUpdate{Update: trieUpdate, ResultCh: resultCh, TrieCh: trieCh}

	newTrie, err := l.forest.NewTrie(trieUpdate)
	walError := <-resultCh

	if err != nil {
		return ledger.State(hash.DummyHash), fmt.Errorf("cannot update state: %w", err)
	}
	if walError != nil {
		return ledger.State(hash.DummyHash), fmt.Errorf("error while writing LedgerWAL: %w", walError)
	}

	err = l.forest.AddTrie(newTrie)
	if err != nil {
		return ledger.State(hash.DummyHash), fmt.Errorf("failed to add new trie to forest: %w", err)
	}

	trieCh <- newTrie

	return ledger.State(newTrie.RootHash()), nil
}

func (w *DiskWAL) RecordUpdate(update *ledger.TrieUpdate) (segmentNum int, skipped bool, err error) {
	if w.paused {
		return 0, true, nil
	}

	bytes := EncodeUpdate(update)

	locations, err := w.wal.Log(bytes)

	if err != nil {
		return 0, false, fmt.Errorf("error while recording update in LedgerWAL: %w", err)
	}
	if len(locations) != 1 {
		return 0, false, fmt.Errorf("error while recording update in LedgerWAL: got %d location, expect 1 location", len(locations))
	}

	return locations[0].Segment, false, nil
}
