### Title
Unpermissioned `deploy_erc20_token()` Allows Any Caller to Permanently Occupy the NEP-141→ERC-20 Bridge Slot - (`engine/src/contract_methods/connector.rs`)

---

### Summary

`deploy_erc20_token()` in the Aurora Engine connector has no caller access control. Any NEAR account can call it with an arbitrary NEP-141 token account ID, permanently registering a NEP-141→ERC-20 mapping. Because `register_token()` rejects any subsequent registration attempt for the same NEP-141 ID with `TokenAlreadyRegistered`, the legitimate protocol can never deploy the canonical ERC-20 bridge contract for that token. The analogous sibling function `mirror_erc20_token()` correctly enforces `require_owner_only`, making the missing guard on `deploy_erc20_token()` a clear inconsistency.

---

### Finding Description

The public NEAR contract method `deploy_erc20_token` is exposed without any caller restriction:

```rust
// engine/src/lib.rs  lines 612-621
/// Deploy ERC20 token mapped to a NEP141
#[unsafe(no_mangle)]
pub extern "C" fn deploy_erc20_token() {
    let io = Runtime;
    let env = Runtime;
    let mut handler = Runtime;
    contract_methods::connector::deploy_erc20_token(io, &env, &mut handler)
        .map_err(ContractError::msg)
        .sdk_unwrap();
}
``` [1](#0-0) 

The implementation in `connector.rs` only checks `require_running` — there is no `require_owner_only` guard:

```rust
// engine/src/contract_methods/connector.rs  lines 111-159
pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I, env: &E, handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;   // ← only guard
        // ... no require_owner_only ...
        match args {
            DeployErc20TokenArgs::Legacy(nep141) => {
                let address = engine::deploy_erc20_token(nep141, None, io, env, handler)?;
                ...
            }
            DeployErc20TokenArgs::WithMetadata(nep141) => {
                // comment says "owner only" but is NOT enforced
                ...
            }
        }
    })
}
``` [2](#0-1) 

The comment on line 149 explicitly acknowledges the intent ("this transaction could be executed by the owner of the contract only") but the check is absent: [3](#0-2) 

Once called, `engine::deploy_erc20_token` calls `register_token`, which permanently locks the NEP-141→ERC-20 slot:

```rust

There is no unregister or override path. The bijection map insert is final: [5](#0-4)

---

### Impact Explanation

An attacker who front-runs a legitimate `deploy_erc20_token("usdc.near")` call causes the NEP-141→ERC-20 mapping to be permanently set to an ERC-20 address derived from the attacker's NEAR account ID and nonce, not the address the protocol intended. Because the registration is irreversible, the protocol can never deploy the canonical ERC-20 bridge contract for that token at the expected address. Any downstream integrations (DEXes, wallets, cross-chain routers) that reference the expected ERC-20 address will be permanently broken. Users holding the NEP-141 token on NEAR who attempt to bridge to the expected ERC-20 address will find no registered mapping, effectively freezing their ability to use the bridge at the intended address. This constitutes **permanent freezing of bridge access** for any targeted NEP-141 token.

---

### Likelihood Explanation

The attack requires only a standard NEAR transaction with no special privileges, no tokens, and no deposit. The attacker only needs to know the NEP-141 account ID of a token the protocol intends to bridge — information that is public. The attack can be executed by monitoring the NEAR transaction pool for pending `deploy_erc20_token` calls and submitting a competing transaction first, or proactively targeting well-known tokens (e.g., `usdc.near`, `wrap.near`) before the protocol deploys them. Cost is a single NEAR gas fee.

---

### Recommendation

Add `require_owner_only` to `deploy_erc20_token` in `engine/src/contract_methods/connector.rs`, consistent with the existing guard in `mirror_erc20_token`:

```rust
pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I, env: &E, handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;  // ← add this
        ...
    })
}
``` [7](#0-6)

### Proof of Concept

1. Protocol announces intent to bridge `usdc.near` to Aurora.
2. Attacker calls `deploy_erc20_token` on the Aurora Engine contract with `DeployErc20TokenArgs::Legacy("usdc.near")` from any NEAR account.
3. `engine::deploy_erc20_token` deploys an `EvmErc20` contract at address `X` (derived from `near_account_to_evm_address(attacker_account)` + nonce) and calls `register_token(X, "usdc.near")`.
4. The bijection map now permanently holds `usdc.near → X`.
5. Protocol's subsequent `deploy_erc20_token("usdc.near")` call reaches `register_token` and returns `Err(RegisterTokenError::TokenAlreadyRegistered)`, causing the transaction to fail.
6. The protocol's intended ERC-20 address `Y` is never registered. All integrations referencing `Y` are permanently broken. Users cannot bridge `usdc.near` to the expected address. [8](#0-7) [9](#0-8)

#[named]
pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let bytes = io.read_input().to_vec();
        let args =
            DeployErc20TokenArgs::deserialize(&bytes).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

        match args {
            DeployErc20TokenArgs::Legacy(nep141) => {
                let address = engine::deploy_erc20_token(nep141, None, io, env, handler)?;

                io.return_output(
                    &borsh::to_vec(address.as_bytes()).map_err(|_| errors::ERR_SERIALIZE)?,
                );
                Ok(PromiseOrValue::Value(address))
            }
            DeployErc20TokenArgs::WithMetadata(nep141) => {
                let args = borsh::to_vec(&nep141).map_err(|_| errors::ERR_SERIALIZE)?;
                let base = PromiseCreateArgs {
                    target_account_id: nep141,
                    method: "ft_metadata".to_string(),
                    args: vec![],
                    attached_balance: ZERO_YOCTO,
                    attached_gas: READ_PROMISE_ATTACHED_GAS,
                };
                let callback = PromiseCreateArgs {
                    target_account_id: env.current_account_id(),
                    method: "deploy_erc20_token_callback".to_string(),
                    args,
                    attached_balance: ZERO_YOCTO,
                    attached_gas: DEPLOY_ERC20_TOKEN_CALLBACK_ATTACHED_GAS,
                };
                // Safe because these promises are read-only calls to the main engine contract
                // and this transaction could be executed by the owner of the contract only.
                let promise_args = PromiseWithCallbackArgs { base, callback };
                let promise_id = handler.promise_create_with_callback(&promise_args);

                handler.promise_return(promise_id);

                Ok(PromiseOrValue::Promise(promise_args))
            }
        }
    })
}

pub fn deploy_erc20_token<I: IO + Copy, E: Env, P: PromiseHandler>(
    nep141: AccountId,
    metadata: Option<Erc20Metadata>,
    io: I,
    env: &E,
    handler: &mut P,
) -> Result<Address, DeployErc20Error> {
    let current_account_id = env.current_account_id();
    let input = setup_deploy_erc20_input(&current_account_id, metadata);
    let mut engine: Engine<_, _> = Engine::new(
        aurora_engine_sdk::types::near_account_to_evm_address(
            env.predecessor_account_id().as_bytes(),
        ),
        current_account_id,
        io,
        env,
    )
    .map_err(DeployErc20Error::State)?;

    let address = match engine.deploy_code_with_input(input, None, handler) {
        Ok(result) => match result.status {
            TransactionStatus::Succeed(ret) => {
                Address::new(H160(ret.as_slice().try_into().unwrap()))
            }
            other => return Err(DeployErc20Error::Failed(other)),
        },
        Err(e) => return Err(DeployErc20Error::Engine(e)),
    };

    sdk::log!("Deployed ERC-20 in Aurora at: {:#?}", address);
    engine
        .register_token(address, nep141)
        .map_err(DeployErc20Error::Register)?;

    Ok(address)

pub fn require_owner_only(
    state: &state::EngineState,
