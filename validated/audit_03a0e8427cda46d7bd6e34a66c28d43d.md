### Title
Gateway Stateful Validator Uses Stale Previous-Block L2 Gas Price for Resource Bounds Threshold, Causing Incorrect Admission Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price from the **latest committed (previous) block** and uses it as the reference for the minimum gas price threshold check. The code itself contains a `TODO` acknowledging this is wrong. This causes the gateway to make incorrect admission decisions: when gas prices are rising, underpriced transactions are admitted to the mempool (they will fail at blockifier execution); when gas prices are falling, correctly-priced transactions are rejected at the gateway even though they would succeed at execution.

### Finding Description

In `validate_resource_bounds` (lines 229–240), the code reads `previous_block_l2_gas_price` from `get_block_info()`, which returns the latest committed block's gas prices:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

This value is then used in `validate_tx_l2_gas_price_within_threshold` to compute the admission threshold:

```
threshold = (min_gas_price_percentage / 100) * previous_block_l2_gas_price
``` [2](#0-1) 

The correct reference is the **next block's** L2 gas price — the block the transaction will actually be included in. The `run_validate_entry_point` method already increments the block number with `block_info.block_number.unchecked_next()` when building the blockifier context, confirming the gateway is aware that validation runs against the next block. The gas price used for the threshold check is not updated to match. [3](#0-2) 

The blockifier's authoritative check in `check_fee_bounds` uses the actual next-block gas prices from `block_info.gas_prices.gas_price_vector(fee_type)`, creating a split between what the gateway admits and what the blockifier enforces: [4](#0-3) 

### Impact Explanation

Two distinct incorrect outcomes arise whenever `validate_resource_bounds` is enabled (`config.validate_resource_bounds = true`) and `min_gas_price_percentage > 0`:

**Case 1 — Gas prices rising (invalid transactions admitted):**
- Previous block L2 gas price: P_prev
- Next block L2 gas price: P_next > P_prev
- A transaction with `max_price_per_unit = X` where `(pct/100)*P_prev ≤ X < (pct/100)*P_next` passes the gateway threshold check but fails `check_fee_bounds` in the blockifier with `ResourceBoundsError::MaxGasPriceTooLow`.
- The transaction is incorrectly admitted to the mempool and wastes sequencer execution resources.

**Case 2 — Gas prices falling (valid transactions rejected):**
- Previous block L2 gas price: P_prev
- Next block L2 gas price: P_next < P_prev
- A transaction with `max_price_per_unit = X` where `(pct/100)*P_next ≤ X < (pct/100)*P_prev` is rejected by the gateway even though it would pass `check_fee_bounds` at execution.
- A valid, correctly-priced transaction is incorrectly rejected before sequencing.

This matches: **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

L2 gas prices change between blocks as L1 conditions (base fee, blob fee) and the ETH→STRK oracle rate fluctuate. The `L1GasPriceProvider` computes a rolling mean over recent L1 blocks, so the price can shift meaningfully between consecutive Starknet blocks. Any deployment with `min_gas_price_percentage > 0` is affected on every block where the L2 gas price changes. The trigger requires no special privileges — any user submitting a transaction exercises this path.

### Recommendation

Replace `previous_block_l2_gas_price` with the next block's L2 gas price. The consensus orchestrator already computes the next block's gas prices via `get_l1_prices_in_fri_and_wei_and_conversion_rate` before proposing a block. [5](#0-4) 

The gateway's `GatewayFixedBlockStateReader` should expose the pending/next block's gas prices, or the threshold check should be deferred to a point where the next block's price is known. At minimum, the check should be documented as a best-effort pre-filter with no correctness guarantee, and the blockifier's `check_fee_bounds` should be treated as the sole authoritative gate.

### Proof of Concept

```
Config: min_gas_price_percentage = 100, validate_resource_bounds = true

Scenario A (rising prices — invalid tx admitted):
  previous block strk l2_gas_price = 100
  next block strk l2_gas_price     = 200

  Attacker submits invoke tx with AllResourceBounds { l2_gas.max_price_per_unit = 110 }

  Gateway check (validate_resource_bounds):
    threshold = (100/100) * 100 = 100
    110 >= 100  →  PASS  (tx admitted to mempool)

  Blockifier check (check_fee_bounds, run_validate_entry_point):
    actual l2_gas_price = 200
    110 < 200  →  ResourceBoundsError::MaxGasPriceTooLow  (tx fails execution)

  Result: invalid transaction admitted, sequencer wastes execution resources.

Scenario B (falling prices — valid tx rejected):
  previous block strk l2_gas_price = 200
  next block strk l2_gas_price     = 100

  User submits invoke tx with AllResourceBounds { l2_gas.max_price_per_unit = 110 }

  Gateway check (validate_resource_bounds):
    threshold = (100/100) * 200 = 200
    110 < 200  →  GAS_PRICE_TOO_LOW  (tx rejected)

  Blockifier check (if it were reached):
    actual l2_gas_price = 100
    110 >= 100  →  PASS

  Result: valid transaction incorrectly rejected before sequencing.
```

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
