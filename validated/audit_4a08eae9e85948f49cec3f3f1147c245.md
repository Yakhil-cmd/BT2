### Title
Pool admin bypasses hard spread-fee cap via uncapped `setPoolBinAdditionalFees` — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` forwards `addFeeBuyE6` and `addFeeSellE6` to the pool with **no validation against the hard fee cap** (`HARD_MAX_SPREAD_FEE_E6 = 200_000`). Every other admin fee setter enforces the cap; this one does not. A pool admin can set per-bin additional fees up to `uint16` max (65 535 E6 = 6.5535 %) on top of an already-capped global spread fee, making the effective per-bin fee exceed the 20 % hard ceiling the protocol is supposed to enforce absolutely.

---

### Finding Description

The factory enforces `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20 %) as an absolute ceiling on spread fees through every relevant setter:

- `setFeeCaps` — reverts if any cap exceeds the hard limit [1](#0-0) 
- `setPoolAdminFees` — reverts if `newAdminSpreadFeeE6 > maxAdminSpreadFeeE6` [2](#0-1) 
- `_validatePoolParameters` at creation — reverts if admin or protocol fee exceeds cap [3](#0-2) 

`setPoolBinAdditionalFees`, however, performs **zero fee-value validation** and passes the caller-supplied values straight through:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [4](#0-3) 

The pool's `setBinAdditionalFees` only validates the bin index, not the fee magnitudes:

```solidity
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    ...
}
``` [5](#0-4) 

`addFeeBuyE6` and `addFeeSellE6` are typed `uint16`, so the maximum value a pool admin can write is **65 535** (6.5535 % in E6 units). These values are applied **on top of** the global `spreadFeeE6` during swaps. If the global spread fee is already at the 20 % hard cap, the effective per-bin fee reaches **26.5535 %** — 6.5535 percentage points above the ceiling the protocol guarantees to users.

The interface documentation confirms the additive semantics: `addFeeBuyE6` is described as "Additional fee on buys into the bin (E6) **on top of base spread**." [6](#0-5) 

---

### Impact Explanation

Traders swapping through a bin where the admin has set maximum per-bin fees pay up to 26.5535 % in spread fees instead of the documented 20 % maximum. The excess fee accrues as surplus and is distributed to the protocol and admin on the next `collectPoolFees` call. This is a direct, measurable loss of trader principal above the protocol-guaranteed ceiling, and constitutes an admin-boundary break: the pool admin exceeds the hard fee cap through a path the factory does not guard.

---

### Likelihood Explanation

The trigger requires only the pool admin to call `setPoolBinAdditionalFees` with `addFeeBuyE6 = 65535` (or any value that, combined with the global spread fee, exceeds 200 000 E6). The call is a single transaction with no timelock. Any pool whose admin is compromised, malicious, or simply unaware of the interaction with the global fee can reach this state. The audit scope explicitly lists `setBinAdditionalFees` as a target path.

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool. The combined per-bin effective fee (`spreadFeeE6 + addFeeBuyE6`) must not exceed `HARD_MAX_SPREAD_FEE_E6`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    // Read current total spread fee from the pool and enforce the hard cap
    uint24 currentSpread = IMetricOmmPool(pool).spreadFeeE6();
    if (uint256(currentSpread) + uint256(addFeeBuyE6)  > HARD_MAX_SPREAD_FEE_E6) revert AdminFeeTooHigh();
    if (uint256(currentSpread) + uint256(addFeeSellE6) > HARD_MAX_SPREAD_FEE_E6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, cap `addFeeBuyE6` and `addFeeSellE6` independently against `maxAdminSpreadFeeE6` to be consistent with the other admin fee setters.
