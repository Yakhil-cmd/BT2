### Title
Unauthorized Global Contract Overwrite via `DeployGlobalContractAction` with `AccountId` Mode — (File: `runtime/runtime/src/global_contracts.rs`)

---

### Summary

`action_deploy_global_contract` performs no authorization check when `deploy_mode == GlobalContractDeployMode::AccountId`. Because NEAR allows `signer_id ≠ receiver_id` in a transaction, any unprivileged account can send a `DeployGlobalContractAction(AccountId)` with `receiver_id = victim`, causing the victim's `AccountId`-keyed global contract slot to be overwritten with attacker-controlled WASM. Every account that references the victim's global contract by `AccountId` will subsequently execute the attacker's code.

---

### Finding Description

`GlobalContractDeployMode::AccountId` is documented as "the owner account id … this allows the **owner** to update the contract for all its users." The invariant is that only the account whose `AccountId` is used as the key may write to that slot.

The execution path is:

1. A signed transaction arrives with `signer_id = attacker`, `receiver_id = victim`, `Action::DeployGlobalContract { deploy_mode: AccountId, code: malicious_wasm }`.
2. The runtime dispatches to `action_deploy_global_contract` in `runtime/runtime/src/global_contracts.rs` (called from `runtime/runtime/src/lib.rs` lines 593–603).
3. Inside `action_deploy_global_contract` (lines 23–61), the **only** check is whether `account.amount() >= storage_cost`. There is no check that the deployer is the owner of the `AccountId` namespace.
4. `initiate_distribution` (lines 141–169) constructs the identifier as `GlobalContractIdentifier::AccountId(account_id.clone())` where `account_id` is the receipt's `receiver_id` — the victim's account.
5. The distribution receipt propagates shard-by-shard and `apply_distribution_current_shard` (lines 189–233) writes the attacker's bytecode to `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim) }`, unconditionally overwriting whatever the victim had previously deployed (subject only to the nonce freshness check, which the attacker trivially satisfies by using a higher nonce).

The nonce mechanism (`check_and_update_nonce`, lines 238–256) only prevents a **stale** distribution receipt from overwriting a **newer** one during cross-shard propagation. It provides zero protection against a fresh attacker-initiated deployment, which always carries a nonce higher than the current stored nonce.

---

### Impact Explanation

**Corrupted value:** `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim_account_id) }` — the WASM bytecode stored at this global trie key is replaced with attacker-controlled code.

**Downstream effect:** Every account whose `AccountContract` is `GlobalByAccount(victim_account_id)` (set via `UseGlobalContractAction`) will execute the attacker's WASM on the next `FunctionCall`. The attacker's contract runs with the full host-function API of the calling account, enabling theft of funds, state corruption, or arbitrary cross-contract calls on behalf of those accounts.

Additionally, the storage cost (`global_contract_storage_amount_per_byte × code_len`) is deducted from the **victim's** balance, not the attacker's, draining the victim's NEAR.

**Severity: Critical** — arbitrary code execution across all users of the victim's global contract, plus balance drain of the victim account.

---

### Likelihood Explanation

- Requires only a standard full-access key on the attacker's own account — no special privilege.
- The transaction is structurally valid; it passes all existing verifier checks (`validate_transaction`, `verify_and_charge_tx_ephemeral`) because those only verify the signer's key and balance.
- The attack is a single transaction; no race condition or timing dependency is required.
- Any account that has deployed a global contract in `AccountId` mode and has attracted users is a profitable target.

---

### Recommendation

In `action_deploy_global_contract`, when `deploy_mode == GlobalContractDeployMode::AccountId`, assert that the receipt's `predecessor_id` equals `account_id` (the `receiver_id`). For a direct transaction this equals `signer_id`; for a cross-contract call it equals the calling contract, which is the natural "owner" in that context. Concretely, pass the receipt's `predecessor_id` into the function and add:

```rust
if deploy_contract.deploy_mode == GlobalContractDeployMode::AccountId
    && predecessor_id != account_id
{
    result.result = Err(ActionErrorKind::ActorNoPermission {
        account_id: account_id.clone(),
        actor_id: predecessor_id.clone(),
    }.into());
    return Ok(());
}
```

This mirrors the authorization model used by `DeployContract` (which is always self-targeted in practice) and enforces the documented ownership invariant.

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
}

pub enum GlobalContractDeployMode {
    /// Contract is deployed under its code hash.
    /// Users will be able reference it by that hash.

    CodeHash,
    /// Contract is deployed under the owner account id.

}
