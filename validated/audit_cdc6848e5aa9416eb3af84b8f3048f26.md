### Title
Missing `threshold < 2` Validation in `assert_sign_inputs` Allows Signing Below Minimum Threshold - (File: src/frost/mod.rs)

### Summary
`assert_key_invariants` in `src/dkg.rs` enforces `threshold >= 2` before DKG proceeds. The shared signing-input validator `assert_sign_inputs` in `src/frost/mod.rs`, called by both `sign_v1` and `sign_v2` for EdDSA (and the RedJubjub equivalent), omits this same lower-bound check. A caller can therefore invoke the signing protocol with `threshold = 1`, causing FROST to construct a `KeyPackage` with `min_signers = 1` and accept a single participant's signature share as sufficient for aggregation — violating the threshold security guarantee established at key-generation time.

### Finding Description

`assert_key_invariants` in `src/dkg.rs` explicitly enforces a minimum threshold of 2:

```rust
// src/dkg.rs lines 580-582
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

`assert_sign_inputs` in `src/frost/mod.rs`, which is the sole validation gate for both `sign_v1` and `sign_v2`, checks only that `threshold` is not *too large*, but never that it is at least 2:

```rust
// src/frost/mod.rs lines 144-150
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no ThresholdTooSmall check here
``` [2](#0-1) 

Both `sign_v1` and `sign_v2` delegate all input validation exclusively to `assert_sign_inputs`: [3](#0-2) [4](#0-3) 

When `threshold = 1` passes validation, `construct_key_package` converts it directly into the FROST `KeyPackage`'s `min_signers` field:

```rust
// src/frost/eddsa/sign.rs lines 360-368
Ok(KeyPackage::new(
    identifier,
    signing_share,
    verifying_share,
    *verifying_key,
    u16::try_from(threshold.value())...,  // becomes min_signers = 1
))
``` [5](#0-4) 

With `min_signers = 1`, the FROST library's `aggregate` call accepts a single signature share as sufficient for producing a valid, verifiable EdDSA signature — even though the DKG was performed with `threshold >= 2`.

The same gap exists in `src/frost/mod.rs`'s `presign` function, which also omits the lower-bound check: [6](#0-5) 

### Impact Explanation

A malicious coordinator or application caller can invoke `sign_v1` or `sign_v2` with `threshold = 1`. The FROST `KeyPackage` is constructed with `min_signers = 1`, so the coordinator's own signature share alone satisfies the FROST aggregation requirement. This produces a cryptographically valid EdDSA signature without the quorum of participants mandated by the DKG threshold — an unauthorized creation of a valid threshold signature for attacker-chosen inputs.

**Impact: High** — Corruption of signing outputs: honest parties accept a signature produced with a threshold inconsistent with the established key-generation parameters, violating the threshold security guarantee.

### Likelihood Explanation

Any unprivileged library caller who controls the `threshold` argument to `sign_v1` or `sign_v2` can trigger this. No special cryptographic capability or key material beyond a valid `KeygenOutput` is required. The call path is direct and reachable from the public API.

### Recommendation

Add the same lower-bound check present in `assert_key_invariants` to `assert_sign_inputs` in `src/frost/mod.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Apply the same fix to the `presign` function in `src/frost/mod.rs`.

### Proof of Concept

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
