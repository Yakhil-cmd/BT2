Audit Report

## Title
Partial-Failure in `AccountState::transfer` Permanently Locks GTC Neurons on Transient Governance Errors — (`rs/nns/gtc/src/lib.rs`)

## Summary
`AccountState::transfer` unconditionally removes every neuron from `neuron_ids` and always returns `Ok(())` regardless of per-neuron `transfer_gtc_neuron` failures. `Gtc::donate_account` then unconditionally sets `has_donated = true`. Any neuron whose governance call failed is moved to `failed_transferred_neurons`, still exists in governance under GTC control, but is permanently unreachable because `has_donated = true` blocks every future `claim_neurons` and `donate_account` call on the account, with no admin recovery path exposed.

## Finding Description
**Root cause — `transfer()` always returns `Ok(())`**

In `rs/nns/gtc/src/lib.rs` lines 188–209, the loop over `neuron_ids`:
- Calls `self.neuron_ids.retain(|id| id != &neuron_id)` **unconditionally** at line 192, whether the governance call succeeded or failed.
- On failure, pushes the neuron to `failed_transferred_neurons` (line 204), not back into `neuron_ids`.
- Returns `Ok(())` at line 209 regardless of how many neurons failed.

**`donate_account` sets `has_donated` unconditionally**

Lines 89–90:
```rust
account.transfer(custodian_neuron_id).await?;
account.has_donated = true;
```
Because `transfer()` never returns `Err`, the `?` never short-circuits and `has_donated = true` is always written.

**`has_donated` permanently blocks all recovery paths**

- `claim_neurons` (lines 54–56): returns `Err("Account has previously donated its funds")` if `has_donated`.
- `transfer` (lines 177–178): returns `Err("Account has already donated its funds")` if `has_donated`.

**No admin recovery path**

The DID (`rs/nns/gtc/canister/gtc.did` lines 38–47) exposes no method to retry `failed_transferred_neurons`. The failed neurons remain in governance under GTC canister control, inaccessible to anyone.

## Impact Explanation
Any GTC account whose `donate_account` call experiences even a single per-neuron governance failure ends with `has_donated = true`, an empty `neuron_ids`, and failed neurons permanently locked in governance under GTC control. The ICP staked in those neurons is permanently frozen — neither the original owner nor any custodian can ever access it. This matches the allowed High impact: "Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds" / significant NNS security impact with concrete user or protocol harm. Severity: **High**.

## Likelihood Explanation
The trigger is a transient inter-canister call failure during `donate_account`. Realistic causes include a ledger reject for one neuron, governance being mid-upgrade when the Nth sequential call is dispatched, or message queue saturation. SR accounts have 48 neurons and ECT accounts have 12, meaning 48 or 12 sequential inter-canister calls are made; the probability of at least one transient failure across that sequence is non-negligible. No special attacker privilege is required — the account owner simply calls `donate_account` and a transient environmental failure does the rest.

## Recommendation
1. `transfer()` should return `Err` (or a partial-failure signal) if any neuron fails, so `donate_account` can propagate the error instead of setting `has_donated = true`.
2. `donate_account` should only set `has_donated = true` when `failed_transferred_neurons` is empty after the loop.
3. Alternatively, keep failed neurons in `neuron_ids` so a retry is possible, and only set `has_donated` once all neurons have been successfully transferred.
4. Add a privileged admin method to retry `failed_transferred_neurons` for a given account.

## Proof of Concept
```
1. Create a GTC account with neurons [N1, N2, N3].
2. Mock governance to succeed for N1, fail for N2, succeed for N3.
3. Call donate_account.
4. transfer() loop:
     N1 → Ok  → successfully_transferred_neurons, removed from neuron_ids
     N2 → Err → failed_transferred_neurons,       removed from neuron_ids
     N3 → Ok  → successfully_transferred_neurons, removed from neuron_ids
   transfer() returns Ok(()).
5. donate_account sets has_donated = true.
6. N2 still exists in governance under GTC control.
7. Call claim_neurons  → Err("Account has previously donated its funds")
8. Call donate_account → transfer() returns Err("Account has already donated its funds")
9. N2's ICP is permanently inaccessible.
```
A deterministic unit test in `rs/nns/gtc/src/lib.rs` can reproduce this by injecting a mock `GovernanceCanister::transfer_gtc_neuron` that fails on the second call, then asserting `failed_transferred_neurons` is non-empty and all subsequent calls return the `has_donated` error.
