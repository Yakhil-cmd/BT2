### Title
Unsafe `u64`-to-`i64` Cast in `delta_gas` Computation Corrupts Refund Accounting - (File: `basic_bootloader/src/bootloader/transaction_flow/refund_calculation.rs`)

---

### Summary

`compute_gas_refund` casts two `u64` resource values to `i64` without bounds validation. For L1 (priority) transactions — which are explicitly exempt from the `MAX_BLOCK_GAS_LIMIT` guard applied to L2 transactions — `gas_used` and `native_used / native_per_gas` can exceed `i64::MAX`, causing a sign-flip that corrupts the `delta_gas` adjustment and produces either an inflated gas refund (operator underpaid) or an arithmetic overflow that halts block processing.

---

### Finding Description

In `compute_gas_refund`, the block guarded by `#[cfg(not(feature = "unlimited_native"))]` computes:

```rust
// basic_bootloader/src/bootloader/transaction_flow/refund_calculation.rs, line 69-79
let delta_gas = if native_per_gas == 0 {
    0
} else {
    (native_used / native_per_gas) as i64 - (gas_used as i64)
};

if delta_gas > 0 {
    gas_used += delta_gas as u64;
}
```

Both operands are `u64` values cast to `i64` with no range check. [1](#0-0) 

`gas_used` is derived directly from `gas_limit`:

```rust
let mut gas_used = gas_limit
    .checked_sub(resources.ergs().0.div_floor(ERGS_PER_GAS))
    .ok_or(internal_error!("gas remaining > gas limit"))?;
``` [2](#0-1) 

For **L2 transactions**, `gas_limit` is bounded before reaching this function:

```rust
require!(
    tx_gas_limit.saturating_mul(ERGS_PER_GAS) < u64::MAX,
    InvalidTransaction::CallerGasLimitTooHigh,
    system
)?;
``` [3](#0-2) 

This enforces `gas_limit < u64::MAX / ERGS_PER_GAS = MAX_BLOCK_GAS_LIMIT`. [4](#0-3) 

For **L1 transactions**, no equivalent bound is enforced. The `prepare_and_check_resources` function explicitly uses saturating arithmetic and logs warnings instead of rejecting the transaction, because L1 transactions cannot be invalidated:

```rust
// L1 transactions cannot be invalidated. Therefore, the following
// function makes sure L1 transactions are processable even when
// some checks that should be performed by the L1 don't hold.
``` [5](#0-4) 

The `gas_limit` field is read directly as a raw `u64` from the ABI-encoded transaction with no upper-bound check:

```rust
let gas_limit = transaction.gas_limit.read();
``` [6](#0-5) 

Similarly, `native_per_gas` for L1 transactions saturates to `u64::MAX` on overflow rather than rejecting:

```rust
u256_try_to_u64(&gas_price.div_ceil(native_price)).unwrap_or_else(|| {
    system_log!(..., "Native per gas calculation for L1 tx overflows, using saturated arithmetic instead");
    u64::MAX
})
``` [7](#0-6) 

The `as i64` cast at line 72 is the **only** place in the entire L1 transaction path that does not use saturating or checked arithmetic, making it an inconsistent and unsafe outlier.

---

### Impact Explanation

**Scenario A — Inflated refund / operator underpaid:**

Let `gas_limit = 2^63 + 1000` (just above `i64::MAX`), all gas consumed (`gas_used = 2^63 + 1000`), and `native_used / native_per_gas = 1000` (small, fits in `i64`).

```
(native_used / native_per_gas) as i64  =  1000          (positive, correct)
gas_used as i64                         = -(2^63 - 1000) (sign-flipped, wrong)
delta_gas = 1000 - (-(2^63 - 1000))    =  2^63           (large positive)
gas_used += 2^63 as u64                → u64 overflow
```

In release mode (wrapping), `gas_used` wraps to a small value. Then:

```rust
let total_gas_refund = gas_limit - gas_used;  // ≈ gas_limit (massive refund)
``` [8](#0-7) 

The operator receives almost nothing; the depositor is refunded nearly the entire deposit.

**Scenario B — Block halt (debug/checked builds):**

The `gas_used += delta_gas as u64` addition overflows `u64`, causing a panic in debug mode or a checked-arithmetic build, halting block processing entirely.

---

### Likelihood Explanation

L1 (priority) transactions are submitted through the L1 bridge contract, which is expected to validate `gas_limit`. However:

1. The ZKsync OS codebase **explicitly acknowledges** that L1 validation may be wrong and uses saturating arithmetic throughout `prepare_and_check_resources` as a defensive fallback — except at this cast.
2. The `gas_limit` field is a `u64` in the ABI-encoded format, so values above `i64::MAX` are structurally valid inputs.
3. A misconfigured or outdated L1 bridge, or a direct submission bypassing bridge validation, can produce such a transaction.
4. The `#[cfg(not(feature = "unlimited_native"))]` guard means this code is active in production (proving mode), where `unlimited_native` is not set.

---

### Recommendation

Replace the bare `as i64` casts with explicit bounds-checked conversions, consistent with the saturating pattern used everywhere else in the L1 transaction path:

```rust
let native_gas_equiv = (native_used / native_per_gas).min(i64::MAX as u64) as i64;
let gas_used_signed = gas_used.min(i64::MAX as u64) as i64;
let delta_gas = native_gas_equiv - gas_used_signed;
```

Alternatively, perform the comparison in `u64` space entirely to avoid signed arithmetic:

```rust
let native_gas_equiv = native_used / native_per_gas;
if native_gas_equiv > gas_used {
    gas_used = gas_used.saturating_add(native_gas_equiv - gas_used);
}
```

### Proof of Concept

Analogous to the Chisel demonstration in the original report:

```
// Rust equivalent
let gas_used: u64 = (i64::MAX as u64) + 1000;  // 9_223_372_036_854_776_807
let native_gas_equiv: u64 = 1000;

let delta_gas: i64 = (native_gas_equiv as i64) - (gas_used as i64);
// native_gas_equiv as i64 = 1000
// gas_used as i64          = -9_223_372_036_854_774_809  (sign-flipped!)
// delta_gas                = 1000 - (-9_223_372_036_854_774_809)
//                          = 9_223_372_036_854_775_809   (> i64::MAX, wraps to i64::MIN+1)
// Actually: delta_gas = i64::MIN + 1 (overflow in i64 arithmetic)
// In practice with wrapping: gas_used += delta_gas as u64 overflows u64
```

The root cause is identical to the original report: casting an unsigned integer to a signed type of the same width without validating the value is within the signed range, causing a sign-flip for values ≥ 2^63. [9](#0-8) [6](#0-5) [10](#0-9)

```rust
fn prepare_and_check_resources<
    'a,
    S: EthereumLikeTypes + 'a,
    Config: BasicBootloaderExecutionConfig,
>(
    system: &mut System<S>,
    transaction: &AbiEncodedTransaction<S::Allocator>,
    is_priority_op: bool,
    gas_limit: u64,
    gas_price: U256,
    gas_per_pubdata: u32,
    intrinsic_pubdata: u64,
) -> Result<ResourceAndFeeInfo<S>, BootloaderSubsystemError>
where
    S::IO: IOSubsystemExt,
    S::Metadata: ZkSpecificPricingMetadata
        + BasicMetadata<S::IOTypes, TransactionMetadata = TxLevelMetadata<S::IOTypes>>,
{
    // For L1->L2 txs, we use a constant native price to avoid censorship.
    let native_price = L1_TX_NATIVE_PRICE;
    let native_per_gas = if is_priority_op {
        if gas_price.is_zero() {
            if Config::SIMULATION {
                u256_try_to_u64(&system.get_eip1559_basefee().div_ceil(native_price))
                    .unwrap_or_else(|| {
                        system_log!(
                            system,
                            "Native per gas calculation for L1 tx overflows, using saturated arithmetic instead");
                        u64::MAX
                    })
            } else {
                FREE_L1_TX_NATIVE_PER_GAS
            }
        } else {
            u256_try_to_u64(&gas_price.div_ceil(native_price)).unwrap_or_else(|| {
                system_log!(
                    system,
                    "Native per gas calculation for L1 tx overflows, using saturated arithmetic instead");
                u64::MAX
            })
        }
    } else {
        // Upgrade txs are paid by the protocol, so we use a fixed native per gas
        FREE_L1_TX_NATIVE_PER_GAS
    };

    let native_per_pubdata = (gas_per_pubdata as u64)
        .checked_mul(native_per_gas)
        .unwrap_or_else(|| {
            system_log!(
                system,
                "Native per pubdata calculation for L1 tx overflows, using saturated arithmetic instead");
                u64::MAX
        });

    let native_prepaid_from_gas = native_per_gas.checked_mul(gas_limit)
        .unwrap_or_else(|| {
            system_log!(
                system,
                "Native prepaid from gas calculation for L1 tx overflows, using saturated arithmetic instead");
                u64::MAX
        });

    let (calldata_tokens, minimal_gas_used) =
        compute_calldata_tokens(system, transaction.calldata());

    // L1 transactions never carry an access list or authorization list, so
    // the corresponding counts are 0 and the intrinsic-gas helper collapses
    // to TX_INTRINSIC_GAS + calldata-token cost (deployment is also false).
    let intrinsic_gas = calculate_tx_intrinsic_gas(
        transaction.calldata().len() as u64,
        calldata_tokens,
        false, // is_deployment
        0,     // access_list_accounts
        0,     // access_list_storage_keys
        0,     // authorization_list_num
    );
    let intrinsic_computational_native = calculate_l1_tx_intrinsic_computational_native_resources(
        transaction.calldata().len() as u64,
    );
    // With L1ResourcesPolicy, this returns Result<ResourcesForTx<S>, BootloaderSubsystemError>
    // Validation errors are type-safe impossible - they're logged and saturated instead
    let resources = create_resources_for_tx::<S, L1ResourcesPolicy>(
        system,
        gas_limit,
        native_per_gas == 0,
        native_prepaid_from_gas,
        native_per_pubdata,
        intrinsic_gas,
        intrinsic_computational_native,
        intrinsic_pubdata,
    )?;

    // L1 transactions might have a gas limit < minimal_gas_used. This should be
    // prevented by L1 validation, but we log and saturate if it happens.
    if gas_limit < minimal_gas_used {
        system_log!(
            system,
            "L1 tx gas limit below intrinsic cost, using saturated arithmetic instead"
        );
    }
    // Pick the min to keep processing L1 txs even if the L1 validation is wrong.
    let minimal_gas_used = minimal_gas_used.min(gas_limit);

    Ok(ResourceAndFeeInfo {
        resources,
        native_per_pubdata,
        native_per_gas,
        minimal_gas_used,
    })
}
