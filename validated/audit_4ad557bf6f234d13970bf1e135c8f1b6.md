### Title
Unprivileged User Can Permanently Block Admin Token-Mapping Registration via Pre-emptive Auto-Deployment - (`x/cronos/keeper/evm.go`, `x/cronos/keeper/keeper.go`)

### Summary

When `EnableAutoDeployment=true`, any unprivileged user holding IBC vouchers can call `MsgConvertVouchers` to trigger auto-deployment of a `ModuleCRC21` contract for a new denom. Because `SetAutoContractForDenom` and `SetExternalContractForDenom` both call `ensureDenomNotMapped`, which rejects any denom that already resolves to a contract, the admin's subsequent `MsgUpdateTokenMapping` for that same denom will permanently fail with `ErrDenomAlreadyMapped`. There is no `DeleteAutoContractForDenom` function, so the state is irreversible without a chain upgrade.

### Finding Description

`ConvertCoinFromNativeToCRC21` in `x/cronos/keeper/evm.go` deploys a new `ModuleCRC21` contract and registers it via `SetAutoContractForDenom` when no mapping exists for a denom and `autoDeploy=true`: [1](#0-0) 

`SetAutoContractForDenom` enforces a create-only constraint via `ensureDenomNotMapped`: [2](#0-1) 

`ensureDenomNotMapped` checks both external and auto contract slots: [3](#0-2) 

The admin path `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom` also calls `ensureDenomNotMapped` at the top: [4](#0-3) 

Once an auto contract is registered for a denom, `GetContractByDenom` returns `found=true` for that denom: [5](#0-4) 

There is no `DeleteAutoContractForDenom`. The spec explicitly states "There's no way to delete a token mapping currently": [6](#0-5) 

The ADR-008 documents this as an intentional create-only guardrail, but does not account for the adversarial pre-emption scenario: [7](#0-6) 

### Impact Explanation

An unprivileged attacker who pre-empts the admin by calling `MsgConvertVouchers` for a target denom permanently prevents the admin from registering any external contract for that denom. The admin's `MsgUpdateTokenMapping` will always return `ErrDenomAlreadyMapped`: [8](#0-7) 

This is a permanent bypass of Cronos admin token-mapping authority. The denom is locked to the auto-deployed `ModuleCRC21` contract indefinitely. If the admin intended to register a custom external contract (e.g., one with different decimal handling, additional bridge support, or pre-existing liquidity), that is permanently blocked without a chain upgrade.

### Likelihood Explanation

The attack requires:
1. `EnableAutoDeployment=true` (a governance-enabled production configuration)
2. The attacker holds any nonzero amount of the target IBC denom (obtainable by receiving an IBC transfer)
3. The attacker submits `MsgConvertVouchers` before the admin submits `MsgUpdateTokenMapping`

`MsgConvertVouchers` is open to any address: [9](#0-8) 

The attack is front-runnable: an attacker monitoring the mempool for a pending `MsgUpdateTokenMapping` can race it with a `MsgConvertVouchers` for the same denom. It can also happen accidentally when a user converts tokens before the admin has a chance to register the intended external contract.

### Recommendation

1. Allow `SetExternalContractForDenom` to override an existing auto-deployed contract (i.e., `ensureDenomNotMapped` should only block re-registration of *external* contracts, not auto contracts).
2. Alternatively, add a `DeleteAutoContractForDenom` admin function so the admin can clear a stale auto mapping before registering an external one.
3. Or, in `ConvertCoinFromNativeToCRC21`, check whether an external contract mapping already exists (not just any mapping) before triggering auto-deployment, so that a pending admin registration is not pre-empted.

### Proof of Concept

1. `EnableAutoDeployment=true` is set in chain params.
2. Admin plans to call `MsgUpdateTokenMapping{Denom: "ibc/XXXX", Contract: "0xCustom..."}`.
3. Attacker receives 1 unit of `ibc/XXXX` via IBC and calls `MsgConvertVouchers{Address: attacker, Coins: [{Denom: "ibc/XXXX", Amount: 1}]}`.
4. `ConvertCoinFromNativeToCRC21` finds no mapping, deploys a `ModuleCRC21` contract at address `0xAuto...`, and calls `SetAutoContractForDenom("ibc/XXXX", 0xAuto...)` — succeeds.
5. Admin submits `MsgUpdateTokenMapping{Denom: "ibc/XXXX", Contract: "0xCustom..."}`.
6. `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom` → `ensureDenomNotMapped` → `GetContractByDenom("ibc/XXXX")` returns `(0xAuto..., true)` → returns `ErrDenomAlreadyMapped`.
7. Admin's registration fails permanently. No recovery path exists without a chain upgrade. [10](#0-9) [11](#0-10)

func (k Keeper) ConvertCoinFromNativeToCRC21(ctx sdk.Context, sender common.Address, coin sdk.Coin, autoDeploy bool) error {
	if !types.IsValidCoinDenom(coin.Denom) {
		return fmt.Errorf("coin %s is not supported for conversion", coin.Denom)
	}
	var err error
	// external contract is returned in preference to auto-deployed ones
	contract, found := k.GetContractByDenom(ctx, coin.Denom)
	if !found {
		if !autoDeploy {
			return fmt.Errorf("no contract found for the denom %s", coin.Denom)
		}
		contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
		if err != nil {
			return err
		}
		if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
			return err
		}

		k.Logger(ctx).Info(fmt.Sprintf("contract address %s created for coin denom %s", contract.String(), coin.Denom))
	}

func (k Keeper) ensureDenomNotMapped(ctx sdk.Context, denom string) error {
	if contract, found := k.GetContractByDenom(ctx, denom); found {
		return errors.Wrapf(
			types.ErrDenomAlreadyMapped,
			"denom %s is already mapped to contract %s",
			denom,
			contract.Hex(),
		)
	}
	return nil
}

func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil

func (k Keeper) SetAutoContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	isSource := types.IsSourceCoin(denom)
	if err := validateContractAddressForSourceDenom(denom, address, isSource); err != nil {
		return err
	}
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}
	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToAutoContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil

func (k Keeper) RegisterOrUpdateTokenMapping(ctx sdk.Context, msg *types.MsgUpdateTokenMapping) error {
	if types.IsSourceCoin(msg.Denom) {
		_, err := types.GetContractAddressFromDenom(msg.Denom)
		if err != nil {
			return err
		}

		if !common.IsHexAddress(msg.Contract) {
			return errors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid contract address (%s)", msg.Contract)
		}
		contract := common.HexToAddress(msg.Contract)
		if err := k.ensureContractCode(ctx, contract); err != nil {
			return err
		}
		if err := validateContractAddressForSourceDenom(msg.Denom, contract, true); err != nil {
			return err
		}

		if err := k.SetExternalContractForDenom(ctx, msg.Denom, contract); err != nil {
			return err
		}

func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
