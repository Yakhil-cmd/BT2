### Title
Missing Threshold Lower Bound Validation in FROST Signing Corrupts Signing Outputs - (File: `src/frost/mod.rs`)

### Summary
Both `presign()` and `assert_sign_inputs()` in `src/frost/mod.rs` enforce only the upper bound on the threshold (`threshold <= participants.len()`) but omit the lower bound check (`threshold >= 2`) that the DKG enforces in `assert_key_invariants`. A caller or malicious coordinator can supply `threshold = 1` to FROST signing while the key was generated under `threshold = 2`, causing Lagrange interpolation to use incorrect weights and producing a signature that fails to verify.

### Finding Description
In `src/frost/mod.rs`, the `presign()` function validates:

```rust
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [1](#0-0) 

Similarly, `assert_sign_inputs()` performs the same one-sided check:

```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
