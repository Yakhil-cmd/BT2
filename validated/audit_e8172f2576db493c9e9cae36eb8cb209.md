### Title
Unauthorized Execution of `exit_delegation_pool_action` on Behalf of a Contract Pool Member Can Permanently Freeze Delegated Funds - (File: `src/pool/pool.cairo`)

### Summary
`exit_delegation_pool_action` in `src/pool/pool.cairo` accepts a `pool_member: ContractAddress` argument and transfers the full `unpool_amount` of delegation tokens directly to that address with no caller restriction. Any unprivileged address can trigger this transfer at any time after the exit window elapses. If the pool member is a smart contract whose internal flow depends on calling `exit_delegation_pool_action` itself (e.g., to atomically receive and re-route funds), a malicious actor can front-run that call, deliver the tokens to the contract prematurely, and permanently freeze them if the contract has no rescue path.

An identical pattern exists in `unstake_action` in `src/staking/staking.cairo`, which also carries "Any address can execute" access control and transfers the staker's principal to `staker_address`.

---

### Finding Description

`exit_delegation_pool_action` contains no `get_caller_address()` check:

```
// src/pool/pool.cairo  lines 295-333
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    ...
    // Transfer delegated amount to the pool member.
    let token_dispatcher = self.token_dispatcher.read();
    token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
    unpool_amount
}
```

The spec explicitly documents this as intentional:

> **Access control:** Any address can execute. [1](#0-0) [2](#0-1) 

The only guards are:
1. `pool_member` must have a registered `unpool_time` (set by the pool member itself via `exit_delegation_pool_intent`).
2. The current timestamp must be ≥ `unpool_time`.

Neither guard restricts *who* triggers the transfer. Once the exit window elapses, any external account can push the tokens into `pool_member`.

The same pattern applies to `unstake_action` in `src/staking/staking.cairo` (lines 483–514), which transfers the staker's principal to `staker_address` with no caller check. [3](#0-2) [4](#0-3) 

---

### Impact Explanation

If the pool member is a smart contract (e.g., a DAO treasury, a DeFi vault, or a multisig) that:
- delegates tokens to a pool,
- calls `exit_delegation_pool_intent` to begin withdrawal, and
- expects to call `exit_delegation_pool_action` itself as part of a larger atomic transaction (e.g., receive tokens → immediately re-stake or forward them),

then a malicious actor can call `exit_delegation_pool_action(pool_member: <contract>)` the moment the exit window expires, delivering the tokens to the contract outside of its expected execution context. If the contract has no rescue/sweep function, the tokens are permanently frozen inside it.

This matches the **Permanent freezing of funds** impact category (High). [5](#0-4) 

---

### Likelihood Explanation

- Any smart contract that participates as a pool member is exposed.
- The attacker requires no special privilege: they only need to monitor the chain for `PoolMemberExitIntent` events and call `exit_delegation_pool_action` as soon as `Time::now() >= unpool_time`.
- No capital is required; the attacker pays only gas.
- The attack is deterministic and repeatable. [6](#0-5) 

---

### Recommendation

Restrict `exit_delegation_pool_action` so that only the pool member itself (or an explicitly authorized address stored in `pool_member_info`) can trigger the transfer:

```cairo
let caller = get_caller_address();
assert!(
    caller == pool_member,
    "{}",
    Error::POOL_EXIT_ACTION_FROM_UNAUTHORIZED_ADDRESS,
);
```

Apply the same fix to `unstake_action` in `src/staking/staking.cairo`. [7](#0-6) 

---

### Proof of Concept

1. Deploy `VulnerablePoolMember`, a contract that:
   - holds delegation tokens,
   - calls `pool.enter_delegation_pool(amount)`,
   - calls `pool.exit_delegation_pool_intent(amount)` to begin withdrawal,
   - expects to call `pool.exit_delegation_pool_action(self_address)` itself in a subsequent transaction that atomically re-routes the returned tokens.

2. Attacker monitors the chain for the `PoolMemberExitIntent` event emitted by `VulnerablePoolMember`.

3. Once `Time::now() >= unpool_time`, attacker calls:
   ```
   pool.exit_delegation_pool_action(pool_member: VulnerablePoolMember.address)
   ```

4. `exit_delegation_pool_action` passes all checks (pool member exists, intent set, window elapsed) and executes:
   ```cairo
   token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
   ```
   transferring the full delegated amount to `VulnerablePoolMember`.

5. `VulnerablePoolMember` has no `receive` hook or rescue function. The tokens are permanently locked.

6. When `VulnerablePoolMember` later tries to call `exit_delegation_pool_action` itself, it reverts with `MISSING_UNDELEGATE_INTENT` because `unpool_time` was already cleared in step 4. [8](#0-7) [9](#0-8)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
