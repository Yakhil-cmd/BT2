### Title
Permissionless `exit_delegation_pool_action` Enables Griefing of Pool Members Attempting to Switch Pools - (File: src/pool/pool.cairo)

### Summary
`Pool.exit_delegation_pool_action` accepts a `pool_member` address as a parameter and executes the exit for that member with no check on `get_caller_address()`. Any unprivileged caller can trigger the exit for any pool member once the exit window has elapsed, forcibly clearing the member's `unpool_time` and `unpool_amount` and transferring their principal back to them. This destroys the precondition required by `switch_delegation_pool` (`unpool_time.is_some()`), permanently blocking the pool member from completing an in-flight pool switch without re-entering from scratch.

### Finding Description
`exit_delegation_pool_action` in `src/pool/pool.cairo` takes `pool_member: ContractAddress` as a caller-supplied argument and performs no `get_caller_address()` equality check:

```rust
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    // ... clears unpool_time / unpool_amount, transfers funds to pool_member
``` [1](#0-0) 

The spec explicitly documents this as "Any address can execute": [2](#0-1) 

`switch_delegation_pool`, by contrast, requires the caller to be the pool member themselves AND requires `unpool_time.is_some()`:

```rust
fn switch_delegation_pool(...) {
    let pool_member = get_caller_address();
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(
        pool_member_info.unpool_time.is_some(),
        "{}",
        GenericError::MISSING_UNDELEGATE_INTENT,
    );
``` [3](#0-2) 

Once an attacker calls `exit_delegation_pool_action` for a victim, `unpool_time` is set to `None` and `unpool_amount` is zeroed: [4](#0-3) 

The victim can no longer call `switch_delegation_pool` and must re-enter the pool from scratch, waiting an additional K epochs before their balance becomes reward-eligible.

### Impact Explanation
**Medium — Griefing with no profit motive but damage to users.**

A pool member who has signaled exit intent and is waiting to call `switch_delegation_pool` (to move to a lower-commission staker) can be front-run by any address once the exit window expires. The attacker forces the principal back to the victim's address and clears the exit intent. The victim:
- Loses the ability to switch pools atomically.
- Must re-enter the pool and wait K epochs before their balance earns rewards again, forfeiting K epochs of yield.
- Incurs extra gas costs.

Unclaimed rewards accrued before the forced exit are preserved in storage and remain claimable, so there is no direct theft of already-accrued yield. The harm is the loss of future yield during the mandatory K-epoch re-entry delay and the disruption of the pool-switching flow.

### Likelihood Explanation
**Medium.** The attack requires no special privilege — any EOA or contract can call `exit_delegation_pool_action` with an arbitrary `pool_member` address once the exit window has elapsed. The attacker gains nothing financially, but the attack is cheap (a single transaction) and can be repeated every time the victim re-enters and re-signals exit intent, making it a persistent griefing vector against any pool member attempting to switch pools.

### Recommendation
Add a caller restriction to `exit_delegation_pool_action` so that only the pool member themselves (or optionally their registered `reward_address`) can trigger the exit action:

```rust
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let caller = get_caller_address();
    let pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(
        caller == pool_member || caller == pool_member_info.reward_address,
        "{}",
        Error::CALLER_CANNOT_EXECUTE_EXIT_ACTION,
    );
    // ... rest of function
```

This mirrors the access-control pattern already applied to `add_to_delegation_pool` and `claim_rewards`. [5](#0-4) [6](#0-5) 

### Proof of Concept

1. Alice (pool member) calls `exit_delegation_pool_intent(amount)` on her pool.
2. The exit window elapses (e.g., 21 days).
3. Alice is waiting to call `switch_delegation_pool(to_staker, to_pool, amount)` to move to a lower-commission staker.
4. Bob (attacker, any address) calls `exit_delegation_pool_action(pool_member: Alice)`.
   - No caller check exists; the only guards are that Alice has an intent and the window has passed — both true.
   - Alice's `unpool_amount` is transferred to her address; `unpool_time` and `unpool_amount` are cleared.
5. Alice attempts `switch_delegation_pool(...)` — it reverts with `MISSING_UNDELEGATE_INTENT` because `unpool_time` is now `None`.
6. Alice must call `enter_delegation_pool` again and wait K epochs before her balance earns rewards, losing K epochs of yield.
7. Bob can repeat step 4 every time Alice re-enters and re-signals exit intent. [1](#0-0) [7](#0-6) [8](#0-7)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
