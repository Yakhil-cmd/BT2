### Title
Gateway Stateful Validator Uses Stale Previous-Block L2 Gas Price for Admission, Causing Transactions to Pass Gateway but Fail at Blockifier Pre-Validation - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **previous block's** L2 gas price, while `AccountTransaction::check_fee_bounds` in the blockifier enforces the **current (next) block's** L2 gas price at execution time. When the EIP-1559-style price mechanism raises the L2 gas price between blocks, transactions whose `max_price_per_unit` sits between the previous and current block prices pass gateway admission but are rejected by the blockifier's pre-validation stage, producing a systematic mismatch between what the gateway admits and what the sequencer can actually execute.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, `validate_resource_bounds` reads the **previous** block's price and uses it as the admission threshold:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`validate_tx_l2_gas_price_within_threshold` then computes `threshold = previous_block_price * min_gas_price_percentage / 100` and admits the transaction if `tx_l2_gas_price >= threshold`: [2](#0-1) 

The production deployment sets `min_gas_price_percentage = 100`, so the threshold equals the previous block's price exactly: [3](#0-2) 

Meanwhile, `AccountTransaction::check_fee_bounds` in the blockifier enforces the **current block's** gas price:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { ... }
    );
}
``` [4](#0-3) 

This check is called unconditionally from `perform_pre_validation_stage` when `charge_fee` is true: [5](#0-4) 

The L2 gas price is updated each block via `calculate_next_l2_gas_price` / `calculate_next_base_gas_price`. When gas usage exceeds the target, the price rises by up to `price / 333` per block: [6](#0-5) 

The developers themselves flag this as a known defect with a TODO: [7](#0-6) 

### Impact Explanation

This matches the **High** impact criterion: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

A transaction with `max_price_per_unit = P_prev` (the previous block's price) passes the gateway's threshold check (`P_prev >= P_prev * 100/100`). When the batcher builds the next block at price `P_next > P_prev`, the blockifier's `check_fee_bounds` rejects the transaction with `MaxGasPriceTooLow`. The transaction was admitted to the mempool under a false premise: the gateway certified it as fee-sufficient for the upcoming block, but it is not. The mempool wastes capacity on transactions that cannot be sequenced, and users whose wallets set `max_price_per_unit` to exactly the last observed block price receive unexpected execution failures.

### Likelihood Explanation

The EIP-1559 mechanism raises the price whenever block gas usage exceeds the target. Under normal load this happens regularly. With `min_gas_price_percentage = 100` (the production default), any transaction priced at exactly the previous block's price is vulnerable. The price increase per block is bounded (`price / 333 ≈ 0.3%`), so the window is narrow but persistent across every block where usage is above target. Any unprivileged user submitting a transaction priced at the previous block's exact price triggers this path.

### Recommendation

Replace `get_block_info()` (which returns the **previous** committed block) with the **next** block's projected L2 gas price, as the TODO comment already acknowledges. The `calculate_next_l2_gas_price` result is available in `SequencerConsensusContext` and is passed to the batcher via `propose_block_input.block_info.gas_prices`. The gateway's `GatewayFixedBlockStateReader` should expose the next block's gas price so that `validate_resource_bounds` can use it as the admission threshold, ensuring the gateway and blockifier enforce the same price invariant.

### Proof of Concept

```
Block N:   L2 gas price = 1_000_000_000 fri  (previous block)
Block N+1: L2 gas price = 1_003_003_003 fri  (next block, usage > target)

User submits Invoke V3 with:
  max_price_per_unit (L2) = 1_000_000_000 fri

Gateway stateful check (validate_resource_bounds):
  threshold = 1_000_000_000 * 100 / 100 = 1_000_000_000
  1_000_000_000 >= 1_000_000_000  → ADMITTED to mempool

Blockifier check_fee_bounds (block N+1):
  actual_gas_price = 1_003_003_003
  1_000_000_000 < 1_003_003_003   → REJECTED: MaxGasPriceTooLow

Result: transaction admitted by gateway, rejected by blockifier.
        Mempool slot consumed; user transaction silently dropped from block.
```

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    // Performs static checks before executing validation entry point.
    // Note that nonce is incremented during these checks.
    pub fn perform_pre_validation_stage<S: State + StateReader>(
        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
