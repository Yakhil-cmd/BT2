### Title
Gateway Unconditionally Skips `__validate__` Signature Verification for Invoke Transactions with Nonce=1 When Account Is In-Flight — (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful transaction validator unconditionally skips the `__validate__` entry-point call (the account's signature verification) for any invoke transaction whose nonce equals `1` when the sender account has any transaction in the mempool or a recent block. An unprivileged attacker who controls a freshly-deployed account can exploit this to inject an invoke transaction carrying an arbitrary (invalid) signature into the mempool, bypassing the admission invariant that every accepted transaction has passed account-level signature verification.

### Finding Description

`extract_state_nonce_and_run_validations` calls `run_pre_validation_checks`, which in turn calls `skip_stateful_validations`:

```rust

The three conditions that trigger the skip are:
1. Transaction type is `Invoke`
2. Transaction nonce is exactly `Nonce(Felt::ONE)`
3. `account_tx_in_pool_or_recent_block(sender)` returns `true`

Condition 3 is satisfied as soon as **any** transaction from the sender address is in the mempool — including a `DeployAccount` the attacker themselves submitted. The check does **not** verify that the in-flight transaction is a `DeployAccount`, nor does it verify that the invoke's signature is well-formed.

**Attack path (unprivileged):**

1. Attacker generates a fresh key-pair and constructs a `DeployAccount` for address `A` using any account class (e.g., the standard OpenZeppelin account). The `DeployAccount` carries a valid signature and passes full stateful validation.
2. Attacker submits the `DeployAccount` to the gateway. It is admitted to the mempool; `account_tx_in_pool_or_recent_block(A)` now returns `true`.
3. Attacker immediately submits an `Invoke` from address `A` with `nonce = 1` and a **garbage / intentionally invalid signature** (e.g., all-zero felts).
4. Gateway path for the invoke:
   - Stateless checks pass (resource bounds, calldata length, DA modes, etc.).
   - `validate_state_preconditions`: nonce `1 ≥ account_nonce 0` and `1 ≤ 0 + max_allowed_nonce_gap(200)` → passes.
   - `validate_by_mempool`: duplicate/nonce checks pass (no signature check here).
   - `skip_stateful_validations`: nonce==1, account_nonce==0, `account_tx_in_pool_or_recent_block(A)==true` → returns `true`.
   - `run_validate_entry_point(skip_validate=true)`: `__validate__` is **never called**.
5. The invoke with the invalid signature is forwarded to the mempool and accepted. [4](#0-3) 

### Impact Explanation

The broken invariant is: *every transaction admitted to the mempool has passed account-level signature verification*. An invoke with an arbitrary signature is accepted at the gateway/mempool admission boundary. When the batcher later executes the block, `__validate__` is called and the transaction fails, wasting sequencer execution resources and polluting the mempool with transactions that are guaranteed to revert. Because the attacker only needs to pay for one `DeployAccount` per injected invalid invoke (and can create arbitrarily many fresh accounts), this constitutes a low-cost, unprivileged mempool-admission bypass.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

The trigger is fully unprivileged: any external user can submit a `DeployAccount` + `Invoke(nonce=1, bad_sig)` pair. No special role, no admin key, no privileged RPC endpoint is required. The only cost is the gas for the `DeployAccount` transaction. The condition is reliably reproducible on every node running the production gateway configuration (`max_nonce_for_validation_skip = 0x1`, `max_allowed_nonce_gap = 200`). [5](#0-4) 

### Recommendation

1. **Narrow the skip condition**: instead of checking only that *any* transaction from the sender is in the mempool, verify that a `DeployAccount` transaction specifically is present for the sender address. Add a dedicated mempool query (e.g., `deploy_account_in_pool(address)`) that returns `true` only when a `DeployAccount` — not an arbitrary transaction — is queued.

2. **Alternatively, perform lightweight signature pre-screening**: even without a full `__validate__` execution, the gateway could verify that the signature field is structurally non-trivial (e.g., correct length, non-zero) before skipping the full entry-point call.

3. **Align with `PyValidator`**: the `PyValidator::should_run_stateful_validations` in `crates/native_blockifier/src/py_validator.rs` requires `deploy_account_tx_hash.is_some()` as an explicit caller-supplied proof that a deploy-account was submitted. The gateway's `skip_stateful_validations` has no equivalent requirement and relies solely on the mempool membership check, which is weaker. [6](#0-5) 

### Proof of Concept

```
# 1. Generate fresh keypair → derive contract_address A
# 2. Submit DeployAccount(address=A, class_hash=<any valid class>, sig=<valid>)
#    → gateway admits it; mempool now contains A's deploy_account
# 3. Submit Invoke(sender=A, nonce=1, calldata=[...], sig=[0x0, 0x0])
#    → gateway stateless checks: pass
#    → validate_nonce: 0 <= 1 <= 200: pass
#    → validate_by_mempool: no dup, nonce ok: pass
#    → skip_stateful_validations: nonce==1 ∧ account_nonce==0
#                                  ∧ account_tx_in_pool_or_recent_block(A)==true
#                                  → returns true (SKIP)
#    → run_validate_entry_point(skip_validate=true): __validate__ NOT called
#    → Invoke with sig=[0,0] is forwarded to mempool ✓
# 4. Batcher picks up the invoke; __validate__ is called during execution → REVERT
#    Sequencer wasted execution resources on a provably-invalid transaction.
``` [7](#0-6) [8](#0-7)

        block_info.block_number = block_info.block_number.unchecked_next();

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

```rust
    pub fn perform_validations(&mut self, tx: AccountTransaction) -> StatefulValidatorResult<()> {
        // Deploy account transaction should be fully executed, since the constructor must run
        // before `__validate_deploy__`. The execution already includes all necessary validations,
        // so they are skipped here.
        // Declare transaction should also be fully executed - otherwise, if we only go through
        // the validate phase, we would miss the check that the class was not declared before.
        match tx.tx {
            ApiTransaction::DeployAccount(_) | ApiTransaction::Declare(_) => self.execute(tx),
            ApiTransaction::Invoke(_) => {
                let tx_context = Arc::new(self.tx_executor.block_context.to_tx_context(&tx));
                tx.perform_pre_validation_stage(self.state(), &tx_context)?;
                if !tx.execution_flags.validate {
                    return Ok(());
                }

                // `__validate__` call.
                let (_optional_call_info, actual_cost) = self.validate(&tx, tx_context.clone())?;

                // Post validations.
                PostValidationReport::verify(
                    &tx_context,
                    &actual_cost,
                    tx.execution_flags.charge_fee,
                )?;

                Ok(())
            }
        }
    }

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
