### Title
Missing Lower-Bound Validation on `threshold` in FROST Signing Entry Points Allows Corrupted Signature Output - (File: src/frost/mod.rs)

### Summary
`assert_sign_inputs` and `presign` in `src/frost/mod.rs` validate that `threshold` does not exceed the participant count, but never check that `threshold >= 2`. A caller supplying `threshold = 0` or `threshold = 1` passes all guards and enters the FROST signing protocol with a semantically invalid threshold, producing an unusable or cryptographically incorrect signature that honest parties accept as a completed protocol output.

### Finding Description
`assert_key_invariants` in `src/dkg.rs` enforces both bounds on the threshold:

```rust
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

The FROST signing entry points apply only the upper-bound check and omit the lower-bound check entirely.

`assert_sign_inputs`:
```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no check for threshold < 2
``` [2](#0-1) 

`presign` (FROST):
```rust
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no check for threshold < 2
``` [3](#0-2) 

`ReconstructionLowerBound` is a plain `usize` newtype with no invariant enforcement of its own:

```rust
pub struct ReconstructionLowerBound(usize);
impl ReconstructionLowerBound {
    pub fn value(self) -> usize { self.0 }
}
``` [4](#0-3) 

Any caller can construct `ReconstructionLowerBound(1)` or `ReconstructionLowerBound(0)` and pass it to `assert_sign_inputs` or `presign`. Both values satisfy `threshold.value() <= participants.len()` (since `participants.len() >= 2`), so no error is returned.

With `threshold = 1`, the FROST aggregation performs Lagrange interpolation over a single point, yielding a Lagrange coefficient of 1 and using one participant's raw signing share directly as the reconstructed nonce/response. The resulting signature is not valid for the public key that was generated under a higher threshold, because the secret was split into shares of degree `keygen_threshold - 1`; evaluating only one share at zero does not recover the secret. The protocol completes without error but emits an invalid, unusable signature.

With `threshold = 0`, the aggregation attempts to interpolate over zero points, which triggers `InvalidInterpolationArguments` deep inside `eval_exponent_interpolation` or `compute_lagrange_coefficient` — a mid-protocol crash rather than a clean initialization error. [5](#0-4) 

### Impact Explanation
Honest participants complete the FROST signing protocol and receive a result, but the output signature is either cryptographically invalid (threshold = 1) or the protocol aborts mid-execution with an internal assertion error (threshold = 0). In both cases the signing round is permanently disrupted for all honest parties. This maps to: **High — Corruption of sign outputs so honest parties accept inconsistent or unusable cryptographic outputs.**

### Likelihood Explanation
The entry points `assert_sign_inputs` and `presign` are public library API. Any unprivileged library caller who controls the `threshold` argument — including a malicious coordinator or a misconfigured application — can trigger this path with a single integer value. No cryptographic material or special privilege is required. The inconsistency with `assert_key_invariants` (which does enforce the lower bound) makes accidental triggering plausible as well.

### Recommendation
Add the same lower-bound guard that `assert_key_invariants` already enforces, in both `assert_sign_inputs` and `presign` in `src/frost/mod.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This should be placed immediately after the upper-bound check in each function, mirroring the pattern in `src/dkg.rs`. [6](#0-5) 

### Proof of Concept

```rust
use threshold_signatures::frost::assert_sign_inputs;
use threshold_signatures::participants::Participant;

let participants = vec![Participant::from(1u32), Participant::from(2u32)];

// threshold = 1 passes all current guards

assert!(result.is_ok()); // succeeds — no lower-bound check

// Downstream: FROST aggregation with threshold=1 uses a single raw share,
// producing a signature that is invalid for the public key generated
// under threshold >= 2, silently corrupting the signing output.
