### Title
Staker Can Front-Run Delegators via `set_commission_commitment` + `set_commission` to Steal All Yield — (`File: src/staking/staking.cairo`)

---

### Summary

A malicious staker can atomically call `set_commission_commitment` followed by `set_commission` to raise their commission to 100% in the same block, front-running a delegator's `enter_delegation_pool` transaction (or exploiting existing delegators). Because the commitment mechanism imposes no delay between setting the commitment and applying the new commission, the staker can silently redirect all delegator yield to themselves.

---

### Finding Description

The `set_commission_commitment` function allows a staker to register a `CommissionCommitment { max_commission, expiration_epoch }` that unlocks the ability to *increase* commission up to `max_commission`. [1](#0-0) 

Once an active commitment exists, `set_commission` (via `update_commission`) only checks:
1. `commission <= commission_commitment.max_commission`
2. `commission != old_commission` [2](#0-1) 

There is **no enforced delay** between setting the commitment and applying the new (higher) commission. The minimum `expiration_epoch` is `current_epoch + 1`, meaning the commitment is immediately active.

The code itself acknowledges this gap:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [3](#0-2) 

**Attack path (front-run on new delegator):**

1. Staker S advertises commission = 0%.
2. Delegator D submits `enter_delegation_pool` targeting S's pool.
3. S observes D's pending transaction and, in the same block with higher priority, submits:
   - `set_commission_commitment(max_commission=10000, expiration_epoch=current_epoch+1)`
   - `set_commission(commission=10000)` — raises commission to 100%
4. D's transaction executes; D is now in a pool with 100% commission.
5. All of D's future yield flows to S via the commission split.

**Attack path (existing delegators):**

A staker with existing delegators can execute the same two-transaction sequence at any time. Delegators who wish to exit must wait through the `exit_wait_window`, during which their stake continues to earn rewards — all of which are captured by the staker at 100% commission.

The `claim_rewards` function in the pool sends rewards to `reward_address` after deducting commission: [4](#0-3) 

Commission is read live from the staking contract at reward calculation time, so the raised commission immediately affects all pending and future reward distributions.

---

### Impact Explanation

**High — Theft of unclaimed yield / Temporary freezing of unclaimed yield.**

- A delegator who enters after the front-run earns 0 STRK rewards for as long as they remain in the pool.
- Existing delegators who do not immediately exit lose all yield accrued during the exit wait window (which can be days/weeks).
- The staker captures 100% of the pool's STRK reward stream.

This matches the allowed impact: *"High: Theft of unclaimed yield"* and *"High: Permanent/Temporary freezing of unclaimed yield."*

---

### Likelihood Explanation

**Low.** The attack requires a staker who is willing to act maliciously and sacrifice their long-term reputation and delegator base. However, the mechanism is fully permissionless — no privileged role, no leaked key, and no external dependency is required. Any registered staker can execute it at any time.

---

### Recommendation

Enforce a mandatory delay between setting a commission commitment and being allowed to apply a commission *increase*. For example, require that the commitment was set at least one full epoch before `set_commission` can raise the commission above the previous value. This gives delegators an observable on-chain signal and time to exit before the increase takes effect.

Alternatively, cap the maximum commission increase per epoch, or require that any commission increase only takes effect starting from the *next* epoch, not the current one.

---

### Proof of Concept

```
Epoch N, Block B:
  Staker S has commission = 0%, no active commitment.
  Delegator D submits enter_delegation_pool(pool=S_pool) at Block B.

  S front-runs D in Block B:
    TX1: staking.set_commission_commitment(max_commission=10000, expiration_epoch=N+1)
         → CommissionCommitment { max_commission: 10000, expiration_epoch: N+1 } stored.
         → Commitment is immediately active (expiration_epoch > current_epoch).

    TX2: staking.set_commission(commission=10000)
         → update_commission checks:
             commission (10000) <= max_commission (10000) ✓
             commission (10000) != old_commission (0)     ✓
         → Commission written as 10000 (100%).

  D's TX executes: D enters pool with commission = 100%.

Epoch N+1 onward:
  Staker attests → rewards distributed.
  Pool receives pool_rewards, but commission = 100% → delegator share = 0.
  D earns 0 STRK. All yield goes to S.
``` [5](#0-4) [6](#0-5)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
