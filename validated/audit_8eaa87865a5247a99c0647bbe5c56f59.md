Looking at the `update_rewards` function in the staking contract and comparing it against the spec's stated access control.

### Title
Missing Access Control on `update_rewards` Allows Anyone to Consume Block Reward Slot Without Distributing Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `StakingRewardsManagerImpl` is documented in the spec as restricted to "Only starkware sequencer," but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` to the current block without distributing any rewards. Because the function enforces a one-call-per-block invariant on this global variable, the legitimate sequencer call for that block will revert with `REWARDS_ALREADY_UPDATED`, causing all stakers to lose their block rewards.

### Finding Description

The spec at `docs/spec.md` lines 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1447–1507 has no such guard:

```cairo
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only checks pause + zero-address
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // Update last block rewards.  <-- written BEFORE the disable_rewards branch
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;   // exits without distributing any rewards
        }
        ...
    }
}
``` [1](#0-0) 

`general_prerequisites` is shared with every user-callable function (`stake`, `claim_rewards`, `unstake_intent`, etc.) and therefore cannot contain a sequencer-specific check. [2](#0-1) 

`last_reward_block` is a single global storage slot, not per-staker: [3](#0-2) 

The spec confirms the intended restriction: [4](#0-3) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in a block before the sequencer's legitimate call. The function:
1. Passes all staker-existence checks (any valid active staker address works)
2. Writes `current_block_number` into `last_reward_block`
3. Returns early due to `disable_rewards: true`, distributing **zero** rewards

The sequencer's subsequent call in the same block hits:
```
assert!(current_block_number > self.last_reward_block.read(), REWARDS_ALREADY_UPDATED)
```
and reverts. Every staker loses their block reward for that block. Repeated every block, this permanently freezes all unclaimed yield accrual.

This maps directly to the Zivoe analog: in Zivoe, calling `depositReward` with zero extends `periodFinish` and dilutes the reward rate; here, calling `update_rewards` with `disable_rewards: true` advances `last_reward_block` and voids the block's reward distribution entirely.

**Impact category**: High — Permanent freezing of unclaimed yield / Theft of unclaimed yield.

### Likelihood Explanation

- The function is publicly callable with no token deposit, no stake, and no special role required.
- The attacker only needs to know any valid active staker address (trivially obtained from on-chain events).
- The cost is a single Starknet transaction per block.
- On Starknet the sequencer is centralized, but user transactions can be submitted to the mempool and included before the sequencer's own system transaction if the sequencer does not enforce strict ordering of its reward-update calls.

### Recommendation

Add an explicit caller check at the top of `update_rewards`, analogous to the guard used in `update_current_epoch_block_rewards` in the reward supplier:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),   // or equivalent role
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

Alternatively, restrict via the existing roles system (`only_sequencer` or equivalent), consistent with how `update_current_epoch_block_rewards` restricts to the staking contract: [5](#0-4) 

### Proof of Concept

```
1. Deploy the system (staking contract, reward supplier, attestation).
2. Advance past `consensus_rewards_first_epoch` so rewards are active.
3. As an unprivileged EOA, call:
       staking.update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
4. Observe: `last_reward_block` is now set to the current block; no rewards were distributed.
5. The sequencer's legitimate call in the same block reverts with REWARDS_ALREADY_UPDATED.
6. Repeat step 3 every block; stakers accumulate zero rewards indefinitely.
```

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
