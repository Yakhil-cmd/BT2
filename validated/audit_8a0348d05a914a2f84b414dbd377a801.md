### Title
`BankContract.RequiredGas` panics on short input, enabling chain halt via unprivileged EVM call — (`x/cronos/keeper/precompiles/bank.go`)

### Summary

`BankContract.RequiredGas` evaluates `input[:4]` without a length guard. Any EVM call to the bank precompile (address `0x64`) with fewer than 4 bytes of calldata causes a deterministic Go runtime panic during block execution, halting all validators on that block.

### Finding Description

`BankContract.RequiredGas` at line 84 unconditionally evaluates `input[:4]`: [1](#0-0) 

In Go, `input[:4]` is a slice expression that panics at runtime with `slice bounds out of range [:4] with length N` whenever `len(input) < 4`. The `copy` builtin never gets a chance to execute safely — the panic occurs during argument evaluation.

By contrast, `IcaContract.RequiredGas` and `RelayerContract.RequiredGas` both guard against this: [2](#0-1) [3](#0-2) 

The bank precompile has no such guard.

### Impact Explanation

`RequiredGas` is called by the EVM framework **before** `Run`, as part of gas accounting during transaction execution. A panic here is unrecovered and propagates up through block processing. Because all validators execute the same transaction deterministically, every validator processing the block containing this transaction will panic and halt. This is a **Critical** chain halt.

### Likelihood Explanation

The attack requires only a standard EVM transaction sent to address `0x0000000000000000000000000000000000000064` with calldata of length 0, 1, 2, or 3 bytes. No special privileges, keys, or permissions are required. The attacker pays only the base transaction fee to halt the chain.

### Recommendation

Add the same length guard used in `IcaContract.RequiredGas` and `RelayerContract.RequiredGas`:

```go
func (bc *BankContract) RequiredGas(input []byte) uint64 {
    baseCost := uint64(len(input)) * bc.kvGasConfig.WriteCostPerByte
    var methodID [4]byte
    if len(input) < 4 {          // ADD THIS GUARD
        return baseCost
    }
    copy(methodID[:], input[:4])
    requiredGas, ok := bankGasRequiredByMethod[methodID]
    if ok {
        return requiredGas + baseCost
    }
    return baseCost
}
```
