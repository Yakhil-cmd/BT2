### Title
Bank Precompile `transfer` Accepts Arbitrary `sender` Without Caller Verification — (`File: x/cronos/keeper/precompiles/bank.go`)

### Summary
The `BankContract.Run` handler for `TransferMethodName` in the bank precompile decodes the `sender` address directly from ABI-encoded call input and passes it as the `from` address to `bankKeeper.SendCoins`, without ever verifying that `sender == contract.Caller()`. Any EVM contract can therefore call `bank.transfer(victim, attacker, amount)` and drain `evm/<caller_address>`-denominated tokens from any account that holds them, with no privilege required.

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` branch of `Run` is:

```go
case TransferMethodName:
    args, err := method.Inputs.Unpack(contract.Input[4:])
    sender    := args[0].(common.Address)   // ← fully attacker-controlled
    recipient := args[1].(common.Address)
    amount    := args[2].(*big.Int)
    from := sdk.AccAddress(sender.Bytes())
    to   := sdk.AccAddress(recipient.Bytes())
    ...
    denom := EVMDenom(contract.Caller())    // evm/<calling_contract_addr>
    amt   := sdk.NewCoin(denom, ...)
    err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
        ...
        return bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
    })
``` [1](#0-0) 

The only guard present is `checkBlockedAddr(to)`, which checks whether the *recipient* is a module-blocked address. There is no check that `sender == contract.Caller()`. [2](#0-1) 

The denomination is `EVMDenom(contract.Caller())` = `"evm/" + callerHex`, so the tokens that can be moved are those of the calling contract's own denomination. Any EVM contract is the implicit "issuer" of its own `evm/<addr>` denomination and can therefore move those tokens out of any holder's account without that holder's consent. [3](#0-2) 

The `mint` and `burn` branches correctly derive the subject address from `contract.Caller()` (via `denom := EVMDenom(contract.Caller())`), but the `transfer` branch uniquely exposes a user-supplied `sender` field that is never cross-checked against the caller. [4](#0-3) 

### Impact Explanation

**Critical — Unauthorized transfer of precompile-controlled assets.**

Any EVM contract can call `bank.transfer(victim, self, victim_balance)` and atomically drain all `evm/<contract_address>` tokens from the victim's Cosmos SDK bank account. Because the denomination is tied to the calling contract, the attacker controls both the minting supply and the ability to forcibly move balances. This is a direct, unconditional rug-pull of every holder of that token denomination, executable in a single unprivileged EVM transaction.

This matches the allowed Critical impact: *"Unauthorized … transfer … for … precompile-controlled assets."*

### Likelihood Explanation

Any deployed EVM contract that uses the bank precompile for token management is immediately exploitable by the contract deployer (or by any contract that can call it). No leaked keys, governance action, or cryptographic break is required. The entry path is a standard EVM call to the precompile at address `0x0000000000000000000000000000000000000064`.

### Recommendation

Add a caller-identity check at the top of the `TransferMethodName` branch before using the decoded `sender`:

```go
case TransferMethodName:
    args, err := method.Inputs.Unpack(contract.Input[4:])
    ...
    sender := args[0].(common.Address)
    // Enforce that only the token's issuing contract can move funds on behalf of a holder,
    // OR remove the sender argument entirely and use contract.Caller() as the source.
    if sender != contract.Caller() {
        return nil, errors.New("sender must be the calling contract")
    }
```

Alternatively, redesign the ABI so that `transfer` takes only `(recipient, amount)` and always uses `contract.Caller()` as the source, consistent with how `mint` and `burn` operate.

### Proof of Concept

1. Attacker deploys `MaliciousToken` at address `0xDEAD`.
2. `MaliciousToken` calls `bank.mint(alice, 1000)` → Alice now holds `1000 evm/0xDEAD` in the Cosmos bank module.
3. Attacker calls `bank.transfer(alice, attacker_eoa, 1000)` from `MaliciousToken`.
4. The bank precompile executes `bankKeeper.SendCoins(alice, attacker_eoa, [{denom:"evm/0xDEAD", amount:1000}])` with no authorization check.
5. Alice's entire `evm/0xDEAD` balance is transferred to the attacker. No signature from Alice, no approval, no governance action required. [1](#0-0)

			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err

					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}

				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
