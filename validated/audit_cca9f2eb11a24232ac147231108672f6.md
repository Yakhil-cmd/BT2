### Title
Unauthorized Global Contract Overwrite via Cross-Account Transaction — (`runtime/runtime/src/global_contracts.rs`)

### Summary
When `GlobalContractDeployMode::AccountId` is used, the runtime stores the global contract under `GlobalContractIdentifier::AccountId(receiver_id)` — the receipt's receiver account. There is no check that the deployer (signer/predecessor) is the same account as the one whose `account_id` is used as the identifier. Any unprivileged account can overwrite the global contract registered under any other account's `account_id` by sending a transaction with `receiver_id = victim` and action `DeployGlobalContract { deploy_mode: AccountId }`. All accounts that have adopted the victim's global contract via `UseGlobalContractByAccountId` will subsequently execute the attacker's injected code.

---

### Finding Description

**Invariant broken (global contract/code selection):** The `AccountId`-mode global contract namespace is intended to be owned by the account whose ID is the key. Only that account should be able to publish or update the contract stored under its own `account_id`. The implementation does not enforce this.

**Root cause — `action_deploy_global_contract`:** [1](#0-0) 

The function receives `account_id` — which is the receipt's `receiver_id` — and passes it directly to `initiate_distribution`: [2](#0-1) 

Inside `initiate_distribution`, when the mode is `AccountId`, the identifier is built from `account_id` with no ownership check: [3](#0-2) 

**Attacker-controlled field:** In a NEAR transaction, `signer_id` and `receiver_id` are independent fields. A transaction signed by `attacker.near` can freely set `receiver_id = victim.near`. The resulting receipt has `receiver_id = victim.near`, so `account_id = victim.near` flows into `initiate_distribution`, and the global contract is stored under `GlobalContractIdentifier::AccountId(victim.near)`.

**Exact corrupted value:** `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim.near) }` — the on-trie contract blob for the victim's account-ID-keyed global contract is overwritten with the attacker's payload. [4](#0-3) 

**Existing guard that fails — nonce check:** The only guard against overwrites is a monotone nonce (`check_and_update_nonce`) that prevents *stale* re-deliveries of the same distribution receipt. It does not prevent a *fresh* unauthorized deployment from a different signer: [5](#0-4) 

A fresh transaction from the attacker produces a new, higher nonce, so the nonce check passes and the overwrite succeeds.

**No action-level restriction:** `DeployGlobalContract` is not restricted to self-calls (`signer_id == receiver_id`) anywhere in action validation: [6](#0-5) 

A `FullAccess` key permits `DeployGlobalContract` targeting any `receiver_id`: [7](#0-6) 

**Downstream effect on users:** Every account that called `UseGlobalContract { contract_identifier: AccountId(victim.near) }` has its `AccountContract` field set to `GlobalByAccount(victim.near)`: [8](#0-7) 

After the overwrite, all those accounts execute the attacker's code on every function call.

---

### Impact Explanation

**Scope: global contract/code selection — Critical.**

- Any account that published a global contract in `AccountId` mode (the mutable, "owner-updatable" mode) can have its contract silently replaced by an attacker.
- Every downstream account that adopted the victim's contract via `UseGlobalContractByAccountId` will execute the attacker's injected WebAssembly on every subsequent call — enabling arbitrary state reads/writes, fund theft, and denial of service across all those accounts.
- The storage cost of the attack is charged to the victim's account balance, compounding the harm.

---

### Likelihood Explanation

- Requires only a funded NEAR account and a standard signed transaction — no validator or operator privilege.
- The attack is a single transaction: `signer_id = attacker`, `receiver_id = victim`, action `DeployGlobalContract { code: malicious_wasm, deploy_mode: AccountId }`.
- The victim's account must hold enough balance to cover `global_contract_storage_amount_per_byte × code.len()`. A minimal Wasm binary (tens of bytes) makes this cost negligible.
- No race condition or timing dependency; the overwrite is deterministic and permanent until the victim re-deploys.

---

### Recommendation

In `action_deploy_global_contract`, when `deploy_mode == GlobalContractDeployMode::AccountId`, assert that the receipt's `signer_id` (from `ActionReceipt.signer_id`) equals `account_id` (the receiver). Alternatively, enforce `predecessor_id == receiver_id` at the receipt level for this action. This mirrors the ownership model described in the documentation ("allows the **owner** to update the contract for all its users"). [1](#0-0) 

---

### Proof of Concept

```
// Step 1 – victim legitimately deploys a global contract under their account_id
Transaction {
    signer_id:   "victim.near",
    receiver_id: "victim.near",
    actions: [DeployGlobalContract { code: <legitimate_wasm>, deploy_mode: AccountId }],
}

// Step 2 – other accounts adopt the victim's contract
Transaction {
    signer_id:   "user.near",
    receiver_id: "user.near",
    actions: [UseGlobalContract { contract_identifier: AccountId("victim.near") }],
}

// Step 3 – attacker overwrites the global contract under victim's account_id
Transaction {
    signer_id:   "attacker.near",   // attacker's own key — no victim permission needed
    receiver_id: "victim.near",     // receiver_id drives the AccountId identifier
    actions: [DeployGlobalContract { code: <malicious_wasm>, deploy_mode: AccountId }],
}
// action_deploy_global_contract is called with account_id = "victim.near"
// initiate_distribution stores malicious_wasm under GlobalContractIdentifier::AccountId("victim.near")
// nonce is incremented → check_and_update_nonce passes on all shards

// Step 4 – every call to user.near now executes malicious_wasm
FunctionCall { receiver_id: "user.near", method_name: "any_method", ... }
// → AccountContract::GlobalByAccount("victim.near") → loads malicious_wasm
```

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())

fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
}

/// Validates `AccessKeyPermission`. If the access key permission is `FunctionCall`, checks that the
/// total number of bytes of the method names doesn't exceed the limit and
/// every method name length doesn't exceed the limit.
fn validate_access_key_permission(
    limit_config: &LimitConfig,
    permission: &AccessKeyPermission,
) -> Result<(), ActionsValidationError> {
    if let Some(fc) = permission.function_call_permission() {
        // Check whether `receiver_id` is a valid account_id. Historically, we
        // allowed arbitrary strings there!
        match limit_config.account_id_validity_rules_version {
            near_primitives_core::config::AccountIdValidityRulesVersion::V0 => (),
            near_primitives_core::config::AccountIdValidityRulesVersion::V1
            | near_primitives_core::config::AccountIdValidityRulesVersion::V2 => {
                if let Err(_) = fc.receiver_id.parse::<AccountId>() {
                    return Err(ActionsValidationError::InvalidAccountId {
                        account_id: truncate_string(&fc.receiver_id, AccountId::MAX_LEN * 2),
                    });
                }
            }
        }

        // Checking method name length limits
        let mut total_number_of_bytes = 0;
        for method_name in &fc.method_names {
            let length = method_name.len() as u64;
            if length > limit_config.max_length_method_name {
                return Err(ActionsValidationError::AddKeyMethodNameLengthExceeded {
                    length,
                    limit: limit_config.max_length_method_name,
                });
            }
            // Adding terminating character to the total number of bytes
            total_number_of_bytes += length + 1;
        }
        if total_number_of_bytes > limit_config.max_number_bytes_method_names {
            return Err(ActionsValidationError::AddKeyMethodNamesNumberOfBytesExceeded {
                total_number_of_bytes,
                limit: limit_config.max_number_bytes_method_names,
            });
        }
    }

    Ok(())
}
