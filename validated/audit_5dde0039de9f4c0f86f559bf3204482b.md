### Title
Global `last_reward_block` Consumed by `disable_rewards: true` Allows Any Caller to Permanently Block Consensus Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `IStakingRewardsManager` writes to the global `last_reward_block` storage variable **before** checking `disable_rewards`. Because the function is callable by any address with any valid `staker_address`, an unprivileged attacker can call `update_rewards(victim_staker, disable_rewards: true)` every block to consume the per-block reward slot without distributing any rewards, permanently freezing unclaimed consensus yield for all stakers.

### Finding Description
`update_rewards` enforces a single-call-per-block invariant via a global storage variable `last_reward_block`:

```cairo
// src/staking/staking.cairo ~line 1453
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After the assertion passes, the function unconditionally writes the current block number to `last_reward_block` and **then** checks `disable_rewards`:

```cairo
// src/staking/staking.cairo ~line 1485
self.last_reward_block.write(current_block_number);   // ← written regardless

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing rewards
}
```

Because `last_reward_block` is a global (non-keyed) storage slot, a single call to `update_rewards` for **any** staker with `disable_rewards: true` exhausts the entire block's reward window for **all** stakers. No access control guards the function beyond `general_prerequisites()` (a pause check). [1](#0-0) 

### Impact Explanation
An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block:
1. Sets `last_reward_block` to the current block.
2. Returns without distributing any rewards.
3. Causes every subsequent `update_rewards` call in that block to revert with `REWARDS_ALREADY_UPDATED`.

Repeated every block, this permanently freezes all unclaimed consensus-epoch yield for every staker in the protocol. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- `update_rewards` has no role-based access control; any EOA or contract can call it.
- The attacker only needs one publicly visible staker address (trivially obtained from on-chain events).
- On Starknet, per-transaction fees are low, making sustained block-by-block griefing economically viable.
- The attacker gains nothing financially but inflicts maximum damage at minimal cost.

### Recommendation
Write `last_reward_block` only when rewards are actually distributed (i.e., after the `disable_rewards` / `is_pre_consensus` guard), or make `last_reward_block` a per-staker mapping so one call cannot block all others:

```cairo
// Option A: move the write after the early-return guard
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);  // only written when rewards flow
```

```cairo
// Option B: per-staker tracking
last_reward_block: Map<ContractAddress, BlockNumber>
```

### Proof of Concept
1. Deploy the protocol and advance to the consensus-rewards epoch.
2. Attacker (any address) calls `update_rewards(alice_staker, disable_rewards: true)` at block N.
   - `last_reward_block` is set to N; no rewards are distributed.
3. Alice's node calls `update_rewards(alice_staker, disable_rewards: false)` at block N.
   - Reverts: `REWARDS_ALREADY_UPDATED`.
4. Repeat step 2 every block → Alice (and every other staker) earns zero consensus rewards indefinitely. [2](#0-1) [3](#0-2)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
