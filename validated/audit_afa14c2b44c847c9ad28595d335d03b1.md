### Title
Staker Can Bind Any Operational Address Without Consent in `stake()` — (File: src/staking/staking.cairo)

### Summary
The `stake` function allows a staker to bind any `operational_address` to themselves without requiring the operational address to have consented. This is structurally inconsistent with `change_operational_address`, which enforces a two-step consent mechanism via `declare_operational_address`. An attacker can grief a legitimate validator by front-running their `stake` call with the same `operational_address`, permanently blocking the validator from using their intended operational address.

### Finding Description
`stake()` at [1](#0-0)  accepts a caller-supplied `operational_address` parameter and binds it to the caller's staker identity with only a uniqueness check:

```
assert!(
    self.operational_address_to_staker_address.read(operational_address).is_zero(),
    "{}",
    Error::OPERATIONAL_EXISTS,
);
``` [2](#0-1) 

There is no requirement that the `operational_address` has previously consented to being associated with this staker. The binding is then written unconditionally: [3](#0-2) 

By contrast, `change_operational_address()` enforces a two-step consent mechanism: the operational address must first call `declare_operational_address(staker_address)`, which writes `eligible_operational_addresses[operational_address] = staker_address`. Only then can the staker call `change_operational_address`, which verifies:

```
assert!(
    self.eligible_operational_addresses.read(operational_address) == staker_address,
    "{}",
    Error::OPERATIONAL_NOT_ELIGIBLE,
);
``` [4](#0-3) 

The `declare_operational_address` function itself is: [5](#0-4) 

The `stake` path entirely bypasses this consent gate, creating an asymmetry: initial registration requires no consent, but subsequent changes do.

### Impact Explanation
An attacker who knows (or can observe) which `operational_address` a legitimate validator intends to use can call `stake` with that address first. Once bound, the address is marked in `operational_address_to_staker_address` and the legitimate validator's own `stake` call will revert with `OPERATIONAL_EXISTS`. The validator is forced to use a different operational address. Additionally, if the operational address owner unknowingly continues to submit attestations (believing they are operating for their own stake), those attestations accrue rewards to the attacker's staker record instead.

This maps to **Medium: Griefing with no profit motive but damage to users or protocol** — the attacker must lock up their own stake capital, but the victim validator suffers disruption and potential loss of attestation rewards.

### Likelihood Explanation
Starknet transactions are visible to the sequencer before finalization. A motivated attacker monitoring for `stake` transactions with a target `operational_address` (e.g., a well-known validator key) can submit a competing `stake` transaction. No privileged access is required; any address with sufficient STRK balance can execute this. The attacker does not need to know the victim's private key — only their intended `operational_address`, which may be publicly announced or derivable from off-chain validator metadata.

### Recommendation
Apply the same consent gate to `stake` that is already enforced in `change_operational_address`. Before accepting an `operational_address` in `stake`, require that `eligible_operational_addresses[operational_address] == staker_address` (i.e., the operational address must have previously called `declare_operational_address` naming this staker). This mirrors the fix described in the referenced report: the signer/operational address must itself specify which custodian/staker it consents to serve.

### Proof of Concept
1. Validator V intends to call `stake(reward_addr_V, operational_addr_X, amount_V)`.
2. Attacker A observes this (mempool or off-chain announcement) and calls `stake(reward_addr_A, operational_addr_X, amount_A)` first with any valid `amount_A >= min_stake`.
3. `operational_address_to_staker_address[operational_addr_X]` is now set to attacker A's staker address.
4. Validator V's `stake` transaction reverts: `OPERATIONAL_EXISTS`.
5. Validator V must choose a different operational address. If V's node software is configured to sign attestations from `operational_addr_X`, those attestations now credit attacker A's staker record with rewards.

        fn stake(
            ref self: ContractState,
            reward_address: ContractAddress,
            operational_address: ContractAddress,
            amount: Amount,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let staker_address = get_caller_address();
            assert!(self.staker_info.read(staker_address).is_none(), "{}", Error::STAKER_EXISTS);
            assert!(
                self.operational_address_to_staker_address.read(operational_address).is_zero(),
                "{}",
                Error::OPERATIONAL_EXISTS,
            );
            self.assert_staker_address_not_reused(:staker_address);
            assert!(
                !self.does_token_exist(token_address: staker_address), "{}", Error::STAKER_IS_TOKEN,
            );
            assert!(
                !self.does_token_exist(token_address: reward_address),
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            assert!(
                !self.does_token_exist(token_address: operational_address),
                "{}",
                Error::OPERATIONAL_IS_TOKEN,
            );
            assert!(amount >= self.min_stake.read(), "{}", Error::AMOUNT_LESS_THAN_MIN_STAKE);
            let normalized_amount = NormalizedAmountTrait::from_strk_native_amount(:amount);

            // Transfer funds from staker. Sufficient approvals is a pre-condition.
            let staking_contract = get_contract_address();
            let token_dispatcher = strk_token_dispatcher();
            token_dispatcher
                .checked_transfer_from(
                    sender: staker_address, recipient: staking_contract, amount: amount.into(),
                );

            self
                .initialize_staker_own_balance_trace(
                    :staker_address, own_balance: normalized_amount,
                );

            // Create the record for the staker.
            self
                .staker_info
                .write(
                    staker_address,
                    VInternalStakerInfoTrait::new_latest(:reward_address, :operational_address),
                );

            // Update the operational address mapping, which is a 1 to 1 mapping.
            self.operational_address_to_staker_address.write(operational_address, staker_address);

            // Update total stake.
            self.add_to_total_stake(token_address: STRK_TOKEN_ADDRESS, amount: normalized_amount);

            // Add staker address to the stakers vector.
            self.stakers.push(staker_address);

            // Mark the staker's version.
            self.staker_version.write(staker_address, LATEST_STAKER_VERSION);

            // Emit events.
            self
                .emit(
                    Events::NewStaker {
                        staker_address, reward_address, operational_address, self_stake: amount,
                    },
                );
            self
                .emit(
                    Events::StakeOwnBalanceChanged {
                        staker_address, old_self_stake: Zero::zero(), new_self_stake: amount,
                    },
                );
        }

        fn declare_operational_address(ref self: ContractState, staker_address: ContractAddress) {
            self.general_prerequisites();
            let operational_address = get_caller_address();
            assert!(
                self.operational_address_to_staker_address.read(operational_address).is_zero(),
                "{}",
                Error::OPERATIONAL_IN_USE,
            );
            assert!(
                !self.does_token_exist(token_address: operational_address),
                "{}",
                Error::OPERATIONAL_IS_TOKEN,
            );
            if self.eligible_operational_addresses.read(operational_address) == staker_address {
                return;
            }
            self.eligible_operational_addresses.write(operational_address, staker_address);
            self.emit(Events::OperationalAddressDeclared { operational_address, staker_address });
        }
