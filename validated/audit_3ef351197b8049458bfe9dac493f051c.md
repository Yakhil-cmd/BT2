### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Freeze Staker and Delegator Yield - (`src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is specified as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` to consume the single per-block reward slot without distributing any rewards, permanently preventing the legitimate sequencer from crediting stakers and pool members for that block.

---

### Finding Description

The protocol specification explicitly restricts `update_rewards` to the Starkware sequencer:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. The only guards are a pause check via `general_prerequisites()` and a per-block idempotency guard:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no caller identity check ...
    self.last_reward_block.write(current_block_number);  // <-- slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;  // <-- exits without distributing rewards
    }
    // ...
}
``` [2](#0-1) 

The interface definition likewise carries no access-control annotation:

```cairo
pub trait IStakingRewardsManager<TContractState> {
    fn update_rewards(
        ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
    );
}
``` [3](#0-2) 

The critical sequence is:

1. `last_reward_block` is written to `current_block_number` **before** the `disable_rewards` early-return check (line 1485 before line 1487).
2. Any subsequent call in the same block — including the legitimate sequencer's call — hits `REWARDS_ALREADY_UPDATED` and reverts.
3. The staker and all pool members receive zero rewards for that block.

---

### Impact Explanation

Every block in which an attacker front-runs the sequencer with `update_rewards(staker_address, disable_rewards: true)` causes the staker's `unclaimed_rewards_own` to receive no increment, and no STRK is transferred to delegation pools. Because `last_reward_block` is a single global slot shared across all stakers, one successful front-run per block silences reward distribution for the entire protocol for that block. [4](#0-3) 

This constitutes **permanent freezing of unclaimed yield** — a valid High impact under the allowed scope.

---

### Likelihood Explanation

`update_rewards` is a public, permissionless entry point on a deployed contract. An attacker needs only to:
- Know any active staker address (readable from on-chain events or `get_stakers`).
- Submit a transaction with `disable_rewards: true` before the sequencer's transaction in each block.

On Starknet, transaction ordering within a block is determined by the sequencer, but the function is callable by any externally-owned account. A motivated attacker can submit the griefing transaction at the start of every block at negligible cost (gas on L2 is cheap), making sustained, indefinite denial of yield realistic.

---

### Recommendation

Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured sequencer address (or a role such as `REWARDS_MANAGER`), consistent with the specification:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
}
```

---

### Proof of Concept

```
Block N:
  1. Attacker calls: staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
     - general_prerequisites() passes (not paused)
     - current_block_number (N) > last_reward_block (N-1) → passes
     - staker is active → passes
     - last_reward_block.write(N)   ← slot consumed
     - disable_rewards == true → early return, zero rewards distributed

  2. Sequencer calls: staking.update_rewards(staker_address=<staker>, disable_rewards=false)
     - current_block_number (N) > last_reward_block (N) → FALSE
     - Reverts with REWARDS_ALREADY_UPDATED

Result: staker.unclaimed_rewards_own unchanged; pool members receive no STRK for block N.
Repeat every block → all yield permanently frozen.
``` [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
