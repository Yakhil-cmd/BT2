### Title
Integer Division Truncation Zeroes `native_per_pubdata`, Allowing Pubdata Cost Bypass - (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

In ZKsync OS's L2 transaction validation, the `native_per_pubdata` rate — which controls how many native (proving) resources a transaction must pay per byte of pubdata it generates — is computed using integer floor division. When the operator-set `pubdata_price` is less than `native_price`, the division truncates to **zero**. A zero rate means the system charges no native resources for any amount of pubdata, completely bypassing the operator's pubdata pricing mechanism and allowing any unprivileged transaction sender to generate pubdata at zero native cost.

---

### Finding Description

In `validate_and_compute_fee_for_transaction` (the L2 transaction validation path), the `native_per_pubdata` rate is computed as:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs, line 142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
``` [1](#0-0) 

`wrapping_div` performs standard integer floor division. When `pubdata_price < native_price` (e.g., `pubdata_price = 5`, `native_price = 10`), the result is `5 / 10 = 0`. The rate is silently zeroed.

This is in direct contrast to how `native_per_gas` is computed on the very next lines, which correctly uses ceiling division to prevent a zero rate:

```rust
// line 135
u256_try_to_u64(&gas_price.div_ceil(native_price)).ok_or(...)
``` [2](#0-1) 

The same truncation bug is present in the API helper used by off-chain tooling:

```rust
// api/src/helpers.rs, line 427
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [3](#0-2) 

Once `native_per_pubdata = 0`, it propagates through the entire resource accounting pipeline:

**1. Intrinsic pubdata overhead becomes zero** in `create_resources_for_tx`:
```rust
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// → 0 * intrinsic_pubdata = 0
``` [4](#0-3) 

**2. Execution pubdata cost becomes zero** in `get_resources_to_charge_for_pubdata`:
```rust
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)  // → any_amount * 0 = 0
    .ok_or(out_of_native_resources!())?;
``` [5](#0-4) 

**3. The pubdata sufficiency check always passes** in `check_enough_resources_for_pubdata`:
```rust
let enough = resources.has_enough(&resources_for_pubdata);
// resources_for_pubdata = 0 native → always true
``` [6](#0-5) 

---

### Impact Explanation

When `0 < pubdata_price < native_price`, the operator has intentionally set a non-zero pubdata price to make transactions pay for the cost of publishing state diffs to the settlement layer. However, due to the truncation, `native_per_pubdata = 0` and:

- No native resources are charged for any pubdata generated (neither intrinsic nor execution-phase pubdata).
- The post-execution pubdata check always passes regardless of how much pubdata the transaction produced.
- The prover must still process and prove all generated pubdata, but the transaction paid nothing for it in native resources.
- An attacker can craft transactions that perform many storage writes (each generating 32+ bytes of pubdata per slot) and exhaust the block's pubdata budget at zero native cost, effectively subsidizing their proving costs onto the protocol.

The `delta_gas` adjustment in `compute_gas_refund` does not compensate for this, as it only adjusts gas based on `native_used / native_per_gas` (computational native), not pubdata native:

```rust
let delta_gas = if native_per_gas == 0 { 0 } else {
    (native_used / native_per_gas) as i64 - (gas_used as i64)
};
``` [7](#0-6) 

---

### Likelihood Explanation

The condition `pubdata_price < native_price` is a realistic operator configuration. The operator sets both values independently via block metadata. During periods of low L1 data availability costs, `pubdata_price` could be set lower than `native_price`. Any L2 transaction sender — with no special privileges — can exploit this by submitting transactions with many storage writes. The entry path is a standard EIP-1559 or legacy transaction; no privileged access is required.

---

### Recommendation

Replace `wrapping_div` with ceiling division (`div_ceil`) for the `native_per_pubdata` calculation, consistent with how `native_per_gas` is computed:

```rust
// Before (truncates to zero when pubdata_price < native_price):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (preserves at least 1 native unit per pubdata byte when pubdata_price > 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix to `api/src/helpers.rs` line 427. Alternatively, introduce a fixed-point scaling factor (e.g., multiply `pubdata_price` by `10^8` before dividing) to preserve sub-unit precision, similar to the mitigation recommended in the reference report.

---

### Proof of Concept

**Setup**: Operator sets `native_price = 100`, `pubdata_price = 50` (a plausible configuration where pubdata is cheaper than native computation).

**Calculation**:
- `native_per_pubdata = 50.wrapping_div(100) = 0`
- `intrinsic_pubdata_overhead = 0 * L2_TX_INTRINSIC_PUBDATA = 0` (no native deducted for intrinsic pubdata)
- For a transaction writing to 100 storage slots: `pubdata_used ≈ 3200 bytes`
- `native_for_pubdata = 3200 * 0 = 0` (no native charged for 3200 bytes of pubdata)
- `check_enough_resources_for_pubdata` returns `true` (0 ≤ any remaining native)

**Result**: The transaction generates 3200 bytes of pubdata and pays zero native resources for it. The operator's pubdata pricing is completely bypassed. An attacker can repeat this across many transactions to fill the block's pubdata budget at no native cost, forcing the prover to process pubdata that was not paid for.
