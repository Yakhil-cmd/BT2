### Title
WalletConnect `take_offer` Confirmation Dialog Understates Total Spend for Multi-NFT Royalty Offers via Incorrect `splitAmount` Division in `formatAmountWithRoyalties` — (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

---

### Summary

The `formatAmountWithRoyalties` function, which generates the royalty-inclusive total displayed in the WalletConnect approval dialog for `chia_wallet.take_offer`, divides the fungible amount equally among all NFTs before computing each royalty. The Chia protocol computes each NFT's royalty on the **full** fungible amount. For any offer involving two or more NFTs with royalties, the confirmation dialog systematically understates the actual on-chain cost, causing a user to approve a `take_offer` command under a false impression of the total spend.

---

### Finding Description

In `formatAmountWithRoyalties`:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // floor division
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

The function first computes `splitAmount = floor(amount / N)` where N is the number of NFTs, then applies each NFT's royalty percentage to that split amount. The Chia protocol, however, computes each NFT's royalty on the **full** fungible amount. The two formulas diverge for N > 1:

| | Formula |
|---|---|
| **Display** | `royaltyTotal = Σ floor(floor(A / N) × Rᵢ / 10000)` |
| **Protocol** | `royaltyTotal = Σ floor(A × Rᵢ / 10000)` |

The display formula underestimates royalties by approximately `(N−1)/N × Σ(A × Rᵢ / 10000)` for N > 1.

The `royaltyPercentages` array fed into this function is collected from all NFT items on the receiving side by `royaltyPercentagesForSide`, and `withRoyaltyTotals` passes it unchanged to `formatAmountWithRoyalties` for every fungible spending item: [2](#0-1) [3](#0-2) 

This display value is produced by `parseCommandDisplay` specifically to populate the WalletConnect confirmation dialog before the user approves the command: [4](#0-3) 

---

### Impact Explanation

A user approving a WalletConnect `chia_wallet.take_offer` command for a multi-NFT offer sees an understated `amountWithRoyalties` in the confirmation dialog. The actual on-chain transaction deducts more from their wallet than the dialog indicates. This fits the **High** allowed impact: *"WalletConnect state that causes a user to approve… the wrong… amount."*

---

### Likelihood Explanation

The attacker needs only an approved WalletConnect session (a normal user action) and an offer involving two or more NFTs with royalties. Multi-NFT offers are a supported protocol feature. The attacker constructs the offer; the user's wallet fetches the summary and calls `parseCommandDisplay`, which produces the understated total without any further attacker interaction.

---

### Recommendation

Replace `splitAmount` with `amount` in the inner royalty computation so each NFT's royalty is applied to the full fungible amount:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [5](#0-4) 

---

### Proof of Concept

**Setup**: A WalletConnect dApp with an approved session sends `chia_wallet.take_offer` for an offer: spend 1,000,000 mojos to receive NFT-A (royalty 500 bp = 5%) and NFT-B (royalty 300 bp = 3%).

**Display calculation** (`formatAmountWithRoyalties`):
```
splitAmount = 1,000,000 / 2 = 500,000
royaltyAmount = (500,000 × 500 / 10,000) + (500,000 × 300 / 10,000)
             = 25,000 + 15,000 = 40,000
totalAmount  = 1,040,000  →  dialog shows "1.04 XCH"
```

**Protocol calculation** (each royalty on full amount):
```
royaltyAmount = (1,000,000 × 500 / 10,000) + (1,000,000 × 300 / 10,000)
             = 50,000 + 30,000 = 80,000
totalAmount  = 1,080,000  →  actual spend is 1.08 XCH
```

The user approves believing they spend **1.04 XCH**; the blockchain deducts **1.08 XCH** — a 40,000-mojo discrepancy (≈ 3.8% of the offer price) that scales with the number of NFTs and their royalty rates.

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }
