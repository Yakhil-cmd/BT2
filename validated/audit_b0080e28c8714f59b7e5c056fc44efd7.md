### Title
Cairo0 Contract Bypasses `replace_class` Cairo1-Only Invariant via `DeprecatedSyscallHintProcessor` — (`crates/blockifier/src/execution/deprecated_syscalls/hint_processor.rs`)

### Summary
`DeprecatedSyscallHintProcessor::replace_class` omits the `is_cairo1` check present in `SyscallHandlerBase::replace_class`, allowing a Cairo0 contract to replace its class hash with any declared Cairo0 class hash. This corrupts the on-chain class pointer and causes the wrong compiled class to be selected for all subsequent executions of that contract.

### Finding Description

`SyscallHandlerBase::replace_class` enforces the invariant that only Cairo1 classes may be used as replacement targets: [1](#0-0) 

The check at line 373 returns `ForbiddenClassReplacement` for any non-Cairo1 compiled class.

`DeprecatedSyscallHintProcessor::replace_class`, used exclusively for Cairo0 contract execution, does not call `SyscallHandlerBase::replace_class`. It implements the syscall independently and omits the guard entirely: [2](#0-1) 

The implementation only calls `get_compiled_class` to verify the class is declared, then unconditionally calls `set_class_hash_at`. There is no `is_cairo1` check anywhere in this path.

The `ReplaceClass` selector is fully routed through the deprecated executor: [3](#0-2) 

### Impact Explanation

An attacker controlling a Cairo0 account contract can:
1. Declare any Cairo0 class (a normal, permissionless operation).
2. Submit a transaction whose `__execute__` entry point calls `replace_class` with the Cairo0 class hash.
3. `DeprecatedSyscallHintProcessor::replace_class` accepts the call, writing the Cairo0 class hash to `storage_address` via `set_class_hash_at`.
4. All subsequent invocations of that contract address resolve to the Cairo0 compiled class instead of the expected Cairo1 class.

This satisfies the Critical impact criterion: **wrong compiled class / contract code selected for execution** from an accepted transaction.

### Likelihood Explanation

Any deployed Cairo0 contract can trigger this. No privileged access is required — only the ability to submit a transaction from a Cairo0 account, which is a standard unprivileged operation on Starknet.

### Recommendation

Add the `is_cairo1` check to `DeprecatedSyscallHintProcessor::replace_class`, mirroring `SyscallHandlerBase::replace_class`:

```rust
fn replace_class(
    request: ReplaceClassRequest,
    _vm: &mut VirtualMachine,
    syscall_handler: &mut Self,
) -> DeprecatedSyscallResult<ReplaceClassResponse> {
    let compiled_class = syscall_handler.state.get_compiled_class(request.class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(/* DeprecatedSyscallExecutionError::ForbiddenClassReplacement */);
    }
    syscall_handler
        .state
        .set_class_hash_at(syscall_handler.storage_address, request.class_hash)?;
    Ok(ReplaceClassResponse {})
}
```

### Proof of Concept

1. Deploy a Cairo0 account contract at address `A`.
2. Declare a second Cairo0 class with hash `H_cairo0`.
3. Submit an `InvokeV1` transaction from `A` whose `__execute__` calls `replace_class(H_cairo0)`.
4. The blockifier routes execution through `DeprecatedSyscallHintProcessor::replace_class`.
5. `get_compiled_class(H_cairo0)` succeeds (class is declared); `set_class_hash_at(A, H_cairo0)` is called.
6. Assert `state.get_class_hash_at(A) == H_cairo0` — a Cairo0 class hash, not a Cairo1 class hash.
7. Submit a subsequent transaction to `A`; the blockifier selects the Cairo0 compiled class for execution, not the original Cairo1 class.
