### Title
Unbounded `stakers` Vec Causes `get_stakers()` Unbounded Gas Consumption via Stake/Unstake Spam — (File: `src/staking/staking.cairo`)

---

### Summary
The `stakers` storage Vec in `staking.cairo` is append-only: every new staker is pushed into it, but stakers are **never removed** even after a full unstake. The `get_stakers()` function in `IStakingConsensus` iterates over the entire Vec on every call. An unprivileged attacker can permanently inflate this Vec by cycling through many addresses (stake → wait exit window → unstake → recover funds), making `get_stakers()` progressively more expensive until it exceeds gas limits, denying the consensus layer its validator-set query.

---

### Finding Description

**Root cause — append-only Vec, never pruned:**

The storage declaration at line 169 carries an explicit developer warning:

```
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [1](#0-0) 

Every call to `stake()` unconditionally appends the caller's address:

```cairo
// Add staker address to the stakers vector.
self.stakers.push(staker_address);
``` [2](#0-1) 

No corresponding removal exists anywhere in the contract — not in `unstake_intent`, not in `unstake_action`, not in any migration path.

**Unbounded iteration in `get_stakers()`:**

`get_stakers()` (part of `IStakingConsensus`) iterates over the **full range** of the Vec on every invocation:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    ...
}
``` [3](#0-2) 

Every ghost entry (an address that staked and then fully unstaked) still costs a storage read plus an `is_staker_active` check. There is no pagination, no lazy deletion, and no cap on Vec length.

**Attacker-controlled entry path:**

`stake()` is a fully public, permissionless function. The only guard is `amount >= min_stake`. After the `exit_wait_window` (default 1 week), `unstake_action` returns the full principal to the attacker. The net cost per ghost entry is therefore only the gas for two transactions — the capital is fully recovered.

---

### Impact Explanation

`get_stakers()` is the consensus-layer entry point for the validator set. As the Vec grows, each call consumes more gas. Once the Vec is large enough, the function will exceed the Starknet execution-resource limit and revert on every call. This prevents any consumer of `IStakingConsensus` from obtaining the active staker list, which can block attestation validation and reward distribution.

**Allowed impact matched:** *Medium — Unbounded gas consumption / Griefing with damage to protocol.*

---

### Likelihood Explanation

The attack is always executable by any address that can acquire `min_stake` tokens. Because the principal is fully returned after unstaking, the attacker's sustained cost is only gas. The Vec is permanent storage — once inflated it cannot be deflated without a contract upgrade. Likelihood: **High**.

---

### Recommendation

1. **Lazy deletion / tombstone flag**: When a staker fully unstakes, mark their slot as inactive in a separate `Map<ContractAddress, bool>` and skip them in `get_stakers()` without iterating ghost entries — or, better, swap-and-pop the Vec entry on removal.
2. **Paginated `get_stakers()`**: Accept `offset` and `limit` parameters so callers can chunk the iteration and avoid single-call gas exhaustion.
3. **Minimum stake as a deterrent**: Ensure `min_stake` is set high enough that the gas-only attack cost is economically prohibitive; document this dependency explicitly.

---

### Proof of Concept

```
Attacker setup (repeat N times with fresh EOA):
  1. EOA_i approves staking contract for min_stake STRK
  2. EOA_i calls staking.stake(reward_addr, op_addr, min_stake)
     → self.stakers.push(EOA_i)          // Vec length += 1
  3. EOA_i calls staking.unstake_intent()
  4. Advance time by exit_wait_window
  5. EOA_i calls staking.unstake_action() // full principal returned
     → EOA_i is gone from staker_info but REMAINS in self.stakers

After N iterations:
  - self.stakers.len() == N (all ghost entries)
  - staking_consensus.get_stakers(epoch_id) iterates all N entries,
    paying a storage read + is_staker_active check per entry.
  - At sufficiently large N the call reverts with out-of-gas,
    permanently denying the consensus layer its validator-set query.
``` [4](#0-3) [5](#0-4)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
