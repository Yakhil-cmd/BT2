### Title
All EIP-2537 BLS12-381 Precompiles Charge Zero Native Resources, Bypassing Proving-Cost Limit — (`basic_bootloader/src/bootloader/block_flow/ethereum/hooks/eip_2537/`)

---

### Summary

Every BLS12-381 precompile introduced by EIP-2537 hardcodes `cost_native = 0`. The native resource is ZKsync OS's model for prover complexity (RISC-V cycles). Setting it to zero for computationally intensive elliptic-curve operations means the native resource limit is never enforced for those calls, directly analogous to the original report's precision mismatch that caused a resource value to always be zero.

---

### Finding Description

ZKsync OS tracks two independent resources per transaction:

1. **Ergs** — EVM gas × `ERGS_PER_GAS` (256). Enforces EVM-level gas limits.
2. **Native** — models proving cost (RISC-V cycles). Enforces the `MAX_NATIVE_COMPUTATIONAL` cap (2³⁵) per transaction.

Every other precompile charges a non-zero native cost:

| Precompile | Native cost constant |
|---|---|
| `ecrecover` | `ECRECOVER_NATIVE_COST = native_with_delegations!(350_000, 43_000, 0)` |
| `bn254_ecmul` | `BN254_ECMUL_NATIVE_COST = native_with_delegations!(600_000, 41_000, 0)` |
| `bn254_pairing` | `BN254_PAIRING_BASE_NATIVE_COST = native_with_delegations!(13_000_000, 500_000, 0)` |
| `point_evaluation` | `POINT_EVALUATION_NATIVE_COST = native_with_delegations!(49_900_000, 3_301_000, 0)` | [1](#0-0) 

However, all seven BLS12-381 precompiles hardcode `cost_native = 0`:

**G1/G2 Addition** (`addition.rs`):
```rust
let cost_ergs = Ergs(BLS12_381_G1_ADDITION_GAS * ERGS_PER_GAS);
let cost_native = 0;   // ← always zero
``` [2](#0-1) 

**G1/G2 MSM** (`msm.rs`):
```rust
let cost_ergs = Ergs(cost * ERGS_PER_GAS);
let cost_native = 0;   // ← always zero
``` [3](#0-2) 

**Pairing Check** (`pairing.rs`):
```rust
let cost_ergs = Ergs(((num_pairs as u64) * BLS12_381_PAIRING_PER_PAIR_GAS + BLS12_381_PAIRING_FIXED_GAS) * ERGS_PER_GAS);
let cost_native = 0;   // ← always zero
``` [4](#0-3) 

**Field-to-Curve Mappings** (`mappings.rs`):
```rust
let cost_ergs = Ergs(BLS12_381_FIELD_TO_G1_GAS * ERGS_PER_GAS);
let cost_native = 0;   // ← always zero
``` [5](#0-4) 

No BLS12-381 native-cost constant exists anywhere in `cost_constants.rs`, confirming this is an omission, not an intentional design choice. [6](#0-5) 

The native resource cap is enforced in `create_resources_for_tx` by splitting resources into `main_resources` (capped at `MAX_NATIVE_COMPUTATIONAL`) and `withheld`. Because BLS12-381 operations never consume native resources, this cap is entirely bypassed for them. [7](#0-6) 

---

### Impact Explanation

**Forward/proving divergence.** The forward system (sequencer) allows a transaction to succeed because native resources are not exhausted. The prover, however, must actually execute the BLS12-381 arithmetic in RISC-V and will consume a number of cycles proportional to the real computational cost — which is not bounded by the native resource limit. A transaction containing, for example, 128 G2 MSM pairs or many pairing checks would be accepted by the forward system but could exceed the prover's cycle budget, making the block unprovable.

**Resource accounting bypass.** The native resource is the mechanism that prevents a single transaction from consuming more proving capacity than it paid for. With `cost_native = 0`, BLS12-381 calls are effectively free from the prover's perspective, allowing an attacker to inflate the proving cost of a block without limit.

---

### Likelihood Explanation

Any unprivileged transaction sender can call BLS12-381 precompile addresses (0x0b–0x11) directly. No special role, governance access, or oracle manipulation is required. The attacker only needs to craft a transaction with sufficient EVM gas to call, e.g., `BLS12_PAIRING_CHECK` with the maximum number of pairs that fit within the block gas limit. The EVM gas cost is charged correctly; only the native resource is missing.

---

### Recommendation

Add non-zero native cost constants for all BLS12-381 precompiles in `basic_system/src/cost_constants.rs`, following the same pattern as `BN254_PAIRING_BASE_NATIVE_COST` and `BN254_PAIRING_PER_PAIR_NATIVE_COST`. Then pass those constants instead of the literal `0` in each precompile's `execute` function. The constants should be benchmarked against actual RISC-V cycle counts for each operation (G1Add, G2Add, G1MSM per point, G2MSM per point, pairing fixed + per-pair, MapFpToG1, MapFp2ToG2).

---

### Proof of Concept

1. Attacker sends a transaction to `BLS12_PAIRING_CHECK` (address `0x0f`) with the maximum number of valid G1/G2 pairs that fit within the block gas limit.
   - EVM gas per pair: `32,600`; fixed: `37,700`. At a 30 M gas block limit: ~919 pairs.
2. The precompile charges `(919 * 32600 + 37700) * 256` ergs — within the gas limit.
3. `cost_native = 0` is charged for native resources — the `MAX_NATIVE_COMPUTATIONAL` cap is never approached.
4. The forward system marks the transaction as successful.
5. The prover must execute 919 BLS12-381 pairings in RISC-V. BLS12-381 pairing is significantly more expensive than BN254 pairing (`BN254_PAIRING_BASE_NATIVE_COST ≈ 13 M` native units per pair). The actual proving cost far exceeds `MAX_NATIVE_COMPUTATIONAL` (2³⁵ ≈ 34 B), causing the block to be unprovable or causing forward/proving divergence. [8](#0-7) [9](#0-8)

fn compute_cost(
    input_len: usize,
    pair_len: usize,
    per_pair_cost: u64,
    discounts_table: &[u16; 128],
) -> u64 {
    let num_pairs = input_len / pair_len;
    if num_pairs == 0 {
        return 0;
    }
    let discount = if num_pairs > 128 {
        discounts_table[127]
    } else {
        discounts_table[num_pairs - 1]
    };

    (per_pair_cost * num_pairs as u64) * (discount as u64) / (DISCOUNT_DENOMINATOR as u64)
}

pub const BLS12_381_PAIRING_FIXED_GAS: u64 = 37700;
pub const BLS12_381_PAIRING_PER_PAIR_GAS: u64 = 32600;

pub const BLS12_381_PAIR_LEN: usize = G1_SERIALIZATION_LEN + G2_SERIALIZATION_LEN;

pub struct Bls12381PairingCheckPrecompile;

impl<R: Resources> SystemFunction<R, Bls12PrecompileErrors> for Bls12381PairingCheckPrecompile {
    fn execute<
        D: zk_ee::common_traits::TryExtend<u8> + ?Sized,
        A: core::alloc::Allocator + Clone,
    >(
        input: &[u8],
        output: &mut D,
        resources: &mut R,
        allocator: A,
    ) -> Result<(), zk_ee::system::errors::subsystem::SubsystemError<Bls12PrecompileErrors>> {
        if input.len() == 0 {
            return Err(interface_error!(
                Bls12PrecompileInterfaceError::InvalidInputSize
            ));
        }
        let num_pairs = input.len() / BLS12_381_PAIR_LEN;
        let cost_ergs = Ergs(
            ((num_pairs as u64) * BLS12_381_PAIRING_PER_PAIR_GAS + BLS12_381_PAIRING_FIXED_GAS)
                * ERGS_PER_GAS,
        );
        let cost_native = 0;
        resources.charge(&R::from_ergs_and_native(
            cost_ergs,
            <R::Native as zk_ee::system::Computational>::from_computational(cost_native),
        ))?;
