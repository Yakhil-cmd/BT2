### Title
`MsgTurnBridge` Handler Is a No-Op Stub — Auth Bypass and Permanent Bridge Circuit-Breaker Failure - (File: `x/cronos/keeper/msg_server.go`)

---

### Summary

The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is implemented as a stub that unconditionally returns `nil, nil`. It performs no permission check and applies no state change. Any unprivileged sender can call `MsgTurnBridge` and receive a success response, bypassing the authorization gate that should be there. Simultaneously, the bridge-enabled flag can never actually be toggled, permanently disabling the emergency circuit-breaker that is the only mechanism to halt the Gravity Bridge in an exploit scenario.

---

### Finding Description

`MsgTurnBridge` is the Cronos module's emergency circuit-breaker. Its proto definition declares it as a permissioned operation that sets a boolean `enable` flag to stop or restart the Gravity Bridge:

```proto
// TurnBridge defines a method to disable or enable the gravity bridge
rpc TurnBridge(MsgTurnBridge) returns (MsgTurnBridgeResponse);
```

The Go handler in `msg_server.go` is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

Two defects are present simultaneously:

1. **No permission check.** Every other privileged handler in the same file checks authorization before proceeding (e.g., `UpdateTokenMapping` calls `k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping)`, `UpdatePermissions` and `StoreBlockList` compare `msg.From` against `admin`). `TurnBridge` skips this entirely, so any unprivileged address that submits a `MsgTurnBridge` transaction receives a success response.

2. **No state mutation.** The handler never reads or writes the bridge-enabled parameter. The bridge state is therefore frozen at its genesis/default value and can never be changed by any caller, including the authorized admin.

---

### Impact Explanation

**Auth bypass (High):** The permission guard that should restrict `MsgTurnBridge` to addresses holding `CanTurnBridge` permission is absent. An unprivileged user submitting this message receives `code: 0` (success), which constitutes a bypass of the Cronos bridge authorization check.

**Permanent inability to halt the bridge (High):** The Gravity Bridge circuit-breaker is the sole on-chain mechanism to stop bridge operations during an active exploit. Because the handler is a no-op, the bridge-enabled flag is permanently stuck. Even the authorized admin or governance cannot disable the bridge, leaving all bridge assets exposed for the full duration of any ongoing attack.

Both impacts fall within the allowed scope:
- *High: Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks.*
- *High: Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows, precompile calls, IBC transfers, or block proposals under normal network assumptions.*

---

### Likelihood Explanation

The entry path is a standard Cosmos SDK message transaction, reachable by any unprivileged account with enough gas. No special keys, validator access, or cryptographic assumptions are required. The stub is present in the production message server and is registered in the live gRPC service descriptor, making it immediately callable on-chain.

---

### Recommendation

Implement `TurnBridge` with the same pattern used by the other privileged handlers in `msg_server.go`:

1. **Add a permission check** — verify the sender holds `CanTurnBridge` (or equivalent) before proceeding, consistent with how `UpdateTokenMapping` uses `k.HasPermission`.
2. **Apply the state change** — persist `msg.Enable` to the module parameters (e.g., via `k.SetParams`) so that downstream bridge logic (EVM hooks, end-blocker) can read and enforce the enabled/disabled state.

func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
}
