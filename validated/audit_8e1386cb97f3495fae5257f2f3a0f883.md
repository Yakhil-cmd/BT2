### Title
Missing Lower-Bound Check on Threshold in FROST Presign and Sign Initialization — (File: `src/frost/mod.rs`)

### Summary

The `presign` and `assert_sign_inputs` functions in `src/frost/mod.rs` validate that `threshold <= participants.len()` but omit the symmetric lower-bound check `threshold >= 2`. The DKG entry-point `assert_key_invariants` in `src/dkg.rs` enforces both bounds. A caller who passes `threshold = 1` to the FROST presign or sign entry-points bypasses the minimum-threshold guard, causing the signing protocol to execute with a cryptographically invalid threshold and producing a corrupted or unverifiable signature.

---

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces both bounds on the threshold:

```rust
// Step 1.1 – validate threshold
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
// Step 1.1
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

The FROST `presign` function in `src/frost/mod.rs` only checks the upper bound:

```rust
// validate threshold
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
``` [2](#0-1) 

`assert_sign_inputs`, also in `src/frost/mod.rs`, has the same gap:

```rust
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
``` [3](#0-2) 

Neither function rejects `threshold = 0` or `threshold = 1`. With `participants.len() >= 2`, the check `1 > 2` is `false`, so both functions return `Ok` and the protocol proceeds.

The `PresignArguments<C>` struct stores the threshold alongside the keygen output and passes it directly into the signing pipeline: [4](#0-3) 

---

### Impact Explanation

**High — Corruption of signing outputs so honest parties accept unusable cryptographic outputs.**

The DKG polynomial has degree `threshold_DKG − 1`. When the FROST sign protocol runs with `threshold_sign = 1` while `threshold_DKG = 2`, the Lagrange interpolation uses coefficients computed for a degree-0 polynomial over the actual signing set. The resulting aggregate scalar `z` does not satisfy the standard Schnorr equation `z·G = R + c·X`, so the produced signature fails external verification. Every honest participant who accepted the protocol output receives an unusable, invalid signature. This matches the allowed impact: *"Corruption of … sign … outputs so honest parties accept … unusable cryptographic outputs."*

Additionally, a malicious coordinator who controls the threshold argument can force the signing session to require only one share, effectively reducing the security threshold below the value agreed upon during DKG, which is a permanent denial of valid signing for the honest quorum.

---

### Likelihood Explanation

The `presign` and `assert_sign_inputs` functions are public API entry-points. Any library caller — including a malicious coordinator who orchestrates the signing session — supplies the `threshold` argument directly. No special privilege is required beyond being the party that initiates the signing call. The missing check is a single missing `if threshold < 2` guard, identical in structure to the one already present in `assert_key_invariants`.

---

### Recommendation

Add the lower-bound check to both `presign` and `assert_sign_inputs` in `src/frost/mod.rs`, mirroring the existing check in `assert_key_invariants`:

```rust
// In presign():
if args.threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: args.threshold.value(),
        min: 2,
    });
}

// In assert_sign_inputs():
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

---

### Proof of Concept

1. Run DKG with `participants = [P1, P2, P3]`, `threshold = 2` (valid; `assert_key_invariants` accepts it).
2. Call `frost::presign(&[P1, P2, P3], P1, &PresignArguments { keygen_out, threshold: 1.into() }, rng)`.
   - Check `1 > 3` → false → no error. Protocol proceeds.
3. Call `assert_sign_inputs(&[P1, P2, P3], 1usize, P1, P1)`.
   - Check `1 > 3` → false → no error. Returns `Ok`.
4. The sign protocol executes with `threshold = 1`. Lagrange coefficients are computed for a degree-0 polynomial. The aggregate signature scalar `z` is cryptographically inconsistent with the DKG public key. External verification fails.
5. Contrast: calling `keygen(&[P1, P2], P1, 1usize, rng)` correctly returns `Err(ThresholdTooSmall { threshold: 1, min: 2 })` because `assert_key_invariants` has the lower-bound guard. [5](#0-4) [6](#0-5) [7](#0-6)

pub fn assert_key_invariants(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
) -> Result<ParticipantList, InitializationError> {
    let threshold = usize::from(threshold.into());
    // need enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // Step 1.1
    // validate threshold
    if threshold > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold,
            max: participants.len(),
        });
    }
    // Step 1.1
    if threshold < 2 {
        return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
    }

    // ensure uniqueness of participants in the participant list
    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }
    Ok(participants)
}

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
