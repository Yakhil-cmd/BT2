### Title
Missing Minimum-Threshold Lower-Bound Check in FROST Signing and Presign Entry Points — (`src/frost/mod.rs`)

---

### Summary

`assert_key_invariants` (the DKG entry-point guard) explicitly rejects `threshold < 2`. The analogous guards for FROST signing (`assert_sign_inputs`) and FROST presigning (`presign`) omit this check entirely. A caller can therefore supply `threshold = 1` to the signing/presign path, receive no error, and proceed into a protocol run whose cryptographic outputs are inconsistent with the key material produced by DKG — directly mirroring the "check applied in one place but not another" pattern of the reference report.

---

### Finding Description

**DKG path — check is present:**

`assert_key_invariants` in `src/dkg.rs` enforces both an upper and a lower bound on the threshold:

```rust
// src/dkg.rs  lines 573-582
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

**FROST signing path — lower-bound check is absent:**

`assert_sign_inputs` in `src/frost/mod.rs` validates the upper bound (`threshold > participants.len()`) but never validates the lower bound:

```rust
// src/frost/mod.rs  lines 144-150
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
// ← no ThresholdTooSmall guard here
``` [2](#0-1) 

**FROST presign path — lower-bound check is also absent:**

`presign` in `src/frost/mod.rs` repeats the same pattern — upper-bound only:

```rust
// src/frost/mod.rs  lines 71-77
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
// ← no ThresholdTooSmall guard here
``` [3](#0-2) 

The `assert_reshare_keys_invariants` function in `src/dkg.rs` has the same gap for `old_threshold`: it delegates the *new* threshold to `assert_key_invariants` (which does check `< 2`), but `old_threshold` is used raw in the intersection-size comparison with no lower-bound validation. [4](#0-3) 

---

### Impact Explanation

When a caller passes `threshold = 1` to `assert_sign_inputs` or `presign`:

1. Both functions return `Ok` — no error is surfaced.
2. The caller legitimately believes the configuration is valid and proceeds to run the signing protocol.
3. Inside the protocol, Lagrange interpolation is performed over the supplied participant set. Because the DKG was necessarily run with `threshold ≥ 2` (enforced by `assert_key_invariants`), the shares are evaluations of a polynomial of degree ≥ 1. Signing with `threshold = 1` causes the reconstruction to use coefficients computed for a degree-0 (constant) polynomial, producing a scalar that does not equal the actual secret key.
4. The resulting signature fails external verification — honest parties have completed the protocol and accepted an output that is cryptographically unusable.

This maps to the allowed impact: **High — Corruption of signing outputs so honest parties accept unusable cryptographic outputs.**

---

### Likelihood Explanation

`assert_sign_inputs` is a public, exported function explicitly documented as the pre-flight check callers must invoke before signing. Any integrator who mirrors the DKG pattern (where `threshold = 1` is the only obviously-invalid small value) and tests with `threshold = 1` will observe no error from the signing guard, conclude the value is acceptable, and ship broken signing logic. The inconsistency is a direct API-contract violation that is easy to trigger accidentally.

---

### Recommendation

Add the same lower-bound guard to both `assert_sign_inputs` and `presign` in `src/frost/mod.rs`, mirroring `assert_key_invariants`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Apply the same fix to the `old_threshold` path in `assert_reshare_keys_invariants` in `src/dkg.rs`.

---

### Proof of Concept

1. DKG is run with `participants = [P1, P2, P3]`, `threshold = 3`. `assert_key_invariants` enforces `threshold ≥ 2` — succeeds.
2. Caller invokes `assert_sign_inputs([P1, P2, P3], threshold = 1, me = P1, coordinator = P1)`.
3. Check `1 > 3` → false; check `participants.len() < 2` → false; self/coordinator present → passes. **Returns `Ok`.**
4. Caller proceeds to run the FROST signing protocol with `threshold = 1` over the three participants.
5. Lagrange coefficients are computed assuming a degree-0 sharing; the reconstructed nonce and key scalar are wrong.
6. The coordinator assembles a signature `(R, s)` that fails `verify(public_key, msg, sig)` — the protocol completed without error but produced an unusable output.
7. Contrast: calling `keygen([P1, P2, P3], threshold = 1, …)` immediately returns `Err(ThresholdTooSmall { threshold: 1, min: 2 })` — the inconsistency is explicit. [5](#0-4) [6](#0-5) [7](#0-6)

/// Verifies that the sign inputs are valid
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
