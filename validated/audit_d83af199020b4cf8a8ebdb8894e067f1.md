### Title
Missing `threshold >= 2` Enforcement in OT-Based ECDSA `presign` Allows Corrupted Presignature Output - (File: `src/ecdsa/ot_based_ecdsa/presign.rs`)

---

### Summary

The `presign` function in `src/ecdsa/ot_based_ecdsa/presign.rs` enforces `threshold <= participants.len()` but omits the `threshold >= 2` minimum check that is consistently enforced in both `validate_triple_inputs` (triple generation) and `assert_key_invariants` (keygen). A caller who supplies `PresignArguments` with a `TriplePub` carrying `threshold = 1` can invoke presign with `threshold = 1`, bypassing the minimum threshold constraint. The resulting presignature is cryptographically corrupted because Lagrange interpolation uses degenerate single-participant coefficients, producing an unusable output. The same omission exists in `assert_sign_inputs` in `src/frost/mod.rs`.
