### Title
Signature Validation Bypass via `skip_stateful_validations` Overly Broad `account_tx_in_pool_or_recent_block` Check - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator skips the `__validate__` entry-point call (signature check) for an invoke transaction with `tx_nonce == 1` and `account_nonce == 0` whenever `account_tx_in_pool_or_recent_block` returns `true`. That helper returns `true` for **any** account that has **any** transaction in the mempool pool — not only accounts with a pending `deploy_account` transaction. An attacker who first inserts a valid invoke transaction at nonce ≥ 2 for an already-deployed account (on-chain nonce = 0) can then submit a second invoke transaction at nonce = 1 carrying an **invalid signature**, and the gateway will admit it to the mempool without ever calling `__validate__`.

The code comment in `skip_stateful_validations` states the check is sufficient because the account "either has a deploy_account transaction **or** transactions with future nonces that passed validations." The second branch is the flaw: a future-nonce invoke tx having passed `__validate__` does **not** imply that a different nonce-1 tx with a different (invalid) signature will also pass `__validate__`. The two transactions carry independent signatures.

When `skip_stateful_validations` returns `true`, `run_validate_entry_point` sets `execution_flags.validate = false`:

```rust
let execution_flags =

Inside `StatefulValidator::perform_validations`, when `validate = false` the function returns immediately after `perform_pre_validation_stage`, never reaching the `__validate__` call:

    return Ok(());
}
// `__validate__` call.
let (_optional_call_info, actual_cost) = self.validate(&tx, tx_context.clone())?;
``` [4](#0-3) 

**Precondition**: `max_allowed_nonce_gap ≥ 2` (the gateway nonce check allows future nonces up to `account_nonce + max_allowed_nonce_gap`). [5](#0-4) 

### Impact Explanation

An invoke transaction carrying an **invalid or forged signature** is admitted to the mempool without the account's `__validate__` entry point being called. This breaks the invariant that every transaction in the mempool has passed account-level authorization. The transaction will fail at execution time (the blockifier calls `__validate__` during block building), but it occupies a mempool slot and consumes gateway resources. With `max_allowed_nonce_gap` set to a value ≥ 2 (the default in tests is 10), the attack is repeatable at low cost.

Impact category: **High — Mempool/gateway admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

- Requires an already-deployed account with on-chain nonce = 0 (deployed but never sent a transaction). This is a common state for freshly deployed accounts.
- Requires `max_allowed_nonce_gap ≥ 2`, which is the default configuration.
- The attacker controls both transactions and pays only the fee for the valid nonce-2 tx (the nonce-1 tx with invalid signature will be reverted at execution, but it is still admitted).
- No privileged access is required; any user can submit transactions to the gateway.

### Recommendation

Replace the overly broad `account_tx_in_pool_or_recent_block` check with a check that specifically verifies a **`deploy_account` transaction** is pending for the address. The skip should only apply when the account does not yet exist on-chain and a deploy_account tx is queued, not when any arbitrary future-nonce invoke tx happens to be in the pool.

```rust
// Instead of:
return mempool_client.account_tx_in_pool_or_recent_block(tx.sender_address()).await ...

// Use a dedicated check:
return mempool_client.has_pending_deploy_account(tx.sender_address()).await ...
```

Alternatively, restrict the skip to the case where `account_nonce == 0` **and** the account contract does not yet exist on-chain (class hash at the address is zero), which is the only situation where skipping `__validate__` is semantically safe.

### Proof of Concept

1. Deploy account `A` on-chain (constructor runs, nonce advances to 0 after deployment — account exists, nonce = 0).
2. Submit `invoke_tx_2`: sender = A, nonce = 2, valid signature. This passes `validate_nonce` (0 ≤ 2 ≤ 0 + max_allowed_nonce_gap), passes `validate_by_mempool` (no duplicate), and passes `__validate__` (account exists, signature valid). Tx is stored in `tx_pool`.
3. Submit `invoke_tx_1`: sender = A, nonce = 1, **garbage signature** (e.g., all zeros).
   - `validate_nonce`: 0 ≤ 1 ≤ 0 + gap → passes. [5](#0-4) 
   - `validate_by_mempool`: nonce 1 not in pool → passes. [6](#0-5) 
   - `skip_stateful_validations`: tx_nonce == 1, account_nonce == 0, `account_tx_in_pool_or_recent_block(A)` == **true** (because `invoke_tx_2` is in `tx_pool`) → returns **true**. [7](#0-6) 
   - `run_validate_entry_point` sets `validate = false`; `__validate__` is **never called**. [3](#0-2) 
4. `invoke_tx_1` with invalid signature is now in the mempool. The gateway has accepted an unauthorized transaction.

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
