### Title
Unrestricted `update_rewards` Allows Any Caller to Permanently Freeze Block Rewards for All Stakers - (File: `src/staking/staking.cairo`)

---

### Summary

The `IStakingRewardsManager::update_rewards` function in `src/staking/staking.cairo` has no caller restriction. Any unprivileged address can call it with `disable_rewards=true` to consume the global `last_reward_block` slot for the current block without distributing any rewards, permanently denying block rewards for that block to all stakers.

---

### Finding Description

`update_rewards` is a public function with no access control beyond `general_prerequisites()`, which only checks that the contract is not paused and the caller is not the zero address.

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // global write, before disable check

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // returns with no rewards distributed
    }
    ...
}
```

The critical design flaw is that `last_reward_block` is a **single global storage variable** (not per-staker), and it is written unconditionally at line 1485 — before the `disable_rewards` early-return check at line 1487.

The attack path:
1. Attacker observes a new block N.
2. Attacker calls `update_rewards(any_valid_staker_address, disable_rewards=true)`.
3. The function passes all checks (staker exists, is active, has non-zero balance).
4. `last_reward_block` is set to block N.
5. The function returns early — **no rewards are distributed**.
6. Any subsequent legitimate call to `update_rewards` in block N fails with `REWARDS_ALREADY_UPDATED` because `current_block_number > last_reward_block` is now false.

Since this can be repeated every block, the attacker can permanently prevent all block reward distribution across the entire protocol.

---

### Impact Explanation

This is **permanent freezing of unclaimed yield** for all stakers. Block rewards for every front-run block are permanently lost — they are never credited to any staker's `unclaimed_rewards_own` and never forwarded to any delegation pool. The `last_reward_block` is a global gate; once consumed with `disable_rewards=true`, the reward opportunity for that block is gone forever. This matches the allowed impact: **Permanent freezing of unclaimed yield or unclaimed royalties**.

---

### Likelihood Explanation

Likelihood is **high**:
- The function is fully public with no role check.
- Valid staker addresses are discoverable on-chain (emitted in `NewStaker` events, stored in the public `stakers` vector).
- The attacker needs only to call the function once per block with a valid staker address and `disable_rewards=true`.
- No capital is required; the attacker only pays gas.
- The attack is repeatable every block indefinitely.

---

### Recommendation

Restrict `update_rewards` to an authorized caller. Based on the protocol design (consensus rewards are distributed per block by the block proposer), the function should only be callable by a designated sequencer/proposer address or a trusted contract. Add a check analogous to the existing `assert_caller_is_attestation_contract` pattern:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_authorized_rewards_updater(); // e.g., only block proposer
    ...
}
```

Alternatively, if the function must remain permissionless, the `last_reward_block` write should be moved **after** the `disable_rewards` check, so that a call with `disable_rewards=true` does not consume the block's reward slot.

---

### Proof of Concept

1. Deploy the staking contract with consensus rewards active (`consensus_rewards_first_epoch` set and passed).
2. Wait for a new block N.
3. From any EOA (attacker), call:
   ```
   staking.update_rewards(valid_staker_address, disable_rewards=true)
   ```
4. Observe: `last_reward_block` is now N, no rewards were distributed.
5. From the legitimate block proposer, call:
   ```
   staking.update_rewards(valid_staker_address, disable_rewards=false)
   ```
6. Observe: transaction reverts with `REWARDS_ALREADY_UPDATED`.
7. Staker's `unclaimed_rewards_own` is unchanged — block N rewards are permanently lost.

```text
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
```
