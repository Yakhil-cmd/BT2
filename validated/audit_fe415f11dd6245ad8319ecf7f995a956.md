### Title
`MsgTurnBridge` Handler Has No Permission Check and No Logic, Permanently Breaking the Bridge Circuit Breaker â€” (`File: x/cronos/keeper/msg_server.go`)

### Summary

The `MsgTurnBridge` message handler is completely unimplemented. It performs no authorization check and executes no state change. Any unprivileged user can submit `MsgTurnBridge` and receive a success response, while the bridge state is never actually modified. The bridge circuit breaker is permanently non-functional.

### Finding Description

The Cronos permission system defines a `CanTurnBridge` permission bit specifically to gate the `MsgTurnBridge` message. `MsgUpdateTokenMapping` correctly enforces its analogous permission:

```go
// UpdateTokenMapping implements the grpc method
func (k msgServer) UpdateTokenMapping(...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    ...
}
``` [1](#0-0) 

However, the `TurnBridge` handler is a stub that returns `nil, nil` unconditionally:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [2](#0-1) 

The `CanTurnBridge` permission constant is defined and the permission system is wired up: [3](#0-2) 

ADR-009 explicitly lists `MsgTurnBridge` as one of the two messages that must be gated by the permission system: [4](#0-3) 

The message type, `ValidateBasic`, and CLI command are all fully wired: [5](#0-4) [6](#0-5) 

### Impact Explanation

Two distinct impacts:

1. **Permission bypass (High):** Any unprivileged address can submit `MsgTurnBridge` and receive a `code == 0` success response. The `CanTurnBridge` permission check that the design mandates is entirely absent. This is a direct bypass of the Cronos admin/permission authorization system.

2. **Permanent bridge circuit-breaker failure (High):** Because the handler does nothing, the gravity bridge can never be turned off via `MsgTurnBridge`, even by the admin or a permissioned address. The integration test `test_gravity_turn_bridge` expects that after a successful `turn_bridge("false")` call, subsequent bridge operations fail â€” this invariant is silently violated. [7](#0-6) 

### Likelihood Explanation

The entry path is fully reachable: `MsgTurnBridge` is a registered Cosmos SDK message, callable by any account with gas. No privileged key or special setup is required. The handler is live in production code.

### Recommendation

Implement `TurnBridge` with the same pattern as `UpdateTokenMapping`:

1. Check `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `ErrUnauthorized` on failure.
2. Persist the bridge enable/disable state (e.g., a module param or dedicated KV key).
3. Enforce the bridge state in the gravity bridge end-blocker / outgoing tx logic.

### Proof of Concept

```
# Any unprivileged account can call MsgTurnBridge and receive code=0
cronosd tx cronos turn-bridge false --from=<any_unprivileged_key> -y

# Expected: ErrUnauthorized (code != 0)
# Actual:   code=0, bridge state unchanged, circuit breaker permanently broken
```

The `CanTurnBridge` permission bit exists, `SetPermissions`/`HasPermission` are implemented, and `UpdateTokenMapping` uses them correctly â€” but `TurnBridge` never calls `HasPermission` and never writes any state, making the entire bridge circuit-breaker mechanism inoperative. [2](#0-1) [8](#0-7)

func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {

	}

	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

// CmdTurnBridge returns a CLI command handler for enable or disable the bridge
func CmdTurnBridge() *cobra.Command {
	cmd := &cobra.Command{
		Use:   "turn-bridge [true/false]",
		Short: "Turn Bridge",
		Args:  cobra.ExactArgs(1),
		RunE: func(cmd *cobra.Command, args []string) error {
			clientCtx, err := client.GetClientTxContext(cmd)
			if err != nil {
				return err
			}

			enable, err := strconv.ParseBool(args[0])
			if err != nil {
				return err
			}
			msg := types.NewMsgTurnBridge(clientCtx.GetFromAddress().String(), enable)
			if err := msg.ValidateBasic(); err != nil {
				return err
			}
			return tx.GenerateOrBroadcastTxCLI(clientCtx, cmd.Flags(), msg)
		},
	}

	flags.AddTxFlagsToCmd(cmd)
	return cmd
}
