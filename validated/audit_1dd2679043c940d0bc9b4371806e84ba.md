### Title
Unbounded `stakers` Vec Growth Causes Unbounded Execution Cost in `get_stakers` - (File: `src/staking/staking.cairo`)

### Summary
The `stakers` storage Vec in the Staking contract grows monotonically — every new staker address is appended on `stake()` and **never removed** on `unstake_action()`. The `get_stakers()` function, consumed by the consensus layer every epoch, iterates over the **entire** Vec. Over time this iteration cost grows without bound, eventually exceeding Starknet's execution-step limit and making the consensus query permanently un-callable.

### Finding Description

Every call to `stake()` appends the caller's address to `self.stakers`: [1](#0-0) 

The storage declaration explicitly documents that removal never happens: [2](#0-1) 

`unstake_action()` deletes the staker's info from all other maps but never calls `stakers.pop()` or any equivalent removal. Because `STAKER_ADDRESS_ALREADY_USED` prevents re-use of the same address, every unique staker that has ever participated permanently occupies a slot in the Vec.

`get_stakers()` — the function the consensus layer calls each epoch to build the validator set — iterates over the **full** Vec unconditionally: [3](#0-2) 

Even though inactive stakers are skipped with `continue`, every slot still costs storage-read steps. The loop is explicitly acknowledged as unbounded in the pool's analogous loop: [4](#0-3) 

### Impact Explanation

`get_stakers` is the sole source of truth for the consensus layer's validator set. As the Vec grows, each call to `get_stakers` consumes more execution steps. Once the step count exceeds Starknet's per-call execution limit the function reverts, making it impossible for the consensus layer to retrieve the active staker set. This constitutes **unbounded gas consumption** (Medium) escalating toward a **permanent freeze of the consensus/reward mechanism** if the protocol runs long enough.

### Likelihood Explanation

Any unprivileged actor can call `stake()` with the minimum stake amount from a fresh address, permanently adding one entry to the Vec, then call `unstake_intent()` / `unstake_action()` to recover the capital. Repeating this across many addresses is cheap relative to the permanent damage: each cycle costs only the minimum stake lock-up for the `exit_wait_window` duration. Natural protocol growth (legitimate stakers joining and leaving over years) also inflates the Vec without any adversarial intent.

### Recommendation

Remove the staker address from `self.stakers` inside `unstake_action()` (swap-and-pop pattern), or replace the `Vec` with an `IterableMap` (already used for `btc_tokens`) that supports O(1) removal. Alternatively, change `get_stakers` to iterate only over currently-active stakers tracked in a separate, pruned data structure.

### Proof of Concept

1. Deploy the Staking contract.
2. In a loop, for `N` unique addresses: call `stake(min_stake)`, wait `exit_wait_window`, call `unstake_action()`.
3. After `N` iterations, `self.stakers.length() == N` while zero stakers are active.
4. Call `get_stakers(current_epoch)` — it iterates all `N` slots, each requiring a storage read, even though all are filtered out immediately.
5. As `N` grows, the call's execution-step cost grows linearly until it exceeds the Starknet execution limit and reverts. [5](#0-4)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
