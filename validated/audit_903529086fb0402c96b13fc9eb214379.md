### Title
Presignature One-Time-Use Not Enforced at API Level Enables Secret Key Extraction via Nonce Reuse — (`File: src/ecdsa/mod.rs`, `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both the OT-based ECDSA and Robust ECDSA schemes require that each `PresignOutput` is consumed **exactly once**. However, the library provides no enforcement of this constraint at the API level: `PresignOutput` derives `Clone` and `Serialize/Deserialize`, and `rerandomize_presign` accepts it by shared reference (`&PresignOutput`) rather than consuming it. A malicious coordinator can trivially rerandomize the same presignature for two different messages and run two signing sessions, producing two ECDSA signatures with the same underlying nonce — enabling full secret key extraction via standard nonce-reuse algebra.

---

### Finding Description

The vulnerability class from the external report is **repeated use of a one-time resource without enforcement**: the staking contract allowed `depositTokens` + `withdrawAll` to be called in a loop because no state guard prevented it. The direct analog here is that the library documents presignatures as one-time-use but provides no code-level guard.

**Root cause — OT-based ECDSA:**

`PresignOutput` is freely cloneable and serializable:

```rust
// src/ecdsa/ot_based_ecdsa/mod.rs line 40
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput {
    pub big_r: AffinePoint,
    pub k: Scalar,      // nonce share — must never be reused
    pub sigma: Scalar,
}
```

`rerandomize_presign` takes the presignature by shared reference, leaving the original intact and reusable:

```rust
// src/ecdsa/ot_based_ecdsa/mod.rs line 66
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // <-- shared reference, not consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> { ... }
```

**Root cause — Robust ECDSA:**

Identical pattern:

```rust

### Impact Explanation

**Impact: Critical — Extraction/reconstruction of private signing shares and aggregate secret material.**

When the same presignature nonce `k` (OT-based) or the same `(alpha, beta, c, e)` shares (Robust) are used in two signing sessions for different messages `h1 ≠ h2`, the resulting signature shares satisfy:

```
s1_i = h1 * k_i + R_x * sigma_i   (session 1)
s2_i = h2 * k_i + R_x * sigma_i   (session 2)
```

The coordinator, who receives all shares, can compute:

```
s1 - s2 = (h1 - h2) * k   =>   k = (s1 - s2) / (h1 - h2)
sigma = (s1 - h1 * k) / R_x
```

From `sigma = alpha * x - beta * a + c` (where `x` is the secret key), the private key is fully recoverable. The Robust ECDSA documentation explicitly confirms this:

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks." [7](#0-6) 

---

### Likelihood Explanation

**Likelihood: High** for any deployment where the coordinator role is not fully trusted (which is the stated threat model for threshold signing). The attack requires only:

1. The coordinator calls `rerandomize_presign` twice on the same `PresignOutput` with two different `msg_hash` values — a single function call with a shared reference, trivially repeatable.
2. The coordinator runs two signing sessions and collects all `s_i` shares from honest participants.
3. No participant can detect the reuse because `big_r` is the same in both sessions and the rerandomization produces a different `R'` each time.

The `PresignOutput` type's `Clone` and `Serialize/Deserialize` derives make it straightforward to store and replay presignatures across sessions, including across process restarts. [8](#0-7) [9](#0-8) 

---

### Recommendation

1. **Consume `PresignOutput` by value in `rerandomize_presign`**: Change the signature from `presignature: &PresignOutput` to `presignature: PresignOutput`. This makes the Rust type system enforce one-time use at the call site — the presignature is moved into the function and cannot be used again.

2. **Remove `Clone` from `PresignOutput`** in both schemes. Since presignatures must not be duplicated, `Clone` should not be derived. `Serialize`/`Deserialize` may be retained for storage, but callers should be required to explicitly re-deserialize (and thus consciously re-use) a stored presignature.

3. **Alternatively**, wrap `PresignOutput` in a newtype that tracks usage state and panics or errors on second use. [10](#0-9) [11](#0-10)

### Proof of Concept

```rust
// Attacker is the coordinator. They have run presigning and hold:
//   presign_output: PresignOutput  (OT-based ECDSA)

// Step 1: rerandomize the SAME presignature for two different messages.
// rerandomize_presign takes &PresignOutput — the original is NOT consumed.
let args1 = RerandomizationArguments::new(pk, tweak, msg_hash_1, presign_output.big_r, participants.clone(), entropy1);
let args2 = RerandomizationArguments::new(pk, tweak, msg_hash_2, presign_output.big_r, participants.clone(), entropy2);

let rerand1 = RerandomizedPresignOutput::rerandomize_presign(&presign_output, &args1).unwrap();
let rerand2 = RerandomizedPresignOutput::rerandomize_presign(&presign_output, &args2).unwrap();
// presign_output is still valid here — no enforcement of one-time use.

// Step 2: Run two signing sessions with honest participants.
// Participants sign both sessions, unaware the same nonce is being reused.
let sig1 = run_sign_session(rerand1, msg_hash_1, ...);
let sig2 = run_sign_session(rerand2, msg_hash_2, ...);

// Step 3: Recover the secret key from the two signatures sharing the same nonce.
// k = (s1 - s2) / (h1 - h2)
// x recoverable from sigma = alpha * x - beta * a + c
```

The attack is also possible via `Clone` or deserialization of a stored `PresignOutput`:

```rust
// Clone-based reuse — compiles without warning
let presign_copy = presign_output.clone();
let rerand_a = RerandomizedPresignOutput::rerandomize_presign(&presign_output, &args_a).unwrap();
let rerand_b = RerandomizedPresignOutput::rerandomize_presign(&presign_copy, &args_b).unwrap();

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
