### Title
Staker's Own Balance Passed as Total Stake Denominator in `update_rewards`, Inflating Rewards to 100% of Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` passes the calling staker's own balance (`staker_total_strk_balance`) as the `strk_total_stake` parameter to `_update_rewards`, instead of the total protocol stake. Because `_update_rewards` uses `strk_total_stake` as the denominator for proportional reward computation — as confirmed by the parallel call in `update_rewards_from_attestation_contract` which correctly passes `get_current_total_staking_power()` — any staker can call `update_rewards` once per block and receive 100% of that block's consensus rewards instead of their proportional share.

---

### Finding Description

Two separate code paths call `_update_rewards`. The attestation path passes the correct total protocol stake:

```cairo
// update_rewards_from_attestation_contract (line ~1408)
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
self._update_rewards(
    :staker_address,
    strk_total_rewards: strk_epoch_rewards,
    :strk_total_stake,   // ← total protocol stake (correct denominator)
    ...
);
```

The consensus path (`update_rewards`) passes the staker's own balance instead:

```cairo
// update_rewards (line ~1475)
let (staker_total_strk_balance, staker_total_btc_balance) = self
    .get_staker_total_strk_btc_balance_at_epoch(:staker_address, ...);
self._update_rewards(
    :staker_address,
    strk_total_rewards: strk_block_rewards,
    strk_total_stake: staker_total_strk_balance,  // ← staker's own balance (wrong)
    btc_total_stake: staker_total_btc_balance,
    ...
);
``` [1](#0-0) 

Because `_update_rewards` computes the staker's proportional reward as:

```
staker_reward = staker_balance / strk_total_stake * strk_total_rewards
```

substituting `strk_total_stake = staker_balance` yields:

```
staker_reward = staker_balance / staker_balance * strk_total_rewards = strk_total_rewards
```

The staker receives **100% of block rewards** regardless of their actual share of total stake.

The function is publicly callable (only `general_prerequisites()` is checked, which enforces non-zero caller and non-paused state — no role restriction): [2](#0-1) 

The single-block guard (`last_reward_block`) prevents multiple calls per block but does not prevent the exploit across blocks: [3](#0-2) 

The correct denominator is used in the attestation path for comparison: [4](#0-3) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Any staker, regardless of stake size, can call `update_rewards` once per block during the consensus rewards phase and accumulate 100% of that block's STRK (and BTC) block rewards into their `unclaimed_rewards_own`. Over time this drains the entire consensus reward stream away from all other stakers and delegators. The stolen yield is claimable via `claim_rewards`.

---

### Likelihood Explanation

**High.** The function is permissionless, requires no special role, no flash loan, and no protocol state precondition beyond the consensus rewards phase being active (`!is_pre_consensus()`). Any registered staker with a non-zero balance can exploit this every block. The only cost is the gas for one transaction per block.

---

### Recommendation

Replace the staker's own balance with the total protocol staking power in the `update_rewards` call to `_update_rewards`, consistent with `update_rewards_from_attestation_contract`:

```cairo
// Before (incorrect):
strk_total_stake: staker_total_strk_balance,
btc_total_stake: staker_total_btc_balance,

// After (correct):
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
// ... pass strk_total_stake and btc_total_stake to _update_rewards
``` [5](#0-4) 

---

### Proof of Concept

1. Attacker stakes the minimum required amount via `stake(...)`.
2. After K epochs (when the stake becomes effective), the consensus rewards phase is active.
3. Each block, attacker calls `update_rewards(staker_address: attacker_address, disable_rewards: false)`.
4. `_update_rewards` computes `attacker_reward = attacker_balance / attacker_balance * block_rewards = block_rewards`.
5. Attacker's `unclaimed_rewards_own` accumulates 100% of every block's consensus rewards.
6. Attacker calls `claim_rewards(staker_address: attacker_address)` to withdraw the stolen yield.

All other stakers and delegators receive zero consensus rewards for every block the attacker calls `update_rewards` first. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
