### Title
Missing Slippage Protection in `switch_delegation_pool()` Allows Commission Front-Running - (`src/pool/pool.cairo`)

---

### Summary

`switch_delegation_pool()` in the Pool contract accepts no `max_commission` parameter. A staker holding an active `CommissionCommitment` can increase their commission between the moment a delegator signs a pool-switch transaction and the moment it is included, causing the delegator to enter the destination pool at a higher commission rate than they evaluated.

---

### Finding Description

When a pool member calls `switch_delegation_pool(to_staker, to_pool, amount)`, they select the destination pool based on its current commission rate. The function performs no check that the destination pool's commission is still within the delegator's acceptable range at execution time. [1](#0-0) 

The staking contract's `set_commission()` allows a staker to change their commission at any time. Without a `CommissionCommitment`, commission can only decrease. However, when an active `CommissionCommitment` exists, the staker may set commission to **any value up to `max_commission`**, including values higher than the current rate. [2](#0-1) 

The `CommissionCommitment` is set via `set_commission_commitment()`, which requires only that `current_commission <= max_commission`. A staker can therefore set a low commission (e.g. 5%) to attract delegators, establish a commitment with `max_commission = 50%`, and then increase commission to 50% at any time while the commitment is active. [3](#0-2) 

**Attack scenario:**

1. Staker sets commission = 5% and a `CommissionCommitment` with `max_commission = 50%`.
2. Delegator observes 5% commission and submits `switch_delegation_pool(to_staker, to_pool, amount)`.
3. Before the delegator's transaction is sequenced, the staker calls `set_commission(50%)`.
4. The delegator's transaction executes and they are now a member of a pool charging 50% commission.
5. The delegator must wait the full `exit_wait_window` (at minimum one week) before they can exit, during which all rewards are subject to the 50% commission. [4](#0-3) 

The commission is applied at reward distribution time and takes effect immediately upon the `set_commission()` call, with no epoch delay. [5](#0-4) 

---

### Impact Explanation

The delegator loses a portion of their yield for the entire exit window duration. The staker captures the excess commission. This constitutes theft of unclaimed yield: the delegator would have earned rewards at 5% commission but instead earns them at 50%, with the difference extracted by the staker. The delegator has no recourse during the exit window.

---

### Likelihood Explanation

The preconditions are:
- The destination staker must have an active `CommissionCommitment` with `max_commission` above the current commission (a feature explicitly designed for stakers to attract delegators).
- On Starknet, the sequencer orders transactions, making it feasible for a staker to submit a `set_commission()` call that is sequenced before an observed `switch_delegation_pool()` call.

Both conditions are realistic in production. The `CommissionCommitment` feature is specifically designed to be used by stakers competing for delegators, making the low-commission-then-increase pattern a plausible strategy.

---

### Recommendation

Add a `max_commission` parameter to `switch_delegation_pool()` and assert that the destination pool's current commission does not exceed it at execution time:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
    max_commission: Commission,  // <-- add this
) -> Amount {
    ...
    // After resolving to_staker's commission:
    let current_commission = self.get_commission_from_staking_contract_for(to_staker);
    assert!(current_commission <= max_commission, "COMMISSION_EXCEEDS_MAX");
    ...
}
```

The same protection should be considered for `enter_delegation_pool()`. [1](#0-0) [6](#0-5) 

---

### Proof of Concept

1. Staker A stakes and opens a delegation pool with commission = 500 (5%).
2. Staker A calls `set_commission_commitment(max_commission: 5000, expiration_epoch: current + 10)`.
3. Delegator D calls `exit_delegation_pool_intent(amount)` on their current pool, then submits `switch_delegation_pool(staker_A, pool_A, amount)` targeting the 5% pool.
4. Staker A observes the pending transaction and calls `set_commission(5000)` (50%), which is sequenced first.
5. Delegator D's `switch_delegation_pool` executes successfully — no commission check exists — and D is now in a 50% commission pool.
6. D must wait `exit_wait_window` (≥ 1 week) to exit, losing 45% of all rewards earned during that period to Staker A. [7](#0-6) [8](#0-7)

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
