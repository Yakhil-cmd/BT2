### Title
Malformed Message Body Causes Unrecoverable Protocol Abort via `recv_from_others` — (`src/protocol/helpers.rs`, `src/protocol/internal.rs`)

### Summary

A malicious participant (or any caller of `Protocol::message`) can inject a message with a syntactically valid `MessageHeader` but an invalid msgpack body. `Comms::push_message` accepts it without body validation, and `Comms::recv` propagates the deserialization error via `?` before `recv_from_others` can apply its participant-filtering guard. This permanently aborts the honest participant's DKG (or any protocol using `recv_from_others`).

---

### Finding Description

**Step 1 — Entry point: `Protocol::message` / `Comms::push_message`**

`push_message` performs only two checks on an incoming message:
- `message.len() < MessageHeader::LEN` → drop
- `MessageHeader::from_bytes` fails → drop

If both pass, the raw bytes (including the unvalidated body) are pushed into the `MessageBuffer` keyed by the parsed header. There is no check that `from` is a legitimate participant, and no validation of the payload. [1](#0-0) 

**Step 2 — Deserialization in `Comms::recv`**

When the protocol calls `chan.recv(waitpoint)`, `Comms::recv` pops the buffered message and immediately attempts `rmp_serde::decode::from_slice` on the body. A failure is converted to `ProtocolError::Other` via the blanket `From<Box<dyn error::Error>>` impl and propagated with `?`. [2](#0-1) [3](#0-2) 

**Step 3 — No recovery in `recv_from_others`**

`recv_from_others` calls `chan.recv(waitpoint).await?` inside its collection loop. The `?` propagates any error immediately — the `seen.put(from)` participant-list guard on line 21 is never reached for a malformed message, so it cannot filter out the attacker's message before the abort. [4](#0-3) 

**Step 4 — `ParticipantCounter::put` cannot help**

Even if the attacker uses a `from` ID that is not in the participant list, `put` would return `false` — but only if execution reaches line 21. Since deserialization on line 20 aborts first, this guard is irrelevant. [5](#0-4) 

---

### Impact Explanation

Any protocol that calls `recv_from_others` — including DKG — is permanently terminated for the honest participant. The `ProtocolError` propagates all the way to the caller of `Protocol::poke`, which returns `Action::Return(Err(...))`, ending the protocol with no possibility of resumption. This matches **High: Permanent denial of key generation for honest parties**.

---

### Likelihood Explanation

- A single malicious participant (1-of-n) is sufficient.
- The attacker only needs to know the deterministic `ChannelTag` (SHA-256 of a public domain string) and the sequential waitpoint value — both are fully predictable from the public protocol spec.
- The attack requires sending exactly one message: 40 bytes of valid header followed by any non-msgpack garbage.
- No cryptographic assumption needs to be broken.

---

### Recommendation

In `recv_from_others`, catch deserialization errors and skip the offending message rather than aborting:

```rust
while !seen.full() {
    let recv_result = chan.recv(waitpoint).await;
    let (from, msg) = match recv_result {
        Ok(pair) => pair,
        Err(_) => continue,  // skip malformed messages
    };
    if seen.put(from) {
        messages.push((from, msg));
    }
}
```

Alternatively, move deserialization after the `seen.put(from)` check so that messages from unknown or already-seen participants are discarded before any parsing is attempted.

---

### Proof of Concept

```rust
// Attacker constructs: valid 40-byte header (root_shared channel, waitpoint=2) + garbage body
let mut malicious_msg = Vec::new();
// root_shared ChannelTag = SHA256(NEAR_CHANNEL_TAGS_DOMAIN || "root shared")
malicious_msg.extend_from_slice(&root_shared_channel_tag_bytes); // 32 bytes
malicious_msg.extend_from_slice(&2u64.to_le_bytes());            // 8 bytes waitpoint
malicious_msg.extend_from_slice(b"\xff\xfe\xfd\xfc");            // invalid msgpack

// Inject via the public Protocol::message interface
honest_participant_protocol.message(attacker_participant_id, malicious_msg.into());

// Next poke() call drives the future to chan.recv(2), which pops this message,
// fails rmp_serde deserialization, and returns ProtocolError::Other(...).
// recv_from_others propagates via ?, permanently aborting DKG.
let result = honest_participant_protocol.poke();
assert!(matches!(result, Ok(Action::Return(Err(ProtocolError::Other(_))))));
```
