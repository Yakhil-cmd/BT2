### Title
Missing Caller Restriction on `update_rewards` Allows Anyone to Permanently Freeze Staker Yield - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is intended to be callable only by the Starkware sequencer (per the protocol specification), but no caller check is implemented. Any unprivileged address can call it, consuming the single per-block reward slot (`last_reward_block`) without distributing rewards, permanently preventing all stakers from accruing yield under the V3 consensus rewards mechanism.

---

### Finding Description

The protocol specification for `update_rewards` explicitly states:

> **Access control:** Only starkware sequencer. [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` performs no caller identity check whatsoever. It only checks the contract pause state (via `general_prerequisites()`) and a per-block guard: [2](#0-1) 

Critically, `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

This means any caller can invoke `update_rewards(valid_staker_address, disable_rewards: true)`. The call will:
1. Pass all checks (contract unpaused, block is new).
2. Write `current_block_number` into `last_reward_block`.
3. Return immediately due to `disable_rewards == true`, distributing **zero rewards**.

Any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

---

### Impact Explanation

Under the V3 consensus rewards mechanism, `update_rewards` is the sole path through which block rewards are credited to stakers' `unclaimed_rewards_own` and forwarded to delegation pools. The `last_reward_block` guard is a **global, single-slot** lock — only one call succeeds per block.

An attacker who calls `update_rewards(..., disable_rewards: true)` at the start of every block:
- Consumes the reward slot for that block with zero reward distribution.
- Causes the sequencer's legitimate call to revert every block.
- Permanently freezes all staker and delegator unclaimed yield.

Alternatively, an attacker who is themselves a staker can call `update_rewards(attacker_staker_address, disable_rewards: false)` every block, directing all block rewards exclusively to their own staker, constituting **theft of unclaimed yield** from all other stakers.

**Impact category matched:** High — Permanent freezing of unclaimed yield; Theft of unclaimed yield.

---

### Likelihood Explanation

- No special privilege, token balance, or prior setup is required. Any EOA on Starknet can call this function.
- The attack is trivially repeatable every block at negligible cost.
- The attacker only needs to submit their transaction before the sequencer's `update_rewards` call in each block, which is straightforward given that the sequencer's call is predictable and observable.

---

### Recommendation

Add a caller restriction to `update_rewards` that mirrors the pattern already used for other privileged entry points in the same contract. Introduce a stored `sequencer_address` (or reuse an existing role) and assert it at the top of the function:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

This mirrors the pattern already correctly applied to `update_rewards_from_attestation_contract` (which asserts `assert_caller_is_attestation_contract()`): [5](#0-4) 

---

### Proof of Concept

```cairo
// Any unprivileged address can execute this.
// Step 1: Find any valid, active staker address (publicly readable from contract state).
let victim_staker: ContractAddress = ...; // any registered staker

// Step 2: Call update_rewards with disable_rewards=true at the start of every block.
//         This consumes last_reward_block without distributing any rewards.
IStakingRewardsManagerDispatcher { contract_address: staking_contract }
    .update_rewards(staker_address: victim_staker, disable_rewards: true);

// Step 3: The sequencer's legitimate update_rewards call in the same block now reverts:
//   assert!(current_block_number > self.last_reward_block.read(), ...)
//   => panics with REWARDS_ALREADY_UPDATED

// Repeating this every block permanently freezes all staker and pool rewards.
```

The root cause is the absence of any `get_caller_address()` check in `update_rewards`, confirmed at: [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
