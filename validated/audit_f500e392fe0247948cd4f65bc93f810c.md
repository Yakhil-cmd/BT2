### Title
Unprivileged caller can permanently freeze all stakers' block rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control. It accepts a caller-controlled `disable_rewards: bool` parameter. When `true`, the function updates the global `last_reward_block` to the current block and returns early — without distributing any rewards. Because `last_reward_block` is a single global gate that prevents more than one call per block, an attacker calling this function once per block with `disable_rewards: true` permanently prevents all stakers from ever receiving consensus block rewards.

---

### Finding Description

`update_rewards` is exposed as a public function under `IStakingRewardsManager`. Its only access guard is `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

The function writes `last_reward_block` to the current block number **before** checking `disable_rewards`: [2](#0-1) 

The guard that prevents a second call in the same block: [3](#0-2) 

Because `last_reward_block` is a single global value (not per-staker), one call per block is sufficient to lock out every staker in the system. The attacker only needs to supply any currently-active staker address with non-zero balance (trivially discoverable on-chain via the `stakers` vector and public events). [4](#0-3) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

In consensus rewards mode (V3), `update_rewards` is the sole mechanism by which block rewards are credited to stakers. An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` at the start of every block:

1. Advances `last_reward_block` to the current block.
2. Returns without crediting any rewards to any staker.
3. Blocks every legitimate call for that block (the `REWARDS_ALREADY_UPDATED` assertion fires for all subsequent callers).

Repeated every block, this permanently freezes all block reward accrual for all stakers and their delegators. No funds already staked are stolen, but all future yield is permanently lost. [5](#0-4) 

---

### Likelihood Explanation

**High.** The attack requires:
- No privileged role or special token balance.
- Only gas fees (one transaction per block).
- A single valid active staker address, which is publicly readable from the `stakers` storage vector and emitted in `NewStaker` events.

The attack is fully automatable with a simple script. Any competitor, griever, or protocol adversary can execute it indefinitely at negligible cost. [6](#0-5) 

---

### Recommendation

Restrict who may call `update_rewards` and/or who may set `disable_rewards: true`. Options:

1. **Restrict the caller**: Only allow the staker themselves (or a designated operator) to call `update_rewards` for their own address.
2. **Remove the `disable_rewards` parameter from the public interface**: If `disable_rewards` is only needed during protocol transitions, gate it behind a privileged role (e.g., `only_app_governor`).
3. **Decouple `last_reward_block` update from the `disable_rewards` path**: Only advance `last_reward_block` when rewards are actually distributed, so a no-op call does not consume the block's reward slot.

---

### Proof of Concept

```
// Attacker script (pseudocode, runs every block):
loop every block:
    staking_contract.update_rewards(
        staker_address = <any_known_active_staker>,
        disable_rewards = true
    )
// Cost: gas only (~1 tx/block)
// Effect: last_reward_block = current_block for every block
//         No staker ever receives consensus block rewards again
```

The attacker needs no tokens beyond gas. Any active staker address (e.g., read from `stakers` Vec or `NewStaker` events) satisfies the precondition checks at lines 1460–1482. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
