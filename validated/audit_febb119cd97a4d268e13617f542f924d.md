### Title
Commission Change Without Reward Checkpoint Retroactively Applies New Rate to Entire Epoch - (File: `src/staking/staking.cairo`)

---

### Summary

`set_commission` in `src/staking/staking.cairo` overwrites the commission immediately without first checkpointing rewards for the elapsed epoch. When `update_rewards_from_attestation_contract` is subsequently called, it reads the **current** commission and applies it to the **entire epoch's** pool rewards — not just the portion after the commission change. A staker holding a commission commitment can increase commission mid-epoch, then attest, causing the new higher rate to be applied retroactively to all delegator rewards for that epoch.

---

### Finding Description

In `set_commission` (line 725), the internal `update_commission` helper (line 1573) writes the new commission directly to storage:

```cairo
// src/staking/staking.cairo ~line 1599-1600
staker_pool_info.commission.write(Option::Some(commission));
```

No reward checkpoint is performed before this write. [1](#0-0) 

Later, when the staker calls `attest` on the attestation contract, it triggers `update_rewards_from_attestation_contract` → `_update_rewards` → `calculate_staker_pools_rewards`. At line 1964, the commission is read from storage at the time of the call:

```cairo
let commission = staker_pool_info.commission();
```

This commission is then applied to `strk_total_rewards`, which represents the **entire epoch's** STRK rewards — not just the rewards accrued after the commission change. [2](#0-1) 

The split is performed at line 1989–1991:

```cairo
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [3](#0-2) 

This is the exact analog of the Super DCA bug: a parameter change (commission instead of mint rate) is applied retroactively to a past time window (the current epoch instead of elapsed seconds) because no checkpoint is taken before the change.

**Attack path using `commission_commitment`:**

Without a commitment, `set_commission` can only decrease commission (line 1596), which would benefit delegators. However, `set_commission_commitment` allows a staker to set a `max_commission` ceiling above the current commission, enabling future increases within that ceiling. [4](#0-3) 

The full attack:
1. Staker calls `set_commission_commitment` with `max_commission = 10000` (100%).
2. Staker waits until near the end of an epoch (after most of the epoch has elapsed).
3. Staker calls `set_commission(10000)` — commission jumps to 100% immediately.
4. Staker calls `attest` → `update_rewards_from_attestation_contract` fires.
5. The entire epoch's pool rewards are split at 100% commission → delegators receive **zero** rewards for the full epoch.
6. Staker can then call `set_commission` back to a lower value for the next epoch to retain delegators. [5](#0-4) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Delegators lose all (or a large fraction of) their accrued rewards for the current epoch. The staker captures those rewards as commission. The staker can repeat this every epoch as long as the commitment is active (up to one year). The delegators' funds-at-rest are not stolen, but their unclaimed yield for the epoch is fully redirected to the staker.

---

### Likelihood Explanation

**Medium.** Any staker with a pool can call `set_commission_commitment` at any time (provided no active commitment exists). The commitment setup is a single permissionless transaction. The attack requires the staker to act maliciously against their own delegators, which is a rational choice if the commission gain exceeds the reputational cost of losing delegators. The attack is fully on-chain and requires no external dependencies.

---

### Recommendation

Before writing the new commission, checkpoint rewards for the current epoch:

```cairo
fn set_commission(ref self: ContractState, commission: Commission) {
    self.general_prerequisites();
    // ... existing assertions ...

    // ✅ Checkpoint rewards at old commission before changing it
    self.update_rewards_for_staker(staker_address);  // flush current epoch rewards

    let staker_pool_info = self.staker_pool_info.entry(staker_address);
    if let Option::Some(old_commission) = staker_pool_info.commission.read() {
        self.update_commission(...);
    } else {
        staker_pool_info.commission.write(Option::Some(commission));
        ...
    }
}
```

Alternatively, restrict commission changes to epoch boundaries (e.g., only allow `set_commission` to take effect at the start of the next epoch), so the new rate never applies retroactively to an already-elapsed epoch.

---

### Proof of Concept

**Setup:**
- Staker stakes with commission = 10% and opens a STRK pool.
- Delegator delegates 1,000,000 STRK.
- Epoch rewards = 10,000 STRK total; pool share = 5,000 STRK.

**Expected (without bug):**
- At 10% commission: delegator receives 4,500 STRK; staker commission = 500 STRK.

**Attack:**
1. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current + 100)`.
2. Near end of epoch, staker calls `set_commission(10000)` (100%).
3. Staker calls `attest` → `update_rewards_from_attestation_contract`.
4. `calculate_staker_pools_rewards` reads `commission = 10000`.
5. `split_rewards_with_commission(5000, 10000)` → commission_rewards = 5000, pool_rewards = 0.

**Actual result:** Delegator receives **0 STRK**; staker captures **5,000 STRK** in commission — a full theft of the delegator's epoch yield. [1](#0-0) [2](#0-1)

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );
                total_commission_rewards += commission_rewards;
                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)
