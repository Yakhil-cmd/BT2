### Title
Missing Caller Access Control on `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

The spec mandates "Only starkware sequencer" access for `update_rewards`, but the production implementation enforces **no caller check**. Because `last_reward_block` is a **global** (not per-staker) gate, any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards=true)` once per block, permanently consuming that block's reward slot for the entire protocol and preventing any staker from ever receiving that block's consensus rewards.

---

### Finding Description

The spec explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The production implementation of `update_rewards` calls only `self.general_prerequisites()` (a pause check) and then proceeds with no `get_caller_address()` validation: [2](#0-1) 

A grep for any sequencer-related identifier (`sequencer`, `only_sequencer`, `assert_sequencer`, `get_sequencer`) in `staking.cairo` returns **zero matches**. The tests confirm this: they call `update_rewards` directly from arbitrary addresses with no sequencer impersonation.

The global `last_reward_block` gate is written unconditionally before the `disable_rewards` branch: [3](#0-2) 

This means:
1. The gate is **global** — one call per block for the entire contract, not per staker.
2. Once `last_reward_block` is set to block N, any subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`.
3. Calling with `disable_rewards=true` sets the gate and returns early — **no rewards are distributed** — and the slot is permanently consumed.

---

### Impact Explanation

An attacker front-runs the sequencer every block by calling:

```
update_rewards(any_valid_active_staker, disable_rewards=true)
```

This:
- Sets `last_reward_block = current_block_number` globally.
- Distributes zero rewards to any staker.
- Causes the legitimate sequencer call for that block to revert with `REWARDS_ALREADY_UPDATED`.

Repeated across every block, this permanently zeroes out all consensus block rewards for all stakers. Unclaimed yield that should have accrued is never minted/attributed and cannot be recovered. This matches **High: Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- Requires no privilege, no leaked key, no third-party dependency.
- The call is a single public transaction callable by any EOA or contract.
- The attacker only needs to submit one transaction per block ahead of the sequencer's reward update, which is straightforward to automate on Starknet.
- The cost is only gas; the attacker has no financial incentive requirement (pure griefing).

---

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`, consistent with the spec. On Starknet this is typically enforced by checking `get_caller_address()` against a stored sequencer/operator address, or by using `get_execution_info` to verify the transaction originates from the expected sequencer account:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

The stored `sequencer_address` should be set at initialization and updatable only by governance.

---

### Proof of Concept

1. Deploy the staking contract with two active stakers (A and B), both past the K-epoch activation window, with consensus rewards active.
2. At block N, before the sequencer acts, submit:
   ```
   staking.update_rewards(staker_A_address, disable_rewards=true)
   ```
   from any unprivileged address.
3. Observe: `last_reward_block` is now N; no rewards credited to anyone.
4. The sequencer's legitimate call `update_rewards(staker_B_address, disable_rewards=false)` reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat for every block. After N blocks, both stakers have `unclaimed_rewards_own == 0` despite being eligible for N blocks of consensus rewards.
6. Compare against a baseline run where the attacker does not intervene: stakers accumulate the expected rewards per `calculate_staker_strk_rewards_with_balances_v3`. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
