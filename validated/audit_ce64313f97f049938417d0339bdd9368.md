### Title
Unprivileged Caller Can Grief Consensus Reward Distribution via `update_rewards(disable_rewards: true)` — (File: src/staking/staking.cairo)

### Summary
The public `update_rewards` function unconditionally writes to the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(valid_staker, true)` each block to consume the per-block reward slot without distributing any rewards, permanently blocking all stakers from receiving consensus-phase block rewards.

### Finding Description
`update_rewards` in `IStakingRewardsManager` is a permissionless external function. Its implementation at line 1449 of `src/staking/staking.cairo` follows this sequence:

1. Validates the staker exists and is active (lines 1466–1482).
2. **Unconditionally** writes `self.last_reward_block.write(current_block_number)` at line 1485.
3. Only then checks `if disable_rewards || self.is_pre_consensus() { return; }` at line 1487.

Because `last_reward_block` is updated before the `disable_rewards` guard, any subsequent call to `update_rewards` in the same block fails with `REWARDS_ALREADY_UPDATED` (line 1456–1459). An attacker who calls `update_rewards(any_valid_staker, true)` first in a block consumes the slot without distributing rewards. [1](#0-0) 

The entry-point guard `general_prerequisites()` only checks the contract is unpaused and the caller is non-zero — no role or ownership check exists. [2](#0-1) 

The interface confirms `update_rewards` is fully public with no access restriction: [3](#0-2) 

### Impact Explanation
In the consensus rewards phase (after `consensus_rewards_first_epoch` is set), `update_rewards` is the sole mechanism for distributing per-block STRK and BTC rewards to stakers and their pools. An attacker who front-runs every block with `disable_rewards: true` causes all stakers to receive zero block rewards for those blocks. This constitutes sustained griefing — damage to users and the protocol with no profit motive for the attacker. Matches **Medium: Griefing with no profit motive but damage to users or protocol**.

### Likelihood Explanation
The attack requires only: (a) a valid, active staker address (publicly readable from the `stakers` Vec), and (b) the ability to submit a transaction before legitimate callers each block. On Starknet, transaction ordering is deterministic and front-running is feasible. The attacker pays only gas. No privileged access, leaked keys, or external dependencies are required.

### Recommendation
Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so that a call with `disable_rewards: true` does not consume the per-block reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Update last block rewards only when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... rest of reward distribution logic
```

Alternatively, restrict who may pass `disable_rewards: true` (e.g., only the staker themselves or a privileged role).

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has been set and passed).
2. Attacker identifies any valid, active staker address `S` (e.g., by reading `stakers` Vec).
3. At the start of each new block, attacker calls `update_rewards(S, true)`.
4. `last_reward_block` is set to the current block number; no rewards are distributed.
5. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers accumulate zero new block rewards for every block the attacker front-runs. [4](#0-3) [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
