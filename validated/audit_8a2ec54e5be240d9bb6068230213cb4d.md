### Title
Missing Sequencer-Only Access Control on `update_rewards` Enables Reward-Griefing - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function is documented as callable only by the Starkware sequencer, but the implementation contains no caller check. Any address can call it with `disable_rewards: true`, which advances the global `last_reward_block` cursor without distributing rewards, permanently blocking the sequencer from distributing rewards for that block.

### Finding Description
The specification explicitly states the access control for `update_rewards`:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. It calls `general_prerequisites()` (which only checks the pause flag) and then immediately proceeds to write `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
``` [2](#0-1) 

`last_reward_block` is a single global field shared across all stakers: [3](#0-2) 

The write to `last_reward_block` happens **before** the `disable_rewards` branch. If any caller invokes `update_rewards(any_valid_staker, disable_rewards: true)`, the cursor advances to the current block and the function returns without distributing rewards. The sequencer's subsequent call for the same block fails with `REWARDS_ALREADY_UPDATED`.

The test suite confirms no caller restriction is enforced — `update_rewards` is called in tests without any `cheat_caller_address_once` override: [4](#0-3) 

### Impact Explanation
**Medium — Griefing with no profit motive but damage to users or protocol.**

An attacker can suppress consensus-phase block rewards for every staker in every block by front-running the sequencer's `update_rewards` call with `disable_rewards: true`. Because `last_reward_block` is global, a single call per block is sufficient to zero out rewards for all stakers in that block. Sustained over time this constitutes a permanent, unbounded suppression of yield.

### Likelihood Explanation
Consensus rewards are already live on mainnet (the `consensus_rewards_first_epoch` path is exercised in fork tests). The attacker needs only a valid staker address (publicly readable from `stakers` storage) and the ability to submit a transaction before the sequencer's `update_rewards` transaction lands in each block. Transaction fees are the only cost; no capital or privileged access is required.

### Recommendation
Add an explicit caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured sequencer address (stored in contract state or as a constant), mirroring the pattern used for `update_rewards_from_attestation_contract`: [5](#0-4) 

Alternatively, store the authorized sequencer address during initialization and assert it in `update_rewards` before any state mutation.

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker reads any live staker address from the public `stakers` vector.
3. Attacker submits `update_rewards(staker_address, disable_rewards: true)` at the start of block N.
4. `last_reward_block` is written to N; function returns without distributing rewards.
5. Sequencer submits `update_rewards(staker_address, disable_rewards: false)` for block N.
6. The assertion `current_block_number > self.last_reward_block.read()` fails → `REWARDS_ALREADY_UPDATED`.
7. No staker receives block rewards for block N.
8. Attacker repeats every block; all consensus-phase rewards are permanently suppressed. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
