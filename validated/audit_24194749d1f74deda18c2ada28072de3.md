### Title
Unbounded Loop Over Pool Member Balance Trace in `calculate_rewards` Enables Permanent Freezing of Unclaimed Yield - (`src/pool/pool.cairo`)

---

### Summary

The `calculate_rewards` function in `src/pool/pool.cairo` iterates over every entry in a pool member's `pool_member_epoch_balance` trace without any cap on the number of iterations. Because each balance change in a distinct epoch appends a new checkpoint to this trace, a delegator who makes balance changes across many epochs without claiming rewards will accumulate an unbounded trace. Eventually, calling `claim_rewards` (or `pool_member_info_v1`) will exceed the Starknet transaction gas limit, permanently freezing the delegator's unclaimed yield.

---

### Finding Description

**Root cause — the unbounded loop:**

In `calculate_rewards`, the code explicitly acknowledges the problem:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The loop iterates from `entry_to_claim_from` (the index saved at the last successful `claim_rewards`) up to the current trace length. Each iteration reads a storage slot and calls `find_sigma`, making it non-trivial in gas cost.

**How the trace grows:**

`set_member_balance` inserts a new checkpoint keyed by `current_epoch + K`. The underlying `insert` function only updates the last entry in-place if the epoch key matches; otherwise it appends a new entry:

```cairo
if last.key == key {
    last.value = value;
    checkpoints[len - 1].write(last);
} else {
    checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
}
``` [2](#0-1) 

This means every balance change in a new epoch appends one entry. The trace has no length cap.

**Which user actions grow the trace:**

`add_to_delegation_pool`, `exit_delegation_pool_intent`, `enter_delegation_pool`, and `switch_delegation_pool` all call `set_member_balance` or `increase_member_balance`: [3](#0-2) [4](#0-3) [5](#0-4) 

**`entry_to_claim_from` is only advanced by `claim_rewards`:**

`pool_member_info_v1` (a read-only view) also calls `calculate_rewards` but does **not** update `entry_to_claim_from`, so repeated reads do not help drain the backlog: [6](#0-5) 

Only a successful `claim_rewards` advances the pointer: [7](#0-6) 

**Attack / accumulation path:**

1. Delegator calls `add_to_delegation_pool` (or `exit_delegation_pool_intent`) once per epoch, each time in a new epoch, without ever calling `claim_rewards`.
2. Each call appends one entry to `pool_member_epoch_balance`.
3. After N epochs, the trace has N entries.
4. When `claim_rewards` is finally called, the loop must iterate over all N entries. Beyond a threshold (determined by Starknet's per-transaction gas/step limit), the transaction reverts.
5. Because `entry_to_claim_from` is never advanced (every attempt reverts), the delegator's unclaimed yield is permanently frozen.

---

### Impact Explanation

**Permanent freezing of unclaimed yield.** Once the trace is large enough that `calculate_rewards` exceeds the gas limit, every call to `claim_rewards` reverts. The delegator's accumulated STRK rewards become permanently inaccessible. There is no administrative escape hatch — no function exists to partially drain the trace or reset `entry_to_claim_from`.

This matches the allowed impact: **"Permanent freezing of unclaimed yield or unclaimed royalties."**

---

### Likelihood Explanation

The protocol is designed to run indefinitely. A delegator who actively manages their delegation (adding or partially exiting each epoch) over a long period — or who simply never claims rewards while making periodic balance changes — will naturally grow their trace. The growth rate is at most one entry per epoch. Given Starknet's current step limits and the cost of each loop iteration (storage read + `find_sigma` lookup), the threshold is in the range of thousands of entries, reachable over years of normal protocol operation. No privileged access is required; any delegator can trigger this on their own account.

---

### Recommendation

1. **Cap the trace length** by enforcing a maximum number of unclaimed balance-change entries before allowing further balance modifications (analogous to GMX's `require(swapPath.length < y)`).
2. **Alternatively, checkpoint rewards eagerly**: call an internal reward-settlement step inside `add_to_delegation_pool` and `exit_delegation_pool_intent` so that `entry_to_claim_from` is advanced at every balance change, keeping the pending range bounded to at most K entries.
3. **At minimum**, document and enforce a protocol-level invariant that `pool_member_trace_length - entry_to_claim_from` cannot exceed a safe constant.

---

### Proof of Concept

```
// Pseudocode — no privileged access required
for epoch in 0..N_LARGE:
    advance_epoch()
    delegator.add_to_delegation_pool(pool, small_amount)
    // Each call appends one entry to pool_member_epoch_balance
    // entry_to_claim_from remains 0 (no claim_rewards called)

// After N_LARGE epochs:
delegator.claim_rewards(pool)
// => calculate_rewards loops N_LARGE times
// => exceeds Starknet gas limit => reverts
// => unclaimed yield permanently frozen
```

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

```text
    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
