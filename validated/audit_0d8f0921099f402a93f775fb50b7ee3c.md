### Title
Pool Member Record Not Deleted After Full Exit, Permanently Blocking Re-Entry — (File: src/pool/pool.cairo)

### Summary
When a pool member fully exits a delegation pool (via `exit_delegation_pool_intent` with the full amount followed by `exit_delegation_pool_action`), their `pool_member_info` record is updated but never deleted. Because `enter_delegation_pool` gates re-entry on `pool_member_info.read(pool_member).is_none()`, the same address is permanently blocked from re-entering that pool.

### Finding Description
During `enter_delegation_pool`, a new member record is created and the existence check is strict:

```cairo
assert!(
    self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
);
``` [1](#0-0) 

When a pool member fully exits, `exit_delegation_pool_intent` is called with `amount == total_amount`, setting `new_delegated_stake = 0` and recording the intent: [2](#0-1) 

Then `exit_delegation_pool_action` completes the exit by zeroing `unpool_amount` and `unpool_time`, but only calls `write_pool_member_info` — it never writes `VInternalPoolMemberInfo::None` back to storage:

```cairo
pool_member_info.unpool_amount = Zero::zero();
pool_member_info.unpool_time = Option::None;
self.write_pool_member_info(:pool_member, :pool_member_info);
``` [3](#0-2) 

`write_pool_member_info` always wraps the struct in `VInternalPoolMemberInfoTrait::wrap_latest`, never producing `VInternalPoolMemberInfo::None`: [4](#0-3) 

After the action, `pool_member_info.read(pool_member)` returns `VInternalPoolMemberInfo::V1(...)` — not `None`. Any subsequent call to `enter_delegation_pool` from the same address panics with `POOL_MEMBER_EXISTS`.

By contrast, `enter_delegation_pool_from_staking_contract` (used for pool-switching) explicitly handles the already-exists case and updates the record rather than rejecting it, confirming the inconsistency is unintentional: [5](#0-4) 

The `pool_member_epoch_balance` trace for the member is also never cleared, leaving stale historical balance data permanently in storage. [6](#0-5) 

### Impact Explanation
Any pool member who fully exits a delegation pool is permanently prevented from re-entering that same pool with the same address. They must use a different address or switch to a different staker's pool. This constitutes griefing with damage to users: a delegator who exits and later wishes to re-delegate to the same staker (e.g., after the exit wait window, to re-enter with a larger amount) is silently and permanently blocked. There is no profit motive for the protocol or any attacker — the damage is purely to the affected delegator.

**Allowed impact matched**: Griefing with no profit motive but damage to users or protocol — **Medium**.

### Likelihood Explanation
Any pool member who performs a full exit (`exit_delegation_pool_intent` with `amount == total_amount`) and later attempts to re-enter the same pool triggers this bug. This is a natural usage pattern (exit, wait, re-enter with a different amount). No special privileges or external conditions are required — the pool member triggers it themselves through normal protocol interactions.

**Likelihood**: Medium.

### Recommendation
In `exit_delegation_pool_action`, after transferring funds, check whether the pool member's remaining balance is zero and no unclaimed rewards remain. If so, delete the record by writing `VInternalPoolMemberInfo::None`:

```cairo
self.pool_member_info.write(pool_member, VInternalPoolMemberInfo::None);
```

Alternatively, modify `enter_delegation_pool` to allow re-entry when the existing record has zero balance and no pending exit intent, treating it as a fresh entry.

### Proof of Concept
1. Pool member `P` calls `enter_delegation_pool(reward_addr, 1000)` — record created, balance = 1000.
2. `P` calls `exit_delegation_pool_intent(1000)` — full exit intent, `unpool_amount = 1000`, `new_delegated_stake = 0`.
3. After the exit wait window, `P` (or anyone) calls `exit_delegation_pool_action(P)` — funds returned, `unpool_amount = 0`, `unpool_time = None`. Record is **updated but not deleted**.
4. `P` calls `claim_rewards(P)` — rewards claimed. Record still exists with zero balance.
5. `P` calls `enter_delegation_pool(reward_addr, 500)` — **panics** with `POOL_MEMBER_EXISTS` because `pool_member_info.read(P)` is `VInternalPoolMemberInfo::V1(...)`, not `None`.

`P` is permanently blocked from re-entering this pool with the same address, despite having a zero balance and no pending state.

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }
