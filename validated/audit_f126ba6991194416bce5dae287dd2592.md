Audit Report

## Title
Dissolved Neuron with Only Staked Maturity Permanently Skipped by Unstake Timer Task — (`rs/nns/governance/src/neuron_store.rs`)

## Summary
`list_neurons_ready_to_unstake_maturity` iterates only over active neurons via `with_active_neurons_iter_sections`, which filters out neurons where `is_inactive()` returns `true`. The `is_inactive` condition 2 delegates to `is_funded()`, which — per its inline doc comment — covers only `cached_neuron_stake_e8s` and `maturity_e8s_equivalent` (unstaked maturity), explicitly excluding `staked_maturity_e8s_equivalent`. A dissolved neuron holding only staked maturity is therefore classified as inactive and permanently skipped by the sole timer task responsible for converting staked maturity to liquid maturity, locking those ICP-equivalent funds indefinitely.

## Finding Description

`is_inactive` at [1](#0-0)  documents condition 2 as *"No stake, and no **(unstaked)** maturity"* — the parenthetical explicitly excludes `staked_maturity_e8s_equivalent`. The implementation at [2](#0-1)  calls `is_funded()` for this check, which does not account for `staked_maturity_e8s_equivalent`.

`with_active_neurons_iter_sections` applies this filter at [3](#0-2)  with `.filter(|n| !n.is_inactive(now))`, so any neuron classified inactive is excluded from the iterator entirely.

`list_neurons_ready_to_unstake_maturity` at [4](#0-3)  feeds exclusively from this filtered iterator, meaning the `ready_to_unstake_maturity` check at [5](#0-4)  is never reached for inactive neurons.

`unstake_maturity_of_dissolved_neurons` at [6](#0-5)  is the sole consumer of `list_neurons_ready_to_unstake_maturity`, and the timer task at [7](#0-6)  is the sole caller of `unstake_maturity_of_dissolved_neurons`. There is no alternative code path.

**Exploit path:**
1. Controller stakes maturity (`stake_maturity = true`).
2. Controller starts dissolving; waits for dissolve delay to expire.
3. Controller disburses the ICP stake → `cached_neuron_stake_e8s = 0`.
4. Controller waits 2 weeks → neuron satisfies all four `is_inactive` conditions.
5. Timer fires every 60 seconds but silently skips the neuron forever. `staked_maturity_e8s_equivalent` is permanently inaccessible.

## Impact Explanation
`staked_maturity_e8s_equivalent` represents ICP-equivalent value that the protocol is supposed to convert to liquid maturity upon dissolution. After step 4 above, no user-facing operation (disburse, spawn, merge) can recover it, and the timer task will never process it. The funds are effectively permanently locked until a governance canister upgrade retroactively fixes and reprocesses affected neurons. This constitutes a **significant NNS security impact with concrete user fund harm** — matching the High ($2,000–$10,000) impact tier for significant NNS security impact with concrete user or protocol harm.

## Likelihood Explanation
The triggering sequence uses only standard, unprivileged neuron management operations available to any neuron controller. No special access, no social engineering, and no external dependencies are required. The bug is entirely silent: no error is logged, no metric is emitted, and the timer task gives no indication that a neuron was skipped. Any neuron controller who stakes maturity, dissolves, disburses ICP, and waits two weeks will silently lose access to their staked maturity.

## Recommendation
Fix `is_funded()` (or the `is_inactive` condition 2 check directly) to include `staked_maturity_e8s_equivalent`:

```rust
// Option A: extend is_funded()
pub fn is_funded(&self) -> bool {
    self.cached_neuron_stake_e8s > 0
        || self.maturity_e8s_equivalent > 0
        || self.staked_maturity_e8s_equivalent.unwrap_or(0) > 0
}
```

Alternatively, `unstake_maturity_of_dissolved_neurons` could use the existing `first_neuron_id` / `range_neurons_sections` full-range scan pattern (as seen at [8](#0-7) ) instead of the active-only iterator, ensuring all neurons are considered regardless of inactive status.

## Proof of Concept

```rust
#[test]
fn test_inactive_dissolved_neuron_with_staked_maturity_is_not_skipped() {
    let mut neuron_store = NeuronStore::new(BTreeMap::new());
    let now = neuron_store.now();
    // Dissolved 30 days ago, no ICP stake, no unstaked maturity, but has staked maturity
    let neuron = NeuronBuilder::new(
        NeuronId { id: 1 },
        subaccount,
        controller,
        DissolveStateAndAge::DissolvingOrDissolved {
            when_dissolved_timestamp_seconds: now - 30 * ONE_DAY_SECONDS,
        },
        now,
    )
    .with_cached_neuron_stake_e8s(0)
    .with_staked_maturity_e8s_equivalent(1_000_000)
    .build();
    neuron_store.add_neuron(neuron).unwrap();

    // Confirm neuron is classified inactive
    neuron_store.with_neuron(&NeuronId { id: 1 }, |n| assert!(n.is_inactive(now))).unwrap();

    // Run timer task multiple times
    for _ in 0..5 {
        neuron_store.unstake_maturity_of_dissolved_neurons(now, 100);
    }

    // Staked maturity should have been moved to liquid maturity — this assertion FAILS on current code
    neuron_store.with_neuron(&NeuronId { id: 1 }, |n| {
        assert_eq!(n.staked_maturity_e8s_equivalent.unwrap_or(0), 0,
            "staked maturity was never unstaked");
        assert_eq!(n.maturity_e8s_equivalent, 1_000_000);
    }).unwrap();
}
```

This test fails on the current codebase, confirming the bug.
