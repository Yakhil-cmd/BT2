### Title
Missing Native Resource Charge for Automatic Return Data Copy — (`evm_interpreter/src/interpreter.rs`)

---

### Summary

The `copy_returndata_to_heap` function in the EVM interpreter performs a raw memory copy of a callee's return data into the caller's pre-allocated output buffer without charging any native resources (RISC-V cycles). A parallel uncharged copy also occurs in `copy_into_return_memory` in the bootloader runner. Because native resources model actual RISC-V proving cycles, this creates a systematic discrepancy between the resources charged during execution and the cycles the prover must actually consume, constituting a resource accounting bug with valid-execution unprovability potential.

---

### Finding Description

**Root cause — `copy_returndata_to_heap`**

When any EVM `CALL`/`STATICCALL`/`DELEGATECALL`/`CALLCODE` opcode completes, `continue_after_preemption` calls `copy_returndata_to_heap`:

```rust
// evm_interpreter/src/interpreter.rs:439-453
pub(crate) fn copy_returndata_to_heap(&mut self, returndata_region: &'ee [u8]) {
    if !self.returndata_location.is_empty() {
        unsafe {
            let to_copy =
                core::cmp::min(returndata_region.len(), self.returndata_location.len());
            let src = returndata_region.as_ptr();
            let dst = self.heap.as_mut_ptr().add(self.returndata_location.start);
            core::ptr::copy_nonoverlapping(src, dst, to_copy);   // ← raw copy, zero native charge
        }
    }
    self.returndata = returndata_region;
}
```

`to_copy` can be up to `out_len` bytes (the output-buffer size the caller specified in the CALL opcode). The copy is a real `core::ptr::copy_nonoverlapping` that the RISC-V prover must execute, yet **no native resources are charged**.

Compare this to every explicit copy opcode (`RETURNDATACOPY`, `CALLDATACOPY`, `CODECOPY`, `EXTCODECOPY`, `MCOPY`), all of which charge:

```rust
// evm_interpreter/src/gas.rs:126-136
let num_words = len.checked_next_multiple_of(32)? / 32;
let gas    = crate::gas_constants::COPY.checked_mul(num_words)?;
let native = crate::native_resource_constants::COPY_BYTE_NATIVE_COST
                 .checked_mul(len)?
                 .checked_add(crate::native_resource_constants::COPY_BASE_NATIVE_COST)?;
```

`COPY_BYTE_NATIVE_COST = 1` and `COPY_BASE_NATIVE_COST = 80` (from `evm_interpreter/src/native_resource_constants.rs:69-70`). For a 1 MB output buffer the missing native charge is ≈ 1 048 656 native units — equal in magnitude to the memory-expansion native cost that *is* charged.

**Secondary uncharged copy — `copy_into_return_memory`**

After the callee's frame ends, the runner copies the callee's full returndata slice into the dedicated return-memory buffer:

```rust
// basic_bootloader/src/bootloader/runner.rs:92-107
fn copy_into_return_memory<'a>(
    &mut self,
    return_values: ReturnValues<'a, S>,
) -> Result<ReturnValues<'external, S>, BootloaderSubsystemError> {
    let return_memory = core::mem::take(&mut self.return_memory);
    if return_values.returndata.len() > return_memory.len() {
        return Err(out_of_return_memory!().into());
    }
    let (output, rest) = return_memory.split_at_mut(return_values.returndata.len());
    self.return_memory = rest;
    Ok(ReturnValues {
        returndata: output.write_copy_of_slice(return_values.returndata),  // ← uncharged copy
        ..return_values
    })
}
```

This copy is bounded by the callee's returndata length (callee-controlled), is executed after the callee's resources have already been taken (`self.gas.take_resources()`), and is charged to no frame at all. The callee paid for heap expansion to *store* the return data but not for the copy into the return-memory buffer.

**Combined effect**

For a single CALL with `out_len = N` bytes and a callee that returns `N` bytes, the prover must execute two uncharged copies totalling `2N` bytes, while the charged native cost covers only the memory-expansion of `N` bytes.

---

### Impact Explanation

ZKsync OS uses "Double Resource Accounting": EVM gas and native resources (RISC-V cycles). Native resources are the binding constraint for the prover. If the prover must execute more RISC-V cycles than were charged, one of two outcomes follows:

1. **Valid-execution unprovability**: the transaction passes all native-resource checks during execution (because the uncharged copy work is invisible to the accounting), but the actual RISC-V cycle count exceeds the proving budget, making the block unprovable.
2. **Underpriced computation**: the prover is forced to do work it was not compensated for, making proving economically unprofitable and enabling a sustained griefing attack against the sequencer/prover.

For a 30 M gas block the maximum heap is ≈ 4 MB. The missing native charge per transaction is ≈ 8 M native units (two copies of 4 MB). An attacker can fill a block with such transactions, multiplying the proving cost far beyond what was paid.

---

### Likelihood Explanation

The trigger is any EVM `CALL` with a non-zero `out_len`. This is the normal case for virtually every contract interaction. No special privilege is required; any unprivileged EOA can submit such a transaction. The severity scales linearly with `out_len`, which is bounded only by the block gas limit.

---

### Recommendation

In `copy_returndata_to_heap`, charge native resources for the copy before performing it, consistent with all other copy operations:

```rust
let native_cost = COPY_BASE_NATIVE_COST
    + COPY_BYTE_NATIVE_COST * (to_copy as u64);
self.gas.spend_gas_and_native(0, native_cost)?;
```

Similarly, in `copy_into_return_memory`, charge the callee's remaining resources (or the caller's resources after reclaim) for the copy proportional to `return_values.returndata.len()`.

---

### Proof of Concept

1. Attacker deploys **Callee** with bytecode that writes `N` bytes to memory and executes `RETURN 0 N` (e.g., `N = 1 048 576`).
2. Attacker deploys **Caller** with bytecode:
   ```
   PUSH N        // out_len = 1 MB
   PUSH 0        // out_offset = 0
   PUSH 0        // in_len
   PUSH 0        // in_offset
   PUSH 0        // value
   PUSH <callee>
   PUSH <gas>
   CALL
   ```
3. Attacker submits a transaction calling **Caller**.
4. During execution:
   - `resize_heap(0, N)` charges memory-expansion native cost ≈ `N` native units — **paid**.
   - `copy_into_return_memory` copies `N` bytes from callee's heap to return buffer — **uncharged**.
   - `copy_returndata_to_heap` copies `N` bytes from return buffer to caller's heap — **uncharged**.
5. Total native units charged: ≈ `N`. Total RISC-V copy cycles consumed by prover: ≈ `2N`. The prover performs 2× the work it was paid for. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)
