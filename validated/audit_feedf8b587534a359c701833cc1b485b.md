### Title
Missing Zero-Address Validation in `change_reward_address()` Allows Permanent Freezing of Unclaimed Yield - (File: src/staking/staking.cairo, src/pool/pool.cairo)

### Summary
Both `Staking.change_reward_address()` and `Pool.change_reward_address()` accept a `reward_address` parameter without validating that it is non-zero. A staker or pool member can set their reward address to the zero address (`0x0`). When rewards are subsequently claimed, the contract attempts to transfer STRK to the zero address. If the STRK ERC20 implementation reverts on zero-address transfers (standard behavior), `claim_rewards` will permanently revert for that user, freezing all accrued unclaimed yield with no recovery path.

### Finding Description
In `src/staking/staking.cairo`, `change_reward_address()` validates only that the new address is not a registered token address (`REWARD_ADDRESS_IS_TOKEN`), but performs no zero-address check:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    assert!(
        !self.does_token_exist(token_address: reward_address),
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    let staker_address = get_caller_address();
    let mut staker_info = self.internal_staker_info(:staker_address);
    // reward_address = 0x0 is accepted here
    staker_info.reward_address = reward_address;
    self.write_staker_info(:staker_address, :staker_info);
    ...
}
```

The identical gap exists in `src/pool/pool.cairo`:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    assert!(
        self.token_dispatcher.contract_address.read() != reward_address,
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    let pool_member = get_caller_address();
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    // reward_address = 0x0 is accepted here
    pool_member_info.reward_address = reward_address;
    self.write_pool_member_info(:pool_member, :pool_member_info);
    ...
}
```

Once `reward_address` is stored as zero, every downstream reward transfer targets the zero address. In `pool.cairo`, `claim_rewards` unconditionally calls:

```cairo
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
```

If the STRK ERC20 reverts on a zero-address recipient (standard ERC20 behavior), this call always reverts, permanently blocking reward withdrawal. The `GenericError::ZERO_ADDRESS` variant already exists in `src/errors.cairo` (line 20), confirming the codebase has a pattern for this check — it is simply absent here.

### Impact Explanation
All STRK rewards accrued by the affected staker or pool member become permanently unclaimable. The staker's `unclaimed_rewards_own` and the pool member's `unclaimed_rewards` fields continue to accumulate in storage but can never be transferred out. This constitutes **permanent freezing of unclaimed yield**, which is an allowed impact.

### Likelihood Explanation
Any active staker or pool member can call `change_reward_address()` at any time with `reward_address = 0x0`. No special privilege, external dependency, or third-party compromise is required. The call succeeds silently, with no on-chain warning. A user who makes this mistake (e.g., passing a zero address due to a front-end bug, a scripting error, or a misunderstanding) has no recovery path — there is no mechanism to reclaim frozen rewards after the fact.

### Recommendation
Add a zero-address guard at the top of both `change_reward_address()` implementations, mirroring the existing `assert_caller_is_not_zero()` pattern:

```cairo
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

This should be added in `src/staking/staking.cairo` before the token-address check, and in `src/pool/pool.cairo` at the start of the function body.

### Proof of Concept
1. Staker calls `Staking.change_reward_address(reward_address: 0x0)`.
   - The `REWARD_ADDRESS_IS_TOKEN` check passes (zero is not a registered token).
   - `staker_info.reward_address` is written as `0x0`.
2. Staker accrues rewards via attestation / `update_rewards`.
3. Staker (or anyone) calls `Staking.claim_rewards(staker_address)`.
   - `send_rewards_to_staker` calls `token_dispatcher.checked_transfer(recipient: 0x0, amount: rewards)`.
   - Standard ERC20 reverts on zero-address recipient → transaction reverts.
4. All accumulated `unclaimed_rewards_own` is permanently frozen; no recovery path exists.

The same sequence applies to a pool member calling `Pool.change_reward_address(0x0)` followed by `Pool.claim_rewards(pool_member)`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
