### Title
Single Malformed Message Permanently Aborts Protocol for All Honest Parties via Unguarded Deserialization Error Propagation in `recv_from_others` — (File: `src/protocol/helpers.rs`)

---

### Summary

The `recv_from_others` helper, used in virtually every protocol round that collects contributions, propagates deserialization errors immediately via `?`. A single malicious participant can inject a message with a valid header but an invalid MessagePack payload, causing deserialization to fail and aborting the entire protocol for all honest parties. Unlike `reliable_broadcast_receive_all`, which silently skips malformed messages (`_ => continue`), `recv_from_others` has no skip or quarantine mechanism — a direct head-of-line blocking analog.

---

### Finding Description

`recv_from_others` in `src/protocol/helpers.rs` drives every round that collects one contribution per participant:

```rust
while !seen.full() {
    let (from, msg) = chan.recv(waitpoint).await?;   // <-- error propagates here
    if seen.put(from) {
        messages.push((from, msg));
    }
}
``` [1](#0-0) 

`chan.recv(waitpoint)` resolves through `SharedChannel::recv` → `Comms::recv`:

```rust
async fn recv<T: DeserializeOwned>(&self, header: MessageHeader) -> Result<(Participant, T), ProtocolError> {
    let (from, data) = self.incoming.pop(header).await;          // pops from FIFO queue
    let message_data = data.get(MessageHeader::LEN..).ok_or_else(|| ...)?;
    let decoded: Result<T, ...> = rmp_serde::decode::from_slice(message_data)...;
    Ok((from, decoded?))                                          // error propagates on bad payload
}
``` [2](#0-1) 

`self.incoming.pop(header)` pops the **head entry** of the `SubMessageQueue` — a per-`MessageHeader` FIFO channel: [3](#0-2) 

The `MessageHeader` is composed of a `ChannelTag` and a `Waitpoint`. For the shared channel, the tag is a **public constant** (`SHA-256(DOMAIN || "root shared")`), and the waitpoint is a **sequential counter starting at 0**: [4](#0-3) [5](#0-4) 

`push_message` accepts any message whose first 40 bytes parse as a valid header — **no payload validation**: [6](#0-5) 

A malicious participant can therefore:
1. Compute the correct `MessageHeader` for any round (constant tag + predictable waitpoint).
2. Craft `[40-byte header] || [0xFF 0xFF 0xFF]` (invalid MessagePack).
3. Deliver it to an honest participant via `Protocol::message(from, data)`.
4. The honest party's `recv_from_others` pops this head entry; `rmp_serde::decode::from_slice` fails; the `?` propagates `ProtocolError::DeserializationError`; the protocol aborts.

The contrast with `reliable_broadcast_receive_all` is explicit — that function guards the same `chan.recv` call:

```rust
match chan.recv(wait).await {
    Ok(value) => (from, (sid, vote)) = value,
    _ => continue,   // malformed messages are silently skipped
};
``` [7](#0-6) 

`recv_from_others` has no equivalent guard. The existing test `attacker_can_fill_message_buffer_with_unused_waitpoints` acknowledges that an attacker can inject arbitrary messages into the buffer, but does not test the deserialization-abort path through `recv_from_others`: [8](#0-7) 

---

### Impact Explanation

**High — Permanent denial of signing, key generation, reshare, refresh, or CKD for honest parties.**

A single malicious participant can abort any protocol run for all honest parties by sending one crafted message per round. Because the attack is repeatable on every restart, the denial is effectively permanent. This violates the liveness guarantee even when the number of malicious parties is within the documented tolerance (`floor((N-1)/3)`).

---

### Likelihood Explanation

**High.** The attack requires only:
- Participant status (any enrolled participant qualifies).
- Knowledge of the shared channel tag (a deterministic public constant) and the waitpoint (sequential from 0, predictable before the protocol starts).
- The ability to send one message with a valid 40-byte header and an invalid MessagePack body.

No cryptographic material, no side-channel, no coordination with other attackers is needed.

---

### Recommendation

Mirror the guard already present in `reliable_broadcast_receive_all`. In `recv_from_others`, skip malformed messages instead of propagating the error:

```rust
while !seen.full() {
    let Ok((from, msg)) = chan.recv(waitpoint).await else {
        continue; // skip malformed / undeserializable messages
    };
    if seen.put(from) {
        messages.push((from, msg));
    }
}
```

Optionally log the skipped sender for observability. This matches the design intent stated in the echo-broadcast comment: *"The recv should be failure-free — this translates to ignoring the received message when deemed wrong."* [9](#0-8)

### Proof of Concept

```
Setup: participants P1 (honest), P2 (honest), M (malicious), running keygen or sign.

1. M computes the shared channel header for waitpoint 0:
       channel = SHA-256(DOMAIN || "root shared")   // constant
       waitpoint = 0                                 // first round
       header_bytes = channel[0..32] || waitpoint.to_le_bytes()

2. M crafts the attack message:
       msg = header_bytes || [0xFF, 0xFF, 0xFF]     // invalid MessagePack payload

3. M delivers msg to P1:
       P1.protocol.message(M_participant_id, msg)

4. P1's recv_from_others calls chan.recv(0):
       → SubMessageQueue for header pops msg (FIFO head)
       → rmp_serde::decode::from_slice([0xFF,0xFF,0xFF]) → Err(...)
       → Comms::recv returns Err(ProtocolError::DeserializationError(...))
       → recv_from_others propagates via `?`

5. P1's protocol returns Err. Signing / keygen aborts for all honest parties.
   M repeats on every restart → permanent denial.
```

struct SubMessageQueue {
    sender: futures::channel::mpsc::UnboundedSender<(Participant, MessageData)>,
    receiver: Arc<Mutex<futures::channel::mpsc::UnboundedReceiver<(Participant, MessageData)>>>,
}

impl SubMessageQueue {
    pub fn send(&self, from: Participant, message: MessageData) {
        // This cannot fail because the receiver is also alive.
        self.sender
            .unbounded_send((from, message))
            .expect("unbound_send should not fail");
    }
}

impl Default for SubMessageQueue {
    fn default() -> Self {
        let (sender, receiver) = futures::channel::mpsc::unbounded();
        Self {
            sender,
            receiver: Arc::new(Mutex::new(receiver)),
        }
    }
}
