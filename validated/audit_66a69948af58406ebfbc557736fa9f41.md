### Title
Missing Zero Address Validation on `reward_address` Causes Permanent Freezing of Unclaimed Yield and Principal - (File: src/staking/staking.cairo)

---

### Summary
Both `stake()` and `change_reward_address()` in `src/staking/staking.cairo` (and their pool counterparts in `src/pool/pool.cairo`) validate that `reward_address` is not a token contract address, but neither validates that it is non-zero. A staker who sets `reward_address` to the zero address will find that every subsequent call to `claim_rewards` and `unstake_action` reverts, because the STRK ERC20 token (OpenZeppelin Cairo) unconditionally rejects transfers to the zero address — permanently freezing their unclaimed yield and temporarily (or permanently) freezing their staked principal.

---

### Finding Description

**Root cause — `stake()`:** [1](#0-0) 

The only guard on `reward_address` is `!self.does_token_exist(token_address: reward_address)`. There is no `assert!(reward_address.is_non_zero(), ...)` check. A caller may pass `reward_address = 0` and the transaction succeeds.

**Root cause — `change_reward_address()` (staking):** [2](#0-1) 

Same single guard: only the token-address check. Zero address is accepted.

**Root cause — `change_reward_address()` (pool):** [3](#0-2) 

Identical omission in the delegation pool contract.

**Downstream failure — `send_rewards_to_staker()`:** [4](#0-3) 

`checked_transfer(recipient: reward_address, amount: amount.into())` is called unconditionally. OpenZeppelin Cairo ERC20 asserts `recipient != 0` inside `_transfer` regardless of the transfer amount, so this call reverts whenever `reward_address` is zero.

**Downstream failure — `claim_rewards()`:** [5](#0-4) 

Calls `send_rewards_to_staker` → reverts.

**Downstream failure — `unstake_action()`:** [6](#0-5) 

`send_rewards_to_staker` is called *before* the principal transfer. If it reverts, the entire `unstake_action` reverts, blocking the return of the staked principal.

**Pool-side downstream failure — `claim_rewards()` in pool:** [7](#0-6) 

`checked_transfer(recipient: reward_address, ...)` reverts when `reward_address` is zero.

The `ZERO_ADDRESS` error variant exists in the shared error enum but is never enforced at these entry points: [8](#0-7) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield:** Once `reward_address` is set to zero, every call to `claim_rewards` reverts. Accrued STRK rewards accumulate in `unclaimed_rewards_own` but can never be transferred out. If the staker loses their signing key after setting the zero address (or if the zero address was set at `stake()` time), the yield is permanently frozen.

**High — Temporary/permanent freezing of staked principal:** `unstake_action` calls `send_rewards_to_staker` as its first step. With a zero `reward_address`, this reverts, blocking the return of the staked principal. The staker must call `change_reward_address` to a valid address before `unstake_action` can succeed. If the staker's key is lost, the principal is permanently frozen.

---

### Likelihood Explanation

Likelihood is **low-to-medium**. The zero address is the default/uninitialized value in Cairo (`ContractAddress` defaults to `0`). A developer integrating via a script or a user making a mistake in a UI could easily pass `0` as `reward_address`. The `REWARD_ADDRESS_IS_TOKEN` check creates a false sense of completeness, making the missing zero-address check easy to overlook. The scenario is self-inflicted but realistic for accidental misuse.

---

### Recommendation

Add a non-zero assertion on `reward_address` in all four entry points:

1. `stake()` in `src/staking/staking.cairo` — after the existing token-address check
2. `change_reward_address()` in `src/staking/staking.cairo`
3. `enter_delegation_pool()` in `src/pool/pool.cairo`
4. `change_reward_address()` in `src/pool/pool.cairo`

```cairo
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

The `GenericError::ZERO_ADDRESS` variant already exists and carries the message `"Address is zero"`. [9](#0-8) 

---

### Proof of Concept

1. Deploy the system normally.
2. Call `staking.stake(reward_address: 0, operational_address: <valid>, amount: <min_stake>)` — succeeds (no zero-address guard).
3. Advance epochs and attest to accrue rewards.
4. Call `staking.claim_rewards(staker_address)` — **reverts** with `"ERC20: transfer to 0"` (or equivalent OZ Cairo panic) because `send_rewards_to_staker` calls `checked_transfer(recipient: 0, ...)`.
5. Call `staking.unstake_intent()` — succeeds (no transfer here).
6. Advance past the exit window.
7. Call `staking.unstake_action(staker_address)` — **reverts** for the same reason; the staked principal cannot be recovered.
8. Demonstrate recovery: call `staking.change_reward_address(<valid_address>)` from the staker key, then retry step 7 — succeeds. This confirms the freeze is permanent only if the staker key is unavailable.

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
