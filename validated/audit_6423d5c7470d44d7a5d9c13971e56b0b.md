### Title
Unprivileged Caller Can Permanently Suppress Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `IStakingRewardsManager::update_rewards` function has no access control and accepts a caller-controlled `disable_rewards` boolean. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` to consume the single per-block reward slot and update the global `last_reward_block` without distributing any rewards, permanently preventing all stakers from receiving block rewards for that block.

### Finding Description
`update_rewards` is a publicly callable function with no caller restriction beyond `general_prerequisites()` (not paused, non-zero caller). [1](#0-0) 

The function accepts a `disable_rewards: bool` parameter that is entirely caller-controlled. [2](#0-1) 

The critical state update happens **before** the `disable_rewards` branch: `last_reward_block` is written to the current block number unconditionally. [3](#0-2) 

`last_reward_block` is a **single global** storage variable, not per-staker. [4](#0-3) 

The guard at the top of the function enforces that only one call per block can succeed: [5](#0-4) 

Combining these facts: an attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at block N will:
1. Pass the `current_block_number > last_reward_block` guard.
2. Write `last_reward_block = N`.
3. Hit the early return at line 1487, distributing zero rewards.
4. Cause every subsequent call to `update_rewards` at block N to revert with `REWARDS_ALREADY_UPDATED`.

The attacker only needs to supply any currently-active staker address with non-zero STRK balance, which is trivially discoverable from on-chain events (`NewStaker`).

### Impact Explanation
**High — Permanent freezing / theft of unclaimed yield.**

Block rewards for the affected block are never credited to any staker's `unclaimed_rewards_own` and never forwarded to any delegation pool. Because `last_reward_block` is global, a single attacker call per block suppresses rewards for **all** stakers and all pool members simultaneously. Repeated across every block, this constitutes a complete, permanent denial of consensus-phase block rewards with no recovery path for the missed blocks.

### Likelihood Explanation
**High.** The attack requires:
- No funds or stake.
- No privileged role.
- Only gas to call `update_rewards` once per block with `disable_rewards: true`.

The attacker can automate this with a simple bot. The only cost is transaction fees, which are negligible on Starknet. A competitor, short-seller, or griever has clear motivation. The `staker_address` argument can be any active staker found from public `NewStaker` events.

### Recommendation
1. **Remove the `disable_rewards` parameter from the public interface.** If the protocol needs a "no-reward" update path, gate it behind an authorized role (e.g., `only_app_governor`) or derive the flag from on-chain state rather than caller input.
2. **Add access control** so only the staker themselves (or the attestation contract) can call `update_rewards` for a given `staker_address`.
3. **Make `last_reward_block` per-staker** so one call cannot block all others.

### Proof of Concept

```
// Setup: Alice is a valid staker with non-zero STRK balance.
// Attacker (Bob) has no stake and no special role.

// At block N, Bob calls:
staking.update_rewards(alice_address, disable_rewards: true);
// Result: last_reward_block = N, no rewards distributed.

// Alice (or anyone) now tries:
staking.update_rewards(alice_address, disable_rewards: false);
// Reverts: REWARDS_ALREADY_UPDATED (current_block_number == last_reward_block)

// Alice loses all block rewards for block N.
// Bob repeats this every block → Alice (and all stakers) receive zero
// consensus block rewards indefinitely.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
