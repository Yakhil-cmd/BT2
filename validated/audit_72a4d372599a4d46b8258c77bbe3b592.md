### Title
Missing Zero-Check for `strk_total_stake` Causes Division by Zero in `calculate_staker_total_staking_power` - (File: src/staking/utils.cairo)

---

### Summary

`calculate_staker_total_staking_power` in `src/staking/utils.cairo` divides by `strk_total_stake` without a zero-guard. The analogous BTC branch explicitly guards against a zero divisor, but the STRK branch does not. When all active stakers have called `unstake_intent` — removing their stake from the total-stake trace while remaining "active" for `K` more epochs — `strk_total_stake` becomes zero and any call to `get_stakers` for those epochs panics.

---

### Finding Description

`calculate_staker_total_staking_power` computes each staker's proportional staking power:

```cairo
// src/staking/utils.cairo  lines 130-146
let strk_staking_power = mul_wide_and_div(
    lhs: staker_strk_total_amount.to_amount_18_decimals(),
    rhs: STRK_WEIGHT_FACTOR,
    div: strk_total_stake.to_amount_18_decimals(),   // ← no zero-check
)
    .unwrap();
let btc_staking_power = if btc_total_stake.is_zero() {   // ← BTC IS guarded
    Zero::zero()
} else {
    mul_wide_and_div(
        lhs: staker_btc_total_amount.to_amount_18_decimals(),
        rhs: BTC_WEIGHT_FACTOR,
        div: btc_total_stake.to_amount_18_decimals(),
    )
        .unwrap()
};
``` [1](#0-0) 

The BTC branch is explicitly guarded with `if btc_total_stake.is_zero()`, but the STRK branch has no equivalent guard. [2](#0-1) 

`unstake_intent` removes the staker's own balance **and** all delegated balances from the total-stake trace immediately, but writes `staker_unstake_intent_epoch = current_epoch + K`, meaning the staker remains "active" for `K` more epochs:

```cairo
// src/staking/staking.cairo  unstake_intent
self.staker_unstake_intent_epoch.write(staker_address, self.get_epoch_plus_k());
// ...
self.remove_from_total_stake(token_address: STRK_TOKEN_ADDRESS, amount: old_self_stake);
``` [3](#0-2) 

`get_stakers` first reads `strk_total_stake` from the trace (now 0), then iterates over all stakers and calls `get_staker_staking_power_at_epoch` for every staker that `is_staker_active` returns true for — which includes the staker who just called `unstake_intent` for the next `K` epochs:

```cairo
// src/staking/staking.cairo  get_stakers
let (strk_total_stake, btc_total_stake) = self.get_total_staking_power_at_epoch(:epoch_id);
// ...
let staking_power = self.get_staker_staking_power_at_epoch(
    :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
);
``` [4](#0-3) 

`get_staker_staking_power_at_epoch` calls `calculate_staker_total_staking_power` with `strk_total_stake = 0`, triggering a division-by-zero panic. [5](#0-4) 

---

### Impact Explanation

`get_stakers` is the consensus-layer entry point for determining per-staker staking power. A panic here prevents the off-chain consensus layer from computing the validator set for any epoch in the `[unstake_intent_epoch, unstake_intent_epoch + K)` window. This blocks attestation submission and therefore blocks reward accrual for all stakers during that window — a **temporary freezing of unclaimed yield** (High).

---

### Likelihood Explanation

Low. The trigger requires that the last remaining STRK staker (or all stakers simultaneously) calls `unstake_intent`, reducing `strk_total_stake` to zero while at least one staker is still in the active window. This is an edge case but is fully reachable by an unprivileged staker with no special access.

---

### Recommendation

Mirror the BTC zero-guard for the STRK branch:

```cairo
let strk_staking_power = if strk_total_stake.is_zero() {
    Zero::zero()
} else {
    mul_wide_and_div(
        lhs: staker_strk_total_amount.to_amount_18_decimals(),
        rhs: STRK_WEIGHT_FACTOR,
        div: strk_total_stake.to_amount_18_decimals(),
    )
        .unwrap()
};
``` [6](#0-5) 

---

### Proof of Concept

1. Deploy the staking contract with a single staker S holding the minimum STRK stake.
2. S calls `unstake_intent()` in epoch E.
   - `tokens_total_stake_trace[STRK]` is updated: `strk_total_stake = 0` from epoch E onward.
   - `staker_unstake_intent_epoch[S] = E + K`, so S is still "active" for epochs `E … E+K-1`.
3. The consensus layer calls `get_stakers(epoch_id = E)`.
4. `get_total_staking_power_at_epoch(E)` returns `strk_total_stake = 0`.
5. `is_staker_active(S, E)` returns `true` (since `E < E + K`).
6. `get_staker_staking_power_at_epoch` → `calculate_staker_total_staking_power(..., strk_total_stake = 0)`.
7. `mul_wide_and_div(..., div: 0).unwrap()` panics — `get_stakers` reverts.
8. The consensus layer cannot build the validator set; attestations for epoch E are blocked; no rewards accrue. [5](#0-4) [7](#0-6)

    staker_btc_total_amount: NormalizedAmount,
    strk_total_stake: NormalizedAmount,
    btc_total_stake: NormalizedAmount,

        lhs: staker_strk_total_amount.to_amount_18_decimals(),
        rhs: STRK_WEIGHT_FACTOR,
        div: strk_total_stake.to_amount_18_decimals(),
    )
        .unwrap();
    let btc_staking_power = if btc_total_stake.is_zero() {
        Zero::zero()

            rhs: BTC_WEIGHT_FACTOR,
            div: btc_total_stake.to_amount_18_decimals(),
        )
            .unwrap()
    };
    strk_staking_power + btc_staking_power
}

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
