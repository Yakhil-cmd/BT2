### Title
Missing Authorization Check in `TurnBridge` Allows Unprivileged Bridge State Bypass — (`File: x/cronos/keeper/msg_server.go`)

### Summary

`MsgTurnBridge` is a privileged message intended to be restricted to addresses holding the `CanTurnBridge` permission. However, the `TurnBridge` handler in `msg_server.go` is an unimplemented stub that returns `nil, nil` with no authorization check and no state mutation. Any unprivileged user can submit a `MsgTurnBridge` transaction and receive a success response, bypassing the bridge authorization gate entirely. Simultaneously, the bridge circuit breaker is permanently non-functional: even the admin cannot actually disable the bridge.

### Finding Description

`UpdateTokenMapping` — the comparable privileged message — correctly enforces its permission gate: [1](#0-0) 

The `CanTurnBridge` permission constant is defined and the permission system is wired up: [2](#0-1) 

But the `TurnBridge` handler is a stub: [3](#0-2) 

It performs no authorization check against `CanTurnBridge`, makes no call to `HasPermission`, and writes no state. The `BlockAddressesDecorator` ante handler — which does enforce the admin check for `MsgStoreBlockList` — contains no corresponding guard for `MsgTurnBridge`: [4](#0-3) 

The proto definition confirms `MsgTurnBridge` is a live, registered, publicly-routable message: [5](#0-4) 

The integration test explicitly asserts that a non-admin sender must be rejected, but the current implementation would return success for any sender: [6](#0-5) 

### Impact Explanation

Two distinct High-severity impacts arise:

1. **Bridge authorization bypass.** Any unprivileged address can submit `MsgTurnBridge` and receive a success response (`code == 0`). The `CanTurnBridge` permission check — the intended gate — is never evaluated. This directly satisfies: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."*

2. **Permanent non-functional bridge circuit breaker.** Because the handler is a no-op, the admin also cannot disable the bridge. In an active bridge exploit scenario, the emergency circuit breaker is silently inoperative, leaving bridge/conversion flows permanently open. This satisfies: *"Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows."*

### Likelihood Explanation

The message is publicly registered on the Cosmos SDK message router and callable by any funded address. No special access, leaked keys, or cryptographic break is required. The attacker only needs to submit a signed `MsgTurnBridge` transaction.

### Recommendation

Implement the `TurnBridge` handler body analogously to `UpdateTokenMapping`:

1. Check `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `sdkerrors.ErrUnauthorized` on failure.
2. Persist the bridge enable/disable state to the KV store.
3. Enforce the bridge state in the relevant gravity/IBC hooks so that the flag is actually respected at runtime.

### Proof of Concept

1. Any address `attacker` (no `CanTurnBridge` permission, not the `CronosAdmin`) submits:
   ```
   cronosd tx cronos turn-bridge false --from attacker
   ```
2. The transaction is accepted into the mempool (no ante-handler guard for `MsgTurnBridge`).
3. The transaction is included in a block and `TurnBridge` executes, returning `nil, nil` — `code == 0` success.
4. The bridge state is unchanged (no state write), so the bypass is silent.
5. Separately, the legitimate admin submits the same message and also receives `code == 0` — but the bridge is still not disabled, confirming the circuit breaker is non-functional. [3](#0-2) [7](#0-6)

```go
// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
