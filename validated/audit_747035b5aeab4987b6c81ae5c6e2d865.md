### Title
Stale Delegated Permissions Persist After `CronosAdmin` Rotation, Enabling Unauthorized Token Mapping Corruption - (File: `x/cronos/keeper/permissions.go`, `x/cronos/keeper/msg_server.go`)

---

### Summary

When governance rotates `CronosAdmin` via `MsgUpdateParams`, all sub-addresses previously granted `CanChangeTokenMapping` by the old admin retain their stored permissions in the KVStore. Because there is no enumeration or bulk-revocation mechanism, the new admin cannot clear these stale grants. Those addresses remain able to call `MsgUpdateTokenMapping` and delete active denom→contract bindings, corrupting the token-mapping state that IBC voucher conversion depends on.

---

### Finding Description

Cronos uses two separate authority layers:

1. **`k.authority`** — set at keeper initialization (always the `x/gov` module account); controls `MsgUpdateParams`.
2. **`params.CronosAdmin`** — stored in state; controls `MsgUpdatePermissions`, `MsgUpdateTokenMapping` (via `HasPermission`), and `MsgStoreBlockList`.

`MsgUpdateParams` (requiring `k.authority`) can change `params.CronosAdmin` to a new address. This is the governance-controlled admin rotation path.

The permissions sub-system (`SetPermissions` / `GetPermissions`) stores per-address permission bitmasks in the KVStore under `AdminToPermissionsKey(address)`. The old admin can grant `CanChangeTokenMapping` (bit 1) or `CanTurnBridge` (bit 2) to arbitrary addresses via `MsgUpdatePermissions`.

`HasPermission` checks:
1. Is the caller the current `params.CronosAdmin`? → implicit all-permissions.
2. Does the caller have the required bit set in the KVStore? → explicit grant.

When governance rotates `CronosAdmin`, step 1 correctly stops recognizing the old admin. However, **step 2 is never cleared**: the KVStore entries written by the old admin for sub-addresses are never deleted. There is no `GetAllPermissions` iterator or bulk-revoke function anywhere in the keeper.

`MsgUpdateTokenMapping` enforces only `HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping)`. A stale-permissioned address passes this check and can call `RegisterOrUpdateTokenMapping`. Passing an empty `Contract` field for a non-source denom triggers `DeleteExternalContractForDenom`, removing the active denom→contract binding from the KVStore. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**High — Corruption of token mappings with direct security impact.**

Deleting an active denom→contract binding via `DeleteExternalContractForDenom` causes `GetContractByDenom` to return `not found` for that denom. Any subsequent IBC voucher arrival for that denom will fail to convert to a CRC21 token (the `OnRecvVouchers` → `ConvertVouchersToEvmCoins` path calls `GetContractByDenom`). Tokens remain stranded as native Cosmos coins with no EVM representation, breaking the IBC bridge conversion flow for all users of that token.

This maps to the allowed High impact: **"Corruption of token mappings, denom/contract binding… with direct security impact"** and **"Bypass of Cronos admin… authorization checks"**. [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

Admin rotation via governance is an explicitly documented and tested operational scenario. The integration test `test_gov_update_params.py` demonstrates governance changing `cronos_admin`. The permissions sub-system (ADR-009) explicitly anticipates the old admin granting sub-addresses operational permissions. Both conditions (admin rotation + prior permission grants) are normal, expected operations. No special attacker capability is required beyond holding an address that was previously granted `CanChangeTokenMapping` by the old admin. [6](#0-5) [7](#0-6) 

---

### Recommendation

When `MsgUpdateParams` changes `params.CronosAdmin`, the keeper should iterate over all stored permission entries and delete them (or set them to zero). This requires adding a KVStore iterator over the `AdminToPermissionsKey` prefix in `SetParams` or in a dedicated `RotateAdmin` helper. Alternatively, `HasPermission` should additionally verify that the granting admin (stored alongside the permission) matches the current `CronosAdmin`, invalidating grants from a superseded admin without requiring enumeration. [8](#0-7) [9](#0-8) 

---

### Proof of Concept

1. Governance sets `CronosAdmin = adminA` via `MsgUpdateParams`.
2. `adminA` calls `MsgUpdatePermissions` granting address `X` permission `CanChangeTokenMapping` (bit 1). KVStore now holds `AdminToPermissionsKey(X) → 1`.
3. Governance passes a new `MsgUpdateParams` rotating `CronosAdmin` to `adminB`. `adminA` loses implicit all-permissions. `adminB` is the new admin.
4. `adminB` has no way to enumerate which addresses hold stored permissions; no iterator exists over `AdminToPermissionsKey`.
5. Address `X` submits `MsgUpdateTokenMapping{Denom: "ibc/...", Contract: ""}`. `HasPermission` checks: `adminB != X` (step 1 fails), then reads `GetPermissions(X) = 1 = CanChangeTokenMapping` (step 2 passes). The call succeeds.
6. `RegisterOrUpdateTokenMapping` sees empty contract → calls `DeleteExternalContractForDenom("ibc/...")`, removing the denom→CRC21 contract binding.
7. All subsequent IBC arrivals of that denom fail to convert to EVM tokens; the bridge conversion is permanently broken for that denom until a new mapping is registered by `adminB`. [1](#0-0) [10](#0-9)

```go
	store := ctx.KVStore(k.storeKey)

func (k Keeper) GetPermissions(ctx sdk.Context, address sdk.AccAddress) uint64 {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := store.Get(types.AdminToPermissionsKey(address))
	return sdk.BigEndianToUint64(permissionsBytes)
}
```

```go
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

```go
	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
```

```go
// DeleteExternalContractForDenom delete the external contract mapping for native denom,
// returns false if mapping not exists.
func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
	store := ctx.KVStore(k.storeKey)
	contract, found := k.getExternalContractByDenom(ctx, denom)
	if !found {
		return false
	}
	store.Delete(types.DenomToExternalContractKey(denom))
	deleteReverseIfOwned(store, contract, denom)
	if auto, found := k.getAutoContractByDenom(ctx, denom); found {
		bz := store.Get(types.ContractToDenomKey(auto.Bytes()))
		if len(bz) == 0 {
			store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
		} else if existingDenom := string(bz); existingDenom != denom {
			if k.contractOwnedByDenom(ctx, existingDenom, auto) {
				// auto address is already owned by another denom; drop local auto mapping
				store.Delete(types.DenomToAutoContractKey(denom))
			} else {
				// stale reverse entry
				store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
			}
		}
	}
	return true
```

```go
func (k Keeper) RegisterOrUpdateTokenMapping(ctx sdk.Context, msg *types.MsgUpdateTokenMapping) error {
	if types.IsSourceCoin(msg.Denom) {
		_, err := types.GetContractAddressFromDenom(msg.Denom)
		if err != nil {
			return err
		}

		if !common.IsHexAddress(msg.Contract) {
			return errors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid contract address (%s)", msg.Contract)
		}
		contract := common.HexToAddress(msg.Contract)
		if err := k.ensureContractCode(ctx, contract); err != nil {
			return err
		}
		if err := validateContractAddressForSourceDenom(msg.Denom, contract, true); err != nil {
			return err
		}

		if err := k.SetExternalContractForDenom(ctx, msg.Denom, contract); err != nil {
			return err
		}

		// check that the coin is registered, otherwise register it
		metadata, exist := k.bankKeeper.GetDenomMetaData(ctx, msg.Denom)
		if !exist {
			// create new metadata
			metadata = banktypes.Metadata{
				Base: msg.Denom,
				Name: msg.Denom,
			}
		}
		// update existing metadata
		metadata.Symbol = msg.Symbol
		metadata.Display = strings.ToLower(msg.Symbol)
		if msg.Decimal != 0 {
			metadata.DenomUnits = []*banktypes.DenomUnit{
				{
					Denom:    metadata.Base,
					Exponent: 0,
				},
				{
					Denom:    metadata.Display,
					Exponent: msg.Decimal,
				},
			}
		} else {
			metadata.DenomUnits = []*banktypes.DenomUnit{
				{
					Denom:    metadata.Base,
					Exponent: 0,
				},
			}
		}
		k.bankKeeper.SetDenomMetaData(ctx, metadata)
	} else {
		if len(msg.Contract) == 0 {
			// delete existing mapping
			k.DeleteExternalContractForDenom(ctx, msg.Denom)
		} else {
			if !common.IsHexAddress(msg.Contract) {
				return errors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid contract address (%s)", msg.Contract)
			}
			// update the mapping
			contract := common.HexToAddress(msg.Contract)
			if err := k.ensureContractCode(ctx, contract); err != nil {
				return err
			}
			if err := k.SetExternalContractForDenom(ctx, msg.Denom, contract); err != nil {
				return err
			}
		}
	}
```

```go
func (k Keeper) ConvertCoinFromNativeToCRC21(ctx sdk.Context, sender common.Address, coin sdk.Coin, autoDeploy bool) error {
	if !types.IsValidCoinDenom(coin.Denom) {
		return fmt.Errorf("coin %s is not supported for conversion", coin.Denom)
	}
	var err error
	// external contract is returned in preference to auto-deployed ones
	contract, found := k.GetContractByDenom(ctx, coin.Denom)
	if !found {
		if !autoDeploy {
			return fmt.Errorf("no contract found for the denom %s", coin.Denom)
		}
		contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
		if err != nil {
			return err
		}
		if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
			return err
		}

		k.Logger(ctx).Info(fmt.Sprintf("contract address %s created for coin denom %s", contract.String(), coin.Denom))
	}
```

```markdown
- Keep the admin address in Cronos space. The admin has all the permissions and has the ability to change the permission of an address.
- By default, all addresses in Cronos are non-permissioned
- Assign to each "restricted" messages in Cronos a permission (integer value) and create in Cronos module a mapping between addresses and permissions that is stored in memory. For now, there are only two messages that require permission : MsgUpdateTokenMapping and MsgTurnBridge.
- Create a msg type "MsgUpdatePermissions" that only admin can use and allow to update the address permission mapping.
- Change the logic to always check for the permission before processing the restricted messages.
```
