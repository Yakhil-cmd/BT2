### Title
Shared `*flate.Writer` in deflate pool `New` closure causes concurrent gRPC response cross-contamination — (`engine/common/grpc/compressor/deflate/deflate.go`)

### Summary

The deflate compressor's `init()` function creates a single `*flate.Writer` and captures it in the `sync.Pool.New` closure. Every invocation of `New` returns a new `*writer` wrapper struct, but all wrappers embed the **same** underlying `*flate.Writer` pointer. Under concurrent gRPC load, multiple goroutines can each receive a distinct `*writer` from the pool (all sharing the same `*flate.Writer`), then race on `Reset(dest)`, redirecting the shared writer's output to the wrong destination buffer.

### Finding Description

In `deflate.go`, `init()` allocates one `*flate.Writer` and closes over it: [1](#0-0) 

Every call to `poolCompressor.New` returns `&writer{Writer: w, ...}` where `w` is always the same pointer. `sync.Pool.New` is not serialized — when the pool is empty and N goroutines call `Get()` simultaneously, `New` is invoked N times, producing N distinct `*writer` structs that all embed the same `*flate.Writer`.

`Compress` then calls `dw.Reset(dest)` on the returned wrapper: [2](#0-1) 

`(*flate.Writer).Reset` redirects the writer's internal output to the new `io.Writer`. With two goroutines racing:

- Goroutine A: `dw_A.Reset(dest_A)` → `w` now writes to `dest_A`
- Goroutine B: `dw_B.Reset(dest_B)` → `w` now writes to `dest_B`
- Goroutine A then writes payload A through `dw_A` (which is `w`) → bytes land in `dest_B`

The snappy compressor does **not** have this bug — its `New` closure calls `snappy.NewBufferedWriter(nil)` each time, producing an independent writer per pool slot: [3](#0-2) 

### Impact Explanation

The deflate compressor is registered globally via `encoding.RegisterCompressor` and imported as a side-effect in the access node's gRPC server, the execution node's gRPC server, and the connection manager: [4](#0-3) [5](#0-4) [6](#0-5) 

In gRPC-Go, a client that sends `grpc-encoding: deflate` causes the server to compress its response using the registered deflate compressor. An attacker controlling one concurrent client can therefore force the server into the buggy code path without any operator configuration change (the `--grpc-compressor` flag governs outbound client compression, not inbound server-side response compression). Under concurrent requests, the attacker's response buffer receives bytes compressed from another client's payload — execution results, register values, or script outputs belonging to a different account.

### Likelihood Explanation

The pool is empty at startup and after any GC cycle (Go's runtime periodically clears `sync.Pool` between GC runs). Any burst of concurrent deflate-compressed gRPC calls — trivially achievable by an attacker opening multiple connections simultaneously — will trigger `New` multiple times before any writer is returned to the pool, reliably reproducing the shared-writer state.

### Recommendation

Fix the `New` closure to allocate a fresh `*flate.Writer` per call, matching the snappy pattern:

```go
c.poolCompressor.New = func() any {
    w, _ := flate.NewWriter(nil, flate.DefaultCompression)
    return &writer{Writer: w, pool: &c.poolCompressor}
}
```

This ensures each pool slot owns an independent writer, eliminating the shared-state race.

### Proof of Concept

```go
// Run 100 concurrent Compress calls through the deflate compressor.
// Each goroutine compresses a unique payload and decompresses the result.
// Without the fix, decompressed output will not match the original payload
// for some goroutines, demonstrating cross-contamination.
import (
    "bytes"; "compress/flate"; "io"; "sync"; "testing"
    _ "github.com/onflow/flow-go/engine/common/grpc/compressor/deflate"
    "google.golang.org/grpc/encoding"
)

func TestDeflateRace(t *testing.T) {
    c := encoding.GetCompressor("deflate")
    var wg sync.WaitGroup
    for i := 0; i < 100; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            payload := []byte(fmt.Sprintf("payload-%d", id))
            var buf bytes.Buffer
            w, _ := c.Compress(&buf)
            w.Write(payload); w.Close()
            r, _ := c.Decompress(&buf)
            got, _ := io.ReadAll(r)
            if !bytes.Equal(got, payload) {
                t.Errorf("id=%d: got %q, want %q", id, got, payload)
            }
        }(i)
    }
    wg.Wait()
}
```
