### Title
No Grace Period After EVM Unpause Exposes Users to Immediate Liquidation - (File: fvm/evm/stdlib/contract.cdc)

### Summary
The Flow EVM system contract (`fvm/evm/stdlib/contract.cdc`) implements a pause mechanism via `isPaused()` that blocks all state-mutating EVM operations. When EVM is paused, users cannot call EVM contracts, withdraw assets from their COA, or bridge tokens out. There is no grace period after unpausing: the moment EVM is unpaused, any unprivileged liquidator can immediately liquidate EVM DeFi positions that became undercollateralized during the pause window, before affected users have any opportunity to add collateral or close positions.

### Finding Description
The `isPaused()` function reads a boolean from `/storage/evmOperationsPaused` and returns `false` by default if absent. [1](#0-0) 

Every state-mutating user-facing function in the contract gates on this check:

- `CadenceOwnedAccount.withdraw` (FLOW withdrawal from COA) [2](#0-1) 
- `CadenceOwnedAccount.call` / `callWithSigAndArgs` (EVM contract interaction) [3](#0-2) 
- `CadenceOwnedAccount.withdrawTokens` / `withdrawNFT` (bridge-out) [4](#0-3) 
- `EVM.run` / `EVM.batchRun` (raw EVM transaction submission) [5](#0-4) 

Only read-only `dryRun` / `dryCall` functions are explicitly exempted from the pause guard. [6](#0-5) 

The pause is toggled by saving/removing a `Bool` at `/storage/evmOperationsPaused`. There is no `minUnpauseTimestamp`, no grace-period counter, and no event or mechanism that would delay liquidation eligibility after the flag is cleared. The contract has no `unpause()` function that could enforce a delay — the state simply flips from paused to unpaused atomically. [7](#0-6) 

### Impact Explanation
EVM DeFi protocols deployed on Flow EVM (lending, perpetuals, etc.) rely on users being able to call `EVM.run`, `EVM.batchRun`, or `COA.call` to add collateral, repay debt, or close positions. During a pause:

1. Prices can move adversely (oracle feeds continue updating off-chain).
2. Time-based accrual (interest, funding rates) continues inside EVM state.
3. Users cannot submit any protective EVM transaction.

The instant EVM is unpaused, any unprivileged liquidator can call the EVM DeFi protocol's liquidation function (via `EVM.run` or `EVM.batchRun`) and seize collateral from positions that became undercollateralized during the pause. The affected user had no opportunity to act. This constitutes **cross-VM asset loss** for COA owners and EVM DeFi participants.

### Likelihood Explanation
The Governance Committee can pause EVM for legitimate maintenance. Pauses are expected to occur. The duration of a pause can be hours or days. Any liquidator monitoring the chain can observe the unpause transaction in the same block it is finalized and front-run affected users in the very next block. No special privilege is required to execute the liquidation — only the ability to submit a Cadence transaction calling `EVM.run` or `EVM.batchRun` with a valid liquidation payload.

### Recommendation
Introduce a `minUnpauseTimestamp` storage variable that is set to `block.timestamp + GRACE_PERIOD` when EVM is unpaused. Gate liquidation-eligible calls (or all `run`/`batchRun`/`call` calls) on this timestamp, giving users a window (e.g., 6–24 hours) to restore their positions before liquidators can act. Alternatively, allow `withdraw` and bridge-out operations (`withdrawTokens`, `withdrawNFT`) to remain available while EVM is paused, so users can at minimum exit their COA positions even if they cannot interact with EVM DeFi contracts.

### Proof of Concept

1. Governance Committee saves `true` to `/storage/evmOperationsPaused` on the EVM contract account, pausing EVM.
2. Over the next N hours, ETH/FLOW price drops 20%. A user's COA-backed lending position on a Flow EVM lending protocol becomes undercollateralized.
3. The user attempts to call `coa.call(to: lendingProtocol, data: addCollateralCalldata, ...)` — this panics with `"EVM operations are temporarily paused"`. [3](#0-2) 
4. Governance Committee saves `false` to `/storage/evmOperationsPaused`, unpausing EVM.
5. In the same block (or the next), an unprivileged liquidator submits:
   ```cadence
   import EVM from <EVMContractAddress>
   transaction(liquidateTx: [UInt8], coinbaseBytes: [UInt8; 20]) {
       prepare(acct: &Account) {
           let coinbase = EVM.EVMAddress(bytes: coinbaseBytes)
           let result = EVM.run(tx: liquidateTx, coinbase: coinbase)
           // result.status == successful
       }
   }
   ``` [8](#0-7) 
6. The lending protocol liquidates the user's position. The user loses their collateral with no opportunity to have protected it.
