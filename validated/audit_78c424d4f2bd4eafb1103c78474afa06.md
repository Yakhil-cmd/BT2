### Title
Gateway Stateful Validator Checks Only L2 Gas Price, Admitting Transactions With Zero L1/L1-Data Gas Prices That Blockifier Rejects - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds validation only checks the transaction's `l2_gas.max_price_per_unit` against the previous block's L2 gas price. It never checks `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The blockifier's `check_fee_bounds`, called during `perform_pre_validation_stage`, enforces all three prices. A transaction with zero L1 and L1-data gas prices passes every gateway admission check but is unconditionally rejected by the blockifier, creating a systematic mismatch between what the gateway admits and what execution enforces.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads only the previous block's STRK L2 gas price and delegates to `validate_tx_l2_gas_price_within_threshold`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:223-243
async fn validate_resource_bounds(&self, executable_tx: &ExecutableTransaction) -> ... {
    if self.config.validate_resource_bounds {
        let previous_block_l2_gas_price = self
            .gateway_fixed_block_state_reader
            .get_block_info().await?
            .gas_prices.strk_gas_prices.l2_gas_price;
        self.validate_tx_l2_gas_price_within_threshold(
            executable_tx.resource_bounds(),
            previous_block_l2_gas_price,
        )?;
    }
    Ok(())
}
```

`validate_tx_l2_gas_price_within_threshold` only inspects `l2_gas.max_price_per_unit` and explicitly skips legacy `L1Gas` transactions entirely:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:358-390

    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // ... only l2_gas checked
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }

```

The stateless validator's `validate_resource_bounds` also only checks `l2_gas.max_price_per_unit >= min_gas_price` and `l2_gas.max_amount <= max_l2_gas_amount`, with no check on L1 or L1-data gas prices:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs:56-88
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(...MaxGasPriceTooLow...);
}
// l1_gas.max_price_per_unit and l1_data_gas.max_price_per_unit: never checked
```

In contrast, the blockifier's `check_fee_bounds` (called unconditionally from `perform_pre_validation_stage` when `charge_fee` is true) validates all three prices against the current block:

```rust
// crates/blockifier/src/transaction/account_transaction.rs:398-424
ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_gas_resource_bounds,
    l2_gas: l2_gas_resource_bounds,
    l1_data_gas: l1_data_gas_resource_bounds,
}) => {
    // checks l1_gas_price, l1_data_gas_price, l2_gas_price
    if resource_bounds.max_price_per_unit < actual_gas_price.get() {
        // InsufficientResourceBounds error
    }
}
```

A transaction with `l1_gas.max_price_per_unit = 0`, `l1_data_gas.max_price_per_unit = 0`, and `l2_gas.max_price_per_unit >= min_gas_price` passes every gateway check (the total fee is non-zero because L2 gas is non-zero, satisfying the `max_possible_fee != 0` guard) but is rejected by the blockifier at `check_fee_bounds` because the actual block L1 gas price is non-zero.

### Impact Explanation

Any unprivileged user can craft `AllResources` transactions with zero L1 and L1-data gas prices that pass both stateless and stateful gateway admission, enter the mempool, and are then unconditionally rejected by the blockifier during batcher execution. This allows systematic flooding of the mempool with transactions that will never execute, wasting mempool capacity and batcher execution resources, and potentially delaying or displacing legitimate transactions. This matches the "High. Mempool/gateway/RPC admission accepts invalid transactions before sequencing" impact category.

### Likelihood Explanation

The attack requires only a standard RPC call with crafted resource bounds. No privileged access, special account state, or race condition is needed. The TODO comment in the production code (`// TODO(Arni): Consider running this validation for all gas prices.`) confirms the gap is known but unaddressed. Any attacker who reads the gateway validation code can exploit this immediately.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also check `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK prices, applying the same `min_gas_price_percentage` threshold. This mirrors what `check_fee_bounds` enforces in the blockifier and closes the admission/execution gap.

### Proof of Concept

1. Construct a V3 invoke transaction with:
   - `l1_gas.max_price_per_unit = 0`
   - `l1_data_gas.max_price_per_unit = 0`
   - `l2_gas.max_price_per_unit = min_gas_price` (passes stateless check)
   - `l2_gas.max_amount > 0` (makes `max_possible_fee != 0`)
2. Submit via the gateway RPC endpoint.
3. `StatelessTransactionValidator::validate_resource_bounds` passes: total fee is non-zero (L2 gas contributes), L2 gas price meets `min_gas_price`.
4. `StatefulTransactionValidator::validate_resource_bounds` passes: only L2 gas price is checked against the previous block threshold.
5. Transaction enters the mempool.
6. Batcher picks it up and calls `AccountTransaction::execute` → `perform_pre_validation_stage` → `check_fee_bounds`.
7. `check_fee_bounds` finds `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (non-zero)` and returns `InsufficientResourceBounds`, rejecting the transaction.
8. Repeat at high volume to exhaust mempool capacity with permanently-invalid transactions. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,

                let gas_price_threshold_multiplier =

                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());

        let resource_bounds = *tx.resource_bounds();

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
