### Title
`validate_resource_bounds` Unconditionally Checks L2 Gas Price Against `min_gas_price`, Rejecting Valid L1-Only Transactions at Gateway Admission — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

`StatelessTransactionValidator::validate_resource_bounds` applies a `min_gas_price` floor check to `l2_gas.max_price_per_unit` for every V3 transaction, regardless of whether the transaction actually allocates any L2 gas. A transaction that legitimately sets only L1 (or L1-data) gas bounds — leaving `l2_gas` at its zero default — is unconditionally rejected at the gateway stateless path because `0 < min_gas_price (8_000_000_000)`. The test suite masks this by setting `min_gas_price: 0` in `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING`, so the `valid_l1_gas` positive test passes in CI while the production default silently rejects the same transaction class.

---

### Finding Description

In `validate_resource_bounds`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs  line 64-76
let resource_bounds = *tx.resource_bounds();          // always AllResourceBounds for V3
// line 66: passes if any resource is non-zero
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(...ZeroResourceBounds...);
}
// line 71: unconditional L2 price floor — no guard on l2_gas.max_amount
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
``` [1](#0-0) 

The production default is `min_gas_price: 8_000_000_000`: [2](#0-1) 

The test config deliberately sets `min_gas_price: 0` and `validate_resource_bounds: false`, which hides the production mismatch: [3](#0-2) 

The `valid_l1_gas` positive test case constructs a transaction with `l2_gas = Default::default()` (price = 0) and expects `Ok(())`: [4](#0-3) 

That test only passes because `min_gas_price = 0` in the test config. Under the production default of `8_000_000_000`, the same transaction hits `0 < 8_000_000_000 → true` and is rejected with `MaxGasPriceTooLow`.

The same structural flaw exists in the stateful path: `validate_tx_l2_gas_price_within_threshold` also checks `l2_gas.max_price_per_unit` for all `AllResources` transactions without guarding on `l2_gas.max_amount`: [5](#0-4) 

---

### Impact Explanation

**High — Mempool/gateway admission rejects valid transactions before sequencing.**

Any V3 transaction that allocates zero L2 gas (setting only `l1_gas` or `l1_data_gas` bounds, which is a valid and protocol-conformant choice) is permanently blocked at the stateless gateway check. The transaction never reaches the mempool, blockifier, or execution. The user receives a `MaxGasPriceTooLow` error that is misleading because the transaction carries no L2 gas at all. This is a denial-of-service on a valid transaction class, matching the "rejects valid transactions before sequencing" impact criterion.

---

### Likelihood Explanation

Any unprivileged user can trigger this by submitting a V3 `Invoke`, `Declare`, or `DeployAccount` transaction with `l2_gas = AllResourceBounds::default()` (i.e., `max_amount = 0`, `max_price_per_unit = 0`) and non-zero `l1_gas` or `l1_data_gas` bounds. No special account state, privileged access, or race condition is required. The gateway is publicly reachable via HTTP.

---

### Recommendation

Guard the L2 gas price floor check on whether the transaction actually requests L2 gas. The check should only apply when `l2_gas.max_amount > 0`:

```rust
// Only enforce the L2 gas price floor when the transaction actually uses L2 gas.
if resource_bounds.l2_gas.max_amount.0 > 0
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
```

Apply the same fix to `validate_tx_l2_gas_price_within_threshold` in the stateful validator. Also update the `valid_l1_gas` test to use the production `min_gas_price` value so the regression is caught by CI.

---

### Proof of Concept

1. Construct a V3 Invoke transaction with:
   - `l1_gas: ResourceBounds { max_amount: GasAmount(1000), max_price_per_unit: GasPrice(1) }`
   - `l2_gas: ResourceBounds::default()` (price = 0, amount = 0)
   - `l1_data_gas: ResourceBounds::default()`

2. Submit to the gateway running with the production `StatelessTransactionValidatorConfig::default()` (`min_gas_price = 8_000_000_000`, `validate_resource_bounds = true`).

3. `max_possible_fee(Tip::ZERO)` = `1000 * 1 = 1000 > 0` → first check passes.

4. `resource_bounds.l2_gas.max_price_per_unit.0 = 0 < 8_000_000_000` → second check fires → transaction rejected with `MaxGasPriceTooLow { gas_price: 0, min_gas_price: 8_000_000_000 }`.

5. The transaction is valid (non-zero fee, correct format) and should have been admitted.

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
