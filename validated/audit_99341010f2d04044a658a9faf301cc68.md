### Title
Secret Signing Share Disclosed via `Debug` Trait on `KeygenOutput` — (File: `src/lib.rs`)

---

### Summary

`KeygenOutput<C>`, the primary output of DKG/reshare/refresh, derives `Debug` while containing `private_share: SigningShare<C>` as a public field. Any caller that formats this type with `{:?}` — through logging, panic output, test assertion failures, or tracing frameworks — will emit the raw secret signing share into whatever sink receives that output. The same pattern repeats across every presign output type in the library.

---

### Finding Description

`KeygenOutput<C>` is defined in `src/lib.rs` as:

```rust
#[derive(Debug, Clone, Deserialize, Serialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct KeygenOutput<C: Ciphersuite> {
    pub private_share: SigningShare<C>,   // ← secret scalar
    #[zeroize[skip]]
    pub public_key: VerifyingKey<C>,
}
```

The `Debug` derive is unconditional and recursive: it will call `Debug` on `SigningShare<C>`, which in `frost_core` exposes the underlying scalar bytes. Because `private_share` is also a `pub` field, any downstream struct that wraps `KeygenOutput` and itself derives `Debug` will transitively leak the share.

The same issue exists in every presign output type:

| Type | File | Secret fields exposed via `Debug` |
|---|---|---|
| `PresignOutput` (OT-based) | `src/ecdsa/ot_based_ecdsa/mod.rs:40` | `k` (nonce share), `sigma` |
| `RerandomizedPresignOutput` (OT-based) | `src/ecdsa/ot_based_ecdsa/mod.rs:54` | `k`, `sigma` |
| `PresignOutput` (Robust ECDSA) | `src/ecdsa/robust_ecdsa/mod.rs:26` | `c`, `e`, `alpha`, `beta` |
| `RerandomizedPresignOutput` (Robust ECDSA) | `src/ecdsa/robust_ecdsa/mod.rs:42` | `e`, `alpha`, `beta` |
| `PresignOutput<C>` (FROST) | `src/frost/mod.rs:36` | `nonces` (`SigningNonces` contains secret scalars) |

The developers clearly recognize the sensitivity of these fields — `ZeroizeOnDrop` is applied to `KeygenOutput`, `PresignOutput`, and `RerandomizedPresignOutput` — yet `Debug` is derived without redaction, creating a contradiction: the memory is zeroed on drop, but the value is freely printable while alive.

---

### Impact Explanation

**Critical — Disclosure of private signing shares and presign secret material.**

- `private_share` in `KeygenOutput` is the per-participant secret key share. If ≥ `threshold` shares are disclosed, the master private key can be reconstructed via Lagrange interpolation, enabling unauthorized creation of valid signatures for any message.
- `k` / `sigma` / `c` / `e` / `alpha` / `beta` in presign outputs are one-time secret nonce material. Disclosure of `k` (the nonce share) combined with the corresponding signature allows recovery of the signing share via the ECDSA equation `s = k⁻¹(hash + r·x)`, directly leaking the private key share.

Both impact classes map to the Critical tier: *Extraction, reconstruction, or disclosure of private signing shares, presign secrets, or nonce material.*

---

### Likelihood Explanation

**Medium-to-High.** The `Debug` derive is the default Rust mechanism for diagnostic output. Library callers routinely:

- Wrap `KeygenOutput` in their own structs and derive `Debug` on those structs.
- Use `tracing` or `log` crates with `{:?}` formatting in error paths.
- Rely on test frameworks (e.g., `assert_eq!`) that print both sides of a failed comparison using `Debug`, which would emit the private share to stdout/stderr in CI logs.
- Store `KeygenOutput` in `Result<KeygenOutput, _>` and call `.unwrap()`, whose panic message uses `Debug`.

No attacker capability beyond reading log output or CI artifacts is required.

---

### Recommendation

1. **Remove `Debug` from all types containing secret scalars**, or implement it manually to redact secret fields:

```rust
impl<C: Ciphersuite> std::fmt::Debug for KeygenOutput<C> {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.debug_struct("KeygenOutput")
            .field("private_share", &"<redacted>")
            .field("public_key", &self.public_key)
            .finish()
    }
}
```

2. Apply the same manual `Debug` implementation to `PresignOutput` and `RerandomizedPresignOutput` in all three scheme modules.

3. Consider using the [`secrecy`](https://crates.io/crates/secrecy) crate's `Secret<T>` wrapper, which explicitly opts out of `Debug` and `Display` for wrapped values.

---

### Proof of Concept

```rust
use threshold_signatures::{keygen, KeygenOutput};
use threshold_signatures::participants::Participant;

// After a successful DKG run, a caller stores the output:
let output: KeygenOutput<_> = run_keygen(...);

// Any of the following leaks the raw private_share scalar:
println!("{:?}", output);                    // direct log
tracing::debug!(?output, "keygen done");     // tracing subscriber
assert_eq!(output, expected_output);         // test failure prints both sides
let s = format!("{output:?}");               // stored in a string, e.g. for an error report
```

The `private_share` scalar value appears verbatim in the formatted output. An attacker with access to logs, CI output, or error reports for ≥ `threshold` participants can reconstruct the master private key and forge signatures for arbitrary messages.
