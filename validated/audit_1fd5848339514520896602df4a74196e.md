### Title
Blocklist Update Takes Effect Only After EndBlocker, Allowing Same-Block Bypass by To-Be-Blocked Address - (File: app/app.go, app/proposal.go, x/cronos/keeper/msg_server.go)

### Summary
Cronos's dynamic on-chain blocklist (`MsgStoreBlockList`) is enforced exclusively through the in-memory `ProposalHandler.blocklist` map, which is refreshed only in `EndBlocker` via `RefreshBlockList`. Because `EndBlocker` runs after all transactions in a block have already been executed, any transaction from a to-be-blocked address that lands in the same block as the `MsgStoreBlockList` update executes without restriction. This is a direct analog of the oracle pause-bypass: the "pause" (blocklist update) is forward-facing only and does not affect the current block.

### Finding Description

The dynamic blocklist flow is:

1. Admin submits `MsgStoreBlockList` → `StoreBlockList` in `x/cronos/keeper/msg_server.go` writes the encrypted blob to `KeyPrefixBlockList` in the KV store. [1](#0-0) 

2. `EndBlocker` calls `RefreshBlockList`, which calls `blockProposalHandler.SetBlockList(app.CronosKeeper.GetBlockList(ctx))` to decrypt and load the new addresses into the in-memory `h.blocklist` map. [2](#0-1) 

3. `PrepareProposalHandler` and `ProcessProposalHandler` both read from `h.blocklist` to filter/reject blocked-address transactions. [3](#0-2) 

The critical ordering is:

```
Block N: FinalizeBlock (all txs execute) → EndBlocker (RefreshBlockList updates h.blocklist)
Block N+1: PrepareProposal/ProcessProposal use the updated h.blocklist
```

`EndBlocker` runs **after** all transactions in block N have already been executed. Therefore, the `MsgStoreBlockList` tx in block N updates the KV store, but `h.blocklist` is not refreshed until after execution completes. Any transaction from the to-be-blocked address that is also in block N executes against the **old** (pre-update) in-memory blocklist.

The `BlockAddressesDecorator` ante handler — the only other enforcement point — only runs when `ctx.IsCheckTx()` is true, providing no protection during `FinalizeBlock`. [4](#0-3) 

`ProcessProposalHandler` for block N uses the blocklist from block N-1's `EndBlocker`, which does not yet include the attacker's address, so it accepts the block. [3](#0-2) 

### Impact Explanation

An address that the admin is attempting to block can monitor the mempool for a `MsgStoreBlockList` transaction targeting it, then submit a transaction (token transfer, bridge operation, EVM call, IBC transfer) in the same block. Because `h.blocklist` is only updated in `EndBlocker` — after all block transactions execute — the attacker's transaction executes without the blocklist check applying. This constitutes a **bypass of the Cronos admin blocklist authorization check**, allowing the attacker to perform actions (e.g., drain funds, bridge assets) that the blocklist is specifically intended to prevent.

This matches the allowed High impact: "Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."

### Likelihood Explanation

The attacker is the address being blocked — an unprivileged actor. The attack requires only:
1. Monitoring the public mempool for a `MsgStoreBlockList` transaction
2. Submitting a transaction with sufficient gas priority to land in the same block

No privileged access, leaked keys, or validator compromise is required. The window is exactly one block, but that is sufficient to execute a transfer or bridge operation.

### Recommendation

Move `RefreshBlockList` from `EndBlocker` to `PreBlocker` (before transaction execution), so the in-memory blocklist is updated before any transactions in the block are processed:

```go
func (app *App) PreBlocker(ctx sdk.Context, req *abci.RequestFinalizeBlock) (*sdk.ResponsePreBlock, error) {
    if err := app.RefreshBlockList(ctx); err != nil {
        app.Logger().Error("failed to update blocklist", "error", err)
    }
    return app.ModuleManager.PreBlock(ctx)
}
```

This ensures that when `MsgStoreBlockList` is committed in block N, the updated blocklist is active for all transactions in block N+1 **and** the in-memory state is consistent with the committed KV state before any tx in the same block executes. Additionally, `BlockAddressesDecorator` should remove the `ctx.IsCheckTx()` guard so the static blocklist is also enforced during `FinalizeBlock`. [5](#0-4) 

### Proof of Concept

1. Admin prepares `MsgStoreBlockList` tx adding attacker address `A` to the blocklist and broadcasts it to the mempool.
2. Attacker monitors the mempool, sees the `MsgStoreBlockList` tx, and immediately broadcasts a `MsgConvertVouchers` (or EVM transfer) tx from address `A` with a gas price high enough to land in the same block.
3. Both transactions are included in block N. `ProcessProposalHandler` uses the blocklist from block N-1's `EndBlocker`, which does not contain `A`, so the block is accepted. [6](#0-5) 
4. During `FinalizeBlock` for block N, the attacker's tx executes (ante handler skips blocklist check because `IsCheckTx()` is false), then `MsgStoreBlockList` executes and writes the new blob to the KV store. [1](#0-0) 
5. `EndBlocker` calls `RefreshBlockList` — only now is `h.blocklist` updated with address `A`. [2](#0-1) 
6. Block N+1 onward: address `A` is blocked. But the attacker's transaction in block N has already executed successfully, bypassing the intended blocklist restriction.

func (k msgServer) StoreBlockList(goCtx context.Context, msg *types.MsgStoreBlockList) (*types.MsgStoreBlockListResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	ctx.KVStore(k.storeKey).Set(types.KeyPrefixBlockList, msg.Blob)
	return &types.MsgStoreBlockListResponse{}, nil

// EndBlocker application updates every end block
func (app *App) EndBlocker(ctx sdk.Context) (sdk.EndBlock, error) {
	rsp, err := app.ModuleManager.EndBlock(ctx)
	if err := app.RefreshBlockList(ctx); err != nil {
		app.Logger().Error("failed to update blocklist", "error", err)
	}
	return rsp, err
}

func (app *App) RefreshBlockList(ctx sdk.Context) error {
	// refresh blocklist
	return app.blockProposalHandler.SetBlockList(app.CronosKeeper.GetBlockList(ctx))
}

func (h *ProposalHandler) ProcessProposalHandler() sdk.ProcessProposalHandler {
	return func(ctx sdk.Context, req *abci.RequestProcessProposal) (*abci.ResponseProcessProposal, error) {
		if len(h.blocklist) == 0 {
			// fast path, accept all txs
			return &abci.ResponseProcessProposal{Status: abci.ResponseProcessProposal_ACCEPT}, nil
		}

		for _, txBz := range req.Txs {
			if err := h.ValidateTransaction(nil, txBz); err != nil {
				return &abci.ResponseProcessProposal{Status: abci.ResponseProcessProposal_REJECT}, nil
			}
		}

		return &abci.ResponseProcessProposal{Status: abci.ResponseProcessProposal_ACCEPT}, nil
	}
}

func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)
