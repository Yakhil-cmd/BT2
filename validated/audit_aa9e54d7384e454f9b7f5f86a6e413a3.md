## Key Findings

### 1. Missing Access Control

The spec at line 1645 states access control is **"Only starkware sequencer"**, but the implementation has **no caller check**: [1](#0-0) 

`general_prerequisites()` only checks the pause flag: [2](#0-1) 

There is no `get_caller_address()` check, no role assertion, and no sequencer-only gate anywhere in `update_rewards`.

### 2. Global `last_reward_block` Written Before the `disable_rewards` Guard

The critical ordering flaw: [3](#0-2) 

`last_reward_block` is written at line 1485 **unconditionally**, before the `disable_rewards` early-return at line 1487. This is a single global slot — not per-staker.

### 3. The Attack

In any block N where consensus rewards are active:
1. Attacker calls `update_rewards(any_valid_active_staker, disable_rewards=true)`
2. The block check passes (`current_block_number > last_reward_block`)
3. `last_reward_block` is set to block N
4. Function returns early — **zero rewards distributed**
5. Any subsequent call in block N (including the legitimate sequencer call) reverts with `REWARDS_ALREADY_UPDATED` [4](#0-3) 

The block's reward opportunity is permanently consumed. There is no recovery path — `last_reward_block` cannot be reset, and the block cannot be revisited.

### Title
Unprivileged caller can permanently suppress per-block consensus rewards via `update_rewards` with `disable_rewards=true` — (`src/staking/staking.cairo`)

### Summary
`update_rewards` is documented as sequencer-only but has no caller check. Any address can call it with `disable_rewards=true`, which writes the global `last_reward_block` before the early-return guard, permanently consuming the block's reward slot and zeroing out that block's yield for all stakers.

### Finding Description
`IStakingRewardsManager::update_rewards` is exposed as a public ABI function with no role or caller restriction. The spec mandates "Only starkware sequencer" access, but `general_prerequisites()` only asserts the contract is unpaused. The global `last_reward_block` is written unconditionally at line 1485 before the `disable_rewards` guard at line 1487. Since only one `update_rewards` call can succeed per block (enforced by the `REWARDS_ALREADY_UPDATED` check), an attacker who calls first with `disable_rewards=true` permanently discards that block's rewards.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.** An attacker can call this every block, suppressing 100% of consensus rewards for all stakers indefinitely. The lost rewards are never redistributed; `last_reward_block` is already set and the block cannot be replayed.

### Likelihood Explanation
High. The call requires no stake, no funds, no privileged role — just a valid active staker address (publicly enumerable via `get_stakers`). The attacker pays only gas. The attack is repeatable every block.

### Recommendation
Add a sequencer-only caller check at the top of `update_rewards`, consistent with the spec's stated access control. For example, assert `get_caller_address() == sequencer_address` where `sequencer_address` is a stored, governance-controlled value. Alternatively, move the `last_reward_block.write` to after the `disable_rewards` guard so that a suppressed call does not consume the block slot.

### Proof of Concept
1. Deploy with two active stakers A and B, consensus rewards active.
2. In block N, attacker (any EOA) calls `update_rewards(staker_A, disable_rewards=true)`.
3. `last_reward_block` is now N; call returns with no rewards distributed.
4. Legitimate sequencer calls `update_rewards(staker_A, disable_rewards=false)` → reverts `REWARDS_ALREADY_UPDATED`.
5. Advance to block N+1; repeat step 2 for staker B.
6. After many blocks, both stakers have zero accumulated rewards despite being eligible every block. [5](#0-4) [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
