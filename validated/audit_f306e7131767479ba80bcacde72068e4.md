### Title
Accrued Admin Fees Are Not Collected Before Destination Change in `setPoolAdminFeeDestination` - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary
`MetricOmmPoolFactory::setPoolAdminFeeDestination` updates `poolAdminFeeDestination[pool]` without first flushing accrued admin fees to the old destination. Any spread-fee surplus and notional-fee accumulator balances that were earned under the old destination are silently redirected to the new destination on the next `collectPoolFees` call, permanently depriving the old destination of its owed fees.

### Finding Description
The factory maintains two types of accrued admin fees on each pool:

1. **Spread fees** — held as a token surplus (`balance - binTotals - notionalFees`) on the pool contract, split at collection time using the stored `adminSpreadFeeE6` ratio.
2. **Notional fees** — tracked in `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` on the pool, split at collection time using the stored `adminNotionalFeeE8` ratio.

Both are only realized when `collectFees` is called (either via `collectPoolFees` or as a side-effect of `setPoolAdminFees` / `setPoolProtocolFee`). The factory correctly guards every other fee-state mutation with a prior collection:

- `setPoolAdminFees` calls `collectFees` before updating rates. [1](#0-0) 
- `setPoolProtocolFee` calls `collectFees` before updating rates. [2](#0-1) 

However, `setPoolAdminFeeDestination` simply overwrites `poolAdminFeeDestination[pool]` with no prior collection:

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;          // ← no collectFees first
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
``` [3](#0-2) 

After this call, the next invocation of `collectPoolFees` reads the updated `poolAdminFeeDestination[pool]` and sends the entire accumulated admin share — including fees earned before the destination change — to the new address:

```solidity
function collectPoolFees(address pool) external override nonReentrant {
    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6, c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8, c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]   // ← now points to new address
    );
}
``` [4](#0-3) 

The old destination has no mechanism to claim its owed share after the pointer is overwritten; the fees are permanently lost to it.

### Impact Explanation
The old `adminFeeDestination` (e.g., a treasury contract or multisig) loses 100% of its accrued admin fees — both spread-fee surplus and notional-fee accumulator balances — that were earned before the destination change. These fees are irrecoverably redirected to the new destination. Per the contest severity definitions, a loss of to-be-claimed fees exceeding 20% and 100 USD qualifies as **High** severity. [5](#0-4) 

### Likelihood Explanation
The trigger is a valid, permissioned pool-admin action (`setPoolAdminFeeDestination`) that any pool admin may call at any time. No external attacker is required. The loss occurs whenever the pool admin changes the fee destination after fees have accrued — a routine operational event (treasury rotation, multisig migration). The pool admin is semi-trusted and this call is within their documented role, but the missing pre-collection is an unintended omission inconsistent with every other fee-state mutation in the factory.

### Recommendation
Add a `collectFees` call before updating the destination, mirroring the pattern used in `setPoolAdminFees` and `setPoolProtocolFee`:

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();

    // Flush accrued fees to the OLD destination before rotating the pointer
    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6,
        c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8,
        c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]   // old destination
    );

    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

### Proof of Concept

```solidity
function testAdminFeesLostOnDestinationChange() public {
    // 1. Pool has accrued admin fees (spread surplus + notional accumulator)
    //    after several swaps with non-zero adminSpreadFeeE6 / adminNotionalFeeE8.
    address oldDest = poolAdminFeeDestination[pool];
    address newDest = makeAddr("newTreasury");

    uint256 oldDestBefore0 = token0.balanceOf(oldDest);
    uint256 oldDestBefore1 = token1.balanceOf(oldDest);

    // 2. Pool admin rotates the fee destination WITHOUT collecting first.
    vm.prank(admin);
    factory.setPoolAdminFeeDestination(pool, newDest);

    // 3. Anyone collects fees — they go entirely to newDest.
    factory.collectPoolFees(pool);

    // 4. Old destination received nothing despite having earned fees.
    assertEq(token0.balanceOf(oldDest), oldDestBefore0, "old dest lost token0 fees");
    assertEq(token1.balanceOf(oldDest), oldDestBefore1, "old dest lost token1 fees");

    // 5. New destination received fees it did not earn.
    assertGt(token0.balanceOf(newDest), 0, "new dest got token0 fees it didn't earn");
}
```
