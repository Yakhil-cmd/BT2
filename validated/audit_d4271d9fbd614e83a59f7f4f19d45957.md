### Title
`SyncStateReader::get_compiled_class_hash` is `todo!()` — any declare transaction panics the gateway process - (File: crates/apollo_gateway/src/sync_state_reader.rs)

### Summary

`SyncStateReader` is the production `BlockifierStateReader` implementation used inside the gateway's stateful validation path. Its `get_compiled_class_hash` method body is `todo!()`, which unconditionally panics at runtime. Because the blockifier calls `get_compiled_class_hash` during declare-transaction execution (to check for re-declaration), any unprivileged user who submits a syntactically valid declare transaction will crash the gateway worker thread, making the gateway unavailable.

### Finding Description

`SyncStateReader` implements the `BlockifierStateReader` trait for the gateway's stateful validator. Every method is properly implemented except `get_compiled_class_hash`:

```rust
// crates/apollo_gateway/src/sync_state_reader.rs  line 197-199
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
```

The production enum `SyncOrGenesisStateReader` wraps `SyncStateReader` and delegates the same method directly to it:

```rust
// line 443-450
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    match self {
        Self::Sync(state_reader) => state_reader.get_compiled_class_hash(class_hash),
        Self::Genesis(genesis_state_reader) => {
            genesis_state_reader.get_compiled_class_hash(class_hash)
        }
    }
}
```

`SyncOrGenesisStateReader` is the concrete type returned by `SyncStateReaderFactory::get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block` and is passed into `StatefulTransactionValidator` as its state reader.

The gateway stateful validation path for a declare transaction is:

1. `extract_state_nonce_and_run_validations` → `run_validate_entry_point`
2. `run_validate_entry_point` creates a `CachedState<SyncOrGenesisStateReader>` and calls `StatefulValidator::validate(account_tx)`
3. `StatefulValidator::perform_validations` matches on `ApiTransaction::Declare(_)` and calls `self.execute(tx)` — the **full** execution path, not just the validate entry point
4. During declare execution the blockifier calls `state.get_compiled_class_hash(class_hash)` to detect re-declaration; `CachedState` forwards the miss to the underlying `SyncOrGenesisStateReader`, which delegates to `SyncStateReader::get_compiled_class_hash`, which panics

Note that `versioned_constants.disable_casm_hash_migration()` is applied only inside `run_validate_entry_point` for the invoke branch; it does not suppress the re-declaration check that calls `get_compiled_class_hash` during declare execution.

The `GenesisStateReader` counterpart has a real implementation that returns `CompiledClassHash::default()`, confirming the `SyncStateReader` implementation is simply missing.

### Impact Explanation

Any unprivileged user can submit a well-formed declare transaction (passes stateless validation and class-manager compilation) to the gateway. When stateful validation runs, the `todo!()` panic propagates out of the `spawn_blocking` task. The gateway worker handling that request terminates abnormally. Depending on the runtime configuration this either crashes the gateway process entirely or kills the blocking thread pool, making the gateway unable to accept further transactions — a complete denial of service for transaction admission.

**Impact category matched:** *High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.*

### Likelihood Explanation

Triggering the bug requires only a syntactically valid declare transaction that passes stateless checks (valid Sierra version, class size within limits, sorted entry points, matching compiled class hash). No privileged access, special account, or unusual network condition is needed. The bug is reachable on every gateway node running against a non-empty chain (i.e., the `SyncStateReader` branch, not the genesis branch).

### Recommendation

Implement `get_compiled_class_hash` in `SyncStateReader` by querying the state-sync client for the compiled class hash at the relevant block number, mirroring the pattern used for `get_nonce_at`, `get_storage_at`, and `get_class_hash_at`. Until a proper RPC exists in the state-sync client, a safe interim implementation can return `CompiledClassHash::default()` (matching the `GenesisStateReader` behaviour) and log a warning, which is semantically equivalent to "class not yet declared" and avoids the panic.

### Proof of Concept

1. Start a gateway node backed by `SyncStateReaderFactory` (any non-genesis state).
2. Submit a declare transaction that passes stateless validation (valid Sierra class, correct `compiled_class_hash`, resource bounds above minimum).
3. The transaction converter calls `class_manager_client.add_class(...)` successfully and returns an `InternalRpcTransaction`.
4. `extract_state_nonce_and_run_validations` is called; it reaches `run_validate_entry_point`.
5. Inside `spawn_blocking`, `StatefulValidator::perform_validations` matches `ApiTransaction::Declare(_)` and calls `self.execute(tx)`.
6. The blockifier's declare execution calls `CachedState::get_compiled_class_hash(class_hash)` → cache miss → `SyncOrGenesisStateReader::get_compiled_class_hash` → `SyncStateReader::get_compiled_class_hash` → **`todo!()` panic**.
7. The `spawn_blocking` task returns a `JoinError`; the gateway maps it to `StarknetErrorCode::InternalError` and the worker thread is terminated.

    pub fn perform_validations(&mut self, tx: AccountTransaction) -> StatefulValidatorResult<()> {
        // Deploy account transaction should be fully executed, since the constructor must run
        // before `__validate_deploy__`. The execution already includes all necessary validations,
        // so they are skipped here.
        // Declare transaction should also be fully executed - otherwise, if we only go through
        // the validate phase, we would miss the check that the class was not declared before.
        match tx.tx {
            ApiTransaction::DeployAccount(_) | ApiTransaction::Declare(_) => self.execute(tx),
            ApiTransaction::Invoke(_) => {
                let tx_context = Arc::new(self.tx_executor.block_context.to_tx_context(&tx));
                tx.perform_pre_validation_stage(self.state(), &tx_context)?;
                if !tx.execution_flags.validate {
                    return Ok(());
                }

                // `__validate__` call.
                let (_optional_call_info, actual_cost) = self.validate(&tx, tx_context.clone())?;

                // Post validations.
                PostValidationReport::verify(
                    &tx_context,
                    &actual_cost,
                    tx.execution_flags.charge_fee,
                )?;

                Ok(())
            }
        }
    }
