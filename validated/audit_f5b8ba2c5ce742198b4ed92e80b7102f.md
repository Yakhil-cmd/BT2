### Title
Unpermissioned `update_rewards` with `disable_rewards: true` Allows Any Caller to Permanently Freeze All Stakers' Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
The public `update_rewards` function in the `IStakingRewardsManager` interface accepts a caller-controlled `disable_rewards: bool` parameter and has no access control. Because the global `last_reward_block` is written unconditionally before the `disable_rewards` guard, any unprivileged caller can invoke `update_rewards(any_staker, disable_rewards: true)` once per block to consume the block's reward slot without distributing any rewards, permanently freezing consensus-phase yield for all stakers.

### Finding Description
`update_rewards` is a fully public function with no role check beyond `general_prerequisites` (which only asserts the contract is unpaused and the caller is non-zero). [1](#0-0) 

The function first asserts the current block is newer than `last_reward_block`, then **unconditionally writes** the current block number to `last_reward_block` before checking `disable_rewards`: [2](#0-1) 

When `disable_rewards` is `true`, the function returns immediately after updating `last_reward_block`, distributing nothing. Because `last_reward_block` is a **single global storage slot** (not per-staker), this one call blocks every other staker from calling `update_rewards` in the same block: [3](#0-2) 

An attacker who calls `update_rewards(any_staker, disable_rewards: true)` once per block permanently prevents all stakers from ever accumulating consensus rewards.

### Impact Explanation
This is a **permanent freezing of unclaimed yield** (High). Consensus rewards are distributed exactly once per block per staker via `update_rewards`. If the global `last_reward_block` is consumed with `disable_rewards: true` every block, no staker can ever receive consensus-phase STRK rewards. The `unclaimed_rewards_own` field in every staker's `InternalStakerInfoLatest` will never increase, and pool rewards forwarded via `update_pool_rewards` will never be sent. [4](#0-3) 

### Likelihood Explanation
The attack requires only a single cheap transaction per block on Starknet (low gas cost). No privileged access, leaked key, or external dependency is needed. Any address can execute it. The attacker has no profit motive but causes total loss of yield for all stakers and delegators, making this a realistic griefing scenario.

### Recommendation
1. **Restrict `disable_rewards: true` to a trusted caller** (e.g., the attestation contract or a governance-controlled role). Add an access-control assertion before the `last_reward_block` write when `disable_rewards` is `true`.
2. Alternatively, **only write `last_reward_block` when rewards are actually distributed** (i.e., move the write inside the `!disable_rewards && !is_pre_consensus()` branch), so a `disable_rewards: true` call does not consume the block slot.
3. Consider making `disable_rewards` an internal-only parameter or removing it from the public interface entirely if it is only needed during migration.

### Proof of Concept

1. Consensus rewards are activated (`consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. Attacker (any EOA) submits one transaction per block:
   ```
   staking_contract.update_rewards(
       staker_address: any_valid_staker,
       disable_rewards: true
   )
   ```
3. Each call passes the `current_block_number > last_reward_block` check (line 1454–1458), writes `last_reward_block = current_block_number` (line 1485), then returns at line 1487–1489 without distributing rewards.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block: no staker ever accumulates `unclaimed_rewards_own`; no pool ever receives rewards via `update_pool_rewards`. All consensus-phase yield is permanently frozen. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
