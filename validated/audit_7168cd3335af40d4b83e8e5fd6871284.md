### Title
Permissionless `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards for All Stakers - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `StakingRewardsManagerImpl` is documented as "Only starkware sequencer" in the protocol spec, but the implementation contains **no caller check**. Any unprivileged address can call it with `disable_rewards: true` on every block, consuming the global `last_reward_block` slot and permanently preventing the legitimate sequencer from distributing consensus rewards to all stakers and delegators.

---

### Finding Description

The spec at `docs/spec.md:1644-1645` explicitly states:

> **access control:** Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo:1449-1507` only calls `general_prerequisites()`, which checks two things: the contract is not paused, and the caller is not the zero address.

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks: unpaused + caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);  // global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;   // exits without distributing any rewards
    }
    ...
}
```

`general_prerequisites` at `src/staking/staking.cairo:1794-1797` confirms there is no sequencer check:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

`last_reward_block` is a **single global storage variable** (not per-staker). Once it is written to the current block number, the `REWARDS_ALREADY_UPDATED` assertion prevents any further call to `update_rewards` in the same block — including the legitimate sequencer call.

**Attack path:**

1. Attacker monitors the mempool / block production.
2. On every new block, attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)`.
3. The function passes all checks (unpaused, non-zero caller, staker active, block is new), writes `last_reward_block = current_block`, then returns early with no rewards distributed.
4. The sequencer's subsequent call for the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block → **zero consensus rewards ever distributed** to any staker or delegator.

---

### Impact Explanation

This is a **permanent freezing of unclaimed yield** for all stakers and delegators in the protocol. Consensus rewards (`update_rewards` path, V3) are the primary reward mechanism after `consensus_rewards_first_epoch`. An attacker who front-runs every block's `update_rewards` call with `disable_rewards: true` causes:

- All stakers' `unclaimed_rewards_own` to never increase.
- All delegation pool balances to never receive reward transfers.
- The `RewardSupplier`'s `unclaimed_rewards` to never be incremented, so no L1 mint requests are triggered.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is publicly callable by any non-zero address.
- The attacker needs only a valid active staker address (publicly readable from `get_stakers()`).
- The cost is one transaction per block. On Starknet, transaction fees are low.
- No special knowledge, keys, or privileges are required.
- The attack is fully automatable with a simple bot.

Likelihood: **High**.

---

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, consistent with the spec. The check should assert that `get_caller_address() == get_sequencer_address()` (or use an equivalent role-based guard already present in the contract's roles system).

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == starknet::get_execution_info().unbox().block_info.unbox().sequencer_address,
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

### Proof of Concept

```
Block N arrives.

Attacker tx (submitted first):
  staking.update_rewards(
      staker_address = <any valid active staker>,
      disable_rewards = true
  )
  → general_prerequisites() passes (unpaused, caller != 0)
  → block_number (N) > last_reward_block (N-1) → passes
  → last_reward_block := N
  → disable_rewards == true → return (no rewards distributed)

Sequencer tx (submitted after):
  staking.update_rewards(staker_address = ..., disable_rewards = false)
  → block_number (N) > last_reward_block (N) → FALSE
  → PANIC: REWARDS_ALREADY_UPDATED

Result: No rewards distributed for block N.
Repeat every block → all stakers and delegators receive zero consensus rewards indefinitely.
```
