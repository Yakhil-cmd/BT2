### Title
Missing Inverse Validation in `assert_reshare_keys_invariants` Allows New Joiner with Non-None `old_signing_key` to Bypass Pre-Validation, Causing Reshare DoS - (File: `src/dkg.rs`)

---

### Summary

`assert_reshare_keys_invariants` contains a comment describing a check that is never actually implemented. The code checks only one direction of the participant-membership/key-presence invariant, leaving the inverse case unguarded. A new joiner (not in `old_participants`) who supplies a non-`None` `old_signing_key` passes pre-validation, receives `Ok(protocol)` from `reshare()`, but then causes the async protocol future to abort before sending any messages — leaving all honest participants blocked indefinitely in the first broadcast round.

---

### Finding Description

`assert_reshare_keys_invariants` in `src/dkg.rs` is the sole pre-flight guard called by the public `reshare()` entry point before the protocol future is spawned. [1](#0-0) 

The comment at line 662 reads:

```
// if me is not in the old participant set then ensure that old_signing_key is None
```

But the code that follows implements the **opposite** direction only:

```rust
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(...));
}
```

This catches: *old member supplies no key* → error.  
It does **not** catch: *new joiner supplies a key* → silently passes.

The missing guard is:

```rust
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(...));
}
```

Because this check is absent, `assert_reshare_keys_invariants` returns `Ok(...)` for a new joiner who passes a `Some(old_signing_key)`. [2](#0-1) 

`reshare()` then calls `do_reshare` with that `old_signing_key`: [3](#0-2) 

Inside `do_reshare`, `intersection = old_participants ∩ participants`. Since `me` is a new joiner, `me ∉ old_participants`, so `me ∉ intersection`. However, `ParticipantList::lagrange` does **not** require the queried participant to be a member of the list — it computes the Lagrange coefficient formula for any scalar: [4](#0-3) 

So `intersection.lagrange::<C>(me)` succeeds and returns a non-zero scalar. The product `lambda * x_i.to_scalar()` yields a non-zero `secret`. Control then reaches `do_keyshare`: [5](#0-4) 

`assert_keyshare_inputs` is the first thing called inside `do_keyshare`, **before any network communication**: [6](#0-5) 

Because `secret ≠ 0` and `me ∉ old_participants`, this inner guard fires and returns `Err(ProtocolError::AssertionFailed(...))`. The protocol future terminates immediately — `me` never reaches `do_broadcast` and never sends its Round-1 session ID to the other participants. [7](#0-6) 

All honest participants are now blocked waiting for `me`'s broadcast message with no timeout mechanism in the protocol loop.

---

### Impact Explanation

**High — Permanent denial of reshare for honest parties.**

Every honest participant in the reshare session blocks indefinitely in `do_broadcast` waiting for the malicious new joiner's Round-1 message that never arrives. There is no timeout in the broadcast receive loop. The reshare session cannot complete, and the key set cannot be updated. This maps directly to the allowed High impact: *"Permanent denial of reshare for honest parties under valid protocol inputs and documented trust assumptions."*

---

### Likelihood Explanation

Any participant invited as a new joiner into a reshare session can trigger this. The attacker only needs to call `reshare()` with their own `old_signing_key` set to `Some(...)` instead of `None`. The pre-validation function explicitly passes this input, so no special knowledge or privilege beyond being a listed new participant is required. The attack is trivially reproducible.

---

### Recommendation

Add the missing inverse check inside `assert_reshare_keys_invariants`, immediately after the existing check:

```rust
// if me is not in the old participant set then ensure that old_signing_key is None
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not in the old participant list but provided a share"
    )));
}
```

This mirrors the existing guard and ensures both directions of the invariant are enforced at initialization time, before the protocol future is created.

---

### Proof of Concept

```
Participants: [A, B, C] (old), [A, B, C, D] (new), threshold = 2
Attacker: D (new joiner)

D calls reshare() with old_signing_key = Some(<any SigningShare>)

1. assert_reshare_keys_invariants:
   - old_participants.contains(D) == false  → existing check skipped
   - Missing check: !old_participants.contains(D) && old_signing_key.is_some() → NOT checked
   - Returns Ok(...)

2. reshare() returns Ok(protocol) — no error surfaced to caller

3. Protocol future runs do_reshare:
   - intersection = {A,B,C} ∩ {A,B,C,D} = {A,B,C}
   - intersection.lagrange::<C>(D) → succeeds (D not in list, but formula computes)
   - secret = lambda_D * old_signing_key.to_scalar() ≠ 0

4. do_keyshare called with secret ≠ 0, old_participants = {A,B,C}:
   - assert_keyshare_inputs: secret ≠ 0 AND D ∉ old_participants → Err(AssertionFailed)
   - Returns before do_broadcast — D sends nothing

5. A, B, C block forever in do_broadcast waiting for D's Round-1 session ID
   → Reshare permanently stalled
```
