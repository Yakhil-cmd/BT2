### Title
Wrong Royalty Amount Displayed in WalletConnect Offer Confirmation When Multiple NFTs Are Involved - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
`formatAmountWithRoyalties` in `parseCommandDisplay.ts` incorrectly divides the fungible amount by the number of NFTs before applying each royalty percentage. When a WalletConnect dApp sends a `take_offer` command involving multiple NFTs with royalties, the confirmation dialog shows the user a significantly underestimated total spend amount, causing them to approve a transaction under false pretenses.

### Finding Description

In `parseCommandDisplay.ts`, the function `formatAmountWithRoyalties` computes the total amount a user will spend (including NFT creator royalties) for display in the WalletConnect confirmation dialog: [1](#0-0) 

The critical error is on line 363:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

Each royalty percentage is applied to `splitAmount` (the total divided by the number of NFTs), not to the full `amount`. The correct behavior is to apply each NFT's royalty percentage to the full fungible amount, since each NFT independently demands its royalty from the full purchase price.

**Concrete example** — offer: 0.0001 XCH for two NFTs with royalties 500 (5%) and 10 (0.1%):

| | Displayed (buggy) | Correct |
|---|---|---|
| splitAmount | 50,000,000 mojos | N/A |
| Royalty NFT1 (5%) | 2,500,000 | 5,000,000 |
| Royalty NFT2 (0.1%) | 50,000 | 100,000 |
| **Total shown** | **0.00010255 XCH** | **0.0001051 XCH** |

The existing test suite confirms the buggy value is the expected output: [2](#0-1) 

The `amountWithRoyalties` field produced here is surfaced directly in the WalletConnect confirmation dialog (`Confirm.tsx`) under "Total Amount with Royalties": [3](#0-2) 

This value flows through `walletDeltaToDisplay` → `parseCommandDisplay` → the Electron confirmation dialog shown to the user before they approve the WalletConnect request: [4](#0-3) 

### Impact Explanation

A user approves a WalletConnect `take_offer` request believing they are spending the displayed total (which includes underestimated royalties). The actual on-chain spend is larger — potentially far larger with many high-royalty NFTs. The user's informed consent is obtained under materially false information about the amount being authorized. This matches the High impact criterion: *WalletConnect state that causes a user to approve the wrong amount*.

The discrepancy scales with both the number of NFTs and their royalty percentages. With 10 NFTs each carrying a 10% royalty, the displayed royalty burden is 10× lower than the actual burden.

### Likelihood Explanation

Any dApp connected via WalletConnect can send a `take_offer` command. An attacker who has minted NFTs with royalties (permissionless on Chia) can craft an offer involving multiple such NFTs and present it to a victim through a WalletConnect-connected dApp. No special privileges are required. The victim sees a plausible but incorrect total and approves.

### Recommendation

Replace `splitAmount` with the full `amount` in the royalty accumulation:

```typescript
// Before (incorrect):
const splitAmount = amount / BigInt(royaltyPercentages.length);
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);

// After (correct):
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

The corresponding test in `parseCommandDisplay.test.ts` must also be updated to reflect the correct expected value (0.0001051 XCH, not 0.00010255 XCH).

### Proof of Concept

1. Attacker mints two NFTs: NFT-A (royalty 500 = 5%) and NFT-B (royalty 1000 = 10%).
2. Attacker creates an offer: sell both NFTs for 1 XCH.
3. Attacker's dApp sends `chia_wallet.take_offer` via WalletConnect to the victim's wallet.
4. `parseCommandDisplay` calls `formatAmountWithRoyalties(line, 1_000_000_000_000n, [500, 1000])`.
5. Buggy code: `splitAmount = 500_000_000_000n`; displayed total ≈ **1.075 XCH**.
6. Correct total: `(1T × 500/10000) + (1T × 1000/10000)` = 50B + 100B = 150B royalty → **1.15 XCH**.
7. User sees 1.075 XCH, approves; wallet actually spends 1.15 XCH — a 7% underdisclosure. [5](#0-4)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }
