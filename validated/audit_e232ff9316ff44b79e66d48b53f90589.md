### Title
Anyone Can Call `update_rewards` with `disable_rewards: true` to Permanently Freeze All Staker Consensus Rewards — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and exposes a public `disable_rewards` boolean parameter. Because the function unconditionally writes to the global `last_reward_block` storage slot before checking `disable_rewards`, any unprivileged caller can invoke `update_rewards(any_valid_staker, true)` once per block to consume the block's reward slot without distributing any rewards. Sustained across every block, this permanently freezes unclaimed yield for all stakers in the protocol.

---

### Finding Description

`update_rewards` is declared in `IStakingRewardsManager` and implemented at `src/staking/staking.cairo` lines 1449–1507. Its only gate is `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no role check, no staker-ownership check.

The function accepts an arbitrary `staker_address` (not `get_caller_address()`) and a caller-controlled `disable_rewards: bool`.

The critical ordering is:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← global slot consumed HERE

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← rewards skipped, slot already taken
}
``` [1](#0-0) 

`last_reward_block` is a single global storage variable shared across all stakers:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The per-block exclusivity guard is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

Because the write to `last_reward_block` happens **before** the `disable_rewards` branch, a caller who passes `disable_rewards: true` burns the block's reward slot for the entire protocol while distributing nothing.

The only prerequisite the attacker must satisfy is supplying a `staker_address` that is active and has non-zero STRK balance at the current epoch — both conditions are trivially met by reading any entry from the public `stakers` vector:

```cairo
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [4](#0-3) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

In consensus-rewards mode, every block that passes without a successful `update_rewards` call means that block's STRK and BTC rewards are never credited to any staker or pool. Because `last_reward_block` is already advanced, no legitimate staker can retroactively claim those rewards. An attacker who calls `update_rewards(valid_staker, true)` in every block denies the entire staker set their yield indefinitely. The only protocol-level escape is a privileged `pause()` call, which itself halts all staking operations.

---

### Likelihood Explanation

**High.** The attack requires no capital, no special role, and no coordination. The attacker pays only Starknet L2 gas per block. Valid staker addresses are publicly enumerable from the `stakers` storage vector. The attack is fully automatable with a simple bot.

---

### Recommendation

1. **Restrict the caller**: Add a check that `get_caller_address() == staker_address || get_caller_address() == staker_info.operational_address` before writing to `last_reward_block`.
2. **Remove the public `disable_rewards` parameter**: If the pre-consensus short-circuit is needed, handle it internally via `self.is_pre_consensus()` only, without exposing a caller-controlled bypass.
3. **Write `last_reward_block` only after rewards are confirmed to be distributed**: Move the `self.last_reward_block.write(current_block_number)` call to after the `disable_rewards` / `is_pre_consensus` guard so that an early return does not consume the slot.

---

### Proof of Concept

```
// Attacker bot, runs once per block:
// 1. Read any active staker address from staking_contract.stakers[0].
// 2. Call: staking_contract.update_rewards(staker_address, disable_rewards=true)
//    - general_prerequisites() passes (contract unpaused, caller non-zero).
//    - current_block > last_reward_block passes (first call this block).
//    - staker_info lookup passes (valid staker).
//    - staker_total_strk_balance > 0 passes (staker has stake).
//    - last_reward_block is written to current_block.
//    - Function returns early; zero rewards distributed.
// 3. Any legitimate staker who now calls update_rewards in the same block
//    hits REWARDS_ALREADY_UPDATED and reverts.
// 4. Repeat every block → all consensus rewards permanently frozen.
``` [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
