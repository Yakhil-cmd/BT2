### Title
Commission Increased Mid-Epoch Without Settling Pending Rewards — (`src/staking/staking.cairo`)

### Summary

A staker holding an active `commission_commitment` can call `set_commission` to raise their commission mid-epoch and then immediately attest. Because reward settlement only occurs at attestation time, the entire epoch's rewards are calculated at the new, higher commission rate — retroactively reducing delegator yield for work already done at the lower rate.

### Finding Description

The `set_commission` function in `staking.cairo` updates the commission value in storage immediately with no epoch-delay and no prior reward settlement: [1](#0-0) 

The internal `update_commission` helper enforces that without a commitment, commission can only decrease. However, when a `commission_commitment` is active, the staker may set commission to **any value ≤ `max_commission`**, including values higher than the current commission: [2](#0-1) 

The commission is not applied at the time it is set. It is read and applied only when `_update_rewards` is triggered by attestation: [3](#0-2) 

Because `calculate_staker_pools_rewards` reads `staker_pool_info.commission()` at attestation time and applies it to the **entire epoch's** `pool_rewards_including_commission`, any commission increase that occurs between epoch start and attestation is retroactively applied to rewards that delegators earned under the old, lower commission.

The developers themselves acknowledge this gap in a code comment: [4](#0-3) 

### Impact Explanation

Delegators who delegated for an entire epoch at commission rate `C_old` receive rewards calculated at `C_new > C_old` if the staker raises commission before attesting. The difference `(C_new − C_old) × pool_rewards_including_commission` is silently redirected to the staker as extra commission. This constitutes **theft of unclaimed yield** from delegators.

### Likelihood Explanation

The attack requires:
1. The staker sets a `commission_commitment` with `max_commission` above the current commission — a normal, publicly documented protocol feature.
2. Near the end of an epoch (maximising the retroactive effect), the staker calls `set_commission` to raise commission up to `max_commission`.
3. The staker attests in the same or next block.

All three steps are callable by the staker address alone, with no privileged role or external dependency. The `commission_commitment` mechanism is explicitly designed to allow commission increases, so the precondition is reachable in normal protocol operation.

### Recommendation

Apply commission changes with an epoch delay (analogous to how balance changes use `epoch + K` checkpoints), so that any commission increase only affects rewards accrued **after** the epoch in which the change was made. Alternatively, record the commission value at the start of each epoch and use that snapshot — rather than the live storage value — when computing rewards at attestation time.

### Proof of Concept

1. Staker stakes with commission = 10% and opens a pool.
2. Delegators join the pool.
3. Staker calls `set_commission_commitment(max_commission: 9000, expiration_epoch: current + 5)`.
4. At the end of epoch N (after most of the epoch has elapsed), staker calls `set_commission(9000)`. Commission is immediately written to storage.
5. Staker attests for epoch N. `calculate_staker_pools_rewards` reads `commission = 9000` and splits the entire epoch's pool rewards at 90% commission.
6. Delegators receive only 10% of the pool rewards for epoch N, despite having been in the pool the entire epoch at 10% commission.
7. The staker captures the extra `(9000 − 1000) / 10000 × pool_rewards_including_commission` as commission — yield that belonged to delegators. [5](#0-4)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );
                total_commission_rewards += commission_rewards;
                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)
