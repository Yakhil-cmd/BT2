### Title
Staker Can Instantly Raise Commission to 100% in a Single Transaction via `set_commission_commitment` + `set_commission` — (File: `src/staking/staking.cairo`)

---

### Summary

A staker can atomically call `set_commission_commitment` with `expiration_epoch = current_epoch + 1` and then immediately call `set_commission(10000)` in the same transaction. Because the commitment is active the moment it is written (the active check is `current_epoch < expiration_epoch`, which is satisfied immediately), there is no delay or notice period before the staker can raise commission to 100%. Delegators have no opportunity to exit before their rewards are fully captured by the staker.

---

### Finding Description

Without a commitment, `set_commission` enforces a strict decrease: `commission < old_commission`. [1](#0-0) 

The `set_commission_commitment` mechanism was introduced to allow controlled commission increases. It requires only that `expiration_epoch > current_epoch`: [2](#0-1) 

The active-check helper is: [3](#0-2) 

Because `current_epoch < current_epoch + 1` is always true, a commitment set with `expiration_epoch = current_epoch + 1` is **immediately active**. There is no cooldown, no waiting period, and no block-gap enforced between writing the commitment and consuming it. The code itself acknowledges this gap: [4](#0-3) 

Once the commitment is active, `update_commission` permits any value up to `max_commission` (which can be `COMMISSION_DENOMINATOR` = 10 000, i.e. 100%): [5](#0-4) 

A staker therefore executes two calls in one transaction:
1. `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`
2. `set_commission(10000)`

Commission is read live from the staking contract at reward-distribution time: [6](#0-5) 

So the 100% commission takes effect for the very next reward epoch, and delegators receive zero pool rewards.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

With commission at 100%, the pool forwards zero rewards to delegators. The flow test at lines 895–904 of `src/flow_test/flows.cairo` explicitly demonstrates that a 100% commission results in a zero pool balance (all rewards go to the staker): [7](#0-6) 

Delegators who trusted the staker and have accruing unclaimed rewards lose all future yield for the epoch(s) during which the inflated commission is active. The staker captures those rewards as commission.

---

### Likelihood Explanation

**Medium-High.** Any registered staker who has opened a delegation pool can execute this in a single transaction with no preconditions beyond having a pool. The only cost is the gas for two contract calls. A rational attacker would time the attack just before a large attestation reward is distributed, maximising the captured yield. Delegators have no on-chain mechanism to detect and exit before the same block settles.

---

### Recommendation

Enforce a minimum delay between the epoch in which a commitment is set and the epoch in which it may first be used to increase commission. For example, require that `set_commission` with an upward change can only be called when `current_epoch >= commitment_set_epoch + MIN_DELAY_EPOCHS` (where `MIN_DELAY_EPOCHS >= 1` gives delegators at least one full epoch to observe the pending increase and exit). Alternatively, record the epoch in which the commitment was written and reject `set_commission` calls that would raise the commission until that epoch has fully elapsed.

---

### Proof of Concept

```
// Staker has current commission = 500 (5%), has a pool with delegators.
// current_epoch = E

// Step 1 — set commitment (immediately active)
staking.set_commission_commitment(
    max_commission: 10000,          // 100%
    expiration_epoch: E + 1,        // satisfies expiration_epoch > current_epoch
);
// is_commission_commitment_active returns: E < E+1 == true  ← active immediately

// Step 2 — raise commission to 100% in the same transaction
staking.set_commission(commission: 10000);
// update_commission: commitment active, 10000 <= 10000, 10000 != 500 → passes

// Result: commission is now 100%.
// Next attestation reward for epoch E is distributed with commission = 10000.
// Pool receives 0 STRK; all pool rewards flow to the staker's reward address.
// Delegators' unclaimed yield for epoch E is stolen.
```
