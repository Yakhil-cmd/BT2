### Title
Unprivileged Caller Can Permanently Consume Block Reward Slot via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract writes `last_reward_block` **before** checking the `disable_rewards` flag. Because the function is public with no access control, any unprivileged caller can invoke it with `disable_rewards: true` to consume the block's single reward slot without distributing any rewards, permanently blocking legitimate reward distribution for that block.

### Finding Description
`update_rewards` enforces a "one call per block" invariant via the global `last_reward_block` storage variable:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After the guard passes, the function unconditionally writes the current block number to `last_reward_block` and **then** checks `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← slot consumed here

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← no rewards distributed
}
``` [1](#0-0) 

Because `IStakingRewardsManager` carries no role restriction and `general_prerequisites()` only checks the pause flag, any address can call `update_rewards(staker_address: <any active staker>, disable_rewards: true)`. [2](#0-1) 

The result: `last_reward_block` is advanced to the current block, the early-return fires, zero rewards are distributed, and every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

This is structurally identical to the reported `conditionallySingleCallPerTransaction()` pattern: a per-slot guard is consumed by a no-op invocation, making the slot unavailable for the legitimate caller.

### Impact Explanation
Consensus-era block rewards are calculated for the specific block in which `update_rewards` is called. There is no accumulation mechanism that retroactively credits skipped blocks to a future call. For every block in which the attacker front-runs with `disable_rewards: true`, the staker's (and delegators') block reward is permanently destroyed — it is never credited to `unclaimed_rewards_own` and never forwarded to the pool's `cumulative_rewards_trace`. [4](#0-3) 

Sustained execution (one cheap Starknet transaction per block) permanently freezes all consensus-era unclaimed yield for all stakers, matching the **High** impact tier: *Permanent freezing of unclaimed yield or unclaimed royalties*.

### Likelihood Explanation
- `update_rewards` is fully public; no role, no signature, no fee is required beyond gas.
- The attacker only needs to supply any currently-active staker address (readable from on-chain events or `get_stakers()`).
- Starknet transaction fees are low, making per-block griefing economically feasible for a motivated adversary.
- The attack requires no privileged access, no bridge compromise, and no external dependency.

### Recommendation
Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so that a no-op call does not consume the block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only advance the slot when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... reward calculation continues
```

Alternatively, restrict `disable_rewards: true` to a privileged role (e.g., `only_app_governor`) so unprivileged callers cannot invoke the no-op path.

### Proof of Concept
1. Consensus rewards are active (`current_epoch >= consensus_rewards_first_epoch`).
2. Attacker observes block `N` is produced.
3. Attacker sends `update_rewards(staker_address: <any active staker>, disable_rewards: true)` in block `N`.
4. `last_reward_block` is written to `N`; function returns with zero rewards distributed.
5. The legitimate sequencer/staker operator sends `update_rewards(staker_address: <staker>, disable_rewards: false)` in block `N` — it reverts with `REWARDS_ALREADY_UPDATED`.
6. Block `N`'s reward is permanently lost for all stakers and their delegators.
7. Repeating steps 2–6 every block eliminates all consensus-era yield indefinitely. [5](#0-4)
