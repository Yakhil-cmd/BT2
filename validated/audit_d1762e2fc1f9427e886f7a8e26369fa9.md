### Title
Single Malicious Participant Can Permanently Abort DKG, Reshare, and Refresh via `broadcast_success` Veto - (File: src/dkg.rs)

### Summary
The `broadcast_success` function in `src/dkg.rs` requires **unanimous** agreement from all participants at the final round of DKG, Reshare, and Refresh. Because any single participant can broadcast `false` instead of `true`, a single malicious participant can abort the protocol for all honest parties at the last step — after all key shares have been computed but before they are returned. This is a permanent denial-of-service against key generation, resharing, and refresh under valid protocol inputs.

### Finding Description

`broadcast_success` is called at the very end of `do_keyshare` (line 531), which is the shared implementation for DKG, Reshare, and Refresh: [1](#0-0) 

The function always broadcasts `(true, session_id)` on behalf of the calling participant, then collects votes from all other participants via reliable broadcast, and then enforces that **every single vote is `true`**: [2](#0-1) 

The check at line 329 is:

```rust
if !vote_list.iter().all(|&(boolean, _)| boolean) {
    return Err(ProtocolError::AssertionFailed(
        "A participant seems to have failed its checks. Aborting Protocol!"
    ));
}
```

Because `broadcast_success` itself always sends `true`, the only way this check can fail is if a **malicious participant deviates from the protocol and sends `false`**. The underlying `do_broadcast` uses the echo-broadcast (reliable broadcast) primitive, which guarantees consistency: if a malicious participant broadcasts `false`, all honest parties will consistently receive `false` from that participant and will all abort.

The abort happens **after** all key material has been computed (polynomial evaluations, secret shares, verifying key), but **before** `KeygenOutput` is returned to the caller. Honest parties therefore lose all computed state and cannot proceed.

This affects all three protocol entry points that call `do_keyshare`:
- `do_keygen` → `do_keyshare` → `broadcast_success`
- `do_reshare` → `do_keyshare` → `broadcast_success`
- Refresh (also routes through `do_keyshare`) [3](#0-2) [4](#0-3) 

### Impact Explanation

**High: Permanent denial of key generation, reshare, and refresh for honest parties under valid protocol inputs.**

A single malicious participant (one out of N) can veto every DKG/reshare/refresh attempt indefinitely. Since the malicious participant can repeat this on every retry, honest parties can never complete key generation or resharing as long as the malicious participant is included in the participant set. This maps directly to the allowed impact: *"Permanent denial of signing, key generation, reshare, refresh, or CKD for honest parties under valid protocol inputs and documented trust assumptions."*

### Likelihood Explanation

Any participant in the DKG/reshare/refresh session can trigger this. No special privilege is required beyond being listed in the `participants` array. The attacker only needs to send a single `false` boolean in the final broadcast round — a trivial protocol deviation. The attack is repeatable on every retry attempt.

### Recommendation

Replace the unanimous-agreement check with a threshold-based check. If at least `threshold` participants broadcast success, the protocol should proceed. Participants who broadcast failure should be identified and excluded from future sessions rather than causing a global abort. Alternatively, remove the `broadcast_success` round entirely: by the time `broadcast_success` is reached, each honest party has already independently verified all received shares via `validate_received_share` and all commitments via `verify_proof_of_knowledge` and `verify_commitment_hash`. The final success broadcast adds no cryptographic guarantee and only introduces a veto vector.

### Proof of Concept

1. N honest participants and 1 malicious participant `M` initiate DKG.
2. All participants complete rounds 1–5 of `do_keyshare` successfully: session IDs are exchanged, commitments broadcast, proofs verified, secret shares sent and validated.
3. In the `broadcast_success` round (line 531), all honest participants broadcast `(true, session_id)`. Malicious participant `M` broadcasts `(false, session_id)`.
4. The echo-broadcast protocol delivers `M`'s `false` vote consistently to all honest participants.
5. Each honest participant reaches the check at line 329:
   ```rust
   if !vote_list.iter().all(|&(boolean, _)| boolean) { ... }
   ```
   This evaluates to `true` (not all are `true`), so every honest participant returns `Err(ProtocolError::AssertionFailed(...))`.
6. No honest participant receives a `KeygenOutput`. The DKG is aborted.
7. `M` repeats this on every retry, permanently blocking key generation. [5](#0-4)

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
