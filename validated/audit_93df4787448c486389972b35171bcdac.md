### Title
Missing Commitment Self-Verification in FROST EdDSA Signing Participant Enables Malicious Coordinator to Corrupt Signing Transcripts - (File: `src/frost/eddsa/sign.rs`)

---

### Summary

In `do_sign_participant_v1`, a participant sends its nonce commitment to the coordinator in Round 1, then receives a `SigningPackage` back in Round 2. The participant validates only that the message in the signing package matches the expected message, but **never verifies that its own commitment inside the signing package matches what it sent**. A malicious coordinator can substitute any participant's commitment in the signing package, causing that participant to compute a signature share under an inconsistent binding factor, corrupting the signing transcript and causing the aggregate to fail for all honest parties.

---

### Finding Description

In `do_sign_participant_v1` the protocol proceeds as follows:

**Round 1 — participant sends its commitment to the coordinator:** [1](#0-0) 

```rust
let (nonces, commitments) = round1::commit(&signing_share, rng);
let nonces = Zeroizing::new(nonces);
// ...
chan.send_private(commit_waitpoint, coordinator, &commitments)?;
```

**Round 2 — participant receives the signing package and validates only the message:** [2](#0-1) 

```rust
let signing_package = loop {
    let (from, signing_package): (_, frost_ed25519::SigningPackage) =
        chan.recv(r2_wait_point).await?;
    if from != coordinator { continue; }
    break signing_package;
};

if signing_package.message() != message.as_slice() {
    return Err(ProtocolError::AssertionFailed(...));
}
```

There is **no check** that `signing_package.signing_commitments().get(&me.to_identifier()?)` equals the `commitments` value sent in Round 1. The participant then immediately proceeds to compute its signature share using its actual nonces against the (potentially tampered) signing package: [3](#0-2) 

RFC 9591 (FROST) §5.2 explicitly mandates: *"Each signer MUST verify their commitment is in the signing package."* This check is absent.

The coordinator collects commitments from all participants via `recv_from_others` and inserts them into the signing package keyed by participant identifier: [4](#0-3) 

Nothing prevents the coordinator from substituting a different commitment for any participant before broadcasting the signing package at line 137: [5](#0-4) 

Note that `sign_v2` / `do_sign_participant_v2` is **not** affected because it constructs the signing package locally from a pre-computed `presignature.commitments_map` rather than receiving it from the coordinator: [6](#0-5) 

---

### Impact Explanation

In FROST, the binding factor for participant *i* is:

> ρ_i = H(i, M, B)  where B = list of all (identifier, commitment) pairs

If the coordinator replaces participant P's commitment `(D_P, E_P)` with `(D_P′, E_P′)` in the signing package:

- P computes `ρ_P′ = H(P, M, B′)` using the substituted commitment list.
- P computes its signature share `s_P = d_P + e_P·ρ_P′ + λ_P·x_P·c′` using its **actual** nonces `(d_P, e_P)` but the **wrong** binding factor.
- The verifier checks `g^{s_P} == D_P′ + E_P′·ρ_P′ + Y_P^{λ_P·c′}` (using the substituted commitment), which fails because P's actual nonces correspond to `(D_P, E_P)`, not `(D_P′, E_P′)`.
- The `aggregate` call at the coordinator fails, and no valid signature is produced.

Furthermore, the coordinator passes an **empty** `verifying_shares` map to `aggregate`, disabling per-share cheater detection: [7](#0-6) 

This means honest participants cannot identify which share is invalid, and the protocol silently aborts with no actionable attribution. A malicious coordinator can repeatably trigger this to permanently deny signing for any honest participant set.

**Impact: High** — Corruption of signing outputs so honest parties accept inconsistent transcripts / permanent denial of signing for honest parties under valid protocol inputs.

---

### Likelihood Explanation

The coordinator is a full participant in the protocol and is explicitly within the threat model (the code already guards against non-coordinator messages at line 276). Substituting a commitment in the signing package requires only that the coordinator modify the `BTreeMap` it constructs before calling `chan.send_many`. No cryptographic capability is required. The attack is trivially repeatable across every signing session.

**Likelihood: High**

---

### Recommendation

In `do_sign_participant_v1`, immediately after receiving the signing package and verifying the message, add a self-commitment check:

```rust
// Verify that our own commitment in the signing package matches what we sent
let my_id = me.to_identifier()?;
if signing_package.signing_commitments().get(&my_id) != Some(&commitments) {
    return Err(ProtocolError::AssertionFailed(
        "Coordinator substituted my commitment in the signing package".to_string(),
    ));
}
```

This check is mandated by RFC 9591 §5.2 and closes the coordinator-impersonation vector for participant commitments.

---

### Proof of Concept

1. All participants call `sign_v1`; participant P sends commitment `(D_P, E_P)` privately to the coordinator at `commit_waitpoint`.
2. The coordinator collects all commitments via `recv_from_others` and inserts them into `commitments_map`.
3. Before calling `chan.send_many(r2_wait_point, &signing_package)`, the coordinator replaces P's entry in `commitments_map` with an arbitrary `(D_P′, E_P′)`.
4. All participants receive the modified signing package. P checks only `signing_package.message() == message` (passes) and proceeds.
5. P computes `round2::sign(&signing_package, &nonces, &key_package)` using its actual nonces `(d_P, e_P)` but the modified binding factor derived from `(D_P′, E_P′)`.
6. The coordinator calls `aggregate(&signing_package, &signature_shares, &public_key_package)` — the aggregate signature is invalid because P's share is inconsistent with the signing package, and `aggregate` returns an error.
7. No valid signature is produced. The coordinator can repeat this for every signing session, permanently denying signing to the honest participant set. [8](#0-7) [9](#0-8)

async fn do_sign_coordinator_v1(
    mut chan: SharedChannel,
    participants: ParticipantList,
    threshold: ReconstructionLowerBound,
    me: Participant,
    keygen_output: KeygenOutput,
    message: Vec<u8>,
    rng: &mut impl CryptoRngCore,
) -> Result<SignatureOption, ProtocolError> {
    // --- Round 1.
    // * Wait for other parties' commitments.

    let mut commitments_map: BTreeMap<frost_ed25519::Identifier, round1::SigningCommitments> =
        BTreeMap::new();

    // signing share is the private_share
    let signing_share = keygen_output.private_share;

    // Step 1.1 (and implicitely 1.2)
    let (nonces, commitments) = round1::commit(&signing_share, rng);
    let nonces = Zeroizing::new(nonces);
    commitments_map.insert(me.to_identifier()?, commitments);

    // Step 1.3
    let commit_waitpoint = chan.next_waitpoint();

    // Step 1.4
    for (from, commitment) in recv_from_others(&chan, commit_waitpoint, &participants, me).await? {
        commitments_map.insert(from.to_identifier()?, commitment);
    }

    let signing_package = frost_ed25519::SigningPackage::new(commitments_map, message.as_slice());

    let mut signature_shares: BTreeMap<frost_ed25519::Identifier, round2::SignatureShare> =
        BTreeMap::new();

    // Step 1.5
    let r2_wait_point = chan.next_waitpoint();
    chan.send_many(r2_wait_point, &signing_package)?;

    // --- Round 2
    // * Wait for each other's signature share
    // Step 2.3 (2.1 and 2.2 are implicit)
    let vk_package = keygen_output.public_key;
    let key_package = construct_key_package(threshold, me, signing_share, &vk_package)?;
    let key_package = Zeroizing::new(key_package);
    let signature_share = round2::sign(&signing_package, &nonces, &key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;

    // Step 2.5 (2.4 is implicit)
    signature_shares.insert(me.to_identifier()?, signature_share);
    for (from, signature_share) in recv_from_others(&chan, r2_wait_point, &participants, me).await?
    {
        signature_shares.insert(from.to_identifier()?, signature_share);
    }

    // --- Signature aggregation.
    // * Converted collected signature shares into the signature.
    // * Signature is verified internally during `aggregate()` call.

    // Step 2.6 and 2.7
    // We supply empty map as `verifying_shares` because we have disabled "cheater-detection" feature flag.
    // Feature "cheater-detection" only points to a malicious participant, if there's such.
    // It doesn't bring any additional guarantees.
    let public_key_package = PublicKeyPackage::new(BTreeMap::new(), vk_package);
    let signature = aggregate(&signing_package, &signature_shares, &public_key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;

    Ok(Some(signature))

async fn do_sign_participant_v1(
    mut chan: SharedChannel,
    threshold: ReconstructionLowerBound,
    me: Participant,
    coordinator: Participant,
    keygen_output: KeygenOutput,
    message: Vec<u8>,
    rng: &mut impl CryptoRngCore,
) -> Result<SignatureOption, ProtocolError> {
    // --- Round 1.
    if coordinator == me {
        return Err(ProtocolError::AssertionFailed(
            "the do_sign_participant function cannot be called
            for a coordinator"
                .to_string(),
        ));
    }

    // signing share is the private_share
    let signing_share = keygen_output.private_share;

    // Step 1.1
    let (nonces, commitments) = round1::commit(&signing_share, rng);
    // Ensures the values are zeroized on drop
    let nonces = Zeroizing::new(nonces);

    // * Wait for an initial message from a coordinator.
    // * Send coordinator our commitment.

    // Step 1.2
    let commit_waitpoint = chan.next_waitpoint();
    chan.send_private(commit_waitpoint, coordinator, &commitments)?;

    // --- Round 2.
    // * Wait for a signing package.
    // * Send our signature share.

    // Step 2.1
    let r2_wait_point = chan.next_waitpoint();
    let signing_package = loop {
        let (from, signing_package): (_, frost_ed25519::SigningPackage) =
            chan.recv(r2_wait_point).await?;
        if from != coordinator {
            continue;
        }
        break signing_package;
    };

    // Step 2.2
    if signing_package.message() != message.as_slice() {
        return Err(ProtocolError::AssertionFailed(
            "Expected message doesn't match with the actual message received in a signing package"
                .to_string(),
        ));
    }

    // Step 2.3
    let vk_package = keygen_output.public_key;
    let key_package = construct_key_package(threshold, me, signing_share, &vk_package)?;
    // Ensures the values are zeroized on drop
    let key_package = Zeroizing::new(key_package);
    let signature_share = round2::sign(&signing_package, &nonces, &key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;

    // Step 2.4
    chan.send_private(r2_wait_point, coordinator, &signature_share)?;

    Ok(None)
}
