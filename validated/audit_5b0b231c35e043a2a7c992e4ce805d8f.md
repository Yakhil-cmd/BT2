### Title
Unprivileged Caller Can Suppress All Consensus Block Rewards by Calling `update_rewards` with `disable_rewards: true` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is publicly callable and accepts a `disable_rewards: bool` parameter. The global `last_reward_block` storage variable is written unconditionally **before** the `disable_rewards` guard, meaning any unprivileged caller can invoke `update_rewards(any_active_staker, disable_rewards: true)` once per block to consume the per-block "reward slot" without distributing any rewards. Because only one call per block is permitted (enforced by the `last_reward_block` check), repeating this every block permanently suppresses consensus block rewards for all stakers.

---

### Finding Description

`update_rewards` is the V3 consensus-rewards entry point. Its logic is:

```
1. Check contract not paused, caller not zero.
2. Assert current_block_number > last_reward_block  ← one-call-per-block gate
3. Assert staker exists and is active with non-zero balance.
4. last_reward_block.write(current_block_number)    ← slot consumed HERE
5. if disable_rewards || is_pre_consensus() { return; }  ← rewards skipped
6. calculate_block_rewards(...)
7. _update_rewards(...)                              ← actual distribution
```

The critical flaw is that step 4 (consuming the per-block slot) executes unconditionally, while step 5 allows an early return that skips steps 6–7 entirely. There is no access-control check on the `disable_rewards` parameter — `general_prerequisites()` only verifies the contract is unpaused and the caller is non-zero.

`last_reward_block` is a **single global** storage variable, not per-staker:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
```

Once it is written to block N, no further call to `update_rewards` can succeed in block N (the assertion `current_block_number > last_reward_block` fails). Rewards for block N are permanently lost — they cannot be retroactively distributed.

---

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` once per block:

- Consumes the global per-block slot for all stakers.
- Prevents any legitimate call from distributing consensus block rewards.
- Causes permanent, irrecoverable loss of block rewards for every staker and every delegation pool for each affected block.

This matches **High: Theft of unclaimed yield / Permanent freezing of unclaimed yield** — stakers and delegators never receive the STRK block rewards they are entitled to for the suppressed blocks.

---

### Likelihood Explanation

- The function is `#[abi(embed_v0)]` with no role restriction.
- Any address (including a bot) can call it.
- The only cost is Starknet gas per block, which is negligible.
- A single attacker can sustain the attack indefinitely.
- No special knowledge, leaked keys, or privileged access is required.

---

### Recommendation

1. **Remove `disable_rewards` from the public interface entirely.** The pre-consensus early-return is already handled by `is_pre_consensus()` inside the function; the explicit `disable_rewards` flag adds no legitimate public use case.
2. **If `disable_rewards` must remain**, gate it behind a privileged role (e.g., `only_app_governor`) or restrict callers to the attestation contract.
3. **Move `last_reward_block.write` after the `disable_rewards` guard**, so the slot is only consumed when rewards are actually distributed.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` is set and current epoch ≥ it).
2. Attacker (any EOA) calls `update_rewards(valid_staker_address, disable_rewards: true)` at the start of block N.
3. `last_reward_block` is written to N; the function returns early — no rewards distributed.
4. Any legitimate call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`.
5. Attacker repeats in block N+1, N+2, … indefinitely.
6. All stakers and pool members receive zero consensus block rewards for every suppressed block.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
