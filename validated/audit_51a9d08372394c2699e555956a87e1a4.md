### Title
Hardcoded Testing Private Key Used in Production Signature Manager — (`crates/apollo_signature_manager/src/lib.rs`)

### Summary
The production sequencer node unconditionally instantiates `LocalKeyStore::new_for_testing()` with the publicly known private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` for signing precommit votes. Because the key is embedded in public source code and `new_for_testing` is not `#[cfg(test)]`-gated, any party can derive valid ECDSA precommit vote signatures over arbitrary `block_hash` values that will pass `verify_precommit_vote_signature` with the corresponding public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a`.

### Finding Description
`LocalKeyStore::new_for_testing` hardcodes both the private and public key as `const` values. [1](#0-0) 

`create_signature_manager` — the sole production factory — calls `SignatureManager::new()` which calls `LocalKeyStore::new_for_testing()` with no configuration path to supply a real key. [3](#0-2) 

The production node wires this directly: [5](#0-4) 

`sign_precommit_vote` uses the keystore's private key to ECDSA-sign the Blake2s digest of `PRECOMMIT_VOTE || block_hash.to_bytes_be()`. [6](#0-5) 

`verify_precommit_vote_signature` accepts any `(block_hash, signature, public_key)` triple and returns `Ok(true)` if the ECDSA check passes — it has no binding to a trusted key registry. [7](#0-6) 

Because the private key is public knowledge, an attacker can compute `ecdsa_sign(PRIVATE_KEY, blake2s("PRECOMMIT_VOTE" || arbitrary_block_hash))` offline and produce a signature that `verify_precommit_vote_signature` accepts as `Ok(true)`.

### Impact Explanation
Any peer connected to the consensus P2P network can inject forged precommit votes for arbitrary `block_hash` values that are cryptographically indistinguishable from legitimate sequencer votes. If the consensus layer accepts these votes (e.g., in a single-validator testnet or when the attacker controls enough peer slots), the finalized block hash is corrupted, and RPC endpoints serving state, storage, and class hashes will return values anchored to the wrong block.

### Likelihood Explanation
The private key is in public source code. The `new_for_testing` constructor is not `#[cfg(test)]`-gated and is called on every production startup. The TODO comment in `lib.rs` ("understand how key store would look in production") confirms this is a known placeholder that was never replaced. [8](#0-7) 

### Recommendation
1. Gate `LocalKeyStore::new_for_testing` with `#[cfg(any(test, feature = "testing"))]` so it cannot be called from production code.
2. Implement a production `KeyStore` that loads the private key from a secrets manager, HSM, or environment variable at startup, and make `create_signature_manager` require an explicit key source.
3. Add a compile-time or startup assertion that prevents the node from starting if the active signing key matches the hardcoded testing value.

### Proof of Concept
```rust
#[test]
fn attacker_forges_precommit_vote_with_known_key() {
    use apollo_signature_manager::signature_manager::{
        LocalKeyStore, SignatureManager, verify_precommit_vote_signature,
    };
    use starknet_api::block::BlockHash;
    use starknet_api::felt;

    // Private key is public knowledge — taken directly from source.
    let key_store = LocalKeyStore::new_for_testing();
    let sm = SignatureManager::new(key_store);

    // Attacker chooses an arbitrary block hash.
    let fake_block_hash = BlockHash(felt!("0xdeadbeef"));

    let rt = tokio::runtime::Runtime::new().unwrap();
    let forged_sig = rt.block_on(sm.sign_precommit_vote(fake_block_hash)).unwrap();

    // Verification passes with the well-known public key.
    assert_eq!(
        verify_precommit_vote_signature(fake_block_hash, forged_sig, key_store.public_key),
        Ok(true)
    );
}
```
