### Title
Missing Minimum Threshold Enforcement in FROST Presign and Sign Validation Allows Single-Participant Signature — (File: `src/frost/mod.rs`)

### Summary
The `frost::presign` and `assert_sign_inputs` functions in `src/frost/mod.rs` do not enforce the minimum threshold constraint (`threshold >= 2`) that is explicitly enforced at DKG time in `assert_key_invariants` (`src/dkg.rs`). A caller can bypass this constraint by passing `threshold = 1` (or `0`) to presign/sign, allowing a single participant to produce a valid threshold signature without the required quorum — directly analogous to the external report's pattern of a type-specific constraint enforced in one code path but bypassable in another.

---

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces a hard lower bound on the threshold:

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

However, `frost::presign` in `src/frost/mod.rs` only validates the upper bound:

```rust
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [2](#0-1) 

Identically, `assert_sign_inputs` — the public validation gate for EdDSA/RedJubjub signing — only checks the upper bound:

```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [3](#0-2) 

Neither function checks `threshold >= 2`. The `ReconstructionLowerBound` type is a plain `usize` wrapper with no enforced minimum:

```rust
pub struct ReconstructionLowerBound(usize);
``` [4](#0-3) 

This is structurally identical to the external report's pattern: the constraint exists in one code path (DKG) but is absent in the presign/sign code paths, allowing a caller to create a broken protocol state.

Note: the OT-based ECDSA presign is incidentally protected because it cross-checks the presign threshold against the triple thresholds, and `validate_triple_inputs` enforces `threshold >= 2`. [5](#0-4)  The FROST presign has no equivalent cross-check.

---

### Impact Explanation

FROST (used for EdDSA/RedJubjub) is designed to operate at any threshold ≥ 1. If `threshold = 1` is accepted by `frost::presign` and `assert_sign_inputs`, the underlying `frost-core` library will require only **one** participant's signature share to produce a valid, verifiable threshold signature. A single participant acting as coordinator can:

1. Participate in the presign phase (collecting all commitments into `commitments_map`)
2. Call signing with `threshold = 1`, contributing only their own share
3. Aggregate and output a cryptographically valid signature

This constitutes **unauthorized creation of a valid threshold signature** — the full quorum is bypassed. Impact: **Critical**.

---

### Likelihood Explanation

The entry path is direct: any library caller invoking `frost::presign` or `assert_sign_inputs` with a `ReconstructionLowerBound(1)` argument triggers the bypass. No special privilege is required beyond being a participant in the presign round. The `PresignArguments` struct accepts the threshold as a plain caller-supplied field with no type-level minimum. [6](#0-5) 

---

### Recommendation

Add the same lower-bound check present in `assert_key_invariants` to both `frost::presign` and `assert_sign_inputs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This mirrors the existing guard in `src/dkg.rs` and closes the inconsistency. [1](#0-0) 

---

### Proof of Concept

```rust
// DKG runs correctly with threshold=2
let keygen_out = run_keygen(&participants, 2, &mut rng);

// Presign accepts threshold=1 — no error returned
let presign_args = PresignArguments {
    keygen_out: keygen_out[0].1.clone(),
    threshold: ReconstructionLowerBound(1), // bypasses minimum check
};
let presign_protocol = frost::presign(&participants, me, &presign_args, rng).unwrap();
// presign_protocol runs successfully, collecting all commitments

// assert_sign_inputs accepts threshold=1 — no error returned
let signing_participants = assert_sign_inputs(
    &participants,
    ReconstructionLowerBound(1), // bypasses minimum check
    me,
    coordinator,
).unwrap();

// Coordinator produces a valid signature using only their own share
// FROST requires only 1 share when threshold=1
```

The coordinator can now produce a cryptographically valid EdDSA/RedJubjub signature without the participation of the required quorum, violating the threshold security guarantee established at DKG time.
