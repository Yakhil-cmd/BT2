### Title
Arbitrary `participants` in `RerandomizationArguments` Not Validated Against Signing Set, Enabling Malicious-Coordinator Split-View Secret Key Recovery - (`File: src/ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/sign.rs`)

---

### Summary

`RerandomizationArguments` accepts an arbitrary `participants` field that is hashed into the HKDF-derived `delta` scalar. Neither `RerandomizedPresignOutput::rerandomize_presign` nor `sign()` verify that the `participants` used during rerandomization match the actual signing participant set. A malicious coordinator can distribute different `RerandomizationArguments` (with different `participants` lists) to different honest signers, causing each to compute a distinct `delta`, producing mutually inconsistent rerandomized presignature shares. The coordinator then collects shares from two such sessions and recovers the aggregate secret key via standard ECDSA nonce-reuse.

---

### Finding Description

`RerandomizationArguments` is a caller-constructed struct with a public `participants: ParticipantList` field. This field is concatenated into the HKDF info string inside `derive_randomness()`, so any difference in `participants` across callers produces a different `delta`. [1](#0-0) 

`derive_randomness()` appends each participant's bytes to the HKDF input: [2](#0-1) 

`rerandomize_presign` applies `delta` to the raw presignature shares: [3](#0-2) 

`RerandomizedPresignOutput` stores only the rerandomized scalars `(big_r, k, sigma)` — it does **not** record which `participants` were used to derive `delta`: [4](#0-3) 

`sign()` accepts a `presignature: RerandomizedPresignOutput` and a separate `participants: &[Participant]` slice, but performs **no check** that the two are consistent: [5](#0-4) 

The security documentation explicitly requires that all participants agree on `(h, ε, participants)` when creating `RerandomizedPresignOutput`, but the library provides no enforcement mechanism: [6](#0-5) 

---

### Impact Explanation

**Critical — Extraction/reconstruction of aggregate secret key material.**

If a malicious coordinator sends signer A `RerandomizationArguments { participants: {A,B,C}, ... }` and signer B `RerandomizationArguments { participants: {A,B,D}, ... }` (same `big_r`, same `msg_hash`, same `tweak`, different `participants`), then:

- Signer A computes `delta_A = HKDF(..., {A,B,C})` → `k_A' = k_A · delta_A^{-1}`
- Signer B computes `delta_B = HKDF(..., {A,B,D})` → `k_B' = k_B · delta_B^{-1}`

The resulting shares are rerandomized with **different** nonce scalars. The coordinator collects two signing sessions whose nonces are multiplicatively related (both derived from the same raw presignature `R`). This is exactly the nonce-reuse condition documented in the security notes: [7](#0-6) 

Two such signatures suffice to solve for the secret key `x` using standard ECDSA nonce-reuse algebra.

---

### Likelihood Explanation

The coordinator role is reachable without privileged assumptions — it is selected from the participant set and is the only party that distributes `RerandomizationArguments` to each signer. A single malicious participant who becomes coordinator can execute this attack in one signing session by sending subtly different `participants` lists (e.g., differing by one phantom or reordered-but-distinct ID) to different honest signers. Honest signers have no way to detect the discrepancy because `RerandomizedPresignOutput` carries no record of the `participants` used.

---

### Recommendation

1. **Bind participants to the rerandomized output.** Store the `participants` (or a commitment to them) inside `RerandomizedPresignOutput` so that `sign()` can assert they match the signing set.

2. **Enforce consistency in `sign()`.** Add a check:
   ```rust
   if presignature.participants != participants {
       return Err(InitializationError::BadParameters(
           "presignature participants must match signing participants".to_string(),
       ));
   }
   ```

3. **Alternatively**, require participants to broadcast their `RerandomizationArguments` via the echo-broadcast channel before signing, so all honest parties can verify agreement on `(h, ε, participants)` before contributing shares.

---

### Proof of Concept

```
Setup: N=5, t=2, so signing requires exactly 2t+1=5 participants {P1..P5}.

1. Presign: all 5 run presign → each holds PresignOutput with shared big_r.

2. Malicious coordinator (P1) distributes RerandomizationArguments:
   - To P2, P3, P4, P5: args_A = { participants: {P1,P2,P3,P4,P5}, msg_hash: h, tweak: ε, ... }
   - To itself (P1):    args_B = { participants: {P1,P2,P3,P4,X},  msg_hash: h, tweak: ε, ... }
     where X is any byte-distinct phantom ID.

3. Each signer calls rerandomize_presign(presig, &their_args):
   - P2..P5 compute delta_A = HKDF(..., {P1,P2,P3,P4,P5})
   - P1     computes delta_B = HKDF(..., {P1,P2,P3,P4,X})  ≠ delta_A

4. sign() is called with participants={P1..P5} and each party's RerandomizedPresignOutput.
   No check is performed that presignature.participants == signing participants.

5. Coordinator collects s_i shares. Because P1's share uses delta_B and others use delta_A,
   the combined s values from two such sessions satisfy the nonce-reuse relation:
   s1 * k = h + r*x  and  s2 * k' = h + r*x  where k'/k = delta_A/delta_B (known).
   Solving the two equations yields x (the aggregate secret key).

    pub fn derive_randomness(&self) -> Result<Scalar, ProtocolError> {
        // create a string containing (pk, msg_hash, big_r, sorted(participants))
        let pk_encoded_point = self.pk.to_encoded_point(true);
        let encoded_pk: &[u8] = pk_encoded_point.as_bytes();
        let encoded_tweak: &[u8] = &<Secp256K1ScalarField as Field>::serialize(&self.tweak.value());
        let encoded_msg_hash: &[u8] = &self.msg_hash;
        let big_r_encoded_point = self.big_r.to_encoded_point(true);
        let encoded_big_r: &[u8] = big_r_encoded_point.as_bytes();

        // concatenate all the bytes
        let mut concatenation = Vec::new();
        // 1 byte counter, used in the unlikely case that the hash result is 0
        concatenation.extend_from_slice(&[0u8, 1]);
        concatenation.extend_from_slice(encoded_pk);
        concatenation.extend_from_slice(encoded_tweak);
        concatenation.extend_from_slice(encoded_msg_hash);
        concatenation.extend_from_slice(encoded_big_r);
        // Append each ParticipantId's
        for participant in self.participants.participants() {
            concatenation.extend_from_slice(&participant.bytes());
        }

        // initiate hkdf with the salt and with some `good' entropy
        let hk = Hkdf::<sha3::Sha3_256>::new(Some(&Self::SALT), &self.entropy);

        let mut delta = Scalar::ZERO;
        // If the randomness created is 0 then we want to generate a new randomness
        while bool::from(delta.is_zero()) {
            // Generate randomization out of HKDF(counter, entropy, pk, msg_hash, big_r, participants, )
            // where entropy is a public but unpredictable random string.
            // The counter depends on the number of times we enter into this loop
            let mut okm = [0u8; 32];

            hk.expand(&concatenation, &mut okm)
                .map_err(|_| ProtocolError::HashingError)?;

            // derive the randomness delta
            delta = Scalar::from_repr(okm.into()).unwrap_or(
                // if delta falls outside the field
                // probability is negligible: in the order of 1/2^224
                Scalar::ZERO,
            );
            // Increment the counter, the probability that this overflows is astronomically low
            let concatenation_0 = concatenation
                .first_mut()
                .ok_or(ProtocolError::InvalidIndex)?;
            *concatenation_0 += 1;
        }
        Ok(delta)
    }

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    max_malicious: impl Into<MaxMalicious>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= 2 * max_malicious + 1
    let robust_ecdsa_threshold = max_malicious
        .into()
        .value()
        .checked_mul(2)
        .and_then(|v| v.checked_add(1))
        .ok_or_else(|| {
            InitializationError::BadParameters(
                "2*threshold+1 must be less than usize::MAX".to_string(),
            )
        })?;
    if robust_ecdsa_threshold > participants.len() {
        return Err(InitializationError::BadParameters(
            "2*max_malicious+1 must be less than or equals to participant count".to_string(),
        ));
    }

    // The next two conditions prevent split-view attacks
    // documented in docs/ecdsa/robust_ecdsa/signing.md
    if participants.len() != robust_ecdsa_threshold {
        return Err(InitializationError::BadParameters(
            "the number of participants during signing must be exactly 2*max_malicious+1 to avoid split view attacks".to_string(),
        ));
    }
    if bool::from(msg_hash.is_zero()) {
        return Err(InitializationError::BadParameters(
            "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
        ));
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
