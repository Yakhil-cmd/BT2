### Title
Anyone Can Call `update_rewards` with `disable_rewards: true` to Permanently Freeze Unclaimed Yield ‚Äî (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is publicly callable by any address and accepts a `disable_rewards: bool` parameter with no access control. An attacker can call this function every block with `disable_rewards: true`, consuming the protocol's single per-block reward-distribution slot and preventing any legitimate reward distribution, permanently freezing unclaimed yield for all stakers.

---

### Finding Description

`update_rewards` is exposed via `#[abi(embed_v0)]` under `IStakingRewardsManager` with no caller restriction beyond `general_prerequisites()` (which only checks pause state and non-zero caller). [1](#0-0) 

The function enforces a global, single-update-per-block invariant via `last_reward_block`: [2](#0-1) 

After validating the staker is active and has non-zero balance, it unconditionally writes the current block number to `last_reward_block`: [3](#0-2) 

Then, if `disable_rewards` is `true`, it returns immediately without distributing any rewards: [4](#0-3) 

Because `disable_rewards` is a plain public parameter with no access control, any unprivileged caller can pass `true`. Once `last_reward_block` is set to the current block, the assertion on line 1455 (`current_block_number > self.last_reward_block.read()`) will revert for every subsequent call in the same block ‚Äî including the legitimate keeper/consensus caller.

---

### Impact Explanation

An attacker who front-runs the legitimate `update_rewards` call every block with `disable_rewards: true` causes `last_reward_block` to be stamped without any rewards being calculated or transferred. No staker or pool receives block rewards for that block. Repeated across every block, this permanently freezes all unclaimed yield accrual for the entire protocol.

**Impact: High ‚Äî Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

Starknet transaction fees are low. A griefing attacker needs only to submit one transaction per block (or per epoch boundary, depending on the reward cadence). There is no economic barrier preventing a motivated attacker from sustaining this indefinitely. The attack requires no privileged role, no leaked key, and no external dependency ‚Äî only a valid staker address (publicly readable from `stakers` storage) and the ability to submit transactions.

**Likelihood: Medium.**

---

### Recommendation

Restrict who may call `update_rewards` with `disable_rewards: true`. Options:

1. **Remove `disable_rewards` from the public ABI** ‚Äî make it an internal flag set only by the contract itself during pre-consensus transitions.
2. **Gate `disable_rewards: true` behind a privileged role** (e.g., `only_app_governor` or `only_security_agent`).
3. **Restrict the entire `update_rewards` entrypoint** to a whitelisted caller (e.g., the attestation contract or a designated keeper address), analogous to how `update_rewards_from_attestation_contract` is gated: [5](#0-4) 

---

### Proof of Concept

```
// Attacker repeats this every block:
staking_contract.update_rewards(

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
