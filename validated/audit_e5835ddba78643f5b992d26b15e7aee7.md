### Title
Declare transactions bypass the `max_l2_gas_amount` gateway admission cap — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a `max_l2_gas_amount` ceiling on the `l2_gas.max_amount` field for every transaction type **except** `Declare`. The exemption is acknowledged only with a TODO comment. Any unprivileged sender can submit a `Declare` transaction whose `l2_gas.max_amount` is `u64::MAX` (or any value exceeding the configured cap), and the gateway will admit it to the mempool, breaking the invariant that every admitted transaction satisfies `l2_gas.max_amount ≤ max_l2_gas_amount`.

### Finding Description

In `validate_resource_bounds` the guard is written as an explicit no-op for `Declare`:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← nothing; the cap is never checked
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
```

The stateful validator's `validate_resource_bounds` / `validate_tx_l2_gas_price_within_threshold` only inspects `max_price_per_unit`, never `max_amount`, so the gap is not closed there either. The test `valid_l2_gas_amount_on_declare` explicitly asserts that a `Declare` with `max_amount: GasAmount(200)` passes when `max_l2_gas_amount: 100`, confirming the bypass is reachable.

At blockifier time, `perform_pre_validation_stage` calls `verify_can_pay_committed_bounds`, which computes `max_possible_fee = l2_gas.max_amount × l2_gas.max_price_per_unit`. With `max_amount = u64::MAX` and any non-zero price, this produces a fee that no real account can cover, so the transaction is ultimately rejected — but only **after** it has been admitted to the mempool by the gateway.

### Impact Explanation

The gateway admits `Declare` transactions that violate the `max_l2_gas_amount` invariant it enforces for every other transaction type. An attacker can flood the mempool with such transactions (each with a valid signature and nonce but an astronomically large gas claim), occupying mempool capacity and displacing legitimate transactions. This matches the allowed impact: **"Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."**

### Likelihood Explanation

No privilege is required. Any account can craft a `Declare` transaction with `l2_gas.max_amount = u64::MAX` and a `max_price_per_unit` at or above `min_gas_price`. Both the stateless and stateful validators will accept it. The attack is stateless and repeatable.

### Recommendation

Remove the `Declare` exemption and apply the same cap uniformly:

```rust
// Remove the special-case branch:
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If `Declare` transactions legitimately require a higher ceiling (e.g., for large Sierra programs), introduce a separate `max_l2_gas_amount_declare` config field rather than removing the check entirely.

### Proof of Concept

1. Build a `RpcDeclareTransactionV3` with `resource_bounds.l2_gas = ResourceBounds { max_amount: GasAmount(u64::MAX), max_price_per_unit: GasPrice(min_gas_price) }` and any valid Sierra class, nonce, and signature.
2. POST it to the gateway's `add_transaction` endpoint.
3. **Stateless check** (`validate_resource_bounds`): the `Declare` branch is taken; the `MaxGasAmountTooHigh` error is never raised. ✓ passes.
4. **Stateful check** (`validate_tx_l2_gas_price_within_threshold`): only `max_price_per_unit ≥ threshold` is verified. ✓ passes.
5. **Mempool admission** (`validate_by_mempool`): nonce is valid; transaction is inserted.
6. **Blockifier pre-validation** (`verify_can_pay_committed_bounds`): `committed_fee = u64::MAX × min_gas_price` far exceeds any real balance → transaction is rejected here, but it has already occupied a mempool slot.
7. Repeat with fresh nonces to exhaust mempool capacity. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}

    pub fn perform_pre_validation_stage<S: State + StateReader>(
        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }
