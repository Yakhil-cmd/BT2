Audit Report

## Title
Uninitialized `end_timestamp_seconds` in SNS `RewardEvent` Inflates Rewards Purse on Upgrade - (File: `rs/sns/governance/src/governance.rs`)

## Summary

`Governance::new()` only seeds `end_timestamp_seconds: Some(now)` into `latest_reward_event` when the field is entirely `None`. Any SNS whose persisted proto already holds a `RewardEvent` with `end_timestamp_seconds: None` (pre-dating the field's introduction) survives upgrade with the `None` intact. On the next periodic-task heartbeat, `distribute_rewards` treats Unix epoch 0 as the last reward boundary, computes ~20,254 elapsed rounds for a daily-round SNS, and accumulates a rewards purse of approximately 5.5× the total token supply, which is then distributed in full to voting neurons as maturity — permanently inflating the SNS token supply.

## Finding Description

**Root cause — `Governance::new()` guard is too narrow** [1](#0-0) 

The guard `if proto.latest_reward_event.is_none()` only fires when the entire field is absent. When the deserialized proto contains `Some(RewardEvent { end_timestamp_seconds: None, … })` — the state of every SNS deployed before `end_timestamp_seconds` was added as an `optional` proto field — the block is skipped and the `None` persists unchanged.

**`canister_post_upgrade` passes the raw proto straight through** [2](#0-1) [3](#0-2) 

`canister_post_upgrade` → `canister_init_()` → `Governance::new()`. No migration or back-fill step exists between deserialization and construction.

**`should_distribute_rewards` fires immediately** [4](#0-3) 

`end_timestamp_seconds.unwrap_or_default()` yields 0, so `seconds_since_last_reward_event ≈ now ≈ 1,750,000,000 s`, which vastly exceeds any `round_duration_seconds`. The function returns `true` on the first heartbeat.

**`distribute_rewards` computes ~20,254 rounds** [5](#0-4) 

`reward_start_timestamp_seconds = 0`, so `new_rounds_count = now / 86_400 ≈ 20,254` for a daily-round SNS.

**Rewards purse loop accumulates 20,000+ rounds** [6](#0-5) 

For early rounds where `round_duration_seconds * i < genesis_timestamp_seconds`, `saturating_sub` clamps `seconds_since_genesis` to 0, so `reward_rate_at(0)` returns the maximum initial rate. Every iteration adds a full round's reward to the purse.

**Overflow guard does not protect against realistic supplies** [7](#0-6) 

The guard only aborts if the purse exceeds `u64::MAX`. For any SNS with a supply below ~33 billion tokens (in e8s), the inflated purse fits in a `u64` and is distributed in full.

## Impact Explanation

All neurons that voted on any settled proposal in the inflated reward round receive maturity proportional to the bloated purse (~5.5× total supply for a 10%/year initial rate, daily rounds). Maturity can be converted to governance tokens via the spawn-neuron path, permanently minting tokens and destroying the economic model of the affected SNS. This constitutes **illegal minting / protocol insolvency** for in-scope SNS framework canisters — matching the **Critical** impact tier: *"Theft, permanent loss, illegal minting, or protocol insolvency involving exorbitant ICP/Cycles or in-scope chain-key/ledger assets."*

## Likelihood Explanation

The precondition — a persisted `RewardEvent` with `end_timestamp_seconds: None` — applies to every SNS deployed before the field was introduced to the proto. No attacker action is required beyond the SNS governance canister being upgraded to the current codebase (a routine NNS-driven operation). The periodic task fires automatically every ~10 seconds after upgrade with no user interaction. The exploit is deterministic and repeatable on any qualifying SNS instance.

## Recommendation

In `Governance::new()`, extend the initialization guard to back-fill `end_timestamp_seconds` when it is `None` inside an existing `latest_reward_event`:

```rust
if proto.latest_reward_event.is_none() {
    proto.latest_reward_event = Some(RewardEvent {
        end_timestamp_seconds: Some(now),
        // ... other fields
    });
} else if let Some(ref mut event) = proto.latest_reward_event {
    if event.end_timestamp_seconds.is_none() {
        // Back-fill for SNS instances deployed before this field existed.
        // Use genesis_timestamp_seconds as a safe lower bound.
        event.end_timestamp_seconds = Some(proto.genesis_timestamp_seconds);
    }
}
```

Alternatively, replace every `end_timestamp_seconds.unwrap_or_default()` call with `end_timestamp_seconds.unwrap_or(self.proto.genesis_timestamp_seconds)` so a missing value is treated as genesis rather than epoch 0.

## Proof of Concept

1. Construct or locate an SNS governance proto with `latest_reward_event: Some(RewardEvent { round: 1, end_timestamp_seconds: None, … })`.
2. Upgrade the SNS governance canister to the current code. `canister_post_upgrade` deserializes the proto and calls `Governance::new()`; the `is_none()` guard is `false`, so `end_timestamp_seconds` remains `None`.
3. Trigger (or wait for) `run_periodic_tasks`. `should_distribute_rewards` returns `true` because `now - 0 >> round_duration_seconds`.
4. `distribute_rewards` computes `new_rounds_count ≈ now / round_duration_seconds ≈ 20,254` and loops that many times, accumulating a rewards purse ≈ 5.55× total supply.
5. Voting neurons receive maturity proportional to the inflated purse; spawning those neurons mints tokens, permanently inflating the SNS token supply.

A deterministic integration test using PocketIC can reproduce this by: (a) initializing SNS governance with a proto containing `latest_reward_event { end_timestamp_seconds: None }`, (b) advancing the canister clock past one `round_duration_seconds`, and (c) asserting that the resulting `RewardEvent.distributed_e8s_equivalent` exceeds the total token supply.
