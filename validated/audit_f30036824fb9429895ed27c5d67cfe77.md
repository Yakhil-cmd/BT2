Audit Report

## Title
Missing `Command::Split` guard in `validate_manage_neuron_proposal` allows followees to split non-not-for-profit neurons via governance proposal — (`rs/nns/governance/src/governance.rs`)

## Summary
The comment at line 4686 of `governance.rs` explicitly states that only not-for-profit neurons may be targeted by `disburse/split/disburse-to-neuron` commands through a ManageNeuron proposal, but the guard block at lines 4688–4704 only rejects `Command::Disburse` and `Command::DisburseToNeuron`. `Command::Split` falls through the `_ => {}` arm. When the proposal is adopted, `perform_action` injects the managed neuron's own controller as `caller`, causing `split_neuron`'s sole authorization check (`is_controlled_by(caller)`) to always pass, executing the split unconditionally on any normal neuron.

## Finding Description
In `validate_manage_neuron_proposal` (`governance.rs` lines 4686–4704), the comment reads:

> "Only not-for-profit neurons can issue disburse/**split**/disburse-to-neuron commands through a proposal."

The guard checks `!is_managed_neuron_not_for_profit` and returns `NotAuthorized` for `Command::Disburse` and `Command::DisburseToNeuron`, but `Command::Split` is absent and falls through `_ => {}`. Validation therefore succeeds for a `Split` command targeting any neuron regardless of its `not_for_profit` flag.

Upon proposal adoption, `perform_action` (lines 4164–4168) retrieves the managed neuron's controller via `with_neuron_by_neuron_id_or_subaccount` and passes it as `caller` to `self.manage_neuron(&controller, &mgmt)`. This routes to `split_neuron` (lines 6167–6170). Inside `split_neuron`, the only authorization check is `if !parent_neuron.is_controlled_by(caller)` (line 2175). Because `caller` is the managed neuron's own controller — injected by `perform_action` — this check always passes and the split executes.

The existing test `test_not_for_profit_neurons` (test file lines 6792–6793) even comments "A normal neuron can't issue a manage neuron proposal to disburse, **split** or disburse-to-neuron" but only tests `Disburse`, leaving the `Split` path untested and the gap undetected.

## Impact Explanation
A registered followee of a normal (non-not-for-profit) neuron on `Topic::NeuronManagement` can force an unauthorized split of that neuron: the parent neuron's staked ICP is reduced by the chosen `amount_e8s`, and a new child neuron is created. No ICP is stolen (the child neuron's controller remains the victim's principal), but the victim's voting power is permanently reduced until they take corrective action (merge or dissolve the child neuron), and an attacker-chosen `memo` field is embedded in the child neuron. This constitutes unauthorized manipulation of a governance asset (neuron stake and voting power) contrary to the explicitly stated invariant in the code. This matches the **Medium** bounty tier ($200–$2,000): the attack requires strict preconditions (being a registered followee) but produces concrete, meaningful harm to the victim neuron's governance participation.

## Likelihood Explanation
The attacker must already be a registered followee of the victim neuron on `Topic::NeuronManagement` — a position the victim explicitly granted. If the attacker is the sole followee, the proposal is immediately self-adopted (the ballot set contains only the attacker's neuron, which votes `Yes` by default as proposer). If there are multiple followees, the attacker needs a majority. This is a realistic scenario for neurons that delegate management to a team or a single trusted entity. The attack is repeatable and requires no special technical infrastructure beyond a standard NNS interaction.

## Recommendation
Add `Command::Split(_)` to the not-for-profit guard in `validate_manage_neuron_proposal`, matching the stated invariant in the comment:

```rust
// Only not-for-profit neurons can issue disburse/split/disburse-to-neuron
// commands through a proposal.
if !is_managed_neuron_not_for_profit {
    match command {
        Command::Disburse(_) => {
            return Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                "Cannot issue a disburse command through a proposal",
            ));
        }
        Command::DisburseToNeuron(_) => {
            return Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                "Cannot issue a disburse to neuron command through a proposal",
            ));
        }
        Command::Split(_) => {
            return Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                "Cannot issue a split command through a proposal",
            ));
        }
        _ => {}
    }
}
```

Additionally, extend `test_not_for_profit_neurons` to assert that `Command::Split` is also rejected for a normal neuron, closing the test coverage gap.

## Proof of Concept
1. Create neuron A (normal, `not_for_profit = false`) with sufficient stake and set neuron B as its sole followee on `Topic::NeuronManagement`.
2. From neuron B's controller, call `manage_neuron` with `MakeProposal` wrapping `ManageNeuron { neuron_id: A, command: Split { amount_e8s: X } }`.
3. `validate_manage_neuron_proposal` is called; `Command::Split` falls through `_ => {}` at line 4702 — proposal is accepted.
4. `compute_ballots_for_manage_neuron_proposal` (lines 5427–5467) creates a ballot set containing only neuron B; B is the proposer so it votes `Yes` immediately — proposal is adopted.
5. `perform_action` (lines 4164–4168) fetches A's controller and calls `manage_neuron(&controller_of_A, split_cmd)`.
6. `split_neuron` line 2175 checks `is_controlled_by(controller_of_A)` — passes trivially.
7. Assert: neuron A's `minted_stake_e8s` is reduced by X and a new child neuron exists with A's controller. The split executed on a non-not-for-profit neuron via proposal, contrary to the stated invariant.
