### Title
No Admin Mechanism to Remove Pending Pool Exit Intents When Paused — (File: `src/staking/staking.cairo`)

### Summary
The Staking contract stores pending undelegation intents in the `pool_exit_intents` map. When the contract is paused, `general_prerequisites()` blocks all state-changing functions including `remove_from_delegation_pool_intent` — the only path through which an intent can be cancelled. No privileged function exists to clear entries from `pool_exit_intents` while paused. Pool members with active exit intents cannot cancel them during a pause, temporarily freezing their funds in the exit-intent state.

### Finding Description
The `pool_exit_intents` storage map holds `UndelegateIntentValue` entries (containing `unpool_time`, `amount`, and `token_address`) keyed by `UndelegateIntentKey` (`pool_contract` + `identifier`). [1](#0-0) 

The only way to cancel an existing intent is to call `remove_from_delegation_pool_intent` with `amount = 0`, which resets the entry: [2](#0-1) 

This function calls `general_prerequisites()` on line 1055 before doing anything else. The `is_paused` flag is set by the security agent via `pause()`: [3](#0-2) 

`general_prerequisites()` is the standard guard used across all state-changing functions in the contract. When the contract is paused, it reverts, blocking the cancellation path entirely.

The full call chain a pool member must traverse to cancel an intent is:

1. `Pool::exit_delegation_pool_intent(amount=0)` — `pool.cairo:256`
2. → `Pool::undelegate_from_staking_contract_intent(pool_member, 0)` — `pool.cairo:637` [4](#0-3) 

3. → `IStakingPool::remove_from_delegation_pool_intent(staker_address, identifier, 0)` — `staking.cairo:1048`
4. → `general_prerequisites()` — **reverts when paused**

There is no `IStakingPause`, `IStakingConfig`, or any other interface that exposes an admin-callable function to remove or clear entries from `pool_exit_intents` while the contract is in a paused state. The `IStakingPause` interface only exposes `pause` and `unpause`: [5](#0-4) 

Additionally, `remove_from_delegation_pool_action` — the execution path — also calls `general_prerequisites()`: [6](#0-5) 

This means that during a pause, exit intents are completely immutable: they cannot be cancelled by the pool member, executed by the pool member, or cleared by any admin role.

### Impact Explanation
Pool members who have called `exit_delegation_pool_intent` before a pause cannot cancel their intent during the pause window. Their funds are locked in the exit-intent state for the entire duration of the pause. They cannot re-delegate, increase their position, or recover their funds until the contract is unpaused. This constitutes **Temporary freezing of funds** (High). Furthermore, the `unpool_time` timer continues to advance during the pause; when the contract is unpaused, intents whose timers have already expired become immediately executable, removing any ability for the protocol to intervene.

### Likelihood Explanation
The security agent can pause the contract at any time for any security reason. Pool members routinely call `exit_delegation_pool_intent` as part of normal protocol usage. The overlap — a pool member holding an active intent at the moment of a pause — is a common operational scenario, not a contrived edge case. No attacker action is required beyond having previously called `exit_delegation_pool_intent`.

### Recommendation
Add a privileged function (callable only by the security admin or app governor, and only when the contract is paused) that accepts an `UndelegateIntentKey` and clears the corresponding entry from `pool_exit_intents`. This mirrors the recommendation in the original report: an owner-callable, pause-gated removal function for queued state. The function should also restore the delegated balance in the staker's trace and in `tokens_total_stake_trace` to maintain accounting invariants.

### Proof of Concept
1. Pool member calls `Pool::exit_delegation_pool_intent(amount)` — intent is written to `pool_exit_intents` in the Staking contract with a future `unpool_time`.
2. Security agent calls `Staking::pause()` — contract enters paused state.
3. Pool member changes their mind and calls `Pool::exit_delegation_pool_intent(0)` to cancel.
4. Pool contract calls `undelegate_from_staking_contract_intent(pool_member, 0)`.
5. Since `is_staker_active()` is true, this calls `staking_pool_dispatcher.remove_from_delegation_pool_intent(staker_address, identifier, 0)`.
6. Inside `Staking::remove_from_delegation_pool_intent`, `general_prerequisites()` reverts because `is_paused == true`.
7. The cancellation fails. The intent remains in `pool_exit_intents`.
8. No admin role has a function to clear the intent.
9. Pool member's funds remain frozen in the exit-intent state for the entire pause duration.
10. When the contract is unpaused and `unpool_time` has elapsed, the pool member can immediately execute `exit_delegation_pool_action`, withdrawing funds the protocol had no ability to intercept. [7](#0-6) [8](#0-7)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }
