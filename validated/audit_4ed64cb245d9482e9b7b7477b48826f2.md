### Title
Unprotected `new()` Initialization Allows Any Caller to Seize Engine Ownership - (File: `engine/src/contract_methods/admin.rs`)

---

### Summary

The `new()` function that initializes the Aurora Engine's on-chain state performs no caller authentication. Any NEAR account that calls it before the legitimate deployer does will become the `owner_id` of the engine, gaining full administrative control including the ability to add a full-access key to the engine account, redirect the ETH connector bridge, pause the contract, and deploy arbitrary replacement code.

---

### Finding Description

`engine/src/contract_methods/admin.rs` exposes the initialization entry point:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input)...;
    // ...
    state::set_state(&mut io, &state)?;
    Ok(())
}
``` [1](#0-0) 

The only guard is the `ERR_ALREADY_INITIALIZED` check — there is no `env.predecessor_account_id()` validation. The function accepts caller-supplied `owner_id`, `chain_id`, `upgrade_delay_blocks`, and `key_manager` from `NewCallArgs` and writes them directly to state. [2](#0-1) 

This function is exposed as the public NEAR contract method `new`: [3](#0-2) 

The official deployment helper `EngineContractBuilder::deploy_and_init()` issues the WASM deploy and the `new()` call as **two separate transactions** (two separate `.await` points), creating a window between them: [4](#0-3) 

---

### Impact Explanation

An attacker who front-runs `new()` and sets themselves as `owner_id` gains access to every owner-gated function. The most severe escalation path is `attach_full_access_key`: [5](#0-4) 

The code comment itself acknowledges: *"This action is dangerous because it adds a new full access key (FAK) to the Engine account."* With a FAK on the engine's NEAR account, the attacker has unrestricted control — they can transfer all NEAR tokens held by the engine, deploy arbitrary replacement WASM, and delete the account.

Additional owner-only paths that directly affect user funds:

- `set_eth_connector_contract_account` — redirects the ETH bridge connector to an attacker-controlled contract, enabling theft of all bridged ETH and ERC-20 tokens. [6](#0-5) 
- `pause_contract` — freezes all EVM execution, temporarily freezing all user funds. [7](#0-6) 
- `upgrade` — deploys malicious replacement contract code. [8](#0-7) 

**Impact tier: Critical** — direct theft of all at-rest bridged funds and permanent/temporary fund freeze.

---

### Likelihood Explanation

NEAR Protocol has a ~1-second block time. The deployment pattern in `engine-workspace/src/lib.rs` issues `deploy` and `new` as separate transactions across separate blocks. Any NEAR account that observes the deployment transaction (which is public on-chain) can submit a `new()` call in the next block before the deployer does. No special privileges, leaked keys, or social engineering are required — only the ability to submit a NEAR transaction. [4](#0-3) 

---

### Recommendation

Add a predecessor check inside `new()` that restricts the caller to the contract's own account (`current_account_id`), which is the only account that can act on itself in a NEAR batch deployment action:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // Only the contract account itself may initialize (enforced by batch atomicity)
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    // ...
}
```

Alternatively, enforce that deploy and `new()` are always issued in a single atomic NEAR batch action so no external account can interleave a call. [1](#0-0) 

---

### Proof of Concept

1. Attacker watches the NEAR blockchain for a `DeployContract` action targeting the Aurora Engine account ID.
2. In the next block, attacker submits a transaction calling `new` on that account with:
   ```json
   {
     "chain_id": "0x...",
     "owner_id": "attacker.near",
     "upgrade_delay_blocks": 0,
     "key_manager": "attacker.near"
   }
   ```
3. Because no state exists yet, `state::get_state(&io).is_ok()` returns `false` and the call succeeds, writing `owner_id = "attacker.near"` to storage. [9](#0-8) 
4. The legitimate deployer's subsequent `new()` call returns `ERR_ALREADY_INITIALIZED` and fails.
5. Attacker calls `attach_full_access_key` with their own public key. The `require_owner_only` check passes because `predecessor == owner_id == "attacker.near"`. [10](#0-9) 
6. Attacker now holds a full-access key on the Aurora Engine NEAR account and can drain all funds, redirect the bridge, or deploy malicious code.

#[named]
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())

pub fn set_eth_connector_contract_account<I: IO + Copy, E: Env>(
    io: I,
    env: &E,
) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let is_private = env.assert_private_call();

        if is_private.is_err() {
            require_owner_only(&state, &env.predecessor_account_id())?;

        let args: SetEthConnectorContractAccountArgs = io.read_input_borsh()?;

        set_connector_account_id(io, &args.account);
        set_connector_withdraw_serialization_type(io, &args.withdraw_serialize_type);

        Ok(())
    })
}
