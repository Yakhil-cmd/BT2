### Title
`max_l2_gas_amount` Upper-Bound Check Skipped for `Declare` Transactions in Stateless Validator — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a `max_l2_gas_amount` ceiling on every transaction type **except** `Declare`. An attacker with sufficient balance can submit a `Declare` transaction whose `l2_gas.max_amount` exceeds the configured gateway limit, bypassing the admission guard that exists for `Invoke` and `DeployAccount`. The gap is acknowledged by a developer TODO comment and is confirmed by a dedicated positive test.

### Finding Description

`validate_resource_bounds` in `StatelessTransactionValidator` applies the `max_l2_gas_amount` ceiling to `Invoke` and `DeployAccount` transactions but explicitly skips it for `Declare`:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { ... });
}
``` [1](#0-0) 

The production `max_l2_gas_amount` is `1,210,000,000` (from `config_schema.json`). [2](#0-1) 

The test `valid_l2_gas_amount_on_declare` explicitly confirms that a `Declare` with `l2_gas.max_amount = 200` passes when `max_l2_gas_amount = 100`: [3](#0-2) 

The stateful validator's `validate_resource_bounds` only checks the L2 gas **price** against the previous block's price — it never checks the gas **amount**: [4](#0-3) 

The only downstream guard that could catch an oversized `max_amount` is `verify_can_pay_committed_bounds`, which rejects the transaction when `max_amount × max_price_per_unit > account_balance`. However, this guard is bypassable: a well-funded account can set `max_amount` to any value above the limit (e.g., `max_l2_gas_amount + N`) while keeping `max_price_per_unit` at the minimum threshold, and the product will still be within the account's balance. [5](#0-4) 

`perform_pre_validation_stage` calls `verify_can_pay_committed_bounds` only when `charge_fee = true`, and the stateful validator runs full execution for `Declare` (not just the validate entry point): [6](#0-5) [7](#0-6) 

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions before sequencing.**

A `Declare` transaction whose `l2_gas.max_amount` exceeds `max_l2_gas_amount` passes the stateless validator and, if the sender's balance covers `max_amount × max_price_per_unit`, also passes the stateful validator. The transaction is then admitted to the mempool. The gateway's per-transaction L2 gas ceiling — the primary DoS admission guard — is rendered ineffective for the `Declare` transaction type. An attacker with a funded account can flood the mempool with `Declare` transactions carrying arbitrarily large `max_l2_gas_amount` values, bypassing the admission control that protects `Invoke` and `DeployAccount`.

### Likelihood Explanation

Any user who can submit a `Declare` transaction (i.e., any Starknet account holder) can trigger this. No privileged access is required. The only prerequisite is holding enough STRK to satisfy `verify_can_pay_committed_bounds` for the chosen `max_amount × max_price_per_unit` product, which is trivially achievable by choosing a small `max_price_per_unit` just above the stateful validator's threshold.

### Recommendation

Remove the `Declare` exemption in `validate_resource_bounds` and apply the same `max_l2_gas_amount` ceiling to all transaction types:

```rust
// Remove the Declare carve-out:
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If `Declare` transactions legitimately require a higher L2 gas ceiling (e.g., due to compilation overhead), introduce a separate `max_l2_gas_amount_declare` config parameter rather than removing the check entirely.

### Proof of Concept

The existing test `valid_l2_gas_amount_on_declare` already demonstrates the bypass:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator_test.rs
fn valid_l2_gas_amount_on_declare(config, rpc_tx_args) {
    // config: max_l2_gas_amount = 100
    // rpc_tx_args: l2_gas.max_amount = 200  <-- exceeds limit
    let tx_type = TransactionType::Declare;
    let tx_validator = StatelessTransactionValidator { config };
    let tx = rpc_tx_for_testing(tx_type, rpc_tx_args);
    assert_matches!(tx_validator.validate(&tx), Ok(()));  // passes!
}
``` [3](#0-2) 

To demonstrate full mempool admission: submit a `Declare` V3 transaction with `l2_gas.max_amount = max_l2_gas_amount + 1` and `l2_gas.max_price_per_unit` set to the minimum threshold. The stateless validator accepts it (no `max_l2_gas_amount` check for `Declare`), the stateful validator accepts it (only checks gas price, not amount), and `verify_can_pay_committed_bounds` accepts it (balance covers `(max_l2_gas_amount + 1) × threshold`). The transaction enters the mempool, bypassing the admission guard that would have rejected an equivalent `Invoke` or `DeployAccount`.

pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}

    pub fn perform_validations(&mut self, tx: AccountTransaction) -> StatefulValidatorResult<()> {
        // Deploy account transaction should be fully executed, since the constructor must run
        // before `__validate_deploy__`. The execution already includes all necessary validations,
        // so they are skipped here.
        // Declare transaction should also be fully executed - otherwise, if we only go through
        // the validate phase, we would miss the check that the class was not declared before.
        match tx.tx {
            ApiTransaction::DeployAccount(_) | ApiTransaction::Declare(_) => self.execute(tx),
            ApiTransaction::Invoke(_) => {
                let tx_context = Arc::new(self.tx_executor.block_context.to_tx_context(&tx));
                tx.perform_pre_validation_stage(self.state(), &tx_context)?;
                if !tx.execution_flags.validate {
                    return Ok(());
                }

                // `__validate__` call.
                let (_optional_call_info, actual_cost) = self.validate(&tx, tx_context.clone())?;

                // Post validations.
                PostValidationReport::verify(
                    &tx_context,
                    &actual_cost,
                    tx.execution_flags.charge_fee,
                )?;

                Ok(())
            }
        }
    }

    pub fn perform_pre_validation_stage<S: State + StateReader>(
        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }
