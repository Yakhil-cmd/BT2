### Title
Wrong Variable Passed to `_update_rewards` in Consensus Path Inflates Staker Rewards - (File: src/staking/staking.cairo)

### Summary

In `StakingRewardsManagerImpl::update_rewards`, the staker's own balance (`staker_total_strk_balance`, `staker_total_btc_balance`) is passed as the `strk_total_stake` / `btc_total_stake` arguments to `_update_rewards`, instead of the global total staking power. The attestation path (`update_rewards_from_attestation_contract`) correctly fetches the global total via `get_current_total_staking_power()`, but the consensus path substitutes the individual staker's balance — the exact same class of wrong-variable bug as the reference report.

### Finding Description

In `update_rewards_from_attestation_contract` (the pre-consensus path), the global total staking power is correctly retrieved and forwarded:

```cairo
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
// ...
self._update_rewards(
    :staker_address,
    strk_total_rewards: strk_epoch_rewards,
    btc_total_rewards: btc_epoch_rewards,
    :strk_total_stake,      // ← global total
    :btc_total_stake,       // ← global total
    ...
);
``` [1](#0-0) 

In `update_rewards` (the consensus path), the code fetches the **staker's own balance** for the purpose of validating non-zero stake, but then mistakenly forwards those same staker-scoped variables as the `strk_total_stake` / `btc_total_stake` arguments:

```cairo
let (staker_total_strk_balance, staker_total_btc_balance) = self
    .get_staker_total_strk_btc_balance_at_epoch(
        :staker_address, :staker_pool_info, epoch_id: curr_epoch,
    );
// ...
self._update_rewards(
    :staker_address,
    strk_total_rewards: strk_block_rewards,
    btc_total_rewards: btc_block_rewards,
    strk_total_stake: staker_total_strk_balance,   // ← BUG: staker's own balance
    btc_total_stake: staker_total_btc_balance,     // ← BUG: staker's own balance
    ...
);
``` [2](#0-1) 

`_update_rewards` uses `strk_total_stake` as the denominator when computing each staker's proportional share of block rewards. When the denominator equals the staker's own balance rather than the global total, the ratio collapses to 1 and the staker receives the **entire** block reward instead of their proportional fraction.

### Impact Explanation

- **Theft of unclaimed yield (High):** A staker who calls `update_rewards` in the consensus phase receives 100 % of the block reward for that block. All other stakers' proportional yield for that block is stolen.
- **Protocol insolvency risk (Critical):** Because `update_rewards` is callable once per block by any staker, every block's full reward can be claimed by a single actor, draining the reward supplier far faster than the minting curve intends.

### Likelihood Explanation

- `update_rewards` is a public, permissionless function — any registered staker can call it.
- The consensus rewards path activates after `consensus_rewards_first_epoch` is set, which is a planned protocol milestone.
- The only per-block guard is `last_reward_block`, which prevents double-calling within the same block but does not restrict who calls it.
- No special role, leaked key, or external dependency is required; a normal staker triggers the bug by calling `update_rewards` for themselves.

### Recommendation

Replace the staker-scoped balance variables with the global total staking power, mirroring the attestation path:

```cairo
// Before (buggy):
strk_total_stake: staker_total_strk_balance,
btc_total_stake: staker_total_btc_balance,

// After (correct):
let (strk_total_stake, btc_total_stake) =
    self.get_total_staking_power_at_epoch(epoch_id: curr_epoch);
// ...
strk_total_stake: strk_total_stake,
btc_total_stake: btc_total_stake,
```

### Proof of Concept

1. Protocol enters consensus rewards phase (`consensus_rewards_first_epoch` is reached).
2. Attacker (a registered staker with, say, 1 % of total stake) calls `update_rewards(staker_address: attacker, disable_rewards: false)` once per block.
3. Inside `update_rewards`, `staker_total_strk_balance` = attacker's 1 % share is passed as `strk_total_stake` to `_update_rewards`.
4. `_update_rewards` computes `attacker_reward = block_reward * attacker_balance / strk_total_stake = block_reward * X / X = block_reward` — the full block reward.
5. All other stakers receive zero reward for that block; the attacker accumulates unclaimed yield equal to the full block reward every block they call the function. [3](#0-2) [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
