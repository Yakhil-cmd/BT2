### Title
Unprivileged Caller Can Distribute Block Rewards Without Attestation Linkage via `update_rewards` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control. The `disable_rewards: bool` parameter — the sole mechanism that prevents unearned reward distribution — is entirely caller-controlled. Any unprivileged address can call `update_rewards(staker_address, false)` for any active staker in any block, crediting that staker with block rewards they never earned through consensus attestation.

---

### Finding Description

`update_rewards` is exposed via `IStakingRewardsManager` with only two guards: the contract must not be paused, and the caller must not be the zero address. [1](#0-0) 

The function accepts `staker_address: ContractAddress` and `disable_rewards: bool` from the caller. In the consensus-rewards phase (after `consensus_rewards_first_epoch`), when `disable_rewards == false`, the function computes and accumulates block rewards for the named staker and forwards pool rewards to pool contracts: [2](#0-1) 

The global `last_reward_block` is written **before** the `disable_rewards` branch, so a single call — regardless of outcome — locks out all subsequent calls for that block: [3](#0-2) 

There is no on-chain check that the named staker actually submitted an attestation for the current block. The only mechanism that was intended to prevent unearned rewards is the `disable_rewards` flag, which the caller supplies freely.

This is structurally identical to the reported PayrollManager pattern: funds are retrieved/distributed from a shared pool without verifying that the recipient is linked to a validated record (a signed transaction there; a consensus attestation here).

---

### Impact Explanation

**High — Theft of unclaimed yield.**

When an attacker calls `update_rewards(victim_staker, false)` for a staker who did not attest:

1. `staker_info.unclaimed_rewards_own` is incremented with block rewards the staker did not earn.
2. Pool rewards are immediately transferred (in STRK) to the staker's delegation pool contracts via `send_rewards_to_delegation_pool` / `update_rewards_from_staking_contract`. [4](#0-3) 

Pool members can then call `claim_rewards` on the pool and withdraw those unearned STRK tokens: [5](#0-4) 

The reward supplier's `unclaimed_rewards` balance is drawn down for rewards that were never legitimately earned, constituting direct theft of yield from the protocol's reward pool. [6](#0-5) 

---

### Likelihood Explanation

**High.** The entry point is a public, permissionless function. No special role, key, or privilege is required. The attacker only needs to know any active staker address (all staker addresses are emitted in `NewStaker` events and stored in the public `stakers` vector). The attack is executable in every block of the consensus-rewards phase.

---

### Recommendation

Restrict `update_rewards` to a single authorized caller (e.g., the consensus/sequencer contract), or replace the caller-supplied `disable_rewards` flag with an on-chain lookup of whether the staker's operational address submitted a valid attestation for the current block — analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract()` in the pre-consensus phase. [7](#0-6) 

---

### Proof of Concept

```
// Attacker is any non-zero address.
// staker_A is any active staker with non-zero STRK balance who did NOT attest this block.

// Step 1: Attacker calls update_rewards before the consensus mechanism does.
staking_contract.update_rewards(staker_A, disable_rewards: false);

// Result:
// - staker_A.unclaimed_rewards_own += block_rewards (unearned)
// - Pool rewards transferred to staker_A's pool contracts (unearned)
// - last_reward_block = current_block  →  consensus mechanism is now locked out for this block

// Step 2: Pool member (or attacker who is also a pool member) calls claim_rewards.
pool_contract.claim_rewards(pool_member);
// Unearned STRK rewards are transferred to pool_member's reward_address.
```

The `last_reward_block` global lock means the legitimate consensus call for that block is permanently blocked, compounding the damage across every block in the epoch. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

                        pool_balance: pool_balance.to_native_amount(:decimals),
                    );

            }
            pool_rewards_list
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn claim_rewards(ref self: ContractState, amount: Amount) {
            // Asserts.
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            let unclaimed_rewards = self.unclaimed_rewards.read();
            assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Update unclaimed_rewards and transfer the requested rewards to the staking contract.
            self.unclaimed_rewards.write(unclaimed_rewards - amount);
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
        }
