### Title
Unpermissioned `update_rewards` Allows Any Caller to Permanently Freeze Unclaimed Yield for All Stakers - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is documented in the protocol spec as restricted to "Only starkware sequencer," but the implementation enforces no such check. Any non-zero address can call `update_rewards` with `disable_rewards: true` to advance the global `last_reward_block` counter without distributing any rewards, permanently blocking the legitimate sequencer from distributing rewards for that block. Because `last_reward_block` is a single contract-wide value, one attacker call per block freezes unclaimed yield for every staker simultaneously.

### Finding Description
`docs/spec.md` line 1645 states the access control for `update_rewards` is "Only starkware sequencer." The implementation at `src/staking/staking.cairo` lines 1447–1507 only calls `general_prerequisites()`, which checks two things: the contract is not paused, and the caller is not the zero address. There is no sequencer identity check.

```cairo
// src/staking/staking.cairo lines 1447-1507
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.          ← written BEFORE disable_rewards check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                            ← returns without distributing rewards
    }
    ...
}
```

`general_prerequisites` is:

```cairo
// src/staking/staking.cairo lines 1793-1797
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

The critical ordering flaw: `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated. An attacker who calls `update_rewards(valid_staker, disable_rewards: true)` at block N will:

1. Pass all assertions (contract not paused, caller non-zero, staker active, staker has non-zero balance).
2. Advance `last_reward_block` to N.
3. Return immediately without distributing any rewards.

When the legitimate sequencer subsequently attempts `update_rewards(any_staker, disable_rewards: false)` at block N, the assertion `current_block_number > last_reward_block` evaluates to `N > N`, which is false, and the call reverts with `REWARDS_ALREADY_UPDATED`.

Because `last_reward_block` is a single global storage slot (not per-staker), one attacker call per block is sufficient to deny rewards to every staker for that block.

### Impact Explanation
An attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block. Each call costs only gas. The result is that the sequencer's legitimate reward-distribution call is permanently blocked for every block the attacker front-runs. All stakers lose their per-block unclaimed yield indefinitely. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The attack requires no privileged role, no staked capital, and no leaked key. The attacker only needs:
- A valid staker address (all staker addresses are publicly visible on-chain via `NewStaker` events).
- Enough gas to call `update_rewards` once per block.

The function is publicly callable by any non-zero address. The attack is mechanically simple and can be automated with a bot. Likelihood is **High**.

### Recommendation
Add an access control check inside `update_rewards` that asserts the caller is the designated starkware sequencer address, consistent with the protocol specification. For example, store the sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` asserts `get_caller_address() == self.attestation_contract.read()`.

Alternatively, move the `self.last_reward_block.write(current_block_number)` call to after the `disable_rewards` guard so that a call with `disable_rewards: true` does not consume the block's reward slot.

### Proof of Concept

**Setup**: Consensus rewards are active (`is_pre_consensus()` returns false). At least one staker exists with non-zero balance.

**Attack steps**:

1. Attacker observes a new block N is produced.
2. Attacker calls `update_rewards(valid_staker_address, disable_rewards: true)`.
   - `general_prerequisites()` passes (contract not paused, attacker address ≠ 0).
   - `current_block_number (N) > last_reward_block (N-1)` passes.
   - Staker is active with non-zero balance — passes.
   - `last_reward_block` is written to N.
   - `disable_rewards == true` → function returns without distributing rewards.
3. Legitimate sequencer calls `update_rewards(staker_address, disable_rewards: false)` at block N.
   - `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
4. No staker receives rewards for block N.
5. Attacker repeats steps 1–4 for every subsequent block.

**Result**: All stakers' unclaimed yield is permanently frozen. The stakers' principal is not at risk, but all per-block reward accrual is halted for as long as the attacker sustains the griefing.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
