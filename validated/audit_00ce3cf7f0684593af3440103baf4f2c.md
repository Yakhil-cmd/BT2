### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Block Rewards for All Stakers - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `src/staking/staking.cairo` uses a global `last_reward_block` storage variable to enforce that rewards are distributed at most once per block. However, the function has no access control beyond a non-zero caller check, despite the specification explicitly stating it should be callable only by the Starknet sequencer. Any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` at every block, consuming the single per-block reward slot without distributing any rewards, permanently destroying block rewards for all stakers.

### Finding Description

The `update_rewards` function enforces a global one-call-per-block invariant via `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← global slot consumed HERE

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits with no rewards distributed
    }
    ...
```

`last_reward_block` is written **before** the `disable_rewards` guard. Calling `update_rewards(any_active_staker, true)` therefore:
1. Passes all checks (any non-zero caller, any active staker address).
2. Marks the current block as "already rewarded" globally.
3. Returns immediately without distributing rewards to anyone.

Any subsequent legitimate call in the same block (e.g., from the sequencer for the correct staker) reverts with `REWARDS_ALREADY_UPDATED`.

The specification explicitly restricts this function:

> **access control**: Only starkware sequencer.

But `general_prerequisites` only enforces pause state and non-zero caller — no sequencer check exists in the code.

### Impact Explanation

Block rewards in the consensus-rewards phase (V3) are calculated per block and distributed only when `update_rewards` succeeds. If the slot is consumed with `disable_rewards: true`, the block's rewards are **permanently lost** — there is no catch-up mechanism, no accumulation of missed blocks, and no way to retroactively distribute them. An attacker calling this every block causes a complete, permanent freeze of all stakers' unclaimed block yield.

This maps to the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The function is publicly callable by any non-zero address with no staking requirement.
- The attacker only needs to know any currently active staker address (publicly observable on-chain).
- On Starknet L2, transaction fees are low, making sustained block-by-block griefing economically viable.
- No privileged role, leaked key, or external dependency is required.

### Recommendation

Add an explicit sequencer-only access control check at the top of `update_rewards`, consistent with the specification. For example, store the authorized sequencer address in contract storage and assert `get_caller_address() == sequencer_address` before any state mutation. Alternatively, move the `last_reward_block.write` to after the `disable_rewards` guard so that a `disable_rewards: true` call does not consume the global slot.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker observes any active staker address `S` with non-zero STRK balance.
3. At every new block, attacker submits: `staking.update_rewards(S, disable_rewards: true)`.
4. `last_reward_block` is set to the current block number; function returns with no rewards distributed.
5. The sequencer's intended call to `update_rewards(legitimate_staker, false)` reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers lose their block rewards for that block permanently.
