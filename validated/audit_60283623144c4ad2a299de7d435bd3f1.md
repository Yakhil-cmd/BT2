### Title
Staking Contract Pause Causes Permanent Loss of Epoch Rewards Due to Expired Attestation Windows - (File: src/staking/staking.cairo)

### Summary
When the staking contract is paused, stakers cannot complete attestations because `update_rewards_from_attestation_contract` is blocked by `general_prerequisites()`. Since attestation windows are strictly time-bounded to a fixed block range per epoch, if a pause spans a staker's attestation window, the staker permanently loses that epoch's rewards with no recourse. No grace period or retroactive compensation mechanism exists.

### Finding Description
The `Staking` contract stores an `is_paused` flag and enforces it in `general_prerequisites()`, which is called at the top of every state-changing function. [1](#0-0) 

The attestation flow in `attestation.cairo` is a single atomic transaction:

1. `get_attestation_info_by_operational_address` — view call, not blocked by pause.
2. `_validate_attestation` — internal to attestation contract, not blocked.
3. `_mark_attestation_is_done` — writes to attestation contract storage.
4. `update_rewards_from_attestation_contract` on the staking contract — **blocked when paused**. [2](#0-1) 

Because step 4 reverts when the staking contract is paused, the entire `attest()` transaction reverts, including step 3. The attestation is never recorded.

The attestation window is strictly bounded: a staker may only attest in blocks `[target_block + MIN_ATTESTATION_WINDOW, target_block + attestation_window]`. [3](#0-2) 

`MIN_ATTESTATION_WINDOW` is 11 blocks. The window is narrow and epoch-specific. If the staking contract is paused for the duration of a staker's window, the staker cannot attest in that epoch and permanently loses the epoch's rewards. There is no mechanism to retroactively credit rewards for epochs missed due to a pause.

The `update_rewards_from_attestation_contract` function explicitly checks `general_prerequisites()` first: [4](#0-3) 

The pause and unpause functions have no grace period: [5](#0-4) 

The test suite confirms `update_rewards_from_attestation_contract` is blocked when paused: [6](#0-5) 

### Impact Explanation
Every staker whose attestation window falls entirely within a pause period permanently loses that epoch's STRK rewards. Rewards are only credited via `update_rewards_from_attestation_contract` (pre-consensus) or `update_rewards` (consensus). There is no retroactive path. This constitutes **permanent freezing of unclaimed yield** for affected stakers.

### Likelihood Explanation
The security agent can pause the contract at any time without notice. Attestation windows are narrow (as few as 11 blocks above `MIN_ATTESTATION_WINDOW`). A pause of even moderate duration (e.g., hours during an incident response) will span the attestation windows of many stakers across one or more epochs, causing irreversible reward loss for all of them simultaneously.

### Recommendation
Add a post-unpause grace period analogous to the recommendation in the original report. Track the total paused duration and either:
- Exclude paused epochs from the attestation requirement, or
- Record a `paused_until` timestamp on unpause and allow retroactive attestation credit for epochs that were fully covered by the pause window.

self.min_attestation_timestamp.write(Time::now().add(delta: ATTESTATION_UNPAUSE_GRACE));
self.is_paused.write(false);

assert!(Time::now() >= self.min_attestation_timestamp.read(), ...);

### Proof of Concept
1. Staker S stakes and has an attestation target block `X` in epoch E, with window `[X+11, X+20]`.
2. Security agent calls `pause()` at block `X+10`.
3. Staker S calls `attest(block_hash)` at block `X+12` — the call to `update_rewards_from_attestation_contract` reverts with `"Contract is paused"`, rolling back the entire transaction including `_mark_attestation_is_done`.
4. Security agent calls `unpause()` at block `X+21` (after the window closes).
5. Staker S attempts `attest()` — now fails with `ATTEST_OUT_OF_WINDOW` because `current_block > X + attestation_window`.
6. Staker S has permanently lost epoch E rewards. No recovery path exists.

    impl StakingPauseImpl of IStakingPause<ContractState> {
        fn pause(ref self: ContractState) {
            self.roles.only_security_agent();
            if self.is_paused() {
                return;
            }
            self.is_paused.write(true);
            self.emit(PauseEvents::Paused { account: get_caller_address() });
        }

        fn unpause(ref self: ContractState) {
            self.roles.only_security_admin();
            if !self.is_paused() {
                return;
            }
            self.is_paused.write(false);
            self.emit(PauseEvents::Unpaused { account: get_caller_address() });
        }

        fn attest(ref self: ContractState, block_hash: felt252) {
            let operational_address = get_caller_address();
            let staking_dispatcher = IStakingAttestationDispatcher {
                contract_address: self.staking_contract.read(),
            };
            // Note: This function checks for a zero staker address and will panic if so.
            let staking_attestation_info = staking_dispatcher
                .get_attestation_info_by_operational_address(:operational_address);
            self._validate_attestation(:block_hash, :staking_attestation_info);
            // Work is one tx per epoch.
            self
                ._mark_attestation_is_done(
                    staker_address: staking_attestation_info.staker_address(),
                    current_epoch: staking_attestation_info.epoch_id(),
                );
            staking_dispatcher
                .update_rewards_from_attestation_contract(
                    staker_address: staking_attestation_info.staker_address(),
                );
        }
