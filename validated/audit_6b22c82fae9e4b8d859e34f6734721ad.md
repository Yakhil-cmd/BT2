### Title
Unbounded Loop in `calculate_rewards` Enables Permanent Freezing of Pool Member Unclaimed Yield — (File: `src/pool/pool.cairo`)

---

### Summary

The `calculate_rewards` internal function in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop that iterates over every balance-change entry in a pool member's `pool_member_epoch_balance` trace since their last reward claim. Because each call to `add_to_delegation_pool` or `exit_delegation_pool_intent` in a distinct epoch appends a new entry to this trace, and because the progress pointer `entry_to_claim_from` is only advanced on a *successful* `claim_rewards` execution, a pool member who accumulates enough trace entries between claims will eventually cause every future `claim_rewards` call to exceed Starknet's per-transaction gas limit. The transaction reverts, the pointer is never advanced, and the pool member's unclaimed yield is permanently frozen.

---

### Finding Description

**Root cause — unbounded loop in `calculate_rewards`**

`calculate_rewards` is called from both `claim_rewards` (state-mutating) and `pool_member_info_v1` (view):

```
// src/pool/pool.cairo  lines 857-877
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    rewards += compute_rewards_rounded_down(...);
    from_sigma = to_sigma;
    from_balance = pool_member_checkpoint.balance();
    entry_to_claim_from += 1;
}
```

The loop iterates from `entry_to_claim_from` (the last successfully processed index) up to the first entry whose epoch ≥ current epoch. Every storage read inside the loop (`pool_member_trace.at`, `find_sigma`) is expensive on Starknet.

**How the trace grows**

`set_member_balance` (called by both `add_to_delegation_pool` and `exit_delegation_pool_intent`) inserts with key `current_epoch + K`:

```
// src/pool/pool.cairo  lines 718-729
fn set_member_balance(ref self: ContractState, pool_member: ContractAddress, amount: Amount) {
    let trace = self.pool_member_epoch_balance.entry(pool_member);
    let pool_member_balance = PoolMemberBalanceTrait::new(
        balance: amount,
        cumulative_rewards_trace_idx: self.cumulative_rewards_trace_length() + 1,
    );
    trace.insert(key: self.get_epoch_plus_k(), value: pool_member_balance);
}
```

The `insert` implementation in `src/pool/pool_member_balance_trace/trace.cairo` (lines 152–175) only overwrites the last entry when the key matches; otherwise it **appends** a new checkpoint. Therefore, one balance change per epoch = one new trace entry per epoch.

**Why the freeze is permanent**

`entry_to_claim_from` is stored in `pool_member_info` and is only written back on a *successful* `claim_rewards` call:

```
// src/pool/pool.cairo  lines 349-359
let (mut rewards, updated_entry_to_claim_from) = self.calculate_rewards(
    :pool_member,
    from_checkpoint: pool_member_info.reward_checkpoint,
    :until_checkpoint,
    entry_to_claim_from: pool_member_info.entry_to_claim_from,
);
// ...
pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
pool_member_info.reward_checkpoint = until_checkpoint;
```

There is no partial-range claim mechanism. If the loop exceeds the gas limit, the entire transaction reverts, `entry_to_claim_from` is not advanced, and every subsequent `claim_rewards` attempt will encounter the same (or larger) trace and revert again. The pool member's accumulated yield is permanently inaccessible.

**Attacker-controlled entry path**

An unprivileged pool member (or their `reward_address`) can trigger this by:
1. Entering the delegation pool via `enter_delegation_pool`.
2. Calling `add_to_delegation_pool` or `exit_delegation_pool_intent` once per epoch across many epochs without ever calling `claim_rewards`.
3. After enough epochs, `claim_rewards` (and even the view `pool_member_info_v1`) will always revert due to gas exhaustion.

---

### Impact Explanation

**Permanent freezing of unclaimed yield** for the affected pool member. The principal (staked tokens) can still be recovered via `exit_delegation_pool_intent` / `exit_delegation_pool_action`, but all accumulated STRK rewards are irrecoverably locked in the contract. This matches the allowed High-severity impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

---

### Likelihood Explanation

**Medium.** The code itself acknowledges the risk with the comment *"This loop is unbounded but unlikely to exceed gas limits."* A pool member who actively manages their delegation (e.g., topping up or partially exiting once per epoch) over a period of months without claiming rewards will naturally accumulate hundreds of trace entries. Each iteration involves multiple storage reads (`pool_member_trace.at`, `find_sigma` with its own binary-search-like logic over `cumulative_rewards_trace`), making the per-iteration gas cost non-trivial. No special privilege is required; only normal delegator actions are needed.

---

### Recommendation

1. **Enforce periodic claiming**: Require or incentivize `claim_rewards` to be called before balance changes, resetting `entry_to_claim_from` and keeping the trace short.
2. **Batch/paginated claiming**: Add a `claim_rewards_up_to(epoch)` variant that processes only a bounded window of entries per call, advancing `entry_to_claim_from` incrementally.
3. **Cap trace growth**: Automatically consolidate or prune trace entries that have already been processed (i.e., entries with index < `entry_to_claim_from`) so the live trace length is bounded.

```text
    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
```
