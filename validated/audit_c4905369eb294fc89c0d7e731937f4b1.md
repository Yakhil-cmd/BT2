### Title
Unprivileged Caller Can Permanently Freeze Consensus Block Rewards via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control. It unconditionally writes `last_reward_block` to the current block number **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(valid_staker, disable_rewards: true)` at the start of every block, consuming the per-block reward slot without distributing any rewards. Every subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`, permanently freezing block-level reward distribution.

---

### Finding Description

`update_rewards` is exposed as a public entry point under `StakingRewardsManagerImpl`. Its only gate is `general_prerequisites()`, which checks for pause state and a non-zero caller — no role or identity check is performed. [1](#0-0) 

The function body executes in this order:

1. **Guard check** — reverts if `current_block_number <= last_reward_block`.
2. **Staker validation** — asserts the staker exists, is active, and has non-zero balance.
3. **State mutation** — writes `last_reward_block = current_block_number`. This happens unconditionally, before any reward-distribution logic.
4. **Early return** — if `disable_rewards || is_pre_consensus()`, the function returns immediately without distributing rewards. [2](#0-1) 

Because step 3 (the state mutation) precedes step 4 (the `disable_rewards` branch), an attacker who calls `update_rewards(valid_staker, disable_rewards: true)` first in a block:

- Consumes the single per-block reward slot (`last_reward_block` is now the current block).
- Distributes zero rewards.
- Causes every subsequent legitimate call in the same block to revert at the guard check with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The attacker only needs to supply any currently-active staker address with non-zero STRK balance — information that is fully public on-chain. There is no economic cost beyond gas.

---

### Impact Explanation

Each missed `update_rewards` call means the block's share of consensus rewards is never credited to `unclaimed_rewards_own` for the staker or forwarded to the delegation pool. Because `last_reward_block` is a single global slot, one attacker transaction per block is sufficient to suppress rewards for **all** stakers system-wide for that block. Repeated every block, this permanently freezes all consensus-era block rewards.

This matches: **High — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

- No privileged role is required; any EOA or contract can call `update_rewards`.
- The attacker only needs to know one valid, active staker address (trivially obtained from `NewStaker` events or `get_stakers`).
- The cost is one cheap transaction per block; it can be fully automated.
- On Starknet, transaction ordering within a block is sequencer-controlled, but the attacker does not need to frontrun in the traditional sense — calling `update_rewards(staker, true)` at any point in the block before the legitimate call is sufficient.

---

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a call with `disable_rewards: true` does not consume the per-block slot:

```cairo
// BEFORE (vulnerable):
self.last_reward_block.write(current_block_number);   // line 1485
if disable_rewards || self.is_pre_consensus() {        // line 1487
    return;
}

// AFTER (fixed):
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
```

Alternatively, restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a designated operator role), consistent with how `update_rewards_from_attestation_contract` is already gated. [4](#0-3) 

---

### Proof of Concept

1. Attacker identifies any active staker address `S` with non-zero STRK balance (e.g., from `NewStaker` events).
2. At the start of each block (or any time before the legitimate sequencer/consensus call), attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. The call passes all guards, writes `last_reward_block = current_block_number`, and returns without distributing rewards.
4. The legitimate `update_rewards` call (with `disable_rewards: false`) reverts:
   ```
   assert!(current_block_number > self.last_reward_block.read(), Error::REWARDS_ALREADY_UPDATED)
   ```
   because `last_reward_block` already equals `current_block_number`.
5. No rewards are credited for that block. Repeating this every block permanently freezes all consensus block rewards for all stakers. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
