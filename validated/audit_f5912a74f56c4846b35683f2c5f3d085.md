### Title
Gateway Signature-Verification Bypass via `skip_stateful_validations` Allows Unsigned Invoke Transactions Into Mempool for Victim Accounts — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`skip_stateful_validations` in the gateway's stateful validator skips the blockifier `__validate__` entry-point call (i.e., account signature verification) for any invoke transaction whose nonce is 1 and whose sender account has *any* transaction in the mempool or a recent block. Because the check is keyed only on `tx.sender_address()` and not on the identity of the submitter, an unprivileged attacker can inject an invoke transaction bearing an arbitrary (wrong) signature for a victim account into the mempool, bypassing the only on-chain signature check performed at admission time.

---

### Finding Description

**Root cause — `skip_stateful_validations`** [1](#0-0) 

```rust

    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {

            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())   // ← keyed on sender, not submitter
                .await

    }
    Ok(false)
}

**The mempool check is not identity-bound**

`account_tx_in_pool_or_recent_block` returns `true` for *any* transaction in the pool or recent block for the queried address: [4](#0-3) 

```rust
pub fn account_tx_in_pool_or_recent_block(&self, account_address: ContractAddress) -> bool {
    self.state.contains_account(account_address)
        || self.tx_pool.contains_account(account_address)
}
```

The comment in `skip_stateful_validations` assumes the only way an account can appear in the mempool with `account_nonce == 0` is if the *same user* submitted a `deploy_account`. That assumption is correct for the honest user, but it does not prevent a third-party attacker from submitting an invoke that names the victim as `sender_address`.

**Attack path**

1. Victim submits `deploy_account` for address `V` (account nonce on-chain = 0; deploy_account enters mempool). Victim pre-funds address `V` with fee tokens (required to pass `verify_can_pay_committed_bounds`).
2. Attacker submits `Invoke { sender_address: V, nonce: 1, signature: <garbage>, calldata: <arbitrary>, resource_bounds: <valid> }`.
3. Gateway stateful validation:
   - `validate_nonce`: nonce 1 ≥ account_nonce 0, within `max_allowed_nonce_gap` → passes. [5](#0-4) 
   - `validate_resource_bounds`: attacker sets valid L2 gas price → passes.
   - `validate_by_mempool`: checks duplicate hash / nonce range only, no signature check → passes.
   - `skip_stateful_validations`: `nonce == 1`, `account_nonce == 0`, `account_tx_in_pool_or_recent_block(V)` = **true** (victim's deploy_account is in pool) → returns `true`.
   - `run_validate_entry_point` with `validate = false`: `perform_pre_validation_stage` runs (nonce bump with `strict_nonce_check=false`, fee/balance check against victim's pre-funded balance) → passes; `__validate__` is **never called**.
4. Attacker's transaction is added to the mempool with `tx_hash` derived from the attacker-controlled fields.
5. Batcher executes the block: deploy_account (nonce 0) runs first, deploying `V`. Then the attacker's invoke (nonce 1) runs: `perform_pre_validation_stage` passes (nonce 1 == 1), `validate_tx` calls `__validate__` on the newly deployed contract, which rejects the garbage signature → transaction is **rejected** (not reverted).

**Broken invariant**: The gateway's stateful validation path guarantees that every transaction admitted to the mempool has passed `__validate__` (or is a deploy_account/declare that is fully executed). The `skip_stateful_validations` path breaks this guarantee for invoke transactions targeting accounts that are being deployed by a third party.

---

### Impact Explanation

Matches **High — "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

- An unsigned (garbage-signature) invoke transaction for a victim account is admitted to the mempool.
- If the attacker's transaction occupies nonce 1 for the victim's address before the victim's own legitimate invoke, the victim's transaction is rejected by the mempool as a duplicate nonce (or must pay a higher tip to replace it), constituting a targeted denial-of-service against the victim's first post-deploy transaction.
- The batcher wastes execution resources on the invalid transaction.
- The rejected transaction is not included in the block, so no fee is charged and no nonce is permanently incremented; however the window of disruption is real and repeatable.

---

### Likelihood Explanation

- Requires the victim to have a `deploy_account` in the mempool and a pre-funded account address — both are standard conditions for any new Starknet account.
- The attacker needs only to monitor the mempool for new `deploy_account` transactions (publicly visible) and submit the spoofed invoke before the victim does.
- No privileged access, no special contract, no on-chain funds required from the attacker.

---

### Recommendation

In `skip_stateful_validations`, replace the generic `account_tx_in_pool_or_recent_block` check with a check that the mempool contains a **`deploy_account` transaction specifically** for the sender address. Alternatively, require that the `deploy_account` transaction hash be supplied by the submitter (as the legacy `native_blockifier` `PyValidator` does via `deploy_account_tx_hash`) and verify it against the mempool before skipping `__validate__`. [6](#0-5) 

The `PyValidator` version correctly requires the caller to supply `deploy_account_tx_hash` and verifies it is non-`None` before skipping validation, preventing a third party from triggering the skip for a victim's address.

    async fn run_validate_entry_point(
        &mut self,
        executable_tx: &ExecutableTransaction,
        skip_validate: bool,
    ) -> StatefulTransactionValidatorResult<()> {
        let only_query = false;
        let charge_fee = enforce_fee(executable_tx, only_query);
        let strict_nonce_check = false;
        let execution_flags =
            ExecutionFlags { only_query, charge_fee, validate: !skip_validate, strict_nonce_check };

        let account_tx = AccountTransaction { tx: executable_tx.clone(), execution_flags };

        // Build block context.
        let mut versioned_constants = VersionedConstants::get_versioned_constants(
            self.config.versioned_constants_overrides.clone(),
        );
        // The validation of a transaction is not affected by the casm hash migration.
        versioned_constants.disable_casm_hash_migration();

        let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
        block_info.block_number = block_info.block_number.unchecked_next();
        let block_context = BlockContext::new(
            block_info,
            self.chain_info.clone(),
            versioned_constants,
            BouncerConfig::max(),
        );

        // Move state into the blocking task and run CPU-heavy validation.
        let state_reader_and_contract_manager = self.take_state_reader_and_contract_manager();

        let cur_span = Span::current();
        #[allow(clippy::result_large_err)]
        tokio::task::spawn_blocking(move || {
            cur_span.in_scope(|| {
                let state = CachedState::new(state_reader_and_contract_manager);
                let mut blockifier_validator = StatefulValidator::create(state, block_context);
                blockifier_validator.validate(account_tx)
            })
        })
        .await
        .map_err(|e| StarknetError {
            code: StarknetErrorCode::UnknownErrorCode(
                "StarknetErrorCode.InternalError".to_string(),
            ),
            message: format!("Blocking task join error when running the validate entry point: {e}"),
        })?
        .map_err(|e| StarknetError {
            code: StarknetErrorCode::KnownErrorCode(KnownStarknetErrorCode::ValidateFailure),
            message: e.to_string(),
        })?;
        Ok(())
    }

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

```rust
    pub fn perform_validations(&mut self, tx: AccountTransaction) -> StatefulValidatorResult<()> {
        // Deploy account transaction should be fully executed, since the constructor must run
        // before `__validate_deploy__`. The execution already includes all necessary validations,
        // so they are skipped here.
        // Declare transaction should also be fully executed - otherwise, if we only go through
        // the validate phase, we would miss the check that the class was not declared before.
        match tx.tx {
            ApiTransaction::DeployAccount(_) | ApiTransaction::Declare(_) => self.execute(tx),
            ApiTransaction::Invoke(_) => {
                let tx_context = Arc::new(self.tx_executor.block_context.to_tx_context(&tx));
                tx.perform_pre_validation_stage(self.state(), &tx_context)?;
                if !tx.execution_flags.validate {
                    return Ok(());
                }

                // `__validate__` call.
                let (_optional_call_info, actual_cost) = self.validate(&tx, tx_context.clone())?;

                // Post validations.
                PostValidationReport::verify(
                    &tx_context,
                    &actual_cost,
                    tx.execution_flags.charge_fee,
                )?;

                Ok(())
            }
        }
    }

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
