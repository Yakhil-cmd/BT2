### Title
Secret Cryptographic Material Exposed via `Debug` Trait on `TripleShare` and `KeygenOutput` - (File: `src/ecdsa/ot_based_ecdsa/triples/mod.rs`, `src/lib.rs`)

### Summary
The `TripleShare` and `KeygenOutput` structs derive Rust's `Debug` trait while containing secret cryptographic scalars. Any caller, logging framework, panic handler, or error-propagation path that formats these types with `{:?}` will emit the raw secret material in plaintext — the direct analog of the eth2deposit password-in-shell-history issue, transposed to Rust's standard observable channel.

### Finding Description
`TripleShare` in `src/ecdsa/ot_based_ecdsa/triples/mod.rs` derives `Debug` unconditionally:

```rust
#[derive(Clone, Debug, Serialize, Deserialize, ZeroizeOnDrop)]
pub struct TripleShare {
    pub a: Scalar,
    pub b: Scalar,
    pub c: Scalar,
}
```

The module's own documentation explicitly states the consequence:

> "It's important that the value of the underlying scalars in the triple is kept secret, **otherwise the private key used to create a signature with that triple could be recovered**."

`KeygenOutput<C>` in `src/lib.rs` also derives `Debug` while holding `private_share: SigningShare<C>`:

```rust
#[derive(Debug, Clone, Deserialize, Serialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct KeygenOutput<C: Ciphersuite> {
    pub private_share: SigningShare<C>,
    pub public_key: VerifyingKey<C>,
}
```

Additionally, `PresignOutput` and `RerandomizedPresignOutput` for both OT-based ECDSA (`src/ecdsa/ot_based_ecdsa/mod.rs`) and Robust ECDSA (`src/ecdsa/robust_ecdsa/mod.rs`) derive `Debug` while containing secret nonce shares (`k`, `sigma`, `alpha`, `beta`, `c`, `e`).

In Rust, `#[derive(Debug)]` produces a `fmt::Debug` implementation that prints every field by name and value. There is no redaction. Any of the following reachable paths will emit the secret scalars in plaintext:
- `println!("{:?}", triple_share)` or `format!("{:?}", keygen_output)`
- `tracing::debug!(?triple_share, ...)` or `log::debug!("{:?}", ...)` in an integrating application
- `assert_eq!(a, b)` failure messages (Rust prints both sides with `Debug`)
- `unwrap()`/`expect()` on a `Result` that wraps one of these types
- Any error type that `#[derive(Debug)]` and contains a `TripleShare` or `KeygenOutput`

### Impact Explanation
**Critical — Extraction/disclosure of private signing shares and presign nonce material.**

For `TripleShare`: the codebase itself documents that exposing `a`, `b`, `c` allows full private key recovery from any signature produced with that triple. A single debug log line containing a `TripleShare` is sufficient for an attacker with log access to reconstruct the aggregate signing key.

For `KeygenOutput`: `private_share` is the participant's secret DKG share. Leaking it across all participants (e.g., via a shared logging sink) allows reconstruction of the aggregate secret key above the threshold.

For `PresignOutput` / `RerandomizedPresignOutput`: leaking nonce shares `k`/`sigma` (OT-based) or `alpha`/`beta`/`c`/`e` (Robust) enables private key extraction from the corresponding signature.

### Likelihood Explanation
**High.** The `Debug` derive is present in the released library API. Integrating applications routinely use `{:?}` in logs, test output, and error messages. The `PresignArguments` struct for OT-based ECDSA also derives `Debug` and holds a `KeygenOutput` directly, meaning a single debug log of `PresignArguments` leaks the private share. The `ZeroizeOnDrop` present on these types shows the authors are aware of the sensitivity, making the `Debug` derive an oversight rather than an intentional design choice.

### Recommendation
- Remove `Debug` from `TripleShare`, `KeygenOutput`, `PresignOutput`, and `RerandomizedPresignOutput` (all variants).
- If debug formatting is needed for development, implement a custom `fmt::Debug` that redacts all secret scalar fields, e.g., printing `"<redacted>"` for `a`, `b`, `c`, `private_share`, `k`, `sigma`, `alpha`, `beta`, `e`, `c`.
- Apply the same treatment to `PresignArguments` (OT-based), which derives `Debug` and transitively exposes `KeygenOutput` and `TripleShare`.

### Proof of Concept

```rust
// Any caller of the library can trigger this:
use threshold_signatures::ecdsa::ot_based_ecdsa::triples::TripleShare;

let share = TripleShare { a: secret_a, b: secret_b, c: secret_c };

// Path 1: direct debug print
println!("{:?}", share);
// Output: TripleShare { a: <raw scalar bytes>, b: <raw scalar bytes>, c: <raw scalar bytes> }

// Path 2: logging framework (tracing)
tracing::debug!(?share, "triple share after generation");

// Path 3: assert failure in test or integration code
assert_eq!(share, other_share);
// Panic message prints both sides with Debug, emitting all secret scalars

// Path 4: KeygenOutput
use threshold_signatures::KeygenOutput;
let out: KeygenOutput<_> = run_keygen(...);
log::debug!("keygen result: {:?}", out);
// Emits private_share scalar in plaintext to any log sink
```
