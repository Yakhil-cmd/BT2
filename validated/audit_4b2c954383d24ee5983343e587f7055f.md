### Title
Unprivileged Caller Can Suppress All Consensus Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function is callable by any unprivileged address. When called with `disable_rewards: true`, it advances the global `last_reward_block` to the current block without distributing any rewards. Because `last_reward_block` is a protocol-wide gate that allows only one reward update per block, an attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` in every block to permanently suppress all consensus block reward distribution for every staker in the protocol.

### Finding Description
`IStakingRewardsManager.update_rewards` in `src/staking/staking.cairo` has no caller access control. Its only guards are the standard `general_prerequisites()` (unpaused + non-zero caller) and a check that the targeted staker is active with non-zero balance. [1](#0-0) 

The critical sequence inside the function is:

```
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← always written

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits before distributing
}
``` [2](#0-1) 

`last_reward_block` is a **global** storage slot shared across all stakers: [3](#0-2) 

The guard that prevents double-updates reads this same slot:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

Because `last_reward_block` is written unconditionally before the `disable_rewards` branch, any subsequent legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`. The attacker only needs to front-run the first legitimate reward update in each block.

### Impact Explanation
Every block in the consensus-rewards phase produces STRK (and BTC) block rewards for all stakers. By calling `update_rewards(any_active_staker, true)` once per block, an attacker prevents every staker from ever receiving consensus block rewards. This constitutes **griefing with no profit motive but measurable damage to users and the protocol** (continuous loss of block rewards for all stakers). It also qualifies as **temporary — or, if sustained, permanent — freezing of unclaimed yield** because rewards that are never distributed can never be claimed.

### Likelihood Explanation
Medium. The attacker needs only:
1. Knowledge of any one active staker address (trivially obtained from on-chain events such as `NewStaker`).
2. One transaction per block — a low and constant cost on Starknet.

No privileged role, no leaked key, and no third-party compromise is required. Any EOA or contract can execute this attack from the moment consensus rewards are activated (`consensus_rewards_first_epoch` is set).

### Recommendation
Add caller authorization to `update_rewards`. Only the targeted staker address (or their registered `reward_address`) should be permitted to call it, consistent with the pattern used in `claim_rewards` and `increase_stake`. Alternatively, remove the `disable_rewards` parameter entirely if it has no legitimate public use, or gate it behind a privileged role.

### Proof of Concept

1. Governance activates consensus rewards: `set_consensus_rewards_first_epoch(E)`.
2. Attacker observes any active staker address `S` from the `NewStaker` event log.
3. In every block `B ≥ E_start`:
   - Attacker submits `update_rewards(S, disable_rewards: true)`.
   - `last_reward_block` is set to `B`; no rewards are distributed.
4. Any honest call to `update_rewards(S, false)` or `update_rewards(S2, false)` in block `B` reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers accumulate zero block rewards indefinitely.

The attack is permissionless, requires no stake, and costs only the gas of one Starknet transaction per block.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
