### Title
`TurnBridge` Permission Check and State Update Completely Missing — Any Caller Succeeds, Bridge Circuit Breaker Permanently Broken - (File: `x/cronos/keeper/msg_server.go`)

### Summary

`MsgTurnBridge` is the designated emergency circuit breaker for the Cronos bridge. Its `msg_server.go` implementation is a stub that returns `nil, nil` unconditionally — no permission check, no state mutation. Any unprivileged address can call it and receive a success response, while the bridge active/inactive state is never actually changed.

### Finding Description

`UpdateTokenMapping` correctly enforces its permission gate:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [1](#0-0) 

`TurnBridge`, which is supposed to require the symmetric `CanTurnBridge` permission, is instead a no-op stub:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [2](#0-1) 

ADR-009 explicitly designates `MsgTurnBridge` as one of the two restricted messages requiring `CanTurnBridge` permission, and the permission constant is defined alongside `CanChangeTokenMapping`: [3](#0-2) 

The proto definition confirms `MsgTurnBridge` is a live, routed message with a `sender` signer field: [4](#0-3) 

### Impact Explanation

Two distinct impacts:

1. **Auth bypass (High):** Any unprivileged address can submit `MsgTurnBridge` and receive a success response. The `CanTurnBridge` permission gate that ADR-009 mandates is entirely absent. This directly matches: *"High: Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."*

2. **Permanent inability to disable the bridge (High):** Because the function never mutates `bridge_active` in params, the bridge circuit breaker is permanently broken. Even the legitimate admin or a permissioned address cannot disable the bridge during an ongoing exploit or emergency. This matches: *"High: Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows..."*

### Likelihood Explanation

The entry path is a standard Cosmos SDK `MsgTurnBridge` transaction, reachable by any unprivileged address with no preconditions. The stub is in production keeper code, not a test or mock.

### Recommendation

Implement `TurnBridge` analogously to `UpdateTokenMapping`:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    params := k.GetParams(ctx)
    params.EnableAutoDeployment = msg.Enable // or the correct bridge_active field
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

The exact param field to toggle should match the `bridge_active` parameter referenced in ADR-001.

### Proof of Concept

1. Any address (no keys, no permissions) submits `MsgTurnBridge{sender: attacker, enable: false}`.
2. The message is routed to `TurnBridge` in `msg_server.go`.
3. The function returns `nil, nil` — tx succeeds with code 0.
4. No permission check fires; no state is written.
5. The bridge remains active regardless of the `enable` field.
6. A legitimate admin subsequently submits the same message to disable the bridge during an emergency — it also silently succeeds and does nothing, leaving the bridge permanently active. [5](#0-4) [6](#0-5) [7](#0-6)

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
