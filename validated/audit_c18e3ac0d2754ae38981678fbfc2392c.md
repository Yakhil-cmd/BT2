### Title
Missing Validation of New Participant's Signing Key in Reshare Initialization Causes Permanent Denial of Reshare for Honest Parties - (File: src/dkg.rs)

---

### Summary

`assert_reshare_keys_invariants` in `src/dkg.rs` validates that an old participant must supply a signing key, but contains no symmetric check to reject a **new** participant (one absent from `old_participants`) who supplies a non-`None` signing key. The developer's own comment at line 661 describes this missing guard. Without it, the invalid state is not caught at initialization time; instead the protocol future is handed to the caller, and the failure surfaces only inside `do_keyshare` — after all honest parties have already entered the broadcast wait-loop. Because the library is a pure state machine with no built-in timeouts, those honest parties hang indefinitely, permanently denying the reshare session.

---

### Finding Description

`assert_reshare_keys_invariants` enforces only one direction of the signing-key invariant:

```rust
// Step 1.1
// if me is not in the old participant set then ensure that old_signing_key is None
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
``` [1](#0-0) 

The comment at line 661 explicitly states the symmetric guard that is missing: *"if me is not in the old participant set then ensure that `old_signing_key` is None"*. The code never implements that guard. [2](#0-1) 

When a new participant (not in `old_participants`) calls `reshare()` with `old_signing_key = Some(x_i)`, `assert_reshare_keys_invariants` returns `Ok`, and `reshare()` returns `Ok(protocol)` to the caller. [3](#0-2) 

Inside `do_reshare`, the secret is computed by calling `intersection.lagrange::<C>(me)` where `me` is absent from `intersection` (because `me ∉ old_participants`). This produces a mathematically arbitrary non-zero scalar, so `secret` is non-zero. [4](#0-3) 

`do_keyshare` is then called with this non-zero `secret` and `old_reshare_package = Some(...)`. The very first thing `do_keyshare` does is call `assert_keyshare_inputs`, which detects the contradiction and returns a `ProtocolError`:

```rust
} else {
    //  return error if me is part of the old participants set
    if !old_participants.contains(me) {
        return Err(ProtocolError::AssertionFailed(
            format!("{me:?} is running Resharing with a non-zero share but does not belong to the old participant set")));
    }
}
``` [5](#0-4) 

This error is returned **before** any channel message is sent. The first network operation in `do_keyshare` is the session-ID broadcast:

```rust
let session_ids = do_broadcast(&mut chan, &participants, me, my_session_id).await?;
``` [6](#0-5) 

Because `me` aborts before this broadcast, all honest participants block indefinitely waiting for `me`'s broadcast message. The library is a pure state machine with no built-in timeouts, so there is no recovery path within the protocol itself. [7](#0-6) 

---

### Impact Explanation

A malicious new participant who deliberately passes `old_signing_key = Some(arbitrary_share)` to `reshare()` will cause every honest participant in the reshare session to block permanently on the first broadcast wait-point. Because the existing key material is unaffected, signing with the old keys remains possible, but the reshare (and any dependent refresh or re-keying) is permanently denied for this session. This maps to:

> **High: Permanent denial of reshare for honest parties under valid protocol inputs and documented trust assumptions.**

---

### Likelihood Explanation

Any participant designated as a *new joiner* in a reshare has the ability to call `reshare()` with a non-`None` signing key. The initialization guard does not reject this. The attacker needs no special privilege beyond being included in the new participant set. A single malicious new participant is sufficient to stall all honest parties.

---

### Recommendation

Add the symmetric guard immediately after the existing check in `assert_reshare_keys_invariants`:

```rust
// if me is not in the old participant set then ensure that old_signing_key is None
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not present in the old participant list but provided a share"
    )));
}
``` [8](#0-7) 

This mirrors the existing check and catches the invalid state at initialization time, before the protocol future is created and before honest parties begin waiting for messages.

---

### Proof of Concept

1. Run a standard keygen with participants `[P1, P2, P3]`, threshold 2.
2. Initiate a reshare to `[P1, P2, P3, P4]` (P4 is new).
3. P4 calls `reshare()` with `old_signing_key = Some(SigningShare::new(random_scalar))` instead of `None`.
4. `assert_reshare_keys_invariants` passes — no error is returned.
5. `reshare()` returns `Ok(protocol)`.
6. When the protocol is driven, `do_reshare` computes a non-zero `secret` for P4 (via a spurious Lagrange evaluation), then `do_keyshare` calls `assert_keyshare_inputs`, which returns `ProtocolError::AssertionFailed` before any message is sent.
7. P1, P2, P3 are blocked indefinitely in `do_broadcast` waiting for P4's session-ID message that never arrives. [8](#0-7) [9](#0-8)

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
}

pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],
) -> Result<(ParticipantList, ParticipantList), InitializationError> {
    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

    let participants = assert_key_invariants(participants, me, threshold)?;

    let old_participants =
        ParticipantList::new(old_participants).ok_or(InitializationError::DuplicateParticipants)?;

    // Step 1.1
    if old_participants.intersection(&participants).len() < old_threshold {
        return Err(InitializationError::NotEnoughParticipantsForNewThreshold {
            threshold: old_threshold,
            participants: old_participants.intersection(&participants).len(),
        });
    }
    // Step 1.1
    // if me is not in the old participant set then ensure that old_signing_key is None
    if old_participants.contains(me) && old_signing_key.is_none() {
        return Err(InitializationError::BadParameters(format!(
            "party {me:?} is present in the old participant list but provided no share"
        )));
    }
    Ok((participants, old_participants))
}
