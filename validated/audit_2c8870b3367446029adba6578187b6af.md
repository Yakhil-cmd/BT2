### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged caller can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` checkpoint without distributing any rewards. Because the per-block guard is global, a single such call per block permanently destroys that block's yield for every staker in the protocol.

---

### Finding Description

The specification for `update_rewards` states:

> **Access control:** Only starkware sequencer.

The implementation, however, only calls `general_prerequisites()`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: unpaused + caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← global state written

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits with no rewards
    }
    ...
```

`general_prerequisites` only asserts the contract is unpaused and the caller is non-zero — there is no sequencer role check:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

`last_reward_block` is a single global storage slot, not per-staker:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
```

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at block N:

1. `current_block_number (N) > last_reward_block` — passes.
2. `last_reward_block` is written to N.
3. Function returns early — **zero rewards distributed**.

When the legitimate sequencer subsequently calls `update_rewards` at the same block N, the assertion `current_block_number > last_reward_block` evaluates to `N > N` → **false**, reverting with `REWARDS_ALREADY_UPDATED`. The rewards for block N are permanently destroyed; they are not carried forward because `last_reward_block` was already advanced.

---

### Impact Explanation

**Permanent freezing of unclaimed yield (High).**

Because `last_reward_block` is global, one attacker call per block is sufficient to prevent every staker from receiving rewards for that block. If repeated every block, no staker ever accumulates yield. Even sporadic attacks permanently destroy rewards for the targeted blocks — unlike a genuine missed block (where the reward supplier compensates on the next call), a `disable_rewards: true` call advances the checkpoint so the compensation never occurs.

---

### Likelihood Explanation

**High.** The entry point is fully public — any non-zero address can call it. The attacker only needs to supply any currently-active staker address (obtainable from on-chain events or `get_stakers`). The cost is a single transaction per block. No privileged key, bridge access, or external dependency is required.

---

### Recommendation

Add an access-control guard at the top of `update_rewards` that restricts the caller to the authorized sequencer role, consistent with every other privileged function in the contract (e.g., `self.roles.only_sequencer()`). Alternatively, store the sequencer address and assert `get_caller_address() == sequencer_address`.

---

### Proof of Concept

```
Block N:
  Attacker → update_rewards(staker_A, disable_rewards: true)
    ✓ current_block_number (N) > last_reward_block (N-1)
    → last_reward_block := N
    → returns early, 0 rewards distributed

  Sequencer → update_rewards(staker_A, disable_rewards: false)
    ✗ current_block_number (N) > last_reward_block (N)  →  REWARDS_ALREADY_UPDATED
    → reverts, staker_A earns nothing for block N

Block N+1:
  Attacker repeats → last_reward_block := N+1, 0 rewards
  Sequencer reverts again

... (every block) → stakers permanently receive zero yield

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
