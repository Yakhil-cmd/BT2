### Title
Gateway L2 gas price admission check uses current block's `l2_gas_price` instead of `next_l2_gas_price`, admitting transactions that fail at blockifier execution - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful L2 gas price threshold check reads `l2_gas_price` (the price of the **current** committed block) from `GatewayFixedBlockStateReader::get_block_info()` to validate incoming transactions. However, transactions are executed in the **next** block, whose L2 gas price is `next_l2_gas_price` (the EIP-1559 adjusted price stored separately in `StorageBlockHeader`). When the network is busy and `next_l2_gas_price > l2_gas_price`, the gateway admits transactions whose `max_price_per_unit` satisfies the gateway threshold but falls below `next_l2_gas_price`, causing them to fail at blockifier pre-validation with `MaxGasPriceTooLow`. A developer TODO comment in the code explicitly acknowledges this wrong field is being read.

### Finding Description

In `stateful_transaction_validator.rs`, `validate_resource_bounds` reads the L2 gas price reference for admission as:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client()` populates `BlockInfo.gas_prices.strk_gas_prices.l2_gas_price` from `block_header.l2_gas_price.price_in_fri` — the **current** block's L2 gas price:

```rust
strk_gas_prices: GasPriceVector {
    ...
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
},
``` [2](#0-1) 

`StorageBlockHeader` stores a distinct `next_l2_gas_price` field — the EIP-1559 computed price for the **next** block — which is never surfaced through `BlockInfo`: [3](#0-2) 

The blockifier's `check_fee_bounds`, executed during block building, checks the transaction's `max_price_per_unit` against the **new block's** gas price (which equals `next_l2_gas_price` from the previous block header):

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [4](#0-3) 

The gateway threshold check is:

```
tx.max_price_per_unit >= l2_gas_price * min_gas_price_percentage / 100
```

The blockifier check is:

```
tx.max_price_per_unit >= next_l2_gas_price
```

Under EIP-1559, when a block is full (`gas_used > gas_target`), `next_l2_gas_price` increases by up to `price / gas_price_max_change_denominator` per block: [5](#0-4) 

Any transaction with `l2_gas_price * min_gas_price_percentage/100 ≤ max_price_per_unit < next_l2_gas_price` passes gateway admission but fails blockifier pre-validation.

### Impact Explanation

When `validate_resource_bounds = true` and `min_gas_price_percentage > 0`, the gateway admits transactions that will be rejected by the blockifier with `TransactionPreValidationError::TransactionFeeError(InsufficientResourceBounds { MaxGasPriceTooLow })`. These transactions occupy mempool slots, consume gateway validation resources, and are handed to the batcher only to fail immediately. Users receive a gateway-accepted response but their transactions never execute. This matches **High: Mempool/gateway/RPC admission accepts invalid transactions before sequencing**.

### Likelihood Explanation

The condition requires: (1) `validate_resource_bounds = true` in the gateway config, (2) `min_gas_price_percentage > 0`, and (3) the network is busy enough that `next_l2_gas_price > l2_gas_price`. Condition (3) is normal during any period of sustained high throughput. The developer TODO comment confirms the wrong field is being used and a fix is planned, indicating this is a known production gap rather than an intentional design.

### Recommendation

In `validate_resource_bounds`, read `next_l2_gas_price` from the block header instead of `l2_gas_price`. This requires either:

1. Extending `BlockInfo` with a `next_l2_gas_price` field and populating it in `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client()` from `block_header.next_l2_gas_price`, or
2. Adding a dedicated `get_next_l2_gas_price()` method to `GatewayFixedBlockStateReader` that reads `block_header.next_l2_gas_price` directly.

The threshold check should then be:

```rust
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()
    .await?;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    next_block_l2_gas_price,
)?;
```

### Proof of Concept

1. Observe the latest committed block has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block was full).
2. Submit an `InvokeV3` transaction with `AllResources` bounds where `l2_gas.max_price_per_unit = P * min_gas_price_percentage / 100` (just at the gateway threshold).
3. Gateway calls `validate_resource_bounds`: `P * pct/100 >= P * pct/100` → **passes**.
4. Transaction enters mempool and is handed to the batcher.
5. Batcher builds the next block with `l2_gas_price = P'`.
6. Blockifier `check_fee_bounds`: `P * pct/100 < P'` → **fails** with `MaxGasPriceTooLow`.
7. Transaction is rejected at execution time despite having been admitted by the gateway. [6](#0-5) [7](#0-6) [8](#0-7)

    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
