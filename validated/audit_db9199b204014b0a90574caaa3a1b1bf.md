### Title
Pool admin bypasses `maxAdminSpreadFeeE6` cap via uncapped `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary
`MetricOmmPoolFactory.setPoolBinAdditionalFees` forwards `addFeeBuyE6`/`addFeeSellE6` to the pool with no validation against `maxAdminSpreadFeeE6`, allowing a pool admin to write per-bin spread fees up to `type(uint16).max` (65 535 = 6.5535%) regardless of the factory-owner-imposed admin fee ceiling.

### Finding Description
The factory enforces `maxAdminSpreadFeeE6` in `setPoolAdminFees`:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
```

But the sibling function `setPoolBinAdditionalFees` (lines 450-457) performs no such check — it calls `setBinAdditionalFees` on the pool directly. The pool-level `setBinAdditionalFees` (lines 464-474) only validates the bin index, not the fee magnitude. The stored `addFeeBuyE6`/`addFeeSellE6` values are then added unconditionally to `baseFeeX64` in all four swap paths (lines 910, 999, 1088, 1177).

### Impact Explanation
Traders swapping through a bin whose `addFeeBuyE6` or `addFeeSellE6` has been set to `type(uint16).max` pay up to 6.5535% additional spread on top of the oracle spread and the global `spreadFeeE6`. If the factory owner has set `maxAdminSpreadFeeE6` to, say, 50 000 (5%), the pool admin can still impose 6.5535% per-bin, making the effective admin-controlled fee 11.5535% — more than double the documented ceiling. Traders suffer direct principal loss on every swap through the affected bin.

### Likelihood Explanation
Any pool admin can call `setPoolBinAdditionalFees` at any time on an active pool (pause level 0). No timelock, no two-step proposal, no factory-owner approval is required. The factory owner's cap is the only intended guard, and it is simply absent from this code path.

### Recommendation
Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (uint256(addFeeBuyE6) > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (uint256(addFeeSellE6) > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

### Proof of Concept
1. Factory owner calls `setFeeCaps(..., maxAdminSpreadFeeE6 = 50_000, ...)`.
2. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` — succeeds with no revert.
3. `_binStates[0].addFeeBuyE6 == 65535`, `addFeeSellE6 == 65535`.
4. A trader swaps through bin 0; the effective spread fee is `baseFeeX64 + 65535/1e6`, which exceeds the 5% admin cap.
5. Assert: trader's input cost exceeds what `maxAdminSpreadFeeE6 = 50_000` would permit.
