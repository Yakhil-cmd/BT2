### Title
Missing Caller Authorization in `exit_delegation_pool_action` Allows Anyone to Force a Pool Member's Exit — (File: `src/pool/pool.cairo`)

---

### Summary

`Pool.exit_delegation_pool_action` accepts an arbitrary `pool_member` address as a parameter but never verifies that the caller is that pool member. Any unprivileged address can call this function for any pool member who has a pending exit intent, forcibly completing their withdrawal and destroying their ability to switch delegation pools.

---

### Finding Description

`exit_delegation_pool_action` is the second step of the two-step undelegation flow. A pool member first calls `exit_delegation_pool_intent` (which uses `get_caller_address()` as the pool member identity), then later calls `exit_delegation_pool_action` to receive their funds.

The intent step correctly derives the pool member from the caller:

```cairo
fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
    let pool_member = get_caller_address();   // ← caller-derived, safe
    ...
}
``` [1](#0-0) 

The action step, however, accepts `pool_member` as a caller-supplied parameter and performs **no authorization check**:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    // ← no assert that get_caller_address() == pool_member
    ...
    token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
``` [2](#0-1) 

Compare this with `add_to_delegation_pool` and `claim_rewards`, which both explicitly guard against unauthorized callers:

```cairo
assert!(
    caller_address == pool_member || caller_address == pool_member_info.reward_address,
    "{}", Error::CALLER_CANNOT_ADD_TO_POOL,
);
``` [3](#0-2) 

```cairo
assert!(
    caller_address == pool_member || caller_address == reward_address,
    "{}", Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
);
``` [4](#0-3) 

The interface documentation for `exit_delegation_pool_action` lists no access-control restriction, confirming the omission is not intentional: [5](#0-4)

### Impact Explanation

When an attacker calls `exit_delegation_pool_action(pool_member: victim)` after the exit window has elapsed:

1. `pool_member_info.unpool_amount` and `unpool_time` are cleared in storage.
2. `unpool_amount` tokens are transferred to `pool_member` (the victim's address — no direct theft).
3. The victim's pending `switch_delegation_pool` opportunity is permanently destroyed: `switch_delegation_pool` requires `pool_member_info.unpool_time.is_some()` and `unpool_amount >= amount`.

```cairo
assert!(
    pool_member_info.unpool_time.is_some(),
    "{}", GenericError::MISSING_UNDELEGATE_INTENT,
);
assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
``` [6](#0-5) 

The victim must now re-enter the delegation pool from scratch. Because staking power is credited at `epoch + K` (where K > 0), the victim misses at least K epochs of delegation rewards — a concrete, quantifiable financial loss. [7](#0-6) 

**Impact class**: Griefing with no profit motive but measurable financial damage to users (lost delegation rewards over K epochs). Matches the allowed impact: *"Griefing with no profit motive but damage to users or protocol."*

---

### Likelihood Explanation

- The entry point is fully public; no role, key, or privileged access is required.
- The only precondition is that the victim has a pending exit intent and the exit window has elapsed — both are observable on-chain.
- A griefing attacker (e.g., a competing staker who wants to prevent delegators from switching to a rival pool) has a clear motive.
- The attack is a single transaction with negligible gas cost.

---

### Recommendation

Add a caller check at the top of `exit_delegation_pool_action`, consistent with the pattern used in `add_to_delegation_pool` and `claim_rewards`:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let caller_address = get_caller_address();
    assert!(
        caller_address == pool_member,
        "{}", Error::CALLER_CANNOT_EXIT_POOL,
    );
    ...
}
```

Alternatively, derive `pool_member` from `get_caller_address()` directly (as `exit_delegation_pool_intent` does), removing the parameter entirely.

---

### Proof of Concept

1. Alice calls `exit_delegation_pool_intent(amount: X)` on Pool P. Her `unpool_time` is set to `T`.
2. Time advances past `T`.
3. Alice intends to call `switch_delegation_pool(to_staker: S2, to_pool: P2, amount: X)` to move to a better staker.
4. Bob (attacker) front-runs Alice and calls `exit_delegation_pool_action(pool_member: Alice)`.
5. The call succeeds: Alice's `unpool_amount` is cleared, `X` tokens are sent to Alice's address.
6. Alice's subsequent `switch_delegation_pool` call reverts with `MISSING_UNDELEGATE_INTENT`.
7. Alice must re-enter Pool P2 via `enter_delegation_pool`, waiting K epochs before her stake earns rewards — losing K epochs of delegation yield.

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
```
