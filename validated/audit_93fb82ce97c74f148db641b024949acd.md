### Title
Unauthorized Transfer of `evm/<contract>` Native Tokens via Missing Caller Authentication in Bank Precompile — (File: `x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The `transfer` (and `burn`) method in the bank precompile's `BankContract.Run` accepts an arbitrary `sender` address from ABI-decoded input without verifying it equals `contract.Caller()`. Any unprivileged EVM contract can move `evm/<itsAddress>` native tokens from **any holder** to any recipient without the holder's consent.

---

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` case decodes `sender` directly from the caller-supplied ABI input and passes it as the `from` address to `bankKeeper.SendCoins`:

```go
sender := args[0].(common.Address)   // ← fully attacker-controlled
...
from := sdk.AccAddress(sender.Bytes())
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

There is **no check** that `sender == contract.Caller()`. The denom is derived from the calling contract's address (`EVMDenom(contract.Caller())` → `"evm/<callerAddr>"`), so the calling contract controls the token namespace, but it can freely drain any address that holds those tokens. [2](#0-1) 

The `burn` case has the same flaw: the first argument (`recipient`, confusingly named) is the address to burn **from**, and it is also never validated against `contract.Caller()`:

```go
recipient := args[0].(common.Address)   // ← attacker-controlled burn target
...
bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, ...)
bc.bankKeeper.BurnCoins(ctx, types.ModuleName, ...)
``` [3](#0-2) 

The intended usage pattern (shown in the integration test contract) is for the calling contract to pass `msg.sender` as the `sender` argument:

```solidity
return bank.transfer(msg.sender, recipient, amount);
``` [4](#0-3) 

Without enforcement, a malicious contract can substitute any victim address for `msg.sender`.

---

### Impact Explanation

**Critical** — Unauthorized transfer of precompile-controlled assets (`evm/<contract>` native tokens).

Any address that holds `evm/<MaliciousContract>` tokens (e.g., a DeFi protocol that accepted them as collateral, or a user who received them as payment) can have those tokens drained to the attacker's address in a single call to the bank precompile, with no approval or signature from the victim. The `burn` variant allows the same contract to destroy a victim's balance. Both operations bypass the holder's consent entirely.

---

### Likelihood Explanation

**Medium** — The victim must hold `evm/<attackerContract>` tokens. This occurs whenever a user or protocol interacts with the attacker's contract and receives those tokens (e.g., liquidity provision, airdrop, collateral deposit). Once any value is locked in `evm/<attackerContract>` denomination, the attacker can drain it atomically.

---

### Recommendation

Add a caller-authentication guard in the `TransferMethodName` case (and symmetrically in the `BurnMethodName` case):

```go
// transfer: sender must be the calling contract itself
if sender != contract.Caller() {
    return nil, errors.New("bank precompile: sender must equal calling contract")
}
``` [5](#0-4) 

For `burn`, the address-to-burn-from should similarly be restricted to `contract.Caller()` unless an explicit approval mechanism is introduced.

---

### Proof of Concept

1. Attacker deploys `MaliciousContract` (any EVM contract that calls the bank precompile at `0x0000…0064`).
2. `MaliciousContract.mint(victimAddress, 1000)` → mints 1000 `evm/MaliciousContract` tokens to `victimAddress` (e.g., as a DeFi deposit receipt).
3. `victimAddress` now holds 1000 `evm/MaliciousContract` native tokens.
4. Attacker calls `MaliciousContract.steal(victimAddress, attackerAddress, 1000)`, which internally calls:
   ```solidity
   bank.transfer(victimAddress, attackerAddress, 1000);
   ```
5. The bank precompile executes `SendCoins(victimAddress → attackerAddress, 1000 evm/MaliciousContract)` with no consent from `victimAddress`.
6. `victimAddress` loses all 1000 tokens; attacker gains them.

The root cause — `sender` taken from attacker-controlled ABI input with no `sender == contract.Caller()` guard — maps directly to the external report's vulnerability class: a missing safety/authorization check during a token operation that allows assets to be moved without the rightful owner's consent. [6](#0-5)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
