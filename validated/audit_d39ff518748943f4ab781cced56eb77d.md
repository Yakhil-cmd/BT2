### Title
Missing Zero-Address Validation in `change_reward_address` Permanently Freezes Unclaimed Yield — (`src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

Both `change_reward_address` functions in `staking.cairo` and `pool.cairo` only guard against setting the reward address to a known token address. They do **not** reject `address(0)`. A staker or pool member who accidentally (or intentionally) sets their reward address to zero will have all future reward transfers directed to the zero address, permanently destroying unclaimed yield with no on-chain recovery path.

---

### Finding Description

**Root cause — `staking.cairo` `change_reward_address`:**

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    assert!(
        !self.does_token_exist(token_address: reward_address),
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    // ← no assert!(reward_address.is_non_zero(), ...)
    staker_info.reward_address = reward_address;
    self.write_staker_info(:staker_address, :staker_info);
    ...
}
``` [1](#0-0) 

**Root cause — `pool.cairo` `change_reward_address`:**

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    assert!(
        self.token_dispatcher.contract_address.read() != reward_address,
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    // ← no assert!(reward_address.is_non_zero(), ...)
    pool_member_info.reward_address = reward_address;
    self.write_pool_member_info(:pool_member, :pool_member_info);
    ...
}
``` [2](#0-1) 

The same gap exists at initial entry points. `stake` in `staking.cairo` only checks `!self.does_token_exist(token_address: reward_address)`: [3](#0-2) 

And `enter_delegation_pool` in `pool.cairo` only checks `token_address != reward_address`: [4](#0-3) 

Once `reward_address` is zero, every downstream reward transfer targets `address(0)`:

- **Staker path** — `send_rewards_to_staker` → `token_dispatcher.checked_transfer(recipient: reward_address, ...)`: [5](#0-4) 

- **Pool member path** — `claim_rewards` → `reward_token.checked_transfer(recipient: reward_address, ...)`: [6](#0-5) 

- **Unstake path** — `unstake_action` calls `send_rewards_to_staker` before returning principal, so rewards are dispatched to `address(0)` automatically when the staker exits: [7](#0-6) 

---

### Impact Explanation

**Impact: High — Permanent freezing of unclaimed yield.**

If the underlying STRK ERC-20 permits transfers to `address(0)` (burn semantics), all accrued rewards are irreversibly destroyed the moment `claim_rewards` or `unstake_action` executes. Even if the token reverts on zero-address transfers, `unstake_action` becomes permanently bricked for that staker until they notice and call `change_reward_address` again — constituting **temporary freezing of funds** (also High per the allowed scope). In either case there is no protocol-level rescue function analogous to what the external report recommends.

---

### Likelihood Explanation

**Likelihood: Low-Medium.**

- Any staker or pool member can call `change_reward_address` permissionlessly at any time.
- A single fat-finger (passing `0` instead of a valid address) is sufficient to trigger the condition.
- Because `unstake_action` is callable by **any address** (spec: "Any address can execute"), an observer who notices a staker has set `reward_address = 0` and has an active `unstake_intent` can immediately call `unstake_action` to force the reward transfer to `address(0)` before the staker can correct the mistake. [8](#0-7) 

---

### Recommendation

1. Add a non-zero guard in both `change_reward_address` implementations and in `stake` / `enter_delegation_pool`:
   ```cairo
   assert!(reward_address.is_non_zero(), "Reward address cannot be zero");
   ```
2. Consider adding a protocol-level `rescue_rewards` function (analogous to the external report's recommendation) that allows recovery when the reward address is demonstrably uncontrolled.

---

### Proof of Concept

```
1. Staker calls staking.stake(reward_address=0, ...) — succeeds (no zero check).
   OR staker calls staking.change_reward_address(reward_address=0) — succeeds.

2. Several epochs pass; staker accumulates unclaimed_rewards_own > 0.

3. Anyone calls staking.unstake_action(staker_address) after the exit window.
   → send_rewards_to_staker fires:
       token.checked_transfer(recipient: 0x0, amount: rewards)
   → If token allows: rewards are burned permanently.
   → If token reverts: unstake_action reverts, principal is frozen.

4. Pool member path is identical via pool.change_reward_address(0)
   followed by pool.claim_rewards(pool_member).
``` [9](#0-8) [10](#0-9)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

            // Asserts.

                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,

            // Calculate rewards and update entry_to_claim_from.

                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
