### Title
Unrestricted `update_rewards` with `disable_rewards=true` Enables Permanent Griefing of All Stakers' Block Rewards — (File: src/staking/staking.cairo)

---

### Summary
Any unprivileged caller can invoke `update_rewards` with `disable_rewards=true`, which advances the **global** `last_reward_block` counter without distributing any rewards. Because `last_reward_block` is a single contract-wide value, one such call per block permanently prevents every staker from receiving block rewards for that block.

---

### Finding Description
`update_rewards` is a public function gated only by `general_prerequisites` (contract not paused + caller non-zero). It accepts an arbitrary `staker_address` and a boolean `disable_rewards`. The critical execution path is:

1. Assert `current_block_number > last_reward_block` — prevents double-call in the same block.
2. Validate the supplied staker exists, is active, and has non-zero balance.
3. **Unconditionally write `last_reward_block = current_block_number`.**
4. If `disable_rewards || is_pre_consensus()` → **return immediately** without distributing rewards.

Step 3 occurs *before* the `disable_rewards` guard in step 4. Any caller therefore consumes the "reward slot" for the current block without paying out any rewards. Any legitimate call to `update_rewards` in the same block then reverts at step 1 with `REWARDS_ALREADY_UPDATED`. [1](#0-0)

---

### Impact Explanation
An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` once per block causes every staker to permanently lose their block rewards for that block. Rewards are not deferred or recoverable — they are simply never credited to `unclaimed_rewards_own` and never forwarded to delegation pools. Sustained over many blocks this suppresses all consensus-era block rewards across the entire protocol.

This matches: **Griefing with no profit motive but damage to users or protocol (Medium)**, and at sustained cadence approaches **permanent freezing of unclaimed yield (High)**.

---

### Likelihood Explanation
The function is fully public with no role check beyond a non-zero caller. The attacker pays only the gas cost of one transaction per block. No privileged access, leaked key, bridge compromise, or external dependency is required. Any address can execute this attack continuously and indefinitely.

---

### Recommendation
1. **Add access control**: restrict `update_rewards` (or at minimum the `disable_rewards=true` path) to an authorized caller — e.g., the staker themselves, a designated relayer role, or a governance-controlled address.
2. **Decouple the slot-consumption from the no-reward path**: only write `last_reward_block` when rewards are actually distributed, or use a per-staker `last_reward_block` so one call cannot block all others.

---

### Proof of Concept

```
Block N:
  Attacker calls: update_rewards(active_staker_address, disable_rewards=true)
    → general_prerequisites: passes (not paused, caller ≠ 0)
    → current_block_number (N) > last_reward_block: passes
    → staker validity checks: passes (any active staker with non-zero balance)
    → last_reward_block := N          ← slot consumed
    → disable_rewards == true → return (no rewards distributed)

  Legitimate call: update_rewards(staker, disable_rewards=false)
    → current_block_number (N) > last_reward_block (N): FAILS → REWARDS_ALREADY_UPDATED

Block N+1, N+2, … : attacker repeats → all block rewards suppressed indefinitely
``` [4](#0-3) [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
