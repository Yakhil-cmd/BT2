### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Suppress Block Rewards for Any Staker - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the staking contract is callable by any unprivileged address despite the protocol specification requiring "Only starkware sequencer." An attacker can invoke it with `disable_rewards: true` to permanently consume the per-block reward slot for any staker without distributing rewards, causing irreversible loss of block rewards.

---

### Finding Description

The protocol specification explicitly restricts `update_rewards` to the Starknet sequencer:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` contains no caller check whatsoever:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks paused state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    ...
``` [2](#0-1) 

The critical sequence is:

1. `last_reward_block` is **unconditionally written** to `current_block_number` before the `disable_rewards` branch is evaluated.
2. If `disable_rewards == true`, the function returns immediately — no rewards are calculated or distributed.
3. The guard `current_block_number > self.last_reward_block.read()` then prevents any subsequent call in the same block, including the legitimate sequencer call. [3](#0-2) 

The `general_prerequisites` helper only checks the paused flag and does not validate the caller: [4](#0-3) 

The `IStakingRewardsManager` interface exposes this function publicly with no documented restriction in the interface itself: [5](#0-4) 

---

### Impact Explanation

**Medium — Griefing with no profit motive but damage to users or protocol.**

An attacker can call `update_rewards(victim_staker_address, disable_rewards: true)` on every block. Each such call:

- Permanently marks that block as "rewards already updated" for the entire staking contract (the `last_reward_block` is a single global slot, not per-staker).
- Prevents the sequencer from distributing block rewards for that block to any staker.
- The rewards for that block are never calculated or credited — they are permanently lost, not merely delayed.

Sustained griefing across many blocks causes cumulative, irreversible loss of consensus block rewards for all stakers. [3](#0-2) 

---

### Likelihood Explanation

**High.** The function is permissionlessly callable by any EOA or contract on Starknet. No stake, role, or special privilege is required. The only cost to the attacker is gas per call. A bot can automate this on every block with negligible overhead.

---

### Recommendation

Add an explicit caller check at the top of `update_rewards` to enforce the spec's "Only starkware sequencer" requirement. The sequencer address should be stored in contract storage (set during initialization or via a privileged setter) and validated:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    self.general_prerequisites();
    ...
}
```

Alternatively, if the sequencer address is not known at deploy time, restrict the call to a designated `rewards_manager` role that is set by governance.

---

### Proof of Concept

```
Block N:
  Attacker calls: staking.update_rewards(victim_staker, disable_rewards=true)
    → last_reward_block is written to N
    → no rewards distributed (early return)

  Sequencer calls: staking.update_rewards(victim_staker, disable_rewards=false)
    → REVERTS with REWARDS_ALREADY_UPDATED (current_block_number == last_reward_block)

Block N+1:
  Attacker repeats the same call.
  Sequencer is again blocked.

Result: Victim staker earns zero block rewards indefinitely.
        All block rewards for the affected blocks are permanently lost.
``` [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
