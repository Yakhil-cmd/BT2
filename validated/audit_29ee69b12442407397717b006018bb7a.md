### Title
Unbounded `stakers` Vec Growth Allows Griefing of `get_stakers()` — (File: `src/staking/staking.cairo`)

---

### Summary

Any unprivileged user can permanently inflate the `stakers` storage vector by staking and unstaking with multiple addresses. Because stakers are **never removed** from this vector, the `get_stakers()` function must iterate over an ever-growing list on every call, eventually causing unbounded resource consumption and threatening the consensus layer's ability to read the validator set.

---

### Finding Description

Every call to `stake()` appends the caller's address to the `stakers` Vec: [1](#0-0) 

The storage declaration carries an explicit developer note that stakers are **never** removed from this vector: [2](#0-1) 

`get_stakers()` iterates over the **full range** of this vector on every invocation: [3](#0-2) 

For each entry it reads `staker_info` from storage to call `is_staker_active()`, then reads the balance traces and key maps for active stakers. Inactive (unstaked) entries are skipped only after a storage read per entry.

An attacker's flow:
1. Create N fresh addresses, each funded with `min_stake` STRK.
2. Each address calls `stake()` → appended to `stakers` Vec.
3. Each address calls `unstake_intent()` → waits `exit_wait_window` (≥ 1 week).
4. Each address calls `unstake_action()` → STRK is **fully returned**; staker info is erased from `staker_info` map but the address **remains** in `stakers` Vec.
5. Repeat with new addresses.

The only cost to the attacker is gas per cycle; the principal is recovered. The `stakers` Vec grows monotonically with no bound.

---

### Impact Explanation

`get_stakers()` is the function the Starknet consensus layer calls to determine the active validator set for a given epoch. As the `stakers` Vec grows, every call must perform O(N) storage reads across all historical staker addresses (active and inactive). At sufficient scale this causes:

- **Unbounded resource consumption** on every `get_stakers()` call.
- Potential inability of the consensus layer to retrieve the validator set within its resource budget, disrupting liveness of the Starknet network.

This maps to the allowed impact: **Griefing with no profit motive but damage to users or protocol; Unbounded gas consumption**.

---

### Likelihood Explanation

Medium. The attacker recovers all staked STRK after each cycle; the only recurring cost is gas. The `min_stake` threshold slows the attack but does not prevent it — a well-funded attacker can run many cycles in parallel across many addresses. The attack is permissionless and requires no privileged access.

---

### Recommendation

1. **Track active stakers separately**: Maintain a count or a separate iterable set of *currently active* staker addresses, and remove entries on `unstake_action()`. `get_stakers()` should iterate only over active stakers.
2. **Pagination**: Add a paginated variant of `get_stakers()` so the consensus layer can retrieve the set in bounded chunks.
3. **Tombstone cleanup**: At minimum, compact the `stakers` Vec periodically by removing zero-info entries, or replace the `Vec` with a data structure that supports O(1) deletion.

---

            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
