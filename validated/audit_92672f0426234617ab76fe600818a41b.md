### Title
Lack of Incentivization for `update_rewards` with Global `last_reward_block` Enables Griefing Attack That Permanently Freezes All Stakers' Unclaimed Yield - (File: src/staking/staking.cairo)

### Summary
The `IStakingRewardsManager::update_rewards` function is a public, unincentivized function that enforces a **global** single-call-per-block constraint via `last_reward_block`. Any unprivileged caller can invoke it with `disable_rewards = true` every block, consuming the sole allowed slot without distributing any rewards, permanently denying all stakers their consensus block rewards at the cost of only gas.

### Finding Description
`update_rewards` in `src/staking/staking.cairo` is the sole mechanism for distributing consensus block rewards to stakers. It is gated only by `general_prerequisites()`, which checks pause state and non-zero caller — no role or ownership check. [1](#0-0) 

The function reads and writes a **single global** storage variable `last_reward_block`: [2](#0-1) 

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After the guard passes, `last_reward_block` is immediately written to the current block number: [3](#0-2) 

This means **only one call per block succeeds for the entire contract**, regardless of which staker is targeted.

The function also accepts a public `disable_rewards: bool` parameter. When `true`, the function returns early after updating `last_reward_block` — no rewards are distributed to anyone: [4](#0-3) 

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
```

The storage declaration confirms `last_reward_block` is a single global slot, not per-staker: [5](#0-4) 

There is no incentive for any external party to call this function. The protocol presumably relies on a centralized off-chain bot. Because the function is fully public and `disable_rewards` is an unchecked caller-supplied parameter, an attacker can:

1. Call `update_rewards(any_valid_active_staker, true)` every block.
2. Each call consumes the block's single allowed slot and writes `last_reward_block = current_block`.
3. Every other call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`.
4. No staker receives consensus block rewards for that block.

The attacker does not need to be a staker. They only need to supply any valid, active staker address with non-zero balance (trivially obtained from on-chain events or `get_stakers`). [6](#0-5) 

### Impact Explanation
Every block the attacker executes this call, **all stakers permanently lose their consensus block rewards** for that block. Block rewards are computed per-block and are not retroactively recoverable — a missed block is a missed reward. Sustained over many blocks, this constitutes **permanent freezing of unclaimed yield** for the entire staker set.

This matches the allowed High impact: *Permanent freezing of unclaimed yield or unclaimed royalties*.

### Likelihood Explanation
- The attacker requires no privileged role, no staked capital, and no special knowledge beyond a valid staker address (publicly available on-chain).
- The only cost is gas per block on Starknet L2, which is negligible.
- The attack is trivially automatable with a simple script.
- There is no on-chain defense: the contract cannot distinguish a legitimate `disable_rewards = true` call from a malicious one.

Likelihood: **High**.

### Recommendation
1. **Remove the global `last_reward_block` constraint** or make it per-staker, so one call cannot block all others.
2. **Restrict `disable_rewards = true` to a privileged role** (e.g., `only_app_governor`) so unprivileged callers cannot suppress reward distribution.
3. **Introduce caller incentivization**: reward the caller with a small fraction of the distributed rewards, creating a natural economic incentive for decentralized keepers to call `update_rewards` honestly.
4. Consider a pull-based model where stakers trigger their own reward updates, eliminating the need for an external keeper entirely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
