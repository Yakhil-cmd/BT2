### Title
Unprivileged Caller Can Permanently Freeze All Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
`StakingRewardsManagerImpl::update_rewards` writes `last_reward_block` unconditionally before checking the `disable_rewards` flag, and the function has no access control beyond a zero-address check. Any unprivileged caller can invoke `update_rewards(any_active_staker, disable_rewards: true)` once per block to consume that block's global reward slot without distributing any rewards, permanently freezing unclaimed yield for every staker and delegator.

### Finding Description
In `update_rewards`, the global storage variable `last_reward_block` is updated **before** the early-return guard on `disable_rewards`:

```rust
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← written unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← no rewards distributed
}
```

The only access control is `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. There is no role check, no restriction to a consensus contract address, and no restriction to the staker themselves.

Because `last_reward_block` is a **single global variable** (not per-staker), one call per block is sufficient to lock out all stakers. The guard at the top of the function:

```rust
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

will revert every subsequent legitimate call for that block number.

The attacker only needs to supply any currently-active `staker_address` (the staker list is public via `stakers` storage and `get_stakers`). The function then validates the staker exists and has non-zero balance before writing `last_reward_block`, so the write always succeeds for a valid staker.

### Impact Explanation
An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` once per block permanently prevents every staker and delegator from receiving consensus-phase block rewards. This is a **permanent freezing of unclaimed yield** for the entire protocol — matching the allowed High-severity impact. Unlike the external report's lending scenario, here the attacker does not even need to lock up collateral; a single cheap transaction per block suffices.

### Likelihood Explanation
The entry path is fully permissionless. Any externally-owned account or contract can call `update_rewards`. The attacker pays gas for one transaction per block but inflicts unbounded yield loss on all protocol participants. A motivated griever, a competing protocol, or an MEV bot can sustain this attack indefinitely. The staker address required is trivially obtained from on-chain state.

### Recommendation
Restrict `update_rewards` to an authorized caller (e.g., a designated consensus contract address stored in the staking contract, enforced with a role check analogous to `assert_caller_is_attestation_contract`). Alternatively, only advance `last_reward_block` when rewards are actually distributed — i.e., move the write inside the block that executes `_update_rewards`, so that a `disable_rewards: true` call does not consume the slot.

### Proof of Concept
1. Protocol transitions to consensus phase (`is_pre_consensus()` returns `false`).
2. Attacker reads any active staker address from `stakers` storage.
3. At block N, attacker calls `update_rewards(active_staker, disable_rewards: true)`.
4. `last_reward_block` is set to N; `_update_rewards` is never reached; no rewards are minted or credited.
5. Any legitimate call to `update_rewards` for block N now reverts with `REWARDS_ALREADY_UPDATED`.
6. Attacker repeats step 3 for every subsequent block.
7. All stakers and delegators receive zero consensus rewards indefinitely; their accrued yield is permanently frozen. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
