### Title
Gateway L2 Gas Price Threshold Check Uses Wrong Reference Price (`l2_gas_price` Instead of `next_l2_gas_price`) — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful `validate_resource_bounds` check compares a transaction's `max_price_per_unit` against the **current block's** `l2_gas_price`, but the price that will actually be enforced when the transaction executes is the **next block's** `next_l2_gas_price` (the EIP-1559-computed value stored in `BlockHeaderWithoutHash`). The code even contains a `TODO` acknowledging this exact mistake. The result is a systematic mismatch between the gateway admission decision and the blockifier execution decision, causing the gateway to admit transactions that will be rejected at execution time (rising price) or to reject transactions that would have been valid (falling price).

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price reference from `gateway_fixed_block_state_reader.get_block_info()`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `BlockInfo.gas_prices.strk_gas_prices.l2_gas_price` from `block_header.l2_gas_price.price_in_fri` — the price **used in the previous (latest committed) block**: [2](#0-1) 

However, `BlockHeaderWithoutHash` also carries a distinct field `next_l2_gas_price: GasPrice`, which is the EIP-1559-adjusted price that will be enforced in the **next** block being built: [3](#0-2) 

`next_l2_gas_price` is computed by `calculate_next_l2_gas_price_for_fin` and written into the block header at proposal finalization time: [4](#0-3) 

The `GatewayFixedBlockStateReader` trait returns only `BlockInfo`, which does not expose `next_l2_gas_price`. Consequently, the gateway threshold check is always performed against the stale `l2_gas_price` of the previous block rather than the price that will actually be applied during execution.

The blockifier's `perform_pre_validation_stage` (called during actual execution) checks `resource_bounds.max_price_per_unit < actual_gas_price.get()` where `actual_gas_price` comes from the **next block's** `BlockContext`, which correctly uses `next_l2_gas_price`: [5](#0-4) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions or rejects valid transactions before sequencing.**

Two concrete scenarios arise from the price mismatch:

1. **Rising price (EIP-1559 upward adjustment):** `next_l2_gas_price > l2_gas_price`. A transaction with `max_price_per_unit ∈ [l2_gas_price × threshold%, next_l2_gas_price × threshold%)` passes the gateway check but fails the blockifier's `check_fee_bounds` at execution time. The gateway admits a transaction that will be rejected, wasting mempool resources and potentially causing the user's transaction to be silently dropped after sequencing begins.

2. **Falling price (EIP-1559 downward adjustment):** `next_l2_gas_price < l2_gas_price`. A transaction with `max_price_per_unit ∈ [next_l2_gas_price × threshold%, l2_gas_price × threshold%)` is rejected by the gateway even though it would have passed blockifier validation. Valid transactions are incorrectly denied admission.

### Likelihood Explanation

**Medium.** The EIP-1559 mechanism (`calculate_next_base_gas_price`) adjusts the L2 gas price every block based on gas usage relative to the target. Any block that is not exactly at the gas target produces a `next_l2_gas_price ≠ l2_gas_price`. Given that the default `min_gas_price_percentage` is 100 (full threshold), even a small EIP-1559 adjustment creates a window where the gateway and blockifier disagree. The discrepancy is proportional to the price change per block, which is bounded by `gas_price_max_change_denominator` but is non-zero in normal operation. [6](#0-5) 

### Recommendation

1. Extend `GatewayFixedBlockStateReader` to expose `next_l2_gas_price` from `BlockHeaderWithoutHash`, or return the full `BlockHeaderWithoutHash` alongside `BlockInfo`.
2. Replace the `l2_gas_price` reference in `validate_resource_bounds` with `next_l2_gas_price` from the latest committed block header, resolving the acknowledged `TODO`.
3. Add a test that constructs a scenario where `next_l2_gas_price ≠ l2_gas_price` and verifies that the gateway threshold matches the blockifier's admission decision.

### Proof of Concept

1. Observe that `BlockHeaderWithoutHash` stores both `l2_gas_price` (current block price) and `next_l2_gas_price` (next block price): [3](#0-2) 

2. `GatewayFixedBlockSyncStateClient` maps only `l2_gas_price` into `BlockInfo`, discarding `next_l2_gas_price`: [7](#0-6) 

3. `validate_resource_bounds` reads `strk_gas_prices.l2_gas_price` (the previous block's price) and the `TODO` comment confirms the correct value should be `next_l2_gas_price`: [8](#0-7) 

4. Craft a transaction with `l2_gas.max_price_per_unit = P` where `l2_gas_price × threshold% ≤ P < next_l2_gas_price × threshold%` (achievable whenever EIP-1559 raises the price). The gateway's `validate_tx_l2_gas_price_within_threshold` passes because it compares against the lower `l2_gas_price`. The blockifier's `check_fee_bounds` then rejects the transaction because it compares against the higher `next_l2_gas_price`. The transaction is admitted to the mempool but fails at execution. [9](#0-8)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
