Audit Report

## Title
Stale 5% Worst-Case Modulation Bound in `spawn_neuron` Allows Sub-Minimum-Stake Neurons Under Mission 70's −10% Floor — (`rs/nns/governance/src/governance.rs`)

## Summary

`spawn_neuron` validates maturity sufficiency using a hardcoded 5% worst-case modulation factor, but Mission 70 expanded the actual worst-case floor to −10% (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). The validation gate and the execution path now use different bounds, allowing any NNS neuron owner to spawn a child neuron whose maturity passes the pre-check but whose resulting `cached_neuron_stake_e8s` falls below `neuron_minimum_stake_e8s` when the live modulation is near −10%. A parallel issue exists in `initiate_maturity_disbursement`, where the comment explicitly states the check must account for worst-case modulation but the implementation applies no modulation factor at all.

## Finding Description

**Root cause 1 — `spawn_neuron` hardcoded 5% floor**

`rs/nns/governance/src/governance.rs` L2664–2673 contains:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

The literal `0.05` encodes the old ±5% CMC-based modulation range.

Mission 70 defines a new floor in `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs` L46–50:

```rust
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000; // −10%
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 = 200;    // +2%
```

`maybe_spawn_neurons` reads the live modulation value at L6427–6435 and applies it via `apply_maturity_modulation` at L6484–6487. The sanity check at L6438–6447 explicitly references `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` and `MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70` as the valid range, confirming the execution path accepts values down to −1000 permyriad (−10%).

The gap: any maturity `M` satisfying `M × 0.95 ≥ neuron_minimum_stake_e8s` (passes the gate) but `M × 0.90 < neuron_minimum_stake_e8s` (fails at actual spawn) produces a child neuron with `cached_neuron_stake_e8s` below the minimum. This range is `M ∈ [neuron_minimum_stake_e8s / 0.95, neuron_minimum_stake_e8s / 0.90)`.

**Root cause 2 — `initiate_maturity_disbursement` ignores modulation entirely**

`rs/nns/governance/src/governance/disburse_maturity.rs` L41–45 states:

> "A neuron can only disburse an amount of maturity that results in minting at least this many ICP (in e8) **assuming the worst case maturity modulation**."

But the actual check at L291–298 is:

```rust
if disbursement_maturity_e8s < MINIMUM_DISBURSEMENT_E8S {
```

No modulation factor is applied. A disbursement of exactly `MINIMUM_DISBURSEMENT_E8S` (1 ICP = 1e8 e8s) at −10% modulation yields 0.9 ICP — below the stated minimum, violating the invariant the comment documents.

## Impact Explanation

**Sub-minimum-stake neurons (High — Significant NNS governance impact with concrete protocol harm):** A child neuron can be created with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`, bypassing the minimum-stake invariant enforced everywhere else in governance. Such a neuron may be ineligible to vote, earn rewards, or be merged, yet it permanently occupies governance state and holds a ledger subaccount. This is a concrete NNS protocol invariant violation reachable by any unprivileged neuron owner.

**Sub-minimum disbursements:** A maturity disbursement of exactly `MINIMUM_DISBURSEMENT_E8S` at −10% modulation mints 0.9 ICP — below the floor the constant's own comment guarantees — violating the conservation property `initiate_maturity_disbursement` is designed to enforce.

## Likelihood Explanation

- Mission 70 is live on mainnet (Proposal 141441, April 2026). The −10% floor is reachable whenever the 7-day ICP price average falls significantly below the 365-day average.
- Any NNS neuron owner can call `manage_neuron { Spawn { ... } }` as an unprivileged ingress message. No special role or key is required.
- The exploitable maturity range (`[neuron_minimum_stake_e8s/0.95, neuron_minimum_stake_e8s/0.90)`) is narrow (~5.5% wide relative to the minimum stake) but reachable by any neuron with accumulated maturity near the minimum stake threshold.
- The modulation value and its daily trajectory (speed-limited to 30 permyriad/day per `MATURITY_MODULATION_DAILY_SPEED_LIMIT_PERMYRIAD`) are publicly queryable, making the timing of near-floor conditions predictable.

## Recommendation

1. **Fix `spawn_neuron`**: Replace the hardcoded `0.05` with the actual Mission 70 worst-case floor using `apply_maturity_modulation`:
   ```rust
   use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;
   let least_possible_stake = apply_maturity_modulation(
       maturity_to_spawn,
       MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
   ).unwrap_or(0);
   ```

2. **Fix `initiate_maturity_disbursement`**: Apply the worst-case modulation before comparing to `MINIMUM_DISBURSEMENT_E8S`, consistent with the comment's stated intent:
   ```rust
   let worst_case_amount = apply_maturity_modulation(
       disbursement_maturity_e8s,
       MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
   ).unwrap_or(0);
   if worst_case_amount < MINIMUM_DISBURSEMENT_E8S { ... }
   ```

3. **Introduce a named constant** for the worst-case bound used in validation so it stays in sync with `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` as the range evolves.

## Proof of Concept

Assume `neuron_minimum_stake_e8s = 100_000_000` (1 ICP, the default).

**Step 1**: Neuron owner accumulates maturity of `106_000_000` e8s (~1.06 ICP).

**Step 2**: Owner calls `manage_neuron { id, Spawn { percentage_to_spawn: 100, ... } }`.

**Step 3**: Validation in `spawn_neuron` (L2666):
```
least_possible_stake = 106_000_000 × 0.95 = 100_700_000 ≥ 100_000_000  ✓ passes
```

**Step 4**: Child neuron is created in spawning state with `maturity_e8s_equivalent = 106_000_000`.

**Step 5**: 7 days later, `maybe_spawn_neurons` fires. Suppose `maturity_modulation.current_value_permyriad = -1_000` (−10%, the Mission 70 floor, accepted by the sanity check at L6438–6447).

**Step 6**: `apply_maturity_modulation(106_000_000, -1_000)` (L6484–6487):
```
106_000_000 × (10_000 − 1_000) / 10_000 = 106_000_000 × 0.90 = 95_400_000
```

**Step 7**: `cached_neuron_stake_e8s = 95_400_000 < 100_000_000 = neuron_minimum_stake_e8s`.

The child neuron now exists on the ledger with a sub-minimum stake, bypassing the invariant that `spawn_neuron` was designed to enforce.

A deterministic integration test can reproduce this by: (1) setting `neuron_minimum_stake_e8s = 100_000_000`, (2) creating a neuron with `maturity_e8s_equivalent = 106_000_000`, (3) calling `spawn_neuron`, (4) advancing time by 7 days, (5) setting `maturity_modulation.current_value_permyriad = -1_000`, (6) calling `maybe_spawn_neurons`, and (7) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.
