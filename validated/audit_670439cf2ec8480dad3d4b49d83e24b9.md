### Title
Unhandled `ft_transfer_call` Partial Refund in `ExitToNear` Omni Path Causes Permanent User Fund Freeze - (File: `engine-precompiles/src/native.rs`)

---

### Summary

When the `ExitToNear` precompile is invoked with an Omni message, it burns the user's full ERC-20 (or ETH) balance in the EVM and schedules a NEAR `ft_transfer_call` promise. The NEP-141 `ft_transfer_call` standard allows the receiver's `ft_on_transfer` to return a non-zero refund amount (partial acceptance). The NEP-141 contract then refunds those tokens back to Aurora engine's own NEP-141 balance. However, Aurora Engine registers no callback to detect this partial refund and re-mint the corresponding ERC-20 tokens. The burned ERC-20 tokens are permanently unrecoverable for the user.

---

### Finding Description

In `engine-precompiles/src/native.rs`, the `ExitToNear::run` function handles the Omni message path by selecting `ft_transfer_call` as the NEAR method:

- For base-token Omni: `exit_base_token_to_near` returns `method = "ft_transfer_call"` and `transfer_near_args = None`.
- For ERC-20 Omni: `exit_erc20_token_to_near` returns `method = "ft_transfer_call"` and `transfer_near_args = None`. [1](#0-0) [2](#0-1) 

After computing the method, the precompile constructs `callback_args`:

```rust
let callback_args = ExitToNearPrecompileCallbackArgs {
    #[cfg(feature = "error_refund")]
    refund: refund_call_args(&exit_to_near_params, &exit_event),
    #[cfg(not(feature = "error_refund"))]
    refund: None,
    transfer_near: transfer_near_args,  // None for Omni case
};
``` [3](#0-2) 

The promise is then created conditionally:

```rust
let promise = if callback_args == ExitToNearPrecompileCallbackArgs::default() {
    PromiseArgs::Create(transfer_promise)   // no callback
} else {
    PromiseArgs::Callback(PromiseWithCallbackArgs { base: transfer_promise, callback: ... })
};
``` [4](#0-3) 

**Without `error_refund` feature**: `refund = None`, `transfer_near = None` → `callback_args == default()` → **no callback is attached at all**. Any outcome of `ft_transfer_call`, including partial refund, is silently ignored.

**With `error_refund` feature**: `refund = Some(...)`, `transfer_near = None` → `callback_args != default()` → a callback **is** attached. However, `exit_to_near_precompile_callback` only distinguishes between `PromiseResult::Successful` and `PromiseResult::Failed`:

```rust
let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
    // Promise succeeded → do nothing (no partial-refund handling)
    None
} else if let Some(args) = args.refund {
    // Promise failed entirely → refund ERC-20 tokens
    engine::refund_on_error(...)
} else {
    None
};
``` [5](#0-4) 

A `ft_transfer_call` that partially accepts tokens **succeeds** at the NEAR promise level (it does not panic). The NEP-141 contract refunds the unaccepted portion back to Aurora engine's own NEP-141 balance. The callback sees `PromiseResult::Successful` and takes no action. The refunded NEP-141 tokens accumulate in Aurora engine's account as orphaned funds, while the user's ERC-20 tokens were already burned for the full requested amount.

---

### Impact Explanation

**Permanent fund freeze.** The user burns 100 ERC-20 tokens in the EVM. The receiver accepts only 50 NEP-141 tokens and refunds 50 back to Aurora engine. The user receives 50 NEP-141 tokens at the destination but has no ERC-20 tokens left and no mechanism to reclaim the 50 NEP-141 tokens sitting in Aurora engine's account. Those 50 tokens are permanently inaccessible to the user. This matches the "permanent freezing of funds" impact class.

---

### Likelihood Explanation

**Medium.** The Omni `ft_transfer_call` path is a standard, user-accessible feature. Any NEAR contract implementing `ft_on_transfer` may legitimately return a non-zero refund (e.g., contracts with deposit caps, rate limits, or conditional acceptance logic). This is a normal part of the NEP-141 `ft_transfer_call` specification and does not require any malicious actor — a well-intentioned receiver contract with a cap is sufficient to trigger the loss.

---

### Recommendation

Register a dedicated callback for the `ft_transfer_call` Omni path that:
1. Reads the actual amount transferred (full amount minus the refund returned by `ft_on_transfer`).
2. Re-mints ERC-20 tokens (or credits ETH) to the original sender for the refunded portion.

Alternatively, parse the `PromiseResult::Successful(bytes)` return value of `ft_transfer_call` (which encodes the refunded amount as a JSON string) inside `exit_to_near_precompile_callback` and invoke `refund_on_error` for the partial amount.

---

### Proof of Concept

1. User calls the `ExitToNear` precompile (`0xe9217bc70b7ed1f598ddd3199e80b093fa71124f`) from an EVM transaction with flag `0x01` (ERC-20), amount = 100, Omni message targeting a NEAR receiver contract.
2. The EVM burns 100 ERC-20 tokens from the user's balance (state committed atomically with the EVM transaction).
3. Aurora Engine schedules `ft_transfer_call` on the NEP-141 contract for 100 tokens to the receiver.
4. The receiver's `ft_on_transfer` returns `"50"` (accepts only 50, refunds 50).
5. The NEP-141 contract transfers 50 tokens to the receiver and refunds 50 tokens to Aurora engine's NEP-141 balance.
6. The NEAR promise resolves as `PromiseResult::Successful`. The callback (if present) takes no action. Without a callback, nothing happens.
7. The user has lost 100 ERC-20 tokens. They received 50 NEP-141 tokens at the destination. The remaining 50 NEP-141 tokens are permanently stranded in Aurora engine's account with no corresponding ERC-20 backing and no user-accessible recovery path. [6](#0-5) [7](#0-6)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
