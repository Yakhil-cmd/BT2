### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Deny Block Rewards to Stakers - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is documented as callable only by the Starknet sequencer, but the implementation enforces no such restriction. Any unprivileged caller can invoke it with `disable_rewards: true` to consume the per-block reward slot (`last_reward_block`) without distributing rewards, permanently denying stakers the yield they were entitled to for that block.

### Finding Description
The `update_rewards` function in `StakingRewardsManagerImpl` is the consensus-era mechanism by which the Starknet sequencer distributes per-block rewards to stakers. The protocol spec explicitly states its access control is **"Only starkware sequencer"**.

However, the implementation only calls `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only: not paused + caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.  <-- written BEFORE the disable_rewards check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;   // returns without distributing any rewards
    }
    ...
``` [1](#0-0) 

The global `last_reward_block` is written to the current block number **before** the `disable_rewards` branch is evaluated. Because `last_reward_block` is a single contract-wide value (not per-staker), once it is set to block N, no further call to `update_rewards` can succeed in block N — the `REWARDS_ALREADY_UPDATED` assertion will revert any subsequent attempt. [2](#0-1) 

The spec's access-control requirement is documented but never enforced in code: [3](#0-2) 

### Impact Explanation
An attacker front-runs the sequencer's legitimate `update_rewards` call in block N by submitting `update_rewards(any_active_staker, disable_rewards: true)`. This:

1. Passes all checks (contract unpaused, caller non-zero, block N > `last_reward_block`, staker active with non-zero balance).
2. Writes `last_reward_block = N` and returns without distributing rewards.
3. The sequencer's call for block N reverts with `REWARDS_ALREADY_UPDATED`.

The staker that was scheduled to receive rewards in block N permanently loses those block rewards — there is no retroactive catch-up mechanism. Repeating this every block causes **permanent, continuous loss of unclaimed yield** for all stakers. This maps to **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- The entry path requires only a non-zero, non-paused caller and a valid active staker address — both trivially satisfied.
- Active staker addresses are fully public (emitted in `NewStaker` events and stored in the public `stakers` Vec).
- The attacker's cost is one transaction per block; the damage (lost yield) is irreversible for each targeted block.
- No privileged access, leaked key, or external dependency is required. [4](#0-3) 

### Recommendation
Enforce the sequencer-only access control that the spec mandates. Add a role check at the top of `update_rewards`, analogous to how other privileged functions use `self.roles.only_*()` guards:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer(); // enforce spec-mandated access control
    self.general_prerequisites();
    ...
```

Alternatively, restrict the caller to the known sequencer address stored in contract configuration, or introduce a dedicated `SEQUENCER_ROLE` enforced via the existing `RolesComponent`.

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. Attacker observes the mempool or simply submits at the start of every block.
3. Attacker calls `staking.update_rewards(victim_staker_address, disable_rewards: true)`.
4. `last_reward_block` is set to the current block; no rewards are distributed.
5. The sequencer's `update_rewards(victim_staker_address, disable_rewards: false)` reverts with `REWARDS_ALREADY_UPDATED`.
6. `victim_staker_address` receives zero block rewards for that block — permanently lost.
7. Repeating step 3 every block causes the staker to accrue zero yield indefinitely. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
