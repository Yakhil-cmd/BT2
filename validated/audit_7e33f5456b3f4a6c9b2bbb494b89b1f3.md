### Title
Unauthenticated Invoke Transaction Admitted to Mempool via `skip_stateful_validations` Signature-Skip Bypass - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator skips the `__validate__` entry-point call (i.e., signature verification) for any Invoke transaction with `nonce == 1` when the sender address already appears in the mempool. Because the sender address in an Invoke transaction is an unauthenticated field, any adversary can submit a fake-signature Invoke for a victim's address immediately after the victim's `DeployAccount` transaction enters the mempool, causing the invalid transaction to be admitted without signature verification.

### Finding Description

The `skip_stateful_validations` function implements a UX feature: when a user submits a `DeployAccount` and an `Invoke` (nonce=1) simultaneously, the gateway skips the `__validate__` entry-point for the Invoke because the account contract does not yet exist on-chain. [1](#0-0) 

The skip condition is:
1. Transaction is `Invoke`
2. `tx.nonce() == Nonce(Felt::ONE)`
3. `account_nonce == Nonce(Felt::ZERO)` (account not yet deployed)
4. `account_tx_in_pool_or_recent_block(sender_address)` returns `true` [2](#0-1) 

Condition 4 is satisfied whenever **any** transaction from `sender_address` is present in the mempool pool or state: [3](#0-2) 

The `sender_address` field of an Invoke transaction is never authenticated at the gateway level — authentication is delegated entirely to the `__validate__` entry point. The stateless validator only checks that the address is structurally valid (non-zero): [4](#0-3) 

The mempool's `validate_tx` only checks nonce ordering, not signatures: [5](#0-4) 

**Attack sequence:**

1. Alice submits `DeployAccount` for address `A`. It passes full validation and enters the mempool.
2. Bob observes Alice's `DeployAccount` in the mempool (transactions are P2P-propagated).
3. Bob crafts an `Invoke` with `sender_address = A`, `nonce = 1`, arbitrary calldata, and a **fake signature**, with a fee high enough to win fee escalation.
4. Gateway stateful validation: `account_nonce = 0`, `tx_nonce = 1`, `account_tx_in_pool_or_recent_block(A) = true` → `skip_stateful_validations` returns `true`.
5. `run_validate_entry_point` is **not called**; Bob's fake-signature Invoke is admitted to the mempool.
6. Alice submits her legitimate Invoke at nonce=1 → rejected with `DuplicateNonce` (Bob's fake Invoke already occupies that slot), or Alice must pay a higher fee to escalate past it. [6](#0-5) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions and rejects valid transactions before sequencing.**

- An Invoke transaction with a completely fake signature is admitted to the mempool, violating the invariant that every admitted transaction has passed account-level signature verification.
- Alice's legitimate first Invoke after account deployment is either rejected outright (`DuplicateNonce`) or forced to pay a higher fee to displace Bob's fake transaction.
- Bob can repeat this attack indefinitely to permanently grief Alice's first post-deployment transaction, since each time Alice's `DeployAccount` enters the mempool, Bob can immediately occupy nonce=1.
- The fake Invoke will fail during blockifier execution (the blockifier calls `__validate__` unconditionally during sequencing), but by then Alice's legitimate transaction has already been displaced from the mempool.

### Likelihood Explanation

**Medium.** The attack requires Bob to observe Alice's `DeployAccount` in the mempool and submit his fake Invoke before Alice submits her legitimate Invoke. Since mempool transactions are propagated via P2P to all nodes, any network participant can observe pending `DeployAccount` transactions and race to occupy nonce=1. The window is the time between Alice's `DeployAccount` entering the mempool and Alice's Invoke being submitted — a window that is explicitly widened by the UX feature itself (users are encouraged to submit both transactions together, meaning the Invoke may arrive slightly after the `DeployAccount`).

### Recommendation

The `skip_stateful_validations` check should verify that the transaction in the mempool for `sender_address` is specifically a `DeployAccount` transaction, not just any transaction. The mempool's `account_tx_in_pool_or_recent_block` API should be replaced with a more specific query such as `has_pending_deploy_account(address)` that only returns `true` when a `DeployAccount` transaction is present in the pool for that address. This prevents an adversary from satisfying the skip condition by injecting an arbitrary transaction for the victim's address.

    async fn run_validate_entry_point(
        &mut self,
        executable_tx: &ExecutableTransaction,
        skip_validate: bool,
    ) -> StatefulTransactionValidatorResult<()> {
        let only_query = false;
        let charge_fee = enforce_fee(executable_tx, only_query);
        let strict_nonce_check = false;
        let execution_flags =
            ExecutionFlags { only_query, charge_fee, validate: !skip_validate, strict_nonce_check };

        let account_tx = AccountTransaction { tx: executable_tx.clone(), execution_flags };

        // Build block context.
        let mut versioned_constants = VersionedConstants::get_versioned_constants(
            self.config.versioned_constants_overrides.clone(),
        );
        // The validation of a transaction is not affected by the casm hash migration.
        versioned_constants.disable_casm_hash_migration();

        let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
        block_info.block_number = block_info.block_number.unchecked_next();
        let block_context = BlockContext::new(
            block_info,
            self.chain_info.clone(),
            versioned_constants,
            BouncerConfig::max(),
        );

        // Move state into the blocking task and run CPU-heavy validation.
        let state_reader_and_contract_manager = self.take_state_reader_and_contract_manager();

        let cur_span = Span::current();
        #[allow(clippy::result_large_err)]
        tokio::task::spawn_blocking(move || {
            cur_span.in_scope(|| {
                let state = CachedState::new(state_reader_and_contract_manager);
                let mut blockifier_validator = StatefulValidator::create(state, block_context);
                blockifier_validator.validate(account_tx)
            })
        })
        .await
        .map_err(|e| StarknetError {
            code: StarknetErrorCode::UnknownErrorCode(
                "StarknetErrorCode.InternalError".to_string(),
            ),
            message: format!("Blocking task join error when running the validate entry point: {e}"),
        })?
        .map_err(|e| StarknetError {
            code: StarknetErrorCode::KnownErrorCode(KnownStarknetErrorCode::ValidateFailure),
            message: e.to_string(),
        })?;
        Ok(())

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
