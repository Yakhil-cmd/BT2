### Title
Missing Validation of Identity Element for `app_pk` in CKD Protocol Allows Coordinator to Learn Confidential Derived Secret - (File: `src/confidential_key_derivation/protocol.rs`)

---

### Summary

The `ckd()` function in `src/confidential_key_derivation/protocol.rs` does not validate that the `app_pk` (application public key) parameter is not the identity element of the G1 group. If `app_pk` is the identity element, the ElGamal masking term `y_i * app_pk` evaluates to the group identity, completely stripping the blinding from every participant's share. The coordinator then directly observes `msk * H(pk || app_id)` — the exact confidential derived secret the protocol is designed to keep hidden from MPC nodes.

---

### Finding Description

The CKD protocol's confidentiality guarantee rests on each participant masking their share of the derived key with a random ElGamal blinding term. In `compute_signature_share()`:

```rust
// C <- S + y . A
let big_c = big_s + app_pk * y.0;
``` [1](#0-0) 

Here `big_s = x_i * H(pk || app_id)` is the participant's secret share of the derived key, and `app_pk * y.0` is the blinding term. If `app_pk` is the G1 identity element, then `app_pk * y.0 = identity` for any scalar `y`, so:

```
big_c_i = x_i * H(pk || app_id)   (blinding completely removed)
```

After the coordinator sums all Lagrange-weighted shares:

```
big_c = Σ λ_i * big_c_i = msk * H(pk || app_id)
``` [2](#0-1) 

This is the confidential derived key itself. The coordinator, who aggregates all shares, directly observes it in plaintext.

The `ckd()` entry point performs several input checks — minimum participants, duplicate detection, presence of `me` and `coordinator` — but performs **no check** that `app_pk` is not the identity element: [3](#0-2) 

The `compute_signature_share()` function likewise performs no such check before using `app_pk`: [4](#0-3) 

---

### Impact Explanation

**Critical — Extraction or disclosure of confidential derived secrets.**

The entire purpose of CKD is stated explicitly in the module documentation: deliver deterministic secrets to applications "without MPC nodes learning the derived secret." [5](#0-4) 

When `app_pk = identity`, the coordinator (an MPC node) directly reads `msk * H(pk || app_id)` from the aggregated `big_c` field of `CKDOutput`. This is the confidential derived key. The `unmask()` function confirms this is the target secret:

```rust
pub fn unmask(&self, secret_scalar: Scalar) -> Signature {
    self.big_c - self.big_y * secret_scalar
}
``` [6](#0-5) 

With `app_pk = identity`, `big_c` already equals `msk * H(pk || app_id)` before `unmask()` is even called.

---

### Likelihood Explanation

**Medium.** The `app_pk` parameter is supplied by the library caller at each participant node. In a realistic deployment, `app_pk` originates from an external application or is transmitted over a network. A malicious application, a misconfigured caller, or a man-in-the-middle that substitutes the identity point for `app_pk` triggers this path with no cryptographic barrier. The library provides no defense because it performs zero validation on `app_pk` before using it in the blinding computation.

---

### Recommendation

Add an explicit check in `ckd()` (or at the top of `compute_signature_share()`) that rejects `app_pk` equal to the G1 identity element:

```rust
if app_pk == ElementG1::identity() {
    return Err(InitializationError::BadParameters(
        "app_pk must not be the identity element".to_string(),
    ));
}
```

This mirrors the pattern already used elsewhere in the library for analogous zero-value guards (e.g., `assert_keyshare_inputs` rejecting a zero secret scalar): [7](#0-6)

```rust

use elliptic_curve::Group;

// Attacker supplies the identity element as app_pk
let app_pk = ElementG1::identity();

let protocol = ckd(
    &participants,
    coordinator,
    me,
    key_pair,
    AppId::try_from(b"victim-app").unwrap(),
    app_pk,   // <-- identity element, no validation rejects this
    rng,
).unwrap();

// After protocol completes, coordinator's CKDOutput.big_c == msk * H(pk || app_id)
// The coordinator has directly learned the confidential derived key.
// ckd_output.unmask(Scalar::ZERO) == msk * H(pk || app_id)
```

The blinding term `y * identity = identity` for every participant, so `big_c` aggregates to the unmasked `msk * H(pk || app_id)` with no further computation required by the coordinator.

pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    let comms = Comms::new();
    let chan = comms.shared_channel();

    let fut = run_ckd_protocol(
        chan,
        coordinator,
        me,
        participants,
        key_pair,
        app_id.into(),
        app_pk,
        rng,
    );
    Ok(make_protocol(comms, fut))

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
