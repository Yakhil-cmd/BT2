### Title
Unprivileged Caller Can Force Pool Member Exit, Permanently Blocking Atomic Pool Switch — (File: `src/pool/pool.cairo`)

### Summary
`exit_delegation_pool_action` in the Pool contract has no caller validation. Any address can invoke it for any pool member once the exit window has elapsed. An attacker can exploit this to forcibly complete a pool member's exit, clearing `unpool_time` and `unpool_amount`, which permanently blocks the pool member from calling `switch_delegation_pool` — the only mechanism for atomically moving stake to another staker's pool without a reward gap.

---

### Finding Description

`exit_delegation_pool_action` accepts a `pool_member` address as a parameter and performs no `get_caller_address()` check: [1](#0-0) 

The function:
1. Verifies the exit window has elapsed (`Time::now() >= unpool_time`)
2. Clears `pool_member_info.unpool_time` → `None`
3. Clears `pool_member_info.unpool_amount` → `Zero`
4. Transfers `unpool_amount` to `pool_member`

The spec explicitly documents this as open to any caller: [2](#0-1) 

`switch_delegation_pool`, the function a pool member uses to atomically move stake to a new staker's pool, requires `unpool_time` to be `Some`: [3](#0-2) 

Once an attacker calls `exit_delegation_pool_action`, `unpool_time` is `None`, and any pending or future `switch_delegation_pool` call by the pool member will revert with `MISSING_UNDELEGATE_INTENT`. [4](#0-3) 

---

### Impact Explanation

A pool member who called `exit_delegation_pool_intent` intending to subsequently call `switch_delegation_pool` (to move stake to a better-performing staker without a reward gap) is instead forced into a full exit. Consequences:

- The pool member's `unpool_amount` is sent to their address; they cannot redirect it atomically.
- `switch_delegation_pool` is permanently blocked for that intent cycle.
- To re-enter a pool, the member must call `enter_delegation_pool` or `add_to_delegation_pool`, which schedules their balance for `current_epoch + K`. They earn **zero rewards for K epochs** — a direct, quantifiable loss of unclaimed yield.
- The attacker gains nothing; this is pure griefing.

This matches the **Medium** allowed impact: *Griefing with no profit motive but damage to users or protocol*, and also touches **High** (temporary loss of staking yield during the forced K-epoch gap).

---

### Likelihood Explanation

- No special privilege is required. Any EOA or contract can call `exit_delegation_pool_action(pool_member)`.
- The only precondition is that the exit window has elapsed — a publicly observable on-chain state.
- An attacker can monitor `PoolMemberExitIntent` events, wait for `unpool_time` to pass, and immediately call the function before the pool member can execute `switch_delegation_pool`.
- Front-running is straightforward on Starknet given public mempool visibility.

---

### Recommendation

Restrict `exit_delegation_pool_action` to the pool member or their registered reward address, mirroring the access control already applied to `claim_rewards`:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let pool_member_info = self.internal_pool_member_info(:pool_member);
    let caller = get_caller_address();
    assert!(
        caller == pool_member || caller == pool_member_info.reward_address,
        "{}",
        Error::UNAUTHORIZED,
    );
    // ... rest of function
}
```

Alternatively, if permissionless execution is desired for UX reasons (e.g., relayers), add a flag allowing the pool member to opt in.

---

### Proof of Concept

**Attack flow:**

1. `pool_member` calls `exit_delegation_pool_intent(amount)` — intending to later call `switch_delegation_pool` to move to a higher-yield staker.
2. The exit window (`exit_wait_window`) elapses.
3. Before `pool_member` can call `switch_delegation_pool`, `attacker` calls:
   ```
   pool.exit_delegation_pool_action(pool_member: pool_member_address)
   ```
4. `pool_member_info.unpool_time` is now `None`, `unpool_amount` is `0`, and `unpool_amount` tokens are sent to `pool_member`.
5. `pool_member` attempts `switch_delegation_pool(to_staker, to_pool, amount)` — it reverts with `MISSING_UNDELEGATE_INTENT` because `unpool_time.is_some()` is `false`. [5](#0-4) 

6. `pool_member` must re-delegate via `enter_delegation_pool`, scheduling their balance for `epoch + K`, losing K epochs of staking rewards. [6](#0-5)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
