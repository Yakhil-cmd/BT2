### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards by Calling `update_rewards` with `disable_rewards: true` Every Block — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control beyond "caller is not zero." It unconditionally writes `last_reward_block` to the current block **before** checking the caller-supplied `disable_rewards` flag. Any unprivileged address can call this function every block with `disable_rewards: true`, consuming the block's single reward-distribution slot without distributing any rewards. All subsequent legitimate calls in the same block revert with `REWARDS_ALREADY_UPDATED`, permanently freezing consensus-era yield for every staker and delegator.

---

### Finding Description

`update_rewards` is exposed as part of `IStakingRewardsManager` with no role guard:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only: not-paused + caller != 0
``` [1](#0-0) 

`general_prerequisites` enforces only two conditions — the contract is not paused and the caller is non-zero: [2](#0-1) 

After validating the staker, the function writes `last_reward_block` **unconditionally**, then branches on `disable_rewards`:

```cairo
        // Update last block rewards.
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;
        }
        // ... actual reward distribution follows
``` [3](#0-2) 

The per-block guard is checked at the top:

```cairo
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
``` [4](#0-3) 

Because `last_reward_block` is written before the `disable_rewards` branch, a call with `disable_rewards: true` marks the block as "processed" while distributing nothing. Any legitimate call later in the same block reverts.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Every block in the consensus-rewards era produces STRK (and BTC) block rewards via `calculate_block_rewards` / `_update_rewards`. If an attacker calls `update_rewards(valid_staker, disable_rewards: true)` once per block, the entire consensus reward stream is silently discarded. Stakers accumulate zero `unclaimed_rewards_own`; delegators receive zero pool rewards. Because the attack is cheap (one transaction per block) and repeatable indefinitely, the freeze is permanent for as long as the attacker continues.

---

### Likelihood Explanation

**High.** The function is permissionlessly callable by any non-zero address. The attacker only needs to supply any currently-active staker address (public on-chain information). No stake, no role, no special setup is required. On Starknet, where sequencer ordering is deterministic, the attacker can reliably place their call before the legitimate consensus-mechanism call in every block.

---

### Recommendation

1. **Add access control**: Restrict `update_rewards` to a designated role (e.g., `REWARDS_MANAGER` or the attestation/consensus contract address), consistent with how `update_rewards_from_attestation_contract` is guarded by `assert_caller_is_attestation_contract`.
2. **Move the state write**: If open-call semantics are intentional, move `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` check so that a no-op call does not consume the block's reward slot.

---

### Proof of Concept

1. Attacker identifies any active staker address `S` (readable from `stakers` vector or events).
2. Each block, attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. Inside the call:
   - `current_block_number > last_reward_block` passes (new block).
   - Staker validation passes (S is valid and active).
   - `last_reward_block` is written to `current_block_number`.
   - `disable_rewards == true` → early return, zero rewards distributed.
4. The consensus mechanism's legitimate `update_rewards` call in the same block hits:
   ```
   assert!(current_block_number > self.last_reward_block.read(), ...)
   // → REWARDS_ALREADY_UPDATED → revert
   ```
5. Repeated every block: all stakers and delegators earn zero consensus rewards indefinitely.
