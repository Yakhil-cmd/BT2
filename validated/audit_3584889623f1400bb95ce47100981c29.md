### Title
Unprivileged cross-contract call can overwrite any account's `AccountId`-mode global contract, corrupting code identity for all referencing accounts - (File: runtime/runtime/src/global_contracts.rs)

### Summary
`action_deploy_global_contract` in `runtime/runtime/src/global_contracts.rs` performs no authorization check to verify that the receipt's `predecessor_id` matches the `receiver_id`. Any contract can create a cross-contract call to any target account with a `DeployGlobalContractByAccountId` action, overwriting the target's global contract. All accounts that reference the target's global contract by `AccountId` will then execute the attacker's injected code.

### Finding Description

`GlobalContractDeployMode::AccountId` is documented as allowing the account owner to deploy a global contract under their own account ID so that all referencing accounts can be updated atomically:

> "Contract is deployed under the owner account id. Users will be able reference it by that account id. This allows the owner to update the contract for all its users."

The invariant is that only the account whose ID is used as the key may write to `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(account_id) }`.

`action_deploy_global_contract` enforces no such invariant:

```rust
pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,          // == receipt.receiver_id()
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    // Only check: does the account have enough balance?
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else { ... };
    ...
    initiate_distribution(state_update, account_id.clone(), ..., &deploy_contract.deploy_mode, ...)?;
    Ok(())
}
``` [1](#0-0) 

Inside `initiate_distribution`, when `deploy_mode == AccountId`, the global contract is keyed by `account_id` (the receipt's `receiver_id`):

```rust
GlobalContractDeployMode::AccountId => {
    GlobalContractIdentifier::AccountId(account_id.clone())
}
``` [2](#0-1) 

The host functions `promise_batch_action_deploy_global_contract_by_account_id` (both in `VMLogic` and the wasmtime runner) allow any executing contract to attach a `DeployGlobalContractAction` with `AccountId` mode to a promise targeting **any** account: [3](#0-2) 

The resulting receipt has `receiver_id = victim_account`. When it executes, `action_deploy_global_contract` is called with `account_id = "victim.near"`, and the global contract is stored under `GlobalContractIdentifier::AccountId("victim.near")`, overwriting whatever the victim had previously deployed.

Neither `validate_deploy_global_contract_action` nor `apply_action` adds any predecessor-vs-receiver check: [4](#0-3) 

**Exact attack path:**

1. Attacker deploys `attacker.near` containing:
   ```rust
   pub fn overwrite(&self) {
       let p = env::promise_batch_create("victim.near");
       env::promise_batch_action_deploy_global_contract_by_account_id(
           p, MALICIOUS_WASM.len() as u64, MALICIOUS_WASM.as_ptr() as u64,
       );
   }
   ```
2. Attacker calls `overwrite()`. This creates a receipt:
   - `predecessor_id = attacker.near`
   - `receiver_id = victim.near`
   - `actions = [DeployGlobalContractAction { code: malicious_wasm, deploy_mode: AccountId }]`
3. Receipt executes on `victim.near`. `action_deploy_global_contract` is called with `account_id = "victim.near"`. Storage cost is deducted from `victim.near`'s balance. The global contract under `GlobalContractIdentifier::AccountId("victim.near")` is overwritten with the attacker's code and a distribution receipt is queued to propagate it to all shards.
4. Every account that previously called `UseGlobalContract(AccountId("victim.near"))` now executes the attacker's WASM on every function call.

**Existing guards and why they fail:**

- `validate_deploy_global_contract_action` only checks `code.len() <= max_contract_size`. [5](#0-4) 
- `validate_actions_with_mode` applies no predecessor-vs-receiver constraint for `DeployGlobalContract`. [6](#0-5) 
- The nonce-based idempotency in `check_and_update_nonce` only prevents stale in-flight distribution receipts from overwriting a newer version; it does not prevent the initial hostile overwrite. [7](#0-6) 

### Impact Explanation

**Corrupted value:** `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId("victim.near") }` — the WASM bytecode executed by every account that holds `AccountContract::GlobalByAccount("victim.near")`.

**Concrete consequences:**
- Every account that opted into `victim.near`'s global contract now executes attacker-controlled WASM. The attacker can drain those accounts' balances, corrupt their storage, or brick them permanently.
- The victim's own account loses balance (storage cost charged from `victim.near`).
- The overwrite propagates to all shards via the distribution receipt, making it network-wide and irreversible without a new legitimate deploy.

Severity: **Critical** — unprivileged permanent corruption of a shared code artifact affecting an unbounded number of accounts.

### Likelihood Explanation

Preconditions:
- Attacker has a NEAR account and can deploy a contract (standard user capability).
- `victim.near` must have sufficient balance to cover the storage cost of the attacker's code (true for any active account holding a global contract).
- No privileged keys, no validator access, no admin role required.

The attack is a single cross-contract call. The only cost to the attacker is gas; the storage cost is borne by the victim.

### Recommendation

In `action_deploy_global_contract`, before calling `initiate_distribution`, add an authorization check for `AccountId` mode:

```rust
if let GlobalContractDeployMode::AccountId = &deploy_contract.deploy_mode {
    // The receipt's predecessor must be the account itself (self-call only).
    // Cross-contract calls from a different predecessor are not permitted to
    // overwrite another account's named global contract.
    if receipt.predecessor_id() != account_id {
        result.result = Err(ActionErrorKind::ActorNoPermission {
            account_id: account_id.clone(),
            actor_id: receipt.predecessor_id().clone(),
        }.into());
        return Ok(());
    }
}
```

`CodeHash` mode requires no such check because the key is the content hash of the code itself, not the deployer's identity, so there is no ownership invariant to protect.

### Proof of Concept

```rust
// attacker.near contract
#[near_bindgen]
impl Attacker {
    pub fn overwrite_victim(&self) {
        // Malicious WASM that drains caller balance on every call
        let malicious: &[u8] = include_bytes!("drain.wasm");
        let p = env::promise_batch_create("victim.near");
        // This host function is unrestricted — no predecessor check
        env::promise_batch_action_deploy_global_contract_by_account_id(
            p,
            malicious.len() as u64,
            malicious.as_ptr() as u64,
        );
    }
}
```

After `overwrite_victim` executes:
- `GlobalContractCode { AccountId("victim.near") }` in the trie contains `drain.wasm`.
- Every account `A` where `A.contract == GlobalByAccount("victim.near")` now executes `drain.wasm` on every function call.
- The distribution receipt propagates the overwrite to all shards automatically via `initiate_distribution`.

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

pub(crate) fn validate_actions_with_mode(
    limit_config: &LimitConfig,
    actions: &[Action],
    receiver: &AccountId,
    current_protocol_version: ProtocolVersion,
    mode: ValidateReceiptMode,
) -> Result<(), ActionsValidationError> {
    if actions.len() as u64 > limit_config.max_actions_per_receipt {
        return Err(ActionsValidationError::TotalNumberOfActionsExceeded {
            total_number_of_actions: actions.len() as u64,
            limit: limit_config.max_actions_per_receipt,
        });
    }

    // Centralized post-quantum gate. Mirrors the tx-admission gate in
    // `check_valid_for_config`, and is load-bearing for actions emitted by
    // contracts via host functions: those actions create new receipts that
    // never go through tx admission, so on a pre-feature protocol they must
    // be rejected here. The exhaustive match in
    // `Action::post_quantum_signatures_required` (including the recursive
    // walk into `Delegate`) forces every future action variant to make an
    // explicit decision at compile time.
    if !ProtocolFeature::PostQuantumSignatures.enabled(current_protocol_version)
        && actions.iter().any(Action::post_quantum_signatures_required)
    {
        return Err(ActionsValidationError::UnsupportedProtocolFeature {
            protocol_feature: "PostQuantumSignatures".to_owned(),
            version: current_protocol_version,
        });
    }

    if mode == ValidateReceiptMode::NewReceipt {
        validate_number_of_deploy_actions(actions, limit_config.max_deploy_actions_per_receipt)?;
    }

    let mut found_delegate_action = false;
    let mut iter = actions.iter().peekable();
    while let Some(action) = iter.next() {
        if let Action::DeleteAccount(_) = action {
            if iter.peek().is_some() {
                return Err(ActionsValidationError::DeleteActionMustBeFinal);
            }
        } else {
            if let Action::Delegate(_) | Action::DelegateV2(_) = action {
                if found_delegate_action {
                    return Err(ActionsValidationError::DelegateActionMustBeOnlyOne);
                }
                found_delegate_action = true;
            }
        }
        validate_action_with_mode(limit_config, action, receiver, current_protocol_version, mode)?;
    }

    let total_prepaid_gas =
        total_prepaid_gas(actions).map_err(|_| ActionsValidationError::IntegerOverflow)?;
    if total_prepaid_gas > limit_config.max_total_prepaid_gas {
        return Err(ActionsValidationError::TotalPrepaidGasExceeded {
            total_prepaid_gas,
            limit: limit_config.max_total_prepaid_gas,
        });
    }

    Ok(())
}
