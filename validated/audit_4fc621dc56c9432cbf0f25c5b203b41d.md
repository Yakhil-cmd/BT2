### Title
Unprotected `disable_rewards` Flag in `update_rewards` Permanently Destroys Per-Block Yield for Any Staker - (File: src/staking/staking.cairo)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. Because `last_reward_block` is written unconditionally before the early-return guard, any unprivileged caller can front-run the legitimate block-reward call with `disable_rewards = true`, permanently consuming the block slot and destroying the staker's (and their pool members') yield for that block with no possibility of recovery.

---

### Finding Description

`update_rewards` in `src/staking/staking.cairo` is declared in the public `IStakingRewardsManager` interface with no role check beyond `general_prerequisites()` (which only asserts the contract is unpaused and the caller is non-zero):

```cairo
// src/staking/staking.cairo  lines 1447-1507
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // only: not paused, caller != 0
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... staker existence / balance checks ...

        // *** last_reward_block is committed BEFORE the disable_rewards guard ***
        self.last_reward_block.write(current_block_number);   // line 1485

        if disable_rewards || self.is_pre_consensus() {
            return;                                           // line 1488 – exits with no rewards
        }
        // ... reward calculation and distribution ...
    }
}
```

`general_prerequisites` (lines 1794-1797) performs no role check:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

The `last_reward_block` storage slot is a **single global value** shared across all stakers. Once it is written to block `N`, the assertion `current_block_number > last_reward_block` prevents any further call in the same block from succeeding. Because the write happens unconditionally before the `disable_rewards` branch, a call with `disable_rewards = true` permanently marks block `N` as processed while distributing zero rewards.

This is structurally identical to the Origin Protocol "Trapped Liquidity Rewards" class: tokens/yield that are due for distribution are rendered permanently inaccessible because a state variable is advanced without the corresponding distribution taking place, and there is no recovery path.

---

### Impact Explanation

**High – Permanent freezing of unclaimed yield.**

For every block in the consensus-rewards phase:

- The staker loses their entire `strk_block_rewards` share for that block.
- All pool members of that staker lose their proportional pool rewards for that block.
- `unclaimed_rewards` in `RewardSupplier` is never incremented for those rewards, so the tokens are never minted/requested from L1 for that block.
- There is no replay mechanism: `last_reward_block` is a monotonically advancing cursor; once block `N` is consumed, it cannot be revisited.

An attacker who repeats this every block can permanently zero out a targeted staker's yield indefinitely.

---

### Likelihood Explanation

**Medium.**

- The function is fully public with no access control.
- The attacker only needs to submit a transaction in the same block as the legitimate `update_rewards` call, with `disable_rewards = true` and any valid active `staker_address`.
- On Starknet, transaction ordering within a block is determined by the sequencer; a griefing actor can submit the malicious call with a higher fee to be sequenced first.
- The cost to the attacker is only gas per block; there is no capital requirement.
- The attack is sustainable indefinitely and requires no special knowledge beyond knowing a valid staker address (which is public on-chain).

---

### Recommendation

1. **Restrict who may pass `disable_rewards = true`.** The parameter should only be settable by a trusted role (e.g., the attestation contract, a designated rewards manager role, or the staker themselves). Add a caller check:

   ```cairo
   if disable_rewards {
       self.roles.only_rewards_manager(); // or assert caller == attestation_contract
   }
   ```

2. **Alternatively, move `last_reward_block.write` after the `disable_rewards` guard**, so that a call with `disable_rewards = true` does not consume the block slot. This preserves the ability for the legitimate caller to still process the block.

3. **Or remove `disable_rewards` from the public interface entirely** and handle reward-skipping logic internally (e.g., based on attestation results), so no external caller can influence whether rewards are distributed.

---

### Proof of Concept

**Setup:** Consensus rewards are active (`is_pre_consensus()` returns `false`). Staker `S` is active with non-zero balance. Block `N` is the current block.

**Attack steps:**

1. Attacker observes block `N` is new (i.e., `N > last_reward_block`).
2. Attacker calls:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
   with a fee high enough to be sequenced before the staker's legitimate call.
3. Inside `update_rewards`:
   - `general_prerequisites()` passes (attacker is not zero, contract is not paused).
   - `current_block_number (N) > last_reward_block` assertion passes.
   - Staker `S` existence and balance checks pass.
   - **`last_reward_block.write(N)` executes** — block `N` is now consumed.
   - `disable_rewards == true` → early return, zero rewards distributed.
4. Staker `S` (or the consensus mechanism) now calls `update_rewards(S, false)`.
   - `current_block_number (N) > last_reward_block (N)` → **assertion fails** with `REWARDS_ALREADY_UPDATED`.
5. Block `N`'s rewards for staker `S` and all their pool members are permanently lost.
6. Attacker repeats every block at negligible cost.

            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();

            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

                return;

            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
