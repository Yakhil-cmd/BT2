### Title
Unprivileged Caller Can Permanently Deny Block Rewards to All Stakers via `disable_rewards` Parameter - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` is a public function that accepts a caller-controlled `disable_rewards: bool` parameter. When `true`, the function updates the global `last_reward_block` storage variable but skips reward distribution entirely. Because `last_reward_block` is a single global slot enforcing a one-call-per-block invariant, any unprivileged address can call `update_rewards(valid_staker, disable_rewards: true)` each block to permanently deny consensus block rewards to all stakers for that block.

### Finding Description

`update_rewards` in `src/staking/staking.cairo` is exposed through the `IStakingRewardsManager` interface with no access control beyond `general_prerequisites()` (not paused, caller non-zero). [1](#0-0) 

The function first writes the current block number into the global `last_reward_block` storage slot: [2](#0-1) 

Only *after* that write does it check `disable_rewards`:

```
if disable_rewards || self.is_pre_consensus() {
    return;
}
```

Because `last_reward_block` is a single global `BlockNumber` (not per-staker), once it is set to block N, the assertion `current_block_number > self.last_reward_block.read()` will reject every subsequent call to `update_rewards` in block N with `REWARDS_ALREADY_UPDATED`. [3](#0-2) [4](#0-3) 

The `disable_rewards` parameter is the direct analog of the unvalidated off-chain parameters in the original report: the caller supplies a value that bypasses the core accounting logic, and the contract has no on-chain mechanism to verify the parameter is legitimate.

The interface definition confirms no access restriction: [5](#0-4) 

### Impact Explanation

An attacker calling `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block:

1. Consumes the one permitted `update_rewards` slot for that block (global `last_reward_block` is updated).
2. Skips `_update_rewards`, so no STRK block rewards are calculated or credited to any staker or pool.
3. Makes every legitimate reward-update call for that block revert with `REWARDS_ALREADY_UPDATED`.

Rewards for each skipped block are permanently lost — they are never added to `unclaimed_rewards_own` or forwarded to pools. This constitutes **permanent freezing of unclaimed yield** for all stakers and all pool members, which is a **High** impact under the allowed scope.

### Likelihood Explanation

- The function is fully public; no role, key, or privileged access is required.
- The attacker only needs to know one active staker address (publicly readable from the `stakers` vector or events).
- The cost is a single cheap transaction per block.
- Front-running the legitimate sequencer/validator call is straightforward on Starknet where transaction ordering is observable.

### Recommendation

Add an access-control check so that only a trusted role (e.g., `security_agent`, `app_governor`, or a designated sequencer address) may call `update_rewards`. Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "skip rewards" logic through a separate privileged function or an internal flag that cannot be set by unprivileged callers.

### Proof of Concept

```
// Attacker script — run once per block before the legitimate update
let staking = IStakingRewardsManagerDispatcher { contract_address: staking_contract };

// Any valid, active staker address suffices
let victim_staker = <any staker from the public `stakers` vec>;

// Consumes last_reward_block for this block; no rewards distributed
staking.update_rewards(staker_address: victim_staker, disable_rewards: true);

// All subsequent legitimate calls in this block revert:
// "Rewards already updated for this block"
staking.update_rewards(staker_address: victim_staker, disable_rewards: false); // PANICS
```

Repeating this every block permanently prevents all consensus block rewards from accruing to any staker or pool member. [6](#0-5) [5](#0-4)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
