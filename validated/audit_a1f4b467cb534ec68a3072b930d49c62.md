### Title
Unprivileged Caller Can Permanently Freeze All Staker Consensus Rewards via `update_rewards(disable_rewards=true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable with no access control beyond a non-zero caller check. It accepts a `disable_rewards` boolean from any caller. When called with `disable_rewards=true`, it advances the global `last_reward_block` sentinel to the current block without distributing any rewards. Because the function enforces a strict "once per block" gate on this sentinel, an attacker can call `update_rewards(any_active_staker, disable_rewards=true)` in every block to permanently consume each block's reward slot, freezing all staker consensus rewards indefinitely.

### Finding Description

`update_rewards` is part of `StakingRewardsManagerImpl`, embedded as a public ABI entry point with no role restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker validity checks ...
    self.last_reward_block.write(current_block_number);   // ← sentinel advanced HERE
    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits before any reward math
    }
    // ... actual reward distribution ...
}
``` [1](#0-0) 

The sentinel `last_reward_block` is written unconditionally at line 1485, **before** the `disable_rewards` branch at line 1487. [2](#0-1) 

`general_prerequisites` only asserts the contract is unpaused and the caller is non-zero — no role check exists. [3](#0-2) 

The attack path:
1. Attacker observes any active staker address (emitted publicly via `NewStaker` events).
2. In every new block, attacker calls `update_rewards(active_staker, disable_rewards=true)`.
3. `last_reward_block` is set to the current block; the function returns without distributing rewards.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated across every block → all stakers receive zero consensus rewards indefinitely.

The state-transition bug is structurally identical to the reference report: a caller performs an action that advances a critical state variable (`last_reward_block` ↔ `liquidationStartTime`) without completing the intended state transition (reward distribution ↔ liquidation closure), leaving the system in a broken state that blocks all subsequent legitimate operations.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

All stakers and their delegators are denied consensus block rewards for every block in which the attacker front-runs the legitimate `update_rewards` call. Because the attacker can repeat this indefinitely (one cheap transaction per block), the yield freeze is permanent for as long as the attack continues. Delegators in pool contracts also lose their share of rewards since pool rewards flow through `_update_rewards`. [4](#0-3) 

### Likelihood Explanation
**High.** Any unprivileged address can call `update_rewards`. The only prerequisite is a valid active staker address, which is trivially obtained from on-chain `NewStaker` events. The attacker pays only transaction fees per block; no capital is required. The attack is fully permissionless and requires no special knowledge beyond the contract ABI.

### Recommendation
1. **Restrict `disable_rewards=true` to privileged callers** (e.g., a `SECURITY_AGENT` or `TOKEN_ADMIN` role), or remove the parameter from the public interface entirely.
2. **Move `last_reward_block.write` after the `disable_rewards` guard**, so the sentinel is only advanced when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // moved here
// ... reward distribution ...
```

3. Alternatively, gate the entire function with a role check so only the attestation contract or a trusted operator can invoke it.

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(alice_staker, disable_rewards=true)
    → last_reward_block = N  (no rewards sent)

  Legitimate tx (same block): update_rewards(alice_staker, disable_rewards=false)
    → PANICS: "Rewards already updated for this block"

Block N+1:
  Attacker repeats → last_reward_block = N+1, no rewards

... repeated every block → all stakers receive 0 consensus rewards forever
```

The attacker needs only one active staker address (public from events) and the ability to submit one transaction per block before any legitimate reward update.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
