### Title
`reverted_sierra_gas` Excluded from Bouncer `sierra_gas` Weight Causes Block Proving-Capacity Overrun — (`File: crates/blockifier/src/bouncer.rs`)

### Summary

`compute_sierra_gas` in the bouncer reads only `tx_resources.computation.sierra_gas` when computing the per-transaction `sierra_gas` bouncer weight. It silently omits `tx_resources.computation.reverted_sierra_gas`, which is the sierra gas consumed during the blockifier revert flow for Cairo 1 contracts. The fee-charging path (`ComputationResources::to_gas_vector`) correctly sums both fields, so the user is charged for the full amount, but the bouncer only accounts for a fraction of the actual proving work. This is the direct sequencer analog of the external "excess funds locked" report: a partial amount is recorded while the remainder is silently dropped.

### Finding Description

`ComputationResources` holds two distinct sierra-gas fields:

```rust
// crates/blockifier/src/fee/resources.rs  lines 59-68
pub struct ComputationResources {
    pub tx_extended_vm_resources: ExtendedExecutionResources,
    pub os_vm_resources: ExecutionResources,
    pub n_reverted_steps: usize,
    pub sierra_gas: GasAmount,
    pub reverted_sierra_gas: GasAmount,   // ← gas consumed in blockifier revert flow
}
``` [1](#0-0) 

The fee-calculation path (`ComputationResources::to_gas_vector`) correctly adds both:

```rust
// lines 92-98
let total_sierra_gas =
    self.sierra_gas.checked_add(self.reverted_sierra_gas).unwrap_or_else(|| { panic!(...) });
``` [2](#0-1) 

The bouncer path (`compute_sierra_gas`) reads only `sierra_gas`:

```rust
// crates/blockifier/src/bouncer.rs  lines 858-860
let sierra_gas = tx_resources.computation.sierra_gas;   // reverted_sierra_gas never read
vm_resources_sierra_gas =
    vm_resources_sierra_gas.checked_add_panic_on_overflow(sierra_gas);
``` [3](#0-2) 

`reverted_sierra_gas` is never referenced anywhere inside `compute_sierra_gas` or in `get_tx_weights`, so it contributes zero to `BouncerWeights::sierra_gas`. [4](#0-3) 

The `receipt_l2_gas` bouncer dimension does include `reverted_sierra_gas` (it comes from the full receipt gas vector), but its block-level cap (default 5 800 000 000) is **higher** than the `sierra_gas` cap (default 5 000 000 000):

```rust
// crates/blockifier/src/bouncer.rs  lines 219-230
sierra_gas: GasAmount(5000000000),
// ...
receipt_l2_gas: GasAmount(5800000000),
``` [5](#0-4) 

Because `sierra_gas` is the tighter dimension, the block is closed by `sierra_gas` first in the normal case. But when `reverted_sierra_gas` is large, the bouncer's `sierra_gas` counter is underestimated, so the block is instead closed by `receipt_l2_gas` at 5.8 B — at which point the actual sierra gas consumed can reach up to 5.8 B, exceeding the 5 B prover limit.

### Impact Explanation

An attacker submits Cairo 1 invoke transactions that:
1. Pass `__validate__` (small `sierra_gas`).
2. Revert during `__execute__` after consuming a large amount of sierra gas (large `reverted_sierra_gas`).

Each such transaction is underweighted in the bouncer. The block fills to the `receipt_l2_gas` ceiling (5.8 B) while the actual sierra gas consumed can be up to 5.8 B — 800 M units above the 5 B prover limit. The prover receives a block it cannot prove within its resource budget, causing a proving failure or forced block re-execution. This is a **Critical** incorrect bouncer/resource-accounting issue with direct economic impact on the sequencer operator and network liveness.

### Likelihood Explanation

Any reverted Cairo 1 transaction produces non-zero `reverted_sierra_gas`. The `SierraGasRevertTracker` records gas consumed during the revert flow and this value is stored in `ComputationResources::reverted_sierra_gas`. No special privilege is required; any user can submit a transaction that reverts. The omission is unconditional — it applies to every reverted Cairo 1 transaction in every block.

### Recommendation

In `compute_sierra_gas`, add `reverted_sierra_gas` alongside `sierra_gas`:

```rust
let sierra_gas = tx_resources.computation.sierra_gas
    .checked_add(tx_resources.computation.reverted_sierra_gas)
    .expect("sierra_gas + reverted_sierra_gas overflow");
```

This mirrors the existing logic in `ComputationResources::to_gas_vector` and ensures the bouncer's `sierra_gas` weight reflects the full proving cost of the transaction.

### Proof of Concept

1. Deploy a Cairo 1 contract whose `execute` entry point loops until it exhausts nearly all available sierra gas, then reverts.
2. Submit an invoke transaction against it with `l2_gas.max_amount` set to the full execute cap (1 110 000 000).
3. Observe: `receipt.resources.computation.reverted_sierra_gas` ≈ 1 110 000 000; `receipt.resources.computation.sierra_gas` ≈ 0 (validate only).
4. Call `get_tx_weights` with these resources; observe `bouncer_weights.sierra_gas` ≈ 0 while `bouncer_weights.receipt_l2_gas` ≈ 1 110 000 000.
5. Fill a block with ~5 such transactions (5 × 1.1 B = 5.5 B receipt_l2_gas < 5.8 B cap, but actual sierra gas = 5.5 B > 5 B sierra_gas cap).
6. The bouncer accepts the block; the prover receives a block whose actual sierra gas exceeds its 5 B limit.

fn compute_sierra_gas(
    vm_resources: &ExtendedExecutionResources,
    sierra_builtin_gas_costs: &BuiltinGasCosts,
    versioned_constants: &VersionedConstants,
    tx_resources: &TransactionResources,
    migration_gas: GasAmount,
    class_hash_to_casm_hash_computation_resources: &HashMap<ClassHash, ExtendedExecutionResources>,
) -> (GasAmount, CasmHashComputationData, GasAmount) {
    let mut vm_resources_sierra_gas = extended_execution_resources_to_gas(
        vm_resources,
        sierra_builtin_gas_costs,
        versioned_constants,
    );
    let sierra_gas = tx_resources.computation.sierra_gas;

    vm_resources_sierra_gas = vm_resources_sierra_gas.checked_add_panic_on_overflow(sierra_gas);

    let sierra_gas_without_casm_hash_computation =
        vm_resources_sierra_gas.checked_add_panic_on_overflow(migration_gas);

    let (total_sierra_gas, casm_hash_computation_data_sierra_gas) =
        add_casm_hash_computation_gas_cost(
            class_hash_to_casm_hash_computation_resources,
            sierra_gas_without_casm_hash_computation,
            sierra_builtin_gas_costs,
            versioned_constants,
        );
    (total_sierra_gas, casm_hash_computation_data_sierra_gas, vm_resources_sierra_gas)
}

#[allow(clippy::too_many_arguments)]
pub fn get_tx_weights<S: StateReader>(
    state_reader: &S,
    executed_class_hashes: &HashSet<ClassHash>,
    n_visited_storage_entries: usize,
    tx_resources: &TransactionResources,
    state_changes_keys: &StateChangesKeys,
    versioned_constants: &VersionedConstants,
    tx_cairo_primitives_counters: &CairoPrimitiveCounterMap,
    bouncer_config: &BouncerConfig,
    receipt_l2_gas: GasAmount,
) -> TransactionExecutionResult<TxWeights> {
    let message_resources = &tx_resources.starknet_resources.messages;
    let message_starknet_l1gas = usize_from_u64(message_resources.get_starknet_gas_cost().l1_gas.0)
        .expect("This conversion should not fail as the value is a converted usize.");

    // Casm hash resources.
    let class_hash_to_casm_hash_computation_resources =
        map_class_hash_to_casm_hash_computation_resources(state_reader, executed_class_hashes)?;

    // Patricia update + transaction resources.
    let patricia_update_resources = get_patricia_update_resources(
        n_visited_storage_entries,
        // TODO(Yoni): consider counting here the global contract tree and the aliases as well.
        state_changes_keys.storage_keys.len(),
    );
    let vm_resources =
        &tx_resources.computation.total_extended_vm_resources() + &patricia_update_resources;

    // Builtin gas costs for stone and for stwo.
    let sierra_builtin_gas_costs = &versioned_constants.os_constants.gas_costs.builtins;
    let proving_builtin_gas_costs = &bouncer_config.builtin_gas_costs();

    // Casm hash migration resources.
    let migration_data = CasmHashMigrationData::from_state(
        state_reader,
        executed_class_hashes,
        versioned_constants,
    )?;
    // Total state changes keys are the sum of marginal state changes keys and the
    // migration state changes.
    let mut total_state_changes_keys = StateChangesKeys {
        compiled_class_hash_keys: migration_data.class_hashes_to_migrate.keys().cloned().collect(),
        ..Default::default()
    };
    total_state_changes_keys.extend(state_changes_keys);

    // Migration occurs once per contract and is not included in the CASM hash computation, which
    // is performed every time a contract is loaded.
    let sierra_migration_gas = migration_data.to_gas(sierra_builtin_gas_costs, versioned_constants);
    let proving_migration_gas =
        migration_data.to_gas(proving_builtin_gas_costs, versioned_constants);

    // Sierra gas computation.
    let (total_sierra_gas, casm_hash_computation_data_sierra_gas, vm_resources_sierra_gas) =
        compute_sierra_gas(
            &vm_resources,
            sierra_builtin_gas_costs,
            versioned_constants,
            tx_resources,
            sierra_migration_gas,
            &class_hash_to_casm_hash_computation_resources,
        );

    // Proving gas computation.
    let cairo_primitives_for_proving_gas = get_cairo_primitives_for_proving_gas_computation(
        patricia_update_resources.prover_builtins(),
        tx_resources.computation.os_vm_resources.prover_builtins(),
        tx_cairo_primitives_counters,
    );

    let (total_proving_gas, casm_hash_computation_data_proving_gas) = compute_proving_gas(
        &cairo_primitives_for_proving_gas,
        vm_resources_sierra_gas,
        versioned_constants,
        proving_builtin_gas_costs,
        sierra_builtin_gas_costs,
        proving_migration_gas,
        &class_hash_to_casm_hash_computation_resources,
    );

    let bouncer_weights = BouncerWeights {
        l1_gas: message_starknet_l1gas,
        message_segment_length: message_resources.message_segment_length,
        n_events: tx_resources.starknet_resources.archival_data.event_summary.n_events,
        state_diff_size: get_onchain_data_segment_length(&total_state_changes_keys.count()),
        sierra_gas: total_sierra_gas,
        n_txs: 1,
        proving_gas: total_proving_gas,
        receipt_l2_gas,
    };

    Ok(TxWeights {
        bouncer_weights,
        casm_hash_computation_data_sierra_gas,
        casm_hash_computation_data_proving_gas,
        class_hashes_to_migrate: migration_data.class_hashes_to_migrate,
    })
}
