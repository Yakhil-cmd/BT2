### Title
Incorrect Royalty Accumulation in `formatAmountWithRoyalties` Causes WalletConnect Confirmation to Display Understated Spend Total - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary
`formatAmountWithRoyalties` in `parseCommandDisplay.ts` divides the fungible spend amount by the number of NFTs before applying each royalty percentage. For any offer involving more than one royalty-bearing NFT, the "Total Amount with Royalties" shown in the WalletConnect `Confirm` dialog is a factor of N smaller than the real on-chain cost, where N is the number of NFTs. A dapp with an active WalletConnect session can exploit this to make the user approve a `take_offer` or `create_offer_for_ids` transaction while believing they are spending significantly less than they actually will.

### Finding Description

`formatAmountWithRoyalties` is called by `withRoyaltyTotals` → `walletDeltaToDisplay` → `parseCommandDisplay` for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids`. Its result is stored in the `amountWithRoyalties` field of each `DisplayWalletDeltaItem` and rendered verbatim in `Confirm.tsx` under the label **"Total Amount with Royalties"** in the "You Spend" section of the approval dialog.

The flawed accumulation:

```typescript
// packages/gui/src/electron/commands/parseCommandDisplay.ts  lines 363-367
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← divides by N
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) =>
    total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,  // ← each royalty on 1/N of amount
  0n,
);
``` [1](#0-0) 

The code first computes `splitAmount = amount / N`, then applies every royalty percentage to that reduced base. The net effect is that the total royalty displayed equals `amount × Σ(royaltyPct) / (10 000 × N)` instead of the correct `amount × Σ(royaltyPct) / 10 000`. For N = 1 the result is accurate; for N ≥ 2 the displayed total is understated by a factor of N.

The rendered label and the rendering path are confirmed in `Confirm.tsx`:

```tsx
// packages/gui/src/electron/dialogs/Confirm/Confirm.tsx  lines 109-114
{line.amountWithRoyalties && (
  <div className="text-xs text-chia-text-secondary">
    {i18n._({ id: 'Total Amount with Royalties' })}: {line.amountWithRoyalties}{' '}
    {networkPrefix ? networkPrefix.toUpperCase() : 'XCH'}
  </div>
)}
``` [2](#0-1) 

The `royaltyPercentages` array is populated from the NFTs on the opposite side of the trade via `royaltyPercentagesForSide` → `withRoyaltyTotals`: [3](#0-2) 

### Impact Explanation

The user's only source of the total cost (base price + royalties) for a WalletConnect-initiated offer is the `amountWithRoyalties` line in the approval dialog. When that figure is wrong, the user approves a spend they would not have approved had they seen the true total. The actual deduction from the wallet is determined by the Chia blockchain and will be the correct (higher) amount; the GUI simply misled the user into consenting.

For two NFTs each carrying a 10 % royalty (1 000 basis points) and a 1 XCH base price:

| | Displayed | Actual |
|---|---|---|
| splitAmount | 0.5 XCH | — |
| Royalty per NFT | 0.05 XCH | 0.1 XCH |
| **Total shown** | **1.1 XCH** | **1.2 XCH** |

The discrepancy grows linearly with N and with royalty magnitude.

### Likelihood Explanation

Any dapp that has obtained a WalletConnect session (a normal, user-initiated pairing) can send `chia_wallet.take_offer` or `chia_wallet.create_offer_for_ids` with an offer that bundles two or more royalty-bearing NFTs. No additional privilege, key material, or host access is required. The attacker controls the offer payload and therefore controls N and the royalty percentages embedded in the driver dict or offer summary infos.

### Recommendation

Remove the per-NFT split. Each royalty percentage should be applied to the full fungible amount:

```typescript
// packages/gui/src/electron/commands/parseCommandDisplay.ts
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) =>
    total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
```

This matches the single-NFT behaviour already validated by the existing test (1 XCH + 2.5 % = 1.025 XCH) and extends it correctly to the multi-NFT case.

### Proof of Concept

1. Attacker pairs a dapp with the victim's wallet via WalletConnect.
2. Attacker calls `chia_wallet.take_offer` with an offer string whose decoded summary contains:
   - `requested.xch = "1000000000000"` (1 XCH)
   - `offered` = two NFT launcher IDs, each with `royalty_percentage: 1000` (10 %) in their `infos` entries.
3. `parseCommandDisplay` calls `offerSummaryRoyaltyPercentages`, producing `royaltyPercentages = [1000, 1000]`.
4. `formatAmountWithRoyalties` computes `splitAmount = 500_000_000_000n`, royalty per NFT = `50_000_000_000n`, total royalty = `100_000_000_000n`, displayed total = **1.1 XCH**.
5. The Confirm dialog shows **"Total Amount with Royalties: 1.1 XCH"**.
6. The victim approves; the blockchain deducts **1.2 XCH** (each NFT's 10 % applied to the full 1 XCH).
7. The victim's wallet is short 0.1 XCH relative to what the approval dialog indicated. [4](#0-3) [5](#0-4)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

```typescript
function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}
