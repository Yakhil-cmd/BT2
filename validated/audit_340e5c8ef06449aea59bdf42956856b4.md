### Title
Gateway Stateful Validator Ignores `l1_gas` and `l1_data_gas` Price Bounds in Resource Bounds Admission Check — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful and stateless resource-bounds validators only inspect `l2_gas.max_price_per_unit` against a minimum threshold. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields of an `AllResourceBounds` transaction are silently ignored at admission time. Because the blockifier's `check_fee_bounds` does enforce all three gas-price bounds during pre-validation, any transaction whose L1 or L1DataGas price is below the current block price will pass gateway admission, enter the mempool, and then be rejected by the blockifier — the gateway invariant "admit only transactions that can pass pre-validation" is broken.

### Finding Description

**Stateless path** — `StatelessTransactionValidator::validate_resource_bounds`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs  lines 56-88
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { … });
}
// l1_gas.max_price_per_unit and l1_data_gas.max_price_per_unit are never read here.
```

Only `l2_gas.max_price_per_unit` is compared against the static `min_gas_price` config. `l1_gas` and `l1_data_gas` price bounds are not touched. [1](#0-0) 

**Stateful path** — `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold`:

The developer-acknowledged TODO confirms the gap: `l1_gas` and `l1_data_gas` prices are never compared against the previous block's corresponding prices. [2](#0-1) 

**Blockifier enforcement** — `AccountTransaction::check_fee_bounds` (called from `perform_pre_validation_stage`):

```rust
// crates/blockifier/src/transaction/account_transaction.rs  lines 398-458
ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_gas_resource_bounds,
    l2_gas: l2_gas_resource_bounds,
    l1_data_gas: l1_data_gas_resource_bounds,
}) => {
    // checks max_price_per_unit < actual_gas_price for ALL THREE resources
    if resource_bounds.max_price_per_unit < actual_gas_price.get() {
        insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { … });
    }
}
```

The blockifier enforces all three price bounds. A transaction with `l1_gas.max_price_per_unit = 0` will be rejected here with `InsufficientResourceBounds` → `MaxGasPriceTooLow { resource: L1Gas }`. [3](#0-2) 

The gap between the two layers is the root cause: the gateway admits what the blockifier will reject.

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An unprivileged user can submit `AllResourceBounds` invoke/declare/deploy-account transactions with a valid `l2_gas.max_price_per_unit` (to pass both gateway checks) but with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0`. These transactions:

1. Pass `StatelessTransactionValidator::validate_resource_bounds` (only l2_gas price checked).
2. Pass `StatefulTransactionValidator::validate_resource_bounds` (only l2_gas price checked against previous block threshold).
3. Enter the mempool.
4. Are pulled by the batcher and fail `check_fee_bounds` with `MaxGasPriceTooLow` for L1Gas, wasting batcher execution cycles and occupying mempool slots.

Because L1 gas prices are non-zero in production today, every such transaction is guaranteed to fail execution. The gateway's admission invariant is broken in the current live configuration, not merely as a future concern.

### Likelihood Explanation

High. The attack requires no privilege, no special account, and no on-chain state. Any caller of `POST /add_transaction` can craft a V3 `AllResourceBounds` transaction with zero L1/L1DataGas prices. The stateless check is purely local and the stateful check only reads the previous block's L2 gas price — neither touches L1 gas prices.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel helper) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK prices (applying the same `min_gas_price_percentage` threshold). The previous block's `strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price` are already available from `get_block_info()`. Similarly, extend `StatelessTransactionValidator::validate_resource_bounds` to apply `min_gas_price` to all three price fields, mirroring the completeness of `check_fee_bounds` in the blockifier.

### Proof of Concept

```
// Craft an invoke V3 transaction with AllResourceBounds:
resource_bounds = AllResourceBounds {
    l2_gas: ResourceBounds {
        max_price_per_unit: <current_l2_gas_price>,  // passes stateless + stateful check
        max_amount: 1,
    },
    l1_gas: ResourceBounds {
        max_price_per_unit: 0,   // NOT checked by either gateway validator
        max_amount: 1,
    },
    l1_data_gas: ResourceBounds {
        max_price_per_unit: 0,   // NOT checked by either gateway validator
        max_amount: 1,
    },
}

// Step 1: POST /add_transaction → StatelessTransactionValidator::validate_resource_bounds
//   → only l2_gas.max_price_per_unit checked → PASSES

// Step 2: StatefulTransactionValidator::validate_resource_bounds
//   → validate_tx_l2_gas_price_within_threshold checks only l2_gas price → PASSES

// Step 3: Transaction enters mempool.

// Step 4: Batcher pulls transaction, blockifier calls check_fee_bounds:
//   l1_gas.max_price_per_unit (0) < actual_l1_gas_price (non-zero)
//   → InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }
//   → Transaction fails pre-validation, nonce is NOT incremented.
//   → Mempool slot was wasted; batcher cycle was consumed.
```

Repeat at scale to degrade sequencer throughput without any on-chain cost.

        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
