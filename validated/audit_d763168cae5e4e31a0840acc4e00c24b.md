### Title
Payer Can Evade Transaction Fees via Cross-Transaction Balance Drain — (File: `fvm/transactionInvoker.go`)

---

### Summary

A transaction payer can avoid paying Flow transaction fees by draining their balance in a separate transaction (using a different proposer key) that executes before the target transaction. When the FVM's payer balance check fails at execution time, the subsequent fee deduction also fails silently in `errorExecution()`, and the transaction is committed to state with no fee deduction. The sequence number is still consumed. This is a direct analog of the external report's "requester ensures balance is 0 so the delivery transaction fails" scenario.

---

### Finding Description

The FVM transaction execution pipeline in `normalExecution()` follows this order:

1. **Balance check** (`CheckPayerBalanceAndReturnMaxFees`, line 334) — reads payer's current balance from state.
2. **Body execution** (`txnBodyExecutor.Execute()`, line 350) — runs the Cadence transaction script.
3. **Fee deduction** (`deductTransactionFees()`, line 398) — deducts fees from the payer. [1](#0-0) 

If the balance check at step 1 fails (returns `InsufficientPayerBalanceError`), `normalExecution()` returns early without executing the body. `ExecuteTransactionBody()` then calls `errorExecution()`: [2](#0-1) 

Inside `errorExecution()`, the nested transaction is restarted (discarding any preprocessing changes), and `deductTransactionFees()` is attempted again. If the payer's balance is still zero, this second deduction also fails. Critically, `TransactionFeeDeductionFailedError` is a non-fatal `CodedError`: [3](#0-2) 

Because it is non-fatal, `executor.errs.Collect(feesError).CollectedFailure()` returns `false`, the delta is dropped via a second `RestartNestedTransaction`, and execution falls through to `commit()`. The transaction is committed to state with **no fee deduction** and **no body execution**, but with the sequence number already incremented (which happened in `preprocess()` before `execute()` was called). [4](#0-3) 

The Access node's pre-submission balance check reads state at the indexed height, which can lag behind the execution height: [5](#0-4) 

This creates a TOCTOU window: the check passes at indexed height, but by execution time the payer's balance has been drained by a separately submitted transaction.

---

### Impact Explanation

A payer can avoid paying transaction fees (both inclusion and execution fees) for a transaction that was included in a block and consumed a sequence number. This violates the protocol's economic invariant that every included transaction must pay fees. At scale, repeated fee evasion reduces fee revenue to the network. If the payer uses a high `GasLimit` on the target transaction (T1), the avoided fee can be substantial relative to the cost of the drain transaction (T2).

---

### Likelihood Explanation

Flow natively supports multi-key accounts and allows the proposer key to differ from the payer. Submitting two transactions with different proposer keys is a standard, unprivileged operation. The attacker needs no special privileges, no admin keys, and no compromised nodes. The only requirement is controlling an account with at least two keys (or using a separate proposer account). This is a realistic, low-barrier attack.

---

### Recommendation

1. **Treat fee deduction failure as fatal in `errorExecution()`**: If `deductTransactionFees()` fails in the error path, the transaction should be treated as a fatal failure rather than silently dropping the delta. This ensures no transaction is ever committed without fee deduction.

2. **Reserve fees at inclusion time**: Analogous to the external report's recommendation, the protocol could lock the fee amount in the payer's account at the time the transaction is included in a collection, preventing the balance from being drained before execution.

3. **Atomic balance reservation**: The FVM could reserve `maxTxFees` from the payer's balance at the start of `normalExecution()` (before body execution), ensuring the fee amount is unavailable for the body to drain or for concurrent transactions to consume.

---

### Proof of Concept

```
1. Payer account P has balance B (sufficient for T1's fees, e.g., 0.001 FLOW).

2. Submit T1:
   - proposer = P.key0, seq = N
   - payer = P
   - gas_limit = 9999 (high fees)
   - body = no-op (or any script)
   Access node checks P's balance at indexed height → passes.

3. Submit T2 (before T1 is executed):
   - proposer = P.key1, seq = 0   ← different proposer key, independent ordering
   - payer = P
   - body = transfer all FLOW from P to P2 (another account P controls)
   T2 has low gas_limit → low fees.

4. T2 is included in a block before T1 (different proposer keys allow independent
   ordering across collections). T2 executes, drains P's balance to 0.

5. T1 executes:
   - normalExecution() → CheckPayerBalanceAndReturnMaxFees → InsufficientPayerBalanceError
   - normalExecution() returns early (body never runs)
   - errorExecution() called:
       - RestartNestedTransaction(nestedTxnId) → discards preprocessing changes
       - deductTransactionFees() → fails (P.balance == 0)
       - TransactionFeeDeductionFailedError is non-fatal → CollectedFailure() == false
       - RestartNestedTransaction(nestedTxnId) again → drops delta
   - commit() → commits empty delta

6. Final state:
   - P.key0 sequence number incremented (consumed)
   - No fee deduction from P
   - T1 body never executed
   - P's tokens are in P2 (still under P's control)

Net result: P paid fees only for T2 (cheap), avoided fees for T1 (expensive).
``` [6](#0-5) [7](#0-6)

func (executor *transactionExecutor) preprocess() error {
	if executor.AuthorizationChecksEnabled {
		err := executor.CheckAuthorization(
			executor.ctx.TracerSpan,
			executor.proc,
			executor.txnState,
			executor.AccountKeyWeightThreshold)
		if err != nil {
			executor.errs.Collect(err)
			return executor.errs.ErrorOrNil()
		}
	}

	if executor.SequenceNumberCheckAndIncrementEnabled {
		err := executor.CheckAndIncrementSequenceNumber(
			executor.ctx.TracerSpan,
			executor.proc,
			executor.txnState)
		if err != nil {
			executor.errs.Collect(err)
			return executor.errs.ErrorOrNil()
		}
	}

	if !executor.TransactionBodyExecutionEnabled {
		return nil
	}

	executor.errs.Collect(executor.preprocessTransactionBody())
	if executor.errs.CollectedFailure() {
		return executor.errs.ErrorOrNil()
	}

	return nil
}

func (executor *transactionExecutor) normalExecution() (
	invalidator derived.TransactionInvalidator,
	err error,
) {
	var maxTxFees uint64
	// run with limits disabled since this is a static cost check
	// and should be accounted for in the inclusion cost.
	executor.txnState.RunWithMeteringDisabled(func() {
		maxTxFees, err = executor.CheckPayerBalanceAndReturnMaxFees(
			executor.proc,
			executor.txnState,
			executor.env)
	})

	if err != nil {
		return
	}

	var bodyTxnId state.NestedTransactionId
	bodyTxnId, err = executor.txnState.BeginNestedTransaction()
	if err != nil {
		return
	}

	err = executor.txnBodyExecutor.Execute()
	if err != nil {
		err = fmt.Errorf("transaction execute failed: %w", err)
		return
	}

	// Before checking storage limits, we must apply all pending changes
	// that may modify storage usage.
	var contractUpdates environment.ContractUpdates
	contractUpdates, err = executor.env.FlushPendingUpdates()
	if err != nil {
		err = fmt.Errorf(
			"transaction invocation failed to flush pending changes from "+
				"environment: %w",
			err)
		return
	}

	var bodySnapshot *snapshot.ExecutionSnapshot
	bodySnapshot, err = executor.txnState.CommitNestedTransaction(bodyTxnId)
	if err != nil {
		return
	}

	invalidator = environment.NewDerivedDataInvalidator(
		contractUpdates,
		bodySnapshot,
		executor.executionStateRead,
	)

	// Check if all account storage limits are ok
	//
	// The storage limit check is performed for all accounts that were touched during the transaction.
	// The storage capacity of an account depends on its balance and should be higher than the accounts storage used.
	// The payer account is special cased in this check and its balance is considered max_fees lower than its
	// actual balance, for the purpose of calculating storage capacity, because the payer will have to pay for this tx.
	err = executor.CheckStorageLimits(
		executor.ctx,
		executor.env,
		bodySnapshot,
		executor.proc.Transaction.Payer,
		maxTxFees)

	if err != nil {
		return
	}

	executor.txnState.RunWithMeteringDisabled(func() {
		err = executor.deductTransactionFees()
	})

	return

// Clear changes and try to deduct fees again.
func (executor *transactionExecutor) errorExecution() {
	// log transaction as failed
	log := executor.env.Logger()
	log.Info().
		Err(executor.errs.ErrorOrNil()).
		Msg("transaction executed with error")

	executor.env.Reset()

	// drop delta since transaction failed
	restartErr := executor.txnState.RestartNestedTransaction(executor.nestedTxnId)
	if executor.errs.Collect(restartErr).CollectedFailure() {
		return
	}

	// try to deduct fees again, to get the fee deduction events
	feesError := executor.deductTransactionFees()

	// if fee deduction fails just do clean up and exit
	if feesError != nil {
		log.Info().
			Err(feesError).
			Msg("transaction fee deduction executed with error")

		if executor.errs.Collect(feesError).CollectedFailure() {
			return
		}

		// drop delta
		executor.errs.Collect(
			executor.txnState.RestartNestedTransaction(executor.nestedTxnId))
	}
}

func (v *TransactionValidator) checkSufficientBalanceToPayForTransaction(ctx context.Context, tx *flow.TransactionBody) error {
	if v.options.CheckPayerBalanceMode == Disabled {
		return nil
	}

	header, err := v.blocks.SealedHeader()
	if err != nil {
		return fmt.Errorf("could not fetch block header: %w", err)
	}

	indexedHeight, err := v.blocks.IndexedHeight()
	if err != nil {
		return fmt.Errorf("could not get indexed height: %w", err)
	}

	// we use latest indexed block to get the most up-to-date state data available for executing scripts.
	// check here to make sure indexing is within an acceptable tolerance of sealing to avoid issues
	// if indexing falls behind
	sealedHeight := header.Height
	if indexedHeight < sealedHeight-DefaultSealedIndexedHeightThreshold {
		return IndexedHeightFarBehindError{SealedHeight: sealedHeight, IndexedHeight: indexedHeight}
	}

	payerAddress := cadence.NewAddress(tx.Payer)
	inclusionEffort := cadence.UFix64(tx.InclusionEffort())
	gasLimit := cadence.UFix64(tx.GasLimit)

	args, err := cadenceutils.EncodeArgs([]cadence.Value{payerAddress, inclusionEffort, gasLimit})
	if err != nil {
		return fmt.Errorf("failed to encode cadence args for script executor: %w", err)
	}

	result, err := v.scriptExecutor.ExecuteAtBlockHeight(ctx, v.verifyPayerBalanceScript, args, indexedHeight)
	if err != nil {
		return fmt.Errorf("script finished with error: %w", err)
	}

	value, err := jsoncdc.Decode(nil, result)
	if err != nil {
		return fmt.Errorf("could not decode result value returned by script executor: %w", err)
	}

	canExecuteTransaction, requiredBalance, _, err := fvm.DecodeVerifyPayerBalanceResult(value)
	if err != nil {
		return fmt.Errorf("could not parse cadence value returned by script executor: %w", err)
	}

	// return no error if payer has sufficient balance
	if canExecuteTransaction {
		return nil
	}

	return InsufficientBalanceError{Payer: tx.Payer, RequiredBalance: requiredBalance}
}

func (_ TransactionPayerBalanceChecker) CheckPayerBalanceAndReturnMaxFees(
	proc *TransactionProcedure,
	txnState storage.TransactionPreparer,
	env environment.Environment,
) (uint64, error) {
	if !env.TransactionFeesEnabled() {
		// if the transaction fees are not enabled,
		// the payer can pay for the transaction,
		// and the max transaction fees are 0.
		// This is also the condition that gets hit during bootstrapping.
		return 0, nil
	}

	var resultValue cadence.Value
	var err error
	txnState.RunWithMeteringDisabled(func() {
		// Don't meter the payer balance check.
		// It has a static cost, and its cost should be part of the inclusion fees, not part of execution fees.
		resultValue, err = env.CheckPayerBalanceAndGetMaxTxFees(
			proc.Transaction.Payer,
			proc.Transaction.InclusionEffort(),
			uint64(txnState.TotalComputationLimit()),
		)
	})
	if err != nil {
		return 0, errors.NewPayerBalanceCheckFailure(proc.Transaction.Payer, err)
	}

	payerCanPay, requiredBalance, maxFees, err := DecodeVerifyPayerBalanceResult(resultValue)
	if err != nil {
		return 0, errors.NewPayerBalanceCheckFailure(proc.Transaction.Payer, err)
	}

	if !payerCanPay {
		return 0, errors.NewInsufficientPayerBalanceError(proc.Transaction.Payer, requiredBalance)
	}

	return uint64(maxFees), nil
}
