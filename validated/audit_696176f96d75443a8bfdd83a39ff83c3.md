### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Block Per-Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary
`update_rewards` in the staking contract enforces a single-call-per-block gate via `last_reward_block`, but has no caller restriction. Any unprivileged address can call it with `disable_rewards: true`, consuming the block's reward slot and updating `last_reward_block` without distributing any rewards. The sequencer's legitimate call for that block then reverts with `REWARDS_ALREADY_UPDATED`, permanently destroying that block's yield.

### Finding Description
`update_rewards` is the sole mechanism for distributing per-block consensus rewards to stakers and their delegation pools. The function enforces a global one-call-per-block invariant by asserting `current_block_number > last_reward_block` and then immediately writing `last_reward_block = current_block_number` — **before** checking the `disable_rewards` flag. [1](#0-0) 

The only access guard is `general_prerequisites`, which checks only that the contract is unpaused and the caller is non-zero: [2](#0-1) 

The spec documents the intended access control as "Only starkware sequencer": [3](#0-2) 

But no such check exists in the implementation. The execution path is:

1. Attacker calls `update_rewards(valid_staker, disable_rewards: true)` in block N.
2. `last_reward_block` is written to N at line 1485.
3. Function returns early at line 1487–1489 — zero rewards distributed.
4. Sequencer's legitimate `update_rewards(staker, disable_rewards: false)` for block N reverts: `REWARDS_ALREADY_UPDATED`.
5. Block N's rewards are permanently lost. [4](#0-3) 

The attacker only needs to supply any currently-active staker address with non-zero balance (publicly readable from chain state).

### Impact Explanation
Every griefed block permanently destroys the STRK rewards that would have been distributed to the targeted staker and all their delegation pool members. Sustained griefing (e.g., every block) causes **permanent freezing of all unclaimed yield** for the protocol. This matches the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

### Likelihood Explanation
The attack requires no special privilege, no capital, and no leaked key — only a valid staker address (public on-chain) and a transaction submitted in the same block as the sequencer's `update_rewards`. On Starknet, transaction ordering within a block is sequencer-controlled, but the attacker can submit the call at any time; the sequencer may not always front-run it. Repeated automated submission makes sustained griefing realistic.

### Recommendation
Add a sequencer-only access control check at the top of `update_rewards`, consistent with the spec's stated intent:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();
    self.roles.only_sequencer(); // or equivalent role check
    ...
}
```

Alternatively, separate the `last_reward_block` write so it only occurs after confirming the caller is authorized, preventing unauthorized consumption of the per-block reward slot.

### Proof of Concept

```
Block N arrives.

Attacker (any EOA):
  → calls staking.update_rewards(staker=<any_active_staker>, disable_rewards=true)
  → general_prerequisites() passes (contract unpaused, caller != 0)
  → assert current_block_number(N) > last_reward_block(N-1) → passes
  → last_reward_block.write(N)          ← slot consumed
  → disable_rewards == true → return    ← no rewards distributed

Sequencer (legitimate):
  → calls staking.update_rewards(staker=<staker>, disable_rewards=false)
  → assert current_block_number(N) > last_reward_block(N) → FAILS
  → reverts: REWARDS_ALREADY_UPDATED

Result: Block N rewards permanently lost for all stakers and delegators.
``` [5](#0-4) [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
