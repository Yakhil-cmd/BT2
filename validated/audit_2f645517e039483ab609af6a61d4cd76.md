### Title
Commission Increase Without Prior Reward Settlement Allows Staker to Steal Pool Members' Unclaimed Yield - (File: src/staking/staking.cairo)

### Summary
In `set_commission`, when a staker increases their commission rate using an active commission commitment, the pending epoch rewards are not settled at the old rate before the new commission takes effect. In V2 (pre-consensus) mode, rewards are calculated once per epoch at attestation time. A staker can increase commission mid-epoch and then attest, causing the entire epoch's pool rewards to be split using the new higher commission — retroactively stealing yield that pool members earned under the lower rate.

### Finding Description
The `set_commission` function in `src/staking/staking.cairo` delegates to `update_commission` when a commission already exists. `update_commission` validates the new commission value and writes it directly to storage with no prior reward settlement:

```cairo
// Update commission in storage.
staker_pool_info.commission.write(Option::Some(commission));
``` [1](#0-0) 

There is no call to `update_rewards_from_attestation_contract` or any equivalent reward-settlement step before the commission is overwritten.

Commission can be **increased** (not just decreased) when an active `CommissionCommitment` exists. `set_commission_commitment` allows `max_commission >= current_commission`, and with an active commitment `set_commission` accepts any value `<= max_commission` (and `!= current_commission`):

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission, ...
    );
    assert!(commission != old_commission, ...);
``` [2](#0-1) 

The code itself acknowledges this gap with the comment:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [3](#0-2) 

In V2 (pre-consensus) mode, `update_rewards_from_attestation_contract` is the sole reward-settlement path and is called **once per epoch** at attestation time:

```cairo
fn update_rewards_from_attestation_contract(
    ref self: ContractState, staker_address: ContractAddress,
) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
    ...
    self._update_rewards(...);
}
``` [4](#0-3) 

`_update_rewards` reads the commission from `staker_pool_info` at call time — after the commission has already been raised — and applies it to the **entire epoch's** pool rewards: [5](#0-4) 

The pool's `cumulative_rewards_trace` is updated only when rewards are transferred to the pool contract via `update_rewards_from_staking_contract`:

```cairo
self.cumulative_rewards_trace.insert(
    key: self.get_current_epoch(),
    value: last + self.compute_rewards_per_unit(
        staking_rewards: rewards, total_stake: pool_balance,
    ),
);
``` [6](#0-5) 

Because no settlement occurs at the moment of commission change, the `cumulative_rewards_trace` is never updated at the old rate for the pre-change portion of the epoch. Pool members' reward calculations therefore use only the new (higher) commission for the full epoch.

### Impact Explanation
**High — Theft of unclaimed yield.** Pool members who delegated under a lower commission rate lose the yield they accrued for the portion of the epoch before the commission increase. The staker captures the difference as additional commission. For a pool with significant delegated stake, the stolen amount equals `(new_commission - old_commission) / COMMISSION_DENOMINATOR * epoch_pool_rewards`, which can be material over a full epoch.

### Likelihood Explanation
The attack requires a staker to:
1. Have an active pool with delegators.
2. Call `set_commission_commitment` with `max_commission > current_commission` — a publicly available function callable by any staker with a pool.
3. Wait for the commitment to become active.
4. Call `set_commission` (increasing) at any point before attesting in the current epoch.
5. Attest, triggering reward settlement at the inflated commission.

Steps 1–4 are entirely within the staker's control and require no privileged access. The staker can repeat this every epoch for as long as the commitment is valid (up to one year per commitment). Likelihood is **medium-high** given the deliberate setup required but the clear financial incentive.

### Recommendation
Before writing the new commission to storage in `update_commission`, settle any pending rewards for the current epoch at the existing commission rate. Concretely, call the equivalent of `update_rewards_from_attestation_contract` (or an internal reward-flush helper) prior to overwriting `staker_pool_info.commission`. This mirrors the fix described in the reference report: update the share/reward index at the current rate before changing the rate.

### Proof of Concept
1. Staker stakes and opens a STRK pool; delegators enter.
2. Staker calls `set_commission_commitment(max_commission: current + 500, expiration_epoch: current + 10)`.
3. Epoch advances; commitment becomes active.
4. Near the end of epoch `E`, staker calls `set_commission(commission: current + 500)` — commission is written to storage immediately, no rewards settled.
5. Staker attests within the attestation window of epoch `E`.
6. `update_rewards_from_attestation_contract` → `_update_rewards` → `calculate_staker_pools_rewards` reads the new (higher) commission and applies it to the full epoch `E` pool rewards.
7. Pool's `cumulative_rewards_trace` is updated with the reduced net pool reward (after higher commission deduction).
8. Pool members claim rewards and receive less than they earned for the pre-change portion of epoch `E`; the staker receives the difference as commission.

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
