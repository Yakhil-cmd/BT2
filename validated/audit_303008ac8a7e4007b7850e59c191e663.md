### Title
Silent Fee Bypass in `errorExecution()` When Fallback `deductTransactionFees()` Fails with Non-Fatal Error - (File: `fvm/transactionInvoker.go`)

---

### Summary

In `fvm/transactionInvoker.go`, the FVM uses a two-phase fee deduction pattern analogous to the `_postOp` callback in the external report. When the primary fee deduction in `normalExecution()` cannot run (because the transaction body failed), a fallback fee deduction is attempted in `errorExecution()`. If this fallback `deductTransactionFees()` call fails, the error is classified as **non-fatal** (`TransactionFeeDeductionFailedError` is a `CodedError`, not a `CodedFailure`), causing `CollectedFailure()` to return `false`. Execution then falls through to `RestartNestedTransaction`, which **drops the fee deduction delta entirely**. The transaction is committed with no fees charged.

---

### Finding Description

`ExecuteTransactionBody()` implements a two-phase fee deduction:

**Phase 1 — `normalExecution()`** (lines 326–401):
- Calls `CheckPayerBalanceAndReturnMaxFees` (on-chain pre-check)
- Executes the transaction body
- Calls `deductTransactionFees()` on success

**Phase 2 — `errorExecution()`** (lines 404–437, called when Phase 1 fails):

```go
// try to deduct fees again, to get the fee deduction events
feesError := executor.deductTransactionFees()

// if fee deduction fails just do clean up and exit
if feesError != nil {
    if executor.errs.Collect(feesError).CollectedFailure() {
        return
    }
    // drop delta
    executor.errs.Collect(
        executor.txnState.RestartNestedTransaction(executor.nestedTxnId))
}
```

`deductTransactionFees()` wraps any failure as `NewTransactionFeeDeductionFailedError`, which returns a `CodedError` (non-fatal):

```go
func NewTransactionFeeDeductionFailedError(...) CodedError {
    return WrapCodedError(ErrCodeTransactionFeeDeductionFailedError, ...)
}
```

Because `IsFailure()` returns `false` for `CodedError`, `CollectedFailure()` returns `false`, and execution falls through to `RestartNestedTransaction(executor.nestedTxnId)`, which drops the fee deduction state delta. The transaction is then committed via `commit()` with no fees charged.

The `ErrorsCollector.Collect()` logic confirms this:

```go
if IsFailure(err) {
    collector.failure = err   // only fatal errors set this
    return collector
}
collector.errors = multierror.Append(collector.errors, err)  // non-fatal: just logged
return collector
```

The concrete trigger path:
1. Payer account has zero or insufficient FLOW balance.
2. Transaction reaches an execution node (Access node `CheckPayerBalanceMode` is `Disabled` or `WarnCheck`, or a race condition exists between the indexed-height check and actual execution).
3. On-chain `CheckPayerBalanceAndReturnMaxFees` fails → `normalExecution()` returns `ErrCodeInsufficientPayerBalance` (non-fatal).
4. `errorExecution()` is invoked.
5. `deductTransactionFees()` fails (payer has no balance) → `TransactionFeeDeductionFailedError` (non-fatal).
6. `CollectedFailure()` → `false` → falls through to `RestartNestedTransaction` → fee delta dropped.
7. `commit()` is called; transaction is finalized with no fees deducted.

---

### Impact Explanation

An unprivileged transaction sender can submit transactions that are fully processed by execution nodes without paying any fees. Fee collectors (the protocol's `FlowFees` contract recipients) are deprived of fees they are entitled to. Because the execution node performs real work (running the pre-check, state transitions, event emission) without compensation, this is equivalent to the paymaster-theft scenario in the external report: the protocol subsidizes execution while the payer pays nothing.

---

### Likelihood Explanation

The `CheckPayerBalanceMode` on Access nodes is configurable (`Disabled`, `WarnCheck`, `EnforceCheck`). The `WarnCheck` mode explicitly accepts and forwards transactions from underfunded payers. Even with `EnforceCheck`, the check is performed against the **indexed height**, which can lag behind the sealed height by up to `DefaultSealedIndexedHeightThreshold` blocks — creating a race window where a payer drains their balance after the check but before execution. Both conditions are reachable by an unprivileged transaction sender with no special privileges.

---

### Recommendation

1. **Treat fee deduction failure in `errorExecution()` as a fatal failure** (`CodedFailure`) rather than a non-fatal `CodedError`. This prevents the fee delta from being silently dropped and halts the transaction with a protocol-level failure.
2. Alternatively, if the fee deduction fails in the fallback path, do **not** call `RestartNestedTransaction` — preserve the (empty) fee deduction attempt so the committed state reflects the failure.
3. Harden the on-chain `CheckPayerBalanceAndReturnMaxFees` pre-check to be the authoritative gate: if it fails, the transaction should be rejected outright (not routed to `errorExecution()` where fee deduction is silently skipped).

---

### Proof of Concept

```
1. Create a Flow account with 0 FLOW balance (payer).
2. Submit a valid transaction signed by payer, with any script body.
3. Ensure the Access node has CheckPayerBalanceMode = WarnCheck or Disabled
   (or exploit the indexed-height lag window with EnforceCheck).
4. Execution node receives the transaction.
5. normalExecution() → CheckPayerBalanceAndReturnMaxFees fails →
   ErrCodeInsufficientPayerBalance (non-fatal CodedError).
6. errorExecution() is called.
7. deductTransactionFees() → FlowFees.deductTransactionFee fails (zero balance) →
   TransactionFeeDeductionFailedError (non-fatal CodedError).
8. CollectedFailure() == false → RestartNestedTransaction drops fee delta.
9. commit() finalizes the transaction. No FlowToken.TokensWithdrawn event
   for fees is emitted. Payer balance unchanged at 0.
10. Repeat indefinitely — execution nodes process transactions for free.
```

func (executor *transactionExecutor) ExecuteTransactionBody() error {
	var invalidator derived.TransactionInvalidator
	if !executor.errs.CollectedError() {

		var txError error
		invalidator, txError = executor.normalExecution()
		if executor.errs.Collect(txError).CollectedFailure() {
			return executor.errs.ErrorOrNil()
		}
	}

	if executor.errs.CollectedError() {
		invalidator = nil
		executor.txnState.RunWithMeteringDisabled(executor.errorExecution)
		if executor.errs.CollectedFailure() {
			return executor.errs.ErrorOrNil()
		}
	}

	// log the execution intensities here, so that they do not contain data
	// from transaction fee deduction, because the payer is not charged for that.
	executor.logExecutionIntensities()

	executor.errs.Collect(executor.commit(invalidator))

	return executor.errs.ErrorOrNil()
}

func (executor *transactionExecutor) deductTransactionFees() (err error) {
	if !executor.env.TransactionFeesEnabled() {
		return nil
	}

	computationLimit := executor.txnState.TotalComputationLimit()

	computationUsed, err := executor.env.ComputationUsed()
	if err != nil {
		return errors.NewTransactionFeeDeductionFailedError(
			executor.proc.Transaction.Payer,
			computationLimit,
			err)
	}

	if computationUsed > computationLimit {
		computationUsed = computationLimit
	}

	_, err = executor.env.DeductTransactionFees(
		executor.proc.Transaction.Payer,
		executor.proc.Transaction.InclusionEffort(),
		computationUsed)

	if err != nil {
		return errors.NewTransactionFeeDeductionFailedError(
			executor.proc.Transaction.Payer,
			computationUsed,
			err)
	}
	return nil
}

// Clear changes and try to deduct fees again.
func (executor *transactionExecutor) errorExecution() {
	// log transaction as failed
	log := executor.env.Logger()
	log.Info().
		Err(executor.errs.ErrorOrNil()).
		Msg("transaction executed with error")

	executor.env.Reset()

	// drop delta since transaction failed
	restartErr := executor.txnState.RestartNestedTransaction(executor.nestedTxnId)
	if executor.errs.Collect(restartErr).CollectedFailure() {
		return
	}

	// try to deduct fees again, to get the fee deduction events
	feesError := executor.deductTransactionFees()

	// if fee deduction fails just do clean up and exit
	if feesError != nil {
		log.Info().
			Err(feesError).
			Msg("transaction fee deduction executed with error")

		if executor.errs.Collect(feesError).CollectedFailure() {
			return
		}

		// drop delta
		executor.errs.Collect(
			executor.txnState.RestartNestedTransaction(executor.nestedTxnId))
	}
}

// This returns collector for chaining purposes.
func (collector *ErrorsCollector) Collect(err error) *ErrorsCollector {
	if err == nil {
		return collector
	}

	if collector.failure != nil {
		collector.errors = multierror.Append(collector.errors, err)
		return collector
	}

	if IsFailure(err) {
		collector.failure = err
		return collector
	}

	collector.errors = multierror.Append(collector.errors, err)
	return collector
