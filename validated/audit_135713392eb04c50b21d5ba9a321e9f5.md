### Title
Unprivileged Caller Can Permanently Suppress Per-Block Reward Distribution via `disable_rewards=true` — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is publicly callable with no access control beyond the generic pause/zero-address check. When called with `disable_rewards = true`, it writes the current block number into the global `last_reward_block` storage slot and returns early — without distributing any rewards. Because the slot is global and the function enforces `current_block_number > last_reward_block`, a single attacker transaction per block permanently forfeits that block's rewards for every staker.

---

### Finding Description

The V2 reward path (`update_rewards_from_attestation_contract`) and the V3 consensus reward path (`update_rewards`) are structurally parallel functions — both accept a `staker_address`, validate it, and call the shared `_update_rewards` helper. The V2 function is correctly gated:

```cairo
fn update_rewards_from_attestation_contract(
    ref self: ContractState, staker_address: ContractAddress,
) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
    self.assert_caller_is_attestation_contract();   // ← access control
    ...
}
``` [1](#0-0) 

The V3 analogue omits any equivalent restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks pause + non-zero caller
    ...
    self.last_reward_block.write(current_block_number);   // ← written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;   // ← exits without distributing rewards
    }
    ...
}
``` [2](#0-1) 

The global `last_reward_block` is written **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

The guard at the top of the function then prevents any second call in the same block:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

The interface exposes `update_rewards` with no documented caller restriction: [5](#0-4) 

---

### Impact Explanation

An attacker calls `update_rewards(any_active_staker, disable_rewards=true)` once per block. This:

1. Passes all staker-validity checks (any publicly known staker address with non-zero balance suffices — the staker list is readable via `get_stakers`).
2. Writes `last_reward_block = current_block`.
3. Returns without distributing STRK or BTC block rewards.
4. Causes every subsequent legitimate call to `update_rewards` in that block to revert with `REWARDS_ALREADY_UPDATED`.

Because `last_reward_block` is a single global slot, one transaction per block suppresses rewards for **all** stakers in that block. Sustained over many blocks this constitutes permanent freezing of unclaimed yield — a **High** impact under the allowed scope.

---

### Likelihood Explanation

- No privileged role is required; any EOA or contract can call `update_rewards`.
- The attacker only needs to know one active staker address (trivially obtained from `get_stakers` or on-chain events).
- On Starknet, per-transaction costs are low, making a sustained per-block attack economically viable.
- The attack is front-runnable: the attacker monitors the mempool for legitimate `update_rewards` calls and front-runs them with `disable_rewards=true`.

---

### Recommendation

Apply the same access-control pattern used in the V2 path. Either:

1. **Restrict the caller** — add an `assert_caller_is_consensus_contract()` guard analogous to `assert_caller_is_attestation_contract()` in `update_rewards_from_attestation_contract`.
2. **Separate concerns** — move the `last_reward_block` write to after the `disable_rewards` branch so that a `disable_rewards=true` call does not consume the block's reward slot.
3. **Remove the public `disable_rewards` parameter** — if the flag is only needed internally, make it an internal function and expose only the no-disable variant publicly.

---

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(staker = <any active staker>, disable_rewards = true)
    → last_reward_block := N
    → returns early, zero rewards distributed

  Consensus tx: update_rewards(staker = <target>, disable_rewards = false)
    → assert!(N > N)  ← FAILS with REWARDS_ALREADY_UPDATED
    → staker receives 0 rewards for block N

Block N+1, N+2, …: attacker repeats → all stakers permanently frozen out of block rewards.
```

The attacker-controlled entry path is the public `IStakingRewardsManager::update_rewards` entrypoint. [2](#0-1) 
The root cause is the missing access-control check that exists in the structurally identical V2 function. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
