### Title
ETH Permanently Locked in `RewardSupplier` When `tick()` Called With No Pending Mint Requests — (`L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

`tick()` is declared `external payable` but only forwards `msg.value` inside the `if (amountToMint > 0)` branch. When there are no pending L2 mint-request messages, the branch is skipped entirely, `msg.value` is silently accepted, and no refund or withdrawal path exists anywhere in the contract or its parents. Any ETH sent in that call is permanently locked.

---

### Finding Description

`tick()` at line 107 is `payable` with no access control: [1](#0-0) 

`msg.value` is only consumed inside the `if (amountToMint > 0)` block: [2](#0-1) 

When `requiredMinting()` returns `(0, 0)` — which happens whenever `messagingContract().l2ToL1Messages(msgHash) == 0` — the entire block is skipped: [3](#0-2) 

A full search of all L1 Solidity files confirms there is no `receive()`, `fallback()`, `withdraw`, or ETH-rescue function anywhere in `RewardSupplier.sol`, `RewardSupplierStorage.sol`, or `ProxySupportImpl.sol`: [4](#0-3) [5](#0-4) 

---

### Impact Explanation

Any caller — privileged or not — who sends ETH to `tick()` when no L2 mint requests are pending loses that ETH permanently. There is no on-chain path to recover it. This matches **Medium: Griefing with no profit motive but damage to users or protocol**.

---

### Likelihood Explanation

`tick()` is a public, permissionless keeper function. Off-chain bots and integrators are expected to call it with ETH to cover StarkNet messaging fees. A bot that calls `tick{value: fee}()` without first checking `requiredMinting()` off-chain will lose its ETH silently. This is a realistic operational scenario, not a contrived one.

---

### Recommendation

Add a refund for unused ETH at the end of `tick()`:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();
    if (amountToMint > 0) {
        // ... existing logic ...
    } else if (msg.value > 0) {
        // Refund ETH when there is nothing to mint.
        (bool ok, ) = msg.sender.call{value: msg.value}("");
        require(ok, "ETH_REFUND_FAILED");
    }
}
```

Alternatively, revert when `msg.value > 0` and `amountToMint == 0`, making the no-op case explicit.

---

### Proof of Concept

```solidity
// 1. Deploy RewardSupplier (proxy) with a messaging contract that has 0 pending messages.
// 2. Call tick with ETH:
rewardSupplier.tick{value: 1 ether}();
// 3. Assert ETH is locked:
assert(address(rewardSupplier).balance == 1 ether);
// 4. Confirm no withdrawal path exists — no withdraw(), receive(), or fallback() function.
```

The `if (amountToMint > 0)` guard at line 111 is never entered, `msg.value` is never forwarded or refunded, and the ETH is irrecoverable. [6](#0-5)

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
