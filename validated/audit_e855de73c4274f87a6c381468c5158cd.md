### Title
Governance `CronosAdmin` replacement does not clear pre-existing permissioned addresses, allowing a malicious admin to retain token-mapping control after emergency replacement - (File: x/cronos/keeper/msg_server.go)

---

### Summary

When governance replaces a malicious `CronosAdmin` via `MsgUpdateParams`, the handler calls `SetParams`, which only overwrites the params blob. It never iterates or clears the KV-store entries written by prior `MsgUpdatePermissions` calls. Any addresses the malicious admin pre-granted `CanChangeTokenMapping` to retain that permission indefinitely, allowing them to corrupt denom/contract bindings after the emergency admin replacement takes effect.

---

### Finding Description

**Permission grant path**

`MsgUpdatePermissions` is gated to the current `CronosAdmin` and writes a `uint64` bitmask into the KV store under `AdminToPermissionsKey(address)`: [1](#0-0) 

`HasPermission` then checks **both** the current `CronosAdmin` param **and** any address whose stored bitmask satisfies the required bits: [2](#0-1) 

**Emergency replacement path**

Governance replaces a malicious admin by passing `MsgUpdateParams` with a new `CronosAdmin` value. The handler calls `SetParams`, which only marshals and stores the new params blob: [3](#0-2) [4](#0-3) 

`SetParams` performs no iteration over `AdminToPermissionsKey` entries. All previously granted permissions survive the admin change untouched.

**Exploitation**

A malicious `CronosAdmin` can pre-grant `CanChangeTokenMapping` to addresses they control before governance acts. Because the Cosmos SDK governance voting period is public and takes time, the attacker has a reliable window to submit `MsgUpdatePermissions` transactions. After governance replaces the admin, those pre-granted addresses still satisfy `HasPermission` and can call `MsgUpdateTokenMapping`: [5](#0-4) 

`MsgUpdateTokenMapping` with an empty `contract` field deletes an existing denom→contract mapping; a subsequent call with a malicious contract address re-maps the denom. The `OnRecvVouchers` path then converts incoming IBC vouchers to the attacker-controlled ERC20 contract: [6](#0-5) 

---

### Impact Explanation

After governance executes the emergency admin replacement, the attacker's pre-granted addresses can:

1. Delete legitimate IBC-voucher → CRC20 mappings via `MsgUpdateTokenMapping` with empty contract.
2. Re-register those denoms to attacker-controlled ERC20 contracts.
3. All subsequent `OnRecvVouchers` calls for those denoms will mint tokens into the malicious contract instead of the legitimate one, permanently corrupting IBC voucher accounting for affected users.

This satisfies **High: Corruption of token mappings, denom/contract binding with direct security impact** and **High: Bypass of Cronos admin / governance authority checks**, because the governance emergency action is structurally undermined by surviving KV-store state.

---

### Likelihood Explanation

The Cosmos SDK governance voting period is public and on-chain. A malicious `CronosAdmin` observing a pending `MsgUpdateParams` proposal (or simply acting pre-emptively) can submit `MsgUpdatePermissions` for any number of addresses they control before the proposal passes. No specialized knowledge or timing precision is required; the attacker only needs to act before the governance proposal's voting period ends.

---

### Recommendation

`UpdateParams` should detect when `CronosAdmin` is changing and clear all `AdminToPermissionsKey` entries from the KV store before writing the new params. A helper that iterates the `AdminToPermissionsKey` prefix and deletes every entry is sufficient. This mirrors the recommendation in the external report: the emergency replacement function should clear all delegated permissions granted by the outgoing admin. [7](#0-6) [3](#0-2)

```go
func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
```

```go
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

```go
func (k Keeper) SetParams(ctx sdk.Context, params types.Params) error {
	if err := params.Validate(); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&params)
	store.Set(types.ParamsKey, bz)

	return nil
}
```
