### Title
Missing Caller-Staker Association Check in `update_rewards` Enables Permanent Freezing of All Consensus Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract performs no check that the caller is associated with the supplied `staker_address`. Because the global `last_reward_block` variable is unconditionally written on every successful call, an unprivileged attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` every block to permanently prevent all stakers from receiving consensus rewards.

---

### Finding Description

`update_rewards` is a public function with no caller-identity restriction beyond `general_prerequisites` (not paused, caller non-zero): [1](#0-0) 

The function accepts an arbitrary `staker_address` and a caller-controlled `disable_rewards` boolean. After validating that the staker exists and is active, it unconditionally writes the current block number to the **global** `last_reward_block` storage slot: [2](#0-1) 

`last_reward_block` is a single global value, not per-staker: [3](#0-2) 

The guard at the top of the function enforces that only one call per block can succeed: [4](#0-3) 

When `disable_rewards` is `true`, the function returns immediately after writing `last_reward_block`, distributing nothing: [5](#0-4) 

There is no check that the caller is the staker's operational address, the staker itself, or any other entity associated with `staker_address`. The function is callable by any address.

**Analog to the reported vulnerability:** Just as `process_cpi_context` failed to verify that compressed accounts belonged to the `CpiContextAccount.associated_merkle_tree`, `update_rewards` fails to verify that the caller is associated with the `staker_address` it is acting on. In both cases, an unrelated party can inject inputs that corrupt a shared state variable, invalidating the integrity of the system for all participants.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block:

1. Consumes the single allowed `update_rewards` slot for that block (enforced by `last_reward_block`).
2. Distributes zero rewards to any staker.
3. Causes every legitimate call to `update_rewards` in that block to revert with `REWARDS_ALREADY_UPDATED`.

Repeated every block, this permanently prevents all consensus-phase reward accrual for every staker in the protocol. Unclaimed yield is frozen indefinitely.

---

### Likelihood Explanation

**High.** The function is fully public. The attacker needs only a valid, active staker address with non-zero balance (trivially obtained from on-chain events or `get_stakers`). The cost is one transaction per block. No privileged access, leaked key, or external dependency is required.

---

### Recommendation

Add an association check requiring the caller to be the staker's registered operational address (mirroring the attestation design):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Enforce caller is the staker's operational address.
    let staker_info = self.internal_staker_info(:staker_address);
    assert!(
        get_caller_address() == staker_info.operational_address,
        "{}",
        Error::CALLER_IS_NOT_OPERATIONAL_ADDRESS,
    );
    ...
}
```

This ensures only the entity legitimately associated with `staker_address` can consume the per-block reward slot and control the `disable_rewards` flag.

---

### Proof of Concept

1. Attacker observes any active staker `S` with non-zero STRK balance (e.g., from `NewStaker` events).
2. Each block, attacker submits: `staking.update_rewards(S, disable_rewards: true)`.
3. The call passes all guards (staker exists, is active, has balance, block is new).
4. `last_reward_block` is set to the current block; no rewards are distributed.
5. Any legitimate call to `update_rewards` for any staker in that block reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeated indefinitely, zero consensus rewards are ever distributed to any staker in the protocol. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
