### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Block Consensus Reward Distribution - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` has no access control enforcement despite the specification requiring "Only starkware sequencer." Because `last_reward_block` is a **global** (not per-staker) uniqueness guard, any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` at the start of every block, consuming the per-block slot and permanently preventing the sequencer from distributing consensus rewards to any staker.

---

### Finding Description

The `update_rewards` function in `StakingRewardsManagerImpl` enforces a global uniqueness constraint via `last_reward_block`:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global write

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits without distributing rewards
    }
    // ... distribute rewards
``` [1](#0-0) 

The specification explicitly states the access control for this function is **"Only starkware sequencer"**: [2](#0-1) 

However, the implementation only calls `self.general_prerequisites()`, which checks only the pause flag. There is no `self.roles.only_sequencer()` or equivalent guard.

The `last_reward_block` storage variable is **global** — a single value shared across all stakers: [3](#0-2) 

This means:
1. An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at block N.
2. `last_reward_block` is written to N; no rewards are distributed.
3. The sequencer's legitimate call to `update_rewards(target_staker, disable_rewards: false)` at block N reverts with `REWARDS_ALREADY_UPDATED`.
4. No staker earns rewards for block N.

The attacker repeats this every block, permanently starving all stakers of consensus rewards.

---

### Impact Explanation

This is a **permanent freezing of unclaimed yield** for all stakers. Once consensus rewards are active (`is_pre_consensus()` returns false), the only mechanism to distribute per-block rewards is `update_rewards`. An attacker who front-runs every block with `disable_rewards: true` ensures the sequencer can never distribute rewards. Stakers accumulate zero `unclaimed_rewards_own` indefinitely, matching the "Permanent freezing of unclaimed yield" impact category.

---

### Likelihood Explanation

The attack requires no privileged access, no capital at risk, and no complex setup. The attacker only needs to submit one cheap Starknet transaction per block (gas cost only). On Starknet, transaction fees are very low. The attack is fully permissionless and can be automated. Any motivated griever — including a competing validator or someone shorting the STRK token — can sustain this indefinitely.

---

### Recommendation

Add the missing access control check at the top of `update_rewards`, consistent with the specification:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer();   // <-- add this
    self.general_prerequisites();
    ...
```

Alternatively, if a dedicated sequencer role is not yet defined, restrict callers to a configurable sequencer address stored in contract state.

---

### Proof of Concept

```
Block N arrives:
  Attacker tx:   update_rewards(staker=Alice, disable_rewards=true)
                 → last_reward_block = N, no rewards distributed

  Sequencer tx:  update_rewards(staker=Alice, disable_rewards=false)
                 → ASSERT FAILS: current_block_number (N) > last_reward_block (N) is FALSE
                 → REWARDS_ALREADY_UPDATED, reverts

Block N+1 arrives:
  Attacker tx:   update_rewards(staker=Alice, disable_rewards=true)
                 → last_reward_block = N+1, no rewards distributed

  Sequencer tx:  update_rewards(staker=Alice, disable_rewards=false)
                 → REWARDS_ALREADY_UPDATED, reverts

... (repeated every block indefinitely)
```

All stakers earn zero consensus rewards for the entire duration of the attack. The attacker's only cost is Starknet gas per block.
