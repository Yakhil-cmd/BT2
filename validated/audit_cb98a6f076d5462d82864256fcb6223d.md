### Title
Unprivileged Caller Can Invoke `update_rewards` with `disable_rewards: true` to Permanently Block Per-Block Reward Distribution — (File: src/staking/staking.cairo)

---

### Summary

The public `update_rewards` function in the Staking contract accepts a caller-controlled `disable_rewards` boolean. When set to `true`, the function still writes the current block number to the global `last_reward_block` storage slot — but returns before distributing any rewards. Because `last_reward_block` is a single global gate (not per-staker), any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to permanently exhaust each block's reward slot without paying out any yield, denying rewards to every staker for that block.

---

### Finding Description

`update_rewards` is an unrestricted public function (no role check beyond `general_prerequisites`, which only asserts the contract is unpaused and the caller is non-zero):

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

    // ← global gate written unconditionally
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits with no rewards paid
    }
    ...
    self._update_rewards(...);
}
``` [1](#0-0) 

The critical ordering is:

1. `last_reward_block` is written to `current_block_number` (line 1485).
2. Only *then* is `disable_rewards` checked (line 1487).

Because `last_reward_block` is a **single global slot** shared across all stakers, any subsequent call to `update_rewards` in the same block — regardless of which staker it targets — will revert with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

There is no access control preventing an arbitrary address from supplying `disable_rewards: true`. The only precondition is that `staker_address` must be a currently active staker with non-zero STRK balance — information that is publicly observable on-chain.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` once per Starknet block:

- Marks every block as "already processed" without distributing any STRK block rewards.
- Prevents every legitimate staker from calling `update_rewards` in that block (they receive `REWARDS_ALREADY_UPDATED`).
- Accumulates zero `unclaimed_rewards_own` for all stakers and zero pool rewards for all delegation pools.

Sustained over time, this freezes all per-block yield accrual for the entire protocol. The attacker gains nothing financially; the damage is purely to stakers and delegators.

---

### Likelihood Explanation

**Medium-High.**

- Requires no privileged role, no leaked key, no bridge compromise.
- Requires only: (a) knowledge of any active staker address (publicly readable from `stakers` Vec or events), and (b) the ability to submit one transaction per block.
- On Starknet, transaction fees are low, making sustained block-by-block griefing economically feasible.
- The attack is fully permissionless and repeatable.

---

### Recommendation

1. **Restrict who may pass `disable_rewards: true`**: Only the staker's own address or their registered operational address should be permitted to disable rewards for a given staker.
2. **Decouple the gate write from the reward path**: Move `self.last_reward_block.write(current_block_number)` to *after* the `disable_rewards` guard, so that a no-op call does not consume the block's reward slot.
3. **Alternatively**, remove the `disable_rewards` parameter from the public interface entirely and handle the pre-consensus / migration path through a separate privileged function.

---

### Proof of Concept

```
// Attacker (any EOA) executes once per block:
staking_contract.update_rewards(
    staker_address: any_active_staker,   // publicly known
    disable_rewards: true,               // no access control
);
// Effect:
//   last_reward_block = current_block   ← slot consumed
//   _update_rewards() never called      ← no yield distributed
//
// All subsequent calls in this block by legitimate stakers:
//   assert!(current_block_number > last_reward_block) → PANIC: REWARDS_ALREADY_UPDATED
``` [3](#0-2) 

The attacker repeats this every block. No staker ever accumulates `unclaimed_rewards_own`; no pool ever receives `pool_rewards`. All yield is permanently frozen without any funds leaving the protocol.

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
