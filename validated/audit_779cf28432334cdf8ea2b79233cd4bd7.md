### Title
Staker Can Unilaterally Increase Commission to 100% After Delegation via Commitment Mechanism - (`src/staking/staking.cairo`)

---

### Summary

A staker can attract delegators with a low commission, then use `set_commission_commitment` + `set_commission` in two back-to-back transactions to raise commission to 100%, stealing all delegator yield for up to one year. This is a direct analog to the oracle-swap attack: a critical parameter agreed upon at delegation time is mutated unilaterally by one party to extract value from the other.

---

### Finding Description

Without a commitment, `set_commission` enforces that commission can only **decrease**: [1](#0-0) 

With an active commitment, `update_commission` drops that protection entirely and only checks two things: [2](#0-1) 

The commitment itself is set with no delay and no upper-bound restriction on `max_commission` beyond `COMMISSION_DENOMINATOR` (10 000 = 100%): [3](#0-2) 

The only constraint is `current_commission <= max_commission`, so a staker at 1% commission can immediately set `max_commission = 10000`. The commitment becomes active the moment it is written (active while `current_epoch < expiration_epoch`): [4](#0-3) 

The code itself acknowledges the gap: [5](#0-4) 

**Attack sequence (two transactions, same block):**

1. Staker advertises 1% commission and opens a pool; delegators join.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + epochs_in_year)`.
3. Staker immediately calls `set_commission(commission: 10000)`.
4. Commission is now 100%. All pool rewards flow to the staker as commission; delegators receive 0 yield.
5. Delegators must wait `exit_wait_window` (1 week) before they can recover principal. During that window — and for up to one year if they do not notice — all yield is stolen.

The commission is read live from the staking contract at reward-distribution time: [6](#0-5) [7](#0-6) 

So every reward update after the commission change sends 100% of pool rewards to the staker.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Delegators lose all accrued yield from the moment of the commission change until they complete the two-step exit (intent + action, minimum 1 week). A staker who sets `expiration_epoch = current_epoch + epochs_in_year` can sustain 100% commission for up to one year, draining the entire yield of every delegator in the pool.

---

### Likelihood Explanation

Any registered staker can execute this in two transactions with no special privileges. The only prerequisite is having an active pool with delegators. The attack is economically rational whenever the stolen yield exceeds the reputational cost of being identified on-chain. Because the commitment mechanism is a legitimate protocol feature, the transaction sequence raises no on-chain alarms.

---

### Recommendation

1. **Prohibit commitment-enabled commission increases above the commission at the time delegators joined**, or at minimum cap the single-step increase (e.g., no more than X% per epoch).
2. **Add a time-lock between setting a commitment and being allowed to increase commission** — at least one full `exit_wait_window` — so delegators can exit before the increase takes effect.
3. Alternatively, **restrict `max_commission` in `set_commission_commitment` to the current commission**, making the commitment a pure "I will not raise above current" guarantee rather than a license to raise.

---

### Proof of Concept

```
// Epoch N, staker has commission = 100 (1%), pool has delegators.

// Tx 1: set commitment with max = 10000 (100%), valid for 1 year
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: N + epochs_in_year);

// Tx 2: immediately raise commission to 100%
staking.set_commission(commission: 10000);

// From this point, every call to update_rewards / update_rewards_from_attestation_contract
// routes 100% of pool rewards to the staker via split_rewards_with_commission.
// Delegators earn zero yield until they complete exit_delegation_pool_intent +
// exit_delegation_pool_action (minimum exit_wait_window = 1 week).
```

The existing test `test_set_commission_with_commitment` already demonstrates that a staker can raise commission to `max_commission` in a single step immediately after setting the commitment: [8](#0-7)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}

fn test_set_commission_with_commitment() {
    let mut cfg: StakingInitConfig = Default::default();
    let staking_contract = deploy_staking_contract(:cfg);
    cfg.test_info.staking_contract = staking_contract;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    stake_with_strk_pool_enabled(:cfg);

    // Set commitment.
    let staker_address = cfg.test_info.staker_address;
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    let max_commission = staker_info.get_pool_info().commission + 2;
    let expiration_epoch = staking_dispatcher.get_current_epoch() + 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission_commitment(:max_commission, :expiration_epoch);

    // Update commission.
    let mut commission = max_commission;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission(:commission);

    // Assert commission is updated.
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info.get_pool_info().commission == commission);

    // Advance to the expiration epoch.
    advance_epoch_global();

    // Lower commission.
    commission = commission - 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission(:commission);

    // Assert commission is updated.
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info.get_pool_info().commission == commission);
}
