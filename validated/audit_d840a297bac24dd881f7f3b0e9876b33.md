### Title
Missing Caller Authorization in `update_rewards` Allows Any Address to Freeze or Steal Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `Staking.cairo` enforces no restriction on who may call it. Unlike its pre-consensus counterpart `update_rewards_from_attestation_contract`, which explicitly asserts `assert_caller_is_attestation_contract()`, `update_rewards` accepts calls from any non-zero address. Because a single global `last_reward_block` slot gates all reward updates to one per block, an unprivileged caller can either (a) permanently deny all stakers their consensus-phase block rewards by calling with `disable_rewards: true` every block, or (b) a registered staker can call it for themselves every block to claim block rewards they did not earn, starving all other stakers.

### Finding Description

`update_rewards_from_attestation_contract` (the pre-consensus path) correctly restricts its caller:

```cairo
fn update_rewards_from_attestation_contract(
    ref self: ContractState, staker_address: ContractAddress,
) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
    self.assert_caller_is_attestation_contract();   // ← access control present
    ...
}
``` [1](#0-0) 

The consensus-phase counterpart `update_rewards` has no equivalent guard:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    ...
    self._update_rewards(...);
}
``` [2](#0-1) 

`general_prerequisites` only checks the pause flag and a non-zero caller: [3](#0-2) 

The global `last_reward_block` is written unconditionally before the early-return branch, so a single call per block permanently consumes the reward slot for that block regardless of whether rewards are actually distributed: [4](#0-3) 

### Impact Explanation

**Attack path A – Griefing / permanent yield freeze (High):**
Any address (including a non-staker) calls `update_rewards(any_valid_staker, disable_rewards: true)` at every block. `last_reward_block` is updated to the current block, the function returns early, and no rewards are distributed. No legitimate caller can succeed in the same block. Repeated every block, this permanently freezes the accrual of consensus-phase block rewards for every staker in the protocol.

**Attack path B – Theft of unclaimed yield (High):**
A registered staker calls `update_rewards(self, disable_rewards: false)` at every block. In V3 consensus rewards the calculation is:

```
staker_own_rewards = block_rewards × staker_own_stake / staker_total_stake
``` [5](#0-4) 

By calling every block the attacker claims the full per-block STRK reward for every block, regardless of whether they were the designated block producer. All other stakers receive zero consensus rewards because the one-per-block slot is consumed.

Both impacts fall within the allowed scope: **Theft of unclaimed yield** and **Permanent freezing of unclaimed yield**.

### Likelihood Explanation

The function is `#[abi(embed_v0)]` and publicly callable with no role check. The only prerequisite is a non-zero caller address and that the current block number exceeds `last_reward_block`. On Starknet, transaction fees are low enough that calling once per block is economically viable for a motivated attacker. No privileged access, leaked key, or external dependency is required.

### Recommendation

Add an authorized-caller check analogous to the one in `update_rewards_from_attestation_contract`. The consensus contract address should be stored at construction time and asserted at the top of `update_rewards`:

```cairo
fn assert_caller_is_consensus_contract(self: @ContractState) {
    assert!(
        get_caller_address() == self.consensus_contract.read(),
        "{}",
        Error::CALLER_IS_NOT_CONSENSUS_CONTRACT,
    );
}
```

Then call it at the start of `update_rewards`, mirroring the pattern already used for the attestation path.

### Proof of Concept

1. Consensus rewards are activated (`consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. Attacker (any EOA or contract) calls:
   ```
   staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
   ```
   at every new block.
3. Each call passes all checks (`general_prerequisites`, block-number guard, staker-active guard) and writes `last_reward_block = current_block`.
4. The function returns before `_update_rewards` is reached.
5. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. No staker ever accumulates `unclaimed_rewards_own` during the consensus phase; all consensus-phase yield is permanently frozen. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
