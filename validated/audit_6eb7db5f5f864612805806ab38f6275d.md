### Title
Unauthorized Interop Root Injection via Unfiltered Event Hook - (`system_hooks/src/event_hooks/interop_root_reporter.rs`)

### Summary

The `interop_root_reporter_event_hook` function processes `InteropRootAdded` events and injects interop roots into the batch public input's `interop_roots_rolling_hash` without verifying that the event was emitted by the authorized `L2_INTEROP_ROOT_STORAGE_ADDRESS` contract. Any unprivileged user can deploy a contract that emits a spoofed `InteropRootAdded` event, causing fake interop roots to be included in the cryptographic commitment sent to the settlement layer.

### Finding Description

`interop_root_reporter_event_hook` in `system_hooks/src/event_hooks/interop_root_reporter.rs` is triggered whenever any EVM contract emits an event whose first topic matches `INTEROP_ROOT_ADDED_EVENT_SIG`. The hook signature accepts `_caller_ee` (execution environment type) but **not** the emitting contract's address, and the parameter is explicitly ignored (`_caller_ee`):

```rust
pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<...>,
    data: &[u8],
    _caller_ee: u8,          // ignored — no address filtering
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
```

The hook only validates the event signature and data format, then unconditionally calls `system.io.add_interop_root()`:

```rust
system.io.add_interop_root(
    ExecutionEnvironmentType::NoEE,
    resources,
    InteropRoot { root, block_or_batch_number, chain_id },
)?;
```

`InteropRootStorage::push_root()` performs no duplicate or source check — it simply appends to a `HistoryList`:

```rust
pub fn push_root(&mut self, interop_root: InteropRoot) -> Result<(), SystemError> {
    self.list.push(interop_root, ());
    Ok(())
}
```

At block finalization, `calculate_interop_roots_rolling_hash` iterates over **all** accumulated roots (including attacker-injected ones) and folds them into the `interop_roots_rolling_hash` field of `BatchOutput`, which is then hashed into `BatchPublicInput` and submitted to the settlement layer:

```rust
let interop_roots_rolling_hash = calculate_interop_roots_rolling_hash(
    Bytes32::zero(),
    io.interop_root_storage.iter(),   // includes attacker roots
    &mut crypto::sha3::Keccak256::new(),
);
```

The `BatchOutput::hash()` commits this value on-chain:

```rust
hasher.update(self.interop_roots_rolling_hash.as_u8_ref());
```

### Impact Explanation

An attacker who injects fake interop roots corrupts the `interop_roots_rolling_hash` committed in the batch public input. This has two concrete consequences:

1. **Batch rejection / chain halt (DoS):** The settlement layer verifies the rolling hash against its own computation. A corrupted hash causes every batch containing the spoofed event to be rejected, halting the chain.
2. **False cross-chain message acceptance:** If the settlement layer uses the rolling hash to verify Merkle inclusion proofs for cross-chain messages, an attacker who controls the injected root value can construct a fake root that proves inclusion of a fabricated cross-chain message (e.g., a fraudulent deposit), enabling theft of bridged funds.

### Likelihood Explanation

The attack requires only a standard L2 user transaction. The attacker deploys a contract that emits `InteropRootAdded(chain_id, block_or_batch_number, [root])` with the exact 4-byte signature `0x6b451b84...`. No privileged access, leaked key, or governance majority is needed. The event hook fires for every block that contains such a transaction, making this continuously exploitable.

### Recommendation

Add an emitting-contract address check inside `interop_root_reporter_event_hook`. The hook already has access to `system`, which can be used to retrieve the current call context. Reject the event if it was not emitted by `L2_INTEROP_ROOT_STORAGE_ADDRESS`:

```rust
// Reject events not emitted by the authorized interop root storage contract
if caller_address != L2_INTEROP_ROOT_STORAGE_ADDRESS {
    return Ok(());
}
```

Alternatively, pass the emitting contract address as a parameter to the event hook and enforce the check there. The `InteropRootStorage::push_root()` could also validate that `chain_id != 0` and `root != Bytes32::zero()` as defense-in-depth, consistent with the struct-level documentation that already states these invariants.

### Proof of Concept

1. Attacker deploys an EVM contract `FakeInteropEmitter` with a function:
   ```solidity
   function inject(uint256 chainId, uint256 batchNum, bytes32 root) external {
       emit InteropRootAdded(chainId, batchNum, [root]);
   }
   ```
   where `InteropRootAdded` has the exact signature `0x6b451b8422636e45b93bf7f594fa2c1769d039766c4254a6e7f9c0ee1715cdb0`.

2. Attacker submits a regular L2 transaction calling `inject(attacker_chain_id, attacker_batch_num, attacker_root)`.

3. During block execution, `interop_root_reporter_event_hook` fires for this event. Since there is no address check, it calls `system.io.add_interop_root(...)` with the attacker-controlled values.

4. At block finalization in `post_tx_op_proving_singleblock_batch.rs`, `calculate_interop_roots_rolling_hash` folds the fake root into `interop_roots_rolling_hash`.

5. The corrupted hash is committed in `BatchOutput` and submitted to the settlement layer, causing batch rejection or enabling fraudulent cross-chain message proofs.

```rust
pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<<S::IOTypes as SystemIOTypesConfig>::EventKey, MAX_EVENT_TOPICS>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,

    // First, ensure we're capturing the InteropRootAdded event
    if topics.is_empty() || topics[0].as_u8_array() != INTEROP_ROOT_ADDED_EVENT_SIG {
        return Ok(());
    }
    // Internal error if the data supplied doesn't match the expected value
    if data.len() != 96 {
        return Err(internal_error!("Interop root reporter event hook received bad data").into());
    }

    // Parse data
    let offset: u32 = match U256::from_be_slice(&data[..32]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad offset").into(),
            );
        }
    };
    // This event is part of the system, but we check it anyways
    if offset != 32 {
        return Err(internal_error!("Interop root reporter event hook received bad offset").into());
    }

    let len: u32 = match U256::from_be_slice(&data[32..64]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad length").into(),
            );
        }
    };
    // It should have exactly one side
    if len != 1 {
        return Err(internal_error!("Interop root reporter event hook received bad length").into());
    }
    // Validate topics length
    if topics.len() != 3 {
        return Err(internal_error!("Interop root reporter event hook received bad topics").into());
    }

    let root = Bytes32::from_array(data[64..96].try_into().unwrap());
    let chain_id = U256::from_be_bytes(topics[1].as_u8_array());
    let block_or_batch_number = U256::from_be_bytes(topics[2].as_u8_array());
    system.io.add_interop_root(
        ExecutionEnvironmentType::NoEE,
        resources,
        InteropRoot {
            root,
            block_or_batch_number,
            chain_id,
        },
    )?;

    Ok(())
```

```rust
pub fn calculate_interop_roots_rolling_hash<'a>(
    old_rolling_hash: Bytes32,
    roots: impl Iterator<Item = &'a InteropRoot>,
    hasher: &mut crypto::sha3::Keccak256,
) -> Bytes32 {
    let mut data = [0u8; 96];

    let mut rolling_hash = old_rolling_hash;
    for root in roots {
        data[0..32].copy_from_slice(&rolling_hash.as_u8_ref());
        data[32..64].copy_from_slice(&root.chain_id.to_be_bytes::<{ U256::BYTES }>());
        data[64..96].copy_from_slice(&root.block_or_batch_number.to_be_bytes::<{ U256::BYTES }>());
        hasher.update(data);

        // Note: now we have only one side
        hasher.update(root.root.as_u8_ref());

        rolling_hash = hasher.finalize_reset().into()
    }

    rolling_hash
}
```
