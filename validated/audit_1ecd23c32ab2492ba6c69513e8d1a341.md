### Title
Unprivileged Caller Can Permanently Freeze Block Rewards by Calling `update_rewards` with `disable_rewards: true` - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the Staking contract updates the global `last_reward_block` state variable **before** distributing rewards, and accepts a caller-controlled `disable_rewards` flag with no access control enforcement. Any unprivileged address can call `update_rewards(any_staker, disable_rewards: true)` each block to permanently consume the per-block reward slot without distributing any rewards, causing all stakers to permanently lose their consensus block rewards.

### Finding Description

The `IStakingRewardsManager::update_rewards` function is documented in the spec as "Only starkware sequencer" access, but the implementation enforces no such restriction — only `general_prerequisites()` (a pause check) is called. [1](#0-0) 

The function immediately writes `last_reward_block = current_block_number` **before** any reward distribution occurs: [2](#0-1) 

After this write, if `disable_rewards` is `true` (a caller-supplied boolean with no validation), the function returns early with no rewards distributed: [3](#0-2) 

`last_reward_block` is a single global storage variable (not per-staker): [4](#0-3) 

Any subsequent call to `update_rewards` for the same block reverts with `REWARDS_ALREADY_UPDATED`: [5](#0-4) 

The spec confirms `update_rewards` is intended to be sequencer-only: [6](#0-5) 

The tests confirm the function is callable by any address (no `cheat_caller_address` is used): [7](#0-6) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

For every block in the consensus rewards epoch, an attacker calls `update_rewards(any_staker_address, disable_rewards: true)`. This:
1. Sets `last_reward_block` to the current block number.
2. Returns without distributing any rewards.
3. Permanently prevents any legitimate call to `update_rewards` for that block (REWARDS_ALREADY_UPDATED).

Because `last_reward_block` is global, a single attacker call per block freezes rewards for **all** stakers for that block. Repeated across every block, this permanently prevents all consensus block rewards from ever being distributed to any staker or pool.

### Likelihood Explanation

**High.** The attack requires no funds, no special role, and no prior setup. Any EOA can call `update_rewards` with `disable_rewards: true` on every new block. The attacker can automate this with a simple script. There is no cost to the attacker and no recovery mechanism for the lost rewards (the block slot is permanently consumed).

### Recommendation

Enforce the intended access control. Add a sequencer-only (or at minimum a non-zero-address) check at the top of `update_rewards`, consistent with the spec:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.roles.only_sequencer(); // enforce spec: "Only starkware sequencer"
    ...
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface and handle the pre-consensus early-return internally, so callers cannot influence whether rewards are distributed.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker (any address) monitors the mempool for new blocks.
3. At each new block N, attacker calls:
   ```
   staking.update_rewards(any_valid_staker_address, disable_rewards: true)
   ```
4. `last_reward_block` is written to N; function returns with no rewards distributed.
5. The legitimate sequencer's call to `update_rewards` for block N reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers permanently lose their block N rewards.
7. Attacker repeats every block — all consensus block rewards are permanently frozen. [8](#0-7) [2](#0-1)

```text
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
```
