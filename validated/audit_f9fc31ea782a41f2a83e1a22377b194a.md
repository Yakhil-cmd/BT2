### Title
Bank Precompile `transfer` Accepts Arbitrary `sender` — Unauthorized Transfer of `evm/` Denom Native Tokens - (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The `BankContract` precompile's `transfer` method accepts a caller-supplied `sender` address and uses it as the source of a `bankKeeper.SendCoins` call without verifying that `sender == contract.Caller()`. Any EVM contract can therefore move `evm/<callerContract>` native bank tokens out of any holder's account to any destination, without the holder's authorization.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` case unpacks `sender` directly from the ABI-encoded call arguments and passes it as the `from` address to `bankKeeper.SendCoins`:

```go
case TransferMethodName:
    args, err := method.Inputs.Unpack(contract.Input[4:])
    sender    := args[0].(common.Address)   // ← attacker-controlled
    recipient := args[1].(common.Address)
    amount    := args[2].(*big.Int)
    from := sdk.AccAddress(sender.Bytes())
    to   := sdk.AccAddress(recipient.Bytes())
    denom := EVMDenom(contract.Caller())    // "evm/<callerContract>"
    // NO check: sender == contract.Caller()
    bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

The `denom` is bound to the calling contract (`EVMDenom(contract.Caller())`), so the calling contract is the sole authority over that denomination. However, the `sender` field — the account whose balance is debited — is taken entirely from the call input with no authorization check. [2](#0-1) 

The same pattern exists in the `BurnMethodName` branch: the address to burn *from* is taken from `args[0]` without verifying it is the caller. [3](#0-2) 

### Impact Explanation
An unprivileged attacker deploys a malicious EVM contract, mints `evm/<maliciousContract>` tokens to victims (e.g., as DeFi rewards or LP shares), then calls `transfer(victimAddress, attackerAddress, amount)` on the bank precompile. The precompile executes `bankKeeper.SendCoins(victim, attacker, amount)` with no consent from the victim, draining their entire balance of that denomination. This constitutes an **unauthorized transfer of native bank-module assets** — a Critical/High impact under the allowed scope.

### Likelihood Explanation
Contract deployment on Cronos is permissionless. Any unprivileged user can deploy a contract, distribute `evm/<contractAddress>` tokens to users (legitimately or as part of a scheme), and later invoke the precompile to steal them. No privileged access, leaked keys, or governance action is required.

### Recommendation
Add an authorization check in the `TransferMethodName` case to ensure the `sender` argument matches `contract.Caller()`:

```go
case TransferMethodName:
    ...
    sender := args[0].(common.Address)
    // Add:
    if sender != contract.Caller() {
        return nil, errors.New("sender must be the calling contract")
    }
    ...
```

Similarly, the `BurnMethodName` branch should verify that the address being burned from is `contract.Caller()` (or an address that has explicitly authorized the burn). [4](#0-3) 

### Proof of Concept

1. Attacker deploys `MaliciousToken` contract on Cronos.
2. `MaliciousToken` calls `bankPrecompile.mint(victimAddress, 1000)` — victim now holds 1000 `evm/<MaliciousToken>` native tokens.
3. Attacker calls `bankPrecompile.transfer(victimAddress, attackerAddress, 1000)` from `MaliciousToken`.
4. The precompile executes `bankKeeper.SendCoins(victim, attacker, 1000 evm/<MaliciousToken>)` with no check on victim's consent.
5. Victim's balance is zero; attacker holds 1000 tokens. No approval or signature from the victim was required. [1](#0-0)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
