### Title
Unprivileged Caller Can Permanently Deny Block Rewards for All Stakers via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in `IStakingRewardsManager` is a public external function with no access control. It accepts a caller-controlled `disable_rewards` boolean. When called with `disable_rewards: true`, it advances the global `last_reward_block` to the current block without distributing any rewards, permanently consuming the one-call-per-block slot for all stakers.

### Finding Description

`update_rewards` is exposed as a fully public external function: [1](#0-0) 

Its only access guard is `general_prerequisites`, which checks only that the contract is unpaused and the caller is non-zero: [2](#0-1) 

Inside the function, `last_reward_block` — a **global** (not per-staker) storage variable — is unconditionally written to the current block number before the `disable_rewards` branch is evaluated: [3](#0-2) 

The guard that prevents double-processing is: [4](#0-3) 

Because `last_reward_block` is global, once any caller writes it for block N, **no other call** to `update_rewards` can succeed in block N, regardless of which staker is targeted. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` first in a block:

1. Passes all validation (staker exists, is active, has non-zero balance).
2. Writes `last_reward_block = current_block_number`.
3. Returns early at the `disable_rewards` branch without distributing rewards. [5](#0-4) 

Every legitimate consensus-layer call to `update_rewards` in that same block then reverts with `REWARDS_ALREADY_UPDATED`. The block's rewards are permanently lost — there is no mechanism to retroactively claim them.

### Impact Explanation

`last_reward_block` is a single global slot. One attacker transaction per block is sufficient to deny block rewards to **every** staker in the protocol for that block. Rewards for a missed block are never recoverable. Repeated across blocks, this constitutes permanent, protocol-wide freezing and theft of unclaimed yield.

Allowed impact matched: **High — Theft of unclaimed yield / Permanent freezing of unclaimed yield**.

### Likelihood Explanation

The entry path requires only a non-zero caller address and knowledge of any currently active staker address (trivially obtained from `NewStaker` events). No privileged role, no capital, no bridge interaction is needed. The attack is cheap (one transaction per block) and fully permissionless.

### Recommendation

1. Restrict `update_rewards` to a trusted caller (e.g., the consensus layer address, or the staker themselves), analogous to how `update_rewards_from_attestation_contract` is restricted to the attestation contract via `assert_caller_is_attestation_contract`.
2. Remove the caller-controlled `disable_rewards` parameter from the public interface, or gate it behind a privileged role check.
3. Consider making `last_reward_block` per-staker rather than global, so one call cannot block all others.

### Proof of Concept

```
// Block N begins.
// Attacker (any EOA) calls:
staking.update_rewards(staker_address: alice, disable_rewards: true);
// → last_reward_block is now N; no rewards distributed.

// Consensus layer (or any legitimate caller) calls:
staking.update_rewards(staker_address: bob, disable_rewards: false);
// → PANICS: current_block_number (N) is NOT > last_reward_block (N).
// Bob and all other stakers permanently lose block N rewards.
``` [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
