### Title
Unconstrained `disable_rewards` Parameter in `update_rewards` Allows Any Caller to Permanently Deny Consensus Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract accepts a caller-supplied `disable_rewards: bool` parameter that is completely unconstrained. Because `last_reward_block` is written unconditionally before the `disable_rewards` guard, any unprivileged address can call `update_rewards(..., disable_rewards: true)` once per block to consume the block's reward slot without distributing any rewards. Since `last_reward_block` is a single global variable shared across all stakers, one such call per block permanently denies consensus rewards to every staker for that block.

---

### Finding Description

`update_rewards` is a public function gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

The function writes `last_reward_block` to the current block number **before** checking `disable_rewards`: [2](#0-1) 

After that write, the guard `current_block_number > self.last_reward_block.read()` will revert for any subsequent caller in the same block: [3](#0-2) 

`last_reward_block` is a single global storage slot, not per-staker: [4](#0-3) 

The `disable_rewards` flag is never validated against any on-chain condition or restricted to a privileged caller. This is the direct analog to the unconstrained `imm_sign` in the referenced report: a boolean that controls whether a critical computation (reward distribution) executes, but whose value is freely chosen by the caller.

---

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` once per block:

1. Atomically marks the block as "rewards already updated" (`last_reward_block = current_block`).
2. Causes the function to return early with no rewards distributed to any staker or pool.
3. Prevents any legitimate call to `update_rewards` for that block (the `REWARDS_ALREADY_UPDATED` assertion reverts all subsequent attempts).

Repeated every block, this permanently prevents all consensus-phase block rewards from ever accruing. This maps to **High: Permanent freezing of unclaimed yield** — stakers and delegators never receive the STRK rewards they are owed for consensus participation. [5](#0-4) 

---

### Likelihood Explanation

- The function is fully public with no role check.
- The attack costs only gas (one transaction per block).
- The attacker needs no stake, no delegation, and no privileged key — only a funded address.
- The attack is deterministic and repeatable every block.

---

### Recommendation

Either:

1. **Restrict the caller**: Add an access-control check so only an authorized consensus contract or operator role can call `update_rewards`.
2. **Remove the unconstrained parameter**: Derive the `disable_rewards` condition entirely from on-chain state (e.g., a flag set by a privileged role) rather than accepting it as a caller-supplied argument.
3. **Write `last_reward_block` only after the `disable_rewards` guard**: At minimum, move the `self.last_reward_block.write(current_block_number)` to after the early-return check so that a `disable_rewards: true` call does not consume the block's reward slot.

---

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
