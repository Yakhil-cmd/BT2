### Title
Missing Caller Guard on `update_rewards` Allows Any Address to Freeze or Redirect Staker Yield - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` is specified to be callable only by the Starknet sequencer, but the implementation contains no `get_caller_address()` check. Any unprivileged address can call it first in any block, either with `disable_rewards: true` to silently consume the block's reward slot without distributing anything, or with `disable_rewards: false` for an attacker-controlled staker to redirect that block's rewards away from the intended recipient.

---

### Finding Description

The spec for `update_rewards` states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. The only guard is a per-block deduplication check on `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    ...
``` [2](#0-1) 

`last_reward_block` is a single global storage variable shared across all stakers. Whoever calls `update_rewards` first in a given block wins the slot. There is no `assert!(get_caller_address() == sequencer_address, ...)` or equivalent. [3](#0-2) 

---

### Impact Explanation

**Attack vector A — Permanent yield freeze (High):**
An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block. This writes `last_reward_block = current_block_number` and returns immediately without distributing any rewards. When the legitimate sequencer attempts the same call in the same block, it reverts with `REWARDS_ALREADY_UPDATED`. No staker or delegator accumulates any yield for that block. Repeated every block, this permanently freezes all unclaimed yield across the entire protocol.

**Attack vector B — Yield redirection (High):**
An attacker who is also a staker calls `update_rewards(attacker_staker_address, disable_rewards: false)` before the sequencer. The attacker's staker receives the full block reward. The sequencer's subsequent call for the intended staker reverts. The attacker captures rewards that should have gone to other stakers.

Both impacts fall within the allowed scope:
- *High: Permanent freezing of unclaimed yield* (Attack A)
- *High: Theft of unclaimed yield* (Attack B)

---

### Likelihood Explanation

The function is publicly callable with no access restriction. Any EOA or contract can invoke it. The only cost is the gas for a Starknet transaction. An attacker with a staked position has direct financial incentive (Attack B). A griefing attacker (Attack A) has no cost barrier beyond gas. The attack is repeatable every block indefinitely.

---

### Recommendation

Add a sequencer-address check at the top of `update_rewards`, analogous to the existing `assert_caller_is_attestation_contract` pattern used elsewhere in the same contract:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
``` [4](#0-3) 

The sequencer address should be stored in contract storage and set during initialization, following the same pattern as `attestation_contract`.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. A valid staker `S` exists with non-zero balance.
3. Attacker deploys a script that, at the start of every block, calls:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(S, disable_rewards: true)
   ```
4. `last_reward_block` is set to the current block; the function returns without distributing rewards.
5. The sequencer's call for the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. No staker or pool member accumulates any yield. `claim_rewards` returns zero for all participants indefinitely.

For Attack B, replace step 3 with `update_rewards(attacker_staker, disable_rewards: false)` — the attacker's staker receives the block reward while the sequencer's intended staker receives nothing.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
```
