### Title
`createPool` Updates Registry Storage After External Extension `initialize` Calls, Enabling Re-Entrancy and `isPool` Read Re-Entrancy — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.createPool` calls external extension `initialize` functions before writing any factory-side registry state (`poolAdmin`, `poolFeeConfig`, `poolAdminFeeDestination`, `poolToIdx`, `idxToPool`, `nextPoolIdx`). Because `createPool` carries no `nonReentrant` guard, a malicious extension can re-enter `createPool` during its `initialize` callback, corrupting the pool-index registry. Simultaneously, any observer that reads `factory.isPool(pool)` during the callback window receives `false` for a pool that is already deployed and accepting interactions.

---

### Finding Description

`createPool` follows this order:

1. Deploy pool via `MetricOmmPoolDeployer.deploy(...)` — external call.
2. For each extension, call `CallExtension.callExtension(extensions[i], abi.encodeCall(IMetricOmmExtensions.initialize, (pool, initData[i])))` — external call to an arbitrary, caller-supplied address.
3. **Only then** write factory storage:

```
poolAdmin[pool]              = params.admin;
priceProviderTimelock[pool]  = params.priceProviderTimelock;
poolFeeConfig[pool]          = PoolFeeConfig({...});
poolAdminFeeDestination[pool] = params.adminFeeDestination;
nextPoolIdx++;
idxToPool[poolIdx]           = pool;
poolToIdx[pool]              = poolIdx;
``` [1](#0-0) 

`createPool` has no `nonReentrant` modifier: [2](#0-1) 

`CallExtension.callExtension` performs a raw `.call` to the extension address, giving the extension full control during the callback: [3](#0-2) 

The factory's `ReentrancyGuardTransient` (OpenZeppelin standard) is only armed on functions that explicitly use the `nonReentrant` modifier. Because `createPool` does not use it, the guard slot is `0` during the entire call, and any `nonReentrant`-protected factory function (e.g., `collectPoolFees`) can be freely entered from within the extension callback. [4](#0-3) 

---

### Impact Explanation

**1. Pool-registry index corruption (re-entrant `createPool`)**

A malicious extension's `initialize` re-enters `createPool` with a different salt to deploy pool B. At that moment `nextPoolIdx` is still at its pre-increment value (e.g., `1`). Pool B's registration runs to completion and consumes index `1`; `nextPoolIdx` becomes `2`. Control returns to pool A's registration, which now assigns pool A index `2`. The `idxToPool` / `poolToIdx` mappings are permanently misordered. Any off-chain or on-chain enumeration that relies on monotonic index assignment (e.g., "pool at index 1 was created first") is silently wrong.

**2. `isPool(pool)` read re-entrancy**

`isPool` is implemented as:

```solidity
function isPool(address pool) external view override returns (bool) {
    return poolToIdx[pool] != 0;
}
``` [5](#0-4) 

Because `poolToIdx[pool]` is written only after the extension loop, any contract that calls `factory.isPool(pool)` during the extension `initialize` window — including the extension itself, a router, or a periphery contract — receives `false` for a pool that is already deployed and operational. This breaks the invariant that a deployed, functional pool is always recognized as canonical by the factory.

---

### Likelihood Explanation

`createPool` is permissionless once `poolDeployer` is set. `ValidateExtensionsConfig` only checks that extension addresses are non-zero, non-duplicate, and that orders are structurally valid — it does not restrict extension code: [6](#0-5) 

Any caller can supply a malicious extension contract. The re-entrancy window is open on every `createPool` call that includes at least one extension.

---

### Recommendation

Apply the checks-effects-interactions pattern: write all factory storage **before** calling external extension `initialize` functions. Additionally, add `nonReentrant` to `createPool`.

```diff
-    pool = MetricOmmPoolDeployer(poolDeployer).deploy(...);
+    pool = MetricOmmPoolDeployer(poolDeployer).deploy(...);
 
+    // Write all registry state before any external extension calls
+    poolAdmin[pool] = params.admin;
+    priceProviderTimelock[pool] = params.priceProviderTimelock;
+    poolFeeConfig[pool] = PoolFeeConfig({...});
+    poolAdminFeeDestination[pool] = params.adminFeeDestination;
+    uint256 poolIdx = nextPoolIdx;
+    nextPoolIdx++;
+    idxToPool[poolIdx] = pool;
+    poolToIdx[pool] = poolIdx;
+
     for (uint256 i = 0; i < params.extensions.length; i++) {
       CallExtension.callExtension(
         params.extensions[i], abi.encodeCall(IMetricOmmExtensions.initialize, (pool, params.extensionInitData[i]))
       );
     }
 
-    poolAdmin[pool] = params.admin;
-    priceProviderTimelock[pool] = params.priceProviderTimelock;
-    poolFeeConfig[pool] = PoolFeeConfig({...});
-    poolAdminFeeDestination[pool] = params.adminFeeDestination;
-    uint256 poolIdx = nextPoolIdx;
-    nextPoolIdx++;
-    idxToPool[poolIdx] = pool;
-    poolToIdx[pool] = poolIdx;
```

Also add `nonReentrant` to the `createPool` signature to prevent re-entrant pool creation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

import {IMetricOmmExtensions} from "metric-core/contracts/interfaces/extensions/IMetricOmmExtensions.sol";
import {IMetricOmmPoolFactory} from "metric-core/contracts/interfaces/IMetricOmmPoolFactory/IMetricOmmPoolFactory.sol";
import {PoolParameters} from "metric-core/contracts/types/FactoryOperation.sol";

contract MaliciousExtension is IMetricOmmExtensions {
    IMetricOmmPoolFactory public factory;
    PoolParameters public secondParams;
    bool public reentered;

    constructor(address _factory, PoolParameters memory _secondParams) {
        factory = IMetricOmmPoolFactory(_factory);
        secondParams = _secondParams;
    }

    function initialize(address pool, bytes calldata) external returns (bytes4) {
        if (!reentered) {
            reentered = true;
            // During pool A's extension init, nextPoolIdx is still 1.
            // Re-enter createPool to deploy pool B, which steals index 1.
            factory.createPool(secondParams); // pool B gets poolToIdx = 1
            // After this returns, pool A will get poolToIdx = 2 (wrong order).

            // Also demonstrate read re-entrancy: isPool(pool) is false here
            assert(!factory.isPool(pool)); // pool A is deployed but not registered yet
        }
        return IMetricOmmExtensions.initialize.selector;
    }

    // ... other IMetricOmmExtensions hooks returning their selectors
}
```

After both `createPool` calls complete:
- `factory.poolToIdx(poolA) == 2` (should be 1)
- `factory.poolToIdx(poolB) == 1` (should be 2)
- `factory.idxToPool(1) == poolB` (wrong)
- `factory.idxToPool(2) == poolA` (wrong)
