### Title
Missing Upper-Bound Constraint on `min_gas_price_percentage` Causes Gateway to Reject Valid Transactions - (File: `crates/apollo_gateway_config/src/config.rs`)

---

### Summary

`StatefulTransactionValidatorConfig::min_gas_price_percentage` is a `u8` field (range 0â€“255) that is semantically a percentage (0â€“100). It carries no `#[validate(range(max = 100))]` annotation, so a value above 100 is accepted by the config validation framework. When consumed in `validate_tx_l2_gas_price_within_threshold`, a value > 100 inflates the admission threshold above the previous block's L2 gas price, causing the gateway to reject transactions whose gas price exactly matches or slightly exceeds the previous block price â€” transactions that are fully valid under the protocol.

---

### Finding Description

`StatefulTransactionValidatorConfig` derives `validator::Validate` but places no upper-bound annotation on `min_gas_price_percentage`:

```rust
// crates/apollo_gateway_config/src/config.rs
#[derive(Clone, Debug, Serialize, Deserialize, Validate, PartialEq)]
pub struct StatefulTransactionValidatorConfig {
    ...
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}
``` [1](#0-0) 

The field is consumed here:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...); // reject
}
``` [2](#0-1) 

When `min_gas_price_percentage = 150`, the multiplier becomes `150/100 = 1.5`, so `threshold = 1.5 Ă— previous_block_l2_gas_price`. Any transaction whose `max_price_per_unit` is between 100% and 149% of the previous block price is rejected, even though it satisfies the protocol's actual requirement.

The analogous field in `MempoolStaticConfig` â€” `fee_escalation_percentage` â€” correctly carries `#[validate(range(min = 1, max = 100))]`, demonstrating the intended pattern is known and applied elsewhere:

```rust
// crates/apollo_mempool_config/src/config.rs
#[validate(range(min = 1, max = 100))]
pub fee_escalation_percentage: u8,
``` [3](#0-2) 

`StatefulTransactionValidatorConfig` has no equivalent upper-bound guard.

---

### Impact Explanation

**High. Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

With `min_gas_price_percentage > 100`, the gateway's stateful validator computes a threshold that exceeds the previous block's L2 gas price. Transactions priced at or near the current market rate â€” which are valid under the Starknet protocol â€” are rejected at the admission stage with `GAS_PRICE_TOO_LOW`. This silently drops legitimate user transactions from the network without any on-chain evidence, degrading liveness for all users of the affected gateway instance.

---

### Likelihood Explanation

**Low.** Exploiting this requires a misconfigured or malicious node operator to set `min_gas_price_percentage` to a value above 100 in the deployed configuration. No unprivileged user can trigger this directly. However, the absence of a compile-time or startup-time guard means a configuration mistake (e.g., entering `150` intending `15`) silently passes all validation and takes effect immediately.

---

### Recommendation

Add `#[validate(range(min = 0, max = 100))]` to `min_gas_price_percentage` in `StatefulTransactionValidatorConfig`, mirroring the existing constraint on `fee_escalation_percentage`:

```rust
#[validate(range(min = 0, max = 100))]
pub min_gas_price_percentage: u8,
``` [4](#0-3) 

Also add a corresponding test in the gateway config test suite (analogous to `zero_fee_escalation_percentage_fails_validation` in `crates/apollo_mempool_config/src/config_test.rs`) that asserts `min_gas_price_percentage = 101` fails validation. [5](#0-4) 

---

### Proof of Concept

1. Set `gateway_config.static_config.stateful_tx_validator_config.min_gas_price_percentage = 150` in the node config (e.g., `crates/apollo_node/resources/config_schema.json` or a deployment override).
2. Start the gateway. `GatewayStaticConfig::validate()` propagates `#[validate(nested)]` to `StatefulTransactionValidatorConfig::validate()`, but because `min_gas_price_percentage` has no range annotation, validation passes silently.
3. Submit an `AllResources` invoke transaction with `max_price_per_unit` equal to the previous block's L2 gas price (i.e., 100% of the threshold â€” a fully valid price).
4. `validate_tx_l2_gas_price_within_threshold` computes `threshold = 1.5 Ă— previous_block_l2_gas_price`, finds `tx_l2_gas_price < threshold`, and returns `GAS_PRICE_TOO_LOW`.
5. The transaction is rejected at the gateway despite being protocol-valid, matching the **High / Mempool-gateway admission rejects valid transactions** impact category. [6](#0-5)

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
