### Title
Missing Caller Authorization in `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is documented as "Only starkware sequencer" in the protocol spec, but the implementation contains **no caller authorization check**. Any unprivileged address can call it with `disable_rewards: true`, which writes `last_reward_block` to the current block number and returns early — permanently preventing the sequencer from distributing rewards for that block to all stakers.

---

### Finding Description

The `update_rewards` function in `StakingRewardsManagerImpl` is the consensus-phase reward distribution entry point. The spec explicitly restricts it to the Starkware sequencer:

> **access control**: Only starkware sequencer.

However, the implementation only calls `general_prerequisites()`, which checks for pause state and a non-zero caller — it does **not** verify the caller's identity.

The critical ordering flaw is that `last_reward_block` is written to storage **before** the `disable_rewards` guard:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // line 1485

if disable_rewards || self.is_pre_consensus() {
    return;                                            // line 1487-1489
}
```

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` will:
1. Pass the `current_block_number > last_reward_block` check.
2. Commit `last_reward_block = current_block_number`.
3. Return early without distributing any rewards.

The sequencer's subsequent call for the same block then fails with `REWARDS_ALREADY_UPDATED` because the strict-greater-than check `current_block_number > last_reward_block` is now false. Since `last_reward_block` is a **single global field** (not per-staker), one attacker call per block suppresses rewards for **every** staker in the system.

Contrast this with `update_rewards_from_attestation_contract`, which correctly enforces:

```cairo
self.assert_caller_is_attestation_contract();   // line 1400
```

No equivalent guard exists in `update_rewards`.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Block rewards that are not distributed in a given block are permanently lost; there is no catch-up mechanism. An attacker who calls `update_rewards(staker, disable_rewards: true)` once per block eliminates all consensus-phase block rewards for all stakers indefinitely. The stakers' `unclaimed_rewards_own` fields are never incremented, and the STRK that would have been minted/transferred is never claimed.

---

### Likelihood Explanation

**High.** The function is public, requires no stake, no role, and no special state. The only cost is the gas for one transaction per block. A motivated attacker (e.g., a competing validator, a griefing party) can sustain this attack cheaply and continuously. The attack is also front-runnable: the attacker simply monitors the mempool and submits before the sequencer's `update_rewards` call.

---

### Recommendation

Add an explicit caller authorization check at the top of `update_rewards`, mirroring the pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
```

Alternatively, move the `last_reward_block.write` to **after** the `disable_rewards` branch so that a call with `disable_rewards: true` does not consume the block's reward slot.

---

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`).
2. A new block `N` is produced. `last_reward_block < N`.
3. Attacker (any address) calls `update_rewards(valid_staker, disable_rewards: true)`.
   - `current_block_number (N) > last_reward_block` → passes.
   - `last_reward_block` is written to `N`.
   - Function returns early; no rewards distributed.
4. Sequencer calls `update_rewards(valid_staker, disable_rewards: false)`.
   - `current_block_number (N) > last_reward_block (N)` → **false** → panics with `REWARDS_ALREADY_UPDATED`.
5. All stakers lose block `N` rewards permanently.
6. Repeat each block.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
