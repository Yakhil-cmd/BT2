### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the Staking contract is documented as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` once per block, consuming the per-block reward slot and permanently preventing the sequencer from distributing staker and pool rewards for that block.

### Finding Description

The protocol specification explicitly states:

> `update_rewards` — **access control: Only starkware sequencer.**

However, the implementation at `src/staking/staking.cairo` lines 1449–1507 contains no such enforcement:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no caller check ...
    self.last_reward_block.write(current_block_number);  // consumes the block slot
    if disable_rewards || self.is_pre_consensus() {
        return;  // exits without distributing rewards
    }
    // ... reward distribution ...
}
```

The `last_reward_block` storage variable is a **single global value** (not per-staker). Once it is written to the current block number, the `REWARDS_ALREADY_UPDATED` assertion prevents any further call to `update_rewards` within the same block — for any staker.

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block. This:
1. Passes all checks (contract not paused, block is new, staker exists and is active).
2. Writes `last_reward_block = current_block_number`.
3. Returns immediately without distributing rewards.
4. Blocks the sequencer from calling `update_rewards` for any staker in that block.

This is the exact same false-assumption pattern as the PasswordStore report: the code assumes a restriction exists (`private` / "only sequencer") but provides no on-chain enforcement.

### Impact Explanation

If an attacker calls `update_rewards(staker, disable_rewards: true)` every block, no staker or delegation pool ever receives consensus-era block rewards. Accumulated `unclaimed_rewards_own` for all stakers and `cumulative_rewards_trace` entries for all pool members are never updated. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators in the protocol.

Even sporadic calls cause permanent loss of yield for the targeted blocks, since missed blocks are never retroactively compensated.

### Likelihood Explanation

- No special role, token balance, or privileged access is required.
- The only cost is the L2 gas for one transaction per block.
- The attacker needs only to know any valid active staker address (publicly readable from on-chain events or the `stakers` vector).
- The attack is trivially automatable with a simple bot.

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, analogous to the existing `assert_caller_is_attestation_contract` pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
```

Alternatively, restrict the function to be callable only by the sequencer address stored in contract configuration, or use Starknet's native sequencer address opcode (`get_sequencer_address`) to validate the caller.

### Proof of Concept

1. Deploy the staking system with consensus rewards active.
2. From any EOA (attacker), call:
   ```
   staking.update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)
   ```
3. Observe that `last_reward_block` is now set to the current block.
4. The sequencer attempts to call `update_rewards` in the same block and receives `REWARDS_ALREADY_UPDATED`.
5. Stakers and pool members receive zero rewards for that block.
6. Repeat every block — all yield is permanently frozen.

The existing test `test_update_rewards_assertions` at `src/staking/tests/test.cairo` lines 3841–3903 confirms that `update_rewards` succeeds when called from an arbitrary address (no `cheat_caller_address` is needed to pass the test), demonstrating the absence of any caller restriction.         fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
