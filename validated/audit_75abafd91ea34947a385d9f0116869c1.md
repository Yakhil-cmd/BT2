### Title
Unprivileged Caller Can Permanently Freeze Consensus Reward Distribution via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `staking.cairo` writes to the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. Because `update_rewards` is publicly callable by any non-zero address with no further access control, an attacker can call it every block with `disable_rewards: true` and a valid staker address, consuming the block's single reward slot without distributing any rewards. This permanently freezes consensus reward accumulation for all stakers in the protocol.

### Finding Description

`update_rewards` is exposed as a public ABI function under `IStakingRewardsManager`. Its access guard, `general_prerequisites`, only checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

The function validates that the current block is strictly greater than `last_reward_block` — a **single global variable** shared across all stakers — and then immediately writes the current block number to it: [2](#0-1) 

Only **after** this write does the function check `disable_rewards`: [3](#0-2) 

Because `last_reward_block` is already updated at this point, any subsequent call to `update_rewards` in the same block — by any legitimate staker — will fail with `REWARDS_ALREADY_UPDATED`. The `disable_rewards` parameter carries no access restriction; any caller may supply `true`.

The storage layout confirms `last_reward_block` is a single protocol-wide slot: [4](#0-3) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

After consensus rewards are activated (`is_pre_consensus()` returns `false`), an attacker calling `update_rewards(any_valid_staker, true)` every block ensures:

1. `last_reward_block` is always equal to the current block.
2. No rewards are ever distributed to any staker.
3. All legitimate `update_rewards` calls revert with `REWARDS_ALREADY_UPDATED`.

Stakers accumulate zero consensus rewards indefinitely. The attack is sustained at the cost of one transaction per block.

### Likelihood Explanation

**High.** The attacker requires only:
- A non-zero address (trivially satisfied).
- Any valid, active staker address with non-zero balance (all staker addresses are publicly observable from on-chain `NewStaker` events).

No privileged role, leaked key, or external dependency is needed. The attack is cheap (one call per block) and fully permissionless.

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a call with `disable_rewards: true` does not consume the block's reward slot:

```cairo
// Update last block rewards only when rewards are actually distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... rest of reward distribution
```

Alternatively, restrict who may pass `disable_rewards: true` (e.g., only the staker address or their reward address), or remove the parameter from the public interface entirely if it is only needed internally.

### Proof of Concept

1. Consensus rewards are activated (`consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. Attacker identifies any valid, active staker with non-zero STRK balance (e.g., from `NewStaker` events).
3. Every block, attacker submits: `staking.update_rewards(valid_staker_address, disable_rewards: true)`.
4. Inside the call: staker validation passes → `last_reward_block` is written to current block → `disable_rewards` check triggers early return → **zero rewards distributed**.
5. Any legitimate staker or keeper attempting `update_rewards` in the same block hits the `current_block_number > self.last_reward_block.read()` assertion and reverts.
6. Repeated every block: all stakers are permanently frozen out of consensus reward accumulation. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
