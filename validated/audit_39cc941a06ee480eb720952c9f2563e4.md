### Title
Staker Can Increase Commission After Delegator Commits Funds, Stealing Unclaimed Yield - (File: `src/staking/staking.cairo`)

### Summary
A staker can advertise a low commission rate to attract delegators, then immediately increase the commission to up to 100% after delegators have committed their funds. Delegators cannot exit immediately — they are subject to the exit wait window — and lose all accrued rewards to the inflated commission during that period. The code itself acknowledges this gap with the comment: *"Current commission increase safeguards still allow for sudden commission changes."*

### Finding Description
The commission system has two modes:

**Without a commitment** (`update_commission`, line 1595–1596): commission can only be *decreased*.

**With an active commitment** (`update_commission`, lines 1583–1589): commission can be changed to *any value up to `max_commission`*, including values higher than the current commission.

The `set_commission_commitment` function (lines 748–785) allows a staker to set `max_commission` up to 10000 (100%) while currently charging only 1% commission. The only constraint is `current_commission <= max_commission`. Once the commitment is active, the staker can call `set_commission(10000)` to jump from 1% to 100% in a single transaction.

The commitment can be set for up to one year (`expiration_epoch - current_epoch <= epochs_in_year()`), and after it expires the staker can set a new one — enabling the attack to repeat indefinitely. [2](#0-1) 

Delegators who entered the pool based on the advertised low commission cannot exit immediately. They must call `exit_delegation_pool_intent` and wait for the full `exit_wait_window` (default: 1 week, up to 12 weeks). During this window their stake remains in the pool and all rewards are consumed by the 100% commission. [3](#0-2) 

The pool's `claim_rewards` distributes rewards to the delegator's `reward_address` only after deducting commission. With 100% commission, the delegator receives zero. [4](#0-3) 

### Impact Explanation
**High — Theft of unclaimed yield.**

A delegator who joined a pool at 1% commission and is now subject to 100% commission loses 100% of all rewards accrued from the moment of the commission increase until they can complete the exit flow. With a 1-week exit window and typical staking yields, this represents a material, quantifiable loss of earned rewards. The staker captures those rewards as commission. The attack can be repeated after each commitment expiry, making the theft ongoing.

### Likelihood Explanation
**Medium-High.** The attack requires only that the staker:
1. Set a commission commitment with `max_commission = 10000` before or after attracting delegators.
2. Call `set_commission(10000)` once delegators have entered.

No privileged role, leaked key, or external dependency is needed. The staker is an unprivileged participant. The only on-chain signal a delegator has is the `CommissionCommitmentSet` event, which most delegators will not monitor in real time. The code comment at line 745 explicitly acknowledges this gap exists. [5](#0-4) 

### Recommendation
1. **Enforce a delay on commission increases.** When a commitment allows raising commission, the new higher rate should only take effect after `K` epochs (matching the existing epoch-delay pattern used for stake changes), giving delegators time to observe the change and exit before it applies.
2. **Cap `max_commission` relative to current commission.** Limit how much a commitment can raise commission in a single step (e.g., no more than X% above the current rate).
3. **Document the risk prominently** so delegators know to monitor `CommissionChanged` events and understand the exit window implications.

### Proof of Concept
```
// Step 1: Staker stakes and sets 1% commission
staking.stake(reward_addr, op_addr, amount);
staking.set_commission(commission: 100);          // 1%
staking.set_open_for_delegation(STRK_TOKEN);

// Step 2: Staker sets commitment with max_commission = 100%
// expiration_epoch = current_epoch + 1 (minimum valid)
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1);

// Step 3: Delegator sees 1% commission and enters pool
pool.enter_delegation_pool(reward_address: delegator_reward, amount: 1_000_000_STRK);

// Step 4: Staker immediately raises commission to 100%
staking.set_commission(commission: 10000);        // 100% — allowed by active commitment

// Step 5: Rewards are distributed — delegator receives 0
// update_rewards / update_rewards_from_attestation_contract runs:
//   staker_commission_rewards = pool_rewards * 10000 / 10000 = pool_rewards
//   delegator_rewards = 0

// Step 6: Delegator calls exit intent — must wait exit_wait_window (≥ 1 week)
pool.exit_delegation_pool_intent(amount: 1_000_000_STRK);
// During the wait, all rewards continue to flow to the staker as commission.
// Delegator's unclaimed yield for the entire exit window is stolen.
```

The root cause is that `set_commission_commitment` places no upper bound on `max_commission` relative to the current commission, and `update_commission` enforces no epoch delay when a commitment permits an increase. [6](#0-5) [7](#0-6)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);

            // Calculate rewards and update entry_to_claim_from.

            // Emit event.

                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
