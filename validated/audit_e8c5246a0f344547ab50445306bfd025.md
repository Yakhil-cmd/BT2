### Title
Staker Can Increase Commission After Commitment Expiration by Setting New Commitment with Higher Max — (File: `src/staking/staking.cairo`)

---

### Summary

After a `CommissionCommitment` expires, a staker can bypass the "commission can only decrease" restriction by first calling `set_commission_commitment` with a higher `max_commission`, then calling `set_commission` to raise commission up to that new ceiling. This allows a staker to significantly increase their commission rate at the expense of delegators' yield.

---

### Finding Description

The protocol has two functions that interact to govern commission changes:

**`set_commission_commitment`** (lines 748–785 of `src/staking/staking.cairo`):

When an existing commitment is expired, the only guard before writing a new one is: [1](#0-0) 

```cairo
if let Option::Some(commission_commitment) = staker_pool_info.commission_commitment.read() {
    assert!(
        !self.is_commission_commitment_active(:commission_commitment),
        "{}",
        Error::COMMISSION_COMMITMENT_EXISTS,
    );
}
```

After passing that check, the only constraint on the new `max_commission` is: [2](#0-1) 

```cairo
let current_commission = staker_pool_info.commission();
assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
```

There is **no upper bound** on `max_commission` relative to the expired commitment's `max_commission` or any other ceiling.

**`update_commission`** (lines 1573–1609 of `src/staking/staking.cairo`):

When a commitment is **active**, commission can be set to any value ≤ `max_commission` (including increases): [3](#0-2) 

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
} else {
    assert!(commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED);
}
```

When the commitment is **expired**, commission can only decrease (`commission < old_commission`). This is the intended protection. However, the staker can trivially circumvent it:

**Exploit path:**
1. Staker has `commission = 50%`, commitment: `max_commission = 60%`, `expiration_epoch = E`.
2. Epoch `E` passes; commitment expires.
3. Staker calls `set_commission_commitment(max_commission: 90%, expiration_epoch: E+100)`.
   - Old commitment is expired → `!is_commission_commitment_active` passes.
   - `50% <= 90%` passes. New commitment written.
4. Staker calls `set_commission(commission: 85%)`.
   - New commitment is active → `85% <= 90%` passes, `85% != 50%` passes.
   - Commission jumps from **50% → 85%**.

The missing check is that `set_commission_commitment` does not enforce `max_commission <= old_commitment.max_commission` (or `max_commission <= current_commission`) when replacing an expired commitment.

---

### Impact Explanation

Delegators who joined a pool expecting commission to only decrease (or stay bounded) after commitment expiration will instead face a sudden, large commission increase. This directly reduces the yield they receive from the pool, constituting **theft of unclaimed yield**. The staker captures the difference.

This matches the allowed impact: **High — Theft of unclaimed yield**.

---

### Likelihood Explanation

Any staker with an active pool can execute this after their commitment expires. No privileged role, leaked key, or external dependency is required. The call to `set_commission_commitment` is permissionless for the staker: [4](#0-3) 

The staker simply waits for their commitment to expire (at most one year, per the `expiration_epoch` bound), then executes the two-step sequence. This is a realistic, low-effort attack.

---

### Recommendation

In `set_commission_commitment`, when replacing an expired commitment, add a check that the new `max_commission` does not exceed the expired commitment's `max_commission`:

```cairo
if let Option::Some(old_commitment) = staker_pool_info.commission_commitment.read() {
    if !self.is_commission_commitment_active(old_commitment) {
        // Expired: new max_commission must not exceed the old max_commission
        assert!(
            max_commission <= old_commitment.max_commission,
            "{}",
            Error::MAX_COMMISSION_EXCEEDS_EXPIRED_COMMITMENT,
        );
    } else {
        panic!("{}", Error::COMMISSION_COMMITMENT_EXISTS);
    }
}
```

This ensures that once a commitment expires, the staker cannot use the commitment mechanism to raise commission above the previously committed ceiling.

---

### Proof of Concept

1. Staker stakes and opens a pool. Commission = 50%.
2. Staker calls `set_commission_commitment(max_commission: 60%, expiration_epoch: curr_epoch + 10)`.
3. Advance past `expiration_epoch`.
4. Staker calls `set_commission_commitment(max_commission: 95%, expiration_epoch: curr_epoch + 10)`.
   - Passes: old commitment is expired, `50% <= 95%`.
5. Staker calls `set_commission(commission: 90%)`.
   - Passes: new commitment active, `90% <= 95%`, `90% != 50%`.
6. Commission is now 90%. Delegators' yield is cut by ~(90−50)/(100−50) = 80% relative to what they expected. [5](#0-4) [6](#0-5)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
