### Title
Missing `app_pk` Consistency Enforcement in CKD Protocol Allows Single Malicious Participant to Corrupt Derived Key — (`src/confidential_key_derivation/protocol.rs`)

---

### Summary

The CKD protocol passes `app_pk` as an independent, unchecked parameter to each participant's `ckd()` call. No commitment, broadcast, or cross-participant consistency check is performed on `app_pk` at any point in the protocol. A single malicious MPC participant can call `ckd()` with a different `app_pk'` than the one agreed upon, causing the coordinator's aggregated `CKDOutput` to be permanently unmasked incorrectly by the application.

---

### Finding Description

In `compute_signature_share`, each participant computes:

```
C_i = S_i + y_i * app_pk      // line 174
Y_i = y_i * G                  // line 165
```

and sends `(λ_i * Y_i, λ_i * C_i)` to the coordinator. [1](#0-0) 

The coordinator in `do_ckd_coordinator` blindly sums all received shares:

```rust
norm_big_y += participant_output.big_y();
norm_big_c += participant_output.big_c();
``` [2](#0-1) 

There is no check that the received `(Y_i, C_i)` pairs were computed with the same `app_pk`. The `ckd()` entry point performs only structural checks (participant count, duplicates, self/coordinator presence) and accepts any `app_pk` value without validation: [3](#0-2) 

`unmask` recovers the confidential key as:

```
C - app_sk * Y = msk * H(pk||app_id)
``` [4](#0-3) 

This cancellation only works if every participant used the same `app_pk = app_sk * G`. If participant `j` used `app_pk'` instead, the aggregated `C` becomes:

```
C = msk*H(pk||app_id) + app_pk * Σ_{i≠j}(λ_i*y_i) + app_pk' * λ_j*y_j
```

Then `unmask(app_sk)` yields:

```
msk*H(pk||app_id) + λ_j*y_j*(app_pk' - app_pk)
```

which is a random, non-zero garbage point — not the expected confidential key.

---

### Impact Explanation

The application receives a `CKDOutput` whose `unmask()` returns a corrupted point. The derived confidential key is permanently wrong for that request. The corruption is silent — the coordinator returns `Ok(Some(ckd_output))` with no error, and the application has no way to detect the corruption without an independent oracle for the expected key. This matches the **High** impact: *Corruption of CKD outputs so honest parties accept unusable cryptographic outputs*.

---

### Likelihood Explanation

A single malicious MPC node (one of `n` participants) can trigger this by passing any `app_pk' ≠ app_pk` to its own `ckd()` call. No coordination with other attackers is needed. The attack requires no cryptographic break, no leaked keys, and no external assumptions — only the ability to call the public `ckd()` API with a chosen `app_pk` argument, which every participant does independently. The protocol has zero defenses against this deviation.

---

### Recommendation

Include `app_pk` in the protocol's consistency-enforcement layer. Concretely:

1. **Commit-and-reveal**: Before computing shares, have each participant broadcast a commitment to their `app_pk` (e.g., a hash). The coordinator verifies all commitments match before aggregating.
2. **Echo broadcast**: Have the coordinator echo the `app_pk` it received (or intends to use) back to all participants before they send their shares, so participants can abort if it differs from their own.
3. **Transcript binding**: Include `app_pk` in a session transcript that is signed or hashed into the protocol, so any deviation is detectable.

---

### Proof of Concept

```rust
// Differential test: one participant uses wrong app_pk
// All participants use correct app_pk except participant[1] who uses app_pk_wrong

let app_sk = Scalar::random(&mut rng);
let app_pk = ElementG1::generator() * app_sk;
let app_pk_wrong = ElementG1::generator() * Scalar::random(&mut rng); // attacker's key

for (i, p) in participants.iter().enumerate() {
    let chosen_pk = if i == 1 { app_pk_wrong } else { app_pk };
    let protocol = ckd(&participants, coordinator, *p, key_pairs[i].clone(),
                       app_id.clone(), chosen_pk, rng_p).unwrap();
    protocols.push((*p, Box::new(protocol)));
}

let result = run_protocol(protocols).unwrap();
let ckd_output = check_one_coordinator_output(result, coordinator).unwrap();

// unmask returns garbage, not msk * H(pk || app_id)
let confidential_key = ckd_output.unmask(app_sk);
let expected = hash_app_id_with_pk(&pk, &app_id) * msk;

assert_ne!(confidential_key, expected); // PASSES — output is corrupted
```
