### Title
ETH Permanently Locked in `RewardSupplier` When `tick()` Called With No Pending Mint Requests - (File: L1/starkware/solidity/stake/RewardSupplier.sol)

### Summary
`RewardSupplier.tick()` is `payable` and uses `msg.value` to pay bridge and messaging fees, but when `amountToMint == 0` (no pending L2 mint requests), any ETH sent by the caller is silently accepted and permanently locked in the contract. There is no `receive()` function, no ETH withdrawal function, and no refund path.

### Finding Description
`tick()` is declared `external payable` and splits `msg.value` between two fee-bearing calls when `amountToMint > 0`:

```solidity
uint256 msgFee = msg.value / 2;
bridge().depositWithMessage{value: msgFee}(...);
msgFee = msg.value - msgFee;
messagingContract().sendMessageToL2{value: msgFee}(...);
```

However, when `requiredMinting()` returns `amountToMint == 0`, the entire `if (amountToMint > 0)` block is skipped. Any ETH attached to the call is accepted by the `payable` function but never forwarded, refunded, or otherwise recoverable. The contract has no `receive()` fallback, no `withdraw()` function, and no other ETH egress path. [1](#0-0) 

The `requiredMinting()` function returns `(0, 0)` whenever there are no pending L2→L1 mint-request messages: [2](#0-1) 

The spec confirms `tick()` is callable by `anyAccount`: [3](#0-2) 

### Impact Explanation
Any ETH sent to `tick()` when there are no pending mint requests is permanently locked in the `RewardSupplier` contract. There is no administrative withdrawal, no `receive()` function, and no refund logic. The caller suffers a direct, irrecoverable ETH loss. This maps to **Medium: Griefing with no profit motive but damage to users or protocol**.

### Likelihood Explanation
**High.** `tick()` is a public, permissionless function intended to be called by any account. Callers must supply ETH to cover StarkGate bridge fees and Starknet messaging fees. A caller who sends ETH when the L2 mint-request queue happens to be empty (e.g., between reward epochs, or after another caller already drained the queue in the same block) loses all attached ETH with no warning or revert. This is a routine operational scenario.

### Recommendation
Add a guard that reverts (or refunds) when ETH is sent but there is nothing to mint:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();
    if (amountToMint == 0) {
        require(msg.value == 0, "NO_MINT_REQUIRED_ETH_REFUNDED");
        return;
    }
    // ... existing logic ...
}
```

Alternatively, add a `receive()` function that reverts to prevent accidental ETH deposits, and ensure the guard above is in place.

### Proof of Concept
1. L2 mint-request queue is empty (no pending `MintRequest` messages from L2).
2. Caller invokes `RewardSupplier.tick{value: 1 ether}()`.
3. `requiredMinting()` returns `(0, 0)` → `amountToMint == 0`.
4. The `if (amountToMint > 0)` block is skipped entirely.
5. The function returns normally; `1 ether` is now held by `RewardSupplier` with no withdrawal path.
6. The ETH is permanently locked. [1](#0-0)

```text
    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
```
