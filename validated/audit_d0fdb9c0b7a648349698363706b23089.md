### Title
Unverified NFT media rendered in `SandboxedIframe` when hash check fails — (`packages/gui/src/components/nfts/NFTPreview.tsx`)

### Summary
`NFTPreview.preparePreview` renders NFT media content fetched via `getURI` without checking `preview.isVerified`, allowing an NFT minter who serves hash-mismatched content to display arbitrary attacker-controlled images/video/audio in the wallet UI.

### Finding Description
`useNFTVerifyHash.findValidUri` returns `{isVerified: false, uri}` when all hash checks fail. The `previewState` memo falls back to this unverified state. `preparePreview` calls `getURI(preview.uri)` and renders the result in `SandboxedIframe` with no guard on `preview.isVerified` — only checking `cachedURI.startsWith('cache://')`.

### Impact Explanation
An attacker-minted NFT can display a spoofed image (e.g., a fake Bored Ape) in offer/transfer confirmation flows, causing users to accept offers for a worthless NFT. Impact is bounded to media spoofing — no script execution is possible due to `sandbox=""` and `script-src 'none'`.

### Likelihood Explanation
Requires minting an NFT with a mismatched hash (trivial on-chain operation) and the victim opening the NFT detail or offer confirmation view. The `NFTHashStatus` warning is present but small and likely ignored by non-technical users. Exploitability is contingent on `getChecksum` caching content on hash-mismatch (standard cache behavior, but unconfirmed from available code).

### Recommendation
Add an `isVerified` guard in `preparePreview` before rendering:
```typescript
if (!preview.isVerified) {
  setPreviewContent(undefined, signal);
  return;
}
```
Or alternatively, display a placeholder/blur when `isVerified` is false, similar to how `blurPreview` works for sensitive content.

### Proof of Concept
1. Mint an NFT with `dataHash = sha256("legitimate_image")` but serve a different image at `dataUri`.
2. Open the NFT detail view in the wallet.
3. Observe: `NFTHashStatus` shows "Invalid hash" (error chip), but `SandboxedIframe` renders the attacker-supplied image.
4. Confirm `blurPreview` is not applied (no `sensitive_content` in metadata).
5. Open an offer for this NFT — the spoofed image appears in the offer confirmation UI.
