### Title
Unrestricted `update_rewards(disable_rewards: true)` Permanently Freezes Per-Block Yield for All Stakers — (File: `src/staking/staking.cairo`)

---

### Summary

The public `update_rewards` function in the Staking contract accepts a caller-controlled `disable_rewards` boolean. When any unprivileged caller invokes it with `disable_rewards = true`, the function advances the **global** `last_reward_block` checkpoint without minting or distributing any rewards. Because the per-block guard is global (not per-staker), that block's rewards are permanently unrecoverable for every staker in the protocol.

---

### Finding Description

`update_rewards` is part of the `IStakingRewardsManager` interface and carries no caller restriction beyond `general_prerequisites()` (unpaused + non-zero caller). [1](#0-0) 

The function first writes the current block number into the **single, global** `last_reward_block` storage slot: [2](#0-1) 

Then, if `disable_rewards` is `true`, it returns immediately — no rewards are calculated, no `unclaimed_rewards` are incremented in the `RewardSupplier`, and no STRK is transferred: [3](#0-2) 

Because `last_reward_block` is a scalar (not a per-staker map): [4](#0-3) 

any subsequent call to `update_rewards` for the **same block** — including the legitimate block-proposer's call — will revert with `REWARDS_ALREADY_UPDATED`: [5](#0-4) 

The rewards that should have been minted for that block are never requested from the `RewardSupplier` and are permanently lost. The `RewardSupplier`'s `unclaimed_rewards` counter is never incremented, so no L1 mint request is triggered either. [6](#0-5) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(<any_active_staker>, true)` once per block causes the entire block's STRK (and BTC) reward allocation to be silently discarded. Because `last_reward_block` is global, this simultaneously blocks every other staker from receiving rewards for that block. Repeated across many blocks, this drains the protocol's yield stream entirely. The lost rewards are not deferred — they are gone, because the per-block reward calculation is never executed and the `RewardSupplier` is never notified.

---

### Likelihood Explanation

**High.** The function is permissionless; the only prerequisites are that the contract is unpaused and the caller is non-zero. The attacker must supply any currently-active staker address (trivially obtained from on-chain events or the public `stakers` vector). The gas cost per call is the only barrier. The attack is repeatable every block indefinitely. [7](#0-6) 

---

### Recommendation

1. **Restrict the caller** of `update_rewards` to a trusted role (e.g., the block-proposer's operational address, or a dedicated consensus contract), mirroring how `update_rewards_from_attestation_contract` is restricted to the attestation contract.
2. **Remove or gate the `disable_rewards` parameter** — if skipping reward distribution is needed for migration, handle it inside a privileged path rather than exposing it to arbitrary callers.
3. Alternatively, make `last_reward_block` a **per-staker** mapping so that one caller's no-op call cannot consume the global slot for all other stakers.

---

### Proof of Concept

```
// Attacker (any EOA with a non-zero address) executes once per block:

// 1. Pick any address currently in `stakers` vec — call it `victim`.
// 2. Call:
staking.update_rewards(victim, disable_rewards: true);
// → last_reward_block is set to current_block_number.
// → No rewards are distributed; RewardSupplier.unclaimed_rewards unchanged.

// 3. The legitimate block-proposer now calls:
staking.update_rewards(proposer_staker, disable_rewards: false);
// → PANICS: "REWARDS_ALREADY_UPDATED"
// → Block proposer receives zero rewards for this block.
// → Rewards are permanently lost (never minted, never distributed).

// Repeat every block → all consensus-era block rewards are frozen.
``` [8](#0-7)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
