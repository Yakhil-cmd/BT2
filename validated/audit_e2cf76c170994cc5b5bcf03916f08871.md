### Title
Unprivileged Caller Can Permanently Freeze All Stakers' Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
`update_rewards` is a publicly callable function that accepts a user-controlled `disable_rewards` boolean. When set to `true`, it advances the global `last_reward_block` checkpoint without distributing any rewards, permanently consuming that block's reward slot for every staker in the protocol.

### Finding Description
`update_rewards` in `StakingRewardsManagerImpl` carries no role guard — `general_prerequisites` only asserts the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function writes `last_reward_block` to the current block **before** the early-return branch: [2](#0-1) 

`last_reward_block` is a single global storage slot shared across all stakers: [3](#0-2) 

Because the guard `current_block_number > self.last_reward_block.read()` enforces exactly one call per block: [4](#0-3) 

…once an attacker calls `update_rewards(any_active_staker, disable_rewards: true)` in block N, no legitimate caller can distribute rewards for block N. The slot is gone. Repeating this every block permanently starves the entire protocol of consensus-era block rewards.

The analog to the ZKPay bug is exact: in ZKPay a user-supplied `asset == NATIVE_ADDRESS` parameter bypasses the actual-transfer check and inflates accounting; here a user-supplied `disable_rewards: true` parameter bypasses the reward-distribution step while still advancing the accounting checkpoint (`last_reward_block`), producing the inverse effect — deflating rewards to zero.

### Impact Explanation
`last_reward_block` is global. A single call per block with `disable_rewards: true` silently discards that block's STRK (and BTC) rewards for **every** staker and every delegation pool in the protocol. Sustained over time this constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

### Likelihood Explanation
High. The function is permissionlessly callable by any non-zero address. No stake, no role, no special state is required. Gas cost is the only barrier, and on Starknet L2 that is negligible.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a dedicated consensus-rewards caller role), mirroring the access control already applied to `update_rewards_from_attestation_contract`: [5](#0-4) 

Add an analogous `assert_caller_is_consensus_contract()` (or reuse the attestation contract check) at the top of `update_rewards`.

### Proof of Concept
1. Any address `attacker` calls `staking.update_rewards(any_active_staker_address, disable_rewards: true)` in block N.
2. `last_reward_block` is written to N; the function returns before `_update_rewards` is reached.
3. Any subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`.
4. No staker or pool receives block rewards for block N.
5. Attacker repeats in block N+1, N+2, … — zero gas profit, total protocol reward freeze.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
