### Title
Unprivileged Caller Can Freeze All Staker Rewards Per Block via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable with no access control beyond a zero-address check. It accepts a `disable_rewards: bool` parameter that, when set to `true`, advances the global `last_reward_block` cursor without distributing any rewards. Because `last_reward_block` is a single global slot shared across all stakers, an attacker can call `update_rewards(any_active_staker, disable_rewards: true)` once per block to permanently prevent reward distribution for every staker in the protocol.

### Finding Description

`update_rewards` is exposed as a public ABI function under `IStakingRewardsManager`:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only checks: not paused, caller != zero
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // ← last_reward_block is written BEFORE the disable_rewards branch
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;   // ← exits without distributing any rewards
        }
        ...
    }
``` [1](#0-0) 

`general_prerequisites` only enforces two invariants — the contract is not paused and the caller is not the zero address: [2](#0-1) 

`last_reward_block` is a single global storage slot, not per-staker: [3](#0-2) 

The write to `last_reward_block` at line 1485 occurs unconditionally, before the `disable_rewards` guard at line 1487. Any subsequent call to `update_rewards` in the same block — regardless of which staker is targeted — will revert with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

### Impact Explanation

An attacker who calls `update_rewards(active_staker, disable_rewards: true)` once per block:

1. Advances `last_reward_block` to the current block number.
2. Skips the entire `_update_rewards` path, so no STRK rewards are credited to any staker's `unclaimed_rewards_own` and no pool rewards are forwarded.
3. Blocks every other legitimate call to `update_rewards` for that block.

Because the attack costs only one transaction per block and requires no capital, it can be sustained indefinitely. The result is a **permanent (sustained) freezing of unclaimed yield** for all stakers and pool members in the protocol. This matches the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"* and *"Griefing with no profit motive but damage to users or protocol"*.

### Likelihood Explanation

- No special role, key, or capital is required — any non-zero address suffices.
- The attacker only needs to know one active staker address, which is trivially discoverable from `NewStaker` events.
- The per-block gas cost on Starknet is low enough to make sustained automation economically viable.
- The attack is deterministic and cannot be front-run away from by the protocol.

### Recommendation

Restrict `update_rewards` to a trusted caller. The most straightforward fix is to require the caller to be either the staker themselves, their operational address, or a designated rewards-updater role. Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "no reward" case internally based on on-chain attestation state, so that the `last_reward_block` cursor is only advanced when rewards are actually computed.

### Proof of Concept

1. Attacker monitors the mempool for the start of each new block.
2. Attacker submits `staking_contract.update_rewards(known_active_staker, disable_rewards: true)` as the first transaction in every block.
3. `last_reward_block` is set to the current block number; no rewards are distributed.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Stakers and pool members accumulate zero rewards indefinitely while the attack is active.

The root cause — `last_reward_block` being written before the `disable_rewards` guard, with no caller restriction — is entirely within `src/staking/staking.cairo` and requires no external dependency or privileged access to exploit. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
