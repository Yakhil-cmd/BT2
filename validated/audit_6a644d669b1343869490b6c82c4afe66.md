### Title
Unconstrained `disable_rewards` Parameter in `update_rewards` Allows Permanent Freezing of All Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract unconditionally advances the global `last_reward_block` state variable, but conditionally skips the actual reward calculation when `disable_rewards = true`. Because the function is publicly callable with no access control, any unprivileged caller can invoke it with `disable_rewards: true` on every new block, permanently preventing all stakers from ever receiving consensus-phase rewards.

### Finding Description
In `src/staking/staking.cairo`, the `StakingRewardsManagerImpl::update_rewards` function is exposed via `#[abi(embed_v0)]` with no caller restriction beyond the generic pause check (`general_prerequisites()`). [1](#0-0) 

The function body:

1. Asserts `current_block_number > last_reward_block` — enforcing a one-call-per-block invariant.
2. **Unconditionally** writes `last_reward_block = current_block_number`.
3. **Conditionally** skips the entire reward calculation when `disable_rewards || is_pre_consensus()`. [2](#0-1) 

The `last_reward_block` write at line 1485 is the "reward debt" equivalent from the Limbo pattern: it permanently marks the block as processed, making it impossible for any subsequent caller to trigger reward distribution for that block. The actual reward accumulation in `_update_rewards` is only reached when `disable_rewards == false`. [3](#0-2) 

Because `last_reward_block` is a **single global** (not per-staker), a single call with `disable_rewards: true` for block N blocks reward distribution for **all stakers** for that block.

### Impact Explanation
An attacker who calls `update_rewards(any_staker_address, disable_rewards: true)` on every new block will:

- Advance `last_reward_block` to the current block on each call.
- Prevent any legitimate call (from the staker, a keeper, or the protocol) from distributing consensus rewards for that block.
- Cause **permanent freezing of all unclaimed consensus-phase yield** for every staker and every delegation pool in the protocol, as long as the attack continues.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- The function is publicly callable with no access control.
- Starknet transaction fees are low, making sustained block-by-block griefing economically viable.
- The attacker needs no tokens, no privileged role, and no prior state — any EOA can execute this.
- The attack is trivially automatable with a simple script.

### Recommendation
Restrict who may pass `disable_rewards: true`. Options:

1. **Access-control gate**: Only allow the staking contract itself, the attestation contract, or a governance-controlled address to call `update_rewards` with `disable_rewards: true`.
2. **Separate the two concerns**: Split `update_rewards` into a public `distribute_rewards(staker_address)` (always distributes) and an internal/privileged `advance_reward_block_without_distribution(staker_address)` used only during exit windows.
3. **Remove the parameter entirely**: Derive `disable_rewards` internally from staker state (e.g., `staker_info.unstake_time.is_some()`) rather than accepting it as a caller-supplied argument.

### Proof of Concept

```
// Attacker script (pseudocode, runs once per block):
loop {
    wait_for_new_block();
    staking_contract.update_rewards(
        staker_address: any_valid_staker,
        disable_rewards: true,   // attacker-controlled
    );
    // last_reward_block is now == current_block
    // _update_rewards is never called
    // All stakers earn zero consensus rewards for this block
}
```

Step-by-step:
1. Consensus rewards are activated (`set_consensus_rewards_first_epoch` has been called).
2. Block N arrives. `last_reward_block < N`.
3. Attacker calls `update_rewards(staker_X, disable_rewards: true)`.
4. `last_reward_block` is written to N. [4](#0-3) 
5. Function returns early; `_update_rewards` is never reached. [5](#0-4) 
6. Any legitimate call to `update_rewards` for block N now reverts with `REWARDS_ALREADY_UPDATED`. [6](#0-5) 
7. Repeat for every block → zero consensus rewards ever distributed.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
