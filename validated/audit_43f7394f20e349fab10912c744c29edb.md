### Title
Unprivileged Caller Can Consume the Per-Block Reward Slot via `update_rewards`, Freezing the Legitimate Block Producer's Unclaimed Yield — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and unconditionally writes the current block number into the **global** `last_reward_block` storage variable. Because the function accepts any `staker_address` from any caller, an attacker can call it for an arbitrary staker in any block, consuming that block's single reward slot and permanently preventing the legitimate block producer from claiming their per-block reward.

---

### Finding Description

`update_rewards` is exposed as a fully public entry point with no caller restriction beyond the generic `general_prerequisites` check (non-zero caller, not paused). [1](#0-0) 

The very first substantive action after the prerequisites is to assert that the current block has not yet been processed, then immediately write the current block number into the **single global** `last_reward_block`: [2](#0-1) 

Because `last_reward_block` is a single contract-wide slot (not per-staker), whichever call arrives first in a given block wins the slot for the entire contract: [3](#0-2) 

The attack scenario mirrors the AlchemixGovernor pattern exactly:

| AlchemixGovernor | Starknet Staking |
|---|---|
| `proposalThreshold()` is computed from current `totalSupply` (manipulable) | `last_reward_block` is a global slot (manipulable by any caller) |
| Attacker inflates `totalSupply` so `getVotes(bob, x) < proposalThreshold()` | Attacker calls `update_rewards(staker_B, false)` to set `last_reward_block = N` |
| Bob's `propose()` reverts | Block producer's `update_rewards` reverts with `REWARDS_ALREADY_UPDATED` |

Concretely:

1. Block N is produced by staker A (the legitimate block producer).
2. Attacker calls `update_rewards(staker_B, false)` in block N for any valid staker B.
3. `last_reward_block` is written to N; staker B receives block N's rewards (rewards are misrouted).
4. Staker A calls `update_rewards(staker_A, false)` in block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker A's block N reward is permanently lost.

The attacker can also call with `disable_rewards = true` to purely consume the slot without routing rewards anywhere — pure griefing at zero benefit to anyone. [4](#0-3) 

---

### Impact Explanation

- **Theft of unclaimed yield (High):** When called with `disable_rewards = false`, block rewards are credited to an arbitrary staker B who did not produce the block, diverting yield that belongs to the actual block producer.
- **Permanent freezing of unclaimed yield (High):** If the attacker repeats this every block, the legitimate block producer never accumulates block rewards. Because the slot is consumed before the producer's transaction, the producer's yield is frozen indefinitely.

Both impacts are within the allowed High-severity scope.

---

### Likelihood Explanation

- `update_rewards` is callable by any non-zero address with no special role or token holding required.
- The attacker only needs to submit a transaction in the same block as the block producer's reward claim — feasible on Starknet where transaction ordering is controlled by the sequencer and any party can submit transactions.
- Continuous execution requires gas expenditure but zero capital and zero protocol privilege.
- A competing staker has a direct economic incentive: by blocking all other producers they increase their own relative share of epoch rewards.

---

### Recommendation

1. **Add access control:** Restrict `update_rewards` so only the staker identified by `staker_address` (or a designated consensus caller) can invoke it for themselves.
2. **Per-staker `last_reward_block`:** Replace the single global `last_reward_block` with a `Map<ContractAddress, BlockNumber>` so each staker's reward slot is independent and one staker's call cannot block another's.

---

### Proof of Concept

```
// Pseudocode — no test harness needed; the logic follows directly from the code.

Block N is produced by staker_A.

// Attacker (any address) front-runs staker_A:
staking.update_rewards(staker_B, false);   // succeeds; last_reward_block = N
                                            // staker_B credited with block N rewards

// Staker_A attempts to claim their block reward:
staking.update_rewards(staker_A, false);   // REVERTS: "REWARDS_ALREADY_UPDATED"
                                            // staker_A receives nothing for block N

// Repeat every block → staker_A's unclaimed yield is permanently frozen.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);
