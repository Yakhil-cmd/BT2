### Title
Declare V3 Transactions Bypass `max_l2_gas_amount` Stateless Gateway Check — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary
`StatelessTransactionValidator::validate_resource_bounds` explicitly skips the `max_l2_gas_amount` upper-bound check for `Declare` transactions. No downstream layer re-applies this upper-bound check. A Declare V3 transaction with `l2_gas.max_amount` exceeding `config.max_l2_gas_amount` is admitted to the mempool while an equivalent Invoke or DeployAccount transaction would be rejected.

### Finding Description

In `validate_resource_bounds`, lines 78–80:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { ... });
}
``` [1](#0-0) 

The empty `if let RpcTransaction::Declare(_) = tx {}` arm is a deliberate no-op that causes the entire upper-bound check to be skipped for Declare. The default production value of `max_l2_gas_amount` is `1_210_000_000`. [2](#0-1) 

The existing test `valid_l2_gas_amount_on_declare` already proves the bypass: it constructs a Declare with `max_amount = 200` against a config with `max_l2_gas_amount = 100` and asserts `Ok(())`. [3](#0-2) 

**Downstream pipeline does not compensate:**

- **Stateful validator** `validate_resource_bounds`: only checks `l2_gas.max_price_per_unit` against the previous block's price threshold; `max_amount` is never inspected. [4](#0-3) 

- **Blockifier `check_fee_bounds`**: only enforces a *lower* bound (`minimal_gas_amount > resource_bounds.max_amount` → reject). There is no upper-bound check on `max_amount`. [5](#0-4) 

- **`verify_can_pay_committed_bounds`**: checks `balance >= max_amount × max_price_per_unit`. This is a *balance* constraint, not a policy constraint. For `max_amount = max_l2_gas_amount + 1`, the product is only marginally larger than the normal limit and passes for any account with sufficient balance. [6](#0-5) 

- **Bouncer**: tracks actual gas consumed during execution, not the declared `max_amount`. [7](#0-6) 

### Impact Explanation

The concrete corrupted value is the **admission decision**: a Declare V3 transaction with `l2_gas.max_amount = config.max_l2_gas_amount + 1` is admitted to the mempool when the gateway's own policy mandates rejection. This falls under **High — Mempool/gateway/RPC admission accepts invalid transactions**. The bypass is unconditional for all Declare transactions regardless of how far `max_amount` exceeds the limit, as long as the account balance covers `max_amount × max_price_per_unit`.

### Likelihood Explanation

Any unprivileged user who can submit a Declare transaction and hold sufficient STRK balance to cover the declared fee can trigger this. No special privileges are required. The bypass is structural (not probabilistic), so it is reliably reproducible.

### Recommendation

Remove the empty `if let RpcTransaction::Declare(_) = tx {}` arm and apply the same `max_l2_gas_amount` upper-bound check uniformly to all transaction types, or introduce a separate (potentially higher) `max_l2_gas_amount_declare` config field if Declare transactions legitimately require a different limit.

pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}
