### Title
Staker Can Instantly Spike Commission via Commitment Mechanism, Stealing Delegator Yield — (File: `src/staking/staking.cairo`)

---

### Summary

The `set_commission_commitment` + `set_commission` combination allows a staker to bypass the normal "commission can only decrease" rule and instantly raise their commission from a low value (e.g. 1%) to a near-maximum value (e.g. 99%) within a single block. Delegators have no time to react, and the new commission is applied immediately at the next attestation, causing delegators to receive drastically less yield than expected. The codebase itself acknowledges this gap in a comment.

---

### Finding Description

Under normal conditions, `set_commission` enforces that commission can only decrease: [1](#0-0) 

However, when a staker first calls `set_commission_commitment`, an active commitment unlocks the ability to **increase** commission up to `max_commission`: [2](#0-1) 

The only constraint on `max_commission` when creating the commitment is that it must be ≥ the current commission: [3](#0-2) 

This means a staker with commission = 100 (1%) can:
1. Call `set_commission_commitment(max_commission: 9900, expiration_epoch: current_epoch + 1)` — sets a commitment allowing commission up to 99%.
2. Immediately call `set_commission(commission: 9900)` — jumps commission from 1% to 99%.

Both calls can be submitted in the same block. The codebase explicitly acknowledges this gap: [4](#0-3) 

The commission is read from storage at attestation time inside `update_rewards_from_attestation_contract`, which calls `split_rewards_with_commission` with the live commission value: [5](#0-4) 

`split_rewards_with_commission` deducts commission from pool rewards before forwarding them to the pool contract: [6](#0-5) 

Because the commission change takes effect immediately in storage, the very next attestation distributes rewards using the inflated commission, and delegators receive only the residual `pool_rewards` after the staker has taken 99%.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Delegators who delegated expecting a 1% commission receive approximately 1% of the yield they were entitled to. The staker captures the remaining ~98% as commission. For a pool with large delegated balances, this represents a direct, one-epoch theft of delegator yield. The funds are not frozen; they are redirected to the staker's reward address.

---

### Likelihood Explanation

**Medium.**

The attack requires a staker who is willing to sacrifice their reputation and future delegations for a one-time extraction. On Starknet, blocks are produced in seconds, making it practically impossible for delegators to observe the commitment being set and exit before the commission increase lands. The two transactions (set commitment, increase commission) can be submitted back-to-back, giving delegators zero reaction time. The staker's only cost is loss of future delegations; the gain is the commission difference on all currently delegated funds for the epoch.

---

### Recommendation

1. **Enforce a minimum delay** between setting a `commission_commitment` and being permitted to increase commission — at least one full epoch, so delegators can observe the commitment and exit before it takes effect.
2. **Alternatively**, require that any commission *increase* only takes effect starting from `current_epoch + K` (matching the balance-trace delay already used elsewhere in the protocol), giving delegators the same window they have for other state changes.
3. **Or**, disallow setting `max_commission` more than a small delta above the current commission in a single commitment, preventing large single-step jumps.

---

### Proof of Concept

```
1. Staker stakes with commission = 100 (1%).
   Delegators observe 1% commission and delegate large amounts.

2. Staker calls:
     set_commission_commitment(max_commission: 9900, expiration_epoch: current_epoch + 1)
   This is valid because 9900 >= 100 (current commission).

3. In the same block, staker calls:
     set_commission(commission: 9900)
   This is valid because an active commitment exists and 9900 <= max_commission and 9900 != 100.
   Commission is now 99% in storage.

4. Staker submits attestation → update_rewards_from_attestation_contract executes →
   split_rewards_with_commission(rewards, commission=9900) is called →
   staker receives 99% of pool rewards as commission →
   pool contract receives only 1% of pool rewards.

5. Delegators call claim_rewards and receive ~1% of the yield they expected.
   The remaining ~98% has been transferred to the staker's reward address.
``` [7](#0-6) [8](#0-7) [9](#0-8)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);

                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
