### Title
Missing Zero-Address Check in `change_reward_address` Permanently Freezes Unclaimed Yield - (File: src/staking/staking.cairo, src/pool/pool.cairo)

### Summary
Both `change_reward_address` in the Staking contract and in the Delegation Pool contract accept a zero address as a valid `reward_address` without any guard. Once set to zero, all subsequent reward claims silently transfer accumulated STRK yield to the zero address, permanently destroying it.

### Finding Description
`change_reward_address` in `staking.cairo` performs only one validation — that the new address is not a registered token address — before overwriting the staker's `reward_address` in storage: [1](#0-0) 

The identical gap exists in the pool contract: [2](#0-1) 

Neither function contains `assert!(reward_address.is_non_zero(), ...)` or any equivalent guard.

When `claim_rewards` is later invoked on the staking contract, the internal helper `send_rewards_to_staker` reads the stored `reward_address` and unconditionally transfers the full `unclaimed_rewards_own` balance to it: [3](#0-2) 

In the pool contract, `claim_rewards` performs the same unconditional transfer directly: [4](#0-3) 

A transfer to the zero address on Starknet does not revert; the tokens are irrecoverably lost.

### Impact Explanation
Any accumulated STRK yield that has not yet been claimed is permanently sent to address `0` the next time `claim_rewards` is executed. Because `unclaimed_rewards_own` is zeroed out after the transfer, there is no recovery path. This constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

### Likelihood Explanation
The call is open to every registered staker and every pool member — no privileged role is required. A user who accidentally passes an uninitialised or default-zero variable (a common mistake in scripting or front-end integrations) will silently burn all future rewards with no on-chain warning. The probability of accidental misuse is non-trivial given the function is a routine self-service operation.

### Recommendation
Add an explicit non-zero guard at the top of both `change_reward_address` implementations, mirroring the pattern already used elsewhere in the codebase (e.g., `assert!(amount.is_non_zero(), ...)`):

```cairo
// staking.cairo  and  pool.cairo — change_reward_address
assert!(reward_address.is_non_zero(), "Reward address cannot be zero");
```

Apply the same guard to the `reward_address` parameter of `stake` in `staking.cairo` and `enter_delegation_pool` in `pool.cairo` for defence-in-depth.

### Proof of Concept
1. Staker calls `stake(reward_address: 0x0, ...)` **or** later calls `change_reward_address(reward_address: 0x0)`.
   - `staking.cairo` line 520–523 passes (zero is not a token address).
   - `staker_info.reward_address` is written as `0x0`.
2. Attestation contract calls `update_rewards_from_attestation_contract`, crediting `unclaimed_rewards_own` for the staker.
3. Anyone calls `claim_rewards(staker_address)`.
   - `send_rewards_to_staker` reads `reward_address = 0x0`.
   - `token_dispatcher.checked_transfer(recipient: 0x0, amount: rewards)` executes successfully.
   - `unclaimed_rewards_own` is set to zero.
4. All accrued STRK yield is permanently destroyed; the staker has no recourse.

The same four-step sequence applies to a pool member via `pool.cairo` `change_reward_address` → `claim_rewards`. [5](#0-4) [6](#0-5)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
