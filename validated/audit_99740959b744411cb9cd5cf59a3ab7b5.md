### Title
Unprivileged Caller Can Freeze Consensus Rewards by Calling `update_rewards` with `disable_rewards: true` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function validates a condition on a **specific staker** (must be active, non-zero balance) but unconditionally updates the **global** `last_reward_block` state before checking the `disable_rewards` flag. Because the function has no access control beyond basic prerequisites, any unprivileged caller can invoke it with `disable_rewards: true` to mark a block as "processed" without distributing any rewards, permanently blocking legitimate reward distribution for that block.

---

### Finding Description

In `src/staking/staking.cairo`, the `StakingRewardsManagerImpl::update_rewards` function performs per-staker validation but writes a global side-effect before the early-return guard:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... validates specific staker is active and has non-zero balance ...

    // ← GLOBAL STATE WRITTEN HERE, before disable_rewards is checked
    self.last_reward_block.write(current_block_number);   // line 1485

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // line 1487-1489 — no rewards sent
    }
    // ... actual reward distribution ...
}
``` [1](#0-0) 

`general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero — there is no role check. [2](#0-1) 

The global `last_reward_block` is the one-per-block gate for the entire protocol: [3](#0-2) 

Because `last_reward_block.write(current_block_number)` executes **before** the `disable_rewards` branch, any caller who passes the per-staker validation can consume the block's reward slot without distributing anything.

**Analog mapping to the external report:**

| External (vePeg) | Starknet Staking |
|---|---|
| `_delegate()` validates `_from` is perpetually locked | `update_rewards()` validates the named staker is active and has non-zero balance |
| `_moveAllDelegates()` then moves **all** tokens, ignoring the restriction | `last_reward_block.write()` then updates the **global** block gate, ignoring `disable_rewards` |
| Non-perpetual locks gain delegation power they should not have | All other stakers lose their reward slot for that block |

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block:

1. Passes the per-staker assertions (staker is active, non-zero balance).
2. Sets `last_reward_block` to the current block number.
3. Returns early — zero rewards distributed.
4. Every subsequent legitimate `update_rewards` call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

Repeated every block, this **permanently freezes all consensus-era unclaimed yield** for every staker and pool member in the protocol. This matches the allowed impact: *Permanent freezing of unclaimed yield*. [4](#0-3) 

---

### Likelihood Explanation

**High.** The function is a public external entry point with no privileged-role guard. The attacker needs only:
- A valid (active, non-zero-balance) staker address to name — trivially obtained from on-chain events.
- Gas to front-run the legitimate `update_rewards` call each block.

No key compromise, bridge access, or governance action is required.

---

### Recommendation

Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so a no-op call does not consume the block's reward slot:

```cairo
// Update last block rewards only when rewards will actually be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... reward distribution ...
```

Alternatively, restrict `update_rewards` with `disable_rewards: true` to a privileged role (e.g., the attestation contract or a dedicated operator role).

---

### Proof of Concept

1. Staker B is active with non-zero balance (legitimate reward recipient for block N).
2. Attacker calls `update_rewards(staker_B, disable_rewards: true)` in block N.
3. Per-staker assertions pass (staker B is valid).
4. `last_reward_block` is written to N — **global slot consumed**.
5. Function returns early; no rewards sent to staker B or any pool.
6. The consensus mechanism (or any other caller) attempts `update_rewards(staker_B, disable_rewards: false)` in block N.
7. Assertion `current_block_number > last_reward_block` → `N > N` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
8. All stakers and pool members receive zero rewards for block N.
9. Repeating this every block permanently freezes all consensus rewards. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
