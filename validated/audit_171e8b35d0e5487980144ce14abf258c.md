### Title
Griefing via `update_rewards(disable_rewards: true)` permanently blocks consensus reward distribution for all stakers - (File: src/staking/staking.cairo)

### Summary
The public `update_rewards` function writes the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. Because the function has no access control, any unprivileged caller can front-run the legitimate consensus-layer call with `disable_rewards: true`, consuming the one-per-block slot without distributing any rewards. Every subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`, causing stakers to permanently lose the rewards for that block.

### Finding Description
`update_rewards` is part of `StakingRewardsManagerImpl`, an `#[abi(embed_v0)]` public interface. Its only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero — no role restriction exists.

The critical ordering inside the function is:

```
// 1. Block-number guard
assert!(current_block_number > self.last_reward_block.read(), …REWARDS_ALREADY_UPDATED);

// 2. Staker validity checks (staker must exist, be active, have non-zero balance)
…

// 3. ← GLOBAL STATE WRITTEN HERE, before the disable check
self.last_reward_block.write(current_block_number);   // line 1485

// 4. Conditional early-return that skips all reward distribution
if disable_rewards || self.is_pre_consensus() {
    return;                                            // line 1488
}
``` [1](#0-0) 

Because `last_reward_block` is a **single global** variable (not per-staker), writing it in step 3 and then returning in step 4 consumes the entire block's reward-update budget for every staker in the system.

An attacker's recipe:
1. Identify any valid, active staker address with non-zero balance (trivially available from `NewStaker` events).
2. At the start of each block, submit `update_rewards(valid_staker, disable_rewards: true)`.
3. `last_reward_block` is set to the current block; the function returns without distributing rewards.
4. The legitimate call (from the consensus layer or any staker) reverts with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

### Impact Explanation
In the consensus-rewards phase (`!is_pre_consensus()`), block rewards are the sole mechanism for distributing STRK to stakers and their delegators. Each missed block means permanently lost yield — the reward for that block is never re-queued. If the attacker repeats this every block, all consensus-phase yield is permanently frozen for every staker and delegator in the protocol. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- The function is unconditionally public; no role or whitelist check exists.
- A valid staker address is trivially obtained from on-chain `NewStaker` events.
- The attacker pays only the gas cost of one `update_rewards` call per block; no capital is at risk.
- On Starknet the sequencer orders transactions, so a well-positioned attacker (or a malicious sequencer) can reliably place their call first in every block.
- Likelihood: **Medium-High** — economically cheap, technically straightforward.

### Recommendation
Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` check, so that a call that skips reward distribution does not consume the block's update slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only mark the block as processed when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
…
```

Alternatively, restrict `update_rewards` to a privileged caller (e.g., the consensus contract address stored in `attestation_contract`) so that unprivileged accounts cannot invoke it at all.

### Proof of Concept
```
Block N arrives.

Attacker tx (lands first):
  staking.update_rewards(
      staker_address = <any valid staker>,
      disable_rewards = true
  )
  → last_reward_block := N
  → returns early, zero rewards distributed

Legitimate consensus tx (lands second, same block):
  staking.update_rewards(
      staker_address = <intended staker>,
      disable_rewards = false
  )
  → assert!(N > last_reward_block /* N */) FAILS
  → reverts with REWARDS_ALREADY_UPDATED

Result: all stakers lose block-N rewards permanently.
Repeat every block → permanent freeze of all consensus-phase yield.
``` [3](#0-2)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
