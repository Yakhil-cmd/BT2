### Title
Unrestricted `update_rewards` with `disable_rewards` Flag Allows Any Caller to Permanently Freeze Staker Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract carries no caller restriction and exposes a `disable_rewards: bool` parameter. A single global `last_reward_block` variable gates the function to one call per block. An unprivileged attacker can call `update_rewards(victim_staker, disable_rewards: true)` before the legitimate sequencer call in any block, consuming the block's reward slot without distributing rewards. Repeated every block, this permanently freezes the targeted staker's consensus-phase yield.

### Finding Description
`update_rewards` is part of the public `IStakingRewardsManager` interface and is designed to be invoked by the sequencer/consensus layer once per block to credit the block-producing staker. However, no access control is applied.

The function first enforces a per-block gate using a **single global** `last_reward_block`:

```cairo
// src/staking/staking.cairo  lines 1452-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

It then unconditionally advances the gate:

```cairo
// line 1485
self.last_reward_block.write(current_block_number);
```

Only after that does it branch on `disable_rewards`:

```cairo
// lines 1487-1489
if disable_rewards || self.is_pre_consensus() {
    return;
}
```

Because the gate is written **before** the rewards branch, any caller who arrives first in a block and passes `disable_rewards: true` will:
1. Advance `last_reward_block` to the current block.
2. Return immediately without distributing any rewards.
3. Cause every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

This is the on-chain analog of the "event-based" authentication described in the external report: the authorization check (`current_block_number > last_reward_block`) is a plain state comparison, not cryptographically bound to the identity of the legitimate caller. Just as the mobile wallet's `if result.success` check can be bypassed by modifying state before the check runs, the reward-distribution gate here can be consumed by an adversary before the sequencer's legitimate call executes.

### Impact Explanation
An attacker who front-runs the sequencer's `update_rewards` call with `disable_rewards: true` causes the targeted staker to receive zero consensus rewards for that block. Because the gate is global and per-block, the loss is irrecoverable for that block. Repeated every block, this constitutes **permanent freezing of unclaimed yield** for the victim staker — a High-severity impact under the allowed scope.

### Likelihood Explanation
On Starknet the sequencer currently controls transaction ordering, so reliable front-running requires either (a) the attacker being the sequencer, (b) a future decentralised sequencer environment where mempool ordering is contestable, or (c) any window in which the sequencer does not include its own `update_rewards` call in a given block. The function is fully public and callable by any non-zero address, so the attack surface is permanently open. Likelihood is **Medium**: not trivially exploitable today under a centralised sequencer, but structurally guaranteed to be exploitable as sequencer decentralisation progresses.

### Recommendation
Restrict `update_rewards` to a single authorised caller (e.g., a dedicated consensus-layer address stored in contract state, analogous to how `update_rewards_from_attestation_contract` is gated by `assert_caller_is_attestation_contract`). The `disable_rewards` flag should only be settable by that trusted caller. This is the "result-based" fix: binding the reward-distribution outcome to a cryptographically verified caller identity rather than a plain state check.

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. In block N, the sequencer prepares to call `update_rewards(staker_A, disable_rewards: false)`.
3. Attacker submits `update_rewards(staker_A, disable_rewards: true)` and it is included first in block N.
4. `last_reward_block` is written to N; function returns without distributing rewards.
5. The sequencer's call in block N reverts: `REWARDS_ALREADY_UPDATED`.
6. `staker_A` receives zero rewards for block N; the block's reward allocation is permanently lost.
7. Attacker repeats steps 2–6 every block, permanently freezing `staker_A`'s consensus yield. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
