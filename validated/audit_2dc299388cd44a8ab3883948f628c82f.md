### Title
Anyone Can Call `update_rewards()` With `disable_rewards=true` to Permanently Freeze Consensus Block Reward Distribution — (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. When called with `disable_rewards: true`, the function writes the current block number to the **global** `last_reward_block` storage slot *before* checking the flag, then returns early without distributing any rewards. Because `last_reward_block` is a single global gate that prevents any reward distribution in the same block, an unprivileged attacker can call this function with `disable_rewards: true` in every block to permanently freeze all consensus block reward distribution for every staker.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` has no caller-identity check. Its execution path is:

1. `general_prerequisites()` — only checks the pause flag. [1](#0-0) 
2. Assert `current_block_number > last_reward_block` — a **global** gate. [2](#0-1) 
3. Assert staker exists and is active with non-zero balance. [3](#0-2) 
4. **Write** `last_reward_block = current_block_number` — unconditionally, before the flag check. [4](#0-3) 
5. `if disable_rewards || self.is_pre_consensus() { return; }` — exits without distributing rewards. [5](#0-4) 

`last_reward_block` is a single global `BlockNumber` field, not a per-staker map: [6](#0-5) 

Because step 4 commits the global gate **before** step 5 checks `disable_rewards`, any caller who passes `disable_rewards: true` consumes the block's reward slot for every staker without distributing anything. Any subsequent legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`.

The analog to the external report is exact:
- **External**: `executeMigration()` checks a timelock and a "done" flag but never checks whether the required quorum was reached, so the critical precondition is silently bypassed.
- **Analog**: `update_rewards()` checks the block gate and staker validity but never restricts who may set `disable_rewards`, so the critical reward-distribution step is silently bypassed while the global gate is consumed.

---

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker, true)` in every block permanently prevents all consensus block rewards from being distributed to every staker in the protocol. This matches the allowed High impact: **Permanent freezing of unclaimed yield**.

Even sporadic calls (e.g., targeting the first block of each epoch, when `calculate_block_rewards` is most valuable) constitute **griefing with damage to users**, matching the Medium impact tier.

---

### Likelihood Explanation

- The function is fully public with no role or identity check.
- The only precondition is a valid, active staker address — trivially obtained from on-chain events (`NewStaker`).
- The cost is one transaction per block (gas only); no capital is required.
- The attack is silent: no on-chain signal distinguishes a legitimate `disable_rewards: false` call from a malicious `disable_rewards: true` call.

---

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a call that skips reward distribution does not consume the block's reward slot:

```cairo
// Check disable flag BEFORE committing last_reward_block.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... proceed with reward distribution
```

Alternatively, restrict `update_rewards` to a permissioned caller (e.g., the attestation contract or an operator role), or remove the `disable_rewards` parameter from the public ABI entirely and handle the skip logic internally.

---

### Proof of Concept

```
Setup:
  - victim_staker = any active staker address (read from NewStaker events)

Attack loop (executed by attacker in every block N):
  1. staking.update_rewards(victim_staker, disable_rewards=true)
     → last_reward_block := N          (global gate consumed)
     → early return, zero rewards distributed

  2. Any legitimate keeper call to update_rewards in block N:
     → assert(N > last_reward_block)   FAILS: REWARDS_ALREADY_UPDATED

Result:
  - last_reward_block advances every block
  - No consensus block rewards are ever distributed
  - All stakers' unclaimed consensus yield is permanently frozen
```
