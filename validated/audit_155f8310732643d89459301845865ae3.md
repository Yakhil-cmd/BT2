### Title
`#[derive(Debug)]` on Secret-Bearing Output Types Exposes Presign Nonce Shares and Private Signing Shares — (File: `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`, `src/lib.rs`)

### Summary
`PresignOutput` (OT-based), `RerandomizedPresignOutput`, `PresignOutput` (Robust), and `KeygenOutput` all derive `Debug` while simultaneously implementing `ZeroizeOnDrop`. The auto-derived `Debug` implementation prints the raw scalar values of every field, including secret nonce shares (`k`, `sigma`) and the private signing share (`private_share`). Any caller that formats one of these values with `{:?}` — in a log line, a panic message, or an error wrapper — discloses the secret material verbatim.

### Finding Description

Three distinct output types carry secret scalar fields and derive `Debug`:

**OT-based ECDSA — `src/ecdsa/ot_based_ecdsa/mod.rs`**

    pub sigma: Scalar,  // secret sigma share

The `ZeroizeOnDrop` derive on all four types is an explicit acknowledgement by the library authors that these fields are secret and must be erased from memory. The simultaneously present `#[derive(Debug)]` contradicts that intent: Rust's auto-derived `Debug` implementation iterates over every field and formats it with its own `Debug` impl, which for `frost_core::Scalar` and `frost_core::keys::SigningShare` prints the underlying byte representation of the scalar. The result is that a single `format!("{:?}", presign_output)` or `log::debug!("{presign_output:?}")` call anywhere in the call stack — including inside error-handling glue code, tracing middleware, or a `?`-propagated `unwrap_or_else` — emits the raw secret to whatever sink the caller uses (logs, error strings, panic messages, test output).

The vulnerability class is identical to the external report: a standard, expected interface (HTTP error body there; Rust `Debug` formatting here) returns raw internal sensitive details to the caller without sanitization.

### Impact Explanation

- **`PresignOutput.k` and `PresignOutput.sigma`** are the per-party nonce share and sigma share. In the OT-based scheme, `sigma = k⁻¹ · (m + r · x)` where `x` is the private key share. Disclosure of `k` and `sigma` together allows direct recovery of the private key share `x`. This is a Critical impact: extraction of private signing shares and presign nonce material.
- **`KeygenOutput.private_share`** is the raw private signing share produced by DKG. Its disclosure is a direct Critical impact: extraction of the private signing share.
- **Robust ECDSA `PresignOutput.c`, `.e`, `.alpha`, `.beta`** are the secret nonce and masking shares. Their disclosure similarly enables private key reconstruction.

### Likelihood Explanation

Likelihood is **medium**. The trigger is not exotic: `{:?}` formatting is the default Rust idiom for logging, panic messages, and error wrapping. Production integrators routinely write `tracing::debug!(?presign_output, "presign complete")` or `anyhow::bail!("presign failed: {presign_output:?}")`. The `Debug` derive is explicitly present on the public types, so nothing prevents or warns against this usage. The `ZeroizeOnDrop` protection is silently bypassed before the value is dropped.
