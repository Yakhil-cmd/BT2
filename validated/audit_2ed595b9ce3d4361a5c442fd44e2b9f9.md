### Title
Unrestricted `update_rewards` Permanently Consumes Block Reward Slot Without Distributing Rewards - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the Staking contract has no access control. Any caller can invoke it with `disable_rewards=true`, which causes `last_reward_block` to be written (consuming the block's reward slot) before an early return that skips all reward distribution. Because `last_reward_block` is a single global value and re-entry in the same block is blocked by an assertion, the rewards for that block are permanently lost to all stakers.

### Finding Description

The analog vulnerability class is **execution flow bypass**: a pre-action state write executes, then an early `return` skips the post-action, and the pre-action's side-effect makes the post-action permanently unreachable for that execution context.

In `update_rewards` (lines 1449–1507 of `src/staking/staking.cairo`):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: unpaused + non-zero caller
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // PRE-ACTION: consumes the block slot globally
    self.last_reward_block.write(current_block_number);   // ← written BEFORE early return

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← POST-ACTION (reward distribution) skipped
    }
    // reward calculation and distribution only reached here
    ...
    self._update_rewards(...);
}
```

The structural parallel to the reported Nexus bug:

| Nexus `fallback()` | Starknet `update_rewards` |
|---|---|
| `preCheck()` runs | `last_reward_block.write(block)` runs |
| assembly `return` exits early | `if disable_rewards { return; }` exits early |
| `postCheck()` never reached | `_update_rewards(...)` never reached |
| block consumed, hook bypassed | block slot consumed, rewards lost |

`general_prerequisites()` only enforces pause state and non-zero caller — no role check, no operator restriction. [1](#0-0) [2](#0-1) 

### Impact Explanation

`last_reward_block` is a **single global storage slot** shared across all stakers: [3](#0-2) 

When an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N:

1. `last_reward_block` is set to N.
2. The function returns early — `_update_rewards` is never called, so no staker receives rewards for block N.
3. Any subsequent call in block N (including the legitimate consensus mechanism call) hits `REWARDS_ALREADY_UPDATED` and reverts.

Repeating this in every block permanently freezes all unclaimed consensus-epoch yield for all stakers. This matches **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The function is `#[abi(embed_v0)]` — publicly callable with no role guard.
- The attacker only needs any valid, active staker address (all are emitted in `NewStaker` events).
- The attack costs only gas per block and can be fully automated.
- There is no economic barrier or slashing risk for the attacker. [4](#0-3) 

### Recommendation

Two complementary fixes:

1. **Access control**: Restrict `update_rewards` to a trusted consensus-layer caller (e.g., a dedicated role, similar to how `update_rewards_from_attestation_contract` restricts to `assert_caller_is_attestation_contract`). [5](#0-4) 

2. **Move the state write after the guard**: Write `last_reward_block` only after the `disable_rewards` check, so that a no-op call does not consume the block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // moved here
// ... reward distribution
```

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(valid_staker, disable_rewards=true)
    → last_reward_block := N
    → returns early, no rewards distributed

  Consensus tx: update_rewards(staker_A, disable_rewards=false)
    → assert!(N > N)  ← FAILS: REWARDS_ALREADY_UPDATED
    → staker_A receives zero rewards for block N

Repeat every block → all stakers permanently frozen out of consensus rewards.
```

            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();

            let staker_info = self.internal_staker_info(:staker_address);

                "{}",
