### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the Staking contract is documented as callable only by the Starkware sequencer, but the implementation contains no such access control check. Any unprivileged caller can invoke it with `disable_rewards: true` at the start of each block, consuming the single-use `last_reward_block` slot for that block and permanently preventing the legitimate sequencer from distributing rewards for that block. Repeated across every block, this permanently freezes all staker and delegator yield.

### Finding Description

The spec for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation at `StakingRewardsManagerImpl::update_rewards` performs no caller identity check: [2](#0-1) 

The function accepts two attacker-controlled parameters: `staker_address: ContractAddress` and `disable_rewards: bool`. After passing the paused-contract check and the per-block deduplication guard, it unconditionally writes the current block number into the global `last_reward_block` storage slot: [3](#0-2) 

Because `last_reward_block` is a single global variable (not per-staker), writing it once per block with `disable_rewards: true` causes the function to return early — distributing zero rewards — and simultaneously locks out any subsequent call in the same block with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

The attacker only needs to supply any currently-active staker address with non-zero balance (publicly readable via `staker_info_v1`) to pass the remaining guards: [5](#0-4) 

**Attack path:**
1. Attacker monitors the mempool / block production.
2. At the start of each new block, attacker calls `update_rewards(any_active_staker, disable_rewards: true)`.
3. `last_reward_block` is set to the current block; no rewards are emitted.
4. The sequencer's subsequent call with `disable_rewards: false` reverts with `REWARDS_ALREADY_UPDATED`.
5. Rewards for that block are permanently lost — they are never retroactively credited.
6. Attacker repeats every block.

### Impact Explanation

Every block where the attacker front-runs the sequencer, all stakers and delegators permanently lose their block rewards. Because `last_reward_block` is global, a single attacker transaction per block is sufficient to freeze yield for the entire protocol. The lost rewards are never recovered; the `_update_rewards` path that credits `unclaimed_rewards_own` and transfers pool rewards is simply never reached.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**. [6](#0-5) 

### Likelihood Explanation

The function is public with no access control. The only precondition is that a valid active staker exists (trivially satisfied on a live network). The attacker needs no stake, no privileged key, and no special role — only the ability to submit a transaction. On Starknet, front-running the sequencer's own transaction is straightforward since the attacker can submit the griefing call in the same block before the sequencer's call is processed. The cost per block is one cheap transaction with no token transfer.

### Recommendation

Add a sequencer-only access control guard at the top of `update_rewards`, analogous to the pattern used in `update_current_epoch_block_rewards` which restricts callers to the staking contract:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer(); // or assert!(get_caller_address() == SEQUENCER_ADDRESS)
    self.general_prerequisites();
    ...
}
```

Alternatively, split the deduplication guard (`last_reward_block`) from the reward-distribution logic so that a `disable_rewards: true` call does not consume the block slot for a `disable_rewards: false` call.

### Proof of Concept

```
// Attacker script (pseudocode, one call per block):
let active_staker = staking.staker_info_v1(known_staker_address); // any active staker
// Submit at block N, before sequencer:
staking.update_rewards(staker_address: known_staker_address, disable_rewards: true);
// Result: last_reward_block = N, no rewards distributed.
// Sequencer's call at block N:
staking.update_rewards(staker_address: known_staker_address, disable_rewards: false);
// Reverts: REWARDS_ALREADY_UPDATED
// All stakers miss block N rewards permanently.
``` [7](#0-6)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
