### Title
Gateway resource-bounds admission checks only L2 gas price, allowing V3 transactions with zero L1/L1-data gas price to bypass all gateway guards and enter the mempool — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

Both the stateless and stateful gateway validators enforce a minimum price floor only on the **L2 gas** component of `AllResourceBounds`. The L1 gas and L1 data gas `max_price_per_unit` fields are never checked against any threshold at the gateway level. A V3 transaction that sets `l1_gas.max_price_per_unit = 0` (and/or `l1_data_gas.max_price_per_unit = 0`) while keeping `l2_gas.max_price_per_unit ≥ min_gas_price` passes every gateway check and is admitted to the mempool. The blockifier's `check_fee_bounds` — which does validate all three prices — is only reached during batcher execution, not during admission. The transaction is therefore guaranteed to be rejected at execution time with `MaxGasPriceTooLow` for L1Gas, but only after it has already consumed mempool capacity and batcher CPU.

---

### Finding Description

**Stateless validator — only L2 gas price is checked**

`StatelessTransactionValidator::validate_resource_bounds` applies two checks to `AllResourceBounds`:

1. `max_possible_fee(Tip::ZERO) != 0` — passes as long as *any* resource has a non-zero `max_amount × max_price_per_unit`. A transaction with `l2_gas.max_amount = 1, l2_gas.max_price_per_unit = 8_000_000_000` satisfies this even when `l1_gas.max_price_per_unit = 0`.
2. `l2_gas.max_price_per_unit >= config.min_gas_price` (default 8 Gwei) — only L2 gas price is tested.

L1 gas and L1 data gas prices are never compared to any floor. [1](#0-0) 

**Stateful validator — same single-resource gap, acknowledged by a TODO**

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` reads the previous block's `strk_gas_prices.l2_gas_price` and checks only `tx_resource_bounds.l2_gas.max_price_per_unit` against a percentage of that price. The function carries an explicit developer note:

```
// TODO(Arni): Consider running this validation for all gas prices.
```

L1 gas and L1 data gas prices are silently skipped. [2](#0-1) 

**Blockifier — checks all three prices, but only at execution time**

`AccountTransaction::check_fee_bounds` (called from `perform_pre_validation_stage`) iterates over all three resources and compares each `max_price_per_unit` against the actual block gas price:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
```

Because the actual L1 gas price is always non-zero in production, a transaction with `l1_gas.max_price_per_unit = 0` will always fail here with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`. But this check is only reached after the transaction has already been admitted to the mempool and picked up by the batcher. [3](#0-2) 

**Execution flow for the malformed transaction**

```
HTTP → Gateway stateless validate  ✓ (l2_gas price ≥ 8 Gwei)
     → Gateway stateful validate   ✓ (l2_gas price ≥ threshold)
     → Mempool admit               ✓
     → Batcher picks up
     → perform_pre_validation_stage
         handle_nonce              ✓ (nonce consumed)
         check_fee_bounds          ✗ MaxGasPriceTooLow(L1Gas)
     → Transaction rejected, never included in block
``` [4](#0-3) 

---

### Impact Explanation

**High — Mempool/gateway/RPC admission accepts invalid transactions.**

Any unprivileged user can craft a V3 `InvokeTransaction` or `DeclareTransaction` with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` while setting `l2_gas.max_price_per_unit` just above the 8 Gwei floor. Such transactions:

- Pass all gateway admission checks and enter the mempool.
- Are indistinguishable from valid transactions at the mempool level.
- Consume batcher execution slots when dequeued.
- Always fail at `check_fee_bounds` and are never included in any block.
- Consume the sender's nonce (incremented before the fee check fails), which can be used to grief specific accounts.

A sustained flood of such transactions degrades mempool throughput and batcher efficiency without any on-chain cost to the attacker (no fee is charged for a transaction that fails pre-validation before execution).

---

### Likelihood Explanation

**High.** The attack requires no special privilege, no contract deployment, and no knowledge of internal state. Any RPC client can submit a V3 transaction with zero L1 gas price. The gateway config `validate_resource_bounds = true` (the production default) does not prevent this because the flag only gates the existing L2-only checks. [5](#0-4) 

---

### Recommendation

Extend both validators to cover all three gas-price dimensions:

1. **Stateless validator**: add parallel checks for `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against configurable minimums (or at minimum against zero), mirroring the existing L2 check at line 71.

2. **Stateful validator**: resolve the acknowledged TODO — compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the corresponding previous-block prices (`strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price`) using the same percentage-threshold logic already applied to L2 gas. [6](#0-5) 

---

### Proof of Concept

Submit the following V3 invoke transaction via the JSON-RPC `starknet_addInvokeTransaction` endpoint:

```json
{
  "type": "INVOKE",
  "version": "0x3",
  "sender_address": "<any funded account>",
  "calldata": ["<any valid calldata>"],
  "resource_bounds": {
    "l1_gas":      { "max_amount": "0x1", "max_price_per_unit": "0x0" },
    "l2_gas":      { "max_amount": "0x1", "max_price_per_unit": "0x1DCD65000" },
    "l1_data_gas": { "max_amount": "0x1", "max_price_per_unit": "0x0" }
  },
  "tip": "0x0",
  "nonce": "<current account nonce>",
  "nonce_data_availability_mode": "L1",
  "fee_data_availability_mode": "L1",
  "paymaster_data": [],
  "account_deployment_data": [],
  "signature": ["<valid ECDSA signature over the tx hash>"]
}
```

**Expected (broken) behaviour:**
- Gateway stateless check passes: `max_possible_fee = 1 × 0x1DCD65000 > 0`; `l2_gas.max_price_per_unit = 0x1DCD65000 ≥ min_gas_price (8 Gwei)`.
- Gateway stateful check passes: `l2_gas.max_price_per_unit` meets the previous-block threshold.
- Transaction is accepted into the mempool.
- Batcher dequeues it; `check_fee_bounds` fires `MaxGasPriceTooLow { resource: L1Gas }` because the actual L1 gas price is non-zero.
- Transaction is discarded; account nonce is consumed; no fee is charged.

**Expected (correct) behaviour:** The gateway should reject the transaction at admission time with a `GAS_PRICE_TOO_LOW` error for L1Gas, before it ever reaches the mempool.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
