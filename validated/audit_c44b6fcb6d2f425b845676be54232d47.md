### Title
Unbounded `stakers` Vec Growth Causes Unbounded Gas Consumption in `get_stakers` - (File: src/staking/staking.cairo)

### Summary

The `stakers` storage vector in the Staking contract grows permanently with every new staker and is **never pruned on unstake**. The `get_stakers` function iterates over the entire vector on every call. An attacker can cycle through many addresses — staking and then fully unstaking to recover funds — to inflate the vector indefinitely, eventually making `get_stakers` prohibitively expensive or unusable.

### Finding Description

Every call to `stake()` unconditionally appends the new staker address to `self.stakers`: [1](#0-0) 

The storage declaration explicitly documents that entries are never removed: [2](#0-1) 

`get_stakers` then iterates over the **full range** of this ever-growing vector on every invocation: [3](#0-2) 

For each entry it reads storage, checks `is_staker_active`, and conditionally computes staking power — all per-element work that scales linearly with the total number of historical stakers.

An attacker's cycle:
1. Stake with address A (minimum stake required).
2. Call `unstake_intent()`, wait for `exit_wait_window`, call `unstake_action()` — funds returned.
3. Stake with address B using the same funds.
4. Repeat — each cycle permanently adds one entry to `self.stakers`.

### Impact Explanation

`get_stakers` is the consensus query used to determine the active validator set for each epoch. As the vector grows, the gas cost of each call grows proportionally. Once the vector is large enough, the function will exceed the Starknet transaction gas limit, making it impossible to retrieve the staker set. This constitutes **unbounded gas consumption** and can escalate to a permanent freeze of the consensus/attestation mechanism.

**Allowed impact matched**: Unbounded gas consumption (Medium); Temporary/permanent freezing of protocol functionality.

### Likelihood Explanation

- The attacker only needs to cover gas costs per cycle; the staked principal is fully recovered after each `unstake_action`.
- No privileged role is required — any address holding the minimum stake can participate.
- The minimum stake requirement raises the capital cost per cycle but does not prevent the attack; a single attacker with `min_stake` tokens can execute cycles serially.
- Starknet's relatively low gas costs make this practical over time.

### Recommendation

1. **Remove stakers from the vector on `unstake_action`**: Replace the `Vec<ContractAddress>` with a data structure that supports O(1) removal (e.g., a swap-and-pop pattern or an `IterableMap`).
2. **Alternatively, cap the maximum number of registered stakers** and enforce the cap in `stake()`.
3. At minimum, add a guard in `stake()` that rejects re-registration of an address that already exists in the vector (preventing the cycling attack).

### Proof of Concept

```
// Attacker holds `min_stake` tokens.
// Repeat N times:
for i in 0..N {
    // 1. Stake with a fresh address (or the same address after full exit).
    staking.stake(reward_address, operational_address_i, min_stake);

    // 2. Signal exit.
    staking.unstake_intent();

    // 3. Wait exit_wait_window, then recover funds.
    advance_time(exit_wait_window);
    staking.unstake_action(attacker_address_i);

    // Each iteration: self.stakers.len() += 1, funds returned.
}

// After N iterations, get_stakers iterates over N dead entries.
// Gas cost of get_stakers ≈ O(N).
// At large N, the call exceeds the block gas limit.
let stakers = staking.get_stakers(current_epoch); // OOG

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
