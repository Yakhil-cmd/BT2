### Title
Silent CRC21 Mint Failure on Destroyed Contract Causes Permanent IBC Voucher Loss - (File: x/cronos/keeper/evm.go)

### Summary

`CallModuleCRC21` in `x/cronos/keeper/evm.go` does not verify contract existence before dispatching an EVM call. If a registered external CRC21 contract is selfdestructed after its token mapping is recorded, `ConvertCoinFromNativeToCRC21` silently returns success — the EVM treats calls to codeless addresses as successful — while the user's IBC vouchers are permanently consumed without any CRC21 tokens being minted.

### Finding Description

`RegisterOrUpdateTokenMapping` calls `ensureContractCode` at registration time to confirm the target contract has deployed bytecode: [1](#0-0) [2](#0-1) 

However, `ConvertCoinFromNativeToCRC21` — the function that actually moves user funds — performs no such check at call time. It retrieves the stored contract address and immediately calls `CallModuleCRC21`: [3](#0-2) 

For a non-source IBC voucher denom, the function first transfers the user's native coins to the contract's bank address, then calls `mint_by_cronos_module`: [4](#0-3) 

`CallModuleCRC21` calls `CallEVM`, which calls `k.evmKeeper.ApplyMessage`. Per the EVM specification (and the Solidity documentation warning cited in the original report), a call to an address with no deployed code returns `success = true` with empty return data. `res.Failed()` is therefore `false`, and `CallModuleCRC21` returns `([]byte{}, nil)`: [5](#0-4) 

The result: the bank transfer of IBC vouchers to the dead contract address succeeds, `mint_by_cronos_module` is never executed, no CRC21 tokens are minted, and the function returns `nil` — signalling success to all callers.

The same silent-success path applies to source-coin conversion, where `bankKeeper.BurnCoins` destroys the user's coins before `transfer_from_cronos_module` is called on the destroyed contract: [6](#0-5) 

### Impact Explanation

**Critical — Unauthorized burn / permanent accounting loss for IBC vouchers and CRC21 source tokens.**

- For non-source IBC vouchers: the user's vouchers are sent to the dead contract's bank address and are permanently locked there (no code exists to release them). The user receives zero CRC21 tokens.
- For source coins (`cronos0x…` denoms): the user's coins are burned from the bank module before the EVM call. The call silently succeeds, no CRC21 tokens are unlocked, and the burned coins are unrecoverable.

Both outcomes represent a permanent, irreversible loss of user assets with no on-chain recourse.

### Likelihood Explanation

The precondition is that an external CRC21 contract registered via `MsgUpdateTokenMapping` is later selfdestructed. This is realistic because:

1. The admin registers an external contract (a legitimate, privileged action).
2. The contract owner — a separate entity from the admin — calls `selfdestruct` on the contract (e.g., due to a bug, upgrade, or malicious intent). This requires no admin or validator compromise.
3. Any unprivileged user who subsequently calls `MsgConvertVouchers`, or who receives IBC tokens that trigger auto-conversion via `IBCConversionModule.OnRecvPacket` → `ConvertVouchersToEvmCoins`, becomes a victim. [7](#0-6) [8](#0-7) 

The IBC auto-conversion path is particularly dangerous: a user receiving tokens over IBC has no ability to opt out of the conversion, and the `OnRecvPacket` middleware will silently consume their vouchers.

### Recommendation

**Short term:** Add a `ensureContractCode` check inside `ConvertCoinFromNativeToCRC21` immediately after `GetContractByDenom` returns a contract address, before any bank operations are performed. This mirrors the check already present at registration time and ensures that a destroyed contract causes an explicit error rather than a silent fund loss.

```go
contract, found := k.GetContractByDenom(ctx, coin.Denom)
if !found { ... }
// ADD: verify the contract still has deployed code
if err := k.ensureContractCode(ctx, contract); err != nil {
    return fmt.Errorf("mapped contract for denom %s no longer exists: %w", coin.Denom, err)
}
```

**Long term:** Consider adding a periodic or on-demand liveness check for all registered external contracts, and emit a governance-visible event when a mapped contract loses its code. Evaluate whether auto-removal of stale mappings (contracts with no code) is appropriate to prevent future silent failures.

### Proof of Concept

1. Admin calls `MsgUpdateTokenMapping` to register an external CRC21 contract `C` for IBC denom `D`. `ensureContractCode` passes at this point.
2. The owner of contract `C` calls `selfdestruct(owner)`. Contract `C` now has no code in the EVM state.
3. User Alice holds 1000 `D` IBC vouchers and calls `MsgConvertVouchers` with `coins = [{denom: D, amount: 1000}]`.
4. `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` is invoked.
5. `GetContractByDenom` returns `C` (the mapping is still in KV store).
6. `bankKeeper.SendCoins(Alice → C, 1000 D)` succeeds — Alice's 1000 `D` vouchers are now at address `C`.
7. `CallModuleCRC21(C, "mint_by_cronos_module", Alice, 1000)` → `CallEVM` → `ApplyMessage` → EVM call to codeless address → `success = true`, empty return data.
8. `res.Failed()` is `false`; `CallModuleCRC21` returns `(nil, nil)`.
9. `ConvertCoinFromNativeToCRC21` returns `nil`.
10. Alice's 1000 `D` vouchers are permanently locked at the dead contract address. Alice receives 0 CRC21 tokens. [9](#0-8) [10](#0-9)

func (k Keeper) ensureContractCode(ctx sdk.Context, contract common.Address) error {
	if contract.Big().Cmp(big.NewInt(256)) < 0 {
		return errors.Wrapf(sdkerrors.ErrInvalidAddress,
			"crc21 contract must not be in precompile range: %s", contract.Hex())
	}
	resp, err := k.evmKeeper.Code(ctx, &evmtypes.QueryCodeRequest{
		Address: contract.Hex(),
	})
	if err != nil {
		return errors.Wrapf(sdkerrors.ErrInvalidAddress, "failed to query contract code (%s): %v", contract.Hex(), err)
	}
	if resp == nil || len(resp.Code) == 0 {
		return errors.Wrapf(sdkerrors.ErrInvalidRequest, "no contract code at address (%s)", contract.Hex())
	}
	return nil
}

// CallModuleCRC21 call a method of ModuleCRC21 contract
func (k Keeper) CallModuleCRC21(ctx sdk.Context, contract common.Address, method string, args ...interface{}) ([]byte, error) {
	data, err := types.ModuleCRC21Contract.ABI.Pack(method, args...)
	if err != nil {
		return nil, err
	}
	_, res, err := k.CallEVM(ctx, &contract, data, big.NewInt(0), DefaultGasCap)
	if err != nil {
		return nil, err
	}
	if res.Failed() {
		return nil, fmt.Errorf("call contract failed: %s, %s, %s", contract.Hex(), method, res.Ret)
	}
	return res.Ret, nil
}

// ConvertCoinFromNativeToCRC21 convert native token to erc20 token
func (k Keeper) ConvertCoinFromNativeToCRC21(ctx sdk.Context, sender common.Address, coin sdk.Coin, autoDeploy bool) error {
	if !types.IsValidCoinDenom(coin.Denom) {
		return fmt.Errorf("coin %s is not supported for conversion", coin.Denom)
	}
	var err error
	// external contract is returned in preference to auto-deployed ones
	contract, found := k.GetContractByDenom(ctx, coin.Denom)
	if !found {
		if !autoDeploy {
			return fmt.Errorf("no contract found for the denom %s", coin.Denom)
		}
		contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
		if err != nil {
			return err
		}
		if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
			return err
		}

		k.Logger(ctx).Info(fmt.Sprintf("contract address %s created for coin denom %s", contract.String(), coin.Denom))
	}

	isSource := types.IsSourceCoin(coin.Denom)
	coins := sdk.NewCoins(coin)
	if isSource {
		// burn coins
		err = k.bankKeeper.SendCoinsFromAccountToModule(ctx, sdk.AccAddress(sender.Bytes()), types.ModuleName, sdk.NewCoins(coin))
		if err != nil {
			return err
		}
		err = k.bankKeeper.BurnCoins(ctx, types.ModuleName, coins)
		if err != nil {
			return err
		}
		// unlock crc tokens
		_, err = k.CallModuleCRC21(ctx, contract, "transfer_from_cronos_module", sender, coin.Amount.BigInt())
		if err != nil {
			return err
		}
	} else {
		// send coins to contract address
		err = k.bankKeeper.SendCoins(ctx, sdk.AccAddress(sender.Bytes()), sdk.AccAddress(contract.Bytes()), coins)
		if err != nil {
			return err
		}
		// mint crc tokens
		_, err = k.CallModuleCRC21(ctx, contract, "mint_by_cronos_module", sender, coin.Amount.BigInt())
		if err != nil {
			return err
		}
	}

	return nil
}
