### Title
Duplicate Key Bypass: Revoking keyIndex=0 Does Not Revoke Duplicate Keys Pointing to storedKeyIndex=0 — (`fvm/environment/accounts.go`, `fvm/environment/account_public_key_util.go`)

### Summary
When a duplicate key is added at keyIndex=N (mapping to storedKeyIndex=0), revoking keyIndex=0 only sets `Revoked=true` in the `apk_0` register. The `revoked` flag for keyIndex=N in the key metadata weight-and-revoked section remains `false`. `GetRuntimeAccountPublicKey(keyIndex=N)` reads `revoked` from key metadata (not from `apk_0`), so it returns `revoked=false` with key0's public key bytes. The transaction verifier accepts signatures from keyIndex=N using key0's private key, even after keyIndex=0 has been revoked.

### Finding Description

The storage model separates revocation state into two independent locations:

- **keyIndex=0**: revocation is stored inside the `apk_0` register (encoded with the key itself via `flow.EncodeAccountPublicKey`).
- **keyIndex>0**: revocation is stored in the key metadata weight-and-revoked section (in the account status register).

When `AppendDuplicateKeyMetadata` records keyIndex=N as a duplicate of storedKeyIndex=0, it writes `revoked=false` into the weight-and-revoked section for keyIndex=N. [5](#0-4) 

When keyIndex=0 is later revoked, `revokeAccountPublicKey0` only updates the `apk_0` register: [6](#0-5) 

`GetRuntimeAccountPublicKey` for keyIndex=N reads `revoked` from key metadata (line 319), then calls `getStoredPublicKey(storedKeyIndex=0)` which reads only the cryptographic material from `apk_0` — not its `Revoked` field (lines 203–215): [7](#0-6) 

The returned `RuntimeAccountPublicKey` has `Revoked: false` (from key metadata) and `PublicKey: <key0 bytes>` (from `apk_0`). The transaction verifier sees `Revoked=false` and accepts the signature. [8](#0-7) 

### Impact Explanation

An account owner who revokes keyIndex=0 to invalidate a compromised private key believes the cryptographic material is fully revoked. However, any duplicate key at keyIndex=N (added before or during the compromise) continues to authorize transactions with the same private key. The invariant "revoking a key's primary index prevents all authorization using that cryptographic material" is broken. This enables unauthorized transaction authorization on behalf of the victim account.

### Likelihood Explanation

The attack requires that a duplicate key at keyIndex=N was added to the account before keyIndex=0 is revoked. This is realistic in a key-compromise scenario: an attacker who has already obtained key0's private key can add a duplicate key at keyIndex=1 (using a transaction signed with key0), then wait for the account owner to revoke keyIndex=0. After revocation, the attacker retains signing authority via keyIndex=1. The Cadence `AuthAccount.keys.add()` function is the standard production entrypoint.

### Recommendation

When revoking keyIndex=0, also scan the key metadata mapping section for any keyIndex=N entries that map to storedKeyIndex=0, and set their `revoked` flag to `true` as well.

Alternatively, in `GetRuntimeAccountPublicKey` (and `GetAccountPublicKey`) for keyIndex=N where `storedKeyIndex==0`, additionally check the `Revoked` field from `apk_0` and OR it with the key metadata revoked flag:

```go
// After getStoredPublicKey returns for storedKeyIndex==0,
// also read the revoked flag from apk_0 and combine:
if storedKeyIndex == 0 {
    key0, err := getAccountPublicKey0(a, address)
    if err != nil { ... }
    revoked = revoked || key0.Revoked
}
```

### Proof of Concept

```
1. Create account A with key0 (ECDSA P-256, weight=1000).
2. Sign and submit a transaction from A (keyIndex=0) that calls:
       AuthAccount(borrow: A).keys.add(
           publicKey: key0.publicKey,   // same bytes as key0
           hashAlgorithm: HashAlgorithm.SHA3_256,
           weight: 500.0
       )
   This adds keyIndex=1 as a duplicate of key0 (storedKeyIndex=0).
3. Sign and submit a transaction from A (keyIndex=0) that calls:
       AuthAccount(borrow: A).keys.revoke(keyIndex: 0)
   This sets Revoked=true in apk_0. Key metadata for keyIndex=1 is unchanged.
4. Construct a new transaction with:
       proposalKey: {address: A, keyIndex: 1, sequenceNumber: <current>}
       authorizers: [A]
       envelopeSignatures: [{address: A, keyIndex: 1, sig: <signed with key0 private key>}]
5. Submit to Access node. The FVM calls GetRuntimeAccountPublicKey(A, 1),
   receives {Revoked: false, PublicKey: key0.bytes}, verifies the signature,
   and executes the transaction — despite key0 being revoked at keyIndex=0.
```

func revokeAccountPublicKey0(
	a Accounts,
	address flow.Address,
) error {
	accountPublicKey0RegisterID := flow.AccountPublicKey0RegisterID(address)

	publicKey, err := a.GetValue(accountPublicKey0RegisterID)
	if err != nil {
		return err
	}

	const keyIndex = uint32(0)

	if len(publicKey) == 0 {
		return errors.NewAccountPublicKeyNotFoundError(
			address,
			keyIndex)
	}

	decodedPublicKey, err := flow.DecodeAccountPublicKey(publicKey, keyIndex)
	if err != nil {
		return fmt.Errorf(
			"failed to decode account public key 0: %w",
			err)
	}

	decodedPublicKey.Revoked = true

	encodedPublicKey, err := flow.EncodeAccountPublicKey(decodedPublicKey)
	if err != nil {
		encoded, _ := decodedPublicKey.MarshalJSON()
		return errors.NewValueErrorf(
			string(encoded),
			"failed to encode revoked account public key 0: %w",
			err)
	}

	return a.SetValue(accountPublicKey0RegisterID, encodedPublicKey)

// AppendDuplicateKeyMetadata appends duplicate key metadata.
func (m *KeyMetadataAppender) AppendDuplicateKeyMetadata(
	keyIndex uint32,
	duplicateStoredKeyIndex uint32,
	revoked bool,
	weight uint16,
) (err error) {

	// Append revoked status and weight
	m.weightAndRevokedStatusBytes, err = appendWeightAndRevokedStatus(m.weightAndRevokedStatusBytes, revoked, weight)
	if err != nil {
		return err
	}

	if !m.deduplicated {
		// Set deduplication flag
		m.deduplicated = true

		// Save mapping start key index.
		m.startIndexForMapping = keyIndex
	}

	// Append duplicate stored key index to mapping
	m.mappingBytes, err = appendStoredKeyIndexToMappings(m.mappingBytes, duplicateStoredKeyIndex)
	if err != nil {
		return err
	}

	return nil
}
