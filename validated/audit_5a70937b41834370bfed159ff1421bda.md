### Title
Unprivileged Caller Can Permanently Suppress Per-Block Reward Distribution by Consuming `last_reward_block` Slot with `disable_rewards: true` - (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` in `staking.cairo` updates `last_reward_block` unconditionally before checking the `disable_rewards` flag. Because there is no access-control guard on the function, any unprivileged caller can invoke it with `disable_rewards: true` at the start of any block, consuming the one-update-per-block slot without distributing rewards. The legitimate sequencer call in the same block then reverts with `REWARDS_ALREADY_UPDATED`, and stakers permanently lose that block's yield.

---

### Finding Description

`update_rewards` enforces a single-call-per-block invariant via `last_reward_block`:

```cairo
// src/staking/staking.cairo  lines 1452-1507
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();                          // only checks paused + non-zero caller
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker validation ...

    // ← ALWAYS written, regardless of disable_rewards
    self.last_reward_block.write(current_block_number);   // line 1485

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // line 1488 – exits without distributing
    }
    // ... reward distribution ...
}
```

`general_prerequisites` contains no role check:

```cairo
// src/staking/staking.cairo  lines 1793-1797
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

The spec documents `update_rewards` as "Only starkware sequencer", but this is never enforced in code. Any address can call the function.

**Attack path:**

1. Attacker observes any valid, active staker address (public via `get_stakers`).
2. At the first transaction of each block, attacker calls `update_rewards(valid_staker, disable_rewards: true)`.
3. `last_reward_block` is written to the current block number; no rewards are distributed.
4. The legitimate sequencer call in the same block hits `REWARDS_ALREADY_UPDATED` and reverts.
5. All stakers lose that block's rewards permanently.

The cost per block is a single Starknet transaction fee (fractions of a cent). The attack can be sustained indefinitely.

**Structural analogy to the TraderJoe finding:** In TraderJoe, `_fp.time` (the timestamp) was updated even when `_deltaT < filterPeriod`, so the volatility reference never decayed. Here, `last_reward_block` is updated even when `disable_rewards = true`, so the reward distribution never fires. In both cases a cheap, repeated state-write prevents the intended accumulation/distribution from occurring.

---

### Impact Explanation

Every block in which the attacker front-runs the sequencer results in a permanent, irrecoverable loss of block rewards for all stakers and pool members. The `last_reward_block` guard is a one-shot mechanism per block; once consumed with `disable_rewards: true`, the rewards for that block are gone. Over time this constitutes continuous theft of unclaimed yield from every staker in the protocol.

**Impact category:** High — Theft of unclaimed yield.

---

### Likelihood Explanation

- No privileged role is required; any EOA can call `update_rewards`.
- The attacker only needs to know one valid active staker address, which is publicly readable.
- The cost is one cheap Starknet L2 transaction per block.
- The attack is fully automated and can run indefinitely.
- There is no on-chain mechanism to detect or prevent it.

**Likelihood: High.**

---

### Recommendation

Enforce that only the designated sequencer (or a whitelisted caller) may invoke `update_rewards`, consistent with the specification. Add a role check at the top of the function:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();
    self.roles.only_sequencer();   // enforce the documented access control
    ...
}
```

Alternatively, move the `last_reward_block.write` to after the `disable_rewards` guard so that a no-op call does not consume the per-block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);  // only write when rewards are actually distributed
```

### Proof of Concept

```
Block N begins.

1. Attacker (any address) calls:
       staking.update_rewards(any_valid_staker, disable_rewards: true)

   → general_prerequisites() passes (not paused, caller non-zero)
   → current_block_number (N) > last_reward_block (N-1) → passes
   → staker validation passes (staker is active, non-zero balance)
   → last_reward_block.write(N)          ← slot consumed
   → disable_rewards == true → return    ← no rewards distributed

2. Sequencer calls:
       staking.update_rewards(staker, disable_rewards: false)

   → current_block_number (N) > last_reward_block (N) → FALSE
   → assert fails: REWARDS_ALREADY_UPDATED

All stakers lose block N's rewards permanently.
Repeat every block at negligible cost.

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,

            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();

                current_block_number > self.last_reward_block.read(),

            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake

            self.last_reward_block.write(current_block_number);

                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,

                    :staker_pool_info,
                    :reward_supplier_dispatcher,
