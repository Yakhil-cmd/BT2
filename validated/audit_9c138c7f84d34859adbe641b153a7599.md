### Title
Missing Access Control on `update_rewards` Allows Any Caller to Front-Run Sequencer and Permanently Freeze Staker Yield - (File: `src/staking/staking.cairo`)

### Summary
`IStakingRewardsManager::update_rewards` is callable by any address. It uses a global `last_reward_block` storage variable as a one-time-per-block gate. An attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block, consuming the gate and returning early without distributing rewards. The sequencer's subsequent call with `disable_rewards: false` then reverts with `REWARDS_ALREADY_UPDATED`, permanently denying all stakers their block rewards.

### Finding Description

The spec at `docs/spec.md` line 1645 explicitly states that `update_rewards` access control is **"Only starkware sequencer."** However, the implementation at `src/staking/staking.cairo` lines 1449–1507 contains only `self.general_prerequisites()` (a pause check) and no caller identity check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);   // global gate consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits without distributing
    }
    // ... reward distribution only reached if disable_rewards == false
``` [1](#0-0) 

The `last_reward_block` is a **single global** storage variable (not per-staker). Once it is written to `current_block_number`, every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`, regardless of which staker is passed. [2](#0-1) 

The spec's intended access control is documented but never enforced: [3](#0-2) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield / theft of unclaimed yield.**

An attacker who calls `update_rewards(valid_staker, disable_rewards: true)` at the start of every block:
1. Writes `last_reward_block = current_block` and returns early — no rewards are distributed.
2. The sequencer's call with `disable_rewards: false` reverts with `REWARDS_ALREADY_UPDATED`.
3. All stakers and their delegators receive zero block rewards for that block.
4. Repeated every block, this permanently freezes all unclaimed yield across the entire protocol.

The `disable_rewards: true` path is the critical enabler: it satisfies all preconditions (valid staker, non-zero balance) while deliberately skipping reward distribution, yet still consuming the global gate. [4](#0-3) 

### Likelihood Explanation

**High.** The entry point is fully public — no special role, key, or privilege is required. Any Starknet account can call `update_rewards` with any valid staker address and `disable_rewards: true`. The attacker has no cost beyond gas and gains a griefing outcome (or competitive advantage if they are a staker themselves, since they can selectively freeze competitors' rewards while timing their own). The attack is trivially automatable. [5](#0-4) 

### Recommendation

Enforce the access control that the spec mandates. Add a sequencer-role check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract()`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_sequencer();   // enforce "Only starkware sequencer"
    ...
``` [6](#0-5) 

### Proof of Concept

```
Block N begins.

1. Attacker (any address) calls:
       update_rewards(staker_address = <any valid staker>, disable_rewards = true)

   Inside the call:
   - general_prerequisites() passes (contract not paused).
   - current_block_number (N) > last_reward_block (N-1) → assertion passes.
   - staker_info loaded, is_staker_active → true.
   - staker_total_strk_balance is non-zero → assertion passes.
   - last_reward_block.write(N)   ← global gate consumed.
   - disable_rewards == true → early return, NO rewards distributed.

2. Sequencer calls:
       update_rewards(staker_address = <any staker>, disable_rewards = false)

   Inside the call:
   - current_block_number (N) > last_reward_block (N) → FALSE → PANIC: REWARDS_ALREADY_UPDATED.

Result: Zero rewards distributed to any staker or delegator for block N.
Repeat every block → permanent yield freeze for the entire protocol.
``` [7](#0-6) [8](#0-7)

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self

                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.

            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);
