### Title
Missing Lower-Bound Validation on `threshold` in FROST Signing Allows Corrupted Signature Output — (File: `src/frost/mod.rs`)

### Summary

`assert_sign_inputs` and `presign` in `src/frost/mod.rs` validate that the threshold does not exceed the participant count, but omit the symmetric lower-bound check (`threshold < 2`) that is present in `assert_key_invariants` in `src/dkg.rs`. A malicious coordinator can supply `threshold = 1` (or `0`) to the FROST signing or presigning entry points, causing the protocol to proceed with incorrect Lagrange coefficients and produce a signature that is cryptographically invalid against the established public key.

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces both bounds on the threshold:

```rust
if threshold > participants.len() { … ThresholdTooLarge … }
if threshold < 2 { … ThresholdTooSmall … }
``` [1](#0-0) 

`assert_sign_inputs` in `src/frost/mod.rs` only enforces the upper bound:

```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
``` [2](#0-1) 

The lower-bound guard (`threshold < 2`) is entirely absent. The same omission exists in the `presign` entry point: [3](#0-2) 

`ReconstructionLowerBound` is a plain `usize` newtype with no invariant enforcement at the type level, so any value including `0` or `1` is accepted: [4](#0-3) 

When `threshold = 1` is passed to the FROST signing protocol, Lagrange coefficients are computed for a degree-0 polynomial (single-point interpolation), whereas the key was generated with a degree-`(t-1)` polynomial where `t ≥ 2`. The resulting signature shares are weighted incorrectly, and the aggregated signature does not verify against the established public key. Honest participants complete the protocol without an error, receiving an unusable `SignatureOption`.

### Impact Explanation

**High — Corruption of sign outputs so honest parties accept unusable cryptographic outputs.**

A malicious coordinator drives the FROST signing round with `threshold = 1`. All honest participants contribute valid nonce commitments and signature shares (consuming their one-time presign nonces), but the coordinator aggregates them with wrong Lagrange weights. The returned signature fails verification. Because FROST nonces must never be reused, the consumed presign material is permanently invalidated, and the signing session must be restarted from presign with fresh nonces.

### Likelihood Explanation

The coordinator role is explicitly within the threat model (the library is designed to tolerate a malicious coordinator up to the documented trust boundary). The attack requires only that the coordinator supply an integer `1` instead of the correct threshold — no cryptographic capability is needed. The missing check is a single-line omission that is invisible to callers who rely on the library's input validation.

### Recommendation

Add the same lower-bound guard to `assert_sign_inputs` and to the `presign` function in `src/frost/mod.rs` that already exists in `assert_key_invariants`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Apply the same fix to the OT-based ECDSA `sign` function in `src/ecdsa/ot_based_ecdsa/sign.rs`, which has the identical omission. [5](#0-4) 

### Proof of Concept

```rust

// Malicious coordinator now calls assert_sign_inputs with threshold = 1.

    &participants,
    1usize,   // threshold = 1, below the minimum of 2
    p1,
    p1,       // coordinator

// Returns Ok(…) — no error, lower-bound check is absent.

    let threshold = usize::from(threshold.into());

/// Verifies that the sign inputs are valid
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
