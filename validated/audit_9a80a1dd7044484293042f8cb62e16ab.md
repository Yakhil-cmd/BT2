### Title
Unguarded `disable_rewards` Flag in `update_rewards` Allows Any Caller to Permanently Suppress Reward Distribution - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` accepts a caller-supplied `disable_rewards: bool` flag and writes `last_reward_block` to the current block **before** checking that flag. Because there is no access-control guard on the function, any unprivileged address can call it with `disable_rewards = true`, consuming the per-block reward slot for every staker without distributing any rewards. The legitimate sequencer is then blocked from distributing rewards for that block by the `REWARDS_ALREADY_UPDATED` assertion.

---

### Finding Description

The spec for `update_rewards` states its access control is **"Only starkware sequencer"**, but the implementation contains no such check. `general_prerequisites()` only asserts the contract is unpaused; it does not restrict the caller. [1](#0-0) 

The critical ordering is:

1. `last_reward_block` is unconditionally written to `current_block_number` (line 1485).
2. Only *after* that write does the code branch on `disable_rewards`:

```cairo
self.last_reward_block.write(current_block_number);   // ← slot consumed

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← no rewards distributed
}
``` [2](#0-1) 

Because `last_reward_block` is a **single global value** shared across all stakers, one call with `disable_rewards = true` for any `staker_address` burns the reward slot for the entire block. Any subsequent call in the same block hits:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

The spec documents this as sequencer-only, but the implementation enforces no such restriction. [4](#0-3) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_staker_address, disable_rewards: true)` once per block permanently denies all stakers their block rewards for that block. Repeated across every block, this completely halts reward accrual for the entire protocol. Rewards that are never accrued are never claimable; the loss is permanent for each affected block.

---

### Likelihood Explanation

**High.** The function is publicly callable with no authentication. The attacker needs only a valid (active) `staker_address` to pass the existence check, which is public information from on-chain events. The attack costs only gas and can be automated to run every block.

---

### Recommendation

Add an access-control guard to `update_rewards` that restricts callers to the authorized sequencer address (or the attestation contract for the attestation-based path). For example:

```cairo
fn update_rewards(...) {
    self.assert_caller_is_sequencer(); // new guard
    self.general_prerequisites();
    ...
}
```

Alternatively, move the `last_reward_block.write` to *after* the `disable_rewards` branch so that a suppressed call does not consume the block slot.

---

### Proof of Concept

1. Staker S is active and in the consensus-rewards phase.
2. Attacker A (any address) calls:
   ```
   IStakingRewardsManager(staking).update_rewards(S, disable_rewards: true)
   ```
3. `last_reward_block` is set to the current block; no rewards are distributed.
4. The legitimate sequencer attempts to call `update_rewards(S, disable_rewards: false)` in the same block and receives `REWARDS_ALREADY_UPDATED`.
5. Staker S (and all other stakers, since `last_reward_block` is global) receives zero rewards for that block.
6. Repeating step 2 every block permanently freezes all unclaimed yield. [5](#0-4) [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
