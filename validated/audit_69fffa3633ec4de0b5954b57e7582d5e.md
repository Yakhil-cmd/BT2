### Title
Missing Minimum Validation of `old_threshold` in `assert_reshare_keys_invariants` Allows Reshare with Insufficient Old Participants — (File: src/dkg.rs)

---

### Summary

`assert_reshare_keys_invariants` enforces a minimum of 2 on the **new** threshold but applies **no minimum bound** to `old_threshold`. A malicious coordinator or participant can supply `old_threshold = 0` or `old_threshold = 1`, trivially satisfying the intersection-size guard and causing the reshare to proceed with fewer old participants than the actual reconstruction threshold requires. The resulting Lagrange interpolation is cryptographically incorrect, producing new key shares that do not correspond to the original master public key.

---

### Finding Description

`assert_key_invariants` (called for the new threshold) explicitly rejects values below 2:

```rust
// src/dkg.rs:580-581
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
```

`assert_reshare_keys_invariants` converts `old_threshold` to `usize` and uses it only in one comparison:

```rust
// src/dkg.rs:647, 655
let old_threshold = usize::from(old_threshold.into());
...
if old_participants.intersection(&participants).len() < old_threshold {
    return Err(InitializationError::NotEnoughParticipantsForNewThreshold { ... });
}
```

There is **no `old_threshold < 2` guard**. Two exploitable cases follow directly:

| `old_threshold` supplied | Arithmetic effect | Guard outcome |
|---|---|---|
| `0` | `usize < 0` is always `false` | Guard **never fires**; any intersection (even empty) passes |
| `1` | `intersection.len() >= 1` suffices | Guard passes with a single old participant |

`KeygenOutput` (src/lib.rs:51-55) stores only `private_share` and `public_key`; the threshold is **not persisted** in the keygen output, so callers must supply it externally on every reshare call — making it straightforward to pass a wrong value.

The protocol specification (docs/dkg.md:50) states: *"Each $P_i$ asserts that $1 < \mathsf{threshold} < N$"* for the new threshold, and (docs/dkg.md:54): *"Each $P_i$ asserts that $\mathsf{OldThreshold} \leq |I|$"* for the old threshold. The implementation enforces the first assertion but not the implicit lower bound on `OldThreshold`.

Once the guard is bypassed, `do_reshare` (src/dkg.rs:600-635) computes each old participant's Lagrange-weighted contribution over the intersection set. With fewer participants than the actual reconstruction threshold, the Lagrange interpolation reconstructs a **different** scalar than the original secret, so every new share is a share of a corrupted key. The public-key equality check at step 4.5 of `do_keyshare` (src/dkg.rs:489-496) then catches the mismatch and aborts — but only after the protocol has run, and only for participants who hold the correct old public key. A participant who does not independently verify the old public key will accept the corrupted output.

---

### Impact Explanation

Corruption of reshare outputs: honest parties who accept the reshare receive signing shares that are inconsistent with the original master public key. Any subsequent signing attempt with those shares produces invalid signatures, permanently denying signing capability. This matches the allowed High impact: *"Corruption of DKG, reshare, refresh, presign, sign, or CKD outputs so honest parties accept inconsistent public keys, participant sets, transcripts, or unusable cryptographic outputs."*

---

### Likelihood Explanation

The `reshare` public API (src/lib.rs:106-141) accepts `old_threshold` as a plain caller-supplied integer with no type-level constraint. Any participant or coordinator orchestrating a reshare can trivially pass `old_threshold = 0` or `old_threshold = 1`. Because `KeygenOutput` does not embed the threshold, there is no authoritative on-chain or in-struct record to compare against. The attack requires no cryptographic capability — only the ability to call the library's public API with a crafted parameter.

---

### Recommendation

Add the same lower-bound check for `old_threshold` that already exists for the new threshold:

```rust
// src/dkg.rs — inside assert_reshare_keys_invariants, after line 647
if old_threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: old_threshold,
        min: 2,
    });
}
```

Alternatively, embed the threshold inside `KeygenOutput` so it can be verified against the caller-supplied value at reshare time, eliminating the reliance on the caller to remember and correctly pass the original threshold.

---

### Proof of Concept

```
Setup:
  old_participants = [P1, P2]   (keygen ran with threshold = 2)
  new_participants = [P1, P3]   (P2 is replaced by P3)
  intersection     = [P1]       (only 1 common participant)

Attack:
  Malicious coordinator calls reshare(..., old_threshold = 1, ...)

Execution of assert_reshare_keys_invariants (src/dkg.rs:655):
  intersection.len() = 1
  old_threshold      = 1
  1 < 1  →  false   →  guard does NOT fire

do_reshare proceeds (src/dkg.rs:611-620):
  intersection = [P1]
  P1 computes: lambda_{P1}([P1]) * secret_{P1}
  With only one point, Lagrange gives lambda = 1, so secret = secret_{P1}
  But secret_{P1} ≠ original_secret  (original_secret = f(0) requires ≥ 2 shares)

Result:
  New participants receive shares of secret_{P1}, not the original key.
  The public-key check at do_keyshare step 4.5 (src/dkg.rs:489-496) aborts
  for participants who hold the correct old_public_key, but any participant
  that skips this check accepts corrupted shares — permanently losing the
  ability to produce valid signatures under the original key.

pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],
) -> Result<(ParticipantList, ParticipantList), InitializationError> {
    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

    let participants = assert_key_invariants(participants, me, threshold)?;

    let old_participants =
        ParticipantList::new(old_participants).ok_or(InitializationError::DuplicateParticipants)?;

    // Step 1.1
    if old_participants.intersection(&participants).len() < old_threshold {
        return Err(InitializationError::NotEnoughParticipantsForNewThreshold {
            threshold: old_threshold,
            participants: old_participants.intersection(&participants).len(),
        });
    }
    // Step 1.1
    // if me is not in the old participant set then ensure that old_signing_key is None
    if old_participants.contains(me) && old_signing_key.is_none() {
        return Err(InitializationError::BadParameters(format!(
            "party {me:?} is present in the old participant list but provided no share"
        )));
    }
    Ok((participants, old_participants))
}

pub fn reshare<C: Ciphersuite>(
    old_participants: &[Participant],
    old_threshold: impl Into<ReconstructionLowerBound> + Send + 'static,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    new_participants: &[Participant],
    new_threshold: impl Into<ReconstructionLowerBound> + Copy + Send + 'static,
    me: Participant,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = KeygenOutput<C>>, InitializationError>
where
    Element<C>: Send,
    Scalar<C>: Send,
{
    let comms = Comms::new();
    let threshold = new_threshold;
    let (participants, old_participants) = assert_reshare_keys_invariants::<C>(
        new_participants,
        me,
        threshold,
        old_signing_key,
        old_threshold,
        old_participants,
    )?;
    let fut = do_reshare(
        comms.shared_channel(),
        participants,
        me,
        threshold,
        old_signing_key,
        old_public_key,
        old_participants,
        rng,
    );
    Ok(make_protocol(comms, fut))
}
