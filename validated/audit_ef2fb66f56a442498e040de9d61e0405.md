### Title
Unbounded Per-Staker Pool Count Causes Linear Gas Growth in Core Functions — (`src/staking/staking.cairo`)

### Summary
`set_open_for_delegation` enforces only a one-pool-per-token guard (`has_pool_for_token`), with no cap on total pools per staker. Since `does_token_exist` passes for any **added** token regardless of enabled/disabled status, a staker can create one pool for every token the admin has ever added. Multiple functions then iterate over all pools with no bound, causing O(N) gas consumption where N = number of tokens added.

### Finding Description

**Guard in `set_open_for_delegation`** — the only check preventing duplicate pools is:

```rust
assert!(self.does_token_exist(:token_address), "{}", Error::TOKEN_NOT_EXISTS);
assert!(!staker_pool_info.has_pool_for_token(:token_address), "{}", Error::STAKER_ALREADY_HAS_POOL);
``` [1](#0-0) 

`does_token_exist` returns `true` for any token in `btc_tokens`, regardless of enabled/disabled status:

```rust
fn does_token_exist(self: @ContractState, token_address: ContractAddress) -> bool {
    token_address == STRK_TOKEN_ADDRESS || self.btc_tokens.read(token_address).is_some()
}
``` [2](#0-1) 

This is confirmed by the existing test `test_set_open_for_delegation_with_disabled_btc_token`, which explicitly verifies a pool can be opened for a disabled token. [3](#0-2) 

`add_token` has no cap on the number of tokens that can be added: [4](#0-3) 

**Functions that iterate over all pools without a bound:**

1. **`unstake_intent`** — iterates over all pools to remove delegated stake from total stake: [5](#0-4) 

2. **`staker_migration`** — iterates over all pools, making **two cross-contract calls per pool** (`add_new_implementation` + `replace_to`): [6](#0-5) 

3. **`calculate_staker_pools_rewards`** (called by both `update_rewards` and `update_rewards_from_attestation_contract`) — iterates over all pools: [7](#0-6) 

4. **`update_pool_rewards`** — iterates over the rewards array built from pool iteration, making one cross-contract call per pool: [8](#0-7) 

5. **`get_staker_total_strk_btc_balance_at_epoch`** — iterates over all pools (called inside `update_rewards`): [9](#0-8) 

6. **`_staker_pool_info`** — iterates over all pools (called by `staker_pool_info` and `staker_pool_info_v3`): [10](#0-9) 

### Impact Explanation

The gas cost of `unstake_intent`, `staker_migration`, `update_rewards`, and `staker_pool_info` scales linearly with the staker's pool count. `staker_migration` is the most severe: it makes 2 cross-contract calls per pool (each involving storage writes and class hash replacement), so N pools = 2N external calls in a single transaction. `update_rewards` is called by the sequencer per block per staker; a staker with many pools inflates the sequencer's per-block cost for that staker. There is no explicit per-staker pool count cap anywhere in the contract.

### Likelihood Explanation

The precondition — multiple tokens added by the token admin — is a normal operational scenario the protocol is explicitly designed for (STRK + multiple BTC variants). The staker's action (`set_open_for_delegation` once per token) is fully unprivileged and public. The test suite already exercises stakers with 3 pools (`StakerMigrationMultiplePoolsFlow`). [11](#0-10) 

The practical severity today is limited because the token admin is expected to add only a small number of tokens. However, there is no on-chain enforcement of this assumption, and the missing invariant (per-staker pool count cap) is a concrete design gap.

### Recommendation

Add a `MAX_POOLS_PER_STAKER` constant and enforce it in `set_open_for_delegation`:

```rust
const MAX_POOLS_PER_STAKER: u32 = 10; // or whatever the protocol intends

assert!(
    staker_pool_info.pools.len() < MAX_POOLS_PER_STAKER,
    "{}",
    Error::MAX_POOLS_REACHED,
);
```

This bounds the gas cost of all pool-iterating functions and makes the invariant explicit and enforceable on-chain.

### Proof of Concept

1. Token admin calls `add_token` + `enable_token` for N BTC tokens (normal operational flow).
2. Staker calls `set_commission` once, then `set_open_for_delegation(token_i)` for each of the N tokens. Each call succeeds because `does_token_exist` passes and `has_pool_for_token` is false for each new token.
3. Staker now has N pools. Measure gas for:
   - `unstake_intent` — O(N) storage reads + events
   - `staker_migration` — O(2N) cross-contract calls
   - `update_rewards` (sequencer call) — O(N) balance reads + O(N) cross-contract calls to pools
4. With N large enough (bounded by the number of tokens the admin has added), any of these calls can become prohibitively expensive or exceed Starknet's per-transaction gas limit.

fn test_set_open_for_delegation_with_disabled_btc_token() {
    let mut cfg: StakingInitConfig = Default::default();

    cfg.test_info.staking_contract = staking_contract;
    stake_for_testing_using_dispatcher(:cfg);
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    let staker_address = cfg.test_info.staker_address;
    let commission = cfg.staker_info._deprecated_get_pool_info()._deprecated_commission;
    let disabled_btc_token_address = deploy_mock_erc20_decimals_contract(
        initial_supply: cfg.test_info.initial_supply,
        owner_address: cfg.test_info.owner_address,
        name: BTC_TOKEN_NAME_2(),
        decimals: constants::TEST_BTC_DECIMALS,
    );
    // Only add the token but not enable it.
    cheat_caller_address_once(
        contract_address: staking_contract, caller_address: cfg.test_info.token_admin,
    );
    let staking_token_dispatcher = IStakingTokenManagerDispatcher {
        contract_address: staking_contract,
    };
    staking_token_dispatcher.add_token(token_address: disabled_btc_token_address);
    // Open pool for the token.
    cheat_caller_address(
        contract_address: staking_contract,
        caller_address: staker_address,
        span: CheatSpan::TargetCalls(2),

    let pool_contract = staking_dispatcher
        .set_open_for_delegation(token_address: disabled_btc_token_address);
    // Test.
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info.pool_info.is_none());
    let staker_pool_info = staking_dispatcher.staker_pool_info(:staker_address);

        pool_contract, token_address: disabled_btc_token_address, amount: Zero::zero(),
    };
    assert!(*staker_pool_info.pools[0] == expected_pool_info);
}
