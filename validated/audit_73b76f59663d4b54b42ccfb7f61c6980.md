### Title
Unauthenticated `sender` in `BankContract.Run` / `TransferMethodName` allows arbitrary drain of victim's `evm/<callerContract>` balance — (`x/cronos/keeper/precompiles/bank.go`)

### Summary
The bank precompile's `transfer(sender, recipient, amount)` handler accepts an arbitrary `sender` address from calldata and passes it directly to `bankKeeper.SendCoins` as the `from` account, with no check that `sender` equals `contract.Caller()`. Any EVM contract can therefore transfer tokens out of any victim address, provided the victim holds a balance of the denom `evm/<callerContract>`.

### Finding Description
In `BankContract.Run`, the `TransferMethodName` branch unpacks `sender` from ABI calldata and constructs `from := sdk.AccAddress(sender.Bytes())`. [3](#0-2) 

It then calls: [4](#0-3) 

There is no assertion that `sender == contract.Caller()`. The denom is `evm/<contract.Caller()>`, so the attacker controls both the denom (by choosing which contract calls the precompile) and the `from` address (by passing any victim as `sender`).

The `exec` utility in `utils.go` demonstrates the correct pattern — it extracts the message signer and rejects calls where `caller != signer` — but the `transfer` branch does not use it. [5](#0-4) 

### Impact Explanation
An attacker who deploys contract `A` can call `bankPrecompile.transfer(victim, attacker, amount)` from `A`. This triggers `SendCoins(victim, attacker, amount)` for denom `evm/A`. Any address that holds a balance of `evm/A` (e.g., from a prior `mint` call by `A`, or from receiving such tokens) can be fully drained without their consent. This is an unauthorized transfer of native bank assets — Critical impact under the allowed scope.

### Likelihood Explanation
The precondition (victim holds `evm/<attackerContract>` tokens) is reachable: the attacker mints tokens to the victim via the `mint` branch (which correctly uses `contract.Caller()` as the denom), then immediately calls `transfer` to drain them back. No privileged access is required.

### Recommendation
In the `TransferMethodName` branch, enforce that the decoded `sender` equals `contract.Caller()`:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be the caller")
}
```

This mirrors the invariant already enforced by the `exec` helper in `utils.go`. [5](#0-4) 

### Proof of Concept
1. Attacker deploys contract `A`.
2. `A` calls `bankPrecompile.mint(victim, 1000)` → victim receives 1000 `evm/A`.
3. `A` calls `bankPrecompile.transfer(victim, attacker_eoa, 1000)` → `SendCoins(victim, attacker_eoa, 1000 evm/A)` executes with no auth check.
4. Victim's balance is zero; attacker holds 1000 `evm/A`.

A unit test asserting that `transfer` with `sender != contract.Caller()` returns an error would fail against the current code, confirming the bug. [6](#0-5)
