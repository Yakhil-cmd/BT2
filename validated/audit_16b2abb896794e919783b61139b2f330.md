### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Grief Reward Distribution - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is specified to be callable only by the Starknet sequencer, but the implementation contains **no caller check**. Any unprivileged address can call it with `disable_rewards: true` once per block to consume the global `last_reward_block` slot, permanently preventing the sequencer from distributing rewards for that block to all stakers and delegators.

---

### Finding Description

The protocol specification explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the actual implementation of `update_rewards` in `StakingRewardsManagerImpl` performs no caller identity check whatsoever: [2](#0-1) 

The function only checks:
1. Contract is not paused (`general_prerequisites()`)
2. `current_block_number > self.last_reward_block.read()` — i.e., not already called this block
3. The supplied `staker_address` is a valid, active staker with non-zero balance

After passing these checks, it unconditionally writes `self.last_reward_block.write(current_block_number)` at line 1485, then returns early if `disable_rewards == true`: [3](#0-2) 

`last_reward_block` is a **global** (not per-staker) storage slot. A single call per block — from any address — exhausts the slot for that block.

---

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` once per block. This:

1. Passes all validation (no caller check, staker is valid, block is new).
2. Writes `last_reward_block = current_block_number`.
3. Returns immediately without distributing any rewards.

Any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

Since consensus rewards are distributed per-block, every block the attacker griefs causes **all stakers and all delegators** to permanently lose that block's rewards. The attack is repeatable every block at minimal cost (one transaction per block). This constitutes **permanent freezing of unclaimed yield** for the entire protocol.

**Allowed impact matched**: *Permanent freezing of unclaimed yield or unclaimed royalties* (High).

---

### Likelihood Explanation

- **Entry path**: Fully permissionless. Any EOA or contract can call `update_rewards` on the public `IStakingRewardsManager` interface.
- **Prerequisites**: Only a valid, active staker address with non-zero balance is needed — trivially obtained from on-chain events or `get_stakers`.
- **Cost**: One transaction per block. No capital at risk for the attacker.
- **Detectability**: The attacker's transaction is indistinguishable from a legitimate sequencer call until the sequencer's call fails.

Likelihood is **High**.

---

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, analogous to the existing `CALLER_IS_NOT_STAKING_CONTRACT` pattern used elsewhere in the codebase: [5](#0-4) 

Store the authorized sequencer address in contract storage during initialization and assert `get_caller_address() == sequencer_address` before any state mutation in `update_rewards`. Alternatively, use Starknet's native `get_execution_info` to verify the transaction originates from the sequencer account.

---

### Proof of Concept

```
1. Deploy the staking system with consensus rewards active.
2. Identify any valid staker address S with non-zero balance (from NewStaker events).
3. Each block, before the sequencer's transaction, submit:
       staking.update_rewards(staker_address: S, disable_rewards: true)
4. The call succeeds: last_reward_block is set to the current block, no rewards distributed.
5. The sequencer's update_rewards call for the same block reverts with REWARDS_ALREADY_UPDATED.
6. Repeat every block → all stakers and delegators receive zero rewards indefinitely.
```

The test suite confirms `update_rewards` is callable by any address without a caller check — tests call it directly without impersonating a sequencer: [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
