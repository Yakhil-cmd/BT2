### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Admitting Transactions That Fail at Execution - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check reads `l2_gas_price` (the gas price *of* the last committed block) from the block header, but every transaction submitted to the gateway will be executed in the *next* block, whose gas price is `next_l2_gas_price`. When the network is congested and `next_l2_gas_price > l2_gas_price`, the gateway admits transactions whose `max_price_per_unit` falls between the two values. Those transactions pass both the gateway's threshold check and its own blockifier pre-validation (which also uses the stale price), but are rejected by the blockifier at actual execution time with `MaxGasPriceTooLow`. The mempool therefore accepts transactions that will never be included in a block.
