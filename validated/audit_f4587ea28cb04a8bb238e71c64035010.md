### Title
Missing Caller Authorization in `update_rewards` Allows Any Address to Permanently Suppress Block Rewards ŌĆö (`File: src/staking/staking.cairo`)

---

### Summary

The spec mandates "Only starkware sequencer" access for `update_rewards`, but the implementation enforces **no caller check**. Because `last_reward_block` is a **global** (not per-staker) storage slot, any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block, permanently consuming that block's reward slot for all stakers.

---

### Finding Description

The spec explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The production implementation of `update_rewards` in `StakingRewardsManagerImpl` performs only three checks before writing `last_reward_block`:

1. `self.general_prerequisites()` ŌĆö only checks the pause flag.
2. `current_block_number > self.last_reward_block.read()` ŌĆö a global, not per-staker, gate.
3. Staker existence and active-balance assertions.

There is **no `get_caller_address()` check, no role assertion, and no sequencer identity verification** anywhere in the function body or in `general_prerequisites`. [2](#0-1) 

After passing those checks, the function unconditionally writes the current block number to the global `last_reward_block` storage slot: [3](#0-2) 

When `disable_rewards` is `true`, the function returns immediately after that write ŌĆö no rewards are distributed to any staker for that block.

A grep for any sequencer-identity enforcement (`sequencer`, `only_sequencer`, `assert_sequencer`, `get_sequencer`) across the entire repository returns **zero matches**, confirming the guard is entirely absent.

---

### Impact Explanation

Because `last_reward_block` is global, a single call per block with `disable_rewards: true` targeting **any** valid active staker:

- Permanently marks that block as "already processed."
- Prevents the legitimate sequencer from calling `update_rewards` for the intended staker in the same block (reverts with `REWARDS_ALREADY_UPDATED`).
- Causes that block's consensus rewards to be **irrecoverably lost** for all stakers ŌĆö there is no catch-up or retroactive distribution mechanism.

An attacker who does this every block starves the entire validator set of all consensus-layer block rewards indefinitely. This matches the allowed impact: **Permanent freezing of unclaimed yield (High)**. [4](#0-3) 

---

### Likelihood Explanation

- Requires no privileged role, no leaked key, no bridge compromise.
- Requires only the ability to submit a Starknet transaction ŌĆö any EOA or contract can do this.
- The cost is one transaction per block; the attacker gains nothing financially but inflicts permanent yield loss on all stakers and delegators.
- The attack is deterministic and requires no race condition beyond being first in the block, which is trivially achievable by front-running or simply submitting early.

---

### Recommendation

Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the authorized sequencer address (stored in contract storage or derived from a role). For example:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

This aligns the implementation with the spec's stated access control.

### Proof of Concept

```

  Attacker calls: staking.update_rewards(staker=<any_valid_staker>, disable_rewards=true)
  ŌåÆ general_prerequisites() passes (not paused)
  ŌåÆ current_block_number (N) > last_reward_block (N-1) ŌåÆ passes

  ŌåÆ last_reward_block.write(N)   ŌåÉ global slot consumed

  Sequencer calls: staking.update_rewards(staker=<intended_staker>, disable_rewards=false)
  ŌåÆ current_block_number (N) > last_reward_block (N) ŌåÆ FALSE
  ŌåÆ PANIC: REWARDS_ALREADY_UPDATED

Repeat every block ŌåÆ complete, indefinite yield suppression.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
