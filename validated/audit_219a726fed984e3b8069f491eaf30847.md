### Title
Malicious Staker Can Permanently Freeze Delegators' Principal via Reverting `reward_address` in `unstake_action` - (File: src/staking/staking.cairo)

### Summary
`unstake_action` in `staking.cairo` pushes staker rewards to a user-controlled `reward_address` **before** returning delegated principal to pool contracts. A malicious staker can set `reward_address` to a contract that reverts on ERC20 `transfer`, permanently blocking `unstake_action` and trapping all delegators' principal inside the staking contract with no recovery path.

### Finding Description

`unstake_action` executes the following sequence: [1](#0-0) 

1. `send_rewards_to_staker` — transfers accumulated STRK rewards to `staker_info.reward_address` (fully user-controlled).
2. `checked_transfer` — returns the staker's own principal to `staker_address`.
3. `transfer_to_pools_when_unstake` — loops over all delegation pools and transfers each pool's delegated principal back to the respective pool contract.

`send_rewards_to_staker` performs a direct ERC20 push to `reward_address`: [2](#0-1) 

`reward_address` is set by the staker and can be changed at any time via `change_reward_address`, which has **no restriction** preventing a change after `unstake_intent` has been called: [3](#0-2) 

The only guard is that `reward_address` must not be a registered token address: [4](#0-3) 

There is no check preventing `reward_address` from being an arbitrary contract that reverts on ERC20 receipt.

If `send_rewards_to_staker` reverts, `transfer_to_pools_when_unstake` is never reached: [5](#0-4) 

The delegated funds remain locked inside the staking contract. Delegators' `exit_action` on the pool contract subsequently fails because the pool contract holds no balance (funds were never returned from the staking contract).

### Impact Explanation

Delegators' principal is permanently frozen inside the staking contract. The pool contract's `exit_action` transfers from the pool contract's own balance: [6](#0-5) 

Since `transfer_to_pools_when_unstake` never executes, the pool contract has no balance to return. Delegators who have already called `exit_intent` and waited out the exit window cannot recover their principal. There is no admin or governance function to force-complete an `unstake_action` or redirect the reward transfer.

**Impact**: Permanent freezing of delegators' principal — matches **High: Temporary/Permanent freezing of funds**.

### Likelihood Explanation

The attack requires a staker who is willing to sacrifice their own stake (which is also locked). This could be:
- A staker whose key is compromised by an attacker who wants to grief delegators.
- A staker acting maliciously to harm specific delegators (e.g., a competitor).
- A staker who has already recouped value through other means (e.g., commission rewards already claimed).

The entry path is fully unprivileged: `change_reward_address` is callable by any staker at any time, including after `unstake_intent`. No governance or admin access is required.

### Recommendation

Move `send_rewards_to_staker` to **after** `transfer_to_pools_when_unstake`, or — more robustly — adopt a pull-based pattern: instead of pushing rewards to `reward_address` inside `unstake_action`, accumulate them in storage and let the staker claim separately. This ensures that a reverting `reward_address` cannot block the return of delegated principal to pools.

### Proof of Concept

1. Staker calls `stake(...)` and attracts delegators via a pool.
2. Staker calls `change_reward_address(reverting_contract)` where `reverting_contract` is a Cairo contract whose `on_erc20_received` (or fallback) always panics.
3. Staker calls `unstake_intent()`. Exit window begins.
4. Delegators call `exit_intent(...)` on the pool contract.
5. Exit window elapses.
6. Any caller invokes `unstake_action(staker_address)` → `send_rewards_to_staker` calls `checked_transfer(recipient: reverting_contract, ...)` → the reverting contract panics → entire transaction reverts.
7. Delegators call `exit_action(...)` on the pool contract → pool has zero balance → transfer fails.
8. Delegators' principal is permanently locked in the staking contract with no recovery path. [7](#0-6) [8](#0-7) [5](#0-4)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn transfer_to_pools_when_unstake(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
        ) {
            for (pool_contract, token_address) in staker_pool_info.pools {
                let pool_balance = self.get_delegated_balance(:staker_address, :pool_contract);
                let token_dispatcher = IERC20Dispatcher { contract_address: token_address };
                let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };
                pool_dispatcher.set_staker_removed();
                self
                    .insert_staker_delegated_balance(
                        :staker_address, :pool_contract, delegated_balance: Zero::zero(),
                    );
                let decimals = self.get_token_decimals(:token_address);
                token_dispatcher
                    .checked_transfer(
                        recipient: pool_contract,
                        amount: pool_balance.to_native_amount(:decimals).into(),
                    );
            }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;

                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,

            // Calculate rewards and update entry_to_claim_from.

                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,

            rewards += pool_member_info._unclaimed_rewards_from_v0;
