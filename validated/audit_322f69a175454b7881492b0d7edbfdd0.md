### Title
Unprivileged Caller Can Permanently Suppress Consensus Rewards via Unvalidated `disable_rewards` Parameter in `update_rewards` — (File: src/staking/staking.cairo)

---

### Summary

`IStakingRewardsManager::update_rewards` accepts a caller-controlled `disable_rewards: bool` parameter with no access control. The function unconditionally writes `last_reward_block` to the current block number before checking `disable_rewards`, consuming the single per-block reward slot. Any unprivileged address can call `update_rewards(valid_staker, true)` every block to permanently prevent all stakers from receiving consensus rewards.

---

### Finding Description

`update_rewards` is a public entrypoint gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. No role or identity check restricts who may call it or what value `disable_rewards` may take.

The critical sequence inside the function is:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← global slot consumed

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing
}
```

`last_reward_block` is a **single global** `BlockNumber` field in storage, not a per-staker mapping. The guard at the top of the function enforces that only one call per block can proceed:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

Because `last_reward_block` is written before the `disable_rewards` branch, an attacker who calls `update_rewards(any_valid_active_staker, true)` first in a block:

1. Passes all precondition checks (staker exists, is active, has non-zero STRK balance — all public on-chain).
2. Writes `last_reward_block = current_block`.
3. Returns immediately without distributing rewards.
4. Causes every subsequent `update_rewards` call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

Repeating this every block permanently freezes consensus reward accrual for all stakers.

**Root cause**: The `disable_rewards` parameter is accepted from any caller without validation, and the side-effect of updating the global `last_reward_block` is not gated on whether rewards are actually distributed.

---

### Impact Explanation

- **Permanent freezing of unclaimed yield** for all stakers in the consensus rewards phase (`is_pre_consensus() == false`).
- Stakers' `unclaimed_rewards_own` and pool `cumulative_rewards_trace` are never updated, so no rewards accumulate and none can be claimed.
- The attack is protocol-wide: a single call per block suppresses rewards for every staker simultaneously.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is fully public; any non-zero address can call it.
- The attacker needs only one valid, active staker address with non-zero STRK balance — trivially obtained from on-chain events (`NewStaker`).
- Gas cost is the only barrier; on Starknet L2 this is negligible.
- The attack is sustainable indefinitely and requires no privileged access, no leaked keys, and no external dependencies.

---

### Recommendation

Restrict who may supply `disable_rewards: true`. Two complementary fixes:

1. **Access control**: Require the caller to be the staker themselves (or a designated sequencer/operator role) before accepting `disable_rewards: true`.
2. **Decouple the write**: Only update `last_reward_block` when rewards are actually distributed (move the write after the `disable_rewards` guard), or use a per-staker last-rewarded-block mapping so one call cannot block others.

---

### Proof of Concept

```
// Attacker script — run once per block:
// 1. Read any active staker address from on-chain events.
staker = <any active staker with non-zero STRK balance>

// 2. Call update_rewards with disable_rewards = true.
staking_contract.update_rewards(staker_address: staker, disable_rewards: true)

// Effect:
//   - last_reward_block is set to current block.
//   - No rewards are distributed.
//   - All subsequent update_rewards calls in this block revert with REWARDS_ALREADY_UPDATED.
//   - Stakers' unclaimed_rewards_own never increases.
//   - claim_rewards() always returns 0 for all stakers.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
