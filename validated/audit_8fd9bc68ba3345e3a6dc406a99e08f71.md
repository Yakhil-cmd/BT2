### Title
Missing Access Control on `update_rewards` Allows Any Caller to Deny Block Rewards to Stakers — (`File: src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, consuming the single per-block reward slot and preventing the legitimate sequencer call from ever executing for that block.

---

### Finding Description

The specification for `update_rewards` states:

> **Access control:** Only starkware sequencer.

However, the implementation in `StakingRewardsManagerImpl` performs no such check:

```rust
// src/staking/staking.cairo:1449-1488
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks is_paused
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);  // global, not per-staker

    if disable_rewards || self.is_pre_consensus() {
        return;                            // exits without distributing rewards
    }
```

`last_reward_block` is a **single global storage variable**, not per-staker. Once any call to `update_rewards` succeeds in a given block, every subsequent call in that same block reverts with `REWARDS_ALREADY_UPDATED`. This means a single attacker transaction per block is sufficient to consume the reward slot for all stakers.

The `general_prerequisites()` helper only enforces the pause state; it contains no sequencer identity check. The test suite confirms this: `update_rewards` is invoked throughout without any `cheat_caller_address_once` to impersonate the sequencer.

---

### Impact Explanation

**Impact: Medium — Griefing with no profit motive but damage to users or protocol.**

An attacker who calls `update_rewards(staker_address: <any_active_staker>, disable_rewards: true)` at the start of every block:

1. Writes `last_reward_block = current_block_number`.
2. Returns early without distributing rewards (the `disable_rewards` branch).
3. Causes every subsequent sequencer call in that block to revert with `REWARDS_ALREADY_UPDATED`.

No staker earns block rewards for that block. Sustained over many blocks, this constitutes effective permanent denial of consensus-era yield for all stakers and their delegators. The attacker gains nothing financially; the damage is entirely to stakers and delegators.

---

### Likelihood Explanation

**Likelihood: Medium.**

- The function is publicly callable with no role gate.
- The attacker only needs to submit one transaction per block, which is cheap on Starknet.
- The attack is detectable on-chain but not preventable by stakers themselves.
- A motivated griever (e.g., a competing validator set or a staker who was slashed) has clear incentive.

---

### Recommendation

Add an explicit sequencer-only guard to `update_rewards`. The staking contract already has a roles component; a `SEQUENCER_ROLE` (or equivalent) should be checked at the top of the function:

```rust
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
```

Alternatively, restrict the call to the attestation contract address (which is already trusted and stored in `self.attestation_contract`), consistent with how `update_rewards_from_attestation_contract` is guarded.

---

### Proof of Concept

```
Block N begins.

Attacker (any EOA):
  → calls staking.update_rewards(
        staker_address = <any valid active staker>,
        disable_rewards = true
    )
  ← succeeds; last_reward_block := N; returns without distributing rewards.

Sequencer (legitimate):
  → calls staking.update_rewards(
        staker_address = <scheduled attesting staker>,
        disable_rewards = false
    )
  ← REVERTS with REWARDS_ALREADY_UPDATED.

Result: No staker earns rewards for block N.
Repeat every block → sustained denial of all consensus-era yield.
```
