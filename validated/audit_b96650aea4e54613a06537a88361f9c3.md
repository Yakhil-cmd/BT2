### Title
Gateway Stateful Validator Checks Only L2 Gas Price, Admitting Transactions That Deterministically Fail Blockifier Pre-Validation — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check validates only `l2_gas.max_price_per_unit` against the previous block's L2 gas price. It never validates `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The blockifier's `check_fee_bounds`, however, enforces all three prices against the current block's actual prices. A transaction with `l1_gas.max_price_per_unit = 0` (or below the live L1 gas price) passes every gateway check, enters the mempool, and then deterministically fails at blockifier pre-validation — without consuming the sender's nonce or charging any fee — enabling a free, repeatable DoS against the mempool and batcher.

### Finding Description

**Stateless validator gap** — `validate_resource_bounds` in `StatelessTransactionValidator` checks only `l2_gas.max_price_per_unit` against a static floor (`min_gas_price`, default 8 Gwei). No floor is applied to `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. [1](#0-0) 

**Stateful validator gap** — `validate_tx_l2_gas_price_within_threshold` (called from `validate_resource_bounds`) checks only `l2_gas.max_price_per_unit` against the previous block's L2 gas price. The function carries an explicit TODO acknowledging the missing coverage: [2](#0-1) 

**Blockifier enforces all three prices** — `check_fee_bounds` inside `perform_pre_validation_stage` iterates over L1Gas, L1DataGas, and L2Gas and rejects any resource whose `max_price_per_unit` is below the current block's actual price: [3](#0-2) 

**Nonce is incremented before the price check** — inside `perform_pre_validation_stage`, `handle_nonce` runs first, then `check_fee_bounds`. If `check_fee_bounds` fails the transactional state is rolled back, so the nonce increment is undone and no fee is charged. The transaction is therefore free to resubmit indefinitely with the same nonce: [4](#0-3) 

**Attack construction** — craft a V3 `AllResources` transaction with:
- `l2_gas.max_price_per_unit ≥ min_gas_price` (passes stateless check)
- `l2_gas.max_amount > 0` (passes zero-bounds check)
- `l1_gas.max_price_per_unit = 0` (not checked anywhere in the gateway)

Such a transaction passes `StatelessTransactionValidator::validate`, passes `StatefulTransactionValidator::validate_resource_bounds`, enters the mempool, is picked up by the batcher, and then fails at `check_fee_bounds` with `MaxGasPriceTooLow { resource: L1Gas }`. Because the state is rolled back, the attacker can resubmit the same transaction (same nonce) in the next block for free.

### Impact Explanation

**Allowed scope match**: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

Concrete effects:
1. **Free mempool flooding** — an attacker with a single account can continuously submit the same invalid transaction (nonce never advances on failure), saturating the mempool with entries that will never execute.
2. **Batcher CPU waste** — the batcher dequeues each such transaction, runs the full blockifier pre-validation path (state reads, nonce handling, transactional state creation/rollback), and discards it — all at zero cost to the attacker.
3. **Valid-transaction starvation** — if the mempool enforces a per-account or global size cap, flooding it with zero-cost invalid transactions can crowd out legitimate transactions.

### Likelihood Explanation

- **Unprivileged trigger**: any Starknet account can craft the transaction; no special role or privileged access is required.
- **Zero cost to attacker**: failed pre-validation does not charge a fee and does not advance the nonce, so the same transaction can be resubmitted indefinitely.
- **Trivial construction**: setting `l1_gas.max_price_per_unit = 0` in an otherwise valid V3 invoke transaction is a one-field change.
- **No existing guard**: neither the stateless nor the stateful gateway validator checks L1 or L1DataGas prices; the only defense is the blockifier, which runs after mempool admission.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also validate `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding prices, mirroring the existing L2 gas price check:

### Proof of Concept

```
1. Build a valid Starknet V3 invoke transaction for an account with STRK balance:
     resource_bounds = AllResourceBounds {
         l2_gas: { max_amount: 1_000_000, max_price_per_unit: 8_000_000_000 },  // passes stateless
         l1_gas: { max_amount: 1_000,     max_price_per_unit: 0 },              // NOT checked by gateway
         l1_data_gas: { max_amount: 100,  max_price_per_unit: 0 },              // NOT checked by gateway
     }

2. Submit to the gateway → StatelessTransactionValidator::validate passes
   (max_possible_fee > 0 because l2_gas contribution is nonzero; l2_gas price ≥ min_gas_price).

3. StatefulTransactionValidator::validate_resource_bounds passes
   (only l2_gas.max_price_per_unit is compared against previous block price).

4. Transaction enters the mempool.

5. Batcher dequeues the transaction and calls blockifier execute().

6. perform_pre_validation_stage → check_fee_bounds:
     l1_gas.max_price_per_unit (0) < actual_l1_gas_price (e.g. 10_000_000_000)
     → Err(MaxGasPriceTooLow { resource: L1Gas })

7. TransactionalState is rolled back; nonce unchanged; no fee charged.

8. Attacker resubmits the identical transaction (same nonce) → repeat from step 2.
```

        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();

        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
