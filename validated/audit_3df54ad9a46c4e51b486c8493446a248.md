### Title
Missing Active-Token Validation in `set_open_for_delegation` Allows Pool Creation for Disabled Tokens — (File: src/staking/staking.cairo)

---

### Summary
`set_open_for_delegation` validates that a `token_address` is registered in the system but does not validate that the token is currently **active/enabled**. A staker can therefore open a delegation pool for a disabled BTC token. Delegators who join that pool transfer real funds but earn zero rewards, because the reward-distribution logic silently skips inactive tokens.

---

### Finding Description

In `set_open_for_delegation`, the only token-validity check performed is:

```cairo
assert!(self.does_token_exist(:token_address), "{}", Error::TOKEN_NOT_EXISTS);
``` [1](#0-0) 

`does_token_exist` returns `true` for any token that is registered — whether enabled or disabled:

```cairo
fn does_token_exist(self: @ContractState, token_address: ContractAddress) -> bool {
    token_address == STRK_TOKEN_ADDRESS || self.btc_tokens.read(token_address).is_some()
}
``` [2](#0-1) 

There is no call to `is_active_token` at pool-creation time. The function that would perform the correct check is:

```cairo
fn is_active_token(self: @ContractState, token_address: ContractAddress, epoch_id: Epoch) -> bool {
    token_address == STRK_TOKEN_ADDRESS
        || is_btc_active(active_status: self.btc_tokens.read(token_address).unwrap(), :epoch_id)
}
``` [3](#0-2) 

A BTC token can be in a disabled state — either because it was added via `add_token` but never enabled, or because it was subsequently disabled via `disable_token`. [4](#0-3) 

Once the pool is deployed, `enter_delegation_pool` in `pool.cairo` performs no active-token check either — it only verifies the staker is not removed:

```cairo
self.assert_staker_is_active();
``` [5](#0-4) 

Delegators' funds are transferred to the staking contract and recorded. However, during reward distribution, `calculate_staker_pools_rewards` explicitly skips pools whose token is inactive:

```cairo
if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
    continue;
}
``` [6](#0-5) 

The result is that delegators in a pool backed by a disabled token earn **zero rewards** for every epoch they remain in the pool.

---

### Impact Explanation

Delegators who join a pool for a disabled token lose all yield they would have earned in a legitimate active pool. Their principal is recoverable (they can call `exit_delegation_pool_intent` / `exit_delegation_pool_action`), but the yield for every epoch spent in the pool is permanently lost. This constitutes **griefing with damage to users** — delegators suffer real economic harm (lost yield) with no corresponding gain to the staker.

---

### Likelihood Explanation

A staker can call `set_open_for_delegation` with a disabled BTC token address at any time without any privileged role. The pool contract itself gives delegators no on-chain signal that the underlying token is inactive; they must query the staking contract separately. Unsophisticated delegators are likely to join without performing this check, especially if the staker advertises the pool off-chain.

---

### Recommendation

Add an `is_active_token` check inside `set_open_for_delegation` before deploying the pool contract:

```cairo
assert!(self.does_token_exist(:token_address), "{}", Error::TOKEN_NOT_EXISTS);
assert!(
    self.is_active_token(:token_address, epoch_id: self.get_current_epoch()),
    "{}",
    Error::TOKEN_NOT_ACTIVE,
);
```

This mirrors the guard already present in `get_total_stake_for_token`:

```cairo
assert!(
    self.is_active_token(:token_address, epoch_id: curr_epoch),
    "{}",
    Error::TOKEN_NOT_ACTIVE,
);
``` [7](#0-6) 

---

### Proof of Concept

1. Governance adds a BTC token via `add_token(btc_token)` — token is registered but **not** enabled (`is_active = false`). [8](#0-7) 
2. A staker calls `set_open_for_delegation(token_address: btc_token)`. The call succeeds because `does_token_exist` returns `true` and no active-status check is performed. [9](#0-8) 
3. Delegators call `enter_delegation_pool` on the newly deployed pool. The call succeeds; their BTC is transferred to the staking contract. [10](#0-9) 
4. Each epoch, `calculate_staker_pools_rewards` iterates over the staker's pools and hits `if !self.is_active_token(...) { continue; }` for the disabled BTC pool — **zero rewards are distributed**. [11](#0-10) 
5. Delegators earn no yield for every epoch they remain in the pool. Their only recourse is to exit, but the yield for elapsed epochs is irrecoverably lost.

        fn disable_token(ref self: ContractState, token_address: ContractAddress) {
            self.roles.only_security_agent();

            assert!(is_active, "{}", Error::TOKEN_ALREADY_DISABLED);
            let next_is_active_first_epoch = self.get_epoch_plus_k();
            self.btc_tokens.write(token_address, (next_is_active_first_epoch, false));
            self.emit(TokenManagerEvents::TokenDisabled { token_address });
        }

        fn enter_delegation_pool(
            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.
            self
                .pool_member_info
                .write(pool_member, VInternalPoolMemberInfoTrait::new_latest(:reward_address));

            // Emit events.
            self
                .emit(
                    Events::NewPoolMember { pool_member, staker_address, reward_address, amount },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake: Zero::zero(), new_delegated_stake: amount,
                    },
                );
        }
