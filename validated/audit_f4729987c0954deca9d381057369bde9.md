### Title
Missing Access Control on `update_rewards` Allows Any Caller to Block Consensus Reward Distribution - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the staking contract is documented in the spec as callable only by the Starkware sequencer, but the implementation contains **no caller access control**. Any unprivileged address can call it with `disable_rewards: true`, which updates the global `last_reward_block` without distributing rewards. This permanently blocks the legitimate sequencer call for that block, causing all stakers to lose their consensus rewards for every block the attacker front-runs.

### Finding Description
The spec at `docs/spec.md` line 1644–1645 states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1447–1507 contains no such check:

```cairo
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only checks pause flag
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // Update last block rewards — written BEFORE the disable_rewards check
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;   // returns without distributing any rewards
        }
        ...
    }
}
```

`last_reward_block` is a **global** (non-per-staker) storage variable. Writing it to the current block number before the `disable_rewards` guard means:

1. Attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)`.
2. `last_reward_block` is set to the current block.
3. Function returns early — no rewards distributed.
4. The legitimate sequencer call in the same block hits `REWARDS_ALREADY_UPDATED` and reverts.
5. All stakers lose their consensus block rewards for that block.

Because the attacker can repeat this every block at negligible cost, the entire consensus reward pipeline can be permanently silenced.

### Impact Explanation
All stakers and delegators are denied consensus-phase block rewards. Since `last_reward_block` is global, a single attacker transaction per block is sufficient to suppress rewards for every staker in the protocol. Rewards that should have accrued are never credited to `unclaimed_rewards_own` and never transferred to delegation pools. This constitutes **griefing with no profit motive but material damage to all users and the protocol** (Medium), and at sustained attack cadence approaches permanent freezing of all future unclaimed yield (High).

### Likelihood Explanation
The function is publicly callable with no authentication. The only cost to the attacker is gas per block. There is no economic barrier. Any motivated adversary — a competitor protocol, a disgruntled staker, or a pure griefing actor — can execute this indefinitely.

### Recommendation
Add a sequencer-only access control guard at the top of `update_rewards`, consistent with the spec. For example, using the existing `RolesComponent`:

```cairo
fn update_rewards(...) {
    self.roles.only_sequencer();   // enforce spec-mandated access control
    self.general_prerequisites();
    ...
}
```

Alternatively, if the sequencer role is not yet wired into `RolesComponent`, add a dedicated `sequencer_address` storage slot and assert `get_caller_address() == self.sequencer_address.read()`.

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`).
2. A valid staker `S` exists with non-zero balance.
3. Attacker `A` (any address) calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. `last_reward_block` is written to the current block number; function returns without distributing rewards.
5. Sequencer calls `update_rewards(staker_address: S, disable_rewards: false)` in the same block → panics with `REWARDS_ALREADY_UPDATED`.
6. No staker receives block rewards for this block.
7. Attacker repeats step 3 every block → zero consensus rewards ever distributed.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,

            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),

                Error::REWARDS_ALREADY_UPDATED,

            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
