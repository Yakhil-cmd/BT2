### Title
Gateway Stateful L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful resource-bounds validator computes its admission threshold from the **current block's** `l2_gas_price`, but the blockifier will execute the transaction in the **next block** whose L2 gas price is `next_l2_gas_price` (the EIP-1559 computed value stored in the same block header). The two values diverge by up to `1/gas_price_max_change_denominator` per block. This mismatch causes the gateway to admit V3 transactions that the blockifier will reject (`InsufficientResourceBounds`) and to reject V3 transactions that the blockifier would accept.

---

The block header stores **both** fields:

```rust
pub l2_gas_price: GasPricePerToken,   // price used in this block
pub next_l2_gas_price: GasPrice,      // EIP-1559 price for the NEXT block
``` [4](#0-3) 

When the consensus orchestrator commits a block it writes `self.l2_gas_price` (the EIP-1559 result) as `next_l2_gas_price` and uses it as the `l2_gas_price` of the **following** block:

```rust
l2_gas_price,
l2_gas_consumed: l2_gas_used,
next_l2_gas_price: self.l2_gas_price,
``` [5](#0-4) 

`self.l2_gas_price` is updated by `calculate_next_base_gas_price`, which can move the price by up to `price / gas_price_max_change_denominator` per block:

```rust
let price_change = (price_u256 * gas_delta) / denominator;
let adjusted_price_u256 =
    if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };
``` [6](#0-5) 

The blockifier's `perform_pre_validation_stage` checks `max_price_per_unit >= actual_gas_price` where `actual_gas_price` is the **next block's** `l2_gas_price` (i.e., `next_l2_gas_price` from the previous header):

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { ... }
    );
}
``` [7](#0-6)

### Impact Explanation

**Scenario A — gateway admits, blockifier rejects (mempool pollution / wasted batcher work):**

When the previous block was full, `next_l2_gas_price > l2_gas_price`. Any V3 transaction with `max_price_per_unit` in the half-open interval `[l2_gas_price, next_l2_gas_price)` passes the gateway threshold (`>= 100% × l2_gas_price`) but fails the blockifier check (`< next_l2_gas_price`). The transaction enters the mempool, consumes batcher resources, and is then discarded with `InsufficientResourceBounds`.

**Scenario B — gateway rejects, blockifier would accept (valid transactions silently dropped):**

When the previous block was empty, `next_l2_gas_price < l2_gas_price`. A transaction with `max_price_per_unit` in `[next_l2_gas_price, l2_gas_price)` would be accepted by the blockifier but is rejected at the gateway with `GAS_PRICE_TOO_LOW`. Legitimate users whose wallets price at the EIP-1559 market rate are denied service.

Both outcomes match the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The EIP-1559 mechanism guarantees that `next_l2_gas_price` diverges from `l2_gas_price` whenever a block is not exactly at the gas target. With `gas_price_max_change_denominator` set to 8 (typical), a single full block raises the price by 12.5 %. The gap is therefore present in every non-target-utilisation block, which is the common case. The attacker needs only to observe the public block header (both fields are on-chain) and submit a V3 transaction priced in the gap. No privileged access is required.

---

### Recommendation

Resolve the acknowledged TODO: expose `next_l2_gas_price` through `GatewayFixedBlockStateReader` and use it as the admission anchor instead of `l2_gas_price`. Concretely:

1. Add `next_l2_gas_price: GasPrice` to the return value of `get_block_info` (or add a dedicated `get_next_l2_gas_price` method to the trait).
2. Populate it from `block_header.next_l2_gas_price` in `GatewayFixedBlockSyncStateClient`.
3. Replace `previous_block_l2_gas_price` in `validate_resource_bounds` with the `next_l2_gas_price` value.

### Proof of Concept

```
State:
  Latest committed block header:
    l2_gas_price.price_in_fri     = 100   (price used in block N)
    next_l2_gas_price             = 113   (EIP-1559 result; block N was full)

Gateway config:
  validate_resource_bounds = true
  min_gas_price_percentage = 100

Attacker submits V3 invoke with:
  l2_gas.max_price_per_unit = 100

Gateway check (validate_tx_l2_gas_price_within_threshold):
  threshold = 100% × 100 = 100
  100 >= 100  →  PASS  (transaction admitted to mempool)

Blockifier check (perform_pre_validation_stage, block N+1):
  actual l2_gas_price = next_l2_gas_price = 113
  100 < 113  →  FAIL  InsufficientResourceBounds { MaxGasPriceTooLow }
  Transaction is discarded after consuming batcher resources.
```

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
