### Title
Unprivileged Caller Can Permanently Freeze All Staker Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract has no access control beyond "not paused" and "caller != zero address." Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` once per block to consume the global `last_reward_block` slot without distributing rewards, permanently preventing all stakers from earning consensus block rewards.

---

### Finding Description

`update_rewards` is the entry point through which the consensus mechanism distributes per-block STRK/BTC rewards to stakers. The function is gated only by `general_prerequisites()`:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

No role check, no allowlist, no caller identity verification beyond "not zero."

Inside `update_rewards`, the global `last_reward_block` is written **before** the `disable_rewards` branch is evaluated:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

The guard at the top of the function enforces that only one call per block can succeed:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

`last_reward_block` is a **single global field** shared across all stakers:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [4](#0-3) 

**Attack sequence (repeatable every block):**

1. Attacker picks any currently-active staker `S` with non-zero STRK balance.
2. Attacker calls `update_rewards(S, disable_rewards: true)` at block `N`.
3. `last_reward_block` is set to `N`; the function returns early without distributing any rewards.
4. Any subsequent legitimate call to `update_rewards` for block `N` (by the consensus mechanism or any other caller) reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers — not just `S` — receive zero rewards for block `N`.
6. Repeating this every block permanently prevents the entire protocol from accumulating consensus rewards.

The analog to the external report is direct: just as repeated partial liquidations each award a liquidation incentive that progressively worsens the collateral ratio until the position is permanently stuck, here each block's reward slot is consumed by a no-op call that progressively drains the reward stream until stakers are permanently frozen out of yield.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Every staker in the protocol is affected simultaneously. Because `last_reward_block` is global, a single attacker call per block is sufficient to zero out consensus rewards for the entire protocol indefinitely. Stakers' `unclaimed_rewards_own` fields and pool `cumulative_rewards_trace` entries never increase, making all accrued yield permanently inaccessible.

---

### Likelihood Explanation

- **No privilege required.** Any EOA or contract with a non-zero address can call `update_rewards`.
- **Minimal cost.** One transaction per block; gas is the only expense.
- **Fully deterministic.** The attacker needs only to know any valid, active staker address (publicly observable on-chain).
- **No recovery path.** Once `last_reward_block` is set for a block, that block's rewards are gone; there is no retroactive distribution mechanism.

---

### Recommendation

Restrict `update_rewards` to an authorized caller — for example, the designated consensus/attestation contract or a role held by the block-reward distributor:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_consensus_contract(); // add this guard
    ...
}
```

Alternatively, move the `last_reward_block.write` to **after** the `disable_rewards` branch so that a disabled call does not consume the block slot.

---

### Proof of Concept

```
Block N:
  attacker.call(staking.update_rewards(valid_staker, disable_rewards=true))
  → last_reward_block = N
  → returns early, zero rewards distributed

  consensus_mechanism.call(staking.update_rewards(any_staker, disable_rewards=false))
  → assert!(N > N)  ← FAILS with REWARDS_ALREADY_UPDATED

Block N+1:
  attacker repeats → last_reward_block = N+1
  ...

Result: all stakers earn 0 consensus rewards indefinitely.
```

The root cause is in `src/staking/staking.cairo` at the `update_rewards` entry point [5](#0-4)  where the absence of an access-control guard combined with the global `last_reward_block` write-before-branch pattern creates a reachable, unprivileged griefing path that permanently freezes all staker yield.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
