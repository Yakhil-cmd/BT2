### Title
Missing Zero-Address Validation in `change_reward_address` Permanently Blocks `unstake_action` and `claim_rewards` - (File: src/staking/staking.cairo, src/pool/pool.cairo)

---

### Summary

Both `change_reward_address` in `staking.cairo` and `pool.cairo` (and the initial entry-points `stake` / `enter_delegation_pool`) accept `reward_address = zero` without validation. Once stored, every downstream function that transfers to `reward_address` — `claim_rewards`, `send_rewards_to_staker`, and critically `unstake_action` — will revert unconditionally, because the OpenZeppelin ERC20 `_transfer` implementation rejects the zero address as recipient regardless of amount. For a staker this blocks the only path to recover their principal stake.

---

### Finding Description

**Staking contract (`staking.cairo`)**

`change_reward_address` only asserts the new address is not a registered token:

```cairo
// src/staking/staking.cairo  line 517-540
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    assert!(
        !self.does_token_exist(token_address: reward_address),
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    ...
    staker_info.reward_address = reward_address;   // zero is accepted
    self.write_staker_info(:staker_address, :staker_info);
}
```

The same gap exists in `stake()` (line 307-311), which checks only `REWARD_ADDRESS_IS_TOKEN`.

`send_rewards_to_staker` (line 1614-1629) is called unconditionally — even when `unclaimed_rewards_own == 0` — and always executes:

```cairo
// src/staking/staking.cairo  line 1625
token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
```

`unstake_action` (line 483-515) calls `send_rewards_to_staker` **before** returning the principal stake and before erasing the staker record:

```cairo
// src/staking/staking.cairo  line 494-506
self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
self.write_staker_info(:staker_address, :staker_info);
...
token_dispatcher.checked_transfer(recipient: staker_address, amount: staker_amount.into());
```

If `reward_address` is zero, `send_rewards_to_staker` reverts, the entire `unstake_action` reverts, and the staker's principal is stuck.

**Pool contract (`pool.cairo`)**

`change_reward_address` (line 505-526) only checks the token address:

```cairo
// src/pool/pool.cairo  line 506-510
assert!(
    self.token_dispatcher.contract_address.read() != reward_address,
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
```

`claim_rewards` (line 335-377) then calls:

```cairo
// src/pool/pool.cairo  line 366
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
```

With `reward_address = zero` this reverts, permanently blocking reward withdrawal until the member corrects the address.

---

### Impact Explanation

**Staking contract** — A staker who sets (or initially stakes with) `reward_address = zero` will find `unstake_action` always reverts. Because `send_rewards_to_staker` is called unconditionally before the principal transfer, the staker cannot recover their locked STRK until they call `change_reward_address` with a valid address. This constitutes **temporary freezing of funds** (principal stake).

**Pool contract** — A pool member with `reward_address = zero` cannot call `claim_rewards`; all accrued yield is inaccessible until the address is corrected. This constitutes **temporary freezing of unclaimed yield**.

Both impacts are in the allowed scope.

---

### Likelihood Explanation

Any staker or pool member can trigger this by calling `change_reward_address(0)` or by entering with `reward_address = 0`. No privileged role is required. The call is a single, unprivileged transaction. Accidental misconfiguration (e.g., a front-end bug passing a zero address) is a realistic path. Likelihood is **medium**.

---

### Recommendation

Add a zero-address guard in both `change_reward_address` implementations and in `stake` / `enter_delegation_pool`:

```cairo
// staking.cairo  change_reward_address
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);

// pool.cairo  change_reward_address
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);

// staking.cairo  stake()
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);

// pool.cairo  enter_delegation_pool
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

---

### Proof of Concept

1. Deploy the staking system normally.
2. Staker calls `stake(reward_address: 0, operational_address: X, amount: MIN_STAKE)` — succeeds (no zero-address check).
3. Staker calls `unstake_intent()` — succeeds.
4. Wait for `exit_wait_window`.
5. Anyone calls `unstake_action(staker_address)` — **reverts** because `send_rewards_to_staker` calls `checked_transfer(recipient: 0, amount: ...)` which the ERC20 rejects.
6. Staker's principal STRK is frozen until they call `change_reward_address(valid_address)`.

Alternatively, an already-staked user calls `change_reward_address(0)` (line 517 of `staking.cairo`) — accepted — then attempts `claim_rewards` or `unstake_action` — both revert. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
