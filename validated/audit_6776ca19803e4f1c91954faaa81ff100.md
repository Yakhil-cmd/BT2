### Title
Unprivileged Caller Can Permanently Freeze All Staker Rewards via `disable_rewards` Flag in `update_rewards` Б─■ (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is publicly callable with no role-based access control. It accepts a caller-controlled `disable_rewards` boolean. When `true`, the function still unconditionally writes the current block number to the **global** `last_reward_block` storage variable before returning early Б─■ consuming the protocol-wide one-per-block reward slot without distributing any rewards. An attacker can call this every block to permanently freeze yield for all stakers and pool members.

---

### Finding Description

`update_rewards` performs several security checks before acting: [1](#0-0) 

1. `general_prerequisites()` Б─■ contract not paused, caller non-zero
2. `current_block_number > self.last_reward_block.read()` Б─■ global one-per-block lock
3. Staker exists, is active, has non-zero STRK balance

After all checks pass, at line 1485 the function **unconditionally** writes:

```cairo
self.last_reward_block.write(current_block_number);
```

Then immediately checks:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

`last_reward_block` is a **global** storage variable (not per-staker): [3](#0-2) 

Writing it before the early return means the one-per-block reward slot is consumed even when `disable_rewards: true`. Since there is no access control on `update_rewards` beyond `general_prerequisites()`, any unprivileged caller can:

1. Pick any valid, active staker with non-zero balance
2. Call `update_rewards(staker_address, disable_rewards: true)` once per block
3. `last_reward_block` advances to the current block
4. All subsequent calls to `update_rewards` in the same block revert with `REWARDS_ALREADY_UPDATED`
5. No staker receives block rewards for that block

The `general_prerequisites()` guard only checks pause state and non-zero caller Б─■ no role restriction: [4](#0-3) 

---

### Impact Explanation

By repeating this call every block, an attacker permanently freezes unclaimed yield for all stakers and pool members. This matches the allowed **High** impact: **"Permanent freezing of unclaimed yield or unclaimed royalties."**

Pool members' rewards flow through `update_rewards_from_staking_contract` on the pool, which is only triggered when `_update_rewards` runs Б─■ a path that is entirely skipped when `disable_rewards: true` is passed. [5](#0-4) 

---

### Likelihood Explanation

The function has no role-based access control. Any externally-owned account can call it. The attacker must submit one transaction per block, which is economically feasible for a motivated adversary (e.g., a competing validator or protocol griever). No privileged key, leaked secret, or special role is required Б─■ the entry path is fully unprivileged.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` check, so the global lock is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only consume the per-block slot when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... rest of reward logic
```

Alternatively, add access control restricting who may pass `disable_rewards: true`.

---

### Proof of Concept

1. Attacker (any non-zero EOA) calls `update_rewards(valid_staker, disable_rewards: true)` at block N
2. All checks pass Б─■ staker exists, is active, has non-zero balance
3. `last_reward_block` is written to N (line 1485) Б─■ global slot consumed
4. Function returns early at line 1487 Б─■ zero rewards distributed
5. Any legitimate call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`
6. Attacker repeats at block N+1, N+2, Б─╕
7. No staker or pool member ever accumulates block rewards; all yield is permanently frozen [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
```
