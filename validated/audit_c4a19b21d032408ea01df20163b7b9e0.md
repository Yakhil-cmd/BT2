### Title
Global `last_reward_block` Consumed by Unprivileged `disable_rewards: true` Call, Permanently Freezing All Staker Consensus Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` is a public, access-control-free function. It writes to the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` once per block, consuming the single per-block reward slot for the entire protocol without distributing any rewards. This permanently freezes all stakers' consensus-phase (V3) unclaimed yield at negligible cost to the attacker.

---

### Finding Description

In V3 (consensus rewards), the staking contract distributes per-block rewards through `update_rewards`. The function enforces a global one-call-per-block invariant via `last_reward_block`:

```
// src/staking/staking.cairo:1453-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After validating the staker is active and has non-zero balance, the function unconditionally commits the block number to storage:

```
// src/staking/staking.cairo:1484-1488
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [1](#0-0) 

The write to `last_reward_block` occurs at line 1485, **before** the early-return guard at line 1487. When `disable_rewards: true` is passed, the block slot is consumed and the function returns without computing or crediting any rewards.

`update_rewards` is declared under `#[abi(embed_v0)]` with no caller restriction:

```
// src/staking/staking.cairo:1447-1451
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
``` [2](#0-1) 

The interface confirms any address may call it:

```
// src/staking/interface.cairo:308-310
fn update_rewards(
    ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
);
``` [3](#0-2) 

Because `last_reward_block` is a single global (not per-staker), one call per block locks out every staker in the protocol for that block. [4](#0-3) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Once consensus rewards are active (`is_pre_consensus()` returns `false`), every block in which the attacker calls `update_rewards(..., disable_rewards: true)` is a block for which **no staker in the protocol earns rewards**. By repeating this every block, the attacker halts all consensus-reward accrual indefinitely. Stakers' `unclaimed_rewards_own` balances stop growing; yield is permanently frozen (not merely delayed) for as long as the attack continues. The attacker spends only transaction fees.

---

### Likelihood Explanation

**High.** The entry path requires no privilege, no capital, no leaked key, and no external dependency. Any EOA or contract can call `update_rewards` with a valid staker address and `disable_rewards: true`. Valid staker addresses are publicly enumerable from the `stakers` Vec in storage and from on-chain events. The cost per block is a single Starknet transaction fee. The attack is trivially automatable.

---

### Recommendation

Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` / `is_pre_consensus()` guard, so the block slot is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only consume the slot when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... rest of reward calculation
```

Alternatively, restrict `update_rewards` to be callable only by the staker's registered operational address or by a trusted consensus layer contract, consistent with how `update_rewards_from_attestation_contract` is restricted to the attestation contract. [5](#0-4) 

---

### Proof of Concept

1. Consensus rewards are activated (`start_consensus_rewards()` called, `is_pre_consensus()` = false).
2. At least one staker `S` exists with non-zero effective balance (past the K-epoch delay).
3. Each block, attacker `A` (any EOA) submits:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Execution path:
   - `current_block_number > last_reward_block` → passes (new block).
   - Staker `S` is active with non-zero balance → passes.
   - `last_reward_block.write(current_block_number)` → **slot consumed**.
   - `disable_rewards == true` → early return, zero rewards distributed.
5. Any legitimate call to `update_rewards` for **any** staker in the same block now reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeated every block: all stakers earn zero consensus rewards indefinitely. [6](#0-5)
