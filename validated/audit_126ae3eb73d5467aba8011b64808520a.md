### Title
WalletConnect `chia_sendTransaction` Hides `wallet_id`, Allowing a dApp to Drain CAT Wallets While Displaying a Misleading XCH Amount - (File: `packages/gui/src/electron/commands/Commands.ts`)

### Summary

The `chia_sendTransaction` WalletConnect command schema marks `wallet_id` as `hide: true` in the confirmation dialog and unconditionally humanizes `amount` as `mojo-to-xch` regardless of the actual wallet type. A dApp that has been granted `chia_sendTransaction` permission can supply any `wallet_id` (e.g., a CAT wallet) while the user sees only a misleading XCH-denominated amount and no indication of which wallet is being spent. The user approves what appears to be a negligible XCH transfer but actually authorizes a large CAT spend.

### Finding Description

In `Commands.ts`, the `chia_wallet.send_transaction` command schema defines `wallet_id` with `hide: true` and `amount` with `humanize: 'mojo-to-xch'`: [1](#0-0) 

The `humanizeParams` function in `humanizeParams.ts` filters out any param with `hide: true` before building the confirmation dialog rows: [2](#0-1) 

The `humanizeParamValue` function then formats `amount` unconditionally as XCH regardless of the wallet type: [3](#0-2) 

The `parseCommandDisplay` function, which generates the richer wallet-delta display, only handles `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` — it returns `undefined` for `chia_wallet.send_transaction`, so no asset-type context is shown: [4](#0-3) 

The `dispatchPairRequest` authorization layer checks pair existence, command allowlist, network, and fingerprint — but performs no validation that the supplied `wallet_id` belongs to the expected wallet type or is the standard XCH wallet: [5](#0-4) 

The dApp command defaults `wallet_id` to `1` (XCH), but a dApp can override this with any numeric wallet ID: [6](#0-5) 

### Impact Explanation

A malicious dApp with `chia_sendTransaction` permission calls it with `wallet_id: 2` (a CAT wallet) and `amount: 1_000_000_000` (1,000,000 CAT in mojos, since CAT uses 1000 mojos/token). The confirmation dialog shows "Amount: 0.000001 XCH" (1,000,000,000 ÷ 10¹²) and the destination address — `wallet_id` is invisible. The user approves, believing they are sending a negligible XCH dust amount, but the Chia wallet daemon executes `send_transaction` on the CAT wallet and transfers 1,000,000 CAT to the attacker's address. This is a direct, unauthorized transfer of CAT assets caused by WalletConnect state spoofing that causes the user to display and approve the wrong asset and amount.

### Likelihood Explanation

Any WalletConnect-connected dApp that has been granted `chia_sendTransaction` permission can exploit this. Users routinely grant this permission to dApps they interact with for XCH payments. The dApp does not need any additional privileges — it only needs to supply a non-default `wallet_id` in its RPC call, which is a single integer parameter change. The user has no way to detect the substitution from the confirmation UI.

### Recommendation

1. **Show `wallet_id` in the confirmation dialog** — remove `hide: true` from the `wallet_id` param in `chia_wallet.send_transaction`, or at minimum display the resolved wallet name and asset type.
2. **Humanize `amount` based on the actual wallet type** — look up the wallet type for the given `wallet_id` and apply `mojo-to-cat` (or the appropriate unit) when the target wallet is not the standard XCH wallet.
3. **Extend `parseCommandDisplay`** to handle `chia_wallet.send_transaction` and render a `walletDelta` showing the correct asset type and amount, analogous to how offer commands are displayed.
4. **Optionally restrict `chia_sendTransaction` to wallet_id: 1** — if the intent is that this command only sends XCH, enforce that `wallet_id` must equal `1` (the standard wallet) and reject other values.

### Proof of Concept

1. User connects a dApp via WalletConnect and grants `chia_sendTransaction` permission, expecting it to be used for XCH payments.
2. The dApp calls `chia_sendTransaction` with:
   ```json
   {
     "walletId": 2,
     "amount": "1000000000",
     "fee": "0",
     "address": "xch1attacker..."
   }
   ```
3. `parseDappParams` accepts `wallet_id: 2` (it is in the allowed param list).
4. `humanizeDappCommand` builds the confirmation rows: `wallet_id` is filtered out (`hide: true`); `amount` is formatted as `0.000001 XCH`.
5. `parseCommandDisplay` returns `undefined` for `send_transaction` — no wallet delta is shown.
6. The user sees: **"Amount: 0.000001 XCH"**, **"Address: xch1attacker..."** — no wallet type, no CAT label.
7. User clicks **Send**.
8. `sendCommand('send_transaction', 'chia_wallet', { wallet_id: 2, amount: 1000000000, ... })` executes, transferring 1,000,000 CAT from the user's CAT wallet to the attacker.

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }

  return undefined;
}
