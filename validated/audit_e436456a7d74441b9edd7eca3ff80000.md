### Title
Unprivileged Caller Can Permanently Suppress Per-Block Reward Distribution via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `IStakingRewardsManager::update_rewards` function has no caller access control and accepts a `disable_rewards: bool` parameter. The global `last_reward_block` storage slot is written **before** the `disable_rewards` guard is evaluated. Any unprivileged address can call `update_rewards(valid_staker, disable_rewards: true)` once per block to consume the per-block reward slot without distributing any rewards, permanently preventing legitimate callers from distributing rewards for that block.

### Finding Description
`update_rewards` is exposed as a public, permissionless entry point:

```cairo
pub trait IStakingRewardsManager<TContractState> {
    fn update_rewards(
        ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
    );
}
``` [1](#0-0) 

Inside the implementation, the contract writes `last_reward_block` to the current block number **before** branching on `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

The guard that prevents double-calling is checked at the top of the function:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

Because `last_reward_block` is a **global** (single-slot) storage variable — not keyed per staker — writing it with `disable_rewards: true` exhausts the one allowed call for that block for **all** stakers. There is no retroactive mechanism: rewards for a missed block are permanently unrecoverable.

The consensus-rewards path (`update_rewards`, V3) is the only mechanism for distributing per-block rewards once `consensus_rewards_first_epoch` is active: [4](#0-3) 

### Impact Explanation
An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` once per block permanently destroys the unclaimed yield that would have accrued to every staker and every delegation pool for that block. Sustained over many blocks, this constitutes **permanent freezing of unclaimed yield** for all protocol participants — a High-severity impact under the allowed scope.

### Likelihood Explanation
- The function is fully permissionless; no role, key, or privileged access is required.
- The attacker only needs to supply a valid (active) staker address, which is publicly observable on-chain.
- On Starknet, per-transaction gas costs are low, making continuous block-by-block griefing economically viable.
- The attacker gains nothing financially, but the damage to stakers and delegators is real and irreversible per block.

### Recommendation
Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that consuming the per-block slot only occurs when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only update last_reward_block when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... reward calculation and distribution ...
```

Alternatively, restrict `disable_rewards: true` calls to the staker address itself or a designated operator role.

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. At block N, attacker calls `staking.update_rewards(alice_staker, disable_rewards: true)`.
3. `last_reward_block` is written to N; the function returns immediately with no rewards distributed.
4. Alice (or anyone else) attempts `staking.update_rewards(alice_staker, disable_rewards: false)` at block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. All per-block rewards for block N are permanently lost for every staker and pool.
6. Attacker repeats at block N+1, N+2, … at negligible gas cost, indefinitely suppressing all consensus-era yield.
