### Title
Single Malicious Participant Can Permanently Abort Presign for All Honest Parties via Malformed MessagePack Payload — (`src/ecdsa/ot_based_ecdsa/presign.rs`)

### Summary

A single malicious participant can craft a message with a valid protocol header but an invalid MessagePack payload at `wait0`. When `recv_from_others` calls `chan.recv`, the `rmp_serde` deserialization failure propagates immediately via `?` through the entire call stack, permanently aborting the presign for all honest parties. There is no per-sender error isolation, no retry, and no skip-and-continue logic anywhere in the path.

### Finding Description

The call chain is fully traceable:

**Step 1 — Message ingestion (`push_message`):** The only validation performed on an incoming message is that it is at least `MessageHeader::LEN` (40) bytes long and that the first 40 bytes parse as a valid `MessageHeader`. The payload bytes after the header are accepted unconditionally. [1](#0-0) 

**Step 2 — Deserialization in `Comms::recv`:** After popping the raw bytes, `rmp_serde::decode::from_slice` is called on the payload. If it fails, the function returns `Err(decoded?)` — a `ProtocolError::DeserializationError` — with no attribution to the sender and no opportunity to skip the message. [2](#0-1) 

**Step 3 — No error isolation in `recv_from_others`:** The helper calls `chan.recv(waitpoint).await?` inside a `while !seen.full()` loop. The `?` operator means the very first deserialization failure from any participant immediately exits the function with an error. There is no `match`, no `continue`, and no per-sender fault tracking. [3](#0-2) 

**Step 4 — Error propagates out of `do_presign`:** The `?` on `recv_from_others::<Scalar>(...).await?` at Round 1 of `do_presign` propagates the error to the caller, terminating the presign permanently. [4](#0-3) 

**Crafting the attack message:** The channel tag is a deterministic SHA-256 hash of a fixed domain string (`root_shared`), and `wait0` is always `0` (the first call to `next_waitpoint()`). Both values are fully predictable by any protocol participant. The attacker constructs a 40-byte valid header followed by any bytes that fail `rmp_serde` deserialization as a `Scalar` (e.g., a single `0xFF` byte, an empty payload, or a valid msgpack encoding of a different type). [5](#0-4) [6](#0-5) 

### Impact Explanation

Every honest participant running `do_presign` will receive `Err(ProtocolError::DeserializationError)` and terminate. The presign output is never produced. Because presign outputs are consumed on use and cannot be retried without fresh triple material, this constitutes a **permanent denial of signing** for the affected session. The same attack applies to every subsequent presign attempt as long as the malicious participant remains in the participant set.

### Likelihood Explanation

The attack requires only that the malicious participant be a registered member of the participant list — the normal precondition for running the protocol. No cryptographic material needs to be leaked. The header values needed to craft the message are fully deterministic and publicly derivable from the protocol specification. The attack is trivially repeatable.

### Recommendation

`recv_from_others` (and `Comms::recv`) must not treat a per-sender deserialization failure as a fatal protocol error. The fix should:

1. Return a typed error that identifies the offending sender (e.g., `ProtocolError::MaliciousParticipant(from)`) rather than an anonymous `DeserializationError`, so callers can exclude or report the sender.
2. Alternatively, have `recv_from_others` skip messages that fail deserialization and record the offending participant, continuing to collect from the remaining honest participants.

The `robust_ecdsa` presign and all other protocols using `recv_from_others` share the same structural issue and should be audited for the same pattern. [7](#0-6) 

### Proof of Concept

```rust
// Pseudocode — inject at the network layer before honest parties receive wait0 messages
let channel_tag = sha256(NEAR_CHANNEL_TAGS_DOMAIN || "root shared"); // deterministic
let waitpoint: u64 = 0;                                               // first next_waitpoint()

let mut malicious_msg = Vec::new();
malicious_msg.extend_from_slice(&channel_tag);          // 32 bytes
malicious_msg.extend_from_slice(&waitpoint.to_le_bytes()); // 8 bytes
malicious_msg.push(0xFF);                               // invalid msgpack Scalar payload

// Deliver this message from the malicious participant to all honest participants.
// Each honest participant's recv_from_others::<Scalar> will call rmp_serde::decode::from_slice
// on the 0xFF byte, fail, and return ProtocolError::DeserializationError,
// aborting do_presign for every honest party.
```

Assert: all honest `presign(...)` calls return `Err(ProtocolError::DeserializationError)`.
