Looking at the Starknet Staking codebase for an analog to M-03's pattern: **an external module failure blocking a critical withdrawal/unwind operation**.

The vulnerability class is a **token-freeze bug**: a critical user exit path (`unstake_action`) unconditionally calls an external contract (`reward_supplier`) whose failure reverts the entire operation, permanently blocking the staker from recovering their principal.

---

### Title
`unstake_action` Permanently Blocked When `reward_supplier.claim_rewards` Reverts, Freezing Staker Principal — (`src/staking/staking.cairo`)

### Summary

`unstake_action` unconditionally calls `send_rewards_to_staker` before returning the staker's principal. `send_rewards_to_staker` calls `claim_from_reward_supplier`, which calls `reward_supplier.claim_rewards(amount)`. If the reward supplier's token balance is insufficient (e.g., due to L1 bridge delay) or its `unclaimed_rewards` accounting is inconsistent, the entire `unstake_action` reverts. The staker — who has already waited the full `exit_wait_window` — cannot recover their principal.

### Finding Description

In `unstake_action`, the reward-claiming step is tightly coupled with the principal-return step:

```cairo
fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    self.general_prerequisites();
    // ...
    // Send rewards to staker's reward address.
    // It must be part of this function's flow because staker_info is about to be erased.
    let token_dispatcher = strk_token_dispatcher();
    self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
    // ...
    // Return stake to staker.
    token_dispatcher.checked_transfer(recipient: staker_address, amount: staker_amount.into());
``` [1](#0-0) 

`send_rewards_to_staker` calls `claim_from_reward_supplier` and then transfers tokens to the reward address:

```cairo
fn send_rewards_to_staker(...) {
    let amount = staker_info.unclaimed_rewards_own;
    let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
    claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
    staker_info.unclaimed_rewards_own = Zero::zero();
``` [2](#0-1) 

`claim_from_reward_supplier` calls `reward_supplier.claim_rewards(amount)`, which:
1. Asserts `amount <= unclaimed_rewards` — reverts if accounting is off.
2. Calls `token_dispatcher.checked_transfer(recipient: staking_contract, amount)` — reverts if the reward supplier's actual STRK token balance is insufficient.

```cairo
fn claim_rewards(ref self: ContractState, amount: Amount) {
    // ...
    let unclaimed_rewards = self.unclaimed_rewards.read();
    assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);
    self.unclaimed_rewards.write(unclaimed_rewards - amount);
    let token_dispatcher = self.token_dispatcher.read();
    token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
}
``` [3](#0-2) 

The reward supplier proactively requests tokens from L1 via `send_mint_request_to_l1_reward_supplier` when `unclaimed_rewards` grows, but there is an inherent L1→L2 bridge delay. During this window, `unclaimed_rewards` can exceed the actual STRK token balance held by the reward supplier contract:

```cairo
fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
    let balance: Amount = token_dispatcher.balance_of(account: get_contract_address())...;
    let credit = balance + l1_pending_requested_amount;
    let debit = unclaimed_rewards;
    if credit < debit + threshold {
        // send L1 mint request — tokens arrive later
    }
}
``` [4](#0-3) 

If a staker calls `unstake_action` during this window, `checked_transfer` inside `claim_rewards` reverts, which propagates up through `send_rewards_to_staker` and causes `unstake_action` to revert entirely. The staker's principal remains locked in the staking contract with no alternative exit path.

### Impact Explanation

**Temporary (potentially extended) freezing of staker principal funds.** A staker who has completed `unstake_intent` and waited the full `exit_wait_window` cannot execute `unstake_action` to recover their principal. The principal remains locked until the reward supplier's token balance is replenished by the L1 bridge. In a scenario where the L1 bridge is congested or the L1 `RewardSupplier`/`MintManager` is paused for an emergency, this freeze becomes indefinite. This matches the allowed impact: *Temporary freezing of funds*.

### Likelihood Explanation

The reward supplier's `unclaimed_rewards` is updated eagerly (on every `_update_rewards` call), but the actual STRK token balance only increases when the L1 bridge delivers tokens. The L1→L2 bridge introduces a non-trivial delay. Any staker calling `unstake_action` while the reward supplier's balance is below their `unclaimed_rewards_own` will be blocked. This is a realistic, non-adversarial condition that can arise during normal protocol operation, and is especially likely during high-reward periods or L1 congestion.

### Recommendation

Decouple reward claiming from principal withdrawal in `unstake_action`. If `claim_from_reward_supplier` fails, preserve the unclaimed rewards in `staker_info` (or a separate claimable mapping) and allow the principal transfer to proceed. This mirrors the M-03 fix: wrap the external call so that a failure does not block the critical exit path.

```cairo
// In send_rewards_to_staker (or directly in unstake_action):
// Attempt to claim rewards; if it fails, preserve unclaimed_rewards_own for later.
let claimed = try_claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
if claimed {
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
    staker_info.unclaimed_rewards_own = Zero::zero();
}
// Principal transfer proceeds regardless.
```

### Proof of Concept

1. Staker calls `unstake_intent()` — sets `unstake_time`.
2. Staker waits `exit_wait_window`.
3. Meanwhile, `_update_rewards` is called, increasing `staker_info.unclaimed_rewards_own` and calling `update_unclaimed_rewards_from_staking_contract` on the reward supplier, which increases `unclaimed_rewards` and sends an L1 mint request.
4. The L1 bridge has not yet delivered the tokens; reward supplier's STRK balance < `unclaimed_rewards`.
5. Staker calls `unstake_action()`.
6. `send_rewards_to_staker` → `claim_from_reward_supplier` → `reward_supplier.claim_rewards(amount)` → `checked_transfer` reverts (insufficient balance).
7. `unstake_action` reverts. Staker's principal remains locked. No alternative exit path exists. [5](#0-4) [3](#0-2)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }
