### Title
Gateway Stateful Validator Skips L1 Gas and L1 Data Gas Price Threshold Checks, Admitting Transactions That Fail Blockifier Pre-Validation — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful validator enforces a minimum gas price threshold only for the L2 gas component of `AllResources` transactions. L1 gas and L1 data gas prices are never validated at the gateway level. A transaction with a valid L2 gas price but zero (or sub-market) L1 gas price passes all gateway admission checks and enters the mempool, but is then rejected by the blockifier's `check_fee_bounds` during `perform_pre_validation_stage` with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`. No fee is charged for such failures, enabling free mempool spam.

---

### Finding Description

`validate_tx_l2_gas_price_within_threshold` in `stateful_transaction_validator.rs` matches on `ValidResourceBounds` and only validates the L2 gas price:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(...) {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // threshold computed from previous block l2_gas_price only
            if tx_l2_gas_price.0 < threshold { return Err(...); }
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
    Ok(())
}
``` [1](#0-0) 

The stateless validator similarly only checks `l2_gas.max_price_per_unit` against the static `min_gas_price` floor:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
``` [2](#0-1) 

Neither `l1_gas.max_price_per_unit` nor `l1_data_gas.max_price_per_unit` is checked at any gateway validation stage.

In contrast, the blockifier's `check_fee_bounds` (called from `perform_pre_validation_stage`) validates all three gas prices against the actual block prices:

```rust
ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_gas_resource_bounds,
    l2_gas: l2_gas_resource_bounds,
    l1_data_gas: l1_data_gas_resource_bounds,
}) => {
    // checks l1_gas_price, l1_data_gas_price, l2_gas_price
    if resource_bounds.max_price_per_unit < actual_gas_price.get() {
        insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
    }
}
``` [3](#0-2) 

The `perform_pre_validation_stage` is called during blockifier execution: [4](#0-3) 

The production deployment config confirms `min_gas_price_percentage: 100` (full L2 gas price enforcement) but has no corresponding L1 gas price enforcement: [5](#0-4) 

The `StatefulTransactionValidatorConfig` default also confirms `validate_resource_bounds: true` and `min_gas_price_percentage: 100`: [6](#0-5) 

---

### Impact Explanation

**High. Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

An attacker submits V3 invoke/declare/deploy-account transactions with:
- `l2_gas.max_price_per_unit ≥ min_gas_price` (passes stateless check)
- `l2_gas.max_price_per_unit ≥ threshold` (passes stateful check)
- `l1_gas.max_price_per_unit = 0` (not checked at gateway)
- `l1_data_gas.max_price_per_unit = 0` (not checked at gateway)

These transactions pass all gateway validation and enter the mempool. When the batcher attempts execution, `check_fee_bounds` rejects them with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }` — a hard pre-validation failure that charges no fee. The batcher discards the transaction, and the attacker can repeat indefinitely at zero cost beyond network bandwidth.

---

### Likelihood Explanation

High. Any unprivileged user can craft such a transaction. The attack requires no special account state, no privileged access, and no knowledge beyond the public minimum L2 gas price. The `max_possible_fee` check in the stateless validator passes as long as any single resource bound is non-zero, so a transaction with only L2 gas set non-zero satisfies it. [7](#0-6) 

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` to also validate `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against their respective previous-block prices (analogous to the existing L2 check). The TODO comment in the code already acknowledges this gap:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
``` [8](#0-7) 

The fix should mirror the blockifier's `check_fee_bounds` logic, reading `l1_gas_price` and `l1_data_gas_price` from `block_info.gas_prices.strk_gas_prices` and applying the same `min_gas_price_percentage` multiplier.

---

### Proof of Concept

1. Construct a V3 invoke transaction with:
   - `resource_bounds.l2_gas.max_price_per_unit = 8_000_000_000` (≥ `min_gas_price`)
   - `resource_bounds.l1_gas.max_price_per_unit = 0`
   - `resource_bounds.l1_data_gas.max_price_per_unit = 0`
   - `resource_bounds.l2_gas.max_amount > 0` (so `max_possible_fee > 0`)

2. Submit to the gateway. `StatelessTransactionValidator::validate_resource_bounds` passes: `max_possible_fee > 0` and `l2_gas.max_price_per_unit ≥ min_gas_price`.

3. `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` passes: only `l2_gas.max_price_per_unit` is compared to the threshold; L1 prices are not checked.

4. Transaction is admitted to the mempool.

5. Batcher calls `AccountTransaction::perform_pre_validation_stage` → `check_fee_bounds`. With actual `l1_gas_price` from the block (e.g., `1_000_000_000`), the check `0 < 1_000_000_000` fails, producing `InsufficientResourceBounds { errors: [MaxGasPriceTooLow { resource: L1Gas, ... }] }`.

6. Transaction is discarded with no fee charged. Repeat from step 1.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
