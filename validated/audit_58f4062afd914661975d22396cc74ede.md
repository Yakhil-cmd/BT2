### Title
Staker Can Register With Unowned `operational_address` in `stake()`, Bypassing Ownership Verification — (File: `src/staking/staking.cairo`)

---

### Summary
`stake()` accepts an arbitrary `operational_address` without verifying the caller controls that address. Unlike `change_operational_address()`, which enforces a two-step ownership proof via `declare_operational_address()`, the initial registration path has no such guard. An attacker who learns a victim's intended operational address can register it first, causing the victim's `stake()` to revert with `OPERATIONAL_EXISTS`.

---

### Finding Description
In `stake()`, the `operational_address` parameter is stored after only a uniqueness check — there is no proof that the caller controls the address: [1](#0-0) 

The only guard is that the address is not already in use. No prior declaration from the operational address itself is required.

By contrast, `change_operational_address()` enforces a two-step ownership proof: [2](#0-1) 

This check requires the operational address to have previously called `declare_operational_address()` — proving it controls the address — before the staker can adopt it. [3](#0-2) 

The `declare_operational_address()` function is the ownership proof mechanism: the operational address itself must be the caller, binding it to a specific staker. This mechanism is entirely absent from the initial `stake()` path. [4](#0-3) 

The `operational_address` is critical because it is the identity used for attestation: `attest()` in the Attestation contract resolves the staker by `get_caller_address()` as the operational address. [5](#0-4) 

---

### Impact Explanation
An attacker who learns a victim's intended `operational_address` can call `stake()` with that address first. The victim's `stake()` then reverts with `OPERATIONAL_EXISTS`. The attacker's staker record is useless for attestation (they do not hold the private key for the operational address), so no rewards are earned by the attacker — the sole purpose is denial. The victim is forced to generate a new operational key pair and retry staking.

This matches **Medium impact**: griefing with no profit motive but concrete damage to users (failed transactions, forced key rotation, operational disruption).

---

### Likelihood Explanation
Starknet does not have a public mempool in the same way as Ethereum, making classic front-running harder. However, the attack is reachable when:
1. The victim announces their operational address publicly before staking (e.g., in a validator setup guide or forum post).
2. The attacker monitors the sequencer's pending transaction queue if it is exposed.

The attacker must also meet the minimum stake requirement (`min_stake`), which is a cost but not a prohibitive barrier for a well-resourced griever. [6](#0-5) 

---

### Recommendation
Apply the same two-step ownership verification to `stake()` as is enforced in `change_operational_address()`. Before accepting an `operational_address` in `stake()`, require that the address has previously called `declare_operational_address(staker_address)` to prove it controls the address:

```cairo
assert!(
    self.eligible_operational_addresses.read(operational_address) == staker_address,
    "{}",
    Error::OPERATIONAL_NOT_ELIGIBLE,
);
```

This ensures the operational address has actively consented to being associated with the staker, mirroring the existing protection on `change_operational_address()`.

---

### Proof of Concept
1. Victim prepares to call `stake(reward_address=V_reward, operational_address=X, amount=min_stake)`.
2. Attacker observes the intended `operational_address=X` (e.g., from a public announcement or sequencer queue).
3. Attacker calls `stake(reward_address=A_reward, operational_address=X, amount=min_stake)` first; this succeeds because `X` is not yet in use.
4. Victim's `stake()` reverts: `assert!(self.operational_address_to_staker_address.read(operational_address).is_zero(), ...)` fails with `OPERATIONAL_EXISTS`.
5. Victim must generate a new operational key pair and retry, while the attacker holds a staker record with `operational_address=X` that cannot be used for attestation (attacker lacks the private key for `X`). [1](#0-0) [7](#0-6)

        fn declare_operational_address(ref self: ContractState, staker_address: ContractAddress) {
            self.general_prerequisites();
            let operational_address = get_caller_address();
            assert!(
                self.operational_address_to_staker_address.read(operational_address).is_zero(),
                "{}",
                Error::OPERATIONAL_IN_USE,
            );
            assert!(
                !self.does_token_exist(token_address: operational_address),
                "{}",
                Error::OPERATIONAL_IS_TOKEN,
            );
            if self.eligible_operational_addresses.read(operational_address) == staker_address {
                return;
            }
            self.eligible_operational_addresses.write(operational_address, staker_address);
            self.emit(Events::OperationalAddressDeclared { operational_address, staker_address });
