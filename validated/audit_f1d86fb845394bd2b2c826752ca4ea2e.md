### Title
Missing Access Control on `update_rewards` Allows Any Caller to Suppress Block Rewards for All Stakers — (File: `src/staking/staking.cairo`)

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable with no access control and accepts a caller-controlled `disable_rewards: bool` parameter. An unprivileged attacker can call it with `disable_rewards: true` once per block, consuming the global `last_reward_block` slot and preventing any staker from receiving consensus block rewards for that block.

---

### Finding Description

`IStakingRewardsManager::update_rewards` is exposed as a public ABI function: [1](#0-0) 

Its implementation in `StakingRewardsManagerImpl`: [2](#0-1) 

The critical sequence is:

1. The function checks that `current_block_number > self.last_reward_block.read()` — this is a **global, single-slot** guard shared across all stakers.
2. It unconditionally writes `self.last_reward_block.write(current_block_number)` **before** checking `disable_rewards`.
3. If `disable_rewards == true`, it returns immediately without distributing any rewards. [3](#0-2) 

There is no `assert_caller_is_attestation_contract()` or any other role check — unlike `update_rewards_from_attestation_contract`, which correctly enforces: [4](#0-3) 

Because `last_reward_block` is global, once an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at block N, every subsequent call to `update_rewards` at block N for any staker will revert with `REWARDS_ALREADY_UPDATED`: [5](#0-4) 

---

### Impact Explanation

An attacker calling `update_rewards(staker_address, true)` once per block permanently prevents all stakers from earning consensus block rewards for those blocks. The rewards are never credited to `staker_info.unclaimed_rewards_own` and never forwarded to delegation pools. This constitutes **griefing with no profit motive but direct, ongoing damage to all stakers and pool members** in the protocol — matching the Medium impact tier: *"Griefing with no profit motive but damage to users or protocol."*

---

### Likelihood Explanation

The entry path requires no privilege, no stake, and no special setup. Any EOA can call `update_rewards` with an arbitrary valid `staker_address` and `disable_rewards: true`. The cost is one transaction per block. The attack is trivially automatable and economically viable for a motivated griever.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a dedicated rewards-manager role), or remove the `disable_rewards` parameter from the public interface entirely and handle the disable logic internally based on protocol state. For example:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.roles.only_rewards_manager(); // add appropriate role check
    ...
}
```

Alternatively, if `disable_rewards` is only ever meant to be `true` during the pre-consensus phase, replace the parameter with an internal check against `self.is_pre_consensus()` and remove the external parameter.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` is set and current epoch ≥ that value).
2. Attacker (any address) calls `update_rewards(valid_staker_address, disable_rewards: true)` at block N.
3. `last_reward_block` is written to N; function returns early — no rewards distributed.
4. Legitimate staker or protocol bot calls `update_rewards(staker_B, false)` at block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Stakers earn zero block rewards for block N.
6. Attacker repeats at every block → all stakers earn zero consensus rewards indefinitely. [2](#0-1)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
