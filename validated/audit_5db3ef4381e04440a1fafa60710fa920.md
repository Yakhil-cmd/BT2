### Title
Unguarded `disable_rewards` Parameter in `update_rewards` Allows Any Caller to Permanently Freeze All Consensus Rewards — (`File: src/staking/staking.cairo`)

---

### Summary

`update_rewards` is a publicly callable function that accepts a caller-controlled `disable_rewards: bool` parameter. The function unconditionally advances the global `last_reward_block` cursor **before** checking `disable_rewards`. An unprivileged attacker can call this function every block with `disable_rewards: true`, consuming each block's reward slot without distributing any rewards, permanently freezing all unclaimed consensus-phase yield for every staker and delegator in the protocol.

---

### Finding Description

`IStakingRewardsManager::update_rewards` is embedded in the public ABI with no role-based access control. Its only gate is `general_prerequisites()`, which checks that the contract is not paused and that the caller is not the zero address. [1](#0-0) 

Inside `update_rewards`, the global `last_reward_block` storage slot is written at line 1485 **unconditionally**, before the `disable_rewards` branch is evaluated at line 1487: [2](#0-1) 

`last_reward_block` is a single protocol-wide field, not per-staker: [3](#0-2) 

The guard at the top of the function enforces that only one call per block can succeed: [4](#0-3) 

Because `last_reward_block` is advanced before the `disable_rewards` check, a call with `disable_rewards: true` marks the block as "processed" while distributing zero rewards. No subsequent call in the same block can pass the `current_block_number > last_reward_block` guard, so the rewards for that block are permanently lost.

---

### Impact Explanation

This is a **permanent freezing of unclaimed yield** for all stakers and delegators in the consensus-rewards phase. Every block's STRK (and BTC) rewards that would have been distributed via `_update_rewards` are silently discarded. Because `last_reward_block` is global, a single attacker transaction per block is sufficient to suppress rewards for the entire protocol indefinitely. Delegators' `cumulative_rewards_trace` entries are never updated, and stakers' `unclaimed_rewards_own` fields are never incremented. The lost yield cannot be recovered.

**Allowed impact matched:** *Permanent freezing of unclaimed yield* — **High**.

---

### Likelihood Explanation

- The function is publicly callable with no role restriction.
- The attacker needs only a valid (active, non-zero-balance) staker address to pass the staker-existence checks — any staker address visible on-chain suffices.
- The cost is one cheap transaction per block; no capital is required.
- The attack is fully automated and can run indefinitely.
- There is no on-chain mechanism to detect or revert the griefing after the fact.

**Likelihood: High.**

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` and `is_pre_consensus()` early-return checks, so the cursor is only advanced when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Only advance the cursor when rewards are actually distributed.
self.last_reward_block.write(current_block_number);

// Get current block data and update rewards.
...
```

Alternatively, restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a dedicated consensus rewards contract) via a role check, mirroring the pattern used in `update_rewards_from_attestation_contract`: [5](#0-4) 

---

### Proof of Concept

1. Consensus rewards are activated (`consensus_rewards_first_epoch` is set to a past epoch).
2. Attacker identifies any active staker address `S` with non-zero balance (readable from public events or view functions).
3. Each block, attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. The call passes all checks: contract is unpaused, caller is non-zero, `current_block_number > last_reward_block`, staker `S` is active with non-zero balance.
5. `last_reward_block` is set to `current_block_number` at line 1485.
6. Execution hits `if disable_rewards || self.is_pre_consensus() { return; }` at line 1487 and exits.
7. No rewards are computed or distributed; `_update_rewards` is never reached.
8. Any legitimate call to `update_rewards` in the same block fails with `REWARDS_ALREADY_UPDATED`.
9. Repeating this every block permanently prevents all consensus rewards from accruing to any staker or delegator. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
