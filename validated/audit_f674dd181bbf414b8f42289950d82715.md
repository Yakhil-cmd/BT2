### Title
`EVM.batchRun` Intended Non-Atomic Batch Reverts Entirely on Malformed RLP Input - (File: `fvm/evm/handler/handler.go`)

---

### Summary

`EVM.batchRun` is designed to be non-atomic: individual EVM transactions that fail validation (bad nonce, bad signature) are marked `invalid` while the rest of the batch proceeds. However, a malformed RLP-encoded transaction byte payload causes `decodeTransaction` to return an error that reverts the **entire** batch, breaking the non-atomic guarantee. Any unprivileged Flow transaction sender can exploit this by including one malformed RLP blob in a batch.

---

### Finding Description

`ContractHandler.BatchRun` calls `batchRun`, which first decodes every RLP-encoded transaction before executing any of them. If any single transaction fails RLP decoding, the function immediately returns a fatal error:

```go
// fvm/evm/handler/handler.go  lines 310-315
for i, rlpEncodedTx := range rlpEncodedTxs {
    tx, err := h.decodeTransaction(rlpEncodedTx)
    // if any tx fails decoding revert the batch
    if err != nil {
        return nil, err   // ŌåÉ entire batch aborted
    }
``` [1](#0-0) 

That error propagates back to `BatchRun`, which calls `panicOnError(err)`, panicking the entire Flow transaction:

```go
// fvm/evm/handler/handler.go  lines 282-286
h.runWithGasFeeRefund(gasFeeCollector, func() {
    results, err = h.batchRun(rlpEncodedTxs)
    panicOnError(err)
})
``` [2](#0-1) 

By contrast, `BatchRunTransactions` in the emulator layer correctly handles the analogous per-transaction error (`TransactionToMessage` failure, which covers bad signatures) by marking only that transaction as invalid and continuing:

```go
// fvm/evm/emulator/emulator.go  lines 242-249
msg, err := gethCore.TransactionToMessage(tx, GetSigner(bl.config), ...)
if err != nil {
    batchResults[i] = types.NewInvalidResult(tx.Type(), tx.Hash(), err)
    continue   // ŌåÉ non-atomic: skip this tx, keep going
}
``` [3](#0-2) 

The existing integration tests only exercise invalid-nonce and out-of-gas failures, both of which are caught at the `TransactionToMessage` level (inside `BatchRunTransactions`) and handled gracefully. No test covers a malformed RLP payload, which is caught earlier in `batchRun` and is **not** handled gracefully. [4](#0-3) 

---

### Impact Explanation

The entire Flow transaction calling `EVM.batchRun` reverts. All valid EVM transactions in the batch are discarded, no state changes are committed, and no events are emitted. This breaks the non-atomic contract of `batchRun` and can be used to grief any Flow transaction that relies on batch EVM execution completing partially.

---

### Likelihood Explanation

`EVM.batchRun` is callable by any unprivileged Flow transaction sender. The attacker only needs to include one zero-byte or otherwise malformed RLP blob in the batch. No special privileges, staked nodes, or key material are required. The attack is trivially constructable.

---

### Recommendation

Handle RLP decode failures per-transaction, the same way `BatchRunTransactions` handles `TransactionToMessage` failures. Instead of returning a fatal error, record an `InvalidResult` for the malformed entry and continue decoding the rest:

```go
for i, rlpEncodedTx := range rlpEncodedTxs {
    tx, err := h.decodeTransaction(rlpEncodedTx)
    if err != nil {
        // treat as invalid, not fatal ŌĆö mirror BatchRunTransactions behaviour
        txs[i] = nil  // sentinel; handle below
        invalidResults[i] = types.NewInvalidResult(0, gethCommon.Hash{}, err)
        continue
    }
    txs[i] = tx
    ...
}
```

Alternatively, move the RLP decode step inside `BatchRunTransactions` so that all per-transaction error handling is centralised in one place.

---

### Proof of Concept

1. Construct a Flow transaction that calls `EVM.batchRun` with a batch of `N` valid RLP-encoded EVM transactions plus one entry that is `[]byte{0xFF, 0xFF}` (invalid RLP).
2. Submit the Flow transaction.
3. Observe that the entire Flow transaction reverts (output error, empty write-set) rather than returning `N` results where the malformed entry has `status == EVM.Status.invalid`.

The decode path that panics: [5](#0-4) 

`tx.UnmarshalBinary` returns an error for any malformed input, which propagates through `decodeTransaction ŌåÆ batchRun ŌåÆ panicOnError` and reverts the entire Flow transaction.

func (h *ContractHandler) decodeTransaction(encodedTx []byte) (*gethTypes.Transaction, error) {
	usage := common.ComputationUsage{
		Kind:      environment.ComputationKindRLPDecoding,
		Intensity: uint64(len(encodedTx)),

	if err != nil {
		return nil, err
	}

	tx := gethTypes.Transaction{}
	if err := tx.UnmarshalBinary(encodedTx); err != nil {
		return nil, err
	}

	return &tx, nil
