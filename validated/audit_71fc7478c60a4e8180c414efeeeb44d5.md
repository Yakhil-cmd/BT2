### Title
Unguarded Slice in `BankContract.RequiredGas` Causes Panic on Short Calldata — (`File: x/cronos/keeper/precompiles/bank.go`)

### Summary
`BankContract.RequiredGas` performs `input[:4]` without a prior length check. An unprivileged attacker can send an EVM call to the bank precompile address (`0x0000000000000000000000000000000000000064`) with fewer than 4 bytes of calldata, triggering a Go runtime index-out-of-range panic. Because the Cosmos SDK's `runTx` panic recovery re-panics for non-`OutOfGas` panics, this crashes the validator process and can halt the chain.

### Finding Description
`BankContract.RequiredGas` in `x/cronos/keeper/precompiles/bank.go` reads:

```go
func (bc *BankContract) RequiredGas(input []byte) uint64 {
    baseCost := uint64(len(input)) * bc.kvGasConfig.WriteCostPerByte
    var methodID [4]byte
    copy(methodID[:], input[:4])   // panics if len(input) < 4
    ...
}
``` [1](#0-0) 

The expression `input[:4]` is a Go slice expression that panics at runtime with `runtime error: index out of range` whenever `len(input) < 4`. There is no guard before this line.

The sibling precompiles fixed exactly this class of bug in PR #1941 (CHANGELOG entry: *"fix: return calculated gas instead of panic for RelayerContract, add guards for ica precompile"*):

- `IcaContract.RequiredGas` guards with `if len(input) < 4 { return baseCost }` [2](#0-1) 
- `RelayerContract.RequiredGas` guards with `if inputLen < 4 { ... return ... }` [3](#0-2) 

`BankContract.RequiredGas` was not patched. The `Run` function has the same unguarded slice at line 105, but `RequiredGas` is called first by the EVM framework. [4](#0-3) 

### Impact Explanation
`RequiredGas` is called by go-ethereum's `RunPrecompiledContract` before `Run`. The panic propagates up through the EVM call stack into ethermint's `ApplyTransaction` and then into the Cosmos SDK's `runTx`. The SDK's panic recovery re-panics for any panic that is not `sdk.ErrorOutOfGas`, crashing the validator process. If a sufficient number of validators crash before the block is finalized, the chain halts. This matches the **High** impact category: *"Permanent or long-lived inability for honest users or validators to process valid transactions… under normal network assumptions."* It may also qualify as **Critical** (consensus divergence) if validators diverge on whether the block containing the crafted tx is accepted.

### Likelihood Explanation
The bank precompile address (`0x64`) is a fixed, publicly known address. Any unprivileged EVM user can send a transaction to it with 0–3 bytes of calldata. No special privilege, key, or prior state is required. The attack is trivially repeatable.

### Recommendation
Add the same length guard that was applied to `IcaContract` and `RelayerContract`:

```go
func (bc *BankContract) RequiredGas(input []byte) uint64 {
    baseCost := uint64(len(input)) * bc.kvGasConfig.WriteCostPerByte
    if len(input) < 4 {
        return baseCost
    }
    var methodID [4]byte
    copy(methodID[:], input[:4])
    ...
}
```

Apply the same fix to the `Run` function's unguarded `contract.Input[:4]` slice at line 105. [4](#0-3) 

### Proof of Concept
1. Obtain any funded EVM account on Cronos.
2. Send an EVM transaction to `0x0000000000000000000000000000000000000064` (the bank precompile) with calldata of length 0, 1, 2, or 3 bytes (e.g., `0xAB`).
3. The EVM framework calls `BankContract.RequiredGas([]byte{0xAB})`.
4. `input[:4]` panics: `runtime error: index out of range [3] with length 1`.
5. The panic propagates through `RunPrecompiledContract` → `ApplyTransaction` → `runTx`; the SDK re-panics because it is not `sdk.ErrorOutOfGas`.
6. The validator process crashes. Repeat from any account to prevent recovery.
