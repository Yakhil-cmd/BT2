Audit Report

## Title
Direct Participant Using NNS Governance Canister ID Causes Neuron ID Collision with Neurons' Fund Participants — (`rs/sns/swap/src/swap.rs`)

## Summary
`refresh_buyer_tokens` accepts any principal as `buyer` with no restriction, allowing an attacker to register the NNS governance canister ID as a direct participant. Because `create_sns_neuron_recipes` assigns direct-participant memos starting at `NEURON_BASKET_MEMO_RANGE_START` keyed on `buyer_principal`, and NF participant memos also start at `NEURON_BASKET_MEMO_RANGE_START` keyed on `nns_governance_canister_id`, both paths produce byte-for-byte identical SNS ledger subaccounts. This causes `sweep_sns` to double-transfer SNS tokens to the same subaccount and `claim_swap_neurons` to fail for the NF participant's neuron, permanently locking their SNS tokens.

## Finding Description

**Root cause — no guard in `refresh_buyer_tokens`:**

The canister endpoint accepts any `buyer` string:

```rust
// rs/sns/swap/canister/canister.rs:130-133
let p: PrincipalId = if arg.buyer.is_empty() {
    caller_principal_id()
} else {
    PrincipalId::from_str(&arg.buyer).unwrap()
};
``` [1](#0-0) 

`refresh_buyer_token_e8s` contains no check rejecting `buyer == nns_governance_canister_id`. An attacker transfers ICP to `principal_to_subaccount(NNS_GOVERNANCE_CANISTER_ID)` on the ICP ledger and calls `refresh_buyer_tokens` with `buyer = NNS_GOVERNANCE_CANISTER_ID`. [2](#0-1) 

**Memo collision in `create_sns_neuron_recipes`:**

Direct participants start at `NEURON_BASKET_MEMO_RANGE_START` keyed on `buyer_principal`:

```rust
// rs/sns/swap/src/swap.rs:858-862
match create_sns_neuron_basket_for_direct_participant(
    &buyer_principal,
    amount_sns_e8s,
    neuron_basket_construction_parameters,
    NEURON_BASKET_MEMO_RANGE_START,
)
``` [3](#0-2) 

NF participants' global memo counter also starts at `NEURON_BASKET_MEMO_RANGE_START`, and the subaccount is keyed on `nns_governance_canister_id`:

```rust
// rs/sns/swap/src/swap.rs:891
let mut global_neurons_fund_memo: u64 = NEURON_BASKET_MEMO_RANGE_START;
``` [4](#0-3) 

Per-neuron memos for NF: `memo = memo_offset + i as u64`, identical to the direct-participant path when `buyer_principal == nns_governance_canister_id`. [5](#0-4) 

**`sweep_sns` confirms the collision:**

```rust
// rs/sns/swap/src/swap.rs:2216-2231
Some(Investor::Direct(...)) => compute_neuron_staking_subaccount_bytes(p, neuron_memo),
Some(Investor::CommunityFund(_)) => compute_neuron_staking_subaccount_bytes(nns_governance.into(), neuron_memo)
``` [6](#0-5) 

When `buyer_principal == nns_governance_canister_id` and both memo sequences start at `NEURON_BASKET_MEMO_RANGE_START`, the destination subaccounts are byte-for-byte identical for every neuron in the first basket. `compute_neuron_staking_subaccount_bytes` is a deterministic SHA-256 hash of `(domain, principal_bytes, nonce)` — no randomness, no disambiguation. [7](#0-6) 

**Why existing checks fail:**

The comment at swap.rs L886-890 acknowledges the NF memo-collision problem among NF participants and addresses it with a global counter — but it never considers that a direct participant could supply `buyer_principal = nns_governance_canister_id`, which is an entirely separate code path that resets to `NEURON_BASKET_MEMO_RANGE_START` independently. [8](#0-7) 

## Impact Explanation

1. `sweep_sns` transfers SNS tokens to the same subaccount twice; both ledger transfers succeed (the second simply adds tokens to the already-funded subaccount).
2. `claim_swap_neurons` submits two `NeuronRecipe` entries with the same `neuron_id`; the second claim fails with a duplicate-neuron error.
3. The NF participant's neuron is never created. Their SNS tokens are permanently locked in the SNS governance canister's subaccount with no recovery path.
4. The attacker's direct-participant neuron also fails (they don't control NNS governance), so the attacker loses their ICP — but the NF participant suffers permanent token loss.

This matches **High ($2,000–$10,000): Significant SNS security impact with concrete user or protocol harm** — specifically, permanent loss of NF participants' SNS tokens and destruction of their governance participation rights in the newly launched SNS.

## Likelihood Explanation

- Requires only publicly obtainable ICP tokens and a single `refresh_buyer_tokens` call.
- No privileged access, no key material, no social engineering.
- The NNS governance canister ID (`rrkah-fqaaa-aaaaa-aaaaq-cai`) is a well-known public constant.
- The attack is fully deterministic and locally testable before execution.
- The only cost to the attacker is the minimum ICP participation amount (lost, not stolen).
- Executable during any open SNS swap that has Neurons' Fund participation.

## Recommendation

Add a guard in `refresh_buyer_token_e8s` (or at the canister endpoint) rejecting any `buyer` principal equal to `nns_governance_canister_id`:

```rust
if buyer == nns_governance_canister_id.get() {
    return Err("Direct participation is not allowed for the NNS governance canister ID".to_string());
}
```

Alternatively, offset the direct-participant memo range to a non-overlapping region (e.g., use a separate constant `DIRECT_PARTICIPANT_MEMO_RANGE_START` distinct from `NEURON_BASKET_MEMO_RANGE_START`), or add a uniqueness assertion over all `(principal, memo)` pairs in `create_sns_neuron_recipes` before committing recipes.

## Proof of Concept

```rust
let nns_gov_id = NNS_GOVERNANCE_CANISTER_ID.get();

// Attacker registers as direct participant with NNS governance principal
swap.refresh_buyer_token_e8s(
    nns_gov_id,
    None,
    SWAP_CANISTER_ID,
    &mock_icp_ledger_with_balance(min_icp),
).await.unwrap();

// Add one NF participant
swap.cf_participants = vec![cf_participant_with_one_neuron()];

// Commit and create recipes
swap.create_sns_neuron_recipes();

// Collect all subaccounts
let subaccounts: Vec<[u8;32]> = swap.neuron_recipes.iter().map(|r| {
    let memo = r.neuron_attributes.as_ref().unwrap().memo;
    match r.investor.as_ref().unwrap() {
        Investor::Direct(d) => compute_neuron_staking_subaccount_bytes(
            PrincipalId::from_str(&d.buyer_principal).unwrap(), memo),
        Investor::CommunityFund(_) => compute_neuron_staking_subaccount_bytes(nns_gov_id, memo),
    }
}).collect();

let unique: HashSet<_> = subaccounts.iter().collect();
assert_eq!(unique.len(), subaccounts.len(), "COLLISION DETECTED");
// ^^^ This assertion FAILS — proving the vulnerability
```
