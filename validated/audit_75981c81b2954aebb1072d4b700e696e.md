### Title
Unbounded Iteration Over `stakers` Vec in `get_stakers()` Causes Unbounded Gas Consumption - (File: src/staking/staking.cairo)

### Summary
`get_stakers()` iterates over the entire `stakers` storage `Vec` using `into_iter_full_range()`. Every call to `stake()` appends to this Vec, and stakers are **never removed** from it even after unstaking. As the staker count grows, the gas cost of `get_stakers()` grows linearly and will eventually exceed Starknet's transaction resource bounds, permanently bricking the function.

### Finding Description
In `src/staking/staking.cairo`, the `stakers` storage field is declared as a `Vec<ContractAddress>`: [1](#0-0) 

Every call to `stake()` unconditionally appends the new staker's address to this Vec: [2](#0-1) 

The comment explicitly acknowledges that stakers are never removed:

> **Note**: Stakers are not removed from this vector when they unstake.

`get_stakers()`, a public function in `IStakingConsensus`, iterates over the **entire** Vec on every call: [3](#0-2) 

The loop at line 918 uses `into_iter_full_range()` with no upper bound, performing per-staker storage reads (`is_staker_active`, `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, `get_peer_id_at_epoch`) for every entry ever added. Because unstaked stakers remain in the Vec, the effective iteration count only ever increases.

### Impact Explanation
Once the `stakers` Vec grows large enough, any call to `get_stakers()` will exceed Starknet's per-transaction resource bounds (gas/steps limit), causing the transaction to revert. This permanently prevents the function from returning the validator set for any epoch. `get_stakers()` is the consensus-layer entrypoint for determining which validators are active and their staking power — its unavailability constitutes **unbounded gas consumption** and effectively freezes the consensus validator-set query capability of the protocol.

### Likelihood Explanation
The Starknet staking protocol is designed to onboard a large number of stakers over time. Each `stake()` call by any unprivileged user permanently increments the Vec length. There is no cap on the number of stakers, no pruning mechanism, and no pagination in `get_stakers()`. The growth is monotonic and irreversible. Given sufficient protocol adoption, this threshold will be reached.

### Recommendation
1. **Paginate `get_stakers()`**: Accept `offset` and `limit` parameters so callers can retrieve the validator set in chunks.
2. **Track active stakers separately**: Maintain a separate, pruned data structure (e.g., a separate Vec or IterableMap) that removes stakers upon `unstake_action`, so the iteration set is bounded by the number of *currently active* stakers rather than all historical stakers.
3. **Alternatively**, add a `max_stakers` cap enforced in `stake()` to bound the Vec size.

### Proof of Concept
1. Deploy the staking contract.
2. Have N distinct addresses each call `stake()` with the minimum stake, then call `unstake_intent()` and `unstake_action()`. Each address remains in `self.stakers` permanently.
3. Call `get_stakers(epoch_id)` with a valid epoch. The transaction iterates over all N entries, performing multiple storage reads per entry.
4. As N grows (e.g., tens of thousands of historical stakers), the step count exceeds Starknet's transaction resource limit and the call reverts unconditionally.

The root cause is the combination of:
- `self.stakers.push(staker_address)` on every `stake()` [4](#0-3) 
- `for staker_address_ptr in self.stakers.into_iter_full_range()` with no bound in `get_stakers()` [5](#0-4) 
- No removal of stakers from the Vec on `unstake_action()` [1](#0-0)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
