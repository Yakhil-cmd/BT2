### Title
Unprivileged Caller Can Permanently Deny Block Rewards via `disable_rewards` Flag - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract accepts a caller-controlled `disable_rewards: bool` parameter with no access control. Any unprivileged address can call `update_rewards(staker_address, disable_rewards: true)` to consume the per-block reward slot for a target staker while suppressing reward distribution, permanently denying that staker their block rewards for that block.

---

### Finding Description

`update_rewards` is a public function gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. There is no role check. [1](#0-0) 

The function enforces a single-call-per-block invariant by asserting `current_block_number > last_reward_block`, then **immediately** writing `last_reward_block = current_block_number` before checking `disable_rewards`: [2](#0-1) 

The critical sequence is:

1. **Line 1485**: `self.last_reward_block.write(current_block_number)` — slot consumed.
2. **Line 1487**: `if disable_rewards || self.is_pre_consensus() { return; }` — rewards skipped.

Because the slot is consumed before the `disable_rewards` branch is evaluated, any subsequent call in the same block — including the legitimate consensus-layer call — reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The `disable_rewards` flag is a caller-supplied parameter with no commitment, no prior authorization, and no on-chain validation of who is permitted to set it to `true`. This is structurally identical to the Buffer Protocol issue: a settlement-time parameter provided by an untrusted party that controls whether value is distributed.

---

### Impact Explanation

**High — Permanent freezing / theft of unclaimed yield.**

Each block in the consensus-rewards phase produces block rewards for active stakers. An attacker who front-runs the legitimate `update_rewards` call with `disable_rewards: true` causes the staker to permanently lose those block rewards — the slot cannot be reclaimed because `last_reward_block` is already set to the current block. Repeated across every block, this completely denies all consensus-phase rewards to any targeted staker.

---

### Likelihood Explanation

**High.** The entry path requires no privilege, no stake, and no special role — only a non-zero address and knowledge of the target staker's address (which is public on-chain). The attacker simply monitors the mempool or submits a transaction at the start of each block. The cost is only gas. The attack is repeatable indefinitely.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the attestation contract, a designated consensus contract, or an operator role). Add a role check analogous to `assert_caller_is_attestation_contract` used elsewhere in the same file: [4](#0-3) 

Alternatively, remove the `disable_rewards` parameter from the public interface entirely and derive the disable condition from on-chain state (e.g., whether the staker attested in the current epoch), so no caller-supplied flag can suppress reward distribution.

---

### Proof of Concept

1. Staker `S` is active in the consensus-rewards phase. Block `B` is the current block.
2. Attacker `A` (any non-zero address) calls `update_rewards(S, disable_rewards: true)` in block `B`.
3. Inside the function:
   - `current_block_number (B) > last_reward_block` → passes.
   - `last_reward_block` is written to `B`.
   - `disable_rewards == true` → function returns with no rewards distributed.
4. The legitimate consensus caller attempts `update_rewards(S, disable_rewards: false)` in the same block `B`.
5. `current_block_number (B) > last_reward_block (B)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker `S` receives zero block rewards for block `B`. The loss is permanent.
7. Repeat for every block to fully deny `S` all consensus-phase rewards. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
