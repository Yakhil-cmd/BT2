### Title
Missing Permission Check in `MsgTurnBridge` Allows Unprivileged Bypass of Bridge Authorization - (File: `x/cronos/keeper/msg_server.go`)

### Summary
The `TurnBridge` message handler is a stub that performs no permission check and makes no state change, returning `nil, nil` unconditionally. Any unprivileged user can submit `MsgTurnBridge` and receive a success response without holding `CanTurnBridge` permission. Simultaneously, the Gravity bridge circuit breaker is permanently non-functional — no actor, including the admin, can disable the bridge.

### Finding Description
ADR-009 explicitly designates both `MsgUpdateTokenMapping` and `MsgTurnBridge` as permission-gated operations requiring `CanChangeTokenMapping` and `CanTurnBridge` respectively. `UpdateTokenMapping` is correctly implemented:

```go
// x/cronos/keeper/msg_server.go:69-82
func (k msgServer) UpdateTokenMapping(...) (...) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    ...
}
```

`TurnBridge`, however, is a no-op stub:

```go
// x/cronos/keeper/msg_server.go:84-87
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

No call to `HasPermission` with `CanTurnBridge` is made. The function returns `(nil, nil)` — a nil response and nil error — which the Cosmos SDK router treats as a successful transaction. The `CanTurnBridge` bit defined in `permissions.go` is never consulted. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation
**High — Bypass of Cronos permission authorization check.**

1. **Auth bypass**: Any unprivileged address can broadcast `MsgTurnBridge` and receive `code: 0` (success) without holding `CanTurnBridge`. The permission system for this message is entirely absent.
2. **Broken circuit breaker**: Because the handler is a no-op, the Gravity bridge state is never mutated. Even the admin or a permissioned address cannot disable the bridge. In an emergency (e.g., a bridge exploit in progress), the intended kill-switch produces a misleading success response while leaving the bridge fully operational.

The integration test in `test_gravity.py` asserts `rsp["code"] != 0` for an unprivileged caller of `turn_bridge`, but the live implementation returns `nil` error unconditionally, meaning this assertion would fail against the actual code. [1](#0-0) [4](#0-3) 

### Likelihood Explanation
The `MsgTurnBridge` RPC endpoint is publicly registered and reachable by any network participant. No special setup is required. The attacker simply submits a signed `MsgTurnBridge` transaction from any funded address. [5](#0-4) 

### Recommendation
Implement `TurnBridge` with the same pattern as `UpdateTokenMapping`:

1. Call `k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `ErrUnauthorized` on failure.
2. Implement the actual bridge enable/disable state mutation (e.g., set a parameter or flag in the keeper store).
3. Return a non-nil `*MsgTurnBridgeResponse{}` on success.

### Proof of Concept
```
# Any unprivileged address can call MsgTurnBridge and receive code 0:
cronosd tx cronos turn-bridge false --from <any_address> --chain-id cronos_777-1

# Expected: code != 0 (ErrUnauthorized)
# Actual:   code == 0, bridge state unchanged, no permission check performed
```

The admin calling `turn-bridge false` in an emergency also receives `code: 0` but the bridge remains enabled, silently defeating the circuit breaker. [1](#0-0) [6](#0-5)

```go
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {

	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
```

```go
// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
}
```
