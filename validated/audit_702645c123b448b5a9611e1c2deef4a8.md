### Title
Unprivileged Caller Can Permanently Freeze Per-Block Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `Staking` contract is publicly callable by any non-zero address and accepts a caller-controlled `disable_rewards` boolean. It unconditionally writes the current block number to the global `last_reward_block` storage slot **before** checking `disable_rewards`. An attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` every block to consume the global reward slot without distributing any rewards, permanently destroying all consensus block rewards for every staker.

### Finding Description

`update_rewards` is exposed as a public function in `IStakingRewardsManager`: [1](#0-0) 

Its only access guard is `general_prerequisites()`, which only checks the contract is unpaused and the caller is non-zero: [2](#0-1) 

Inside `update_rewards`, the global `last_reward_block` is written **before** the `disable_rewards` branch: [3](#0-2) 

The critical sequence is:
1. Assert `current_block_number > last_reward_block` (line 1454–1458)
2. **Write** `last_reward_block = current_block_number` (line 1485)
3. If `disable_rewards || is_pre_consensus()` → **return early, no rewards distributed** (line 1487–1489)

`last_reward_block` is a single global slot, not per-staker: [4](#0-3) 

Because the slot is consumed before the early return, any subsequent legitimate call to `update_rewards` for the same block will fail with `REWARDS_ALREADY_UPDATED`. The block's rewards are not accumulated anywhere — they are simply never distributed.

### Impact Explanation

An attacker who calls `update_rewards(valid_staker_address, disable_rewards: true)` once per block:
- Permanently destroys all consensus block rewards for that block for every staker in the protocol.
- The rewards are not deferred or queued — `_update_rewards` is never reached, so `unclaimed_rewards_own` and pool reward traces are never incremented.
- This constitutes **permanent freezing of unclaimed yield** for all stakers, matching the High-severity allowed impact.

### Likelihood Explanation

- The function is fully public; no role, key, or capital is required.
- The only prerequisite is passing a valid, active staker address — all staker addresses are discoverable from on-chain `NewStaker` events.
- The cost is one transaction per block (gas only). A motivated attacker can sustain this indefinitely.
- Likelihood: **High**.

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus/sequencer contract or a designated rewards manager role), or remove the caller-controlled `disable_rewards` parameter and derive the disable condition entirely from on-chain state (e.g., `is_pre_consensus()`). The global `last_reward_block` slot must not be writable by an unprivileged path that skips reward distribution.

### Proof of Concept

```
// Attacker script (pseudocode, run once per block):
staking.update_rewards(
    staker_address = <any valid active staker>,
    disable_rewards = true   // caller-controlled
)
// Effect:
//   last_reward_block = current_block  (written before early return)
//   _update_rewards never called       (early return taken)
//   All legitimate update_rewards calls for this block now revert
//   with REWARDS_ALREADY_UPDATED.
//   Block rewards are permanently lost.
``` [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
