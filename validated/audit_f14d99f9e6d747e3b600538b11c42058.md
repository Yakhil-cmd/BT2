### Title
Presignature Reuse Enabled by Non-Consuming `rerandomize_presign` API and `Clone` Derivation — (`src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both OT-based and Robust ECDSA presignature types (`PresignOutput`, `RerandomizedPresignOutput`) derive `Clone`, and the `rerandomize_presign()` function accepts `&PresignOutput` by shared reference rather than consuming it. This means a single completed presignature can be rerandomized and fed into `sign()` an unlimited number of times. The library's own documentation explicitly states this leads to private key extraction, yet no enforcement mechanism exists in the API.

---

### Finding Description

The analog to the external report's "completed proposals can be voted on and executed again" is: a completed presignature (the cryptographic analog of a "proposal") can be "executed" (used for signing) multiple times because the library never invalidates or consumes the presignature state.

**Root cause 1 — `rerandomize_presign` takes `&PresignOutput` (shared reference, not consumed):**

In `src/ecdsa/ot_based_ecdsa/mod.rs`:
```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // <-- reference, never consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

In `src/ecdsa/robust_ecdsa/mod.rs`:
```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // <-- reference, never consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

A caller can call `rerandomize_presign()` on the same `PresignOutput` any number of times, producing multiple distinct `RerandomizedPresignOutput` values, each of which can be passed to `sign()`.

**Root cause 2 — `PresignOutput` and `RerandomizedPresignOutput` both derive `Clone`:**

OT-based (`src/ecdsa/ot_based_ecdsa/mod.rs`):
```rust
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput { ... }

#[derive(Debug, Clone, Serialize, Deserialize, ZeroizeOnDrop)]
pub struct RerandomizedPresignOutput { ... }
```

Robust (`src/ecdsa/robust_ecdsa/mod.rs`):
```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, ZeroizeOnDrop)]
pub struct PresignOutput { ... }

#[derive(Debug, Clone, Serialize, Deserialize, ZeroizeOnDrop)]
pub struct RerandomizedPresignOutput { ... }
```

Even though `sign()` takes `presignature: RerandomizedPresignOutput` by value (a partial mitigation), `Clone` allows a caller to trivially copy the value before passing it.

**The complete reuse path:**
```rust
// Step 1: generate presignature once
let presign_out: PresignOutput = run_presign_protocol(...);

// Step 2: rerandomize and sign — presign_out is NOT consumed
let rerandomized1 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args1)?;
sign(..., rerandomized1, msg_hash1)?;  // first signing

// Step 3: reuse the same presign_out for a second signing session
let rerandomized2 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args2)?;
sign(..., rerandomized2, msg_hash2)?;  // second signing — nonce reuse!
```

Or, using `Clone` directly on `RerandomizedPresignOutput`:
```rust
let rerandomized = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args)?;
let rerandomized_clone = rerandomized.clone();
sign(..., rerandomized, msg_hash1)?;
sign(..., rerandomized_clone, msg_hash2)?;  // identical nonce, different message
```

---

### Impact Explanation

The library's own security documentation explicitly states the consequence:

> "Never reuse a presignature, even across failed, aborted, or partially completed signing sessions."

And:

> "a novel split-view attack exists that can extract the secret key using as few as 2t+2 presigning participants, with as few as two signing sessions."

In standard ECDSA, reusing the nonce `k` across two signatures `(r, s1)` and `(r, s2)` for messages `h1` and `h2` allows solving for the private key `x`:
```
s1 = h1*k + r*x  (mod q)
s2 = h2*k + r*x  (mod q)
=> x = (s1 - s2) / (h1 - h2) * (1/r)  (mod q)
```

Even with rerandomization (different `delta` values), the underlying nonce material is algebraically related across rerandomizations of the same `PresignOutput`. Two signing sessions using the same base presignature expose the private key.

**Impact class: Critical — Extraction of private signing shares / aggregate secret material.**

---

### Likelihood Explanation

The attack is reachable by any library caller — no special privilege is required. The `rerandomize_presign()` function's reference-based API makes reuse the natural, ergonomic path: a caller who stores a `PresignOutput` and calls `rerandomize_presign()` twice (e.g., to retry a failed signing session, or to sign two messages "efficiently") triggers the vulnerability without any malicious intent. The `Clone` derive on both output types further lowers the barrier. The library's warning exists only in documentation and comments, not in the type system.

---

### Recommendation

1. **Make `rerandomize_presign` consume `PresignOutput` by value** so that the presignature is moved and cannot be reused:
   ```rust
   pub fn rerandomize_presign(
       presignature: PresignOutput,  // consume, not borrow
       args: &RerandomizationArguments,
   ) -> Result<Self, ProtocolError>
   ```

2. **Remove `Clone` from `PresignOutput` and `RerandomizedPresignOutput`** so that the type system enforces single-use at compile time. The `ZeroizeOnDrop` derive already signals these are sensitive single-use secrets; `Clone` contradicts that intent.

3. **Add a `used` flag or use a wrapper type** (e.g., `SingleUse<PresignOutput>`) that panics or errors on second use if consuming by value is not feasible for API compatibility reasons.

---

### Proof of Concept

```rust
// Attacker-controlled coordinator runs two signing sessions
// using the same presignature output.

let presign_out: PresignOutput = /* result of presign protocol */;

// Session 1: sign message h1
let rerandomized1 = RerandomizedPresignOutput::rerandomize_presign(
    &presign_out,  // presign_out is NOT consumed
    &args,
)?;
let sig1 = run_sign_protocol(rerandomized1, h1);  // produces (r, s1)

// Session 2: sign message h2 — same presign_out, same nonce material
let rerandomized2 = RerandomizedPresignOutput::rerandomize_presign(
    &presign_out,  // reused!
    &args,
)?;
let sig2 = run_sign_protocol(rerandomized2, h2);  // produces (r, s2)

// Key extraction:
// x = (s1 - s2) * (h1 - h2)^{-1} * r^{-1}  (mod q)
```
