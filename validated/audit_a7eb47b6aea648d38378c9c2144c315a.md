### Title
Reward Address Can Be Set to the Staking or Pool Contract Itself, Permanently Trapping Unclaimed Yield - (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

Both `Staking.change_reward_address` and `Pool.change_reward_address` only validate that the provided `reward_address` is not a registered token address. Neither checks whether the address equals the contract itself (`get_contract_address()`). A staker or pool member can set their `reward_address` to the staking contract or pool contract address respectively. When rewards are subsequently claimed, STRK tokens are transferred into the contract with no recovery path, permanently freezing unclaimed yield.

---

### Finding Description

**Root cause in `staking.cairo` — `change_reward_address` and `stake`:**

`change_reward_address` performs only one address validation:

```cairo
assert!(
    !self.does_token_exist(token_address: reward_address),
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
```

The staking contract address is not a registered token, so this check passes when `reward_address == get_contract_address()`. The same single-check pattern appears in `stake()`.

When `claim_rewards` is later called, `send_rewards_to_staker` unconditionally transfers STRK to whatever `reward_address` is stored:

```cairo
token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
```

If `reward_address` is the staking contract itself, the STRK is deposited into the contract's own balance with no mechanism to recover it.

**Root cause in `pool.cairo` — `change_reward_address`:**

```cairo
assert!(
    self.token_dispatcher.contract_address.read() != reward_address,
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
```

Only the token address is blocked. The pool contract address is not the token address, so a pool member can set `reward_address = pool_contract_address`. When `claim_rewards` executes:

```cairo
let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
```

STRK rewards are sent to the pool contract itself, where they are permanently mixed with delegated stake balances and irrecoverable.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Any staker or pool member can deliberately (or accidentally) route their earned STRK rewards into the staking or pool contract. Once transferred, there is no admin withdrawal function, no sweep mechanism, and no way to distinguish trapped reward tokens from legitimate staked balances. The yield is permanently lost to the affected user and effectively burned from the protocol's perspective.

---

### Likelihood Explanation

**Medium-High.** The entry path requires only a single permissionless call (`change_reward_address`) by the staker or pool member themselves — no privileged role is needed. The missing check is a simple address equality guard that is absent from both contracts. Any staker or delegator who either deliberately griefs themselves or makes an input error (e.g., copy-pasting the wrong address) triggers the freeze. The pool case is particularly accessible since any delegator can call `Pool.change_reward_address` with the pool's own contract address.

---

### Recommendation

1. In `Staking.stake` and `Staking.change_reward_address`, add:
   ```cairo
   assert!(reward_address != get_contract_address(), "Reward address cannot be staking contract");
   ```

2. In `Pool.change_reward_address`, add:
   ```cairo
   assert!(reward_address != get_contract_address(), "Reward address cannot be pool contract");
   ```

3. Consider also blocking the `reward_supplier` contract address and other known protocol contract addresses from being used as reward destinations.

---

### Proof of Concept

**Staking contract scenario:**

1. Alice calls `Staking.stake(reward_address: staking_contract_address, ...)`.
   - The only check at [1](#0-0)  passes because the staking contract is not a registered token.
2. Alice earns STRK rewards over several epochs via attestation.
3. Alice (or anyone) calls `Staking.claim_rewards(staker_address: alice)`.
4. `send_rewards_to_staker` executes [2](#0-1) , transferring STRK to `staking_contract_address`.
5. The staking contract has no function to withdraw arbitrary STRK balances. Alice's yield is permanently frozen.

**Pool contract scenario:**

1. Bob delegates to a pool and calls `Pool.change_reward_address(pool_contract_address)`.
   - The only check at [3](#0-2)  passes because the pool contract is not the STRK token address.
2. Bob earns pool rewards over several epochs.
3. Bob (or anyone) calls `Pool.claim_rewards(pool_member: bob)`.
4. The transfer at [4](#0-3)  sends STRK to the pool contract itself.
5. The pool contract has no sweep or recovery function. Bob's unclaimed yield is permanently frozen inside the pool contract.

The missing guard in `Staking.change_reward_address` [5](#0-4)  and `Pool.change_reward_address` [6](#0-5)  is the necessary vulnerable step — both functions accept any non-token address without checking for the contract's own address.

```text
        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                !self.does_token_exist(token_address: reward_address),
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let old_address = staker_info.reward_address;

            // Update reward_address and commit to storage.
            staker_info.reward_address = reward_address;
            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.
            self
                .emit(
                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,
                    },
                );
        }
```
