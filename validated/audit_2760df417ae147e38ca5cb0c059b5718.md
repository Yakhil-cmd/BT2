### Title
Missing Caller Validation in `update_rewards` Allows Anyone to Permanently Skip Block Reward Distribution - (File: `src/staking/staking.cairo`)

### Summary

The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which writes `last_reward_block` to the current block number without distributing any rewards. Because `last_reward_block` is a single global storage variable, this permanently prevents the sequencer from distributing rewards for that block to any staker.

### Finding Description

The protocol specification explicitly states that `update_rewards` has access control "Only starkware sequencer": [1](#0-0) 

The actual implementation in `StakingRewardsManagerImpl` performs no such check: [2](#0-1) 

The function accepts a `disable_rewards: bool` parameter. When `true`, it still writes `last_reward_block` to the current block number but returns before distributing any rewards: [3](#0-2) 

Because `last_reward_block` is a single global storage variable (not per-staker), any subsequent call in the same block — including the legitimate sequencer call — fails with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

Compare this to `update_rewards_from_attestation_contract`, which correctly enforces its caller restriction via `assert_caller_is_attestation_contract()`: [5](#0-4) 

### Impact Explanation

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` in block N:

1. Sets `last_reward_block = N` with no rewards distributed.
2. Blocks the sequencer from calling `update_rewards` for block N (reverts with `REWARDS_ALREADY_UPDATED`).
3. Rewards for block N are permanently lost — there is no recovery path since `last_reward_block` can only advance forward.

Repeated every block, this permanently freezes all staker and pool member unclaimed yield. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation

The entry point is a public external function with no authentication. Any address on Starknet can call it. The only cost is the gas for the transaction. A motivated attacker (e.g., a competing validator or a griefing actor) can automate this call every block. The `disable_rewards: true` path is a valid code path that passes all existing guards (staker existence, active status, non-zero balance), so no special setup is required beyond having any valid active staker address to pass as the argument.

### Recommendation

Add a caller check enforcing that only the designated sequencer address can call `update_rewards`, mirroring the pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // Add this check
    // ... rest of function
}
```

The sequencer address should be stored in contract storage and validated here, consistent with how `assert_caller_is_attestation_contract` works. [6](#0-5) 

### Proof of Concept

```
1. Deploy the staking system with an active staker (K epochs after stake).
2. Advance to the consensus rewards epoch.
3. From any unprivileged address `attacker`:
   - Call staking.update_rewards(staker_address: any_active_staker, disable_rewards: true)
   - This succeeds: last_reward_block is set to current block, no rewards distributed.
4. The sequencer attempts:
   - Call staking.update_rewards(staker_address: any_staker, disable_rewards: false)
   - This reverts with REWARDS_ALREADY_UPDATED.
5. Rewards for this block are permanently lost.
6. Repeat step 3 every block to freeze all reward accrual indefinitely.
```

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
