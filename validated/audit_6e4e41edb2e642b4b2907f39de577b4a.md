### Title
Floor Rounding in `validate_tx_l2_gas_price_within_threshold` Allows Transactions Below the Intended Gas Price Threshold — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The stateful gateway validator computes the minimum acceptable L2 gas price threshold using floor (truncating) integer division via `.to_integer()` on a `Ratio<u128>`. The invariant requires that a transaction's `max_price_per_unit` be at least `min_gas_price_percentage / 100 * previous_block_l2_gas_price`. Because the threshold is rounded **down** instead of **up**, a transaction whose gas price equals `floor(percentage * price / 100)` passes the check even when the mathematically correct threshold is `ceil(percentage * price / 100)`, i.e., one FRI unit higher.

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← truncates (floor)
if tx_l2_gas_price.0 < threshold { ... }
```

`Ratio::to_integer()` truncates toward zero. When `min_gas_price_percentage * previous_block_l2_gas_price` is not divisible by 100, the computed `threshold` is one FRI unit below the true rational threshold. A transaction with `tx_l2_gas_price = floor(pct * price / 100)` satisfies `tx_l2_gas_price >= threshold` and is admitted, even though the intended policy requires `tx_l2_gas_price >= ceil(pct * price / 100)`.

Concrete example:
- `min_gas_price_percentage = 90`, `previous_block_l2_gas_price = 101 FRI/gas`
- True threshold: `ceil(90 * 101 / 100) = ceil(90.9) = 91`
- Computed threshold: `floor(90.9) = 90`
- A transaction with `tx_l2_gas_price = 90` passes the check and is admitted to the mempool. [1](#0-0) 

The correct rounding direction for a **minimum** threshold is ceiling: the threshold should be the smallest integer that is **at least** `pct/100 * price`. Using floor makes the enforced threshold strictly smaller than the policy intends whenever the product is non-integer.

Compare with the rest of the codebase, which consistently uses ceiling rounding when converting gas amounts upward to protect the protocol: [2](#0-1) [3](#0-2) 

### Impact Explanation

The gateway's stateful path is supposed to gate mempool admission on the transaction's L2 gas price being at least `min_gas_price_percentage %` of the previous block's price. Floor rounding silently lowers that gate by up to 1 FRI unit. Any transaction whose `max_price_per_unit` equals `floor(pct * prev_price / 100)` is admitted when it should be rejected.

Impact: **High — Mempool/gateway admission accepts invalid transactions before sequencing.** [4](#0-3) 

### Likelihood Explanation

The condition is triggered whenever `min_gas_price_percentage * previous_block_l2_gas_price` is not divisible by 100, which is the common case for arbitrary gas prices. Any unprivileged user can craft a transaction with `max_price_per_unit = floor(pct * prev_price / 100)` and submit it to the gateway. No special privileges or knowledge beyond the public previous-block gas price are required.

### Recommendation

Replace `.to_integer()` (floor) with ceiling division. Using `num_rational::Ratio::ceil().to_integer()` or an equivalent integer ceiling formula:

```rust
// ceiling of (pct * price) / 100
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This matches the rounding convention used everywhere else in the codebase where a minimum bound is computed from a ratio.

### Proof of Concept

1. Observe the previous block's L2 gas price: `prev = 101 FRI/gas`.
2. Gateway config: `min_gas_price_percentage = 90`.
3. Compute `floor(90 * 101 / 100) = 90`.
4. Submit an `AllResources` V3 invoke transaction with `l2_gas.max_price_per_unit = 90`.
5. `validate_tx_l2_gas_price_within_threshold` computes `threshold = 90`, checks `90 < 90` → false → **transaction admitted**.
6. The intended policy threshold is `ceil(90.9) = 91`; the transaction should have been rejected with `GAS_PRICE_TOO_LOW`. [5](#0-4)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
