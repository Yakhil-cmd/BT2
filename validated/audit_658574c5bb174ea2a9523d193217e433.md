### Title
Gas Fee Permanent Lock via Self-Transfer When `gasFeeCollector == CoinbaseAddress` - (File: `fvm/evm/handler/handler.go`)

---

### Summary

`runWithGasFeeRefund` in `fvm/evm/handler/handler.go` uses a snapshot-diff pattern to forward accumulated gas fees from the internal `CoinbaseAddress` accumulator to a caller-supplied `gasFeeCollector`. When the caller supplies `gasFeeCollector == types.CoinbaseAddress`, the resulting `cb.Transfer(CoinbaseAddress, diff)` is a self-transfer: the EVM deducts `diff` from `CoinbaseAddress` and immediately credits it back. The net balance is unchanged, the gas fees are never forwarded, and they are permanently locked at `CoinbaseAddress` — a special protocol address that no user or contract can withdraw from.

---

### Finding Description

`runWithGasFeeRefund` is the single code path used by both `Run` and `BatchRun` to collect and forward EVM gas fees:

```go
// fvm/evm/handler/handler.go:252-266
func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
    cb := h.AccountByAddress(types.CoinbaseAddress, true)
    initCoinbaseBalance := cb.Balance()          // snapshot before execution
    f()                                           // gas fees accumulate at CoinbaseAddress
    afterBalance := cb.Balance()
    diff := new(big.Int).Sub(afterBalance, initCoinbaseBalance)
    if diff.Sign() > 0 {
        cb.Transfer(gasFeeCollector, diff)        // forward fees to caller-supplied address
    }
    ...
}

When `gasFeeCollector == CoinbaseAddress`, `from == to` in the DirectCall. The EVM processes this as a valid self-transfer: it deducts `diff` from `CoinbaseAddress` and immediately credits `diff` back. The net balance of `CoinbaseAddress` is unchanged at `afterBalance`. The `diff` is never forwarded anywhere. On every subsequent call to `runWithGasFeeRefund`, the locked amount is absorbed into the new `initCoinbaseBalance` snapshot and is never recovered.

---

### Impact Explanation

Any gas fees paid by EVM transactions executed under a call where `coinbase == CoinbaseAddress` are permanently locked at `CoinbaseAddress`. Because `CoinbaseAddress` is a special protocol-internal address (prefix `0x00…03`) with no user-accessible withdrawal path, the locked attoflow is irrecoverable. This is a direct cross-VM asset loss: FLOW tokens that entered the EVM bridge as gas payments are destroyed from the perspective of every user and contract on the network.

func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
	// capture coinbase init balance
	cb := h.AccountByAddress(types.CoinbaseAddress, true)
	initCoinbaseBalance := cb.Balance()
	f()
	// transfer the gas fees collected to the gas fee collector address
	afterBalance := cb.Balance()
	diff := new(big.Int).Sub(afterBalance, initCoinbaseBalance)
	if diff.Sign() > 0 {
		cb.Transfer(gasFeeCollector, diff)
	}
	if diff.Sign() < 0 { // this should never happen but in case
		panic(fvmErrors.NewEVMError(fmt.Errorf("negative balance change on coinbase")))
	}
}
