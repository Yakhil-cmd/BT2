### Title
Stale Pending Block Synthesizes `DeprecatedPendingBlock` with Zero `l2_gas_price`, Causing `estimate_fee`/`simulate_transactions` to Use `NonzeroGasPrice::MIN` (= 1) ‚Äî (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

When the in-memory pending data is stale (its `parent_block_hash` does not match the latest committed block hash), `read_pending_data` synthesizes a `DeprecatedPendingBlock`. That type has no `l2_gas_price` field; its `l2_gas_price()` accessor unconditionally returns `GasPricePerToken::default()` (zero). `create_block_context` then substitutes `NonzeroGasPrice::MIN` (= 1) for both the wei and fri L2 gas price slots in the `BlockContext`. Every `estimate_fee` or `simulate_transactions` call with `block_id = Tag(Pending)` that lands during this window executes against a `BlockContext` whose L2 gas price is 1 instead of the real value (typically on the order of 10‚Åπ fri on mainnet), producing an authoritative-looking fee estimate that is off by ~10‚Åπ√ó.

### Finding Description

**Step 1 ‚Äî `read_pending_data` synthesizes a `DeprecatedPendingBlock` on hash mismatch.** [1](#0-0) 

When `pending_data.block.parent_block_hash() != latest_header.block_hash`, the function returns a freshly constructed `DeprecatedPendingBlock`. It copies `eth_l1_gas_price`, `strk_l1_gas_price`, `timestamp`, `sequencer_address`, and `starknet_version` from the latest committed header, but fills everything else with `..Default::default()`. `DeprecatedPendingBlock` has no `l2_gas_price` or `l1_data_gas_price` fields, so those are structurally absent.

**Step 2 ‚Äî `l2_gas_price()` on a `Deprecated` variant returns zero.** [2](#0-1) 

The accessor unconditionally returns `GasPricePerToken::default()` for the `Deprecated` arm. Both `price_in_wei` and `price_in_fri` are zero.

**Step 3 ‚Äî `client_pending_data_to_execution_pending_data` forwards the zero price.** [3](#0-2) 

The zero `GasPricePerToken` is placed verbatim into `ExecutionPendingData.l2_gas_price`.

**Step 4 ‚Äî `create_block_context` substitutes `NonzeroGasPrice::MIN` for zero.** [4](#0-3) 

`NonzeroGasPrice::new(GasPrice(0))` returns `Err`, so `.unwrap_or(NonzeroGasPrice::MIN)` is taken. `NonzeroGasPrice::MIN` is defined as `GasPrice(1)`. [5](#0-4) 

**Step 5 ‚Äî `estimate_fee` and `simulate_transactions` use this corrupted `BlockContext`.** [6](#0-5) 

Both `estimate_fee` and `simulate_transactions` call `read_pending_data` ‚Üí `client_pending_data_to_execution_pending_data` ‚Üí `exec_estimate_fee` / `exec_simulate_transactions`, which internally call `create_block_context` with the zero-l2_gas_price `PendingData`. [7](#0-6) 

### Impact Explanation

Any `estimate_fee` or `simulate_transactions` RPC call with `block_id = Tag(Pending)` that is served during the stale window receives a fee estimate computed with `l2_gas_price = 1` (wei and fri) instead of the real pending-block value. On a live network the real L2 gas price is on the order of 10‚Åπ fri; the returned estimate for the L2 gas component is therefore off by a factor of ~10‚Åπ. The response is structurally valid and indistinguishable from a correct answer, so callers (wallets, dApps, relayers) will act on it. This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

### Likelihood Explanation

The stale window is a normal operational state. It occurs on every block transition: the moment a new block is committed to storage, `latest_header.block_hash` advances, but the in-memory `pending_data` (updated by the pending-sync loop) still carries the old `parent_block_hash`. Any RPC call that arrives in this interval ‚Äî which can span multiple seconds under load or network delay ‚Äî hits the bug. No special privileges or adversarial setup are required; any unprivileged caller issuing `estimate_fee` with `block_id = Pending` can observe the corrupted value.

### Recommendation

In the `else` branch of `read_pending_data`, populate `l2_gas_price` (and `l1_data_gas_price`) from the latest committed header rather than relying on the `DeprecatedPendingBlock` fallback, which structurally cannot carry those fields. One approach is to return a `PendingBlockOrDeprecated::Current(PendingBlock { ‚Ä¶ })` in the fallback path, copying all gas-price fields from `latest_header.block_header_without_hash`. Alternatively, extend the synthesized struct to carry the missing prices explicitly before passing it downstream.

### Proof of Concept

```
1. Node has committed block N with block_hash = H_N and l2_gas_price_fri = 1_000_000_000.
2. The pending-sync loop has not yet updated pending_data; pending_data.block.parent_block_hash = H_{N-1}.
3. Unprivileged caller sends:
       starknet_estimateFee(transactions=[‚Ä¶], block_id="pending")
4. read_pending_data: H_{N-1} != H_N ‚Üí synthesizes DeprecatedPendingBlock.
5. l2_gas_price() ‚Üí GasPricePerToken { price_in_wei: 0, price_in_fri: 0 }.
6. create_block_context: NonzeroGasPrice::new(0).unwrap_or(MIN) ‚Üí 1.
7. BlockContext.gas_prices.strk_gas_prices.l2_gas_price = 1.
8. estimate_fee returns a fee where the L2 gas component is computed at price 1 fri/gas
   instead of 1_000_000_000 fri/gas ‚Äî a ~10^9√ó underestimate.
```
