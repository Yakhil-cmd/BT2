### Title
Any unprivileged caller can permanently freeze per-block consensus rewards by calling `update_rewards` with `disable_rewards: true` ‚Äî (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the Staking contract is callable by any non-zero address. The `last_reward_block` storage variable is written **before** the `disable_rewards` guard is evaluated. An attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block, consuming the per-block reward slot without distributing any rewards, permanently destroying that block's yield for all stakers and delegators.

### Finding Description

`update_rewards` is exposed through `IStakingRewardsManager` with no access control beyond `general_prerequisites()` (which only checks `is_paused` and `caller != zero`):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, not zero caller
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // ‚Üê last_reward_block is committed HERE, before the disable_rewards check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ‚Üê exits without distributing rewards
    }
    ...
    self._update_rewards(...);
}
```

The ordering is the root cause: `last_reward_block` is updated unconditionally, then the function exits early when `disable_rewards == true`. Because the per-block guard (`current_block_number > last_reward_block`) is now satisfied for no further call in the same block, the block's reward slot is consumed and the rewards are never credited to `unclaimed_rewards` in `RewardSupplier`, never transferred to any staker, and never forwarded to any pool. There is no retroactive mechanism to reclaim skipped blocks.

The `disable_rewards` flag is a plain `bool` parameter with no caller restriction. Any externally-owned address can supply `true`.

### Impact Explanation

Every block in the consensus-rewards phase carries a fixed STRK (and BTC) reward computed by `calculate_block_rewards` ‚Üí `update_current_epoch_block_rewards`. If an attacker calls `update_rewards(valid_staker, disable_rewards: true)` once per block:

- `last_reward_block` advances to the current block, locking out all legitimate callers for that block.
- `_update_rewards` is never reached, so `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` is never called.
- The yield for that block is permanently destroyed ‚Äî it is never added to any staker's `unclaimed_rewards_own`, never forwarded to any pool's `cumulative_rewards_trace`, and never requested from L1.

Sustained over time this constitutes **permanent freezing of unclaimed yield** for all stakers and pool members, matching the High-impact category.

### Likelihood Explanation

- **Entry point**: `update_rewards` is a public external function; any non-zero EOA or contract can call it.
- **Required knowledge**: a valid, active staker address ‚Äî trivially obtained from the public `stakers` Vec or from `NewStaker` events.
- **Cost**: one Starknet transaction per block (~2‚Äì3 s block time). Starknet L2 gas fees are low; the attack is economically viable with no profit requirement (pure griefing).
- **No special role, no leaked key, no bridge compromise** is needed.

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a call with `disable_rewards: true` does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);  // ‚Üê moved here
...
self._update_rewards(...);
```

Alternatively, restrict who may pass `disable_rewards: true` (e.g., only the staker themselves or a designated operator role), or remove the parameter entirely and derive the flag from on-chain state.

### Proof of Concept

1. Attacker reads any active staker address from `stakers` Vec or `NewStaker` events.
2. Each block, attacker submits: `staking.update_rewards(valid_staker, disable_rewards: true)`.
3. `last_reward_block` is set to `current_block_number` (line 1485).
4. Function returns at line 1487 without calling `_update_rewards`.
5. Any legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. The block's STRK/BTC rewards are permanently lost ‚Äî never credited to any staker or pool.
7. Repeat every block; all consensus-phase yield is frozen indefinitely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
