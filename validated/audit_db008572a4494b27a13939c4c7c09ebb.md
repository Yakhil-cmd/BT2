### Title
Unprivileged Caller Can Suppress Consensus Reward Distribution via `disable_rewards` Parameter — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no authorization check and exposes a caller-controlled `disable_rewards` boolean. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)`, which writes the current block number to the global `last_reward_block` storage slot **before** the `disable_rewards` guard is evaluated. This consumes the single per-block reward slot for the entire protocol without distributing any rewards. A subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`. Repeated every block, this permanently freezes all consensus-era unclaimed yield for all stakers.

---

### Finding Description

`update_rewards` is part of `StakingRewardsManagerImpl` and is the entry point for distributing per-block consensus rewards (V3 mode). Its only access guard is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

The function writes `current_block_number` to the global `last_reward_block` unconditionally, **before** the `disable_rewards` branch: [2](#0-1) 

Only after that write does the early-return guard fire: [3](#0-2) 

`last_reward_block` is a single global storage slot, not per-staker: [4](#0-3) 

Because the slot is global, one call with any valid active staker and `disable_rewards: true` blocks every staker from receiving rewards in that block. The attacker needs only to supply any staker address that passes the two validity checks (staker exists, non-zero STRK balance at current epoch). [5](#0-4) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

If an attacker calls `update_rewards(valid_staker, disable_rewards: true)` at the start of every block, no staker ever receives consensus block rewards. The rewards are never minted/transferred to stakers; they remain unclaimed indefinitely. This matches the allowed impact: *"Permanent freezing of unclaimed yield."*

---

### Likelihood Explanation

**Medium.** The attack requires no privileged key, no token balance, and no special setup — only the address of any currently active staker with non-zero STRK balance (publicly readable from chain state). The cost is one cheap Starknet transaction per block. The attacker gains nothing financially, making it a pure griefing vector, but the barrier to execution is very low.

---

### Recommendation

Restrict who may call `update_rewards` to a trusted role (e.g., the consensus/attestation system, a designated keeper role, or the staker themselves). At minimum, the `disable_rewards: true` path must not be reachable by arbitrary callers. One approach:

1. Add a role check (e.g., `only_app_governor` or a dedicated `REWARDS_MANAGER_ROLE`) to `update_rewards`.
2. Alternatively, move the `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard so that a suppressed call does not consume the per-block slot.

---

### Proof of Concept

1. Deploy the Staking contract in consensus-rewards mode (`consensus_rewards_first_epoch` is set and current epoch ≥ that value).
2. Identify any active staker `S` with non-zero STRK balance (readable via `staker_info_v3`).
3. At the start of block `N`, call `update_rewards(S, disable_rewards: true)` from any EOA.
   - `last_reward_block` is written to `N`.
   - No rewards are distributed.
4. The legitimate consensus caller attempts `update_rewards(S, disable_rewards: false)` in the same block `N`.
   - The call reverts: `assert!(current_block_number > self.last_reward_block.read(), ...)` fails because `current_block_number == N == last_reward_block`.
5. Repeat step 3 every block. All stakers permanently receive zero consensus rewards. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
