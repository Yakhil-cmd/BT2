### Title
`MsgTurnBridge` Handler Is a No-Op: Missing Permission Check and Missing State Update Allow Any Unprivileged Caller to Silently Succeed While the Bridge Circuit Breaker Remains Permanently Broken — (File: `x/cronos/keeper/msg_server.go`)

---

### Summary

The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is completely unimplemented. It performs no permission check and makes no state change, returning `nil, nil` unconditionally. Any unprivileged address can submit `MsgTurnBridge` and receive a successful on-chain result, while the bridge's `bridge_active` parameter is never actually modified. The intended circuit breaker for the Gravity bridge is therefore permanently non-functional.

---

### Finding Description

The `MsgTurnBridge` message is the designated on-chain circuit breaker for the Gravity bridge. It is supposed to be restricted to addresses holding the `CanTurnBridge` permission bit, and it is supposed to update the `bridge_active` parameter in the cronos module params.

The actual implementation is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Two things are absent:

1. **No permission check.** Every other restricted handler in the same file calls `k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping)` or compares `msg.From` against the admin. `TurnBridge` does neither. The `CanTurnBridge` permission constant is defined and the `HasPermission` helper exists, but they are never invoked here. [2](#0-1) [3](#0-2) 

2. **No state change.** The handler never reads `msg.Enable`, never calls `k.SetParams`, and never modifies `bridge_active`. The `bridge_active` flag that the EVM hooks and gravity hooks consult to gate bridge operations is therefore immutable via this message path.

Compare with `UpdateTokenMapping`, which correctly gates on permission before acting: [4](#0-3) 

The proto definition and the permission system both expect `TurnBridge` to be a privileged, state-mutating operation: [5](#0-4) [6](#0-5) 

---

### Impact Explanation

**Permission bypass (High):** Any unprivileged address can submit `MsgTurnBridge` with any `enable` value and receive a successful transaction result (`code = 0`). The `CanTurnBridge` authorization check is entirely absent. This directly matches the allowed High impact: *Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks.*

**Permanent bridge circuit-breaker failure (High):** Because the handler never writes `bridge_active`, the Gravity bridge cannot be disabled via the intended emergency mechanism. If the bridge is active and a security incident occurs (e.g., a malicious batch relay, an oracle manipulation, or a contract exploit), the admin or any permissioned address calling `TurnBridge(false)` will receive a success response but the bridge will remain fully operational. The only remaining path to disable the bridge is a governance proposal through `MsgUpdateParams`, which is a slow multi-block process. This matches the allowed High impact: *Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows … under normal network assumptions* — specifically, the inability of the authorized party to halt bridge/conversion flows in an emergency.

---

### Likelihood Explanation

The `TurnBridge` message is a live, registered gRPC endpoint reachable by any on-chain transaction. The permission bypass requires no special access — any address with gas can call it. The circuit-breaker failure is triggered the moment any authorized party attempts to use the intended emergency stop. Both conditions are trivially reachable.

---

### Recommendation

Implement `TurnBridge` fully:

1. Add the `CanTurnBridge` permission check (mirroring `UpdateTokenMapping`).
2. Unwrap the SDK context, read `msg.Enable`, and call `k.SetParams` to update `bridge_active`.

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    params := k.GetParams(ctx)
    params.EnableGravity = msg.Enable   // or bridge_active field
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

### Proof of Concept

1. Any address (no `CanTurnBridge` permission) submits `MsgTurnBridge{Sender: attacker, Enable: false}`.
2. The transaction is included in a block and returns `code = 0` (success).
3. Query `bridge_active` via `QueryParams` — it is unchanged; the bridge remains active.
4. Separately, the legitimate admin submits `MsgTurnBridge{Sender: admin, Enable: false}` during an emergency.
5. The transaction also returns `code = 0`, but `bridge_active` is still unchanged.
6. The Gravity bridge continues processing batches and relaying tokens despite the admin's belief that it has been halted. [1](#0-0) [2](#0-1)

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
