### Title
`max_l2_gas_amount` Admission Control Bypassed for Declare Transactions — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary
`StatelessTransactionValidator::validate_resource_bounds` contains an explicit no-op branch for `RpcTransaction::Declare` that unconditionally skips the `max_l2_gas_amount` upper-bound check. Any attacker can submit a Declare transaction with `l2_gas.max_amount = u64::MAX` and it will be admitted by the gateway regardless of the configured `max_l2_gas_amount` limit.

### Finding Description

In `validate_resource_bounds`, the check for whether `l2_gas.max_amount` exceeds the configured ceiling is structured as:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { ... });
}
``` [1](#0-0) 

The `if let RpcTransaction::Declare(_) = tx { }` arm is an empty block — it matches Declare transactions and does nothing, falling through to `Ok(())`. The `max_l2_gas_amount` check in the `else if` branch is therefore **never reached** for Declare transactions.

This is confirmed by the dedicated test `valid_l2_gas_amount_on_declare`, which asserts `Ok(())` for a Declare transaction with `l2_gas.max_amount = 200` when `max_l2_gas_amount = 100`: [2](#0-1) 

The counterpart test `test_invalid_max_l2_gas_amount` explicitly excludes `TransactionType::Declare` from its `#[values(...)]` list, confirming the bypass is known: [3](#0-2) 

The production `max_l2_gas_amount` is configured at `1,210,000,000`: [4](#0-3) 

### Impact Explanation

The blockifier's `check_fee_bounds` in `perform_pre_validation_stage` only checks that `max_amount >= minimal_required_amount` (a lower-bound check), not that `max_amount` is below any ceiling: [5](#0-4) 

There is no downstream enforcement of the gateway's `max_l2_gas_amount` policy. A Declare transaction with `l2_gas.max_amount = u64::MAX` will:
1. Pass stateless validation (the no-op branch)
2. Pass blockifier pre-validation (u64::MAX ≥ any minimal amount)
3. Be admitted to the mempool and sequenced

The concrete corrupted admission value is: a Declare transaction with `l2_gas.max_amount` arbitrarily exceeding `max_l2_gas_amount` is admitted when it must be rejected. This satisfies the **High** impact criterion: *"Mempool/gateway/RPC admission accepts invalid transactions before sequencing."*

### Likelihood Explanation

Trivially exploitable. Any unprivileged user submitting an `RpcDeclareTransactionV3` with `l2_gas.max_amount = GasAmount(u64::MAX)` will bypass the check with no special knowledge or access required.

### Recommendation

Remove the no-op Declare branch and apply the same `max_l2_gas_amount` check uniformly across all transaction types:

```rust
// Remove the Declare exemption entirely:
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If Declare transactions legitimately require a higher L2 gas ceiling (e.g., for large class compilation), introduce a separate `max_l2_gas_amount_declare` config field rather than removing the check entirely.

### Proof of Concept

```rust
#[test]
fn declare_bypasses_max_l2_gas_amount() {
    use apollo_gateway::stateless_transaction_validator::StatelessTransactionValidator;
    use apollo_gateway_config::config::StatelessTransactionValidatorConfig;
    use starknet_api::transaction::fields::{GasAmount, GasPrice, ResourceBounds, AllResourceBounds};

    let config = StatelessTransactionValidatorConfig {
        validate_resource_bounds: true,
        max_l2_gas_amount: 1,  // extremely tight limit
        min_gas_price: 1,
        ..Default::default()
    };
    let validator = StatelessTransactionValidator { config };

    // Build a Declare tx with l2_gas.max_amount = u64::MAX
    let tx = rpc_tx_for_testing(TransactionType::Declare, RpcTransactionArgs {
        resource_bounds: AllResourceBounds {
            l2_gas: ResourceBounds {
                max_amount: GasAmount(u64::MAX),
                max_price_per_unit: GasPrice(1),
            },
            ..Default::default()
        },
        ..Default::default()
    });

    // Asserts Ok(()) — the limit is not enforced
    assert_matches!(validator.validate(&tx), Ok(()));
}
```
