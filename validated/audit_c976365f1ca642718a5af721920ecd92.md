### Title
Unprivileged Caller Can Permanently Suppress Block Rewards for All Stakers via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract carries no access control and exposes a caller-controlled `disable_rewards: bool` parameter. Because `last_reward_block` is a single global storage slot and the function enforces a strict "one call per block" invariant, any unprivileged address can call `update_rewards(any_staker, disable_rewards: true)` to consume the per-block slot without distributing rewards, permanently destroying the block's yield for every staker.

---

### Finding Description

`update_rewards` is declared in `IStakingRewardsManager` and implemented without any role guard:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);   // slot consumed here
    if disable_rewards || self.is_pre_consensus() {
        return;                                            // rewards never calculated
    }
    // ... calculate_block_rewards + _update_rewards ...
}
``` [1](#0-0) 

`general_prerequisites` enforces only two things — the contract is not paused and the caller is non-zero: [2](#0-1) 

`last_reward_block` is a single global field (not per-staker): [3](#0-2) 

The per-block guard is: [4](#0-3) 

Once the slot is consumed with `disable_rewards: true`, the `calculate_block_rewards` path is never reached for that block number, so the STRK/BTC block rewards are never computed or forwarded to stakers or pools. [5](#0-4) 

---

### Impact Explanation

Every block for which an attacker fires `update_rewards(any_active_staker, disable_rewards: true)` first, the entire block's reward allocation is permanently destroyed for all stakers and all delegation pools. The rewards are never claimed from the reward supplier and never credited to `unclaimed_rewards_own` or forwarded via `update_rewards_from_staking_contract` to pool contracts. This constitutes **permanent freezing / destruction of unclaimed yield** for every participant in the protocol for each attacked block.

Matched allowed impact: **High — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

- No special role, stake, or prior state is required.
- The attacker only needs a non-zero address and gas.
- The attack can be repeated every block at negligible cost.
- On Starknet, where the sequencer orders transactions, a determined attacker can simply submit the griefing call at the start of every block before any legitimate proposer call lands.

---

### Recommendation

Restrict `update_rewards` to a designated privileged caller (e.g., the consensus/block-proposer role, or the attestation contract). Add a role check analogous to the existing `only_security_agent` / `only_app_governor` guards used elsewhere:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_rewards_updater();   // new role, or reuse an existing operator role
    // ...
}
```

Alternatively, remove the `disable_rewards` parameter entirely and derive the "skip rewards" condition solely from on-chain state (`is_pre_consensus()`), which is already checked inside the function.

---

### Proof of Concept

```
// Attacker script — repeat every block
loop {
    staking_contract.update_rewards(
        staker_address = <any valid staker>,
        disable_rewards = true,
    );
    // last_reward_block is now == current_block_number
    // Any subsequent legitimate update_rewards call in this block reverts with
    // REWARDS_ALREADY_UPDATED
    // All stakers earn zero rewards for this block
    wait_for_next_block();
}
```

1. Attacker calls `update_rewards(valid_staker, true)` — passes `general_prerequisites`, writes `last_reward_block = current_block`, returns early.
2. Legitimate consensus call `update_rewards(staker, false)` hits `assert!(current_block_number > last_reward_block)` → reverts.
3. `calculate_block_rewards` is never reached; no STRK/BTC rewards are minted or distributed for this block.
4. Repeated every block → all stakers and delegators receive zero rewards indefinitely.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
