### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards via Early-Return Accounting Skip - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in `staking.cairo` writes `last_reward_block` to the current block number **before** an early-return guard that skips actual reward distribution when `disable_rewards=true`. Because no access control enforces the "Only starkware sequencer" requirement stated in the spec, any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards=true)` once per block, consuming the per-block reward slot for all stakers without distributing any rewards.

### Finding Description

In `src/staking/staking.cairo`, `update_rewards` performs the following sequence:

1. Validates the staker exists and is active.
2. **Writes `last_reward_block` to the current block number** (line 1485).
3. Immediately checks `if disable_rewards || self.is_pre_consensus() { return; }` (line 1487–1488), returning without calling `_update_rewards`. [1](#0-0) 

`last_reward_block` is a **global** (not per-staker) storage variable. The guard at line 1454–1458 rejects any subsequent call in the same block with `REWARDS_ALREADY_UPDATED`: [2](#0-1) 

The spec declares access control as "Only starkware sequencer": [3](#0-2) 

However, the implementation contains **no sequencer check**. `general_prerequisites()` only enforces the contract-is-paused invariant; a `grep` for any sequencer assertion in `staking.cairo` returns zero matches. The function is therefore callable by any address. [4](#0-3) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker, disable_rewards=true)` once per block. This:

- Sets `last_reward_block = current_block`, consuming the slot.
- Returns early, skipping `_update_rewards` and therefore skipping `update_unclaimed_rewards_from_staking_contract` and the transfer of block rewards to stakers and pools. [5](#0-4) 

The sequencer's legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`. The rewards for that block are **permanently lost** — they are never credited to `unclaimed_rewards_own` and never transferred to delegation pools. Repeated across every block, this constitutes a complete, sustained suppression of all staker and delegator yield.

Impact: **High — Theft/permanent freezing of unclaimed yield** for all stakers and pool members.

### Likelihood Explanation

The entry point is fully public (no role check, no signature, no privileged key). Any address can call it. The cost is one Starknet transaction per block. The attack is straightforward to automate and requires no special knowledge beyond knowing a valid staker address (which is public on-chain). Likelihood: **High**.

### Recommendation

1. **Enforce access control**: Add an assertion that `get_caller_address() == sequencer_address` (or an equivalent role check) at the top of `update_rewards`, consistent with the spec.
2. **Move `last_reward_block` write after the early-return guard**: Only mark the block as processed when rewards are actually distributed (or when the call is legitimately authorized), so that a no-op early return does not consume the slot.

```cairo
// Corrected ordering:
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... proceed with _update_rewards
```

### Proof of Concept

1. Staker `S` is active with non-zero STRK balance; consensus rewards are live.
2. Attacker `A` (any address) calls `staking.update_rewards(S, disable_rewards=true)` at block `N`.
3. `last_reward_block` is set to `N`; function returns early — no rewards distributed.
4. Sequencer calls `staking.update_rewards(S, disable_rewards=false)` at block `N` → reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker `S` and all pool members earn zero rewards for block `N`.
6. Attacker repeats at block `N+1`, `N+2`, … — all staker and delegator yield is permanently suppressed. [6](#0-5)
