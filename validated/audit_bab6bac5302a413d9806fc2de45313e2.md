### Summary

Because `ReconstructionLowerBound` is a plain newtype wrapper with no enforced minimum: [5](#0-4) 

â€¦a value of `1` passes all validation in `assert_sign_inputs` and `presign`, and is forwarded into the FROST signing protocol. FROST uses Lagrange interpolation over the supplied threshold, so `threshold = 1` means a single participant's share is sufficient to reconstruct the aggregate signature scalar â€” collapsing the multi-party scheme to a single-party operation.

---

### Impact Explanation

**Critical â€” Unauthorized creation of a valid threshold signature.**

With `threshold = 1`, a single participant (or a colluding coordinator) can unilaterally produce a cryptographically valid FROST/EdDSA or RedJubjub threshold signature over any attacker-chosen message, without the cooperation of any other honest party. The entire security assumption of the threshold scheme â€” that at least `t` parties must cooperate â€” is voided.

---

### Likelihood Explanation

The `threshold` parameter is fully attacker-controlled at the call site. Any participant acting as coordinator, or any library caller who constructs `PresignArguments`, can set `threshold = 1`. No special privilege, leaked key, or cryptographic break is required. The missing check is a single missing `if threshold < 2` guard, identical in structure to the guard already present in `assert_key_invariants`.

---

### Recommendation

Add the same lower-bound check present in `assert_key_invariants` to both `assert_sign_inputs` and `presign` in `src/frost/mod.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Additionally, consider cross-validating the caller-supplied `threshold` against the threshold embedded in `keygen_out` to prevent mismatched threshold attacks.

---

### Proof of Concept

1. Run DKG with participants `[P1, P2, P3]` and `threshold = 2`. All three receive valid key shares.
2. Construct `PresignArguments` with `keygen_out = P1.keygen_out` and `threshold = 1`.
3. Call `presign(participants, P1, &args, rng)` â€” `assert_sign_inputs` / `presign` accept `threshold = 1` with no error.
4. Call the FROST signing function with `threshold = 1` and only `P1` as participant.
5. FROST performs Lagrange interpolation with a single share at `threshold = 1`, producing a valid aggregate signature.
6. The resulting signature verifies correctly against the group public key, despite only one party participating â€” the threshold guarantee is broken. [6](#0-5) [7](#0-6)

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
