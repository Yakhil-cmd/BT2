### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Deny Block Rewards to All Stakers - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the Staking contract uses a **global** `last_reward_block` variable as its deduplication guard, but imposes no access control. Any unprivileged caller can invoke `update_rewards(any_active_staker, disable_rewards=true)` in a given block, consuming the global slot and writing `last_reward_block` before any rewards are distributed. The sequencer's subsequent call for the same block then reverts with `REWARDS_ALREADY_UPDATED`, and the block's rewards are permanently lost for all stakers.

### Finding Description

`update_rewards` is documented as callable only by the Starknet sequencer, but the implementation enforces no such restriction. `general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

The deduplication guard in `update_rewards` is keyed on a single global `last_reward_block` field, not on a per-staker basis:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

Critically, `last_reward_block` is written **before** the reward distribution logic executes, and the function returns early when `disable_rewards=true`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← global slot consumed

if disable_rewards || self.is_pre_consensus() {
    return;   // ← no rewards distributed
}
``` [3](#0-2) 

The storage field is declared as a single scalar, confirming it is global across all stakers:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [4](#0-3) 

The spec explicitly states the intended access control is "Only starkware sequencer", but this is never enforced in code: [5](#0-4) 

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker, disable_rewards=true)` in block N:

1. Passes the guard (`N > last_reward_block`)
2. Writes `last_reward_block = N`
3. Returns immediately — zero rewards distributed

The sequencer's legitimate call for block N then fails with `REWARDS_ALREADY_UPDATED`. The block rewards for that slot are **permanently lost** — they are never minted or credited to any staker. Because block rewards are computed per-epoch and each block represents a discrete reward unit, every griefed block results in irreversible loss of yield for all active stakers and their delegators.

An attacker who is themselves an active staker can additionally call with `disable_rewards=false` to collect their own rewards while blocking the sequencer from distributing rewards to all other stakers in that block.

**Impact: High — Permanent freezing/loss of unclaimed yield for all stakers.**

### Likelihood Explanation

- Entry point is fully public; any non-zero address can call it.
- The attacker only needs to supply any currently-active staker address (trivially discoverable on-chain via `NewStaker` events).
- Cost is a single Starknet transaction fee per block griefed.
- The attack can be sustained indefinitely, denying all consensus-era block rewards.

**Likelihood: High.**

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` restricts callers to the attestation contract:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_sequencer();   // enforce spec-mandated access control
    // ...
}
```

Alternatively, if a sequencer role is not yet defined, consider making `last_reward_block` a per-staker mapping so that a call for one staker cannot block reward distribution for others.

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`).
2. Attacker observes block N is about to be produced.
3. Attacker submits: `update_rewards(alice_staker_address, disable_rewards=true)`.
4. Transaction succeeds: `last_reward_block` is set to N, no rewards distributed.
5. Sequencer submits: `update_rewards(bob_staker_address, disable_rewards=false)`.
6. Transaction reverts: `REWARDS_ALREADY_UPDATED` — Bob's block-N rewards are permanently lost.
7. Attacker repeats every block; all stakers lose all consensus block rewards indefinitely. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
