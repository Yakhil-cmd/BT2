### Title
Unprivileged Overwrite of Any Account's E2EE Encryption Key via `RegisterEncryptionKey` - (File: x/e2ee/keeper/keeper.go)

### Summary
`RegisterEncryptionKey` in the `x/e2ee` keeper performs a blind `store.Set` that both registers a new key and silently overwrites an existing one, with no check that `req.Address` matches the transaction signer. Any unprivileged account can overwrite any other account's registered encryption key, corrupting e2ee key state with direct security impact.

### Finding Description
The `registerEncryptionKey` internal function unconditionally writes to the KV store:

```go
// x/e2ee/keeper/keeper.go:32-43
func (k Keeper) registerEncryptionKey(
    ctx context.Context,
    address string,
    key []byte,
) error {
    bz, err := k.addressCodec.StringToBytes(address)
    if err != nil {
        return err
    }
    sdk.UnwrapSDKContext(ctx).KVStore(k.storeKey).Set(types.KeyPrefix(bz), key)
    return nil
}
```

The public message handler calls it directly with `req.Address` — the address field supplied by the caller — without verifying that the signer of the transaction is the owner of that address:

```go
// x/e2ee/keeper/keeper.go:45-53
func (k Keeper) RegisterEncryptionKey(
    ctx context.Context,
    req *types.MsgRegisterEncryptionKey,
) (*types.MsgRegisterEncryptionKeyResponse, error) {
    if err := k.registerEncryptionKey(ctx, req.Address, []byte(req.Key)); err != nil {
        return nil, err
    }
    return &types.MsgRegisterEncryptionKeyResponse{}, nil
}
```

There is no guard of the form `if signer != req.Address { return error }`, and no existence check that would distinguish "first-time registration" from "overwrite of an existing key." The function is a single upsert path reachable by any account. [1](#0-0) [2](#0-1) 

### Impact Explanation
An attacker submits `MsgRegisterEncryptionKey{Address: victim_address, Key: attacker_pubkey}`. The keeper overwrites the victim's stored encryption key with the attacker's public key. Any sender who subsequently queries the victim's registered key and encrypts a message with it will encrypt to the attacker's key instead. The attacker can decrypt all such messages. This is direct corruption of e2ee key/message state with security impact, matching the High impact category: *"Corruption of … e2ee key/message state with direct security impact."* [3](#0-2) 

### Likelihood Explanation
The entry point is a standard Cosmos SDK message (`MsgRegisterEncryptionKey`) reachable by any account with a small transaction fee. No privilege, leaked key, or special role is required. The attacker only needs to know the victim's bech32 address, which is public on-chain. [2](#0-1) 

### Recommendation
1. **Separate registration from update.** Add a `RegisterEncryptionKey` path that checks the key slot is empty (returns an error if a key already exists for `req.Address`), and a separate `UpdateEncryptionKey` path that requires the key slot to be non-empty.
2. **Enforce signer == address.** In both paths, verify that the signer of the transaction equals `req.Address` before writing to the store.

```go
// Proposed guard in RegisterEncryptionKey
existing := sdk.UnwrapSDKContext(ctx).KVStore(k.storeKey).Get(types.KeyPrefix(bz))
if len(existing) != 0 {
    return errors.Wrap(sdkerrors.ErrUnauthorized, "key already registered; use UpdateEncryptionKey")
}
// signer check
if signer != req.Address {
    return errors.Wrap(sdkerrors.ErrUnauthorized, "signer must match address")
}
``` [4](#0-3) 

### Proof of Concept

1. Alice has a registered encryption key `alice_pubkey` stored at `KeyPrefix(alice_bytes)`.
2. Attacker (any account) broadcasts:
   ```
   MsgRegisterEncryptionKey{
       Address: alice_bech32,
       Key:     attacker_pubkey,
   }
   ```
3. `RegisterEncryptionKey` calls `registerEncryptionKey(ctx, alice_bech32, attacker_pubkey)`.
4. `store.Set(KeyPrefix(alice_bytes), attacker_pubkey)` executes unconditionally — no signer check, no existence check.
5. Alice's key slot now holds `attacker_pubkey`.
6. Any party querying Alice's key via `Key(ctx, &KeyRequest{Address: alice_bech32})` receives `attacker_pubkey`.
7. Messages encrypted to "Alice" are now decryptable only by the attacker. [5](#0-4) [1](#0-0)

	if err != nil {
		return nil, err
	}
	sdkCtx := sdk.UnwrapSDKContext(ctx)
	value := sdkCtx.KVStore(k.storeKey).Get(types.KeyPrefix(bz))
	return &types.KeyResponse{Key: string(value)}, nil
