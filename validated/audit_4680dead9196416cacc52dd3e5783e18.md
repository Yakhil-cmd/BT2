### Title
Unauthorized Global Contract Overwrite via Missing `predecessor_id == receiver_id` Guard on `DeployGlobalContract(AccountId)` — (File: `runtime/runtime/src/global_contracts.rs`)

---

### Summary

Any unprivileged account can deploy an arbitrary WASM binary as the global contract stored under **any victim account's** `AccountId`-mode identifier. The runtime charges the **victim's** balance for storage and propagates the attacker's code to every shard. Every account that has called `UseGlobalContract(AccountId(victim))` will subsequently execute the attacker's code. No special privilege is required; the only cost to the attacker is gas.

---

### Finding Description

`DeployGlobalContract` is **not** listed among the actions that require `predecessor_id == receiver_id`:

> For the following actions, `predecessor_id` and `receiver_id` are required to be equal: `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, `DeleteAccount`. [1](#0-0) 

`validate_deploy_global_contract_action` enforces only a code-size limit — it performs no ownership check: [2](#0-1) 

`action_deploy_global_contract` receives `account_id` (the receipt's `receiver_id`) and passes it directly to `initiate_distribution` without verifying that `predecessor_id == receiver_id`: [3](#0-2) 

Inside `initiate_distribution`, when `deploy_mode == AccountId`, the identifier is set to `GlobalContractIdentifier::AccountId(account_id.clone())` — i.e., the **receiver's** account ID, not the signer's: [4](#0-3) 

The storage cost is then deducted from the **receiver's** (victim's) account balance: [5](#0-4) 

A `GlobalContractDistributionReceipt` is emitted and propagated shard-by-shard, writing the attacker's code to `TrieKey::GlobalContractCode { identifier: AccountId(victim) }` on every shard: [6](#0-5) 

The design intent of `AccountId` mode is explicit — "This allows the **owner** to update the contract for all its users" — but the code enforces no such ownership invariant: [7](#0-6) 

---

### Impact Explanation

**Corrupted value**: `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim.near) }` is overwritten with the attacker's arbitrary WASM on every shard.

**Affected accounts**: Every account that has executed `UseGlobalContract(AccountId(victim.near))` stores `AccountContract::GlobalByAccount(victim.near)` and will execute the attacker's code on the next function call: [8](#0-7) 

**Balance drain**: The victim's account balance is reduced by `global_contract_storage_amount_per_byte × code_len` without the victim's consent.

**Persistence**: The nonce mechanism only prevents *stale* overwrites (older nonce < stored nonce). The attacker can always increment the nonce by deploying again, continuously racing against the victim's redeployments: [9](#0-8) 

Scope match: **global contract/code selection** and **authorization**.

---

### Likelihood Explanation

- Any account with a non-zero balance can perform the attack; no validator, operator, or trusted-service privilege is required.
- The attacker pays only gas; storage cost is charged to the victim.
- The attack is replayable at any block — not limited to a registration window.
- The victim's only recourse is to redeploy (paying their own storage cost again), creating a continuous griefing loop.

---

### Recommendation

Add a guard in `action_deploy_global_contract` (or in the receipt-processing dispatch in `runtime/runtime/src/lib.rs`) that rejects `DeployGlobalContract` with `deploy_mode == AccountId` when `predecessor_id != receiver_id`, mirroring the existing ownership requirement for `DeployContract`. Alternatively, enforce this at the action-validation layer in `validate_deploy_global_contract_action` by threading the `predecessor_id` through and checking equality when the mode is `AccountId`.

---

### Proof of Concept

```
// Setup: Alice has deployed her global contract and users rely on it.
tx1: signer=alice.near, receiver=alice.near,
     actions=[DeployGlobalContract(alice_code, AccountId)]
// → TrieKey::GlobalContractCode{AccountId(alice.near)} = alice_code, nonce=1

tx2: signer=user.near, receiver=user.near,
     actions=[UseGlobalContract(AccountId(alice.near))]
// → user.near.contract = GlobalByAccount(alice.near)

// Attack: Bob sends a transaction TO alice.near's account.
tx3: signer=bob.near, receiver=alice.near,
     actions=[DeployGlobalContract(malicious_code, AccountId)]

// Runtime applies tx3:
// 1. action_deploy_global_contract(account_id=alice.near, ...)
// 2. alice.near balance -= storage_cost  (victim pays)
// 3. initiate_distribution(account_id=alice.near, malicious_code, AccountId, ...)
//    → id = GlobalContractIdentifier::AccountId(alice.near)
//    → nonce incremented to 2
//    → GlobalContractDistributionReceipt emitted
// 4. Receipt propagates to all shards:
//    TrieKey::GlobalContractCode{AccountId(alice.near)} = malicious_code

// Consequence: user.near's next function call executes malicious_code.
// Alice must redeploy (nonce=3) to recover, but bob can immediately
// overwrite again (nonce=4), creating an unbounded griefing loop.
```

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}
