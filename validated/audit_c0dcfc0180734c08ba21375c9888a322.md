### Title
Push-based token transfer in `exit_delegation_pool_action` is incompatible with tokens that implement a block list — (File: src/pool/pool.cairo)

---

### Summary

The `exit_delegation_pool_action` function in the delegation pool contract unconditionally pushes the pool token directly to the `pool_member` address. If the pool token (e.g., a wrapped BTC token with an admin-controlled block list) has `pool_member` on its block list, every call to `exit_delegation_pool_action` for that member will revert. Because there is no alternative withdrawal path and the `pool_member` address is immutable, the delegator's principal is permanently frozen inside the pool contract.

---

### Finding Description

The Starknet Staking protocol supports multi-token delegation pools, including BTC-class tokens added via `IStakingTokenManager`. Wrapped BTC tokens (e.g., WBTC-style tokens on Starknet) commonly implement an admin-controlled address block list where all transfers to a blocked address revert.

In `exit_delegation_pool_action`, after the intent window has elapsed, the function performs a push-based transfer of the pool token directly to `pool_member`:

```cairo
// src/pool/pool.cairo  line 328-330
// Transfer delegated amount to the pool member.
let token_dispatcher = self.token_dispatcher.read();
token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
``` [1](#0-0) 

`pool_member` is the delegator's wallet address, set at `enter_delegation_pool` time and never changeable. There is no access control on `exit_delegation_pool_action`; any caller can invoke it for any `pool_member`:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
``` [2](#0-1) 

If the pool token's block list includes `pool_member`, the `checked_transfer` call reverts unconditionally. Because the pool member's address is fixed and there is no pull-based fallback, the delegated principal is permanently locked in the pool contract.

The same structural issue exists in `unstake_action` in `staking.cairo`, which pushes STRK directly to `staker_address` (also immutable) and pushes STRK rewards to `reward_address` (changeable) in a single atomic transaction:

```cairo
// src/staking/staking.cairo  line 504-506
// Return stake to staker.
token_dispatcher
    .checked_transfer(recipient: staker_address, amount: staker_amount.into());
``` [3](#0-2) 

If `staker_address` is on the STRK block list, `unstake_action` always reverts, permanently freezing the staker's own stake. The staker has no mechanism to redirect the principal to a different address.

---

### Impact Explanation

**High — Permanent freezing of delegated funds / staker principal.**

A pool member whose address is on the pool token's block list cannot complete `exit_delegation_pool_action`. Their `unpool_amount` is recorded in storage but can never be transferred out. Because `pool_member` is immutable and no pull-based escape hatch exists, the funds are permanently frozen inside the pool contract. The same applies to a staker whose `staker_address` is on the STRK block list: `unstake_action` will always revert, trapping the staker's principal indefinitely. [4](#0-3) 

---

### Likelihood Explanation

**Low.** Two conditions must hold simultaneously:

1. The pool token (STRK or a BTC-class token) must implement an admin-controlled address block list.
2. The `pool_member` (or `staker_address`) must appear on that block list at the time `exit_delegation_pool_action` (or `unstake_action`) is called.

BTC-class tokens added via `add_token` / `enable_token` are the most realistic vector, as wrapped BTC tokens commonly carry block list functionality. STRK itself may also gain such a feature in future upgrades. [5](#0-4) 

---

### Recommendation

Apply the **Pull over Push** pattern for principal withdrawals:

1. In `exit_delegation_pool_action`, instead of calling `checked_transfer` immediately, record the owed amount in a per-member mapping (e.g., `pending_withdrawals: Map<ContractAddress, Amount>`).
2. Expose a separate `withdraw()` function that the pool member calls at their convenience to pull their own funds.
3. Apply the same pattern to `unstake_action` for the staker principal return.

This decouples the protocol's state transitions from the token's transfer restrictions, ensuring that a block-listed address cannot freeze the exit flow for other participants or for the protocol itself.

---

### Proof of Concept

1. A BTC-class token with an admin block list is added and enabled via `add_token` / `enable_token`.
2. A delegator calls `enter_delegation_pool` on a pool backed by that BTC token, depositing `amount` tokens.
3. The delegator calls `exit_delegation_pool_intent(amount)`, starting the exit window.
4. The token admin adds the delegator's address to the block list.
5. After the exit window elapses, anyone calls `exit_delegation_pool_action(pool_member: delegator_address)`.
6. `token_dispatcher.checked_transfer(recipient: pool_member, ...)` reverts because `pool_member` is blocked.
7. The delegator's `unpool_amount` remains non-zero in storage; the funds are permanently frozen.
8. No alternative function exists to redirect or recover the principal. [6](#0-5)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
