### Title
P2P Sync Drops All Transaction Events from Receipts, Causing RPC to Return Authoritative-Looking Empty Events - (File: crates/apollo_protobuf/src/converters/receipt.rs)

### Summary

Every `TryFrom<protobuf::receipt::*>` implementation in the protobuf receipt converter unconditionally sets `events = vec![]`, and the corresponding `From<*TransactionOutput>` serializers never write events into the protobuf wire format. Because the P2P sync client has no separate event channel, any node that syncs blocks via P2P stores permanently empty event lists for every transaction. Subsequent RPC calls (`starknet_getTransactionReceipt`, `starknet_getEvents`) on that node return authoritative-looking but completely wrong results.

### Finding Description

**Root cause — serialization side (events silently dropped):**

All five `From<*TransactionOutput>` implementations call `create_proto_receipt_common_from_txn_output_fields`, which only encodes `actual_fee`, `messages_sent`, `execution_resources`, and `execution_status`. The `events` field of every `TransactionOutput` variant is never written into the protobuf message. [1](#0-0) [2](#0-1) 

**Root cause — deserialization side (events hardcoded to empty):**

All five `TryFrom<protobuf::receipt::*>` implementations explicitly set `let events = vec![]` before constructing the output struct. The comments even document this: `// The output will have an empty events vec`. [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

**No compensating event channel in the P2P sync client:**

The P2P sync server exposes a dedicated `EventReceiver` channel and serves `(Event, TransactionHash)` tuples by reading `transaction_output.events()` from storage. [7](#0-6) [8](#0-7) 

However, `P2pSyncClientChannels` has no corresponding event sender, and the production `new_p2p_state_sync_client` never registers an event protocol client: [9](#0-8) [10](#0-9) 

**Storage writes empty events:**

When the P2P sync client writes a `FullTransaction` to storage, `write_events` is called with the deserialized `transaction_output`, whose `events()` slice is always empty: [11](#0-10) 

**Confirmed by a suppressed test assertion:**

The transaction sync test explicitly comments out the assertion that verifies stored `transaction_outputs` match the sent ones, with a TODO acknowledging the broken protobuf conversion: [12](#0-11) 

### Impact Explanation

Any node that syncs blocks via the P2P protocol stores permanently empty event lists for every transaction in every block. RPC endpoints `starknet_getTransactionReceipt` and `starknet_getEvents` on such a node return authoritative-looking responses with zero events, even for transactions that emitted many events. The block header's `event_commitment` (synced separately via the header protocol) is non-zero and correct, but the stored events are empty — a silent, undetected mismatch.

This matches: **High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

Any production node configured to use P2P sync (the default for distributed deployments) is affected for every block it syncs. No attacker action is required; the corruption is structural and automatic. The trigger is unprivileged: any user querying the RPC of a P2P-synced node receives wrong data.

### Recommendation

1. Add an event field to the protobuf `Receipt` common message and populate it in all `From<*TransactionOutput>` serializers.
2. Add an event sender channel to `P2pSyncClientChannels` and register the event protocol client in `new_p2p_state_sync_client`, mirroring the server-side `EventReceiver`.
3. Alternatively, reconstruct events on the client side from the separately-synced event stream and merge them into the stored `TransactionOutput` before writing to storage.
4. Uncomment and fix the suppressed assertion in `transaction_test.rs` to prevent regression.

### Proof of Concept

1. Start two nodes: node A as sequencer (produces blocks with events), node B configured for P2P sync from node A.
2. Submit an Invoke transaction on node A that emits events (e.g., an ERC-20 transfer emitting a `Transfer` event).
3. Wait for node B to sync the block via P2P.
4. Call `starknet_getTransactionReceipt` on node B for that transaction hash.
5. **Observed**: `"events": []` — empty, despite the transaction having emitted events.
6. **Expected**: the full list of emitted events matching node A's response.
7. Call `starknet_getEvents` with the block range on node B: returns zero events.
8. Verify: node B's stored `event_commitment` in the block header is non-zero (correct), but `get_block_transaction_outputs` returns outputs with empty `events` vecs — confirming the structural mismatch introduced by the protobuf round-trip at lines 79, 127, 174, 202, and 230 of `crates/apollo_protobuf/src/converters/receipt.rs`.

fn create_proto_receipt_common_from_txn_output_fields(
    actual_fee: Fee,
    messages_sent: Vec<MessageToL1>,
    execution_resources: ExecutionResources,
    execution_status: TransactionExecutionStatus,
) -> protobuf::receipt::Common {
    let actual_fee = Felt::from(actual_fee).into();
    let messages_sent = messages_sent.into_iter().map(protobuf::MessageToL1::from).collect();
    let execution_resources = execution_resources.into();
    let revert_reason =
        if let TransactionExecutionStatus::Reverted(reverted_status) = execution_status {
            Some(reverted_status.revert_reason)
        } else {
            None
        };
    protobuf::receipt::Common {
        actual_fee: Some(actual_fee),
        price_unit: 0,
        messages_sent,
        execution_resources: Some(execution_resources),
        revert_reason,
    }
}

impl FetchBlockData for (Event, TransactionHash) {
    async fn fetch_block_data(
        block_number: BlockNumber,
        txn: &StorageTxn<'_, db::RO>,
        _class_manager_client: &mut SharedClassManagerClient,
    ) -> Result<Vec<Self>, P2pSyncServerError> {

                block_hash_or_number: BlockHashOrNumber::Number(block_number),

        let transaction_hashes = txn.get_block_transaction_hashes(block_number)?.ok_or(
            P2pSyncServerError::BlockNotFound {
                block_hash_or_number: BlockHashOrNumber::Number(block_number),
            },
        )?;

        let mut result = Vec::new();
        for (transaction_output, transaction_hash) in
            transaction_outputs.into_iter().zip(transaction_hashes)
        {
            for event in transaction_output.events() {
                result.push((event.clone(), transaction_hash));
            }
        }
        Ok(result)
    }

pub struct P2pSyncClientChannels {
    header_sender: HeaderSqmrSender,
    state_diff_sender: StateSqmrDiffSender,
    transaction_sender: TransactionSqmrSender,
    class_sender: ClassSqmrSender,
}

impl P2pSyncClientChannels {
    pub fn new(
        header_sender: HeaderSqmrSender,
        state_diff_sender: StateSqmrDiffSender,
        transaction_sender: TransactionSqmrSender,
        class_sender: ClassSqmrSender,
    ) -> Self {
        Self { header_sender, state_diff_sender, transaction_sender, class_sender }
    }
    pub(crate) fn create_stream(
        self,
        storage_reader: StorageReader,
        config: P2pSyncClientConfig,
        internal_blocks_receivers: InternalBlocksReceivers,
    ) -> impl Stream<Item = BlockDataResult> + Send + 'static {
        let header_stream = HeaderStreamBuilder::create_stream(
            self.header_sender,
            storage_reader.clone(),
            Some(internal_blocks_receivers.header_receiver),
            config.wait_period_for_new_data,
            config.wait_period_for_other_protocol,
            config.num_headers_per_query,
        );

        let state_diff_stream = StateDiffStreamBuilder::create_stream(
            self.state_diff_sender,
            storage_reader.clone(),
            Some(internal_blocks_receivers.state_diff_receiver),
            config.wait_period_for_new_data,
            config.wait_period_for_other_protocol,
            config.num_block_state_diffs_per_query,
        );

        let transaction_stream = TransactionStreamFactory::create_stream(
            self.transaction_sender,
            storage_reader.clone(),
            Some(internal_blocks_receivers.transaction_receiver),
            config.wait_period_for_new_data,
            config.wait_period_for_other_protocol,
            config.num_block_transactions_per_query,
        );

        let class_stream = ClassStreamBuilder::create_stream(
            self.class_sender,
            storage_reader.clone(),
            Some(internal_blocks_receivers.class_receiver),
            config.wait_period_for_new_data,
            config.wait_period_for_other_protocol,
            config.num_block_classes_per_query,
        );

        header_stream.merge(state_diff_stream).merge(transaction_stream).merge(class_stream)
    }
