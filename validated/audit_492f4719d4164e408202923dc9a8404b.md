### Title
Unprivileged Caller Can Permanently Suppress Block Reward Distribution via `update_rewards` - (File: `src/staking/staking.cairo`)

### Summary
`update_rewards` in `staking.cairo` is documented as callable only by the Starknet sequencer, but the implementation enforces no such restriction. Any non-zero address can call it with `disable_rewards: true`, which marks the current block as processed (`last_reward_block` updated) while distributing zero rewards. Because only one call per block is permitted, the legitimate sequencer call is then blocked for that block. An attacker repeating this every block permanently freezes all unclaimed yield for every staker and their delegators.

---

### Finding Description

The spec for `update_rewards` states:

> **Access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, only calls `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address — no sequencer identity check exists: [2](#0-1) 

The function then unconditionally writes `current_block_number` to `last_reward_block` **before** the `disable_rewards` guard: [3](#0-2) 

Because `last_reward_block` is already updated, any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

When `disable_rewards: true` is passed, the function returns immediately after updating `last_reward_block`, distributing nothing: [5](#0-4) 

The rewards that would have been distributed via `_update_rewards` → `update_pool_rewards` → `update_rewards_from_staking_contract` are permanently lost for that block: [6](#0-5) 

Pool members' `cumulative_rewards_trace` is never updated for the suppressed block, so `calculate_rewards` in the pool contract computes zero for that interval: [7](#0-6) 

---

### Impact Explanation

Every block whose rewards are suppressed is permanently unrecoverable. There is no catch-up mechanism: `last_reward_block` is a monotone write and the per-block reward amount is not buffered. Stakers lose `unclaimed_rewards_own` accrual and pool delegators lose their share of `pool_rewards` for every griefed block. Sustained griefing (one cheap transaction per block) results in **permanent freezing of all unclaimed yield** for every staker and delegator in the protocol.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

The entry point is fully public — any EOA or contract with a non-zero address can call `update_rewards`. The only cost is gas per block. No capital, no privileged role, no leaked key, and no external dependency is required. The attacker gains nothing financially but can inflict unbounded yield loss on all protocol participants. The attack is trivially scriptable and sustainable indefinitely.

---

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, consistent with the specification. For example, store the authorized sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation. Alternatively, update `last_reward_block` only after confirming rewards will actually be distributed (i.e., move the write past the `disable_rewards` guard), so that a suppressed call does not consume the block's reward slot.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker monitors the chain. At the start of every new block, attacker calls:
   ```
   staking.update_rewards(any_valid_staker_address, disable_rewards: true)
   ```
3. Inside `update_rewards`:
   - `general_prerequisites()` passes (contract not paused, caller non-zero). [8](#0-7) 
   - `current_block_number > last_reward_block` passes (new block). [4](#0-3) 
   - `last_reward_block` is written to `current_block_number`. [9](#0-8) 
   - `disable_rewards == true` → early return, zero rewards distributed. [5](#0-4) 
4. The legitimate sequencer attempts `update_rewards(staker, disable_rewards: false)` in the same block → reverts with `REWARDS_ALREADY_UPDATED`. [10](#0-9) 
5. No staker's `unclaimed_rewards_own` is incremented; no pool's `cumulative_rewards_trace` is updated; no STRK is transferred to any pool. All yield for that block is permanently lost.
6. Repeated every block → complete, permanent suppression of all protocol rewards.
