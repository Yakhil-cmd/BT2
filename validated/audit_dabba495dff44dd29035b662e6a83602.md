### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Wrong Mempool Admission Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check validates a transaction's `max_price_per_unit` against the **previous block's `l2_gas_price`** field, but the block being built uses **`next_l2_gas_price`** (also stored in the previous block's header) as its actual L2 gas price. This is the direct Sequencer analog of the Wildcat "stale timestamp" bug: a checkpoint value is used instead of the current value, producing wrong admission decisions. The code even carries an explicit TODO acknowledging the error.

---

### Finding Description

In `validate_resource_bounds`, the gateway reads the latest committed block's `BlockInfo` and extracts `gas_prices.strk_gas_prices.l2_gas_price`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:229-240
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `BlockInfo.gas_prices` exclusively from the block header's `l2_gas_price` field — the price that governed transactions **in that already-committed block**:

```rust
// crates/apollo_gateway/src/gateway_fixed_block_state_reader.rs:44,49
l2_gas_price: block_header.l2_gas_price.price_in_wei.try_into()?,
// ...
l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
``` [2](#0-1) 

The block header also carries a **distinct** `next_l2_gas_price` field — the price that will govern the **next** block being built:

```rust
// crates/starknet_api/src/block.rs:239
pub next_l2_gas_price: GasPrice,
``` [3](#0-2) 

`next_l2_gas_price` is computed by the consensus orchestrator's EIP-1559-style fee market and written into every committed block header:

```rust
// crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs:405
next_l2_gas_price: self.l2_gas_price,
``` [4](#0-3) 

When the batcher builds the next block it uses `self.l2_gas_price` — which equals `previous_block.next_l2_gas_price` — as the block context's L2 gas price. The blockifier's `check_fee_bounds` then enforces:

```
tx.max_price_per_unit >= block_context.gas_prices.l2_gas_price
                       = previous_block.next_l2_gas_price
``` [5](#0-4) 

But the gateway's admission check enforces only:

```
tx.max_price_per_unit >= min_gas_price_percentage% × previous_block.l2_gas_price
``` [6](#0-5) 

The two prices diverge every block because `calculate_next_base_gas_price` adjusts the price up or down based on gas usage relative to the target: [7](#0-6) 

---

### Impact Explanation

**When the L2 gas price is rising** (network congestion, gas usage above target — the common case):

`previous_block.next_l2_gas_price > previous_block.l2_gas_price`

A transaction with:
```
previous_block.l2_gas_price  ≤  tx.max_price_per_unit  <  previous_block.next_l2_gas_price
```
passes the gateway's `validate_resource_bounds` check, is admitted to the mempool, and is handed to the batcher. The batcher's blockifier then rejects it in `check_fee_bounds` with `MaxGasPriceTooLow` for L2Gas. The transaction was admitted but cannot execute — the gateway accepted an invalid transaction.

**When the L2 gas price is falling** (gas usage below target):

`previous_block.next_l2_gas_price < previous_block.l2_gas_price`

A transaction with:
```
previous_block.next_l2_gas_price  ≤  tx.max_price_per_unit  <  previous_block.l2_gas_price
```
is rejected by the gateway even though it would pass the batcher's fee check — the gateway rejected a valid transaction.

Both directions match the "High" impact scope: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

The `min_gas_price_percentage` config (default 100) does not mitigate this: the wrong reference price is used regardless of the percentage.

---

### Likelihood Explanation

- The L2 gas price changes every block via the EIP-1559-style fee market. Any period of non-zero network activity causes `next_l2_gas_price ≠ l2_gas_price`.
- Any user submitting a V3 (`AllResources`) transaction with `max_price_per_unit` set to exactly the current block's L2 gas price (the natural value a wallet would use) triggers the discrepancy during a rising-price period.
- The bug is self-documenting: the TODO comment `// TODO(Arni): getnext_l2_gas_price from the block header.` at line 229 explicitly acknowledges the wrong value is being used.
- No privilege is required; any unprivileged transaction submission suffices.

---

### Recommendation

In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, expose `next_l2_gas_price` from the block header (it is already present in `BlockHeaderWithoutHash`). Either:

1. Add a `next_l2_gas_price` field to the returned `BlockInfo` (or a wrapper), and use it in `validate_resource_bounds` instead of `gas_prices.strk_gas_prices.l2_gas_price`; or
2. Extend the `GatewayFixedBlockStateReader` trait with a `get_next_l2_gas_price()` method and call it directly in `validate_resource_bounds`.

The corrected check should be:

```rust
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()   // reads block_header.next_l2_gas_price
    .await?;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    next_block_l2_gas_price,
)?;
```

This mirrors the fix recommended for the Wildcat bug: use the value that applies at the time of actual processing, not the stale checkpoint value.

---

### Proof of Concept

1. Observe the latest committed block N with:
   - `block_N.l2_gas_price = 1000 fri`
   - `block_N.next_l2_gas_price = 1003 fri` (price rose ~0.3% due to above-target gas usage)

2. Submit a V3 invoke transaction with:
   ```
   l2_gas.max_price_per_unit = 1000 fri   // exactly matches l2_gas_price, not next_l2_gas_price
   ```

3. Gateway `validate_resource_bounds` computes threshold = `100% × 1000 = 1000`. Check passes (`1000 >= 1000`). Transaction is admitted to the mempool. [8](#0-7) 

4. Batcher builds block N+1 with `block_context.gas_prices.strk_gas_prices.l2_gas_price = 1003 fri` (= `block_N.next_l2_gas_price`).

5. Blockifier `check_fee_bounds` evaluates `1000 < 1003` → `ResourceBoundsError::MaxGasPriceTooLow { resource: L2Gas, max_gas_price: 1000, actual_gas_price: 1003 }`. Transaction fails at execution time. [9](#0-8) 

The gateway admitted a transaction that the blockifier correctly rejects, violating the admission invariant. The root cause is the use of `block_header.l2_gas_price` (stale) instead of `block_header.next_l2_gas_price` (the price for the block being built).

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
