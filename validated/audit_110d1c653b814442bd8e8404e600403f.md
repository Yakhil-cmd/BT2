### Title
Unprivileged Caller Can Permanently Deny All Stakers' Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the `Staking` contract accepts a caller-controlled `disable_rewards` boolean with no access control. Because the global `last_reward_block` state is written **before** the `disable_rewards` check, any unprivileged address can front-run every block's reward update to permanently deny consensus rewards to all stakers.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` is the entry point for distributing per-block consensus rewards. Its execution order is:

1. `general_prerequisites()` — only checks pause state and non-zero caller (no role check).
2. Assert `current_block_number > last_reward_block` (gate on double-spend).
3. Assert staker exists, is active, and has non-zero balance.
4. **Write `self.last_reward_block.write(current_block_number)`** — global state change.
5. `if disable_rewards || self.is_pre_consensus() { return; }` — early exit with no rewards.
6. Calculate and distribute block rewards via `_update_rewards`. [1](#0-0) 

The critical ordering flaw: the global `last_reward_block` is committed at step 4, **before** the `disable_rewards` guard at step 5. Because `general_prerequisites()` imposes no role restriction, any address can call:

```
update_rewards(any_valid_staker_address, disable_rewards: true)
```

This atomically:
- Consumes the current block (sets `last_reward_block = current_block`)
- Returns without distributing any rewards

All subsequent legitimate calls to `update_rewards` for that block — from the staker, operational address, or anyone else — revert with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

`last_reward_block` is a **single global storage slot**, not per-staker: [3](#0-2) 

One attacker call per block silences rewards for **every staker** in the protocol.

The `general_prerequisites` guard: [4](#0-3) 

---

### Impact Explanation

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` once per block permanently prevents all stakers from accumulating consensus rewards. Because `last_reward_block` is global, a single call per block is sufficient to deny rewards protocol-wide. This constitutes **permanent freezing of unclaimed yield** for all stakers — matching the allowed High impact category.

---

### Likelihood Explanation

The attack requires:
- No capital or privileged role
- No leaked keys
- One transaction per block targeting any valid staker address

It is trivially automatable by monitoring the chain and submitting the griefing call at the start of each block. The attacker bears only gas costs.

---

### Recommendation

Add authorization to `update_rewards`. Options:

1. Restrict the call to the staker's registered operational address (consistent with the attestation model).
2. Introduce a dedicated privileged role (e.g., `CONSENSUS_REPORTER`) that is the only address permitted to pass `disable_rewards: true`.
3. Remove the `disable_rewards` parameter from the public interface entirely and handle reward suppression through a separate access-controlled function.

The core fix mirrors the external report's recommendation: move the authorization check **before** the state-mutating write to `last_reward_block`, or gate the entire function on an authorized caller.

---

### Proof of Concept

```
// Setup: StakerA is registered, active, has non-zero balance, consensus rewards are live.

// Block N arrives.
// Attacker (any EOA) calls:
staking.update_rewards(staker_a_address, disable_rewards: true);
// → last_reward_block = N
// → no rewards distributed (early return)

// StakerA's legitimate call in the same block:
staking.update_rewards(staker_a_address, disable_rewards: false);
// → PANICS: REWARDS_ALREADY_UPDATED (current_block_number == last_reward_block)

// Attacker repeats every block.
// Result: StakerA (and all other stakers) accumulate zero consensus rewards indefinitely.
``` [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
