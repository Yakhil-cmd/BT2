### Title
Non-Participant Message Injection Aborts DKG/Reshare/Sign via Unguarded `participants.index(from)?` in `reliable_broadcast_receive_all` — (File: src/protocol/echo_broadcast.rs)

---

### Summary

The `reliable_broadcast_receive_all` function in `src/protocol/echo_broadcast.rs` calls `participants.index(from)?` when processing a `MessageType::Send` message without first verifying that `from` is a member of the participant list. If `from` is a non-participant, `participants.index(from)` returns an error, and the `?` operator propagates it out of the loop and out of the function, aborting the entire DKG, reshare, or sign protocol for the honest party. The public `Protocol::message()` API accepts messages from any `Participant` without filtering, and the library does not document that callers must authenticate senders before delivery. This creates a reachable, attacker-controlled path to permanent denial of key generation and signing.
