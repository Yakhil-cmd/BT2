### Title
Missing Caller Restriction on `update_rewards` Allows Any User to Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `src/staking/staking.cairo` is documented in the spec as callable by "Only starkware sequencer," but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block `last_reward_block` slot without distributing rewards, permanently denying a staker their yield for that block.

### Finding Description

The spec at `docs/spec.md` line 1644–1645 states:

> **access control:** Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 is:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // consumes the block slot

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits without distributing
    }
    // ... distribute rewards
}
``` [1](#0-0) 

There is no `assert_caller_is_sequencer()` or equivalent check. `general_prerequisites()` only enforces the pause flag. [2](#0-1) 

`last_reward_block` is a single global storage variable (not per-staker). Once written for block N, the check `current_block_number > self.last_reward_block.read()` causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The spec confirms this is the intended per-block gate: [4](#0-3) 

### Impact Explanation

An attacker calls `update_rewards(victim_staker, disable_rewards: true)` at block N before the sequencer does. This:

1. Passes all validation (staker exists, has balance, block is new).
2. Writes `last_reward_block = N`.
3. Returns immediately without distributing any rewards (because `disable_rewards == true`).

When the sequencer subsequently calls `update_rewards(victim_staker, disable_rewards: false)` at block N, it reverts with `REWARDS_ALREADY_UPDATED`. The staker's `unclaimed_rewards_own` is never incremented for that block. The yield for block N is permanently lost — it is never minted or credited.

This maps to **High: Theft of unclaimed yield / Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The function is publicly callable with no role check.
- The attacker only needs to submit a transaction before the sequencer in each target block — a standard front-run on Starknet L2.
- No capital, no special role, and no setup is required.
- The attack can be repeated every block to continuously deny rewards to one or more stakers.

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract()`: [5](#0-4) 

Concretely, store the authorized sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` as the first check in `update_rewards`.

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. Staker Alice has staked and has a non-zero balance.
3. At block N, attacker Eve calls:
   ```
   staking.update_rewards(alice_address, disable_rewards: true)
   ```
4. `last_reward_block` is written to N; no rewards are distributed.
5. The sequencer calls:
   ```
   staking.update_rewards(alice_address, disable_rewards: false)
   ```
   → reverts with `REWARDS_ALREADY_UPDATED`.
6. Alice's `unclaimed_rewards_own` is unchanged. The block N yield is permanently lost.
7. Eve repeats step 3 every block to continuously freeze Alice's yield. [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
