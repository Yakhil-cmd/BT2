### Title
Gateway Stateful Validator Checks Only L2 Gas Price Threshold, Admitting Transactions With Zero L1/L1-Data Gas Price That Will Fail at Execution - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds validator enforces a minimum price threshold only for `l2_gas.max_price_per_unit`, while applying no analogous threshold check to `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. A transaction with `l1_gas.max_price_per_unit = 0` passes every gateway admission check and enters the mempool, but is deterministically rejected by the blockifier's `check_fee_bounds` at execution time. The code itself acknowledges the gap with a `TODO` comment.

### Finding Description

**Stateless validator** (`validate_resource_bounds`):

```rust
// Only L2 gas price is checked against a static floor.
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { … });
}
// No analogous check for l1_gas.max_price_per_unit or l1_data_gas.max_price_per_unit.
``` [1](#0-0) 

**Stateful validator** (`validate_tx_l2_gas_price_within_threshold`):

```rust
// TODO(Arni): Consider running this validation for all gas prices.

    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // threshold computed from previous block L2 gas price only
            if tx_l2_gas_price.0 < threshold { return Err(…); }
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
}
``` [2](#0-1) 

The stateful validator fetches only `strk_gas_prices.l2_gas_price` from the previous block and compares it against `l2_gas.max_price_per_unit`. Neither `l1_gas_price` nor `l1_data_gas_price` from the previous block is fetched or compared. [3](#0-2) 

**Blockifier pre-validation** (`check_fee_bounds`) — the check that *does* cover all three resources — runs only after the transaction has already been admitted to the mempool:

```rust
// AllResources path checks L1Gas, L1DataGas, AND L2Gas prices.
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { … });
}
``` [4](#0-3) 

**Attack path:**

1. Attacker submits an `InvokeV3` transaction with:
   - `l2_gas.max_price_per_unit` ≥ `min_gas_price` (passes stateless check)
   - `l2_gas.max_price_per_unit` ≥ previous-block threshold (passes stateful check)
   - `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0`
2. Both gateway validators pass; the transaction is forwarded to the mempool.
3. The batcher dequeues the transaction and calls `perform_pre_validation_stage` → `check_fee_bounds`, which rejects it with `MaxGasPriceTooLow { resource: L1Gas }`.
4. The transaction is discarded after consuming mempool and batcher resources.

With `max_allowed_nonce_gap = 200`, a single account can inject 200 such permanently-invalid transactions per admission cycle. Across many accounts this constitutes a sustained mempool-poisoning vector.

### Impact Explanation

**High. Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

Transactions with `l1_gas.max_price_per_unit = 0` are structurally invalid for any block whose actual L1 gas price is non-zero, yet they pass every gateway admission gate. The mempool accumulates entries that will never execute, wasting mempool slots, batcher dequeue cycles, and blockifier pre-validation work. The invariant "every transaction admitted to the mempool is executable under current block prices" is broken for the L1 and L1-data gas dimensions.

### Likelihood Explanation

Any unprivileged user can craft such a transaction. No special account state, privileged key, or race condition is required. The `min_gas_price` floor in the stateless validator is `8_000_000_000` for L2 gas but there is no floor at all for L1 gas, so the attack requires only setting `l1_gas.max_price_per_unit = 0` while keeping `l2_gas` fields valid. The TODO comment in the production source confirms the developers are aware the check is incomplete. [5](#0-4) 

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or introduce a new helper) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the corresponding previous-block prices (`strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price`), applying the same `min_gas_price_percentage` multiplier. This mirrors the pattern already used for L2 gas and closes the inconsistency flagged by the existing TODO comment.

### Proof of Concept

```

resource_bounds = AllResourceBounds {
    l2_gas: ResourceBounds {
        max_price_per_unit: 8_000_000_000,  // passes stateless min_gas_price

    },
    l1_gas: ResourceBounds {
        max_price_per_unit: 0,              // zero — no gateway check exists
        max_amount: 10_000,

    l1_data_gas: ResourceBounds {
        max_price_per_unit: 0,              // zero — no gateway check exists
        max_amount: 10_000,

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
