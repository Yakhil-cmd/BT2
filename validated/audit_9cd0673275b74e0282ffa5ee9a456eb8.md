### Title
Unrestricted `update_rewards` with `disable_rewards=true` Allows Any Caller to Permanently Freeze Consensus Rewards for All Stakers - (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and unconditionally writes `last_reward_block` **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(valid_staker, disable_rewards: true)` once per block to mark the block as processed without distributing any rewards, permanently blocking all stakers from receiving consensus-phase block rewards.

---

### Finding Description

`update_rewards` is a publicly callable function (embedded via `#[abi(embed_v0)]`) with no role check. Its body is:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker existence / activity checks ...

    // Update last block rewards.  <-- STATE WRITTEN BEFORE disable_rewards CHECK
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits with NO rewards distributed
    }
    // ... actual reward calculation and distribution ...
}
```

The critical ordering flaw: `last_reward_block` is committed to storage **before** the `disable_rewards` branch. Once written, the per-block guard `current_block_number > self.last_reward_block.read()` causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

`general_prerequisites()` only enforces two conditions:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

There is no role check, no restriction to the attestation contract, and no restriction to the staker themselves.

---

### Impact Explanation

An attacker submits `update_rewards(any_active_staker, disable_rewards: true)` as the first transaction of every block. The result:

1. `last_reward_block` is set to the current block number.
2. No rewards are computed or distributed (early return).
3. Every legitimate call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`.

Repeated across every block, this **permanently freezes all consensus-phase block rewards** for every staker in the protocol. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The attacker requires only a non-zero address and gas to pay for one transaction per block.
- Any active staker address (publicly enumerable from the `stakers` Vec in storage) can be supplied as the argument.
- On Starknet, transaction ordering within a block is controlled by the sequencer; an attacker can submit with a sufficiently high fee to be sequenced first.
- No privileged access, no leaked key, no external dependency is required.

---

### Recommendation

1. **Add access control**: Restrict `update_rewards` to an authorized caller (e.g., the attestation contract, a designated relayer role, or the staker's operational address).
2. **Fix the ordering**: Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` check, so that a disabled call does not consume the block's reward slot.

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only mark the block as processed when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... reward calculation ...
```

---

### Proof of Concept

1. Attacker monitors the Starknet mempool / block production.
2. At the start of each new block, attacker calls:
   ```
   staking_contract.update_rewards(
       staker_address = <any active staker>,
       disable_rewards = true
   )
   ```
3. `last_reward_block` is set to the current block; no rewards are distributed.
4. Any legitimate `update_rewards` call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block → all stakers accumulate zero consensus rewards indefinitely.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
