### Title
Unrestricted `update_rewards` with `disable_rewards=true` Permanently Blocks All Block Reward Distribution - (File: `src/staking/staking.cairo`)

### Summary

`IStakingRewardsManager::update_rewards` in `src/staking/staking.cairo` has no caller restriction. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` to the current block without distributing any rewards. Because `last_reward_block` is a single contract-wide variable, this permanently prevents every staker from receiving block rewards for that block.

### Finding Description

`update_rewards` is the consensus-era reward distribution entry point. Its first meaningful action after validating the staker is to write the current block number into the global `last_reward_block` storage slot: [1](#0-0) 

Immediately after that write, the function checks `disable_rewards`: [2](#0-1) 

If `disable_rewards` is `true`, the function returns without distributing any rewards. The guard at the top of the function prevents a second call in the same block: [3](#0-2) 

`last_reward_block` is a single global slot, not per-staker: [4](#0-3) 

There is no `only_*` role guard anywhere in `update_rewards`: [5](#0-4) 

Compare this with every other sensitive function in the same file, which all call a role guard (e.g., `self.roles.only_token_admin()`, `self.roles.only_app_governor()`, `self.roles.only_security_agent()`): [6](#0-5) 

The public interface exposes `disable_rewards` as a plain boolean with no documentation restricting who may pass `true`: [7](#0-6) 

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block:

1. Advances `last_reward_block` to the current block number.
2. Skips all reward computation and distribution.
3. Causes every subsequent legitimate call to `update_rewards` in that block to revert with `REWARDS_ALREADY_UPDATED`.

Block rewards for that block are permanently lost for all stakers — they cannot be retroactively recovered because the reward calculation is block-scoped and the slot is already consumed. This constitutes **permanent freezing of unclaimed yield** for all stakers, which is a High-severity impact under the allowed scope.

### Likelihood Explanation

- The function is publicly callable with no role restriction.
- The attacker only needs one valid, active staker address with non-zero balance — all staker addresses are observable on-chain.
- The cost is one transaction per block; no capital is required.
- The attack can be automated to run every block, continuously denying all stakers their consensus block rewards.

### Recommendation

Restrict `update_rewards` to a privileged caller (e.g., the sequencer address, a dedicated consensus contract, or a role such as `only_app_governor`). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "no-reward" case internally based on protocol state (e.g., `is_pre_consensus()`), so that no external caller can suppress reward distribution.

### Proof of Concept

1. Alice observes that staker `S` is active with non-zero balance.
2. At the start of block `N`, Alice calls:
   ```
   Staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `last_reward_block` is written to `N`; no rewards are distributed.
4. The legitimate sequencer/consensus layer calls `update_rewards(S, false)` in the same block → reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers permanently lose their block rewards for block `N`.
6. Alice repeats this every block. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
