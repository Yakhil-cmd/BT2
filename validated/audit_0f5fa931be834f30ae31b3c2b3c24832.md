### Title
Pool Admin Can Set Malicious `adminFeeDestination` to Permanently DoS Protocol Fee Collection and Block Factory Owner Fee Updates — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

A pool admin (semi-trusted) can set `adminFeeDestination` to a contract that reverts on ERC20 token receipt. Because `collectFees` transfers admin fees **before** protocol fees, a reverting destination causes the entire fee-collection call to fail. This permanently blocks `collectPoolFees` and `setPoolProtocolFee` for the affected pool, locking accrued protocol fees inside the pool with no factory-owner recovery path.

---

### Finding Description

`setPoolAdminFeeDestination` accepts any non-zero address as the new admin fee destination with no further validation: [1](#0-0) 

The stored address is passed verbatim into every `collectFees` call originating from the factory: [2](#0-1) 

Inside `collectFees`, the admin-fee transfer executes **before** the protocol-fee transfer: [3](#0-2) 

`collectFees` uses `SafeERC20.safeTransfer`, which propagates any revert from the recipient. If `adminFeeDestination_` is a contract that reverts on receipt, the entire `collectFees` call reverts, and the protocol-fee transfers on lines 422–427 are never reached.

Both `collectPoolFees` and `setPoolProtocolFee` call `collectFees` as their **first** action: [4](#0-3) 

`setPoolAdminFees` (pool-admin path) also calls `collectFees` first: [5](#0-4) 

The factory owner has **no independent path** to collect protocol fees or update protocol fees without going through `collectFees`. `collectFees` on the pool is `onlyFactory`, so the factory owner cannot call it directly — only the factory contract can, and the factory always passes `poolAdminFeeDestination[pool]`. [6](#0-5) 

---

### Impact Explanation

- Accrued protocol spread fees and notional fees are permanently locked inside the pool — the factory can never transfer them to itself.
- The factory owner cannot call `setPoolProtocolFee` to adjust protocol fees on the affected pool, breaking the fee-governance invariant.
- `collectPoolFees` (callable by anyone) is permanently bricked for the pool.
- The only recovery requires the pool admin to voluntarily restore a valid destination — a malicious admin will not do so.

---

### Likelihood Explanation

- The pool admin is explicitly semi-trusted; the task's own audit pivots call out "fee collection destinations" as a bypass surface.
- The pool admin needs only two actions: set a non-zero `adminSpreadFeeE6` or `adminNotionalFeeE8` (within caps, no timelock), then point `adminFeeDestination` at a reverting contract.
- No timelock, no factory-owner approval, and no cap governs the destination address change.
- Once fees accumulate (which happens automatically with every swap), the DoS is triggered on the next `collectPoolFees` or `setPoolProtocolFee` call.

---

### Recommendation

1. **Separate admin and protocol fee transfers**: collect protocol fees first (or in an independent sub-call) so a reverting admin destination cannot block protocol fee collection.
2. **Use `try/catch` for admin fee transfers**: wrap the `transferToken0/1` to `adminFeeDestination_` in a try/catch; emit an event on failure rather than reverting the entire call.
3. **Add a factory-owner override**: allow the factory owner to reset `poolAdminFeeDestination` to a safe address when the pool admin is unresponsive or malicious.

---

### Proof of Concept

```
1. Pool admin calls setPoolAdminFees(pool, 10_000, 0)
   → adminSpreadFeeE6 = 10_000 (within cap), fees will accrue.

2. Pool admin deploys MaliciousReceiver:
     receive() external { revert("no tokens"); }
     // also reverts on ERC20 safeTransfer via onERC20Received or fallback

3. Pool admin calls setPoolAdminFeeDestination(pool, address(MaliciousReceiver))
   → poolAdminFeeDestination[pool] = MaliciousReceiver

4. Swaps occur; spread fees accumulate in the pool.

5. Anyone calls collectPoolFees(pool):
   → factory calls pool.collectFees(..., MaliciousReceiver)
   → collectFees computes totalFee0ToAdmin > 0
   → transferToken0(MaliciousReceiver, amount) → safeTransfer reverts
   → collectPoolFees reverts

6. Factory owner calls setPoolProtocolFee(pool, newFee, newFee):
   → same collectFees call → same revert
   → factory owner cannot update protocol fees for this pool

Result: protocol fees permanently locked in pool; factory owner governance over
        this pool's protocol fee is permanently broken.
```
