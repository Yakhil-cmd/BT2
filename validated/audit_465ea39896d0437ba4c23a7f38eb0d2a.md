### Title
Expired Offer Accept Button Enabled When Blockchain Time Is Zero/Loading — (`packages/gui/src/components/offers2/OfferBuilderViewer.tsx`)

### Summary
When `useCurrentBlockchainTime` returns `timestamp=0` (during initial load or when height RPC returns 0), the expiration check `isExpired = expirationTime < currentTime` evaluates to `expirationTime < 0`, which is always `false` for any real Unix timestamp. This causes an expired offer to appear non-expired, enabling the Accept button and allowing the user to submit a spend bundle that will be rejected on-chain, burning the transaction fee.

### Finding Description
`OfferBuilderViewer` computes `isExpired = expirationTime < currentTime` where `currentTime = blockchainTime`. When `blockchainTime = 0` (the default while `isHeightInfoLoading = true`), any positive `expirationTime` produces `isExpired = false`. The `isBlockchainTimeLoading` flag is forwarded only to the display component `OfferBuilderExpirationSection`, never incorporated into `disableAccept`. The component's own `isLoading` guard also omits `isBlockchainTimeLoading`. The `handleAcceptOffer` guard checks only `isWalletSynced` (a separate RPC), which can be `true` while height info is still loading.

### Impact Explanation
A user who opens an imported offer file with a past `maxTime` during the brief window when blockchain time is loading (wallet SYNCED, height info not yet returned) will see the Accept button enabled. Clicking it submits a spend bundle that the full node rejects due to `ASSERT_SECONDS_ABSOLUTE` failure, consuming the transaction fee in XCH. This is direct asset loss triggered by an external offer file/import payload.

### Likelihood Explanation
The window is transient (height info loads quickly), but it is reliably reproducible on every app start when an offer file is pre-loaded or opened immediately. An attacker can craft an offer with a past `maxTime` and share it; the victim's fee is lost on every acceptance attempt during the loading window.

### Recommendation
Add `isBlockchainTimeLoading` to the `disableAccept` expression and treat `blockchainTime === 0` as an unknown/unsafe state:

```javascript
const disableAccept = missingOfferedCATs || showInvalid || isExpired
  || (hasExpiration && (isBlockchainTimeLoading || blockchainTime === 0));
```

Also guard `isExpired` explicitly:

```javascript
if (expirationTime !== 0 && expirationTime !== null && currentTime !== null && currentTime !== 0) {
  isExpired = expirationTime < currentTime;
}
```
