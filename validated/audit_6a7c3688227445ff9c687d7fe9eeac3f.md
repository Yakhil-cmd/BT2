### Title
Off-by-one in `remove_txs_older_than` sentinel construction expires transactions at exact TTL boundary — (`crates/apollo_mempool/src/transaction_pool.rs`)

### Summary

`TimedTransactionMap::remove_txs_older_than` constructs a sentinel `SubmissionID` with `tx_hash: Default::default()` (`Felt::ZERO`, the minimum possible hash) to drive `BTreeMap::split_off`. Because the `SubmissionID::Ord` implementation breaks ties on equal `submission_time` by ascending `tx_hash`, every real transaction whose `submission_time` equals exactly `now - duration` has a key **strictly greater** than the sentinel and is therefore included in the "old" partition and expired — even though its age equals the TTL, not exceeds it.

### Finding Description

**`SubmissionID` ordering** (lines 382–388):

```rust
impl Ord for SubmissionID {
    fn cmp(&self, other: &Self) -> Ordering {
        self.submission_time
            .cmp(&other.submission_time)
            .reverse()                                    // older time → larger key
            .then_with(|| self.tx_hash.cmp(&other.tx_hash)) // tie-break ascending
    }
}
``` [1](#0-0) 

In the `BTreeMap<SubmissionID, TransactionReference>`, newer transactions have smaller keys (reversed time), older transactions have larger keys. Within the same timestamp, keys are ordered by ascending `tx_hash`.

**Sentinel construction** (lines 439–444):

```rust
let split_off_value = SubmissionID {
    submission_time: self.clock.now() - duration,
    tx_hash: Default::default(),   // Felt::ZERO — the minimum
    batching_time: None,
};
let old_txs = self.txs_by_submission_time.split_off(&split_off_value);
``` [2](#0-1) 

`BTreeMap::split_off(k)` returns all entries with key **≥ k**. The sentinel sits at `(submission_time = now - duration, tx_hash = Felt::ZERO)`.

**Boundary case:** A real transaction T with `submission_time = now - duration` and any `tx_hash H > Felt::ZERO` (every real transaction hash) compares against the sentinel as:

1. `submission_time` equal → reversed → still `Equal`
2. `tx_hash`: `H.cmp(Felt::ZERO)` = `Greater`
3. Therefore T's key **> sentinel's key** → T is included in `old_txs` → T is expired.

The function is named `remove_txs_older_than`, implying strict inequality (`age > duration`). A transaction with `age == duration` should be retained, but the sentinel construction with the minimum `tx_hash` causes it to be swept into the expired partition. [3](#0-2) 

### Impact Explanation

A valid, in-flight transaction is silently dropped from the mempool at the exact TTL boundary. The submitting account must detect the absence and resubmit. This maps to:

> **High. Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

The corruption is concrete: the exact `TransactionReference` removed is deterministic given the clock value and the transaction's hash.

### Likelihood Explanation

Low-to-medium in practice. The cleanup must fire at the exact clock tick when `now - submission_time == duration`. With a coarse-grained clock (seconds) and a TTL of e.g. 60 s, the window is one full second wide — not a single nanosecond. Any transaction submitted in that second and still in the pool when cleanup runs is affected. The effect is systematic (not probabilistic) for all transactions at the boundary second.

### Recommendation

Use `Felt::MAX` (or the maximum representable hash) as the sentinel's `tx_hash` so that all transactions at the exact boundary timestamp fall **below** the split point and are retained:

```rust
let split_off_value = SubmissionID {
    submission_time: self.clock.now() - duration,
    tx_hash: TransactionHash(Felt::MAX),  // maximum sentinel
    batching_time: None,
};
```

With this change, only transactions with `submission_time < now - duration` (strictly older) are included in `old_txs`, matching the documented semantics of `remove_txs_older_than`.

### Proof of Concept

```rust
#[test]
fn boundary_tx_not_expired_at_exact_ttl() {
    let fake_clock = Arc::new(FakeClock::default());
    let ttl = Duration::from_secs(60);
    let mut pool = TransactionPool::new(fake_clock.clone());

    // Submit a transaction at T=0.
    let tx = /* build InternalRpcTransaction with known hash */;
    pool.insert(tx.clone()).unwrap();

    // Advance clock by exactly TTL.
    fake_clock.advance(ttl);

    // remove_txs_older_than should NOT remove a tx whose age == TTL.
    let removed = pool.remove_txs_older_than(ttl, &HashMap::new());

    // With the current code this assertion FAILS — the tx is in `removed`.
    assert!(removed.is_empty(), "tx at exact TTL boundary must not be expired");
}
```

The test will fail with the current sentinel (`tx_hash = Felt::ZERO`) and pass after the fix (`tx_hash = Felt::MAX`).

    pub fn remove_txs_older_than(
        &mut self,
        duration: Duration,
        exclude_txs: &HashMap<ContractAddress, Nonce>,
    ) -> Vec<TransactionReference> {
        let split_off_value = SubmissionID {
            submission_time: self.clock.now() - duration,
            tx_hash: Default::default(),
            batching_time: None,
        };
        let old_txs = self.txs_by_submission_time.split_off(&split_off_value);

        let mut removed_txs = Vec::new();
        for (submission_id, tx) in old_txs.into_iter() {
            if exclude_txs.get(&tx.address).is_some_and(|nonce| tx.nonce < *nonce) {
                // The transaction should be preserved. Add it back.
                self.txs_by_submission_time.insert(submission_id, tx);
            } else {
                self.hash_to_submission_id.remove(&tx.tx_hash).expect(
                    "Transaction should have a submission ID if it is in the timed transaction \
                     map.",
                );
                removed_txs.push(tx);
            }
        }

        removed_txs
    }
