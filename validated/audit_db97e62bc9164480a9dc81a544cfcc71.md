### Title
`StatelessTransactionValidator::validate_resource_bounds` Unconditionally Skips `max_l2_gas_amount` Ceiling for Declare Transactions — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

The `max_l2_gas_amount` ceiling check in `StatelessTransactionValidator::validate_resource_bounds` is gated behind an empty `if let RpcTransaction::Declare(_) = tx {}` branch that does nothing, causing the check to be silently skipped for every Declare transaction. An unprivileged user can submit an `RpcDeclareTransactionV3` with `l2_gas.max_amount = u64::MAX` and receive `Ok(())` from stateless validation, while the identical resource bounds on an Invoke or DeployAccount would return `MaxGasAmountTooHigh`. The gateway's own admission rule is therefore applied inconsistently, and the Declare enters the mempool pipeline with resource bounds the gateway is explicitly configured to reject.

---

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs`, the relevant block is:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

The `if let` arm is an empty block. Rust evaluates it as: "if this is a Declare, do nothing; otherwise, check the ceiling." The ceiling check is therefore unreachable for any Declare variant. The TODO comment confirms this is a known, unresolved gap, not an intentional design decision.

The test suite makes the bypass explicit in two ways:

1. `valid_l2_gas_amount_on_declare` asserts `Ok(())` for a Declare with `max_amount = 200` when `max_l2_gas_amount = 100` — a 2× overage passes. [2](#0-1) 

2. `test_invalid_max_l2_gas_amount` only parameterises over `TransactionType::DeployAccount` and `TransactionType::Invoke`, explicitly omitting Declare from the rejection test. [3](#0-2) 

The default production ceiling is `max_l2_gas_amount = 1_210_000_000`. [4](#0-3) 

---

### Impact Explanation

The gateway's stateless validator is the first admission gate. Its purpose is to reject structurally invalid transactions before they consume stateful resources (nonce lookup, balance check, blockifier execution). By skipping the `max_l2_gas_amount` ceiling for Declare, the gateway admits Declare transactions with arbitrarily large `l2_gas.max_amount` values — up to `u64::MAX` — that it is explicitly configured to reject for every other transaction type. The corrupted admission value is concrete: `Ok(())` is returned where `Err(MaxGasAmountTooHigh)` is required by the gateway's own policy.

Downstream, the stateful validator will eventually reject the transaction because no account holds enough balance to cover `u64::MAX × min_gas_price` in fees. However, the transaction has already passed the stateless gate and entered the mempool pipeline, consuming stateful validation resources (state reads, blockifier pre-validation) for a transaction the gateway should have dropped at the door. An attacker can continuously submit such Declares to amplify stateful validation load without any stateless cost.

---

### Likelihood Explanation

Trivially exploitable. No privileges are required. The attacker constructs a valid `RpcDeclareTransactionV3` (any minimal Sierra class suffices) and sets `resource_bounds.l2_gas.max_amount` to any value above `max_l2_gas_amount`. The bypass is unconditional — it applies to every Declare regardless of sender, class content, or network state.

---

### Recommendation

Remove the empty `if let` arm and apply the ceiling check uniformly:

```rust
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If Declare transactions legitimately require a higher ceiling (e.g., because class compilation consumes more L2 gas), introduce a separate `max_l2_gas_amount_declare` config field with an explicit, documented value rather than an unchecked bypass. Update `valid_l2_gas_amount_on_declare` to assert rejection, and add Declare to the `test_invalid_max_l2_gas_amount` parameterisation.

### Proof of Concept

```rust
#[test]
fn declare_bypasses_max_l2_gas_amount_ceiling() {
    use apollo_gateway::stateless_transaction_validator::StatelessTransactionValidator;
    use apollo_gateway_config::config::StatelessTransactionValidatorConfig;
    use starknet_api::transaction::fields::{AllResourceBounds, GasAmount, GasPrice, ResourceBounds};

    let config = StatelessTransactionValidatorConfig {
        validate_resource_bounds: true,
        max_l2_gas_amount: 1_210_000_000,
        min_gas_price: 8_000_000_000,
        ..Default::default()
    };
    let validator = StatelessTransactionValidator { config };

    // Build a Declare with l2_gas.max_amount = u64::MAX (far above the ceiling).
    let declare_tx = rpc_tx_for_testing(
        TransactionType::Declare,
        RpcTransactionArgs {
            resource_bounds: AllResourceBounds {
                l2_gas: ResourceBounds {
                    max_amount: GasAmount(u64::MAX),
                    max_price_per_unit: GasPrice(8_000_000_000),
                },
                ..Default::default()
            },
            ..Default::default()
        },
    );

    // Gateway admits the Declare — bypass confirmed.
    assert_matches!(validator.validate(&declare_tx), Ok(()));

    // Same bounds on Invoke are correctly rejected.
    let invoke_tx = rpc_tx_for_testing(
        TransactionType::Invoke,
        RpcTransactionArgs {
            resource_bounds: AllResourceBounds {
                l2_gas: ResourceBounds {
                    max_amount: GasAmount(u64::MAX),
                    max_price_per_unit: GasPrice(8_000_000_000),
                },
                ..Default::default()
            },
            ..Default::default()
        },
    );
    assert_matches!(
        validator.validate(&invoke_tx),
        Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { .. })
    );
}
```
