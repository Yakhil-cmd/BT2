### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Consensus Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is specified as "Only starkware sequencer" but has no on-chain access control enforcement. It unconditionally writes `last_reward_block` to the current block number **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke it with `disable_rewards=true`, consuming the one-call-per-block slot without distributing any rewards. The sequencer's subsequent legitimate call for the same block is then rejected with `REWARDS_ALREADY_UPDATED`, permanently destroying that block's consensus rewards for all stakers.
