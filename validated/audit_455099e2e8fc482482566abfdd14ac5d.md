### Title
Staker With Active Commission Commitment Can Front-Run Delegators by Instantly Raising Commission to `max_commission` - (File: src/staking/staking.cairo)

### Summary

A staker who holds an active `CommissionCommitment` can call `set_commission` at any time to raise their commission up to `max_commission`. Because the commission is stored at the pool level and applied immediately to all reward distributions, a malicious staker can front-run a delegator's `enter_delegation_pool` transaction — or any pending reward-distribution transaction — by spiking the commission to `max_commission`. The delegator has no way to specify a maximum acceptable commission in `enter_delegation_pool`, so they are silently subjected to the higher rate.

### Finding Description

The `set_commission` function in `src/staking/staking.cairo` delegates to the internal `update_commission` helper. When a `CommissionCommitment` is active, the only constraint is that the new commission must not exceed `max_commission` and must differ from the old value:

```rust
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
}
``` [1](#0-0) 

There is no delay, no per-epoch grace period, and no per-delegator snapshot. The updated commission is written directly to `staker_pool_info.commission`:

```rust
staker_pool_info.commission.write(Option::Some(commission));
``` [2](#0-1) 

Every subsequent reward distribution reads this single pool-level value:

```rust
let commission = staker_pool_info.commission();
...
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [3](#0-2) 

The `enter_delegation_pool` function in `src/pool/pool.cairo` accepts no `max_commission` parameter, so a delegator cannot protect themselves from a commission change that occurs between when they sign their transaction and when it is included in a block. [4](#0-3) 

The developers themselves acknowledge this gap with an explicit code comment:

```
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
``` [5](#0-4) 

### Impact Explanation

When a staker front-runs a delegator's `enter_delegation_pool` (or any pending attestation/reward-update transaction) by raising commission from `X` to `max_commission`, the delegator earns proportionally fewer rewards for every epoch they remain delegated. Because the commission is applied to the entire pool balance — not just new delegations — all existing delegators are simultaneously harmed. The staker captures the difference as additional commission rewards. This constitutes **theft of unclaimed yield** from delegators.

Mapped to the allowed impact scope: **High — Theft of unclaimed yield**.

### Likelihood Explanation

The attack requires two preconditions:
1. The staker must have an active `CommissionCommitment` with `max_commission > current_commission`. This is a voluntary, publicly visible on-chain state that a staker can set up in advance.
2. The staker must be able to order their `set_commission` transaction before the delegator's `enter_delegation_pool` transaction. On Starknet, sequencers can reorder transactions within a block, making this feasible for a staker who also controls or monitors the mempool.

Both conditions are realistic and require no privileged protocol access beyond the staker's own key.

### Recommendation

1. **Add a `max_commission` parameter to `enter_delegation_pool`**: Allow delegators to specify the maximum commission they are willing to accept. Revert if the pool's current commission exceeds this value at execution time.

2. **Enforce a time-lock on commission increases**: When a staker raises their commission (even within a commitment), require the new rate to take effect only after a delay of at least one epoch, giving delegators time to exit before the higher rate applies.

3. **Store commission per-epoch in the pool's cumulative-rewards trace**: Snapshot the commission at the epoch boundary so that reward calculations for past epochs always use the commission that was in effect at that time, preventing retroactive harm to delegators.

### Proof of Concept

1. Staker calls `set_commission_commitment(max_commission: 5000, expiration_epoch: current + 10)` to establish a commitment allowing commission up to 50%.
2. Staker currently has `commission = 100` (1%).
3. Delegator submits `enter_delegation_pool(reward_address, amount)` targeting this pool.
4. Staker observes the pending transaction and immediately submits `set_commission(5000)` with higher gas priority.
5. `set_commission(5000)` is included first; `update_commission` passes because `5000 <= max_commission` and `5000 != 100`.
6. Delegator's `enter_delegation_pool` is included next; it succeeds with no commission check.
7. On the next attestation/reward-update call, `split_rewards_with_commission` uses `commission = 5000`, so the staker takes 50% of pool rewards instead of 1%, and the delegator receives only 50% of what they expected. [6](#0-5) [7](#0-6) [8](#0-7)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
