### Title
Gateway Skips `__validate__` Signature Check for Any Invoke with Nonce=1 When Victim Has a Pending Deploy-Account — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator unconditionally skips the `__validate__` entry-point (and therefore all signature verification) for any Invoke transaction whose nonce is 1 and whose sender address has *any* transaction in the mempool or a recent block. Because the mempool check is keyed only on the sender address supplied in the attacker-controlled transaction fields, an unprivileged attacker can submit an Invoke transaction bearing a victim's address and a garbage signature, and the gateway will admit it without ever calling `__validate__`.

### Finding Description

`skip_stateful_validations` is the Sequencer-native analog of the `voteExtraordinary(account_)` bug: both functions accept an attacker-supplied address and perform a privileged action (skipping signature validation / casting a vote) on behalf of that address without verifying that the caller is actually the account owner.

The relevant code path is:

1. `extract_state_nonce_and_run_validations` calls `run_pre_validation_checks`, which calls `skip_stateful_validations`. [1](#0-0) 

2. `skip_stateful_validations` returns `true` (skip) when:
   - the transaction is an `Invoke`,
   - the transaction nonce equals `Nonce(Felt::ONE)`,
   - the on-chain account nonce equals `Nonce(Felt::ZERO)` (account not yet deployed), and
   - `account_tx_in_pool_or_recent_block(tx.sender_address())` returns `true`. [2](#0-1) 

3. When `skip_validate = true`, `run_validate_entry_point` sets `execution_flags.validate = false`. [3](#0-2) 

4. Inside `StatefulValidator::perform_validations`, when `execution_flags.validate` is `false`, the function returns `Ok(())` immediately after `perform_pre_validation_stage`, never calling `validate_tx` (the `__validate__` entry point). [4](#0-3) 

5. `account_tx_in_pool_or_recent_block` checks only whether *any* transaction from the given address is known to the mempool — it does not verify that the transaction is a `DeployAccount` or that it was submitted by the same party. [5](#0-4) 

The comment in the code acknowledges the intent: "It is sufficient to check if the account exists in the mempool since it means that either it has a deploy_account transaction or transactions with future nonces that passed validations." This reasoning is correct for the legitimate user but fails to account for an attacker who can freely choose the `sender_address` field of an Invoke transaction.

### Impact Explanation

An attacker who observes a victim's `DeployAccount` transaction in the mempool can immediately submit an `Invoke` transaction with:
- `sender_address = victim_address`
- `nonce = 1`
- arbitrary (invalid) `signature`

The gateway's stateless validator does not check signatures. [6](#0-5) 

The gateway's stateful nonce check passes because `0 ≤ 1 ≤ max_allowed_nonce_gap`. [7](#0-6) 

The mempool's `validate_tx` passes because there is no duplicate hash and nonce=1 is a valid future nonce. [8](#0-7) 

The invalid transaction is admitted to the mempool. When the batcher later executes it, `__validate__` is called (the skip is gateway-only), the signature check fails, and the transaction reverts — but the fee for the failed validation is charged to the victim's newly-deployed account. Additionally, the attacker's transaction occupies nonce slot 1 for the victim, forcing the victim to pay a higher fee to replace it via fee escalation.

**Matching impact**: *High — Mempool/gateway admission accepts an invalid transaction (one that would fail `__validate__`) before sequencing.*

### Likelihood Explanation

The attack requires only:
1. Monitoring the public mempool for `DeployAccount` transactions (trivially observable).
2. Submitting a single crafted `Invoke` transaction with the victim's address and nonce=1.

No privileged access, no special contract, no prior relationship with the victim is needed. The window of opportunity is the time between the victim's `DeployAccount` entering the mempool and being included in a block.

### Recommendation

Replace the `account_tx_in_pool_or_recent_block` check with a check that specifically verifies a `DeployAccount` transaction for the sender address is pending, and additionally verify that the `DeployAccount` transaction hash matches the one the user claims (e.g., pass the expected deploy-account tx hash as part of the invoke submission, as the `native_blockifier` `PyValidator` already does via `deploy_account_tx_hash`). [9](#0-8) 

Alternatively, require that the invoke transaction carry a valid signature even when `skip_validate` is set, and only skip the on-chain `__validate__` call (not the gateway-level signature pre-check).

### Proof of Concept

```
1. Victim submits RpcTransaction::DeployAccount for address V (nonce=0, valid signature).
   → Mempool now contains a tx from address V.

2. Attacker submits RpcTransaction::Invoke with:
     sender_address = V
     nonce          = 1
     signature      = [0xdead, 0xbeef]   // garbage

3. Gateway stateless validator: passes (no signature check).

4. Gateway stateful validator:
   a. get_nonce_from_state(V) → Nonce(0)   (V not deployed yet)
   b. validate_nonce: 0 ≤ 1 ≤ max_gap     → Ok
   c. validate_by_mempool: nonce=1 is a valid future nonce → Ok
   d. skip_stateful_validations:
        tx.nonce() == 1  ✓
        account_nonce == 0  ✓
        account_tx_in_pool_or_recent_block(V) == true  ✓  (victim's DeployAccount is there)
      → returns true (skip __validate__)
   e. run_validate_entry_point(skip_validate=true):
        execution_flags.validate = false
        StatefulValidator::perform_validations returns Ok(()) without calling __validate__

5. Invalid Invoke is added to the mempool (nonce slot 1 for address V is now occupied).

6. Victim tries to submit their legitimate Invoke (nonce=1, valid signature):
   → Mempool rejects it as DuplicateNonce, or victim must pay higher fee for replacement.

7. Batcher executes the block:
   - DeployAccount(V, nonce=0) succeeds → V is deployed.
   - Attacker's Invoke(V, nonce=1, bad sig) → __validate__ is called → fails → reverts.
   - Fee for failed validation is charged to V's account.
```

```rust
        skip_validate: bool,

        let charge_fee = enforce_fee(executable_tx, only_query);
        let strict_nonce_check = false;
        let execution_flags =
            ExecutionFlags { only_query, charge_fee, validate: !skip_validate, strict_nonce_check };

        let account_tx = AccountTransaction { tx: executable_tx.clone(), execution_flags };

        // Build block context.
        let mut versioned_constants = VersionedConstants::get_versioned_constants(
            self.config.versioned_constants_overrides.clone(),
        );
        // The validation of a transaction is not affected by the casm hash migration.
        versioned_constants.disable_casm_hash_migration();

        let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
        block_info.block_number = block_info.block_number.unchecked_next();
        let block_context = BlockContext::new(
            block_info,
            self.chain_info.clone(),
            versioned_constants,
            BouncerConfig::max(),
        );

        // Move state into the blocking task and run CPU-heavy validation.
        let state_reader_and_contract_manager = self.take_state_reader_and_contract_manager();

        let cur_span = Span::current();
        #[allow(clippy::result_large_err)]
        tokio::task::spawn_blocking(move || {
            cur_span.in_scope(|| {
                let state = CachedState::new(state_reader_and_contract_manager);
                let mut blockifier_validator = StatefulValidator::create(state, block_context);
                blockifier_validator.validate(account_tx)
            })
        })
        .await
        .map_err(|e| StarknetError {
            code: StarknetErrorCode::UnknownErrorCode(
```

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
```

```rust
    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
```
