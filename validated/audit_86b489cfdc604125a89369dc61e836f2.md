### Title
No Maximum Commission Guard in `enter_delegation_pool` Allows Staker to Front-Run Delegators and Steal Yield - (File: src/pool/pool.cairo)

### Summary
`enter_delegation_pool` and `add_to_delegation_pool` in `src/pool/pool.cairo` accept no `max_commission` parameter. A staker who holds an active `commission_commitment` can front-run a delegator's delegation transaction by calling `set_commission` to raise the commission up to `max_commission`, causing the delegator to receive materially less yield than they anticipated with no ability to revert.

### Finding Description
The `update_commission` internal function in `src/staking/staking.cairo` enforces two distinct code paths:

- **Without an active commitment** (line 1596): commission can only decrease (`commission < old_commission`).
- **With an active commitment** (lines 1583–1589): commission can be set to any value up to `commission_commitment.max_commission`, including values **higher** than the current commission. [1](#0-0) 

The developers themselves acknowledge this in a code comment directly above `set_commission_commitment`:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [2](#0-1) 

`enter_delegation_pool` performs no check on the pool's current commission rate before accepting the delegator's funds: [3](#0-2) 

The same omission exists in `add_to_delegation_pool`: [4](#0-3) 

### Impact Explanation
A delegator who observes a pool with 5% commission and submits `enter_delegation_pool` can be front-run by the staker calling `set_commission(50%)` (permitted by an active commitment). The delegator's transaction executes and they are locked into a pool with 50% commission. All future yield accrued by that delegator is split 50/50 with the staker instead of 95/5. The excess commission flows directly to the staker's reward address — this is a direct, ongoing theft of unclaimed yield from the delegator.

**Impact: High — Theft of unclaimed yield.**

### Likelihood Explanation
The precondition is that the staker has an active `commission_commitment` with a `max_commission` higher than the current commission. This is a standard, publicly visible on-chain feature that any staker can set at any time. A malicious staker can set a commitment with `max_commission = 10000` (100%) immediately after deploying their pool, making the attack permanently available. Starknet's public mempool means the staker can observe and front-run any `enter_delegation_pool` transaction. No privileged access or external dependency is required beyond the staker's own key.

### Recommendation
Add a `max_commission: Commission` parameter to both `enter_delegation_pool` and `add_to_delegation_pool`. At the start of each function, read the current commission from the staking contract and assert it does not exceed the caller-supplied maximum:

```cairo
fn enter_delegation_pool(
    ref self: ContractState,
    reward_address: ContractAddress,
    amount: Amount,
    max_commission: Commission,   // <-- add this
) {
    // ... existing asserts ...
    let current_commission = /* read from staking contract */;
    assert!(current_commission <= max_commission, Error::COMMISSION_TOO_HIGH);
    // ... rest of function ...
}
```

This mirrors the fix applied in the referenced Pool.buy report: add a caller-supplied ceiling and revert if the actual rate exceeds it.

### Proof of Concept

1. Staker deploys a pool with commission = 500 (5%).
2. Staker calls `set_commission_commitment(max_commission: 9000, expiration_epoch: current + 52)` — this is publicly visible on-chain.
3. Delegator reads commission = 500, approves tokens, and submits `enter_delegation_pool(reward_address, amount)`.
4. Staker observes the pending transaction and front-runs with `set_commission(9000)` (90%), which succeeds because an active commitment permits it. [5](#0-4) 
5. Delegator's `enter_delegation_pool` executes. No commission check exists in the function body. [3](#0-2) 
6. Delegator is now a pool member at 90% commission. For every 100 STRK of pool rewards, 90 STRK flows to the staker and only 10 STRK to the delegator — a permanent, ongoing theft of unclaimed yield until the delegator manually exits.

        fn add_to_delegation_pool(
            ref self: ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Amount {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            assert!(
                caller_address == pool_member || caller_address == pool_member_info.reward_address,
                "{}",
                Error::CALLER_CANNOT_ADD_TO_POOL,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);

            // Transfer funds from the delegator to the staking contract.
            let token_dispatcher = self.token_dispatcher.read();
            let staker_address = self.staker_address.read();
            transfer_from_delegator(pool_member: caller_address, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            // Update the pool member's balance checkpoint.
            let old_delegated_stake = self.increase_member_balance(:pool_member, :amount);
            let new_delegated_stake = old_delegated_stake + amount;

            // Emit events.
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );

            new_delegated_stake
        }
