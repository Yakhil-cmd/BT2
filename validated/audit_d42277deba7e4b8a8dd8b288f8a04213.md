### Title
Missing Caller Authentication in `update_rewards` Allows Any Address to Permanently Freeze Staker Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` without distributing rewards, permanently blocking the legitimate sequencer from distributing rewards for that block.

### Finding Description
The protocol specification explicitly restricts `update_rewards` to "Only starkware sequencer": [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` performs no such check. The function begins with only `self.general_prerequisites()` (a pause check) and a block-number guard, then immediately proceeds to write `last_reward_block`: [2](#0-1) 

The critical state mutation is at line 1485, where `last_reward_block` is unconditionally updated to the current block number before the `disable_rewards` branch: [3](#0-2) 

When `disable_rewards` is `true`, the function returns immediately after writing `last_reward_block`, distributing zero rewards. Because the block-number guard (`current_block_number > self.last_reward_block.read()`) is now satisfied for the current block, any subsequent call — including the legitimate sequencer's — will revert with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

`last_reward_block` is a single global variable, so one attacker call per block poisons the entire reward distribution for all stakers in that block.

### Impact Explanation
An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` once per block permanently forfeits all staker and delegator block rewards for that block. Because the guard is block-scoped and irreversible, the lost rewards can never be recovered. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators, matching the High-severity impact category.

### Likelihood Explanation
The function is publicly callable with no authentication. The attacker needs only to know any active staker address (trivially discoverable from on-chain events) and to front-run or race the sequencer's transaction each block. The cost is only gas. A motivated griever can sustain this attack indefinitely, and there is no on-chain mechanism to recover the skipped rewards.

### Recommendation
Add a caller check at the top of `update_rewards` that asserts `get_caller_address() == sequencer_address`, consistent with the specification's "Only starkware sequencer" access control. The sequencer address should be stored in contract storage during initialization and updatable only by a privileged governance role.

### Proof of Concept
1. Staker Alice stakes and waits K epochs for her stake to become effective.
2. Consensus rewards are activated (`set_consensus_rewards_first_epoch` is called).
3. Attacker Bob calls `staking.update_rewards(alice_address, disable_rewards: true)` at block N.
   - `last_reward_block` is set to N; no rewards are distributed.
4. The legitimate sequencer attempts `staking.update_rewards(alice_address, disable_rewards: false)` at block N.
   - Reverts with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. Alice's block-N rewards are permanently lost.
6. Bob repeats step 3 at every subsequent block, continuously suppressing all reward distribution at negligible cost. [5](#0-4) [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
