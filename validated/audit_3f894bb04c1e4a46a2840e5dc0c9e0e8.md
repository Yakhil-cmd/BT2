### Title
Missing Zero Address Validation for `reward_address` Allows Temporary Freezing of Staker Principal - (File: src/staking/staking.cairo, src/pool/pool.cairo)

### Summary
The `stake()` and `change_reward_address()` functions in `staking.cairo`, and `enter_delegation_pool()` and `change_reward_address()` in `pool.cairo`, accept a `reward_address` parameter without checking whether it is the zero address. If a staker or delegator sets their reward address to zero, the `unstake_action()` (or `exit_delegation_pool_action()`) flow will revert when it attempts to transfer rewards to address zero, temporarily freezing the user's principal until the reward address is corrected.

### Finding Description
The `stake()` function in `staking.cairo` validates that `reward_address` is not a registered token address, but performs no zero-address check: [1](#0-0) 

Similarly, `change_reward_address()` in `staking.cairo` only checks for the token-address collision: [2](#0-1) 

And in `pool.cairo`, `enter_delegation_pool()` and `change_reward_address()` only guard against the token address: [3](#0-2) [4](#0-3) 

The `GenericError::ZERO_ADDRESS` variant already exists in the codebase and is used elsewhere (e.g., `add_token` checks `token_address.is_non_zero()`), confirming the developers are aware of this pattern but omitted it for reward addresses: [5](#0-4) [6](#0-5) 

### Impact Explanation
When `unstake_action()` is called, it internally calls `send_rewards_to_staker()`, which calls `checked_transfer(recipient: reward_address, ...)`. The OpenZeppelin ERC20 implementation on Starknet rejects transfers to address zero. If `reward_address` is zero, `unstake_action()` reverts entirely, preventing the staker from recovering their principal: [7](#0-6) [8](#0-7) 

The staker's principal remains locked in the contract until they call `change_reward_address()` with a valid address and retry `unstake_action()`. This constitutes **temporary freezing of funds** (High impact per the allowed scope). The same applies to pool members via `exit_delegation_pool_action()` → `claim_rewards()` → transfer to zero reward address. [9](#0-8) 

### Likelihood Explanation
A staker or delegator must pass `reward_address = 0` either at registration (`stake()` / `enter_delegation_pool()`) or via a subsequent `change_reward_address()` call. This can occur through a scripting error, a front-end bug, or a misconfigured deployment. The likelihood is **low** but non-zero, and the consequence (inability to unstake principal) is disproportionate to the mistake.

### Recommendation
Add a zero-address guard in all four entry points:

- `stake()` in `staking.cairo`: assert `reward_address.is_non_zero()`
- `change_reward_address()` in `staking.cairo`: assert `reward_address.is_non_zero()`
- `enter_delegation_pool()` in `pool.cairo`: assert `reward_address.is_non_zero()`
- `change_reward_address()` in `pool.cairo`: assert `reward_address.is_non_zero()`

Use the already-defined `GenericError::ZERO_ADDRESS` error for consistency.

### Proof of Concept

1. Staker calls `stake(reward_address: 0, operational_address: X, amount: MIN_STAKE)` — passes all existing checks, staker is registered with `reward_address = 0`.
2. Staker accumulates rewards over several epochs.
3. Staker calls `unstake_intent()` — succeeds, exit timestamp is set.
4. After the exit window elapses, staker calls `unstake_action(staker_address)`.
5. Inside `unstake_action`, `send_rewards_to_staker` is called, which executes `token_dispatcher.checked_transfer(recipient: 0, amount: rewards)`.
6. The OZ ERC20 `_transfer` asserts `!recipient.is_zero()` → transaction reverts.
7. The staker's principal remains locked in the staking contract. The staker must first call `change_reward_address(valid_address)` and then retry `unstake_action`. [10](#0-9) [11](#0-10)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
