### Title
Unprivileged Auto-Deployment Permanently Blocks Admin/Governance from Registering External Token Mappings - (File: `x/cronos/keeper/keeper.go`, `x/cronos/keeper/evm.go`)

### Summary
When `EnableAutoDeployment` is true, any unprivileged user can trigger auto-deployment of a CRC21 contract for a new denom by sending an IBC transfer. The `ensureDenomNotMapped` guard (introduced in the 2026-04 ADR-008 update) then permanently prevents the admin or governance from registering an external contract for that denom, bypassing their token-mapping authority with no available remediation path.

### Finding Description

**Root cause — `ensureDenomNotMapped` blocks all re-registration, including admin overrides:**

`SetExternalContractForDenom` unconditionally calls `ensureDenomNotMapped`, which returns `ErrDenomAlreadyMapped` if *any* contract — external **or** auto-deployed — is already mapped to the denom:

```go
func (k Keeper) ensureDenomNotMapped(ctx sdk.Context, denom string) error {
    if contract, found := k.GetContractByDenom(ctx, denom); found {
        return errors.Wrapf(
            types.ErrDenomAlreadyMapped,
            "denom %s is already mapped to contract %s",
            denom, contract.Hex(),
        )
    }
    return nil
}
``` [1](#0-0) 

`GetContractByDenom` returns the auto contract if no external contract exists:

```go
func (k Keeper) GetContractByDenom(ctx sdk.Context, denom string) (contract common.Address, found bool) {
    contract, found = k.getExternalContractByDenom(ctx, denom)
    if !found {
        contract, found = k.getAutoContractByDenom(ctx, denom)
    }
    return contract, found
}
``` [2](#0-1) 

**Unprivileged trigger — IBC transfer causes auto-deployment:**

`ConvertCoinFromNativeToCRC21` with `autoDeploy=true` deploys a new contract and registers it via `SetAutoContractForDenom` when no mapping exists:

```go
contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
...
if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
    return err
}
``` [3](#0-2) 

This is triggered by any IBC transfer when `EnableAutoDeployment` is true, via `ConvertVouchersToEvmCoins`: [4](#0-3) 

And via the IBC middleware `OnRecvPacket`: [5](#0-4) 

**No admin escape hatch — `DeleteExternalContractForDenom` does not remove auto mappings:**

The only "delete" path in `RegisterOrUpdateTokenMapping` (when `msg.Contract == ""`) calls `DeleteExternalContractForDenom`, which only removes the external mapping key, leaving the auto mapping intact: [6](#0-5) 

After deletion of the external mapping, `GetContractByDenom` still returns the auto contract, so `ensureDenomNotMapped` still returns `ErrDenomAlreadyMapped`. There is no `DeleteAutoContractForDenom` accessible to the admin or governance.

**Governance path is equally blocked:**

The governance `TokenMappingChangeProposal` handler also calls `RegisterOrUpdateTokenMapping`: [7](#0-6) 

### Impact Explanation

**High — Bypass of Cronos admin and governance token-mapping authorization checks.**

Once an attacker triggers auto-deployment for a target denom, neither the admin (`MsgUpdateTokenMapping`) nor governance (`TokenMappingChangeProposal`) can ever register an external contract for that denom. The admin's intended authority to control token mappings — including deploying contracts with custom logic such as blocklists, fee mechanisms, or migration paths — is permanently nullified for any denom the attacker targets. This matches the allowed impact: *"Bypass of Cronos admin, governance authority, permission, token-mapping... authorization checks."*

### Likelihood Explanation

Moderate. The attack requires `EnableAutoDeployment = true` (a configurable parameter) and the attacker to possess or receive any amount of the target denom. Any IBC transfer of a new denom to Cronos is sufficient. The attacker only needs to act before the admin submits `MsgUpdateTokenMapping`, making frontrunning straightforward since admin transactions are visible in the mempool.

### Recommendation

1. In `RegisterOrUpdateTokenMapping`, distinguish between "denom has an external mapping" (block re-registration) and "denom has only an auto mapping" (allow admin override). The `ensureDenomNotMapped` check should only block re-registration when an *external* contract is already active.
2. Alternatively, expose an admin-accessible `DeleteAutoContractForDenom` function so the admin can clear an auto mapping before registering an external one.
3. The `SetExternalContractForDenom` call path from `RegisterOrUpdateTokenMapping` should be permitted to supersede auto-deployed mappings, consistent with the original design intent where external contracts take precedence over auto-deployed ones.

### Proof of Concept

1. `EnableAutoDeployment = true` on the Cronos chain.
2. Admin prepares `MsgUpdateTokenMapping{Denom: "ibc/XXXX", Contract: "0xCustom..."}` to register a custom external contract.
3. Attacker observes the pending admin transaction in the mempool and frontruns it by sending 1 unit of `ibc/XXXX` to Cronos via IBC.
4. IBC middleware calls `OnRecvPacket` → `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21(autoDeploy=true)` → `DeployModuleCRC21` + `SetAutoContractForDenom`. Auto contract `0xAutoDeployed` is now mapped to `ibc/XXXX`.
5. Admin's `MsgUpdateTokenMapping` executes: `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom` → `ensureDenomNotMapped` → finds `0xAutoDeployed` → returns `ErrDenomAlreadyMapped`. Transaction fails.
6. Admin attempts to clear the mapping with `MsgUpdateTokenMapping{Denom: "ibc/XXXX", Contract: ""}` → `DeleteExternalContractForDenom` → no external mapping exists, no-op. Auto mapping remains.
7. Admin's `MsgUpdateTokenMapping{Denom: "ibc/XXXX", Contract: "0xCustom..."}` fails again with `ErrDenomAlreadyMapped`. The admin is permanently blocked.

func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
	store := ctx.KVStore(k.storeKey)
	contract, found := k.getExternalContractByDenom(ctx, denom)
	if !found {
		return false
	}
	store.Delete(types.DenomToExternalContractKey(denom))
	deleteReverseIfOwned(store, contract, denom)
	if auto, found := k.getAutoContractByDenom(ctx, denom); found {
		bz := store.Get(types.ContractToDenomKey(auto.Bytes()))
		if len(bz) == 0 {
			store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
		} else if existingDenom := string(bz); existingDenom != denom {
			if k.contractOwnedByDenom(ctx, existingDenom, auto) {
				// auto address is already owned by another denom; drop local auto mapping
				store.Delete(types.DenomToAutoContractKey(denom))
			} else {
				// stale reverse entry
				store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
			}
		}
	}
	return true
```
