### Title
Global `last_reward_block` Tracker Updated Before `disable_rewards` Check Allows Unprivileged Griefing of Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract uses a single global `last_reward_block` storage variable to prevent double-counting of block rewards. Because `last_reward_block` is written **before** the `disable_rewards` guard is evaluated, any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` to consume the block's reward slot without distributing any rewards. This mirrors the Pepper bug's root cause: a shared global counter is updated by both legitimate and illegitimate paths, causing legitimate operations to be blocked.

---

### Finding Description

`update_rewards` is a publicly callable function (guarded only by `general_prerequisites`, which checks for non-zero caller and unpaused state — no role restriction). [1](#0-0) 

Inside the function, the critical ordering flaw is:

```
// 1. Check that this block hasn't been processed yet.
assert!(current_block_number > self.last_reward_block.read(), ...REWARDS_ALREADY_UPDATED);

// ... staker validation ...

// 2. Write the block number — BEFORE checking disable_rewards.
self.last_reward_block.write(current_block_number);

// 3. Only now check whether rewards should actually be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

`last_reward_block` is a **single global value** shared across all stakers: [3](#0-2) 

Because step 2 (write) happens before step 3 (guard), calling `update_rewards(any_active_staker, disable_rewards: true)` in block N will:
- Set `last_reward_block = N`
- Return immediately without distributing rewards

Any subsequent legitimate call to `update_rewards` in block N — whether by the consensus mechanism or the staker — will revert with `REWARDS_ALREADY_UPDATED` because `N > N` is false.

This is the direct analog to the Pepper bug: `totalSupply()` (updated by both `mint` and `claim`) was used as the limit tracker instead of a separate counter for role-based mints. Here, `last_reward_block` (updated by both legitimate reward calls and `disable_rewards: true` calls) is used as the per-block gate instead of a tracker that only advances on actual reward distribution.

---

### Impact Explanation

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` in every block prevents all stakers from accumulating block rewards for the duration of the attack. Stakers' unclaimed yield stops growing. This constitutes **temporary (continuously renewable) freezing of unclaimed yield**, matching the High-severity impact tier.

---

### Likelihood Explanation

- No special role or privilege is required — `general_prerequisites` only rejects zero-address callers and paused state.
- The attacker only needs the address of any currently active staker with non-zero balance, which is publicly readable on-chain from the `stakers` vector and `staker_info` map.
- The per-block cost is one transaction. On Starknet, this is economically feasible for a motivated griever.
- The attack is immediately effective in the consensus rewards phase (V3, after `consensus_rewards_first_epoch` is set). [4](#0-3) 

---

### Recommendation

Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` and `is_pre_consensus` guard, so that a no-op call does not consume the block's reward slot:

```cairo
// Update last block rewards only when rewards are actually distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... proceed with reward calculation ...
```

Alternatively, add access control to `update_rewards` restricting callers to the designated consensus mechanism address, or use a per-staker `last_reward_block` map instead of a single global value.

---

### Proof of Concept

```
1. Consensus rewards phase is active (consensus_rewards_first_epoch has been set).
2. Attacker identifies any active staker address S with non-zero STRK balance
   (readable from stakers[] Vec and staker_own_balance_trace).
3. In block N, attacker submits:
       staking.update_rewards(staker_address: S, disable_rewards: true)
   - assert(N > last_reward_block) passes (last_reward_block < N).
   - last_reward_block is written to N.
   - Function returns early; no rewards distributed.
4. Consensus mechanism submits:
       staking.update_rewards(staker_address: S, disable_rewards: false)
   - assert(N > last_reward_block) FAILS (N > N is false).
   - Reverts with REWARDS_ALREADY_UPDATED.
5. No staker receives block rewards for block N.
6. Attacker repeats step 3 in every subsequent block.
   All stakers are denied block rewards for the duration of the attack.
``` [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
