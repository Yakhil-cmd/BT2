### Title
Missing Access Control on `update_rewards` Allows Permanent Freezing of All Staker Yield - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is specified as callable only by the Starkware sequencer, but the implementation contains no access control check. Any unprivileged caller can invoke it with `disable_rewards: true` every block, consuming the global `last_reward_block` slot without distributing rewards, permanently preventing all stakers from receiving consensus-phase yield.

### Finding Description
The protocol specification at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 enforces no such restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks: paused + caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);  // global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;   // exits without distributing anything
    }
    ...
``` [1](#0-0) 

`last_reward_block` is a **global** storage variable, not per-staker. [2](#0-1) 

Once it is written to the current block number, the guard `current_block_number > self.last_reward_block.read()` causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

The spec confirms this is intentional design — exactly one reward update per block, for exactly one staker, by the sequencer. [3](#0-2) 

`general_prerequisites()` only asserts the contract is not paused and the caller is not the zero address — it does **not** assert the caller is the sequencer. [4](#0-3) 

### Impact Explanation
An attacker calls `update_rewards(any_active_staker, disable_rewards: true)` once per block, indefinitely. Each call:
1. Passes all guards (contract unpaused, new block, valid staker with non-zero balance).
2. Writes `last_reward_block = current_block`.
3. Returns immediately without distributing any rewards (due to `disable_rewards: true`).

No legitimate sequencer call can succeed in that block. Repeated across every block, this **permanently freezes all unclaimed consensus-phase yield** for every staker and every delegation pool in the protocol. The attacker needs no funds and gains nothing — pure griefing.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- Entry path is fully permissionless: any EOA or contract can call `update_rewards`.
- The attacker only needs to know one valid active staker address, which is publicly observable from `NewStaker` events.
- Cost per block is a single transaction fee on Starknet (negligible).
- No flash loan, no capital, no privileged role required.
- The attack is sustainable indefinitely.

### Recommendation
Add a sequencer-only guard at the top of `update_rewards`, consistent with the spec:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer();   // <-- add this
    ...
```

Alternatively, restrict via the existing roles component (e.g., `self.roles.only_operator()`) and assign the sequencer address to that role at deployment.

### Proof of Concept
```
Block N:
  Attacker calls: staking.update_rewards(staker=<any_valid_staker>, disable_rewards=true)
  → general_prerequisites() passes (contract not paused, caller != zero)
  → assert!(block_N > last_reward_block) passes (new block)
  → last_reward_block := block_N
  → disable_rewards == true → return (no rewards distributed)

Block N+1:
  Sequencer attempts: staking.update_rewards(staker=<real_staker>, disable_rewards=false)
  → Attacker front-runs with same call (disable_rewards=true)
  → last_reward_block := block_N+1
  → Sequencer's call reverts: REWARDS_ALREADY_UPDATED

Repeat every block → all stakers permanently receive zero consensus rewards.
``` [5](#0-4) [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
