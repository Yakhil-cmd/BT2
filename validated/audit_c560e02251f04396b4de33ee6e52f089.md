### Title
Gateway Skips `__validate__` Signature Verification for Invoke Transactions When Deploy-Account Is In Mempool — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator unconditionally skips the `__validate__` entry-point call (and therefore all signature verification) for any Invoke transaction with `nonce = 1` submitted against an account whose on-chain nonce is `0`, provided *any* transaction from that account address already exists in the mempool or a recent block. Because the deploy-account transaction that seeds the mempool is publicly observable, an unprivileged attacker can submit an Invoke transaction carrying an arbitrary (invalid) signature for a victim's not-yet-deployed account and have it admitted to the mempool without any cryptographic check.

---

### Finding Description

**Relevant code — `skip_stateful_validations`:** [1](#0-0) 

The function returns `true` (skip validation) when all three conditions hold:
1. The transaction is an `Invoke`.
2. `tx.nonce() == Nonce(Felt::ONE)`.
3. `account_nonce == Nonce(Felt::ZERO)`.
4. `mempool_client.account_tx_in_pool_or_recent_block(sender_address)` returns `true`.

The returned boolean is used directly to set `validate: !skip_validate` in the `ExecutionFlags`: [2](#0-1) 

When `validate = false`, `StatefulValidator::perform_validations` returns immediately after `perform_pre_validation_stage` without ever calling `__validate__`: [3](#0-2) 

`perform_pre_validation_stage` only checks the nonce and fee bounds; it does **not** verify the signature: [4](#0-3) 

The mempool's `validate_tx` path also does not inspect the signature — it only checks nonce ordering and fee escalation: [5](#0-4) 

`account_tx_in_pool_or_recent_block` returns `true` for *any* transaction from the address, regardless of who submitted it: [6](#0-5) 

**Attack path:**

1. Victim broadcasts a `deploy_account` transaction for address `A`; it enters the mempool.
2. Attacker observes the mempool, learns address `A`.
3. Attacker crafts an `Invoke` with `sender_address = A`, `nonce = 1`, arbitrary `calldata`, and a garbage `signature`.
4. Gateway stateless checks pass (signature length, resource bounds, DA mode, etc.).
5. `extract_state_nonce_and_run_validations` reads `account_nonce = 0` from state.
6. `validate_nonce` passes because `0 ≤ 1 ≤ max_allowed_nonce_gap` (default gap ≥ 1, confirmed by the `should_skip_validation` test case): [7](#0-6) 

7. `skip_stateful_validations` returns `true` → `__validate__` is never called.
8. The attacker's unsigned Invoke is forwarded to the mempool and accepted.
9. When the victim subsequently submits their own valid Invoke with `nonce = 1`, the mempool rejects it as `DuplicateNonce`: [8](#0-7) 

10. The attacker can continuously re-submit to maintain the nonce-squatting DoS until the victim out-bids via fee escalation.

---

### Impact Explanation

An unprivileged attacker can inject an Invoke transaction carrying an invalid (or absent) signature into the mempool for any account whose `deploy_account` is publicly visible. This satisfies the **High** impact criterion:

> *Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.*

The concrete harm is nonce squatting: the victim's first post-deployment Invoke (nonce = 1) is blocked from entering the mempool, delaying or preventing its execution until the attacker's invalid transaction is eventually rejected by the blockifier and the victim resubmits.

---

### Likelihood Explanation

- The `deploy_account` transaction is broadcast publicly (mempool P2P propagation).
- The attacker needs no privileged access, no special contract, and no funds beyond the gas to submit the Invoke.
- The condition (`nonce = 1`, `account_nonce = 0`, any mempool entry) is deterministically satisfied for every new account deployment.
- The attack is repeatable: after the invalid Invoke is rejected at execution time, the attacker can immediately re-submit.

---

### Recommendation

Replace the coarse `account_tx_in_pool_or_recent_block` membership check with a check that the *specific* `deploy_account` transaction hash provided by the submitter is present in the mempool and belongs to the same sender address. Alternatively, require the `deploy_account` transaction hash to be co-signed or otherwise cryptographically bound to the Invoke, so that only the legitimate account owner can trigger the skip. At minimum, the skip should only be granted when the gateway can confirm that the pending deploy-account transaction was submitted by the same party (e.g., same IP/session, or by requiring the Invoke to carry a reference to the deploy-account tx hash that is verified against the mempool).

---

### Proof of Concept

```
1. Victim submits:
     deploy_account { class_hash=C, salt=S, constructor_calldata=[], nonce=0, signature=<valid> }
   → address A is deterministically computed; tx enters mempool.

2. Attacker queries mempool, learns address A.

3. Attacker submits:
     invoke { sender_address=A, nonce=1, calldata=[...], signature=[0xdead, 0xbeef] }

4. Gateway stateful path:
     account_nonce = get_nonce(A) = 0          // account not yet deployed
     validate_nonce(nonce=1, account_nonce=0)   // passes (gap ≥ 1)
     validate_by_mempool(...)                   // passes (no dup nonce yet)
     skip_stateful_validations(...)
       → nonce==1 && account_nonce==0 && account_tx_in_pool(A)==true
       → returns true
     run_validate_entry_point(skip_validate=true)
       → validate flag = false → __validate__ NOT called
     → transaction admitted to mempool

5. Victim submits:
     invoke { sender_address=A, nonce=1, calldata=[...], signature=<valid> }
   → mempool rejects: DuplicateNonce { address: A, nonce: 1 }

6. Attacker's invalid invoke is eventually executed by the blockifier:
     __validate__ runs → fails (wrong signature) → transaction rejected
   Victim must resubmit; attacker can repeat step 3 indefinitely.
```

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    pub fn perform_pre_validation_stage<S: State + StateReader>(
        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }
