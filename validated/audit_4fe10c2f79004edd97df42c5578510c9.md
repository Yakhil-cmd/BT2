### Title
Stale KV Permissions Survive Admin Rotation, Granting Old Admin Persistent `CanChangeTokenMapping` Access — (`x/cronos/keeper/permissions.go`, `x/cronos/keeper/msg_server.go`)

### Summary

When `CronosAdmin` is rotated via governance (`UpdateParams`), the old admin's explicitly-stored KV permissions are never cleared. Because `HasPermission` checks both the live `CronosAdmin` param **and** the KV-stored permission bits independently, the old admin retains any permission bit they granted themselves before rotation — indefinitely.

### Finding Description

`HasPermission` has two independent grant paths: [1](#0-0) 

1. **Admin path** (line 39): if the account string equals the current `CronosAdmin` param → grant all permissions.
2. **KV path** (lines 42–44): if the account's stored permission bits satisfy the check → grant permission.

`UpdatePermissions` lets the current admin write any permission bits for any address, including themselves: [2](#0-1) 

`SetParams` (called by `UpdateParams`) only overwrites the params blob; it performs no cleanup of the permissions KV store: [3](#0-2) 

There is no `DeletePermissions` or equivalent call anywhere in the keeper.

### Impact Explanation

After admin rotation, the old admin A retains `CanChangeTokenMapping` via their KV entry and can successfully call `UpdateTokenMapping`: [4](#0-3) 

This allows A to arbitrarily remap denom↔contract bindings — corrupting IBC voucher conversion, gravity bridge accounting, and CRC20/CRC21 token flows — even though governance explicitly revoked A's admin status.

### Likelihood Explanation

The scenario is realistic: a departing or malicious admin can pre-emptively call `UpdatePermissions{address: self, permissions: CanChangeTokenMapping}` before the governance proposal that rotates the admin takes effect. The window is any time while A is still admin. The KV entry then persists forever with no expiry or cleanup mechanism.

### Recommendation

One or more of the following mitigations should be applied:

1. **Clear old admin's KV permissions on rotation**: In `SetParams`, compare the incoming `CronosAdmin` with the current one; if changed, call `SetPermissions(ctx, oldAdmin, 0)`.
2. **Remove the KV path from `HasPermission` for the admin address**: Require that the admin always uses the admin path, and that delegated permissions are only valid for non-admin addresses. On rotation, the old admin's KV bits become irrelevant because they are no longer admin.
3. **Scope `UpdatePermissions` to prevent self-grant**: Disallow `msg.From == msg.Address` so the admin cannot pre-stage their own KV permissions.

### Proof of Concept

```go
// Unit test sketch (x/cronos/keeper/msg_server_test.go)
func TestStalePermissionsAfterAdminRotation(t *testing.T) {
    // 1. Set CronosAdmin = addrA
    params := types.DefaultParams()
    params.CronosAdmin = addrA.String()
    k.SetParams(ctx, params)

    // 2. Admin A grants themselves CanChangeTokenMapping in KV
    _, err := srv.UpdatePermissions(ctx, &types.MsgUpdatePermissions{
        From:        addrA.String(),
        Address:     addrA.String(),
        Permissions: keeper.CanChangeTokenMapping,
    })
    require.NoError(t, err)

    // 3. Governance rotates admin to addrB
    params.CronosAdmin = addrB.String()
    k.SetParams(ctx, params)

    // 4. Old admin A still has CanChangeTokenMapping via KV
    require.True(t, k.HasPermission(ctx, []sdk.AccAddress{addrA}, keeper.CanChangeTokenMapping))

    // 5. A can still call UpdateTokenMapping — should fail but doesn't
    _, err = srv.UpdateTokenMapping(ctx, &types.MsgUpdateTokenMapping{
        Sender: addrA.String(),
        // ... valid denom/contract fields ...
    })
    require.NoError(t, err) // BUG: should be ErrUnauthorized
}
```

```go
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
```

```go
func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
```

```go
	if err := params.Validate(); err != nil {

	bz := k.cdc.MustMarshal(&params)
	store.Set(types.ParamsKey, bz)

	return nil
}
```
