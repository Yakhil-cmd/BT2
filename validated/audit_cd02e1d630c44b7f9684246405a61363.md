The vulnerability is **real and valid**. Here is the full finding:

---

### Title
`OracleBase.register` Fee Bypass via `msg.value` Reuse in Inherited OpenZeppelin `Multicall` — (`smart-contracts-poc/contracts/oracles/providers/OracleBase.sol`)

---

### Summary

`OracleBase` inherits OpenZeppelin's `Multicall`, whose `multicall()` dispatches each sub-call via `functionDelegateCall`. Because `delegatecall` preserves the caller's `msg.value` unchanged across every sub-call, an attacker can send a single registration fee `F` and call `register()` N times in one `multicall` batch, passing the `msg.value >= registrationFee` guard every time while the contract receives only `F` ETH total.

---

### Finding Description

`OracleBase` is declared as:

```solidity
contract OracleBase is AccessControl, Multicall, IOffchainOracle {
``` [1](#0-0) 

The inherited OpenZeppelin `Multicall.multicall()` loops over calldata and dispatches each element with `Address.functionDelegateCall(address(this), data[i])`. `delegatecall` does **not** deduct or forward `msg.value` per sub-call; it simply exposes the original transaction's `msg.value` to every frame. The `register` function's only fee guard is:

```solidity
require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
``` [2](#0-1) 

There is no per-call accounting, no counter, and no deduction of already-consumed ETH. Every delegatecall'd `register` frame reads the same `msg.value` and passes the check independently.

---

### Impact Explanation

- The contract receives `F` ETH but grants N registrations, losing `(N-1)*F` in fee revenue per batch.
- The abuse-protection whitelist (`registeredPool`) is populated at `1/N` of the intended cost, undermining the economic deterrent that is the system's primary defence against oracle-read abuse.
- `registrationFee` defaults to 1 wei but is configurable via `setRegistrationFee` to any meaningful amount; the bypass works regardless of the fee level.

This is a direct loss of protocol fee revenue, qualifying as Medium under Sherlock thresholds (the fee is configurable and the bypass is unconditional).

---

### Likelihood Explanation

The attack requires no privilege. Any address can call `multicall` with a batch of `register` calls. The only prerequisite is that the attacker supplies a valid `factory` (approved by ADMIN) and valid pool addresses recognised by that factory — both of which are normal preconditions for any legitimate registrant. The attack is therefore available to every pool deployer and is trivially repeatable.

---

### Recommendation

Override `multicall` in `OracleBase` (or in a shared base) to reject payable batches, or replace the per-call `msg.value` check with an explicit ETH accounting pattern that deducts the fee from a running balance:

```solidity
// Option A: disallow ETH in multicall
function multicall(bytes[] calldata data) public payable override returns (bytes[] memory) {
    require(msg.value == 0, "no ETH in multicall");
    return super.multicall(data);
}
```

Or restructure `register` to accept an explicit `uint256 payment` parameter pulled from a pre-deposited balance, so `msg.value` is not the fee signal inside delegatecall frames.

---

### Proof of Concept

```solidity
// Foundry test sketch
function test_multicall_register_fee_bypass() public {
    uint256 fee = 1 ether;
    oracle.setRegistrationFee(fee);          // ADMIN sets meaningful fee

    bytes[] memory calls = new bytes[](2);
    calls[0] = abi.encodeCall(oracle.register, (feedId, pool1, factory));
    calls[1] = abi.encodeCall(oracle.register, (feedId, pool2, factory));

    // Send only 1 × fee, get 2 registrations
    oracle.multicall{value: fee}(calls);

    assertTrue(oracle.registeredPool(feedId, pool1));
    assertTrue(oracle.registeredPool(feedId, pool2));
    assertEq(address(oracle).balance, fee);   // received 1 ETH, not 2 ETH
}
```

Both `register` frames see `msg.value == 1 ether`, both pass the guard, both pools are whitelisted, and the contract balance is `1 ether` instead of `2 ether`. The protocol loses `(N-1) * fee` per batch.
