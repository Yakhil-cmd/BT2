### Title
`MsgTurnBridge` Handler Has No Permission Check and No Implementation — Any Caller Succeeds, Bridge Circuit Breaker Is Non-Functional - (File: x/cronos/keeper/msg_server.go)

### Summary

The `TurnBridge` gRPC message handler in `x/cronos/keeper/msg_server.go` is a stub that returns `nil, nil` unconditionally. It performs no authorization check and makes no state change. Any unprivileged address can invoke `MsgTurnBridge` and receive a success response, bypassing the `CanTurnBridge` permission system entirely. Simultaneously, the intended emergency circuit breaker for the Gravity bridge is permanently non-functional.

### Finding Description

`MsgTurnBridge` is defined in the proto service and is intended to require the `CanTurnBridge` permission (bit `2` in the permission bitmask), as documented in ADR-009 and enforced for the analogous `MsgUpdateTokenMapping` handler. The permissions system is fully implemented: `CanTurnBridge` is declared in `x/cronos/keeper/permissions.go`, `HasPermission` checks it correctly, and `UpdateTokenMapping` uses it properly.

However, the `TurnBridge` handler itself is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Compare with `UpdateTokenMapping`, which correctly gates on `HasPermission`:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [2](#0-1) 

The permission constants and `HasPermission` logic are fully in place: [3](#0-2) [4](#0-3) 

The proto definition confirms `MsgTurnBridge` is a live, registered RPC endpoint: [5](#0-4) 

The gRPC handler is wired and reachable: [6](#0-5) 

### Impact Explanation

**Auth bypass (High):** Any unprivileged address can submit a `MsgTurnBridge` transaction and receive a success response (`code == 0`). The `CanTurnBridge` permission check — which is supposed to restrict this to the `CronosAdmin` or explicitly permissioned addresses — is entirely absent. This directly matches the "Bypass of Cronos admin, governance authority, permission … authorization checks" High impact criterion.

**Non-functional circuit breaker (High):** The Gravity bridge circuit breaker is permanently broken. Even the `CronosAdmin` or a permissioned address cannot actually disable the bridge via `MsgTurnBridge`, because the handler makes no state change. ADR-009 explicitly identifies `MsgTurnBridge` as a critical emergency stop for the Gravity module. This matches the "Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows" High impact criterion.

### Likelihood Explanation

The entry path is fully unprivileged: any address can broadcast a `MsgTurnBridge` transaction on-chain. No special keys, validator access, or governance participation is required. The gRPC endpoint is registered and reachable. The integration test `test_gravity_turn_bridge` confirms the message is exercised in production scenarios. [7](#0-6) 

### Recommendation

Implement `TurnBridge` with the same pattern as `UpdateTokenMapping`:

1. Add a `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` guard at the top of the handler.
2. Implement the actual bridge-enable/disable state change (store the bridge-enabled flag in module params or a dedicated KV key and check it in the Gravity end-blocker).
3. Emit an appropriate event.

### Proof of Concept

1. Any address (e.g., `signer1`) submits:
   ```
   cronosd tx cronos turn-bridge false --from signer1
   ```
2. The transaction is accepted on-chain with `code == 0` because `TurnBridge` returns `nil, nil` with no auth check.
3. The bridge state is unchanged (no state write occurs), so the circuit breaker has no effect.
4. Simultaneously, the `CronosAdmin` calling the same message also gets `code == 0` but the bridge is still not disabled — the emergency stop is permanently inoperative. [1](#0-0) [8](#0-7)

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
