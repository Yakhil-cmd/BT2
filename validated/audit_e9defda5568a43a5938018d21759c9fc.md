### Title
`TurnBridge` Is a No-Op Stub — Bridge Circuit Breaker Completely Non-Functional - (File: x/cronos/keeper/msg_server.go)

### Summary
The `MsgTurnBridge` handler in `msg_server.go` is an unimplemented stub that returns `nil, nil` without performing any state change or permission check. As a result, the bridge circuit breaker is entirely non-functional: no bridge-enabled flag is ever written, no `CanTurnBridge` permission is enforced, and every bridge path (EVM hooks, IBC middleware, `ConvertVouchers`, `TransferTokens`) continues to operate unconditionally. An unprivileged user can bridge assets freely even after an authorized admin has called `TurnBridge(false)` in an emergency.

### Finding Description
The Cronos permission system defines a `CanTurnBridge` bit-flag and the `MsgTurnBridge` message type, both intended to give authorized operators an emergency circuit breaker over bridge operations. [1](#0-0) 

The `TurnBridge` gRPC handler, however, is a stub: [2](#0-1) 

It neither:
1. Checks whether the caller holds the `CanTurnBridge` permission (so any address can call it), nor
2. Writes any bridge-enabled state to the KV store.

Because no state is ever written, every downstream bridge path has nothing to check. `ConvertVouchersToEvmCoins` and `IbcTransferCoins` execute unconditionally: [3](#0-2) 

The IBC conversion middleware's `OnRecvPacket` calls `ConvertVouchersToEvmCoins` with no bridge-enabled guard: [4](#0-3) 

The EVM log handlers (`SendToIbcHandler`, `SendToIbcV2Handler`) similarly perform no bridge-enabled check before initiating IBC transfers: [5](#0-4) 

The contrast with `UpdateTokenMapping`, which correctly enforces its permission, confirms the pattern: [6](#0-5) 

The integration test `test_gravity_turn_bridge` explicitly expects bridge operations to fail after `turn_bridge("false")` is called, confirming the intended behavior is absent: [7](#0-6) 

### Impact Explanation
**High.** The bridge circuit breaker — a core admin/governance safety mechanism — is completely bypassed. In an emergency (e.g., an active bridge exploit draining CRC20/CRC21 or IBC voucher balances), an authorized operator calling `TurnBridge(false)` achieves nothing. Unprivileged users continue to execute `send_to_ibc`, `send_to_evm_chain`, `ConvertVouchers`, and `TransferTokens` without restriction, enabling continued unauthorized bridge/conversion flows and potential asset loss. This matches the allowed High impact: "Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."

### Likelihood Explanation
The `CanTurnBridge` permission is already granted to trusted operators in production deployments. The only trigger required is an emergency scenario where an operator calls `TurnBridge(false)` — a call that succeeds silently and does nothing. Any unprivileged user who continues normal bridge usage after that call exploits the gap without any special knowledge or privilege.

### Recommendation
Implement `TurnBridge` fully:
1. Check that `msg.Sender` holds the `CanTurnBridge` permission via `HasPermission`.
2. Persist a bridge-enabled boolean to the KV store (e.g., add a `BridgeEnabled` field to `Params` or a dedicated store key).
3. Add a guard at the top of `ConvertVouchersToEvmCoins`, `IbcTransferCoins`, and the EVM log handlers (`SendToIbcHandler.handle`, `SendToIbcV2Handler.Handle`) that returns an error when the bridge is disabled.
4. Add the same guard in `IBCConversionModule.OnRecvPacket` before calling `ConvertVouchersToEvmCoins`.

### Proof of Concept
1. Operator (holding `CanTurnBridge`) submits `MsgTurnBridge{Sender: operator, Enable: false}`.
2. Handler executes `return nil, nil` — tx succeeds, no state written.
3. Unprivileged user calls `MsgConvertVouchers` or triggers `send_to_ibc` from a CRC20/CRC21 contract.
4. `ConvertVouchersToEvmCoins` / `SendToIbcHandler.handle` execute without any bridge-enabled check — bridge operation succeeds.
5. Bridge remains fully operational despite the operator's emergency disable attempt.

func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
	}
	defer func() {
		for _, a := range coins {
			if a.Amount.IsInt64() {
				telemetry.SetGaugeWithLabels(
					[]string{"tx", "msg", "ConvertVouchersToEvmCoins"},
					float32(a.Amount.Int64()),
					[]metrics.Label{telemetry.NewLabel("denom", a.Denom)},
				)
			}
		}
	}()
	return nil
}

// OnRecvPacket implements the IBCModule interface.
func (im IBCConversionModule) OnRecvPacket(
	ctx sdk.Context,
	channelVersion string,
	packet channeltypes.Packet,
	relayer sdk.AccAddress,
) exported.Acknowledgement {
	cacheCtx, commit := ctx.CacheContext()
	ack := im.app.OnRecvPacket(cacheCtx, channelVersion, packet, relayer)
	if !ack.Success() {
		// Underlying transfer failed: discard cacheCtx writes and return the
		// failure ack. Committing would persist a half-applied transfer.
		return ack
	}
	data, err := transferTypes.UnmarshalPacketData(packet.GetData(), channelVersion, "")
	if err != nil {
		return channeltypes.NewErrorAcknowledgement(errors.Wrap(sdkerrors.ErrUnknownRequest,
			"cannot unmarshal ICS-20 transfer packet data in middleware"))
	}
	denom := im.getIbcDenomFromPacketAndData(packet, data.Token)
	if im.canBeConverted(cacheCtx, denom) {
		transferAmount, ok := sdkmath.NewIntFromString(data.Token.Amount)
		if !ok {
			return channeltypes.NewErrorAcknowledgement(errors.Wrapf(
				transferTypes.ErrInvalidAmount,
				"unable to parse transfer amount (%s) into sdk.Int in middleware",
				data.Token.Amount,
			))
		}
		token := sdk.NewCoin(denom, transferAmount)
		if err := im.cronoskeeper.ConvertVouchersToEvmCoins(cacheCtx, data.Receiver, sdk.NewCoins(token)); err != nil {
			im.cronoskeeper.Logger(ctx).Error(
				"failed to convert vouchers on recv",
				"denom", denom,
				"receiver", data.Receiver,
				"error", err,
			)
			return channeltypes.NewErrorAcknowledgement(err)
		}
	}
	commit()
	return ack
}

func (h SendToIbcHandler) handle(
	ctx sdk.Context,
	contract common.Address,
	senderAddress common.Address,
	recipient string,
	amountInt *big.Int,
	id *big.Int,
) error {
	denom, found := h.cronosKeeper.GetDenomByContract(ctx, contract)
	if !found {
		return fmt.Errorf("contract %s is not connected to native token", contract)
	}

	if !types.IsValidIBCDenom(denom) && !types.IsValidCronosDenom(denom) {
		return fmt.Errorf("the native token associated with the contract %s is neither an ibc voucher or a cronos token", contract)
	}

	contractAddr := sdk.AccAddress(contract.Bytes())
	sender := sdk.AccAddress(senderAddress.Bytes())
	amount := sdkmath.NewIntFromBigInt(amountInt)
	coins := sdk.NewCoins(sdk.NewCoin(denom, amount))

	var err error
	if types.IsSourceCoin(denom) {
		// it is a source token, we need to mint coins
		if err = h.bankKeeper.MintCoins(ctx, types.ModuleName, coins); err != nil {
			return err
		}
		// send the coin to the user
		if err = h.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, sender, coins); err != nil {
			return err
		}
	} else {
		// First, transfer IBC coin to user so that he will be the refunded address if transfer fails
		if err = h.bankKeeper.SendCoins(ctx, contractAddr, sender, coins); err != nil {
			return err
		}
	}

	channelId := ""
	if id != nil {
		channelId = "channel-" + id.String()
	}
	// Initiate IBC transfer from sender account
	if err = h.cronosKeeper.IbcTransferCoins(ctx, sender.String(), recipient, coins, channelId); err != nil {
		return err
	}
	return nil
