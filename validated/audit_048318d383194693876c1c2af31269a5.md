### Title
Presigning Participant Set Not Enforced During Signing in Robust ECDSA — (File: `src/ecdsa/robust_ecdsa/sign.rs`)

---

### Summary

The robust ECDSA `sign` function accepts a caller-supplied `participants` list without validating it against the participant set used during `presign`. The `RerandomizedPresignOutput` struct carries no record of the presigning participant set. A malicious coordinator can therefore present different participant lists to different signers in the same signing session, enabling the documented split-view attack that can extract the aggregate secret key.
