### Title
Gateway L2 Gas Price Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Systematic Admission/Rejection Mismatch - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check validates a transaction's `max_price_per_unit` against the **current block's** `l2_gas_price` (the price that was used for transactions in the block that was just committed). The correct reference is `next_l2_gas_price` — the EIP-1559-derived price stored in that same block header that the batcher will use when executing the transaction in the **next** block. The code itself contains a TODO acknowledging this: `// TODO(Arni): getnext_l2_gas_price from the block header.` With `min_gas_price_percentage = 100` in production, the threshold is exactly `previous_block.l2_gas_price`, creating a systematic mismatch between what the gateway admits and what the batcher will accept.

### Finding Description

In `validate_resource_bounds` the gateway reads:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;          // ← price OF the previous block
``` [1](#0-0) 

It then rejects any transaction whose `l2_gas.max_price_per_unit` is below `previous_block.l2_gas_price * (min_gas_price_percentage / 100)`. [2](#0-1) 

The production config sets `min_gas_price_percentage = 100`, so the threshold equals `previous_block.l2_gas_price` exactly. [3](#0-2) 

However, `BlockHeaderWithoutHash` carries two distinct price fields:

```
l2_gas_price      – the price used for transactions IN this block
next_l2_gas_price – the EIP-1559-adjusted price FOR the next block
``` [4](#0-3) 

The batcher receives `propose_block_input.block_info` whose `l2_gas_price` is set to `next_l2_gas_price` from the previous block header (i.e., the price the batcher will actually use for execution): [5](#0-4) 

The EIP-1559 formula in `calculate_next_base_gas_price` adjusts the price up when `gas_used > gas_target` and down when `gas_used < gas_target`: [6](#0-5) 

This creates two failure modes:

**Mode A — Gas price rising (blocks are full):**
`next_l2_gas_price > previous_block.l2_gas_price`. A transaction with `max_price_per_unit = previous_block.l2_gas_price` passes the gateway check but fails `check_fee_bounds` in the blockifier when the batcher executes it at `next_l2_gas_price`. The gateway has admitted a transaction that will be dropped during block building. [7](#0-6) 

**Mode B — Gas price falling (blocks are underfull):**
`next_l2_gas_price < previous_block.l2_gas_price`. A transaction with `max_price_per_unit = next_l2_gas_price` would be perfectly valid for the next block but is rejected by the gateway because it is below the stale threshold. The gateway has rejected a valid transaction.

### Impact Explanation

**Mode A** causes the gateway/mempool to admit transactions that will fail during sequencing — matching the "Mempool/gateway/RPC admission accepts invalid transactions … before sequencing" High impact criterion. The batcher wastes execution resources on transactions that cannot be included, and the mempool accumulates stale entries.

**Mode B** causes the gateway to reject transactions that are valid for the next block — matching "rejects valid transactions before sequencing." Users whose wallets set `max_price_per_unit` to exactly the current `next_l2_gas_price` (the correct value) are systematically denied admission during any period of falling gas prices.

### Likelihood Explanation

The L2 gas price changes every block under EIP-1559. Any block that is not exactly at the gas target produces a `next_l2_gas_price ≠ l2_gas_price`. This is the normal operating condition of the network. The production config (`min_gas_price_percentage = 100`) maximises the mismatch window. No special attacker capability is required; the bug is triggered by ordinary network load variation.

### Recommendation

Replace the `l2_gas_price` field read with `next_l2_gas_price` from the block header. The `GatewayFixedBlockStateReader` trait needs to expose this field (it is already stored in `BlockHeaderWithoutHash.next_l2_gas_price` and serialised in the storage layer): [8](#0-7) 

The fix in `validate_resource_bounds` should read:

```rust
// Correct: use the price FOR the next block, not the price OF the previous block.
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_header()   // new method returning BlockHeaderWithoutHash
    .await?
    .next_l2_gas_price;
```

### Proof of Concept

1. Observe the previous committed block has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block was full).
2. Submit an `InvokeV3` transaction with `AllResourceBounds { l2_gas: { max_price_per_unit: P, … }, … }`.
3. Gateway `validate_resource_bounds` computes threshold = `P * 100% = P`; transaction passes (`P >= P`).
4. Transaction enters the mempool.
5. Batcher starts building the next block with `block_info.gas_prices.strk_gas_prices.l2_gas_price = P'`.
6. Blockifier `check_fee_bounds` evaluates `resource_bounds.max_price_per_unit (P) < actual_gas_price (P')` → `MaxGasPriceTooLow` error.
7. Transaction is dropped from the block despite having passed gateway admission.

The reverse (Mode B) is symmetric: set `max_price_per_unit = P'` when `P' < P`; the gateway rejects it with `GAS_PRICE_TOO_LOW` even though the batcher would have accepted it.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
