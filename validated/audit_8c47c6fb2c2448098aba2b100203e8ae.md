### Title
Signing Approval Bypass via `checkNFTOwnership` Using `sign_message_by_id` Without User Consent — (`packages/gui/src/electron/api/checkNFTOwnership.ts`)

### Summary
`checkNFTOwnership` uses `sign_message_by_id` as an ownership probe, producing real cryptographic signatures without user consent whenever a user opens an offer containing NFT IDs. This bypasses the signing approval guards the codebase explicitly enforces for this command.

### Finding Description
`checkNFTOwnership` [5](#0-4)  calls `sign_message_by_id` with a fixed message `'x'` and discards the resulting `pubkey`/`signature`/`latest_coin_id`, using only `response.success` to determine ownership. This is called automatically for every NFT ID in an offer via `useResolveNFTOffer` [6](#0-5)  with no user prompt. The command is listed in `BlockedCommands` [3](#0-2)  and `SIGN_COMMANDS` [4](#0-3)  precisely because it is a privileged signing operation — but `checkNFTOwnership` bypasses both guards by calling `sendCommand` directly.

### Impact Explanation
Real cryptographic signatures are produced by the wallet daemon without user awareness or consent, triggered by opening any offer file containing NFT IDs. The signing approval invariant is violated. The signatures are discarded by the GUI and are not directly accessible to a remote attacker through this path, which limits the exploitability compared to the report's framing. The direct impact is the signing approval bypass itself.

### Likelihood Explanation
Any user who opens a crafted offer file (a normal, expected workflow) triggers this for every NFT ID in the offer. No special permissions or local access are required to trigger the signing; local access would be required to observe the signatures or ownership results.

### Recommendation
Replace `sign_message_by_id` with a read-only ownership check — e.g., `nft_get_nfts` filtered by wallet, or a dedicated non-signing RPC — to determine NFT ownership without producing cryptographic signatures.

### Proof of Concept
Mock `sendCommand` in a test environment, construct an `OfferSummaryRecord` with N singleton NFT IDs, render a component that uses `useResolveNFTOffer`, and assert that `sendCommand` is called with `sign_message_by_id` and `message: 'x'` for each NFT ID, with no user confirmation dialog shown at any point.
