### Title
Missing `msg_hash == 0` Guard in OT-Based ECDSA `sign()` Enables Private Key Extraction via Presignature Reuse - (File: `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

The OT-based ECDSA `sign()` function accepts `msg_hash = 0` without rejection. The sibling robust ECDSA `sign()` function explicitly rejects this input to prevent split-view and nonce-reuse attacks. A malicious coordinator who can cause presignature reuse — which the library does not prevent — can exploit the missing guard to extract the aggregate private key.

---

### Finding Description

`src/ecdsa/robust_ecdsa/sign.rs` lines 91–95 enforce:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
    ));
}
``` [1](#0-0) 

The OT-based ECDSA `sign()` function at `src/ecdsa/ot_based_ecdsa/sign.rs` lines 22–76 performs participant-set and threshold checks but contains **no equivalent `msg_hash == 0` rejection**: [2](#0-1) 

The function's own docstring acknowledges the risk: *"Allowing the signing of arbitrary scalars is a security risk."* [3](#0-2) 

The `RerandomizedPresignOutput` struct is plain data with no consumption tracking, so the library provides no mechanism to prevent a participant from using the same `PresignOutput` in two separate signing sessions: [4](#0-3) 

---

### Impact Explanation

**Critical — private key extraction.**

The OT-based ECDSA signing equation is `s_i = h · k_i + R_x · σ_i`, where `σ_i` is the participant's share of `σ = k · x`.

**Attack with presignature reuse across two sessions (same raw `PresignOutput`, different `msg_hash`):**

Let `δ₀ = HKDF(entropy, pk, 0, R, participants)` and `δ_h = HKDF(entropy, pk, h, R, participants)` be the rerandomization scalars for the two sessions.

- **Session 1 (`h = 0`):** Each participant computes `s_i⁰ = R₀_x · σ_i⁰` where `σ_i⁰ = (σ_i + ε·k_i)/δ₀`. The coordinator receives `s_i⁰` and can compute `σ_i + ε·k_i = s_i⁰ · δ₀ / R₀_x`.

- **Session 2 (`h ≠ 0`, same raw presignature):** Each participant computes `s_i^h = h · k_i/δ_h + R_h_x · (σ_i + ε·k_i)/δ_h`. The coordinator already knows `σ_i + ε·k_i`, so it solves: `k_i = (s_i^h · δ_h − R_h_x · s_i⁰ · δ₀ / R₀_x) / h`.

With `k_i` and `σ_i + ε·k_i` known for every participant, the coordinator reconstructs the aggregate nonce `k` and `σ = k·x` via Lagrange interpolation, then computes `x = σ / k` — the aggregate private key.

This falls squarely within the allowed Critical impact: *"Extraction, reconstruction, or disclosure of private signing shares, aggregate secret material, presign secrets, nonce material, or confidential derived secrets."*

---

### Likelihood Explanation

The coordinator is an explicitly allowed attacker profile per `RESEARCHER.md`. The coordinator controls which `msg_hash` is presented to each participant and can initiate multiple signing sessions. The library provides no presignature-consumption tracking, so participants have no in-library mechanism to detect reuse. A coordinator claiming a first session "failed" and requesting a retry with the same presignature is a realistic operational scenario. [5](#0-4) 

---

### Recommendation

Add the same `msg_hash == 0` guard to `ot_based_ecdsa::sign::sign()` that already exists in `robust_ecdsa::sign::sign()`:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0: signing with a zero hash enables private key extraction via presignature reuse".to_string(),
    ));
}
```

This should be inserted after the threshold check at line 63 and before the protocol is constructed. [6](#0-5) 

---

### Proof of Concept

1. Run DKG to produce key shares for participants `P₁…Pₙ`.
2. Run presigning to produce a `PresignOutput` for each participant.
3. **Session 1:** Coordinator presents `msg_hash = 0` and `entropy = E` to all participants. Each participant calls `RerandomizedPresignOutput::rerandomize_presign` with `msg_hash = [0u8; 32]`, then calls `ot_based_ecdsa::sign::sign(…, msg_hash = Scalar::ZERO)`. Because no `is_zero()` check exists, the protocol proceeds. The coordinator collects `s_i⁰ = R₀_x · σ_i⁰` from every participant.
4. **Session 2:** Coordinator presents `msg_hash = h ≠ 0` and the **same** `entropy = E` to all participants, who reuse the same raw `PresignOutput`. Each participant calls `sign()` with `msg_hash = h`. The coordinator collects `s_i^h`.
5. For each participant `i`, the coordinator computes:
   - `σ_i + ε·k_i = s_i⁰ · δ₀ / R₀_x`
   - `k_i = (s_i^h · δ_h − R_h_x · (σ_i + ε·k_i)) / h`
6. Lagrange-interpolate `k` and `σ` from the per-participant shares; compute `x = σ / k`.

The robust ECDSA variant would have rejected step 3 at initialization with `BadParameters("msg_hash cannot be 0…")`. [7](#0-6) [8](#0-7)

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
