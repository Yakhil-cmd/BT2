### Title
Missing Maximum Commission Guard in Delegation Entry Points Allows Staker to Front-Run Delegators and Steal Yield - (File: src/pool/pool.cairo)

---

### Summary

`enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool` in `src/pool/pool.cairo` accept no `max_commission` parameter. A staker who holds an active `CommissionCommitment` can increase their commission to any value up to `max_commission` at any time. By front-running a delegator's entry transaction with a commission increase, the staker forces the delegator into a pool whose commission is far higher than the one the delegator observed, permanently reducing the delegator's yield share.

---

### Finding Description

The three delegation entry points in `Pool` take only `reward_address` / `amount` / destination identifiers:

```cairo
fn enter_delegation_pool(
    ref self: ContractState, reward_address: ContractAddress, amount: Amount,
) { … }

fn add_to_delegation_pool(
    ref self: ContractState, pool_member: ContractAddress, amount: Amount,
) -> Amount { … }

fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
) -> Amount { … }
```

None of them accept a `max_commission` the caller is willing to tolerate. [1](#0-0) [2](#0-1) [3](#0-2) 

Meanwhile, `update_commission` in the staking contract explicitly permits a commission **increase** whenever an active `CommissionCommitment` exists:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, …);
    assert!(commission != old_commission, …);   // any change ≤ max is fine
}
``` [4](#0-3) 

A staker can therefore:
1. Call `set_commission_commitment(max_commission=9000, expiration_epoch=future)` to arm the ability to raise commission up to 90 %.
2. Observe a delegator's pending `enter_delegation_pool` transaction (or simply race it in the same block).
3. Call `set_commission(commission=9000)` — valid because the commitment is active.
4. The delegator's transaction executes and they are locked into a 90 % commission pool, even though they observed 10 % when they signed.

The same race applies to `add_to_delegation_pool` (existing member topping up) and `switch_delegation_pool` (member migrating to a new pool). [5](#0-4) [6](#0-5) 

---

### Impact Explanation

Every epoch the delegator remains in the pool, the staker's `split_rewards_with_commission` takes the inflated commission fraction from the delegator's share of pool rewards. The delegator receives a fraction `(1 - commission/10000)` of what they would have earned at the advertised rate. At 90 % commission the delegator loses 80 percentage points of yield per epoch for as long as they stay in the pool (until they notice and exit, which itself requires waiting through the `exit_wait_window`). This constitutes **theft of unclaimed yield** — a High-severity impact under the allowed scope. [7](#0-6) 

---

### Likelihood Explanation

- The attack requires only that the staker has previously called `set_commission_commitment` with a high `max_commission`. This is a single permissionless transaction available to any staker.
- On Starknet the sequencer orders transactions; a staker who controls their own node or monitors the mempool can reliably insert the commission-raise transaction before the delegator's entry transaction in the same block.
- Even without strict front-running, a staker can raise commission between the moment a delegator decides to delegate (off-chain) and the moment the transaction is included, because there is no on-chain slippage guard.
- The `CommissionCommitment` is visible on-chain, but delegators have no way to atomically assert "commission must be ≤ X when I enter."

---

### Recommendation

Add a `max_commission: Commission` parameter to `enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool`. Before transferring funds, assert the pool's current commission does not exceed the caller's stated maximum:

```cairo
fn enter_delegation_pool(
    ref self: ContractState,
    reward_address: ContractAddress,
    amount: Amount,
+   max_commission: Commission,
) {
+   let current_commission = self.get_commission_from_staking_contract();
+   assert!(current_commission <= max_commission, "Commission exceeds max");
    …
}
```

Apply the same guard in `add_to_delegation_pool` and `switch_delegation_pool`.

---

### Proof of Concept

1. Staker deploys pool with commission = 1000 (10 %).
2. Staker calls `set_commission_commitment(max_commission=9000, expiration_epoch=current+10)`.
3. Delegator signs `enter_delegation_pool(reward_address, amount=1_000_000)` seeing commission = 10 %.
4. Before the delegator's tx is sequenced, staker calls `set_commission(commission=9000)`. This succeeds because `9000 <= max_commission` and `9000 != 1000`. [8](#0-7) 
5. Delegator's `enter_delegation_pool` executes. No commission check exists in the function body. [9](#0-8) 
6. Each subsequent epoch, `split_rewards_with_commission` deducts 90 % of the delegator's pool rewards as commission, forwarding only 10 % to the pool and 90 % to the staker's `unclaimed_rewards_own`. [7](#0-6) 
7. Delegator loses the majority of their yield until they notice and complete the full `exit_delegation_pool_intent` → wait window → `exit_delegation_pool_action` cycle. [10](#0-9)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }
