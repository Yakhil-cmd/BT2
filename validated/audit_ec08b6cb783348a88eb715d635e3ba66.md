### Title
Wrong Royalty Total Displayed in WalletConnect Offer Confirmation Due to `splitAmount` Division by NFT Count - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary
In `parseCommandDisplay.ts`, the `formatAmountWithRoyalties` function divides the fungible amount by the **count** of NFTs (`royaltyPercentages.length`) before computing each NFT's royalty contribution. This causes the `amountWithRoyalties` value shown in the WalletConnect offer-confirmation dialog to be systematically understated whenever an offer involves multiple NFTs with royalties. The user approves believing they will pay less than the blockchain will actually deduct.

### Finding Description

The function `formatAmountWithRoyalties` at lines 354–375 of `parseCommandDisplay.ts` computes the displayed royalty total as follows:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // BUG: divides by NFT count
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

The correct formula is to apply each NFT's royalty percentage to the **full** fungible amount, not to `amount / N`. The current code effectively computes:

```
royaltyAmount = Σ( (amount / N) × royalty_i )
             = (amount / N) × Σ(royalty_i)
```

whereas the correct on-chain calculation is:

```
royaltyAmount = Σ( amount × royalty_i )
```

**Concrete example** (confirmed by the existing test at lines 251–335):

| Scenario | Displayed total | Correct total |
|---|---|---|
| 0.0001 XCH, NFT1 royalty=5%, NFT2 royalty=0.1% | 0.00010255 XCH | 0.0001051 XCH |

The test itself encodes the buggy value as the expected output, confirming the defect is present and untested-against-correct-behavior. [2](#0-1) 

This display value is produced by `walletDeltaToDisplay` → `withRoyaltyTotals` → `formatAmountWithRoyalties` and is surfaced in the WalletConnect confirmation dialog for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` commands. [3](#0-2) [4](#0-3) 

### Impact Explanation

The Chia wallet daemon computes royalties correctly on-chain (full amount × each royalty percentage). The GUI confirmation dialog, however, shows the understated `amountWithRoyalties`. A user who relies on the displayed total to decide whether to approve the offer will consent to paying less than the blockchain actually deducts. With multiple high-royalty NFTs, the gap can be significant. This constitutes **corruption of WalletConnect offer state that causes a user to approve the wrong amount**, matching the High impact category.

### Likelihood Explanation

Any WalletConnect-connected dApp can send a `chia_wallet.take_offer` payload containing an offer with two or more NFTs that carry royalties. No special privileges, leaked keys, or social engineering are required beyond a normal WalletConnect session. The bug is triggered automatically by the multi-NFT code path.

### Recommendation

Remove the `splitAmount` division. Each NFT's royalty must be computed against the full fungible `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

The corresponding test expectation at line 320 of `parseCommandDisplay.test.ts` must also be updated to reflect the correct value (`0.0001051` instead of `0.00010255`). [5](#0-4) 

### Proof of Concept

1. Establish a WalletConnect session with the Chia GUI.
2. Send `chia_wallet.take_offer` with an offer string that encodes:
   - Two NFTs offered, each with a royalty (e.g., 5% and 10%).
   - 1 XCH requested.
3. Observe the confirmation dialog shows `amountWithRoyalties ≈ 1.075 XCH` (split-based, wrong).
4. Approve the offer.
5. Observe the actual blockchain transaction deducts `1.15 XCH` (correct on-chain royalties).

The discrepancy grows linearly with the number of NFTs and their royalty percentages, giving a malicious dApp a reliable mechanism to cause users to approve offers that cost materially more than displayed.

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }
