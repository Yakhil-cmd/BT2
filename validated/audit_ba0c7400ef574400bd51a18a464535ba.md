Audit Report

## Title
SNS Tokens Permanently Locked in Swap Canister on Aborted Swap - (File: rs/sns/swap/src/swap.rs)

## Summary
When an SNS decentralization swap is aborted, SNS tokens pre-loaded into the swap canister's SNS ledger account are never returned to the SNS governance treasury. The `finalize_inner` function returns early after restoring dapp controllers without sweeping SNS tokens back, and no recovery function exists in the swap canister. Since the swap canister is eventually deleted per the documented lifecycle, these tokens are permanently lost.

## Finding Description
The root cause is a missing SNS token return step in the ABORTED finalization path of `finalize_inner`.

**Code path:**

`finalize_inner` (rs/sns/swap/src/swap.rs, L1544–1624) executes the following steps on abort:
1. `sweep_icp` — returns ICP to buyers. [1](#0-0) 
2. `settle_neurons_fund_participation` — refunds Neurons' Fund maturity. [2](#0-1) 
3. `should_restore_dapp_control()` returns `true` (because `lifecycle == Aborted`), restores dapp controllers, and **returns early at line 1583**. [3](#0-2) 

The `sweep_sns` call only exists after this early return, in the COMMITTED-only path. [4](#0-3) 

`should_restore_dapp_control` is confirmed to trigger exclusively on `Lifecycle::Aborted`: [5](#0-4) 

`sweep_sns` itself also guards against being called outside COMMITTED, so it cannot be repurposed for the abort path: [6](#0-5) 

No `sweep_sns_tokens_back` or equivalent function exists anywhere in the swap source. The proto documentation states: *"The swap canister can be deleted when all tokens registered with the swap canister have been disbursed to their rightful owners"* — but the ABORTED path only mentions returning ICP, not SNS tokens. [7](#0-6) 

The lifecycle diagram explicitly shows `ABORTED -> <DELETED>`, meaning the swap canister (and its locked SNS token balance) is eventually destroyed: [8](#0-7) 

**Why existing checks fail:** The swap canister holds the SNS tokens in its own SNS ledger account. Only the swap canister can initiate a transfer from that account. After finalization, the swap canister has no function to return those tokens, and the SNS governance canister cannot unilaterally transfer tokens from the swap canister's account. Recovery would require extraordinary NNS governance action (upgrading the swap canister), which is not part of the normal lifecycle.

## Impact Explanation
This is a **High** severity finding. SNS tokens pre-loaded into the swap canister's SNS ledger account are permanently inaccessible after an aborted swap. This is a direct ledger conservation violation: tokens enter the swap canister's SNS ledger account but have no exit path in the ABORTED lifecycle. The impact matches: *"Significant SNS security impact with concrete user or protocol harm"* — the SNS project loses its entire token allocation for the swap, which represents real economic value. The loss is permanent once the swap canister is deleted per the documented lifecycle.

## Likelihood Explanation
Swap abortion is a normal, documented lifecycle outcome — it occurs automatically whenever `min_participants` or `min_icp_e8s` is not reached before the swap deadline. [9](#0-8)  No attacker action is required; the loss occurs automatically upon `finalize` being called on any aborted swap. Every historical and future aborted SNS swap triggers this loss.

## Recommendation
Add a SNS token return step in the ABORTED branch of `finalize_inner`, before the early return, that transfers the full SNS token balance held by the swap canister back to the SNS governance canister's treasury account:

```rust
if self.should_restore_dapp_control() {
    // Return SNS tokens to SNS governance treasury
    finalize_swap_response.set_sweep_sns_tokens_back_result(
        self.sweep_sns_tokens_back(environment.sns_ledger()).await
    );
    finalize_swap_response.set_set_dapp_controllers_result(
        self.restore_dapp_controllers_for_finalize(environment.sns_root_mut()).await,
    );
    return finalize_swap_response;
}
```

The `sweep_sns_tokens_back` function should transfer the swap canister's entire SNS ledger balance to the SNS governance canister's treasury subaccount, analogous to how `sweep_icp` returns ICP to buyers.

## Proof of Concept
1. Deploy an SNS with `sns_token_e8s = 1_000_000_000` and `min_icp_e8s = 500_000_000`.
2. Pre-load the swap canister with 1,000,000,000 SNS tokens (PENDING state).
3. Open the swap.
4. Have participants contribute only 100,000,000 ICP (below minimum).
5. Allow the swap deadline to pass → swap transitions to `Lifecycle::Aborted`.
6. Call `finalize_swap`.
7. Observe: `sweep_icp` returns ICP to buyers; dapp controllers are restored; no SNS token transfer occurs.
8. Query the swap canister's SNS ledger account balance → still holds 1,000,000,000 SNS tokens.
9. Confirm no callable function exists to recover these tokens.
10. Delete the swap canister → 1,000,000,000 SNS tokens are permanently destroyed.

This is reproducible as a deterministic integration test using PocketIC by asserting the swap canister's SNS ledger balance is zero after `finalize` on an aborted swap.
