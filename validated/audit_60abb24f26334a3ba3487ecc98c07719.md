### Title
Pool admin bypasses `maxAdminSpreadFeeE6` cap via uncapped `setPoolBinAdditionalFees`, charging up to 6.5535% additional fee per bin — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary
`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6`/`addFeeSellE6` (both `uint16`, max 65535) directly to the pool with no validation against `maxAdminSpreadFeeE6`. Every swap adds `Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)` on top of `baseFeeX64`, so the pool admin can impose up to 6.5535% additional fee per bin beyond whatever the factory owner has configured as the spread cap.

### Finding Description
`setPoolAdminFees` enforces `if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh()` before updating the pool's global spread fee. `setPoolBinAdditionalFees` has no analogous check — it calls `setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6)` on the pool unconditionally. The pool's swap routines then compute the effective bin fee as `baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)`, where `baseFeeX64` already encodes the oracle-derived spread plus the global admin spread. The bin additional fee is purely additive and uncapped.

A pool admin can call:
```
factory.setPoolBinAdditionalFees(pool, 0, 65535, 65535)
```
This sets `addFeeBuyE6 = addFeeSellE6 = 65535` (6.5535%) on bin 0. Every subsequent swap through that bin pays 6.5535% more than the oracle spread alone, regardless of what `maxAdminSpreadFeeE6` is set to.

### Impact Explanation
Traders executing swaps through affected bins pay up to 6.5535% more than the factory-enforced cap permits. This is a direct loss of user principal on every swap. The factory owner's ability to set `maxAdminSpreadFeeE6` to protect users is rendered ineffective for any pool whose admin uses this path. The impact is per-bin but can cover all active bins simultaneously.

### Likelihood Explanation
The pool admin is a semi-trusted role. The factory's cap system (`maxAdminSpreadFeeE6`) is the only on-chain protection users have against admin fee extraction. The bypass requires a single transaction from the pool admin with no timelock, no precondition, and no factory-level guard. Any pool admin who wishes to extract value from traders can do so immediately.

### Recommendation
Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, define a separate `maxBinAdditionalFeeE6` cap that the factory owner can configure independently.
