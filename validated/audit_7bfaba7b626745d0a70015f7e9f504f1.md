### Title
Unprivileged Caller Can Permanently Freeze Block Rewards via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is callable by any non-zero address. When invoked with `disable_rewards: true`, it writes the current block number to `last_reward_block` **before** checking the flag, then returns without distributing any rewards. Because the contract enforces exactly one reward update per block, a single attacker call per block permanently forfeits all staker and delegator block rewards for that block.

---

### Finding Description

`update_rewards` is exposed through the public `IStakingRewardsManager` interface with no role-based access control — only `general_prerequisites()` is checked, which enforces "not paused" and "caller ≠ zero address".

The critical ordering inside the function is:

```
// 1. Validate staker exists and has non-zero balance
// 2. *** Write last_reward_block = current_block_number ***
self.last_reward_block.write(current_block_number);   // line ~1485

// 3. Early-return if disable_rewards OR pre-consensus
if disable_rewards || self.is_pre_consensus() {
    return;                                            // line ~1488
}
// 4. (never reached when disable_rewards = true)
// calculate_block_rewards + _update_rewards
```

The guard that prevents double-processing is:

```
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

Once `last_reward_block` is stamped with the current block, no subsequent call in the same block can pass this assertion. Therefore, an attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` first in a block consumes the single allowed slot and discards all rewards for that block permanently.

---

### Impact Explanation

Every block in which the attacker fires this call, the entire block's STRK (and BTC) rewards for **all** stakers and their delegators are silently dropped. The `reward_supplier` is never asked to mint or transfer anything, and `unclaimed_rewards_own` / the pool's `cumulative_rewards_trace` are never updated. The lost yield cannot be recovered retroactively. This constitutes **permanent freezing of unclaimed yield** across the whole protocol.

---

### Likelihood Explanation

- No special role or privilege is required — any externally-owned account suffices.
- The attacker only needs to submit one transaction per block; on Starknet the sequencer processes transactions in order, so the attacker simply needs to be included before any legitimate `update_rewards` call.
- The attacker bears only gas costs; there is no economic barrier proportional to the damage caused.
- The attack is fully automatable with a simple bot.

---

### Recommendation

1. **Restrict `disable_rewards: true` to a privileged role** (e.g., `only_security_agent` or a dedicated rewards-manager role), or remove the parameter entirely and handle the pre-consensus no-op path internally.
2. Alternatively, move the `last_reward_block.write` **after** the `disable_rewards` guard so that a disabled call does not consume the block's reward slot.

---

### Proof of Concept

1. Attacker monitors the chain; at the start of each new block (i.e., `current_block_number > last_reward_block`), attacker submits:
   ```
   staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
   ```
2. Inside the call: staker validity checks pass → `last_reward_block` is written to `current_block_number` → function returns early.
3. Any subsequent legitimate call in the same block hits `REWARDS_ALREADY_UPDATED` and reverts.
4. Block rewards are permanently lost; `unclaimed_rewards_own` and pool `cumulative_rewards_trace` are never incremented.
5. Repeated every block → all consensus-phase block rewards are frozen indefinitely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
