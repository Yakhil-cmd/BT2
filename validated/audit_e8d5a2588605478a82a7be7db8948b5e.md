### Title
Gateway Stateful Validator Uses Stale Previous-Block L2 Gas Price Instead of Pre-Computed `next_l2_gas_price` for Admission Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads the **current block's** `l2_gas_price` from the latest committed block header to gate admission of incoming V3 transactions. The block header already contains a pre-computed `next_l2_gas_price` field (the EIP-1559 price for the block the transaction will actually execute in), but this field is never read. Because the L2 gas price can change by up to ~9.5% per block under full load, the gateway's admission decision is systematically wrong: it admits transactions whose `max_price_per_unit` is too low for the next block (they will fail blockifier pre-validation and cannot be sequenced), and it rejects transactions whose `max_price_per_unit` is sufficient for the next block (they are valid but turned away). The code itself acknowledges the problem with a TODO comment: `// TODO(Arni): getnext_l2_gas_price from the block header.`

---

### Finding Description

**Root cause — wrong field read in `validate_resource_bounds`**

`StatefulTransactionValidator::validate_resource_bounds` fetches block info from `GatewayFixedBlockStateReader` and extracts `gas_prices.strk_gas_prices.l2_gas_price`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold check then requires `tx.l2_gas.max_price_per_unit >= min_gas_price_percentage% × previous_block_l2_gas_price`. In production `min_gas_price_percentage = 100`, so the threshold equals the previous block's price exactly. [2](#0-1) [3](#0-2) 

**The correct value is already in the block header but is never surfaced**

`BlockHeaderWithoutHash` carries both `l2_gas_price` (the price of the block just committed) and `next_l2_gas_price` (the EIP-1559 price pre-computed for the *next* block):

```rust
pub struct BlockHeaderWithoutHash {
    pub l2_gas_price: GasPricePerToken,
    pub l2_gas_consumed: GasAmount,
    pub next_l2_gas_price: GasPrice,   // ← correct value to use
    ...
}
``` [4](#0-3) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` reads the block header from state sync but constructs a `BlockInfo` that only copies `l2_gas_price`, silently discarding `next_l2_gas_price`:

```rust
gas_prices: GasPrices {
    strk_gas_prices: GasPriceVector {
        ...
        l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
        // next_l2_gas_price is never read
    },
},
``` [5](#0-4) 

**EIP-1559 price dynamics make the discrepancy material**

The next-block L2 gas price is computed by `calculate_next_base_gas_price` using `gas_price_max_change_denominator = 48` and `gas_target = 1_040_000_000` (v0.14.3/v0.14.4). When a block is fully saturated (`gas_used = max_block_size = 5_800_000_000`):

```
gas_delta = 5.8B − 1.04B = 4.76B
price_change = price × 4.76B / (1.04B × 48) ≈ price × 9.54%
``` [6](#0-5) [7](#0-6) 

So `next_l2_gas_price` can be up to ~9.5% higher than `l2_gas_price` in a single block.

**Divergence between gateway admission and blockifier execution**

The gateway's `run_validate_entry_point` also builds its `BlockContext` from the same stale `get_block_info()` call (only incrementing the block number, not the gas price), so the blockifier validation inside the gateway also uses the previous block's price:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
// gas_prices are NOT updated — still previous block's prices
let block_context = BlockContext::new(block_info, ...);
``` [8](#0-7) 

When the batcher later executes the transaction, it uses the actual next-block gas prices (computed by the consensus orchestrator via `update_l2_gas_price`). The blockifier's `check_fee_bounds` then enforces `tx.l2_gas.max_price_per_unit >= actual_next_block_l2_gas_price`:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { ... }
    );
}
``` [9](#0-8) 

A transaction with `max_price_per_unit = P_prev` passes the gateway (threshold = 100% × P_prev) but fails blockifier pre-validation when `P_next > P_prev`. The transaction is admitted to the mempool but cannot be sequenced into any block.

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions or rejects valid transactions before sequencing.**

Two concrete effects:

1. **False admission (invalid tx accepted):** A user sets `l2_gas.max_price_per_unit` exactly at the previous block's price (the natural choice). If the next block's price is higher (high-demand period), the gateway admits the transaction but the batcher's blockifier rejects it at `check_fee_bounds`. The transaction occupies a mempool slot indefinitely and is never sequenced. An adversary can deliberately flood the mempool with such transactions to exhaust mempool capacity.

2. **False rejection (valid tx refused):** A user sets `l2_gas.max_price_per_unit` slightly below the previous block's price (e.g., during a price-decrease period). The gateway rejects it with `GAS_PRICE_TOO_LOW` even though the next block's price is lower and the transaction would have been valid. Legitimate transactions are silently dropped at the gateway.

---

### Likelihood Explanation

The L2 gas price changes every block via EIP-1559. During any period of above-target block utilization the next-block price is strictly higher than the current-block price. With `min_gas_price_percentage = 100` in production, any transaction whose `max_price_per_unit` equals the previous block's price (the value a well-behaved wallet would use) is at risk of false admission whenever the network is busy. This is a normal operating condition, not an edge case.

---

### Recommendation

In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, expose `next_l2_gas_price` through the `GatewayFixedBlockStateReader` interface (or as a separate method), and use it in `validate_resource_bounds` instead of `l2_gas_price`. The value is already present in the stored block header:

```rust
// In get_block_info_from_sync_client, surface next_l2_gas_price:
let next_l2_gas_price = block_header.next_l2_gas_price;

// In validate_resource_bounds, use it:
// (replacing the current l2_gas_price read)
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()   // new method
    .await?;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    next_block_l2_gas_price,
)?;
```

The same correction should be applied to `run_validate_entry_point`'s `BlockContext` construction so that gateway-side blockifier validation uses the same price the batcher will enforce.

---

### Proof of Concept

1. Query the latest committed block header; read `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (any block with above-target gas usage).
2. Construct a V3 Invoke transaction with `l2_gas.max_price_per_unit = P` (exactly the previous block's price).
3. Submit to the gateway. `validate_resource_bounds` computes threshold = 100% × P = P; the transaction passes (`P >= P`).
4. The transaction is admitted to the mempool.
5. When the batcher attempts to include the transaction, `check_fee_bounds` compares `P` against the actual next-block price `P'`. Since `P < P'`, the transaction fails with `MaxGasPriceTooLow` and is dropped from the block.
6. The transaction remains in the mempool, consuming a slot, and will never be sequenced unless the L2 gas price falls back to or below `P`.

The `next_l2_gas_price` field is already serialized in the P2P protobuf (`field 19` in `SignedBlockHeader`) and stored in the MDBX header table, confirming it is available at the point `GatewayFixedBlockSyncStateClient` reads the block. [10](#0-9) [11](#0-10)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
