### Title
`GlobalContractIdentifier::len()` Undercharges `storage_usage` by 5 Bytes for `AccountId` Variant — (`core/primitives-core/src/global_contract.rs`)

---

### Summary

`use_global_contract` increments `account.storage_usage` by `contract_identifier.len()`. For the `GlobalContractIdentifier::AccountId` variant, that method returns only `account_id.len()`, omitting the 1-byte enum discriminant and the 4-byte Borsh `u32` string-length prefix that are actually written into the account struct. Every account that adopts a global contract by AccountId has its `storage_usage` permanently underreported by exactly **5 bytes**, allowing it to hold less NEAR than the storage-staking invariant requires.

---

### Finding Description

`GlobalContractIdentifier::len()` is defined in `core/primitives-core/src/global_contract.rs`:

```rust
// core/primitives-core/src/global_contract.rs  lines 33-38
pub fn len(&self) -> usize {
    match self {
        GlobalContractIdentifier::CodeHash(_) => 32,
        GlobalContractIdentifier::AccountId(account_id) => account_id.len(),  // ← missing 5 bytes
    }
}
``` [1](#0-0) 

`use_global_contract` uses this value directly to update `account.storage_usage`:

```rust
// runtime/runtime/src/global_contracts.rs  lines 97-104
account.set_storage_usage(
    account.storage_usage().checked_add(contract_identifier.len() as u64)...
);
``` [2](#0-1) 

The account struct stores `AccountContract::GlobalByAccount(id)` in Borsh encoding, which is:

| Field | Bytes |
|---|---|
| enum discriminant | 1 |
| `u32` string-length prefix | 4 |
| account ID bytes | `account_id.len()` |

Total: **`5 + account_id.len()`** bytes.

The sibling type `GlobalContractCodeIdentifier::len()` (used for trie-key sizing) gets this right:

```rust
// core/primitives/src/trie_key.rs  lines 141-149
pub fn len(&self) -> usize {
    1 + match self {
        Self::CodeHash(hash) => hash.as_bytes().len(),
        Self::AccountId(account_id) => {
            // Corresponds to String repr in borsh spec
            size_of::<u32>() + account_id.len()   // 4 + account_id.len()
        }
    }
}
``` [3](#0-2) 

`GlobalContractIdentifier::len()` for `AccountId` returns only `account_id.len()`, missing the 5-byte overhead. The same value is used symmetrically in `clear_account_contract_storage_usage` → `get_contract_storage_usage` → `account.contract().identifier_storage_usage()`, so the add/subtract cycle is internally consistent but both sides are wrong by 5 bytes relative to the actual on-disk representation. [4](#0-3) 

---

### Impact Explanation

`account.storage_usage` is the sole input to the storage-staking check (`check_storage_stake`). With `storage_usage` 5 bytes below the true value, an account can maintain a balance that is `5 × storage_amount_per_byte` yoctoNEAR below the required minimum. At the current mainnet rate (~10^20 yoctoNEAR/byte) this is ~0.0005 NEAR per affected account — a small but real, permanent ledger invariant violation that accumulates across every account that calls `UseGlobalContractAction` with the `AccountId` variant.

---

### Likelihood Explanation

Any unprivileged user can submit a `UseGlobalContractAction` with `GlobalContractIdentifier::AccountId(...)` once a global contract has been deployed in `AccountId` mode. The action is validated and processed unconditionally through `action_use_global_contract` → `use_global_contract`. No special privilege is required. [5](#0-4) 

---

### Recommendation

Change `GlobalContractIdentifier::len()` for the `AccountId` arm to match the actual Borsh encoding:

```rust
GlobalContractIdentifier::AccountId(account_id) => {
    1 + std::mem::size_of::<u32>() + account_id.len()  // discriminant + u32 prefix + bytes
}
```

Align the `CodeHash` arm similarly (currently returns `32`, which already omits the 1-byte discriminant — verify whether that is intentional or a parallel bug). Add a test asserting that `GlobalContractIdentifier::len()` equals `borsh::object_length(&AccountContract::from(identifier))` for both variants.

---

### Proof of Concept

1. Deploy a global contract in `AccountId` mode from account `deployer.near`.
2. Submit `UseGlobalContractAction { contract_identifier: GlobalContractIdentifier::AccountId("deployer.near") }` targeting account `victim.near`.
3. Read `victim.storage_usage()` before and after; observe the delta is `len("deployer.near") = 12` bytes.
4. Borsh-serialize `AccountContract::GlobalByAccount("deployer.near")`; observe the serialized length is `1 + 4 + 12 = 17` bytes.
5. Assert `delta == 12` and `actual_overhead == 17`; the 5-byte gap is the undercharge.
6. Confirm `check_storage_stake` passes with a balance 5 × `storage_amount_per_byte` below what the true storage usage would require.

pub(crate) fn use_global_contract(
    state_update: &mut TrieUpdate,
    account_id: &AccountId,
    account: &mut Account,
    contract_identifier: &GlobalContractIdentifier,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let key = TrieKey::GlobalContractCode { identifier: contract_identifier.clone().into() };
    if !state_update.contains_key(&key, AccessOptions::DEFAULT)? {
        result.result = Err(ActionErrorKind::GlobalContractDoesNotExist {
            identifier: contract_identifier.clone(),
        }
        .into());
        return Ok(());
    }
    clear_account_contract_storage_usage(state_update, account_id, account)?;
    if account.contract().is_local() {
        state_update.remove(TrieKey::ContractCode { account_id: account_id.clone() });
    }
    let contract = match contract_identifier {
        GlobalContractIdentifier::CodeHash(code_hash) => AccountContract::Global(*code_hash),
        GlobalContractIdentifier::AccountId(id) => AccountContract::GlobalByAccount(id.clone()),
    };
    account.set_storage_usage(
        account.storage_usage().checked_add(contract_identifier.len() as u64).ok_or_else(|| {
            StorageError::StorageInconsistentState(format!(
                "Storage usage integer overflow for account {}",
                account_id
            ))
        })?,
    );
    account.set_contract(contract);
    Ok(())
}
```

fn get_contract_storage_usage(
    state_update: &TrieUpdate,
    account_id: &AccountId,
    account: &Account,
) -> Result<StorageUsage, StorageError> {
    Ok(match account.contract().as_ref() {
        AccountContract::None => 0,
        AccountContract::Local(code_hash) => {
            get_code_len_or_default(state_update, account_id.clone(), *code_hash)?
        }
        AccountContract::Global(_) | AccountContract::GlobalByAccount(_) => {
            account.contract().identifier_storage_usage()
        }
    })
}

/// Clears the contract storage usage based on type for an account.
pub(crate) fn clear_account_contract_storage_usage(
    state_update: &TrieUpdate,
    account_id: &AccountId,
    account: &mut Account,
) -> Result<(), StorageError> {
    let contract_storage = get_contract_storage_usage(state_update, account_id, account)?;
    account.set_storage_usage(account.storage_usage().saturating_sub(contract_storage));
    Ok(())
