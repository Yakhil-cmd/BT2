### Title
Malicious Participant Sends Correct R_me but Tampered w_me to Permanently Abort Presign for All Honest Parties — (`src/ecdsa/robust_ecdsa/presign.rs`)

### Summary
In `do_presign`, a single malicious participant can broadcast a correct `R_me = g^{k_me}` alongside a tampered `w_me' = w_me + delta`. The R-consistency check passes, but the final `W == g^w` check fails for every honest party, causing a universal `AssertionFailed` abort with no mechanism to identify or exclude the cheater.

### Finding Description
In Round 2, each participant broadcasts `(R_me, w_me)` via `chan.send_many`. [1](#0-0) 

The only downstream check on the R values is an exponent-interpolation consistency test over the first `t+1` entries: [2](#0-1) 

This check verifies that the `R_i` values lie on a common degree-t polynomial in the exponent. A correct `R_me` passes trivially. There is **no proof** that `w_me` was computed as `a_me * k_me + b_me` using the same `k_me` whose discrete log is `R_me`.

The `w` scalar is then interpolated from all `2t+1` w-shares: [3](#0-2) 

The W shares are computed independently from `R` and each party's own `a_me`: [4](#0-3) 

The final cross-check compares `W` (from W-shares, correct) against `g^w` (from the corrupted interpolation, wrong): [5](#0-4) 

When this fails, every honest party returns `Err(ProtocolError::AssertionFailed(...))` with no per-sender attribution.

### Impact Explanation
All `2t+1` honest parties abort presign. Because the protocol enforces exactly `N = 2t+1` participants: [6](#0-5) 

there is no smaller honest subset to fall back to. If the DKG was run with exactly `2t+1` participants, presign is permanently blocked as long as the malicious party participates. Even with a larger DKG, honest parties cannot identify the cheater to exclude them from a retry.

### Likelihood Explanation
Any participant who can inject messages into the broadcast channel (i.e., any protocol participant) can execute this attack with a single-bit change to their `w_me` value. No cryptographic assumption needs to be broken. The attacker only needs to be one of the `2t+1` presign participants.

### Recommendation
Require each participant to accompany `w_me` with a zero-knowledge proof of correct computation — specifically, a proof that `w_me = a_me * k_me + b_me` where `R_me = g^{k_me}`. A Maurer-style NIZK (already used elsewhere in the library, see `docs/crypto/proofs.md`) of the form `Mau((·G, ·R_me), (R_me, W_me_candidate); k_me, a_me)` would bind the scalar `w_me` to the broadcast `R_me` and allow honest parties to reject and attribute a cheating `w_me` before the interpolation step.
