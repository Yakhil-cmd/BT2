### Title
Missing `max_l2_gas_amount` Bound Check for Declare Transactions in Stateless Validator — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a `max_l2_gas_amount` ceiling on every transaction type **except** `RpcTransaction::Declare`. A Declare transaction with `l2_gas.max_amount` arbitrarily exceeding the configured limit passes stateless validation, passes stateful resource-bounds validation (which only checks the L2 gas *price* threshold, not the amount), and — if the sender holds sufficient balance to cover the committed fee — is admitted to the mempool. The gateway's own admission-control invariant is therefore broken for one transaction type.

### Finding Description

In `validate_resource_bounds`, the check that rejects transactions whose `l2_gas.max_amount` exceeds `self.config.max_l2_gas_amount` is guarded by an explicit empty `if let` branch that short-circuits for Declare:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← nothing; check is skipped
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

The developer acknowledged the gap with a `TODO` comment. The test suite explicitly documents and asserts the bypass:

```rust
fn valid_l2_gas_amount_on_declare(…) {
    // max_l2_gas_amount: 100, tx max_amount: 200
    assert_matches!(tx_validator.validate(&tx), Ok(()));
}
``` [2](#0-1) 

The production default for `max_l2_gas_amount` is `1_210_000_000`: [3](#0-2) 

The stateful validator's `validate_resource_bounds` only checks the L2 gas *price* threshold — it never checks `max_amount`: [4](#0-3) 

The only downstream guard that could stop the transaction is `verify_can_pay_committed_bounds`, which checks `balance >= max_amount × max_price`. For a `max_amount` only slightly above the configured limit (e.g., `1_210_000_001`) with a realistic `max_price_per_unit`, the required balance is well within reach of a normal account, so the transaction proceeds into the mempool. [5](#0-4) 

The full stateless validation entry point calls `validate_resource_bounds` for all transaction types, but the Declare branch silently exits: [6](#0-5) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid Declare transactions.**

The `max_l2_gas_amount` parameter is described as "Maximum allowed L2 gas amount for transactions" and is enforced as a hard admission-control ceiling for Invoke and DeployAccount. Bypassing it for Declare means:

1. A Declare transaction with `l2_gas.max_amount > 1_210_000_000` (production limit) is admitted to the mempool.
2. The bouncer uses resource bounds to estimate block capacity; an over-declared gas amount causes it to reserve more L2 gas budget than the transaction actually needs, reducing effective block throughput.
3. The gateway's own stated invariant — that no admitted transaction may claim more than `max_l2_gas_amount` of L2 gas — is violated for one transaction type.

### Likelihood Explanation

Any unprivileged user can submit a Declare transaction via the public RPC endpoint. The only prerequisite is holding enough STRK to cover `(max_l2_gas_amount_config + 1) × max_price_per_unit` as a committed fee, which is a normal account balance for a slightly-above-limit amount. No special access or privileged role is required.

### Recommendation

Remove the empty `if let RpcTransaction::Declare(_) = tx {}` branch and apply the `max_l2_gas_amount` check uniformly:

```rust
// Before (broken):
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(…);
}

// After (fixed):
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If Declare transactions legitimately require a higher ceiling (e.g., because compilation is more gas-intensive), introduce a separate `max_l2_gas_amount_declare` config field rather than removing the check entirely.

### Proof of Concept

1. Configure the gateway with the production default: `max_l2_gas_amount = 1_210_000_000`.
2. Construct a valid `RpcDeclareTransactionV3` with `resource_bounds.l2_gas = { max_amount: 1_210_000_001, max_price_per_unit: P }` where `P` is at or above `min_gas_price`.
3. Ensure the sender account holds balance ≥ `1_210_000_001 × P`.
4. Submit via `starknet_addDeclareTransaction`.
5. `StatelessTransactionValidator::validate_resource_bounds` hits the `if let RpcTransaction::Declare(_) = tx {}` branch and returns `Ok(())` — the `MaxGasAmountTooHigh` error is never raised.
6. `StatefulTransactionValidator::validate_resource_bounds` checks only the L2 gas price threshold — `max_amount` is not inspected.
7. `verify_can_pay_committed_bounds` passes because the account balance covers the committed fee.
8. The transaction enters the mempool with `l2_gas.max_amount = 1_210_000_001`, one unit above the configured admission limit, in violation of the gateway's own invariant. [7](#0-6) [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

```rust
pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}
