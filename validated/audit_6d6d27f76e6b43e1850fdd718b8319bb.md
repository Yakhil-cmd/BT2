### Title
Unprivileged Bypass of `MsgTurnBridge` Authorization and Permanent Disablement of Bridge Circuit Breaker - (File: `x/cronos/keeper/msg_server.go`)

### Summary
`MsgTurnBridge` is the Cronos emergency circuit breaker for the Gravity bridge. Its handler is a complete no-op stub that performs no authorization check and changes no state. Any unprivileged address can submit `MsgTurnBridge` and receive a success response, while no authorized party can ever actually disable the bridge.

### Finding Description

The `TurnBridge` handler in `x/cronos/keeper/msg_server.go` is implemented as:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

It returns `(nil, nil)` unconditionally — no permission check, no state mutation, no error.

The design intent is explicit. ADR-009 states that `MsgUpdateTokenMapping` and `MsgTurnBridge` are the two messages that require the `CanChangeTokenMapping` and `CanTurnBridge` permissions respectively: [2](#0-1) 

`UpdateTokenMapping` correctly enforces this:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [3](#0-2) 

The `CanTurnBridge` permission constant is defined and the `HasPermission` infrastructure is fully in place: [4](#0-3) 

But `TurnBridge` never calls `HasPermission` at all.

The integration test for `test_gravity_turn_bridge` expects `from_="community"` to fail with a non-zero code and `from_="validator"` to succeed: [5](#0-4) 

With the stub implementation, both calls return success (`nil` error), and neither actually changes bridge state.

### Impact Explanation

Two distinct impacts:

1. **Auth bypass (High):** Any unprivileged address can submit `MsgTurnBridge` and receive an on-chain success. The `CanTurnBridge` permission check — the exact authorization guard that ADR-009 mandates — is completely absent. This is a direct bypass of Cronos bridge authorization checks.

2. **Permanent bridge circuit breaker failure (High):** Because the function is a no-op, no one — including the admin or a permissioned address — can ever disable the Gravity bridge. In an emergency (e.g., a bridge exploit draining CRC20/CRC21 assets), the intended emergency stop mechanism is permanently non-functional, allowing continued unauthorized minting and burning of bridge tokens.

### Likelihood Explanation

The entry path is a standard Cosmos SDK message transaction, reachable by any address with a funded account. No special privileges, leaked keys, or cryptographic breaks are required. The `MsgTurnBridge` RPC is registered and live on the message server.

### Recommendation

Implement `TurnBridge` with the same pattern as `UpdateTokenMapping`:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // set bridge enabled/disabled state
    params := k.GetParams(ctx)
    params.EnableAutoDeployment = msg.Enable // or the appropriate bridge-enable field
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

The exact state field to toggle depends on the Gravity bridge integration, but the authorization guard must be added unconditionally.

### Proof of Concept

1. Any address (e.g., `community`) submits `MsgTurnBridge{Sender: "<community_addr>", Enable: false}`.
2. The message is routed to `msgServer.TurnBridge`, which immediately returns `(nil, nil)`.
3. The transaction is included in a block with code `0` (success).
4. No bridge state changes. The bridge remains enabled regardless of the `Enable` field.
5. Even the admin submitting `Enable: false` produces the same no-op result — the bridge cannot be stopped.
