### Title
Missing Access Control on `update_rewards` Allows Anyone to Suppress Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in `staking.cairo` is specified as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` sentinel to the current block without distributing any rewards. Because the function enforces a strict one-call-per-block invariant, the legitimate sequencer is then blocked from distributing rewards for that block.

### Finding Description

The spec for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. After `general_prerequisites()` (which only verifies the contract is not paused and the caller is non-zero), the function immediately writes `last_reward_block` to the current block number — **before** the `disable_rewards` branch — and then returns early when `disable_rewards == true`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits without distributing
    }
    ...
``` [2](#0-1) 

The one-call-per-block guard is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

Once `last_reward_block` equals the current block, no further call — including the legitimate sequencer call — can succeed in that block.

### Impact Explanation

When an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N:

1. `last_reward_block` is set to N.
2. No rewards are computed or transferred.
3. The sequencer's subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`.
4. All stakers and delegators lose their consensus block rewards for block N.

If repeated across many blocks, this constitutes a **permanent freezing of unclaimed yield** for stakers and delegators. This matches the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"*.

### Likelihood Explanation

Any address can call `update_rewards`. The attacker only needs to supply a valid (non-exiting, non-zero-balance) staker address, which is publicly observable on-chain. On Starknet the sequencer controls ordering, so reliable front-running requires the attacker to submit the transaction in a block where the sequencer has not yet included its own `update_rewards` call (e.g., sequencer downtime, missed blocks, or if the sequencer call is submitted as a regular user transaction rather than a forced first-in-block transaction). The missing guard means the attack surface is permanently open.

### Recommendation

Add a caller check at the top of `update_rewards` that restricts execution to the designated sequencer address (stored in contract storage or enforced via a role), mirroring the pattern used for other privileged callers in the system:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_operator();   // or assert!(get_caller_address() == sequencer_address)
    self.general_prerequisites();
    ...
```

### Proof of Concept

1. Deploy the system; wait until consensus rewards are active (`is_pre_consensus()` returns false).
2. In block N, before the sequencer transaction lands, call:
   ```
   staking.update_rewards(valid_staker_address, disable_rewards: true)
   ```
3. Observe `last_reward_block` is now N and the call succeeds with no reward transfer.
4. The sequencer's `update_rewards` call in block N reverts with `REWARDS_ALREADY_UPDATED`.
5. Stakers and delegators receive zero rewards for block N.
6. Repeat each block to suppress all consensus reward distribution indefinitely. [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
