### Title
Unprivileged Caller Can Permanently Freeze All Staker Consensus Rewards via `update_rewards` - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `staking.cairo` accepts a caller-controlled `disable_rewards: bool` parameter with no access control. Any unprivileged address can call `update_rewards(valid_staker, disable_rewards: true)` every block, consuming the global `last_reward_block` slot without distributing any rewards, permanently freezing consensus-era yield for all stakers.

### Finding Description

`update_rewards` is part of `IStakingRewardsManager` and is callable by any non-zero address. Its only gate is `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

Inside the function, `last_reward_block` — a **single global storage slot** shared across all stakers — is written unconditionally before the `disable_rewards` branch is evaluated: [2](#0-1) 

The guard that prevents a second call in the same block compares against this global slot: [3](#0-2) 

Because `last_reward_block` is written **before** the `disable_rewards` check, any caller who passes `disable_rewards: true` will:

1. Consume the block's reward slot (set `last_reward_block = current_block`).
2. Return immediately without distributing any STRK rewards.
3. Cause every subsequent legitimate call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

The `disable_rewards` flag has no access-control guard — there is no assertion that the caller is the staker, the staker's operational address, or any privileged role. [4](#0-3) 

### Impact Explanation

`last_reward_block` is a global variable, not per-staker: [5](#0-4) 

A single attacker call per block is sufficient to prevent **every** staker in the protocol from receiving consensus rewards in that block. Repeated across every block, this permanently freezes all unclaimed consensus-era yield. Stakers' `unclaimed_rewards_own` fields are never incremented, and pool reward traces are never updated, so delegators also lose all yield.

This matches the allowed impact: **Permanent freezing of unclaimed yield** (High).

### Likelihood Explanation

- The function is fully public; no role, signature, or stake is required.
- The attacker only needs to supply any currently-active staker address (readable from the public `stakers` vector or events).
- The cost is one transaction per block — trivially affordable given the value of frozen rewards across the entire protocol.
- The attack is silent: no on-chain alarm is raised, and stakers have no way to distinguish a griefed block from a normal missed-reward block.

### Recommendation

1. **Remove `disable_rewards` from the public interface entirely**, or restrict it to a privileged role (e.g., `only_security_agent`).
2. Alternatively, write `last_reward_block` **only after** confirming that rewards will actually be distributed (i.e., move the write past the `disable_rewards` branch), so that a no-op call does not consume the block slot.
3. If `disable_rewards` is needed for legitimate operational use, gate it with `assert!(caller == staker_address || caller == staker_info.operational_address, ...)`.

### Proof of Concept

```
// Attacker script — run once per block:
// 1. Read any active staker address from the public `stakers` Vec.
let victim_staker = staking.stakers[0];

// 2. Call update_rewards with disable_rewards = true.
//    No special role or stake required.
staking.update_rewards(staker_address: victim_staker, disable_rewards: true);

// Effect:
//   - last_reward_block is now set to the current block.
//   - No rewards were distributed.
//   - Any legitimate update_rewards call in this block reverts with REWARDS_ALREADY_UPDATED.
//   - Repeat every block → all stakers and delegators receive zero consensus rewards indefinitely.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
