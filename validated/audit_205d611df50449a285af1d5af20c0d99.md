### Title
`set_commission` Applies Commission Increase to Pending Epoch Rewards Without Prior Settlement - (File: `src/staking/staking.cairo`)

---

### Summary

The `set_commission` function in the Staking contract immediately overwrites the stored commission rate without first settling pending rewards. Because `calculate_staker_pools_rewards` reads the commission at the moment of reward calculation (during attestation), a staker who uses `set_commission_commitment` to unlock a commission increase and then calls `set_commission` mid-epoch causes the new, higher commission to be applied retroactively to the entire epoch's pending delegator rewards — rewards that were accruing under the old, lower rate.

---

### Finding Description

**Root cause — `set_commission` writes the new rate with no prior reward settlement:** [1](#0-0) 

The internal helper `update_commission` also writes directly to storage with no settlement step: [2](#0-1) 

**Commission is consumed at reward-calculation time, not at the time it was set:**

When attestation fires, `_update_rewards` calls `calculate_staker_pools_rewards`, which reads the *current* commission from storage: [3](#0-2) 

The commission read at line 1964 is whatever value is in storage at that moment — not the value that was in effect when the epoch began.

**The commitment mechanism explicitly enables mid-epoch commission increases:**

`set_commission_commitment` lets a staker pre-authorize any commission value up to `max_commission`, which may be *higher* than the current commission: [4](#0-3) 

The protocol's own inline note acknowledges the gap: [5](#0-4) 

**Attack sequence:**

1. Staker is running at 5 % commission; delegators have been staking for most of the current epoch.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — this is permissionless for any staker with a pool.
3. Staker immediately calls `set_commission(10000)` — raises commission to 100 %.
4. Attestation fires later in the same epoch; `calculate_staker_pools_rewards` reads `commission = 10000` and splits the *entire epoch's* pool rewards: `commission_rewards = pool_rewards_including_commission * 10000 / 10000 = pool_rewards_including_commission`, leaving `pool_rewards = 0`.
5. Delegators receive zero rewards for the epoch despite having staked the whole time at 5 %.

Delegators cannot escape: the two-step exit (`exit_delegation_pool_intent` → wait window → `exit_delegation_pool_action`) prevents them from withdrawing before the attestation settles rewards.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

A malicious staker can drain the entire epoch's delegator reward allocation into their own `unclaimed_rewards_own` by raising commission to 100 % immediately before attestation. Delegators who staked in good faith at a low commission rate receive nothing for that epoch. The staker profits by the full amount that should have gone to the pool.

---

### Likelihood Explanation

**Medium.**

Any staker with an active delegation pool can execute this in two transactions (`set_commission_commitment` then `set_commission`) with no external dependencies. The only prerequisite is having delegators whose yield is worth stealing. The protocol's own comment at line 745–747 acknowledges that "current commission increase safeguards still allow for sudden commission changes," confirming the path is reachable and unmitigated.

---

### Recommendation

Before writing the new commission value, settle all pending rewards for the staker's pools using the *current* commission. This ensures that rewards accrued under the old rate are correctly distributed before the rate changes. Concretely, `set_commission` (and its internal `update_commission` helper) should trigger the equivalent of `_update_rewards` for the calling staker prior to overwriting `staker_pool_info.commission`.

---

### Proof of Concept

```
Epoch N begins. Staker commission = 500 (5 %).
Delegator stakes 1 000 000 STRK.

// Mid-epoch, staker executes:
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: N+1);
staking.set_commission(commission: 10000);   // commission now 100 %

// Attestation fires (still epoch N):
// calculate_staker_pools_rewards reads commission = 10000
// commission_rewards = pool_rewards_including_commission * 10000 / 10000
//                    = pool_rewards_including_commission
// pool_rewards = 0
// Delegator receives 0 STRK for epoch N.
// Staker's unclaimed_rewards_own += full pool allocation.
```

The delegator's `unclaimed_rewards` in the pool contract is never incremented for epoch N because `pool_rewards = 0` is passed to `update_pool_rewards`, while the staker pockets the commission portion equal to 100 % of what the pool earned. [6](#0-5)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
