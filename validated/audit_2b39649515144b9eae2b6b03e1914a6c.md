### Title
Missing Zero-Address Validation for `_admin` in Oracle Factory Constructors Permanently Bricks `AnchoredProviderFactory` - (File: smart-contracts-poc/contracts/AnchoredProviderFactory.sol)

### Summary

`AnchoredProviderFactory`, `PriceProviderFactory`, and `PriceProviderFactoryL2` all grant `ADMIN_ROLE` to a caller-supplied `_admin` address in their constructors without validating that it is non-zero. For `AnchoredProviderFactory` this is fatal: every oracle-creation path requires an admin-curated allow-list entry, so a zero-address admin permanently bricks the factory — no `AnchoredPriceProvider` can ever be deployed through it.

### Finding Description

All three oracle factory constructors follow the same pattern:

```solidity
constructor(address _admin) {
    _grantRole(ADMIN_ROLE, _admin);       // no address(0) guard
    _setRoleAdmin(ADMIN_ROLE, ADMIN_ROLE);
}
``` [1](#0-0) [2](#0-1) [3](#0-2) 

`_setRoleAdmin(ADMIN_ROLE, ADMIN_ROLE)` makes `ADMIN_ROLE` self-administering: only an existing `ADMIN_ROLE` holder can grant the role to anyone else. If `_admin == address(0)`, the role is irrevocably held by `address(0)` — an address that can never send a transaction — so no real account can ever acquire `ADMIN_ROLE`.

In `AnchoredProviderFactory` every privileged entry point requires `ADMIN_ROLE`:

- `addOracle` — populates the oracle allow-list
- `removeOracle`
- `setEnvelope` — configures parameter envelopes for feed classes
- `removeEnvelope`
- `setFeedClass` [4](#0-3) [5](#0-4) 

The oracle allow-list starts **empty** at construction by design:

> "The allow-list starts EMPTY at construction and is populated/curated via addOracle / removeOracle (admin)" [6](#0-5) 

`createAnchoredProvider` — the only public deployment path — immediately checks:

```solidity
if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);
``` [7](#0-6) 

Because no oracle can ever be added (allow-list permanently empty), every call to `createAnchoredProvider` reverts with `OracleNotAllowed`. The factory is permanently non-functional from the moment of deployment.

`PriceProviderFactory` and `PriceProviderFactoryL2` share the same constructor flaw but their `createPriceProvider` is permissionless, so provider creation still works; only `addProvider`/`removeProvider` are permanently locked. [8](#0-7) 

### Impact Explanation

If `AnchoredProviderFactory` is deployed with `_admin == address(0)`:

- No oracle can ever be added to the allow-list.
- No `AnchoredPriceProvider` can ever be deployed through the factory.
- All admin-gated configuration (`setEnvelope`, `setFeedClass`) is permanently inaccessible.
- The factory contract is permanently bricked; redeployment is the only recovery, losing any state or integrations pointing to the original address.

This breaks the core oracle infrastructure that pools rely on for anchored price providers, satisfying the "broken core pool functionality" impact criterion.

### Likelihood Explanation

The trigger is passing `address(0)` as `_admin` at deployment — a straightforward input-validation mistake with no on-chain guard to catch it. The `MetricOmmPoolFactory` constructor correctly delegates to `Ownable(initialOwner)` which in OpenZeppelin v5 reverts on zero address, but the oracle factories use raw `AccessControl` with no equivalent protection. The contrast makes the omission easy to miss during review.

### Recommendation

Add an explicit zero-address guard in each factory constructor before granting the role:

```solidity
constructor(address _admin) {
    require(_admin != address(0), "AnchoredProviderFactory: zero admin");
    _grantRole(ADMIN_ROLE, _admin);
    _setRoleAdmin(ADMIN_ROLE, ADMIN_ROLE);
}
```

Apply the same fix to `PriceProviderFactory` and `PriceProviderFactoryL2`.

### Proof of Concept

```solidity
// Deploy with zero admin
AnchoredProviderFactory factory = new AnchoredProviderFactory(address(0));

// ADMIN_ROLE is held by address(0) — no real account can acquire it
assert(factory.hasRole(factory.ADMIN_ROLE(), address(0)));

// addOracle reverts: AccessControl: account 0x... is missing role ADMIN_ROLE
vm.expectRevert();
factory.addOracle(someOracle);

// createAnchoredProvider always reverts: OracleNotAllowed
vm.expectRevert(IAnchoredProviderFactory.OracleNotAllowed.selector);
factory.createAnchoredProvider(
    someOracle, feedId, bytes32(0), minMargin, staleness, maxSpread,
    false, 0, baseToken, quoteToken
);
// Factory is permanently bricked — no AnchoredPriceProvider can ever be deployed.
