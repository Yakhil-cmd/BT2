### Title
Paused Engine Blocks `exit_to_near_precompile_callback` Refund Path, Causing Permanent ERC-20 Token Loss — (`engine/src/contract_methods/connector.rs`)

---

### Summary

When the Aurora Engine is paused via `pause_contract`, the callback `exit_to_near_precompile_callback` is blocked by `require_running`. If the underlying NEP-141 transfer fails while the engine is paused, the refund path that would restore a user's burned ERC-20 tokens is never executed. Because NEAR callbacks execute exactly once and cannot be retried, the user's ERC-20 tokens are permanently destroyed with no corresponding NEP-141 tokens received.

---

### Finding Description

The exit-to-NEAR flow works as follows:

1. A user submits an EVM transaction calling the `ExitToNear` precompile via `submit` / `submit_with_args`.
2. The precompile burns the user's ERC-20 tokens in EVM state and schedules a NEAR promise to transfer the equivalent NEP-141 tokens to the user.
3. `exit_to_near_precompile_callback` is registered as the promise callback to handle success or failure.

The callback is defined in `engine/src/contract_methods/connector.rs`:

```rust
pub fn exit_to_near_precompile_callback<...>(...) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // <-- BLOCKED WHEN PAUSED
        env.assert_private_call()?;
        ...
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
            ...
        }
    })
}

### Impact Explanation

If the engine is paused between the moment the `submit` call commits EVM state (burning ERC-20 tokens) and the moment `exit_to_near_precompile_callback` executes:

- The NEP-141 transfer promise may fail (e.g., the eth-connector is also paused, the recipient account is not registered, or any transient error).
- The callback is invoked by the NEAR runtime exactly once.
- `require_running` returns `ERR_PAUSED`, causing the callback to panic/error.
- The `engine::refund_on_error` branch is never reached.
- The user's ERC-20 tokens remain burned in EVM state with no NEP-141 tokens received and no refund issued.

NEAR promise callbacks are not retried. Once the callback fails, the refund opportunity is permanently lost. This constitutes **permanent freezing (destruction) of user funds**.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

- The `pause_contract` function is callable by the owner at any time for maintenance or security incidents.
- The exit-to-NEAR flow is a standard, publicly documented user operation.
- The window between `submit` committing EVM state and the callback executing spans at least one NEAR block, making the race condition realistic during any pause event.
- The NEP-141 transfer can fail for independent reasons (connector paused separately via `pa_pause_feature`, account not registered, etc.), making the failure path reachable without attacker action.

**Likelihood: Medium** — Requires a pause event to coincide with an in-flight exit-to-NEAR operation, but both events are independently plausible during normal protocol operation.

---

### Recommendation

Remove `require_running` from `exit_to_near_precompile_callback`. This callback is a private NEAR callback (`env.assert_private_call()` is enforced immediately after), so it cannot be called by an external actor. Its sole purpose is to finalize or refund an already-committed EVM state transition. Blocking it when paused provides no security benefit and directly causes irreversible fund loss.

Alternatively, introduce a separate `require_running_or_private_callback` guard that allows private callbacks to proceed regardless of pause state, similar to the recommendation in the reference report to create dedicated functions for critical operations.

---

### Proof of Concept

1. Alice calls `submit` with an EVM transaction invoking the `ExitToNear` precompile to bridge 100 ERC-20 tokens to NEAR. The ERC-20 tokens are burned in EVM state. A NEP-141 transfer promise is scheduled.
2. The owner calls `pause_contract`, setting `state.is_paused = true`.
3. The NEP-141 transfer promise fails (e.g., Alice's NEAR account is not registered with the NEP-141 contract).
4. The NEAR runtime invokes `exit_to_near_precompile_callback` as the scheduled callback.
5. `require_running(&state)?` returns `Err(ERR_PAUSED)` — the callback panics.
6. `engine::refund_on_error` is never called. Alice's 100 ERC-20 tokens remain burned. She receives no NEP-141 tokens. The refund window is permanently closed. [5](#0-4) [6](#0-5) [7](#0-6)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
