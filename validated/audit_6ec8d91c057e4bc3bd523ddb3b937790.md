Audit Report

## Title
Missing Validation of New `ReportedToExecution` Entries Allows Byzantine Block Proposer to Permanently Suppress XNet Reshare Requests — (`rs/consensus/idkg/src/payload_verifier.rs`)

## Summary

`validate_reshare_dealings` applies no checks to `CompletedReshareRequest::ReportedToExecution` entries appearing in `curr_payload.xnet_reshare_agreements`. A single Byzantine block proposer can inject a fabricated `ReportedToExecution` entry keyed to any active `idkg_dealings_context`, causing it to survive the honest cleanup pass in `update_completed_reshare_requests` and permanently prevent `initiate_reshare_requests` from ever starting the legitimate XNet reshare. The target subnet never receives the reshared key material, permanently blocking any Chain Fusion operation that depends on it.

## Finding Description

**Root cause — `validate_reshare_dealings` (`payload_verifier.rs` L546–554):**

```rust
for (request, dealings) in curr_payload.xnet_reshare_agreements.iter() {
    if let idkg::CompletedReshareRequest::Unreported(dealings) = &dealings
        && !prev_payload.xnet_reshare_agreements.contains_key(request)
    {
        // validation only runs here
    }
    // ReportedToExecution falls through silently — no checks at all
}
```

The `if let` arm matches only `Unreported`. A `ReportedToExecution` variant is never examined. There is no check that the entry existed in `prev_payload.xnet_reshare_agreements`, and no check that it was ever in `prev_payload.ongoing_xnet_reshares`. A Byzantine proposer can freely insert `(R, ReportedToExecution)` for any request key `R`.

The second loop (L557–571) iterates only over `prev_payload.ongoing_xnet_reshares`. A fabricated entry for a request that was never in that map is invisible to both loops, so the function returns `Ok(...)` without error.

**Persistence — `update_completed_reshare_requests` (`resharing.rs` L153–163):**

```rust
payload.xnet_reshare_agreements = payload
    .xnet_reshare_agreements
    .keys()
    .filter(|&request| {
        idkg_dealings_contexts.values()
            .any(|context| *request == reshare_request_from_dealings_context(context))
    })
    .cloned()
    .map(|request| (request, idkg::CompletedReshareRequest::ReportedToExecution))
    .collect();
```

The cleanup retains any entry whose key matches an active `idkg_dealings_context`. Because the Byzantine proposer can observe replicated state and craft `R` to match an active context, the spurious entry survives every subsequent honest block.

**Suppression — `initiate_reshare_requests` (`resharing.rs` L44–47):**

```rust
if payload.ongoing_xnet_reshares.contains_key(&request)
    || payload.xnet_reshare_agreements.contains_key(&request)
{
    continue;
}
```

Because `R` is already in `xnet_reshare_agreements`, the legitimate reshare is never added to `ongoing_xnet_reshares`. The context in execution state waits indefinitely for a response that never arrives, keeping the spurious entry alive and the suppression permanent.

## Impact Explanation

The target subnet never receives the reshared threshold key material. Any Chain Fusion operation (threshold ECDSA/Schnorr signing for Bitcoin, Ethereum, etc.) that depends on that reshare is permanently blocked on the target subnet. This constitutes a significant, concrete Chain Fusion and cross-chain infrastructure security impact with direct, lasting harm to users whose assets on the target subnet become inaccessible. This matches the **High** bounty impact: *"Significant Chain Fusion, ck-token, ledger, Rosetta, boundary/API, XRC, Internet Identity, NNS, SNS, or infrastructure security impact with concrete user or protocol harm."*

## Likelihood Explanation

- The attacker needs only to be selected as block proposer for a single round — no privileged role, no threshold corruption required.
- Active `idkg_dealings_contexts` are part of replicated state visible to all subnet nodes, so the attacker can deterministically craft the matching request key.
- The attack requires constructing a single malicious block payload; it is fully deterministic.
- Honest nodes will notarize the block because `validate_reshare_dealings` returns `Ok(...)` for the fabricated entry.
- The suppression is self-reinforcing: the spurious entry is re-emitted as `ReportedToExecution` by every subsequent honest block proposer until the context is removed from execution state — which never happens because no response is ever delivered.

## Recommendation

In `validate_reshare_dealings`, add an explicit check over `curr_payload.xnet_reshare_agreements` that rejects any `ReportedToExecution` entry that is absent from `prev_payload.xnet_reshare_agreements`:

```rust
for (request, dealings) in curr_payload.xnet_reshare_agreements.iter() {
    match dealings {
        idkg::CompletedReshareRequest::Unreported(dealings) => {
            if !prev_payload.xnet_reshare_agreements.contains_key(request) {
                if !prev_payload.ongoing_xnet_reshares.contains_key(request) {
                    return Err(XNetReshareAgreementWithoutRequest(request.clone()).into());
                }
                new_reshare_agreement.insert(request.clone(), dealings);
            }
        }
        idkg::CompletedReshareRequest::ReportedToExecution => {
            // A ReportedToExecution entry must have existed in the previous payload.
            if !prev_payload.xnet_reshare_agreements.contains_key(request) {
                return Err(XNetReshareAgreementWithoutRequest(request.clone()).into());
            }
        }
    }
}
```

This ensures that `ReportedToExecution` entries can only persist across blocks, never appear fresh.

5. Simulate subsequent honest block production: call `update_completed_reshare_requests` with the active context for `R` still present; assert the spurious entry survives in `xnet_reshare_agreements` as `ReportedToExecution`.
6. Call `initiate_reshare_requests` with `R` in the reshare request set; assert `R` is **not** added to `ongoing_xnet_reshares` because `xnet_reshare_agreements.contains_key(&R)` is true.
7. Repeat steps 5–6 for multiple rounds; assert the suppression is permanent as long as the context remains active.
