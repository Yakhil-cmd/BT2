### Title
WalletConnect `chia_cancelOffer` Approval Dialog Omits Offer Asset Details, Enabling Blind Cancellation — (`packages/gui/src/electron/commands/Commands.ts`, `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary

The `chia_cancelOffer` WalletConnect command approval dialog shows only a raw `trade_id` hex string and a fee amount. It never resolves the trade_id to the underlying offer's assets and amounts. A connected dApp can target any of the user's open offers by trade_id, and the user has no way to identify which offer they are cancelling before clicking "Proceed."

### Finding Description

**Step 1 — dApp command schema hides `secure`, shows only raw `trade_id`**

In `Commands.ts`, the `chia_cancelOffer` dapp entry overrides the parent params list to expose only `trade_id` (type `string`) and `fee` (type `bigint`). The `secure` param is hidden and hard-coded to `true` via `defaults`. [1](#0-0) 

**Step 2 — `humanizeParams` renders `trade_id` as a raw string**

`humanizeDappCommand` calls `humanizeParams` with the dapp's param list. For `trade_id` (type `'string'`), no resolution occurs — the raw hex value is passed through verbatim as a label/value row. [2](#0-1) 

**Step 3 — `parseCommandDisplay` has no handler for `chia_wallet.cancel_offer`**

`parseCommandDisplay` is the function that enriches the confirmation dialog with resolved asset details. It explicitly handles `chia_wallet.take_offer` (resolves offer summary → spending/receiving assets) and `chia_wallet.create_offer_for_ids` (resolves wallet delta), but returns `undefined` for all other commands, including `chia_wallet.cancel_offer`. [3](#0-2) 

The `display` prop passed to the `Confirm` dialog is therefore `undefined` for cancel-offer, so no `walletDelta` panel is rendered. [4](#0-3) 

**Step 4 — Confirmation dialog shows only `Trade Id: <hex>` and `Fee: X XCH`**

The `Confirm` component renders the `rows` array (label/value pairs from `humanizeParams`) and, when `display` is present, a rich wallet-delta panel. For `chia_cancelOffer`, `display` is `undefined`, so the user sees only:

```
Trade Id: a3f9...c12b
Fee:      0 XCH
```

There is no indication of which assets or amounts are locked in the offer being cancelled. [5](#0-4) 

**Step 5 — Attack path**

1. Malicious dApp connects via WalletConnect and is granted `chia_cancelOffer` (and optionally `chia_getOffers` to enumerate trade_ids).
2. dApp calls `chia_cancelOffer({ trade_id: victimTradeId, fee: 0 })`.
3. Approval dialog shows only `Trade Id: victimTradeId` — no assets, no amounts, no counterparty.
4. User, unable to identify the offer, approves.
5. `cancel_offer` RPC is dispatched with `secure: true` (on-chain cancellation), permanently cancelling the offer and spending a blockchain transaction.

### Impact Explanation

A user unknowingly cancels a valuable open offer (XCH, CAT, NFT) via WalletConnect because the approval dialog provides no actionable information about what is being cancelled. The cancellation is on-chain and irreversible. This fits the High impact category: *WalletConnect state causes a user to approve the wrong action without being shown the correct asset/amount/status.*

### Likelihood Explanation

Any dApp that has been granted `chia_cancelOffer` permission (which the user must approve during pairing, but may do so without understanding the implication) can exploit this. The dApp can enumerate trade_ids via `chia_getOffers` if that command is also permitted. The user has no defense at the confirmation step because the dialog is informationally blind.

### Recommendation

Add a `chia_wallet.cancel_offer` branch to `parseCommandDisplay` that:
1. Calls `get_offer` (or `get_all_offers`) with the provided `trade_id` to fetch the offer record.
2. Resolves the offer summary to a `walletDelta` (spending/receiving assets and amounts).
3. Returns a `display.walletDelta` so the `Confirm` dialog renders the full asset panel — identical to what `chia_wallet.take_offer` already does. [6](#0-5) 

### Proof of Concept

```
1. Connect a WalletConnect dApp that requests chia_cancelOffer permission.
2. User approves the pairing.
3. dApp calls: chia_cancelOffer({ trade_id: "<any open offer trade_id>", fee: 0 })
4. Observe the confirmation dialog: only "Trade Id: <hex>" and "Fee: 0 XCH" are shown.
5. User clicks Proceed.
6. cancel_offer RPC fires with secure=true; offer is cancelled on-chain.
7. Assert: no asset/amount information was shown before the user confirmed.
```

export async function humanizeParams(params: ParamSchema[], data: Record<string, unknown>, networkPrefix?: string) {
  // hidden and optional params that are not presented are not shown in the confirm dialog
  const visibleParams = params.filter((param) => {
    const { hide, isOptional, name } = param;

    if (hide === true) {
      return false;
    }

    const isDefined = name in data;
    if (isOptional && !isDefined) {
      return false;
    }

    return true;
  });

  // humanize the visible params
  const humanizedParams = await Promise.all(
    visibleParams.map(async (param) => {
      const value = data[param.name];

      const humanizedValue = await humanizeParamValue(param, value, data, networkPrefix);

      return {
        field: param.name,
        label: param.label(),
        value: humanizedValue,
      };
    }),
  );

  return humanizedParams;

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }

  return undefined;
