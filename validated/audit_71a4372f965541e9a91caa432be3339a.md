### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The gateway's stateful admission check for V3 transactions validates the transaction's `max_price_per_unit` against the **previous block's** `l2_gas_price` (block N-1's execution price). However, the block being built (block N) will execute at `next_l2_gas_price`, which is stored in block N-1's header and is computed from block N-1's gas usage via the EIP-1559 formula. Using the wrong price as the reference causes the gateway to reject valid transactions (when price decreases) and admit transactions that will fail blockifier pre-validation (when price increases). A developer TODO in the code explicitly acknowledges this is wrong.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, `validate_resource_bounds` reads the L2 gas price from `get_block_info()`:

```rust
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;   // ← price used in block N-1, NOT the price for block N
``` [1](#0-0) 

This value is then used as the reference in `validate_tx_l2_gas_price_within_threshold`:

```rust
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0).to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...)
}
``` [2](#0-1) 

The correct reference is `next_l2_gas_price`, which is computed by `calculate_next_l2_gas_price_for_fin` at the end of block N-1 and stored in block N-1's header as `next_l2_gas_price`. This is the price at which block N will actually execute. The TODO comment in the code confirms the intent:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
``` [3](#0-2) 

The `next_l2_gas_price` is computed by the EIP-1559 oracle and stored in the block header:

```rust
let block_header_without_hash = BlockHeaderWithoutHash {
    l2_gas_price,
    l2_gas_consumed: l2_gas_used,
    next_l2_gas_price: self.l2_gas_price,  // ← this is what the gateway should use
    ...
};
``` [4](#0-3) 

The `next_l2_gas_price` is calculated from block N-1's actual gas usage:

```rust
let next_l2_gas_price = calculate_next_l2_gas_price_for_fin(
    args.l2_gas_price,
    args.build_param.height,
    info.l2_gas_used,
    ...
);
``` [5](#0-4) 

The EIP-1559 formula adjusts the price up when `gas_used > gas_target` and down when `gas_used < gas_target`:

```rust
let adjusted_price_u256 =
    if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };
``` [6](#0-5) 

### Impact Explanation

**Scenario A — price decreases (block N-1 was under-utilized):** `next_l2_gas_price < l2_gas_price`. The gateway's threshold is computed as `min_gas_price_percentage * l2_gas_price`, which is higher than the correct threshold `min_gas_price_percentage * next_l2_gas_price`. A user submitting a V3 transaction with `max_price_per_unit` in the range `[correct_threshold, stale_threshold)` has a transaction that is economically valid for block N but is **rejected by the gateway**. The user receives a `GAS_PRICE_TOO_LOW` error and cannot submit the transaction.

**Scenario B — price increases (block N-1 was over-utilized):** `next_l2_gas_price > l2_gas_price`. The gateway's threshold is lower than the correct threshold. A transaction with `max_price_per_unit` in the range `[stale_threshold, correct_threshold)` passes the gateway but will fail the blockifier's `check_fee_bounds` pre-validation. The gateway **admits an invalid transaction** into the mempool.

Both scenarios match the allowed High impact: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."

### Likelihood Explanation

The price divergence between `l2_gas_price` and `next_l2_gas_price` occurs every block where gas usage deviates from the target. The EIP-1559 formula can move the price by up to `price / gas_price_max_change_denominator` per block. With a default `gas_price_max_change_denominator` and a `min_gas_price_percentage` of e.g. 90%, any block that is significantly under- or over-utilized will create a window where the gateway's admission decision diverges from the blockifier's. This is a persistent, deterministic condition that any user can observe and exploit by monitoring block gas usage.

### Recommendation

Replace the `l2_gas_price` read with `next_l2_gas_price` from the block header. The `GatewayFixedBlockStateReader::get_block_info` interface (or a new method) should expose `next_l2_gas_price` from the stored `BlockHeaderWithoutHash`, and `validate_resource_bounds` should use that value as the reference price. This is already acknowledged by the TODO comment in the code.

### Proof of Concept

1. Observe that block N-1 had very low gas usage (e.g., near-empty block). The EIP-1559 formula will compute `next_l2_gas_price` significantly lower than `l2_gas_price`.
2. Construct a V3 `INVOKE` transaction with `l2_gas.max_price_per_unit = P` where `min_gas_price_percentage * next_l2_gas_price <= P < min_gas_price_percentage * l2_gas_price`.
3. Submit the transaction to the gateway's `add_transaction` endpoint.
4. The gateway calls `validate_tx_l2_gas_price_within_threshold` with `previous_block_l2_gas_price = l2_gas_price` and computes `threshold = min_gas_price_percentage * l2_gas_price > P`.
5. The gateway returns `StarknetErrorCode::GAS_PRICE_TOO_LOW` and rejects the transaction.
6. However, the blockifier would accept this transaction in block N (since `P >= min_gas_price_percentage * next_l2_gas_price`), demonstrating that the gateway incorrectly rejected a valid transaction.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
