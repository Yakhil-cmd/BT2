### Title
Gateway Rejects Valid Transactions When L2 Gas Price Decreases Due to Stale Reference Price - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The gateway's stateful validator compares a transaction's `max_price_per_unit` for L2 gas against the **previous committed block's** L2 gas price as the admission threshold. When the L2 gas price decreases between blocks (normal under EIP-1559 dynamics), transactions priced at the new valid (lower) price are incorrectly rejected at the gateway, even though they would pass blockifier execution validation.

### Finding Description

In `validate_tx_l2_gas_price_within_threshold` (lines 359–390 of `crates/apollo_gateway/src/stateful_transaction_validator.rs`), the gateway fetches `previous_block_l2_gas_price` from the last committed block and uses it as the sole reference for the minimum gas price threshold:

```rust
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

The threshold is then computed as:

```rust
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0).to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(StarknetError { code: GAS_PRICE_TOO_LOW, ... });
}
``` [2](#0-1) 

With the production default `min_gas_price_percentage = 100`, the threshold equals the previous block's L2 gas price exactly. [3](#0-2) 

The code itself contains a TODO acknowledging the wrong reference is being used:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
``` [4](#0-3) 

The **correct** reference is the **next block's** L2 gas price (the block being built), not the previous block's price. The next block's price is computed by the consensus orchestrator via EIP-1559 dynamics and can be lower than the previous block's price when block utilization is low.

Meanwhile, the blockifier's `check_fee_bounds` — which runs during actual execution — correctly uses the **current block's** gas price as the lower bound:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    // MaxGasPriceTooLow
}
``` [5](#0-4) 

This creates a split: the gateway rejects a transaction that the blockifier would accept.

### Impact Explanation

A user submitting a transaction priced at the correct next-block L2 gas price is rejected at the gateway admission layer with `GAS_PRICE_TOO_LOW` when the gas price has decreased. The transaction is economically valid and would execute successfully, but it never reaches the mempool. This is a **High** impact admission failure: the gateway rejects valid transactions before sequencing.

### Likelihood Explanation

The L2 gas price decreases whenever block gas usage falls below the EIP-1559 target. This is a routine network condition (e.g., during low-traffic periods). With `min_gas_price_percentage = 100` (the production default), any decrease — even by 1 FRI — causes the rejection of transactions priced at the new valid level.

### Recommendation

Replace `previous_block_l2_gas_price` with `next_l2_gas_price` from the block header (as the TODO already notes). The gateway should validate the transaction's gas price against the price that will actually govern the block in which the transaction will be included, not the price of the already-committed previous block.

### Proof of Concept

1. Previous committed block L2 gas price: **100 FRI**
2. Next block L2 gas price (EIP-1559 decrease due to low usage): **90 FRI**
3. User submits `AllResources` invoke transaction with `l2_gas.max_price_per_unit = 90 FRI`
4. Gateway threshold: `(100/100) × 100 = 100 FRI`
5. Gateway check: `90 < 100` → **rejected** with `GAS_PRICE_TOO_LOW`
6. Blockifier check (if it were reached): `90 >= 90` (next block price) → **would pass**

The transaction is valid for the next block but is silently dropped at the gateway. The user must wait for the next block to be committed before the gateway's reference price updates, at which point they can resubmit — but by then the gas price may have changed again. [6](#0-5)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
