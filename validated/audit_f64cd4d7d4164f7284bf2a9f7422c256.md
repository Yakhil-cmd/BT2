### Title
Bank Precompile `transfer` Allows Any Contract to Drain `evm/<contract>` Native Tokens from Arbitrary User Accounts Without Authorization - (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary

The bank precompile's `transfer` method accepts an arbitrary `sender` address from calldata and moves native bank tokens of denom `evm/<caller_contract>` from that address to any recipient — with no check that `contract.Caller()` equals `sender` and no allowance/approval mechanism. Any unprivileged contract can therefore drain `evm/<contract>` native bank tokens from any user who holds them.

### Finding Description

The bank precompile at address `0x0000000000000000000000000000000000000064` exposes four methods: `mint`, `burn`, `balanceOf`, and `transfer`. The denom managed by each call is always `evm/<contract.Caller()>`, so each deployed contract controls its own native denomination.

In the `transfer` case the implementation is:

```go
// x/cronos/keeper/precompiles/bank.go  lines 175-192
sender    := args[0].(common.Address)   // ← taken from calldata, not from contract.Caller()
recipient := args[1].(common.Address)
amount    := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())    // evm/<calling_contract>
amt   := sdk.NewCoin(denom, ...)
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

`from` is derived entirely from the first argument supplied by the calling contract, not from `contract.Caller()`. There is no guard of the form `require(sender == contract.Caller())` and no allowance table. The only check performed is `checkBlockedAddr(to)`, which only validates the recipient.

Compare with `mint`/`burn`, where the target address is also taken from args — but those operations are semantically "the contract mints/burns its own token for a user", which is intentional. `transfer`, however, moves tokens *between two arbitrary third-party accounts* at the contract's sole discretion.

### Impact Explanation

**Critical — Unauthorized transfer of native bank assets.**

Any EVM contract can call `bank.transfer(victim, attacker, amount)` and the bank module will execute `SendCoins(victim, attacker, amount_of_evm/<contract>)` without any consent from `victim`. This is a direct, unconditional theft of native Cosmos bank-module tokens.

The affected asset class (`evm/<contract_address>`) is a real Cosmos SDK bank denomination. It is minted by the same contract via `bank.mint(user, amount)` and is visible in the bank module's balance store. Any protocol or dApp that uses the bank precompile to represent user balances (the intended use-case, as shown in `TestBank.sol`) is fully exposed: the contract deployer — or any logic path inside the contract — can drain every user's native balance at will, with no on-chain recourse.

### Likelihood Explanation

The entry path is fully unprivileged: anyone can deploy an EVM contract on Cronos. The exploit requires only two transactions:
1. Accumulate victim balances via `bank.mint(victim, amount)` (or wait for users to call `moveToNative`).
2. Call `bank.transfer(victim, attacker, amount)` from the same contract.

No admin key, governance vote, or special permission is needed. The `TestBank.sol` integration test contract already demonstrates the exact pattern (`nativeTransfer` calls `bank.transfer(msg.sender, recipient, amount)`), confirming the code path is live and reachable.

### Recommendation

Enforce that the `sender` argument must equal `contract.Caller()`:

```go
case TransferMethodName:
    ...
    sender    := args[0].(common.Address)
    // Guard: only the calling contract may initiate a transfer from an account
    if sender != contract.Caller() {
        return nil, errors.New("transfer: sender must be the calling contract")
    }
    ...
```

Alternatively, if peer-to-peer transfers between arbitrary user accounts are desired, implement an ERC-20-style allowance table keyed by `(owner, spender=contract.Caller())` so that users must explicitly approve the contract before it can move their tokens.
