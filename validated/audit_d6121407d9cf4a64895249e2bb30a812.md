### Title
Staker Unstake Intent Never Expires and Is Executable by Any Caller, Enabling Forced Permanent Removal - (File: src/staking/staking.cairo)

### Summary
Once a staker calls `unstake_intent()`, the intent is stored permanently with no upper-bound expiration. `unstake_action` is callable by **any address** and only enforces a lower-bound time check. Unlike pool members who can cancel their exit intent by calling `exit_delegation_pool_intent(amount: 0)`, stakers have no cancellation mechanism. After the exit window elapses, any unprivileged caller can permanently evict a staker who may have changed their mind, destroying their staking position forever.

### Finding Description

`unstake_intent()` sets an absolute `unstake_time = now + exit_wait_window` and immediately removes the staker's own and delegated balances from the total stake. The intent is written to storage with no deadline: [1](#0-0) 

`unstake_action` enforces only a **lower-bound** check — there is no upper-bound expiration: [2](#0-1) 

The spec confirms `unstake_action` is open to any caller: [3](#0-2) 

Critically, the spec also states that calling `unstake_intent()` permanently blocks the staker from ever re-staking from the same address: [4](#0-3) 

Pool members have a cancellation path (`exit_delegation_pool_intent(0)` sets `unpool_time = None`): [5](#0-4) 

No equivalent cancel path exists for stakers. There is no `cancel_unstake_intent` function anywhere in the staking contract.

### Impact Explanation

After the exit window passes, any unprivileged caller can invoke `unstake_action(staker_address)` on a staker who has changed their mind. The staker:
- Is permanently deleted from the protocol (`remove_staker` is called, the record is erased).
- Can **never** re-stake from that address — the protocol enforces this as a permanent invariant.
- Loses all future staking rewards from that address.

The staker's principal and accrued-to-date rewards are returned, so there is no direct fund theft, but the permanent destruction of the staking position and all future yield constitutes griefing with concrete, irreversible damage to the user.

### Likelihood Explanation

A staker may call `unstake_intent()` under duress, by mistake, or while evaluating an exit that they later decide against. Since the intent cannot be cancelled and the exit window is a fixed delay (not a deadline), the staker is trapped indefinitely in a state where they cannot attest, cannot earn rewards, and cannot re-stake. Any observer — including a competitor validator or a griefing bot — can monitor the chain for stakers whose `unstake_time` has elapsed and call `unstake_action` to permanently evict them. The entry path requires no privilege, no key, and no external dependency.

### Recommendation

1. **Add an expiration deadline** to the unstake intent (e.g., `unstake_time + 2 * exit_wait_window`). After expiration, `unstake_action` should revert, and the staker should be allowed to cancel or re-activate.
2. **Add a `cancel_unstake_intent` function** for stakers, symmetric to the pool member's ability to cancel via `exit_delegation_pool_intent(0)`. This would restore the staker's balance to the total stake and clear `unstake_time`.

### Proof of Concept

1. Staker calls `unstake_intent()` — `unstake_time` is set, stake is removed from total.
2. The staker reconsiders (market conditions changed, mistake, etc.) but finds no cancellation path.
3. `exit_wait_window` elapses (e.g., 21 days).
4. Attacker calls `unstake_action(staker_address)` — permissioned to any caller per spec.
5. `unstake_action` passes the only guard (`Time::now() >= unstake_time`) and permanently deletes the staker record.
6. The staker's address is now permanently blacklisted from re-staking; all future yield from that position is lost. [6](#0-5)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }
