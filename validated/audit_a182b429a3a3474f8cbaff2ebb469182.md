### Title
Cloneable `PresignOutput` and Non-Consuming `rerandomize_presign` API Allow Presignature Reuse, Enabling Private Key Extraction — (`src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both the OT-based and Robust ECDSA schemes document that each presignature must be consumed **exactly once**. However, the library's own API design provides no type-level or runtime enforcement of this invariant: `PresignOutput` derives `Clone` in both modules, and `rerandomize_presign` accepts `&PresignOutput` (a shared reference, not an owned value). A malicious coordinator can instruct honest participants to rerandomize and sign with the same presignature for two different `(h, ε)` pairs. The resulting signatures expose multiplicatively related nonces, and the aggregate secret key can be recovered via standard ECDSA nonce-reuse algebra.

---

### Finding Description

**Root cause — OT-based ECDSA (`src/ecdsa/ot_based_ecdsa/mod.rs`):**

`PresignOutput` is declared with `#[derive(Clone)]`:

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput {
    pub big_r: AffinePoint,
    pub k: Scalar,      // nonce share — MUST be one-time
    pub sigma: Scalar,
}
``` [1](#0-0) 

`rerandomize_presign` takes the presignature by shared reference, leaving the original intact and reusable:

```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // ← shared ref, NOT consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> {
``` [2](#0-1) 

**Root cause — Robust ECDSA (`src/ecdsa/robust_ecdsa/mod.rs`):**

Identical pattern: `PresignOutput` derives `Clone` and `rerandomize_presign` takes `&PresignOutput`: [3](#0-2) [4](#0-3) 

**What the library itself says must not happen:**

The orchestration document states:

> "It's **critical** that the output is then destroyed, so that no other group of parties attempts to re-use that output for another phase." [5](#0-4) 

The robust ECDSA security section states:

> "**Never reuse a presignature**, even across failed, aborted, or partially completed signing sessions." [6](#0-5) 

Both `presign.rs` files carry the same comment in their doc-strings:

> "it's crucial that a presignature is never reused." [7](#0-6) [8](#0-7) 

Despite these warnings, the library provides **zero enforcement**. `PresignOutput` can be `.clone()`d freely, and `rerandomize_presign(&presign_output, &args)` can be called any number of times on the same value.

**Contrast with the signing phase:** `sign()` in both modules correctly takes `presignature: RerandomizedPresignOutput` **by value** (consuming it), so a `RerandomizedPresignOutput` cannot be directly reused in a second `sign()` call. The gap is entirely at the earlier `rerandomize_presign` step, where the original `PresignOutput` is not consumed. [9](#0-8) [10](#0-9) 

---

### Impact Explanation

In ECDSA, the nonce `k` (embedded in `PresignOutput.k` / the shares `alpha, beta, c, e`) must be unique per signature. If the same presignature is rerandomized for two different `(h₁, ε₁)` and `(h₂, ε₂)` pairs, the resulting nonces are multiplicatively related by a publicly computable factor (the ratio of the two `delta` values, both derived from public HKDF inputs). Given two signatures `(R, s₁)` and `(R', s₂)` sharing a related nonce, the aggregate secret key `x` is recoverable by standard linear algebra over the scalar field.

The library's own security section confirms this:

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks." [11](#0-10) 

**Impact category:** Critical — Extraction/reconstruction of the aggregate private signing key.

---

### Likelihood Explanation

The coordinator is a designated, trusted role in both signing protocols. A malicious coordinator can:

1. Obtain or observe participants' `PresignOutput` shares (participants hold them locally and use them when instructed).
2. Instruct participants to sign message `h₁` with tweak `ε₁` using presignature `P`.
3. Immediately instruct the same participants to sign message `h₂` with tweak `ε₂` using the **same** presignature `P` (e.g., by claiming the first session failed or was aborted).
4. Each honest participant, holding a cloneable `PresignOutput`, calls `rerandomize_presign` a second time — the library does not prevent this.
5. The coordinator collects both sets of signature shares and recovers the private key offline.

No cryptographic break, no leaked keys, and no external dependency is required. The attack is entirely within the library's documented API surface and requires only a malicious coordinator, which is an explicitly considered adversary in the security model. [12](#0-11) 

---

### Recommendation

1. **Remove `Clone` from `PresignOutput`** in both `src/ecdsa/ot_based_ecdsa/mod.rs` and `src/ecdsa/robust_ecdsa/mod.rs`. A presignature share is a one-time secret; cloneability directly contradicts this invariant.

2. **Change `rerandomize_presign` to consume `PresignOutput` by value** (take `presignature: PresignOutput`, not `presignature: &PresignOutput`). This makes single-use a compile-time guarantee: once rerandomized, the original presignature is moved and cannot be rerandomized again.

3. **Remove `Clone` from `RerandomizedPresignOutput`** as well, since a rerandomized presignature is equally one-time-use.

These changes align the API with the documented invariant and leverage Rust's ownership system — the same mechanism already used correctly for `RerandomizedPresignOutput` in the `sign()` call — to enforce the security property at compile time rather than relying solely on caller discipline.

#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput {
    /// The public nonce commitment.
    #[zeroize[skip]]
    pub big_r: AffinePoint,
    /// Our share of the nonce value.
    pub k: Scalar,
    /// Our share of the sigma value.
    pub sigma: Scalar,
}

/// The output of the presigning protocol.
/// Contains the signature precomputed elements
/// independently of the message
#[derive(Debug, Clone, Serialize, Deserialize, ZeroizeOnDrop)]
pub struct RerandomizedPresignOutput {
    /// The rerandomized public nonce commitment.
    #[zeroize[skip]]
    pub big_r: AffinePoint,
    /// Our rerandomized share of the nonce value.
    pub k: Scalar,
    /// Our rerandomized share of the sigma value.
    pub sigma: Scalar,
}

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
