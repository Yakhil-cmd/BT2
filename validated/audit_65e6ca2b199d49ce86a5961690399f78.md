### Title
Blocklist Enforcement Skipped During DeliverTx, Allowing Blocked Addresses to Transact via Validator Inclusion - (File: app/block_address.go)

### Summary
The `BlockAddressesDecorator` ante handler gates **all** blocklist enforcement inside `if ctx.IsCheckTx()`. During block execution (`DeliverTx`), the guard is never entered and the handler unconditionally calls `next`. A malicious or colluding validator can include a blocked address's transaction directly in a block proposal; every node will execute it without any blocklist check.

### Finding Description
In `app/block_address.go`, `AnteHandle` is structured as:

```go
func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
    if ctx.IsCheckTx() {
        // signer blocklist check
        // EVM destination blocklist check
        // EIP-7702 authority/target blocklist check
        // MsgStoreBlockList admin check
    }
    return next(ctx, tx, simulate)   // ← always reached during DeliverTx
}
``` [1](#0-0) 

Every check — signer, EVM destination, EIP-7702 authorization list, and `MsgStoreBlockList` sender — lives exclusively inside the `ctx.IsCheckTx()` branch. During `DeliverTx` (actual block execution), `ctx.IsCheckTx()` returns `false`, so the entire body is skipped and `next` is called unconditionally. [2](#0-1) 

The blocklist itself is stored and managed via `MsgStoreBlockList`, which is an admin-only operation: [3](#0-2) 

The intent is clearly to prevent blocked addresses from transacting. However, the enforcement point is only the mempool gate (`CheckTx`), not the execution gate (`DeliverTx`).

### Impact Explanation
Any validator (or a blocked address that is itself a validator, or one that bribes/colludes with a validator) can include a blocked address's transaction in a block proposal. All other validators execute the block via `DeliverTx`, where `BlockAddressesDecorator` performs zero checks. The blocked address's `MsgConvertVouchers`, `MsgTransferTokens`, EVM transactions, or any other message executes successfully on every node. This is a direct, complete bypass of the Cronos admin-controlled blocklist — a High-severity authorization bypass explicitly listed in the allowed impact scope ("block-list … authorization checks"). [4](#0-3) 

### Likelihood Explanation
The attacker needs block-proposal rights. In Cosmos, any bonded validator can propose blocks. A blocked address with sufficient stake is itself a validator; alternatively, a blocked address can bribe or collude with any single validator. No key compromise, no cryptographic break, and no admin access is required — only normal validator behavior (proposing a block with chosen transactions).

### Recommendation
Remove the `if ctx.IsCheckTx()` guard. The blocklist checks must run unconditionally on every call to `AnteHandle`, covering both `CheckTx` and `DeliverTx`. If performance during simulation is a concern, the `simulate` parameter (already passed to `next`) can be used to skip the check only during gas-estimation simulations, but never during actual execution.

```go
func (bad BlockAddressesDecorator) AnteHandle(...) (...) {
    // Remove the `if ctx.IsCheckTx()` wrapper:
    if sigTx, ok := tx.(signing.SigVerifiableTx); ok { ... }
    for _, msg := range tx.GetMsgs() { ... }
    return next(ctx, tx, simulate)
}
```

### Proof of Concept

1. Admin calls `MsgStoreBlockList` to add Alice's Cosmos/EVM address to the blocklist.
2. Alice constructs a valid `MsgConvertVouchers` (or any EVM tx) signed by her key.
3. Alice (as a validator, or via a colluding validator) proposes a block containing her transaction. The transaction bypasses `CheckTx` on the proposer node because the proposer can inject it directly into the block without going through the mempool.
4. Every validator executes the block. `BlockAddressesDecorator.AnteHandle` is called with `ctx.IsCheckTx() == false`.
5. The `if ctx.IsCheckTx()` branch is not entered; `next(ctx, tx, simulate)` is called immediately.
6. Alice's transaction executes successfully — her vouchers are converted, her IBC transfer goes through, or her EVM transaction is processed — despite her address being on the blocklist. [1](#0-0)

func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)
```

func (k msgServer) StoreBlockList(goCtx context.Context, msg *types.MsgStoreBlockList) (*types.MsgStoreBlockListResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	ctx.KVStore(k.storeKey).Set(types.KeyPrefixBlockList, msg.Blob)
	return &types.MsgStoreBlockListResponse{}, nil
