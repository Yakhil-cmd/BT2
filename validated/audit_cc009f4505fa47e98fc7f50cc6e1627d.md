### Title
Blocked Destination Address Receives CRO/CRC20 Tokens via Internal EVM Sub-Calls, Bypassing the Blocklist - (File: app/proposal.go)

---

### Summary

Cronos's blocklist enforces that a blocked address cannot be the direct `To` destination of an EVM transaction. However, this check only inspects the **top-level transaction destination** (`ethTx.To()`). Internal EVM sub-calls made by smart contracts during execution are never validated against the blocklist. An unprivileged attacker can deploy an intermediate contract and route CRO value transfers or CRC20/ERC20 token transfers to a blocked destination address, fully bypassing the restriction.

---

### Finding Description

`ValidateTransaction` in `app/proposal.go` and `AnteHandle` in `app/block_address.go` both check the blocklist against the outer transaction's `ethTx.To()` field and the tx signer. Neither function inspects internal EVM call destinations (sub-calls generated by CALL/DELEGATECALL opcodes, or ERC20 `transfer`/`transferFrom` invocations made inside a contract). [1](#0-0) 

The check is:

```go
if ethTx.To() != nil {
    encoded, err := h.addressCodec.BytesToString(ethTx.To().Bytes())
    ...
    if _, ok := h.blocklist[encoded]; ok {
        return fmt.Errorf("destination address is blocked: %s", encoded)
    }
}
``` [2](#0-1) 

This check fires only on the outer `To` field of the `MsgEthereumTx`. When a smart contract executes and internally calls `payable(blocked).transfer(value)` or calls `erc20.transfer(blocked, amount)`, those sub-call destinations are invisible to `ValidateTransaction` and `AnteHandle`. The blocklist is enforced at the mempool/proposal layer, not inside the EVM execution engine.

The destination-blocklist feature was added in v1.6.0 (PR #1922) and patched for EIP-7702 authorization lists (PR #1924), but the internal-call gap was never addressed. [3](#0-2) 

---

### Impact Explanation

**High — Bypass of Cronos block-list authorization checks.**

The blocklist is a compliance/sanctions enforcement mechanism. The destination check is specifically intended to prevent blocked addresses from receiving funds. An attacker can trivially circumvent this by routing any CRO (native token) or CRC20/ERC20 token transfer through an intermediate contract. The blocked address receives the assets as if it were never listed, defeating the entire purpose of the destination restriction.

This maps directly to the allowed High impact: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."*

---

### Likelihood Explanation

**High.** No special privileges are required. Any Ethereum-compatible address can deploy a proxy contract on Cronos. The attacker only needs to know the blocked address and deploy a one-function contract. The bypass works for both native CRO transfers and any ERC20/CRC20 token transfer.

---

### Recommendation

The blocklist destination check must be enforced inside EVM execution, not only at the transaction boundary. Options:

1. **EVM StateDB hook**: Intercept `AddBalance` / `SubBalance` calls in the StateDB and revert if the recipient is a blocked address.
2. **Custom EVM tracer**: Reject execution if any internal call's `to` field matches a blocked address.
3. **Post-execution log scan**: After EVM execution, scan all internal call traces for blocked destinations and revert the transaction.

Option 1 is the most robust because it catches both native-value transfers and token transfers that ultimately settle through the bank module.

---

### Proof of Concept

**Setup**: Admin blocks address `B` via `MsgStoreBlockList`.

**Attacker deploys proxy contract `C`**:
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;
contract Proxy {
    function forwardNative(address payable blocked) external payable {
        blocked.transfer(msg.value); // internal sub-call, not checked by blocklist
    }
    function forwardToken(address token, address blocked, uint256 amount) external {
        IERC20(token).transfer(blocked, amount); // internal sub-call, not checked
    }
}
```

**Attacker sends EVM tx**:
- `from`: attacker (not blocked)
- `to`: `C` (not blocked — passes `ValidateTransaction`)
- `data`: `forwardNative(B)` with `value = X`

**Result**: `ValidateTransaction` passes because `ethTx.To() == C` (not blocked). The EVM executes `C`, which internally calls `B.transfer(X)`. Address `B` receives `X` CRO despite being on the blocklist.

The same path works for CRC20/ERC20 tokens: attacker holds tokens, calls `C.forwardToken(crc20Addr, B, amount)`, and `B` receives the tokens via the internal `transfer(B, amount)` sub-call. [1](#0-0) [2](#0-1) [4](#0-3)

func (h *ProposalHandler) ValidateTransaction(tx sdk.Tx, txBz []byte) error {
	if len(h.blocklist) == 0 {
		// fast path, accept all txs
		return nil
	}

	var err error
	if tx == nil {
		tx, err = h.TxDecoder(txBz)
		if err != nil {
			return err
		}
	}

	sigTx, ok := tx.(signing.SigVerifiableTx)
	if !ok {
		return fmt.Errorf("tx of type %T does not implement SigVerifiableTx", tx)
	}

	signers, err := sigTx.GetSigners()
	if err != nil {
		return err
	}
	for _, signer := range signers {
		encoded, err := h.addressCodec.BytesToString(signer)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", signer, err)
		}
		if _, ok := h.blocklist[encoded]; ok {
			return fmt.Errorf("signer is blocked: %s", encoded)
		}
	}

	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if ok {
			ethTx := msgEthTx.AsTransaction()
			// check the destination address
			if ethTx.To() != nil {
				encoded, err := h.addressCodec.BytesToString(ethTx.To().Bytes())
				if err != nil {
					return fmt.Errorf("invalid bech32 address: %s, err: %w", ethTx.To(), err)
				}
				if _, ok := h.blocklist[encoded]; ok {
					return fmt.Errorf("destination address is blocked: %s", encoded)
				}
			}
			// check EIP-7702 authorisation list
			if ethTx.SetCodeAuthorizations() != nil {
				for _, auth := range ethTx.SetCodeAuthorizations() {
					addr, err := auth.Authority()
					if err == nil {
						encoded, err := h.addressCodec.BytesToString(addr.Bytes())
						if err != nil {
							return fmt.Errorf("invalid bech32 address: %s, err: %w", addr, err)
						}
						if _, ok := h.blocklist[encoded]; ok {
							return fmt.Errorf("signer is blocked: %s", encoded)
						}
					}
					// check the target address
					encoded, err := h.addressCodec.BytesToString(auth.Address.Bytes())
					if err != nil {
						return fmt.Errorf("invalid bech32 address: %s, err: %w", auth.Address, err)
					}
					if _, ok := h.blocklist[encoded]; ok {
						return fmt.Errorf("authorisation address is blocked: %s", encoded)
					}
				}
			}
		}
	}

	return nil
}
