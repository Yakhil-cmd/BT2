### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (`src/staking/staking.cairo`)

### Summary

`update_rewards` is specified as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged address can call it once per block with `disable_rewards: true`, consuming the global `last_reward_block` slot and permanently discarding that block's rewards for every staker.

### Finding Description

The spec explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation's only gate is `general_prerequisites()`: [2](#0-1) 

`general_prerequisites` checks only that the contract is unpaused and the caller is non-zero — no role or identity check:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [3](#0-2) 

`last_reward_block` is a **single global** storage slot shared across all stakers: [4](#0-3) 

The guard that prevents double-updates compares against this global value: [5](#0-4) 

The global slot is written unconditionally before the `disable_rewards` branch: [6](#0-5) 

**Attack sequence per block:**
1. Attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)`.
2. `last_reward_block` is set to `current_block_number`.
3. The function returns early — zero rewards distributed.
4. Every subsequent call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. That block's rewards are permanently lost for all stakers and delegators.

The attacker only needs to know one valid, active staker address (publicly discoverable from `NewStaker` events) and pay gas once per block.

### Impact Explanation

Every block's consensus rewards can be suppressed indefinitely at negligible cost. Stakers and delegators accumulate zero `unclaimed_rewards_own` for every attacked block. Because `last_reward_block` is never reset and rewards are not retroactively recalculated for missed blocks, the yield is permanently destroyed — not deferred.

**Impact: High — Permanent freezing of unclaimed yield.**

### Likelihood Explanation

The call requires no privilege, no stake, and no special setup. The only cost is one transaction per block. A motivated attacker (e.g., a competing protocol, a griefing actor) can sustain this indefinitely. The attack is fully automatable.

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, consistent with the spec. For example, using the existing roles component:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert_only_sequencer(); // enforce "Only starkware sequencer"
    ...
}
```

Alternatively, restrict the `IStakingRewardsManager` interface so it is only callable from the attestation contract or a designated sequencer address stored in contract state.

### Proof of Concept

```
// Block N:
// Attacker (any EOA) calls:
staking.update_rewards(staker_address: any_valid_staker, disable_rewards: true);
// → last_reward_block = N, no rewards distributed, returns early.

// Legitimate sequencer then tries:
staking.update_rewards(staker_address: validator_A, disable_rewards: false);
// → PANICS: REWARDS_ALREADY_UPDATED (current_block_number == last_reward_block)

// Repeat every block → all consensus rewards permanently suppressed.
```

The `REWARDS_ALREADY_UPDATED` revert path is confirmed by the existing test suite: [7](#0-6)
