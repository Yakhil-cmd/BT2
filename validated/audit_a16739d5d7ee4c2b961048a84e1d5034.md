### Title
Pending Epoch Rewards Not Settled Before Commission Update Allows Staker to Retroactively Steal Delegator Yield - (File: src/staking/staking.cairo)

### Summary

`set_commission` updates the staker's commission immediately in storage without first settling pending epoch rewards. In the attestation-based (V2/pre-consensus) reward model, rewards for an entire epoch are calculated in a single call to `update_rewards_from_attestation_contract` at attestation time using the commission value stored at that moment. A staker holding an active commission commitment can raise their commission to `max_commission` at any point mid-epoch and then immediately attest, causing the elevated commission to be applied retroactively to the entire epoch's delegator rewards.

---

### Finding Description

`set_commission` in `src/staking/staking.cairo` writes the new commission directly to storage with no prior reward settlement step: [1](#0-0) 

The internal `update_commission` helper enforces that commission can only decrease **unless** an active `CommissionCommitment` exists, in which case any value up to `max_commission` is accepted — including values higher than the current commission: [2](#0-1) 

`max_commission` is constrained only to be `>= current_commission` at commitment creation time, so a staker can legitimately set `max_commission = 10000` (100%) while running at `commission = 100` (1%): [3](#0-2) 

When `update_rewards_from_attestation_contract` is called (once per epoch, at attestation), it reads the commission from storage at that instant and applies it to the **entire epoch's** delegated rewards: [4](#0-3) 

Inside `calculate_staker_pools_rewards`, the commission is fetched once and used for all pool reward splits: [5](#0-4) 

There is no mechanism to record when within an epoch the commission changed, so the new rate is applied to the full epoch retroactively. The developers themselves note this gap: [6](#0-5) 

---

### Impact Explanation

A staker can steal the entire epoch's delegator yield:

1. Staker operates at `commission = 100` (1%) to attract delegators.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: E+1)`.
3. Delegators join the pool expecting 1% commission.
4. At the very end of epoch E, just before attesting, staker calls `set_commission(commission: 10000)`.
5. Staker attests; `update_rewards_from_attestation_contract` calculates the full epoch's pool rewards with 100% commission.
6. Delegators receive zero rewards for the entire epoch; the staker captures all pool-side yield as commission.

This is **theft of unclaimed yield** from delegators — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

**Medium.** The attack requires:
- The protocol to be in pre-consensus (V2/attestation) mode — currently the active mode.
- The staker to hold an active `CommissionCommitment` with `max_commission > current_commission`.

Both conditions are realistic and reachable by any staker without any privileged access. The staker is the sole caller of `set_commission` and `set_commission_commitment`; no admin or bridge compromise is needed.

---

### Recommendation

Before writing the new commission to storage in `set_commission`, settle any pending epoch rewards for the staker (analogous to adding a `processInterests`-style guard). Concretely, if the staker has already attested in the current epoch, no action is needed; if they have not, the pending rewards should be calculated and credited at the old commission rate before the new rate takes effect. Alternatively, record the block/epoch at which the commission changed and apply the old rate to the portion of the epoch that elapsed before the change.

---

### Proof of Concept

```
Epoch E begins.
  Staker commission = 100 (1%).
  Delegator delegates 1,000,000 STRK.
  Epoch rewards for pool = 10,000 STRK.

  [Block near end of epoch E]
  Staker calls set_commission(commission: 10000).   // 100%
  → update_commission writes 10000 to storage immediately.
  → No pending rewards are settled.

  Staker attests.
  → update_rewards_from_attestation_contract fires.
  → calculate_staker_pools_rewards reads commission = 10000.
  → split_rewards_with_commission(10000 STRK, commission=10000)
       commission_rewards = 10000 STRK  (staker keeps all)
       pool_rewards       = 0 STRK      (delegators get nothing)

Expected: delegators receive ~9900 STRK (99% of pool rewards).
Actual:   delegators receive 0 STRK.
``` [7](#0-6) [8](#0-7)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

            ref self: ContractState, staker_address: ContractAddress,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
            self.assert_caller_is_attestation_contract();
            let mut staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            // Get current epoch data.
            let (strk_epoch_rewards, btc_epoch_rewards) = reward_supplier_dispatcher
                .calculate_current_epoch_rewards();
            let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let curr_epoch = self.get_current_epoch();
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_epoch_rewards,
                    btc_total_rewards: btc_epoch_rewards,
                    :strk_total_stake,
                    :btc_total_stake,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
