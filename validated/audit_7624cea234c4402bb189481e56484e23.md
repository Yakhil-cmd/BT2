### Title
Unprivileged Caller Can Permanently Freeze All Staker Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract carries no access control. Any unprivileged caller can invoke it with `disable_rewards: true` once per block, consuming the per-block reward slot without distributing any rewards. Repeated across every block, this permanently freezes all staker consensus-phase yield.

### Finding Description
`StakingRewardsManagerImpl::update_rewards` is exposed as a fully public entry point through the `IStakingRewardsManager` interface: [1](#0-0) 

The implementation enforces only a pause check via `general_prerequisites()` and a single-call-per-block guard, then unconditionally writes `current_block_number` to `last_reward_block` **before** inspecting the `disable_rewards` flag: [2](#0-1) 

The critical sequence is:

1. `last_reward_block` is updated to `current_block_number` at line 1485.
2. Only then is `disable_rewards` checked at line 1487; if `true`, the function returns without distributing any rewards.

Because `last_reward_block` is a **global** storage slot shared across all stakers, any caller who invokes `update_rewards(any_active_staker, true)` at block N exhausts the reward slot for that block. Every subsequent call in block N fails with `REWARDS_ALREADY_UPDATED`: [3](#0-2) 

An attacker repeating this call at the first transaction of every block prevents the consensus reward pipeline from ever executing `_update_rewards`, meaning no staker accumulates `unclaimed_rewards_own` and no pool receives its share. The rewards for each skipped block are not deferred or buffered anywhere — they are simply never computed or credited.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

All stakers operating in the consensus-rewards phase lose their per-block yield indefinitely. The `unclaimed_rewards_own` field in each staker's info is never incremented, and pool contracts never receive their reward share via `update_rewards_from_staking_contract`. Because the block-reward calculation is point-in-time (it uses the current epoch's cached block rewards), missed blocks cannot be retroactively compensated. The loss is permanent for every block the attacker covers. [4](#0-3) 

### Likelihood Explanation
The attack requires no tokens, no privileged role, and no special setup — only the ability to submit one transaction per Starknet block. The attacker needs only a valid active staker address (publicly readable from the `stakers` vector or emitted `NewStaker` events). Gas cost per call is low relative to the aggregate yield stolen from all stakers. The function is callable by any EOA or contract.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the attestation contract, a designated consensus relay address, or a `security_agent`/`app_governor` role). Alternatively, split the interface: keep a permissionless `distribute_rewards(staker_address)` that always distributes, and gate the `disable_rewards` path behind `only_app_governor()` or `only_security_agent()`.

### Proof of Concept

```
// Attacker script (pseudocode, one tx per block)
loop every block N:
    staking.update_rewards(
        staker_address = <any active staker>,
        disable_rewards = true
    )
    // Effect: last_reward_block = N, no rewards distributed
    // All legitimate update_rewards calls in block N now revert with REWARDS_ALREADY_UPDATED
```

Step-by-step:
1. Attacker calls `update_rewards(alice, true)` at block 1000.
2. `last_reward_block` is written to 1000; function returns early — Alice receives nothing.
3. Alice's node operator calls `update_rewards(alice, false)` in the same block → reverts (`REWARDS_ALREADY_UPDATED`).
4. Attacker repeats at block 1001, 1002, … indefinitely.
5. No staker ever accumulates consensus rewards; `unclaimed_rewards_own` stays at zero for all stakers across all epochs.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
