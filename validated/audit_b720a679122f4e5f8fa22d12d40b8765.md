### Title
Unprivileged Caller Can Permanently Suppress Block Rewards via Unguarded `disable_rewards` Parameter — (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` accepts a caller-controlled `disable_rewards: bool` parameter with no authorization check. Any unprivileged address can call `update_rewards(valid_staker, disable_rewards: true)`, which advances the global `last_reward_block` to the current block without distributing any rewards. Because the assertion `current_block_number > last_reward_block` prevents a second call in the same block, the rewards for that block are permanently destroyed for every staker in the protocol.

---

### Finding Description

`update_rewards` is part of the public `IStakingRewardsManager` interface and is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

`general_prerequisites` itself performs no role check: [2](#0-1) 

Inside the function, `last_reward_block` is written **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

If `disable_rewards` is `true`, the function returns immediately without computing or distributing any rewards: [4](#0-3) 

Because `last_reward_block` is a **single global slot** (not per-staker), any subsequent legitimate call to `update_rewards` in the same block fails the guard: [5](#0-4) 

The attacker's pattern mirrors the external report exactly: a parameter that controls the effective outcome of the transaction (`disable_rewards`) is not included in any authorization check, so an unprivileged caller can set it adversarially.

---

### Impact Explanation

Block rewards that are skipped are never accumulated in `unclaimed_rewards` inside `RewardSupplier` and are never credited to any staker's `unclaimed_rewards_own`. They are permanently lost — there is no recovery path. An attacker who calls `update_rewards(any_valid_staker, true)` once per block destroys 100 % of block rewards for every staker and every delegation pool in the protocol for that block.

This maps to **High: Permanent freezing of unclaimed yield** and, at sustained cadence, approaches **Critical: Protocol insolvency** (continuous yield destruction).

---

### Likelihood Explanation

The call requires no special role, no stake, and no prior state. The only cost is gas. The attacker needs one valid `staker_address` (trivially obtained from on-chain events) and one transaction per block. On Starknet, where transaction ordering within a block is controlled by the sequencer, a single well-timed transaction per block is sufficient. The attack is cheap, repeatable, and leaves no on-chain trace beyond a benign-looking `update_rewards` call.

---

### Recommendation

Add an authorization check so that only the staker themselves, their operational address, or a designated keeper role may call `update_rewards`. Alternatively, remove the `disable_rewards` parameter from the public interface entirely and handle the "skip rewards" logic through an internal, access-controlled path. At minimum, `disable_rewards: true` must never be accepted from an arbitrary caller.

---

### Proof of Concept

1. Attacker observes any active staker address `S` from on-chain `NewStaker` events.
2. At the start of block `N` (before any legitimate reward update), attacker calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. Inside the call:
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → function returns; no rewards computed or distributed.
4. Any legitimate call to `update_rewards` in block `N` now hits:
   ```
   assert!(current_block_number > self.last_reward_block.read(), ...)
   // N > N  →  false  →  REVERT
   ```
5. Block `N` rewards are permanently lost for all stakers and pools.
6. Repeating step 2 every block continuously destroys all consensus block rewards at the cost of one transaction per block. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
