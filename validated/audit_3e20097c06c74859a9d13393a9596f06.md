### Title
`MsgTurnBridge` Handler Is a No-Op — Bridge Disable Mechanism Completely Bypassed - (File: `x/cronos/keeper/msg_server.go`)

### Summary
The `TurnBridge` message handler in the Cronos module is a stub that returns `nil, nil` without performing any state change. Any admin attempt to disable the Gravity Bridge via `MsgTurnBridge` silently succeeds on-chain but has zero effect, leaving the bridge permanently enabled and allowing any unprivileged user to continue bridging CRC20/CRC21 tokens to Ethereum even after an emergency shutdown was intended.

### Finding Description

The `TurnBridge` gRPC handler in `x/cronos/keeper/msg_server.go` is implemented as a complete no-op:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

The message type, CLI command, and proto definition are all fully wired: [2](#0-1) [3](#0-2) 

The integration test `test_gravity_turn_bridge` explicitly expects that after a successful `turn_bridge("false")` call, a subsequent `send_to_evm_chain` on a CRC21 contract **fails**: [4](#0-3) 

Because the handler writes no state, the bridge-enabled flag is never updated. The `__CronosSendToEvmChain` event path (emitted by `send_to_evm_chain` in `ModuleCRC21.sol` / `ModuleCRC20Proxy.sol`) and the Gravity module's batch creation logic never see a disabled state, so they continue to process bridge-out requests normally. [5](#0-4) [6](#0-5) 

### Impact Explanation

**High — Bypass of Cronos admin/bridge authorization checks.**

The `TurnBridge` mechanism is the sole on-chain emergency stop for the Gravity Bridge. With the handler being a no-op:

- The bridge can never be disabled regardless of how many times an authorized admin submits `MsgTurnBridge{enable: false}`.
- Any unprivileged user holding CRC20/CRC21 tokens can call `send_to_evm_chain` and successfully bridge assets to Ethereum indefinitely, even during a declared emergency.
- Token burns/transfers that back the bridge-out event are irreversible once processed by the Gravity orchestrator, meaning funds leave Cronos permanently.

This directly matches the allowed High impact: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."*

### Likelihood Explanation

The entry path is fully reachable by any unprivileged user: call `send_to_evm_chain` on any mapped CRC20/CRC21 contract. No special role, leaked key, or cryptographic break is required. The only precondition is that an admin has attempted (and failed) to disable the bridge — a scenario the protocol explicitly anticipates and provides a UI/CLI for.

### Recommendation

Implement `TurnBridge` to actually persist the bridge-enabled state. The handler should:

1. Verify the caller has the required permission (analogous to how `UpdateTokenMapping` checks `HasPermission`).
2. Update a persistent parameter (e.g., `params.EnableGravityBridge` or a dedicated `BridgeEnabled` key in the KV store).
3. Return an error if the caller is unauthorized.

The `SendToEvmChain` EVM hook handler (or the Gravity module's batch-creation path) must then read this flag and reject bridge-out requests when the bridge is disabled.

### Proof of Concept

1. Deploy a CRC21-mapped token on Cronos.
2. Admin submits `MsgTurnBridge{sender: admin, enable: false}` — transaction succeeds with code 0.
3. Unprivileged user calls `crc21.send_to_evm_chain(recipient, amount, chain_id, 0, "")` — transaction succeeds and the `__CronosSendToEvmChain` log is emitted.
4. Gravity orchestrator picks up the event and creates a batch; tokens are released on Ethereum.
5. The bridge was never actually disabled because `TurnBridge` wrote no state. [1](#0-0) [5](#0-4)

// CmdTurnBridge returns a CLI command handler for enable or disable the bridge
func CmdTurnBridge() *cobra.Command {
	cmd := &cobra.Command{
		Use:   "turn-bridge [true/false]",
		Short: "Turn Bridge",
		Args:  cobra.ExactArgs(1),
		RunE: func(cmd *cobra.Command, args []string) error {
			clientCtx, err := client.GetClientTxContext(cmd)
			if err != nil {
				return err
			}

			enable, err := strconv.ParseBool(args[0])
			if err != nil {
				return err
			}
			msg := types.NewMsgTurnBridge(clientCtx.GetFromAddress().String(), enable)
			if err := msg.ValidateBasic(); err != nil {
				return err
			}
			return tx.GenerateOrBroadcastTxCLI(clientCtx, cmd.Flags(), msg)
		},
	}

	flags.AddTxFlagsToCmd(cmd)
	return cmd
}
