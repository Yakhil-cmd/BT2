### Title
Any Contract Can Overwrite Another Account's `AccountId`-Mode Global Contract — (`File: runtime/runtime/src/global_contracts.rs`)

### Summary

`action_deploy_global_contract` contains no authorization check binding the deployer to the target account when `GlobalContractDeployMode::AccountId` is used. Any unprivileged user can deploy a malicious contract that issues a cross-contract call carrying `DeployGlobalContractAction(mode=AccountId)` to a victim account, overwriting the victim's global contract entry in the trie. Every account on every shard that has called `UseGlobalContract(AccountId(victim))` will subsequently execute attacker-controlled Wasm.

---

### Finding Description

`GlobalContractDeployMode::AccountId` stores the uploaded Wasm under the key `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(account_id) }`, where `account_id` is the **receipt's `receiver_id`**, not the original transaction signer. [1](#0-0) 

The function `action_deploy_global_contract` receives only `account_id` (the receipt receiver), `account` (the receiver's state), `apply_state`, and the action payload. It never inspects `receipt.predecessor_id()` and has no parameter for it: [2](#0-1) 

The call site in `lib.rs` passes `account_id` (= `receiver_id`) but does not forward `receipt.predecessor_id()`: [3](#0-2) 

In NEAR, any contract can create a promise to an arbitrary account and append any action, including `DeployGlobalContract`. The resulting action receipt is executed unconditionally on the target account. The only runtime guards are:

1. **Balance check** — storage cost is deducted from the *victim's* account (not the attacker's).
2. **Size check** — code must be ≤ `max_contract_size`.
3. **Nonce freshness** — prevents stale re-delivery of the same distribution receipt, but a fresh deployment with a higher nonce always succeeds. [4](#0-3) 

None of these guards verify that the entity initiating the deployment is the account owner.

The `validate_deploy_global_contract_action` validator also performs only a size check: [5](#0-4) 

---

### Impact Explanation

**Scope: global contract/code selection.**

The exact corrupted value is the trie entry `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim) }`. After the attack:

- Every account that previously called `UseGlobalContract(AccountId(victim))` has `account.contract = AccountContract::GlobalByAccount(victim)`.
- On the next function call to any such account, the runtime resolves the contract via `GlobalContractCodeIdentifier::AccountId(victim)` and loads the attacker-supplied Wasm.
- The attacker controls execution for all those accounts — arbitrary state reads/writes, token transfers, key additions/deletions — within the gas limit of each call.

The `AccountId` deploy mode is explicitly designed for mutable, owner-updatable global contracts. Allowing any third party to overwrite it breaks the entire trust model of that mode.

---

### Likelihood Explanation

The attack requires only:
1. A deployed contract (standard user operation, costs gas + storage).
2. The victim account to hold enough balance to cover `global_contract_storage_amount_per_byte × code_len` (the storage cost is charged to the victim, not the attacker).
3. One function call that issues `promise_batch_action_deploy_global_contract` targeting the victim.

No validator privilege, no special network access, and no admin capability is required. Any NEAR account with a deployed contract can execute this.

---

### Recommendation

Add an authorization guard inside `action_deploy_global_contract` for `AccountId` mode. Before calling `initiate_distribution`, verify that the receipt's `predecessor_id` equals the `receiver_id` (i.e., the action was self-initiated):

```rust
if deploy_contract.deploy_mode == GlobalContractDeployMode::AccountId {
    // Only the account itself may register a global contract under its own name.
    if receipt.predecessor_id() != account_id {
        result.result = Err(ActionErrorKind::ActorNoPermission {
            account_id: account_id.clone(),
            actor_id: receipt.predecessor_id().clone(),
        }.into());
        return Ok(());
    }
}
```

The `receipt` (or at minimum `predecessor_id`) must be threaded into `action_deploy_global_contract`. Alternatively, enforce the restriction at the action-validation layer by rejecting `DeployGlobalContract(mode=AccountId)` in any receipt whose `predecessor_id ≠ receiver_id`.

---

### Proof of Concept

```rust
// Attacker's contract (deployed on attacker.near):

impl AttackerContract {
    pub fn overwrite_victim_global_contract(&self) {
        // Create a promise targeting victim.near
        let promise = env::promise_batch_create("victim.near");
        // Append DeployGlobalContract(AccountId) with malicious Wasm
        env::promise_batch_action_deploy_global_contract(
            promise,
            &MALICIOUS_WASM_BYTES,
            GlobalContractDeployMode::AccountId,
        );
    }
}
```

After `overwrite_victim_global_contract` executes:

1. An action receipt is created: `receiver_id = victim.near`, action = `DeployGlobalContract(mode=AccountId, code=MALICIOUS_WASM)`.
2. `action_deploy_global_contract` runs with `account_id = victim.near`, deducts storage cost from `victim.near`'s balance, and calls `initiate_distribution(account_id = victim.near, ...)`.
3. `GlobalContractDistribution` receipts propagate to all shards, writing `MALICIOUS_WASM` under `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }`.
4. All accounts that previously called `UseGlobalContract(AccountId("victim.near"))` now execute `MALICIOUS_WASM` on their next function call. [6](#0-5) [7](#0-6)

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

pub(crate) fn use_global_contract(
    state_update: &mut TrieUpdate,
    account_id: &AccountId,
    account: &mut Account,
    contract_identifier: &GlobalContractIdentifier,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let key = TrieKey::GlobalContractCode { identifier: contract_identifier.clone().into() };
    if !state_update.contains_key(&key, AccessOptions::DEFAULT)? {
        result.result = Err(ActionErrorKind::GlobalContractDoesNotExist {
            identifier: contract_identifier.clone(),
        }
        .into());
        return Ok(());
    }
    clear_account_contract_storage_usage(state_update, account_id, account)?;
    if account.contract().is_local() {
        state_update.remove(TrieKey::ContractCode { account_id: account_id.clone() });
    }
    let contract = match contract_identifier {
        GlobalContractIdentifier::CodeHash(code_hash) => AccountContract::Global(*code_hash),
        GlobalContractIdentifier::AccountId(id) => AccountContract::GlobalByAccount(id.clone()),
    };
    account.set_storage_usage(
        account.storage_usage().checked_add(contract_identifier.len() as u64).ok_or_else(|| {
            StorageError::StorageInconsistentState(format!(
                "Storage usage integer overflow for account {}",
                account_id
            ))
        })?,
    );
    account.set_contract(contract);
    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())

// Checks if the incoming nonce is fresh and updates the stored nonce. Returns
// true if the nonce is fresh, false if it's stale. The nonce is set
// immediately and the freshness check allows the same nonce (>=).
fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
}

/// Validates `DeployGlobalContractAction`. Checks that the given contract size doesn't exceed the limit.
fn validate_deploy_global_contract_action(
    limit_config: &LimitConfig,
    action: &DeployGlobalContractAction,
) -> Result<(), ActionsValidationError> {
    if action.code.len() as u64 > limit_config.max_contract_size {
        return Err(ActionsValidationError::ContractSizeExceeded {
            size: action.code.len() as u64,
            limit: limit_config.max_contract_size,
        });
    }

    Ok(())
}
