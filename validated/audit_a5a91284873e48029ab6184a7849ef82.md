### Title
Unprivileged Caller Can Permanently Skip Consensus Block Rewards for All Stakers via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards` boolean. Because `last_reward_block` is a global, single-value storage slot that is unconditionally written on every call, any unprivileged caller can invoke `update_rewards(any_active_staker, disable_rewards: true)` once per block to permanently skip that block's consensus rewards for every staker in the protocol.

### Finding Description

`update_rewards` is exposed as a public interface with no role guard beyond `general_prerequisites` (which only checks the contract is unpaused and the caller is non-zero):

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // no role check
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        self.last_reward_block.write(current_block_number);   // always written

        if disable_rewards || self.is_pre_consensus() {
            return;                                           // rewards skipped
        }
        // reward distribution only reached when disable_rewards == false
``` [1](#0-0) 

`last_reward_block` is a single global storage slot, not a per-staker map: [2](#0-1) 

The one-call-per-block invariant is enforced by the `REWARDS_ALREADY_UPDATED` assertion. Once `last_reward_block` is written for block `N`, no subsequent call in block `N` can pass the assertion, regardless of `disable_rewards`. The rewards for that block are irrevocably skipped.

`general_prerequisites` imposes no role restriction: [3](#0-2) 

### Impact Explanation

When an attacker calls `update_rewards(any_active_staker, disable_rewards: true)` at the first transaction of every block, the global `last_reward_block` is advanced to the current block and the function returns before distributing any STRK or BTC block rewards. Because the slot is already written, no legitimate caller can recover those rewards in the same block. Repeated across every block, this permanently destroys all consensus-phase yield for every staker and delegator in the protocol — a permanent freezing of unclaimed yield.

### Likelihood Explanation

The entry path requires no privilege, no leaked key, and no external dependency. Any EOA or contract can call `update_rewards` with `disable_rewards: true`. The only cost is one transaction per block. The attack is trivially automatable (a simple bot calling the function at the start of each block). Because the function is public and the parameter is caller-controlled, the likelihood is high.

### Recommendation

1. **Remove `disable_rewards` as a caller-controlled parameter.** The flag should be derived internally from protocol state (e.g., whether the staker is in an unstake-intent window), not supplied by an arbitrary caller.
2. **Restrict `update_rewards` to a privileged role** (e.g., the attestation contract or a designated sequencer address) so that only trusted callers can advance `last_reward_block`.
3. Alternatively, **separate the `last_reward_block` write from the reward-distribution logic** so that skipping distribution does not consume the block's reward slot.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker monitors the mempool; at the start of block `N`, attacker submits:
   ```
   staking_contract.update_rewards(any_active_staker, disable_rewards: true)
   ```
3. Inside `update_rewards`:
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → early return, no rewards distributed.
4. Any legitimate call to `update_rewards` in block `N` now reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers and delegators receive zero block rewards for block `N`.
6. Attacker repeats every block, permanently zeroing all consensus-phase yield. [4](#0-3) [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
