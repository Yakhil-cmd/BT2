### Title
Unprivileged Caller Can Permanently Block Consensus Reward Distribution via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `src/staking/staking.cairo` is callable by any non-zero address with no access control. It unconditionally writes `last_reward_block = current_block_number` **before** checking the `disable_rewards` flag. An attacker can call `update_rewards(valid_staker, disable_rewards: true)` in every block to consume the per-block reward slot without distributing any rewards, causing all legitimate `update_rewards` calls in the same block to revert with `REWARDS_ALREADY_UPDATED`. Sustained over time, this permanently freezes unclaimed yield for all stakers and pool members.

---

### Finding Description

`update_rewards` enforces a global, per-block guard using `last_reward_block`: [1](#0-0) 

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After passing this guard, the function writes the current block number to `last_reward_block` **unconditionally**, before the `disable_rewards` branch: [2](#0-1) 

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

The only access control on the function is `general_prerequisites()`, which only checks the pause flag and that the caller is non-zero: [3](#0-2) 

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

There is no check that `get_caller_address() == staker_address` or any role requirement. Any address can call `update_rewards(any_valid_staker, disable_rewards: true)`.

The result is a race condition analogous to the Derby Finance bug: just as TrueFi/Idle vaults block a second operation in the same block after a first conflicting one, `last_reward_block` blocks any second `update_rewards` call in the same block after the attacker's first call — regardless of whether rewards were actually distributed.

---

### Impact Explanation

If an attacker calls `update_rewards(valid_staker, disable_rewards: true)` in every block:

- `last_reward_block` is set to the current block with no rewards distributed.
- Every legitimate `update_rewards` call in that block reverts with `REWARDS_ALREADY_UPDATED`.
- Consensus block rewards (`strk_block_rewards`, `btc_block_rewards`) are never credited to any staker's `unclaimed_rewards_own` or forwarded to pool contracts.
- All stakers' and pool members' unclaimed yield is permanently frozen.

This matches the **High** impact category: *Permanent freezing of unclaimed yield*.

---

### Likelihood Explanation

- The attacker only needs to know one valid, active staker address with non-zero balance — trivially discoverable from on-chain events (`NewStaker`).
- The cost is one Starknet transaction per block. Starknet transaction fees are low.
- No privileged access, no leaked keys, no external dependency compromise is required.
- The attack is fully permissionless and reachable by any unprivileged public caller.

---

### Recommendation

1. **Move `last_reward_block.write` after the `disable_rewards` check**, so that a no-op call does not consume the per-block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // moved here
// ... distribute rewards
```

2. **Or add caller authorization**: require `get_caller_address() == staker_address` or a designated role, so only the staker (or an authorized distributor) can call `update_rewards` for a given staker.

3. **Or remove `disable_rewards` from the public interface** if it has no legitimate external use case.

---

### Proof of Concept

```
Block N:
  Attacker calls: update_rewards(valid_staker_S, disable_rewards=true)
    → assert(N > last_reward_block)  ✓ (passes)
    → last_reward_block = N          ← slot consumed
    → disable_rewards=true → return  ← no rewards distributed

  Legitimate staker calls: update_rewards(staker_S, disable_rewards=false)
    → assert(N > last_reward_block)  ✗ REVERT: REWARDS_ALREADY_UPDATED

Block N+1: attacker repeats → same outcome
...
All consensus rewards permanently frozen.
```

The root cause is identical in structure to the Derby Finance M-5 bug: a shared state variable (`last_reward_block`, analogous to `latestJoinBlock` in TrueFi) is written by an operation that does not complete its intended effect (`disable_rewards=true`), and this write blocks the legitimate operation that follows in the same block. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
