### Title
Unprotected `update_rewards` with Caller-Controlled `disable_rewards` Parameter Allows Permanent Griefing of All Stakers' Consensus Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public function with no access control beyond a non-zero caller check. It accepts a caller-controlled `disable_rewards: bool` parameter and unconditionally writes the current block number to the **global** `last_reward_block` storage variable before checking `disable_rewards`. Any unprivileged caller can invoke this with `disable_rewards: true` every block, consuming the single per-block reward slot and permanently preventing consensus block rewards from being distributed to any staker.

---

### Finding Description

`update_rewards` is defined in `IStakingRewardsManager` and implemented in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.  <-- written BEFORE the disable_rewards check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits with no rewards distributed
    }
    ...
}
``` [1](#0-0) 

`general_prerequisites` imposes no role check — it only asserts the contract is unpaused and the caller is non-zero: [2](#0-1) 

`last_reward_block` is a **single global** storage slot (not per-staker): [3](#0-2) 

The enforcement gate is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

Because `last_reward_block` is global and is written before the `disable_rewards` branch, a single call per block with `disable_rewards: true` exhausts the slot for that block. Any subsequent legitimate call (e.g., from a block proposer) reverts with `REWARDS_ALREADY_UPDATED`.

The interface explicitly exposes this to any caller: [5](#0-4) 

---

### Impact Explanation

An attacker who submits `update_rewards(any_valid_active_staker, disable_rewards: true)` as the first transaction of every block:

1. Writes `last_reward_block = current_block_number` with no rewards distributed.
2. Causes every subsequent legitimate `update_rewards` call in that block to revert.
3. Stakers accumulate zero consensus block rewards for as long as the attack is sustained.

This constitutes **permanent freezing of unclaimed yield** (consensus block rewards) for all stakers in the protocol, matching the High-severity allowed impact.

---

### Likelihood Explanation

On Starknet, the sequencer orders transactions within a block. An attacker can submit a low-cost transaction at the start of every block. The cost is one transaction per block; the damage is the complete suppression of consensus rewards for the entire protocol. There is no economic barrier preventing a motivated attacker (e.g., a competing validator, a short-seller of STRK, or a pure griefer) from sustaining this indefinitely.

---

### Recommendation

1. **Restrict the caller**: Add an access-control check so only the designated consensus/block-proposer role (or the attestation contract) can call `update_rewards`. This mirrors how `update_rewards_from_attestation_contract` is already gated: [6](#0-5) 

2. **Remove the `disable_rewards` parameter from the public interface**: If the intent is to allow a block proposer to signal "no rewards this block," that signal should come from a trusted source, not an arbitrary caller.

3. **Make `last_reward_block` per-staker** if the function must remain public, so one call cannot block all other stakers.

---

### Proof of Concept

```
Block N arrives.

Attacker (any EOA) calls:
  staking.update_rewards(staker=<any_active_staker>, disable_rewards=true)

Execution path:
  1. general_prerequisites() → passes (contract not paused, caller != 0)
  2. assert(block_N > last_reward_block) → passes (first call this block)
  3. last_reward_block.write(block_N)   ← slot consumed
  4. disable_rewards == true → return   ← no rewards distributed

Block proposer (legitimate) calls:
  staking.update_rewards(staker=<proposer_staker>, disable_rewards=false)

Execution path:
  1. general_prerequisites() → passes
  2. assert(block_N > last_reward_block) → FAILS (last_reward_block == block_N)
  → reverts with REWARDS_ALREADY_UPDATED

Repeat every block → all stakers receive zero consensus rewards indefinitely.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
