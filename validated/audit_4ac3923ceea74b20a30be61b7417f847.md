### Title
Ejected Consensus Nodes Retain Full Epoch Voting Power via Stale Initial-Committee Snapshot — (`consensus/hotstuff/committees/consensus_committee.go`)

### Summary
`Consensus.IdentityByEpoch` resolves voter identity from a static snapshot of the committee captured at epoch-setup time. It does not consult the current ejection status of a node. Consequently, `Validator.ValidateVote` accepts cryptographically valid votes from consensus nodes that have been ejected mid-epoch, and those votes count toward the quorum weight needed to form a QC — exactly mirroring the DAO bug where an expelled validator retains influence over active proposals through a back-dated snapshot.

### Finding Description

**Root cause — static initial-committee map ignores ejection**

`newEpochInfo` builds the committee cache used for all vote validation:

```go
initialCommittee := initialIdentities.Filter(filter.IsConsensusCommitteeMember)
```

`IsConsensusCommitteeMember` is defined over `flow.IdentitySkeleton`, which carries no `EpochParticipationStatus` field:

```go
var IsConsensusCommitteeMember = And(
    HasRole[flow.IdentitySkeleton](flow.RoleConsensus),
    HasInitialWeight[flow.IdentitySkeleton](true),
)
```

Any consensus node with positive `InitialWeight` that was present in the `EpochSetup` event is unconditionally placed into `initialCommitteeMap`. A node ejected *after* the `EpochSetup` event is still in that map for the remainder of the epoch.

`IdentityByEpoch` performs a plain map lookup with no ejection guard:

```go
func (c *Consensus) IdentityByEpoch(view uint64, participantID flow.Identifier) (*flow.IdentitySkeleton, error) {
    epochInf, err := c.epochInfoByView(view)
    ...
    identity, ok := epochInf.initialCommitteeMap[participantID]
    if !ok {
        return nil, model.NewInvalidSignerErrorf("id %v is not a valid node id", participantID)
    }
    return identity, nil   // ejected node returned without error
}
```

**Vote validation path accepts ejected-node votes**

`Validator.ValidateVote` calls `IdentityByEpoch` as its sole membership check:

```go
func (v *Validator) ValidateVote(vote *model.Vote) (*flow.IdentitySkeleton, error) {
    voter, err := v.committee.IdentityByEpoch(vote.View, vote.SignerID)
    if model.IsInvalidSignerError(err) {
        return nil, newInvalidVoteError(vote, err)
    }
    ...
    err = v.verifier.VerifyVote(voter, vote.SigData, vote.View, vote.BlockID)
    ...
    return voter, nil
}
```

Because `IdentityByEpoch` returns the ejected node's skeleton without error, the vote proceeds to cryptographic verification and, if the signature is valid, is accepted and counted toward the quorum weight.

**Asymmetry with proposal validation**

Proposal validation uses `IdentityByBlock`, which queries the live protocol state and applies `filter.IsVotingConsensusCommitteeMember` — a filter that *does* require `EpochParticipationStatusActive`:

```go
var IsVotingConsensusCommitteeMember = And[flow.Identity](
    IsValidCurrentEpochParticipant,    // EpochParticipationStatusActive
    Adapt(IsConsensusCommitteeMember),
)
```

So ejected nodes are immediately barred from having their *proposals* certified, but their *votes* remain valid for the entire epoch. The interface comment in `committee.go` explicitly documents this asymmetry:

> "Any consensus committee member who was authorized to contribute to consensus AT THE BEGINNING of the epoch may produce valid votes and timeouts for the entire epoch, even if they are later ejected."

The quorum weight threshold (`weightThresholdForQC`) is also frozen at epoch-start and never adjusted for ejections, so the ejected node's weight continues to count toward the supermajority denominator.

**Attacker-controlled entry path**

A malicious node operator whose node has been ejected retains their private staking key. They can craft a `model.Vote` directly (bypassing the local `SafetyRules`, which would block vote production via `IdentityByBlock`), sign it with the ejected node's staking key, and relay it to the network. Honest peers validate the vote with `ValidateVote → IdentityByEpoch`, which returns the ejected node's identity without error, and the vote is accepted.

### Impact Explanation

An ejected consensus node's historical voting weight remains fully effective for QC and TC formation for the rest of the epoch. A malicious ejected node (or an attacker who obtained the ejected node's staking key — the typical reason for ejection) can:

- **Contribute to a malicious QC**: if the ejected node's weight, combined with other Byzantine nodes, crosses the >2/3 supermajority threshold, a safety-violating QC can be formed for a fork.
- **Block honest QC formation**: if the Byzantine coalition (including the ejected node) holds >1/3 of the epoch weight, they can prevent honest nodes from reaching quorum, causing a liveness failure.

The ejected node's weight is not subtracted from the threshold denominator, so even a modest ejected-node weight meaningfully shifts the effective Byzantine tolerance of the epoch.

### Likelihood Explanation

Medium. Nodes are ejected for protocol violations (e.g., equivocation), meaning the ejected operator is already demonstrated to be adversarial. The operator retains their private key after ejection. Honest nodes disconnect from ejected nodes at the networking layer, but the attacker can relay votes through other paths (e.g., via an access node or a colluding peer) before the ejection propagates. The window is bounded by the epoch length (~500 k views), which is substantial.

### Recommendation

1. Extend `IdentityByEpoch` (or `ValidateVote`) to cross-check the current ejection status via the live protocol state, analogous to how `IdentityByBlock` applies `IsVotingConsensusCommitteeMember`.
2. Alternatively, adjust the quorum weight threshold dynamically to exclude ejected nodes' weight, so their votes cannot contribute to reaching the supermajority.
3. At minimum, document the residual risk explicitly and ensure the ejection-propagation path at the networking layer is fast enough to close the window before the ejected node can accumulate meaningful vote weight.

### Proof of Concept

1. Epoch E begins; consensus committee includes node Alice with 10 % of total weight. `initialCommitteeMap` is populated with Alice's `IdentitySkeleton`.
2. Mid-epoch, Alice is ejected via an `EjectNode` service event for equivocation. `IdentityByBlock` now returns `InvalidSignerError` for Alice, so her proposals are rejected.
3. Alice's operator retains the private staking key. They craft a `model.Vote` for a malicious block B at view V (within epoch E), sign it with Alice's staking key, and relay it to honest peers.
4. Honest peers call `Validator.ValidateVote`:
   - `v.committee.IdentityByEpoch(V, Alice.NodeID)` → returns Alice's `IdentitySkeleton` (no error, because `initialCommitteeMap` is not updated on ejection).
   - `v.verifier.VerifyVote(alice, vote.SigData, V, B)` → signature is valid.
   - Vote is accepted.
5. Alice's 10 % weight is added to the staking aggregator for block B. If a Byzantine coalition already holds 57 % of epoch weight, Alice's contribution pushes them past the >2/3 threshold, enabling a malicious QC for block B. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

```go
// newEpochInfo retrieves the committee information and computes leader selection.
// This can be cached and used for all by-view queries for this epoch.
// No errors are expected during normal operation.
func newEpochInfo(epoch protocol.CommittedEpoch) (*epochInfo, error) {
	randomSeed := epoch.RandomSource()
	firstView := epoch.FirstView()
	finalView := epoch.FinalView()
	initialIdentities := epoch.InitialIdentities()
	leaders, err := leader.SelectionForConsensus(initialIdentities, randomSeed, firstView, finalView)
	if err != nil {
		return nil, fmt.Errorf("could not get leader selection: %w", err)
	}

	initialCommittee := initialIdentities.Filter(filter.IsConsensusCommitteeMember)
	dkg, err := epoch.DKG()
	if err != nil {
		return nil, fmt.Errorf("could not get dkg: %w", err)
	}

	totalWeight := initialCommittee.TotalWeight()
	ei := &epochInfo{
		LeaderSelection:      leaders,
		randomSeed:           randomSeed,
		initialCommittee:     initialCommittee,
		initialCommitteeMap:  initialCommittee.Lookup(),
		weightThresholdForQC: WeightThresholdToBuildQC(totalWeight),
		weightThresholdForTO: WeightThresholdToTimeout(totalWeight),
		dkg:                  dkg,
	}
	return ei, nil
```

```go
// IdentitiesByBlock returns the identities of all authorized consensus participants at the given block.
// The order of the identities is the canonical order.
// ERROR conditions:
//   - state.ErrUnknownSnapshotReference if the blockID is for an unknown block
func (c *Consensus) IdentitiesByBlock(blockID flow.Identifier) (flow.IdentityList, error) {
	il, err := c.state.AtBlockID(blockID).Identities(filter.IsVotingConsensusCommitteeMember)
	if err != nil {
		return nil, fmt.Errorf("could not identities at block %x: %w", blockID, err) // state.ErrUnknownSnapshotReference or exception
	}
	return il, nil
}

// IdentityByBlock returns the identity of the node with the given node ID at the given block.
// ERROR conditions:
//   - model.InvalidSignerError if participantID does NOT correspond to an authorized HotStuff participant at the specified block.
//   - state.ErrUnknownSnapshotReference if the blockID is for an unknown block
func (c *Consensus) IdentityByBlock(blockID flow.Identifier, nodeID flow.Identifier) (*flow.Identity, error) {
	identity, err := c.state.AtBlockID(blockID).Identity(nodeID)
	if err != nil {
		if protocol.IsIdentityNotFound(err) {
			return nil, model.NewInvalidSignerErrorf("id %v is not a valid node id: %w", nodeID, err)
		}
		return nil, fmt.Errorf("could not get identity for node ID %x: %w", nodeID, err) // state.ErrUnknownSnapshotReference or exception
	}
	if !filter.IsVotingConsensusCommitteeMember(identity) {
		return nil, model.NewInvalidSignerErrorf("node %v is not an authorized hotstuff voting participant", nodeID)
	}
	return identity, nil
}
```

```go
// IsConsensusCommitteeMember is an identity filter for all members of the consensus committee.
// Formally, a Node X is a Consensus Committee Member if and only if X is a consensus node with
// positive initial weight. This is specified by the EpochSetup Event and remains static
// throughout the epoch.
var IsConsensusCommitteeMember = And(
	HasRole[flow.IdentitySkeleton](flow.RoleConsensus),
	HasInitialWeight[flow.IdentitySkeleton](true),
)

// IsVotingConsensusCommitteeMember is an identity filter for all members of
// the consensus committee allowed to vote.
// Formally, a Node X has authority to vote in the consensus process, if and only if
//  1. Node X is an active member of the current epoch AND
//  2. X is a consensus node with positive initial weight in the current Epoch. This
//     is specified by the EpochSetup Event for the current epoch and remains static
//     throughout the epoch.
var IsVotingConsensusCommitteeMember = And[flow.Identity](
	IsValidCurrentEpochParticipant,    // enforces 1.
	Adapt(IsConsensusCommitteeMember), // enforces 2.
)
```

```go
func (v *Validator) ValidateVote(vote *model.Vote) (*flow.IdentitySkeleton, error) {
	voter, err := v.committee.IdentityByEpoch(vote.View, vote.SignerID)
	if model.IsInvalidSignerError(err) {
		return nil, newInvalidVoteError(vote, err)
	}
	if err != nil {
		return nil, fmt.Errorf("error retrieving voter Identity at view %d: %w", vote.View, err)
	}

	// check whether the signature data is valid for the vote in the hotstuff context
	err = v.verifier.VerifyVote(voter, vote.SigData, vote.View, vote.BlockID)
	if err != nil {
		// Theoretically, `VerifyVote` could also return a `model.InvalidSignerError`. However,
		// for the time being, we assume that _every_ HotStuff participant is also a member of
		// the random beacon committee. Consequently, `InvalidSignerError` should not occur atm.
		// TODO: if the random beacon committee is a strict subset of the HotStuff committee,
		//       we expect `model.InvalidSignerError` here during normal operations.
		if model.IsInvalidFormatError(err) || errors.Is(err, model.ErrInvalidSignature) {
			return nil, newInvalidVoteError(vote, err)
		}
		if errors.Is(err, model.ErrViewForUnknownEpoch) {
			return nil, fmt.Errorf("no Epoch information available for vote; symptom of internal bug or invalid bootstrapping information: %s", err.Error())
		}
		return nil, fmt.Errorf("cannot verify signature for vote (%x): %w", vote.ID(), err)
	}

	return voter, nil
}
```
