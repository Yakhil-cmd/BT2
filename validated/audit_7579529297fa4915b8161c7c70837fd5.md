### Title
Reward Supplier Insufficient Token Balance Causes `unstake_action` to Revert, Temporarily Freezing Staker Funds - (File: src/staking/staking.cairo)

### Summary
`unstake_action` in `staking.cairo` unconditionally calls `send_rewards_to_staker`, which calls `claim_from_reward_supplier`, which calls `reward_supplier.claim_rewards`. If the reward supplier's actual on-chain token balance is less than its `unclaimed_rewards` accounting value — a window that exists whenever L1-minted funds have not yet arrived — the inner `checked_transfer` reverts, propagating the failure all the way up and blocking the staker from exiting.

### Finding Description
`unstake_action` (staking.cairo lines 483–515) must distribute accumulated rewards before erasing the staker record. It does so by calling `send_rewards_to_staker` (lines 1614–1629):

```cairo
fn send_rewards_to_staker(...) {
    let amount = staker_info.unclaimed_rewards_own;
    let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
    claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
    staker_info.unclaimed_rewards_own = Zero::zero();
}
```

`claim_from_reward_supplier` (utils.cairo lines 50–60) calls `reward_supplier_dispatcher.claim_rewards(amount)` and then asserts the staking contract's balance increased by exactly `amount`:

```cairo
pub(crate) fn claim_from_reward_supplier(...) {
    let balance_before = token_dispatcher.balance_of(account: staking_contract);
    reward_supplier_dispatcher.claim_rewards(:amount);
    let balance_after = token_dispatcher.balance_of(account: staking_contract);
    assert!(balance_after - balance_before == amount.into(), "{}", Error::UNEXPECTED_BALANCE);
}
```

Inside the reward supplier, `claim_rewards` (reward_supplier.cairo lines 205–220) first checks the accounting variable:

```cairo
let unclaimed_rewards = self.unclaimed_rewards.read();
assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);
self.unclaimed_rewards.write(unclaimed_rewards - amount);
token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
```

The accounting check (`amount <= unclaimed_rewards`) can pass while the actual ERC-20 balance of the reward supplier is lower than `amount`. This happens because `update_unclaimed_rewards_from_staking_contract` (lines 189–202) increases `unclaimed_rewards` immediately when rewards are calculated, then calls `request_funds` to send a message to L1 requesting a mint. The L1 tokens only arrive later via `on_receive`. During the entire interval between the reward update and the L1 bridge delivery, `unclaimed_rewards > actual_token_balance`. In that window, `checked_transfer` reverts with `INSUFFICIENT_BALANCE`, which propagates through `claim_from_reward_supplier` → `send_rewards_to_staker` → `unstake_action`, causing the staker's exit transaction to revert.

There is no exception handling or fallback path: `unstake_action` does not skip reward distribution when the reward supplier is temporarily underfunded, nor does it defer the reward and proceed with returning the principal.

### Impact Explanation
A staker who has completed the exit-wait window and calls `unstake_action` will have the transaction revert for as long as the reward supplier's actual token balance is below the staker's `unclaimed_rewards_own`. The staker's principal remains locked in the staking contract. This constitutes **temporary freezing of funds** (staked principal cannot be recovered), which is within the allowed impact scope.

### Likelihood Explanation
The reward supplier is designed to hold a running buffer and proactively request L1 mints, but the gap between accounting and actual balance is structural and always present after any reward update. During periods of high attestation activity, L1 bridge congestion, or simply the normal latency of L1→L2 message delivery, the reward supplier's balance can lag behind `unclaimed_rewards` for multiple blocks or even epochs. Any staker whose `unclaimed_rewards_own` exceeds the reward supplier's current balance will be unable to exit during that window.

### Recommendation
Decouple reward distribution from the exit flow. Two options:

1. **Skip rewards on insufficient balance**: In `send_rewards_to_staker`, check whether the reward supplier has sufficient balance before calling `claim_from_reward_supplier`. If not, leave `unclaimed_rewards_own` intact and proceed with returning the principal. The staker can claim rewards separately once the reward supplier is funded.

2. **Separate reward claim from unstake**: Do not call `send_rewards_to_staker` inside `unstake_action`. Instead, preserve `unclaimed_rewards_own` in storage and allow the staker (or anyone) to call `claim_rewards` independently after the staker record is removed.

### Proof of Concept
1. Staker stakes and accumulates rewards over several epochs via attestation.
2. `update_rewards` is called; `reward_supplier.unclaimed_rewards` is increased by the staker's share, and a `request_funds` L1 message is sent.
3. The L1 mint has not yet been delivered via `on_receive`; the reward supplier's actual STRK balance is less than `unclaimed_rewards`.
4. The staker's exit window has elapsed; they call `unstake_action(staker_address)`.
5. `unstake_action` → `send_rewards_to_staker` → `claim_from_reward_supplier` → `reward_supplier.claim_rewards` → `checked_transfer` reverts with `INSUFFICIENT_BALANCE` (or `claim_from_reward_supplier` reverts with `UNEXPECTED_BALANCE`).
6. The entire `unstake_action` transaction reverts. The staker's principal remains locked in the staking contract until the L1 funds arrive and the transaction is retried. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn claim_rewards(ref self: ContractState, amount: Amount) {
            // Asserts.
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            let unclaimed_rewards = self.unclaimed_rewards.read();
            assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Update unclaimed_rewards and transfer the requested rewards to the staking contract.
            self.unclaimed_rewards.write(unclaimed_rewards - amount);
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
        }
