### Title
`CanTurnBridge` Permission Check Completely Bypassed — Bridge Circuit Breaker Non-Functional (`x/cronos/keeper/msg_server.go`)

---

### Summary

The `MsgTurnBridge` message handler is an unimplemented stub that returns `nil, nil` with no permission check and no state mutation. The `CanTurnBridge` permission bit is defined and documented as the guard for the bridge circuit breaker, but it is never enforced. Any unprivileged user can submit `MsgTurnBridge` and receive a success response, while the bridge state is never actually changed — permanently disabling the emergency halt mechanism.

---

### Finding Description

The `CanTurnBridge` permission is defined in the permissions bitmask: [1](#0-0) 

The ADR-009 permissioning system and the wiki document that `CanTurnBridge` grants the ability to trigger the bridge circuit breaker, and that `HasPermission` enforces this: [2](#0-1) 

However, the actual `TurnBridge` message handler in `msg_server.go` is a complete stub: [3](#0-2) 

It returns `nil, nil` — no permission check against `CanTurnBridge`, no bridge state write, no error. Compare this to `UpdateTokenMapping`, which correctly calls `HasPermission`: [4](#0-3) 

And `UpdatePermissions`, which correctly checks `CronosAdmin`: [5](#0-4) 

The consequence is twofold:

1. **Permission bypass**: Any unprivileged address can submit `MsgTurnBridge` and receive a success response, completely bypassing the `CanTurnBridge` guard.
2. **Circuit breaker is dead**: Because no bridge-enabled state is ever written, `ConvertVouchersToEvmCoins` and `IbcTransferCoins` have no state to read — the bridge can never be halted regardless of who calls `TurnBridge`. [6](#0-5) 

---

### Impact Explanation

**High — Bypass of Cronos admin/permission authorization check.**

The `CanTurnBridge` permission is the documented and intended guard for the bridge circuit breaker. Because the handler is a stub, this guard is never evaluated. Any unprivileged user can call `MsgTurnBridge` with a success result, which constitutes a direct bypass of the permission system defined in ADR-009.

**High — Permanent inability to halt bridge/conversion flows under emergency conditions.**

The bridge circuit breaker is the only documented mechanism to stop `ConvertVouchersToEvmCoins` and `IbcTransferCoins` in an emergency (e.g., a bridge exploit draining CRO or IBC vouchers). Because the handler never writes bridge state, this mechanism is permanently non-functional. Honest validators and the admin cannot halt asset flows even if a critical exploit is actively occurring.

---

### Likelihood Explanation

The entry path is fully open: `MsgTurnBridge` is a standard Cosmos SDK message reachable by any unprivileged user via the CLI or gRPC. No special access, leaked keys, or cryptographic break is required. The stub handler guarantees the permission check is never reached.

---

### Recommendation

Implement `TurnBridge` to:
1. Call `k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `sdkerrors.ErrUnauthorized` on failure.
2. Persist the bridge-enabled boolean to the module's KV store.
3. Add a bridge-state guard at the top of `ConvertVouchersToEvmCoins` and `IbcTransferCoins` that returns an error when the bridge is disabled.

---

### Proof of Concept

1. Any unprivileged user submits `MsgTurnBridge{From: <anyAddress>, Enable: false}`.
2. The handler at `msg_server.go:85–87` returns `nil, nil` immediately — no permission check, no state write.
3. The transaction is accepted on-chain with a success status.
4. `ConvertVouchersToEvmCoins` and `IbcTransferCoins` continue to execute normally because no bridge-disabled state was ever written.
5. In a real bridge emergency, the admin's only recourse (the circuit breaker) is silently inoperative.

```go
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

```go
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {

	}

	// msg is already validated

		return nil, err
	}
```

```go
func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
}
```

```go
func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
	}
	defer func() {
		for _, a := range coins {
			if a.Amount.IsInt64() {
				telemetry.SetGaugeWithLabels(
					[]string{"tx", "msg", "ConvertVouchersToEvmCoins"},
					float32(a.Amount.Int64()),
					[]metrics.Label{telemetry.NewLabel("denom", a.Denom)},
				)
			}
		}
	}()
	return nil
}
```
