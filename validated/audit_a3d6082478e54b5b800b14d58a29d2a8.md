### Title
Missing Zero-Address Validation for `reward_address` Enables Temporary Freezing of Staker/Delegator Funds - (File: src/staking/staking.cairo, src/pool/pool.cairo)

---

### Summary

Neither `stake()` nor `change_reward_address()` in the Staking contract, nor `enter_delegation_pool()` in the Pool contract, validate that the supplied `reward_address` is non-zero. Because `send_rewards_to_staker` and `claim_rewards` unconditionally call `checked_transfer` to `reward_address`, setting it to the zero address causes those transfers to revert, temporarily freezing unclaimed yield and blocking `unstake_action` from completing.

---

### Finding Description

The `stake()` function in `staking.cairo` accepts a `reward_address` parameter and stores it without checking for the zero address:

```cairo
// src/staking/staking.cairo  lines 288-316
fn stake(ref self: ContractState, reward_address: ContractAddress, ...) {
    ...
    assert!(!self.does_token_exist(token_address: reward_address), ...); // only token check
    // ← no assert!(reward_address.is_non_zero(), ...)
    ...
    self.staker_info.write(staker_address,
        VInternalStakerInfoTrait::new_latest(:reward_address, :operational_address));
}
```

`change_reward_address()` has the same gap:

```cairo
// src/staking/staking.cairo  lines 517-540
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    assert!(!self.does_token_exist(token_address: reward_address), ...);
    // ← no zero-address check
    staker_info.reward_address = reward_address;
    self.write_staker_info(:staker_address, :staker_info);
}
```

`enter_delegation_pool()` in `pool.cairo` has the same gap:

```cairo
// src/pool/pool.cairo  lines 182-206
fn enter_delegation_pool(ref self: ContractState, reward_address: ContractAddress, amount: Amount) {
    ...
    assert!(token_address != reward_address, ...); // only token check
    // ← no zero-address check
    self.pool_member_info.write(pool_member,
        VInternalPoolMemberInfoTrait::new_latest(:reward_address));
}
```

When rewards are later claimed, `send_rewards_to_staker` calls `checked_transfer` directly to `reward_address`:

```cairo
// src/staking/staking.cairo  lines 1620-1625
let reward_address = staker_info.reward_address;
let amount = staker_info.unclaimed_rewards_own;
claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
```

And `unstake_action` calls `send_rewards_to_staker` **before** returning the principal:

```cairo
// src/staking/staking.cairo  lines 492-506
self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
...
token_dispatcher.checked_transfer(recipient: staker_address, amount: staker_amount.into());
```

If `reward_address` is zero and `unclaimed_rewards_own > 0`, the `checked_transfer` to the zero address reverts (OpenZeppelin Cairo ERC20 rejects transfers to address zero), causing `unstake_action` to revert entirely and blocking the staker from recovering their principal.

The same pattern applies in the Pool contract's `claim_rewards`:

```cairo
// src/pool/pool.cairo  line 366
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
```

---

### Impact Explanation

A staker who (accidentally or deliberately) supplies `reward_address = 0` at `stake()` time, or later calls `change_reward_address(0)`, will find:

1. **`claim_rewards` reverts** — unclaimed STRK yield is frozen in the staking contract.
2. **`unstake_action` reverts** — because it calls `send_rewards_to_staker` first, the staker's principal is also temporarily frozen.

The staker can recover by calling `change_reward_address` (which has no `unstake_time` guard) to set a valid address, then retrying `unstake_action`. The freeze is therefore **temporary**, but it blocks the full exit flow until corrected. The same applies to pool members via `Pool::claim_rewards`.

Impact category: **Temporary freezing of funds** (staker principal locked in exit window; unclaimed yield inaccessible).

---

### Likelihood Explanation

Any unprivileged staker or pool member can trigger this by passing `ContractAddress { address: 0 }` as `reward_address` in `stake()` or `enter_delegation_pool()`, or by calling `change_reward_address(0)`. No privileged access is required. The scenario is realistic as an accidental misconfiguration (e.g., a front-end bug or scripting error passing a default/uninitialized address).

---

### Recommendation

Add a non-zero guard in all three entry points:

```cairo
// In stake(), change_reward_address() (staking.cairo), and enter_delegation_pool(), 
// change_reward_address() (pool.cairo):
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

The error string `ZERO_ADDRESS` ("Zero address") is already defined in the errors module, confirming the protocol authors anticipated this class of check.

---

### Proof of Concept

1. Call `stake(reward_address: 0, operational_address: <valid>, amount: <min_stake>)`.
2. Advance epochs and attest to accumulate non-zero `unclaimed_rewards_own`.
3. Call `unstake_intent()` — succeeds.
4. Advance past `exit_wait_window`.
5. Call `unstake_action(staker_address)` — **reverts** because `send_rewards_to_staker` calls `checked_transfer(recipient: 0, amount: rewards)` which the ERC20 rejects.
6. Principal remains locked in the staking contract until the staker calls `change_reward_address(<valid>)` and retries. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());

        fn send_rewards_to_staker(
            ref self: ContractState,
            staker_address: ContractAddress,
            ref staker_info: InternalStakerInfoLatest,

        ) {
            let reward_address = staker_info.reward_address;
            let amount = staker_info.unclaimed_rewards_own;

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
