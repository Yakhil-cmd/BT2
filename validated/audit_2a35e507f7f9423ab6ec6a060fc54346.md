### Title
`meta_tx_v0` Syscall Constructs a Nonce-Free Transaction Hash, Enabling Unlimited Signature Replay Against Account `__execute__` — (`crates/blockifier/src/execution/syscalls/syscall_base.rs`)

### Summary

The `meta_tx_v0` syscall builds a synthetic `InvokeTransactionV0` context whose hash is computed without any nonce, and hardcodes `nonce = 0` in the resulting `TxInfo`. Because `__validate__` is entirely bypassed and the hash is fully determined by `(contract_address, calldata, chain_id)`, any `(calldata, signature)` pair that was valid once remains valid forever. An unprivileged attacker who observes a single on-chain `meta_tx_v0` invocation can replay it in any subsequent outer transaction, causing the target account's `__execute__` to run again with the identical hash and signature.

### Finding Description

**Root cause — hash excludes nonce**

`get_common_invoke_transaction_v0_hash` in `crates/starknet_api/src/transaction_hash.rs` computes:

```
INVOKE | version | contract_address | entry_point_selector | calldata_hash | max_fee | chain_id
```

No nonce field is present. [1](#0-0) 

**Root cause — nonce hardcoded to zero in the synthetic context**

`SyscallBaseHandler::meta_tx_v0` constructs the replacement `TransactionInfo` with `nonce: Nonce(0.into())` and never calls `increment_nonce`. [2](#0-1) 

The OS Cairo implementation mirrors this: `nonce=0` is written into the new `TxInfo` struct unconditionally. [3](#0-2) 

**Root cause — `__validate__` is skipped**

`meta_tx_v0` calls `execute_inner_call` directly on the `__execute__` entry point. The standard OS flow that calls `__validate__` (which is where account contracts check nonces) is never invoked. [4](#0-3) 

**Root cause — V0 nonce handling is a no-op**

Even if `handle_nonce` were reached for the meta-tx context, it returns immediately for any V0 transaction, so no nonce increment or check would occur. [5](#0-4) 

**Exploit path**

1. A public relayer contract `R` exposes `relay(address, calldata, signature)` which calls `meta_tx_v0(address, __execute__, calldata, signature)`.
2. Alice signs `sig = sign(H(INVOKE, 0, alice_account, __execute__, calldata_hash, 0, chain_id))` and submits an outer transaction to `R`.
3. The meta-tx hash `H` is computed identically every time from the same `(alice_account, calldata, chain_id)` tuple — no nonce is mixed in.
4. Alice's `__execute__` runs; the OS presents `(transaction_hash=H, nonce=0, signature=sig)` to the contract.
5. An attacker observes the call on-chain and submits a new outer transaction (with the attacker's own nonce) to `R` with the identical `(alice_account, calldata, sig)`.
6. `meta_tx_v0` recomputes the same `H`, presents the same `(H, nonce=0, sig)` to Alice's `__execute__`, which passes signature verification again.
7. Alice's `__execute__` runs a second time — replay complete.

The attacker requires no privileged keys; only the ability to submit a normal outer transaction.

### Impact Explanation

The target account's `__execute__` function is invoked with an identical `(transaction_hash, nonce, signature)` triple on every replay. Standard Starknet account contracts rely on the OS to enforce nonce uniqueness via `__validate__`; they do not re-check nonces inside `__execute__`. Consequently, any state-mutating action encoded in the meta-tx calldata (token transfers, approvals, governance votes, etc.) can be repeated an unlimited number of times by any unprivileged observer. This constitutes unauthorized transaction acceptance through broken nonce/signature logic — matching the **Critical** impact tier: *"Invalid or unauthorized Starknet transaction accepted through account validation, signature, nonce … logic."*

### Likelihood Explanation

The `meta_tx_v0` syscall is a standard, publicly accessible syscall available to any deployed contract. Any relayer or batching contract that exposes a public entry point wrapping `meta_tx_v0` is immediately exploitable by any network participant who can read on-chain calldata. No leaked secrets, no admin keys, and no special network position are required.

### Recommendation

Include the outer transaction's nonce (or a monotonically increasing per-account meta-tx nonce stored in the relayer or the target account) in the meta-tx hash computation. Concretely, `get_common_invoke_transaction_v0_hash` should accept and commit to a nonce field when used in the `meta_tx_v0` context, and the OS should read and increment a dedicated meta-tx nonce from the target account's storage before executing `__execute__`. Alternatively, require that `__validate__` is called before `__execute__` even in the meta-tx flow, so the account's own replay-protection logic is exercised.

### Proof of Concept

```
// Outer transaction (attacker-controlled, any nonce):
//   calls relayer.relay(alice_account, calldata, alice_sig)
//
// Relayer contract (public, no access control):
//   fn relay(address, calldata, signature) {
//       meta_tx_v0_syscall(address, __execute__, calldata, signature)
//   }
//
// Meta-tx hash (computed identically on every replay):
//   H = Pedersen(INVOKE, 0, alice_account, __execute__selector,
//                Pedersen(calldata...), 0 /*max_fee*/, chain_id)
//   -- nonce is absent --
//
// Alice's __execute__ receives:
//   tx_info.transaction_hash = H   (same every replay)
//   tx_info.nonce            = 0   (same every replay)
//   tx_info.signature        = alice_sig (same every replay)
//
// Signature check inside __execute__: verify(alice_sig, H) → PASS (every time)
// Result: alice's action executes on every replay with no additional authorization.

    pub fn meta_tx_v0(
        &mut self,
        contract_address: ContractAddress,
        entry_point_selector: EntryPointSelector,
        calldata: Calldata,
        signature: TransactionSignature,
        remaining_gas: &mut u64,
    ) -> SyscallResult<Vec<Felt>> {
        self.increment_syscall_linear_factor_by(&SyscallSelector::MetaTxV0, calldata.0.len());
        if self.context.execution_mode == ExecutionMode::Validate {
            self.reject_syscall_in_validate_mode("meta_tx_v0")?;
        }
        if entry_point_selector != selector_from_name(EXECUTE_ENTRY_POINT_NAME) {
            return Err(SyscallExecutionError::Revert { error_data: vec![INVALID_ARGUMENT_FELT] });
        }
        let entry_point = CallEntryPoint {
            class_hash: None,
            code_address: Some(contract_address),
            entry_point_type: EntryPointType::External,
            entry_point_selector,
            calldata: calldata.clone(),
            storage_address: contract_address,
            caller_address: ContractAddress::default(),
            call_type: CallType::Call,
            // NOTE: this value might be overridden later on.
            initial_gas: *remaining_gas,
        };

        let old_tx_context = self.context.tx_context.clone();
        let only_query = old_tx_context.tx_info.only_query();

        // Compute meta-transaction hash.
        let transaction_hash = InvokeTransactionV0 {
            max_fee: Fee(0),
            signature: signature.clone(),
            contract_address,
            entry_point_selector,
            calldata,
        }
        .calculate_transaction_hash(
            &self.context.tx_context.block_context.chain_info.chain_id,
            &signed_tx_version(&TransactionVersion::ZERO, &TransactionOptions { only_query }),
        )?;

        let class_hash = self.state.get_class_hash_at(contract_address)?;

        // Replace `tx_context`.
        let new_tx_info = TransactionInfo::Deprecated(DeprecatedTransactionInfo {
            common_fields: CommonAccountFields {
                transaction_hash,
                version: TransactionVersion::ZERO,
                signature,
                nonce: Nonce(0.into()),
                sender_address: contract_address,
                only_query,
            },
            max_fee: Fee(0),
        });
        self.context.tx_context = Arc::new(TransactionContext {
            block_context: old_tx_context.block_context.clone(),
            tx_info: new_tx_info,
        });

        // No error should be propagated until we restore the old `tx_context`.
        let result = self.execute_inner_call(entry_point, remaining_gas).map_err(|error| {
            SyscallExecutionError::from_self_or_revert(error.try_extract_revert().map_original(
                |error| {
                    // TODO(lior): Change to meta-tx specific error.
                    error.as_call_contract_execution_error(
                        class_hash,
                        contract_address,
                        entry_point_selector,
                    )
                },
            ))
        });

        // Restore the old `tx_context`.
        self.context.tx_context = old_tx_context;

        result
    }
