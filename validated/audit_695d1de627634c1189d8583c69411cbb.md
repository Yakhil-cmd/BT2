### Title
Unbounded Loop Over `pool_member_epoch_balance` Trace in `calculate_rewards` Can Permanently Freeze Delegator's Unclaimed Yield - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` function in `Pool` iterates over a pool member's entire balance-change trace without any cap. The trace grows by one entry per epoch in which the member changes their balance. A pool member who makes balance changes across many epochs without claiming rewards will eventually be unable to call `claim_rewards` because the loop exhausts the Starknet transaction gas limit, permanently freezing their accrued yield. The code itself acknowledges this: *"This loop is unbounded but unlikely to exceed gas limits."*

### Finding Description
`calculate_rewards` in `src/pool/pool.cairo` (lines 837–888) drives reward computation for every `claim_rewards` call. It iterates over the `pool_member_epoch_balance` trace from `entry_to_claim_from` to the current epoch:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch { break; }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    ...
    entry_to_claim_from += 1;
}

Each call to `set_member_balance` appends a new checkpoint to the `PoolMemberBalanceTrace` storage vector whenever the target epoch differs from the last stored key:

```cairo

The `insert` implementation only deduplicates when the incoming key equals the last stored key; otherwise it unconditionally appends:

`set_member_balance` is called by `increase_member_balance` (used in `add_to_delegation_pool` and `enter_delegation_pool_from_staking_contract`) and directly by `exit_delegation_pool_intent`. Each of these is a normal, unprivileged pool-member action. [4](#0-3) 

The `entry_to_claim_from` cursor stored in `pool_member_info` advances only when `claim_rewards` succeeds. If the member never claims, the cursor stays at 0 and every future `claim_rewards` call must traverse the entire accumulated trace from the beginning. [5](#0-4) 

Each iteration of the loop also calls `find_sigma`, which performs multiple storage reads from `cumulative_rewards_trace`, making each iteration non-trivial in gas cost. [6](#0-5) 

### Impact Explanation
When the trace length exceeds the gas budget for a single Starknet transaction, `claim_rewards` reverts on every attempt. Because `exit_delegation_pool_action` does **not** call `claim_rewards` — it only returns the principal — the pool member can recover their staked tokens but can never collect the accumulated yield. The unclaimed yield is permanently frozen in the contract with no recovery path. [7](#0-6) 

This matches the allowed High impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The trace grows at most one entry per epoch (same-epoch changes are deduplicated). The rate of growth is therefore bounded by how often the member changes their balance across distinct epochs. A member who adds to or partially exits their delegation once per epoch — a plausible DCA or rebalancing pattern — and defers reward claims for an extended period will accumulate entries linearly with time. The developers themselves flagged this as a known risk ("unlikely to exceed gas limits"), acknowledging that the bound is not enforced by the protocol. Likelihood is **low-to-medium** over a multi-year horizon for active delegators who do not claim regularly.

### Recommendation
1. **Enforce a maximum trace length per pool member.** Reject `set_member_balance` calls that would push the trace beyond a safe constant (e.g., 500 entries), or require the member to claim rewards before further balance changes are accepted.
2. **Alternatively, paginate `calculate_rewards`.** Accept a `max_iterations` parameter and allow partial reward claims, advancing `entry_to_claim_from` incrementally across multiple transactions.
3. **Emit a warning or revert** when `pool_member_trace_length - entry_to_claim_from` exceeds a threshold, prompting the member to claim before the trace grows further.

### Proof of Concept

**Setup:**
- Staker stakes and opens a delegation pool.
- Delegator enters the pool with the minimum delegation amount.
- For N consecutive epochs (e.g., N = 600), the delegator calls `add_to_delegation_pool` with a small amount each epoch, creating one new trace entry per epoch, and **never** calls `claim_rewards`.

**Trigger:**
- After N epochs, the delegator calls `claim_rewards`.
- `calculate_rewards` enters the `while` loop with `pool_member_trace_length = N` and `entry_to_claim_from = 0`.
- Each of the N iterations reads from `pool_member_epoch_balance` storage and calls `find_sigma` (which reads from `cumulative_rewards_trace`).
- At sufficiently large N the transaction exceeds the Starknet gas limit and reverts.

**Outcome:**
- Every subsequent `claim_rewards` call also reverts (the cursor never advances).
- `exit_delegation_pool_action` succeeds and returns the principal, but the accumulated yield remains locked in the contract forever.

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

#[generate_trait]
pub impl MutablePoolMemberBalanceTraceImpl of MutablePoolMemberBalanceTraceTrait {
    /// Inserts a (`key`, `value`) pair into a Trace so that it is stored as the checkpoint.
    /// This is done by either inserting a new checkpoint, or updating the last one.
    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
