### Title
Anyone Can Call `update_rewards` with `disable_rewards: true` to Permanently Freeze Block Rewards for All Stakers — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and accepts a caller-controlled `disable_rewards` boolean. Any unprivileged address can invoke it with `disable_rewards: true` for any valid staker. The function unconditionally writes the current block number to the global `last_reward_block` storage variable **before** checking `disable_rewards`, permanently consuming the block's reward slot without distributing any rewards.

---

### Finding Description

`update_rewards` is part of the `IStakingRewardsManager` interface and is the sole mechanism for distributing per-block consensus rewards to stakers.

The execution order is:

1. **Line 1454–1458** — Assert `current_block_number > last_reward_block` (global, once-per-block gate).
2. **Lines 1466–1482** — Assert the supplied `staker_address` exists and is active with non-zero balance.
3. **Line 1485** — `self.last_reward_block.write(current_block_number)` — **global state is mutated here, unconditionally.**
4. **Line 1487** — `if disable_rewards || self.is_pre_consensus() { return; }` — reward distribution is skipped. [1](#0-0) 

Because step 3 precedes step 4, any caller who passes `disable_rewards: true` permanently exhausts the block's reward slot. `last_reward_block` is a **single global** storage variable: [2](#0-1) 

Once it equals the current block number, every subsequent call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`. The block's rewards are irrecoverably lost for **all** stakers and delegators.

There is no role check, no ownership check, and no restriction on who may supply `disable_rewards: true`. The only guard is `general_prerequisites()`, which merely asserts the contract is unpaused and the caller is non-zero: [3](#0-2) 

This is the direct analog of the reported pattern: a function that performs a privileged state mutation (consuming the per-block reward slot) without verifying that the caller is authorized to suppress reward distribution for that resource.

---

### Impact Explanation

**Permanent freezing of unclaimed yield** (allowed High impact).

An attacker calling `update_rewards(any_valid_staker, disable_rewards: true)` once per block:

- Updates `last_reward_block` to the current block.
- Skips `_update_rewards`, so no staker accumulates `unclaimed_rewards_own` and no pool receives rewards via `update_rewards_from_staking_contract`.
- Blocks every legitimate `update_rewards` call for that block.

Repeated every block, this permanently halts all consensus reward accrual for every staker and every delegation pool in the protocol.

---

### Likelihood Explanation

The attack requires one transaction per Starknet block (~3-second block times). No special role, capital, or privileged access is needed — any externally owned address suffices. The gas cost per block is minimal relative to the protocol-wide yield destroyed. The attacker has no profit motive but causes maximum damage to all stakers and delegators, satisfying the griefing criterion.

---

### Recommendation

Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` branch, so that a call with `disable_rewards: true` does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only mark the block as processed when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... _update_rewards ...
```

Alternatively, add a dedicated role check (e.g., `only_app_governor()` or a new `REWARDS_MANAGER_ROLE`) to gate the `disable_rewards: true` path.

---

### Proof of Concept

1. Attacker identifies any currently active staker address `S` with non-zero balance.
2. At the first transaction of each new block, attacker calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `last_reward_block` is set to the current block number; the function returns without distributing rewards.
4. Any legitimate sequencer or staker call to `update_rewards` for that block reverts with `REWARDS_ALREADY_UPDATED`.
5. The block's STRK and BTC consensus rewards are permanently lost.
6. Repeating step 2 every block freezes all future consensus reward accrual across the entire protocol. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
