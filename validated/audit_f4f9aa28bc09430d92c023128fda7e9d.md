Audit Report

## Title
SNS Developer Neuron Stake Locked Indefinitely After Decentralization Swap Abort - (`rs/sns/swap/src/swap.rs`, `rs/sns/governance/src/types.rs`)

## Summary
When an SNS decentralization swap is aborted, `finalize_inner` returns early after `restore_dapp_controllers_for_finalize` and never calls `set_sns_governance_to_normal_mode`. The SNS governance canister remains in `PreInitializationSwap` mode indefinitely, blocking `StartDissolving`, `Disburse`, `Split`, `MergeMaturity`, and `DisburseMaturity` for all developer neurons. Developer neurons created at SNS genesis with real staked SNS tokens therefore cannot be dissolved or disbursed until manual intervention (a canister upgrade) is performed.

## Finding Description
In `rs/sns/swap/src/swap.rs`, `finalize_inner` (L1572–1584) checks `self.should_restore_dapp_control()`, which returns `true` on the aborted lifecycle path. It then calls `restore_dapp_controllers_for_finalize` and unconditionally returns at L1583. The call to `set_sns_governance_to_normal_mode` at L1610–1612 is only reachable on the committed path, never on the aborted path.

In `rs/sns/governance/src/types.rs`, `allows_manage_neuron_command_or_err` (L163–211) enforces that in `PreInitializationSwap` mode only `Follow`, `MakeProposal`, `RegisterVote`, `AddNeuronPermissions`, `RemoveNeuronPermissions`, and `ClaimOrRefresh` (swap canister only) are permitted. `Configure` (containing `StartDissolving`), `Disburse`, `Split`, `MergeMaturity`, and `DisburseMaturity` all fall through to the `_ => false` arm and return `PreconditionFailed`.

Because `StartDissolving` is blocked, a developer neuron can never transition from `NotDissolving` to `Dissolved`. Because `Disburse` in `rs/sns/governance/src/governance.rs` (L1129–1136) requires `NeuronState::Dissolved`, the staked tokens cannot be retrieved. The integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` (L1268–1289) explicitly asserts this behavior: after an abort, `start_dissolving_neuron` returns `PreconditionFailed` containing `"PreInitializationSwap"`.

There is no governance proposal type that transitions SNS governance mode from `PreInitializationSwap` back to `Normal`, and no automatic post-upgrade hook performs this transition. Recovery requires a manual canister upgrade.

## Impact Explanation
Developer neurons created at SNS genesis hold real staked SNS governance tokens. After a swap abort, those tokens are inaccessible for an unbounded period — the owner cannot start dissolving, disburse, split, or disburse maturity. The tokens are not permanently destroyed (they remain in the neuron's ledger subaccount) but are locked until a manual canister upgrade is performed. This constitutes a moderate user-funds/security impact matching the Medium ($200–$2,000) bounty tier: "moderate user-funds/security impact."

## Likelihood Explanation
A swap aborts whenever `min_participants`, `min_icp_e8s`, or `min_direct_participation_icp_e8s` thresholds are not met before `swap_due_timestamp_seconds`. This is a normal, expected lifecycle outcome. No privileged access, key compromise, or majority attack is required. Any SNS project whose swap fails minimum participation triggers this condition for all developer neurons automatically.

## Recommendation
In `finalize_inner` (`rs/sns/swap/src/swap.rs`), call `set_sns_governance_to_normal_mode` on the aborted path as well, or introduce a dedicated mode (e.g., `PostAbortedSwap`) that lifts neuron-management restrictions while retaining other `PreInitializationSwap` guards (e.g., treasury transfer blocks). Alternatively, add a `release_developer_neurons` endpoint callable only by the swap canister after an abort that explicitly permits `StartDissolving` and `Disburse` for developer neurons.

## Proof of Concept
1. Initialize an SNS; developer neurons are created with `dissolve_delay_seconds = ONE_YEAR` and staked tokens.
2. Open the decentralization swap; let it close without meeting `min_participants`.
3. Call `try_abort`; lifecycle transitions to `Aborted`.
4. Call `finalize`; `finalize_inner` executes `sweep_icp`, `settle_neurons_fund_participation`, `restore_dapp_controllers_for_finalize`, then returns early — `set_sns_governance_to_normal_mode` is never called.
5. Developer neuron owner calls `manage_neuron` with `Command::Configure(StartDissolving)` → `PreconditionFailed: "PreInitializationSwap"`.
6. Developer neuron owner calls `manage_neuron` with `Command::Disburse` → same rejection.
7. Staked tokens remain locked until a manual canister upgrade changes the governance mode.

This exact behavior is already asserted as passing in `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` (L1268–1289), providing a ready-made reproducible test.
