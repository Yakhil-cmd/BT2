### Title
Unbounded loop in `calculate_rewards` can permanently freeze a delegator's unclaimed yield — (File: src/pool/pool.cairo)

---

### Summary

The `calculate_rewards` internal function in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop that iterates over every balance-change entry in a pool member's `pool_member_epoch_balance` trace. A delegator who repeatedly enters and exits the delegation pool across many epochs accumulates an unbounded number of trace entries. When `claim_rewards` is eventually called, the loop iterates over all of them with no cap, and beyond a certain trace length the transaction will exhaust the Starknet block gas limit and revert — permanently freezing the delegator's unclaimed yield.

---

### Finding Description

`calculate_rewards` (an internal function called during reward claiming) iterates over the entire historical balance-change trace of a pool member:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    rewards += compute_rewards_rounded_down(...);
    from_sigma = to_sigma;
    from_balance = pool_member_checkpoint.balance();
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The developers themselves flag this as unbounded at line 857–858. Every time a delegator changes their balance (each `enter_delegation_pool`, `exit_delegation_pool_intent`/`exit_delegation_pool_action` cycle), a new checkpoint is appended to `pool_member_epoch_balance`. There is no maximum on how many checkpoints a single pool member may accumulate, and there is no pagination or batch-claim mechanism. Additionally, `find_sigma` is called once per loop iteration, and `find_sigma_standard_case` (in `src/pool/utils.cairo`) performs its own trace scan, making the per-iteration cost non-trivial. [2](#0-1) 

---

### Impact Explanation

Once the trace is long enough that a single `claim_rewards` call exhausts the block gas limit, every future call to claim rewards for that pool member will revert. The delegator's accrued yield becomes permanently unclaimable — a **permanent freezing of unclaimed yield**, which is an explicitly allowed High-severity impact in the bounty scope.

---

### Likelihood Explanation

The attack surface is reachable by any unprivileged delegator with no special permissions. A delegator who actively manages their position — entering and exiting the pool across many epochs — will organically grow their trace. The protocol has no minimum time between balance changes and no cap on the number of checkpoints. Over a sufficiently long protocol lifetime (many epochs), even a moderately active delegator can reach the threshold. The developers' own comment acknowledges the loop is unbounded, confirming awareness of the risk.

---

### Recommendation

1. **Introduce a maximum iteration cap** per `claim_rewards` call (analogous to `maxGas` in Morpho), storing `entry_to_claim_from` in persistent state so subsequent calls resume where the previous one stopped.
2. **Alternatively**, implement a paginated `claim_rewards(max_entries: u32)` interface that allows partial reward collection, preventing any single call from exceeding gas limits.
3. **Enforce a maximum number of balance-change checkpoints** per pool member (e.g., by requiring a minimum epoch gap between balance changes), bounding the trace length at the protocol level.

---

### Proof of Concept

1. Deploy the protocol and register a staker with a delegation pool.
2. As an unprivileged delegator, repeatedly call `enter_delegation_pool` followed by `exit_delegation_pool_intent` + `exit_delegation_pool_action` across consecutive epochs. Each full cycle appends one entry to `pool_member_epoch_balance`.
3. After N such cycles (where N is large enough to exhaust Starknet block gas in the loop), call `claim_rewards` on the pool contract.
4. The `calculate_rewards` loop iterates over all N checkpoints, calling `find_sigma` on each. The transaction runs out of gas and reverts.
5. Every subsequent `claim_rewards` call for this pool member also reverts — the delegator's accumulated yield is permanently frozen.

The root cause is the absence of any bound on `pool_member_trace_length` in the loop at `src/pool/pool.cairo` line 859, directly analogous to `maxGas = 0` or an excessively low `maxGas` in the Morpho report. [3](#0-2)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
