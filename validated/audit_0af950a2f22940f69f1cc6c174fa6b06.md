### Title
Gateway Stateful Validator Admits Transactions With Insufficient L1/L1-Data Gas Price — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check only validates the L2 gas price against the previous block's price. It performs **no validation** of `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` for `AllResources` transactions, and **no validation at all** for `L1Gas` (legacy) transactions. The blockifier's `check_fee_bounds` enforces all three gas prices against the current block. The result is that the gateway admits transactions the blockifier will deterministically reject, breaking the admission invariant.

### Finding Description

`validate_tx_l2_gas_price_within_threshold` in `StatefulTransactionValidator` only inspects `l2_gas.max_price_per_unit`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:364-388
match tx_resource_bounds {
    ValidResourceBounds::AllResources(tx_resource_bounds) => {
        let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
        // ... only l2_gas_price is checked against threshold
    }
    ValidResourceBounds::L1Gas(_) => {
        // No validation required for legacy transactions.
    }
}
```

The blockifier's `check_fee_bounds` (called from `perform_pre_validation_stage`) checks **all three** gas prices against the current block:

```rust
// crates/blockifier/src/transaction/account_transaction.rs:441-449
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow {
            resource: *resource,
            max_gas_price: resource_bounds.max_price_per_unit,
            actual_gas_price: (*actual_gas_price).into(),
        },
    );
}
```

This is applied to L1Gas, L1DataGas, and L2Gas. A transaction with `l1_gas.max_price_per_unit = 0` (or any value below the current block's L1 gas price) passes both the stateless validator and the stateful gateway validator, is admitted to the mempool, and is then deterministically rejected by the blockifier with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`.

The TODO comment at line 358 acknowledges the gap: `// TODO(Arni): Consider running this validation for all gas prices.`

### Impact Explanation

This matches the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

An unprivileged sender can flood the mempool with transactions that carry a valid L2 gas price (passing the gateway threshold check) but a zero or sub-market L1 gas price. Every such transaction passes gateway admission, occupies a mempool slot, and is handed to the batcher, which then discards it after a failed `check_fee_bounds` call. Because the blockifier rejection happens in `perform_pre_validation_stage` (before any fee is charged), the attacker bears no cost beyond the gateway's stateless size/calldata checks.

The structural analog to the Frax `liquidateClean` bug is direct: Frax accepted a repayment of X shares without verifying that X ≤ available collateral; the gateway here accepts a transaction without verifying that `l1_gas.max_price_per_unit ≥ current_block_l1_gas_price`, the exact bound the blockifier will enforce.

### Likelihood Explanation

- Trigger requires only a standard V3 `invoke` or `declare` transaction with `l1_gas.max_price_per_unit` set below the current L1 gas price.
- No privileged access, special account class, or unusual network condition is required.
- The stateless validator does not check L1 gas price at all; the stateful validator explicitly skips it.
- L1 gas prices fluctuate with Ethereum base fee, so any period of rising L1 fees creates a natural window where previously-valid price values become sub-market.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or replace it with a unified helper) to check all three gas prices against the previous block's corresponding prices, mirroring the logic already present in `check_fee_bounds`:

```rust
// For AllResources: also check l1_gas and l1_data_gas prices
let tx_l1_gas_price = tx_resource_bounds.l1_gas.max_price_per_unit;
let prev_l1_gas_price = block_info.gas_prices.strk_gas_prices.l1_gas_price;
if tx_l1_gas_price.0 < threshold_for(prev_l1_gas_price, percentage) {
    return Err(...);
}
// similarly for l1_data_gas
```

For `L1Gas` (legacy) transactions, apply the same threshold check to `l1_gas.max_price_per_unit`.

### Proof of Concept

1. Observe the current block's L1 gas price (e.g., `prev_block.gas_prices.strk_gas_prices.l1_gas_price = 100`).
2. Submit a V3 invoke transaction with:
   - `l2_gas.max_price_per_unit = 200` (above the L2 threshold → passes `validate_tx_l2_gas_price_within_threshold`)
   - `l1_gas.max_price_per_unit = 0` (below the L1 gas price → not checked by gateway)
3. The gateway calls `validate_resource_bounds` → only L2 price is checked → **admitted**.
4. The batcher calls `perform_pre_validation_stage` → `check_fee_bounds` checks L1 gas price → `MaxGasPriceTooLow { resource: L1Gas, max_gas_price: 0, actual_gas_price: 100 }` → **rejected**.
5. The transaction occupied a mempool slot and consumed batcher CPU with zero cost to the attacker.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
