### Title
Unbounded Griefing via `disable_rewards: true` in `update_rewards` Permanently Freezes Consensus Yield — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable by any unprivileged actor. It accepts a caller-controlled `disable_rewards` boolean. Because `last_reward_block` is written to storage **before** the `disable_rewards` guard is evaluated, any attacker can call `update_rewards(valid_staker, disable_rewards: true)` once per block to consume the block's single reward-update slot without distributing any rewards. This permanently freezes consensus-phase yield for all stakers.

---

### Finding Description

`update_rewards` is exposed through `IStakingRewardsManager` with no role-based access control. Its only caller gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function enforces a single-call-per-block invariant by asserting `current_block_number > last_reward_block`. Immediately after validating the staker, it commits the current block number to `last_reward_block`: [2](#0-1) 

Only **after** this write does the function branch on `disable_rewards`:

```cairo
self.last_reward_block.write(current_block_number);   // slot consumed

if disable_rewards || self.is_pre_consensus() {
    return;   // no rewards distributed
}
```

Because the slot is consumed unconditionally, any subsequent call in the same block — even a legitimate one with `disable_rewards: false` — reverts with `REWARDS_ALREADY_UPDATED`. The attacker does not need to be a staker; they only need to supply any valid, active staker address (trivially discoverable from on-chain events). [3](#0-2) 

---

### Impact Explanation

During the consensus-rewards phase, every block that is "consumed" by an attacker's `disable_rewards: true` call produces zero yield for all stakers. The rewards for those blocks are never accrued and cannot be recovered retroactively. If the attacker repeats this every block, **all consensus-phase staking yield is permanently frozen** — matching the allowed impact: *High: Permanent freezing of unclaimed yield*.

---

### Likelihood Explanation

The attack requires:
- No stake, no privileged role, no special token balance.
- One cheap transaction per block (Starknet L2 fees are minimal).
- A single valid active staker address, which is publicly observable from `NewStaker` events.

The cost to the attacker is negligible; the damage to all stakers is total and ongoing.

---

### Recommendation

1. **Remove the `disable_rewards` parameter from the public interface.** Handle the pre-consensus early-return internally (the `is_pre_consensus()` branch already covers it).
2. If `disable_rewards` must remain, gate it behind an access-control check so only the staker themselves (or a designated operator role) can pass `true`.
3. Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard so that a no-op call does not consume the block's reward slot.

---

### Proof of Concept

1. Consensus rewards become active (i.e., `current_epoch >= consensus_rewards_first_epoch`).
2. Attacker observes any valid, active staker address `S` from on-chain events.
3. At the start of every block, attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. `last_reward_block` is set to the current block number; the function returns early.
5. Any legitimate call `update_rewards(S, disable_rewards: false)` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. No staker receives block rewards for that block. Repeating every block yields zero consensus rewards for the entire protocol indefinitely. [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
