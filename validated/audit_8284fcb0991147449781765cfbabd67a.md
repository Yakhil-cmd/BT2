### Title
Silo Fixed-Gas Fee Bypass via Zero `max_fee_per_gas` — (`engine/src/engine.rs`)

### Summary

In Silo mode, the Aurora Engine charges a per-transaction fee using a `fixed_gas` amount set by the operator. However, the fee is computed as `fixed_gas * effective_gas_price`, and `effective_gas_price` is derived entirely from the user-controlled `max_fee_per_gas` field. When a user submits a transaction with `max_fee_per_gas = 0`, the `effective_gas_price` collapses to zero, making `prepaid_amount = 0` regardless of the configured `fixed_gas`. The user executes the transaction for free, and the relayer receives no reward.

### Finding Description

In `submit_with_alt_modexp` (`engine/src/engine.rs`), the Silo fixed gas is fetched and passed to `charge_gas`:

```rust
let fixed_gas = silo::get_fixed_gas(&io);
// ...
let prepaid_amount = match engine.charge_gas(&sender, &transaction, max_gas_price, fixed_gas) {
``` [1](#0-0) 

Inside `charge_gas`, the early-return guard correctly excludes the case where `fixed_gas` is set:

```rust
if transaction.max_fee_per_gas.is_zero()
    && fixed_gas.is_none()          // ← guard: only skips when no fixed_gas
    && block_base_fee_per_gas.is_zero()
{
    return Ok(GasPaymentResult::default());
}
``` [2](#0-1) 

However, the `effective_gas_price` is still computed purely from the transaction's `max_fee_per_gas` and the block base fee:

```rust
let priority_fee_per_gas = transaction
    .max_priority_fee_per_gas
    .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;
``` [3](#0-2) 

When `max_fee_per_gas = 0` and `block_base_fee_per_gas = 0` (Aurora's default), `effective_gas_price = 0`. The prepaid amount then becomes:

```rust
let prepaid_amount = fixed_gas
    .map_or(transaction.gas_limit, EthGas::as_u256)
    .checked_mul(effective_gas_price)   // fixed_gas * 0 = 0
    .map(Wei::new)
    .ok_or(GasPaymentError::EthAmountOverflow)?;
``` [4](#0-3) 

`prepaid_amount = 0`, so no ETH is deducted from the sender. In `refund_unused_gas`, the early return on `effective_gas_price.is_zero()` also prevents the relayer from receiving any reward:

```rust
if gas_result.effective_gas_price.is_zero() {
    return Ok(());
}
``` [5](#0-4) 

There is no validation that enforces `max_fee_per_gas > 0` when `fixed_gas` is configured. The only existing check (`fixed_gas <= gas_limit`) does not address this:

```rust
if fixed_gas.is_some_and(|gas| gas.as_u256() > transaction.gas_limit) {
    return Err(EngineErrorKind::FixedGasOverflow.into());
}
``` [6](#0-5) 

### Impact Explanation

In Silo mode, the operator configures `fixed_gas` to guarantee a minimum fee per transaction, paid to the relayer. A user who submits a transaction with `max_fee_per_gas = 0` bypasses this fee entirely: they pay nothing, and the relayer receives nothing. This constitutes theft of the relayer's expected yield on every such transaction. The Silo fee mechanism is rendered ineffective.

**Impact: High — Theft of unclaimed yield (relayer fees).**

### Likelihood Explanation

Any unprivileged EVM user can craft a signed Ethereum transaction with `max_fee_per_gas = 0` and submit it via the public `submit` or `submit_with_args` entrypoints. No special access is required. Aurora's `block_base_fee_per_gas` is zero by default, so the condition is trivially satisfied. The attack is repeatable on every transaction.

**Likelihood: High.**

### Recommendation

Add a validation in `charge_gas` (or in `submit_with_alt_modexp` before calling it) that rejects transactions where `effective_gas_price` is zero but `fixed_gas` is set:

```rust
let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;

// Analog fix: if fixed_gas is set, a zero effective_gas_price must be rejected
// (mirrors the external report's pattern: revert when net is zero but total is positive)
if effective_gas_price.is_zero() && fixed_gas.is_some() {
    return Err(GasPaymentError::ZeroGasPriceWithFixedGas);
}
```

Alternatively, treat `fixed_gas` as a flat fee deducted independently of `effective_gas_price`, so that the Silo fee is always charged regardless of the user-supplied gas price.

### Proof of Concept

1. Silo operator calls `set_silo_params` with `fixed_gas = EthGas::new(1_000_000)`.
2. Attacker constructs a Legacy or EIP-1559 transaction with `max_fee_per_gas = 0`, `max_priority_fee_per_gas = 0`, and any valid `gas_limit >= fixed_gas`.
3. Attacker signs and submits via `submit` (public NEAR entrypoint).
4. In `submit_with_alt_modexp`: `fixed_gas = Some(1_000_000)`, early-return guard is NOT triggered.
5. `effective_gas_price = min(0, 0 - 0) + 0 = 0`.
6. `prepaid_amount = 1_000_000 * 0 = 0`. Sender balance unchanged.
7. `refund_unused_gas` returns immediately because `effective_gas_price.is_zero()`. Relayer receives nothing.
8. Transaction executes successfully with zero fee paid. [7](#0-6) [8](#0-7) [9](#0-8)

    pub fn charge_gas(
        &mut self,
        sender: &Address,
        transaction: &NormalizedEthTransaction,
        max_gas_price: Option<U256>,
        fixed_gas: Option<EthGas>,
    ) -> Result<GasPaymentResult, GasPaymentError> {
        let block_base_fee_per_gas = self.block_base_fee_per_gas();
        if transaction.max_fee_per_gas.is_zero()
            && fixed_gas.is_none()
            && block_base_fee_per_gas.is_zero()
        {
            return Ok(GasPaymentResult::default());
        }

        if transaction.max_fee_per_gas < block_base_fee_per_gas {
            return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
        }

        let priority_fee_per_gas = transaction
            .max_priority_fee_per_gas
            .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
        let priority_fee_per_gas = max_gas_price.map_or(priority_fee_per_gas, |price| {
            price.min(priority_fee_per_gas)
        });
        let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;
        // First, we try to use `fixed_gas`. At this point we already know that the `fixed_gas` is
        // less than the `gas_limit`. It allows avoiding refunding unused gas to the sender later.
        let prepaid_amount = fixed_gas
            .map_or(transaction.gas_limit, EthGas::as_u256)
            .checked_mul(effective_gas_price)
            .map(Wei::new)
            .ok_or(GasPaymentError::EthAmountOverflow)?;

        let new_balance = get_balance(&self.io, sender)
            .checked_sub(prepaid_amount)
            .ok_or(GasPaymentError::OutOfFund)?;

        set_balance(&mut self.io, sender, &new_balance);

        self.gas_price = effective_gas_price;

        Ok(GasPaymentResult {
            prepaid_amount,
            effective_gas_price,
            priority_fee_per_gas,
        })
    }
