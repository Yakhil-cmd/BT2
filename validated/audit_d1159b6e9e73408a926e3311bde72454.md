### Title
`update_rewards` can be DoS'd by any caller passing `disable_rewards: true` — (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the Staking contract is publicly callable by any non-zero address and accepts a caller-controlled `disable_rewards` boolean. Because the global `last_reward_block` storage variable is written **unconditionally** before the `disable_rewards` guard, any unprivileged caller can front-run the legitimate block-proposer call with `disable_rewards: true`, permanently preventing reward distribution for that block across all stakers.

### Finding Description
`update_rewards` is the consensus-rewards entry point for V3. Its logic is:

1. Assert `current_block_number > last_reward_block` (enforces one call per block, **globally**).
2. Validate the staker is active and has non-zero balance.
3. **Write `last_reward_block = current_block_number` unconditionally.**
4. If `disable_rewards || is_pre_consensus()` → return early, distributing nothing.
5. Otherwise, calculate and distribute block rewards. [1](#0-0) 

The only caller gate is `general_prerequisites()`, which checks the contract is unpaused and the caller is non-zero — no role or identity check: [2](#0-1) 

Because `last_reward_block` is a single global slot (not per-staker), a single adversarial call with any valid active staker address and `disable_rewards: true` consumes the entire block's reward slot for **all** stakers: [3](#0-2) 

Any subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

### Impact Explanation
Every block in which the adversary front-runs the call, **all stakers permanently lose their block rewards** — the rewards are never credited and are not deferred. Sustained over time this constitutes permanent freezing of unclaimed yield for the entire protocol. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The attack requires:
- No privileged role or capital.
- A valid active staker address (trivially readable from on-chain `NewStaker` events).
- One low-cost transaction per block.

On Starknet, transaction fees are low enough to make sustained per-block griefing economically feasible. The adversary gains nothing directly, but the damage to stakers and the protocol is severe and irreversible per block.

### Recommendation
Restrict `update_rewards` to a trusted caller — for example, require the caller to be the operational address of the staker being rewarded, or introduce a dedicated `REWARDS_DISTRIBUTOR` role. Alternatively, remove `disable_rewards` from the public ABI and derive the no-reward condition internally from attestation state, so external callers cannot suppress reward distribution.

### Proof of Concept
1. Staker `S` is active with non-zero STRK balance in epoch `E`.
2. In block `B`, adversary calls `staking.update_rewards(S, disable_rewards: true)`.
   - `last_reward_block` is set to `B`.
   - No rewards are distributed; function returns early.
3. Block proposer calls `staking.update_rewards(S, disable_rewards: false)` in the same block `B`.
   - Reverts: `current_block_number (B) > last_reward_block (B)` is `false` → `REWARDS_ALREADY_UPDATED`.
4. Staker `S` (and every other staker) receives zero rewards for block `B`.
5. Adversary repeats in every block → all stakers permanently lose all consensus block rewards. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
