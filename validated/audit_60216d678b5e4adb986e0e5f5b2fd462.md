### Title
Unprivileged Caller Can Permanently Suppress Consensus Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public function with no access control. It accepts a caller-controlled `disable_rewards: bool` parameter. When called with `disable_rewards: true`, it writes the current block number into the **global** `last_reward_block` storage slot *before* checking the flag, then returns without distributing any rewards. Because the global slot is now consumed, every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. An unprivileged attacker can call this every block to permanently freeze all consensus-era staker rewards.

---

### Finding Description

`update_rewards` is exposed as a fully public entry point under `IStakingRewardsManager`: [1](#0-0) 

The function begins with only `general_prerequisites()` (unpaused + non-zero caller) and a staker-validity check — no role guard of any kind. [2](#0-1) 

The global `last_reward_block` is written **unconditionally**, before the `disable_rewards` branch: [3](#0-2) 

Only after that write does the code branch on `disable_rewards`: [4](#0-3) 

`last_reward_block` is a single global slot shared across all stakers: [5](#0-4) 

Because the slot is written before the guard, calling `update_rewards(any_valid_staker, disable_rewards: true)` in block N:

1. Passes all validity checks (staker exists, is active, has non-zero balance).
2. Writes `last_reward_block = N`.
3. Returns immediately — zero rewards distributed.
4. Makes every subsequent `update_rewards` call in block N revert with `REWARDS_ALREADY_UPDATED`.

The block's rewards are permanently lost; there is no recovery path.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield / theft of unclaimed yield.**

An attacker who front-runs the legitimate `update_rewards` call every block with `disable_rewards: true` causes all consensus-era block rewards to be permanently unclaimable. Because `last_reward_block` is global, a single griefing transaction per block is sufficient to deny rewards to every staker and every delegation pool in the protocol indefinitely.

A staker-attacker can additionally call `update_rewards(own_staker, disable_rewards: false)` to collect their own block reward while the slot is still open, then the global lock prevents all other stakers from claiming theirs — a direct, repeatable theft of other stakers' unclaimed yield.

---

### Likelihood Explanation

- The function is unconditionally public; no role, no signature, no stake required.
- The only prerequisite is knowing any valid, active staker address — all staker addresses are emitted as `NewStaker` events and are fully public.
- The attack costs one cheap transaction per block and can be automated trivially.
- There is no on-chain mechanism to detect or revert the griefing after the fact.

---

### Recommendation

Apply an access-control guard to `update_rewards` so that only the staker themselves, their operational address, or a designated keeper role may call it. At minimum, move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` branch so that a call with `disable_rewards: true` does not consume the global block slot:

```cairo
// Move this write to after the disable_rewards check
// self.last_reward_block.write(current_block_number);  // <-- remove from here

if disable_rewards || self.is_pre_consensus() {
    // Do NOT write last_reward_block here; the slot remains open
    return;
}

// Write only when rewards are actually distributed
self.last_reward_block.write(current_block_number);
// ... distribute rewards ...
```

---

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(staker=<any_valid_staker>, disable_rewards=true)
    → general_prerequisites() passes (not paused, caller != 0)
    → current_block_number (N) > last_reward_block (N-1) → passes
    → staker is active, has non-zero balance → passes
    → last_reward_block.write(N)          ← slot consumed
    → disable_rewards == true → return    ← no rewards sent

  Legitimate tx: update_rewards(staker=<victim>, disable_rewards=false)
    → current_block_number (N) > last_reward_block (N) → FAILS: REWARDS_ALREADY_UPDATED

Repeat every block → all consensus rewards permanently frozen.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
