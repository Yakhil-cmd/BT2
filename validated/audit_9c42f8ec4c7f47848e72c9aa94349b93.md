### Title
Unprivileged Caller Can Permanently Freeze Per-Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary
The `update_rewards` function in the Staking contract is publicly callable with no access control. Any non-zero address can invoke it with `disable_rewards: true` to consume the global `last_reward_block` slot for the current block without distributing any rewards, permanently destroying that block's yield for all stakers.

---

### Finding Description
`update_rewards` is exposed via `#[abi(embed_v0)]` under `IStakingRewardsManager` and enforces only two prerequisites: the contract is not paused and the caller is not the zero address. [1](#0-0) 

The function accepts a caller-controlled `disable_rewards: bool` parameter. When `true`, the function still writes the current block number into the global `last_reward_block` storage variable, but then returns immediately without distributing any rewards. [2](#0-1) 

Because `last_reward_block` is a global guard that enforces at most one reward update per block: [3](#0-2) 

once an attacker calls `update_rewards(any_active_staker, disable_rewards: true)` at block N, the slot for block N is permanently consumed. Any subsequent legitimate call at block N reverts with `REWARDS_ALREADY_UPDATED`. The block's rewards are never distributed and cannot be recovered.

The only prerequisite the attacker must satisfy is passing a currently active staker address with non-zero balance (checked before the `disable_rewards` branch): [4](#0-3) 

Any active staker's address is publicly observable on-chain, so this is trivially satisfied.

---

### Impact Explanation
Each block in the consensus rewards phase carries a fixed `block_rewards` amount computed once per epoch by `calculate_block_rewards` / `update_current_epoch_block_rewards`. [5](#0-4) 

When a block's slot is consumed with `disable_rewards: true`, those rewards are never added to any staker's `unclaimed_rewards_own` or forwarded to any delegation pool. They are permanently lost — not deferred, not accumulated. This constitutes **permanent freezing of unclaimed yield** for every staker and delegator in the protocol.

An attacker who repeats this every block eliminates all consensus-phase staking rewards indefinitely.

---

### Likelihood Explanation
- The function is unconditionally public; no role, no signature, no stake required.
- Active staker addresses are emitted as `NewStaker` events and are trivially enumerable.
- The attacker does not need to front-run; they simply need to be the first transaction in any block, which is achievable by submitting a transaction at the start of each block.
- There is no economic cost beyond gas fees, and no profit motive is required — pure griefing suffices.

---

### Recommendation
Restrict who may call `update_rewards` with `disable_rewards: true`. Options include:

1. **Require caller to be the staker or their reward address** when `disable_rewards: true` (analogous to the `claim_rewards` guard at line 417–420).
2. **Remove `disable_rewards` from the public ABI** and handle the pre-consensus / migration skip internally.
3. **Add a dedicated role check** (e.g., `only_staker_or_operator`) before allowing `disable_rewards: true` to consume a block slot.

---

### Proof of Concept

```
Block N:
  Attacker → Staking.update_rewards(alice_staker_address, disable_rewards=true)
    → general_prerequisites() passes (not paused, caller ≠ 0)
    → current_block_number (N) > last_reward_block → passes
    → staker_total_strk_balance is non-zero → passes
    → last_reward_block.write(N)          ← slot consumed
    → disable_rewards == true → return    ← no rewards distributed

  Legitimate caller → Staking.update_rewards(alice_staker_address, disable_rewards=false)
    → current_block_number (N) > last_reward_block (N) → FAILS: REWARDS_ALREADY_UPDATED

Block N rewards: permanently lost for all stakers and delegators.
Repeat every block → protocol yields zero rewards indefinitely.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

        fn calculate_block_rewards(
            ref self: ContractState,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) -> (Amount, Amount) {
            if curr_epoch > self.last_calculated_epoch.read() {
                self.last_calculated_epoch.write(curr_epoch);
                let block_rewards = reward_supplier_dispatcher.update_current_epoch_block_rewards();
                self.block_rewards.write(block_rewards);
                block_rewards
            } else {
                self.block_rewards.read()
            }
        }
