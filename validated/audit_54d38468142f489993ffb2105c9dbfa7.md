### Title
Unbounded `stakers` Vec Iteration in `get_stakers` Enables Unbounded Gas Consumption — (File: src/staking/staking.cairo)

### Summary
The `get_stakers` function iterates over the entire `stakers` Vec on every call. This Vec grows monotonically — every `stake()` call appends to it, and the contract explicitly documents that stakers are **never removed** when they unstake. An unprivileged actor can inflate this Vec by cycling many staker addresses through stake/unstake, making `get_stakers` progressively more expensive and eventually causing gas exhaustion.

### Finding Description
In `src/staking/staking.cairo`, the `stakers` storage field is declared as a `Vec<ContractAddress>`: [1](#0-0) 

Every `stake()` call unconditionally appends the caller to this Vec: [2](#0-1) 

`get_stakers` then iterates over the **full range** of the Vec — including all historical stakers who have long since unstaked: [3](#0-2) 

For every entry in the Vec (active or not), the function performs multiple storage reads: `is_staker_active`, `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, and `get_peer_id_at_epoch`. The gas cost scales linearly with the total number of addresses ever staked, not just the current active set.

This is structurally identical to the reported bug: an array that only grows, iterated in full on every invocation.

### Impact Explanation
`get_stakers` is a public function on `IStakingConsensus`. Any on-chain caller (e.g., a future consensus contract, an integrator, or the attestation contract) that invokes it will pay gas proportional to the lifetime size of the `stakers` Vec. As the Vec grows large enough, the call will exceed the block gas limit, causing the transaction to revert. This constitutes **unbounded gas consumption** and potential griefing of any on-chain consumer of `get_stakers`.

### Likelihood Explanation
The Vec grows naturally over the protocol's lifetime as stakers join and leave. An adversary can accelerate this by registering many distinct staker addresses (each meeting `min_stake`), waiting out the `exit_wait_window`, and unstaking — recovering their principal. The only sustained cost is the opportunity cost of locked capital during the exit window. No privileged role is required; `stake()` is open to any address.

### Recommendation
Replace the `Vec<ContractAddress>` with a pattern that supports O(1) active-staker lookup and bounded iteration:
- Maintain a separate `active_stakers: Map<ContractAddress, bool>` and a compact active-staker list that is pruned on `unstake_action`.
- Or swap-and-pop the unstaking address out of the Vec at unstake time to keep the Vec bounded to the live set.

The analog fix from the report applies directly: replace the unbounded array with a mapping (`Map<ContractAddress, bool>`) so membership checks and iteration are O(active stakers), not O(all-time stakers).

### Proof of Concept
1. Attacker controls N addresses, each funded with `min_stake` STRK.
2. Each address calls `stake()` → appended to `self.stakers` Vec.
3. After `exit_wait_window`, each address calls `unstake_intent` + `unstake_action` → principal recovered, but Vec entry remains permanently.
4. `self.stakers` now contains N dead entries.
5. Any subsequent call to `get_stakers` iterates all N entries, performing ~5 storage reads each.
6. At sufficient N, the call exhausts the transaction gas budget, reverting any on-chain consumer. [4](#0-3) [5](#0-4)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
