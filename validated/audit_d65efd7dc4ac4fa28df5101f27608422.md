### Title
Single Malicious Participant Can Abort Entire Triple Generation via Crafted dlogeq Proof Triggering `IdentityElement` Error — (`src/ecdsa/ot_based_ecdsa/triples/generation.rs`)

---

### Summary

In `do_generation_many`, when verifying a remote participant's `dlogeq` proof, a crafted proof can cause `dlogeq::verify` to return `Err(ProtocolError::IdentityElement)`. This error propagates via `?` as a fatal error, aborting the entire triple generation future for all honest participants. The protocol has no mechanism to treat this as a per-participant proof failure and continue.

---

### Finding Description

**Exact call site** — `generation.rs` lines 483–491:

```rust
if !dlogeq::verify(
    &mut transcript.fork(b"dlogeq0", &from.bytes()),
    statement,
    their_phi_proof,
)? {                          // <-- ? propagates Err(IdentityElement) fatally
    return Err(ProtocolError::AssertionFailed(...));
}
``` [1](#0-0) 

The `?` operator propagates any `Err` variant from `dlogeq::verify` directly. `Err(ProtocolError::AssertionFailed)` is only returned when `verify` returns `Ok(false)`.

**How `Err(IdentityElement)` is triggered** — inside `dlogeq::verify`:

```rust
let (phi0, phi1) = statement.phi(&proof.s.0);
let big_k0 = phi0 - *statement.public0 * proof.e.0;
let big_k1 = phi1 - *statement.public1 * proof.e.0;
let enc = encode_two_points::<C>(&big_k0, &big_k1)?;
``` [2](#0-1) 

`encode_two_points` calls `C::Group::serialize(point_1).map_err(|_| ProtocolError::IdentityElement)?`. If `big_k0` is the identity element (point at infinity), secp256k1 serialization fails and `Err(IdentityElement)` is returned. [3](#0-2) 

**The statement constructed by the honest verifier** (lines 477–481):

```rust
let statement = dlogeq::Statement::<C> {
    public0: &big_e_j_zero.index(from)?.value(),   // = G * a_j  (from's committed value)
    generator1: &big_f.eval_at_zero()?.value(),
    public1: &big_c_j,                              // from's claimed big_c_j
};
``` [4](#0-3) 

`public0 = G * a_j` where `a_j = e_j(0)` is the malicious participant's own polynomial constant term — a value they chose and know.

**The exploit**: The malicious participant picks any scalar `e_val` and sets `s = a_j * e_val` in their proof. The honest verifier then computes:

```
phi0  = G * s         = G * (a_j * e_val)
big_k0 = phi0 - public0 * e_val
       = G * a_j * e_val - G * a_j * e_val
       = identity
```

`encode_two_points` fails with `Err(IdentityElement)`, which propagates via `?` through `dlogeq::verify` and then via `?` at the call site in `do_generation_many`, terminating the entire async future.

The same pattern exists for `dlog::verify` at lines 368–376 and 381–389, where `big_k = G * s - public * e` can similarly be forced to the identity. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

A single malicious participant aborts the entire `do_generation_many` execution for every honest participant. Triple generation is a prerequisite for OT-based ECDSA signing. The malicious participant can repeat this on every attempt, permanently blocking triple generation as long as they remain in the participant set.

---

### Likelihood Explanation

The attack requires only that the malicious participant know their own polynomial constant term `a_j`, which they chose themselves. No cryptographic assumptions need to be broken. The crafted proof is trivially constructable: pick any `e_val`, set `s = a_j * e_val`. The attack is deterministic and repeatable.

---

### Recommendation

In `do_generation_many`, catch `Err(ProtocolError::IdentityElement)` (and `Err(ProtocolError::PointSerialization)`) from `dlogeq::verify` and `dlog::verify` and treat them as proof verification failures (equivalent to `Ok(false)`), returning `Err(ProtocolError::AssertionFailed(...))` identifying the offending participant rather than propagating the error fatally. Alternatively, modify `dlogeq::verify` and `dlog::verify` to return `Ok(false)` instead of `Err(IdentityElement)` when `big_k` computes to the identity, since that condition is itself evidence of an invalid proof.

---

### Proof of Concept

```rust
// Malicious participant's proof construction:
// a_j = e_j(0) — their own polynomial constant term (known to them)
// Choose e_val = Scalar::ONE
// Set s = a_j * e_val = a_j
// Send Proof { e: Scalar::ONE, s: a_j }
//
// Honest verifier computes:
//   phi0   = G * s = G * a_j
//   big_k0 = phi0 - public0 * e_val
//          = G * a_j - (G * a_j) * 1
//          = identity
// => encode_two_points returns Err(IdentityElement)
// => dlogeq::verify propagates Err(IdentityElement)
// => do_generation_many ? propagates Err(IdentityElement)
// => all honest participants' futures terminate with error
```
