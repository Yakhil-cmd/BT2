### Title
Missing Access Control on `update_rewards` Allows Any Caller to Redirect or Permanently Block Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Because a single global `last_reward_block` lock allows only one successful call per block, any unprivileged address can call `update_rewards` first in a block — either redirecting block rewards to an attacker-controlled staker, or permanently suppressing all reward distribution by passing `disable_rewards: true`.

### Finding Description

**Specification vs. Implementation mismatch**

The spec at `docs/spec.md` line 1644–1645 states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation at `src/staking/staking.cairo` lines 1449–1507 performs no such check. The only guards are a pause check via `general_prerequisites()`, a per-block lock on `last_reward_block`, and staker validity assertions: [2](#0-1) 

**The global per-block lock**

`last_reward_block` is a single contract-wide storage variable. The first call in any block writes the current block number to it; all subsequent calls in the same block revert with `REWARDS_ALREADY_UPDATED`: [3](#0-2) 

**The `disable_rewards` griefing path**

The caller supplies the `disable_rewards` flag. When `true`, the function writes `last_reward_block` (consuming the per-block slot) and then returns immediately without distributing any rewards: [4](#0-3) 

**Two concrete attack paths**

1. **Reward redirection (theft of yield):** An attacker who controls a staker calls `update_rewards(attacker_staker, disable_rewards: false)` before the legitimate sequencer call. The attacker's staker receives the block rewards; the intended attesting staker receives nothing for that block.

2. **Permanent reward freeze (griefing):** An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` every block. `last_reward_block` is updated each time, but no rewards are ever distributed. All stakers are permanently denied consensus-era block rewards.

### Impact Explanation

- **Theft of unclaimed yield (High):** An attacker-controlled staker can capture block rewards that should go to the legitimately attesting staker. Over many blocks this constitutes direct, ongoing theft of yield from honest stakers.
- **Permanent freezing of unclaimed yield (High):** The `disable_rewards: true` griefing path requires no profit motive and can be sustained indefinitely. Every staker in the protocol is denied all block rewards for as long as the attacker continues the attack.

### Likelihood Explanation

The entry point is fully public — no tokens, no stake, no role is required to call `update_rewards`. The only cost is the gas for one Starknet transaction per block. The attack is mechanically simple and can be automated. Likelihood is high once the consensus rewards epoch is active.

### Recommendation

Enforce the access control stated in the specification. Add a sequencer-address check at the top of `update_rewards`, analogous to the pattern used in `update_rewards_from_attestation_contract` which correctly restricts its caller to the attestation contract:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   assert!(
+       get_caller_address() == self.sequencer_address.read(),
+       "{}",
+       Error::CALLER_IS_NOT_SEQUENCER,
+   );
    ...
}
```

Alternatively, if the sequencer address is not stored on-chain, use Starknet's `get_execution_info` syscall to verify the transaction originates from the expected system account.

### Proof of Concept

**Attack path 1 — reward redirection:**
1. Consensus rewards epoch is active.
2. Attacker observes that staker `S_honest` is the attesting staker for block `N`.
3. Attacker submits `update_rewards(S_attacker, disable_rewards: false)` targeting block `N`.
4. If the attacker's transaction is included before the sequencer's system call, `last_reward_block` is set to `N` and `S_attacker` receives the block rewards.
5. The sequencer's subsequent call for `S_honest` reverts with `REWARDS_ALREADY_UPDATED`.
6. `S_honest` receives zero rewards for block `N`.

**Attack path 2 — permanent freeze:**
1. Attacker runs a bot that, for every new block `N`, submits `update_rewards(any_valid_staker, disable_rewards: true)`.
2. Each call succeeds (staker is valid, block is new), writes `last_reward_block = N`, and returns without distributing rewards.
3. The sequencer's legitimate call for the same block reverts with `REWARDS_ALREADY_UPDATED`.
4. No staker ever accumulates `unclaimed_rewards_own` during the consensus rewards era.
5. All yield is permanently frozen at zero. [5](#0-4) [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
