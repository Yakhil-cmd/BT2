### Title
Unrestricted `update_rewards` Allows Any Caller to Freeze or Steal Consensus Block Rewards - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in `StakingRewardsManagerImpl` carries no caller validation. Any address can invoke it with `disable_rewards: true` to permanently block reward distribution for a block (griefing), or any registered staker can invoke it with `disable_rewards: false` to capture consensus block rewards they did not earn (yield theft).

### Finding Description
`update_rewards` is the entry point for distributing per-block consensus rewards to a staker. It is intended to be called by the block proposer, but the implementation contains no check that the caller is the block proposer or even the staker being rewarded. [1](#0-0) 

The function signature accepts an arbitrary `staker_address` and a caller-controlled `disable_rewards` flag: [2](#0-1) 

The only guard against repeated calls is a single global `last_reward_block` storage variable: [3](#0-2) [4](#0-3) 

Because `last_reward_block` is global (not per-staker), exactly one call is accepted per block. Whoever calls first wins.

**Attack vector 1 – Griefing / permanent reward freeze:**
Any address (no stake required) calls `update_rewards(any_valid_staker_address, disable_rewards: true)`. The function validates the staker, writes `last_reward_block = current_block`, then returns early without distributing rewards. The legitimate block proposer's subsequent call reverts with `REWARDS_ALREADY_UPDATED`. The block's rewards are permanently lost.

**Attack vector 2 – Yield theft:**
A registered staker calls `update_rewards(attacker_staker_address, disable_rewards: false)` before the legitimate block proposer. The function distributes block rewards proportional to the attacker's stake to the attacker's `unclaimed_rewards_own`, and the legitimate proposer's call reverts. The attacker accumulates rewards for blocks they never proposed. [5](#0-4) 

Compare with the attestation path, which correctly enforces that only the registered operational address of a staker may trigger reward updates: [6](#0-5) 

`update_rewards` has no equivalent guard.

### Impact Explanation
- **Griefing / permanent freeze of unclaimed yield (High):** A single transaction per block permanently destroys that block's consensus rewards for all stakers. Sustained over time this drains the entire consensus reward stream.
- **Theft of unclaimed yield (High):** A staker with any non-zero balance can front-run every block proposer and accumulate rewards they did not earn, directly stealing yield from legitimate proposers.

Both impacts fall within the allowed scope: *"High: Theft of unclaimed yield"* and *"High: Permanent freezing of unclaimed yield"*.

### Likelihood Explanation
Starknet currently uses a centralized sequencer that orders transactions within a block. An attacker who submits a `update_rewards` transaction at the very start of each block (e.g., as the first transaction in the block) can reliably win the race. No privileged access, leaked key, or external dependency is required — only a valid staker address (publicly readable on-chain) and the ability to submit transactions.

### Recommendation
Add a caller check analogous to the attestation flow: require that `get_caller_address()` maps to the `staker_address` via the `operational_address_to_staker_address` mapping, or introduce a dedicated role/flag that only the consensus layer can set. At minimum, verify that the caller's operational address corresponds to the `staker_address` argument before processing rewards.

### Proof of Concept

**Griefing (no stake needed):**
```
// Attacker reads any active staker address from on-chain events.
let victim_staker: ContractAddress = <any active staker>;

// At the start of every block, attacker submits:
staking_rewards_dispatcher.update_rewards(
    staker_address: victim_staker,
    disable_rewards: true,   // no rewards distributed
);
// last_reward_block is now set; legitimate proposer's call reverts.
```

**Yield theft (attacker must be a staker):**
```
// Attacker stakes the minimum amount and registers as a staker.
staking_dispatcher.stake(reward_address: attacker, operational_address: attacker_op, amount: min_stake);

// At the start of every block, attacker submits:
staking_rewards_dispatcher.update_rewards(
    staker_address: attacker_staker_address,
    disable_rewards: false,  // rewards flow to attacker
);
// Attacker's unclaimed_rewards_own increases each block.
// Legitimate block proposer's call reverts with REWARDS_ALREADY_UPDATED.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
