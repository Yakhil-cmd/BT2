### Title
Staker Can Retroactively Increase Commission Mid-Epoch to Steal Delegators' Unclaimed Yield — (`src/staking/staking.cairo`)

---

### Summary

In V2 (pre-consensus) mode, a staker can use the commission commitment mechanism to increase their commission mid-epoch and then attest, triggering reward calculation with the new higher commission. Because `update_rewards_from_attestation_contract` calculates the **entire epoch's rewards** using the commission value in storage at attestation time, the staker retroactively captures yield that delegators had already earned during the epoch.

---

### Finding Description

`set_commission` immediately writes the new commission to storage without triggering any reward settlement for the current epoch: [1](#0-0) 

With an active commission commitment, the staker is permitted to **increase** commission up to `max_commission`: [2](#0-1) 

The commitment itself only requires `max_commission >= current_commission`, so a staker at 0% commission can set `max_commission = 10000`: [3](#0-2) 

In V2 (pre-consensus) mode, `update_rewards_from_attestation_contract` is called once per epoch when the staker attests. It reads the commission from storage at that moment and applies it to the **entire epoch's rewards**: [4](#0-3) 

Inside `calculate_staker_pools_rewards`, the commission is read live from storage: [5](#0-4) 

Because there is no reward settlement checkpoint before the commission change, the new commission applies retroactively to all rewards accrued since the epoch began.

The `ALPHA` constant and the reward split formula are: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

**High — Theft of unclaimed yield from delegators.**

A staker can capture up to 100% of the pool's epoch rewards by increasing commission to 10000 (100%) just before attestation. Delegators who staked for the entire epoch expecting 0% commission receive zero rewards. The stolen yield flows to the staker's `unclaimed_rewards_own`.

---

### Likelihood Explanation

**Medium.** The staker must first call `set_commission_commitment` (a public on-chain action), then call `set_commission` to increase it, then attest — all within the same epoch. Delegators can observe the commitment on-chain but cannot exit before the attestation fires within the same epoch because the `exit_wait_window` is at minimum one week: [8](#0-7) 

The attack is fully self-contained and requires no external coordination.

---

### Recommendation

Apply commission changes with a K-epoch delay (analogous to how balance changes and token enable/disable are handled), or force a reward settlement checkpoint before writing the new commission to storage. This ensures delegators' accrued yield is locked in at the old commission rate before any increase takes effect.

---

### Proof of Concept

1. Staker stakes with `commission = 0`. Delegators enter the pool expecting 0% commission.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
3. Near the end of the epoch (inside the attestation window), staker calls `set_commission(commission: 10000)`. Storage is updated immediately; no reward settlement occurs.
4. Staker calls `attest` on the attestation contract, which calls `update_rewards_from_attestation_contract` on the staking contract.
5. The function reads `commission = 10000` from storage and applies it to the full epoch's pool rewards via `split_rewards_with_commission`.
6. `commission_rewards = pool_rewards_including_commission * 10000 / 10000 = pool_rewards_including_commission`.
7. `pool_rewards = 0`. Delegators receive zero yield for the epoch. The staker captures all pool rewards as commission.

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
