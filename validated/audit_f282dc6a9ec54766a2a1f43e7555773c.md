### Title
EIP-7623 Floor Gas Not Enforced in Fee Accounting — Users Underpay Relayers on Calldata-Heavy Transactions - (`engine/src/engine.rs`)

---

### Summary

The `submit_with_alt_modexp` function computes `floor_gas` (EIP-7623) and validates that `gas_limit >= max(intrinsic_gas, floor_gas)`, but never enforces the floor gas minimum when computing the actual fee charged to the user. The `refund_unused_gas` call uses the raw SputnikVM `gas_used` value, which can be far below `floor_gas`. The difference is refunded to the user instead of being retained as fee revenue for the relayer, allowing any user to systematically underpay for calldata-heavy transactions.

---

### Finding Description

In `submit_with_alt_modexp` (`engine/src/engine.rs`), the transaction processing flow is:

1. `floor_gas` is computed from the transaction's calldata per EIP-7623.
2. The check `gas_limit >= max(intrinsic_gas, floor_gas)` is enforced — the user must declare enough gas.
3. `charge_gas` prepays `gas_limit * effective_gas_price` from the sender.
4. The EVM executes and returns `gas_used` from SputnikVM.
5. `refund_unused_gas` is called with the raw `gas_used`.

```rust
// engine/src/engine.rs:1073-1075
let floor_gas = transaction
    .floor_gas(CONFIG)
    .map_err(|_| EngineErrorKind::GasOverflow)?;

// engine/src/engine.rs:1077-1080
// Check that the max value of intrinsic gas and floor gas is covered by the transaction
// gas limit, EIP-7623 https://eips.ethereum.org/EIPS/eip-7623
if transaction.gas_limit < core::cmp::max(intrinsic_gas, floor_gas).into() {
    return Err(EngineErrorKind::IntrinsicGasNotMet.into());
}
```

After execution, `floor_gas` is never used again:

```rust
// engine/src/engine.rs:1143-1155
let gas_used = match &result {
    Ok(submit_result) => submit_result.gas_used,
    Err(engine_err) => engine_err.gas_used,
};

refund_unused_gas(
    &mut io,
    &sender,
    gas_used,          // ← raw SputnikVM value, floor_gas not applied
    &prepaid_amount,
    &relayer_address,
    fixed_gas,
)
```

Inside `refund_unused_gas`, the relayer reward and sender refund are both computed from `gas_used`:

```rust
// engine/src/engine.rs:1274-1291
let gas_to_wei = |price: U256| {
    fixed_gas
        .map_or_else(|| gas_used.into(), EthGas::as_u256)
        .checked_mul(price)
        ...
};
let spent_amount = gas_to_wei(gas_result.effective_gas_price)?;
let reward_amount = gas_to_wei(gas_result.priority_fee_per_gas)?;
let refund = gas_result.prepaid_amount.checked_sub(spent_amount)...;
```

EIP-7623 requires that when `gas_used < floor_gas`, the transaction must consume `floor_gas` gas. The `floor_gas` formula is:

```rust
// engine-transactions/src/lib.rs:238-247
let base_gas = config.gas_transaction_call;
let tokens_in_calldata = num_non_zero_bytes.checked_mul(4)
    .and_then(|gas| gas.checked_add(num_zero_bytes))...;
tokens_in_calldata
    .checked_mul(config.total_cost_floor_per_token)
    .and_then(|gas| gas.checked_add(base_gas))...
```

For any transaction with calldata, `floor_gas > intrinsic_gas` always holds because `tokens_in_calldata * 10 > non_zero_bytes * 16 + zero_bytes * 4` simplifies to `24 * non_zero + 6 * zero > 0`. This means for every calldata transaction, there is a gap between `intrinsic_gas` and `floor_gas` that the user should pay but does not.

---

### Impact Explanation

The relayer receives `gas_used * priority_fee_per_gas` instead of the correct `floor_gas * priority_fee_per_gas`. The user is refunded `(floor_gas - gas_used) * effective_gas_price` that should have been burned/retained. This is a direct theft of relayer fee revenue by the transaction sender.

**Impact: High — Theft of unclaimed yield (relayer fee revenue).**

---

### Likelihood Explanation

This is trivially exploitable by any EVM user. Any transaction with calldata where execution is cheap (e.g., calling a contract that does nothing, or a simple token transfer with extra calldata padding) will trigger the underpayment. The attacker controls both the calldata size (which sets `floor_gas` high) and the execution complexity (which keeps `gas_used` low). No special privileges are required.

---

### Recommendation

Before calling `refund_unused_gas`, enforce the floor gas minimum:

```rust
let effective_gas_used = core::cmp::max(gas_used, floor_gas);
refund_unused_gas(
    &mut io,
    &sender,
    effective_gas_used,
    &prepaid_amount,
    &relayer_address,
    fixed_gas,
)
```

This ensures the user is charged at least `floor_gas * effective_gas_price`, consistent with EIP-7623 semantics.

---

### Proof of Concept

- User submits a transaction with 1,000 non-zero calldata bytes to a contract that does nothing.
- `intrinsic_gas` = 21,000 + 1,000 × 16 = 37,000
- `floor_gas` = 21,000 + (1,000 × 4) × 10 = 61,000
- `gas_limit` must be ≥ 61,000 (validation passes).
- User prepays `61,000 × price`.
- EVM executes; `gas_used` ≈ 37,000 (intrinsic only, no meaningful execution).
- `refund_unused_gas` refunds `(61,000 − 37,000) × price = 24,000 × price` to the user.
- Relayer receives `37,000 × priority_fee` instead of `61,000 × priority_fee`.
- Relayer loses `24,000 × priority_fee` per transaction.
- Repeating this with maximum calldata amplifies the loss proportionally. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    #[allow(clippy::naive_bytecount)]
    pub fn floor_gas(&self, config: &aurora_evm::Config) -> Result<u64, Error> {
        if config.has_floor_gas {
            let num_zero_bytes = u64::try_from(self.data.iter().filter(|b| **b == 0).count())
                .map_err(|_e| Error::IntegerConversion)?;
            let data_len = u64::try_from(self.data.len()).map_err(|_e| Error::IntegerConversion)?;
            let num_non_zero_bytes = data_len
                .checked_sub(num_zero_bytes)
                .ok_or(Error::GasOverflow)?;

            let base_gas = config.gas_transaction_call;
            let tokens_in_calldata = num_non_zero_bytes
                .checked_mul(4)
                .and_then(|gas| gas.checked_add(num_zero_bytes))
                .ok_or(Error::GasOverflow)?;

            tokens_in_calldata
                .checked_mul(config.total_cost_floor_per_token)
                .and_then(|gas| gas.checked_add(base_gas))
                .ok_or(Error::GasOverflow)
        } else {
            Ok(0)
        }
    }
