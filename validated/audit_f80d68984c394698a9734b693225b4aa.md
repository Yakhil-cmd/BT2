### Title
`resume_precompiles` Requires Contract Running State, Blocking Exit Precompile Recovery When Contract Is Paused - (File: `engine/src/contract_methods/admin.rs`)

### Summary

`resume_precompiles` enforces `require_running`, meaning it can only be called when the Aurora Engine contract is **not** paused. If exit precompiles (`ExitToNear`, `ExitToEthereum`) are paused while the contract is also paused, the owner cannot selectively restore exit functionality without first resuming the entire contract — re-enabling all operations including potentially unsafe ones.

### Finding Description

`resume_precompiles` in `engine/src/contract_methods/admin.rs` unconditionally calls `require_running` before allowing the owner to resume any precompile:

```rust
pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← blocks when contract is paused
        require_owner_only(&state, &predecessor_account_id)?;
        ...
        pauser.resume_precompiles(flags);
    })
}
``` [1](#0-0) 

`require_running` returns `ERR_PAUSED` whenever `state.is_paused == true`: [2](#0-1) 

`pause_contract` sets `is_paused = true` and is callable by the owner independently of the precompile pause state: [3](#0-2) 

The `ExitToNear` and `ExitToEthereum` precompiles are the only user-facing mechanisms to withdraw funds from Aurora. When a precompile is paused, the EVM executor returns a fatal `ERR_PAUSED` error for any call to that precompile address: [4](#0-3) 

These are the only two precompiles that can be paused, as defined by `PrecompileFlags`: [5](#0-4) 

### Impact Explanation

**Temporary freezing of funds (High).**

When both the contract and exit precompiles are paused simultaneously, users cannot withdraw their ETH or ERC-20 tokens from Aurora. The owner cannot call `resume_precompiles` to restore exit functionality because that function requires the contract to be running. The owner is forced to call `resume_contract` first — re-enabling all contract operations including `submit`, `deploy_code`, and `call` — before being able to selectively restore exit precompile access. This means there is no way to restore only exit functionality while keeping the rest of the contract paused, which is the exact operational intent of having two independent pause mechanisms.

### Likelihood Explanation

Both pause mechanisms exist for independent security reasons: exit precompiles may be paused due to a bridge exploit, while the contract may be paused due to an EVM-level vulnerability. Both being active simultaneously is a realistic and expected scenario. In that scenario, the owner is forced into a binary choice: keep everything paused (funds frozen) or resume everything (potentially re-exposing the EVM vulnerability) just to restore exit access.

### Recommendation

Remove the `require_running` guard from `resume_precompiles`, or add an explicit exception allowing the owner to call `resume_precompiles` even when the contract is paused. The analogous fix to the Reserve Protocol mitigation would be:

```rust
pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        // Remove require_running — owner should be able to resume exit precompiles
        // even when the contract is paused, to restore user exit access independently.
        require_owner_only(&state, &env.predecessor_account_id())?;
        ...
    })
}
```

### Proof of Concept

1. Contract is running; exit precompiles are unpaused (normal state).
2. Owner (or authorized account) calls `pause_precompiles` with `EXIT_TO_NEAR | EXIT_TO_ETHEREUM` mask. Exit precompiles are now paused.
3. Owner calls `pause_contract`. Contract is now paused.
4. Owner attempts to call `resume_precompiles` to restore exit access for users.
5. `resume_precompiles` calls `require_running(&state)?` → returns `Err(ERR_PAUSED)`. Call fails.
6. Users cannot exit funds via `ExitToNear` or `ExitToEthereum` (precompiles paused → `ERR_PAUSED`).
7. Owner cannot restore exit access without first calling `resume_contract`, which re-enables all EVM operations (`submit`, `deploy_code`, `call`), defeating the purpose of the contract-level pause. [1](#0-0) [6](#0-5) [7](#0-6)

#[named]
pub fn pause_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        require_running(&state::get_state(&io)?)?;
        let authorizer: EngineAuthorizer = engine::get_authorizer(&io);

        if !authorizer.is_authorized(&env.predecessor_account_id()) {
            return Err(b"ERR_UNAUTHORIZED".into());

        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.pause_precompiles(flags);
        Ok(())
    })
}

#[named]
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
