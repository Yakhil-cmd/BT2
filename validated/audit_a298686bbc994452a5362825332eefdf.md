### Title
Missing Caller Authorization in `update_rewards` Allows Anyone to Permanently Freeze Staker Yield ŌĆö (`File: src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract accepts a caller-controlled `disable_rewards: bool` parameter and enforces no access control beyond the generic pause check. Any unprivileged address can call `update_rewards(staker_address, disable_rewards: true)` to advance the global `last_reward_block` while skipping reward calculation, permanently destroying the yield for that block for all stakers.

---

### Finding Description

`update_rewards` is part of `IStakingRewardsManager` and is the per-block reward distribution entry point for the consensus rewards phase. Its implementation reads: [1](#0-0) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global write, always executed

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // reward calculation skipped
    }
    ...
```

Two properties combine to create the vulnerability:

1. **`last_reward_block` is a single global storage slot** ŌĆö not per-staker. Writing it prevents any subsequent call to `update_rewards` for the same block number from succeeding. [2](#0-1) 

2. **`disable_rewards` is a raw caller-supplied boolean** ŌĆö there is no role check, no `assert_caller_is_attestation_contract`, and no other guard restricting who may pass `true`. [3](#0-2) 

The analog to the external report is exact: just as `partial_close_position` was missing the owner check that `close_amount` enforced, `update_rewards` is missing the caller restriction that `update_rewards_from_attestation_contract` correctly enforces: [4](#0-3) 

```cairo
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), ...);
    self.assert_caller_is_attestation_contract();   // ŌåÉ access control present here
    ...
```

---

### Impact Explanation

**High ŌĆö Permanent freezing of unclaimed yield.**

Each Starknet block, an attacker calls:

```
update_rewards(any_valid_staker_address, disable_rewards: true)
```

This:
- Writes `last_reward_block = current_block_number` (global, irreversible for that block).
- Returns early, skipping `_update_rewards` and the reward supplier call.
- Permanently destroys the block reward for every staker in the protocol for that block.

Because `last_reward_block` is global, a single call per block is sufficient to deny rewards to the entire protocol. The lost yield cannot be recovered retroactively. [5](#0-4) 

---

### Likelihood Explanation

**High.** The function is publicly callable (no role gate, no allowlist). The attacker needs only a funded Starknet account and the ability to submit one transaction per block. There is no economic cost beyond gas. The attack is trivially scriptable and can be sustained indefinitely.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the attestation contract, a designated rewards manager role, or the staker themselves), mirroring the pattern already used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_rewards_manager();   // add appropriate role check
    ...
```

Alternatively, derive `disable_rewards` from on-chain state rather than accepting it as a caller-supplied parameter.

---

### Proof of Concept

1. Attacker monitors the Starknet mempool for each new block.
2. For every block `B`, attacker submits:
   ```
   staking_contract.update_rewards(any_active_staker, disable_rewards=true)
   ```
3. The call passes `general_prerequisites()` (contract not paused).
4. `last_reward_block` is set to `B`; reward calculation is skipped.
5. Any legitimate call to `update_rewards` for block `B` now reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers permanently lose their yield for block `B`.
7. Repeating each block freezes all consensus-phase staking rewards indefinitely. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
