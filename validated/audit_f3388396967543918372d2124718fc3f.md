### Title
Commission Front-Running in `enter_delegation_pool` Allows Staker to Steal Delegator Yield - (File: src/pool/pool.cairo)

### Summary
A staker with an active `CommissionCommitment` can front-run a delegator's `enter_delegation_pool` (or `add_to_delegation_pool`) transaction by calling `set_commission` to raise the commission to the commitment's `max_commission` ceiling. The delegator enters the pool at a far higher commission rate than they observed, permanently redirecting the majority of their future yield to the staker.

### Finding Description
The `CommissionCommitment` mechanism allows a staker to set a `max_commission` ceiling and then freely raise or lower commission within that ceiling at any time with no delay. [1](#0-0) 

When a delegator reads the current commission and submits `enter_delegation_pool`, the staker can observe the pending transaction in the mempool and front-run it with a `set_commission` call that raises commission to `max_commission`. The delegator's transaction then executes against the new, higher commission with no slippage protection. [2](#0-1) 

Neither `enter_delegation_pool` nor `add_to_delegation_pool` accepts a `max_commission` parameter, so there is no on-chain guard the delegator can set. [3](#0-2) 

The commission is read lazily from the staking contract at reward-claim time, not locked at entry time, so the delegator is permanently bound to whatever commission the staker sets going forward. [4](#0-3) 

### Impact Explanation
**High — Theft of unclaimed yield.**

The staker captures the delegator's future reward share via the inflated commission. For example, a staker advertising 5% commission but holding a commitment with `max_commission = 9000` (90%) can redirect 85% of every future reward epoch from the delegator to themselves. The delegator's only recourse is to exit the pool, which requires waiting out the full `exit_wait_window` (default one week, up to twelve weeks), during which the inflated commission continues to drain yield. [5](#0-4) 

### Likelihood Explanation
**High.** The attack requires no special privilege beyond being a registered staker with a pool and an active `CommissionCommitment`. Starknet's public mempool makes pending transactions observable. The staker has a direct financial incentive (capturing delegator yield). The `CommissionCommitment` feature was designed to protect delegators by capping commission, but it simultaneously enables sudden upward changes within the cap, which is the root cause. Any staker who sets a high `max_commission` ceiling can exploit this against every new delegator. [6](#0-5) 

### Recommendation
Add a `max_commission: Commission` parameter to both `enter_delegation_pool` and `add_to_delegation_pool`. After reading the staker's current commission inside the function, assert:

```cairo
assert!(
    current_commission <= max_commission,
    "{}",
    Error::COMMISSION_EXCEEDS_MAX,
);
```

This mirrors the fix recommended in the external report (adding a `maxAmount` slippage guard) and gives delegators an on-chain guarantee that their transaction reverts rather than silently executing at an unexpected commission rate. [7](#0-6) 

### Proof of Concept

1. Staker calls `set_commission(500)` → commission = 5 % (500 / 10 000). [8](#0-7) 
2. Staker calls `set_commission_commitment(max_commission: 9000, expiration_epoch: current_epoch + 50)` → commitment ceiling = 90 %. [6](#0-5) 
3. Delegator observes 5 % commission and broadcasts `enter_delegation_pool(reward_address, 1_000_000_STRK)`.
4. Staker sees the pending transaction and broadcasts `set_commission(9000)` with higher gas priority. The `update_commission` path allows this because `9000 <= max_commission` and `9000 != 500`. [1](#0-0) 
5. Staker's `set_commission` is included first; commission is now 90 %.
6. Delegator's `enter_delegation_pool` executes. Funds are transferred and the pool member record is created with no commission check. [9](#0-8) 
7. Every subsequent reward epoch, 90 % of the delegator's share flows to the staker instead of the expected 5 %, constituting ongoing theft of unclaimed yield until the delegator exits after the full exit wait window.

            // Prerequisites and asserts.
            self.general_prerequisites();

                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn enter_delegation_pool(
            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.
            self
                .pool_member_info
                .write(pool_member, VInternalPoolMemberInfoTrait::new_latest(:reward_address));

            // Emit events.
            self
                .emit(
                    Events::NewPoolMember { pool_member, staker_address, reward_address, amount },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake: Zero::zero(), new_delegated_stake: amount,
                    },
                );
        }
