### Title
Anyone Can Trigger `exit_delegation_pool_action` on Behalf of a Pool Member, Blocking Pool Switching - (File: src/pool/pool.cairo)

### Summary

`Pool.exit_delegation_pool_action` accepts a `pool_member` address as a parameter but enforces no caller restriction. Any external address can call it once the exit window has elapsed, forcing the pool member's funds to be transferred out at an unintended time and permanently clearing the exit intent state that is required to execute a pool switch.

### Finding Description

`exit_delegation_pool_action` in `src/pool/pool.cairo` performs two critical state mutations: it transfers `unpool_amount` tokens to `pool_member` and clears both `unpool_time` and `unpool_amount` from the pool member's record. [1](#0-0) 

The function has no access control check — it does not assert that `get_caller_address() == pool_member` or any other authorized party. Compare this to every other state-mutating function in the same contract:

- `add_to_delegation_pool`: restricted to pool member or reward address [2](#0-1) 
- `claim_rewards`: restricted to pool member or reward address [3](#0-2) 
- `switch_delegation_pool`: restricted to pool member (uses `get_caller_address()`) [4](#0-3) 

The interface documentation for `exit_delegation_pool_action` lists no "Access control" section, confirming the omission is structural. [5](#0-4) 

After a pool member calls `exit_delegation_pool_intent`, they hold two mutually exclusive options until they act:

1. Call `exit_delegation_pool_action` — withdraw funds and leave the pool.
2. Call `switch_delegation_pool` — redirect the `unpool_amount` to a different staker's pool without going through a new exit window.

`switch_delegation_pool` requires `unpool_time.is_some()`: [6](#0-5) 

`exit_delegation_pool_action` sets `unpool_time = None` and `unpool_amount = 0`: [7](#0-6) 

An attacker who calls `exit_delegation_pool_action` after the exit window expires but before the pool member calls `switch_delegation_pool` atomically destroys the pool member's ability to switch pools. The pool member must now re-enter the delegation pool and wait through a full new exit window before they can switch — a delay measured in weeks (`DEFAULT_EXIT_WAIT_WINDOW = WEEK`). [8](#0-7) 

### Impact Explanation

The pool member's funds are sent to their own address (no theft), but their position management strategy is permanently disrupted: the ability to atomically switch delegation pools is destroyed. The pool member must re-enter and wait through a full new exit window. This is **griefing with no profit motive but concrete damage to users** — matching the allowed Medium impact: *"Griefing with no profit motive but damage to users or protocol."*

### Likelihood Explanation

The entry path requires no privilege: any externally owned account can call `Pool.exit_delegation_pool_action(pool_member)` on any pool contract once `Time::now() >= unpool_time`. The attacker needs only to monitor `PoolMemberExitIntent` events on-chain and front-run or race the pool member's `switch_delegation_pool` call after the exit window expires. This is straightforward with standard mempool/event monitoring.

### Recommendation

Add a caller restriction to `exit_delegation_pool_action` so that only the pool member (or optionally their reward address) can trigger the action:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let caller = get_caller_address();
    let pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(
        caller == pool_member || caller == pool_member_info.reward_address,
        "{}",
        Error::CALLER_CANNOT_EXECUTE_EXIT_ACTION,
    );
    // ... rest of function
}
```

This mirrors the access control pattern already used in `claim_rewards` and `add_to_delegation_pool`.

### Proof of Concept

1. Alice (pool member) calls `exit_delegation_pool_intent(amount)` on her pool contract. The exit window is set to `Time::now() + WEEK`.
2. Alice intends to call `switch_delegation_pool(to_staker, to_pool, amount)` after the window expires to move to a better-performing staker without waiting for a new exit window.
3. Bob (attacker) monitors `PoolMemberExitIntent` events. After the exit window expires, Bob calls `Pool.exit_delegation_pool_action(alice_address)` before Alice can act.
4. The function executes: `unpool_amount` is transferred to Alice, `unpool_time` is set to `None`, `unpool_amount` is set to `0`.
5. Alice attempts `switch_delegation_pool` — it panics at `assert!(pool_member_info.unpool_time.is_some(), ...)` because the intent state was cleared.
6. Alice must call `exit_delegation_pool_intent` again and wait another full week before she can switch pools.

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
