### Title
Commission Not Cached Per-Epoch Allows Staker to Retroactively Increase Commission Before Reward Settlement - (File: `src/staking/staking.cairo`)

---

### Summary

The commission used for pool reward calculation is read from live storage at the moment `update_rewards` is called, not cached at epoch start or at delegation time. A staker who holds an active `CommissionCommitment` can increase their commission to `max_commission` immediately before reward settlement, retroactively redirecting delegators' earned yield to themselves as commission. The codebase itself acknowledges this gap at `src/staking/staking.cairo:745–746`.

---

### Finding Description

**Root cause — commission is never snapshotted per epoch.**

In `calculate_staker_pools_rewards` the commission is fetched live from storage:

```
let commission = staker_pool_info.commission();   // line 1964
``` [1](#0-0) 

That value is immediately used to split every pool's rewards between delegators and the staker:

```
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [2](#0-1) 

No per-epoch snapshot of the commission is ever written. The commission that was in effect when delegators joined, or at the start of the epoch, is irrelevant; only the value present in storage at settlement time matters.

**The commission commitment feature enables a sudden increase.**

Without a commitment, `set_commission` enforces `commission < old_commission` — commission can only fall:

```
} else {
    assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
}
``` [3](#0-2) 

But when an active `CommissionCommitment` exists, the only constraints are `commission <= max_commission` and `commission != old_commission` — the commission may be raised to any value up to `max_commission` in a single transaction:

```
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission, ...
    );
    assert!(commission != old_commission, ...);
``` [4](#0-3) 

The write takes effect immediately:

```
staker_pool_info.commission.write(Option::Some(commission));
``` [5](#0-4) 

The protocol's own inline comment acknowledges the gap:

```
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
``` [6](#0-5) 

**Reward settlement path (V2 — attestation-based).**

In V2 the staker controls the exact moment rewards are settled: they call `attest`, which triggers `update_rewards_from_attestation_contract` on the staking contract, which calls `_update_rewards` → `calculate_staker_pools_rewards`. The staker can therefore:

1. Raise commission to `max_commission` in the same block.
2. Immediately call `attest` — all epoch rewards are calculated with the inflated commission.

**Reward settlement path (V3 — consensus rewards).**

`update_rewards` is callable by anyone, but the staker can front-run any external caller with a commission increase in the same block.

---

### Impact Explanation

Every STRK token that should have flowed to delegators as pool rewards is instead captured by the staker as commission. This is a direct, quantifiable theft of unclaimed yield. In the extreme case (`max_commission = 10000`, i.e. 100 %), delegators receive zero rewards for the entire epoch despite having staked throughout it.

Impact: **High — Theft of unclaimed yield.**

---

### Likelihood Explanation

The attack requires only that the staker previously called `set_commission_commitment` with a `max_commission` higher than the current commission. This is a permissionless, single-transaction setup. The staker can advertise a low commission to attract delegators, set a commitment with a high ceiling, and execute the attack at any epoch boundary. No privileged key, bridge access, or external dependency is needed.

Likelihood: **Medium** — requires deliberate malicious intent from the staker, but the on-chain mechanics are fully available to any staker.

---

### Recommendation

Apply the same K-epoch delay that governs staking-power changes to commission increases. Specifically, when `set_commission` raises the commission (only possible under an active commitment), the new value should not take effect until `current_epoch + K`, mirroring the balance-trace pattern already used elsewhere in the contract. Alternatively, snapshot the commission once per epoch at the start of reward calculation and use that snapshot throughout the epoch, so mid-epoch changes cannot affect already-accruing rewards.

---

### Proof of Concept

```
// 1. Staker stakes and opens a STRK pool with commission = 0.
staking_dispatcher.set_commission(commission: 0);
staking_dispatcher.set_open_for_delegation(token_address: STRK_TOKEN_ADDRESS);

// 2. Staker sets a commitment: max_commission = 10000 (100%), valid for 1 epoch.
let current_epoch = staking_dispatcher.get_current_epoch();
staking_dispatcher.set_commission_commitment(
    max_commission: 10000,
    expiration_epoch: current_epoch + 1,
);

// 3. Delegators join the pool, expecting 0% commission.
pool_dispatcher.enter_delegation_pool(reward_address: delegator_reward, amount: LARGE_AMOUNT);

// 4. Epoch progresses; delegators earn rewards.
advance_epoch_global();

// 5. Staker raises commission to 100% — takes effect immediately.
staking_dispatcher.set_commission(commission: 10000);

// 6. Staker attests (V2) or calls update_rewards (V3).
//    calculate_staker_pools_rewards reads commission = 10000.
//    split_rewards_with_commission returns (all_rewards, 0) for the pool.
attestation_dispatcher.attest(block_hash: ...);

// 7. Pool receives 0 rewards. Staker captures 100% as commission.
assert!(pool_dispatcher.pool_member_info_v1(pool_member: delegator).unclaimed_rewards == 0);
```

The `commission` read at line 1964 of `src/staking/staking.cairo` is `10000`; `split_rewards_with_commission` in `src/staking/utils.cairo` returns `(pool_rewards_including_commission, 0)` for the pool, and the delegators receive nothing. [7](#0-6) [8](#0-7)

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );
                total_commission_rewards += commission_rewards;
                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
