### Title
Pool Admin Can Set Per-Bin Additional Fees to `uint16.max` (6.5535%) With No Cap, Bypassing the Protocol Fee Cap System — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6` / `addFeeSellE6` directly to the pool with zero upper-bound validation. The analogous `setPoolAdminFees` enforces `maxAdminSpreadFeeE6` / `maxAdminNotionalFeeE8` caps. A pool admin can therefore set per-bin fees to `type(uint16).max` = 65 535 E6 (6.5535%) on any bin, causing every trader who swaps through that bin to pay an uncapped additional spread fee on top of the global spread fee.

### Finding Description

**Factory entry point — no cap check:**

`MetricOmmPoolFactory.setPoolBinAdditionalFees` only enforces `onlyPoolAdmin(pool)` and forwards the values verbatim:

```solidity

  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}

**Pool — only bin-range check, no fee cap:**

```solidity
// MetricOmmPool.sol lines 464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)

  s.addFeeSellE6 = addFeeSellE6;

}

**Contrast — `setPoolAdminFees` has explicit caps:**

The hard ceiling for admin spread fees is `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20%). [4](#0-3) 

**Swap math — per-bin fee is additive to global spread fee:**

```solidity
// MetricOmmPool.sol line 910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [5](#0-4) 

`baseFeeX64` is already derived from the oracle bid/ask spread plus the global `spreadFeeE6`. The per-bin fee is added on top, so the effective fee for a swap in that bin is `globalSpreadFee + addFeeBuyE6`. With `addFeeBuyE6 = 65535`, the per-bin component alone is 6.5535%, and the combined fee exceeds the 20% hard cap the protocol enforces everywhere else. [6](#0-5) 

### Impact Explanation

Every trader whose swap routes through the affected bin pays an extra 6.5535% fee (at `uint16.max`) on the input token. This fee is retained by the pool as LP balance (it is not separately tracked as a protocol or admin fee), directly reducing the trader's received output. The pool admin can apply this to the active bin (bin 0 or whichever bin the current price cursor sits in), ensuring every swap is affected. This constitutes direct loss of trader principal above Sherlock Medium thresholds.

### Likelihood Explanation

The pool admin is a semi-trusted role that the protocol explicitly constrains to operate "inside caps." The factory already enforces caps on global admin spread and notional fees. The absence of an equivalent cap on per-bin fees is an inconsistency that any pool admin can exploit immediately, with no timelock, no co-signer, and no protocol-owner involvement. The Smart Audit Pivots for this contest explicitly flag `setBinAdditionalFees` as a boundary to check.

### Recommendation

Add a cap check in `MetricOmmPoolFactory.setPoolBinAdditionalFees` before forwarding to the pool, mirroring the pattern in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, introduce a dedicated `maxAdminBinFeeE6` cap so per-bin fees can be governed independently.

### Proof of Concept

1. Deploy a pool via `MetricOmmPoolFactory.createPool` with default fee settings.
2. As pool admin, call `factory.setPoolBinAdditionalFees(pool, 0, 65535, 0)` — succeeds with no revert.
3. Read bin 0 state via `IExtsload(pool).extsload(PoolStateLibrary._calculateBinStateSlot(0))` and decode `addFeeBuyE6`; assert it equals `65535`.
4. Execute a `swap` buying token0 through bin 0 and compare the received output against the same swap with `addFeeBuyE6 = 0`; the output is reduced by ≈6.5535%, confirming direct principal loss to the trader.
