### Title
`validate_resource_bounds` rejects valid fee-paying transactions when fee is funded entirely by tip — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The stateless gateway validator checks whether a transaction has non-zero resource bounds by computing `max_possible_fee(Tip::ZERO)`. However, the blockifier's fee-enforcement decision uses `max_possible_fee(context.tip)` — the actual tip. A V3 transaction that sets all `max_price_per_unit` fields to zero but carries a non-zero `tip` and a non-zero `l2_gas.max_amount` will compute a zero fee under `Tip::ZERO` and be rejected with `ZeroResourceBounds`, even though the blockifier would enforce fees for it (because `l2_gas.max_amount × tip > 0`). This is the direct analog of the Venus M-01 bug: an early-return guard that ignores a boundary parameter (tip, analogous to alpha=1) causes a valid input to be incorrectly rejected.

### Finding Description

In `validate_resource_bounds`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs, lines 64-69
let resource_bounds = *tx.resource_bounds();
// The resource bounds should be positive even without the tip.
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
{
    return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
}
``` [1](#0-0) 

The guard hard-codes `Tip::ZERO`. The `max_possible_fee` implementation adds the tip only to the L2 gas term:

```rust
// crates/starknet_api/src/transaction/fields.rs, lines 398-413
l2_gas
    .max_amount
    .saturating_mul(l2_gas.max_price_per_unit.saturating_add(tip.into())),
``` [2](#0-1) 

So for a transaction with `l2_gas.max_price_per_unit = 0`, `l2_gas.max_amount = N > 0`, `tip = T > 0`, and all other bounds zero:

- `max_possible_fee(Tip::ZERO)` = `N × (0 + 0)` = **0** → rejected with `ZeroResourceBounds`
- `max_possible_fee(T)` = `N × (0 + T)` = `N × T > 0` → **fee enforced by blockifier**

The blockifier's `enforce_fee` uses the actual tip:

```rust
// crates/blockifier/src/transaction/objects.rs, lines 105-113
pub fn enforce_fee(&self) -> bool {
    match self {
        TransactionInfo::Current(context) => {
            // Assumes that the tip is enabled, as it is in the OS.
            context.resource_bounds.max_possible_fee(context.tip) > Fee(0)
        }
        TransactionInfo::Deprecated(context) => context.max_fee != Fee(0),
    }
}
``` [3](#0-2) 

The stateful validator's `run_validate_entry_point` derives `charge_fee` from `enforce_fee`, which also uses the real tip:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs, lines 308-312
let only_query = false;
let charge_fee = enforce_fee(executable_tx, only_query);
``` [4](#0-3) 

The invariant broken: the gateway's zero-fee guard and the blockifier's fee-enforcement decision use different tip values, creating a class of valid fee-paying transactions that are permanently rejected at the admission layer.

### Impact Explanation

A V3 `AllResources` transaction with `l2_gas.max_price_per_unit = 0`, `l2_gas.max_amount > 0`, and `tip > 0` is a structurally valid Starknet transaction that would pay fees (`l2_gas.max_amount × tip` STRK). The gateway rejects it with `ZeroResourceBounds` before it ever reaches the mempool or blockifier. This is a gateway admission decision that incorrectly rejects a valid transaction — matching the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The scenario requires `l2_gas.max_price_per_unit = 0` with a non-zero tip. This is an unusual but protocol-valid configuration (tip is a first-class signed field in V3 transactions). The bug is unconditional: it fires regardless of the `min_gas_price` config value because the `ZeroResourceBounds` check executes before the `MaxGasPriceTooLow` check. [5](#0-4) 

When `min_gas_price = 0` (a valid operator configuration), no subsequent check would have rejected the transaction, making the `ZeroResourceBounds` rejection the sole and incorrect gate.

### Recommendation

Replace `Tip::ZERO` with the transaction's actual tip when computing the zero-fee guard:

```rust
// Before (incorrect):
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {

// After (correct):
let tip = tx.tip(); // extract tip from the RpcTransaction
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(tip) == Fee(0) {
```

This mirrors the logic in `TransactionInfo::enforce_fee`, which already uses the actual tip, and eliminates the inconsistency between the gateway admission check and the blockifier fee-enforcement decision.

### Proof of Concept

```rust
// Construct a V3 invoke transaction with:
//   l2_gas.max_price_per_unit = 0
//   l2_gas.max_amount         = 1_000_000
//   tip                       = 1_000_000_000  (1 Gwei)
//   all other bounds          = 0

let resource_bounds = AllResourceBounds {
    l2_gas: ResourceBounds {
        max_amount: GasAmount(1_000_000),
        max_price_per_unit: GasPrice(0),   // zero price
    },
    ..Default::default()
};
let tip = Tip(1_000_000_000);

// max_possible_fee(Tip::ZERO) = 1_000_000 * (0 + 0) = 0  → ZeroResourceBounds error
// max_possible_fee(tip)       = 1_000_000 * (0 + 1_000_000_000) = 1e15 > 0 → fee enforced

// The stateless validator rejects this transaction:
let config = StatelessTransactionValidatorConfig {
    validate_resource_bounds: true,
    min_gas_price: 0,   // no price floor
    ..Default::default()
};
let validator = StatelessTransactionValidator { config };
let tx = rpc_invoke_v3_with(resource_bounds, tip);
assert_matches!(
    validator.validate(&tx),
    Err(StatelessTransactionValidatorError::ZeroResourceBounds { .. })
    // BUG: should be Ok(()) because max_possible_fee(tip) > 0
);

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    pub fn max_possible_fee(&self, tip: Tip) -> Fee {
        match self {
            ValidResourceBounds::L1Gas(l1_bounds) => {
                l1_bounds.max_amount.saturating_mul(l1_bounds.max_price_per_unit)
            }
            ValidResourceBounds::AllResources(AllResourceBounds {
                l1_gas,
                l2_gas,
                l1_data_gas,

                )
