### Title
Unbounded `stakers` Vector Growth via Stake/Unstake Cycling Causes `get_stakers` Unbounded Gas Consumption - (File: src/staking/staking.cairo)

### Summary
The `stakers` vector in the Staking contract is append-only: staker addresses are pushed on `stake()` but **never removed** on `unstake_action()`. The `get_stakers` function iterates over the entire vector on every call. An unprivileged attacker with only `min_stake` tokens can repeatedly stake with a fresh address, wait out the exit window, unstake (recovering their tokens), transfer to a new address, and repeat — permanently inflating the vector. Over time this makes `get_stakers` prohibitively expensive, breaking the consensus layer's ability to query the validator set.

### Finding Description

The storage declaration explicitly documents the invariant:

```
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [1](#0-0) 

Every call to `stake()` appends the caller's address unconditionally:

```cairo
self.stakers.push(staker_address);
``` [2](#0-1) 

`get_stakers` iterates over the **full range** of this vector on every invocation:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    ...
}
``` [3](#0-2) 

Even though inactive stakers are skipped via `continue`, each entry still requires a storage read for `is_staker_active`. The gas cost is O(N) in the total number of addresses ever staked, not just currently active ones.

The `assert_staker_address_not_reused` guard prevents reusing the same address, but does **not** prevent an attacker from cycling tokens through fresh addresses:

```cairo
fn assert_staker_address_not_reused(self: @ContractState, staker_address: ContractAddress) {
    assert!(
        self.staker_own_balance_trace.entry(key: staker_address).is_empty(),
        "{}", Error::STAKER_ADDRESS_ALREADY_USED,
    );
}
``` [4](#0-3) 

**Attack cycle (one iteration):**
1. Attacker holds `min_stake` STRK tokens.
2. Call `stake()` from address `A_n` → `A_n` appended to `stakers`.
3. Call `unstake_intent()` → starts the `exit_wait_window` (default 1 week).
4. After the window, call `unstake_action()` → tokens returned to `A_n`.
5. Transfer tokens to fresh address `A_{n+1}`.
6. Repeat from step 2.

Each cycle permanently adds one entry to `stakers`. The attacker only needs gas fees and one week per cycle. The vector never shrinks.

### Impact Explanation

`get_stakers` is the consensus-layer entry point used to determine the active validator set for each epoch:

```rust
fn get_stakers(self: @TContractState, epoch_id: Epoch)
    -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)>
``` [5](#0-4) 

As the `stakers` vector grows without bound, the gas cost of `get_stakers` grows linearly. Once the vector is large enough, the call exceeds the Starknet block gas limit, permanently preventing the consensus layer from querying the validator set. This constitutes **unbounded gas consumption** leading to a griefing DOS on the protocol's consensus infrastructure — matching the allowed Medium impact: *"Griefing with no profit motive but damage to users or protocol; Unbounded gas consumption."*

### Likelihood Explanation

The attack requires only `min_stake` STRK tokens (a small, recoverable capital cost since tokens are returned after each unstake) and gas fees. The exit wait window of one week per cycle means the vector grows slowly, but the damage is **permanent and cumulative** — each cycle irreversibly adds one entry. A well-funded attacker can run many parallel accounts to accelerate the attack. No privileged access is required; `stake()` and `unstake_action()` are both open to any address.

### Recommendation

1. **Remove stakers from the vector on `unstake_action`**: Replace the `Vec<ContractAddress>` with a data structure that supports O(1) deletion (e.g., a swap-and-pop pattern or an `IterableMap`).
2. **Alternatively, cap the `stakers` vector size** and enforce a minimum stake high enough to make the attack economically infeasible.
3. **Short-term mitigation**: Add a maximum iteration count inside `get_stakers` or paginate the result to bound per-call gas usage.

### Proof of Concept

```
// Attacker setup: one address with min_stake tokens
// Repeat N times:
for i in 0..N {
    let addr = fresh_address(i);
    transfer(tokens, addr);
    // addr calls stake()  → stakers.push(addr)  [vector grows by 1]
    stake(addr, min_stake);
    // addr calls unstake_intent()
    unstake_intent(addr);
    // advance time by exit_wait_window (1 week)
    advance_time(exit_wait_window);
    // addr calls unstake_action() → tokens returned
    unstake_action(addr);
    // tokens now back in addr, transfer to addr+1 next iteration
}
// After N cycles: stakers.len() == N
// get_stakers() now requires N storage reads → O(N) gas
// At sufficient N, get_stakers() exceeds block gas limit → consensus DOS
```

The `stakers` vector grows by one entry per cycle with no upper bound, and `get_stakers` iterates the full range on every call. [1](#0-0) [3](#0-2)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
