### Title
Missing Authorization Check in `action_deploy_global_contract` Allows Any Account to Overwrite Another Account's Global Contract (AccountId Mode) — (File: `runtime/runtime/src/global_contracts.rs`)

---

### Summary

`action_deploy_global_contract` in `runtime/runtime/src/global_contracts.rs` contains no check that the receipt's `predecessor_id` equals the `receiver_id` when `deploy_mode == GlobalContractDeployMode::AccountId`. Any unprivileged account can therefore overwrite the global contract stored under a victim's account ID by sending a transaction (or cross-contract call) whose `receiver_id` is the victim. All accounts that reference the victim's global contract via `GlobalContractIdentifier::AccountId` will subsequently execute the attacker-supplied code.

---

### Finding Description

When `DeployGlobalContractAction { deploy_mode: AccountId }` executes, `initiate_distribution` derives the trie identifier from `account_id` — which is `receipt.receiver_id()`, fully attacker-controlled: [1](#0-0) 

The resulting trie key written on every shard is:

```
TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim_account_id) }
```

The only guard in `action_deploy_global_contract` is a balance check on the receiver's account: [2](#0-1) 

There is no check that `predecessor_id == account_id`. The action-level validator `validate_deploy_global_contract_action` only enforces a size limit: [3](#0-2) 

The dispatch site in `apply_action_receipt` passes `account_id` (the receiver) directly without any ownership assertion: [4](#0-3) 

**Attack path (direct transaction):**

1. Attacker constructs a `SignedTransaction` with `signer_id = attacker`, `receiver_id = victim`, `actions = [DeployGlobalContract { code: malicious_wasm, deploy_mode: AccountId }]`.
2. The runtime converts it to a receipt; `predecessor_id = attacker`, `receiver_id = victim`.
3. `action_deploy_global_contract` deducts storage cost from the victim's balance and calls `initiate_distribution` with `account_id = victim`.
4. The distribution receipt propagates to every shard and writes `malicious_wasm` under `GlobalContractCode::AccountId(victim)`.

**Attack path (cross-contract call):**

The host function `promise_batch_action_deploy_global_contract_by_account_id` lets any deployed contract create a promise targeting an arbitrary account and attach `DeployGlobalContract(AccountId)`: [5](#0-4) 

The `AccountId` mode is documented as "allows the **owner** to update the contract for all its users": [6](#0-5) 

The ownership invariant is stated but never enforced.

---

### Impact Explanation

- **Code injection (Critical):** Every account that has called `UseGlobalContract { contract_identifier: AccountId(victim) }` now executes the attacker's Wasm. The attacker can steal tokens, corrupt storage, or brick all dependent accounts.
- **Balance drain:** Storage cost for the malicious code is deducted from the victim's account, not the attacker's.
- **Scope:** `global contract/code selection` — the corrupted value is `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim_account_id) }` on every shard.

---

### Likelihood Explanation

- No special privilege is required; any account with a positive balance can sign a transaction.
- The attacker controls `receiver_id` freely.
- The victim only needs to have enough balance to cover `global_contract_storage_amount_per_byte × len(malicious_code)`. A minimal valid Wasm module is a few dozen bytes, making the cost negligible.
- The attack is a single transaction; no mempool race or validator collusion is needed.

---

### Recommendation

Add an authorization guard inside `action_deploy_global_contract` (or in `validate_deploy_global_contract_action` at receipt-validation time) that rejects `AccountId` mode deployments where `predecessor_id != account_id`:

```rust
if deploy_contract.deploy_mode == GlobalContractDeployMode::AccountId {
    // Only the account itself may publish a global contract under its own ID.
    if receipt.predecessor_id() != account_id {
        result.result = Err(ActionErrorKind::ActorNoPermission {
            account_id: account_id.clone(),
            actor_id: receipt.predecessor_id().clone(),
        }
        .into());
        return Ok(());
    }
}
```

Alternatively, enforce `signer_id == receiver_id` for `DeployGlobalContract(AccountId)` at transaction-admission time in `validate_verify_and_charge_transaction`.

---

```rust
fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())
```

```rust
fn apply_distribution_current_shard(
    receipt: &Receipt,
    global_contract_data: &GlobalContractDistributionReceipt,
    apply_state: &ApplyState,
    state_update: &mut TrieUpdate,
) -> Result<Compute, RuntimeError> {
    let identifier = match &global_contract_data.id() {
        GlobalContractIdentifier::CodeHash(hash) => GlobalContractCodeIdentifier::CodeHash(*hash),
        GlobalContractIdentifier::AccountId(account_id) => {
            GlobalContractCodeIdentifier::AccountId(account_id.clone())
        }
    };

    let is_nonce_fresh = check_and_update_nonce(global_contract_data, &identifier, state_update)?;
    if !is_nonce_fresh {
        return Ok(0);
    }

    let config = apply_state.config.wasm_config.clone();
    let trie_key = TrieKey::GlobalContractCode { identifier };
    let code_len = global_contract_data.code().len() as u64;
    state_update.set(trie_key, global_contract_data.code().to_vec());
    state_update.commit(StateChangeCause::ReceiptProcessing { receipt_hash: receipt.get_hash() });
    let code_hash = match global_contract_data.id() {
        GlobalContractIdentifier::CodeHash(hash) => Some(*hash),
        GlobalContractIdentifier::AccountId(_) => None,
    };
    precompile_contract_with_warming(
        &ContractCode::new(global_contract_data.code().to_vec(), code_hash),
        config,
        apply_state.next_wasm_config.clone(),
        apply_state.cache.as_deref(),
    );
    near_vm_runner::report_metrics(apply_state.shard_id, "global_contract");
    let fees = &apply_state.config.fees;
    let per_byte_total = fees
        .deploy_global_contract_execution_per_byte
        .checked_mul(code_len)
        .ok_or(IntegerOverflowError)?;
    let compute = fees
        .deploy_global_contract_execution_base
        .checked_add(per_byte_total)
        .ok_or(IntegerOverflowError)?;
    Ok(compute)
}
```
