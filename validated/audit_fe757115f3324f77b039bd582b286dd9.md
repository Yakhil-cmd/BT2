### Title
Unprivileged Caller Can Permanently Suppress Per-Block Consensus Rewards via `disable_rewards` Bypass - (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` is a publicly callable function in `IStakingRewardsManager`. It accepts a caller-controlled `disable_rewards: bool` flag. When `true`, the function updates the global `last_reward_block` checkpoint but returns before distributing any rewards. Because `last_reward_block` is consumed, no subsequent call can distribute rewards for that block. Any unprivileged address can call this once per block to permanently destroy all consensus-phase block rewards for every staker and delegator.

---

### Finding Description

`update_rewards` is exposed as a public ABI entry point with no role guard — `general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero. [1](#0-0) 

Inside the implementation, the function unconditionally writes `current_block_number` to `last_reward_block` before checking `disable_rewards`: [2](#0-1) 

The early-return branch is:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [3](#0-2) 

The guard that prevents a second call in the same block is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

Because `last_reward_block` is written **before** the `disable_rewards` check, calling `update_rewards(any_active_staker, disable_rewards: true)` atomically:

1. Marks the block as "already processed" in global state.
2. Returns without invoking `_update_rewards`, so `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` is never called and no STRK is ever credited to any staker or pool. [5](#0-4) 

The rewards for that block are irrecoverably lost — there is no mechanism to retroactively distribute them.

The structural analog to the Gramine finding is exact:

| Gramine | Starknet Staking |
|---|---|
| `trusted_files` (specific integrity check) | Consensus reward distribution in `_update_rewards` |
| `allowed_files: /etc` (broad bypass) | `disable_rewards: true` (broad bypass flag) |
| Broader rule silently overrides specific check | `disable_rewards` path consumes `last_reward_block` without distributing rewards |
| SSL cert integrity never verified | Block rewards permanently destroyed |

---

### Impact Explanation

Every block in the consensus-rewards phase carries a STRK reward computed by `calculate_block_rewards` and distributed proportionally to all active stakers and their delegators. When an attacker calls `update_rewards(staker, true)` once per block, every block's reward is permanently destroyed. No staker or delegator ever receives consensus-phase yield. This constitutes **permanent freezing of unclaimed yield** for all protocol participants. [6](#0-5) 

---

### Likelihood Explanation

The function is publicly callable by any non-zero address with no role restriction. The only cost to the attacker is gas per block. The attacker has no requirement to hold stake, be a staker, or have any privileged role. The target `staker_address` only needs to be any currently active staker (trivially discoverable from on-chain events). The attack is sustainable indefinitely. [7](#0-6) 

---

### Recommendation

Remove the `disable_rewards` parameter from the public interface entirely, or restrict `update_rewards` to a trusted caller (e.g., the sequencer or an authorized contract). If `disable_rewards` must remain for internal use (e.g., during staker migration), gate it with a role check such as `self.roles.only_app_governor()` and ensure `last_reward_block` is only written when rewards are actually distributed.

---

### Proof of Concept

1. Consensus rewards are activated (`consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. Attacker (any EOA) identifies any active staker address `S` from `NewStaker` events.
3. Each block, attacker calls: `staking_contract.update_rewards(S, disable_rewards: true)`.
4. `last_reward_block` is set to the current block number.
5. Any legitimate call to `update_rewards` for any staker in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. `_update_rewards` is never reached; `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` is never called.
7. All stakers' `unclaimed_rewards_own` and all pool `cumulative_rewards_trace` entries remain frozen at zero for every block the attacker targets.
8. Repeated every block → all consensus-phase yield is permanently destroyed. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
