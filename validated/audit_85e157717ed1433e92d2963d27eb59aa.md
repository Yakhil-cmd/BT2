### Title
Factory Protocol Owner Cannot Fully Restore Pool to Active State Without Pool Admin Cooperation — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory` exposes `setPause(uint8)` on every deployed pool as an `onlyFactory` entrypoint that accepts levels 0 (active), 1 (admin-paused), and 2 (protocol-paused). However, the factory itself only wires two of those three values: `protocolPausePool` calls `setPause(2)` and `protocolUnpausePool` calls `setPause(1)`. There is no factory-owner path that calls `setPause(0)`. The only path to level 0 is `unpausePool`, which is `onlyPoolAdmin`. If the pool admin is unavailable — lost key, dead contract, or compromised address — the protocol owner cannot fully restore the pool to operational state after a protocol pause, permanently disabling all swaps.

---

### Finding Description

`MetricOmmPool.setPause` is gated `onlyFactory` and accepts any value in `[0, 2]`:

```solidity
// MetricOmmPool.sol
function setPause(uint8 newLevel) external onlyFactory {
    if (newLevel > 2) revert InvalidPauseLevel();
    ...
    pauseLevel = newLevel;
}
``` [1](#0-0) 

The factory's protocol-owner pause functions are:

```solidity
// MetricOmmPoolFactory.sol
function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    IMetricOmmPoolFactoryActions(pool).setPause(2);          // ← only ever writes 2
}

function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1);          // ← only ever writes 1
}
``` [2](#0-1) 

The only path to level 0 is the pool-admin function:

```solidity
function unpausePool(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 1) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);          // ← pool admin only
}
``` [3](#0-2) 

`_checkNotPaused()` blocks swaps at any non-zero level:

```solidity
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
``` [4](#0-3) 

`swap` enforces this check via `whenNotPaused`:

```solidity
function swap(...) external whenNotPaused nonReentrant(PoolActions.SWAP) ...
``` [5](#0-4) 

The interface comment acknowledges the design but does not account for the admin-unavailability scenario:

> "Intentionally transitions only **2 → 1**. Full resume to level **0** requires the pool admin to call `unpausePool`." [6](#0-5) 

The factory owner has no function that calls `setPause(0)` on any pool. The pool's `onlyFactory` guard means no external actor other than the factory can invoke it. The factory's `onlyPoolAdmin` guard on `unpausePool` means the factory owner cannot bypass the pool admin. The result is a dead-end: after `protocolUnpausePool` the pool is at level 1 and the factory owner has exhausted all available transitions.

---

### Impact Explanation

All swaps are permanently disabled for any pool whose admin is unavailable after a protocol-level pause/unpause cycle. `removeLiquidity` is not gated by `_checkNotPaused`, so LP principal is recoverable, but the pool's core swap functionality is irreversibly broken. Traders cannot execute any trades; the pool is effectively dead as an AMM. Protocol fee revenue from that pool is permanently lost.

---

### Likelihood Explanation

The trigger requires two conditions: (1) the factory owner exercises `protocolPausePool` on a pool (an emergency action), and (2) the pool admin is subsequently unavailable — a lost private key, a multisig that loses quorum, a smart-contract admin with a bug, or a deliberately uncooperative admin. Both conditions are realistic in production. The factory owner has no way to detect or prevent this at pause time.

---

### Recommendation

Add a factory-owner function that calls `setPause(0)` directly, bypassing the pool-admin requirement in genuine emergencies:

```solidity
/// @notice Force-restore pool to fully active state (level 0).
/// @dev Only callable by factory owner; intended for cases where pool admin is unavailable.
function protocolForceUnpausePool(address pool) external nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 1 && cur != 2) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
    emit PoolForcefullyUnpaused(pool);
}
```

Alternatively, modify `protocolUnpausePool` to accept a target level parameter so the owner can choose between 1 and 0 depending on whether the pool admin is available.

---

### Proof of Concept

```
1. Factory owner calls protocolPausePool(pool)
   → pool.pauseLevel = 2 (swaps blocked)

2. Emergency resolved. Factory owner calls protocolUnpausePool(pool)
   → pool.pauseLevel = 1 (swaps still blocked)

3. Pool admin key is lost / admin contract is bricked.

4. Factory owner attempts to call unpausePool(pool)
   → reverts: NotPoolAdmin

5. Factory owner attempts to call protocolUnpausePool(pool) again
   → reverts: InvalidPauseTransition(1, 1)  [cur != 2]

6. Factory owner attempts to call protocolPausePool(pool) then protocolUnpausePool(pool)
   → pool.pauseLevel = 2, then back to 1 — still stuck

7. No factory-owner path exists to call setPause(0).
   Pool is permanently at level 1. All swaps revert PoolPaused forever.
``` [2](#0-1) [7](#0-6) [1](#0-0)
