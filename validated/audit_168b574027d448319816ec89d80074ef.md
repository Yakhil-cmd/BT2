### Title
Missing Lower-Bound Validation on `threshold` in FROST Signing Initialization — (`src/frost/mod.rs`, `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

`assert_sign_inputs` in `src/frost/mod.rs` and `sign` in `src/ecdsa/ot_based_ecdsa/sign.rs` validate that `threshold` does not exceed the participant count, but impose **no lower-bound check** (`threshold >= 2`). The DKG initialization function `assert_key_invariants` in `src/dkg.rs` correctly enforces `threshold >= 2`, but this invariant is absent from the signing-phase entry points. A caller (including a malicious coordinator) can pass `threshold = 1` (or `threshold = 0`) to the signing API, causing Lagrange interpolation to use the wrong reconstruction degree and producing a cryptographically invalid or unusable signature output.

---

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces a minimum threshold of 2:

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

However, `assert_sign_inputs` in `src/frost/mod.rs` only checks the upper bound:

```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [2](#0-1) 

There is no corresponding `threshold < 2` guard. The same omission exists in `frost::presign`:

```rust
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [3](#0-2) 

And in `ot_based_ecdsa::sign`:

```rust
if participants.len() < threshold {
    return Err(InitializationError::NotEnoughParticipantsForThreshold { ... });
}
``` [4](#0-3) 

No `threshold < 2` check is present in either signing entry point. [5](#0-4) 

The `ReconstructionLowerBound` type itself imposes no minimum:

```rust
pub struct ReconstructionLowerBound(usize);
impl ReconstructionLowerBound {
    pub fn value(self) -> usize { self.0 }
}
``` [6](#0-5) 

---

### Impact Explanation

Keys are always generated with `threshold >= 2` (enforced by `assert_key_invariants`), meaning every signing share lies on a polynomial of degree `T-1 >= 1`. When a caller passes `threshold = 1` to the signing functions, the Lagrange interpolation inside the signing protocol uses a degree-0 reconstruction (a single point with coefficient 1). This does not reconstruct the correct secret, so the aggregated signature fails verification. Honest participants complete the protocol and accept the output, but the resulting signature is cryptographically unusable — a corruption of the signing output.

With `threshold = 0`, `threshold.value() > participants.len()` evaluates to `0 > N = false`, so the check passes silently. Downstream Lagrange interpolation over zero points is undefined and may panic or produce garbage output.

This maps to: **High — Corruption of sign outputs so honest parties accept inconsistent or unusable cryptographic outputs.**

---

### Likelihood Explanation

The signing API is a public library entry point callable by any application integrating the crate. A misconfigured caller (e.g., accidentally passing `1` instead of the keygen threshold) or a malicious coordinator controlling the signing session parameters can trivially trigger this path. No privileged access or cryptographic break is required — only passing an integer argument.

---

### Recommendation

Add the same lower-bound guard that `assert_key_invariants` already enforces, to both `assert_sign_inputs` and `frost::presign` in `src/frost/mod.rs`, and to `ot_based_ecdsa::sign` in `src/ecdsa/ot_based_ecdsa/sign.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This mirrors the existing guard in `assert_key_invariants`: [7](#0-6) 

---

### Proof of Concept

1. Run DKG with `threshold = 2`, `participants = [P1, P2, P3]` — succeeds because `assert_key_invariants` enforces `threshold >= 2`.
2. Call `frost::assert_sign_inputs(&[P1, P2, P3], 1usize, P1, P1)` — passes all current checks (`1 <= 3`, participants valid).
3. Proceed to FROST signing with `threshold = 1`: the coordinator collects only its own share and applies Lagrange coefficient `λ = 1` (degree-0 interpolation).
4. The reconstructed nonce/secret is wrong (the key polynomial has degree 1), so the aggregated signature fails standard Ed25519 verification.
5. Honest participants have completed the protocol and accepted the output — the signing session is permanently corrupted.

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
