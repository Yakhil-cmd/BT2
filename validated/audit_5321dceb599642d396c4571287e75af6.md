### Title
STRK Reward Tokens Permanently Frozen in Pool Contract When Pool Balance Falls Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo)

### Summary

The `Pool` contract receives STRK reward tokens from the staking contract even when the pool's delegated balance is below `min_delegation_for_rewards`. Because `compute_rewards_per_unit` returns zero in that case, the cumulative rewards trace is never updated, making those STRK tokens permanently unclaimable by any delegator. There is no withdrawal or recovery function in the pool contract.

### Finding Description

**Step 1 – Staking contract sends STRK unconditionally.**

In `_update_rewards` → `calculate_staker_pools_rewards` (`src/staking/staking.cairo`), the staking contract computes `pool_rewards_including_commission` using `pool_balance_curr_epoch.to_amount_18_decimals()` with no lower-bound guard: [1](#0-0) 

If `pool_rewards` is non-zero (any non-zero pool balance with commission < 100 %), the tuple is appended to `pools_rewards_data` and `update_pool_rewards` is called.

**Step 2 – STRK is transferred to the pool contract before the trace is updated.**

Inside `update_pool_rewards`, `send_rewards_to_delegation_pool` executes the ERC-20 transfer *first*, then calls `update_rewards_from_staking_contract`: [2](#0-1) 

**Step 3 – Pool contract silently discards the rewards in the trace.**

`update_rewards_from_staking_contract` delegates to `compute_rewards_per_unit`, which returns **zero** when `total_stake < min_delegation_for_rewards`: [3](#0-2) 

The cumulative rewards trace is therefore incremented by zero: [4](#0-3) 

The developer comment on `compute_rewards_per_unit` explicitly acknowledges the mismatch:

> **Note**: Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case. [5](#0-4) 

**Step 4 – `claim_rewards` can never distribute the stuck STRK.**

`claim_rewards` computes payouts solely from the cumulative rewards trace. Because the trace was not advanced, the calculation returns zero for those epochs, and the STRK balance sitting in the pool contract is never touched: [6](#0-5) 

There is no sweep, rescue, or admin-withdrawal function anywhere in the pool contract.

### Impact Explanation

Every epoch in which a pool's delegated balance is below `min_delegation_for_rewards` (1 STRK for STRK pools; `10^(decimals−5)` for BTC pools, e.g. 1 000 satoshis for 8-decimal BTC), the STRK rewards forwarded by the staking contract accumulate in the pool contract with no path to recovery. This constitutes **permanent freezing of unclaimed yield** for all delegators of that pool.

### Likelihood Explanation

The threshold is low (1 STRK or ~1 000 satoshis). Any newly opened pool, any pool whose delegators have partially exited, or any pool that was deliberately seeded with a dust amount will fall below the threshold. An unprivileged delegator can deliberately keep a pool below the threshold by delegating just under `min_delegation_for_rewards`, causing every subsequent attestation to silently burn the pool's share of epoch rewards into the pool contract.

### Recommendation

Add a symmetric guard in the staking contract so that `pool_rewards` is only forwarded when `pool_balance.to_native_amount(decimals) >= pool_contract.min_delegation_for_rewards()`. Alternatively, when `compute_rewards_per_unit` returns zero, the pool contract should refund the received STRK back to the staking contract (or to the reward supplier) rather than silently retaining it.

### Proof of Concept

1. Deploy the system; open a BTC pool with 8-decimal token.
2. Delegator calls `enter_delegation_pool` with `amount = 999` satoshis (below `min_delegation_for_rewards = 1000`).
3. Advance K epochs; staker attests → `update_rewards_from_attestation_contract` → `_update_rewards` → `calculate_staker_pools_rewards` computes non-zero `pool_rewards` → `update_pool_rewards` transfers STRK to the pool contract.
4. Inside the pool contract, `update_rewards_from_staking_contract` calls `compute_rewards_per_unit(staking_rewards, total_stake=999)` → returns 0 → cumulative trace unchanged.
5. Delegator calls `claim_rewards` → receives 0 STRK.
6. Inspect `IERC20(STRK).balance_of(pool_contract)` → non-zero and growing each epoch.
7. No function exists to recover this balance; it is permanently frozen. [7](#0-6) [8](#0-7)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        /// Compute the rewards for the pool trace.
        ///
        /// `staking_rewards` is in `STRK_DECIMALS` decimals.
        /// `total_stake` is in the contract's token decimals.
        /// **Note**: Delegation rewards lost when pool balance is less than
        /// `min_delegation_for_rewards`. The staking contract continues to forward
        /// `pool_rewards` to the pool contract even in this case.
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
