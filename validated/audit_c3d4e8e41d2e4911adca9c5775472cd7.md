### Title
Missing Identity-Point Validation on `app_pk` Strips ElGamal Blinding, Exposing Unblinded BLS Signature to Coordinator — (`src/confidential_key_derivation/protocol.rs`)

---

### Summary

`ckd()` accepts `app_pk: PublicKey` with no check that it is a non-identity G1 point. When `app_pk = G1Projective::identity()` is supplied, the ElGamal blinding term `y_i · A` collapses to the identity in every participant's `compute_signature_share`, so the coordinator aggregates the raw, unblinded BLS signature `msk · H(pk ‖ app_id)` directly in the `C` field of `CKDOutput`.

---

### Finding Description

**Entry point — `ckd()` validation block** [1](#0-0) 

The four guards in `ckd()` check participant count, duplicates, self-presence, and coordinator-presence. There is no guard on `app_pk`. Any `G1Projective` value — including `G1Projective::identity()` — is accepted and forwarded unchanged to `run_ckd_protocol` → `compute_signature_share`.

**Root cause — blinding computation in `compute_signature_share`** [2](#0-1) 

```
S_i  = x_i · H(pk ‖ app_id)
C_i  = S_i + y_i · app_pk          // line 174
```

When `app_pk = identity`:

```
y_i · identity  =  identity   (scalar mult of ∞ is always ∞)
C_i             =  S_i + identity  =  S_i
```

The blinding term vanishes entirely. Each participant sends `(λ_i · Y_i, λ_i · S_i)` to the coordinator.

**Coordinator aggregation** [3](#0-2) 

```
C = Σ λ_i · C_i = Σ λ_i · x_i · H(pk ‖ app_id) = msk · H(pk ‖ app_id)
```

The coordinator's `CKDOutput.big_c` is now the raw BLS signature — the exact value the protocol is designed to hide from every MPC node.

**Protocol invariant (from spec)** [4](#0-3) 

The spec requires `C = msk · H(pk, app_id) + a · Y`. The `a · Y` term is the ElGamal mask. With `app_pk = identity` that term is zero, and the invariant is broken.

**Security requirement violated** [5](#0-4) 

> "secrecy does not [depend on all nodes being honest]. Example values that should not be leaked even if a node is malicious are `s`, `msk` and private shares of other nodes."

The coordinator is a single MPC node. After this attack it holds `C = msk · H(pk ‖ app_id)` in plaintext, directly violating the stated secrecy guarantee.

---

### Impact Explanation

**Critical — disclosure of a confidential derived secret.**

The coordinator observes `C = msk · H(pk ‖ app_id)` without needing `a`. Because `unmask(a) = C − a · Y`, and with `app_pk = identity` the `Y` aggregation is still a random non-zero point, the coordinator cannot trivially verify it has the right value — but it can verify by checking `e(C, G2) == e(H(pk ‖ app_id), pk)` using the public BLS pairing. The coordinator therefore obtains and can verify the full confidential derived secret `s`, which is supposed to be known only to the TEE app.

---

### Likelihood Explanation

The `app_pk` value `A` is submitted by the TEE app to the blockchain and read by MPC nodes before calling `ckd()`. A malicious or compromised TEE app (or any caller of the public `ckd()` API) can trivially supply `G1Projective::identity()`. No cryptographic assumption needs to be broken; it is a single-line API call. The library performs no downstream serialization check that would reject the identity point before `compute_signature_share` runs (the `BLS12381G1Group::serialize` identity check exists only for FROST group elements, not for the `app_pk` parameter path). [6](#0-5) 

---

### Recommendation

Add an identity-point check at the top of `ckd()`, immediately after the existing participant-list guards:

```rust
if app_pk.is_identity().into() {
    return Err(InitializationError::InvalidInput(
        "app_pk must not be the identity point".into(),
    ));
}
```

This mirrors the pattern already used in `BLS12381G1Group::serialize` and `BLS12381G1Group::deserialize`.

### Proof of Concept

```rust
#[test]
fn test_identity_app_pk_strips_blinding() {
    use blstrs::{G1Projective, G2Projective, Scalar};
    use elliptic_curve::Group;
    use rand_core::SeedableRng;
    use threshold_signatures::{
        confidential_key_derivation::{
            protocol::ckd, AppId, CKDOutputOption,
            hash_app_id_with_pk,
        },
        test_utils::{MockCryptoRng, generate_participants,
                     run_protocol, check_one_coordinator_output, GenProtocol},
        confidential_key_derivation::{KeygenOutput, SigningShare, VerifyingKey},
    };
    use threshold_signatures::participants::ParticipantList;
    use threshold_signatures::confidential_key_derivation::BLS12381SHA256;

    let mut rng = MockCryptoRng::seed_from_u64(99);

    // Malicious app_pk = identity
    let app_pk = G1Projective::identity();

    let app_id = AppId::try_from(b"evil app").unwrap();
    let participants = generate_participants(3);
    let participant_list = ParticipantList::new(&participants).unwrap();
    let coordinator = participants[0];

    let mut private_shares = Vec::new();
    let mut msk = Scalar::ZERO;
    for (i, _) in participants.iter().enumerate() {
        let ps = SigningShare::new(Scalar::random(&mut rng));
        let li = participant_list
            .lagrange::<BLS12381SHA256>(participant_list.get_participant(i).unwrap())
            .unwrap();
        msk += li * ps.to_scalar();
        private_shares.push(ps);
    }
    let pk = VerifyingKey::new(G2Projective::generator() * msk);

    let mut protocols: GenProtocol<CKDOutputOption> = Vec::new();
    for (i, p) in participants.iter().enumerate() {
        let rng_p = MockCryptoRng::seed_from_u64(i as u64 + 1);
        let kp = KeygenOutput { public_key: pk, private_share: private_shares[i] };
        let proto = ckd(&participants, coordinator, *p, kp,
                        app_id.clone(), app_pk, rng_p).unwrap();
        protocols.push((*p, Box::new(proto)));
    }

    let result = run_protocol(protocols).unwrap();
    let ckd_output = check_one_coordinator_output(result, coordinator).unwrap();

    // Coordinator's big_c should equal the raw BLS signature (no blinding)
    let expected = hash_app_id_with_pk(&pk, &app_id) * msk;
    assert_eq!(
        ckd_output.big_c(), expected,
        "Coordinator sees unblinded BLS signature — confidentiality broken"
    );
}
```

The assertion passes because `big_c = msk · H(pk ‖ app_id)` with no `a · Y` mask, confirming the coordinator directly observes the confidential derived secret.
