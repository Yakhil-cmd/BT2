### Title
Zero Reward Address Accepted Without Validation, Causing Reward Transfer Revert and Temporary Freeze of Unclaimed Yield — (`File: src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

Both `change_reward_address` in the Staking contract and `change_reward_address` in the Pool contract accept the zero address as a valid reward address. The initial entry points `stake` and `enter_delegation_pool` share the same gap. When rewards are later claimed, the ERC20 `checked_transfer` to the zero address reverts, temporarily freezing the staker's or pool member's unclaimed yield and blocking `unstake_action` for stakers.

---

### Finding Description

`change_reward_address` in `staking.cairo` validates only that the new address is not a registered token address:

```cairo
assert!(
    !self.does_token_exist(token_address: reward_address),
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
```

There is no `assert!(reward_address.is_non_zero(), ...)` guard. The same single-check pattern exists in `pool.cairo`'s `change_reward_address`, and in the initial `stake` and `enter_delegation_pool` entry points. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

Once a zero reward address is stored, every downstream reward-transfer path hits the same wall. In `send_rewards_to_staker`:

```cairo
token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
``` [5](#0-4) 

And in `pool.cairo`'s `claim_rewards`:

```cairo
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
``` [6](#0-5) 

The STRK token is an OpenZeppelin ERC20 that asserts `recipient.is_non_zero()` inside `_transfer`. A transfer to the zero address therefore reverts, causing `claim_rewards` and `unstake_action` to revert entirely. [7](#0-6) 

---

### Impact Explanation

**Temporary freezing of unclaimed yield and principal (High).**

- A staker with `reward_address == 0` cannot call `claim_rewards` — the call reverts at the ERC20 transfer step.
- `unstake_action` calls `send_rewards_to_staker` before returning the principal, so it also reverts, temporarily locking the staker's entire stake.
- A pool member with `reward_address == 0` cannot call `claim_rewards` in the pool contract.

The freeze is temporary because the affected party can call `change_reward_address` with a valid address and retry. However, during the window between setting the zero address and correcting it, all reward claims and (for stakers) the unstake flow are blocked. [8](#0-7) [9](#0-8) 

---

### Likelihood Explanation

**Low-Medium.** The most plausible path is a staker or delegator who passes `Zero::zero()` as `reward_address` at entry time (`stake` or `enter_delegation_pool`), intending it as "no separate reward address" or as a placeholder. A user who later calls `change_reward_address(Zero::zero())` to "clear" the field faces the same outcome. No privileged role is required; any unprivileged staker or pool member can trigger this state. [10](#0-9) [11](#0-10) 

---

### Recommendation

Add a non-zero guard in every function that writes a `reward_address`:

1. `Staking::stake` — before storing `reward_address`.
2. `Staking::change_reward_address` — alongside the existing token check.
3. `Pool::enter_delegation_pool` — before creating the pool member record.
4. `Pool::change_reward_address` — alongside the existing token check.

Example addition (mirrors the existing `assert_caller_is_not_zero` pattern already used in `utils.cairo`):

```cairo
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
``` [12](#0-11) 

---

### Proof of Concept

**Staker path:**

1. Staker calls `stake(reward_address: Zero::zero(), operational_address: <valid>, amount: <min_stake>)`.
   - Passes all current checks (zero is not a token address).
   - Staker record is created with `reward_address = 0`.
2. Staker accrues rewards over one or more epochs.
3. Staker calls `claim_rewards(staker_address)`.
   - `send_rewards_to_staker` executes `checked_transfer(recipient: 0, amount: rewards)`.
   - OZ ERC20 asserts `recipient.is_non_zero()` → transaction reverts.
4. Staker calls `unstake_intent()` then `unstake_action(staker_address)`.
   - `unstake_action` calls `send_rewards_to_staker` first → same revert.
   - Principal is locked until the staker calls `change_reward_address(<valid>)`.

**Pool member path:**

1. Pool member calls `enter_delegation_pool(reward_address: Zero::zero(), amount: <valid>)`.
2. Pool member accrues rewards.
3. Pool member calls `claim_rewards(pool_member)`.
   - `checked_transfer(recipient: 0, amount: rewards)` → reverts.
4. Pool member calls `change_reward_address(<valid>)` to recover. [13](#0-12) [14](#0-13)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
