### Title
Missing Access Control on `disable_rewards` Parameter Allows Permanent Freezing of Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `Staking` is callable by any unprivileged address and accepts a `disable_rewards: bool` parameter. Because `last_reward_block` is a single global variable updated on every call, an attacker can call `update_rewards(any_staker, disable_rewards: true)` once per block to permanently prevent all reward distribution across the entire protocol.

### Finding Description
`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. There is no check that the caller is the consensus layer, the staker themselves, or any other authorized party. [1](#0-0) 

The function enforces a single-call-per-block invariant via a global `last_reward_block` storage variable: [2](#0-1) 

When `disable_rewards` is `true`, the function writes `current_block_number` to `last_reward_block` and returns immediately without distributing any rewards: [3](#0-2) 

Because `last_reward_block` is global (not per-staker), a single call with `disable_rewards: true` for any `staker_address` consumes the entire block's reward slot for all stakers. [4](#0-3) 

### Impact Explanation
An attacker who calls `update_rewards(any_staker_address, disable_rewards: true)` at the start of every block permanently prevents all stakers from accumulating unclaimed yield. The rewards are never credited to `staker_info.unclaimed_rewards_own` and never forwarded to delegation pools. Because the reward supplier's `unclaimed_rewards` is only incremented inside `_update_rewards` (which is skipped), the yield is permanently lost to all participants.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**. [5](#0-4) 

### Likelihood Explanation
The entry point is fully public — any EOA or contract can call `update_rewards`. The attacker only needs to submit one cheap transaction per block. On Starknet the sequencer orders transactions, so a well-funded attacker (or a malicious sequencer) can reliably front-run every legitimate `update_rewards` call. The attack requires no privileged access, no leaked keys, and no external dependency.

### Recommendation
Restrict `update_rewards` so that only an authorized caller (e.g., the consensus layer contract, or the staker's own operational address) may invoke it. Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "no reward" case internally based on verifiable on-chain data (e.g., whether the staker produced the block).

### Proof of Concept
1. Attacker monitors the chain. At the start of block `N`, before any legitimate call arrives, attacker submits:
   ```
   staking.update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
   ```
2. The call passes `general_prerequisites()` (contract not paused, caller non-zero).
3. `current_block_number (N) > last_reward_block` → assertion passes.
4. `last_reward_block` is written to `N`.
5. `disable_rewards == true` → function returns early; `_update_rewards` is never called; no yield is credited.
6. Any subsequent `update_rewards` call in block `N` reverts with `REWARDS_ALREADY_UPDATED`.
7. Attacker repeats step 1 every block. All stakers accumulate zero unclaimed yield indefinitely. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
