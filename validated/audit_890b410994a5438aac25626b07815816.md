### Title
Missing Caller Authorization in `unstake_action` Allows Anyone to Force a Staker's Exit — (File: `src/staking/staking.cairo`)

---

### Summary

`unstake_action` in the `Staking` contract accepts a `staker_address` parameter but performs no check that the caller is the staker (or any authorized party). After a staker's `exit_wait_window` has elapsed, any unprivileged address can call `unstake_action(staker_address)` and permanently remove the staker from the protocol.

---

### Finding Description

`unstake_intent` correctly restricts itself to the staker by deriving the address from `get_caller_address()`: [1](#0-0) 

`unstake_action`, however, accepts `staker_address` as a caller-supplied argument and contains no authorization guard: [2](#0-1) 

The only preconditions checked are:
1. The staker has a pending unstake intent (`unstake_time` is `Some`).
2. The current time is past `unstake_time`.

Neither condition restricts who may call the function. Any address that passes `general_prerequisites()` (non-zero caller, contract not paused) can execute the action on behalf of any staker whose wait window has expired.

The function then:
- Sends unclaimed rewards to `staker_info.reward_address`
- Returns the staked principal to `staker_address`
- Calls `remove_staker`, which writes `VInternalStakerInfo::None` and zeroes the operational address mapping [3](#0-2) 

After removal, `assert_staker_address_not_reused` permanently blocks the same address from re-staking: [4](#0-3) 

The check inspects `staker_own_balance_trace`, which is written during `stake()` and never cleared during `unstake_action`, so the address is blacklisted forever.

---

### Impact Explanation

An attacker can grief any staker who has declared an unstake intent by calling `unstake_action` the moment the wait window expires. The staker:

- Receives their funds at the correct addresses (no direct theft), but
- Is **permanently removed** from the protocol under that staker address — they cannot re-stake with the same address due to `STAKER_ADDRESS_ALREADY_USED` / `STAKER_ADDRESS_ALREADY_USED_IN_V1` guards.
- Must re-register under a new address, losing their operational address mapping, public key, peer ID, and delegation pool associations.

This matches the **Medium — Griefing with no profit motive but damage to users or protocol** impact category.

---

### Likelihood Explanation

- The attack requires no special privilege: any non-zero address can call `unstake_action`.
- The attacker only needs to monitor the chain for stakers whose `unstake_time` has passed.
- The cost is a single transaction; the attacker bears only gas.
- Stakers who declared intent but later changed their mind (e.g., to re-stake under the same identity) are the primary victims.

---

### Recommendation

Add a caller check analogous to the one used in `claim_rewards`:

```cairo
fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    self.general_prerequisites();
    let caller = get_caller_address();
    let mut staker_info = self.internal_staker_info(:staker_address);
    assert!(
        caller == staker_address || caller == staker_info.reward_address,
        "{}",
        Error::UNSTAKE_ACTION_FROM_UNAUTHORIZED_ADDRESS,
    );
    // ... rest of function
}
```

This mirrors the pattern already used in `claim_rewards`: [5](#0-4) 

The same fix should be applied to `exit_delegation_pool_action` in `src/pool/pool.cairo`, which has the identical missing-authorization pattern: [6](#0-5) 

---

### Proof of Concept

1. Staker `S` calls `unstake_intent()`. Their stake is removed from total stake; `unstake_time` is set to `now + exit_wait_window`.
2. Attacker `A` waits until `Time::now() >= unstake_time`.
3. `A` calls `unstake_action(S)`.
4. The function passes all checks (no caller guard), sends `S`'s rewards to `S`'s reward address, returns principal to `S`, and calls `remove_staker`.
5. `S`'s `staker_own_balance_trace` is non-empty; `assert_staker_address_not_reused` will now permanently reject any future `stake()` call from address `S`.
6. `S` is forced to re-register under a new address, losing their on-chain identity (operational address, public key, peer ID, pool associations). [7](#0-6)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
