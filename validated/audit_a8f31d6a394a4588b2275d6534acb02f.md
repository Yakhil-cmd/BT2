### Title
Missing Zero `msg_hash` Validation in OT-Based ECDSA `sign` Entry Point — (File: `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary
The OT-based ECDSA `sign` function accepts a zero `msg_hash` without rejection. The robust ECDSA `sign` function explicitly guards against this. A malicious caller can pass `msg_hash = 0`, causing the protocol to produce and return a cryptographically valid threshold ECDSA signature for the zero message hash — an attacker-chosen degenerate input.

---

### Finding Description

In `src/ecdsa/ot_based_ecdsa/sign.rs`, the public entry point `sign` (lines 22–76) validates participant count, duplicates, self-presence, coordinator-presence, and threshold, but performs **no check that `msg_hash` is non-zero**: [1](#0-0) 

By contrast, the robust ECDSA `sign` function in `src/ecdsa/robust_ecdsa/sign.rs` explicitly rejects zero message hashes at lines 91–95: [2](#0-1) 

When `msg_hash = 0`, the per-party signature share computation in `compute_signature_share` (lines 139–159) reduces to:

```
s_i = 0 * k_i + R_x * sigma_i  ==>  s_i = R_x * sigma_i
``` [3](#0-2) 

After aggregation, the coordinator holds `s = R_x * sigma`. The coordinator then calls `sig.verify(&public_key, &msg_hash)`: [4](#0-3) 

The `Signature::verify` function (lines 65–79) does not reject a zero `msg_hash`: [5](#0-4) 

Verification with `h = 0` reduces to checking `public_key * (r * s_inv) == R`. Substituting `s = R_x * k * x` (since `sigma = k * x` from the presigning triple relationship) and `R_x = r`:

```
x * r * s_inv = x * r / (r * k * x) = 1/k  =>  k^{-1} * G = R  ✓
```

The verification passes, and the coordinator returns a valid `Signature` struct. The protocol completes successfully with a fully valid threshold ECDSA signature over the zero message hash.

Additionally, the signature leaks `sigma = s / R_x = k * x` — the product of the secret nonce and the private key — to any observer of the output.

---

### Impact Explanation

**Impact: Critical — Unauthorized creation of a valid threshold signature for attacker-chosen inputs.**

A malicious caller supplies `msg_hash = Scalar::ZERO` to `ot_based_ecdsa::sign::sign`. The protocol runs to completion and the coordinator outputs a valid, verifiable ECDSA signature `(R, R_x * sigma)` for the zero message hash. This is a valid threshold signature produced for an attacker-chosen degenerate input that the library's own documentation and the sibling robust ECDSA implementation explicitly prohibit. The output signature also exposes `sigma = k * x` to any party that observes it.

---

### Likelihood Explanation

`msg_hash` is a plain `Scalar` parameter supplied directly by the caller with no type-level or runtime constraint preventing zero. Any unprivileged library caller invoking `ot_based_ecdsa::sign::sign` can trivially pass `Scalar::ZERO`. The function's own doc-comment warns that "allowing the signing of arbitrary scalars *is* a security risk," yet no guard is present. [6](#0-5) 

---

### Recommendation

Add the same guard that exists in the robust ECDSA `sign` function, immediately before the protocol is launched:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0".to_string(),
    ));
}
```

This mirrors the existing check at lines 91–95 of `src/ecdsa/robust_ecdsa/sign.rs`. [2](#0-1) 

---

### Proof of Concept

1. Obtain a valid `RerandomizedPresignOutput` through the normal presigning flow.
2. Call:
   ```rust
   ot_based_ecdsa::sign::sign(
       &participants,
       coordinator,
       threshold,
       me,
       public_key,
       presignature,
       Scalar::ZERO,   // attacker-chosen zero hash
   )
   ```
3. Run the protocol to completion. The coordinator's `do_sign_coordinator` computes `s = R_x * sigma`, calls `sig.verify(&public_key, &Scalar::ZERO)`, which returns `true`, and returns `Ok(Some(sig))`.
4. The returned `Signature { big_r: R, s: R_x * sigma }` is a valid, verifiable ECDSA signature for the zero message hash — produced by the full threshold protocol with no party detecting the anomaly. [7](#0-6)

    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

    mut chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    public_key: AffinePoint,

    msg_hash: Scalar,

    // Round 1
    let s_i = compute_signature_share(&participants, me, &presignature, msg_hash)?;
    // Spec 1.4 is non-existent for a coordinator

    let wait0 = chan.next_waitpoint();
    // Receive sj
    // Spec 1.5
    let mut s = s_i;
    for (_, s_j) in recv_from_others::<Scalar>(&chan, wait0, &participants, me).await? {
        // Spec 1.6
        s += s_j;
    }

    // Spec 1.7
    s.conditional_assign(&(-s), s.is_high());

    if !sig.verify(&public_key, &msg_hash) {
        return Err(ProtocolError::AssertionFailed(
            "signature failed to verify".to_string(),

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    presignature: &RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<Scalar, ProtocolError> {
    // Round 1
    // Linearize ki
    // Spec 1.1
    let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
    let k_i = lambda * presignature.k;

    // Linearize sigmai
    // Spec 1.2
    let sigma_i = lambda * presignature.sigma;

    // Compute si = h * ki + Rx * sigmai
    // Spec 1.3
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)
}

    pub fn verify(&self, public_key: &AffinePoint, msg_hash: &Scalar) -> bool {
        let r: Scalar = x_coordinate(&self.big_r);
        if r.is_zero().into() || self.s.is_zero().into() {
            return false;
        }
        // Check if s has been normalized
        if self.s.is_high().into() {
            return false;
        }
        // tested earlier is not zero, so inversion will not raise an error and unwrap cannot panic
        let s_inv = self.s.invert_vartime().unwrap();
        let reproduced = (ProjectivePoint::GENERATOR * (*msg_hash * s_inv))
            + (ProjectivePoint::from(*public_key) * (r * s_inv));
        x_coordinate(&reproduced.into()) == r
    }
