### Title
Gateway Admits Signature-Unverified Invoke Transaction for Undeployed Account via `skip_stateful_validations` UX Bypass - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator unconditionally suppresses the `__validate__` entry-point call for any Invoke transaction whose `nonce == 1` and whose sender address already appears in the mempool or a recent block. Because the only signature check for an Invoke transaction is that `__validate__` call, an unprivileged attacker can craft an Invoke with an arbitrary (invalid) signature targeting any undeployed account that has a pending `deploy_account` in the mempool, and the gateway will admit it without ever verifying the signature.

### Finding Description

The gateway stateful-validation path is:

```
extract_state_nonce_and_run_validations
  └─ run_pre_validation_checks
       ├─ validate_state_preconditions   (nonce range, resource bounds)
       ├─ validate_by_mempool            (duplicate hash / nonce-gap check only)
       └─ skip_stateful_validations      ← decides whether __validate__ runs
  └─ run_validate_entry_point(skip_validate)
```

### Impact Explanation

An attacker who observes a `deploy_account` transaction for address X in the mempool can immediately submit an Invoke with `sender_address = X`, `nonce = 1`, and a completely fabricated signature. All three skip conditions are satisfied:

| Condition | Value |
|---|---|
| `tx.nonce() == 1` | ✓ attacker sets it |
| `account_nonce == 0` | ✓ account not yet deployed |
| `account_tx_in_pool_or_recent_block(X)` | ✓ victim's deploy_account is there |

The gateway admits the transaction to the mempool without ever running `__validate__`. The broken invariant is: *every Invoke transaction admitted to the mempool must have its signature verified by the account's `__validate__` entry point*. This matches the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

Secondary consequence: if the attacker's transaction occupies the (address, nonce=1) slot in the mempool before the victim's legitimate Invoke, the victim's transaction may be rejected as a duplicate nonce or displaced by fee-escalation logic, causing the victim's legitimate post-deploy Invoke to be dropped.

### Likelihood Explanation

- The target account address is fully deterministic from the public `deploy_account` fields (`class_hash`, `contract_address_salt`, `constructor_calldata`), so the attacker can compute it before the victim even submits.
- The attacker only needs to watch the public mempool for a `deploy_account` and race-submit an Invoke with `nonce=1` — no privileged access required.
- The nonce-range check (`account_nonce=0 ≤ tx_nonce=1 ≤ max_allowed_nonce_gap`) passes trivially for any non-zero `max_allowed_nonce_gap`. [6](#0-5) 

### Recommendation

**Short term:** In `skip_stateful_validations`, additionally verify that the incoming Invoke's `sender_address` matches the `contract_address` stored inside the pending `deploy_account` transaction in the mempool, not merely that *some* transaction from that address exists. This prevents an attacker from piggybacking on a victim's `deploy_account`.

**Long term:** Consider requiring a lightweight off-chain signature check (e.g., verifying the transaction hash against the supplied signature using the declared class's public key) before skipping the on-chain `__validate__` call, so that the UX bypass cannot be exploited by parties who do not control the private key for the target address.

### Proof of Concept

1. Alice broadcasts `deploy_account` for address `X` (class_hash `C`, salt `S`). The mempool now contains Alice's transaction; `account_tx_in_pool_or_recent_block(X)` returns `true`.

2. Eve computes `X = calculate_contract_address(C, S, constructor_calldata)` (public information).

3. Eve submits an `RpcInvokeTransactionV3` with:
   - `sender_address = X`
   - `nonce = 1`
   - `signature = [0xdead, 0xbeef]` (arbitrary garbage)
   - `calldata` = anything

4. Gateway stateful path:
   - `get_nonce_from_state(X)` → `Nonce(0)` (not deployed yet)
   - `validate_nonce`: `0 ≤ 1 ≤ max_allowed_nonce_gap` → passes
   - `validate_by_mempool`: no duplicate hash → passes
   - `skip_stateful_validations`: `nonce==1 && account_nonce==0 && account_tx_in_pool(X)==true` → returns `true`
   - `run_validate_entry_point(skip_validate=true)`: sets `validate=false`, `StatefulValidator` returns `Ok(())` without calling `__validate__`

5. Eve's transaction is admitted to the mempool with an invalid signature, violating the admission invariant. [7](#0-6) [8](#0-7) [9](#0-8)

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
```

```rust
    pub fn perform_validations(&mut self, tx: AccountTransaction) -> StatefulValidatorResult<()> {
        // Deploy account transaction should be fully executed, since the constructor must run
        // before `__validate_deploy__`. The execution already includes all necessary validations,
        // so they are skipped here.
        // Declare transaction should also be fully executed - otherwise, if we only go through
        // the validate phase, we would miss the check that the class was not declared before.
        match tx.tx {
            ApiTransaction::DeployAccount(_) | ApiTransaction::Declare(_) => self.execute(tx),
            ApiTransaction::Invoke(_) => {
                let tx_context = Arc::new(self.tx_executor.block_context.to_tx_context(&tx));
                tx.perform_pre_validation_stage(self.state(), &tx_context)?;
                if !tx.execution_flags.validate {
                    return Ok(());
                }

                // `__validate__` call.
                let (_optional_call_info, actual_cost) = self.validate(&tx, tx_context.clone())?;

                // Post validations.
                PostValidationReport::verify(
                    &tx_context,
                    &actual_cost,
                    tx.execution_flags.charge_fee,
                )?;

                Ok(())
            }
        }
    }
```
