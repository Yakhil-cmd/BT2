### Title
Protocol Has No Escape Hatch from Admin-Level Pause: Lost Pool Admin Key Permanently Disables Swaps — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

The three-level pause state machine enforces a strict transition graph where only the pool admin can clear pause level 1 (admin pause) back to level 0 (active). The protocol owner (factory `onlyOwner`) has no path to restore a pool from level 1 to level 0. If the pool admin loses key access while the pool is at pause level 1, all swaps are permanently disabled with no on-chain recovery mechanism.

---

### Finding Description

The pause state machine in `MetricOmmPoolFactory` defines three levels and enforces the following transitions:

| Caller | Function | Allowed transition |
|---|---|---|
| Pool admin | `pausePool` | `0 → 1` |
| Pool admin | `unpausePool` | `1 → 0` |
| Protocol owner | `protocolPausePool` | `0 or 1 → 2` |
| Protocol owner | `protocolUnpausePool` | `2 → 1` |

`unpausePool` is gated exclusively by `onlyPoolAdmin`:

```solidity
function unpausePool(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 1) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
}
``` [1](#0-0) 

`protocolUnpausePool` can only transition from level 2 to level 1, never to level 0:

```solidity
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1);
}
``` [2](#0-1) 

The protocol owner is therefore trapped in a `1 → 2 → 1 → 2 → …` loop and can never restore the pool to active state without the pool admin's cooperation.

The pool's `swap` function enforces `whenNotPaused`, which reverts on any non-zero `pauseLevel`:

```solidity
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
``` [3](#0-2) 

The `swap` entry point applies this check:

```solidity
function swap(...) external whenNotPaused nonReentrant(PoolActions.SWAP) returns (int128, int128) {
``` [4](#0-3) 

The existing test suite explicitly confirms that after a protocol pause-then-unpause cycle, the pool lands at level 1 and requires the pool admin to complete the final step:

```solidity
function test_protocolPausePool_fromZero_skipsAdminLevel() public {
    address pool = _createPool();
    factory.protocolPausePool(pool);   // 0 → 2
    factory.protocolUnpausePool(pool); // 2 → 1
    vm.prank(admin);
    factory.unpausePool(pool);         // 1 → 0  ← only admin can do this
    assertEq(_pauseLevel(pool), 0);
}
``` [5](#0-4) 

There is no `forceUnpause`, emergency override, or time-based fallback anywhere in the factory or pool.

---

### Impact Explanation

If the pool admin's key is lost or the admin account is compromised/bricked while `pauseLevel == 1`:

- All calls to `swap` revert with `PoolPaused` permanently.
- The protocol owner cannot restore the pool to active state.
- The pool's core trading functionality is permanently broken.

`removeLiquidity` is not blocked by the pause check (intentional per the comment: *"liquidity mutators are intentionally not blocked by the same check"*), so LP principal is not directly lost — LPs can still withdraw. [6](#0-5) 

However, the pool is permanently unusable for its primary purpose (swapping), which constitutes broken core pool functionality under the contest's allowed impact gate.

---

### Likelihood Explanation

The trigger requires two conditions:

1. The pool admin calls `pausePool` (a routine operational action during incidents or upgrades).
2. The pool admin subsequently loses access to their key (key loss, multisig threshold failure, smart-contract wallet upgrade gone wrong, etc.).

Alternatively, the protocol owner calls `protocolPausePool` on an already-active pool (level 0 → 2), then `protocolUnpausePool` (2 → 1), and the pool admin is unavailable to complete the final step. This second path requires no admin error at all — it is a normal protocol operation that leaves the pool in a state only the admin can exit.

Pool admin is described as "semi-trusted" and is a per-pool EOA or multisig, making key loss a realistic operational risk.

---

### Recommendation

Add a protocol-level emergency override that allows the factory owner to force-clear any pause level directly to 0, bypassing the layered transition rules. For example:

```solidity
/// @notice Emergency: protocol owner can force-clear any pause level to 0.
function protocolForceUnpausePool(address pool) external onlyOwner {
    IMetricOmmPoolFactoryActions(pool).setPause(0);
    emit PoolForcedUnpaused(pool);
}
```

Alternatively, add a time-based escape hatch: if the pool has been at pause level 1 for longer than a governance-defined period (e.g., 30 days), allow the protocol owner to transition directly to level 0.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

// Scenario: pool admin pauses, then loses key access.
// Protocol owner is unable to restore the pool.

function test_permanentPause_adminKeyLoss() public {
    address pool = _createPool();

    // Step 1: Pool admin pauses the pool (routine operation)
    vm.prank(admin);
    factory.pausePool(pool);
    assertEq(_pauseLevel(pool), 1); // paused at level 1

    // Step 2: Pool admin loses key access — simulate by using a different address
    // Protocol owner attempts to restore the pool to active
    // Only option: escalate to level 2 then back to level 1 (no progress)
    factory.protocolPausePool(pool);   // 1 → 2
    assertEq(_pauseLevel(pool), 2);

    factory.protocolUnpausePool(pool); // 2 → 1 (back to admin-paused, no improvement)
    assertEq(_pauseLevel(pool), 1);

    // Protocol owner cannot go 1 → 0 directly
    vm.expectRevert(); // InvalidPauseTransition(1, 0) — only admin can do this
    factory.unpausePool(pool); // factory owner is NOT the pool admin

    // Pool is permanently stuck at level 1; all swaps revert
    vm.expectRevert(IMetricOmmPoolActions.PoolPaused.selector);
    _swap(SWAPPER_INDEX, swapper, false, 1000, type(uint128).max);
}
```
