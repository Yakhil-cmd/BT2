### Title
Incomplete Method Dispatch in `RelayerContract.Run` Permanently Blocks `registerPayee`, `registerCounterpartyPayee`, and `submitMisbehaviour` Precompile Calls — (`File: x/cronos/keeper/precompiles/relayer.go`)

---

### Summary

The `RelayerContract.Run` function in the Cronos relayer precompile contains a `switch` statement that acts as a method-type whitelist. Three methods — `RegisterPayee`, `RegisterCounterpartyPayee`, and `SubmitMisbehaviour` — are defined as named constants, are present in the ABI (and therefore appear in the gas table populated from the ABI), but are **absent from the `Run` dispatch switch**. Any call to these methods always falls through to the `default` branch and returns `"unknown method"`, permanently blocking their execution through the precompile.

---

### Finding Description

In `x/cronos/keeper/precompiles/relayer.go`, three method name constants are declared: [1](#0-0) 

`SubmitMisbehaviour`, `RegisterPayee`, and `RegisterCounterpartyPayee` are all defined. The `init()` function populates `relayerGasRequiredByMethod` by iterating over **all** methods in `irelayerABI.Methods`: [2](#0-1) 

`RegisterPayee` and `RegisterCounterpartyPayee` receive explicit gas entries (38000 and 37000 respectively); `SubmitMisbehaviour` falls into the `default` gas bucket. All three are therefore reachable through `RequiredGas`.

However, the `Run` dispatch switch only handles 17 methods and terminates with: [3](#0-2) 

`RegisterPayee`, `RegisterCounterpartyPayee`, and `SubmitMisbehaviour` are **not present** in the switch. Every call to these methods succeeds the ABI lookup (`irelayerABI.MethodById` returns no error because the method is in the ABI), charges gas via `RequiredGas`, then hits `default: return nil, fmt.Errorf("unknown method: %s", method.Name)` and reverts.

This is structurally identical to the external report: a base dispatch table (the `Run` switch) is the effective whitelist, and it is incomplete — the extended/missing cases are never handled.

---

### Impact Explanation

**High** — Permanent inability for honest users and EVM contracts to execute valid precompile calls:

- `registerPayee` / `registerCounterpartyPayee`: IBC fee-middleware functions that allow relayers to register payee addresses for fee distribution. Any EVM contract or EOA invoking these through the precompile will always revert, making the IBC fee-middleware path through the precompile permanently non-functional.
- `submitMisbehaviour`: Allows submission of light-client misbehavior evidence. Blocking this through the precompile prevents EVM-side contracts from triggering misbehavior reporting, which has direct security implications for IBC client integrity.

Gas is consumed (charged by `RequiredGas`) before the revert, so callers also lose gas on every failed attempt.

---

### Likelihood Explanation

Any unprivileged user or EVM contract calling `registerPayee`, `registerCounterpartyPayee`, or `submitMisbehaviour` on the relayer precompile (address `0x65`) triggers this. No special privileges are required. The failure is deterministic and 100% reproducible.

---

### Recommendation

Add the missing cases to the `Run` switch statement in `x/cronos/keeper/precompiles/relayer.go`, dispatching to the corresponding `ibcKeeper` methods (e.g., `bc.ibcKeeper.RegisterPayee`, `bc.ibcKeeper.RegisterCounterpartyPayee`, `bc.ibcKeeper.SubmitMisbehaviour`), consistent with how all other methods are handled. Alternatively, if any of these methods are intentionally unsupported, remove them from the ABI and the gas table to prevent misleading callers.

---

### Proof of Concept

1. Deploy any EVM contract (or use an EOA) on Cronos.
2. ABI-encode a call to `registerPayee(bytes)` with a valid `MsgRegisterPayee` protobuf payload.
3. Call the relayer precompile at address `0x65` with this calldata.
4. `RequiredGas` returns 38000 (method is in the gas table).
5. `Run` is entered; `irelayerABI.MethodById` succeeds (method is in the ABI).
6. The switch hits `default` → `return nil, fmt.Errorf("unknown method: registerPayee")`.
7. The transaction reverts. Repeat identically for `registerCounterpartyPayee` and `submitMisbehaviour`. [4](#0-3)
