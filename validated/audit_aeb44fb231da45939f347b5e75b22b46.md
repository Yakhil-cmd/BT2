### Title
Global `last_reward_block` Lock Allows Any Caller to Grief Per-Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in `StakingRewardsManagerImpl` uses a single global `last_reward_block` storage variable to enforce a one-call-per-block limit. Because the function is callable by any unprivileged address and accepts any valid `staker_address` as a parameter, an attacker can call it with `disable_rewards: true` (or for their own staker) in any block, consuming the global slot and causing every other legitimate `update_rewards` call in that block to revert with `REWARDS_ALREADY_UPDATED`. Since missed blocks are never retroactively compensated, this permanently destroys unclaimed yield for all stakers.

### Finding Description

`update_rewards` is the sole mechanism for distributing per-block consensus rewards to stakers. Its implementation enforces a global, contract-wide one-call-per-block invariant:

```cairo
// src/staking/staking.cairo  lines 1453-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After passing this check, the function unconditionally writes the current block number to the global slot before any early-return path:

```cairo
// src/staking/staking.cairo  line 1485
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;   // returns WITHOUT distributing rewards
}
```

There is no access control on `update_rewards`. Any caller may invoke it with any valid `staker_address`. The function only requires that the supplied `staker_address` is an active staker with non-zero balance — information that is entirely public on-chain.

An attacker therefore:
1. Picks any existing, active staker address (publicly readable).
2. Calls `update_rewards(staker_address: <victim_or_any_valid_staker>, disable_rewards: true)` in block N.
3. The call passes all assertions, writes `last_reward_block = N`, and returns early — distributing **zero** rewards.
4. Every subsequent `update_rewards` call in block N reverts with `REWARDS_ALREADY_UPDATED`.

Because the reward model is strictly per-block (each block's rewards are calculated and credited only when `update_rewards` is called in that block), any block for which the slot is consumed by the attacker results in **permanent, irrecoverable loss** of that block's rewards for all stakers and their delegators.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

The flow test `test_update_rewards_miss_blocks_flow_test` explicitly confirms that missed blocks are never compensated: a staker that misses a block's `update_rewards` call earns `(epoch_length - 1) * block_rewards` instead of `epoch_length * block_rewards`, and the difference is gone forever. An attacker who calls `update_rewards` with `disable_rewards: true` every block can reduce every staker's yield to zero at the cost of only gas fees, with no profit motive required.

### Likelihood Explanation

**High.** The entry point is fully public, requires no privileged role, no token balance, and no special setup beyond knowing a valid staker address (trivially obtained from on-chain events). The attacker pays only transaction fees. On Starknet the sequencer orders transactions within a block, so the attacker simply needs to submit their griefing transaction in the same block as a legitimate `update_rewards` call, or submit it first in every block to suppress all reward accrual system-wide.

### Recommendation

Replace the single global `last_reward_block` with a **per-staker** mapping, e.g. `last_reward_block: Map<ContractAddress, BlockNumber>`. The check and write become:

```cairo
assert!(
    current_block_number > self.last_reward_block.entry(staker_address).read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
// ...
self.last_reward_block.entry(staker_address).write(current_block_number);
```

This ensures that a call for staker A cannot block a call for staker B in the same block, eliminating the griefing surface entirely.

### Proof of Concept

1. Staker Alice is active with non-zero balance.
2. Attacker (any address) calls `update_rewards(staker_address: Alice, disable_rewards: true)` in block N.
   - `current_block_number (N) > last_reward_block (N-1)` → passes.
   - `last_reward_block` is written to N.
   - `disable_rewards == true` → returns early, no rewards credited.
3. Alice's node operator (or any legitimate caller) calls `update_rewards(staker_address: Alice, disable_rewards: false)` in block N.
   - `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
4. Alice permanently loses block N's rewards.
5. Repeating step 2 every block reduces Alice's (and every other staker's) annual yield to zero. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
