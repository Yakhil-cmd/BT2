### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is a public function with no caller restriction. Any address can invoke it with `disable_rewards: true`, which writes the current block number into the global `last_reward_block` storage slot and returns without distributing rewards. Because `last_reward_block` is a single global value, this permanently consumes the reward slot for that block — no subsequent call in the same block can distribute rewards. An attacker repeating this every block permanently freezes all staker and delegator yield.

### Finding Description
The spec explicitly states the access control for `update_rewards` is **"Only starkware sequencer"**: [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` contains **no caller check whatsoever**: [2](#0-1) 

The function accepts two attacker-controlled parameters: `staker_address: ContractAddress` and `disable_rewards: bool`. After validating that the staker exists and is active (which any attacker can satisfy by picking any live staker), it unconditionally writes the current block number to the **global** `last_reward_block` storage variable: [3](#0-2) 

`last_reward_block` is a single global slot, not per-staker: [4](#0-3) 

When `disable_rewards: true` is passed, the function returns immediately after writing `last_reward_block`, skipping all reward calculation and distribution. Any legitimate call to `update_rewards` in the same block then fails with `REWARDS_ALREADY_UPDATED` because the block number is no longer greater than `last_reward_block`. The rewards for that block are permanently lost — there is no catch-up or recovery path.

The interface definition confirms there is no access-control annotation: [5](#0-4) 

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

Each block's rewards are computed once and only once. If `update_rewards` is called with `disable_rewards: true` before the legitimate sequencer call, the block's reward slot is consumed with zero distribution. The staker's `unclaimed_rewards_own` is never incremented, pool reward indices are never updated, and the reward supplier's `unclaimed_rewards` is never increased for that block. This loss is irreversible. An attacker calling this function every block causes 100% of all staker and delegator yield to be permanently frozen.

### Likelihood Explanation
**High.** The function is fully public, costs only a standard transaction fee, and requires no special setup beyond knowing any active staker address (which is publicly observable on-chain). The attacker needs no stake, no privileged role, and no capital at risk. The attack is trivially automatable: a bot monitors the mempool or block production and front-runs the sequencer's `update_rewards` call each block with `disable_rewards: true`.

### Recommendation
Add a caller check at the top of `update_rewards` enforcing that only the authorized sequencer (or a designated role) may call it, matching the spec's stated access control:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Add: assert caller is the authorized sequencer
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

Alternatively, restrict via the existing roles component (e.g., a `REWARDS_MANAGER_ROLE`) consistent with how other privileged functions are gated in the contract.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Staker `S` is registered and active with non-zero balance.
3. Attacker `A` (any EOA) calls:
   ```
   update_rewards(staker_address: S, disable_rewards: true)
   ```
   in block `N`.
4. Inside the function:
   - `current_block_number (N) > last_reward_block` — passes.
   - Staker `S` exists and is active — passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → early return, zero rewards distributed.
5. The legitimate sequencer now calls `update_rewards(staker_address: S, disable_rewards: false)` in the same block `N`.
6. The check `current_block_number (N) > last_reward_block (N)` **fails** → reverts with `REWARDS_ALREADY_UPDATED`.
7. Block `N`'s rewards are permanently lost for all stakers and delegators.
8. Repeating steps 3–7 every block freezes 100% of protocol yield indefinitely. [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
