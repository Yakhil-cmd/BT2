### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Block Consensus Reward Distribution — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `staking.cairo` is specified to be callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged caller can invoke it with `disable_rewards: true` for any valid staker, consuming the block's reward slot without distributing rewards. The legitimate sequencer's subsequent call in the same block then reverts with `REWARDS_ALREADY_UPDATED`, causing stakers to permanently lose block rewards.

---

### Finding Description

The `update_rewards` function is the consensus-era reward distribution entry point. Its spec explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation only calls `general_prerequisites()`, which checks for pause state and a non-zero caller — no role check is performed:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only: assert_is_unpaused + assert_caller_is_not_zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;   // <-- returns here with no rewards distributed
    }
    ...
``` [1](#0-0) 

`general_prerequisites` is:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [2](#0-1) 

The `last_reward_block` storage variable is a **single global value** shared across all stakers: [3](#0-2) 

An attacker who calls `update_rewards(valid_staker_address, disable_rewards: true)`:

1. Passes all checks (staker exists, is active, has non-zero balance at current epoch).
2. Writes `last_reward_block = current_block_number`.
3. Returns early — **no rewards are distributed**.

The legitimate sequencer's call in the same block then hits:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

and reverts. Because `last_reward_block` is global, **one attacker call per block blocks reward distribution for all stakers in that block**.

The spec confirms the intended access control restriction: [5](#0-4) 

---

### Impact Explanation

- **Per block**: every staker misses their consensus block rewards — theft of unclaimed yield.
- **Sustained (every block)**: stakers permanently receive zero consensus rewards — permanent freezing of unclaimed yield.
- Both map to **High** in the allowed impact scope: *"Theft of unclaimed yield"* and *"Permanent freezing of unclaimed yield"*.

The attacker has no profit motive but can inflict severe, sustained damage to all stakers and delegators at the cost of gas fees only.

---

### Likelihood Explanation

- Any non-zero address can call `update_rewards` — no role, no key, no privilege required.
- Valid staker addresses are publicly discoverable from on-chain `NewStaker` events.
- The attacker must call once per block to sustain the attack; this is cheap on Starknet.
- No external dependency or third-party compromise is needed.

---

### Recommendation

Add a sequencer-only access control guard at the top of `update_rewards`, consistent with the spec:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer();   // <-- add this
    self.general_prerequisites();
    ...
```

---

### Proof of Concept

1. Staker stakes and becomes active (K epochs pass so balance is non-zero at current epoch).
2. At the start of every block, attacker calls `update_rewards(staker_address, disable_rewards: true)`.
3. `last_reward_block` is set to the current block number; no rewards are distributed.
4. The legitimate sequencer's `update_rewards` call reverts with `REWARDS_ALREADY_UPDATED`.
5. Stakers and their delegators receive zero consensus rewards for every block the attacker front-runs.

**Analog mapping to the external report:**

| HatsSignerGate | Starknet Staking |
|---|---|
| `signerCount` read without calling `reconcileSignerCount` first | `last_reward_block` written by attacker without distributing rewards |
| Stale `signerCount` allows extra signer to be added | Stale `last_reward_block` blocks legitimate reward distribution |
| `reconcileSignerCount` reverts: `validSignerCount > maxSigners` | `update_rewards` reverts: `current_block_number <= last_reward_block` |
| Valid signers lose their signer hat | Stakers lose all consensus block rewards |
