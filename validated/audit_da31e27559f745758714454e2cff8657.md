### Title
Pool Contract `claim_rewards` Bypasses Staking Contract Emergency Pause - (File: src/pool/pool.cairo)

### Summary

The `Pool` contract's `claim_rewards` function has no pause check and does not call the staking contract, allowing pool members to claim unclaimed yield even when the staking contract is paused. This is inconsistent with the staking contract's own `claim_rewards`, which is explicitly blocked during a pause.

### Finding Description

The staking contract (`src/staking/staking.cairo`) implements a pause mechanism via `is_paused` storage and `StakingPauseImpl`. The pause test suite confirms that `claim_rewards` on the staking contract reverts with `"Contract is paused"` when the contract is paused. [1](#0-0) 

The staking contract's `pause` and `unpause` functions are restricted to `SECURITY_AGENT` and `SECURITY_ADMIN` roles respectively. [2](#0-1) 

However, the `Pool` contract has **no pause mechanism of its own** and its `claim_rewards` function does not call the staking contract at any point. It reads directly from the pool's own `cumulative_rewards_trace` storage and transfers STRK tokens directly from the pool contract's balance to the reward address: [3](#0-2) 

Contrast this with `exit_delegation_pool_action` and `exit_delegation_pool_intent`, which call `remove_from_delegation_pool_action` / `remove_from_delegation_pool_intent` on the staking contract and are therefore indirectly blocked by the staking contract's pause. `claim_rewards` makes no such call and is entirely self-contained within the pool contract. [4](#0-3) 

The pool contract's `Storage` struct contains no `is_paused` field and no reference to the staking contract's pause state. [5](#0-4) 

The pool contract's `IPool` interface for `claim_rewards` lists no `CONTRACT_IS_PAUSED` error, confirming the omission is not guarded at the interface level either. [6](#0-5) 

### Impact Explanation

When the `SECURITY_AGENT` pauses the staking contract in an emergency (e.g., a reward calculation bug, an exploit in progress, or a governance incident), the intent is to freeze all reward flows. Stakers cannot call `claim_rewards` on the staking contract. However, pool members can still call `claim_rewards` on any pool contract and receive STRK tokens from the pool's existing reward balance. The pool contract accumulates STRK rewards via `update_rewards_from_staking_contract` (which is itself paused, so no new rewards flow in), but the existing balance already held by the pool contract is fully drainable by pool members during the pause window.

This maps to **High: Theft of unclaimed yield** — pool members can extract unclaimed yield that the security pause was intended to freeze. [7](#0-6) 

### Likelihood Explanation

- Any pool member (unprivileged delegator) can call `claim_rewards` on the pool contract at any time.
- No special role, leaked key, or privileged access is required.
- The only precondition is that the pool member exists and has accrued rewards — a normal operational state.
- The attack path is a single direct call to `Pool::claim_rewards` while the staking contract is paused. [8](#0-7) 

### Recommendation

The pool contract's `claim_rewards` (and `change_reward_address`, which also makes no staking contract call) should query the staking contract's pause state before proceeding. Add a check at the top of `claim_rewards`:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
    // ... rest of function
}
```

This mirrors the pattern already used in `get_current_epoch` and `get_commission_from_staking_contract`, which already call back into the staking contract. [9](#0-8) 

### Proof of Concept

1. Security agent calls `staking.pause()` due to an emergency.
2. Staking contract is now paused; `staking.claim_rewards(staker)` reverts.
3. Pool member calls `pool.claim_rewards(pool_member)` directly on the pool contract.
4. The pool contract reads its own `cumulative_rewards_trace`, computes rewards, and calls `STRK_TOKEN.transfer(reward_address, rewards)` — no staking contract interaction occurs.
5. Pool member successfully receives STRK rewards despite the emergency pause, draining the pool contract's reward balance. [10](#0-9)

    impl StakingPauseImpl of IStakingPause<ContractState> {
        fn pause(ref self: ContractState) {
            self.roles.only_security_agent();
            if self.is_paused() {
                return;
            }
            self.is_paused.write(true);
            self.emit(PauseEvents::Paused { account: get_caller_address() });
        }

        fn unpause(ref self: ContractState) {
            self.roles.only_security_admin();
            if !self.is_paused() {
                return;
            }
            self.is_paused.write(false);
            self.emit(PauseEvents::Unpaused { account: get_caller_address() });
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
