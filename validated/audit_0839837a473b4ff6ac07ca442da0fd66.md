### Title
Signature Validation Bypass via `skip_stateful_validations` Allows Unsigned Invoke Transactions to Enter Mempool — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator unconditionally skips the `__validate__` entry-point call for any Invoke V3 transaction with `nonce == 1` when the sender's on-chain nonce is `0` and the mempool reports any transaction for that address. An unprivileged attacker who observes a victim's `deploy_account` transaction in the mempool can immediately submit a second Invoke transaction from the victim's address carrying an arbitrary (invalid) signature. The gateway accepts it without ever verifying the signature, admitting an invalid transaction into the mempool and causing the victim's account to be charged a fee for the reverted execution.

### Finding Description

The `skip_stateful_validations` helper is called inside `run_pre_validation_checks` and returns `true` (skip) when all three conditions hold:

1. The transaction is an `Invoke`.
2. `tx.nonce() == Nonce(Felt::ONE)`.
3. `account_nonce == Nonce(Felt::ZERO)`.
4. `mempool_client.account_tx_in_pool_or_recent_block(sender)` returns `true`. [1](#0-0) 

When `skip_stateful_validations` returns `true`, `run_validate_entry_point` is called with `skip_validate = true`, which sets `ExecutionFlags { validate: false, … }` on the `AccountTransaction` passed to the blockifier's `StatefulValidator`. [2](#0-1) 

Inside `StatefulValidator::perform_validations`, when `execution_flags.validate == false`, the function returns immediately after `perform_pre_validation_stage` without ever calling `tx.validate_tx(…)` (the `__validate__` entry point): [3](#0-2) 

`perform_pre_validation_stage` only checks the nonce (non-strict: `account_nonce <= tx_nonce`) and fee bounds; it never inspects the signature bytes. [4](#0-3) 

The stateless validator only checks the *length* of the signature, not its cryptographic validity: [5](#0-4) 

The `account_tx_in_pool_or_recent_block` check that gates the skip is satisfied by the *victim's own* `deploy_account` transaction already sitting in the pool: [6](#0-5) 

### Impact Explanation

**High — Mempool/gateway admission accepts an invalid transaction before sequencing.**

An attacker can inject an Invoke transaction bearing a completely fabricated signature into the mempool for any address whose `deploy_account` is pending. When the batcher later executes the block, the `deploy_account` runs first (nonce 0), deploying the account; then the Invoke runs (nonce 1), calls `__validate__`, fails signature verification, reverts, and charges the victim's account a fee. The attacker pays nothing; the victim loses funds to a transaction they never signed.

### Likelihood Explanation

**Medium.** The mempool is observable (transactions are broadcast). Any address that follows the common UX pattern of submitting `deploy_account` + `invoke` together is vulnerable during the window between mempool admission and block finalization. No privileged access is required; the attacker only needs to know the victim's sender address (visible in the `deploy_account` transaction) and that the address has been pre-funded (a prerequisite for the deploy flow itself).

### Recommendation

Remove the unconditional signature-skip for the deploy-account UX path, or restrict it so that only the *same sender* that submitted the `deploy_account` can benefit from the skip (e.g., by binding the skip to the `deploy_account` transaction hash rather than a bare mempool-presence check). At minimum, add a check inside `skip_stateful_validations` that the incoming Invoke transaction's `tx_hash` was submitted by the same key that signed the pending `deploy_account`.

### Proof of Concept

```
1. Victim broadcasts:
     deploy_account { sender: A, nonce: 0, signature: <valid> }

2. Attacker observes A in the mempool.

3. Attacker broadcasts:
     invoke { sender: A, nonce: 1, calldata: <anything>, signature: [0x0, 0x0] }

4. Gateway stateful validator:
     account_nonce(A) == 0  ✓
     tx.nonce()       == 1  ✓
     account_tx_in_pool_or_recent_block(A) == true  ✓
     → skip_stateful_validations returns true
     → run_validate_entry_point called with skip_validate=true
     → __validate__ is NEVER called
     → gateway returns Ok, invoke tx enters mempool

5. Batcher executes block:
     deploy_account(A, nonce=0) → account deployed
     invoke(A, nonce=1)         → __validate__ called → FAIL (bad sig) → reverted
     → A is charged fee for the reverted invoke

6. Victim's account loses fee for a transaction they never signed.
``` [1](#0-0) [7](#0-6)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

        &self,
        state: &mut S,

        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;
