### Title
`protocolUnpausePool` Always Transitions to Pause Level 1 Regardless of Pre-Pause State, Leaving Originally-Active Pools Stuck in Admin-Paused State - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

---

### Summary

`protocolUnpausePool` unconditionally calls `setPause(1)` after removing the protocol's pause (level 2). When the pool was originally active (level 0) before the protocol paused it, the pool is left in admin-paused state (level 1) — a state the pool admin never set — blocking all swaps until the admin takes a separate, unrelated action.

---

### Finding Description

The factory defines three pause levels:

- `0` — active (swaps allowed)
- `1` — paused by admin (swaps blocked)
- `2` — paused by protocol (swaps blocked)

`protocolPausePool` accepts pools at **either** level 0 or level 1:

```solidity
// MetricOmmPoolFactory.sol line 394
if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
IMetricOmmPoolFactoryActions(pool).setPause(2);
```

`protocolUnpausePool` always restores to level **1**, never to level 0:

```solidity
// MetricOmmPoolFactory.sol line 401
if (cur != 2) revert InvalidPauseTransition(cur, 1);
IMetricOmmPoolFactoryActions(pool).setPause(1);
```

The original pre-pause state is never recorded. When the pool was at level 0 (active) before the protocol paused it, `protocolUnpausePool` leaves it at level 1 (admin-paused) — a state the admin never chose. The pool's `_checkNotPaused` guard blocks swaps at any non-zero level:

```solidity
// MetricOmmPool.sol line 643-645
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
```

The admin can recover by calling `unpausePool` (level 1 → 0), but only if they are aware the pool is stuck and have the operational capacity to act. If the admin is a slow multisig, a DAO, or a contract without an `unpausePool` call path, the pool remains permanently swap-disabled.

---

### Impact Explanation

All `swap` calls revert with `PoolPaused` for the entire period between `protocolUnpausePool` and the admin's subsequent `unpausePool`. `addLiquidity` and `removeLiquidity` are not gated by `whenNotPaused`, so LP positions can be modified but cannot be traded against. This constitutes an **unusable swap flow** — a broken core pool functionality — matching the allowed impact gate.

---

### Likelihood Explanation

The trigger is any sequence where:
1. The pool is active (level 0), and
2. The protocol owner calls `protocolPausePool` followed by `protocolUnpausePool`.

This is a routine operational pattern (e.g., emergency pause during an oracle incident, then resumption). The protocol owner is semi-trusted and acts in good faith; the broken invariant is a silent implementation error, not a malicious act. Likelihood is **medium-high**.

---

### Recommendation

Record the pre-pause level in factory storage when `protocolPausePool` is called, and restore it in `protocolUnpausePool`:

```solidity
mapping(address => uint8) public prePausePauseLevel; // new storage

function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    prePausePauseLevel[pool] = cur;          // save original level
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}

function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    uint8 restore = prePausePauseLevel[pool]; // 0 or 1
    delete prePausePauseLevel[pool];
    IMetricOmmPoolFactoryActions(pool).setPause(restore);
}
```

---

### Proof of Concept

1. Pool is deployed and active — `pauseLevel == 0`.
2. Protocol owner calls `protocolPausePool(pool)`:
   - Guard: `cur == 0` → passes (`cur != 0 && cur != 1` is false).
   - Pool transitions: `0 → 2`.
3. Protocol owner calls `protocolUnpausePool(pool)`:
   - Guard: `cur == 2` → passes.
   - Pool transitions: `2 → 1` (hardcoded).
4. Pool is now at level 1 — **admin-paused** — even though the admin never called `pausePool`.
5. Any user calling `swap(...)` receives `PoolPaused`.
6. Admin must independently discover the state and call `unpausePool(pool)` (level 1 → 0) to restore swap functionality.
7. If the admin is a slow multisig or a contract without this call path, swaps remain permanently blocked. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
