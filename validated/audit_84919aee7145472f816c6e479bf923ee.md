### Title
Missing Input Length Validation in `BankContract.RequiredGas` and `BankContract.Run` Causes Unrecovered Panic — (`File: x/cronos/keeper/precompiles/bank.go`)

---

### Summary

`BankContract.RequiredGas` and `BankContract.Run` both slice `input[:4]` / `contract.Input[:4]` without first checking that the input is at least 4 bytes long. In Go, this is an out-of-bounds slice operation that causes a runtime `panic`. No `recover()` exists in the production precompile execution path. An unprivileged user can send an EVM transaction to the bank precompile address with fewer than 4 bytes of calldata to trigger this panic on every validator, halting consensus.

---

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, both entry points into the bank precompile perform an unconditional 4-byte slice before any length guard:

**`RequiredGas` (line 84):**
```go
func (bc *BankContract) RequiredGas(input []byte) uint64 {
    baseCost := uint64(len(input)) * bc.kvGasConfig.WriteCostPerByte
    var methodID [4]byte
    copy(methodID[:], input[:4])   // ← panics if len(input) < 4
    ...
}
```

**`Run` (line 105):**
```go
func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
    methodID := contract.Input[:4]  // ← panics if len(contract.Input) < 4
    ...
}
```

The sibling precompiles `IcaContract` and `RelayerContract` both guard against this explicitly:

- `IcaContract.RequiredGas` (line 109): `if len(input) < 4 { return baseCost }`
- `IcaContract.Run` (line 125): `if len(contract.Input) < 4 { return nil, errors.New("input too short") }`
- `RelayerContract.RequiredGas` (line 145): `if inputLen < 4 { ... return ... }`
- `RelayerContract.Run` (line 203): `if len(contract.Input) < 4 { return nil, errors.New("input too short") }`

`BankContract` is the only precompile missing these guards. A grep across all production Go files confirms there is no `recover()` in the precompile execution path — only in test files (`app/mempool/manager_test.go`, `app/sim_test.go`). A Go runtime panic from `RequiredGas` or `Run` therefore propagates unhandled up through the EVM call stack and into the block-processing loop.

---

### Impact Explanation

**Critical — Consensus halt triggered by an unprivileged EVM transaction.**

`RequiredGas` is called first (before `Run`) during EVM precompile dispatch. Sending any EVM transaction to the bank precompile address (`0x0000000000000000000000000000000000000064`) with 0–3 bytes of calldata causes `input[:4]` to panic at the `RequiredGas` stage. Because there is no `recover()` in the production execution path, the panic propagates through the block-processing loop. Every validator that processes this transaction crashes identically and deterministically, halting the chain.

---

### Likelihood Explanation

The bank precompile address is a fixed, well-known address (`common.BytesToAddress([]byte{100})`). Any unprivileged account can send an EVM transaction to it with empty calldata at negligible cost. The trigger is a single transaction; no special privileges, keys, or prior state are required.

---

### Recommendation

Add the same length guard that `IcaContract` and `RelayerContract` already use, in both `RequiredGas` and `Run`:

```go
// RequiredGas
func (bc *BankContract) RequiredGas(input []byte) uint64 {
    baseCost := uint64(len(input)) * bc.kvGasConfig.WriteCostPerByte
    if len(input) < 4 {
        return baseCost
    }
    var methodID [4]byte
    copy(methodID[:], input[:4])
    ...
}

// Run
func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
    if len(contract.Input) < 4 {
        return nil, errors.New("input too short")
    }
    methodID := contract.Input[:4]
    ...
}
```

---

### Proof of Concept

1. Obtain any funded EVM account on Cronos.
2. Send an EVM transaction to `0x0000000000000000000000000000000000000064` (bank precompile) with `data = 0x` (empty calldata) or any 1–3 byte payload.
3. The EVM dispatcher calls `BankContract.RequiredGas([]byte{})`.
4. Line 84 evaluates `input[:4]` → Go runtime panic: `runtime error: slice bounds out of range [:4] with length 0`.
5. No `recover()` exists in the production path; the panic propagates through the block-processing loop.
