### Title
Presignature Nonce Material Not Consumed on Rerandomization, Enabling Reuse and Private Key Extraction — (File: `src/ecdsa/ot_based_ecdsa/mod.rs`)

---

### Summary

`RerandomizedPresignOutput::rerandomize_presign` accepts `presignature: &PresignOutput` by shared reference rather than by value. Combined with `Clone` being derived on both `PresignOutput` and `RerandomizedPresignOutput`, the library provides no mechanism to enforce its own documented invariant that "a presignature is never reused." A caller or malicious coordinator can rerandomize the same `PresignOutput` twice with identical arguments — or clone a `RerandomizedPresignOutput` before passing it to `sign()` — producing two ECDSA signatures under the same effective nonce, from which the aggregate private key is algebraically recoverable.

---

### Finding Description

In `src/ecdsa/ot_based_ecdsa/mod.rs`, `RerandomizedPresignOutput::rerandomize_presign` is defined as:

```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // shared reference — original is NOT consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> {
``` [1](#0-0) 

Because the function borrows `presignature` rather than moving it, the caller retains full ownership of the `PresignOutput` after the call and can invoke `rerandomize_presign` again with identical `args`. This produces a second `RerandomizedPresignOutput` encoding the same effective nonce `k_eff = k · δ⁻¹` and the same `r = x_coord(δ · R)`.

Both structs additionally derive `Clone`:

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput { pub big_r: AffinePoint, pub k: Scalar, pub sigma: Scalar }

#[derive(Debug, Clone, Serialize, Deserialize, ZeroizeOnDrop)]
pub struct RerandomizedPresignOutput { pub big_r: AffinePoint, pub k: Scalar, pub sigma: Scalar }
``` [2](#0-1) 

`sign()` correctly takes `presignature: RerandomizedPresignOutput` by value to consume it after one use, but this protection is trivially bypassed by cloning before the call. [3](#0-2) 

The code comment in `presign.rs` explicitly states: *"it's crucial that a presignature is never reused."* The API design directly contradicts this invariant. [4](#0-3) 

---

### Impact Explanation

In ECDSA, if the same effective nonce `k_eff` (and therefore the same `r = x_coord(R_eff)`) is used to sign two distinct messages `m₁` and `m₂`:

```
s₁ = (h₁ + r·x) / k_eff
s₂ = (h₂ + r·x) / k_eff
```

Then:
```
k_eff = (h₁ − h₂) / (s₁ − s₂)
x     = (s₁·k_eff − h₁) / r
```

The aggregate private key `x` is fully recovered from two public signatures. This matches the Critical allowed impact: **Extraction, reconstruction, or disclosure of private signing shares or aggregate secret material.**

---

### Likelihood Explanation

The attack requires control over which presignature is passed to `sign()`. In a realistic deployment, the application layer manages presignature storage and assignment. A malicious coordinator can initiate two signing sessions referencing the same presignature (same `big_r`), and participants — having no library-enforced guard against reuse — will process both. The `Clone` derive makes accidental reuse equally likely without malicious intent (e.g., retry logic after a failed signing attempt, or serialization/deserialization of the presignature for storage).

---

### Recommendation

1. Change `rerandomize_presign` to consume the `PresignOutput` by value, making reuse a compile-time error:
   ```rust
   pub fn rerandomize_presign(
       presignature: PresignOutput,   // moved, not borrowed
       args: &RerandomizationArguments,
   ) -> Result<Self, ProtocolError>
   ```
2. Remove `Clone` from `PresignOutput` and `RerandomizedPresignOutput` so that cloning before `sign()` is also a compile-time error.
3. Apply the same fix to the Robust ECDSA `PresignOutput` and its rerandomization path in `src/ecdsa/robust_ecdsa/`.

let presign_out: PresignOutput = run_presign(participants, me, args);

// Rerandomize twice with the same args → same delta → same r, same k_eff
let rerandomized_1 =
    RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args).unwrap();
let rerandomized_2 =
    RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args).unwrap();

// Sign two different messages with the same effective nonce
let sig1 = run_sign(rerandomized_1, msg_hash_1, ...);
let sig2 = run_sign(rerandomized_2, msg_hash_2, ...);

// Both signatures share r = x_coord(delta * R)
// k_eff = (h1 - h2) / (s1 - s2)
// x     = (s1 * k_eff - h1) / r   ← aggregate private key fully extracted

    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
