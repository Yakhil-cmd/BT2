### Title
`skip_stateful_validations` Bypasses `__validate__` Entry Point for Nonce-1 Invoke Transactions, Allowing Invalid Signatures to Pass Gateway Admission — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful validator contains a UX shortcut (`skip_stateful_validations`) that completely skips the account's `__validate__` entry point for Invoke transactions with `nonce == 1` whenever the sender address appears in the mempool or a recent block. Because the mempool presence check (`account_tx_in_pool_or_recent_block`) accepts *any* prior transaction for that address — not only a `deploy_account` — an attacker can submit an Invoke transaction carrying an arbitrary (invalid) signature and have it admitted to the mempool without any signature verification.

### Finding Description

**Step 1 — Nonce-1 skip path.**

`skip_stateful_validations` returns `true` (skip) when all three conditions hold:

```
tx is Invoke  AND  tx.nonce() == 1  AND  account_nonce == 0
AND  account_tx_in_pool_or_recent_block(sender) == true
``` [1](#0-0) 

**Step 2 — `validate` flag set to `false`.**

When `skip_validate == true`, `run_validate_entry_point` constructs `ExecutionFlags` with `validate: false`: [2](#0-1) 

**Step 3 — `StatefulValidator` returns early without calling `__validate__`.**

Inside `StatefulValidator::perform_validations`, the Invoke branch checks the flag and returns `Ok(())` immediately, never reaching the `validate_tx` call: [3](#0-2) 

**Step 4 — Overly broad mempool-presence check.**

The code comment claims the check is equivalent to "a deploy_account exists," but `account_tx_in_pool_or_recent_block` returns `true` for *any* prior transaction from that address, including future-nonce Invoke transactions that were admitted earlier: [4](#0-3) 

The comment in `skip_stateful_validations` acknowledges this explicitly: [5](#0-4) 

**Broken invariant:** Every Invoke transaction admitted to the mempool must have its account's `__validate__` entry point executed to verify the signature. This invariant is unconditionally broken for any Invoke with `nonce == 1` from an account whose address appears anywhere in the mempool.

### Impact Explanation

An attacker can submit an Invoke transaction with a completely fabricated signature (or zeroed signature) for any account that already has *any* pending transaction in the mempool. The gateway accepts it, the mempool stores it, and it is forwarded toward sequencing. This matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

The transaction will revert during block execution (the OS-level `__validate__` will catch the bad signature), but the admission decision itself is wrong: an unauthenticated transaction from an arbitrary sender is accepted as if it were valid.

### Likelihood Explanation

- Requires only that the target account has *any* transaction already in the mempool (a common condition for active accounts).
- No privileged access is needed; any external caller can submit an RPC transaction.
- The `max_nonce_for_validation_skip` config (default `0x1`) limits the skip to `nonce == 1`, but that is precisely the first post-deploy nonce — the most common slot for a legitimate first Invoke.

### Recommendation

Replace the broad `account_tx_in_pool_or_recent_block` check with a check that specifically confirms a `deploy_account` transaction is pending for the sender. Alternatively, still run `__validate__` even in the skip path (the skip was intended to avoid a *failure* due to the account not being deployed yet, not to skip signature verification entirely). A minimal fix:

```rust
// In skip_stateful_validations: verify the pending tx is a deploy_account,
// not just any tx from that address.
return mempool_client
    .has_pending_deploy_account(tx.sender_address())   // new, narrower query
    .await
    ...
```

Or keep the existing check but do not set `validate: false` — instead, tolerate the expected execution failure (account not yet deployed) and treat it as a skip condition rather than skipping the call entirely.

### Proof of Concept

1. Account `A` has on-chain nonce `0` (not yet deployed).
2. Legitimate user submits a valid `deploy_account` tx for `A` (nonce `0`). It enters the mempool and `account_tx_in_pool_or_recent_block(A)` now returns `true`.
3. Attacker submits an Invoke tx: `sender = A`, `nonce = 1`, `signature = [0x0, 0x0]` (invalid).
4. Gateway calls `skip_stateful_validations`: `tx.nonce() == 1`, `account_nonce == 0`, `account_tx_in_pool_or_recent_block(A) == true` → returns `true`.
5. `run_validate_entry_point` is called with `skip_validate = true` → `validate = false` → `StatefulValidator::perform_validations` returns `Ok(())` without ever calling `__validate__`.
6. The invalid Invoke tx is admitted to the mempool and forwarded to the batcher. [6](#0-5) [7](#0-6)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
