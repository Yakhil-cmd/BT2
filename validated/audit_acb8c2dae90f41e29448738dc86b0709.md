### Title
Malicious Participant at Index > Threshold Causes Unconditional Presign Abort for All Honest Parties — (`src/ecdsa/robust_ecdsa/presign.rs`)

### Summary

A participant whose sorted index falls in the "checked" range (positions `threshold+1` through `2*threshold`) can send an arbitrary `big_r_p` value that fails the exponent interpolation consistency check in Round 3 of `do_presign`. Because every honest party independently runs the same check and the implementation unconditionally returns `ProtocolError::AssertionFailed` on mismatch — with no culprit identification or exclusion mechanism — the entire presign session aborts for all honest parties. The `participants.len() == 2*t+1` hard constraint prevents any retry with a smaller honest subset.

### Finding Description

**Participant ordering.** `ParticipantList::new` sorts participants before storing them. [1](#0-0) 

This means the first `t+1` sorted participants always form the interpolation basis, and participants at sorted positions `t+1` through `2t` are always in the "checked" range.

**The check.** In `do_presign`, after collecting all `big_r_p` values, the code splits the sorted list at `threshold+1` and iterates over the remainder: [2](#0-1) 

For each participant `p` at index `> threshold`, it computes `ExponentInterpolation(R_0, ..., R_t; p)` from the basis and compares it to the received `big_r_p`. A mismatch causes an immediate unconditional abort with `AssertionFailed("Exponent interpolation check failed.")`. There is no culprit identification, no exclusion, and no retry path.

**The hard-size constraint.** The `presign()` entry point enforces `participants.len() == 2*t+1` exactly: [3](#0-2) 

This means honest parties cannot simply restart with the remaining `2t` honest participants — they are one short of the required quorum.

**Attack.** A malicious participant at any sorted index `> threshold` sends `big_r_p = g^(k_p + δ)` for any `δ ≠ 0` (e.g., `g^(k_p + 1)`). Every honest party computes the interpolated value from the honest basis, finds it does not match the received value, and aborts. The malicious party is never identified in the error.

### Impact Explanation

All honest parties permanently lose the ability to complete presign with this participant set. Because the protocol mandates exactly `2t+1` participants and provides no mechanism to identify and exclude the cheater, the session cannot be salvaged. Signing is blocked until a new participant set (excluding the malicious party) is assembled and a fresh presign is initiated — which itself requires out-of-band coordination not provided by the library.

### Likelihood Explanation

The attack requires only that a registered participant (already part of the `2t+1` set) send a single malformed group element. No cryptographic assumption needs to be broken. Any participant at a sorted index `> threshold` can execute this trivially. The sorted-index assignment is deterministic and known to all parties before the protocol starts, so the attacker can predict whether they occupy a "checked" position.

### Recommendation

1. **Identify the cheater before aborting.** When the interpolation check fails for participant `p`, record `p` as the culprit and surface it in the error (e.g., `ProtocolError::MaliciousParticipant(p)`). This allows callers to exclude the identified party and retry.
2. **Consider a complaint/accusation round.** After detecting a mismatch, broadcast the accusation so all honest parties agree on the culprit before aborting, preventing a split-view on who cheated.
3. **Document the abort-on-any-mismatch behavior** explicitly in the API if full robustness is not intended, so callers understand that any participant in the checked range can unilaterally abort the session.

### Proof of Concept

```
Setup: N=5, t=2 → participants sorted as [P0, P1, P2, P3, P4]
       Basis = [P0, P1, P2] (indices 0..=2)
       Checked = [P3, P4]   (indices 3..=4)

Malicious party: P3 (sorted index 3 = threshold+1)

Round 2 send (P3, malicious):
  big_r_p3 = g^(k_p3 + 1)   ← one generator added to the honest value

Round 3 check (every honest party):
  interpolated = ExponentInterpolation(R_P0, R_P1, R_P2; scalar(P3))
               = g^(k_P3)                    ← correct value
  received     = g^(k_P3 + 1)               ← malicious value
  interpolated != received
  → return Err(AssertionFailed("Exponent interpolation check failed."))

Result: all 4 honest parties abort; presign permanently fails for this session.
```
