### Title
Withdrawal Blocked by `require_running` Pause Check Freezes User Funds - (File: `engine/src/contract_methods/connector.rs`)

### Summary
When the Aurora Engine contract is paused via `pause_contract()`, the `withdraw()` function in `connector.rs` unconditionally fails because it calls `require_running()` before any withdrawal logic executes. Users who hold ETH or tokens in Aurora cannot exit the protocol while the contract is paused, resulting in a temporary but complete fund freeze for all users.

### Finding Description
The `require_running()` guard in `contract_methods/mod.rs` returns `ERR_PAUSED` whenever `state.is_paused == true`: [1](#0-0) 

The `is_paused` flag is set to `true` by `pause_contract()`, which is callable by the owner: [2](#0-1) 

The `withdraw()` function — the primary mechanism for users to bridge ETH from Aurora back to Ethereum — calls `require_running()` as its very first operation: [3](#0-2) 

Every other user-facing exit and transfer function is equally blocked: [4](#0-3) [5](#0-4) 

Additionally, `exit_to_near_precompile_callback()` — the callback that finalises EVM-level exits and is responsible for both completing NEAR token transfers and issuing refunds on failure — also calls `require_running()` before any of its logic: [6](#0-5) 

If the contract is paused between a user's initial `submit()` call (which burns EVM tokens and schedules the exit promise) and the execution of this callback, the callback panics before it can either deliver the NEAR tokens or refund the burned EVM tokens, creating a permanent loss scenario for in-flight exits.

The `EngineState` struct confirms `is_paused` is a persistent flag with no automatic expiry: [7](#0-6) 

### Impact Explanation
**High — Temporary (potentially permanent for in-flight exits) freezing of funds.**

- All users with ETH or tokens in Aurora are unable to call `withdraw()`, `ft_transfer()`, or `storage_withdraw()` while the contract is paused.
- For users who initiated an `exit_to_near` EVM transaction that is mid-flight when the pause occurs, the callback failure means their EVM tokens are already burned but neither the NEAR delivery nor the EVM refund executes — a permanent fund loss.

### Likelihood Explanation
The owner can legitimately pause the contract for operational reasons (e.g., during upgrades, incident response). The existence of a separate, granular `pause_precompiles()` mechanism for `EXIT_TO_NEAR` and `EXIT_TO_ETHEREUM` demonstrates that the design anticipates selective pausing of exit paths — yet the contract-level pause additionally blocks the NEAR-native `withdraw()` path with no emergency bypass. Any pause of non-trivial duration affects all users simultaneously. [8](#0-7) [9](#0-8) 

### Recommendation
Remove `require_running()` from `withdraw()`, `storage_withdraw()`, and `exit_to_near_precompile_callback()` (specifically its refund branch), or introduce an `emergency_withdraw` path that bypasses the pause check. The contract-level pause should freeze new deposits and EVM execution, not prevent users from reclaiming funds they already own.

### Proof of Concept
1. Alice bridges 10 ETH into Aurora; her balance is recorded in the engine's EVM state.
2. The owner calls `pause_contract()` → `state.is_paused = true`.
3. Alice calls `withdraw(recipient_address, 10 ETH)`.
4. `require_running(&state::get_state(&io)?)` returns `Err(ERR_PAUSED)`.
5. The transaction reverts. Alice cannot recover her ETH until the owner calls `resume_contract()`.

For the in-flight variant:
1. Alice submits an EVM transaction calling the `exit_to_near` precompile, burning her EVM tokens.
2. The engine schedules `exit_to_near_precompile_callback` as a NEAR promise callback.
3. Before the callback executes, the owner calls `pause_contract()`.
4. The callback hits `require_running()` and panics — neither the NEAR transfer nor the EVM refund executes.
5. Alice's tokens are permanently lost.

pub fn pause_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        require_running(&state::get_state(&io)?)?;
        let authorizer: EngineAuthorizer = engine::get_authorizer(&io);

        if !authorizer.is_authorized(&env.predecessor_account_id()) {
            return Err(b"ERR_UNAUTHORIZED".into());

        let flags = PrecompileFlags::from_bits_truncate(args.paused_mask);
        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.pause_precompiles(flags);
        Ok(())
    })
}

pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

bitflags! {
    /// Wraps unsigned integer where each bit identifies a different precompile.
    #[derive(BorshSerialize, BorshDeserialize, Default)]
    #[borsh(crate = "aurora_engine_types::borsh")]
    pub struct PrecompileFlags: u32 {
        const EXIT_TO_NEAR        = 0b01;
        const EXIT_TO_ETHEREUM    = 0b10;
    }
}

impl PrecompileFlags {
    #[must_use]
    pub fn from_address(address: &Address) -> Option<Self> {
        Some(if address == &exit_to_ethereum::ADDRESS {
            Self::EXIT_TO_ETHEREUM
        } else if address == &exit_to_near::ADDRESS {
            Self::EXIT_TO_NEAR
        } else {
            return None;
        })
    }

    /// Checks if the precompile belonging to the `address` is marked as paused.
    #[must_use]
    pub fn is_paused_by_address(&self, address: &Address) -> bool {
        Self::from_address(address).is_some_and(|precompile_flag| self.contains(precompile_flag))
    }
}
