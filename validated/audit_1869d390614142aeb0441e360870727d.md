### Title
Missing Maximum Commission Guard in `switch_delegation_pool` Enables Staker to Extract Yield from Delegators - (File: `src/pool/pool.cairo`)

---

### Summary

The `switch_delegation_pool` function in `src/pool/pool.cairo` accepts no `max_commission` parameter. A staker who holds an active `CommissionCommitment` can legally raise their commission to `max_commission` at any time. Because the pool-switch path performs no on-chain check against the destination pool's commission, a staker can increase their commission between the moment a delegator initiates the switch and the moment the transaction executes, silently redirecting a larger share of future rewards to themselves.

---

### Finding Description

`switch_delegation_pool` in `src/pool/pool.cairo` (lines 379–429) is the only path a delegator can use to move staked funds from one pool to another without going through the full exit-wait window. Its signature is:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
) -> Amount
``` [1](#0-0) 

The function validates that the caller has an active undelegate intent and that `amount ≤ unpool_amount`, but it never reads or constrains the commission of the destination pool. [2](#0-1) 

Commission is stored in the staking contract and is read at reward-distribution time:

```cairo
let commission = staker_pool_info.commission();
...
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [3](#0-2) 

A staker may raise their commission above its current value only when an active `CommissionCommitment` exists. `set_commission_commitment` lets any staker set `max_commission` up to `COMMISSION_DENOMINATOR` (10 000 = 100 %) for up to one year:

```cairo
fn set_commission_commitment(
    ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
) {
    ...
    assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
    assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
    assert!(
        expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(), ...
    );
``` [4](#0-3) 

Once the commitment is active, `set_commission` allows any value ≤ `max_commission`:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
``` [5](#0-4) 

The same absence of a commission guard exists in `enter_delegation_pool` and `add_to_delegation_pool`. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

A delegator who switches to a pool expecting commission `C` may instead land in a pool charging `max_commission` (up to 100 %). All future rewards accrued in that pool are split at the higher rate. Because the commission is applied at every reward-distribution call and the delegator cannot retroactively recover the over-charged commission, this constitutes **theft of unclaimed yield** — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

The attack requires the staker to have previously called `set_commission_commitment` with a `max_commission` materially higher than their advertised commission. This is a one-time, permissionless setup step. Once the commitment is in place, the staker can raise their commission at any block. On Starknet the sequencer controls ordering, but the staker does not need to front-run in the traditional sense: they can simply raise their commission during the delegator's exit-wait window (up to 12 weeks) before the delegator calls `switch_delegation_pool`. The delegator has no on-chain mechanism to enforce a maximum acceptable commission at execution time.

---

### Recommendation

Add a `max_commission: Commission` parameter to `switch_delegation_pool`, `enter_delegation_pool`, and `add_to_delegation_pool`. Before completing the operation, assert:

```cairo
let actual_commission = self.get_commission_from_staking_contract();
assert!(actual_commission <= max_commission, Error::COMMISSION_EXCEEDS_MAX);
```

This mirrors Uniswap V2's `amountAMin`/`amountBMin` pattern: the caller declares the worst-case terms they will accept, and the transaction reverts if on-chain state has moved beyond that bound.

---

### Proof of Concept

1. **Setup** – Staker deploys a STRK pool, sets commission to 2 % (`200`), then calls `set_commission_commitment(max_commission: 9000, expiration_epoch: current + epochs_in_year)`.
2. **Delegator initiates exit** – Delegator calls `exit_delegation_pool_intent(amount)` on their current pool, entering the exit-wait window (up to 12 weeks).
3. **Staker raises commission** – At any point during the exit window, staker calls `set_commission(9000)`. The commitment is active, so this succeeds immediately.
4. **Delegator switches** – Delegator calls `switch_delegation_pool(to_staker, to_pool, amount)`. No commission check is performed; the call succeeds.
5. **Outcome** – Delegator is now a member of a pool charging 90 % commission. Every subsequent `update_rewards` call routes 90 % of the delegator's share to the staker as commission, leaving the delegator with 10 % of their entitled yield. The delegator has no recourse for rewards already distributed at the inflated rate. [8](#0-7) [4](#0-3) [5](#0-4)

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }
