### Title
Unprivileged Caller Can Permanently Deny All Stakers' Block Rewards by Calling `update_rewards` with `disable_rewards: true` — (File: src/staking/staking.cairo)

---

### Summary

The public `update_rewards` function in `StakingRewardsManagerImpl` writes the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. Because the function has no access control, any unprivileged caller can invoke it with `disable_rewards: true` to consume the per-block reward slot without distributing any rewards, permanently denying all stakers their consensus block rewards for that block.

---

### Finding Description

In `update_rewards` (`src/staking/staking.cairo`, lines 1449–1507), the execution order is:

1. Check `current_block_number > last_reward_block` (line ~1454–1458) — gate to prevent double-processing.
2. Validate the staker is active and has non-zero balance (lines ~1460–1482).
3. **Write `last_reward_block = current_block_number`** (line 1485).
4. If `disable_rewards || self.is_pre_consensus()` → **return early, distributing zero rewards** (lines 1487–1489).
5. Otherwise, compute and distribute block rewards via `_update_rewards`. [1](#0-0) 

Because `last_reward_block` is a **single global** storage slot (not per-staker): [2](#0-1) 

…any call that reaches step 3 — even one that immediately returns at step 4 — permanently consumes the reward slot for that block. All subsequent calls in the same block fail with `REWARDS_ALREADY_UPDATED`: [3](#0-2) 

The function carries **no role check** — it is part of `IStakingRewardsManager` embedded via `#[abi(embed_v0)]` and is callable by any address: [4](#0-3) 

An attacker therefore only needs to supply any valid active staker address (publicly available from `NewStaker` events) and call `update_rewards(active_staker, disable_rewards: true)` once per block to prevent every staker from receiving consensus block rewards in that block.

---

### Impact Explanation

Consensus block rewards are distributed exclusively through `update_rewards`. If the attacker calls it with `disable_rewards: true` at the start of every block, **no staker ever receives consensus rewards**. The rewards for each consumed block are permanently lost — they cannot be retroactively claimed. This constitutes **permanent freezing of unclaimed yield** for all stakers, matching the High impact tier.

---

### Likelihood Explanation

The attack requires:
- No special role or privilege.
- Knowledge of one active staker address — trivially obtained from on-chain `NewStaker` events.
- Gas to call the function once per block.

The attacker gains nothing financially (pure griefing), but the cost is low and the attack is fully automatable. Any motivated adversary (e.g., a competing protocol, a censorship actor) can sustain it indefinitely.

---

### Recommendation

Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` early-return check, so the slot is only consumed when rewards are actually distributed:

```cairo
// Update last block rewards ONLY when rewards will be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... proceed with _update_rewards
```

Alternatively, add access control (e.g., `only_app_governor` or a dedicated `REWARDS_UPDATER` role) to restrict who may call `update_rewards`.

---

### Proof of Concept

1. Attacker observes any active staker address `S` from `NewStaker` events.
2. At the start of each block, attacker calls `update_rewards(S, disable_rewards: true)`.
3. `last_reward_block` is written to the current block number (line 1485).
4. Function returns early — zero rewards distributed.
5. Any legitimate call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED` (line 1455–1458).
6. All stakers lose their consensus block rewards for that block.
7. Repeated every block → permanent, protocol-wide denial of all consensus rewards. [5](#0-4)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
