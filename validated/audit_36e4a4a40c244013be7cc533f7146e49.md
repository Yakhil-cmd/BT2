### Title
Gateway Stateless Validator Skips `max_l2_gas_amount` Enforcement for Declare Transactions - (File: crates/apollo_gateway/src/stateless_transaction_validator.rs)

### Summary
`StatelessTransactionValidator::validate_resource_bounds` explicitly skips the `max_l2_gas_amount` upper-bound check for `RpcTransaction::Declare` transactions. Any user can submit a Declare transaction with `l2_gas.max_amount` set to an arbitrary value up to `u64::MAX`, bypassing the gateway's configured admission limit (`1_210_000_000` in production). The gateway admits the transaction; the blockifier's balance check then rejects it only if the user cannot cover `max_possible_fee = max_amount × max_price_per_unit`, but the admission invariant is broken at the gateway layer.

### Finding Description

In `validate_resource_bounds`, the check for `max_l2_gas_amount` is guarded by a type-dispatch that silently no-ops for Declare:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← no check at all
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

The production default is `max_l2_gas_amount = 1_210_000_000` and `min_gas_price = 8_000_000_000`. [2](#0-1) 

The existing test `valid_l2_gas_amount_on_declare` explicitly asserts that a Declare with `l2_gas.max_amount = 200` passes when `max_l2_gas_amount = 100`, confirming the bypass is real and exercised: [3](#0-2) 

The `l2_gas.max_amount` field is the direct source of the initial Sierra gas budget passed to the Cairo VM. In the blockifier, `initial_sierra_gas()` returns `l2_gas.max_amount` for `AllResources` bounds, and the validate/execute phases cap it with `min(validate_max_sierra_gas, user_initial_gas)` and `min(execute_max_sierra_gas, remaining_gas)` respectively: [4](#0-3) 

The OS Cairo code confirms `l2_gas.max_amount` is the initial gas bound: [5](#0-4) 

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions.**

A Declare transaction with `l2_gas.max_amount` exceeding `max_l2_gas_amount` passes `StatelessTransactionValidator::validate` and is admitted to the mempool. The gateway's own admission invariant — that no transaction may claim more than `max_l2_gas_amount` L2 gas — is violated for every Declare transaction that sets a larger value.

Downstream mitigations exist but do not restore the gateway invariant:
- `verify_can_pay_committed_bounds` in `perform_pre_validation_stage` rejects the transaction at blockifier pre-validation if `balance < max_amount × max_price_per_unit`. Because `min_gas_price = 8_000_000_000`, any `max_amount` meaningfully above `max_l2_gas_amount` produces a `max_possible_fee` that exceeds any realistic balance.
- The blockifier caps actual execution gas at `validate_max_sierra_gas = 100_000_000` and `execute_max_sierra_gas = 1_110_000_000`, so no extra computation is performed even if the transaction were executed. [6](#0-5) 

The broken invariant is at the gateway admission layer: the gateway accepts and forwards to the mempool a Declare transaction that its own configuration declares invalid.

### Likelihood Explanation

Any unprivileged user can craft a Declare V3 transaction with `l2_gas.max_amount = u64::MAX` and submit it to the gateway. The gateway will accept it through stateless validation. The transaction will subsequently be rejected by the blockifier's balance check (because `max_possible_fee` saturates to an astronomically large value), so no state change occurs. The likelihood of gateway-level admission is **certain** for any Declare with `max_amount > max_l2_gas_amount`; the likelihood of the transaction reaching execution is low due to the balance guard.

### Recommendation

Remove the type-dispatch exemption and apply the same `max_l2_gas_amount` check to Declare transactions:

```rust
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

This resolves the acknowledged TODO and closes the admission gap at the gateway layer. [1](#0-0) 

### Proof of Concept

1. Construct a valid Declare V3 transaction with:
   - `l2_gas.max_amount = u64::MAX` (or any value > `1_210_000_000`)
   - `l2_gas.max_price_per_unit = 8_000_000_000` (satisfies `min_gas_price` check)
   - Any non-zero `l1_gas` or `l1_data_gas` bounds (to pass `ZeroResourceBounds` check)
   - A valid Sierra class, valid signature, valid nonce

2. Submit to the gateway's `add_declare_transaction` endpoint.

3. Observe: `StatelessTransactionValidator::validate` returns `Ok(())` — the transaction is admitted to the mempool despite `l2_gas.max_amount` exceeding `max_l2_gas_amount = 1_210_000_000`.

4. The blockifier subsequently rejects it at `verify_can_pay_committed_bounds` because `max_possible_fee = u64::MAX × 8_000_000_000` exceeds any realistic balance — but the gateway admission invariant has already been violated. [7](#0-6) [2](#0-1) [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
