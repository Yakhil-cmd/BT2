### Title
Unrestricted `update_rewards` Allows Any Caller to Steal or Permanently Freeze Per-Block Staker Yield — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract enforces no meaningful caller restriction. A global `last_reward_block` lock allows only one call per block. Any unprivileged address can front-run the consensus mechanism's call, either redirecting the full block reward to their own staker or suppressing it entirely with `disable_rewards=true`, permanently denying yield to the intended recipient.

---

### Finding Description

`update_rewards` is the consensus-based reward distribution entry point. Its only gate is `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

The global `last_reward_block` enforces a single call per block: [2](#0-1) 

When `disable_rewards=false`, the function computes block rewards using the **caller-supplied** `staker_address`'s own total balance as the denominator, then distributes the full block reward to that staker: [3](#0-2) 

Because `last_reward_block` is written unconditionally before the `disable_rewards` branch: [4](#0-3) 

…any subsequent call in the same block—including the legitimate consensus call—reverts with `REWARDS_ALREADY_UPDATED`.

The `disable_rewards=true` path is equally dangerous: it advances `last_reward_block` and returns immediately, distributing nothing, so the block reward is silently discarded: [5](#0-4) 

---

### Impact Explanation

**High — Theft of unclaimed yield / Permanent freezing of unclaimed yield.**

- **Theft path**: Attacker front-runs the consensus mechanism with `update_rewards(attacker_staker, disable_rewards=false)`. The attacker's staker receives the full block reward. The intended staker receives nothing for that block. Repeated every block, this constitutes continuous theft of yield from the targeted staker.
- **Freeze path**: Attacker calls `update_rewards(any_valid_staker, disable_rewards=true)`. `last_reward_block` advances, no rewards are minted or distributed. The block reward is permanently lost. Repeated every block, this permanently freezes all yield for any staker.

Both paths cause irreversible per-block yield loss with no recovery mechanism.

---

### Likelihood Explanation

**Medium.** The attack requires front-running the consensus mechanism's `update_rewards` call each block. On Starknet, transaction ordering within a block is sequencer-controlled, but the function is fully public and callable by any EOA or contract. The attacker must pay gas per block but gains yield (theft path) or achieves targeted griefing (freeze path). No privileged access, leaked key, or external dependency is required—only a valid staker address and a non-zero caller.

---

### Recommendation

Restrict `update_rewards` to a single authorized caller (e.g., the attestation contract, a dedicated consensus contract, or an `OPERATOR` role). Add a caller check analogous to the existing `assert_caller_is_attestation_contract` pattern: [6](#0-5) 

Alternatively, record the authorized caller address in storage during construction and assert `get_caller_address() == authorized_consensus_caller` at the top of `update_rewards`, mirroring the pool-contract caller check: [7](#0-6) 

---

### Proof of Concept

1. Attacker deploys or controls a valid staker `A` with minimal self-stake (above `min_stake`).
2. Each block, attacker monitors the pending transaction queue for the consensus mechanism's `update_rewards(legitimate_staker, false)` call.
3. Attacker submits `update_rewards(A, false)` with higher priority (or simply submits it first).
4. `last_reward_block` is set to the current block; full block rewards flow to staker `A`.
5. Consensus mechanism's call reverts: `current_block_number > self.last_reward_block.read()` is false.
6. `legitimate_staker` receives zero yield for that block—permanently, since there is no retroactive reward mechanism.
7. For the freeze variant, replace step 3 with `update_rewards(any_valid_staker, true)`: `last_reward_block` advances, zero rewards distributed, block reward is destroyed. [8](#0-7)

```text
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
```
