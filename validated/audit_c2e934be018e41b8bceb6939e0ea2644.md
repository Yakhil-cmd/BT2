### Title
Public `update_rewards` Caller Can Permanently Freeze All Staker Rewards via `disable_rewards=true` - (File: src/staking/staking.cairo)

### Summary

`update_rewards` is a public ABI function that accepts a caller-controlled `disable_rewards: bool` parameter. When called with `disable_rewards = true`, it advances the global `last_reward_block` checkpoint without distributing any rewards, consuming the per-block reward slot for the entire protocol. Any unprivileged address can call this every block to permanently freeze all staker and delegator yield.

### Finding Description

`StakingRewardsManagerImpl::update_rewards` is decorated with `#[abi(embed_v0)]`, making it callable by any address. Its only access control is `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

The function enforces a single global gate: `current_block_number > self.last_reward_block.read()`. It then unconditionally writes `last_reward_block = current_block_number` **before** checking `disable_rewards`. [2](#0-1) 

If `disable_rewards == true`, the function returns immediately after updating `last_reward_block`, skipping the entire `_update_rewards` call path. [3](#0-2) 

Because `last_reward_block` is a **single global storage slot** (not per-staker), one call with `disable_rewards = true` for any valid active staker exhausts the reward slot for that block for **all** stakers. Any subsequent legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

The attacker only needs to supply any valid, active `staker_address` — they do not need to be the staker themselves.

### Impact Explanation

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` on every block permanently prevents all consensus-phase reward distribution for every staker and delegator in the protocol. Stakers' `unclaimed_rewards_own` and pool `cumulative_rewards_trace` are never updated, freezing all yield indefinitely. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation

The function is public with no role check. The only cost is gas per block. The attacker needs no special privileges, no capital, and no coordination. A single bot running continuously is sufficient to sustain the attack.

### Recommendation

Access-control `update_rewards` so only the attestation contract or a designated consensus rewards caller can invoke it, or remove the `disable_rewards` parameter from the public ABI and handle that logic internally via a privileged path. At minimum, `disable_rewards = true` should require a trusted caller role.

### Proof of Concept

1. Attacker deploys a bot that monitors the Starknet mempool.
2. Each block, the bot calls `staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)`.
3. `last_reward_block` is set to the current block number; no rewards are computed or distributed.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. After sustained execution, `unclaimed_rewards_own` for all stakers remains zero; pool `cumulative_rewards_trace` is never advanced; delegators' `claim_rewards` returns zero.
6. All staked STRK principal remains withdrawable (no theft of principal), but all yield is permanently frozen — matching the **High: Permanent freezing of unclaimed yield** impact category.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
