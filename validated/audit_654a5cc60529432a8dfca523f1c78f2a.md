### Title
Rejected L1 Handler Transactions Are Permanently Removed from `proposable_index` with No Recovery Path — (`crates/apollo_l1_events/src/transaction_manager.rs`)

---

### Summary

`TransactionManager::commit_txs` calls `mark_rejected()` on each tx in `rejected_txs`, which sets `state = TransactionState::Rejected`. The subsequent `maintain_indices` call then removes the tx from `proposable_index` because `is_proposable()` only matches `Pending`. No code path transitions a `Rejected` record back to `Pending`, so the tx is permanently unproposable. The design document explicitly states the intended behavior is the opposite: *"For rejected txs: Unstage, keep as Pending."*

**Step 5 — `get_txs` reads exclusively from `proposable_index`; a tx absent from it is never returned:** [6](#0-5) 

**Step 6 — No state transition from `Rejected` back to `Pending` exists anywhere in `TransactionRecord`.** The only state-mutating methods are `mark_committed`, `mark_rejected`, `mark_cancellation_request`, `mark_cancellation_finalized_on_l1`, `mark_consumed`, and `update_time_based_state`. None of them transition from `Rejected` to `Pending`. [7](#0-6) 

**Step 7 — `add_tx` cannot rescue a rejected tx.** `create_record_if_not_exist` is a no-op when the record already exists, and the `Full` payload branch only warns and returns — it never resets state. [8](#0-7) 

**Step 8 — The design document contradicts the implementation.** The sequence diagram for block commit explicitly annotates: *"For rejected txs: Unstage, keep as Pending"* — meaning the intended behavior is that rejected txs remain in `proposable_index` for re-proposal. The code does the opposite. [9](#0-8) 

**Step 9 — `is_validatable()` explicitly lists `Rejected` as a validatable state** (its docstring says *"this includes states like: a rejected transaction"*), yet `is_proposable()` excludes it. This inconsistency confirms the `Rejected` state was intended to be temporary and re-proposable, not terminal. [10](#0-9) 

---

### Impact Explanation

An L1 handler transaction that fails execution in a proposed block (e.g., due to a transient execution error, a gas/resource edge case, or a blockifier revert) is passed to `commit_txs` in `rejected_txs`. After that call, the tx's record remains in `records` with state `Rejected`, but it is permanently absent from `proposable_index`. `get_txs` will never return it again. The L1 message it represents is permanently stuck: it cannot be re-proposed, re-validated for inclusion, or recovered by any existing code path. The user's L1→L2 message is silently dropped with no on-chain or off-chain recovery mechanism.

---

### Likelihood Explanation

Any L1 handler tx that experiences a transient execution failure (revert, out-of-gas, blockifier error) during a block proposal triggers this path. This is a normal operational scenario, not an adversarial one. The L1 message queue on Ethereum continues to hold the unconsumed message, but the sequencer will never re-propose it.

---

### Recommendation

In `commit_txs`, rejected L1 handler txs should not call `mark_rejected()`. Instead, the staging epoch rollback (`rollback_staging()`, already called at the top of `commit_txs`) is sufficient to unstage them, leaving their state as `Pending` and keeping them in `proposable_index` for re-proposal in the next block — which is exactly what the design document specifies. If a `Rejected` terminal state is needed for other purposes (e.g., operator-initiated permanent rejection), it should be a separate, explicitly privileged operation distinct from the normal block-commit path.

---

// Pseudocode unit test (matches the suggested proof idea exactly)
#[test]
fn rejected_tx_is_permanently_unproposable() {
    let mut mgr = TransactionManager::default();
    let tx = make_l1_handler_tx(); // full payload
    let tx_hash = tx.tx_hash;
    let scrape_ts = 0u64;
    let block_ts = BlockTimestamp(0);

    mgr.add_tx(tx, block_ts, scrape_ts);
    // tx is now Pending and in proposable_index

    // Simulate: tx was proposed, executed, and rejected
    mgr.commit_txs(&[], &[tx_hash]);

    // After commit_txs, state is Rejected, proposable_index is empty
    let snapshot = mgr.snapshot();
    assert!(snapshot.proposable_index.is_empty());          // PASSES — index cleared
    assert_eq!(snapshot.rejected, vec![tx_hash]);           // PASSES — permanently Rejected

    // get_txs returns nothing — tx is permanently lost
    let returned = mgr.get_txs(10, u64::MAX);
    assert!(returned.is_empty());                           // PASSES — tx never re-proposed
}

    pub fn mark_committed(&mut self) {
        // Can't return error because committing only part of a block leaves the provider in an
        // undetermined state.
        assert!(
            !self.committed,
            "L1 handler transaction {} committed twice, this may lead to l2 reorgs,",
            self.tx.tx_hash()
        );
        self.state = TransactionState::Committed;
        self.committed = true;
    }

    // Note: double reject not currently checked.
    pub fn mark_rejected(&mut self) {
        // Pedantic, this is unlikely to happen.
        assert!(
            !self.committed,
            "Attempted to reject a committed transaction {}",
            self.tx.tx_hash()
        );
        self.state = TransactionState::Rejected;
        self.rejected = true;
    }

    /// Mark a cancellation request for this transaction.
    /// Returns the existing cancellation timestamp if one exists, `None` if this is the first
    /// request.
    pub fn mark_cancellation_request(
        &mut self,
        timestamp: BlockTimestamp,
    ) -> Option<BlockTimestamp> {
        let tx_hash = self.tx.tx_hash();
        // Once committed on L2, cancellation requests are only recorded for debugging purposes, but
        // not processed.
        if self.is_committed() {
            warn!(
                "L1 handler transaction {tx_hash} was not marked for cancellation started on L2 \
                 as it is already committed."
            )
        } else {
            info!("Marking L1 handler transaction {tx_hash} as cancellation started on L2.");
            self.state = TransactionState::CancellationStartedOnL2;
        }

        match self.cancellation_requested_at {
            Some(existing) => Some(existing),
            None => {
                self.cancellation_requested_at = Some(timestamp);
                None
            }
        }
    }

    /// Mark a transaction as cancelled on L1.
    // This happens just before the tx is removed from the transaction manager.
    // It is useful to change the state so that maintain_indices can remove it from the proposable
    // index.
    pub fn mark_cancellation_finalized_on_l1(&mut self) {
        self.state = TransactionState::CancellationFinalizedOnL1;
    }

    /// Mark a transaction as consumed on L1.
    /// The timestamp is the L1 block timestamp where this tx was marked consumed.
    /// If tx was not already consumed (expected result), return None.
    /// If tx was already consumed (double consumption), return the time when it was previously
    /// consumed. Note that double consumption is a bug.
    pub fn mark_consumed(&mut self, timestamp: BlockTimestamp) -> Option<BlockTimestamp> {
        if self.is_committed() {
            debug!("Marking a committed transaction {} as consumed.", self.tx.tx_hash());
        } else {
            // TODO(guyn): check if this situation should be an error.
            // TODO(guyn): check other state combinations that may be worth an error/warning/debug
            // log.
            debug!(
                "Marking a non-committed transaction {} as consumed. Previous state: {:?}",
                self.tx.tx_hash(),
                self.state
            );
        }
        self.state = TransactionState::Consumed;
        // First check if the tx was already consumed. Double consumption is a bug!
        // If None, it wasn't previously consumed: mark the time and return None to signal
        // everything is ok. If Some, it was already consumed: report the time when it was
        // previously consumed, the caller decides what to do.
        match self.consumed_at {
            Some(existing) => Some(existing),
            None => {
                self.consumed_at = Some(timestamp);
                None
            }
        }
    }
