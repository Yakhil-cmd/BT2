### Title
Revoking keyIndex=0 Does Not Revoke Duplicate Keys Pointing to storedKeyIndex=0 — (`fvm/environment/account_public_key_util.go`, `fvm/environment/accounts.go`)

### Summary

When a duplicate of key 0 is added at keyIndex=N, the revoked status for keyIndex=N is stored independently in the key metadata section (weight-and-revoked group). Revoking keyIndex=0 only sets `Revoked=true` in the `apk_0` register. It does not update the weight-and-revoked entry for keyIndex=N. As a result, `GetRuntimeAccountPublicKey(keyIndex=N)` returns `Revoked=false` with the same public key bytes as key 0, and the transaction verifier accepts a transaction signed with key 0's private key at keyIndex=N.

### Finding Description

**Storage layout for key 0 vs. duplicate keys:**

Key 0's revoked status is stored inline in the `apk_0` register (encoded `AccountPublicKey` struct). `revokeAccountPublicKey0` reads this register, sets `Revoked=true`, and writes it back — it touches nothing else. [1](#0-0) 

For keyIndex=N (N>0), the revoked status is stored in the weight-and-revoked section of the key metadata blob inside the account status register. When a duplicate of key 0 is added at keyIndex=N, `AppendDuplicateKeyMetadata` writes `revoked=false` (the value passed at add-time) into that section, and records the mapping `keyIndex=N → storedKeyIndex=0`. [2](#0-1) 

**Lookup path for keyIndex=N:**

`GetRuntimeAccountPublicKey(address, N)` (N>0) reads `revoked` from the weight-and-revoked section via `status.AccountPublicKeyMetadata(N)`, then fetches the stored public key bytes via `getStoredPublicKey(a, address, storedKeyIndex=0)`. `getStoredPublicKey` with `storedKeyIndex=0` reads from the `apk_0` register and returns the raw public key bytes — but **not** the revoked flag from that register. [3](#0-2) [4](#0-3) 

The returned `RuntimeAccountPublicKey` has `Revoked` sourced from the metadata section (still `false`) and `PublicKey` sourced from the `apk_0` register (key 0's bytes). [5](#0-4) 

**Transaction verifier:**

`getAccountKeysAndAggregateWeights` calls `GetRuntimeAccountPublicKey` and rejects only if `accountKey.Revoked == true`. Since the duplicate key's metadata still says `false`, the check passes. [6](#0-5) 

**The missing update:**

`RevokeAccountPublicKey(address, 0)` dispatches to `revokeAccountPublicKey0`, which only writes the `apk_0` register. There is no code that iterates over duplicate-key mappings and sets the revoked flag in the weight-and-revoked section for any keyIndex that maps to `storedKeyIndex=0`. [7](#0-6) 

### Impact Explanation

An attacker who controls an account (or whose account has been set up with a duplicate key) can:
1. Add a duplicate of key 0 at keyIndex=1 (permitted by the protocol — deduplication is a supported feature).
2. Revoke keyIndex=0 (or have it revoked by the account owner).
3. Sign and submit transactions using key 0's private key at keyIndex=1.
4. The transaction verifier accepts the transaction because `GetRuntimeAccountPublicKey(keyIndex=1).Revoked == false`.

This violates the invariant that revoking a key's cryptographic material prevents all authorization using that material. The impact is **unauthorized account manipulation**: a transaction is authorized using a key whose primary index has been revoked.

### Likelihood Explanation

- Adding duplicate keys is a supported, production code path with no restrictions.
- The attacker only needs to control their own account (no admin/operator access required).
- The exploit is deterministic and locally reproducible on an unmodified Flow emulator.

### Recommendation

When revoking keyIndex=0, also scan the key metadata mappings for all keyIndexes that resolve to `storedKeyIndex=0` and set their revoked flag in the weight-and-revoked section. Alternatively, when `GetRuntimeAccountPublicKey` resolves a duplicate key to `storedKeyIndex=0`, it should additionally read the revoked flag from the `apk_0` register and OR it with the metadata revoked flag before returning.

### Proof of Concept

```
1. Create account A with key K (keyIndex=0, stored in apk_0).
2. Call AddAccountKey on A with the same public key bytes as K (weight=1000).
   → keyIndex=1 is added as a duplicate; metadata records revoked=false, storedKeyIndex=0.
3. Call RevokeAccountKey on A at keyIndex=0.
   → apk_0 register: Revoked=true.
   → Key metadata weight-and-revoked for keyIndex=1: Revoked=false (unchanged).
4. Submit a transaction signed by K's private key, referencing keyIndex=1.
5. TransactionVerifier calls GetRuntimeAccountPublicKey(A, 1):
   → revoked=false (from metadata), PublicKey=K's bytes (from apk_0).
6. Revocation check passes (revoked==false).
7. Signature verification passes (signed with K's private key).
8. Transaction is accepted — key 0's cryptographic material is still active via keyIndex=1.
```

func revokeAccountPublicKey0(
	a Accounts,
	address flow.Address,
) error {
	accountPublicKey0RegisterID := flow.AccountPublicKey0RegisterID(address)

	publicKey, err := a.GetValue(accountPublicKey0RegisterID)
	if err != nil {
		return err
	}

	const keyIndex = uint32(0)

	if len(publicKey) == 0 {
		return errors.NewAccountPublicKeyNotFoundError(
			address,
			keyIndex)
	}

	decodedPublicKey, err := flow.DecodeAccountPublicKey(publicKey, keyIndex)
	if err != nil {
		return fmt.Errorf(
			"failed to decode account public key 0: %w",
			err)
	}

	decodedPublicKey.Revoked = true

	encodedPublicKey, err := flow.EncodeAccountPublicKey(decodedPublicKey)
	if err != nil {
		encoded, _ := decodedPublicKey.MarshalJSON()
		return errors.NewValueErrorf(
			string(encoded),
			"failed to encode revoked account public key 0: %w",
			err)
	}

	return a.SetValue(accountPublicKey0RegisterID, encodedPublicKey)

func (m *KeyMetadataAppender) AppendDuplicateKeyMetadata(
	keyIndex uint32,
	duplicateStoredKeyIndex uint32,
	revoked bool,
	weight uint16,
) (err error) {

	// Append revoked status and weight
	m.weightAndRevokedStatusBytes, err = appendWeightAndRevokedStatus(m.weightAndRevokedStatusBytes, revoked, weight)
	if err != nil {
		return err
	}

	if !m.deduplicated {
		// Set deduplication flag
		m.deduplicated = true

		// Save mapping start key index.
		m.startIndexForMapping = keyIndex
	}

	// Append duplicate stored key index to mapping
	m.mappingBytes, err = appendStoredKeyIndexToMappings(m.mappingBytes, duplicateStoredKeyIndex)
	if err != nil {
		return err
	}

	return nil
}

func (a *StatefulAccounts) RevokeAccountPublicKey(
	address flow.Address,
	keyIndex uint32,
) error {
	err := a.accountPublicKeyIndexInRange(address, keyIndex)
	if err != nil {
		return err
	}

	if keyIndex == 0 {
		return revokeAccountPublicKey0(a, address)
	}

	status, err := a.getAccountStatus(address)
	if err != nil {
		return err
	}

	err = status.RevokeAccountPublicKey(keyIndex)
	if err != nil {
		return fmt.Errorf("failed to revoke public key at index %d for %s: %w", keyIndex, address, err)
	}

	return a.setAccountStatusAfterAccountStatusSizeChange(address, status)
