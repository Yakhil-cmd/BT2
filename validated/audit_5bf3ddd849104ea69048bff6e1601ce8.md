### Title
Gateway Stateful Validator Compares Transaction L2 Gas Price Against Stale Current-Block Price Instead of Next-Block Price, Causing Wrong Admission Decisions — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` checks whether a transaction's offered L2 gas price meets the admission threshold by comparing it against `previous_block.gas_prices.strk_gas_prices.l2_gas_price` — the **current** (already-committed) block's L2 gas price. However, the transaction will actually be executed in the **next** block, whose L2 gas price is computed by the EIP-1559 fee-market formula and stored in `BlockHeaderWithoutHash::next_l2_gas_price`. The two values can diverge by up to `1/gas_price_max_change_denominator` per block. Using the wrong reference price causes the gateway to admit transactions that will fail `check_fee_bounds` during batcher execution (price rising) or to reject transactions that are perfectly valid for the next block (price falling). A developer TODO comment in the source code explicitly acknowledges this gap.

---

### Finding Description

`validate_resource_bounds` reads the block info from `GatewayFixedBlockStateReader::get_block_info()`, which returns the **last committed block's** `BlockInfo`. It then extracts `strk_gas_prices.l2_gas_price` from that struct and uses it as the reference price for the threshold check:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold is then:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold { return Err(...); }
``` [2](#0-1) 

The `GatewayFixedBlockSyncStateClient` builds `BlockInfo` from `block_header_without_hash` and populates `l2_gas_price` from `block_header.l2_gas_price.price_in_fri` — the **current** block's price — while the `next_l2_gas_price` field of the same header (the EIP-1559-adjusted price for the next block) is silently discarded: [3](#0-2) 

The consensus orchestrator computes the next block's L2 gas price via `calculate_next_base_gas_price` (EIP-1559) and stores it in `BlockHeaderWithoutHash::next_l2_gas_price`. This value is what the batcher will actually use when building the block context for transaction execution: [4](#0-3) 

The batcher's `check_fee_bounds` (called inside `perform_pre_validation_stage`) compares `tx.resource_bounds.l2_gas.max_price_per_unit` against `block_info.gas_prices.gas_price_vector(fee_type).l2_gas_price` — the **next** block's price supplied by the consensus orchestrator, not the previous block's price used by the gateway: [5](#0-4) 

The two reference prices are structurally different values: `previous_block.l2_gas_price` vs. `previous_block.next_l2_gas_price` (= EIP-1559 output). The TODO comment in the gateway source confirms the intended fix is to use `next_l2_gas_price`.

---

### Impact Explanation

**Stale price too low (price rising, high congestion):** When the network is congested, `next_l2_gas_price > l2_gas_price`. A transaction with `tx_l2_gas_price` in the range `[threshold_based_on_current_price, actual_next_block_price)` passes the gateway's admission check but fails `check_fee_bounds` during batcher execution. The transaction is included in a block, reverts, and the sender is charged a fee — an incorrect admission decision.

**Stale price too high (price falling, low congestion):** When congestion drops, `next_l2_gas_price < l2_gas_price`. A transaction with `tx_l2_gas_price` in the range `[threshold_based_on_next_price, threshold_based_on_current_price)` is rejected by the gateway even though it would satisfy the actual next-block price. This is an incorrect rejection of a valid transaction.

Both directions match the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The EIP-1559 formula can move the L2 gas price by up to `price / gas_price_max_change_denominator` per block. With the default `min_gas_price_percentage = 100` (no discount), any upward price movement — however small — creates a window where transactions admitted by the gateway will fail `check_fee_bounds` in the batcher. The window grows proportionally with congestion. The trigger requires no privilege: any user submitting a V3 (`AllResources`) transaction with `max_price_per_unit` set to exactly the previous block's price exercises this path.

---

### Recommendation

Replace the use of `l2_gas_price` with `next_l2_gas_price` from the block header in `validate_resource_bounds`. The `GatewayFixedBlockStateReader` trait and its `GatewayFixedBlockSyncStateClient` implementation should expose `next_l2_gas_price` (already present in `BlockHeaderWithoutHash`) so that `validate_resource_bounds` can compare the transaction's offered price against the price that will actually govern the block the transaction enters.

---

### Proof of Concept

1. The last committed block has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (high congestion, EIP-1559 raised the price).
2. A user submits a V3 invoke transaction with `l2_gas.max_price_per_unit = P` (exactly equal to the previous block's price, so `threshold = P` with `min_gas_price_percentage = 100`).
3. `validate_resource_bounds` computes `threshold = P`, sees `tx_l2_gas_price (P) >= threshold (P)`, and **admits** the transaction.
4. The transaction enters the mempool and is picked up by the batcher.
5. The batcher builds the block context with `l2_gas_price = P'` (from the consensus orchestrator).
6. `check_fee_bounds` compares `tx_l2_gas_price (P) < actual_gas_price (P')` and **rejects** the transaction with `MaxGasPriceTooLow`.
7. The transaction is included in the block as a failed/reverted transaction; the sender is charged a fee for a transaction that should never have been admitted.

The inverse scenario (price falling) causes valid transactions to be incorrectly rejected at the gateway, matching the "rejects valid transactions" branch of the allowed impact.

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
