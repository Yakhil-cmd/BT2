### Title
Missing Maximum Commission Guard in Delegation Entry Points Allows Staker to Steal Delegator Yield - (File: src/pool/pool.cairo)

### Summary
`enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool` in `pool.cairo` accept no `max_commission` parameter. A staker can use the `set_commission_commitment` + `set_commission` mechanism to raise commission to 100% immediately after a delegator commits funds, with no on-chain protection for the delegator. The codebase itself acknowledges this gap with an explicit developer note.

### Finding Description

The three delegation entry-points in `pool.cairo` take no slippage-equivalent guard:

```cairo
fn enter_delegation_pool(ref self: ContractState, reward_address: ContractAddress, amount: Amount)
fn add_to_delegation_pool(ref self: ContractState, pool_member: ContractAddress, amount: Amount) -> Amount
fn switch_delegation_pool(ref self: ContractState, to_staker: ContractAddress, to_pool: ContractAddress, amount: Amount) -> Amount
``` [1](#0-0) [2](#0-1) [3](#0-2) 

None of these functions accept a `max_commission` argument, so a delegator cannot express "revert if commission exceeds X" at the protocol level.

The commission-increase path is enabled by `set_commission_commitment` + `set_commission` in `staking.cairo`. When an active commitment exists, `update_commission` enforces only `commission <= commitment.max_commission` and `commission != old_commission` — it does **not** require the new value to be lower than the current one:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
``` [4](#0-3) 

`max_commission` may be set to `COMMISSION_DENOMINATOR` (10 000 = 100 %): [5](#0-4) [6](#0-5) 

The developers themselves flag this in a code comment immediately above `set_commission_commitment`:

```cairo
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
``` [7](#0-6) 

### Impact Explanation

**Impact: High — Theft of unclaimed yield.**

Attack sequence:

1. Pool is live with commission = 5 % (500). Delegators have accrued yield that will be distributed on the next `update_rewards_from_attestation_contract` / `update_rewards` call.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
3. In the same block (or the next), staker calls `set_commission(commission: 10000)`. Commission is now 100 %.
4. The next reward distribution call applies the new commission: `split_rewards_with_commission` routes 100 % of pool rewards to the staker as commission; the pool contract receives 0 STRK.
5. Delegators call `claim_rewards` and receive nothing for that distribution period.

The yield that delegators earned through the staker's attestation/consensus participation is redirected entirely to the staker. Delegators cannot recover it; they can only exit (subject to the exit wait window of at least one week), during which further distributions at 100 % commission continue to drain their yield. [8](#0-7) [9](#0-8) 

### Likelihood Explanation

Any registered staker who has opened a delegation pool can execute this attack without any privileged access. The two-step sequence (`set_commission_commitment` then `set_commission`) is callable by the staker address alone and can be executed in consecutive transactions within the same epoch. Delegators have no on-chain mechanism to prevent it; monitoring off-chain and submitting an exit intent still leaves them exposed for the full exit wait window. [10](#0-9) 

### Recommendation

Add a `max_commission` parameter to `enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool`. Each function should assert that the pool's current commission does not exceed the caller-supplied bound before accepting funds:

```cairo
fn enter_delegation_pool(
    ref self: ContractState,
    reward_address: ContractAddress,
    amount: Amount,
    max_commission: Commission,   // <-- new
) {
    let current_commission = self.get_commission();
    assert!(current_commission <= max_commission, Error::COMMISSION_TOO_HIGH);
    ...
}
```

This mirrors the standard slippage-guard pattern and gives delegators an atomic, trust-minimised protection equivalent to `amountOutMin` in AMM swap interfaces.

### Proof of Concept

```
// Staker has commission = 500 (5 %).
// Delegator enters pool with 1_000_000 STRK.
pool.enter_delegation_pool(reward_address, amount: 1_000_000);

// --- Staker executes attack in two txs ---
staking.set_commission_commitment(max_commission: 10_000, expiration_epoch: current_epoch + 1);
staking.set_commission(commission: 10_000);   // commission is now 100 %

// --- Next epoch reward distribution ---
// update_rewards_from_attestation_contract is called.
// split_rewards_with_commission routes 100 % to staker; pool receives 0 STRK.

// Delegator claims rewards.
let rewards = pool.claim_rewards(pool_member: delegator);
assert!(rewards == 0);   // delegator receives nothing
``` [11](#0-10) [12](#0-11) [13](#0-12)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
