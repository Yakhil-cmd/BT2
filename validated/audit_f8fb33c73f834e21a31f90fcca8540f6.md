### Title
Switching ETH Connector Account to Non-Functional Address Blocks All Bridge Withdrawals Without Guard - (File: `engine/src/contract_methods/connector.rs`)

### Summary
`set_eth_connector_contract_account` allows the owner to redirect the engine's ETH connector reference to any arbitrary account ID with no existence check or interface validation. Because every bridge operation (`withdraw`, `ft_metadata`, `ft_balance_of`, etc.) unconditionally forwards to the stored connector account via `return_promise`, switching to a non-functional account immediately blocks all user withdrawals, causing a temporary fund freeze with no guard preventing the inconsistent state.

### Finding Description
`set_eth_connector_contract_account` writes the caller-supplied account ID directly to storage with no validation:

```rust

```
