### Title
Unprivileged Caller Can Permanently Freeze Block Rewards via `disable_rewards` Flag in `update_rewards` - (File: src/staking/staking.cairo)

### Summary
`update_rewards` merges two distinct behaviors — distributing consensus block rewards and marking a block as processed without distributing rewards — under a single public function controlled by a `disable_rewards` boolean. Because the function has no access control beyond `general_prerequisites` (unpaused + non-zero caller), any unprivileged address can call it with `disable_rewards: true` every block to advance the global `last_reward_block` counter without ever distributing rewards, permanently freezing unclaimed yield for all stakers.

### Finding Description
`update_rewards` is exposed in the public ABI via `StakingRewardsManagerImpl`: [1](#0-0) 

The function's only gate is `general_prerequisites`, which checks only that the contract is unpaused and the caller is non-zero: [2](#0-1) 

Inside the function, `last_reward_block` — a **single global** storage slot shared across all stakers — is unconditionally written before the `disable_rewards` branch: [3](#0-2) 

The guard that prevents a second call in the same block checks this global value: [4](#0-3) 

So the two distinct behaviors merged into one function are:
1. **Normal path** (`disable_rewards: false`): distributes STRK/BTC block rewards to the staker and pools.
2. **No-op path** (`disable_rewards: true`): advances `last_reward_block` to the current block and returns immediately, distributing nothing.

Because `last_reward_block` is global, whichever path runs first in a block wins. An attacker who calls path 2 first causes every subsequent legitimate call in that block to revert with `REWARDS_ALREADY_UPDATED`.

### Impact Explanation
- `last_reward_block` is a single global slot; one attacker call per block blocks **all** stakers from receiving rewards for that block.
- The attacker can repeat this every block at negligible cost on Starknet, causing **permanent freezing of unclaimed yield** for the entire protocol.
- This maps directly to the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"* (High).

### Likelihood Explanation
- The function is in the public ABI with no role check.
- The attacker only needs to supply any currently-active staker address (publicly readable from `stakers` vec) and pass `disable_rewards: true`.
- No capital, no privileged key, no external dependency is required.
- Likelihood is **High**.

### Recommendation
Separate the two behaviors into distinct functions with appropriate access control:
1. A public `update_rewards(staker_address)` that always distributes rewards (remove the `disable_rewards` parameter).
2. A restricted `mark_block_no_rewards(staker_address)` callable only by a trusted role (e.g., `security_agent` or `attestation_contract`), if the no-op path is genuinely needed.

Alternatively, if `disable_rewards: true` is only ever intended to be called by a specific privileged contract, add an explicit caller assertion (similar to `assert_caller_is_attestation_contract`) before the `last_reward_block` write.

### Proof of Concept
1. Attacker observes that `last_reward_block < current_block_number` (i.e., rewards have not yet been updated this block).
2. Attacker reads any active staker address `S` from the public `stakers` storage vector.
3. Attacker calls `update_rewards(staker_address: S, disable_rewards: true)`.
4. `last_reward_block` is written to `current_block_number`; no rewards are distributed.
5. Any legitimate call to `update_rewards` for any staker in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. Attacker repeats step 3 every block → all stakers permanently receive zero consensus block rewards. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
