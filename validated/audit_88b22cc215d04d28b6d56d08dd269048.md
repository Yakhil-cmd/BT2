### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Wrong Admission Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check compares a transaction's `max_price_per_unit` against the **current block's** `l2_gas_price`, but the transaction will actually be executed in the **next block** at `next_l2_gas_price`. Because EIP-1559 can move the price up or down between blocks, the gateway admits transactions that will fail at execution (price rising) and rejects transactions that would succeed (price falling). A TODO comment in the code explicitly acknowledges this is wrong.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price reference from `get_block_info()`:

```rust
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

The `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` builds a `BlockInfo` from the block header, mapping `block_header.l2_gas_price.price_in_fri` into `strk_gas_prices.l2_gas_price`:

```rust
l2_gas_price: block_header.l2_gas_price.price_in_wei.try_into()?,
// ...
l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
``` [2](#0-1) 

However, `BlockHeaderWithoutHash` carries **two distinct price fields**:

```rust
pub l2_gas_price: GasPricePerToken,      // price used in THIS block
pub next_l2_gas_price: GasPrice,          // price that will apply to the NEXT block
``` [3](#0-2) 

`next_l2_gas_price` is computed by the EIP-1559 fee market (`calculate_next_base_gas_price`) and stored in the block header and in `StorageBlockHeader`: [4](#0-3) 

The `get_block_info` conversion **silently drops** `next_l2_gas_price`; it is never placed into the returned `BlockInfo`. The gateway therefore validates against the **current block's** price, while the blockifier's `check_fee_bounds` at execution time uses the **next block's** price (the one actually in effect when the transaction runs).

The TODO comment in the validator code explicitly acknowledges this:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

The threshold check then becomes:

```rust
if tx_l2_gas_price.0 < threshold {   // threshold = % of CURRENT block price
    return Err(GAS_PRICE_TOO_LOW);
}
``` [5](#0-4) 

But the blockifier's `check_fee_bounds` at execution time checks against the **next block's** actual price: [6](#0-5) 

The production default for `min_gas_price_percentage` is 100, meaning the gateway requires `tx.max_price_per_unit >= current_block.l2_gas_price` exactly, while execution requires `tx.max_price_per_unit >= next_block.l2_gas_price`. [7](#0-6) 

### Impact Explanation

**When `next_l2_gas_price > l2_gas_price` (price rising, e.g. block is over gas target):**
Any transaction with `l2_gas_price <= tx.max_price_per_unit < next_l2_gas_price` passes the gateway check but will be rejected by the blockifier's `check_fee_bounds` at execution time. The gateway admits transactions that will fail — wasting mempool slots and misleading users.

**When `next_l2_gas_price < l2_gas_price` (price falling, e.g. block is under gas target):**
Any transaction with `next_l2_gas_price <= tx.max_price_per_unit < l2_gas_price` would succeed at execution but is rejected by the gateway. Valid transactions are denied admission.

Both cases match the **High** impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The EIP-1559 fee market (`calculate_next_base_gas_price`) adjusts the price every block based on gas usage vs. the gas target. Under normal load the price moves continuously. The divergence between `l2_gas_price` and `next_l2_gas_price` is bounded per block by `1/gas_price_max_change_denominator`, but over multiple blocks (or during rapid load changes) the gap can be substantial. Any unprivileged user submitting a V3 (`AllResources`) transaction with a gas price in the gap between the two values triggers the wrong admission decision. No special privileges are required.

### Recommendation

Replace the reference price in `validate_resource_bounds` with `next_l2_gas_price` from the block header, as the TODO comment already indicates:

```rust
// Use next_l2_gas_price (the price for the block this tx will execute in),
// not l2_gas_price (the price of the block already committed).
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()   // new method, or extend BlockInfo
    .await?;
```

Concretely:
1. Extend `GatewayFixedBlockStateReader::get_block_info` (or add a new method) to also return `block_header.next_l2_gas_price`.
2. Update `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` to include `next_l2_gas_price` in the returned value.
3. Change `validate_resource_bounds` to compare against `next_l2_gas_price` instead of `l2_gas_price`.

### Proof of Concept

1. Observe the current block N has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block N was over the gas target).
2. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = P` (exactly equal to current block price, passes the 100% threshold check).
3. Gateway admits the transaction: `P >= 100% * P` ✓.
4. Transaction enters the mempool and is included in block N+1, which uses `next_l2_gas_price = P'`.
5. Blockifier's `check_fee_bounds` rejects: `P < P'` → `MaxGasPriceTooLow` → transaction reverts or is dropped.

The gateway has admitted a transaction that cannot execute, breaking the admission invariant. The converse (valid transaction rejected) follows symmetrically when `P' < P`. [8](#0-7) [9](#0-8) [10](#0-9)

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
