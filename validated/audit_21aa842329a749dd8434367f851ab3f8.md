### Title
Unprivileged Caller Can Invoke `update_rewards` with `disable_rewards: true` to Block All Consensus Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `StakingRewardsManagerImpl` has no caller authorization check. Any address can call it with `disable_rewards: true`, which advances the global `last_reward_block` sentinel without distributing any rewards. Because the contract enforces a strict one-call-per-block invariant on that sentinel, a griefing attacker who front-runs the legitimate consensus caller every block permanently prevents all stakers from accumulating consensus rewards.

### Finding Description
`update_rewards` is part of the public `IStakingRewardsManager` interface and is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

The function unconditionally writes the current block number to `last_reward_block` before it checks `disable_rewards`: [2](#0-1) 

The one-call-per-block guard is: [3](#0-2) 

Because `last_reward_block` is a single global slot shared across all stakers, a successful call in block N — regardless of `disable_rewards` — makes every subsequent call in block N revert with `REWARDS_ALREADY_UPDATED`. There is no check that the caller is the consensus mechanism, the staker, or any privileged role. [4](#0-3) 

### Impact Explanation
An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` once per block:

1. Advances `last_reward_block` to the current block without distributing rewards.
2. Causes every legitimate consensus-triggered call in the same block to revert.
3. Prevents all stakers from ever accumulating consensus-mode block rewards.

This constitutes **permanent freezing of unclaimed yield** for the entire staker set, matching the allowed High impact category.

### Likelihood Explanation
- The function is publicly callable with no role restriction.
- The attacker only needs to submit one transaction per block, which is cheap on Starknet L2.
- No special knowledge, tokens, or privileged access is required — any EOA can execute the attack.
- The attack is sustainable indefinitely as long as the attacker pays gas.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the consensus/sequencer address stored in contract configuration, similar to how `update_rewards_from_attestation_contract` is restricted): [5](#0-4) 

Add an analogous `assert_caller_is_consensus_contract` guard at the top of `update_rewards`, or introduce a dedicated role (e.g., `REWARDS_MANAGER_ROLE`) that must be held by the caller.

### Proof of Concept

```
// Attacker script — run once per block before the consensus sequencer call
loop every block:
    staking_contract.update_rewards(
        staker_address = <any_active_staker>,
        disable_rewards = true
    )
    // last_reward_block is now == current_block
    // Consensus mechanism's call in the same block reverts with REWARDS_ALREADY_UPDATED
    // No staker receives block rewards for this block
```

The attacker requires only:
1. A valid active `staker_address` (readable from public `get_stakers` or events).
2. Enough gas to submit one transaction per block.

The root cause is the absence of any caller authorization in `update_rewards` at `src/staking/staking.cairo` lines 1448–1507, directly analogous to the missing access-control check described in the reference report. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
