### Title
Missing Caller Validation in `update_rewards` Allows Anyone to Freeze Consensus Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `staking.cairo` is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it once per block, consuming the global `last_reward_block` slot and permanently preventing the sequencer from distributing consensus block rewards to all stakers for that block. Repeated across every block, this permanently freezes unclaimed yield for all stakers.

---

### Finding Description

The specification at `docs/spec.md` line 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `update_rewards` in `src/staking/staking.cairo` at lines 1449–1470 contains no such check:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... proceeds to distribute rewards and write last_reward_block
```

The only guard is `general_prerequisites()` (which checks the pause flag) and the single-call-per-block guard `current_block_number > self.last_reward_block.read()`. There is no `assert!(get_caller_address() == sequencer_address, ...)` or equivalent.

The error list for `update_rewards` in the spec (`CONTRACT_IS_PAUSED`, `REWARDS_ALREADY_UPDATED`, `STAKER_NOT_EXISTS`, `INVALID_STAKER`) contains no "CALLER_IS_NOT_SEQUENCER" error, confirming the check was never implemented.

Contrast this with `update_rewards_from_attestation_contract`, which correctly enforces `CALLER_IS_NOT_ATTESTATION_CONTRACT` at line 1153 of the spec and in the implementation.

The `last_reward_block` storage variable is a single global value (not a per-staker map), as evidenced by the read `self.last_reward_block.read()` (no key argument). Writing it for any staker blocks all subsequent calls in the same block.

---

### Impact Explanation

An attacker calls `update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)` at the start of every block. This:

1. Passes all checks (contract unpaused, block number advanced, staker exists and active).
2. Sets `last_reward_block` to the current block number.
3. Distributes zero rewards because `disable_rewards = true`.
4. Causes every subsequent call by the sequencer in that block to revert with `REWARDS_ALREADY_UPDATED`.

Repeated every block, this permanently freezes all consensus block rewards (`unclaimed_rewards_own`) for every staker and every delegation pool. No staker or delegator can ever accumulate new yield from the consensus reward path.

This matches the allowed impact: **Permanent freezing of unclaimed yield or unclaimed royalties (High)**.

---

### Likelihood Explanation

- Requires no privileged role, no leaked key, no bridge compromise.
- Requires only a valid staker address (publicly readable from on-chain events) and enough gas to call the function once per block.
- The attacker has no cost beyond gas; the damage is protocol-wide and irreversible without a contract upgrade.
- Likelihood: **High**.

---

### Recommendation

Add a caller check at the top of `update_rewards` that asserts the caller is the registered sequencer address (stored in contract storage, analogous to how `staking_contract` is stored in `reward_supplier.cairo`):

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    self.general_prerequisites();
    // ...
}
```

### Proof of Concept

1. Deploy the system normally; stakers are registered and active.
2. At the start of block N, any unprivileged address calls:
   ```
   staking.update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)
   ```
3. `last_reward_block` is written to block N; zero rewards are distributed.
4. The sequencer attempts to call `update_rewards` for any staker in block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat step 2 every block.
6. After K epochs, all stakers' `unclaimed_rewards_own` remain at zero; no yield has accrued. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
