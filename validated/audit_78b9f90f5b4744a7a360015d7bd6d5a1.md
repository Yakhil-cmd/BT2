### Title
Griefing Attack via `update_rewards(disable_rewards: true)` Exhausts the Global Per-Block Reward Slot — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract enforces a global, per-block reward-distribution slot via `last_reward_block`. Any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` for any active staker, consuming that slot without distributing any rewards. Because the check is global and there is no access control on the `disable_rewards` flag, an attacker can permanently freeze consensus-phase reward accrual for all stakers at the cost of one cheap transaction per block.

---

### Finding Description

`update_rewards` in `src/staking/staking.cairo` enforces a single-call-per-block invariant through a global storage variable `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // no rewards distributed
    }
    // ... actual reward distribution follows
``` [1](#0-0) 

Two properties combine to create the vulnerability:

1. **No access control on `disable_rewards`.** `general_prerequisites()` only checks the pause flag; there is no role check preventing an arbitrary caller from passing `disable_rewards: true`. The caller does not need to be the staker, the staker's operational address, or any privileged role.

2. **`last_reward_block` is a global, shared slot.** Once any call to `update_rewards` succeeds in a given block, every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. The attacker's call with `disable_rewards: true` writes `last_reward_block` to the current block number and returns early — no rewards are distributed, but the slot is permanently consumed for that block.

The attacker only needs to supply any currently-active staker address (readable from public state) and pay the gas cost of one transaction per block. No token holdings, no stake, no privileged key is required. [2](#0-1) [3](#0-2) 

---

### Impact Explanation

This maps to **High: Permanent freezing of unclaimed yield**. In the consensus phase (`is_pre_consensus()` returns false), `update_rewards` is the sole mechanism by which per-block staking rewards are computed and credited. If the attacker front-runs every block's first `update_rewards` call with `disable_rewards: true`, no staker ever accumulates consensus-phase rewards. Stakers and delegators lose all yield for the duration of the attack with no recourse other than a governance intervention (pause/upgrade).

---

### Likelihood Explanation

**Medium-High.** Starknet transaction fees are low. The attacker needs no capital — only a script that submits one transaction per block. Any active staker address (publicly visible on-chain) serves as the required argument. The attack is sustainable indefinitely and requires no coordination or privileged access.

---

### Recommendation

Restrict who may supply `disable_rewards: true`. The simplest fix is to require that the caller is either the staker's own operational address or a designated consensus/sequencer role when `disable_rewards` is `true`:

```cairo
if disable_rewards {
    // Only the staker's operational address or an authorized consensus caller
    // may disable rewards.
    assert!(
        get_caller_address() == staker_info.operational_address
            || self.roles.is_consensus_caller(get_caller_address()),
        "{}",
        Error::UNAUTHORIZED_DISABLE_REWARDS,
    );
}
```

Alternatively, split the function into two entry points — one for normal reward distribution (public) and one for penalizing a staker (restricted to the consensus/attestation contract).

### Proof of Concept

```
// Attacker script — runs once per block, costs only gas

loop {
    wait_for_new_block();
    let active_staker = read_any_active_staker_from_chain();
    staking_contract.update_rewards(
        staker_address: active_staker,
        disable_rewards: true,   // no access control prevents this
    );
    // last_reward_block is now set to current_block_number
    // all legitimate update_rewards calls this block revert with REWARDS_ALREADY_UPDATED
    // zero rewards distributed to any staker this block
}
```

Step-by-step:
1. Attacker reads any active staker address from public chain state.
2. At the start of each block, attacker calls `update_rewards(active_staker, disable_rewards: true)`.
3. `last_reward_block` is written to the current block number; the function returns early.
4. Every legitimate `update_rewards` call in the same block fails with `REWARDS_ALREADY_UPDATED`.
5. No consensus-phase rewards are distributed for that block.
6. Repeated every block → permanent freezing of all staker and delegator yield.
