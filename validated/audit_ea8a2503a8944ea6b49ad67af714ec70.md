### Title
Malicious Participant Can Pre-Inject Messages at Deterministic Waitpoints to Permanently Deny DKG/Signing Sessions — (File: `src/protocol/internal.rs`, `src/protocol/helpers.rs`)

---

### Summary
The `MessageBuffer` in `src/protocol/internal.rs` routes incoming messages using a `(ChannelTag, Waitpoint)` header that is fully deterministic and publicly computable from participant identities. A malicious participant can pre-inject crafted messages at future protocol waitpoints before the honest protocol logic reaches those rounds. The first-come-first-served deduplication in `recv_from_others` then silently drops the legitimate message when it arrives, causing the protocol to operate on attacker-supplied data and ultimately abort — permanently denying the DKG, reshare, refresh, presign, or signing session for all honest parties.

---

### Finding Description

**Root cause — deterministic, unauthenticated channel addressing**

`ChannelTag::root_shared()` always produces the same 32-byte tag: [1](#0-0) 

Waitpoints are assigned sequentially from 0 via `next_waitpoint()`: [2](#0-1) 

Together, every `MessageHeader` used by every protocol round is fully predictable before the session begins. Any participant who knows the participant list can compute the exact `(root_shared_tag, waitpoint_N)` header for every future round.

**Root cause — unbounded, unauthenticated message acceptance**

`MessageBuffer::push` inserts any message at any header without validating that the round has started or that the sender is expected: [3](#0-2) 

`Comms::push_message` (called by `Protocol::message`) routes every incoming byte directly into this buffer: [4](#0-3) 

**Root cause — first-come-first-served deduplication drops the legitimate message**

`recv_from_others` accepts exactly one message per sender using `seen.put(from)`: [5](#0-4) 

`MessageBuffer::pop` returns the *first* queued message for a header: [6](#0-5) 

If the attacker's pre-injected message is already in the queue when the honest protocol calls `recv_from_others`, it is dequeued first, `seen.put(attacker_id)` returns `true`, and the attacker's slot is marked consumed. When the attacker's legitimate message (or any corrective message) arrives later, `seen.put(attacker_id)` returns `false` and the message is silently discarded.

**Concrete protocol path — DKG commitment-hash round**

In `do_keyshare`, the commitment-hash exchange occupies `wait_round_1`, which is the second waitpoint (index 1) on the shared channel — after the first `do_broadcast` consumes index 0: [7](#0-6) 

The attacker pre-injects a syntactically valid but semantically wrong `HashOutput` at `(root_shared, 1)` before the session starts. When the honest code reaches `wait_round_1`, it dequeues the attacker's stale value. Later, `verify_commitment_hash` computes `H(commitment)` and compares it against the pre-injected hash — they differ, and the session aborts: [8](#0-7) 

The same pre-injection pattern applies to `wait_round_3` (secret-share exchange), the `do_broadcast` rounds for commitments and proofs, and the equivalent waitpoints in presign, sign, and CKD protocols.

The codebase itself contains a test that acknowledges the unbounded injection surface for *unused* waitpoints: [9](#0-8) 

The more severe case — pre-occupying *used* waitpoints — is not tested or mitigated.

---

### Impact Explanation
A single malicious participant can pre-inject one wrong message per future round before the session begins, then go offline. Every honest participant's session aborts when verification fails on the attacker-supplied data. The session cannot be recovered; a new session with a fresh `Comms` instance must be initiated. This constitutes **permanent denial of DKG, reshare, refresh, presign, sign, and CKD** for all honest parties in the affected session, matching the High impact tier.

---

### Likelihood Explanation
Any participant who has been admitted to the protocol knows the full participant list and can compute every future `(ChannelTag, Waitpoint)` header before the session starts. No cryptographic material or privileged access is required beyond legitimate participation. The attack requires sending one message per targeted round before the honest protocol reaches it — a trivially low bar.

---

### Recommendation
1. **Session-scoped channel tags**: Derive the root channel tag from a freshly negotiated, unpredictable session identifier (e.g., the broadcast session-ID established in round 1 of DKG). This makes future headers uncomputable until the session ID is known, eliminating pre-injection.
2. **Round-open gating**: Do not accept messages for a waitpoint until the protocol has explicitly opened that round. Reject or quarantine messages whose waitpoint exceeds the current protocol state.
3. **Sender-round binding**: Verify that the `from` field in each received message matches the expected sender for that round before buffering, rather than relying solely on post-hoc deduplication.

---

### Proof of Concept

```
Setup: participants = [A (attacker), B, C]; threshold = 2; DKG session about to start.

Step 1 — Attacker computes future header:
  channel_tag = SHA256(NEAR_CHANNEL_TAGS_DOMAIN || "root shared")   // constant
  waitpoint   = 1   // index after first do_broadcast consumes index 0

Step 2 — Attacker constructs a syntactically valid but wrong commitment hash:
  fake_hash = [0u8; 32]   // or any value ≠ H(real_commitment)

Step 3 — Attacker calls Protocol::message(from=A, data=encode(header(1), fake_hash))
  on each honest participant's protocol instance BEFORE the session starts.
  MessageBuffer now holds fake_hash at (root_shared, 1) for sender A.

Step 4 — DKG session starts. Round 1 (do_broadcast for session IDs) completes normally.

Step 5 — Round 2: honest code calls recv_from_others at waitpoint 1.
  First dequeue for sender A → fake_hash (pre-injected). seen.put(A) = true.
  A's real commitment hash arrives later → seen.put(A) = false → DROPPED.

Step 6 — Round 3: verify_commitment_hash(A, real_commitment, all_hash_commitments)
  computes H(real_commitment) ≠ fake_hash → ProtocolError::InvalidCommitmentHash.
  Session aborts for all honest participants B and C.

pub async fn recv_from_others<T>(
    chan: &SharedChannel,
    waitpoint: u64,
    participants: &ParticipantList,
    me: Participant,
) -> Result<Vec<(Participant, T)>, ProtocolError>
where
    T: serde::de::DeserializeOwned,
{
    let mut seen = ParticipantCounter::new(participants);
    seen.put(me);
    let mut messages = Vec::with_capacity(participants.others(me).count());

    while !seen.full() {
        let (from, msg) = chan.recv(waitpoint).await?;
        if seen.put(from) {
            messages.push((from, msg));
        }
    }

    Ok(messages)

fn verify_commitment_hash<C: Ciphersuite>(
    session_id: &HashOutput,
    participant: Participant,
    domain_separator: &mut DomainSeparator,
    commitment: &VerifiableSecretSharingCommitment<C>,
    all_hash_commitments: &ParticipantMap<'_, HashOutput>,
) -> Result<(), ProtocolError> {
    let actual_commitment_hash = all_hash_commitments.index(participant)?;
    let commitment_hash =
        domain_separate_hash(domain_separator, &(&participant, &commitment, &session_id))?;
    if *actual_commitment_hash != commitment_hash {
        return Err(ProtocolError::InvalidCommitmentHash);
    }
    Ok(())
}
