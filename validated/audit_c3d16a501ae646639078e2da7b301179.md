### Title
Unchecked Return Value of Exit Precompile `call()` in `EvmErc20.sol` / `EvmErc20V2.sol` Causes Permanent Bridged Fund Loss - (File: `etc/eth-contracts/contracts/EvmErc20.sol`, `etc/eth-contracts/contracts/EvmErc20V2.sol`)

---

### Summary

Both `EvmErc20.sol` and `EvmErc20V2.sol` irreversibly burn user ERC-20 tokens **before** making a low-level assembly `call()` to the Aurora exit precompile. The return value `res` of that `call()` is captured but **never checked**. If the precompile call fails for any reason (e.g., invalid NEAR recipient account ID, out-of-gas in the precompile, or any other precompile-level error), the ERC-20 tokens are permanently destroyed on Aurora while the corresponding NEP-141 tokens remain locked in the Aurora bridge contract on NEAR — a permanent, irrecoverable fund freeze.

---

### Finding Description

In `EvmErc20.sol`, both `withdrawToNear()` and `withdrawToEthereum()` follow this pattern:

```solidity
// EvmErc20.sol lines 53-62
function withdrawToNear(bytes memory recipient, uint256 amount) external override {
    _burn(_msgSender(), amount);          // <-- tokens destroyed here, irreversibly

    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", amount_b, recipient);
    uint input_size = 1 + 32 + recipient.length;

    assembly {
        let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
        // res is NEVER CHECKED — silent failure is indistinguishable from success
    }
}
``` [1](#0-0) 

The identical pattern appears in `withdrawToEthereum()`: [2](#0-1) 

And in both functions of `EvmErc20V2.sol`: [3](#0-2) 

The precompile addresses called are:
- `0xe9217bc70b7ed1f598ddd3199e80b093fa71124f` → `ExitToNear` precompile
- `0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab` → `ExitToEthereum` precompile

Both precompiles are implemented in `engine-precompiles/src/native.rs` and can return errors (e.g., `ERR_INVALID_RECEIVER_ACCOUNT_ID`, `ERR_INVALID_RECIPIENT_ADDRESS`, `OutOfGas`, `ERR_INVALID_IN_STATIC`, etc.): [4](#0-3) 

When the precompile returns an error, the EVM `call()` opcode returns `0` into `res`. Because `res` is never inspected, the Solidity function returns normally — **after the burn has already committed**.

---

### Impact Explanation

**Critical — Permanent freezing of bridged funds.**

The ERC-20 token supply on Aurora is reduced (tokens burned), but the corresponding NEP-141 token balance held by the Aurora contract on NEAR is never released. The user's funds are permanently inaccessible on both sides of the bridge:

- On Aurora: ERC-20 tokens are burned → gone.
- On NEAR: NEP-141 tokens remain locked in the Aurora bridge contract → no mechanism to recover them without a contract upgrade.

This is a connector/bridge accounting bug resulting in permanent fund freeze.

---

### Likelihood Explanation

**Medium.**

The `recipient` parameter in `withdrawToNear()` is a raw `bytes memory` value supplied entirely by the caller. The `ExitToNear` precompile parses this as a NEAR account ID. If the bytes do not form a valid NEAR account ID (e.g., too long, contains invalid characters, or is empty), the precompile returns an error. Since `res` is never checked, the burn succeeds silently and the funds are lost.

A user who mistypes or programmatically constructs an invalid NEAR account ID will lose their tokens with no revert and no warning. This is a realistic, unprivileged, user-triggered path requiring no special access.

Additionally, any future condition that causes the precompile to fail (e.g., gas exhaustion in a tight-gas context, or the precompile being in an unexpected state) would trigger the same permanent loss.

---

### Recommendation

Reorder operations so that the precompile call is made **before** the burn, and revert if the call fails:

```solidity
function withdrawToNear(bytes memory recipient, uint256 amount) external override {
    address sender = _msgSender();

    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", amount_b, recipient);
    uint input_size = 1 + 32 + recipient.length;

    bool success;
    assembly {
        success := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
    }
    require(success, "ExitToNear precompile call failed");

    _burn(sender, amount);  // only burn after confirming the exit was accepted
}
```

Apply the same fix to `withdrawToEthereum()` in both `EvmErc20.sol` and `EvmErc20V2.sol`.

---

### Proof of Concept

1. Deploy `EvmErc20` (or `EvmErc20V2`) as a bridged token on Aurora.
2. Mint tokens to `victim` address.
3. From `victim`, call `withdrawToNear(bytes("INVALID\x00ACCOUNT\xFF"), amount)` — a byte sequence that is not a valid NEAR account ID.
4. Observe:
   - `_burn(victim, amount)` executes: victim's ERC-20 balance is reduced to zero.
   - The `call()` to `0xe9217bc70b7ed1f598ddd3199e80b093fa71124f` returns `0` (failure) because the precompile rejects the invalid account ID.
   - `res` is never checked; the transaction succeeds with no revert.
   - No NEAR-side `ft_transfer` promise is scheduled.
   - The victim's ERC-20 tokens are permanently destroyed; the NEP-141 tokens remain locked in the Aurora contract on NEAR with no recovery path. [1](#0-0) [5](#0-4) [6](#0-5)

impl<I: IO> Precompile for ExitToNear<I> {
    fn required_gas(_input: &[u8]) -> Result<EthGas, ExitError> {
        Ok(costs::EXIT_TO_NEAR_GAS)
    }

    #[allow(clippy::too_many_lines)]
    fn run(
        &self,
        input: &[u8],
        target_gas: Option<EthGas>,
        context: &Context,
        is_static: bool,
    ) -> EvmPrecompileResult {
        // ETH (base) transfer input format: (85 bytes)
        //  - flag (1 byte)
        //  - refund_address (20 bytes), present if the feature "error_refund" is enabled
        //  - recipient_account_id (max MAX_INPUT_SIZE - 20 - 1 bytes)
        // ERC-20 transfer input format: (124 bytes)
        //  - flag (1 byte)
        //  - refund_address (20 bytes), present if the feature "error_refund" is enabled.
        //  - amount (32 bytes)
        //  - recipient_account_id (max MAX_INPUT_SIZE - 1 - (20) - 32 bytes)
        //  - `:unwrap` suffix in a case of wNEAR (7 bytes)
        let required_gas = Self::required_gas(input)?;

        if let Some(target_gas) = target_gas
            && required_gas > target_gas
        {
            return Err(ExitError::OutOfGas);
        }

        // It's not allowed to call exit precompiles in static mode
        if is_static {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_STATIC")));
        } else if context.address != exit_to_near::ADDRESS.raw() {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_DELEGATE")));
        }

        let exit_to_near_params = ExitToNearParams::try_from(input)?;

        let (nep141_address, args, exit_event, method, transfer_near_args) =
            match exit_to_near_params {
                // ETH(base) token transfer
                //
                // Input slice format:
                //  recipient_account_id (bytes) - the NEAR recipient account which will receive
                //  NEP-141 (base) tokens, or also can contain the `:unwrap` suffix in case of
                //  withdrawing wNEAR, or another message of JSON in case of OMNI, or address of
                //  receiver in case of transfer tokens to another engine contract.
                ExitToNearParams::BaseToken(ref exit_params) => {
                    let eth_connector_account_id = self.get_eth_connector_contract_account()?;
                    exit_base_token_to_near(eth_connector_account_id, context, exit_params)?
                }
                // ERC-20 token transfer
                //
                // This precompile branch is expected to be called from the ERC-20 burn function.
                //
                // Input slice format:
                //  amount (U256 big-endian bytes) - the amount that was burned
                //  recipient_account_id (bytes) - the NEAR recipient account which will receive
                //  NEP-141 tokens, or also can contain the `:unwrap` suffix in case of withdrawing
                //  wNEAR, or another message of JSON in case of OMNI, or address of receiver in case
                //  of transfer tokens to another engine contract.
                ExitToNearParams::Erc20TokenParams(ref exit_params) => {
                    exit_erc20_token_to_near(context, exit_params, &self.io)?
                }
