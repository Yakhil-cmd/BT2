### Title
Unprivileged Caller Can Freeze All Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager.update_rewards` is a public function with no access control that accepts a caller-controlled `disable_rewards` parameter. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` in every block, consuming the global `last_reward_block` slot without distributing rewards, permanently denying all stakers their consensus-phase yield.

### Finding Description
`update_rewards` is exposed as a public ABI function under `IStakingRewardsManager` with no caller restriction beyond `general_prerequisites()` (contract not paused, caller not zero address). [1](#0-0) 

The implementation unconditionally writes `last_reward_block = current_block_number` **before** checking `disable_rewards`: [2](#0-1) 

The critical sequence is:
1. Assert `current_block_number > last_reward_block` (global, not per-staker)
2. Validate the provided `staker_address` is active with non-zero balance
3. **Unconditionally write `last_reward_block = current_block_number`**
4. If `disable_rewards == true` **or** `is_pre_consensus()`, return immediately — zero rewards distributed [3](#0-2) 

`last_reward_block` is a single global storage slot: [4](#0-3) 

Because it is global, any call to `update_rewards` in a given block — regardless of which staker is named — exhausts the reward slot for that entire block. A subsequent call in the same block fails with `REWARDS_ALREADY_UPDATED`: [5](#0-4) 

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block. Each call:
- Costs only the gas of one transaction
- Sets `last_reward_block` to the current block
- Distributes zero rewards to anyone
- Blocks every legitimate `update_rewards` call for that block

Note: `update_rewards_from_attestation_contract` (the pre-consensus path) is correctly protected by `assert_caller_is_attestation_contract()`. The vulnerability is exclusive to the consensus-rewards path via `update_rewards`. [6](#0-5) 

### Impact Explanation
In the consensus rewards phase (`is_pre_consensus()` returns false), `update_rewards` is the sole mechanism for distributing per-block STRK rewards to stakers and their delegation pools via `_update_rewards`. If an attacker front-runs this function in every block with `disable_rewards: true`, no staker ever accumulates consensus rewards. This constitutes **permanent freezing of unclaimed yield** for all stakers and pool members — matching the High impact tier. [7](#0-6) 

### Likelihood Explanation
Medium. The attacker requires no special role, no funds beyond gas, and no privileged access — only the ability to submit one transaction per block (~3 seconds on Starknet). Any currently-active staker address with non-zero balance (publicly observable on-chain) satisfies the preconditions. The attack has no profit motive but is trivially cheap to sustain.

### Recommendation
Add access control to `update_rewards` restricting callers to the attestation contract or another designated trusted address. Alternatively, remove the `disable_rewards` parameter entirely and derive the reward-distribution decision internally (e.g., based on whether the staker attested in the current epoch), so that no external caller can suppress reward distribution while still consuming the global block slot.

### Proof of Concept
1. Consensus rewards are active: `consensus_rewards_first_epoch` is set and `current_epoch >= consensus_rewards_first_epoch`.
2. Attacker observes any active staker address `S` with non-zero STRK balance (readable from public events or view functions).
3. In block N, attacker calls `update_rewards(S, disable_rewards: true)`.
4. `last_reward_block` is written to N; no rewards are distributed.
5. Any legitimate call to `update_rewards` in block N fails: `current_block_number (N) > last_reward_block (N)` is false → `REWARDS_ALREADY_UPDATED`.
6. Attacker repeats in block N+1, N+2, …
7. All stakers and pool members are permanently denied consensus rewards for every griefed block.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
