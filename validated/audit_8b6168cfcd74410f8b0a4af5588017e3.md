### Title
Pool Admin Can Set Uncapped Per-Bin Additional Fees via `setPoolBinAdditionalFees`, Bypassing `maxAdminSpreadFeeE6` - (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` forwards `addFeeBuyE6` and `addFeeSellE6` directly to the pool with no upper-bound check, while the analogous `setPoolAdminFees` enforces `maxAdminSpreadFeeE6`. A pool admin can set either value to `type(uint16).max = 65535` (6.5535% in E6), which is then added on top of the base spread fee in every swap through that bin, extracting excess fees from traders beyond the documented admin fee cap boundary.

---

### Finding Description

`setPoolAdminFees` enforces the cap system:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
```

`setPoolBinAdditionalFees` has no equivalent check:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

The pool-side `setBinAdditionalFees` only validates the bin index range, not the fee values:

```solidity
// MetricOmmPool.sol:469-472
if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
BinState storage s = _binStates[bin];
s.addFeeBuyE6 = addFeeBuyE6;
s.addFeeSellE6 = addFeeSellE6;
```

During every swap, the per-bin additional fee is added directly to the base fee passed into `SwapMath`:

```solidity
// MetricOmmPool.sol:910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6),
// MetricOmmPool.sol:999
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6),
// MetricOmmPool.sol:1088
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6),
```

The hard cap for global spread fees is `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20%). The per-bin additional fee field is `uint16`, so its maximum is `65535` (6.5535%), which is entirely uncapped and additive on top of whatever global spread fee is already set.

---

### Impact Explanation

Traders swapping through a bin where the pool admin has set `addFeeBuyE6 = 65535` pay up to 6.5535% additional fee on top of the base spread fee. This fee is charged on every swap through that bin and accrues to LPs (not the protocol), meaning the pool admin can silently redirect excess value from traders to LP positions they control. This is a direct loss of trader principal beyond any documented admin fee boundary, satisfying the "admin-boundary break: pool admin exceeds caps" impact gate.

---

### Likelihood Explanation

The pool admin is a designated per-pool role reachable via `poolAdmin[pool]`. The call path is a single transaction with no timelock. Any pool admin — including one who obtained the role legitimately — can invoke this at any time on any bin in the pool's configured range. No special conditions are required beyond holding the pool admin role.

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` analogous to the one in `setPoolAdminFees`. Either reuse `maxAdminSpreadFeeE6` or introduce a dedicated `maxAdminBinAdditionalFeeE6` constant:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

---

### Proof of Concept

1. Pool admin calls `factory.setPoolBinAdditionalFees(pool, 0, type(uint16).max, type(uint16).max)` — no revert.
2. `BinState.addFeeBuyE6` for bin 0 is now `65535`.
3. A trader executes a buy-token0 swap through bin 0.
4. Swap math computes effective fee as `baseFeeX64 + mulDiv(65535, ONE_X64, 1e6)` — 6.5535% additional fee charged on top of the base spread.
5. Trader pays 6.5535% more than the documented admin fee cap permits; the excess accrues to bin liquidity (LP-controlled).
