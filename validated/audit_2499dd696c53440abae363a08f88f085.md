### Title
`exit_to_near_precompile_callback` `require_running` Guard Permanently Freezes In-Flight wNEAR Exit Funds When Engine Is Paused — (`engine/src/contract_methods/connector.rs`)

### Summary

When a user exits wNEAR from the EVM via the `ExitToNear` precompile, the engine schedules a two-step NEAR promise chain: first `near_withdraw` on the wNEAR contract (which burns the NEP-141 wNEAR and credits NEAR to the Aurora account), then a callback `exit_to_near_precompile_callback` that transfers the unwrapped NEAR to the user. If the engine is paused between these two receipts, the callback fails immediately at the `require_running` guard before the NEAR transfer executes. The NEAR is permanently stranded in the Aurora contract with no recovery path, while the user's EVM-side wNEAR tokens are already burned.

### Finding Description

The `ExitToNear` precompile, when handling a wNEAR unwrap, constructs a two-promise chain:

1. **Base promise**: `near_withdraw` on the wNEAR NEP-141 contract — burns wNEAR and deposits NEAR into the Aurora engine account.
2. **Callback**: `exit_to_near_precompile_callback` on the engine — transfers the unwrapped NEAR to the user's target NEAR account. [1](#0-0) 

The callback is implemented as: [2](#0-1) 

The critical guard is at line 203:

```rust
require_running(&state)?;
``` [3](#0-2) 

`require_running` returns an error (causing a panic) if `state.is_paused == true`: [4](#0-3) 

If the engine owner calls `pause_contract` between the `near_withdraw` receipt and the `exit_to_near_precompile_callback` receipt, the callback panics before reaching the `transfer_near` branch:

```rust
if let Some(args) = args.transfer_near {
    // Transfer NEAR to user — NEVER REACHED when paused
    let promise_id = handler.promise_create_batch(&promise);
    handler.promise_return(promise_id);
}
``` [5](#0-4) 

At this point:
- The user's wNEAR ERC-20 balance in the EVM is already burned (irreversible EVM state change).
- The `near_withdraw` already executed: wNEAR NEP-141 is burned and NEAR is credited to the Aurora engine account.
- The NEAR transfer to the user never happens.
- The NEAR is permanently stuck in the Aurora contract with no recovery mechanism.

The same issue applies to the `error_refund` path: if `ft_transfer` fails and the engine is paused, the refund branch is also unreachable. [6](#0-5) 

### Impact Explanation

**Critical — Permanent freezing of user funds.**

The NEAR obtained from `near_withdraw` is credited to the Aurora engine account. Once the callback fails, there is no re-entrant mechanism to retry the transfer. The NEAR remains locked in the Aurora contract indefinitely. The user has lost both their EVM-side wNEAR tokens (burned) and the corresponding NEAR (stranded). This matches the "Permanent freezing of funds" impact class.

### Likelihood Explanation

The engine can be legitimately paused by the owner via `pause_contract` for security incidents. NEAR receipts are processed asynchronously across blocks; the window between the `near_withdraw` receipt and the callback receipt spans at least one block. Any pause issued in that window triggers the loss. The owner does not need to act maliciously — a routine emergency pause during a security event is sufficient. Users who happen to have in-flight wNEAR exits at that moment permanently lose their funds.

### Recommendation

Remove or relocate the `require_running` guard in `exit_to_near_precompile_callback`. The callback is a system-internal receipt (enforced by `env.assert_private_call()`) and its sole purpose is to settle an already-committed state change. It should be allowed to execute regardless of the engine's pause state. Alternatively, the `transfer_near` and `refund` branches should be extracted into a separate function that bypasses the pause check, similar to how `assert_private_call` already restricts the caller.

### Proof of Concept

1. User calls the `ExitToNear` precompile with wNEAR, specifying a NEAR recipient. EVM wNEAR balance is burned.
2. Engine schedules `near_withdraw` on the wNEAR contract → callback `exit_to_near_precompile_callback`.
3. `near_withdraw` executes successfully: wNEAR NEP-141 burned, NEAR credited to Aurora engine account.
4. Before the callback receipt is processed, the engine owner calls `pause_contract`.
5. `exit_to_near_precompile_callback` is invoked; `require_running` returns `Err(ERR_PAUSED)`; the function panics.
6. The `transfer_near` branch is never reached. NEAR remains in the Aurora account. User's wNEAR ERC-20 tokens are gone. Funds are permanently frozen. [7](#0-6) [8](#0-7)

```rust
pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

#[named]
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
