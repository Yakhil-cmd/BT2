### Title
Unprivileged Caller Can Permanently Freeze All Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable and accepts a `disable_rewards` parameter. When called with `disable_rewards: true`, it writes the global `last_reward_block` to the current block **before** checking the flag, then returns early without distributing any rewards. Because `last_reward_block` is a single global slot and only one call per block is permitted, any unprivileged caller can invoke this every block to permanently prevent all consensus rewards from reaching any staker or pool member.

---

### Finding Description

`update_rewards` is exposed via `IStakingRewardsManager` with no role-based access control — only `general_prerequisites()` (not-paused + caller-not-zero) guards it. [1](#0-0) 

The function accepts a caller-supplied `disable_rewards: bool` parameter. [2](#0-1) 

The critical ordering flaw: `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

`last_reward_block` is a single global `BlockNumber` field — not per-staker: [4](#0-3) 

The guard that enforces one call per block reads this same global: [5](#0-4) 

**Attack sequence:**

1. Consensus rewards are active (`is_pre_consensus()` returns false).
2. Attacker calls `update_rewards(any_active_staker, disable_rewards: true)` at the start of block N.
3. `last_reward_block` is set to N; the function returns early — zero rewards distributed.
4. Any legitimate call to `update_rewards` for any staker in block N now reverts with `REWARDS_ALREADY_UPDATED`.
5. Attacker repeats in every subsequent block.

All block rewards for every staker and every delegation pool are permanently lost. The attacker pays only gas; there is no profit motive.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Consensus block rewards are the primary yield mechanism in V3. By consuming the global `last_reward_block` slot each block with `disable_rewards: true`, an attacker permanently prevents `_update_rewards` from executing for any staker. Pool members' `cumulative_rewards_trace` is never updated, so `calculate_rewards` always returns zero for epochs after the attack begins. Stakers' `unclaimed_rewards_own` is never incremented. The yield is not deferred — it is permanently lost because block rewards are not retroactively recalculated. [6](#0-5) 

---

### Likelihood Explanation

**High.** The function is part of the public ABI with no privileged caller restriction. Any EOA or contract can call it. The only preconditions are: (a) the contract is not paused, (b) the caller is non-zero, and (c) a valid active staker address is supplied — all trivially satisfiable. The attacker's cost is one transaction per block of gas.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so that consuming the block slot only occurs when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Update last block rewards.
self.last_reward_block.write(current_block_number);

// ... proceed with reward calculation
```

Alternatively, add role-based access control (e.g., `only_operator` or `only_app_governor`) to the `disable_rewards: true` path, so only the consensus mechanism can suppress rewards for a given block.

---

### Proof of Concept

```
// Precondition: consensus rewards are active, at least one active staker exists.

// Attacker (any address) calls in block N:
staking_contract.update_rewards(
    staker_address: any_active_staker,
    disable_rewards: true,
);
// Result: last_reward_block = N, no rewards distributed.

// Legitimate node operator tries to distribute rewards in block N:
staking_contract.update_rewards(
    staker_address: legitimate_staker,
    disable_rewards: false,
);
// Reverts: REWARDS_ALREADY_UPDATED

// Attacker repeats every block → all consensus rewards permanently frozen.
``` [7](#0-6)

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
