### Title
Missing Zero-Address Validation on `reward_address` Enables Permanent Freezing of Unclaimed Yield - (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary
Both `staking.cairo::change_reward_address` and `pool.cairo::change_reward_address` (and the initial `stake` entrypoint) accept any `ContractAddress` as `reward_address` with only a token-address exclusion check. There is no zero-address guard. A staker or pool member can register `reward_address = 0`, causing all subsequent reward-claim and unstake flows to either permanently burn unclaimed STRK (if the token allows zero-address transfers) or permanently revert until the address is corrected.

---

### Finding Description

**`staking.cairo::stake`** accepts `reward_address` with a single guard:

```cairo
assert!(
    !self.does_token_exist(token_address: reward_address),
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
``` [1](#0-0) 

No zero-address check is present. The address is stored directly into `InternalStakerInfo`. [2](#0-1) 

**`staking.cairo::change_reward_address`** carries the same single guard and no zero-address assertion: [3](#0-2) 

**`pool.cairo::change_reward_address`** only checks the token address, not zero: [4](#0-3) 

When rewards are later disbursed, `send_rewards_to_staker` blindly transfers to whatever `reward_address` is stored:

```cairo
token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
``` [5](#0-4) 

Likewise in `pool.cairo::claim_rewards`:

```cairo
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
``` [6](#0-5) 

---

### Impact Explanation

Two outcomes depending on the STRK token's zero-address behavior:

1. **If the STRK ERC20 reverts on `transfer` to address 0** (standard OpenZeppelin Cairo behavior): every call to `claim_rewards` and `unstake_action` reverts. `unstake_action` calls `send_rewards_to_staker` before returning stake, so the staker is blocked from completing their exit until they call `change_reward_address` — **temporary freezing of unclaimed yield and staked funds**. [7](#0-6) 

2. **If the STRK ERC20 does not revert on `transfer` to address 0**: rewards are silently sent to the zero address and are permanently unrecoverable — **permanent loss of unclaimed yield**.

Both outcomes fall within the allowed impact scope: *"High: Permanent freezing of unclaimed yield"* / *"Temporary freezing of funds"*.

---

### Likelihood Explanation

Any active staker or pool member can call `change_reward_address(0)` at any time — no privilege required. The call succeeds and emits a `StakerRewardAddressChanged` / `PoolMemberRewardAddressChanged` event with `new_address: 0`. A user could do this accidentally (e.g., a frontend bug passing an uninitialized value) or a griefing actor could do it to a staker they control. The path is fully reachable and requires no special role. [8](#0-7) [9](#0-8) 

---

### Recommendation

Add a zero-address guard in all three entry points:

```cairo
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

This should be added to:
- `staking.cairo::stake` alongside the existing token-address check
- `staking.cairo::change_reward_address` alongside the existing token-address check
- `pool.cairo::change_reward_address` alongside the existing token-address check

The error variant `GenericError::ZERO_ADDRESS` already exists in the codebase. [10](#0-9) 

---

### Proof of Concept

1. Staker calls `stake(reward_address: 0, operational_address: X, amount: MIN_STAKE)` — succeeds, no revert.
2. Staker earns rewards over several epochs via attestation.
3. Staker calls `claim_rewards(staker_address)` — `send_rewards_to_staker` attempts `checked_transfer(recipient: 0, amount: rewards)`.
   - If STRK ERC20 reverts: `claim_rewards` reverts; staker cannot claim until they call `change_reward_address`.
   - If STRK ERC20 does not revert: rewards are sent to address 0 and permanently lost.
4. Staker calls `unstake_intent()` then `unstake_action()` — `unstake_action` also calls `send_rewards_to_staker` before returning stake, so the entire exit is blocked (or rewards are burned) for the same reason. [11](#0-10) [12](#0-11)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                !self.does_token_exist(token_address: reward_address),
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let old_address = staker_info.reward_address;

            // Update reward_address and commit to storage.
            staker_info.reward_address = reward_address;
            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.
            self
                .emit(
                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,
                    },
                );
        }

        ) {
            let reward_address = staker_info.reward_address;

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
