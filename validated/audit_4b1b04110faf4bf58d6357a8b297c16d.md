### Title
Gateway Stateful Validator Checks Only L2 Gas Price, Admitting Transactions With Insufficient L1/L1-Data Gas Prices - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds validation only compares the transaction's L2 gas `max_price_per_unit` against the previous block's L2 gas price. It never validates L1 gas price or L1 data gas price. This is the direct analog of the external report's root cause: an accounting check that only reads one component of a multi-component balance, leaving the rest unchecked. A transaction with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` passes every gateway admission check but is rejected by the blockifier's `check_fee_bounds` at execution time, meaning the gateway admits transactions that are invalid under the sequencer's own execution rules.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads only `strk_gas_prices.l2_gas_price` from the previous block and delegates to `validate_tx_l2_gas_price_within_threshold`: [1](#0-0) 

`validate_tx_l2_gas_price_within_threshold` matches only on `AllResources` and inspects only `l2_gas.max_price_per_unit`. The `L1Gas` arm is a no-op, and within `AllResources` neither `l1_gas` nor `l1_data_gas` prices are touched. The developer TODO at line 358 explicitly acknowledges the gap: [2](#0-1) 

In contrast, the blockifier's `check_fee_bounds` — called inside `perform_pre_validation_stage` during actual execution — iterates over **all three** resource types and rejects any transaction whose `max_price_per_unit` is below the current block's gas price for that resource: [3](#0-2) 

The mempool-level config description mirrors the same gap — it also only mentions L2 gas price as the admission criterion: [4](#0-3) 

**Attack path (unprivileged):**

1. Craft a V3 `AllResources` transaction with:
   - `l2_gas.max_price_per_unit` ≥ `min_gas_price_percentage × previous_block_l2_gas_price` (passes stateful check)
   - `l2_gas.max_amount > 0` (makes `max_possible_fee > 0`, passes stateless check)
   - `l1_gas.max_price_per_unit = 0`
   - `l1_data_gas.max_price_per_unit = 0`
2. Submit via the gateway. `StatelessTransactionValidator::validate_resource_bounds` passes because `max_possible_fee(Tip::ZERO) > 0`.
3. `validate_tx_l2_gas_price_within_threshold` passes because only L2 price is checked.
4. Transaction is forwarded to the mempool and admitted.
5. At execution time, `check_fee_bounds` compares `l1_gas.max_price_per_unit (= 0)` against the current block's non-zero L1 gas price and raises `ResourceBoundsError::MaxGasPriceTooLow`, rejecting the transaction.

The stateless validator's `min_gas_price` guard only covers L2 gas price: [5](#0-4) 

No existing guard at the gateway or mempool layer checks L1 or L1-data gas prices against the current block's prices.

### Impact Explanation

Transactions that are structurally valid (correct signature, nonce, non-zero fee) but economically invalid (L1 gas price below the current block's price) are admitted to the mempool. They consume mempool capacity and are only rejected when the batcher attempts to execute them. An unprivileged attacker can flood the mempool with such transactions at negligible cost (only L2 gas price must be non-trivial), degrading throughput and displacing legitimate transactions. This matches the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

The attack requires no special privilege. Any user with a valid account can submit a V3 transaction with zero L1 and L1-data gas prices. The condition is trivially constructable and the gateway provides no feedback that distinguishes this case from a valid submission.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or introduce a parallel function) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK prices, applying the same `min_gas_price_percentage` threshold. The previous block's full `GasPriceVector` is already available via `get_block_info().gas_prices.strk_gas_prices`. The existing TODO comment at line 358 already identifies this as a known gap.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
