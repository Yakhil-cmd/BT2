### Title
System-Wide Pause Permanently Blocks Fund Withdrawal for Stakers and Delegators Who Have Completed Their Exit Window - (File: src/staking/staking.cairo)

### Summary
The staking contract's `general_prerequisites()` guard unconditionally reverts on every state-changing call when `is_paused` is `true`. This includes `unstake_action()` and `remove_from_delegation_pool_action()` — the two functions that transfer principal back to stakers and delegators. A staker or delegator who has already signalled exit intent and waited out the full `exit_wait_window` has a vested right to withdraw their principal, yet the pause blocks that withdrawal with no exception, temporarily freezing their funds.

### Finding Description
`general_prerequisites()` is the single shared guard called at the top of every mutating staking function: [1](#0-0) [2](#0-1) 

Both `unstake_action` and `remove_from_delegation_pool_action` call `self.general_prerequisites()` as their very first statement. When `is_paused` is `true` the call reverts with `"Contract is paused"`.

The pool-level withdrawal path (`exit_delegation_pool_action` in `pool.cairo`) does not check the pause flag itself, but it internally calls `staking_pool_dispatcher.remove_from_delegation_pool_action(...)` on the staking contract: [3](#0-2) 

That cross-contract call hits `general_prerequisites()` in the staking contract and reverts, so the entire pool withdrawal reverts too.

The pause is set by the `security_agent` role and can only be lifted by the `security_admin` role: [4](#0-3) 

There is no bypass for users whose exit window has already elapsed.

### Impact Explanation
Any staker or delegator who called `unstake_intent()` / `exit_delegation_pool_intent()` before the pause, and whose `exit_wait_window` has since elapsed, holds a matured withdrawal right but cannot execute it. Their principal (STRK or BTC tokens) remains locked in the staking contract until the `security_admin` unpauses. This is a **temporary freezing of funds** for an unbounded number of users simultaneously.

The spec itself documents this as a hard pre-condition: [5](#0-4) [6](#0-5) 

### Likelihood Explanation
The `security_agent` is expected to pause the contract in response to any detected anomaly. Because the `exit_wait_window` defaults to one week, there will almost always be stakers or delegators in the exit window at any given moment. A pause of even a few hours after their window has elapsed freezes their principal with no recourse. No attacker action is required — the freeze is a direct consequence of a legitimate security response.

### Recommendation
Exempt matured withdrawal calls from the pause guard. Specifically, `unstake_action` and `remove_from_delegation_pool_action` (and by extension `exit_delegation_pool_action`) should be allowed to execute when the caller's exit window has already elapsed, even while the contract is paused. One approach is to skip `general_prerequisites()` in those two functions and instead only assert the pause for operations that add new stake or modify staking parameters.

### Proof of Concept
1. Staker calls `unstake_intent()` while the contract is unpaused. The `exit_wait_window` (e.g., 1 week) begins.
2. One week passes. The staker's `unstake_time` is now in the past; they have a matured right to withdraw.
3. The `security_agent` pauses the contract (`is_paused = true`).
4. Staker calls `unstake_action(staker_address)`.
5. `general_prerequisites()` fires, reads `is_paused == true`, and panics with `"Contract is paused"`.
6. The staker's principal remains locked in the staking contract.

The same path applies to a pool member: `exit_delegation_pool_action` → `remove_from_delegation_pool_action` → `general_prerequisites()` → revert. [7](#0-6)

```text
        fn pause(ref self: ContractState) {
            self.roles.only_security_agent();
            if self.is_paused() {
                return;
            }
            self.is_paused.write(true);
            self.emit(PauseEvents::Paused { account: get_caller_address() });
        }

        fn unpause(ref self: ContractState) {
            self.roles.only_security_admin();
            if !self.is_paused() {
                return;
            }
            self.is_paused.write(false);
            self.emit(PauseEvents::Unpaused { account: get_caller_address() });
        }
```
