### Title
Malicious Participant Can Abort Presign for All Honest Parties via Unauthenticated Alpha/Beta Shares — (`src/ecdsa/ot_based_ecdsa/presign.rs`)

### Summary
In `do_presign`, each participant broadcasts `(alpha_i, beta_i)` in round 2. These values are summed and checked against public triple commitments (`K + A`, `X + B`). Because the check is only on the aggregate and no per-party commitment or proof is required, a single malicious participant can send arbitrary (e.g., zero) values, causing the aggregate check to fail for every honest party with no ability to identify the culprit.

### Finding Description
`do_presign` collects `(alpha_j, beta_j)` from all other participants via `recv_from_others` and accumulates them into `alpha` and `beta`. The only validation is the aggregate equality check at lines 162–168. There is no per-party binding: no commitment to `alpha_j` is published before the value is revealed, and no zero-knowledge proof ties each party's broadcast to their Lagrange-weighted shares of the triple. A malicious participant sends `alpha_j = 0, beta_j = 0`. The aggregate `alpha` is now missing that party's contribution, so `G * alpha ≠ K + A`, the check fails, and all honest parties abort. [5](#0-4) 

### Impact Explanation
Every honest party aborts with `ProtocolError::AssertionFailed("received incorrect shares of additive triple phase.")`. Because the check is aggregate-only, no party can identify the malicious participant. The malicious party can repeat this attack on every presign attempt, permanently denying signing. This matches **High: Permanent denial of signing for honest parties under valid protocol inputs**.

### Likelihood Explanation
Any participant in the presign session can trigger this. No special privilege is required beyond being a valid participant. The attack requires only sending a single wrong scalar pair in round 2, which is trivially achievable by a malicious library caller.

### Recommendation
Before accumulating `alpha_j` and `beta_j`, each party should publish a commitment (e.g., `G * alpha_j`, `G * beta_j`) in a prior broadcast round. After all values are revealed, each party verifies that `G * alpha_j == K_j + A_j` (using per-party public triple share commitments) before accepting the value. This allows honest parties to identify and exclude the malicious participant rather than aborting globally.

### Proof of Concept
In a test with 3 participants, intercept one party's outgoing `(alpha_i, beta_i)` message and replace it with `(Scalar::ZERO, Scalar::ZERO)`. Assert that all other parties' `do_presign` futures resolve to `Err(ProtocolError::AssertionFailed("received incorrect shares of additive triple phase."))`. The existing test harness in `src/ecdsa/ot_based_ecdsa/presign.rs` (lines 202–269) already sets up the exact participant/triple structure needed to reproduce this. [6](#0-5)
