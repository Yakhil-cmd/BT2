### Title
Single-Step Owner Transfer Allows Irrecoverable Loss of Contract Control - (File: engine/src/contract_methods/admin.rs)

### Summary
The `set_owner` function in the Aurora Engine immediately replaces `state.owner_id` with the caller-supplied account ID in a single transaction, with no confirmation step from the new owner. An accidental typo or wrong account ID permanently transfers all owner privileges — including the ability to upgrade the contract and attach full access keys — to an unintended account with no recovery path.

### Finding Description
`set_owner` reads the new owner from input and writes it directly to state in one call:

```rust
state.owner_id = args.new_owner;
state::set_state(&mut io, &state)?;
```

There is no pending-owner staging, no acceptance call from the new account, and no time-lock. The only guard is `require_owner_only` (verifying the caller is the current owner) and a same-owner check. Once the call succeeds, the old owner loses all privileges immediately and irrevocably.

The owner account controls every critical administrative function:
- `upgrade()` — deploys arbitrary new WASM bytecode to the contract
- `attach_full_access_key()` — adds a full-access key to the contract account
- `pause_contract()` / `resume_contract()`
- `set_key_manager()` — controls which account can add/remove relayer function-call keys
- `stage_upgrade()` / `resume_precompiles()`

The same single-step pattern also exists in `set_key_manager`:

```rust
state.key_manager = key_manager;
state::set_state(&mut io, &state)?;
```

And in `set_eth_connector_contract_account`, which immediately overwrites the account that handles all ETH bridge deposits and withdrawals.

### Impact Explanation
**Critical — Direct theft of user funds / Permanent freezing of funds.**

If the owner accidentally supplies a wrong `new_owner` account ID:

- **Scenario A (wrong account is attacker-controlled):** The attacker immediately calls `upgrade()` to deploy malicious WASM that redirects all bridged ETH and ERC-20 balances, or calls `attach_full_access_key()` to gain a full-access key and drain the contract account. All user funds at rest in the Aurora bridge are at risk.
- **Scenario B (wrong account is non-existent or uncontrolled):** No one can ever call `upgrade()`, `pause_contract()`, or any other owner-gated function again. If a critical bug is later discovered, the contract cannot be patched or paused, permanently freezing all user funds.

The same applies to `set_eth_connector_contract_account`: pointing it at a wrong account immediately breaks all bridge deposit/withdrawal flows, freezing bridged ETH.

### Likelihood Explanation
NEAR account IDs are human-readable strings (e.g., `aurora.near`, `aurora-ops.near`). A single character typo produces a syntactically valid but wrong account ID that passes all current validation. The `require_owner_only` check only verifies the *caller*, not the *new* account. The operation is irreversible the moment the transaction is finalized.

### Recommendation
Implement a two-step ownership transfer:
1. The current owner calls `propose_owner(new_owner)`, which stores a *pending* owner in state.
2. The pending owner calls `accept_owner()`, which finalizes the transfer only if `predecessor_account_id == pending_owner`.

Apply the same pattern to `set_key_manager` and `set_eth_connector_contract_account`. This mirrors the `approve`/`transferFrom` pattern cited in the original report and ensures the new account demonstrably exists and is accessible before privileges are transferred.

#[named]
pub fn set_owner<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;

        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;

        let args: SetOwnerArgs = io.read_input_borsh()?;
        if state.owner_id == args.new_owner {
            return Err(errors::ERR_SAME_OWNER.into());
        }

        state.owner_id = args.new_owner;
        state::set_state(&mut io, &state)?;

        Ok(())
    })
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn set_key_manager<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;

        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;

        let key_manager =
            serde_json::from_slice::<RelayerKeyManagerArgs>(&io.read_input().to_vec())
                .map(|args| args.key_manager)
                .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;

        if state.key_manager == key_manager {
            return Err(errors::ERR_SAME_KEY_MANAGER.into());
        }

        state.key_manager = key_manager;
        state::set_state(&mut io, &state)?;

        Ok(())
    })
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn set_eth_connector_contract_account<I: IO + Copy, E: Env>(
    io: I,
    env: &E,
) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let is_private = env.assert_private_call();

        if is_private.is_err() {
            require_owner_only(&state, &env.predecessor_account_id())?;
        }

        let args: SetEthConnectorContractAccountArgs = io.read_input_borsh()?;

        set_connector_account_id(io, &args.account);
        set_connector_withdraw_serialization_type(io, &args.withdraw_serialize_type);

        Ok(())
    })
}
