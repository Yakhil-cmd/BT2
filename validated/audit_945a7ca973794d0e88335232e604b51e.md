### Title
Missing Access Control on `update_rewards` Allows Any Caller to Redirect or Permanently Freeze Block Rewards - (File: `src/staking/staking.cairo`)

### Summary

`IStakingRewardsManager::update_rewards` is publicly callable by any non-zero address despite the specification requiring it to be restricted to the Starkware sequencer. Because `last_reward_block` is a single global lock, any caller can consume the per-block reward slot — either redirecting rewards to an arbitrary staker or permanently destroying the block's rewards by passing `disable_rewards: true`.

### Finding Description

`update_rewards` is defined in `IStakingRewardsManager` and implemented in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks: unpaused + caller != zero
    ...
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distributes rewards to staker_address
``` [1](#0-0) 

`general_prerequisites` performs no role check — it only asserts the contract is unpaused and the caller is non-zero: [2](#0-1) 

The protocol specification explicitly states the access control for this function is **"Only starkware sequencer"**: [3](#0-2) 

`last_reward_block` is a single global `BlockNumber` storage variable — not per-staker: [4](#0-3) 

The guard at the top of `update_rewards` enforces that only one call can succeed per block: [5](#0-4) 

Once `last_reward_block` is written, every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

### Impact Explanation

Two distinct attack paths exist:

**Path 1 — Permanent freezing of unclaimed yield (High):**
An attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)`. The function writes `last_reward_block` and returns early without distributing any rewards. The sequencer's subsequent call for the legitimate block-producing staker reverts. The block's entire reward allocation is permanently lost — no staker receives it.

**Path 2 — Theft / redirection of unclaimed yield (High):**
An attacker front-runs the sequencer's `update_rewards(staker_A, false)` with `update_rewards(staker_B, false)`. Staker B receives rewards it did not earn; staker A (the actual block producer) receives nothing for that block. The attacker does not need to control staker B — any valid active staker address suffices.

Both paths match the allowed impact categories: **Permanent freezing of unclaimed yield** and **Theft of unclaimed yield**.

### Likelihood Explanation

The function is part of the public ABI (`#[abi(embed_v0)]`), callable by any EOA or contract on Starknet. No special privilege, leaked key, or bridge compromise is required. An attacker only needs to observe the mempool (or simply call the function at the start of each block) to reliably execute either attack. The attack is repeatable every block once consensus rewards are active.

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` restricts callers to the attestation contract: [6](#0-5) 

Concretely, introduce a stored `sequencer_address` (or reuse an existing privileged role) and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation in `update_rewards`.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Staker A produces block N and the sequencer prepares to call `update_rewards(staker_A, false)`.
3. Attacker observes the pending transaction and submits `update_rewards(staker_A, disable_rewards: true)` (or `update_rewards(staker_B, false)`) with higher gas priority.
4. Attacker's transaction executes first: `last_reward_block` is set to block N.
5. Sequencer's transaction reverts with `REWARDS_ALREADY_UPDATED`.
6. In the `disable_rewards: true` variant: no rewards are distributed for block N — permanently frozen.
7. In the `staker_B` variant: staker B's `unclaimed_rewards_own` is incremented by the full block reward; staker A receives zero.
8. The attack is repeatable every block at negligible cost. [7](#0-6) [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
