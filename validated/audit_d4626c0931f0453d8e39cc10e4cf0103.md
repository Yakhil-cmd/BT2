### Title
Multi-NFT Royalty Total Underestimated in WalletConnect Confirmation Dialog - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary
`formatAmountWithRoyalties` in `parseCommandDisplay.ts` divides the fungible payment amount equally across all NFTs before applying each NFT's royalty percentage, instead of applying each royalty to the full amount. When a WalletConnect `take_offer` or `create_offer_for_ids` command involves multiple NFTs with royalties, the "Total Amount with Royalties" shown in the confirmation dialog is materially lower than the amount the blockchain will actually deduct. A connected dapp can exploit this to get the user to approve a spend larger than what is displayed.

### Finding Description

`formatAmountWithRoyalties` computes the royalty total as:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // divides by NFT count
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [1](#0-0) 

Each NFT's royalty percentage is applied to `amount / N` (where N is the number of NFTs) rather than to the full `amount`. The correct formula is to sum `(amount × royaltyPct_i) / 10_000` for each NFT independently.

The test suite itself encodes the buggy result as the expected value: for 2 NFTs with royalty percentages 500 (5%) and 10 (0.1%) and a payment of 100,000,000 mojos, the test expects `amountWithRoyalties: '0.00010255'` (split-based), whereas the correct on-chain total is `0.0001051` (full-amount-based). [2](#0-1) 

`withRoyaltyTotals` feeds this value into the `DisplayWalletDelta` that is rendered in the WalletConnect confirmation dialog under "Total Amount with Royalties": [3](#0-2) 

The dialog renders `line.amountWithRoyalties` as the authoritative total the user is asked to approve: [4](#0-3) 

### Impact Explanation

The blockchain enforces royalties independently of what the GUI displays. When the user clicks "Confirm", the actual on-chain deduction equals `amount + Σ(amount × royaltyPct_i / 10_000)`. The dialog shows a smaller number. The gap scales with the number of NFTs and their royalty percentages. A dapp that crafts an offer with many high-royalty NFTs can make the displayed total arbitrarily lower than the real spend, causing the user to approve a transaction they would have rejected had they seen the correct figure.

This matches the allowed High impact: *"Corruption … of … offer … state that causes a user to approve … the wrong … amount."*

### Likelihood Explanation

Any WalletConnect-paired dapp can send a `take_offer` or `create_offer_for_ids` command. The command always triggers the confirmation dialog (it is not in the bypass list). The dapp only needs to include two or more NFTs with non-zero royalties in the offer. No special privileges beyond an established WalletConnect pairing are required. [5](#0-4) 

### Recommendation

Remove the division by `royaltyPercentages.length`. Apply each royalty percentage to the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

Update the corresponding test expectation to `'0.0001051'` (5% + 0.1% of 0.0001 XCH).

### Proof of Concept

Existing test at line 251–334 of `parseCommandDisplay.test.ts` demonstrates the bug:

- Payment: 100,000,000 mojos (0.0001 XCH)
- NFT 1 royalty: 500 (5%), NFT 2 royalty: 10 (0.1%)
- **Buggy display**: `splitAmount = 50,000,000`; royalties = `2,500,000 + 50,000 = 2,550,000`; total shown = **0.00010255 XCH**
- **Correct on-chain deduction**: royalties = `5,000,000 + 100,000 = 5,100,000`; actual total = **0.0001051 XCH**
- **Discrepancy**: user approves thinking they pay 0.00010255 XCH but the blockchain deducts 0.0001051 XCH — ~2.4% more than shown, growing proportionally with NFT count and royalty rates. [6](#0-5)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}
