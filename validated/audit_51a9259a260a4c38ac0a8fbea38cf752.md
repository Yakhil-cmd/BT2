Looking at the external report's vulnerability class — **authorization bypass on a state-mutating function that anyone can call** — I need to find an analogous missing access control in the Starknet Staking codebase that allows an unprivileged caller to damage other users.

The closest analog is in `update_rewards` inside `IStakingRewardsManager`.

---

### Title
Unprivileged Caller Can Permanently Suppress Block Reward Distribution via `update_rewards(disable_rewards: true)` - (File: `src/staking/staking.cairo`)

### Summary

`update_rewards` in the `StakingRewardsManagerImpl` is callable by any address with no access control. It accepts a caller-controlled `disable_rewards: bool` parameter. Critically, the global `last_reward_block` storage variable is written **before** the `disable_rewards` guard is evaluated. An attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` every block to permanently prevent reward distribution for all stakers, causing permanent loss of unclaimed yield.

### Finding Description

The function `update_rewards` in `src/staking/staking.cairo` has no caller restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker validity checks ...

    // *** last_reward_block is committed HERE, before disable_rewards is checked ***
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;   // returns with NO rewards distributed
    }
    // ... actual reward distribution ...
}

Compare this with `update_rewards_from_attestation_contract`, which correctly gates itself:

```cairo
self.assert_caller_is_attestation_contract();
``` [2](#0-1) 

`general_prerequisites` only asserts the contract is not paused and the caller is non-zero — it provides no role or identity check: [3](#0-2) 

The `last_reward_block` field is a single global value shared across all stakers: [4](#0-3) 

### Impact Explanation

Once `last_reward_block` is set to block `N`, the assertion `current_block_number > self.last_reward_block.read()` will reject any further call to `update_rewards` in block `N`. Because the write happens before the early return, calling with `disable_rewards: true` permanently consumes the reward slot for that block without distributing anything. Rewards for block `N` are irrecoverably lost for every staker in the system.

An attacker who front-runs the legitimate reward update every block causes **permanent, continuous suppression of all staking yield** — matching the allowed impact: **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The function is fully public; any EOA or contract can call it.
- The only cost is gas.
- The attacker needs only to supply any currently-active staker address (readable from on-chain events or `get_stakers()`).
- Front-running is straightforward on Starknet (sequencer ordering) or the attacker can simply be the first caller each block.
- No special setup, no tokens, no privileged role required.

### Recommendation

Add an access-control guard to `update_rewards` so that only an authorized caller (e.g., the attestation contract, or a dedicated rewards-manager role) can invoke it with `disable_rewards: true`. At minimum, restrict the `disable_rewards: true` path to privileged callers while allowing public callers only with `disable_rewards: false`. Alternatively, move the `self.last_reward_block.write(current_block_number)` call to **after** the `disable_rewards` check so that a suppressed call does not consume the block's reward slot.

### Proof of Concept

1. Staker Alice is active with non-zero STRK balance.
2. At block `N`, before any legitimate reward update, attacker calls:
   ```
   staking.update_rewards(alice_address, disable_rewards: true)
   ```
3. `last_reward_block` is set to `N`; no rewards are distributed.
4. Any subsequent call to `update_rewards` in block `N` reverts with `REWARDS_ALREADY_UPDATED`.
5. Rewards for block `N` are permanently lost for all stakers.
6. Attacker repeats step 2 every block, permanently suppressing all yield.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
