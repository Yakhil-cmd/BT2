### Title
Unbounded Pool Member Balance Trace Causes Permanent Freezing of Unclaimed Yield - (File: `src/pool/pool.cairo`)

### Summary
The `calculate_rewards` function in `src/pool/pool.cairo` contains an explicitly unbounded loop that iterates over every entry in a pool member's balance trace. Because an unprivileged delegator can grow this trace without limit by repeatedly changing their delegated balance, they can permanently brick their own reward-claiming path — or be griefed into that state — causing permanent freezing of unclaimed yield.

### Finding Description
In `src/pool/pool.cairo`, the internal `calculate_rewards` function iterates over the entire `pool_member_epoch_balance` trace for a given pool member: [1](#0-0) 

The developers themselves annotate this loop with: *"This loop is unbounded but unlikely to exceed gas limits."* The trace grows by one entry each time the pool member changes their delegated balance (stake/unstake operations). There is no cap on the number of entries that can accumulate in `pool_member_epoch_balance` for a given address. [2](#0-1) 

The `calculate_rewards` function is called as part of the reward-claiming flow. Once the trace is large enough, any transaction that invokes `calculate_rewards` for that pool member will exceed the Starknet block gas limit and revert, permanently preventing the member from ever claiming their accrued rewards.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

A pool member whose balance trace has grown beyond the gas-safe threshold can never successfully call the reward-claiming entry point. Their accrued STRK rewards are permanently locked in the pool contract with no recovery path, since every claim attempt will revert due to out-of-gas.

### Likelihood Explanation
A delegator who frequently changes their stake (e.g., partial top-ups, partial exits, re-entries across many epochs) will naturally accumulate a large trace over time. The protocol has no mechanism to prune or checkpoint the trace, and no on-chain guard prevents the trace from growing. The risk increases linearly with protocol age and delegator activity.

### Recommendation
1. **Cap trace growth per pool member**: Enforce a maximum number of entries in `pool_member_epoch_balance` per address, similar to the `MAX_WHITELISTED_TOKENS` pattern recommended in the reference report.
2. **Checkpoint-and-prune**: After a successful `claim_rewards`, truncate all trace entries up to and including the claimed checkpoint, so the iterable window stays bounded.
3. **Paginated claiming**: Allow `calculate_rewards` to accept a `max_iterations` parameter and resume from a stored cursor, so reward claiming never requires a single unbounded transaction.

### Proof of Concept
1. Attacker (or any delegator) calls `enter_delegation_pool` to join a pool.
2. Across many epochs, the delegator repeatedly calls partial `exit_delegation_pool_intent` followed by `enter_delegation_pool` (or any balance-changing operation), adding one or more entries to `pool_member_epoch_balance` per epoch.
3. After accumulating sufficiently many entries (exact threshold depends on Starknet's per-transaction gas limit), the delegator attempts to claim rewards.
4. The `calculate_rewards` loop at `src/pool/pool.cairo:859` iterates over all accumulated entries. The transaction runs out of gas and reverts.
5. Every subsequent claim attempt also reverts. The delegator's unclaimed yield is permanently frozen.

The entry path is entirely unprivileged: `enter_delegation_pool` and `exit_delegation_pool_intent` are public functions callable by any address. [3](#0-2)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
