### Title
Missing Lower-Bound Threshold Validation in FROST Signing Allows Corrupted Signature Output — (`File: src/frost/mod.rs`)

### Summary
`assert_sign_inputs` and `presign` in `src/frost/mod.rs` validate that the threshold does not exceed the participant count, but never enforce the minimum threshold of 2. A caller can supply `threshold = 0` or `threshold = 1` to the FROST signing pipeline, causing Lagrange interpolation to run with a degree inconsistent with the DKG output and producing a cryptographically invalid signature that honest parties accept as a completed protocol result.

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces both an upper and a lower bound on the threshold: [1](#0-0) 

```rust
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
```

The analogous validation gate for FROST signing, `assert_sign_inputs`, only enforces the upper bound: [2](#0-1) 

```rust
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no lower-bound check; threshold = 0 or 1 passes silently
```

The same omission exists in the FROST `presign` entry point: [3](#0-2) 

```rust
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no lower-bound check
```

`ReconstructionLowerBound` is a plain `usize` newtype with no internal invariant, so any value including 0 can be constructed and passed: [4](#0-3) 

### Impact Explanation

When `threshold = 1` (or `0`) is supplied to the FROST signing path, the Lagrange interpolation inside `compute_signature_share` / the aggregation step runs with a degree-0 (or degenerate) polynomial basis. Because the DKG was performed with `threshold ≥ 2`, the Lagrange coefficients computed during signing are inconsistent with the secret-sharing polynomial used at key generation. The aggregated signature fails cryptographic verification against the group public key. Honest participants complete the protocol and receive what appears to be a finished signature, but the output is unusable.

**Mapped impact:** High — Corruption of sign outputs so honest parties accept unusable cryptographic outputs.

### Likelihood Explanation

The FROST signing API is a public library entry point. Any library caller — including a malicious coordinator orchestrating a signing session — can pass an arbitrary `ReconstructionLowerBound` value. No privilege or key material beyond participation in the signing session is required to trigger the bug. The DKG phase does not embed the threshold into the `KeygenOutput` struct in a way that is re-validated at signing time, so the mismatch is not caught downstream.

### Recommendation

Add the same lower-bound check that `assert_key_invariants` already performs to both `assert_sign_inputs` and `presign` in `src/frost/mod.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This mirrors the existing guard in `src/dkg.rs` at lines 580–582 and in `validate_triple_inputs` in `src/ecdsa/ot_based_ecdsa/triples/generation.rs` at lines 699–703. [5](#0-4) 

### Proof of Concept

```rust
use threshold_signatures::frost::{assert_sign_inputs, presign, PresignArguments};
use threshold_signatures::ReconstructionLowerBound;

// DKG was run with threshold = 2 (enforced).
// Attacker-controlled coordinator now calls signing with threshold = 1.
let bad_threshold: ReconstructionLowerBound = 1usize.into(); // passes the upper-bound check

// assert_sign_inputs returns Ok(participants) — no error
let participants = assert_sign_inputs(&participants, bad_threshold, me, coordinator).unwrap();

// presign also accepts threshold = 1 without error
let presign_args = PresignArguments { keygen_out, threshold: bad_threshold };
let protocol = presign(&participants_slice, me, &presign_args, rng).unwrap();

// Protocol runs to completion; Lagrange coefficients are computed for degree-0 basis,
// inconsistent with the degree-(threshold-1) = degree-1 polynomial from DKG.
// The resulting aggregated signature fails verification against the group public key.
```
