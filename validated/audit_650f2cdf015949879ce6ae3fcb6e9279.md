The key analog is in `AnchoredPriceProvider._getBidAndAskPrice()` at the synthetic ratio computation. Let me verify the exact behavior of `_computeBidAsk` when `mid = 0` is passed.

### Title
Synthetic Ratio Truncates to Zero in `AnchoredPriceProvider`, Permanently DoS-ing Pool Swaps — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider._getBidAndAskPrice()` computes a synthetic cross-pair mid price as `Math.mulDiv(mid, ORACLE_DECIMALS, mid2)` (i.e., `mid1 × 1e8 / mid2`). When the base-feed price is more than eight orders of magnitude smaller than the quote-feed price, integer division truncates the result to `0`. No post-ratio zero check exists. The zero `mid` propagates into `_computeBidAsk`, which returns the stalled sentinel `(0, type(uint128).max)`, causing `getBidAndAskPrice()` to revert with `FeedStalled`. Every swap through the affected pool is permanently blocked for as long as the price relationship holds.

---

### Finding Description

In the synthetic (two-feed) path of `_getBidAndAskPrice`:

```solidity
// AnchoredPriceProvider.sol line 263-269
if (_quote != bytes32(0)) {
    (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
    if (!ok2 || mid2 == 0) return (0, type(uint128).max);
    // Synthetic ratio (8-decimal): mid1 / mid2.
    mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // ← no post-result zero check
    spreadBps += spreadBps2;
}
return _computeBidAsk(mid, spreadBps);
``` [1](#0-0) 

`_readLeg` validates that each individual leg's `mid` is non-zero, but it cannot prevent the ratio from truncating to zero. After line 267, `mid` can be `0` with both legs individually valid.

`_computeBidAsk` then computes:

```solidity
uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
    return (0, type(uint128).max);
}
``` [2](#0-1) 

`_bandEdge(0, ...)` is `Math.mulDiv(0, Q64 * edgeFactor, STEP_DENOM)` = `0`, so `refBid == 0` is immediately true and the stalled sentinel is returned. [3](#0-2) 

`getBidAndAskPrice()` then reverts:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [4](#0-3) 

The same unguarded ratio computation is present in the in-repo test mock `FaithfulAnchoredPriceProvider` at line 114, confirming this is the intended production pattern, not a test-only artifact. [5](#0-4) 

---

### Impact Explanation

Every call to `pool.swap()` routes through `provider.getBidAndAskPrice()`. When that call reverts with `FeedStalled`, the pool's swap function propagates the revert (confirmed by the e2e test `test_e2e_staleFeedHaltsSwap`). [6](#0-5) 

The result is a complete, permanent DoS of all swaps through the affected pool for as long as the price relationship `mid1 × 1e8 < mid2` holds. This is broken core pool functionality per the allowed impact gate.

---

### Likelihood Explanation

The condition `Math.mulDiv(mid1, 1e8, mid2) == 0` requires `mid1 < mid2 / 1e8`. In 8-decimal oracle terms this means the base token's USD price is less than `(quote token USD price) × 1e-8`. Concrete examples:

| Base | Quote | mid1 (8-dec) | mid2 (8-dec) | Ratio |
|---|---|---|---|---|
| SHIB ($8e-9) | BTC ($60k) | 800 | 6e12 | `800 × 1e8 / 6e12 = 0` |
| any token < $0.0001 | BTC | < 1e4 | 6e12 | 0 |

The `AnchoredPriceProvider` is documented as "the one standard provider for public pools" and synthetic ratio mode is an explicit production feature (e.g., BTC/ETH = BTC/USD ÷ ETH/USD). Any pool pairing a low-value token against a high-value denominator feed hits this condition. No privileged action is required — the trigger is purely market price data. [7](#0-6) 

---

### Recommendation

Add a zero check on the computed ratio immediately after line 267:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
if (mid == 0) return (0, type(uint128).max);   // ratio underflows 8-decimal precision
```

Alternatively, scale the numerator before dividing (e.g., multiply by `1e18` instead of `1e8` and adjust downstream consumers) to preserve sub-unit precision for cheap-base / expensive-quote pairs.

### Proof of Concept

Using the existing `AnchoredPriceProviderSyntheticTest` harness:

```solidity

    // SHIB = $0.000000008 → mid1 = 800 (8-decimal)

    // Ratio = 800 * 1e8 / 6_000_000_000_000 = 0 → FeedStalled
    oracle.setData(FEED1, uint64(800),              2, 0, block.timestamp); // SHIB/USD
    oracle.setData(FEED2, uint64(6_000_000_000_000), 2, 0, block.timestamp); // BTC/USD

    // Both legs individually pass _readLeg (non-zero, fresh, within guard).
    // The ratio truncates to 0 → _computeBidAsk returns stalled sentinel.
    _expectStalled(); // reverts FeedStalled → pool swap is permanently DoS'd

```

The `_readLeg` guard at line 287 passes for both legs (each `mid > 0`, each `spreadBps < ORACLE_BPS`), but the ratio at line 267 produces `0`, which `_computeBidAsk` catches at the `refBid == 0` guard and returns the stalled sentinel, causing `getBidAndAskPrice` to revert. [8](#0-7)
