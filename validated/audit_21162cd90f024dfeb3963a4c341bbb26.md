### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` in `staking.cairo` has no caller restriction. Any unprivileged address can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` sentinel to the current block without distributing any rewards. Because the function enforces `current_block_number > last_reward_block`, every subsequent legitimate call in the same block is rejected. An attacker who front-runs this call every block permanently destroys all consensus-era block rewards for every staker.

---

### Finding Description

`update_rewards` is the sole external entry point for distributing per-block consensus rewards to stakers. Its only gate is `general_prerequisites()`, which checks the pause flag and that the caller is non-zero: [1](#0-0) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
```

There is no `assert_caller_is_attestation_contract()` or equivalent role check. Compare this with every pool-facing function in the same contract, which all enforce `CALLER_IS_NOT_POOL_CONTRACT`: [2](#0-1) 

And every staking-contract-facing function in the pool, which all enforce `assert_caller_is_staking_contract()`: [3](#0-2) 

The `disable_rewards` flag is the critical lever. When `true`, the function writes `last_reward_block` and returns immediately without distributing anything: [4](#0-3) 

```cairo
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

`last_reward_block` is a **global** storage slot, not per-staker. Once it equals the current block number, the `REWARDS_ALREADY_UPDATED` assertion blocks every other call to `update_rewards` for that block, for every staker.

---

### Impact Explanation

An attacker who calls `update_rewards(any_staker_address, disable_rewards: true)` at the start of every block:

1. Advances `last_reward_block` to the current block.
2. Causes the function to return before any reward calculation or distribution.
3. Blocks all legitimate calls to `update_rewards` for that block (they revert with `REWARDS_ALREADY_UPDATED`).
4. The skipped block's rewards are **permanently lost** — there is no mechanism to retroactively distribute rewards for a past block.

Repeated across every block, this permanently freezes all consensus-era unclaimed yield for all stakers and their delegators. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is public and callable by any non-zero address.
- The attacker needs only to submit a transaction per block — no capital, no privileged key, no special state.
- On Starknet, transaction fees are low, making sustained griefing economically viable.
- The attack is fully permissionless and requires no coordination or prior setup.

---

### Recommendation

Add a caller restriction to `update_rewards` so it can only be invoked by the designated consensus/sequencer contract (or the attestation contract). For example, add an `assert_caller_is_rewards_manager()` check analogous to the existing `assert_caller_is_attestation_contract()` used in `update_rewards_from_attestation_contract`: [5](#0-4) 

Alternatively, if `update_rewards` is intended to be callable by anyone (e.g., as a permissionless keeper), the `disable_rewards` parameter must be removed or restricted to a privileged caller, since it is the mechanism that allows rewards to be silently suppressed.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has been set and passed).
2. Attacker monitors the mempool/block production.
3. At the start of block `N`, attacker calls:
   ```
   staking.update_rewards(any_registered_staker, disable_rewards: true)
   ```
4. `last_reward_block` is set to `N`; no rewards are distributed.
5. Any legitimate call to `update_rewards` for block `N` reverts with `REWARDS_ALREADY_UPDATED`.
6. Block `N`'s rewards are permanently lost.
7. Attacker repeats for block `N+1`, `N+2`, … indefinitely.

The root cause is the absence of any access control on `update_rewards`, directly analogous to M-06's missing `onlyRegistry` modifier: a function intended to be called only from a specific privileged path is instead callable by any address, allowing an unprivileged actor to corrupt protocol state. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
