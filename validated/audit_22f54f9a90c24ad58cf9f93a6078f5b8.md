### Title
`protocolUnpausePool` Unconditionally Transitions to Admin-Paused Level 1, Permanently Blocking Swaps When Pool Admin Is Unavailable — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

The `protocolUnpausePool` function always transitions the pool to pause level `1` (admin-paused), regardless of the pool's state before the protocol pause. When the protocol owner protocol-pauses a pool that was originally active (level `0`) and then protocol-unpauses it, the pool is left at level `1` — a state the pool admin never requested. There is no direct path from level `2` to level `0`; the only route is `2 → 1 → 0`, and the final step requires the pool admin to cooperate. If the pool admin is unavailable, the pool is permanently stuck at level `1`, blocking all swaps indefinitely.

---

### Finding Description

The pause mechanism uses a `uint8 pauseLevel` with three values: `0` (active), `1` (admin-paused), `2` (protocol-paused). The valid factory-enforced transitions are:

| Caller | From | To | Function |
|---|---|---|---|
| Pool admin | `0` | `1` | `pausePool` |
| Pool admin | `1` | `0` | `unpausePool` |
| Factory owner | `0` or `1` | `2` | `protocolPausePool` |
| Factory owner | `2` | `1` | `protocolUnpausePool` |

There is **no** `2 → 0` transition. The `protocolUnpausePool` function unconditionally sets the level to `1`:

```solidity
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1);   // always → 1, never → 0
}
``` [1](#0-0) 

The `unpausePool` path (the only way to reach level `0`) is exclusively gated by `onlyPoolAdmin`:

```solidity
function unpausePool(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 1) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
}
``` [2](#0-1) 

The pool-level check that blocks swaps is:

```solidity
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
``` [3](#0-2) 

`swap` carries `whenNotPaused`, while `addLiquidity` and `removeLiquidity` do not: [4](#0-3) [5](#0-4) 

This is the direct analog to the Drift enum-status bug: the pause state is a single scalar value, not a bitmask. A protocol-level "unpause" action unconditionally writes level `1`, overwriting the prior state (`0`) and leaving the pool in an admin-paused condition that the admin never set.

---

### Impact Explanation

- **Swaps are permanently blocked** (`PoolPaused` revert on every `swap` call) until the pool admin explicitly calls `unpausePool`.
- `addLiquidity` and `removeLiquidity` are **not** gated by pause, so LP principal is not trapped — but the pool's core trading function is entirely unusable.
- The factory owner has **no unilateral path** to restore the pool to active. The only route is `2 → 1` (owner) then `1 → 0` (pool admin). If the pool admin key is lost, the multisig is bricked, or the admin is simply unresponsive, the pool is permanently stuck at level `1` with swaps disabled.

---

### Likelihood Explanation

The trigger sequence is:
1. Pool is active (level `0`).
2. Factory owner calls `protocolPausePool` (emergency or routine) → level `2`.
3. Factory owner calls `protocolUnpausePool` → level `1` (not `0`).
4. Pool admin is unavailable (lost key, dead multisig, governance delay, etc.).

This is a realistic operational scenario. Protocol-level emergency pauses are exactly the situations where pool admins may be unreachable or where the protocol owner expects to be able to restore service unilaterally. The design forces a dependency on the pool admin for the final step, which is undocumented at the code level and creates a latent DoS path.

---

### Recommendation

**Short term:** Store the pre-protocol-pause level in factory state and restore it on `protocolUnpausePool`:

```solidity
mapping(address => uint8) public prePausePauseLevel; // 0 or 1

function protocolPausePool(address pool) external onlyOwner nonReentrant {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    prePausePauseLevel[pool] = cur;          // remember original level
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}

function protocolUnpausePool(address pool) external onlyOwner nonReentrant {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 0);
    uint8 restore = prePausePauseLevel[pool]; // 0 or 1
    delete prePausePauseLevel[pool];
    IMetricOmmPoolFactoryActions(pool).setPause(restore);
}
```

**Long term:** Periodically audit the pause transition matrix to ensure the factory owner can always restore a pool to its pre-pause state without requiring pool-admin cooperation.

### Proof of Concept

```solidity
// 1. Pool starts active (level 0)
assertEq(pauseLevel(pool), 0);

// 2. Factory owner protocol-pauses
factory.protocolPausePool(pool);
assertEq(pauseLevel(pool), 2);

// 3. Factory owner protocol-unpauses
factory.protocolUnpausePool(pool);
assertEq(pauseLevel(pool), 1);   // ← stuck at admin-paused; admin never requested this

// 4. Swap attempt reverts with PoolPaused even though the protocol "unpaused"
vm.expectRevert(IMetricOmmPoolActions.PoolPaused.selector);
pool.swap(recipient, true, 1e18, 0, "", "");

// 5. Factory owner cannot fix it — protocolUnpausePool reverts (cur != 2)
vm.expectRevert();
factory.protocolUnpausePool(pool);

// 6. Only the pool admin can resolve it
vm.prank(poolAdmin);
factory.unpausePool(pool);
assertEq(pauseLevel(pool), 0);   // finally active — but only with admin cooperation
```
