### Title
`DelegateAction` Expiry Off-by-One Allows Execution at `max_block_height` â€” (`runtime/runtime/src/actions.rs`)

### Summary

`apply_delegate_action` uses a strict-greater-than (`>`) comparison to check whether a `DelegateAction` has expired, but the field's documented invariant requires expiry at `block_height >= max_block_height`. The result is that any `DelegateAction` is executable for one extra block beyond the authorization window the user signed.

### Finding Description

`DelegateAction.max_block_height` is documented as:

> "The maximal height of the block in the blockchain **below which** the given DelegateAction is valid."

The runtime spec (`docs/RuntimeSpec/Actions.md`) states the expiry condition as:

> "If the current block is **equal or greater than** `max_block_height`" â†’ `DelegateActionExpired`

The production check in `apply_delegate_action` is:

```rust
if apply_state.block_height > delegate_action.max_block_height() {
    result.result = Err(ActionErrorKind::DelegateActionExpired.into());
    return Ok(());
}
``` [1](#0-0) 

The operator is `>` (strictly greater than), not `>=`. When `apply_state.block_height == max_block_height`, the condition is `false` and the action proceeds. The action should be expired at that exact height per the documented invariant.

The existing unit test `test_delegate_action_max_height` only verifies the `max_block_height + 1` case:

```rust
// Setup current block as higher than max_block_height. Must fail.
let apply_state = create_apply_state(signed_delegate_action.delegate_action.max_block_height + 1);
``` [2](#0-1) 

The boundary case `block_height == max_block_height` is never tested for expiry; instead, `test_delegate_action` and `test_delegate_action_validate_sender_account` both call `create_apply_state(max_block_height)` and expect the action to succeed past the expiry check. [3](#0-2) [4](#0-3) 

The `DelegateAction` struct definition and the field's docstring confirm the intended semantics: [5](#0-4) 

### Impact Explanation

A user signs a `DelegateAction` with `max_block_height = N` to authorize a relayer to execute their meta-transaction only for blocks with height `< N`. Due to the off-by-one, the relayer can hold the signed action and submit it at exactly block `N`, executing it one block after the user's intended authorization window closes. The user's signed authorization is valid for one more block than they intended, violating the authorization boundary they explicitly set.

The corrupted value is the authorization decision: the action is admitted (`Ok`) at `block_height == max_block_height` when it must be rejected (`DelegateActionExpired`).

### Likelihood Explanation

Any relayer processing a `DelegateAction` can trivially exploit this by timing submission to the exact `max_block_height` block. No special privilege is required â€” the relayer is an unprivileged participant who receives the signed `DelegateAction` from the user. The user-controlled field `max_block_height` is the attacker-controlled input.

### Recommendation

Change the comparison from strict-greater-than to greater-than-or-equal:

```rust
// Before (off-by-one):
if apply_state.block_height > delegate_action.max_block_height() {

// After (correct):
if apply_state.block_height >= delegate_action.max_block_height() {
``` [6](#0-5) 

Add a test asserting that `block_height == max_block_height` produces `DelegateActionExpired`, mirroring the existing `test_delegate_action_max_height` test which only covers `max_block_height + 1`.

### Proof of Concept

```rust
// Existing test already demonstrates the off-by-one implicitly:
// test_delegate_action uses block_height == max_block_height and expects Ok.
// The following test would expose the bug:
#[test]
fn test_delegate_action_expires_at_max_block_height() {
    let mut result = ActionResult::default();
    let (action_receipt, signed_delegate_action) = create_delegate_action_receipt();
    let sender_id = signed_delegate_action.delegate_action.sender_id.clone();
    let sender_pub_key = signed_delegate_action.delegate_action.public_key.clone();
    let access_key = AccessKey { nonce: 19000000, permission: AccessKeyPermission::FullAccess };

    // block_height == max_block_height: per docs, must expire.
    let apply_state =
        create_apply_state(signed_delegate_action.delegate_action.max_block_height);
    let mut state_update = setup_account(&sender_id, &sender_pub_key, &access_key);

    apply_delegate_action(
        &mut state_update,
        &apply_state,
        &VersionedActionReceipt::from(action_receipt),
        &sender_id,
        (&signed_delegate_action).into(),
        &mut result,
    ).expect("Expect ok");

    // This assertion FAILS with the current code (bug):
    assert_eq!(result.result, Err(ActionErrorKind::DelegateActionExpired.into()));
}
```
