### Title
Missing Minimum Threshold Lower-Bound Check in `assert_sign_inputs` Allows Single-Party Signature Bypass — (File: `src/frost/mod.rs`)

---

### Summary

`assert_sign_inputs`, the shared validation gate for all EdDSA (`sign_v1`, `sign_v2`) and RedJubjub (`sign`) signing entry-points, enforces an upper bound on `threshold` but omits the lower-bound check (`threshold >= 2`) that is present in the DKG entry-point `assert_key_invariants`. A caller who supplies `threshold = 1` passes all validation, causing FROST's `KeyPackage` to be constructed with `min_signers = 1`, after which the coordinator can aggregate a cryptographically valid signature from a single share — bypassing the quorum entirely.

---

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces both bounds:

```rust
// src/dkg.rs  line 573-581
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

`assert_sign_inputs` in `src/frost/mod.rs`, shared by all FROST signing functions, only enforces the upper bound:

```rust
// src/frost/mod.rs  lines 144-150
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no check for threshold < 2
``` [2](#0-1) 

All three public signing entry-points delegate to this function:

- `sign_v1` → `assert_sign_inputs` [3](#0-2) 
- `sign_v2` → `assert_sign_inputs` [4](#0-3) 
- `sign` (RedJubjub) → `assert_sign_inputs` [5](#0-4) 

When `threshold = 1` is accepted, `construct_key_package` converts it directly to `u16` without any floor check:

```rust
// src/frost/eddsa/sign.rs  line 365
u16::try_from(threshold.value()).map_err(|_| { ... })?
``` [6](#0-5) 

The same pattern exists in the RedJubjub variant: [7](#0-6) 

A `KeyPackage` with `min_signers = 1` is then handed to `frost_core::round2::sign` and `aggregate`. FROST's aggregation succeeds with a single share, producing a signature that verifies against the group public key.

---

### Impact Explanation

A malicious coordinator calls `sign_v1`, `sign_v2`, or `sign` (RedJubjub) with `threshold = 1`. Validation passes. The coordinator constructs its own `KeyPackage` with `min_signers = 1`, computes its own signature share, and calls `aggregate` with only that share. The resulting signature is a fully valid ECDSA/EdDSA/RedJubjub signature over the group public key — produced unilaterally by one party, without the quorum that the DKG was configured to require.

**Impact class:** Critical — Unauthorized creation of a valid threshold signature for attacker-chosen inputs.

---

### Likelihood Explanation

The coordinator role is a documented, reachable caller of these functions. No key compromise or cryptographic break is required. The only precondition is that the coordinator supplies `threshold = 1` instead of the correct value. Because `ReconstructionLowerBound` is a plain `usize` newtype with no enforced invariant, any caller can construct it freely: [8](#0-7) 

There is no runtime check anywhere between the caller and the FROST signing core that would catch this.

---

### Recommendation

Add the same lower-bound guard that exists in `assert_key_invariants` to `assert_sign_inputs`:

```rust
// src/frost/mod.rs — inside assert_sign_inputs
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Apply the same fix to the `presign` function in `src/frost/mod.rs` (line 72), which has the identical omission. [9](#0-8) 

---

### Proof of Concept

1. Run DKG with 3 participants and `threshold = 2` (valid, enforced by `assert_key_invariants`).
2. Call `sign_v1(&participants, /*threshold=*/1usize, me, coordinator, keygen_output, message, rng)`.
3. `assert_sign_inputs` returns `Ok` — `1 <= 3` passes the only check.
4. `construct_key_package` builds a `KeyPackage` with `min_signers = 1`.
5. The coordinator computes its own signature share and calls `aggregate` with only that share.
6. `aggregate` returns a valid signature that passes `public_key.verify(message, &signature)` — produced by one party, bypassing the 2-of-3 quorum.
