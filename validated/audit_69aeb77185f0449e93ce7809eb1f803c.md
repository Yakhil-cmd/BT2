### Title
Unvalidated Identity `app_pk` in CKD Protocol Discloses Confidential Derived Key to Coordinator — (`src/confidential_key_derivation/protocol.rs`)

---

### Summary

The `ckd()` entry point in the Confidential Key Derivation protocol accepts an `app_pk` (application ElGamal public key) without checking whether it is the group identity element. If `app_pk` is the identity, the ElGamal masking term vanishes and the coordinator directly reconstructs the confidential derived key `msk · H(pk ‖ app_id)` — the exact secret CKD is designed to keep hidden from MPC nodes.

---

### Finding Description

`ckd()` in `src/confidential_key_derivation/protocol.rs` validates participant counts and membership but performs no check on `app_pk`: [1](#0-0) 

The value flows unchanged into `compute_signature_share`, where it is used as the ElGamal public key: [2](#0-1) 

The critical line is:

```rust
let big_c = big_s + app_pk * y.0;   // line 174
```

When `app_pk` is the group identity `G1::identity()`, the term `app_pk * y.0` evaluates to the identity, so `big_c = big_s = x_i · H(pk ‖ app_id)`. Each participant's normalized share becomes `norm_big_c = λ_i · x_i · H(pk ‖ app_id)`. The coordinator sums all shares:

```
Σ λ_i · x_i · H(pk ‖ app_id)  =  msk · H(pk ‖ app_id)
```

which is the confidential derived key — received in the clear.

The `CKDOutput` type and its `unmask` method are defined in `src/confidential_key_derivation/mod.rs`: [3](#0-2) 

With `app_pk = identity`, `app_sk = 0` is the matching secret key, so `unmask(0)` still returns the correct key to the application, meaning the protocol completes without error while the coordinator has already learned the secret.

---

### Impact Explanation

The entire security guarantee of CKD is that MPC nodes never learn the derived secret. With `app_pk = identity`, the coordinator (an MPC node) directly reconstructs `msk · H(pk ‖ app_id)` — a **confidential derived secret** — from the protocol messages. This matches the Critical allowed impact: *"Extraction, reconstruction, or disclosure of … confidential derived secrets."*

---

### Likelihood Explanation

The `app_pk` parameter is caller-supplied with no library-side validation. A malicious application, a misconfigured client, or a default-initialized `ElementG1` value (whose zero/identity is the natural Rust default for many group types) can trigger this path. No privileged access is required; any caller of the public `ckd()` API suffices.

---

### Recommendation

Add an explicit identity check on `app_pk` at the start of `ckd()`, analogous to the existing `msg_hash == 0` guard in the robust ECDSA `sign()` function: [4](#0-3) 

For CKD, the equivalent guard should be placed in `ckd()` after participant validation:

```rust
use elliptic_curve::Group;
if app_pk == ElementG1::identity() {
    return Err(InitializationError::BadParameters(
        "app_pk must not be the identity element".to_string(),
    ));
}
```

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))

pub struct CKDOutput {
    big_y: ElementG1,
    big_c: ElementG1,

impl CKDOutput {
    pub fn new(big_y: ElementG1, big_c: ElementG1) -> Self {
        Self { big_y, big_c }
    }

    /// Outputs `big_y`
    pub fn big_y(&self) -> ElementG1 {
        self.big_y
    }

    /// Outputs `big_c`
    pub fn big_c(&self) -> ElementG1 {
        self.big_c
    }

    /// Takes a secret scalar and returns
    /// s <- C − a ⋅ Y = msk ⋅ H ( `app_id` )
    pub fn unmask(&self, secret_scalar: Scalar) -> Signature {
        self.big_c - self.big_y * secret_scalar
    }
}
