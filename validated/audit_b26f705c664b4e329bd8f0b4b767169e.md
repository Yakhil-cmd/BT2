### Title
Missing Caller Validation on `update_rewards` Allows Any Address to Permanently Suppress Consensus Reward Distribution - (File: `src/staking/staking.cairo`)

### Summary

The `update_rewards` function in the Staking contract accepts an attacker-controlled `disable_rewards: bool` parameter with no caller validation. The protocol specification explicitly restricts this function to "Only starkware sequencer," but the implementation enforces no such check. Any unprivileged address can call `update_rewards(..., disable_rewards: true)` each block, consuming the single per-block reward slot while distributing zero rewards, permanently freezing all staker and delegator unclaimed yield.

### Finding Description

The `IStakingRewardsManager::update_rewards` function is the sole mechanism for distributing per-block consensus rewards to stakers and their delegation pools. The function enforces a global one-call-per-block invariant via `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);  // slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;  // rewards skipped, slot already consumed
    }
    // ... distribute rewards
``` [1](#0-0) 

The only prerequisite check is `self.general_prerequisites()`, which only verifies the contract is not paused. There is no assertion that `get_caller_address()` equals the authorized sequencer address. The `disable_rewards` boolean is accepted verbatim from any caller.

The protocol specification explicitly states the intended access control:

> **access control**: Only starkware sequencer. [2](#0-1) 

Once `last_reward_block` is written to `current_block_number`, the `REWARDS_ALREADY_UPDATED` assertion blocks every subsequent call in the same block: [3](#0-2) 

### Impact Explanation

An attacker calling `update_rewards(valid_staker, disable_rewards: true)` once per block:

1. Writes `last_reward_block = current_block_number`, consuming the block's reward slot.
2. Returns immediately without distributing any STRK or BTC rewards.
3. Blocks the legitimate sequencer call for that block with `REWARDS_ALREADY_UPDATED`.

Repeated across every block, this permanently freezes all unclaimed consensus yield for every staker and every delegation pool member. The `unclaimed_rewards_own` field of stakers and the `cumulative_rewards_trace` of pools are never updated, so `claim_rewards` on both the staking contract and pool contract will always return zero.

This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation

The attack requires no stake, no privileged role, and no special setup. Any EOA or contract can call `update_rewards` with `disable_rewards: true` on any valid, active staker address. The cost is one transaction per block. The attack is trivially automatable and economically rational for any party wishing to grief the protocol or its stakers.

### Recommendation

Add a caller check at the top of `update_rewards` to enforce that only the authorized Starkware sequencer address may invoke it. The sequencer address should be stored in contract storage (similar to how `attestation_contract` is stored) and validated:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
}
```

Alternatively, if the sequencer address is not known at deploy time, restrict `disable_rewards: true` to a privileged role (e.g., `security_agent`) while allowing `disable_rewards: false` only from the sequencer.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. A valid staker `S` exists with non-zero balance effective in the current epoch.
3. Attacker `A` (any address) calls `update_rewards(staker_address: S, disable_rewards: true)` at block `N`.
4. `last_reward_block` is set to `N`; no rewards are distributed.
5. The legitimate sequencer attempts `update_rewards(staker_address: S, disable_rewards: false)` at block `N` — it reverts with `REWARDS_ALREADY_UPDATED`.
6. Attacker repeats step 3 at every subsequent block.
7. `S.unclaimed_rewards_own` remains zero indefinitely; `S.claim_rewards()` always returns zero.
8. All delegation pool members of `S` also receive zero rewards, as `update_rewards_from_staking_contract` on the pool is never called. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
