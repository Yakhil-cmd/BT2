### Title
`last_reward_block` Updated Before `disable_rewards` Guard Allows Any Caller to Permanently Suppress Block Rewards - (File: src/staking/staking.cairo)

### Summary
In `StakingRewardsManagerImpl::update_rewards`, the global `last_reward_block` storage variable is written **before** the `disable_rewards || self.is_pre_consensus()` early-return guard is evaluated. Because `update_rewards` has no access control beyond a non-zero caller check, any unprivileged address can call `update_rewards(any_staker, disable_rewards: true)`, atomically consuming the current block number and returning without distributing any rewards. Subsequent legitimate calls for the same block revert with `REWARDS_ALREADY_UPDATED`. The attacker can repeat this every block, permanently freezing all consensus-era staker and pool rewards.

### Finding Description
`update_rewards` is a public function gated only by `general_prerequisites()`, which checks the contract is unpaused and the caller is non-zero. [1](#0-0) 

Inside `update_rewards`, the sequence is:

1. Assert `current_block_number > last_reward_block` (line 1454–1458).
2. Validate staker existence and activity (lines 1466–1482).
3. **Write `last_reward_block = current_block_number`** (line 1485).
4. **Then** check `if disable_rewards || self.is_pre_consensus() { return; }` (lines 1487–1489). [2](#0-1) 

Because step 3 commits the block number to storage before step 4 evaluates whether rewards should actually be distributed, passing `disable_rewards: true` permanently marks the block as processed while skipping all reward computation and transfer. No subsequent call can re-process the same block.

`last_reward_block` is a single global (not per-staker) storage slot: [3](#0-2) 

Therefore one call with any valid `staker_address` and `disable_rewards: true` silences rewards for **all** stakers for that block.

### Impact Explanation
Every block in the consensus-rewards era produces STRK (and BTC) block rewards for every active staker and their delegation pools. By front-running or simply being the first caller each block, an attacker can permanently zero out those rewards. Stakers and pool members accumulate no `unclaimed_rewards_own` and pools receive no `update_rewards_from_staking_contract` call. The loss is irreversible because the block number can never be re-used. This constitutes **permanent freezing of unclaimed yield** (High impact per the allowed scope).

### Likelihood Explanation
`update_rewards` is permissionless. The attacker needs only a non-zero address and enough gas to call it once per block. There is no economic barrier; the attacker spends a small gas fee while denying rewards to the entire staker set. The attack is trivially automatable and can be sustained indefinitely.

### Recommendation
Move the `last_reward_block` write to **after** the `disable_rewards` guard, so the block slot is only consumed when rewards are actually going to be distributed:

```cairo
// Check early-exit conditions BEFORE consuming the block slot.
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Only now mark the block as processed.
self.last_reward_block.write(current_block_number);

// ... proceed with reward calculation
```

Alternatively, restrict `update_rewards` to a trusted caller (e.g., the block proposer's operational address or a dedicated role) so that `disable_rewards: true` cannot be injected by an arbitrary party.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. At block `N`, attacker calls:
   ```
   staking.update_rewards(any_active_staker_address, disable_rewards: true)
   ```
3. Inside `update_rewards`:
   - `current_block_number (N) > last_reward_block` → passes.
   - Staker validation passes.
   - `last_reward_block` is written to `N`. ← **block slot consumed**
   - `disable_rewards == true` → early return, zero rewards distributed.
4. Legitimate block-proposer calls `update_rewards(staker, disable_rewards: false)` in the same block `N`:
   - `current_block_number (N) > last_reward_block (N)` → **false → reverts with `REWARDS_ALREADY_UPDATED`**.
5. All stakers and pools receive zero rewards for block `N`.
6. Attacker repeats at block `N+1`, `N+2`, … indefinitely. [4](#0-3)

```text
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
```
