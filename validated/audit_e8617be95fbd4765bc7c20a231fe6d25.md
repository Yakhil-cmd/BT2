### Title
Unprivileged IBC transfer permanently blocks admin `SetExternalContractForDenom` via auto-deployment griefing - (File: x/cronos/keeper/keeper.go)

### Summary
An unprivileged attacker can permanently prevent the Cronos admin (or governance) from registering a custom external CRC21 contract for any IBC denom. By sending a dust IBC transfer of the target denom to Cronos before the admin's `MsgUpdateTokenMapping` is processed, the attacker triggers auto-deployment of a standard ModuleCRC21 contract. The `ensureDenomNotMapped` guardrail then permanently blocks every subsequent `SetExternalContractForDenom` call for that denom. No keeper function exists to remove an auto-deployed contract mapping, so the lock-out is irreversible.

### Finding Description

**Auto-deployment path (unprivileged trigger)**

When an IBC packet carrying a new token denom arrives on Cronos and `EnableAutoDeployment` is `true`, the IBC middleware calls `OnRecvVouchers`, which calls `ConvertVouchersToEvmCoins`, which calls `ConvertCoinFromNativeToCRC21` with `autoDeploy=true`:

```go
// evm.go:98-110
contract, found := k.GetContractByDenom(ctx, coin.Denom)
if !found {
    if !autoDeploy {
        return fmt.Errorf("no contract found for the denom %s", coin.Denom)
    }
    contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
    ...
    if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
        return err
    }
}
```

`SetAutoContractForDenom` writes the auto-deployed contract into the KV store under `DenomToAutoContractKey`.

**The blocking guardrail**

`SetExternalContractForDenom` — called by every admin/governance token-mapping path — unconditionally calls `ensureDenomNotMapped`:

```go
// keeper.go:204-216
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
    if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
        return err
    }
    ...
}
