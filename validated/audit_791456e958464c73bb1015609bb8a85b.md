### Title
`unstake_action()` Reverts When Reward Supplier Lacks Actual Token Balance, Temporarily Freezing Staker Principal — (File: src/staking/staking.cairo)

---

### Summary

`unstake_action()` in `Staking` unconditionally calls `send_rewards_to_staker()` before returning the staker's principal. `send_rewards_to_staker()` calls `claim_from_reward_supplier()` → `RewardSupplier.claim_rewards()` → `checked_transfer()`. If the reward supplier's actual STRK token balance is less than the owed reward amount (because L1 bridge funds have not yet arrived), the `checked_transfer` reverts, causing the entire `unstake_action()` to revert. The staker's principal — already held in the staking contract — cannot be withdrawn until the reward supplier is sufficiently funded.

---

### Finding Description

The call chain is:

```
unstake_action()                          [staking.cairo:483]
  └─ send_rewards_to_staker()             [staking.cairo:1614]
       └─ claim_from_reward_supplier()    [staking/utils]
            └─ RewardSupplier.claim_rewards(amount)  [reward_supplier.cairo:205]
                 ├─ assert!(amount <= unclaimed_rewards)   ← passes (accounting is correct)
                 └─ token_dispatcher.checked_transfer(...)  ← REVERTS if actual balance < amount
```

`RewardSupplier` tracks owed rewards in the storage variable `unclaimed_rewards` and proactively sends L1 mint requests via `request_funds()`. However, L1→L2 message delivery is asynchronous. Between the moment rewards are credited to `unclaimed_rewards` and the moment the bridged tokens actually arrive via `on_receive()`, the contract's real STRK balance can be lower than `unclaimed_rewards`.

When `unstake_action()` is called during this window, the accounting assertion at line 214 passes:

```cairo
assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);
``` [1](#0-0) 

But the subsequent `checked_transfer` reverts with `INSUFFICIENT_BALANCE` because the actual token balance is lower than `amount`:

```cairo
token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
``` [2](#0-1) 

This revert propagates through `send_rewards_to_staker()`:

```cairo
claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
``` [3](#0-2) 

And through `unstake_action()`:

```cairo
self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
``` [4](#0-3) 

The staker's principal — already held in the staking contract — cannot be returned until the reward supplier is funded. The staker has no way to skip reward payout and retrieve only their principal.

---

### Impact Explanation

A staker who has completed the exit window (`unstake_time` has passed) is unable to execute `unstake_action()` and recover their staked STRK. Their principal is temporarily frozen inside the staking contract. This matches the allowed impact: **Temporary freezing of funds**.

---

### Likelihood Explanation

L1→L2 message delivery on Starknet has inherent latency (minutes to hours). The reward supplier requests funds from L1 only when `credit < debit + threshold` in `request_funds()`:

```cairo
if credit < debit + threshold {
    ...
    self.send_mint_request_to_l1_reward_supplier();
``` [5](#0-4) 

During the bridging window, any staker with non-zero `unclaimed_rewards_own` who calls `unstake_action()` will be blocked. This is a realistic scenario during normal protocol operation, especially after epochs with large reward accruals or when many stakers exit simultaneously.

---

### Recommendation

Decouple reward payout from principal withdrawal in `unstake_action()`. One approach: if `claim_from_reward_supplier` fails (or if the reward supplier balance is insufficient), allow `unstake_action()` to proceed with returning the principal and leave `unclaimed_rewards_own` intact for a separate `claim_rewards()` call. Alternatively, add a check before calling `send_rewards_to_staker()` to skip the reward transfer if the reward supplier cannot cover it, deferring it to a later `claim_rewards()` call.

---

### Proof of Concept

1. Staker stakes and accrues rewards over several epochs (`unclaimed_rewards_own > 0`).
2. Staker calls `unstake_intent()` and waits for the exit window to expire.
3. During the exit window, the reward supplier's `unclaimed_rewards` is updated (accounting is correct) but the L1 bridge has not yet delivered the corresponding STRK tokens (actual balance < `unclaimed_rewards`).
4. Staker calls `unstake_action()`.
5. `send_rewards_to_staker()` → `claim_from_reward_supplier()` → `RewardSupplier.claim_rewards(amount)` → `checked_transfer()` reverts with `INSUFFICIENT_BALANCE`.
6. `unstake_action()` reverts. The staker's principal remains locked in the staking contract.
7. The staker must wait until `on_receive()` is called by StarkGate to fund the reward supplier before `unstake_action()` can succeed. [6](#0-5) [7](#0-6) [8](#0-7)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn send_rewards_to_staker(
            ref self: ContractState,
            staker_address: ContractAddress,
            ref staker_info: InternalStakerInfoLatest,
            token_dispatcher: IERC20Dispatcher,
        ) {
            let reward_address = staker_info.reward_address;
            let amount = staker_info.unclaimed_rewards_own;
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();

            claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
            token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
            staker_info.unclaimed_rewards_own = Zero::zero();

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
