### Title
`EvmErc20` Burns User Tokens Before Precompile Call Without Checking Result, Causing Permanent Fund Loss When Exit Precompile Is Paused — (File: `etc/eth-contracts/contracts/EvmErc20.sol`)

---

### Summary

`EvmErc20.sol` and `EvmErc20V2.sol` burn the caller's ERC-20 tokens **before** invoking the `ExitToNear` or `ExitToEthereum` precompile, and do **not** check the return value of that precompile call. When the exit precompile is paused via `pause_precompiles` as an emergency measure, the burn still executes and is never reverted, permanently destroying user funds with no refund path. This mirrors the external report's pattern exactly: a privileged burn operation proceeds despite the corresponding authorization mechanism being deactivated.

---

### Finding Description

In `EvmErc20.sol`, `withdrawToNear` (and `withdrawToEthereum`) follows this sequence:

```solidity
function withdrawToNear(bytes memory recipient, uint256 amount) external override {
    _burn(_msgSender(), amount);          // ← tokens destroyed here, unconditionally

    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", amount_b, recipient);
    uint input_size = 1 + 32 + recipient.length;

    assembly {
        let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
        // res is NEVER checked
    }
}
``` [1](#0-0) 

The identical pattern exists in `EvmErc20V2.sol`: [2](#0-1) 

And for `withdrawToEthereum` in both contracts: [3](#0-2) [4](#0-3) 

The `pause_precompiles` function in `admin.rs` allows any account in the `EngineAuthorizer` ACL to pause `EXIT_TO_NEAR` or `EXIT_TO_ETHEREUM`: [5](#0-4) 

The pause flags are stored and checked via `EnginePrecompilesPauser`: [6](#0-5) 

When the precompile is paused, the `call(...)` in the assembly block returns `0` (failure). Because `res` is never inspected, the outer EVM call does **not** revert. The `_burn` that already executed is committed to state.

The only refund path is `exit_to_near_precompile_callback`, which handles re-minting on promise failure: [7](#0-6) 

This callback is only scheduled if the precompile **successfully** creates a NEAR promise. When the precompile is paused and returns an error, no promise is created, no callback is scheduled, and no refund occurs.

---

### Impact Explanation

**Critical — Permanent freezing/theft of user funds.**

Any user who calls `withdrawToNear` or `withdrawToEthereum` while the corresponding exit precompile is paused will:
- Have their ERC-20 tokens permanently burned (destroyed on the Aurora/EVM side).
- Receive zero NEP-141 tokens on the NEAR side.
- Have no on-chain recovery path, because no refund promise is ever created.

The NEP-141 tokens that were supposed to be released remain locked in the Aurora engine contract indefinitely.

---

### Likelihood Explanation

- `pause_precompiles` is callable by any account in the `EngineAuthorizer` ACL — not exclusively the owner — making activation realistic during incident response.
- Users have no on-chain signal that the precompile is paused before submitting `withdrawToNear`; the transaction appears to succeed (EVM status = success) even though funds are lost.
- The pause is an emergency mechanism, so it may be activated abruptly without user-facing notification, maximizing the window of exposure.
- Both `EvmErc20.sol` (legacy) and `EvmErc20V2.sol` (current) are affected, covering all bridged NEP-141 tokens.

---

### Recommendation

Check the assembly `call` return value and revert if the precompile call fails, so that `_burn` is rolled back atomically:

```solidity
assembly {
    let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
    if iszero(res) { revert(0, 0) }
}
```

This ensures the burn and the cross-chain transfer are atomic: either both succeed or neither does.

---

### Proof of Concept

1. An authorized account calls `pause_precompiles` with the `EXIT_TO_NEAR` bitmask, pausing the `ExitToNear` precompile.
2. A user calls `withdrawToNear(recipient, 1000)` on a deployed `EvmErc20` contract.
3. `_burn(_msgSender(), 1000)` executes — 1000 ERC-20 tokens are destroyed.
4. The assembly `call` to `0xe921...124f` fails (precompile is paused), returning `res = 0`.
5. `res` is not checked; the EVM transaction completes with status `success`.
6. The user's 1000 ERC-20 tokens are gone. No NEP-141 tokens are transferred. No refund promise is created.
7. Funds are permanently lost with no recovery mechanism.

pub fn pause_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        require_running(&state::get_state(&io)?)?;
        let authorizer: EngineAuthorizer = engine::get_authorizer(&io);

        if !authorizer.is_authorized(&env.predecessor_account_id()) {
            return Err(b"ERR_UNAUTHORIZED".into());
        }

        let args: PausePrecompilesCallArgs = io.read_input_borsh()?;
        let flags = PrecompileFlags::from_bits_truncate(args.paused_mask);
        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.pause_precompiles(flags);
        Ok(())
    })
}

bitflags! {
    /// Wraps unsigned integer where each bit identifies a different precompile.
    #[derive(BorshSerialize, BorshDeserialize, Default)]
    #[borsh(crate = "aurora_engine_types::borsh")]
    pub struct PrecompileFlags: u32 {
        const EXIT_TO_NEAR        = 0b01;
        const EXIT_TO_ETHEREUM    = 0b10;
    }
}

impl PrecompileFlags {
    #[must_use]
    pub fn from_address(address: &Address) -> Option<Self> {
        Some(if address == &exit_to_ethereum::ADDRESS {
            Self::EXIT_TO_ETHEREUM
        } else if address == &exit_to_near::ADDRESS {
            Self::EXIT_TO_NEAR
        } else {
            return None;
        })
    }

    /// Checks if the precompile belonging to the `address` is marked as paused.
    #[must_use]
    pub fn is_paused_by_address(&self, address: &Address) -> bool {
        Self::from_address(address).is_some_and(|precompile_flag| self.contains(precompile_flag))
    }
}

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
