### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Block Block Reward Distribution - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` is specified as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged caller can invoke it with `disable_rewards: true` in every block, setting the global `last_reward_block` without distributing any rewards, permanently preventing the sequencer from distributing block rewards to all stakers and pool members.

---

### Finding Description

The specification for `update_rewards` in `docs/spec.md` explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation's only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero: [2](#0-1) 

There is no sequencer identity check anywhere in `update_rewards`: [3](#0-2) 

The critical state variable `last_reward_block` is a **global** (not per-staker) storage field: [4](#0-3) 

Once any caller writes to it in a block, the check `current_block_number > self.last_reward_block.read()` fails for every subsequent call in that same block, including the sequencer's legitimate call: [5](#0-4) 

The `disable_rewards: true` path returns immediately after writing `last_reward_block`, distributing zero rewards to zero stakers, making the griefing completely free beyond gas cost: [6](#0-5) 

---

### Impact Explanation

An attacker calling `update_rewards(any_valid_staker, disable_rewards: true)` once per block:

1. Sets `last_reward_block` to the current block number.
2. Distributes no rewards (early return due to `disable_rewards: true`).
3. Causes every subsequent call in that block — including the sequencer's — to revert with `REWARDS_ALREADY_UPDATED`.

All stakers and pool members permanently lose their block rewards for every block the attacker griefs. Because block rewards are never retroactively compensated (the system only distributes rewards for the current block at call time), the lost yield is unrecoverable.

**Impact category: High — Permanent freezing of unclaimed yield for all stakers and pool members.**

---

### Likelihood Explanation

- The function is publicly callable by any non-zero address with no token deposit, no stake, and no special role.
- The only cost is gas per block.
- On Starknet, transaction fees are low, making sustained griefing economically viable.
- A competitor, a malicious actor, or even a protocol-level bot could automate this indefinitely.

**Likelihood: High.**

---

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, consistent with the specification. For example, store the authorized sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation. Alternatively, restrict the function to be callable only by the staking contract's designated operator role using the existing `RolesComponent`.

---

### Proof of Concept

```
// Attacker (any non-zero address) runs this every block:
// 1. Pick any valid, active staker address S.
// 2. Call: staking_contract.update_rewards(staker_address: S, disable_rewards: true)
//    - general_prerequisites() passes (contract unpaused, caller != 0)
//    - current_block_number > last_reward_block  ✓  (first call this block)
//    - last_reward_block := current_block_number  ← global lock set
//    - disable_rewards == true → early return, zero rewards distributed
// 3. Sequencer now calls: staking_contract.update_rewards(staker_address: T, disable_rewards: false)
//    - current_block_number > last_reward_block  ✗  (same block)
//    - PANICS: REWARDS_ALREADY_UPDATED
// Result: No staker receives block rewards this block. Repeated every block = permanent yield freeze.
``` [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
