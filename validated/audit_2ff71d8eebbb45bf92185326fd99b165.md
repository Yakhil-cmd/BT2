### Title
Unbounded `stakers` Vec Inflation Allows Permanent DoS of `get_stakers()` - (File: `src/staking/staking.cairo`)

---

### Summary

The `stakers` storage vector in the `Staking` contract grows monotonically and is never pruned. Any unprivileged caller can permanently append entries to it by staking and unstaking with fresh addresses. The `get_stakers()` function iterates over the **entire** vector on every call, so a sufficiently large vector causes an out-of-gas revert, permanently breaking the consensus-layer view that enumerates active validators.

---

### Finding Description

The `Staking` contract maintains a `stakers: Vec<ContractAddress>` storage field. The inline comment is explicit:

> **"Note: Stakers are not removed from this vector when they unstake."** [1](#0-0) 

Every call to `stake()` unconditionally appends the caller's address to this vector: [2](#0-1) 

The `remove_staker()` internal function, called during `unstake_action()`, clears `staker_info` and the operational-address mapping but **never touches `stakers`**: [3](#0-2) 

`get_stakers()` iterates over `self.stakers.into_iter_full_range()` — the full, ever-growing vector — on every invocation: [4](#0-3) 

An attacker creates N distinct Starknet accounts, stakes the protocol minimum from each, waits for the `exit_wait_window`, and calls `unstake_action()` to recover the principal. Each cycle costs only gas; the staked tokens are returned in full. After enough cycles the vector is so large that `get_stakers()` exhausts the Starknet gas limit and reverts on every call.

---

### Impact Explanation

`get_stakers()` is the canonical entry point used by the consensus layer to enumerate the active validator set for a given epoch. Permanently breaking this call prevents the protocol from producing the validator committee, which disrupts block production and consensus. This matches the **"Unbounded gas consumption / Griefing with damage to the protocol"** impact category.

---

### Likelihood Explanation

- **Entry point is fully permissionless**: `stake()` requires no role, no whitelist, and no approval beyond the token transfer.
- **Attacker recovers principal**: After `unstake_action()` the staked tokens are returned, so the net cost per cycle is only gas.
- **No address-reuse barrier matters**: Even if `assert_staker_address_not_reused` blocks the same address from re-staking, each fresh Starknet account address adds one permanent entry. Accounts are cheap to create.
- **Starknet gas limits are finite**: The loop in `get_stakers()` performs multiple storage reads per entry (staker info, balance traces, public key, peer id). At scale this will exceed the block/call gas cap.

---

### Recommendation

1. **Track active staker count separately** and remove stakers from the `stakers` vector (or use a tombstone flag) when `unstake_action()` is called, so `get_stakers()` only visits live entries.
2. **Alternatively**, replace the full-range iteration with a paginated or off-chain-indexed approach, and enforce a hard cap on the number of registered stakers.
3. **Impose a minimum re-stake cooldown or address-reuse fee** to raise the economic cost of inflating the vector.

---

### Proof of Concept

```
// Pseudocode – repeat N times with fresh accounts
for i in 0..N:
    account_i = create_new_starknet_account()
    strk_token.approve(staking, min_stake, from=account_i)
    staking.stake(reward_address=account_i, operational_address=op_i, amount=min_stake, from=account_i)
    staking.unstake_intent(from=account_i)
    advance_time(exit_wait_window + 1)
    staking.unstake_action(staker_address=account_i)   // principal returned
    // stakers Vec now has N permanent dead entries

// After N iterations:
staking.get_stakers(epoch_id=current_epoch)  // OUT OF GAS
```

Each iteration permanently increments `stakers.len()` while recovering the staked tokens. Once the vector is large enough, every call to `get_stakers()` reverts, preventing the consensus layer from reading the active validator set. [5](#0-4)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
