### Title
Missing Minimum Threshold Enforcement in FROST Signing/Presigning Allows Corrupted Signature Output - (File: `src/frost/mod.rs`)

---

### Summary

The DKG initialization path enforces a minimum threshold of 2 (`threshold >= 2`), but the FROST signing and presigning entry points omit this lower-bound check entirely. A malicious coordinator or caller can invoke signing with `threshold=1`, bypassing the minimum constraint enforced at key generation time and causing the protocol to produce a cryptographically invalid, unusable signature.

---

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces both bounds on the threshold:

```rust
// Step 1.1
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

This guarantees that no key is ever generated with a threshold below 2.

However, `assert_sign_inputs` in `src/frost/mod.rs` — the validation gate for FROST signing — only checks the **upper** bound:

```rust
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [2](#0-1) 

There is **no** corresponding lower-bound check (`threshold >= 2`). The same omission exists in the FROST `presign` function:

```rust
// validate threshold
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [3](#0-2) 

`ReconstructionLowerBound` is a plain `usize` newtype with no internal invariant:

```rust
pub struct ReconstructionLowerBound(usize);
``` [4](#0-3) 

A caller can construct `ReconstructionLowerBound(1)` and pass it directly to `assert_sign_inputs` or `presign`. Both functions accept it without error.

---

### Impact Explanation

In FROST, each participant computes a partial signature using Lagrange coefficients derived from the **signing threshold** and the signing participant set. If `threshold=1` is passed at signing time but the keygen polynomial had degree `T-1 >= 1` (because keygen enforced `T >= 2`), the Lagrange interpolation is performed over the wrong polynomial degree. The aggregated signature scalar satisfies:

```
z * G = R + c * VK_i   (participant's verifying share)
```

instead of:

```
z * G = R + c * PK     (group public key)
```

The resulting signature is **cryptographically invalid** against the group public key. Honest participants who completed the protocol receive an unusable output. Any nonces consumed during the signing round are permanently spent, and if the session is retried with the same nonces (a common failure-recovery pattern), secret key material can be extracted via standard FROST nonce-reuse attacks.

**Impact:** High — Corruption of sign/presign outputs so honest parties accept unusable cryptographic outputs.

---

### Likelihood Explanation

The entry point is the public `assert_sign_inputs` function and the public `presign` function, both callable by any library user or coordinator. No special privilege is required. The `ReconstructionLowerBound` type imposes no compile-time or runtime minimum, so passing `1` is trivial. A malicious coordinator controlling the signing session parameters can trigger this on every signing round.

---

### Recommendation

Add the same lower-bound check present in `assert_key_invariants` to both `assert_sign_inputs` and `presign` in `src/frost/mod.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This mirrors the existing guard in `assert_key_invariants`: [1](#0-0) 

Optionally, encode the invariant directly in `ReconstructionLowerBound`'s constructor so it is impossible to construct a value below 2 at the type level.

---

### Proof of Concept

```rust
use threshold_signatures::frost::assert_sign_inputs;
use threshold_signatures::ReconstructionLowerBound;
use threshold_signatures::participants::Participant;

let p1 = Participant::from(1u32);
let p2 = Participant::from(2u32);
let participants = vec![p1, p2];

// Keygen was run with threshold=2 (enforced minimum).
// Signing is called with threshold=1 — passes all validation:
let result = assert_sign_inputs(
    &participants,
    ReconstructionLowerBound::from(1usize), // below enforced minimum
    p1,
    p1,
);
assert!(result.is_ok()); // succeeds — no lower-bound check

// Downstream: FROST partial signatures are computed with wrong
// Lagrange coefficients, producing an invalid aggregated signature.
```

The bypass is structurally identical to the external report: the minimum is enforced at the "entry" operation (keygen via `assert_key_invariants`) but not at the "exit" operation (signing via `assert_sign_inputs`/`presign`), allowing a caller to operate below the security minimum after the initial constraint has been satisfied. pub fn assert_key_invariants(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
) -> Result<ParticipantList, InitializationError> {
    let threshold = usize::from(threshold.into());
    // need enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // Step 1.1
    // validate threshold
    if threshold > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold,
            max: participants.len(),
        });
    }
    // Step 1.1
    if threshold < 2 {
        return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
    }

    // ensure uniqueness of participants in the participant list
    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }
    Ok(participants)
}

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
