### Title
Signature Validation Bypass via `skip_stateful_validations` — Any Invoke with Nonce=1 Admitted Without `__validate__` When Any Tx Exists for the Sender - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`skip_stateful_validations` skips the `__validate__` entry-point call for an invoke transaction with `nonce=1` whenever `account_tx_in_pool_or_recent_block` returns `true` for the sender. That helper returns `true` for **any** transaction in the pool for that address — not exclusively a `deploy_account`. An attacker who observes a legitimate `deploy_account` in the mempool can immediately submit a crafted invoke (nonce=1, invalid signature, high fee) for the same address. The gateway admits it without signature verification, it enters the mempool, and — after the legitimate `deploy_account` executes and funds the account — the invalid invoke executes, fails in `__validate__`, increments the nonce, and charges a fee, permanently displacing the legitimate user's invoke.

---

### Finding Description

**Invariant broken:** Every invoke transaction admitted to the mempool must have its `__validate__` entry point verified by the blockifier before admission.

**Root cause — `skip_stateful_validations`:** [1](#0-0) 

The function returns `true` (skip validation) when:
1. The transaction is an `Invoke`
2. `tx.nonce() == Nonce(Felt::ONE)`
3. `account_nonce == Nonce(Felt::ZERO)` (account not yet deployed in committed state)
4. `account_tx_in_pool_or_recent_block(sender)` returns `true`

**The flawed guard — `account_tx_in_pool_or_recent_block`:** [2](#0-1) 

This returns `true` if the address has **any** transaction in the pool (`tx_pool.contains_account`) or in the committed state (`state.contains_account`). It does **not** verify that the existing transaction is specifically a `deploy_account`. The code comment claims this is sufficient: [3](#0-2) 

This reasoning is incorrect. An attacker who observes a legitimate `deploy_account` in the mempool satisfies condition 4 without owning the account.

**Effect of skip — `run_validate_entry_point`:** [4](#0-3) 

When `skip_validate=true`, `ExecutionFlags { validate: false, ... }` is set. Inside `StatefulValidator::perform_validations`: [5](#0-4) 

The function returns `Ok(())` after `perform_pre_validation_stage` without ever calling `__validate__`. The signature is never checked.

**`perform_pre_validation_stage` does not check signatures:** [6](#0-5) 

Only nonce, fee bounds, and proof facts are checked — no signature verification.

---

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An attacker submits an invoke with an invalid signature and a fee higher than the legitimate user's invoke. The gateway admits it (signature check bypassed). The mempool's fee-escalation logic replaces the legitimate invoke with the attacker's. After the `deploy_account` executes and funds the account, the blockifier executes the attacker's invalid invoke: `__validate__` fails, the nonce is incremented to 2, and a fee is charged. The legitimate user's invoke (nonce=1) is now permanently displaced — the user must resubmit with nonce=2, having lost the fee from the failed transaction.

---

### Likelihood Explanation

**Likelihood: Medium.** The attack requires observing a `deploy_account` in the mempool (trivially possible on a public node), crafting an invoke with nonce=1 and a higher fee (no special knowledge required), and submitting it before the `deploy_account` is committed. The window is the time between `deploy_account` admission and block commitment. The attack is cheap (only the attacker's fee is at risk, and the attacker can set it to the minimum viable escalation amount).

---

### Recommendation

Replace the `account_tx_in_pool_or_recent_block` check with a check that specifically verifies a `deploy_account` transaction exists for the sender address. The mempool should expose a dedicated `deploy_account_in_pool_or_recent_block(address)` query that only returns `true` when the pooled or recently committed transaction for that address is of type `DeployAccount`. Alternatively, store the transaction type alongside the address in `MempoolState::committed`/`staged` so the gateway can distinguish deploy-account entries from invoke entries.

---

### Proof of Concept

1. Legitimate user broadcasts `deploy_account(class_hash=C, salt=S, nonce=0)` → address `X` is computed and the tx enters the mempool.
2. Attacker observes `X` in the mempool via any RPC node.
3. Attacker crafts `invoke(sender=X, nonce=1, calldata=<arbitrary>, signature=<garbage>, tip=<high>)` and submits it to the gateway.
4. Gateway path:
   - `get_nonce_from_state(X)` → `Nonce(0)` (X not yet deployed)
   - `validate_nonce`: nonce=1 is within `[0, 0+max_allowed_nonce_gap]` ✓
   - `validate_by_mempool`: no duplicate, nonce not too old ✓
   - `skip_stateful_validations`: `tx.nonce()==1`, `account_nonce==0`, `account_tx_in_pool_or_recent_block(X)==true` (deploy_account is in pool) → returns `true`
   - `run_validate_entry_point(skip_validate=true)` → `validate: false` → `perform_pre_validation_stage` passes, `__validate__` is **never called**
   - Attacker's invoke is admitted to the mempool.
5. Mempool fee-escalation: attacker's invoke (higher fee) replaces the legitimate user's invoke (nonce=1).
6. Batcher executes: `deploy_account` (nonce=0) succeeds, account is funded. Attacker's invoke (nonce=1) is executed: `__validate__` fails (invalid signature), nonce incremented to 2, fee charged.
7. Legitimate user's invoke is gone; nonce is now 2. User must resubmit.

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
