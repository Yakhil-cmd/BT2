### Title
`update_rewards` Marks Block as Processed Without Distributing Rewards Due to Missing Access Control — (`File: src/staking/staking.cairo`)

### Summary

`StakingRewardsManagerImpl::update_rewards` writes `last_reward_block` to storage (marking the block as "processed") before checking the `disable_rewards` flag, and has no access-control guard. Any unprivileged caller can invoke it with `disable_rewards = true`, consuming the per-block reward slot without distributing any rewards. Because `last_reward_block` is a single global value, the legitimate sequencer is then blocked from distributing rewards for that block to any staker.

### Finding Description

`update_rewards` is specified as callable only by the Starknet sequencer (spec: "Only starkware sequencer"). However, the implementation only calls `general_prerequisites()`, which checks `is_paused` and `caller_is_not_zero` — no role check is performed. [1](#0-0) 

The critical ordering issue is:

```
self.last_reward_block.write(current_block_number);   // ← state written unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← rewards NOT written, but block is consumed
}
``` [2](#0-1) 

`last_reward_block` is a single global storage slot, not per-staker: [3](#0-2) 

The guard that prevents double-processing reads this same slot: [4](#0-3) 

So once an attacker writes `last_reward_block = N`, no call to `update_rewards` in block `N` can succeed — for any staker.

The actual reward accumulation (`staker_info.unclaimed_rewards_own += staker_rewards`) only happens inside `_update_rewards`, which is never reached when `disable_rewards = true`: [5](#0-4) 

### Impact Explanation

An unprivileged attacker calls `update_rewards(any_active_staker, disable_rewards: true)` once per block. Each such call:

1. Writes `last_reward_block = current_block` — the block is permanently marked as processed.
2. Skips `_update_rewards` — no staker's `unclaimed_rewards_own` is incremented.
3. Blocks the sequencer from calling `update_rewards` again in the same block (reverts with `REWARDS_ALREADY_UPDATED`).

Repeated across every block, this permanently freezes all consensus-era block rewards for all stakers. This matches **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- `update_rewards` is a public, permissionless entry point (no role check).
- The attacker only needs to submit one transaction per block with a valid (active, non-zero-balance) staker address and `disable_rewards = true`.
- The cost is one transaction per block; the damage is all staker rewards for that block.
- On Starknet L2 the sequencer controls intra-block ordering, but the attacker can submit their transaction in the same block before the sequencer's inclusion, or in any block where the sequencer has not yet called `update_rewards`.

### Recommendation

Add a sequencer-only role check at the top of `update_rewards`, consistent with the spec's stated access control ("Only starkware sequencer"):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer();   // add role guard
    self.general_prerequisites();
    ...
}
```

Alternatively, move `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` / `is_pre_consensus` check so that the block slot is only consumed when rewards are actually distributed.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Staker `S` is active with non-zero balance at the current epoch.
3. Attacker calls `update_rewards(S, disable_rewards: true)` in block `N`.
   - `last_reward_block` is written to `N`.
   - Function returns early; `S.unclaimed_rewards_own` is unchanged.
4. Sequencer attempts `update_rewards(S, disable_rewards: false)` in block `N`.
   - Reverts: `REWARDS_ALREADY_UPDATED` (`current_block_number > last_reward_block` fails).
5. Block `N` rewards are permanently lost for all stakers.
6. Attacker repeats step 3 every block → all future rewards are frozen.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
