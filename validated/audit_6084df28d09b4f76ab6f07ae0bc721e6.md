### Title
P2P Sync Receipt Deserialization Unconditionally Drops All Transaction Events - (File: `crates/apollo_protobuf/src/converters/receipt.rs`)

### Summary
Every `TryFrom<protobuf::receipt::*>` implementation for transaction output types in the P2P sync receipt converter hardcodes `let events = vec![];`. Because the P2P sync client has no separate event-protocol channel, any node syncing blocks via P2P stores empty event lists for every transaction in every synced block. RPC queries on those nodes return authoritative-looking empty event arrays for transactions that actually emitted events.

### Finding Description

**Root cause — serialization side drops events**

`crates/apollo_protobuf/src/converters/receipt.rs` serializes a `TransactionOutput` into a `protobuf::Receipt` by calling `create_proto_receipt_common_from_txn_output_fields`, which accepts only `actual_fee`, `messages_sent`, `execution_resources`, and `execution_status`. The `events` field of every output variant is silently discarded: [1](#0-0) 

**Root cause — deserialization side hardcodes empty events**

Every `TryFrom` implementation for the five receipt types explicitly sets `let events = vec![];` before constructing the output struct: [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

**P2P sync client has no event channel**

The P2P sync client's channel set contains only `header_sender`, `state_diff_sender`, `transaction_sender`, and `class_sender`. There is no event receiver: [6](#0-5) 

The `TransactionStreamFactory` receives `FullTransaction` objects, pushes the deserialized (events-empty) `transaction_output` directly into `block_body.transaction_outputs`, and writes the whole body to storage: [7](#0-6) 

A TODO comment in the same file acknowledges the gap: `// TODO(Eitan): Add events protocol to BlockBody or split their write to storage`.

**P2P server has an event channel the client never consumes**

The P2P sync server registers and serves a separate `Event` protocol channel: [8](#0-7) 

It serves events by reading `transaction_output.events()` from storage: [9](#0-8) 

But the client never registers a corresponding receiver, so those events are never consumed and never written to the synced node's storage.

**Downstream: storage writes empty events**

`write_transactions` calls `write_events(tx_output, ...)` for each transaction output. Because `tx_output.events()` is always `[]` on a P2P-synced node, the events index is never populated: [10](#0-9) 

**Downstream: block hash event_commitment computed from empty set**

`calculate_block_commitments` derives `event_leaf_elements` from `transaction_data.transaction_output.events`. On a P2P-synced node this slice is always empty, so `event_commitment` is the Poseidon root of an empty tree — a value that diverges from the canonical commitment stored in the block header: [11](#0-10) 

The test for transaction outputs over P2P is commented out with a matching TODO: [12](#0-11) 

### Impact Explanation

Any node that syncs blocks via the P2P protocol (the intended mechanism for non-sequencer full nodes) will store `events = []` for every transaction in every synced block. Subsequent RPC calls (`starknet_getTransactionReceipt`, `starknet_getEvents`) on that node return empty event arrays for transactions that actually emitted events. The response is structurally valid and carries no error indicator, so callers treat it as authoritative. Additionally, when such a node acts as a P2P sync server for downstream peers, it propagates the empty-event state further.

This matches: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

The trigger is simply syncing any block that contains transactions with events via P2P — which is every non-trivial block on Starknet. No special transaction crafting or privileged access is required. Any operator running a full node with P2P sync enabled is affected.

### Recommendation

1. **Implement the event protocol on the client side**: Register an event-protocol receiver in `P2pSyncClientChannels` and implement a corresponding `EventStreamFactory` that fetches events from the server and merges them into the stored `transaction_output` for each block.
2. **Alternatively, embed events in the `Transaction` protocol receipt**: Extend `protobuf::receipt::Common` with a repeated `events` field, update `create_proto_receipt_common_from_txn_output_fields` to serialize events, and update all `TryFrom` implementations to deserialize them instead of hardcoding `vec![]`.
3. **Add a round-trip test**: Uncomment and fix the commented-out assertion in `transaction_test.rs` that checks `actual_transaction_outputs == transaction_outputs`.

### Proof of Concept

1. Start a P2P sync client node and sync any block containing ERC-20 transfer transactions (which emit `Transfer` events).
2. Query `starknet_getTransactionReceipt` for one of those transactions on the synced node.
3. Observe `"events": []` in the response.
4. Issue the same query against the sequencer node; observe the correct non-empty event list.
5. Confirm the discrepancy: the block header's `event_commitment` on the synced node (received from the sequencer) is non-zero, but recomputing it locally from the stored (empty) events yields the Poseidon root of an empty tree — a mismatch that proves the stored events are wrong.

    fn parse_data_for_block<'a>(
        transactions_response_manager: &'a mut ClientResponsesManager<DataOrFin<FullTransaction>>,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut block_body = BlockBody::default();
            let mut current_transaction_len = 0;
            let target_transaction_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .n_transactions;
            while current_transaction_len < target_transaction_len {
                let maybe_transaction = transactions_response_manager.next().await.ok_or(
                    ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }),
                )?;
                let Some(FullTransaction { transaction, transaction_output, transaction_hash }) =
                    maybe_transaction?.0
                else {
                    if current_transaction_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::NotEnoughTransactions {
                            expected: target_transaction_len,
                            actual: current_transaction_len,
                            block_number: block_number.0,
                        }));
                    }
                };
                block_body.transactions.push(transaction);
                block_body.transaction_outputs.push(transaction_output);
                // TODO(eitan): Validate transaction hash from untrusted sources
                block_body.transaction_hashes.push(transaction_hash);
                current_transaction_len += 1;
            }
            Ok(Some((block_body, block_number)))
        }
        .boxed()
    }

#[async_trait]
impl FetchBlockData for (Event, TransactionHash) {
    async fn fetch_block_data(
        block_number: BlockNumber,
        txn: &StorageTxn<'_, db::RO>,
        _class_manager_client: &mut SharedClassManagerClient,
    ) -> Result<Vec<Self>, P2pSyncServerError> {
        let transaction_outputs = txn.get_block_transaction_outputs(block_number)?.ok_or(
            P2pSyncServerError::BlockNotFound {
                block_hash_or_number: BlockHashOrNumber::Number(block_number),
            },
        )?;
        let transaction_hashes = txn.get_block_transaction_hashes(block_number)?.ok_or(
            P2pSyncServerError::BlockNotFound {
                block_hash_or_number: BlockHashOrNumber::Number(block_number),
            },
        )?;

        let mut result = Vec::new();
        for (transaction_output, transaction_hash) in
            transaction_outputs.into_iter().zip(transaction_hashes)
        {
            for event in transaction_output.events() {
                result.push((event.clone(), transaction_hash));
            }
        }
        Ok(result)
    }
