### Title
`set_commission` Does Not Settle Current Epoch Rewards Before Updating Commission, Allowing Retroactive Commission Increase to Steal Delegator Yield - (File: `src/staking/staking.cairo`)

---

### Summary

`set_commission` in `src/staking/staking.cairo` updates the commission rate immediately in storage without first settling the current epoch's pending rewards at the old rate. When a staker uses a `commission_commitment` to raise their commission mid-epoch, the new higher rate retroactively applies to the **entire epoch's** rewards when `update_rewards_from_attestation_contract` is next called, causing delegators to receive less yield than they earned.

---

### Finding Description

`set_commission` (lines 725–743) calls `update_commission` (lines 1573–1609), which writes the new commission directly to storage:

```cairo
// Update commission in storage.
staker_pool_info.commission.write(Option::Some(commission));
```

No reward settlement occurs before this write.

The commission is consumed in `calculate_staker_pools_rewards` (line 1964):

```cairo
let commission = staker_pool_info.commission();
```

This is called from `_update_rewards`, which is called from `update_rewards_from_attestation_contract` — **once per epoch**, at attestation time. The commission value read at that moment is applied to the **entire epoch's** pool rewards, regardless of when during the epoch the commission was changed.

The `set_commission_commitment` mechanism (lines 748–785) explicitly allows a staker to set a `max_commission` higher than their current commission. With an active commitment, `update_commission` permits raising commission up to `max_commission` (lines 1583–1589):

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
}
```

There is no guard requiring `commission < old_commission` when a commitment is active. A staker can therefore raise commission at any point during an epoch, then attest, causing the elevated rate to apply retroactively to all rewards accumulated since the epoch began.

---

### Impact Explanation

This is **theft of unclaimed yield** from delegators. The staker captures commission on rewards that accrued while the lower rate was in effect. For example, if epoch rewards attributable to a pool are 1,000 STRK and commission is raised from 10% to 50% just before attestation, delegators receive 500 STRK instead of the 900 STRK they earned — a loss of 400 STRK per epoch per pool. The staker pockets the difference.

This matches the allowed impact: **High — Theft of unclaimed yield**.

---

### Likelihood Explanation

The attack requires:
1. The staker to call `set_commission_commitment` with a `max_commission` above the current rate (a one-time setup, callable by any staker with a pool).
2. The staker to call `set_commission` to raise the rate at any point before attesting in the target epoch.
3. The staker to attest normally.

Steps 2 and 3 are routine staker operations. The setup in step 1 is a single permissionless transaction. Any staker operating a pool can execute this attack at will once the commitment is in place. The attack is profitable and repeatable every epoch.

---

### Recommendation

Before writing the new commission to storage in `set_commission` / `update_commission`, settle the current epoch's rewards using the old commission rate (analogous to calling `applyInterestForToken` in the referenced report). Alternatively, buffer commission changes so they only take effect at the start of the next epoch, ensuring the old rate applies to all rewards accrued in the current epoch.

---

### Proof of Concept

1. Staker has `commission = 10%` and an active pool with delegators.
2. Staker calls `set_commission_commitment(max_commission: 50%, expiration_epoch: current_epoch + 1)`.
3. Epoch N begins; delegators hold positions expecting ≤10% commission.
4. Staker calls `set_commission(commission: 50%)` — permitted by the active commitment.
   - `update_commission` writes `50%` to storage immediately. [1](#0-0) 
5. Staker attests; `update_rewards_from_attestation_contract` is triggered.
6. `calculate_staker_pools_rewards` reads `commission = 50%` from storage. [2](#0-1) 
7. `split_rewards_with_commission` splits the **entire epoch's** pool rewards at 50%. [3](#0-2) 
8. Delegators receive only 50% of pool rewards instead of 90%, losing yield for the full epoch.

The root cause — commission written without prior reward settlement — is at: [4](#0-3) 

The consuming site that applies the commission to the full epoch's rewards is: [5](#0-4)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );
                total_commission_rewards += commission_rewards;
                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)
