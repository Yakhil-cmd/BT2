### Title
Gateway Stateful Validation Checks Only L2 Gas Price Threshold, Silently Skipping L1 Gas and L1 Data Gas Price Bounds — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The Apollo Gateway's stateful resource-bounds validation only checks the transaction's `l2_gas.max_price_per_unit` against the previous block's L2 gas price. It performs **no analogous check** for `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. A transaction whose L1 gas or L1 data gas price is below the current block's actual prices will pass all gateway admission checks and be inserted into the mempool, but will be unconditionally rejected by the blockifier's `perform_pre_validation_stage` at execution time with `InsufficientResourceBounds`. This is a direct structural analog to the SuperPool M-8 bug: the gateway checks a partial/local bound (L2 gas price) while ignoring the global/actual bounds (L1 gas price, L1 data gas price), admitting transactions that are guaranteed to fail.

---

### Finding Description

**Stateless validator** (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, lines 56–88) checks:
1. `max_possible_fee(Tip::ZERO) != Fee(0)` — rejects only if the *total* fee across all resources is zero.
2. `l2_gas.max_price_per_unit >= config.min_gas_price` — a static floor on L2 gas price only.
3. `l2_gas.max_amount <= config.max_l2_gas_amount` — an upper bound on L2 gas amount only.

No check is performed on `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. [1](#0-0) 

**Stateful validator** (`crates/apollo_gateway/src/stateful_transaction_validator.rs`, lines 223–243 and 358–390) calls `validate_tx_l2_gas_price_within_threshold`, which matches on `ValidResourceBounds`:

```rust
ValidResourceBounds::AllResources(tx_resource_bounds) => {
    let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
    // ... checks only l2_gas price against previous block threshold
}
ValidResourceBounds::L1Gas(_) => {
    // No validation required for legacy transactions.
}
```

The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields of `AllResources` are **never validated** against the current block's actual prices. The code even carries an explicit acknowledgment of this gap:

```
// TODO(Arni): Consider running this validation for all gas prices.
``` [2](#0-1) 

**Blockifier pre-validation** (`crates/blockifier/src/transaction/account_transaction.rs`, lines 377–475) *does* check all three resource bounds against actual block gas prices:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [3](#0-2) 

This means the gateway and blockifier apply **different, inconsistent caps** to the same field — exactly the structural pattern of the SuperPool M-8 bug.

**Attack path:**

1. Attacker submits an `AllResources` invoke transaction with:
   - `l2_gas.max_price_per_unit` ≥ gateway threshold (passes stateful check)
   - `l1_gas.max_price_per_unit = 0` (below actual block L1 gas price)
   - `l1_data_gas.max_price_per_unit = 0` (below actual block L1 data gas price)
2. Stateless validator passes: `max_possible_fee > 0` (L2 gas contribution is nonzero), L2 gas price ≥ floor.
3. Stateful validator passes: only L2 gas price is checked against the previous block.
4. Mempool admits the transaction.
5. Batcher fetches the transaction and calls blockifier execution.
6. `perform_pre_validation_stage` → `check_fee_bounds` rejects with `InsufficientResourceBounds { errors: [MaxGasPriceTooLow { resource: L1Gas, ... }] }`. [4](#0-3) 

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

Transactions with zero or sub-threshold L1 gas or L1 data gas prices are admitted to the mempool despite being guaranteed to fail at blockifier execution. This:

- Pollutes the mempool with permanently unexecutable transactions.
- Forces the batcher to fetch, convert, and attempt execution of these transactions before discarding them, wasting sequencer CPU and I/O.
- Provides an unprivileged DoS vector: an attacker can continuously submit transactions that pass the L2 gas price threshold (paying only the L2 gas price cost in terms of mempool admission) while setting L1 gas price to zero, filling the mempool with transactions that will never execute.

The mempool's capacity limit (`capacity_in_bytes`, default 1 GB) bounds the total damage per epoch, but the attacker can sustain the attack across blocks.

---

### Likelihood Explanation

**Medium.** The attack requires no privileged access. Any user can craft an `AllResources` V3 transaction with a valid L2 gas price and zero L1 gas price. The only cost is meeting the L2 gas price threshold. The gap is confirmed by the in-code TODO comment, indicating it is a known but unaddressed omission.

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a new `validate_all_gas_prices` function) to check `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding prices, mirroring the existing L2 gas price check:

```rust
fn validate_tx_l2_gas_price_within_threshold(
    &self,
    tx_resource_bounds: ValidResourceBounds,
    previous_block_gas_prices: GasPriceVector,  // pass full vector, not just L2
) -> StatefulTransactionValidatorResult<()> {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(bounds) => {
            // Check L2 gas price (existing)
            self.check_price(bounds.l2_gas.max_price_per_unit,
                             previous_block_gas_prices.l2_gas_price)?;
            // Check L1 gas price (new)
            self.check_price(bounds.l1_gas.max_price_per_unit,
                             previous_block_gas_prices.l1_gas_price)?;
            // Check L1 data gas price (new)
            self.check_price(bounds.l1_data_gas.max_price_per_unit,
                             previous_block_gas_prices.l1_data_gas_price)?;
        }
        ValidResourceBounds::L1Gas(_) => { /* legacy */ }
    }
    Ok(())
}
```

This ensures the gateway's admission invariant matches the blockifier's execution invariant, eliminating the class of transactions that pass gateway validation but fail at execution.

### Proof of Concept

```
1. Block N has: l1_gas_price = 100, l1_data_gas_price = 50, l2_gas_price = 10.
2. Gateway config: validate_resource_bounds = true, min_gas_price_percentage = 100.
   → L2 gas price threshold = 10.

3. Attacker submits AllResources invoke tx:
   resource_bounds = AllResourceBounds {
       l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 0 },  // zero!
       l2_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 10 }, // meets threshold
       l1_data_gas: ResourceBounds { max_amount: 1000, max_price_per_unit: 0 },  // zero!
   }

4. Stateless validator:
   - max_possible_fee(Tip::ZERO) = 0*0 + 1000*10 + 0*0 = 10000 ≠ 0 → PASS
   - l2_gas.max_price_per_unit (10) >= min_gas_price (0) → PASS

5. Stateful validator:
   - validate_tx_l2_gas_price_within_threshold:
     l2_gas.max_price_per_unit (10) >= threshold (10) → PASS
     l1_gas.max_price_per_unit: NOT CHECKED → PASS (silently)
     l1_data_gas.max_price_per_unit: NOT CHECKED → PASS (silently)

6. Mempool admits transaction.

7. Batcher fetches tx, blockifier calls perform_pre_validation_stage:
   - l1_gas.max_price_per_unit (0) < actual_l1_gas_price (100) → FAIL
   - Error: InsufficientResourceBounds { errors: [MaxGasPriceTooLow { resource: L1Gas }] }

8. Transaction is discarded. Attacker repeats indefinitely.
``` [5](#0-4) [6](#0-5)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)

                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
