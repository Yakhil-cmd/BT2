### Title
Integer Division Truncation in `native_per_pubdata` Computation Silently Zeros Pubdata Cost, Enabling Free State Writes — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

In `validate_and_compute_fee_for_transaction`, the per-pubdata-byte native resource rate is computed with floor (truncating) division: `pubdata_price.wrapping_div(native_price)`. When `pubdata_price < native_price`, this silently produces `native_per_pubdata = 0`. Every downstream pubdata-cost function then multiplies by zero, making all pubdata completely free for the transaction. Any unprivileged sender can exploit this to write unlimited storage without paying the corresponding native resource cost.

---

### Finding Description

The root cause is in `validate_and_compute_fee_for_transaction`:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs, line 142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`wrapping_div` is integer floor division. Whenever `pubdata_price < native_price` (e.g., `pubdata_price = 1`, `native_price = 2`), the result is `0`. No error is returned; the zero propagates silently. [1](#0-0) 

This zero then flows into two critical cost-charging sites:

**1. Intrinsic pubdata overhead in `create_resources_for_tx`:**

```rust
// gas_helpers.rs, line 352
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
```

With `native_per_pubdata_byte = 0`, the intrinsic pubdata overhead is zero, so the native limit is not reduced for the mandatory per-transaction pubdata. [2](#0-1) 

**2. Execution pubdata charging in `get_resources_to_charge_for_pubdata`:**

```rust
// gas_helpers.rs, lines 430-432
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)
    .ok_or(out_of_native_resources!())?;
```

With `native_per_pubdata = 0`, `native = 0` regardless of how many bytes of pubdata the transaction actually wrote. The returned `S::Resources` object carries zero native cost. [3](#0-2) 

This function is called both during execution (in `check_enough_resources_for_pubdata`) and at the refund step (in `before_refund` for ZK transactions and `compute_gas_refund`), so the zero cost is applied at every pubdata-charging point. [4](#0-3) 

By contrast, `native_per_gas` is computed with **ceiling** division (`div_ceil`), making the asymmetry explicit:

```rust
// validation_impl.rs, line 135
u256_try_to_u64(&gas_price.div_ceil(native_price))
``` [5](#0-4) 

The same floor-division truncation exists in the `api/src/helpers.rs` helper that mirrors bootloader logic for off-chain validation:

```rust
// api/src/helpers.rs, line 427
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [6](#0-5) 

---

### Impact Explanation

When `native_per_pubdata` truncates to zero:

- **Pubdata is completely free.** A transaction can perform an arbitrary number of `SSTORE` operations (each generating 32+ bytes of pubdata) without consuming any native resource budget for them.
- **Operator financial loss.** The operator must pay L1 data-availability costs for every pubdata byte published, but collects zero native-resource fees for them from the sender.
- **State bloat.** An attacker can inflate the L2 state at no cost beyond EVM gas, which is bounded by the block gas limit but does not account for the L1 publication cost.
- **Resource accounting divergence.** The native resource model is supposed to reflect the true off-chain proving and DA cost. Silently zeroing pubdata cost breaks this invariant and can cause the prover's actual cost to exceed what was collected.

---

### Likelihood Explanation

The condition `pubdata_price < native_price` is reachable in practice:

- Both values are operator-set block-context parameters. An operator may legitimately set a low `pubdata_price` relative to `native_price` during periods of cheap L1 DA.
- The truncation is silent — no log, no rejection, no minimum enforcement — so neither the operator nor the sender is alerted.
- Once the condition holds, **every** L2 transaction in that block benefits from free pubdata, not just a specially crafted one. Any unprivileged sender can exploit it by simply writing storage.

---

### Recommendation

Replace `wrapping_div` with `div_ceil` for `native_per_pubdata`, consistent with how `native_per_gas` is computed:

```rust
// validation_impl.rs line 142 — fix
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix to `api/src/helpers.rs` line 427. This ensures that even when `pubdata_price < native_price`, at least 1 native unit is charged per pubdata byte, preserving the cost floor.

---

### Proof of Concept

1. Operator sets block context: `native_price = 100`, `pubdata_price = 50`.
2. `native_per_pubdata = 50.wrapping_div(100) = 0`.
3. Attacker submits an L2 EIP-1559 transaction with `gas_limit = 300_000`, `max_fee_per_gas = 200` (satisfies `gas_price >= native_price` so `native_per_gas = ceil(200/100) = 2 > 0`).
4. Transaction body executes 100 `SSTORE` operations → ~3,200 bytes of pubdata.
5. `get_resources_to_charge_for_pubdata` returns `native = 3200 * 0 = 0`.
6. Transaction succeeds; attacker pays only EVM gas, zero native for pubdata.
7. Operator publishes 3,200 bytes to L1 at their own cost with no fee recovery. [7](#0-6) [1](#0-0)

pub fn get_resources_to_charge_for_pubdata<S: EthereumLikeTypes>(
    system: &mut System<S>,
    native_per_pubdata: u64,
    base_pubdata: Option<u64>,
) -> Result<(u64, S::Resources), SystemError> {
    let current_pubdata_spent = system
        .net_pubdata_used()?
        .saturating_sub(base_pubdata.unwrap_or(0));
    let native = current_pubdata_spent
        .checked_mul(native_per_pubdata)
        .ok_or(out_of_native_resources!())?;
    let native = <S::Resources as zk_ee::system::Resources>::Native::from_computational(native);
    Ok((current_pubdata_spent, S::Resources::from_native(native)))
}
