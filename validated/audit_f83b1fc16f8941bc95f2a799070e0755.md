### Title
Missing Caller Authorization on `update_rewards` Allows Any Account to Freeze or Redirect Staker Rewards - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the `Staking` contract is callable by any address. The protocol specification explicitly restricts this function to "Only starkware sequencer," but the implementation contains no such check. An unprivileged attacker can exploit this to either permanently freeze all staker reward distribution or redirect block rewards exclusively to an attacker-controlled staker.

### Finding Description
The `IStakingRewardsManager::update_rewards` function is the consensus-era mechanism by which the Starkware sequencer distributes per-block rewards to stakers. The spec at `docs/spec.md` line 1644–1645 states:

> **access control:** Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1447–1507 performs no caller identity check whatsoever:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no get_caller_address() check ...
``` [1](#0-0) 

The only guard is `REWARDS_ALREADY_UPDATED`, which checks that `current_block_number > last_reward_block`. The `last_reward_block` is a **single global storage variable** (not per-staker), meaning only one call per block is possible — and whoever calls first wins. [2](#0-1) [3](#0-2) 

The spec requirement is documented clearly: [4](#0-3) 

### Impact Explanation
Two distinct high-impact attack paths exist:

**Path 1 — Permanent Freezing of Unclaimed Yield (High)**
An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` on every block. The function updates `last_reward_block` to the current block and returns early without distributing any rewards. The legitimate sequencer then receives `REWARDS_ALREADY_UPDATED` for that block. Repeated every block, this permanently prevents all stakers from accruing consensus-era rewards.

**Path 2 — Theft of Unclaimed Yield (High)**
An attacker who is also a staker calls `update_rewards(attacker_staker_address, disable_rewards: false)` on every block, front-running the legitimate sequencer. Because `last_reward_block` is global, only one staker receives rewards per block. The attacker's staker captures 100% of block reward distributions while all other stakers receive nothing.

Both impacts fall within the allowed scope: "High: Theft of unclaimed yield" and "High: Permanent freezing of unclaimed yield."

### Likelihood Explanation
- Requires no special role, key, or privilege — any EOA can call this function.
- The attack is trivially automated: submit one transaction per block.
- The `disable_rewards: true` griefing path requires no stake at all.
- The reward-redirection path requires only that the attacker is a registered staker (a permissionless action).

### Recommendation
Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured sequencer address (or a dedicated rewards-manager role). This mirrors the pattern already used in `update_rewards_from_attestation_contract`, which correctly calls `self.assert_caller_is_attestation_contract()`: [5](#0-4) 

A similar `assert_caller_is_sequencer()` guard should be introduced and called at the start of `update_rewards`.

### Proof of Concept

```cairo
// Any unprivileged account can call this every block to freeze all rewards:
staking_rewards_dispatcher.update_rewards(
    staker_address: any_valid_staker,
    disable_rewards: true,   // no rewards distributed, last_reward_block updated
);

// Next block: repeat. The legitimate sequencer always gets REWARDS_ALREADY_UPDATED.
// Result: all stakers permanently earn zero consensus rewards.
```

```cairo
// Alternatively, an attacker who is a staker calls this every block to steal yield:
staking_rewards_dispatcher.update_rewards(
    staker_address: attacker_staker_address,
    disable_rewards: false,  // rewards flow only to attacker's staker
);
// All other stakers receive zero rewards for every block.
```

The `_update_rewards` internal function confirms that rewards are credited exclusively to the single `staker_address` argument passed in, and `last_reward_block` is then locked for that block: [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
