## Verdict

### Title
`get_nonce(pending, contract_deployed_only_in_pending)` returns `CONTRACT_NOT_FOUND` instead of `Nonce::default()` — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary
When a contract is deployed in the pending block via a `deploy` syscall (appearing in `deployed_contracts` of the pending state diff but with no explicit `nonces` entry, because nonce 0 is the default and is not emitted), `starknet_getNonce` with `block_id = Tag(Pending)` returns `CONTRACT_NOT_FOUND` instead of `Nonce(0)`. This is a concrete wrong value returned by the RPC pending view.

### Finding Description
`get_nonce` reads only `state_diff.nonces` from the pending data. `execution_utils::get_nonce_at` checks that map; on a miss it queries committed storage, which returns `None` for a pending-only contract. The `None` is mapped to `CONTRACT_NOT_FOUND`. By contrast, `get_class_hash_at` explicitly iterates `deployed_contracts` in the pending diff and returns the correct result. The asymmetry means a contract that is visible via `get_class_hash_at(pending, addr)` is invisible via `get_nonce(pending, addr)`.

### Impact Explanation
**High — RPC pending view returns an authoritative-looking wrong value.** Any client (wallet, SDK, dApp) that queries the nonce of a freshly deployed pending contract to construct the next transaction will receive `CONTRACT_NOT_FOUND` and cannot proceed, even though the contract legitimately exists in the pending state with nonce 0.

### Likelihood Explanation
Every `deploy` syscall that lands in the pending block and whose target address has never transacted produces this condition. The trigger is fully unprivileged: any user can send a transaction that calls `deploy`, then immediately call `starknet_getNonce` on the new address with `block_id = pending`.

### Recommendation
In `execution_utils::get_nonce_at`, add a `pending_deployed_contracts` parameter (mirroring `get_class_hash_at`). After the pending-nonces miss and before the storage fallback, check whether the address appears in `pending_deployed_contracts`; if so, return `Ok(Some(Nonce::default()))`. Update `get_nonce` in `api_impl.rs` to pass `state_diff.deployed_contracts` alongside `state_diff.nonces`.

pub fn get_nonce_at<Mode: TransactionKind>(
    txn: &StorageTxn<'_, Mode>,
    state_number: StateNumber,
    pending_nonces: Option<&IndexMap<ContractAddress, Nonce>>,
    contract_address: ContractAddress,
) -> StorageResult<Option<Nonce>> {
    if let Some(pending_nonces) = pending_nonces {
        if let Some(nonce) = pending_nonces.get(&contract_address) {
            return Ok(Some(*nonce));
        }
    }

    txn.get_state_reader()?.get_nonce_at(state_number, &contract_address)
}
