### Title
Unrestricted `update_rewards` Allows Any Staker to Monopolize Consensus Block Rewards Per Block — (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `staking.cairo` is publicly callable with no access control. A global `last_reward_block` guard ensures only one call succeeds per block, but there is no check that the caller is the actual block proposer or that `staker_address` corresponds to the proposer. Any staker can call `update_rewards(their_address, false)` every block in consensus mode, consuming the per-block reward slot for themselves and permanently denying all other stakers their block rewards.

### Finding Description
`update_rewards` is the consensus-mode reward distribution entry point. Its only replay guard is a global storage variable `last_reward_block`:
