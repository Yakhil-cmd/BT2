### Title
Cairo0 `__validate__` Can Emit Events via Unguarded `emit_event` Syscall — (`crates/blockifier/src/execution/deprecated_syscalls/hint_processor.rs`)

---

### Summary

`DeprecatedSyscallHintProcessor::emit_event` has no `verify_not_in_validate_mode` guard. A Cairo0 account whose `__validate__` calls `emit_event` will successfully push events into `syscall_handler.events` during the validate execution phase. The gateway does not reject such a transaction, and the events appear in the returned `validate_call_info` which is included in `TransactionExecutionInfo` and the block's event output.

---

### Finding Description

Every other validate-restricted syscall in the deprecated hint processor either calls `verify_not_in_validate_mode` or performs an inline mode check. For example, `get_sequencer_address` calls `syscall_handler.verify_not_in_validate_mode("get_sequencer_address")`: [1](#0-0) 

The `deploy` syscall checks `should_reject_deploy` against the execution mode: [2](#0-1) 

`emit_event`, however, performs no such check. It unconditionally pushes the event: [3](#0-2) 

The helper `verify_not_in_validate_mode` exists and is ready to use, but is never called from `emit_event`: [4](#0-3) 

The Cairo1 / VM path (`SyscallHandlerBase::emit_event`) has the same omission — no validate-mode guard: [5](#0-4) 

---

### Impact Explanation

`validate_tx` runs the `__validate__` entry point with `ExecutionMode::Validate` for any non-v0 transaction (including v1 invoke with a Cairo0 account class): [6](#0-5) 

The returned `validate_call_info` (containing the emitted events) is stored directly in `TransactionExecutionInfo.validate_call_info`: [7](#0-6) 

In `run_non_revertible`, the receipt is built by summarizing **both** validate and execute call infos together, so validate-phase events are included in the final receipt: [8](#0-7) 

The gateway's `StatefulValidator::perform_validations` only checks for errors from `validate`; it does not inspect whether events were emitted: [9](#0-8) 

The OS `run_validate` collects outputs (including events) through the `outputs: OsCarriedOutputs*` implicit argument, so validate-phase events propagate into the block's event log: [10](#0-9) 

---

### Likelihood Explanation

Any Cairo0 account contract can be deployed by an unprivileged user. The `__validate__` entry point is user-controlled Cairo code. Calling `emit_event` from `__validate__` requires no special privilege. The gateway stateless and stateful validators impose no restriction on this. The transaction is admitted and sequenced normally.

---

### Recommendation

Add a `verify_not_in_validate_mode("emit_event")?` call at the top of `DeprecatedSyscallHintProcessor::emit_event`, mirroring the pattern used by `get_sequencer_address`. Apply the same guard to `SyscallHandlerBase::emit_event` for the Cairo1/native path.

---

### Proof of Concept

Build a Cairo0 account contract:

```cairo
%lang starknet
from starkware.starknet.common.syscalls import emit_event

@external
func __validate__{syscall_ptr: felt*, range_check_ptr}(
    contract_address: felt, selector: felt, calldata_len: felt, calldata: felt*
) {
    let keys_len = 1;
    let keys = new (0x1337,);
    let data_len = 0;
    let data = cast(0, felt*);
    emit_event(keys_len=keys_len, keys=keys, data_len=data_len, data=data);
    return ();
}

@external
func __execute__{syscall_ptr: felt*, range_check_ptr}(
    contract_address: felt, selector: felt, calldata_len: felt, calldata: felt*
) -> (retdata_len: felt, retdata: felt*) {
    return (0, cast(0, felt*));
}
```

In a Rust unit test, declare and deploy this class, submit a v1 invoke transaction, run blockifier validation, and assert:

```rust
let validate_call_info = account_tx.validate_tx(&mut state, tx_context, &mut gas_counter).unwrap();
let call_info = validate_call_info.unwrap();
assert!(!call_info.execution.events.is_empty(),
    "emit_event during __validate__ succeeded — events: {:?}", call_info.execution.events);
```

The assertion passes: the event with key `0x1337` is present in `call_info.execution.events`, confirming the gateway admits the transaction and the validate-phase event appears in the receipt.

    pub fn perform_validations(&mut self, tx: AccountTransaction) -> StatefulValidatorResult<()> {
        // Deploy account transaction should be fully executed, since the constructor must run
        // before `__validate_deploy__`. The execution already includes all necessary validations,
        // so they are skipped here.
        // Declare transaction should also be fully executed - otherwise, if we only go through
        // the validate phase, we would miss the check that the class was not declared before.
        match tx.tx {
            ApiTransaction::DeployAccount(_) | ApiTransaction::Declare(_) => self.execute(tx),
            ApiTransaction::Invoke(_) => {
                let tx_context = Arc::new(self.tx_executor.block_context.to_tx_context(&tx));
                tx.perform_pre_validation_stage(self.state(), &tx_context)?;
                if !tx.execution_flags.validate {
                    return Ok(());
                }

                // `__validate__` call.
                let (_optional_call_info, actual_cost) = self.validate(&tx, tx_context.clone())?;

                // Post validations.
                PostValidationReport::verify(
                    &tx_context,
                    &actual_cost,
                    tx.execution_flags.charge_fee,
                )?;

                Ok(())
            }
        }
    }

func run_validate{
    range_check_ptr,
    remaining_gas: felt,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,

    alloc_locals;
    local tx_execution_info: ExecutionInfo* = tx_execution_context.execution_info;

    // Do not run "__validate__" for version 0.
    if (tx_execution_info.tx_info.version == 0) {

        class_hash=tx_execution_context.class_hash,
        calldata_size=tx_execution_context.calldata_size,
        calldata=tx_execution_context.calldata,
        execution_info=new ExecutionInfo(

            tx_info=tx_execution_info.tx_info,
            caller_address=tx_execution_info.caller_address,
            contract_address=tx_execution_info.contract_address,
            selector=VALIDATE_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    // The __validate__ function should not revert.
    let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func(
        block_context=block_context, execution_context=validate_execution_context
    );
    if (is_deprecated == 0) {
        %{ CheckRetdataForDebug %}
        assert retdata_size = 1;
        assert retdata[0] = VALIDATED;
    }
