**Key facts established:**

1. `setPoolBinAdditionalFees` in `MetricOmmPoolFactory.sol` (lines 450–457) passes `addFeeBuyE6` / `addFeeSellE6` directly to the pool with **zero cap validation**. [1](#0-0)

4. The bin additional fee is applied directly in swap math on top of the base spread fee. [4](#0-3) 

5. `setBinAdditionalFees` on the pool itself also has no cap — it stores whatever the factory sends. [5](#0-4) 

**Rejection analysis:**

The pool admin is explicitly described as **"semi-trusted only inside caps"** in the Smart Audit Pivots, and `setBinAdditionalFees` is explicitly called out as a bypass surface. The factory owner (trusted/rejected) is a separate role from the pool admin (semi-trusted). This is not a malicious pool setup at creation time — it is a post-creation admin action that bypasses a cap the protocol demonstrably intended to enforce (evidenced by the parallel cap system for global fees). ### Title
Pool admin can set per-bin additional fees to `uint16` max with no factory-level cap, bypassing the protocol fee-cap system — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary
`MetricOmmPoolFactory.setPoolBinAdditionalFees` accepts arbitrary `uint16` values for `addFeeBuyE6` / `addFeeSellE6` and forwards them to the pool without any cap check, while the parallel global-fee setter `setPoolAdminFees` enforces `maxAdminSpreadFeeE6`. A pool admin can set per-bin fees to 65535 (~6.55% in E6 units) on any bin, causing traders to pay that fee on top of the global spread fee with no protocol-enforced ceiling.

### Finding Description
The factory maintains a layered fee-cap system: `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20%) is the absolute ceiling, `maxAdminSpreadFeeE6` is the per-pool admin cap, and `setPoolAdminFees` enforces it:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
```

But `setPoolBinAdditionalFees` has no equivalent guard:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

`setBinAdditionalFees` on the pool also stores the values unconditionally. During a swap, the bin additional fee is added directly to the base fee:

```solidity
// MetricOmmPool.sol:910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
```

A pool admin can call `setPoolBinAdditionalFees(pool, bin, 65535, 65535)`, setting ~6.55% additional fee on a specific bin. Any swap routed through that bin pays this fee on top of the global `spreadFeeE6`, with no revert.

### Impact Explanation
Traders executing swaps through the affected bin pay `addFeeBuyE6 = 65535` (~6.55%) on top of the global spread fee. This is direct loss of user principal via excessive fee extraction. The bin additional fee accrues to LPs (including potentially the pool admin), making this a wealth transfer from traders to LPs that bypasses the protocol's intended fee ceiling.

### Likelihood Explanation
Any pool admin — a semi-trusted role, not the factory owner — can execute this in a single transaction with no timelock. The pool admin role is set at pool creation and can be any address. The attack requires no special conditions beyond holding the pool admin role.

### Recommendation
Add a cap check in `setPoolBinAdditionalFees` mirroring the check in `setPoolAdminFees`. Introduce a `maxAdminBinFeeE6` state variable (or reuse `maxAdminSpreadFeeE6`) and enforce it:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminBinFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminBinFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```
