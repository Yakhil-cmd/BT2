### Title
Caller-Controlled `coinbase` in `EVM.run` / `EVM.batchRun` Allows Arbitrary EVM Gas Fee Redirection - (File: fvm/evm/stdlib/contract.cdc)

### Summary

`EVM.run` and `EVM.batchRun` are declared `access(all)` in the EVM Cadence contract and accept a fully caller-controlled `coinbase: EVMAddress` parameter with no validation. Any unprivileged Cadence transaction author can supply any EVM address as the fee recipient, redirecting all EVM gas fees away from the legitimate relayer or application operator to an address they control.

### Finding Description

In Flow EVM, EVM transactions are submitted by wrapping them in a Cadence transaction that calls `EVM.run(tx: [UInt8], coinbase: EVMAddress)` or `EVM.batchRun(txs: [[UInt8]], coinbase: EVMAddress)`. The `coinbase` argument determines where the collected EVM gas fees are sent.

Both functions are declared `access(all)`:

```cadence
access(all)
fun run(tx: [UInt8], coinbase: EVMAddress): Result {
    ...
    return InternalEVM.run(tx: tx, coinbase: coinbase.bytes) as! Result
}

### Impact Explanation

EVM gas fees (denominated in FLOW tokens within the EVM environment) are paid by the EVM transaction signer (the EOA). Those fees accumulate at `CoinbaseAddress` during execution and are then forwarded to the `coinbase` address supplied by the Cadence transaction author.

Because `EVM.run` is `access(all)`, any Cadence transaction author — including the EVM transaction signer themselves — can wrap a signed EVM transaction and specify their own EVM address as `coinbase`. This lets them capture 100% of the EVM gas fees that would otherwise go to the legitimate relayer or application operator.

Concretely:
- A relayer service submits EVM transactions on behalf of users and earns gas fees by specifying its own address as `coinbase`.
- Any user who holds a signed EVM transaction (or who signs their own) can instead submit a competing Cadence transaction with `coinbase` set to their own EVM address, stealing the gas fees.
- For high-volume contracts, an attacker can build a Cadence wrapper that routes EVM calls through `EVM.run` with an attacker-controlled coinbase, capturing fees from all users who interact through that wrapper.

This is a direct loss of funds for legitimate relayers and application operators.

### Likelihood Explanation

The attack requires only the ability to submit a Cadence transaction — the lowest possible privilege level on Flow. No special keys, staked nodes, or admin access are needed. The EVM transaction bytes are observable on-chain (or can be self-generated). The economic incentive is proportional to EVM gas fees, which grow with network usage.

### Recommendation

Restrict the `coinbase` parameter so it cannot be freely chosen by an unprivileged caller. Options include:

1. **Fix the coinbase to the Cadence transaction proposer's COA address** — derive the fee recipient from the signing account rather than accepting it as a free argument.
2. **Require an entitlement or capability** — change `access(all)` to `access(Relayer)` or similar, so only accounts holding a protocol-issued capability can specify a custom coinbase.
3. **Validate the coinbase against a registry** — only allow addresses that have been registered through a permissioned process (analogous to the Canto mitigation of making CSR participation permissioned).

### Proof of Concept

Any unprivileged Cadence transaction can redirect EVM gas fees:

```cadence
import EVM from <EVM_CONTRACT_ADDRESS>

// Attacker submits this transaction instead of using the legitimate relayer.
// `rlpEncodedEVMTx` is a validly signed EVM transaction (e.g., self-signed).
// `attackerEVMAddress` is the attacker's own EVM address.
transaction(rlpEncodedEVMTx: [UInt8], attackerEVMAddress: [UInt8; 20]) {
    execute {
        let coinbase = EVM.EVMAddress(bytes: attackerEVMAddress)
        // All EVM gas fees from this transaction go to attackerEVMAddress,
        // not to the legitimate relayer.
        let result = EVM.run(tx: rlpEncodedEVMTx, coinbase: coinbase)
    }
}
```

The `runWithGasFeeRefund` function will transfer the full gas fee delta from `CoinbaseAddress` to `attackerEVMAddress` with no further checks. [7](#0-6) [8](#0-7)

// runWithGasFeeRefund runs a method and transfers the balance changes of the
// coinbase address to the provided gas fee collector
func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
	// capture coinbase init balance
	cb := h.AccountByAddress(types.CoinbaseAddress, true)
	initCoinbaseBalance := cb.Balance()
	f()
	// transfer the gas fees collected to the gas fee collector address
	afterBalance := cb.Balance()
	diff := new(big.Int).Sub(afterBalance, initCoinbaseBalance)
	if diff.Sign() > 0 {
		cb.Transfer(gasFeeCollector, diff)
	}
	if diff.Sign() < 0 { // this should never happen but in case
		panic(fvmErrors.NewEVMError(fmt.Errorf("negative balance change on coinbase")))
	}
}
