### Title
`setPoolAdminFees` and `setPoolProtocolFee` Permanently Blocked When `collectFees` Reverts, Trapping Protocol Fees - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary

Both `setPoolAdminFees` and `setPoolProtocolFee` in `MetricOmmPoolFactory` unconditionally call `collectFees` on the pool **before** writing the new fee configuration. If `collectFees` reverts — for example because the `adminFeeDestination` or the `FACTORY` address is USDC/USDT-blacklisted — the fee update is permanently blocked and all accrued protocol fees are trapped inside the pool.

### Finding Description

`setPoolAdminFees` (pool-admin path) and `setPoolProtocolFee` (protocol-owner path) both follow the same pattern:

1. Call `pool.collectFees(currentRates, adminFeeDestination)` — which performs live ERC-20 `safeTransfer` calls to `adminFeeDestination` and to `FACTORY`.
2. Only if that succeeds: write the new fee config and call `pool.setPoolFees(newTotals)`. [1](#0-0) [2](#0-1) 

Inside `collectFees`, the pool executes:

```solidity
if (totalFee0ToAdmin > 0) transferToken0(adminFeeDestination_, totalFee0ToAdmin);
if (totalFee1ToAdmin > 0) transferToken1(adminFeeDestination_, totalFee1ToAdmin);
if (totalFee0ToProtocol > 0) transferToken0(FACTORY, totalFee0ToProtocol);
if (totalFee1ToProtocol > 0) transferToken1(FACTORY, totalFee1ToProtocol);
``` [3](#0-2) 

`transferToken0`/`transferToken1` use OpenZeppelin `safeTransfer`: [4](#0-3) 

USDC and USDT maintain on-chain blacklists. A `safeTransfer` to a blacklisted address reverts. If `adminFeeDestination` or `FACTORY` is ever USDC/USDT-blacklisted:

- `collectFees` reverts on every call.
- `setPoolAdminFees` reverts → pool admin cannot change admin fee rates.
- `setPoolProtocolFee` reverts → protocol owner cannot change protocol fee rates.
- `collectPoolFees` reverts → no one can extract accrued fees. [5](#0-4) 

The `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` accumulators and the spread surplus continue to grow with every swap but can never be drained. [6](#0-5) 

### Impact Explanation

**Protocol fees are permanently trapped.** All spread-surplus and notional-fee balances accumulate inside the pool contract but cannot be transferred out. This is a direct loss of protocol and admin fee revenue.

**Fee governance is frozen.** Neither the pool admin nor the protocol owner can update fee rates for the affected pool. The protocol owner cannot enforce fee-cap reductions on existing pools. The pool admin cannot reduce fees to zero to protect LPs.

The `FACTORY` address is an immutable pool parameter — if it is blacklisted, every pool using USDC or USDT as a token is permanently affected with no on-chain remedy.

### Likelihood Explanation

USDC/USDT blacklisting of the `adminFeeDestination` is a realistic scenario: the destination is a mutable address set by the pool admin (potentially a multisig or treasury contract), and USDC/USDT Centre has blacklisted hundreds of addresses. The pool admin can mitigate by calling `setPoolAdminFeeDestination` first (that function does **not** call `collectFees`), but this requires the admin to be responsive. The protocol owner has no independent remedy — they depend on the pool admin to rotate the destination before `setPoolProtocolFee` can succeed.

Blacklisting of `FACTORY` itself is lower probability but would be catastrophic and irreversible for all USDC/USDT pools.

### Recommendation

Decouple fee collection from fee-rate updates. The fee-rate write should never be gated on a successful token transfer. Two options:

1. **Skip collection on revert**: wrap the `collectFees` call in a `try/catch`; if it reverts, proceed with the rate update and leave fees to be collected later via `collectPoolFees`.
2. **Separate the operations**: remove the embedded `collectFees` call from `setPoolAdminFees` and `setPoolProtocolFee` entirely. Callers who want to collect before changing rates can call `collectPoolFees` explicitly first.

### Proof of Concept

1. Pool is created with USDC as `token0` and a non-zero admin spread fee.
2. Swaps occur; spread surplus and notional fees accumulate.
3. USDC Centre blacklists `adminFeeDestination` (e.g., a sanctioned treasury address).
4. Pool admin calls `setPoolAdminFees(pool, 0, 0)` to zero out fees.
5. Transaction reverts inside `collectFees` → `transferToken0(adminFeeDestination, amount)` → USDC reverts.
6. Pool admin calls `setPoolAdminFeeDestination(pool, newDest)` (succeeds — no `collectFees` call here).
7. Pool admin retries `setPoolAdminFees` → now succeeds.
8. **However**: protocol owner calls `setPoolProtocolFee(pool, newRate, newRate)` before step 6 completes → permanently reverts if pool admin is unresponsive.
9. If `FACTORY` itself is blacklisted: steps 6–7 do not help; `collectFees` reverts on the `transferToken0(FACTORY, ...)` leg regardless of `adminFeeDestination`, and there is no on-chain remedy. [7](#0-6) [8](#0-7) [9](#0-8)

  /// @inheritdoc IMetricOmmPoolCollectFees
  function collectFees(
    uint256 protocolSpreadFeeE6_,
    uint256 adminSpreadFeeE6_,
    uint256 protocolNotionalFeeE8_,
    uint256 adminNotionalFeeE8_,
    address adminFeeDestination_
  ) external onlyFactory nonReentrant(PoolActions.COLLECT_FEES) {
    uint256 spreadSumE6;
    uint256 notionalSumE8;
    unchecked {
      spreadSumE6 = protocolSpreadFeeE6_ + adminSpreadFeeE6_;
      notionalSumE8 = protocolNotionalFeeE8_ + adminNotionalFeeE8_;
      if (spreadSumE6 == 0 && notionalSumE8 == 0) {
        return;
      }
    }

    uint256 notionalFee0AmountScaled = notionalFeeToken0Scaled;
    uint256 notionalFee1AmountScaled = notionalFeeToken1Scaled;

    uint256 surplus0Scaled =
      balance0() * TOKEN_0_SCALE_MULTIPLIER - uint256(binTotals.scaledToken0) - notionalFee0AmountScaled;
    uint256 surplus1Scaled =
      balance1() * TOKEN_1_SCALE_MULTIPLIER - uint256(binTotals.scaledToken1) - notionalFee1AmountScaled;

    unchecked {
      uint256 spreadFee0ToAdminScaled = spreadSumE6 == 0 ? 0 : (surplus0Scaled * adminSpreadFeeE6_) / spreadSumE6;
      uint256 spreadFee1ToAdminScaled = spreadSumE6 == 0 ? 0 : (surplus1Scaled * adminSpreadFeeE6_) / spreadSumE6;

      uint256 spreadFee0ToProtocolScaled = spreadSumE6 == 0 ? 0 : (surplus0Scaled * protocolSpreadFeeE6_) / spreadSumE6;
      uint256 spreadFee1ToProtocolScaled = spreadSumE6 == 0 ? 0 : (surplus1Scaled * protocolSpreadFeeE6_) / spreadSumE6;

      uint256 notionalFee0ToAdminScaled =
        notionalSumE8 == 0 ? 0 : (notionalFee0AmountScaled * adminNotionalFeeE8_) / notionalSumE8;
      uint256 notionalFee1ToAdminScaled =
        notionalSumE8 == 0 ? 0 : (notionalFee1AmountScaled * adminNotionalFeeE8_) / notionalSumE8;

      uint256 notionalFee0ToProtocolScaled = notionalFee0AmountScaled - notionalFee0ToAdminScaled;
      uint256 notionalFee1ToProtocolScaled = notionalFee1AmountScaled - notionalFee1ToAdminScaled;

      uint256 totalFee0ToAdminScaled = spreadFee0ToAdminScaled + notionalFee0ToAdminScaled;
      uint256 totalFee1ToAdminScaled = spreadFee1ToAdminScaled + notionalFee1ToAdminScaled;

      uint256 totalFee0ToProtocolScaled = spreadFee0ToProtocolScaled + notionalFee0ToProtocolScaled;
      uint256 totalFee1ToProtocolScaled = spreadFee1ToProtocolScaled + notionalFee1ToProtocolScaled;

      (uint256 totalFee0ToAdmin, uint256 totalFee1ToAdmin) =
        deltasScaledToExternal(totalFee0ToAdminScaled, totalFee1ToAdminScaled, Math.Rounding.Floor);
      (uint256 totalFee0ToProtocol, uint256 totalFee1ToProtocol) =
        deltasScaledToExternal(totalFee0ToProtocolScaled, totalFee1ToProtocolScaled, Math.Rounding.Floor);

      if (totalFee0ToAdmin > 0) {
        transferToken0(adminFeeDestination_, totalFee0ToAdmin);
      }
      if (totalFee1ToAdmin > 0) {
        transferToken1(adminFeeDestination_, totalFee1ToAdmin);
      }
      if (totalFee0ToProtocol > 0) {
        transferToken0(FACTORY, totalFee0ToProtocol);
      }
      if (totalFee1ToProtocol > 0) {
        transferToken1(FACTORY, totalFee1ToProtocol);
      }

      notionalFeeToken0Scaled = 0;
      notionalFeeToken1Scaled = 0;

      emit ProtocolFeesCollected(totalFee0ToProtocol, totalFee1ToProtocol, totalFee0ToAdmin, totalFee1ToAdmin);
    }
  }
