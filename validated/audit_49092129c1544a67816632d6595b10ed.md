### Title
Invalid-Signature Invoke Admitted to Mempool via `skip_stateful_validations` UX Bypass — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`skip_stateful_validations` unconditionally skips the blockifier `__validate__` entry point for any invoke with `nonce=1` and `account_nonce=0` whenever `account_tx_in_pool_or_recent_block` returns `true` for the sender address. An unprivileged attacker who observes a legitimate `deploy_account` transaction in the mempool for address A can submit an invoke with `nonce=1` and an **arbitrary/invalid signature** for address A. The gateway admits it to the mempool without any signature verification.

### Finding Description

The execution path through `extract_state_nonce_and_run_validations`: [1](#0-0) 

calls `run_pre_validation_checks`, which calls `skip_stateful_validations`: [2](#0-1) 

When `tx.nonce() == 1` and `account_nonce == 0`, the function queries `mempool_client.account_tx_in_pool_or_recent_block(sender_address)`. If that returns `true` (i.e., any transaction from that address is in the pool or a recent block), it returns `true` — meaning "skip validation."

The returned `skip_validate = true` is then passed to `run_validate_entry_point`: [3](#0-2) 

which sets `ExecutionFlags { validate: false, ... }` and calls `blockifier_validator.validate(account_tx)`.

Inside the blockifier's `perform_validations`: [4](#0-3) 

when `execution_flags.validate == false`, the function returns `Ok(())` immediately after `perform_pre_validation_stage`, **without ever calling `tx.validate_tx()`** (the `__validate__` entry point). `perform_pre_validation_stage` only checks nonce, fee bounds, and proof facts — it does not verify the signature: [5](#0-4) 

The `validate_by_mempool` call that precedes `skip_stateful_validations` also performs no signature check — `ValidationArgs` carries only address, nonce, hash, tip, and gas price: [6](#0-5) 

and `Mempool::validate_tx` only checks for duplicate hashes and fee escalation: [7](#0-6) 

### Impact Explanation

An attacker can submit an `InvokeV3` transaction with `nonce=1`, `sender_address=A`, and a completely invalid/random signature for any address A that has a `deploy_account` (or any other transaction) in the mempool. The gateway's stateful validator returns `Ok(account_nonce)` and the transaction is forwarded to the mempool and admitted. The concrete corrupted admission value is: **an invoke transaction bearing an invalid signature is accepted into the mempool as if it were valid**, matching the High impact scope "Mempool/gateway/RPC admission accepts invalid transactions before sequencing."

The invalid invoke will fail during block execution (the sequencer does run `__validate__` at that stage), but the admission decision itself is corrupted. Secondary effects include: mempool slot consumption, potential displacement of the legitimate user's nonce-1 invoke via fee escalation/replacement, and griefing of the deploy-account UX flow.

### Likelihood Explanation

The mempool is observable on the P2P network. Any peer receiving a `deploy_account` broadcast for address A has all the information needed to craft the attack. No privileged access is required. The only precondition is that address A has a transaction in the mempool or a recent committed block, which is a public, observable fact.

### Recommendation

The `skip_stateful_validations` check should verify that the **existing mempool entry for the address is specifically a `deploy_account` transaction** (not just any transaction), and should additionally perform a lightweight stateless signature check (e.g., ECDSA format validity) before skipping the full blockifier `__validate__` call. Alternatively, the skip should only be applied when the gateway itself submitted the `deploy_account` in the same request batch (i.e., tie the skip to a session/request context rather than a global mempool query).

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
