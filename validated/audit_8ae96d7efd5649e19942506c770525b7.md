### Title
Gateway Admits Unsigned Invoke Transactions via Overly Broad `skip_stateful_validations` Check — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`skip_stateful_validations` skips the `__validate__` entry-point call (i.e., signature verification) for any invoke transaction with `nonce=1` when the account's on-chain nonce is `0` and `account_tx_in_pool_or_recent_block` returns `true`. The pool check returns `true` for **any** transaction from that address in the pool — including a `deploy_account` transaction submitted by the legitimate owner. An unprivileged attacker who observes a victim's `deploy_account` transaction in the mempool can immediately submit a forged invoke transaction (nonce=1, arbitrary calldata, arbitrary signature) for the victim's address. The gateway skips `__validate__`, admits the forged transaction, and it enters the mempool — potentially displacing the victim's legitimate nonce=1 invoke transaction via fee escalation.

### Finding Description

`skip_stateful_validations` in `crates/apollo_gateway/src/stateful_transaction_validator.rs` (lines 429–461) implements the "deploy_account + invoke UX" shortcut:

                .await
                ...

    Ok(false)

The pool membership check `account_tx_in_pool_or_recent_block` returns `true` if the account has **any** transaction in the pool:

```rust
pub fn account_tx_in_pool_or_recent_block(&self, account_address: ContractAddress) -> bool {
    self.state.contains_account(account_address)
        || self.tx_pool.contains_account(account_address)
}
``` [4](#0-3) 

The code comment claims this is sufficient because the account "either has a deploy_account transaction or transactions with future nonces that passed validations." However, the check does not distinguish between a `deploy_account` transaction (which proves the legitimate owner submitted it) and any other transaction type. A `deploy_account` transaction for address A is publicly visible in the mempool and was submitted by the legitimate owner. An attacker observing it can immediately satisfy the `account_tx_in_pool_or_recent_block` condition for A without possessing A's private key.

**Attack path:**

1. Victim submits `deploy_account` for address A (nonce=0). It enters the mempool.
2. Attacker submits `invoke` with `sender_address=A`, `nonce=1`, arbitrary calldata, arbitrary/empty signature, fee higher than victim's pending invoke (if any).
3. Gateway stateless validation passes (valid address, valid resource bounds, no signature check).
4. Gateway stateful validation:
   - `get_nonce_from_state(A)` → `0` (A not yet deployed)
   - `validate_nonce`: `0 ≤ 1 ≤ max_allowed_nonce_gap` → passes
   - `validate_by_mempool`: no existing nonce=1 tx for A → passes (or fee-escalates victim's tx out)
   - `skip_stateful_validations`: `nonce==1`, `account_nonce==0`, `account_tx_in_pool_or_recent_block(A)==true` (deploy_account is in pool) → returns `true`
   - `run_validate_entry_point(skip_validate=true)`: `__validate__` not called → passes
5. Forged transaction admitted to mempool.
6. If attacker's fee > victim's invoke fee, victim's legitimate nonce=1 invoke

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
