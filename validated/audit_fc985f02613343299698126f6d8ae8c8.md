Now I have enough context to write up the finding. Let me verify one more detail about how `next_l2_gas_price` is used vs. `l2_gas_price` in the block context for execution.

### Title
Gateway Resource-Bounds Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` from Block Header — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads the **current block's** `l2_gas_price` from the latest committed block to check whether an incoming transaction's `max_price_per_unit` meets the admission threshold. The correct value is `next_l2_gas_price`, which is the EIP-1559-computed price that the sequencer will actually use for the **next** block. The code itself contains a TODO acknowledging this: `// TODO(Arni): getnext_l2_gas_price from the block header.` The mismatch causes the gateway to admit transactions that will fail at blockifier execution and to reject transactions that would succeed.

---

### Finding Description

**Root cause — wrong field read at admission time**

`validate_resource_bounds` calls `get_block_info()` and reads:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  lines 229-236
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
```

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` constructs a `BlockInfo` from the latest committed block header, mapping `block_header.l2_gas_price.price_in_fri` into `strk_gas_prices.l2_gas_price`:

```rust
// crates/apollo_gateway/src/gateway_fixed_block_state_reader.rs  lines 46-49
strk_gas_prices: GasPriceVector {
    l1_gas_price: block_header.l1_gas_price.price_in_fri.try_into()?,
    l1_data_gas_price: block_header.l1_data_gas_price.price_in_fri.try_into()?,
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
},
```

`BlockHeaderWithoutHash` carries **two distinct L2 gas price fields**:

```rust
// crates/starknet_api/src/block.rs  lines 237-239
pub l2_gas_price: GasPricePerToken,   // price used IN this block
pub l2_gas_consumed: GasAmount,
pub next_l2_gas_price: GasPrice,      // EIP-1559 price for the NEXT block
```

The consensus orchestrator stores `next_l2_gas_price` in the block header as the price it will use for the following block:

```rust
// crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs  line 405
next_l2_gas_price: self.l2_gas_price,
```

`BlockInfo` (the struct returned by `get_block_info()`) does not expose `next_l2_gas_price`, so the gateway validator never reads it. The admission check therefore compares the transaction's `max_price_per_unit` against the **current block's** price, not the price that will govern execution.

The same stale price is also used in `run_validate_entry_point` when constructing the `BlockContext` for the blockifier's `__validate__` call:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  lines 323-330
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
```

This means `check_fee_bounds` inside `perform_pre_validation_stage` also runs against the current block's price, not the next block's price.

---

### Impact Explanation

**Scenario A — price rising (high utilization)**

Block N has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P`.

1. User submits a transaction with `max_price_per_unit = X` where `P ≤ X < P'`.
2. Gateway admission threshold = `(min_gas_price_percentage / 100) * P`. With `min_gas_price_percentage = 100`, threshold = `P`. Since `X ≥ P`, the transaction passes.
3. The blockifier's `check_fee_bounds` at the gateway also uses `P`, so it passes.
4. Transaction enters the mempool.
5. Block N+1 is built with `l2_gas_price = P'`. The batcher's blockifier runs `check_fee_bounds` against `P'`. Since `X < P'`, the transaction fails with `MaxGasPriceTooLow`.

**Result**: Gateway admits a transaction that will fail at execution — invalid transaction accepted before sequencing.

**Scenario B — price falling (low utilization)**

Block N has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' < P`.

1. User submits a transaction with `max_price_per_unit = X` where `P' ≤ X < threshold_based_on_P`.
2. Gateway rejects with `GAS_PRICE_TOO_LOW`.
3. The transaction would have succeeded in block N+1 (where `l2_gas_price = P'`).

**Result**: Gateway rejects a valid transaction — valid transaction rejected before sequencing.

The EIP-1559 formula can change the price by up to ~0.3% per block (`MIN_GAS_PRICE_INCREASE_DENOMINATOR = 333`). Over multiple blocks of sustained high or low utilization, the divergence between `l2_gas_price` and `next_l2_gas_price` compounds, widening the window of incorrectly admitted or rejected transactions.

---

### Likelihood Explanation

This is a normal operating condition. Every block that is not exactly at the gas target produces a `next_l2_gas_price` that differs from `l2_gas_price`. The divergence is small per block but persistent during sustained load changes. Any user submitting a transaction whose `max_price_per_unit` falls in the gap between the two prices triggers the incorrect admission decision. No special privileges or adversarial setup are required — ordinary transaction submission is sufficient.

---

### Recommendation

1. Expose `next_l2_gas_price` through `GatewayFixedBlockStateReader`. Add a `get_next_l2_gas_price()` method or include it in the returned `BlockInfo`-equivalent struct.

2. In `validate_resource_bounds`, replace the read of `block_info.gas_prices.strk_gas_prices.l2_gas_price` with the `next_l2_gas_price` from the block header:

```rust
// Replace:
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// With:
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()
    .await?;
```

3. Apply the same fix to `run_validate_entry_point` so the `BlockContext` used for `__validate__` also carries `next_l2_gas_price` as its `l2_gas_price`.

### Proof of Concept

**Setup**: Block N is committed with `l2_gas_price_fri = 1_000_000` and `next_l2_gas_price = 1_003_000` (0.3% increase due to full block).

**Step 1**: Submit an `InvokeV3` transaction with:
```
resource_bounds.l2_gas.max_price_per_unit = 1_001_000
```

**Step 2**: Gateway `validate_resource_bounds` computes:
```
threshold = (100 / 100) * 1_000_000 = 1_000_000
1_001_000 >= 1_000_000  →  PASS
```
Transaction is admitted to the mempool.

**Step 3**: Batcher builds block N+1 with `l2_gas_price = 1_003_000` (taken from block N's `next_l2_gas_price`). Blockifier runs `check_fee_bounds`:
```
resource_bounds.l2_gas.max_price_per_unit (1_001_000) < actual_gas_price (1_003_000)
→  MaxGasPriceTooLow error
```

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
