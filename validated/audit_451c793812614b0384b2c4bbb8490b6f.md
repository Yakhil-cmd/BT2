### Title
Arithmetic Underflow in `find_sigma_edge_cases` Causes Unexpected Panic When Pool Has Never Received Rewards - (File: `src/pool/utils.cairo`)

### Summary
In `find_sigma_edge_cases`, edge case 4 performs `cumulative_rewards_trace_vec.at(cumulative_rewards_trace_len - 1)` without first checking that `cumulative_rewards_trace_len > 0`. When a pool member joins a pool whose cumulative rewards trace is still empty (`len == 0`), the stored `cumulative_rewards_trace_idx` is `1` (i.e., `len + 1`). If no rewards are ever distributed to that pool before the member attempts to claim or undelegate, edge case 4 is triggered and `0_u64 - 1` underflows, producing an unexpected arithmetic panic instead of the correct result of zero rewards.

### Finding Description
`find_sigma_edge_cases` in `src/pool/utils.cairo` dispatches on the relationship between `cumulative_rewards_trace_idx` (stored per pool-member checkpoint) and `cumulative_rewards_trace_len` (the current length of the pool's cumulative rewards trace vector). [1](#0-0) 

Edge case 4 is entered when `cumulative_rewards_trace_idx == cumulative_rewards_trace_len + 1`. The comment documents the intended scenario: the member joined when the trace had `len` entries, and no further rewards have been given since. The code then tries to read the last entry of the trace:

```cairo
let (epoch, sigma) = cumulative_rewards_trace_vec.at(cumulative_rewards_trace_len - 1);
```

When `cumulative_rewards_trace_len == 0` (the trace is still empty), this subtraction is `0_u64 - 1`, which underflows and panics in Cairo's checked arithmetic. The correct behaviour in this state is to return `Some(Zero::zero())` — the member has no accrued rewards — but instead the transaction reverts with an arithmetic underflow error.

The reachable path is:

1. A staker opens a delegation pool.
2. A pool member delegates before any epoch reward has been distributed to that pool (`len == 0`). The pool contract writes `cumulative_rewards_trace_idx = len + 1 = 1`.
3. No rewards are distributed to the pool (e.g., the staker has not yet attested, or the pool is brand-new).
4. The pool member calls `claim_rewards`, `undelegate_intent`, or any other pool function that internally invokes `find_sigma`.
5. `find_sigma_edge_cases` is called with `idx = 1`, `len = 0`. Edge cases 1–3 do not match; edge case 4 matches (`1 == 0 + 1`). The subtraction `0 - 1` underflows. [2](#0-1) 

### Impact Explanation
Any pool-contract function that settles rewards before acting (e.g., `undelegate_intent`, `switch_delegation_pool`) will revert with an arithmetic underflow panic rather than the expected domain error or a clean zero-reward result. If `undelegate_intent` is blocked, the pool member cannot initiate withdrawal of their delegated funds, constituting a **temporary freeze of funds**. The member's principal is locked in the pool until either rewards are distributed (advancing `len` past 0) or the contract is upgraded.

This matches the allowed impact: **Temporary freezing of funds (High)** and **Permanent/temporary freezing of unclaimed yield (High)**.

### Likelihood Explanation
The scenario is realistic and requires no privileged access:

- Any unprivileged delegator can trigger it by joining a newly deployed pool before the first reward epoch completes.
- Newly created pools (e.g., a staker who just called `set_open_for_delegation`) will have `len == 0` for at least one full epoch.
- The attacker does not need to be malicious; any ordinary pool member who joins early and then tries to claim or undelegate will hit this path.

### Recommendation
Add a guard at the start of edge case 4 to handle the empty-trace sub-case:

```cairo
if cumulative_rewards_trace_idx == cumulative_rewards_trace_len + 1 {
    // If the trace is empty, no rewards have ever been given; sigma is zero.
    if cumulative_rewards_trace_len == 0 {
        return Some(Zero::zero());
    }
    let (epoch, sigma) = cumulative_rewards_trace_vec.at(cumulative_rewards_trace_len - 1);
    assert!(epoch < target_epoch, "{}", InternalError::INVALID_EPOCH_IN_TRACE);
    return Some(sigma);
}
```

This mirrors the intent of edge case 1 (which returns zero when `idx == 0`) and avoids the underflow.

### Proof of Concept

1. Deploy staking + pool contracts.
2. Staker calls `set_open_for_delegation` → pool deployed, `cumulative_rewards_trace` length = 0.
3. Pool member calls `enter_delegation_pool` → pool contract writes `cumulative_rewards_trace_idx = 1` (= `len + 1 = 0 + 1`).
4. No epoch reward is distributed to the pool (staker has not attested / epoch has not advanced).
5. Pool member calls `claim_rewards` (or `undelegate_intent`).
6. Internally, `find_sigma_edge_cases` is invoked with `cumulative_rewards_trace_idx = 1`, `cumulative_rewards_trace_len = 0`.
7. Edge case 4 condition `1 == 0 + 1` is true.
8. `cumulative_rewards_trace_vec.at(0 - 1)` → `u64` underflow → transaction reverts with arithmetic panic instead of returning `Some(0)`. [3](#0-2)

pub(crate) fn find_sigma_edge_cases(
    cumulative_rewards_trace_vec: StorageBase<Trace>,
    cumulative_rewards_trace_idx: VecIndex,
    target_epoch: Epoch,
) -> Option<Amount> {
    // Edge case 1: Pool member enter delegation before any rewards given to the pool.
    if cumulative_rewards_trace_idx == 0 {
        return Some(Zero::zero());
    }

    let cumulative_rewards_trace_len = cumulative_rewards_trace_vec.length();

    // Edge case 2: `idx = len`.
    // In this version: `len + 1` was written, and rewards given to pool from that moment
    // only once.
    // In old version: `len` was written, and no rewards given to pool from that moment.
    if cumulative_rewards_trace_idx == cumulative_rewards_trace_len {
        // Two entries in the cumulative rewards trace are relevant (`idx - 1`, `idx - 2`).
        let (epoch, sigma) = cumulative_rewards_trace_vec.at(cumulative_rewards_trace_idx - 1);
        // In case `idx == 1`, `(epoch, sigma)` at `idx - 1` is `(0,0)` (the first trace
        // entry), so always return here.
        // This case only occurs in old-version checkpoints. In the current version, `idx =
        // 1` implies `len - 1` was written, so `len > idx` and we never reach this point.
        if epoch < target_epoch {
            return Some(sigma);
        }
        // Note: When handling a checkpoint from the old version, it never reaches here.
        assert!(cumulative_rewards_trace_idx > 1, "{}", InternalError::INVALID_REWARDS_TRACE_IDX);
        let (epoch, sigma) = cumulative_rewards_trace_vec.at(cumulative_rewards_trace_idx - 2);
        assert!(epoch < target_epoch, "{}", InternalError::INVALID_EPOCH_IN_TRACE);
        return Some(sigma);
    }

    // Edge case 3: `idx = 1`.
    // In this version: `len - 1` was written for the current checkpoint. (`len + 1` wasn't
    // written since `len >= 1`).
    // In old version: `len` was written, or `len - 1` was written for the current
    // checkpoint.
    // TODO: Use helper function that gets index and looks at two entries in the cumulative
    // rewards trace here and in edge case 2.
    if cumulative_rewards_trace_idx == 1 && cumulative_rewards_trace_len > 1 {
        // Two entries in the cumulative rewards trace are relevant (`idx`, `idx - 1`).
        let (epoch, sigma) = cumulative_rewards_trace_vec.at(cumulative_rewards_trace_idx);
        if epoch < target_epoch {
            return Some(sigma);
        }
        let (epoch, sigma) = cumulative_rewards_trace_vec.at(cumulative_rewards_trace_idx - 1);
        assert!(epoch < target_epoch, "{}", InternalError::INVALID_EPOCH_IN_TRACE);
        return Some(sigma);
    }

    // Edge case 4: `idx = len + 1`.
    // In this version: `len + 1` was written, and no rewards given to pool from that
    // moment.
    // In old version: never reached here (`len` or `len - 1` was written).
    if cumulative_rewards_trace_idx == cumulative_rewards_trace_len + 1 {
        // Only one entry in the cumulative rewards trace is relevant (`idx - 2`).
        let (epoch, sigma) = cumulative_rewards_trace_vec.at(cumulative_rewards_trace_len - 1);
        assert!(epoch < target_epoch, "{}", InternalError::INVALID_EPOCH_IN_TRACE);
        return Some(sigma);
    }

    None
}
