### Title
Missing Presign/Sign Participant-Set Binding Causes Signing Session Failure — (`src/frost/eddsa/sign.rs`)

### Summary

`sign_v2` accepts a `PresignOutput` whose `commitments_map` was produced for a different (larger) participant set without any validation. The `SigningPackage` passed to `aggregate()` therefore contains commitments for participants who never submitted signature shares, causing `aggregate()` to return an error and the entire signing session to fail.

### Finding Description

`presign` in `src/frost/mod.rs` collects one `SigningCommitments` per participant and stores all of them in `PresignOutput.commitments_map`. [1](#0-0) 

`sign_v2` in `src/frost/eddsa/sign.rs` delegates all input validation to `assert_sign_inputs`. [2](#0-1) 

`assert_sign_inputs` checks only: minimum participant count, no duplicates, self-inclusion, threshold bound, and coordinator inclusion. It performs **no check** that `presignature.commitments_map.keys()` is consistent with the `participants` slice. [3](#0-2) 

Inside `do_sign_coordinator_v2`, the `SigningPackage` is built directly from the presignature's `commitments_map` — whatever size it happens to be: [4](#0-3) 

Signature shares are then collected only from the (smaller) `participants` list: [5](#0-4) 

`aggregate()` is then called with a `signing_package` containing N_presign commitments but a `signature_shares` map containing only N_sign < N_presign entries. The frost-ed25519 `aggregate()` implementation requires a share for every identifier present in the signing package; missing entries cause it to return an error, which is propagated as `ProtocolError::AssertionFailed`: [6](#0-5) 

The same structural flaw exists in `do_sign_participant_v2`, which also builds its own `SigningPackage` from the raw presignature commitments map without any participant-set check: [7](#0-6) 

### Impact Explanation

Any caller who can invoke `sign_v2` (i.e., any participant acting as coordinator) can supply a `PresignOutput` from a session with a larger participant set. The signing session terminates with an error and produces no usable signature. This matches **High: Corruption of sign outputs so honest parties receive unusable cryptographic outputs**.

### Likelihood Explanation

The `PresignOutput` struct is `pub`, `Clone`, and `Serialize/Deserialize`. Nothing in the type system or runtime guards prevents reuse across sessions with different participant sets. A misconfigured or malicious coordinator can trigger this trivially. [8](#0-7) 

### Recommendation

In `sign_v2` (or inside `assert_sign_inputs` / `fut_wrapper_v2`), add an explicit check that the set of identifiers in `presignature.commitments_map` exactly equals the set derived from the `participants` slice:

```rust
let expected_ids: BTreeSet<_> = participants
    .iter()
    .map(|p| p.to_identifier())
    .collect::<Result<_, _>>()?;
let actual_ids: BTreeSet<_> = presignature.commitments_map.keys().cloned().collect();
if expected_ids != actual_ids {
    return Err(InitializationError::BadParameters(
        "presignature participant set does not match sign participant set".into(),
    ));
}
```

    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,

    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)

async fn do_sign_coordinator_v2(
    mut chan: SharedChannel,
    participants: ParticipantList,
    threshold: ReconstructionLowerBound,
    me: Participant,
    keygen_output: KeygenOutput,
    presignature: PresignOutput,
    message: Vec<u8>,
) -> Result<SignatureOption, ProtocolError> {
    // --- Round 1
    let signing_package =
        frost_ed25519::SigningPackage::new(presignature.commitments_map, message.as_slice());

    let mut signature_shares: BTreeMap<frost_ed25519::Identifier, round2::SignatureShare> =
        BTreeMap::new();

    let vk_package = keygen_output.public_key;

    let key_package =
        construct_key_package(threshold, me, keygen_output.private_share, &vk_package)?;

    let key_package = Zeroizing::new(key_package);
    let signature_share = round2::sign(&signing_package, &presignature.nonces, &key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;
    signature_shares.insert(me.to_identifier()?, signature_share);

    let sign_waitpoint = chan.next_waitpoint();
    for (from, signature_share) in
        recv_from_others(&chan, sign_waitpoint, &participants, me).await?
    {
        signature_shares.insert(from.to_identifier()?, signature_share);
    }

    // --- Signature aggregation.
    // * Converted collected signature shares into the signature.
    // * Signature is verified internally during `aggregate()` call.
    // We supply empty map as `verifying_shares` because we have disabled "cheater-detection" feature flag.
    // Feature "cheater-detection" only points to a malicious participant, if there's such.
    // It doesn't bring any additional guarantees.
    let public_key_package = PublicKeyPackage::new(BTreeMap::new(), vk_package);
    let signature = aggregate(&signing_package, &signature_shares, &public_key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;

    Ok(Some(signature))
