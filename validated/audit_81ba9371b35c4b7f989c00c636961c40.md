### Title
Unprivileged Caller Can Permanently Suppress All Staker Block Rewards via `update_rewards` - (File: `src/staking/staking.cairo`)

---

### Summary
The `update_rewards` function in the Staking contract is publicly callable with no access control. Any unprivileged caller can invoke it with `disable_rewards: true`, which marks the current block as processed in the global `last_reward_block` storage slot while skipping reward distribution entirely. Because `last_reward_block` is a single global value, this permanently prevents any staker from receiving consensus rewards for that block.

---

### Finding Description
`StakingRewardsManagerImpl::update_rewards` (lines 1449–1500 of `src/staking/staking.cairo`) is exposed as a public ABI entry point with no caller restriction:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // ← last_reward_block is written BEFORE the disable_rewards branch
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;   // ← exits without distributing any rewards
        }
        ...
    }
}
``` [1](#0-0) 

The critical sequence is:

1. `last_reward_block` is unconditionally overwritten with the current block number.
2. If `disable_rewards == true`, the function returns immediately — no rewards are computed or transferred.
3. The guard `current_block_number > self.last_reward_block.read()` then blocks every subsequent call for the same block with `REWARDS_ALREADY_UPDATED`.

Because `last_reward_block` is a **single global slot** (not per-staker), a single call with `disable_rewards: true` at block N silences reward distribution for **all stakers** at block N.

Contrast this with the pre-consensus path, `update_rewards_from_attestation_contract`, which correctly enforces `assert_caller_is_attestation_contract()`:

```cairo
fn update_rewards_from_attestation_contract(
    ref self: ContractState, staker_address: ContractAddress,
) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
    self.assert_caller_is_attestation_contract();   // ← access control present
``` [2](#0-1) 

`update_rewards` has no equivalent guard, making `disable_rewards` an attacker-controlled kill switch for consensus-era block rewards.

---

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, true)` once per block permanently erases the block reward for every staker in the protocol for that block. Repeated across every block, this completely halts consensus reward accrual for all stakers and delegators without any privileged access. The lost yield is irrecoverable because `last_reward_block` cannot be rolled back.

---

### Likelihood Explanation
**High.** The function is a standard public ABI call. No special role, token balance, or prior state is required — only a valid active staker address (readable from on-chain events or `get_stakers()`). The attacker's only cost is gas per block. A griefing bot can sustain the attack indefinitely.

---

### Recommendation
Add the same caller restriction used by `update_rewards_from_attestation_contract`. Restrict `update_rewards` to the attestation contract:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_attestation_contract(); // add this guard
    ...
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface and let the attestation contract choose which internal path to invoke.

---

### Proof of Concept

1. Attacker reads any active staker address `S` from on-chain events or `get_stakers()`.
2. At block N, attacker calls `staking.update_rewards(S, disable_rewards: true)`.
3. `last_reward_block` is written to N; the function returns without distributing rewards.
4. The legitimate attestation contract call at block N hits `REWARDS_ALREADY_UPDATED` and reverts.
5. All stakers lose their block-N consensus rewards permanently.
6. Repeating step 2 every block freezes all future consensus yield. [3](#0-2)
