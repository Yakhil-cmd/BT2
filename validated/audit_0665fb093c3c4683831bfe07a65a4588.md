### Title
Missing Access Control on `update_rewards` Enables Griefing to Permanently Deny Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the staking contract is documented as callable only by the Starkware sequencer, but the implementation contains **no access control check**. Any unprivileged address can call it with `disable_rewards: true` before the sequencer does, setting the global `last_reward_block` to the current block without distributing rewards. The sequencer's subsequent legitimate call then reverts with `REWARDS_ALREADY_UPDATED`, permanently denying all stakers their block rewards for that block.

---

### Finding Description

The `update_rewards` function in `StakingRewardsManagerImpl` is the consensus-phase reward distribution entry point. Per the spec, it is restricted to "Only starkware sequencer": [1](#0-0) 

However, the implementation contains no such check. The only guard is a block-level deduplication check against the global `last_reward_block`: [2](#0-1) 

After passing that check, the function unconditionally writes the current block number to `last_reward_block`: [3](#0-2) 

`last_reward_block` is a **single global field** shared across all stakers: [4](#0-3) 

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in the same block as the sequencer's intended call. This:
1. Passes all checks (staker exists, block is new).
2. Writes `last_reward_block = current_block`.
3. Returns early without distributing any rewards (because `disable_rewards == true`).

When the sequencer then calls `update_rewards(staker_address, disable_rewards: false)`, the assertion `current_block_number > last_reward_block` fails and the call reverts with `REWARDS_ALREADY_UPDATED`. The block rewards for every staker in that block are permanently lost.

The `general_prerequisites()` call only checks the pause flag — it performs no caller identity check: [5](#0-4) 

Compare with `update_rewards_from_attestation_contract`, which correctly enforces its caller restriction: [6](#0-5) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

When the griefer wins the race, the block rewards for that block are permanently unrecoverable. `last_reward_block` is advanced to the current block, so the sequencer cannot retry in the same block. The rewards that would have been minted and credited to stakers and pool members for that block are simply never distributed. Because `last_reward_block` is global, a single griefing call denies rewards to **all** stakers simultaneously for that block.

---

### Likelihood Explanation

**High.** Any address can call `update_rewards` at any time. The attack costs only a single transaction (cheap on Starknet). The attacker needs only to submit the call in the same block as the sequencer's reward distribution transaction. This can be repeated every block, continuously denying all stakers their consensus-phase block rewards indefinitely.

---

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, analogous to the attestation contract check already present in `update_rewards_from_attestation_contract`:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // ADD THIS
    ...
}
```

Alternatively, restrict the `IStakingRewardsManager` interface so that `update_rewards` is only callable from a whitelisted sequencer address stored in contract storage, following the same pattern used for the attestation contract address.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. Staker `S` has been staking for `K` epochs and is eligible for block rewards.
3. In block `N`, the sequencer prepares to call `update_rewards(S, disable_rewards: false)`.
4. Attacker `A` (any address) submits `update_rewards(S, disable_rewards: true)` in the same block `N` with higher priority.
5. Attacker's call succeeds: `last_reward_block` is set to `N`, no rewards distributed.
6. Sequencer's call reverts: `current_block_number (N) > last_reward_block (N)` is `false` → `REWARDS_ALREADY_UPDATED`.
7. Staker `S` (and all other stakers) receive zero rewards for block `N`.
8. Attacker repeats every block, permanently denying all stakers their consensus rewards. [7](#0-6)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
