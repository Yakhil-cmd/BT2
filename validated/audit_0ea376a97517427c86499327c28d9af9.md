### Title
`rerandomize_presign()` Does Not Consume `PresignOutput`, Enabling Presignature Reuse and Private Key Extraction — (`src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both `RerandomizedPresignOutput::rerandomize_presign()` implementations (OT-based and Robust ECDSA) accept `presignature: &PresignOutput` by immutable reference instead of consuming it by value. This is the direct analog of the external report's bug: just as `earlyExecute()` calls `_executeOperations()` (which does not set `executed = true`) instead of `execute()` (which does), `rerandomize_presign()` borrows the presignature without invalidating it, leaving the one-time-use invariant unenforced at the API level. A malicious coordinator can rerandomize the same `PresignOutput` twice with different public arguments, run two signing sessions, and recover the private key from the resulting nonce-related signatures.

---

### Finding Description

The library's own documentation and README state unambiguously that each presignature is one-time use:

> "It's **critical** that the output is then destroyed, so that no other group of parties attempts to re-use that output for another phase."
> "Each presignature is consumed **exactly once** (one-time use)."

Despite this, both `rerandomize_presign()` functions take the presignature by shared reference:

**OT-based ECDSA** (`src/ecdsa/ot_based_ecdsa/mod.rs`, lines 66–96):
```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // ← borrowed, not consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> { ... }
```

**Robust ECDSA** (`src/ecdsa/robust_ecdsa/mod.rs`, lines 54–86):
```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // ← borrowed, not consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> { ... }
```

`PresignOutput` also derives `Clone` in both schemes, making it trivially copyable. The downstream `sign()` functions correctly consume `RerandomizedPresignOutput` by value (OT-based: `sign.rs` line 28; Robust: `sign.rs` line 39), but this protection is rendered meaningless because the upstream `PresignOutput` is never invalidated.

The rerandomization factor `delta` is derived entirely from public inputs (`pk`, `tweak`, `msg_hash`, `big_r`, `participants`, `entropy`). If a coordinator supplies two different `RerandomizationArguments` (e.g., different `msg_hash` or `entropy`) for the same presignature, each participant will call `rerandomize_presign()` twice on the same `PresignOutput`, producing two `RerandomizedPresignOutput` values with effective nonces:

```
k_eff1 = k · δ₁⁻¹
k_eff2 = k · δ₂⁻¹
```

where `δ₁` and `δ₂` are both public. The ratio `k_eff1 / k_eff2 = δ₂ / δ₁` is known to the attacker. Two ECDSA signatures with multiplicatively related nonces allow full private key recovery via standard linear algebra — a fact the library's own security documentation explicitly acknowledges:

> "the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks."

---

### Impact Explanation

A malicious coordinator who controls `RerandomizationArguments` can trigger two signing sessions from the same presignature. After collecting both completed signatures, the coordinator solves for the private key `x` using the known nonce relationship. This constitutes **full extraction of the aggregate secret key material**, matching the Critical impact tier: *"Extraction, reconstruction, or disclosure of private signing shares, aggregate secret material, presign secrets, nonce material, or confidential derived secrets."*

---

### Likelihood Explanation

The coordinator role is a normal participant in both signing protocols and requires no privileged access beyond participation in the signing phase. The attack requires only that the coordinator issue two signing requests referencing the same presignature with different public parameters — a straightforward API-level action. The library provides no runtime guard, no consumed-flag, and no type-system enforcement to prevent this. The `Clone` derivation on `PresignOutput` further lowers the barrier.

---

### Recommendation

Change both `rerandomize_presign()` signatures to consume `PresignOutput` by value, leveraging Rust's ownership system to enforce the one-time-use invariant at compile time:

```rust
// OT-based: src/ecdsa/ot_based_ecdsa/mod.rs
pub fn rerandomize_presign(
-   presignature: &PresignOutput,
+   presignature: PresignOutput,
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>

// Robust: src/ecdsa/robust_ecdsa/mod.rs
pub fn rerandomize_presign(
-   presignature: &PresignOutput,
+   presignature: PresignOutput,
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

Additionally, consider removing the `Clone` derive from both `PresignOutput` types to prevent callers from cloning the value before passing it, which would otherwise circumvent the ownership-based protection.

### Proof of Concept

1. Complete presigning to obtain `presig: PresignOutput` for each participant.
2. Coordinator constructs `args1` (with `msg_hash1`, `entropy1`) and `args2` (with `msg_hash2`, `entropy2`).
3. Each participant calls:
   ```rust
   let rerand1 = RerandomizedPresignOutput::rerandomize_presign(&presig, &args1)?;
   let rerand2 = RerandomizedPresignOutput::rerandomize_presign(&presig, &args2)?;
   ```
   Both calls succeed because `presig` is borrowed, not consumed.
4. Coordinator runs signing session 1 with `rerand1` → signature `(r1, s1)`.
5. Coordinator runs signing session 2 with `rerand2` → signature `(r2, s2)`.
6. Coordinator computes `δ₁` and `δ₂` (both public via `derive_randomness()`), then solves:
   ```
   s1 = (h1 + r1·x) · δ1 / k
   s2 = (h2 + r2·x) · δ2 / k
   ```

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
