### Title
Unprivileged `MsgConvertVouchers` Front-Runs Admin `MsgUpdateTokenMapping`, Permanently Blocking External Contract Registration - (`x/cronos/keeper/keeper.go`, `x/cronos/keeper/evm.go`)

---

### Summary

When `EnableAutoDeployment` is `true`, any unprivileged user who holds IBC/gravity tokens can call `MsgConvertVouchers` to trigger auto-deployment of a module CRC21 contract and permanently bind a denom to it via `SetAutoContractForDenom`. Because `SetExternalContractForDenom` enforces a create-only invariant (`ErrDenomAlreadyMapped`), the admin's subsequent `MsgUpdateTokenMapping` for the same denom is permanently rejected. There is no `DeleteAutoContractForDenom` path, so the corruption is irreversible without a chain upgrade.

---

### Finding Description

**Step 1 – Unprivileged entry point.**
`MsgConvertVouchers` has no permission check:

```go
// x/cronos/keeper/msg_server.go:27-44

    ctx := sdk.UnwrapSDKContext(goCtx)
    err := k.ConvertVouchersToEvmCoins(ctx, msg.Address, msg.Coins)
    ...
}
``` [1](#0-0) 

**Step 2 – Auto-deployment path.**
`ConvertVouchersToEvmCoins` calls `ConvertCoinFromNativeToCRC21` with `params.EnableAutoDeployment`. When no mapping exists and auto-deploy is on, it deploys a module CRC21 and calls `SetAutoContractForDenom`:

```go

    }
    contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
    ...
    if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {

``` [2](#0-1) 

**Step 3 – Create-only guard blocks the admin.**
`SetExternalContractForDenom` (called by the admin's `RegisterOrUpdateTokenMapping`) calls `ensureDenomNotMapped`, which checks both external and auto mappings via `GetContractByDenom`:

```go
// x/cronos/keeper/keeper.go:172-181

    if contract, found := k.GetContractByDenom(ctx, denom); found {
        return errors.Wrapf(types.ErrDenomAlreadyMapped, ...)
    }
    return nil
}
``` [3](#0-2) 

`GetContractByDenom` returns the auto-contract if no external contract exists: [4](#0-3) 

**Step 4 – No recovery path.**
There is no `DeleteAutoContractForDenom`. `DeleteExternalContractForDenom` only removes external mappings. The auto-mapping is permanent. [5](#0-4) 

---

### Impact Explanation

The denom is permanently bound to the auto-deployed module CRC21 contract instead of the admin's intended external contract. All future `ConvertVouchers` and IBC-middleware auto-conversion flows for that denom route through the wrong contract. The admin's `MsgUpdateTokenMapping` is permanently rejected with `ErrDenomAlreadyMapped`. This is:

- **High: Corruption of token mappings / denom-contract binding with direct security impact** — the wrong contract handles all mint/burn/transfer operations for the affected denom.
- **High: Bypass of Cronos admin authority** — the admin's privileged mapping operation is permanently blocked by an unprivileged transaction.

---

### Likelihood Explanation

- Requires `EnableAutoDeployment = true` (a module parameter, can be set by governance).
- Attacker only needs to hold a nonzero amount of the target IBC/gravity token — trivially obtainable via a standard IBC transfer.
- The attack is a single `MsgConvertVouchers` transaction, costing only gas.
- The admin's intent is observable from the mempool (pending `MsgUpdateTokenMapping`), enabling precise front-running. Even without mempool observation, any user who converts vouchers before the admin registers the external contract achieves the same permanent block.

---

### Recommendation

1. **In `SetExternalContractForDenom`**: if an auto-mapping already exists for the denom and no external mapping exists, allow the admin to promote it to an external mapping (i.e., move the auto-contract entry to the external slot) rather than rejecting with `ErrDenomAlreadyMapped`.
2. **Alternatively**, add a privileged `DeleteAutoContractForDenom` callable only by the admin, so the admin can clear a stale auto-mapping before registering the intended external contract.
3. **Longer-term**: consider whether `MsgConvertVouchers` should be restricted to only convert denoms that already have an existing mapping, deferring auto-deployment exclusively to the IBC middleware path (which is not user-triggered).

---

### Proof of Concept

```
1. EnableAutoDeployment = true (governance param)
2. Admin broadcasts MsgUpdateTokenMapping{denom: "ibc/XXXX", contract: "0xExternalCRC21"}
3. Attacker observes pending tx in mempool; attacker holds 1 ibc/XXXX token
4. Attacker broadcasts MsgConvertVouchers{address: attacker, coins: [{denom:"ibc/XXXX", amount:1}]}
   with higher gas price → lands first in the block
   → ConvertCoinFromNativeToCRC21 → DeployModuleCRC21 → SetAutoContractForDenom("ibc/XXXX", 0xAutoContract)
5. Admin's MsgUpdateTokenMapping lands next:
   → RegisterOrUpdateTokenMapping → SetExternalContractForDenom
   → ensureDenomNotMapped → GetContractByDenom finds 0xAutoContract
   → returns ErrDenomAlreadyMapped → admin tx fails
6. "ibc/XXXX" is now permanently mapped to 0xAutoContract (module CRC21),
   not the admin's intended 0xExternalCRC21.
   No DeleteAutoContractForDenom exists; state is irreversible.
```

// DeleteExternalContractForDenom delete the external contract mapping for native denom,
// returns false if mapping not exists.
func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
	store := ctx.KVStore(k.storeKey)
	contract, found := k.getExternalContractByDenom(ctx, denom)
	if !found {
		return false
	}
	store.Delete(types.DenomToExternalContractKey(denom))
	deleteReverseIfOwned(store, contract, denom)
	if auto, found := k.getAutoContractByDenom(ctx, denom); found {
		bz := store.Get(types.ContractToDenomKey(auto.Bytes()))
		if len(bz) == 0 {
			store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
		} else if existingDenom := string(bz); existingDenom != denom {
			if k.contractOwnedByDenom(ctx, existingDenom, auto) {
				// auto address is already owned by another denom; drop local auto mapping
				store.Delete(types.DenomToAutoContractKey(denom))
			} else {
				// stale reverse entry
				store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
			}
		}
	}
	return true
}
