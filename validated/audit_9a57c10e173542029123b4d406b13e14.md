### Title
`TurnBridge` Message Handler is a No-Op, Permanently Disabling the Gravity Bridge Circuit Breaker and Bypassing Permission Enforcement - (File: `x/cronos/keeper/msg_server.go`)

### Summary

The `TurnBridge` message handler, designed as an emergency circuit breaker for the Gravity Bridge, is implemented as a stub that returns `(nil, nil)` — performing no permission check and no state change. This is structurally identical to the H-02 class: a security control that appears to exist but is rendered non-functional by a missing/default implementation. Any unprivileged user can call `MsgTurnBridge` and receive a success response, while the bridge itself can never be disabled by any actor.

### Finding Description

`ADR-009` explicitly designates `MsgTurnBridge` as one of two restricted messages requiring the `CanTurnBridge` permission bit. The companion message `UpdateTokenMapping` correctly enforces this:

```go
// x/cronos/keeper/msg_server.go:72-75
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
```

The `TurnBridge` handler, however, is:

```go
// x/cronos/keeper/msg_server.go:84-87
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

No permission check. No state mutation. The handler is fully registered in the proto service, has a working CLI command (`CmdTurnBridge`), and `ValidateBasic` accepts any valid sender address — so the message is reachable by any unprivileged user and will succeed silently.

The `CanTurnBridge` permission constant is defined and the `HasPermission` infrastructure exists:

```go
// x/cronos/keeper/permissions.go:13-17
const (
    CanChangeTokenMapping uint64 = 1 << iota // 1
    CanTurnBridge                             // 2
    All = CanChangeTokenMapping | CanTurnBridge // 3
)
```

But it is never consulted in `TurnBridge`. The integration test `test_gravity_turn_bridge` expects `turn_bridge("false", from_="community")` to fail and `turn_bridge("false", from_="validator")` to succeed and actually halt the bridge — neither behavior is implemented.

### Impact Explanation

**High — Bypass of bridge authorization check + permanent inability to stop the bridge.**

1. **Permission bypass**: Any unprivileged address can submit `MsgTurnBridge` and receive a success response (`code == 0`), bypassing the `CanTurnBridge` authorization gate that ADR-009 mandates.
2. **Emergency circuit breaker permanently non-functional**: The Gravity Bridge cannot be halted by any actor — admin, validator, or governance — because the handler never writes bridge state. In the event of a live bridge exploit (e.g., unauthorized minting of gravity-wrapped tokens), the intended emergency stop mechanism is silently inoperative, allowing the exploit to continue unchecked.

### Likelihood Explanation

Certainty: the handler is reachable by any user with a valid Cosmos address and a small gas fee. The proto endpoint `/cronos.Msg/TurnBridge` is live. No preconditions beyond a funded account are required.

### Recommendation

Implement `TurnBridge` with:
1. A permission check mirroring `UpdateTokenMapping`: `if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) { return nil, ErrUnauthorized }`.
2. Actual bridge state mutation (e.g., setting a `BridgeEnabled` flag in the cronos KV store and having the gravity end-blocker/hook respect it).

### Proof of Concept

1. Attacker holds any funded Cronos address `A` (no special permissions).
2. Attacker submits `MsgTurnBridge{Sender: A, Enable: false}`.
3. `ValidateBasic` passes (valid bech32 address).
4. Handler executes: `return nil, nil` — tx succeeds, `code == 0`.
5. No permission check was performed; `A` is not the admin and holds no `CanTurnBridge` bit.
6. Bridge state is unchanged; the bridge remains active regardless of the `Enable` field.
7. Simultaneously, a legitimate admin submitting `MsgTurnBridge{Sender: admin, Enable: false}` also receives `code == 0` but the bridge is still not stopped — the circuit breaker is permanently broken for all callers. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

```go
// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
