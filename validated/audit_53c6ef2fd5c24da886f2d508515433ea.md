### Title
Missing Caller Validation in `update_rewards` Allows Any Address to Permanently Freeze All Staker Consensus Rewards — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract enforces a global per-block reward slot via `last_reward_block`, but performs **no caller validation**. The specification explicitly restricts this function to "Only starkware sequencer," yet any unprivileged address can invoke it. An attacker can call `update_rewards` with `disable_rewards: true` at the start of every block, consuming the reward slot without distributing rewards, permanently freezing all stakers' consensus yield.

---

### Finding Description

`update_rewards` uses a single global `last_reward_block` storage variable to gate reward distribution to once per block: [1](#0-0) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
```

After passing the staker validity checks, the function unconditionally writes the current block to `last_reward_block` and then branches on `disable_rewards`: [2](#0-1) 

```cairo
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

There is **no check** that `get_caller_address()` equals the authorized sequencer. The specification is unambiguous: [3](#0-2) 

> **access control**: Only starkware sequencer.

Because `last_reward_block` is a single global (not per-staker), a single call per block with any valid `staker_address` and `disable_rewards: true` consumes the slot for **all** stakers in that block. The legitimate sequencer call then reverts with `REWARDS_ALREADY_UPDATED`.

The vulnerability class is identical to the reference report: **missing enforcement of who is permitted to invoke a state-changing function**, allowing an unprivileged caller to corrupt the protocol's accounting without holding any privileged role.

---

### Impact Explanation

- **Category**: Permanent freezing of unclaimed yield (consensus rewards).
- An attacker calling `update_rewards(staker_address: any_active_staker, disable_rewards: true)` once per block prevents every staker from ever accumulating consensus-era block rewards.
- `unclaimed_rewards_own` in every staker's record remains at zero indefinitely; `claim_rewards` returns nothing.
- The attack is permanent and affects the entire protocol, not a single staker.

---

### Likelihood Explanation

- **High.** No special role, token balance, or prior interaction is required.
- Valid staker addresses are publicly enumerable from the `stakers` storage vector.
- The cost is one cheap transaction per block; the attacker gains nothing but the damage to all stakers is total.
- The attack is front-runnable: the attacker simply submits the call before the sequencer in every block.

---

### Recommendation

Add an explicit caller check at the top of `update_rewards`, analogous to the pattern already used in `update_unclaimed_rewards_from_staking_contract` and `claim_rewards` in `RewardSupplier`: [4](#0-3) 

```cairo
// Example pattern already used in the codebase:
assert!(
    get_caller_address() == staking_contract,
    "{}",
    GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
);
```

For `update_rewards`, store the authorized sequencer address in storage during construction and assert it at the function entry, or use the existing roles component to gate access to a dedicated `SEQUENCER_ROLE`.

---

### Proof of Concept

```
1. Deploy the full staking system (Staking + RewardSupplier + MintingCurve).
2. Register a staker and wait K epochs for consensus rewards to become active
   (consensus_rewards_first_epoch is set and passed).
3. At the start of block N, from any EOA (attacker), call:
     staking.update_rewards(staker_address: <any_active_staker>, disable_rewards: true)
   - Passes general_prerequisites() (contract not paused).
   - Passes REWARDS_ALREADY_UPDATED check (first call this block).
   - Passes INVALID_STAKER checks (valid, active staker with non-zero balance).
   - Writes last_reward_block = N.
   - Returns early due to disable_rewards: true — no rewards distributed.
4. The Starkware sequencer now calls:
     staking.update_rewards(staker_address: <legitimate_staker>, disable_rewards: false)
   - Reverts: current_block_number (N) is NOT > last_reward_block (N).
   - Error: REWARDS_ALREADY_UPDATED.
5. Repeat step 3 every block. All stakers receive zero consensus rewards indefinitely.
6. Confirm: staking.staker_info_v1(staker_address).unclaimed_rewards_own == 0 after
   many blocks despite consensus rewards being active.
```
