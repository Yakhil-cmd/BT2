### Title
`validateIsUint64` Accepts Zero for `MaxCallbackGas`, Enabling Governance Proposal to Permanently Block All IBC Ack/Timeout Callbacks — (`x/cronos/types/params.go`, `x/cronos/keeper/keeper.go`)

---

### Summary

`validateIsUint64` only checks the Go type of `MaxCallbackGas`, not that it is `> 0`. A governance proposal setting `MaxCallbackGas=0` passes `SetParams` without error. Subsequently, every IBC acknowledgement and timeout callback for ICA contracts calls `CallEVM` with `gasLimit=0`, causing immediate out-of-gas failure and permanently blocking IBC callback processing for all registered contracts.

---

### Finding Description

**Root cause — `validateIsUint64` does not enforce a minimum value:** [1](#0-0) 

The function only asserts the Go type is `uint64`. The value `0` is a valid `uint64`, so `MaxCallbackGas=0` passes both the per-field validator registered in `ParamSetPairs` and the `Validate()` call in `SetParams`. [2](#0-1) 

**Governance entry point — `UpdateParams` requires only governance authority:** [3](#0-2) 

Any governance proposal that passes with majority vote can call this path. The scope rules explicitly list "proposal path" as a valid entry point.

**Propagation — `onPacketResult` uses `MaxCallbackGas` directly as the EVM gas limit:** [4](#0-3) 

With `gasLimit=0`, `CallEVM` constructs a `core.Message` with `GasLimit: 0`: [5](#0-4) 

`ApplyMessage` with `GasLimit=0` immediately exhausts gas before any EVM opcode executes, returning an out-of-gas error. `onPacketResult` propagates this error upward, causing `IBCOnAcknowledgementPacketCallback` and `IBCOnTimeoutPacketCallback` to both fail: [6](#0-5) 

---

### Impact Explanation

Once `MaxCallbackGas=0` is set, every IBC ack and timeout packet delivered to any ICA contract that registered a callback fails at the `onPacketResult` step. Relayers cannot successfully deliver these packets. IBC channels for ICA contracts become stuck. The condition persists until a second governance proposal restores a non-zero value. This matches the High impact criterion: **"Permanent or long-lived inability for honest users or validators to process valid transactions... IBC transfers."**

---

### Likelihood Explanation

The entry point is a governance proposal, which requires a majority of staked tokens. This is a meaningful barrier, but the proposal path is explicitly in scope. The bug is a single missing `> 0` check, and the effect is immediate and total once the proposal passes.

---

### Recommendation

Replace `validateIsUint64` for `MaxCallbackGas` with a dedicated validator that enforces a minimum value of 1:

```go
func validateMaxCallbackGas(i interface{}) error {
    v, ok := i.(uint64)
    if !ok {
        return fmt.Errorf("invalid parameter type: %T", i)
    }
    if v == 0 {
        return fmt.Errorf("MaxCallbackGas must be greater than 0")
    }
    return nil
}
```

Apply this validator in both `ParamSetPairs` and `Validate()`: [2](#0-1) [7](#0-6)
