### Title
EVM `Heartbeat.heartbeat()` Advances Block Time During Pause, Enabling Interest Accrual While User Operations Are Blocked - (File: `fvm/evm/stdlib/contract.cdc`)

---

### Summary

The `EVM.Heartbeat.heartbeat()` function does not check `EVM.isPaused()`, so EVM block height and timestamp continue to advance every Flow block even when EVM operations are paused. All user-facing EVM operations are blocked during pause, but EVM-based lending protocols (e.g., Aave, Compound deployed on Flow EVM) continue to accrue interest based on elapsed `block.number`/`block.timestamp`. When EVM unpauses, users face retroactive interest for the entire pause period and may be liquidated, losing their collateral.

---

### Finding Description

The `EVM.isPaused()` mechanism guards every user-facing EVM operation. When `evmOperationsPaused` is `true`, the following are all blocked with `"EVM operations are temporarily paused"`:

- `EVMAddress.deposit` [1](#0-0) 
- `CadenceOwnedAccount.withdraw` [2](#0-1) 
- `CadenceOwnedAccount.call`, `deploy`, `callWithSigAndArgs` [3](#0-2) 
- `CadenceOwnedAccount.depositNFT`, `withdrawNFT`, `depositTokens`, `withdrawTokens` [4](#0-3) 
- `EVM.run`, `EVM.batchRun`, `EVM.createCadenceOwnedAccount` [5](#0-4) 

However, the `Heartbeat` resource's `heartbeat()` function has **no `isPaused()` check**:

```cadence
access(all) resource Heartbeat {
    access(all)
    fun heartbeat() {
        InternalEVM.commitBlockProposal()   // ← no isPaused() guard
    }
}
``` [6](#0-5) 

The system chunk transaction — executed unconditionally every Flow block — calls this heartbeat:

```cadence
let evmHeartbeat = serviceAccount.storage
    .borrow<&EVM.Heartbeat>(from: /storage/EVMHeartbeat)
evmHeartbeat?.heartbeat()
``` [7](#0-6) 

`commitBlockProposal()` advances the EVM block height and sets the EVM block timestamp to the current Flow block time: [8](#0-7) 

`isPaused()` reads from `/storage/evmOperationsPaused` and is intentionally designed to put EVM into a "read-only mode where all EVM state is available for reading, but no state updates are executed": [9](#0-8) 

The design intent is clearly to freeze EVM state during pause. But because `Heartbeat.heartbeat()` is exempt from this check, EVM block height and timestamp keep advancing — violating the stated invariant.

---

### Impact Explanation

EVM lending protocols (Aave, Compound, etc.) deployed on Flow EVM calculate interest as a function of elapsed `block.number` or `block.timestamp` since the last accrual checkpoint. Interest is not stored continuously; it is computed lazily on the next user interaction. During a pause:

1. EVM blocks continue to be committed → `block.number` and `block.timestamp` advance.
2. All user operations are blocked → users cannot call `repay()` or add collateral.
3. When EVM unpauses, the first interaction with the lending protocol computes interest for the **entire pause duration**, including the period during which the user was locked out.
4. If the accrued interest pushes the user's health factor below the liquidation threshold, their collateral is seized.

This is a direct cross-VM asset loss: FLOW tokens or ERC-20 collateral locked in EVM lending protocols can be liquidated due to interest accrued during a period when the user had no ability to act.

---

### Likelihood Explanation

- EVM pauses are a documented, expected governance mechanism ("maintenance or any situation that requires special governance handling").
- Flow EVM is EVM-compatible; deploying Aave/Compound forks is straightforward and already occurring on similar chains.
- A pause of even a few hours at a high interest rate (e.g., during a liquidity crisis, which is precisely when pauses are most likely) can push leveraged positions into insolvency.
- No attacker action is required beyond having an open leveraged position before the pause.

---

### Recommendation

Add an `isPaused()` guard to `Heartbeat.heartbeat()` so that EVM block production is frozen when EVM operations are paused:

```cadence
access(all)
fun heartbeat() {
    if !EVM.isPaused() {
        InternalEVM.commitBlockProposal()
    }
}
```

This ensures that EVM block height and timestamp do not advance during a pause, preventing time-based interest accrual in EVM contracts from running against users who are locked out from taking protective actions. This mirrors the recommendation in the original report: the "accrual" mechanism (`commitBlockProposal`) should not run while the protocol is paused.

---

### Proof of Concept

1. User deposits 10,000 FLOW into a COA and borrows 8,000 USDC from an Aave fork deployed on Flow EVM, at 90% collateral ratio (near liquidation threshold).
2. Governance pauses EVM by saving `true` to `/storage/evmOperationsPaused` (multi-sig governance transaction).
3. Every subsequent Flow block, the system chunk transaction calls `evmHeartbeat?.heartbeat()` → `InternalEVM.commitBlockProposal()` → EVM block height increments, EVM timestamp advances.
4. User attempts `coa.call(to: lendingProtocol, data: repayCalldata, ...)` → panics with `"EVM operations are temporarily paused"` due to the guard at line 644.
5. After 24 hours of pause (at a 20% APR during a liquidity crisis), ~0.055% interest accrues in the lending protocol's accounting.
6. EVM unpauses. A liquidator calls `liquidate(user)` on the lending protocol. The protocol computes 24 hours of interest retroactively, finds the position undercollateralized, and seizes the user's 10,000 FLOW collateral.
7. The user loses their collateral despite having had no opportunity to repay during the pause.

The root cause — `Heartbeat.heartbeat()` at `fvm/evm/stdlib/contract.cdc:1200` calling `InternalEVM.commitBlockProposal()` without an `isPaused()` check — is the necessary and sufficient vulnerable step. [10](#0-9)

```go
func (h *ContractHandler) commitBlockProposal() (err error) {
	defer func() {
		if err == nil {
			// Invalidate drycall cache if EVM state is changed (commitBlockProposal is successful).
			h.invalidateDryCallCache()
		}
	}()

	// load latest block proposal
	bp, err := h.backend.BlockProposal()
	if err != nil {
		return err
	}

	// commit the proposal
	err = h.backend.CommitBlockProposal(bp)
	if err != nil {
		return err
	}

	// emit block executed event
	err = h.emitEvent(events.NewBlockEvent(&bp.Block))
	if err != nil {
		return err
	}

	// report metrics
	h.backend.EVMBlockExecuted(
		len(bp.TxHashes),
		bp.TotalGasUsed,
		types.UnsafeCastOfBalanceToFloat64(bp.TotalSupply),
	)

	// log evm block commitment
	logger := h.backend.Logger()
	logger.Info().
		Uint64("evm_height", bp.Height).
		Int("tx_count", len(bp.TxHashes)).
		Uint64("total_gas_used", bp.TotalGasUsed).
		Uint64("total_supply", bp.TotalSupply.Uint64()).
		Msg("EVM Block Committed")

	return nil
```
