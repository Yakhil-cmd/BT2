### Title
Unauthorized Burn and Transfer of Any User's Native Tokens via Bank Precompile — (`x/cronos/keeper/precompiles/bank.go`)

### Summary
The `BankContract` precompile at address `0x0000000000000000000000000000000000000064` exposes `burn(address, uint256)` and `transfer(address, address, uint256)` methods that accept an arbitrary victim address without verifying the caller is authorized to act on that address. Any unprivileged EVM contract can burn or transfer native bank module tokens (denom `evm/<callerAddress>`) from any account that holds them.

### Finding Description
In `BankContract.Run()`, the shared `MintMethodName, BurnMethodName` case unpacks `args[0]` as the address to burn from and immediately calls `bankKeeper.SendCoinsFromAccountToModule(ctx, addr, ...)` with no check that `contract.Caller() == addr`:

```go
// bank.go lines 121-149
recipient := args[0].(common.Address)   // victim address — caller-supplied
...
addr := sdk.AccAddress(recipient.Bytes())
...
// No check: contract.Caller() == recipient
if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
    ...
}
if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
    ...
}
```

The `transfer` case is identical in structure — `sender` (args[0]) is the address funds are moved *from*, but there is no check that `contract.Caller() == sender`:

```go
// bank.go lines 175-192
sender := args[0].(common.Address)   // victim address — caller-supplied
...
from := sdk.AccAddress(sender.Bytes())
...
// No check: contract.Caller() == sender
if err := bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt)); err != nil {
    ...
}
```

The denom is derived from `contract.Caller()` — `EVMDenom(contract.Caller())` = `"evm/" + callerAddress`. So the burn/transfer operates on the `evm/<callerContractAddress>` denom. Any contract that calls the precompile can burn or transfer tokens of its own denom from any victim address that holds them. [1](#0-0) [2](#0-1) 

The precompile interface is publicly documented and callable by any EVM contract: [3](#0-2) 

### Impact Explanation
**Critical.** An unprivileged attacker contract can:
1. Call `bank.burn(victimAddress, amount)` to destroy any victim's `evm/<attackerContract>` native tokens without consent — an unauthorized burn of CRC20/native assets.
2. Call `bank.transfer(victimAddress, attackerAddress, amount)` to steal any victim's `evm/<attackerContract>` native tokens — an unauthorized transfer of CRC20/native assets.

Both operations bypass all ownership checks and directly manipulate the Cosmos bank module state for the victim's account.

### Likelihood Explanation
**High.** The bank precompile is at a fixed, well-known address (`0x0000000000000000000000000000000000000064`). Any EVM contract can call it with no special permissions. Victims who hold `evm/<contractAddress>` tokens (e.g., received via IBC auto-conversion or direct transfer) are exposed to any contract at that address burning or stealing their balance. [4](#0-3) 

### Recommendation
For `burn`: verify `contract.Caller()` equals the address being burned from:
```go
if contract.Caller() != recipient {
    return nil, errors.New("caller is not authorized to burn from this address")
}
```
For `transfer`: verify `contract.Caller()` equals the `sender` address:
```go
if contract.Caller() != sender {
    return nil, errors.New("caller is not authorized to transfer from this address")
}
```

### Proof of Concept
1. Alice holds 1000 `evm/0xAttackerContract` tokens (received via IBC conversion or direct transfer).
2. Attacker deploys `MaliciousContract` at address `0xAttackerContract`.
3. `MaliciousContract` calls:
   ```solidity
   IBankModule(0x0000000000000000000000000000000000000064)
       .burn(aliceAddress, 1000);
   ```
4. `BankContract.Run()` computes `denom = "evm/" + address(this)` = `"evm/0xAttackerContract"`, then calls `SendCoinsFromAccountToModule(ctx, alice, ...)` followed by `BurnCoins(...)` — Alice's 1000 tokens are destroyed with no authorization check.
5. Alternatively, step 3 uses `.transfer(aliceAddress, attackerAddress, 1000)` to steal Alice's tokens instead of burning them. [5](#0-4) [6](#0-5)
