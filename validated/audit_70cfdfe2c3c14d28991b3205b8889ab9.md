### Title
`MsgTurnBridge` Handler Is a No-Op Stub With No Permission Check — Bypasses Bridge Circuit-Breaker Authorization (`File: x/cronos/keeper/msg_server.go`)

### Summary

The `TurnBridge` gRPC message handler is implemented as a stub that unconditionally returns `nil, nil`. It performs no permission check and mutates no state. Any unprivileged address can submit `MsgTurnBridge` and receive a success response, while the bridge circuit-breaker is permanently non-functional for everyone, including the admin.

### Finding Description

`MsgTurnBridge` is the designated emergency circuit-breaker for the Gravity bridge. ADR-009 explicitly lists it as one of the two messages that must be restricted to addresses holding the `CanTurnBridge` permission bit. The integration test `test_gravity_turn_bridge` confirms the intended behavior: a community address must be rejected, and only the validator (admin) must succeed.

The actual keeper implementation, however, is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Compare this with `UpdateTokenMapping`, which correctly gates on `HasPermission`:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [2](#0-1) 

The permission constants and the `HasPermission` function are fully implemented and ready to use: [3](#0-2) 

The proto definition and the gRPC service descriptor register `TurnBridge` as a live, callable endpoint: [4](#0-3) 

### Impact Explanation

Two distinct High-severity impacts apply simultaneously:

1. **Auth bypass**: Any unprivileged address can call `MsgTurnBridge` and receive a `200 OK` (nil error). The `CanTurnBridge` permission check that ADR-009 mandates is entirely absent. This directly bypasses the Cronos admin/governance authorization layer for bridge control.

2. **Permanent inability to disable the bridge**: Because the handler writes no state, the bridge can never actually be disabled — not by an unprivileged attacker, and not by the admin either. In a security incident (e.g., a Gravity bridge exploit draining funds), the intended emergency stop is silently inoperative. This constitutes a permanent inability for honest users and validators to invoke a valid bridge-control flow under normal network assumptions. [5](#0-4) 

### Likelihood Explanation

The endpoint is publicly registered and callable by any address that can pay gas. No special privilege, leaked key, or cryptographic break is required. The stub was likely left in place during a refactor and was never completed.

### Recommendation

Replace the stub with a real implementation that:
1. Checks `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and returns `ErrUnauthorized` on failure.
2. Persists the `enable` flag to a dedicated KV store key (e.g., `KeyPrefixBridgeEnabled`).
3. Has all bridge-outbound paths (Gravity end-blocker, EVM hooks) read that flag and reject operations when the bridge is disabled.

### Proof of Concept

```
# Step 1 – any unprivileged address submits MsgTurnBridge{enable: false}
cronosd tx cronos turn-bridge false --from attacker --chain-id cronos_777-1

# Expected (per ADR-009 and integration test): ErrUnauthorized
# Actual: tx succeeds (code 0), bridge state unchanged, circuit-breaker silently broken

# Step 2 – admin also cannot disable the bridge
cronosd tx cronos turn-bridge false --from admin --chain-id cronos_777-1
# Also succeeds with code 0, also changes nothing
```

The integration test `test_gravity_turn_bridge` already asserts `rsp["code"] == 0` for the admin path and `rsp["code"] != 0` for the community path, but the stub makes both return `code 0` with no state effect, meaning the test itself would fail against the live implementation — confirming the regression. const (
	CanChangeTokenMapping uint64                                  = 1 << iota // 1
	CanTurnBridge                                                             // 2
	All                   = CanChangeTokenMapping | CanTurnBridge             // 3
)

func (k Keeper) SetPermissions(ctx sdk.Context, address sdk.AccAddress, permissions uint64) {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := sdk.Uint64ToBigEndian(permissions)
	store.Set(types.AdminToPermissionsKey(address), permissionsBytes)
}

func (k Keeper) GetPermissions(ctx sdk.Context, address sdk.AccAddress) uint64 {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := store.Get(types.AdminToPermissionsKey(address))
	return sdk.BigEndianToUint64(permissionsBytes)
}

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
