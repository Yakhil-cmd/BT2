### Title
No Enforcement of One-Time Presignature Use Enables Nonce Reuse and Full Private Key Extraction — (`File: src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both the OT-based and Robust ECDSA implementations document that each `PresignOutput` must be consumed **exactly once**, but the library provides no type-system or runtime mechanism to enforce this. `PresignOutput` is `Clone + Serialize + Deserialize`, and `rerandomize_presign` accepts it by shared reference (`&PresignOutput`), allowing the same presignature to be rerandomized and signed with an unlimited number of times. A malicious coordinator can exploit this to produce two valid signatures from the same underlying nonce, enabling full private key recovery via standard ECDSA nonce-reuse algebra.

---

### Finding Description

**Vulnerability class:** Threshold-state bug — missing one-time-use enforcement (direct analog to "eternal free mint": no restriction on how many times a privileged one-use resource can be consumed).

**Root cause — OT-based ECDSA:**

`PresignOutput` is declared with `Clone`, `Serialize`, and `Deserialize`: [1](#0-0) 

`rerandomize_presign` accepts it by shared reference, not by value: [2](#0-1) 

This means the same `PresignOutput` can be passed to `rerandomize_presign` an unlimited number of times with different `(msg_hash, tweak, entropy)` arguments, producing multiple distinct `RerandomizedPresignOutput` values — all sharing the same underlying secret nonce `k`.

**Root cause — Robust ECDSA:**

Identical pattern: `PresignOutput` is `Clone + Serialize + Deserialize`: [3](#0-2) 

And `rerandomize_presign` again takes `&PresignOutput`: [4](#0-3) 

**Documentation acknowledges the requirement but does not enforce it:**

The presign docstring in both modules states "it's crucial that a presignature is never reused": [5](#0-4) [6](#0-5) 

The orchestration documentation explicitly marks presignature outputs as one-time-use ("red boxes") and states "It's **critical** that the output is then destroyed": [7](#0-6) 

The security documentation for Robust ECDSA explicitly warns that reuse enables split-view attacks and private key recovery: [8](#0-7) 

None of these warnings are enforced in code. The library exposes `rerandomize_presign` as a public API that silently accepts the same `PresignOutput` repeatedly.

---

### Impact Explanation

**Impact: Critical — Extraction of aggregate private signing key material.**

When the same `PresignOutput` is rerandomized twice with different `(h, tweak, entropy)` tuples producing `delta1` and `delta2`, both resulting signatures share the same underlying nonce `k`. Using the OT-based ECDSA signing equation:

```
s · delta = h·k + r·sigma + r·tweak·k
```

From two signing sessions:
```
s1·delta1 = h1·k + r·sigma + r·tweak1·k
s2·delta2 = h2·k + r·sigma + r·tweak2·k
```

Subtracting eliminates `sigma`:
```
k = (s1·delta1 - s2·delta2) / (h1 - h2 + r·(tweak1 - tweak2))
```

With `k` recovered, `sigma = (s1·delta1 - h1·k - r·tweak1·k) / r`, and since `sigma = k · x`, the full private key `x` is recovered. The Robust ECDSA scheme is equally vulnerable per its own security documentation. [9](#0-8) 

---

### Likelihood Explanation

**Likelihood: High.**

The attack requires only that a coordinator (or any participant holding a `PresignOutput`) calls `rerandomize_presign` twice on the same object — a trivially reachable code path with no privilege beyond normal protocol participation. The `PresignOutput` type is `Clone` and `Serialize`/`Deserialize`, making it straightforward to store and reuse. No cryptographic break, no leaked keys, and no external assumptions are required. The only barrier is the documentation warning, which is not machine-enforced.

---

### Recommendation

Enforce one-time use at the type level by consuming `PresignOutput` by value in `rerandomize_presign`:

```rust
// OT-based: src/ecdsa/ot_based_ecdsa/mod.rs
pub fn rerandomize_presign(
    presignature: PresignOutput,   // consume by value, not &PresignOutput
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>

// Robust: src/ecdsa/robust_ecdsa/mod.rs
pub fn rerandomize_presign(
    presignature: PresignOutput,   // consume by value
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

Additionally, remove `Clone`, `Serialize`, and `Deserialize` from `PresignOutput` in both modules, or at minimum gate serialization behind a clearly documented "unsafe" feature flag. This mirrors the fix described in the external report: replacing an "eternal" unlimited-use resource with a counted/consumed one. [1](#0-0) [3](#0-2)

### Proof of Concept

**Entry path (OT-based ECDSA):**

```rust
// Attacker holds a PresignOutput from a legitimate presign session
let presign_out: PresignOutput = /* obtained from presign protocol */;

// Rerandomize the SAME presignature twice with different args
let args1 = RerandomizationArguments::new(pk, tweak1, msg_hash1, presign_out.big_r, participants.clone(), entropy1);
let args2 = RerandomizationArguments::new(pk, tweak2, msg_hash2, presign_out.big_r, participants.clone(), entropy2);

// Both calls succeed — no error, no consumption
let rerand1 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args1).unwrap();
let rerand2 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args2).unwrap();

// Run two signing sessions with the two rerandomized presignatures
let sig1 = /* sign with rerand1, msg_hash1 */;
let sig2 = /* sign with rerand2, msg_hash2 */;

// Recover k from the two signatures (all values on the right are known to the attacker):
// k = (s1·delta1 - s2·delta2) / (h1 - h2 + r·(tweak1 - tweak2))
// x = sigma / k  (sigma recovered from either signature equation)
```

The `rerandomize_presign` function accepts `&presign_out` both times without error, as there is no consumed-flag, no atomic swap, and no type-system barrier preventing it. [10](#0-9) [11](#0-10)

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
