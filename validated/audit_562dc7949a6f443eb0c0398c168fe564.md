### Title
Pool Member Cannot Cancel Exit Intent After Staker Removal, Temporarily Freezing Remaining Funds - (File: src/pool/pool.cairo)

### Summary
`exit_delegation_pool_intent(0)` — the mechanism for canceling a pending exit intent — reverts when the staker has been removed and the pool member already has an active intent. The guard in `undelegate_from_staking_contract_intent` applies identically to both "set intent" and "cancel intent" paths, mirroring the external report's pattern where `setApprovalForAll(operator, false)` was blocked by the same check as `setApprovalForAll(operator, true)`.

### Finding Description

`exit_delegation_pool_intent` serves two roles depending on the `amount` argument:
- `amount > 0`: set or update an exit intent
- `amount == 0`: cancel an existing intent

Both paths unconditionally call `undelegate_from_staking_contract_intent`, which contains the following guard when the staker is inactive:

```cairo
// src/pool/pool.cairo
fn undelegate_from_staking_contract_intent(
    self: @ContractState, pool_member: ContractAddress, amount: Amount,
) -> Timestamp {
    if !self.is_staker_active() {
        assert!(
            self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
            "{}",
            Error::UNDELEGATE_IN_PROGRESS,
        );
        return Time::now();
    }
    ...
``` [1](#0-0) 

When `unpool_time.is_some()` (an intent is active) and the staker is removed, this assert fires for **any** `amount` value, including `0`. The cancellation path in the caller is never reached:

```cairo
// src/pool/pool.cairo
if amount.is_zero() {
    pool_member_info.unpool_time = Option::None;  // never executed
}
``` [2](#0-1) 

The existing test suite explicitly confirms this revert path:

```cairo
pool_dispatcher.exit_delegation_pool_intent(:amount);
snforge_std::store(pool_contract, selector!("staker_removed"), array![true.into()].span());
let result = pool_safe_dispatcher.exit_delegation_pool_intent(:amount);
assert_panic_with_error(:result, expected_error: Error::UNDELEGATE_IN_PROGRESS.describe());
``` [3](#0-2) 

The spec's own logic section states: *"If `amount` is zero, remove request for intent (if exists)"*, but the precondition *"Pool member is not in an exit window or staker is active"* overrides it, creating a contradiction. [4](#0-3) 

### Impact Explanation

Consider a pool member with 100 tokens who set a partial exit intent for 10 tokens, then the staker is removed via `unstake_action`. The pool member now:

1. **Cannot cancel** the intent (`exit_delegation_pool_intent(0)` → `UNDELEGATE_IN_PROGRESS`)
2. **Cannot increase** the intent to cover the full 100 tokens (same revert)
3. **Cannot set a new intent** for the remaining 90 tokens (same revert, since `unpool_time.is_some()`)
4. **Cannot switch** the remaining 90 tokens to another pool — `switch_delegation_pool` requires `unpool_time.is_some()`, but only the 10-token intent qualifies

The 90 tokens are frozen until the pool member completes the 10-token exit via `exit_delegation_pool_action` (which clears `unpool_time`), then waits through a full new exit window for the remaining 90 tokens. The maximum additional delay equals the `exit_wait_window`, configurable up to `MAX_EXIT_WAIT_WINDOW` (12 weeks). [5](#0-4) 

This constitutes **temporary freezing of funds** (High impact per the allowed scope).

### Likelihood Explanation

The scenario requires:
1. A pool member to have called `exit_delegation_pool_intent` with a partial amount — a normal, expected operation.
2. The staker to subsequently be removed via `unstake_action` — also a normal, expected operation.

Both steps are routine protocol flows with no special preconditions. Any pool member who set a partial intent before their staker exits is affected.

### Recommendation

Apply the `UNDELEGATE_IN_PROGRESS` guard only when `amount > 0`. When `amount == 0`, the intent is being canceled, which should always be permitted regardless of staker status:

```cairo
fn undelegate_from_staking_contract_intent(
    self: @ContractState, pool_member: ContractAddress, amount: Amount,
) -> Timestamp {
    if !self.is_staker_active() {
+       if amount.is_non_zero() {
            assert!(
                self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                "{}",
                Error::UNDELEGATE_IN_PROGRESS,
            );
+       }
        return Time::now();
    }
    ...
``` [6](#0-5) 

### Proof of Concept

```cairo
// Scenario: pool member has partial intent, staker is removed, remaining funds frozen

// 1. Pool member sets a partial exit intent (10 out of 100 tokens)
pool_dispatcher.exit_delegation_pool_intent(amount: 10);
// pool_member_info.unpool_time = Some(T), unpool_amount = 10

// 2. Staker calls unstake_intent + unstake_action → staker_removed = true

// 3. Pool member tries to cancel intent to switch all 100 tokens to a new pool
// → REVERTS with UNDELEGATE_IN_PROGRESS
pool_dispatcher.exit_delegation_pool_intent(amount: 0);

// 4. Pool member tries to increase intent to cover all 100 tokens
// → REVERTS with UNDELEGATE_IN_PROGRESS
pool_dispatcher.exit_delegation_pool_intent(amount: 100);

// 5. Pool member can only switch the 10-token intent; remaining 90 tokens are frozen
pool_dispatcher.switch_delegation_pool(to_staker, to_pool, amount: 10);
// The 90 remaining tokens cannot be switched or exited until the 10-token
// exit_delegation_pool_action completes and a new intent is set.
```

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }
