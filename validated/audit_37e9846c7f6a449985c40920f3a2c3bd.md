### Title
Staker Can Retroactively Apply Increased Commission to Current Epoch's Pending Rewards - (File: src/staking/staking.cairo)

### Summary
`set_commission()` in `staking.cairo` immediately overwrites the stored commission rate without first checkpointing (distributing) pending rewards for the current epoch. A staker who holds an active commission commitment can raise their commission to `max_commission` (up to 100%) at any point within an epoch, then attest — causing the elevated rate to be applied to the **entire epoch's** pool rewards, including the portion earned before the change. Delegators lose their unclaimed yield for that epoch.

### Finding Description
In the V2 (attestation-based) reward path, rewards are calculated once per epoch at attestation time. The commission is read from storage at that moment and applied to the full epoch's pool rewards.

`set_commission` writes the new rate directly to storage with no reward checkpoint: [1](#0-0) 

The internal `update_commission` helper enforces commitment bounds but performs no reward settlement before writing: [2](#0-1) 

When `_update_rewards` → `calculate_staker_pools_rewards` runs at attestation time, it reads the **current** commission and applies it to the entire epoch's pool rewards: [3](#0-2) 

The commission split utility: [4](#0-3) 

The code itself carries an explicit acknowledgement of this gap: [5](#0-4) 

**Attack path:**

1. Staker stakes with commission `200` (2 %). Delegators join the pool.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current + 1)`. This is valid because `max_commission >= current_commission`. [6](#0-5) 

3. Near the end of the epoch (before attesting), staker calls `set_commission(commission: 10000)`. The commitment is active, so the assertion `commission <= max_commission` passes.
4. Staker attests. `update_rewards_from_attestation_contract` fires `_update_rewards`, which reads commission `10000` and routes 100 % of pool rewards to the staker as commission. Delegators receive zero for the epoch. [7](#0-6) 

The pool contract's `update_rewards_from_staking_contract` receives the already-commission-deducted (zero) amount and updates the cumulative sigma accordingly, permanently locking delegators out of that epoch's yield: [8](#0-7) 

### Impact Explanation
Delegators lose 100 % of their unclaimed yield for the epoch in which the commission was raised. Because the pool's `cumulative_rewards_trace` is updated with the post-commission (zero) amount, the loss is permanent — delegators cannot recover the yield by waiting or claiming later. This maps directly to **High: Theft of unclaimed yield**.

### Likelihood Explanation
The staker must first call `set_commission_commitment` with a high `max_commission`, which is a public, permissionless action available to any staker. The subsequent `set_commission` call is also permissionless for the staker. No external dependency, oracle, or admin action is required. The only constraint is that the staker must attest within the same epoch after raising the commission, which they control via their operational address. Likelihood is **Medium** (requires deliberate setup but no privileged access beyond owning the staker key).

### Recommendation
Mirror the M-5 fix: before writing the new commission to storage, checkpoint pending rewards by distributing the current epoch's accrued pool rewards at the old rate. Concretely, `set_commission` should call the equivalent of `update_rewards_from_attestation_contract` (or a shared internal helper) for the calling staker before overwriting `staker_pool_info.commission`. This ensures the old rate is applied to all rewards earned up to the moment of the change, and the new rate only affects future epochs.

### Proof of Concept

```
Epoch N, block B1: Staker stakes, commission = 200 (2%).
Epoch N, block B2: Delegator delegates 1 000 000 STRK.
Epoch N, block B3: Staker calls set_commission_commitment(max_commission=10000, expiration_epoch=N+1).
Epoch N, block B4: Staker calls set_commission(commission=10000).
                   Storage: staker_pool_info.commission = Some(10000).
Epoch N, block B5: Staker attests.
                   _update_rewards reads commission = 10000.
                   split_rewards_with_commission(pool_rewards_including_commission, 10000)
                     → commission_rewards = pool_rewards_including_commission
                     → pool_rewards = 0
                   Pool receives 0 STRK.
                   cumulative_rewards_trace updated with delta = 0.
Epoch N+1:         Delegator calls claim_rewards → receives 0 STRK for epoch N.
```

The delegator's entire epoch-N yield is redirected to the staker's `unclaimed_rewards_own` as commission, with no recourse.

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();

                    );

                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }

        :rewards_including_commission, :commission,

}
