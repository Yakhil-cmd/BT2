### Title
Floor-rounding in `validate_tx_l2_gas_price_within_threshold` admits transactions below the intended minimum L2 gas price threshold - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary
The stateful gateway's L2 gas price admission check uses `Ratio::to_integer()` — which performs **floor (truncation) division** — to compute the minimum acceptable gas price threshold. When `min_gas_price_percentage × previous_block_l2_gas_price` is not evenly divisible by 100, the threshold is silently rounded down by 1 fri, allowing a transaction whose `max_price_per_unit` is 1 fri below the operator-intended minimum to pass the admission check and enter the mempool.

### Finding Description
In `validate_tx_l2_gas_price_within_threshold`, the threshold is computed as:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();   // ← floor division
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
```

`Ratio::to_integer()` in Rust's `num_rational` crate returns `⌊numerator / denominator⌋` (floor). The operator-intended invariant is:

```
tx_l2_gas_price ≥ ⌈min_gas_price_percentage × previous_price / 100⌉
```

But the actual check enforces:

```
tx_l2_gas_price ≥ ⌊min_gas_price_percentage × previous_price / 100⌋
```

Whenever `min_gas_price_percentage × previous_price` is not divisible by 100, the floor is strictly less than the true fractional threshold. A transaction with `tx_l2_gas_price = ⌊threshold⌋` satisfies `tx_l2_gas_price >= threshold` and is admitted, even though it is below the operator-intended percentage minimum.

This is the direct Sequencer analog of the external report's Bug 1: implicit floor rounding in `QuantMath.mul`/`div` caused the collateral requirement to be computed as 0 instead of a small positive amount. Here, implicit floor rounding in the threshold computation causes the admission gate to accept a gas price that is 1 fri below the intended minimum. [1](#0-0) 

### Impact Explanation
A transaction whose L2 gas price is 1 fri below the operator-configured minimum percentage of the previous block's L2 gas price is admitted to the mempool. This breaks the admission invariant and maps to: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

The `validate_resource_bounds` call is the only stateful gas-price gate before `run_validate_entry_point`; a transaction that passes it proceeds to blockifier validation and, if that succeeds, is forwarded to the mempool via `mempool_client.add_tx`. [2](#0-1) [3](#0-2) 

### Likelihood Explanation
Triggered whenever `min_gas_price_percentage × previous_block_l2_gas_price` is not divisible by 100. With the default `min_gas_price_percentage = 100`, the product is always divisible by 100 (no rounding occurs). However, any operator-configured non-100% value (e.g., 50, 80, 90) combined with a previous price that is not a multiple of `100 / gcd(percentage, 100)` produces a fractional threshold that is silently floored. An attacker who observes the previous block's L2 gas price can craft a transaction with exactly the floored threshold to bypass the intended minimum. [4](#0-3) 

### Recommendation
Replace `to_integer()` with ceiling division. Using `num_rational::Ratio::ceil()`:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

Or equivalently in integer arithmetic: `(min_gas_price_percentage * previous_price + 99) / 100`.

This mirrors the pattern already used correctly elsewhere in the codebase for gas conversions: [5](#0-4) 

### Proof of Concept
- `previous_block_l2_gas_price = 101 fri`
- `min_gas_price_percentage = 50` (operator wants ≥ 50% of previous price)
- True threshold = `50 × 101 / 100 = 50.5 fri` → intended minimum is 51 fri
- Computed threshold = `⌊50.5⌋ = 50 fri`
- A transaction with `max_price_per_unit = 50 fri` satisfies `50 >= 50` and is admitted
- But `50 < 50.5`, so it is below the operator-intended 50% minimum

The attacker submits a transaction with `l2_gas.max_price_per_unit = 50` when the previous block's L2 gas price is 101 fri and `min_gas_price_percentage = 50`. The gateway admits it despite it being below the intended threshold. The test suite confirms the boundary at even-divisible values only: [6](#0-5)
