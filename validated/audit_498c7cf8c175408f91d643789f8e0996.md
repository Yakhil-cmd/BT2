### Title
Missing Access Control on `deploy_erc20_token` Allows Any User to Deploy ERC-20 Mirrors for Arbitrary NEP-141 Tokens - (File: `engine/src/contract_methods/connector.rs`)

---

### Summary

The `deploy_erc20_token` function in `engine/src/contract_methods/connector.rs` is a public NEAR contract method that lacks a `require_owner_only` guard. The code's own inline comment explicitly states the function "could be executed by the owner of the contract only," but no such check is enforced. Any unprivileged NEAR account can call it to deploy an ERC-20 mirror contract for any NEP-141 token, including front-running legitimate admin deployments with incorrect parameters.

---

### Finding Description

`deploy_erc20_token` is exposed as a public NEAR contract entrypoint via `lib.rs`: [1](#0-0) 

The implementation only checks `require_running` and performs no caller authorization: [2](#0-1) 

The comment at line 149 explicitly acknowledges the intended restriction:

> "Safe because these promises are read-only calls to the main engine contract and **this transaction could be executed by the owner of the contract only**."

Despite this, no `require_owner_only` call is present, unlike every other privileged function in the same codebase: [3](#0-2) 

For comparison, `mirror_erc20_token` — a closely related function — correctly enforces the owner check: [4](#0-3) 

---

### Impact Explanation

An attacker can call `deploy_erc20_token` with the `Legacy` variant for any NEP-141 token before the admin does. This creates the NEP-141 → ERC-20 address mapping in the engine's persistent state. Once the mapping is set:

- A subsequent legitimate admin call using `WithMetadata` (which fetches correct decimals and symbol from the NEP-141 contract) will either fail or be a no-op because the mapping already exists.
- Users who bridge NEP-141 tokens to Aurora will receive tokens from the attacker-influenced ERC-20 contract, which was deployed without correct metadata (e.g., wrong decimals).
- Incorrect decimal accounting causes bridged token balances to be permanently misstated, effectively freezing or destroying user funds at the bridge layer.

**Impact: High — Permanent freezing of bridged user funds / ERC-20 mirror accounting corruption.**

---

### Likelihood Explanation

- The function is callable by any NEAR account with no preconditions beyond the contract being unpaused.
- The attacker only needs to know the NEP-141 account ID of a token that has not yet been mirrored — this is public on-chain information.
- Front-running a legitimate `deploy_erc20_token` call on NEAR is straightforward since transaction ordering within a block is observable.
- No special privileges, keys, or capital are required.

**Likelihood: High.**

---

### Recommendation

Add `require_owner_only` (or an equivalent authorized-caller check) at the top of `deploy_erc20_token`, consistent with the pattern used in `mirror_erc20_token` and all other privileged connector/admin functions:

```rust
pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?; // ADD THIS
        // ...
    })
}
```

---

### Proof of Concept

Any NEAR account (`attacker.near`) can execute the following sequence:

1. Observe that `usdc.near` has not yet been mirrored on Aurora.
2. Call `deploy_erc20_token` on the Aurora Engine contract with `DeployErc20TokenArgs::Legacy("usdc.near")` from `attacker.near` — no owner key required.
3. The engine stores the NEP-141 → ERC-20 mapping with no metadata (wrong decimals default).
4. The admin's subsequent `deploy_erc20_token` with `WithMetadata` fails because the mapping already exists.
5. All users who bridge USDC via `ft_on_transfer` receive tokens from the malformed ERC-20 contract, with permanently incorrect decimal accounting, freezing their bridged value. [5](#0-4)

pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let bytes = io.read_input().to_vec();
        let args =
            DeployErc20TokenArgs::deserialize(&bytes).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

        match args {
            DeployErc20TokenArgs::Legacy(nep141) => {
                let address = engine::deploy_erc20_token(nep141, None, io, env, handler)?;

                io.return_output(
                    &borsh::to_vec(address.as_bytes()).map_err(|_| errors::ERR_SERIALIZE)?,
                );
                Ok(PromiseOrValue::Value(address))
            }
            DeployErc20TokenArgs::WithMetadata(nep141) => {
                let args = borsh::to_vec(&nep141).map_err(|_| errors::ERR_SERIALIZE)?;
                let base = PromiseCreateArgs {
                    target_account_id: nep141,
                    method: "ft_metadata".to_string(),
                    args: vec![],
                    attached_balance: ZERO_YOCTO,
                    attached_gas: READ_PROMISE_ATTACHED_GAS,
                };
                let callback = PromiseCreateArgs {
                    target_account_id: env.current_account_id(),
                    method: "deploy_erc20_token_callback".to_string(),
                    args,
                    attached_balance: ZERO_YOCTO,
                    attached_gas: DEPLOY_ERC20_TOKEN_CALLBACK_ATTACHED_GAS,
                };
                // Safe because these promises are read-only calls to the main engine contract
                // and this transaction could be executed by the owner of the contract only.
                let promise_args = PromiseWithCallbackArgs { base, callback };
                let promise_id = handler.promise_create_with_callback(&promise_args);

                handler.promise_return(promise_id);

                Ok(PromiseOrValue::Promise(promise_args))
            }
        }
    })
}

pub fn require_owner_only(
    state: &state::EngineState,
    predecessor_account_id: &AccountId,
) -> Result<(), ContractError> {
    if &state.owner_id != predecessor_account_id {
        return Err(errors::ERR_NOT_ALLOWED.into());
    }
    Ok(())
}
