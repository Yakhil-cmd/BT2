### Title
`last_reward_block` Updated Unconditionally Before `disable_rewards` Guard, Enabling Permanent Griefing of All Staker Yield — (File: `src/staking/staking.cairo`)

---

### Summary

In `StakingRewardsManagerImpl::update_rewards`, the global `last_reward_block` storage variable is written **before** the `disable_rewards` early-return guard. Because `general_prerequisites()` enforces no caller restriction beyond "non-zero address," any unprivileged caller can invoke `update_rewards(any_staker, disable_rewards=true)` once per block to permanently consume that block's reward slot without distributing any yield. The legitimate sequencer's subsequent call for the same block is rejected with `REWARDS_ALREADY_UPDATED`, and the block's rewards are irrecoverably lost for all stakers.

---

### Finding Description

In `src/staking/staking.cairo`, `update_rewards` (lines 1449–1507):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: unpaused + caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );

    // ... staker existence / active checks ...

    // *** BUG: last_reward_block written BEFORE the disable_rewards guard ***
    self.last_reward_block.write(current_block_number);   // line 1485

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // no rewards distributed
    }

    // ... calculate and distribute block rewards ...
}
``` [1](#0-0) 

`general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [2](#0-1) 

The spec states access is "Only starkware sequencer," but **no such check exists in the deployed code**. The `IStakingRewardsManager` interface exposes `update_rewards` as a public ABI entry point with no role guard. [3](#0-2) 

`last_reward_block` is a **single global** storage variable (not per-staker). Writing it with `disable_rewards=true` blocks reward distribution for **all stakers** for that block in a single transaction.

The attack path:
1. Consensus rewards are active (`!is_pre_consensus()`).
2. Attacker calls `update_rewards(any_valid_staker, disable_rewards=true)` at the start of block N.
3. `last_reward_block` is set to block N; zero rewards are distributed.
4. The legitimate sequencer's call `update_rewards(staker, disable_rewards=false)` for block N reverts with `REWARDS_ALREADY_UPDATED`.
5. Block N's rewards are permanently lost for every staker.
6. Attacker repeats every block.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Each griefed block's rewards are irrecoverable: the `last_reward_block` guard is a one-way latch per block. An attacker running this every block can reduce all stakers' yield to zero indefinitely at the cost of gas alone. There is no recovery mechanism — missed blocks are not retroactively compensated.

---

### Likelihood Explanation

**High.** The function is a public ABI entry point callable by any non-zero address. No token, stake, or privileged role is required. The attack is mechanically trivial (one transaction per block) and economically rational for any party wishing to harm stakers or the protocol.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards || is_pre_consensus()` guard, so the block slot is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Only mark the block as processed when rewards are actually distributed.
self.last_reward_block.write(current_block_number);

// ... calculate and distribute block rewards ...
```

Additionally, enforce the sequencer-only access control in code (matching the spec) to eliminate the open-caller surface entirely.

---

### Proof of Concept

```
1. Deploy system; staker stakes min_stake; advance K epochs (stake becomes effective).
2. Call set_consensus_rewards_first_epoch to activate consensus rewards.
3. Advance to consensus epoch.
4. From any non-zero EOA (attacker), call:
       staking.update_rewards(staker_address, disable_rewards=true)
   → Succeeds. last_reward_block = current_block. Zero rewards distributed.
5. From the legitimate sequencer, call:
       staking.update_rewards(staker_address, disable_rewards=false)
   → Reverts: "REWARDS_ALREADY_UPDATED"
6. Advance one block; repeat steps 4–5 every block.
7. After N blocks, staker.claim_rewards() returns 0.
   Expected (without attack): N × block_rewards > 0.
```

This mirrors the existing flow test `update_rewards_disable_rewards_consensus_rewards_flow_test` (lines 2882–2895), which already demonstrates that a `disable_rewards=true` call in a block blocks a subsequent `disable_rewards=false` call in the same block — the only missing piece is that the test does not model an unprivileged attacker making the first call.
