### Title
Unprivileged Caller Can Permanently Freeze All Consensus Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function is publicly callable with no access control. It accepts a caller-controlled `disable_rewards: bool` parameter. When called with `disable_rewards: true`, it advances the global `last_reward_block` to the current block but skips all reward distribution. Because `last_reward_block` is a single contract-wide value, any subsequent legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`. An unprivileged attacker can call this function once per block, every block, to permanently deny all consensus-phase block rewards to every staker in the protocol.

---

### Finding Description

`update_rewards` is exposed on the public `IStakingRewardsManager` interface with no role guard beyond the generic pause check (`general_prerequisites`):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // ← global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits without distributing
    }
    ...
``` [1](#0-0) 

The storage field `last_reward_block` is a single `BlockNumber` — not a per-staker mapping:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The one-call-per-block invariant is enforced globally:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

The attack sequence is:

1. Attacker calls `update_rewards(any_active_staker, disable_rewards: true)` at the first transaction of block N.
2. `last_reward_block` is written to N (line 1485).
3. The function returns early — zero rewards distributed.
4. Every subsequent call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block, all consensus block rewards are permanently lost.

The `disable_rewards` parameter is intended to signal that a staker did not perform their duties, but there is no check that the caller is an authorized consensus-layer contract or operator. Any EOA can supply `disable_rewards: true`.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Once consensus rewards are active (`is_pre_consensus()` returns `false`), every block produces STRK block rewards computed by `calculate_block_rewards` and distributed via `_update_rewards`. By consuming the global `last_reward_block` slot each block with `disable_rewards: true`, the attacker causes every block's reward allocation to be silently discarded. Stakers and delegators accumulate zero yield indefinitely. The rewards are not deferred — they are simply never credited, and the `unclaimed_rewards` balance in the `RewardSupplier` is never incremented for those blocks. [4](#0-3) 

---

### Likelihood Explanation

**High.** The attack requires no capital, no privileged role, and no protocol knowledge beyond the public ABI. It costs only gas per block. It can be automated with a trivial script that monitors the mempool and front-runs any legitimate `update_rewards` call, or simply submits the griefing call as the first transaction of each block. There is no economic disincentive.

---

### Recommendation

Add an access-control guard to `update_rewards` so that only an authorized caller (e.g., the consensus layer contract, a designated sequencer address, or a role such as `APP_GOVERNOR`) can invoke it. Alternatively, remove the `disable_rewards` parameter from the public interface and expose a separate privileged function for the "staker missed duties" path, keeping the reward-distribution path open only to authorized callers.

---

### Proof of Concept

```
// Precondition: consensus rewards are active (is_pre_consensus() == false)
// Attacker script, run once per block:

loop {
    wait_for_new_block();
    staking_contract.update_rewards(
        staker_address = any_active_staker,   // any valid staker address
        disable_rewards = true,
    );
    // last_reward_block is now == current_block
    // All legitimate update_rewards calls this block revert with REWARDS_ALREADY_UPDATED
    // Zero block rewards are distributed to any staker
}
```

Every staker's `unclaimed_rewards_own` and every pool's cumulative reward trace stop growing. The `RewardSupplier.unclaimed_rewards` is never incremented for those blocks. The yield is permanently frozen. [5](#0-4) [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
