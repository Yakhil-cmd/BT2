### Title
Unprivileged Caller Can Permanently Deny Block Rewards via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` is a publicly callable function that accepts a caller-controlled `disable_rewards` boolean. When called with `disable_rewards: true`, the function updates `last_reward_block` to the current block number — consuming the one-per-block reward slot — but skips all reward distribution. Because the slot is consumed, no subsequent call can distribute rewards for that block. Any unprivileged address can call this once per block to permanently deny a staker's block rewards.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` is exposed as a public ABI entry point with no caller restriction beyond "not zero address": [1](#0-0) 

The function enforces a strict one-call-per-block invariant by asserting `current_block_number > last_reward_block` and then immediately writing the current block number to storage: [2](#0-1) 

After writing `last_reward_block`, the function checks `disable_rewards`: [3](#0-2) 

If `disable_rewards` is `true`, the function returns immediately — no rewards are calculated, no `unclaimed_rewards` is incremented in the `RewardSupplier`, and no staker or pool balances are updated. Because `last_reward_block` was already written, any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. The block's reward opportunity is permanently gone.

`general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero: [4](#0-3) 

There is no role check, no whitelist, and no restriction on who may supply `disable_rewards: true`.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

For every block in which an attacker front-runs the legitimate `update_rewards` call with `disable_rewards: true`, the staker and its delegators permanently lose the block rewards that would have been distributed. The rewards are not deferred or locked — they are simply never minted into `unclaimed_rewards` in the `RewardSupplier`. There is no recovery path: no admin function can retroactively credit a missed block.

If sustained across many blocks, this amounts to a complete denial of consensus-era block rewards for any targeted staker.

---

### Likelihood Explanation

**High.** The attack requires:
- Any non-zero Starknet address (no stake, no role, no cost beyond gas).
- A single transaction per block, front-running the legitimate `update_rewards` call.

Front-running on Starknet is feasible via mempool observation or by simply submitting the transaction at the start of each block. The attacker has no financial incentive requirement — pure griefing suffices.

---

### Recommendation

Remove the `disable_rewards` parameter from the public interface, or gate it behind a privileged role (e.g., `only_app_governor` or `only_security_agent`). If the parameter is needed for migration scenarios, it should be callable only by a trusted address. Alternatively, restructure the function so that `last_reward_block` is only written after rewards are successfully distributed, preventing slot consumption on a no-op path.

---

### Proof of Concept

1. Staker `S` is registered and active. Consensus rewards are live (`is_pre_consensus()` returns `false`).
2. At block `N`, attacker `A` (any non-zero address) calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. Inside `update_rewards`:
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → early return, no rewards distributed.
4. The legitimate operator (or anyone else) attempts:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: false)
   ```
5. This reverts: `current_block_number (N) > last_reward_block (N)` is `false` → `REWARDS_ALREADY_UPDATED`.
6. Block `N`'s rewards for staker `S` and all its delegators are permanently lost.
7. Repeat every block to continuously deny all rewards. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
