### Title
Unauthenticated HTTP Admin Server Allows Any Network-Reachable Caller to Execute Privileged Node Commands - (`admin/command_runner.go`)

### Summary

The `CommandRunner` HTTP admin server starts without any authentication by default. When `tlsConfig` is `nil` (the default unless all three of `AdminCert`, `AdminKey`, and `AdminClientCAs` are explicitly configured), any HTTP client that can reach the admin port can invoke privileged commands — including weakening consensus security parameters, crashing execution nodes, and manipulating rate-limit allowlists — with no caller identity check at any layer.

### Finding Description

`CommandRunner.runAdminServer` in `admin/command_runner.go` starts an HTTP server bound to `r.httpAddress` (configured as `0.0.0.0:9002` in both localnet and testnet deployments):

```go
if r.tlsConfig == nil {
    err = httpServer.ListenAndServe()   // plain HTTP, no auth
} else {
    err = httpServer.ListenAndServeTLS("", "")
}
``` [1](#0-0) 

The gRPC server that backs the HTTP gateway is also created with no authentication interceptors:

```go
opts := []grpc.ServerOption{
    grpc.MaxRecvMsgSize(r.maxMsgSize),
    grpc.MaxSendMsgSize(r.maxMsgSize),
}
grpcServer := grpc.NewServer(opts...)
``` [2](#0-1) 

`runCommand`, the function that dispatches every admin command, performs zero caller identity verification:

```go
func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
    r.logger.Info().Str("command", command).Msg("received new command")
    req := &CommandRequest{Data: data}
    if validator := r.getValidator(command); validator != nil { ... }
    if handler := r.getHandler(command); handler != nil { ... }
    ...
}
``` [3](#0-2) 

TLS mutual authentication is opt-in and requires all three flags (`AdminCert`, `AdminKey`, `AdminClientCAs`) to be set simultaneously. If any one is missing, the server falls back to unauthenticated plain HTTP:

```go
if (fnb.AdminCert != NotSet || fnb.AdminKey != NotSet || fnb.AdminClientCAs != NotSet) &&
    !(fnb.AdminCert != NotSet && fnb.AdminKey != NotSet && fnb.AdminClientCAs != NotSet) {
    return fmt.Errorf("admin cert / key and client certs must all be provided to enable mutual TLS")
}
``` [4](#0-3) 

If none of the three flags are set (the default), the server starts with no TLS and no authentication at all. The localnet bootstrap explicitly binds to `0.0.0.0`:

```go
fmt.Sprintf("--admin-addr=0.0.0.0:%s", testnet.AdminPort),
``` [5](#0-4) 

The registered privileged commands include:

- `set-config` — can set `consensus-required-approvals-for-sealing` to `0`, bypassing verification node approval requirements for sealing
- `stop-at-height` — can crash or halt an execution node at an attacker-chosen block height
- `ingest-tx-rate-limit` — can add/remove addresses from the collection node rate-limit allowlist
- `trigger-checkpoint` — can force WAL checkpoint creation [6](#0-5) [7](#0-6) [8](#0-7) 

### Impact Explanation

An attacker who can reach the admin port (e.g., via a misconfigured firewall, a co-located container, or a compromised internal network segment) can:

1. **Weaken consensus integrity**: `set-config {"consensus-required-approvals-for-sealing": 0}` removes the requirement for verification node approvals before sealing, allowing execution results to be sealed without independent verification — a direct protocol integrity violation.
2. **Halt execution nodes**: `stop-at-height` with `crash: true` terminates the execution node process at an attacker-chosen height, halting block execution for that node.
3. **Bypass transaction rate limits**: `ingest-tx-rate-limit` can remove rate-limit entries, allowing previously throttled accounts to flood the collection node mempool.

The root cause is identical to the reported Solidity vulnerability: a privileged state-mutating function is callable by any external caller with no authorization check. The `runCommand` function is the direct analog of the unguarded `proposeUpdateTransmitters`.

### Likelihood Explanation

The admin server binds to `0.0.0.0:9002` by default in both localnet and testnet configurations, and the same default is used in integration test network setups. Production operators who do not explicitly configure all three TLS flags run an unauthenticated HTTP server. Any attacker with network access to port 9002 — including a co-located process, a container on the same Docker network, or a host on the same subnet with a misconfigured firewall — can exploit this without any credentials.

### Recommendation

Require authentication for all admin server requests. The minimal fix is to enforce that the HTTP server **always** requires mutual TLS client certificate authentication (not opt-in), or add an authentication middleware (e.g., a shared secret token checked in `runCommand`) that rejects unauthenticated callers before any command handler is invoked. The gRPC server should also add an authentication interceptor rather than relying solely on Unix socket filesystem permissions for the local path.

### Proof of Concept

```bash
# No credentials required. Any network-reachable client can weaken consensus:
curl http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config", "data": {"consensus-required-approvals-for-sealing": 0}}'

# Or crash an execution node:
curl http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "stop-at-height", "data": {"height": 1000, "crash": true}}'
```

The `runCommand` dispatcher at `admin/command_runner.go:304` will execute either command without checking the caller's identity. [3](#0-2)

func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
	r.logger.Info().Str("command", command).Msg("received new command")

	req := &CommandRequest{Data: data}

	if validator := r.getValidator(command); validator != nil {
		if validationErr := validator(req); validationErr != nil {
			// for expected validation errors, return code InvalidArgument and the error text
			if IsInvalidAdminParameterError(validationErr) {
				return nil, status.Error(codes.InvalidArgument, validationErr.Error())
			}
			// for unexpected errors, return code Internal and log a warning
			r.logger.Err(validationErr).Msg("unexpected error validating admin request")
			return nil, status.Error(codes.Internal, validationErr.Error())
		}
	}

	var handleResult any
	var handleErr error

	if handler := r.getHandler(command); handler != nil {
		if handleResult, handleErr = handler(ctx, req); handleErr != nil {
			if errors.Is(handleErr, context.Canceled) {
				return nil, status.Error(codes.Canceled, "client canceled")
			} else if errors.Is(handleErr, context.DeadlineExceeded) {
				return nil, status.Error(codes.DeadlineExceeded, "request timed out")
			} else {
				r.logger.Err(handleErr).Msg("unexpected error handling admin request")
				s, _ := status.FromError(handleErr)
				return nil, s.Err()
			}
		}
	} else {
		return nil, status.Error(codes.Unimplemented, "invalid command")
	}

	return handleResult, nil
}

	validatedReq := req.ValidatorData.(validatedSetConfigData)

	oldValue := validatedReq.field.Get()

	err := validatedReq.field.Set(validatedReq.value)
	if err != nil {
		if updatable_configs.IsValidationError(err) {
			return nil, fmt.Errorf("config update failed due to invalid input: %w", err)
		}
		return nil, fmt.Errorf("unexpected error setting config field %s: %w", validatedReq.field.Name, err)

	res := map[string]any{
		"oldValue": oldValue,
		"newValue": validatedReq.value,
	}
