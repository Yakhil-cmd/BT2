Audit Report

## Title
Missing `http2_max_header_list_size` cap on SOCKS-proxied HTTP/2 client allows malicious server to exhaust adapter memory — (`rs/https_outcalls/adapter/src/rpc_server.rs`)

## Summary
`CanisterHttp::new` sets `.http2_max_header_list_size(MAX_HEADER_LIST_SIZE)` (52 KiB) on the direct HTTPS client, but `create_socks_proxy_client` omits this call entirely, leaving the `h2` crate's default (`u32::MAX`) in place. Any canister that issues an HTTPS outcall to an IPv4-only attacker-controlled server will trigger the SOCKS fallback path, where the server can send arbitrarily large HTTP/2 HEADERS frames that the adapter buffers without bound, potentially exhausting adapter process memory.

## Finding Description
`MAX_HEADER_LIST_SIZE` is defined as `52 * 1024` at line 50. [1](#0-0) 

The direct client at lines 88–90 applies the cap: [2](#0-1) 

`create_socks_proxy_client` at lines 119–127 builds a `Client` without `.http2_max_header_list_size(...)`: [3](#0-2) 

The SOCKS path is invoked as a fallback whenever the direct connection fails (lines 336–365), which is the normal case for IPv4-only destinations: [4](#0-3) 

The `h2` crate advertises `SETTINGS_MAX_HEADER_LIST_SIZE` to the server and enforces the limit on received HEADERS frames. With the cap set, oversized headers produce a stream-level `PROTOCOL_ERROR` before full allocation; without it (effectively `u32::MAX`), `h2` allocates the full header block. The SOCKS client is cached per proxy address (`get_socks_client`, lines 131–147), so the uncapped client persists across requests. [5](#0-4) 

## Impact Explanation
Repeated requests to an attacker-controlled IPv4-only HTTPS server drive the adapter process toward OOM. An adapter crash on a replica causes that replica to fail all HTTPS outcall attempts until the process restarts. Because HTTPS outcalls require a threshold of replicas to agree, targeting enough replicas' adapters can degrade or stall the feature subnet-wide. This matches the allowed High impact: **Application/platform-level DoS, crash, or subnet availability impact not based on raw volumetric DDoS** ($2,000–$10,000).

## Likelihood Explanation
The attacker only needs to operate an HTTPS server on an IPv4-only address and have any canister issue an outcall to it. No privileged access, key material, or network-level attack is required. The SOCKS fallback is the documented normal path for IPv4 destinations. The attack is repeatable and low-cost.

## Recommendation
Add `.http2_max_header_list_size(MAX_HEADER_LIST_SIZE)` to the `Client::builder` call inside `create_socks_proxy_client`, mirroring the direct client:

```rust
Client::builder(TokioExecutor::new())
    .http2_max_header_list_size(MAX_HEADER_LIST_SIZE)   // add this
    .build::<_, Full<Bytes>>(
        builder
            .enable_all_versions()
            .wrap_connector(SocksConnector { … }),
    )
```

## Proof of Concept
1. Stand up an HTTPS server (self-signed cert accepted by the adapter) on an IPv4-only address.
2. Configure it to respond with an HTTP/2 HEADERS frame containing ~200 KiB of header data.
3. Have a canister issue an HTTPS outcall to that server.
4. The direct connection fails (no IPv4 on the replica interface); `do_https_outcall_socks_proxy` is invoked.
5. `get_socks_client` → `create_socks_proxy_client` returns a client with no header size cap.
6. `h2` buffers the full 200 KiB header block; the adapter returns no error.
7. Repeat to drive memory usage up; compare with the direct path, which returns a `PROTOCOL_ERROR` at 52 KiB.
8. A deterministic integration test can be written using a local mock HTTPS server (e.g., `hyper` server with a custom `h2` frame writer) and `PocketIC` to issue the outcall, asserting that the SOCKS path does not error while the direct path does.

```rust
    fn get_socks_client(
        &self,
        socks_proxy_uri: Uri,
    ) -> Client<HttpsConnector<SocksConnector<HttpConnector>>, OutboundRequestBody> {
        let cache_guard = self.cache.upgradable_read();

        if let Some(client) = cache_guard.get(&socks_proxy_uri.to_string()) {
            client.clone()
        } else {
            let mut cache_guard = RwLockUpgradableReadGuard::upgrade(cache_guard);
            self.metrics.socks_cache_misses.inc();
            let client = self.create_socks_proxy_client(socks_proxy_uri.clone());
            cache_guard.insert(socks_proxy_uri.to_string(), client.clone());
            self.metrics.socks_cache_size.set(cache_guard.len() as i64);
            client
        }
    }
```
