### Title
`MsgTurnBridge` Handler Is Completely Unimplemented, Disabling the Documented Emergency Bridge-Disable Mechanism - (File: `x/cronos/keeper/msg_server.go`)

### Summary

`MsgTurnBridge` is the documented emergency mechanism to disable or enable the Gravity bridge in Cronos. The proto service, the `CanTurnBridge` permission constant, and the permission-check infrastructure are all fully wired. However, the actual message handler in `x/cronos/keeper/msg_server.go` is a stub that unconditionally returns `nil, nil` — it performs no permission check, writes no state, and silently succeeds for any caller. The bridge can never be disabled through this path.

---

### Finding Description

ADR-009 explicitly identifies `MsgTurnBridge` as one of the two restricted messages whose purpose is "disabling the bridge module in case of emergency":

> "For example, changing the mapping to support new CRC20 auto-conversion contracts or **disabling the bridge module in case of emergency**."

The proto definition confirms the intent:

```proto
// TurnBridge defines a method to disable or enable the gravity bridge
rpc TurnBridge(MsgTurnBridge) returns (MsgTurnBridgeResponse);
```

The `MsgTurnBridge` message carries a `bool enable` field, clearly designed to toggle bridge state on/off.

The permission constant `CanTurnBridge` is defined and the `HasPermission` guard is fully functional — it is correctly used by `UpdateTokenMapping`. But the `TurnBridge` handler itself is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

Two concrete defects:

1. **No permission check** — `HasPermission(ctx, ..., CanTurnBridge)` is never called. Any address can submit `MsgTurnBridge` and receive a success response, completely bypassing the `CanTurnBridge` permission system.
2. **No state change** — No bridge-enabled flag is read or written. The `bool enable` field in the message is ignored. The bridge state is never modified regardless of who calls it or what value they pass.

Compare with `UpdateTokenMapping`, which correctly guards its action:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
```

`TurnBridge` has no equivalent guard and no equivalent action.

---

### Impact Explanation

**High — Bypass of Cronos admin bridge authorization checks; permanent inability to exercise the documented emergency bridge-disable control.**

If a bridge exploit is in progress (e.g., a vulnerability in IBC voucher conversion, gravity bridge relay logic, or CRC20/CRC21 minting), the `CronosAdmin` or any address holding `CanTurnBridge` permission has no functional path to halt the bridge. The only documented emergency stop for the bridge is `MsgTurnBridge`, and it is a no-op. Recovery would require a full governance parameter update (`MsgUpdateParams`), which carries governance voting delay and is far slower than the intended single-admin emergency action.

Additionally, because there is no permission check, the `CanTurnBridge` permission system is entirely bypassed — any unprivileged address can call `TurnBridge` and receive a success response, creating false confidence that the bridge state was changed.

---

### Likelihood Explanation

The entry path is fully reachable: `MsgTurnBridge` is a registered gRPC message, exposed via the standard Cosmos SDK tx pipeline, with no ante-handler or other guard blocking it. Any unprivileged address can submit it. The admin's inability to disable the bridge is permanent until the handler is implemented.

---

### Recommendation

Implement the `TurnBridge` handler to:
1. Check that the sender holds `CanTurnBridge` permission via `HasPermission`.
2. Persist the `enable` boolean into module params (or a dedicated KV key).
3. Have the IBC middleware and gravity bridge hooks read that flag and reject operations when the bridge is disabled.

Example skeleton:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    params := k.GetParams(ctx)
    params.EnableBridge = msg.Enable   // persist bridge on/off state
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

---

### Proof of Concept

1. Any address submits `MsgTurnBridge{sender: <any>, enable: false}`.
2. The handler returns `nil, nil` — success, no error.
3. The bridge continues to process IBC voucher conversions and gravity relay events unchanged.
4. The `CronosAdmin` submits the same message — same result: silent no-op.
5. No bridge state is ever toggled; the `CanTurnBridge` permission is never evaluated.
