### Title
Gateway Stateful Validator Admits Transactions With Insufficient L1/L1_data Gas Price Bounds That Will Fail at Blockifier Pre-Validation - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The gateway's stateful resource-bounds check only validates the L2 gas price against the previous block's L2 gas price. It explicitly skips L1 gas and L1 data gas price validation. A transaction with `l1_gas.max_price_per_unit = 0` (or below the actual block L1 gas price) but a valid `l2_gas.max_price_per_unit` passes all gateway checks and is admitted to the mempool, but then fails at blockifier `perform_pre_validation_stage` with `MaxGasPriceTooLow` for L1Gas. The gateway's admission decision is structurally incapable of enforcing the L1 fee requirement that the blockifier enforces downstream — a direct analog to the Wormhole bridge connector lacking `@payable` while calling a `payable` function.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` calls `validate_tx_l2_gas_price_within_threshold`, which contains an explicit no-op branch for the L1 gas dimension:

```rust

The stateless validator similarly only checks `l2_gas.max_price_per_unit` against a static `min_gas_price`, with no check on `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`:

Downstream, `AccountTransaction::perform_pre_validation_stage` calls `check_fee_bounds` when `charge_fee` is true. For `AllResources` bounds, it checks **all three** gas prices against actual block prices: [3](#0-2) [4](#0-3) [5](#0-4) 

The `enforce_fee` flag (and thus `charge_fee`) is `true` whenever `max_possible_fee > 0`. A transaction with `l2_gas = {max_amount: 1, max_price_per_unit: min_gas_price}` and `l1_gas = {max_amount: 0, max_price_per_unit: 0}` has `max_possible_fee > 0`, so `charge_fee = true`, and `check_fee_bounds` runs and fails for L1Gas because `0 < actual_l1_gas_price`.

### Impact Explanation

The gateway's admission invariant — "admitted transactions should be executable" — is broken. Transactions with `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` below the actual block price pass both stateless and stateful gateway validation, enter the mempool, and are then discarded by the batcher at pre-validation. This matches **High: Mempool/gateway/RPC admission accepts invalid transactions before sequencing**.

### Likelihood Explanation

Any unprivileged user can craft a V3 `AllResources` transaction with a valid `l2_gas.max_price_per_unit` (meeting the gateway threshold) but `l1_gas.max_price_per_unit = 0`. The gateway has no mechanism to detect this. The L1 gas price is non-zero in any live network block, so these transactions will always fail at blockifier pre-validation. This is trivially reproducible and requires no special privileges.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also validate `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding prices, mirroring the L2 gas price check already in place. The previous block's `l1_gas_price` and `l1_data_gas_price` are already available from `get_block_info()` in the same code path. [7](#0-6) 

### Proof of Concept

1. Read the previous block's `l1_gas_price` (e.g., `P_l1 = 100`).
2. Submit a V3 invoke transaction with:
   - `l2_gas = { max_amount: 1, max_price_per_unit: min_gas_price }` (passes stateless + stateful L2 check)
   - `l1_gas = { max_amount: 0, max_price_per_unit: 0 }` (zero price, zero amount)
   - `l1_data_gas = { max_amount: 0, max_price_per_unit: 0 }`
3. Gateway stateless check: `max_possible_fee = 1 * min_gas_price > 0` ✓; `l2_gas.max_price_per_unit >= min_gas_price` ✓ → **admitted**.
4. Gateway stateful check: `l2_gas.max_price_per_unit >= threshold` ✓; L1 gas price not checked → **admitted to mempool**.
5. Batcher calls `perform_pre_validation_stage`: `charge_fee = true` (max_possible_fee > 0); `check_fee_bounds` checks `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (P_l1 = 100)` → `MaxGasPriceTooLow` for L1Gas → **transaction discarded**.

The gateway admitted a transaction that the blockifier will always reject, wasting mempool and batcher resources. An attacker can repeat this at scale to degrade sequencer throughput. [8](#0-7) [9](#0-8)

        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,

        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);

                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
