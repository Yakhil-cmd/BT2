### Title
Unauthorized Transfer and Burn of Native Bank Tokens via Unauthenticated `sender` in Bank Precompile — (`x/cronos/keeper/precompiles/bank.go`)

### Summary
The `transfer` and `burn` methods of the bank precompile accept an arbitrary address from calldata as the token source (`sender` / `recipient`) without verifying that the EVM caller is authorized to act on behalf of that address. Any unprivileged contract can drain or destroy another account's native bank tokens of the denom `evm/<callerContractAddress>`.

### Finding Description
In `BankContract.Run`, the `TransferMethodName` case unpacks a `sender` address directly from calldata and passes it as the `from` argument to `bankKeeper.SendCoins`:

```go
// x/cronos/keeper/precompiles/bank.go  lines 175–192
sender := args[0].(common.Address)
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())          // "evm/0x<callerContract>"
...
bc.bankKeeper.SendCoins(ctx, from, to, ...)   // no check: caller == sender
```

There is no guard asserting `contract.Caller() == sender`. The denom is derived from the calling contract's address (`contract.Caller()`), so the attacker controls both the denom and the `from` address.

The same pattern exists for `BurnMethodName`: `recipient` (the address whose tokens are burned) is taken from calldata with no check that `contract.Caller() == recipient`:

```go
// lines 121–144
recipient := args[0].(common.Address)
...
addr := sdk.AccAddress(recipient.Bytes())
...
bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, ...)  // burns addr's tokens
``` [1](#0-0) [2](#0-1) 

### Impact Explanation
**Critical — Unauthorized transfer/burn of EVM-backed native bank tokens.**

Any user who has converted EVM tokens to native bank tokens (e.g., via `bank.mint(msg.sender, amount)` as shown in `TestBank.sol`) holds a balance of denom `evm/0x<contractAddress>`. An attacker deploys a contract at address `A` and calls:

```
bank.transfer(victim, attacker, victimBalance)
```

The precompile derives denom `evm/0xA`, then calls `bankKeeper.SendCoins(victim, attacker, ...)` with no authorization check. The victim's entire native balance of that denom is transferred to the attacker.

Similarly, `bank.burn(victim, amount)` destroys the victim's tokens with no authorization. [3](#0-2) [4](#0-3) 

### Likelihood Explanation
The bank precompile is reachable by any unprivileged EVM transaction. The attacker only needs to deploy a contract and call the precompile at address `0x0000000000000000000000000000000000000064`. The only precondition is that the victim holds a native bank balance of the denom `evm/<attackerContractAddress>`, which is the normal state for any user who has used the `moveToNative`-style pattern. [5](#0-4) 

### Recommendation
Add an authorization check in both `TransferMethodName` and `BurnMethodName` to ensure the EVM caller is only permitted to act on its own behalf:

```go
// TransferMethodName
if contract.Caller() != sender {
    return nil, errors.New("caller is not authorized to transfer on behalf of sender")
}

// BurnMethodName
if contract.Caller() != recipient {
    return nil, errors.New("caller is not authorized to burn on behalf of recipient")
}
```

This mirrors the fix in the external report: use the authenticated identity (`address(this)` / `contract.Caller()`) rather than a caller-supplied address. [6](#0-5) 

### Proof of Concept

1. Deploy attacker contract `Exploit` at address `0xATTACKER`.
2. Victim calls `bank.mint(victim, 1000)` from `Exploit` (or any contract at `0xATTACKER`), giving victim a balance of `evm/0xATTACKER = 1000`.
3. Attacker calls from `Exploit`:
   ```solidity
   IBankModule(0x64).transfer(victim, attacker, 1000);
   ```
4. The precompile computes `denom = "evm/0xATTACKER"`, then executes `bankKeeper.SendCoins(victim, attacker, 1000 evm/0xATTACKER)` — no authorization check is performed.
5. Victim's balance is drained to zero; attacker receives 1000 tokens.

The same call with `burn(victim, 1000)` destroys the victim's tokens entirely. [7](#0-6)

```go
func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
```
