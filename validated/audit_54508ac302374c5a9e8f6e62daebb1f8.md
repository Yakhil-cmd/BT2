### Title
Floored Threshold in `validate_tx_l2_gas_price_within_threshold` Admits Transactions Below the Intended Minimum Gas Price - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price by multiplying the previous block's gas price by a configured percentage and then calling `.to_integer()` on the resulting `Ratio`, which **floors** (truncates) the result. This makes the enforced threshold strictly lower than the intended minimum, allowing transactions whose `max_price_per_unit` is below the true threshold to pass the gateway's stateful admission check.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, the threshold is computed as:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();   // ← floors the rational result
if tx_l2_gas_price.0 < threshold { ... }
```

`Ratio::to_integer()` (from the `num_rational` crate) returns `⌊numerator / denominator⌋`. Whenever `min_gas_price_percentage × prev_price` is not exactly divisible by 100, the computed `threshold` is one FRI unit lower than the true rational threshold.

**Concrete example:**
- `previous_block_l2_gas_price = 8_000_000_001` FRI (just above 8 gwei)
- `min_gas_price_percentage = 10`
- True threshold = `10 × 8_000_000_001 / 100 = 800_000_000.1` FRI
- Computed threshold = `⌊800_000_000.1⌋ = 800_000_000` FRI
- A transaction with `max_price_per_unit = 800_000_000` passes (`800_000_000 >= 800_000_000`), but the intended policy requires `> 800_000_000.0`, so it should be rejected.

The maximum per-unit error is `(100 − 1) / 100 < 1` FRI, occurring whenever `min_gas_price_percentage × prev_price mod 100 ≠ 0`.

### Impact Explanation

The gateway's stateful path is the last admission gate before a transaction enters the mempool. By rounding the threshold down, the check admits transactions whose offered gas price is below the operator-configured minimum fraction of the previous block's L2 gas price. This breaks the invariant that `tx_l2_gas_price ≥ (min_gas_price_percentage / 100) × prev_price` and maps to the **High** impact category: *Mempool/gateway/RPC admission accepts invalid transactions before sequencing*.

In production the default `min_gas_price_percentage` is 100, which yields an exact integer and no rounding error. The bug is latent but activates whenever the operator sets a non-100 percentage whose product with the current gas price is not divisible by 100 — a common scenario for any odd gas price value.

### Likelihood Explanation

- The default configuration (`min_gas_price_percentage = 100`) is immune, so the bug is dormant in the standard deployment.
- Any operator-configured percentage other than 100 (e.g., 80, 50, 10) combined with a gas price not divisible by `100 / gcd(percentage, 100)` triggers the rounding error.
- Any unprivileged user can craft a transaction with `max_price_per_unit` equal to the floored threshold to exploit the weakened check.

### Recommendation

Replace `.to_integer()` (floor) with ceiling division so the enforced threshold is never lower than the intended minimum:

```rust
// Instead of:
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0)
    .to_integer();

// Use ceiling:
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

Alternatively, avoid `Ratio` entirely and use integer arithmetic with ceiling division:

```rust
let threshold = previous_block_l2_gas_price.get().0
    .saturating_mul(u128::from(self.config.min_gas_price_percentage))
    .div_ceil(100);
```

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
