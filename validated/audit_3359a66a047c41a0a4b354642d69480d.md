### Title
Missing Access Control on `update_rewards` Allows Any Staker to Steal Block Rewards — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `staking.cairo` is specified to be callable only by the Starknet sequencer, but the implementation enforces no such restriction. Any registered staker can invoke it to claim block rewards for themselves without having attested, stealing yield that should accrue to the legitimately attesting staker.

---

### Finding Description

The protocol specification explicitly states:

> **access control: Only starkware sequencer.**

However, the implementation of `update_rewards` at `src/staking/staking.cairo` lines 1449–1500 contains no caller check beyond `general_prerequisites()`, which only asserts the contract is unpaused and the caller is non-zero:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: unpaused + non-zero caller
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global, one write per block
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distributes block rewards to staker_address
    ...
}
``` [1](#0-0) 

The spec-mandated "only sequencer" guard is entirely absent. [2](#0-1) 

`general_prerequisites()` only checks pause state and zero-address:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [3](#0-2) 

The global `last_reward_block` variable enforces that only **one** call to `update_rewards` succeeds per block. Whoever calls it first for a given block wins the rewards for that block. The caller supplies the `staker_address` that receives the rewards — there is no validation that the supplied address is the staker that actually attested.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

An attacker who is a registered staker with effective balance (i.e., K epochs after staking) can:

1. Call `update_rewards(staker_address: attacker_address, disable_rewards: false)` for any block in which the sequencer has not yet called it.
2. Receive the full block's STRK rewards (and pool rewards if the attacker has a pool) without having submitted any attestation.
3. The sequencer's subsequent call for the legitimate attesting staker reverts with `REWARDS_ALREADY_UPDATED`, permanently denying that staker their yield for that block.

The rewards are minted from the `RewardSupplier` and transferred to the attacker's `unclaimed_rewards_own`, then claimable via `claim_rewards`. [4](#0-3) 

---

### Likelihood Explanation

**Medium.**

- Any address that has staked the minimum amount and waited K epochs qualifies as an attacker — no special privilege is required.
- The sequencer controls transaction ordering on Starknet and can front-run the attacker for blocks where it actively calls `update_rewards`. However, the sequencer only calls `update_rewards` for blocks in which a staker attested. For any block where no attestation occurs (or where the sequencer is delayed), the window is open.
- The attacker does not need to compromise any trusted role; they only need to submit a valid transaction in a block where `last_reward_block < current_block_number`.

---

### Recommendation

Add an explicit access-control guard to `update_rewards` that restricts the caller to the authorized sequencer address (stored in contract storage and settable by governance), mirroring the pattern used for `update_rewards_from_attestation_contract`, which correctly enforces `assert_caller_is_attestation_contract()`: [5](#0-4) 

A similar `assert_caller_is_sequencer()` check should be added at the top of `update_rewards`.

---

### Proof of Concept

1. Attacker calls `stake(reward_address, operational_address, min_stake)` on `staking.cairo`.
2. Attacker waits K epochs for their balance to become effective in the trace.
3. In any block where `get_block_number() > last_reward_block` and consensus rewards are active, attacker calls:
   ```
   update_rewards(staker_address: attacker_address, disable_rewards: false)
   ```
4. `_update_rewards` executes: calculates block rewards proportional to attacker's stake, increments `attacker_info.unclaimed_rewards_own`, transfers pool rewards, and updates `RewardSupplier.unclaimed_rewards`. [6](#0-5) 
5. `last_reward_block` is set to the current block number. [7](#0-6) 
6. The sequencer's call for the legitimate attesting staker reverts with `REWARDS_ALREADY_UPDATED`.
7. Attacker calls `claim_rewards(attacker_address)` to withdraw the stolen yield. [8](#0-7)

        /// **Note**: `curr_epoch` must be `get_current_epoch()`, it's passed as a param to save
        /// storage reads.
        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
