### Title
Missing Zero-Address Check in `change_reward_address` Allows Permanent Burning of Unclaimed Yield - (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

Both the staking contract and the pool contract allow a staker or pool member to set their `reward_address` to the zero address (`0x0`). There is no zero-address guard in `change_reward_address` (or in the initial `stake` call). When rewards are subsequently claimed, `checked_transfer` sends the accumulated yield to address zero, permanently destroying it.

---

### Finding Description

**In `src/staking/staking.cairo`**, `change_reward_address` validates only that the new address is not a registered token address. There is no `is_non_zero()` guard:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    assert!(
        !self.does_token_exist(token_address: reward_address),
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    // ← no zero-address check
    let staker_address = get_caller_address();
    let mut staker_info = self.internal_staker_info(:staker_address);
    staker_info.reward_address = reward_address;
    self.write_staker_info(:staker_address, :staker_info);
    ...
}
``` [1](#0-0) 

The same pattern exists in the initial `stake` entrypoint, which also only checks `REWARD_ADDRESS_IS_TOKEN`: [2](#0-1) 

**In `src/pool/pool.cairo`**, `change_reward_address` has the identical gap — only the token-address check, no zero-address check:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    assert!(
        self.token_dispatcher.contract_address.read() != reward_address,
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    // ← no zero-address check
    let pool_member = get_caller_address();
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    pool_member_info.reward_address = reward_address;
    self.write_pool_member_info(:pool_member, :pool_member_info);
    ...
}
``` [3](#0-2) 

Once `reward_address` is zero, every downstream reward transfer path sends tokens to `0x0`:

- `send_rewards_to_staker` → `token_dispatcher.checked_transfer(recipient: reward_address, ...)` [4](#0-3) 
- `pool::claim_rewards` → `reward_token.checked_transfer(recipient: reward_address, ...)` [5](#0-4) 

`send_rewards_to_staker` is also called unconditionally inside `unstake_action`, so even a staker who forgets they set `reward_address = 0` will have all accumulated rewards burned at exit time.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

The STRK token (OpenZeppelin ERC20 on Starknet) does not revert on `transfer` to address zero; it silently burns the tokens. Any unclaimed STRK rewards accumulated by the staker or pool member are permanently destroyed and irrecoverable. This matches the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

---

### Likelihood Explanation

The entry path requires only the staker or pool member themselves — an unprivileged role — to call `change_reward_address(ContractAddress { 0 })`. This can happen:

1. **Accidentally** — a user passes a zero address by mistake (e.g., a front-end bug or a scripting error).
2. **Intentionally as self-griefing** — a user who wants to forfeit rewards (e.g., to avoid tax events) can do so with no protocol-level protection.

No privileged role, no bridge compromise, and no external dependency is required. The call is directly reachable by any registered staker or pool member.

---

### Recommendation

Add an `is_non_zero()` assertion in both `change_reward_address` implementations and in the `stake` / `enter_delegation_pool` entrypoints before accepting a `reward_address`:

```cairo
assert!(reward_address.is_non_zero(), "Reward address cannot be zero");
```

This mirrors the mitigation applied in the referenced Badger Finance report: add an explicit zero-check before any address is stored as a reward destination.

---

### Proof of Concept

**Staker path:**
1. Staker calls `staking.change_reward_address(reward_address: 0)`. Passes the only guard (`REWARD_ADDRESS_IS_TOKEN`). [1](#0-0) 
2. Staker (or anyone) calls `staking.claim_rewards(staker_address)`. Access control passes because `caller == staker_address`. [6](#0-5) 
3. `send_rewards_to_staker` executes `checked_transfer(recipient: 0, amount: unclaimed_rewards)`. All accumulated STRK rewards are sent to address zero and permanently burned. [7](#0-6) 

**Pool member path:**
1. Pool member calls `pool.change_reward_address(reward_address: 0)`. Passes the only guard. [8](#0-7) 
2. Pool member calls `pool.claim_rewards(pool_member)`. Access control passes because `caller == pool_member`. [9](#0-8) 
3. `reward_token.checked_transfer(recipient: 0, amount: rewards)` burns all accumulated STRK rewards. [5](#0-4)
