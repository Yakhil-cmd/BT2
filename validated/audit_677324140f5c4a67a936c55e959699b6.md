### Title
Gateway L2 Gas Price Admission Check Uses Stale Current-Block Price Instead of Next-Block Execution Price — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful `validate_resource_bounds` reads the **current committed block's** L2 gas price as the admission threshold, but every transaction it admits will be executed in the **next** block, which carries a different L2 gas price computed by the EIP-1559 fee-market. The code itself acknowledges the mismatch with a TODO comment. The result is a systematic split between what the gateway accepts and what the blockifier will actually allow at execution time.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` fetches the block info from `gateway_fixed_block_state_reader.get_block_info()` and names the result `previous_block_l2_gas_price`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold check then accepts any transaction whose `max_price_per_unit` ≥ `min_gas_price_percentage/100 × current_block_price`: [2](#0-1) 

`run_validate_entry_point` also builds its `BlockContext` from the same current-block info (only bumping the block number, not the gas prices):

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
``` [3](#0-2) 

So `check_fee_bounds` inside `perform_pre_validation_stage` also runs against the current block's prices: [4](#0-3) 

The batcher, however, builds its `BlockContext` from `propose_block_input.block_info`, which carries the **next block's** gas prices — the value computed by `calculate_next_l2_gas_price_for_fin` and placed in `ProposalInit.l2_gas_price_fri`: [5](#0-4) [6](#0-5) 

The next-block price is computed by the EIP-1559 formula and can differ from the current-block price by up to `price / gas_price_max_change_denominator` per block: [7](#0-6) 

### Impact Explanation

**Case A — price rising (gas usage above target):** A transaction submitted with `max_price_per_unit = current_block_price` passes both the gateway pre-check and the gateway blockifier validation (both use current prices). It is admitted to the mempool. When the batcher executes it, `check_fee_bounds` compares `max_price_per_unit` against `next_block_price > current_block_price` and rejects the transaction at pre-validation. The mempool fills with transactions that are structurally invalid for the block they will be sequenced in.

**Case B — price falling (gas usage below target):** A transaction with `max_price_per_unit = next_block_price` (which is lower than `current_block_price`) is rejected by the gateway's threshold check even though it would pass `check_fee_bounds` at execution time. Valid transactions are denied admission.

Both cases match the allowed High impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The condition is triggered whenever the EIP-1559 fee market moves the L2 gas price between blocks — a normal, continuous occurrence under any non-trivial load. With `min_gas_price_percentage = 100` (the default), the window equals the full per-block price delta. The trigger requires no privilege: any unprivileged user submitting a standard V3 invoke transaction with `max_price_per_unit` set to the current block's price will hit Case A whenever the network is busy.

### Recommendation

Replace the current-block price lookup with the `next_l2_gas_price` field already stored in the block header (as the TODO comment itself prescribes). The `BlockHeaderWithoutHash` struct already carries `next_l2_gas_price`:

```rust
// Instead of:
let previous_block_l2_gas_price = ...get_block_info()...strk_gas_prices.l2_gas_price;
// Use:
let next_l2_gas_price = ...get_block_header()...next_l2_gas_price;
```

Similarly, `run_validate_entry_point` should populate the `BlockContext` gas prices with the next-block prices so that the gateway's blockifier validation is consistent with what the batcher will enforce.

### Proof of Concept

1. Observe the current committed block's STRK L2 gas price `P_cur` and the `next_l2_gas_price` field `P_next` stored in its header (available via any block-header RPC call). Confirm `P_next > P_cur` (network is above gas target).
2. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = P_cur`. The gateway's `validate_tx_l2_gas_price_within_threshold` computes `threshold = 100% × P_cur = P_cur` and accepts the transaction. `run_validate_entry_point` also passes because it uses `P_cur` for `check_fee_bounds`.
3. The transaction enters the mempool.
4. When the batcher calls `propose_block` for the next block, it sets `block_info.gas_prices.strk_gas_prices.l2_gas_price = P_next`. Inside `perform_pre_validation_stage → check_fee_bounds`, the condition `max_price_per_unit (= P_cur) < actual_gas_price (= P_next)` is true, and the transaction is rejected with `ResourceBoundsError::MaxGasPriceTooLow`.
5. The transaction is dropped from the block despite having passed all gateway checks — confirming that the gateway admitted an invalid transaction. [8](#0-7) [9](#0-8)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
