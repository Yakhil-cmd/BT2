### Title
Unprivileged Caller Can Front-Run `update_rewards` with `disable_rewards: true` to Permanently Deny Block Rewards - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` has no access control. Any address can call it with `disable_rewards: true` for any active staker. Because the global `last_reward_block` is written **before** the `disable_rewards` branch is evaluated, a front-running attacker can consume the per-block reward slot without distributing rewards, causing the legitimate consensus-layer call to revert with `REWARDS_ALREADY_UPDATED` and the targeted staker to lose all block rewards for that block.

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` is the consensus-era reward distribution entry point. It is callable by any address with no role or caller check: [1](#0-0) 

The function writes the current block number to the global `last_reward_block` storage slot unconditionally, before branching on `disable_rewards`: [2](#0-1) 

When `disable_rewards = true` (or `is_pre_consensus()` is true), the function returns immediately after updating `last_reward_block`, distributing nothing. Any subsequent call in the same block — including the legitimate sequencer/consensus call with `disable_rewards: false` — hits the guard: [3](#0-2) 

and reverts with `REWARDS_ALREADY_UPDATED`.

The interface confirms there is no access control: [4](#0-3) 

The `disable_rewards` parameter is intended to let the consensus layer signal "this staker did not perform their duties this block." An unprivileged caller can abuse it to signal that for any staker, every block.

### Impact Explanation

**High — Theft of unclaimed yield.**

In the consensus rewards era, each block the sequencer calls `update_rewards(block_producer, disable_rewards: false)` to credit the block producer with their proportional share of the epoch's STRK (and BTC) block rewards. An attacker who front-runs this call with `update_rewards(victim, disable_rewards: true)` causes:

1. `last_reward_block` is set to the current block number.
2. No rewards are distributed.
3. The legitimate sequencer call reverts.
4. The staker permanently loses the yield for that block (there is no catch-up mechanism).

Repeated every block, this drains 100 % of a targeted staker's consensus-era yield. Pool delegators whose rewards flow through the staker also lose their proportional share. [5](#0-4) 

### Likelihood Explanation

**High.** The function is public, requires no stake or special role, and the only cost to the attacker is gas. The attack is mechanically simple: submit a transaction with `disable_rewards: true` targeting any active staker before the sequencer's transaction lands. On Starknet, transaction ordering within a block is controlled by the sequencer, but the function is also callable by L2 users in normal transactions. A motivated attacker (e.g., a competing validator) can sustain this indefinitely.

### Recommendation

Add an access-control guard so that only the designated consensus/sequencer address (or a trusted role) may call `update_rewards`. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_consensus_contract(); // <-- add this
    ...
}
```

Alternatively, restrict the `disable_rewards = true` path to a privileged caller while allowing the `disable_rewards = false` path to remain open, if open participation is desired.

### Proof of Concept

```
Epoch N, consensus rewards active.
Block B is produced by StakerX.

1. Attacker calls:
   update_rewards(staker_address: StakerX, disable_rewards: true)
   → last_reward_block := B
   → no rewards distributed (early return at line 1487)

2. Sequencer calls:
   update_rewards(staker_address: StakerX, disable_rewards: false)
   → assert!(B > last_reward_block.read())  // B > B is FALSE
   → PANIC: REWARDS_ALREADY_UPDATED

3. StakerX receives 0 block rewards for block B.
   Pool delegators of StakerX also receive 0 rewards for block B.

Repeat every block → StakerX earns 0 consensus rewards for the entire epoch.
``` [2](#0-1) [3](#0-2)
