### Title
Any Unprivileged Caller Can Invoke `update_rewards` with `disable_rewards: true` to Permanently Suppress Per-Block Reward Distribution — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the Staking contract enforces a strict one-call-per-block limit via `last_reward_block`, but imposes **no access control** on who may call it or what value they pass for `disable_rewards`. An attacker can front-run every block by calling `update_rewards(valid_staker, disable_rewards: true)`, consuming the block's reward slot without distributing any rewards. All subsequent legitimate calls in that block revert with `REWARDS_ALREADY_UPDATED`, and the block's rewards are permanently lost.

---

### Finding Description

`update_rewards` is the entry point for consensus-phase (V3) per-block reward distribution: [1](#0-0) 

The function's only gate is `general_prerequisites()`, which checks only that the contract is not paused and the caller is non-zero: [2](#0-1) 

There is no role check — no `only_operator`, no `only_staker`, nothing. Any EOA or contract may call it.

After passing `general_prerequisites`, the function asserts the current block has not yet been processed and immediately writes `last_reward_block`: [3](#0-2) 

Then, if `disable_rewards` is `true`, it returns **before** distributing any rewards: [4](#0-3) 

Because `last_reward_block` is already written to the current block number, every subsequent call in the same block hits the `REWARDS_ALREADY_UPDATED` assertion and reverts. The block's rewards are never credited to any staker — they are permanently lost.

---

### Impact Explanation

Each block in the consensus rewards phase is entitled to a share of the yearly mint, calculated by `calculate_block_rewards` → `update_current_epoch_block_rewards`. When an attacker calls `update_rewards(..., disable_rewards: true)` first in a block, that block's entire reward allocation is silently discarded. Repeated across many blocks, this constitutes **permanent freezing of unclaimed yield** for all stakers and delegators.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield or unclaimed royalties**.

---

### Likelihood Explanation

- The attacker needs only a valid (active, non-zero-balance) staker address, which is publicly readable from on-chain events (`NewStaker`).
- The cost is a single L2 transaction per block — extremely cheap on Starknet.
- No tokens, no stake, no privileged role is required.
- A motivated actor (e.g., a competing validator, a governance adversary, or a griever) can sustain this attack indefinitely.

Likelihood: **High**.

---

### Recommendation

Restrict `update_rewards` to a trusted caller — either the sequencer/operator role, the attestation contract, or a designated rewards-distributor address. Alternatively, remove the `disable_rewards` parameter from the public interface and handle that logic internally through a privileged path. At minimum, add a role check analogous to the one already present in `update_rewards_from_attestation_contract`: [5](#0-4) 

---

### Proof of Concept

1. Observe a valid staker address `S` from on-chain `NewStaker` events.
2. At the start of each new block `B`, submit a transaction calling:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. The function passes `general_prerequisites()` (no role check), asserts `B > last_reward_block` (true, new block), writes `last_reward_block = B`, then returns early because `disable_rewards == true`.
4. Any legitimate call to `update_rewards` later in block `B` reverts with `REWARDS_ALREADY_UPDATED`.
5. Block `B`'s reward allocation is permanently lost. Repeating this every block freezes all consensus-phase staking rewards indefinitely at the cost of L2 gas only. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
