### Title
Gateway `validate_resource_bounds` Skips L1Gas and L1DataGas Price Floor Checks, Admitting Transactions That Fail Blockifier Pre-Validation — (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The `StatelessTransactionValidator::validate_resource_bounds` function enforces a minimum price floor only for `l2_gas.max_price_per_unit`, leaving `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` unchecked. The stateful validator's `validate_tx_l2_gas_price_within_threshold` has the same gap. A transaction with zero L1/L1DataGas prices but a valid L2 gas price passes both gateway validators and enters the mempool, only to be rejected by the blockifier's `check_fee_bounds` at pre-validation time. This is the direct analog to the external report: just as OTM options have delta = 0 (no rebalancing), L1/L1DataGas prices contribute zero to the gateway's price validation, breaking the invariant that admitted transactions have sufficient prices for all signed resources.

### Finding Description

**Stateless validator — only L2 gas price is checked:**

In `StatelessTransactionValidator::validate_resource_bounds`, after the aggregate zero-fee guard, only `l2_gas.max_price_per_unit` is compared against `min_gas_price`:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never checked against any floor. [1](#0-0) 

The aggregate zero-fee guard:

```rust
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) { ... }
```

only requires that the sum of all resource fees is non-zero. A transaction with `l2_gas` fee > 0 but `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` passes this guard. [2](#0-1) 

**Stateful validator — same gap, explicitly noted:**

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` carries a TODO acknowledging the gap and only checks `l2_gas.max_price_per_unit` in the `AllResources` branch; the `L1Gas` legacy branch explicitly skips all validation:

```rust
ValidResourceBounds::L1Gas(_) => {
    // No validation required for legacy transactions.
}
``` [3](#0-2) 

**Blockifier pre-validation catches what the gateway misses:**

`AccountTransaction::check_fee_bounds` checks all three resources against actual block gas prices:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
```

A transaction with `l1_gas.max_price_per_unit = 0` when the actual L1 gas price is non-zero fails here with `TransactionFeeError::InsufficientResourceBounds`. This check only runs when `charge_fee = true`, which is determined by `enforce_fee()`. Since `enforce_fee()` returns `true` whenever `max_possible_fee > 0` (satisfied by the non-zero L2 gas fee), `check_fee_bounds` does run and rejects the transaction — but only after it has already been admitted to the mempool. [4](#0-3) 

**The broken invariant:** The gateway is supposed to reject transactions that will fail at blockifier pre-validation due to insufficient resource prices. By only checking L2 gas price, the gateway admits transactions with zero L1/L1DataGas prices that will fail at blockifier pre-validation. This is structurally identical to the external report: the "delta = 0" for L1/L1DataGas prices means those resources contribute nothing to the gateway's price validation, just as OTM options contribute nothing to vault rebalancing.

### Impact Explanation

The gateway admits transactions that will fail at blockifier pre-validation. This matches **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.** An unprivileged attacker can flood the mempool with transactions that have zero L1/L1DataGas prices, causing mempool pollution and wasting sequencer resources. The mempool and batcher will process these transactions up to the blockifier pre-validation stage before discarding them, consuming CPU and I/O on every such transaction.

### Likelihood Explanation

Any user can craft such a transaction with no special privileges. The attacker sets `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` while keeping `l2_gas.max_price_per_unit >= min_gas_price` and `l2_gas.max_amount > 0`. This passes all gateway checks. The attack is trivially repeatable at low cost since the transactions are rejected before fee transfer.

### Recommendation

Extend the gas price validation in both `StatelessTransactionValidator::validate_resource_bounds` and `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` to check all three resource prices (L1Gas, L2Gas, L1DataGas) against their respective minimum thresholds. The existing TODO comment in `stateful_transaction_validator.rs` (`// TODO(Arni): Consider running this validation for all gas prices.`) already identifies this gap. [5](#0-4) 

### Proof of Concept

Submit a V3 `AllResources` invoke transaction with:
- `l1_gas.max_price_per_unit = 0`
- `l1_data_gas.max_price_per_unit = 0`
- `l2_gas.max_price_per_unit = min_gas_price` (e.g., the configured minimum)
- `l2_gas.max_amount = 1`

**Step 1 — Stateless validator:** `max_possible_fee(Tip::ZERO) = min_gas_price > 0` → passes zero-bounds check. `l2_gas.max_price_per_unit >= min_gas_price` → passes price check. L1/L1DataGas prices are never checked. Transaction admitted. [1](#0-0) 

**Step 2 — Stateful validator:** `validate_tx_l2_gas_price_within_threshold` checks only `l2_gas.max_price_per_unit` against the previous block's L2 gas price. L1/L1DataGas prices are not checked. Transaction admitted to mempool. [6](#0-5) 

**Step 3 — Blockifier pre-validation:** `enforce_fee()` returns `true` (max_possible_fee > 0). `check_fee_bounds` runs. Actual L1 gas price (non-zero) > `l1_gas.max_price_per_unit` (0) → `MaxGasPriceTooLow` for L1Gas → `TransactionFeeError::InsufficientResourceBounds`. Transaction rejected after consuming mempool and batcher resources. [7](#0-6)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
