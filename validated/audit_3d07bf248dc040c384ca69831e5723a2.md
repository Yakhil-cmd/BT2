### Title
Unpermissioned `update_rewards` with `disable_rewards=true` Permanently Freezes All Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `IStakingRewardsManager` interface has no access control and accepts a caller-controlled `disable_rewards: bool` parameter. A global `last_reward_block` variable enforces a strict one-call-per-block limit. Any unprivileged actor can call `update_rewards(any_valid_staker, disable_rewards: true)` in every block, consuming the single allowed slot without distributing rewards, permanently freezing all consensus-era unclaimed yield for every staker and delegator in the protocol.

### Finding Description

`update_rewards` is the sole mechanism for distributing consensus-era block rewards to stakers and their delegation pools. The function is publicly callable with no `assert_caller_is_*` guard:

```
// src/staking/interface.cairo lines 304-311
pub trait IStakingRewardsManager<TContractState> {
    fn update_rewards(
        ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
    );
}
```

Inside the implementation the function:

1. Checks that the current block is strictly greater than the global `last_reward_block` (one call per block, across all stakers):

```
// src/staking/staking.cairo lines 1453-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

2. Unconditionally advances `last_reward_block` to the current block **before** checking `disable_rewards`:

```
// src/staking/staking.cairo line 1485
self.last_reward_block.write(current_block_number);
```

3. Returns early without distributing any rewards when `disable_rewards` is `true`:

```
// src/staking/staking.cairo lines 1487-1489
if disable_rewards || self.is_pre_consensus() {
    return;
}
```

Because `last_reward_block` is a single global slot (not per-staker), a call with `disable_rewards: true` for **any** valid staker exhausts the per-block quota for **all** stakers. Any subsequent call in the same block — including a legitimate call with `disable_rewards: false` — reverts with `REWARDS_ALREADY_UPDATED`. This is confirmed by the existing test suite:

```
// src/staking/tests/test.cairo lines 3887-3894
staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: true);
// Both subsequent calls in the same block revert:
.update_rewards(:staker_address, disable_rewards: true);  // REWARDS_ALREADY_UPDATED
.update_rewards(:staker_address, disable_rewards: false); // REWARDS_ALREADY_UPDATED
```

The `update_rewards_from_attestation_contract` path is explicitly gated to the pre-consensus era (`assert!(self.is_pre_consensus(), ...)`) and cannot substitute for `update_rewards` once consensus rewards are active.

### Impact Explanation

Once consensus rewards are activated (`consensus_rewards_first_epoch` is reached), `update_rewards` is the **only** code path that credits stakers and pools with block rewards. An attacker who calls `update_rewards(valid_staker, disable_rewards: true)` in every block:

- Advances `last_reward_block` each block, permanently blocking legitimate reward distribution.
- Causes `staker_info.unclaimed_rewards_own` and the pool's `cumulative_rewards_trace` to never be updated.
- All stakers and delegators accumulate zero consensus rewards indefinitely.

This constitutes **permanent freezing of unclaimed yield** for the entire protocol — a High-severity impact under the allowed scope.

### Likelihood Explanation

The attack requires only a single cheap L2 transaction per block (Starknet fees are low). No privileged role, leaked key, or external dependency is needed. The attacker only needs to know one valid, active staker address (trivially obtained from on-chain events). The attack is sustainable indefinitely and has no meaningful cost barrier.

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., a designated sequencer address, the attestation contract, or a governance-controlled role). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "disable" logic through a privileged admin function, so that the per-block slot can only be consumed by an authorized party.

### Proof of Concept

1. Consensus rewards are activated (governance calls `set_consensus_rewards_first_epoch`).
2. Attacker identifies any valid, active staker address `S` from on-chain `NewStaker` events.
3. In every new block `N`, attacker submits:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. This sets `last_reward_block = N` and returns without distributing rewards.
5. Any legitimate call to `update_rewards(S, disable_rewards: false)` in block `N` reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeated every block: no staker or delegator ever receives consensus rewards.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
