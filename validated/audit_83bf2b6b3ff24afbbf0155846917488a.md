### Title
`protocolUnpausePool` Hardcodes Wrong Target Pause Level, Leaving Pool Permanently Swap-Blocked After Protocol Unpause - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

---

### Summary

`MetricOmmPoolFactory.protocolUnpausePool` calls `setPause(1)` (admin-paused) instead of `setPause(0)` (active). Because `MetricOmmPool._checkNotPaused()` reverts for any `pauseLevel != 0`, every pool that the protocol owner "unpauses" remains fully paused for swaps. The pool admin must separately call `unpausePool` to reach level 0; if the admin is unresponsive, the protocol owner has no further recourse.

---

### Finding Description

`MetricOmmPool` uses three pause levels:

| Level | Meaning |
|---|---|
| 0 | Active |
| 1 | Admin-paused |
| 2 | Protocol-paused |

The staleness check that gates swaps is:

```solidity
// MetricOmmPool.sol
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
``` [1](#0-0) 

`swap` carries `whenNotPaused`, so any non-zero pause level blocks all trading. [2](#0-1) 

The factory's `protocolUnpausePool` is:

```solidity
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1);   // ŌåÉ hardcoded 1, not 0
}
``` [3](#0-2) 

The hardcoded argument `1` means the pool transitions `2 ŌåÆ 1` (protocol-paused ŌåÆ admin-paused), never reaching `0` (active). Swaps remain blocked after the call returns.

For comparison, `protocolPausePool` correctly accepts both pre-states (0 or 1) and targets 2:

```solidity
function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}
``` [4](#0-3) 

The pool admin's `unpausePool` can only be called from level 1 and targets 0:

```solidity
function unpausePool(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 1) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
}
``` [5](#0-4) 

So the only path from level 2 back to level 0 requires two separate actors: the protocol owner (`protocolUnpausePool`) and then the pool admin (`unpausePool`). The protocol owner alone cannot restore a pool to active state.

---

### Impact Explanation

After `protocolUnpausePool`, `pauseLevel == 1 != 0`, so `_checkNotPaused()` still reverts on every `swap` call. Core swap functionality is broken for the pool. `removeLiquidity` is not gated by `whenNotPaused`, so LP principal withdrawal is unaffected, but all trading is halted. If the pool admin is unresponsive or adversarial, the protocol owner has no mechanism to restore the pool to active state unilaterally.

---

### Likelihood Explanation

This triggers on every invocation of `protocolUnpausePool` for a pool that was active (level 0) before the protocol pause. The hardcoded `1` is unconditional ŌĆö there is no code path through `protocolUnpausePool` that reaches `setPause(0)`.

---

### Recommendation

Change the hardcoded target in `protocolUnpausePool` from `1` to `0`:

```solidity
// metric-core/contracts/MetricOmmPoolFactory.sol
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 0);   // update error arg too
    IMetricOmmPoolFactoryActions(pool).setPause(0);         // ŌåÉ was 1
}
```

---

### Proof of Concept

1. Pool is deployed; `pauseLevel = 0` (active). Swaps succeed.
2. Protocol owner calls `protocolPausePool(pool)` ŌåÆ `pauseLevel = 2`. Swaps revert `PoolPaused`.
3. Protocol owner calls `protocolUnpausePool(pool)` ŌåÆ `setPause(1)` executes ŌåÆ `pauseLevel = 1`.
4. User calls `swap(...)` ŌåÆ `_checkNotPaused()` fires: `pauseLevel (1) != 0` ŌåÆ `revert PoolPaused()`.
5. Pool admin must separately call `unpausePool(pool)` to reach `pauseLevel = 0`.
6. If pool admin is absent, the protocol owner has no further lever ŌĆö the pool is permanently swap-blocked.
