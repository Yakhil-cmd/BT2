### Title
Unrestricted `update_rewards` with Caller-Controlled `disable_rewards` Enables Permanent Griefing of Consensus Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is callable by any unprivileged address. It accepts a caller-controlled `disable_rewards: bool` parameter with no validation that the caller is authorized to suppress reward distribution. When called with `disable_rewards: true`, the function writes the current block number into the global `last_reward_block` slot without distributing any rewards, permanently consuming the one-call-per-block budget and blocking every legitimate call for that block.

### Finding Description
`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

There is no role check, no check that the caller is the staker, and no check that the caller is any trusted consensus participant. The function signature accepts two fully attacker-controlled inputs: [2](#0-1) 

The critical gate that makes this exploitable is the global `last_reward_block` write, which happens unconditionally before the `disable_rewards` branch: [3](#0-2) 

Once `last_reward_block` is written to the current block, the assertion at the top of the function: [4](#0-3) 

causes every subsequent call to `update_rewards` in the same block to revert with `REWARDS_ALREADY_UPDATED`. The `last_reward_block` field is a single global slot, not per-staker: [5](#0-4) 

The attacker only needs to supply any currently active staker address (all staker addresses are public on-chain) to pass the staker-existence and non-zero-balance checks, then pass `disable_rewards: true`. The function returns early after writing `last_reward_block`, distributing nothing. [6](#0-5) 

### Impact Explanation
An attacker calling `update_rewards(staker_address: <any_active_staker>, disable_rewards: true)` once per block permanently prevents all stakers from receiving consensus block rewards for that block. Repeated every block, this completely halts the consensus reward stream for the entire protocol. Stakers and their delegators accrue zero rewards despite the protocol operating normally. This is griefing with no profit motive but direct, measurable damage to every participant in the protocol.

**Matched allowed impact:** *Griefing with no profit motive but damage to users or protocol* (Medium).

### Likelihood Explanation
The entry path requires no privilege, no stake, no special role, and no leaked key. Any externally owned account can call `update_rewards` with an arbitrary valid staker address and `disable_rewards: true`. Staker addresses are discoverable from on-chain events (`NewStaker`). The cost is one transaction per block. The attack is trivially automatable.

### Recommendation
Add an access-control check to `update_rewards` so that only the authorized consensus/block-reward caller (e.g., a designated `REWARDS_MANAGER` role, or the attestation contract, or a specific sequencer address) can invoke it. The `disable_rewards` flag should not be a caller-supplied parameter at all; the authorization logic for whether to disable rewards for a given block should be determined internally or by a trusted caller, not by an arbitrary external address.

### Proof of Concept
1. Attacker observes any active staker address `S` from on-chain `NewStaker` events.
2. At the start of block `N`, attacker calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `general_prerequisites()` passes (contract not paused, caller non-zero).
4. `current_block_number > last_reward_block` passes (first call this block).
5. Staker `S` existence and active-balance checks pass.
6. `last_reward_block` is written to block `N`.
7. `disable_rewards == true` → function returns early; zero rewards distributed.
8. Any legitimate call to `update_rewards` for block `N` now reverts with `REWARDS_ALREADY_UPDATED`.
9. Attacker repeats every block → all stakers receive zero consensus rewards indefinitely.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
