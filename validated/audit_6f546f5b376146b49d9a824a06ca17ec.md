### Title
Sequencer Timestamp Manipulation Inflates Per-Block Rewards via `get_block_timestamp()` in `set_avg_block_duration` — (File: src/reward_supplier/reward_supplier.cairo)

---

### Summary

`RewardSupplier::set_avg_block_duration()` uses `starknet::get_block_timestamp()` to measure elapsed time between epoch snapshots and derive `avg_block_duration`. This value directly scales per-block reward amounts. Because Starknet block timestamps are set by the sequencer (block proposer), a malicious sequencer proposing the epoch-boundary block can inflate the timestamp, pushing `avg_block_duration` to its configured maximum and causing all stakers to receive up to ~67% excess rewards per epoch.

---

### Finding Description

In `src/reward_supplier/reward_supplier.cairo`, `set_avg_block_duration()` is called from `update_current_epoch_block_rewards()` at the start of each epoch:

```
fn set_avg_block_duration(ref self: ContractState) {
    let current_block_number = starknet::get_block_number();
    let current_timestamp = starknet::get_block_timestamp();          // <-- attacker-controlled
    let (snapshot_block_number, snapshot_timestamp) = self.block_snapshot.read();
    ...
    self.block_snapshot.write((current_block_number, Timestamp { seconds: current_timestamp }));
    ...
    let time_delta = current_timestamp - snapshot_timestamp.into();
    let num_blocks = current_block_number - snapshot_block_number;
    let mut calculated_block_duration = mul_wide_and_div(
        lhs: time_delta, rhs: BLOCK_DURATION_SCALE, div: num_blocks,
    )...;
    calculated_block_duration = calculated_block_duration
        .clamp(block_duration_config.min_block_duration, block_duration_config.max_block_duration);
    self.avg_block_duration.write(calculated_block_duration);
}
``` [1](#0-0) 

The resulting `avg_block_duration` is then used directly to compute per-block rewards:

```
let total_rewards = mul_wide_and_div(
    lhs: yearly_mint,
    rhs: avg_block_duration.into(),
    div: BLOCK_DURATION_SCALE.into() * SECONDS_IN_YEAR.into(),
)
``` [2](#0-1) 

The only guard on the timestamp is:

```
assert!(current_timestamp > snapshot_timestamp.into(), "{}", InternalError::INVALID_BLOCK_TIMESTAMP);
``` [3](#0-2) 

There is no upper-bound check. The clamp limits `avg_block_duration` to `[min_block_duration, max_block_duration]`, with defaults:

```
pub(crate) const DEFAULT_AVG_BLOCK_DURATION: u64 = 3 * BLOCK_DURATION_SCALE;   // 300 = 3 s
pub(crate) const DEFAULT_BLOCK_DURATION_CONFIG: BlockDurationConfig = BlockDurationConfig {
    min_block_duration: 2 * BLOCK_DURATION_SCALE, max_block_duration: 5 * BLOCK_DURATION_SCALE,
};
``` [4](#0-3) 

A sequencer proposing the epoch-boundary block sets `current_timestamp` to any value satisfying `current_timestamp > snapshot_timestamp`. By setting it high enough that `time_delta * 100 / num_blocks >= 500`, the clamp saturates at `max_block_duration = 500` (5 seconds). The per-block reward becomes `yearly_mint * 500 / (100 * SECONDS_IN_YEAR)` instead of the intended `yearly_mint * 300 / (100 * SECONDS_IN_YEAR)` — a **5/3 ≈ 1.67× inflation**.

---

### Impact Explanation

**Impact: High — Theft of unclaimed yield.**

With actual 3-second blocks but `avg_block_duration` forced to 5 seconds:

- Intended annual distribution: `yearly_mint * 3/SECONDS_IN_YEAR * (SECONDS_IN_YEAR/3) = yearly_mint`
- Manipulated annual distribution: `yearly_mint * 5/SECONDS_IN_YEAR * (SECONDS_IN_YEAR/3) = yearly_mint * 5/3`

Every epoch where the attack succeeds, stakers collectively receive **~67% more STRK/BTC rewards** than the minting curve intends. This excess is minted from the protocol's inflation budget, constituting theft of unclaimed yield at the protocol level. A sequencer who is also a staker (or who colludes with stakers) directly profits.

---

### Likelihood Explanation

In Starknet's PoS model, the sequencer proposing the first block of each epoch controls `block.timestamp` for that block. The attack requires no special privilege beyond being the epoch-boundary block proposer — a role that rotates among validators. The manipulation is a single-field change (inflating the timestamp) with no on-chain detection. The only constraint is `current_timestamp > snapshot_timestamp`, which is trivially satisfied. The attack is repeatable every epoch.

---

### Recommendation

1. **Bound `time_delta` against `num_blocks`**: Before computing `avg_block_duration`, assert that `time_delta` is within a plausible range relative to `num_blocks`, e.g., `time_delta <= num_blocks * MAX_EXPECTED_BLOCK_SECONDS`. This prevents a single inflated timestamp from saturating the clamp.

2. **Tighten the clamp**: Reduce the gap between `min_block_duration` and `max_block_duration` to limit the reward variance a manipulated timestamp can cause.

3. **Use a median or TWAP**: Accumulate multiple block-timestamp samples across the epoch and use a median or time-weighted average rather than a single epoch-boundary measurement.

---

### Proof of Concept

1. Epoch N ends; the sequencer for the first block of epoch N+1 is a staker (or colluding with stakers).
2. The sequencer sets `block.timestamp = snapshot_timestamp + 5 * num_blocks` (where `num_blocks` is the epoch length in blocks and `snapshot_timestamp` is the stored snapshot from epoch N's start).
3. `set_avg_block_duration()` computes `time_delta = 5 * num_blocks`, `avg_block_duration = 5 * num_blocks * 100 / num_blocks = 500`, clamped to `max_block_duration = 500`.
4. `update_current_epoch_block_rewards()` stores `block_rewards = yearly_mint * 500 / (100 * SECONDS_IN_YEAR)` — 5/3× the intended value.
5. Every attestation in epoch N+1 triggers `update_rewards_from_attestation_contract`, distributing inflated rewards to all stakers.
6. The sequencer (as a staker) collects their proportional share of the ~67% excess rewards with no additional cost.

        fn set_avg_block_duration(ref self: ContractState) {
            let current_block_number = starknet::get_block_number();
            let current_timestamp = starknet::get_block_timestamp();
            let (snapshot_block_number, snapshot_timestamp) = self.block_snapshot.read();
            // Sanity asserts.
            assert!(
                current_block_number > snapshot_block_number,
                "{}",
                InternalError::INVALID_BLOCK_NUMBER,
            );
            assert!(
                current_timestamp > snapshot_timestamp.into(),
                "{}",
                InternalError::INVALID_BLOCK_TIMESTAMP,
            );
            self
                .block_snapshot
                .write((current_block_number, Timestamp { seconds: current_timestamp }));
            // If this is the first time we're setting the block snapshot, can't calculate avg block
            // time yet.
            if snapshot_block_number.is_zero() || snapshot_timestamp.is_zero() {
                return;
            }
            let time_delta = current_timestamp - snapshot_timestamp.into();
            // *Note*: `num_blocks` should match the epoch length in blocks. This calculation is
            // expected to run on the first block of each epoch, assuming `update_rewards` is called
            // every block.
            // We calculate `num_blocks` instead of using the configured value to keep the average
            // accurate even if some calls are missed.
            let num_blocks = current_block_number - snapshot_block_number;
            let mut calculated_block_duration = mul_wide_and_div(
                lhs: time_delta, rhs: BLOCK_DURATION_SCALE, div: num_blocks,
            )
                .expect_with_err(err: Error::BLOCK_DURATION_OVERFLOW);
            // Adjust calculated_block_duration with min and max block duration.
            let block_duration_config = self.block_duration_config.read();
            calculated_block_duration = calculated_block_duration
                .clamp(
                    block_duration_config.min_block_duration,
                    block_duration_config.max_block_duration,
                );
            self.avg_block_duration.write(calculated_block_duration);
        }
