### Title
Revoking key 0 does not revoke duplicate key entries sharing the same key material — (`fvm/environment/account_public_key_util.go`, `fvm/environment/accounts.go`)

### Summary

When an account has a duplicate key (key index 1 pointing to the same stored key bytes as key index 0), revoking key 0 only sets `Revoked=true` in the `apk_0` register. The revoked status for key index 1 is stored independently in the account status key metadata and remains `false`. The transaction verifier and sequence number checker both read the revoked status per key index independently, so key 1 continues to pass all authorization checks using the same private key.

### Finding Description

The revocation path for key index 0 is handled by `revokeAccountPublicKey0`, which exclusively writes to the `apk_0` register: [1](#0-0) 

For key index > 0, `RevokeAccountPublicKey` writes only to the account status key metadata for that specific key index: [2](#0-1) 

These two storage locations are completely independent. When key 1 is a duplicate of key 0 (detected via `FindDuplicateKey` and stored with `AppendDuplicateKeyMetadata`), its `revoked` bit in key metadata is initialized to `false` and is never touched by `revokeAccountPublicKey0`: [3](#0-2) 

`GetRuntimeAccountPublicKey` for key index 1 reads `revoked` from account status metadata (not from `apk_0`), so it returns `Revoked: false` even after key 0 has been revoked: [4](#0-3) 

The transaction verifier calls `GetRuntimeAccountPublicKey` per signature and checks `accountKey.Revoked`: [5](#0-4) 

The sequence number checker calls `GetAccountPublicKeyRevokedStatus` for the proposal key, which for key index 1 also reads from account status metadata: [6](#0-5) [7](#0-6) 

Both checks return `false` (not revoked) for key 1, so the transaction proceeds normally.

### Impact Explanation

An attacker who holds signing authority on account A (e.g., one key holder in a multi-sig setup, or an attacker who temporarily compromised the account) can pre-register a duplicate key (key 1 = same key material as key 0) before key 0 is revoked. After key 0 is revoked, the attacker retains full signing authority via key 1 using the same private key. This violates the security invariant that revoking a key prevents all use of that key material, resulting in unauthorized account mutation — the attacker retains signing authority on account A after revocation.

### Likelihood Explanation

The attack requires the attacker to have had signing authority on the account at some point (to add key 1). This is realistic in multi-sig accounts or accounts that were temporarily compromised. The `AppendAccountPublicKey` Cadence built-in is accessible to any authorized signer. The duplicate key detection mechanism (`FindDuplicateKey`) is designed for storage deduplication, not security enforcement, and explicitly stores independent revoked status per key index. [8](#0-7) [9](#0-8) 

### Recommendation

When revoking key index 0, scan the account status key metadata for all key indices whose `storedKeyIndex` maps to 0 (i.e., duplicate entries pointing to the same stored key) and set their revoked bit to `true` as well. Alternatively, revocation should be keyed on the stored key index rather than the account key index, so that revoking any key index that maps to a given stored key index revokes all key indices sharing that stored key.

### Proof of Concept

```
1. Create account A with key0 (weight=1000, private key K).
2. Using key0, call addPublicKey(key1) where key1 has the same
   PublicKey/SignAlgo/HashAlgo bytes as key0 (weight=1000).
   → AppendAccountPublicKeyMetadata detects duplicate, stores
     storedKeyIndex=0 for key1, revoked=false in key metadata.
3. Call revokeAccountKey(keyIndex=0).
   → revokeAccountPublicKey0 sets apk_0.Revoked=true.
   → Key metadata for key1 is untouched; revoked=false remains.
4. Construct a transaction with:
   - ProposalKey: {Address: A, KeyIndex: 1, SequenceNumber: 0}
   - EnvelopeSignature signed with private key K under key index 1.
5. Submit transaction.
   → GetAccountPublicKeyRevokedStatus(A, 1) → reads key metadata → false
   → GetRuntimeAccountPublicKey(A, 1) → Revoked=false, PublicKey=K
   → Signature verification passes (same private key K)
   → Transaction executes successfully.
6. Assert: key0 is revoked, key1 is not, and private key K is still usable.
```

func revokeAccountPublicKey0(
	a Accounts,
	address flow.Address,
) error {
	accountPublicKey0RegisterID := flow.AccountPublicKey0RegisterID(address)

	publicKey, err := a.GetValue(accountPublicKey0RegisterID)
	if err != nil {
		return err
	}

	const keyIndex = uint32(0)

	if len(publicKey) == 0 {
		return errors.NewAccountPublicKeyNotFoundError(
			address,
			keyIndex)
	}

	decodedPublicKey, err := flow.DecodeAccountPublicKey(publicKey, keyIndex)
	if err != nil {
		return fmt.Errorf(
			"failed to decode account public key 0: %w",
			err)
	}

	decodedPublicKey.Revoked = true

	encodedPublicKey, err := flow.EncodeAccountPublicKey(decodedPublicKey)
	if err != nil {
		encoded, _ := decodedPublicKey.MarshalJSON()
		return errors.NewValueErrorf(
			string(encoded),
			"failed to encode revoked account public key 0: %w",
			err)
	}

	return a.SetValue(accountPublicKey0RegisterID, encodedPublicKey)
}

func (a *StatefulAccounts) GetAccountPublicKeyRevokedStatus(address flow.Address, keyIndex uint32) (bool, error) {
	err := a.accountPublicKeyIndexInRange(address, keyIndex)
	if err != nil {
		return false, err
	}

	if keyIndex == 0 {
		accountPublicKey, err := getAccountPublicKey0(a, address)
		if err != nil {
			return false, err
		}
		return accountPublicKey.Revoked, nil
	}

	status, err := a.getAccountStatus(address)
	if err != nil {
		return false, err
	}

	return status.AccountPublicKeyRevokedStatus(keyIndex)
}

func (a *StatefulAccounts) RevokeAccountPublicKey(
	address flow.Address,
	keyIndex uint32,
) error {
	err := a.accountPublicKeyIndexInRange(address, keyIndex)
	if err != nil {
		return err
	}

	if keyIndex == 0 {
		return revokeAccountPublicKey0(a, address)
	}

	status, err := a.getAccountStatus(address)
	if err != nil {
		return err
	}

	err = status.RevokeAccountPublicKey(keyIndex)
	if err != nil {
		return fmt.Errorf("failed to revoke public key at index %d for %s: %w", keyIndex, address, err)
	}

	return a.setAccountStatusAfterAccountStatusSizeChange(address, status)
}
