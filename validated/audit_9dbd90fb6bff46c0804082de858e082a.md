### Title
`validate_tx` Skips Expired-Transaction Pruning That `add_tx` Performs, Causing False `DuplicateNonce` Rejections at Gateway Admission - (File: `crates/apollo_mempool/src/mempool.rs`)

---

### Summary

`Mempool::validate_tx` — called by the gateway's stateful validator before `add_tx` — does not prune TTL-expired transactions from the pool before running fee-escalation checks. `Mempool::add_tx` does prune them first. The asymmetry means a transaction that `add_tx` would accept (because the conflicting entry is expired and would be removed) is rejected by `validate_tx` with `DuplicateNonce`, causing the gateway to return an error to the user. The user is forced to pay ≥10 % more tip than the expired transaction, or is permanently blocked when fee escalation is disabled.

---

### Finding Description

`add_tx` opens with an explicit cleanup step:

```rust
pub fn add_tx(&mut self, args: AddTransactionArgs) -> MempoolResult<()> {
    // First remove old transactions from the pool.
    let mut account_nonce_updates = self.remove_expired_txs();
    ...
    self.add_tx_validations(tx_reference, &args.tx, args.account_state.nonce)?;
``` [1](#0-0) 

`validate_tx`, the read-only pre-check called by the gateway, performs the same logical checks (`validate_incoming_tx` + `validate_fee_escalation`) but **without** calling `remove_expired_txs()` first:

```rust
pub fn validate_tx(&mut self, args: ValidationArgs) -> MempoolResult<()> {
    let tx_reference = (&args).into();
    self.validate_incoming_tx(tx_reference, args.account_nonce)?;
    self.validate_fee_escalation(tx_reference)?;
    Ok(())
}
``` [2](#0-1) 

`validate_fee_escalation` checks whether an existing pool entry at `(address, nonce)` can be replaced. When fee escalation is enabled (the default), it requires the incoming tip and max-L2-gas-price to each be ≥ `existing × (1 + fee_escalation_percentage/100)`: [3](#0-2) 

The default `fee_escalation_percentage` is 10 and `transaction_ttl` is 60 seconds: [4](#0-3) 

The gateway calls `validate_by_mempool` (→ `validate_tx`) during stateful validation, then calls `add_tx` only if validation passes: [5](#0-4) [6](#0-5) 

**Attack / trigger path (unprivileged):**

1. Attacker or any user submits transaction T1 for account A, nonce N, tip 100.
2. T1 sits in the mempool for > 60 s (TTL) without being sequenced.
3. User submits T2 for account A, nonce N, tip 100 (a legitimate replacement or re-submission).
4. Gateway calls `validate_tx` → T1 is still in the pool (expired but not yet pruned) → `validate_fee_escalation` computes required tip = 110 → 100 < 110 → returns `MempoolError::DuplicateNonce`.
5. Gateway converts this to `StarknetErrorCode::InvalidTransactionNonce` / `DuplicateNonce` and rejects T2.
6. If the user retried with tip 110, `validate_tx` would pass, then `add_tx` would prune T1 first and insert T2 — so the user is forced to overpay.

When `enable_fee_escalation = false`, step 4 always returns `DuplicateNonce` regardless of tip, permanently blocking the user until some other `add_tx` call triggers `remove_expired_txs`. [7](#0-6) 

---

### Impact Explanation

The gateway's admission path (`validate_tx`) rejects a transaction that the insertion path (`add_tx`) would accept. This is a **false rejection**: a structurally valid transaction with a correct nonce and sufficient fee is turned away at the RPC/gateway boundary. The user either pays an unnecessary 10 % fee premium or, when fee escalation is disabled, cannot submit at all until an unrelated `add_tx` call happens to trigger cleanup. This matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

---

### Likelihood Explanation

Any transaction that remains unsequenced for longer than `transaction_ttl` (default 60 s) creates the condition. This is a normal operational scenario: network congestion, low-tip transactions, or a mempool near capacity can all cause transactions to age past the TTL. The trigger requires no special privilege — any user who previously submitted a transaction can reproduce it.

---

### Recommendation

Call `remove_expired_txs()` at the start of `validate_tx`, mirroring `add_tx`:

```rust
pub fn validate_tx(&mut self, args: ValidationArgs) -> MempoolResult<()> {
    self.remove_expired_txs(); // prune stale entries before checking
    let tx_reference = (&args).into();
    self.validate_incoming_tx(tx_reference, args.account_nonce)?;
    self.validate_fee_escalation(tx_reference)?;
    Ok(())
}
```

This ensures the fee-escalation and nonce checks in `validate_tx` operate on the same pool state that `add_tx` would see, eliminating the divergence.

// Setup
let fake_clock = Arc::new(FakeClock::default());
let mut mempool = Mempool::new(
    MempoolConfig {
        dynamic_config: MempoolDynamicConfig { transaction_ttl: Duration::from_secs(60) },
        static_config: MempoolStaticConfig {
            enable_fee_escalation: true,
            fee_escalation_percentage: 10,
            ..Default::default()
        },
        ..Default::default()
    },
    fake_clock.clone(),
);

// Step 1: add T1 (nonce=0, tip=100)
let t1 = add_tx_input!(tx_hash: 1, address: "0x0", tx_nonce: 0, account_nonce: 0, tip: 100);
add_tx(&mut mempool, &t1);

// Step 2: advance clock past TTL
fake_clock.advance(Duration::from_secs(65));

// Step 3: validate T2 (nonce=0, tip=100) — same tip, should be accepted because T1 is expired
// Expected: Ok(())  Actual: Err(DuplicateNonce) because validate_tx sees T1 still in pool
let t2_args = ValidationArgs {
    address: contract_address!("0x0"),
    tx_nonce: nonce!(0),
    account_nonce: nonce!(0),
    tip: Tip(100),
    tx_hash: tx_hash!(2),
    max_l2_gas_price: GasPrice(1),
};
let result = mempool.validate_tx(t2_args);
assert!(result.is_err()); // DuplicateNonce — false rejection

// Step 4: add_tx with same tip succeeds because it prunes T1 first
let t2 = add_tx_input!(tx_hash: 2, address: "0x0", tx_nonce: 0, account_nonce: 0, tip: 100);
let result = mempool.add_tx(t2);
assert!(result.is_ok()); // accepted after pruning T1

    fn validate_fee_escalation(
        &self,
        incoming_tx_reference: TransactionReference,
    ) -> MempoolResult<Option<TransactionReference>> {
        let TransactionReference { address, nonce, .. } = incoming_tx_reference;

        self.validate_no_delayed_declare_front_run(incoming_tx_reference)?;

        if !self.config.static_config.enable_fee_escalation {
            if self.tx_pool.get_by_address_and_nonce(address, nonce).is_some() {
                return Err(MempoolError::DuplicateNonce { address, nonce });
            };

            return Ok(None);
        }

        let Some(existing_tx_reference) = self.tx_pool.get_by_address_and_nonce(address, nonce)
        else {
            // Replacement irrelevant: no existing transaction with the same nonce for address.
            return Ok(None);
        };

        if !self.should_replace_tx(&existing_tx_reference, &incoming_tx_reference) {
            info!(
                "{existing_tx_reference} was not replaced by {incoming_tx_reference} due to \
                 insufficient fee escalation."
            );
            // TODO(Elin): consider adding a more specific error type / message.
            return Err(MempoolError::DuplicateNonce { address, nonce });
        }

        Ok(Some(existing_tx_reference))
    }

    fn remove_expired_txs(&mut self) -> AddressToNonce {
        let removed_txs = self
            .tx_pool
            .remove_txs_older_than(self.config.dynamic_config.transaction_ttl, &self.state.staged);

        for tx_ref in &removed_txs {
            self.decrement_stuck_txs_if_gap_account(tx_ref.address, 1);
        }

        let queued_txs = self.tx_queue.remove_txs(&removed_txs);

        self.log_and_count_expired_txs(&removed_txs);
        self.update_state_metrics();
        queued_txs
            .into_iter()
            .map(|tx| (tx.address, self.state.resolve_nonce(tx.address, tx.nonce)))
            .collect::<AddressToNonce>()
    }
