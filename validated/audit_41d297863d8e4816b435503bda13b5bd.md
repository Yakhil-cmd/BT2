### Title
Signature Verification Bypass via Stale Mempool Presence Check in `skip_stateful_validations` — (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

---

### Summary

The `skip_stateful_validations` function skips the `__validate__` entry-point call (i.e., signature verification) for an invoke transaction with nonce=1 when the on-chain account nonce is 0, based solely on whether **any** transaction for that address exists in the mempool. The check does not verify that the pooled transaction is specifically a `deploy_account`. An unprivileged attacker who observes a victim's pending `deploy_account` in the mempool can submit an invoke with an arbitrary/fake signature for the victim's address, and the gateway will admit it without ever verifying the signature.

---

### Finding Description

In `skip_stateful_validations`:

`ValidationArgs` carries only `address`, `account_nonce`, `tx_hash`, `tx_nonce`, `tip`, and `max_l2_gas_price` — no signature field: [5](#0-4) 

**Attack sequence:**

1. Alice submits `deploy_account` (nonce=0) for address A. It enters the mempool pool (`tx_pool.contains_account(A)` → `true`).
2. Bob submits `invoke` (nonce=1, sender=A) with an arbitrary/fake signature.
3. Gateway stateful validation:
   - Nonce check: `account_nonce=0`, `tx_nonce=1`, `0 ≤ 1 ≤ max_allowed_nonce_gap` → passes. [6](#0-5) 
   - `validate_by_mempool`: no duplicate nonce=1 for address A → passes.
   - `skip_stateful_validations`: `nonce==1`, `account_nonce==0`, `account_tx_in_pool_or_recent_block(A)==true` → returns `true`.
   - `run_validate_entry_point` with `validate=false` → `__validate__` is **never called**.
4. Bob's invoke is admitted to the mempool with no signature check.

**At execution time**, the batcher sets `validate=true` by default, so `__validate__` is called on Alice's deployed account, which rejects Bob's fake signature. The transaction fails and no fee is charged. Bob pays nothing for the failed transaction.

**Fee-escalation displacement sub-case:** If Alice also submitted a legitimate invoke (nonce=1) for address A, Bob can supply a tip/gas_price exceeding Alice's by the `fee_escalation_percentage` threshold (10% in production config). The mempool's `validate_fee_escalation` will accept Bob's fake invoke as a replacement, atomically removing Alice's legitimate invoke from the pool. Alice's transaction is permanently lost from the mempool. [7](#0-6) 

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

- An unprivileged attacker can inject invoke transactions with arbitrary signatures into the mempool for any address that has a pending `deploy_account`.
- Via fee escalation, the attacker can displace a victim's legitimate nonce=1 invoke, causing it to be permanently dropped from the mempool.
- Because failed `__validate__` incurs no fee, the attacker bears zero cost per injected transaction, enabling sustained mempool pollution.
- The corrupted value is the admission decision: the gateway returns a valid `tx_hash` for a transaction whose signature was never verified.

---

### Likelihood Explanation

`deploy_account` transactions are publicly visible in the mempool (propagated via P2P). Any observer can extract the sender address and immediately submit a competing nonce=1 invoke. The condition `nonce==1 && account_nonce==0` is the exact window every new account passes through. No special privilege or prior relationship with the victim is required.

---

### Recommendation

Replace the coarse `account_tx_in_pool_or_recent_block` check with a type-specific query that confirms a `deploy_account` transaction (not just any transaction) is pending for the address. Concretely, expose a `has_pending_deploy_account(address)` method on the mempool that inspects the transaction type stored in `tx_pool`, and use that in `skip_stateful_validations` instead of the generic presence check.

```rust
// Proposed replacement condition
if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
    return mempool_client
        .has_pending_deploy_account(tx.sender_address())  // type-specific check
        .await
        ...
}
```

This preserves the UX improvement (deploy_account + invoke submitted simultaneously) while closing the path by which an attacker can exploit any arbitrary pending transaction to bypass signature verification.

### Proof of Concept

```
1. Alice calls POST /add_transaction with:
     type: DEPLOY_ACCOUNT, nonce: 0, class_hash: <any>, salt: S,
     constructor_calldata: [...], signature: <valid Alice sig>
   → Gateway admits it; mempool now has tx_pool entry for address A = hash(class, salt, calldata).

2. Bob calls POST /add_transaction with:
     type: INVOKE, sender_address: A, nonce: 1,
     calldata: [<arbitrary>], signature: [0xdeadbeef, 0xdeadbeef],
     resource_bounds: { l2_gas: { max_amount: 1000, max_price_per_unit: <above threshold> } }

3. Gateway stateful path:
   a. get_nonce_from_state(A) → Nonce(0)          [account not deployed]
   b. validate_nonce: 0 ≤ 1 ≤ max_gap             [passes]
   c. validate_resource_bounds                     [passes]
   d. validate_by_mempool(nonce=1, account_nonce=0)[passes, no dup]
   e. skip_stateful_validations:
        nonce==1 ✓, account_nonce==0 ✓
        account_tx_in_pool_or_recent_block(A) → true  [Alice's deploy_account is pooled]
        → returns true
   f. run_validate_entry_point(skip_validate=true)
        ExecutionFlags { validate: false, ... }
        → __validate__ is NEVER called
   → Gateway returns tx_hash for Bob's fake invoke. ✓ admitted.

4. Batcher later executes:
   - Alice's deploy_account → account A deployed with Alice's public key
   - Bob's invoke → __validate__ called → rejects 0xdeadbeef signature → REVERTED, no fee

5. Net effect: Bob's invalid transaction was admitted to the mempool.
   If Bob used a tip 10%+ above Alice's nonce=1 invoke (if any), Alice's invoke
   was atomically replaced and is now gone from the mempool.
```

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
```

```rust
    /// `(address, nonce)` via fee escalation, without mutating any state. Returns the existing
    /// transaction to be replaced when a valid replacement exists, `None` when there is nothing to
    /// replace, or an error when a replacement is present but not permitted.
    fn validate_fee_escalation(
        &self,
        incoming_tx_reference: TransactionReference,
    ) -> MempoolResult<Option<TransactionReference>> {
        let TransactionReference { address, nonce, .. } = incoming_tx_reference;

        self.validate_no_delayed_declare_front_run(incoming_tx_reference)?;

        if !self.config.static_config.enable_fee_escalation {
            if self.tx_pool.get_by_address_and_nonce(address, nonce).is_some() {
                return Err(MempoolError::DuplicateNonce { address, nonce });
            };

            return Ok(None);
        }

        let Some(existing_tx_reference) = self.tx_pool.get_by_address_and_nonce(address, nonce)
        else {
            // Replacement irrelevant: no existing transaction with the same nonce for address.
            return Ok(None);
        };

        if !self.should_replace_tx(&existing_tx_reference, &incoming_tx_reference) {
            info!(
                "{existing_tx_reference} was not replaced by {incoming_tx_reference} due to \
                 insufficient fee escalation."
            );
            // TODO(Elin): consider adding a more specific error type / message.
            return Err(MempoolError::DuplicateNonce { address, nonce });
        }

        Ok(Some(existing_tx_reference))
    }
```
