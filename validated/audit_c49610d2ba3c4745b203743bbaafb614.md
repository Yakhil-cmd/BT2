### Title
Missing Access Control on `update_rewards` Allows Griefing of Consensus Reward Distribution — (`File: src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that writes to the global `last_reward_block` storage slot **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke it with `disable_rewards: true` once per block, consuming the per-block reward slot without distributing any rewards, and permanently blocking the legitimate consensus node from doing so for that block.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` has no role-based access control. Its only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function writes `last_reward_block` to the current block number unconditionally, **before** the `disable_rewards` branch: [2](#0-1) 

`last_reward_block` is a single global slot (not per-staker): [3](#0-2) 

Once written, any subsequent call in the same block fails immediately: [4](#0-3) 

The attacker only needs to supply any currently-active staker address with non-zero STRK balance (both are publicly readable via `get_stakers` / `staker_info_v3`). [5](#0-4) 

---

### Impact Explanation

For every block where the attacker fires first, `last_reward_block` is stamped and the early-return path is taken:

```
if disable_rewards || self.is_pre_consensus() {
    return;   // ← no rewards distributed, but slot is consumed
}
```

The legitimate consensus node's subsequent call reverts with `REWARDS_ALREADY_UPDATED`. The block's reward allocation is permanently lost — it is never credited to `unclaimed_rewards_own` for any staker. Repeated every block, this constitutes **permanent freezing of all consensus-era unclaimed yield** for the entire protocol.

This matches the allowed impact: *Permanent freezing of unclaimed yield or unclaimed royalties*.

---

### Likelihood Explanation

- The function is fully public; no key, role, or privileged position is required.
- The attacker only needs to know one valid, active staker address (trivially obtained from on-chain events or `get_stakers`).
- The cost is a single cheap transaction per block.
- The attack is reliable and deterministic: front-running the consensus node's call is sufficient.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus node's registered operational address, or a dedicated role). Alternatively, separate the `last_reward_block` write so it only occurs when rewards are actually distributed, preventing the slot from being consumed by a no-op call.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. A new block `N` is produced. `last_reward_block < N`.
3. Attacker calls:
   ```
   staking.update_rewards(staker_address: <any_active_staker>, disable_rewards: true)
   ```
4. Inside the function:
   - All precondition checks pass (staker exists, is active, has non-zero STRK balance).
   - `last_reward_block` is written to `N`.
   - Function returns early — zero rewards distributed.
5. The legitimate consensus node calls `update_rewards(staker_address: X, disable_rewards: false)` in the same block.
6. The call reverts: `current_block_number (N) > last_reward_block (N)` is **false** → `REWARDS_ALREADY_UPDATED`.
7. Block `N`'s reward allocation is permanently lost for all stakers.
8. Repeat for every block. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
