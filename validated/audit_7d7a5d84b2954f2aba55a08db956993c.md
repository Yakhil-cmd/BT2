### Title
`setPoolAdminFeeDestination` Redirects Already-Accrued Admin Fees to New Destination Without Prior Collection - (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`MetricOmmPoolFactory.setPoolAdminFeeDestination` updates `poolAdminFeeDestination[pool]` without first calling `collectFees` at the old destination. All spread-fee surplus and notional-fee accumulator amounts that accrued before the destination change are subsequently paid out to the **new** destination, permanently depriving the **old** destination of its owed admin fees.

### Finding Description

Every other fee-configuration mutator in the factory that changes how fees are split or where they go first flushes accrued fees at the current rates and destination before applying the new configuration:

- `setPoolAdminFees` calls `collectFees(…, poolAdminFeeDestination[pool])` before updating `poolFeeConfig`. [1](#0-0) 
- `setPoolProtocolFee` does the same before updating the protocol component. [2](#0-1) 

`setPoolAdminFeeDestination` does not follow this pattern:

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;   // ← no collectFees first
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
``` [3](#0-2) 

When `collectFees` is eventually called (permissionlessly via `collectPoolFees`, or implicitly via the next `setPoolAdminFees`/`setPoolProtocolFee`), it uses `adminFeeDestination_` — which is now the **new** address — to transfer both:

1. The **spread-fee surplus** (`balance - binTotals - notionalAccumulator`) accrued from all swaps before the destination change. [4](#0-3) 
2. The **notional-fee accumulator** (`notionalFeeToken0Scaled` / `notionalFeeToken1Scaled`) built up from all swaps before the destination change. [5](#0-4) 

Both accumulators are then zeroed, so the old destination can never recover its share. [6](#0-5) 

### Impact Explanation

The old `adminFeeDestination` — which may be a DAO treasury, a separate multisig, or any contract that is not the pool admin itself — permanently loses all admin-fee revenue that accrued between the last collection and the destination change. The total loss equals `adminSpreadFeeE6 / totalSpreadFeeE6 × surplus + adminNotionalFeeE8 / totalNotionalFeeE8 × notionalAccumulator` at the moment of the call, denominated in both pool tokens. For an active pool this can be a material amount of protocol fee revenue.

### Likelihood Explanation

The trigger is the pool admin calling `setPoolAdminFeeDestination`, which is a normal operational action (treasury rotation, multisig migration). The pool admin is semi-trusted and acts within their authority. The loss is automatic and silent — no separate attacker step is required. Any pool admin rotation or treasury migration that does not manually call `collectPoolFees` first will silently redirect all accrued admin fees.

### Recommendation

Mirror the pattern used by `setPoolAdminFees` and `setPoolProtocolFee`: collect fees at the **current** destination before updating it.

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();

    // Flush accrued fees to the old destination before switching.
    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6,
        c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8,
        c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]   // old destination
    );

    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

### Proof of Concept

1. Pool is deployed with `adminFeeDestination = treasuryA`, `adminSpreadFeeE6 = 5_000` (0.5%), `adminNotionalFeeE8 = 500_000` (0.5%).
2. Traders execute swaps; `notionalFeeToken0Scaled` accumulates to `N` and a spread surplus `S` builds up in the pool balance.
3. Pool admin calls `setPoolAdminFeeDestination(pool, treasuryB)` — no `collectFees` is triggered.
4. Anyone calls `collectPoolFees(pool)`. The factory reads `poolAdminFeeDestination[pool] == treasuryB` and transfers the admin share of both `S` and `N` to `treasuryB`. [7](#0-6) 
5. `notionalFeeToken0Scaled` is zeroed. `treasuryA` receives nothing; its entire accrued admin-fee entitlement is gone. [8](#0-7)
