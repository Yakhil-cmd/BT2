### Title
Malicious Participant Causes Unconditional Liveness DoS in DKG/Reshare/Refresh via `broadcast_success` False Vote — (`src/dkg.rs`)

### Summary

The `broadcast_success` function in `src/dkg.rs` is the final gate of every `do_keyshare` execution, which underlies `do_keygen`, `do_reshare`, and `do_refresh`. It unconditionally aborts the entire protocol for all honest participants if **any single participant** broadcasts `false` or a mismatched session ID. A malicious participant who participates honestly through all prior rounds can trivially broadcast `(false, session_id)` in this final step, causing every honest participant to receive a `ProtocolError::AssertionFailed` and permanently blocking key generation, resharing, and refresh for the entire group.

### Finding Description

`broadcast_success` is called at the very end of `do_keyshare`: [1](#0-0) 

Its implementation: [2](#0-1) 

The function:
1. Calls `do_broadcast(chan, participants, me, (true, session_id))` — each participant reliably broadcasts their success vote.
2. Collects all votes into `vote_list`.
3. Checks `vote_list.iter().all(|(_, ref sid)| sid == &session_id)` — aborts if any session ID mismatches.
4. Checks `vote_list.iter().all(|&(boolean, _)| boolean)` — aborts if **any** participant voted `false`.

The underlying `do_broadcast` is the reliable echo-broadcast protocol: [3](#0-2) 

Reliable broadcast guarantees that if an honest participant delivers a value for a given session, **all** honest participants deliver the same value. Therefore, if a malicious participant reliably broadcasts `(false, session_id)`, every honest participant will receive and deliver that `false` vote, and every honest participant will hit the `all(|&(boolean, _)| boolean)` check and return `Err(ProtocolError::AssertionFailed(...))`.

**Attack path:**
1. Attacker joins the protocol as a legitimate participant (no privileged access needed).
2. Attacker participates honestly through all prior rounds (session ID exchange, commitment broadcast, proof-of-knowledge, share distribution, share verification).
3. In the final `broadcast_success` round, attacker broadcasts `(false, session_id)` instead of `(true, session_id)`.
4. Reliable broadcast delivers the `false` vote to all honest participants.
5. All honest participants abort with `ProtocolError::AssertionFailed("A participant seems to have failed its checks. Aborting Protocol!")`.
6. Attacker can repeat this on every retry attempt, permanently blocking the protocol.

The same `do_keyshare` function is the shared implementation for all three lifecycle operations: [4](#0-3) [5](#0-4) 

### Impact Explanation

A single malicious participant can permanently prevent DKG, reshare, and refresh from completing for all honest parties. No honest party can obtain a valid `KeygenOutput` (signing share + public key) as long as the attacker remains in the participant set. This maps directly to the allowed impact: **High — Permanent denial of key generation, reshare, and refresh for honest parties under valid protocol inputs.**

### Likelihood Explanation

The attacker requires only a valid slot in the participant list — no leaked keys, no privileged access, no cryptographic breaks. The attack is trivially executable: participate honestly for all expensive rounds (to avoid early detection), then send a single `false` bit in the last round. The cost to the attacker is negligible; the cost to honest parties is the full protocol execution wasted on every attempt. Any participant who wishes to block a reshare (e.g., to prevent being removed from the signer set) has a direct economic incentive to exploit this.

### Recommendation

Replace the unanimous-abort check with a fault-tolerant approach:

1. **Identify and exclude the liar, do not abort.** After collecting all votes, identify which participants broadcast `false` (or a wrong session ID), mark them as malicious, and continue with the remaining honest quorum — provided the quorum still meets the threshold.
2. **Remove the `broadcast_success` gate entirely** if the prior rounds already provide sufficient cryptographic assurance (share verification via `validate_received_share` already catches invalid shares at line 522).
3. **If a success-confirmation round is required**, use a threshold-based check: abort only if more than `f` participants report failure, where `f` is the maximum tolerated number of malicious participants, rather than requiring unanimous success. [6](#0-5) 

The two `all(...)` predicates at lines 321 and 329 are the root cause. They should be replaced with threshold-tolerant checks that identify the specific malicious participant(s) rather than aborting the entire protocol.

### Proof of Concept

```
Setup: 4 participants [P0, P1, P2, P3], threshold = 2.
P3 is malicious.

Round 1 (session IDs): P3 participates honestly.
Round 2 (commitment hashes): P3 participates honestly.
Round 3 (commitments + proofs): P3 participates honestly.
Round 4 (secret share distribution): P3 participates honestly.
Round 5 (share verification): P3 participates honestly.

Final round (broadcast_success):
  P0 broadcasts (true,  session_id) ← honest
  P1 broadcasts (true,  session_id) ← honest
  P2 broadcasts (true,  session_id) ← honest
  P3 broadcasts (false, session_id) ← MALICIOUS

Reliable broadcast delivers P3's (false, session_id) to P0, P1, P2.

P0: vote_list.iter().all(|&(b,_)| b) → false → Err(AssertionFailed)
P1: vote_list.iter().all(|&(b,_)| b) → false → Err(AssertionFailed)
P2: vote_list.iter().all(|&(b,_)| b) → false → Err(AssertionFailed)

Result: do_keygen / do_reshare / do_refresh returns Err for all honest parties.
P3 repeats on every retry → permanent DoS.
```

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)

pub async fn do_broadcast<'a, T>(
    chan: &mut SharedChannel,
    participants: &'a ParticipantList,
    me: Participant,
    data: T,
) -> Result<ParticipantMap<'a, T>, ProtocolError>
where
    T: Serialize + Clone + DeserializeOwned + PartialEq,
{
    let wait_broadcast = chan.next_waitpoint();
    let send_vote = reliable_broadcast_send(chan, wait_broadcast, participants, me, data)?;
    let vote_list =
        reliable_broadcast_receive_all(chan, wait_broadcast, participants, me, send_vote).await?;
    Ok(vote_list)
}
