### Title
Unprivileged caller can permanently freeze staker consensus block rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `IStakingRewardsManager::update_rewards` function carries no access control. Any non-zero address may call it with `disable_rewards: true`, which advances the global `last_reward_block` counter without distributing rewards. Because the function enforces a strict "one call per block" invariant, a single attacker transaction per block permanently consumes the reward slot for that block, preventing the legitimate block proposer from ever receiving their consensus reward.

---

### Finding Description

`update_rewards` is the sole entry point for distributing per-block consensus rewards to stakers. Its only guards are `general_prerequisites()` (contract not paused, caller not zero) and the single-call-per-block assertion:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

Regardless of the `disable_rewards` argument, the function unconditionally writes the current block number to the global `last_reward_block` storage slot before branching:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;          // ← exits here; rewards never distributed
}
``` [1](#0-0) 

Because `last_reward_block` is a **single global value** (not per-staker), one call with any valid `staker_address` and `disable_rewards: true` exhausts the reward slot for the entire block. Any subsequent call in the same block — including the legitimate block proposer's call with `disable_rewards: false` — reverts with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

The `general_prerequisites` check imposes no role restriction:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [3](#0-2) 

The public interface confirms the function is exposed to all callers with no role annotation:

```cairo
fn update_rewards(
    ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
);
``` [4](#0-3) 

---

### Impact Explanation

When `disable_rewards: true` is used, `_update_rewards` is never reached. The reward supplier is never notified, no STRK is minted or transferred, and `staker_info.unclaimed_rewards_own` is never incremented. The block reward is not deferred — it is simply never created. A persistent attacker who submits one cheap transaction per block causes the targeted staker (and all other stakers, since `last_reward_block` is global) to receive **zero consensus rewards indefinitely**.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**. [5](#0-4) 

---

### Likelihood Explanation

- The attacker needs only one transaction per block on Starknet (cheap).
- Any non-zero address qualifies; no special role or stake is required.
- The attacker must supply a valid, active `staker_address` with non-zero balance — all staker addresses are publicly observable on-chain.
- On Starknet's current sequencer model, transaction ordering within a block is controlled by the sequencer. A malicious or bribed sequencer, or a competing staker willing to spend gas, can reliably front-run the block proposer's legitimate `update_rewards` call.
- The attacker has no profit motive but can eliminate a competitor's yield entirely.

---

### Recommendation

Restrict `update_rewards` to an authorized caller (e.g., a designated consensus contract or a role such as `BLOCK_PROPOSER`). At minimum, `disable_rewards: true` should require a privileged role, while `disable_rewards: false` may remain callable by the staker themselves. Example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_consensus_contract(); // add role check
    ...
}
```

Alternatively, split the function: one privileged function for the consensus mechanism (with `disable_rewards` flag) and one unprivileged function callable only by the staker themselves (always distributes rewards).

---

### Proof of Concept

1. Staker A is the block proposer for block N and intends to call `update_rewards(A, false)` to collect their block reward.
2. Attacker observes the pending transaction and submits `update_rewards(A, true)` (or `update_rewards(any_active_staker, true)`) with higher priority.
3. Attacker's transaction executes first: `last_reward_block` is set to block N, function returns early — no rewards distributed.
4. Staker A's transaction reverts: `current_block_number > self.last_reward_block.read()` is false.
5. Staker A receives zero rewards for block N.
6. Attacker repeats for every block → Staker A's `unclaimed_rewards_own` never increases → permanent yield freeze. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
