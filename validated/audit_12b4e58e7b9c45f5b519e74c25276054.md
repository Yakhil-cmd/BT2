The `tick()` function in `RewardSupplier.sol` is confirmed vulnerable. Here is the full analysis:

### Title
ETH Sent to `tick()` Is Permanently Locked When `amountToMint == 0` — (`L1/starkware/solidity/stake/RewardSupplier.sol`)

### Summary
`tick()` is declared `payable` and uses `msg.value` to pay L1→L2 messaging fees, but only inside the `if (amountToMint > 0)` branch. When no pending L2→L1 mint-request messages exist, the branch is skipped entirely and `msg.value` is neither consumed nor refunded. No `receive()`, `fallback()`, or `withdraw` function exists anywhere in the contract or its inheritance chain, so any ETH sent in that call is permanently locked.

### Finding Description
`tick()` at line 107 is `external payable`. [1](#0-0) 

`msg.value` is only forwarded to `bridge().depositWithMessage` and `messagingContract().sendMessageToL2` inside the `if (amountToMint > 0)` block. [2](#0-1) 

When `requiredMinting()` returns `(0, 0)` — i.e., `messagingContract().l2ToL1Messages(msgHash) == 0` — the function returns at line 142 without touching `msg.value`. [3](#0-2) 

A grep across all L1 Solidity files for `withdraw`, `receive()`, `fallback()`, and `transfer(` returns zero hits in `RewardSupplier.sol`, `RewardSupplierStorage.sol`, and `ProxySupportImpl.sol`. There is no ETH recovery path. [4](#0-3) [5](#0-4) 

### Impact Explanation
Any ETH attached to a `tick()` call that resolves with `amountToMint == 0` is permanently locked in the contract. The most realistic trigger is a race condition: a caller reads pending messages off-chain, submits `tick{value: X}()`, but a competing transaction mines first and consumes all messages. The caller's ETH is then locked with no recovery mechanism. This is a concrete, permanent loss of caller funds — Medium severity under the allowed scope (griefing / damage to users with no attacker profit).

### Likelihood Explanation
`tick()` is a permissionless, public function intended to be called by any keeper or bot. Keepers routinely monitor pending messages and race to call `tick()`. Front-running or simultaneous submission is a normal operational condition, making this scenario realistic rather than theoretical.

### Recommendation
Add a refund at the end of `tick()` when `amountToMint == 0`:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ... existing logic ...
    } else if (msg.value > 0) {
        // Refund ETH sent when there is nothing to mint.
        (bool ok, ) = msg.sender.call{value: msg.value}("");
        require(ok, "ETH_REFUND_FAILED");
    }
}
```

Alternatively, remove `payable` from `tick()` entirely and require callers to pre-fund the contract via a separate deposit function, decoupling fee payment from the minting trigger.

### Proof of Concept
```solidity

// 2. Call tick{value: 1 ether}() — amountToMint == 0, branch skipped.
// 3. Assert: address(rewardSupplier).balance == 1 ether.
// 4. Assert: no withdraw/receive function exists → ETH is permanently locked.

```

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
