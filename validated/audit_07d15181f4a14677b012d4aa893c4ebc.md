### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. It unconditionally writes `current_block_number` to the global `last_reward_block` storage slot **before** checking `disable_rewards`. An attacker can call `update_rewards(any_valid_staker, true)` every block, consuming the global "reward slot" for that block without distributing any rewards, permanently denying all consensus-mode staking and delegation rewards.

---

### Finding Description

`update_rewards` in `src/staking/staking.cairo` is the sole mechanism for distributing per-block consensus rewards to stakers and their delegation pools. The function enforces a one-call-per-block invariant via a global `last_reward_block` field:

```
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

Critically, `last_reward_block` is written **before** the `disable_rewards` branch:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← written unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing
}
``` [1](#0-0) 

The function has no role guard — `general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero: [2](#0-1) [3](#0-2) 

`last_reward_block` is a single global slot, not per-staker: [4](#0-3) 

**Attack path:**

1. Attacker identifies any valid, active staker address (trivially available from on-chain events).
2. Every block, attacker calls `update_rewards(valid_staker_address, disable_rewards: true)`.
3. `last_reward_block` is set to the current block number; no rewards are distributed.
4. Any legitimate call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`.
5. The block's reward allocation is permanently lost — it is not carried forward.

`calculate_block_rewards` returns a fixed per-block amount; unclaimed blocks are not accumulated: [5](#0-4) 

Rewards forwarded to delegation pools via `update_rewards_from_staking_contract` are also blocked, so delegators' `cumulative_rewards_trace` is never updated: [6](#0-5) 

---

### Impact Explanation

All consensus-mode staking rewards and delegation pool rewards are permanently frozen. Stakers' `unclaimed_rewards_own` and delegators' `cumulative_rewards_trace` are never incremented. Rewards are not deferred — each skipped block's allocation is gone. This matches the **High** impact category: *Permanent freezing of unclaimed yield*.

---

### Likelihood Explanation

The attack requires one transaction per block (~6 s on Starknet). Gas costs on Starknet L2 are very low. The attacker needs no privileged role, no tokens, and no special setup — only a valid staker address, which is public. The attack is indefinitely sustainable at negligible cost, analogous to the BathBuddy griefing on Optimism described in the reference report.

---

### Recommendation

Remove the `disable_rewards` parameter from the public interface, or restrict calls with `disable_rewards: true` to a trusted role (e.g., `security_agent`). Alternatively, only write `last_reward_block` when rewards are actually distributed (move the write after the `disable_rewards` guard). A minimal fix:

```cairo
// Only consume the block slot when rewards are actually distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // moved here
// ... distribute rewards
```

### Proof of Concept

```
// Pseudocode — executable with Starknet Foundry (snforge)
// Precondition: consensus rewards are active (post consensus_rewards_first_epoch).

let attacker = some_unprivileged_address;
let valid_staker = any_active_staker_from_events();

// Attacker calls every block:
loop {
    staking.update_rewards(valid_staker, disable_rewards: true);
    advance_block();
}

// After N blocks, legitimate staker tries to claim:
staking.update_rewards(valid_staker, disable_rewards: false);
// → reverts: REWARDS_ALREADY_UPDATED (attacker already consumed this block)

// Staker's unclaimed_rewards_own remains 0.
// Pool's cumulative_rewards_trace is never updated.
// Delegators receive 0 rewards indefinitely.
```
