### Title
Gateway L2 Gas Price Admission Check Silently Skipped for Legacy `ValidResourceBounds::L1Gas` Transactions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful admission check for L2 gas price (`validate_tx_l2_gas_price_within_threshold`) explicitly skips all `ValidResourceBounds::L1Gas` transactions with the comment "No validation required for legacy transactions." This means a legacy-format transaction can carry an arbitrarily low (including zero) `max_price_per_unit` for L1 gas and still be admitted to the mempool, bypassing the price-floor guard that exists to prevent underpriced transactions from being sequenced at a loss.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, the function `validate_tx_l2_gas_price_within_threshold` enforces that a transaction's stated gas price is at or above a configurable percentage of the previous block's L2 gas price. However, the match arm for `ValidResourceBounds::L1Gas` unconditionally returns `Ok(())`:

```rust
ValidResourceBounds::L1Gas(_) => {
    // No validation required for legacy transactions.
}
``` [1](#0-0) 

This is the **only** gas-price check performed in `validate_resource_bounds` at the stateful gateway layer. The stateless validator (`StatelessTransactionValidator::validate_resource_bounds`) checks `l2_gas.max_price_per_unit` against a static `min_gas_price` floor, but for `L1Gas`-variant transactions the stateless validator only checks the `AllResourceBounds` path — the `l2_gas` field of an `L1Gas` variant is always `ResourceBounds::default()` (zero), so the static floor check is also bypassed. [2](#0-1) 

The `ValidResourceBounds::L1Gas` variant is explicitly documented as the pre-0.13.3 format where "L2 bounds are signed but never used": [3](#0-2) 

The `get_l2_bounds()` method returns `ResourceBounds::default()` (all zeros) for `L1Gas` variants: [4](#0-3) 

The stateful validator's `validate_resource_bounds` reads the previous block's L2 gas price and calls `validate_tx_l2_gas_price_within_threshold`, but the L1Gas branch exits immediately without any price comparison: [5](#0-4) 

The analog to the external bug is exact: just as the Oracle's hardcoded stable price allowed collateral to be deposited at an inflated value, the gateway's hardcoded exemption for legacy transactions allows them to be admitted at an arbitrarily low (zero) gas price — a fixed "peg" of zero — while the actual block gas price may be 8 Gwei or higher.

### Impact Explanation

An attacker can submit `ValidResourceBounds::L1Gas` transactions with `max_price_per_unit = 0` (or any value below the current block's L2 gas price floor). These transactions:

1. **Pass stateless validation** — the stateless check only validates `l2_gas.max_price_per_unit` against `min_gas_price`, but for `L1Gas` variants `l2_gas` is always zero and the check is not applied to the L1 gas price.
2. **Pass stateful gateway admission** — `validate_tx_l2_gas_price_within_threshold` explicitly skips L1Gas variants.
3. **Enter the mempool** — the mempool's own `validate_resource_bounds` flag follows the same shared pointer target and has the same gap.

The sequencer then sequences these transactions. During blockifier execution, `check_fee_bounds` in `AccountTransaction::perform_pre_validation_stage` does compare `resource_bounds.max_price_per_unit` against the actual block gas price and will reject the transaction — but only at execution time, after the transaction has already consumed mempool capacity and sequencer resources. This is a **High** impact admission-layer failure: the gateway/mempool accepts transactions that will predictably fail at execution, enabling mempool spam and denial-of-service against the sequencer. [6](#0-5) 

### Likelihood Explanation

The `ValidResourceBounds::L1Gas` variant is the standard format for all pre-0.13.3 Starknet transactions (V1/V2 invoke, V1 declare). These transaction types are still accepted by the gateway. Any user can craft such a transaction with `max_price_per_unit = 0`. The bypass requires no special privilege — it is triggered by the normal transaction submission path with a legacy transaction type.

### Recommendation

In `validate_tx_l2_gas_price_within_threshold`, add a price check for the `ValidResourceBounds::L1Gas` arm that compares `l1_bounds.max_price_per_unit` against the previous block's L1 gas price (not L2 gas price). The TODO comment at line 358 already acknowledges this gap:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
``` [7](#0-6) 

The fix should mirror the `AllResources` branch but use `block_info.gas_prices.l1_gas_price(fee_type)` as the reference price and `l1_bounds.max_price_per_unit` as the transaction's stated price.

### Proof of Concept

1. Construct a V1 invoke transaction (which uses `ValidResourceBounds::L1Gas`) with `max_price_per_unit = 0` and any valid nonce/signature.
2. Submit it to the gateway's `add_transaction` endpoint.
3. Observe that `StatelessTransactionValidator::validate_resource_bounds` passes (the `l2_gas.max_price_per_unit` check does not apply to the L1Gas variant's zero l2 field).
4. Observe that `StatefulTransactionValidator::validate_resource_bounds` passes (the `L1Gas` match arm returns `Ok(())` unconditionally).
5. The transaction enters the mempool.
6. When the sequencer attempts to execute it, `AccountTransaction::check_fee_bounds` rejects it with `MaxGasPriceTooLow`, but only after it has already consumed admission resources. [8](#0-7) [9](#0-8)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
