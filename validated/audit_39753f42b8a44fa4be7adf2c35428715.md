### Title
Unprivileged Caller Can Permanently Freeze All Consensus-Era Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` has no access control and accepts a caller-controlled `disable_rewards` boolean. Because `last_reward_block` is written **before** the `disable_rewards` guard, any unprivileged address can call `update_rewards(any_active_staker, disable_rewards: true)` once per block to consume the block's reward slot without distributing any rewards, permanently freezing all consensus-era block rewards for every staker in the protocol.

### Finding Description
`update_rewards` in `StakingRewardsManagerImpl` is a public, permissionless function. Its execution order is:

1. Check `current_block_number > last_reward_block` (gate).
2. Validate staker exists and is active.
3. **Write `last_reward_block = current_block_number`** (state mutation).
4. `if disable_rewards || self.is_pre_consensus() { return; }` — exits without distributing rewards. [1](#0-0) 

Because step 3 commits the block number before step 4 checks `disable_rewards`, any caller who passes `disable_rewards: true` permanently marks the current block as "already rewarded" while distributing nothing. The global `last_reward_block` storage field is shared across all stakers: [2](#0-1) 

There is no `assert_caller_is_*` guard anywhere in `update_rewards`. The spec confirms the function is intentionally public: [3](#0-2) 

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

In the consensus era (after `consensus_rewards_first_epoch` activates), block rewards are the sole reward mechanism for all stakers. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block:

- Consumes the single per-block reward slot (`last_reward_block` is set).
- Distributes zero rewards to any staker.
- Blocks every subsequent legitimate `update_rewards` call in that block with `REWARDS_ALREADY_UPDATED`.

Repeating this every block permanently freezes all consensus-era yield for every staker and every pool member in the protocol. The rewards are not deferred — they are silently skipped and never recoverable for that block.

### Likelihood Explanation
**High.** The attacker needs no stake, no special role, and no prior relationship with any staker. The only cost is gas. The attack is mechanically simple: submit one transaction per block. A motivated griefer (e.g., a competing validator or a protocol adversary) can sustain this indefinitely.

### Recommendation
Add an access-control check to `update_rewards` so that only an authorized caller (e.g., the attestation contract, a designated operator role, or the staker/operational address itself) can invoke it. Alternatively, move the `last_reward_block.write(...)` call to **after** the `disable_rewards` guard so that a no-op call does not consume the block's reward slot.

### Proof of Concept
```
// Attacker (any EOA) calls every block in the consensus era:
IStakingRewardsManagerDispatcher { contract_address: staking }
    .update_rewards(staker_address: any_active_staker, disable_rewards: true);

// Result:
//   last_reward_block = current_block  (slot consumed)
//   No STRK/BTC rewards distributed
//
// Any legitimate call in the same block reverts with REWARDS_ALREADY_UPDATED.
// Repeating each block → all consensus block rewards permanently frozen.
``` [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
