### Title
Pool Contract `claim_rewards` Bypasses Staking Contract Pause Mechanism - (File: src/pool/pool.cairo)

---

### Summary

The `Pool` contract's `claim_rewards` function does not check the staking contract's `is_paused` state before transferring STRK rewards to pool members. This is the direct analog to the reported vulnerability class: a state variable (`is_paused`) controls intended behavior, but a critical code path has no mechanism to enforce it, leaving the protection incomplete.

---

### Finding Description

The staking contract (`src/staking/staking.cairo`) implements a pause mechanism via the `is_paused` storage variable and the `IStakingPause` interface. Every state-changing function in the staking contract calls `self.general_prerequisites()` first, which enforces the pause. [1](#0-0) [2](#0-1) 

The staking contract's own `claim_rewards` is correctly gated: [3](#0-2) 

However, the pool contract's `claim_rewards` function has **no pause check whatsoever**. It reads directly from the pool's internal `cumulative_rewards_trace` and transfers STRK to the pool member's reward address without ever querying the staking contract's `is_paused` state: [4](#0-3) 

The STRK balance available for claiming in the pool contract was deposited there by the staking contract via `update_rewards_from_staking_contract`. Once deposited, the staking contract's pause has no reach over it: [5](#0-4) 

All other pool functions that interact with the staking contract (`enter_delegation_pool`, `add_to_delegation_pool`, `exit_delegation_pool_intent`, `exit_delegation_pool_action`, `switch_delegation_pool`) are indirectly blocked by the pause because they call staking contract functions (`add_stake_from_pool`, `remove_from_delegation_pool_intent`, `remove_from_delegation_pool_action`, `switch_staking_delegation_pool`) that each call `general_prerequisites()`: [6](#0-5) [7](#0-6) [8](#0-7) 

`pool.claim_rewards` is the sole exception — it never calls back into the staking contract and therefore is never gated by the pause. [9](#0-8) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

The pause is the protocol's primary emergency brake. Its documented purpose is to prevent any state changes (balance changes, reward claims, etc.) during a security incident. If a reward calculation bug causes inflated rewards to be forwarded to pool contracts via `update_rewards_from_staking_contract`, the security agent can pause the staking contract to freeze all claims while the bug is investigated. However, pool members can immediately call `pool.claim_rewards` to drain the inflated STRK rewards from the pool contract before the issue is remediated. Stakers are blocked from claiming their own inflated rewards (staking `claim_rewards` checks `general_prerequisites()`), but pool members face no such restriction. This asymmetry allows pool members to extract incorrectly minted yield that the pause was specifically intended to protect. [10](#0-9) 

---

### Likelihood Explanation

**Medium.** The attack requires: (1) a reward calculation bug that causes inflated rewards to be sent to pool contracts, and (2) the security agent pausing the staking contract in response. Both conditions are realistic — the protocol explicitly provides a pause mechanism for exactly this class of incident. Once paused, any pool member can immediately exploit the gap with a single `claim_rewards` call, requiring no special privileges.

---

### Recommendation

Add a check in `pool.claim_rewards` that queries the staking contract's `is_paused()` view function and reverts if the staking contract is paused:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    // Add pause check:
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
    // ... rest of function
}
```

Alternatively, expose a `pause`/`unpause` interface on the pool contract itself, callable only by the staking contract, so the pool's pause state is kept in sync.

---

### Proof of Concept

1. Reward calculation bug causes inflated rewards to be forwarded to a pool contract via `staking._update_rewards` → `pool.update_rewards_from_staking_contract`. The pool's `cumulative_rewards_trace` now reflects inflated values.
2. Security agent calls `staking.pause()`. `is_paused` is set to `true`.
3. Staker calls `staking.claim_rewards(staker_address)` → reverts at `general_prerequisites()` because `is_paused == true`.
4. Pool member calls `pool.claim_rewards(pool_member)` → **succeeds**. No pause check exists. The function reads from `cumulative_rewards_trace`, computes inflated rewards, and transfers STRK to the pool member's reward address.
5. Pool member has extracted inflated unclaimed yield that the pause was intended to freeze. [4](#0-3) [3](#0-2)

    impl StakingPauseImpl of IStakingPause<ContractState> {
        fn pause(ref self: ContractState) {
            self.roles.only_security_agent();
            if self.is_paused() {
                return;
            }
            self.is_paused.write(true);
            self.emit(PauseEvents::Paused { account: get_caller_address() });
        }

        fn unpause(ref self: ContractState) {
            self.roles.only_security_admin();
            if !self.is_paused() {
                return;
            }
            self.is_paused.write(false);
            self.emit(PauseEvents::Unpaused { account: get_caller_address() });
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
