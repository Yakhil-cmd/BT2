### Title
Any address can force `exit_delegation_pool_action` on a delegator with no upper deadline, blocking pool switching and causing reward loss - (File: src/pool/pool.cairo)

### Summary
`exit_delegation_pool_action` in `pool.cairo` accepts any caller and has no upper deadline. Once a delegator's exit wait window expires, any third party can force the delegator's exit at any time, clearing the intent before the delegator can invoke `switch_delegation_pool`. This causes the delegator to miss rewards during the forced re-delegation period and pay extra gas to re-enter a pool.

### Finding Description
`exit_delegation_pool_action` (pool.cairo:295–333) takes `pool_member: ContractAddress` as a parameter and performs no `get_caller_address()` check. The spec explicitly states "Any address can execute." The only guard is the minimum wait window (`Time::now() >= unpool_time`), but there is no upper deadline — the intent stays open indefinitely.

```cairo

        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

```

A delegator who calls `exit_delegation_pool_intent` with the intention of switching to a better pool (via `switch_delegation_pool`) is racing against any address that can call `exit_delegation_pool_action` first. If the action is forced before the delegator's switch transaction executes, the intent is cleared, the funds are returned, and the delegator loses the ability to switch atomically. The delegator must then re-delegate from scratch, missing rewards during the gap.

The root cause is the same vulnerability class as the external report: **missing time-bound validation on a user-initiated operation**. The intent has no expiry (no upper deadline), and the action is callable by any address, meaning the delegator's pending operation can be executed at an unexpected time by a third party.

### Impact Explanation
**Medium — Griefing with no profit motive but damage to users.**

The delegator's principal is returned intact, but:
1. The delegator loses the ability to switch pools atomically.
2. The delegator must re-delegate to the target pool in a separate transaction, incurring extra gas.
3. During the re-delegation period the delegator earns zero rewards, whereas a direct `switch_delegation_pool` would have resumed reward accrual immediately in the new pool.

The reward loss is proportional to the delegator's stake and the length of the forced re-delegation gap.

### Likelihood Explanation
**Medium.** No special privileges are required. Any address can call `exit_delegation_pool_action` after the wait window. A griefer only needs to monitor on-chain events for `PoolMemberExitIntent` emissions, wait for the wait window to expire, and submit the action before the delegator's switch transaction. On Starknet the sequencer is currently centralized, reducing front-running risk, but the griefer does not need to front-run — they only need to act before the delegator does, which is possible any time after the wait window.

### Recommendation
Introduce an upper deadline on the intent, or restrict `exit_delegation_pool_action` to be callable only by the pool member (or their reward address) unless a configurable grace period after the wait window has elapsed. This mirrors the recommendation in the external report: add a deadline parameter so the user controls the execution window.

### Proof of Concept

1. Delegator holds 1 000 STRK delegated to Staker A's pool.
2. Delegator calls `exit_delegation_pool_intent(1000)` intending to switch to Staker B.
   - `unpool_time` is set to `Time::now() + exit_wait_window`.
   - Delegator's balance is removed from the staking pool; rewards stop accruing.
3. The exit wait window elapses (e.g., 1 week).
4. Delegator prepares to call `switch_delegation_pool(to_staker=B, to_pool=B_pool, amount=1000)`.
5. **Griefer** calls `pool.exit_delegation_pool_action(pool_member=delegator)` first.
   - `Time::now() >= unpool_time` passes.
   - Intent is cleared; 1 000 STRK is transferred back to the delegator.
6. Delegator's `switch_delegation_pool` call now reverts with `MISSING_UNDELEGATE_INTENT` because the intent was already consumed.
7. Delegator must call `add_to_delegation_pool` on Staker B's pool from scratch, paying extra gas and missing rewards for the entire re-delegation period.

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );
