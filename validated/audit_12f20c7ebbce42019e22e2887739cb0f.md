### Title
Missing Authorization Status Check in Seal Signature Verification Allows Ejected Verifiers' Signatures to Count Toward Sealing Threshold - (File: `module/validation/seal_validator.go`)

---

### Summary

`sealValidator.verifySealSignature` retrieves a verifier's identity via `identityForNode` but does not check whether the signer is ejected, has positive weight, or holds the Verification role. A Byzantine consensus node proposer can include a seal whose chunk signatures come from ejected verification nodes. Because ejected nodes retain their staking keys, their signatures pass cryptographic verification, and the missing authorization re-check allows those signatures to count toward the sealing threshold.

---

### Finding Description

`sealValidator.verifySealSignature` in `module/validation/seal_validator.go` is called during seal payload validation to confirm that each signer's BLS signature is cryptographically valid:

```go
nodeIdentity, err := identityForNode(s.state, chunk.BlockID, signerId)
if err != nil {
    return err
}
valid, err := nodeIdentity.StakingPubKey.Verify(signature, atstID[:], s.signatureHasher)
``` [1](#0-0) 

`identityForNode` only checks that the node exists in the protocol state at `chunk.BlockID`. It does **not** check ejection status, weight, or role:

```go
func identityForNode(...) (*flow.Identity, error) {
    identity, err := state.AtBlockID(blockID).Identity(nodeID)
    if err != nil {
        if protocol.IsIdentityNotFound(err) { ... }
        ...
    }
    return identity, nil   // returned regardless of ejection status
}
``` [2](#0-1) 

By contrast, `validateReceipt` in `module/validation/receipt_validator.go` correctly calls both `identityForNode` **and** `ensureNodeHasWeightAndRole`, which enforces role, positive weight, and `EpochParticipationStatusActive`: [3](#0-2) 

`ensureNodeHasWeightAndRole` explicitly rejects ejected nodes: [4](#0-3) 

The chunk assignment check inside `validateSeal` uses the protocol state at the **incorporating block**:

```go
assignments, err := s.assigner.Assign(executionResult, incorporatedResult.IncorporatedBlockID)
``` [5](#0-4) 

If a verifier was authorized at the incorporating block but was subsequently ejected, it remains in the computed assignment. The ejection status is never re-checked at seal validation time, so the ejected verifier's signature passes both the assignment check and the signature check.

The `VerifyingAssignmentCollector` does pre-select authorized verifiers at construction time via `authorizedVerifiersAtBlock`, which filters for non-ejected, positive-weight nodes: [6](#0-5) 

However, `sealValidator.verifySealSignature` is a **separate code path** used when consensus nodes validate seals in candidate blocks, and it does not perform these checks.

---

### Impact Explanation

A Byzantine consensus node proposer can craft a seal whose `AggregatedApprovalSigs` include signatures from ejected verification nodes. Because:

1. Ejected nodes retain their staking keys and can produce valid BLS signatures.
2. The assignment check passes (assignment computed at incorporating block, before ejection).
3. `verifySealSignature` accepts the signature without checking ejection status.

Ejected verifiers' signatures count toward the `requireApprovalsForSealVerification` threshold. In the worst case, a proposer can construct a seal that meets the threshold entirely with signatures from ejected (and therefore untrusted) verifiers, allowing an incorrect execution result to be sealed and finalized. Sealing an incorrect execution result is a critical protocol violation: it permanently commits a wrong state root to the chain.

---

### Likelihood Explanation

Exploitation requires a Byzantine consensus node proposer and a window where a verifier is ejected between the incorporating block and the sealing block. Node ejection is a real protocol mechanism (`EjectNode` service event). The window can span many blocks (sealing lags finalization by design). The ejected verifier must produce a `ResultApproval` (possible before or after ejection, since keys are unchanged). This is a realistic scenario with medium likelihood.

---

### Recommendation

In `verifySealSignature`, after calling `identityForNode`, add a call to `ensureNodeHasWeightAndRole(nodeIdentity, flow.RoleVerification)` to enforce that the signer is an active, non-ejected verification node with positive weight — consistent with how `validateReceipt` handles this check.

```go
nodeIdentity, err := identityForNode(s.state, chunk.BlockID, signerId)
if err != nil {
    return err
}
// ADD: reject ejected / zero-weight / wrong-role signers
if err := ensureNodeHasWeightAndRole(nodeIdentity, flow.RoleVerification); err != nil {
    return engine.NewInvalidInputErrorf("signer %x is not an authorized verifier: %w", signerId, err)
}
valid, err := nodeIdentity.StakingPubKey.Verify(signature, atstID[:], s.signatureHasher)
```

---

### Proof of Concept

1. Verifier V is authorized at incorporating block `B_inc` and assigned to verify chunk C.
2. An `EjectNode` service event for V is processed at block `B_eject > B_inc`. V's `EpochParticipationStatus` becomes `EpochParticipationStatusEjected`.
3. V (still holding its staking key) produces a `ResultApproval` for chunk C and sends it to the consensus node.
4. A Byzantine consensus node proposer builds a candidate block containing a seal for the result. The seal's `AggregatedApprovalSigs[C.Index]` includes V's `SignerID` and V's BLS signature.
5. `sealValidator.Validate` → `validateSeal` → `verifySealSignature` is called.
6. `assignments.HasVerifier(C.Index, V.NodeID)` returns `true` — assignment was computed at `B_inc`, before ejection. [7](#0-6) 
7. `identityForNode(s.state, chunk.BlockID, V.NodeID)` returns V's identity without error — ejected nodes remain in the protocol state. [2](#0-1) 
8. `nodeIdentity.StakingPubKey.Verify(signature, ...)` returns `true` — V's key is unchanged. [8](#0-7) 
9. The seal is accepted. V's ejected-verifier signature counted toward the sealing threshold, despite V no longer being an authorized protocol participant.

func identityForNode(state protocol.State, blockID flow.Identifier, nodeID flow.Identifier) (*flow.Identity, error) {
	// get the identity of the origin node
	identity, err := state.AtBlockID(blockID).Identity(nodeID)
	if err != nil {
		if protocol.IsIdentityNotFound(err) {
			return nil, engine.NewInvalidInputErrorf("unknown node identity: %w", err)
		}
		if errors.Is(err, protocolstate.ErrUnknownSnapshotReference) {
			return nil, module.NewUnknownBlockError("block %v is unknown: %w", blockID, err)
		}
		return nil, fmt.Errorf("unexpected exception retrieving node identity: %w", err)
	}

	return identity, nil
}

func ensureNodeHasWeightAndRole(identity *flow.Identity, expectedRole flow.Role) error {
	// check that the role is expected
	if identity.Role != expectedRole {
		return engine.NewInvalidInputErrorf("expected node %x to have role %s but got %s", identity.NodeID, expectedRole, identity.Role)
	}
	// check if the identity has non-zero weight
	if identity.InitialWeight == 0 {
		return engine.NewInvalidInputErrorf("node %x has zero weight", identity.NodeID)
	}
	// check if the identity is a valid epoch participant(is active in the current epoch + not ejected)
	if !filter.IsValidCurrentEpochParticipant(identity) {
		return engine.NewInvalidInputErrorf("node %x is not an active participant, instead has status: %s", identity.NodeID, identity.EpochParticipationStatus.String())
	}
	return nil
```

// authorizedVerifiersAtBlock pre-select all authorized Verifiers at the block that incorporates the result.
// The method returns the set of all node IDs that:
//   - are authorized members of the network at the given block and
//   - have the Verification role and
//   - have _positive_ weight and
//   - are not ejected
func authorizedVerifiersAtBlock(state protocol.State, blockID flow.Identifier) (map[flow.Identifier]*flow.Identity, error) {
	authorizedVerifierList, err := state.AtBlockID(blockID).Identities(
		filter.And(
			filter.HasRole[flow.Identity](flow.RoleVerification),
			filter.HasInitialWeight[flow.Identity](true),
			filter.IsValidCurrentEpochParticipant,
		))
	if err != nil {
		return nil, fmt.Errorf("failed to retrieve Identities for block %v: %w", blockID, err)
	}
	if len(authorizedVerifierList) == 0 {
		return nil, fmt.Errorf("no authorized verifiers found for block %v", blockID)
	}

	return authorizedVerifierList.Lookup(), nil
}
