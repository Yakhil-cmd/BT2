### Title
Gateway Stateful Validator Admits Transactions with Zero L1/L1-Data Gas Price That Will Always Fail Blockifier Pre-Validation - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check validates only the L2 gas price against the previous block's price. It does not validate `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The blockifier's `check_fee_bounds`, called during `perform_pre_validation_stage`, validates all three gas prices against the actual block prices. Any user can submit a V3 transaction with `l1_gas.max_price_per_unit = 0` that passes gateway admission, enters the mempool, and then deterministically fails at blockifier pre-validation — enabling costless mempool flooding and batcher resource exhaustion.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` calls `validate_tx_l2_gas_price_within_threshold`, which only inspects `l2_gas.max_price_per_unit`: [1](#0-0) 

The function explicitly skips L1 and L1-data gas price checks, with a developer TODO acknowledging the gap: [2](#0-1) 

The stateless validator also only checks `l2_gas.max_price_per_unit` against a static floor and `l2_gas.max_amount` against a ceiling — no L1 gas price check: [3](#0-2) 

In contrast, the blockifier's `check_fee_bounds` (called inside `perform_pre_validation_stage`) checks all three gas prices against the actual block prices: [4](#0-3) 

The check compares `resource_bounds.max_price_per_unit < actual_gas_price.get()` for L1Gas, L1DataGas, and L2Gas. A transaction with `l1_gas.max_price_per_unit = 0` will always fail this check because the actual L1 gas price is always nonzero.

The execution path is:

1. `StatelessTransactionValidator::validate` — passes (no L1 gas price check).
2. `StatefulTransactionValidator::validate_resource_bounds` → `validate_tx_l2_gas_price_within_threshold` — passes (only L2 gas price checked).
3. Transaction admitted to mempool.
4. Batcher calls `AccountTransaction::execute` → `perform_pre_validation_stage` → `check_fee_bounds` — **fails** with `TransactionFeeError::InsufficientResourceBounds { errors: [ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, ... }] }`. [5](#0-4) 

The nonce is incremented before `check_fee_bounds` is called, but since the batcher wraps execution in a `TransactionalState`, the nonce increment is rolled back on failure. The transaction is simply dropped from the block — but it already consumed a mempool slot and batcher processing time.

### Impact Explanation

An attacker can submit an unbounded number of V3 invoke transactions with `l1_gas.max_price_per_unit = 0` and a valid L2 gas price. Each transaction:

- Passes all gateway checks and enters the mempool.
- Occupies a mempool slot, displacing legitimate transactions once the mempool capacity is reached.
- Forces the batcher to attempt execution, fail at pre-validation, and discard the transaction — wasting CPU and I/O on every block.

Because gateway submission carries no fee and pre-validation failures charge no fee, the attack is costless. The invariant broken is: *the gateway must not admit transactions that are statically guaranteed to fail at blockifier pre-validation*.

This matches the allowed impact: **High — Mempool/gateway admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

The trigger is unprivileged: any caller of the `add_tx` RPC endpoint can submit a V3 transaction with `l1_gas.max_price_per_unit = 0`. The L1 gas price on Starknet mainnet is always nonzero, so the failure at `check_fee_bounds` is deterministic. No special account state, timing, or chain condition is required.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding prices, mirroring the logic already applied to L2 gas. The existing TODO comment at line 358 already flags this gap:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
```

The fix should apply the same percentage-threshold logic used for L2 gas to L1 gas and L1 data gas, reading `previous_block_l1_gas_price` and `previous_block_l1_data_gas_price` from the `GasPrices` struct already fetched via `get_block_info()`.

### Proof of Concept

Submit the following V3 invoke transaction to the gateway's `add_tx` endpoint:

```json
{
  "type": "INVOKE",
  "version": "0x3",
  "sender_address": "<any valid deployed account>",
  "calldata": ["..."],
  "resource_bounds": {
    "l1_gas":      { "max_amount": "0x100", "max_price_per_unit": "0x0" },
    "l2_gas":      { "max_amount": "0x100000", "max_price_per_unit": "<above_threshold>" },
    "l1_data_gas": { "max_amount": "0x100", "max_price_per_unit": "0x0" }
  },
  "nonce": "<current_nonce>",
  "signature": ["..."]
}
```

**Expected (current behaviour):** Gateway returns success; transaction enters the mempool. When the batcher executes it, `check_fee_bounds` returns `InsufficientResourceBounds { errors: [MaxGasPriceTooLow { resource: L1Gas }] }`; the transaction is dropped without being included in any block.

**Expected (after fix):** Gateway rejects the transaction at stateful validation with a `GAS_PRICE_TOO_LOW` error for L1Gas, mirroring the existing L2 gas price rejection path shown in: [6](#0-5)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;
