### Title
Silo Mode `fixed_gas` Bypass via Zero `max_fee_per_gas` Allows Free EVM Execution — (`engine/src/engine.rs`)

---

### Summary

In silo mode, when `fixed_gas` is set, an attacker can submit a transaction with `max_fee_per_gas = 0` to bypass the fee-enforcement logic in `charge_gas`. Because `block_base_fee_per_gas` is always zero in the current implementation, `effective_gas_price` resolves to zero, `prepaid_amount` becomes zero, the sender's balance is not reduced, and `refund_unused_gas` exits immediately without paying the relayer. The result is fully free EVM execution with zero relayer reward.

---

### Finding Description

**Early-return guard in `charge_gas`** (lines 476–481):

```rust
if transaction.max_fee_per_gas.is_zero()
    && fixed_gas.is_none()          // ← false in silo mode
    && block_base_fee_per_gas.is_zero()
{
    return Ok(GasPaymentResult::default());
}
```

The guard requires all three conditions. In silo mode `fixed_gas.is_none()` is `false`, so the guard never fires even when `max_fee_per_gas = 0`. [1](#0-0) 

**`effective_gas_price` derivation** (lines 487–493):

```rust
let priority_fee_per_gas = transaction
    .max_priority_fee_per_gas
    .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
// With max_fee_per_gas=0, block_base_fee=0 → priority_fee_per_gas = 0
let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas; // = 0
``` [2](#0-1) 

**`prepaid_amount` computation** (lines 496–500):

```rust
let prepaid_amount = fixed_gas
    .map_or(transaction.gas_limit, EthGas::as_u256)
    .checked_mul(effective_gas_price)   // fixed_gas * 0 = 0
    .map(Wei::new)
    .ok_or(GasPaymentError::EthAmountOverflow)?;
```

`prepaid_amount = 0`, so `checked_sub(0)` leaves the sender's balance unchanged. [3](#0-2) 

**`refund_unused_gas` early exit** (lines 1270–1272):

```rust
if gas_result.effective_gas_price.is_zero() {
    return Ok(());
}
```

Because `effective_gas_price = 0`, the function returns immediately. No refund is computed, no relayer reward is paid, and no base fee is burned. [4](#0-3) 

The full `refund_unused_gas` function that would otherwise distribute rewards is skipped entirely. [5](#0-4) 

---

### Impact Explanation

The primary confirmed impact is **theft of relayer rewards (High — theft of unclaimed yield)**. The relayer performs work (submitting the NEAR transaction, paying NEAR gas) and expects to receive `fixed_gas × gas_price` in ETH. With `gas_price = 0`, the relayer receives nothing while still bearing NEAR costs.

A secondary effect is **free arbitrary EVM computation**: the attacker can execute any EVM call — including value-moving calls (ETH transfers, ERC-20 transfers, DeFi interactions) — without paying any fee. This breaks the silo operator's economic model entirely and enables spam/DoS at zero cost.

The "Critical — direct theft of user funds" framing in the question is overstated for the gas-bypass alone. The attacker does not gain funds they did not already hold; they avoid paying fees. The clearest in-scope impact is **High — theft of unclaimed yield** (relayer rewards).

---

### Likelihood Explanation

- Preconditions are minimal: silo mode active, attacker is whitelisted (or whitelist disabled), attacker submits a standard EIP-1559 tx with `max_fee_per_gas = 0`.
- No admin compromise, no key leakage, no external oracle required.
- `block_base_fee_per_gas` is structurally zero in the current codebase, making the third guard condition permanently satisfied.
- The path is fully exercisable with the existing test harness (`AuroraRunner` + `submit_with_signer`).

---

### Recommendation

Add a minimum gas price floor check inside `charge_gas` when `fixed_gas` is set. If `fixed_gas.is_some()` and `effective_gas_price.is_zero()`, the function should either return an error or enforce a minimum price configured by the silo operator. For example:

```rust
if fixed_gas.is_some() && effective_gas_price.is_zero() {
    return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
}
```

Alternatively, the silo configuration could include a `min_gas_price` field that is enforced before computing `prepaid_amount`.

### Proof of Concept

Using the existing unit-test infrastructure in `engine/src/engine.rs`:

```rust
#[test]
fn test_silo_fixed_gas_zero_price_bypass() {
    let origin = Address::zero();
    let current_account_id = AccountId::default();
    let env = Fixed::default();
    let storage = RefCell::new(Storage::default());
    let mut io = StoragePointer(&storage);
    let initial_balance = Wei::new_u64(1_000_000);
    add_balance(&mut io, &origin, initial_balance).unwrap();

    let mut engine: Engine<_, _> =
        Engine::new_with_state(EngineState::default(), origin, current_account_id, io, &env);

    let transaction = NormalizedEthTransaction {
        address: Address::default(),
        chain_id: None,
        nonce: U256::default(),
        gas_limit: 300_000.into(),
        max_priority_fee_per_gas: U256::zero(), // attacker sets 0
        max_fee_per_gas: U256::zero(),           // attacker sets 0
        to: None,
        value: Wei::default(),
        data: vec![],
        access_list: vec![],
        authorization_list: vec![],
    };

    let fixed_gas = Some(EthGas::new(1_000)); // silo operator set fixed gas

    let result = engine.charge_gas(&origin, &transaction, None, fixed_gas).unwrap();

    // prepaid_amount = fixed_gas * 0 = 0 → sender pays nothing
    assert_eq!(result.prepaid_amount, Wei::zero());
    assert_eq!(result.effective_gas_price, U256::zero());

    // Balance is unchanged — sender executed for free
    let balance_after = get_balance(&engine.io, &origin);
    assert_eq!(balance_after, initial_balance); // FAILS the invariant: should have decreased
}
```

This test passes on unmodified code, confirming the bypass. [6](#0-5)

```rust
pub fn refund_unused_gas<I: IO>(
    io: &mut I,
    sender: &Address,
    gas_used: u64,
    gas_result: &GasPaymentResult,
    relayer: &Address,
    fixed_gas: Option<EthGas>,
) -> Result<(), GasPaymentError> {
    if gas_result.effective_gas_price.is_zero() {
        return Ok(());
    }

    let (refund, relayer_reward) = {
        let gas_to_wei = |price: U256| {
            fixed_gas
                .map_or_else(|| gas_used.into(), EthGas::as_u256)
                .checked_mul(price)
                .map(Wei::new)
                .ok_or(GasPaymentError::EthAmountOverflow)
        };

        let spent_amount = gas_to_wei(gas_result.effective_gas_price)?;
        let reward_amount = gas_to_wei(gas_result.priority_fee_per_gas)?;

        let refund = gas_result
            .prepaid_amount
            .checked_sub(spent_amount)
            .ok_or(GasPaymentError::EthAmountOverflow)?;

        (refund, reward_amount)
    };

    if !refund.is_zero() {
        add_balance(io, sender, refund)?;
    }

    if !relayer_reward.is_zero() {
        add_balance(io, relayer, relayer_reward)?;
    }

    Ok(())
}
