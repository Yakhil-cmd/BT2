### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via `setPoolBinAdditionalFees` — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

The factory enforces `maxAdminSpreadFeeE6` and `maxAdminNotionalFeeE8` caps on `setPoolAdminFees`, but the parallel admin entry-point `setPoolBinAdditionalFees` carries **no cap check at all**. A pool admin can therefore charge per-bin fees up to `uint16.max = 65 535` (≈ 6.55 % in E6 units) on every bin, regardless of what the factory owner has set as the maximum allowed admin fee.

---

### Finding Description

The factory owner controls fee caps via `setFeeCaps`: [1](#0-0) 

When the pool admin calls `setPoolAdminFees`, the cap is enforced: [2](#0-1) 

However, the second admin fee-setting path, `setPoolBinAdditionalFees`, passes the values straight through to the pool with **no cap validation**: [3](#0-2) 

The pool's `setBinAdditionalFees` only validates the bin index range, not the fee magnitudes: [4](#0-3) 

The bin additional fees are then added directly on top of the base spread fee during every swap: [5](#0-4) 

---

### Impact Explanation

**Impact: Medium–High.** Traders pay more fees than the factory owner's cap permits. Every swap through an affected bin incurs `addFeeBuyE6` or `addFeeSellE6` (up to 6.55 %) on top of the base spread fee, directly reducing the output token amount received. This is a direct loss of user principal on every swap, not a theoretical edge case.

---

### Likelihood Explanation

**Likelihood: High.** The pool admin is a semi-trusted role that is explicitly expected to be constrained by factory-owner caps. The call to `setPoolBinAdditionalFees` requires no timelock, no proposal step, and no co-signer — the pool admin can execute it in a single transaction at any time after pool creation.

---

### Recommendation

Add a cap check inside `setPoolBinAdditionalFees` before forwarding to the pool, analogous to the check already present in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(
    address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6
) external override nonReentrant onlyPoolAdmin(pool) {
    if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce the cap inside `MetricOmmPool.setBinAdditionalFees` by reading the factory's `maxAdminSpreadFeeE6` directly.

---

### Proof of Concept

1. Factory owner calls `setFeeCaps(0, 0, 0, 0)` to zero out all admin fee caps.
2. Pool admin attempts `setPoolAdminFees(pool, 1, 0)` → reverts with `AdminFeeTooHigh`.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` → **succeeds** with no revert.
4. Every swap through bin 0 now pays an additional ≈ 6.55 % fee on top of the base spread, directly extracted from the trader's output amount, in violation of the factory owner's explicit cap of 0. [3](#0-2) [4](#0-3)
