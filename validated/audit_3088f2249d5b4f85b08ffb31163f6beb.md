### Title
Unbounded `stakers` Vec Growth Causes Permanent Griefing of `get_stakers()` - (File: src/staking/staking.cairo)

### Summary
The `stakers` storage Vec in `staking.cairo` grows monotonically — stakers are explicitly never removed from it after unstaking. The `get_stakers()` function iterates over the **entire** Vec on every call, including all historical stakers who have long since exited. An unprivileged attacker can inflate this Vec at the cost of gas fees alone (stake funds are recovered after unstaking), eventually making `get_stakers()` prohibitively expensive or impossible to execute within Starknet's execution-step limits.

### Finding Description

The storage field is self-documented as permanently append-only: [1](#0-0) 

Every call to `stake()` unconditionally pushes the new staker address: [2](#0-1) 

`get_stakers()` then iterates over the **full range** of this Vec on every invocation: [3](#0-2) 

Inactive/unstaked stakers are skipped with `continue`, but every entry is still read from storage, so the gas cost scales linearly with the total number of addresses ever staked — not just the currently active set.

### Impact Explanation

`get_stakers()` is the `IStakingConsensus` interface function used by the consensus layer to determine the active validator set for a given epoch. [4](#0-3) 

If the Vec grows large enough to exceed Starknet's per-call execution-step limit, `get_stakers()` will revert on every call. The consensus layer loses the ability to read the validator set, constituting **unbounded gas consumption** and **griefing with damage to the protocol** — both within the allowed Medium impact scope.

### Likelihood Explanation

The attack is economically cheap: an attacker stakes the minimum amount from address A, waits for the exit window, unstakes (recovering all principal), then repeats with address B, C, … Each cycle permanently adds one entry to the Vec at the cost of gas fees only. Because `assert_staker_address_not_reused` prevents re-staking the same address, each cycle requires a fresh address, but no capital is permanently lost. [5](#0-4) 

The flow test confirms that fully-exited stakers remain in the Vec indefinitely: [6](#0-5) 

### Recommendation

1. **Remove stakers from the Vec on `unstake_action`**: Replace the `Vec` with a data structure that supports O(1) removal (e.g., a swap-and-pop pattern or an `IterableMap`).
2. **Alternatively, maintain a separate active-staker count/set** and iterate only over active stakers in `get_stakers()`, rather than the full historical Vec.
3. **Add a maximum staker cap** as a short-term mitigation to bound Vec growth.

### Proof of Concept

```
1. Attacker creates addresses A_1 … A_N, each funded with min_stake + gas.
2. Each A_i calls stake(min_stake, ...) → stakers Vec grows by 1.
3. After exit_wait_window, each A_i calls unstake_action() → funds returned.
4. stakers Vec now has N permanent entries for fully-exited stakers.
5. get_stakers(epoch_id) must read all N entries on every call.
6. Repeat until get_stakers() exceeds the Starknet execution-step limit and reverts.
7. Consensus layer can no longer query the validator set.
```

The cost per cycle is gas fees only; principal is fully recovered. The Vec size is permanently inflated with no protocol mechanism to prune it. [7](#0-6)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
