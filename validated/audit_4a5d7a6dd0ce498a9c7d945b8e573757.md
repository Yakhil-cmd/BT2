### Title
Staker Can Instantly Raise Commission to 100% via `set_commission_commitment` + `set_commission`, Stealing Delegator Yield — (File: `src/staking/staking.cairo`)

### Summary
A staker can raise their commission from any value to up to 100% in two back-to-back transactions by first calling `set_commission_commitment` (which takes effect immediately) and then calling `set_commission`. Because delegators are subject to a mandatory `exit_wait_window` of at least one week, they cannot exit before the next reward distribution, and all yield accrued during that window is redirected to the staker.

### Finding Description
The `set_commission_commitment` function allows a staker to register a `CommissionCommitment` with an arbitrary `max_commission` (up to `COMMISSION_DENOMINATOR = 10000`, i.e. 100%). The commitment becomes active the moment it is written to storage — there is no activation delay or epoch offset. [1](#0-0) 

Once the commitment is active, `update_commission` (called internally by `set_commission`) permits raising the commission to any value `<= max_commission`, as long as it differs from the current value: [2](#0-1) 

Without a commitment, `set_commission` can only **decrease** commission: [3](#0-2) 

The developers themselves acknowledge the gap in a code comment directly above `set_commission_commitment`: [4](#0-3) 

The only guard on `set_commission_commitment` is that `max_commission >= current_commission`, meaning a staker at 5% commission can immediately set `max_commission = 10000` and then call `set_commission(10000)` in the very next transaction. [5](#0-4) 

### Impact Explanation
Delegators who joined the pool at a low commission rate (e.g. 5%) cannot exit before the next reward distribution. The mandatory `exit_wait_window` is at least `DEFAULT_EXIT_WAIT_WINDOW = 1 week`: [6](#0-5) 

During that entire window, all pool rewards are split with a 100% commission, meaning delegators receive zero yield. The staker captures all rewards that would have accrued to delegators — this is **theft of unclaimed yield** (High impact).

### Likelihood Explanation
Any staker who has opened a delegation pool can execute this in two sequential transactions with no special privileges. The attack is cheap, deterministic, and requires no external coordination. Stakers are economically incentivised to do this when their pool holds large delegated balances.

### Recommendation
Introduce an activation delay on `set_commission_commitment` so that the commitment only becomes usable after at least `K` epochs (matching the balance-trace delay already used elsewhere in the protocol). This gives delegators time to observe the pending commitment on-chain and exit before it can be exercised. Alternatively, require that any commission increase via a commitment is announced at least one `exit_wait_window` in advance.

### Proof of Concept
```
// Precondition: staker has commission = 500 (5%), pool has delegators.

// Step 1 — staker sets a commitment allowing commission up to 100%.
//   Passes because current_commission (500) <= max_commission (10000).
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1);

// Step 2 — staker immediately raises commission to 100%.
//   Passes because active commitment exists and 10000 <= 10000 and 10000 != 500.
staking.set_commission(commission: 10000);

// Result: commission is now 100%.
// Delegators call exit_delegation_pool_intent() but must wait >= 1 week.
// All rewards distributed during the exit window go entirely to the staker.
// Delegators receive 0 yield on their locked capital — their unclaimed yield is stolen.
``` [7](#0-6) [8](#0-7)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }
