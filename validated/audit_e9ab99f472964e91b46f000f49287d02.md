### Title
Missing Inverse Condition in `assert_reshare_keys_invariants` Allows New Participant to Abort Resharing Protocol - (File: src/dkg.rs)

### Summary
The `assert_reshare_keys_invariants` function in `src/dkg.rs` validates only one direction of the participant-key invariant: it rejects an old participant that omits a signing key, but never rejects a **new** participant (absent from `old_participants`) that **supplies** a signing key. A malicious new participant can pass this incomplete guard, inject an attacker-chosen scalar into the resharing computation, and force the public-key consistency check to fail, aborting the resharing for every honest party.

### Finding Description

`assert_reshare_keys_invariants` (lines 638–669) contains the following guard:

```rust
// if me is not in the old participant set then ensure that old_signing_key is None
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
```

The developer comment describes the **missing** check ("if me is **not** in the old participant set then ensure that `old_signing_key` is `None`"), but the implemented predicate is the **opposite** direction (`old_participants.contains(me) && old_signing_key.is_none()`). The inverse case — `!old_participants.contains(me) && old_signing_key.is_some()` — is never rejected.

When a new participant `D` (not in `old_participants`) supplies a non-`None` `old_signing_key`, the guard passes and `do_reshare` is called. Inside `do_reshare` (lines 600–635):

```rust
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)          // me is NOT in intersection
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(<C::Group as Group>::Field::zero);
```

Because `D` is absent from `intersection`, `intersection.lagrange::<C>(me)` computes a Lagrange basis polynomial evaluated at 0 for a point **outside** the interpolation set. The standard formula `∏_j (-x_j)/(p - x_j)` has no zero denominator here (since `p ≠ x_j` for all `j`), so it returns a well-defined but cryptographically meaningless scalar `λ_wrong`. The attacker's contribution becomes `λ_wrong × fake_x_i`, a non-zero value that corrupts the aggregate polynomial.

After all shares are combined, `do_keyshare` reaches the public-key consistency check (lines 489–496):

```rust
if let Some(old_vk) = old_verification_key {
    if old_vk != verifying_key {
        return Err(ProtocolError::AssertionFailed(
            "new public key does not match old public key".to_string(),
        ));
    }
}
```

The reconstructed public key no longer matches `old_public_key`, so every honest party's protocol instance aborts.

### Impact Explanation

A single malicious new participant can abort the resharing for all honest parties. Because the attacker only needs to be listed in `new_participants` — a role they legitimately hold — they can repeat this on every resharing attempt, permanently preventing the key set from being updated. This matches **High: Permanent denial of reshare for honest parties under valid protocol inputs and documented trust assumptions**.

### Likelihood Explanation

The attacker requires no privileged access, no leaked secrets, and no cryptographic break. Any participant that is new to the resharing (i.e., absent from `old_participants`) can trigger the bug by passing any non-`None` value as `old_signing_key` when calling the public `reshare()` API. The entry path is direct and requires a single malformed API call.

### Recommendation

Add the missing inverse guard immediately after the existing check in `assert_reshare_keys_invariants`:

```rust
// Existing check: old participant must supply a key
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
// Missing check: new participant must NOT supply a key
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not in the old participant list but provided a share"
    )));
}
```

This mirrors the comment's stated intent and closes the bypass path before the protocol executes.

### Proof of Concept

1. Run a DKG with old participants `{A, B, C}` and threshold 2.
2. Initiate resharing to new participants `{A, B, C, D}` with the same threshold.
3. Participant `D` (new, not in old set) calls `reshare()` supplying any non-`None` `old_signing_key` (e.g., a random `SigningShare`).
4. `assert_reshare_keys_invariants` passes — the missing inverse check does not fire.
5. `do_reshare` computes `intersection.lagrange::<C>(D)` where `D ∉ intersection`; this returns a non-zero scalar without error.
6. `D`'s contribution `λ_wrong × fake_share` is added to the aggregate polynomial.
7. The reconstructed verifying key diverges from `old_public_key`.
8. Every honest party's instance hits the assertion at `src/dkg.rs:491–495` and aborts.
9. The resharing fails for all participants; `D` can repeat this on every subsequent attempt. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],
) -> Result<(ParticipantList, ParticipantList), InitializationError> {
    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

    let participants = assert_key_invariants(participants, me, threshold)?;

    let old_participants =
        ParticipantList::new(old_participants).ok_or(InitializationError::DuplicateParticipants)?;

    // Step 1.1
    if old_participants.intersection(&participants).len() < old_threshold {
        return Err(InitializationError::NotEnoughParticipantsForNewThreshold {
            threshold: old_threshold,
            participants: old_participants.intersection(&participants).len(),
        });
    }
    // Step 1.1
    // if me is not in the old participant set then ensure that old_signing_key is None
    if old_participants.contains(me) && old_signing_key.is_none() {
        return Err(InitializationError::BadParameters(format!(
            "party {me:?} is present in the old participant list but provided no share"
        )));
    }
    Ok((participants, old_participants))
}

pub fn reshare<C: Ciphersuite>(
    old_participants: &[Participant],
    old_threshold: impl Into<ReconstructionLowerBound> + Send + 'static,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    new_participants: &[Participant],
    new_threshold: impl Into<ReconstructionLowerBound> + Copy + Send + 'static,
    me: Participant,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = KeygenOutput<C>>, InitializationError>
where
    Element<C>: Send,
    Scalar<C>: Send,
{
    let comms = Comms::new();
    let threshold = new_threshold;
    let (participants, old_participants) = assert_reshare_keys_invariants::<C>(
        new_participants,
        me,
        threshold,
        old_signing_key,
        old_threshold,
        old_participants,
    )?;
    let fut = do_reshare(
        comms.shared_channel(),
        participants,
        me,
        threshold,
        old_signing_key,
        old_public_key,
        old_participants,
        rng,
    );
    Ok(make_protocol(comms, fut))
}
