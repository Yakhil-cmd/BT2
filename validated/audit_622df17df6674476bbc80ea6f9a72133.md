### Title
Operator-Configurable `native_per_pubdata` Silently Truncates to Zero, Zeroing All Pubdata Costs for Every Transaction - (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

`native_per_pubdata` is derived via integer (`wrapping_div`) division of two operator-supplied block-level parameters. When `pubdata_price < native_price`, the result silently truncates to `0`. With `native_per_pubdata == 0`, every downstream pubdata-cost check returns zero, so any unprivileged transaction sender can generate arbitrarily large pubdata at zero cost for the entire block.

---

### Finding Description

In `validate_and_compute_fee_for_transaction`, the per-pubdata-byte native cost is computed as:

```rust
// validation_impl.rs line 142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`wrapping_div` performs integer (floor) division. Whenever `pubdata_price < native_price` (e.g. `pubdata_price = 50`, `native_price = 100`), the result is `0`. No validation rejects or warns about this outcome; the only guard (`PubdataPriceTooHigh`) fires only on overflow, not on truncation to zero.

This zero propagates through every pubdata-cost site:

**1. Intrinsic pubdata overhead in `create_resources_for_tx`:**
```rust
// gas_helpers.rs line 352
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// → 0 * intrinsic_pubdata = 0  → no native deducted for intrinsic pubdata
```

**2. Post-execution pubdata charge in `get_resources_to_charge_for_pubdata`:**
```rust
// gas_helpers.rs line 430-431
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)   // 0 → always Ok(0)
    .ok_or(out_of_native_resources!())?;
```

**3. Pubdata sufficiency check in `check_enough_resources_for_pubdata` / `execute_or_deploy_inner`:**
```rust
// mod.rs line 880-885
let (has_enough, to_charge_for_pubdata, pubdata_used) = check_enough_resources_for_pubdata(
    system,
    context.native_per_pubdata,   // 0
    &resources_for_check,
    Some(context.validation_pubdata),
)?;
// resources_for_pubdata = 0 → has_enough always true
```

The same `wrapping_div` pattern is replicated in the public API helper:
```rust
// api/src/helpers.rs line 427
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
```

There is no constraint anywhere in `BlockMetadataFromOracle` deserialization or in the block-context construction path that enforces `pubdata_price >= native_price`.

---

### Impact Explanation

When `native_per_pubdata == 0`, every transaction in the block can write to an unbounded number of storage slots, emit unlimited events, and produce arbitrarily large state diffs — all at zero native cost. The protocol must still publish this data to the settlement layer (L1), bearing the full L1 data-availability cost with no corresponding fee collected from the sender. This is a direct, per-block funds-loss path for the protocol/operator: pubdata costs that should be charged to transaction senders are silently waived, and the operator subsidises all L1 DA costs for the block.

---

### Likelihood Explanation

`native_price` reflects the cost of one RISC-V proving cycle; `pubdata_price` reflects the cost of one byte of L1 data. These two quantities are denominated in different units and can legitimately differ by orders of magnitude. An operator who sets, for example, `native_price = 1000` (high proving cost) and `pubdata_price = 500` (moderate DA cost) will silently produce `native_per_pubdata = 0`. No log, no error, and no on-chain enforcement prevents this. The default test `BlockContext` already sets `pubdata_price = U256::default() = 0`, demonstrating that the zero case is a normal operating mode — making it easy to accidentally leave it zero or set it below `native_price` in production.

---

### Recommendation

1. **Enforce a minimum ratio**: after computing `native_per_pubdata`, reject (or at minimum log a hard warning for) the case where `pubdata_price > 0` but `native_per_pubdata == 0`:
   ```rust
   if pubdata_price > U256::ZERO && native_per_pubdata == 0 {
       return Err(TxError::Validation(InvalidTransaction::PubdataPriceTooLow));
   }
   ```
2. **Use ceiling division** (`div_ceil`) instead of floor division so that any non-zero `pubdata_price` yields at least `native_per_pubdata = 1`.
3. **Add a block-context invariant check** in the oracle/metadata layer that asserts `pubdata_price == 0 || pubdata_price >= native_price` before the block is accepted.

---
