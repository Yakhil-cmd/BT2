### Title
Missing Caller Access Control in `update_rewards` Allows Any Address to Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary
`StakingRewardsManagerImpl::update_rewards` has no caller access control despite the specification requiring "Only starkware sequencer." Any unprivileged address can call it with `disable_rewards: true`, consuming the global `last_reward_block` slot for the current block without distributing rewards, permanently preventing the sequencer from distributing staker rewards for that block. Repeated every block, this freezes all unclaimed yield indefinitely.

### Finding Description
The specification at `docs/spec.md:1644-1645` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation in `src/staking/staking.cairo` contains no such check:

```cairo
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only checks paused state
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... no get_caller_address() check anywhere
        self.last_reward_block.write(current_block_number);  // global slot consumed
        if disable_rewards || self.is_pre_consensus() {
            return;   // exits without distributing rewards
        }
``` [1](#0-0) 

`last_reward_block` is a single global storage variable (not per-staker). Once it is written to `current_block_number`, the check `current_block_number > self.last_reward_block.read()` will fail for any subsequent call in the same block, including the legitimate sequencer call. [2](#0-1) 

The spec's access control requirement is documented but never enforced in code: [3](#0-2) 

### Impact Explanation
An attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)` once per block. Each call:
1. Passes all precondition checks (staker exists, is active, has non-zero balance).
2. Writes `current_block_number` into the global `last_reward_block`.
3. Returns immediately without updating `unclaimed_rewards_own` or transferring pool rewards.

The sequencer's subsequent call for the same block reverts with `REWARDS_ALREADY_UPDATED`. Because the attacker can repeat this every block at negligible cost, all stakers' `unclaimed_rewards_own` never increases. This constitutes **permanent freezing of unclaimed yield** for all stakers and all delegation pools simultaneously. [4](#0-3) 

### Likelihood Explanation
The entry point is fully public — no token balance, no staking position, no privileged role is required. The attacker only needs to know one valid active staker address (trivially obtained from on-chain events). The gas cost per block is a single contract call. The attack is sustainable indefinitely and is front-runnable against every sequencer transaction.

### Recommendation
Add a caller check at the top of `update_rewards`, mirroring the pattern used in other privileged internal entry points (e.g., `update_rewards_from_attestation_contract` which asserts `get_caller_address() == self.attestation_contract.read()`):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
``` [5](#0-4) 

### Proof of Concept

```
1. Deploy the system (consensus rewards active, staker staked K+ epochs ago).
2. Attacker (any EOA) calls:
       staking.update_rewards(staker_address, disable_rewards: true)
   → last_reward_block = current_block_number
   → staker_info.unclaimed_rewards_own unchanged
3. Sequencer calls:
       staking.update_rewards(staker_address, disable_rewards: false)
   → PANICS: REWARDS_ALREADY_UPDATED
4. Advance one block. Attacker repeats step 2.
5. After N blocks: staker's unclaimed_rewards_own == 0 despite N blocks of earned rewards.
```

The test suite itself demonstrates that `update_rewards` is callable by any address with no caller restriction — every test invokes it directly without `cheat_caller_address_once`: [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
