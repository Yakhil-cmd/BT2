Audit Report

## Title
Spawn Neuron Minimum-Stake Guard Uses Stale −5% Worst-Case While Mission 70 Minimum Modulation Is −10% — (`rs/nns/governance/src/governance.rs`)

## Summary

The `spawn_neuron` guard at line 2666 of `rs/nns/governance/src/governance.rs` hardcodes a worst-case maturity modulation of −5% (`1_f64 - 0.05`), which matched the old `MIN_MATURITY_MODULATION_PERMYRIAD = -500` constant. Following the Mission 70 switchover (Proposal 141779), `maybe_spawn_neurons` now reads a locally computed modulation whose minimum is −10% (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). Any neuron holder can call `spawn_neuron` with maturity in the range `[min_stake / 0.90, min_stake / 0.95)`, pass the guard, and have `maybe_spawn_neurons` mint a neuron with a stake below `neuron_minimum_stake_e8s`, violating the NNS governance accounting invariant.

## Finding Description

**Root cause — stale constant in `spawn_neuron` guard:**

`rs/nns/governance/src/governance.rs` line 2666:
```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
```
The `0.05` was calibrated against the old CMC-polled modulation range, whose lower bound was `MIN_MATURITY_MODULATION_PERMYRIAD = -500` (−5%), defined in `rs/nervous_system/governance/src/maturity_modulation/mod.rs` line 4.

**Mission 70 extended the minimum to −10%:**

`rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs` lines 46–50:
```rust
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;  // -10%
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 = 200;     // +2%
```

**`maybe_spawn_neurons` now uses the Mission 70 modulation:**

`rs/nns/governance/src/governance.rs` lines 6427–6435 read `heap_data.maturity_modulation.current_value_permyriad` (the Mission 70 field, range [−1000, 200]), and lines 6484–6487 pass that value directly to `apply_maturity_modulation`. The CHANGELOG (`rs/nns/governance/CHANGELOG.md` lines 20–22) confirms: *"Neuron spawning and maturity disbursement finalization now read the locally computed Mission 70 maturity modulation … instead of the CMC-polled `cached_daily_maturity_modulation_basis_points`."*

**The gap:** For maturity `M` in `[min_stake / 0.90, min_stake / 0.95)`:
- Guard at spawn time: `M × 0.95 ≥ min_stake` → passes.
- Minting at `maybe_spawn_neurons` with modulation = −1000 permyriad: `apply_maturity_modulation(M, -1000)` = `M × 9000 / 10000 = M × 0.90 < min_stake` → below-minimum stake neuron created.

`apply_maturity_modulation` in `rs/nervous_system/governance/src/maturity_modulation/mod.rs` lines 11–29 accepts any `i32` basis-point value including −1000 and will return a valid (reduced) `u64` without error, so no error path prevents the minting.

## Impact Explanation

A neuron is created on-chain with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`, incrementing `neurons_with_invalid_stake_count` in NNS governance cached metrics (`rs/nns/governance/src/neuron_store/metrics.rs` line 35). Such neurons may be ineligible to vote or participate in governance proposals, silently reducing the effective voting pool. The ICP is not permanently lost (the neuron can be dissolved), but the NNS governance accounting invariant — that every spawned neuron has at least `neuron_minimum_stake_e8s` — is concretely violated. This constitutes a **High** impact: significant NNS governance security impact with concrete protocol harm, reachable by any unprivileged neuron holder.

## Likelihood Explanation

Trigger condition 1: ICP price must be persistently below its 365-day moving average, driving `compute_maturity_modulation_permyriad` toward −1000 permyriad. This is a realistic bear-market condition. Trigger condition 2: A neuron holder calls `spawn_neuron` with maturity in the narrow window `[min_stake / 0.90, min_stake / 0.95)`. With `neuron_minimum_stake_e8s = 100_000_000` e8s (1 ICP), this window is approximately 1.00–1.11 ICP of maturity — easily reachable by any active voter. The entry path is the standard unprivileged `manage_neuron` ingress call; no special role or key is required.

## Recommendation

Replace the hardcoded `0.05` in `spawn_neuron` with the actual Mission 70 minimum modulation constant:

```rust
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;
use ic_nervous_system_governance::maturity_modulation::apply_maturity_modulation;

let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);
```

This ensures the guard at spawn time is consistent with the modulation that will actually be applied at minting time.

## Proof of Concept

1. Assume `neuron_minimum_stake_e8s = 100_000_000` (1 ICP).
2. ICP price is in a sustained bear market; `compute_maturity_modulation_permyriad` has reached −1000 permyriad.
3. User calls `spawn_neuron` with `maturity_to_spawn = 106_000_000` e8s (~1.06 ICP).
4. Guard check (`governance.rs` line 2666): `106_000_000 × 0.95 = 100_700_000 ≥ 100_000_000` → **passes**.
5. Neuron enters spawning state. After 7 days, `maybe_spawn_neurons` fires.
6. `maturity_modulation` = −1000 permyriad (from `heap_data.maturity_modulation.current_value_permyriad`).
7. `apply_maturity_modulation(106_000_000, -1000)` = `106_000_000 × (10_000 − 1_000) / 10_000` = `95_400_000` e8s.
8. `neuron.cached_neuron_stake_e8s` is set to `95_400_000` (`governance.rs` line 6517).
9. `95_400_000 < 100_000_000 = neuron_minimum_stake_e8s` → neuron created with invalid stake, `neurons_with_invalid_stake_count` incremented.

A deterministic integration test can reproduce this by: (a) setting `heap_data.maturity_modulation.current_value_permyriad = Some(-1000)`, (b) creating a neuron with `maturity_e8s_equivalent = 106_000_000` and `spawn_at_timestamp_seconds` in the past, (c) calling `maybe_spawn_neurons`, and (d) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.
