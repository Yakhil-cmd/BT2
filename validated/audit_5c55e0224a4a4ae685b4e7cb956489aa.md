### Title
Unprivileged Caller Can Permanently Suppress Block Rewards for All Stakers via `update_rewards` - (File: src/staking/staking.cairo)

---

### Summary
`update_rewards` in the Staking contract is documented as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which writes the current block number into the global `last_reward_block` storage slot and returns without distributing rewards. Because `last_reward_block` is a single contract-wide variable, this permanently prevents any subsequent legitimate call from distributing rewards for that block to any staker.

---

### Finding Description

The `IStakingRewardsManager::update_rewards` function is the consensus-era entry point for distributing per-block staking rewards. The protocol specification (`docs/spec.md`, line 1645) states:

> **Access control: Only starkware sequencer.**

The implementation, however, performs no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks is_paused
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // ← global, one per block

    if disable_rewards || self.is_pre_consensus() {
        return;                            // ← exits with no rewards distributed
    }
    ...
``` [1](#0-0) 

`last_reward_block` is a single global `BlockNumber` field in contract storage, not scoped per staker: [2](#0-1) 

The spec-mandated access control is absent. `get_caller_address()` is never read inside `update_rewards`.

---

### Impact Explanation

An attacker calls `update_rewards(any_active_staker, disable_rewards: true)` at the beginning of any block. This:

1. Passes all guards (contract not paused, staker active, block number is new).
2. Writes `current_block_number` into `last_reward_block`.
3. Returns immediately without distributing any rewards.

Every subsequent call to `update_rewards` in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`: [3](#0-2) 

Because Starknet blocks are final and `last_reward_block` is never decremented, the block rewards for that block are **permanently lost** for every staker in the protocol. This maps directly to the allowed impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- **No privilege required**: any EOA or contract can call `update_rewards`.
- **No capital required**: the call costs only gas.
- **Repeatable**: the attacker can suppress rewards every single block.
- **Front-running is straightforward**: the attacker monitors the mempool for the sequencer's `update_rewards` transaction and submits their own with `disable_rewards: true` at a higher gas price, or simply calls it first in any block where the sequencer has not yet acted.

The spec explicitly names the intended caller ("Only starkware sequencer"), confirming this is an unintended open surface. [4](#0-3) 

---

### Recommendation

Add a caller check at the top of `update_rewards` that restricts execution to the designated sequencer address (stored in contract configuration), mirroring the pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer();   // add this
    ...
}
```

Alternatively, store the sequencer address in an immutable config field and assert `get_caller_address() == self.sequencer_address.read()`.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has been set and passed).
2. Staker S is active with non-zero balance.
3. Attacker A (any address) calls:
   ```
   update_rewards(staker_address: S, disable_rewards: true)
   ```
   at block B.
4. `last_reward_block` is now B; no rewards are distributed.
5. The legitimate sequencer attempts:
   ```
   update_rewards(staker_address: S, disable_rewards: false)
   ```
   in the same block B → reverts with `REWARDS_ALREADY_UPDATED`.
6. Block B's rewards are permanently unclaimable by all stakers.
7. Attacker repeats every block at negligible cost, draining the entire expected yield stream. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
