### Title
`MsgTurnBridge` Handler Is a No-Op with No Permission Check, Rendering the Bridge Circuit Breaker Non-Functional — (`File: x/cronos/keeper/msg_server.go`)

### Summary

The `MsgTurnBridge` message handler in `x/cronos/keeper/msg_server.go` is completely unimplemented: it performs no permission check and makes no state change, returning `nil, nil` unconditionally. Any unprivileged user can call it successfully, and no authorized party can ever actually disable the bridge.

### Finding Description

The Cronos module defines a permission system with two restricted operations: `CanChangeTokenMapping` and `CanTurnBridge`. The `UpdateTokenMapping` handler correctly enforces its permission:

```go
func (k msgServer) UpdateTokenMapping(...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    ...
}
```

The `TurnBridge` handler, however, is a stub:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

It neither checks `CanTurnBridge` nor updates any bridge state. The `HasPermission` / `CanTurnBridge` infrastructure exists and is wired up, but is never invoked for this message. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation

Two distinct impacts arise:

1. **Permission bypass (High):** Any unprivileged address can submit `MsgTurnBridge` and receive a success response (code 0). The `CanTurnBridge` authorization check — the only mechanism restricting who may operate the bridge circuit breaker — is completely absent from the handler. This directly satisfies: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."*

2. **Bridge circuit breaker permanently non-functional (High):** Because the handler makes no state change, even a legitimately authorized party (admin or address with `CanTurnBridge`) cannot disable the bridge. In an emergency (e.g., active exploit draining bridge funds), the intended emergency stop mechanism silently succeeds but does nothing. This satisfies: *"Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows."* [4](#0-3) [5](#0-4) 

### Likelihood Explanation

The entry path is fully unprivileged: any account can broadcast a `MsgTurnBridge` transaction. The proto service, CLI command, and message type are all wired and reachable on-chain. No special conditions are required. [6](#0-5) [7](#0-6) 

### Recommendation

Implement `TurnBridge` analogously to `UpdateTokenMapping`:

1. Check `CanTurnBridge` permission via `k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge)`.
2. Persist the bridge-enabled state (e.g., update a `BridgeActive` parameter or dedicated store key).
3. Ensure downstream bridge logic (EVM hooks, gravity end-blocker) reads and respects this state flag. [1](#0-0) 

### Proof of Concept

1. Any account (no `CanTurnBridge` permission) submits:
   ```
   cronosd tx cronos turn-bridge false --from <any_unprivileged_key>
   ```
2. Transaction is accepted and returns code 0 — no `ErrUnauthorized` is returned.
3. Bridge state is unchanged; `send_to_evm_chain` / `send_to_ethereum` continue to function normally, confirming the circuit breaker had no effect.
4. Even the admin calling `turn-bridge false` produces the same no-op result, confirming the bridge can never be disabled. [1](#0-0) [2](#0-1)

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

// Permissions returns the permissions of a specific account
func (k Keeper) Permissions(goCtx context.Context, req *types.QueryPermissionsRequest) (*types.QueryPermissionsResponse, error) {
	if req == nil {
		return nil, status.Error(codes.InvalidArgument, "empty request")
	}
	ctx := sdk.UnwrapSDKContext(goCtx)
	acc, err := sdk.AccAddressFromBech32(req.Address)
	if err != nil {
		return nil, err
	}
	admin := k.GetParams(ctx).CronosAdmin
	if admin == acc.String() {
		return &types.QueryPermissionsResponse{
			CanChangeTokenMapping: true,
			CanTurnBridge:         true,
		}, nil
	}
	permissions := k.GetPermissions(ctx, acc)
	return &types.QueryPermissionsResponse{
		CanChangeTokenMapping: CanChangeTokenMapping == (permissions & CanChangeTokenMapping),
		CanTurnBridge:         CanTurnBridge == (permissions & CanTurnBridge),
	}, nil
```

// CmdTurnBridge returns a CLI command handler for enable or disable the bridge
func CmdTurnBridge() *cobra.Command {
	cmd := &cobra.Command{
		Use:   "turn-bridge [true/false]",
		Short: "Turn Bridge",
		Args:  cobra.ExactArgs(1),
		RunE: func(cmd *cobra.Command, args []string) error {
			clientCtx, err := client.GetClientTxContext(cmd)
			if err != nil {
				return err
			}

			enable, err := strconv.ParseBool(args[0])
			if err != nil {
				return err
			}
			msg := types.NewMsgTurnBridge(clientCtx.GetFromAddress().String(), enable)
			if err := msg.ValidateBasic(); err != nil {
				return err
			}
			return tx.GenerateOrBroadcastTxCLI(clientCtx, cmd.Flags(), msg)
		},
	}

	flags.AddTxFlagsToCmd(cmd)
	return cmd
}
