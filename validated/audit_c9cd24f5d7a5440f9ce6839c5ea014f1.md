### Title
Unprivileged Front-Running of `MsgUpdateTokenMapping` via Auto-Deployment Permanently Locks Token Mapping - (File: x/cronos/keeper/keeper.go)

### Summary

An unprivileged attacker can permanently prevent the Cronos admin from registering an external contract for any IBC or gravity denom by front-running the admin's `MsgUpdateTokenMapping` transaction with a `MsgConvertVouchers` call (or a small IBC receive). When `EnableAutoDeployment` is true, this triggers `SetAutoContractForDenom` for the target denom. The 2026-04 guardrail in `SetExternalContractForDenom` — which calls `ensureDenomNotMapped` and rejects any denom already mapped — then permanently blocks the admin from overriding the auto-deployed contract with the intended external contract. There is no `DeleteAutoContractForDenom` function, making the lock irreversible.

### Finding Description

`SetExternalContractForDenom` enforces a create-only policy via `ensureDenomNotMapped`:

```go
// keeper.go:202-216
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
    if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
        return err
    }
    ...
}
```

`ensureDenomNotMapped` rejects the call if `GetContractByDenom` returns any mapping — including auto-deployed ones:

```go
// keeper.go:172-182
func (k Keeper) ensureDenomNotMapped(ctx sdk.Context, denom string) error {
    if contract, found := k.GetContractByDenom(ctx, denom); found {
        return errors.Wrapf(types.ErrDenomAlreadyMapped, ...)
    }
    return nil
}
```

`GetContractByDenom` checks both external and auto-deployed contracts:

```go
// keeper.go:114-120
func (k Keeper) GetContractByDenom(ctx sdk.Context, denom string) (contract common.Address, found bool) {
    contract, found = k.getExternalContractByDenom(ctx, denom)
    if !found {
        contract, found = k.getAutoContractByDenom(ctx, denom)
    }
    return contract, found
}
```

`MsgConvertVouchers` has no permission check and is callable by any user:

```go
// msg_server.go:27-44
func (k msgServer) ConvertVouchers(goCtx context.Context, msg *types.MsgConvertVouchers) (*types.MsgConvertVouchersResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    err := k.ConvertVouchersToEvmCoins(ctx, msg.Address, msg.Coins)
    ...
}
```

`ConvertVouchersToEvmCoins` calls `ConvertCoinFromNativeToCRC21` with `params.EnableAutoDeployment`. When `EnableAutoDeployment` is true and the denom has no existing mapping, a CRC20 contract is auto-deployed and `SetAutoContractForDenom` is called, permanently mapping the denom. The ADR explicitly confirms this path:

> "When auto-deployment is enabled and `MsgConvertVouchers` message executed with a new token" — `x/cronos/spec/03_state_transitions.md`

There is no `DeleteAutoContractForDenom` function anywhere in the keeper. Once an auto-contract is set, the admin has no mechanism to clear it and register the intended external contract.

### Impact Explanation

**High — Permanent corruption of token mappings with direct security impact.**

The attacker permanently locks a denom to an auto-deployed module CRC20 contract instead of the admin's intended external contract (e.g., a custom CRC21 with specific bridge logic, fee handling, or access controls). All subsequent bridge operations for that denom — IBC receives, gravity transfers, `MsgConvertVouchers` — will route through the wrong contract. The admin's `MsgUpdateTokenMapping` will always revert with `ErrDenomAlreadyMapped` and cannot be retried. This matches the allowed impact: "Corruption of token mappings, denom/contract binding … with direct security impact" and "Permanent or long-lived inability for honest users or validators to process valid bridge/conversion flows."

### Likelihood Explanation

- `EnableAutoDeployment` is a standard operational parameter expected to be enabled on mainnet.
- The attacker only needs a dust amount of the target IBC/gravity denom (obtainable via any IBC transfer from a counterparty chain).
- `MsgConvertVouchers` is permissionless and cheap to call.
- The attack is a single transaction that permanently forecloses the admin's ability to act.
- The attacker can monitor the mempool for pending `MsgUpdateTokenMapping` transactions and front-run them, or preemptively attack any denom they anticipate the admin will want to map.

### Recommendation

`SetExternalContractForDenom` should be permitted to override an **auto-deployed** contract (but not another external contract). Specifically, `ensureDenomNotMapped` should be split into two checks:

1. Reject if an **external** contract is already mapped (create-only for external→external).
2. Allow override if only an **auto-deployed** contract is mapped (external overrides auto).

Alternatively, expose a `DeleteAutoContractForDenom` admin function so the admin can clear a stale auto-mapping before registering the intended external contract.

### Proof of Concept

1. `EnableAutoDeployment` is `true` on the chain.
2. Admin prepares `MsgUpdateTokenMapping` for denom `ibc/XXXX` → `contractA` (a custom CRC21).
3. Attacker obtains 1 unit of `ibc/XXXX` via a normal IBC transfer.
4. Attacker submits `MsgConvertVouchers{Address: attacker, Coins: [{denom: "ibc/XXXX", amount: 1}]}` before the admin's tx lands.
5. `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21(enableAutoDeployment=true)` → auto-deploys a CRC20 → `SetAutoContractForDenom("ibc/XXXX", autoContract)`.
6. Admin's `MsgUpdateTokenMapping` executes: `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom` → `ensureDenomNotMapped` → finds `autoContract` → returns `ErrDenomAlreadyMapped` → **permanent revert**.
7. Admin retries indefinitely; every attempt fails. The denom is permanently bound to the auto-deployed contract. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

```go
func (k Keeper) ensureContractNotMapped(ctx sdk.Context, denom string, address common.Address) error {
	store := ctx.KVStore(k.storeKey)
	bz := store.Get(types.ContractToDenomKey(address.Bytes()))
	if len(bz) == 0 {
		return nil
	}
	existingDenom := string(bz)
	if existingDenom == denom {
		return nil
	}
	if k.contractOwnedByDenom(ctx, existingDenom, address) {
		return errors.Wrapf(types.ErrContractAlreadyRegistered, "contract %s is already registered for denom %s", address.Hex(), existingDenom)
	}
	// stale reverse entry
	store.Delete(types.ContractToDenomKey(address.Bytes()))
	return nil
}
```

```go
// SetExternalContractForDenom sets denom→external CRC21 mapping for an unmapped denom.
// Caller is responsible for source-denom specific validation before calling this method.
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}
```

```go
// SetAutoContractForDenom set the auto deployed contract for native denom
func (k Keeper) SetAutoContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	isSource := types.IsSourceCoin(denom)
	if err := validateContractAddressForSourceDenom(denom, address, isSource); err != nil {
		return err
	}
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}
	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToAutoContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}
```
