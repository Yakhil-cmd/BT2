### Title
Gateway L2 Gas Price Threshold Reads Current Block Price Instead of Next Block Price — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` compares a transaction's `l2_gas.max_price_per_unit` against `block_info.gas_prices.strk_gas_prices.l2_gas_price` — the **current** (already-committed) block's L2 gas price. The correct reference is `next_l2_gas_price` stored in the same block header, which is the EIP-1559-derived price that will govern the **next** block the transaction will actually be included in. A developer TODO in the code explicitly acknowledges this discrepancy.

### Finding Description

`validate_resource_bounds` fetches the latest committed block via `GatewayFixedBlockStateReader::get_block_info()` and extracts the threshold price:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  lines 229-236
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;          // ← current block's price, NOT next block's price
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `BlockInfo` from `block_header.l2_gas_price.price_in_fri` (the current block's price) and silently drops `block_header.next_l2_gas_price`, which is a distinct field in `BlockHeaderWithoutHash`:

```rust
// crates/apollo_gateway/src/gateway_fixed_block_state_reader.rs  lines 46-50
strk_gas_prices: GasPriceVector {
    ...
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
    // next_l2_gas_price is never read here
},
``` [2](#0-1) 

`next_l2_gas_price` is a real, populated field in `BlockHeaderWithoutHash` (confirmed by the sync-context test that sets it): [3](#0-2) 

The threshold check then compares the transaction's `max_price_per_unit` against the **wrong** price:

```rust
// lines 366-383
let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0).to_integer();
if tx_l2_gas_price.0 < threshold { return Err(...GAS_PRICE_TOO_LOW...) }
``` [4](#0-3) 

The blockifier's pre-validation (`check_fee_bounds`) uses the **actual** block's gas prices at execution time: [5](#0-4) 

### Impact Explanation

Two distinct wrong-source outcomes arise:

**Case A — Admission of invalid transactions (next block price rises):**  
EIP-1559 can increase the next block's L2 gas price relative to the current block's price. A transaction with `max_price_per_unit` satisfying `current_price × threshold% ≤ tx_price < next_price × threshold%` passes the gateway check (wrong source) but fails the blockifier's `MaxGasPriceTooLow` pre-validation at execution time. The mempool accepts transactions that will deterministically fail, polluting the mempool and misleading users.

**Case B — Rejection of valid transactions (next block price falls):**  
When the next block's price is lower than the current block's price, a transaction with `max_price_per_unit` satisfying `next_price × threshold% ≤ tx_price < current_price × threshold%` is rejected by the gateway even though it would pass blockifier pre-validation. Valid transactions are incorrectly excluded from sequencing.

Both outcomes match the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

- Triggerable by any unprivileged user submitting a V3 transaction with `AllResources` bounds.
- The EIP-1559 mechanism (`calculate_next_base_gas_price`) adjusts the price every block based on gas usage vs. target. During periods of sustained high or low usage the gap between `l2_gas_price` and `next_l2_gas_price` grows, widening the exploitable window.
- The TODO comment confirms the team is aware the wrong field is being read, meaning no guard currently closes this gap.

### Recommendation

1. Expose `next_l2_gas_price` through `GatewayFixedBlockStateReader` (either as a new method or by extending the returned struct).
2. In `validate_resource_bounds`, replace `block_info.gas_prices.strk_gas_prices.l2_gas_price` with the `next_l2_gas_price` read from the block header.
3. Update `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` to propagate `block_header.next_l2_gas_price`.

### Proof of Concept

1. Read the latest committed block header; note `l2_gas_price.price_in_fri = P` and `next_l2_gas_price = P'` where `P' > P` (high-usage block).
2. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = P` (equals current price, passes gateway threshold at 100%).
3. Gateway calls `validate_tx_l2_gas_price_within_threshold(P, P)` → passes.
4. Transaction enters the mempool.
5. Batcher picks the transaction; blockifier builds block context with `strk_gas_prices.l2_gas_price = P'`.
6. `check_fee_bounds` fires: `max_price_per_unit (P) < actual_gas_price (P')` → `MaxGasPriceTooLow` error.
7. Transaction reverts at pre-validation; the gateway admitted an invalid transaction.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
