### Title
Missing Active-Token Check in `set_open_for_delegation` Allows Pool Creation for Disabled Tokens - (File: src/staking/staking.cairo)

### Summary
`set_open_for_delegation` validates only that a token *exists* in the registry, but never checks whether it is currently *active*. A staker can therefore open a delegation pool for a disabled (or never-yet-enabled) BTC token. Delegators who enter that pool have their funds locked for the full exit-wait window while earning zero rewards, because the reward-distribution loop silently skips inactive tokens.

### Finding Description
`set_open_for_delegation` in `src/staking/staking.cairo` performs the following guard:

```cairo
assert!(self.does_token_exist(:token_address), "{}", Error::TOKEN_NOT_EXISTS);
``` [1](#0-0) 

`does_token_exist` returns `true` for any address that is present in the `btc_tokens` iterable map, regardless of its active/inactive state:

```cairo
fn does_token_exist(self: @ContractState, token_address: ContractAddress) -> bool {
    token_address == STRK_TOKEN_ADDRESS || self.btc_tokens.read(token_address).is_some()
}
``` [2](#0-1) 

A token is added in the *disabled* state by default (`(STARTING_EPOCH, false)`) and can be re-disabled at any time via `disable_token`: [3](#0-2) [4](#0-3) 

The reward-distribution path in `calculate_staker_pools_rewards` explicitly skips pools whose token is inactive:

```cairo
if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
    continue;
}
``` [5](#0-4) 

`enter_delegation_pool` in the pool contract performs no active-token check either; it simply transfers the delegator's tokens and records the balance: [6](#0-5) 

### Impact Explanation
A staker opens a delegation pool for a disabled BTC token. Delegators who join the pool transfer real tokens to the staking contract. Because the token is inactive, every reward-distribution call silently skips the pool. Delegators earn nothing and must wait the full `exit_wait_window` (default 1 week, up to 12 weeks) before they can recover their principal. This constitutes a **temporary freeze of delegator funds** with zero yield accrual during the lock period — matching the "Temporary freezing of funds" impact tier.

### Likelihood Explanation
The attack requires only an unprivileged staker calling `set_open_for_delegation` with a token address that exists in the registry but is currently disabled. No privileged key, bridge compromise, or external dependency is needed. The scenario is realistic whenever:
- A token has been added but not yet enabled (the default state after `add_token`), or
- A token has been disabled by the security agent and a staker (malicious or confused) opens a new pool for it.

Delegators have no on-chain guard preventing them from entering such a pool.

### Recommendation
Add an active-token check inside `set_open_for_delegation`, mirroring the pattern already used in `get_total_stake_for_token`:

```diff
 assert!(self.does_token_exist(:token_address), "{}", Error::TOKEN_NOT_EXISTS);
+assert!(
+    self.is_active_token(:token_address, epoch_id: self.get_current_epoch()),
+    "{}",
+    Error::TOKEN_NOT_ACTIVE,
+);
``` [7](#0-6) 

The same guard should be considered for `enter_delegation_pool` in the pool contract, or the pool contract should query the staking contract for the token's active status before accepting new delegations.

### Proof of Concept
1. Admin calls `add_token(btc_address)` — token is stored as `(STARTING_EPOCH, false)` (disabled).
2. Staker calls `set_open_for_delegation(btc_address)` — passes `does_token_exist` check, pool is deployed.
3. Delegator calls `pool.enter_delegation_pool(reward_addr, 1_000_000)` — BTC tokens are transferred to the staking contract, balance recorded.
4. Attestation / `update_rewards` fires — `calculate_staker_pools_rewards` hits `!is_active_token(btc_address)` and `continue`s; pool receives zero rewards.
5. Delegator calls `exit_delegation_pool_intent(1_000_000)` — must wait `exit_wait_window` (≥ 1 week).
6. During the entire lock period the delegator earns no yield and cannot access their funds.

        fn set_open_for_delegation(
            ref self: ContractState, token_address: ContractAddress,

            self.general_prerequisites();
            let staker_address = get_caller_address();

        fn disable_token(ref self: ContractState, token_address: ContractAddress) {
            self.roles.only_security_agent();
            let is_active_opt: Option<(Epoch, bool)> = self.btc_tokens.read(token_address);
            assert!(is_active_opt.is_some(), "{}", Error::TOKEN_NOT_EXISTS);
            let (is_active_first_epoch, is_active) = is_active_opt.unwrap();
            let curr_epoch = self.get_current_epoch();
            assert!(curr_epoch >= is_active_first_epoch, "{}", Error::INVALID_EPOCH);
            assert!(is_active, "{}", Error::TOKEN_ALREADY_DISABLED);
            let next_is_active_first_epoch = self.get_epoch_plus_k();
            self.btc_tokens.write(token_address, (next_is_active_first_epoch, false));
            self.emit(TokenManagerEvents::TokenDisabled { token_address });
        }

            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.

                .pool_member_info

            self
                .emit(

                );
