### Title
Silo Whitelist Bypass via `call` and `deploy_code` Entry Points — (`engine/src/contract_methods/evm_transactions.rs`)

---

### Summary

The Aurora Engine's silo whitelist system is enforced only inside the `engine::submit` execution path via `assert_access`. The `call` and `deploy_code` contract methods — which are publicly callable by any NEAR account — skip this check entirely, allowing non-whitelisted accounts to execute EVM transactions and deploy EVM contracts even when all four whitelist tiers are active.

---

### Finding Description

The silo whitelist system defines four tiers of access control:

- `WhitelistKind::Account` — NEAR accounts allowed to submit transactions
- `WhitelistKind::Address` — EVM addresses allowed to submit transactions
- `WhitelistKind::Admin` — NEAR accounts allowed to deploy EVM code
- `WhitelistKind::EvmAdmin` — EVM addresses allowed to deploy EVM code

These are enforced through `is_allow_submit` and `is_allow_deploy` in `silo/mod.rs`, which are called exclusively from `assert_access` in `engine.rs`: [1](#0-0) 

`assert_access` is invoked only within the `engine::submit` internal function, which is reached via the `submit` and `submit_with_args` contract methods.

However, the `call` and `deploy_code` contract methods in `evm_transactions.rs` only call `require_running` and then proceed directly to EVM execution — **no whitelist check is performed**: [2](#0-1) [3](#0-2) 

Both are exposed as public NEAR contract entry points with no caller restriction: [4](#0-3) 

The whitelist functions themselves are correct in isolation: [5](#0-4) 

But they are never reached from the `call` or `deploy_code` paths.

---

### Impact Explanation

When silo mode is active with whitelists enabled, the intended invariant is that only explicitly approved NEAR accounts and EVM addresses may interact with the EVM. This invariant is broken for `call` and `deploy_code`. An attacker can:

1. **Execute arbitrary EVM calls** against any deployed contract (e.g., ERC-20 `transfer`, `approve`, or custom logic) by calling `call` with crafted `CallArgs`, bypassing the `Account` and `Address` whitelists.
2. **Deploy arbitrary EVM bytecode** by calling `deploy_code`, bypassing the `Admin` and `EvmAdmin` whitelists.

This enables direct theft of user funds held in EVM contracts, or deployment of malicious contracts that drain balances — matching the **Critical: Direct theft of any user funds** impact tier.

---

### Likelihood Explanation

- `call` and `deploy_code` are unconditionally exposed as WASM entry points callable by any NEAR account.
- No deposit, key, or role requirement gates them.
- Any NEAR account that discovers the bypass can exploit it immediately without any privileged access.
- The existing silo tests (`test_submit_with_disabled_whitelist`, `test_deploy_with_disabled_whitelist`) only exercise the `submit` path, leaving the `call`/`deploy_code` bypass untested and undetected. [6](#0-5) [7](#0-6) 

---

### Recommendation

Add a whitelist access check inside `call` and `deploy_code` analogous to `assert_access`. Specifically, after `require_running`, derive the caller's EVM address from `predecessor_account_id` and invoke `silo::is_allow_submit` (for `call`) and `silo::is_allow_deploy` (for `deploy_code`), returning `ERR_NOT_ALLOWED` if the check fails.

---

### Proof of Concept

1. Deploy the Aurora Engine and enable silo mode with all four whitelists active (no entries added).
2. From a non-whitelisted NEAR account, call `submit` with a valid signed EVM transaction → **rejected** with `ERR_NOT_ALLOWED` (whitelist enforced).
3. From the same non-whitelisted NEAR account, call `call` with equivalent `CallArgs` targeting the same EVM contract → **succeeds** (whitelist bypassed).
4. Repeat step 3 with `deploy_code` to deploy arbitrary EVM bytecode — also succeeds despite the `Admin`/`EvmAdmin` whitelists being active. [8](#0-7) [9](#0-8)

fn assert_access<I: IO + Copy, E: Env>(
    io: &I,
    env: &E,
    transaction: &NormalizedEthTransaction,
) -> Result<(), EngineError> {
    let allowed = if transaction.to.is_some() {
        silo::is_allow_submit(io, &env.predecessor_account_id(), &transaction.address)
    } else {
        silo::is_allow_deploy(io, &env.predecessor_account_id(), &transaction.address)
    };

    if !allowed {
        return Err(EngineError {
            kind: EngineErrorKind::NotAllowed,
            gas_used: 0,
        });
    }

    Ok(())
}

pub fn call<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let bytes = io.read_input().to_vec();
        let args = CallArgs::deserialize(&bytes).ok_or(errors::ERR_BORSH_DESERIALIZE)?;
        let current_account_id = env.current_account_id();
        let predecessor_account_id = env.predecessor_account_id();

        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&predecessor_account_id),
            current_account_id,
            io,
            env,
        );
        let result = engine.call_with_args(args, handler)?;
        let result_bytes = borsh::to_vec(&result).map_err(|_| errors::ERR_SERIALIZE)?;
        io.return_output(&result_bytes);
        Ok(result)
    })
}

        .submit_transaction(&signer.secret_key, transaction.clone())
        .unwrap_err();
    assert_eq!(err.kind, EngineErrorKind::NotAllowed);

    // validate post-state
    validate_address_balance_and_nonce(&runner, sender, INITIAL_BALANCE, INITIAL_NONCE.into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, ZERO_BALANCE, INITIAL_NONCE.into())
        .unwrap();

    // Disable whitelists.
    disable_whitelist(&mut runner, WhitelistKind::Account);
    disable_whitelist(&mut runner, WhitelistKind::Address);

    // perform transfer
    let result = runner
        .submit_transaction(&signer.secret_key, transaction.clone())
        .unwrap();
    assert!(matches!(result.status, TransactionStatus::Succeed(_)));

    // validate post-state

#[test]
fn test_deploy_with_disabled_whitelist() {
    let (mut runner, signer, _) = initialize_transfer();
    let sender = utils::address_from_secret_key(&signer.secret_key);
    let code: Vec<u8> = {
        let mut rng = rand::rng();
        let len = rand::random_range(512..=1024);
        let mut buf = vec![0u8; len];
        rng.fill_bytes(&mut buf);
        buf
    };
    let deploy_tx = utils::create_deploy_transaction_with_price(
        code.clone(),
        INITIAL_NONCE.into(),
        ONE_GAS_PRICE.raw(),
    );
    // Check that caller's balance is enough.
    let balance = runner.get_balance(sender);
    assert_eq!(balance, INITIAL_BALANCE);

    set_silo_params(&mut runner, Some(SILO_PARAMS_ARGS));
    enable_all_whitelists(&mut runner);

    // Try to deploy code without adding to admins white list.
    let err = runner
        .submit_transaction(&signer.secret_key, deploy_tx.clone())
        .unwrap_err();
    assert_eq!(err.kind, EngineErrorKind::NotAllowed);

    // Check that the balance and the nonce haven't been changed.
    validate_address_balance_and_nonce(&runner, sender, INITIAL_BALANCE, INITIAL_NONCE.into())
        .unwrap();

    // Disable whitelists.
    disable_whitelist(&mut runner, WhitelistKind::Admin);
    disable_whitelist(&mut runner, WhitelistKind::EvmAdmin);

    // Deploy that code
    let result = runner
        .submit_transaction(&signer.secret_key, deploy_tx)
        .unwrap();
    let address = Address::try_from_slice(utils::unwrap_success_slice(&result)).unwrap();

    // Confirm the code stored at that address is equal to the input code.
    let stored_code = runner.get_code(address);
    assert_eq!(code, stored_code);

    // Check that the balance and the nonce haven't been changed.
    validate_address_balance_and_nonce(
        &runner,
        sender,
        INITIAL_BALANCE - FIXED_GAS * ONE_GAS_PRICE,
        (INITIAL_NONCE + 1).into(),
    )
    .unwrap();
}
