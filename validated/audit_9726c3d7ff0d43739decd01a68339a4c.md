### Title
`setPoolAdminFeeDestination` Does Not Flush Accrued Fees Before Updating Destination, Causing Admin Fee Misdirection - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary
`MetricOmmPoolFactory.setPoolAdminFeeDestination` updates `poolAdminFeeDestination[pool]` without first collecting accrued fees at the old address. All spread-surplus and notional-accumulator fees earned under the previous destination are subsequently swept to the new destination on the next `collectPoolFees` call, permanently depriving the original fee recipient of its earned share.

### Finding Description
The factory enforces a consistent "collect-before-mutate" pattern in `setPoolAdminFees`: it calls `collectFees` with the current `poolAdminFeeDestination[pool]` before writing new fee rates, ensuring accrued fees are settled to the correct address before the configuration changes.

`setPoolAdminFeeDestination` breaks this invariant. It overwrites `poolAdminFeeDestination[pool]` immediately, with no prior flush:

```solidity
// MetricOmmPoolFactory.sol lines 438-447
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;   // ← no collectFees() first
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

Contrast with `setPoolAdminFees` (lines 417–425), which calls `collectFees` before any state mutation
