### Title
Unrestricted `disable_rewards` Parameter in `update_rewards` Allows Permanent Freezing of All Staker Consensus Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is callable by any unprivileged address and accepts a caller-controlled `disable_rewards: bool` parameter. When called with `disable_rewards: true`, the function advances the **global** `last_reward_block` counter without distributing any rewards. Because `last_reward_block` is a single global slot (not per-staker), one call per block is sufficient for an attacker to permanently freeze consensus-mode rewards for every staker in the protocol.

---

### Finding Description

`update_rewards` is exposed as a public ABI function under `StakingRewardsManagerImpl` with no role-based access control beyond `general_prerequisites()` (which only asserts the contract is unpaused and the caller is non-zero):

```cairo
// src/staking/staking.cairo  ~L1449
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global write

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits WITHOUT distributing rewards
    }
    // ... reward distribution follows
``` [1](#0-0) 

The storage field `last_reward_block` is a single global `BlockNumber`, not a per-staker map:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

**Attack path:**

1. Attacker picks any currently-active staker address `S`.
2. At block `N`, attacker calls `update_rewards(S, disable_rewards: true)`.
3. The function passes all staker-validity checks, writes `last_reward_block = N`, then returns early — **zero rewards distributed**.
4. Any legitimate caller (validator, keeper, staker) who subsequently calls `update_rewards(S', false)` in the same block `N` hits `REWARDS_ALREADY_UPDATED` and reverts.
5. Attacker repeats once per block, indefinitely.

Because `last_reward_block` is global, a single call per block is sufficient to deny rewards to **all** stakers simultaneously.

---

### Impact Explanation

In consensus-rewards mode (after `consensus_rewards_first_epoch` is set), `update_rewards` is the sole mechanism by which per-block STRK and BTC rewards are calculated and credited to stakers and their delegation pools. Permanently suppressing it means:

- `staker_info.unclaimed_rewards_own` is never incremented for any staker.
- Pool `cumulative_rewards_trace` is never updated, so delegators also receive zero rewards.
- The `reward_supplier` is never notified, so no L1 mint requests are triggered.

This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators — a **High** impact per the allowed scope.

---

### Likelihood Explanation

- No special privilege, stake, or capital is required.
- The call costs only a single transaction per block (~2-second blocks on Starknet).
- The attacker needs only a valid (non-zero) staker address, which is publicly observable on-chain.
- There is no economic disincentive; the attacker spends only gas.

Likelihood is **High**.

---

### Recommendation

Restrict `update_rewards` to an authorized caller (e.g., the consensus/sequencer role, or a dedicated `REWARDS_MANAGER` role). The simplest fix is to add a role check at the top of the function:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_rewards_manager();   // add role check
    self.general_prerequisites();
    ...
```

Alternatively, remove the `disable_rewards` parameter entirely and handle the "no-reward block" case through a separate privileged function that does not advance `last_reward_block`.

---

### Proof of Concept

```
Block N:
  attacker → staking.update_rewards(staker=<any_active_staker>, disable_rewards=true)
    → passes general_prerequisites()
    → passes REWARDS_ALREADY_UPDATED check (N > last_reward_block)
    → writes last_reward_block = N
    → hits `if disable_rewards { return; }` → exits, no rewards distributed

  validator → staking.update_rewards(staker=<any_staker>, disable_rewards=false)
    → PANICS: REWARDS_ALREADY_UPDATED (N == last_reward_block)

Block N+1:
  attacker repeats → last_reward_block = N+1
  ... (every block, indefinitely)

Result: all stakers accumulate zero unclaimed_rewards_own forever.
``` [3](#0-2)
