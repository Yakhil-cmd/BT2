### Title
Block-List Authorization Check Enforced Only at CheckTx, Bypassed at DeliverTx — (File: `app/block_address.go`)

### Summary
`BlockAddressesDecorator.AnteHandle` wraps every block-list enforcement check inside `if ctx.IsCheckTx()`. Because Cosmos SDK ante handlers execute at both CheckTx (mempool admission) and DeliverTx/FinalizeBlock (block execution), the block-list is silently skipped during actual block processing. Any validator can include a transaction from a blocked signer or to a blocked destination address, and it will execute without restriction.

### Finding Description
In `app/block_address.go`, the entire body of `AnteHandle` — including the signer block-list check, the EVM destination address check, and the EIP-7702 authorization-list check — is gated on a single `if ctx.IsCheckTx()` condition: [1](#0-0) 

All three enforcement paths (signer check at lines 34–44, EVM destination/EIP-7702 check at lines 46–71, and `MsgStoreBlockList` admin check at lines 74–81) are inside this single `if ctx.IsCheckTx()` block. When the ante handler is invoked during `FinalizeBlock` / `DeliverTx`, `ctx.IsCheckTx()` returns `false`, so the entire function falls through to `return next(ctx, tx, simulate)` with no checks performed.

The block list itself is stored and managed by `MsgStoreBlockList` / `StoreBlockList`: [2](#0-1) 

The `StoreBlockList` msg-server handler does enforce the admin check independently, so the block list can be correctly written. The problem is exclusively in the read/enforcement path inside the ante handler.

### Impact Explanation
A blocked address (or a transaction targeting a blocked destination) is only rejected at mempool admission (CheckTx). During block execution the check is absent. A validator — including one operated by the attacker — can include such a transaction directly in a proposed block. The transaction will pass the ante handler (because `ctx.IsCheckTx()` is `false`) and execute fully, including any EVM calls, ERC-20/CRC-20 transfers, or interactions with blocked contracts. This is a direct bypass of the Cronos admin block-list authorization mechanism.

**Impact category:** High — Bypass of Cronos admin block-list authorization check.

### Likelihood Explanation
The attacker must control or collude with a single validator to include the transaction in a block. On a permissionless PoS network this is achievable by anyone who can stake enough to become a validator, or by bribing an existing validator. No cryptographic break or key compromise is required. The vulnerability is reachable by an unprivileged party.

### Recommendation
Remove the `if ctx.IsCheckTx()` guard so that block-list enforcement runs unconditionally on every ante-handler invocation, including during block execution:

```go
func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
    // Remove the `if ctx.IsCheckTx()` wrapper; enforce on every execution path.
    if sigTx, ok := tx.(signing.SigVerifiableTx); ok { ... }
    for _, msg := range tx.GetMsgs() { ... }
    return next(ctx, tx, simulate)
}
```

The admin-authority check for `MsgStoreBlockList` is already duplicated in the msg-server handler (`msg_server.go` lines 118–125), so removing it from the ante handler does not weaken that path.

### Proof of Concept

1. Admin calls `MsgStoreBlockList` to add attacker address `A` to the block list.
2. Attacker submits `MsgEthereumTx` from address `A` to any node's mempool → rejected at CheckTx (correct).
3. Attacker runs (or bribes) a validator node. The validator constructs a block that directly includes the raw transaction bytes for address `A` without going through the mempool.
4. During `FinalizeBlock`, the Cosmos SDK ante handler chain is invoked. `BlockAddressesDecorator.AnteHandle` is called with `ctx.IsCheckTx() == false`. The entire enforcement body is skipped; `next(ctx, tx, simulate)` is called.
5. The EVM transaction from blocked address `A` executes successfully, transferring tokens, interacting with contracts, or performing any other EVM action the block list was intended to prevent. [3](#0-2) [4](#0-3)

```go
func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)
```

```go
func (k msgServer) StoreBlockList(goCtx context.Context, msg *types.MsgStoreBlockList) (*types.MsgStoreBlockListResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	ctx.KVStore(k.storeKey).Set(types.KeyPrefixBlockList, msg.Blob)
	return &types.MsgStoreBlockListResponse{}, nil
```
