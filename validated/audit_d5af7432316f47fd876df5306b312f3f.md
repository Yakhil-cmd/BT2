### Title
`Debug` Derive on `KeygenOutput` Exposes Private Signing Share in Plaintext - (`File: src/lib.rs`)

### Summary
`KeygenOutput<C>`, the primary output of DKG, reshare, and refresh protocols, derives `Debug`. This struct contains `private_share: SigningShare<C>` — the participant's secret signing share. Any code path that formats this struct with `{:?}` (logging, panic output, error formatting) will emit the raw secret scalar in plaintext. The same pattern is repeated across `PresignOutput` and `RerandomizedPresignOutput` in both ECDSA variants, exposing presign nonce secrets (`k`, `sigma`, `c`, `e`, `alpha`, `beta`) through the same mechanism.

### Finding Description

`KeygenOutput<C>` is declared in `src/lib.rs` as:

```rust
#[derive(Debug, Clone, Deserialize, Serialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct KeygenOutput<C: Ciphersuite> {
    pub private_share: SigningShare<C>,
    #[zeroize[skip]]
    pub public_key: VerifyingKey<C>,
}
``` [1](#0-0) 

The `#[derive(Debug)]` macro generates a `Debug` implementation that recursively formats all fields, including `private_share`. Since `SigningShare<C>` wraps a scalar and its own `Debug` implementation (from `frost_core`) exposes the underlying scalar bytes, any `{:?}` formatting of `KeygenOutput` prints the secret signing share in full.

The same `Debug` derive is present on:

- `PresignArguments` in `src/ecdsa/ot_based_ecdsa/mod.rs`, which embeds `keygen_out: KeygenOutput` alongside secret `TripleShare` material: [2](#0-1) 

- `PresignOutput` (OT-based) containing secret nonce shares `k` and `sigma`: [3](#0-2) 

- `RerandomizedPresignOutput` (OT-based) containing rerandomized `k` and `sigma`: [4](#0-3) 

- `PresignOutput` (robust ECDSA) containing secret shares `c`, `e`, `alpha`, `beta`: [5](#0-4) 

- `RerandomizedPresignOutput` (robust ECDSA) containing `e`, `alpha`, `beta`: [6](#0-5) 

- `PresignOutput<C>` (FROST) containing `nonces: SigningNonces<C>` — the secret per-round nonces: [7](#0-6) 

The library already applies `ZeroizeOnDrop` to protect secrets in memory at drop time, but `Debug` bypasses this protection entirely by serializing the secret to a human-readable string before any drop occurs.

### Impact Explanation

**Critical — Extraction or disclosure of private signing shares and presign nonce material.**

If `KeygenOutput` is formatted with `{:?}` (e.g., via a logging call, a `dbg!()` macro, a panic handler, or an error type that wraps it), the raw `private_share` scalar is emitted in plaintext. An adversary who can read log output, crash dumps, or error messages obtains the full signing share. With `t` or more such shares (where `t` is the threshold), the aggregate private key is reconstructable and the attacker can forge arbitrary signatures without participation of honest parties.

For `PresignOutput`, exposure of `k` (the nonce share) combined with a known signature allows direct recovery of the signing key via the standard nonce-reuse equation `x = (s·k − h) / r`.

### Likelihood Explanation

**Medium.** `KeygenOutput` is a public library type returned directly to callers of `keygen`, `reshare`, and `refresh`. Library integrators routinely log protocol outputs during development and integration testing, and logging frameworks that capture `Debug` output are standard practice. The `Debug` bound is also required by the test infrastructure (`<C::Group as Group>::Element: std::fmt::Debug` appears in test helpers), normalizing the expectation that these types are freely printable. A single misplaced `log::debug!("{:?}", keygen_output)` or inclusion of `KeygenOutput` in an error type that derives `Debug` is sufficient to trigger the leak.

### Recommendation

1. Remove `Debug` from `KeygenOutput<C>`, `PresignOutput`, `RerandomizedPresignOutput`, and `PresignArguments`. Implement a manual `Debug` that redacts secret fields, e.g.:
   ```rust
   impl<C: Ciphersuite> fmt::Debug for KeygenOutput<C> {
       fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
           f.debug_struct("KeygenOutput")
               .field("private_share", &"[REDACTED]")
               .field("public_key", &self.public_key)
               .finish()
       }
   }
   ```
2. Apply the same manual `Debug` to all `PresignOutput` and `RerandomizedPresignOutput` types, redacting every scalar field that is a secret share or nonce.
3. Audit all error types and wrapper structs that embed these types to ensure they do not transitively expose secret material through `Debug`.

### Proof of Concept

```rust
use threshold_signatures::{keygen, KeygenOutput};
// After running DKG...
let output: KeygenOutput<_> = /* result of keygen protocol */;
// The following line prints the private_share scalar in full:
println!("{:?}", output);
// Output: KeygenOutput { private_share: SigningShare(<raw scalar bytes>), public_key: ... }
```

Any logging call, `dbg!(output)`, or error type wrapping `KeygenOutput` with a derived `Debug` produces the same disclosure. The root cause is `#[derive(Debug)]` at `src/lib.rs:48`, with the same pattern repeated across `src/ecdsa/ot_based_ecdsa/mod.rs:23,40,54` and `src/ecdsa/robust_ecdsa/mod.rs:26,42` and `src/frost/mod.rs:36`.
