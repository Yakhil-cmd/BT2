### Title
Attacker Can Frontrun `update_rewards` with `disable_rewards: true` to Permanently Freeze Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is callable by any unprivileged address. It accepts a `disable_rewards` boolean that, when `true`, still writes to the global `last_reward_block` storage variable but distributes zero rewards. Because `last_reward_block` is a single global slot enforcing a one-call-per-block invariant, an attacker can frontrun every block with `update_rewards(any_valid_staker, true)`, consuming the per-block slot without distributing any yield, and permanently starving all stakers of consensus-phase block rewards.

### Finding Description

`update_rewards` is exposed publicly via `#[abi(embed_v0)]` with only `general_prerequisites()` as a guard, which checks only that the contract is unpaused and the caller is non-zero:

```cairo
// src/staking/staking.cairo:1449-1507
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits with no rewards paid
    }
    ...
}
```

`last_reward_block` is a single global `BlockNumber` field in contract storage:

```cairo
// src/staking/staking.cairo:187
last_reward_block: BlockNumber,
```

The invariant `current_block_number > self.last_reward_block.read()` means the slot can be consumed exactly once per block, for any staker, by any caller. Once consumed, every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

An attacker who calls `update_rewards(any_active_staker, true)` at the top of block N:
1. Passes all checks (valid staker, non-zero balance, block advanced).
2. Writes `last_reward_block = N`.
3. Returns immediately — no rewards are distributed.
4. Every legitimate staker call in block N now reverts.

The attacker repeats this every block. The only requirement is a valid, active staker address (publicly readable from events or `staker_info_v1`) and enough gas to submit one transaction per block. On Starknet, per-transaction fees are fractions of a cent, making sustained griefing economically trivial.

### Impact Explanation

In the consensus-rewards phase (`is_pre_consensus() == false`), `update_rewards` is the sole mechanism by which stakers accumulate block rewards. If the per-block slot is consumed with `disable_rewards: true`, no staker receives any yield for that block — the rewards are simply never minted/claimed. Sustained across every block, this constitutes **permanent freezing of unclaimed yield** for all stakers, matching the High-severity impact category.

### Likelihood Explanation

- No privileged role, no token ownership, and no stake is required — any EOA suffices.
- The attacker only needs one valid staker address, which is trivially obtained from on-chain events (`NewStaker`).
- Gas cost on Starknet is negligible; the attacker has no financial cost beyond transaction fees.
- The attack is fully automatable with a simple bot that submits one transaction per block.

### Recommendation

Restrict who may call `update_rewards` with `disable_rewards: true`, or remove the parameter entirely and always distribute rewards. The minimal fix is to require the caller to be the staker or their registered reward address:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let staker_info = self.internal_staker_info(:staker_address);
    let caller = get_caller_address();
    assert!(
        caller == staker_address || caller == staker_info.reward_address,
        "{}",
        Error::CALLER_CANNOT_UPDATE_REWARDS,
    );
    ...
}
```

Alternatively, make `last_reward_block` a per-staker mapping so that one staker's call cannot block another's.

### Proof of Concept

1. Attacker observes any active staker address `S` from on-chain events.
2. At the start of block N (before any legitimate transaction), attacker submits:
   `staking.update_rewards(S, disable_rewards: true)`
3. `last_reward_block` is written to N; no rewards are distributed.
4. Staker `S` (or any other staker) attempts `update_rewards(S, false)` in block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Attacker repeats step 2 in every subsequent block.

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,

            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,

            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

                return;

                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
