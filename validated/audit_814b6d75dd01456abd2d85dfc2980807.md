### Title
Missing Presignature Invalidation Mechanism Enables Secret Key Extraction via Nonce Reuse - (File: src/ecdsa/ot_based_ecdsa/mod.rs, src/ecdsa/robust_ecdsa/mod.rs, src/frost/mod.rs)

### Summary
The library computes presignatures containing secret nonce material but provides no mechanism to invalidate or mark them as consumed. `PresignOutput` and `RerandomizedPresignOutput` derive `Clone` across all three signing schemes (OT-based ECDSA, Robust ECDSA, FROST/EdDSA), allowing a malicious coordinator to silently reuse a presignature across multiple signing sessions with different messages. This enables secret key extraction via standard ECDSA nonce-reuse attacks, a risk the documentation explicitly acknowledges but the code does not enforce.

### Finding Description

**Root cause — `Clone` on secret nonce structs with no use-tracking:**

`PresignOutput` (OT-based ECDSA) derives `Clone`: [1](#0-0) 

`RerandomizedPresignOutput` (OT-based ECDSA) also derives `Clone`: [2](#0-1) 

`PresignOutput` (Robust ECDSA) derives `Clone`: [3](#0-2) 

`PresignOutput<C>` (FROST/EdDSA) derives `Clone`: [4](#0-3) 

Although the `sign` function takes the presignature by value (move semantics), the `Clone` implementation trivially defeats this protection — the caller can clone before passing. There is no registry, "used" flag, or any other mechanism to detect or prevent reuse.

**The documentation explicitly acknowledges the danger but provides zero enforcement:** [5](#0-4) [6](#0-5) 

**Attack path (OT-based ECDSA):**

1. Malicious coordinator initiates presigning with honest participants; all parties produce `PresignOutput { big_r, k, sigma }`.
2. Coordinator calls `RerandomizedPresignOutput::rerandomize_presign` with a fixed `RerandomizationArguments` to obtain `rerandomized`.
3. Coordinator **clones** `rerandomized` → `rerandomized_copy`.
4. Coordinator calls `sign(..., rerandomized, h1)` — honest participants contribute shares; coordinator collects signature `(r, s1)`.
5. Coordinator calls `sign(..., rerandomized_copy, h2)` — honest participants contribute shares again; coordinator collects `(r, s2)`.
6. Both signatures share the same `r` (same nonce `k`). Standard ECDSA nonce-reuse recovery: `x = (s1·h2 − s2·h1) / (r·(s1 − s2))` yields the aggregate secret key.

Honest participants have no visibility into whether the presignature was previously used; the `sign` function performs no such check: [7](#0-6) 

For Robust ECDSA the documentation further notes a novel split-view variant requiring only `2t+2` presigning participants and two signing sessions: [8](#0-7) 

For FROST/EdDSA, `PresignOutput.nonces` holds `SigningNonces<C>` (the secret per-round nonces). FROST's security proof requires each nonce pair is used exactly once; reuse directly leaks the signing share. [9](#0-8) 

### Impact Explanation
**Critical.** A malicious coordinator — a regular participant with no privileged keys — can extract the aggregate secret key by reusing a presignature across two signing sessions. This maps directly to the allowed impact: *"Extraction, reconstruction, or disclosure of private signing shares, aggregate secret material, presign secrets, nonce material, or confidential derived secrets."* All three signing schemes are affected.

### Likelihood Explanation
**Medium.** The coordinator role is reachable by any participant without privileged assumptions (it is simply the aggregator). The exploit requires only two signing sessions and a single `.clone()` call — no cryptographic capability beyond normal protocol participation. The only friction is that the attacker must be selected as coordinator, which in many deployment models rotates or is chosen by the signing initiator.

### Recommendation
1. **Remove `Clone`** from `PresignOutput` and `RerandomizedPresignOutput` in all three schemes. Move semantics alone are insufficient while `Clone` exists, but removing `Clone` forces callers to explicitly opt into copying and makes accidental reuse a compile error.
2. **Consume on use at the type level**: ensure the `sign` entry point takes ownership and the type is `!Clone`, so the compiler enforces single-use.
3. **Add a presignature-ID / nonce-commitment registry** at the application layer (documented as a required integration step) so that any attempt to reuse a `big_r` value is detected and rejected before shares are contributed.

### Proof of Concept

```rust
// After a normal presigning run:
let presign_out: PresignOutput = run_presign(...);

// Rerandomize once for a specific context
let args = RerandomizationArguments { big_r: presign_out.big_r, tweak: t, .. };
let rerandomized = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args).unwrap();

// *** Root cause: Clone is available ***
let rerandomized_copy = rerandomized.clone();

// Session 1: sign message h1
let sig1 = run_sign(rerandomized,      h1, ...).unwrap(); // (r, s1)

// Session 2: sign message h2 — same nonce, different message
let sig2 = run_sign(rerandomized_copy, h2, ...).unwrap(); // (r, s2)

// Both signatures share the same r.
assert_eq!(sig1.big_r, sig2.big_r);

// Standard ECDSA nonce-reuse key recovery:
// x = (s1*h2 - s2*h1) * modular_inverse(r * (s1 - s2))
let secret_key = recover_key(r, s1, h1, s2, h2);
```

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
