### Title
Unauthenticated `LedgerService.Set()` gRPC Endpoint Allows Arbitrary Execution-State Corruption When Bound to TCP - (File: `cmd/ledger/main.go`, `ledger/remote/service.go`)

---

### Summary

The standalone ledger service (`cmd/ledger`) exposes a gRPC `LedgerService` that includes a `Set()` RPC which writes arbitrary key-value pairs directly into the execution-state MTrie. The gRPC server is constructed with **no authentication interceptors, no TLS, and no caller-identity check**. The service is explicitly documented and supported for TCP binding (e.g., `0.0.0.0:9000`). Any network-reachable caller can invoke `Set()` to overwrite account balances, contract storage, and any other on-chain state in the execution ledger — without holding any Flow keys or node credentials.

---

### Finding Description

In `cmd/ledger/main.go`, the gRPC server is created as:

```go
grpcServer := grpc.NewServer(
    grpc.MaxRecvMsgSize(int(*maxRequestSize)),
    grpc.MaxSendMsgSize(int(*maxResponseSize)),
)
```

No authentication interceptor, no TLS credential, and no per-call identity check is added. [1](#0-0) 

The service is then registered and started on a plain TCP listener:

```go
lis, err := net.Listen("tcp", *ledgerServiceTCP)
...
grpcServer.Serve(lis)
``` [2](#0-1) 

The README explicitly documents and demonstrates binding to `0.0.0.0:9000`:

```
./flow-ledger-service \
  -triedir /path/to/trie \
  -ledger-service-tcp 0.0.0.0:9000
``` [3](#0-2) 

The server-side `Set()` handler in `ledger/remote/service.go` performs only input-shape validation (non-nil state hash, matching key/value lengths) and then calls `s.ledger.Set(update)` directly — there is no check on who the caller is:

```go
func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
    if req.State == nil || len(req.State.Hash) != len(ledger.State{}) {
        return nil, status.Error(codes.InvalidArgument, "invalid state")
    }
    ...
    newState, trieUpdate, err := s.ledger.Set(update)
``` [4](#0-3) 

The `LedgerService` proto definition confirms `Set` is a plain unary RPC with no security annotations:

```proto
rpc Set(SetRequest) returns (SetResponse);
``` [5](#0-4) 

The `Set()` RPC is intended to be called exclusively by the execution node acting as the sole authorized client. The `ledger/remote/client.go` `Client.Set()` is the only intended caller. [6](#0-5) 

The `Ledger.Set()` interface documents that it "updates a list of keys with new values at specific state and returns a new state" — it is the write path for all execution-state mutations. [7](#0-6) 

---

### Impact Explanation

An attacker who can reach the TCP port can:

1. Call the unauthenticated `InitialState()` or `HasState()` RPCs (also unauthenticated) to obtain a valid current state hash.
2. Craft a `SetRequest` with attacker-controlled keys and values targeting any account's balance register, contract storage slot, or protocol-state register.
3. Call `Set()` — the server writes the attacker's payload into the MTrie and returns a new state root.

The execution node uses this ledger as its authoritative state backend. Corrupted state propagates into all subsequent block executions: account balances are falsified, contract storage is overwritten, and execution receipts produced from the corrupted state will diverge from honest nodes. This constitutes unauthorized mutation of on-chain assets and execution-state integrity.

---

### Likelihood Explanation

The `cmd/ledger` service is a production component explicitly designed for deployment as a remote backend for execution nodes. The README's primary TCP example uses `0.0.0.0:9000`, which binds to all interfaces. Cloud or container deployments that expose this port (e.g., via a misconfigured security group or service mesh) are directly exploitable by any network-reachable attacker — no Flow keys, no node credentials, and no privileged access are required. The attacker only needs a valid state hash, which is freely obtainable from the same unauthenticated `InitialState()` RPC on the same server.

---

### Recommendation

Add mandatory mutual-TLS authentication to the gRPC server in `cmd/ledger/main.go`, mirroring the optional mTLS pattern already present in the main admin server (`cmd/scaffold.go`). At minimum:

- Require a TLS server certificate and enforce `tls.RequireAndVerifyClientCert` so only the execution node (holding the corresponding client certificate) can connect.
- Alternatively, restrict TCP binding to loopback/Unix socket only and document that TCP mode requires network-level isolation (firewall rules, VPC peering, etc.).
- Add a gRPC unary interceptor that rejects any call that does not present a verified client identity.

---

// Set updates keys with new values at a specific state and returns the new state
func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
	if req.State == nil || len(req.State.Hash) != len(ledger.State{}) {
		return nil, status.Error(codes.InvalidArgument, "invalid state")
	}

	if len(req.Keys) == 0 {
		return nil, status.Error(codes.InvalidArgument, "keys cannot be empty")
	}

	if len(req.Keys) != len(req.Values) {
		return nil, status.Error(codes.InvalidArgument, "keys and values length mismatch")
	}

	var state ledger.State
	copy(state[:], req.State.Hash)

	keys := make([]ledger.Key, len(req.Keys))
	for i, protoKey := range req.Keys {
		key, err := protoKeyToLedgerKey(protoKey)
		if err != nil {
			return nil, err // protoKeyToLedgerKey already returns status.Error
		}
		keys[i] = key
	}

	values := make([]ledger.Value, len(req.Values))
	for i, protoValue := range req.Values {
		var value ledger.Value
		// Reconstruct the original value type using is_nil flag
		// This preserves the distinction between nil and []byte{} that protobuf loses
		if len(protoValue.Data) == 0 {
			if protoValue.IsNil {
				// Original value was nil
				value = nil
			} else {
				// Original value was []byte{} (empty slice)
				value = ledger.Value([]byte{})
			}
		} else {
			// Non-empty value, use data as-is
			value = ledger.Value(protoValue.Data)
		}
		values[i] = value
	}

	update, err := ledger.NewUpdate(state, keys, values)
	if err != nil {
		return nil, status.Error(codes.InvalidArgument, err.Error())
	}

	newState, trieUpdate, err := s.ledger.Set(update)
	if err != nil {
		return nil, status.Error(codes.Internal, err.Error())
	}

// Set updates keys with new values at a specific state and returns the new state.
func (c *Client) Set(update *ledger.Update) (ledger.State, *ledger.TrieUpdate, error) {
	// Handle empty updates locally without RPC call.
	// This matches the behavior of the local ledger implementations which return
	// the same state when there are no keys to update.
	if update.Size() == 0 {
		return update.State(),
			&ledger.TrieUpdate{
				RootHash: ledger.RootHash(update.State()),
				Paths:    []ledger.Path{},
				Payloads: []*ledger.Payload{},
			},
			nil
	}

	ctx, cancel := c.callCtx()
	defer cancel()
	state := update.State()
	req := &ledgerpb.SetRequest{
		State: &ledgerpb.State{
			Hash: state[:],
		},
		Keys:   make([]*ledgerpb.Key, len(update.Keys())),
		Values: make([]*ledgerpb.Value, len(update.Values())),
	}

	for i, key := range update.Keys() {
		req.Keys[i] = ledgerKeyToProtoKey(key)
	}

	for i, value := range update.Values() {
		// Distinguish between nil and []byte{} for protobuf encoding
		// Protobuf cannot distinguish them, so we use is_nil flag
		isNil := value == nil
		req.Values[i] = &ledgerpb.Value{
			Data:  value,
			IsNil: isNil,
		}
	}

	resp, err := c.client.Set(ctx, req)
	if err != nil {
		return ledger.DummyState, nil, fmt.Errorf("failed to set values: %w", err)
	}

	if resp == nil || resp.NewState == nil {
		return ledger.DummyState, nil, fmt.Errorf("invalid response: missing new state")
	}

	var newState ledger.State
	if len(resp.NewState.Hash) != len(newState) {
		return ledger.DummyState, nil, fmt.Errorf("invalid new state hash length")
	}
	copy(newState[:], resp.NewState.Hash)

	// Decode trie update using centralized decoding function to ensure
	// client and server use the same encoding method
	trieUpdate, err := decodeTrieUpdateFromTransport(resp.TrieUpdate)
	if err != nil {
		return ledger.DummyState, nil, fmt.Errorf("failed to decode trie update: %w", err)
	}

	return newState, trieUpdate, nil
}
