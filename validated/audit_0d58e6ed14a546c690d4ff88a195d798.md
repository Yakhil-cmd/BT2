### Title
Missing Caller Authorization on `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

`update_rewards` is documented as restricted to "Only starkware sequencer," but the implementation enforces no such check. Any non-zero address can call it first in any block with `disable_rewards: true`, which writes the global `last_reward_block` and returns early — permanently discarding that block's rewards for all stakers. No subsequent call in the same block can succeed.

---

### Finding Description

The spec for `update_rewards` states:

> **access control**: Only starkware sequencer.

The actual implementation delegates all access control to `general_prerequisites`: [1](#0-0) 

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

There is no sequencer check — only a non-zero caller check. The full `update_rewards` body is: [2](#0-1) 

The critical sequence is:

1. **Line 1454–1458**: Reverts if `current_block_number <= last_reward_block` (global, not per-staker).
2. **Line 1485**: Unconditionally writes `last_reward_block = current_block_number` before the `disable_rewards` check.
3. **Lines 1487–1489**: If `disable_rewards == true`, returns immediately without distributing any rewards.

Because `last_reward_block` is a **global** storage variable (not keyed per staker), once any caller sets it to the current block, every subsequent call in that block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The spec also confirms the error: [4](#0-3) 

---

### Impact Explanation

An attacker who front-runs the sequencer's `update_rewards` call in every block with `disable_rewards: true` causes every block's consensus rewards to be permanently discarded. The `last_reward_block` gate is a one-shot per block; once consumed with `disable_rewards: true`, the rewards for that block are gone forever. This maps directly to **High: Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- No privilege required — any EOA or contract with a non-zero address qualifies.
- The attacker only needs to submit a transaction before the sequencer's `update_rewards` call each block.
- On Starknet, transaction ordering within a block is controlled by the sequencer, which partially mitigates this — **but** the sequencer is not the only entity that can submit transactions, and the function is fully public on-chain with no access gate.
- The attack is cheap (one call per block) and can be sustained indefinitely.

---

### Recommendation

Add a sequencer-only guard to `update_rewards`, consistent with the spec. For example, assert that `get_caller_address() == get_sequencer_address()` (or an equivalent stored sequencer address), matching the documented access control.

---

### Proof of Concept

```
Block N:
  Attacker tx:    update_rewards(staker=<any_valid_active_staker>, disable_rewards=true)
                  → last_reward_block written to N
                  → returns early, zero rewards distributed

  Sequencer tx:   update_rewards(staker=<intended_staker>, disable_rewards=false)
                  → REVERTS: REWARDS_ALREADY_UPDATED (current_block_number == last_reward_block)

Result: Block N rewards permanently lost for all stakers.
Repeat every block → all consensus rewards suppressed indefinitely.
```

The test suite itself confirms that once `update_rewards` is called in a block (even with `disable_rewards: true`), any second call in the same block reverts: [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
