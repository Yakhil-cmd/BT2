### Title
Missing Caller Access Control on `update_rewards` Allows Any Address to Block or Misroute Reward Distribution - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no such check. Any unprivileged caller can invoke it with `disable_rewards: true` to consume the per-block reward slot without distributing rewards, permanently preventing stakers and delegators from earning yield.

---

### Finding Description

The protocol specification explicitly states:

> **access control:** Only starkware sequencer. [1](#0-0) 

However, the `StakingRewardsManagerImpl::update_rewards` implementation performs no caller identity check whatsoever. The only guard is a per-block deduplication check on `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
``` [2](#0-1) 

`general_prerequisites()` only checks the paused flag — it does not restrict the caller.

`last_reward_block` is a single contract-wide storage slot. Once it is written for a given block number, no further call to `update_rewards` can succeed for that block (the `REWARDS_ALREADY_UPDATED` assertion fires).

An attacker exploits this by calling `update_rewards(any_valid_staker, disable_rewards: true)` in every block:

1. The call passes all checks (contract unpaused, staker active, block not yet processed).
2. `last_reward_block` is updated to the current block.
3. The early-return path is taken — zero rewards are distributed.
4. The legitimate sequencer's subsequent call for the same block reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

A secondary attack vector: the attacker calls with `disable_rewards: false` and an arbitrary `staker_address` (not the one who attested). This misroutes block rewards to a staker who did not perform attestation, while the attesting staker receives nothing for that block. [4](#0-3) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

By calling `update_rewards(..., disable_rewards: true)` once per block, an attacker can ensure that no staker or delegator ever accumulates rewards. The `last_reward_block` slot is shared across all stakers; a single attacker call per block is sufficient to block the entire reward pipeline for all participants indefinitely. Stakers and delegators lose all yield for every block the attack is active.

---

### Likelihood Explanation

Any address can submit a transaction on Starknet. The call is cheap (no token transfer, no stake required). The attacker needs only to submit one transaction per block targeting any currently-active staker. Because the spec intended this to be sequencer-only, no on-chain enforcement exists to prevent it. The attack is repeatable, low-cost, and requires no privileged access.

---

### Recommendation

Add an explicit caller check at the top of `update_rewards`, analogous to the checks already present in `update_rewards_from_attestation_contract` and the reward supplier functions:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    self.general_prerequisites();
    ...
}
```

Alternatively, store the authorized sequencer address at construction time and enforce it here, consistent with how `update_rewards_from_attestation_contract` enforces `CALLER_IS_NOT_ATTESTATION_CONTRACT`. [5](#0-4) 

---

### Proof of Concept

1. Deploy the system with a staker active and consensus rewards enabled.
2. From any EOA (attacker), call:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(
       staker_address,   // any valid active staker
       disable_rewards: true
   )
   ```
3. Observe that `last_reward_block` is now set to the current block.
4. The legitimate sequencer attempts the same call — it reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat step 2 every block. Stakers and delegators accumulate zero rewards indefinitely.

The existing test suite confirms `update_rewards` is callable without any caller restriction: [6](#0-5) 

No `cheat_caller_address` is needed — the call succeeds from any address.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
