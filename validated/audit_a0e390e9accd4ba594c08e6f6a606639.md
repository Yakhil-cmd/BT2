### Title
`collectFees` Unconditionally Zeroes Notional Fee Accumulators Even When Floor Rounding Produces Zero Transfers — (`metric-core/contracts/MetricOmmPool.sol`)

---

### Summary

`collectFees` clears `notionalFeeToken0Scaled` and `notionalFeeToken1Scaled` to zero unconditionally at the end of every call, regardless of whether any tokens were actually transferred. On pools with high-scale-multiplier tokens (e.g., USDC/USDT with `TOKEN_0_SCALE_MULTIPLIER = 1e12`), if the accumulated scaled fee is below the scale threshold, Floor-division rounds the external transfer amounts to zero — but the accumulators are still wiped, permanently destroying the accrued fee value.

---

### Finding Description

`MetricOmmPoolFactory.collectPoolFees` is a permissionless `external` function with no access control: [1](#0-0) 

It delegates to `MetricOmmPool.collectFees`, which:

1. Reads the notional fee accumulators: [2](#0-1) 

2. Converts scaled fee amounts to external token units using **Floor rounding**: [3](#0-2) 

The Floor-rounding path is: [4](#0-3) 

For a USDC/USDT pool, `TOKEN_0_SCALE_MULTIPLIER = 10^(18-6) = 1e12` (confirmed by factory logic): [5](#0-4) 

If `totalFee0ToAdminScaled < 1e12` and `totalFee0ToProtocolScaled < 1e12`, both Floor-divided results are `0`. The conditional transfers are skipped: [6](#0-5) 

But the accumulators are **unconditionally** zeroed regardless: [7](#0-6) 

---

### Impact Explanation

Any scaled fee value below `1e12` (for 6-decimal tokens) that has been collected from real swaps and stored in `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` is permanently destroyed — no tokens are transferred to protocol or admin, and the accumulator is reset to zero. This is a direct loss of protocol and admin fee revenue.

The notional fee accumulates in scaled units during swaps: [8](#0-7) 

A swap output of up to ~100 native USDC units (0.0001 USDC) with a 1% notional fee produces `< 1e12` scaled fee units — well within normal micro-swap territory.

---

### Likelihood Explanation

`collectPoolFees` is callable by **anyone** with no restriction. An attacker (or even a well-meaning bot) can call it after each small swap, preventing the notional fee accumulator from ever growing large enough to produce a non-zero Floor-divided transfer. This is a repeatable griefing path that permanently suppresses protocol and admin fee collection on any 6-decimal token pool.

---

### Recommendation

Instead of unconditionally zeroing the accumulators, subtract only the scaled amount that was actually transferred. Compute the scaled equivalent of the transferred external amounts (using Ceil to be conservative) and deduct only that from the accumulators, leaving any sub-threshold remainder intact:

```solidity
// Instead of:
notionalFeeToken0Scaled = 0;
notionalFeeToken1Scaled = 0;

// Use:
uint256 transferred0Scaled = (totalFee0ToAdmin + totalFee0ToProtocol) * TOKEN_0_SCALE_MULTIPLIER;
uint256 transferred1Scaled = (totalFee1ToAdmin + totalFee1ToProtocol) * TOKEN_1_SCALE_MULTIPLIER;
notionalFeeToken0Scaled = uint128(notionalFee0AmountScaled > transferred0Scaled
    ? notionalFee0AmountScaled - transferred0Scaled : 0);
notionalFeeToken1Scaled = uint128(notionalFee1AmountScaled > transferred1Scaled
    ? notionalFee1AmountScaled - transferred1Scaled : 0);
```

---

### Proof of Concept

1. Deploy a USDC/USDT pool (`TOKEN_0_SCALE_MULTIPLIER = 1e12`, `notionalFeeE8 = 1e6` i.e. 1%).
2. Execute a swap with output of 50 native USDC units → notional fee = `50 * 1e12 * 1e6 / 1e8 = 5e11` scaled units (< 1e12).
3. Call `MetricOmmPoolFactory.collectPoolFees(pool)` (no auth required).
4. Observe: `totalFee0ToAdmin = 5e11 / 1e12 = 0`, `totalFee0ToProtocol = 0` → no transfer.
5. Assert: `notionalFeeToken0Scaled == 0` (wiped), USDC balance of factory and admin unchanged.
6. The 5e11 scaled units of accrued fee are permanently destroyed.
