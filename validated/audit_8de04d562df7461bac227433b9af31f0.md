### Title
Incorrect Multi-NFT Royalty Accounting in WalletConnect Offer Confirmation Understates Total Cost - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
`formatAmountWithRoyalties` in `parseCommandDisplay.ts` divides the fungible amount by the number of NFTs before computing each NFT's royalty, producing a displayed `amountWithRoyalties` that is materially lower than the actual on-chain cost. A user presented with a multi-NFT WalletConnect offer confirmation approves a transaction believing they will spend less than they actually will.

### Finding Description
`formatAmountWithRoyalties` is called during WalletConnect command display for `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids`. It is responsible for computing the "total including royalties" figure shown to the user before they approve the offer.

The function contains the following logic:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

When `royaltyPercentages` contains more than one entry (i.e., the offer involves multiple NFTs each with royalties), the code first divides `amount` by the number of NFTs, then computes each NFT's royalty on that reduced `splitAmount`. This is mathematically wrong: each NFT's royalty is a percentage of the **full** sale price, not a fraction of it.

**Concrete example** (reproduced by the existing test at line 251):
- Amount: 100,000,000 mojos (0.0001 XCH)
- NFT 1 royalty: 500 basis points (5%)
- NFT 2 royalty: 10 basis points (0.1%)

*Current (buggy) display:*
- `splitAmount` = 50,000,000
- royalty₁ = 50,000,000 × 500 / 10,000 = 2,500,000
- royalty₂ = 50,000,000 × 10 / 10,000 = 50,000
- Displayed total = **102,550,000 mojos (0.00010255 XCH)**

*Correct (royalties on full amount):*
- royalty₁ = 100,000,000 × 500 / 10,000 = 5,000,000
- royalty₂ = 100,000,000 × 10 / 10,000 = 100,000
- Correct total = **105,100,000 mojos (0.0001051 XCH)**

The test at line 320 asserts `amountWithRoyalties: '0.00010255'`, confirming the buggy value is the one actually displayed. [2](#0-1) 

The `amountWithRoyalties` field is surfaced directly in the WalletConnect confirmation dialog via `walletDeltaToDisplay` → `withRoyaltyTotals` → `formatAmountWithRoyalties`. [3](#0-2) 

### Impact Explanation
A user reviewing a WalletConnect `take_offer` or `create_offer_for_ids` confirmation for a multi-NFT offer sees an understated total cost. The actual on-chain royalty payments are computed by the Chia node on the full fungible amount for each NFT, so the user's wallet is debited more than the confirmation dialog indicated. This constitutes displaying the wrong amount in an offer approval flow, causing the user to approve a transaction under false accounting.

### Likelihood Explanation
Any WalletConnect-connected dApp or wallet that presents a multi-NFT offer (two or more NFTs with non-zero royalties in a single offer) triggers this path. The attacker only needs to craft such an offer and deliver it to the victim via WalletConnect. No special privileges are required. The discrepancy grows with the number of NFTs and the magnitude of royalty percentages.

### Recommendation
Remove the `splitAmount` division. Each NFT's royalty should be calculated on the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

Update the corresponding test expectation at line 320 from `'0.00010255'` to the correct value `'0.0001051'`. [4](#0-3) 

### Proof of Concept
1. Attacker constructs a WalletConnect `chia_wallet.take_offer` payload containing an offer where two NFTs (royalties 5% and 0.1%) are offered in exchange for 0.0001 XCH.
2. Victim's GUI calls `parseCommandDisplay('chia_wallet.take_offer', { offer: '...' })`.
3. `formatAmountWithRoyalties` computes `amountWithRoyalties = '0.00010255'` (splitting the amount by 2 before royalty calculation).
4. The WalletConnect confirmation dialog displays "Total including royalties: 0.00010255 XCH".
5. Victim approves. The Chia node applies royalties on the full 0.0001 XCH for each NFT, deducting 0.0001051 XCH — approximately 2.4% more than displayed. [1](#0-0) [5](#0-4)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}
