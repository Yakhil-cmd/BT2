### Title
Missing Proof of Possession in `set_public_key` Enables Rogue Key Attack on Consensus BLS Aggregation - (File: src/staking/staking.cairo)

---

### Summary

`set_public_key()` in the Staking contract accepts any arbitrary `felt252` value as a staker's BLS public key with no proof that the caller controls the corresponding private key. An attacker who stakes the minimum amount can register the additive inverse of a victim staker's public key. When the off-chain consensus layer aggregates public keys from `get_stakers()`, the two keys cancel to the point at infinity, allowing the attacker to forge aggregate BLS signatures that appear to include the victim's participation — without ever holding the victim's private key.

---

### Finding Description

`PublicKey` is typed as `felt252` (a Starknet field element): [1](#0-0) 

`set_public_key()` performs only two semantic checks — non-zero and differs from the previous value — with no signature or proof-of-possession verification: [2](#0-1) 

The stored key is later served verbatim to the consensus layer via `get_stakers()`: [3](#0-2) 

The protocol explicitly allows two different stakers to register the same public key (confirmed by the existing flow test `set_same_public_key_for_2_different_stakers_flow_test`), and there is no uniqueness or curve-validity check anywhere: [4](#0-3) 

The `get_stakers()` interface documents that the returned `(staker_address, staking_power, Option<PublicKey>, Option<PeerId>)` tuples are consumed by the consensus layer for BLS signature aggregation: [5](#0-4) 

---

### Impact Explanation

**High — Theft of unclaimed consensus yield.**

In the consensus phase (after `consensus_rewards_first_epoch` is activated), block rewards are distributed per-block via `update_rewards()` to whichever staker the sequencer names: [6](#0-5) 

The sequencer selects which staker produced a block based on the consensus committee, which is derived from `get_stakers()`. If the attacker's rogue key causes the aggregate BLS verification to accept forged signatures that include the victim's staking power, the sequencer will attribute block production to the attacker's address rather than the victim's. The victim's `unclaimed_rewards_own` is never incremented; the attacker's is. This constitutes direct theft of unclaimed yield.

Additionally, the victim's legitimate key is cryptographically neutralised in the aggregate (their contribution collapses to the identity element), so they cannot produce valid aggregate signatures for any epoch in which the attacker's rogue key is active — permanently freezing the victim's yield for those epochs.

---

### Likelihood Explanation

**High.**

- Any address that meets `min_stake` can call `stake()` and then `set_public_key()` with no additional prerequisites.
- The victim's public key is publicly readable on-chain via `get_current_public_key()` or from `get_stakers()` events.
- Computing the additive inverse of a `felt252` is trivial: `-pk mod p`.
- No privileged role, leaked key, or external dependency is required.
- The attack is fully self-contained within the on-chain contract and the off-chain consensus layer that consumes `get_stakers()`.

---

### Recommendation

1. **Require proof of possession**: When a staker calls `set_public_key(public_key)`, require them to also supply a BLS signature over a domain-separated message (e.g., `H("set_public_key" || staker_address || public_key)`) that verifies under the submitted `public_key`. This proves the caller holds the corresponding private key before the key is stored.

2. **Enforce key uniqueness**: Maintain a `public_key_to_staker` map and reject registration of a key already in use by another staker. This prevents the rogue-key scenario even if proof-of-possession is not yet implemented.

3. **Validate curve membership**: Reject `public_key` values that do not correspond to a valid point on the target BLS curve before storing them.

---

### Proof of Concept

```
1. Victim staker V is registered with public key pk_V (readable via get_current_public_key(V)).

2. Attacker A stakes the minimum amount:
       staking.stake(reward_address=A, operational_address=A_op, amount=min_stake)

3. Attacker calls set_public_key with the additive inverse of pk_V:
       staking.set_public_key(public_key = (-pk_V) mod STARK_PRIME)
   This passes all checks:
     - is_non_zero(): -pk_V mod p ≠ 0 (since pk_V ≠ 0)
     - prev_public_key != public_key: attacker had no prior key (default 0 ≠ -pk_V)

4. After K epochs, get_stakers(epoch_id) returns:
       [..., (V, power_V, Some(pk_V), ...), (A, power_A, Some(-pk_V), ...), ...]

5. The consensus layer computes the aggregate public key:
       pk_agg = pk_V + (-pk_V) = O  (point at infinity / identity)

6. The attacker constructs a valid aggregate signature σ = O (point at infinity),
   which satisfies e(σ, G2) = e(O, G2) = 1 = e(pk_agg, H(m)) for any message m.

7. The forged aggregate signature is accepted by the consensus layer as if both
   V and A signed the block. The sequencer calls update_rewards(staker_address=A),
   crediting A with block rewards that should have gone to V.

8. V's key is neutralised; V earns zero consensus rewards for every epoch the
   attacker's rogue key remains active.
``` [2](#0-1) [3](#0-2)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

fn set_same_public_key_for_2_different_stakers_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let amount = cfg.staking_contract_info.min_stake;
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    let staker_1 = system.new_staker(:amount);
    let staker_2 = system.new_staker(:amount);
    let public_key = PUBLIC_KEY;
    let staking_address = system.staking.address;
    let staking = system.staking.dispatcher();
    let staking_consensus = system.staking.consensus_dispatcher();

    // Staker 1 stake and set public key.
    system.stake(staker: staker_1, :amount, pool_enabled: false, commission: 200);
    cheat_caller_address_once(
        contract_address: staking_address, caller_address: staker_1.staker.address,
    );
    staking.set_public_key(:public_key);

    // Staker 2 stake and set public key.
    system.stake(staker: staker_2, :amount, pool_enabled: false, commission: 200);
    cheat_caller_address_once(
        contract_address: staking_address, caller_address: staker_2.staker.address,
    );
    staking.set_public_key(:public_key);

    // Test get_stakers.
    system.advance_k_epochs();
    let expected_stakers = array![
        (staker_1.staker.address, STRK_WEIGHT_FACTOR / 2, Option::Some(public_key), Option::None),
        (staker_2.staker.address, STRK_WEIGHT_FACTOR / 2, Option::Some(public_key), Option::None),
    ]
        .span();
    let epoch_id = staking.get_current_epoch();
    assert!(staking_consensus.get_stakers(:epoch_id) == expected_stakers);
    assert!(staking.get_current_public_key(staker_address: staker_1.staker.address) == public_key);
    assert!(staking.get_current_public_key(staker_address: staker_2.staker.address) == public_key);
}
