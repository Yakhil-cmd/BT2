### Title
Misleading `PoolMemberExitIntent` Event Emitted on No-Op / Intent-Cancellation Call ŌĆö (File: src/pool/pool.cairo)

---

### Summary

`exit_delegation_pool_intent(amount: 0)` is a valid, non-reverting call that is documented to *cancel* an existing exit intent. However, the function unconditionally emits `PoolMemberExitIntent` (and `PoolMemberBalanceChanged`) regardless of whether `amount` is zero or non-zero. When `amount = 0`, the emitted event carries `exit_timestamp: 0` and `amount: 0` ŌĆö signalling an exit intent that was never actually registered. Any off-chain system (indexer, UI, monitoring tool) that listens for `PoolMemberExitIntent` will be misled into believing a pool member has initiated an exit, when in reality nothing changed on-chain.

---

### Finding Description

In `src/pool/pool.cairo`, `exit_delegation_pool_intent` emits two events unconditionally at the end of the function body:

```cairo
// Emit events.
self.emit(Events::PoolMemberExitIntent {
    pool_member, exit_timestamp: unpool_time, amount,
});
self.emit(Events::PoolMemberBalanceChanged {
    pool_member, old_delegated_stake, new_delegated_stake,
});
``` [1](#0-0) 

When `amount = 0`:

1. `unpool_time` is the return value of `undelegate_from_staking_contract_intent(pool_member, amount: 0)`, which calls `remove_from_delegation_pool_intent` with `amount = 0`. That function clears any existing intent and returns `Timestamp { seconds: 0 }` (the zero value of the cleared intent).
2. `pool_member_info.unpool_time` is set to `Option::None` and `unpool_amount` to `0` ŌĆö no intent is stored.
3. `new_delegated_stake = total_amount - 0 = total_amount` ŌĆö the balance is unchanged. [2](#0-1) 

Despite no state change, the contract emits:
- `PoolMemberExitIntent { pool_member, exit_timestamp: 0, amount: 0 }` ŌĆö looks like an exit intent was registered.
- `PoolMemberBalanceChanged { pool_member, old_delegated_stake: X, new_delegated_stake: X }` ŌĆö looks like a balance update occurred.

Additionally, `remove_from_delegation_pool_intent` in the staking contract also emits `RemoveFromDelegationPoolIntent` and `StakeDelegatedBalanceChanged` with `old == new` for the same no-op call: [3](#0-2) 

The spec explicitly documents `amount = 0` as a cancellation path ("If `amount` is zero, remove request for intent (if exists)"), yet no separate event is emitted for cancellation, and the exit-intent event is still fired. [4](#0-3) 

---

### Impact Explanation

Off-chain systems (indexers, dashboards, alerting bots) that subscribe to `PoolMemberExitIntent` events will observe events with `amount: 0` and `exit_timestamp: 0`. These systems cannot distinguish a genuine exit intent from a no-op call. A malicious pool member can:

- Repeatedly call `exit_delegation_pool_intent(amount: 0)` to flood event logs with spurious exit-intent signals.
- Cause indexers to mark a pool member as "exiting" when they are not, corrupting UI state and off-chain accounting.
- Trigger automated reactions in third-party integrations (e.g., liquidation bots, re-delegation logic) that act on `PoolMemberExitIntent` events.

This is **griefing with no profit motive but concrete damage to users and protocol observability** ŌĆö matching the **Medium** impact tier.

---

### Likelihood Explanation

The preconditions are minimal:
- Caller must be a registered pool member (any delegator qualifies).
- No minimum amount, no cooldown, no pausing check specific to this path.
- The call succeeds even when the pool member has no existing intent and `amount = 0`.

Any pool member can execute this at will, making the attack trivially reachable.

---

### Recommendation

1. **Guard the event emission**: Only emit `PoolMemberExitIntent` when `amount > 0`.
2. **Emit a distinct cancellation event**: When `amount = 0` and an existing intent is cleared, emit a `PoolMemberExitIntentCancelled` event instead, analogous to the 1inch fix that introduced `BitInvalidatorUpdated` for the non-order-hash path.
3. **Guard `PoolMemberBalanceChanged`**: Skip emission when `old_delegated_stake == new_delegated_stake`.

---

### Proof of Concept

1. Alice is a registered pool member with `amount = 1000` and no existing exit intent.
2. Alice calls `exit_delegation_pool_intent(amount: 0)`.
3. The call succeeds (no revert ŌĆö `0 <= 1000` passes the `AMOUNT_TOO_HIGH` guard).
4. On-chain state is **unchanged**: `unpool_time = None`, `unpool_amount = 0`, balance = 1000.
5. The following events are emitted:
   - (Staking contract) `RemoveFromDelegationPoolIntent { old_intent_amount: 0, new_intent_amount: 0 }`
   - (Staking contract) `StakeDelegatedBalanceChanged { old_delegated_stake: 1000, new_delegated_stake: 1000 }`
   - (Pool contract) `PoolMemberExitIntent { pool_member: Alice, exit_timestamp: 0, amount: 0 }`
   - (Pool contract) `PoolMemberBalanceChanged { old_delegated_stake: 1000, new_delegated_stake: 1000 }`
6. Any indexer watching `PoolMemberExitIntent` records Alice as having initiated an exit with timestamp 0 ŌĆö a false signal.
7. Alice can repeat this indefinitely to spam the event log.

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }
