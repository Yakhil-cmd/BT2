### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Suppress Per-Block Rewards - (File: src/staking/staking.cairo)

### Summary

The spec mandates "Only starkware sequencer" access control for `update_rewards`, but the implementation contains **no caller check**. Because `last_reward_block` is a **global** (not per-staker) storage variable that is written unconditionally before the `disable_rewards` branch, any unprivileged caller can call `update_rewards(valid_staker, disable_rewards: true)` once per block to permanently discard that block's rewards for every staker in the protocol.

### Finding Description

`StakingRewardsManagerImpl::update_rewards` in `src/staking/staking.cairo` performs the following steps:

1. `self.general_prerequisites()` — only checks the pause flag, no caller identity check.
2. Asserts `current_block_number > self.last_reward_block.read()` (REWARDS_ALREADY_UPDATED).
3. Validates the supplied `staker_address` is active with non-zero balance.
4. **Writes `self.last_reward_block` to `current_block_number`** — unconditionally, before the `disable_rewards` branch.
5. `if disable_rewards || self.is_pre_consensus() { return; }` — exits without distributing rewards. [1](#0-0) 

The spec explicitly states the access control for this function is "Only starkware sequencer": [2](#0-1) 

No such check exists in the implementation. The `last_reward_block` gate is **global** — one successful call per block locks out all subsequent calls for that block: [3](#0-2) 

### Impact Explanation

An attacker who calls `update_rewards(any_valid_active_staker, disable_rewards: true)` in block N:

- Sets `last_reward_block = N` (line 1485).
- Returns immediately without distributing rewards (line 1487–1488).
- Every subsequent call to `update_rewards` in block N — for **any** staker — reverts with `REWARDS_ALREADY_UPDATED` (line 1454–1458).

Block N's consensus rewards are permanently lost. Repeating this every block causes continuous, unbounded suppression of all staker yield. This matches **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- Requires no privilege, no stake, no funds — only a valid `staker_address` (publicly readable from events or `get_stakers`).
- One transaction per block suffices; gas cost is negligible relative to the yield destroyed.
- The attack is fully permissionless and can be automated.

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, consistent with the spec. For example, assert `get_caller_address() == sequencer_address` (or use the appropriate role/access-control primitive already used elsewhere in the contract) before any state mutation.

### Proof of Concept

```
// Block N:
// Attacker (any EOA) calls:
staking.update_rewards(staker_address: any_valid_staker, disable_rewards: true);
// → last_reward_block is now N, no rewards distributed.

// Legitimate sequencer then tries:
staking.update_rewards(staker_address: intended_staker, disable_rewards: false);
// → REVERTS with REWARDS_ALREADY_UPDATED.
// Block N rewards are gone for all stakers.

// Repeat every block → continuous yield suppression.
```

The test suite itself demonstrates that `update_rewards` is callable by arbitrary addresses with no `cheat_caller_address` setup for a sequencer role: [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
