### Title
Variable-Length Contract Name Without Delimiter Causes Storage Key Prefix Collision, Returning Wrong Contract Data from Extended API - (File: storage/indexes/contracts.go)

### Summary
The `ContractDeploymentsIndex` in `storage/indexes/contracts.go` encodes the contract name as a raw variable-length byte sequence directly between the fixed-length address (8 bytes) and the fixed-length suffix (`~height(8) + txIndex(4) + eventIndex(4)` = 16 bytes), with no length prefix or separator. When two contracts at the same address have names where one is a byte-level prefix of the other (e.g., `"Token"` and `"TokenV2"`), the storage key for the longer-named contract sorts lexicographically before the shorter-named contract's key. As a result, `ByContract(addr, "Token")` returns a `ContractDeployment` record for `"TokenV2"` instead of `"Token"`, and `DeploymentsByContract(addr, "Token", nil)` silently includes deployments of all contracts whose names begin with `"Token"`.

### Finding Description

The primary key format is:

```
[codeContractDeployment(1)][address(8)][contract name(variable)][~height(8)][txIndex(4)][eventIndex(4)]
``` [1](#0-0) 

The contract name is embedded with no length field or separator. The `makeContractDeploymentContractPrefix` function produces the prefix `[code][addr][name_bytes]`, which is used as both the start and end key for a prefix scan in `rangeKeysByContract`: [2](#0-1) [3](#0-2) 

Because the contract name is immediately followed by the one's-complement height (whose first byte is always `0xFF` for any valid height), any contract name that is a byte-level prefix of another name will have its key sort **after** the longer-named contract's key. Concretely:

- Key for `"Token"` at height H: `...[T][o][k][e][n][0xFF ...]`
- Key for `"TokenV2"` at height H': `...[T][o][k][e][n][V(0x56)][2][0xFF ...]`

At the byte position immediately after `"Token"`, the `"TokenV2"` key has `0x56` (`'V'`) while the `"Token"` key has `0xFF` (first byte of `~height`). Since `0x56 < 0xFF`, all `"TokenV2"` keys sort **before** all `"Token"` keys within the same prefix scan range.

`DeploymentsByContract` uses `iterator.Build` (not `BuildPrefixIterator`), so it does not deduplicate by contract-name prefix — it returns every key in the scanned range: [4](#0-3) 

`ByContract` returns the first item from this iterator without validating that the decoded `ContractName` matches the requested name: [5](#0-4) 

The backend layer also performs no such validation: [6](#0-5) 

The only name validation at write time rejects empty names and names containing `"."`, but does not prevent two contracts at the same address from having names where one is a prefix of the other: [7](#0-6) 

### Impact Explanation

Any API caller issuing `GET /experimental/v1/contracts/A.{addr}.Token` or `GET /experimental/v1/contracts/A.{addr}.Token/deployments` receives data for `"TokenV2"` (or any other contract at the same address whose name starts with `"Token"`), not for `"Token"`. The returned `ContractDeployment` carries the wrong `ContractName`, wrong `CodeHash`, wrong `Code`, and wrong `TransactionID`. A consumer relying on this API to verify which contract is deployed at an address is silently given incorrect information — a direct analog to the external report's concern about confusion over which application is actually deployed.

### Likelihood Explanation

The on-chain condition (two contracts at the same address where one name is a prefix of the other, e.g., `"Token"` / `"TokenV2"`, `"Vault"` / `"VaultV2"`, `"NFT"` / `"NFTMinter"`) is a realistic and common naming pattern. No malicious actor is required; the bug is triggered by ordinary legitimate deployments. Any unprivileged caller of the Access Node's extended REST API can observe the incorrect result.

### Recommendation

Separate the variable-length contract name from the fixed-length height/index suffix with a length-prefix or a fixed-width separator byte that cannot appear in a valid contract name. For example, encode the name as `[len(name)(2 bytes)][name bytes]`, making the boundary between the name and the `~height` field unambiguous regardless of name content. Alternatively, add a post-decode assertion in `ByContract` and `DeploymentsByContract` that the decoded `ContractName` exactly equals the requested name, and return `ErrNotFound` if it does not.

### Proof of Concept

Given an account at address `addr` with two deployed contracts `"Token"` (height 100) and `"TokenV2"` (height 50):

Storage keys (schematic, after `[code][addr]`):
```
"TokenV2" h=50:  [T][o][k][e][n][V=0x56][2][~50=0xFF...CD][0][0]
"Token"   h=100: [T][o][k][e][n][~100=0xFF...9B][0][0]
```

Prefix scan for `[code][addr][T][o][k][e][n]` returns `"TokenV2"` entries first (because `0x56 < 0xFF`).

`ByContract(addr, "Token")` → returns `ContractDeployment{ContractName: "TokenV2", BlockHeight: 50, ...}` instead of `ContractDeployment{ContractName: "Token", BlockHeight: 100, ...}`.

API call `GET /experimental/v1/contracts/A.{addr}.Token` → response body contains `"contract_id": "A.{addr}.TokenV2"` with `"TokenV2"`'s code hash and transaction ID. [8](#0-7) [9](#0-8)

	if err != nil {
		return access.ContractDeployment{}, fmt.Errorf("could not get deployments for %s.%s: %w", account.Hex(), name, err)

	// iterate over deployments for the contract, and return the first one (most recent)

		}

	}

	// no deployments were found
	return access.ContractDeployment{}, storage.ErrNotFound
}

func makeContractDeploymentKey(addr flow.Address, name string, height uint64, txIndex, eventIndex uint32) []byte {
	nameBytes := []byte(name)
	key := make([]byte, contractDeploymentKeyOverhead+len(nameBytes))
	offset := 0

	key[offset] = codeContractDeployment
	offset++

	copy(key[offset:], addr[:])
	offset += flow.AddressLength

	copy(key[offset:], nameBytes)
	offset += len(nameBytes)

	binary.BigEndian.PutUint64(key[offset:], ^height) // one's complement for descending height order
	offset += heightLen

	binary.BigEndian.PutUint32(key[offset:], txIndex)
	offset += txIndexLen

	binary.BigEndian.PutUint32(key[offset:], eventIndex)

	return key
}

// makeContractDeploymentContractPrefix returns the prefix used to iterate over all deployments
// of a specific contract:
//
//	[codeContractDeployment][address bytes(8)][contract name bytes]
func makeContractDeploymentContractPrefix(addr flow.Address, name string) []byte {
	nameBytes := []byte(name)
	prefix := make([]byte, 1+flow.AddressLength+len(nameBytes))
	prefix[0] = codeContractDeployment
	copy(prefix[1:], addr[:])
	copy(prefix[1+flow.AddressLength:], nameBytes)
	return prefix
}

func contractDeploymentKeyPrefix(key []byte) ([]byte, error) {
	if len(key) < minValidKeyLen {
		return nil, fmt.Errorf("key too short: expected at least %d bytes, got %d", minValidKeyLen, len(key))
	}
	if key[0] != codeContractDeployment {
		return nil, fmt.Errorf("invalid key prefix: expected %d, got %d", codeContractDeployment, key[0])
	}
	return key[:len(key)-heightLen-txIndexLen-eventIndexLen], nil
}
