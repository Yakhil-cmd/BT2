### Title
Missing Caller Validation in `update_rewards` Allows Griefing and Reward Misrouting — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is specified to be callable **only by the Starknet sequencer**, but the implementation contains no caller validation. Any unprivileged address can invoke it, consuming the single per-block reward slot (`last_reward_block`) and either preventing the legitimate block producer from receiving rewards or redirecting rewards to an attacker-controlled staker.

---

### Finding Description

The protocol spec at `docs/spec.md:1645` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `update_rewards` at `src/staking/staking.cairo:1449–1507` only calls `self.general_prerequisites()` (a pause check) and validates the supplied `staker_address`, with no check that `get_caller_address()` is the authorized sequencer:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker existence/active checks ...
    self.last_reward_block.write(current_block_number);   // consumed for this block
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distribute rewards ...
}
``` [1](#0-0) 

The critical invariant is that `last_reward_block` is a **single global variable** written unconditionally at line 1485, even when `disable_rewards: true`. Once written, the `REWARDS_ALREADY_UPDATED` assertion at line 1454–1458 blocks any further call in the same block. [2](#0-1) 

The spec's access control requirement is documented but not enforced: [3](#0-2) 

---

### Impact Explanation

Two distinct attack paths exist:

**Path A — Freeze unclaimed yield (High):**
An attacker calls `update_rewards(valid_staker, disable_rewards: true)` in every block before the sequencer. This sets `last_reward_block` to the current block, causing the sequencer's subsequent call to revert with `REWARDS_ALREADY_UPDATED`. No staker accumulates rewards for that block. Repeated every block, this permanently freezes all unclaimed yield.

**Path B — Misroute rewards (High):**
An attacker calls `update_rewards(attacker_staker, disable_rewards: false)` before the sequencer. Rewards for that block are distributed to the attacker's staker instead of the legitimate block producer. The attacker's staker receives `block_rewards * attacker_stake / total_stake`, stealing yield that should have gone to the block producer's staker.

Both paths match the allowed impact: **Theft of unclaimed yield** and **Permanent/temporary freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is part of a public ABI (`IStakingRewardsManager`) callable by any address.
- The attacker only needs to know any valid, active staker address — this is fully public on-chain via `get_stakers`.
- On Starknet, transaction fees are low enough to sustain a per-block front-running campaign.
- No privileged access, leaked key, or external dependency is required. [4](#0-3) 

---

### Recommendation

Add an explicit sequencer caller check at the top of `update_rewards`, analogous to the existing `assert_caller_is_attestation_contract` pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
}
``` [5](#0-4) 

Store the authorized sequencer address in contract storage (set at initialization) and validate `get_caller_address() == self.sequencer_address.read()`.

---

### Proof of Concept

1. Staker A (attacker-controlled) stakes the minimum amount and waits K epochs for their balance to become effective.
2. Consensus rewards are activated (`set_consensus_rewards_first_epoch`).
3. In every new block, the attacker submits a transaction calling `update_rewards(staker_A_address, disable_rewards: false)`.
4. The call passes all checks: contract not paused, `current_block > last_reward_block`, staker A exists and is active, staker A has non-zero balance.
5. `last_reward_block` is set to the current block; rewards proportional to staker A's stake are credited to staker A.
6. The sequencer's subsequent `update_rewards` call for the legitimate block producer reverts with `REWARDS_ALREADY_UPDATED`.
7. The legitimate block producer receives zero rewards for that block; staker A receives rewards they did not earn by producing the block. [1](#0-0)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
