### Title
Access Node Program Cache Not Invalidated on Contract Update Due to Insufficient Gate Condition - (`module/state_synchronization/indexer/util.go`)

### Summary

The `programInvalidator` used by the Access node's indexer to invalidate its derived-data program cache has a gate function (`ShouldInvalidateEntries`) that only checks the `invalidateAll` boolean flag, completely ignoring the populated `invalidated` map of specific contract locations. As a result, any contract update that does not involve the service account silently fails to evict the stale cached program, and subsequent script executions on that Access node return results computed against the old contract code.

### Finding Description

`IndexerCore.updateProgramCache` in `module/state_synchronization/indexer/indexer_core.go` calls `findContractUpdates` to collect every `AccountContractUpdated` event from the block and stores the resulting `map[common.Location]struct{}` in the `programInvalidator.invalidated` field. It then calls `tx.AddInvalidator` and `tx.Commit()` to apply the invalidation to the `DerivedBlockData` cache. [1](#0-0) 

The `programInvalidator` struct in `module/state_synchronization/indexer/util.go` implements the `derived.ProgramInvalidator` interface. Its gate method is:

```go
func (inv *programInvalidator) ShouldInvalidateEntries() bool {
    return inv.invalidateAll
}
``` [2](#0-1) 

`invalidateAll` is set to `hasAuthorizedTransaction(collections, c.serviceAddress)` â€” it is `true` only when the service account signed a transaction in the block. [3](#0-2) 

The `DerivedDataTable` invalidation machinery calls `ShouldInvalidateEntries()` as a mandatory gate before ever calling `ShouldInvalidateEntry()` on individual cache entries. Because `ShouldInvalidateEntries()` returns `false` for any ordinary contract update (no service account involvement), the per-entry check is never reached and the `invalidated` map is never consulted. The populated map is dead data.

Compare this with the FVM's own `ProgramInvalidator`, which correctly gates on whether any contract was touched at all:

```go
func (invalidator ProgramInvalidator) ShouldInvalidateEntries() bool {
    return invalidator.ExecutionParametersUpdated ||
        invalidator.ContractUpdates.Any()
}
``` [4](#0-3) 

The Access node's `programInvalidator` is missing the equivalent `len(inv.invalidated) > 0` check in its gate.

The program cache itself is keyed by `common.AddressLocation` (address + contract name), so the cache key granularity is correct. The bug is that the gate that decides whether to walk the cache at all is keyed only on a single boolean (`invalidateAll`), discarding the richer per-contract information already collected. [5](#0-4) 

### Impact Explanation

An Access node with program caching enabled (`programCacheSize > 0`) will serve stale Cadence program bytecode for any contract updated by a non-service-account transaction. Scripts executed against that node â€” including balance checks, price queries, access-control checks, and any other read-only logic â€” will silently compute results against the pre-update contract code. The stale entry persists until the next service-account transaction triggers `invalidateAll = true`, or until the LRU cache naturally evicts the entry. Applications and wallets that rely on Access node script results to construct or validate on-chain transactions may act on incorrect data.

### Likelihood Explanation

Every ordinary contract update (the common case â€” no service account involvement) triggers this path. Any Cadence contract author can update their own contract via an unprivileged transaction, which is a normal, permissionless operation. Access nodes with the program cache enabled are the recommended production configuration. The bug is therefore reachable on every block that contains a contract update not signed by the service account.

### Recommendation

Change `ShouldInvalidateEntries` in `programInvalidator` to also return `true` when the `invalidated` map is non-empty:

```go
func (inv *programInvalidator) ShouldInvalidateEntries() bool {
    return inv.invalidateAll || len(inv.invalidated) > 0
}
```

This mirrors the FVM's own `ProgramInvalidator.ShouldInvalidateEntries` pattern and ensures the per-entry check is reached for targeted contract-update invalidations.

### Proof of Concept

1. Access node starts with `programCacheSize > 0`; program cache is empty.
2. A script calls contract `A` at address `0x01`; the program is loaded and cached under `AddressLocation{Address: 0x01, Name: "A"}`.
3. A transaction (not signed by the service account) updates contract `A`. The execution data contains an `AccountContractUpdated` event for `(0x01, "A")`.
4. `IndexerCore.updateProgramCache` runs: `findContractUpdates` returns `{AddressLocation{0x01,"A"}: {}}`, `hasAuthorizedTransaction` returns `false`.
5. `programInvalidator{invalidateAll: false, invalidated: {loc: {}}}` is committed.
6. `ShouldInvalidateEntries()` returns `false`; the cache walk is skipped entirely.
7. A subsequent script call to contract `A` hits the cache and executes the old bytecode, returning results from the pre-update contract. [6](#0-5) [2](#0-1) [7](#0-6)

func (c *IndexerCore) updateProgramCache(header *flow.Header, events []flow.Event, collections []*flow.Collection) error {
	if c.derivedChainData == nil {
		return nil
	}

	derivedBlockData := c.derivedChainData.GetOrCreateDerivedBlockData(
		header.ID(),
		header.ParentID,
	)

	// get a list of all contracts that were updated in this block
	updatedContracts, err := findContractUpdates(events)
	if err != nil {
		return fmt.Errorf("could not find contract updates for block %d: %w", header.Height, err)
	}

	// invalidate cache entries for all modified programs
	tx, err := derivedBlockData.NewDerivedTransactionData(0, 0)
	if err != nil {
		return fmt.Errorf("could not create derived transaction data for block %d: %w", header.Height, err)
	}

	tx.AddInvalidator(&accessInvalidator{
		programs: &programInvalidator{
			invalidated:   updatedContracts,
			invalidateAll: hasAuthorizedTransaction(collections, c.serviceAddress),
		},
		executionParameters: &executionParametersInvalidator{
			invalidateAll: hasAuthorizedTransaction(collections, c.serviceAddress),
		},
	})

	err = tx.Commit()
	if err != nil {
		return fmt.Errorf("could not commit derived transaction data for block %d: %w", header.Height, err)
	}

	return nil
}
```

// findContractUpdates returns a map of common.AddressLocation for all contracts updated within the
// provided events.
// No errors are expected during normal operation and indicate an invalid protocol event was encountered
func findContractUpdates(events []flow.Event) (map[common.Location]struct{}, error) {
	invalidatedPrograms := make(map[common.Location]struct{})
	for _, event := range events {
		if event.Type == accountContractUpdated {
			location, err := parseAccountContractUpdated(&event)
			if err != nil {
				return nil, fmt.Errorf("could not parse account contract updated event: %w", err)
			}
			invalidatedPrograms[location] = struct{}{}
		}
	}
	return invalidatedPrograms, nil
}
