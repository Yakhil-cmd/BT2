### Title
Global `last_reward_block` Prevents Multiple Stakers From Receiving Rewards Per Block - (File: src/staking/staking.cairo)

### Summary
`update_rewards` uses a single global `last_reward_block` state variable to gate reward distribution. Because this variable is shared across all stakers rather than tracked per-staker, only one staker can receive rewards per block. Any unprivileged caller can exploit this by calling `update_rewards` with `disable_rewards: true`, consuming the per-block slot without distributing any rewards, permanently starving all stakers of their unclaimed yield.

### Finding Description

In `staking.cairo`, the `update_rewards` function enforces a global "one call per block" invariant using a single storage slot:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // ... distribute rewards to staker_address only
}
``` [1](#0-0) 

The storage field `last_reward_block: BlockNumber` is a single global value, not a `Map<ContractAddress, BlockNumber>`: [2](#0-1) 

The function accepts an arbitrary `staker_address` parameter and is designed to be called once per block per staker. However, the guard `current_block_number > self.last_reward_block.read()` is evaluated against a single global value. The first call in any block — regardless of which staker is targeted — writes `current_block_number` into `last_reward_block`, causing every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

Critically, the function has **no access control** beyond `general_prerequisites()` (which only checks the pause flag). The test suite confirms this: `update_rewards` is called throughout without any `cheat_caller_address_once`, meaning any address can invoke it: [4](#0-3) 

The `disable_rewards: bool` parameter is accepted from any caller with no restriction. When `disable_rewards: true` is passed, the function writes `last_reward_block` and returns immediately — no rewards are distributed, but the per-block slot is consumed: [5](#0-4) 

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker_address, disable_rewards: true)` every block:

1. Consumes the global per-block slot (`last_reward_block` = current block).
2. Distributes zero rewards to anyone.
3. Causes every subsequent `update_rewards` call in that block to revert with `REWARDS_ALREADY_UPDATED`.

All stakers are permanently denied their per-block reward accrual for every block the attacker front-runs. Because rewards are never retroactively compensated for missed blocks, this constitutes **permanent freezing of unclaimed yield** for all stakers and their delegators.

This matches the allowed High impact: *Permanent freezing of unclaimed yield or unclaimed royalties*.

### Likelihood Explanation

- `update_rewards` is a public, permissionless entry point — no role check exists in the implementation.
- The attacker only needs a valid (active) staker address, which is publicly observable on-chain.
- The cost per block on Starknet is low, making sustained griefing economically viable.
- A single malicious staker can execute this attack against the entire protocol.

### Recommendation

Track `last_reward_block` per staker rather than globally:

```cairo
// Change:
last_reward_block: BlockNumber,

// To:
last_reward_block: Map<ContractAddress, BlockNumber>,
```

Then gate the check and write against `staker_address`:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(staker_address),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
// ...
self.last_reward_block.write(staker_address, current_block_number);
```

Additionally, enforce access control (e.g., only the sequencer/attestation contract) on `update_rewards` to prevent the `disable_rewards: true` griefing vector entirely.

### Proof of Concept

1. Stakers A, B, and C are all active with non-zero balances.
2. Consensus rewards are active (`!is_pre_consensus()`).
3. Attacker (any address) calls `update_rewards(staker_A, disable_rewards: true)` at block N.
   - `last_reward_block` is written to N.
   - No rewards are distributed.
4. Sequencer calls `update_rewards(staker_B, disable_rewards: false)` at block N.
   - Reverts: `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. Sequencer calls `update_rewards(staker_C, disable_rewards: false)` at block N.
   - Reverts: same reason.
6. Attacker repeats step 3 at every block.
7. Stakers B and C (and their delegators) permanently accumulate zero rewards.

The `REWARDS_ALREADY_UPDATED` revert path is confirmed by the existing test suite: [6](#0-5) [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
