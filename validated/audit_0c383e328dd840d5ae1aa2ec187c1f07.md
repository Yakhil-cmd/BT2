### Title
Missing Eth Connector Account Initialization in `new()` Causes All Bridge Functions to Fail — (File: `engine/src/contract_methods/admin.rs`, `engine/src/contract_methods/connector.rs`)

---

### Summary

The `new()` initialization function does not set the eth connector contract account in storage. Every bridge-facing function (`withdraw`, `ft_transfer`, `ft_transfer_call`, `ft_on_transfer`, `ft_metadata`) depends on `get_connector_account_id()`, which reads that storage key. If `set_eth_connector_contract_account` is never called after `new()`, all bridge operations fail with `ERR_CONNECTOR_STORAGE_KEY_NOT_FOUND`, freezing user funds.

---

### Finding Description

`new()` in `engine/src/contract_methods/admin.rs` only writes the engine state (chain ID, owner, upgrade delay): [1](#0-0) 

It does not set the eth connector account. The `state_migration()` entry point is also a no-op: [2](#0-1) 

The connector account is read from storage by `get_connector_account_id()`, which returns `ERR_CONNECTOR_STORAGE_KEY_NOT_FOUND` if the key is absent: [3](#0-2) 

Every bridge function routes through `return_promise()`, which calls `get_connector_account_id()` as its first step: [4](#0-3) 

Affected functions:

- `withdraw()` — calls `return_promise(io, env, "engine_withdraw", …)`: [5](#0-4) 

- `ft_transfer()` — calls `return_promise(io, env, "engine_ft_transfer", …)`: [6](#0-5) 

- `ft_transfer_call()` — calls `return_promise(io, env, "engine_ft_transfer_call", …)`: [7](#0-6) 

- `ft_metadata()` — calls `return_promise(io, env, "ft_metadata", …)`: [8](#0-7) 

- `ft_on_transfer()` — calls `get_connector_account_id()` directly with `?`, propagating the error: [9](#0-8) 

The only way to set the connector account is via a separate, post-deployment call to `set_eth_connector_contract_account`, which is not enforced or referenced anywhere in `new()`: [10](#0-9) 

---

### Impact Explanation

If `set_eth_connector_contract_account` is omitted after deployment:

- `withdraw()` fails — users cannot redeem bridged ETH back to Ethereum.
- `ft_transfer()` / `ft_transfer_call()` fail — ETH token transfers on NEAR are blocked.
- `ft_on_transfer()` fails — ETH deposits into Aurora are rejected (tokens returned to sender).
- `ft_metadata()` fails — metadata queries revert.

All bridge functionality is frozen. Funds already held in the eth connector contract cannot be moved through the engine until the owner calls `set_eth_connector_contract_account`. This constitutes **High — Temporary freezing of funds**.

---

### Likelihood Explanation

The `new()` function gives no indication that a second initialization call is required. There is no guard, no assertion, and no documentation comment in the function body pointing to `set_eth_connector_contract_account`. An operator following only the `new()` call signature would have no reason to know this step is mandatory. The deployment utilities in tests show the two-step pattern is required but it is not enforced on-chain: [11](#0-10) 

Likelihood is **Medium**: any deployment that omits the second step produces a fully broken bridge.

---

### Recommendation

Include the eth connector account (and withdraw serialization type) as required parameters of `new()`, and call `set_connector_account_id` / `set_connector_withdraw_serialization_type` inside `new()` before returning. This mirrors the pattern already used in test setup and eliminates the silent omission risk.

---

### Proof of Concept

1. Deploy Aurora Engine WASM.
2. Call `new(chain_id, owner_id, upgrade_delay_blocks)` — **do not** call `set_eth_connector_contract_account`.
3. A user deposits ETH via the eth connector; the connector calls `ft_on_transfer` on the engine.
4. `ft_on_transfer` calls `get_connector_account_id(&io)?` — storage key absent → `ERR_CONNECTOR_STORAGE_KEY_NOT_FOUND` propagates → NEAR transaction panics → deposit rejected.
5. User calls `withdraw(recipient, amount)` with 1 yoctoNEAR attached.
6. `withdraw` → `return_promise` → `get_connector_account_id` → same error → transaction fails.
7. All bridge operations remain frozen until the owner separately calls `set_eth_connector_contract_account`.

#[named]
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

#[named]
pub fn set_eth_connector_contract_account<I: IO + Copy, E: Env>(
    io: I,
    env: &E,
) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let is_private = env.assert_private_call();

        if is_private.is_err() {
            require_owner_only(&state, &env.predecessor_account_id())?;
        }

        let args: SetEthConnectorContractAccountArgs = io.read_input_borsh()?;

        set_connector_account_id(io, &args.account);
        set_connector_withdraw_serialization_type(io, &args.withdraw_serialize_type);

        Ok(())
    })
}

fn get_connector_account_id<I: IO>(io: &I) -> Result<AccountId, ContractError> {
    io.read_storage(&construct_contract_key(
        EthConnectorStorageId::EthConnectorAccount,
    ))
    .ok_or(errors::ERR_CONNECTOR_STORAGE_KEY_NOT_FOUND)
    .and_then(|x| {
        x.to_value()
            .map_err(|_| errors::ERR_BORSH_DESERIALIZE.as_bytes())
    })
    .map_err(Into::into)
}

fn return_promise<I: IO + PromiseHandler, E: Env>(
    mut io: I,
    env: &E,
    method: &str,
    args: Vec<u8>,
    deposit: Yocto,
) -> Result<(), ContractError> {
    let promise_args = PromiseCreateArgs {
        target_account_id: get_connector_account_id(&io)?,
        method: method.to_string(),
        args,
        attached_balance: deposit,
        attached_gas: calculate_attached_gas(env),
    };
    let promise_id = io.promise_create_call(&promise_args);

    io.promise_return(promise_id);

    Ok(())
