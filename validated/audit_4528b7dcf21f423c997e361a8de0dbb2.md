### Title
Presignature Reuse Enabled by `Clone` Derivation Allows Private Key Extraction — (File: src/ecdsa/ot_based_ecdsa/mod.rs)

### Summary
Both `PresignOutput` and `RerandomizedPresignOutput` derive `Clone`, silently undermining the move-semantics-based single-use guarantee the library relies on. The library provides no runtime tracking mechanism to detect or prevent presignature reuse. A malicious coordinator can orchestrate two signing sessions using the same presignature with different message hashes, causing honest participants to sign two distinct messages under the same ECDSA nonce, enabling full private key recovery via standard nonce-reuse algebra.

### Finding Description

**Root cause — `Clone` derivation on secret presignature material:**

`PresignOutput` and `RerandomizedPresignOutput` in the OT-based ECDSA module both derive `Clone`:

```rust

pub struct PresignOutput {
    pub big_r: AffinePoint,
    pub k: Scalar,       // secret nonce share
    pub sigma: Scalar,
}

// lines 54-63
#[derive(Debug, Clone, Serialize, Deserialize, ZeroizeOnDrop)]

``` [1](#0-0) 

The same pattern applies to the robust ECDSA `PresignOutput` and to the FROST `PresignOutput` (confirmed by `.clone()` usage in tests). [2](#0-1) 

**The library's only protection is documentation:**

Both presign functions carry the comment "it's crucial that a presignature is never reused," and the security documentation states "Never reuse a presignature, even across failed, aborted, or partially completed signing sessions." However, no runtime enforcement exists. [3](#0-2) [4](#0-3) 

**The `sign()` functions accept presignatures by value (move), but `Clone` trivially defeats this:**

```rust
// Caller can do:
let presig_clone = presig.clone();
sign(..., presig, h1)?;       // session 1
sign(..., presig_clone, h2)?; // session 2 — same nonce k, different message
``` [5](#0-4) 

**Concrete malicious coordinator attack path:**

1. Presign phase completes; each participant holds a `PresignOutput` share containing their nonce share `k_i`.
2. Malicious coordinator initiates signing session 1 with `msg_hash = h1`. Participants call `sign()`, compute their signature share `s_i = f(k_i, h1, x_i)`, and send it to the coordinator. The coordinator receives all shares but deliberately aborts without producing the final signature.
3. Participants have no way to know their presignature was consumed — the library provides no "used" flag, no nonce registry, and no consumed-token pattern.
4. Coordinator initiates signing session 2 with `msg_hash = h2`. Participants, holding the same `PresignOutput` (or a clone of it), call `sign()` again. They compute `s_i = f(k_i, h2, x_i)` using the same nonce share `k_i`.
5. The coordinator now holds two sets of signature shares over the same nonce `R = k·G` but different messages. Standard ECDSA nonce-reuse algebra recovers the aggregate private key `x`. [6](#0-5) [7](#0-6) 

The robust ECDSA documentation explicitly acknowledges this class of attack: "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks." [8](#0-7) 

### Impact Explanation

**Critical — private key extraction.** Two signing sessions over the same nonce `k` with distinct message hashes `h1`, `h2` yield:

```
s1 = k⁻¹(h1 + r·x)
s2 = k⁻¹(h2 + r·x)
```

Subtracting: `s1 - s2 = k⁻¹(h1 - h2)` → `k = (h1 - h2)/(s1 - s2)` → `x = (s1·k - h1)/r`.

This recovers the aggregate private key, matching the allowed Critical impact: *"Extraction, reconstruction, or disclosure of private signing shares, aggregate secret material, presign secrets, nonce material, or confidential derived secrets."*

### Likelihood Explanation

**Medium-High.** The attack is realistic in any production deployment that retries failed signing sessions — a common operational pattern in distributed systems. The malicious coordinator role is reachable without privileged assumptions: any participant can be designated coordinator. The coordinator needs only to abort one session after collecting shares, then initiate a second session with a different message hash. No cryptographic break is required; the algebra is elementary.

### Recommendation

1. **Remove `Clone` from `PresignOutput` and `RerandomizedPresignOutput`** (and the FROST equivalents). Rust's move semantics then enforce single-use at compile time, making reuse a compile error rather than a runtime risk.
2. **If `Clone` must be retained** for serialization or storage purposes, wrap presignatures in a `OnceCell`-style type or a `consumed: AtomicBool` guard that panics or errors on second use.
3. **Provide a nonce registry** at the library level that participants can use to record and check `big_r` values before signing, rejecting any presignature whose `R` has been seen before.
4. **Document the retry hazard explicitly** in the `sign()` function signatures, not only in the presign docstring and a separate markdown file.

### Proof of Concept

```rust
// Attacker-controlled coordinator flow (pseudocode)
let presig_output = run_presign(participants, ...);

// Clone the presignature before the first session
let presig_clone = presig_output.clone();

// Session 1: coordinator sends h1 to participants, collects shares, then aborts
let shares_h1 = collect_participant_shares(participants, presig_output, h1);
// coordinator does NOT call aggregate() — session silently fails

// Session 2: coordinator sends h2; participants reuse their PresignOutput
// (library provides no mechanism to detect reuse)
let shares_h2 = collect_participant_shares(participants, presig_clone, h2);

// Recover private key from two share sets over the same nonce R
let k = (h1 - h2) * (shares_h1 - shares_h2).invert();
let x = (shares_h1 * k - h1) * r.invert();
// x is the aggregate private signing key
```

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
