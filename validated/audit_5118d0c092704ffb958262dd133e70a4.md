### Title
Missing Participant Contribution Validation in CKD Protocol Allows Malicious Participant to Corrupt Confidential Key Derivation Output — (File: `src/confidential_key_derivation/protocol.rs`)

---

### Summary

The CKD coordinator in `do_ckd_coordinator` aggregates participant-supplied ElGamal shares `(big_y, big_c)` with no proof of correctness. Any single malicious participant can substitute arbitrary group elements, causing the coordinator to produce a silently corrupted `CKDOutput`. Honest callers who unmask the result obtain a wrong confidential key with no indication of failure — a direct analog to accepting a swap with `minAmountOut = 0`.

---

### Finding Description

The CKD protocol derives a confidential key `msk · H(pk, app_id)` via a distributed ElGamal construction. Each participant is supposed to contribute:

```
norm_big_y = λᵢ · yᵢ · G
norm_big_c = λᵢ · (xᵢ · H(pk, app_id) + yᵢ · app_pk)
```

The coordinator sums all contributions and returns `(Y, C)` such that `C − app_sk · Y = msk · H(pk, app_id)`.

In `do_ckd_coordinator` the aggregation loop is:

```rust
for (_, participant_output) in
    recv_from_others::<CKDOutput>(&chan, waitpoint, &participants, me).await?
{
    norm_big_y += participant_output.big_y();
    norm_big_c += participant_output.big_c();
}
let ckd_output = CKDOutput::new(norm_big_y, norm_big_c);
Ok(Some(ckd_output))
```

There is **no ZK proof** that `big_c` was formed with the participant's actual private share `xᵢ`, and **no range or consistency check** on the received group elements. The function unconditionally adds whatever values arrive from the network.

Contrast this with the DKG protocol, which enforces proof-of-knowledge verification (`verify_proof_of_knowledge`) and commitment-hash binding (`verify_commitment_hash`) before accepting any participant contribution. The CKD protocol has neither.

A malicious participant can send any pair `(big_y', big_c')` with `big_c' ≠ λᵢ(xᵢ · H(pk, app_id) + yᵢ · app_pk)`. The resulting output satisfies:

```
C' − app_sk · Y' = msk · H(pk, app_id) + Δc − app_sk · Δy
```

where `Δc` and `Δy` are the attacker-controlled deltas. Because the attacker does not know `app_sk`, they cannot cancel the error term, so the unmasked key is wrong. The caller receives `Ok(Some(ckd_output))` with no error.

---

### Impact Explanation

**High — Corruption of CKD outputs so honest parties accept unusable cryptographic outputs.**

The coordinator and every downstream consumer of the CKD result silently obtain an incorrect confidential key. There is no post-hoc verification step: `CKDOutput::unmask` simply computes `C − app_sk · Y` and returns the result. The corrupted key is indistinguishable from a legitimate one without an independent oracle for `msk · H(pk, app_id)`.

---

### Likelihood Explanation

Any participant in the CKD session is a valid attacker. The exploit requires only sending two arbitrary non-identity `G1` points instead of the honest contribution — a single-message, zero-cryptographic-effort attack. No special privilege, leaked key, or external assumption is needed. The CKD protocol has no threshold tolerance: even one malicious participant out of N corrupts the output.

---

### Recommendation

Add a ZK proof of correct ElGamal share formation to each participant's message. Concretely, each participant should prove knowledge of `(xᵢ, yᵢ)` such that:

```
big_c = xᵢ · H(pk, app_id) + yᵢ · app_pk
big_y = yᵢ · G
```

This is a standard Chaum–Pedersen / DLog-equality proof (the `dlogeq` module already exists in `src/crypto/proofs/dlogeq.rs`). The coordinator must verify each proof before adding the contribution, mirroring the pattern used in `do_keyshare` for DKG.

---

### Proof of Concept

1. Malicious participant `P_m` intercepts the protocol and, instead of calling `compute_signature_share`, sends `(G₁, G₁)` — two copies of the G1 generator — to the coordinator.
2. `do_ckd_coordinator` adds these values unconditionally:
   ```rust
   norm_big_y += participant_output.big_y();  // += G₁
   norm_big_c += participant_output.big_c();  // += G₁
   ```
3. The coordinator returns `CKDOutput::new(Y + G₁, C + G₁)` with `Ok(Some(...))`.
4. The application calls `ckd_output.unmask(app_sk)`:
   ```
   (C + G₁) − app_sk · (Y + G₁) = msk · H(pk, app_id) + G₁ − app_sk · G₁
                                  = msk · H(pk, app_id) + (1 − app_sk) · G₁
   ```
5. The returned confidential key is wrong by `(1 − app_sk) · G₁`. No error is raised anywhere in the call chain. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}

pub fn verify<C: Ciphersuite>(
    transcript: &mut Transcript,
    statement: Statement<'_, C>,
    proof: &Proof<C>,
) -> Result<bool, ProtocolError>
where
    Element<C>: ConstantTimeEq,
{
    if statement.generator1.ct_eq(&C::Group::identity()).into() {
        return Err(ProtocolError::IdentityElement);
    }

    transcript.message(NEAR_DLOGEQ_STATEMENT_LABEL, &statement.encode()?);

    let (phi0, phi1) = statement.phi(&proof.s.0);
    let big_k0 = phi0 - *statement.public0 * proof.e.0;
    let big_k1 = phi1 - *statement.public1 * proof.e.0;

    let enc = encode_two_points::<C>(&big_k0, &big_k1)?;

    transcript.message(NEAR_DLOGEQ_COMMITMENT_LABEL, &enc);
    let mut rng = transcript.challenge_then_build_rng(NEAR_DLOGEQ_CHALLENGE_LABEL);
    let e = frost_core::random_nonzero::<C, _>(&mut rng);

    Ok(e == proof.e.0)
