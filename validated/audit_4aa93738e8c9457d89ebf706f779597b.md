### Title
Missing Declaration Check for Cairo V0 Classes in `get_compiled_class` Allows Undeclared Class Execution via RPC — (`crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

When `class_manager_handle` is `Some`, `ExecutionStateReader::get_compiled_class` enforces `is_contract_class_declared` for `ContractClass::V1` but unconditionally skips that check for `ContractClass::V0`. A caller-controlled `class_hash` that resolves to a Cairo V0 class in the class manager but was not declared at the queried `state_number` is returned as a valid `RunnableCompiledClass::V0`, causing RPC execution, fee estimation, and simulation to operate on an undeclared class.

---

### Finding Description

In `get_compiled_class`, the class-manager branch handles the two contract class variants asymmetrically: [1](#0-0) 

For `ContractClass::V1`, `is_contract_class_declared` is called and the function returns `StateError::UndeclaredClassHash` if the class is not present in storage at `state_number`: [2](#0-1) 

For `ContractClass::V0`, no such check exists — the class is returned directly: [3](#0-2) 

The TODO comment in the source explicitly acknowledges this gap: [4](#0-3) 

`is_contract_class_declared` checks `get_class_definition_block_number` against `state_number.is_after(block_number)`: [5](#0-4) 

This check is entirely absent for the V0 arm. The class manager stores classes globally without filtering by `state_number`, so a V0 class declared in block N is returned by `get_executable` even when the reader's `state_number` is before block N.

---

### Impact Explanation

`ExecutionStateReader` is used directly in `execute_call`, `estimate_fee`, and `simulate_transactions`: [6](#0-5) [7](#0-6) 

An attacker who submits an RPC request (`starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`) targeting a historical `state_number` and includes a library call to a V0 `class_hash` that was declared in a later block will receive execution results computed against an undeclared class. The concrete corrupted values are: wrong call return data, wrong fee estimate, and wrong simulation trace — all presented as authoritative RPC responses.

**Impact: High** — RPC execution, fee estimation, and simulation return an authoritative-looking wrong value (undeclared Cairo V0 class executed as if declared).

---

### Likelihood Explanation

- Requires only an unprivileged RPC call with an attacker-chosen `class_hash`.
- The attacker only needs to know any V0 `class_hash` that exists in the class manager but was declared after the queried `state_number` — a condition that is trivially observable from chain history.
- The `class_manager_handle` is `Some` in the standard RPC execution configuration.
- The bug is acknowledged by the developers (TODO comment) and has no existing mitigation.

---

### Recommendation

Apply the same `is_contract_class_declared` guard to the `ContractClass::V0` arm, mirroring the V1 logic:

```rust
ContractClass::V0(deprecated_contract_class) => {
    let is_declared = is_contract_class_declared(
        &self.storage_reader.begin_ro_txn().map_err(storage_err_to_state_err)?,
        &class_hash,
        self.state_number,
    )
    .map_err(|e| StateError::StateReadError(e.to_string()))?;

    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
```

This resolves the TODO and makes V0 and V1 handling consistent.

---

### Proof of Concept

The existing test infrastructure in `state_reader_test.rs` already provides `set_execution_state_reader` and `MockClassManagerClient`. A confirming test:

1. Write a block 1 to storage with a V0 class declared in block 1 (`deprecated_declared_classes: vec![class_hash_v0]`).
2. Configure `MockClassManagerClient::get_executable` to return `ContractClass::V0(deprecated_class)` for `class_hash_v0`.
3. Set `state_number = StateNumber::unchecked_right_after_block(BlockNumber(0))` (before block 1).
4. Call `exec_state_reader.get_compiled_class(class_hash_v0)`.
5. **Current behavior**: returns `Ok(RunnableCompiledClass::V0(...))` — undeclared class executed.
6. **Expected behavior**: returns `Err(StateError::UndeclaredClassHash(class_hash_v0))`. [8](#0-7)

pub(crate) fn is_contract_class_declared(
    txn: &StorageTxn<'_, RO>,
    class_hash: &ClassHash,
    state_number: StateNumber,
) -> Result<bool, ExecutionUtilsError> {
    Ok(txn
        .get_state_reader()?
        .get_class_definition_block_number(class_hash)?
        .is_some_and(|block_number| state_number.is_after(block_number)))
}
