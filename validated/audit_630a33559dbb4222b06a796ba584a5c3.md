### Title
Unprivileged Caller Can Permanently Suppress Block Reward Distribution via `update_rewards(disable_rewards: true)` - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` accepts a user-controlled `disable_rewards: bool` parameter with no access control beyond a non-zero caller check. Any unprivileged address can call it with `disable_rewards: true` every block, permanently preventing block rewards from being distributed to all stakers. Because `last_reward_block` is a global singleton updated on every call, a single attacker transaction per block is sufficient to freeze the entire reward stream indefinitely.

---

### Finding Description

`update_rewards` is exposed as a public function in `IStakingRewardsManager`. Its only guard is `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero.

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);   // ← global singleton updated

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits before distributing rewards
    }
    // ... reward distribution logic
``` [1](#0-0) 

The `last_reward_block` field is a **global** storage variable, not per-staker:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The guard `current_block_number > self.last_reward_block.read()` means only one call per block is possible. Once an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)`, `last_reward_block` is set to the current block and no legitimate call can distribute rewards for that block. The attacker repeats this every block.

The analog to the referral bug is direct: just as `mintWithReferralFor()` accepted both a `referralCode` and a `referrer` as unconstrained user inputs (allowing the user to route the referral payout to themselves), `update_rewards` accepts both a `staker_address` and a `disable_rewards` flag as unconstrained user inputs, allowing any caller to route the reward distribution outcome to "nothing."

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Block rewards (`strk_block_rewards`, `btc_block_rewards`) are computed per block and distributed through `_update_rewards` → `send_rewards_to_delegation_pool` / `send_rewards_to_staker`. If `disable_rewards: true` is passed, the function returns before any of this logic executes. Because `last_reward_block` is already stamped, no other caller can recover that block's rewards. An attacker repeating this every block causes a total, permanent freeze of all consensus-phase block rewards for every staker in the protocol. [3](#0-2) 

---

### Likelihood Explanation

**High.** The entry point is fully public (no role check, no allowlist). The only cost to the attacker is gas per block. The attacker needs no stake, no special role, and no prior setup. Any EOA or contract can execute this attack continuously. [4](#0-3) 

---

### Recommendation

Restrict `update_rewards` to authorized callers only (e.g., the attestation contract, a designated rewards manager role, or the staker/operational address). The `disable_rewards` flag should not be a free parameter exposed to arbitrary callers. If the flag is needed for protocol-internal flows (e.g., migration), gate it behind `only_app_governor()` or an equivalent role check, consistent with how other sensitive config functions are protected. [5](#0-4) 

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker (any non-zero address, no stake required) calls:
   ```
   staking.update_rewards(staker_address: any_active_staker, disable_rewards: true)
   ```
   at block `N`.
3. `last_reward_block` is written to `N`; the function returns before distributing rewards.
4. Any legitimate call to `update_rewards` at block `N` now reverts with `REWARDS_ALREADY_UPDATED`.
5. Attacker repeats at block `N+1`, `N+2`, … indefinitely.
6. All stakers and delegators receive zero block rewards for every block the attacker targets. The lost rewards are unrecoverable because the per-block reward window has passed.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
