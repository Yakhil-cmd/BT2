### Title
`MsgTurnBridge` Handler is a No-Op with No Permission Check, Permanently Breaking the Bridge Circuit Breaker - (File: `x/cronos/keeper/msg_server.go`)

### Summary
The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is implemented as a complete stub (`return nil, nil`). It performs no permission check and no actual bridge enable/disable logic. This means: (1) any unprivileged user can call `MsgTurnBridge` without the `CanTurnBridge` permission, bypassing the authorization check; and (2) the bridge circuit breaker is permanently non-functional, preventing the admin or permissioned addresses from ever disabling the Gravity Bridge in an emergency.

### Finding Description

The proto definition explicitly documents the intent:

```proto
// TurnBridge defines a method to disable or enable the gravity bridge
rpc TurnBridge(MsgTurnBridge) returns (MsgTurnBridgeResponse);
``` [1](#0-0) 

The message carries a `sender` signer and an `enable` boolean: [2](#0-1) 

The permission system defines `CanTurnBridge` as a dedicated permission bit, and the query server exposes it as a queryable permission for any address: [3](#0-2) [4](#0-3) 

The admin can grant `CanTurnBridge` to specific addresses via `MsgUpdatePermissions`, and ADR-009 explicitly names `MsgTurnBridge` as one of the two restricted messages requiring this permission: [5](#0-4) 

However, the actual handler implementation is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [6](#0-5) 

Compare this to `UpdateTokenMapping`, the other restricted message, which correctly enforces its permission: [7](#0-6) 

The `TurnBridge` handler:
1. Does **not** call `HasPermission(ctx, ..., CanTurnBridge)` — any unprivileged sender passes silently.
2. Does **not** write any state — the bridge enable/disable flag is never toggled.
3. Returns `nil, nil` — the transaction is accepted on-chain and fees are consumed, but nothing happens.

The handler is registered in the live gRPC service descriptor and is reachable by any user: [8](#0-7) 

### Impact Explanation

**High — Bypass of Cronos admin/permission authorization checks:** The `CanTurnBridge` permission is defined, grantable, and queryable, but is never enforced in the handler. Any unprivileged address can submit `MsgTurnBridge` and receive a successful on-chain response without holding the permission, directly violating the permission model described in ADR-009.

**High — Permanent inability to halt bridge/conversion flows:** The Gravity Bridge circuit breaker is permanently broken. No matter what the admin or a `CanTurnBridge`-permissioned address submits, the bridge state is never changed. In an active bridge exploit scenario, the intended emergency stop mechanism is completely inoperative, leaving bridge/conversion flows running indefinitely with no on-chain remedy.

### Likelihood Explanation

Certainty is high. The handler is a two-line stub with no conditional logic. Any user on mainnet can submit `MsgTurnBridge` today and observe a successful transaction that changes no state. The broken circuit breaker is a latent, permanent condition that activates the moment a bridge emergency arises.

### Recommendation

Implement `TurnBridge` with:
1. A `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` guard, mirroring `UpdateTokenMapping`.
2. Actual state mutation — e.g., persisting the `enable` boolean to module params or a dedicated store key, and having the Gravity Bridge end-blocker/hooks read that flag before processing transfers.

### Proof of Concept

1. Any unprivileged address submits `MsgTurnBridge{sender: attacker, enable: false}`.
2. The handler executes `return nil, nil` — no permission check, no state write.
3. The transaction is committed on-chain with code `0` (success); fees are deducted.
4. The Gravity Bridge continues operating normally; the `enable: false` intent is silently discarded.
5. Simultaneously, the legitimate admin or a `CanTurnBridge`-permissioned address submitting the same message to halt the bridge during an exploit receives the same no-op response — the bridge cannot be stopped.

// UpdateTokenMapping implements the grpc method
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}
