### Title
Paused BTC Token Blocks `unstake_action`, Temporarily Freezing Staker's STRK Funds - (File: src/staking/staking.cairo)

### Summary
The `unstake_action` function tightly couples BTC token transfers to pool contracts with the staker's STRK withdrawal. If a BTC token supported by the protocol has pause functionality and is paused by its token admin, `unstake_action` reverts entirely, temporarily freezing the staker's STRK funds in the staking contract.

### Finding Description
The Starknet Staking protocol explicitly supports multi-token staking, including BTC tokens added by the `token_admin` via `add_token`. When a staker has a BTC delegation pool and calls `unstake_action`, the function executes the following sequence:

1. Sends STRK rewards to the staker's reward address via `send_rewards_to_staker`
2. Removes the staker record from storage via `remove_staker`
3. Transfers the staker's STRK stake back to the staker
4. Calls `transfer_to_pools_when_unstake`, which iterates over **all** pools (STRK and BTC) and transfers each pool's delegated token balance to the respective pool contract [1](#0-0) 

Inside `transfer_to_pools_when_unstake`, for each pool the function calls `token_dispatcher.checked_transfer` to push the delegated token balance to the pool contract: [2](#0-1) 

If the BTC token contract has pause functionality (common in many real-world tokens, as demonstrated by Axie Infinity in the external report) and the BTC token admin pauses the token, the `checked_transfer` call at line 1676–1680 will revert. Because this transfer is tightly coupled inside `unstake_action` with no fallback or decoupling mechanism, the **entire transaction reverts**, including the STRK transfer back to the staker and the `remove_staker` storage changes.

The staker is left in a state where:
- `unstake_intent` has already been called (their stake is removed from `tokens_total_stake_trace`)
- The exit window has already elapsed
- `unstake_action` cannot be completed
- Their STRK principal remains locked in the staking contract

The staker has no recourse within the protocol to bypass this — they must wait for the BTC token to be unpaused by its external admin.

The `add_token` function does not validate whether the token has pause functionality: [3](#0-2) 

The `disable_token` function can disable a BTC token going forward, but it does not help stakers who already have active BTC pools with non-zero delegated balances — `transfer_to_pools_when_unstake` iterates over all pools regardless of active status: [4](#0-3) 

### Impact Explanation
**Temporary freezing of funds.** The staker's STRK principal is locked in the staking contract for the duration of the BTC token pause. The staker has already completed the intent step and waited the `exit_wait_window` (default 1 week, up to 12 weeks), yet cannot complete the exit. Since `unstake_intent` already removed the staker's stake from `tokens_total_stake_trace`, the staker also earns no further rewards during this period.

### Likelihood Explanation
The Starknet Staking protocol explicitly supports BTC tokens as a first-class feature. Many real-world BTC-wrapped tokens (e.g., WBTC, tBTC) include pause/emergency-stop functionality controlled by their respective token admins. The `token_admin` of the Starknet Staking protocol adds these tokens without any check for pause functionality. A pause event on any supported BTC token — whether due to a security incident, regulatory action, or upgrade — would trigger this freeze for all stakers with active BTC pools at the time of their `unstake_action` call.

### Recommendation
Decouple the BTC token transfer from `unstake_action` by implementing a pull-based mechanism. Instead of pushing BTC tokens to pool contracts during `unstake_action`, record the owed amount in storage and allow pool contracts to claim their BTC tokens independently via a separate function. This mirrors the recommendation in the external report: introduce a timestamp tracker when the staker calls `unstake_action` and allow the BTC transfer to be finalized separately once the token is unpaused.

### Proof of Concept
1. `token_admin` calls `add_token(btc_token_address)` where `btc_token_address` is a pausable BTC token.
2. Staker calls `set_open_for_delegation(btc_token_address)`, creating a BTC delegation pool.
3. Delegators call `enter_delegation_pool` on the BTC pool, depositing BTC. The pool now has a non-zero `delegated_balance` in `staker_delegated_balance_trace`.
4. Staker calls `unstake_intent()`. Their STRK and BTC delegated stakes are removed from `tokens_total_stake_trace`.
5. Staker waits for `exit_wait_window` to elapse.
6. BTC token admin pauses the BTC token contract.
7. Staker calls `unstake_action(staker_address)`.
   - `send_rewards_to_staker` succeeds (STRK transfer).
   - `remove_staker` succeeds (storage update).
   - `checked_transfer(recipient: staker_address, amount: staker_amount)` succeeds (STRK transfer).
   - `transfer_to_pools_when_unstake` is called; when it reaches the BTC pool, `token_dispatcher.checked_transfer(recipient: btc_pool_contract, amount: btc_balance)` **reverts** because the BTC token is paused.
   - The entire transaction reverts. The staker's STRK remains locked.
8. The staker cannot retry `unstake_action` until the BTC token admin unpauses the token. There is no protocol-level mechanism to bypass this. [5](#0-4) [2](#0-1)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount

        fn add_token(ref self: ContractState, token_address: ContractAddress) {
            self.roles.only_token_admin();
            assert!(token_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
            assert!(self.staker_info.read(token_address).is_none(), "{}", Error::TOKEN_IS_STAKER);
            assert!(token_address != STRK_TOKEN_ADDRESS, "{}", Error::INVALID_TOKEN_ADDRESS);
            assert!(
                self.btc_tokens.read(token_address).is_none(), "{}", Error::TOKEN_ALREADY_EXISTS,
            );
            let token_dispatcher = IERC20MetadataDispatcher { contract_address: token_address };
            let decimals = token_dispatcher.decimals();
            assert!(decimals >= 5 && decimals <= 18, "{}", GenericError::INVALID_TOKEN_DECIMALS);
            self.btc_tokens.write(token_address, (STARTING_EPOCH, false));
            self.token_decimals.write(token_address, decimals);
            // Initialize the token total stake trace.
            self
                .tokens_total_stake_trace
                .entry(token_address)
                .insert(key: STARTING_EPOCH, value: Zero::zero());
            self.emit(TokenManagerEvents::TokenAdded { token_address });
        }

        fn disable_token(ref self: ContractState, token_address: ContractAddress) {
            self.roles.only_security_agent();
            let is_active_opt: Option<(Epoch, bool)> = self.btc_tokens.read(token_address);
            assert!(is_active_opt.is_some(), "{}", Error::TOKEN_NOT_EXISTS);
            let (is_active_first_epoch, is_active) = is_active_opt.unwrap();
            let curr_epoch = self.get_current_epoch();
            assert!(curr_epoch >= is_active_first_epoch, "{}", Error::INVALID_EPOCH);
            assert!(is_active, "{}", Error::TOKEN_ALREADY_DISABLED);
            let next_is_active_first_epoch = self.get_epoch_plus_k();
            self.btc_tokens.write(token_address, (next_is_active_first_epoch, false));
            self.emit(TokenManagerEvents::TokenDisabled { token_address });
        }

        fn transfer_to_pools_when_unstake(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
        ) {
            for (pool_contract, token_address) in staker_pool_info.pools {
                let pool_balance = self.get_delegated_balance(:staker_address, :pool_contract);
                let token_dispatcher = IERC20Dispatcher { contract_address: token_address };
                let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };
                pool_dispatcher.set_staker_removed();
                self
                    .insert_staker_delegated_balance(
                        :staker_address, :pool_contract, delegated_balance: Zero::zero(),
                    );
                let decimals = self.get_token_decimals(:token_address);
                token_dispatcher
                    .checked_transfer(
                        recipient: pool_contract,
                        amount: pool_balance.to_native_amount(:decimals).into(),
                    );
            }
        }
