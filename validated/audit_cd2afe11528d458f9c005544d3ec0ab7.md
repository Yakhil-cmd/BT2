### Title
Unauthorized Burn/Transfer of Native Tokens via Bank Precompile Missing Caller Authorization - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The bank precompile's `burn` and `transfer` methods accept an arbitrary `address` argument specifying whose native tokens to burn or transfer, but never verify that the EVM contract invoking the precompile is authorized to act on that address. Any EVM contract can burn or transfer `evm/<callerContractAddress>` native tokens from any victim address without the token holder's consent.

### Finding Description
In `BankContract.Run()`, the `burn` case unpacks `recipient` (the address to burn from) and the `transfer` case unpacks `sender` (the address to transfer from). In both cases the denom is derived from `contract.Caller()` — the address of the EVM contract calling the precompile — but there is **no check** that `recipient`/`sender` equals `contract.Caller()` or that the token holder has granted any approval.

For `burn`: [1](#0-0) 

`recipient` is the account whose native tokens are burned. Nothing prevents `recipient` from being an arbitrary victim address.

For `transfer`: [2](#0-1) 

`sender` is the account whose native tokens are sent. Nothing prevents `sender` from being an arbitrary victim address.

Compare this with the generic `exec` helper used by other precompiles, which explicitly enforces `caller == signer`: [3](#0-2) 

The bank precompile performs no equivalent check.

### Impact Explanation
**Critical** — unauthorized burn and transfer of precompile-controlled assets (`evm/<contractAddress>` native tokens).

A malicious EVM contract `0xMAL` can:
- Call `bank.burn(victimAddress, amount)` → destroys `evm/0xMAL` native tokens held by `victimAddress` with no consent.
- Call `bank.transfer(victimAddress, attackerAddress, amount)` → steals `evm/0xMAL` native tokens from `victimAddress`.

These are Cosmos-side native bank balances, not EVM state. The `bankKeeper.SendCoinsFromAccountToModule` and `bankKeeper.SendCoins` calls execute unconditionally against the victim's Cosmos account: [4](#0-3) [5](#0-4) 

### Likelihood Explanation
Any unprivileged actor can deploy an EVM contract and call the bank precompile at `0x0000000000000000000000000000000000000064`. The interface is publicly documented: [6](#0-5) 

The attacker must first lure victims into holding `evm/<attackerContract>` native tokens (e.g., by presenting a legitimate-looking DeFi contract that calls `bank.mint` on deposit). Once victims hold the denom, the attacker can drain them at will.

### Recommendation
Add an authorization check in both the `burn` and `transfer` cases to ensure the address being debited equals `contract.Caller()`:

```go
// burn case
if recipient != contract.Caller() {
    return nil, errors.New("burn: caller not authorized to burn from this address")
}

// transfer case
if sender != contract.Caller() {
    return nil, errors.New("transfer: caller not authorized to transfer from this address")
}
```

Alternatively, adopt the same signer-verification pattern used in `exec()` in `utils.go`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function burn(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
}

contract BankPrecompileExploit {
    // Cronos bank precompile address
    IBankModule constant bank = IBankModule(0x0000000000000000000000000000000000000064);

    // Step 1: Lure victim into holding evm/<this contract address> native tokens
    // (e.g., expose a mint function that calls bank.mint(msg.sender, amount))

    // Step 2: Drain victim's native tokens without their consent
    function stealTokens(address victim, uint256 amount, address attacker) external {
        // Transfers evm/<address(this)> native tokens from victim to attacker
        // No approval from victim required — precompile does not check
        bank.transfer(victim, attacker, amount);
    }

    function burnVictimTokens(address victim, uint256 amount) external {
        // Burns evm/<address(this)> native tokens from victim
        // No approval from victim required
        bank.burn(victim, amount);
    }
}
```

The `denom` resolved inside the precompile will be `evm/<address(BankPrecompileExploit)>`. Any victim holding that denom's native tokens can have them stolen or burned by the contract deployer with no on-chain authorization check. [7](#0-6)
