### Title
Signature Verification Bypass via `skip_stateful_validations` Allows Unsigned Invoke Transactions into Mempool — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function skips the `__validate__` entry-point (signature check) for any invoke transaction with `nonce == 1` when `account_tx_in_pool_or_recent_block` returns `true` for the sender address. That helper returns `true` for **any** transaction type in the pool, not specifically a `deploy_account`. An unprivileged attacker who observes a victim's pending `deploy_account` in the public mempool can immediately submit a forged invoke with `nonce = 1` for the victim's address and have it admitted without any signature verification.

---

### Finding Description

`skip_stateful_validations` is the UX shortcut that lets a user broadcast `deploy_account` + `invoke` simultaneously before the account is on-chain:

```rust

            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())   // ← the guard

`tx_pool.contains_account` is true for **any** transaction type stored for that address — invoke, declare, or deploy_account — and `state.contains_account` is true for any address that has ever appeared in a committed or staged block:

The code comment claims this is sufficient because the account "either has a deploy_account transaction or transactions with future nonces that passed validations." The second branch is circular (the transaction being validated has not yet passed), and the first branch is never actually verified — the check does not distinguish `deploy_account` from any other transaction type.

When `skip_stateful_validations` returns `true`, `run_validate_entry_point` sets `execution_flags.validate = false` and calls the blockifier validator, which then skips the `__validate__` entry-point entirely:

---

### Impact Explanation

An attacker can inject an invoke transaction carrying an arbitrary (garbage) signature into the mempool for any account whose `deploy_account` is currently pending. The transaction is admitted without any cryptographic check. Consequences:

- **Mempool pollution / DoS**: If fee escalation is disabled (`enable_fee_escalation = false`), the mempool returns `DuplicateNonce` for the victim's legitimate `nonce = 1` invoke, blocking it from admission until the attacker's transaction is evicted or executed.
- **Forced fee escalation**: If fee escalation is enabled, the victim must outbid the attacker's tip to replace the forged transaction.
- **Batcher waste**: The batcher picks up the forged invoke, executes the `deploy_account` first (deploying the account), then calls `__validate__` on the forged invoke — which fails — causing the transaction to be rejected and the slot wasted.

This matches the allowed High impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The attack requires no privileges. The Starknet P2P layer propagates pending transactions publicly, so any observer can detect a `deploy_account` for address X and immediately race an unsigned invoke with `nonce = 1` for X. The window is the entire time the `deploy_account` sits in the mempool before being committed.

---

### Recommendation

Replace the coarse `account_tx_in_pool_or_recent_block` guard with a check that specifically confirms a `deploy_account` transaction is pending for the sender address. Add a dedicated mempool API, e.g. `has_pending_deploy_account(address: ContractAddress) -> bool`, that inspects the transaction type stored in `tx_pool` for that address, and use it in `skip_stateful_validations` instead of the current generic presence check.

---

### Proof of Concept

```
1. Victim broadcasts:
       deploy_account { sender = X, nonce = 0, class_hash = C, salt = S, ... }
   → admitted to mempool; account_tx_in_pool_or_recent_block(X) now returns true.

2. Attacker (observing the P2P mempool) broadcasts:
       invoke { sender_address = X, nonce = 1, signature = [0xdeadbeef] }

3. Gateway stateful validation for attacker's invoke:
   a. validate_nonce:  0 ≤ 1 ≤ max_allowed_nonce_gap  → OK
   b. validate_by_mempool: no dup hash, nonce 1 ≥ account_nonce 0 → OK
   c. skip_stateful_validations:
        tx is Invoke ✓, tx.nonce == 1 ✓, account_nonce == 0 ✓,
        account_tx_in_pool_or_recent_block(X) == true ✓
        → returns true  (SKIP __validate__)
   d. run_validate_entry_point: execution_flags.validate = false → __validate__ NOT called

4. Attacker's invoke is added to the mempool with no signature check.

5. Victim tries to submit their own invoke { sender = X, nonce = 1, valid_sig }:
   → mempool returns DuplicateNonce (fee escalation disabled) or forces higher tip.

6. Batcher execution:
   - deploy_account(X) executes → account deployed at X.
   - invoke(X, nonce=1, garbage_sig) executes → __validate__ called → FAILS.
   - Transaction rejected; victim's nonce not incremented, no fee charged,
     but the slot was wasted and the victim's tx was blocked.
```

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
