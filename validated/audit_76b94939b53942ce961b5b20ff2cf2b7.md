### Title
`switch_delegation_pool` Lacks Maximum Commission Protection, Enabling Staker Front-Running - (File: src/pool/pool.cairo)

### Summary
The `switch_delegation_pool` function accepts no `max_commission` parameter for the destination pool. A staker who holds an active `commission_commitment` can front-run a delegator's pool-switch transaction by raising their commission to `max_commission`, forcing the delegator into the new pool at a higher commission rate than they intended to accept.

### Finding Description
When a delegator calls `switch_delegation_pool`, the function signature is:

```rust
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
) -> Amount
``` [1](#0-0) 

No commission constraint is checked anywhere in this function or in the downstream `switch_staking_delegation_pool` call in the staking contract. [2](#0-1) 

The protocol's `set_commission_commitment` feature explicitly allows a staker to set a `max_commission` ceiling that is **higher** than their current commission:

```rust
let current_commission = staker_pool_info.commission();
assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
``` [3](#0-2) 

Once a commitment is active, `update_commission` permits raising commission to any value up to `max_commission`:

```rust
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
``` [4](#0-3) 

This means a staker can legitimately increase their commission at any time while a commitment is active. Because `switch_delegation_pool` reads the destination pool's commission only at execution time (via `get_commission_from_staking_contract` called during reward calculations), a delegator who signs a switch transaction based on the current commission has no on-chain guarantee that the commission will remain at that level when the transaction is included.

### Impact Explanation
A malicious staker can extract yield that would otherwise flow to the delegator. After the switch executes at the inflated commission, every future reward epoch diverts a larger fraction of pool rewards to the staker. The delegator cannot undo the switch once `switch_delegation_pool` has been executed. This constitutes ongoing theft of the delegator's unclaimed yield for as long as they remain in the pool.

### Likelihood Explanation
The attack requires the destination staker to hold an active `commission_commitment` with `max_commission` materially above the current commission. `set_commission_commitment` is a public, permissionless function callable by any staker who has a pool. [5](#0-4) 

On Starknet the sequencer orders transactions, so a staker monitoring the mempool can reliably insert a `set_commission` call ahead of the delegator's `switch_delegation_pool` call. No privileged access, leaked key, or external dependency is required.

### Recommendation
Add a `max_commission: Commission` parameter to `switch_delegation_pool`. Before forwarding the call to the staking contract, assert that the destination pool's current commission does not exceed the caller's stated maximum:

```rust
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
    max_commission: Commission,   // <-- new
) -> Amount {
    ...
    let actual_commission = self.get_commission_from_staking_contract_for(to_staker);
    assert!(actual_commission <= max_commission, Error::COMMISSION_EXCEEDS_MAX);
    ...
}
```

An optional `deadline` (timestamp or epoch) parameter should also be considered to prevent the transaction from being held and executed under significantly different conditions.

### Proof of Concept
1. Staker B deploys a pool with commission 5 % (500 / 10 000).
2. Staker B calls `set_commission_commitment(max_commission: 5000, expiration_epoch: E+100)`, publicly committing to a ceiling of 50 %.
3. Delegator D is in pool A and decides to switch to pool B, attracted by the 5 % commission. D calls `exit_delegation_pool_intent` on pool A and then submits `switch_delegation_pool(to_staker: B, to_pool: pool_B, amount: X)`.
4. Staker B observes D's pending transaction and front-runs it with `set_commission(commission: 5000)` — valid because the commitment is active and `5000 <= max_commission`. [6](#0-5) 
5. D's `switch_delegation_pool` executes. D is now a member of pool B at 50 % commission. [7](#0-6) 
6. Every subsequent reward epoch, 50 % of pool B's rewards flow to staker B instead of the 5 % D expected. D cannot exit immediately (exit wait window applies) and has already lost the ability to stay in pool A. The yield difference — 45 % of pool rewards attributable to D's stake — is continuously diverted to staker B.

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }

        fn switch_staking_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            switched_amount: Amount,
            data: Span<felt252>,
            identifier: felt252,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            if switched_amount.is_zero() {
                return;
            }
            let from_pool = get_caller_address();
            let undelegate_intent_key = UndelegateIntentKey {
                pool_contract: from_pool, identifier,
            };
            let mut undelegate_intent_value = self.get_pool_exit_intent(:undelegate_intent_key);
            assert!(
                undelegate_intent_value.is_non_zero(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            // Extract the token address of the `from_pool` contract.
            let token_address = get_undelegate_intent_token(
                undelegate_intent: undelegate_intent_value,
            );
            let old_intent_amount = undelegate_intent_value.amount;
            assert!(to_pool != from_pool, "{}", Error::SELF_SWITCH_NOT_ALLOWED);
            let decimals = self.get_token_decimals(:token_address);
            let normalized_switched_amount = NormalizedAmountTrait::from_native_amount(
                amount: switched_amount, :decimals,
            );
            assert!(
                normalized_switched_amount <= old_intent_amount,
                "{}",
                GenericError::AMOUNT_TOO_HIGH,
            );

            let to_staker_info = self.internal_staker_info(staker_address: to_staker);

            // More asserts.
            assert!(to_staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let to_token_address = self
                .staker_pool_info
                .entry(to_staker)
                .get_pool_token(pool_contract: to_pool)
                .expect_with_err(Error::DELEGATION_POOL_MISMATCH);
