Looking at the `update_rewards` function in `src/staking/staking.cairo` and comparing it to `update_rewards_from_attestation_contract`, I can identify a direct analog to the FloodPlain incomplete-check vulnerability.

### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Freeze Block Rewards - (File: `src/staking/staking.cairo`)

### Summary
The `IStakingRewardsManager::update_rewards` function has no caller access control despite the specification requiring "Only starkware sequencer." A parallel function, `IStakingAttestation::update_rewards_from_attestation_contract`, correctly enforces `CALLER_IS_NOT_ATTESTATION_CONTRACT`. Because `update_rewards` unconditionally writes `last_reward_block` to the current block number even when `disable_rewards: true`, any unprivileged caller can consume the per-block reward slot before the sequencer, permanently destroying the rewards for that block.

### Finding Description

The staking contract exposes two reward-update entry points:

1. `update_rewards_from_attestation_contract` — restricted to the attestation contract via an explicit caller check.
2. `update_rewards` — **no caller check at all** in the implementation, despite the spec mandating "Only starkware sequencer." [1](#0-0) 

The function writes `last_reward_block` to the current block number **before** the `disable_rewards` branch:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

The guard at the top of the function prevents a second call in the same block:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

`last_reward_block` is a single global storage slot, so one call for any staker blocks all reward distribution for that block.

The spec explicitly states the access control requirement:

> **access control:** Only starkware sequencer. [4](#0-3) 

By contrast, `update_rewards_from_attestation_contract` enforces its caller restriction and the test suite confirms it: [5](#0-4) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` in every block before the sequencer. Each such call:
- Passes all validation (staker exists, non-zero balance, new block)
- Writes `last_reward_block = current_block`
- Returns immediately without distributing rewards

The sequencer's subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. The rewards for that block are permanently lost — they are never credited to `unclaimed_rewards_own` and never transferred to pools. Repeated across every block, this permanently freezes all unclaimed yield for all stakers.

**Impact: High — Permanent freezing of unclaimed yield.**

### Likelihood Explanation

The function is publicly callable with no authentication. The attacker only needs to know any valid, active staker address (trivially obtained from on-chain events). The attack requires no capital, no privileged role, and no complex setup. It can be automated to run every block.

**Likelihood: High.**

### Recommendation

Add a caller restriction to `update_rewards` matching the specification. The simplest fix mirrors the pattern already used elsewhere in the contract:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Add: assert caller is the authorized sequencer address
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

Alternatively, store the authorized sequencer address in contract storage (set at construction/configuration time) and enforce it here, consistent with how `staking_contract` is enforced in `RewardSupplier::claim_rewards`. [6](#0-5) 

### Proof of Concept

```cairo
// Any unprivileged address can call this:
let staking_rewards = IStakingRewardsManagerDispatcher { contract_address: staking_contract };

// Attacker front-runs the sequencer every block:
loop {
    // disable_rewards: true → last_reward_block is set, no rewards distributed
    staking_rewards.update_rewards(
        staker_address: any_valid_staker,
        disable_rewards: true,
    );
    // Sequencer's call now reverts with REWARDS_ALREADY_UPDATED
    // Rewards for this block are permanently lost
    advance_to_next_block();
}
```

The existing test `test_update_rewards_disable_rewards_consensus_rewards_flow_test` already demonstrates that calling `update_rewards` with `disable_rewards: true` produces zero rewards and that a second call in the same block panics with `REWARDS_ALREADY_UPDATED` — confirming the mechanism. The missing piece is that the first caller is never validated. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
