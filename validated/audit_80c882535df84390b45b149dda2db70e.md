### Title
Pool Rewards Permanently Frozen When Delegated Balance Is Below `min_delegation_for_rewards` — (`src/pool/pool.cairo`)

---

### Summary

When a delegation pool's total balance is below `min_delegation_for_rewards`, the staking contract still calculates non-zero pool rewards and **transfers STRK tokens to the pool contract**, but the pool contract's `compute_rewards_per_unit` silently returns zero, leaving the `cumulative_rewards_trace` unchanged. The transferred STRK is permanently stuck in the pool contract with no recovery path.

---

### Finding Description

The vulnerability involves a mismatch between two layers of the reward pipeline:

**Layer 1 — Staking contract (`staking.cairo`): rewards are calculated and sent unconditionally**

In `calculate_staker_pools_rewards`, pool rewards are computed proportionally to `pool_balance_curr_epoch` relative to the global `total_stake`. There is **no check** for whether `pool_balance_curr_epoch` is below `min_delegation_for_rewards`: [1](#0-0) 

If `pool_rewards.is_non_zero()`, the pool is added to `pool_rewards_array` and `update_pool_rewards` transfers STRK to the pool contract and calls `update_rewards_from_staking_contract`: [2](#0-1) 

**Layer 2 — Pool contract (`pool.cairo`): rewards are silently discarded**

Inside `update_rewards_from_staking_contract`, the cumulative trace is updated by adding the result of `compute_rewards_per_unit`: [3](#0-2) 

`compute_rewards_per_unit` returns **zero** when `total_stake < min_delegation_for_rewards`: [4](#0-3) 

The code itself acknowledges this in a comment: [5](#0-4) 

The result: STRK tokens are transferred into the pool contract, but the `cumulative_rewards_trace` is not advanced. Since `claim_rewards` computes payouts exclusively from the trace, those tokens are permanently unclaimable.

For STRK pools, `min_delegation_for_rewards = 10^18` (1 STRK): [6](#0-5) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

STRK reward tokens are transferred into the pool contract but the `cumulative_rewards_trace` is not updated. No pool member can ever claim them. There is no sweep or recovery function in the pool contract. The tokens are frozen at the pool contract address indefinitely.

---

### Likelihood Explanation

**Medium.** There is no minimum delegation amount enforced in `enter_delegation_pool` or `add_to_delegation_pool`. Any delegator who enters a pool with less than 1 STRK (or the BTC equivalent threshold) triggers this path. This is a realistic scenario: a single delegator with a small balance, or a pool whose last remaining delegator partially exits leaving a sub-threshold residual, will cause every subsequent reward distribution to silently lose funds. [7](#0-6) 

---

### Recommendation

In `calculate_staker_pools_rewards` (staking contract), skip pools whose `pool_balance_curr_epoch` is below the pool's `min_delegation_for_rewards` threshold before computing and forwarding rewards. Alternatively, expose a `min_delegation_for_rewards` getter on the pool contract and gate the reward transfer in `update_pool_rewards` on that value. This mirrors the fix suggested in the Blend report: skip distribution to zero/sub-threshold reserves rather than forwarding tokens that will be silently discarded. [8](#0-7) 

---

### Proof of Concept

1. Staker stakes and opens a STRK delegation pool.
2. Delegator enters the pool with `amount = 10^18 - 1` (just below `min_delegation_for_rewards`).
3. Staker attests (or `update_rewards` is called by the sequencer).
4. `calculate_staker_pools_rewards` computes a non-zero `pool_rewards` (proportional to the small balance vs. global stake).
5. `update_pool_rewards` transfers STRK to the pool contract and calls `update_rewards_from_staking_contract(rewards=R, pool_balance=10^18-1)`.
6. `compute_rewards_per_unit` returns 0 because `10^18 - 1 < min_delegation_for_rewards`.
7. `cumulative_rewards_trace` is updated with `last + 0`.
8. `claim_rewards` returns 0 for the delegator.
9. `R` STRK tokens are permanently stuck in the pool contract.

        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,

            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
