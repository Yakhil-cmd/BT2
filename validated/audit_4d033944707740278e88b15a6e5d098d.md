### Title
`staker_migration()` Does Not Push Staker to `stakers` Vec, Permanently Excluding Pre-V3 Stakers from Consensus Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

`staker_migration()` migrates a staker from V2 to V3 but never pushes the staker's address into the `stakers` Vec. The `stakers` Vec is the sole source iterated by `get_stakers()`, which the consensus layer uses to determine which stakers participate in block production and receive consensus rewards. Any staker who staked before V3 and is migrated via `staker_migration()` is permanently invisible to the consensus layer and receives no consensus rewards.

---

### Finding Description

The `stakers` Vec is a new storage field introduced in V3: [1](#0-0) 

When a new staker calls `stake()` in V3, their address is pushed into this Vec: [2](#0-1) 

`get_stakers()` — the function called by the consensus layer to enumerate all active stakers — iterates exclusively over this Vec: [3](#0-2) 

However, `staker_migration()`, which migrates pre-V3 stakers to V3, only upgrades pool contracts and writes the new `staker_version`. It never calls `self.stakers.push(staker_address)`: [4](#0-3) 

The `StakingEIC` that runs during the V2→V3 upgrade also does not populate `stakers` with existing stakers: [5](#0-4) 

The result: every staker who staked before V3 is absent from `stakers` after `staker_migration()` completes, and `get_stakers()` will never return them.

---

### Impact Explanation

`get_stakers()` is the consensus layer's authoritative list of stakers eligible for block rewards. A staker absent from `stakers` is never included in the committee, so `update_rewards()` is never triggered for them by the consensus layer. This is a **permanent, irrecoverable loss of all consensus-era unclaimed yield** for every pre-V3 staker — matching the allowed impact: **High: Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- `staker_migration()` is permissionless — any address can call it for any staker (per spec: "Any address can execute").
- Every staker who existed before V3 is affected; this is the entire pre-upgrade staker population.
- There is no recovery path: once migrated, there is no function that adds a staker to `stakers` except `stake()`, which requires the staker to not already exist.
- The test `MultipleStakersMigrationVecFlow` does not catch this because `set_staker_for_migration` in the test harness operates on a V3 deployment where `stake()` already pushed the staker into `stakers` — it does not simulate the real upgrade scenario where `stakers` was empty before V3. [6](#0-5) 

---

### Recommendation

Add `self.stakers.push(staker_address)` inside `staker_migration()`, guarded to avoid double-insertion (e.g., only push if the staker is not already present, or rely on the version check that already prevents double migration):

```cairo
fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
    // ... existing checks ...

    // Upgrade pools.
    // ... existing pool upgrade logic ...

    // Add staker to the stakers vector (was not present for pre-V3 stakers).
+   self.stakers.push(staker_address);

    // Mark the staker's version.
    self.staker_version.write(staker_address, LATEST_STAKER_VERSION);
}
```

---

### Proof of Concept

1. Deploy the V2 staking contract. Staker A calls `stake()`. Staker A is **not** in `stakers` (Vec did not exist in V2).
2. Upgrade to V3 via `StakingEIC`. `stakers` Vec is initialized empty; no existing stakers are added.
3. Anyone calls `staker_migration(staker_A)`. The function upgrades pools and sets `staker_version = V3`. It does **not** push `staker_A` into `stakers`.
4. Consensus layer calls `get_stakers(epoch_id)`. The function iterates `self.stakers` — Staker A is absent. Staker A receives zero consensus rewards for all future epochs despite having active stake.
5. Staker A cannot remedy this: `stake()` reverts with `STAKER_EXISTS`, and no other public function pushes to `stakers`. [4](#0-3) [7](#0-6)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }

        fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
            // Assert the staker exists.
            self._internal_staker_info(:staker_address);

            // Assert staker already migrated to V2.
            assert!(
                !self.staker_own_balance_trace.entry(staker_address).is_empty(),
                "{}",
                Error::STAKER_NOT_MIGRATED,
            );

            // Assert the staker is not migrated to V3 yet.
            assert!(
                !self.staker_version.read(staker_address).is_latest(),
                "{}",
                Error::STAKER_ALREADY_MIGRATED,
            );

            // Prepare the implementation data.
            let pool_class_hash = self.pool_contract_class_hash.read();
            let pool_eic_class_hash = self.pool_eic_class_hash.read();
            // Sanity checks.
            assert!(pool_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            assert!(pool_eic_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            let eic_data = EICData { eic_hash: pool_eic_class_hash, eic_init_data: [].span() };
            let implementation_data = ImplementationData {
                impl_hash: pool_class_hash, eic_data: Option::Some(eic_data), final: false,
            };

            // Upgrade pools.
            let pools = self.staker_pool_info.entry(staker_address).pools;
            for pool_contract_ptr in pools.keys_iter() {
                let pool_contract = pool_contract_ptr.read();
                let pool_replaceable_dispatcher = IReplaceableDispatcher {
                    contract_address: pool_contract,
                };
                pool_replaceable_dispatcher.add_new_implementation(:implementation_data);
                pool_replaceable_dispatcher.replace_to(:implementation_data);
            }

            // Mark the staker's version.
            self.staker_version.write(staker_address, LATEST_STAKER_VERSION);
        }

pub(crate) impl MultipleStakersMigrationVecFlowImpl of FlowTrait<MultipleStakersMigrationVecFlow> {
    fn setup_v1(ref self: MultipleStakersMigrationVecFlow, ref system: SystemState) {
        let amount = system.staking.get_min_stake();
        let staker1 = system.new_staker(:amount);
        let staker2 = system.new_staker(:amount);
        let commission = 200;
        system.stake(staker: staker1, :amount, pool_enabled: false, :commission);
        system.stake(staker: staker2, :amount, pool_enabled: false, :commission);

        system.set_staker_for_migration(staker_address: staker1.staker.address);
        system.set_staker_for_migration(staker_address: staker2.staker.address);
        self.old_stakers = Option::Some((staker1, staker2));
    }

    fn test(self: MultipleStakersMigrationVecFlow, ref system: SystemState) {
        let old_stakers = self.old_stakers.unwrap();
        let (staker1, staker2) = old_stakers;

        let actual_stakers = system.staking.get_stakers();
        assert!(actual_stakers == array![staker1.staker.address, staker2.staker.address].span());

        let amount = system.staking.get_min_stake();
        let staker3 = system.new_staker(:amount);
        let commission = 200;
        system.stake(staker: staker3, :amount, pool_enabled: false, :commission);

        let actual_stakers = system.staking.get_stakers();
        assert!(
            actual_stakers == array![
                staker1.staker.address, staker2.staker.address, staker3.staker.address,
            ]
                .span(),
        );
    }
