### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Unchecked `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6`/`addFeeSellE6` directly to the pool with no validation against `maxAdminSpreadFeeE6`. The parallel function `setPoolAdminFees` enforces the cap; `setPoolBinAdditionalFees` does not. A pool admin can therefore set per-bin additional fees to any value up to `uint16.max` (65 535 E6 ≈ 6.55%) regardless of what `maxAdminSpreadFeeE6` is configured to, violating the factory's explicit admin-fee ceiling.

---

### Finding Description

`setPoolAdminFees` enforces the cap:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees` has no such check — it passes the caller-supplied values straight through:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

`MetricOmmPool.setBinAdditionalFees` also performs no cap check — it only validates the bin index:

```solidity
// MetricOmmPool.sol:464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    ...
}
``` [3](#0-2) 

During a swap, the per-bin fee is added directly on top of the oracle-derived base fee:

```solidity
// MetricOmmPool.sol:910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [4](#0-3) 

This means the per-bin fee is a real, swap-time cost paid by traders, not a view-only value.

---

### Impact Explanation

The contest Q&A explicitly states:

> "Pool Admin — semi-trusted, bounded … **Cannot exceed caps or bypass timelocks.** If they can exceed caps or bypass timelocks and it leads to Medium or higher severity issue, then it can be valid." [5](#0-4) 

When `maxAdminSpreadFeeE6` is set to a value below 65 535 (e.g., 5 000 = 0.5%), the pool admin can call `setPoolBinAdditionalFees(pool, bin, 65535, 65535)` and impose ≈6.55% additional spread per bin — 13× the factory-enforced ceiling. Traders pay this excess on every swap through the affected bin, constituting a direct loss of user principal above Sherlock Medium thresholds.

---

### Likelihood Explanation

The path requires only the pool admin role (semi-trusted, not fully trusted) and a single transaction with no timelock. Any pool whose `maxAdminSpreadFeeE6` is set below `uint16.max` is affected. The factory constructor initialises `maxAdminSpreadFeeE6 = HARD_MAX_SPREAD_FEE_E6` (20%), but the owner can lower it via `setFeeCaps`; once lowered, the bypass is immediately exploitable. [6](#0-5) 

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
```

Alternatively, define a dedicated `maxAdminBinAdditionalFeeE6` cap if per-bin fees are intentionally allowed to differ from the global spread cap.

---

### Proof of Concept

1. Deploy factory; owner calls `setFeeCaps(..., newMaxAdminSpreadFeeE6 = 5_000, ...)` (0.5% cap).
2. Create a pool with `adminSpreadFeeE6 = 5_000` (at the cap).
3. Pool admin calls `factory.setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)`.
4. No revert occurs; `BinState.addFeeBuyE6 == 65_535` (6.55%) is stored.
5. A swap through bin 0 pays `baseFeeX64 + 6.55%` spread — 13× the factory-enforced 0.5% admin ceiling — confirming the bypass. [2](#0-1) [3](#0-2)
