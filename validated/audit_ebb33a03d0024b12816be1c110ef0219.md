### Title
Missing Zero-Address Check in `change_reward_address` and `stake` Allows Permanent Freezing of Unclaimed Yield and Principal — (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

Neither `stake()` nor `change_reward_address()` in `staking.cairo`, nor `enter_delegation_pool()` nor `change_reward_address()` in `pool.cairo`, validate that the supplied `reward_address` is non-zero. If a staker or pool member sets their `reward_address` to the zero address, every subsequent call to `claim_rewards` and `unstake_action` will revert because the underlying ERC-20 `checked_transfer` to address zero panics. This permanently freezes both unclaimed yield and staked principal for any staker whose `reward_address` is zero at the time `unstake_action` is executed.

---

### Finding Description

**Root cause — `staking.cairo` `change_reward_address`:**

`change_reward_address` performs only one validation: the new address must not be a registered token address. There is no check that `reward_address.is_non_zero()`. [1](#0-0) 

**Root cause — `staking.cairo` `stake`:**

The initial `stake()` entry-point applies the same single guard (`does_token_exist`) and writes the caller-supplied `reward_address` directly into storage without a zero-address check. [2](#0-1) 

**Root cause — `pool.cairo` `change_reward_address` and `enter_delegation_pool`:**

The pool-side functions mirror the same pattern: only a token-address check, no zero-address guard. [3](#0-2) [4](#0-3) 

**Propagation — `send_rewards_to_staker` called inside `unstake_action`:**

`unstake_action` unconditionally calls `send_rewards_to_staker`, which calls `checked_transfer(recipient: reward_address, ...)`. If `reward_address` is zero and `unclaimed_rewards_own > 0`, the ERC-20 transfer panics, causing `unstake_action` to revert permanently. [5](#0-4) [6](#0-5) 

**Propagation — `claim_rewards` in `pool.cairo`:**

Pool-side `claim_rewards` transfers directly to `pool_member_info.reward_address`; if that is zero, the transfer panics and all accrued pool rewards are frozen. [7](#0-6) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield; Temporary freezing of funds.**

1. **Unclaimed yield frozen**: `claim_rewards` reverts on every call, so accrued STRK rewards can never be withdrawn.
2. **Principal frozen via `unstake_action`**: Because `unstake_action` calls `send_rewards_to_staker` before returning the principal, a non-zero reward balance with a zero `reward_address` causes the entire exit flow to revert. The staker's principal remains locked in the contract.

The staker *can* recover by calling `change_reward_address` again with a valid address (no `unstake_time` guard prevents this), making the principal freeze technically temporary. However, if the staker account is a contract without that capability, or the key is lost, the freeze becomes permanent. Unclaimed yield is permanently frozen in all cases where the staker cannot or does not correct the address before `unstake_action` is triggered by a third party (the function is callable by any address). [8](#0-7) 

---

### Likelihood Explanation

**Low-to-Medium.** The zero address is a valid `ContractAddress` value in Cairo and is the default/uninitialized value. A staker or delegator could pass it accidentally (e.g., a deployment script bug, an uninitialized variable, or a front-end that submits a blank field). The `change_reward_address` function is callable at any time by the staker, including after `unstake_intent`, so a deliberate self-grief or accidental misconfiguration is realistic. No privileged role is required.

---

### Recommendation

Add a zero-address assertion in all four entry points that accept or update `reward_address`:

- `stake()` in `src/staking/staking.cairo`
- `change_reward_address()` in `src/staking/staking.cairo`
- `enter_delegation_pool()` in `src/pool/pool.cairo`
- `change_reward_address()` in `src/pool/pool.cairo`

```cairo
assert!(reward_address.is_non_zero(), "Reward address cannot be zero");
```

Additionally, consider adding a defensive guard in `send_rewards_to_staker` and pool `claim_rewards` to skip the transfer (or assert) when `amount` is zero, which would prevent a zero-amount transfer to zero address from causing an unexpected revert in edge cases.

---

### Proof of Concept

1. Staker calls `stake(reward_address: 0, operational_address: valid_addr, amount: min_stake)`. The call succeeds — no zero-address check exists. [9](#0-8) 

2. After epochs pass, the staker accrues `unclaimed_rewards_own > 0`.

3. Staker calls `unstake_intent()`. Succeeds. [10](#0-9) 

4. After the exit window, any address calls `unstake_action(staker_address)`.

5. `unstake_action` calls `send_rewards_to_staker` (line 495), which executes:
   ```
   token_dispatcher.checked_transfer(recipient: 0, amount: rewards)
   ```
   The ERC-20 panics on transfer to zero address. `unstake_action` reverts. [11](#0-10) 

6. The staker's principal and all accrued rewards remain locked in the staking contract. The staker must call `change_reward_address` with a valid address to unblock the exit — but if the staker account is a non-upgradeable contract or the key is lost, the funds are permanently frozen.

        fn unstake_intent(ref self: ContractState) -> Timestamp {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            // Set the unstake time.
            let unstake_time = Time::now().add(delta: self.exit_wait_window.read());
            staker_info.unstake_time = Option::Some(unstake_time);
            self.write_staker_info(:staker_address, :staker_info);

            // Write the unstake intent epoch.
            self.staker_unstake_intent_epoch.write(staker_address, self.get_epoch_plus_k());

            // Write off the delegated stake from the total stake.
            for (pool_contract, token_address) in self
                .staker_pool_info
                .entry(staker_address)
                .pools {
                let amount = self.get_delegated_balance(:staker_address, :pool_contract);
                self.remove_from_total_stake(:token_address, :amount);
                let decimals = self.get_token_decimals(:token_address);
                self
                    .emit(
                        Events::StakeDelegatedBalanceChanged {
                            staker_address,
                            token_address,
                            old_delegated_stake: amount.to_native_amount(:decimals),
                            new_delegated_stake: Zero::zero(),
                        },
                    );
            }
            // Write off the self stake from the total stake.
            let old_self_stake = self.get_own_balance(:staker_address);
            self.remove_from_total_stake(token_address: STRK_TOKEN_ADDRESS, amount: old_self_stake);

            // Emit events.
            self.emit(Events::StakerExitIntent { staker_address, exit_timestamp: unstake_time });
            self
                .emit(
                    Events::StakeOwnBalanceChanged {
                        staker_address,
                        old_self_stake: old_self_stake.to_strk_native_amount(),
                        new_self_stake: Zero::zero(),
                    },
                );
            unstake_time
