### Title
Signature Validation Bypass via `skip_stateful_validations` Allows Admission of Unsigned Invoke Transactions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's `skip_stateful_validations` function skips the `__validate__` entry-point call for any invoke transaction whose nonce is `1` and whose sender's on-chain nonce is `0`, provided `account_tx_in_pool_or_recent_block` returns `true` for that sender. Because `account_tx_in_pool_or_recent_block` returns `true` for **any** address that has **any** transaction in the mempool pool — not only the legitimate owner's deploy-account — an unprivileged attacker can submit an invoke transaction carrying an arbitrary or invalid signature for a victim address that has a pending deploy-account, and the gateway will admit it to the mempool without ever calling `__validate__`.

**The overly-broad membership check**

`account_tx_in_pool_or_recent_block` returns `true` if the address has **any** transaction in the pool, regardless of who submitted it:

```
crates/apollo_mempool/src/mempool.rs  lines 697-700
```

```rust
pub fn account_tx_in_pool_or_recent_block(&self, account_address: ContractAddress) -> bool {
    self.state.contains_account(account_address)
        || self.tx_pool.contains_account(account_address)
}
```

**Attack path**

1. Alice submits a `DeployAccount` for address `A`. The mempool now contains Alice's deploy-account, so `tx_pool.contains_account(A)` is `true`.
2. Attacker observes `A` in the mempool (P2P gossip propagates it) and submits an `Invoke` with `sender_address = A`, `nonce = 1`, and a completely fabricated signature.
3. Gateway stateless validation passes (signature length is within bounds, resource bounds are non-zero, DA modes are L1).
4. `convert_rpc_tx_to_internal` computes the tx hash and stores the transaction; no signature cryptographic check occurs here.
5. `extract_state_nonce_and_run_validations` reads `account_nonce = 0` from state, calls `validate_state_preconditions` (nonce `1` is within `[0, max_gap]`, so it passes), then calls `skip_stateful_validations` → `account_tx_in_pool_or_recent_block(A)` returns `true` → returns `true` (skip).
6. `run_validate_entry_point` is called with `validate = false` → `__validate__` is **never invoked** → the transaction is admitted to the mempool with an invalid signature.

**Execution-time consequence**

When the batcher later executes the block, it creates a fresh `AccountTransaction` with `validate = true`. After Alice's deploy-account executes (giving `A` nonce `1`), the attacker's invoke is the next transaction at nonce `1`. The blockifier calls `__validate__` on it; the invalid signature causes it to fail. The transaction is reverted and the execution fee is charged to Alice's account. If the attacker submitted the invoke with a higher tip than Alice's own nonce-1 invoke, Alice's legitimate transaction is displaced from the mempool (fee-escalation replaces it), and Alice's account is charged for the attacker's failed transaction.

---

### Impact Explanation

**Broken invariant**: Every transaction admitted to the mempool must have passed account-contract signature verification (`__validate__`). The `skip_stateful_validations` path violates this invariant for any address whose deploy-account is visible in the mempool.

**Concrete wrong outcomes**:
- **Mempool admission of invalid transactions** (High impact per scope): an invoke with an arbitrary signature is accepted.
- **Fee drain / DoS**: the attacker's invalid invoke executes at nonce `1` after deploy-account, fails `__validate__`, and the fee is charged to the victim's account.
- **Displacement of legitimate invoke**: if the attacker bids a higher tip, the victim's own nonce-1 invoke is evicted from the mempool via fee escalation and never executed.

---

### Likelihood Explanation

- The mempool propagates transactions over P2P gossip; any node can observe pending deploy-account transactions.
- The attacker needs no privileged access: submitting an RPC transaction is a public operation.
- The condition (`nonce == 1`, `account_nonce == 0`, address in pool) is reliably satisfied for every new account during the deploy-account + invoke UX window.
- The attack requires only one well-timed RPC call per victim address.

---

### Recommendation

Replace the coarse `account_tx_in_pool_or_recent_block` membership check with a check that verifies the **specific deploy-account transaction hash** belongs to the same sender. Concretely:

- The gateway should look up the pending deploy-account for the sender address from the mempool and verify that the invoke's sender address matches the contract address computed from that deploy-account's class hash, salt, and constructor calldata.
- Alternatively, require the caller to supply the deploy-account transaction hash (as `native_blockifier`'s `PyValidator` already does via `deploy_account_tx_hash`) and verify it matches a known pending deploy-account for the same address before skipping `__validate__`.

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
```
