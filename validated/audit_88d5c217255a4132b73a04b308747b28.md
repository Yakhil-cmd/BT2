### Title
Unchecked Zero `reward_address` Causes Permanent Loss of Unclaimed Yield - (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

### Summary
Neither `Staking::stake()`, `Staking::change_reward_address()`, `Pool::enter_delegation_pool()`, nor `Pool::change_reward_address()` validate that the supplied `reward_address` is non-zero. Because all reward-transfer paths unconditionally send tokens to the stored `reward_address`, setting it to the zero address permanently destroys any accrued yield.

### Finding Description
The `reward_address` parameter is the sole destination for all staker and pool-member reward transfers. Four public entry points accept this parameter without a zero-address guard:

**`Staking::stake()`** — checks only that `reward_address` is not a registered token address: [1](#0-0) 

**`Staking::change_reward_address()`** — same single check, no zero guard: [2](#0-1) 

**`Pool::enter_delegation_pool()`** — checks only against the token address: [3](#0-2) 

**`Pool::change_reward_address()`** — same single check: [4](#0-3) 

When rewards are eventually claimed, both transfer paths use the stored `reward_address` directly with no further validation:

`Staking::send_rewards_to_staker()`: [5](#0-4) 

`Pool::claim_rewards()`: [6](#0-5) 

The error catalogue already defines `ZERO_ADDRESS` as a recognised error string, confirming the protocol authors are aware of the pattern — but it is never applied to `reward_address` validation: [7](#0-6) 

### Impact Explanation
Any staker or pool member who registers or later sets `reward_address = 0` will have all future reward claims silently transferred to the zero address. Because the zero address has no known private key on Starknet, those tokens are irrecoverable. This constitutes **permanent freezing of unclaimed yield** (High). The principal amount is unaffected (it is returned to `staker_address`, not `reward_address`), but the entire accumulated reward stream is destroyed.

### Likelihood Explanation
The entry paths are fully unprivileged — any caller can invoke `stake()`, `change_reward_address()`, or `enter_delegation_pool()`. The most realistic trigger is accidental misconfiguration (e.g., a front-end submitting a default-zero address, or a script omitting the field). A malicious actor could also deliberately grief a victim by front-running a `change_reward_address` call if they control the staker key, though the primary risk is accidental self-harm. Given the absence of any guard and the existence of the `ZERO_ADDRESS` error constant that is never used here, likelihood is **medium**.

### Recommendation
Add an explicit non-zero assertion for `reward_address` in all four entry points, mirroring the existing `CALLER_IS_ZERO_ADDRESS` pattern already used elsewhere in the codebase:

```cairo
assert!(reward_address.is_non_zero(), "{}", Error::ZERO_ADDRESS);
```

Apply this check in:
- `Staking::stake()` — before writing `staker_info`
- `Staking::change_reward_address()` — before updating storage
- `Pool::enter_delegation_pool()` — alongside the existing token-address check
- `Pool::change_reward_address()` — alongside the existing token-address check

### Proof of Concept

1. Staker calls `Staking::stake(reward_address: 0, operational_address: X, amount: MIN_STAKE)`. No revert occurs. [8](#0-7) 

2. Rewards accrue over epochs via the attestation/consensus path and are stored in `staker_info.unclaimed_rewards_own`.

3. Staker (or anyone) calls `Staking::claim_rewards(staker_address)`. The authorization check passes because `caller == staker_address`. [9](#0-8) 

4. `send_rewards_to_staker` executes `checked_transfer(recipient: 0, amount: rewards)`, sending all yield to the zero address permanently. [10](#0-9) 

The identical path exists for pool members via `Pool::enter_delegation_pool(reward_address: 0, ...)` followed by `Pool::claim_rewards()`. [11](#0-10) [6](#0-5)

        fn claim_rewards(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let caller_address = get_caller_address();
            let reward_address = staker_info.reward_address;
            assert!(
                caller_address == staker_address || caller_address == reward_address,
                "{}",
                Error::CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            // Transfer rewards to staker's reward address and write updated staker info to storage.
            // Note: `send_rewards_to_staker` alters `staker_info` thus commit to storage is
            // performed only after that.
            let amount = staker_info.unclaimed_rewards_own;
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            self.write_staker_info(:staker_address, :staker_info);
            amount
        }
