### Title
Dynamic Blocklist Bypass via Nil Identity — Blocked Address Transactions Execute in FinalizeBlock — (`app/block_address.go`, `app/proposal.go`)

### Summary

`BlockAddressesDecorator.AnteHandle` wraps **all** blocklist enforcement in `if ctx.IsCheckTx()`. During `FinalizeBlock`, `IsCheckTx()` returns `false`, so the ante handler performs zero checks. The dynamic on-chain blocklist is only enforced at the proposal layer (`ValidateTransaction` / `ProcessProposalHandler`), but both fast-path to "accept all" when `h.blocklist` is empty. `h.blocklist` is never populated when `Identity == nil` because `SetBlockList` returns early. The result: a blocked address whose address is absent from the static config blacklist can have its transaction finalized on any validator node that has no `age` identity configured.

### Finding Description

**Root cause 1 — Ante handler skips all checks outside CheckTx:** [1](#0-0) 

The entire body of `AnteHandle` — static blacklist, destination address, EIP-7702 auth list, and `MsgStoreBlockList` admin check — is gated on `ctx.IsCheckTx()` (line 33). During `FinalizeBlock` the condition is `false`, so `next(ctx, tx, simulate)` is called unconditionally.

**Root cause 2 — SetBlockList is a no-op when Identity is nil:** [2](#0-1) 

If the validator node has no `age` identity configured, `SetBlockList` returns `nil` immediately, leaving `h.blocklist` as the empty map initialised in `NewProposalHandler`.

**Root cause 3 — ValidateTransaction and ProcessProposalHandler fast-path on empty blocklist:** [3](#0-2) [4](#0-3) 

Both `ValidateTransaction` and `ProcessProposalHandler` return "accept" immediately when `len(h.blocklist) == 0`. A proposer with `Identity == nil` therefore includes the blocked-address tx in its proposal, and any other validator that also has an empty blocklist (also `Identity == nil`, or blocklist not yet refreshed) will accept the block via `ProcessProposalHandler`.

### Impact Explanation

Once the block is accepted by ≥ 2/3 of validators (all of whom fast-path through `ProcessProposalHandler` when their own blocklist is empty), `FinalizeBlock` runs the ante handler with `IsCheckTx() == false`, which skips every check and executes the transaction. The blocked address successfully transacts on-chain, bypassing the entire compliance/sanctions enforcement mechanism. This is a direct bypass of the block-list authorization check.

### Likelihood Explanation

The `ProposalHandler` comment explicitly states `"Identity is nil if it's not a validator node"`. [5](#0-4) 

Any validator that is not configured with an `age` identity — including newly onboarded validators, validators that have not yet set up the e2ee key, or any node running in a configuration where the identity is intentionally omitted — will silently have an empty dynamic blocklist. If such a node is the block proposer, and a sufficient fraction of the validator set also has an empty blocklist, the blocked address's transaction is finalized.

### Recommendation

1. **Remove the `ctx.IsCheckTx()` gate in `BlockAddressesDecorator.AnteHandle`.** The static blacklist check must run unconditionally during both `CheckTx` and `FinalizeBlock`. The current guard means the ante handler provides zero enforcement at execution time.
2. **Enforce the dynamic blocklist during `FinalizeBlock` as well.** Either pass the dynamic blocklist into the ante handler (reading it from on-chain state via `getParams` or a keeper), or add a separate `FinalizeBlock` hook that rejects blocked-address transactions before they execute.
3. **Do not fast-path `ProcessProposalHandler` to ACCEPT when the blocklist is empty.** A validator that cannot decrypt the blocklist should either reject the proposal or abstain, not silently accept it.

### Proof of Concept

1. Start a single-validator devnet with `Identity = nil` (the default for a node without e2ee configured).
2. Submit a `MsgStoreBlockList` from the admin address containing an encrypted blob that lists address `A`.
3. Submit any transaction signed by address `A`.
4. Observe: `CheckTx` passes (A is not in the static config blacklist; `IsCheckTx()` is `true` but `bad.blockedMap` does not contain A).
5. `PrepareProposal` calls `ValidateTransaction` → `len(h.blocklist) == 0` → fast-path accept.
6. `ProcessProposal` → same fast-path accept.
7. `FinalizeBlock` runs `AnteHandle` with `IsCheckTx() == false` → entire body skipped → tx executes.
8. Assert the transaction appears in the committed block and its state changes are applied.

func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)
