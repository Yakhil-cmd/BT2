### Title
Unprivileged Front-Run of `update_rewards` via `disable_rewards: true` Permanently Freezes Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `StakingRewardsManagerImpl` enforces a single reward update per block using the global `last_reward_block` state variable. Because the function is callable by any non-zero address and updates `last_reward_block` **before** checking the `disable_rewards` flag, an attacker can front-run a legitimate `update_rewards(staker, false)` call with `update_rewards(staker, true)`, consuming the block's reward slot without distributing any rewards. The legitimate call then reverts with `REWARDS_ALREADY_UPDATED`. Repeated across every block, this permanently freezes all stakers' unclaimed block rewards.

---

### Finding Description

`update_rewards` is a public function with no caller-identity access control — `general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function first checks the monotonic guard: [2](#0-1) 

Then, after validating the staker exists and has non-zero balance, it **unconditionally writes** `last_reward_block` before branching on `disable_rewards`: [3](#0-2) 

`last_reward_block` is a single global storage slot shared across all stakers: [4](#0-3) 

**Attack path:**

1. In block N, the legitimate staker (or their keeper) submits `update_rewards(staker_A, disable_rewards: false)`.
2. The attacker front-runs with `update_rewards(staker_A, disable_rewards: true)`.
   - Passes the `current_block_number > last_reward_block` check.
   - Passes the staker-active and non-zero-balance checks.
   - Writes `last_reward_block = N`.
   - Hits `if disable_rewards { return; }` — **no rewards distributed**.
3. The legitimate call now fails: `current_block_number (N) > last_reward_block (N)` is false → `REWARDS_ALREADY_UPDATED`.
4. Staker A receives zero block rewards for block N.
5. Repeated every block, the staker's yield is permanently frozen.

Because `last_reward_block` is global, a single attacker call with **any** valid staker address blocks **all** stakers from receiving rewards in that block.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Block rewards for every staker are permanently lost for any block in which the attacker front-runs. Since the attacker can repeat this every block at low cost (Starknet gas is cheap), the effective result is a complete, indefinite denial of consensus block rewards to all stakers. The lost rewards are never recoverable because `last_reward_block` is monotonically increasing and the missed block slot cannot be revisited.

---

### Likelihood Explanation

**Medium.** The attacker needs only one valid, active staker address (publicly readable on-chain) and the ability to submit a transaction before the legitimate one in the same block. On Starknet, the sequencer orders transactions; an attacker who submits with higher priority fees or who is the sequencer can reliably execute this. No privileged access, leaked keys, or external dependencies are required.

---

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` branch, so that consuming the block slot is only possible when rewards are actually being distributed:

```cairo
// Update last block rewards ONLY when rewards are being distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}

self.last_reward_block.write(current_block_number); // moved here

let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
...
```

Alternatively, restrict `update_rewards` so that `disable_rewards: true` can only be passed by the staker themselves or a designated operator, preventing an unprivileged caller from consuming the block slot without distributing rewards.

---

### Proof of Concept

```
Given:
  staker_A is active with non-zero STRK balance
  last_reward_block = N-1
  current_block = N

Step 1 (attacker, block N):
  call update_rewards(staker_A, disable_rewards: true)
  → current_block_number (N) > last_reward_block (N-1)  ✓
  → staker_A exists and is active                        ✓
  → staker_A has non-zero balance                        ✓
  → last_reward_block.write(N)
  → disable_rewards == true → return (no rewards sent)

Step 2 (legitimate staker, same block N):
  call update_rewards(staker_A, disable_rewards: false)
  → current_block_number (N) > last_reward_block (N)  ✗
  → PANIC: REWARDS_ALREADY_UPDATED

Result: staker_A receives 0 block rewards for block N.
Repeated every block → permanent yield freeze for all stakers.
```
