### Title
Missing Access Control on `update_rewards` Allows Any Caller to Consume the Global Block Reward Slot, Permanently Freezing Unclaimed Yield — (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` in the Staking contract is documented as callable only by the Starkware sequencer, but the code enforces no such restriction. Any address can call it. The function writes to a single global `last_reward_block` storage variable before checking `disable_rewards`. An attacker who calls `update_rewards(staker_address: any_valid_staker, disable_rewards: true)` first in a block consumes the block's reward slot without distributing any yield, permanently blocking the sequencer from distributing rewards for that block. Repeated every block, this permanently freezes unclaimed yield for all stakers.

---

### Finding Description

`update_rewards` at `src/staking/staking.cairo:1449` is gated only by `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

There is no role check, no sequencer address check, and no `only_sequencer` guard anywhere in `update_rewards`.

The function writes to the global `last_reward_block` **before** the `disable_rewards` branch:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

`last_reward_block` is a single global storage slot shared across all stakers:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [3](#0-2) 

The guard at the top of `update_rewards` enforces that only one call per block can succeed:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

Once any caller writes `last_reward_block = N`, every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`. The sequencer's legitimate call is blocked for that block.

The spec explicitly states the intended access control:

> **access control**: Only starkware sequencer. [5](#0-4) 

The code does not implement this restriction.

---

### Impact Explanation

An attacker calls `update_rewards(staker_address: any_valid_active_staker, disable_rewards: true)` in block N:

1. `last_reward_block` is written to N.
2. No rewards are distributed (early return due to `disable_rewards: true`).
3. The sequencer's call for block N reverts with `REWARDS_ALREADY_UPDATED`.
4. All stakers receive zero yield for block N.

Repeated every block, this permanently freezes unclaimed yield for every staker and every delegation pool in the protocol. This matches the allowed impact: **High — Permanent freezing of unclaimed yield or unclaimed royalties**.

Additionally, an attacker can call with `disable_rewards: false` for a staker the sequencer did not intend to reward in that block, redirecting block rewards to a chosen staker. This matches **High — Theft of unclaimed yield**.

---

### Likelihood Explanation

- **Entry path**: `update_rewards` is a public function with no role guard. Any non-zero address can call it with any valid active staker address. Staker addresses are public (emitted in `NewStaker` events).
- **Precondition**: The attacker's transaction must land in the block before the sequencer's `update_rewards` call. On Starknet the sequencer controls ordering, but the code places no enforcement on this; if the sequencer ever includes `update_rewards` as a non-system transaction, or if the sequencer skips a block, the slot is open to any caller.
- **Repeatability**: The attack is trivially repeatable every block at negligible cost.
- **No privileged access required**: The attacker is an ordinary address with no staked funds.

---

### Recommendation

Add an access control check to `update_rewards` that restricts callers to the authorized sequencer address. For example, store the sequencer address at construction time and assert:

```cairo
assert!(
    get_caller_address() == self.sequencer_address.read(),
    "{}",
    Error::CALLER_IS_NOT_SEQUENCER,
);
```

Alternatively, use an existing role (e.g., a new `SEQUENCER_ROLE`) and call `self.roles.only_sequencer()` at the top of `update_rewards`, consistent with how other privileged functions are guarded in the contract.

---

### Proof of Concept

1. Deploy the staking system. Have staker `S` stake and advance `K` epochs so `S` is active with non-zero balance.
2. Start consensus rewards (`set_consensus_rewards_first_epoch`).
3. In block `N`, from **any** address `A` (not the sequencer), call:
   ```
   update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Observe: `last_reward_block` is now `N`; no rewards were distributed; `S.unclaimed_rewards_own` is unchanged.
5. The sequencer now attempts `update_rewards(staker_address: S, disable_rewards: false)` in block `N` — it reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeat step 3 in every subsequent block. Staker `S` (and any other staker, since the slot is global) permanently accumulates zero yield. [6](#0-5)

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;

            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
