### Title
Permissionless `exit_delegation_pool_action` Can Front-Run a Pool Member's `switch_delegation_pool` - (File: src/pool/pool.cairo)

### Summary

`exit_delegation_pool_action` carries no caller restriction — any address may execute it for any pool member once the exit window has elapsed. `switch_delegation_pool`, by contrast, requires the pool member to be the caller and requires `unpool_time` to be set. After the exit window passes, a third party can race the pool member's switch transaction, forcibly completing the exit and clearing the intent state, causing the switch to revert with `MISSING_UNDELEGATE_INTENT`.

### Finding Description

`exit_delegation_pool_action` in `pool.cairo` accepts an arbitrary `pool_member` address and has no `get_caller_address()` guard:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    ...
    pool_member_info.unpool_amount = Zero::zero();
    pool_member_info.unpool_time = Option::None;   // ← clears the intent
    ...
    token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
``` [1](#0-0) 

The spec explicitly documents this as intentional: *"Any address can execute."* [2](#0-1) 

`switch_delegation_pool`, however, requires `unpool_time.is_some()` as a hard precondition:

```cairo
assert!(
    pool_member_info.unpool_time.is_some(),
    "{}",
    GenericError::MISSING_UNDELEGATE_INTENT,
);
``` [3](#0-2) 

Critically, `switch_delegation_pool` does **not** require the exit window to have elapsed — it can be called at any time after the intent is set. The staking spec confirms: *"Note: It does not matter if the unpool_time has passed or not."* [4](#0-3) 

This creates an asymmetric race window: once `Time::now() >= unpool_time`, the exit action becomes executable by anyone, but the pool member may still intend to switch rather than exit.

### Impact Explanation

When the attacker executes `exit_delegation_pool_action` first:

1. The pool member's `unpool_time` and `unpool_amount` are zeroed.
2. The principal is transferred back to the pool member.
3. The pool member's subsequent `switch_delegation_pool` reverts with `MISSING_UNDELEGATE_INTENT`.

The pool member must now re-delegate from scratch. New delegations take effect only at `current_epoch + K` (where `K = 2`): [5](#0-4) 

During those K epochs the re-delegated stake earns no rewards. The pool member also pays additional gas for re-entry. This constitutes **griefing with damage to users** — the pool member loses K epochs of yield and is denied the ability to atomically switch delegation pools.

**Impact: Medium** — Griefing with no direct profit motive but concrete damage to the pool member (lost yield for K epochs, extra gas costs, disrupted delegation strategy).

### Likelihood Explanation

The attack requires no privileged access. Any unprivileged address can:

1. Monitor on-chain state for pool members whose `unpool_time` has passed.
2. Submit `exit_delegation_pool_action(pool_member)` before the pool member submits `switch_delegation_pool`.

On Starknet the sequencer currently processes transactions in submission order, so a watcher who submits the exit action before the pool member's switch transaction is included will succeed. The attack is cheap (one transaction) and requires no capital.

### Recommendation

Restrict `exit_delegation_pool_action` so that only the pool member (or their reward address) may trigger it:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let caller = get_caller_address();
    let pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(
        caller == pool_member || caller == pool_member_info.reward_address,
        "Only pool member or reward address can execute exit action"
    );
    ...
```

Alternatively, if permissionless execution is desired for automation purposes, add a time-based grace period after `unpool_time` during which only the pool member may act, allowing them to switch before anyone else can force the exit.

### Proof of Concept

```
1. Pool member calls exit_delegation_pool_intent(amount=X)
   → pool_member_info.unpool_time = now + exit_wait_window (≥ 1 week)
   → pool_member_info.unpool_amount = X

2. Time advances past unpool_time.

3. Pool member decides to switch and submits:
   switch_delegation_pool(to_staker=B, to_pool=B_pool, amount=X)

4. Attacker (any address) observes the pending switch and submits first:
   exit_delegation_pool_action(pool_member=victim)
   → pool_member_info.unpool_time = None
   → pool_member_info.unpool_amount = 0
   → X tokens transferred back to pool member

5. Pool member's switch_delegation_pool reverts:
   assert!(pool_member_info.unpool_time.is_some(), ...)  ← MISSING_UNDELEGATE_INTENT

6. Pool member must re-enter the pool and wait K=2 epochs before earning rewards.
``` [1](#0-0) [6](#0-5)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }
