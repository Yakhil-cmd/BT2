### Title
Stale KV-Store Permissions Survive Admin Nullification, Allowing Unauthorized Token-Mapping Remaps — (`x/cronos/keeper/permissions.go`, `x/cronos/keeper/msg_server.go`)

### Summary

When governance sets `CronosAdmin` to `""` via `MsgUpdateParams`, previously granted per-address permissions (e.g., `CanChangeTokenMapping`) remain intact in the KV store. `HasPermission` checks the KV store independently of the admin field, so any address that was granted a permission before the admin was nullified retains that permission indefinitely and can continue to remap token-contract mappings.

### Finding Description

`HasPermission` has two independent authorization paths: [1](#0-0) 

1. **Admin path** (line 39): `admin == account.String()` — fails when admin is `""`.
2. **KV-store path** (lines 42–44): reads the permission bitmask stored by `SetPermissions` and checks the bit.

`UpdateParams` / `SetParams` only writes the new params blob to the store: [2](#0-1) 

There is no logic to iterate and clear the `AdminToPermissionsKey(address)` entries when `CronosAdmin` changes. The KV-store permissions are completely orthogonal to the admin field.

`UpdateTokenMapping` gates solely on `HasPermission`: [3](#0-2) 

`UpdatePermissions` correctly blocks new grants when admin is `""` (line 106 comment: "if admin is empty, no sender could be equal to it"): [4](#0-3) 

But it cannot revoke existing grants because the new admin (nobody) cannot call `UpdatePermissions` either.

### Impact Explanation

Address X retains `CanChangeTokenMapping` and can call `UpdateTokenMapping` to remap any denom↔contract binding. This corrupts the token-mapping state that governs IBC voucher conversion and CRC20 auto-deployment, with direct security impact on asset accounting. This is a bypass of the governance authority that nullified the admin, and corruption of token mappings — both listed High impacts.

### Likelihood Explanation

The precondition is realistic: an admin routinely delegates `CanChangeTokenMapping` to operational addresses (e.g., a hot-wallet relayer). Governance later decides to remove the admin entirely (e.g., transitioning to a fully on-chain governance model). The stale permissions are invisible to governance and cannot be revoked because `UpdatePermissions` requires a non-empty admin. The window is permanent — there is no expiry or cleanup mechanism.

### Recommendation

In `SetParams`, compare the old and new `CronosAdmin`. If the admin changes (or is set to `""`), iterate all `AdminToPermissionsKey` entries and delete them (or set their bitmask to `0`). Alternatively, add a `MsgRevokePermissions` that governance itself can authorize, or make `HasPermission` short-circuit to `false` for all KV-store entries when `CronosAdmin == ""`.

### Proof of Concept

```
1. Admin = "cosmos1admin..." is set in params.
2. Admin calls MsgUpdatePermissions{from: admin, address: X, permissions: 1 (CanChangeTokenMapping)}.
   → SetPermissions writes CanChangeTokenMapping to KV store for X.
3. Governance submits MsgUpdateParams{params: {CronosAdmin: ""}}.
   → SetParams writes new params; KV store entry for X is untouched.
4. X submits MsgUpdateTokenMapping{sender: X, denom: "ibc/...", contract: "0xAttacker"}.
   → HasPermission: admin="" ≠ X.String() (path 1 fails).
   → GetPermissions(X) = 1 = CanChangeTokenMapping (path 2 succeeds).
   → Returns true → RegisterOrUpdateTokenMapping executes.
5. Token mapping is corrupted; X has effectively admin-level token-mapping power
   with no way for governance to revoke it.
```

func (k Keeper) SetParams(ctx sdk.Context, params types.Params) error {
	if err := params.Validate(); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&params)
	store.Set(types.ParamsKey, bz)

	return nil
}

func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
}
