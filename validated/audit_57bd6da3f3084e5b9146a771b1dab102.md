Looking at the `update_rewards` function in `IStakingRewardsManager` and the global `last_reward_block` storage variable, I can identify a valid analog.

### Title
Unrestricted `update_rewards` with `disable_rewards: true` Permanently Destroys Per-Block Consensus Rewards for All Stakers — (File: `src/staking/staking.cairo`)

---

### Summary
The public `update_rewards` function accepts a caller-controlled `disable_rewards` flag with no access control. When called with `disable_rewards: true`, it advances the global `last_reward_block` counter without distributing any rewards. Because the contract enforces a strict one-call-per-block invariant on this global counter, any attacker can permanently destroy the block reward for every staker by front-running the legitimate call.

---

### Finding Description

`update_rewards` is declared in `IStakingRewardsManager` and is fully public — the only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function enforces a global, single-use-per-block invariant via `last_reward_block`: [2](#0-1) 

`last_reward_block` is a single global storage slot, not per-staker: [3](#0-2) 

When `disable_rewards` is `true`, the function writes the current block number to `last_reward_block` and returns immediately without distributing anything: [4](#0-3) 

Because `last_reward_block` is global, a single call with `disable_rewards: true` for **any** valid active staker exhausts the reward slot for the entire block. No subsequent call in the same block can pass the `current_block_number > last_reward_block` check, so the block's rewards are permanently unrecoverable.

The interface definition confirms there is no documented or enforced caller restriction: [5](#0-4) 

The block rewards themselves are computed once per epoch and cached: [6](#0-5) 

Once the block slot is consumed with `disable_rewards: true`, the cached per-block reward amount for that block is never credited to any staker's `unclaimed_rewards_own`, and it is never claimed from the reward supplier. The yield is permanently frozen.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in block N:
- Advances `last_reward_block` to N.
- Distributes zero rewards to any staker.
- Prevents every other staker from receiving rewards for block N, because the one-per-block gate is now closed for that block.

Repeated across many blocks, this permanently destroys consensus-phase block rewards for the entire staker set. Stakers who later call `unstake_action` will receive only the `unclaimed_rewards_own` that was accumulated before the attack, with the destroyed blocks' rewards gone forever. [7](#0-6) 

---

### Likelihood Explanation

**Medium.** The attacker needs to submit one transaction per block ahead of any legitimate `update_rewards` call. On Starknet, transaction ordering is controlled by the sequencer, so front-running requires either (a) submitting with a higher fee/priority, or (b) being the sequencer itself. The attack is cheap (gas cost of one failing-path call per block) and requires no capital. Any unprivileged address can execute it.

---

### Recommendation

1. **Add caller restriction**: Restrict `update_rewards` to be callable only by a designated consensus/sequencer address (e.g., stored in an `authorized_rewards_updater` storage slot), or by the staker themselves.
2. **Remove the `disable_rewards` parameter from the public interface**: If `disable_rewards` is needed for migration/initialization, handle it via a separate privileged admin function.
3. **Make `last_reward_block` per-staker**: If the function must remain public, scope the one-per-block guard to the individual staker rather than a global slot, so one call cannot block all others.

---

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(staker=<any_active_staker>, disable_rewards=true)
    → last_reward_block := N
    → no rewards distributed
    → returns OK

  Legitimate tx (staker or sequencer):  update_rewards(staker=<victim>, disable_rewards=false)
    → assert(N > N)  ← FAILS with REWARDS_ALREADY_UPDATED
    → victim receives 0 rewards for block N

Block N+1, N+2, ...:
  Attacker repeats → all block rewards permanently destroyed.
```

The attacker needs only a non-zero address and a small amount of gas per block. No stake, no privileged role, no external dependency is required. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
