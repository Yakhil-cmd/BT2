### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Unclaimed Yield — (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` is a public, permissionless entry point that writes to the shared `last_reward_block` storage slot before checking the `disable_rewards` flag. Any unprivileged caller can invoke it with `disable_rewards: true` every block, consuming the one-call-per-block slot without distributing rewards, permanently denying all stakers their consensus-era block rewards.

---

### Finding Description

`update_rewards` is the consensus-era reward distribution entry point. It is intended to be called once per block by the consensus mechanism for the block-producing staker. The function enforces a single-call-per-block invariant via `last_reward_block`: [1](#0-0) 

The only gate applied is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero: [2](#0-1) 

Critically, `last_reward_block` is written **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

This means an attacker who calls `update_rewards(valid_staker, disable_rewards: true)` will:
1. Pass all validation (staker exists, has non-zero balance, block is new).
2. Advance `last_reward_block` to the current block.
3. Return immediately without distributing any rewards.
4. Block every subsequent call in the same block with `REWARDS_ALREADY_UPDATED`.

Compare this to the attestation-era equivalent, which correctly restricts its caller: [4](#0-3) 

`update_rewards_from_attestation_contract` calls `assert_caller_is_attestation_contract()`. `update_rewards` has no analogous guard — this is the missing modifier.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

`last_reward_block` is a single contract-wide slot. One successful attacker call per block is sufficient to prevent the legitimate consensus mechanism from distributing rewards to any staker for that block. Repeated every block (trivially automatable), this permanently zeroes all consensus-era staker and pool rewards. Stakers and pool members accumulate zero `unclaimed_rewards_own` / pool rewards indefinitely, constituting permanent freezing of unclaimed yield.

---

### Likelihood Explanation

**High.** The function is fully public, requires no stake, no role, and no special state. The only cost is gas. A bot calling `update_rewards` with `disable_rewards: true` at the start of every block is a realistic, low-cost, sustained attack.

---

### Recommendation

Add a caller restriction to `update_rewards` analogous to `assert_caller_is_attestation_contract()` used in `update_rewards_from_attestation_contract`. Introduce an authorized consensus contract address in storage and assert it at the top of `update_rewards`:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();
    self.assert_caller_is_consensus_contract(); // <-- add this
    ...
}
```

Alternatively, move the `self.last_reward_block.write(current_block_number)` call to after the `disable_rewards` guard so that a no-op call does not consume the block slot.

---

### Proof of Concept

```
Block N:
  Attacker → Staking::update_rewards(alice_staker, disable_rewards=true)
    ✓ general_prerequisites passes (unpaused, caller != 0)
    ✓ current_block_number (N) > last_reward_block (N-1)
    ✓ alice_staker exists and is active with non-zero balance
    → last_reward_block := N          ← slot consumed
    → disable_rewards == true → return (no rewards distributed)

  Consensus mechanism → Staking::update_rewards(block_producer, disable_rewards=false)
    ✗ assert!(N > N) → REWARDS_ALREADY_UPDATED → reverts

  Result: block_producer receives 0 rewards for block N.

Repeat every block → all stakers permanently receive 0 consensus rewards.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
