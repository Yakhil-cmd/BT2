### Title
Unrestricted `update_rewards` with `disable_rewards: true` Permanently Freezes Staker Yield — (File: src/staking/staking.cairo)

### Summary

Any unprivileged caller can invoke `update_rewards(staker_address, disable_rewards: true)` on the Staking contract. Because the global `last_reward_block` is written **before** the `disable_rewards` guard is evaluated, the call consumes the per-block reward slot without distributing any rewards. Since `last_reward_block` is a single contract-wide value, one such call per block is sufficient to permanently freeze unclaimed yield for every staker in the protocol.

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` is a public, permissionless function. Its implementation writes `last_reward_block` unconditionally, then branches on `disable_rewards`:

```cairo
// src/staking/staking.cairo  ~L1484-1489
self.last_reward_block.write(current_block_number);   // ← written first

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing
}
``` [1](#0-0) 

The storage field `last_reward_block` is a single global `BlockNumber`, not scoped per staker:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The only guard at the top of the function is `general_prerequisites()`, which checks only for pause state and a non-zero caller — no role or identity check:

```cairo
self.general_prerequisites();
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

Because `last_reward_block` is global, a single call `update_rewards(any_valid_staker, disable_rewards: true)` at block `B` sets `last_reward_block = B` for the entire contract. Any subsequent legitimate call at block `B` — whether by the staker themselves or by the attestation path — reverts with `REWARDS_ALREADY_UPDATED`. The rewards for block `B` are permanently lost for every staker.

### Impact Explanation

An attacker calling this function once per block (at minimal Starknet L2 gas cost) can suppress all consensus-era block rewards indefinitely. Stakers and delegators accumulate zero `unclaimed_rewards_own` and zero pool balance increments. This constitutes **permanent freezing of unclaimed yield** for all participants, matching the allowed High impact category. Even sporadic calls constitute **griefing with damage to users and the protocol** (Medium impact).

### Likelihood Explanation

The function is public, requires no tokens, no stake, and no privileged role. The only cost is the L2 transaction fee per block. On Starknet, fees are low enough that a motivated attacker (e.g., a competing validator or a protocol adversary) can sustain the attack indefinitely. The attack is also trivially automatable via a simple contract that calls `update_rewards` on every new block.

### Recommendation

1. **Restrict `disable_rewards: true` to privileged callers** (e.g., `only_security_agent` or `only_app_governor`), or remove the parameter from the public interface entirely.
2. **Write `last_reward_block` only when rewards are actually distributed** — move the `self.last_reward_block.write(current_block_number)` call to after the `disable_rewards` guard, so that a no-op call does not consume the block slot.
3. Alternatively, make `last_reward_block` per-staker if the intent is to track per-staker reward windows.

### Proof of Concept

```
// Attacker script (pseudocode, runs once per block)
loop {
    staking.update_rewards(
        staker_address: any_valid_staker,
        disable_rewards: true,   // no rewards distributed
    );
    // last_reward_block is now set to current block for the ENTIRE contract.
    // Any legitimate update_rewards call this block reverts with REWARDS_ALREADY_UPDATED.
    wait_for_next_block();
}
```

Concrete call path:
1. Attacker (any EOA) calls `update_rewards(victim_staker, true)` at block N.
2. `last_reward_block` is set to N; function returns early — no rewards minted or transferred.
3. Victim staker (or anyone) calls `update_rewards(victim_staker, false)` at block N → reverts `REWARDS_ALREADY_UPDATED`.
4. Victim permanently loses block-N rewards. Repeat every block.
