### Title
`native_per_pubdata` Truncates to Zero via Floor Division When `pubdata_price < native_price`, Enabling Free Pubdata Publication - (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

When the block-level `pubdata_price` is less than `native_price` (a realistic and commonly observed configuration), the integer floor division `pubdata_price.wrapping_div(native_price)` truncates to zero. This sets `native_per_pubdata = 0` for the entire transaction, meaning **zero native resources are charged for any amount of pubdata the transaction generates**. An unprivileged L2 transaction sender can exploit this to publish the maximum allowed pubdata per transaction at zero native resource cost, forcing the operator/sequencer to bear L1 data publication and proving costs without compensation.

---

### Finding Description

In `validation_impl.rs`, the ratio of pubdata cost to native cost is computed as:

```rust
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
``` [1](#0-0) 

`wrapping_div` performs **floor (truncating) integer division**. Whenever `pubdata_price < native_price`, the result is `0`. This is not a corner case — the test suite itself uses exactly this configuration:

```rust
let native_price = U256::from(100);
let pubdata_price = U256::from(2);  // pubdata_price < native_price → native_per_pubdata = 0
``` [2](#0-1) 

And the default test metadata also sets `pubdata_price = 0`: [3](#0-2) 

The resulting `native_per_pubdata = 0` propagates through two critical charging paths:

**1. Intrinsic pubdata overhead (pre-execution):**
```rust
let intrinsic_pubdata_overhead = native_per_pubdata.saturating_mul(intrinsic_pubdata);
// → 0 * intrinsic_pubdata = 0
``` [4](#0-3) 

**2. Post-execution pubdata charging:**
```rust
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)  // → current_pubdata_spent * 0 = 0
    .ok_or(out_of_native_resources!())?;
``` [5](#0-4) 

The entire pubdata native cost is zero regardless of how many bytes of pubdata the transaction writes.

**Contrast with `native_per_gas`:** The analogous ratio for gas uses **ceiling division** (`div_ceil`), explicitly preventing truncation to zero:

```rust
u256_try_to_u64(&gas_price.div_ceil(native_price))
``` [6](#0-5) 

The same asymmetry exists in the API-level helper:

```rust
// native_per_gas = ceil(gas_price / native_price)   ← ceiling
let native_per_gas = u256_try_to_u64(&gas_price.div_ceil(native_price)).ok_or(())?;

// native_per_pubdata = pubdata_price / native_price  ← floor, can be 0
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [7](#0-6) 

The `native_per_gas` path also has an explicit guard: when `native_per_gas == 0`, the system grants unlimited native resources (`u64::MAX - 1`). No equivalent guard exists for `native_per_pubdata == 0`. [8](#0-7) 

---

### Impact Explanation

When `pubdata_price < native_price`:

- Every L2 transaction pays **zero native resources** for all pubdata it generates, regardless of volume.
- An attacker submits transactions that write to as many distinct storage slots as possible (up to the block pubdata limit), paying only EVM gas for `SSTORE` opcodes but zero native resource cost for the resulting pubdata.
- The operator must pay L1 data publication costs and the prover must prove the pubdata without receiving native resource compensation from the transaction fee.
- This is a direct resource accounting loss: the operator subsidizes L1 data costs that the protocol intended to charge to the transaction sender.

The `deltaGas` adjustment in `compute_gas_refund` also uses `native_per_gas` (not `native_per_pubdata`) for its correction, so the pubdata native cost is never recovered through the gas refund path either. [9](#0-8) 

---

### Likelihood Explanation

The condition `pubdata_price < native_price` is realistic and common. The `native_price` reflects the cost of a single RISC-V proving cycle (a small unit), while `pubdata_price` reflects the cost of one byte of L1 calldata. In periods of low L1 gas prices, `pubdata_price` can easily be smaller than `native_price`. The test suite itself demonstrates this configuration as a normal operating scenario. No special attacker setup is required beyond submitting a standard L2 transaction during such a block.

---

### Recommendation

Replace `wrapping_div` with `div_ceil` for `native_per_pubdata`, mirroring the treatment of `native_per_gas`:

```rust
// Before (floor division — truncates to 0 when pubdata_price < native_price):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (ceiling division — guarantees at least 1 native unit per pubdata byte
// when pubdata_price > 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix in `api/src/helpers.rs` line 427 to keep the off-chain validation helper consistent with the on-chain bootloader behavior. [10](#0-9) 

---

### Proof of Concept

The following scenario demonstrates the issue:

1. Block is produced with `native_price = 100`, `pubdata_price = 50` (both non-zero; operator intends to charge 0.5 native units per pubdata byte).
2. `native_per_pubdata = 50.wrapping_div(100) = 0`.
3. Attacker submits a transaction that writes to 1000 distinct storage slots (generating ~32,000 bytes of pubdata).
4. `get_resources_to_charge_for_pubdata` computes `32000 * 0 = 0` native resources charged for pubdata.
5. Attacker pays only EVM gas for `SSTORE` operations; the operator bears the full L1 data cost.
6. Repeat across every block where `pubdata_price < native_price`.

The existing test at `tests/instances/transactions/src/lib.rs:1235` already runs with `native_price=100, pubdata_price=2` and expects the transaction to succeed — confirming this path is exercised without any native resource charge for pubdata. [11](#0-10)
