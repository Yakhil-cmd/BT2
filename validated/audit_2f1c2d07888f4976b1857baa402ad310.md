### Title
Missing Caller Authorization on `update_rewards` Allows Any Address to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the staking contract is documented to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block reward slot and permanently preventing legitimate reward distribution for all stakers.

### Finding Description

The specification at `docs/spec.md` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `update_rewards` in `src/staking/staking.cairo` performs no such check. The only gate is `general_prerequisites()`, which only verifies the contract is unpaused and the caller is non-zero:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

The `update_rewards` function uses a single global `last_reward_block` storage variable as a one-call-per-block rate limiter:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
...
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

An attacker calls `update_rewards(any_staker_address, disable_rewards: true)` in block N. This:
1. Passes all checks (not paused, caller non-zero, block N > last block)
2. Writes `last_reward_block = N`
3. Returns immediately without distributing any rewards

When the sequencer subsequently attempts `update_rewards(staker, disable_rewards: false)` in the same block N, the assertion `N > N` fails with `REWARDS_ALREADY_UPDATED`. Because `last_reward_block` is a **single global variable** (not per-staker), one attacker call per block blocks reward distribution for **all stakers** in that block.

### Impact Explanation

The attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block. Each call costs only gas. The result is that `last_reward_block` is always equal to the current block number before the sequencer can act, so `_update_rewards` is never reached. Stakers' `unclaimed_rewards_own` never increases and pools never receive `update_rewards_from_staking_contract` calls. This constitutes **permanent freezing of unclaimed yield** for all stakers and their delegators.

**Impact**: High — Permanent freezing of unclaimed yield.

### Likelihood Explanation

The function is publicly callable by any non-zero address with no role or identity requirement. The cost per block is a single cheap transaction. No special knowledge, privileged access, or external dependency is required. The attacker does not need to hold any stake.

**Likelihood**: High.

### Recommendation

Add an explicit sequencer-only access control check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract()`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
```

Alternatively, store the authorized sequencer address in contract storage and validate `get_caller_address() == sequencer_address`.

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. Staker `S` has been staked for `K` epochs and has non-zero balance.
3. Attacker `A` (any address, no stake required) calls in every block:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Each call writes `last_reward_block = current_block`.
5. The sequencer's call `update_rewards(S, disable_rewards: false)` in the same block fails: `REWARDS_ALREADY_UPDATED`.
6. `S.unclaimed_rewards_own` remains zero indefinitely; pool receives no rewards.
7. `S` and all delegators are permanently denied yield with no recourse.
