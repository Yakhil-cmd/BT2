### Title
Unauthorized Native Token Transfer via Bank Precompile `transfer` Method Missing Caller Authorization - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The `transfer` method of the bank precompile (`BankContract.Run`) accepts an arbitrary `sender` address as an ABI-decoded input argument and uses it directly as the `from` address in `bankKeeper.SendCoins`, without ever verifying that `contract.Caller()` (the EVM `msg.sender`) matches the supplied `sender`. Any EVM contract can therefore transfer `evm/<caller_contract>` native tokens out of any victim account that holds that denom, without the victim's authorization.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` branch of `BankContract.Run` decodes `sender` from the raw ABI input and passes it directly to `bankKeeper.SendCoins` as the source account:

```go
sender := args[0].(common.Address)   // attacker-controlled
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

There is no guard of the form `contract.Caller() == sender`. The denom being transferred is `EVMDenom(contract.Caller())` = `"evm/" + contract.Caller().Hex()`, so the attacker controls both the denom (by choosing which contract calls the precompile) and the victim address (via the `sender` argument). [2](#0-1) 

The same missing-authorization pattern also affects the `burn` branch: `addr` (the address to burn from) is taken from ABI input with no check that it equals `contract.Caller()`. [3](#0-2) 

The correct pattern — verifying the signer matches the caller — is already implemented in the generic `exec` helper used by other precompiles:

```go
caller := common.BytesToAddress(signers[0])
if caller != e.caller {
    return nil, fmt.Errorf("caller is not authenticated: ...")
}
``` [4](#0-3) 

The bank precompile's `transfer` and `burn` methods never apply this check.

### Impact Explanation
**Critical — Unauthorized transfer and burn of precompile-controlled native assets.**

`evm/<contract_address>` tokens are native Cosmos-layer assets managed exclusively through the bank precompile. Any user who holds `evm/<attacker_contract>` tokens (e.g., after calling a contract's `moveToNative`-style function) can have those tokens stolen or destroyed by the contract owner without consent. The `bankKeeper.SendCoins` call succeeds because the Cosmos bank module has no EVM-level authorization concept; it trusts the precompile to enforce caller identity. The precompile fails to do so. [5](#0-4) 

### Likelihood Explanation
Any deployed EVM contract that calls the bank precompile at address `0x0000000000000000000000000000000000000064` can trigger this. The attacker needs only to deploy a contract and have victims hold `evm/<attacker_contract>` native tokens — a realistic scenario for any token contract that exposes a `moveToNative` flow (as shown in `TestBank.sol`). No special privilege, leaked key, or governance action is required. [6](#0-5) 

### Recommendation
In the `TransferMethodName` branch, assert that the decoded `sender` equals `contract.Caller()` before executing the transfer:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender is not authenticated")
}
```

Apply the same guard in the `BurnMethodName` branch, asserting `recipient == contract.Caller()`. This mirrors the authentication pattern already used in `exec` in `utils.go`. [4](#0-3) 

### Proof of Concept

1. Attacker deploys contract `M` on Cronos EVM. `M` exposes a `moveToNative(amount)` function that calls `bank.mint(msg.sender, amount)`, minting `evm/0xM...` native tokens to callers.

2. Victim calls `M.moveToNative(1000)`. Victim now holds 1000 `evm/0xM...` native tokens on the Cosmos layer.

3. Attacker adds a `steal(address victim, address attacker, uint256 amount)` function to `M` (or a second contract that calls the precompile with `M`'s address as caller via a delegatecall/proxy):

```solidity
IBankModule bank = IBankModule(0x0000000000000000000000000000000000000064);
function steal(address victim, address to, uint256 amount) external {
    bank.transfer(victim, to, amount);
    // denom = "evm/" + address(this) — victim's tokens
}
```

4. Attacker calls `M.steal(victim, attacker, 1000)`. The bank precompile executes `bankKeeper.SendCoins(victim_cosmos_addr, attacker_cosmos_addr, 1000 evm/0xM...)` with no authorization check.

5. Victim's 1000 `evm/0xM...` native tokens are transferred to the attacker. The victim has no recourse. [7](#0-6) [8](#0-7)
