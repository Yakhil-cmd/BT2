### Title
Unbounded Loop in `calculate_rewards` Over `pool_member_epoch_balance` Trace Can Permanently Freeze Unclaimed Yield - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` internal function in `src/pool/pool.cairo` contains an unbounded `while` loop that iterates over every entry in a pool member's `pool_member_epoch_balance` trace. A pool member who makes many balance changes (deposits, withdrawals, increases) across many epochs without claiming rewards will accumulate an arbitrarily large trace. When they eventually call `claim_rewards` or `exit_delegation_pool_action`, the loop will exceed Starknet's gas limit, permanently freezing their unclaimed yield.

### Finding Description
In `src/pool/pool.cairo`, the internal function `calculate_rewards` (lines 837–888) iterates over the entire `pool_member_epoch_balance` trace for a given pool member to compute accumulated rewards:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    ...
    entry_to_claim_from += 1;
}
```

The codebase itself acknowledges the issue with the comment "This loop is unbounded but unlikely to exceed gas limits." [1](#0-0) 

The `pool_member_epoch_balance` trace is a `Vec`-backed checkpoint structure (`PoolMemberBalanceTrace`) that appends a new entry every time a pool member's balance changes at a new epoch. [2](#0-1) 

Each call to `set_member_balance` (invoked by `enter_delegation_pool`, `increase_delegation_pool`, `exit_delegation_pool_intent`) inserts a new checkpoint keyed by `current_epoch + K`. A pool member who performs balance changes across N distinct epochs accumulates N trace entries. [3](#0-2) 

`calculate_rewards` is called from both `claim_rewards` and `exit_delegation_pool_action`, which are the only ways a pool member can recover their funds or yield. [4](#0-3) 

### Impact Explanation
If the `pool_member_epoch_balance` trace grows large enough, the `while` loop in `calculate_rewards` will consume more gas than Starknet's per-transaction limit. Since `calculate_rewards` is on the critical path of both `claim_rewards` and `exit_delegation_pool_action`, the pool member's unclaimed yield becomes permanently inaccessible — every attempt to claim or exit will revert with an out-of-gas error. This matches the **High** impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
Any pool member who actively manages their delegation (frequent deposits, partial withdrawals, or re-entries) across many epochs without claiming rewards will grow their trace. Starknet epochs are short (on the order of hours), so a delegator active over months can accumulate hundreds of trace entries. The risk is realistic for long-term, active delegators. The protocol provides no mechanism to claim rewards in partial batches (no `from`/`to` epoch parameters on `claim_rewards`), so there is no way to recover once the trace is too large.

### Recommendation
1. **Add a checkpoint parameter to `claim_rewards`**: Allow callers to pass an explicit `until_checkpoint` or a maximum number of trace entries to process per call, so rewards can be claimed incrementally.
2. **Enforce a maximum trace length**: Cap the number of unclaimed balance-change entries per pool member (e.g., by requiring a claim before a new balance change is accepted once the trace exceeds a threshold).
3. **Update the comment**: At minimum, replace the dismissive comment with a documented invariant that bounds the expected trace length, and add a protocol-level enforcement of that bound.

### Proof of Concept
1. A pool member calls `enter_delegation_pool` in epoch 1, then calls `increase_delegation_pool` in epochs 2, 3, 4, …, N — each call at a new epoch appends one entry to `pool_member_epoch_balance` via `set_member_balance` → `pool_member_trace.insert(key: epoch + K, ...)`. [3](#0-2) 
2. The pool member never calls `claim_rewards`, so `entry_to_claim_from` (stored in their checkpoint) remains at 0.
3. After N epochs, the pool member calls `claim_rewards`. This invokes `calculate_rewards` with `entry_to_claim_from = 0` and `pool_member_trace_length = N`.
4. The `while` loop at line 859 iterates N times, each iteration performing storage reads (`pool_member_trace.at(entry_to_claim_from)`) and a `find_sigma` call. [5](#0-4) 
5. For sufficiently large N (bounded only by the number of epochs the protocol has been live), the transaction exceeds the Starknet gas limit and reverts.
6. All subsequent calls to `claim_rewards` and `exit_delegation_pool_action` also revert, permanently freezing the pool member's accumulated yield.

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
