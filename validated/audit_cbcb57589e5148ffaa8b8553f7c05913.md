### Title
Missing Caller Validation in `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `src/staking/staking.cairo` is documented as restricted to "Only starkware sequencer" but contains **no caller access control** in its implementation. Any unprivileged address can call it with `disable_rewards: true` for any valid staker, which advances the global `last_reward_block` checkpoint without distributing rewards. Because the guard `REWARDS_ALREADY_UPDATED` is keyed on a single global `last_reward_block` value, a single such call permanently forfeits all staker and pool rewards for that block — the sequencer's legitimate call for the same block will always revert.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` is the sole entry point for distributing per-block consensus rewards to stakers and their delegation pools. Its implementation begins:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global checkpoint written here

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits without distributing rewards
    }
    ...
}
``` [1](#0-0) 

`general_prerequisites()` only asserts the contract is not paused — there is no `assert!(get_caller_address() == sequencer_address, ...)` or equivalent guard. The spec explicitly states access is "Only starkware sequencer": [2](#0-1) 

The `last_reward_block` storage slot is a **single global value** (not per-staker). Writing it with `disable_rewards: true` consumes the block's reward slot for every staker simultaneously. Any subsequent call in the same block — including the sequencer's legitimate call — reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Block rewards that are not distributed in the block they are earned are permanently lost; there is no catch-up mechanism. An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` in block N causes:

1. `last_reward_block` is set to N.
2. No STRK or BTC rewards are minted or transferred to any staker or pool for block N.
3. The sequencer's own `update_rewards` call for block N reverts with `REWARDS_ALREADY_UPDATED`.
4. Rewards for block N are gone for **all** stakers and all delegation pools.

Because `last_reward_block` is global, a single griefing transaction affects the entire protocol for that block.

---

### Likelihood Explanation

**High.** The function is publicly callable with no authentication. The only inputs required are any valid, active `staker_address` (all staker addresses are public on-chain) and the boolean `disable_rewards: true`. On Starknet L2, while the sequencer controls ordering, any user-submitted transaction can be included in a block before the sequencer's own system transaction. An attacker can submit this call every block at negligible cost to continuously suppress all consensus rewards across the protocol.

---

### Recommendation

Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured sequencer address (or a dedicated rewards-manager role), consistent with the spec's stated access control:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

Alternatively, restrict the `IStakingRewardsManager` interface to an internal-only call path that is not externally dispatchable.

---

### Proof of Concept

1. Staker S is active with non-zero balance at epoch E (past the K-epoch delay).
2. Consensus rewards are active (`is_pre_consensus()` returns false).
3. Attacker A (any address) calls:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(
       staker_address: S,
       disable_rewards: true
   )
   ```
   in block N.
4. Inside `update_rewards`: `general_prerequisites()` passes (not paused); `current_block_number (N) > last_reward_block` passes; `last_reward_block` is written to N; `disable_rewards == true` → early return, zero rewards distributed.
5. Sequencer calls `update_rewards(S, disable_rewards: false)` in the same block N.
6. `assert!(N > last_reward_block.read())` → `assert!(N > N)` → **reverts with `REWARDS_ALREADY_UPDATED`**.
7. All stakers and pools receive zero rewards for block N. The rewards are permanently lost. [4](#0-3)
