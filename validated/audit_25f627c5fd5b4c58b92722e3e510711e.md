Audit Report

## Title
NNS Governance `disburse_neuron` Unconditionally Zeroes `neuron_fees_e8s` Even When Fee Burn Is Skipped, Causing Permanent Ledger/Governance Accounting Divergence - (File: rs/nns/governance/src/governance.rs)

## Summary
In `disburse_neuron`, the neuron accounting fields `cached_neuron_stake_e8s` and `neuron_fees_e8s` are updated unconditionally outside the burn-conditional block. When `fees_amount_e8s <= transaction_fee_e8s`, the burn is skipped but governance still zeroes `neuron_fees_e8s` and decrements `cached_neuron_stake_e8s`, permanently diverging governance state from the actual ICP ledger balance. The SNS governance canister contains an explicit fix and comment for this exact pattern, confirming the NNS code is incorrect.

## Finding Description
In `rs/nns/governance/src/governance.rs`, the burn transfer is correctly gated:

```rust
// Line 2046
if fees_amount_e8s > transaction_fee_e8s {
    self.ledger.transfer_funds(fees_amount_e8s, 0, ...).await?;
}
``` [1](#0-0) 

Immediately after, outside the `if` block, the neuron state is updated unconditionally:

```rust
// Lines 2067-2076
self.with_neuron_mut(id, |neuron| {
    if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
        neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
    } else {
        neuron.cached_neuron_stake_e8s = 0;
    }
    neuron.neuron_fees_e8s = 0;  // ← always executed
})
``` [2](#0-1) 

When the burn is skipped (`fees_amount_e8s <= transaction_fee_e8s`), the ICP ledger subaccount still holds `fees_amount_e8s` tokens, but governance records them as gone. The subsequent disburse transfer at lines 2091–2100 then moves `disburse_amount_e8s` (computed as `neuron_minted_stake_e8s = cached_neuron_stake_e8s - neuron_fees_e8s`) out of the subaccount. [3](#0-2)  After both operations, governance believes `cached_neuron_stake_e8s = 0` and `neuron_fees_e8s = 0`, but the ledger subaccount retains `fees_amount_e8s` tokens with no recovery path.

The SNS governance canister fixes this exactly by placing the state update inside the burn-conditional block with an explicit comment:

```rust
// Lines 1193-1194
// We only update the cached_neuron_stake_e8s and neuron_fees_e8s if we actually
// burn fees, otherwise this leads to ledger and governance getting out of sync.
``` [4](#0-3) 

The TLA+ formal model in `Disburse_Neuron.tla` also models the correct behavior: in the `else` branch (burn skipped), it calls `update_fees(neuron_id, fees_amount)` before proceeding to the stake transfer, meaning the formal specification does not match the Rust implementation. [5](#0-4) 

## Impact Explanation
Every dissolved NNS neuron with `0 < neuron_fees_e8s <= transaction_fee_e8s` that is disbursed permanently strands up to `transaction_fee_e8s` (10 000 e8s = 0.0001 ICP) in its ICP ledger subaccount. Governance records the neuron as fully emptied, but the ledger disagrees. The discrepancy is irreversible: no governance command can recover the stranded balance because governance believes the neuron has zero stake. This constitutes a permanent, irreversible loss of user funds (stranded ICP) and a persistent governance/ledger accounting divergence. While the per-neuron amount is small (≤ 0.0001 ICP), the condition is reachable by any unprivileged user and accumulates across the entire NNS neuron population over time. This matches the **High** impact class: unauthorized/unrecoverable loss of user funds in the NNS governance system with concrete, demonstrable harm.

## Likelihood Explanation
The trigger condition is reachable by any unprivileged NNS neuron owner through a normal user flow: submit a proposal that gets rejected (accumulating `reject_cost_e8s = transaction_fee_e8s = 10 000 e8s` in `neuron_fees_e8s`), then dissolve and disburse the neuron. With exactly one rejected proposal, `neuron_fees_e8s == transaction_fee_e8s`, which fails the `> transaction_fee_e8s` check and skips the burn. No special privileges, no coordination, and no victim mistakes are required. The condition is deterministic and repeatable.

## Recommendation
Mirror the SNS governance fix: move the `neuron_fees_e8s` and `cached_neuron_stake_e8s` update inside the `if fees_amount_e8s > transaction_fee_e8s` block in `rs/nns/governance/src/governance.rs` around lines 2067–2076, so accounting is only adjusted when the burn actually executes. [2](#0-1)  When the burn is skipped, leave `neuron_fees_e8s` unchanged and do not decrement `cached_neuron_stake_e8s` by `fees_amount_e8s`.

## Proof of Concept
The SNS governance unit test `test_disburse_neuron_small_fees_not_burned` in `rs/sns/governance/src/governance/disburse_neuron_tests.rs` (lines 540–580) explicitly validates the correct behavior and can serve as a template for an equivalent NNS test. [6](#0-5) 

**Minimal NNS reproduction:**
1. Create a neuron with `cached_neuron_stake_e8s = 100_000_000` and `neuron_fees_e8s = 5_000` (< `transaction_fee_e8s = 10_000`).
2. Set neuron state to `Dissolved`.
3. Call `disburse_neuron` with `amount = None` (full disburse).
4. Observe: only one ledger transfer occurs (no burn). Ledger subaccount retains 5 000 e8s. Governance records `cached_neuron_stake_e8s = 0`, `neuron_fees_e8s = 0`.
5. Assert: ledger subaccount balance > 0 while governance believes the neuron is empty — permanent divergence confirmed.

#[test]
fn test_disburse_neuron_small_fees_not_burned() {
    // Test that disburse_neuron doesn't burn fees that are too small and preserves accounting
    let (mut governance, neuron_id, ledger) =
        setup_disburse_neuron_test(DissolveState::WhenDissolvedTimestampSeconds(0), 1000);

    let disburse = manage_neuron::Disburse {
        amount: None,
        to_account: None,
    };

    // This should succeed but not burn any fees
    let result = governance
        .disburse_neuron(&neuron_id, &A_NEURON_PRINCIPAL_ID, &disburse)
        .now_or_never()
        .unwrap();

    assert_eq!(result, Ok(1)); // Mock ledger returns block height 1

    // Verify that only one transfer was made (no burn), just the disburse transfer
    let transfer_calls = ledger.get_transfer_calls();
    assert_eq!(transfer_calls.len(), 1); // Only one transfer (disburse), no burn

    // Check disburse call
    let disburse_call = &transfer_calls[0];
    assert!(disburse_call.is_transfer());
    // Disburse: (500M stake - 1K fees) - 10K tx_fee = 499,989,000
    disburse_call.assert_amount_and_fee(499_989_000, 10_000);

    // Check that the neuron fees were NOT reduced (preserved for future)
    let updated_neuron = governance
        .proto
        .neurons
        .get(&neuron_id.to_string())
        .unwrap();
    // Fees should remain unchanged since they were too small to burn
    assert_eq!(updated_neuron.neuron_fees_e8s, 1_000);

    // Check cached_neuron_stake_e8s: 500M - 499.989M disbursed - 10K tx_fee = 1K (equals fees)
    assert_eq!(updated_neuron.cached_neuron_stake_e8s, 1_000);
}
