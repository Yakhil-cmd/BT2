### Title
WalletConnect `chia_wallet.cat_spend` Confirmation Omits CAT Identity — Malicious dApp Can Cause User to Spend Wrong CAT Asset - (`packages/gui/src/electron/commands/humanizeParamValue.ts`)

---

### Summary

The WalletConnect confirmation dialog for `chia_wallet.cat_spend` displays the `wallet_id` as a raw integer and the amount without any CAT name or symbol. Because `lookupCat` is permanently disabled (commented out with a TODO), a malicious dApp can substitute any `wallet_id` — including one belonging to a high-value CAT — and the user has no way to identify which CAT is actually being spent from the confirmation screen.

---

### Finding Description

**Root cause — `formatMojoCat` never resolves the CAT name:**

In `humanizeParamValue.ts`, `formatMojoCat` reads `wallet_id` from the request data but then unconditionally returns only the numeric locale string. The `lookupCat` call that would append the CAT display name is commented out with a `// TODO add lookupCat` marker and is unreachable dead code: [1](#0-0) 

**Command schema — `wallet_id` rendered as a raw number:**

The `chia_wallet.cat_spend` schema in `Commands.ts` declares `wallet_id` as `type: 'number'`, which `humanizeParamValue` converts to a plain `String(value)` (e.g., `"3"`). The `amount` field uses `humanize: 'mojo-to-cat'`, which calls the broken `formatMojoCat` above: [2](#0-1) 

**No enriched display for `cat_spend`:**

`parseCommandDisplay` — which provides the richer "You Spend / You Receive" wallet-delta view with resolved names — only handles `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids`. For `chia_wallet.cat_spend` it returns `undefined`: [3](#0-2) 

**Confirmation dialog renders only raw rows:**

Because `display` is `undefined`, the `WalletDeltaSection` (which shows resolved asset names) is never rendered. The user sees only the raw label/value rows: [4](#0-3) 

The resulting dialog shows:
```
Wallet Id:  3          ← opaque integer, no CAT name
Amount:     1.000      ← no symbol, no ticker
Address:    xch1...
Fee:        0.000001 XCH
```

---

### Impact Explanation

A malicious WalletConnect dApp can send `chia_wallet.cat_spend` with `wallet_id` set to the user's high-value CAT wallet instead of the low-value one the user expects. Because the confirmation dialog shows only a raw integer for `wallet_id` and a bare number for `amount` (no CAT name, no asset ID, no ticker), the user cannot distinguish a spend of a $0.01 CAT from a spend of a $1,000 CAT. If the user approves, the high-value CAT is irreversibly transferred.

This is a direct, unauthorized asset transfer caused by WalletConnect state spoofing — fitting the **High** impact category: *"WalletConnect state that causes a user to approve the wrong asset."*

---

### Likelihood Explanation

- Any dApp that has been granted `chia_spendCAT` permission via WalletConnect can trigger this immediately.
- No additional privileges, leaked keys, or host compromise are required.
- Users with multiple CAT wallets (a common configuration for DeFi participants) are the target population.
- The `lookupCat` TODO has been present since the feature was introduced, meaning no version of this code has ever shown the CAT name in the confirmation dialog.

---

### Recommendation

1. **Implement `lookupCat`** in `formatMojoCat` (`humanizeParamValue.ts` lines 28–30): resolve `wallet_id` to the wallet's CAT name/asset ID via the daemon API and append it to the displayed amount string (e.g., `"1.000 SBX"`).
2. **Extend `parseCommandDisplay`** to handle `chia_wallet.cat_spend`: look up the wallet by `wallet_id`, resolve its CAT asset ID and display name, and populate a `walletDelta` entry so the richer "You Spend" section is shown with the resolved CAT identity.
3. **Validate `wallet_id` type**: confirm the wallet identified by `wallet_id` is actually a CAT wallet before executing the spend, and surface a warning if it is not.

---

### Proof of Concept

**Preconditions:** User has two CAT wallets — wallet ID 3 (low-value token, e.g., 1 mojo = $0.00001) and wallet ID 7 (high-value token, e.g., 1 CAT = $500). dApp has been granted `chia_spendCAT` permission.

**Steps:**
1. dApp sends via WalletConnect:
   ```json
   {
     "method": "chia_spendCAT",
     "params": {
       "wallet_id": 7,
       "address": "xch1attacker...",
       "amount": "1000",
       "fee": "1000000"
     }
   }
   ```
2. GUI confirmation dialog opens showing:
   - **Wallet Id:** `7` (user does not know wallet 7 is their high-value CAT)
   - **Amount:** `0.001` (no CAT name — user thinks this is the low-value token)
3. User clicks **Send**.
4. `cat_spend` executes against wallet ID 7 — the high-value CAT is transferred to the attacker's address.

**Assert:** The confirmation dialog never displays the CAT name or asset ID for `wallet_id: 7`, confirmed by the dead-code path in `humanizeParamValue.ts` lines 28–30 and the `undefined` return from `parseCommandDisplay` for this command.

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }

  return undefined;
