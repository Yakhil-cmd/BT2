### Title
Single Malicious Participant Can Permanently Abort DKG, Reshare, and Refresh via False Success Broadcast — (File: src/dkg.rs)

---

### Summary

The `broadcast_success` function in `src/dkg.rs` requires **unanimous** agreement from all `n` participants that DKG succeeded. A single malicious participant can broadcast `(false, session_id)` in the final round, causing every honest party to abort with a non-attributable error, permanently denying key generation.

---

### Finding Description

`broadcast_success` (called at line 531 of `do_keyshare`) is the final step of `do_keygen`, `do_reshare`, and `do_refresh`. It always broadcasts `(true, session_id)` and then asserts that **every** participant did the same:

```rust
// src/dkg.rs lines 329-335
if !vote_list.iter().all(|&(boolean, _)| boolean) {
    return Err(ProtocolError::AssertionFailed(
        "A participant
            seems to have failed its checks. Aborting Protocol!"
            .to_string(),
    ));
}
``` [1](#0-0) 

This unanimity requirement is structurally inconsistent with the library's own echo-broadcast layer, which is explicitly designed to tolerate up to `(n−1)/3` malicious parties:

```rust
// src/protocol/echo_broadcast.rs lines 67-78
fn echo_ready_thresholds(n: usize) -> (usize, usize) {
    if n <= 3 { return (0, 0); }
    let broadcast_threshold = (n - 1) / 3;
    let echo_threshold = usize::midpoint(n, broadcast_threshold);
    (echo_threshold, broadcast_threshold)
}
``` [2](#0-1) 

**Attack path:**

1. Malicious participant `P_m` participates honestly in all prior DKG rounds (commitment hash broadcast, commitment + proof-of-knowledge broadcast, private share exchange) to avoid early detection.
2. In the final `broadcast_success` round, `P_m` deviates and broadcasts `(false, session_id)` instead of `(true, session_id)`.
3. The echo-broadcast protocol guarantees consistency: because `P_m` sends the same `false` value to all peers, every honest party reliably delivers `(false, session_id)` from `P_m`.
4. The `.all(|&(boolean, _)| boolean)` check fails for every honest party simultaneously.
5. All honest parties return `ProtocolError::AssertionFailed("A participant seems to have failed its checks. Aborting Protocol!")`. [3](#0-2) 

Critically, the error message does **not** identify which participant broadcast `false`. Honest parties have no way to attribute the failure and exclude `P_m` from a retry. `P_m` can repeat this attack on every subsequent DKG attempt it is invited to join.

The same `do_keyshare` path is shared by `do_keygen`, `do_reshare`, and `do_refresh`, so all three protocols are affected. [4](#0-3) [5](#0-4) 

---

### Impact Explanation

**High — Permanent denial of key generation, reshare, and refresh for honest parties.**

A single participant below the fault-tolerance threshold of the echo-broadcast layer can unilaterally abort every DKG attempt. Because the culprit is not identified in the error, honest parties cannot remove them from the participant set without out-of-band coordination. This maps directly to the allowed impact: *"Permanent denial of signing, key generation, reshare, refresh, or CKD for honest parties under valid protocol inputs and documented trust assumptions."*

---

### Likelihood Explanation

**High.** The attack requires no cryptographic capability, no leaked material, and no external dependency. Any enrolled participant executes it by sending a single bit (`false`) in the last broadcast round. The attacker can sustain the denial indefinitely at negligible cost.

---

### Recommendation

1. **Replace the unanimity check with a threshold check.** Require only a quorum (e.g., `> 2*(n−1)/3`) of `true` votes before declaring success, consistent with the fault-tolerance already assumed by the echo-broadcast layer.
2. **Attribute the failure.** When a `false` vote is received, record and surface the sender's `Participant` identity (analogous to `ProtocolError::MaliciousParticipant(participant)` already used elsewhere in `dkg.rs`) so callers can exclude the culprit.
3. **Structural analog to the BvbProtocol fix:** just as the recommended fix removed reliance on the bull's cooperation by routing the NFT through the contract itself, `broadcast_success` should be restructured so that a single participant's refusal to cooperate cannot abort the protocol for all honest parties. [1](#0-0)

DKG fails for all honest parties. No culprit is identified.
P_m repeats on every retry → permanent DoS.

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)

fn echo_ready_thresholds(n: usize) -> (usize, usize) {
    // case where no malicious parties are assumed: when n <= 3/
    // In this case the echo and ready thresholds are both 0
    // later we compare if we have collected more votes than these thresholds
    if n <= 3 {
        return (0, 0);
    }
    // we should always have n >= 3*threshold + 1
    let broadcast_threshold = (n - 1) / 3;
    let echo_threshold = usize::midpoint(n, broadcast_threshold);
    (echo_threshold, broadcast_threshold)
}
