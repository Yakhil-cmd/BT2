### Title
Single Malicious Participant Aborts DKG/Reshare/Refresh for All Honest Parties via `broadcast_success` — (File: `src/dkg.rs`)

---

### Summary

The final round of the DKG, reshare, and refresh protocols calls `broadcast_success`, which requires **every single participant** to broadcast `(true, session_id)`. A single malicious participant — even one within the documented `n/3` fault-tolerance bound — can broadcast `(false, session_id)` instead, causing all honest parties to abort the protocol and permanently lose the ability to complete key generation or resharing.

---

### Finding Description

At the end of `do_keyshare`, after all local computations succeed, every participant calls `broadcast_success`: [1](#0-0) 

`broadcast_success` always sends `(true, session_id)` via `do_broadcast` (which runs the full echo-broadcast protocol), then checks that **all** received votes are `true`: [2](#0-1) 

The check at line 329 is:

```rust
if !vote_list.iter().all(|&(boolean, _)| boolean) {
    return Err(ProtocolError::AssertionFailed(
        "A participant seems to have failed its checks. Aborting Protocol!"
    ));
}
```

This requires **unanimous** success. A malicious participant can instead send `(false, session_id)` through the echo-broadcast protocol. Because `reliable_broadcast_receive_all` faithfully echoes whatever SEND message it receives from each sender, if the malicious participant sends `(false, session_id)` to all other parties in the SEND round, all honest parties will echo it, accumulate enough echo votes to exceed `echo_t`, and ultimately deliver `(false, session_id)` for that participant's broadcast session. [3](#0-2) 

Once `vote_list` contains even one `false` entry, `broadcast_success` returns `Err`, and `do_keyshare` propagates that error to the caller, aborting DKG for every honest participant. [4](#0-3) 

This affects `do_keygen`, `do_reshare`, and (via `do_keyshare`) key refresh, since all three converge on the same final step. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

**High — Permanent denial of DKG, reshare, and refresh for honest parties.**

The README documents that the DKG tolerates up to `floor(n/3)` malicious parties. For `n ≥ 7`, a single malicious party is within that tolerance, yet it can unilaterally abort DKG for all `n-1` honest parties. No key material is ever produced, and the protocol must be restarted from scratch — which the same attacker can abort again indefinitely at negligible cost (they only need to participate honestly through all prior rounds and then send one `false` vote in the final broadcast). [2](#0-1) 

---

### Likelihood Explanation

**High.** The attacker only needs to be a registered participant. They participate honestly through all expensive rounds (polynomial commitment, proof-of-knowledge, secret share distribution) and then, in the final `broadcast_success` round, send `(false, session_id)` instead of `(true, session_id)`. This is a single-message deviation at the very last step, requiring no cryptographic capability and no prior compromise. The attack is repeatable across every DKG/reshare attempt.

---

### Recommendation

Replace the unanimous-success check with a threshold-based check. Honest parties should complete DKG successfully if at least `n - floor(n/3)` participants broadcast success. Specifically:

1. Count the number of `true` votes in `vote_list`.
2. Proceed if that count meets or exceeds `n - max_faulty` (where `max_faulty = floor((n-1)/3)`).
3. Return an error only if the count falls below that threshold, indicating more malicious parties than the protocol can tolerate.

This mirrors the fault-tolerance guarantee already provided by the echo-broadcast layer itself.

---

### Proof of Concept

Setup: 7 participants, threshold 4 (so `max_faulty = floor(6/3) = 2`; 1 malicious party is within tolerance).

1. Malicious participant `M` runs `do_keyshare` honestly through rounds 1–5 (session-id broadcast, commitment hash, commitment+PoK broadcast, secret share exchange).
2. In the `broadcast_success` call, instead of sending `(true, session_id)`, `M` sends `(false, session_id)` to all other 6 participants in the SEND phase of the echo-broadcast.
3. All 6 honest parties receive `(false, session_id)` from `M`, echo it, accumulate `> echo_t` echo votes, and deliver `(false, session_id)` for `M`'s broadcast session.
4. Each honest party's `vote_list` now contains `(false, session_id)` for `M`.
5. The check `vote_list.iter().all(|&(boolean, _)| boolean)` fails for all 6 honest parties.
6. All 6 honest parties return `Err(ProtocolError::AssertionFailed(...))` and discard their computed key shares.
7. No key is generated. The attack can be repeated on every subsequent attempt. [7](#0-6) [8](#0-7)

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
