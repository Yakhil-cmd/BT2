### Title
Wrong Rounding Direction in `validate_tx_l2_gas_price_within_threshold` Threshold Computation — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price threshold by multiplying the previous block's L2 gas price by a configured percentage ratio and calling `.to_integer()`, which truncates (rounds down). Because the threshold is a lower bound that must be enforced strictly, it should round **up**. Rounding down produces a threshold that is 1 FRI lower than intended whenever the product is non-integer, allowing transactions with an L2 gas price below the true intended minimum to pass gateway admission.

---

### Finding Description

In `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold`:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← truncates toward zero (floor)
if tx_l2_gas_price.0 < threshold { ... }
```

`Ratio::to_integer()` from `num_rational` truncates toward zero, i.e., it floors the result for positive values. The threshold is the minimum L2 gas price a transaction must offer. To enforce the invariant conservatively (reject anything below the intended minimum), the threshold must be a **ceiling** of the fractional value, not a floor.

**Concrete example**:
- `previous_block_l2_gas_price = 101` FRI, `min_gas_price_percentage = 50`
- True threshold = `50 × 101 / 100 = 50.5` FRI → correct enforcement requires ceiling → **51** FRI
- Computed threshold = `floor(50.5)` = **50** FRI
- A transaction offering exactly 50 FRI passes the check (`50 < 50` is false), but the operator intended to require at least 51 FRI

The off-by-one occurs whenever `min_gas_price_percentage × previous_block_l2_gas_price` is not divisible by 100. With the default `min_gas_price_percentage = 100` the product is always exact, so the default deployment is unaffected. Any non-default percentage (e.g., 50, 80, 90) triggers the rounding error on every block whose price is not a multiple of `100 / gcd(percentage, 100)`. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions.**

The gateway's stateful path is the first line of defense for enforcing the minimum L2 gas price policy. When the threshold is under-estimated by 1 FRI, transactions whose `max_price_per_unit` equals the floored threshold (but is below the true ceiling threshold) are admitted into the mempool and forwarded to the batcher. These transactions carry an L2 gas price that violates the operator's configured minimum, breaking the admission invariant.

---

### Likelihood Explanation

Triggered on every block where `min_gas_price_percentage × previous_block_l2_gas_price` has a non-zero fractional part when divided by 100. This is the common case for any non-100% percentage setting. Any operator who configures `min_gas_price_percentage` to a value other than 100 (e.g., 50 or 80, which are natural choices for a "require at least 80% of last block's price" policy) will experience this on most blocks. An attacker who knows the previous block's L2 gas price can deterministically craft a transaction at exactly the floored threshold to bypass the check.

---

### Recommendation

Replace `.to_integer()` (floor) with `.ceil().to_integer()` (ceiling) so the threshold is always at least as large as the true fractional value:

```rust
// Before (rounds down):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();

// After (rounds up):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This ensures that any transaction whose `max_price_per_unit` is strictly below the ceiling of the intended threshold is rejected, preserving the admission invariant regardless of whether the product is an integer. [3](#0-2)

    50_u128.into(),                 // tx_l2_gas_price = floored threshold

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
