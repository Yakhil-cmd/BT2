### Title
Presignature Reuse Enabled by Non-Consuming `rerandomize_presign` API Allows Private Key Extraction — (File: `src/ecdsa/ot_based_ecdsa/mod.rs`)

---

### Summary

`RerandomizedPresignOutput::rerandomize_presign` accepts a `&PresignOutput` by shared reference rather than consuming it by value, and `PresignOutput` is `Clone`. The library provides no mechanism to enforce single-use of a presignature. A malicious coordinator can instruct participants to rerandomize and sign with the same presignature under two different `(msg_hash, tweak, entropy)` tuples, producing two ECDSA signatures whose nonces are multiplicatively related. Standard nonce-reuse algebra then recovers the private key.

---

### Finding Description

`PresignOutput` is declared with `Clone`:

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput {
    pub big_r: AffinePoint,
    pub k: Scalar,
    pub sigma: Scalar,
}
``` [1](#0-0) 

`rerandomize_presign` takes the presignature by shared reference, leaving the original intact and reusable:

```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // ← shared reference, not consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> {
``` [2](#0-1) 

The rerandomization derives a scalar `delta` from `(pk, tweak, msg_hash, big_r, entropy)` via HKDF and produces:

```
k_rerandomized  = k  * delta^{-1}
sigma_rerandomized = (sigma + tweak*k) * delta^{-1}
big_r_rerandomized = big_r * delta
``` [3](#0-2) 

If the same `PresignOutput` is rerandomized twice with different `(msg_hash₁, tweak₁, entropy₁)` and `(msg_hash₂, tweak₂, entropy₂)`, the two resulting nonce shares satisfy:

```
k₁ = k / delta₁
k₂ = k / delta₂
⟹  k₁ · delta₁ = k₂ · delta₂  (both equal k)
```

Both `delta` values are known to the coordinator (they are derived from public inputs). Given two ECDSA signatures `(r₁, s₁)` and `(r₂, s₂)` with multiplicatively related nonces, the private key is recoverable by standard nonce-reuse algebra.

The security documentation acknowledges this exact risk but relies entirely on caller discipline:

> "Never reuse a presignature, even across failed, aborted, or partially completed signing sessions." [4](#0-3) 

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks." [5](#0-4) 

The library provides no type-level or runtime enforcement of single-use. `PresignOutput` is `Clone`, `rerandomize_presign` does not consume its argument, and there is no used-flag or session registry. This is the direct analog of the DCA bug: the code assumes the resource (presignature) is available and unused, without tracking in-flight commitments.

---

### Impact Explanation

A malicious coordinator who controls which `RerandomizationArguments` are distributed to participants can trigger two concurrent or sequential signing sessions that both consume the same `PresignOutput`. The two resulting signatures expose the aggregate private key via multiplicatively related nonces. This constitutes **Critical** impact: extraction of the aggregate secret signing material. [6](#0-5) 

---

### Likelihood Explanation

The coordinator role is a documented trust boundary in both OT-based and Robust ECDSA. A malicious coordinator is explicitly within the threat model. The attack requires no cryptographic break, no leaked keys, and no external dependency — only the ability to present two different `RerandomizationArguments` referencing the same `big_r` to the same set of participants. Because `PresignOutput` is `Clone` and `rerandomize_presign` takes `&PresignOutput`, participants have no way to detect or refuse the second use. [7](#0-6) 

---

### Recommendation

1. **Consume by value**: Change `rerandomize_presign` to take `presignature: PresignOutput` (owned), so the compiler enforces single-use at the call site.
2. **Remove `Clone` from `PresignOutput` and `RerandomizedPresignOutput`**: Removing the `Clone` derive prevents callers from duplicating presignatures before passing them.
3. **Consume in `sign()`**: The `sign()` function already takes `presignature: RerandomizedPresignOutput` by value; ensuring `RerandomizedPresignOutput` is not `Clone` closes the remaining gap. [8](#0-7)

### Proof of Concept

```
1. Run presigning: each participant P_i obtains PresignOutput { big_r: R, k: k_i, sigma: σ_i }.

2. Coordinator constructs args1 = RerandomizationArguments {
       msg_hash: h1, tweak: ε1, entropy: ρ1, big_r: R, ...
   }
   Each P_i calls rerandomize_presign(&presign_output, &args1)
   → RerandomizedPresignOutput { big_r: R·δ1, k: k_i/δ1, sigma: σ_i/δ1 }
   Signing session 1 completes → signature (r1, s1).

3. Coordinator constructs args2 = RerandomizationArguments {
       msg_hash: h2, tweak: ε2, entropy: ρ2, big_r: R, ...   ← same big_r
   }
   Each P_i calls rerandomize_presign(&presign_output, &args2)  ← same presign_output, not consumed
   → RerandomizedPresignOutput { big_r: R·δ2, k: k_i/δ2, sigma: σ_i/δ2 }
   Signing session 2 completes → signature (r2, s2).

4. Coordinator knows δ1, δ2 (derived from public inputs).
   From (r1,s1) and (r2,s2) with nonces k/δ1 and k/δ2:
       s1 = (h1 + r1·x) · (k/δ1)^{-1}  mod q
       s2 = (h2 + r2·x) · (k/δ2)^{-1}  mod q
   Two equations, two unknowns (k, x) → private key x recovered.
``` [7](#0-6) [1](#0-0)

    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
