### Title
Unbounded `stakers` Vec Growth Enables Attacker to Permanently DoS `get_stakers` via Gas Exhaustion - (File: src/staking/staking.cairo)

### Summary
An unprivileged attacker can register an arbitrarily large number of stakers (each with minimum stake), then unstake them all. Because staker addresses are pushed into a `Vec` that is **never pruned on unstake**, the `get_stakers` function must iterate over every historical staker address — including all inactive ones — causing unbounded gas consumption that can permanently render `get_stakers` uncallable.

### Finding Description
In `staking.cairo`, every call to `stake()` appends the new staker address to a persistent `stakers` storage vector: [1](#0-0) [2](#0-1) 

The comment at line 168 explicitly states: *"Stakers are not removed from this vector when they unstake."*

The `get_stakers` function (part of `IStakingConsensus`) iterates over the **full range** of this vector on every call: [3](#0-2) 

For each entry it calls `is_staker_active`, which reads storage, but the loop itself has no upper bound and no pruning. As the vector grows, the gas cost of `get_stakers` grows linearly and without ceiling.

### Impact Explanation
`get_stakers` is the consensus-layer entry point that returns the validator set for a given epoch. If it becomes uncallable due to gas exhaustion, consensus nodes cannot retrieve the staker list, breaking the liveness of the consensus mechanism. This constitutes **unbounded gas consumption** causing **damage to the protocol** — matching the Medium impact category.

### Likelihood Explanation
The attack is cheap to sustain: the attacker only needs to lock `min_stake` per address for one `exit_wait_window` (default 1 week), then recovers all funds. With a low `min_stake`, an attacker can register thousands of stakers at modest capital cost. The attack is fully permissionless — no privileged role is required — and is irreversible because the `stakers` Vec is never compacted.

### Recommendation
Remove staker addresses from the `stakers` Vec (or mark them as deleted with a tombstone) when `unstake_action` is called, so that `get_stakers` only iterates over currently-registered stakers. Alternatively, maintain a separate active-staker count and enforce a cap, or use a compactable data structure instead of an append-only `Vec`.

### Proof of Concept
1. Attacker controls N addresses, each pre-funded with `min_stake` STRK.
2. For each address, attacker calls `stake(reward_address, operational_address, min_stake)` — each call executes `self.stakers.push(staker_address)`.
3. After `exit_wait_window` elapses, attacker calls `unstake_intent()` then `unstake_action()` for each address, recovering all funds.
4. The `stakers` Vec now contains N entries, all inactive.
5. Any call to `get_stakers(epoch_id)` must iterate all N entries:
   ```
   for staker_address_ptr in self.stakers.into_iter_full_range() { ... }
   ```
6. For sufficiently large N, the transaction exceeds the Starknet gas limit and reverts, permanently preventing consensus nodes from querying the validator set. [4](#0-3)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
