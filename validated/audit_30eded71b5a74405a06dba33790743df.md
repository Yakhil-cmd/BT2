### Title
Front-Runnable `UpdatePermissions` Allows Permissioned Address to Corrupt Token Mapping Before Revocation - (File: x/cronos/keeper/msg_server.go)

### Summary
`MsgUpdatePermissions` directly overwrites the permission bitmask for an address via `SetPermissions`. When the Cronos admin reduces or revokes a `CanChangeTokenMapping` permission, the address being demoted can observe the pending revocation transaction in the public mempool and front-run it with a `MsgUpdateTokenMapping` call, mapping a legitimate IBC denom to a malicious contract before losing the right to do so.

### Finding Description
`UpdatePermissions` in `x/cronos/keeper/msg_server.go` calls `k.SetPermissions(ctx, acc, msg.Permissions)`, which unconditionally overwrites the stored permission bitmask:

```go
// msg_server.go
func (k msgServer) UpdatePermissions(...) {
    ...
    k.SetPermissions(ctx, acc, msg.Permissions)   // direct overwrite
    ...
}
```

`SetPermissions` itself performs a raw KV store write with no check against the current value:

```go
// permissions.go
func (k Keeper) SetPermissions(ctx sdk.Context, address sdk.AccAddress, permissions uint64) {
    store := ctx.KVStore(k.storeKey)
    permissionsBytes := sdk.Uint64ToBigEndian(permissions)
    store.Set(types.AdminToPermissionsKey(address), permissionsBytes)
}
```

There is no atomic "revoke-and-act" primitive. The admin must submit a standalone `MsgUpdatePermissions` transaction, which is visible in the public mempool before it is included in a block.

A holder of `CanChangeTokenMapping` (value `1`) who sees a pending revocation (new value `0` or `CanTurnBridge=2`) can submit a `MsgUpdateTokenMapping` transaction with higher gas priority in the same block, mapping a legitimate IBC denom to an attacker-controlled contract before the revocation lands. Because `SetExternalContractForDenom` enforces a **create-only** guard (`ensureDenomNotMapped`), the mapping, once written, cannot be overwritten by the admin without first deleting it — and the delete+re-register two-step itself opens a second race window.

### Impact Explanation
Once a legitimate IBC denom (e.g., `ibc/C4CFF46FD6DE35CA4CF4CE031E643C8FDC9BA4B99AE598E9B0ED98FE3A2319F9`) is mapped to an attacker-controlled contract, every subsequent IBC packet received for that denom triggers `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21`, which calls `mint_by_cronos_module` on the malicious contract. The malicious contract can mint CRC20 tokens to arbitrary addresses, constituting an unauthorized mint of CRC20/IBC-voucher-backed assets. This matches the **High** impact category: corruption of token mappings with direct security impact (unauthorized minting of CRC20 tokens backed by IBC vouchers).

### Likelihood Explanation
The Cosmos mempool is public. Any address that currently holds `CanChangeTokenMapping` and monitors the mempool can detect a pending `MsgUpdatePermissions` targeting their address. Submitting a competing `MsgUpdateTokenMapping` with a higher gas price is straightforward. The admin has no atomic primitive to simultaneously revoke the permission and prevent the race.

### Recommendation
Replace the direct overwrite in `UpdatePermissions` with a compare-and-set or delta-based approach:
- Require the caller to supply the **expected current** permission value; reject if it does not match (optimistic concurrency).
- Or provide separate `GrantPermission` / `RevokePermission` messages that add/remove individual bits rather than replacing the whole bitmask, so a revocation of `CanChangeTokenMapping` cannot be raced against a `MsgUpdateTokenMapping` in the same block.
- Alternatively, allow the admin to bundle a `MsgUpdatePermissions` and a `MsgUpdateTokenMapping` (or mapping deletion) in a single atomic transaction to close the window.

### Proof of Concept
1. Admin grants address A `CanChangeTokenMapping` (permissions = 1) via `MsgUpdatePermissions`.
2. Admin later decides to revoke A's permission and submits `MsgUpdatePermissions(address=A, permissions=0)`.
3. A monitors the mempool, sees the pending revocation, and immediately submits `MsgUpdateTokenMapping(denom="ibc/ATOM...", contract=maliciousContract)` with higher gas.
4. A's `MsgUpdateTokenMapping` is ordered before the admin's revocation in the same block. `SetExternalContractForDenom` succeeds because the denom was previously unmapped.
5. The admin's revocation lands; A now has no permissions.
6. Every subsequent IBC ATOM transfer to Cronos triggers `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` on `maliciousContract`, which mints CRC20 tokens to A's address.
7. A has permanently corrupted the ATOM token mapping and can drain all future IBC ATOM inflows as unauthorized CRC20 mints. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

```go
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
```

```go
func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
```

```go
// SetExternalContractForDenom sets denom→external CRC21 mapping for an unmapped denom.
// Caller is responsible for source-denom specific validation before calling this method.
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
```
