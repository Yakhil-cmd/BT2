### Title
Gateway Stateful Validator Skips L1/L1-Data Gas Price Admission Check, Admitting Transactions the Blockifier Rejects - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `validate_resource_bounds` only validates the transaction's L2 gas price against the previous block's L2 gas price. It does not check L1 gas price or L1 data gas price. The blockifier's `check_fee_bounds` checks all three gas prices. A transaction with a valid L2 gas price but zero (or below-market) L1 gas price passes gateway admission but fails blockifier execution, polluting the mempool with permanently-unexecutable transactions.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, `validate_tx_l2_gas_price_within_threshold` only validates the L2 gas price:

```rust

    &self,
    tx_resource_bounds: ValidResourceBounds,
    previous_block_l2_gas_price: NonzeroGasPrice,
) -> StatefulTransactionValidatorResult<()> {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // ... only checks L2 gas price against threshold
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
    Ok(())
}

The stateless validator in `crates/apollo_gateway/src/stateless_transaction_validator.rs` also only checks L2 gas price:

In contrast, the blockifier's `check_fee_bounds` in `crates/blockifier/src/transaction/account_transaction.rs` checks **all three** gas prices for `AllResources` transactions:

```rust
ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_gas_resource_bounds,
    l2_gas: l2_gas_resource_bounds,
    l1_data_gas: l1_data_gas_resource_bounds,
}) => {
    let GasPriceVector { l1_gas_price, l1_data_gas_price, l2_gas_price } =
        block_info.gas_prices.gas_price_vector(fee_type);
    vec![
        (L1Gas, l1_gas_resource_bounds, minimal_gas_amount_vector.l1_gas, *l1_gas_price),
        (L1DataGas, l1_data_gas_resource_bounds, minimal_gas_amount_vector.l1_data_gas, *l1_data_gas_price),
        (L2Gas, l2_gas_resource_bounds, minimal_gas_amount_vector.l2_gas, *l2_gas_price),
    ]
}
``` [3](#0-2) 

The price check applied to each resource is unconditional:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [4](#0-3) 

This check fires regardless of `max_amount`. A transaction with `l1_gas.max_price_per_unit = 0` will always fail this check because `actual_l1_gas_price` is a `NonzeroGasPrice` (always > 0).

The `validate_resource_bounds` call chain in the stateful validator reads only `strk_gas_prices.l2_gas_price` from the previous block:

```rust
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [5](#0-4) 

The `l1_gas_price` and `l1_data_gas_price` fields of the previous block are never read for admission purposes.

**Broken invariant:** Every transaction admitted to the mempool by the gateway must satisfy the blockifier's `check_fee_bounds` at the current block's gas prices. This invariant is violated because the gateway only enforces the L2 gas price dimension while the blockifier enforces all three.

**Exact corrupted value:** The gateway's admission decision — it returns "accepted" for a transaction where `l1_gas.max_price_per_unit < actual_l1_gas_price`, a condition the blockifier will unconditionally reject.

### Impact Explanation

A transaction with `l2_gas.max_price_per_unit >= min_gas_price` (passes gateway) but `l1_gas.max_price_per_unit = 0` (fails blockifier) passes both stateless and stateful gateway validation, enters the mempool, and is permanently unexecutable. An unprivileged attacker can flood the mempool with such transactions at zero cost (no gateway submission fee), causing mempool pollution and wasting batcher resources during block building as the blockifier rejects each one with `MaxGasPriceTooLow` for `L1Gas`.

This matches the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

The attack is trivial to construct. The stateless validator's zero-fee guard is bypassed because `l2_gas.max_amount * l2_gas.max_price_per_unit > 0`. The stateful validator's threshold check is bypassed because it only reads `l2_gas.max_price_per_unit`. No privileged access is required. The TODO comment in the production code (`// TODO(Arni): Consider running this validation for all gas prices.`) confirms the gap is known but unmitigated. [6](#0-5) 

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or introduce a parallel function) to also check `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's `strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price` respectively, using the same percentage-threshold logic already applied to L2 gas. This mirrors what the blockifier enforces and closes the admission gap.

### Proof of Concept

1. Craft an `AllResources` V3 invoke transaction with:
   - `l2_gas.max_price_per_unit = min_gas_price` (passes stateless `MaxGasPriceTooLow` check)
   - `l2_gas.max_amount = 1` (makes `max_possible_fee > 0`, bypassing zero-fee guard)
   - `l1_gas.max_price_per_unit = 0` (below any real L1 gas price; not checked by gateway)
   - `l1_gas.max_amount = 0`, `l1_data_gas.*= 0`
2. Submit to the gateway's `add_tx` endpoint.
3. The stateless validator accepts: total fee = `1 * min_gas_price > 0`; L2 gas price ≥ `min_gas_price`. [7](#0-6) 
4. The stateful validator accepts: `l2_gas.max_price_per_unit >= threshold * previous_block_l2_gas_price`; L1 gas price not checked. [8](#0-7) 
5. Transaction enters the mempool.
6. The batcher dequeues the transaction and passes it to the blockifier.
7. `perform_pre_validation_stage` → `check_fee_bounds` fires: `0 < actual_l1_gas_price` → `MaxGasPriceTooLow` for `L1Gas`. [9](#0-8) 
8. The transaction is permanently unexecutable; it occupies mempool space until eviction.

This is the direct sequencer analog of the WstEth `withdraw` bug: just as WstEth computes `minOut` using only the stETH balance (assuming 1 stETH = 1 ETH and ignoring the ETH/stETH market rate), the gateway computes its admission decision using only the L2 gas price (assuming L1 gas price validity follows from L2 gas price validity), ignoring the independent L1 gas price dimension that the blockifier enforces.
