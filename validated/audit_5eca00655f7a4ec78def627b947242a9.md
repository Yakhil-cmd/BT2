### Title
Staker Can Instantly Raise Commission to 100% via Commitment Mechanism, Stealing Delegator Epoch Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

A staker can call `set_commission_commitment` to authorize a `max_commission` of 10000 (100%), then immediately call `set_commission(10000)` in the same block. Because commission changes take effect in storage with no time-lock and `calculate_staker_pools_rewards` reads the live commission value at reward-distribution time, the staker captures 100% of pool rewards as commission for that epoch, leaving delegators with zero yield.

The developers themselves acknowledge this gap in a code comment directly above `set_commission_commitment`:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [1](#0-0) 

---

### Finding Description

**Step 1 — Commitment creation (no delay).**
`set_commission_commitment` stores a `CommissionCommitment { max_commission, expiration_epoch }` immediately. The only constraints are that `max_commission >= current_commission`, `expiration_epoch > current_epoch`, and `expiration_epoch - current_epoch <= epochs_in_year()`. There is no waiting period before the commitment becomes usable. [2](#0-1) 

**Step 2 — Instant commission increase.**
`set_commission` calls the internal `update_commission`. When a commitment is active, the only checks are `commission <= commitment.max_commission` and `commission != old_commission`. If `max_commission = 10000`, the staker can set commission to 10000 in the very next transaction. The new value is written to storage immediately:

```
staker_pool_info.commission.write(Option::Some(commission));
``` [3](#0-2) 

**Step 3 — Commission read at reward-distribution time.**
`calculate_staker_pools_rewards` (called by both `update_rewards_from_attestation_contract` and `update_rewards`) reads the commission from live storage at the moment rewards are split:

```cairo
let commission = staker_pool_info.commission();
...
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [4](#0-3) 

With commission = 10000, `split_rewards_with_commission` returns `commission_rewards = total_pool_rewards` and `pool_rewards = 0`. All delegator yield is redirected to the staker. [5](#0-4) 

**Step 4 — Staker resets commission after the attack.**
Without a commitment (or after it expires), `set_commission` only allows decreases (`commission < old_commission`). The staker can lower commission back to an attractive rate to retain delegators, then repeat the attack in a future epoch. [6](#0-5) 

---

### Impact Explanation

Every delegator in the staker's pool receives **zero STRK rewards** for any epoch in which the staker executes this attack. The staker receives the full pool reward as commission. This is direct theft of unclaimed yield from delegators. The attack can be repeated every epoch as long as the staker renews the commitment (which is trivially possible since `max_commission >= current_commission` is always satisfiable when `current_commission = 10000`).

**Impact: High — Theft of unclaimed yield.**

---

### Likelihood Explanation

- The staker is an unprivileged, self-interested actor with full control over when to call `set_commission_commitment` and `set_commission`.
- Both calls can be made in the same block, or even in the same transaction via a wrapper contract, atomically before the attestation window opens.
- The epoch structure is public and predictable; the staker can time the attack precisely.
- No external dependency, oracle, or privileged role is required.
- The developer note at line 745–747 confirms the team is aware the safeguard is incomplete.

**Likelihood: High.**

---

### Recommendation

Apply a time-lock to commission increases analogous to the fix in Celo PR #2913: when `set_commission` would result in a commission higher than the previous value, queue the change and only apply it after at least one full epoch has elapsed. This gives delegators time to observe the pending increase and exit before it takes effect. Alternatively, prohibit commission increases entirely (only allow decreases without a commitment), which is the current behavior when no commitment exists.

---

### Proof of Concept

```
Epoch N, block B (before attestation window):

1. staker calls set_commission_commitment(max_commission=10000, expiration_epoch=N+1)
   → CommissionCommitment { max_commission: 10000, expiration_epoch: N+1 } stored immediately.

2. staker calls set_commission(commission=10000)
   → update_commission checks: 10000 <= 10000 ✓, 10000 != old_commission ✓
   → staker_pool_info.commission = 10000 written to storage.

Epoch N, attestation block:

3. Attestation contract calls update_rewards_from_attestation_contract(staker_address)
   → calculate_staker_pools_rewards reads commission = 10000
   → split_rewards_with_commission(pool_rewards_including_commission, 10000)
     → commission_rewards = pool_rewards_including_commission * 10000 / 10000 = total
     → pool_rewards = 0
   → Staker receives all pool rewards; delegation pool receives 0.

4. Delegators call claim_rewards → receive 0 for epoch N.

Epoch N+1:

5. staker calls set_commission(commission=200)  // decrease allowed without commitment
   → Commission back to 2%, delegators stay.

6. Repeat from step 1 in any future epoch.
``` [7](#0-6) [8](#0-7)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
