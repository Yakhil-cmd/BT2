### Title
Unprivileged caller can permanently freeze consensus rewards by calling `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` writes the global `last_reward_block` checkpoint **unconditionally** before checking the `disable_rewards` flag. Because the function has no access control beyond "caller is not zero," any unprivileged actor can call it with `disable_rewards: true` in every block, consuming each block's reward slot without distributing any rewards, permanently starving all stakers of consensus-era yield.

---

### Finding Description

`update_rewards` is the entry point for consensus-era (V3) per-block reward distribution. Its execution order is:

1. Assert `current_block_number > last_reward_block` — ensures only one reward update per block.
2. Assert staker exists, is active, and has non-zero STRK balance.
3. **Write `last_reward_block = current_block_number`** — unconditionally, at line 1485.
4. `if disable_rewards || self.is_pre_consensus() { return; }` — early exit with no rewards distributed. [1](#0-0) 

The critical ordering flaw: `last_reward_block` is committed to storage at step 3 **before** the `disable_rewards` guard at step 4. Once `last_reward_block` equals the current block, the assertion at step 1 will reject any further call in the same block — including a legitimate call from the staker.

The function carries no role check beyond `general_prerequisites()`, which only verifies the contract is unpaused and the caller is non-zero: [2](#0-1) 

The analog to the reference report is exact: the reference code checks `trove_ltv > MAX_PENALTY_LTV` but omits the complementary bound `threshold < MAX_PENALTY_LTV`, allowing a healthy loan to be treated as insolvent. Here, the code checks `current_block > last_reward_block` but omits the complementary guard that `disable_rewards` may only be set by a privileged caller, allowing a block's reward slot to be consumed without distributing rewards.

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block:

- Advances `last_reward_block` to the current block each time.
- Causes the early return before any STRK is credited to stakers or pools.
- Blocks every legitimate `update_rewards` call in that block with `REWARDS_ALREADY_UPDATED`.

Sustained over time this permanently freezes all unclaimed consensus-era yield for every staker and every delegation pool. This matches the **High** impact: *Permanent freezing of unclaimed yield*. [3](#0-2) 

---

### Likelihood Explanation

- The function is publicly callable with no role restriction.
- The attacker only needs to supply any currently-active staker address (readable from on-chain events or `get_stakers`).
- The cost is one transaction per block; on Starknet L2 this is cheap enough to sustain indefinitely.
- No special knowledge or privileged access is required.

---

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so the block slot is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Update last block rewards only when rewards are actually distributed.
self.last_reward_block.write(current_block_number);

// ... calculate and distribute rewards
```

Alternatively, restrict the `disable_rewards: true` path to a privileged role (e.g., `security_agent`) so an unprivileged caller cannot invoke it.

---

### Proof of Concept

1. Staker Alice stakes STRK and the protocol enters the consensus-rewards epoch.
2. Attacker Eve, in every new block, calls:
   ```
   staking.update_rewards(alice_address, disable_rewards: true)
   ```
3. Each call passes all assertions (Alice is a valid active staker with non-zero balance), writes `last_reward_block = current_block`, then returns early.
4. Alice (or anyone else) attempts `update_rewards(alice_address, disable_rewards: false)` in the same block — the call reverts with `REWARDS_ALREADY_UPDATED`.
5. No STRK is ever credited to `staker_info.unclaimed_rewards_own` or forwarded to any delegation pool.
6. Alice's yield is permanently frozen as long as Eve continues the attack. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
