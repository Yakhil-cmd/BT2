### Title
Excess Native ETH Not Returned to Swapper After WETH-Input Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`PeripheryPayments.pay()` wraps only the exact amount of ETH the pool requests and leaves any surplus native ETH in the router. Because `refundETH()` sends the entire router ETH balance to whoever calls it, a user who sends more ETH than the swap consumes (the normal pattern for exact-output swaps) loses the excess to any third party who calls `refundETH()` first.

---

### Finding Description

`PeripheryPayments.pay()` handles the WETH leg of a swap as follows: [1](#0-0) 

```solidity
} else if (token == WETH) {
    uint256 nativeBalance = address(this).balance;
    if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();   // wraps exactly `value`
        IERC20(WETH).safeTransfer(recipient, value);
    } else if (nativeBalance > 0) {
        ...
    }
}
```

When `nativeBalance >= value`, only `value` wei is wrapped and forwarded to the pool. The remainder (`nativeBalance - value`) is silently left in the router.

None of the four swap entry-points — `exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput` — call `refundETH()` before returning: [2](#0-1) 

The only recovery path is `refundETH()`, which is a public function that sends the **entire** router ETH balance to `msg.sender`: [3](#0-2) 

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
        _transferETH(msg.sender, balance);   // sends to whoever calls, not the original swapper
    }
}
```

There is no access control and no association between the deposited ETH and the original sender. Any address that calls `refundETH()` after the victim's swap transaction is confirmed receives the stranded ETH.

---

### Impact Explanation

A user executing an exact-output WETH swap sends up to `amountInMaximum` in native ETH. The pool callback pulls only the actual `amountIn ≤ amountInMaximum`. The delta `amountInMaximum − amountIn` remains in the router. If the user does not bundle a `refundETH()` call in the same multicall transaction, any third party can immediately call `refundETH()` and receive the surplus. This is a **direct, permanent loss of user principal** with no protocol-side recovery mechanism.

---

### Likelihood Explanation

Exact-output swaps are a standard pattern: users routinely set `amountInMaximum` above the expected cost to absorb price movement, exactly as described in the SpeedBumpPriceGate report. The `refundETH()` function is public, permissionless, and well-known to MEV searchers. Any unrefunded ETH left in the router between the swap call and the next block is extractable by a bot. Users who call `exactOutputSingle` or `exactOutput` directly (not via multicall) are fully exposed.

---

### Recommendation

After the swap callback settles, automatically refund any remaining native ETH balance to the original caller. The simplest fix is to add a refund step at the end of each payable swap function:

```solidity
// inside exactOutputSingle / exactOutput / exactInputSingle / exactInput
// after _clearExpectedCallbackPool():
uint256 surplus = address(this).balance;
if (surplus > 0) {
    _transferETH(msg.sender, surplus);
}
```

Alternatively, enforce that `msg.value` equals the exact amount consumed (revert if `msg.value > amountIn`), or document that callers **must** append a `refundETH()` call and enforce this in the interface.

---

### Proof of Concept

1. Pool has a WETH/token1 pair. Current oracle price implies `exactOutputSingle` for 1 token1 costs 0.5 WETH.
2. Alice calls `exactOutputSingle` with `amountOut = 1e18`, `amountInMaximum = 1 ETH`, sending `msg.value = 1 ETH`.
3. The pool callback fires; `pay(WETH, Alice, pool, 0.5e18)` is called. `nativeBalance = 1 ETH ≥ value = 0.5 ETH`, so 0.5 ETH is wrapped and sent to the pool. The remaining 0.5 ETH stays in the router.
4. `exactOutputSingle` returns. Alice received her token1. The router holds 0.5 ETH.
5. Bob (a bot) calls `router.refundETH()` in the next block. The router's entire 0.5 ETH balance is transferred to Bob.
6. Alice has lost 0.5 ETH with no recourse.

The test file confirms `refundETH()` sends the full balance to any caller: [4](#0-3)
