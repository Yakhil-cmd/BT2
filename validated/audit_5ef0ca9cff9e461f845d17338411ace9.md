### Title
Unbounded `pool_member_epoch_balance` Trace Growth Enables Permanent Freezing of Delegator Unclaimed Rewards - (File: `src/pool/pool.cairo`)

---

### Summary

A pool member (delegator) can grow their `pool_member_epoch_balance` trace without bound by making one balance-changing call per epoch (`add_to_delegation_pool` or `exit_delegation_pool_intent`). The `calculate_rewards` function in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop that iterates over every trace entry since the last claim. If a delegator accumulates enough entries without claiming rewards, the loop will exceed the Starknet block gas limit, permanently freezing their unclaimed yield.

---

### Finding Description

The `calculate_rewards` internal function iterates over the entire `pool_member_epoch_balance` trace between the delegator's last reward checkpoint and the current epoch:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The trace is stored as a `Vec<PoolMemberBalanceCheckpoint>` keyed by epoch. The `insert` function appends a **new** checkpoint whenever the epoch key differs from the last stored key:

```cairo
if last.key == key {
    last.value = value;
    checkpoints[len - 1].write(last);
} else {
    checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
}
``` [2](#0-1) 

Every call to `add_to_delegation_pool` or `exit_delegation_pool_intent` writes to the trace at key `current_epoch + K`:

```cairo
fn set_member_balance(ref self: ContractState, pool_member: ContractAddress, amount: Amount) {
    let trace = self.pool_member_epoch_balance.entry(pool_member);
    ...
    trace.insert(key: self.get_epoch_plus_k(), value: pool_member_balance);
}
``` [3](#0-2) 

Because the key advances by one each epoch, a delegator who makes one balance-changing call per epoch adds exactly one new checkpoint per epoch. There is no cap on the trace length and no minimum amount enforced beyond `amount > 0`.

The `entry_to_claim_from` cursor (saved in `pool_member_info`) advances only when `claim_rewards` succeeds:

```cairo
pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
pool_member_info.reward_checkpoint = until_checkpoint;
``` [4](#0-3) 

If the delegator never calls `claim_rewards` while accumulating entries, the cursor stays at zero. When `claim_rewards` is eventually called, the loop must process every accumulated entry in a single transaction. Once the entry count is large enough to exhaust the block gas limit, the transaction reverts every time, and the delegator's rewards are permanently frozen.

---

### Impact Explanation

**Permanent freezing of unclaimed yield (High).**

A delegator whose trace has grown beyond the gas-processable threshold can never successfully call `claim_rewards`. The rewards are locked in the pool contract with no recovery path: there is no partial-claim mechanism, no admin escape hatch, and no way to prune the trace. The delegator's accrued STRK rewards are permanently inaccessible.

---

### Likelihood Explanation

**Medium-to-High likelihood over the protocol's lifetime.**

- The attack requires only one `add_to_delegation_pool` call per epoch with a minimal token amount (1 unit). No special privilege is needed.
- The attacker is the delegator themselves (self-griefing) or a third party who front-runs the delegator's first deposit and then repeatedly tops up on their behalf (the `add_to_delegation_pool` caller can be the `reward_address`, not just the pool member).
- The protocol is designed to run indefinitely. Over hundreds or thousands of epochs, a delegator who simply never claims rewards accumulates entries organically, eventually hitting the gas ceiling without any deliberate attack.
- The codebase itself acknowledges the risk: *"This loop is unbounded but unlikely to exceed gas limits."* This is an optimistic assumption that fails for long-lived delegators.

---

### Recommendation

1. **Enforce a maximum trace length** by capping the number of entries per pool member (e.g., reject `add_to_delegation_pool` / `exit_delegation_pool_intent` if the trace already has more than `MAX_TRACE_LEN` entries ahead of `entry_to_claim_from`).
2. **Require periodic reward claims**: reject balance-changing operations if the delegator's unclaimed trace window exceeds a configurable epoch threshold, forcing them to claim before adding more entries.
3. **Implement partial/paginated reward claiming**: allow `claim_rewards` to accept a `max_entries` parameter and process only a bounded number of entries per call, advancing `entry_to_claim_from` incrementally.

---

### Proof of Concept

1. Delegator calls `enter_delegation_pool` with `amount = MIN_AMOUNT`.
2. Each epoch, delegator calls `add_to_delegation_pool(pool_member, amount: 1)`. Each call inserts a new checkpoint at `current_epoch + K` (a distinct key each epoch), growing the trace by one entry.
3. After `N` epochs without calling `claim_rewards`, the trace has `N` entries and `entry_to_claim_from = 0`.
4. Delegator (or anyone) calls `claim_rewards(pool_member)`. The `calculate_rewards` loop must iterate `N` times, each iteration performing storage reads via `pool_member_trace.at(entry_to_claim_from)` and `find_sigma(...)`.
5. For sufficiently large `N` (determined by Starknet's per-transaction gas limit divided by the per-iteration gas cost), the transaction reverts with an out-of-gas error.
6. Every subsequent `claim_rewards` call also reverts. The delegator's accumulated STRK rewards are permanently frozen in the pool contract. [5](#0-4) [6](#0-5) [7](#0-6)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

#[generate_trait]
pub impl MutablePoolMemberBalanceTraceImpl of MutablePoolMemberBalanceTraceTrait {
    /// Inserts a (`key`, `value`) pair into a Trace so that it is stored as the checkpoint.
    /// This is done by either inserting a new checkpoint, or updating the last one.
    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
