### Title
Execution Effort Truncates to Zero for Small EVM Gas Transactions, Enabling Fee Evasion - (`fvm/meter/computation_meter.go`)

### Summary

The FVM computation meter uses a fixed-point internal precision scheme where all accumulated computation is right-shifted by 16 bits (`MeterExecutionInternalPrecisionBytes = 16`) before being reported as execution effort. When the product of a weight and its intensity is less than 65536, `TotalComputationUsed()` returns 0. The `MainnetExecutionEffortWeights` table assigns `ComputationKindEVMGasUsage` a weight of **3**, meaning any EVM transaction consuming fewer than 21,846 EVM gas units reports zero execution effort. A simple EVM ETH transfer uses ~21,000 gas — below this threshold — so its execution effort fee component is always zero.

### Finding Description

`TotalComputationUsed()` in `fvm/meter/computation_meter.go` computes the externally visible execution effort by right-shifting the raw internal accumulator:

```go
func (m *ComputationMeter) TotalComputationUsed() uint64 {
    return m.computationUsed >> MeterExecutionInternalPrecisionBytes  // >> 16
}
``` [1](#0-0) 

The raw accumulator is built by `MeterComputation`:

```go
m.computationUsed += w * intensity
``` [2](#0-1) 

For the result to be non-zero after the right-shift, `w * intensity` must be ≥ 65536. The `MainnetExecutionEffortWeights` table assigns:

```go
ComputationKindEVMGasUsage: 3,
``` [3](#0-2) 

With weight 3, the threshold is `65536 / 3 = 21845` EVM gas units. Any EVM transaction consuming ≤ 21845 gas produces `3 * gas < 65536`, which right-shifts to 0.

This zero value is then passed directly to `DeductTransactionFees` as the `executionEffort` argument:

```go
computationUsed, err := executor.env.ComputationUsed()
...
_, err = executor.env.DeductTransactionFees(
    executor.proc.Transaction.Payer,
    executor.proc.Transaction.InclusionEffort(),
    computationUsed)
``` [4](#0-3) 

The on-chain fee contract computes: `fee = surgeFactor * (inclusionEffortCost * inclusionEffort + executionEffortCost * executionEffort)`. With `executionEffort = 0`, the execution effort cost component is entirely waived.

The existing test suite explicitly documents and accepts this truncation behavior:

```go
// meter computation with weights lower than MeterInternalPrecisionBytes
require.Equal(t, uint64(0), m.TotalComputationUsed())
``` [5](#0-4) 

### Impact Explanation

Any unprivileged user submitting a Flow EVM transaction that consumes fewer than 21,846 EVM gas units pays zero execution effort fees. A standard EVM ETH transfer costs exactly 21,000 gas — below the threshold — so its execution effort fee is always zero. This is a systematic, repeatable fee evasion for the most common class of EVM transactions on Flow. The execution effort cost is a non-zero protocol parameter on mainnet (set via `SetupFeesTransaction`), so the waived amount is real FLOW value.

### Likelihood Explanation

High. Every simple EVM token transfer or ETH transfer (21,000 gas) automatically triggers this condition with no special setup. The attacker only needs to submit ordinary EVM transactions. No privileged access, leaked keys, or social engineering is required. The condition is structural and deterministic.

### Recommendation

Scale small weights before storing them, or accumulate the raw `weight * intensity` product and only apply the precision shift at the point where the fee contract receives the value — not at the metering layer. Alternatively, enforce a minimum effective weight of `1 << MeterExecutionInternalPrecisionBytes` (65536) for any weight entry in `MainnetExecutionEffortWeights`, or add a rounding-up step in `TotalComputationUsed()`:

```go
func (m *ComputationMeter) TotalComputationUsed() uint64 {
    // Round up instead of truncating
    used := m.computationUsed
    if used > 0 {
        return (used + (1<<MeterExecutionInternalPrecisionBytes - 1)) >> MeterExecutionInternalPrecisionBytes
    }
    return 0
}
```

### Proof of Concept

With `ComputationKindEVMGasUsage` weight = 3 and a standard EVM ETH transfer consuming 21,000 gas:

```
internal_computation = 3 * 21_000 = 63_000
TotalComputationUsed = 63_000 >> 16 = 0   // integer truncation
executionEffort passed to fee contract = 0
executionEffortFee = executionEffortCost * 0 = 0 FLOW
```

The payer pays only the inclusion effort cost, regardless of the actual EVM work performed. Any EVM transaction under 21,846 gas (the vast majority of simple transfers and token approvals) is permanently exempt from execution effort fees. [6](#0-5) [7](#0-6)

// MeterExecutionInternalPrecisionBytes are the amount of bytes that are used internally by the
// WeigthedMeter to allow for metering computation smaller than one unit of computation.
// This allows for more fine weights. A weight of 1 unit of computation is equal to 1<<16.
// The minimum possible weight is 1/65536.
const MeterExecutionInternalPrecisionBytes = 16

type ExecutionEffortWeights map[common.ComputationKind]uint64

func (weights ExecutionEffortWeights) ComputationFromIntensities(intensities MeteredComputationIntensities) uint64 {
	var result uint64
	for kind, weight := range weights {
		intensity := uint64(intensities[kind])
		result += weight * intensity
	}
	return result >> MeterExecutionInternalPrecisionBytes
```
