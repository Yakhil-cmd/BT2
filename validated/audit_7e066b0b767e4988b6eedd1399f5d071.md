### Title
Unprivileged Account Can Overwrite Any Account's `GlobalContractCode` Trie Entry via Cross-Account `DeployGlobalContractByAccountId` Promise — (`runtime/runtime/src/global_contracts.rs`)

### Summary
Account A can create a promise targeting account B and append a `DeployGlobalContract { deploy_mode: AccountId }` action to it. When the resulting receipt executes on B's shard, the runtime writes attacker-supplied code to `TrieKey::GlobalContractCode { identifier: AccountId(B) }` using B's `account_id` as the key — because `action_deploy_global_contract` derives the trie key from the receipt's `receiver_id`, not from the `predecessor_id`. There is no `predecessor_id == receiver_id` guard for `DeployGlobalContract`, unlike `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, and `DeleteAccount`.

### Finding Description

The host function `promise_batch_action_deploy_global_contract_by_account_id` appends a `DeployGlobalContractAction { deploy_mode: AccountId }` to any receipt index returned by `promise_idx_to_receipt_idx_with_sir`. That receipt can target an arbitrary account B (created via `promise_batch_create("b.near")`). The `sir` flag only affects gas pricing; it does not gate the action.

When the receipt executes, `apply_action` calls `check_actor_permissions`, which does **not** require `predecessor_id == receiver_id` for `DeployGlobalContract`. It then calls `action_deploy_global_contract(state_update, account_B, account_id=B, ...)`.

Inside `action_deploy_global_contract`, `initiate_distribution` is called with `account_id = B`:

```rust
let id = match deploy_mode {
    GlobalContractDeployMode::AccountId => {
        GlobalContractIdentifier::AccountId(account_id.clone())  // = B
    }
};
```

This produces `TrieKey::GlobalContractCode { identifier: AccountId(B) }` and overwrites it with A's supplied code. B's balance is also charged for the storage cost. [7](#0-6) 

### Impact Explanation

The exact corrupted ledger value is `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(B) }`. After the attack:

- Every account that references `GlobalContractIdentifier::AccountId(B)` via `UseGlobalContract` or `GlobalByAccount` now executes A's malicious code instead of B's legitimate code.
- B's balance is drained by the storage cost without B's consent.
- B's previously deployed global contract (if any) is silently replaced.
- The `GlobalContractNonce` for `AccountId(B)` is incremented by A, which can also suppress B's own future legitimate re-deployments if the nonce check is used for idempotency. [8](#0-7) 

### Likelihood Explanation

Any account with a deployed contract can execute this attack in a single transaction. No special privileges are required. The only precondition is that B is an existing account with sufficient balance to cover the storage cost (which is true for any active account). The attack is cheap and deterministic.

### Recommendation

Add a `predecessor_id == receiver_id` check for `DeployGlobalContract` with `deploy_mode: AccountId`, analogous to the existing guard for `DeployContract`. In `apply_action` or `action_deploy_global_contract`, verify that `receipt.predecessor_id() == account_id` (or equivalently that `actor_id == account_id`) before allowing the `AccountId`-mode deployment. The `CodeHash` mode does not need this restriction since the key is content-addressed and cannot be hijacked.

### Proof of Concept

1. Deploy a malicious contract on account `attacker.near`.
2. The contract's method calls:

3. Submit a transaction calling that method from `attacker.near`.
4. After execution, read `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }` from the trie.
5. Assert it contains `malicious_code`, not `victim.near`'s original code.
6. Assert that any account using `GlobalContractIdentifier::AccountId("victim.near")` now executes `malicious_code`.
7. Assert `victim.near`'s balance was reduced by the storage cost. [1](#0-0) [5](#0-4) [9](#0-8)

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())
