### Title
Gateway Stateful Validator Uses Stale L2 Gas Price for Admission Check — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` checks a transaction's `max_price_per_unit` against the **current (already-committed) block's** L2 gas price, but the transaction will be executed in the **next block** at the **next block's** L2 gas price. The two values diverge under EIP-1559 dynamics. The developer left an explicit `TODO` acknowledging the wrong field is used. This causes the gateway to admit transactions that will fail at batcher execution and to reject transactions that would succeed.

---

### Finding Description

In `validate_resource_bounds`, the gateway reads the L2 gas price from the latest committed block's `BlockInfo`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `BlockInfo` from `block_header.l2_gas_price.price_in_fri` — the price **used in** the current block:

```rust
strk_gas_prices: GasPriceVector {
    ...
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
},
``` [2](#0-1) 

The `BlockHeaderWithoutHash` struct carries a separate `next_l2_gas_price` field — the price that will govern the **next** block — which is never read here. This field is populated by the consensus orchestrator via `calculate_next_l2_gas_price_for_fin` / `update_l2_gas_price` and stored in the block header: [3](#0-2) 

The same stale `BlockInfo` is also forwarded into `run_validate_entry_point`, which increments only the block number but leaves the gas prices unchanged:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
``` [4](#0-3) 

So both the early admission check and the blockifier `check_fee_bounds` inside the gateway use the current block's L2 gas price, while the batcher will execute the transaction against the next block's L2 gas price.

---

### Impact Explanation

Under EIP-1559 dynamics, `next_l2_gas_price` diverges from `l2_gas_price` whenever gas usage deviates from the target:

**Case A — next block price rises (congested block):**
A transaction with `max_price_per_unit = current_block_price` passes `validate_tx_l2_gas_price_within_threshold` (threshold = 100 % × current price) and passes the gateway's blockifier `check_fee_bounds`. It is admitted to the mempool. At batcher execution, `check_fee_bounds` compares against `next_block_price > current_block_price` and rejects the transaction with `MaxGasPriceTooLow`. The gateway admitted an invalid transaction.

**Case B — next block price falls (under-utilised block):**
A transaction with `max_price_per_unit` between `next_block_price` and `current_block_price` fails `validate_tx_l2_gas_price_within_threshold` at the gateway and is rejected. It would have passed `check_fee_bounds` at batcher execution. The gateway rejected a valid transaction.

Both cases match the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The `gas_price_max_change_denominator` versioned constant bounds the per-block price change, so a single block's discrepancy is small. However, under sustained congestion or sustained under-utilisation the gap compounds across consecutive blocks. With `min_gas_price_percentage = 100` (the production default), the threshold is exactly the current block's price, so any upward movement in the next block's price creates a window of admitted-but-invalid transactions. An unprivileged user can trigger Case A simply by submitting a transaction priced at the current block's L2 gas price during a period of rising congestion.

---

### Recommendation

Replace the read of `block_header.l2_gas_price.price_in_fri` with `block_header.next_l2_gas_price` in `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, or expose `next_l2_gas_price` as a separate accessor on `GatewayFixedBlockStateReader` and use it directly in `validate_resource_bounds`. The same correction should be applied to the `BlockInfo` passed into `run_validate_entry_point` so that the blockifier's `check_fee_bounds` at the gateway uses the same price the batcher will enforce.

```diff
- l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
+ l2_gas_price: block_header.next_l2_gas_price.try_into()
+     .unwrap_or_else(|_| block_header.l2_gas_price.price_in_fri.try_into()?),
```

---

### Proof of Concept

1. Observe the current committed block's L2 gas price `P_cur` and compute `P_next = calculate_next_base_gas_price(P_cur, gas_used, gas_target, min)` where `gas_used > gas_target` (congested block), so `P_next > P_cur`.
2. Submit an `InvokeV3` transaction with `l2_gas.max_price_per_unit = P_cur`.
3. Gateway `validate_resource_bounds` computes `threshold = 100% × P_cur = P_cur`; the transaction passes.
4. Gateway `run_validate_entry_point` runs blockifier `check_fee_bounds` against `P_cur`; the transaction passes.
5. Transaction is admitted to the mempool.
6. Batcher builds the next block with `l2_gas_price = P_next > P_cur`; `check_fee_bounds` fires `MaxGasPriceTooLow` and the transaction is rejected at pre-validation.
7. The gateway admitted a transaction that the batcher cannot include. [5](#0-4) [6](#0-5) [7](#0-6) [8](#0-7)

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
