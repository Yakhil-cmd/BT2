### Title
Integer Division Truncation in XCC EVM Gas Accounting Allows Users to Obtain NEAR Gas for Free - (File: engine-precompiles/src/xcc.rs)

### Summary

The `CrossContractCall` precompile charges EVM gas for the NEAR gas attached to a user-specified promise using integer division by `CROSS_CONTRACT_CALL_NEAR_GAS = 175_000_000`. Because Rust integer division truncates, any user-controlled `call_gas` value whose addition to `router_exec_cost` does not change the integer quotient is effectively free. An unprivileged EVM user can attach up to 174,999,999 NEAR gas (~0.175 TGas) to an XCC promise without paying any additional EVM gas for it, causing the relayer/engine to subsidize that NEAR gas cost.

### Finding Description

In `engine-precompiles/src/xcc.rs` line 174, after constructing the promise with `attached_gas = router_exec_cost + call_gas`, the EVM gas cost is computed as:

```rust
cost += EthGas::new(promise.attached_gas.as_u64() / costs::CROSS_CONTRACT_CALL_NEAR_GAS);
``` [1](#0-0) 

The divisor is:

```rust
pub const CROSS_CONTRACT_CALL_NEAR_GAS: u64 = 175_000_000;
``` [2](#0-1) 

For the `Eager` variant, `promise.attached_gas = router_exec_cost.saturating_add(call_gas)`, where `call_gas = call.total_gas()` is entirely user-controlled via the `CrossContractCallArgs::Eager` input: [3](#0-2) 

`router_exec_cost` is at minimum `ROUTER_EXEC_BASE = 7_000_000_000_000` (7 TGas): [4](#0-3) 

Because integer division truncates, any `call_gas` value in the range `[0, 174_999_999]` that keeps `(router_exec_cost + call_gas)` within the same multiple of 175,000,000 costs the user **zero additional EVM gas**. Concretely:

- `call_gas = 0`: `7_000_000_000_000 / 175_000_000 = 40_000` EVM gas
- `call_gas = 174_999_999`: `7_174_999_999 / 175_000_000 = 40_000` EVM gas (same — truncated)
- `call_gas = 175_000_000`: `7_175_000_000 / 175_000_000 = 41_000` EVM gas (correctly charged)

The engine still attaches the full `call_gas` to the NEAR promise and the NEAR runtime consumes it, but the user paid nothing for it.

### Impact Explanation

**High — Theft of unclaimed yield.**

The relayer who submits the NEAR transaction pays the NEAR gas cost. The EVM gas charged to the user is the mechanism by which the relayer is compensated. When the user attaches up to 174,999,999 NEAR gas (~0.175 TGas) without paying EVM gas for it, the relayer subsidizes that NEAR gas. At NEAR's gas price of 100 million yoctoNEAR per gas unit, 174,999,999 NEAR gas ≈ 0.0175 NEAR per call. Repeated across many XCC calls, this constitutes systematic underpayment and theft of relayer fee income.

### Likelihood Explanation

**High.** Any unprivileged EVM user can trigger this by calling the XCC precompile with `CrossContractCallArgs::Eager` and setting `call_gas` to any value below 175,000,000. No special privileges, governance access, or external conditions are required. The XCC precompile is a production-deployed, user-facing interface.

### Recommendation

Replace the truncating integer division with a ceiling division to ensure users always pay for the full NEAR gas they consume:

```rust
// Instead of:
cost += EthGas::new(promise.attached_gas.as_u64() / costs::CROSS_CONTRACT_CALL_NEAR_GAS);

// Use ceiling division:
cost += EthGas::new(
    promise.attached_gas.as_u64().div_ceil(costs::CROSS_CONTRACT_CALL_NEAR_GAS)
);
```

This ensures that any non-zero `call_gas` always results in at least 1 EVM gas being charged, eliminating the free-gas window.

### Proof of Concept

1. Deploy Aurora Engine with XCC enabled and a wNEAR ERC-20 set.
2. As an unprivileged EVM user, approve the XCC precompile to spend wNEAR.
3. Construct a `CrossContractCallArgs::Eager` with a `PromiseCreateArgs` where `attached_gas = NearGas::new(174_999_999)`.
4. Submit the XCC transaction.
5. Observe that the EVM gas charged for the `attached_gas` component is `(7_000_000_000_000 + 174_999_999) / 175_000_000 = 40_000` — identical to a call with `attached_gas = 0`.
6. The NEAR promise is dispatched with the full 174,999,999 NEAR gas attached, but the user paid 0 EVM gas for it.

The attacker-controlled entry path is `CrossContractCall::run_with_handle` → line 174 division: [5](#0-4)

impl<I: IO> HandleBasedPrecompile for CrossContractCall<I> {
    #[allow(clippy::too_many_lines)]
    fn run_with_handle(
        &self,
        handle: &mut impl PrecompileHandle,
    ) -> Result<PrecompileOutput, PrecompileFailure> {
        let input = handle.input();
        let target_gas = handle.gas_limit().map(EthGas::new);
        let context = handle.context();
        utils::validate_no_value_attached_to_precompile(context.apparent_value)?;
        let is_static = handle.is_static();

        // This only includes the cost we can easily derive without parsing the input.
        // This allows failing fast without wasting computation on parsing.
        let input_len = u64::try_from(input.len()).map_err(utils::err_usize_conv)?;
        let mut cost =
            costs::CROSS_CONTRACT_CALL_BASE + costs::CROSS_CONTRACT_CALL_BYTE * input_len;
        let check_cost = |cost: EthGas| -> Result<(), PrecompileFailure> {
            if let Some(target_gas) = target_gas
                && cost > target_gas
            {
                return Err(PrecompileFailure::Error {
                    exit_status: ExitError::OutOfGas,
                });
            }
            Ok(())
        };
        check_cost(cost)?;

        // It's not allowed to call cross contract call precompile in static or delegate mode
        if is_static {
            return Err(revert_with_message(consts::ERR_STATIC));
        } else if context.address != cross_contract_call::ADDRESS.raw() {
            return Err(revert_with_message(consts::ERR_DELEGATE));
        }

        let sender = context.caller;
        let target_account_id = create_target_account_id(sender, self.engine_account_id.as_ref())?;
        let args = CrossContractCallArgs::try_from_slice(input)
            .map_err(|_| ExitError::Other(Cow::from(consts::ERR_INVALID_INPUT)))?;
        let (promise, attached_near) = match args {
            CrossContractCallArgs::Eager(call) => {
                let call_gas = call.total_gas();
                let attached_near = call.total_near();
                let callback_count = call
                    .promise_count()
                    .checked_sub(1)
                    .ok_or_else(|| ExitError::Other(Cow::from(consts::ERR_INVALID_INPUT)))?;
                let router_exec_cost = costs::ROUTER_EXEC_BASE
                    + NearGas::new(callback_count * costs::ROUTER_EXEC_PER_CALLBACK.as_u64());
                let promise = PromiseCreateArgs {
                    target_account_id,
                    method: consts::ROUTER_EXEC_NAME.into(),
                    args: borsh::to_vec(&call)
                        .map_err(|_| ExitError::Other(Cow::from(consts::ERR_SERIALIZE)))?,
                    attached_balance: ZERO_YOCTO,
                    attached_gas: router_exec_cost.saturating_add(call_gas),
                };
                (promise, attached_near)
            }
            CrossContractCallArgs::Delayed(call) => {
                let attached_near = call.total_near();
                let promise = PromiseCreateArgs {
                    target_account_id,
                    method: consts::ROUTER_SCHEDULE_NAME.into(),
                    args: borsh::to_vec(&call)
                        .map_err(|_| ExitError::Other(Cow::from(consts::ERR_SERIALIZE)))?,
                    attached_balance: ZERO_YOCTO,
                    // We don't need to add any gas to the amount need for the schedule call
                    // since the promise is not executed right away.
                    attached_gas: costs::ROUTER_SCHEDULE,
                };
                (promise, attached_near)
            }
        };
        cost += EthGas::new(promise.attached_gas.as_u64() / costs::CROSS_CONTRACT_CALL_NEAR_GAS);
        check_cost(cost)?;
