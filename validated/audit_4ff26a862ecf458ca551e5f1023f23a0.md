### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the staking contract is specified to be callable "Only starkware sequencer" but contains no caller validation in its implementation. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block reward slot and permanently preventing the legitimate sequencer from distributing rewards for that block.

### Finding Description
The spec for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1447–1510 contains no `get_caller_address()` check against any sequencer address. The function only validates:

1. Contract is not paused (`general_prerequisites`)
2. `current_block_number > self.last_reward_block.read()` — the per-block gate
3. Staker exists and is active
4. Staker has non-zero balance [1](#0-0) 

The critical state mutation is:

```cairo
self.last_reward_block.write(current_block_number);
```

This write happens unconditionally before the `disable_rewards` branch check. [2](#0-1) 

`last_reward_block` is a single global storage variable. Once it is set to block `N`, the guard `current_block_number > self.last_reward_block.read()` rejects all further calls for block `N` with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The spec confirms this is the intended one-call-per-block design:

> **pre-condition**: Rewards did not distributed for the current block yet. [4](#0-3) 

### Impact Explanation
An attacker calls `update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)` at the start of any block. This:

1. Passes all validation checks (staker exists, block is new).
2. Writes `current_block_number` into `last_reward_block`.
3. Returns early due to `disable_rewards: true` — **no rewards are credited**.

The legitimate sequencer's subsequent call for the same block fails with `REWARDS_ALREADY_UPDATED`. The block's reward allocation is permanently lost; it is never credited to `unclaimed_rewards_own` for the staker or to the pool. This constitutes **permanent freezing of unclaimed yield** for every block the attacker targets.

**Impact: High** — Permanent freezing of unclaimed yield (stakers and delegators lose block rewards for every griefed block).

### Likelihood Explanation
The function is public and permissionless on-chain. No special role, key, or capital is required. The attacker only needs to submit a transaction before the sequencer's reward-distribution transaction in each block. On Starknet, transaction ordering within a block is controlled by the sequencer, but the function is callable by any externally-owned account. A persistent attacker can automate this for every block, causing continuous yield loss.

### Recommendation
Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured sequencer address (or a dedicated rewards-manager role stored in contract state), mirroring the pattern already used for the attestation contract:

```cairo
fn assert_caller_is_attestation_contract(self: @ContractState) {
    assert!(
        get_caller_address() == self.attestation_contract.read(),
        "{}",
        Error::CALLER_IS_NOT_ATTESTATION_CONTRACT,
    );
}
``` [5](#0-4) 

A dedicated `sequencer_address` storage variable should be introduced and checked at the entry of `update_rewards`.

### Proof of Concept

1. Deploy the system in consensus-rewards mode (after `consensus_rewards_first_epoch`).
2. Staker stakes and waits K epochs for balance to become effective.
3. At block `N`, attacker calls:
   ```cairo
   IStakingRewardsManagerDispatcher { contract_address: staking_contract }
       .update_rewards(staker_address: victim_staker, disable_rewards: true);
   ```
4. `last_reward_block` is now `N`; no rewards credited.
5. Sequencer attempts the same call for block `N` → reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker calls `claim_rewards` — receives zero for block `N`.
7. Repeat for every block to permanently deny all yield.

The test suite itself demonstrates that `update_rewards` is callable without any caller restriction from arbitrary addresses: [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
