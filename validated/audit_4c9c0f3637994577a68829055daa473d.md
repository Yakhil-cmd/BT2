### Title
Missing Access Control on `update_rewards` Allows Any Caller to Grief Consensus Reward Distribution — (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the `IStakingRewardsManager` interface has no caller restriction. A global `last_reward_block` guard allows only one call per block. Any unprivileged address can front-run the legitimate consensus caller with `disable_rewards: true`, consuming the per-block slot and preventing reward distribution for that block across all stakers.

### Finding Description
`update_rewards` is exposed publicly with no `only_*` access-control guard:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);   // global, not per-staker

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits without distributing rewards
    }
    // ... distribute block rewards
``` [1](#0-0) 

`last_reward_block` is a single global storage slot, not per-staker. The first call in any block wins the slot. If an attacker calls `update_rewards(valid_staker, disable_rewards: true)` before the consensus mechanism does, the slot is consumed and the legitimate call reverts with `REWARDS_ALREADY_UPDATED`. No rewards are distributed for that block.

The test suite confirms the function is callable by any address without special setup: [2](#0-1) [3](#0-2) 

### Impact Explanation
An attacker who calls `update_rewards(..., disable_rewards: true)` in every block prevents all stakers from accruing consensus-phase block rewards for those blocks. Because `last_reward_block` is global, a single attacker transaction per block is sufficient to freeze yield for the entire protocol. This maps to **Temporary freezing of unclaimed yield** (High) under the allowed impact scope.

### Likelihood Explanation
Staker addresses are public on-chain. Starknet transaction fees are low. An attacker can sustain this griefing indefinitely at modest cost. No privileged key, bridge compromise, or external dependency is required — only the ability to submit a transaction before the consensus mechanism in each block.

### Recommendation
Add an access-control guard to `update_rewards` restricting callers to the designated consensus mechanism address (analogous to how `update_rewards_from_attestation_contract` restricts callers to the attestation contract via `assert_caller_is_attestation_contract()`): [4](#0-3) 

Apply the same pattern to `update_rewards`:
```cairo
fn update_rewards(...) {
    self.assert_caller_is_consensus_contract(); // add this guard
    ...
}
```

### Proof of Concept
1. Attacker observes any active staker address `S` on-chain.
2. Each block, attacker submits `staking.update_rewards(staker_address: S, disable_rewards: true)` with higher gas priority than the consensus mechanism.
3. `last_reward_block` is set to the current block; the function returns early without distributing rewards.
4. The consensus mechanism's subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. No staker earns block rewards for that block. Repeated every block, this permanently suppresses yield accrual for all stakers.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
