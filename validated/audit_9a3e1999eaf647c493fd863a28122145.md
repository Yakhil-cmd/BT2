### Title
Gateway L2 Gas Price Threshold Uses Current Block Price Instead of Next Block Price, Causing Valid Transactions to Be Rejected - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **current (latest committed) block's** L2 gas price, but the transaction will be executed in the **next** block whose price is the EIP-1559-adjusted `next_l2_gas_price`. When the price is falling (lightly-used blocks), the gateway rejects transactions whose `max_price_per_unit` is correct for the next block but below the stale current-block threshold. When the price is rising, the gateway admits transactions that will fail `check_fee_bounds` at execution time. A developer TODO in the source code explicitly acknowledges this is wrong.

### Finding Description

`validate_resource_bounds` reads the L2 gas price from the latest committed block:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:229-236
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
```

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates this field from `block_header.l2_gas_price.price_in_fri`:

```rust
// crates/apollo_gateway/src/gateway_fixed_block_state_reader.rs:46-49
strk_gas_prices: GasPriceVector {
    ...
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
},
```

This is the price **used in the current block**, not the price for the next block. The `StorageBlockHeader` already carries the correct value:

```rust
// crates/apollo_storage/src/header.rs:88-89
/// The next L2 gas price.
pub next_l2_gas_price: GasPrice,
```

`next_l2_gas_price` is the EIP-1559-computed price for the next block, produced by `calculate_next_base_gas_price` in `crates/apollo_consensus_orchestrator/src/fee_market/mod.rs`. The gateway never reads this field; `GatewayFixedBlockSyncStateClient` does not expose it.

The threshold check then compares the transaction's `max_price_per_unit` against the wrong reference:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:367-383
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(StarknetError { ... "GAS_PRICE_TOO_LOW" ... });
}
```

**Falling-price scenario (lightly-used blocks):** EIP-1559 lowers `next_l2_gas_price` below `current_l2_gas_price`. A user who correctly sets `max_price_per_unit = next_l2_gas_price` satisfies the execution-time check (`check_fee_bounds` in `AccountTransaction::perform_pre_validation_stage`) but fails the gateway check because `next_l2_gas_price < current_l2_gas_price = threshold`. The transaction is rejected at admission even though it is valid.

**Rising-price scenario (heavily-used blocks):** EIP-1559 raises `next_l2_gas_price` above `current_l2_gas_price`. A user who sets `max_price_per_unit = current_l2_gas_price` passes the gateway check but fails `check_fee_bounds` at execution time because `max_price_per_unit < next_l2_gas_price`. The gateway admits a transaction that will fail.

### Impact Explanation

**Falling-price path (valid transactions rejected):** Any user who prices their transaction at the correct next-block L2 gas price is denied admission. This is a systematic, unprivileged, externally-triggerable denial of service: any period of low block utilization causes the gateway to reject correctly-priced transactions. Impact matches: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

**Rising-price path (invalid transactions accepted):** The gateway admits transactions that will fail `check_fee_bounds` at execution. The user loses the fee charged for the validation step, and the sequencer wastes execution resources. Impact matches: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price every block. Any block with gas usage below the target (`gas_target`) produces a lower `next_l2_gas_price`. With `gas_price_max_change_denominator` controlling the rate, the discrepancy per block is bounded but non-zero and cumulative across consecutive low-usage blocks. The condition is reachable by any user submitting a transaction during a period of below-target block utilization — no special privileges required. The developer TODO comment at line 229 confirms the team is aware the wrong field is being used.

### Recommendation

1. Expose `next_l2_gas_price` from `StorageBlockHeader` through `GatewayFixedBlockStateReader`. Add a `get_next_l2_gas_price` method or include it in the returned `BlockInfo`.
2. In `validate_resource_bounds`, replace `block_info.gas_prices.strk_gas_prices.l2_gas_price` with the `next_l2_gas_price` value from the block header.
3. Remove the TODO comment once fixed.

### Proof of Concept

Scenario (falling price, valid transaction rejected):

1. Latest committed block N has `l2_gas_price = 1000 FRI` and was lightly used (gas_used < gas_target).
2. EIP-1559 computes `next_l2_gas_price = 990 FRI` for block N+1.
3. User submits an invoke transaction with `l2_gas.max_price_per_unit = 990 FRI` (correct for block N+1).
4. `validate_resource_bounds` reads `previous_block_l2_gas_price = 1000 FRI` (block N's price).
5. With `min_gas_price_percentage = 100`, `threshold = 1000 FRI`.
6. `990 < 1000` → gateway returns `GAS_PRICE_TOO_LOW` and rejects the transaction.
7. The transaction would have passed `check_fee_bounds` at execution time (990 ≥ 990).

```rust
    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
```

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
```
