### Title
Missing Caller Authorization on `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` enforces no caller check. The spec explicitly states access control is **"Only starkware sequencer"**, but the implementation omits this guard entirely. Because `last_reward_block` is a **global** (not per-staker) storage slot, any unprivileged caller can consume the single per-block reward slot with `disable_rewards=true`, permanently discarding that block's rewards for all stakers.

---

### Finding Description

The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 contains no such check:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}", Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);  // global write

    if disable_rewards || self.is_pre_consensus() {
        return;                            // exits without distributing rewards
    }
    ...
}
``` [1](#0-0) 

`general_prerequisites()` only checks the pause flag — confirmed by the complete absence of any `sequencer`, `only_sequencer`, or `SEQUENCER` reference in `staking.cairo`. There is no role gate, no `get_caller_address()` comparison, and no tx-origin check.

`last_reward_block` is declared as a single global `BlockNumber` field (not a map keyed by staker): [2](#0-1) 

The spec confirms the invariant: "Rewards did not distributed for the current block yet" is the only precondition, and the gate is this single global slot. [3](#0-2) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

For every block in which the attacker front-runs the sequencer:

1. Attacker calls `update_rewards(any_valid_active_staker, disable_rewards=true)`.
2. `last_reward_block` is written to the current block number (line 1485).
3. The function returns early at line 1487–1489 without distributing any rewards.
4. Every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED` (line 1454–1458).
5. That block's rewards are **permanently lost** — there is no carry-forward or catch-up mechanism.

The attacker does not need to control the `staker_address` argument; any currently active staker address satisfies the validity checks. The attacker gains nothing financially (pure griefing), but every targeted block's yield is irrecoverably destroyed for all stakers and delegators.

---

### Likelihood Explanation

Starknet is a public mempool environment. Any EOA can submit a transaction. The attacker only needs to:
- Know one valid active staker address (publicly readable via `get_stakers`).
- Submit `update_rewards(staker, true)` before the sequencer's own call in each block.

The cost is one transaction per block. The attack is sustainable indefinitely and requires no privileged access, no capital, and no protocol knowledge beyond the public ABI.

---

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, consistent with the spec. On Starknet, the sequencer address is available via `starknet::get_execution_info().tx_info.account_contract_address` or a stored authorized-caller address. For example:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.roles.only_sequencer();   // or assert!(get_caller_address() == self.sequencer_address.read(), ...)
    ...
}
```

Alternatively, restrict the `IStakingRewardsManager` interface so it is only callable from a known sequencer address stored at construction time.

---

### Proof of Concept

```
// Setup: two active stakers A and B, consensus rewards active.
// Block N:
//   Attacker (any address) calls:
//     staking.update_rewards(staker_address=A, disable_rewards=true)
//   → last_reward_block = N, returns early, zero rewards distributed.
//
//   Sequencer then calls:
//     staking.update_rewards(staker_address=B, disable_rewards=false)
//   → PANICS with REWARDS_ALREADY_UPDATED
//
// Block N's rewards are permanently lost for all stakers.
// Repeat every block to suppress all consensus rewards indefinitely.
```

This is directly confirmed by the existing test at `src/staking/tests/test.cairo` lines 3956–3973, which shows that after one `update_rewards` call (regardless of `disable_rewards` value), all subsequent calls in the same block revert with `REWARDS_ALREADY_UPDATED`. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
