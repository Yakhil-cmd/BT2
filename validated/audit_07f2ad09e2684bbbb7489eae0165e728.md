### Title
Opaque Spend Bundle Display in `chia_pushTransactions` Dapp Command Allows Malicious Dapp to Trick User into Approving Arbitrary Asset Transfers ‚Äî (File: `packages/gui/src/electron/commands/Commands.ts`)

---

### Summary

A paired dapp can invoke `chia_pushTransactions` (`chia_wallet.push_transactions`) with an arbitrary spend bundle. The confirmation dialog renders the `transactions` parameter only as a raw JSON blob ‚Äî no human-readable amount, destination address, or asset type is derived and displayed. A malicious dapp can therefore describe a small, benign-looking operation to the user while embedding a large, draining spend bundle in the actual call. The user has no way to verify the true effect of what they are approving.

---

### Finding Description

The `chia_wallet.push_transactions` command is registered as a dapp-callable command (`chia_pushTransactions`) in `Commands.ts`. [1](#0-0) 

Its `params` list exposes `transactions` (type `json`), `fee`, `push`, and `sign` ‚Äî all without `hide: true`. The `humanizeParams` function therefore renders `transactions` verbatim as a raw JSON string in the confirmation dialog. [2](#0-1) 

`parseCommandDisplay`, which is the only mechanism that produces a human-readable `walletDelta` (spending/receiving breakdown), handles only `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids`. It returns `undefined` for every other command, including `chia_wallet.push_transactions`. [3](#0-2) 

When `display` is `undefined`, the `Confirm` dialog renders no `WalletDeltaSection` ‚Äî only the raw `rows` (the opaque JSON blob) and a collapsible "Raw data" dump. [4](#0-3) 

The spend bundle JSON contains hex-encoded coin IDs, puzzle reveals, and CLVM solutions ‚Äî completely unreadable to a normal user. The `sign` flag (optional, shown only if provided) means the wallet can also be asked to sign the bundle before pushing, compounding the risk. [5](#0-4) 

---

### Impact Explanation

A malicious dapp that has been paired with the user's wallet can call `chia_pushTransactions` with a crafted spend bundle that transfers any amount of XCH, CAT tokens, or NFTs to an attacker-controlled address. The confirmation dialog will show only the raw JSON blob of the spend bundle. The user cannot determine the actual amounts, asset types, or destination addresses from the dialog and may approve the operation believing it to be benign. This constitutes an unauthorized transfer of XCH, CAT, or NFT assets ‚Äî a Critical/High impact under the allowed scope.

---

### Likelihood Explanation

Any dapp that has been granted the `chia_pushTransactions` permission via the WalletConnect pairing flow can trigger this. The pairing flow itself is user-initiated, but once a dapp is paired and the command is in its granted set, it can call `chia_pushTransactions` at any time and present the opaque confirmation. The attack requires no leaked keys, no host compromise, and no cryptographic break ‚Äî only a paired malicious dapp.

---

### Recommendation

Extend `parseCommandDisplay` to handle `chia_wallet.push_transactions`. Parse the `transactions` JSON to extract coin amounts and destination puzzle hashes, resolve them to human-readable XCH/CAT/NFT rows, and populate a `walletDelta` display ‚Äî mirroring the existing logic for `take_offer` and `create_offer_for_ids`. Until a full parser is in place, at minimum display a prominent warning that the raw spend bundle cannot be verified and that the user should only approve if they trust the requesting application completely. [6](#0-5) 

---

### Proof of Concept

1. A malicious dapp pairs with the user's Chia wallet via WalletConnect and is granted the `chia_pushTransactions` command.
2. The dapp constructs a spend bundle that spends a coin owned by the user and sends 1000 XCH to an attacker address. It sets `sign: true` so the wallet signs the bundle.
3. The dapp calls `chia_pushTransactions` with this bundle.
4. `parseDappParams` validates the parameter types and passes the bundle through.
5. `humanizeDappCommand` calls `humanizeParams`, which renders `transactions` as a raw JSON string (coin IDs, puzzle reveals, solutions in hex).
6. `parseCommandDisplay('chia_wallet.push_transactions', ...)` returns `undefined` ‚Äî no `walletDelta` section is generated.
7. The `Confirm` dialog opens showing: command name, the raw JSON blob, and a "Raw data" collapsible ‚Äî no spending/receiving summary.
8. The user, unable to parse the hex-encoded spend bundle, approves.
9. The wallet signs and pushes the transaction; 1000 XCH is transferred to the attacker.

export async function humanizeParams(params: ParamSchema[], data: Record<string, unknown>, networkPrefix?: string) {
  // hidden and optional params that are not presented are not shown in the confirm dialog
  const visibleParams = params.filter((param) => {
    const { hide, isOptional, name } = param;

    if (hide === true) {
      return false;
    }

    const isDefined = name in data;
    if (isOptional && !isDefined) {
      return false;
    }

    return true;
  });

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }

  return undefined;
}
