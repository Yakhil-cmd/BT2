### Title
Gateway L2 gas price threshold computed with floor truncation instead of ceiling, admitting transactions below the configured minimum — (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary
`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price using `Ratio::to_integer()`, which truncates toward zero (floor). When `min_gas_price_percentage × previous_block_l2_gas_price` is not evenly divisible by 100, the computed threshold is strictly lower than the true mathematical threshold, so a transaction whose `max_price_per_unit` is below the operator-configured percentage of the previous block's L2 gas price passes gateway admission.

### Finding Description

```rust
fn validate_tx_l2_gas_price_within_threshold(

            if tx_l2_gas_price.0 < threshold {
                return Err(...)

```

`Ratio::to_integer()` in `num_rational` rounds toward zero. For positive values this is floor division. The invariant the check is meant to enforce is:

```
tx_l2_gas_price ≥ ceil(min_gas_price_percentage × previous_block_l2_gas_price / 100)
```

Because floor is used instead of ceiling, the enforced threshold is:

```
threshold = floor(min_gas_price_percentage × previous_block_l2_gas_price / 100)
```

Whenever `min_gas_price_percentage × previous_block_l2_gas_price mod 100 ≠ 0`, the computed threshold is strictly less than the true threshold, and the strict-less-than guard (`< threshold`) passes for a gas price that is below the true minimum.

**Concrete example:**
- `min_gas_price_percentage = 80` (a plausible non-default operator setting)
- `previous_block_l2_gas_price = 101`
- True threshold = 80 × 101 / 100 = **80.8**
- Computed threshold = floor(80.8) = **80**
- A transaction with `tx_l2_gas_price = 80` satisfies `80 < 80 → false`, so it **passes**
- But 80 / 101 ≈ 79.2 % < 80 %, so it **should fail**

The function is reached via `validate_resource_bounds` → `validate_state_preconditions` → `run_pre_validation_checks` → `extract_state_nonce_and_run_validations` in the gateway's `add_tx_inner` path. No downstream guard re-checks this invariant; once the transaction is admitted to the mempool it proceeds to execution. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation

The gateway stateful path is the last admission gate before a transaction enters the mempool and is sequenced. A transaction whose `max_price_per_unit` for L2 gas is below the operator-configured minimum percentage of the previous block's price is, by definition, invalid under the configured policy. The floor truncation allows such transactions to bypass the check and be admitted. This matches the allowed impact:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

- The default `min_gas_price_percentage` is **100**. At exactly 100 %, `100 × price / 100 = price` with no remainder, so the default configuration is unaffected.
- Any operator-configured value that is not a multiple of 100 (e.g., 80, 90, 95) combined with a previous-block gas price that is not a multiple of `100 / gcd(percentage, 100)` triggers the off-by-one. Values like 80 % are natural choices for a "require at least 80 % of last block's price" policy.
- An attacker only needs to observe the previous block's L2 gas price (public on-chain data) and the configured percentage (public config), then submit a transaction with `max_price_per_unit` equal to the truncated threshold.

### Recommendation

Replace `to_integer()` (floor) with `ceil().to_integer()` so the threshold is always rounded up:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This ensures that a transaction's gas price must be at least the ceiling of the true threshold, preserving the intended invariant for all values of `min_gas_price_percentage`.

### Proof of Concept

1. Deploy a sequencer node with `min_gas_price_percentage = 80`.
2. Wait for a block whose L2 gas price is any value not divisible by 5 (e.g., 101 fri).
3. Compute `floor(80 × 101 / 100) = 80`.
4. Submit an `AllResources` V3 invoke transaction with `l2_gas.max_price_per_unit = 80`.
5. `validate_tx_l2_gas_price_within_threshold` evaluates `80 < 80 → false` and returns `Ok(())`.
6. The transaction is admitted to the mempool despite offering only ≈79.2 % of the previous block's L2 gas price, below the configured 80 % floor. [5](#0-4) [6](#0-5)
