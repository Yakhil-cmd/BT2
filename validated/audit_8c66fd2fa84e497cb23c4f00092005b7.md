### Title
Unrestricted `update_rewards` with `disable_rewards` Flag Allows Any Caller to Permanently Block Consensus Reward Distribution - (File: src/staking/staking.cairo)

---

### Summary

`IStakingRewardsManager::update_rewards` in the `Staking` contract carries no role-based access control. It enforces a strict one-call-per-block invariant via `last_reward_block`. When called with `disable_rewards: true`, it advances `last_reward_block` to the current block but skips all reward distribution. Any subsequent legitimate call in the same block is rejected with `REWARDS_ALREADY_UPDATED`. An unprivileged attacker can exploit this to permanently suppress consensus block rewards for all stakers.

---

### Finding Description

`update_rewards` is declared in the public `IStakingRewardsManager` interface with no caller restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // rewards skipped
    }
    // ... distribute block rewards to staker and pools
``` [1](#0-0) 

The `general_prerequisites()` call only asserts the contract is not paused — there is no `only_*` role guard. [2](#0-1) 

The public interface confirms the function is intentionally exposed to any caller: [3](#0-2) 

The one-call-per-block gate is the critical shared state: [4](#0-3) 

Once `last_reward_block` equals the current block number, every other call in that block reverts unconditionally — there is no recovery path within the same block. [5](#0-4) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_valid_active_staker, disable_rewards: true)` in every block:

1. Consumes the block's single reward slot.
2. Distributes zero rewards (the `disable_rewards` branch returns immediately after writing `last_reward_block`).
3. Causes every legitimate consensus-reward call in that block to revert with `REWARDS_ALREADY_UPDATED`.

Because consensus block rewards are per-block and non-retroactive, missed blocks are permanently lost. Sustained execution across all blocks means all stakers and pool members receive zero consensus rewards indefinitely — a permanent freeze of all unclaimed yield generated by the consensus reward mechanism.

---

### Likelihood Explanation

**High.** The attacker requires only:

- A valid, active staker address with non-zero STRK balance (trivially obtained from `NewStaker` events on-chain).
- Gas to submit one transaction per block.

No tokens, no privileged role, no special setup. The attack is cheap relative to the value of rewards suppressed across the entire staker set.

---

### Recommendation

Add an authorized-caller check to `update_rewards`. The function should only be callable by the consensus mechanism (e.g., a designated `REWARDS_MANAGER` role or a specific contract address stored at construction time). Example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_rewards_manager();   // add role guard
    self.general_prerequisites();
    ...
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface entirely and handle that logic internally or via a separate privileged entry point.

---

### Proof of Concept

1. Attacker reads any `NewStaker` event to obtain a valid `staker_address` that is currently active and has non-zero STRK balance.
2. In every Starknet block, attacker submits:
   ```
   staking_contract.update_rewards(staker_address=<valid_staker>, disable_rewards=true)
   ```
3. Inside `update_rewards`:
   - `general_prerequisites()` passes (contract is not paused).
   - `current_block_number > last_reward_block` passes (new block).
   - `internal_staker_info` and `is_staker_active` pass (valid staker).
   - `last_reward_block` is written to `current_block_number`.
   - `disable_rewards == true` → function returns, **zero rewards distributed**.
4. The consensus mechanism (or any staker) attempts `update_rewards(..., disable_rewards=false)` in the same block → reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers and pool members receive no consensus block rewards for that block.
6. Repeated every block → permanent suppression of all consensus yield.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
