### Title
Unprivileged User Can Permanently Block Admin External Token-Mapping Registration via Auto-Deployment Front-Running — (`x/cronos/keeper/keeper.go`, `x/cronos/keeper/ibc.go`)

### Summary

When `enable_auto_deployment` is `true`, any unprivileged user can trigger auto-deployment of a CRC21 contract for any IBC denom by sending `MsgConvertVouchers` or by initiating an IBC transfer to themselves. Once an auto contract is recorded for a denom, the `ensureDenomNotMapped` guard in `SetExternalContractForDenom` permanently prevents the Cronos admin from registering a different external contract for that same denom. There is no admin-accessible path to delete an auto contract mapping, so the block is irreversible.

### Finding Description

`ConvertVouchersToEvmCoins` passes `params.EnableAutoDeployment` as the `autoDeploy` flag to `ConvertCoinFromNativeToCRC21`: [1](#0-0) 

When `autoDeploy` is `true` and no contract is yet mapped, `ConvertCoinFromNativeToCRC21` deploys a new CRC21 contract and calls `SetAutoContractForDenom`: [2](#0-1) 

`SetAutoContractForDenom` writes both `DenomToAutoContractKey` and `ContractToDenomKey` entries: [3](#0-2) 

Later, when the Cronos admin calls `MsgUpdateTokenMapping` → `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom`, the first thing that function does is call `ensureDenomNotMapped`: [4](#0-3) 

`ensureDenomNotMapped` calls `GetContractByDenom`, which returns the auto contract if no external contract exists: [5](#0-4) [6](#0-5) 

Because the auto contract is found, `ensureDenomNotMapped` returns `ErrDenomAlreadyMapped` and the admin's registration is rejected. There is no `DeleteAutoContractForDenom` function exposed to the admin, and passing an empty contract to `RegisterOrUpdateTokenMapping` only calls `DeleteExternalContractForDenom`, which does not touch auto contracts: [7](#0-6) 

The spec confirms: "There's no way to delete a token mapping currently." [8](#0-7) 

### Impact Explanation

**High — Bypass of Cronos admin token-mapping authorization and permanent inability to register external contracts for affected denoms.**

The admin's `MsgUpdateTokenMapping` authority is completely bypassed for any denom an attacker pre-occupies. The admin intended to bind a specific external CRC21 contract (e.g., one with gravity bridge support, existing liquidity, or custom logic) to a denom. After the attack, the denom is permanently bound to the attacker-triggered auto-deployed contract. Bridge/conversion flows that depend on the admin-chosen external contract are permanently disrupted for that denom.

### Likelihood Explanation

`enable_auto_deployment` is `true` in the default production genesis config: [9](#0-8) 

The attacker needs only a minimal balance of the target IBC denom. The easiest path is to send themselves a tiny IBC transfer from another chain, which triggers `OnRecvVouchers` → `ConvertVouchersToEvmCoins` → auto-deployment automatically via the IBC middleware, with no direct `MsgConvertVouchers` required: [10](#0-9) 

The admin's pending `MsgUpdateTokenMapping` is visible in the mempool, making targeted front-running straightforward.

### Recommendation

- **Short term**: In `SetExternalContractForDenom`, allow the admin to override an existing *auto* contract (but not an existing *external* contract owned by another denom). Separate the `ensureDenomNotMapped` check so it only blocks re-registration of external mappings, not auto mappings.
- **Long term**: Expose an admin-only `DeleteAutoContractForDenom` operation, or require that auto-deployment only occurs when no pending admin registration exists for that denom (e.g., via a governance-controlled allowlist of denoms eligible for auto-deployment).

### Proof of Concept

1. Admin broadcasts `MsgUpdateTokenMapping{denom: "ibc/ATOM...", contract: "0xAdminContract"}` (visible in mempool).
2. Attacker sends themselves 1 `ibc/ATOM...` token from Cosmos Hub via IBC transfer.
3. IBC middleware calls `OnRecvVouchers` → `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21(autoDeploy=true)` → `DeployModuleCRC21` → `SetAutoContractForDenom("ibc/ATOM...", 0xAutoContract)`.
4. Admin's transaction executes: `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom` → `ensureDenomNotMapped` → finds `0xAutoContract` → returns `ErrDenomAlreadyMapped`. Admin's registration fails.
5. Admin has no recourse: `DeleteExternalContractForDenom` does not remove auto contracts; no `DeleteAutoContractForDenom` exists. The denom is permanently bound to `0xAutoContract`.

// SetExternalContractForDenom sets denom→external CRC21 mapping for an unmapped denom.
// Caller is responsible for source-denom specific validation before calling this method.
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}

// DeleteExternalContractForDenom delete the external contract mapping for native denom,
// returns false if mapping not exists.
func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
	store := ctx.KVStore(k.storeKey)
	contract, found := k.getExternalContractByDenom(ctx, denom)
	if !found {
		return false
	}
	store.Delete(types.DenomToExternalContractKey(denom))
	deleteReverseIfOwned(store, contract, denom)
	if auto, found := k.getAutoContractByDenom(ctx, denom); found {
		bz := store.Get(types.ContractToDenomKey(auto.Bytes()))
		if len(bz) == 0 {
			store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
		} else if existingDenom := string(bz); existingDenom != denom {
			if k.contractOwnedByDenom(ctx, existingDenom, auto) {
				// auto address is already owned by another denom; drop local auto mapping
				store.Delete(types.DenomToAutoContractKey(denom))
			} else {
				// stale reverse entry
				store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
			}
		}
	}
	return true
}

// SetAutoContractForDenom set the auto deployed contract for native denom
func (k Keeper) SetAutoContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	isSource := types.IsSourceCoin(denom)
	if err := validateContractAddressForSourceDenom(denom, address, isSource); err != nil {
		return err
	}
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}
	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToAutoContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}
