### Title
`IsValidEpochSetup` Does Not Enforce Uniqueness of `NetworkPubKey` Across Epoch Participants — (`File: state/protocol/validity.go`)

### Summary
`IsValidEpochSetup` in `state/protocol/validity.go` validates the `EpochSetup` service event before it is accepted into the protocol state. It enforces uniqueness of `NodeID` and, conditionally, uniqueness of the network `Address`. However, it never checks for duplicate `NetworkPubKey` values across participants. A staked node operator can register two nodes with different staking keys (different `NodeID`s) but the same `NetworkPubKey`, and the Go protocol state will accept both into the epoch without error.

### Finding Description
`IsValidEpochSetup` performs three participant-level checks:

1. **(a)** Unique `NodeID` — always enforced.
2. **(b)** Unique network `Address` — enforced only when `verifyNetworkAddress == true`.
3. **(c)** Canonical ordering — always enforced. [1](#0-0) 

There is no check for duplicate `NetworkPubKey` or `StakingPubKey`. Because `NodeID` is derived from the staking key (`nodeID = hash(StakingPubKey)`), duplicate staking keys are implicitly caught by check (a). But `NetworkPubKey` is independent of `NodeID` and is never checked for uniqueness. [2](#0-1) 

The `NetworkPubKey` is the key used to derive the LibP2P peer ID for each node. Two nodes with the same `NetworkPubKey` are indistinguishable at the network layer. The fixture data in `utils/unittest/service_events_fixtures.go` demonstrates this gap concretely: nodes 1–4 (all collection nodes) share an identical `NetworkPubKey` in the test fixture, and `IsValidEpochSetup` accepts this without error. [3](#0-2) 

The `EpochSetup` event originates from the `FlowIDTableStaking` system smart contract and is converted to Go via `convertParticipants` in `model/convert/service_event.go`, which also performs no uniqueness check on `NetworkPubKey`. [4](#0-3) 

The final Go-side validation gate before the event is committed to the protocol state is `IsValidEpochSetup`, called from `IsValidExtendingEpochSetup` (with `verifyNetworkAddress=true`) and from `state/protocol/badger/state.go` during bootstrap. [5](#0-4) 

### Impact Explanation
Two nodes with the same `NetworkPubKey` share the same LibP2P peer ID. The networking layer (`network/underlay/network.go`) uses the peer ID derived from the `NetworkPubKey` for all peer routing, message delivery, and authentication. Accepting two distinct protocol participants with the same peer ID causes:

1. **Network-layer impersonation**: A malicious node operator can register a second node with the same `NetworkPubKey` as a legitimate consensus, collection, execution, or verification node. At the LibP2P layer, the attacker's node is indistinguishable from the victim node. Protocol messages (HotStuff votes, collection guarantees, execution receipts, result approvals) addressed to the victim's peer ID can be intercepted or answered by the attacker's node.

2. **Consensus disruption**: If a consensus node's `NetworkPubKey` is duplicated, the attacker can receive HotStuff messages intended for that node, selectively drop or replay them, and degrade or stall consensus liveness.

3. **Epoch-wide persistence**: Once accepted into the `EpochSetup` event, the duplicate key persists for the entire epoch (potentially weeks), giving the attacker a sustained window of network-layer access. [6](#0-5) 

### Likelihood Explanation
**Low.** The attack requires the adversary to stake FLOW tokens and successfully register a node with a duplicate `NetworkPubKey`. The `FlowIDTableStaking` Cadence smart contract may independently enforce network key uniqueness (this contract is not in scope here), which would be the primary guard. However, the Go protocol state provides no defense-in-depth check. If the smart contract guard is absent, incomplete, or bypassed (e.g., via a future upgrade or a bug in the contract), the Go layer will silently accept the duplicate. The gap is a missing validation in a critical security boundary.

### Recommendation
Add a uniqueness check for `NetworkPubKey` inside `IsValidEpochSetup`, analogous to the existing `NodeID` and `Address` checks:

```go
// (d) enforce uniqueness of NetworkPubKey
networkKeyLookup := make(map[string]struct{})
for _, participant := range setup.Participants {
    encoded := participant.NetworkPubKey.Encode()
    key := string(encoded)
    if _, ok := networkKeyLookup[key]; ok {
        return fmt.Errorf("duplicate network public key for participant (%x)", participant.NodeID)
    }
    networkKeyLookup[key] = struct{}{}
}
```

This check should be unconditional (not gated by a boolean parameter like `verifyNetworkAddress`) because a duplicate `NetworkPubKey` is always a protocol violation with no legitimate use case. [7](#0-6) 

### Proof of Concept
The existing test fixture in `utils/unittest/service_events_fixtures.go` already demonstrates the gap: four collection nodes (NodeIDs `0x01`–`0x04`) are all assigned the identical `NetworkPubKey` `378dbf45...`. Calling `IsValidEpochSetup(setup, true)` on this fixture returns `nil` (no error), confirming that duplicate `NetworkPubKey` values pass all current validation checks. [3](#0-2) [1](#0-0)

```go
// IsValidEpochSetup checks whether an `EpochSetup` event is syntactically correct. The boolean parameter `verifyNetworkAddress`
// controls, whether we want to permit nodes to share a networking address.
// This is a side-effect-free function. Any error return indicates that the EpochSetup event is not compliant with protocol rules.
func IsValidEpochSetup(setup *flow.EpochSetup, verifyNetworkAddress bool) error {
	// 1. CHECK: Enforce protocol compliance of Epoch parameters:
	// - RandomSource of entropy in Epoch Setup event should the protocol-prescribed length
	// - first view must be before final view
	if len(setup.RandomSource) != flow.EpochSetupRandomSourceLength {
		return fmt.Errorf("seed has incorrect length (%d != %d)", len(setup.RandomSource), flow.EpochSetupRandomSourceLength)
	}
	if setup.FirstView >= setup.FinalView {
		return fmt.Errorf("first view (%d) must be before final view (%d)", setup.FirstView, setup.FinalView)
	}

	// 2. CHECK: Enforce protocol compliance active participants:
	// (a) each has a unique node ID,
	// (b) each has a unique network address (if `verifyNetworkAddress` is true),
	// (c) participants are sorted in canonical order.
	//     Note that the system smart contracts manage the identity table as an unordered set! For the protocol state, we desire a fixed
	//     ordering to simplify various implementation details, like the DKG. Therefore, we order identities in `flow.EpochSetup` during
	//     conversion from cadence to Go in the function `convert.ServiceEvent(flow.ChainID, flow.Event)` in package `model/convert`
	identLookup := make(map[flow.Identifier]struct{})
	for _, participant := range setup.Participants { // (a) enforce uniqueness of NodeIDs
		_, ok := identLookup[participant.NodeID]
		if ok {
			return fmt.Errorf("duplicate node identifier (%x)", participant.NodeID)
		}
		identLookup[participant.NodeID] = struct{}{}
	}

	if verifyNetworkAddress { // (b) enforce uniqueness of networking address
		addrLookup := make(map[string]struct{})
		for _, participant := range setup.Participants {
			_, ok := addrLookup[participant.Address]
			if ok {
				return fmt.Errorf("duplicate node address (%x)", participant.Address)
			}
			addrLookup[participant.Address] = struct{}{}
		}
	}

	if !setup.Participants.Sorted(flow.Canonical[flow.IdentitySkeleton]) { // (c) enforce canonical ordering
		return fmt.Errorf("participants are not canonically ordered")
	}
```

```go
func convertParticipants(cdcParticipants []cadence.Value) (flow.IdentitySkeletonList, error) {
	participants := make(flow.IdentitySkeletonList, 0, len(cdcParticipants))

	for _, value := range cdcParticipants {
		// checking compliance with expected format
		cdcNodeInfoStruct, ok := value.(cadence.Struct)
		if !ok {
			return nil, invalidCadenceTypeError(
				"cdcNodeInfoFields",
				value,
				cadence.Struct{},
			)
		}

		if cdcNodeInfoStruct.Type() == nil {
			return nil, fmt.Errorf("nodeInfo struct doesn't have type")
		}

		fields := cadence.FieldsMappedByName(cdcNodeInfoStruct)

		const expectedFieldCount = 14
		if len(fields) < expectedFieldCount {
			return nil, fmt.Errorf(
				"insufficient fields (%d < %d)",
				len(fields),
				expectedFieldCount,
			)
		}

		nodeIDHex, err := getField[cadence.String](fields, "id")
		if err != nil {
			return nil, fmt.Errorf("failed to decode nodeInfo struct: %w", err)
		}

		role, err := getField[cadence.UInt8](fields, "role")
		if err != nil {
			return nil, fmt.Errorf("failed to decode nodeInfo struct: %w", err)
		}
		if !flow.Role(role).Valid() {
			return nil, fmt.Errorf("invalid role %d", role)
		}

		address, err := getField[cadence.String](fields, "networkingAddress")
		if err != nil {
			return nil, fmt.Errorf("failed to decode nodeInfo struct: %w", err)
		}

		networkKeyHex, err := getField[cadence.String](fields, "networkingKey")
		if err != nil {
			return nil, fmt.Errorf("failed to decode nodeInfo struct: %w", err)
		}

		stakingKeyHex, err := getField[cadence.String](fields, "stakingKey")
		if err != nil {
			return nil, fmt.Errorf("failed to decode nodeInfo struct: %w", err)
		}

		initialWeight, err := getField[cadence.UInt64](fields, "initialWeight")
		if err != nil {
			return nil, fmt.Errorf("failed to decode nodeInfo struct: %w", err)
		}

		identity := &flow.IdentitySkeleton{
			InitialWeight: uint64(initialWeight),
			Address:       string(address),
			Role:          flow.Role(role),
		}

		// convert nodeID string into identifier
		identity.NodeID, err = flow.HexStringToIdentifier(string(nodeIDHex))
		if err != nil {
			return nil, fmt.Errorf("could not convert hex string to identifer: %w", err)
		}

		// parse to PublicKey the networking key hex string
		networkKeyBytes, err := hex.DecodeString(string(networkKeyHex))
		if err != nil {
			return nil, fmt.Errorf(
				"could not decode network public key into bytes: %w",
				err,
			)
		}
		identity.NetworkPubKey, err = crypto.DecodePublicKey(
			crypto.ECDSAP256,
			networkKeyBytes,
		)
		if err != nil {
			return nil, fmt.Errorf("could not decode network public key: %w", err)
		}

		// parse to PublicKey the staking key hex string
		stakingKeyBytes, err := hex.DecodeString(string(stakingKeyHex))
		if err != nil {
			return nil, fmt.Errorf(
				"could not decode staking public key into bytes: %w",
				err,
			)
		}
		identity.StakingPubKey, err = crypto.DecodePublicKey(
			crypto.BLSBLS12381,
			stakingKeyBytes,
		)
		if err != nil {
			return nil, fmt.Errorf("could not decode staking public key: %w", err)
		}

		participants = append(participants, identity)
	}

	// IMPORTANT: returned identities must be in *canonical order*
	participants = participants.Sort(flow.Canonical[flow.IdentitySkeleton])
	return participants, nil
```
