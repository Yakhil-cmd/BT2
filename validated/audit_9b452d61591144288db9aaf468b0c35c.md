### Title
Gateway Stateful Validator Checks Only L2 Gas Price, Admitting Transactions with Insufficient L1/L1-Data Gas Prices into the Mempool — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful resource-bounds validation checks only the L2 gas `max_price_per_unit` against the previous block's L2 gas price threshold. It never checks `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The blockifier's `check_fee_bounds`, executed later during sequencing, checks all three resources against the actual block prices. A transaction with `l1_gas.max_price_per_unit = 0` (or any value below the actual block L1 gas price) passes both the stateless and stateful gateway validators, enters the mempool, and is only rejected at blockifier pre-validation — after consuming sequencer resources and occupying a mempool slot.

---

### Finding Description

**Stateless validator** (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, lines 56–88) checks only `l2_gas.max_price_per_unit` against the configured absolute `min_gas_price`. L1 gas and L1 data gas prices are not checked. [1](#0-0) 

**Stateful validator** (`crates/apollo_gateway/src/stateful_transaction_validator.rs`, lines 223–243) fetches only `strk_gas_prices.l2_gas_price` from the previous block and calls `validate_tx_l2_gas_price_within_threshold`, which exclusively inspects `l2_gas.max_price_per_unit`. The developer TODO at line 358 explicitly acknowledges the gap: [2](#0-1) [3](#0-2) 

**Blockifier `check_fee_bounds`** (`crates/blockifier/src/transaction/account_transaction.rs`, lines 374–475), called inside `perform_pre_validation_stage` during sequencing, checks all three resources — L1Gas, L1DataGas, and L2Gas — against the actual block gas prices: [4](#0-3) 

The gap: the gateway's two-stage validation (stateless → stateful) only enforces L2 gas price. L1 gas and L1 data gas prices are enforced only at blockifier execution time, after the transaction has already been admitted to the mempool.

---

### Impact Explanation

A transaction with `AllResourceBounds { l1_gas: { max_price_per_unit: 0, ... }, l2_gas: { max_price_per_unit: <valid>, ... }, l1_data_gas: { max_price_per_unit: 0, ... } }` passes both gateway validators and enters the mempool. When the batcher sequences it, `perform_pre_validation_stage` calls `check_fee_bounds`, which returns `ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, ... }`, causing a `TransactionPreValidationError`. The transaction is not included in the block, but the sequencer has already spent resources fetching, validating, and attempting to execute it.

An unprivileged attacker can flood the mempool with such transactions at negligible cost (they pass all gateway checks), displacing legitimate transactions and degrading sequencer throughput. This matches the allowed impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."** [5](#0-4) 

---

### Likelihood Explanation

The attack requires only submitting a V3 `AllResources` transaction with a valid L2 gas price and zero (or below-threshold) L1 gas price. No privileged access, special account, or on-chain state is required. The gateway's `validate_resource_bounds` flag is enabled by default (`validate_resource_bounds: true`), but it only guards L2 gas price. The stateless validator's `min_gas_price` check is also L2-only. There is no existing guard at the gateway layer for L1 or L1-data gas prices. [6](#0-5) 

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the corresponding previous-block prices (`strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price`), applying the same `min_gas_price_percentage` threshold. The existing TODO comment at line 358 already flags this as a known gap. The stateless validator's `validate_resource_bounds` should similarly enforce absolute minimums for L1 and L1-data gas prices, mirroring the existing L2 gas price check.

---

### Proof of Concept

1. Construct a V3 Invoke transaction with:
   ```
   resource_bounds: AllResourceBounds {
       l1_gas:      { max_amount: 1, max_price_per_unit: 0 },
       l2_gas:      { max_amount: 1_000_000, max_price_per_unit: <previous_block_l2_gas_price> },
       l1_data_gas: { max_amount: 1, max_price_per_unit: 0 },
   }
   ```
2. Submit to the gateway. Both `StatelessTransactionValidator::validate` and `StatefulTransactionValidator::validate_resource_bounds` pass — only L2 gas price is checked.
3. The transaction enters the mempool.
4. When the batcher sequences it, `AccountTransaction::perform_pre_validation_stage` calls `check_fee_bounds`, which compares `l1_gas.max_price_per_unit = 0` against the actual block L1 gas price and returns `TransactionFeeError::InsufficientResourceBounds { errors: [ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, ... }] }`.
5. The transaction fails with `TransactionPreValidationError` and is not included in the block.
6. Repeat at scale to exhaust mempool capacity. [3](#0-2) [7](#0-6)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
