### Title
`skip_stateful_validations` Bypasses Signature Verification via Manipulable Mempool State Check — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `skip_stateful_validations` function decides whether to skip the `__validate__` entry-point (signature check) for an invoke transaction by querying `account_tx_in_pool_or_recent_block`. That query reflects live mempool state that any unprivileged attacker can influence. An attacker who observes a victim's `deploy_account` transaction in the mempool can immediately submit an invoke transaction for the same address with an **arbitrary/invalid signature**, and the gateway will admit it without ever running `__validate__`.

### Finding Description

`extract_state_nonce_and_run_validations` executes three sequential steps:

1. Read `account_nonce` from the committed-state snapshot.
2. `run_pre_validation_checks` → `validate_state_preconditions` (nonce range, resource bounds) → `validate_by_mempool` → `skip_stateful_validations`.
3. `run_validate_entry_point` — runs blockifier validation, which calls `__validate__` only when `execution_flags.validate == true`. [1](#0-0) 

`validate_by_mempool` builds a `ValidationArgs` struct that contains address, nonces, tx hash, tip, and gas price — **no signature field** — and forwards it to the mempool. [2](#0-1) 

The mempool's `validate_tx` only checks for duplicate hash and nonce ordering; it never inspects the signature. [3](#0-2) 

`skip_stateful_validations` then returns `true` (skip `__validate__`) when all three conditions hold:

- tx type is Invoke
- `tx.nonce() == Nonce(Felt::ONE)`
- `account_nonce == Nonce(Felt::ZERO)`
- `account_tx_in_pool_or_recent_block(sender)` returns `true` [4](#0-3) 

`account_tx_in_pool_or_recent_block` returns `true` whenever the address appears in the mempool pool **or** in the committed-nonce history — it does not require a `deploy_account` specifically. [5](#0-4) 

When `skip_validate == true`, `run_validate_entry_point` sets `execution_flags.validate = false`: [6](#0-5) 

Inside `StatefulValidator::perform_validations`, the Invoke branch immediately returns `Ok(())` without calling `validate_tx` (i.e., without running `__validate__`): [7](#0-6) 

The comment in `skip_stateful_validations` claims the mempool-presence check is "sufficient" because it implies the account "either has a deploy_account transaction or transactions with future nonces that passed validations." This reasoning is circular: it does not account for the transaction currently being validated, which has not yet passed any signature check. [8](#0-7) 

### Impact Explanation

An attacker can inject an invoke transaction carrying an **invalid or forged signature** for any undeployed account that has a pending transaction in the mempool. The transaction passes all gateway checks and is admitted. When the batcher later executes it, `__validate__` runs and fails, the transaction is reverted, but the sequencer has already consumed resources and the mempool slot was occupied. This directly violates the admission invariant: only signature-verified transactions should enter the mempool.

Matching impact: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

The mempool is observable (transactions are broadcast over P2P). Any attacker can monitor for `deploy_account` transactions, identify the target address, and immediately submit a crafted invoke with nonce=1 and an arbitrary signature. No privileged access, no special contract, no coordination with the victim is required. The window is open from the moment the victim's `deploy_account` enters the mempool until it is committed and the account nonce advances past 0.

### Recommendation

The `skip_stateful_validations` check must be tightened so that it cannot be triggered by an attacker who merely observes a victim's pending deploy. Two complementary approaches:

1. **Require the deploy_account tx hash to be provided by the caller.** The gateway can only skip `__validate__` if the caller explicitly supplies the hash of a co-submitted `deploy_account` transaction (as the Python gateway's `deploy_account_tx_hash` parameter does in `PyValidator::should_run_stateful_validations`). This ties the skip to the same submission session. [9](#0-8) 

2. **Verify the signature against the class declared in the co-submitted deploy_account.** If the class hash is available from the pending deploy, run a lightweight signature check using that class before skipping `__validate__`.

At minimum, the `account_tx_in_pool_or_recent_block` check should be replaced with a check that the **same gateway request** included a `deploy_account` for the same address, eliminating the ability for an external observer to trigger the skip.

### Proof of Concept

**Setup:**
- Victim account `B` is undeployed; `account_nonce(B) == 0` in committed state.
- Victim submits `deploy_account` for `B`; it enters the mempool. `account_tx_in_pool_or_recent_block(B)` now returns `true`.

**Attack:**
1. Attacker observes the mempool and sees `deploy_account` for address `B`.
2. Attacker constructs an `RpcInvokeTransaction` with `sender_address = B`, `nonce = 1`, and an **arbitrary/invalid signature** (e.g., all zeros).
3. Attacker submits the transaction to the gateway via `add_tx`.
4. Gateway flow:
   - `get_nonce_from_state(B)` → `0` (B not deployed).
   - `validate_nonce`: nonce=1, account_nonce=0, within `max_allowed_nonce_gap` → **passes**.
   - `validate_by_mempool`: no duplicate hash, nonce ordering valid → **passes** (no signature check).
   - `skip_stateful_validations`: nonce=1, account_nonce=0, `account_tx_in_pool_or_recent_block(B)` = `true` → returns **`true`**.
   - `run_validate_entry_point` called with `validate = false` → `__validate__` is **never executed**.
5. Invalid transaction is admitted to the mempool.
6. Batcher executes `deploy_account` (B's nonce → 1), then executes attacker's invoke: `__validate__` fails, tx reverts. Sequencer resources wasted; mempool slot consumed; legitimate nonce-1 invoke from B may be displaced. [10](#0-9) [4](#0-3)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
