### Title
Global `last_reward_block` Lock Consumed by Unprivileged Caller via `disable_rewards: true`, Permanently Blocking Consensus Reward Distribution — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract uses a single global `last_reward_block` storage slot as a one-per-block guard. Any unprivileged caller can consume this guard by calling `update_rewards` with `disable_rewards: true`, preventing every other staker from receiving consensus rewards in that block. Executed continuously, this permanently freezes unclaimed yield for all stakers.

---

### Finding Description

`update_rewards` is a public, permissionless function. It enforces a global "one update per block" invariant via a single storage slot:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [1](#0-0) 

The function reads and asserts against this slot, then **unconditionally writes the current block number to it before checking `disable_rewards`**:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number); // ← lock consumed HERE
    
    if disable_rewards || self.is_pre_consensus() {
        return;                                          // ← no rewards distributed
    }
    ...
}
``` [2](#0-1) 

There is no access-control check on `disable_rewards`. `general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero: [3](#0-2) 

The function also requires a valid, active staker address with non-zero balance — information that is entirely public on-chain: [4](#0-3) 

**Root cause**: `last_reward_block` is written before the `disable_rewards` early-return, so calling `update_rewards(any_valid_staker, disable_rewards: true)` consumes the block's reward slot without distributing a single token. Every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

This is structurally identical to the reported vulnerability class: a state lock that can be triggered by an unprivileged actor during a normal operation, causing all legitimate uses of that lock to revert.

---

### Impact Explanation

If an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block:

- `last_reward_block` is consumed each block with no rewards distributed.
- Every legitimate staker's call to `update_rewards` in that block fails.
- Stakers accumulate zero consensus rewards indefinitely.

Alternatively, if the attacker is themselves a staker, they call `update_rewards(attacker_staker, disable_rewards: false)` in every block: they receive their own proportional rewards while all other stakers are permanently locked out. This constitutes both permanent freezing of unclaimed yield for victims and a competitive reward advantage for the attacker.

**Matched impact**: *High — Permanent freezing of unclaimed yield / Theft of unclaimed yield.*

---

### Likelihood Explanation

- The function is fully public with no role restriction.
- The only prerequisite is a valid, active staker address with non-zero balance — trivially obtained from on-chain events.
- The attack is a single transaction per block, automatable with a bot.
- Starknet transaction fees are low, making continuous execution economically viable.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so the lock is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // ← moved here
// ... reward distribution logic
```

Additionally, consider restricting who may call `update_rewards` with `disable_rewards: true` (e.g., only the consensus layer or the staker themselves).

---

### Proof of Concept

1. Attacker observes any valid, active staker address `S` with non-zero STRK balance (from `NewStaker` events).
2. In every block `N`, attacker sends: `update_rewards(S, disable_rewards: true)`.
3. `last_reward_block` is set to `N`; the function returns early — no rewards distributed.
4. Any legitimate staker calling `update_rewards` in block `N` hits the assertion at line 1454–1458 and reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block → all stakers permanently receive zero consensus rewards. [5](#0-4) [6](#0-5)

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`

            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(

                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self

                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);
