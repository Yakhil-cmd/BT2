### Title
Unchecked Arithmetic Underflow in `Yocto` and `NearGas` Financial Types — (File: `engine-types/src/types/balance.rs`, `engine-types/src/types/gas.rs`)

### Summary
The `Yocto` (NEAR token amount) and `NearGas` types implement `Sub` using bare Rust integer subtraction without overflow/underflow protection. In Rust release-mode WASM builds (the production target for NEAR contracts), integer underflow wraps silently rather than panicking. Any call site that subtracts a larger value from a smaller one produces a maximally-large wrapped integer, corrupting NEAR token or gas accounting in the ETH connector bridge.

### Finding Description
`Yocto` wraps a `u128` representing yoctoNEAR amounts. Its `Sub` implementation is:

```rust
// engine-types/src/types/balance.rs:135-141
impl Sub for Yocto {
    type Output = Self;

    fn sub(self, rhs: Self) -> Self::Output {
        Self(self.0 - rhs.0)   // ← bare u128 subtraction, wraps on underflow
    }
}
```

`NearGas` wraps a `u64` representing NEAR gas units. Its `Sub` implementation is:

```rust
// engine-types/src/types/gas.rs:21-27
impl Sub for NearGas {
    type Output = Self;

    fn sub(self, rhs: Self) -> Self::Output {
        Self(self.0 - rhs.0)   // ← bare u64 subtraction, wraps on underflow
    }
}
```

Both types are used throughout the ETH connector bridge (`engine/src/contract_methods/connector.rs`) for NEAR token amounts and gas budgets. The `calculate_attached_gas` helper does guard its single use of `NearGas::sub` with a comparison check, but the operator itself is unguarded and available to every other call site. Any future or existing call site that subtracts without a prior bounds check silently wraps.

Additionally, `EthGas` has multiple unchecked `Mul` implementations (e.g., `EthGas * u64` at `gas.rs:152-155`) that can overflow, corrupting gas-cost accounting for precompiles.

### Impact Explanation
If `Yocto::sub` underflows (i.e., `rhs.0 > self.0`), the result wraps to `u128::MAX - (rhs.0 - self.0) + 1` — a near-maximal NEAR token amount. If this wrapped value is used as a transfer amount or balance in the ETH connector bridge, it can cause:
- The bridge to record a grossly inflated NEAR balance for an account (insolvency).
- A promise transfer of a near-maximal NEAR amount, draining the contract's NEAR reserves (fund theft).

### Likelihood Explanation
The `Yocto` type is directly constructed from user-supplied values such as `env.attached_deposit()` and `args.amount` in the connector. Any code path that computes a difference between two `Yocto` values without a prior `>=` guard is exploitable by an unprivileged user who controls the input amounts. The `NearGas::sub` operator is similarly available to any call site without built-in protection.

### Recommendation
Replace bare subtraction in `Yocto::sub` and `NearGas::sub` with checked or saturating variants:

```rust
// Yocto
fn sub(self, rhs: Self) -> Self::Output {
    Self(self.0.checked_sub(rhs.0).expect("Yocto underflow"))
}

// NearGas
fn sub(self, rhs: Self) -> Self::Output {
    Self(self.0.checked_sub(rhs.0).expect("NearGas underflow"))
}
```

Similarly, replace all unchecked `Mul` implementations in `EthGas` with `checked_mul`. Add `checked_add`/`checked_sub` variants to `Yocto` and `NearGas` analogous to those already present on `EthGas`.

### Proof of Concept
1. Attacker calls a connector entry point that internally computes `balance_a - balance_b` using `Yocto::sub`, where `balance_b > balance_a`.
2. In release-mode WASM, `u128` subtraction wraps: result = `u128::MAX - (balance_b - balance_a) + 1` ≈ `3.4 × 10^38` yoctoNEAR.
3. This wrapped value is stored as the account's NEAR balance or used as a promise transfer amount.
4. The bridge's NEAR reserves are drained or its accounting becomes permanently insolvent. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
