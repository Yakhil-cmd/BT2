### Title
Unauthorized Bank Transfer via Unchecked `sender` Argument in Bank Precompile `transfer` — (`x/cronos/keeper/precompiles/bank.go`)

### Summary

The `TransferMethodName` case in `BankContract.Run` accepts an arbitrary `sender` address from ABI-decoded input and passes it directly to `bankKeeper.SendCoins` without verifying that `sender == contract.Caller()`. Any EVM contract can therefore drain the `evm/<itself>` denom balance of any account that holds such tokens, with no authorization from the victim.

### Finding Description

In `bank.go`, the `transfer` case: [1](#0-0) 

`sender` is taken verbatim from the ABI payload (`args[0]`), and `from` is derived from it. The denom is correctly locked to `EVMDenom(contract.Caller())`, but `from` is never validated against `contract.Caller()`. The call to `bankKeeper.SendCoins(ctx, from, to, ...)` then moves tokens from the victim's account with no signer check inside `ExecuteNativeAction`.

Compare with `mint`/`burn`, which share the same pattern — `addr` (the first ABI argument) is used as the source for `burn` without any check that it equals the caller: [2](#0-1) 

The missing guard is simply:
```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be caller")
}
```

### Impact Explanation

- **Critical — Unauthorized transfer of native bank tokens.**
- A malicious contract at address `A` can call `bank.transfer(victim, attacker, amount)` via the precompile at `0x64`.
- The denom `evm/A` is controlled by contract `A`. Any account that holds `evm/A` tokens (e.g., received via a prior `mint` or `transfer` from contract `A`) can have their entire balance drained to an arbitrary recipient, with zero authorization from the victim.
- `bankKeeper.SendCoins` performs no signer check; authorization is expected to be enforced by the caller (the precompile), which fails to do so here. [3](#0-2) 

### Likelihood Explanation

Reachable by any unprivileged EVM transaction. The attacker only needs to:
1. Deploy a contract (or use an existing one).
2. Ensure at least one victim holds `evm/<attacker_contract>` tokens (trivially achievable by minting tokens to users as a "legitimate" token offering).
3. Call the bank precompile with a crafted ABI payload setting `sender = victim`.

No privileged access, leaked keys, or external assumptions are required.

### Recommendation

Enforce that the `sender` argument equals `contract.Caller()` before executing the transfer:

```go
case TransferMethodName:
    // ...
    sender := args[0].(common.Address)
    if sender != contract.Caller() {
        return nil, errors.New("sender must be the calling contract")
    }
```

Apply the same fix to the `burn` case, where `addr` (the first argument) is used as the source account without any check against `contract.Caller()`. [4](#0-3) 

### Proof of Concept

1. Deploy attacker contract `A` on Cronos EVM.
2. Call `bank.mint(victim, 1000)` from `A` — victim now holds 1000 `evm/A` tokens.
3. From `A`, call `bank.transfer(victim, attacker_eoa, 1000)` via the precompile at `0x64`.
4. Assert: victim's `evm/A` balance = 0, attacker EOA's `evm/A` balance = 1000.
5. Victim signed nothing; no authorization was obtained.
