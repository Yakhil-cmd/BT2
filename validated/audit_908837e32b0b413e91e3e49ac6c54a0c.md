### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check compares a transaction's `max_price_per_unit` against the **current block's** `l2_gas_price`, but the price that will actually govern execution is the **next block's** `next_l2_gas_price` (computed via EIP-1559 mechanics). During high-load periods when `next_l2_gas_price` > `l2_gas_price`, the gateway admits transactions whose `max_price_per_unit` falls in the gap between the two prices. Those transactions pass admission, enter the mempool, and then fail at blockifier execution with `MaxGasPriceTooLow`. The code itself contains a `TODO` acknowledging the wrong value is being used.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price reference value from `BlockInfo.gas_prices.strk_gas_prices.l2_gas_price`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

This value is the L2 gas price **used inside the latest committed block** (block N). The price that will actually apply to the next block (block N+1) is `next_l2_gas_price`, which is computed by the EIP-1559 formula (`calculate_next_l2_gas_price_for_fin`) and stored as a separate field in the block header: [2](#0-1) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `BlockInfo.gas_prices` from `block_header.l2_gas_price` only; `next_l2_gas_price` is never surfaced into `BlockInfo`: [3](#0-2) 

The threshold check then computes `threshold = min_gas_price_percentage% × l2_gas_price(block N)` and rejects transactions below it: [4](#0-3) 

When the network is under load, `next_l2_gas_price` can be materially higher than `l2_gas_price` (the EIP-1559 formula allows up to ~0.3% increase per block, compounding). A transaction with `max_price_per_unit` in the range `[threshold(l2_gas_price(N)), next_l2_gas_price(N+1))` passes the gateway check but will be rejected by `check_fee_bounds` inside `perform_pre_validation_stage` when the batcher executes it against the actual block context (which carries `next_l2_gas_price`): [5](#0-4) 

The `run_validate_entry_point` path also builds its `BlockContext` from the same stale `block_info` (only bumping `block_number`, not the gas price): [6](#0-5) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

Transactions admitted by the gateway with `max_price_per_unit` between the stale threshold and the actual next-block price will:
1. Pass gateway stateful validation and enter the mempool.
2. Fail at blockifier execution inside the batcher with `InsufficientResourceBounds / MaxGasPriceTooLow`.

This pollutes the mempool with transactions that cannot be included in any block until the gas price falls back below their `max_price_per_unit`. Under sustained high load (where `next_l2_gas_price` keeps rising), a large fraction of admitted transactions become permanently unexecutable, degrading throughput and wasting sequencer resources.

The inverse also holds: when `next_l2_gas_price` < `l2_gas_price` (price decreasing), the gateway rejects transactions that are actually valid for the next block, causing unnecessary user-facing errors.

### Likelihood Explanation

The EIP-1559 L2 gas price mechanism is always active. Any period of above-target gas consumption causes `next_l2_gas_price` to diverge upward from `l2_gas_price`. No special privileges are required — any user submitting a V3 (`AllResources`) transaction with a `max_price_per_unit` in the gap triggers the condition. The `TODO` comment at line 229 confirms the developers are aware the wrong field is being read. [7](#0-6) 

### Recommendation

1. Expose `next_l2_gas_price` from `GatewayFixedBlockSyncStateClient` — either by adding it to `BlockInfo` or by adding a dedicated `get_next_l2_gas_price()` method to `GatewayFixedBlockStateReader`.
2. In `validate_resource_bounds`, replace `block_info.gas_prices.strk_gas_prices.l2_gas_price` with the `next_l2_gas_price` read from `block_header_without_hash.next_l2_gas_price`.
3. Apply the same fix to the `BlockContext` constructed in `run_validate_entry_point` so that the blockifier validation also uses the correct next-block price.

### Proof of Concept

1. Observe the current committed block N has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (high-load scenario).
2. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = X` where `threshold(P) ≤ X < P'`.
3. Gateway `validate_resource_bounds` computes `threshold = min_gas_price_percentage% × P` and accepts the transaction (`X ≥ threshold`).
4. Transaction enters the mempool.
5. Batcher pulls the transaction and executes it against block N+1 whose `l2_gas_price = P'`.
6. `check_fee_bounds` finds `X < P'` and returns `InsufficientResourceBounds { MaxGasPriceTooLow }`.
7. Transaction is reverted/dropped; the gateway admission decision was wrong. [8](#0-7) [9](#0-8)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
