### Title
Missing Upper-Bound Validation Before Bit-Packing `weight` into 15-Bit Field Silently Corrupts Account Key Revoked Status - (File: `fvm/environment/account-key-metadata/weight_and_revoked_group.go`)

### Summary

`encodeWeightAndRevokedStatusGroup` packs a `uint16` weight value (range 0–65535) into only 15 bits of a 2-byte field, sharing the high bit with the `revoked` boolean flag. No bounds check enforces that `weight ≤ 0x7FFF` (32767) before encoding. If a weight value with bit 15 set is stored, the decoded result silently produces an incorrect `revoked = true` and a truncated weight, corrupting account key metadata in on-chain state.

### Finding Description

The 2-byte value field in the weight-and-revoked-status RLE group is laid out as:

- Bit 15 (`revokedMask = 0x8000`): revoked flag
- Bits 0–14 (`weightMask = 0x7FFF`): weight (max 32767)

The encoding function accepts `weight uint16` (max 65535) but writes it directly without checking that it fits in 15 bits:

```go
// weight_and_revoked_group.go
func encodeWeightAndRevokedStatusGroup(runLength uint16, revoked bool, weight uint16) []byte {
    var b [weightAndRevokedStatusGroupSize]byte
    binary.BigEndian.PutUint16(b[:], runLength)
    value := weight          // weight may have bit 15 set
    if revoked {
        value |= revokedMask
    }
    binary.BigEndian.PutUint16(b[weightAndRevokedStatusIndex:], value)
    return b[:]
}
``` [1](#0-0) 

The decoder then masks the stored bytes:

```go
weight  = weightAndRevoked & weightMask   // 0x7FFF — truncates bit 15
revoked = (weightAndRevoked & revokedMask) > 0  // reads bit 15 as revoked
``` [2](#0-1) 

The caller in `accounts.go` performs an unchecked narrowing cast from `int` to `uint16` before passing the value into this path:

```go
storedKeyIndex, saveKey, err := a.appendKeyMetadataToAccountStatusRegister(
    address,
    publicKey.Revoked,
    uint16(publicKey.Weight),   // no bounds check
    encodedKey,
)
``` [3](#0-2) 

`publicKey.Weight` is typed as `int` in `flow.AccountPublicKey` and is populated directly from the `weight int` argument supplied by the Cadence runtime: [4](#0-3) 

The `AccountPublicKey.Validate()` function explicitly does **not** validate the weight range (the TODO comment confirms this): [5](#0-4) 

### Impact Explanation

For any key added at index ≥ 1 with `weight ≥ 32768`:

- The stored byte has bit 15 set, so `parseWeightAndRevokedStatus` returns `revoked = true` regardless of the actual revoked argument.
- The decoded weight is `weight & 0x7FFF`, a silently truncated value.

A non-revoked key is permanently recorded as revoked in the account status register. The FVM reads this metadata when loading account keys for transaction authorization: [6](#0-5) 

A key incorrectly marked revoked cannot contribute its weight to transaction authorization, potentially locking an account out of transactions if the corrupted key was its only signing key above the weight threshold.

### Likelihood Explanation

The standard Flow key weight is 1000 (well below 32768), so ordinary usage is unaffected. However:

- The Cadence `Account.keys.add` function accepts `weight: UFix64`, which has no protocol-enforced ceiling in the FVM layer.
- No check in `NewAccountPublicKey`, `appendPublicKey`, or `encodeWeightAndRevokedStatusGroup` rejects `weight > 32767`.
- Any unprivileged transaction sender who owns an account can call `signer.keys.add(... weight: 40000.0)` to trigger the corruption on their own account's key metadata.

The likelihood of accidental triggering is low (non-standard weight), but the path is fully reachable by a transaction sender without any special privilege.

### Recommendation

Add an explicit upper-bound check in `encodeWeightAndRevokedStatusGroup` (or in `appendWeightAndRevokedStatus`) before encoding:

```go
if weight > weightMask {
    return nil, fmt.Errorf("account key weight %d exceeds maximum encodable value %d", weight, weightMask)
}
```

Additionally, add a corresponding check in `NewAccountPublicKey` or `appendPublicKey` to reject weights that exceed the 15-bit field capacity, and document the invariant (`weight ≤ 0x7FFF`) in the encoding format comment. [7](#0-6) 

### Proof of Concept

```go
func TestEncodeWeightOverflow(t *testing.T) {
    // weight = 40000 = 0x9C40, bit 15 is set
    weight := uint16(40000)
    revoked := false

    encoded := encodeWeightAndRevokedStatusGroup(1, revoked, weight)

    // Decode
    decodedRevoked, decodedWeight := parseWeightAndRevokedStatus(encoded, runLengthSize)

    // These will FAIL:
    assert.Equal(t, revoked, decodedRevoked)  // false != true  (bit 15 set → revoked)
    assert.Equal(t, weight, decodedWeight)    // 40000 != 7232  (truncated to 15 bits)
}
```

The call chain from a Cadence transaction is:

```
signer.keys.add(publicKey: pk, hashAlgorithm: ..., weight: 40000.0)
  → AddAccountKey(weight=40000)
  → addAccountKey → appendPublicKey
  → uint16(40000) = 40000  [unchecked cast]
  → appendKeyMetadataToAccountStatusRegister
  → AppendAccountPublicKeyMetadata
  → appendWeightAndRevokedStatus
  → encodeWeightAndRevokedStatusGroup(1, false, 40000)
  → stores 0x9C40 → decoded as revoked=true, weight=7232
```

func encodeWeightAndRevokedStatusGroup(runLength uint16, revoked bool, weight uint16) []byte {
	var b [weightAndRevokedStatusGroupSize]byte

func parseWeightAndRevokedStatus(b []byte, off int) (revoked bool, weight uint16) {
	_ = b[off+1] // bounds check
	weightAndRevoked := binary.BigEndian.Uint16(b[off:])
	weight = weightAndRevoked & weightMask
	revoked = (weightAndRevoked & revokedMask) > 0
	return revoked, weight
}

	error,

	ok, err := updater.accounts.Exists(address)
	if err != nil {
		return nil, fmt.Errorf("adding account key failed: %w", err)
	}
	if !ok {
		return nil, fmt.Errorf(
			"adding account key failed: %w",
			errors.NewAccountNotFoundError(address))
	}

	keyIndex, err := updater.accounts.GetAccountPublicKeyCount(address)
	if err != nil {
		return nil, fmt.Errorf("adding account key failed: %w", err)
	}

	accountPublicKey, err := NewAccountPublicKey(
		publicKey,
		hashAlgo,
		keyIndex,
		weight)
	if err != nil {
		return nil, fmt.Errorf("adding account key failed: %w", err)
	}

	err = updater.accounts.AppendAccountPublicKey(address, *accountPublicKey)
	if err != nil {
		return nil, fmt.Errorf("adding account key failed: %w", err)
	}

	return &runtime.AccountKey{
		KeyIndex:  accountPublicKey.Index,
		PublicKey: publicKey,
		HashAlgo:  hashAlgo,
		Weight:    accountPublicKey.Weight,
		IsRevoked: accountPublicKey.Revoked,
	}, nil
}
