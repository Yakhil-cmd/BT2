### Title
Gateway Admits Invoke Transactions with Unverified Signatures via Unconfirmed Deploy-Account UX Skip — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `skip_stateful_validations` function bypasses the `__validate__` entry point (the account's signature-verification logic) for invoke transactions with nonce=1 from accounts whose on-chain nonce is 0, based solely on the presence of *any* transaction for that address in the mempool. Because the mempool state is unconfirmed — the deploy-account transaction has not been executed on-chain — an attacker can submit an invoke transaction with an arbitrary/invalid signature and have it admitted to the mempool without signature verification.

### Finding Description

The gateway stateful path in `extract_state_nonce_and_run_validations` calls `run_pre_validation_checks`, which in turn calls `skip_stateful_validations`:

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                ...
        }
    }
    Ok(false)
}

Inside `StatefulValidator::perform_validations`, when `validate=false` for an Invoke transaction, execution returns immediately after `perform_pre_validation_stage` — the `__validate__` call is never reached:

```rust

        return Ok(());

The `account_tx_in_pool_or_recent_block` check used as the skip condition only verifies that the account address appears in the mempool pool or recent-block state — it does not verify that the pending transaction is a deploy-account, nor that the deploy-account has been confirmed on-chain:

```rust
pub fn account_tx_in_pool_or_recent_block(&self, account_address: ContractAddress) -> bool {
    self.state.contains_account(account_address)
        || self.tx_pool.contains_account(account_address)
}
``` [4](#0-3) 

The stateless validator checks only signature *length*, not signature *validity*. The only place signature validity is checked is inside the account's `__validate__` entry point, which is exactly what is skipped.

**Attack path:**
1. Attacker computes the deterministic deploy-account address for a chosen class hash and salt.
2. Attacker pre-funds that address with STRK (so the fee check in `perform_pre_validation_stage` passes).
3. Attacker submits a valid `deploy_account` transaction → it passes all validations and enters the mempool.
4. Attacker immediately submits an `invoke` transaction with nonce=1 from the same address, carrying an **arbitrary/invalid signature**.
5. `skip_stateful_validations` returns `true` (nonce=1, on-chain nonce=0, account in mempool).
6. `run_validate_entry_point` runs with `validate=false`; `__validate__` is never called.
7. The invoke transaction with an invalid signature is admitted to the mempool.

### Impact Explanation

An invalid transaction — one whose signature would be rejected by the account's `__validate__` entry point — is admitted through the gateway and into the mempool. This directly matches the allowed impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."** The invalid transaction will eventually fail during batcher execution, but it occupies a mempool slot, consumes sequencer execution resources, and represents a broken admission invariant: every invoke transaction in the mempool should have a verified signature.

### Likelihood Explanation

The attack requires only:
- Knowledge of the deploy-account address derivation (public, deterministic).
- A small STRK pre-fund to pass the fee check.
- Submission of one valid deploy-account transaction followed by one invalid invoke transaction.

No privileged access, no special network position, and no dependency on external systems is required. The condition (`nonce == 1`, `account_nonce == 0`, account in mempool) is trivially constructable by any user.

### Recommendation

The skip should be conditioned on verifying that the pending mempool transaction for the account is specifically a `deploy_account` transaction, not just any transaction. Alternatively, the skip should be removed and the gateway should defer the invoke transaction (e.g., queue it) until the deploy-account is confirmed on-chain, analogous to how the external report's fix deferred the stake-cache update until P-Chain confirmation.

A minimal fix: in `skip_stateful_validations`, instead of calling `account_tx_in_pool_or_recent_block` (which matches any tx type), query the mempool for a deploy-account transaction specifically for the sender address.

### Proof of Concept

```
1. Compute deploy_account address A for class_hash C, salt S, constructor_calldata D.
2. Send STRK to address A (pre-fund for fee check).
3. Submit deploy_account tx: class_hash=C, salt=S, calldata=D, nonce=0, valid signature.
   → Passes all validations; enters mempool. account_tx_in_pool_or_recent_block(A) = true.
4. Submit invoke tx: sender=A, nonce=1, calldata=<anything>, signature=[0x0, 0x0] (invalid).
   → skip_stateful_validations returns true (nonce==1, account_nonce==0, A in mempool).
   → run_validate_entry_point called with validate=false.
   → __validate__ never runs. Signature never checked.
   → Invoke tx with invalid signature admitted to mempool.
5. Batcher later executes deploy_account (deploys contract), then executes invoke tx.
   → __validate__ runs, rejects invalid signature, tx fails.
   → Invalid tx was in mempool and consumed execution resources.

        let charge_fee = enforce_fee(executable_tx, only_query);
        let strict_nonce_check = false;

            ExecutionFlags { only_query, charge_fee, validate: !skip_validate, strict_nonce_check };

        let account_tx = AccountTransaction { tx: executable_tx.clone(), execution_flags };

        // Build block context.
        let mut versioned_constants = VersionedConstants::get_versioned_constants(

        // The validation of a transaction is not affected by the casm hash migration.
        versioned_constants.disable_casm_hash_migration();

        let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
        block_info.block_number = block_info.block_number.unchecked_next();
        let block_context = BlockContext::new(
            block_info,
            self.chain_info.clone(),
            versioned_constants,
            BouncerConfig::max(),
        );

        // Move state into the blocking task and run CPU-heavy validation.
        let state_reader_and_contract_manager = self.take_state_reader_and_contract_manager();

        let cur_span = Span::current();

        tokio::task::spawn_blocking(move || {

                blockifier_validator.validate(account_tx)

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
