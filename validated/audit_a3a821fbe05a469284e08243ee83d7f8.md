### Title
Commission Change Without Prior Reward Accrual Allows Staker to Retroactively Steal Delegator Yield - (File: src/staking/staking.cairo)

### Summary
`set_commission` updates the commission rate in storage immediately without first accruing rewards at the old rate. Because `update_rewards_from_attestation_contract` (V2) reads the commission from storage at attestation time and applies it to the **entire epoch**, a staker who increases commission mid-epoch (via the commitment mechanism) retroactively applies the higher rate to all delegator rewards for that epoch, stealing yield that delegators had already earned.

### Finding Description
`set_commission` in `src/staking/staking.cairo` calls `update_commission`, which writes the new commission directly to storage with no reward settlement step: [1](#0-0) 

The internal `update_commission` function performs only validation and a storage write: [2](#0-1) 

When `update_rewards_from_attestation_contract` is later called (once per epoch), it reads the commission from storage at that moment and applies it to the full epoch's rewards: [3](#0-2) 

Inside `_update_rewards`, `calculate_staker_pools_rewards` reads the live commission: [4](#0-3) 

The `set_commission_commitment` mechanism explicitly permits a staker to set a `max_commission` up to `COMMISSION_DENOMINATOR` (10 000 = 100%) and then call `set_commission` to raise the rate to any value ≤ `max_commission` during the commitment window: [5](#0-4) 

Because there is no reward accrual before the commission write, the new (higher) commission is retroactively applied to the entire epoch when the staker attests.

### Impact Explanation
**High — Theft of unclaimed yield from delegators.**

A staker with a commitment can raise commission to 100% immediately before attesting. `update_rewards_from_attestation_contract` then calculates the full epoch's pool rewards with `commission = 10000`, leaving delegators with zero yield for that epoch. All pool rewards flow to the staker as commission. Delegators earned those rewards throughout the epoch at the original lower rate; the retroactive commission change confiscates them.

### Likelihood Explanation
**Medium.** The attack requires two deliberate on-chain steps by the staker: (1) call `set_commission_commitment` with a high `max_commission`, and (2) call `set_commission` to the maximum just before attesting. Both calls are permissionless for the staker address. A rational malicious staker can execute this in the same epoch, and there is no time-lock or epoch delay between setting a commitment and using it. The only prerequisite is that the staker has an active delegation pool with delegators.

### Recommendation
Accrue rewards at the current commission rate before writing the new commission. In `set_commission`, call the reward-accrual logic (equivalent to `_update_rewards` / `update_rewards_from_attestation_contract`) for the staker before invoking `update_commission`. This ensures the old rate is settled for all blocks/epochs elapsed so far, and the new rate only applies going forward.

### Proof of Concept
1. Staker stakes with `commission = 500` (5%) and opens a delegation pool.
2. Delegators join the pool. Epoch N begins.
3. Near the end of epoch N (after most rewards have accrued), staker calls:
   - `set_commission_commitment(max_commission: 10000, expiration_epoch: N+1)`
   - `set_commission(commission: 10000)` — commission is now 100%, stored immediately with no reward settlement.
4. Staker attests in epoch N. `update_rewards_from_attestation_contract` is called.
5. `calculate_staker_pools_rewards` reads `commission = 10000` from storage.
6. `split_rewards_with_commission` assigns 100% of pool rewards to the staker as commission; delegators receive 0.
7. Delegators who held funds in the pool for the entire epoch receive no yield for that epoch. [6](#0-5) [7](#0-6)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();

                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,

            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self

                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
