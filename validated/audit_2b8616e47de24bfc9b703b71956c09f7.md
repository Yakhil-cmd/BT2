### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`setPoolAdminFees` enforces `maxAdminSpreadFeeE6` on the global admin spread component, but `setPoolBinAdditionalFees` passes `addFeeBuyE6`/`addFeeSellE6` directly to the pool with **no cap check**, allowing a pool admin to set per-bin additional fees up to `type(uint16).max` = 65535 (6.5535%) and extract fees from traders far beyond the factory-enforced ceiling.

---

### Finding Description

The factory enforces a hard cap on the global admin spread fee in `setPoolAdminFees`:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`maxAdminSpreadFeeE6` is itself bounded by `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20%): [2](#0-1) 

However, `setPoolBinAdditionalFees` performs **no analogous cap check** — it forwards the raw `uint16` values directly to the pool:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [3](#0-2) 

`setBinAdditionalFees` on the pool stores the values without any validation:

```solidity
// MetricOmmPool.sol:464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
  if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
  BinState storage s = _binStates[bin];
  s.addFeeBuyE6 = addFeeBuyE6;
  s.addFeeSellE6 = addFeeSellE6;
  emit BinAdditionalFeesUpdated(bin, addFeeBuyE6, addFeeSellE6);
}
``` [4](#0-3) 

These per-bin fees are then **added on top of** the base spread fee in every swap through that bin:

```solidity
// MetricOmmPool.sol:999
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6),
``` [5](#0-4) 

The `BinState` struct stores `addFeeBuyE6` and `addFeeSellE6` as `uint16`, so the maximum settable value is 65535 (6.5535% in E6 units): [6](#0-5) 

---

### Impact Explanation

A pool admin can set `addFeeBuyE6 = addFeeSellE6 = 65535` on any bin. Every swap routed through that bin pays an additional 6.5535% on top of the base spread fee — far exceeding the `maxAdminSpreadFeeE6` cap (which is at most 20% globally, but typically much lower). The excess fee accrues as LP surplus inside the pool and is later swept to the admin fee destination via `collectPoolFees`. Traders suffer direct, quantifiable loss of principal on every swap through the affected bin, with no on-chain mechanism to prevent or revert it.

---

### Likelihood Explanation

The pool admin is a semi-trusted role explicitly called out in the audit pivots as a boundary to check for `setBinAdditionalFees`. The call path is a single transaction with no timelock. Any pool whose admin is compromised, malicious, or simply misconfigured can immediately exploit this. The asymmetry — global fees capped, per-bin fees uncapped — is a clear design gap rather than an intentional permission.

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` mirroring the one in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

This ensures per-bin additional fees are bounded by the same factory-enforced ceiling as the global admin spread fee.

---

### Proof of Concept

1. Deploy a pool with `maxAdminSpreadFeeE6 = 5_000` (0.5%).
2. Pool admin calls `factory.setPoolBinAdditionalFees(pool, 0, 65535, 65535)` — no revert.
3. A trader swaps through bin 0; the effective buy fee is `baseFeeX64 + mulDiv(65535, ONE_X64, 1e6)` — 6.5535% additional on top of the base spread.
4. Assert the effective fee exceeds `maxAdminSpreadFeeE6` by ~13×.
5. `collectPoolFees` sweeps the excess surplus to the admin fee destination.
