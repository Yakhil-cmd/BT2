### Title
Staker Can Instantly Raise Commission to 100% via `set_commission_commitment`, Stealing Delegator Yield — (`src/staking/staking.cairo`)

---

### Summary

The `set_commission_commitment` mechanism is intended to give delegators advance notice before a staker raises their commission. However, the minimum commitment window is only 1 epoch (`expiration_epoch > current_epoch`). A malicious staker can set a commitment with `max_commission = 10000` (100%) expiring in 1 epoch, then **immediately** call `set_commission(10000)` in the same transaction or block. The new 100% commission applies to all existing delegators at the very next reward calculation, redirecting all pool yield to the staker. Delegators who joined under a low commission rate (e.g., 2%) receive zero yield with no practical ability to exit in time.

---

### Finding Description

Without a commitment, `set_commission` enforces that commission can only decrease: [1](#0-0) 

With an active commitment, the check is relaxed — commission can be set to any value up to `max_commission`, and the only constraint is that it differs from the old value: [2](#0-1) 

The commitment itself only requires `expiration_epoch > current_epoch`, meaning a 1-epoch window is the minimum: [3](#0-2) 

And `max_commission` can be set up to `COMMISSION_DENOMINATOR` (10000 = 100%): [4](#0-3) 

The attack sequence in two transactions:
1. `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`
2. `set_commission(commission: 10000)`

After step 2, the commission stored in `staker_pool_info` is 100%. The next time `calculate_staker_pools_rewards` runs, it reads the current commission and applies it: [5](#0-4) 

The `split_rewards_with_commission` call at line 1989–1990 uses the live commission value, so all pool rewards are routed to the staker as commission, and delegators receive zero.

The developers themselves acknowledge this gap in a code comment: [6](#0-5) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Delegators who joined a pool at a 2% commission rate can have their entire future yield stolen by the staker raising commission to 100% in a single epoch. The commission is applied at reward-distribution time (not at delegation time), so all rewards accruing after the commission change are captured by the staker. Delegators receive `pool_rewards_including_commission * (1 - 10000/10000) = 0`. [7](#0-6) 

---

### Likelihood Explanation

**Medium.** Any staker with an active pool can execute this attack permissionlessly. No leaked keys or external dependencies are required. The only prerequisite is that the staker has a pool (i.e., has called `set_open_for_delegation`). The attack is economically rational for a staker who wants to extract maximum value from their delegators before exiting the protocol. The 1-epoch minimum window is too short for delegators to monitor, react, and submit an `exit_delegation_pool_intent` transaction.

---

### Recommendation

1. **Enforce a meaningful minimum commitment window** — require `expiration_epoch >= current_epoch + MIN_EPOCHS` (e.g., at least 7–14 epochs) so delegators have time to observe the `CommissionCommitmentSet` event and exit before the higher commission takes effect.
2. **Delay commission increases** — apply a raised commission only starting from `expiration_epoch + 1`, not immediately upon calling `set_commission`. This mirrors the recommendation in M-03 to make new rates apply only to future interactions.
3. **Cap the maximum commission increase per commitment** — limit how much commission can be raised in a single commitment cycle (e.g., no more than +500 bps per cycle).

---

### Proof of Concept

```
// Staker has commission = 200 (2%), delegators have joined expecting 2% commission.

// Step 1: Set commitment with max_commission = 10000, expiring next epoch.
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1);
// Emits CommissionCommitmentSet — but delegators may not be monitoring.

// Step 2: Immediately raise commission to 100%.
staking.set_commission(commission: 10000);
// update_commission passes because active commitment allows commission <= max_commission (10000).
// commission != old_commission (10000 != 200) — passes.

// Next attestation/reward update:
// calculate_staker_pools_rewards reads commission = 10000
// split_rewards_with_commission(pool_rewards_including_commission, 10000)
//   => commission_rewards = pool_rewards_including_commission * 10000 / 10000 = all rewards
//   => pool_rewards = 0
// Delegators receive 0 yield. Staker captures 100% of pool rewards.
``` [8](#0-7) [9](#0-8)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
