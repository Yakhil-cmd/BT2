Audit Report

## Title
Unbounded Loop via `Time::add_assign` u128→u64 Truncation Causes OOM Crash in `construction_payloads` — (`rs/rosetta-api/icp/src/request_handler/construction_payloads.rs`)

## Summary

The `construction_payloads` handler builds an `ingress_expiries` vector by looping from `ingress_start` to `ingress_end` in 120-second steps. The `+=` operator on `Time` delegates to `Time::from_duration`, which silently truncates a `u128` nanosecond sum to `u64` via an `as` cast. An unauthenticated caller can supply `ingress_start` and `ingress_end` values near `u64::MAX` that cause `now` to wrap to zero on the first iteration, making the termination condition permanently true and driving the process to OOM.

## Finding Description

**Root cause — truncating cast in `Time::from_duration`:**

`Time::add_assign` at [1](#0-0)  calls `Time::from_duration`, which performs `t.as_nanos() as u64` at [2](#0-1) . `Duration::as_nanos()` returns `u128`; the `as u64` cast silently truncates any value exceeding `u64::MAX`.

**The loop:**

`construction_payloads` at [3](#0-2)  iterates `now` from `ingress_start` to `ingress_end` using `now += interval`, where `interval = MAX_INGRESS_TTL − PERMITTED_DRIFT − 120s = 120_000_000_000 ns` per [4](#0-3)  and [5](#0-4) .

**No bounds validation on caller-supplied values:**

`ingress_start` and `ingress_end` are taken directly from the JSON request body and passed to the bare `Time(nanos)` constructor with no range check at [6](#0-5)  and [7](#0-6) .

**Exploit flow:**

Set `ingress_start = u64::MAX − 120_000_000_000 + 1` and `ingress_end = u64::MAX`.

- Iteration 1: `now < ingress_end` → true; `now += interval` computes `(u64::MAX − 120_000_000_000 + 1 + 120_000_000_000)` = `u64::MAX + 1` as `u128`, which truncates to `0` via `as u64`.
- Iteration 2+: `now = 0 < u64::MAX` → permanently true; `now` increments by 120 s each step, wrapping again after ~153 billion iterations.
- Each iteration pushes a `u64` (8 bytes) onto `ingress_expiries`; the heap is exhausted (OOM / process crash) long before the first wrap cycle completes.

`Time::checked_add` exists at [8](#0-7)  and performs a safe checked conversion, but is not used in the loop.

## Impact Explanation

A single malformed HTTP request crashes the Rosetta node process via OOM. The Rosetta API is an explicitly in-scope financial integration target. This matches the **High ($2,000–$10,000)** impact class: *"Application/platform-level DoS, crash … not based on raw volumetric DDoS"* and *"Significant … Rosetta … infrastructure security impact with concrete user or protocol harm."* Availability of the Rosetta node is directly and completely lost until the process is restarted; the attack is repeatable with a single request.

## Likelihood Explanation

- `/construction/payloads` is a public HTTP endpoint requiring no authentication.
- `ingress_start` and `ingress_end` are optional JSON integer fields; no schema-level constraint prevents near-`u64::MAX` values.
- No server-side guard exists between JSON deserialization and the loop.
- The exploit is deterministic, requires no special privileges, and is reproducible with a single HTTP request.

## Recommendation

1. **Replace the truncating cast** in `Time::from_duration` with a checked or saturating conversion, or remove the function and require callers to use `Time::checked_add`.
2. **Use `Time::checked_add`** (already present at `time.rs` lines 122–128) instead of `+=` in the loop body, returning an error on overflow.
3. **Add an upper-bound guard** before the loop:
   ```rust
   const MAX_EXPIRIES: u64 = 1000;
   if ingress_end > ingress_start + interval * MAX_EXPIRIES {
       return Err(ApiError::invalid_request("ingress window too large"));
   }
   ```

## Proof of Concept

```rust
// Unit test — no network required
use ic_types::time::Time;
use std::time::Duration;

let interval_ns: u64 = 120_000_000_000;
let ingress_start = u64::MAX - interval_ns + 1;
let ingress_end   = u64::MAX;

let mut count = 0usize;
let mut now = Time::from_nanos_since_unix_epoch(ingress_start);
let end     = Time::from_nanos_since_unix_epoch(ingress_end);
let dur     = Duration::from_nanos(interval_ns);

while now < end {
    count += 1;
    assert!(count <= 2, "wrap detected: now wrapped to 0, loop is unbounded");
    now += dur;  // wraps to 0 on first iteration with current code
}
```
