### Title
Permissionless `update_rewards` with global block-level rate limit allows griefing of consensus reward distribution - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the Staking contract has no access control. A global `last_reward_block` storage variable limits reward updates to one call per block. Any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` to consume the block's reward slot without distributing any rewards, causing every subsequent legitimate call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

### Finding Description
`update_rewards` is gated only by `general_prerequisites`, which checks that the contract is not paused and the caller is non-zero — no role or address restriction exists. [1](#0-0) 

The function enforces a **global** per-block rate limit: [2](#0-1) 

It then writes `last_reward_block` unconditionally before checking `disable_rewards`: [3](#0-2) 

If `disable_rewards` is `true`, the function returns immediately after updating `last_reward_block`, distributing nothing. Because `last_reward_block` is now equal to the current block, every subsequent call in the same block fails at the assertion on line 1455–1458.

The attacker only needs to supply any currently-active staker address with non-zero STRK balance (trivially readable from the public `stakers` vector and `staker_info` map) to pass the validity checks at lines 1466–1482. [4](#0-3) 

### Impact Explanation
By calling `update_rewards(any_active_staker, disable_rewards: true)` in every block, an attacker prevents all consensus block rewards from ever being distributed to any staker. This matches the allowed impact **"Griefing with no profit motive but damage to users or protocol"** (Medium) and, if sustained, escalates to **"Permanent freezing of unclaimed yield"** (High), because no staker can accumulate consensus rewards while the attack is active.

### Likelihood Explanation
Starknet transaction fees are low. The attacker needs one cheap transaction per block — no privileged access, no token holdings, no leaked keys. Any anonymous address can execute this indefinitely. The entry path is fully reachable by an unprivileged public caller.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the consensus contract or a designated operator role). Add a role check analogous to the `only_security_agent` / `only_app_governor` pattern already used elsewhere in the contract: [5](#0-4) 

Alternatively, remove the `disable_rewards` parameter from the public interface and expose it only through an internal path called by privileged functions.

### Proof of Concept
1. Attacker reads any active staker address from the public `stakers` vector.
2. Each block, attacker submits: `update_rewards(active_staker_address, disable_rewards: true)`.
3. The call passes all checks, writes `last_reward_block = current_block`, then returns early — zero rewards distributed.
4. The consensus mechanism's legitimate `update_rewards` call in the same block hits the assertion `current_block_number > last_reward_block` → reverts with `REWARDS_ALREADY_UPDATED`.
5. No staker receives consensus block rewards for any block the attacker griefs. Sustained over time, all unclaimed consensus yield is frozen. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
