### Title
Gateway Admission Validates Only L2 Gas Price, Silently Admitting Transactions with Zero L1/L1-Data Gas Price Bounds — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`, `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

Both the stateless and stateful gateway validators check only the L2 gas price component of `AllResourceBounds`. Neither validator checks `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` against the current block prices. A transaction with a valid L2 gas price but zero (or arbitrarily low) L1 gas and L1 data gas prices passes all gateway admission checks and enters the mempool, yet it will fail blockifier pre-validation (`check_fee_bounds`) with `InsufficientResourceBounds` when the sequencer attempts to execute it. The gateway's admission invariant — that admitted transactions have resource bounds sufficient to cover actual block gas prices — is broken for the L1 and L1-data gas dimensions.

---

### Finding Description

**Stateless validator** (`validate_resource_bounds`, lines 56–88):

The gateway validates only one of the three resource-price dimensions that the blockifier enforces, creating a mismatch between what is admitted and what can actually execute.

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

Any unprivileged user can craft a V3 `AllResourceBounds` transaction with:
- `l2_gas.max_price_per_unit` ≥ `min_gas_price` (passes stateless check)
- `l2_gas.max_price_per_unit` ≥ threshold × previous-block L2 price (passes stateful check)
- `l1_gas.max_price_per_unit = 0` and/or `l1_data_gas.max_price_per_unit = 0`

Such a transaction passes all gateway admission checks and is forwarded to the mempool. When the batcher attempts to execute it, `perform_pre_validation_stage` → `check_fee_bounds` fails with `InsufficientResourceBounds` for L1Gas or L1DataGas, causing the transaction to be dropped without inclusion. The sequencer has consumed validation, mempool, and batcher resources for a transaction that was structurally invalid at submission time.

---

### Likelihood Explanation

**High.** The attack requires no privileged access, no special account state, and no coordination. A single HTTP POST to the gateway endpoint with a crafted `AllResourceBounds` payload suffices. The gap is present in both the stateless and stateful paths, so there is no secondary filter.

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or replace it with a new `validate_all_gas_prices_within_threshold`) to also fetch `strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price` from the previous block and apply the same percentage-threshold check to `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit`. The existing TODO comment at line 358 already flags this as a known gap. [4](#0-3) 

Similarly, the stateless validator should add floor checks for `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` analogous to the existing `l2_gas` check. [5](#0-4) 

---

### Proof of Concept

1. Obtain the current L2 gas price from the previous block (e.g., via `starknet_getBlockWithTxHashes`).
2. Construct a V3 Invoke transaction with:
   ```json
   "resource_bounds": {
     "l1_gas":      { "max_amount": "0x1000", "max_price_per_unit": "0x0" },
     "l2_gas":      { "max_amount": "0x1",    "max_price_per_unit": "<valid_l2_price>" },
     "l1_data_gas": { "max_amount": "0x1000", "max_price_per_unit": "0x0" }
   }
   ```
3. Submit via `starknet_addInvokeTransaction`.
4. **Expected (buggy) result**: Gateway returns a transaction hash; the transaction enters the mempool.
5. **Actual blockifier result**: When the batcher processes the transaction, `check_fee_bounds` returns `InsufficientResourceBounds { resource: L1Gas, max_gas_price: 0, actual_gas_price: <non-zero> }` and the transaction is dropped.
6. Repeat at high frequency to exhaust mempool capacity and batcher CPU with zero-cost-to-attacker invalid transactions.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
