### Title
Staker Can Instantly Raise Commission to 100% via `set_commission_commitment` + `set_commission`, Stealing All Delegator Yield - (File: src/staking/staking.cairo)

---

### Summary

The `set_commission_commitment` / `set_commission` pair in `src/staking/staking.cairo` contains no delay between when a commitment is set and when the commission can be raised to `max_commission`. A staker can atomically increase commission from any low value to 100% in a single block, with zero advance notice to delegators, stealing all pool rewards accrued during the mandatory exit window.

---

### Finding Description

`set_commission_commitment` (line 748) lets a staker record a `CommissionCommitment { max_commission, expiration_epoch }`. The only constraints are:

- `current_commission <= max_commission` (line 770)
- `expiration_epoch > current_epoch` (line 771) — minimum is `current_epoch + 1` [1](#0-0) 

Once the commitment is written, `update_commission` (line 1573) — called by `set_commission` — checks only:

```
assert!(commission <= commission_commitment.max_commission, ...);
assert!(commission != old_commission, ...);
``` [2](#0-1) 

There is **no minimum elapsed time** between setting the commitment and raising the commission. A staker can therefore, in a single block:

1. Call `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — valid because `current_commission (e.g. 200) <= 10000`.
2. Immediately call `set_commission(10000)` — valid because `10000 <= max_commission` and `10000 != 200`.

Commission jumps from 2% to 100% with no advance notice. The developers themselves acknowledge this gap in the inline comment:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [3](#0-2) 

Delegators who entered the pool based on the advertised low commission cannot exit immediately. They must call `exit_delegation_pool_intent` and then wait the full `exit_wait_window` (default 1 week, up to 12 weeks) before `exit_delegation_pool_action` is available. All rewards minted during this window are captured entirely by the staker. [4](#0-3) 

The spec's pre-condition for `set_commission` confirms the only guard when a commitment is active is the `max_commission` ceiling — no time-lock exists: [5](#0-4) 

---

### Impact Explanation

All delegator rewards accrued from the moment of the commission change until each delegator completes the full exit process are redirected to the staker's `reward_address`. With a minimum 1-week exit window and a large pool, this constitutes direct, quantifiable **theft of unclaimed yield** from delegators — matching the allowed High impact: *"Theft of unclaimed yield or unclaimed royalties."*

---

### Likelihood Explanation

Any registered staker who has opened a delegation pool can execute this attack. No governance role, leaked key, or external dependency is required. The attack is atomic (two sequential calls in the same block), requires no special timing beyond the current epoch, and leaves no on-chain warning visible to delegators before the commission change takes effect.

---

### Recommendation

Enforce a minimum delay between when a commission commitment is set and when the commission can be raised above its previous value. For example:

- Require that `set_commission_commitment` be called at least `K` epochs before any upward commission change takes effect (mirroring the balance-checkpoint delay already used elsewhere in the protocol).
- Alternatively, make upward commission changes effective only after a delay equal to `exit_wait_window`, giving delegators a full exit window to react before the new rate applies.

---

### Proof of Concept

1. Staker registers with `commission = 200` (2%) and opens a STRK delegation pool.
2. Delegators enter the pool, trusting the 2% commission rate.
3. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — succeeds because `200 <= 10000` and `expiration_epoch > current_epoch`.
4. In the **same block**, staker calls `set_commission(10000)` — succeeds because the commitment is active (`current_epoch < current_epoch + 1`) and `10000 <= 10000`.
5. Commission is now 100%; all pool rewards flow to the staker's reward address.
6. Delegators must wait the full `exit_wait_window` (≥ 1 week) before recovering principal; every reward minted during this period is lost to them.

This exact flow is exercised (with a small increase rather than 100%) in the existing test `test_set_commission_with_commitment`: [6](#0-5)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );

#[test]
fn test_set_commission_with_commitment() {
    let mut cfg: StakingInitConfig = Default::default();
    let staking_contract = deploy_staking_contract(:cfg);
    cfg.test_info.staking_contract = staking_contract;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    stake_with_strk_pool_enabled(:cfg);

    // Set commitment.
    let staker_address = cfg.test_info.staker_address;
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    let max_commission = staker_info.get_pool_info().commission + 2;
    let expiration_epoch = staking_dispatcher.get_current_epoch() + 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission_commitment(:max_commission, :expiration_epoch);

    // Update commission.
    let mut commission = max_commission;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission(:commission);

    // Assert commission is updated.
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info.get_pool_info().commission == commission);

    // Advance to the expiration epoch.
    advance_epoch_global();

    // Lower commission.
    commission = commission - 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission(:commission);

    // Assert commission is updated.
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info.get_pool_info().commission == commission);
}
