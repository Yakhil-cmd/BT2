### Title
Griefing via `update_rewards(disable_rewards=true)` Permanently Freezes Per-Block Unclaimed Yield — (`src/staking/staking.cairo`)

---

### Summary

Any unprivileged caller can invoke `update_rewards` with `disable_rewards = true` for any staker, advancing the global `last_reward_block` checkpoint without distributing rewards. Because the contract enforces exactly one call per block, the legitimate reward-earning call for that block is permanently blocked, and the staker's yield for that block is irrecoverably lost.

---

### Finding Description

`update_rewards` is a public, permissionless function in `StakingRewardsManagerImpl`. It accepts a `disable_rewards: bool` parameter. When `true`, the function still writes the current block number to the global `last_reward_block` storage slot, but returns before computing or distributing any rewards. [1](#0-0) 

The guard that enforces one-call-per-block is: [2](#0-1) 

After the guard passes, `last_reward_block` is unconditionally written: [3](#0-2) 

Then, if `disable_rewards` is `true`, the function returns early with no reward distribution: [4](#0-3) 

`last_reward_block` is a **global** (not per-staker) storage variable: [5](#0-4) 

The only access control applied is `general_prerequisites()`, which only checks for pause state and a non-zero caller: [6](#0-5) 

---

### Impact Explanation

Because `last_reward_block` is global, a single call to `update_rewards(any_staker, true)` in block N:

1. Advances `last_reward_block` to N.
2. Distributes **zero** rewards.
3. Makes every subsequent call to `update_rewards` in block N revert with `REWARDS_ALREADY_UPDATED`.

The rewards that would have been earned in block N are permanently lost — they are not deferred or accumulated; the per-block reward calculation is simply skipped and never retried for that block. An attacker repeating this every block can suppress **all** consensus-era staking rewards indefinitely at the cost of gas alone.

This matches the allowed impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- `update_rewards` is fully public with no role restriction.
- The attacker needs only a non-zero address and enough gas to call it once per block.
- No privileged access, no leaked key, no bridge compromise is required.
- On Starknet, transaction ordering is controlled by the sequencer; a colluding or malicious sequencer, or any actor who submits the transaction before the legitimate consensus call, can trigger this.
- The attack is cheap and repeatable.

---

### Recommendation

Restrict the `disable_rewards = true` path to a privileged role (e.g., `security_agent` or `app_governor`), or remove the `disable_rewards` parameter from the public interface entirely and expose a separate privileged function for the disable-only use case. Alternatively, make `last_reward_block` per-staker so that one call cannot block all others.

---

### Proof of Concept

1. Staker Alice is active and eligible for consensus block rewards in block N.
2. Attacker Bob (any address) calls `update_rewards(alice_address, true)` in block N.
3. `last_reward_block` is written to N; no rewards are distributed.
4. Alice (or the consensus mechanism) calls `update_rewards(alice_address, false)` in block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Alice's rewards for block N are permanently lost.
6. Bob repeats every block → Alice earns zero rewards indefinitely. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
