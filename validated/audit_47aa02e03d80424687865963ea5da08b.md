### Title
Unimplemented `TurnBridge` Handler Permanently Nullifies Bridge Circuit Breaker and Bypasses `CanTurnBridge` Permission — (File: `x/cronos/keeper/msg_server.go`)

---

### Summary

The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is a stub that unconditionally returns `nil, nil` without performing any permission check, writing any bridge-enabled state, or doing anything at all. As a result: (1) any unprivileged address can call `MsgTurnBridge` and it succeeds, bypassing the `CanTurnBridge` permission system; and (2) the bridge circuit breaker is permanently non-functional — no bridge-enabled flag is ever stored, and no EVM hook checks one, so the bridge can never be disabled.

---

### Finding Description

`MsgTurnBridge` is the protocol-defined message for enabling or disabling the Gravity/IBC bridge in an emergency. The proto service declares it as:

```
// TurnBridge defines a method to disable or enable the gravity bridge
rpc TurnBridge(MsgTurnBridge) returns (MsgTurnBridgeResponse);
```

The keeper implementation is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Compare this to `UpdateTokenMapping`, which correctly gates on `HasPermission`:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [2](#0-1) 

`TurnBridge` performs neither the `HasPermission(ctx, ..., CanTurnBridge)` check nor any KV-store write. The `Params` struct has no `bridge_enabled` field: [3](#0-2) 

And none of the EVM hook handlers (`SendToIbcHandler`, `SendToIbcV2Handler`, `SendCroToIbcHandler`) check any bridge-enabled gate before executing bridge transfers: [4](#0-3) [5](#0-4) 

The `CanTurnBridge` permission constant is defined and the permission system is wired up: [6](#0-5) 

But it is never enforced in the handler that is supposed to use it.

---

### Impact Explanation

**High — Bypass of Cronos bridge authorization checks.**

Two distinct bypasses exist:

1. **Permission bypass**: Any unprivileged address can submit `MsgTurnBridge` and receive a success response (`nil, nil`). The `CanTurnBridge` permission — which the protocol explicitly defines and the ADR-009 design requires — is never checked. This is a direct bypass of the bridge authorization control.

2. **Circuit-breaker nullification**: Because the handler writes no state and no EVM hook reads a bridge-enabled flag, the bridge can never be disabled. During a live exploit draining funds through `__CronosSendToIbc` or `__CronosSendCroToIbc` events, the admin's only on-chain emergency lever (`MsgTurnBridge`) is silently swallowed. Bridge flows continue unimpeded regardless of what the admin submits.

This matches: **High: Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks.**

---

### Likelihood Explanation

The permission bypass is reachable by any address with no preconditions — just a valid bech32 sender. The circuit-breaker failure is permanent and structural; it requires no attacker action to trigger. The only precondition for the circuit-breaker impact to matter is an active exploit of the bridge, which is precisely the scenario the mechanism was designed to address.

---

### Recommendation

1. **Implement the handler body**: Check `HasPermission(ctx, []sdk.AccAddress{senderAddr}, CanTurnBridge)` and return `ErrUnauthorized` on failure.
2. **Add a `bridge_enabled` boolean to `Params`** (or a dedicated KV key) and persist `msg.Enable` in the handler.
3. **Gate all EVM bridge hooks** on the stored bridge-enabled flag at the top of each `Handle` method, returning an error (not silently ignoring) when the bridge is disabled.
4. **Add a `ValidateBasic` check** or ante-handler guard so that `MsgTurnBridge` from an unauthorized sender is rejected before it reaches the keeper.

---

### Proof of Concept

```
# Step 1: Any unprivileged address calls TurnBridge — succeeds with code 0
cronosd tx cronos turn-bridge false --from=<any_unprivileged_key> -y
# → tx code: 0  (no permission check, no-op)

# Step 2: Admin also calls TurnBridge to "disable" the bridge
cronosd tx cronos turn-bridge false --from=<admin_key> -y
# → tx code: 0  (no-op, nothing stored)

# Step 3: Any EVM contract emits __CronosSendToIbc — bridge hook fires normally
# SendToIbcHandler.handle() has no bridge-enabled gate; IBC transfer proceeds
# → Bridge is NOT disabled; funds continue to flow
```

The integration test `test_gravity_turn_bridge` asserts that after `turn_bridge("false")` the bridge is disabled: [7](#0-6) 

This test would fail against the current production keeper because `TurnBridge` is a no-op and no bridge-enabled state is ever written or read.

func (h SendToIbcHandler) handle(
	ctx sdk.Context,
	contract common.Address,
	senderAddress common.Address,
	recipient string,
	amountInt *big.Int,
	id *big.Int,
) error {
	denom, found := h.cronosKeeper.GetDenomByContract(ctx, contract)
	if !found {
		return fmt.Errorf("contract %s is not connected to native token", contract)
	}

	if !types.IsValidIBCDenom(denom) && !types.IsValidCronosDenom(denom) {
		return fmt.Errorf("the native token associated with the contract %s is neither an ibc voucher or a cronos token", contract)
	}

	contractAddr := sdk.AccAddress(contract.Bytes())
	sender := sdk.AccAddress(senderAddress.Bytes())
	amount := sdkmath.NewIntFromBigInt(amountInt)
	coins := sdk.NewCoins(sdk.NewCoin(denom, amount))

	var err error
	if types.IsSourceCoin(denom) {
		// it is a source token, we need to mint coins
		if err = h.bankKeeper.MintCoins(ctx, types.ModuleName, coins); err != nil {
			return err
		}
		// send the coin to the user
		if err = h.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, sender, coins); err != nil {
			return err
		}
	} else {
		// First, transfer IBC coin to user so that he will be the refunded address if transfer fails
		if err = h.bankKeeper.SendCoins(ctx, contractAddr, sender, coins); err != nil {
			return err
		}
	}

	channelId := ""
	if id != nil {
		channelId = "channel-" + id.String()
	}
	// Initiate IBC transfer from sender account
	if err = h.cronosKeeper.IbcTransferCoins(ctx, sender.String(), recipient, coins, channelId); err != nil {
		return err
	}
	return nil
}

func (h SendCroToIbcHandler) Handle(
	ctx sdk.Context,
	contract common.Address,
	topics []common.Hash,
	data []byte,
	_ func(contractAddress common.Address, logSig common.Hash, logData []byte),
) error {
	authorizedBridges := h.cronosKeeper.GetParams(ctx).CroBridgeContractAddresses
	if !slices.ContainsFunc(authorizedBridges, func(addr string) bool {
		return common.HexToAddress(addr) == contract
	}) {
		return fmt.Errorf("contract %s is not authorized to use SendCroToIbc hook", contract)
	}

	unpacked, err := SendCroToIbcEvent.Inputs.Unpack(data)
	if err != nil {
		// log and ignore
		h.cronosKeeper.Logger(ctx).Error("log signature matches but failed to decode", "error", err)
		return nil
	}

	contractAddr := sdk.AccAddress(contract.Bytes())
	sender := sdk.AccAddress(unpacked[0].(common.Address).Bytes())
	recipient := unpacked[1].(string)
	amount := sdkmath.NewIntFromBigInt(unpacked[2].(*big.Int))
	evmDenom := h.cronosKeeper.GetEvmParams(ctx).EvmDenom
	coins := sdk.NewCoins(sdk.NewCoin(evmDenom, amount))
	// First, transfer IBC coin to user so that he will be the refunded address if transfer fails
	if err = h.bankKeeper.SendCoins(ctx, contractAddr, sender, coins); err != nil {
		return err
	}
	// Initiate IBC transfer from sender account
	if err = h.cronosKeeper.IbcTransferCoins(ctx, sender.String(), recipient, coins, ""); err != nil {
		return err
	}
	return nil
}
