Audit Report

## Title
SNS Swap Canister Strands SNS Token Allocation on Aborted Swap With No Recovery Path — (File: rs/sns/swap/src/swap.rs)

## Summary
When an SNS decentralization swap is aborted due to insufficient participation, `finalize_inner` executes `sweep_icp`, `settle_neurons_fund_participation`, and `restore_dapp_controllers_for_finalize`, then returns early at L1583 without performing any corresponding SNS token sweep back to the SNS governance treasury. The full `swap_distribution_sns_e8s` allocation remains locked in the swap canister's ledger account with no callable recovery path. The integration test suite explicitly confirms and asserts this behavior.

## Finding Description
In `rs/sns/swap/src/swap.rs`, `finalize_inner` (L1544–1624) branches on `should_restore_dapp_control()` at L1572. For the `Aborted` lifecycle this condition is true, so execution enters the block at L1572–1584, calls `restore_dapp_controllers_for_finalize`, and unconditionally returns at L1583. The `sweep_sns` call at L1593–1598 is never reached.

`sweep_sns` itself (L2165–2178) independently enforces `Lifecycle::Committed` as a precondition and returns `SweepResult::new_with_global_failures(1)` for any other lifecycle, so even a direct call on the aborted canister would fail. No alternative function exists to transfer the SNS token balance back to the SNS governance treasury account.

The integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` L1336–1342 explicitly asserts `swap_canister_balance_sns_e8s == swap_distribution_sns_e8s` after an aborted swap, confirming the full allocation remains in the swap canister. The lifecycle diagram in `rs/sns/swap/proto/ic_sns_swap/pb/v1/swap.proto` L56–66 shows `ABORTED → <DELETED>` as the terminal path, meaning once the canister is deleted the tokens become permanently inaccessible on the SNS ledger.

## Impact Explanation
The entire SNS token swap allocation — which can represent millions of governance tokens with substantial market value — is permanently stranded in the swap canister's ledger account. The swap canister is the sole authorized signer for that account; SNS root and governance canisters hold no authority over it. Upon canister deletion (the documented terminal state), the tokens are irrecoverable. This constitutes permanent loss of in-scope SNS ledger assets, matching the High impact class: "Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds" and "Significant SNS security impact with concrete user or protocol harm." The scale of loss (potentially millions of tokens) could reach Critical thresholds depending on the specific SNS allocation.

## Likelihood Explanation
No attacker action is required. Any SNS swap that fails to reach `min_participants` or `min_icp_e8s` before the deadline automatically transitions to `Aborted` via `try_abort`. This is a documented, explicitly supported lifecycle path. New or low-visibility SNS projects are particularly susceptible. The condition is triggered by ordinary under-participation, making it a realistic and repeatable scenario.

## Recommendation
In `finalize_inner`, within the `should_restore_dapp_control()` branch (the aborted path), add a step before returning to transfer the remaining SNS token balance from the swap canister's account back to the SNS governance treasury account. This should mirror the existing `sweep_icp` pattern and be guarded by an idempotency flag so retried `finalize` calls do not double-transfer. `sweep_sns` should also be updated to permit `Lifecycle::Aborted` for this treasury-return operation, or a separate `sweep_sns_to_treasury` function should be introduced.

## Proof of Concept
1. Deploy an SNS with a swap allocation of `N` SNS tokens loaded into the swap canister.
2. Open the swap; ensure participation remains below `min_participants` until the deadline.
3. Call `try_abort`; lifecycle transitions to `Aborted`.
4. Call `finalize`. Observe `finalize_inner` executes `sweep_icp` → `settle_neurons_fund_participation` → `restore_dapp_controllers_for_finalize` → returns at L1583.
5. Query `icrc1_balance_of(swap_canister_account)` on the SNS ledger; result is `N` (unchanged).
6. The existing integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` L1336–1342 already encodes this assertion and passes, confirming the behavior without any additional test authorship.
7. Delete the swap canister (per the `ABORTED → <DELETED>` lifecycle); `N` SNS tokens are permanently inaccessible.
