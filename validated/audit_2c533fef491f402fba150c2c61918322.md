### Title
Unprivileged Caller Can Permanently Suppress Per-Block Consensus Rewards via Caller-Controlled `disable_rewards` Parameter - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is callable by any non-zero address and accepts a caller-controlled `disable_rewards: bool` parameter. When set to `true`, the function updates `last_reward_block` to the current block but skips all reward distribution. Because the contract enforces a strict one-call-per-block invariant, any attacker who calls `update_rewards(..., disable_rewards: true)` first on a given block permanently forfeits that block's consensus rewards for the targeted staker. Repeated across every block, this constitutes a complete, sustained freeze of unclaimed yield with no recovery path short of a contract upgrade.

### Finding Description
`update_rewards` is part of the `IStakingRewardsManager` public interface. Its only access guard is `general_prerequisites()`, which checks that the contract is not paused and that the caller is not the zero address — no role check, no whitelist. [1](#0-0) 

The function immediately writes the current block number into `last_reward_block`: [2](#0-1) 

Then it branches on the caller-supplied flag:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [3](#0-2) 

Because `last_reward_block` is already committed, the one-call-per-block guard will reject any subsequent call in the same block: [4](#0-3) 

The rewards that would have been distributed via `_update_rewards` → `reward_supplier_dispatcher.claim_rewards` are never claimed and never credited to the staker or their pool. There is no catch-up mechanism; the per-block reward is simply gone.

The `staker_address` argument must resolve to a currently active staker, but all active stakers are enumerated on-chain in the public `stakers` vector, so this is not a barrier. [5](#0-4) 

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

Each block for which an attacker fires `update_rewards(any_active_staker, disable_rewards: true)` first results in that block's STRK (and BTC) consensus rewards being permanently unclaimable. The `unclaimed_rewards` counter in `RewardSupplier` is never incremented for those blocks, so neither the staker nor their delegators can ever recover those tokens. Sustained execution across every block eliminates all consensus-phase yield for the targeted staker(s). [6](#0-5) 

### Likelihood Explanation
**Medium.** Starknet uses a sequencer, so classical mempool frontrunning does not apply. However, the attacker does not need to frontrun — they simply need to submit `update_rewards(..., true)` at the start of each block before the legitimate consensus caller does. The call is cheap (no token transfer, no approval), the target staker address is public, and the attacker gains nothing financially, making this a pure griefing vector. Sustained execution requires automation but no privileged access whatsoever.

### Recommendation
1. **Remove the `disable_rewards` parameter from the public interface.** If the consensus layer needs to signal "no reward this block," that signal should come from a permissioned caller (e.g., the attestation contract or a dedicated consensus role), not from an arbitrary external address.
2. Alternatively, gate `disable_rewards: true` behind a role check (e.g., `only_app_governor` or a new `CONSENSUS_ROLE`) so that only the trusted consensus component can suppress rewards.
3. Consider separating the "mark block as processed" write (`last_reward_block`) from the "distribute rewards" logic so that a suppressed block does not permanently consume the one-call-per-block slot.

### Proof of Concept

```
// Attacker script — runs once per block, no special privileges needed

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
