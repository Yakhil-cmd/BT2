### Title
`RelayerContract.RequiredGas` Incorrectly Subtracts Transaction-Level Intrinsic Gas for Contract-to-Precompile Calls, Enabling IBC Operation Undercharging - (File: x/cronos/keeper/precompiles/relayer.go)

### Summary

`RelayerContract.RequiredGas` subtracts the full transaction-level intrinsic gas (including the 21,000 base `TxGas`) from the precompile's required gas. This subtraction is only valid when an EOA calls the precompile directly, because only then has the intrinsic gas already been charged at the transaction level. When a contract calls the relayer precompile, the inner call's calldata is never charged intrinsic gas, yet `RequiredGas` still subtracts it — causing severe undercharging or a return of 0 gas for multiple IBC operations.

### Finding Description

`RequiredGas` is defined as:

```
max(0, len(input) * DefaultTxSizeCostPerByte + gasRequiredByMethod - intrinsicGas)
```

where `intrinsicGas` is computed via `core.IntrinsicGas(input, nil, nil, false, ...)`. [1](#0-0) 

`core.IntrinsicGas` always includes the 21,000 base `TxGas` plus per-byte calldata costs. This base is only charged for top-level transactions, not for inner EVM calls (CALL opcodes from a contract). [2](#0-1) 

The `getRequiredGas` helper returns 0 whenever `gasRequiredByMethod + baseCost < intrinsicGas`. [2](#0-1) 

For a contract-to-precompile call with small input, `intrinsicGas ≥ 21,000` (the base), while `baseCost ≈ 0`. Any method whose table entry is below 21,000 will always return 0 gas:

| Method | Table Gas | RequiredGas from contract |
|---|---|---|
| `ConnectionOpenConfirm` | 12,865 | **0** (always) |
| `ConnectionOpenInit` | 19,755 | **0** (always) |
| `ChannelOpenConfirm` | 21,190 | ~190 (severely undercharged) |
| `ChannelOpenAck` | 22,127 | ~1,127 (severely undercharged) | [3](#0-2) 

For larger inputs (non-zero bytes), the per-byte intrinsic gas (16 gas/byte post-EIP-2028) exceeds `DefaultTxSizeCostPerByte` (10 gas/byte). For `RecvPacket` (144,025 gas) or `UpdateClient` (111,894 gas), inputs exceeding ~24 KB and ~18.6 KB of non-zero bytes respectively cause `RequiredGas` to return 0. [4](#0-3) 

The `TestRelayer` integration contract confirms that contract-to-precompile calls are a supported and expected pattern — the `batchCall` function calls the relayer precompile in a loop from a contract. [5](#0-4) 

There is no EOA-only restriction in `Run`; any contract can call the relayer precompile. [6](#0-5) 

### Impact Explanation

An unprivileged attacker deploys a wrapper contract and calls IBC operations (e.g., `ConnectionOpenConfirm`, `ConnectionOpenInit`) through it. These operations cost 0 EVM gas from the precompile. The EVM block gas limit is therefore not properly enforced for these operations. An attacker with valid IBC state can pack far more IBC relay operations into a single block than the gas limit should permit, causing validators to spend disproportionate CPU/IO time processing the block. This maps to:

**High: Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows, precompile calls, IBC transfers, or block proposals under normal network assumptions.**

### Likelihood Explanation

The attack requires the attacker to have valid IBC messages (e.g., be a relayer or have access to valid IBC state). This is a realistic assumption — relayers are unprivileged actors. The `TestRelayer` contract already demonstrates the contract-to-precompile call pattern is in active use. The undercharging is deterministic and requires no special conditions beyond a valid IBC message.

### Recommendation

Remove the `intrinsicGas` subtraction from `RequiredGas`. The intrinsic gas deduction was designed to avoid double-counting for direct EOA calls, but the EVM does not charge intrinsic gas for inner CALL operations. The precompile should charge the full `gasRequiredByMethod + baseCost` regardless of whether the caller is an EOA or a contract:

```go
func getRequiredGas(gasRequiredByMethod, baseCost uint64) uint64 {
    return gasRequiredByMethod + baseCost
}
```

If the intent is to avoid double-charging EOA callers, the correct approach is to detect whether the call originates from a top-level transaction (e.g., by checking `evm.Depth() == 1` in `Run`) and apply the discount only in that case — but this is complex and error-prone. The simplest and safest fix is to remove the subtraction entirely.

### Proof of Concept

1. Deploy the following contract on Cronos:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IRelayer {
    function connectionOpenConfirm(bytes calldata data) external payable returns (bytes calldata);
}

contract RelayerAbuser {
    IRelayer constant relayer = IRelayer(0x0000000000000000000000000000000000000065);

    // Call connectionOpenConfirm in a loop — each call costs 0 precompile gas
    function abuse(bytes calldata data, uint256 times) external {
        for (uint256 i = 0; i < times; i++) {
            relayer.connectionOpenConfirm(data);
        }
    }
}
```

2. Call `abuse(validConnectionOpenConfirmMsg, N)` with a valid `MsgConnectionOpenConfirm` protobuf payload and a large `N`.

3. Observe that `RequiredGas` returns 0 for each inner call (12,865 < 21,000), so the EVM gas meter charges 0 gas per IBC operation. The block gas limit is consumed only by the loop overhead (~21 gas per iteration), allowing thousands of IBC operations per block instead of the ~1,650 that the 30M gas limit should permit at 18,000 gas each. // RequiredGas calculates the contract gas use
// `max(0, len(input) * DefaultTxSizeCostPerByte + requiredGasTable[methodPrefix] - intrinsicGas)`
func (bc *RelayerContract) RequiredGas(input []byte) (gas uint64) {
	intrinsicGas, _ := core.IntrinsicGas(input, nil, nil, false, bc.isHomestead, bc.isIstanbul, bc.isShanghai)
	// base cost to prevent large input size
	inputLen := len(input)
	baseCost := uint64(inputLen) * authtypes.DefaultTxSizeCostPerByte
	var methodID [4]byte
	if inputLen < 4 {
		bc.logger.Error("invalid input length", "input", input)
		return getRequiredGas(DefaultGasRequired, baseCost, intrinsicGas)
	}
	defer func() {
		methodName := relayerMethodNamedByMethod[methodID]
		bc.logger.Debug("required", "gas", gas, "method", methodName, "len", inputLen, "intrinsic", intrinsicGas)
	}()
	copy(methodID[:], input[:4])
	gasRequiredByMethod, ok := relayerGasRequiredByMethod[methodID]
	if !ok {
		bc.logger.Error("unknown method", "method", methodID)
		return getRequiredGas(DefaultGasRequired, baseCost, intrinsicGas)
	}

	method, err := irelayerABI.MethodById(methodID[:])
	if err != nil {
		bc.logger.Error("failed to get method by id", "error", err)
		return getRequiredGas(gasRequiredByMethod, baseCost, intrinsicGas)
	}
	if method.Name == RecvPacket {
		args, err := method.Inputs.Unpack(input[4:])
		if err != nil {
			bc.logger.Error("failed to unpack input arguments", "error", err)
			return getRequiredGas(gasRequiredByMethod, baseCost, intrinsicGas)
		}
		i := args[0].([]byte)
		var msg channeltypes.MsgRecvPacket
		if err = bc.cdc.Unmarshal(i, &msg); err != nil {
			bc.logger.Error("failed to unmarshal MsgRecvPacket", "error", err)
			return getRequiredGas(gasRequiredByMethod, baseCost, intrinsicGas)
		}
		var data ibctransfertypes.FungibleTokenPacketData
		if err = ibctransfertypes.ModuleCdc.UnmarshalJSON(msg.Packet.GetData(), &data); err != nil {
			bc.logger.Error("failed to unmarshal FungibleTokenPacketData", "error", err)
			return getRequiredGas(gasRequiredByMethod, baseCost, intrinsicGas)
		}
		denom := ibctransfertypes.ExtractDenomFromPath(data.Denom)
		if denom.HasPrefix(msg.Packet.GetSourcePort(), msg.Packet.GetSourceChannel()) {
			gasRequiredByMethod = GasWhenReceiverChainIsSource
		}
	}

	return getRequiredGas(gasRequiredByMethod, baseCost, intrinsicGas)
}

func getRequiredGas(gasRequiredByMethod, baseCost, intrinsicGas uint64) uint64 {
	total := gasRequiredByMethod + baseCost
	if total < intrinsicGas {
		return 0
	}
	return total - intrinsicGas
}
