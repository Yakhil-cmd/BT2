### Title
Floor Rounding in `validate_tx_l2_gas_price_within_threshold` Admits Transactions Below the Intended Minimum Gas Price Threshold - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price by multiplying `previous_block_l2_gas_price` by a percentage ratio and calling `.to_integer()`, which performs **floor (truncating) division**. This makes the enforced threshold strictly lower than the intended fractional value, so transactions whose gas price falls in the gap `[floor(threshold), true_threshold)` are admitted to the mempool when they should be rejected.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, the gateway stateful validator enforces a minimum L2 gas price for incoming `AllResources` transactions:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← floor division
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
```

`Ratio::to_integer()` from `num_rational` returns the **floor** of the rational number. When `min_gas_price_percentage × previous_block_l2_gas_price` is not evenly divisible by 100, the computed `threshold` is 1 FRI lower than the true fractional threshold.

**Concrete example:**
- `previous_block_l2_gas_price = 101` FRI, `min_gas_price_percentage = 80`
- True threshold = `80 × 101 / 100 = 80.8` FRI
- Computed threshold = `floor(80.8) = 80` FRI
- A transaction with `tx_l2_gas_price = 80` passes the check (`80 >= 80`) but should fail (`80 < 80.8`)

The existing unit tests in `stateful_transaction_validator_test.rs` only exercise cases where `min_gas_price_percentage × previous_block_l2_gas_price` is exactly divisible by 100 (e.g., `100 × 100`, `100 × 50`), so the rounding error is never triggered by the test suite.

This is the direct analog of the external report's rounding error: in the DeFi contract, `lockedEarnUSDCAmount` was rounded **down**, making the withdrawable amount higher than entitled. Here, `threshold` is rounded **down**, making the admission check more permissive than intended.

### Impact Explanation

The gateway's stateful admission path (`extract_state_nonce_and_run_validations` → `run_pre_validation_checks` → `validate_state_preconditions` → `validate_resource_bounds` → `validate_tx_l2_gas_price_within_threshold`) is the production mempool gate for all `AllResources` (V3) transactions. Transactions with an L2 gas price in the range `[floor(P × pct / 100), P × pct / 100)` — i.e., exactly 1 FRI below the true threshold — are admitted when the operator's policy requires rejection. This breaks the invariant that the sequencer only sequences transactions meeting the configured minimum gas price floor, and falls under **High: Mempool/gateway admission accepts invalid transactions before sequencing**.

### Likelihood Explanation

Any user can craft a V3 transaction with `l2_gas.max_price_per_unit` set to exactly `floor(min_gas_price_percentage × previous_block_l2_gas_price / 100)`. This requires no privilege, no front-running, and no special state. The condition is triggered whenever `min_gas_price_percentage × previous_block_l2_gas_price` is not divisible by 100, which is the common case for any non-round gas price (e.g., EIP-1559 adjusted prices).

### Recommendation

Replace `.to_integer()` (floor) with `.ceil().to_integer()` (ceiling) so the threshold is rounded **up**, ensuring the check is at least as strict as the operator's configured percentage:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This matches the rounding convention already used elsewhere in the codebase for conservative gas conversions (e.g., `convert_l1_to_l2_gas_price_round_up`, `sierra_gas_to_l1_gas_amount_round_up`, `l1_gas_to_sierra_gas_amount_round_up` in `crates/blockifier/src/blockifier_versioned_constants.rs`).

### Proof of Concept

```
previous_block_l2_gas_price = NonzeroGasPrice(101)
min_gas_price_percentage     = 80

// Current (buggy) behavior:
threshold = floor(Ratio(80, 100) * 101) = floor(Ratio(8080, 100)) = floor(80.8) = 80
tx with tx_l2_gas_price = 80 → 80 >= 80 → ADMITTED  ← should be rejected

// Correct behavior with ceil:
threshold = ceil(80.8) = 81
tx with tx_l2_gas_price = 80 → 80 < 81 → REJECTED   ← correct
```

The test case `tx_gas_price_meets_threshold_with_factor_pass` in `crates/apollo_gateway/src/stateful_transaction_validator_test.rs` uses `previous_block_l2_gas_price = 100, min_gas_price_percentage = 50, tx_gas_price = 50`, which is exactly divisible and does not expose the rounding error. Adding a case with `previous_block_l2_gas_price = 101, min_gas_price_percentage = 80, tx_gas_price = 80` would demonstrate the bug: the current code returns `Ok(())` but the correct behavior is to return an error. ```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
