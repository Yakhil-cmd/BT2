### Title
Incorrect Multi-NFT Royalty Total Displayed in WalletConnect Offer Confirmation Dialog - (`packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary

The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts` incorrectly calculates the displayed total-with-royalties for WalletConnect `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` confirmation dialogs when multiple NFTs with royalties are involved. The preview divides the fungible amount equally among NFTs before applying each royalty rate, producing a systematically understated total. The actual blockchain execution applies each royalty to the full amount, so the user approves a transaction showing a lower cost than what is actually deducted.

### Finding Description

`formatAmountWithRoyalties` is called from `withRoyaltyTotals` to populate the `amountWithRoyalties` field shown in the WalletConnect approval dialog. When the receiving side of an offer contains N NFTs each with a royalty percentage, the function does:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // line 363
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [1](#0-0) 

It divides `amount` by the number of NFTs, then applies each NFT's royalty to that fraction. The correct behavior is to apply each NFT's royalty to the **full** amount, because each royalty is independently owed on the full sale price.

**Concrete example** (matching the existing test at line 251):
- Spending: 100,000,000 mojos (0.0001 XCH)
- NFT1 royalty: 500 (5%), NFT2 royalty: 10 (0.1%)

| | Displayed (buggy) | Actual on-chain |
|---|---|---|
| splitAmount | 50,000,000 | N/A |
| royaltyAmount | 2,500,000 + 50,000 = 2,550,000 | 5,000,000 + 100,000 = 5,100,000 |
| Total shown | 0.00010255 XCH | 0.0001051 XCH |

The test at line 314-320 asserts `amountWithRoyalties: '0.00010255'`, confirming the buggy value is the one shown to the user. [2](#0-1) 

This `amountWithRoyalties` value is the primary figure shown in the WalletConnect confirmation dialog for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids`. [3](#0-2) 

### Impact Explanation

A user approving a WalletConnect request to take or create an offer involving multiple royalty-bearing NFTs is shown a total spending amount that is lower than what the Chia wallet backend will actually deduct. The discrepancy grows with the number of NFTs and the magnitude of royalty percentages. The user gives informed consent to a lower amount but the actual spend is higher — a direct accounting mismatch in the approval flow.

### Likelihood Explanation

Any paired WalletConnect dApp can send a `chia_wallet.take_offer` request with an offer string containing multiple NFTs. The user has already approved the WalletConnect session (standard pairing), so no further privilege is required. The bug is triggered automatically whenever two or more NFTs with non-zero royalties appear on the receiving side of the offer. The dApp does not need to be malicious — the bug fires for any legitimate multi-NFT offer.

### Recommendation

Remove the `splitAmount` division. Each royalty percentage should be applied to the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [4](#0-3) 

The corresponding test expectation at line 320 (`amountWithRoyalties: '0.00010255'`) must also be updated to the correct value (`'0.0001051'`). [5](#0-4) 

### Proof of Concept

1. A WalletConnect dApp (already paired) sends `chia_wallet.take_offer` with an offer string that, when decoded by `getOfferSummary`, shows:
   - Spending: 100,000,000 mojos XCH
   - Receiving: NFT-A (royalty 5%) + NFT-B (royalty 1%)
2. `parseCommandDisplay` calls `formatAmountWithRoyalties` with `amount = 100000000n`, `royaltyPercentages = [500, 10]`.
3. `splitAmount = 50000000n`; displayed total = `102,550,000` mojos = **0.00010255 XCH**.
4. User sees "Total Amount with Royalties: 0.00010255 XCH" and approves.
5. The Chia wallet backend applies both royalties to the full 100,000,000 mojos: actual deduction = **105,100,000** mojos = **0.0001051 XCH**.
6. User's wallet is debited ~2.4% more than the approved display amount.

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
