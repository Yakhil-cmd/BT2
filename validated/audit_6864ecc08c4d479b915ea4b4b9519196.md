### Title
Missing Validation Allows New Reshare Participant to Supply Superfluous Key Share, Causing Protocol-Wide Abort - (File: src/dkg.rs)

---

### Summary

`assert_reshare_keys_invariants` in `src/dkg.rs` contains a comment stating it will reject a new participant (not in the old set) who supplies an `old_signing_key`, but the actual guard is inverted and only checks the opposite direction. A malicious new participant can therefore pass a non-`None` `old_signing_key` through validation unchallenged, causing `do_reshare` to compute a corrupted secret contribution and abort the reshare for every honest party.

---

### Finding Description

The guard in `assert_reshare_keys_invariants` reads:

```rust
// if me is not in the old participant set then ensure that old_signing_key is None
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(…));
}
``` [1](#0-0) 

The comment describes the intended invariant ("new participant → key must be `None`"), but the code enforces only the symmetric case ("old participant → key must be `Some`"). The complementary guard:

```rust
if !old_participants.contains(me) && old_signing_key.is_some() { … }
```

is entirely absent.

When a new participant `D` (absent from `old_participants`) calls `reshare` with a non-`None` `old_signing_key`, validation passes. Inside `do_reshare`, the secret is computed as:

```rust
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)          // me ∉ intersection
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(…::zero);
``` [2](#0-1) 

Because `D` is not in `old_participants`, `D` is not in `intersection`. `intersection.lagrange::<C>(D)` either:

- **Returns an error** (if `compute_lagrange_coefficient` enforces membership), causing `transpose()?` to propagate a `ProtocolError` before `D` sends any messages — leaving every other participant blocked indefinitely in `recv_from_others`.
- **Succeeds with a non-zero scalar** (if the coefficient is computed for a point outside the set), giving `D` a non-zero secret instead of zero. `D` then broadcasts a commitment derived from this corrupted secret. The final public-key consistency check:

```rust
if old_vk != verifying_key {
    return Err(ProtocolError::AssertionFailed(
        "new public key does not match old public key".to_string(),
    ));
}
```
