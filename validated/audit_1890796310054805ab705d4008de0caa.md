### Title
Permissionless `update_rewards` with Attacker-Controlled `disable_rewards` Allows Permanent Denial of Block Rewards - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` has no caller validation despite the specification requiring "Only starkware sequencer" access. Because the function accepts an attacker-controlled `disable_rewards: bool` parameter and writes `last_reward_block` before checking that flag, any unprivileged caller can front-run the sequencer, mark the current block as "already processed," and permanently prevent all stakers from receiving consensus block rewards for that block.

### Finding Description

The spec at `docs/spec.md:1645` states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo:1449-1488` contains no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // ← last_reward_block is committed HERE, before disable_rewards is checked
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits with no rewards distributed
    }
    ...
``` [1](#0-0) 

`last_reward_block` is a single global storage slot (not per-staker):

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The spec's intended access control is documented but never enforced in code: [3](#0-2) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` once per block before the sequencer's legitimate call. The sequence is:

1. Attacker's call passes the `current_block_number > last_reward_block` guard (first call for this block).
2. `last_reward_block` is written to `current_block_number`.
3. The function returns early — zero rewards are distributed to any staker or pool.
4. The sequencer's subsequent call for the same block reverts with `REWARDS_ALREADY_UPDATED`.

Because `last_reward_block` is global, a single attacker transaction per block permanently denies consensus block rewards to **all** stakers and their delegation pools for that block. Repeated across every block, this constitutes a sustained, permanent freezing of all unclaimed consensus yield.

This maps directly to the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The function is publicly callable with no role or address restriction.
- The attacker only needs to submit one transaction per block with `disable_rewards: true` and any valid `staker_address`.
- Gas cost on Starknet is low, making sustained griefing economically viable.
- No special knowledge, leaked keys, or privileged access is required.

### Recommendation

Enforce the access control that the specification already mandates. Add a sequencer-address check at the top of `update_rewards`, analogous to the pattern used in `update_rewards_from_attestation_contract` and `update_current_epoch_block_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
```

Alternatively, remove the `disable_rewards` parameter from the public interface and have the sequencer signal intent through a separate privileged call, so that the reward-marking step and the reward-distribution step cannot be decoupled by an unprivileged caller.

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. A valid staker `S` exists with non-zero balance.
3. At block `N`, before the sequencer's transaction, attacker `A` submits:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. The call succeeds: `last_reward_block` is set to `N`, no rewards are minted.
5. The sequencer's `update_rewards(S, disable_rewards: false)` for block `N` reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker `S` (and all pool members) receive zero rewards for block `N`.
7. Repeating step 3 every block permanently freezes all consensus block rewards.

The `REWARDS_ALREADY_UPDATED` revert path is confirmed by existing tests: [4](#0-3)

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);
