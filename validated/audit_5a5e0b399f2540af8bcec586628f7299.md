### Title
No Slippage Protection on Commission Rate in `switch_delegation_pool` — (File: src/pool/pool.cairo)

### Summary
When a delegator calls `switch_delegation_pool`, they specify a destination staker and amount but have no way to cap the maximum acceptable commission rate of the destination pool. A staker holding an active `commission_commitment` can front-run the delegator's transaction by raising their commission up to `max_commission`, causing the delegator to earn far less yield than expected with no recourse.

### Finding Description

`switch_delegation_pool` in `src/pool/pool.cairo` accepts only `to_staker`, `to_pool`, and `amount`:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
) -> Amount {
```

There is no `max_commission` guard. The function immediately deducts `amount` from the caller's `unpool_amount` and forwards it to the destination pool via `switch_staking_delegation_pool`, without ever reading or validating the destination staker's current commission.

The commission of any staker can be changed at any time via `set_commission`. Normally commission can only decrease, but when a staker has an **active** `commission_commitment`, `update_commission` permits raising commission to any value ≤ `max_commission`:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
}
```

A malicious staker can therefore:
1. Call `set_commission_commitment(max_commission: 10000, expiration_epoch: ...)` to establish a commitment allowing up to 100% commission.
2. Set current commission to a low value (e.g. 1%) to attract delegators.
3. Observe a pending `switch_delegation_pool` transaction in the mempool.
4. Front-run it with `set_commission(10000)`.
5. The delegator's transaction lands and they are now a member of a 100%-commission pool, earning zero yield.

The same gap exists in `enter_delegation_pool` and `add_to_delegation_pool`, but `switch_delegation_pool` is the most severe case: the delegator has already called `exit_delegation_pool_intent` and deducted their balance from the old pool, so they are committed to the switch.

### Impact Explanation

The delegator's entire future yield from the destination pool is redirected to the staker as commission. In the worst case (commission = 10000) the delegator earns **zero** rewards indefinitely. This constitutes theft of unclaimed yield: rewards the delegator is entitled to earn are instead captured by the staker. The delegator's principal is not at risk, but their yield stream is fully compromised until they notice and exit.

### Likelihood Explanation

The attack requires a staker to have an active `commission_commitment` with a high `max_commission`. A malicious staker can set this up in advance at negligible cost. Starknet transactions are visible before finality, making front-running feasible. The delegator has no on-chain mechanism to atomically verify and enforce a commission bound at switch time.

### Recommendation

Add a `max_commission: Commission` parameter to `switch_delegation_pool` (and, for consistency, to `enter_delegation_pool` and `add_to_delegation_pool`). Before executing the switch, read the destination staker's current commission and assert it does not exceed the caller-supplied bound:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
    max_commission: Commission,   // <-- new
) -> Amount {
    // ... existing checks ...
    let dest_commission = staking_contract.get_commission(to_staker);
    assert!(dest_commission <= max_commission, Error::COMMISSION_EXCEEDS_MAX);
    // ... rest of logic ...
}
```

The same guard should be applied in `enter_delegation_pool` and `add_to_delegation_pool`.

### Proof of Concept

**Setup:**
- Staker B calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current + 100)`.
- Staker B calls `set_commission(100)` (1%) to appear attractive.

**Attack:**
1. Delegator A calls `exit_delegation_pool_intent(amount)` on pool A.
2. Delegator A submits `switch_delegation_pool(to_staker: B, to_pool: pool_B, amount)`.
3. Staker B observes the pending transaction and calls `set_commission(10000)` with higher gas priority.
4. Staker B's `set_commission` is included first; `update_commission` succeeds because `10000 <= max_commission` and `10000 != 100`.
5. Delegator A's `switch_delegation_pool` is included next. No commission check exists, so it succeeds.
6. Delegator A is now a member of pool B with 100% commission. All rewards from pool B flow to staker B.

                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.

                .read()

                    :to_pool,

            self

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
