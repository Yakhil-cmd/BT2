### Title
Unprivileged caller can permanently suppress per-block rewards for all stakers by calling `update_rewards` with `disable_rewards=true` — (`File: src/staking/staking.cairo`)

### Summary

`update_rewards` is documented as "Only starkware sequencer" but the implementation enforces **no caller restriction**. Any address can call it. Because `last_reward_block` is written to storage **before** the `disable_rewards` early-return branch, an attacker can permanently discard an entire block's worth of rewards for every staker by front-running the sequencer with `disable_rewards=true`.

---

### Finding Description

The implementation of `update_rewards` in `StakingRewardsManagerImpl`: [1](#0-0) 

The only gate is `general_prerequisites()`, which exclusively checks the pause flag (confirmed by the pause test suite). [2](#0-1) 

There is no `get_caller_address()` check, no role assertion, and no sequencer-identity verification anywhere in the function. The spec explicitly states access should be restricted: [3](#0-2) 

The critical ordering flaw: `last_reward_block` is written at line 1485, **before** the `disable_rewards` branch at line 1487. [4](#0-3) 

`last_reward_block` is a **global** storage variable — it is not per-staker. Once it is set to the current block number, every subsequent call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`, regardless of which staker is targeted. [5](#0-4) 

---

### Impact Explanation

An attacker calls `update_rewards(any_valid_active_staker, disable_rewards=true)` once per block. This:

1. Passes all validation (staker exists, is active, has non-zero balance).
2. Writes `last_reward_block = current_block`.
3. Returns early — zero rewards distributed to any staker.
4. Blocks the legitimate sequencer call for that block with `REWARDS_ALREADY_UPDATED`.

The block's rewards are **permanently lost** — there is no recovery path. The attacker can repeat this every block, continuously zeroing out all staker and pool rewards. This matches the **High** impact: *Permanent freezing of unclaimed yield*.

---

### Likelihood Explanation

- No privilege required; any EOA can call the function.
- The attacker only needs to submit a transaction before the sequencer's `update_rewards` call in each block.
- On Starknet, transaction ordering within a block is controlled by the sequencer, but the sequencer is the *intended* caller — if the sequencer is the attacker or if the attacker can front-run (e.g., via a higher fee), the attack succeeds.
- Even a single successful call per block permanently discards that block's rewards.

---

### Recommendation

Add a caller restriction to `update_rewards` matching the spec. The simplest fix is to assert the caller holds the sequencer role (or a dedicated `REWARDS_UPDATER` role) at the top of the function, before any state is written:

```cairo
fn update_rewards(...) {
    self.roles.only_sequencer(); // enforce "Only starkware sequencer"
    self.general_prerequisites();
    ...
}
```

Alternatively, move the `last_reward_block.write(...)` to **after** the `disable_rewards` branch so that a suppressed call does not consume the block's reward slot — but the caller restriction is the correct primary fix.

---

### Proof of Concept

```
Block N:
  Attacker tx:    update_rewards(staker=Alice, disable_rewards=true)
                  → last_reward_block := N, returns (no rewards)
  Sequencer tx:   update_rewards(staker=Alice, disable_rewards=false)
                  → assert!(N > N) fails → REWARDS_ALREADY_UPDATED

Result: Alice (and every other staker) receives 0 rewards for block N.
Repeat every block → all staker yield permanently frozen.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
