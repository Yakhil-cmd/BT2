### Title
Unbounded `stakers` Vec Growth Causes Unbounded Gas Consumption in `get_stakers` — (File: `src/staking/staking.cairo`)

### Summary
The `stakers` storage `Vec` in the `Staking` contract is append-only: staker addresses are pushed on `stake()` but are **never removed** when a staker calls `unstake_action`. The `get_stakers` function iterates over the entire Vec on every call. An unprivileged actor can grow this Vec without bound by cycling through many addresses (stake → wait → unstake → repeat), eventually making `get_stakers` consume unbounded gas.

### Finding Description

The storage layout explicitly documents the invariant violation:

```
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [1](#0-0) 

On every `stake()` call, the caller's address is appended unconditionally:

```cairo
// Add staker address to the stakers vector.
self.stakers.push(staker_address);
``` [2](#0-1) 

`remove_staker` (called from `unstake_action`) zeroes out `staker_info` and the operational-address mapping, but never touches `stakers`:

```cairo
fn remove_staker(...) {
    self.insert_staker_own_balance(:staker_address, own_balance: Zero::zero());
    self.staker_info.write(staker_address, VInternalStakerInfo::None);
    self.operational_address_to_staker_address.write(operational_address, Zero::zero());
    ...
}
``` [3](#0-2) 

`get_stakers` then iterates over **every address ever pushed**, including all unstaked ones:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    ...
}
``` [4](#0-3) 

Each inactive entry still costs a storage read and an `is_staker_active` check. As the Vec grows, the per-call gas cost grows linearly with no upper bound.

### Impact Explanation

`get_stakers` is a public entrypoint on `IStakingConsensus` (access: "Any address can execute") used by the consensus layer to enumerate active validators and their staking power for a given epoch. If the Vec is large enough, the function will exceed the Starknet per-transaction gas limit, making it permanently uncallable. This breaks the ability of any on-chain or off-chain consumer (including the sequencer's consensus logic) to enumerate the active validator set, constituting **unbounded gas consumption / griefing with damage to the protocol** — a Medium impact under the allowed scope. [5](#0-4) 

### Likelihood Explanation

The attack requires only the minimum stake amount per address and gas fees. After the `exit_wait_window` elapses, the attacker recovers the staked principal. The net cost per Vec entry is only gas. Because `assert_staker_address_not_reused` prevents address reuse, the attacker needs distinct addresses, but these are trivially generated. The attack is therefore economically feasible for a motivated griever. [6](#0-5) 

### Recommendation

Maintain a parallel `Map<ContractAddress, bool>` (or reuse `staker_info`) as the authoritative membership set, and replace the `Vec`-based iteration in `get_stakers` with an off-chain-indexed or paginated approach. Alternatively, record a "tombstone" count and skip tombstoned entries without iterating them, or remove the address from the Vec at `unstake_action` time (swap-and-pop pattern). The interface comment already warns that the token set must remain small for the same reason:

> "Adding too many tokens can lead to unbounded complexity and potential performance issues." [7](#0-6) 

The same discipline must be applied to `stakers`.

### Proof of Concept

1. Attacker controls addresses `A1 … AN` (N large).
2. For each `Ai`: call `stake(reward_addr, operational_addr_i, min_stake)` → `stakers` Vec grows by 1.
3. After `exit_wait_window`: call `unstake_intent(Ai)` then `unstake_action(Ai)` → principal returned, but `Ai` remains in `stakers`.
4. Repeat until `stakers.len()` is large enough that `get_stakers(epoch_id)` exhausts the block gas limit.
5. `get_stakers` is now permanently DoS'd; the consensus layer cannot enumerate the active validator set.

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
