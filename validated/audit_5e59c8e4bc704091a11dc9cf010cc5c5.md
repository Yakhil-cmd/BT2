### Title
Unbounded loop over all registered stakers in `get_stakers()` causes resource exhaustion - (File: src/staking/staking.cairo)

### Summary
`IStakingConsensus::get_stakers()` iterates over every staker address ever registered in the protocol — including those who have already unstaked — with no pagination or bound. As the staker registry grows monotonically, this loop's resource cost grows without limit, eventually making the function uncallable.

### Finding Description
`StakingConsensusImpl::get_stakers` iterates over the entire `self.stakers` Vec using `into_iter_full_range()`:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    // ... per-staker storage reads for staking_power, public_key, peer_id
    stakers.append((staker_address, staking_power, public_key, peer_id));
}
``` [1](#0-0) 

The `stakers` Vec is append-only: every call to `stake()` pushes a new address into it, and no path ever removes an address from it (unstaking only clears the staker's data, not the Vec entry). This means the Vec length is the **cumulative count of all stakers who have ever staked**, not just the currently active ones. The `is_staker_active` check on line 920 only skips the append, but the storage read for every address still occurs on every iteration.

Each iteration performs multiple storage reads (`is_staker_active`, `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, `get_peer_id_at_epoch`), making the per-iteration cost non-trivial.

### Impact Explanation
`get_stakers` is the function the consensus layer calls to obtain the staker set for a given epoch. It is a public view function (`self: @ContractState`) callable by any address. As the number of historical stakers grows, the execution resources (steps/gas) required to complete the loop grow proportionally. Once the cumulative staker count is large enough, the function will exceed the Starknet execution resource limit and revert, making the staker set permanently unretrievable. This disrupts the consensus layer's ability to determine staking power, which is a core protocol function.

**Impact: Medium — Unbounded gas consumption / griefing with damage to the protocol.**

### Likelihood Explanation
The `stakers` Vec grows naturally over time as participants join and leave the protocol. An adversary can accelerate this by registering many staker accounts at the minimum stake threshold, each of which permanently inflates the Vec. Because the minimum stake is a protocol parameter (not zero), this requires capital, making the likelihood **Low** for a deliberate attack but **Medium-to-High** over the long-term lifecycle of the protocol as organic churn accumulates.

### Recommendation
1. **Pagination**: Add `offset` and `limit` parameters to `get_stakers` so callers can retrieve the staker set in bounded chunks.
2. **Active-staker index**: Maintain a separate, pruned index of currently active stakers (add on `stake`, remove on `unstake_action`) so the loop only visits live entries.
3. **Tombstone removal**: When a staker calls `unstake_action`, swap-and-pop their address out of the `stakers` Vec to keep its length bounded by the number of *active* stakers.

### Proof of Concept
1. Deploy the staking contract.
2. Register `N` staker accounts (each with `min_stake`), then have each call `unstake_intent` + `unstake_action` to exit. Their addresses remain in `self.stakers`.
3. Register `N` more stakers. The Vec now has `2N` entries.
4. Call `get_stakers(epoch_id)`. The loop executes `2N` iterations, each performing multiple storage reads.
5. As `N` grows toward the Starknet step limit, the call reverts with an out-of-resources error, permanently breaking the consensus layer's view of the staker set. [2](#0-1)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
