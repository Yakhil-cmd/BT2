### Title
Unprivileged Caller Can Permanently Freeze All Consensus Block Rewards via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the Staking contract is callable by any non-zero address and accepts a `disable_rewards: bool` parameter. When called with `disable_rewards: true`, the function advances the **global** `last_reward_block` storage variable to the current block number but returns early without calling `_update_rewards`, permanently discarding that block's rewards for every staker. Because the function enforces a one-call-per-block invariant on this global variable, a single attacker call per block is sufficient to deny all consensus rewards to all stakers and delegators indefinitely.

---

### Finding Description

`update_rewards` is implemented in `StakingRewardsManagerImpl` and gated only by `general_prerequisites()`, which checks the pause flag and that the caller is non-zero — no role check exists. [1](#0-0) 

The function enforces a global, single-call-per-block invariant using `last_reward_block`: [2](#0-1) 

After validating the staker, it unconditionally writes the current block number to `last_reward_block`: [3](#0-2) 

Then, if `disable_rewards` is `true`, it returns **before** calling `_update_rewards`: [4](#0-3) 

`last_reward_block` is a single global storage slot — not per-staker: [5](#0-4) 

The consequence is:

1. The attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at the first transaction of a block.
2. `last_reward_block` is set to `current_block_number`.
3. `_update_rewards` is **never called** — no rewards are distributed to any staker or pool for that block.
4. Any subsequent legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. The block's rewards are permanently lost; they are never credited to `unclaimed_rewards` in the reward supplier.

The attacker only needs a valid staker address (publicly enumerable from the `stakers` vector) and enough gas to call once per block. [6](#0-5) 

The contrast with the pre-consensus path is instructive: `update_rewards_from_attestation_contract` is correctly gated by `assert_caller_is_attestation_contract()` before it calls `_update_rewards`. [7](#0-6) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker calling `update_rewards(valid_staker, disable_rewards: true)` once per block permanently eliminates every staker's and every delegator's consensus block rewards. The rewards are never accrued in `unclaimed_rewards` on the reward supplier, so they cannot be claimed later. Delegators' `cumulative_rewards_trace` in pool contracts is never updated, making pool rewards unclaimable as well. [8](#0-7) 

---

### Likelihood Explanation

**High.** The function is permissionless (no role check), the required input (a valid staker address) is publicly readable on-chain, and the attack costs only the gas for one transaction per block. The attacker has no profit motive but can cause complete, continuous denial of yield to all protocol participants.

---

### Recommendation

Add an access-control guard to `update_rewards` so that only the designated consensus/attestation caller (or a specific privileged role) may invoke it, mirroring the guard already present on `update_rewards_from_attestation_contract`. At minimum, the `disable_rewards: true` path must be restricted to a trusted caller.

### Proof of Concept

```
// Attacker script — repeat every block:
staking_contract.update_rewards(
    staker_address = <any valid staker from stakers[]>,
    disable_rewards = true
)
```

1. `current_block_number > last_reward_block` — passes (first call this block).
2. Staker validation passes (any active staker with non-zero balance).
3. `last_reward_block.write(current_block_number)` — global slot advanced.
4. `disable_rewards == true` → early return; `_update_rewards` is **never called**.
5. All subsequent `update_rewards` calls this block revert with `REWARDS_ALREADY_UPDATED`.
6. No staker or pool receives block rewards for this block.
7. Repeat every block → all consensus rewards permanently frozen. [9](#0-8)
