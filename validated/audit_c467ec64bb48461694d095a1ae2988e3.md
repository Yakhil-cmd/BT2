### Title
Unrestricted `update_rewards` with `disable_rewards: true` Permanently Freezes All Staker Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards` boolean. When called with `disable_rewards: true`, it advances the global `last_reward_block` sentinel without distributing any rewards. Because the sentinel is global and enforces a strict one-call-per-block invariant, any unprivileged caller can permanently consume every block's reward slot, causing all stakers to lose their consensus-phase yield indefinitely.

### Finding Description
`update_rewards` in `src/staking/staking.cairo` (lines 1449–1507) is exposed through the `IStakingRewardsManager` interface with no role check or caller restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // ← sentinel advanced unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // ← exits before distributing rewards
    }
    ...
```

`general_prerequisites` only asserts the contract is not paused and the caller is non-zero — there is no `assert_caller_is_attestation_contract()` or any equivalent guard. The `last_reward_block` write happens **before** the `disable_rewards` branch, so the slot is consumed even when no rewards are distributed.

Compare with `update_rewards_from_attestation_contract` (lines 1394–1423), which correctly gates itself:

```cairo
self.assert_caller_is_attestation_contract();
```

No such guard exists on `update_rewards`.

### Impact Explanation
`last_reward_block` is a **global, single-slot** sentinel. Once it is set to block N, no further call to `update_rewards` can succeed in block N (it reverts with `REWARDS_ALREADY_UPDATED`). The rewards that would have been distributed in block N are permanently lost — there is no catch-up mechanism. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block causes **every staker and every delegator** to receive zero consensus rewards for every block the attacker targets. Sustained over time this constitutes permanent freezing of all unclaimed yield for the entire protocol.

**Allowed impact matched:** *High — Permanent freezing of unclaimed yield.*

### Likelihood Explanation
- The function is fully public; any non-zero address can call it.
- The attacker needs no stake, no delegation, and no privileged role.
- The only cost is one Starknet transaction per block. Starknet L2 transaction fees are low enough to make this economically viable as a sustained griefing campaign.
- The attacker needs only to supply any currently-active staker address (readable from on-chain events or `get_stakers`).

### Recommendation
Add an access-control guard to `update_rewards` identical to the one used in `update_rewards_from_attestation_contract`. Either restrict it to the attestation contract, or introduce a dedicated authorized caller role:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_attestation_contract(); // or a dedicated rewards-manager role
    ...
```

Alternatively, move the `last_reward_block.write` to occur **after** the `disable_rewards` guard so that a disabled call does not consume the block slot.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker (any non-zero EOA) observes a valid active staker address `S` from on-chain events.
3. At the start of every block N, attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Inside `update_rewards`:
   - `general_prerequisites()` passes (contract not paused, caller non-zero). [1](#0-0) 
   - `current_block_number > last_reward_block` passes (new block). [2](#0-1) 
   - `last_reward_block` is written to N. [3](#0-2) 
   - `disable_rewards == true` → early return, zero rewards distributed. [4](#0-3) 
5. Any legitimate call to `update_rewards` for block N now reverts with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 
6. Block N's rewards are permanently lost. Repeating every block freezes all consensus yield indefinitely.

The missing access control is confirmed by comparing with `update_rewards_from_attestation_contract`, which correctly enforces `assert_caller_is_attestation_contract()`: [5](#0-4) 

The public interface declaration confirms no restriction is documented or enforced: [6](#0-5)
