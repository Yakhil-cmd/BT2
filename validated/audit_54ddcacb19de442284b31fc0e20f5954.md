### Title
Revoking keyIndex=0 does not revoke duplicate key indices sharing storedKeyIndex=0, allowing continued transaction signing with a revoked key — (`fvm/environment/accounts.go`, `fvm/environment/account_public_key_util.go`)

### Summary
When an account has a duplicate key (keyIndex=N, storedKeyIndex=0) and the owner revokes keyIndex=0, the revocation only updates the `apk_0` register. The weight-and-revoked metadata for keyIndex=N in the account status register is never updated. `GetRuntimeAccountPublicKey` for keyIndex=N reads `revoked=false` from the metadata section and key0's public key bytes from `apk_0` (without the revoked flag), so the transaction verifier sees a non-revoked key with key0's cryptographic material and accepts signatures made with key0's private key.

### Finding Description
The revocation path for keyIndex=0 (`revokeAccountPublicKey0`) writes `Revoked=true` exclusively into the `apk_0` register. No other register is touched. The authorization path for keyIndex=N>0 (`GetRuntimeAccountPublicKey`) reads the `revoked` field from `AccountPublicKeyMetadata(N)` — the weight-and-revoked section of the account status register — and reads the raw key bytes from `getStoredPublicKey(storedKeyIndex=0)`, which extracts only `PublicKey/SignAlgo/HashAlgo` from `apk_0`, discarding the `Revoked` field. The two revocation stores are completely disjoint: revoking keyIndex=0 never propagates to the metadata entries for any keyIndex>0 that shares storedKeyIndex=0.

Relevant code:
- `revokeAccountPublicKey0` — only modifies `apk_0` register [2](#0-1) 
- `GetRuntimeAccountPublicKey` for keyIndex>0 — reads `revoked` from metadata, not from `apk_0` [3](#0-2) 
- `getStoredPublicKey(storedKeyIndex=0)` — discards `Revoked` from `apk_0` [5](#0-4) 
- `getAccountKeysAndAggregateWeights` — checks `accountKey.Revoked` sourced from metadata [6](#0-5) 
- `AppendDuplicateKeyMetadata` — confirms duplicate keys store their own independent revoked bit in metadata [7](#0-6) 

### Impact Explanation
An attacker who controls an account can add a duplicate of key0 as keyIndex=1 (storedKeyIndex=0), then revoke keyIndex=0 (perhaps to appear compliant or after a key compromise is detected), and continue signing transactions with key0's private key via keyIndex=1. The revocation of keyIndex=0 has no effect on keyIndex=1's authorization capability. This constitutes unauthorized transaction authorization after key revocation — a direct bypass of the account key revocation security invariant.

### Likelihood Explanation
The scenario is reachable through normal Cadence transaction operations (`AuthAccount.keys.add`, `AuthAccount.keys.revoke`). No privileged access, staked node control, or external dependency is required. Any account owner can set up the precondition (duplicate key) and exploit it. The deduplication feature that enables this is active whenever a key is added that matches an existing key's digest and bytes.

### Recommendation
When revoking keyIndex=0, also iterate over all key indices in the key metadata section that have `storedKeyIndex=0` in their mapping and set their revoked bit to true. Alternatively, when `getStoredPublicKey(storedKeyIndex=0)` is called, propagate the `Revoked` field from `apk_0` into the returned `RuntimeAccountPublicKey` so that the revoked status of the underlying stored key is always authoritative. A simpler fix: in `GetRuntimeAccountPublicKey` for keyIndex>0 with storedKeyIndex=0, additionally check `apk_0.Revoked` and OR it with the metadata revoked bit.

### Proof of Concept
```
1. Create account A.
2. Add key0 (keyIndex=0, weight=1000) to A.
3. Add key1 = key0 (same public key bytes, keyIndex=1, weight=1000) to A.
   → deduplication assigns storedKeyIndex=0 to keyIndex=1; metadata revoked=false for keyIndex=1.
4. Revoke keyIndex=0: AuthAccount(A).keys.revoke(keyIndex: 0)
   → revokeAccountPublicKey0 sets apk_0.Revoked=true.
   → key metadata for keyIndex=1 is unchanged: revoked=false.
5. Submit a transaction signed with key0's private key, specifying keyIndex=1 as the signing key.
6. GetRuntimeAccountPublicKey(A, 1):
   - reads revoked=false from metadata (keyIndex=1 was never revoked)
   - reads key bytes from apk_0 (key0's public key, without revoked flag)
   - returns RuntimeAccountPublicKey{Revoked: false, PublicKey: key0.PublicKey}
7. getAccountKeysAndAggregateWeights: accountKey.Revoked == false → proceeds.
8. verifySignatures: signature made with key0's private key verifies against key0's public key → valid.
9. Transaction is accepted despite key0 being revoked.
```

func revokeAccountPublicKey0(
	a Accounts,
	address flow.Address,
) error {
	accountPublicKey0RegisterID := flow.AccountPublicKey0RegisterID(address)

	publicKey, err := a.GetValue(accountPublicKey0RegisterID)
	if err != nil {
		return err
	}

	const keyIndex = uint32(0)

	if len(publicKey) == 0 {
		return errors.NewAccountPublicKeyNotFoundError(
			address,
			keyIndex)
	}

	decodedPublicKey, err := flow.DecodeAccountPublicKey(publicKey, keyIndex)
	if err != nil {
		return fmt.Errorf(
			"failed to decode account public key 0: %w",
			err)
	}

	decodedPublicKey.Revoked = true

	encodedPublicKey, err := flow.EncodeAccountPublicKey(decodedPublicKey)
	if err != nil {
		encoded, _ := decodedPublicKey.MarshalJSON()
		return errors.NewValueErrorf(
			string(encoded),
			"failed to encode revoked account public key 0: %w",
			err)
	}

	return a.SetValue(accountPublicKey0RegisterID, encodedPublicKey)

// AppendDuplicateKeyMetadata appends duplicate key metadata.
func (m *KeyMetadataAppender) AppendDuplicateKeyMetadata(
	keyIndex uint32,
	duplicateStoredKeyIndex uint32,
	revoked bool,
	weight uint16,
) (err error) {

	// Append revoked status and weight
	m.weightAndRevokedStatusBytes, err = appendWeightAndRevokedStatus(m.weightAndRevokedStatusBytes, revoked, weight)
	if err != nil {
		return err
	}

	if !m.deduplicated {
		// Set deduplication flag
		m.deduplicated = true

		// Save mapping start key index.
		m.startIndexForMapping = keyIndex
	}

	// Append duplicate stored key index to mapping
	m.mappingBytes, err = appendStoredKeyIndexToMappings(m.mappingBytes, duplicateStoredKeyIndex)
	if err != nil {
		return err
	}

	return nil
}
