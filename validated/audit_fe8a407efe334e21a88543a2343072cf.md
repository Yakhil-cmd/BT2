### Title
Missing Caller Verification in `update_rewards` Allows Any Address to Permanently Deny Staker Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is specified to be callable **only by the Starkware sequencer**, but the implementation contains no caller verification. Any unprivileged address can call `update_rewards` with `disable_rewards: true` to advance `last_reward_block` without distributing rewards, permanently blocking the legitimate sequencer from distributing rewards for that block.

---

### Finding Description

The protocol specification at `docs/spec.md` line 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `StakingRewardsManagerImpl::update_rewards` at `src/staking/staking.cairo:1449–1507` contains **no `get_caller_address()` check** against any sequencer address. The function is publicly callable by any address.

The `disable_rewards` boolean parameter is the direct analog to the `allowAll` flag described in the external report. When an attacker calls `update_rewards(staker_address, disable_rewards: true)`, the function:

1. Passes the `REWARDS_ALREADY_UPDATED` guard (lines 1454–1458), since it is a new block.
2. Validates the staker exists and is active (lines 1466–1482).
3. **Writes `current_block_number` to the global `last_reward_block` storage variable** (line 1485).
4. Hits the early-return branch at line 1487–1489 (`if disable_rewards || self.is_pre_consensus() { return; }`) — **no rewards are distributed**.

After this, the legitimate sequencer's call to `update_rewards` for the same block will revert with `REWARDS_ALREADY_UPDATED` (line 1454–1458), and the staker permanently loses rewards for that block. Because `last_reward_block` is a **single global state variable** (not per-staker), one attacker call per block is sufficient to deny rewards to **all stakers** for that block.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker can front-run the sequencer every block to permanently deny all stakers their consensus-era block rewards. Each successful attacker call irrevocably sets `last_reward_block` to the current block without distributing rewards. The staker's `unclaimed_rewards_own` is never incremented for that block, and there is no recovery path — the block is gone. Sustained over many blocks, this constitutes a complete denial of all staking yield.

---

### Likelihood Explanation

**High.** The function is publicly callable with no access restriction whatsoever. Any address can call it. The attack requires only a single transaction per block, costs only gas, and requires no special privileges, tokens, or prior setup beyond knowing a valid active staker address (which is public on-chain).

---

### Recommendation

1. Add a caller check at the top of `update_rewards` to assert `get_caller_address() == sequencer_address`, where `sequencer_address` is stored in contract storage and set during initialization (analogous to how `update_rewards_from_attestation_contract` checks `CALLER_IS_NOT_ATTESTATION_CONTRACT`).
2. Remove or restrict the `disable_rewards` parameter so it cannot be weaponized by an untrusted caller to suppress reward distribution while still advancing `last_reward_block`.

---

### Proof of Concept

1. Staker stakes and waits K epochs; consensus rewards are activated via `set_consensus_rewards_first_epoch`.
2. A new block arrives. The attacker (any address) calls:
   ```
   IStakingRewardsManagerDispatcher { contract_address: staking }.update_rewards(
       staker_address: victim_staker,
       disable_rewards: true
   );
   ```
3. Inside `update_rewards` (lines 1484–1489):
   - `self.last_reward_block.write(current_block_number)` — block is consumed.
   - `if disable_rewards { return; }` — no rewards distributed.
4. The sequencer subsequently calls `update_rewards(victim_staker, disable_rewards: false)`.
5. The check at line 1454–1458 fires: `current_block_number > self.last_reward_block.read()` is **false** → reverts with `REWARDS_ALREADY_UPDATED`.
6. The staker permanently loses block rewards for that block. Repeat every block.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
