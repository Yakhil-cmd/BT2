### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Grief Staker Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` has no caller access control. Any unprivileged address can call it with `disable_rewards=true`, which writes the current block number into the global `last_reward_block` storage slot and returns without distributing rewards. Because the function enforces `current_block_number > last_reward_block`, the legitimate sequencer's subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`, and the staker's consensus rewards for that block are permanently lost.

### Finding Description
`StakingRewardsManagerImpl::update_rewards` is the entry point through which the Starknet sequencer distributes per-block consensus rewards to stakers. The protocol specification states its access control is "Only starkware sequencer," but the implementation contains no such check. [1](#0-0) 

The function's only guards are:

1. `general_prerequisites()` — checks the contract is not paused.
2. `current_block_number > self.last_reward_block.read()` — enforces one call per block.
3. Staker existence and active-balance checks.

There is no `get_caller_address()` comparison against a sequencer or operator role. After passing those checks, the function unconditionally writes the current block number to the global `last_reward_block` slot: [2](#0-1) 

When `disable_rewards` is `true` (or `is_pre_consensus()` is true), the function returns immediately after that write, distributing nothing: [3](#0-2) 

Because `last_reward_block` is a single global slot (not per-staker), a single call with any valid `staker_address` and `disable_rewards=true` exhausts the reward slot for the entire block for every staker. [4](#0-3) 

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, disable_rewards=true)` once per block:

- Writes `last_reward_block = current_block_number`.
- Causes every subsequent legitimate sequencer call in that block to revert with `REWARDS_ALREADY_UPDATED`.
- Stakers accumulate zero `unclaimed_rewards_own` for every griefed block; those block rewards are never minted or credited and cannot be recovered retroactively.

Sustained over many blocks, this permanently destroys all consensus-phase staking yield for all stakers and their delegators.

### Likelihood Explanation
**High.** The function is publicly callable on-chain with no special role or token balance required. The only cost to the attacker is gas per block. On Starknet, transaction fees are low, making continuous griefing economically viable. The attacker needs only to know one active staker address (trivially obtained from `NewStaker` events) and call `update_rewards` with `disable_rewards=true` once per block.

### Recommendation
Add a caller check inside `update_rewards` that restricts execution to the authorized sequencer address (or a dedicated operator role). For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

Alternatively, restrict the function to a role managed by `RolesComponent` (e.g., `OPERATOR`) consistent with the existing role system used elsewhere in the protocol. [5](#0-4) 

### Proof of Concept

1. Deploy the protocol in consensus-rewards phase (after `set_consensus_rewards_first_epoch` epoch has passed).
2. Advance to a new block.
3. As an arbitrary unprivileged address `ATTACKER`, call:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(
       staker_address = <any_active_staker>,
       disable_rewards = true
   )
   ```
   This succeeds: `last_reward_block` is set to the current block, no rewards are distributed.
4. The legitimate sequencer now calls `update_rewards(staker_address, disable_rewards=false)` in the same block. It reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number == last_reward_block`.
5. Repeat step 2–4 every block. The staker's `unclaimed_rewards_own` never increases despite being an active consensus participant.
6. Confirm: `staking_dispatcher.staker_info_v1(staker_address).unclaimed_rewards_own == 0` after many blocks. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
