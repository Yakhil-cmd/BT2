### Title
Missing Pause Guard in `staker_migration` Allows Pool Upgrades While Contract Is Paused - (File: src/staking/staking.cairo)

### Summary
The `staker_migration` function in the `StakingMigrationImpl` is the only public state-changing function in the staking contract that does not call `general_prerequisites()`, which contains the `assert_is_unpaused()` check. Any unprivileged caller can invoke `staker_migration` to upgrade pool contracts to V3 even when the staking contract is paused, bypassing the protocol's emergency freeze mechanism.

### Finding Description
Every state-mutating function in `src/staking/staking.cairo` begins with `self.general_prerequisites()`, which internally calls `self.assert_is_unpaused()`:

```cairo
fn assert_is_unpaused(self: @ContractState) {
    assert!(!self.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
}
``` [1](#0-0) 

The `staker_migration` function, however, skips this check entirely:

```cairo
fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
    // Assert the staker exists.
    self._internal_staker_info(:staker_address);
    // ... no general_prerequisites() call ...
``` [2](#0-1) 

The function is callable by **any address** (no access control), as confirmed by the spec: [3](#0-2) 

By contrast, every other state-changing function — `stake`, `unstake_intent`, `unstake_action`, `claim_rewards`, `add_stake_from_pool`, `remove_from_delegation_pool_intent`, `remove_from_delegation_pool_action`, `switch_staking_delegation_pool`, `update_rewards`, `update_rewards_from_attestation_contract`, `set_public_key`, `set_peer_id`, etc. — all begin with `self.general_prerequisites()`: [4](#0-3) [5](#0-4) [6](#0-5) 

The pause test suite confirms all of the above functions are blocked when paused, but `staker_migration` has no corresponding pause test: [7](#0-6) 

### Impact Explanation
`staker_migration` upgrades each of a staker's pool contracts from V2 to V3 by calling `add_new_implementation` and `replace_to` on the pool's `IReplaceableDispatcher`, then marks the staker's version as `LATEST_STAKER_VERSION`: [8](#0-7) 

The pause mechanism exists precisely to freeze all state changes during a security incident. If the security agent pauses the contract because a vulnerability was discovered in the new V3 pool implementation (stored in `pool_contract_class_hash` / `pool_eic_class_hash`), an unprivileged attacker can still call `staker_migration` for any staker to force their pool contracts to be upgraded to the vulnerable V3 implementation. This circumvents the emergency freeze and can expose delegators' funds to the vulnerability the pause was meant to contain — resulting in **temporary freezing of funds** or **theft of unclaimed yield** depending on the nature of the vulnerability in the new pool class.

### Likelihood Explanation
- The staking contract's pause is a live, deployed security mechanism triggered by a `security_agent` role.
- `staker_migration` is callable by any address with no restrictions.
- During a real security incident where the pause is active, an attacker (or even a well-meaning but uninformed user) can call `staker_migration` for any staker, forcing pool upgrades that the pause was intended to block.
- The scenario is realistic: a security agent pauses the contract upon discovering a bug in the new pool class hash, but the migration can still proceed.

### Recommendation
Add `self.general_prerequisites()` as the first statement in `staker_migration`, consistent with every other state-changing function in the contract:

```cairo
fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
    self.general_prerequisites(); // ADD THIS
    // Assert the staker exists.
    self._internal_staker_info(:staker_address);
    // ...
}
``` [9](#0-8) 

### Proof of Concept
1. Security agent calls `pause()` on the staking contract due to a discovered vulnerability in the new V3 pool class hash.
2. Staking contract `is_paused` is now `true`; all standard functions revert with `CONTRACT_IS_PAUSED`.
3. Attacker (any address) calls `staker_migration(victim_staker_address)`.
4. The function executes without reverting — no pause check exists.
5. The victim staker's pool contracts are upgraded to the vulnerable V3 implementation via `add_new_implementation` + `replace_to`.
6. The staker's version is marked as `LATEST_STAKER_VERSION`, making the upgrade irreversible through this path.
7. Delegators in the upgraded pool are now exposed to the vulnerability the pause was meant to contain. [2](#0-1)

        fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
            // Assert the staker exists.
            self._internal_staker_info(:staker_address);

            // Assert staker already migrated to V2.
            assert!(
                !self.staker_own_balance_trace.entry(staker_address).is_empty(),
                "{}",
                Error::STAKER_NOT_MIGRATED,
            );

            // Assert the staker is not migrated to V3 yet.
            assert!(
                !self.staker_version.read(staker_address).is_latest(),
                "{}",
                Error::STAKER_ALREADY_MIGRATED,
            );

            // Prepare the implementation data.
            let pool_class_hash = self.pool_contract_class_hash.read();
            let pool_eic_class_hash = self.pool_eic_class_hash.read();
            // Sanity checks.
            assert!(pool_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            assert!(pool_eic_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            let eic_data = EICData { eic_hash: pool_eic_class_hash, eic_init_data: [].span() };
            let implementation_data = ImplementationData {
                impl_hash: pool_class_hash, eic_data: Option::Some(eic_data), final: false,
            };

            // Upgrade pools.
            let pools = self.staker_pool_info.entry(staker_address).pools;
            for pool_contract_ptr in pools.keys_iter() {
                let pool_contract = pool_contract_ptr.read();
                let pool_replaceable_dispatcher = IReplaceableDispatcher {
                    contract_address: pool_contract,
                };
                pool_replaceable_dispatcher.add_new_implementation(:implementation_data);
                pool_replaceable_dispatcher.replace_to(:implementation_data);
            }

            // Mark the staker's version.
            self.staker_version.write(staker_address, LATEST_STAKER_VERSION);
        }
