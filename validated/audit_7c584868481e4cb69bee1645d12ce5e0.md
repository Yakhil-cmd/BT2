Audit Report

## Title
`voting_power_refreshed_timestamp_seconds` Inactivity Penalty Bypassed via Neuron Split - (File: `rs/nns/governance/src/governance.rs`)

## Summary
The NNS governance inactivity penalty reduces a neuron's `deciding_voting_power` to zero after 7 months without a governance action. The `split_neuron` function constructs the child neuron via `NeuronBuilder::new()`, which defaults `voting_power_refreshed_timestamp_seconds` to `created_timestamp_seconds` (i.e., `now()`), and does not copy the parent's stale timestamp. Any neuron holder whose deciding voting power has been reduced to zero can call `ManageNeuron::Split` to create a child neuron with a fresh timestamp, fully recovering governance influence without performing any legitimate governance action.

## Finding Description
In `NeuronBuilder::new()`, `voting_power_refreshed_timestamp_seconds` is initialized to `created_timestamp_seconds`: [1](#0-0) 

In `split_neuron`, the child is built with `created_timestamp_seconds = now()` and copies several parent fields (`hot_keys`, `followees`, `kyc_verified`, `auto_stake_maturity`, `not_for_profit`, `joined_community_fund_timestamp_seconds`, `neuron_type`), but never calls `with_voting_power_refreshed_timestamp_seconds` with the parent's value: [2](#0-1) 

The `deciding_voting_power` is computed by applying `deciding_voting_power_adjustment_factor` to `now - voting_power_refreshed_timestamp_seconds`: [3](#0-2) 

The default thresholds are 6 months (`start_reducing_voting_power_after_seconds`) and 1 month (`clear_following_after_seconds`), so any neuron inactive for 7+ months has `deciding_voting_power = 0`: [4](#0-3) 

The existing test suite explicitly asserts and validates this behavior — the child's `voting_power_refreshed_timestamp_seconds` is always `now()` and always strictly greater than the parent's: [5](#0-4) 

The builder method `with_voting_power_refreshed_timestamp_seconds` exists but is gated to `#[cfg(test)]`, preventing its use in production code: [6](#0-5) 

The same pattern exists in `disburse_to_neuron`, which also creates a child neuron without inheriting the parent's staleness: [7](#0-6) 

## Impact Explanation
A neuron holder with a fully stale neuron (deciding VP = 0) can call `ManageNeuron::Split` to create a child neuron with `voting_power_refreshed_timestamp_seconds = now()`. The child receives full `deciding_voting_power` proportional to its stake. The attacker recovers governance influence that the NNS protocol explicitly reduced to zero, without voting, setting following, or calling `refresh_voting_power`. This is a concrete NNS governance security impact: the inactivity-penalty invariant is broken, allowing inactive participants to maintain full governance sway indefinitely by periodically splitting. This matches the **High ($2,000–$10,000)** impact class: "Significant NNS security impact with concrete user or protocol harm."

## Likelihood Explanation
Any unprivileged NNS governance user who controls a neuron inactive for 6+ months can trigger this. `ManageNeuron::Split` is a standard, permissionless operation requiring no special privileges, key compromise, or subnet-majority. The only precondition — owning a neuron that has not refreshed in 6+ months — is realistic for long-term passive holders. The attack is repeatable: after splitting, the attacker can split again from the child before it goes stale, maintaining full deciding voting power indefinitely.

## Recommendation
In `split_neuron`, after building the child neuron, add a call to inherit the parent's `voting_power_refreshed_timestamp_seconds`:

```rust
.with_voting_power_refreshed_timestamp_seconds(
    parent_neuron.voting_power_refreshed_timestamp_seconds,
)
```

The `#[cfg(test)]` gate on `with_voting_power_refreshed_timestamp_seconds` must be removed (the comment already says "Feel free to use in production code"): [6](#0-5) 

The same fix should be applied to `disburse_to_neuron`. The existing test at line 5602–5616 of `rs/nns/governance/tests/governance.rs` must be updated to assert that the child's `voting_power_refreshed_timestamp_seconds` equals the parent's, not `now()`.

## Proof of Concept
1. Create Neuron A with 1000 ICP staked. Advance time by 7+ months without any governance action → `deciding_voting_power(A) = 0`.
2. Call `ManageNeuron { command: Split { amount_e8s: 999_990_000, memo: None } }` on Neuron A.
3. Observe that child Neuron B is created with `voting_power_refreshed_timestamp_seconds = now()`.
4. Compute `deciding_voting_power(B)` → full deciding VP ≈ 999.99 ICP worth.
5. Neuron A retains `deciding_voting_power = 0`; Neuron B has full deciding VP.
6. Net result: attacker recovered ~999.99 ICP worth of deciding voting power from a neuron that was supposed to have zero governance influence.

This is directly reproducible as a unit test by modifying the existing test at `rs/nns/governance/tests/governance.rs` around line 5560: set the parent neuron's `voting_power_refreshed_timestamp_seconds` to 7+ months in the past before calling `split_neuron`, then assert that the child's `deciding_voting_power` is zero (which it currently is not).
