### Title
Commission Front-Running via `set_commission_commitment` Allows Staker to Steal Delegator Yield - (File: `src/staking/staking.cairo`)

---

### Summary

A staker can front-run a delegator's `enter_delegation_pool` transaction by first establishing a `CommissionCommitment` and then immediately raising the commission to its maximum allowed value. The delegator enters the pool expecting the commission they observed on-chain, but the commission applied to their rewards is the higher value set by the staker. There is no slippage-protection parameter in `enter_delegation_pool`.

---

### Finding Description

The `set_commission_commitment` function allows a staker to register a commitment with a `max_commission` ceiling that can be **higher** than the current commission. While the commitment is active, `set_commission` permits the staker to raise the commission to any value up to `max_commission` in a single transaction. [1](#0-0) 

The code itself acknowledges this gap: [2](#0-1) 

Meanwhile, `enter_delegation_pool` accepts no `expected_commission` parameter and performs no commission check: [3](#0-2) 

The commission stored in the staking contract is read at reward-distribution time via `get_commission_from_staking_contract`, so whatever value is in storage when rewards are split is what the delegator receives: [4](#0-3) 

Reward splitting uses the live commission value: [5](#0-4) 

**Attack sequence:**

1. Staker has commission = 500 (5 %).
2. Staker calls `set_commission_commitment(max_commission: 9000, expiration_epoch: current_epoch + 1)`.
3. Delegator observes 5 % commission and submits `enter_delegation_pool`.
4. Staker front-runs with `set_commission(commission: 9000)` — commission is now 90 %, effective immediately.
5. Delegator's transaction lands; they are now in a pool with 90 % commission.
6. All subsequent rewards are split 90 % to the staker, 10 % to the pool.

---

### Impact Explanation

Every epoch the delegator remains in the pool, 90 % of the yield that should belong to them is redirected to the staker. This is a direct, ongoing theft of unclaimed yield from an unprivileged delegator. The delegator has no on-chain recourse other than exiting, which itself requires waiting through the `exit_wait_window`, during which the inflated commission continues to apply.

**Impact: High — Theft of unclaimed yield.**

---

### Likelihood Explanation

Starknet has a public mempool. A staker who has already set a `CommissionCommitment` (a prerequisite that can be done well in advance) can monitor for incoming `enter_delegation_pool` calls and front-run them. The two-step setup (`set_commission_commitment` then `set_commission`) is entirely permissionless for any staker. The code comment at line 745–746 explicitly acknowledges that "current commission increase safeguards still allow for sudden commission changes," confirming the developers are aware the mechanism does not fully protect delegators. [2](#0-1) 

---

### Recommendation

Add an `expected_commission: Commission` parameter to `enter_delegation_pool` (and `add_to_delegation_pool`). Inside the function, read the current commission from the staking contract and assert it equals `expected_commission`, reverting if it differs. This is the direct analog of the fix proposed in M-7 and eliminates the race condition without changing the commission-update logic. [6](#0-5) 

---

### Proof of Concept

```
// Step 1 – staker sets a commitment allowing commission up to 90%
staking.set_commission_commitment(max_commission: 9000, expiration_epoch: current_epoch + 1);

// Step 2 – delegator submits enter_delegation_pool (5% commission visible on-chain)
// [tx is in mempool]

// Step 3 – staker front-runs: raises commission to 90%
staking.set_commission(commission: 9000);

// Step 4 – delegator's tx lands; pool now has 90% commission
pool.enter_delegation_pool(reward_address, amount);

// Step 5 – after attestation, rewards are split:
//   staker receives 90% of pool rewards (commission) + own rewards
//   delegator receives only 10% of what they expected
split_rewards_with_commission(pool_rewards_including_commission, commission: 9000);
``` [7](#0-6) [8](#0-7)

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

        fn enter_delegation_pool(
            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.
            self
                .pool_member_info
                .write(pool_member, VInternalPoolMemberInfoTrait::new_latest(:reward_address));

            // Emit events.
            self
                .emit(
                    Events::NewPoolMember { pool_member, staker_address, reward_address, amount },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake: Zero::zero(), new_delegated_stake: amount,
                    },
                );
        }

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
