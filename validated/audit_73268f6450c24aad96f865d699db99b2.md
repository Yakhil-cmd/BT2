### Title
Missing Caller Restriction on `update_rewards` Allows Any Staker to Monopolize Block Reward Distribution - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no caller verification, despite the protocol specification explicitly requiring it to be callable only by the Starknet sequencer. Because the protocol enforces a single `update_rewards` call per block via a global `last_reward_block` guard, any registered staker can front-run the sequencer every block, monopolizing the sole per-block reward slot and permanently freezing unclaimed yield for all other stakers.

---

### Finding Description

The specification for `update_rewards` states under **access control**: *"Only starkware sequencer."* [1](#0-0) 

The implementation, however, applies only `general_prerequisites()` (which checks the contract is not paused and the caller is non-zero) and a global `last_reward_block` block-number guard. There is no check that the caller is the sequencer address: [2](#0-1) 

The `general_prerequisites` helper confirms the absence of any privileged-caller check: [3](#0-2) 

The `last_reward_block` guard is a **global, single-slot** lock: once any call to `update_rewards` succeeds in block N, `last_reward_block` is written to N, and every subsequent call in that same block reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

This is structurally identical to the external report's root cause: a function that is supposed to be invoked only by a specific trusted entity (the sequencer / the FA1.2 `getBalance` callback) has no enforcement of that restriction, so any caller can invoke it and supply attacker-controlled parameters (`staker_address`, `disable_rewards`).

Contrast this with `update_rewards_from_attestation_contract`, which correctly enforces its caller restriction: [5](#0-4) 

---

### Impact Explanation

When an attacker calls `update_rewards(attacker_staker_address, false)` as the first transaction of every block:

1. `last_reward_block` is set to the current block number.
2. The sequencer's intended call for any other staker reverts with `REWARDS_ALREADY_UPDATED`.
3. The attacker's own `unclaimed_rewards_own` is incremented every block via `_update_rewards`.
4. Every other staker's `unclaimed_rewards_own` is never incremented for as long as the attack continues.

The attacker's pool also receives STRK rewards every block via `update_pool_rewards` / `send_rewards_to_delegation_pool`: [6](#0-5) 

**Impact class**: *Temporary (but sustained and indefinitely renewable) freezing of unclaimed yield for all stakers except the attacker*, combined with *theft of unclaimed yield* — the attacker captures block rewards that would otherwise be distributed to other stakers in those blocks.

---

### Likelihood Explanation

- **Entry path**: Any registered staker (unprivileged, no special role required).
- **Cost**: One Starknet transaction per block. Gas fees on Starknet are low; the attacker's own block rewards easily exceed the cost if they hold any non-trivial stake.
- **Detectability**: The attack is on-chain visible but unstoppable without a contract upgrade, since there is no sequencer-address check to enforce.
- **Likelihood**: High.

---

### Recommendation

Store the authorized sequencer address in contract storage and add a caller check to `update_rewards`, mirroring the pattern already used for `update_rewards_from_attestation_contract`:

```cairo
fn assert_caller_is_sequencer(self: @ContractState) {
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
}
```

Add `self.assert_caller_is_sequencer();` as the first line of `update_rewards`, analogous to: [7](#0-6) 

---

### Proof of Concept

1. Attacker registers as a staker via `stake(...)` with any amount ≥ `min_stake`.
2. Attacker waits K epochs for their stake to become effective.
3. Every block, attacker submits `update_rewards(attacker_address, false)` as the first transaction in the block.
4. `last_reward_block` is set to the current block; the sequencer's call for any other staker reverts with `REWARDS_ALREADY_UPDATED`.
5. After N blocks: attacker has accumulated N × (attacker_share × block_rewards) in `unclaimed_rewards_own`; every other staker has accumulated 0 new rewards.
6. Attacker calls `claim_rewards(attacker_address)` to withdraw the stolen yield.

The attack requires no privileged access, no key compromise, and no external dependency — only a registered stake and the ability to submit one transaction per block.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);
