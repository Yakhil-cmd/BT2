### Title
Zero Native Resource Charge in BLS12-381 MSM and Pairing Precompiles with Unbounded Loop Over User-Controlled Input — (`basic_bootloader/src/bootloader/block_flow/ethereum/hooks/eip_2537/msm.rs`, `pairing.rs`)

---

### Summary

The BLS12-381 G1 MSM, G2 MSM, and Pairing precompiles each iterate over a user-controlled number of elliptic-curve pairs while hardcoding `cost_native = 0`. This means the native (proving) resource is never charged regardless of how many pairs the attacker supplies, creating an unbounded loop over a dynamic array with no native resource accounting. An attacker can exhaust the prover's native resource budget within a single block at no extra cost beyond EVM gas.

---

### Finding Description

In `Bls12381G1MSMPrecompile::execute`, `Bls12381G2MSMPrecompile::execute`, and `Bls12381PairingCheckPrecompile::execute`, the native resource cost is hardcoded to zero:

```rust
let cost_native = 0;
resources.charge(&R::from_ergs_and_native(
    cost_ergs,
    <R::Native as zk_ee::system::Computational>::from_computational(cost_native),
))?;
``` [1](#0-0) [2](#0-1) 

After charging (only EVM gas), the code enters an unbounded loop over all pairs derived from the user-supplied input:

```rust
for pair_encoding in input.as_chunks::<G1_MSM_PAIR_LEN>().0.iter() {
    let point = parse_g1_with_subgroup_check(...)?;
    let scalar = parse_integer(...);
    points.push(point);
    scalars.push(scalar);
}
let result: G1Projective = msm(&points, scalars, allocator);
``` [3](#0-2) [4](#0-3) 

The `msm()` function itself is O(n · log n) in elliptic-curve group operations, and `multi_pairing` is similarly O(n). The number of pairs `n = input.len() / pair_len` is entirely attacker-controlled via calldata. EVM gas is charged proportionally to `n`, but native (proving) resources — which represent the actual ZK proof cost — are charged zero for any `n`.

`G1_MSM_PAIR_LEN = 160` bytes, `G2_MSM_PAIR_LEN = 288` bytes, `BLS12_381_PAIR_LEN = 384` bytes. A transaction with the maximum calldata that fits within the EVM gas limit can supply hundreds of pairs. [5](#0-4) 

---

### Impact Explanation

ZKsync OS tracks two resource dimensions: EVM gas (ergs) and native (proving) resources. Block limits exist for both. Because `cost_native = 0` is hardcoded, the native resource block limit is never consumed by BLS12-381 precompile calls, no matter how many pairs are processed. A single transaction can force the prover to execute hundreds of expensive BLS12-381 MSM or pairing operations while the native resource counter remains at zero. This can:

1. Cause the prover to exceed its actual computational budget, producing an unprovable block (valid-execution unprovability / resource accounting bug).
2. Allow a block to be constructed that is valid from the sequencer's perspective but cannot be proven, stalling the chain.

---

### Likelihood Explanation

The BLS12-381 precompiles are standard EVM precompiles (EIP-2537) reachable by any unprivileged transaction sender. No special role or key is required. The attacker only needs to send a transaction calling address `BLS12_G1MSM`, `BLS12_G2MSM`, or `BLS12_PAIRING_CHECK` with a large calldata payload. The EVM gas cost scales with input size, but the native resource cost does not, so the attacker pays a bounded EVM gas fee while imposing unbounded proving cost. [6](#0-5) 

---

### Recommendation

Replace `cost_native = 0` with a native cost that scales linearly with `num_pairs`. Determine the empirical native resource cost per BLS12-381 G1 MSM pair, G2 MSM pair, and pairing pair (via benchmarking the prover), then charge:

```rust
let cost_native = PER_PAIR_NATIVE_COST * num_pairs as u64;
resources.charge(&R::from_ergs_and_native(
    cost_ergs,
    <R::Native as zk_ee::system::Computational>::from_computational(cost_native),
))?;
```

Apply the same fix to all three precompiles: `Bls12381G1MSMPrecompile`, `Bls12381G2MSMPrecompile`, and `Bls12381PairingCheckPrecompile`.

---

### Proof of Concept

1. Deploy no contract. Send a raw transaction to `BLS12_G1MSM` (address `BLS12_G1MSM` as defined in `addresses.rs`) with calldata consisting of `K` valid G1 point + scalar pairs (each 160 bytes), where `K` is chosen to consume the full EVM gas limit.
2. The precompile charges EVM gas = `compute_cost(K * 160, 160, 12000, &DISCOUNT_TABLE_G1_MSM)` — a finite, gas-limit-bounded amount.
3. Native resources charged = 0 regardless of `K`.
4. The `msm()` function executes O(K · log K) BLS12-381 group operations.
5. The block is accepted by the sequencer (EVM gas and native resource counters both within limits), but the prover must prove K MSM operations that were never budgeted in native resources, potentially exceeding the prover's actual capacity. [7](#0-6) [8](#0-7)

impl<R: Resources> SystemFunction<R, Bls12PrecompileErrors> for Bls12381G1MSMPrecompile {
    fn execute<
        D: zk_ee::common_traits::TryExtend<u8> + ?Sized,
        A: core::alloc::Allocator + Clone,
    >(
        input: &[u8],
        output: &mut D,
        resources: &mut R,
        allocator: A,
    ) -> Result<(), zk_ee::system::errors::subsystem::SubsystemError<Bls12PrecompileErrors>> {
        if input.len() == 0 {
            return Err(interface_error!(
                Bls12PrecompileInterfaceError::InvalidInputSize
            ));
        }
        let cost = compute_cost(
            input.len(),
            G1_MSM_PAIR_LEN,
            BLS12_381_G1_MSM_PER_POINT_GAS,
            &DISCOUNT_TABLE_G1_MSM,
        );
        let cost_ergs = Ergs(cost * ERGS_PER_GAS);
        let cost_native = 0;
        resources.charge(&R::from_ergs_and_native(
            cost_ergs,
            <R::Native as zk_ee::system::Computational>::from_computational(cost_native),
        ))?;

        if !input.len().is_multiple_of(G1_MSM_PAIR_LEN) {
            return Err(interface_error!(
                Bls12PrecompileInterfaceError::InvalidInputSize
            ));
        }

        let num_pairs = input.len() / G1_MSM_PAIR_LEN;
        let mut scalars = Vec::with_capacity_in(num_pairs, allocator.clone());
        let mut points = Vec::with_capacity_in(num_pairs, allocator.clone());

        // arkworks MSM allocates inside, so we will do it our way, just parse here
        // G1Projective::msm_bigint(bases, bigints)

        // parse to use Peppinger algorithm
        for pair_encoding in input.as_chunks::<G1_MSM_PAIR_LEN>().0.iter() {
            let point = parse_g1_with_subgroup_check(
                pair_encoding[0..G1_SERIALIZATION_LEN].try_into().unwrap(),
            )?;
            let scalar = parse_integer(
                pair_encoding
                    [G1_SERIALIZATION_LEN..(G1_SERIALIZATION_LEN + SCALAR_SERIALIZATION_LEN)]
                    .try_into()
                    .unwrap(),
            );
            points.push(point);
            scalars.push(scalar);
        }

        let result: G1Projective = msm(&points, scalars, allocator);

        let result = result.into_affine();

        write_g1(result, output);

        Ok(())
    }

impl<R: Resources> SystemFunction<R, Bls12PrecompileErrors> for Bls12381PairingCheckPrecompile {
    fn execute<
        D: zk_ee::common_traits::TryExtend<u8> + ?Sized,
        A: core::alloc::Allocator + Clone,
    >(
        input: &[u8],
        output: &mut D,
        resources: &mut R,
        allocator: A,
    ) -> Result<(), zk_ee::system::errors::subsystem::SubsystemError<Bls12PrecompileErrors>> {
        if input.len() == 0 {
            return Err(interface_error!(
                Bls12PrecompileInterfaceError::InvalidInputSize
            ));
        }
        let num_pairs = input.len() / BLS12_381_PAIR_LEN;
        let cost_ergs = Ergs(
            ((num_pairs as u64) * BLS12_381_PAIRING_PER_PAIR_GAS + BLS12_381_PAIRING_FIXED_GAS)
                * ERGS_PER_GAS,
        );
        let cost_native = 0;
        resources.charge(&R::from_ergs_and_native(
            cost_ergs,
            <R::Native as zk_ee::system::Computational>::from_computational(cost_native),
        ))?;

        if !input.len().is_multiple_of(BLS12_381_PAIR_LEN) {
            return Err(interface_error!(
                Bls12PrecompileInterfaceError::InvalidInputSize
            ));
        }

        let mut g1_points = Vec::with_capacity_in(num_pairs, allocator.clone());
        let mut g2_points = Vec::with_capacity_in(num_pairs, allocator.clone());

        // arkworks MSM allocates inside, so we will do it our way, just parse here
        // G1Projective::msm_bigint(bases, bigints)

        // parse to use Peppinger algorithm
        for pair_encoding in input.as_chunks::<BLS12_381_PAIR_LEN>().0.iter() {
            let g1 = parse_g1_with_subgroup_check(
                pair_encoding[0..G1_SERIALIZATION_LEN].try_into().unwrap(),
            )?;
            let g2 = parse_g2_with_subgroup_check(
                pair_encoding[G1_SERIALIZATION_LEN..(G1_SERIALIZATION_LEN + G2_SERIALIZATION_LEN)]
                    .try_into()
                    .unwrap(),
            )?;
            g1_points.push(g1);
            g2_points.push(g2);
        }

        let pairing_result = <Bls12_381 as Pairing>::multi_pairing(g1_points, g2_points);
        output
            .try_extend([0u8; 31])
            .map_err(|_| out_of_return_memory!())?;

        use crypto::ark_ff::Field;
        if pairing_result.0 == <Bls12_381 as Pairing>::TargetField::ONE {
            output
                .try_extend([1u8])
                .map_err(|_| out_of_return_memory!())?;
        } else {
            output
                .try_extend([0u8])
                .map_err(|_| out_of_return_memory!())?;
        }

        Ok(())
    }
