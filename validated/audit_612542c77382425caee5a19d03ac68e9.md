### Title
`validate_resource_bounds` Unconditionally Checks L2 Gas Price Even When L2 Gas Amount Is Zero, Causing Valid L1-Only Transactions to Be Rejected at Gateway Admission — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

`StatelessTransactionValidator::validate_resource_bounds` applies a `min_gas_price` floor check against `l2_gas.max_price_per_unit` unconditionally, even when `l2_gas.max_amount == 0`. A V3 transaction that legitimately carries only L1 gas (or only L1 data gas) will have `l2_gas.max_price_per_unit = 0` by default. With the production default of `min_gas_price = 8_000_000_000`, the check `0 < 8_000_000_000` evaluates to `true` and the gateway rejects the transaction with `MaxGasPriceTooLow`, even though the transaction is structurally valid and the L2 gas price field is irrelevant when no L2 gas is consumed.

---

### Finding Description

In `validate_resource_bounds`:

```rust
// Line 66-69: passes for L1-only tx because l1_gas contributes non-zero fee
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
}

// Line 71-76: runs unconditionally — no guard for l2_gas.max_amount == 0
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
``` [1](#0-0) 

The first guard (lines 66–69) correctly passes for a transaction with `l1_gas = NON_EMPTY_RESOURCE_BOUNDS` and `l2_gas = Default::default()`, because `max_possible_fee` sums all three resources and the L1 gas contribution is non-zero. [2](#0-1) 

The second guard (lines 71–76) then fires unconditionally. `AllResourceBounds::default()` sets `l2_gas.max_price_per_unit = GasPrice(0)`. The production config sets `min_gas_price = 8_000_000_000`, so `0 < 8_000_000_000` is `true` and the transaction is rejected. [3](#0-2) 

The test suite masks this because the positive-flow test cases (`valid_l1_gas`, `valid_l1_data_gas`) inherit `min_gas_price: 0` from `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING`, making `0 < 0` false and the check pass. No test exercises these cases against the production-default `min_gas_price = 8_000_000_000`. [4](#0-3) 

The `NON_EMPTY_RESOURCE_BOUNDS` constant used in those tests is `ResourceBounds { max_amount: GasAmount(1), max_price_per_unit: GasPrice(1) }`, confirming that the L2 gas price in the L1-only cases is `GasPrice(0)` (the default). [5](#0-4) 

The same structural flaw exists in the stateful validator's `validate_tx_l2_gas_price_within_threshold`, which also checks `tx_l2_gas_price < threshold` unconditionally for `AllResources` transactions without guarding on `l2_gas.max_amount`. [6](#0-5) 

---

### Impact Explanation

Any V3 (`AllResources`) transaction that sets `l2_gas.max_amount = 0` and `l2_gas.max_price_per_unit = 0` — a structurally valid configuration when the sender only intends to pay for L1 or L1-data gas — is permanently rejected at the stateless gateway layer with `MaxGasPriceTooLow`. The transaction never reaches the mempool or blockifier. This matches the **High** impact tier: *"Mempool/gateway/RPC admission rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The trigger is unprivileged: any external user can submit a V3 transaction with only L1 gas bounds. The production config has `validate_resource_bounds: true` and `min_gas_price: 8_000_000_000` by default. The bug fires deterministically for every such transaction. [3](#0-2) 

---

### Recommendation

Guard the L2 gas price check on whether L2 gas is actually being used:

```rust
// Only enforce the L2 gas price floor when the sender actually allocates L2 gas.
if resource_bounds.l2_gas.max_amount.0 > 0
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
```

Apply the same fix to `validate_tx_l2_gas_price_within_threshold` in the stateful validator. Add regression tests that run the `valid_l1_gas` and `valid_l1_data_gas` cases against the production-default `min_gas_price = 8_000_000_000`.

---

### Proof of Concept

1. Construct a valid V3 Invoke transaction with:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas:      ResourceBounds { max_amount: 1, max_price_per_unit: 8_000_000_000 },
       l2_gas:      ResourceBounds { max_amount: 0, max_price_per_unit: 0 },  // default
       l1_data_gas: ResourceBounds { max_amount: 0, max_price_per_unit: 0 },  // default
   }
   ```
2. Submit to the gateway with the production-default `StatelessTransactionValidatorConfig` (`validate_resource_bounds: true`, `min_gas_price: 8_000_000_000`).
3. `max_possible_fee(Tip::ZERO)` returns `1 * 8_000_000_000 = 8_000_000_000 > 0`, so the zero-bounds guard passes.
4. The unconditional check `0 < 8_000_000_000` fires and the gateway returns `MaxGasPriceTooLow { gas_price: 0, min_gas_price: 8_000_000_000 }`.
5. The transaction is rejected despite being structurally valid — the L2 gas price is irrelevant when `l2_gas.max_amount = 0`. [7](#0-6) [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
