### Title
Pool Member Can Indefinitely Reset Delegation Exit Timelock via Repeated `exit_delegation_pool_intent` Calls - (File: src/pool/pool.cairo)

### Summary
The `exit_delegation_pool_intent` function in the Pool contract, and the underlying `update_undelegate_intent_value` in the Staking contract, unconditionally overwrite the `unpool_time` with `Time::now() + exit_wait_window` on every call — even when an active intent already exists. A pool member can exploit this to permanently prevent their own exit from completing, freezing their delegated funds in the intent state indefinitely.

### Finding Description
When a pool member calls `exit_delegation_pool_intent`, the pool contract calls `undelegate_from_staking_contract_intent`, which calls `remove_from_delegation_pool_intent` in the staking contract. [1](#0-0) 

`remove_from_delegation_pool_intent` always calls `update_undelegate_intent_value` with the new amount: [2](#0-1) 

`update_undelegate_intent_value` unconditionally recomputes `unpool_time` as `Time::now() + exit_wait_window` and overwrites the stored intent — there is no check to preserve the existing `unpool_time` if an intent is already in progress: [3](#0-2) 

There is no guard in `exit_delegation_pool_intent` preventing re-entry when `unpool_time` is already set (the only guard, `UNDELEGATE_IN_PROGRESS`, only fires when `staker_removed == true`): [4](#0-3) 

The existing test `test_remove_from_delegation_pool_intent` confirms this behavior is reachable: calling the intent a second time produces a fresh `unpool_time = Time::now() + exit_wait_window`, overwriting the previous one: [5](#0-4) 

### Impact Explanation
A pool member (or any account with control of the pool member key, including a buggy delegating smart contract) can call `exit_delegation_pool_intent(amount)` with any non-zero amount just before the wait window expires, resetting `unpool_time` to `now + exit_wait_window`. Repeating this indefinitely keeps the pool member's funds permanently locked in the `pool_exit_intents` map inside the Staking contract. The funds cannot be recovered via `exit_delegation_pool_action` because `Time::now() >= unpool_time` is never satisfied: [6](#0-5) 

This constitutes **temporary (and in practice permanent) freezing of the pool member's delegated funds** — matching the allowed High impact "Temporary freezing of funds."

### Likelihood Explanation
The entry path is fully reachable by an unprivileged pool member with no special role. The cost per reset is only gas. Realistic triggering scenarios include:

1. A pool member whose private key is compromised — the attacker keeps resetting the timer to prevent fund recovery.
2. A delegating smart contract with a logic bug that re-calls `exit_delegation_pool_intent` on each epoch transition.
3. A pool member who intentionally griefs themselves (e.g., under coercion) to prevent a court-ordered or protocol-level exit.

The `exit_wait_window` is at most 12 weeks (`MAX_EXIT_WAIT_WINDOW`), so the attacker must call roughly once per week, which is trivially automatable. [7](#0-6) 

### Recommendation
In `update_undelegate_intent_value`, check whether a valid intent already exists for the given `undelegate_intent_key`. If one does, preserve the original `unpool_time` rather than recomputing it. Only assign a fresh `unpool_time` when no prior intent exists (i.e., `old_intent_amount.is_zero()`). This mirrors the fix suggested in the reference report: track and limit modifications to the timelock-bearing record. [3](#0-2) 

### Proof of Concept

```
// Setup: pool_member has 100 STRK delegated, exit_wait_window = 1 week.

// T=0: pool_member calls exit_delegation_pool_intent(100)
//   → unpool_time = T+1week, unpool_amount = 100

// T=6days23hrs: pool_member calls exit_delegation_pool_intent(100) again
//   → update_undelegate_intent_value recomputes unpool_time = T+6d23h+1week
//   → old unpool_time is discarded; funds remain locked

// Repeat every ~6d23h indefinitely.
// exit_delegation_pool_action always reverts with INTENT_WINDOW_NOT_FINISHED.
// Pool member's 100 STRK are permanently frozen in pool_exit_intents.
```

The call chain is:
`Pool::exit_delegation_pool_intent` → `Pool::undelegate_from_staking_contract_intent` → `Staking::remove_from_delegation_pool_intent` → `Staking::update_undelegate_intent_value` (resets `unpool_time` unconditionally). [8](#0-7) [9](#0-8)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }

        fn remove_from_delegation_pool_intent(
            ref self: ContractState,
            staker_address: ContractAddress,
            identifier: felt252,
            amount: Amount,
        ) -> Timestamp {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let staker_info = self.internal_staker_info(:staker_address);
            let pool_contract = get_caller_address();
            let token_address = self
                .staker_pool_info
                .entry(staker_address)
                .get_pool_token(:pool_contract)
                .expect_with_err(Error::CALLER_IS_NOT_POOL_CONTRACT);
            let decimals = self.get_token_decimals(:token_address);
            let normalized_amount = NormalizedAmountTrait::from_native_amount(:amount, :decimals);

            // Update the delegated stake according to the new intent.
            let undelegate_intent_key = UndelegateIntentKey { pool_contract, identifier };
            let old_intent_amount = self.get_pool_exit_intent(:undelegate_intent_key).amount;
            let new_intent_amount = normalized_amount;
            // After this call, the staker balance will be updated.
            let (old_delegated_stake, new_delegated_stake) = self
                .update_delegated_stake(
                    :staker_address,
                    :token_address,
                    :pool_contract,
                    :staker_info,
                    :old_intent_amount,
                    :new_intent_amount,
                );
            self
                .update_undelegate_intent_value(
                    :token_address, :staker_info, :undelegate_intent_key, :new_intent_amount,
                );

            self
                .emit(
                    Events::RemoveFromDelegationPoolIntent {
                        staker_address,
                        pool_contract,
                        token_address,
                        identifier,
                        old_intent_amount: old_intent_amount.to_native_amount(:decimals),
                        new_intent_amount: amount,
                    },
                );
            // If the staker is in the process of unstaking (intent called),
            // an event indicating the staked amount (own and delegated) to be zero
            // had already been emitted, thus unneeded now.
            if staker_info.unstake_time.is_none() {
                self
                    .emit(
                        Events::StakeDelegatedBalanceChanged {
                            staker_address,
                            token_address,
                            old_delegated_stake: old_delegated_stake.to_native_amount(:decimals),
                            new_delegated_stake: new_delegated_stake.to_native_amount(:decimals),
                        },
                    );
            }
            self.get_pool_exit_intent(:undelegate_intent_key).unpool_time
        }

        fn update_undelegate_intent_value(
            ref self: ContractState,
            token_address: ContractAddress,
            staker_info: InternalStakerInfoLatest,
            undelegate_intent_key: UndelegateIntentKey,
            new_intent_amount: NormalizedAmount,
        ) {
            let undelegate_intent_value = if new_intent_amount.is_zero() {
                Zero::zero()
            } else {
                let unpool_time = staker_info
                    .compute_unpool_time(exit_wait_window: self.exit_wait_window.read());
                assert!(token_address.is_non_zero(), "{}", InternalError::TOKEN_IS_ZERO_ADDRESS);
                UndelegateIntentValue { amount: new_intent_amount, unpool_time, token_address }
            };
            self.pool_exit_intents.write(undelegate_intent_key, undelegate_intent_value);
        }
