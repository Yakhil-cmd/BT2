### Title
Gateway L2 Gas Price Admission Check Uses Stale Previous-Block Price Instead of Next-Block Price - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful admission check validates a transaction's `max_price_per_unit` (L2 gas) against the **previous** block's L2 gas price, but the transaction will be executed in the **next** block whose L2 gas price is computed via EIP-1559 and may differ by up to ~12.5%. This causes the gateway to admit transactions that will fail blockifier pre-validation (when price is rising) and to reject transactions that are actually valid (when price is falling). The code itself contains a TODO acknowledging the wrong reference value is used.

The two checks use different reference prices, creating a systematic mismatch.

### Impact Explanation

**Case 1 — Rising L2 gas price (high congestion):** The next block's price `P'` exceeds the previous block's price `P`. A transaction with `max_price_per_unit` in the range `[P * threshold%, P')` passes the gateway check (admitted to mempool) but fails `check_fee_bounds` during blockifier pre-validation with `InsufficientResourceBounds`. The transaction is never included in a block; the user's nonce is not incremented; the user must resubmit. Sequencer resources are wasted processing transactions that are guaranteed to fail.

**Case 2 — Falling L2 gas price (low congestion):** The next block's price `P'` is below the previous block's price `P`. A transaction with `max_price_per_unit` in the range `[P', P * threshold%)` is rejected at the gateway even though it would pass `check_fee_bounds` during execution. Valid transactions are denied service.

With `min_gas_price_percentage = 100` (the default) and EIP-1559 allowing up to ~12.5% price change per block, the affected range is up to 12.5% of the current L2 gas price on every block where congestion changes.

This matches the impact category: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The L2 gas price changes on every block where `gas_used ≠ gas_target`. This is the normal operating condition of any active network. The discrepancy is bounded but persistent and predictable: an attacker who monitors the mempool can craft transactions that exploit the admission window to flood the mempool with transactions that will fail execution, or can predict when valid transactions will be incorrectly rejected.

### Recommendation

Replace `previous_block_l2_gas_price` with the computed next-block L2 gas price. The consensus orchestrator already computes this value via `calculate_next_l2_gas_price_for_fin`; it should be made available to the gateway (e.g., stored in the block header as the TODO comment suggests: `// TODO(Arni): getnext_l2_gas_price from the block header`). Until then, the gateway should at minimum apply a forward-looking margin that accounts for the maximum EIP-1559 price increase per block.

### Proof of Concept

1. Observe the current L2 gas price `P` from the latest block (e.g., `P = 1_000_000_000` fri).
2. The previous block was at 90% of `gas_target`, so the next block's price will be `P' = P * (1 - delta) ≈ 0.9375 * P` (falling).
3. Submit a transaction with `max_price_per_unit = 0.95 * P`. This is above `P' = 0.9375 * P` so it would pass `check_fee_bounds` during execution.
4. The gateway computes `threshold = P * 100% = P`. Since `0.95 * P < P`, the transaction is **rejected** with `GAS_PRICE_TOO_LOW` even though it is valid for the next block.

Conversely, when price is rising:
1. Previous block price `P`, next block price `P' = 1.0625 * P` (rising).
2. Submit a transaction with `max_price_per_unit = 1.03 * P`. Gateway threshold is `P`; `1.03 * P > P` → **admitted**.
3. During blockifier execution, `check_fee_bounds` checks `1.03 * P >= P' = 1.0625 * P` → **fails**. Transaction is rejected by the blockifier after consuming sequencer resources. [5](#0-4) [6](#0-5)

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
```
