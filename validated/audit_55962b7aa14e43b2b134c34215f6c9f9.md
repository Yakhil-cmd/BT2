### Title
Unprivileged Caller Can Permanently Deny Block Rewards to All Stakers via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and accepts a caller-controlled `disable_rewards: bool` parameter. Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` once per block. This writes the global `last_reward_block` state variable **before** the early-return guard, permanently consuming the block's reward slot without distributing any rewards. All subsequent legitimate calls in the same block revert with `REWARDS_ALREADY_UPDATED`, causing all stakers and delegators to permanently lose that block's yield.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` at `src/staking/staking.cairo` lines 1449–1507 is a public function with no role-based access control. Its only guards are `general_prerequisites()` (contract not paused, caller not zero) and a per-block uniqueness check.

The execution order is:

1. **Line 1453–1458** — asserts `current_block_number > last_reward_block` (one call per block, globally).
2. **Lines 1466–1482** — validates the supplied `staker_address` is an active staker with non-zero STRK balance.
3. **Line 1485** — `self.last_reward_block.write(current_block_number)` — **global state is committed here, unconditionally.**
4. **Lines 1487–1489** — `if disable_rewards || self.is_pre_consensus() { return; }` — exits with no rewards distributed.
5. **Lines 1492–1506** — reward calculation and distribution (never reached when `disable_rewards` is `true`). [1](#0-0) 

`last_reward_block` is a **single global** `BlockNumber` storage slot, not a per-staker map: [2](#0-1) 

Because the write at line 1485 precedes the `disable_rewards` guard at line 1487, any caller who passes `disable_rewards: true` atomically:
- Marks the current block as "processed" for the entire protocol.
- Distributes zero rewards to any staker or delegator.
- Prevents every other call in that block from distributing rewards (they revert with `REWARDS_ALREADY_UPDATED`).

This is only exploitable during the consensus rewards phase (`is_pre_consensus()` returns `false`), i.e., after `consensus_rewards_first_epoch` is set. [3](#0-2) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Each block's rewards are calculated from `avg_block_duration` and `yearly_mint` at the moment `update_current_epoch_block_rewards` is called. There is no mechanism to retroactively distribute rewards for a block whose `last_reward_block` slot has already been consumed. An attacker repeating this call every block permanently freezes all consensus-phase block rewards for every staker and every delegator in every pool. The loss is irreversible. [4](#0-3) 

---

### Likelihood Explanation

**Medium.** The attack requires no privileges, no capital, and no leaked keys — only the ability to submit a transaction. The attacker must supply any currently-active staker address with non-zero STRK balance (publicly queryable via `get_stakers`). The per-block cost on Starknet is negligible. The only natural mitigation would be if the sequencer consistently orders the legitimate consensus call before the attacker's, but the protocol enforces no such ordering guarantee.

---

### Recommendation

Two complementary fixes:

1. **Add access control** to `update_rewards` so only a designated trusted caller (e.g., a consensus operator role) can invoke it.
2. **Move the `last_reward_block` write** to after the `disable_rewards` guard, so a call with `disable_rewards: true` does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only commit last_reward_block when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
``` [5](#0-4) 

---

### Proof of Concept

1. Consensus rewards phase is active (`consensus_rewards_first_epoch` has been set and the current epoch has passed it).
2. Attacker queries `get_stakers(current_epoch)` to obtain any valid active staker address `S`.
3. At the start of block `N`, attacker submits `update_rewards(S, disable_rewards: true)`.
4. `last_reward_block` is written to `N`; function returns early — zero rewards distributed.
5. The legitimate consensus mechanism submits `update_rewards(any_staker, false)` in block `N` — it reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number == last_reward_block`.
6. All stakers and all pool delegators permanently lose their block `N` rewards.
7. Attacker repeats in block `N+1`, `N+2`, … to freeze all future consensus rewards. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
