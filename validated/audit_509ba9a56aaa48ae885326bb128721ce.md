### Title
Gateway `skip_stateful_validations` Admits Invoke Transactions with Arbitrary Signatures for Accounts with Pending Deploy-Account Transactions — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function bypasses the account's `__validate__` entry point for any invoke transaction with `nonce=1` when the sender address has *any* transaction in the mempool. An unprivileged attacker who observes a `deploy_account` transaction in the public mempool can submit an invoke transaction with `nonce=1` and a completely invalid signature for the target account. The gateway admits this transaction without running signature validation, placing an invalid transaction in the mempool and blocking the legitimate user's first post-deployment invoke.

---

### Finding Description

**Trigger path** in `extract_state_nonce_and_run_validations`: [1](#0-0) 

The function calls `run_pre_validation_checks`, which calls `skip_stateful_validations`: [2](#0-1) 

`skip_stateful_validations` returns `true` (skip) when:
- the transaction is an `Invoke`,
- `tx.nonce() == Nonce(Felt::ONE)`, and
- `account_nonce == Nonce(Felt::ZERO)`, and
- `account_tx_in_pool_or_recent_block(sender_address)` returns `true`. [3](#0-2) 

When `skip_validate = true`, `run_validate_entry_point` sets `execution_flags.validate = false`, so the account's `__validate__` entry point is **never called**: [4](#0-3) 

`account_tx_in_pool_or_recent_block` returns `true` if the account has *any* transaction in the pool — not specifically a `deploy_account`: [5](#0-4) 

**Attack sequence:**

1. Alice submits a `deploy_account` tx for address `X` (nonce 0). It enters the mempool.
2. Attacker observes Alice's `deploy_account` tx in the public mempool.
3. Attacker submits an `Invoke` tx from address `X`, `nonce=1`, with an arbitrary/invalid signature and valid resource bounds.
4. Gateway stateless validation passes (signature *size* is checked, not validity).
5. `validate_state_preconditions` passes: nonce 1 is within the allowed gap from account nonce 0.
6. `validate_by_mempool` passes: no duplicate nonce in pool yet, fee escalation not triggered.
7. `skip_stateful_validations`: `nonce==1`, `account_nonce==0`, `account_tx_in_pool_or_recent_block(X)==true` → returns `true`.
8. `run_validate_entry_point` is called with `skip_validate=true` → `__validate__` is **skipped**.
9. Attacker's invalid-signature invoke tx is admitted to the mempool.
10. Alice submits her legitimate invoke tx (nonce 1) → rejected with `MempoolError::DuplicateNonce`.

The `validate_by_mempool` call that precedes `skip_stateful_validations` only checks nonce ordering and fee escalation — it does not verify signatures: [6](#0-5) 

The only signature check in the entire gateway path is the `__validate__` entry point, which is the one being skipped.

---

### Impact Explanation

**Impact: High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

An unprivileged attacker can inject an invoke transaction with an invalid signature into the mempool for any account that has a pending `deploy_account` transaction. Consequences:

- The legitimate user's first post-deployment invoke (nonce 1) is rejected as `DuplicateNonce` because the attacker's invalid tx already occupies that slot.
- With fee escalation enabled (default), the user must pay a higher fee to replace the attacker's tx; the attacker can keep escalating to sustain the DoS.
- With fee escalation disabled, the user's tx is permanently blocked until the attacker's tx is eventually executed (and fails `__validate__` at the blockifier level, leaving nonce unchanged) and the user retries.
- The attacker can repeat this for every block, continuously disrupting the deploy-account + invoke UX flow.

The broken invariant: *every invoke transaction admitted to the mempool must have passed the account's `__validate__` entry point, or have a cryptographically sound reason to skip it.*

---

### Likelihood Explanation

- **Unprivileged**: requires only the ability to submit transactions to the gateway and observe the public mempool.
- **Low cost**: the attacker only needs to pay valid resource bounds; no private key for the victim account is needed.
- **Targeted**: the attacker can select any account currently deploying, making this a targeted griefing attack.
- **Persistent**: the attack window lasts from when the `deploy_account` tx enters the mempool until it is committed.

---

### Recommendation

1. **Narrow the skip condition**: instead of checking `account_tx_in_pool_or_recent_block` (which matches any tx type), explicitly verify that a `deploy_account` transaction for the sender address is present in the mempool. This requires the mempool to expose a `deploy_account_in_pool(address)` query.
2. **Alternatively**, track the `deploy_account` tx hash in the gateway session and pass it as a proof to `skip_stateful_validations`, similar to how `native_blockifier`'s `PyValidator::should_run_stateful_validations` uses `deploy_account_tx_hash`: [7](#0-6) 

The `PyValidator` path requires the caller to supply the actual `deploy_account_tx_hash`, making it impossible to trigger the skip without knowing the legitimate deploy tx. The Apollo gateway path should adopt the same pattern.

---

### Proof of Concept

```
1. Alice: submit DeployAccount(address=X, nonce=0, valid_sig)
   → mempool: pool contains (X, nonce=0)

2. Attacker: submit Invoke(sender=X, nonce=1, signature=[0xDEAD, 0xBEEF], valid_resource_bounds)
   Gateway checks:
     stateless: OK (signature size within limit)
     validate_state_preconditions: nonce=1 in [0, 0+max_gap], OK
     validate_by_mempool: no dup nonce, OK
     skip_stateful_validations:
       nonce==1 ✓, account_nonce==0 ✓
       account_tx_in_pool_or_recent_block(X) → true (deploy_account is in pool) ✓
       → returns true (skip)
     run_validate_entry_point(skip_validate=true): __validate__ NOT called
   → attacker's tx admitted to mempool with invalid signature

3. Alice: submit Invoke(sender=X, nonce=1, valid_sig)
   validate_by_mempool → DuplicateNonce (attacker's tx already at nonce=1)
   → Alice's legitimate invoke is REJECTED

4. Attacker repeats step 2 each time Alice retries, sustaining the DoS.
``` [3](#0-2) [4](#0-3)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
