The `general_prerequisites` function is the decisive piece:

### Title
Missing Caller Authorization on `update_rewards` Allows Any Address to Permanently Suppress Per-Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

`update_rewards` is documented as "Only starkware sequencer" but the implementation enforces no such check. Any non-zero address can call it first for a given block with `disable_rewards: true`, which writes `last_reward_block` to the current block and returns without distributing rewards. Because `last_reward_block` is a **global, single-slot gate**, every subsequent call for that same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`. The block's rewards are permanently discarded.

---

### Finding Description

`general_prerequisites`, the only guard called at the top of `update_rewards`, checks exactly two things:

```
// src/staking/staking.cairo:1794-1797
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

There is no `assert_caller_is_sequencer()` or role check of any kind. The spec at `docs/spec.md:1645` states "Only starkware sequencer" but this is never enforced in code. [2](#0-1) 

Inside `update_rewards`, the critical sequence is:

1. **Line 1454–1458**: Gate check — `current_block_number > last_reward_block`. Passes on the first call per block.
2. **Line 1485**: `self.last_reward_block.write(current_block_number)` — the global slot is stamped **before** the `disable_rewards` branch.
3. **Line 1487–1489**: `if disable_rewards || self.is_pre_consensus() { return; }` — exits without distributing anything. [3](#0-2) 

Because `last_reward_block` is written unconditionally at line 1485 regardless of `disable_rewards`, any caller who wins the race for block N with `disable_rewards: true` permanently consumes that block's reward slot. The sequencer's subsequent call for the same block hits the `REWARDS_ALREADY_UPDATED` assertion and reverts.

---

### Impact Explanation

Every block for which the attacker front-runs the sequencer loses its entire reward distribution permanently. There is no recovery path: `last_reward_block` is never decremented, and there is no "re-distribute missed block" mechanism. Stakers and delegators accumulate zero yield for those blocks. This matches **High — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The attacker only needs to know any valid, active staker address (fully public on-chain).
- The call is cheap (no token transfer, no approval).
- The attacker must front-run the sequencer within each block, which is feasible on Starknet where transaction ordering within a block is observable in the mempool.
- The attack can be automated to fire every block, zeroing all consensus rewards indefinitely.

---

### Recommendation

Add an explicit sequencer-only guard immediately after `general_prerequisites()` in `update_rewards`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

Alternatively, gate via a role (e.g., `REWARDS_MANAGER_ROLE`) that is assigned only to the sequencer address, consistent with how other privileged operations are protected elsewhere in the contract.

---

### Proof of Concept

```
Block N arrives.

Attacker tx (lands first in block N):
  update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
  → general_prerequisites() passes (not paused, caller != 0)
  → current_block_number (N) > last_reward_block (N-1) → passes
  → last_reward_block := N          ← slot consumed
  → disable_rewards == true → return (no rewards distributed)

Sequencer tx (lands second in block N):
  update_rewards(staker_address=<staker>, disable_rewards=false)
  → current_block_number (N) > last_reward_block (N) → FALSE
  → PANIC: REWARDS_ALREADY_UPDATED

Result: block N rewards are permanently lost for all stakers.
Repeat every block → all consensus rewards suppressed indefinitely.
``` [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
