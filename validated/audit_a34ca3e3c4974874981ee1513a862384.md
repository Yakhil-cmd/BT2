### Title
WalletConnect Offer Confirmation Displays Underestimated Royalty Total Due to BigInt Truncation - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts` uses two sequential BigInt integer-division operations that silently truncate, causing the WalletConnect offer-confirmation dialog to display a lower "Total Amount with Royalties" than the user will actually spend. A user who approves based on the displayed figure authorizes a transaction that deducts more XCH/CAT than shown.

### Finding Description

In `formatAmountWithRoyalties` (lines 354–375), when a `take_offer` or `create_offer_for_ids` WalletConnect command involves multiple NFTs with royalties, the displayed royalty total is computed with two lossy BigInt divisions:

```typescript
// Line 363 – first truncation: remainder mojos are silently dropped
const splitAmount = amount / BigInt(royaltyPercentages.length);

// Line 365 – second truncation: per-NFT royalty mojos are silently dropped
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [1](#0-0) 

The result (`totalAmount = amount + royaltyAmount`) is then formatted and stored as `amountWithRoyalties` on the spending line and surfaced in the Confirm dialog that the user must approve before the WalletConnect command executes. [2](#0-1) 

The Confirm dialog is opened at `main.tsx` line 329–352, passing `display` (which contains `amountWithRoyalties`) directly to the user-facing approval UI: [3](#0-2) 

The Chia wallet backend calculates royalties correctly (without truncation). The discrepancy between the displayed figure and the actual on-chain deduction is the vulnerability.

**Concrete example:**
- Offer: 100,000,001 mojos of XCH for 2 NFTs, each with royalty_percentage = 500 (5%)
- `splitAmount = 100_000_001n / 2n = 50_000_000n` (1 mojo dropped)
- Per-NFT royalty: `50_000_000n * 500n / 10_000n = 2_500_000n` each
- Displayed total: `100_000_001 + 2×2_500_000 = 105_000_001` mojos
- Correct total: `100_000_001 + 2×(50_000_000.5 × 0.05) = 105_000_051` mojos (backend rounds up)
- The user sees a lower figure and approves; the wallet deducts more.

The gap grows with larger amounts, more NFTs, and higher royalty percentages.

### Impact Explanation

**High.** The WalletConnect confirmation dialog is the sole security gate between a dApp request and wallet execution. Displaying an underestimated `amountWithRoyalties` causes the user to approve a transaction that deducts more XCH or CAT than they consented to. This is a direct, concrete asset-accounting impact on every `take_offer` / `create_offer_for_ids` WalletConnect call involving multiple royalty-bearing NFTs.

### Likelihood Explanation

**Medium.** Multi-NFT offers with royalties are a normal, supported use case in the Chia ecosystem. Any WalletConnect-connected dApp can craft such an offer. The truncation error is always present when `amount` is not evenly divisible by the number of NFTs, which is the common case.

### Recommendation

Replace the two BigInt truncating divisions with a calculation that preserves full precision. Compute the royalty for each NFT against the full `amount` (not a pre-split value), and accumulate before dividing once:

```typescript
// Correct: one division per NFT against the full amount, rounded correctly
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) =>
    total + (amount * BigInt(royaltyPercentage) + 9_999n) / (10_000n * BigInt(royaltyPercentages.length)),
  0n,
);
```

This matches the semantics the backend uses and eliminates the double-truncation.

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}
