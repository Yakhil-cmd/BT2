### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Block Consensus Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is specified to be callable only by the Starknet sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` every block, permanently preventing legitimate reward distribution to all stakers and their delegators.

### Finding Description
The spec explicitly states the access control for `update_rewards`:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1447–1507 contains no caller validation whatsoever. The function begins with `self.general_prerequisites()` (a pause check) and a `last_reward_block` staleness check, but never asserts that `get_caller_address()` is the authorized sequencer.

The `last_reward_block` storage variable is a **single global slot** shared across all stakers. The staleness guard is:

```
assert!(current_block_number > self.last_reward_block.read(), Error::REWARDS_ALREADY_UPDATED);
```

An attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` at every new block. This:
1. Passes the staleness check (new block).
2. Writes `last_reward_block = current_block_number` (line 1485).
3. Returns early without distributing rewards because `disable_rewards == true`.

When the legitimate sequencer subsequently attempts to call `update_rewards` for the same block, the staleness check reverts with `REWARDS_ALREADY_UPDATED`. The attacker can repeat this every block at negligible cost, permanently starving all stakers and delegators of consensus-phase rewards.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

Once consensus rewards are active (`is_pre_consensus()` returns false), every block's reward distribution is gated on `last_reward_block`. An attacker who wins the race each block (or simply calls first, since there is no sequencer-only guard) can set `last_reward_block` to the current block with `disable_rewards: true`, ensuring no rewards are ever credited to `unclaimed_rewards_own` or transferred to delegation pools. Stakers and delegators accumulate zero yield indefinitely.

### Likelihood Explanation
**High.** The function is publicly callable with no role check. The attacker needs only a valid staker address (readable from on-chain events) and enough gas to submit one transaction per block. No privileged access, no leaked key, no external dependency is required.

### Recommendation
Add a sequencer-only caller check at the top of `update_rewards`, analogous to the existing `assert_caller_is_staking_contract` pattern used elsewhere in the codebase. For example:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.assert_caller_is_sequencer(); // add this
    self.general_prerequisites();
    ...
}
```

The authorized sequencer address should be stored in contract storage and set during initialization, with a governance-controlled setter.

### Proof of Concept

1. Consensus rewards are activated (epoch ≥ `consensus_rewards_first_epoch + K`).
2. A new block `N` is produced.
3. Attacker calls `update_rewards(staker: <any_valid_staker>, disable_rewards: true)` from any EOA.
4. `last_reward_block` is written to `N`; no rewards are distributed.
5. Sequencer calls `update_rewards(staker: <staker>, disable_rewards: false)` — reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeat steps 2–5 every block. All stakers and delegators receive zero consensus rewards indefinitely.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
