Gateway Admits `AllResources` Transactions with Zero L1/L1-Data Gas Prices That Always Fail Blockifier Pre-Validation — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's resource-bounds validation checks only the **L2 gas price** against the previous block's price. It never validates the `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` fields. An unprivileged user can submit an `AllResources` V3 transaction with those two fields set to zero, pass every gateway check, enter the mempool, and then be deterministically rejected by the blockifier's `check_fee_bounds` at pre-validation time — before any nonce increment or fee charge. The mempool can therefore be filled with transactions that can never be sequenced.

---

### Finding Description

**Stateless validator — only L2 gas price is floor-checked**

`StatelessTransactionValidator::validate_resource_bounds` enforces two rules on `AllResourceBounds`:

1. `max_possible_fee(Tip::ZERO) > 0` — at least one resource bound is non-zero.
2. `l2_gas.max_price_per_unit >= config.min_gas_price` (production default: 8 000 000 000 fri). [1](#0-0) 

Neither `l1_gas.max_price_per_unit` nor `l1_data_gas.max_price_per_unit` is compared against any floor. A transaction with `l2_gas.max_price_per_unit = 8_000_000_000`, `l1_gas.max_price_per_unit = 0`, `l1_data_gas.max_price_per_unit = 0` satisfies both rules and passes.

**Stateful validator — only L2 gas price is threshold-checked**

`StatefulTransactionValidator::validate_resource_bounds` reads the previous block's STRK L2 gas price and calls `validate_tx_l2_gas_price_within_threshold`. The function's own TODO comment acknowledges the gap: [2](#0-1) [3](#0-2) 

`l1_gas` and `l1_data_gas` prices are never compared against the block's `l1_gas_price` or `l1_data_gas_price`. The transaction therefore clears both gateway layers and is forwarded to the mempool.

**Blockifier — checks all three prices unconditionally**

`AccountTransaction::check_fee_bounds` (called from `perform_pre_validation_stage`) iterates over all three resources for `AllResources` transactions and rejects any whose `max_price_per_unit < actual_gas_price`: [4](#0-3) 

Because `l1_gas_price > 0` and `l1_data_gas_price > 0` in every real block, a transaction with those fields set to zero will **always** fail here with `InsufficientResourceBounds`. The failure is a pre-validation error: the nonce is not incremented and no fee is charged, so the transaction remains eligible for re-submission and re-processing.

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An attacker can craft `AllResources` invoke transactions that:
- satisfy every gateway check (non-zero fee via `l2_gas`, L2 price above the 8 Gwei floor, valid nonce range, valid signature size, etc.),
- enter the mempool,
- are picked up by the batcher,
- are unconditionally rejected by `check_fee_bounds` without consuming a nonce or paying a fee,
- may be re-queued, causing repeated wasted batcher cycles.

Because the nonce is never bumped on pre-validation failure, a single (address, nonce) slot can be used to inject a permanently-invalid transaction that occupies a mempool slot and forces repeated blockifier rejection.

---

### Likelihood Explanation

Any unprivileged user can trigger this with a standard V3 invoke transaction. No special account, privileged key, or unusual network condition is required. The only cost is the signature and submission overhead. The production `min_gas_price_percentage = 100` and `min_gas_price = 8_000_000_000` are both enforced only on `l2_gas`, leaving `l1_gas` and `l1_data_gas` prices completely unchecked at the gateway. [5](#0-4) 

---

### Recommendation

1. **Stateful validator**: extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's `strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price`, applying the same `min_gas_price_percentage` threshold. The existing TODO comment at line 358 already flags this gap.

2. **Stateless validator**: add floor checks for `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` analogous to the existing `l2_gas` check, or document explicitly that those fields are intentionally unchecked at the stateless layer.

### Proof of Concept

```
Craft an AllResources invoke transaction with:
  l2_gas.max_price_per_unit  = 8_000_000_000   // meets stateless min_gas_price
  l2_gas.max_amount          = 1
  l1_gas.max_price_per_unit  = 0               // not checked by gateway
  l1_gas.max_amount          = 0
  l1_data_gas.max_price_per_unit = 0           // not checked by gateway
  l1_data_gas.max_amount     = 0
  nonce                      = account_nonce   // valid nonce

Gateway stateless path:
  max_possible_fee(Tip::ZERO) = 8_000_000_000 * 1 > 0  → PASS
  l2_gas.max_price_per_unit (8B) >= min_gas_price (8B)  → PASS

Gateway stateful path:
  validate_tx_l2_gas_price_within_threshold:
    tx_l2_gas_price (8B) >= threshold (prev_block_l2_gas_price * 100%)  → PASS
  l1_gas / l1_data_gas prices: not evaluated                            → PASS

Mempool: transaction admitted.

Blockifier check_fee_bounds (AllResources branch):
  L1Gas:     max_price_per_unit (0) < actual_l1_gas_price (> 0)  → MaxGasPriceTooLow
  → InsufficientResourceBounds error, pre-validation failure
  → nonce NOT incremented, fee NOT charged
  → transaction slot remains occupied in mempool
```

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
