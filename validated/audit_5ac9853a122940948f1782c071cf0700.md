### Title
Anyone Can Call `update_rewards` With `disable_rewards: true` to Permanently Block Reward Distribution Per Block - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` lacks access control. It writes to the global `last_reward_block` storage variable regardless of the `disable_rewards` flag. Any unprivileged caller can invoke it with `disable_rewards: true` to mark the current block as "already updated" without distributing any rewards, causing the legitimate sequencer's subsequent call with `disable_rewards: false` to revert with `REWARDS_ALREADY_UPDATED`. This is a direct analog to H-4: a public state-marking function that can be weaponized to invalidate legitimate operations.

### Finding Description

`IStakingRewardsManager::update_rewards` is the per-block reward distribution entry point. Its implementation in `StakingRewardsManagerImpl` (lines 1448–1507 of `src/staking/staking.cairo`) performs the following steps unconditionally:

1. Calls `self.general_prerequisites()` (pause check only).
2. Asserts `current_block_number > self.last_reward_block.read()` — the only guard against double-execution.
3. **Writes `self.last_reward_block.write(current_block_number)`** — this happens before the `disable_rewards` branch.
4. If `disable_rewards || self.is_pre_consensus()` → returns early with no rewards distributed.
5. Otherwise, calculates and distributes block rewards. [1](#0-0) 

The `last_reward_block` field is a **single global** `BlockNumber` in contract storage, not per-staker: [2](#0-1) 

The spec declares access control as "Only starkware sequencer": [3](#0-2) 

However, the implementation contains **no caller check** enforcing this. The test suite confirms this: `update_rewards` is called throughout without any `cheat_caller_address_once` to impersonate a sequencer: [4](#0-3) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at block N:
- `last_reward_block` is written to N.
- The function returns early — zero rewards distributed.
- The legitimate sequencer's call at block N with `disable_rewards: false` reverts with `REWARDS_ALREADY_UPDATED`.
- All stakers lose their block-N rewards permanently.

Repeated every block, this constitutes **permanent freezing of unclaimed yield** for all stakers and delegators. This matches the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"* (High).

### Likelihood Explanation

- No special privilege is required — any EOA or contract can call `update_rewards`.
- The attacker only needs to submit a transaction before the sequencer's reward-distribution transaction in each block.
- Gas cost is low (a single storage write + early return).
- The attack is sustainable indefinitely and requires no capital.

### Recommendation

Add an access control check at the top of `update_rewards` to restrict callers to the authorized sequencer address (or the `STARKWARE_SEQUENCER` role), consistent with the spec's stated intent. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_starkware_sequencer(); // enforce spec access control
    self.general_prerequisites();
    ...
}
```

Alternatively, move the `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` branch so that a no-op call does not consume the block's reward slot.

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`).
2. Staker `S` is registered and has non-zero balance.
3. At block N, attacker calls:
   ```
   IStakingRewardsManager(staking).update_rewards(S, disable_rewards: true)
   ```
   - Passes `REWARDS_ALREADY_UPDATED` check (new block).
   - Writes `last_reward_block = N`.
   - Returns early — no rewards minted.
4. Sequencer calls:
   ```
   IStakingRewardsManager(staking).update_rewards(S, disable_rewards: false)
   ```
   - Fails: `current_block_number (N) > last_reward_block (N)` is false → `REWARDS_ALREADY_UPDATED` panic.
5. Staker S (and all other stakers) receive zero rewards for block N.
6. Attacker repeats step 3 at every subsequent block. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
