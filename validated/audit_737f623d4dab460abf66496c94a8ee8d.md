### Title
Pool admin can set `poolAdminFeeDestination` to a blacklisted token address, permanently locking protocol fees and blocking `setPoolProtocolFee` — (`File: metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`setPoolAdminFeeDestination` accepts any non-zero address as the admin fee destination with no further validation. Because `collectFees` in `MetricOmmPool` transfers to `adminFeeDestination_` **before** transferring to `FACTORY`, a pool admin who sets a USDC-blacklisted address as the destination causes every `collectFees` call to revert. This permanently locks accrued protocol fees inside the pool and prevents the factory owner from executing `setPoolProtocolFee` on that pool.

---

### Finding Description

`setPoolAdminFeeDestination` in `MetricOmmPoolFactory` only rejects `address(0)`:

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    ...
}
``` [1](#0-0) 

Any other address — including a USDC-blacklisted one — is accepted without restriction.

`collectFees` in `MetricOmmPool` then transfers to `adminFeeDestination_` first, before the protocol share reaches `FACTORY`:

```solidity
if (totalFee0ToAdmin > 0) {
    transferToken0(adminFeeDestination_, totalFee0ToAdmin);   // ← reverts if blacklisted
}
if (totalFee1ToAdmin > 0) {
    transferToken1(adminFeeDestination_, totalFee1ToAdmin);
}
if (totalFee0ToProtocol > 0) {
    transferToken0(FACTORY, totalFee0ToProtocol);             // ← never reached
}
if (totalFee1ToProtocol > 0) {
    transferToken1(FACTORY, totalFee1ToProtocol);
}
``` [2](#0-1) 

Every factory-side function that must settle outstanding fees before changing state calls `collectFees` as its first step:

- `collectPoolFees` (callable by anyone) [3](#0-2) 
- `setPoolAdminFees` (pool admin) [4](#0-3) 
- `setPoolProtocolFee` (factory owner / protocol) [5](#0-4) 

Once the admin fee destination is set to a blacklisted address and any admin fee component (`adminSpreadFeeE6 > 0` or `adminNotionalFeeE8 > 0`) is active, all three calls revert permanently.

---

### Impact Explanation

**Protocol fees are permanently locked in the pool.** The `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` accumulators and the spread-fee surplus are never zeroed and never transferred to `FACTORY`. The factory owner has no alternative path to collect these funds.

**The factory owner loses the ability to update protocol fees for the affected pool.** `setPoolProtocolFee` — an `onlyOwner` function — calls `collectFees` unconditionally before writing the new fee config. With a blacklisted destination it always reverts, so the protocol cannot raise, lower, or zero out its own fee rate on that pool. [6](#0-5) 

This is an admin-boundary break: a semi-trusted pool admin can permanently block a factory-owner (`onlyOwner`) operation, which exceeds the pool admin's intended authority.

---

### Likelihood Explanation

The pool admin is semi-trusted and has direct, unrestricted access to `setPoolAdminFeeDestination`. The only precondition is that the pool's `adminSpreadFeeE6` or `adminNotionalFeeE8` is non-zero (which is the normal operating state for any pool that charges admin fees). The attack requires a single transaction and no special market conditions. A malicious or compromised pool admin can execute it at any time, including frontrunning a `setPoolProtocolFee` call from the factory owner.

---

### Recommendation

1. **Validate the destination before storing it.** For tokens known to implement blacklists (USDC, USDT), call the token's `isBlacklisted` method inside `setPoolAdminFeeDestination` and revert if the address is blacklisted.

2. **Decouple admin and protocol fee transfers in `collectFees`.** Transfer the protocol share to `FACTORY` independently of the admin share, so a revert on the admin leg does not block protocol fee collection. For example, catch the admin transfer failure and emit an event rather than reverting the entire call.

3. **Add a factory-owner escape hatch.** Allow the factory owner to override `poolAdminFeeDestination` for a pool in an emergency, or provide a separate `collectProtocolFeesOnly` path that bypasses the admin transfer entirely.

---

### Proof of Concept

```
1. Factory deploys a USDC/WETH pool with adminSpreadFeeE6 = 50_000 (5%).
2. Pool accumulates spread fees through normal swaps.
3. Pool admin calls:
       factory.setPoolAdminFeeDestination(pool, 0x0E6b8E34dC115a2848F585851AF23D99D09b8463)
   (a known USDC-blacklisted address on Arbitrum; only check is != address(0), passes).
4. Factory owner calls factory.setPoolProtocolFee(pool, newRate, newNotional).
   → internally calls pool.collectFees(..., 0x0E6b8E34...),
   → pool calls IERC20(USDC).safeTransfer(0x0E6b8E34..., adminAmount),
   → USDC reverts "Blacklistable: account is blacklisted",
   → entire collectFees reverts,
   → setPoolProtocolFee reverts.
5. Anyone calls factory.collectPoolFees(pool) → same revert.
6. Protocol fees remain locked in the pool indefinitely.
   notionalFeeToken0Scaled and notionalFeeToken1Scaled are never zeroed.
```

    override
    onlyOwner
    nonReentrant

    if (newProtocolNotionalFeeE8 > maxProtocolNotionalFeeE8) revert ProtocolFeeTooHigh();

    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool)
      .collectFees(

        c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8,
        c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]
      );

    uint24 aSpread = c.adminSpreadFeeE6;
    uint24 aNotional = c.adminNotionalFeeE8;
    if (aSpread > maxAdminSpreadFeeE6) {
      aSpread = maxAdminSpreadFeeE6;
      emit PoolAdminSpreadFeeUpdated(pool, aSpread);
    }
    if (aNotional > maxAdminNotionalFeeE8) {
      aNotional = maxAdminNotionalFeeE8;
      emit PoolAdminNotionalFeeUpdated(pool, aNotional);
