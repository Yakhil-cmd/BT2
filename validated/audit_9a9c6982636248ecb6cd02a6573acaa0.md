### Title
Global `last_reward_block` Lock Granularity Mismatch Enables Permanent Griefing of Consensus Reward Distribution — (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` in `staking.cairo` uses a single global `last_reward_block` value as a reentrancy/double-call guard, but the reward state it protects is per-staker. Because the lock key (block number, global) does not match the state key (staker address, per-staker), and because `update_rewards` has no access control beyond "caller is not zero", any unprivileged caller can consume the global lock for the entire contract in a given block by calling with `disable_rewards=true`. This permanently prevents the legitimate sequencer from distributing consensus rewards for that block to any staker.

---

### Finding Description

`update_rewards` is documented in the spec as callable only by the Starknet sequencer, but the implementation enforces no such restriction. The only guard is `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function uses a single global storage variable `last_reward_block` as its idempotency guard: [2](#0-1) 

This guard is written **before** the `disable_rewards` branch: [3](#0-2) 

The structural mismatch is:

| Dimension | Lock | Protected State |
|---|---|---|
| Key | `last_reward_block` (global, one value) | `staker_info.unclaimed_rewards_own` (per `staker_address`) |
| Granularity | Contract-wide | Per-staker |

Because the lock is global but the state is per-staker, a single call with **any** valid staker address and `disable_rewards=true` consumes the lock for the entire contract in that block. The sequencer's subsequent call for the intended staker fails with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

The `last_reward_block` storage field: [5](#0-4) 

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards=true)` at the start of every block:

1. Sets `last_reward_block` to the current block number.
2. Returns early — no rewards are computed or distributed.
3. The sequencer's legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`.
4. The block rewards for that block are permanently lost — `_update_rewards` is never called, `update_unclaimed_rewards_from_staking_contract` is never invoked, and the STRK/BTC rewards for that block are never minted or credited to any staker or delegator. [6](#0-5) 

Sustained over time, this permanently freezes all unclaimed yield for all stakers and delegators in the consensus rewards phase. This matches the **High** impact category: *Permanent freezing of unclaimed yield*.

---

### Likelihood Explanation

- `update_rewards` is a public function with no role check.
- The attacker only needs to know any valid, active staker address with non-zero balance — all of which are public on-chain via events and the `stakers` vector.
- Gas costs on Starknet are low enough to make per-block griefing economically viable for a motivated attacker.
- The attack requires no special privileges, no token ownership, and no coordination. [7](#0-6) 

---

### Recommendation

1. **Add access control**: Restrict `update_rewards` to the authorized sequencer address (e.g., via a `SEQUENCER_ROLE` or a stored `sequencer_address`).
2. **Fix the lock granularity**: Change `last_reward_block` from a global scalar to a per-staker map (`Map<ContractAddress, BlockNumber>`) so that consuming the lock for one staker does not block reward distribution for all others. This directly resolves the key-granularity mismatch analogous to the reported `ERC20BalanceGteEnforcer` issue.

---

### Proof of Concept

```
Block N begins.

1. Attacker observes any valid staker address S with non-zero balance.

2. Attacker calls:
   update_rewards(staker_address=S, disable_rewards=true)

   Inside the function:
   - current_block_number = N
   - last_reward_block = N-1  →  assertion passes
   - staker S is valid and active  →  assertion passes
   - last_reward_block.write(N)   ← lock consumed
   - disable_rewards=true         → early return, no rewards distributed

3. Sequencer calls:
   update_rewards(staker_address=S, disable_rewards=false)

   Inside the function:
   - current_block_number = N
   - last_reward_block = N  →  N > N is FALSE  →  REWARDS_ALREADY_UPDATED panic

4. Block N's rewards are permanently lost for all stakers and delegators.

5. Attacker repeats every block → all consensus rewards are permanently frozen.
``` [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
