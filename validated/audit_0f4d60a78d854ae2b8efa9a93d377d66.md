### Title
Missing Sequencer Access Control on `update_rewards` Allows Any Caller to Freeze Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it first in any block, either suppressing reward distribution entirely (`disable_rewards: true`) or locking the sequencer out after distributing rewards for only one chosen staker. Because `last_reward_block` is a single global variable, one call per block exhausts the slot for all stakers.

---

### Finding Description

The specification explicitly states:

> **Access control:** Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. `StakingRewardsManagerImpl::update_rewards` only verifies the contract is not paused, that the current block is newer than `last_reward_block`, and that the supplied `staker_address` is active: [2](#0-1) 

There is no `get_caller_address()` assertion anywhere in the function, and a repository-wide search for any sequencer-related identifier (`sequencer`, `CALLER_IS_NOT_SEQUENCER`, `sequencer_address`) returns zero matches across all Cairo source files.

The critical side-effect is on line 1485:

```cairo
self.last_reward_block.write(current_block_number);
``` [3](#0-2) 

`last_reward_block` is a single global storage slot. Once it is written to the current block number, the guard on line 1454–1458 causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

The interface definition confirms the function is publicly exposed with no modifier: [5](#0-4) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker can call `update_rewards(any_active_staker, disable_rewards: true)` at the start of every block. This:

1. Writes `last_reward_block = current_block`, consuming the single per-block slot.
2. Returns immediately at line 1487 without distributing any rewards (`disable_rewards || self.is_pre_consensus()` is true).
3. Prevents the legitimate sequencer from calling `update_rewards` for that block — it will always revert with `REWARDS_ALREADY_UPDATED`. [6](#0-5) 

Repeated across every block, this permanently freezes all consensus-era block rewards for every staker and every delegation pool. No funds are stolen, but all unclaimed yield is permanently withheld — matching the allowed impact "Permanent freezing of unclaimed yield or unclaimed royalties."

A secondary variant: the attacker calls with `disable_rewards: false` for a staker of their choice, distributing rewards to exactly one staker and locking out all others for that block.

---

### Likelihood Explanation

**High.** The function is part of the public ABI (`#[abi(embed_v0)]`), callable by any EOA or contract on Starknet with no preconditions beyond having an active staker address to supply. The attacker needs no stake, no privileged role, and no capital. The only cost is the gas for one transaction per block. The attack is trivially automatable.

---

### Recommendation

Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the stored sequencer address (or a dedicated role). For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    self.general_prerequisites();
    // ... rest of function
}
```

This mirrors the pattern already used in `update_current_epoch_block_rewards` and `update_unclaimed_rewards_from_staking_contract` in the Reward Supplier contract, which both assert `get_caller_address() == staking_contract`: [7](#0-6) 

---

### Proof of Concept

```
1. Deploy or use any existing active staker address S.
2. In every new block, call:
       staking.update_rewards(S, disable_rewards=true)
   from any unprivileged address.
3. last_reward_block is set to the current block number.
4. The sequencer's subsequent call to update_rewards for the same block
   reverts with REWARDS_ALREADY_UPDATED.
5. No block rewards are ever distributed; all stakers' unclaimed_rewards_own
   remain permanently at zero for the consensus reward period.
```

The `REWARDS_ALREADY_UPDATED` guard that enforces the one-call-per-block invariant is the mechanism that makes this griefing permanent — it is the same check that is supposed to protect against double-distribution, but it is exploitable because the caller restriction is absent. [4](#0-3)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
