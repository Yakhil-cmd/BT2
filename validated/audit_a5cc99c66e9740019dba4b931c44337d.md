Audit Report

## Title
Zero-Cost `UpdateSettings` Delayed-Induction Path Enables Sustained Subnet Queue Spam — (`rs/cycles_account_manager/src/cycles_account_manager.rs`, `rs/messaging/src/scheduling/valid_set_rule.rs`)

## Summary
`UpdateSettings` ingress messages with payloads ≤ 338 bytes are intentionally inducted into the subnet queue at zero upfront cost to support the canister-unfreeze use case. When the target canister has zero cycles, the deferred execution-time charge is silently dropped. An attacker who controls a cycles-drained canister can exploit this to continuously fill block slots with zero-cost messages, degrading subnet throughput for legitimate users.

## Finding Description
`ingress_induction_cost` returns `IngressInductionCost::Free` for any `UpdateSettings` call whose `arg` length is ≤ `MAX_DELAYED_INGRESS_COST_PAYLOAD_SIZE` (338 bytes): [1](#0-0) [2](#0-1) [3](#0-2) 

In `ValidSetRuleImpl::enqueue`, the `Free` branch pushes the message directly to the subnet queue with no cycles deducted and no balance check: [4](#0-3) 

The consensus-layer ingress selector also performs no action for `Free` messages: [5](#0-4) 

The deferred charge is applied at execution time in `execute_subnet_message`. If the canister has zero cycles, the error is explicitly ignored: [6](#0-5) 

The HTTP-handler controller check (`canister_manager.rs` lines 211–236) is a real guard — the sender must be a controller of the target canister: [7](#0-6) 

However, this check is trivially satisfied: canister creation is permissionless, the attacker is their own controller, and they can register multiple principal keys as controllers to bypass per-sender ingress pool quotas. The consensus layer does not re-verify controller status; it only checks cycles via `ingress_induction_cost`, which returns `Free` for this path.

**Attack path:**
1. Attacker creates a canister (permissionless) and drains it to zero cycles.
2. Attacker registers multiple controller keys on the canister.
3. Each key sends `UpdateSettings` messages with a minimal payload (e.g., only `freezing_threshold`). The HTTP handler admits them because each sender is a controller.
4. `ingress_induction_cost` returns `Free`; `enqueue` pushes to the subnet queue at zero cost.
5. At execution, `update_settings` succeeds, the deferred charge fails silently, and the attacker pays nothing.
6. Repeat across rounds using different sender keys to bypass per-sender round-robin limits.

The existing test `management_message_update_setting_is_inducted_but_not_charged` explicitly confirms zero balance is deducted at induction time: [8](#0-7) 

## Impact Explanation
The attacker can sustain a steady-state of spam messages consuming up to `MAX_INGRESS_MESSAGES_PER_BLOCK = 1000` slots per block: [9](#0-8) 

The ingress history is bounded at `INGRESS_HISTORY_MAX_MESSAGES = 600,000`: [10](#0-9) 

The queue does not grow unboundedly, but the attacker can occupy a large fraction of the per-block message budget at zero cost, starving legitimate messages and causing significant subnet throughput degradation. This matches the allowed impact: **Application/platform-level DoS or subnet availability impact not based on raw volumetric DDoS — High ($2,000–$10,000).**

## Likelihood Explanation
- Canister creation is permissionless; draining cycles to zero is trivial for the controller.
- Generating many signed ingress messages is cheap computation.
- Multiple controller keys can be registered on a single canister to bypass per-sender pool quotas.
- No privileged access, node compromise, or governance majority is required.
- The attack is repeatable every block indefinitely.

## Recommendation
1. **Charge a minimum non-zero base fee at induction time** for the delayed-cost path — one that does not depend on canister balance. This preserves the unfreeze use-case while imposing a spam cost.
2. **Alternatively**, at induction time check whether the target canister has sufficient balance to cover the deferred cost and reject if not (mirroring the `IngressInductionCost::Fee` path in `enqueue`).
3. **Remove the silent `_ignore_error` drop** or convert it to a logged metric/counter so operators can detect and alert on the condition.

## Proof of Concept
```rust
// 1. Create canister, drain cycles to zero.
// 2. Register N controller keys on the canister.
// 3. In a loop across rounds:
let payload = UpdateSettingsArgs {
    canister_id: drained_canister_id.get(),
    settings: CanisterSettingsArgsBuilder::new()
        .with_freezing_threshold(1u64)
        .build(),
    sender_canister_version: None,
}.encode(); // payload.len() << 338

// For each controller key k:
let ingress = SignedIngressBuilder::new()
    .canister_id(IC_00)
    .method_name("update_settings")
    .method_payload(payload.clone())
    .sender(controller_key_k)
    .nonce(round_number)
    .build();

// assert: ingress_induction_cost returns Free
// assert: enqueue succeeds with no balance deducted
// assert: after N rounds, drained_canister balance remains 0
// assert: block ingress slots are saturated with attacker messages
```

The existing unit test `management_message_update_setting_is_inducted_but_not_charged` already validates the free-induction path and zero-balance deduction, confirming the precondition for the attack.

#[test]
fn management_message_update_setting_is_inducted_but_not_charged() {
    let ingress_history_writer = MockIngressHistory::new();
    let metrics_registry = MetricsRegistry::new();
    let subnet_id = subnet_test_id(1);
    let valid_set_rule = ValidSetRuleImpl::new(
        Arc::new(ingress_history_writer),
        Arc::new(
            CyclesAccountManagerBuilder::new()
                .with_subnet_id(subnet_id)
                .build(),
        ),
        &metrics_registry,
        no_op_logger(),
    );

    let mut state = ReplicatedStateBuilder::new().build();
    let canister_id = canister_test_id(0);
    let canister = get_running_canister(canister_id);
    let balance_before = canister.system_state.balance();
    state.put_canister_state(canister);

    let payload = UpdateSettingsArgs {
        canister_id: canister_id.get(),
        settings: CanisterSettingsArgsBuilder::new()
            .with_freezing_threshold(1 << 20)
            .build(),
        sender_canister_version: None,
    }
    .encode();
    let ingress = SignedIngressBuilder::new()
        .canister_id(IC_00)
        .method_name("update_settings")
        .method_payload(payload)
        .build();
    assert!(valid_set_rule.enqueue(&mut state, ingress).is_ok());

    let balance_after = state
        .canister_state(&canister_id)
        .unwrap()
        .system_state
        .balance();

    assert_eq!(balance_after, balance_before);
}
