### Title
Gateway Skips Signature Verification for Invoke Transactions with Nonce=1 on Undeployed Accounts, Allowing Unauthorized Transaction Admission — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function in the gateway stateful validator bypasses the `__validate__` entry point (and therefore all signature verification) for any Invoke transaction with `nonce=1` when the sender account has a transaction in the mempool or a recent block. An unprivileged attacker who observes a victim's `deploy_account` in the mempool can submit an Invoke with an arbitrary invalid signature for the victim's address, have it admitted to the mempool without any signature check, block the victim's legitimate first Invoke, and cause the victim's account to be charged fees for a reverted transaction the victim never authorized.

---

### Finding Description

**Root cause — `skip_stateful_validations`** [1](#0-0) 

The function returns `true` (skip) when all three conditions hold:
1. The transaction is an `Invoke`.
2. `tx.nonce() == Nonce(Felt::ONE)`.
3. `account_nonce == Nonce(Felt::ZERO)` (account not yet deployed).
4. `account_tx_in_pool_or_recent_block(sender)` returns `true`.

The comment states the check is "sufficient" because the account "either has a deploy_account transaction or transactions with future nonces that passed validations." This reasoning is incorrect: the check only verifies that *some* transaction from that address exists in the pool — it does not verify that the *current* Invoke was submitted by the account owner.

**How the skip propagates to disable signature verification**

When `skip_validate = true`, `run_validate_entry_point` sets `validate: !skip_validate = false`: [2](#0-1) 

This flag is forwarded into `StatefulValidator::perform_validations`, which short-circuits before calling `__validate__`: [3](#0-2) 

`perform_pre_validation_stage` is still called (nonce increment + fee-balance check), but the `__validate__` entry point — the only place the account's signature is verified — is never reached. [4](#0-3) 

**Fee balance check does not substitute for signature verification**

`verify_can_pay_committed_bounds` only confirms the account has enough tokens to cover the declared resource bounds. It says nothing about whether the transaction was authorized by the account owner. [5](#0-4) 

**Attack path**

1. Victim pre-funds address `A` and submits `deploy_account` (nonce 0). `A` is now visible in the mempool.
2. Attacker calls `account_tx_in_pool_or_recent_block(A)` → `true`.
3. Attacker submits `Invoke` for `A`, nonce=1, with a garbage signature.
4. Gateway stateful path:
   - `validate_nonce`: nonce 1 ≥ account_nonce 0 ✓
   - `validate_resource_bounds`: passes if attacker sets a plausible L2 gas price ✓
   - `validate_by_mempool`: no duplicate nonce yet ✓
   - `skip_stateful_validations` → `true` → `validate: false` → **`__validate__` never called** ✓
   - `verify_can_pay_committed_bounds`: victim pre-funded `A` ✓
   - **Attacker's Invoke admitted to mempool.**
5. Victim submits their legitimate Invoke (nonce=1) → rejected: `DuplicateNonce`.
6. Block execution (`new_for_sequencing` sets `validate: true`):
   - `deploy_account` executes; `A` is now deployed.
   - Attacker's Invoke executes; `__validate__` is called; invalid signature → **revert**.
   - Nonce of `A` is incremented (1 → 2); fee is charged from `A`.
7. Victim's account is deployed, their first Invoke is permanently blocked at nonce 1, and they have been charged fees for a transaction they never signed. [6](#0-5) 

---

### Impact Explanation

An unauthorized Invoke transaction — carrying an invalid signature from a non-owner — is accepted through the gateway's account validation logic without any signature check. This satisfies:

- **High: Mempool/gateway/RPC admission accepts invalid transactions before sequencing** — the transaction is admitted to the mempool despite having an invalid signature.
- **Critical: Invalid or unauthorized Starknet transaction accepted through account validation, signature logic** — the signature invariant ("every admitted transaction must carry a valid `__validate__` result") is broken at the gateway boundary.

Concrete harms to the victim:
- Their legitimate nonce-1 Invoke is permanently blocked (duplicate nonce).
- Their account balance is debited for a reverted transaction they never authorized.
- They must resubmit at nonce 2, paying additional fees.

---

### Likelihood Explanation

- No special privileges are required; any observer of the public mempool can execute this attack.
- The attacker only needs to detect a `deploy_account` transaction in the mempool and submit a malicious Invoke before the victim's legitimate Invoke is processed.
- The deploy_account + invoke UX pattern is explicitly documented and encouraged, making victims predictable.
- The race window is the time between the victim's `deploy_account` appearing in the mempool and their `invoke` being processed — typically milliseconds to seconds, but sufficient for an automated attacker.

---

### Recommendation

**Short term**: Do not use `account_tx_in_pool_or_recent_block` as a complete substitute for signature verification. At minimum, perform a stateless signature-format check (e.g., verify the signature is well-formed and matches the declared account class's expected format) even when skipping the full `__validate__` execution.

**Long term**: Redesign the deploy-account + invoke UX skip so that the gateway either:
(a) Requires both transactions to arrive in the same atomic submission, binding them together before admission; or
(b) Defers the Invoke to a pending queue and only admits it after the `deploy_account` is committed and `__validate__` can be run against the deployed account.

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}
