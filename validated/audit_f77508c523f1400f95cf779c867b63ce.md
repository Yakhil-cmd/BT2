Audit Report

## Title
State Mutation Before Inter-Canister Ledger Call Enables Permanent Neuron Lock and Maturity Loss - (File: `rs/nns/governance/src/governance/disburse_maturity.rs`)

## Summary
In `try_finalize_maturity_disbursement`, the maturity disbursement entry is popped from the neuron's queue (a committed state mutation) before the inter-canister ledger mint call. If the ledger call fails and the subsequent reversal via `with_neuron_mut` also fails, `neuron_lock.retain()` is called, permanently retaining the neuron lock and leaving the user's maturity irrecoverably lost. The TLA+ formal model at `rs/nns/governance/tla/Disburse_Maturity_Timer.tla` explicitly models the reversal failure as a reachable non-deterministic outcome via a `skip` branch.

## Finding Description
**Root cause**: `try_finalize_maturity_disbursement` mutates neuron state before the inter-canister call, violating the checks-effects-interactions pattern.

**Exact code path**:

1. **Lines 615–623** (`disburse_maturity.rs`): `neuron.pop_maturity_disbursement_in_progress()` is called inside `governance.with_borrow_mut(...)` — a committed state mutation — before any inter-canister call. The disbursement entry is removed from the queue.

2. **Lines 642–644**: `mint_icp_with_ledger(...).await` is called. The ledger can return a reject response during subnet upgrades, ledger maintenance, or transient network partitions.

3. **Lines 650–668** (failure path): On ledger failure, the code attempts reversal via `governance.with_neuron_mut(&neuron_id, |neuron| { neuron.push_front_maturity_disbursement_in_progress(...) })`. If this returns `Err` (e.g., `NeuronStoreError::InvalidData` from a stable memory serialization failure, or `NeuronNotFound` from an index inconsistency), the `else` branch is not taken and execution falls through to `neuron_lock.retain()` at line 668.

4. **`neuron_lock.retain()`** (`neuron_lock.rs` lines 43–67): Sets `self.retain = true`. The `Drop` implementation returns early without calling `unlock_neuron`, so `in_flight_commands` retains the entry for this neuron ID permanently.

**Why existing checks fail**: The neuron lock prevents concurrent deletion but does not prevent `with_neuron_mut` from failing due to stable memory errors (`NeuronStoreError::InvalidData`, `NeuronIdModified`, `SubaccountModified`). The `NeuronStoreError` enum explicitly includes `CorruptedNeuronIndexes` and `InvalidData` variants that can be triggered by stable memory inconsistencies independent of the lock.

**TLA+ confirmation**: `Disburse_Maturity_Timer.tla` lines 62–68 use PlusCal `either/or` with a `skip` branch on ledger failure, explicitly modeling the case where the reversal fails and the lock is not released. This is not a modeling artifact — it directly corresponds to the `Err` branch at line 657 of the Rust code.

**Contrast with correct SNS pattern**: `rs/sns/governance/src/governance.rs` lines 5047–5069 calls `neuron.disburse_maturity_in_progress.remove(0)` only inside the `Ok(block_index)` branch after the `await`, eliminating the reversal logic and the permanent-lock failure path entirely.

## Impact Explanation
**Permanent neuron lock (DoS)**: The `in_flight_commands` entry for the neuron is never cleared. All subsequent `manage_neuron` operations (`disburse`, `split`, `merge`, `stake_maturity`, `disburse_maturity`) return `LedgerUpdateOngoing`. The neuron is inaccessible without a canister upgrade to manually clear the lock.

**Maturity loss**: Maturity was already deducted from `neuron.maturity_e8s_equivalent` when `initiate_maturity_disbursement` was called. The popped disbursement entry is the only record of the pending ICP mint. If it cannot be restored, the user's maturity is permanently gone and no ICP is ever minted.

This matches the allowed impact: **High — Significant NNS governance security impact with concrete user harm** (permanent loss of neuron control and maturity funds) and **High — Application-level DoS on a neuron** (permanent lock without canister upgrade).

## Likelihood Explanation
- **Ledger call failure**: Realistic during ICP subnet upgrades, ledger canister maintenance windows, or transient message routing failures. Any NNS neuron owner whose disbursement timer fires during such a window is exposed.
- **Reversal failure**: `with_neuron_mut` (neuron_store.rs lines 720–731) calls `load_neuron_all_sections` and `update_neuron`, both of which return `Result`. `NeuronStoreError::InvalidData` can be triggered by stable memory serialization failures. The neuron lock prevents deletion but does not prevent deserialization errors.
- **TLA+ acknowledgment**: The DFINITY formal verification team explicitly included the `skip` branch (reversal failure, lock not released) as a reachable non-deterministic outcome in the formal model, confirming this is considered a real execution path.
- **Trigger**: Unprivileged NNS neuron owner. No special privileges required beyond owning a neuron with maturity and waiting 7 days for the disbursement timer.

## Recommendation
Apply the checks-effects-interactions pattern: do not pop the disbursement from the queue before the ledger call. Keep the disbursement in the queue during the ledger call and only remove it after the ledger call succeeds. This is exactly the pattern used in the SNS governance analog (`rs/sns/governance/src/governance.rs` lines 5047–5069).

Concretely, restructure `try_finalize_maturity_disbursement` so that:
1. Lock is acquired (Step 1 — unchanged)
2. Ledger mint call is made (Step 2 — moved before state mutation, no pop yet)
3. Only on `Ok`: pop the disbursement from the queue (Step 3 — moved after the `await`)

This eliminates the reversal logic entirely, removes the permanent-lock failure path, and aligns NNS governance with the SNS governance implementation.

## Proof of Concept
**Minimal integration test plan (PocketIC or local replica)**:

1. Create an NNS neuron with maturity. Call `manage_neuron` → `disburse_maturity`. Verify `MaturityDisbursement` entry is enqueued and maturity is deducted.
2. Advance time by 7+ days so the disbursement timer fires.
3. Configure the mock ledger to return a reject response for the mint call.
4. Inject a fault into the neuron store so that `with_neuron_mut` returns `Err` on the next call for this neuron ID (e.g., by corrupting the stable memory index or using a test hook).
5. Trigger `finalize_maturity_disbursement`.
6. Assert: `in_flight_commands` still contains the neuron ID (lock retained). Assert: `maturity_disbursements_in_progress` is empty (disbursement not restored). Assert: subsequent `manage_neuron` calls return `LedgerUpdateOngoing`.

**Simpler unit test**: Mock `with_neuron_mut` to return `Err` on the second call (reversal attempt) while the first call (pop) succeeds. Verify `neuron_lock.retain()` is called and the lock is permanently held.
