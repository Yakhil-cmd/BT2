### Title
Staker Can Front-Run Delegator's Pool Entry by Increasing Commission Within Active Commitment Window — (File: `src/staking/staking.cairo`)

---

### Summary

`enter_delegation_pool` and `switch_delegation_pool` in the Pool contract accept no `max_commission` parameter. When a staker holds an active `commission_commitment`, they can increase their commission up to `max_commission` at any time — including in the same block as a delegator's entry transaction. The delegator has no slippage protection and may enter the pool at a far higher commission rate than they observed.

---

### Finding Description

**Root cause — `update_commission` allows immediate commission increases under an active commitment:** [1](#0-0) 

When `commission_commitment` is active, the only constraints are `commission <= max_commission` and `commission != old_commission`. There is no lower-bound check, no delay, and no epoch-gating. The new commission is written to storage immediately: [2](#0-1) 

The developers themselves note this gap: [3](#0-2) 

**Victim entry points — no `max_commission` guard:**

`enter_delegation_pool` accepts only `reward_address` and `amount`: [4](#0-3) 

`switch_delegation_pool` accepts only `to_staker`, `to_pool`, and `amount`: [5](#0-4) 

Neither function checks the current commission or accepts a caller-supplied maximum. The commission is read dynamically from the staking contract at reward-distribution time, not at entry time.

**Commission is read dynamically at reward time:** [6](#0-5) 

---

### Impact Explanation

A delegator who observes a staker at 5% commission and submits `enter_delegation_pool` can have their transaction front-run by the staker calling `set_commission(max_commission)` (e.g. 50%). The delegator enters the pool at 50% commission. All future rewards accrued until the delegator exits are split at the higher rate, with the excess going to the staker. This constitutes **theft of unclaimed yield** from the delegator.

The exit wait window (default 1 week, up to 12 weeks) means the delegator cannot immediately escape; they continue to lose yield during the entire exit window.

---

### Likelihood Explanation

Requires a staker to have an active `commission_commitment` with `max_commission` materially above the current commission. A staker sets this commitment voluntarily, so a malicious staker can pre-arm the attack. On Starknet the sequencer is centralised, but a staker can still observe the public mempool and submit a `set_commission` transaction before the delegator's transaction is included. Alternatively, the staker can raise commission at any time after a delegator enters, capturing yield for the full exit window. The precondition (active commitment) is a normal, documented protocol feature, making this a realistic attack path.

---

### Recommendation

Add a `max_commission` parameter to `enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool`. Revert if the pool's current commission exceeds the caller-supplied maximum at the time of execution:

```cairo
fn enter_delegation_pool(
    ref self: ContractState,
    reward_address: ContractAddress,
    amount: Amount,
    max_commission: Commission,   // <-- add
) {
    let current_commission = self.get_commission_from_staking_contract();
    assert!(current_commission <= max_commission, "commission exceeds max");
    // ... rest of logic
}
```

This mirrors the standard AMM slippage-protection pattern and ensures a delegator's transaction reverts rather than silently executing at an unfavourable commission.

---

### Proof of Concept

1. Staker calls `set_commission_commitment(max_commission: 5000, expiration_epoch: current + 10)` while current commission is 100 (1%).
2. Delegator observes 1% commission and submits `enter_delegation_pool(reward_address, 1_000_000_STRK)`.
3. Staker observes the pending transaction and submits `set_commission(5000)` (50%) with higher priority.
4. Sequencer includes the staker's `set_commission` first; commission is now 50%.
5. Delegator's `enter_delegation_pool` executes — no commission check, succeeds.
6. For every reward epoch until the delegator exits (minimum 1 week exit window), 50% of the delegator's share is taken as commission instead of the 1% the delegator expected.
7. On a 1M STRK delegation earning 10% APY, the delegator loses ~(50%-1%) × 10% × 1M STRK / 52 ≈ 942 STRK per week to the staker, with no recourse.

        fn enter_delegation_pool(
            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.
            self
                .pool_member_info
                .write(pool_member, VInternalPoolMemberInfoTrait::new_latest(:reward_address));

            // Emit events.
            self
                .emit(
                    Events::NewPoolMember { pool_member, staker_address, reward_address, amount },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake: Zero::zero(), new_delegated_stake: amount,
                    },
                );
        }

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }

        fn get_commission_from_staking_contract(self: @ContractState) -> Commission {
            if self.staker_removed.read() {
                return Zero::zero();
            }
            let staking_dispatcher = IStakingDispatcher {
                contract_address: self.staking_pool_dispatcher.contract_address.read(),
            };
            // The staker must have commission since it has a pool (this contract). So unwrap is
            // safe.
            staking_dispatcher
                .staker_pool_info(staker_address: self.staker_address.read())
                .commission
                .unwrap()
        }
