### Title
Weak CREATE2 Salt in `MetricOmmPoolFactory::createPool` Enables DoS and Hijacking of Pool Admin Role - (File: `metric-core/contracts/MetricOmmPoolDeployer.sol`)

### Summary

`MetricOmmPoolFactory::createPool` is permissionless and uses only the caller-supplied `params.salt` as the CREATE2 salt, without committing any other pool parameters into it. An attacker can front-run any legitimate `createPool` transaction with the same `salt` but attacker-controlled `admin`, `adminFeeDestination`, `priceProviderTimelock`, fees, and oracle, either DoS-ing the legitimate creator or hijacking the pool admin role.

### Finding Description

In `MetricOmmPoolDeployer.deploy`, the pool is deployed via CREATE2 using only `params.salt` as the salt:

```solidity
pool = address(
  new MetricOmmPool{salt: params.salt}(...)
);
``` [1](#0-0) 

The pool address is therefore determined solely by `(deployer_address, params.salt, bytecode_hash)`. None of the governance-critical parameters — `admin`, `adminFeeDestination`, `priceProvider`, `priceProviderTimelock`, `adminSpreadFeeE6`, `adminNotionalFeeE8`, `extensions`, `initialAmount0PerShareE18`, `initialAmount1PerShareE18`, `minimalMintableLiquidity`, `curBinDistFromProvidedPriceE6`, `nonNegativeBinDataArray`, `negativeBinDataArray` — are committed into the salt. [2](#0-1) 

`createPool` is explicitly permissionless once `poolDeployer` is set: [3](#0-2) 

After a successful `createPool`, the factory records the attacker-supplied `admin`, `adminFeeDestination`, and `priceProviderTimelock` as the authoritative governance state for that pool: [4](#0-3) 

Attack scenario:
1. Alice submits `createPool` with `salt = S`, `admin = alice_multisig`, `priceProviderTimelock = 7 days`.
2. Bob front-runs with the same `salt = S` but `admin = bob`, `adminFeeDestination = bob`, `priceProviderTimelock = 0`, `adminSpreadFeeE6 = maxAdminSpreadFeeE6`.
3. Bob's transaction succeeds; the pool is deployed at the deterministic address.
4. Alice's transaction reverts because CREATE2 at that address already exists.

Two outcomes:
- **DoS**: Alice's transaction fails. Bob's pool is never used by Alice. An attacker can repeat this for every `createPool` call.
- **Admin hijack**: If Alice computed the pool address off-chain and already wired integrations to it, she proceeds to use the pool — but `poolAdmin[pool]` is Bob, `poolAdminFeeDestination[pool]` is Bob, and `priceProviderTimelock[pool]` is 0. Bob can immediately rotate the oracle, drain admin fees to his address, pause/unpause the pool, and set maximum admin fees.

### Impact Explanation

If the admin hijack path is taken, Bob controls all privileged pool operations gated by `onlyPoolAdmin`:
- `setPoolAdminFees` — can raise admin fees to the maximum cap, extracting value from every swap and liquidity operation.
- `setPoolAdminFeeDestination` — redirects admin fee sweeps to Bob's address.
- `proposePoolPriceProvider` / `executePoolPriceProviderUpdate` — with `priceProviderTimelock = 0`, Bob can immediately rotate the oracle to a malicious price provider, enabling bad-price execution on every swap.
- `pausePool` / `unpausePool` — Bob can freeze the pool at will. [5](#0-4) 

This constitutes a direct loss of admin fee revenue, broken core pool functionality, and potential bad-price execution — all matching the allowed impact gate.

### Likelihood Explanation

This attack can be performed by any unprivileged address whenever a new pool is deployed. Front-running is straightforward on any public mempool chain (Ethereum, Base). The attacker only needs to observe a pending `createPool` transaction and replay it with the same `salt` and modified governance parameters.

### Recommendation

Include all pool parameters in the CREATE2 salt so that any parameter change produces a different pool address. Replace the raw `params.salt` with a hash that commits all governance-critical fields:

```solidity
bytes32 derivedSalt = keccak256(abi.encode(
    params.salt,
    params.admin,
    params.adminFeeDestination,
    params.priceProvider,
    params.priceProviderTimelock,
    params.adminSpreadFeeE6,
    params.adminNotionalFeeE8,
    params.token0,
    params.token1,
    params.extensions,
    params.extensionOrders,
    params.initialAmount0PerShareE18,
    params.initialAmount1PerShareE18,
    params.minimalMintableLiquidity,
    params.curBinDistFromProvidedPriceE6,
    params.nonNegativeBinDataArray,
    params.negativeBinDataArray
));
```

Alternatively, restrict `createPool` to a trusted deployer address (e.g., `onlyOwner`).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;
import {Test} from "forge-std/Test.sol";
import {MetricOmmPoolFactory} from "contracts/MetricOmmPoolFactory.sol";
import {MetricOmmPoolDeployer} from "contracts/MetricOmmPoolDeployer.sol";
import {PoolParameters} from "contracts/types/FactoryOperation.sol";
import {ExtensionOrders} from "contracts/types/PoolExtensionsConfig.sol";

contract PoolDosTest is Test {
    // ... (setup: deploy factory, deployer, mock oracle, mock tokens)

    function testPoc_poolDoS() public {
        bytes32 salt = keccak256("POOL_SALT");

        // Alice's intended params
        PoolParameters memory params = _defaultPoolParams();
        params.salt = salt;
        params.admin = alice;
        params.adminFeeDestination = alice;
        params.priceProviderTimelock = 7 days;
        params.adminSpreadFeeE6 = 1000;

        // Bob front-runs with same salt, different admin/fees/timelock
        PoolParameters memory bobParams = params;
        bobParams.admin = bob;
        bobParams.adminFeeDestination = bob;
        bobParams.priceProviderTimelock = 0;          // immediate oracle rotation
        bobParams.adminSpreadFeeE6 = maxAdminFee;     // max fee extraction

        vm.prank(bob);
        address pool = factory.createPool(bobParams);

        // Bob controls the pool admin
        assertEq(factory.poolAdmin(pool), bob);
        assertEq(factory.poolAdminFeeDestination(pool), bob);
        assertEq(factory.priceProviderTimelock(pool), 0);

        // Alice's transaction reverts
        vm.prank(alice);
        vm.expectRevert(); // CREATE2 collision
        factory.createPool(params);
    }
}

  function proposePoolPriceProvider(address pool, address newPriceProvider)
    external
    override
    nonReentrant
    onlyPoolAdmin(pool)
  {
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    uint256 timelock = priceProviderTimelock[pool];
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, newPriceProvider);

    address mutableProvider = PoolStateLibrary._slot3(pool);
    address current = mutableProvider != address(0) ? mutableProvider : p.immutablePriceProvider;
    uint256 executeAfter = block.timestamp + timelock;
    pendingPriceProvider[pool] = newPriceProvider;
    pendingPriceProviderExecuteAfter[pool] = executeAfter;
    emit PoolPriceProviderChangeProposed(pool, current, newPriceProvider, executeAfter);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function executePoolPriceProviderUpdate(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPriceProvider[pool];
    if (pending == address(0)) revert NoPriceProviderChangeProposed();
    uint256 execAfter = pendingPriceProviderExecuteAfter[pool];
    // forge-lint: disable-next-line(block-timestamp) -- timelock enforcement legitimately relies on `block.timestamp`.
    if (block.timestamp < execAfter) revert PriceProviderTimelockNotElapsed(execAfter, block.timestamp);
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, pending);
    IMetricOmmPoolFactoryActions(pool).setPriceProvider(pending);
    delete pendingPriceProvider[pool];
    delete pendingPriceProviderExecuteAfter[pool];
    emit PoolPriceProviderUpdated(pool, pending);
  }
