### Title
NFT Seller Displayed Full Pre-Royalty Amount as Net Proceeds Due to Commented-Out Royalty Subtraction — (`packages/gui/src/components/offers/utils.ts`, `NFTOfferViewer.tsx`)

### Summary

In `calculateNFTRoyalties`, the line that computes `nftSellerNetAmount` as `amount - royalties - makerFee` is commented out. As a result, `nftSellerNetAmount` is always set to the full `amount`. This value propagates to `overrideNFTSellerAmount` (converted to mojos) and is displayed in `OfferSummaryTokenRow` as "You will receive", and separately rendered as "Net Proceeds" — both showing the full pre-royalty amount. The NFT seller is misled into believing they will receive the full offered amount, when on-chain they receive `amount - royalties`.

---

### Finding Description

**Root cause — `utils.ts` lines 314–317:**

```typescript
const nftSellerNetAmount: number = amount;
// : parseFloat(
//     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
//   );
``` [1](#0-0) 

The royalty subtraction is commented out. `nftSellerNetAmount` always equals `amount` regardless of `royaltyPercentage`.

**Propagation — `NFTOfferViewer.tsx` lines 397–402:**

```typescript
const overrideNFTSellerAmount =
  exchangeType === NFTOfferExchangeType.TokenForNFT
    ? assetType === OfferAsset.CHIA
      ? chiaToMojo(nftSaleInfo?.nftSellerNetAmount ?? 0)
      : catToMojo(nftSaleInfo?.nftSellerNetAmount ?? 0)
    : undefined;
``` [2](#0-1) 

For `TokenForNFT` offers, `overrideNFTSellerAmount = chiaToMojo(amount)` — the full pre-royalty amount in mojos.

**Display — `OfferSummaryRow.tsx` line 146:**

```typescript
const amount = overrideNFTSellerAmount ?? originalAmount;
``` [3](#0-2) 

`OfferSummaryTokenRow` replaces the raw offer amount with `overrideNFTSellerAmount`, then formats and displays it. Since `overrideNFTSellerAmount` equals the full amount (not net), the "You will receive" row shows the full pre-royalty figure.

**Second misleading display — `NFTOfferViewer.tsx` lines 594–597:**

```tsx
<Typography variant="h5" fontWeight="bold">
  <FormatLargeNumber value={new BigNumber(nftSaleInfo?.nftSellerNetAmount ?? 0)} /> {displayName}
</Typography>
``` [4](#0-3) 

The "Net Proceeds" label explicitly claims to show the net amount after royalties, but renders `nftSellerNetAmount` = full `amount`. Both the "You will receive" row and the "Net Proceeds" section are wrong simultaneously.

The "Creator Fee" line does correctly display the royalty amount: [5](#0-4) 

However, the explicit "Net Proceeds" label overrides any inference a user might make from seeing the Creator Fee separately.

---

### Impact Explanation

An NFT seller viewing a `TokenForNFT` offer sees:

| UI Label | Displayed Value | Actual On-Chain Value |
|---|---|---|
| "You will receive" | `amount` XCH | `amount - royalties` XCH |
| "Net Proceeds" | `amount` XCH | `amount - royalties` XCH |
| "Creator Fee (X%)" | `royalties` XCH | correct |

With a 50% royalty, the seller sees "Net Proceeds: 10 XCH" but receives 5 XCH on-chain. The "Net Proceeds" label is specifically designed to communicate the post-royalty figure; its incorrectness directly causes the seller to accept under false pretense, resulting in concrete XCH loss.

This fits the High impact category: display of the wrong amount causes a user to approve an offer with an incorrect understanding of what they will receive.

---

### Likelihood Explanation

Any NFT with a non-zero `royaltyPercentage` triggers this bug when a `TokenForNFT` offer is viewed. No special attacker action is required — the bug fires for every such offer. NFTs with royalties are common in the Chia ecosystem. The victim only needs to open an imported offer file or a shared offer link.

---

### Recommendation

Uncomment and restore the royalty subtraction in `calculateNFTRoyalties`:

```typescript
const nftSellerNetAmount: number = parseFloat(
  (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
);
``` [1](#0-0) 

Add a unit test asserting that for a 50% royalty on a 10 XCH offer, `nftSellerNetAmount` equals 5 (not 10), and `overrideNFTSellerAmount` equals `chiaToMojo(5)`.

---

### Proof of Concept

1. Mint an NFT with `royaltyPercentage = 5000` (50%).
2. Create a `TokenForNFT` offer offering 10 XCH for that NFT.
3. Open the offer file in `NFTOfferViewer` (imported mode).
4. Observe: "You will receive" shows **10 XCH**, "Net Proceeds" shows **10 XCH**.
5. Accept the offer.
6. On-chain result: NFT seller receives **5 XCH**; 5 XCH goes to the royalty address.

Programmatic assertion:

```typescript
const result = calculateNFTRoyalties(10, 0, 50, NFTOfferExchangeType.TokenForNFT);
// Bug: result.nftSellerNetAmount === 10
// Expected: result.nftSellerNetAmount === 5
assert.strictEqual(result.nftSellerNetAmount, 5); // FAILS with current code
```
