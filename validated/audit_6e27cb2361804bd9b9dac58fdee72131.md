### Title
Unrestricted `update_rewards` Allows Any Caller to Permanently Block Consensus Reward Distribution for All Stakers - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the `Staking` contract is specified to be callable only by the Starkware sequencer, but contains no caller access control. Any unprivileged address can call it with `disable_rewards: true` to advance the global `last_reward_block` without distributing rewards, permanently preventing the legitimate sequencer from distributing consensus block rewards to any staker for that block.

### Finding Description

The spec explicitly restricts `update_rewards` to the Starkware sequencer:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` contains no such check. The function only validates that the contract is unpaused, that `current_block_number > last_reward_block`, and that the supplied staker is active: [2](#0-1) 

The critical state mutation is the global `last_reward_block` write, which occurs **before** the `disable_rewards` early-return branch: [3](#0-2) 

`last_reward_block` is a single global `BlockNumber` field (not per-staker): [4](#0-3) 

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` at the start of every block. This:
1. Passes all checks (staker is valid, block is new)
2. Writes `last_reward_block = current_block_number`
3. Returns early — no rewards distributed

When the legitimate sequencer then calls `update_rewards` for any staker in the same block, the assertion `current_block_number > last_reward_block` fails with `REWARDS_ALREADY_UPDATED`, and no staker receives consensus block rewards for that block.

The interface definition confirms no caller restriction exists: [5](#0-4) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield / Theft of unclaimed yield.**

By calling `update_rewards(valid_staker, disable_rewards: true)` once per block, an attacker permanently prevents the sequencer from distributing consensus block rewards to **all** stakers for every blocked block. Stakers and delegators lose all consensus-era block rewards for the duration of the attack. The rewards are never minted/distributed — they are permanently lost to the stakers.

### Likelihood Explanation

**High.** The entry point is fully public with no authentication. Valid staker addresses are observable on-chain (emitted in `NewStaker` events and stored in the public `stakers` vector). The attacker pays only gas per block. The attack is cheap, deterministic, and requires no special privileges or capital.

### Recommendation

Add a caller check to `update_rewards` analogous to the existing `assert_caller_is_attestation_contract` pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
}
```

Store the authorized sequencer address in contract storage (set at construction or by governance) and assert `get_caller_address() == sequencer_address`. [6](#0-5) 

### Proof of Concept

1. Deploy the system with consensus rewards active.
2. Staker stakes and waits K epochs for balance to become effective.
3. Attacker (any EOA) calls `update_rewards(staker_address, disable_rewards: true)` at block N.
   - `last_reward_block` is written to N; no rewards distributed.
4. Sequencer attempts `update_rewards(staker_address, disable_rewards: false)` at block N.
   - Reverts with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. Staker calls `claim_rewards` — receives zero rewards for block N.
6. Repeat step 3 every block: staker permanently receives zero consensus rewards.

This is confirmed by the existing test structure which uses `cheat_caller_address` to simulate the sequencer — demonstrating the function has no on-chain caller enforcement: [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

#[feature("safe_dispatcher")]
fn test_update_rewards_assertions_before_consensus() {

    general_contract_system_deployment(ref :cfg);
    let staking_contract = cfg.test_info.staking_contract;

    let staking_rewards_dispatcher = IStakingRewardsManagerDispatcher {

        contract_address: staking_contract,
    };
    let staker_address = cfg.test_info.staker_address;

    // Catch STAKER_NOT_EXISTS.
    let result = staking_rewards_safe_dispatcher
        .update_rewards(:staker_address, disable_rewards: true);

    let result = staking_rewards_safe_dispatcher
        .update_rewards(:staker_address, disable_rewards: false);
    assert_panic_with_error(:result, expected_error: Error::STAKER_NOT_EXISTS.describe());
