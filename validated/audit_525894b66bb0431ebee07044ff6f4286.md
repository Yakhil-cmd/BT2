### Title
Blocklist Enforcement Skipped During Block Execution Allows Blocked Addresses to Bypass Sanctions - (File: app/block_address.go)

### Summary
`BlockAddressesDecorator.AnteHandle` wraps every blocklist check inside `if ctx.IsCheckTx()`, meaning the entire enforcement logic is skipped during `DeliverTx` (actual block execution). A blocked address whose transaction is included directly in a block by a validator â€” bypassing the public mempool â€” will have its transaction executed without any blocklist check.

### Finding Description
In `app/block_address.go`, the `AnteHandle` function is the sole enforcement point for the Cronos blocklist:

```go
func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
    if ctx.IsCheckTx() {
        // signer check
        // EVM destination check
        // EIP-7702 authorization list check
        // MsgStoreBlockList admin check
    }
    return next(ctx, tx, simulate)
}
``` [1](#0-0) 

All four enforcement branches â€” signer blocklist, EVM destination blocklist, EIP-7702 authorization list, and `MsgStoreBlockList` admin guard â€” are gated behind `ctx.IsCheckTx()`. In the Cosmos SDK ante handler pipeline, `ctx.IsCheckTx()` returns `false` during `DeliverTx` (block execution). The final `return next(ctx, tx, simulate)` is unconditional, so every transaction passes through without any blocklist check during actual execution. [2](#0-1) 

The blocklist itself is stored and managed via `MsgStoreBlockList` and `GetParams`: [3](#0-2) 

The `BlockAddressesDecorator` is the only place this blocklist is consulted for transaction filtering. There is no equivalent check in any keeper message handler or EVM hook.

**Attacker path:**
1. Attacker's address is added to the Cronos blocklist by the admin via `MsgStoreBlockList`.
2. Attacker submits a signed transaction to a validator out-of-band (directly to the validator's node, bypassing the public mempool, or the attacker is themselves a validator).
3. The validator proposes a block containing the transaction.
4. During `DeliverTx`, `BlockAddressesDecorator.AnteHandle` is called with `ctx.IsCheckTx() == false`; the entire `if` body is skipped.
5. `next(ctx, tx, simulate)` is called unconditionally, and the transaction executes successfully.

### Impact Explanation
The blocklist is a Cronos admin security control explicitly designed to prevent specific addresses from transacting â€” including sending EVM transactions, interacting with contracts, and performing IBC/bridge operations. Bypassing it allows a blocked address to:
- Send EVM transactions to any destination, including CRC20/CRC21 token contracts (unauthorized transfers)
- Interact with the IBC transfer module and bridge flows
- Perform any on-chain action that the blocklist was intended to prevent

This matches the allowed High impact: **Bypass of Cronos block-list authorization checks**.

### Likelihood Explanation
Any validator (including the attacker if they operate a validator node) can include transactions directly in a proposed block without routing them through the public mempool. The `CheckTx` path is a mempool admission gate, not a consensus-level enforcement. The bypass requires no privileged keys beyond block proposal rights, which are available to any bonded validator.

### Recommendation
Remove the `if ctx.IsCheckTx()` wrapper so that blocklist checks apply during both `CheckTx` and `DeliverTx`. The signer and destination checks should execute unconditionally:

```go
func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
    // enforce blocklist on ALL execution paths, not just CheckTx
    if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
        // ... signer checks ...
    }
    for _, msg := range tx.GetMsgs() {
        // ... EVM destination and EIP-7702 checks ...
    }
    // MsgStoreBlockList admin check can remain CheckTx-only if desired,
    // but the address blocklist checks must not be.
    return next(ctx, tx, simulate)
}
```

### Proof of Concept
1. Deploy a local Cronos devnet.
2. Have the admin call `MsgStoreBlockList` to add address `A` to the blocklist.
3. Confirm that a normal mempool submission from `A` is rejected (the `CheckTx` path works).
4. Construct a valid signed transaction from `A` (e.g., a CRC20 `transfer` call).
5. Submit it directly to a validator via `broadcast_tx_sync` on a node configured to skip mempool checks, or inject it into a block proposal directly.
6. Observe that `DeliverTx` succeeds: `ctx.IsCheckTx()` is `false`, the entire `if` block in `AnteHandle` is skipped, and `next` is called unconditionally.
7. The blocked address successfully executes the transaction. [1](#0-0)

func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)
}

func (k msgServer) StoreBlockList(goCtx context.Context, msg *types.MsgStoreBlockList) (*types.MsgStoreBlockListResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	ctx.KVStore(k.storeKey).Set(types.KeyPrefixBlockList, msg.Blob)
	return &types.MsgStoreBlockListResponse{}, nil
}
