### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Freeze Staker Yield — (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` in `src/staking/staking.cairo` has no caller restriction. It accepts a `disable_rewards` boolean parameter. Any unprivileged caller can invoke it with `disable_rewards: true` for any valid staker, which writes `last_reward_block` to the current block without distributing any rewards. Because `last_reward_block` is a global state variable and the function asserts `current_block_number > last_reward_block`, only one successful call is possible per block. An attacker who front-runs every block with `disable_rewards: true` permanently prevents the targeted staker from ever accumulating consensus-phase rewards.

---

### Finding Description

`update_rewards` is the public entry point for distributing per-block consensus rewards to a staker:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← state written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // ← early exit, no rewards distributed
    }
    ...
    self._update_rewards(...);
}
``` [1](#0-0) 

Key observations:

1. **No access control.** `general_prerequisites()` only checks the pause flag. There is no `assert_caller_is_X` guard. [2](#0-1) 

2. **`last_reward_block` is written before the `disable_rewards` branch.** The global sentinel is updated regardless of whether rewards are actually distributed. [3](#0-2) 

3. **`disable_rewards: true` suppresses all reward accounting.** When the flag is set, the function returns immediately after writing `last_reward_block`, so `_update_rewards` is never reached and `unclaimed_rewards_own` is never incremented. [4](#0-3) 

4. **One call per block is enforced globally.** The `REWARDS_ALREADY_UPDATED` assertion means that once an attacker's call succeeds in block N, every legitimate call in block N reverts. [5](#0-4) 

The contrast with `update_current_epoch_block_rewards` on the reward supplier — which correctly enforces `CALLER_IS_NOT_STAKING_CONTRACT` — makes the missing guard on `update_rewards` stand out. [6](#0-5) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(target_staker, disable_rewards: true)` every block:

- Writes `last_reward_block` to the current block without crediting any rewards to `unclaimed_rewards_own`.
- Blocks every legitimate `update_rewards` call for that block with `REWARDS_ALREADY_UPDATED`.
- Repeating this every block means the staker's `unclaimed_rewards_own` never increases; all consensus-phase block rewards are permanently lost.

The staker has no recourse: they cannot override the attacker's call within the same block, and the attacker can repeat the attack indefinitely at low cost.

---

### Likelihood Explanation

**High.** The function is fully public, requires no special role or token balance, and the only prerequisite is that a valid, active staker with non-zero balance exists — a condition trivially satisfied on a live network. The attacker pays only the gas cost of one call per block (~6 s on Starknet), which is negligible compared to the yield stolen from a large staker.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the staking contract itself, the attestation contract, or the targeted staker's own address). At minimum, the `disable_rewards: true` path must never be reachable by an arbitrary external caller. A simple guard analogous to the one already present on `update_rewards_from_attestation_contract` would suffice:

```cairo
fn update_rewards(...) {
    self.assert_caller_is_staking_contract(); // or appropriate role
    ...
}
``` [7](#0-6) 

---

### Proof of Concept

1. Deploy the system and advance past `consensus_rewards_first_epoch + K` so consensus rewards are active.
2. Stake as `staker_address` and wait `K` epochs for the balance to become effective.
3. As an unprivileged attacker, call every block:
   ```cairo
   IStakingRewardsManagerDispatcher { contract_address: staking }
       .update_rewards(staker_address: victim, disable_rewards: true);
   ```
4. Observe that `last_reward_block` advances each block but `staker_info.unclaimed_rewards_own` remains zero.
5. Any legitimate call by the victim in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. After N blocks the victim's claimable rewards are 0 instead of the expected `N × block_rewards × (victim_stake / total_stake)`. [1](#0-0) [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
