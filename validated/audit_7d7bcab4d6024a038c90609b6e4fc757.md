### Title
Pool Admin Can Set Per-Bin Additional Fees to Uncapped `uint16` Max, Bypassing Factory Fee Cap System — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` forwards `addFeeBuyE6` / `addFeeSellE6` directly to the pool with **no cap check**, while the parallel admin fee setter `setPoolAdminFees` explicitly enforces `maxAdminSpreadFeeE6` / `maxAdminNotionalFeeE8`. A pool admin can therefore write `65535` (6.5535 %) into any bin's additional-fee fields, and every swap routed through that bin will pay that surcharge on top of the configured global spread fee.

---

### Finding Description

`setPoolAdminFees` guards both fee components before touching the pool:

```solidity
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees`, by contrast, performs **no validation** on the fee values and calls straight through to the pool:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

`MetricOmmPool.setBinAdditionalFees` likewise performs no cap check — it only validates the bin index — and writes the caller-supplied values directly into `BinState`:

```solidity
s.addFeeBuyE6 = addFeeBuyE6;
s.addFeeSellE6 = addFeeSellE6;
``` [3](#0-2) 

During every swap the per-bin fee is **added** to `baseFeeX64` before the swap-math call:

```solidity
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6),
``` [4](#0-3) 

The same pattern applies to the buy direction in `getSellAndBuyPrices`:

```solidity
uint256 buyFeeX64  = baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6,  ONE_X64, 1e6);
uint256 sellFeeX64 = baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6);
``` [5](#0-4) 

`uint16` max is `65535`, which in E6 notation equals **6.5535 %**. There is no factory state variable, no hard constant, and no revert path that prevents a pool admin from writing this value.

---

### Impact Explanation

Every trader whose swap crosses a bin whose `addFeeBuyE6` or `addFeeSellE6` has been set to `65535` pays an additional 6.5535 % on top of the global spread fee. The extra fee is collected as LP fee inside the bin (it inflates `binLpFeeAmountScaled`), so it accrues to LPs rather than being returned to the trader. This is a **direct, permanent loss of trader principal** on every affected swap, with no on-chain mechanism to detect or revert it.

The factory's cap system (`maxAdminSpreadFeeE6`, hard ceiling `HARD_MAX_SPREAD_FEE_E6 = 200_000` = 20 %) is designed to bound the total fee a pool admin can impose. [6](#0-5) 

Per-bin additional fees are additive on top of the global spread fee and are entirely outside that cap system, so the admin-boundary the factory is meant to enforce is broken.

---

### Likelihood Explanation

The pool admin is a semi-trusted role that is explicitly in scope for admin-boundary checks. The call path is two transactions (set fees, then any swap through the bin) and requires no special pool state. Any pool whose admin key is compromised, misconfigured, or acting adversarially can exploit this immediately.

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool, mirroring the pattern in `setPoolAdminFees`:

```solidity
if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
```

Alternatively, introduce a dedicated `maxAdminBinAdditionalFeeE6` cap variable (owner-settable, hard-capped) so the protocol can tune the per-bin ceiling independently of the global spread cap.

---

### Proof of Concept

1. Deploy a standard pool; note `maxAdminSpreadFeeE6` (e.g. 20 000 = 2 %).
2. As pool admin call `factory.setPoolBinAdditionalFees(pool, 0, 65535, 65535)`.
3. Observe `BinAdditionalFeesUpdated(0, 65535, 65535)` emitted — no revert.
4. Execute a swap through bin 0.
5. Assert the trader's output is reduced by ≈ 6.5535 % beyond what the global spread fee alone would produce, confirming the uncapped per-bin fee is applied.
