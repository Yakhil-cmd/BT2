### Title
Unprivileged `MsgConvertVouchers` Auto-Deployment Permanently Blocks Admin from Registering External Token Mapping - (File: `x/cronos/keeper/evm.go`, `x/cronos/keeper/msg_server.go`)

---

### Summary

When `EnableAutoDeployment` is true, any unprivileged user holding at least 1 unit of a valid IBC or gravity token can call `MsgConvertVouchers` to trigger auto-deployment of a `ModuleCRC21` contract for that denom. Because `SetAutoContractForDenom` writes a permanent denom→contract entry and `SetExternalContractForDenom` (used by the admin's `MsgUpdateTokenMapping`) unconditionally rejects any denom that already resolves to a contract via `ensureDenomNotMapped`, the admin is permanently and irrecoverably blocked from registering a legitimate external contract for that denom.

---

### Finding Description

`MsgConvertVouchers` has no permission check: [1](#0-0) 

It calls `ConvertVouchersToEvmCoins`, which for any non-CRO denom calls `ConvertCoinFromNativeToCRC21` with `autoDeploy = params.EnableAutoDeployment`: [2](#0-1) 

Inside `ConvertCoinFromNativeToCRC21`, when no mapping exists and `autoDeploy` is true, a `ModuleCRC21` contract is deployed and `SetAutoContractForDenom` is called to permanently record the mapping: [3](#0-2) 

`SetAutoContractForDenom` calls `ensureDenomNotMapped`, which succeeds only if no mapping exists yet, then writes both the forward and reverse KV entries: [4](#0-3) 

Later, when the admin calls `MsgUpdateTokenMapping` → `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom`, the same `ensureDenomNotMapped` guard fires and returns `ErrDenomAlreadyMapped`: [5](#0-4) [6](#0-5) 

There is no `DeleteAutoContractForDenom` function and `DeleteExternalContractForDenom` only removes external entries, leaving the auto-contract mapping intact. The state is irrecoverable without a governance upgrade.

---

### Impact Explanation

**High — Bypass of Cronos admin authority for token-mapping; permanent corruption of token mappings with direct security impact.**

Once an attacker front-runs the admin for a given denom:

1. The denom is permanently bound to an auto-deployed `ModuleCRC21` contract the admin did not choose.
2. The admin's `MsgUpdateTokenMapping` is permanently rejected with `ErrDenomAlreadyMapped` for that denom.
3. Any project that intended to use a custom external CRC20/CRC21 contract (with custom logic, access controls, or supply management) for that denom is forced to use the generic auto-deployed contract instead.
4. If the intended external contract had already been deployed and announced, users may interact with it expecting it to be the canonical mapping — but it never gets registered, leading to accounting divergence between the EVM and Cosmos layers.

---

### Likelihood Explanation

- `MsgConvertVouchers` is a standard, publicly documented Cosmos SDK message requiring no special privilege.
- The attacker only needs 1 unit of any valid IBC or gravity token, obtainable by sending a trivial IBC transfer to themselves from any connected chain.
- The attack is a single transaction and costs only standard gas fees.
- The condition `EnableAutoDeployment = true` is the default production configuration for Cronos.
- The attack can be executed preemptively against any denom before the admin acts, or reactively by monitoring the mempool for pending `MsgUpdateTokenMapping` transactions.

---

### Recommendation

Add a permission check to `MsgConvertVouchers` before allowing auto-deployment, or decouple auto-deployment from the user-facing conversion message. Specifically:

1. **Restrict auto-deployment to privileged callers only** (admin or IBC/gravity hooks). The `ConvertCoinFromNativeToCRC21` `autoDeploy` flag should be `false` when called from `MsgConvertVouchers`; auto-deployment should only occur from the IBC `OnRecvVouchers` hook (which is triggered by the IBC module, not by arbitrary users).
2. **Allow the admin to override auto-deployed mappings.** `SetExternalContractForDenom` should be permitted to replace an auto-deployed contract when called by an authorized admin, rather than treating auto-contracts and external contracts identically in `ensureDenomNotMapped`.

---

### Proof of Concept

1. `EnableAutoDeployment` is `true` (default). Denom `ibc/HASH` has no mapping yet.
2. Attacker receives 1 unit of `ibc/HASH` via a standard IBC transfer.
3. Attacker submits `MsgConvertVouchers{Address: attacker, Coins: [{1, "ibc/HASH"}]}`.
4. `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21(ctx, attacker, {1, "ibc/HASH"}, true)`.
5. No contract found → `DeployModuleCRC21` deploys a new contract at address `0xAUTO`.
6. `SetAutoContractForDenom("ibc/HASH", 0xAUTO)` writes `DenomToAutoContractKey("ibc/HASH") → 0xAUTO` and `ContractToDenomKey(0xAUTO) → "ibc/HASH"` to the KV store.
7. Admin later submits `MsgUpdateTokenMapping{Denom: "ibc/HASH", Contract: "0xLEGIT"}`.
8. `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom` → `ensureDenomNotMapped("ibc/HASH")` → `GetContractByDenom` returns `(0xAUTO, true)` → returns `ErrDenomAlreadyMapped`.
9. Admin's transaction is permanently rejected. `0xLEGIT` is never registered. [7](#0-6) [8](#0-7) [5](#0-4) [4](#0-3)

func (k Keeper) ConvertCoinFromNativeToCRC21(ctx sdk.Context, sender common.Address, coin sdk.Coin, autoDeploy bool) error {
	if !types.IsValidCoinDenom(coin.Denom) {
		return fmt.Errorf("coin %s is not supported for conversion", coin.Denom)
	}
	var err error
	// external contract is returned in preference to auto-deployed ones
	contract, found := k.GetContractByDenom(ctx, coin.Denom)
	if !found {
		if !autoDeploy {
			return fmt.Errorf("no contract found for the denom %s", coin.Denom)
		}
		contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
		if err != nil {
			return err
		}
		if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
			return err
		}

		k.Logger(ctx).Info(fmt.Sprintf("contract address %s created for coin denom %s", contract.String(), coin.Denom))
	}

func (k Keeper) ensureDenomNotMapped(ctx sdk.Context, denom string) error {
	if contract, found := k.GetContractByDenom(ctx, denom); found {
		return errors.Wrapf(
			types.ErrDenomAlreadyMapped,
			"denom %s is already mapped to contract %s",
			denom,
			contract.Hex(),
		)
	}
	return nil
}

func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}

func (k Keeper) SetAutoContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	isSource := types.IsSourceCoin(denom)
	if err := validateContractAddressForSourceDenom(denom, address, isSource); err != nil {
		return err
	}
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}
	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToAutoContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}
