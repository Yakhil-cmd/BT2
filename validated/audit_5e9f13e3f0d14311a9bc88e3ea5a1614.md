### Title
Unprivileged Caller Can Permanently Freeze Block Rewards via `update_rewards` with `disable_rewards: true` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control beyond a pause-and-non-zero-caller check. Any unprivileged address can call it with `disable_rewards: true`, which consumes the global `last_reward_block` slot for the current block without distributing any rewards. Because the guard `current_block_number > last_reward_block` is a strict greater-than check on a single global variable, the legitimate staker can never reclaim that block's reward — it is permanently lost.

---

### Finding Description

`update_rewards` is the entry point for distributing per-block consensus rewards to a staker. Its only gate is `general_prerequisites()`, which checks that the contract is not paused and that the caller is non-zero:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // pause + non-zero only
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits with no rewards paid
    }
    ...
``` [1](#0-0) 

`last_reward_block` is a **single global** storage variable, not per-staker: [2](#0-1) 

The attack sequence is:

1. Attacker (any non-zero address) calls `update_rewards(valid_staker_address, disable_rewards: true)` at block N.
2. The function passes the `current_block_number > last_reward_block` check (assuming no prior call this block).
3. `last_reward_block` is written to N.
4. The function returns early — **no rewards are distributed**.
5. Any subsequent call to `update_rewards` at block N (by the legitimate staker or anyone else) reverts with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
6. Block N's reward is permanently unrecoverable.

The `disable_rewards` boolean is a caller-supplied parameter with no validation of who is allowed to set it to `true`. There is no role check, no whitelist, and no on-chain proof that the staker actually failed to perform their duties.

---

### Impact Explanation

Each block in the consensus rewards phase has a fixed reward budget. Once `last_reward_block` is set to block N, that budget is gone. The staker's `unclaimed_rewards_own` is never incremented for that block, and there is no recovery path. An attacker can repeat this every block, permanently freezing all future block rewards for every staker in the system. This matches **"Permanent freezing of unclaimed yield"** (High severity).

---

### Likelihood Explanation

The function is fully public — no role, no signature, no proof of work required. The only cost to the attacker is gas. The attacker does not need to hold any stake, be registered as a staker, or control any privileged key. They only need to know a valid (active, non-zero-balance) staker address, which is publicly observable from on-chain events (`NewStaker`). The attack can be automated to fire every block.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus layer contract, an operator role, or the staker themselves). At minimum, add a role check analogous to `assert_caller_is_attestation_contract` used in `update_rewards_from_attestation_contract`: [3](#0-2) 

Additionally, the `disable_rewards` flag should not be a free parameter supplied by an arbitrary caller. If the intent is to allow the consensus layer to penalise a staker, that signal must come from a verified, access-controlled source — not from any address that happens to call the function first.

---

### Proof of Concept

```
// Attacker is any EOA with enough gas.
// Precondition: current_block > last_reward_block (true at the start of any new block).

1. attacker.call(
       staking_contract,
       update_rewards(
           staker_address = <any valid, active staker>,
           disable_rewards = true
       )
   )

2. Inside update_rewards:
   - general_prerequisites() passes (not paused, caller != 0).
   - current_block_number (N) > last_reward_block (N-1) → passes.
   - staker validated as active with non-zero balance → passes.
   - last_reward_block.write(N)  ← slot consumed.
   - disable_rewards == true → return (no rewards distributed).

3. Legitimate staker calls update_rewards at block N:
   - current_block_number (N) > last_reward_block (N) → FALSE → REVERT.

4. Block N rewards are permanently lost.
   Repeat every block to freeze all consensus rewards indefinitely.
```
