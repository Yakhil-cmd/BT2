### Title
Unrestricted `update_rewards` with `disable_rewards=true` Permanently Denies Consensus Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary
The `update_rewards` function in the Staking contract has no access control. Any unprivileged caller can invoke it with `disable_rewards: true` for any active staker. Because the function writes to the global `last_reward_block` storage slot **before** checking the `disable_rewards` flag, the block's single reward-distribution slot is consumed without distributing any rewards. No subsequent call to `update_rewards` can succeed for that block. Repeated across every block, this permanently denies all consensus-phase block rewards to all stakers and their delegators.

---

### Finding Description

`update_rewards` is the sole mechanism for distributing consensus-phase block rewards. Its implementation is:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.  ← written BEFORE the disable check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits with no rewards paid
    }
    // ... reward calculation and distribution ...
}
```

The critical ordering flaw:

1. `last_reward_block` is updated at line 1485 unconditionally.
2. The `disable_rewards` guard at line 1487 fires **after** the slot is consumed.
3. `general_prerequisites` (line 1794) enforces only "not paused" and "caller ≠ 0" — no role check, no whitelist.

Because `last_reward_block` is a **global** (not per-staker) counter, exactly one call to `update_rewards` can succeed per block. An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` first in a block:

- Marks the block as "already updated."
- Distributes zero rewards.
- Causes every subsequent call in that block to revert with `REWARDS_ALREADY_UPDATED`.

The attacker needs only a valid active staker address (all staker addresses are publicly enumerable from the `stakers` Vec in storage) and enough gas to front-run the legitimate consensus-layer call.

---

### Impact Explanation

Each block for which the attacker fires first is permanently lost: there is no catch-up mechanism, no retroactive reward distribution, and no way to "replay" a block. Stakers and all their delegators receive zero block rewards for every griefed block. Because the attack is repeatable every block at low cost, the entire consensus reward stream can be frozen indefinitely. This matches **"Permanent freezing of unclaimed yield"** (High).

---

### Likelihood Explanation

- **No access control** — the function is callable by any EOA or contract.
- **No economic barrier** — Starknet gas fees are low; the attacker spends a small, fixed cost per block.
- **Staker addresses are public** — the `stakers` Vec is iterable on-chain; finding a valid active staker is trivial.
- **Race condition is winnable** — the attacker only needs to submit the transaction before the legitimate consensus-layer call in the same block. On a permissionless mempool this is straightforward; even with a centralized sequencer, a malicious or bribed sequencer can order the attacker's transaction first.

Likelihood: **Medium–High**.

---

### Recommendation

Restrict `update_rewards` to a designated consensus-layer caller (e.g., a role such as `REWARDS_MANAGER` or a hardcoded address stored in the contract). Add a role check at the top of the function, analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract`.

---

### Proof of Concept

```
// Attacker script — run once per block in the consensus-rewards phase

// 1. Read any active staker address from the public `stakers` Vec.
let victim_staker = staking_contract.stakers[0];

// 2. Call update_rewards with disable_rewards = true.
//    No special role required; general_prerequisites only checks pause + zero-address.
staking_contract.update_rewards(
    staker_address: victim_staker,
    disable_rewards: true,   // ← rewards suppressed
);
// last_reward_block is now set to current_block_number.

// 3. Any legitimate call by the consensus layer in the same block now reverts:
//    assert!(current_block_number > self.last_reward_block.read(), ...)
//    → REWARDS_ALREADY_UPDATED

// 4. Repeat every block → all consensus block rewards are permanently denied.
```
