### Title
Signature Bypass via `skip_stateful_validations`: Attacker-Controlled `sender_address` Admitted to Mempool Without `__validate__` - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's UX-skip path for the deploy-account + invoke flow allows any unprivileged caller to submit an `InvokeTransaction` with an **arbitrary victim `sender_address`** and a garbage signature. Because `skip_stateful_validations` returns `true` whenever the victim's address appears in the mempool (regardless of who submitted the transaction), the gateway skips the `__validate__` entry-point call entirely and admits the transaction to the mempool without any signature check.

### Finding Description

`skip_stateful_validations` in `crates/apollo_gateway/src/stateful_transaction_validator.rs` (lines 429–461) returns `true` — causing `run_validate_entry_point` to set `execution_flags.validate = false` — whenever **all three** of the following hold:

1. The incoming transaction is an `Invoke`.
2. `tx.nonce() == Nonce(Felt::ONE)`.
3. `account_nonce == Nonce(Felt::ZERO)` (account not yet on-chain).
4. `account_tx_in_pool_or_recent_block(tx.sender_address())` returns `true`. [1](#0-0) 

Condition 4 is checked against the **attacker-supplied** `sender_address` field. Any observer of the public mempool can see which addresses have a pending `DeployAccount` transaction and immediately submit a spoofed invoke with that address as `sender_address`, `nonce = 1`, and any garbage signature.

When `skip_validate = true`, `run_validate_entry_point` constructs an `AccountTransaction` with `execution_flags.validate = false` and calls `StatefulValidator::perform_validations`, which returns `Ok(())` early without ever calling the account's `__validate__` entry point: [2](#0-1) [3](#0-2) 

The transaction is then forwarded to the mempool via `validate_by_mempool` / `add_tx`, which performs only nonce-gap and fee-price checks — no signature verification. [4](#0-3) 

When the batcher later converts the stored `InternalRpcTransaction` to an executable transaction via `convert_internal_rpc_tx_to_executable_tx`, it produces a `blockifier::AccountTransaction` using `derive_more::From<Transaction>`, which fills `execution_flags` with `ExecutionFlags::default()` — where `validate: true`: [5](#0-4) [6](#0-5) 

So `__validate__` is called during batcher execution and the spoofed transaction is **rejected** (no fee charged, no state change). However, the transaction was already admitted to the mempool and occupies the `(victim_address, nonce=1)` slot.

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

1. **Nonce-slot squatting**: The attacker's spoofed transaction occupies `(victim, nonce=1)` in the mempool. The victim's legitimate first post-deploy invoke is rejected by the mempool with `DuplicateNonce` until the attacker's transaction is eventually rejected by the batcher and the mempool is notified.
2. **Free mempool spam**: Because rejected transactions incur no fee, the attacker can continuously flood the mempool with spoofed invokes targeting every account that has a pending `DeployAccount`, at zero cost.
3. **Delayed UX for new accounts**: Every new account deployment is a target window during which the attacker can block the account's first transaction.

### Likelihood Explanation

**High.** The mempool is observable (P2P propagation, RPC `pending` block). The attacker needs only to watch for `DeployAccount` transactions and immediately submit a spoofed invoke. No privileged access, no special knowledge, and no fee is required. The attack is fully automatable.

### Recommendation

The `skip_stateful_validations` check must bind the skip decision to the **transaction hash signed by the deployer**, not merely to the presence of any transaction for that address in the mempool. Concretely:

- Require the caller to supply the `deploy_account_tx_hash` (as the native-blockifier path already does in `PyValidator::should_run_stateful_validations`), and verify that the invoke transaction's hash is signed by the same key that signed the deploy-account transaction.
- Alternatively, restrict the skip to cases where the gateway itself received the `DeployAccount` in the same request batch (atomic pairing), so the `sender_address` is bound to the submitter's session. [7](#0-6) 

### Proof of Concept

```
1. Alice submits DeployAccount(address=0xALICE, nonce=0, ...) to the gateway.
   → Mempool now contains 0xALICE; account_tx_in_pool_or_recent_block(0xALICE) == true.

2. Attacker observes the mempool and sees 0xALICE has a pending DeployAccount.

3. Attacker submits:
     InvokeTransaction {
         sender_address: 0xALICE,   // victim's address
         nonce: 1,
         signature: [0xDEAD, 0xBEEF],  // garbage
         calldata: [drain_alice_funds...],
         resource_bounds: { ... valid ... },
     }

4. Gateway stateless validator: passes (no signature check here).

5. Gateway stateful validator:
   - account_nonce = 0  (Alice not deployed yet)
   - tx.nonce() = 1
   - account_tx_in_pool_or_recent_block(0xALICE) = true
   → skip_stateful_validations returns true
   → run_validate_entry_point sets validate=false, returns Ok(()) without calling __validate__

6. Transaction admitted to mempool at slot (0xALICE, nonce=1).

7. Alice submits her legitimate Invoke(sender=0xALICE, nonce=1, ...).
   → Mempool rejects: DuplicateNonce { address: 0xALICE, nonce: 1 }.

8. Batcher eventually executes attacker's tx:
   - __validate__ called → signature check fails → REJECTED (no fee, no state change).
   - Mempool notified; slot freed.
   - But Alice's window for her first post-deploy invoke was blocked.
```

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
