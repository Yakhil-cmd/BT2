### Title
Unprivileged Caller Can Permanently Suppress Block Rewards via `disable_rewards` Flag in `update_rewards` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is callable by any unprivileged address and accepts a caller-controlled `disable_rewards` boolean. Because `last_reward_block` is written **before** the `disable_rewards` guard, an attacker can call `update_rewards(staker_address: victim, disable_rewards: true)` on any block to permanently suppress that block's reward distribution for the targeted staker, with no way to recover the lost yield.

### Finding Description
`IStakingRewardsManager.update_rewards` carries no access-control check — only `general_prerequisites()` (a pause guard) is enforced. The function accepts an arbitrary `staker_address` and a caller-supplied `disable_rewards` flag. [1](#0-0) 

The critical ordering is:

1. `last_reward_block` is unconditionally written to `current_block_number` (line 1485).
2. Only **after** that write does the code branch on `disable_rewards` and return early without distributing rewards (line 1487). [2](#0-1) 

Because `last_reward_block` is a **global** single-slot value (not per-staker), once it is set to the current block, the guard at line 1454–1458 will reject any subsequent call for that same block: [3](#0-2) 

The legitimate reward-distribution call for that block is therefore permanently blocked. The rewards that should have been minted and credited are never added to `unclaimed_rewards` in the Reward Supplier and never credited to the staker's `unclaimed_rewards_own`.

Compare this with the protected path `update_rewards_from_attestation_contract`, which correctly enforces `assert_caller_is_attestation_contract()`: [4](#0-3) 

`update_rewards` has no equivalent guard.

### Impact Explanation
For every block in which the attacker front-runs the legitimate `update_rewards` call with `disable_rewards: true`:

- `last_reward_block` advances to that block number.
- No block rewards are computed or forwarded to the Reward Supplier.
- The staker's `unclaimed_rewards_own` is never incremented for that block.
- The loss is **permanent**: the slot is consumed and cannot be reclaimed.

Repeated across every block, this constitutes **permanent freezing of unclaimed yield** for any targeted staker. This matches the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

### Likelihood Explanation
- The function is fully public with no role restriction.
- The attacker needs only to submit a transaction before the legitimate caller each block — a standard front-running scenario on Starknet.
- The cost is one transaction per block; no capital is required.
- Any address (delegator, competing staker, or anonymous actor) can execute this.

### Recommendation
Add an access-control check so that only the attestation contract (or another trusted, protocol-controlled address) may call `update_rewards`. Alternatively, remove the `disable_rewards` parameter entirely and handle the "skip rewards" case internally based on on-chain state (e.g., `is_pre_consensus()`), so no external caller can influence whether rewards are distributed.

### Proof of Concept

1. Staker Alice is active and eligible for block rewards at block `N`.
2. Attacker Bob monitors the mempool and sees (or anticipates) the legitimate `update_rewards(alice, false)` call.
3. Bob submits `update_rewards(alice, true)` with higher gas priority.
4. Bob's transaction executes first:
   - `last_reward_block` is set to `N`.
   - The function returns early; no rewards are distributed.
5. Alice's (or the protocol's) legitimate call arrives and reverts with `REWARDS_ALREADY_UPDATED`.
6. Alice permanently loses her block-`N` rewards.
7. Bob repeats this every block at negligible cost, continuously suppressing Alice's yield. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
