### Title
PSMT.Update() Fails to Update Node Payload Field, Creating Stale-Read Inconsistency After Leaf Updates - (File: ledger/partial/ptrie/partialTrie.go)

### Summary

`PSMT.Update()` in the Partial Sparse Merkle Tree updates only `node.hashValue` for each updated leaf, but never updates `node.payload`. After an update, the root hash correctly reflects the new state, but any subsequent call to `Get()` or `GetSinglePayload()` returns the pre-update (stale) payload. This is a direct structural analog to the reported canopy-node omission: one component of the node's state (the hash) is kept current while another (the payload) is left inconsistent.

---

### Finding Description

`PSMT.Update()` iterates over the provided paths, looks up each leaf node via `pathLookUp`, and writes a new `hashValue`:

```go
node.hashValue = ledger.ComputeCompactValue(hash.Hash(path), payload.Value(), node.height)
``` [1](#0-0) 

It then calls `p.root.forceComputeHash()` to propagate the new leaf hash up to the root. The root hash returned is therefore correct.

However, `node.payload` — set during `NewPSMT` construction — is never touched:

```go
currentNode.payload = payload   // set once at construction
``` [2](#0-1) 

`forceComputeHash()` for a leaf node returns `n.hashValue` directly and never consults `n.payload`: [3](#0-2) 

But `Get()` and `GetSinglePayload()` return `node.payload`: [4](#0-3) 

After any call to `Update()`, the PSMT is in an inconsistent state: the Merkle root reflects the new values, but reads return the old values.

---

### Impact Explanation

The `PSMT` is the backing store for `partial.Ledger`, which is constructed fresh for every chunk verification in `ChunkVerifier.verifyTransactionsInContext()`: [5](#0-4) 

The verifier builds a `snapshotTree` over the partial ledger, re-executes all transactions in the chunk reading register values through that snapshot, then applies all register writes in a single `psmt.Set()` call at the end: [6](#0-5) 

`partial.Ledger.Set()` delegates directly to `ptrie.Update()`: [7](#0-6) 

In the current single-pass flow (all reads before the single write), the stale-payload inconsistency is not triggered. However, the `partial.Ledger` interface is public and general-purpose. Any caller that interleaves `Set()` and `Get()` — for example, a multi-step execution model where transaction N+1 reads a register written by transaction N through the same partial ledger instance — will silently receive the pre-update value. Because the root hash is computed from `hashValue` (correct) while reads return `payload` (stale), the ledger presents two contradictory views of state simultaneously. This can cause a verifier to re-execute transactions against wrong initial register values, producing a root hash that diverges from the execution node's result and causing legitimate execution receipts to be incorrectly challenged or rejected.

---

### Likelihood Explanation

The current `ChunkVerifier` applies a single batch write after all reads, so the bug is dormant in the existing code path. Likelihood rises to medium if:
- The execution model is extended to support intra-chunk state checkpointing (reads after intermediate writes), or
- Any other consumer of `partial.Ledger` calls `Get()` after `Set()`.

The `partial.Ledger` API provides no documentation warning that `Get()` is invalid after `Set()`, making accidental misuse likely in future development.

---

### Recommendation

In `PSMT.Update()`, update `node.payload` alongside `node.hashValue` for each updated path:

```go
node.hashValue = ledger.ComputeCompactValue(hash.Hash(path), payload.Value(), node.height)
node.payload = payload   // add this line
``` [8](#0-7) 

This keeps the payload field consistent with the hash field after every update, matching the invariant established at construction time in `NewPSMT`.

---

### Proof of Concept

1. Construct a PSMT with a single path `p` holding payload `A`.
2. Call `psmt.Update([]ledger.Path{p}, []*ledger.Payload{B})` — root hash now reflects `B`.
3. Call `psmt.Get([]ledger.Path{p})` — returns `A` (stale), not `B`.
4. The PSMT simultaneously reports a root hash consistent with `B` and a payload value of `A`, violating the invariant that the root hash commits to the current register values.

This is directly reproducible using the existing test harness in `ledger/partial/ptrie/partialTrie_test.go` by adding a `Get()` assertion after the `Update()` call in `TestPartialTrieLeafUpdates`. [9](#0-8)

func (p *PSMT) Get(paths []ledger.Path) ([]*ledger.Payload, error) {
	var failedPaths []ledger.Path
	payloads := make([]*ledger.Payload, len(paths))
	for i, path := range paths {
		// lookup the path for the payload
		node, found := p.pathLookUp[path]
		if !found {
			failedPaths = append(failedPaths, path)
			continue
		}
		payloads[i] = node.payload
	}
	if len(failedPaths) > 0 {
		return nil, &ErrMissingPath{Paths: failedPaths}
	}
	return payloads, nil
}

func (p *PSMT) Update(paths []ledger.Path, payloads []*ledger.Payload) (ledger.RootHash, error) {
	var failedPaths []ledger.Path
	for i, path := range paths {
		payload := payloads[i]
		// lookup the path and update the value
		node, found := p.pathLookUp[path]
		if !found {
			failedPaths = append(failedPaths, path)
			continue
		}
		node.hashValue = ledger.ComputeCompactValue(hash.Hash(path), payload.Value(), node.height)
	}
	if len(failedPaths) > 0 {
		return ledger.RootHash(hash.DummyHash), &ErrMissingPath{Paths: failedPaths}
	}
	// after updating all the nodes, compute the value recursively only once
	return ledger.RootHash(p.root.forceComputeHash()), nil
}

	if update.Size() == 0 {
		return update.State(),
			&ledger.TrieUpdate{
				RootHash: ledger.RootHash(update.State()),
				Paths:    []ledger.Path{},
				Payloads: []*ledger.Payload{},
			},
			nil
	}

	trieUpdate, err = pathfinder.UpdateToTrieUpdate(update, l.pathFinderVersion)
	if err != nil {
		return ledger.DummyState, nil, err
	}

	newRootHash, err := l.ptrie.Update(trieUpdate.Paths, trieUpdate.Payloads)
	if err != nil {
		if pErr, ok := err.(*ptrie.ErrMissingPath); ok {
			//store mappings and restore keys from missing paths
			pathToKey := make(map[ledger.Path]ledger.Key)

			for i, key := range update.Keys() {
				path := trieUpdate.Paths[i]
				pathToKey[path] = key
			}

			keys := make([]ledger.Key, len(pErr.Paths))
			for i, path := range pErr.Paths {
				keys[i] = pathToKey[path]
			}
			return ledger.DummyState, nil, &ledger.ErrMissingKeys{Keys: keys}
		}
		return ledger.DummyState, nil, err
	}

	// TODO log info state
	return ledger.State(newRootHash), trieUpdate, nil

func TestPartialTrieLeafUpdates(t *testing.T) {

	pathByteSize := 32
	withForest(t, pathByteSize, 10, func(t *testing.T, f *mtrie.Forest) {

		path1 := testutils.PathByUint16(0)
		payload1 := testutils.LightPayload('A', 'a')
		updatedPayload1 := testutils.LightPayload('B', 'b')

		path2 := testutils.PathByUint16(1)
		payload2 := testutils.LightPayload('C', 'c')
		updatedPayload2 := testutils.LightPayload('D', 'd')

		path3 := testutils.PathByUint16(2)
		payload3 := testutils.LightPayload('E', 'e')

		paths := []ledger.Path{path1, path2}
		payloads := []*ledger.Payload{payload1, payload2}

		u := &ledger.TrieUpdate{RootHash: f.GetEmptyRootHash(), Paths: paths, Payloads: payloads}
		rootHash, err := f.Update(u)
		require.NoError(t, err, "error updating trie")

		r := &ledger.TrieRead{RootHash: rootHash, Paths: paths}
		bp, err := f.Proofs(r)
		require.NoError(t, err, "error getting batch proof")

		psmt, err := NewPSMT(rootHash, bp)
		require.NoError(t, err, "error building partial trie")
		ensureRootHash(t, rootHash, psmt)

		payloads = []*ledger.Payload{updatedPayload1, updatedPayload2}
		rootHash, err = f.Update(&ledger.TrieUpdate{RootHash: rootHash, Paths: paths, Payloads: payloads})
		require.NoError(t, err, "error updating trie")

		_, err = psmt.Update(paths, payloads)
		require.NoError(t, err, "error updating psmt")
		ensureRootHash(t, rootHash, psmt)

		// Update on non-existent leafs
		_, err = psmt.Update([]ledger.Path{path3}, []*ledger.Payload{payload3})
		missingPathErr, ok := err.(*ErrMissingPath)
		require.True(t, ok)
		require.Equal(t, 1, len(missingPathErr.Paths))
		require.Equal(t, path3, missingPathErr.Paths[0])
	})
