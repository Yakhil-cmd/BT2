### Title
Blocklist Destination Check Bypassed via ERC20 `transfer`/`transferFrom` to Blocked Addresses - (File: `app/proposal.go`, `app/block_address.go`)

### Summary
Cronos enforces a blocklist that is intended to prevent blocked addresses from both sending transactions and receiving assets. The destination check only inspects the top-level EVM transaction `To` field. Any ERC20/CRC20 token transfer routed through a contract call — where the EVM tx `To` is the contract, not the blocked address — bypasses the blocklist entirely, allowing blocked addresses to receive CRC20/ERC20 tokens.

### Finding Description
The blocklist is enforced at three points, all using the same logic:

1. **`BlockAddressesDecorator.AnteHandle`** (CheckTx only) — checks tx signers and `ethTx.To()`.
2. **`ProposalHandler.ValidateTransaction`** (PrepareProposal) — checks tx signers and `ethTx.To()`.
3. **`ProposalHandler.ProcessProposalHandler`** (ProcessProposal) — calls `ValidateTransaction` per tx.

In both enforcement paths, the destination check is:

```go
if ethTx.To() != nil {
    if _, ok := h.blocklist[encoded]; ok {
        return fmt.Errorf("destination address is blocked: %s", encoded)
    }
}
``` [1](#0-0) [2](#0-1) 

This only inspects the **outer EVM transaction's `To` field** — the contract being called. When a user calls `crc20.transfer(blocked_address, amount)` or `crc20.transferFrom(from, blocked_address, amount)`, the EVM tx `To` is the CRC20 contract address (not blocked), so both checks pass. The actual token recipient — the blocked address — is encoded only in the calldata and is invisible to the blocklist enforcement layer.

The intent to block destinations is explicit: the `test_block_list_contract` integration test and CHANGELOG entry `[#1922] Feat: check destination address in the blocklist` confirm this was a deliberate security control. The `ModuleCRC21` and `ModuleCRC20` contracts expose standard DSToken `transfer` and `transferFrom` functions with no override or restriction: [3](#0-2) 

DSToken's inherited `transfer` and `transferFrom` are fully functional and unrestricted at the contract level, meaning any caller can route tokens to a blocked address through a contract call.

Additionally, the `BlockAddressesDecorator` ante handler wraps its entire check inside `if ctx.IsCheckTx()`, meaning **no blocklist check runs at all during DeliverTx (block execution)**: [4](#0-3) 

Enforcement relies entirely on PrepareProposal/ProcessProposal filtering, which only inspects the outer tx `To`.

### Impact Explanation
**High** — Bypass of Cronos blocklist authorization checks. An unprivileged attacker can transfer CRC20/ERC20 tokens to any blocked address by calling a token contract's `transfer(blocked_address, amount)`. The blocked address receives the tokens despite being on the blocklist. Additionally, if a blocked address had previously approved a spender, that spender can call `transferFrom(blocked_address, recipient, amount)` to drain the blocked address's CRC20 holdings, since the tx signer and `To` are both unblocked.

### Likelihood Explanation
High. The attack requires no special privileges. Any user holding CRC20 tokens can call `transfer(blocked_address, amount)` on any deployed CRC20/CRC21 contract. The bypass is structural and requires no exploit of edge cases — it is the normal ERC20 transfer flow.

### Recommendation
The blocklist destination enforcement must be extended beyond the outer EVM tx `To` field. Options include:
- Implementing an EVM state-transition hook (e.g., in `PostTxProcessing` or via a custom EVM tracer) that inspects ERC20 `Transfer` events and rejects transactions where the recipient is a blocked address.
- Overriding `transfer` and `transferFrom` in the `ModuleCRC21`/`ModuleCRC20` contracts to check a blocklist stored on-chain, analogous to how the external report recommended disabling `transferFrom`.
- Enforcing the blocklist check during DeliverTx (removing the `ctx.IsCheckTx()` guard in `BlockAddressesDecorator`) so that any tx that slips through proposal filtering is still rejected at execution time.

### Proof of Concept
1. Admin adds `blocked_address` to the blocklist via `MsgStoreBlockList`.
2. Attacker (any unprivileged address) holds CRC20 tokens on a registered `ModuleCRC21` contract.
3. Attacker submits EVM tx: `crc20.transfer(blocked_address, 1000)`.
   - EVM tx `To` = CRC20 contract address (not in blocklist) → `ValidateTransaction` passes.
   - Tx signer = attacker (not in blocklist) → signer check passes.
4. Tx is included in a block and executed. `ModuleCRC21.transfer` (inherited from DSToken) moves 1000 tokens to `blocked_address`.
5. `blocked_address` now holds 1000 CRC20 tokens, bypassing the blocklist destination restriction.

For the `transferFrom` variant:
1. `blocked_address` previously approved `attacker` for 5000 tokens (approval made before blocking).
2. Attacker calls `crc20.transferFrom(blocked_address, attacker, 5000)`.
3. Same bypass: tx `To` is the contract, signer is attacker — both pass the blocklist check.
4. 5000 tokens are drained from `blocked_address`. [5](#0-4) [6](#0-5)

func (h *ProposalHandler) ValidateTransaction(tx sdk.Tx, txBz []byte) error {
	if len(h.blocklist) == 0 {
		// fast path, accept all txs
		return nil
	}

	var err error
	if tx == nil {
		tx, err = h.TxDecoder(txBz)
		if err != nil {
			return err
		}
	}

	sigTx, ok := tx.(signing.SigVerifiableTx)
	if !ok {
		return fmt.Errorf("tx of type %T does not implement SigVerifiableTx", tx)
	}

	signers, err := sigTx.GetSigners()
	if err != nil {
		return err
	}
	for _, signer := range signers {
		encoded, err := h.addressCodec.BytesToString(signer)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", signer, err)
		}
		if _, ok := h.blocklist[encoded]; ok {
			return fmt.Errorf("signer is blocked: %s", encoded)
		}
	}

	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if ok {
			ethTx := msgEthTx.AsTransaction()
			// check the destination address
			if ethTx.To() != nil {
				encoded, err := h.addressCodec.BytesToString(ethTx.To().Bytes())
				if err != nil {
					return fmt.Errorf("invalid bech32 address: %s, err: %w", ethTx.To(), err)
				}
				if _, ok := h.blocklist[encoded]; ok {
					return fmt.Errorf("destination address is blocked: %s", encoded)
				}
			}
			// check EIP-7702 authorisation list
			if ethTx.SetCodeAuthorizations() != nil {
				for _, auth := range ethTx.SetCodeAuthorizations() {
					addr, err := auth.Authority()
					if err == nil {
						encoded, err := h.addressCodec.BytesToString(addr.Bytes())
						if err != nil {
							return fmt.Errorf("invalid bech32 address: %s, err: %w", addr, err)
						}
						if _, ok := h.blocklist[encoded]; ok {
							return fmt.Errorf("signer is blocked: %s", encoded)
						}
					}
					// check the target address
					encoded, err := h.addressCodec.BytesToString(auth.Address.Bytes())
					if err != nil {
						return fmt.Errorf("invalid bech32 address: %s, err: %w", auth.Address, err)
					}
					if _, ok := h.blocklist[encoded]; ok {
						return fmt.Errorf("authorisation address is blocked: %s", encoded)
					}
				}
			}
		}
	}

	return nil
}

func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)
}
