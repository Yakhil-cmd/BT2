### Title
`MsgTurnBridge` Handler is a No-Op, Permanently Bypassing Admin Bridge-Disable Authorization — (`x/cronos/keeper/msg_server.go`)

### Summary
The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is implemented as a stub that returns `nil, nil` without performing any state mutation. The `bridge_active` parameter is never updated, so the admin's emergency bridge-disable capability is silently voided. Any unprivileged user can continue to invoke bridge operations after an admin has submitted — and believes they have executed — a successful `MsgTurnBridge{Enable: false}` transaction.

### Finding Description
The root cause is at `msg_server.go` lines 85–87:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

The handler accepts the message, passes all ante-handler checks (signature, fee, sequence), and returns a success response — but never calls `k.SetParams`, `k.GetParams`, or any other state-writing function. The `bridge_active` field in the Cronos module params is therefore immutable through this message path.

The `MsgTurnBridge` proto definition and the CLI command are fully wired: [2](#0-1) [3](#0-2) 

The integration test `test_gravity_turn_bridge` explicitly verifies that after a successful `turn_bridge("false", from_="validator")` call, `send_to_evm_chain` must fail: [4](#0-3) 

Because the handler is a no-op, the `bridge_active` flag is never written, so the EVM hook that gates `__CronosSendToEvmChain` events on `bridge_active` always sees the initial (enabled) value. Every bridge operation that should be blocked after an admin shutdown continues to succeed.

### Impact Explanation
**High — Bypass of Cronos admin authorization check.**

The `bridge_active` parameter is the sole on-chain mechanism for the admin to halt bridge operations in an emergency (e.g., a discovered exploit, oracle manipulation, or regulatory requirement). Because `TurnBridge` never writes state, this control is permanently non-functional. An unprivileged user who submits bridge transactions after an admin "disable" call faces no on-chain enforcement: the bridge remains open regardless of the admin's intent. This directly matches the allowed High impact category: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."*

### Likelihood Explanation
The admin submits `MsgTurnBridge{Enable: false}`, receives a success response with code 0, and has no on-chain signal that the bridge was not actually disabled. The bypass requires zero attacker privilege — any user who simply continues to call bridge functions after the admin's transaction is confirmed exploits the gap.

### Recommendation
Implement the handler body to actually persist the parameter change:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    // Enforce that only the authorized sender may call this.
    params := k.GetParams(ctx)
    params.BridgeActive = msg.Enable   // or the correct field name
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

Additionally, add an authorization check (admin or governance) before the state write, consistent with how `UpdateTokenMapping` and `UpdatePermissions` guard their handlers. [5](#0-4) 

### Proof of Concept

1. Admin holds the `CronosAdmin` address and submits `MsgTurnBridge{Sender: admin, Enable: false}`.
2. The transaction is accepted on-chain; the response is `MsgTurnBridgeResponse{}` with code 0.
3. The admin queries `bridge_active` — it remains `true` because no state was written.
4. An unprivileged user calls `send_to_evm_chain` (or any other bridge entry point gated on `bridge_active`).
5. The EVM hook reads `bridge_active == true` and processes the bridge operation normally.
6. The admin's emergency shutdown has no effect; bridge operations continue unimpeded.
