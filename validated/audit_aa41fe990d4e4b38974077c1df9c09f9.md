### Title
Unauthenticated Admin HTTP Endpoint Allows Any Caller to Mutate Critical Node Runtime State - (`admin/command_runner.go`)

### Summary
The Flow node admin server exposes an HTTP endpoint at `/admin/run_command` (default port 9002) with no authentication or authorization checks. Any caller that can reach this port can invoke privileged commands including setting `consensus-required-approvals-for-sealing` to 0, crashing execution nodes at arbitrary heights, modifying transaction rate limits, and changing log levels — all without any identity verification.

### Finding Description
The `CommandRunner` in `admin/command_runner.go` starts an HTTP server that proxies all requests to an internal gRPC server. Neither the HTTP server nor the gRPC server applies any authentication middleware or caller identity check.

The HTTP server is created with no TLS by default and no authentication handler:

```go
httpServer := &http.Server{
    Addr:      r.httpAddress,
    Handler:   mux,
    TLSConfig: r.tlsConfig,  // nil by default
}
```

The gRPC server is created with only message-size options — no interceptors for authentication:

```go
opts := []grpc.ServerOption{
    grpc.MaxRecvMsgSize(r.maxMsgSize),
    grpc.MaxSendMsgSize(r.maxMsgSize),
}
grpcServer := grpc.NewServer(opts...)
```

The `runCommand` function dispatches to registered handlers after only validating the *format* of the request data — it never checks who is calling:

```go
func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
    req := &CommandRequest{Data: data}
    if validator := r.getValidator(command); validator != nil {
        if validationErr := validator(req); validationErr != nil { ... }
    }
    if handler := r.getHandler(command); handler != nil {
        handleResult, handleErr = handler(ctx, req)
    }
    ...
}
```

The registered commands that have security impact include:

- **`set-config`** (`admin/commands/common/set_config.go`): Sets any registered updatable config field. On consensus nodes, `consensus-required-approvals-for-sealing` is registered as updatable. Setting it to `0` causes the `ApprovalCollector` to seal results immediately without any verification approvals.
- **`stop-at-height`** (`admin/commands/execution/stop_at_height.go`): Sets a stop height and optionally crashes (`ShouldCrash: true`) the execution node.
- **`ingest-tx-rate-limit`** (`admin/commands/collection/tx_rate_limiter.go`): Adds/removes addresses from the rate-limit list or reconfigures rate limits.
- **`set-log-level`** / **`set-golog-level`** (`admin/commands/common/set_log_level.go`, `set_golog_level.go`): Changes global log verbosity, potentially exposing sensitive debug output.

### Impact Explanation
The most critical impact is via `set-config` targeting `consensus-required-approvals-for-sealing`. The consensus node registers this field as dynamically updatable:

```go
err = node.ConfigManager.RegisterUintConfig("consensus-required-approvals-for-sealing",
    setter.RequireApprovalsForSealConstructionDynamicValue,
    setter.SetRequiredApprovalsForSealingConstruction)
```

When set to `0`, `NewApprovalCollector` immediately seals every incorporated result without waiting for any verification approvals:

```go
if requiredApprovalsForSealConstruction == 0 {
    for i := range numberOfChunks {
        _, err := collector.aggregatedSignatures.PutSignature(i, flow.AggregatedSignature{})
        ...
    }
    err := collector.SealResult()
    ...
}
```

This bypasses the entire verification layer of the Flow protocol, allowing execution results — including potentially invalid or malicious ones — to be sealed and finalized on-chain without any cryptographic verification from Verification nodes. This constitutes unauthorized mutation of critical protocol state with direct on-chain security impact.

### Likelihood Explanation
The admin server binds to a configurable address (default `localhost:9002`). While typically bound to localhost, the code enforces no network restriction. Any process co-located on the node host, any SSRF vulnerability in a co-located service, or any misconfigured network firewall that exposes port 9002 gives an attacker full unauthenticated access to all admin commands. The attack requires only a single HTTP POST with a JSON body — no credentials, tokens, or signatures of any kind.

### Recommendation
Add an authentication layer to the admin HTTP and gRPC servers. At minimum:
1. Require a shared secret token (e.g., a bearer token or API key) validated in an HTTP middleware and a gRPC interceptor before dispatching any command.
2. Restrict the bind address to a Unix domain socket or loopback interface by default, and enforce this in configuration validation.
3. For the gRPC server, add a `grpc.UnaryInterceptor` that validates caller identity before invoking any handler.
4. Consider separating read-only commands (e.g., `read-blocks`, `get-config`) from state-mutating commands (e.g., `set-config`, `stop-at-height`) and applying stricter controls to the latter.

### Proof of Concept
An attacker with network access to port 9002 of a consensus node can reduce the required approvals for sealing to zero with a single unauthenticated HTTP request:

```bash
curl http://<consensus-node>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config", "data": {"consensus-required-approvals-for-sealing": 0}}'
```

After this call, `sealingConfigs.requiredApprovalsForSealConstruction` is atomically set to `0`. Every subsequent `NewApprovalCollector` instantiation will immediately call `SealResult()` without collecting any verification approvals, causing the consensus node to propose seals for all incorporated results regardless of their validity. func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
	r.logger.Info().Str("command", command).Msg("received new command")

	req := &CommandRequest{Data: data}

	if validator := r.getValidator(command); validator != nil {
		if validationErr := validator(req); validationErr != nil {
			// for expected validation errors, return code InvalidArgument and the error text
			if IsInvalidAdminParameterError(validationErr) {
				return nil, status.Error(codes.InvalidArgument, validationErr.Error())
			}
			// for unexpected errors, return code Internal and log a warning
			r.logger.Err(validationErr).Msg("unexpected error validating admin request")
			return nil, status.Error(codes.Internal, validationErr.Error())
		}
	}

	var handleResult any
	var handleErr error

	if handler := r.getHandler(command); handler != nil {
		if handleResult, handleErr = handler(ctx, req); handleErr != nil {
			if errors.Is(handleErr, context.Canceled) {
				return nil, status.Error(codes.Canceled, "client canceled")
			} else if errors.Is(handleErr, context.DeadlineExceeded) {
				return nil, status.Error(codes.DeadlineExceeded, "request timed out")
			} else {
				r.logger.Err(handleErr).Msg("unexpected error handling admin request")
				s, _ := status.FromError(handleErr)
				return nil, s.Err()
			}
		}
	} else {
		return nil, status.Error(codes.Unimplemented, "invalid command")
	}

	return handleResult, nil
}

func (s *SetConfigCommand) Validator(req *admin.CommandRequest) error {
	mval, ok := req.Data.(map[string]any)
	if !ok {
		return admin.NewInvalidAdminReqFormatError("expected map[string]any")
	}

	if len(mval) != 1 {
		return admin.NewInvalidAdminReqErrorf("data field must have 1 entry, got %d", len(mval))
	}

	var (
		configName  string
		configValue any
	)
	// select the single name/value pair from the map
	for k, v := range mval {
		configName = k
		configValue = v
		break
	}

	field, ok := s.configs.GetField(configName)
	if !ok {
		return admin.NewInvalidAdminReqErrorf("unknown config field: %s", configName)
	}

	// we have found a corresponding updatable config field, set it in the ValidatorData
	// field - we will attempt to apply the change in Handler
	req.ValidatorData = validatedSetConfigData{
		field: field,
		value: configValue,
	}
	return nil
}
