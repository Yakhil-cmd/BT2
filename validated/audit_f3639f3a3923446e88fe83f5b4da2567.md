### Title
Unprivileged staker can monopolize `update_rewards` per-block slot to steal all consensus block rewards — (`src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the Staking contract is callable by any non-zero address for any valid staker. A single global `last_reward_block` variable gates the function to one successful call per block. Because there is no access control restricting who may call it or for which staker, an attacker who is a valid staker can call `update_rewards` for themselves in every block, monopolizing all block-reward slots and causing every legitimate consensus-mechanism call for other stakers to revert with `REWARDS_ALREADY_UPDATED`.

---

### Finding Description

`update_rewards` is the V3 consensus-rewards entry point. Each block, the off-chain consensus mechanism is expected to call it once for the staker designated to receive that block's rewards. The function enforces the one-call-per-block invariant through a single global storage variable:

```cairo
// src/staking/staking.cairo  lines 1453-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
...
self.last_reward_block.write(current_block_number);
``` [1](#0-0) 

The only guard before this check is `general_prerequisites`, which only verifies the contract is unpaused and the caller is non-zero:

```cairo
// src/staking/staking.cairo  lines 1793-1797
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [2](#0-1) 

There is no check that the caller is the consensus mechanism, nor that `staker_address` is the staker designated for this block. Any valid staker can therefore call `update_rewards(self_address, false)` in any block.

When the function succeeds it writes `last_reward_block = current_block_number`. Any subsequent call in the same block — including the legitimate consensus call for a different staker — hits the assertion and reverts. The attacker's staker receives the full block rewards for that block; the intended staker receives nothing.

Block rewards are distributed as:

```cairo
// src/staking/staking.cairo  lines 1918-1923
mul_wide_and_div(
    lhs: strk_total_rewards,
    rhs: own_balance_curr_epoch.to_strk_native_amount(),
    div: strk_total_stake.to_strk_native_amount(),
)
``` [3](#0-2) 

where `strk_total_stake` is the calling staker's own total balance (own + delegated), not the protocol-wide total. Each call therefore distributes the entire block-reward budget to one staker. By calling in every block, the attacker captures 100 % of all block rewards regardless of their proportional stake.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

In normal operation each staker receives rewards proportional to their staking power across blocks. An attacker with, say, 1 % of total stake who calls `update_rewards` in every block receives 1 % of each block's reward budget every block, accumulating 100 % of all emitted rewards over time. Every other staker receives zero. The stolen amount scales with total protocol inflation and is unbounded over time.

---

### Likelihood Explanation

Any registered staker can execute this attack with a simple bot that submits `update_rewards(attacker_address, false)` once per block. The only cost is transaction fees. As Starknet moves toward decentralized sequencing, MEV-style ordering makes reliable front-running straightforward. Even under the current centralized sequencer, a staker can submit the call at the start of every block without needing to observe the consensus transaction first.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., a designated consensus contract address stored in the Staking contract), analogous to how `update_rewards_from_attestation_contract` is restricted:

```cairo
// src/staking/staking.cairo  lines 2219-2225
fn assert_caller_is_attestation_contract(self: @ContractState) {
    assert!(
        get_caller_address() == self.attestation_contract.read(),
        "{}",
        Error::CALLER_IS_NOT_ATTESTATION_CONTRACT,
    );
}
``` [4](#0-3) 

Add an analogous `assert_caller_is_consensus_contract` check at the top of `update_rewards`, and store the consensus contract address in storage during construction/configuration.

---

### Proof of Concept

1. Attacker registers as a valid staker (`stake(...)`) with any amount ≥ `min_stake`.
2. Protocol transitions to V3 consensus rewards (`consensus_rewards_first_epoch` is set).
3. Each block, before the consensus mechanism submits its transaction, the attacker's bot calls:
   ```
   update_rewards(attacker_staker_address, false)
   ```
4. `last_reward_block` is set to the current block number; attacker's staker accrues `block_rewards * own_balance / total_balance` in `unclaimed_rewards_own`.
5. The consensus mechanism's call `update_rewards(intended_staker, false)` reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number == last_reward_block`.
6. Repeated every block: attacker accumulates all emitted block rewards; all other stakers accumulate zero. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
