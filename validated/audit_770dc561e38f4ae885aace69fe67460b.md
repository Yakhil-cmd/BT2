### Title
Immutable `starkgate_address` in `RewardSupplier` Permanently Blocks `on_receive`, Freezing Reward Delivery - (File: src/reward_supplier/reward_supplier.cairo)

---

### Summary

`RewardSupplier` stores `starkgate_address` as a plain storage variable written once in the constructor. It is used as the sole access guard in `on_receive`, the callback that StarkGate invokes when minted STRK arrives from L1. No setter exists anywhere in the contract or its interface. If the StarkGate bridge address ever changes (upgrade, migration, or redeployment), every future `on_receive` call will revert, permanently severing the L1→L2 reward-token pipeline and freezing all unclaimed staker yield.

---

### Finding Description

In `src/reward_supplier/reward_supplier.cairo`, the constructor writes `starkgate_address` once:

```cairo
// constructor, line 132
self.starkgate_address.write(starkgate_address);
``` [1](#0-0) 

`on_receive` then uses it as a hard gate:

```cairo
// on_receive, line 233-237
assert!(
    get_caller_address() == self.starkgate_address.read(),
    "{}",
    Error::ON_RECEIVE_NOT_FROM_STARKGATE,
);
``` [2](#0-1) 

The `IRewardSupplierConfig` interface exposes only `set_block_duration_config`; there is no `set_starkgate_address` or equivalent: [3](#0-2) 

The same pattern exists for `l1_reward_supplier` in `MintingCurve`: written in the constructor and used as the sole guard in the `update_total_supply` L1 handler, with no setter in `IMintingCurveConfig`: [4](#0-3) [5](#0-4) [6](#0-5) 

---

### Impact Explanation

`on_receive` is the only entry point through which minted STRK tokens bridged from L1 reach the `RewardSupplier`. When it reverts:

1. `l1_pending_requested_amount` is never decremented, so the credit/debit accounting in `request_funds` becomes permanently skewed.
2. No new STRK can be deposited into the contract.
3. The existing balance drains as stakers call `claim_rewards` via the staking contract.
4. Once the balance is exhausted, `claim_rewards` reverts for every staker.

This constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

StarkGate has been upgraded on Starknet mainnet before and is expected to be upgraded again as the protocol matures. The `RewardSupplier` has no mechanism to track or react to such a change. Because the staking protocol is intended to run indefinitely, the probability that the hardcoded address becomes stale over the contract's lifetime is non-trivial. The secondary instance (`l1_reward_supplier` in `MintingCurve`) carries the same risk for the `update_total_supply` L1 handler, causing `total_supply` to become permanently stale and distorting `yearly_mint` for all reward calculations.

---

### Recommendation

1. Add a privileged setter (e.g., `set_starkgate_address`, restricted to `app_governor` or `token_admin`) to `RewardSupplier` and expose it in `IRewardSupplierConfig`.
2. Add a corresponding `set_l1_reward_supplier` setter to `MintingCurve` and expose it in `IMintingCurveConfig`.
3. Emit an event on each update for auditability.

---

### Proof of Concept

**State before StarkGate upgrade:**
- `starkgate_address` = `0xOLD_GATE`
- StarkGate at `0xOLD_GATE` calls `on_receive` → succeeds, `l1_pending_requested_amount` decrements normally.

**After StarkGate migrates to `0xNEW_GATE`:**
1. L1 reward supplier mints STRK and bridges it via `0xNEW_GATE`.
2. `0xNEW_GATE` calls `RewardSupplier::on_receive(...)`.
3. `get_caller_address()` returns `0xNEW_GATE`; `self.starkgate_address.read()` returns `0xOLD_GATE`.
4. `assert!` fires → `"Only StarkGate can call on_receive"` → revert.
5. Deposit fails; `l1_pending_requested_amount` is never decremented.
6. `request_funds` keeps issuing new mint requests (credit never grows), flooding L1 with messages that also fail.
7. Existing STRK balance drains epoch by epoch as stakers claim; once empty, `claim_rewards` reverts for all stakers → all unclaimed yield is frozen. [9](#0-8)

        // This function is called by the staking contract, claiming an amount of owed rewards.
        fn claim_rewards(ref self: ContractState, amount: Amount) {
            // Asserts.
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            let unclaimed_rewards = self.unclaimed_rewards.read();
            assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Update unclaimed_rewards and transfer the requested rewards to the staking contract.
            self.unclaimed_rewards.write(unclaimed_rewards - amount);
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
        }

        fn on_receive(
            ref self: ContractState,
            l2_token: ContractAddress,
            amount: u256,
            depositor: EthAddress,
            message: Span<felt252>,
        ) -> bool {
            // Note that the deposit can be done by anyone (not just the L1 reward supplier), so
            // depositor is not checked.

            // These messages accepted only from the token bridge.
            assert!(
                get_caller_address() == self.starkgate_address.read(),
                "{}",
                Error::ON_RECEIVE_NOT_FROM_STARKGATE,
            );
            // The bridge may serve multiple tokens, only the correct token may be received.
            assert!(
                l2_token == self.token_dispatcher.contract_address.read(),
                "{}",
                Error::UNEXPECTED_TOKEN,
            );
            let amount_u128: Amount = amount
                .try_into()
                .expect_with_err(GenericError::AMOUNT_TOO_HIGH);
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            if amount_u128 > l1_pending_requested_amount {
                self.l1_pending_requested_amount.write(Zero::zero());
            } else {
                l1_pending_requested_amount -= amount_u128;
                self.l1_pending_requested_amount.write(l1_pending_requested_amount);
            }
            true
        }

        /// Requests funds from L1 to account for new rewards, if the contract's balance is too low.
        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }

    /// Message updating the total supply, sent by the L1 reward supplier.
    #[l1_handler]
    fn update_total_supply(ref self: ContractState, from_address: felt252, total_supply: Amount) {
        assert!(
            from_address == self.l1_reward_supplier.read(),
            "{}",
            Error::UNAUTHORIZED_MESSAGE_SENDER,
        );
        let old_total_supply = self.total_supply.read();
        // Note that the total supply may only increase.
        // Check that total_supply > old_total_supply to handle possible message reordering.
        if total_supply > old_total_supply {
            self.total_supply.write(total_supply);
            self
                .emit(
                    Events::TotalSupplyChanged { old_total_supply, new_total_supply: total_supply },
                );
        }
    }
