### Title
Stale Delegated Permissions Persist After `CronosAdmin` Is Cleared, Allowing Unauthorized `MsgUpdateTokenMapping` — (`x/cronos/keeper/permissions.go`, `x/cronos/keeper/msg_server.go`)

### Summary

When governance clears `CronosAdmin` (sets it to `""`) via `MsgUpdateParams`, previously-delegated `CanChangeTokenMapping` permission bits stored in the KV store are never revoked. `HasPermission` checks stored bits independently of whether an admin exists, so any address that was previously granted `CanChangeTokenMapping` can still successfully call `MsgUpdateTokenMapping` indefinitely after admin is cleared.

### Finding Description

`Params.Validate` explicitly allows an empty `CronosAdmin`:

```go
// x/cronos/types/params.go:70-74
if len(p.CronosAdmin) > 0 {
    if _, err := sdk.AccAddressFromBech32(p.CronosAdmin); err != nil {
        return err
    }
}
```

`HasPermission` has two independent authorization paths:

```go
// x/cronos/keeper/permissions.go:37-46
admin := k.GetParams(ctx).CronosAdmin
for _, account := range accounts {
    if admin == account.String() {   // path 1: admin match
        return true
    }
    permission := k.GetPermissions(ctx, account)
    if permission&permissionsToCheck == permissionsToCheck { // path 2: KV bits
        return true
    }
}
```

When admin is cleared to `""`, path 1 never fires (no valid bech32 address equals `""`). But path 2 still reads from the KV store, which is never cleared. `SetPermissions` writes to KV; nothing in `UpdateParams`, `SetParams`, or any other code path clears those entries when admin changes.

`UpdatePermissions` and `StoreBlockList` both gate on `admin != msg.From`, so once admin is empty they are permanently locked — no one can revoke the stale bits either.

### Impact Explanation

An address that was previously granted `CanChangeTokenMapping` retains the ability to call `MsgUpdateTokenMapping` and arbitrarily remap denom-to-contract bindings after governance has cleared the admin. This is a direct bypass of the intended authorization revocation and constitutes corruption of token mappings / denom-contract bindings with direct security impact (IBC voucher conversion, CRC20 auto-deployment, and bridge accounting all depend on correct token mappings).

### Likelihood Explanation

Governance clearing `CronosAdmin` is a documented, supported operation ("empty means no admin"). Any operator who clears admin expecting a clean slate will unknowingly leave stale permission holders active. The stale holders need only submit a standard `MsgUpdateTokenMapping` transaction — no special tooling required.

### Recommendation

When `UpdateParams` changes `CronosAdmin`, iterate over all stored permission entries and clear them (or set to `0`). Alternatively, add a guard in `HasPermission` that returns `false` for the KV-bit path when `admin == ""`, treating an empty admin as "permission system disabled." The simplest safe fix is to wipe all stored permissions inside `SetParams` whenever `CronosAdmin` changes.

### Proof of Concept

1. Governance submits `MsgUpdateParams` with a valid `CronosAdmin = addrAdmin`.
2. `addrAdmin` submits `MsgUpdatePermissions` granting `CanChangeTokenMapping` (bit `1`) to `addrA`.
3. Governance submits `MsgUpdateParams` with `CronosAdmin = ""`.
4. `addrA` submits `MsgUpdateTokenMapping` — `HasPermission` reads KV bit `1` for `addrA`, returns `true`, and the mapping update succeeds.
5. No one can revoke `addrA`'s permission because `UpdatePermissions` requires a non-empty admin match.

```go
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

```go
func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
}

func (k msgServer) StoreBlockList(goCtx context.Context, msg *types.MsgStoreBlockList) (*types.MsgStoreBlockListResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	ctx.KVStore(k.storeKey).Set(types.KeyPrefixBlockList, msg.Blob)
	return &types.MsgStoreBlockListResponse{}, nil
}
```
