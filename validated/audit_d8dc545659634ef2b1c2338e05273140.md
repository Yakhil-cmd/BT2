### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards - (File: src/staking/staking.cairo)

### Summary
`StakingRewardsManager::update_rewards` is documented as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block reward slot and permanently discarding that block's rewards for all stakers and delegators.

### Finding Description
The specification for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation only calls `general_prerequisites()`, which checks two things: the contract is not paused, and the caller is not the zero address. [1](#0-0) 

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

There is no check that `get_caller_address()` equals the sequencer address. The full `update_rewards` entry point: [2](#0-1) 

The critical state mutation is at line 1485: [3](#0-2) 

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

When `disable_rewards = true`, the function:
1. Writes `current_block_number` into `last_reward_block`
2. Returns immediately — no rewards are calculated, no `update_unclaimed_rewards_from_staking_contract` is called, no pool rewards are forwarded

The guard that prevents double-calling is: [4](#0-3) 

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

Once any caller has written `last_reward_block = N`, the legitimate sequencer's call at block `N` will revert with `REWARDS_ALREADY_UPDATED`. The rewards for block `N` are permanently lost.

### Impact Explanation
An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` at block `N` before the sequencer:

- Permanently discards all STRK and BTC block rewards that would have been distributed at block `N` to the targeted staker and all delegators in their pools.
- The sequencer cannot recover those rewards; `last_reward_block` is already set to `N`.
- Repeated across every block, this permanently freezes all consensus-era yield for every staker.

This maps to the allowed impact: **High — Permanent freezing of unclaimed yield**.

The spec confirms the intended design: [5](#0-4) 

> **access control**: Only starkware sequencer.

### Likelihood Explanation
- No funds, no privileged role, and no special setup are required.
- The call is cheap (gas cost only).
- The attacker only needs to submit a transaction before the sequencer's `update_rewards` transaction lands in the same block.
- On Starknet the sequencer controls ordering, but the sequencer role is not enforced on-chain, so any address — including a malicious or compromised one — can exploit this.
- The `staker_address` argument can be any currently active staker; the attacker does not need to know a specific target.

### Recommendation
Add an explicit sequencer-only access control check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract`: [6](#0-5) 

```cairo
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), ...);
    self.assert_caller_is_attestation_contract();
    ...
}
```

Apply the same pattern to `update_rewards`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this guard
    ...
}
```

Store the sequencer address in contract storage (set at construction or via a privileged config function) and verify `get_caller_address() == self.sequencer_address.read()`.

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`).
2. Attacker (any EOA) calls `update_rewards(active_staker_address, disable_rewards: true)` at block `N`.
3. `last_reward_block` is written to `N`; function returns with no rewards distributed.
4. Sequencer calls `update_rewards(active_staker_address, disable_rewards: false)` at block `N` → reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker's `unclaimed_rewards_own` is unchanged; pool contracts receive no reward transfer for block `N`.
6. Attacker repeats every block → stakers and delegators accumulate zero consensus rewards indefinitely. [7](#0-6) [8](#0-7)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
