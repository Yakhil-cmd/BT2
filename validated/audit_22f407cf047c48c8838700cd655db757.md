### Title
Global `last_reward_block` Consumed by Any Caller Permanently Freezes Staker Yield - (File: `src/staking/staking.cairo`)

### Summary

`update_rewards` in `staking.cairo` uses a single **global** `last_reward_block` storage variable to gate reward distribution. There is no access control on the function. Any unprivileged caller can invoke `update_rewards(staker_address: valid_staker, disable_rewards: true)` in every block, consuming the global block slot without distributing rewards, permanently preventing any staker from receiving yield for those blocks.

### Finding Description

`update_rewards` is gated by a single global `last_reward_block` field:

```cairo
// src/staking/staking.cairo (storage struct)
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [1](#0-0) 

The function reads and writes this single global value:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    ...
}
``` [2](#0-1) 

`general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero — **no sequencer-only access control is enforced**:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [3](#0-2) 

The spec documents the intended access control as "Only starkware sequencer" but the code does not implement it. [4](#0-3) 

**Analog to M-15:** In M-15, `totalDepositedAmountPerUser[L1WethBridge]` is a shared counter for all depositors — once the bridge's cumulative limit is hit, no individual user can deposit. Here, `last_reward_block` is a shared counter for all stakers — once any caller consumes it in a block (with `disable_rewards: true`), no staker can receive rewards for that block.

### Impact Explanation

An attacker calling `update_rewards(staker_address: any_valid_staker, disable_rewards: true)` in every block:

1. Sets `last_reward_block` to the current block number.
2. Causes the legitimate sequencer's subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`.
3. Since `disable_rewards: true` skips reward distribution, **zero rewards are minted for that block**.
4. Block rewards are never retroactively recoverable — the yield for each griefed block is permanently lost.

This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators across the entire protocol.

### Likelihood Explanation

- The entry point (`update_rewards`) is publicly callable by any non-zero address.
- The attacker only needs to front-run the sequencer once per block with a valid staker address and `disable_rewards: true`.
- The cost is only Starknet transaction fees; there is no capital requirement.
- A valid staker address is trivially discoverable from on-chain events (`NewStaker`).

### Recommendation

Enforce that only the authorized sequencer address can call `update_rewards`, consistent with the spec's stated access control. Add a check analogous to `assert_caller_is_attestation_contract` used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
```

Alternatively, track `last_reward_block` per staker (`Map<ContractAddress, BlockNumber>`) so that one staker's update cannot block another's — though access control is the primary fix.

### Proof of Concept

1. Consensus rewards are active; stakers A and B both have effective balances.
2. In block N, attacker calls `update_rewards(staker_address: A, disable_rewards: true)`.
   - `last_reward_block` is written to N.
   - No rewards are distributed (early return due to `disable_rewards: true`).
3. The sequencer attempts `update_rewards(staker_address: B, disable_rewards: false)` in block N.
   - `assert!(N > last_reward_block.read())` → `assert!(N > N)` → **reverts with `REWARDS_ALREADY_UPDATED`**.
4. Staker B receives zero rewards for block N. The attacker repeats this every block.
5. All stakers' unclaimed yield is permanently frozen. [5](#0-4) [6](#0-5)

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,

            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);

                "{}",

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
