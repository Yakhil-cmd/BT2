### Title
Unrestricted `DeployGlobalContractByAccountId` Allows Any Contract to Overwrite Another Account's Global Contract Namespace - (File: runtime/runtime/src/global_contracts.rs)

### Summary

`action_deploy_global_contract` contains no ownership check when `deploy_mode = AccountId`. Any contract can create a cross-contract promise targeting a victim account and append a `DeployGlobalContractByAccountId` action, overwriting the global contract stored under the victim's account ID. Every account that has called `UseGlobalContract(AccountId(victim))` will silently execute the attacker's code on the next invocation.

### Finding Description

`GlobalContractDeployMode::AccountId` is documented as an owner-controlled namespace: the contract is stored under the deployer's account ID so that the owner can update it for all users. The invariant is that only the account owner can write to `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(owner) }`.

`action_deploy_global_contract` enforces only a balance check: [1](#0-0) 

`initiate_distribution` derives the trie key directly from `account_id`, which is the receipt's `receiver_id`, not the original signer: [2](#0-1) 

The host function that appends this action to a cross-contract promise checks only `is_view()`: [3](#0-2) 

The action validator checks only code size: [4](#0-3) 

There is no guard that `receipt.predecessor_id()` or `receipt.signer_id()` equals `account_id` (the receiver) when `deploy_mode = AccountId`. A malicious contract on `attacker.near` can call `promise_batch_create("victim.near")` followed by `promise_batch_action_deploy_global_contract_by_account_id(promise_idx, malicious_wasm)`. The resulting receipt is executed on `victim.near`'s shard with `account_id = victim.near`, so the global contract trie key `GlobalContractCodeIdentifier::AccountId("victim.near")` is overwritten with the attacker's WASM. The storage cost is deducted from `victim.near`'s balance.

### Impact Explanation

Every account that previously executed `UseGlobalContract(AccountId("victim.near"))` has `AccountContract::GlobalByAccount("victim.near")` stored in its account record: [5](#0-4) 

After the overwrite, those accounts resolve to the attacker's WASM at call time. The corrupted value is `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim) }` on every shard (the distribution receipt propagates globally). This is a global code-identity substitution affecting all users of the victim's global contract — Critical severity.

### Likelihood Explanation

The attack requires only a funded account and a deployed contract. No validator, admin, or privileged role is needed. The attacker pays only gas; storage cost is charged to the victim. The path is reachable on mainnet today via any contract that calls `promise_batch_action_deploy_global_contract_by_account_id`.

### Recommendation

In `action_deploy_global_contract`, when `deploy_mode = AccountId`, assert that the receipt's `predecessor_id` equals `account_id` (i.e., the action originates from the account itself, not a cross-contract call):

```rust
if deploy_contract.deploy_mode == GlobalContractDeployMode::AccountId {
    // Only the account itself may register a global contract under its own ID.
    if receipt.predecessor_id() != account_id {
        result.result = Err(ActionErrorKind::ActorNoPermission {
            account_id: account_id.clone(),
            actor_id: receipt.predecessor_id().clone(),
        }.into());
        return Ok(());
    }
}
```

Alternatively, enforce this at the action-validation layer so it is rejected before execution.

### Proof of Concept

1. Attacker deploys a contract on `attacker.near` containing:
   ```rust
   pub fn attack() {
       let promise = env::promise_batch_create("victim.near");
       env::promise_batch_action_deploy_global_contract_by_account_id(
           promise,
           MALICIOUS_WASM,
       );
   }
   ```
2. Attacker calls `attacker.near::attack()`.
3. The runtime creates a receipt with `receiver_id = victim.near`, `predecessor_id = attacker.near`, action = `DeployGlobalContract { code: MALICIOUS_WASM, deploy_mode: AccountId }`.
4. `action_deploy_global_contract` is called with `account_id = victim.near`. No ownership check fires.
5. `initiate_distribution` stores `GlobalContractIdentifier::AccountId("victim.near")` → `MALICIOUS_WASM` in the trie and emits a distribution receipt that propagates to all shards.
6. All accounts holding `AccountContract::GlobalByAccount("victim.near")` now execute `MALICIOUS_WASM`. [6](#0-5) [7](#0-6)

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())
