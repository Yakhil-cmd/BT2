### Title
Unpermissioned `update_rewards` with Attacker-Controlled `disable_rewards` Allows Permanent Freezing of All Staker Consensus Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts an arbitrary `staker_address` and a caller-controlled `disable_rewards` flag. When called with `disable_rewards: true`, it advances the global `last_reward_block` to the current block without distributing any rewards, then returns early. Because the guard `current_block_number > self.last_reward_block.read()` is checked globally (not per-staker), any subsequent legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`. An attacker can repeat this every block at negligible cost, permanently preventing all stakers from accumulating consensus block rewards.

---

### Finding Description

`update_rewards` is defined in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global write, always executed

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // early return, no rewards distributed
    }
    ...
    self._update_rewards(...);
}
``` [1](#0-0) 

The storage field `last_reward_block` is a single contract-wide value, not per-staker: [2](#0-1) 

`general_prerequisites` imposes no caller identity check: [3](#0-2) 

The public interface exposes `update_rewards` with no access-control annotation: [4](#0-3) 

**Attack path:**

1. Attacker monitors the mempool / block production.
2. At the start of every block (or just before the legitimate consensus call), attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)`.
3. The function passes the `current_block_number > last_reward_block` guard (first call in the block), writes `last_reward_block = current_block_number`, then returns early — zero rewards distributed.
4. Every subsequent call to `update_rewards` in the same block (by the legitimate consensus mechanism or any staker) reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block, no staker ever accumulates consensus block rewards.

The analog to the Shardeum `setCertTime` bug is exact:
- **Shardeum**: anyone can submit a `setCertTime` tx for a victim node; the system checks signature validity but not that the signer owns the nominee key → attacker extends cert, drains fees, locks stake.
- **Starknet Staking**: anyone can call `update_rewards` for any staker with `disable_rewards: true`; the system checks the caller is non-zero but not that the caller is authorized to control reward distribution → attacker poisons the global `last_reward_block`, freezing all yield.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

All stakers' consensus block rewards are permanently suppressed. `unclaimed_rewards_own` is only incremented inside `_update_rewards`: [5](#0-4) 

If `_update_rewards` is never reached (because `disable_rewards: true` causes an early return every block), stakers accumulate zero rewards indefinitely. Pool delegators are equally affected because pool rewards flow through the same path: [6](#0-5) 

---

### Likelihood Explanation

**High.** The attack requires only:
- A valid (non-zero) caller address.
- Knowledge of any active staker address (publicly readable from `stakers` vec or events).
- Gas to submit one transaction per block.

No privileged role, no leaked key, no external dependency. The attacker has no financial cost beyond gas and gains a griefing outcome (or competitive advantage if they are themselves a staker whose rivals are frozen out). The attack is trivially automatable.

---

### Recommendation

1. **Restrict the caller.** `update_rewards` should only be callable by the attestation contract or a designated consensus contract, mirroring the pattern already used for `update_rewards_from_attestation_contract`: [7](#0-6) 

2. **Make `last_reward_block` per-staker**, or remove the `disable_rewards` parameter from the public interface and handle the "no reward" case internally based on verified attestation data.

3. **Remove caller control over `disable_rewards`.** The decision of whether a staker deserves rewards for a block should be made by the trusted attestation/consensus contract, not by an arbitrary caller.

---

### Proof of Concept

```python
# Pseudocode — callable by any EOA each block
staking_contract.update_rewards(
    staker_address = any_known_active_staker,  # readable from public state
    disable_rewards = True,                    # attacker-controlled flag
    caller = attacker_address                  # no restriction
)
# Result: last_reward_block = current_block, no rewards distributed.
# All subsequent update_rewards calls in this block revert with REWARDS_ALREADY_UPDATED.
# Repeat every block → all stakers earn 0 consensus rewards indefinitely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
