### Title
Insufficient Reward Supplier Token Balance Causes `unstake_action` and `claim_rewards` to Permanently Revert, Temporarily Freezing Staker Principal and Yield - (File: `src/reward_supplier/reward_supplier.cairo`)

---

### Summary

The `RewardSupplier` contract tracks owed rewards via an accounting variable (`unclaimed_rewards`) and separately requests actual tokens from L1 via a bridge. During the L1→L2 bridge delay, the contract's real token balance can be less than `unclaimed_rewards`. Any call to `reward_supplier.claim_rewards(amount)` with a non-zero `amount` will then revert at the `checked_transfer` step. Because `unstake_action` and `claim_rewards` in the staking contract unconditionally call through this path, stakers who have already completed the exit wait window are blocked from retrieving their principal, and all reward claims are frozen until L1 funds arrive.

---

### Finding Description

**Root cause chain:**

`unstake_action` (staking.cairo:495) calls `send_rewards_to_staker` (staking.cairo:1614–1628), which calls `claim_from_reward_supplier` (utils.cairo:50–60), which calls `reward_supplier_dispatcher.claim_rewards(amount)` (utils.cairo:57). Inside `reward_supplier.claim_rewards` (reward_supplier.cairo:205–220), after passing the accounting check `amount <= unclaimed_rewards` (line 214), the function calls:

```cairo
token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
``` [1](#0-0) 

The `unclaimed_rewards` storage variable is a pure accounting counter — it is incremented whenever `update_unclaimed_rewards_from_staking_contract` is called, but the actual STRK token balance of the `RewardSupplier` contract is only replenished when the L1 bridge delivers tokens in response to a `MintRequest` message. The `request_funds` function (reward_supplier.cairo:301–331) sends L1 messages but does not guarantee the balance is immediately available. [2](#0-1) 

The gap between `unclaimed_rewards` (accounting) and the real token balance is explicitly tracked as `l1_pending_requested_amount`. During the entire L1→L2 bridge delay, `actual_balance < unclaimed_rewards` is a normal operating state. [3](#0-2) 

`claim_from_reward_supplier` in utils.cairo adds a second layer of enforcement: it asserts that the staking contract's balance increased by exactly `amount` after the call, so even a partial transfer would revert. [4](#0-3) 

**Blocked operations:**

1. `unstake_action` — calls `send_rewards_to_staker` unconditionally before returning the staker's principal. If the staker has any `unclaimed_rewards_own > 0`, the entire transaction reverts and the staker cannot recover their principal even after the exit wait window has elapsed. [5](#0-4) 

2. `claim_rewards` (staking) — calls `send_rewards_to_staker` for the same path. [6](#0-5) 

3. `send_rewards_to_staker` — the shared internal function that always calls `claim_from_reward_supplier` with the full `unclaimed_rewards_own` amount, with no zero-amount short-circuit guard that would skip the external call. [7](#0-6) 

---

### Impact Explanation

- **Temporary freezing of staker principal**: A staker who has signaled `unstake_intent`, waited the full exit window, and is entitled to call `unstake_action` is blocked from recovering their staked tokens for as long as the reward_supplier's actual balance is below `unclaimed_rewards`. The staker's principal remains locked in the staking contract with no alternative withdrawal path.
- **Temporary freezing of unclaimed yield**: All `claim_rewards` calls revert for any staker with non-zero `unclaimed_rewards_own` during the same window.

Both match the allowed impact: **Temporary freezing of funds** and **Temporary freezing of unclaimed yield** (High severity).

---

### Likelihood Explanation

The L1→L2 bridge delay is a structural, recurring property of the system — not an edge case. Every epoch in which rewards are accrued triggers a `MintRequest` to L1, and the corresponding tokens arrive only after the L1 transaction is confirmed and the StarkGate bridge processes the deposit. During this window (which can span multiple Starknet epochs), the reward_supplier's real balance is below `unclaimed_rewards`. Any staker attempting `unstake_action` or `claim_rewards` during this window with non-zero accrued rewards will be blocked. The likelihood is **high** because this is a normal, expected operating state of the system, not a rare race condition.

---

### Recommendation

Add a zero-amount short-circuit in `send_rewards_to_staker` so that if `unclaimed_rewards_own == 0`, the external call to `reward_supplier.claim_rewards` is skipped entirely. More importantly, decouple reward claiming from the principal-return path in `unstake_action`. Specifically:

1. Allow `unstake_action` to succeed and return the staker's principal even when the reward transfer would revert, by separating the reward claim into a distinct, independently callable step (similar to the `emergencyWithdraw` pattern in the referenced report).
2. Alternatively, if `claim_from_reward_supplier` fails due to insufficient balance, catch the failure and defer the reward transfer rather than reverting the entire `unstake_action`.

---

### Proof of Concept

1. Staker stakes and accrues rewards over several epochs (so `unclaimed_rewards_own > 0`).
2. Staker calls `unstake_intent()`. The exit wait window begins.
3. During the wait window, the staking contract calls `update_unclaimed_rewards_from_staking_contract` on the reward_supplier, incrementing `unclaimed_rewards` and emitting a `MintRequest` to L1. The L1 bridge has not yet delivered the tokens, so `reward_supplier.balance < unclaimed_rewards`.
4. The exit wait window elapses. Staker calls `unstake_action(staker_address)`.
5. `unstake_action` calls `send_rewards_to_staker` → `claim_from_reward_supplier` → `reward_supplier.claim_rewards(amount)`.
6. Inside `reward_supplier.claim_rewards`, `amount <= unclaimed_rewards` passes (accounting is correct), but `token_dispatcher.checked_transfer(staking_contract, amount)` reverts because the actual token balance is insufficient.
7. The entire `unstake_action` transaction reverts. The staker's principal remains locked. The staker must retry after the L1 bridge delivers the tokens — with no guaranteed timeline and no alternative exit path. [8](#0-7) [4](#0-3) [9](#0-8)

        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }

}

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn send_rewards_to_staker(
            ref self: ContractState,
            staker_address: ContractAddress,
            ref staker_info: InternalStakerInfoLatest,
            token_dispatcher: IERC20Dispatcher,
        ) {
            let reward_address = staker_info.reward_address;
            let amount = staker_info.unclaimed_rewards_own;
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();

            claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
            token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
            staker_info.unclaimed_rewards_own = Zero::zero();

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
