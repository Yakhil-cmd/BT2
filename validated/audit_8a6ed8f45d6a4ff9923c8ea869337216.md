### Title
Staker's `reward_address` Lacks Zero-Address Validation, Blocking `unstake_action` Principal Recovery - (File: `src/staking/staking.cairo`)

---

### Summary

`unstake_action` uses a push-payment pattern to send accumulated rewards to `reward_address` **before** returning the staker's principal. Neither `stake()` nor `change_reward_address()` validate that `reward_address` is non-zero. If a staker registers or updates to a zero `reward_address`, the STRK token's zero-address guard causes `unstake_action` to revert, temporarily freezing the staker's principal in the contract.

---

### Finding Description

`unstake_action` in `src/staking/staking.cairo` calls the internal helper `send_rewards_to_staker` as the **first** step before returning the principal:

```cairo
fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    ...
    let token_dispatcher = strk_token_dispatcher();
    self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
    ...
    // Return stake to staker.
    token_dispatcher.checked_transfer(recipient: staker_address, amount: staker_amount.into());
    ...
}
``` [1](#0-0) 

`send_rewards_to_staker` unconditionally calls `checked_transfer` to `reward_address`:

```cairo
fn send_rewards_to_staker(...) {
    let reward_address = staker_info.reward_address;
    let amount = staker_info.unclaimed_rewards_own;
    claim_from_reward_supplier(...);
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
    ...
}
``` [2](#0-1) 

Both entry points that set `reward_address` only guard against the address being a registered token address — **there is no zero-address check**:

`stake()`:
```cairo
assert!(
    !self.does_token_exist(token_address: reward_address),
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
``` [3](#0-2) 

`change_reward_address()`:
```cairo
assert!(
    !self.does_token_exist(token_address: reward_address),
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
``` [4](#0-3) 

The spec also lists only `REWARD_ADDRESS_IS_TOKEN` as an error for `change_reward_address`, confirming no zero-address guard exists: [5](#0-4) 

The OpenZeppelin ERC20 used for STRK reverts on any transfer to the zero address (including zero-amount transfers), so `checked_transfer(recipient: 0x0, amount: ...)` always panics. Because `send_rewards_to_staker` is called before the principal transfer, the entire `unstake_action` reverts and the staker's principal remains locked.

---

### Impact Explanation

A staker whose `reward_address` is the zero address cannot complete `unstake_action`. Their principal (which may be the minimum stake or far more) is temporarily frozen in the staking contract. The staker can recover by calling `change_reward_address` with a valid address and then retrying `unstake_action`, so the freeze is **temporary** rather than permanent. This matches the allowed impact: **Temporary freezing of funds**.

---

### Likelihood Explanation

- A staker can supply `reward_address = 0x0` at `stake()` time or later via `change_reward_address()` — both are accepted by the contract.
- A staker might do this accidentally (e.g., passing a default/uninitialized value), or a staker whose reward address is a smart contract that later becomes non-functional (paused, self-destructed equivalent, or upgraded to revert on receipt) faces the same DoS.
- The condition is reachable by any unprivileged staker with no external dependencies.
- Likelihood is **low-to-medium**: it requires a user error or a degraded reward-address contract, but the missing guard makes it a latent footgun.

---

### Recommendation

Add a non-zero address check in both `stake()` and `change_reward_address()` in `src/staking/staking.cairo`, and in `change_reward_address()` in `src/pool/pool.cairo`:

```cairo
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

Alternatively, adopt the withdrawal-pattern analog: decouple reward distribution from principal return in `unstake_action` so that a failing reward transfer does not block the return of principal.

---

### Proof of Concept

1. Staker calls `stake(reward_address: 0x0, ...)` — accepted, no revert.
2. Staker accrues rewards over several epochs (`unclaimed_rewards_own > 0`).
3. Staker calls `unstake_intent()` — succeeds, sets `unstake_time`.
4. After the exit window, staker calls `unstake_action(staker_address)`.
5. Inside `unstake_action`, `send_rewards_to_staker` calls `checked_transfer(recipient: 0x0, amount: rewards)`.
6. The STRK token reverts on transfer to zero address.
7. `unstake_action` reverts; the staker's principal remains locked in the staking contract.
8. The staker must first call `change_reward_address(valid_address)`, then retry `unstake_action`.

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn send_rewards_to_staker(
            ref self: ContractState,
            staker_address: ContractAddress,
            ref staker_info: InternalStakerInfoLatest,
            token_dispatcher: IERC20Dispatcher,
        ) {
            let reward_address = staker_info.reward_address;
            let amount = staker_info.unclaimed_rewards_own;
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();

            claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
            token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
            staker_info.unclaimed_rewards_own = Zero::zero();

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
