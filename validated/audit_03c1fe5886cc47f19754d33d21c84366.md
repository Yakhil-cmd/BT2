### Title
Gateway Stateful Validator Admits Transactions Using Stale Previous-Block Gas Prices, Causing Execution Failures on L1 Gas Price Increase - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `run_validate_entry_point` builds its `BlockContext` from the **previous block's** gas prices. The actual execution block uses gas prices set by the consensus orchestrator from the live L1 feed. When the L1 gas price or L1 data gas price rises between the previous block and the current block, transactions that passed gateway admission will fail `check_fee_bounds` at execution time. Compounding this, the explicit `validate_resource_bounds` pre-check only validates `l2_gas.max_price_per_unit` — L1 gas and L1 data gas prices receive no threshold check at all — so the only guard for those two dimensions is the stale-price blockifier call.

### Finding Description

**Step 1 — Explicit threshold check covers only L2 gas.**

`validate_resource_bounds` fetches `strk_gas_prices.l2_gas_price` from the previous block and calls `validate_tx_l2_gas_price_within_threshold`. Inside that function the `L1Gas` arm is a no-op and the `AllResources` arm only inspects `l2_gas.max_price_per_unit`:

```rust
// TODO(Arni): Consider running this validation for all gas prices.

    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // only l2_gas checked
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
}

**Step 2 — `run_validate_entry_point` uses the previous block's gas prices.**

The block context passed to the blockifier validator is constructed from `gateway_fixed_block_state_reader.get_block_info()`, which returns the **latest committed block**. Only the block number is incremented; all gas prices remain those of the previous block:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);  // gas prices = previous block
``` [2](#0-1) 

**Step 3 — `check_fee_bounds` at execution time uses the current block's gas prices.**

When the batcher executes the transaction, `perform_pre_validation_stage` → `check_fee_bounds` compares `l1_gas.max_price_per_unit`, `l1_data_gas.max_price_per_unit`, and `l2_gas.max_price_per_unit` against the **current block's** gas prices set by the consensus orchestrator:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [3](#0-2) 

**Step 4 — The consensus orchestrator sets the current block's L1 gas price from a live feed.**

The consensus orchestrator fetches the live L1 gas price (ETH base fee + blob fee) and ETH→STRK rate at proposal time. These values are independent of the previous block's prices stored in `gateway_fixed_block_state_reader`: [4](#0-3) 

**The gap:** A transaction submitted with `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` equal to the previous block's L1 gas price passes both the explicit `validate_resource_bounds` check (which ignores L1 gas entirely) and the blockifier validation in `run_validate_entry_point` (which uses the previous block's prices). If the L1 gas price rises in the current block — a routine occurrence during Ethereum congestion — the same transaction fails `check_fee_bounds` at execution time with `MaxGasPriceTooLow`, consuming sequencer resources and the user's nonce slot without executing.

### Impact Explanation

This matches **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."** The gateway's admission decision is authoritative: once a transaction is accepted and placed in the mempool, users and downstream components treat it as sequenceable. When L1 gas prices rise between blocks (a normal event), the mempool fills with transactions that will deterministically fail `check_fee_bounds` at execution time. Because `handle_nonce` runs before `check_fee_bounds` inside `perform_pre_validation_stage`, the nonce is incremented before the price check fires, meaning the account's nonce is advanced without a successful execution or fee payment. [5](#0-4) 

### Likelihood Explanation

The L1 gas price scraper polls Ethereum every 120 seconds and maintains a rolling mean over 300 blocks. The consensus orchestrator queries the live rate at each proposal. Ethereum base fees routinely move 12.5% per block during congestion. A transaction admitted with `l1_gas.max_price_per_unit` at the previous block's mean will fail whenever the current block's mean exceeds that value — a condition that occurs in every period of rising Ethereum gas prices. No privileged access is required; any user can submit a transaction with `l1_gas.max_price_per_unit` set to the last known L1 gas price. [6](#0-5) 

### Recommendation

1. **Extend `validate_resource_bounds` to cover all three gas dimensions.** The existing TODO at line 358 acknowledges this gap. Apply the same percentage-threshold check to `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding prices.

2. **Use a forward-looking price estimate in `run_validate_entry_point`.** Rather than passing the raw previous-block `block_info` to the blockifier validator, apply the same fee-transformation multipliers the consensus orchestrator uses when projecting the next block's gas prices. This closes the gap between admission-time and execution-time price checks.

3. **Re-validate price bounds at block-building time.** Before including a mempool transaction in a block proposal, the batcher should re-check `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the current block's gas prices and evict transactions that no longer satisfy the bounds, rather than letting them reach `perform_pre_validation_stage` and fail after nonce increment.

### Proof of Concept

1. Observe the previous block's L1 gas price `P_prev` from `gateway_fixed_block_state_reader.get_block_info().gas_prices.strk_gas_prices.l1_gas_price`.
2. Submit an `InvokeV3` transaction with:
   - `l2_gas.max_price_per_unit` ≥ `min_gas_price_percentage * P_prev_l2` (passes `validate_resource_bounds`)
   - `l1_gas.max_price_per_unit = P_prev` (passes `check_fee_bounds` in `run_validate_entry_point` using previous-block prices)
   - `l1_data_gas.max_price_per_unit = 0` (ignored by `validate_resource_bounds`; passes `check_fee_bounds` only if previous-block L1 data gas price is also 0)
3. The transaction is admitted to the mempool.
4. The consensus orchestrator builds the next block with a live L1 gas price `P_curr > P_prev` (Ethereum congestion).
5. The batcher calls `perform_pre_validation_stage` → `check_fee_bounds` with `P_curr`; `l1_gas.max_price_per_unit = P_prev < P_curr` triggers `MaxGasPriceTooLow`.
6. The transaction fails at execution; the nonce is incremented; no fee is charged; the sequencer has processed an invalid transaction that the gateway should have rejected. [7](#0-6) [8](#0-7)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
