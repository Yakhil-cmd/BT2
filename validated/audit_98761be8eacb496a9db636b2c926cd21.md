### Title
Excess `msg.value` in `register()` is permanently lost to callers — (`File: smart-contracts-poc/contracts/oracles/providers/OracleBase.sol`)

### Summary
`OracleBase.register()` accepts any `msg.value >= registrationFee` but never refunds the surplus. The excess ETH is locked in the contract and is only recoverable by the ADMIN role via `withdrawEth()`. There is no on-chain path for the overpaying caller to reclaim their funds.

### Finding Description
`OracleBase.register()` is a permissionless, payable function that whitelists a pool for a given feed ID:

```solidity
// OracleBase.sol L199-214
/// @dev    Overpayment is NOT refunded: any msg.value above registrationFee is kept and is
///         withdrawable by ADMIN via withdrawEth. This is intentional.
function register(bytes32 feedId, address pool, address factory) external payable {
    require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
    ...
}
```

The check `msg.value >= registrationFee` is structurally identical to the seeded bug class. Any ETH above `registrationFee` is silently accepted and credited to the contract balance. The only withdrawal path is:

```solidity
// OracleBase.sol L292-297
function withdrawEth() external onlyRole(ADMIN_ROLE) {
    uint256 amount = address(this).balance;
    (bool ok, ) = payable(msg.sender).call{value: amount}("");
    require(ok);
}
```

`withdrawEth()` sweeps the **entire** balance to the ADMIN caller — there is no per-depositor accounting, no partial refund, and no user-callable recovery function. The overpaying caller has zero on-chain recourse.

### Impact Explanation
Any caller of `register()` who sends `msg.value > registrationFee` permanently loses the surplus. The default `registrationFee` is `1 wei`, so a caller who sends even a small amount of ETH (e.g., a round number like `0.01 ETH` or `1 ETH`) loses nearly the entire payment. The funds are not burned — they accumulate in the contract — but the original sender cannot retrieve them. Recovery requires the ADMIN to act off-chain and return funds manually, which is not guaranteed by any protocol invariant.

**Severity: Medium** — direct loss of user ETH; requires a user-side mistake (sending excess), but the protocol provides no safety net.

### Likelihood Explanation
Low-to-medium. `register()` is permissionless and intended to be called by pool deployers or integrators. Callers using scripts, wallets, or front-ends that round up ETH amounts, or that do not precisely compute `registrationFee` at call time (especially if the fee was recently changed via `setRegistrationFee`), will silently overpay. The fee can be changed by ADMIN at any time, creating a race condition where a caller reads the old fee and sends the wrong amount.

### Recommendation
Enforce exact payment, mirroring the fix from the seeded report:

```diff
- require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
+ require(msg.value == registrationFee, InsufficientFee(msg.value, registrationFee));
```

Alternatively, refund the surplus after the fee is collected:

```solidity
require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
uint256 excess = msg.value - registrationFee;
if (excess > 0) {
    (bool ok, ) = payable(msg.sender).call{value: excess}("");
    require(ok);
}
```

### Proof of Concept

1. ADMIN sets `registrationFee = 1 wei` (the default).
2. A pool deployer calls `register(feedId, pool, factory)` with `msg.value = 1 ether` (a common round-number mistake or a script that uses `msg.value = registrationFee + buffer`).
3. The `require(msg.value >= registrationFee)` passes.
4. `registeredPool[feedId][pool] = true` is set; the function returns successfully.
5. `0.999...999 ETH` is now locked in `OracleBase`. The caller has no function to call to recover it.
6. Only the ADMIN can call `withdrawEth()`, which sweeps the full balance to themselves — not back to the original caller. [1](#0-0) [2](#0-1) [3](#0-2)

        require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
        require(pool != address(0));
        require(approvedFactories.contains(factory), FactoryNotApproved(factory));
        require(IPoolFactory(factory).isPool(pool), NotAPool(pool));

        if (blacklisted[pool]) {
            blacklisted[pool] = false;
            emit BlacklistUpdated(pool, false);
        }

        registeredPool[feedId][pool] = true;
        emit PoolRegistered(feedId, pool, msg.sender, msg.value);
    }
