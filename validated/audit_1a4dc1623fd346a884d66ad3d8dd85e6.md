### Title
Wrong Variable Used in Consensus Reward Calculation: Staker Balance Passed as Total Protocol Stake — (`File: src/staking/staking.cairo`)

---

### Summary

In `update_rewards` (the consensus-era reward path), the staker's **individual** balance is passed as the `strk_total_stake` / `btc_total_stake` arguments to `_update_rewards`, instead of the **total protocol staking power**. The pre-consensus path (`update_rewards_from_attestation_contract`) correctly passes the total protocol staking power. This is a direct analog to the external report's "wrong variable used in a critical check" vulnerability class.

---

### Finding Description

Two separate code paths both call `_update_rewards`. They differ in what they pass as `strk_total_stake` / `btc_total_stake`:

**Pre-consensus path — correct:**

```cairo
// src/staking/staking.cairo, lines 1406-1422
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
...
self._update_rewards(
    :staker_address,
    strk_total_rewards: strk_epoch_rewards,
    btc_total_rewards: btc_epoch_rewards,
    :strk_total_stake,   // ← total protocol staking power
    :btc_total_stake,    // ← total protocol staking power
    ...
);
``` [1](#0-0) 

**Consensus path — wrong variable:**

```cairo
// src/staking/staking.cairo, lines 1474-1506
let (staker_total_strk_balance, staker_total_btc_balance) = self
    .get_staker_total_strk_btc_balance_at_epoch(
        :staker_address, :staker_pool_info, epoch_id: curr_epoch,
    );
...
self._update_rewards(
    :staker_address,
    strk_total_rewards: strk_block_rewards,
    btc_total_rewards: btc_block_rewards,
    strk_total_stake: staker_total_strk_balance,  // ← BUG: staker's own balance
    btc_total_stake: staker_total_btc_balance,    // ← BUG: staker's own balance
    ...
);
``` [2](#0-1) 

The parameter name `strk_total_stake` in `_update_rewards` is consistent with the attestation path's usage (total protocol stake as denominator for proportional reward calculation). When `_update_rewards` computes a staker's share as:

```
staker_reward = total_block_rewards × staker_balance / strk_total_stake
```

substituting `strk_total_stake = staker_balance` yields:

```
staker_reward = total_block_rewards × staker_balance / staker_balance
             = total_block_rewards   (100% of block rewards)
```

---

### Impact Explanation

**Critical — Direct theft of user funds.**

Any staker who calls `update_rewards` with their own address during the consensus-rewards era receives the entire block reward allocation instead of their proportional share. All other stakers' and delegators' accrued yield for that block is effectively stolen. Because `last_reward_block` is updated on each call, only one call per block is processed; a malicious staker who calls first each block continuously drains the reward stream. [3](#0-2) 

---

### Likelihood Explanation

**High.** `update_rewards` is a fully public function gated only by `general_prerequisites()` (pause check + non-zero caller). No privileged role is required. [4](#0-3) [5](#0-4) 

Any registered staker can call `update_rewards(their_own_address, false)` once per block, every block, and claim 100% of consensus block rewards. The attack requires no special setup beyond being a staker.

---

### Recommendation

In `update_rewards`, replace the staker-scoped balance variables with the total protocol staking power, consistent with `update_rewards_from_attestation_contract`:

```cairo
// Replace:
strk_total_stake: staker_total_strk_balance,
btc_total_stake: staker_total_btc_balance,

// With:
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
// then pass:
:strk_total_stake,
:btc_total_stake,
```

---

### Proof of Concept

1. Protocol enters consensus-rewards era (`is_pre_consensus()` returns `false`).
2. Malicious staker `A` (with a small stake, e.g. 1% of total) monitors the chain.
3. At the start of each new block, `A` calls `update_rewards(A_address, false)`.
4. Inside `update_rewards`, `staker_total_strk_balance` = A's own balance (e.g. 1,000 STRK).
5. This is passed as `strk_total_stake` to `_update_rewards`.
6. `_update_rewards` computes A's reward as `block_rewards × A_balance / A_balance = block_rewards` — 100% of the block reward.
7. All other stakers receive zero rewards for that block.
8. Repeated every block, `A` drains the entire consensus reward stream. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
