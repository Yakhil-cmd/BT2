### Title
P2P Receipt Deserialization Silently Drops All Transaction Events, Causing RPC to Return Wrong Receipt and Event Data - (File: crates/apollo_protobuf/src/converters/receipt.rs)

### Summary

Every `TryFrom<protobuf::receipt::*>` implementation in the P2P receipt converter unconditionally sets `events = vec![]`, discarding all events for every transaction type. Because no separate events-sync protocol exists on the client side, any node that syncs blocks via P2P stores permanently empty event lists in its `TransactionOutput` records. Subsequent calls to `starknet_getTransactionReceipt` and `starknet_getEvents` on that node return authoritative-looking but completely wrong (empty) event data.

### Finding Description

**Root cause — protobuf deserialization drops events for all five transaction types:**

In `crates/apollo_protobuf/src/converters/receipt.rs`, every `TryFrom` implementation for a receipt type hard-codes `let events = vec![];` and never reads events from the wire message: [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

The symmetric `From<*TransactionOutput>` serialization implementations also omit events from the `Common` message, so events are neither sent nor received over the wire.

**No compensating events-sync protocol exists on the client side:**

The P2P sync client directory contains `header.rs`, `state_diff.rs`, `class.rs`, and `transaction.rs`, but no `event.rs`. A TODO comment in the transaction sync module acknowledges this gap: [6](#0-5) 

**Empty-events outputs are written directly to storage:**

The P2P client collects `FullTransaction` objects (which include the deserialized, empty-events `transaction_output`) and pushes them into a `BlockBody`: [7](#0-6) 

`append_body` then calls `write_transactions`, which calls `write_events` on each output. Because `tx_output.events()` returns an empty slice, nothing is written to the events index table either: [8](#0-7) [9](#0-8) 

**RPC reads events directly from the stored `TransactionOutput`:**

`starknet_getTransactionReceipt` calls `get_non_pending_receipt`, which reads the `TransactionOutput` from the file store via `get_transaction_output`. Because the stored output has `events = vec![]`, the RPC response contains an empty events array: [10](#0-9) 

`starknet_getEvents` similarly iterates over `transaction_output.events()` from storage, returning nothing for P2P-synced blocks.

**The P2P server propagates the corruption:**

When a P2P-synced node acts as a server, it reads `transaction_outputs` from storage and calls `.events()` to serve the events protocol: [11](#0-10) 

Since those outputs have empty events, the node serves empty events to all downstream peers, cascading the corruption.

### Impact Explanation

Any node that syncs blocks via the P2P protocol will permanently store empty event lists for every transaction in every synced block. `starknet_getTransactionReceipt` and `starknet_getEvents` will return authoritative-looking responses with zero events, regardless of what the transactions actually emitted. This matches the allowed impact: **High — RPC returns an authoritative-looking wrong value**.

### Likelihood Explanation

The P2P sync client is a production code path used by every non-sequencer node. The defect is structural and deterministic: every block synced via P2P triggers it. No special transaction type, timing, or attacker action is required.

### Recommendation

1. Add events to the protobuf `Common` message (or a per-type field) and populate them in both the `From` (serialization) and `TryFrom` (deserialization) implementations.
2. Alternatively, implement a dedicated events-sync protocol on the client side (the existing `EventsRequest`/`EventsResponse` protobuf messages already exist for this purpose).
3. Remove the `let events = vec![];` hard-coding in all five `TryFrom` implementations once the wire format carries events.

### Proof of Concept

1. Start a sequencer node that produces a block containing an Invoke transaction that emits events (e.g., an ERC-20 transfer).
2. Start a second node configured to sync via P2P from the first.
3. After the second node syncs the block, call `starknet_getTransactionReceipt` with the invoke transaction hash on the second node.
4. Observe that the `events` array in the response is empty, while the same call on the sequencer node returns the correct events.
5. Confirm by calling `starknet_getEvents` with the emitting contract address on the second node — zero events are returned for the synced block range. [1](#0-0) [12](#0-11) [9](#0-8) [11](#0-10)

pub(crate) struct TransactionStreamFactory;

impl BlockDataStreamBuilder<FullTransaction> for TransactionStreamFactory {
    // TODO(Eitan): Add events protocol to BlockBody or split their write to storage
    type Output = (BlockBody, BlockNumber);

    const TYPE_DESCRIPTION: &'static str = "transactions";
    const BLOCK_NUMBER_LIMIT: BlockNumberLimit = BlockNumberLimit::HeaderMarker;

    fn parse_data_for_block<'a>(
        transactions_response_manager: &'a mut ClientResponsesManager<DataOrFin<FullTransaction>>,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut block_body = BlockBody::default();
            let mut current_transaction_len = 0;
            let target_transaction_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .n_transactions;
            while current_transaction_len < target_transaction_len {
                let maybe_transaction = transactions_response_manager.next().await.ok_or(
                    ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }),
                )?;
                let Some(FullTransaction { transaction, transaction_output, transaction_hash }) =
                    maybe_transaction?.0
                else {
                    if current_transaction_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::NotEnoughTransactions {
                            expected: target_transaction_len,
                            actual: current_transaction_len,
                            block_number: block_number.0,
                        }));
                    }
                };
                block_body.transactions.push(transaction);
                block_body.transaction_outputs.push(transaction_output);
                // TODO(eitan): Validate transaction hash from untrusted sources
                block_body.transaction_hashes.push(transaction_hash);
                current_transaction_len += 1;
            }
            Ok(Some((block_body, block_number)))
        }
        .boxed()
    }

fn write_events<'env>(
    tx_output: &TransactionOutput,
    txn: &DbTransaction<'env, RW>,
    events_table: &'env EventsTable<'env>,
    transaction_index: TransactionIndex,
) -> StorageResult<()> {
    let mut contract_addresses_set = HashSet::new();

    for event in tx_output.events().iter() {
        contract_addresses_set.insert(event.from_address);
    }

    for contract_address in contract_addresses_set {
        let key = (contract_address, transaction_index);
        // Here, we use the function assumption; the append will fail if an older transaction_index
        // is a table.
        events_table.append_greater_sub_key(txn, &key, &NoValue)?;
    }
    Ok(())
}

    async fn get_transaction_receipt(
        &self,
        transaction_hash: TransactionHash,
    ) -> RpcResult<GeneralTransactionReceipt> {
        verify_storage_scope(&self.storage_reader)?;

        let txn = self.storage_reader.begin_ro_txn().map_err(internal_server_error)?;

        if let Some(transaction_index) =
            txn.get_transaction_idx_by_hash(&transaction_hash).map_err(internal_server_error)?
        {
            let tx = txn
                .get_transaction(transaction_index)
                .map_err(internal_server_error)?
                .unwrap_or_else(|| panic!("Should have tx {transaction_hash}"));

            // TODO(Shahak): Add version function to transaction in SN_API.
            let tx_version = match &tx {
                StarknetApiTransaction::Declare(tx) => tx.version(),
                StarknetApiTransaction::Deploy(tx) => tx.version,
                StarknetApiTransaction::DeployAccount(tx) => tx.version(),
                StarknetApiTransaction::Invoke(tx) => tx.version(),
                StarknetApiTransaction::L1Handler(tx) => tx.version,
            };

            let msg_hash = match tx {
                StarknetApiTransaction::L1Handler(l1_handler_tx) => {
                    Some(l1_handler_tx.calc_msg_hash())
                }
                _ => None,
            };

            get_non_pending_receipt(&txn, transaction_index, transaction_hash, tx_version, msg_hash)
        } else {
            // The transaction is not in any non-pending block. Search for it in the pending block
            // and if it's not found, return error.

            // TODO(shahak): Consider cloning the transactions and the receipts in order to free
            // the lock sooner (Check which is better).
            let pending_data = read_pending_data(&self.pending_data, &txn).await?;

            let client_transaction_receipt = pending_data
                .block
                .transaction_receipts()
                .iter()
                .find(|receipt| receipt.transaction_hash == transaction_hash)
                .ok_or_else(|| ErrorObjectOwned::from(TRANSACTION_HASH_NOT_FOUND))?
                .clone();
            let client_transaction = &pending_data
                .block
                .transactions()
                .iter()
                .find(|tx| tx.transaction_hash() == transaction_hash)
                .ok_or_else(|| ErrorObjectOwned::from(TRANSACTION_HASH_NOT_FOUND))?
                .clone();
            client_receipt_to_rpc_pending_receipt(client_transaction, client_transaction_receipt)
        }

#[async_trait]
impl FetchBlockData for (Event, TransactionHash) {
    async fn fetch_block_data(
        block_number: BlockNumber,
        txn: &StorageTxn<'_, db::RO>,
        _class_manager_client: &mut SharedClassManagerClient,
    ) -> Result<Vec<Self>, P2pSyncServerError> {
        let transaction_outputs = txn.get_block_transaction_outputs(block_number)?.ok_or(
            P2pSyncServerError::BlockNotFound {
                block_hash_or_number: BlockHashOrNumber::Number(block_number),
            },
        )?;
        let transaction_hashes = txn.get_block_transaction_hashes(block_number)?.ok_or(
            P2pSyncServerError::BlockNotFound {
                block_hash_or_number: BlockHashOrNumber::Number(block_number),
            },
        )?;

        let mut result = Vec::new();
        for (transaction_output, transaction_hash) in
            transaction_outputs.into_iter().zip(transaction_hashes)
        {
            for event in transaction_output.events() {
                result.push((event.clone(), transaction_hash));
            }
        }
        Ok(result)
    }
