### Title
Staker Can Frontrun Reward Distribution by Spiking Commission Within Active Commitment — (`File: src/staking/staking.cairo`)

---

### Summary

A staker holding an active `commission_commitment` can call `set_commission` to raise their commission to `max_commission` immediately before a reward-distribution event (`update_rewards` / `update_rewards_from_attestation_contract`). Because the commission is read at reward-calculation time rather than being locked at delegation time, and because `enter_delegation_pool` / `add_to_delegation_pool` accept no slippage parameter, delegators receive far less yield than they expected when they joined the pool. The staker can then lower the commission again to attract new delegators and repeat the attack.

The developers themselves flag this in a code comment: *"Current commission increase safeguards still allow for sudden commission changes."*

---

### Finding Description

**Root cause — commission is read at reward-calculation time, not locked at delegation time.**

`calculate_staker_pools_rewards` reads the live commission value every time rewards are distributed:

```cairo
// src/staking/staking.cairo  line 1964
let commission = staker_pool_info.commission();
``` [1](#0-0) 

`update_commission` (called by `set_commission`) allows a commission *increase* whenever a `commission_commitment` is active, as long as the new value does not exceed `max_commission`:

```cairo
// src/staking/staking.cairo  lines 1583-1589
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission, ...
    );
    assert!(commission != old_commission, ...);
}
``` [2](#0-1) 

The developer note at the top of `set_commission_commitment` explicitly acknowledges the gap:

```cairo
// src/staking/staking.cairo  lines 745-747
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
``` [3](#0-2) 

`enter_delegation_pool` and `add_to_delegation_pool` accept no `max_commission` / slippage parameter, so delegators cannot express the commission they agreed to: [4](#0-3) [5](#0-4) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

The staker captures a disproportionate share of every reward event that occurs while the inflated commission is in effect. Delegators who joined expecting a low commission (e.g. 1 %) receive only the residual after the spiked commission (e.g. 99 %) is deducted. The stolen yield is transferred directly to the staker's `unclaimed_rewards_own` and is immediately claimable. The delegators' principal is unaffected, but their accrued yield for the manipulated epoch(s) is permanently redirected to the staker.

---

### Likelihood Explanation

**Medium-High.** Any staker who has opened a delegation pool can execute this attack. The only prerequisite is calling `set_commission_commitment` with a high `max_commission` — a permissionless, public function. The staker can advertise a low commission to attract delegators, set a commitment with `max_commission = COMMISSION_DENOMINATOR` (100 %), then spike the commission to 100 % immediately before each reward event. After the reward is distributed, the staker lowers the commission again (which is always allowed) to maintain the appearance of a fair pool. The attack is repeatable every reward cycle.

---

### Recommendation

1. **Add a `max_commission` slippage guard to `enter_delegation_pool` and `add_to_delegation_pool`**: require the caller to pass the maximum commission they accept; revert if the current commission exceeds it.
2. **Enforce a time-delay on commission increases**: when a commitment is active, require that any increase takes effect only at the start of the *next* epoch, giving delegators time to exit.
3. **Alternatively, prohibit commission increases entirely** (remove the commitment-based increase path) and only allow decreases, consistent with the default path.

---

### Proof of Concept

```
1. Staker calls stake(..., commission=100 /* 1% */, pool_enabled=true).
2. Staker calls set_commission_commitment(
       max_commission = 10000 /* 100% */,
       expiration_epoch = current_epoch + 1
   ).
3. Delegators observe commission=1% and call enter_delegation_pool(reward_address, amount).
4. Before update_rewards / attest is called for the current epoch, staker calls
   set_commission(commission = 10000 /* 100% */).
   → Passes: 10000 <= max_commission (10000) and 10000 != old_commission (100). ✓
5. update_rewards (or attestation contract) triggers _update_rewards →
   calculate_staker_pools_rewards reads commission = 10000.
   → split_rewards_with_commission gives 100% to staker, 0% to pool.
6. Delegators claim_rewards → receive 0 for that epoch.
7. Staker calls set_commission(commission = 99) to restore low commission.
   → Allowed because 99 < 10000 (decrease). ✓
8. Repeat from step 3 each epoch.
``` [6](#0-5) [7](#0-6) [8](#0-7)

            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);

                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();

                    Events::CommissionCommitmentSet {

            // Calculate self rewards.

        fn enter_delegation_pool(
            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.
            self
                .pool_member_info
                .write(pool_member, VInternalPoolMemberInfoTrait::new_latest(:reward_address));

            // Emit events.
            self
                .emit(
                    Events::NewPoolMember { pool_member, staker_address, reward_address, amount },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake: Zero::zero(), new_delegated_stake: amount,
                    },
                );
        }

        fn add_to_delegation_pool(
            ref self: ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Amount {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            assert!(
                caller_address == pool_member || caller_address == pool_member_info.reward_address,
                "{}",
                Error::CALLER_CANNOT_ADD_TO_POOL,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);

            // Transfer funds from the delegator to the staking contract.
            let token_dispatcher = self.token_dispatcher.read();
            let staker_address = self.staker_address.read();
            transfer_from_delegator(pool_member: caller_address, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            // Update the pool member's balance checkpoint.
            let old_delegated_stake = self.increase_member_balance(:pool_member, :amount);
            let new_delegated_stake = old_delegated_stake + amount;

            // Emit events.
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );

            new_delegated_stake
        }
