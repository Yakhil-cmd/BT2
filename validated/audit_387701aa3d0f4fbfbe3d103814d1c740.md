### Title
Gateway L2 Gas Price Threshold Uses Truncating Division, Admitting Transactions Below Intended Minimum — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price by multiplying the previous block's price by a percentage ratio and calling `.to_integer()`, which **truncates toward zero** (floor). The invariant "reject any transaction whose gas price is below `min_gas_price_percentage%` of the previous block price" is therefore violated whenever the product is non-integer: the threshold is one unit too low, and a transaction at the truncated value passes when it should be rejected.

---

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  lines 367-372
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← truncates (floor), not ceiling
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
```

`Ratio::to_integer()` from `num_rational` returns the integer part of the ratio, i.e. `⌊numerator / denominator⌋`. For any `(percentage, price)` pair where `percentage * price` is not divisible by 100, the computed threshold is one unit below the mathematically correct minimum.

**Concrete example**

| Parameter | Value |
|---|---|
| `min_gas_price_percentage` | 50 |
| `previous_block_l2_gas_price` | 101 fri |
| Exact threshold | 50.5 fri |
| `to_integer()` result | **50** (floor) |
| Correct ceiling | **51** |

A transaction with `tx_l2_gas_price = 50` satisfies `50 < 50` → false, so it **passes** the check. The intended policy requires `tx_l2_gas_price ≥ 51`.

The ERC-4626 analog is exact: just as `previewWithdraw` should round up the shares a user must supply (to favour the vault), the gateway threshold should round **up** the minimum price a submitter must offer (to favour the sequencer's admission policy). Both bugs allow a user to satisfy a check with a value one unit below the intended floor. [1](#0-0) 

---

### Impact Explanation

The gateway's stateful path admits transactions whose L2 gas price is one unit below the operator-configured minimum. Such transactions enter the mempool and are forwarded to the batcher. If the actual block gas price at execution time equals the previous block price, the transaction will also pass `check_fee_bounds` (which uses the block's gas price directly, not the percentage-adjusted threshold), so the transaction executes successfully with a gas price the gateway was supposed to reject.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.** [2](#0-1) 

---

### Likelihood Explanation

- Triggered whenever `min_gas_price_percentage × previous_block_l2_gas_price` is not divisible by 100.
- The **default** `min_gas_price_percentage = 100` is immune (100/100 = 1, exact integer).
- Any non-default percentage (e.g. 50, 80, 90) combined with a price not divisible by the corresponding factor produces a non-integer product and exposes the off-by-one.
- The rounding error is bounded to **1 fri**, so the economic impact per transaction is negligible, but the admission invariant is structurally broken for all non-trivial percentage configurations. [3](#0-2) 

---

### Recommendation

Replace `.to_integer()` with `.ceil().to_integer()` so the threshold is the smallest integer **≥** the exact fractional value:

```rust
// Before
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();

// After
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This mirrors the pattern already used throughout the fee-calculation layer (`convert_l1_to_l2_gas_price_round_up`, `sierra_gas_to_l1_gas_amount_round_up`, `calculate_resource_gas_cost`) where every "minimum the user must provide" computation uses ceiling arithmetic. [4](#0-3) 

---

### Proof of Concept

```
min_gas_price_percentage = 50
previous_block_l2_gas_price = 101 fri

gas_price_threshold_multiplier = Ratio(50, 100) = 1/2
product = 1/2 * 101 = Ratio(101, 2) = 50.5

to_integer()  → 50   (floor — current code)
ceil().to_integer() → 51  (correct)

Attacker submits tx with tx_l2_gas_price = 50:
  current check:  50 < 50  → false  → tx ADMITTED  ← wrong
  correct check:  50 < 51  → true   → tx REJECTED
```

The test suite confirms the current (truncating) behaviour is the intended contract:

```
#[case::tx_gas_price_meets_threshold_with_factor_pass(
    100_u128.try_into().unwrap(),   // prev price = 100
    50,                              // percentage = 50%
    50_u128.into(),                  // tx price = 50  → passes
    Ok(())
)]
```

With `prev_price = 101` and `percentage = 50`, the test would still pass at `tx_price = 50` under the current code, demonstrating the off-by-one. [5](#0-4)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
