### Title
Single Malicious Participant Can Permanently Abort DKG/Reshare/Refresh for All Honest Parties via `broadcast_success` Unanimous Vote Requirement - (File: src/dkg.rs)

### Summary
The `broadcast_success` function at the end of `do_keyshare` requires **unanimous** agreement from all participants. A single malicious participant can broadcast `false` or a wrong session ID, causing every honest party to abort DKG, reshare, and refresh — permanently denying key generation for the entire group.

### Finding Description

`do_keyshare` is the core function driving DKG, reshare, and refresh. Its final step calls `broadcast_success`: [1](#0-0) 

`broadcast_success` collects one broadcast value from every participant via `do_broadcast` (the echo-broadcast protocol), then enforces two **unanimous** `all()` checks: [2](#0-1) 

The honest party always sends `(true, session_id)`: [3](#0-2) 

A malicious participant is free to send `(false, session_id)` or `(true, wrong_session_id)`. Because the echo-broadcast protocol faithfully delivers each participant's own broadcast to all others, every honest party will receive the malicious value and hit one of the two `all()` guards, returning `ProtocolError::AssertionFailed` and aborting the protocol.

The `?` propagation at the call site means the error immediately terminates `do_keyshare` for every honest participant: [1](#0-0) 

This affects all three entry points that call `do_keyshare`: [4](#0-3) [5](#0-4) 

### Impact Explanation

**High — Permanent denial of key generation, reshare, and refresh for honest parties.**

A single malicious participant (1-of-n) can prevent the entire group from ever completing DKG, reshare, or refresh. Because the protocol aborts with no mechanism to exclude the offending party and retry, the honest parties are left without a usable key share. This maps directly to the allowed impact: *"Permanent denial of signing, key generation, reshare, refresh, or CKD for honest parties under valid protocol inputs and documented trust assumptions."*

### Likelihood Explanation

**Low.** The attacker must be a registered participant in the protocol session. However, this is a realistic threat: any single compromised or adversarial node in the participant set can trivially trigger the condition by sending one bit (`false`) in the final broadcast round, with no cryptographic barrier.

### Recommendation

Replace the unanimous `all()` requirement with a threshold-based check (e.g., accept success if at least `threshold` participants broadcast `(true, session_id)`). Participants that broadcast `false` or a mismatched session ID should be identified and excluded, allowing the honest majority to complete the protocol. This mirrors the pull-vs-push fix in the original report: do not let a single party's failure veto the entire operation.

### Proof of Concept

1. Participants `[P1, P2, P3]` run DKG with threshold 2.
2. `P1` is malicious. It participates honestly through all share-distribution rounds so that `P2` and `P3` derive valid shares.
3. In the final `broadcast_success` round, `P1` sends `(false, session_id)` instead of `(true, session_id)`.
4. The echo-broadcast protocol faithfully delivers `P1`'s value to `P2` and `P3`.
5. Both `P2` and `P3` reach the check at line 329:
   ```rust
   if !vote_list.iter().all(|&(boolean, _)| boolean) {
       return Err(ProtocolError::AssertionFailed("A participant seems to have failed its checks. Aborting Protocol!"));
   }
   ```
6. Both abort. Neither obtains a `KeygenOutput`. The key is never established, and signing is permanently unavailable. [6](#0-5)

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
