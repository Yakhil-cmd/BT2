### Title
Gateway Stateful Validator Uses Stale Previous-Block L2 Gas Price Instead of `next_l2_gas_price` for Resource Bounds Admission - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check compares an incoming transaction's `max_price_per_unit` against the **current committed block's** L2 gas price, not the **next block's** L2 gas price that will actually govern execution. The code even carries an explicit TODO acknowledging the wrong value is used. This causes the admission gate to be calibrated against a stale price, mirroring the Chainlink `latestAnswer()` pattern: a price is consumed without the freshness metadata needed to make the check meaningful.

### Finding Description

In `validate_resource_bounds` inside `StatefulTransactionValidator`, the reference price for the threshold check is obtained as:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

This reads `gas_prices.strk_gas_prices.l2_gas_price` — the price **of the last committed block** — and uses it as the floor for the incoming transaction's `max_price_per_unit`.

However, the block header already stores the EIP-1559-computed price for the **next** block:

```rust
pub next_l2_gas_price: GasPrice,
``` [2](#0-1) 

The next-block price is computed via `calculate_next_base_gas_price` and can diverge from the current block's price by up to `1/gas_price_max_change_denominator` per block. [3](#0-2) 

The threshold applied is:

```rust
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0).to_integer();
if tx_l2_gas_price.0 < threshold { return Err(...GAS_PRICE_TOO_LOW...) }
``` [4](#0-3) 

Because `next_l2_gas_price` is not used, two admission errors arise:

1. **Accepts transactions that will fail at execution**: When `next_l2_gas_price > current_l2_gas_price`, a transaction with `max_price_per_unit` in the range `[threshold(current), next_l2_gas_price)` passes the gateway check but will be rejected or reverted during block execution because the actual price exceeds the user's declared maximum.

2. **Rejects valid transactions**: When `next_l2_gas_price < threshold(current)` (price is falling), a transaction with `max_price_per_unit` in the range `[next_l2_gas_price, threshold(current))` is refused by the gateway even though it would succeed in the next block.

The `GatewayFixedBlockSyncStateClient` that supplies `get_block_info()` reads the block header but exposes only `l2_gas_price`, not `next_l2_gas_price`: [5](#0-4) 

### Impact Explanation

**High** — Mempool/gateway admission accepts invalid transactions (case 1) or rejects valid transactions (case 2) before sequencing. In case 1, users pay gas for transactions that will fail. In case 2, legitimate transactions are silently dropped at the gateway, degrading liveness. Both outcomes are observable by any unprivileged user submitting a V3 (`AllResources`) transaction during a period of price movement.

### Likelihood Explanation

The EIP-1559 L2 gas price adjusts every block based on gas consumption. Any block that is meaningfully above or below the gas target will produce a `next_l2_gas_price` that differs from `l2_gas_price`. This is the normal operating condition of a live network, so the divergence occurs continuously, not only under adversarial conditions.

### Recommendation

Replace the read of `gas_prices.strk_gas_prices.l2_gas_price` with `next_l2_gas_price` from the block header. The `GatewayFixedBlockStateReader` trait and its `GatewayFixedBlockSyncStateClient` implementation should expose `next_l2_gas_price` from `BlockHeaderWithoutHash`, and `validate_resource_bounds` should use that value as the reference price. This directly resolves the acknowledged TODO.

### Proof of Concept

1. Observe the current committed block has `l2_gas_price = P` and `next_l2_gas_price = P * 1.125` (block was full, EIP-1559 raised the price by the maximum increment).
2. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = P * 1.05` and `min_gas_price_percentage = 100`.
3. Gateway computes `threshold = 1.0 * P = P`. Since `P * 1.05 >= P`, the transaction passes `validate_resource_bounds` and is admitted to the mempool.
4. When the sequencer builds the next block, the actual L2 gas price is `P * 1.125`. The transaction's `max_price_per_unit = P * 1.05 < P * 1.125`, so the transaction is rejected or reverted during execution — it was admitted on a stale price reference.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
