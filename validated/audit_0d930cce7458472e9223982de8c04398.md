Audit Report

## Title
Unbounded Frame-Length Allocation in `FrameDecoder::decode` Allows Compromised Sandbox to Abort the Replica Process — (`rs/canister_sandbox/src/frame_decoder.rs`)

## Summary
`FrameDecoder::decode` reads a raw `u64` length field from the IPC socket, casts it to `usize`, and calls `data.reserve(size)` with no upper-bound check. A compromised sandbox process can send an 8-byte header encoding `u64::MAX`, causing the replica's `CanisterSandboxIPC` thread to invoke the global allocator with an impossible size. Because Rust's allocator calls `handle_alloc_error` on failure — which **aborts** the process rather than unwinding — the entire replica process is terminated.

## Finding Description
In `rs/canister_sandbox/src/frame_decoder.rs`, the `FrameDecoderState::Length` arm casts the wire-supplied `u64` directly to `usize` and calls `data.reserve(size)` with no guard:

```rust
// frame_decoder.rs L44-48
FrameDecoderState::Length(size) => {
    let size: usize = *size as usize;
    if data.len() < size {
        data.reserve(size);   // ← no upper-bound check
        return None;
    }
``` [1](#0-0) 

No `MAX_FRAME_SIZE` constant exists anywhere in the canister sandbox crate — confirmed by search. `SocketReaderConfig` controls only idle-timeout buffer trimming and `malloc_trim`; it imposes no message-size limit: [2](#0-1) 

`socket_read_messages` calls `decoder.decode` in a tight loop with no interposed size check: [3](#0-2) 

The replica spawns a dedicated `CanisterSandboxIPC` thread per sandbox process that runs this loop: [4](#0-3) 

**Exploit path:**
1. Attacker achieves arbitrary code execution inside a Wasmtime sandbox (e.g., via a Wasmtime CVE or memory-safety bug in the embedder).
2. Attacker writes 8 bytes encoding `u64::MAX` in big-endian to the already-open `UnixStream` IPC socket.
3. `socket_read_messages` reads the bytes into `buf`, calls `decoder.decode(&mut buf)`.
4. `decode` enters `FrameDecoderState::Length(u64::MAX)`, computes `size = usize::MAX`, finds `data.len() < usize::MAX`, and calls `data.reserve(usize::MAX)`.
5. `BytesMut::reserve` delegates to the global allocator; the OS rejects the allocation; Rust calls `handle_alloc_error`, which **aborts** (SIGABRT) the process — not a recoverable panic.
6. The entire replica process terminates.

No existing check intercepts this: the `SocketReaderConfig` default used at the call site (`SocketReaderConfig::default()`) sets only `idle_malloc_trim: false` and `idle_timeout: IDLE_TIMEOUT_TO_TRIM_BUFFER`. [5](#0-4) 

## Impact Explanation
The abort terminates the replica process, removing the node from its subnet. If multiple sandboxes are targeted simultaneously (one crafted message per active canister sandbox), several replicas can be taken down in a coordinated fashion, potentially causing a consensus gap or subnet stall. This matches the allowed High impact: **"Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS."** Severity: **High ($2,000–$10,000)**.

## Likelihood Explanation
The precondition is a compromised Wasmtime sandbox — i.e., arbitrary code execution inside the sandbox process. This is non-trivial but is precisely the threat model the sandbox isolation architecture is designed to contain. The IPC channel between sandbox and controller is the explicit trust boundary; hardening it against a malicious sandbox is the entire purpose of the isolation design. Once the sandbox is compromised, sending 8 crafted bytes over the already-open `UnixStream` requires no further privilege and is trivially repeatable. The attack is not volumetric and does not require mainnet access for proof.

## Recommendation
Add a `MAX_FRAME_SIZE` constant and reject frames exceeding it before calling `reserve`:

```rust
const MAX_FRAME_SIZE: usize = 64 * 1024 * 1024; // 64 MiB

FrameDecoderState::Length(size) => {
    let size: usize = *size as usize;
    if size > MAX_FRAME_SIZE {
        // Close the socket gracefully or return an error instead of panicking
        return None; // or propagate an Err
    }
    if data.len() < size {
        data.reserve(size);
        return None;
    }
    // ...
}
```

Alternatively, change `decode` to return `Option<Result<Message, FrameError>>` so oversized frames can be reported and the socket closed without aborting the process.

## Proof of Concept

```rust
use bytes::BytesMut;
use ic_canister_sandbox::frame_decoder::FrameDecoder;

#[test]
fn oversized_frame_aborts() {
    let mut buf = BytesMut::new();
    // 8-byte big-endian u64::MAX — no payload follows
    buf.extend_from_slice(&u64::MAX.to_be_bytes());

    let mut decoder = FrameDecoder::<Vec<u8>>::new();
    // Triggers data.reserve(usize::MAX) → handle_alloc_error → process abort
    decoder.decode(&mut buf);
}
```

Running this test inside the replica's `CanisterSandboxIPC` thread context (or as a standalone unit test with `#[should_panic]` replaced by observing SIGABRT) demonstrates the abort. A safe integration-test variant can use `std::process::Command` to spawn a child process that executes the PoC and assert the child exits with a non-zero status (abort signal). [6](#0-5)

    pub fn decode(&mut self, data: &mut BytesMut) -> Option<Message> {
        loop {
            match &self.state {
                FrameDecoderState::NoLength => {
                    if data.len() < 8 {
                        data.reserve(8);
                        return None;
                    } else {
                        let size = data.get_u64();
                        self.state = FrameDecoderState::Length(size);
                    }
                }
                FrameDecoderState::Length(size) => {
                    let size: usize = *size as usize;
                    if data.len() < size {
                        data.reserve(size);
                        return None;
                    } else {
                        let frame = data.split_to(size);
                        self.state = FrameDecoderState::NoLength;
                        let value = bincode::deserialize(&frame).unwrap();
                        return Some(value);
                    }
                }
            }
        }
