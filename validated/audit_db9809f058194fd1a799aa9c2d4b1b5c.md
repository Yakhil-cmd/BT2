### Title
Gateway Stateful Validator Checks L2 Gas Price Against Stale Previous-Block Price Instead of Next-Block Price, Admitting Transactions That Fail Blockifier Pre-Validation - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price from the **current committed block** (block N) and uses it as the admission threshold. The blockifier's `check_fee_bounds`, called during actual block building (block N+1), checks the transaction's `max_price_per_unit` against the **next block's** L2 gas price. Because the L2 gas price is adjusted per-block via EIP-1559 (`calculate_next_base_gas_price`), the two checks operate on different values. When the network is congested and the price rises, the gateway admits transactions that will fail blockifier pre-validation; when the price falls, the gateway rejects transactions that would succeed.

The gateway also runs blockifier validation using the same stale block info (block N with `block_number.unchecked_next()`), so the blockifier validation at the gateway level is also stale:

This means both the explicit `validate_resource_bounds` check and the blockifier's `check_fee_bounds` during gateway validation use block N's gas prices. The mismatch only manifests when the batcher builds block N+1 with the updated price.

### Impact Explanation

**Congested network (price rising):** A transaction submitted with `tx.l2_gas_price = block_N.l2_gas_price` passes the gateway threshold (`block_N.l2_gas_price >= min_gas_price_percentage * block_N.l2_gas_price / 100` when `min_gas_price_percentage ≤ 100`). When the batcher builds block N+1 with `l2_gas_price = next_l2_gas_price > block_N.l2_gas_price`, the blockifier's `check_fee_bounds` rejects the transaction with `MaxGasPriceTooLow`. The gateway has admitted a transaction that is invalid for the next block.

**Uncongested network (price falling):** A transaction with `tx.l2_gas_price = next_l2_gas_price < block_N.l2_gas_price` would succeed in the blockifier but is rejected by the gateway because `next_l2_gas_price < min_gas_price_percentage * block_N.l2_gas_price / 100`. Valid transactions are incorrectly rejected.

This matches the **High** impact: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."

The analog to BellumNursery is direct: BellumNursery distributes rewards based on the current share ratio (a snapshot at distribution time) rather than the time-weighted stake. Here, the gateway validates fees against the current block's price (a snapshot at admission time) rather than the next block's price at which the transaction will actually execute. In both cases, a stale snapshot causes the wrong value to be used for an economically significant decision.

### Likelihood Explanation

The L2 gas price changes every block via EIP-1559. Under sustained high load (blocks consistently above `gas_target`), the price rises each block. The maximum change per block is `price / gas_price_max_change_denominator`. With `gas_price_max_change_denominator = 8` (a typical EIP-1559 value), the price can rise by up to 12.5% per block. Any transaction submitted with `tx.l2_gas_price` between `min_gas_price_percentage * block_N_price / 100` and `block_N+1_price` will be admitted by the gateway but rejected by the batcher. This is a normal operating condition during network congestion, not an edge case.

### Recommendation

Read `next_l2_gas_price` from the block header instead of `l2_gas_price`, as the TODO comment already identifies:

```rust
// Replace:
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// With:
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .next_l2_gas_price;  // already stored in StorageBlockHeader
```

The `GatewayFixedBlockStateReader::get_block_info` trait method should be extended (or a new method added) to expose `next_l2_gas_price` from `StorageBlockHeader`. This ensures the gateway's admission threshold matches the price the batcher will enforce.

### Proof of Concept

1. Observe the current committed block N's L2 gas price `P_N` and its `next_l2_gas_price` field `P_N1 > P_N` (network is congested).
2. Submit an `AllResources` invoke transaction with `l2_gas.max_price_per_unit = P_N` (equal to the previous block's price, above the gateway threshold when `min_gas_price_percentage ≤ 100`).
3. The gateway's `validate_tx_l2_gas_price_within_threshold` computes `threshold = min_gas_price_percentage * P_N / 100 ≤ P_N` and admits the transaction.
4. The batcher begins building block N+1 with `l2_gas_price = P_N1`.
5. The blockifier's `check_fee_bounds` evaluates `P_N < P_N1` and raises `MaxGasPriceTooLow`, rejecting the transaction from the block.
6. The transaction was admitted to the mempool but cannot be included in any block until the gas price drops back to `P_N` or below. [7](#0-6) [8](#0-7)

```rust
    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
```
