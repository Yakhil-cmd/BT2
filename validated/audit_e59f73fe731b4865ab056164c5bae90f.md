Audit Report

## Title
Byzantine Flex-Committee Member Can Inject Invalid Candid to Silently Drop Flexible HTTP Callback â€” (`rs/https_outcalls/consensus/src/payload_builder.rs`)

## Summary

`validate_canister_http_payload_impl` validates each entry in `payload.flexible_responses` via `validate_flexible_response_with_proof`, which treats `CanisterHttpResponseContent::Success(data)` as an opaque byte blob and never Candid-decodes it. At execution time, `flexible_ok_responses_into_consensus_response` Candid-decodes those bytes, and a single decode failure causes `collect::<Option<_>>()?` to short-circuit the entire group to `None`, silently dropping the `ConsensusResponse`. A Byzantine node that is both block proposer and a member of the `flex_committee` can craft one entry with invalid Candid bytes, sign it with their own key, and include it alongside legitimately-signed entries. The payload passes all validation, but the callback is never delivered â€” it stalls until `CANISTER_HTTP_TIMEOUT_INTERVAL` expires.

## Finding Description

**Validation gap â€” `validate_flexible_response_with_proof` (utils.rs L147â€“198):**

For each `FlexibleCanisterHttpResponseWithProof`, the function checks:
- Callback ID consistency
- Signer is in `flex_committee` and not a duplicate
- `crypto_hash(response)` matches the hash in the proof
- `content.count_bytes()` matches `content_size` in the proof
- `is_reject` flag consistency [1](#0-0) 

None of these checks Candid-decode the raw bytes inside `CanisterHttpResponseContent::Success(data)`. The hash and size checks operate on the raw byte representation, so a blob of `b"this is invalid candid"` passes all checks as long as the proof metadata is computed from those same raw bytes.

**Execution-time drop â€” `flexible_ok_responses_into_consensus_response` (payload_builder.rs L1000â€“1014):** [2](#0-1) 

`Decode!(&data, CanisterHttpResponsePayload).ok()` wraps a failure as `None`. The `.collect::<Option<_>>()?` at line 1014 short-circuits the entire iterator to `None` if any single entry fails. The function returns `None`, and `.flatten()` at line 969 silently drops the group â€” no `ConsensusResponse` is emitted for `callback_id`. [3](#0-2) 

**Developer comment acknowledges the path (L992â€“996):** [4](#0-3) 

The comment treats this as unreachable ("should never occur"), but a Byzantine committee member can deliberately trigger it.

**Exploit flow:**
1. Byzantine node is block proposer for a round and is a member of `flex_committee` for a targeted flexible HTTP request.
2. It constructs a `FlexibleCanisterHttpResponseWithProof` with `CanisterHttpResponseContent::Success(b"invalid candid bytes")`.
3. It computes `crypto_hash` and `count_bytes` over those raw bytes and populates the proof metadata accordingly â€” both checks in `validate_flexible_response_with_proof` pass.
4. It signs the proof with its own key â€” `validate_response_share` passes because the node is a legitimate committee member.
5. It includes this entry alongside legitimately-signed entries from honest nodes in the proposed block.
6. `validate_canister_http_payload_impl` accepts the payload (lines 639â€“657). [5](#0-4) 

7. At execution time, `flexible_ok_responses_into_consensus_response` fails to decode the invalid entry, returns `None`, and the entire group is dropped.

## Impact Explanation

The targeted canister's flexible HTTP callback is not delivered. The callback stalls until `CANISTER_HTTP_TIMEOUT_INTERVAL` (60 seconds) expires, at which point a `FlexibleCanisterHttpError::Timeout` is finalized and the callback receives a timeout reject. The attack is repeatable on every flexible HTTP request the Byzantine node is assigned to, constituting a sustained, targeted subnet availability impact for HTTP outcalls â€” matching the **High** impact class: *Application/platform-level DoS or subnet availability impact not based on raw volumetric DDoS* ($2,000â€“$10,000).

## Likelihood Explanation

Requires a single Byzantine subnet node that is simultaneously the block proposer for a given round (a rotating role â€” a Byzantine node will eventually be proposer) and a member of the `flex_committee` for the targeted request. No threshold majority is needed. The attack is below the consensus fault threshold and is repeatable: the Byzantine node can re-trigger it on every subsequent flexible HTTP request it is assigned to.

## Recommendation

Add Candid-decode validation of `CanisterHttpResponseContent::Success(data)` inside `validate_canister_http_payload_impl` for each entry in `payload.flexible_responses`, mirroring the decode performed in `flexible_ok_responses_into_consensus_response`. Specifically, within the loop at lines 639â€“657, after the existing `validate_flexible_response_with_proof` call, attempt `Decode!(&data, CanisterHttpResponsePayload)` and return `invalid_artifact(InvalidCanisterHttpPayloadReason::...)` if it fails. This closes the gap between what is validated and what is executed.

## Proof of Concept

The existing test `flexible_ok_responses_into_messages_decode_failure_is_skipped` (tests.rs L3002â€“3026) already proves the drop behavior: one valid entry + one `b"this is invalid candid"` entry produces `responses.len() == 0` and `flexible_ok_responses_candid_failures == 1`. [6](#0-5) 

A complementary test should call `validate_payload` with the same payload (one valid entry + one `b"this is invalid candid"` entry, with correctly computed hash/size proof metadata and a valid signature from a committee member acting as block proposer) and assert `Ok(())` â€” confirming that validation passes while execution drops the group. This two-test pair constitutes a complete, reproducible proof of the validation gap.

pub(crate) fn validate_flexible_response_with_proof(
    response_with_proof: &FlexibleCanisterHttpResponseWithProof,
    callback_id: CallbackId,
    flex_committee: &BTreeSet<NodeId>,
    seen_signers: &mut HashSet<NodeId>,
    consensus_registry_version: RegistryVersion,
    per_replica_allowance: Cycles,
) -> Result<(), InvalidCanisterHttpPayloadReason> {
    if response_with_proof.response.id != callback_id {
        return Err(
            InvalidCanisterHttpPayloadReason::FlexibleCallbackIdMismatch {
                callback_id,
                mismatched_id: response_with_proof.response.id,
            },
        );
    }

    validate_response_share(
        &response_with_proof.proof,
        callback_id,
        flex_committee,
        seen_signers,
        consensus_registry_version,
        per_replica_allowance,
    )?;

    let calculated_hash = crypto_hash(&response_with_proof.response);
    if &calculated_hash != response_with_proof.proof.content.content_hash() {
        return Err(InvalidCanisterHttpPayloadReason::ContentHashMismatch {
            metadata_hash: response_with_proof.proof.content.content_hash().clone(),
            calculated_hash,
        });
    }

    let calculated_size = response_with_proof.response.content.count_bytes() as u32;
    if calculated_size != response_with_proof.proof.content.content_size() {
        return Err(InvalidCanisterHttpPayloadReason::ContentSizeMismatch {
            metadata_size: response_with_proof.proof.content.content_size(),
            calculated_size,
        });
    }

    let calculated_is_reject = response_with_proof.response.content.is_reject();
    if calculated_is_reject != response_with_proof.proof.content.is_reject() {
        return Err(InvalidCanisterHttpPayloadReason::IsRejectMismatch {
            metadata_is_reject: response_with_proof.proof.content.is_reject(),
            calculated_is_reject,
        });
    }

    Ok(())
}

#[test]
fn flexible_ok_responses_into_messages_decode_failure_is_skipped() {
    let callback_id = CallbackId::from(42);

    let valid_data = Encode!(&CanisterHttpResponsePayload {
        status: 200,
        headers: vec![],
        body: vec![],
    })
    .unwrap();
    let valid_entry = flexible_response(42, 0, &valid_data);
    let invalid_entry = flexible_response(42, 1, b"this is invalid candid");

    let payload = flexible_payload(vec![FlexibleCanisterHttpResponses {
        callback_id,
        responses: vec![valid_entry, invalid_entry],
    }]);
    let bytes = payload_to_bytes_max_4mb(payload);

    let (responses, stats) = CanisterHttpPayloadBuilderImpl::into_messages(&bytes);

    assert_eq!(responses.len(), 0);
    assert_eq!(stats.flexible_ok_responses, 0);
    assert_eq!(stats.flexible_ok_responses_candid_failures, 1);
}
