### Title
Unbounded `stakers` Vec Growth Causes Unbounded Gas Consumption in `get_stakers` - (File: src/staking/staking.cairo)

### Summary
The `stakers` storage Vec in `staking.cairo` is append-only: every call to `stake()` pushes a new address, but `unstake_action()` never removes it. The `get_stakers()` function iterates over the **entire** Vec on every call. An unprivileged attacker can cheaply inflate this Vec by repeatedly staking with fresh addresses and then unstaking to recover their STRK, leaving permanent ghost entries. As the Vec grows, `get_stakers()` consumes unbounded gas, eventually becoming uncallable.

### Finding Description

Every call to `stake()` appends the caller's address to `self.stakers`: [1](#0-0) 

The storage comment explicitly acknowledges this is permanent: [2](#0-1) 

`unstake_action()` clears `staker_info` and pool data but never removes the address from `self.stakers`: [3](#0-2) 

`get_stakers()` then iterates over every entry ever pushed, including all inactive/removed stakers: [4](#0-3) 

The loop has no bound and performs multiple storage reads per entry (`is_staker_active`, `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, `get_peer_id_at_epoch`).

### Impact Explanation

`get_stakers()` is the `IStakingConsensus` entrypoint used by the consensus layer to determine the active validator set for each epoch. As the `stakers` Vec grows without bound, the gas cost of `get_stakers()` grows linearly with the total number of stakers ever registered. An attacker who inflates the Vec sufficiently can make `get_stakers()` exceed Starknet's per-call gas/step limits, rendering the consensus query permanently uncallable. This constitutes **unbounded gas consumption** and griefing of the protocol with no profit motive.

**Allowed impact matched**: Medium — Griefing with no profit motive but damage to users or protocol; Unbounded gas consumption.

### Likelihood Explanation

The attack is cheap. The attacker stakes `min_stake` STRK from address A₁, waits for the exit window (~1 week), unstakes to recover the STRK, moves it to address A₂, and repeats. Each cycle costs only gas and produces one permanent ghost entry. Because `assert_staker_address_not_reused` prevents address reuse, each cycle requires a fresh address, but the STRK principal is fully recycled. The attacker's only ongoing cost is gas per cycle. No privileged access is required; `stake()` is open to any address. [5](#0-4) 

### Recommendation

1. **Track active staker count separately** and skip inactive entries in `get_stakers` more cheaply, or maintain a separate active-staker set that is pruned on `unstake_action`.
2. **Remove the address from `stakers` on `unstake_action`**, or use a swap-and-pop pattern to keep the Vec compact.
3. **Add a cap** on the total number of registered stakers (analogous to the `add_token` note in `IStakingTokenManager` which warns that unbounded token sets cause unbounded complexity). [6](#0-5) 

### Proof of Concept

```
Attacker controls addresses A₁, A₂, ..., Aₙ (all funded with min_stake STRK).

For i = 1 to N:
  1. Aᵢ calls stake(reward=Aᵢ, operational=Oᵢ, amount=min_stake)
     → self.stakers.push(Aᵢ)   // Vec length = i
  2. Aᵢ calls unstake_intent()
  3. Wait exit_wait_window (~1 week)
  4. Anyone calls unstake_action(Aᵢ)
     → STRK returned to Aᵢ; Vec entry for Aᵢ remains forever
  5. Transfer STRK from Aᵢ to Aᵢ₊₁

After N cycles, self.stakers.len() == N (all inactive).
get_stakers(epoch_id) iterates all N entries, each requiring
multiple storage reads. For large N, the call exceeds gas limits.
``` [7](#0-6)

            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
