### Title
Attestation Transaction Submitted Near Epoch Boundary or Window End Fails On-Chain, Causing Permanent Loss of Epoch Rewards — (File: `src/attestation/attestation.cairo`)

---

### Summary

A staker's `attest()` transaction can pass simulation but revert on-chain because the attestation validity window is evaluated at **execution time**, not at submission time. If the transaction is delayed by even one block past the epoch boundary or past `max_block`, the staker permanently loses all rewards for that epoch with no recourse.

---

### Finding Description

The `attest()` function in `src/attestation/attestation.cairo` calls `get_attestation_info_by_operational_address` on the staking contract, which reads the **current epoch** at the moment of on-chain execution: [1](#0-0) 

Inside `_validate_attestation`, two time-sensitive checks are performed using the live block number:

**Check 1 — Attestation window check** (`_assert_attest_in_window`): [2](#0-1) 

The valid range is `[target_attestation_block + 11, target_attestation_block + attestation_window]`. If the transaction is included even one block after `max_block`, it reverts with `ATTEST_OUT_OF_WINDOW`.

**Check 2 — Block hash check**: [3](#0-2) 

The `target_attestation_block` is computed from a Poseidon hash of `(stake, epoch_id, staker_address)`: [4](#0-3) 

If the transaction crosses an **epoch boundary** between simulation and execution, `epoch_id` and `current_epoch_starting_block` both change, producing a completely different `target_attestation_block`. The staker's pre-computed `block_hash` (for epoch N's target block) will not match the hash of epoch N+1's target block, causing `ATTEST_WRONG_BLOCK_HASH`.

The `get_attestation_info_by_operational_address` function reads the live epoch at execution time: [5](#0-4) 

There is **no guard** in the contract preventing a staker from submitting an attestation when only 1–2 blocks remain in the epoch or in the attestation window. The `MIN_ATTESTATION_WINDOW` constant (11 blocks) only sets the *start* of the valid range, not a minimum remaining time before the window closes: [6](#0-5) 

---

### Impact Explanation

In the pre-consensus phase, `update_rewards_from_attestation_contract` is the **sole mechanism** by which a staker accrues epoch rewards: [7](#0-6) 

A failed `attest()` means this function is never called. The staker cannot retroactively attest for a missed epoch — `_assert_attestation_is_not_done` only checks the current epoch: [8](#0-7) 

**Result**: Permanent, irrecoverable loss of one full epoch's staking rewards. This maps to **High impact** under the allowed scope: *"Theft of unclaimed yield"* / *"Permanent freezing of unclaimed yield."*

---

### Likelihood Explanation

**Medium.** Stakers are incentivized to attest as late as possible in the window to maximize certainty that the target block hash is available. Submitting 1–3 blocks before `max_block` or before the epoch boundary is a natural operational pattern. On Starknet

        fn attest(ref self: ContractState, block_hash: felt252) {
            let operational_address = get_caller_address();
            let staking_dispatcher = IStakingAttestationDispatcher {
                contract_address: self.staking_contract.read(),
            };
            // Note: This function checks for a zero staker address and will panic if so.
            let staking_attestation_info = staking_dispatcher
                .get_attestation_info_by_operational_address(:operational_address);
            self._validate_attestation(:block_hash, :staking_attestation_info);
            // Work is one tx per epoch.
            self
                ._mark_attestation_is_done(
                    staker_address: staking_attestation_info.staker_address(),
                    current_epoch: staking_attestation_info.epoch_id(),
                );
            staking_dispatcher
                .update_rewards_from_attestation_contract(
                    staker_address: staking_attestation_info.staker_address(),
                );
        }

        fn _assert_attest_in_window(self: @ContractState, target_attestation_block: BlockNumber) {
            let attestation_window = self.attestation_window.read();
            let current_block_number = get_block_number();
            let min_block = target_attestation_block + MIN_ATTESTATION_WINDOW.into();
            let max_block = target_attestation_block + attestation_window.into();
            assert!(
                min_block <= current_block_number && current_block_number <= max_block,
                "{}",
                Error::ATTEST_OUT_OF_WINDOW,
            );
        }

        fn get_attestation_info_by_operational_address(
            self: @ContractState, operational_address: ContractAddress,
        ) -> AttestationInfo {
            let staker_address = self.get_staker_address_by_operational(:operational_address);

            // Return the attestation info.
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            let epoch_info = self.get_epoch_info();
            let epoch_len = epoch_info.epoch_len_in_blocks();
            let epoch_id = epoch_info.current_epoch();
            let current_epoch_starting_block = epoch_info.current_epoch_starting_block();
            let stake = self
                .get_staker_total_strk_balance_at_epoch(
                    :staker_address, :staker_pool_info, :epoch_id,
                )

                :staker_address, :stake, :epoch_len, :epoch_id, :current_epoch_starting_block,
