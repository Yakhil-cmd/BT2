### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Block Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the staking contract is specified to be callable only by the Starknet sequencer, but the implementation contains no such access control check. Any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` to consume a block's reward slot without distributing any rewards, permanently freezing that block's yield for all stakers.

---

### Finding Description

`update_rewards` is exposed as part of the `IStakingRewardsManager` interface: [1](#0-0) 

The protocol specification explicitly restricts this function:

> **access control**: Only starkware sequencer. [2](#0-1) 

However, the implementation only calls `self.general_prerequisites()` (a pause check) and a block-number guard — there is no check that the caller is the sequencer: [3](#0-2) 

The critical path is:

1. The function checks `current_block_number > self.last_reward_block.read()` — one call per block is allowed.
2. It unconditionally writes `self.last_reward_block.write(current_block_number)` at line 1485, consuming the block's reward slot.
3. If `disable_rewards` is `true`, it returns immediately without distributing any rewards. [4](#0-3) 

Because `last_reward_block` is now equal to the current block, every subsequent call to `update_rewards` for that block will revert with `REWARDS_ALREADY_UPDATED`: [5](#0-4) 

The block's rewards are permanently unrecoverable — there is no mechanism to retroactively distribute rewards for a past block.

`general_prerequisites` cannot contain a sequencer check because it is also called by user-facing functions such as `stake` and `unstake_intent`, confirming the absence of any caller restriction in `update_rewards`.

---

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` once per block. Each call:
- Marks the block as "already rewarded" (`last_reward_block = current_block`)
- Distributes zero rewards

All stakers and their delegators permanently lose the block rewards that should have accrued. This constitutes **permanent freezing of unclaimed yield** for the entire protocol, matching the allowed High impact.

---

### Likelihood Explanation

- The function is publicly callable with no access control.
- Valid staker addresses are observable on-chain via `NewStaker` events.
- The attack costs only gas per block and requires no capital.
- A single persistent attacker can suppress all consensus block rewards indefinitely.

---

### Recommendation

Add an explicit sequencer-only access control check at the top of `update_rewards`, consistent with the specification. For example, store the sequencer address in contract storage during initialization and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation. Alternatively, use the existing `RolesComponent` to define and enforce a `SEQUENCER_ROLE`.

---

### Proof of Concept

```
// Pseudocode — attacker script running once per block
loop every new block:
    staking_contract.update_rewards(
        staker_address = any_active_staker,   // publicly known
        disable_rewards = true,               // no rewards distributed
    )
    // last_reward_block is now == current block
    // legitimate sequencer call will revert with REWARDS_ALREADY_UPDATED
    // block rewards are permanently lost
```

The only precondition is that `current_block_number > last_reward_block`, which is trivially satisfied once per block. The attacker front-runs the sequencer's legitimate call, consuming the slot with `disable_rewards: true`. [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
