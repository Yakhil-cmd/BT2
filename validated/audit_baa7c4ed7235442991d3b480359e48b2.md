### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6` / `addFeeSellE6` directly to the pool with no cap check, while the parallel `setPoolAdminFees` path is explicitly bounded by `maxAdminSpreadFeeE6`. A pool admin can therefore set per-bin additional fees up to `uint16` max (65 535 ≈ 6.55 % in E6 units) on every bin, completely bypassing the factory-owner-controlled spread-fee cap — including when that cap has been set to zero.

---

### Finding Description

`setPoolAdminFees` enforces the factory-owner cap before writing:

```solidity
// MetricOmmPoolFactory.sol  408-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees` has no equivalent guard — it forwards the caller-supplied values straight to the pool:

```solidity
// MetricOmmPoolFactory.sol  450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool stores the values verbatim and adds them to `baseFeeX64` on every swap through the affected bin:

```solidity
// MetricOmmPool.sol  910, 999
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [3](#0-2) 

`BinState.addFeeBuyE6` / `addFeeSellE6` are `uint16`, so the maximum injectable additional fee is 65 535 / 1 000 000 ≈ **6.55 %** per bin, per direction — on top of whatever `spreadFeeE6` already is. [4](#0-3) 

The factory-owner-controlled `maxAdminSpreadFeeE6` (hard ceiling `HARD_MAX_SPREAD_FEE_E6 = 200 000`) is the intended mechanism to bound all admin-controlled spread costs: [5](#0-4) 

Because `setPoolBinAdditionalFees` never reads that cap, the pool admin can exceed it on any or all bins regardless of what the factory owner has configured.

---

### Impact Explanation

Every swap that crosses a bin with elevated `addFeeBuyE6` / `addFeeSellE6` pays a higher effective spread than the protocol's cap permits. The excess fee is embedded in the swap price (trader receives less output / pays more input) and accrues to LPs. Traders suffer a direct, quantifiable loss on every affected swap. With `maxAdminSpreadFeeE6 = 0` (factory owner fully suppresses admin spread) a pool admin can still impose up to 6.55 % additional spread per bin — a complete bypass of the cap invariant.

---

### Likelihood Explanation

Requires a malicious or compromised pool admin. Pool admins are semi-trusted and are expected to operate within factory-owner-set caps; this path lets them silently exceed those caps without any on-chain signal beyond the `BinAdditionalFeesUpdated` event. Likelihood is low but the trigger is a single permissioned call with no timelock.

---

### Recommendation

Add a cap check inside `setPoolBinAdditionalFees` analogous to the one in `setPoolAdminFees`. The simplest approach is to bound each bin additional fee against `maxAdminSpreadFeeE6`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce a combined cap (global admin spread + per-bin additional) so the total admin-controlled spread on any bin cannot exceed `maxAdminSpreadFeeE6`.

---

### Proof of Concept

1. Factory owner calls `setFeeCaps(200_000, 0, 1_000_000, 1_000_000)` — setting `maxAdminSpreadFeeE6 = 0` to prevent the pool admin from charging any admin spread fee.
2. Pool admin calls `setPoolAdminFees(pool, 1, 0)` → reverts with `AdminFeeTooHigh` as expected.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` → **succeeds**, writing `addFeeBuyE6 = addFeeSellE6 = 65535` to bin 0.
4. Any swap through bin 0 now pays an additional ≈ 6.55 % spread on top of the base fee, despite the factory owner having set the admin spread cap to zero. [2](#0-1) [6](#0-5)
