### Title
Unbounded `pool_member_epoch_balance` Trace Growth Enables Permanent Freezing of Unclaimed Yield - (File: `src/pool/pool.cairo`)

### Summary
A pool member can inflate their own `pool_member_epoch_balance` trace to an arbitrary size by repeatedly calling `exit_delegation_pool_intent(0)` across successive epochs. The `calculate_rewards` function contains an explicitly acknowledged unbounded loop over this trace. Once the trace is large enough, every `claim_rewards` call will exceed the Starknet gas limit, permanently freezing the delegator's unclaimed yield with no recovery path.

### Finding Description

**Root cause — no lower bound on `exit_delegation_pool_intent` amount:**

`exit_delegation_pool_intent` in `src/pool/pool.cairo` only checks that `amount <= total_amount`; it does not require `amount > 0`: [1](#0-0) 

When called with `amount = 0`, the function still calls `set_member_balance` with the unchanged balance, which unconditionally calls `trace.insert(key: current_epoch + K, ...)`: [2](#0-1) 

The `insert` implementation appends a **new** checkpoint whenever the epoch key differs from the last entry's key: [3](#0-2) 

Because the key is always `current_epoch + K`, calling `exit_delegation_pool_intent(0)` once per epoch (after the epoch advances) appends one new checkpoint per call, growing the trace without bound and without moving any tokens.

**Unbounded loop in `calculate_rewards`:**

`claim_rewards` calls `calculate_rewards`, which iterates over every entry in the trace from `entry_to_claim_from` up to the current epoch. The code itself acknowledges the loop is unbounded: [4](#0-3) 

`entry_to_claim_from` is only advanced on a **successful** claim. If the trace has grown large enough that the transaction runs out of gas, the state is never updated, so every subsequent attempt also fails — the delegator is permanently locked out of their rewards.

### Impact Explanation

Once the trace exceeds the gas budget for a single transaction, `claim_rewards` (and `pool_member_info_v1`, which also calls `calculate_rewards`) will always revert. The delegator's accrued STRK rewards become permanently unclaimable. This matches the allowed impact: **Permanent freezing of unclaimed yield (High)**.

### Likelihood Explanation

**Low.** The attack requires the delegator to submit one transaction per epoch for many epochs without ever claiming rewards. There is no direct profit motive; the delegator harms only themselves. However, the attack is fully permissionless, costs only gas, and requires no privileged access.

### Recommendation

1. **Require `amount > 0` in `exit_delegation_pool_intent`** — add `assert!(amount.is_non_zero(), ...)` to prevent zero-amount calls that serve no economic purpose but still write a new trace entry.
2. **Alternatively, add a partial-claim parameter** to `claim_rewards` so a delegator can process a bounded number of trace entries per call, allowing recovery even after trace inflation.
3. **Consider a hard cap** on the number of entries in `pool_member_epoch_balance` per pool member.

### Proof of Concept

```
1. Delegator enters pool with minimum stake.
2. For each epoch i = 1 … N (without ever calling claim_rewards):
     a. Advance epoch.
     b. Call exit_delegation_pool_intent(amount: 0).
        → set_member_balance is called with unchanged balance.
        → trace.insert appends a new checkpoint (key = epoch_i + K).
3. After N epochs, pool_member_epoch_balance has N+1 entries.
4. Call claim_rewards.
   → calculate_rewards loops over all N entries.
   → If N is large enough, the transaction runs out of gas and reverts.
   → entry_to_claim_from is never updated.
5. Every future claim_rewards call also reverts.
   → Delegator's accumulated STRK rewards are permanently frozen.
```

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
