### Title
Atomic Multi-Transfer in `unstake_action` Permanently DoS'd via Zero Reward Address — (File: `src/staking/staking.cairo`)

---

### Summary

`unstake_action` in `src/staking/staking.cairo` performs three sequential fund transfers atomically: (1) rewards to the staker's `reward_address`, (2) principal back to the staker, and (3) delegated stake to pool contracts. Because `change_reward_address` imposes no zero-address guard, a staker can set `reward_address` to `ContractAddress::zero()`. When `unstake_action` is subsequently called, the first transfer (`send_rewards_to_staker`) reverts on the zero-address recipient, permanently blocking the return of the staker's principal **and** the return of all delegated funds to pool contracts — freezing delegators' assets with no recovery path.

---

### Finding Description

`unstake_action` (lines 483–515) executes the following transfers in strict sequence:

```cairo
// 1. Rewards → reward_address
self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);

// 2. Principal → staker_address
token_dispatcher.checked_transfer(recipient: staker_address, amount: staker_amount.into());

// 3. Delegated stake → pool contracts
self.transfer_to_pools_when_unstake(:staker_address, ...);
``` [1](#0-0) 

`send_rewards_to_staker` unconditionally calls `checked_transfer` to `reward_address` with no zero-address guard:

```cairo
fn send_rewards_to_staker(...) {
    let reward_address = staker_info.reward_address;
    let amount = staker_info.unclaimed_rewards_own;
    ...
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
``` [2](#0-1) 

`change_reward_address` only checks that the new address is not a registered token address. There is **no zero-address check**:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    assert!(
        !self.does_token_exist(token_address: reward_address),
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    ...
    staker_info.reward_address = reward_address;
``` [3](#0-2) 

OpenZeppelin Cairo ERC20 (the STRK token) reverts on `transfer` to `ContractAddress::zero()`. Therefore, if `reward_address` is zero, step 1 reverts, and steps 2 and 3 never execute. The staker's principal and all delegated balances remain permanently locked in the staking contract.

---

### Impact Explanation

- **Staker's principal is permanently frozen**: `checked_transfer(recipient: staker_address, ...)` (step 2) never executes.
- **Delegators' funds are permanently frozen**: `transfer_to_pools_when_unstake` (step 3) never executes. Per the spec, this is the mechanism by which delegated stake is returned to pool contracts so delegators can withdraw. With it blocked, delegators have no recovery path.
- **Severity**: High — permanent freezing of delegators' unclaimed principal (funds at rest).

---

### Likelihood Explanation

- `change_reward_address` is callable by any staker at any time, including after `unstake_intent` (no check for `unstake_time.is_none()`).
- A malicious staker can deliberately set `reward_address = 0`, call `unstake_intent`, wait out the exit window, and then allow `unstake_action` to be called (it is permissionless — "Any address can execute").
- The staker sacrifices their own rewards and principal to permanently freeze all delegators' funds — a credible griefing vector with no profit requirement.
- Accidental self-lock is also possible (user sets reward_address to 0 by mistake).

---

### Recommendation

1. Add a zero-address guard in `change_reward_address`:
   ```cairo
   assert!(!reward_address.is_zero(), "{}", Error::REWARD_ADDRESS_IS_ZERO);
   ```
2. Add the same guard in the initial `stake` function for the `reward_address` parameter.
3. Consider skipping the reward transfer in `send_rewards_to_staker` when `amount == 0` to avoid any edge-case revert on zero-amount transfers.
4. As a defense-in-depth measure (analogous to the report's recommendation), consider separating the reward claim from the principal/pool-return transfers so that a failed reward transfer does not block fund recovery.

---

### Proof of Concept

1. Staker calls `stake(...)` with a valid `reward_address`.
2. Staker calls `change_reward_address(reward_address: ContractAddress::zero())` — succeeds because the only check is `!does_token_exist(zero)`, which passes.
3. Staker calls `unstake_intent()`.
4. Exit window elapses.
5. Anyone calls `unstake_action(staker_address)`.
6. `send_rewards_to_staker` calls `token_dispatcher.checked_transfer(recipient: ContractAddress::zero(), amount: ...)` — OZ ERC20 reverts with `TRANSFER_TO_ZERO`.
7. The entire `unstake_action` transaction reverts.
8. Steps 2 and 3 (principal return and pool transfers) never execute.
9. Staker's principal and all delegated balances are permanently locked in the staking contract with no alternative withdrawal path. [1](#0-0) [2](#0-1) [3](#0-2)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.

                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,

        fn send_rewards_to_staker(
            ref self: ContractState,
            staker_address: ContractAddress,
            ref staker_info: InternalStakerInfoLatest,
            token_dispatcher: IERC20Dispatcher,
        ) {
            let reward_address = staker_info.reward_address;
            let amount = staker_info.unclaimed_rewards_own;
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();

            claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
            token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
            staker_info.unclaimed_rewards_own = Zero::zero();

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
