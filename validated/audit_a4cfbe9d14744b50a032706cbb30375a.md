### Title
Malicious Participant in Sorted Positions t+1..2t Can Abort Presign for All Honest Parties via Crafted W Share — (`src/ecdsa/robust_ecdsa/presign.rs`)

---

### Summary

In `do_presign`, the W-share exponent interpolation check at step 3.9 uses the first `t+1` sorted participants as the interpolation basis and verifies the remaining `t` participants against it. A single malicious participant whose sorted index falls in positions `t+1..2t` can send a crafted `W_me = R^{a_me + delta}` (delta ≠ 0), causing the check to fail and returning `Err(AssertionFailed)` for every honest party. No participant is identified or excluded; the entire presign session aborts with no recovery path.

---

### Finding Description

**Protocol invariant**: With N = 2t+1 and at most t malicious parties, presign must complete for all honest parties.

**Participant ordering**: `ParticipantList::new()` sorts participants by their `u32` ID before storing them. [1](#0-0) 

`ParticipantMap::into_vec_or_none()` returns data in that same sorted order. [2](#0-1) 

**Basis vs. checked split**: After collecting all 2t+1 W shares, the code splits them at index `t+1`:
- `threshold_plus1_wshares` = positions `0..=t` (the interpolation basis)
- The loop iterates positions `t+1..2t` (the "checked" set) [3](#0-2) 

**The check**: For each participant `i` in the checked set, the code interpolates `W_i` from the basis and compares it to the received value. If they differ, it immediately returns `Err(AssertionFailed)`. [4](#0-3) 

**Attack**: A malicious participant whose sorted position is `t+1` (or any index in `t+1..2t`) sends `W_me = R^{a_me} * R^delta` for any attacker-chosen `delta ≠ 0`. The interpolation from the t+1 honest basis shares correctly predicts `R^{a_me}`, but the received value is `R^{a_me + delta}`. The equality check fails. Every honest party executing this check returns the same error and aborts.

**No recovery**: The error message is generic (`"Exponent interpolation check failed."`) — it does not identify which participant sent the bad share. There is no mechanism to exclude the offender and retry with the remaining honest parties. [5](#0-4) 

---

### Impact Explanation

**High — Permanent denial of presign for honest parties.** A single malicious participant (one of the t allowed) can unconditionally abort every presign attempt. Because the protocol requires exactly N = 2t+1 participants (enforced at line 75), and the error gives no attribution, honest parties cannot simply restart without the attacker. The attacker can repeat this on every session, permanently blocking signing. [6](#0-5) 

---

### Likelihood Explanation

Any participant whose sorted `u32` ID places them at index `t+1` or higher can execute this attack. Participant IDs are caller-supplied, so an attacker can trivially choose an ID that lands in the checked range. The attack requires only sending one malformed group element in round 3 — no cryptographic computation is needed.

---

### Recommendation

When the interpolation check fails for a participant in the checked set, the protocol should **identify and record the offending participant** rather than aborting immediately. With t+1 honest W shares already in the basis, the protocol can complete successfully using only those shares. Alternatively, implement a blame/attribution round that names the cheating participant so honest parties can restart with a reduced set.

---

### Proof of Concept

```
Setup: t=2, N=5 participants sorted as [P0, P1, P2, P3, P4]
       Attacker controls P3 (sorted index 3 = t+1)

Round 1-2: All parties run honestly, producing valid k_i, a_i, w_i, R_i shares.

Round 3 (step 3.8): P3 broadcasts W_P3 = R^{a_P3 + delta}  (delta = 1, for example)

Step 3.9 at each honest party:
  threshold_plus1_wshares = [W_P0, W_P1, W_P2]   (basis, all honest)
  checked set = [W_P3, W_P4]

  For P3:
    interpolated_W_P3 = eval_exponent_interpolation([P0,P1,P2], [W_P0,W_P1,W_P2], P3)
                      = R^{a_P3}          (correct, from honest basis)
    received W_P3     = R^{a_P3 + delta}  (crafted)
    R^{a_P3} != R^{a_P3+delta}  =>  Err(AssertionFailed)

All 4 honest parties (P0, P1, P2, P4) return Err("Exponent interpolation check failed.")
Presign aborts. No attribution. Attacker repeats on next session.
```
