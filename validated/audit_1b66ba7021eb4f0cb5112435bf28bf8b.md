### Title
WalletConnect Offer Confirmation Understates Total Royalty Cost for Multi-NFT Offers - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
The `formatAmountWithRoyalties` function used to populate the WalletConnect confirmation dialog incorrectly computes the total fungible cost (including royalties) when an offer involves multiple NFTs with royalties. It divides the payment amount equally among NFTs before computing each royalty, producing a displayed total that is `1/N` of the correct royalty for `N` NFTs. A malicious dApp can exploit this to cause a user to approve a WalletConnect offer-acceptance believing they are spending less than the wallet will actually deduct.

### Finding Description
In `parseCommandDisplay.ts`, `formatAmountWithRoyalties` is called for every fungible spending line in a WalletConnect `chia_wallet.take_offer` or `chia_wallet.create_offer_for_ids` command to produce the `amountWithRoyalties` field shown in the confirmation dialog:

```ts
// packages/gui/src/electron/commands/parseCommandDisplay.ts L363-L368
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← integer-divides by NFT count
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

The `splitAmount` is `amount / N` (BigInt truncating division). Each NFT's royalty is then computed on this reduced base. The net effect is:

```
displayed royalty = amount × Σ(rᵢ) / (N × 10 000)
correct royalty   = amount × Σ(rᵢ) / 10 000
```

For `N = 2` NFTs with royalties 500 (5 %) and 10 (0.1 %) and a payment of 100 000 000 mojos (0.0001 XCH):

| | Displayed | Correct |
|---|---|---|
| royaltyAmount | 2 550 000 mojos | 5 100 000 mojos |
| totalAmount | 102 550 000 mojos (0.00010255 XCH) | 105 100 000 mojos (0.0001051 XCH) |

This matches the existing test fixture, which asserts `amountWithRoyalties: '0.00010255'` — the understated value — as the expected output: [2](#0-1) 

The `amountWithRoyalties` string is rendered verbatim in the WalletConnect confirmation dialog under the label **"Total Amount with Royalties"**: [3](#0-2) 

The `parseCommandDisplay` function is the sole source of `walletDelta` for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` WalletConnect commands: [4](#0-3) 

The wallet daemon computes and deducts the **correct** royalty amount on-chain; only the GUI confirmation display is wrong.

### Impact Explanation
A malicious dApp with an established WalletConnect pairing sends `chia_takeOffer` with an offer containing multiple NFTs that each carry royalties. The confirmation dialog shows an understated "Total Amount with Royalties." The user approves, believing they are spending `X` XCH/CAT, but the wallet deducts `X + Δ` where `Δ` grows with the number of NFTs and their royalty percentages. This is a direct, concrete asset loss caused by WalletConnect state causing the user to approve the wrong amount — matching the **High** impact category: *"WalletConnect state that causes a user to approve the wrong amount."*

### Likelihood Explanation
Any dApp that has obtained a WalletConnect pairing (the normal, intended integration path) can trigger this by constructing a multi-NFT offer. No special privileges, leaked keys, or host compromise are required. The discrepancy is invisible to the user in the confirmation dialog and scales with the number of NFTs and royalty percentages involved.

### Recommendation
Remove the `splitAmount` division. Each NFT's royalty should be computed on the full fungible `amount`:

```ts
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

Update the corresponding test fixtures to reflect the corrected (higher) totals.

### Proof of Concept
1. A dApp with WalletConnect pairing sends `chia_takeOffer` with an offer blob where the maker offers 2 NFTs (royalty 500 and 10 respectively) and requests 100 000 000 mojos XCH.
2. `parseCommandDisplay` calls `formatAmountWithRoyalties(line, 100_000_000n, [500, 10])`.
3. `splitAmount = 100_000_000n / 2n = 50_000_000n`.
4. `royaltyAmount = (50_000_000n × 500n / 10_000n) + (50_000_000n × 10n / 10_000n) = 2_500_000n + 50_000n = 2_550_000n`.
5. Dialog displays **0.00010255 XCH** as the total.
6. Correct total is `(100_000_000 × 500 / 10_000) + (100_000_000 × 10 / 10_000) = 5_000_000 + 100_000 = 5_100_000` → **0.0001051 XCH**.
7. User approves at 0.00010255 XCH; wallet deducts 0.0001051 XCH — a ~2.4 % overcharge invisible at confirmation time, scaling arbitrarily with NFT count and royalty rates.

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }
