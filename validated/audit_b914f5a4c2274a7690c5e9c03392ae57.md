### Title
Unprivileged Caller Can Front-Run `update_rewards` with `disable_rewards: true` to Permanently Freeze Block Reward Distribution — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is documented as callable only by the Starknet sequencer, but contains no access-control check. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` gate without distributing any rewards. Because the gate is global and single-valued, the sequencer's legitimate call in the same block is then rejected with `REWARDS_ALREADY_UPDATED`, and no staker receives block rewards for that block. An attacker can repeat this every block to permanently freeze all consensus-era block reward accrual.

---

### Finding Description

**Root cause — missing caller guard in `update_rewards`**

`update_rewards` calls `general_prerequisites()` as its only entry check. Because `general_prerequisites()` is also the first call in every user-facing function (`stake`, `increase_stake`, `claim_rewards`, `unstake_intent`, etc.), it cannot contain a sequencer-only restriction; it only enforces the pause flag. No additional `assert_caller_is_sequencer()` (or equivalent) is present in `update_rewards`, unlike the explicit `assert_caller_is_attestation_contract()` guard in `update_rewards_from_attestation_contract`. [1](#0-0) 

**The gate is written before the `disable_rewards` branch**

```
self.last_reward_block.write(current_block_number);   // ← gate advanced here

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← no rewards distributed
}
``` [2](#0-1) 

Any caller who supplies `disable_rewards: true` advances `last_reward_block` to the current block and returns immediately. The sequencer's subsequent call in the same block hits:

```
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

and reverts, leaving `unclaimed_rewards_own` and all pool reward traces unmodified for that block.

**Contrast with properly guarded functions**

`update_rewards_from_attestation_contract` carries an explicit `assert_caller_is_attestation_contract()` call, demonstrating the pattern the protocol uses when it intends to restrict a function to a specific caller. [4](#0-3) 

`update_rewards` has no equivalent guard.

---

### Impact Explanation

`update_rewards` is the sole mechanism for distributing consensus-era block rewards to stakers and their pools. If it is blocked every block, `unclaimed_rewards_own` for every staker never increases, and `update_rewards_from_staking_contract` on the `RewardSupplier` is never called, so the `unclaimed_rewards` ledger is never credited. [5](#0-4) 

This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators — a **High** severity impact under the allowed scope.

---

### Likelihood Explanation

- The attacker needs only one valid, active staker address (public on-chain information) and enough gas to submit one transaction per block.
- No privileged key, no oracle, no bridge compromise is required.
- The attack is front-running: the attacker watches the mempool for the sequencer's `update_rewards` call and submits the same call with `disable_rewards: true` at a higher fee.
- On Starknet, transaction ordering within a block is controlled by the sequencer, but the sequencer is the same entity that is supposed to call `update_rewards`. If the sequencer is honest but the function is public, any other party can race it. Even without mempool visibility, the attacker can simply call `update_rewards(any_active_staker, disable_rewards: true)` as the first transaction of every block unconditionally.

---

### Recommendation

Add an explicit sequencer-only access-control check at the top of `update_rewards`, analogous to the pattern already used in `update_rewards_from_attestation_contract`:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.assert_caller_is_sequencer(); // add this guard
    self.general_prerequisites();
    ...
}
```

Alternatively, remove the `disable_rewards` parameter entirely and derive the flag internally, so no external caller can suppress reward distribution while still advancing the block gate.

---

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`). Staker `S` is active with non-zero STRK balance.
2. At block `N`, the sequencer prepares to call `update_rewards(S, false)`.
3. Attacker submits `update_rewards(S, true)` in the same block with higher priority.
4. Attacker's call passes all checks, writes `last_reward_block = N`, and returns early — no rewards distributed.
5. Sequencer's call hits `assert!(N > N)` → `REWARDS_ALREADY_UPDATED` → reverts.
6. `S.unclaimed_rewards_own` is unchanged; `RewardSupplier.unclaimed_rewards` is unchanged.
7. Attacker repeats at block `N+1`, `N+2`, … indefinitely.
8. All stakers and pool members are permanently denied block rewards. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
