### Title
Gateway Stateful Admission Accepts Transactions With Zero L1/L1-Data Gas Price That Always Fail Blockifier Pre-Validation — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateless and stateful validators only enforce a minimum price floor on `l2_gas.max_price_per_unit`. Neither validator checks `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` against the current block's L1 gas prices. A transaction carrying `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` (but a valid `l2_gas.max_price_per_unit`) passes every gateway check and is admitted to the mempool. When the blockifier later calls `check_fee_bounds`, it compares each resource's `max_price_per_unit` against the block's actual (always non-zero) gas prices and unconditionally rejects the transaction with `MaxGasPriceTooLow`. Because the failure occurs in `perform_pre_validation_stage` — before any execution — no fee is charged. This is the direct sequencer analog of the external report's `minAmountOut = 0`: a bound that should constrain admission is silently set to zero, allowing execution to proceed under conditions that guarantee failure.

### Finding Description

**Stateless validator** (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, lines 56–88):


Only `l2_gas.max_price_per_unit` is compared against `min_gas_price`. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never checked against any floor.

**Blockifier pre-validation** (`crates/blockifier/src/transaction/account_transaction.rs`, lines 441–448):

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { ... }
    );
}
```

This check runs for **all three** resources (L1Gas, L1DataGas, L2Gas). `actual_gas_price` is typed `NonzeroGasPrice`, so it is always `> 0`. A transaction with `l1_gas.max_price_per_unit = 0` therefore always fails here.

The failure path is `perform_pre_validation_stage` → `check_fee_bounds` → `TransactionPreValidationError`. Because this is a pre-validation error (not a post-execution revert), no fee is charged to the sender.

### Impact Explanation

An attacker can craft `AllResources` transactions where:
- `l2_gas.max_price_per_unit ≥ min_gas_price` (passes both gateway checks)
- `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0`

These transactions pass the full gateway admission pipeline (`stateless_tx_validator.validate` → `stateful_tx_validator.extract_state_nonce_and_run_validations`) and are inserted into the mempool. At sequencing time the blockifier unconditionally rejects them at pre-validation with no fee collected. The mempool is polluted with permanently-invalid, fee-free transactions, constituting a resource-exhaustion vector against the sequencer's mempool and batcher.

**Matching impact**: *High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.*

### Likelihood Explanation

The attack requires only a valid sender address and a valid nonce. No privileged access is needed. The attacker pays no fee for the rejected transactions. The condition is deterministic: L1 gas prices are always non-zero (`NonzeroGasPrice`), so every such transaction will always fail at blockifier pre-validation. The gap is acknowledged by the in-code TODO comment, confirming it is a known, unresolved omission.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or rename it) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK gas prices, mirroring the three-resource check already performed by `check_fee_bounds` in the blockifier. The stateless validator's `min_gas_price` check should similarly be applied to all three resource prices, not only `l2_gas`.

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with:
   ```
   resource_bounds = AllResourceBounds {
       l2_gas: { max_amount: 1_000_000, max_price_per_unit: 8_000_000_000 },  // ≥ min_gas_price
       l1_gas: { max_amount: 1_000_000, max_price_per_unit: 0 },              // zero — not checked
       l1_data_gas: { max_amount: 1_000_000, max_price_per_unit: 0 },         // zero — not checked
   }
   ```
2. Submit to the gateway. `StatelessTransactionValidator::validate` passes: `max_possible_fee > 0` (l2_gas contributes), `l2_gas.max_price_per_unit ≥ min_gas_price`.
3. `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` passes: only `l2_gas.max_price_per_unit` is compared against the previous block's L2 price.
4. Transaction is inserted into the mempool.
5. At sequencing, `AccountTransaction::perform_pre_validation_stage` calls `check_fee_bounds`. For L1Gas: `0 < actual_l1_gas_price` → `MaxGasPriceTooLow`. Transaction fails with a pre-validation error; no fee is charged.

```rust
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
```

```rust
pub struct StatefulTransactionValidatorConfig {
    // If true, ensures the max L2 gas price exceeds (a configurable percentage of) the base gas
    // price of the previous block.
    pub validate_resource_bounds: bool,
    pub max_allowed_nonce_gap: u32,
    pub reject_future_declare_txs: bool,
    pub max_nonce_for_validation_skip: Nonce,
    pub versioned_constants_overrides: Option<VersionedConstantsOverrides>,
    // Minimum gas price as percentage of threshold to accept transactions.
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}

impl Default for StatefulTransactionValidatorConfig {
    fn default() -> Self {

            max_nonce_for_validation_skip: Nonce(Felt::ONE),
            min_gas_price_percentage: 100,
            versioned_constants_overrides: None,
        }
    }
}
```
