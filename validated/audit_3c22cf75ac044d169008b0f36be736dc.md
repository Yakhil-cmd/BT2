### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary

`Staking.update_rewards` is callable by any address with any value of `disable_rewards`. The spec mandates "Only starkware sequencer" access control, but the implementation enforces none. An unprivileged attacker can front-run every legitimate sequencer call with `disable_rewards: true`, advancing `last_reward_block` without distributing rewards, permanently starving all stakers of their yield.

### Finding Description

`IStakingRewardsManager.update_rewards` is the consensus-phase reward distribution entry point. Its implementation in `staking.cairo` is:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← written BEFORE the guard

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // ← exits without distributing
    }
    ...
``` [1](#0-0) 

The critical sequence is:
1. `last_reward_block` is written to `current_block_number` unconditionally.
2. If `disable_rewards == true`, the function returns immediately — no rewards are calculated or transferred.

There is **no caller check** anywhere in this path. `general_prerequisites` only asserts the contract is not paused. [2](#0-1) 

The protocol specification explicitly states the access control for this function is **"Only starkware sequencer"**: [3](#0-2) 

The test suite confirms the absence of any caller restriction — `update_rewards` is invoked in tests with no `cheat_caller_address_once` setup: [4](#0-3) [5](#0-4) 

### Impact Explanation

An attacker who calls `update_rewards(staker_address, disable_rewards: true)` at every block:

- Advances `last_reward_block` to the current block without distributing any rewards.
- Causes every subsequent legitimate sequencer call to revert with `REWARDS_ALREADY_UPDATED` (since `current_block_number > last_reward_block` is now false for that block).
- Stakers accumulate zero `unclaimed_rewards_own`; delegation pools receive zero STRK.

This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators as long as the attacker sustains the attack. On Starknet, transaction fees are low enough to make this economically viable indefinitely.

**Allowed impact matched:** *Permanent freezing of unclaimed yield or unclaimed royalties* (High).

### Likelihood Explanation

- Entry point is fully public — no role, no signature, no stake required.
- The attacker needs only to submit one cheap Starknet transaction per block.
- No special knowledge or setup is required beyond knowing a valid `staker_address` (which is public on-chain).
- The attack is detectable but unstoppable without a contract upgrade, since there is no on-chain mechanism to override `last_reward_block` once it has been advanced.

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, consistent with the spec:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.roles.only_sequencer();   // enforce "Only starkware sequencer" per spec
    ...
```

Alternatively, remove the `disable_rewards` parameter from the public interface and handle reward-skipping logic internally, so external callers cannot influence whether rewards are distributed.

### Proof of Concept

1. Consensus rewards are active (post `consensus_rewards_first_epoch`).
2. Attacker monitors the chain. At block `N`, before the sequencer submits its `update_rewards` call, the attacker submits:
   ```
   staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
   ```
3. `last_reward_block` is set to `N`; no rewards are distributed.
4. The sequencer's `update_rewards(staker_address, disable_rewards=false)` for block `N` reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeating this at every block causes `unclaimed_rewards_own` for all stakers to remain permanently at zero. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
