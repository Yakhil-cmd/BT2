### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Freeze Consensus Yield — (`src/staking/staking.cairo`)

### Summary

The `update_rewards` function in `Staking.cairo` is documented as "Only starkware sequencer" but has **no caller restriction in the implementation**. Because `last_reward_block` is a global (not per-staker) storage variable, any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to consume the per-block reward slot without distributing any rewards, permanently blocking all stakers from receiving consensus-phase yield.

### Finding Description

The spec for `update_rewards` states:

> **Access control:** Only starkware sequencer.

However, the implementation contains no such check:

```cairo
// src/staking/staking.cairo ~line 1449
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks CONTRACT_IS_PAUSED
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... NO get_caller_address() == sequencer check ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;   // exits without distributing rewards
    }
    // reward distribution follows
``` [1](#0-0) 

The `last_reward_block` is a **single global storage slot** (not a per-staker mapping). The guard `current_block_number > self.last_reward_block.read()` means only **one** call to `update_rewards` can succeed per block, for any staker. Once it is consumed with `disable_rewards: true`, the sequencer's legitimate call for the same block reverts with `REWARDS_ALREADY_UPDATED`.

The spec error table for `update_rewards` lists only `CONTRACT_IS_PAUSED`, `REWARDS_ALREADY_UPDATED`, `STAKER_NOT_EXISTS`, and `INVALID_STAKER` — no sequencer-related error exists, confirming the check was never implemented. [2](#0-1) 

The tests themselves confirm the absence of access control: `update_rewards` is called directly from test addresses with no `cheat_caller_address_once` spoofing to a sequencer address. [3](#0-2) 

### Impact Explanation

**High — Permanent/temporary freezing of unclaimed yield.**

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` at every block. Each call:
1. Sets `last_reward_block = current_block_number`.
2. Returns immediately without distributing any rewards (due to `disable_rewards: true`).
3. Causes the sequencer's legitimate call for the same block to revert with `REWARDS_ALREADY_UPDATED`.

All stakers and pool members in the consensus-rewards phase receive zero yield for every block the attacker sustains the attack. The attacker has no profit motive — this is pure griefing — but the damage to the protocol and all stakers is severe and directly maps to the "Permanent/temporary freezing of unclaimed yield" impact category.

### Likelihood Explanation

The entry path is fully permissionless: `update_rewards` is a public function callable by any EOA or contract. The only cost is one Starknet transaction per block. A motivated attacker (e.g., a competing validator, a short-seller of STRK, or a protocol saboteur) can sustain this indefinitely at low cost. No privileged access, leaked keys, or external dependencies are required.

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, consistent with the spec:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
```

Alternatively, if the sequencer address is not stored on-chain, use Starknet's `get_sequencer_address()` syscall for the comparison.

### Proof of Concept

1. Deploy the protocol and advance to the consensus-rewards epoch.
2. From any unprivileged address, call:
   ```
   staking.update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)
   ```
   at every new block.
3. Observe that `last_reward_block` is updated each time, and the sequencer's call for the same block reverts with `REWARDS_ALREADY_UPDATED`.
4. After N blocks, all stakers have `unclaimed_rewards_own` unchanged — zero consensus rewards distributed.

The existing test `test_update_rewards_without_distribute` already demonstrates that calling with `disable_rewards: true` updates `last_reward_block` while leaving `staker_info` unchanged (no rewards credited), confirming the mechanism. [4](#0-3)

#[test]
fn test_update_rewards_without_distribute() {
    let mut cfg: StakingInitConfig = Default::default();
    general_contract_system_deployment(ref :cfg);
    let staking_contract = cfg.test_info.staking_contract;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    let staking_rewards_dispatcher = IStakingRewardsManagerDispatcher {
        contract_address: staking_contract,
    };
    let staking_config_dispatcher = IStakingConfigDispatcher { contract_address: staking_contract };
    stake_for_testing_using_dispatcher(:cfg);
    advance_k_epochs_global();
    let staker_address = cfg.test_info.staker_address;
    let staker_info_before = staking_dispatcher.staker_info_v1(:staker_address);
    // `disable_rewards = true`, and self.is_pre_consensus().
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: true);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);

    // `disable_rewards = false`, and self.is_pre_consensus().
    advance_block_number_global(blocks: 1);
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: false);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);
