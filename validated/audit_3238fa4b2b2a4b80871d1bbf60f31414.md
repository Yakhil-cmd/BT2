### Title
Unprivileged Caller Can Invoke `update_rewards` with `disable_rewards: true` to Permanently Suppress Block Reward Distribution — (File: src/staking/staking.cairo)

---

### Summary

Any unprivileged, non-zero address can call `update_rewards` with `disable_rewards: true` for any valid staker. Because the global `last_reward_block` is written **before** the `disable_rewards` guard, the block is permanently marked as processed with no rewards distributed, and every subsequent `update_rewards` call in that block reverts. Block rewards for the griefed block are irrecoverably lost.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` is a public entry point protected only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

The function accepts a caller-supplied `disable_rewards: bool` parameter. The critical ordering flaw is that `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

Once `last_reward_block` equals the current block number, the guard at the top of the function causes every subsequent call in that block to revert: [3](#0-2) 

Because `last_reward_block` is a **single global variable** (not per-staker), one call with `disable_rewards: true` for any valid staker poisons the entire block for all stakers. [4](#0-3) 

The analog to the reported bug is direct: in the original report an unvalidated input parameter (`authority_bump`) corrupts stored state so that future operations using the stored value fail. Here, the unvalidated `disable_rewards` parameter corrupts `last_reward_block` so that future reward-distribution calls for the same block fail.

---

### Impact Explanation

Block rewards for each griefed block are **permanently lost** — there is no mechanism to retroactively distribute rewards for a block whose `last_reward_block` has already been set. An attacker who calls `update_rewards(..., disable_rewards: true)` once per block, for any valid staker, can suppress all consensus block reward distribution indefinitely. This constitutes **permanent freezing of unclaimed yield** (High impact under the allowed scope).

---

### Likelihood Explanation

The attack requires no privileges, no capital, and no special knowledge beyond knowing a valid staker address (which is public on-chain). The cost is one cheap transaction per block. In Starknet's sequencer model the attacker cannot guarantee ordering against a specific victim transaction, but they do not need to: they can simply submit the griefing call at the start of every block independently, without racing anyone. Likelihood is **High**.

---

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a call that skips reward distribution does not consume the block's reward slot:

```cairo
// Update last block rewards ONLY when rewards are actually processed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... rest of reward distribution
```

Alternatively, restrict who may pass `disable_rewards: true` (e.g., only a privileged role or the staker themselves), or remove the parameter entirely and derive the skip condition from on-chain state.

---

### Proof of Concept

1. Staker `S` is active with a non-zero STRK balance in the consensus-rewards phase.
2. Attacker calls `update_rewards(S, disable_rewards: true)` in block `N`.
3. `last_reward_block` is written to `N`; the function returns early — no rewards distributed.
4. The legitimate sequencer/node call `update_rewards(S, disable_rewards: false)` in block `N` reverts with `REWARDS_ALREADY_UPDATED`.
5. Block `N` rewards for all stakers are permanently lost.
6. Attacker repeats step 2 every block; no staker ever receives consensus block rewards. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
