### Title
Gateway Admits Transactions with Zero/Below-Threshold L1 and L1-Data Gas Prices That Will Deterministically Fail at Blockifier Execution - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful validator enforces a minimum price threshold only for `l2_gas.max_price_per_unit`. It performs **no analogous check** for `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` against the previous block's L1/L1-data gas prices. The blockifier's `check_fee_bounds`, called during `perform_pre_validation_stage`, enforces all three prices against the current block's actual gas prices. The result is a structural gap: the gateway admits `AllResources` V3 transactions whose L1 or L1-data gas price is zero (or below the block price), which will deterministically fail at blockifier pre-validation with `InsufficientResourceBounds`, consuming mempool slots without any fee being charged.

---

### Finding Description

**Gateway stateful path — only L2 gas price is validated:**

`StatefulTransactionValidator::validate_resource_bounds` fetches the previous block's STRK L2 gas price and delegates to `validate_tx_l2_gas_price_within_threshold`. That function explicitly handles only the `AllResources` variant and only reads `tx_resource_bounds.l2_gas.max_price_per_unit`. The `L1Gas` legacy variant is explicitly skipped with the comment "No validation required for legacy transactions." The function carries a developer TODO: `// TODO(Arni): Consider running this validation for all gas prices.` [1](#0-0) 

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared to the previous block's L1 or L1-data gas prices anywhere in the gateway path.

**Stateless path — same gap:**

`StatelessTransactionValidator::validate_resource_bounds` checks `resource_bounds.l2_gas.max_price_per_unit` against `self.config.min_gas_price` (a static floor). No equivalent check exists for `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. [2](#0-1) 

**Blockifier path — all three prices are enforced:**

`AccountTransaction::check_fee_bounds`, called from `perform_pre_validation_stage` during blockifier validation, iterates over all three resources (L1Gas, L1DataGas, L2Gas) and rejects the transaction if `resource_bounds.max_price_per_unit < actual_gas_price.get()` for any resource. [3](#0-2) 

**The gap:** A V3 `AllResources` transaction with `l1_gas.max_price_per_unit = 0` (or any value below the current block's L1 gas price) and `l2_gas.max_price_per_unit ≥ min_gas_price_threshold` passes both the stateless and stateful gateway checks, is admitted to the mempool, and then deterministically fails at blockifier pre-validation with `TransactionFeeError::InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`. Because this is a pre-validation failure (not a revert), **no fee is charged**. [4](#0-3) 

---

### Impact Explanation

**Matching impact:** *High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.*

The gateway unconditionally admits transactions that will never execute successfully. Because blockifier pre-validation failures charge no fee, an attacker can repeatedly submit such transactions at zero cost (beyond the L2 gas price floor, which can be set to the minimum). Each admitted transaction occupies a mempool slot keyed on `(sender_address, nonce)`. With the `max_allowed_nonce_gap = 200` default, a single account can inject up to 200 such permanently-failing transactions into the mempool, displacing legitimate transactions and degrading sequencer throughput without paying any fee for the wasted work. [5](#0-4) 

---

### Likelihood Explanation

The trigger is entirely unprivileged: any account can craft a V3 `AllResources` invoke transaction with `l1_gas.max_price_per_unit = 0` and `l2_gas.max_price_per_unit` set to exactly the minimum threshold. The L1 gas price on Starknet mainnet is always non-zero, so the blockifier rejection is deterministic under normal operating conditions. The developer TODO comment confirms the gap is known but unaddressed. [6](#0-5) 

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or introduce a parallel function) to also compare `tx_resource_bounds.l1_gas.max_price_per_unit` and `tx_resource_bounds.l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK gas prices (`strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price`), applying the same `min_gas_price_percentage` multiplier. This mirrors the existing L2 gas price check and closes the admission gap before transactions reach the mempool. [7](#0-6) 

---

### Proof of Concept

1. Obtain any funded account on the network.
2. Construct a V3 `AllResources` invoke transaction with:
   - `l2_gas.max_price_per_unit` = `previous_block_l2_gas_price * min_gas_price_percentage / 100` (exactly at threshold, passes gateway)
   - `l1_gas.max_price_per_unit` = `0` (zero, never checked by gateway)
   - `l1_data_gas.max_price_per_unit` = `0` (zero, never checked by gateway)
   - `l2_gas.max_amount` = any non-zero value ≤ `max_l2_gas_amount`
   - `l1_gas.max_amount` = any value (e.g., `1`)
3. Submit via `POST /gateway/add_transaction`.
4. **Gateway result:** `Ok(tx_hash)` — transaction admitted to mempool. Both `StatelessTransactionValidator::validate_resource_bounds` and `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` pass because only `l2_gas.max_price_per_unit` is checked.
5. **Blockifier result:** `TransactionExecutionError::TransactionPreValidationError(TransactionFeeError::InsufficientResourceBounds { errors: [MaxGasPriceTooLow { resource: L1Gas, max_gas_price: 0, actual_gas_price: <block_l1_price> }] })` — transaction rejected, **no fee charged**.
6. Repeat with nonce+1 up to `max_allowed_nonce_gap` (200) times from the same account to fill the mempool with permanently-failing transactions at zero net cost. [8](#0-7) [9](#0-8)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())

#[derive(Clone, Debug, Serialize, Deserialize, Validate, PartialEq)]
pub struct StatefulTransactionValidatorConfig {
    // If true, ensures the max L2 gas price exceeds (a configurable percentage of) the base gas
    // price of the previous block.
    pub validate_resource_bounds: bool,
    pub max_allowed_nonce_gap: u32,
    pub reject_future_declare_txs: bool,
    pub max_nonce_for_validation_skip: Nonce,
    pub versioned_constants_overrides: Option<VersionedConstantsOverrides>,
    // Minimum gas price as percentage of threshold to accept transactions.
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}

impl Default for StatefulTransactionValidatorConfig {
    fn default() -> Self {
        StatefulTransactionValidatorConfig {
            validate_resource_bounds: true,
            max_allowed_nonce_gap: 200,
            reject_future_declare_txs: true,
            max_nonce_for_validation_skip: Nonce(Felt::ONE),
            min_gas_price_percentage: 100,
            versioned_constants_overrides: None,
        }
    }
}
