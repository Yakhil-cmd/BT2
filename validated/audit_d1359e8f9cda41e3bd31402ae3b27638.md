### Title
Missing Validation of `old_threshold` in `assert_reshare_keys_invariants` Allows Bypassing Intersection Check — (File: `src/dkg.rs`)

---

### Summary

`assert_reshare_keys_invariants` validates the **new** threshold through `assert_key_invariants` (which enforces `threshold >= 2`), but never validates the caller-supplied `old_threshold`. A malicious caller can pass `old_threshold = 0` or `old_threshold = 1` to trivially bypass the intersection-size guard, causing all honest participants to execute the full 5-round DKG/reshare protocol before the protocol aborts at the public-key equality check.

---

### Finding Description

In `src/dkg.rs`, `assert_reshare_keys_invariants` performs the following intersection check: [1](#0-0) 

```rust
if old_participants.intersection(&participants).len() < old_threshold {
    return Err(InitializationError::NotEnoughParticipantsForNewThreshold {
        threshold: old_threshold,
        participants: old_participants.intersection(&participants).len(),
    });
}
```

The purpose of this check is to ensure that at least `old_threshold` old participants are present in the new participant set — the minimum required to reconstruct the old secret via Lagrange interpolation.

However, `old_threshold` is never independently validated: [2](#0-1) 

```rust
let threshold = usize::from(threshold.into());
let old_threshold = usize::from(old_threshold.into());

let participants = assert_key_invariants(participants, me, threshold)?;
```

`assert_key_invariants` is called only for the **new** threshold. It enforces `threshold >= 2`: [3](#0-2) 

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
```

No equivalent check exists for `old_threshold`. Because `old_threshold` is a `usize`, passing `0` makes the guard `intersection.len() < 0` — which is always `false` in Rust (unsigned arithmetic), so the check **never fires**. Passing `1` requires only a single old participant in the new set, regardless of the actual security threshold of the old key.

The public API entry point `reshare` in `src/lib.rs` passes `old_threshold` directly to `assert_reshare_keys_invariants` without any prior sanitization: [4](#0-3) 

```rust
let (participants, old_participants) = assert_reshare_keys_invariants::<C>(
    new_participants,
    me,
    threshold,
    old_signing_key,
    old_threshold,
    old_participants,
)?;
```

---

### Impact Explanation

When `old_threshold` is set to `0` or `1`, the pre-check passes and all participants enter the full 5-round `do_keyshare` protocol. Because the intersection is smaller than the actual old threshold, the Lagrange interpolation in `do_reshare` produces an incorrect reconstructed secret: [5](#0-4) 

```rust
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(<C::Group as Group>::Field::zero);
```

The resulting public key will not match the old public key, and the protocol aborts at the equality check in `do_keyshare`: [6](#0-5) 

```rust
if let Some(old_vk) = old_verification_key {
    if old_vk != verifying_key {
        return Err(ProtocolError::AssertionFailed(
            "new public key does not match old public key".to_string(),
        ));
    }
}
```

The cryptographic output is therefore never accepted. However, **all honest participants have already executed all five rounds** of the protocol — including echo-broadcast rounds, polynomial commitment generation, proof-of-knowledge verification, and private share exchange — before the abort. This constitutes bounded but non-trivial wasted CPU, memory, and bandwidth for every honest participant.

**Impact class**: Griefing / resource exhaustion — matches the allowed Medium impact: *"Griefing or resource-exhaustion by a malicious caller or participant causing unbounded CPU, memory, bandwidth, or non-terminating work beyond documented behavior."*

---

### Likelihood Explanation

The `reshare` function is a public API entry point. Any unprivileged library caller who initiates a reshare controls the `old_threshold` argument. No authentication or out-of-band check prevents passing `0` or `1`. The attack requires no special privileges, no leaked keys, and no cryptographic breaks — only a single malformed API call.

---

### Recommendation

Add validation of `old_threshold` inside `assert_reshare_keys_invariants`, mirroring the check already applied to the new threshold in `assert_key_invariants`:

```rust
// In assert_reshare_keys_invariants, after converting old_threshold:
if old_threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: old_threshold,
        min: 2,
    });
}
```

This ensures that a caller cannot trivially bypass the intersection-size guard by supplying a semantically invalid `old_threshold`.

---

### Proof of Concept

```
Attacker calls:
  reshare(
      old_participants = [A, B, C],   // actual old threshold was 3
      old_threshold    = 0,           // ← attacker-controlled, not validated
      old_signing_key  = None,
      old_public_key   = <real key>,
      new_participants = [D, E, F],   // zero overlap with old set
      new_threshold    = 2,
      me               = D,
      rng              = ...,
  )

assert_reshare_keys_invariants:
  intersection = {A,B,C} ∩ {D,E,F} = {} → len = 0
  check: 0 < 0  →  false  →  NO ERROR (bypassed)

All of D, E, F enter the 5-round do_keyshare protocol.
Each contributes secret = Field::zero() (no old share).
Resulting public key = identity ≠ old_public_key.
Protocol aborts at do_keyshare line 491 — after all rounds complete.
```
