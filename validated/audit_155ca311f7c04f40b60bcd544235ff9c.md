### Title
Unprivileged Caller Can Force Premature Pool Contract Upgrade via `staker_migration` - (File: src/staking/staking.cairo)

### Summary
The `staker_migration` function in the `StakingMigrationImpl` is publicly callable with no access-control check. Any address can invoke it for any V2 staker, causing the staking contract (which holds upgrade-governor rights over pool contracts) to immediately upgrade the victim staker's delegation pool contracts to V3 and mark the staker as fully migrated.

### Finding Description
`staker_migration` is exposed as `#[abi(embed_v0)]` under `IStakingMigration` with no caller restriction: [1](#0-0) 

The function performs three privileged actions on behalf of the staking contract:
1. Calls `pool_replaceable_dispatcher.add_new_implementation` on every pool contract owned by the target staker.
2. Calls `pool_replaceable_dispatcher.replace_to` on those same pool contracts, executing the EIC initializer.
3. Writes `LATEST_STAKER_VERSION` into `staker_version` for the target staker.

The staking contract is the upgrade governor of pool contracts (granted during `upgrade_pool_implementation_v1`): [2](#0-1) 

Because the staking contract holds that role, the `add_new_implementation` / `replace_to` calls succeed unconditionally when initiated from within `staker_migration`, regardless of who triggered `staker_migration` externally.

The only guards present are: [3](#0-2) 

None of these checks the identity of `get_caller_address()`.

Compare with `update_rewards_from_attestation_contract`, which correctly gates its privileged action: [4](#0-3) 

And `update_rewards_from_staking_contract` in the pool, which also gates correctly: [5](#0-4) 

`staker_migration` has no equivalent guard.

### Impact Explanation
An attacker can force-migrate any eligible staker's pool contracts before the staker or the protocol operator intends. Consequences:

- The pool EIC initializer runs with empty `eic_init_data` at an arbitrary time chosen by the attacker, not by the staker or upgrade governor. If the EIC relies on staker-side state that has not yet been established (e.g., the staker has not yet set a public key or peer ID required by V3 logic), the pool contract can be left in an inconsistent state.
- Once `staker_version` is written to `LATEST_STAKER_VERSION`, the staker is permanently considered V3. Any subsequent call to `internal_staker_info` that expects V3 state will succeed even if the pool's internal state is corrupt, masking the inconsistency.
- Pool members whose funds are held in the prematurely-upgraded pool contract may be unable to exit or claim rewards if the pool's `cumulative_rewards_trace` or `pool_member_epoch_balance` is in an unexpected state post-EIC, constituting temporary freezing of delegator funds.

Impact: **High** — temporary freezing of pool member funds (delegator funds at rest).

### Likelihood Explanation
The entry point is fully public and requires no tokens, no role, and no prior relationship with the target staker. Any address on Starknet can call `staking_contract.staker_migration(victim_staker_address)` for any staker that has completed V2 migration but not yet V3. During the V2→V3 upgrade window, every staker in the system is eligible. Likelihood: **High**.

### Recommendation
Add a caller restriction to `staker_migration`. The caller should be either the staker themselves or a designated upgrade governor:

```cairo
fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
    // Only the staker or the upgrade governor may trigger migration.
    let caller = get_caller_address();
    assert!(
        caller == staker_address || self.roles.is_upgrade_governor(caller),
        "{}",
        Error::UNAUTHORIZED_MIGRATION_CALLER,
    );
    ...
}
```

Alternatively, restrict it to `only_upgrade_governor` if operator-driven batch migration is the intended model.

### Proof of Concept

1. Protocol upgrades staking contract to V3; stakers are now eligible for `staker_migration`.
2. Attacker observes `victim_staker` has `staker_own_balance_trace` non-empty and `staker_version != LATEST_STAKER_VERSION`.
3. Attacker calls:
   ```
   IStakingMigrationDispatcher { contract_address: staking_contract }
       .staker_migration(staker_address: victim_staker);
   ```
4. Staking contract (as upgrade governor) calls `add_new_implementation` + `replace_to` on victim's pool contracts, running the pool EIC with empty init data at attacker-chosen timing.
5. `staker_version[victim_staker]` is set to `LATEST_STAKER_VERSION`.
6. If the pool EIC leaves the pool in an inconsistent state, pool members calling `exit_delegation_pool_action` or `claim_rewards` will revert, freezing their funds until a further upgrade or manual remediation. [1](#0-0)

        fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
            // Assert the staker exists.
            self._internal_staker_info(:staker_address);

            // Assert staker already migrated to V2.
            assert!(
                !self.staker_own_balance_trace.entry(staker_address).is_empty(),
                "{}",
                Error::STAKER_NOT_MIGRATED,
            );

            // Assert the staker is not migrated to V3 yet.
            assert!(
                !self.staker_version.read(staker_address).is_latest(),
                "{}",
                Error::STAKER_ALREADY_MIGRATED,
            );

            // Prepare the implementation data.
            let pool_class_hash = self.pool_contract_class_hash.read();
            let pool_eic_class_hash = self.pool_eic_class_hash.read();
            // Sanity checks.
            assert!(pool_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            assert!(pool_eic_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            let eic_data = EICData { eic_hash: pool_eic_class_hash, eic_init_data: [].span() };
            let implementation_data = ImplementationData {
                impl_hash: pool_class_hash, eic_data: Option::Some(eic_data), final: false,
            };

            // Upgrade pools.
            let pools = self.staker_pool_info.entry(staker_address).pools;
            for pool_contract_ptr in pools.keys_iter() {
                let pool_contract = pool_contract_ptr.read();
                let pool_replaceable_dispatcher = IReplaceableDispatcher {
                    contract_address: pool_contract,
                };
                pool_replaceable_dispatcher.add_new_implementation(:implementation_data);
                pool_replaceable_dispatcher.replace_to(:implementation_data);
            }

            // Mark the staker's version.
            self.staker_version.write(staker_address, LATEST_STAKER_VERSION);
        }
