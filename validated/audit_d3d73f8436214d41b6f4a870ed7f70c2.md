### Title
`last_reward_block` Updated Unconditionally Before `disable_rewards` Check Enables Permanent Reward Griefing — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function writes the global `last_reward_block` storage variable **before** evaluating the `disable_rewards` flag. Because the function has no role-based access control, any unprivileged caller can invoke it with `disable_rewards: true` once per block, advancing `last_reward_block` without distributing any rewards and blocking all subsequent legitimate calls for that block.

---

### Finding Description

In `update_rewards` the ordering is:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← written unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing
}
``` [1](#0-0) 

The function enforces a strict single-call-per-block invariant at its top:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

`last_reward_block` is a **global** (not per-staker) `BlockNumber` in storage:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [3](#0-2) 

A single call per block with `disable_rewards: true` therefore blocks every subsequent `update_rewards` call for that block, for every staker.

The only gate on the function is `general_prerequisites()`:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [4](#0-3) 

No role check exists. Any non-zero address may call `update_rewards` with arbitrary arguments.
