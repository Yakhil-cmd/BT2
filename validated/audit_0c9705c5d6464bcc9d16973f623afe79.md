### Title
`MsgTurnBridge` Handler Missing Permission Check and Implementation Allows Unprivileged Bypass of Bridge Circuit Breaker - (File: `x/cronos/keeper/msg_server.go`)

---

### Summary

The `TurnBridge` gRPC handler in `x/cronos/keeper/msg_server.go` is a stub that unconditionally returns `nil, nil`. It performs no authorization check and modifies no state. Any unprivileged user can submit `MsgTurnBridge` and receive a success response, bypassing the `CanTurnBridge` permission that was explicitly designed to gate this operation. The bridge circuit breaker is permanently non-functional.

---

### Finding Description

The Cronos permission system defines a `CanTurnBridge` bit (value `2`) alongside `CanChangeTokenMapping` (value `1`). The `UpdateTokenMapping` handler correctly enforces its permission gate:

```go
// x/cronos/keeper/msg_server.go:73-74
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
```

The `TurnBridge` handler, however, is a no-op stub:

```go
// x/cronos/keeper/msg_server.go:84-87
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

This means:
1. **No permission check** — any address can call `MsgTurnBridge` and receive a success (`nil` error / code 0), bypassing the `CanTurnBridge` permission entirely.
2. **No state change** — the bridge is never actually disabled or enabled regardless of who calls it or what `Enable` value is passed.

The `CanTurnBridge` permission constant is defined, the `MsgTurnBridge` message is fully registered in the gRPC service descriptor, and `ValidateBasic` accepts any valid bech32 sender — the message is fully reachable by any unprivileged user. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**High — Bypass of Cronos admin/permission authorization checks.**

The `CanTurnBridge` permission bit exists precisely to restrict bridge circuit-breaker control to authorized addresses (per ADR-009). The handler never enforces it. Any unprivileged user can call `MsgTurnBridge` and receive a success response, which constitutes a complete bypass of the permission authorization check for this message.

Additionally, the bridge circuit breaker is permanently non-functional: no authorized party can ever disable the Gravity bridge through this message path, since the handler performs no state mutation. This creates a long-lived inability for honest operators to halt bridge/conversion flows in an emergency. [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

**High.** `MsgTurnBridge` is a fully registered, publicly callable gRPC message. `ValidateBasic` only checks that the sender is a valid bech32 address. Any user on the network can submit this transaction and receive a success response with no special access required. [6](#0-5) 

---

### Recommendation

Implement `TurnBridge` with:
1. A permission check using `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)`, mirroring the pattern in `UpdateTokenMapping`.
2. Actual bridge state modification (e.g., updating a `BridgeEnabled` parameter or equivalent state).

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // ... apply bridge enable/disable state ...
    return &types.MsgTurnBridgeResponse{}, nil
}
``` [2](#0-1) 

---

### Proof of Concept

1. Any unprivileged address `attacker` (no `CanTurnBridge` permission) submits:
   ```
   MsgTurnBridge{ Sender: attacker, Enable: false }
   ```
2. The handler at `msg_server.go:85-87` returns `nil, nil` — the transaction succeeds with code 0.
3. No permission check is performed; `CanTurnBridge` is never consulted.
4. The bridge state is unchanged regardless of the `Enable` field value.
5. Legitimate operators with `CanTurnBridge` permission also cannot disable the bridge, since the handler is a stub. [1](#0-0) [7](#0-6)

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
