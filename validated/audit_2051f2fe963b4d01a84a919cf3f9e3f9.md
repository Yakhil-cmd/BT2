### Title
Raw Internal Storage Error Strings Leaked Verbatim to Unprivileged REST API Callers via Unfiltered gRPC Status Message Forwarding - (File: `engine/access/rest/common/http_request_handler.go`)

---

### Summary

The Access Node REST API unconditionally forwards the full text of internal gRPC status error messages to any unauthenticated HTTP caller. Because the error-conversion helpers (`ConvertStorageError`, `ConvertError`, `ConvertIndexError`, `mapReadError`) embed raw Go error strings — including storage-backend error details, internal file paths, and component identifiers — into those gRPC status messages using `%v`, an unprivileged caller can read internal node implementation details by triggering any error path.

---

### Finding Description

Two cooperating layers produce the leak.

**Layer 1 — error conversion embeds raw internal strings into gRPC status messages.**

`ConvertStorageError` in `engine/common/rpc/errors.go` formats the raw error directly into the gRPC status payload:

```go
// line 67
return status.Errorf(codes.NotFound, "not found: %v", err)
// line 70
return status.Errorf(codes.Internal, "failed to find: %v", err)
```

`ConvertError` does the same for any unrecognised error:

```go
// line 51
return status.Errorf(returnCode, "%s%v", msg, err)
```

`ConvertIndexError` follows the same pattern:

```go
// line 82
return status.Errorf(codes.FailedPrecondition, "data for block is not available: %v", err)
// line 90
return status.Errorf(codes.NotFound, "data not found: %v", err)
```

`mapReadError` in `access/backends/extended/backend.go` does the same for every storage sentinel it handles:

```go
// lines 116-122
return status.Errorf(codes.FailedPrecondition, "%s index not initialized: %v", label, err)
return status.Errorf(codes.OutOfRange, "requested height not indexed: %v", err)
return status.Errorf(codes.InvalidArgument, "invalid query: %v", err)
return status.Errorf(codes.NotFound, "not found: %v", err)
```

**Layer 2 — `ErrorHandler` forwards every gRPC status message verbatim to the HTTP client.**

The comment at line 82 of `engine/access/rest/common/http_request_handler.go` is explicit:

```go
// handle grpc status error returned from the backend calls, we are forwarding the message to the client
if se, ok := status.FromError(err); ok {
    switch se.Code() {
    case codes.NotFound:
        msg := fmt.Sprintf("Flow resource not found: %s", se.Message())   // line 86
    case codes.InvalidArgument:
        msg := fmt.Sprintf("Invalid Flow argument: %s", se.Message())     // line 90
    case codes.Internal:
        msg := fmt.Sprintf("Invalid Flow request: %s", se.Message())      // line 94
    case codes.Unavailable:
        msg := fmt.Sprintf("Failed to process request: %s", se.Message()) // line 98
    case codes.FailedPrecondition:
        msg := fmt.Sprintf("Precondition failed: %s", se.Message())       // line 103
    case codes.OutOfRange:
        msg := fmt.Sprintf("Out of range: %s", se.Message())              // line 107
    }
}
```

Every gRPC code that the backend can produce is covered; none are sanitised before being written into the JSON response body.

A concrete call site that combines both layers is `GetLatestBlock` in `engine/access/rpc/backend/backend_block_details.go`:

```go
// line 55
return nil, flow.BlockStatusUnknown, status.Errorf(codes.Internal, "could not get latest block: %v", err)
```

The raw storage error (e.g., a BadgerDB or Pebble error containing a file-system path) is placed in the gRPC status message and then forwarded by `ErrorHandler` as `"Invalid Flow request: could not get latest block: <raw storage error>"` in the HTTP response body.

The existing test suite confirms this behaviour explicitly:

```go
// engine/access/rest/http/routes/collections_test.go lines 132-134
status.Errorf(codes.Internal, "some internal error"),
`{"code":500,"message":"Invalid Flow request: some internal error"}`,
```

---

### Impact Explanation

An unprivileged REST API caller who triggers any storage error path receives the raw internal error string in the JSON response body. Depending on the underlying storage backend, this string can contain:

- Absolute file-system paths to the node's data directory (e.g., BadgerDB/Pebble paths such as `/home/flow/data/protocol/...`)
- Database-specific error messages that reveal the storage engine version and configuration
- Internal component names, storage key formats, and block/height identifiers

This is the direct Flow analog of the reported vulnerability: the original server leaked its directory structure, OS, and package names through JSON-parse error messages; the Access Node leaks its storage-layer internals through gRPC-to-HTTP error forwarding.

**Impact: 3**

---

### Likelihood Explanation

No authentication or special privilege is required. Any caller with network access to the Access Node REST API can trigger these error paths by:

- Requesting a block at a height that is not yet indexed (`GET /v1/blocks/{height}`)
- Requesting a resource by an ID that does not exist (`GET /v1/blocks/{id}`, `GET /v1/collections/{id}`, etc.)
- Submitting a script against an unindexed height

These are ordinary API operations. The error paths are exercised in normal operation whenever a caller queries data that is absent or not yet available.

**Likelihood: 3**

---

### Recommendation

1. **Sanitise at the forwarding boundary.** In `ErrorHandler` (`engine/access/rest/common/http_request_handler.go`), replace `se.Message()` with a fixed, human-readable string for each code, and log the full message server-side only. For example:

   ```go
   case codes.Internal:
       errorLogger.Error().Str("grpc_message", se.Message()).Msg("internal backend error")
       h.errorResponse(w, http.StatusInternalServerError, "internal server error", errorLogger)
   ```

2. **Strip raw errors from gRPC status messages.** In `ConvertStorageError`, `ConvertError`, `ConvertIndexError`, and `mapReadError`, avoid embedding `%v` of the raw error in the status message. Use a fixed sentinel string in the status message and attach the raw error only to the server-side log.

3. **Audit all `status.Errorf` call sites** that use `%v` or `%s` with a raw error value and that are reachable from the REST or gRPC API surface.

func ConvertError(err error, msg string, defaultCode codes.Code) error {
	if err == nil {
		return nil
	}

	// Handle multierrors separately
	if multiErr, ok := err.(*multierror.Error); ok {
		return ConvertMultiError(multiErr, msg, defaultCode)
	}

	// Already converted
	if status.Code(err) != codes.Unknown {
		return err
	}

	if msg != "" {
		msg += ": "
	}

	var returnCode codes.Code
	switch {
	case errors.Is(err, context.Canceled):
		returnCode = codes.Canceled
	case errors.Is(err, context.DeadlineExceeded):
		returnCode = codes.DeadlineExceeded
	default:
		returnCode = defaultCode
	}

	return status.Errorf(returnCode, "%s%v", msg, err)

func ConvertStorageError(err error) error {
	if err == nil {
		return nil
	}

	// Already converted
	if status.Code(err) == codes.NotFound {
		return err
	}

	if errors.Is(err, storage.ErrNotFound) {
		return status.Errorf(codes.NotFound, "not found: %v", err)
	}

	return status.Errorf(codes.Internal, "failed to find: %v", err)
}

func ConvertIndexError(err error, height uint64, defaultMsg string) error {
	if err == nil {
		return nil
	}

	if errors.Is(err, indexer.ErrIndexNotInitialized) {
		return status.Errorf(codes.FailedPrecondition, "data for block is not available: %v", err)
	}

	if errors.Is(err, storage.ErrHeightNotIndexed) {
		return status.Errorf(codes.OutOfRange, "data for block height %d is not available", height)
	}

	if errors.Is(err, storage.ErrNotFound) {
		return status.Errorf(codes.NotFound, "data not found: %v", err)
	}

	return ConvertError(err, defaultMsg, codes.Internal)

// mapReadError converts storage read errors to appropriate gRPC status errors.
func mapReadError(ctx context.Context, label string, err error) error {
	switch {
	case errors.Is(err, storage.ErrNotBootstrapped):
		return status.Errorf(codes.FailedPrecondition, "%s index not initialized: %v", label, err)
	case errors.Is(err, storage.ErrHeightNotIndexed):
		return status.Errorf(codes.OutOfRange, "requested height not indexed: %v", err)
	case errors.Is(err, storage.ErrInvalidQuery):
		return status.Errorf(codes.InvalidArgument, "invalid query: %v", err)
	case errors.Is(err, storage.ErrNotFound):
		return status.Errorf(codes.NotFound, "not found: %v", err)
	default:
		irrecoverable.Throw(ctx, fmt.Errorf("failed to get %s: %w", label, err))
		return err
	}
}
