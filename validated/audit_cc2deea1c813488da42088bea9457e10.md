### Title
Gas Fee Collector Set to Internal `CoinbaseAddress` Permanently Locks FLOW Tokens in EVM - (`File: fvm/evm/handler/handler.go`, `fvm/evm/stdlib/contract.cdc`)

---

### Summary

`EVM.run` is `access(all)` and accepts any `coinbase: EVMAddress` without validation. When `coinbase` is set to the internal `types.CoinbaseAddress` (the EVM-side gas fee accumulation address), `runWithGasFeeRefund` performs a self-transfer that is a no-op, leaving all gas fees permanently locked in `CoinbaseAddress`. Because `CoinbaseAddress` has no associated Cadence COA resource, those FLOW tokens can never be withdrawn back to Cadence — they are irreversibly lost from the cross-VM accounting.

---

### Finding Description

`EVM.run` in `fvm/evm/stdlib/contract.cdc` is declared `access(all)` and passes the caller-supplied `coinbase` bytes directly to `InternalEVM.run` with no restriction:

```cadence
access(all)
fun run(tx: [UInt8], coinbase: EVMAddress): Result {
    pre { !self.isPaused(): "EVM operations are temporarily paused" }
    return InternalEVM.run(tx: tx, coinbase: coinbase.bytes) as! Result
}
```

This reaches `ContractHandler.Run` → `runWithGasFeeRefund` in `fvm/evm/handler/handler.go`:

```go
func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
    cb := h.AccountByAddress(types.CoinbaseAddress, true)
    initCoinbaseBalance := cb.Balance()
    f()
    afterBalance := cb.Balance()
    diff := new(big.Int).Sub(afterBalance, initCoinbaseBalance)
    if diff.Sign() > 0 {
        cb.Transfer(gasFeeCollector, diff)   // ← no check: gasFeeCollector != CoinbaseAddress
    }
    ...
}
```

During EVM execution, gas fees accumulate in `types.CoinbaseAddress`. After execution, `runWithGasFeeRefund` is supposed to forward those fees to the caller-supplied `gasFeeCollector`. If `gasFeeCollector == types.CoinbaseAddress`, the call `cb.Transfer(CoinbaseAddress, diff)` is an EVM self-transfer — it succeeds without error but leaves the balance of `CoinbaseAddress` unchanged. The gas fees never leave `CoinbaseAddress`.

`CoinbaseAddress` is a protocol-internal EVM address. There is no Cadence COA resource bound to it. The only Cadence-side withdrawal path is through `InternalEVM.withdraw(from: self.addressBytes, ...)` called on a COA resource. Since no COA exists for `CoinbaseAddress`, the FLOW tokens backing those gas fees are permanently inaccessible — they cannot be withdrawn to Cadence.

The same root cause applies to `EVM.batchRun`, which also calls `runWithGasFeeRefund` with the caller-supplied `gasFeeCollector`.

---

### Impact Explanation

Every gas fee paid by EVM transaction senders whose transactions are relayed via `EVM.run(tx, coinbase: CoinbaseAddress)` is permanently locked inside the EVM state. The Cadence-side FlowToken vault that backs EVM balances retains the corresponding FLOW (it was deposited when the sender funded their EVM account), but those tokens can never be reclaimed. Over time, repeated exploitation accumulates an unbounded amount of permanently frozen FLOW. This is a direct cross-VM asset loss: real FLOW tokens become inaccessible to all parties.

---

### Likelihood Explanation

`EVM.run` is `access(all)` — any unprivileged Cadence transaction can call it. The attacker only needs to submit a valid RLP-encoded EVM transaction (signed by any EOA) and pass `CoinbaseAddress` as the coinbase bytes. No special role, staked node, or admin key is required. The attack is cheap (only the gas cost of the EVM transaction itself) and repeatable.

---

### Recommendation

Add a guard in `runWithGasFeeRefund` (and/or at the Cadence `EVM.run` boundary) that rejects or redirects the call when `gasFeeCollector` equals any protocol-reserved address (`CoinbaseAddress`, `NativeTokenBridgeAddress`, etc.):

```go
func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
    if gasFeeCollector == types.CoinbaseAddress {
        panic(fvmErrors.NewEVMError(fmt.Errorf("gasFeeCollector must not be the coinbase address")))
    }
    // ... existing logic
}
```

Equivalently, add a `pre` condition in `EVM.run` / `EVM.batchRun` in `contract.cdc` that rejects reserved addresses.

func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
	// capture coinbase init balance
	cb := h.AccountByAddress(types.CoinbaseAddress, true)
	initCoinbaseBalance := cb.Balance()
	f()
	// transfer the gas fees collected to the gas fee collector address
	afterBalance := cb.Balance()
	diff := new(big.Int).Sub(afterBalance, initCoinbaseBalance)
	if diff.Sign() > 0 {
		cb.Transfer(gasFeeCollector, diff)
	}
	if diff.Sign() < 0 { // this should never happen but in case
		panic(fvmErrors.NewEVMError(fmt.Errorf("negative balance change on coinbase")))
	}
}
