### Title
Gateway Admits Transactions with Zero L1/L1-Data Gas Price Due to Incomplete Resource Bounds Validation — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful resource-bounds check (`validate_tx_l2_gas_price_within_threshold`) enforces a minimum price floor only for `l2_gas.max_price_per_unit`. It does not apply the same check to `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The stateless validator has the same gap. As a result, a transaction with zero L1 and L1-data gas prices passes all gateway admission checks and enters the mempool, but is guaranteed to fail at blockifier pre-validation when the batcher attempts to include it in a block.

---

### Finding Description

**Stateless path** — `StatelessTransactionValidator::validate_resource_bounds`:

```rust
// Line 66-76 (stateless_transaction_validator.rs)
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(...ZeroResourceBounds...);
}
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(...MaxGasPriceTooLow...);
}
```

Only `l2_gas.max_price_per_unit` is compared against `config.min_gas_price`. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` receive no floor check. A transaction with `l2_gas.max_amount = 1`, `l2_gas.max_price_per_unit = min_gas_price`, and both L1 prices set to `0` satisfies `max_possible_fee > 0` and passes the price check. [1](#0-0) 

**Stateful path** — `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold`:

```rust
// Line 358-390 (stateful_transaction_validator.rs)
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(...) {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // threshold derived from previous block L2 gas price only
            if tx_l2_gas_price.0 < threshold { return Err(...); }
        }
        ValidResourceBounds::L1Gas(_) => { /* No validation */ }
    }
}
```

The function explicitly carries a developer TODO acknowledging the gap. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared against the previous block's L1 or L1-data gas prices. [2](#0-1) 

**Blockifier enforcement (too late)** — `AccountTransaction::check_fee_bounds`:

The blockifier's `perform_pre_validation_stage` does check all three gas prices against the block's actual prices:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [3](#0-2) 

But this check runs inside the batcher's execution pipeline, **after** the transaction has already been admitted to the mempool. The gateway never rejects the transaction.

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

A transaction with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` passes every gateway check and occupies a mempool slot. When the batcher pulls it and calls `perform_pre_validation_stage`, the blockifier rejects it with `MaxGasPriceTooLow` for L1Gas and L1DataGas. The transaction is discarded without being included in any block, having consumed mempool capacity and batcher CPU for no purpose.

An attacker can craft an unbounded stream of such transactions, each passing gateway admission, to pollute the mempool and displace legitimate transactions, degrading sequencer throughput.

---

### Likelihood Explanation

**High.** The attacker-controlled fields (`l1_gas.max_price_per_unit`, `l1_data_gas.max_price_per_unit`) are free-form integers in the signed transaction body. Setting them to `0` requires no privilege, no special account state, and no knowledge beyond the current L2 gas price (needed to pass the one check that does exist). The gap is confirmed by the developer TODO comment in the production code.

---

### Recommendation

1. In `validate_tx_l2_gas_price_within_threshold`, extend the check to cover `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK gas prices, applying the same `min_gas_price_percentage` threshold.

2. In `StatelessTransactionValidator::validate_resource_bounds`, add floor checks for `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against `config.min_gas_price` (or a separate configurable floor), mirroring the existing L2 check. [4](#0-3) 

---

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with a valid account signature and:
   ```
   resource_bounds.l2_gas.max_price_per_unit  = <current_l2_gas_price>   // passes stateful check
   resource_bounds.l2_gas.max_amount          = 1                         // max_possible_fee > 0
   resource_bounds.l1_gas.max_price_per_unit  = 0                         // no gateway check
   resource_bounds.l1_data_gas.max_price_per_unit = 0                     // no gateway check
   ```

2. Submit to the gateway HTTP endpoint. Trace through:
   - `StatelessTransactionValidator::validate_resource_bounds`: `max_possible_fee = 1 * current_l2_gas_price > 0` ✓; `l2_gas.max_price_per_unit >= min_gas_price` ✓. **Passes.**
   - `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold`: `l2_gas.max_price_per_unit >= threshold` ✓. **Passes.**
   - Transaction is inserted into the mempool.

3. When the batcher calls `AccountTransaction::perform_pre_validation_stage` → `check_fee_bounds`:
   - `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (> 0)` → `MaxGasPriceTooLow` for L1Gas.
   - `l1_data_gas.max_price_per_unit (0) < actual_l1_data_gas_price (> 0)` → `MaxGasPriceTooLow` for L1DataGas.
   - Transaction is rejected from the block.

4. Repeat at high frequency to fill the mempool with permanently-invalid transactions. [5](#0-4) [6](#0-5)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
