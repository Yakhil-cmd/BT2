### Title
Bank Precompile `transfer` Executes Unauthorized Token Transfers — Missing Caller-Authorization Check on `sender` Field - (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The bank precompile's `transfer` method accepts the `sender` address directly from ABI-decoded calldata without verifying that the calling contract is authorized to move tokens on behalf of that address. Any unprivileged EVM contract can call the precompile and specify an arbitrary victim as `sender`, draining that victim's `evm/<callerContract>` bank-module balance to any destination.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go`, the `Run` function handles `TransferMethodName` as follows:

```go
sender    := args[0].(common.Address)   // fully attacker-controlled
recipient := args[1].(common.Address)
amount    := args[2].(*big.Int)
...
from  := sdk.AccAddress(sender.Bytes())
to    := sdk.AccAddress(recipient.Bytes())
denom := EVMDenom(contract.Caller())    // evm/<callerContract>
amt   := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

`sender` (the `from` address passed to `SendCoins`) is taken verbatim from the ABI input. There is no check that `contract.Caller() == sender`, no allowance/approval lookup, and no other authorization gate. The only field tied to the caller is the token denom (`evm/<callerContract>`), which scopes the token namespace but does not restrict who can be drained within it.

The same pattern applies to `BurnMethodName`: `addr` (the account to burn from) is also taken from ABI input without verifying the caller is authorized to burn on behalf of that address. [1](#0-0) 

### Impact Explanation
**Critical — Unauthorized transfer of precompile-controlled assets.**

`evm/<address>` tokens are precompile-controlled bank-module assets. Any EVM contract that has previously minted `evm/<itself>` tokens to users (e.g., as part of a DeFi protocol, vesting schedule, or reward distribution) can at any time call `transfer(victim, attacker, balance)` on the bank precompile and move those tokens out of the victim's account without consent. The victim has no on-chain mechanism to prevent or revoke this.

This falls squarely within the allowed Critical impact: *"Unauthorized … transfer … for … precompile-controlled assets."*

### Likelihood Explanation
The bank precompile is callable by any deployed EVM contract with no privilege requirement. An attacker needs only to:
1. Deploy a contract.
2. Use the precompile's `mint` to distribute `evm/<contract>` tokens to users (establishing a balance to steal).
3. Later call `transfer(victim, attacker, amount)` — no signature, no approval, no admin key required.

### Recommendation
Add a caller-authorization check before executing the transfer:

```go
if contract.Caller() != sender {
    return nil, errors.New("transfer: caller is not the token sender")
}
```

Or, implement an ERC20-style allowance mapping keyed by `(owner, spender)` within the precompile so that users can explicitly approve a contract to move tokens on their behalf.

The `burn` path should receive the same fix: verify `contract.Caller() == addr` before burning from an account.

### Proof of Concept

1. Attacker deploys contract `M` on Cronos EVM.
2. `M` calls bank precompile `mint(victim, 1_000_000)` → victim now holds `1_000_000 evm/M` tokens.
3. `M` calls bank precompile `transfer(victim, attacker, 1_000_000)`.
4. The precompile executes `bankKeeper.SendCoins(ctx, victim, attacker, [{denom:"evm/M", amount:1_000_000}])` with no authorization check.
5. Victim's entire `evm/M` balance is transferred to attacker. Victim had no opportunity to prevent this.

The root cause — `sender` is accepted from calldata without verifying `contract.Caller() == sender` — is structurally identical to the external report's pattern: a field that participates in a security-critical operation (`SendCoins` source address) is not bound to the authenticated party (`contract.Caller()`), allowing it to be set arbitrarily by the caller. func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
