### Title
Delegation Below `min_delegation_for_rewards` Permanently Freezes Pool Rewards in Pool Contract - (File: src/pool/pool.cairo)

### Summary

`enter_delegation_pool` in `pool.cairo` enforces only a non-zero amount check. When the total pool balance falls below `min_delegation_for_rewards` (1 STRK = `10^18` wei for STRK pools), the staking contract still transfers STRK reward tokens to the pool contract, but the pool's `compute_rewards_per_unit` silently returns zero, leaving those tokens permanently unclaimable by any pool member.

### Finding Description

**Root cause — no minimum delegation floor in `enter_delegation_pool`:** [1](#0-0) 

The only guard is `amount.is_non_zero()`. There is no check that `amount >= min_delegation_for_rewards`.

**The threshold that triggers reward loss:** [2](#0-1) 

`min_for_rewards` is `10^18` (1 STRK). This value is written to storage during pool construction: [3](#0-2) 

**Rewards are transferred to the pool contract unconditionally:**

In `_update_rewards`, the staking contract calls `update_pool_rewards`, which first transfers STRK to the pool contract and then notifies it: [4](#0-3) 

The transfer happens at line 1876–1881 *before* the pool has any chance to reject it.

**The pool silently discards the reward accounting when balance is too small:** [5](#0-4) 

When `total_stake < min_delegation_for_rewards`, the function returns `Zero::zero()`. The cumulative rewards trace is then updated with `last + 0`, i.e., no change: [6](#0-5) 

The STRK tokens are now inside the pool contract but the cumulative index was never incremented, so `calculate_rewards` will always compute zero for any pool member.

**The code itself documents this as a known data loss path:** [7](#0-6) 

### Impact Explanation

Any STRK rewards forwarded to a pool whose total balance is below 1 STRK are permanently frozen inside the pool contract. There is no recovery function; the only token-exit paths (`claim_rewards`, `exit_delegation_pool_action`, `transfer_to_pools_when_unstake`) either rely on the cumulative trace (which was never updated) or return only the delegated principal. The delegator's unclaimed yield is permanently frozen.

**Impact class:** High — Permanent freezing of unclaimed yield.

### Likelihood Explanation

Any unprivileged delegator can call `enter_delegation_pool` with any amount from 1 wei up to `10^18 - 1` wei (just under 1 STRK). This is a realistic amount for a retail user unfamiliar with the threshold. Once the pool's total balance is below 1 STRK, every subsequent reward distribution silently discards the pool's share. The threshold is not surfaced in any user-facing error or event.

### Recommendation

Add a minimum delegation check in `enter_delegation_pool` (and `add_to_delegation_pool`) that ensures the resulting pool balance meets or exceeds `min_delegation_for_rewards`:

```cairo
assert!(
    amount >= self.min_delegation_for_rewards.read(),
    "{}",
    Error::AMOUNT_LESS_THAN_MIN_DELEGATION
);
```

Alternatively, if sub-threshold delegations must be allowed, the staking contract should not forward rewards to a pool whose balance is below the threshold, rather than forwarding tokens that will never be claimable.

### Proof of Concept

1. Staker registers with `min_stake` (e.g. 20,000 STRK) and opens a STRK delegation pool.
2. Delegator calls `enter_delegation_pool(reward_address, amount: 5 * 10^17)` — 0.5 STRK, which passes the `is_non_zero` check.
3. Pool total balance = 0.5 STRK < `min_delegation_for_rewards` (1 STRK).
4. Staker attests; `update_rewards_from_attestation_contract` is called.
5. `calculate_staker_pools_rewards` computes `pool_rewards ≈ epoch_rewards × 0.5 / 20000.5` — a non-zero STRK amount.
6. `update_pool_rewards` transfers that STRK amount to the pool contract via `send_rewards_to_delegation_pool`.
7. `pool.update_rewards_from_staking_contract` is called; `compute_rewards_per_unit` returns 0 because `5×10^17 < 10^18`.
8. Cumulative rewards trace is unchanged; the transferred STRK is now permanently locked in the pool contract.
9. Delegator calls `claim_rewards` → receives 0. Principal can be withdrawn via `exit_delegation_pool_intent/action`, but the yield is gone forever.

```text
        /// Compute the rewards for the pool trace.
        ///
        /// `staking_rewards` is in `STRK_DECIMALS` decimals.
        /// `total_stake` is in the contract's token decimals.
        /// **Note**: Delegation rewards lost when pool balance is less than
        /// `min_delegation_for_rewards`. The staking contract continues to forward
        /// `pool_rewards` to the pool contract even in this case.
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
```
