### Title
Permissionless `update_rewards` with `disable_rewards: true` Allows Any Caller to Block Reward Distribution for All Stakers - (File: src/staking/staking.cairo)

### Summary
`update_rewards` in `Staking` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. When called with `disable_rewards: true`, it advances the global `last_reward_block` checkpoint without distributing any rewards. Because `last_reward_block` is a single global value and the function enforces `current_block_number > last_reward_block`, any unprivileged caller can front-run every legitimate reward-distribution call in a block, permanently suppressing yield for all stakers.

### Finding Description
`update_rewards` is exposed in the `IStakingRewardsManager` public interface with no access-control guard beyond `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function immediately writes the current block number to the global `last_reward_block` storage slot: [2](#0-1) 

The guard at line 1454–1458 ensures only one call per block succeeds: [3](#0-2) 

`last_reward_block` is a single, protocol-wide storage variable: [4](#0-3) 

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` first in a block:
1. Passes the `current_block_number > last_reward_block` check.
2. Writes `current_block_number` into `last_reward_block`.
3. Hits the early-return branch at line 1487 — no rewards are distributed.

Every subsequent legitimate call to `update_rewards` for any staker in the same block reverts with `REWARDS_ALREADY_UPDATED`. Repeating this every block indefinitely freezes all consensus-era staker and pool rewards.

### Impact Explanation
The `last_reward_block` slot is global. A single griefing call per block prevents every staker from accumulating `unclaimed_rewards_own` and prevents every delegation pool from receiving its `update_rewards_from_staking_contract` callback. Sustained execution permanently freezes unclaimed yield for all stakers and pool members — matching **High: Permanent freezing of unclaimed yield or unclaimed royalties; Temporary freezing of funds**.

### Likelihood Explanation
The entry point is fully permissionless; any EOA or contract can call it. The attacker needs only to submit one transaction per block with a valid (existing) `staker_address` and `disable_rewards: true`. No capital, no privileged role, and no external dependency is required. On Starknet L2, transaction fees are low, making sustained griefing economically feasible. Likelihood is **Medium-High**.

### Recommendation
Restrict `update_rewards` to a trusted caller — either the attestation contract, the consensus layer contract, or a dedicated operator role. Add an access-control assertion analogous to `assert_caller_is_attestation_contract` used in `update_rewards_from_attestation_contract`: [5](#0-4) 

Alternatively, remove the `disable_rewards` parameter from the public interface entirely and derive the flag internally from protocol state (e.g., whether the staker attested in the current block).

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has been set and the current epoch has passed it).
2. At the start of block `N`, attacker calls:
   ```
   staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
   ```
3. `general_prerequisites()` passes (contract unpaused, caller non-zero).
4. `current_block_number (N) > last_reward_block` — passes.
5. `last_reward_block` is written to `N`.
6. `disable_rewards == true` → early return; no rewards distributed.
7. Any legitimate node operator calling `update_rewards(their_staker, false)` in block `N` hits `REWARDS_ALREADY_UPDATED` and reverts.
8. Attacker repeats step 2 every block. All stakers' `unclaimed_rewards_own` and all pool `cumulative_rewards_trace` entries stop accumulating indefinitely. [6](#0-5)

```text
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
```
