### Title
Hardcoded Test Private Key Unconditionally Used in Production `create_signature_manager()` Enables Precommit Vote Signature Forgery — (`crates/apollo_signature_manager/src/lib.rs`)

### Summary
`create_signature_manager()` is the sole production factory for the signature manager component. It unconditionally constructs a `LocalKeyStoreSignatureManager` backed by `LocalKeyStore::new_for_testing()`, which embeds the publicly known private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133`. Any party with access to the source code can derive the corresponding public key and forge signatures that pass `verify_precommit_vote_signature`.

### Finding Description
`LocalKeyStore::new_for_testing()` is `pub(crate)`, so it cannot be called from outside the crate directly. However, `LocalKeyStoreSignatureManager::new()` — a **public** type — calls it unconditionally, and `create_signature_manager()` — also **public** — wraps that. [5](#0-4) 

`create_signature_manager()` is imported and called in the production node startup path inside `create_node_components()` whenever `signature_manager.execution_mode` is `LocalExecutionWithRemoteDisabled` or `LocalExecutionWithRemoteEnabled`. [3](#0-2) 

`verify_precommit_vote_signature` takes a caller-supplied `public_key` argument; it does not look up the key from any trusted registry. [6](#0-5) 

Because the production node signs every precommit vote with the hardcoded key, the corresponding public key is also fixed and publicly derivable. A verifier that trusts this public key (e.g., a consensus peer that learned the sequencer's public key from the network) will accept forged signatures.

### Impact Explanation
An attacker who knows the private key can:
1. Construct a `RawSignature` over an arbitrary `block_hash` using `ecdsa_sign` with the hardcoded key.
2. Present that signature alongside the known public key to any caller of `verify_precommit_vote_signature`.
3. Receive `Ok(true)`, making the forged precommit vote indistinguishable from a legitimate one.

If a consensus peer or RPC layer trusts precommit votes verified this way to determine the canonical finalized block, it will accept attacker-chosen `block_hash` values, causing it to serve wrong finalized block state, storage, and class hashes — matching the "High: RPC execution / pending view returns an authoritative-looking wrong value" impact category.

### Likelihood Explanation
The private key is embedded in plain text in the public source repository. No configuration, flag, or feature gate prevents `create_signature_manager()` from using it. Every production node built from this code uses the same key.

### Recommendation
- Remove `LocalKeyStore::new_for_testing()` from any non-`#[cfg(test)]` code path.
- Implement a production `KeyStore` that loads the private key from a secure external source (HSM, encrypted config, environment secret).
- Gate `LocalKeyStore` and `LocalKeyStoreSignatureManager` entirely behind `#[cfg(test)]` or a dedicated `testing` feature flag so they cannot be compiled into release binaries.
- Add a compile-time or startup assertion that the active `KeyStore` is not the test stub.

### Proof of Concept
```rust
#[tokio::test]
async fn forge_precommit_vote_with_hardcoded_key() {
    use starknet_api::block::BlockHash;
    use starknet_api::felt;
    use crate::signature_manager::{
        verify_precommit_vote_signature, LocalKeyStore, SignatureManager,
    };

    // The hardcoded key is publicly known; this is what production uses.
    let key_store = LocalKeyStore::new_for_testing();
    let sm = SignatureManager::new(key_store);

    // Attacker chooses an arbitrary block hash.
    let forged_block_hash = BlockHash(felt!("0xdeadbeef"));
    let forged_sig = sm.sign_precommit_vote(forged_block_hash).await.unwrap();

    // verify_precommit_vote_signature accepts the attacker's public key —
    // which is fixed and derivable from the source — and returns Ok(true).
    let result = verify_precommit_vote_signature(
        forged_block_hash,
        forged_sig,
        key_store.public_key,
    );
    assert_eq!(result.unwrap(), true); // passes: forgery succeeds
}
```

The test passes because `create_signature_manager()` — the only production constructor — unconditionally calls `LocalKeyStore::new_for_testing()`, making the private key a public constant. [1](#0-0)
