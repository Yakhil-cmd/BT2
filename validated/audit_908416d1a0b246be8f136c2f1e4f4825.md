### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`setPoolBinAdditionalFees` imposes no cap on per-bin additional fees, while `setPoolAdminFees` enforces `maxAdminSpreadFeeE6`. A pool admin can set `addFeeBuyE6` / `addFeeSellE6` up to `type(uint16).max` (65 535 = 6.5535 % in E6 scale) on any bin, regardless of what the factory owner has configured as the admin fee ceiling. This breaks the admin-boundary invariant: the factory owner's cap is bypassed through a separate, unchecked code path.

### Finding Description

`MetricOmmPoolFactory.setPoolAdminFees` enforces the factory owner's cap:

```solidity
// MetricOmmPoolFactory.sol L414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees`, however, passes values straight through to the pool with **no cap check**:

```solidity
// MetricOmmPoolFactory.sol L450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` also performs no cap check — only a bin-index range check: [3](#0-2) 

These per-bin fees are **directly additive** to the base spread fee in every swap path:

```solidity
// MetricOmmPool.sol L910 (and L999, L1177 for other directions)
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [4](#0-3) 

### Impact Explanation

If the factory owner sets `maxAdminSpreadFeeE6 = 0` (or any low value) to prevent admin fee extraction, the pool admin can still call `setPoolBinAdditionalFees(pool, bin, 65535, 65535)` and impose a 6.5535 % additional spread on every swap through that bin. Because the bin additional fee is additive to `baseFeeX64` in the swap math with no ceiling, traders receive less output than the factory-owner-enforced cap permits. The excess fee accrues to the pool (and ultimately to the admin via `collectPoolFees`), constituting a direct loss of trader principal above the protocol-sanctioned limit. [5](#0-4) 

### Likelihood Explanation

The pool admin is a semi-trusted role. The factory owner's cap mechanism exists precisely because the admin is not fully trusted. The bypass requires a single transaction from the pool admin — no special setup, no multi-step coordination, and no external dependency. Any pool with a financially motivated admin and a factory owner who has set a meaningful `maxAdminSpreadFeeE6` cap is at risk.

### Recommendation

Add a cap check inside `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce the cap against the **combined** base + bin additional fee so the total admin-controlled spread never exceeds `maxAdminSpreadFeeE6`.

### Proof of Concept

1. Factory owner deploys factory and calls `setFeeCaps(..., maxAdminSpreadFeeE6 = 1000, ...)` — capping admin spread at 0.1 %.
2. Pool creator calls `createPool` with `adminSpreadFeeE6 = 0` (respects the cap).
3. Pool admin calls `setPoolAdminFees(pool, 1000, 0)` — accepted, total admin spread = 0.1 %.
4. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` — **no revert**; bin 0 now carries an additional 6.5535 % fee on both buy and sell.
5. A trader swaps through bin 0 and pays 6.5535 % + 0.1 % = 6.6535 % spread — 65× the factory-owner-sanctioned maximum.
6. The excess fee is retained in the pool and collected by the admin via `collectPoolFees`. [6](#0-5) [2](#0-1)
