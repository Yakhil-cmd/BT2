### Title
Staker's Principal Temporarily Frozen When `RewardSupplier` Lacks Token Balance During `unstake_action` - (File: src/staking/staking.cairo)

### Summary
After a staker calls `unstake_intent`, their state is irrevocably changed and there is no cancel path. When they subsequently call `unstake_action`, the function unconditionally attempts to claim all accumulated rewards from the `RewardSupplier` before returning the principal. If the `RewardSupplier`'s actual STRK token balance is less than the accumulated rewards (a realistic window while L1 bridge delivery is pending), the `checked_transfer` inside `RewardSupplier.claim_rewards` panics, reverting the entire `unstake_action`. The staker's principal remains locked in the staking contract with no recovery path until the reward supplier is sufficiently funded.

### Finding Description

The `unstake_action` function in `src/staking/staking.cairo` bundles the reward payout with the principal return in a single atomic call:

```cairo
fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    ...
    let token_dispatcher = strk_token_dispatcher();
    self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher); // <-- must succeed
    self.write_staker_info(:staker_address, :staker_info);
    let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
    ...
    token_dispatcher.checked_transfer(recipient: staker_address, amount: staker_amount.into());
    ...
}
```

`send_rewards_to_staker` calls `claim_from_reward_supplier`, which calls `RewardSupplier.claim_rewards`:

```cairo
fn claim_rewards(ref self: ContractState, amount: Amount) {
    let unclaimed_rewards = self.unclaimed_rewards.read();
    assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);
    self.unclaimed_rewards.write(unclaimed_rewards - amount);
    let token_dispatcher = self.token_dispatcher.read();
    token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into()); // panics if balance < amount
}
```

The `RewardSupplier` maintains an accounting variable `unclaimed_rewards` that is incremented whenever rewards are calculated, and separately requests STRK from L1 via `request_funds`. The `request_funds` logic counts `l1_pending_requested_amount` (tokens requested but not yet bridged) as part of its "credit":

```cairo
let credit = balance + l1_pending_requested_amount;
let debit = unclaimed_rewards;
if credit < debit + threshold {
    // request more from L1
}
```

This means the actual on-chain STRK balance of the `RewardSupplier` can be less than `unclaimed_rewards` while the system considers itself solvent (because it counts in-flight L1 bridge amounts). During this window, `checked_transfer` in `claim_rewards` will panic because the actual balance is insufficient.

The state-transition lock is in `unstake_intent`:

```cairo
fn unstake_intent(ref self: ContractState) -> Timestamp {
    ...
    assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
    let unstake_time = Time::now().add(delta: self.exit_wait_window.read());
    staker_info.unstake_time = Option::Some(unstake_time);
    self.write_staker_info(:staker_address, :staker_info);
    ...
}
```

Once `unstake_intent` is called, `unstake_time` is set. There is no `cancel_unstake_intent` function anywhere in the codebase. The staker cannot re-stake (blocked by `assert!(staker_info.unstake_time.is_none(), ...)` in `increase_stake`), cannot call `unstake_intent` again (same guard), and cannot call `unstake_action` successfully until the reward supplier has enough tokens. The principal is frozen.

### Impact Explanation

A staker's principal (STRK tokens held in the staking contract) is temporarily frozen. The staker has already committed to exiting via `unstake_intent` and waited through the full `exit_wait_window`, but cannot retrieve their principal because `unstake_action` atomically requires a successful reward payout. The staker is stuck until the L1 bridge delivers the pending STRK to the `RewardSupplier`. This matches the **Temporary freezing of funds** impact category.

### Likelihood Explanation

Medium. The `RewardSupplier` is designed to proactively request L1 funds, but there is always a non-zero delay between the L1 message being sent and the bridge delivering tokens on L2. During high-activity periods (many attestations, many stakers accumulating rewards), the gap between `unclaimed_rewards` and actual balance can be significant. Any staker who calls `unstake_action` during this window will have their principal frozen. This is not a contrived edge case — it is an inherent property of the asynchronous L1→L2 bridge architecture.

### Recommendation

Decouple the reward claim from the principal return in `unstake_action`. Allow `unstake_action` to succeed and return the principal even if the reward claim fails or is deferred. One approach is to leave `unclaimed_rewards_own` intact if the reward supplier cannot pay at the time of `unstake_action`, and allow the staker (or their reward address) to claim rewards separately after the fact. Alternatively, add a `cancel_unstake_intent` function so stakers can revert to an active state if `unstake_action` is blocked.

### Proof of Concept

1. Staker accumulates a large `unclaimed_rewards_own` over many epochs via `update_rewards`.
2. Each reward update calls `update_unclaimed_rewards_from_staking_contract` on the `RewardSupplier`, incrementing `unclaimed_rewards`. The `request_funds` call sends L1 messages but the bridge has not yet delivered.
3. At this point: `RewardSupplier.unclaimed_rewards > RewardSupplier.token_balance`.
4. Staker calls `unstake_intent` → `unstake_time` is set, stake removed from totals. No cancel path exists.
5. Staker waits for `exit_wait_window` to expire.
6. Staker calls `unstake_action`. Execution reaches `send_rewards_to_staker` → `claim_from_reward_supplier` → `RewardSupplier.claim_rewards` → `token_dispatcher.checked_transfer(recipient: staking_contract, amount: rewards)` → **panics** because `token_balance < rewards`.
7. The entire `unstake_action` reverts. The staker's principal remains in the staking contract.
8. The staker cannot call `unstake_intent` again (guard: `unstake_time.is_none()`), cannot `increase_stake` (same guard), and cannot call `unstake_action` successfully until the L1 bridge delivers tokens.

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn claim_rewards(ref self: ContractState, amount: Amount) {
            // Asserts.
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            let unclaimed_rewards = self.unclaimed_rewards.read();
            assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Update unclaimed_rewards and transfer the requested rewards to the staking contract.
            self.unclaimed_rewards.write(unclaimed_rewards - amount);
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
        }

        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }
