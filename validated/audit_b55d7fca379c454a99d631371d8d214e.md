### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `staking.cairo` is documented as restricted to "Only starkware sequencer" but contains no `get_caller_address()` check. Any unprivileged caller can invoke it with `disable_rewards: true`, consuming the single per-block `last_reward_block` slot and preventing the legitimate sequencer from distributing rewards for that block. Repeated every block, this permanently freezes all staker and pool unclaimed yield.

### Finding Description

`IStakingRewardsManager::update_rewards` is the consensus-era reward distribution entry point. The spec explicitly states its access control is "Only starkware sequencer": [1](#0-0) 

The implementation, however, performs no caller identity check: [2](#0-1) 

The function writes `last_reward_block` to the current block number unconditionally before checking `disable_rewards`: [3](#0-2) 

`last_reward_block` is a single global storage slot shared across all stakers: [4](#0-3) 

Once written for block N, the guard `current_block_number > self.last_reward_block.read()` causes every subsequent call in block N to revert with `REWARDS_ALREADY_UPDATED`, including the legitimate sequencer call.

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block:

1. Marks `last_reward_block = current_block` without distributing any rewards.
2. Blocks the sequencer from calling `update_rewards` again in that block.
3. No STRK rewards are minted or credited to `unclaimed_rewards_own` for any staker or pool.
4. Repeated indefinitely, this permanently freezes all unclaimed yield for every staker and every delegation pool.

This matches the allowed impact: **Permanent freezing of unclaimed yield** (High).

### Likelihood Explanation

- The function is a standard external entry point with no role guard.
- Any address holding enough gas can call it.
- The cost per block is a single transaction; the attacker needs no stake, no special role, and no secret.
- The attack is fully automated and sustainable indefinitely.

### Recommendation

Add a sequencer-address check analogous to the pattern used for the attestation contract:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Add: assert caller is the registered sequencer address
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

Alternatively, store the authorized sequencer address at construction time and expose a governance-controlled setter, mirroring how `attestation_contract` is stored and checked: [5](#0-4) 

### Proof of Concept

```
// Attacker script (runs once per block, e.g. via a bot):
// 1. Pick any valid staker_address (readable from on-chain events).
// 2. Call update_rewards(staker_address, disable_rewards: true).
//    - last_reward_block is set to current block.
//    - No rewards are distributed.
// 3. Sequencer attempts update_rewards(...) → reverts REWARDS_ALREADY_UPDATED.
// 4. Repeat next block.
//
// Result: zero rewards ever distributed after consensus rewards epoch begins.
```

The attacker entry path is entirely unprivileged: read a `staker_address` from the `NewStaker` event, call `update_rewards` with `disable_rewards: true` before the sequencer in every block. The root cause is solely the missing `get_caller_address()` guard in `src/staking/staking.cairo` at the `StakingRewardsManagerImpl::update_rewards` function. [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
