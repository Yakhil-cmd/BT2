After exhaustive analysis of the ZKsync OS codebase, I searched for analogs to the VotingEscrow inconsistent-rounding / checkpoint-alignment vulnerability class across all relevant subsystems.

**Vulnerability class mapped**: Inconsistent rounding/alignment in a period-based accounting or checkpoint system where one code path omits a required boundary normalization, causing the system's invariant to be permanently violated and producing incorrect cumulative state.
