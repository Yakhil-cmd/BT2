### Title
Gateway Stateful Validator Checks Only `l2_gas` Price Threshold, Silently Admitting Transactions With Insufficient `l1_gas`/`l1_data_gas` Prices - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check (`validate_tx_l2_gas_price_within_threshold`) validates only the `l2_gas.max_price_per_unit` field against the previous block's L2 gas price threshold. The `l1_gas` and `l1_data_gas` price fields are never checked at the gateway admission layer. A transaction carrying `l1_gas.max_price_per_unit = 0` (or any value below the current block's L1 gas price) passes every gateway check and is forwarded to the mempool. When the batcher later calls `check_fee_bounds` inside `perform_pre_validation_stage`, the transaction is rejected as a pre-validation error and never included in a block, but the mempool slot was already consumed.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` calls `validate_tx_l2_gas_price_within_threshold`, which contains an explicit `match` on `ValidResourceBounds`:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(...) {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // ... threshold check on l2_gas only ...
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
}
``` [1](#0-0) 

The `l1_gas` and `l1_data_gas` price fields inside `AllResources` are never inspected. The TODO comment on line 358 explicitly acknowledges the gap.

The stateless validator's `validate_resource_bounds` similarly only enforces a floor on `l2_gas.max_price_per_unit`:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(...MaxGasPriceTooLow...);
}
``` [2](#0-1) 

No analogous check exists for `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` at either the stateless or stateful gateway layer.

The blockifier's `check_fee_bounds` (called inside `perform_pre_validation_stage`) does check all three resources against actual block gas prices:

```rust
ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_gas_resource_bounds,
    l2_gas: l2_gas_resource_bounds,
    l1_data_gas: l1_data_gas_resource_bounds,
}) => { /* checks all three */ }
``` [3](#0-2) 

However, this check is gated on `self.execution_flags.charge_fee`:

```rust
if self.execution_flags.charge_fee {
    self.check_fee_bounds(tx_context)?;
    verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
}
``` [4](#0-3) 

The gateway sets `charge_fee = enforce_fee(executable_tx, only_query)` in `run_validate_entry_point`. When `charge_fee` evaluates to `false` (bootstrap transactions, zero-fee paths), `check_fee_bounds` is skipped entirely, and the only gas-price gate that ran was the incomplete `validate_tx_l2_gas_price_within_threshold`. Such a transaction is forwarded to the mempool with no `l1_gas`/`l1_data_gas` price validation having occurred. [5](#0-4) 

### Impact Explanation

An attacker can craft a V3 `AllResources` transaction with:
- `l2_gas.max_price_per_unit` ≥ threshold (passes `validate_tx_l2_gas_price_within_threshold`)
- `l1_gas.max_price_per_unit = 0` and/or `l1_data_gas.max_price_per_unit = 0` (no gateway check)

This transaction passes all gateway admission checks and is inserted into the mempool. When the batcher attempts to execute it, `check_fee_bounds` rejects it as a pre-validation error (not a revert), so it is never included in a block. The mempool slot is wasted and the sequencer expends CPU on a transaction that was structurally invalid at admission time. At scale this is a targeted mempool-flooding / DoS vector. The impact matches **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing**.

### Likelihood Explanation

The attack requires only a valid `l2_gas` price and a zero (or below-threshold) `l1_gas`/`l1_data_gas` price. No privileged access, no special account state, and no cryptographic material beyond a normal transaction signature is needed. The TODO comment on line 358 confirms the developers are aware the check is partial.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to enforce a minimum threshold on `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding gas prices, mirroring the existing `l2_gas` check. The TODO comment at line 358 already flags this exact work item.

### Proof of Concept

1. Obtain the previous block's `strk_gas_prices.l2_gas_price` (e.g., 100 fri).
2. Construct an `RpcInvokeTransactionV3` with:
   - `l2_gas.max_price_per_unit = 100` (meets threshold)
   - `l2_gas.max_amount = 1`
   - `l1_gas.max_price_per_unit = 0`
   - `l1_data_gas.max_price_per_unit = 0`
3. Submit via the gateway's `add_tx` endpoint.
4. Observe: `StatelessTransactionValidator::validate` passes (total fee > 0 from `l2_gas`; `l2_gas` price ≥ `min_gas_price`). [6](#0-5) 
5. Observe: `StatefulTransactionValidator::validate_resource_bounds` passes (only `l2_gas` price is checked). [7](#0-6) 
6. When `charge_fee = false` (bootstrap path), `run_validate_entry_point` skips `check_fee_bounds` entirely, and the transaction is forwarded to the mempool with `l1_gas.max_price_per_unit = 0` unchallenged. [8](#0-7) 
7. The batcher later rejects the transaction at `perform_pre_validation_stage` → `check_fee_bounds` with `MaxGasPriceTooLow { resource: L1Gas }`, but the mempool slot has already been consumed. [9](#0-8)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
