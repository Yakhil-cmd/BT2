### Title
Missing Minimum Threshold Bound Check in `assert_sign_inputs` Allows Invalid Threshold to Initiate FROST Signing - (File: src/frost/mod.rs)

### Summary

`assert_sign_inputs` in `src/frost/mod.rs` validates only the upper bound of the `threshold` parameter but omits the lower bound check (`threshold >= 2`) that is consistently enforced in the analogous DKG validation function `assert_key_invariants`. Any caller can pass `threshold = 1` (or `0`) to FROST EdDSA and RedJubjub signing entry points, bypassing the intended security parameter gate and causing the signing protocol to proceed with a cryptographically invalid threshold, producing unusable signature outputs.

### Finding Description

`assert_sign_inputs` is the shared validation gate for all FROST-based signing protocols. It checks:

1. `participants.len() >= 2`
2. No duplicate participants
3. `me` is in the participant list
4. `threshold <= participants.len()` (upper bound only)
5. `coordinator` is in the participant list [1](#0-0) 

The lower bound check — `threshold < 2` → `ThresholdTooSmall` — is entirely absent. Compare this to `assert_key_invariants`, which correctly enforces both bounds: [2](#0-1) 

The same omission exists in the `presign` function for FROST: [3](#0-2) 

`assert_sign_inputs` is called directly by all three FROST signing entry points:

- `sign_v1` and `sign_v2` in EdDSA: [4](#0-3) 

- `sign` in RedJubjub: [5](#0-4) 

`ReconstructionLowerBound` is a plain `usize` newtype with no enforced minimum: [6](#0-5) 

So `threshold = 1` (or `0`) is a valid value at the type level and passes all checks in `assert_sign_inputs`.

### Impact Explanation

When `threshold = 1` is accepted, the FROST signing protocol proceeds with a threshold that is inconsistent with the key generation threshold (which enforces `>= 2`). In FROST, the threshold governs Lagrange interpolation during signature aggregation. With `threshold = 1`, the coordinator aggregates using only a single participant's partial signature share (Lagrange coefficient = 1 for a single-point interpolation), which does not reconstruct the correct group signing scalar. The resulting signature fails to verify against the group public key — an unusable cryptographic output. Honest participants who ran the protocol correctly receive a corrupted, non-verifiable signature.

This maps to: **High — Corruption of sign outputs so honest parties accept unusable cryptographic outputs.**

### Likelihood Explanation

Any unprivileged library caller invoking `sign_v1`, `sign_v2`, or `redjubjub::sign` controls the `threshold` argument directly. No special privilege, key material, or network access is required. The caller simply passes `threshold = 1` (a valid `usize` that converts to `ReconstructionLowerBound` without restriction). The missing check is a single-line omission that is trivially reachable on every invocation of these public API functions.

### Recommendation

Add the same lower bound check present in `assert_key_invariants` to `assert_sign_inputs`:

```rust
// After the upper bound check:
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Apply the same fix to the `presign` function in `src/frost/mod.rs` at line 72.

### Proof of Concept

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)

pub fn assert_key_invariants(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
) -> Result<ParticipantList, InitializationError> {
    let threshold = usize::from(threshold.into());
    // need enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // Step 1.1
    // validate threshold
    if threshold > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold,
            max: participants.len(),
        });
    }
    // Step 1.1
    if threshold < 2 {
        return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
    }

    // ensure uniqueness of participants in the participant list
    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }
    Ok(participants)
}
