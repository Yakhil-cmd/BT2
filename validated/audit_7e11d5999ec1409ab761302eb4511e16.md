### Title
Missing Zero-Address Validation for `reward_address` in `stake()` Allows Permanent Loss of Staker Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `stake()` function in `src/staking/staking.cairo` accepts a `reward_address` parameter without validating that it is non-zero. If a staker accidentally supplies the zero address as their `reward_address`, all accumulated staking rewards will be permanently transferred to address(0) upon any `claim_rewards()` call, resulting in irreversible loss of unclaimed yield.

---

### Finding Description

In `src/staking/staking.cairo`, the `stake()` function performs several address checks on `reward_address`: [1](#0-0) 

It verifies that `reward_address` is not a registered token address, and that `amount >= min_stake`. However, there is **no check that `reward_address` is non-zero**. The zero address (`0x0`) is not a token address, so `does_token_exist(token_address: reward_address)` returns `false` and the check passes silently. [2](#0-1) 

The `reward_address` is then stored in the staker's record: [3](#0-2) 

When `claim_rewards()` is later called, it reads `staker_info.reward_address` and calls `send_rewards_to_staker`, which transfers the accumulated rewards to that address: [4](#0-3) 

On Starknet, address(0) is a valid transfer target — the ERC20 transfer will succeed and the tokens will be permanently unrecoverable.

The same issue exists in `enter_delegation_pool()` in `src/pool/pool.cairo`, which also accepts `reward_address` without a zero-address check: [5](#0-4) 

---

### Impact Explanation

A staker who accidentally passes `reward_address = 0` will have all their earned staking rewards permanently sent to address(0) upon the first `claim_rewards()` call. The rewards are irretrievably burned. This constitutes **permanent freezing / loss of unclaimed yield**, matching the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

While `change_reward_address` exists as a recovery path, it only helps if the staker notices the mistake *before* calling `claim_rewards()`. Any rewards claimed before the correction are permanently lost.

---

### Likelihood Explanation

Passing a zero address by accident is a realistic user error, especially for users interacting directly with the contract (e.g., via a script or CLI) rather than through a frontend that might validate inputs. The `stake()` function is a primary entry point callable by any unprivileged user. No special role or privilege is required to trigger this path.

---

### Recommendation

Add an explicit zero-address check for `reward_address` in both `stake()` and `enter_delegation_pool()`:

```cairo
assert!(reward_address.is_non_zero(), "{}", GenericError::REWARD_ADDRESS_IS_ZERO);
```

This should be placed alongside the existing address validation checks in both functions. A corresponding check should also be added to `change_reward_address()` in `src/staking/staking.cairo` for consistency.

---

### Proof of Concept

1. Attacker/user calls `stake(reward_address: 0, operational_address: <valid>, amount: <min_stake>)` on the `Staking` contract.
2. The call succeeds — `does_token_exist(0)` returns `false`, and `0 >= min_stake` passes if `min_stake` is non-zero (the amount check is on `amount`, not `reward_address`).
3. The staker accrues rewards over epochs via the normal attestation/consensus reward mechanism.
4. The staker (or anyone) calls `claim_rewards(staker_address)`.
5. `send_rewards_to_staker` transfers the accumulated `unclaimed_rewards_own` to address(0).
6. The tokens are permanently lost — no recovery is possible. [6](#0-5)

        fn claim_rewards(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let caller_address = get_caller_address();
            let reward_address = staker_info.reward_address;
            assert!(
                caller_address == staker_address || caller_address == reward_address,
                "{}",
                Error::CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            // Transfer rewards to staker's reward address and write updated staker info to storage.
            // Note: `send_rewards_to_staker` alters `staker_info` thus commit to storage is
            // performed only after that.
            let amount = staker_info.unclaimed_rewards_own;
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            self.write_staker_info(:staker_address, :staker_info);
            amount
        }
