### Title
Unprivileged Caller Can Permanently Suppress Per-Block Consensus Rewards via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `staking.cairo` writes the global `last_reward_block` storage slot **before** evaluating the `disable_rewards` flag. Because the function has no access control, any unprivileged caller can invoke it with `disable_rewards: true` to consume the single per-block reward slot without distributing any rewards. The legitimate block proposer's subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`, burning their transaction fee, while the block's rewards are permanently lost.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

Inside the function, the global `last_reward_block` is written at line 1485, **before** the `disable_rewards` branch at line 1487: [2](#0-1) 

Because `last_reward_block` is a single global value (not per-staker), writing it unconditionally means:

1. Attacker calls `update_rewards(any_active_staker, disable_rewards: true)` — passes all validation, writes `last_reward_block = current_block`, returns early with **zero rewards distributed**.
2. Legitimate block proposer calls `update_rewards(their_staker, disable_rewards: false)` in the same block — fails immediately at the `REWARDS_ALREADY_UPDATED` assert, paying a transaction fee for nothing.
3. The block's rewards are **permanently lost**: they are computed per-block and never accumulated for a future block. [3](#0-2) 

The attacker only needs a valid, active staker address with non-zero STRK balance — any publicly observable staker in the network qualifies, including the attacker's own. [4](#0-3) 

---

### Impact Explanation

- **Permanent freezing / theft of unclaimed yield**: Block rewards for each attacked block are calculated per-block and never re-accumulated. Repeating the attack every block permanently denies all stakers their consensus-phase rewards.
- **Griefing**: Honest block proposers pay Starknet transaction fees for reverted calls with no recourse.

Matches allowed impact: *Permanent freezing of unclaimed yield* (High) and *Griefing with damage to users or protocol* (Medium).

---

### Likelihood Explanation

- No privileged role required — any EOA or contract can call `update_rewards`.
- The attacker only needs to submit one cheap transaction per block before the legitimate proposer.
- Active staker addresses are publicly observable on-chain.
- The attack is profitable for a competing staker who benefits from rivals receiving no rewards.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the early-return guards, so that a call with `disable_rewards: true` does not consume the per-block slot:

```cairo
// BEFORE (vulnerable):
self.last_reward_block.write(current_block_number);   // line 1485
if disable_rewards || self.is_pre_consensus() {        // line 1487
    return;
}

// AFTER (fixed):
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
```

Alternatively, add an access-control check (e.g., only the designated block proposer or a whitelisted caller) before the function body.

---

### Proof of Concept

1. Attacker identifies any valid, active staker address `staker_A` (e.g., from on-chain events).
2. In block N (consensus-rewards phase), attacker submits `update_rewards(staker_A, disable_rewards: true)` with a priority fee to land first.
3. Function passes all checks, writes `last_reward_block = N`, returns early — **no rewards distributed**.
4. Legitimate block proposer submits `update_rewards(staker_B, disable_rewards: false)` in block N.
5. Assert at line 1455 fires: `current_block_number > self.last_reward_block.read()` → false → revert with `REWARDS_ALREADY_UPDATED`.
6. Block proposer's fee is consumed; stakers receive zero rewards for block N.
7. Attacker repeats every block for a sustained denial of all consensus rewards. [5](#0-4)
