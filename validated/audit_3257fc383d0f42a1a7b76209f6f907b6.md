### Title
Unbounded Iteration Over Ever-Growing `stakers` Vec in `get_stakers` Causes Permanent Denial of Service - (File: src/staking/staking.cairo)

### Summary
The `stakers` storage vector in the Staking contract grows monotonically — stakers are pushed on every `stake()` call but **never removed**, even after unstaking. The `get_stakers` function iterates over the full vector on every call using `into_iter_full_range()`. An unprivileged actor can deliberately bloat this vector by cycling through many staker addresses, eventually making `get_stakers` exceed Starknet's gas ceiling and permanently breaking the consensus interface.

### Finding Description
`src/staking/staking.cairo` declares the staker registry as a `Vec<ContractAddress>`: [1](#0-0) 

The comment is explicit: stakers are **never removed** from this vector. Every successful `stake()` call appends to it unconditionally: [2](#0-1) 

`get_stakers`, the public consensus interface, iterates over the **entire** vector on every invocation: [3](#0-2) 

There is no pagination, no cap, and no pruning. The loop body performs multiple storage reads per staker (`is_staker_active`, `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, `get_peer_id_at_epoch`), making each iteration expensive.

### Impact Explanation
Once the `stakers` vector is large enough, every call to `get_stakers` will exceed Starknet's per-transaction gas limit. Because `get_stakers` is the sole source of the active-staker list for the consensus layer, a permanently broken `get_stakers` means:

- The consensus layer can no longer retrieve the staker set for any epoch.
- Attestation-based and block-based reward flows that depend on this list are disrupted.
- Unclaimed yield for all stakers and delegators is permanently frozen.

This matches the **High** allowed impact: *Permanent freezing of unclaimed yield*, and the **Medium** allowed impact: *Unbounded gas consumption*.

### Likelihood Explanation
The minimum stake is a fixed protocol parameter. An attacker with sufficient capital can register thousands of staker addresses (each with the minimum stake), immediately call `unstake_intent` + `unstake_action` to recover principal, and repeat with fresh addresses. Each cycle permanently adds one entry to `stakers`. The cost is only the gas for each round-trip; principal is fully recovered. The attack is permissionless, requires no privileged access, and the damage is irreversible because the vector is never pruned.

### Recommendation
1. **Remove exited stakers from the vector** by swapping the unstaked address with the last element and popping, or by maintaining a separate active-staker count and skipping inactive entries without iterating them.
2. **Alternatively**, replace the `Vec` with an `IterableMap` that supports deletion, mirroring the existing `btc_tokens: IterableMap` pattern already used in the same contract.
3. **Short-term mitigation**: add a hard cap on the number of registered stakers, or paginate `get_stakers` so callers can retrieve results in bounded chunks.

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
