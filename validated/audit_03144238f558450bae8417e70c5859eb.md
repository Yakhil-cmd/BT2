### Title
`get_storage_at` Pending View Returns `CONTRACT_NOT_FOUND` Instead of `Felt::ZERO` for Contracts Deployed Only in Pending State — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

`get_storage_at` with `block_id=Tag(Pending)` performs a contract-existence check against the **committed state only** when the storage value is zero. It does not consult the pending `deployed_contracts` list. A contract deployed exclusively in the pending block with a zero (or absent) storage value at a queried key will cause the endpoint to return `CONTRACT_NOT_FOUND` instead of the correct `Felt::ZERO`.

### Finding Description

In `get_storage_at` (lines 340–383), when `block_id` is `Tag(Pending)`, the handler correctly reads `pending_storage_diffs` and passes them to `execution_utils::get_storage_at`: [1](#0-0) 

`execution_utils::get_storage_at` checks pending storage diffs first; if the key is absent there, it falls back to the committed state reader, which returns `Felt::ZERO` for a contract that does not exist in committed state: [2](#0-1) 

When `res == Felt::ZERO`, the handler then checks whether the contract exists — but it calls `txn.get_state_reader().get_class_hash_at(state_number, &contract_address)` directly, which queries **committed state only**: [3](#0-2) 

The pending-aware equivalent, `execution_utils::get_class_hash_at`, exists and correctly searches `pending_deployed_contracts` and `pending_replaced_classes` before falling back to committed state: [4](#0-3) 

The same pattern is used correctly in `maybe_get_class_hash_at`, which reads pending deployed contracts and passes them through: [5](#0-4) 

**Concrete trigger sequence:**
1. A contract `C` appears in `pending_state_diff.deployed_contracts` but not in committed state.
2. Storage key `K` for `C` is either absent from `pending_storage_diffs` or explicitly set to zero.
3. Caller issues `starknet_getStorageAt(contract_address=C, key=K, block_id=Tag(Pending))`.
4. `execution_utils::get_storage_at` returns `Felt::ZERO` (key absent in pending diffs, contract absent in committed state).
5. The zero-value branch fires; `txn.get_state_reader().get_class_hash_at(state_number, C)` returns `None` because `C` is not in committed state.
6. `.ok_or_else(|| ErrorObjectOwned::from(CONTRACT_NOT_FOUND))?` fires — the caller receives `CONTRACT_NOT_FOUND`.

The correct answer is `Felt::ZERO`: the contract is deployed (in pending), and its storage at `K` is the default zero.

### Impact Explanation

The RPC pending view `starknet_getStorageAt` returns an authoritative-looking wrong value (`CONTRACT_NOT_FOUND`) for a valid query. Any client or dApp that deploys a contract in a pending transaction and immediately reads a zero-valued storage slot will receive a spurious error, misidentifying a live pending contract as non-existent. This falls under: **High — RPC pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

The scenario is straightforward: deploy a contract in a pending block, then query any storage key that has not been explicitly written (the vast majority of keys). This is a normal usage pattern for any dApp that reads contract state immediately after deployment before the block is accepted.

### Recommendation

Replace the raw `txn.get_state_reader().get_class_hash_at(...)` call in the zero-value branch with `execution_utils::get_class_hash_at(...)`, passing the pending `deployed_contracts` and `replaced_classes` from the pending state diff — exactly as `maybe_get_class_hash_at` does. The pending data must be read before the branch (or reused from an earlier read) and passed as `Some((&pending_deployed_contracts, &pending_replaced_classes))`.

pub fn get_class_hash_at<Mode: TransactionKind>(
    txn: &StorageTxn<'_, Mode>,
    state_number: StateNumber,
    pending_deployed_contracts_and_replaced_classes: Option<(
        &Vec<DeployedContract>,
        &Vec<ReplacedClass>,
    )>,
    contract_address: ContractAddress,
) -> StorageResult<Option<ClassHash>> {
    if let Some((pending_deployed_contracts, pending_replaced_classes)) =
        pending_deployed_contracts_and_replaced_classes
    {
        // Searching first in the replaced classes because if the contract was deployed and
        // replaced, the replaced class is the contract's class.
        for ReplacedClass { address, class_hash } in pending_replaced_classes {
            if *address == contract_address {
                return Ok(Some(*class_hash));
            }
        }
        for DeployedContract { address, class_hash } in pending_deployed_contracts {
            if *address == contract_address {
                return Ok(Some(*class_hash));
            }
        }
    }
    txn.get_state_reader()?.get_class_hash_at(state_number, &contract_address)
