### Title
Unbounded `stakers` Vector Growth via Instant Stake-and-Exit Griefs Consensus Mechanism - (File: `src/staking/staking.cairo`)

### Summary
Any unprivileged actor can repeatedly stake with the minimum amount and immediately call `unstake_intent`, permanently bloating the `stakers` vector. Because stakers are never removed from this vector, the `get_stakers` function — which the consensus mechanism calls to enumerate the active validator set — must iterate over every address ever registered, including all exited ones, causing unbounded growth in its execution cost.

### Finding Description
Every call to `stake()` appends the caller's address to the `stakers` storage vector: [1](#0-0) 

Neither `unstake_intent()`, `unstake_action()`, nor the internal `remove_staker()` helper ever removes an entry from this vector. The storage comment explicitly acknowledges this design: [2](#0-1) 

`get_stakers()`, the function used by the consensus layer to enumerate the active validator set, iterates over the **entire** vector unconditionally via `into_iter_full_range()`: [3](#0-2) 

For each entry it calls `is_staker_active()` and skips inactive stakers, but the storage read and branch still execute for every historical address. After `unstake_action()` the staker's `staker_info` is set to `VInternalStakerInfo::None`: [4](#0-3) 

so `is_staker_active` returns `false` immediately, but the slot in the vector is never reclaimed.

Because `assert_staker_address_not_reused` prevents address reuse: [5](#0-4) 

each attack cycle requires a fresh address, but the attacker recovers their stake after the `exit_wait_window` (default one week) and can reuse the same tokens with a new address indefinitely.

### Impact Explanation
`get_stakers` is the entry point the consensus mechanism uses to build the validator set for a given epoch: [6](#0-5) 

As the `stakers` vector grows without bound, the cost of executing `get_stakers` grows linearly with the total number of addresses ever registered. A sufficiently large vector makes the function prohibitively expensive to execute, degrading or halting the consensus layer's ability to enumerate validators — a direct griefing impact on the protocol with no financial gain for the attacker.

**Matched allowed impact:** Medium — Griefing with no profit motive but damage to users or protocol; Unbounded gas consumption.

### Likelihood Explanation
The attack requires only:
- `min_stake` tokens (fully recovered after each `exit_wait_window` cycle, so the recurring cost is gas only)
- A fresh EOA address per cycle (trivially generated)
- Two transactions per cycle: `stake` + `unstake_intent`

There is no rate-limiting, no fee that scales with the number of registrations, and no mechanism to prune the vector. The attack is permissionless, cheap, and repeatable.

### Recommendation
1. **Introduce a minimum lock period** between `stake` and `unstake_intent` (directly analogous to the Casimir fix). A staker should not be able to signal exit intent until at least one full epoch has elapsed since staking.
2. **Prune the `stakers` vector on exit.** In `remove_staker`, swap-and-pop the exited address out of the vector, or maintain a separate active-staker count so `get_stakers` can skip the full scan.
3. **Alternatively**, replace the append-only `Vec` with an iterable map keyed by staker address that supports O(1) deletion, so exited stakers are removed from the iteration domain entirely.

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
