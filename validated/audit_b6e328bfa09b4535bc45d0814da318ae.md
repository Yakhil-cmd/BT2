### Title
Unauthorized Global Contract Overwrite via Cross-Account Promise in `AccountId` Deploy Mode - (File: `runtime/runtime/src/global_contracts.rs`)

### Summary

`action_deploy_global_contract` performs no authorization check when `GlobalContractDeployMode::AccountId` is used. Any contract can create a cross-account promise targeting a victim account and append a `DeployGlobalContract(AccountId)` action, overwriting the victim's globally-shared contract code with attacker-controlled bytes. Every account on every shard that references the victim via `GlobalContractIdentifier::AccountId(victim)` will subsequently execute the attacker's code.

### Finding Description

`GlobalContractDeployMode::AccountId` stores the contract under the key `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(account_id) }`, where `account_id` is the **receipt receiver**. The documentation explicitly states this mode "allows the owner to update the contract for all its users," implying only the owner account should be able to write to that slot.

However, `action_deploy_global_contract` never checks that the action is self-initiated (i.e., that the receipt predecessor equals the receiver). The function signature does not even receive the predecessor:

```rust
// runtime/runtime/src/global_contracts.rs lines 23-61
pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,   // receipt receiver — used as the AccountId key
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    // Only balance check — no predecessor == receiver guard
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else { ... };
    initiate_distribution(state_update, account_id.clone(), ..., &deploy_contract.deploy_mode, ...)?;
    Ok(())
}
```

The host function exposed to Wasm contracts, `promise_batch_action_deploy_global_contract_by_account_id`, places no restriction on the promise target:

```rust
// runtime/near-vm-runner/src/logic/logic.rs lines 2558-2570
pub fn promise_batch_action_deploy_global_contract_by_account_id(
    &mut self, promise_idx: u64, code_len: u64, code_ptr: u64,
) -> Result<()> {
    self.promise_batch_action_deploy_global_contract_impl(
        promise_idx, code_len, code_ptr,
        GlobalContractDeployMode::AccountId,
        "promise_batch_action_deploy_global_contract_by_account_id",

```

Inside `promise_batch_action_deploy_global_contract_impl`, the only guards are a view-mode check and a size check. The `sir` (sender-is-receiver) flag only affects gas pricing, not authorization:

```rust
// runtime/near-vm-runner/src/logic/logic.rs lines 2573-2599
let (receipt_idx, sir) = self.promise_idx_to_receipt_idx_with_sir(promise_idx)?;
self.pay_action_base(ActionCosts::deploy_global_contract_base, sir)?;
self.pay_action_per_byte(ActionCosts::deploy_global_contract_byte, code_len, sir)?;
self.ext.append_action_deploy_global_contract(receipt_idx, code, mode);
```

The nonce mechanism (`increment_nonce` / `check_and_update_nonce`) prevents **stale** distribution receipts from overwriting a newer version, but it does not prevent an **unauthorized** account from initiating a fresh distribution with a higher nonce:

```rust
// runtime/runtime/src/global_contracts.rs lines 238-255
fn check_and_update_nonce(...) -> Result<bool, RuntimeError> {
    ...
    if incoming_nonce < stored_nonce { return Ok(false); }
    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
}
```

An attacker's fresh distribution always carries `stored_nonce + 1`, so it passes the freshness check unconditionally.

### Impact Explanation

An attacker deploys a malicious contract that calls `promise_batch_create(victim_account_id)` followed by `promise_batch_action_deploy_global_contract_by_account_id(promise_idx, malicious_code)`. The resulting action receipt executes on the victim account, charges the victim's balance for storage, and writes the attacker's Wasm binary to `TrieKey::GlobalContractCode { identifier: AccountId(victim) }` on every shard via the distribution pipeline. Every account that has called `UseGlobalContract(AccountId(victim))` now runs the attacker's code on every subsequent function call. The corrupted value is the global contract code slot for the victim's account ID, propagated to all shards.

### Likelihood Explanation

The attack requires only: (1) deploying a contract (any account can do this), and (2) the victim account having sufficient balance to cover the storage fee. No privileged role is needed. The host function is unconditionally available to any executing Wasm contract.

### Recommendation

In `action_deploy_global_contract`, when `deploy_mode == GlobalContractDeployMode::AccountId`, assert that the receipt's `predecessor_id` equals `account_id` (the receiver). This mirrors the ownership semantics stated in the documentation. The `receipt` object is already available in the calling context (`apply_action_receipt` in `runtime/runtime/src/lib.rs`) and can be threaded into `action_deploy_global_contract`.

Alternatively, restrict `promise_batch_action_deploy_global_contract_by_account_id` at the host-function level to only allow self-targeting promises (i.e., `promise_idx` must resolve to a receipt whose receiver equals `context.current_account_id`).

### Proof of Concept

```
// Attacker contract (pseudo-Rust using NEAR SDK)
#[near_bindgen]
impl AttackerContract {
    pub fn overwrite_victim_global_contract(&self, victim: AccountId) {
        // Create a promise targeting the victim account
        let promise = env::promise_batch_create(&victim);
        // Append DeployGlobalContract(AccountId mode) with malicious Wasm
        env::promise_batch_action_deploy_global_contract_by_account_id(
            promise,
            &MALICIOUS_WASM_BYTES,
        );
    }
}
```

1. Attacker calls `overwrite_victim_global_contract("victim.near")`.
2. The runtime creates an action receipt: `predecessor = attacker_contract.near`, `receiver = victim.near`, action = `DeployGlobalContract { code: MALICIOUS_WASM, deploy_mode: AccountId }`.
3. `action_deploy_global_contract` executes on `victim.near`, deducts storage cost from `victim.near`'s balance, and calls `initiate_distribution` with `account_id = "victim.near"`.
4. `initiate_distribution` sets `id = GlobalContractIdentifier::AccountId("victim.near")`, increments the nonce (now higher than any prior nonce), and enqueues a `GlobalContractDistributionReceipt` carrying the malicious Wasm.
5. `apply_distribution_current_shard` writes `MALICIOUS_WASM` to `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }` and forwards the receipt to all remaining shards.

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())

fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)

pub enum GlobalContractDeployMode {
    /// Contract is deployed under its code hash.
    /// Users will be able reference it by that hash.
    /// This effectively makes the contract immutable.
    CodeHash,
    /// Contract is deployed under the owner account id.
    /// Users will be able reference it by that account id.
    /// This allows the owner to update the contract for all its users.
    AccountId,
}
