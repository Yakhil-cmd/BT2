### Title
`PresignOutput` Not Consumed on Rerandomization Enables Presignature Reuse Leading to Private Key Extraction - (`File: src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both the OT-based and Robust ECDSA `RerandomizedPresignOutput::rerandomize_presign` functions accept the `PresignOutput` by shared reference (`&PresignOutput`) rather than by value. Because `PresignOutput` also derives `Clone` and `Serialize/Deserialize`, the presignature is never consumed after use. A malicious coordinator can therefore trigger two independent signing sessions that both rerandomize the same underlying presignature with different `(msg_hash, tweak)` pairs, producing two signatures with multiplicatively related nonces. This is the classical ECDSA nonce-reuse condition and allows full private key extraction.

---

### Finding Description

The protocol documentation is explicit that presignatures are one-time-use material:

> "It's **critical** that the output is then destroyed, so that no other group of parties attempts to re-use that output for another phase."
> "**Never reuse a presignature**, even across failed, aborted, or partially completed signing sessions."

Despite this, the API does not enforce single-use semantics at the type level.

**OT-based ECDSA** (`src/ecdsa/ot_based_ecdsa/mod.rs`):

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput { ... }

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,   // <-- reference, not consumed
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> { ... }
}
```

**Robust ECDSA** (`src/ecdsa/robust_ecdsa/mod.rs`):

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, ZeroizeOnDrop)]
pub struct PresignOutput { ... }

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,   // <-- reference, not consumed
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> { ... }
}
```

In both cases `rerandomize_presign` borrows the `PresignOutput` immutably. After the call returns, the caller still holds a live, valid `PresignOutput` that can be passed to `rerandomize_presign` again with different arguments. The `Clone` and `Serialize/Deserialize` derives make it trivial to duplicate or persist the value across sessions.

The downstream `sign()` functions do accept `RerandomizedPresignOutput` by value (consuming it), but this is too late: the damage is done at the rerandomization step, where the same raw presignature material is used to produce two distinct rerandomized outputs.

---

### Impact Explanation

ECDSA nonce reuse is a well-known total-break condition. If the same nonce `k` (or a linearly related one) is used to sign two different messages `m1` and `m2`, the private key `x` can be recovered algebraically from the two public signatures. The rerandomization step applies a deterministic scalar `δ` derived from `(X, ε, h, R, ρ)`. Two calls to `rerandomize_presign` on the same `PresignOutput` with different `(tweak, msg_hash)` inputs produce two `RerandomizedPresignOutput` values whose nonce shares are scalar multiples of the same underlying `k_i`. The resulting signatures satisfy the nonce-reuse relation, enabling private key extraction.

The Robust ECDSA security documentation explicitly names this:

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the **secret key can be recovered using standard ECDSA nonce-reuse attacks**."

Impact: **Critical** — full extraction of the aggregate private signing key.

---

### Likelihood Explanation

A malicious coordinator controls which presignature is assigned to which signing session. Because the library does not consume `PresignOutput` on rerandomization, the coordinator can:

1. Initiate signing session A → participants rerandomize their `PresignOutput` with `(tweak_A, hash_A)` and send their shares.
2. Abort or complete session A.
3. Initiate signing session B referencing the same presignature → participants rerandomize the same `PresignOutput` with `(tweak_B, hash_B)` and send their shares.

No participant detects the reuse because the library provides no consumed-flag, no ownership transfer, and no session-binding check on the `PresignOutput`. The coordinator is a documented trust boundary in both ECDSA variants, and the attack requires only that the coordinator be malicious — a realistic threat model for MPC deployments.

---

### Recommendation

Change `rerandomize_presign` to consume the `PresignOutput` by value in both ECDSA variants:

```rust
pub fn rerandomize_presign(
    presignature: PresignOutput,   // consume by value
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> { ... }
```

This makes the Rust ownership system enforce single-use semantics at compile time: once a `PresignOutput` has been rerandomized, it is moved and cannot be used again. Additionally, remove or gate the `Clone` derive on `PresignOutput` behind a `#[cfg(test)]` attribute so that production code cannot trivially circumvent the ownership discipline by cloning before passing.

The same recommendation applies to `TripleShare` in `src/ecdsa/ot_based_ecdsa/triples/mod.rs`, which also derives `Clone` and is documented as one-time-use input to presigning.

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
