Let me look at the key reward update flow and commission application timing more carefully.

### Title
Commission Updated Without Settling Current-Epoch Rewards, Enabling Retroactive Commission Extraction - (File: src/staking/staking.cairo)

### Summary
When a staker increases their commission via `set_commission` (permitted under an active `set_commission_commitment`), the new commission is written to storage immediately with no prior settlement of the current epoch's accrued rewards. Because `calculate_staker_pools_rewards` reads the **live** commission value at attestation/reward-update time, the higher commission is applied retroactively to the entire epoch's pool rewards — including rewards earned by delegators before the commission change. This is the direct analog of the external report's "inconsistent shares" bug: a rate parameter is updated without updating the dependent distribution records, causing incorrect reward splits.

### Finding Description
`set_commission` → `update_commission` writes the new commission to `staker_pool_info.commission` immediately:

```cairo
// src/staking/staking.cairo:1599-1600
// Update commission in storage.
staker_pool_info.commission.write(Option::Some(commission));
```

No reward settlement occurs before this write. Later, when `_update_rewards` is triggered (by attestation or `update_rewards`), it calls `calculate_staker_pools_rewards`, which reads the **current** commission from storage:

```cairo
// src/staking/staking.cairo:1964
let commission = staker_pool_info.commission();
```

This commission is then applied to the full epoch's `pool_rewards_including_commission` to compute the staker's `commission_rewards` and the delegators' `pool_rewards`:

```cairo
// src/staking/staking.cairo:1989-1991
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
```

Because `set_commission_commitment` explicitly allows a staker to raise commission up to `max_commission` while the commitment is active:

```cairo
// src/staking/staking.cairo:1583-1589
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission, ...
    );
    assert!(commission != old_commission, ...);
```

a staker can increase commission from `C_old` to `C_max` at any point during an epoch, and the higher rate will be applied to rewards earned by delegators throughout the **entire** epoch, not just from the moment of the change.

The code itself acknowledges this gap:

```cairo
// src/staking/staking.cairo:745-747
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
```

### Impact Explanation
Delegators who entered the pool at the start of an epoch under commission `C_old` have their unclaimed yield for that epoch recalculated at `C_max > C_old` when the staker attests. The difference `(C_max - C_old) × pool_rewards_including_commission` is transferred to the staker's `unclaimed_rewards_own` instead of to the pool. This is a direct theft of unclaimed yield from delegators — a **High** impact.

### Likelihood Explanation
Any staker who has previously called `set_commission_commitment` with a `max_commission` above their current commission can execute this. The commitment window can be up to one year (`expiration_epoch - current_epoch <= epochs_in_year()`). The staker controls the timing of both the commission increase and the attestation, making the attack straightforward to execute within any epoch covered by the commitment.

### Recommendation
Before writing the new commission to storage in `update_commission`, first settle the current epoch's rewards at the old commission rate (analogous to the external report's recommendation to save the share value in the distribution event). Alternatively, record the commission value per epoch and use the epoch-locked commission — not the live value — when computing `calculate_staker_pools_rewards`. This ensures that rewards earned under `C_old` are always split at `C_old`, regardless of when the commission is changed within the epoch.

### Proof of Concept
1. Staker stakes with `commission = 500` (5 %).
2. Staker calls `set_commission_commitment(max_commission: 5000, expiration_epoch: current + 10)`.
3. Delegators enter the pool, expecting ≤ 5 % commission.
4. At the last block of epoch N (just before attestation), staker calls `set_commission(commission: 5000)` — permitted because the commitment is active and `5000 ≤ max_commission`.
5. Staker (or sequencer) calls `update_rewards_from_attestation_contract` / `update_rewards`.
6. `calculate_staker_pools_rewards` reads `commission = 5000` and applies it to the full epoch's `pool_rewards_including_commission`.
7. Delegators receive 50 % of what they earned; the staker captures the other 50 % as commission — 10× the rate delegators agreed to for that epoch. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
