### Title
Unverified Invoke Transaction Admitted to Mempool via `skip_stateful_validations` Enables Griefing of New Account Deployments — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function bypasses the `__validate__` entry point for any invoke transaction whose nonce is exactly `1` and whose sender's on-chain nonce is `0`, provided `account_tx_in_pool_or_recent_block` returns `true` for that address. Because the check does not verify the transaction's signature or confirm that the pooled transaction is a `deploy_account`, an attacker who observes a victim's `deploy_account` in the mempool can immediately submit a second invoke with nonce `1` carrying an arbitrary (invalid) signature from the same address. The gateway admits it without running `__validate__`, placing an unverified transaction in the mempool and forcing the victim to pay a fee-escalation premium to displace it.

`account_tx_in_pool_or_recent_block` (mempool line 697) returns `true` if **any** transaction from the address is in the pool or in the mempool's committed-nonce history:

```rust

    self.state.contains_account(account_address)
        || self.tx_pool.contains_account(account_address)
}
```

It does **not** verify that the pooled transaction is a `deploy_account`, nor does it verify the incoming invoke's signature. The comment's claim — *"it means that either it has a deploy_account transaction or transactions with future nonces that passed validations"* — is the intended reasoning, but the code enforces neither condition.

**Concrete attack sequence:**

1. Victim Alice submits `deploy_account` for address `A` (nonce 0, valid signature). It enters the mempool.
2. Attacker observes `A` in the public mempool.
3. Attacker crafts `invoke { sender=A, nonce=1, signature=[garbage] }` and submits it.
4. Gateway evaluates:
   - `validate_nonce`: `0 ≤ 1 ≤ 200` → passes.
   - `validate_by_mempool` → `validate_fee_escalation`: no existing nonce-1 tx for `A` → passes.
   - `skip_stateful_validations`: `nonce==1`, `account_nonce==0`, `account_tx_in_pool_or_recent_block(A)==true` (Alice's `deploy_account` is in pool) → returns `true`.
   - `run_validate_entry_point`: **skipped**.
5. Attacker's invalid invoke is admitted to the mempool at `(A, nonce=1)`.
6. Alice submits her legitimate invoke at `(A, nonce=1)`. The mempool's `validate_fee_escalation` finds the attacker's tx already occupying that slot and returns `MempoolError::DuplicateNonce` unless Alice's tip and `max_l2_gas_price` both exceed the attacker's values by the configured `fee_escalation_percentage` (default 10%).
7. If Alice cannot or does not pay the premium, her invoke is rejected for that round. The batcher later executes Alice's `deploy_account` (account deployed), then attempts the attacker's invoke — `__validate__` fails, the tx is rejected — and Alice must resubmit.

---

### Impact Explanation

**Broken invariant:** Every transaction admitted to the mempool has passed the account's `__validate__` entry point (or an equivalent signature check). The `skip_stateful_validations` path violates this for invoke transactions with nonce 1 from undeployed accounts.

**Concrete effects:**
- An invalid transaction (unverified signature) is accepted by the gateway and enters the mempool — matching the "High: Mempool/gateway/RPC admission accepts invalid transactions" impact category.
- The attacker occupies nonce 1 for any new account whose `deploy_account` is visible in the mempool, forcing fee escalation or a one-block delay on the victim's first post-deployment invoke.
- The attack is repeatable for every new account deployment and costs the attacker only the minimum gas to submit the invalid invoke (no `deploy_account` fee required on the attacker's side).

---

### Likelihood Explanation

- `deploy_account` transactions are publicly visible in the mempool before they are sequenced.
- The attacker needs only to read the sender address from the pending `deploy_account` and submit a single invoke with nonce 1 and any signature bytes.
- No privileged access, no special contract, no prior on-chain state is required.
- The race window is the entire time the victim's `deploy_account` sits in the mempool (potentially many seconds to minutes).

---

### Recommendation

**Short term:** Inside `skip_stateful_validations`, additionally verify that the pooled transaction for the address is specifically a `deploy_account` transaction (not just any transaction). The mempool already stores transaction type; expose a `has_deploy_account_in_pool(address)` query and use it instead of the generic `account_tx_in_pool_or_recent_block`.

**Medium term:** Even when skipping the blockifier `__validate__` call (because the account contract does not exist yet), perform a lightweight off-chain signature pre-check against the class hash declared in the pending `deploy_account` transaction. This mirrors the intent of the UX feature while closing the unauthenticated-admission window.

**Long term:** Consider requiring the invoke and `deploy_account` to be submitted as an atomic bundle (a single RPC call), so the gateway can verify both together before either enters the mempool.

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    fn validate_fee_escalation(
        &self,
        incoming_tx_reference: TransactionReference,
    ) -> MempoolResult<Option<TransactionReference>> {
        let TransactionReference { address, nonce, .. } = incoming_tx_reference;

        self.validate_no_delayed_declare_front_run(incoming_tx_reference)?;

        if !self.config.static_config.enable_fee_escalation {
            if self.tx_pool.get_by_address_and_nonce(address, nonce).is_some() {
                return Err(MempoolError::DuplicateNonce { address, nonce });
            };

            return Ok(None);
        }

        let Some(existing_tx_reference) = self.tx_pool.get_by_address_and_nonce(address, nonce)
        else {
            // Replacement irrelevant: no existing transaction with the same nonce for address.
            return Ok(None);
        };

        if !self.should_replace_tx(&existing_tx_reference, &incoming_tx_reference) {
            info!(
                "{existing_tx_reference} was not replaced by {incoming_tx_reference} due to \
                 insufficient fee escalation."
            );
            // TODO(Elin): consider adding a more specific error type / message.
            return Err(MempoolError::DuplicateNonce { address, nonce });
        }

        Ok(Some(existing_tx_reference))
    }
