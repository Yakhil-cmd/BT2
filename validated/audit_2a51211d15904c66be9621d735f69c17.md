### Title
Gateway L2 Gas Price Threshold Uses Wrong Block Field, Admitting Transactions That Fail Execution - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads `gas_prices.strk_gas_prices.l2_gas_price` from `BlockInfo` to derive the admission threshold, but the correct field is `next_l2_gas_price` stored in the block header. The two values diverge whenever the EIP-1559 fee market adjusts the price between blocks. When the next block's price is higher than the current block's price, the gateway admits V3 (`AllResources`) transactions whose `max_price_per_unit` satisfies the stale threshold but falls below the actual execution-time price, causing them to fail blockifier pre-validation after mempool admission. When the next block's price is lower, the gateway incorrectly rejects transactions that would be valid for the next block.

### Finding Description

`validate_resource_bounds` is called for every incoming transaction during stateful validation:

```rust
async fn validate_resource_bounds(
    &self,
    executable_tx: &ExecutableTransaction,
) -> StatefulTransactionValidatorResult<()> {
    if self.config.validate_resource_bounds {
        // TODO(Arni): getnext_l2_gas_price from the block header.
        let previous_block_l2_gas_price = self
            .gateway_fixed_block_state_reader
            .get_block_info()
            .await?
            .gas_prices
            .strk_gas_prices
            .l2_gas_price;          // ← reads the price *used in* the latest block
        self.validate_tx_l2_gas_price_within_threshold(
            executable_tx.resource_bounds(),
            previous_block_l2_gas_price,
        )?;
    }
    Ok(())
}

`gas_prices.strk_gas_prices.l2_gas_price` is the price *applied inside* the latest committed block. The fee market computes a separate `next_l2_gas_price` field on the block header, which is the price that will govern the *next* block. These two values diverge whenever the block is over- or under-full. The TODO comment in the code itself acknowledges the correct field is `next_l2_gas_price` from the block header.

The downstream check compares the transaction's `max_price_per_unit` against a percentage of this stale price:

```rust
fn validate_tx_l2_gas_price_within_threshold(
    &self,
    tx_resource_bounds: ValidResourceBounds,
    previous_block_l2_gas_price: NonzeroGasPrice,
) -> StatefulTransactionValidatorResult<()> {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            let threshold = (gas_price_threshold_multiplier
                * previous_block_l2_gas_price.get().0)
                .to_integer();
            if tx_l2_gas_price.0 < threshold { return Err(...); }
        }
        ValidResourceBounds::L1Gas(_) => { /* no check */ }
    }
    Ok(())
}

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

When the fee market raises `next_l2_gas_price` above the current block's `l2_gas_price` (a normal occurrence on a busy network), the gateway's threshold is computed from the stale lower price. Any V3 `AllResources` transaction whose `l2_gas.max_price_per_unit` satisfies `threshold(current_price) ≤ price < next_price` passes gateway admission and enters the mempool, but is then rejected by the blockifier's `check_fee_bounds` pre-validation with `MaxGasPriceTooLow`. The reverse scenario (price falling) causes the gateway to reject transactions that the blockifier would accept, constituting false rejection of valid transactions.

### Likelihood Explanation

The EIP-1559-style fee market (`calculate_next_l2_gas_price_for_fin`) adjusts the L2 gas price every block based on gas usage. On any block that is meaningfully over- or under-full the two prices diverge. The gap is bounded by the EIP-1559 adjustment factor per block, but it is non-zero in normal operation and grows with sustained load. No privileged access is required; any user submitting a V3 transaction with `AllResources` bounds is affected. [6](#0-5) 

### Recommendation

Replace the `get_block_info()` call with a dedicated `get_next_l2_gas_price()` method on `GatewayFixedBlockStateReader` that reads `BlockHeaderWithoutHash::next_l2_gas_price` from storage. The variable should be renamed from `previous_block_l2_gas_price` to `next_block_l2_gas_price` to reflect the correct semantic. The existing TODO comment in the code confirms this is the intended fix.

```rust
// Correct:
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()   // reads BlockHeaderWithoutHash::next_l2_gas_price
    .await?;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    next_block_l2_gas_price,
)?;
```

### Proof of Concept

1. Observe the latest committed block has `gas_prices.strk_gas_prices.l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block was over-full).
2. Compute `threshold = P * min_gas_price_percentage / 100`. With the default `min_gas_price_percentage = 100`, `threshold = P`.
3. Submit a V3 `AllResources` invoke transaction with `l2_gas.max_price_per_unit = P` (satisfies gateway check since `P >= threshold`).
4. Gateway calls `validate_resource_bounds` → reads stale `P` → `P >= P` → **admits the transaction**.
5. Blockifier calls `check_fee_bounds` with actual block price `P'` → `P < P'` → **rejects with `MaxGasPriceTooLow`**.
6. The transaction occupies a mempool slot and consumes gateway validation resources despite being unexecutable. [7](#0-6) [8](#0-7)

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
