### Title
`change_reward_address` Redirects Accumulated Unclaimed Rewards Without First Distributing to Old Reward Address - (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

Both `change_reward_address` implementations — in the staking contract and the delegation pool contract — update the reward destination without first distributing accumulated unclaimed rewards to the old reward address. Any rewards that have accrued but not yet been claimed are silently redirected to the new address, permanently denying the old reward address its earned yield.

---

### Finding Description

**Staking contract** (`src/staking/staking.cairo`, lines 517–540):

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    // ...token check...
    let staker_address = get_caller_address();
    let mut staker_info = self.internal_staker_info(:staker_address);
    let old_address = staker_info.reward_address;

    // Update reward_address and commit to storage.
    staker_info.reward_address = reward_address;   // ← no prior claim
    self.write_staker_info(:staker_address, :staker_info);
    // ...emit event...
}
```

`staker_info.unclaimed_rewards_own` is never flushed to `old_address` before the pointer is overwritten. The next call to `claim_rewards` (or `unstake_action`) will send those rewards to `reward_address` instead. [1](#0-0) 

Compare with `unstake_action`, which correctly calls `send_rewards_to_staker` before erasing the record: [2](#0-1) 

**Delegation pool contract** (`src/pool/pool.cairo`, lines 505–526):

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {

    let pool_member = get_caller_address();
```

All rewards tracked via `_unclaimed_rewards_from_v0`, `reward_checkpoint`, and `entry_to_claim_from` will be paid to the new address on the next `claim_rewards` call. [3](#0-2) 

The `claim_rewards` function in the pool correctly sends rewards to `pool_member_info.reward_address` at the time of the call: [4](#0-3) 

---

### Impact Explanation

The reward address is an independent actor: both `staking.cairo` and `pool.cairo` explicitly allow the reward address itself to call `claim_rewards`, giving it an independent right to collect earned yield. [5](#0-4) [6](#0-5) 

A staker or pool member can therefore:

1. Set `reward_address` to a yield aggregator, DAO treasury, or any third-party contract that has been promised the yield.
2. Allow rewards to accumulate over many epochs (large `unclaimed_rewards_own` / pool reward checkpoints).
3. Call `change_reward_address(attacker_controlled_address)` — zero rewards are distributed to the old address.
4. Call `claim_rewards` — the full accumulated yield is paid to the attacker-controlled address.

The old reward address permanently loses all yield it had earned but not yet claimed. This is **theft of unclaimed yield** (High impact).

---

### Likelihood Explanation

- Callable by any staker or pool member — no privileged role required.
- No time lock, no cooldown, no prior-claim requirement on `change_reward_address`.
- The attack is profitable whenever `unclaimed_rewards_own > 0` (staking) or accumulated pool rewards > 0 (pool), which is the normal steady state after any attestation epoch.
- The spec itself documents the logic as simply "Change registered `reward_address`" with no mention of a prior distribution step, confirming the omission is structural. [7](#0-6) [8](#0-7) 

---

### Recommendation

Before overwriting `reward_address`, both functions should first distribute all accumulated rewards to the old address:

- In `staking.cairo`: call `send_rewards_to_staker` (the same helper used in `unstake_action`) before updating `staker_info.reward_address`.
- In `pool.cairo`: call the internal reward-calculation and transfer logic (the same path used in `claim_rewards`) before updating `pool_member_info.reward_address`.

This mirrors the pattern already established in `unstake_action`: [9](#0-8) 

---

### Proof of Concept

**Staking contract scenario:**

1. Staker stakes and sets `reward_address = yield_aggregator`.
2. Several attestation epochs pass; `staker_info.unclaimed_rewards_own` grows to `R`.
3. Staker calls `staking.change_reward_address(attacker_wallet)`.
   - `unclaimed_rewards_own` is **not** sent to `yield_aggregator`.
4. Staker (or anyone) calls `staking.claim_rewards(staker_address)`.
   - `send_rewards_to_staker` reads `staker_info.reward_address == attacker_wallet` and transfers `R` there.
5. `yield_aggregator` receives 0; `attacker_wallet` receives `R`.

**Pool member scenario:**

1. Pool member enters delegation pool with `reward_address = dao_treasury`.
2. Staker attests; pool rewards accumulate in the pool contract's trace.
3. Pool member calls `pool.change_reward_address(attacker_wallet)`.
   - No rewards are flushed to `dao_treasury`.
4. Pool member calls `pool.claim_rewards(pool_member)`.
   - Rewards are calculated from the trace and transferred to `attacker_wallet`.
5. `dao_treasury` receives 0. [10](#0-9) [11](#0-10)

```text
        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                !self.does_token_exist(token_address: reward_address),
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let old_address = staker_info.reward_address;

            // Update reward_address and commit to storage.
            staker_info.reward_address = reward_address;
            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.
            self
                .emit(
                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,
                    },
                );
        }
```

```text
        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
```
