### Title
Missing Caller Authentication in `update_rewards()` Allows Any Address to Freeze Staker Yield - (`File: src/staking/staking.cairo`)

### Summary

The `update_rewards()` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` to advance the global `last_reward_block` state variable without distributing rewards, permanently denying all stakers their yield for that block.

### Finding Description

The protocol specification at `docs/spec.md` line 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `StakingRewardsManagerImpl::update_rewards()` in `src/staking/staking.cairo` lines 1449–1507 performs no caller validation whatsoever. The only guards are:

1. `general_prerequisites()` — a contract-pause check
2. `current_block_number > self.last_reward_block.read()` — a per-block deduplication check
3. Staker existence and activity checks

There is no `get_sequencer_address()` call, no role check, and no grep match for "sequencer" anywhere in the production source files.

The critical state mutation is at line 1485:

```cairo
self.last_reward_block.write(current_block_number);
```

`last_reward_block` is a **global** (not per-staker) storage variable. Once it is written to the current block number, the deduplication assert at line 1454–1458 will revert any subsequent call in the same block with `REWARDS_ALREADY_UPDATED`. This means a single attacker call per block is sufficient to consume the entire block's reward slot for all stakers.

An attacker calls:

```cairo
update_rewards(staker_address: any_valid_staker, disable_rewards: true)
```

This passes all checks, writes `last_reward_block = current_block`, then hits the early-return branch at line 1487 (`if disable_rewards || self.is_pre_consensus() { return; }`) without distributing any rewards. The legitimate sequencer's subsequent call for the same block reverts.

### Impact Explanation

Every block that the attacker front-runs results in zero rewards distributed to all stakers and all delegation pool members for that block. Because `last_reward_block` is global, a single attacker transaction per block permanently forfeits that block's yield for the entire protocol. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators.

### Likelihood Explanation

The entry point is a public, permissionless function. No stake, no role, and no special setup is required. The attacker only needs to submit a transaction before the sequencer's `update_rewards` call in any given block. The attack is cheap (one transaction per block), repeatable indefinitely, and requires no profit motive.

### Recommendation

Add a sequencer-only guard at the top of `update_rewards()`, analogous to the existing `assert_caller_is_attestation_contract()` pattern used in `update_rewards_from_attestation_contract()`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   assert!(
+       get_caller_address() == starknet::get_sequencer_address(),
+       "{}",
+       Error::CALLER_IS_NOT_SEQUENCER,
+   );
    // ... rest of function
}
```

Alternatively, introduce a dedicated `SEQUENCER` role via the existing `RolesComponent` and gate the function with `self.roles.only_sequencer()`.

### Proof of Concept

1. Staker stakes and K epochs pass so the staker becomes active and eligible for rewards.
2. Consensus rewards are active (`is_pre_consensus()` returns false).
3. Attacker (any EOA) calls `update_rewards(staker_address: valid_staker, disable_rewards: true)` in block N.
4. The function passes all checks, writes `last_reward_block = N`, and returns without distributing rewards.
5. The legitimate sequencer calls `update_rewards(staker_address: valid_staker, disable_rewards: false)` in the same block N.
6. The call reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number (N) > last_reward_block (N)` is false.
7. All stakers and delegators receive zero rewards for block N.
8. Attacker repeats every block to permanently freeze all yield. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
