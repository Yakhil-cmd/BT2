### Title
Two-step `UpdateTokenMapping` can be front-run by another `CanChangeTokenMapping` holder, corrupting denom/contract binding and enabling unauthorized IBC voucher conversion — (File: `x/cronos/keeper/keeper.go`, `x/cronos/keeper/msg_server.go`)

---

### Summary

`RegisterOrUpdateTokenMapping` enforces a strict "denom not already mapped" invariant via `ensureDenomNotMapped`. Updating an existing external contract mapping therefore requires **two separate transactions**: (1) delete the old mapping (empty contract), then (2) set the new one. Because multiple addresses can simultaneously hold the `CanChangeTokenMapping` permission, a second permission holder can observe the delete transaction in the mempool and front-run the re-add with a malicious contract address, permanently corrupting the denom-to-contract binding.

---

### Finding Description

`SetExternalContractForDenom` unconditionally calls `ensureDenomNotMapped` before writing any mapping:

```go
// x/cronos/keeper/keeper.go
func (k Keeper) SetExternalContractForDenom(...) error {
    if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
        return err   // fails if denom already has any mapping
    }
    ...
}
``` [1](#0-0) 

`ensureDenomNotMapped` returns `ErrDenomAlreadyMapped` whenever `GetContractByDenom` finds an existing entry (external **or** auto-deployed):

```go
func (k Keeper) ensureDenomNotMapped(ctx sdk.Context, denom string) error {
    if contract, found := k.GetContractByDenom(ctx, denom); found {
        return errors.Wrapf(types.ErrDenomAlreadyMapped, ...)
    }
    return nil
}
``` [2](#0-1) 

`RegisterOrUpdateTokenMapping` (called by `UpdateTokenMapping`) therefore cannot atomically replace a mapping; the only path to update is:

1. **Tx₁** — send `UpdateTokenMapping(denom, contract="")` → `DeleteExternalContractForDenom` removes the entry.
2. **Tx₂** — send `UpdateTokenMapping(denom, contract=newAddr)` → `SetExternalContractForDenom` writes the new entry. [3](#0-2) 

The permission check in `UpdateTokenMapping` only verifies that the sender holds `CanChangeTokenMapping` — it does **not** restrict the operation to a single address:

```go
func (k msgServer) UpdateTokenMapping(...) (...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, ...)
    }
    ...
}
``` [4](#0-3) 

`HasPermission` iterates over all accounts and returns `true` for **any** account that holds the bit, meaning the admin can grant this permission to an arbitrary number of addresses: [5](#0-4) 

**Attack path:**

1. Admin grants `CanChangeTokenMapping` to addresses A (legitimate operator) and B (attacker).
2. Address A wants to migrate denom `ibc/XXXX` from `contractOld` to `contractNew`. It broadcasts **Tx₁**: `UpdateTokenMapping("ibc/XXXX", "")`.
3. Attacker B observes Tx₁ in the mempool. B deploys `maliciousContract` (any contract with bytecode, passing `ensureContractCode`) and broadcasts **Tx₂**: `UpdateTokenMapping("ibc/XXXX", maliciousContract)` with higher gas priority.
4. Tx₂ is included before Tx₁ in the same or next block. After Tx₁ executes (delete), Tx₂ immediately maps `ibc/XXXX → maliciousContract`.
5. Address A's **Tx₃** (`UpdateTokenMapping("ibc/XXXX", contractNew)`) now fails with `ErrDenomAlreadyMapped` because the denom is already bound to `maliciousContract`.
6. All subsequent IBC receives for `ibc/XXXX` trigger `OnRecvVouchers` → `ConvertVouchersToEvmCoins`, which mints against `maliciousContract`. A malicious contract can redirect minted tokens to the attacker or silently burn them, causing unauthorized conversion of IBC vouchers. [6](#0-5) 

---

### Impact Explanation

Corruption of the denom/contract binding is an explicitly listed **High** impact. Because `OnRecvVouchers` uses the corrupted mapping to mint CRC20 tokens for every subsequent IBC receive of that denom, the attacker's malicious contract can redirect or destroy those minted tokens — constituting an **unauthorized conversion/transfer of IBC vouchers**, which is a **Critical** impact under the allowed scope.

---

### Likelihood Explanation

**Medium.** Requires: (a) the admin to have granted `CanChangeTokenMapping` to at least two addresses, and (b) the attacker to observe and front-run the delete transaction. Cosmos mempool front-running is feasible for any node operator or well-connected participant. The two-step update pattern is the only way to change an existing mapping, making the window predictable and repeatable.

---

### Recommendation

Replace the two-step delete-then-set pattern with an atomic **upsert** operation in `RegisterOrUpdateTokenMapping`. Specifically, add a dedicated `UpdateExternalContractForDenom` path that deletes the old reverse entry and writes the new forward/reverse entries in a single transaction, bypassing the `ensureDenomNotMapped` guard when the caller is explicitly replacing an existing mapping. Alternatively, restrict `CanChangeTokenMapping` to exactly one address at a time (enforced on-chain), mirroring the recommendation in the external report.

---

### Proof of Concept

```
// State: ibc/XXXX → contractOld (external mapping exists)
// Actors: A holds CanChangeTokenMapping (legitimate), B holds CanChangeTokenMapping (attacker)

// Block N mempool:
// A broadcasts: MsgUpdateTokenMapping{Denom:"ibc/XXXX", Contract:""}   // Tx₁ (delete)
// B observes Tx₁, deploys maliciousContract, broadcasts:
//   MsgUpdateTokenMapping{Denom:"ibc/XXXX", Contract:maliciousContract} // Tx₂ (higher gas)

// Block N execution order (Tx₁ then Tx₂):
// After Tx₁: ibc/XXXX → (unmapped)
// After Tx₂: ibc/XXXX → maliciousContract  ← attacker wins the race

// A's follow-up Tx₃: MsgUpdateTokenMapping{Denom:"ibc/XXXX", Contract:contractNew}
// → FAILS: ErrDenomAlreadyMapped

// All future IBC receives for ibc/XXXX:
// OnRecvVouchers → ConvertVouchersToEvmCoins → mints on maliciousContract
// maliciousContract.mint(receiver, amount) → transfers amount to attacker instead
```

func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}

func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}

```go
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
