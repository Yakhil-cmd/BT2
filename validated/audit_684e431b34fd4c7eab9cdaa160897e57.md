### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `Staking` contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` every block, consuming the global `last_reward_block` slot without distributing rewards, permanently denying all stakers their consensus-phase yield.

### Finding Description
The protocol specification at `docs/spec.md` line 1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `StakingRewardsManagerImpl::update_rewards` in `src/staking/staking.cairo` (lines 1447–1488) contains no such check. The function only calls `self.general_prerequisites()` (a pause-check) and then verifies `current_block_number > self.last_reward_block.read()`. There is no `assert!(get_caller_address() == sequencer_address, ...)` or equivalent guard anywhere in the function body. A grep for `sequencer` in `staking.cairo` returns zero matches.

The critical sequence inside `update_rewards` is:

```
// line 1485
self.last_reward_block.write(current_block_number);   // ← written unconditionally

// line 1487
if disable_rewards || self.is_pre_consensus() {
    return;   // ← exits WITHOUT distributing rewards
}
```

`last_reward_block` is a **single global storage slot** shared across all stakers. Once it is written to the current block number, the guard `current_block_number > self.last_reward_block.read()` will revert any subsequent call in the same block with `REWARDS_ALREADY_UPDATED`. Therefore, whichever call arrives first in a block "consumes" the reward slot for that block.

An attacker who calls `update_rewards(any_valid_staker_address, disable_rewards: true)` first in every block will:
1. Advance `last_reward_block` to the current block.
2. Return immediately without distributing any rewards.
3. Block every legitimate sequencer call for that block.

Repeating this every block permanently prevents all stakers and delegators from accumulating consensus rewards.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

All stakers and delegators are denied consensus-phase rewards indefinitely. The `unclaimed_rewards_own` field of every staker and the cumulative sigma in every pool contract will never increase. Rewards are not merely delayed; they are permanently lost for each skipped block because the protocol has no mechanism to retroactively credit missed blocks.

### Likelihood Explanation
**High.** The entry point is a public, permissionless function. No tokens, no stake, no special role is required. The attacker only needs to submit a transaction before the sequencer in each block. On Starknet, where transaction ordering is controlled by the sequencer, a sophisticated attacker could exploit this during any window where the sequencer is not the first to call the function (e.g., during sequencer downtime, network congestion, or by front-running via fee escalation). Even a single missed block per epoch meaningfully reduces staker yield.

### Recommendation
Add a sequencer-only guard at the top of `update_rewards`, consistent with the specification:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    self.general_prerequisites();
    // ... rest of function
}
```

Alternatively, store the sequencer address at construction time and expose a governance function to update it, mirroring the pattern used for `attestation_contract` caller checks elsewhere in the codebase.

### Proof of Concept

1. Consensus rewards are activated (`set_consensus_rewards_first_epoch` has been called and K epochs have passed).
2. A staker with non-zero balance exists at `staker_address`.
3. Attacker (any EOA) calls:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(
       staker_address: staker_address,
       disable_rewards: true
   )
   ```
   — no `cheat_caller_address` needed; the function accepts any caller.
4. `last_reward_block` is set to the current block. No rewards are minted or credited.
5. The legitimate sequencer call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeating step 3 every block results in zero rewards ever reaching stakers or delegators.

This is directly confirmed by the flow test `update_rewards_disable_rewards_consensus_rewards_flow_test` (src/flow_test/test.cairo lines 2808–2917), which calls `system.update_rewards(:staker, disable_rewards: true)` **without any caller restriction**, and the subsequent sequencer call in the same block panics with `REWARDS_ALREADY_UPDATED`.     impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self

            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

fn update_rewards_disable_rewards_consensus_rewards_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    let stake_amount = system.staking.get_min_stake();
    let staker = system.new_staker(amount: stake_amount);
    let commission = 200;
    system.stake(:staker, amount: stake_amount, pool_enabled: false, :commission);
    system.advance_k_epochs();

    // Disable rewards = true with consensus off - no rewards
    system.update_rewards(:staker, disable_rewards: true);
    let rewards = system.staker_claim_rewards(:staker);
    assert!(rewards.is_zero());

    // Attempt again same block - panic
    let result = system
        .staking
        .rewards_manager_safe_dispatcher()
        .update_rewards(staker_address: staker.staker.address, disable_rewards: true);
    assert_panic_with_error(
        :result, expected_error: StakingError::REWARDS_ALREADY_UPDATED.describe(),
    );
    advance_block_number_global(blocks: 1);

    // Disable rewards = false with consensus off - no rewards
    system.update_rewards(:staker, disable_rewards: false);
    let rewards = system.staker_claim_rewards(:staker);
    assert!(rewards.is_zero());

    // Attempt again same block - panic
    let result = system
        .staking
        .rewards_manager_safe_dispatcher()
        .update_rewards(staker_address: staker.staker.address, disable_rewards: false);
    assert_panic_with_error(
        :result, expected_error: StakingError::REWARDS_ALREADY_UPDATED.describe(),
    );
    advance_block_number_global(blocks: 1);
