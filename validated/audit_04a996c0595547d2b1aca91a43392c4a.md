### Title
Lack of Event Type / Contract String Canonicalization in `NewEventFilter` Causes Silent Subscription Misses - (File: `engine/access/state_stream/filter.go`)

### Summary

`NewEventFilter` in the Access Node's state-stream layer accepts event-type and contract filter strings that contain uppercase hex digits in the address segment (e.g., `A.0000000000000ABC.Contract1.EventA`). Validation passes because `flow.HexToAddress` is case-insensitive, but the raw, un-normalized string is stored in the filter map. The Flow runtime always emits event types with lowercase hex addresses (e.g., `A.0000000000000abc.Contract1.EventA`). The subsequent case-sensitive map lookup in `Match` therefore never fires, and the subscriber silently receives zero events for the lifetime of the subscription.

---

### Finding Description

`NewEventFilter` processes three kinds of caller-supplied filter strings: `eventTypes`, `addresses`, and `contracts`.

**Address filter** (lines 93–100): the raw hex string is parsed through `flow.HexToAddress`, and the canonical `addr.String()` (always lowercase, no `0x` prefix) is stored. This path is correctly normalized. [1](#0-0) 

**Event-type filter** (lines 85–91): `validateEventType` → `events.ValidateEvent` → `flow.HexToAddress(parsed.Address)` accepts uppercase hex without error, but the raw string is stored verbatim in `f.EventTypes`. [2](#0-1) 

**Contract filter** (lines 102–107): `validateContract` only checks the structural format (`A.<hex>.<name>`), never normalizes the address segment, and stores the raw string in `f.Contracts`. [3](#0-2) 

During `Match`, both lookups are plain map key comparisons against the stored raw strings: [4](#0-3) 

`ParseEvent` extracts the address segment verbatim from the incoming event's type string (`parts[1]`), which the runtime always emits in lowercase hex: [5](#0-4) 

`validateContract` performs no address normalization: [6](#0-5) 

The same un-normalized path is exposed through the gRPC handler: [7](#0-6) 

and through the WebSocket provider: [8](#0-7) 

---

### Impact Explanation

Any unprivileged caller that submits a `SubscribeEvents` (gRPC) or WebSocket events subscription with an event-type or contract filter whose address segment contains at least one uppercase hex letter (e.g., `A.0000000000000ABC.FungibleToken.TokensDeposited`) will have their filter silently accepted — no error is returned — but the filter will never match any on-chain event for the lifetime of the subscription. Applications that rely on these subscriptions to detect deposits, contract deployments, or other critical on-chain state changes will miss every matching event with no indication that the filter is misconfigured.

**Impact: 3 / 5** — Subscribers silently miss events; no direct on-chain asset loss at the protocol layer, but downstream application logic that depends on event delivery (e.g., crediting deposits, triggering contract interactions) can fail silently.

---

### Likelihood Explanation

Flow addresses are 8-byte values commonly displayed as 16-character hex strings. Hex strings are routinely written in mixed or uppercase by developers and tooling. The Access API accepts the string without complaint, making it easy for an integrator to construct a filter that is structurally valid but semantically dead.

**Likelihood: 3 / 5** — Requires a caller to supply an uppercase hex digit in the address segment of an event-type or contract filter string; plausible for any developer copying an address from a block explorer or tool that renders hex in uppercase.

---

### Recommendation

In `NewEventFilter`, normalize the address segment of every event-type and contract filter string before storing it, mirroring the normalization already applied to the standalone `addresses` filter:

- For `eventTypes`: after `validateEventType` succeeds, reconstruct the canonical form using `addr.String()` for the address segment before inserting into `f.EventTypes`.
- For `contracts`: after `validateContract` succeeds, parse the address segment with `flow.HexToAddress`, call `.String()` on the result, and rebuild the canonical `A.<lowercase_addr>.<ContractName>` string before inserting into `f.Contracts`.

This mirrors the existing correct pattern at lines 93–99 and ensures that filter-time strings always match the lowercase-hex form emitted by the runtime.

---

### Proof of Concept

// Subscriber calls SubscribeEvents (gRPC or WebSocket) with:

// NewEventFilter stores the raw string:

func NewEventFilter(
	config EventFilterConfig,
	chain flow.Chain,
	eventTypes []string,
	addresses []string,
	contracts []string,
) (EventFilter, error) {
	// put some reasonable limits on the number of filters. Lookups use a map so they are fast,
	// this just puts a cap on the memory consumed per filter.
	if len(eventTypes) > config.MaxEventTypes {
		return EventFilter{}, fmt.Errorf("too many event types in filter (%d). use %d or fewer", len(eventTypes), config.MaxEventTypes)
	}

	if len(addresses) > config.MaxAddresses {
		return EventFilter{}, fmt.Errorf("too many addresses in filter (%d). use %d or fewer", len(addresses), config.MaxAddresses)
	}

	if len(contracts) > config.MaxContracts {
		return EventFilter{}, fmt.Errorf("too many contracts in filter (%d). use %d or fewer", len(contracts), config.MaxContracts)
	}

	f := EventFilter{
		EventTypes:        make(map[flow.EventType]struct{}, len(eventTypes)),
		Addresses:         make(map[string]struct{}, len(addresses)),
		Contracts:         make(map[string]struct{}, len(contracts)),
		EventFieldFilters: make(map[flow.EventType]FieldFilter),
	}

	// Check all of the filters to ensure they are correctly formatted. This helps avoid searching
	// with criteria that will never match.
	for _, event := range eventTypes {
		eventType := flow.EventType(event)
		if err := validateEventType(eventType, chain); err != nil {
			return EventFilter{}, err
		}
		f.EventTypes[eventType] = struct{}{}
	}

	for _, address := range addresses {
		addr := flow.HexToAddress(address)
		if err := validateAddress(addr, chain); err != nil {
			return EventFilter{}, err
		}
		// use the parsed address to make sure it will match the event address string exactly
		f.Addresses[addr.String()] = struct{}{}
	}

	for _, contract := range contracts {
		if err := validateContract(contract); err != nil {
			return EventFilter{}, err
		}
		f.Contracts[contract] = struct{}{}
	}

	f.hasFilters = len(f.EventTypes) > 0 || len(f.Addresses) > 0 || len(f.Contracts) > 0
	return f, nil
}

// validateContract ensures that the contract is in the correct format
func validateContract(contract string) error {
	if contract == "flow" {
		return nil
	}

	parts := strings.Split(contract, ".")
	if len(parts) != 3 || parts[0] != "A" {
		return fmt.Errorf("invalid contract: %s", contract)
	}
	return nil
}

func ParseEvent(eventType flow.EventType) (*ParsedEvent, error) {
	parts := strings.Split(string(eventType), ".")

	switch parts[0] {
	case "flow":
		if len(parts) == 2 {
			return &ParsedEvent{
				Type:         ProtocolEventType,
				EventType:    eventType,
				Contract:     parts[0],
				ContractName: parts[0],
				Name:         parts[1],
			}, nil
		}

	case "A":
		if len(parts) >= 4 {
			return &ParsedEvent{
				Type:         AccountEventType,
				EventType:    eventType,
				Address:      parts[1],
				Contract:     fmt.Sprintf("A.%s.%s", parts[1], parts[2]),
				ContractName: parts[2],
				Name:         parts[len(parts)-1],
			}, nil
		}
	}

	return nil, fmt.Errorf("invalid event type: %s", eventType)
