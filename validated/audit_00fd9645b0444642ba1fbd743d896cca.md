### Title
Router's Accumulated ETH Balance Drainable via Zero-ETH WETH Swaps — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`PeripheryPayments.pay()` uses `address(this).balance` — the router's **total** ETH balance — rather than only the current transaction's `msg.value` when settling WETH-input swaps. Any ETH left in the router from prior transactions (e.g., excess from exact-output swaps where `refundETH()` was never called) can be consumed by a subsequent caller who sends `msg.value = 0`, giving them a free swap at the expense of the user who left ETH behind.

---

### Finding Description

In `PeripheryPayments.pay()`, the WETH branch reads the router's full native balance:

```solidity
} else if (token == WETH) {
    uint256 nativeBalance = address(this).balance;   // ← total, not msg.value
    if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
    } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
    } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
    }
``` [1](#0-0) 

`nativeBalance` is not bounded to the ETH the current caller sent. It includes any ETH that accumulated in the router from earlier calls.

**How ETH accumulates between transactions:**

`exactOutputSingle` (and `exactOutput`) are payable and accept ETH upfront because the caller cannot know the exact input amount before execution. A user who sends `1 ETH` for a swap that only consumes `0.9 ETH` leaves `0.1 ETH` stranded in the router if they do not include a `refundETH()` call in the same multicall. The `receive()` guard (`if (msg.sender != WETH) revert NotWETH()`) only blocks bare ETH transfers — it does not prevent ETH from arriving via a payable function call. [2](#0-1) 

**Exploit path:**

1. Victim calls `exactOutputSingle{value: 1 ether}(…, tokenIn=WETH, …)` and omits `refundETH()`. The swap consumes `0.9 ETH`; `0.1 ETH` remains in the router.
2. Attacker calls `exactInputSingle{value: 0}(…, tokenIn=WETH, amountIn=0.1 ETH, …)`.
3. Inside `metricOmmSwapCallback`, `_justPayCallback` calls `pay(WETH, attacker, pool, 0.1 ETH)`.
4. `nativeBalance = 0.1 ETH >= value = 0.1 ETH` → the router wraps and transfers the victim's ETH to the pool on the attacker's behalf.
5. Attacker receives the swap output having paid nothing; victim's ETH is gone. [3](#0-2) 

The same path applies to `exactInput` (multi-hop) and `exactOutput` / `exactOutputSingle`.

---

### Impact Explanation

Any ETH stranded in the router between transactions is freely claimable by the next caller who issues a WETH-input swap with `msg.value = 0`. The victim loses their principal ETH; the attacker receives an equivalent swap output for free. The loss is bounded by the router's accumulated balance at the time of the attack, which can be non-trivial during periods of high exact-output WETH swap volume.

---

### Likelihood Explanation

Exact-output swaps with WETH as input are a standard user pattern (the caller cannot know the precise input amount in advance and must overpay). Users who do not atomically append `refundETH()` — whether through UI error, failed multicall encoding, or a standalone `exactOutputSingle` call — leave ETH in the router. An attacker monitoring the mempool or the router's balance can exploit this opportunistically with zero capital at risk.

---

### Recommendation

Replace the unconditional `address(this).balance` read with a value bounded to the current transaction's `msg.value`. One approach: pass `msg.value` into `pay()` as an explicit `nativeAvailable` parameter and cap the native-ETH branch to that amount, pulling the remainder from the payer via `safeTransferFrom`. Alternatively, enforce that `msg.value` exactly covers the required WETH amount before entering the swap, and revert otherwise — consistent with how the PhiFactory mitigation validates `msg.value` against `mintFee`.

---
