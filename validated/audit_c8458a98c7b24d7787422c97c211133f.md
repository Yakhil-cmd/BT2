### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` in `staking.cairo` has no caller check despite the protocol specification requiring "Only starkware sequencer." Any unprivileged address can call `update_rewards(staker_address, disable_rewards: true)`, which advances the global `last_reward_block` without distributing rewards. Because the function enforces a strict one-call-per-block invariant, the legitimate sequencer is then blocked from distributing rewards for that block. Repeated every block, this permanently freezes all stakers' unclaimed yield at zero cost beyond gas.

### Finding Description

The specification at `docs/spec.md` line 1644–1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1448–1488 contains no such check:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ← no get_caller_address() check here
    ...
    self.last_reward_block.write(current_block_number);   // global write

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits without distributing
    }
    // reward distribution follows only if we reach here
```

`last_reward_block` is a contract-level (global) storage variable — `self.last_reward_block.read()` takes no key argument. When any caller invokes `update_rewards(any_staker, disable_rewards: true)`:

1. `last_reward_block` is set to the current block number.
2. The function returns early — no rewards are distributed to any staker.
3. Any subsequent call in the same block (including by the legitimate sequencer for any staker) reverts with `REWARDS_ALREADY_UPDATED`.

The `disable_rewards` flag is caller-controlled with no validation. The sequencer's intended call with `disable_rewards: false` is permanently pre-empted for that block.

### Impact Explanation

An attacker calling `update_rewards(any_staker, disable_rewards: true)` once per block prevents the sequencer from distributing consensus-era rewards to **all** stakers for that block. Repeated every block, this permanently freezes all unclaimed yield across the entire protocol. This matches the allowed High impact: **Permanent freezing of unclaimed yield**.

Even a single call causes irreversible loss of one block's worth of rewards for all active stakers, since missed blocks are not retroactively compensated (the `last_reward_block` check is strictly `>`, not `>=`).

### Likelihood Explanation

The function is publicly callable with no access restriction. Any address can call it. The only cost is gas per block, which is low on Starknet L2. The attack requires no special knowledge, no privileged role, and no prior setup — a single transaction per block suffices.

### Recommendation

Add an explicit caller check matching the spec. For example, store the authorized sequencer address at initialization and assert it:

```rust
assert!(
    get_caller_address() == self.sequencer_address.read(),
    "{}",
    Error::CALLER_IS_NOT_SEQUENCER,
);
```

Alternatively, use the existing roles infrastructure (e.g., an `OPERATOR` or dedicated `SEQUENCER` role) consistent with the rest of the protocol's access control pattern.

### Proof of Concept

1. Attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` at block N.
2. `last_reward_block` is written to N; no rewards are distributed.
3. Sequencer attempts `update_rewards(staker_address, disable_rewards: false)` at block N → reverts with `REWARDS_ALREADY_UPDATED`.
4. All stakers miss rewards for block N.
5. Attacker repeats steps 1–4 every block → all stakers permanently receive zero consensus rewards.
