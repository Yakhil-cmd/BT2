Let me look at the exact deserialization path and how messages are routed to confirm the full attack chain.

### Title
Single Malicious Participant Can Permanently Abort DKG/Reshare/Refresh for All Honest Parties via Malformed Message — (`src/protocol/helpers.rs`)

---

### Summary

`recv_from_others` propagates deserialization errors via `?` with no retry or skip logic. A single malicious participant in a valid DKG/reshare/refresh session can inject one syntactically malformed (non-deserializable) message at the correct waitpoint, causing every honest party's `do_keyshare` future to return `Err(DeserializationError)` and permanently abort the protocol.

---

### Finding Description

**Root cause — `recv_from_others` propagates `chan.recv` errors unconditionally:** [1](#0-0) 

```rust
while !seen.full() {
    let (from, msg) = chan.recv(waitpoint).await?;   // ← ? aborts on any recv error
    if seen.put(from) {
        messages.push((from, msg));
    }
}
```

**`Comms::recv` returns `Err` on msgpack deserialization failure:** [2](#0-1) 

```rust
async fn recv<T: DeserializeOwned>(...) -> Result<(Participant, T), ProtocolError> {
    let (from, data) = self.incoming.pop(header).await;
    ...
    let decoded: Result<T, ...> = rmp_serde::decode::from_slice(message_data)...;
    Ok((from, decoded?))   // ← returns Err(DeserializationError) on bad msgpack
}
```

**`push_message` only validates the 40-byte routing header, not the msgpack body:** [3](#0-2) 

A malicious participant can craft a message with a valid 40-byte header (deterministic `root_shared()` channel tag + known sequential waitpoint) followed by arbitrary garbage bytes. `push_message` accepts and routes it; `Comms::recv` then fails deserialization and returns `Err`.

**`do_keyshare` calls `recv_from_others` at two waitpoints, both fatal:** [4](#0-3) [5](#0-4) 

Both calls use `await?`, so a deserialization error at either `wait_round_1` (commitment-hash collection) or `wait_round_3` (signing-share collection) immediately terminates `do_keyshare` with an error for the receiving honest party.

**Contrast: `reliable_broadcast_receive_all` correctly skips recv errors:** [6](#0-5) 

```rust
match chan.recv(wait).await {
    Ok(value) => (from, (sid, vote)) = value,
    _ => continue,   // ← silently skips bad messages
};
```

The inconsistency between these two receive patterns is the root cause.

---

### Impact Explanation

Every honest party running `do_keygen`, `do_reshare`, or `do_refresh` calls `do_keyshare`, which calls `recv_from_others` at `wait_round_1` and `wait_round_3`. A single malicious participant sends one crafted message per honest party. Each honest party aborts with `Err(DeserializationError)`. No key material is produced. The protocol cannot be retried without restarting the entire session (and the attacker can repeat the attack). This is permanent denial of DKG/reshare/refresh under valid protocol inputs. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

The attacker only needs to be a registered participant in the session — a condition that is explicitly part of the documented trust model (up to `(n−1)/3` malicious parties tolerated). The channel tag is deterministic and public; the waitpoint is sequential and predictable from the protocol flow. No cryptographic material needs to be known or broken. The attack requires sending one message per honest party.

---

### Recommendation

Change `recv_from_others` to skip messages that fail deserialization (and optionally log them), mirroring the pattern already used in `reliable_broadcast_receive_all`:

```rust
while !seen.full() {
    let Ok((from, msg)) = chan.recv(waitpoint).await else {
        continue;  // skip malformed messages, do not abort
    };
    if seen.put(from) {
        messages.push((from, msg));
    }
}
```

This ensures a single bad message does not abort the entire protocol for honest parties.

```rust
pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
```
