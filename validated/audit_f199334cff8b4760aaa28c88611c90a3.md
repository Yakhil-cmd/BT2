### Title
Staker Can Instantly Raise Commission to 100% via Commitment Mechanism, Stealing All Delegator Yield - (File: src/staking/staking.cairo)

### Summary

A staker can exploit the `set_commission_commitment` + `set_commission` flow to immediately raise their commission to 100%, redirecting all pool rewards to themselves and leaving delegators with zero yield for up to one full year, with no grace period for delegators to exit.

### Finding Description

The `set_commission_commitment` function allows a staker to set a `max_commission` ceiling (up to `COMMISSION_DENOMINATOR = 10000`, i.e. 100%) for a future period. While the mechanism was designed to give delegators assurance about commission bounds, it simultaneously enables the staker to **increase** commission to any value up to `max_commission` — including 100% — with immediate effect and no delay.

The `update_commission` internal function enforces the following logic:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
} else {
    assert!(commission < old_commission, ...);
}
```

When a commitment is active, the only constraints are that the new commission does not exceed `max_commission` and is not identical to the old commission. There is **no lower bound** — the commission can jump from 1% to 100% in a single call.

The developers themselves acknowledge this in a code comment:

```
/// **Note**: Current commission increase safeguards still allow for sudden commission changes.
```

The attack sequence is atomic from the staker's perspective:

1. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + epochs_in_year())` — sets a commitment allowing commission up to 100% for up to one year.
2. Staker immediately calls `set_commission(commission: 10000)` — raises commission to 100%.
3. All subsequent reward distributions use the new 100% commission; delegators receive 0% of pool rewards.
4. Delegators must call `exit_delegation_pool_intent` and wait through the full `exit_wait_window` before recovering their principal — during which they continue earning zero yield.

### Impact Explanation

Delegators lose all accrued yield from the moment of the commission change until they complete the exit process. Since the exit requires an intent + wait window (default one week, up to 12 weeks), and the commitment can last up to one year, delegators can be robbed of yield for an extended period. This constitutes **theft of unclaimed yield** (High severity).

### Likelihood Explanation

Any staker who has opened a delegation pool can execute this attack. It requires no special privileges beyond being a registered staker. The two transactions (`set_commission_commitment` then `set_commission`) can be submitted back-to-back, giving delegators no practical opportunity to react before the commission change takes effect. Delegators monitoring on-chain events would see the `CommissionCommitmentSet` event followed immediately by `CommissionChanged`, but by then the damage is already done for any rewards distributed in that block or later.

### Recommendation

Introduce an epoch-based delay between when a commission increase is committed and when it takes effect. Specifically:

- When `set_commission` is called with a value **higher** than the current commission (under an active commitment), the new commission should only take effect starting at `current_epoch + K` (analogous to how stake balance changes are deferred), giving delegators at least K epochs to observe the change and exit.
- Alternatively, enforce that `max_commission` in a commitment cannot exceed the current commission by more than a bounded increment per epoch, preventing sudden jumps to 100%.

### Proof of Concept

```
1. Staker stakes with commission = 500 (5%) and opens a pool.
2. Delegator delegates 1000 STRK to the pool.
3. Staker calls:
     set_commission_commitment(max_commission: 10000, expiration_epoch: get_current_epoch() + epochs_in_year())
4. Staker immediately calls:
     set_commission(commission: 10000)
5. Commission is now 100% — all pool rewards go to staker.
6. Delegator calls exit_delegation_pool_intent() but must wait exit_wait_window (up to 12 weeks).
7. During the entire wait period, delegator earns 0 rewards.
8. Staker collects 100% of pool rewards for up to one year.
```

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        /// **Note**: Current commission increase safeguards still allow for sudden commission
        /// changes.
        /// **Note**: Updating epoch info can impact the commission commitment expiration date.
        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
