### Title
Unauthorized Caller Can Invoke `update_rewards` with `disable_rewards: true`, Permanently Denying Staker Block Rewards - (File: `src/staking/staking.cairo`)

### Summary

The `update_rewards` function in the Staking contract is documented as callable only by the "starkware sequencer" but has no on-chain caller restriction. Any public address can call it with `disable_rewards: true`, consuming the per-block reward slot and preventing the legitimate sequencer from distributing rewards for that block.

### Finding Description

The spec explicitly states the access control for `update_rewards`:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` contains no caller check whatsoever:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no assert on get_caller_address()
``` [2](#0-1) 

The function unconditionally writes `current_block_number` to `last_reward_block` before checking `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [3](#0-2) 

Because `last_reward_block` is updated before the early return, any caller who invokes `update_rewards(staker_address, disable_rewards: true)` at block N will:
1. Set `last_reward_block = N`
2. Return without distributing any rewards

The legitimate sequencer's subsequent call at the same block N will revert with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

### Impact Explanation

When the attacker front-runs the sequencer every block with `disable_rewards: true`, stakers and their delegation pool members receive zero block rewards indefinitely. The `unclaimed_rewards_own` field of the staker is never incremented, and `update_rewards_from_staking_contract` is never called on pool contracts, so pool members also receive no rewards. [5](#0-4) 

This constitutes **permanent freezing of unclaimed yield** for targeted stakers, matching the High-severity impact category.

### Likelihood Explanation

The function is part of the public ABI (`IStakingRewardsManager`) and requires no special role, token balance, or prior state. Any EOA or contract can call it. The only cost is the gas for a Starknet transaction per block. An attacker targeting a specific staker (e.g., a competitor validator) can do so cheaply and continuously. [6](#0-5) 

### Recommendation

Add a caller restriction to `update_rewards` that asserts `get_caller_address()` equals the designated sequencer address (or a role-controlled address). This mirrors how `update_rewards_from_attestation_contract` is protected:

```cairo
fn update_rewards_from_attestation_contract(...) {
    ...
    self.assert_caller_is_attestation_contract();
    ...
}
``` [7](#0-6) 

A similar `assert_caller_is_sequencer()` guard should be added at the top of `update_rewards`.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Staker S has been staked for K+ epochs and has non-zero balance.
3. At block N, the sequencer prepares to call `update_rewards(S, false)`.
4. Attacker A calls `update_rewards(S, true)` first (front-run or simply in the same block before the sequencer).
5. `last_reward_block` is set to N; no rewards are distributed.
6. Sequencer's call reverts with `REWARDS_ALREADY_UPDATED`.
7. Staker S receives zero rewards for block N.
8. Repeated every block → staker S and all pool members receive zero rewards indefinitely. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
