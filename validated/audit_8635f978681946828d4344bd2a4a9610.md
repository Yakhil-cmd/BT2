### Title
WalletConnect `take_offer` Confirmation Dialog Displays Attacker-Controlled Royalty Percentage, Causing User to Approve Incorrect Spend Amount - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary
The `parseCommandDisplay` function builds the "You Spend / You Receive" confirmation dialog for WalletConnect `chia_wallet.take_offer` commands. The "Total Amount with Royalties" figure shown to the user is derived from the `royalty_percentage` embedded in the offer blob's driver dict — data fully controlled by the dapp/offer creator. A fallback path explicitly leaves this attacker-supplied value in place when the local `nftGetInfo` lookup fails. A malicious dapp can therefore suppress or inflate the displayed royalty, causing the user to approve a spend that differs from what the dialog shows.

### Finding Description

**Data flow for `chia_wallet.take_offer`:**

1. `parseCommandDisplay` calls `getOfferSummary(params.offer)` — the offer string is supplied by the WalletConnect dapp.
2. The daemon returns `summary.infos`, which is the driver dict embedded in the offer blob by the offer creator.
3. `offerSummaryRoyaltyPercentages` reads `royalty_percentage` directly from `summary.infos[assetId].also.also.transfer_program.royalty_percentage` — no cross-check against on-chain NFT state. [1](#0-0) 

4. This value is passed as the initial `royaltyPercentage` to `parseWalletDeltaItem`. The code then calls `nftGetInfo` and, **only if it succeeds and returns `data_uris`**, overrides the value: [2](#0-1) 

5. The `catch` block is silent and the comment reads *"NFT type has already been resolved from offer data; metadata is best effort."* — explicitly acknowledging the fallback to the attacker-supplied value.
6. `withRoyaltyTotals` → `formatAmountWithRoyalties` uses the surviving `royaltyPercentage` to compute `amountWithRoyalties` shown in the `Confirm` dialog: [3](#0-2) 

7. The `Confirm` dialog renders this as the authoritative "Total Amount with Royalties" the user sees before clicking Approve: [4](#0-3) 

**Same root cause for `create_offer_for_ids`:** `params.driver_dict` is taken verbatim from the dapp's WalletConnect params and fed into `createOfferRoyaltyPercentages` with no on-chain validation: [5](#0-4) 

### Impact Explanation

A malicious WalletConnect dapp sends a `take_offer` command whose offer blob contains `royalty_percentage: 0` in the driver dict for an NFT that actually carries, say, a 10 % on-chain royalty. The confirmation dialog shows:

> You Spend: **1 XCH** (Total Amount with Royalties: **1 XCH**)

The user approves. The Chia blockchain enforces the real 10 % royalty at spend time, so the wallet actually deducts **1.1 XCH**. The user approved a spend of 1 XCH but lost 1.1 XCH — a direct, unprivileged-attacker-caused accounting discrepancy affecting XCH balance.

Conversely, an inflated `royalty_percentage` causes the dialog to show a higher total than will actually be spent, potentially causing the user to reject a legitimate offer (denial-of-service on offer acceptance).

### Likelihood Explanation

- Any WalletConnect dapp granted `chia_wallet.take_offer` permission can trigger this path.
- The `nftGetInfo` fallback is explicitly coded as "best effort"; it fails whenever the NFT is not yet indexed in the local wallet, the daemon is unreachable, or `data_uris` is absent from the response.
- Crafting an offer blob with a custom driver dict is straightforward using the Chia SDK.

### Recommendation

Cross-validate the royalty percentage from the offer summary against the authoritative on-chain value returned by `nftGetInfo`. If `nftGetInfo` fails, display the royalty as **unknown / unverified** rather than silently falling back to the offer-embedded value. Specifically:

- In `parseWalletDeltaItem`, if `nftGetInfo` fails or does not return `royalty_percentage`, set `result.royaltyPercentage = undefined` and render the total-with-royalties line as "Unable to verify royalties" rather than computing a figure from untrusted data.
- In `offerSummaryRoyaltyPercentages`, do not propagate `royalty_percentage` from `summary.infos` to the display layer at all; rely solely on the `nftGetInfo` result.

function offerSummaryRoyaltyPercentages(offerSummary: OfferSummaryForDisplay): AssetRoyaltyPercentages {
  const royaltyPercentages: AssetRoyaltyPercentages = {
    spending: {},
    receiving: {},
  };

  const { infos } = offerSummary;
  if (!isPlainObject(infos)) {
    return royaltyPercentages;
  }

  for (const assetId of Object.keys(offerSummary.requested)) {
    const parsedAssetId = assetId === 'xch' ? '1' : assetId;
    royaltyPercentages.spending[parsedAssetId] = royaltyPercentageForDriverInfo(infos[assetId]);
  }

  for (const assetId of Object.keys(offerSummary.offered)) {
    const parsedAssetId = assetId === 'xch' ? '1' : assetId;
    royaltyPercentages.receiving[parsedAssetId] = royaltyPercentageForDriverInfo(infos[assetId]);
  }

  return royaltyPercentages;

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }
