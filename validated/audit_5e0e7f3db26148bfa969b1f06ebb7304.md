### Title
Gateway Admits Invoke Transactions with Unverified Signatures via `skip_stateful_validations` Bypass — (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

---

### Summary

The `skip_stateful_validations` function in the gateway's stateful transaction validator unconditionally skips the account's `__validate__` entry point for any invoke transaction with `nonce=1` when the sender account has any transaction present in the mempool. Because `__validate__` is the sole place where the account contract verifies the transaction signature, an attacker can submit an invoke transaction carrying an arbitrary (invalid) signature that passes all gateway checks and is admitted to the mempool without signature verification.

---

### Finding Description

**Trigger condition** — `skip_stateful_validations` returns `true` (skip validation) when all three hold:

1. The transaction is `ExecutableTransaction::Invoke` with `nonce == Nonce(Felt::ONE)`.
2. The on-chain account nonce is `Nonce(Felt::ZERO)` (account not yet deployed).
3. `mempool_client.account_tx_in_pool_or_recent_block(sender)` returns `true`. [1](#0-0) 

When `skip_stateful_validations` returns `true`, `run_validate_entry_point` is called with `validate: false`: [2](#0-1) 

Inside `StatefulValidator::perform_validations`, the `validate=false` flag causes an early return before the `__validate__` call: [3](#0-2) 

The stateless validator only checks signature **length**, never cryptographic validity: [4](#0-3) 

**Attack path:**

1. Attacker generates a fresh account address (any deployed account class, e.g. OpenZeppelin).
2. Attacker submits a valid `deploy_account` transaction (nonce=0). This passes full gateway validation including `__validate_deploy__`, and is inserted into the mempool.
3. `account_tx_in_pool_or_recent_block` now returns `true` for that address: [5](#0-4) 

4. Attacker immediately submits an invoke transaction with `nonce=1` and a completely fabricated signature (e.g., `[0xdead, 0xbeef]`).
5. Gateway stateful validation: `account_nonce=0`, `tx.nonce()=1`, account is in mempool → `skip_stateful_validations` returns `true` → `__validate__` is never called → transaction is admitted to the mempool.

The full admission path is: [6](#0-5) 

**Broken invariant:** Every transaction admitted to the mempool must have its signature verified by the account's `__validate__` entry point before sequencing. This invariant is violated for invoke transactions with `nonce=1` from undeployed accounts.

---

### Impact Explanation

The gateway admits invoke transactions with unverified (potentially invalid) signatures to the mempool. These transactions will fail during block execution — either at `perform_pre_validation_stage` (insufficient balance, since the account is not yet deployed) or at the `__validate__` entry point — but they are accepted as valid by the admission layer. This matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

An attacker can flood the mempool with structurally valid but cryptographically unsigned invoke transactions at the cost of one `deploy_account` transaction per fresh account address, consuming mempool capacity and batcher execution resources.

---

### Likelihood Explanation

**Medium.** The attacker must submit a valid `deploy_account` transaction (paying its fee) to seed the mempool check. However, account class hashes for standard wallet contracts are publicly known, and the `deploy_account` transaction itself does not require the attacker to control the private key — only the constructor calldata and salt. The attacker can generate arbitrarily many fresh account addresses and repeat the pattern at linear cost.

---

### Recommendation

**Short term:** In `skip_stateful_validations`, do not return `true` (skip validation) solely because the account exists in the mempool. Instead, verify the signature independently (e.g., via a lightweight ECDSA check on the transaction hash) before skipping the full `__validate__` entry point, or restrict the skip to cases where the deploy_account transaction hash is explicitly provided and verified.

**Long term:** Audit all gateway admission paths that set `execution_flags.validate = false`. Any path that bypasses `__validate__` must provide an equivalent cryptographic guarantee that the transaction was authorized by the account owner. Consider whether the UX benefit of the deploy_account + invoke shortcut justifies the admission of unverified signatures.

---

### Proof of Concept

```
1. Choose any known account class hash H (e.g., standard ECDSA account).
2. Pick a random salt S and compute the deploy address A = pedersen(H, S, ...).
3. Submit RpcTransaction::DeployAccount { class_hash: H, salt: S, nonce: 0, signature: <valid> }
   → Gateway runs __validate_deploy__, tx passes, inserted into mempool.
4. Submit RpcTransaction::Invoke {
       sender_address: A,
       nonce: 1,
       signature: [0xdeadbeef, 0xcafebabe],   // arbitrary invalid signature
       calldata: [...],
       resource_bounds: <valid>,
   }
   → Gateway calls skip_stateful_validations:
       tx.nonce() == 1  ✓
       account_nonce == 0  ✓
       account_tx_in_pool_or_recent_block(A) == true  ✓  (deploy_account is in pool)
   → skip_validate = true → __validate__ NOT called → tx admitted to mempool.
5. Batcher picks up both transactions. deploy_account executes (account deployed).
   Invoke tx executes: __validate__ is now called with the invalid signature → FAILS.
   Sequencer wasted execution resources; mempool admitted an unauthorized transaction.

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
