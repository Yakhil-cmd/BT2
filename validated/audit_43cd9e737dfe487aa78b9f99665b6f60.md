### Title
Unprivileged caller can permanently deny block rewards to all stakers by front-running `update_rewards` with `disable_rewards: true` — (File: `src/staking/staking.cairo`)

### Summary

`IStakingRewardsManager.update_rewards` is a public, permissionless function that writes to the global `last_reward_block` storage variable. Because the function accepts a caller-controlled `disable_rewards` flag and has no access-control guard beyond "not paused / not zero address," any unprivileged actor can call it once per block with `disable_rewards: true` to advance `last_reward_block` without distributing any rewards. All subsequent legitimate calls in the same block revert with `REWARDS_ALREADY_UPDATED`, permanently destroying the block's reward for every staker.

### Impact Explanation

`last_reward_block` is a **global** counter. Once it is set to block `N` by any caller, no other call to `update_rewards` can succeed in block `N` (the `current_block_number > last_reward_block` assertion fails). Because the rewards for block `N` are never credited to `staker_info.unclaimed_rewards_own` and never forwarded to pool contracts via `update_pool_rewards`, those rewards are permanently lost — they are never minted/claimed from the reward supplier for that block.

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block:
- Pays only gas (cheap on Starknet L2).
- Permanently destroys every staker's and every pool member's block reward for that block.
- Can repeat this every block, effectively zeroing out all consensus-era reward distribution.

This maps directly to **High: Permanent freezing of unclaimed yield** in the allowed impact scope.

### Likelihood Explanation

- The function is publicly callable with no economic barrier beyond gas.
- A single transaction per block is sufficient to affect all stakers simultaneously.
- The attack is trivially automatable (watch for new blocks, submit one tx).
- There is no on-chain mechanism to recover the lost rewards after `last_reward_block` is advanced.
- Likelihood: **High**.

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus/sequencer contract), analogous to how `update_rewards_from_attestation_contract` is gated:

```cairo
// src/staking/staking.cairo  lines 2219-2225
fn assert_caller_is_attestation_contract(self: @ContractState) {
    assert!(
        get_caller_address() == self.attestation_contract.read(),
        "{}",
        Error::CALLER_IS_NOT_ATTESTATION_CONTRACT,
    );
}
``` [5](#0-4) 

Add an equivalent `assert_caller_is_consensus_contract` guard at the top of `update_rewards`, or store and check an authorized `consensus_contract` address in storage.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Block `N` begins. The consensus contract is about to call `update_rewards(staker_A, false)` to distribute rewards.
3. Attacker (any EOA) front-runs with: `update_rewards(staker_A, disable_rewards: true)`.
   - `current_block_number (N) > last_reward_block` → passes.
   - `last_reward_block` is written to `N`.
   - Function returns early; no rewards are distributed.
4. Consensus contract's call arrives: `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
5. `staker_A.unclaimed_rewards_own` is unchanged; pool contracts receive nothing for block `N`.
6. Attacker repeats step 3 every block → all stakers and delegators earn zero consensus rewards indefinitely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
