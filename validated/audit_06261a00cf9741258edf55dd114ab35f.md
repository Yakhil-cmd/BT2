### Title
Unbounded Growth of `stakers` Vec Causes Unbounded Gas Consumption in `get_stakers()` - (File: src/staking/staking.cairo)

---

### Summary

The `stakers` storage `Vec<ContractAddress>` in `staking.cairo` is append-only: staker addresses are added on `stake()` but **never removed** when a staker calls `unstake_action()`. The `get_stakers()` function iterates over the full range of this Vec on every call, reading every stale entry from storage. As the Vec grows unboundedly, `get_stakers()` consumes unbounded gas, eventually becoming uncallable and breaking the consensus mechanism that depends on it.

---

### Finding Description

The storage layout explicitly documents the design:

```
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [1](#0-0) 

`get_stakers()` iterates over the **entire** Vec unconditionally via `into_iter_full_range()`, performing one storage read per entry to check `is_staker_active()`:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    ...
}
``` [2](#0-1) 

Stale entries (addresses of stakers who have completed `unstake_action()`) are never pruned. Each such entry still costs a storage read on every `get_stakers()` invocation. In Starknet, storage reads are metered; as the Vec grows, the cumulative cost grows linearly with the total number of stakers who have ever participated, not just active ones.

The analog to the original report is direct: the original used Solidity `delete` to zero array slots without shrinking the array; here, `unstake_action()` erases the staker's `VInternalStakerInfo` record but leaves the address permanently in `stakers`, producing the same unbounded-iteration pattern. [3](#0-2) 

---

### Impact Explanation

`get_stakers()` is the sole source of truth for the consensus layer to determine which stakers are eligible to attest in a given epoch:

```
fn get_stakers(self: @ContractState, epoch_id: Epoch)
  -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)>
``` [4](#0-3) 

The spec confirms this is the entry point for consensus staker enumeration: [5](#0-4) 

If `get_stakers()` exceeds the Starknet call gas limit, the consensus mechanism cannot enumerate the active staker set. This constitutes **unbounded gas consumption** leading to a **griefing / protocol damage** scenario (Medium impact per the allowed scope).

---

### Likelihood Explanation

Any unprivileged user can call `stake()` (adding themselves to `stakers`) and later `unstake_action()` (leaving a permanent stale entry). The minimum stake is returned after unstaking, so the net cost per stale entry is only gas. An adversary willing to cycle through many accounts can inflate the Vec at low marginal cost. Even without a deliberate attacker, organic protocol growth over time will cause the same degradation.

---

### Recommendation

When `unstake_action()` removes a staker's `VInternalStakerInfo`, also remove the corresponding address from the `stakers` Vec (swap-and-pop, or maintain a separate index map for O(1) removal). Alternatively, replace the `Vec` with the existing `IterableMap` pattern (already used for `btc_tokens` and `pools`) which supports deletion, so that `get_stakers()` only iterates over currently-registered stakers. [6](#0-5) 

---

### Proof of Concept

1. Deploy the staking contract.
2. In a loop, for `N` accounts: call `stake(min_stake, ...)` → wait exit window → call `unstake_action()`. Each iteration appends one permanent address to `stakers`.
3. After `N` iterations, call `get_stakers(epoch_id)`.
4. Observe that the call performs `N` storage reads for stale entries before returning the (small) active set.
5. As `N` grows, the gas cost of `get_stakers()` grows linearly. At sufficiently large `N`, the call exceeds the Starknet gas cap and reverts, making the consensus staker enumeration permanently unavailable. [1](#0-0) [7](#0-6)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
