### Title
Changing Reward Supplier While Stakers Have Unclaimed Rewards Permanently Freezes Those Rewards - (File: src/staking/staking.cairo)

### Summary
`IStakingConfig::set_reward_supplier` replaces the `reward_supplier_dispatcher` address with no migration of the old supplier's `unclaimed_rewards` state. Any staker who accumulated `unclaimed_rewards_own` before the switch can never claim those rewards from the new supplier, because the new supplier's `unclaimed_rewards` counter starts at only 1 STRK and has no record of the previously accrued obligations.

### Finding Description

`set_reward_supplier` in `StakingConfigImpl` performs a bare address swap: [1](#0-0) 

The reward supplier contract tracks a global `unclaimed_rewards` counter. Every time the staking contract distributes rewards it calls `update_unclaimed_rewards_from_staking_contract(rewards)` on the reward supplier, incrementing that counter: [2](#0-1) 

When a staker claims, the staking contract calls `claim_rewards(amount)` on the reward supplier, which enforces: [3](#0-2) 

A freshly deployed reward supplier initialises `unclaimed_rewards` to exactly 1 STRK (`STRK_IN_FRIS`): [4](#0-3) 

After `set_reward_supplier` is called:

- The **old** supplier retains its `unclaimed_rewards = X` (the sum of all past reward distributions), but the staking contract no longer holds a reference to it.
- The **new** supplier starts with `unclaimed_rewards = 1 STRK`.
- Every staker's `unclaimed_rewards_own` field in `InternalStakerInfo` is unchanged — it still reflects the full historical accumulation.

When any staker whose `unclaimed_rewards_own > 1 STRK` calls `claim_rewards`, the staking contract dispatches `new_supplier.claim_rewards(amount)`. The assertion `amount <= unclaimed_rewards` (1 STRK) fails with `AMOUNT_TOO_HIGH`, reverting the transaction.

The old rewards are **permanently** inaccessible because:
1. The staking contract no longer references the old supplier.
2. The new supplier's `unclaimed_rewards` only grows through future reward distributions; it never retroactively accounts for obligations that were charged to the old supplier.
3. There is no function in either contract to migrate or inject the missing `unclaimed_rewards` balance. [1](#0-0) [5](#0-4) 

### Impact Explanation

Every staker who had non-zero `unclaimed_rewards_own` at the moment of the supplier swap permanently loses access to those rewards. The staking contract's `claim_rewards` path will revert for them indefinitely, because the new supplier's `unclaimed_rewards` counter can never retroactively cover obligations that were recorded against the old supplier. This constitutes **permanent freezing of unclaimed yield** for all affected stakers.

### Likelihood Explanation

`set_reward_supplier` is a documented, callable admin function. It would naturally be invoked during a protocol upgrade or when deploying a new reward supplier contract. At any point after the protocol has been live for more than a trivial period, virtually every active staker will have a non-zero `unclaimed_rewards_own`. The combination of a routine admin action and a universally non-zero staker state makes this a realistic scenario.

### Recommendation

Before the new supplier is activated, migrate the outstanding obligations. Concretely:

1. Read `old_supplier.contract_parameters_v1().unclaimed_rewards` to obtain the total owed amount.
2. Call `new_supplier.update_unclaimed_rewards_from_staking_contract(migrated_amount)` (restricted to the staking contract) so the new supplier's counter reflects the inherited obligations.
3. Ensure the new supplier has sufficient STRK token balance (or has issued the corresponding L1 mint requests) to cover those obligations before the swap takes effect.

Alternatively, record the reward supplier address inside each staker's info at the time rewards are accrued, and use that stored address when the staker later claims — directly mirroring the recommendation in the referenced external report.

### Proof of Concept

1. Staker A stakes and, over several epochs, accumulates `unclaimed_rewards_own = 500 STRK`.
2. Token admin calls `set_reward_supplier(new_supplier)`. New supplier is freshly deployed with `unclaimed_rewards = 1 STRK`.
3. Staker A calls `staking.claim_rewards(staker_A)`.
4. Staking contract internally calls `new_supplier.claim_rewards(500 STRK)`.
5. New supplier evaluates `500 STRK <= 1 STRK` → **reverts** with `AMOUNT_TOO_HIGH`.
6. Staker A's 500 STRK in accumulated rewards is permanently frozen; no path exists to recover it from the new supplier, and the old supplier is no longer reachable from the staking contract. [3](#0-2) [1](#0-0)

        fn set_reward_supplier(ref self: ContractState, reward_supplier: ContractAddress) {
            self.roles.only_token_admin();
            let old_reward_supplier = self.reward_supplier_dispatcher.contract_address.read();
            self.reward_supplier_dispatcher.contract_address.write(reward_supplier);
            self
                .emit(
                    ConfigEvents::RewardSupplierChanged {
                        old_reward_supplier, new_reward_supplier: reward_supplier,
                    },

            ref self: ContractState, rewards: Amount,
        ) {
            assert!(
                get_caller_address() == self.staking_contract.read(),
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,

            self.unclaimed_rewards.write(unclaimed_rewards);

        fn claim_rewards(ref self: ContractState, amount: Amount) {
            // Asserts.
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            let unclaimed_rewards = self.unclaimed_rewards.read();
            assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Update unclaimed_rewards and transfer the requested rewards to the staking contract.
            self.unclaimed_rewards.write(unclaimed_rewards - amount);
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
        }
