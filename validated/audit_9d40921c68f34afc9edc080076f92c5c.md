### Title
Unbounded Recursive Deserialization of `NearPromise::Then` Enables Wasm Stack Overflow in XCC Router — (`File: engine-types/src/parameters/promise.rs`)

---

### Summary

The `NearPromise` type uses a hand-written recursive `BorshDeserialize` implementation with no depth limit. An unprivileged EVM user can craft a deeply-nested `NearPromise::Then { base: Box<NearPromise::Then { ... }> }` payload and submit it via the XCC precompile (`CrossContractCallArgs::Eager` or `Delayed`). When the router contract's `execute` or `execute_scheduled` method processes this payload, the unbounded recursion in `recursive_promise_create` exhausts the Wasm call stack, causing a panic/abort that permanently freezes any NEAR funds attached to the in-flight XCC call.

---

### Finding Description

`NearPromise` is a recursive enum:

```rust
pub enum NearPromise {
    Simple(SimpleNearPromise),
    Then { base: Box<Self>, callback: SimpleNearPromise },
    And(Vec<Self>),
}
```

Its `BorshDeserialize` implementation calls itself recursively without any depth guard:

```rust
0x01 => {
    let base = Self::deserialize_reader(reader)?;   // ← unbounded recursion
    let callback = SimpleNearPromise::deserialize_reader(reader)?;
    Ok(Self::Then { base: Box::new(base), callback })
}
```

A payload of N nested `Then` nodes is encoded as N consecutive `0x01` bytes followed by a terminal `0x00` node. Deserializing it recurses N frames deep on the Wasm stack.

After deserialization, `Router::recursive_promise_create` also recurses over the same structure without a depth limit:

```rust
NearPromise::Then { base, callback } => {
    let base_index = Self::recursive_promise_create(base);  // ← unbounded recursion
    ...
}
```

The XCC precompile accepts `CrossContractCallArgs` (containing a `PromiseArgs::Recursive(NearPromise)`) directly from EVM calldata, with no depth or size validation beyond a byte-cost gas check:

```rust
let args = CrossContractCallArgs::try_from_slice(input)
    .map_err(|_| ExitError::Other(Cow::from(consts::ERR_INVALID_INPUT)))?;
```

The `execute_scheduled` function on the router is callable by **anyone** (not just the parent engine), and it also calls `Self::promise_create(promise)` → `recursive_promise_create` with no depth limit.

---

### Impact Explanation

**High — Temporary (potentially permanent) freezing of funds.**

When a deeply-nested `NearPromise` is submitted via `CrossContractCallArgs::Delayed`, it is stored in the router's `scheduled_promises` map. When `execute_scheduled` is later called, `recursive_promise_create` recurses to the depth of the nesting, overflowing the Wasm stack. The NEAR runtime traps the contract, the transaction fails, and the stored promise is consumed (removed from the map) but never executed. Any NEAR tokens that were attached to the XCC call (withdrawn from the user's wNEAR balance to fund the router) are permanently stranded in the router sub-account with no recovery path, since the only execution path for that nonce is now gone.

For `CrossContractCallArgs::Eager`, the overflow occurs during the same transaction as the EVM call, causing the entire NEAR transaction to abort.

---

### Likelihood Explanation

Any EVM address that has approved the XCC precompile and has wNEAR can trigger this. The attack requires only crafting a Borsh-encoded `CrossContractCallArgs::Delayed(PromiseArgs::Recursive(...))` with sufficient nesting depth (a few hundred `0x01` bytes suffice to overflow a typical Wasm stack frame budget). No privileged access is required. The `execute_scheduled` entry point is explicitly open to all callers.

---

### Recommendation

1. **Add a depth limit during deserialization** of `NearPromise`. Track recursion depth and return an `io::Error` if it exceeds a safe bound (e.g., 64).
2. **Add a depth/count limit in `recursive_promise_create`** in `etc/xcc-router/src/lib.rs` before recursing into `Then` branches.
3. **Validate `promise_count()` in the XCC precompile** before accepting `Recursive` args — reject inputs whose `promise_count()` exceeds a protocol-defined maximum.

---

### Proof of Concept

**Attacker constructs the following Borsh payload** (pseudocode):

```
CrossContractCallArgs::Delayed(
  PromiseArgs::Recursive(
    NearPromise::Then {
      base: NearPromise::Then {
        base: NearPromise::Then {
          ... (500 levels deep) ...
          base: NearPromise::Simple(...),
          callback: SimpleNearPromise::Create(...)
        },
        callback: SimpleNearPromise::Create(...)
      },
      callback: SimpleNearPromise::Create(...)
    }
  )
)
```

Encoded as bytes, the `NearPromise` portion is simply `[0x01, 0x01, 0x01, ...(500 times)..., 0x00, <SimpleNearPromise bytes>, <SimpleNearPromise bytes> * 500]`.

1. Attacker calls the XCC precompile from an EVM contract with this payload as `Delayed`.
2. The engine stores the `PromiseArgs` in the router's `scheduled_promises` at nonce N.
3. Attacker (or anyone) calls `execute_scheduled(nonce: N)` on the router.
4. `Router::promise_create` → `recursive_promise_create` recurses 500 frames deep → Wasm stack overflow → NEAR runtime traps → transaction aborts.
5. The promise at nonce N was already removed from storage (line `self.scheduled_promises.remove(&nonce.0)`) before the trap, so it is permanently lost along with any attached NEAR.

impl BorshDeserialize for NearPromise {
    fn deserialize_reader<R: io::Read>(reader: &mut R) -> io::Result<Self> {
        let variant_byte = {
            let mut buf = [0u8; 1];
            reader.read_exact(&mut buf)?;
            buf[0]
        };
        match variant_byte {
            0x00 => {
                let inner = SimpleNearPromise::deserialize_reader(reader)?;
                Ok(Self::Simple(inner))
            }
            0x01 => {
                let base = Self::deserialize_reader(reader)?;
                let callback = SimpleNearPromise::deserialize_reader(reader)?;
                Ok(Self::Then {
                    base: Box::new(base),
                    callback,
                })
            }
            0x02 => {
                let promises: Vec<Self> = Vec::deserialize_reader(reader)?;
                Ok(Self::And(promises))
            }
            _ => Err(io::Error::new(
                io::ErrorKind::InvalidInput,
                "Invalid variant byte for NearPromise",
            )),
        }
    }
