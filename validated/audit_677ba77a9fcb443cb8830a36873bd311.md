### Title
Unprivileged Caller Can Permanently Freeze Block Rewards by Calling `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` in the Staking contract has no access-control check on the caller. Any unprivileged address can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` sentinel to the current block without distributing any rewards. Because the function enforces a strict "one call per block" invariant, the legitimate caller is permanently locked out for that block and the rewards are irrecoverably lost.

### Finding Description

`update_rewards` is the post-consensus entry point for distributing per-block staking rewards. Its implementation in `StakingRewardsManagerImpl` begins with only two generic guards:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global sentinel updated

    if disable_rewards || self.is_pre_consensus() {
        return;                            // exits WITHOUT distributing rewards
    }
    ...
    self._update_rewards(...);
}
```

`general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero — there is no role check, no `assert_caller_is_attestation_contract()`, and no equivalent of the guard present in the sibling function `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
    self.assert_caller_is_attestation_contract();   // ← guard that is ABSENT in update_rewards
    ...
}
```

The `disable_rewards` boolean is clearly intended to let the attestation contract signal "this staker did not attest; skip reward distribution." Because there is no caller restriction, any address can supply `disable_rewards: true`.

The `last_reward_block` storage variable is **global** (not per-staker). A single call to `update_rewards` with any valid `staker_address` and `disable_rewards: true` advances the sentinel for the entire contract. Every subsequent call in the same block — regardless of which staker it targets — will revert with `REWARDS_ALREADY_UPDATED`.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who front-runs the legitimate per-block reward call with `update_rewards(any_valid_staker, disable_rewards: true)`:

1. Advances `last_reward_block` to the current block.
2. Skips `_update_rewards`, so no staker's `unclaimed_rewards_own` is incremented and no pool receives its share.
3. Blocks every other call to `update_rewards` for the remainder of that block.

Because rewards are computed per-block and the sentinel is never rolled back, the rewards for that block are permanently lost for all stakers and delegators. Repeating this every block eliminates all consensus-era reward distribution.

### Likelihood Explanation

**High.** The entry point is public, requires no tokens, no stake, and no privileged role. The only prerequisite is knowing any active staker address, which is trivially obtained from the public `stakers` vector or on-chain events. A single transaction per block is sufficient to execute the attack indefinitely.

### Recommendation

Add a caller restriction to `update_rewards` analogous to the one in `update_rewards_from_attestation_contract`. The simplest fix is to assert that the caller is the attestation contract:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_attestation_contract();   // add this guard
    ...
}
```

Alternatively, introduce a dedicated `rewards_manager` role and gate the function with `self.roles.only_rewards_manager()`.

### Proof of Concept

1. Deploy the protocol in post-consensus mode (i.e., `consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. From an arbitrary EOA (no stake, no role), call:
   ```
   staking.update_rewards(staker_address: <any_active_staker>, disable_rewards: true)
   ```
3. Observe that `last_reward_block` is now equal to the current block number.
4. The legitimate attestation contract (or any other caller) attempts to call `update_rewards` in the same block → reverts with `REWARDS_ALREADY_UPDATED`.
5. No staker receives rewards for that block. Repeat every block to freeze all future yield. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
