### Title
Unrestricted `update_rewards` with `disable_rewards=true` Allows Any Caller to Permanently Block Per-Block Reward Distribution — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is callable by any unprivileged address. It writes to the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. An attacker can call `update_rewards(valid_staker, disable_rewards: true)` every block to consume the block's single reward slot without distributing any rewards, permanently freezing unclaimed yield for all stakers in the consensus rewards phase.

---

### Finding Description

`update_rewards` is exposed via `IStakingRewardsManager` with no role-based access control. Its only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function body executes in this order:

1. Asserts `current_block_number > last_reward_block` — enforcing one call per block, globally.
2. Validates the supplied `staker_address` is active with non-zero STRK balance.
3. **Writes `current_block_number` into `last_reward_block`** — consuming the block's reward slot.
4. Only then checks `if disable_rewards || self.is_pre_consensus()` and returns early. [2](#0-1) 

The critical ordering flaw: `last_reward_block` is committed to storage at line 1485 **before** the `disable_rewards` guard at line 1487. Because `last_reward_block` is a single global variable (not per-staker), once it is set to block N, every subsequent call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

An attacker can therefore:
1. Obtain any valid, active staker address from on-chain events (public information).
2. Call `update_rewards(valid_staker, disable_rewards: true)` in every block.
3. This passes all validation, sets `last_reward_block = current_block`, and returns without distributing rewards.
4. Any legitimate call to `update_rewards` in the same block reverts.

---

### Impact Explanation

In the consensus rewards phase (V3), `update_rewards` is the sole mechanism by which stakers receive per-block STRK rewards. Blocking it every block permanently freezes unclaimed yield for all stakers. This matches the **High** impact category: *Permanent freezing of unclaimed yield*.

---

### Likelihood Explanation

The attack requires no privilege, no capital, and no special setup — only the gas cost of one Starknet transaction per block. Valid staker addresses are trivially discoverable from `NewStaker` events. The attack can be fully automated and sustained indefinitely at negligible cost.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so that a call with `disable_rewards = true` does not consume the block's reward slot:

```cairo
// Update last block rewards ONLY if rewards will actually be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... proceed with reward calculation
```

Alternatively, restrict who may pass `disable_rewards = true` (e.g., only the staker or their reward address).

---

### Proof of Concept

1. Consensus rewards are active: `current_epoch >= consensus_rewards_first_epoch`.
2. Attacker reads any valid staker address `S` from a `NewStaker` event.
3. **Block N**: Attacker calls `update_rewards(S, disable_rewards: true)`.
   - Passes: `block_number(N) > last_reward_block`.
   - Passes: staker `S` is active with non-zero balance.
   - `last_reward_block` is written to `N`.
   - Function returns early — zero rewards distributed.
4. Legitimate staker calls `update_rewards(S, false)` in block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Attacker repeats step 3 every block. All stakers are permanently denied consensus rewards. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
