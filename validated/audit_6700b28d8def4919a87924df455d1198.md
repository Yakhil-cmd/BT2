### Title
Gateway Stateful Validator Only Checks L2 Gas Price Against Block Threshold, Missing L1 Gas and L1 Data Gas Price Checks, Allowing Admission of Transactions That Will Fail Execution - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` function only validates `l2_gas.max_price_per_unit` against the previous block's L2 gas price. It does not validate `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` against the corresponding block prices. The stateless validator similarly only checks `l2_gas.max_price_per_unit` against a static `min_gas_price` floor. An attacker can submit `AllResources` transactions with valid L2 gas price but zero (or sub-threshold) L1 and L1DataGas prices. These pass all gateway admission checks, enter the mempool, and are only rejected by the blockifier's `check_fee_bounds` during execution — wasting sequencer resources and polluting the mempool with transactions that are guaranteed to fail.

The `ZeroResourceBounds` guard in the stateless validator does not close this gap. `max_possible_fee` is non-zero as long as any single resource has both non-zero amount and non-zero price. A transaction with `l2_gas.max_amount > 0` and `l2_gas.max_price_per_unit > min_gas_price` but `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` passes the zero-fee guard, passes both gateway validators, and is admitted to the mempool.

### Impact Explanation

**High. Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

Any unprivileged user can craft `AllResources` invoke/declare/deploy-account transactions with:
- `l2_gas.max_price_per_unit` ≥ gateway threshold (passes both validators)
- `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0`

These transactions pass the full gateway admission pipeline (`StatelessTransactionValidator::validate` → `StatefulTransactionValidator::extract_state_nonce_and_run_validations`) and are inserted into the mempool. When the batcher dequeues them, the blockifier's `perform_pre_validation_stage` → `check_fee_bounds` rejects them with `InsufficientResourceBounds { MaxGasPriceTooLow }`. The sequencer expends CPU and I/O resources processing transactions that are guaranteed to fail, and the mempool is polluted with invalid entries that displace legitimate transactions.

### Likelihood Explanation

Trivially reachable by any external caller. No privileged access, special account state, or race condition is required. The attacker only needs to set two fields to zero in an otherwise valid transaction. The gateway's `validate_resource_bounds` flag is `true` by default in production configuration, but the check it enables covers only L2 gas price.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or rename it) to also validate `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the corresponding previous-block prices (multiplied by `min_gas_price_percentage`), mirroring the complete three-resource check already performed by the blockifier's `check_fee_bounds`. The TODO comment in the production code already flags this as a known gap.

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with:
   - `resource_bounds.l2_gas.max_price_per_unit` = current L2 gas price (passes stateless `min_gas_price` check and stateful threshold check)
   - `resource_bounds.l2_gas.max_amount` > 0 (passes `ZeroResourceBounds` check)
   - `resource_bounds.l1_gas.max_price_per_unit` = `GasPrice(0)`
   - `resource_bounds.l1_data_gas.max_price_per_unit` = `GasPrice(0)`
   - Valid nonce, signature, calldata
2. Submit to the gateway's `add_tx` endpoint.
3. Observe: `StatelessTransactionValidator::validate_resource_bounds` passes (only L2 price checked). [1](#0-0) 
4. Observe: `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` passes (only L2 price checked). [2](#0-1) 
5. Transaction is admitted to the mempool.
6. When the batcher executes it, `check_fee_bounds` in `perform_pre_validation_stage` rejects it with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas, ... } }`. [3](#0-2)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,

        }

        Ok(())

#[derive(Clone, Debug, Serialize, Deserialize, Validate, PartialEq)]
pub struct StatefulTransactionValidatorConfig {
    // If true, ensures the max L2 gas price exceeds (a configurable percentage of) the base gas
    // price of the previous block.
    pub validate_resource_bounds: bool,
    pub max_allowed_nonce_gap: u32,
    pub reject_future_declare_txs: bool,
    pub max_nonce_for_validation_skip: Nonce,
    pub versioned_constants_overrides: Option<VersionedConstantsOverrides>,
    // Minimum gas price as percentage of threshold to accept transactions.
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}

impl Default for StatefulTransactionValidatorConfig {
    fn default() -> Self {
        StatefulTransactionValidatorConfig {
            validate_resource_bounds: true,
            max_allowed_nonce_gap: 200,
            reject_future_declare_txs: true,
            max_nonce_for_validation_skip: Nonce(Felt::ONE),
            min_gas_price_percentage: 100,
            versioned_constants_overrides: None,
        }
    }
}
