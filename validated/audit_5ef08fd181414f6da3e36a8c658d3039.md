### Title
Interop Root Event Hook Does Not Enforce the Service-Transaction Whitelist at the Point of Root Ingestion — (`system_hooks/src/event_hooks/interop_root_reporter.rs`)

---

### Summary

The `SERVICE_DESTINATION_WHITELIST` in ZKsync OS defines the only permitted `(address, selector)` pairs that may trigger interop-root imports. This whitelist is enforced at service-transaction parse time, but the `interop_root_reporter_event_hook` — the function that actually writes an `InteropRoot` into the system — performs **no caller check whatsoever**. Any EVM transaction that causes `L2_INTEROP_ROOT_STORAGE_ADDRESS` to emit `InteropRootAdded` will have its root unconditionally accepted, bypassing the whitelist entirely.

---

### Finding Description

**Whitelist definition (enforced only at parse time)**

`SERVICE_DESTINATION_WHITELIST` is a compile-time constant that lists every `(B160, [u8;4])` pair a service transaction is allowed to target: [1](#0-0) 

The check is applied inside `ServiceTx::decode_list_body` — i.e., only when the bootloader parses a type-`0x7d` transaction: [2](#0-1) 

**Event hook — no caller validation**

The `interop_root_reporter_event_hook` is registered for every event emitted from `L2_INTEROP_ROOT_STORAGE_ADDRESS`: [3](#0-2) 

When the hook fires it validates only the event signature, ABI offset, array length, and topic count. It never checks who called the contract, never verifies the call originated from a service transaction, and never validates that `chain_id != 0` or `root != 0` — both of which are documented invariants of `InteropRoot`: [4](#0-3) 

The `_caller_ee` parameter is explicitly ignored (prefixed `_`). The `add_interop_root` IO method also performs no field validation: [5](#0-4) 

The `InteropRoot` struct documents the invariants but does not enforce them: [6](#0-5) 

**The gap**

A regular (signed) L2 transaction can call `L2_INTEROP_ROOT_STORAGE_ADDRESS` directly — the whitelist only blocks service-transaction type `0x7d`. If the Solidity contract deployed at that address does not independently enforce an operator-only guard, any user can call `addInteropRootsInBatch`, cause the contract to emit `InteropRootAdded`, and have the hook unconditionally ingest the attacker-supplied `(chain_id, block_or_batch_number, root)` tuple. The whitelist that was supposed to gate this path is never consulted by the hook.

---

### Impact Explanation

The accepted roots are folded into `interop_roots_rolling_hash`, which is committed on-chain as part of `BatchOutput`: [7](#0-6) 

An attacker who can inject arbitrary `(chain_id, block_number, root)` tuples can:

1. **Corrupt the rolling hash** — causing the batch commitment to diverge from the honest value, potentially invalidating the batch on the settlement layer (denial of service / forced re-execution).
2. **Forge cross-chain message proofs** — by injecting a root that corresponds to a Merkle tree the attacker controls, enabling fraudulent cross-chain asset claims against any bridge that trusts the committed interop roots.

---

### Likelihood Explanation

The attack requires that the Solidity contract at `L2_INTEROP_ROOT_STORAGE_ADDRESS` lacks an operator-only guard on `addInteropRootsInBatch`. The Rust layer provides **zero defense in depth**: the hook fires unconditionally for any matching event from that address. Because the Rust code is the authoritative enforcement layer for ZKsync OS security properties, the absence of a caller check here is a structural gap regardless of what the Solidity contract does. The entry path (a normal signed L2 transaction) is fully unprivileged.

---

### Recommendation

Add a caller-origin check inside `interop_root_reporter_event_hook`. The hook already receives `_caller_ee`; the system should also expose the immediate caller address so the hook can assert it equals the expected system contract, or alternatively gate the hook on a flag set only during service-block execution. Additionally, enforce the documented `InteropRoot` invariants (`chain_id != 0`, `root != Bytes32::ZERO`) directly in the hook before calling `add_interop_root`, independent of the Solidity contract.

---

### Proof of Concept

1. Deploy a normal L2 transaction targeting `L2_INTEROP_ROOT_STORAGE_ADDRESS` with selector `0xcca2f7bc` (`addInteropRootsInBatch`) and a crafted payload containing an attacker-chosen `(chain_id, block_number, root)`.
2. If the Solidity contract at that address has no `onlyOperator` guard, it processes the call and emits `InteropRootAdded(chain_id, block_number, [root])`.
3. `try_intercept_event` matches `L2_INTEROP_ROOT_STORAGE_ADDRESS_LOW` and dispatches to `interop_root_reporter_event_hook`.
4. The hook passes all format checks (signature matches, `data.len() == 96`, `offset == 32`, `len == 1`, `topics.len() == 3`) and calls `system.io.add_interop_root(...)` with the attacker-supplied values — no whitelist check occurs.
5. The attacker-controlled root is now included in `interop_roots_rolling_hash` for the batch, corrupting the settlement-layer commitment.

/// Pairs (destination, selector) that service transactions are allowed

const SERVICE_DESTINATION_WHITELIST: &[(B160, [u8; 4])] = &[
    (
        L2_INTEROP_ROOT_STORAGE_ADDRESS,
        ADD_INTEROP_ROOTS_IN_BATCH_SELECTOR,
    ),
    (SYSTEM_CONTEXT_ADDRESS, SET_SL_CHAIN_ID_SELECTOR),
    (L2_INTEROP_CENTER_ADDRESS, SET_INTEROP_FEE_SELECTOR),
];

fn whitelisted(to: B160, data: &[u8]) -> bool {
    let selector: [u8; 4] = match data.get(..4).and_then(|bytes| bytes.try_into().ok()) {
        Some(selector) => selector,
        None => return false,
    };
    SERVICE_DESTINATION_WHITELIST.contains(&(to, selector))
}

impl<'a> RlpListDecode<'a> for ServiceTx<'a> {
    /// Decode the 2-field list body:
    /// [destination, data]
    fn decode_list_body(r: &mut Rlp<'a>) -> Result<Self, InvalidTransaction> {
        let to_slice = r.bytes()?;
        if to_slice.len() != 20 {
            return Err(InvalidTransaction::InvalidStructure);
        }
        let to: &'a [u8; 20] = to_slice
            .try_into()
            .map_err(|_| InvalidTransaction::InvalidStructure)?;

        let to_b160 = B160::from_be_bytes(*to);

        let data = r.bytes()?;
        // Validate whitelist
        if !whitelisted(to_b160, data) {
            return Err(InvalidTransaction::InvalidStructure);
        }
        let salt = r.u64()?;
        Ok(Self { to, data, salt })
    }
}

pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<<S::IOTypes as SystemIOTypesConfig>::EventKey, MAX_EVENT_TOPICS>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
where
{
    // First, ensure we're capturing the InteropRootAdded event
    if topics.is_empty() || topics[0].as_u8_array() != INTEROP_ROOT_ADDED_EVENT_SIG {
        return Ok(());
    }
    // Internal error if the data supplied doesn't match the expected value
    if data.len() != 96 {
        return Err(internal_error!("Interop root reporter event hook received bad data").into());
    }

    // Parse data
    let offset: u32 = match U256::from_be_slice(&data[..32]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad offset").into(),
            );
        }
    };
    // This event is part of the system, but we check it anyways
    if offset != 32 {
        return Err(internal_error!("Interop root reporter event hook received bad offset").into());
    }

    let len: u32 = match U256::from_be_slice(&data[32..64]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad length").into(),
            );
        }
    };
    // It should have exactly one side
    if len != 1 {
        return Err(internal_error!("Interop root reporter event hook received bad length").into());
    }
    // Validate topics length
    if topics.len() != 3 {
        return Err(internal_error!("Interop root reporter event hook received bad topics").into());
    }

    let root = Bytes32::from_array(data[64..96].try_into().unwrap());
    let chain_id = U256::from_be_bytes(topics[1].as_u8_array());
    let block_or_batch_number = U256::from_be_bytes(topics[2].as_u8_array());
    system.io.add_interop_root(
        ExecutionEnvironmentType::NoEE,
        resources,
        InteropRoot {
            root,
            block_or_batch_number,
            chain_id,
        },
    )?;

    Ok(())
}

impl BatchOutput {
    ///
    /// Calculate keccak256 hash of public input
    ///
    pub fn hash(&self) -> [u8; 32] {
        let mut hasher = Keccak256::new();
        hasher.update(self.chain_id.to_be_bytes::<32>());
        hasher.update(&self.first_block_timestamp.to_be_bytes());
        hasher.update(&self.last_block_timestamp.to_be_bytes());
        // Encode DA commitment scheme as U256 BE
        hasher.update([0u8; 31]);
        hasher.update([self.da_commitment_scheme as u8]);
        hasher.update(self.pubdata_commitment.as_u8_ref());
        hasher.update(self.number_of_layer_1_txs.to_be_bytes::<32>());
        hasher.update(self.number_of_layer_2_txs.to_be_bytes::<32>());
        hasher.update(self.priority_operations_hash.as_u8_ref());

    }
