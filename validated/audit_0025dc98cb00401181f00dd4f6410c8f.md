### Title
Gateway Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` for Resource-Bounds Threshold Check — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful admission path reads `gas_prices.strk_gas_prices.l2_gas_price` from the latest committed block to validate a transaction's L2 gas `max_price_per_unit`. That field records the price that was used **for** the already-committed block, not the price that will govern the **next** block. The correct field is `next_l2_gas_price`, which is stored in the same block header and is what the batcher will actually enforce during execution. The developer-authored TODO comment at the exact line of the bug confirms the intent: `// TODO(Arni): getnext_l2_gas_price from the block header.`

---

### Finding Description

In `validate_resource_bounds`, the gateway fetches block info from the latest committed block and extracts `l2_gas_price`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;          // ← price used FOR the committed block
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`validate_tx_l2_gas_price_within_threshold` then computes a threshold as `min_gas_price_percentage / 100 * previous_block_l2_gas_price` and rejects any V3 (`AllResources`) transaction whose `l2_gas.max_price_per_unit` falls below it:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(StarknetError { code: "GAS_PRICE_TOO_LOW", ... });
}
``` [2](#0-1) 

The block header that is committed to storage contains **two** distinct L2 gas price fields:

| Field | Meaning |
|---|---|
| `l2_gas_price` | Price that governed the block just committed |
| `next_l2_gas_price` | EIP-1559-derived price that will govern the **next** block |

The batcher builds the next block using `self.l2_gas_price`, which is updated via `update_l2_gas_price` → `calculate_next_l2_gas_price_for_fin` after each decision. That value equals `next_l2_gas_price` from the committed block header, **not** `l2_gas_price`. [3](#0-2) [4](#0-3) 

The block header written by `update_state_sync_with_new_block` stores both fields separately:

```rust
let block_header_without_hash = BlockHeaderWithoutHash {
    l2_gas_price,                          // price used for THIS block
    next_l2_gas_price: self.l2_gas_price,  // price for the NEXT block
    ...
};
``` [5](#0-4) 

Because the gateway reads `l2_gas_price` instead of `next_l2_gas_price`, the threshold it enforces is anchored to the wrong epoch.

---

### Impact Explanation

Two divergent failure modes arise, both reachable by any unprivileged user submitting a V3 transaction:

**Mode A — Valid transaction rejected (price fell between blocks)**

When the EIP-1559 mechanism decreases the L2 gas price from block N to block N+1 (e.g., because block N was under-full), a user who sets `max_price_per_unit` to a value that satisfies `max_price >= next_l2_gas_price` (valid for execution) but `max_price < threshold * l2_gas_price` (fails the stale threshold) will have their transaction rejected at the gateway with `GAS_PRICE_TOO_LOW`. The transaction is permanently denied admission even though the batcher would have accepted and executed it. This matches the allowed impact: **"Mempool/gateway/RPC admission rejects valid transactions before sequencing."**

**Mode B — Invalid transaction admitted (price rose between blocks)**

When the L2 gas price increases from block N to block N+1 (e.g., because block N was over-full), a user who sets `max_price_per_unit` to a value that satisfies `max_price >= threshold * l2_gas_price` (passes the stale threshold) but `max_price < next_l2_gas_price` (insufficient for execution) will be admitted to the mempool. The batcher's `check_fee_bounds` in `perform_pre_validation_stage` will then reject the transaction during block building, wasting mempool capacity and causing the user's transaction to silently fail after admission. This matches: **"Mempool/gateway/RPC admission accepts invalid transactions."** [6](#0-5) 

---

### Likelihood Explanation

- The L2 gas price changes **every block** via the EIP-1559 mechanism in `calculate_next_base_gas_price`. The price can move up or down by up to `price / gas_price_max_change_denominator` per block.
- The production deployment config sets `min_gas_price_percentage` to a non-trivial value (the config schema and deployment JSON both expose this field), meaning the threshold is a meaningful fraction of the stale price.
- Any block whose gas usage deviates from the target (which is the common case) will produce a `next_l2_gas_price` that differs from `l2_gas_price`, triggering the mismatch.
- The bug is triggered by any V3 (`AllResources`) transaction submitted during normal operation; no special privileges or adversarial setup are required. [7](#0-6) 

---

### Recommendation

Replace the read of `l2_gas_price` with `next_l2_gas_price` from the block header, exactly as the TODO comment instructs:

```rust
// Before (wrong):
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// After (correct):
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .next_l2_gas_price;   // the price the batcher will actually enforce
```

The `GatewayFixedBlockStateReader::get_block_info` return type (`BlockInfo`) must expose `next_l2_gas_price` if it does not already. Rename the local variable to `next_block_l2_gas_price` to make the intent explicit and prevent future confusion.

---

### Proof of Concept

1. Observe the latest committed block has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block N was over-full).
2. Submit a V3 transaction with `l2_gas.max_price_per_unit = X` where `threshold * P <= X < P'` (e.g., `min_gas_price_percentage = 80`, `P = 100`, `P' = 115`, `X = 90`).
3. The gateway computes `threshold = 80` and admits the transaction (`90 >= 80`).
4. The batcher builds block N+1 with `l2_gas_price = P' = 115`.
5. `check_fee_bounds` in `perform_pre_validation_stage` finds `max_price (90) < block_price (115)` and rejects the transaction.
6. The admitted transaction never executes despite passing gateway validation.

Conversely, set `P' < P` (block N was under-full, e.g., `P = 100`, `P' = 85`, `min_gas_price_percentage = 90`, `threshold = 90`, `X = 87`):
- Gateway rejects (`87 < 90`) even though `87 >= 85 = P'` would satisfy the batcher. [1](#0-0) [2](#0-1)

                // No validation required for legacy transactions.
            }
        }

pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
