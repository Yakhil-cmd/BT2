### Title
Missing Zero `msg_hash` Validation in OT-Based ECDSA `sign` Allows Signing Attacker-Chosen Zero Message — (`src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

The `sign` function in the OT-based ECDSA module accepts `msg_hash = 0` without any validation, while the robust ECDSA `sign` function explicitly rejects it. A malicious coordinator or caller can pass a zero scalar as the message hash, causing all participants to produce signature shares that reduce to `s_i = r · σ_i`, yielding a valid threshold ECDSA signature on the zero message. This reveals the product `k · x` (nonce × private key) in the aggregated signature and, when combined with presignature reuse, enables full private key extraction.

---

### Finding Description

**Root cause — missing input guard in OT-based ECDSA `sign`:**

The robust ECDSA `sign` function explicitly rejects a zero message hash:

```rust
// src/ecdsa/robust_ecdsa/sign.rs  lines 91-95
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
    ));
}
``` [1](#0-0) 

The OT-based ECDSA `sign` function has **no equivalent check**:

```rust
// src/ecdsa/ot_based_ecdsa/sign.rs  lines 22-76
pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,          // ← zero is accepted silently
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    // ... participant-count checks only; no msg_hash == 0 guard ...
``` [2](#0-1) 

The function's own doc-comment acknowledges the risk but does not enforce it:

> **WARNING** You must absolutely hash an actual message before passing it to this function. Allowing the signing of arbitrary scalars *is* a security risk, and this function only tolerates this risk to allow for genericity. [3](#0-2) 

**Algebraic consequence of `msg_hash = 0`:**

Each participant's signature share is computed as:

```rust
// src/ecdsa/ot_based_ecdsa/sign.rs  lines 155-158
let r = x_coordinate(&presignature.big_r);
Ok(msg_hash * k_i + r * sigma_i)
``` [4](#0-3) 

When `msg_hash = 0`, every participant's share collapses to `s_i = r · σ_i`. The coordinator aggregates these into `s = r · σ = r · k · x`, which is a **valid ECDSA signature** on the zero message. The aggregated `s` directly encodes `k · x = s / r` — the product of the secret nonce and the private key — in a publicly observable value.

**Key-extraction path (with presignature reuse):**

If the same presignature is reused for a second signing session on a known message `h′ ≠ 0` (a risk already documented in the codebase), an attacker who obtained the zero-message signature `(R, s)` and the second signature `(R, s′)` can solve:

```
s  = r · k · x          (h = 0)
s′ = h′ · k + r · k · x (h′ ≠ 0, same presignature)

→ k  = (s′ − s) / h′
→ x  = s / (r · k)
```

This yields the aggregate private key `x` from two signatures and public values alone.

---

### Impact Explanation

**Direct (Critical — unauthorized threshold signature on attacker-chosen input):** A malicious coordinator or any caller that controls the `msg_hash` argument can pass `Scalar::ZERO` and obtain a valid, verifiable threshold ECDSA signature on the zero message without any protocol error. The `Signature::verify` function confirms the signature is valid:

```rust
// src/ecdsa/mod.rs  lines 65-79
pub fn verify(&self, public_key: &AffinePoint, msg_hash: &Scalar) -> bool {
    let r: Scalar = x_coordinate(&self.big_r);
    if r.is_zero().into() || self.s.is_zero().into() { return false; }
    ...
}
``` [5](#0-4) 

Verification passes for `h = 0` because `u₁ = h · s⁻¹ = 0`, `u₂ = r · s⁻¹`, and `u₁G + u₂X = G/k = R`.

**Indirect (Critical — private key extraction):** Combined with presignature reuse — a risk the codebase already documents — the zero-message signature leaks `k · x`, enabling full private key recovery as shown above.

---

### Likelihood Explanation

The OT-based ECDSA `sign` function is a public API entry point. Any caller — including a malicious coordinator, a compromised participant node, or a malicious API/RPC client — can supply `msg_hash = Scalar::ZERO`. In automated MPC deployments where participants do not independently verify the semantic content of the message hash before calling `sign`, a single malicious coordinator can direct all participants to sign the zero hash. No privileged key material or cryptographic break is required; only the ability to call the public `sign` function with a crafted scalar.

---

### Recommendation

Add the same zero-hash guard that the robust ECDSA variant already enforces, immediately after the participant-count checks in `src/ecdsa/ot_based_ecdsa/sign.rs`:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0".to_string(),
    ));
}
```

This mirrors the existing check in `src/ecdsa/robust_ecdsa/sign.rs` at lines 91–95 and closes the algebraic attack surface without any protocol change.

### Proof of Concept

```rust
// Attacker-controlled call — no error is returned
let protocol = ot_based_ecdsa::sign::sign(
    &participants,
    coordinator,
    threshold,
    me,
    public_key,
    rerandomized_presignature,
    Scalar::ZERO,   // ← zero message hash, accepted silently
).unwrap();

// Protocol runs; coordinator receives s_i = r * sigma_i from every participant.
// Coordinator aggregates: s = r * sigma = r * k * x.
// sig.verify(&public_key, &Scalar::ZERO) returns true.
// k*x = s / r is now public.
```

The robust ECDSA variant rejects the identical call at initialization with `"msg_hash cannot be 0 to avoid potential split view attacks"`. The OT-based variant does not, making it the only reachable path for this attack. [2](#0-1) [1](#0-0)

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

    pub fn verify(&self, public_key: &AffinePoint, msg_hash: &Scalar) -> bool {
        let r: Scalar = x_coordinate(&self.big_r);
        if r.is_zero().into() || self.s.is_zero().into() {
            return false;
        }
        // Check if s has been normalized
        if self.s.is_high().into() {
            return false;
        }
        // tested earlier is not zero, so inversion will not raise an error and unwrap cannot panic
        let s_inv = self.s.invert_vartime().unwrap();
        let reproduced = (ProjectivePoint::GENERATOR * (*msg_hash * s_inv))
            + (ProjectivePoint::from(*public_key) * (r * s_inv));
        x_coordinate(&reproduced.into()) == r
    }
