### Title
Missing Caller Restriction on `update_rewards` Allows Griefing of Consensus Reward Distribution — (`File: src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no such caller check. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block reward slot and permanently preventing the sequencer from distributing consensus rewards for that block.

---

### Finding Description

The protocol specification explicitly states:

> **`update_rewards` — Access control: Only starkware sequencer.** [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` performs no caller identity check. The only guard is a block-number deduplication check: [2](#0-1) 

After that check passes, `last_reward_block` is unconditionally written to the current block number: [3](#0-2) 

The test suite confirms the absence of any caller restriction — `update_rewards` is invoked throughout tests with no `cheat_caller_address` setup for a sequencer address: [4](#0-3) 

The `general_prerequisites()` call only checks the paused flag; it does not enforce a sequencer identity.

---

### Impact Explanation

**Medium — Griefing with no profit motive but damage to users or protocol.**

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at block N before the sequencer does. This:

1. Sets the global `last_reward_block` to N.
2. Distributes **zero** rewards (because `disable_rewards = true`).
3. Causes every subsequent call to `update_rewards` at block N to revert with `REWARDS_ALREADY_UPDATED`.

The sequencer cannot retry for block N. All stakers permanently lose the consensus rewards that should have been distributed at block N. The attacker gains nothing but inflicts a measurable, irreversible loss of yield on all stakers. [5](#0-4) 

---

### Likelihood Explanation

**Low-to-Medium.** On Starknet, the sequencer controls transaction ordering and can place its own `update_rewards` call first in every block. However:

- The function is fully public with no on-chain enforcement of the sequencer restriction.
- Any L2 account can submit the transaction; the sequencer must actively front-run every block to prevent the attack.
- A sequencer under load, or a future decentralized sequencer model, could fail to consistently win the race.
- The attack requires only a valid staker address (publicly readable from events) and a small gas fee.

---

### Recommendation

Add an explicit caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured sequencer address, analogous to the existing `assert_caller_is_attestation_contract` pattern: [6](#0-5) 

Store the sequencer address in contract storage (set during initialization or by a governance role) and assert it in `update_rewards` before any state mutation.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. A valid staker `S` exists with non-zero balance.
3. At block N, attacker submits: `update_rewards(staker_address: S, disable_rewards: true)`.
4. The call passes all checks: contract is unpaused, `current_block_number (N) > last_reward_block`, staker exists and is active, staker has non-zero balance.
5. `last_reward_block` is written to N; no rewards are distributed.
6. The sequencer submits `update_rewards(staker_address: S, disable_rewards: false)` at block N → reverts with `REWARDS_ALREADY_UPDATED`.
7. Block N's rewards are permanently lost for all stakers. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
