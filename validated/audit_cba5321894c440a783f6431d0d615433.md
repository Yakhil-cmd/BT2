Audit Report

## Title
Canister HTTP Outcall Queue Monopolization via Missing Per-Canister Slot Fairness - (File: `rs/execution_environment/src/execution_environment.rs`)

## Summary
`try_add_http_context_to_replicated_state` enforces only a single global cap of 3000 in-flight HTTP outcall slots with no per-canister sub-limit. A single canister with sufficient cycles can occupy all 3000 slots by targeting unreachable URLs, blocking every other canister on the subnet from issuing HTTP outcalls for up to 60 seconds per cycle. The attack is repeatable and can be sustained indefinitely.

## Finding Description
The sole admission gate in `try_add_http_context_to_replicated_state` checks only the global queue length against `max_canister_http_requests_in_flight` (3000): [1](#0-0) 

No per-canister counter, quota, or fairness check exists anywhere in the execution environment HTTP outcall path. The grep search for `canister_http.*per_canister`, `per_canister.*http`, and `http_requests_per_canister` returns zero matches in `rs/execution_environment/`.

The global constant and its rationale: [2](#0-1) 

The consensus-layer timeout that governs slot retention: [3](#0-2) 

Under legacy pricing, the full fee is deducted at submission and `LegacyTracker::create_payment_receipt` returns a default (zero) receipt, meaning no cycles are ever refunded on timeout: [4](#0-3) [5](#0-4) 

**Attack path:**
1. Deploy a canister on the target application subnet.
2. Submit 3000 `HttpRequest` calls targeting an unreachable address (e.g., `https://[40d:40d:40d:40d:40d:40d:40d:40d]:28992`), each with the minimum legacy fee.
3. `canister_http_request_contexts` reaches length 3000; every subsequent `HttpRequest` from any canister is rejected with `CanisterRejectedMessage: "max number (3000) of http requests in-flight reached."`.
4. After 60 seconds the slots expire. The attacker re-submits ~3000 requests to maintain saturation (~50 req/s steady-state).

## Impact Explanation
All canisters on the targeted subnet lose the ability to make HTTP outcalls for the duration of the attack. Applications relying on HTTPS outcalls — price feeds, cross-chain bridges, oracle integrations — are completely disrupted. This is a concrete application/platform-level DoS of a core protocol feature affecting the entire subnet, not a raw volumetric DDoS. It matches the allowed High impact: *"Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS."* Severity: **High ($2,000–$10,000)**.

## Likelihood Explanation
Any canister on the subnet can trigger this without privileged access. The economic cost (tens of trillions of cycles on a 13-node subnet) is non-trivial but fixed and predictable, making it feasible for a motivated attacker. No external dependency, governance majority, or node compromise is required. The attack is repeatable and can be sustained indefinitely at steady-state cost.

## Recommendation
1. **Introduce a per-canister in-flight limit** enforced inside `try_add_http_context_to_replicated_state`, analogous to the `canister_guaranteed_callback_quota` pattern used for inter-canister callbacks.
2. **Charge a non-refundable slot-holding fee** proportional to the time the slot is held (not just transmission cost) to raise the economic cost of sustained saturation.
3. Consider a **per-canister sliding-window rate limit** on HTTP outcall submissions to prevent burst saturation.

## Proof of Concept
The existing test `http_request_bound_holds` directly confirms that a single caller can fill the entire global cap and that subsequent requests from any caller are rejected: [6](#0-5) 

A minimal reproduction: configure `with_max_canister_http_requests_in_flight(10)`, inject 10 requests from canister A (filling the cap), then inject 1 request from canister B and assert it is rejected — demonstrating that canister B is blocked by canister A's monopolization with no per-canister fairness.

#[test]
fn http_request_bound_holds() {
    let own_subnet = subnet_test_id(1);
    let caller_canister = canister_test_id(10);
    let mut test = ExecutionTestBuilder::new()
        .with_own_subnet_id(own_subnet)
        .with_caller(own_subnet, caller_canister)
        // set number of max in-flight calls to 10
        .with_max_canister_http_requests_in_flight(10)
        .build();
    test.state_mut().metadata.own_subnet_features.http_requests = true;

    // Create payload of the request.
    let url = "https://".to_string();
    let response_size_limit = 1000_u64;
    let transform_method_name = "transform".to_string();
    let transform_context = vec![0, 1, 2];
    let args = CanisterHttpRequestArgs {
        url: url.clone(),
        max_response_bytes: Some(response_size_limit),
        headers: BoundedHttpHeaders::new(vec![]),
        body: None,
        method: HttpMethod::GET,
        transform: Some(TransformContext {
            function: TransformFunc(candid::Func {
                principal: caller_canister.get().0,
                method: transform_method_name.clone(),
            }),
            context: transform_context.clone(),
        }),
        is_replicated: None,
        pricing_version: None,
    };

    // Create request to HTTP_REQUEST method.
    let payload = args.clone().encode();
    test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    test.execute_all();
    // Check that the SubnetCallContextManager contains the request.
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;
    assert_eq!(canister_http_request_contexts.len(), 1);

    // Now we try to inject more than the maximum number of requests we allow to be in-flight

    for _ in 0..15 {
        let payload = args.clone().encode();
        test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    }

    test.execute_all();
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;

    // Check that the SubnetCallContextManager contains the maximum number but not more
    assert_eq!(canister_http_request_contexts.len(), 10);
}
