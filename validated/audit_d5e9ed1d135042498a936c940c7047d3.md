### Title
`validateCOAOwnershipProof` Uses Chain-Agnostic Domain Tag Enabling Cross-Chain and Cross-Address Signature Replay — (`File: fvm/evm/stdlib/contract.cdc`)

---

### Summary

`EVM.validateCOAOwnershipProof` verifies COA ownership proofs using the fixed domain separation tag `"FLOW-V0.0-user"`, which does not encode the Flow chain ID or the target EVM address. This allows a valid COA ownership proof signature produced on one Flow network (e.g., testnet) to be replayed on another (e.g., mainnet), and allows a signature produced for one COA to be replayed against a different COA owned by the same Cadence account. The function is `access(all)` and callable by any unprivileged Cadence script or transaction, and is also reachable via the Cadence Arch EVM precompile from any EVM contract.

---

### Finding Description

`EVM.validateCOAOwnershipProof` verifies that a set of signatures over `signedData` are valid against the account keys at `address`, then checks that the COA at `path` has EVM address `evmAddress`. The cryptographic verification uses a hardcoded domain separation tag:

```cadence
let isValid = keyList.verify(
    signatureSet: signatureSet,
    signedData: signedData,
    domainSeparationTag: "FLOW-V0.0-user"
)
``` [1](#0-0) 

The tag `"FLOW-V0.0-user"` is identical across all Flow networks (mainnet, testnet, sandboxnet, emulator, etc.) and does not encode:
1. The Flow chain ID (no network binding)
2. The `evmAddress` parameter (no COA binding)

The code itself acknowledges the second issue in a comment:

> "Note: this function does not enforce that `signedData` includes `evmAddress`. In principle, a signature produced for one purpose could be replayed here against a different COA owned by the same Cadence account." [2](#0-1) 

The function signature accepts fully attacker-controlled inputs:

```cadence
access(all)
fun validateCOAOwnershipProof(
    address: Address,
    path: PublicPath,
    signedData: [UInt8],
    keyIndices: [UInt64],
    signatures: [[UInt8]],
    evmAddress: [UInt8; 20]
): ValidationResult
``` [3](#0-2) 

COA EVM addresses are derived from a UUID using `MakeCOAAddress(uuid)`, which applies a fixed multiplier constant with no chain-specific component:

```go
func MakeCOAAddress(index uint64) types.Address {
    return makePrefixedAddress(mapAddressIndex(index), types.FlowEVMCOAAddressPrefix)
}
func mapAddressIndex(index uint64) uint64 {
    return uint64(index * addressIndexMultiplierConstant)
}
``` [4](#0-3) [5](#0-4) 

This means a COA created with UUID `N` on testnet and a COA created with UUID `N` on mainnet share the same EVM address. A user who creates their first COA on both networks will have the same COA EVM address on both.

The COA's ERC-1271 `isValidSignature` implementation calls the precompile directly:

```solidity
function isValidSignature(bytes32 _hash, bytes memory _sig)
    external view virtual returns (bytes4) {
    (bool ok, bytes memory data) = cadenceArch.staticcall(
        abi.encodeWithSignature(
            "verifyCOAOwnershipProof(address,bytes32,bytes)",
            address(this), _hash, _sig));
    ...
}
``` [6](#0-5) 

The precompile routes to `validateCOAOwnershipProof` via `coaOwnershipProofValidator`: [7](#0-6)

### Impact Explanation

**Cross-chain replay**: A user who registers the same key pair on both testnet and mainnet (common practice) and creates a COA on both networks will have the same COA EVM address on both (UUID-derived, chain-agnostic). A signature over arbitrary `signedData` produced on testnet with tag `"FLOW-V0.0-user"` is cryptographically identical to a valid signature on mainnet. An attacker who observes a testnet COA ownership proof can replay it on mainnet by calling `EVM.validateCOAOwnershipProof` (or triggering `isValidSignature` on the COA contract) with the mainnet Flow address, the same `signedData`, and the same signatures. Any EVM contract on mainnet using ERC-1271 via the COA's `isValidSignature` would accept this replayed proof, bypassing authorization.

**Cross-address replay**: If a Cadence account holds multiple COAs (at different capability paths), a signature produced for one COA's `signedData` can be replayed against a different COA owned by the same account, since the signed bytes do not encode which COA they are intended for.

---

### Likelihood Explanation

The entry path is fully unprivileged: any Cadence script can call `EVM.validateCOAOwnershipProof` with attacker-controlled arguments, and any EVM contract can call `verifyCOAOwnershipProof` on the Cadence Arch precompile. The cross-chain replay requires only that the victim has the same key registered on both networks and has created a COA on both — a realistic scenario for developers and power users who operate on both testnet and mainnet. The cross-address replay requires only that the victim has more than one COA, which is a supported and documented use case.

---

### Recommendation

1. **Bind the domain tag to the chain ID**: Replace the static `"FLOW-V0.0-user"` tag with a chain-specific tag that encodes the Flow chain ID, e.g., `"FLOW-V0.0-user-flow-mainnet"`. This prevents cross-chain replay.

2. **Bind `signedData` to `evmAddress`**: Enforce (or document as a protocol requirement) that `signedData` must encode the `evmAddress` parameter. The function should verify that the first 20 bytes of `signedData` match `evmAddress`, or the protocol should define a canonical signed payload format that includes the EVM address.

3. **Enforce binding in the precompile**: The `coaOwnershipProofValidator` in `fvm/evm/handler/precompiles.go` constructs the `COAOwnershipProofInContext` from caller-supplied data. The precompile should construct `signedData` to canonically include the chain ID and EVM address before passing to `validateCOAOwnershipProof`.

---

### Proof of Concept

**Cross-chain replay**:

1. Alice registers key `K` on both testnet (Flow address `A_test`) and mainnet (Flow address `A_main`).
2. Alice creates her first COA on testnet (UUID = 1, EVM address = `MakeCOAAddress(1)` = `E`).
3. Alice creates her first COA on mainnet (UUID = 1, EVM address = `MakeCOAAddress(1)` = `E` — same address).
4. An application on testnet asks Alice to sign `signedData = bytes("authenticate:app.example.com")` to prove COA ownership. Alice signs with key `K` under tag `"FLOW-V0.0-user"`, producing signature `S`.
5. An attacker calls on mainnet:
   ```cadence
   EVM.validateCOAOwnershipProof(
       address: A_main,
       path: /public/coa,
       signedData: bytes("authenticate:app.example.com"),
       keyIndices: [0],
       signatures: [S],
       evmAddress: E
   )
   ```
6. The function verifies `S` over `signedData` with tag `"FLOW-V0.0-user"` against key `K` — **passes**, because the tag is identical on both networks.
7. The function checks that the COA at `/public/coa` in `A_main` has EVM address `E` — **passes**, because the COA EVM address is chain-agnostic.
8. `ValidationResult(isValid: true)` is returned. The attacker has successfully replayed Alice's testnet signature on mainnet.

func coaOwnershipProofValidator(contractAddress flow.Address, backend backends.Backend) func(proof *types.COAOwnershipProofInContext) (bool, error) {
	return func(proof *types.COAOwnershipProofInContext) (bool, error) {
		value, err := backend.Invoke(
			environment.ContractFunctionSpec{
				AddressFromChain: func(_ flow.Chain) flow.Address {
					return contractAddress
				},
				LocationName: "EVM",
				FunctionName: "validateCOAOwnershipProof",
				ArgumentTypes: []sema.Type{
					types.FlowAddressSemaType,
					types.PublicPathSemaType,
					types.SignedDataSemaType,
					types.KeyIndicesSemaType,
					types.SignaturesSemaType,
					types.AddressBytesSemaType,
				},
			},
			proof.ToCadenceValues(),
		)
		if err != nil {
			if types.IsAFatalError(err) {
				panic(err)
			}
			return false, err
		}
		data, ok := value.(cadence.Struct)
		if !ok {
			return false, fmt.Errorf("invalid output data received from validateCOAOwnershipProof")
		}

		isValidValue := cadence.SearchFieldByName(data, ValidationResultTypeIsValidFieldName)
		if isValidValue == nil {
			return false, fmt.Errorf("invalid output data received from validateCOAOwnershipProof")
		}

		return bool(isValidValue.(cadence.Bool)), nil
	}
