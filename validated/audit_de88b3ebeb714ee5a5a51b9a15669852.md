### Title
L2 `base_mint_amount` and L1 hardcoded `TOKENS_PER_MINT_REQUEST` Mismatch Silently Breaks Reward Minting - (`src/reward_supplier/reward_supplier.cairo` / `L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

The L2 `RewardSupplier` sends mint-request messages to L1 with a payload derived from the configurable storage variable `base_mint_amount`, while the L1 `RewardSupplier` hardcodes `TOKENS_PER_MINT_REQUEST = 1_300_000 * 10**18` as the **only** payload value it will ever look for. If these two values diverge — at deployment or after an upgrade — the L1 silently finds zero matching messages, mints nothing, and the entire reward pipeline is permanently frozen.

---

### Finding Description

**L2 side** — `send_mint_request_to_l1_reward_supplier` constructs the message payload from the storage variable `base_mint_amount`:

```cairo
// src/reward_supplier/reward_supplier.cairo  line 334
let payload: Span<felt252> = array![self.base_mint_amount.read().into()].span();
send_message_to_l1_syscall(:to_address, :payload).unwrap_syscall();
``` [1](#0-0) 

**L1 side** — `tick()` (and `requiredMinting()`) hardcodes the expected payload as the compile-time constant `TOKENS_PER_MINT_REQUEST`:

```solidity
// L1/starkware/solidity/stake/RewardSupplier.sol  line 10
uint256 constant TOKENS_PER_MINT_REQUEST = 1_300_000 * 10**18;
...
// line 113-114
uint256[] memory messagePayload = new uint256[](1);
messagePayload[0] = TOKENS_PER_MINT_REQUEST;
bytes32 msgHash = l2ToL1MsgHash(mintRequestSource(), address(this), messagePayload);
uint256 numMsgsToConsume = Math.min(
    messagingContract().l2ToL1Messages(msgHash),
    MAX_MESSAGES_TO_PROCESS_PER_TICK
);
``` [2](#0-1) [3](#0-2) [4](#0-3) 

The L1 computes a message hash using `TOKENS_PER_MINT_REQUEST` as the payload. If `base_mint_amount` on L2 is anything other than exactly `1_300_000 * 10**18`, the hash computed by L1 will never match the hash of the messages actually sent by L2. `l2ToL1Messages(msgHash)` returns `0`, `numMsgsToConsume = 0`, `amountToMint = 0`, and the entire `if (amountToMint > 0)` block is skipped — **silently, with no revert**.

The L2 `l1_pending_requested_amount` keeps growing with each call to `request_funds`, but no tokens are ever minted or bridged back. Once the contract's existing STRK balance is exhausted, `claim_rewards` will fail for every staker and delegator. [5](#0-4) 

---

### Impact Explanation

**Permanent freezing of unclaimed yield (High).**

All stakers and delegators accumulate rewards that are tracked in `unclaimed_rewards`, but `claim_rewards` transfers tokens from the contract's actual balance. Once the pre-seeded balance is drained and no new tokens arrive from L1, every call to `claim_rewards` reverts. Stakers cannot withdraw earned rewards; the yield is permanently frozen until the mismatch is corrected and a new deployment is made. [6](#0-5) 

---

### Likelihood Explanation

The L2 constructor accepts `base_mint_amount` as a free parameter with no on-chain enforcement that it equals the L1 constant. The L1 constant is buried in a Solidity file and is not surfaced in any interface or documentation visible to the deployer of the L2 contract. A deployment that sets `base_mint_amount` to any value other than exactly `1_300_000 * 10**18` (e.g., a different denomination, a test value, or a future governance-adjusted value) will silently break the entire reward pipeline. Because the failure is silent (no revert on either side), it may go undetected until stakers attempt to claim rewards. [7](#0-6) [8](#0-7) 

---

### Recommendation

1. **Remove the hardcoded constant from L1.** The L1 `RewardSupplier` should not assume a fixed payload value. Instead, it should accept and process any payload value (or read the expected amount from storage, set during initialization).
2. **Alternatively, enforce the invariant on-chain.** Add a view function to the L1 contract that exposes `TOKENS_PER_MINT_REQUEST`, and have the L2 constructor assert that `base_mint_amount == l1_reward_supplier.tokens_per_mint_request()` (via a cross-layer check or off-chain deployment script).
3. **Add a setter with matching L1 update.** If `base_mint_amount` ever needs to change, both sides must be updated atomically. The current design has no setter for `base_mint_amount` on L2 and no mechanism to update the L1 constant, making future governance changes impossible without a full redeployment.

---

### Proof of Concept

1. Deploy L2 `RewardSupplier` with `base_mint_amount = 2_000_000 * 10**18` (any value ≠ `1_300_000 * 10**18`).
2. Staking activity accumulates rewards; `update_unclaimed_rewards_from_staking_contract` is called, triggering `request_funds`, which calls `send_mint_request_to_l1_reward_supplier` with payload `[2_000_000 * 10**18]`.
3. Anyone calls `tick()` on the L1 `RewardSupplier`. It computes `msgHash` using `TOKENS_PER_MINT_REQUEST = 1_300_000 * 10**18`. `l2ToL1Messages(msgHash)` returns `0`. `amountToMint = 0`. Nothing happens.
4. L2 messages accumulate indefinitely; `l1_pending_requested_amount` grows; no tokens are ever bridged back.
5. Once the initial STRK balance of the L2 `RewardSupplier` is exhausted, all calls to `claim_rewards` revert with `AMOUNT_TOO_HIGH` (balance < unclaimed_rewards), permanently freezing all staker and delegator yield. [9](#0-8) [1](#0-0)

        /// Requests funds from L1 to account for new rewards, if the contract's balance is too low.
        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }

    function requiredMinting() public view returns (uint256, uint256) {
        uint256[] memory messagePayload = new uint256[](1);
        messagePayload[0] = TOKENS_PER_MINT_REQUEST;

        bytes32 msgHash = l2ToL1MsgHash(mintRequestSource(), address(this), messagePayload);
        // Limit the number of msgs to consume to limit.
        uint256 numMsgsToConsume = Math.min(
            messagingContract().l2ToL1Messages(msgHash),
            MAX_MESSAGES_TO_PROCESS_PER_TICK
        );

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
