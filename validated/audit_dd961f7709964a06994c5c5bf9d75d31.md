### Title
Zero EVM Gas Cost for ExitToNear/ExitToEthereum Enables Free NEAR Gas Scheduling — (File: engine-precompiles/src/native.rs)

---

### Summary

`EXIT_TO_NEAR_GAS` and `EXIT_TO_ETHEREUM_GAS` are both hardcoded to `EthGas::new(0)`. Each invocation of these precompiles schedules a NEAR promise carrying 10 TGas–100 TGas of real NEAR gas. Because the EVM gas cost is zero, an attacker can invoke either precompile an arbitrary number of times within a single EVM transaction at negligible EVM gas cost, scheduling NEAR gas that the relayer pays for but is never compensated for via EVM gas fees. This breaks the economic invariant that every unit of NEAR gas consumed on behalf of an EVM transaction must be covered by the EVM gas fees paid by the caller.

**Promise scheduling has no NEAR gas budget check:**

After EVM execution completes, `filter_promises_from_logs` in `engine/src/engine.rs` iterates every promise-bearing log and unconditionally calls `schedule_promise` / `schedule_promise_callback` for each one. There is no accounting of total NEAR gas being scheduled against the prepaid NEAR gas budget. [4](#0-3) 

**`Engine::call` does not account for NEAR gas consumed by promises:**

`used_gas` captures only EVM gas. The NEAR gas attached to all scheduled promises is invisible to the EVM gas accounting layer. [5](#0-4) 

**Clarification on the "unbounded" claim:** The NEAR runtime itself enforces that total gas attached to promises cannot exceed the prepaid gas budget; it will panic the transaction if that limit is breached. So the number of schedulable promises per transaction is bounded by `prepaid_gas / gas_per_promise` (e.g., 300 TGas / 10 TGas = 30 calls for `ExitToNear`). The claim of *unbounded* scheduling is therefore false. However, the insolvency issue is real and does not require exceeding the prepaid budget — it only requires that the EVM caller pays 0 EVM gas while the relayer pays real NEAR gas.

---

### Impact Explanation

Aurora's economic model requires that every unit of NEAR gas consumed on behalf of an EVM transaction is covered by the EVM gas fees paid by the transaction sender. The relayer submits the NEAR transaction, prepays NEAR gas, and is compensated by the EVM gas fees (`gas_used × gas_price`).

Because `EXIT_TO_NEAR_GAS = 0` and `EXIT_TO_ETHEREUM_GAS = 0`, an attacker who calls either precompile N times in a loop pays 0 EVM gas for those N calls (only loop overhead: `JUMP`, `CALL`, etc., which is a few hundred EVM gas per iteration). Each call schedules 10–100 TGas of NEAR gas. The relayer pays that NEAR gas but receives 0 EVM gas compensation for it.

Concretely, with a 300 TGas prepaid budget and `FT_TRANSFER_GAS = 10 TGas`, an attacker can schedule 30 NEAR promises for the cost of ~30 × 300 = ~9,000 EVM gas in loop overhead — a negligible EVM gas cost. The relayer absorbs the full 300 TGas NEAR gas cost with zero compensation. Repeated across many transactions, this drains the relayer's NEAR balance.

**Impact: Critical. Insolvency** — the Aurora relayer/engine system pays NEAR gas not covered by the EVM caller.

### Likelihood Explanation

- No special privileges required: any deployed EVM contract can call the `ExitToNear` precompile at `0xe9217bc70b7ed1f598ddd3199e80b093fa71124f` or `ExitToEthereum` at `0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab` directly.
- The ETH (base token) path requires only sending 0 ETH with a valid recipient account ID — no token balance needed.
- The exploit is a simple loop in a deployed Solidity contract.
- The TODO comments (`// TODO(#483)`) confirm the zero gas cost is a known unresolved issue, not an intentional design. [1](#0-0) 

---

### Recommendation

1. **Set non-zero EVM gas costs** for `EXIT_TO_NEAR_GAS` and `EXIT_TO_ETHEREUM_GAS` that reflect the NEAR gas cost of the scheduled promises. A reasonable lower bound is `FT_TRANSFER_GAS / NEAR_GAS_PER_EVM` (where `NEAR_GAS_PER_EVM ≈ 175_000_000` as used elsewhere in the codebase).
2. **Add a NEAR gas budget check** in `filter_promises_from_logs`: track the cumulative NEAR gas being scheduled and abort (or revert the EVM transaction) if it would exceed the remaining prepaid NEAR gas.
3. Resolve the open TODO `#483` which explicitly acknowledges this gap.

fn filter_promises_from_logs<I, T, P>(
    io: &I,
    handler: &mut P,
    logs: T,
    current_account_id: &AccountId,
) -> Vec<ResultLog>
where
    T: IntoIterator<Item = Log>,
    P: PromiseHandler,
    I: IO + Copy,
{
    let mut previous_promise: Option<PromiseId> = None;
    logs.into_iter()
        .filter_map(|log| {
            if log.address == exit_to_near::ADDRESS.raw()
                || log.address == exit_to_ethereum::ADDRESS.raw()
            {
                if log.topics.is_empty() {
                    if let Ok(promise) = PromiseArgs::try_from_slice(&log.data) {
                        match promise {
                            PromiseArgs::Create(promise) => {
                                // Safety: this promise creation is safe because it does not come from
                                // users directly. The exit precompile only create promises which we
                                // are able to execute without violating any security invariants.
                                let id = match previous_promise {
                                    Some(base_id) => {
                                        schedule_promise_callback(handler, base_id, &promise)
                                    }
                                    None => schedule_promise(handler, &promise),
                                };
                                previous_promise = Some(id);
                            }
                            PromiseArgs::Callback(promise) => {
                                // Safety: This is safe because the promise data comes from our own
                                // exit precompiles. See note above.
                                let base_id = match previous_promise {
                                    Some(base_id) => {
                                        schedule_promise_callback(handler, base_id, &promise.base)
                                    }
                                    None => schedule_promise(handler, &promise.base),
                                };
                                let id =
                                    schedule_promise_callback(handler, base_id, &promise.callback);
                                previous_promise = Some(id);
                            }
                            PromiseArgs::Recursive(_) => {
                                unreachable!("Exit precompiles do not produce recursive promises")
                            }
                        }
                    }
                    // do not pass on these "internal logs" to the caller
                    None
                } else {
                    // The exit precompile does produce externally consumable logs in
                    // addition to the promises. The external logs have a non-empty
                    // `topics` field.
                    Some(evm_log_to_result_log(log))
                }
            } else if log.address == cross_contract_call::ADDRESS.raw() {
                if log.topics[0] == cross_contract_call::AMOUNT_TOPIC {
                    // NEAR balances are 128-bit, so the leading 16 bytes of the 256-bit topic
                    // value should always be zero.
                    assert_eq!(&log.topics[1].as_bytes()[0..16], &[0; 16]);
                    let required_near =
                        Yocto::new(U256::from_big_endian(log.topics[1].as_bytes()).low_u128());
                    if let Ok(promise) = PromiseCreateArgs::try_from_slice(&log.data) {
                        let id = crate::xcc::handle_precompile_promise(
                            io,
                            handler,
                            previous_promise,
                            &promise,
                            required_near,
                            current_account_id,
                        );
                        previous_promise = Some(id);
                    }
                }
                // do not pass on these "internal logs" to the caller
                None
            } else {
                Some(evm_log_to_result_log(log))
            }
        })
        .collect()
}
