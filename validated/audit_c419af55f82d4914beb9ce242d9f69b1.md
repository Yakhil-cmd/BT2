### Title
Unprivileged Caller Can Permanently Deny Block Rewards to All Stakers via Unguarded `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `IStakingRewardsManager` has no access control and accepts a caller-controlled `disable_rewards: bool` parameter. Any unprivileged address can call it with `disable_rewards: true` for any valid staker, which advances the global `last_reward_block` to the current block without distributing any rewards. Because `last_reward_block` is a single global variable, this one call permanently blocks every staker from receiving consensus block rewards for that block.

### Finding Description
`update_rewards` is exposed as a public entry point with no role check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global write

    if disable_rewards || self.is_pre_consensus() {
        return;                            // exits before any reward calculation
    }
    ...
``` [1](#0-0) 

`general_prerequisites` only asserts the contract is unpaused and the caller is non-zero — no role or identity check: [2](#0-1) 

`last_reward_block` is a single global storage slot, not per-staker: [3](#0-2) 

**Attack path:**

1. Attacker picks any currently active staker address `S` (readable from public events or `get_stakers`).
2. At the start of block `N`, attacker calls `update_rewards(S, disable_rewards: true)`.
3. The function passes all checks (contract unpaused, staker active, `N > last_reward_block`), writes `last_reward_block = N`, then returns early — zero rewards distributed.
4. Every subsequent call to `update_rewards` in block `N` hits `REWARDS_ALREADY_UPDATED` and reverts.
5. Block `N`'s consensus rewards are permanently lost for all stakers.
6. Attacker repeats at the start of block `N+1`, `N+2`, … to continuously drain all stakers of their yield.

The `disable_rewards` flag is clearly intended for a trusted caller (e.g., the consensus layer signalling a non-attesting block), but the absence of any access control makes it freely exploitable.

### Impact Explanation
**High — Permanent freezing / theft of unclaimed yield.**

Block rewards that should accrue to stakers are irreversibly skipped for every block the attacker targets. Because `last_reward_block` is global, a single transaction per block is sufficient to zero out rewards for the entire staker set. Continuous execution across blocks constitutes a complete, permanent denial of consensus-phase yield with no recovery path.

This falls squarely within: *"High: Permanent freezing of unclaimed yield or unclaimed royalties."*

### Likelihood Explanation
**High.** The attacker requires only:
- A non-zero address (any EOA or contract).
- Knowledge of one valid, active staker address (publicly available on-chain).
- Gas for one transaction per block.

No privileged key, no front-running of a specific user action, no external dependency. The attack is cheap, repeatable, and requires no profit motive.

### Recommendation
Add an access-control guard to `update_rewards` so only a trusted role (e.g., `OPERATOR_ROLE`, the consensus sequencer address, or a dedicated rewards-manager role) may call it. For example:

```cairo
fn update_rewards(...) {
    self.roles.only_operator();   // or equivalent trusted-caller check
    ...
}
```

Alternatively, remove the `disable_rewards` parameter entirely and derive the disable condition from on-chain protocol state (e.g., `is_pre_consensus()`), so no external caller can influence whether rewards are distributed.

### Proof of Concept

```
Block N:
  Attacker tx: staking_contract.update_rewards(
      staker_address = <any valid active staker>,
      disable_rewards = true
  )
  → last_reward_block written to N
  → _update_rewards never called → 0 STRK distributed

  Legitimate staker tx (same block): staking_contract.update_rewards(staker, false)
  → PANICS: "Rewards already updated for this block"

Block N+1: attacker repeats → all stakers receive 0 rewards indefinitely
``` [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
