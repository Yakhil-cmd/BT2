### Title
Missing Minimum Threshold Lower-Bound Check in FROST Presign and Sign Allows `threshold=1` — (File: `src/frost/mod.rs`)

---

### Summary

The `presign()` and `assert_sign_inputs()` functions in `src/frost/mod.rs` validate that `threshold <= participants.len()` but do **not** enforce the minimum lower bound `threshold >= 2`. This is the direct analog to the external report's "no minimum floor enforced" class: DKG enforces the floor, but the downstream signing path silently accepts a threshold of 1, producing a corrupted, unusable signature that honest parties accept as protocol output.

---

### Finding Description

`assert_key_invariants()` in `src/dkg.rs` enforces both bounds on the threshold:

```rust
// src/dkg.rs:573-582
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

However, the FROST `presign()` function only checks the upper bound:

```rust
// src/frost/mod.rs:71-77
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no lower-bound check; threshold=1 is silently accepted
``` [2](#0-1) 

Identically, `assert_sign_inputs()` — the shared validation entry point for both EdDSA and RedJubjub signing — only checks the upper bound:

```rust
// src/frost/mod.rs:144-150
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no lower-bound check
``` [3](#0-2) 

`ReconstructionLowerBound` is a plain `usize` newtype with no embedded invariant:

```rust
// src/thresholds.rs:12
pub struct ReconstructionLowerBound(usize);
``` [4](#0-3) 

So `ReconstructionLowerBound(1)` is a valid value that passes every check in the FROST signing path.

---

### Impact Explanation

When `threshold=1` is supplied to FROST signing:

- The Lagrange interpolation in `participants.lagrange()` is evaluated over a single-point set, assigning that participant a coefficient of 1 — treating their **share** as if it were the **full secret key**.
- The resulting signature share is computed from an incorrect scalar, producing a signature that is cryptographically invalid against the public key that was generated under `threshold >= 2`.
- All honest parties that receive and accept this output obtain an **unusable cryptographic output** — the signature will fail verification.

This maps to the allowed High impact: **Corruption of sign outputs so honest parties accept unusable cryptographic outputs**. [5](#0-4) 

---

### Likelihood Explanation

The entry path is a direct library API call. Any unprivileged library caller — including a malicious coordinator who controls the `threshold` argument passed to `presign()` or the EdDSA/RedJubjub `sign()` wrappers — can supply `threshold=1`. No special privilege, leaked key, or cryptographic break is required. The inconsistency with DKG's enforced minimum makes this easy to trigger accidentally or maliciously. [6](#0-5) 

---

### Recommendation

Add the same lower-bound guard that `assert_key_invariants` applies, in both `presign()` and `assert_sign_inputs()`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Alternatively, encode the invariant directly in `ReconstructionLowerBound::new()` so it is impossible to construct a value below 2, eliminating the class of missing-check bugs across all call sites. [7](#0-6) 

---

### Proof of Concept

```
1. Run DKG for 3 participants with threshold=2 (enforced, succeeds).
2. Call frost::presign() with threshold=1 — passes all validation in src/frost/mod.rs:71-77.
3. Call assert_sign_inputs() / the EdDSA sign wrapper with threshold=1 — passes all validation in src/frost/mod.rs:144-150.
4. The signing protocol proceeds; participant P1's Lagrange coefficient is computed as 1 (single-point interpolation).
5. P1's signature share = share_1 * challenge, not (full_key) * challenge.
6. The aggregated signature fails verification against the public key — honest parties have accepted a corrupted, unusable output.
``` [8](#0-7)

async fn do_presign<C: Ciphersuite + Send>(
    mut chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    signing_share: SigningShare<C>,
    mut rng: impl CryptoRngCore,
) -> Result<PresignOutput<C>, ProtocolError> {
    // Round 1
    let mut commitments_map: BTreeMap<Identifier<C>, SigningCommitments<C>> = BTreeMap::new();

    // Creating two commitments and corresponding nonces
    let (nonces, commitments) = commit(&signing_share, &mut rng);
    commitments_map.insert(me.to_identifier()?, commitments);

    let commit_waitpoint = chan.next_waitpoint();
    // Sending the commitments to all
    chan.send_many(commit_waitpoint, &commitments)?;

    // Collecting the commitments
    for (from, commitment) in recv_from_others(&chan, commit_waitpoint, &participants, me).await? {
        commitments_map.insert(from.to_identifier()?, commitment);
    }

    Ok(PresignOutput {
        nonces,
        commitments_map,
    })
}

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
