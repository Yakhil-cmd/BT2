### Title
Anyone Can Call `update_rewards` With `disable_rewards: true` to Permanently Freeze All Stakers' Block Rewards - (File: `src/staking/staking.cairo`)

### Summary

`update_rewards` is a public, permissionless function in `IStakingRewardsManager`. It writes to the **global** `last_reward_block` storage variable on every call, regardless of the `disable_rewards` flag. An attacker can call it with `disable_rewards: true` at the start of every block, setting `last_reward_block` to the current block number and causing every subsequent legitimate `update_rewards` call in that block to revert with `REWARDS_ALREADY_UPDATED`. Repeated across blocks, this permanently freezes all stakers' consensus block rewards.

---

### Finding Description

`update_rewards` is exposed as a public ABI entry point with no caller restriction: [1](#0-0) 

Its implementation writes to the global `last_reward_block` unconditionally before checking `disable_rewards`: [2](#0-1) 

The guard is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

followed immediately by:

```cairo
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

`last_reward_block` is a **single global** `BlockNumber` in storage, not a per-staker map: [3](#0-2) 

Because the write happens before the `disable_rewards` early-return, any caller who supplies `disable_rewards: true` (and any valid active staker address, which is public on-chain) will:

1. Pass the block-number guard.
2. Write `last_reward_block = current_block_number`.
3. Return without distributing rewards.

Every other call to `update_rewards` in the same block — for any staker — then hits the guard and reverts with `REWARDS_ALREADY_UPDATED`.

The error string confirms the intent: [4](#0-3) 

The flow tests confirm that a second call in the same block always reverts: [5](#0-4) 

---

### Impact Explanation

`update_rewards` is the sole mechanism for distributing per-block consensus rewards to stakers. If an attacker calls it with `disable_rewards: true` at the beginning of every block, no staker ever receives block rewards. The rewards are not accumulated anywhere — they are simply never distributed. This constitutes **permanent freezing of unclaimed yield** for all stakers in the protocol.

**Impact: High** — Permanent freezing of unclaimed yield for all stakers.

---

### Likelihood Explanation

- No special role, key, or privilege is required.
- The attacker only needs to know any valid, active staker address (fully public on-chain).
- The cost is one cheap Starknet transaction per block.
- There are no preconditions beyond the consensus rewards epoch being active.

**Likelihood: High** — Any unprivileged address can execute this attack continuously.

---

### Recommendation

Restrict who may call `update_rewards` with `disable_rewards: true`. The simplest fix is to make `update_rewards` callable only by the staker themselves or their reward address (mirroring the access control on `claim_rewards`), or to split the function into a privileged "disable" variant and a public "enable" variant. Alternatively, move the `last_reward_block.write` to after the `disable_rewards` branch so that a call with `disable_rewards: true` does not consume the block's reward slot.

---

### Proof of Concept

```
Block N:
  Attacker calls: staking.update_rewards(valid_staker_address, disable_rewards=true)
    → last_reward_block is written to N
    → returns early, no rewards distributed

  Staker A calls: staking.update_rewards(staker_A, disable_rewards=false)
    → assert!(N > N) fails → REWARDS_ALREADY_UPDATED panic

  Staker B calls: staking.update_rewards(staker_B, disable_rewards=false)
    → assert!(N > N) fails → REWARDS_ALREADY_UPDATED panic

Block N+1:
  Attacker repeats → same result

Result: No staker ever receives consensus block rewards.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
