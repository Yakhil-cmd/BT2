### Title
Unauthorized Burn and Transfer of Native `evm/` Tokens via Bank Precompile Missing Caller Authorization - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The bank precompile's `burn` and `transfer` methods accept an arbitrary `addr`/`sender` argument without verifying that the calling contract is authorized to act on behalf of that address. Any contract can burn or transfer `evm/<callerContractAddress>` native tokens from any user's account without that user's consent.

### Finding Description

The bank precompile at `x/cronos/keeper/precompiles/bank.go` exposes `burn(address, uint256)` and `transfer(address, address, uint256)` methods callable by any EVM contract. The denom operated on is derived from `contract.Caller()` (the calling contract's address), but the `addr`/`sender` argument — the account whose tokens are actually debited — is taken directly from the user-supplied ABI arguments with no authorization check.

For `burn`: [1](#0-0) 

`recipient` (the address to burn from) is `args[0]`, an attacker-controlled value. The only guard is `checkBlockedAddr`, which only rejects Cosmos module accounts: [2](#0-1) 

The actual burn then debits `addr` (the victim) with no consent check: [3](#0-2) 

For `transfer`: [4](#0-3) 

`sender` is `args[0]`, again fully attacker-controlled. There is no check that `sender == contract.Caller()` or that `sender` has granted any allowance. `SendCoins` is called directly from the victim's account.

The intended usage pattern (as shown in the integration test contract) passes `msg.sender` as the `from` address: [5](#0-4) [6](#0-5) 

But the precompile itself never enforces this constraint.

### Impact Explanation

**Critical.** Any deployed contract can call `bank.burn(victimAddress, amount)` or `bank.transfer(victimAddress, attackerAddress, amount)` to drain `evm/<callerContractAddress>` native tokens from any user who holds that denom. Users who have converted ERC20 tokens to native `evm/` denom tokens (e.g., via `moveToNative`) are directly exposed. This constitutes unauthorized burn and transfer of CRC20/native assets.

### Likelihood Explanation

**High.** The bank precompile is a publicly accessible precompile at address `0x0000000000000000000000000000000000000064`. Any unprivileged EVM contract can call it. No special permissions, leaked keys, or governance access are required. The attacker only needs to deploy a contract and have users hold its denom. [7](#0-6) 

### Recommendation

In both `burn` and `transfer`, verify that the `addr`/`sender` argument equals `contract.Caller()` (the calling contract itself), or implement an allowance/approval mechanism before permitting cross-account debits:

```go
// For burn: enforce addr == contract.Caller()
callerAddr := sdk.AccAddress(contract.Caller().Bytes())
if !addr.Equals(callerAddr) {
    return nil, errors.New("burn: caller is not the token holder")
}

// For transfer: enforce from == contract.Caller()
callerAddr := sdk.AccAddress(contract.Caller().Bytes())
if !from.Equals(callerAddr) {
    return nil, errors.New("transfer: caller is not the sender")
}
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function burn(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
}

contract AttackerContract {
    address constant bankPrecompile = 0x0000000000000000000000000000000000000064;
    IBankModule bank = IBankModule(bankPrecompile);

    // Drain victim's evm/<address(this)> native tokens without consent
    function stealBurn(address victim, uint256 amount) external {
        // denom = "evm/" + address(this)
        // burns victim's native tokens of this contract's denom
        bank.burn(victim, amount);
    }

    function stealTransfer(address victim, address attacker, uint256 amount) external {
        // transfers victim's native tokens to attacker without victim's approval
        bank.transfer(victim, attacker, amount);
    }
}
```

1. Deploy `AttackerContract`; its denom becomes `evm/<AttackerContract address>`.
2. Victim holds `evm/<AttackerContract>` native tokens (e.g., after interacting with the contract).
3. Attacker calls `stealBurn(victimAddress, victimBalance)` — victim's native tokens are burned with no authorization.
4. Or attacker calls `stealTransfer(victimAddress, attackerAddress, victimBalance)` — victim's native tokens are transferred to the attacker.

The precompile executes `SendCoinsFromAccountToModule` / `SendCoins` directly from the victim's account with no consent check. [8](#0-7) [9](#0-8)
