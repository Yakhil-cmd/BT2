### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

The factory enforces `maxAdminSpreadFeeE6` on `setPoolAdminFees` but applies **no equivalent cap** on `setPoolBinAdditionalFees`. Because per-bin additional fees are added directly to the base fee inside the swap math, the pool admin can charge traders more than the factory-owner-imposed cap permits.

### Finding Description

`setPoolAdminFees` guards the admin's base spread component:

```solidity
// MetricOmmPoolFactory.sol L414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees`, callable by the same pool admin, forwards values straight to the pool with **no cap check**:

```solidity
// MetricOmmPoolFactory.sol L450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` only validates the bin index, not the fee magnitudes: [3](#0-2) 

Inside every swap path, the per-bin additional fee is **added to the base fee** before computing the trader's cost:

```solidity
// MetricOmmPool.sol L999
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [4](#0-3) 

The same pattern applies to the sell direction: [5](#0-4) 

`addFeeBuyE6` and `addFeeSellE6` are `uint16`, so their maximum value is 65 535, which in E6 notation equals **6.5535 %**. The hard cap `HARD_MAX_SPREAD_FEE_E6 = 200 000` (20 %) applies only to the base spread fee path: [6](#0-5) 

### Impact Explanation

The factory owner uses `maxAdminSpreadFeeE6` to bound how much the pool admin can extract from traders. If the owner sets `maxAdminSpreadFeeE6 = 0` (or any low value), the pool admin can still call `setPoolBinAdditionalFees(pool, bin, 65535, 65535)` on every bin, imposing up to 6.5535 % additional fee per bin on every swap. Traders pay more than the cap permits; the excess accrues inside the pool as LP surplus and is later swept by the admin via `collectPoolFees`. This is a direct loss of trader principal that the factory-owner cap was designed to prevent.

### Likelihood Explanation

The pool admin is a semi-trusted role that is explicitly constrained to operate "inside caps." The bypass requires only a single permissioned call (`setPoolBinAdditionalFees`) with no timelock. Any pool admin who wishes to extract more fees than the factory owner allows can do so immediately after deployment or after the owner tightens the cap.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` analogous to the one in `setPoolAdminFees`. One approach:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (uint256(addFeeBuyE6)  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (uint256(addFeeSellE6) > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce that `spreadFeeE6 + addFeeBuyE6` (and the sell equivalent) does not exceed the hard cap, so the total effective fee per bin is bounded end-to-end.

### Proof of Concept

1. Factory owner calls `setFeeCaps(…, maxAdminSpreadFeeE6 = 0, …)` to prohibit any admin spread fee.
2. Pool admin calls `setPoolAdminFees(pool, 0, 0)` — succeeds (0 ≤ 0).
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` — **succeeds with no revert** because there is no cap check.
4. A trader swaps through bin 0; the swap math computes `baseFeeX64 + mulDiv(65535, ONE_X64, 1e6)`, charging the trader an extra 6.5535 % that the factory owner's cap was meant to prevent.
5. The surplus accumulates in the pool and is collected by the admin via `collectPoolFees`, transferring trader funds to the admin destination beyond the permitted cap.
