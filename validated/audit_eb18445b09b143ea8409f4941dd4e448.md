### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

`update_rewards` is documented as restricted to "Only starkware sequencer" but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which writes `last_reward_block` to the current block and returns without distributing rewards. Because `last_reward_block` is a **global** singleton, every subsequent call in that block reverts with `REWARDS_ALREADY_UPDATED`, permanently discarding that block's rewards for all stakers.

---

### Finding Description

The specification at `docs/spec.md:1645` declares:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation at `src/staking/staking.cairo` enforces no such restriction. `general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero: [2](#0-1) 

The full `update_rewards` body: [3](#0-2) 

The critical sequence is:

1. Line 1454–1458: gate on `current_block_number > last_reward_block` — one call per block, globally.
2. Line 1485: `last_reward_block.write(current_block_number)` — written **before** the `disable_rewards` branch.
3. Line 1487–1489: if `disable_rewards == true`, return immediately without distributing.

`last_reward_block` is a single global storage slot, not per-staker: [4](#0-3) 

---

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` first in a block:

- Advances `last_reward_block` to the current block.
- Skips all reward distribution.
- Blocks every legitimate call for that block with `REWARDS_ALREADY_UPDATED`.

The block's rewards are **permanently lost** — there is no catch-up mechanism. Repeating this every block permanently freezes all unclaimed yield for all stakers and delegators. This matches the allowed High impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- No special role, key, or privilege is required.
- The call is cheap (a single contract call per block).
- The attacker only needs to know any one active staker address (publicly readable from events or `get_stakers`).
- The attack is sustainable indefinitely at negligible cost.

---

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, consistent with the spec. For example, assert `get_caller_address() == sequencer_address` using a stored sequencer address, or gate via the existing roles component (e.g., an `APP_GOVERNOR` or dedicated `SEQUENCER` role). The check must be placed **before** `last_reward_block.write(current_block_number)` so that unauthorized calls revert without consuming the block's reward slot.

---

### Proof of Concept

```
// Setup: staker_A is an active staker with effective balance (K epochs elapsed).
// Attacker is any EOA with no stake.

// Block N:
attacker.call(
    staking_contract,
    update_rewards(staker_address: staker_A, disable_rewards: true)
);
// → passes general_prerequisites (not paused, caller != 0)
// → passes REWARDS_ALREADY_UPDATED check (block N > last_reward_block)
// → writes last_reward_block = N
// → disable_rewards == true → returns, no rewards distributed

// Same block N, legitimate sequencer call:
sequencer.call(
    staking_contract,
    update_rewards(staker_address: staker_A, disable_rewards: false)
);
// → PANICS: REWARDS_ALREADY_UPDATED (block N == last_reward_block)

// Block N's rewards are permanently lost.
// Repeat every block → all staker yield frozen indefinitely.
``` [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
