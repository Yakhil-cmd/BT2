### Title
Signature Verification Bypass for Invoke Transactions via `skip_stateful_validations` Deploy-Account UX Path - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validation path unconditionally skips the `__validate__` entry point (signature verification) for any invoke transaction with `nonce=1` when the sender's on-chain nonce is `0` and any transaction from that address exists in the mempool or a recent block. Because this check is purely address-based and not caller-authenticated, an unprivileged third party can submit an invoke transaction for a victim account — with an entirely invalid signature — and have it admitted to the mempool without signature verification.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, the function `skip_stateful_validations` implements a UX shortcut: [1](#0-0) 

When an invoke transaction has `nonce == 1` and the account's on-chain nonce is `0`, the function queries the mempool for any existing transaction from that address. If one exists, it returns `true` (skip validation). This result propagates back through `extract_state_nonce_and_run_validations`: [2](#0-1) 

And into `run_validate_entry_point`, which sets `execution_flags.validate = !skip_validate`: [3](#0-2) 

When `validate=false`, the blockifier's `StatefulValidator::perform_validations` returns `Ok(())` immediately without calling `__validate__`: [4](#0-3) 

The `validate_by_mempool` call that precedes this only checks nonce ordering and fee escalation — it does not verify the signature: [5](#0-4) 

**Attack scenario:**

1. Victim account `A` submits a `deploy_account` transaction (nonce=0). It sits in the mempool; on-chain nonce of `A` remains `0`.
2. Attacker observes the mempool and sees `A`'s `deploy_account`.
3. Attacker submits `Invoke(sender=A, nonce=1, signature=GARBAGE)`.
4. Gateway evaluates: `account_nonce=0`, `tx_nonce=1`, `account_tx_in_pool_or_recent_block(A)=true` → `skip_validate=true`.
5. The `__validate__` entry point is never called. The transaction is admitted to the mempool with an invalid signature.

Additionally, the `StatefulTransactionValidatorConfig` field `max_nonce_for_validation_skip` exists and is documented as "Maximum nonce for which the validation is skipped": [6](#0-5) 

But this field is **never read** inside `skip_stateful_validations`. The nonce bound is hardcoded to `== Nonce(Felt::ONE)`, making the config field dead code in the Rust gateway path and leaving no operator-controlled knob to tighten the skip window.

### Impact Explanation

An unprivileged attacker can inject invoke transactions with arbitrary (invalid) signatures for any account that has a pending `deploy_account` in the mempool. These transactions pass all gateway admission checks and enter the mempool. This directly satisfies the High impact criterion: **"Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."**

Secondary effects:
- **Mempool pollution**: invalid transactions occupy mempool capacity, potentially displacing valid transactions via capacity overflow eviction logic.
- **Fee drain**: when the batcher executes the block, the `deploy_account` runs first (creating the account), then the attacker's invoke runs with `validate=true` and `strict_nonce_check=false`. If `__validate__` reverts, the transaction is reverted but a fee is still charged against the newly-funded account.

### Likelihood Explanation

The attack requires only passive mempool observation (public) and submitting a single transaction. No privileged access, no special contract, no cryptographic capability is needed. Any account that submits a `deploy_account` is immediately targetable for the duration the transaction remains unconfirmed.

### Recommendation

1. Remove the unconditional skip or gate it on a cryptographic check. The intended UX (deploy_account + invoke submitted together) can be preserved by verifying the signature even when skipping the on-chain `__validate__` call, or by requiring the invoke to be bundled with the deploy_account in a single RPC call.
2. Wire `max_nonce_for_validation_skip` into `skip_stateful_validations` so operators can at minimum set it to `0` to disable the skip entirely.
3. Alternatively, require that the mempool entry for the address is specifically a `deploy_account` transaction type, not just any transaction, before granting the skip.

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
