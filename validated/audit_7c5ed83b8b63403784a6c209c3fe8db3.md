### Title
Bootstrap Private Key Files Written with World-Readable Permissions (0644) - (File: cmd/util/cmd/common/utils.go)

### Summary
The Flow bootstrap tooling writes all private key material — including node staking keys, networking keys, random beacon private keys, machine account private keys, and the secrets DB encryption key — to disk with `0644` permissions (world-readable). Any unprivileged local user on the same machine where bootstrapping is performed can read these files and obtain the node's private keys.

### Finding Description
The central write utility `WriteText` in `cmd/util/cmd/common/utils.go` unconditionally uses `os.FileMode(0644)` for every file it creates:

```go
err = os.WriteFile(path, data, 0644)
```

`WriteJSON` delegates to `WriteText`, so every JSON-serialized private key file inherits this permission. The bootstrap `key` command (`cmd/bootstrap/cmd/key.go`) calls these functions to write:

- `PathNodeInfoPriv` — contains the node's private staking key and networking key
- `PathSecretsEncryptionKey` — the AES-256 key used to encrypt the node's secrets Badger DB
- `PathNodeMachineAccountPrivateKey` — the machine account private key (Collection/Consensus nodes)
- `PathRandomBeaconPriv` — the random beacon private key (Consensus nodes, via `cmd/bootstrap/cmd/dkg.go`)

A second independent instance exists in `cmd/bootstrap/transit/cmd/utils.go`:

```go
const fileMode = os.FileMode(0644)
```

This constant is used in `generateKeys` to write the transit NaCl private key (`FilenameTransitKeyPriv`) and in `unWrapFile` to write the decrypted random beacon private key to disk — both with `0644`.

By contrast, `cmd/bootstrap/cmd/observer_network_key.go` and `cmd/bootstrap/cmd/access_keygen.go` correctly use `0600` for their respective private key files, demonstrating that the correct permission is known but inconsistently applied.

### Impact Explanation
An unprivileged local user on the bootstrapping machine can read:

- **Staking private key**: enables forging BLS signatures on behalf of the node, allowing the attacker to submit fraudulent staking operations or equivocate.
- **Networking private key**: enables impersonating the node's libp2p identity on the network.
- **Random beacon private key**: enables the attacker to contribute a known share to the DKG threshold signature, potentially biasing or predicting randomness used in leader/collector selection.
- **Machine account private key**: enables draining the on-chain machine account (real FLOW tokens) or submitting unauthorized transactions from that account.
- **Secrets DB encryption key**: enables decrypting the node's entire secrets database at rest.

### Likelihood Explanation
Bootstrapping is routinely performed on shared CI/CD servers, cloud VMs with multiple operator accounts, or Docker build hosts. The `0644` permission is the default for non-sensitive files and is easy to overlook. No special privileges are required to exploit this — any local user with shell access to the machine can `cat` the key files immediately after bootstrap completes.

### Recommendation
Change `WriteText` in `cmd/util/cmd/common/utils.go` to use `0600` instead of `0644`. Since `WriteJSON` delegates to `WriteText`, this single change protects all JSON-serialized private key files written through the common utility. Additionally, change `fileMode` in `cmd/bootstrap/transit/cmd/utils.go` from `0644` to `0600` to protect the transit private key and the decrypted random beacon key.

### Proof of Concept

```bash
# On a shared machine, operator runs bootstrap key:
$ go run ./cmd/bootstrap key --address "node.example.com:3569" --role consensus -o ./bootstrap

# Another local user immediately reads the private key:
$ cat ./bootstrap/private-root-information/private-node-info_<NodeID>/node-info.priv.json
# => Full staking + networking private keys exposed

$ cat ./bootstrap/private-root-information/private-node-info_<NodeID>/secretsdb-key
# => AES-256 secrets DB encryption key exposed

$ cat ./bootstrap/private-root-information/private-node-info_<NodeID>/random-beacon.priv.json
# => Random beacon private key share exposed
```
