### Title
Pool Rewards Permanently Frozen When Pool Balance Is Below `min_delegation_for_rewards` - (`src/pool/pool.cairo`)

### Summary

When a delegation pool's total balance is below `min_delegation_for_rewards` (1 STRK = 10^18 wei), the staking contract still calculates and transfers non-zero STRK reward tokens to the pool contract, but the pool contract's `update_rewards_from_staking_contract` silently drops the accounting update by returning zero from `compute_rewards_per_unit`. The transferred STRK tokens are permanently stuck in the pool contract with no recovery mechanism.

### Finding Description

The root cause is a sequencing mismatch between token transfer and accounting in `update_pool_rewards` in `staking.cairo`, combined with a silent zero-return guard in `compute_rewards_per_unit` in `pool.cairo`.

**Step 1 — Rewards are calculated as non-zero even for small pool balances.**

In `calculate_staker_pools_rewards`, pool rewards are computed proportionally to `pool_balance_curr_epoch` with no minimum balance guard: [1](#0-0) 

If `pool_balance_curr_epoch` is, say, 0.5 STRK (5×10^17 wei) and `total_rewards` is large enough, `pool_rewards_including_commission` is non-zero. After commission split, `pool_rewards` is non-zero and is appended to `pool_rewards_array`.

**Step 2 — Tokens are transferred to the pool contract unconditionally.**

In `update_pool_rewards`, STRK tokens are sent to the pool contract *before* calling `update_rewards_from_staking_contract`: [2](#0-1) 

**Step 3 — The pool contract silently drops the accounting update.**

Inside `update_rewards_from_staking_contract`, `compute_rewards_per_unit` is called with `total_stake = pool_balance`: [3](#0-2) 

`compute_rewards_per_unit` returns zero when `total_stake < min_delegation_for_rewards`: [4](#0-3) 

For STRK, `min_delegation_for_rewards = 10^18` (1 STRK): [5](#0-4) 

The `cumulative_rewards_trace` is updated with a zero increment, so no rewards are ever attributed to pool members.

**Step 4 — No recovery path exists.**

Pool member rewards are computed exclusively from `cumulative_rewards_trace` in `calculate_rewards`: [6](#0-5) 

Since the trace was not updated, `claim_rewards` returns zero for pool members. There is no admin withdrawal or sweep function in the pool contract to recover the stuck STRK tokens.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

STRK reward tokens are transferred into the pool contract but are irrecoverable. Pool members who delegated while the total pool balance was below 1 STRK permanently lose their proportional share of rewards for every epoch in that state. The pool contract has no privileged recovery function.

### Likelihood Explanation

`enter_delegation_pool` enforces only `amount.is_non_zero()` with no minimum delegation floor: [7](#0-6) 

Any pool whose cumulative delegated balance is between 1 wei and 10^18 − 1 wei (i.e., less than 1 STRK) triggers this path. This is reachable by any unprivileged delegator who enters a new pool with a small amount, or by a pool that has had most of its balance withdrawn via `exit_delegation_pool_intent` leaving a residual below 1 STRK.

### Recommendation

In `update_pool_rewards` (or in `calculate_staker_pools_rewards`), skip both the token transfer and the accounting call when `pool_balance.to_native_amount(:decimals) < min_delegation_for_rewards`. The check should mirror the guard already present in `compute_rewards_per_unit` so that no tokens are sent to a pool whose balance is too small to record the rewards:

```cairo
// Before transferring, check if pool_balance meets the minimum threshold.
let native_pool_balance = pool_balance.to_native_amount(:decimals);
if native_pool_balance < pool_dispatcher.get_min_delegation_for_rewards() {
    continue; // skip transfer and accounting for this pool
}
self.send_rewards_to_delegation_pool(...);
pool_dispatcher.update_rewards_from_staking_contract(...);
```

Alternatively, expose `min_delegation_for_rewards` as a view on the pool contract and gate the entire reward distribution path in the staking contract on that threshold.

### Proof of Concept

1. Staker calls `stake` and then `set_open_for_delegation` to create a STRK pool.
2. Pool member calls `enter_delegation_pool` with `amount = 5 * 10^17` (0.5 STRK).
3. Staker attests; `update_rewards_from_attestation_contract` is triggered.
4. `calculate_staker_pools_rewards` computes `pool_rewards > 0` (proportional to 0.5 STRK).
5. `update_pool_rewards` calls `send_rewards_to_delegation_pool`, transferring STRK to the pool contract.
6. `update_rewards_from_staking_contract` is called with `pool_balance = 5 * 10^17 < 10^18`.
7. `compute_rewards_per_unit` returns `0`; `cumulative_rewards_trace` is updated with zero increment.
8. Pool member calls `claim_rewards` → receives `0`.
9. The STRK tokens transferred in step 5 remain permanently locked in the pool contract.

        ) {

            let (_, last) = self.cumulative_rewards_trace.last().unwrap();
            self
                .cumulative_rewards_trace
                .insert(
                    key: self.get_current_epoch(),
                    value: last
                        + self
                            .compute_rewards_per_unit(
                                staking_rewards: rewards, total_stake: pool_balance,
                            ),
                );
        }

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
