### Title
`switch_delegation_pool()` Lacks Commission Slippage Protection, Allowing Staker to Front-Run Delegator with Commission Increase - (File: src/pool/pool.cairo)

### Summary

`switch_delegation_pool()` accepts only `to_staker`, `to_pool`, and `amount` as parameters. It contains no check on the destination pool's commission rate. Because a staker holding an active `commission_commitment` can legitimately increase their commission at any time (up to `max_commission`), a delegator's switch transaction can execute against a materially different commission than the one observed when the decision was made, causing the delegator to lose future yield to the staker.

### Finding Description

The `switch_delegation_pool` function in `src/pool/pool.cairo` allows a pool member to move their delegated stake from one staker's pool to another:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
) -> Amount {
``` [1](#0-0) 

The function performs no check on the commission of the destination pool. Commission is the key economic parameter a delegator uses to choose a pool.

The staking contract's `update_commission` internal function explicitly permits commission **increases** when an active `commission_commitment` is in effect:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
``` [2](#0-1) 

The code comment on `set_commission_commitment` explicitly acknowledges this: *"Current commission increase safeguards still allow for sudden commission changes."* [3](#0-2) 

A staker can therefore set a `commission_commitment` with `max_commission` well above their current commission, then increase commission to `max_commission` at any time — including immediately before a delegator's `switch_delegation_pool` transaction is sequenced.

### Impact Explanation

A delegator who switches to a pool expecting commission `C` may instead be placed in a pool with commission `C_max > C`. The delegator is then locked in that pool for at least `exit_wait_window` (default 1 week, up to 12 weeks) before they can exit. During this window, the staker collects a higher commission share of all rewards earned by the delegator's stake. This constitutes theft of unclaimed yield: yield that would have accrued to the delegator is instead captured by the staker via the elevated commission.

**Impact: High — Theft of unclaimed yield.**

### Likelihood Explanation

The attack requires:
1. The staker has set a `commission_commitment` with `max_commission` above their current commission (a publicly visible, on-chain action).
2. The delegator submits `switch_delegation_pool` targeting that staker's pool.
3. The staker calls `set_commission` to raise commission to `max_commission` before the delegator's transaction is sequenced.

On Starknet, transaction ordering is controlled by the sequencer. A staker who is also a sequencer operator, or who monitors the mempool, can reliably front-run the delegator. Even without mempool visibility, a staker can raise commission speculatively at any time, catching any in-flight switch transactions. The `commission_commitment` mechanism is a standard feature that many stakers will use, making the precondition common.

### Recommendation

Add a `max_commission` parameter to `switch_delegation_pool` (and symmetrically to `enter_delegation_pool`) and assert that the destination pool's current commission does not exceed it at execution time:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
+   max_commission: Commission,
) -> Amount {
    ...
+   let current_commission = self.get_commission_from_staking_contract_for(to_staker);
+   assert!(current_commission <= max_commission, Error::COMMISSION_EXCEEDS_MAX);
    ...
}
```

The same protection should be applied to `enter_delegation_pool`.

### Proof of Concept

1. Staker S deploys a pool with commission 5% (500 bps).
2. Staker S calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current + 50)`. [4](#0-3) 
3. Delegator D observes commission = 5% in the UI and calls `exit_delegation_pool_intent` on their current pool, then submits `switch_delegation_pool(to_staker: S, to_pool: S_pool, amount: X)`.
4. Before D's transaction is sequenced, S calls `set_commission(commission: 10000)` — valid because `10000 <= max_commission` and `10000 != 500`. [5](#0-4) 
5. D's `switch_delegation_pool` executes. The function reads no commission, performs no check, and completes successfully. [6](#0-5) 
6. D is now a member of S's pool at 100% commission. All of D's future rewards are taken by S until D completes the full `exit_delegation_pool_intent` → wait `exit_wait_window` → `exit_delegation_pool_action` cycle. [7](#0-6)

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }
