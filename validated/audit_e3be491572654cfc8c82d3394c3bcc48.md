### Title
Pool Rewards Permanently Frozen When `pool_balance < min_delegation_for_rewards` — (File: src/pool/pool.cairo)

---

### Summary

When the total delegated balance in a pool falls below `min_delegation_for_rewards`, `Pool.compute_rewards_per_unit` returns zero and the cumulative rewards index is not advanced. However, the staking contract has already transferred the calculated STRK reward tokens to the pool contract. Those tokens are permanently frozen: no pool member can claim them and no admin recovery path exists.

---

### Finding Description

**Root cause — `Pool.compute_rewards_per_unit`:**

```rust
fn compute_rewards_per_unit(
    self: @ContractState, staking_rewards: Amount, total_stake: Amount,
) -> Index {
    // Return zero if the total stake is too small, to avoid overflow below.
    if total_stake < self.min_delegation_for_rewards.read() {
        return Zero::zero();   // ← rewards silently discarded from the index
    }
    mul_wide_and_div(
        lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
    )
    .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
}
``` [1](#0-0) 

The code comment directly above this function acknowledges the consequence:

> **Note**: Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case. [2](#0-1) 

**The mismatch — staking contract still transfers tokens:**

In `_update_rewards`, the staking contract computes `pool_rewards` as non-zero whenever `pool_balance_curr_epoch > 0` and `total_stake > 0`, regardless of whether `pool_balance` is below the pool's minimum threshold:

```rust
let pool_rewards_including_commission = if total_stake.is_non_zero() {
    mul_wide_and_div(
        lhs: total_rewards,
        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
        div: total_stake.to_amount_18_decimals(),
    )
    ...
} else {
    Zero::zero()
};
``` [3](#0-2) 

Then `update_pool_rewards` unconditionally transfers STRK to the pool contract and calls `update_rewards_from_staking_contract`:

```rust
self.send_rewards_to_delegation_pool(
    :staker_address, pool_address: pool_contract, amount: pool_rewards, ...
);
pool_dispatcher.update_rewards_from_staking_contract(
    rewards: pool_rewards,
    pool_balance: pool_balance.to_native_amount(:decimals),
);
``` [4](#0-3) 

Inside `update_rewards_from_staking_contract`, the cumulative trace is updated with `last + compute_rewards_per_unit(...)`. When `pool_balance < min_delegation_for_rewards`, `compute_rewards_per_unit` returns 0, so the trace entry becomes `last + 0 = last` — the index does not advance:

```rust
fn update_rewards_from_staking_contract(
    ref self: ContractState, rewards: Amount, pool_balance: Amount,
) {
    self.assert_caller_is_staking_contract();
    let (_, last) = self.cumulative_rewards_trace.last().unwrap();
    self.cumulative_rewards_trace.insert(
        key: self.get_current_epoch(),
        value: last + self.compute_rewards_per_unit(staking_rewards: rewards, total_stake: pool_balance),
    );
}
``` [5](#0-4) 

Because all delegator reward claims (`claim_rewards`, `exit_delegation_pool_action`) derive their payout from the cumulative index delta, and the index was never advanced, the transferred STRK tokens are permanently unclaimable.

**Threshold values:**

For STRK: `min_delegation_for_rewards = 10^18` (1 STRK).
For BTC-8: `min_delegation_for_rewards = 10^3` (1000 satoshis). [6](#0-5) 

---

### Impact Explanation

STRK tokens are transferred from the reward supplier into the pool contract but the cumulative rewards index is not updated. No pool member can ever claim those tokens. There is no admin sweep or recovery function in the pool contract. The frozen amount grows with every epoch in which the pool balance remains below the minimum. This constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

The condition `0 < pool_balance < min_delegation_for_rewards` is reachable by any unprivileged delegator:

- A pool starts with two delegators each holding 0.6 STRK (total 1.2 STRK ≥ 1 STRK minimum). One delegator calls `exit_delegation_pool_intent` then `exit_delegation_pool_action`, leaving 0.6 STRK < 1 STRK in the pool.
- From that epoch onward, every attestation/reward cycle transfers STRK to the pool contract but advances the index by zero.
- The remaining delegator's new rewards are permanently frozen.

No privileged access is required. The exit functions are standard delegator operations callable by any pool member.

---

### Recommendation

Align the token-transfer decision with the index-update decision. Specifically, in `calculate_staker_pools_rewards` (or in `update_pool_rewards`), skip the STRK transfer and the `update_rewards_from_staking_contract` call when `pool_balance.to_native_amount(:decimals) < min_delegation_for_rewards`. Alternatively, credit the undeliverable pool rewards back to the staker or to a recoverable admin reserve rather than sending them to the pool contract where they become unclaimable.

---

### Proof of Concept

1. Deploy the system; staker S creates a STRK delegation pool.
2. Delegator D delegates 0.6 STRK (6 × 10¹⁷ wei) — below `min_delegation_for_rewards = 10¹⁸`.
3. Advance K epochs so the delegation becomes effective.
4. Staker S attests → `update_rewards_from_attestation_contract` is called.
5. Inside `calculate_staker_pools_rewards`: `pool_balance_curr_epoch = 6×10¹⁷`, `total_stake > 0` → `pool_rewards > 0`.
6. `update_pool_rewards` transfers `pool_rewards` STRK to the pool contract.
7. `pool.update_rewards_from_staking_contract(rewards: pool_rewards, pool_balance: 6×10¹⁷)` is called.
8. `compute_rewards_per_unit`: `6×10¹⁷ < 10¹⁸` → returns `0`.
9. Cumulative trace entry: `last + 0 = last` — index unchanged.
10. Delegator D calls `claim_rewards` → receives 0 (index delta = 0).
11. STRK tokens are permanently locked in the pool contract with no recovery path. [5](#0-4) [1](#0-0) [3](#0-2) [4](#0-3)
