### Title
WalletConnect Offer Approval Displays Underestimated Royalty Total for Multi-NFT Offers - (`packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary
`formatAmountWithRoyalties` in `parseCommandDisplay.ts` incorrectly divides the fungible amount by the number of NFTs before computing each NFT's royalty, causing the WalletConnect approval dialog to display a total-with-royalties that is systematically lower than the amount the wallet will actually deduct. A user approving a multi-NFT offer via WalletConnect is shown a misleadingly low cost and may approve a spend they would otherwise reject.

### Finding Description

`formatAmountWithRoyalties` is called during WalletConnect `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` command parsing to compute the `amountWithRoyalties` field shown in the approval dialog. [1](#0-0) 

The function first splits the total fungible amount equally across all NFTs:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← wrong
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [2](#0-1) 

Chia's actual royalty protocol (as implemented in `nft_calculate_royalties` and consumed by `OfferBuilderTokensSection`) calculates each NFT's royalty on the **full** fungible amount, then sums them: [3](#0-2) 

**Concrete example** (from the existing test at line 251):
- 2 NFTs, 100,000,000 mojos (0.0001 XCH), royalties 500 (5%) and 10 (0.1%)
- **Displayed** (split logic): `splitAmount = 50,000,000`; royalties = 2,500,000 + 50,000 = 2,550,000; total shown = **0.00010255 XCH**
- **Actual protocol**: royalties = 5,000,000 + 100,000 = 5,100,000; true total = **0.0001051 XCH** [4](#0-3) 

The underestimation factor is `N` (number of NFTs): with 3 NFTs the displayed royalty is one-third of the actual royalty. The `amountWithRoyalties` field is surfaced directly in the WalletConnect confirmation UI via `withRoyaltyTotals` → `walletDeltaToDisplay`. [5](#0-4) 

### Impact Explanation

The WalletConnect approval dialog shows the user a `amountWithRoyalties` value that is lower than what the Chia wallet daemon will actually deduct when the offer is executed. A user who relies on this figure to decide whether to approve the offer is deceived into approving a spend larger than displayed. With high-royalty NFTs and several NFTs per offer, the gap can be substantial (e.g., 5× underestimate with 5 NFTs each at 5% royalty). This constitutes display of the wrong amount in a WalletConnect signing-approval context, directly matching the High impact class.

### Likelihood Explanation

WalletConnect multi-NFT offers are a normal use case for NFT marketplace dApps. Any dApp (including a malicious one) can craft a `take_offer` payload with multiple high-royalty NFTs. The user has no other in-dialog signal of the true cost; the `amountWithRoyalties` field is the only royalty-inclusive total shown. No special privileges or key access are required.

### Recommendation

Remove the `splitAmount` division. Each NFT's royalty must be calculated on the full fungible amount:

```typescript
// Correct: each NFT royalty is on the full amount
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [6](#0-5) 

### Proof of Concept

The existing test at line 251 of `parseCommandDisplay.test.ts` already demonstrates the discrepancy: it asserts `amountWithRoyalties: '0.00010255'` (the split-based result), while the correct protocol value is `0.0001051`. The test passes only because it was written to match the buggy implementation. [7](#0-6)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}
