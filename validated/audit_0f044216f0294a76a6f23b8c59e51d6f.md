### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Block Rewards - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is specified to be callable only by the Starkware sequencer, but the implementation contains no such access control check. Any unprivileged caller can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` cursor without distributing any rewards, permanently destroying the reward entitlement for that block for all stakers and delegators.

### Finding Description

The specification at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 enforces only `general_prerequisites()`, which at lines 1794–1797 checks only that the contract is unpaused and that the caller is not the zero address. There is no check that the caller is the sequencer.

The function accepts a caller-controlled `disable_rewards: bool` parameter. When `true`, the function:
1. Validates the staker (lines 1466–1482).
2. **Unconditionally writes `current_block_number` to the global `last_reward_block` storage variable** (line 1485).
3. Returns early without computing or distributing any rewards (lines 1487–1489).

Because `last_reward_block` is a single global slot (not per-staker), the subsequent guard at lines 1454–1458 (`assert!(current_block_number > self.last_reward_block.read(), Error::REWARDS_ALREADY_UPDATED)`) will revert any further call to `update_rewards` in the same block — including the legitimate sequencer call. The rewards that would have been distributed for that block are permanently lost; there is no mechanism to retroactively credit them.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

For every block in which an attacker front-runs the sequencer's `update_rewards` call with `disable_rewards: true`, the entire block's reward allocation for the targeted staker (and all delegators in its pools) is irrecoverably destroyed. An attacker willing to pay gas can repeat this every block, continuously suppressing reward accrual across the entire protocol. Delegators' unclaimed STRK yield is permanently frozen with no recovery path.

### Likelihood Explanation

**High.** The function is publicly callable with no role restriction. The only cost to the attacker is gas per block. On Starknet, transaction fees are low, making sustained griefing economically feasible. The attacker needs only to know any valid active staker address (publicly observable on-chain) and submit a transaction before the sequencer's reward-update transaction in each block.

### Recommendation

Enforce the access control stated in the specification. Add a check at the top of `update_rewards` that asserts the caller is the designated sequencer address, analogous to the guard already present in `update_rewards_from_attestation_contract` (which asserts `CALLER_IS_NOT_ATTESTATION_CONTRACT`). Store the sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation.

### Proof of Concept

1. Staker `S` is active and eligible for block rewards in block `B`.
2. Attacker observes the mempool (or simply submits at the start of block `B`) and calls:
   ```
   update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `general_prerequisites()` passes (contract is unpaused, caller ≠ zero).
4. Staker `S` is found active with non-zero balance — all assertions pass.
5. `last_reward_block` is written to `B` (line 1485).
6. Function returns early — no rewards computed or transferred (lines 1487–1489).
7. The sequencer later attempts `update_rewards(staker_address: S, disable_rewards: false)` in block `B`.
8. The guard at line 1454 fires: `B > B` is false → `REWARDS_ALREADY_UPDATED` revert.
9. Block `B`'s reward for staker `S` and all its pool delegators is permanently lost.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
