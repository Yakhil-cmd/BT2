### Title
Missing Caller Authorization on `update_rewards` Allows Any Address to Permanently Freeze Consensus Rewards - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the `Staking` contract is documented to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` every block, consuming the global `last_reward_block` slot without distributing rewards, permanently preventing the legitimate sequencer from ever distributing consensus-era staking rewards.

### Finding Description

The spec explicitly restricts `update_rewards` to the sequencer:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. After `general_prerequisites()` (which only asserts the contract is unpaused), the function immediately writes to the global `last_reward_block` storage variable:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no get_caller_address() check anywhere ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distribute rewards only if we reach here
``` [2](#0-1) 

`last_reward_block` is a **global** (not per-staker) field. A single call to `update_rewards` for any valid staker address in block N sets `last_reward_block = N`, causing every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block. This:

1. Passes all checks (contract unpaused, block number advances, staker exists and is active).
2. Writes `last_reward_block = current_block_number` with no rewards distributed.
3. Causes the legitimate sequencer's call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

Because the attacker can repeat this every block at negligible cost, **no consensus rewards are ever distributed to any staker or pool member**. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The function is part of the public ABI (`IStakingRewardsManager`), callable by any EOA or contract.
- Valid staker addresses are publicly observable from on-chain events and the `stakers` storage vector.
- The attack requires no funds, no special role, and no prior state — just one transaction per block.
- The `disable_rewards` parameter is caller-supplied, so the attacker trivially sets it to `true`.

### Recommendation

Add a sequencer-address check at the top of `update_rewards`, analogous to the existing pattern used for the attestation contract:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
}
```

Store the authorized sequencer address in contract storage (set during construction or via a governance call), and enforce it as the first caller check. [4](#0-3) 

### Proof of Concept

1. Deploy the staking system and advance past `consensus_rewards_first_epoch`.
2. In every block, call:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(
       staker_address = <any active staker>,
       disable_rewards = true
   )
   ```
3. Observe that `last_reward_block` is updated to the current block number but `unclaimed_rewards_own` for the staker remains zero.
4. The sequencer's legitimate `update_rewards` call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat for every block — stakers and pool members accumulate zero consensus rewards indefinitely. [5](#0-4) [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
