### Title
Missing Validation of `app_pk` Allows Coordinator to Extract Confidential Derived Key Directly - (File: src/confidential_key_derivation/protocol.rs)

### Summary

The `ckd()` function in `src/confidential_key_derivation/protocol.rs` accepts the application ElGamal public key `app_pk: PublicKey` (type `blstrs::G1Projective`) without checking whether it is the group identity element. If `app_pk` is the identity, the ElGamal masking term `y * app_pk` collapses to the identity, causing every participant's ciphertext component `big_c` to equal their raw secret share contribution `big_s`. The coordinator then directly holds the unmasked confidential derived key `msk * H(pk || app_id)` in `CKDOutput.big_c` without needing the TEE's secret scalar `a`.

### Finding Description

The CKD protocol is designed so that each participant computes:

```
Y_i = y_i * G
S_i = x_i * H(pk || app_id)
C_i = S_i + y_i * app_pk
```

and sends `(λ_i * Y_i, λ_i * C_i)` to the coordinator. The coordinator sums these to obtain `(Y_total, C_total)` where:

```
C_total = msk * H(pk || app_id) + app_pk * Y_total
```

The TEE then recovers the confidential key as `C_total - a * Y_total = msk * H(pk || app_id)`, where `a` is the TEE's secret scalar corresponding to `app_pk = a * G`.

**Root cause:** The `ckd()` entry point performs no check that `app_pk` is a valid, non-identity group element: [1](#0-0) 

The `compute_signature_share` function uses `app_pk` directly in the masking step: [2](#0-1) 

If `app_pk = G1Projective::identity()`, then `app_pk * y.0 = identity`, so `big_c = big_s = hash_point * private_share.to_scalar()`. After the coordinator sums all Lagrange-weighted contributions:

```
C_total = Σ λ_i * S_i = H(pk || app_id) * Σ λ_i * x_i = msk * H(pk || app_id)
```

The coordinator's `CKDOutput.big_c` is exactly the confidential derived key, with no masking. The coordinator does not need to call `unmask(a)` — it reads `big_c` directly.

By contrast, the `BLS12381G1Group::deserialize()` implementation does reject the identity element when deserializing from bytes: [3](#0-2) 

But `app_pk` is passed as a native `G1Projective` value, bypassing that path entirely. The `PublicKey` type alias confirms there is no wrapper type enforcing non-identity: [4](#0-3) 

### Impact Explanation

**Critical — Extraction/disclosure of a confidential derived secret.**

When `app_pk = identity`, the coordinator receives `CKDOutput { big_y: Y_total, big_c: msk * H(pk || app_id) }`. The field `big_c` is the confidential derived key in plaintext. The coordinator extracts it without possessing the TEE's secret scalar `a`, completely defeating the ElGamal confidentiality guarantee that is the protocol's sole privacy mechanism. [5](#0-4) 

### Likelihood Explanation

The `app_pk` parameter is supplied by the caller at each participant's invocation of `ckd()`. In a realistic deployment the coordinator distributes `app_pk` to all participants (it is a public parameter for the session). A malicious coordinator can trivially substitute `G1Projective::identity()` for the legitimate application public key. No cryptographic break, no leaked secret, and no privileged access beyond the coordinator role (which is an explicitly documented trust boundary) is required.

### Recommendation

- **Short term:** Add an explicit identity check in `ckd()` before the protocol runs:
  ```rust
  if app_pk.is_identity().into() {
      return Err(InitializationError::BadParameters(
          "app_pk must not be the identity element".to_string(),
      ));
  }
  ```
- **Long term:** Introduce a validated `AppPublicKey` newtype that enforces non-identity at construction, mirroring the pattern already used in `BLS12381G1Group::deserialize()`, so the invariant is enforced at the type level rather than at each call site.

### Proof of Concept

```rust
use blstrs::G1Projective;
use threshold_signatures::confidential_key_derivation::{
    protocol::ckd, AppId, ElementG1,
};

// Malicious coordinator passes the identity as app_pk
let identity_pk: ElementG1 = G1Projective::identity();

// ckd() accepts it without error
let protocol = ckd(
    &participants,
    coordinator,
    me,
    key_pair,
    app_id,
    identity_pk,   // <-- identity, no validation triggered
    rng,
).unwrap();  // succeeds

// After running the protocol, coordinator's CKDOutput.big_c
// equals msk * H(pk || app_id) directly.
// coordinator reads big_c without calling unmask(a).
let ckd_output = /* run protocol */;
let confidential_key_plaintext = ckd_output.big_c(); // = msk * H(pk || app_id)
```

The missing check is directly analogous to the external report: just as `MultiSigWallet._externalCall` silently succeeds when the destination is not a contract, `ckd()` silently accepts an identity `app_pk` and produces a protocol output that appears successful but has completely bypassed the confidentiality guarantee, handing the coordinator the protected secret in plaintext.

    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    let comms = Comms::new();
    let chan = comms.shared_channel();

    let fut = run_ckd_protocol(
        chan,
        coordinator,
        me,
        participants,
        key_pair,
        app_id.into(),
        app_pk,
        rng,
    );
    Ok(make_protocol(comms, fut))
}
