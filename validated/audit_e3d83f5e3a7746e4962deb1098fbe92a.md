### Title
Unbounded `pool_member_epoch_balance` Trace Growth Causes Permanent Freezing of Unclaimed Yield via Gas Exhaustion in `claim_rewards` - (File: `src/pool/pool.cairo`)

---

### Summary

Every call to `add_to_delegation_pool` or `exit_delegation_pool_intent` in a new epoch appends a new checkpoint to the per-delegator `pool_member_epoch_balance` trace. The `calculate_rewards` function iterates over **all** accumulated trace entries since the last claim in an unbounded loop. A delegator who makes balance changes across many epochs without claiming rewards will eventually cause `claim_rewards` to exceed Starknet's per-transaction gas limit, permanently freezing their unclaimed yield with no recovery path.

---

### Finding Description

`set_member_balance` is called by both `add_to_delegation_pool` and `exit_delegation_pool_intent`. It inserts a checkpoint keyed by `current_epoch + K` into the `pool_member_epoch_balance` trace: [1](#0-0) 

The underlying `PoolMemberBalanceTrace::insert` only deduplicates if the key (epoch) is identical to the last entry; otherwise it unconditionally appends a new checkpoint: [2](#0-1) 

When `claim_rewards` is called, it invokes `calculate_rewards`, which iterates over every trace entry from `entry_to_claim_from` up to the current epoch in an explicitly acknowledged unbounded loop: [3](#0-2) 

The developers themselves note the risk: *"This loop is unbounded but unlikely to exceed gas limits."* There is no cap on trace length, no mechanism to prune old entries, and no way to claim rewards for a partial epoch range — `claim_rewards` accepts only `pool_member` as a parameter: [4](#0-3) 

After each successful claim, `entry_to_claim_from` advances, so the loop only covers entries since the last claim. However, if a delegator makes balance changes across N epochs without claiming, the next `claim_rewards` call iterates N times, each requiring multiple storage reads from `pool_member_epoch_balance` and `cumulative_rewards_trace`.

---

### Impact Explanation

Once the trace grows large enough that a single `claim_rewards` call exceeds Starknet's per-transaction gas limit, the delegator's accumulated rewards are **permanently frozen**. There is no alternative entry point to claim rewards in smaller batches, no admin recovery function, and no way to prune the trace. The delegator's principal remains accessible (via `exit_delegation_pool_action`), but all unclaimed yield is irrecoverably lost.

**Impact: High — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

- The minimum cost per trace entry is 1 wei of STRK plus gas for one `add_to_delegation_pool` call.
- A delegator who actively manages their stake (e.g., topping up or partially exiting each epoch) over months or years accumulates entries naturally.
- Starknet epochs are short (on the order of hours based on block times), so hundreds to thousands of entries can accumulate within a year of normal use.
- There is no protocol-level warning or cap to prevent this.
- The attack is also achievable deliberately: a delegator calls `add_to_delegation_pool` with 1 wei every epoch, accumulating entries at negligible cost until `claim_rewards` is bricked.

**Likelihood: Medium.**

---

### Recommendation

1. **Enforce a maximum trace length** per pool member (e.g., cap at 256 entries) and require the delegator to call `claim_rewards` before making further balance changes once the cap is reached.
2. **Alternatively**, add a `claim_rewards_until(epoch: Epoch)` entry point that allows partial reward claims up to a specified epoch, so a delegator with a large trace can drain it incrementally.
3. **Automatically checkpoint rewards** (i.e., force a `claim_rewards` internally) whenever `set_member_balance` is called, so the trace never grows beyond 2–3 entries at a time.

---

### Proof of Concept

```
1. Delegator enters pool with amount A via `enter_delegation_pool`.
   → trace length = 1 (entry at epoch E+K)

2. For each subsequent epoch E_i (i = 1 … N), delegator calls:
       pool.add_to_delegation_pool(pool_member, amount: 1)
   Each call is in a new epoch, so `get_epoch_plus_k()` returns a new key,
   and `PoolMemberBalanceTrace::insert` appends a new checkpoint.
   → trace length grows to N+1

3. Delegator never calls `claim_rewards` during this period.
   `entry_to_claim_from` remains at 0.

4. After N epochs, delegator calls `claim_rewards(pool_member)`.
   → `calculate_rewards` enters the while loop:
      while entry_to_claim_from (=0) < pool_member_trace_length (=N+1)
   → iterates N times, each reading from `pool_member_epoch_balance`
      and calling `find_sigma` (which reads `cumulative_rewards_trace`)

5. For sufficiently large N (e.g., N ≈ 5,000–10,000 depending on Starknet
   gas pricing), the transaction exceeds the block gas limit and reverts.

6. Every subsequent `claim_rewards` call also reverts (same loop, same length).
   The delegator's accumulated yield is permanently frozen.
   No recovery path exists in the contract.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
