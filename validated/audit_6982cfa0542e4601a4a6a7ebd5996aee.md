### Title
Pool Contract Lacks Pause Mechanism, Allowing Reward Drainage to Bypass Emergency Stop — (`src/pool/pool.cairo`)

---

### Summary

The `Staking` contract implements a full pause/unpause mechanism guarded by `Security Agent` / `Security Admin` roles. However, the `Pool` contract — which independently holds delegator funds and distributes STRK rewards — has **no pause mechanism at all**. Critically, `Pool::claim_rewards` transfers STRK directly without ever calling into the `Staking` contract, so it is completely unaffected when `Staking` is paused. Any delegator can drain accrued (potentially exploit-inflated) rewards from the Pool even after an emergency pause has been activated on Staking.

---

### Finding Description

The `Staking` contract stores an `is_paused` flag and exposes `pause()` / `unpause()` functions: [1](#0-0) 

The wiki confirms that `stake`, `increase_stake`, `claim_rewards`, and `unstake_intent` on **Staking** are blocked when paused. However, the `Pool` contract embeds `RolesComponent` but defines **no** `is_paused` storage slot, no `pause()` / `unpause()` entry points, and no `assert_not_paused` guard anywhere: [2](#0-1) 

The most critical unguarded function is `Pool::claim_rewards`. It reads from the pool's local `cumulative_rewards_trace`, computes rewards, and transfers STRK **directly** via an ERC-20 dispatcher — it never calls back into the `Staking` contract: [3](#0-2) 

The direct STRK transfer at the end: [4](#0-3) 

Because this path is entirely self-contained within `Pool`, pausing `Staking` has zero effect on it.

Additionally, `Pool::exit_delegation_pool_action` — when the staker has already been removed — also transfers tokens directly without touching `Staking`: [5](#0-4) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

If an exploit inflates the `cumulative_rewards_trace` (e.g., via a bug in `update_rewards_from_staking_contract` or a reward-accounting vulnerability), the Security Agent can pause `Staking` to halt further damage. However, because `Pool::claim_rewards` is unguarded, every delegator (or the attacker themselves, if they hold a pool position) can immediately drain the inflated rewards from the Pool contract before any remediation is applied. The emergency pause is rendered partially ineffective: the attack surface on `Staking` is closed, but the reward-draining path through `Pool` remains wide open.

---

### Likelihood Explanation

**Medium.** The entry path requires only a standard delegator account — no privileged role, no leaked key. Any address that has previously called `enter_delegation_pool` can call `Pool::claim_rewards` at any time. The scenario is realistic: the Staking protocol is a high-value target, and the asymmetry between a paused Staking contract and an unpaused Pool contract is a predictable gap that an attacker aware of the architecture would exploit immediately after triggering a reward-inflation bug.

---

### Recommendation

1. Add an `is_paused` storage slot and `pause()` / `unpause()` functions to `Pool`, mirroring the pattern in `Staking`.
2. Add an `assert_not_paused` guard at the top of every state-changing user-facing function in `Pool`: `enter_delegation_pool`, `add_to_delegation_pool`, `exit_delegation_pool_intent`, `exit_delegation_pool_action`, `claim_rewards`, and `switch_delegation_pool`.
3. Consider a cross-contract pause propagation: when `Staking` is paused, it could call `pause()` on all associated Pool contracts, or Pool could query `Staking::is_paused()` as a guard.

---

### Proof of Concept

1. Attacker or any delegator holds a position in a Pool contract (called `enter_delegation_pool` previously).
2. An exploit inflates `cumulative_rewards_trace` in the Pool (e.g., via a bug in reward accounting).
3. Security Agent calls `Staking::pause()` — `Staking.is_paused` is set to `true`.
4. Attacker immediately calls `Pool::claim_rewards(attacker_address)`.
5. `Pool::claim_rewards` reads the inflated `cumulative_rewards_trace`, computes a large reward, and calls `STRK_TOKEN.transfer(reward_address, inflated_amount)` — **no pause check is ever reached**.
6. Inflated STRK rewards are transferred to the attacker's reward address. The emergency pause on `Staking` did not prevent this. [3](#0-2)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
