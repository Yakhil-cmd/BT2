### Title
Stale-Pending Fallback Synthesizes `DeprecatedPendingBlock` With Zero `l1_data_gas_price`, Causing `estimate_fee` to Return `NonzeroGasPrice::MIN` (1) Instead of the Real Data Gas Price — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

When the in-memory pending data is stale (its `parent_block_hash` does not match the latest stored block hash), `read_pending_data` synthesizes a `DeprecatedPendingBlock`. That type structurally has no `l1_data_gas_price` field; its `l1_data_gas_price()` accessor unconditionally returns `GasPricePerToken::default()` (zero). `client_pending_data_to_execution_pending_data` passes this zero value into `ExecutionPendingData.l1_data_gas_price`. `prepare_block_context` then substitutes `NonzeroGasPrice::MIN` (= 1) for the zero. Any call to `estimate_fee`, `simulate_transactions`, or `call` with `block_id=Tag::Pending` during a stale-pending window returns fee data computed against a data gas price of 1 instead of the real value.

### Finding Description

**Step 1 — `read_pending_data` stale-pending fallback**

When `pending_data.block.parent_block_hash() != latest_header.block_hash`, the function returns a synthesized `DeprecatedPendingBlock`: [1](#0-0) 

The synthesized block copies `eth_l1_gas_price` / `strk_l1_gas_price` from the latest header's `l1_gas_price`, but the `DeprecatedPendingBlock` struct has no `l1_data_gas_price` field, so `..Default::default()` fills the rest.

**Step 2 — `l1_data_gas_price()` always returns zero for `Deprecated` variant** [2](#0-1) 

The comment "In older versions, data gas price was 0" is correct for historical blocks, but the stale-pending fallback uses this type for *current* blocks too, discarding the real `l1_data_gas_price` from the latest header.

**Step 3 — `client_pending_data_to_execution_pending_data` passes zero through** [3](#0-2) 

**Step 4 — `prepare_block_context` substitutes `NonzeroGasPrice::MIN`** [4](#0-3) 

`NonzeroGasPrice::MIN` is defined as `GasPrice(1)`: [5](#0-4) 

**Step 5 — `estimate_fee` uses this corrupted block context** [6](#0-5) 

The resulting `FeeEstimation` carries `l1_data_gas_price = 1` and an `overall_fee` computed against that price, regardless of the real on-chain data gas price (which on mainnet is on the order of 10⁶–10⁹ wei).

### Impact Explanation

Any unprivileged caller invoking `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_call` with `block_id = "pending"` during a stale-pending window receives an authoritative-looking `FeeEstimation` where:

- `l1_data_gas_price` = 1 (instead of the real value, e.g. 10⁶–10⁹)
- `overall_fee` is computed against that price

For transactions using blob DA (`l1_da_mode = Blob`), the `l1_data_gas` component of the fee is multiplied by 1 instead of the real price, making the estimated fee orders of magnitude too low. Wallets and dApps that rely on `estimate_fee` to set resource bounds will underprice transactions.

The same zero-price defect also affects `l2_gas_price` (also returns `GasPricePerToken::default()` for `Deprecated`) and forces `l1_da_mode` to `Calldata` regardless of the real mode: [7](#0-6) 

### Likelihood Explanation

The stale-pending window is a normal operational condition. It occurs every time a new block is finalized and the pending sync loop has not yet fetched fresh pending data. On a busy chain this window is brief but recurrent. No special privileges are required — any public RPC caller can trigger this by querying `estimate_fee` with `block_id=pending` at any time.

### Recommendation

In the stale-pending fallback inside `read_pending_data`, use `PendingBlockOrDeprecated::Current(PendingBlock { ... })` instead of `Deprecated`, and populate `l1_data_gas_price`, `l2_gas_price`, and `l1_da_mode` from the latest stored header — the same way `eth_l1_gas_price` and `strk_l1_gas_price` are already copied. Alternatively, remove the `unwrap_or(NonzeroGasPrice::MIN)` silent fallback in `prepare_block_context` and return an explicit error when a zero gas price is encountered for a modern block.

### Proof of Concept

```
1. Node has latest block N with real l1_data_gas_price = 1_000_000_000 wei.
2. Pending sync has not yet fetched data for block N+1 (stale window).
3. Attacker calls: starknet_estimateFee([blob_da_tx], [], "pending")
4. read_pending_data: pending.parent_hash != block_N.hash → synthesizes DeprecatedPendingBlock
5. l1_data_gas_price() → GasPricePerToken { price_in_wei: 0, price_in_fri: 0 }
6. prepare_block_context: NonzeroGasPrice::new(0).unwrap_or(NonzeroGasPrice::MIN) → 1
7. FeeEstimation returned: { l1_data_gas_price: 1, overall_fee: <severely underestimated> }
```
