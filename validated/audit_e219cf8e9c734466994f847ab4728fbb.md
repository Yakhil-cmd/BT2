### Title
Permissionless `update_rewards` with Global `last_reward_block` Allows Any Staker to Steal All Consensus Block Rewards - (File: src/staking/staking.cairo)

### Summary
`update_rewards` is a public, access-control-free function that uses a single global `last_reward_block` variable to gate reward distribution to one call per block. Because the V3 consensus-rewards formula uses the calling staker's own total stake as both numerator and denominator, any active staker can call `update_rewards` for themselves first in a block and receive 100% of that block's rewards, permanently denying the legitimate block proposer their share.

### Finding Description

`update_rewards` (lines 1449–1507 of `src/staking/staking.cairo`) is a fully public function. Its only per-block guard is a check against a **global** (not per-staker) storage variable:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
...
self.last_reward_block.write(current_block_number);
``` [1](#0-0) 

The first transaction to call `update_rewards` in any block wins: it sets `last_reward_block` to the current block number, and every subsequent call in the same block reverts. `general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero — no role or identity check exists. [2](#0-1) 

In V3 consensus-rewards mode, the reward calculation passes the **staker's own total stake** as both the reward numerator weight and the denominator:

```cairo
let (staker_total_strk_balance, staker_total_btc_balance) = self
    .get_staker_total_strk_btc_balance_at_epoch(
        :staker_address, :staker_pool_info, epoch_id: curr_epoch,
    );
``` [3](#0-2) 

Then inside `calculate_staker_own_rewards`:

```cairo
mul_wide_and_div(
    lhs: strk_total_rewards,
    rhs: own_balance_curr_epoch.to_strk_native_amount(),
    div: strk_total_stake.to_strk_native_amount(),
)
``` [4](#0-3) 

A staker with no delegation pool has `own_balance == staker_total_strk_balance`, so the ratio is `1` and they receive **100% of the block rewards** regardless of their stake size relative to the global total.

**Attack path:**
1. Attacker stakes `min_stake` and waits K epochs for the balance to become effective.
2. Each block N, the attacker submits `update_rewards(attacker_address, disable_rewards=false)` as early as possible.
3. The attacker's transaction lands first, sets `last_reward_block = N`, and credits the attacker with 100% of block N's STRK rewards.
4. The legitimate block proposer's `update_rewards` call reverts with `REWARDS_ALREADY_UPDATED`.
5. Block N's rewards are permanently lost to the legitimate proposer.

Alternatively, the attacker can call `update_rewards(any_active_staker, disable_rewards=true)` to purely grief — no rewards are distributed to anyone, and `last_reward_block` is still consumed for that block. [5](#0-4) 

### Impact Explanation
**High — Theft of unclaimed yield / Permanent freezing of unclaimed yield.** An attacker holding only the minimum stake can steal 100% of every block's consensus rewards indefinitely. Legitimate block proposers permanently lose their per-block rewards for every block the attacker front-runs. Over time, all consensus block rewards are drained to the attacker.

### Likelihood Explanation
- `update_rewards` is permissionless — no special role is required beyond being an active staker with an effective balance.
- The attacker only needs their transaction to land before the legitimate proposer's in each block.
- In Starknet's current centralized-sequencer model, a malicious or bribed sequencer makes this trivially exploitable. Even with an honest sequencer, a staker who submits transactions at the very start of each block has a realistic chance of winning the race.
- As Starknet moves toward decentralized sequencing, this becomes straightforwardly executable by any staker.
- The cost is minimal: `min_stake` (fully recoverable via `unstake_intent`/`unstake_action`) plus gas.

### Recommendation
1. **Add access control to `update_rewards`**: Restrict callers to a trusted consensus contract or verify the caller is the staker whose block is being processed.
2. **Make `last_reward_block` per-staker**: If multiple stakers should be able to call `update_rewards` per block, track the last reward block per `staker_address` rather than globally.
3. **Remove `disable_rewards` as a freely caller-controlled parameter**: If this flag is meant to be set by the protocol, it should not be settable by arbitrary callers.

### Proof of Concept
```
1. Attacker calls stake(reward_addr, op_addr, min_stake).
2. Wait K epochs for balance to become effective.
3. Each block N:
   a. Attacker submits update_rewards(attacker_address, disable_rewards=false).
   b. last_reward_block is set to N; attacker receives 100% of block N's STRK rewards.
   c. Legitimate block proposer's update_rewards(proposer_address, ...) reverts
      with REWARDS_ALREADY_UPDATED.
4. Attacker accumulates all consensus block rewards indefinitely.
   Legitimate stakers receive zero block rewards.
``` [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
