### Title
Unrestricted `exit_delegation_pool_action` Enables Griefing of Pool Members Attempting to Switch Pools - (File: src/pool/pool.cairo)

### Summary

`exit_delegation_pool_action` in `pool.cairo` carries no caller restriction — any address can execute it for any pool member. A pool member who has set an exit intent and is waiting to call `switch_delegation_pool` (to atomically re-delegate to a better staker) can be griefed by an attacker who calls `exit_delegation_pool_action` first, clearing the intent and forcing an unwanted exit. The pool member must then re-delegate from scratch, losing K epochs of rewards on the re-delegated amount.

### Finding Description

`exit_delegation_pool_action` is documented and implemented with "Any address can execute" access control:

```
#### access control
Any address can execute.
```

The implementation confirms this — there is no `get_caller_address()` check against the pool member:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    ...
    pool_member_info.unpool_amount = Zero::zero();
    pool_member_info.unpool_time = Option::None;
    self.write_pool_member_info(:pool_member, :pool_member_info);
    token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
    unpool_amount
}
```

Meanwhile, `switch_delegation_pool` — the alternative action a pool member may wish to take — requires `unpool_time.is_some()`:

```cairo
fn switch_delegation_pool(...) {
    ...
    assert!(
        pool_member_info.unpool_time.is_some(),
        "{}",
        GenericError::MISSING_UNDELEGATE_INTENT,
    );
    ...
}
```

`switch_delegation_pool` imposes **no time constraint** — it can be called before or after the exit window expires, as long as the intent is still set. This is confirmed by flow tests that call `switch_delegation_pool` after `advance_exit_wait_window()`.

The attack window is therefore: **any time after the exit window expires and before the pool member calls `switch_delegation_pool`**. An attacker who monitors on-chain exit intents can call `exit_delegation_pool_action(pool_member)` to clear the intent, making `switch_delegation_pool` revert with `MISSING_UNDELEGATE_INTENT`.

### Impact Explanation

The pool member:
1. Receives their `unpool_amount` back (no direct fund loss).
2. Loses the ability to atomically switch to the target pool.
3. Must re-delegate from scratch, which takes effect only at `current_epoch + K`.
4. Earns **zero rewards on the re-delegated amount for K epochs** from the new pool.

This constitutes griefing with no profit motive but measurable economic damage to the pool member (lost staking yield for K epochs on the re-delegated principal). This fits the **Medium** impact tier: *Griefing with no profit motive but damage to users or protocol*.

### Likelihood Explanation

- Exit intents are fully public on-chain (stored in `pool_exit_intents` map in the staking contract).
- The attacker needs only to watch for intents whose `unpool_time` has passed and call `exit_delegation_pool_action` before the pool member calls `switch_delegation_pool`.
- No privileged access, leaked key, or external dependency is required.
- The attack can be repeated every time the pool member sets a new exit intent, making it a persistent denial-of-pool-switch.
- On Starknet the sequencer is centralized, but the attacker does not need same-block front-running — any block after the exit window suffices.

### Recommendation

Restrict `exit_delegation_pool_action` so that only the pool member (or their designated reward address) can trigger it, mirroring the access control already applied to `claim_rewards` and `switch_delegation_pool`. Alternatively, allow the pool member to call `switch_delegation_pool` without a prior exit intent, removing the dependency on `unpool_time`.

### Proof of Concept

1. Pool member calls `exit_delegation_pool_intent(amount)` on `pool.cairo`.
   - `pool_member_info.unpool_time = Some(now + exit_wait_window)`
   - `pool_member_info.unpool_amount = amount`

2. Time advances past `exit_wait_window`.

3. Pool member constructs a transaction to call `switch_delegation_pool(to_staker, to_pool, amount)`.

4. **Attacker** (any address) calls `exit_delegation_pool_action(pool_member)` in an earlier or same-block transaction:
   - `unpool_time` → `None`
   - `unpool_amount` → `0`
   - `amount` tokens transferred back to pool member.

5. Pool member's `switch_delegation_pool` transaction reverts:
   ```
   assert!(pool_member_info.unpool_time.is_some(), ..., MISSING_UNDELEGATE_INTENT)
   ```

6. Pool member must re-delegate `amount` to `to_pool`, which takes effect at `current_epoch + K`, losing K epochs of rewards on that principal.

```text
        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
```
