### Title
Unrestricted `new()` Initialization Allows Any Caller to Seize Engine Ownership - (File: engine/src/contract_methods/admin.rs)

### Summary

The `new()` function that initializes the Aurora Engine's state performs no caller authentication. Any NEAR account can invoke it before the legitimate deployer does, supplying an attacker-controlled `owner_id`. Because `owner_id` governs all subsequent privileged operations (`upgrade`, `set_eth_connector_contract_account`, `pause_contract`, etc.), a successful front-run grants the attacker full administrative control over the engine and every asset it custodies.

### Finding Description

`engine/src/contract_methods/admin.rs` exposes the `new()` function as the sole initialization path for the `EngineState`:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input)...;
    ...
    state::set_state(&mut io, &state)?;
    Ok(())
}
``` [1](#0-0) 

The only guard is the `ERR_ALREADY_INITIALIZED` check on line 57. There is no check on `env.predecessor_account_id()`. The `NewCallArgs` structure carries the `owner_id` field that becomes the engine's privileged owner: [2](#0-1) 

The `owner_id` stored in `EngineState` is the sole credential checked by every privileged operation: [3](#0-2) 

The deployment flow in `engine-workspace/src/lib.rs` shows that contract deployment and `new()` are **two separate transactions** with an `await` boundary between them, creating an exploitable window: [4](#0-3) 

An attacker monitoring the NEAR chain sees the `DeployContract` action land in a block and immediately submits a `new()` call with their own account as `owner_id` before the legitimate deployer's initialization transaction is included.

### Impact Explanation

Once the attacker controls `owner_id` they can:

1. Call `upgrade()` (owner-only) to deploy arbitrary WASM, replacing the engine with code that drains all EVM balances — **direct theft of all user funds at rest**.
2. Call `set_eth_connector_contract_account()` to point the engine at a malicious ETH connector, redirecting all future deposits and withdrawals — **theft of funds in motion**.
3. Call `pause_contract()` to freeze the engine indefinitely — **permanent fund freeze**. [5](#0-4) [6](#0-5) 

### Likelihood Explanation

NEAR Protocol does not have a public mempool in the Ethereum sense, but blocks are produced and finalized rapidly and all transactions are publicly observable on-chain. An attacker running a NEAR node or watching the RPC can detect the `DeployContract` action in the block it lands and submit a `new()` call in the very next block — a back-run that is straightforward to automate. The window exists on every fresh deployment or re-deployment of the engine. The attack requires no special privilege, no capital, and no prior relationship with the protocol.

### Recommendation

Add a caller check inside `new()` that restricts initialization to the contract account itself (a self-call, which is the NEAR-idiomatic pattern for atomic deploy-and-init batches):

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // Only the contract account itself may initialize it.
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(errors::ERR_NOT_ALLOWED.into());
    }
    ...
}
```

Alternatively, bundle the `DeployContract` and `FunctionCall("new", ...)` actions into a single NEAR batch transaction so they execute atomically, eliminating the window entirely. The XCC router already documents this pattern as the correct approach: [7](#0-6) 

### Proof of Concept

1. Attacker watches the NEAR chain for a `DeployContract` action targeting the `aurora` account.
2. In the next block, attacker submits:
   ```
   near call aurora new '{"chain_id":1313161554,"owner_id":"attacker.near","upgrade_delay_blocks":0,"key_manager":"attacker.near","initial_hashchain":null}' --accountId attacker.near
   ```
3. Because no state exists yet, `state::get_state(&io).is_ok()` returns `false` and the call succeeds, writing `owner_id = "attacker.near"` into storage.
4. The legitimate deployer's subsequent `new()` call fails with `ERR_ALREADY_INITIALIZED`.
5. Attacker calls `upgrade()` with a malicious WASM payload, replacing the engine contract and draining all EVM balances. [1](#0-0) [8](#0-7)

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
