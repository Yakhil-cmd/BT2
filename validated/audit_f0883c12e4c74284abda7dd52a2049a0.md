### Title
Admin fee cap bypassed via `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary
`setPoolAdminFees` enforces `maxAdminSpreadFeeE6` and `maxAdminNotionalFeeE8` caps on every call, but the parallel path `setPoolBinAdditionalFees` forwards per-bin additional fees directly to the pool with **no cap validation**, allowing a pool admin to charge fees that exceed the factory-enforced limits on any targeted bin.

### Finding Description

The factory enforces admin fee caps in `setPoolAdminFees`:

```solidity
// MetricOmmPoolFactory.sol L414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

However, `setPoolBinAdditionalFees` passes `addFeeBuyE6` and `addFeeSellE6` straight through to the pool without any cap check:

```solidity
// MetricOmmPoolFactory.sol L450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` stores the values without any cap check either:

```solidity
// MetricOmmPool.sol L464-473
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
  if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
  BinState storage s = _binStates[bin];
  s.addFeeBuyE6 = addFeeBuyE6;
  s.addFeeSellE6 = addFeeSellE6;
  emit BinAdditionalFeesUpdated(bin, addFeeBuyE6, addFeeSellE6);
}
``` [3](#0-2) 

The `uint16` type permits values up to **65 535**, which in E6 units equals **6.5535%**. The factory owner can set `maxAdminSpreadFeeE6` to any value below that (e.g. 0.5 % = 5 000). The admin is then blocked from setting the global admin spread fee above 0.5 % via `setPoolAdminFees`, but can freely set per-bin additional fees up to 6.5535 % via `setPoolBinAdditionalFees` — a **13× bypass** of the intended cap.

The structural parallel to the external report is exact: the protection (fee cap) is enforced on one code path (`setPoolAdminFees`) but is entirely absent on the alternative path (`setPoolBinAdditionalFees`), mirroring how `ownershipChange` was only updated on `_transferFrom` and not on split/merge.

### Impact Explanation

Traders swapping through the targeted bin pay per-bin additional fees that are additive with the base swap fee. Because these fees are set by the pool admin and flow to the admin fee destination, the admin can extract more value from traders than the factory-enforced cap permits. This is a direct loss of user principal on every swap through the affected bin, constituting an admin-boundary break above Sherlock medium thresholds.

### Likelihood Explanation

Medium. The trigger is a pool admin — a semi-trusted role — calling a single permissioned function. No flash loan, no oracle manipulation, and no external dependency is required. The bypass is reachable in one transaction and leaves no on-chain evidence distinguishing it from a legitimate per-bin fee configuration.

### Recommendation

Add the same cap guards inside `setPoolBinAdditionalFees` that exist in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, if per-bin fees are intended to be additive with the global admin fee, validate that `adminSpreadFeeE6 + addFeeBuyE6 ≤ maxAdminSpreadFeeE6` for the affected bin.

### Proof of Concept

1. Factory owner sets `maxAdminSpreadFeeE6 = 5_000` (0.5 %).
2. Pool admin calls `setPoolAdminFees(pool, 5_001, 0)` → reverts `AdminFeeTooHigh`. Cap is enforced.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` → **succeeds**. Per-bin buy and sell fees on bin 0 are now 6.5535 %, 13× the intended cap.
4. Every trader whose swap routes through bin 0 pays 6.5535 % additional fees on top of the base spread, with the excess flowing to the admin fee destination — a direct loss of user funds above the protocol-permitted level. [4](#0-3) [2](#0-1)
