### Title
`setPoolAdminFeeDestination` Redirects Accrued-But-Uncollected Admin Fees to New Destination Without Prior Collection — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`setPoolAdminFeeDestination` updates `poolAdminFeeDestination[pool]` without first flushing accrued fees to the old destination. Every other fee-configuration mutator in the factory (`setPoolAdminFees`, `setPoolProtocolFee`) calls `collectFees` before updating state. The missing collect-first step in `setPoolAdminFeeDestination` causes all fees that accrued under the old destination to be paid out to the new destination on the next `collectPoolFees` call.

### Finding Description

`setPoolAdminFees` and `setPoolProtocolFee` both follow a collect-first pattern: [1](#0-0) 

```solidity
// setPoolAdminFees — collects first, then updates
PoolFeeConfig memory c = poolFeeConfig[pool];
IMetricOmmPoolCollectFees(pool).collectFees(
    c.protocolSpreadFeeE6,
    c.adminSpreadFeeE6,
    c.protocolNotionalFeeE8,
    c.adminNotionalFeeE8,
    poolAdminFeeDestination[pool]   // ← old destination receives accrued fees
);
// ... then update config
```

`setPoolAdminFeeDestination` does not: [2](#0-1) 

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;   // ← no collect first
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

`collectPoolFees` is permissionless and reads `poolAdminFeeDestination[pool]` at call time: [3](#0-2) 

```solidity
function collectPoolFees(address pool) external override nonReentrant {
    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6, c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8, c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]   // ← now points to new destination
    );
}
```

The pool's `collectFees` then transfers the entire accrued admin share to whatever address `adminFeeDestination_` holds at that moment: [4](#0-3) 

```solidity
if (totalFee0ToAdmin > 0) transferToken0(adminFeeDestination_, totalFee0ToAdmin);
if (totalFee1ToAdmin > 0) transferToken1(adminFeeDestination_, totalFee1ToAdmin);
```

### Impact Explanation

The old fee destination — which may be a treasury contract entirely separate from the pool admin — permanently loses all fees that accrued before the destination change. The new destination receives tokens it did not earn. The loss is bounded by the pool's accrued-but-uncollected admin fee balance at the time of the call, which can be substantial for high-volume pools with infrequent collection.

This is a direct, irreversible loss of owed admin-fee assets, matching the "direct loss of protocol fees or owed LP assets" impact gate.

### Likelihood Explanation

The trigger is a single pool-admin call to `setPoolAdminFeeDestination`. Legitimate scenarios include treasury migrations, multisig rotations, or an admin transfer followed by the new admin redirecting fees. The function is callable at any time with no timelock. The inconsistency with `setPoolAdminFees` (which does collect first) means the loss is easy to overlook.

### Recommendation

Add a collect-first step inside `setPoolAdminFeeDestination`, mirroring the pattern in `setPoolAdminFees`:

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
+   PoolFeeConfig memory c = poolFeeConfig[pool];
+   IMetricOmmPoolCollectFees(pool).collectFees(
+       c.protocolSpreadFeeE6, c.adminSpreadFeeE6,
+       c.protocolNotionalFeeE8, c.adminNotionalFeeE8,
+       poolAdminFeeDestination[pool]   // flush to old destination first
+   );
    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

### Proof of Concept

1. Pool accumulates 1000 token0 in admin spread fees (uncollected).
2. Pool admin calls `setPoolAdminFeeDestination(pool, newTreasury)`.
3. `poolAdminFeeDestination[pool]` is updated; no fees are flushed.
4. Anyone calls `collectPoolFees(pool)`.
5. `collectFees` is invoked with `adminFeeDestination_ = newTreasury`.
6. `newTreasury` receives the 1000 token0 that accrued under `oldTreasury`.
7. `oldTreasury` receives nothing; loss is permanent. [2](#0-1) [5](#0-4)
