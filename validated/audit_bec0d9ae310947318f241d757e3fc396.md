### Title
Simulation Trace Annotates Full Transaction DA Gas on Every `FunctionInvocation` — (`crates/apollo_rpc_execution/src/objects.rs`)

### Summary

`InvokeTransactionTrace::try_from` passes `receipt.da_gas` — the **total transaction-level** DA gas — uniformly to `validate_invocation`, `execute_invocation`, and `fee_transfer_invocation`. Every `FunctionInvocation` node in the trace tree, including all recursive inner calls, is then stamped with this same aggregate value as its `da_gas_consumed`. The result is that `starknet_simulateTransactions` returns an authoritative-looking but structurally wrong trace: the validate call (which makes no state changes) appears to consume the same DA gas as the entire transaction.

### Finding Description

`TransactionReceipt::da_gas` is computed once at the transaction level from the aggregate state diff across all phases: [1](#0-0) 

This single `GasVector` is then passed identically to all three top-level invocations: [2](#0-1) 

Inside `FunctionInvocation::try_from`, the same `gas_vector` is forwarded to every inner call recursively: [3](#0-2) 

And it is written verbatim into `execution_resources.da_gas_consumed` for each node: [4](#0-3) 

The same pattern is present in `DeclareTransactionTrace::try_from` and `DeployAccountTransactionTrace::try_from`: [5](#0-4) 

### Impact Explanation

Any caller of `starknet_simulateTransactions` receives a trace where:

- `validate_invocation.execution_resources.da_gas_consumed` equals the full transaction DA gas, even though `__validate__` makes no state changes and its true DA contribution is zero.
- `fee_transfer_invocation.execution_resources.da_gas_consumed` equals the full transaction DA gas, even though fee transfer only touches two balance storage slots.
- Every inner call at any depth carries the same inflated value.

This is an authoritative-looking wrong value on a public, unprivileged RPC endpoint. Tooling, block explorers, and developers that rely on per-call DA gas attribution for cost analysis, debugging, or gas optimization will receive systematically incorrect data.

### Likelihood Explanation

Triggerable by any unprivileged user submitting any invoke transaction to `starknet_simulateTransactions`. No special privileges, keys, or state are required. The corruption is deterministic and reproducible on every simulation call.

### Recommendation

DA gas is a transaction-level quantity derived from the aggregate state diff; it cannot be meaningfully decomposed per call without tracking per-call storage writes. The correct fix is one of:

1. **Zero out per-call DA gas**: set `da_gas_consumed` to `GasVector::ZERO` for all `FunctionInvocation` nodes and document that DA gas is reported only at the transaction receipt level.
2. **Per-call state-diff tracking**: propagate per-call `StateChanges` through `CallInfo` and compute each call's DA contribution individually before building the trace.

Option 1 is the minimal correct fix; option 2 requires deeper blockifier instrumentation.

    type Error = ExecutionError;
    fn try_from(transaction_execution_info: TransactionExecutionInfo) -> ExecutionResult<Self> {
        let execute_invocation = match transaction_execution_info.revert_error {
            Some(revert_error) => {
                FunctionInvocationResult::Err(RevertReason::RevertReason(revert_error.to_string()))
            }
            None => FunctionInvocationResult::Ok(
                (
                    transaction_execution_info
                        .execute_call_info
                        .expect("Invoke transaction execution should contain execute_call_info."),
                    transaction_execution_info.receipt.da_gas,
                )
                    .try_into()?,
            ),
        };

        Ok(Self {
            validate_invocation: match transaction_execution_info.validate_call_info {
                None => None,
                Some(call_info) => {
                    Some((call_info, transaction_execution_info.receipt.da_gas).try_into()?)
                }
            },
            execute_invocation,
            fee_transfer_invocation: match transaction_execution_info.fee_transfer_call_info {
                None => None,
                Some(call_info) => {
                    Some((call_info, transaction_execution_info.receipt.da_gas).try_into()?)
                }
            },
        })
    }
}
