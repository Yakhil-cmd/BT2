### Title
L2 Gas Price Permanently Stuck at Sub-Minimum Value Due to Integer Division Truncation to Zero - (`crates/apollo_consensus_orchestrator/src/fee_market/mod.rs`)

### Summary

`calculate_next_base_gas_price` contains a precision-loss bug in its below-minimum recovery path. When the current L2 gas price is below both `min_gas_price` and `MIN_GAS_PRICE_INCREASE_DENOMINATOR` (333), integer division truncates `max_increase` to zero, leaving the price permanently frozen at its current sub-minimum value. This is the direct Sequencer analog of the external report's "fee is zero because of precision loss" pattern.

### Finding Description

In `crates/apollo_consensus_orchestrator/src/fee_market/mod.rs`, the constant `MIN_GAS_PRICE_INCREASE_DENOMINATOR = 333` is used to compute a per-block price increase when the current price is below the enforced minimum:

```rust
const MIN_GAS_PRICE_INCREASE_DENOMINATOR: u128 = 333;
``` [1](#0-0) 

The recovery branch is:

```rust
if price < min_gas_price {
    let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
    let adjusted = price.0 + max_increase;
    // Cap at min_gas_price to avoid overshooting
    let adjusted_price = adjusted.min(min_gas_price.0);
    return GasPrice(adjusted_price);
}
``` [2](#0-1) 

When `price.0 < 333`, Rust integer division yields `max_increase = 0`. Then `adjusted = price.0`, and `adjusted_price = min(price.0, min_gas_price.0) = price.0` (since `price < min_gas_price`). The function returns the **same price** it received — the price never moves.

Contrast this with the normal (non-early-return) path, which correctly enforces a floor:

```rust
GasPrice(max(adjusted_price, min_gas_price.0))
``` [3](#0-2) 

The early-return path has no such floor, so a price of 1–332 FRI is frozen indefinitely.

The frozen price then propagates into the gateway's stateful admission check:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
``` [4](#0-3) 

With `previous_block_l2_gas_price = 1 FRI` and `min_gas_price_percentage = 100`, `threshold = 1`. Every transaction with `max_price_per_unit >= 1 FRI` is admitted — effectively bypassing the intended minimum-price guard.

### Impact Explanation

The L2 gas price is permanently frozen at a sub-minimum value (1–332 FRI ≈ 10⁻¹⁸–10⁻¹⁶ STRK). All subsequent blocks inherit this frozen price. The gateway's stateful L2 gas price threshold collapses to 1 FRI, admitting transactions that should be rejected. Actual fees charged for L2 gas are orders of magnitude below the intended minimum, constituting a direct economic loss to the sequencer/protocol. This matches the "Incorrect fee, gas, bouncer, resource accounting … with economic impact" criterion.

### Likelihood Explanation

The trigger condition is `current_l2_gas_price < 333 FRI`. This is reachable when:

1. **Genesis / bootstrap**: The initial L2 gas price in the first `ProposalInit` is set to a very small value. `convert_to_sn_api_block_info` only emits a `warn!` on zero price and returns an error only for exactly zero; any value 1–332 is silently accepted.
2. **`fee_actual` step-up**: If the dynamic `fee_actual` (median of recent proposals) jumps sharply, `effective_min` rises above the current price. If the current price is already below 333, the recovery loop stalls.

### Recommendation

Replace the integer division with a guaranteed minimum increment of 1:

```rust
let max_increase = (price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR).max(1);
```

Alternatively, apply the same `max(adjusted_price, min_gas_price.0)` floor that the normal path already uses, so the early-return path cannot return a value below `min_gas_price`.

### Proof of Concept

```
price = GasPrice(1), min_gas_price = GasPrice(100_000)

max_increase = 1 / 333 = 0          // integer truncation
adjusted     = 1 + 0   = 1
adjusted_price = min(1, 100_000) = 1

return GasPrice(1)                   // identical to input — price never moves
```

Repeat for every subsequent block: the function always returns `GasPrice(1)`. The gateway threshold becomes `(100/100) * 1 = 1 FRI`, so any transaction with `max_price_per_unit >= 1 FRI` is admitted regardless of the true minimum.
