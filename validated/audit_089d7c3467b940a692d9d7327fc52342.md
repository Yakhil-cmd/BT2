### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Other Stakers' Rewards - (File: src/staking/staking.cairo)

### Summary

`update_rewards` is documented as "Only starkware sequencer" but has no on-chain access control enforcement. Any address can call it, consuming the single global `last_reward_block` slot for the current block. An attacker who front-runs the sequencer every block can permanently prevent all other stakers from receiving consensus rewards.

### Finding Description

The spec explicitly states the access control for `update_rewards`:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation only calls `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address — no sequencer role check exists: [2](#0-1) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
```

`general_prerequisites` enforces no privileged role: [3](#0-2) 

The function writes `last_reward_block` to the current block number unconditionally before the `disable_rewards` guard: [4](#0-3) 

`last_reward_block` is a **single global variable** shared across all stakers. Once it is set to block `N`, no further call to `update_rewards` can succeed in block `N` for any staker: [5](#0-4) 

### Impact Explanation

**Attack path A — Theft of unclaimed yield (High):**

1. Attacker registers as a staker with minimum stake.
2. Every block, the attacker calls `update_rewards(attacker_staker, disable_rewards: false)` before the sequencer's transaction lands.
3. `last_reward_block` is set to the current block; the attacker's staker receives its proportional block rewards via `_update_rewards`.
4. The sequencer's legitimate call for any other staker reverts with `REWARDS_ALREADY_UPDATED`.
5. All other stakers receive zero rewards for that block. The attacker captures the entire reward slot every block.

**Attack path B — Permanent freezing of unclaimed yield (High) / Griefing (Medium):**

1. Attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` every block.
2. `last_reward_block` is consumed; no rewards are distributed to anyone.
3. All stakers permanently lose consensus rewards with no profit to the attacker.

The rewards that are not distributed are never added to `unclaimed_rewards` in the reward supplier, so they are effectively lost: [6](#0-5) 

### Likelihood Explanation

- No special privilege is required — any non-zero address can call `update_rewards`.
- The attacker only needs to submit a transaction before the sequencer's reward-distribution transaction each block, which is trivially achievable by monitoring the mempool or simply submitting at the start of every block.
- The minimum stake requirement is the only barrier for Attack path A; Attack path B requires no stake at all (any valid active staker address suffices as the argument).

### Recommendation

Add a sequencer-only role check at the top of `update_rewards`, consistent with the spec's stated access control:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer(); // enforce "Only starkware sequencer"
    self.general_prerequisites();
    ...
}
```

Alternatively, restrict via the existing `RolesComponent` using a dedicated `SEQUENCER_ROLE`, mirroring how `pause` is restricted to `only_security_agent`: [7](#0-6) 

### Proof of Concept

```
Block N:
  Attacker tx:   update_rewards(attacker_staker, disable_rewards=false)
                 → last_reward_block = N
                 → attacker_staker.unclaimed_rewards_own += proportional_block_rewards
                 → reward_supplier.unclaimed_rewards += attacker_rewards

  Sequencer tx:  update_rewards(validator_staker, disable_rewards=false)
                 → PANICS: REWARDS_ALREADY_UPDATED (last_reward_block == N)
                 → validator_staker receives 0 rewards for block N

Repeat every block → validator_staker (and all other stakers) permanently frozen out of consensus rewards.
``` [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
