### Title
Unbounded `pool_member_epoch_balance` Trace Growth Permanently Freezes Delegator Unclaimed Rewards - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` function in the Pool contract iterates over every entry in a delegator's `pool_member_epoch_balance` trace in an unbounded loop. Each balance change in a distinct epoch appends a new checkpoint to this trace. A delegator who accumulates many balance changes across many epochs without claiming rewards will eventually be unable to call `claim_rewards` at all, permanently freezing their unclaimed yield.

### Finding Description
Every call to `add_to_delegation_pool`, `exit_delegation_pool_intent`, or `enter_delegation_pool` invokes `set_member_balance`, which calls `trace.insert(key: self.get_epoch_plus_k(), value: pool_member_balance)`. The `insert` function only overwrites the last entry if the key matches; otherwise it appends a new checkpoint. [1](#0-0) 

Because the key is `current_epoch + K`, balance changes made in different epochs always produce distinct keys and thus always append new entries, growing the trace without bound. [2](#0-1) 

When `claim_rewards` is called, it invokes `calculate_rewards`, which contains an explicitly acknowledged unbounded loop:

```
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
``` [3](#0-2) 

The `entry_to_claim_from` cursor advances only on a **successful** `claim_rewards` call. If the loop exceeds the gas limit, the transaction reverts, `entry_to_claim_from` is never updated, and every subsequent `claim_rewards` call will attempt the same oversized loop and revert again — permanently. [4](#0-3) 

### Impact Explanation
Once the trace is large enough to exceed the Starknet transaction gas limit, `claim_rewards` becomes permanently uncallable for that delegator. All accumulated STRK rewards are irreversibly frozen in the pool contract with no recovery path, because there is no partial-claim mechanism and no administrative escape hatch.

This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
A delegator who makes one balance change per epoch (e.g., a small `add_to_delegation_pool` or `exit_delegation_pool_intent`) without ever claiming rewards will grow their trace by one entry per epoch. The loop also calls `find_sigma` on each iteration, which performs multiple storage reads per entry. [5](#0-4) 

Additionally, a delegator's `reward_address` can call `add_to_delegation_pool` on their behalf, spending the reward address's own tokens to inflate the victim's trace across epochs — a targeted griefing attack at a cost proportional to the minimum delegation amount per epoch. [6](#0-5) 

The protocol's own comment acknowledges the risk but dismisses it as "unlikely." Given that Starknet epochs are short and the minimum delegation amount is small, the threshold for gas exhaustion is reachable over a realistic protocol lifetime.

### Recommendation
1. **Short term**: Add a `max_entries` parameter to `claim_rewards` so the loop processes at most N entries per call, allowing delegators to drain their trace incrementally.
2. **Long term**: Implement trace compaction — when `claim_rewards` is called, collapse processed entries so the trace length is bounded by the number of unclaimed epochs, not the total lifetime of balance changes.

### Proof of Concept

```
// Attacker (or negligent delegator) scenario:
// 1. Delegator enters pool with minimum amount.
// 2. Every epoch, delegator calls add_to_delegation_pool(amount: 1)
//    → set_member_balance is called with key = current_epoch + K (distinct each epoch)
//    → pool_member_epoch_balance trace grows by 1 entry per epoch.
// 3. Delegator never calls claim_rewards.
// 4. After N epochs (N large enough to exceed gas limit in the loop):
//    claim_rewards is called.
//    → calculate_rewards loops N times, each iteration reading storage + calling find_sigma.
//    → Transaction reverts due to gas exhaustion.
//    → entry_to_claim_from is NOT updated (revert).
// 5. All future claim_rewards calls revert identically.
//    → Delegator's accumulated STRK rewards are permanently frozen.
//
// Targeted variant: reward_address calls add_to_delegation_pool for victim
// across many epochs, spending small amounts to inflate victim's trace.
```

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
