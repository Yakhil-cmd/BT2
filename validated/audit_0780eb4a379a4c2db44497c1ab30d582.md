### Title
`set_commission` applies updated commission retroactively to entire epoch rewards without checkpointing accrued yield - (File: `src/staking/staking.cairo`)

### Summary
When a staker calls `set_commission` mid-epoch (enabled by an active `commission_commitment`), the updated commission rate is applied to the **entire epoch's rewards** when `update_rewards_from_attestation_contract` is subsequently called. Because no reward checkpoint is taken before the commission change, rewards accrued at the old rate are retroactively recalculated at the new (higher) rate, allowing a staker to steal unclaimed yield from delegators.

### Finding Description

`set_commission` at line 725 of `src/staking/staking.cairo` simply writes the new commission value to storage and emits an event. It performs no checkpoint or settlement of pending epoch rewards before the update:

```rust
fn set_commission(ref self: ContractState, commission: Commission) {
    // ...
    staker_pool_info.commission.write(Option::Some(commission));
    // No reward checkpoint here
}
```

`update_rewards_from_attestation_contract` at line 1394 calculates rewards for the **entire current epoch** by calling `calculate_current_epoch_rewards()` and passing the result to `_update_rewards`, which calls `calculate_staker_pools_rewards`. Inside `calculate_staker_pools_rewards` at line 1964, the commission is read from storage at the time of the call — i.e., the post-change value:

```rust
let commission = staker_pool_info.commission();
```

This single commission value is then applied to the full epoch reward bucket via `split_rewards_with_commission`, regardless of when during the epoch the commission was changed.

`set_commission_commitment` at line 748 allows a staker to set a commitment with `max_commission >= current_commission`, explicitly enabling future commission increases. The code itself acknowledges the gap at lines 745–747:

> **Note**: Current commission increase safeguards still allow for sudden commission changes.

The attack path is:
1. Epoch N begins with commission = 10% (1000).
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: N+1)`.
3. Near the end of epoch N (but before attesting), staker calls `set_commission(commission: 10000)`.
4. Staker attests, triggering `update_rewards_from_attestation_contract`.
5. `calculate_staker_pools_rewards` reads `commission = 10000` (100%).
6. `split_rewards_with_commission` routes 100% of pool rewards to the staker and 0% to the pool.
7. Delegators receive nothing for epoch N, despite having earned rewards at 10% commission for the entire epoch.

### Impact Explanation

This is a direct theft of unclaimed yield from delegators. A staker can retroactively apply a higher commission rate to rewards that were earned at a lower rate, capturing the difference. In the worst case (commission raised to 100%), delegators lose their entire epoch's yield. The stolen amount scales with the commission delta and the total delegated stake.

### Likelihood Explanation

Any staker with a pool can execute this attack. The only prerequisite is setting a `commission_commitment` (which the staker controls entirely) and then calling `set_commission` before attesting. No external dependency, leaked key, or privileged role is required. The staker is a valid unprivileged participant per the access rules.

### Recommendation

Commission changes should not retroactively affect the current epoch's rewards. Two mitigations:

1. **Checkpoint first**: Before writing the new commission in `set_commission`, settle any pending epoch rewards using the old commission rate (analogous to the `_regenerate` call suggested in the Olympus report).
2. **Epoch-delayed application**: Store the new commission alongside an `effective_epoch` field and apply it only from the next epoch onward in `calculate_staker_pools_rewards`.

### Proof of Concept

```
Epoch N:
  commission = 1000 (10%)
  delegated_amount = 1_000_000 STRK
  epoch_pool_rewards_including_commission = 10_000 STRK

Step 1: staker.set_commission_commitment(max_commission=10000, expiration_epoch=N+1)
Step 2: [wait until attestation window opens in epoch N]
Step 3: staker.set_commission(commission=10000)  // 100%
Step 4: staker.attest(block_hash)
  -> update_rewards_from_attestation_contract called
  -> calculate_staker_pools_rewards reads commission=10000
  -> commission_rewards = 10_000 * 10000 / 10000 = 10_000 STRK  (all to staker)
  -> pool_rewards = 0 STRK  (nothing to delegators)

Expected (at 10%): delegators receive 9_000 STRK
Actual (at 100%): delegators receive 0 STRK
Stolen: 9_000 STRK
```

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();

                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }

        fn update_rewards_from_attestation_contract(
            ref self: ContractState, staker_address: ContractAddress,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
            self.assert_caller_is_attestation_contract();
            let mut staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            // Get current epoch data.
            let (strk_epoch_rewards, btc_epoch_rewards) = reward_supplier_dispatcher
                .calculate_current_epoch_rewards();
            let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let curr_epoch = self.get_current_epoch();
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_epoch_rewards,

                    :strk_total_stake,
                    :btc_total_stake,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
