Based on my investigation, I found a valid analog. The `update_rewards` function in `src/staking/staking.cairo` is specified as "Only starkware sequencer" in the spec but has **no caller authorization check** in the implementation.

### Title
Missing Authorization on `update_rewards` Allows Any Caller to Permanently Freeze Staker Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `StakingRewardsManagerImpl` is documented as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards = true`, updating the global `last_reward_block` without distributing rewards. This permanently blocks the legitimate sequencer call for that block, causing all stakers to lose their block rewards.

### Finding Description
The spec at `docs/spec.md` line 1645 states:
> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1448–1507 contains no such check:

```rust
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... no caller identity check ...
        self.last_reward_block.write(current_block_number);
        if disable_rewards || self.is_pre_consensus() {
            return;
        }
        // distribute rewards ...
    }
}
``` [1](#0-0) 

The `last_reward_block` is a **global** storage variable (not per-staker). Any call to `update_rewards` — regardless of `staker_address` — writes the current block number to it. The guard `current_block_number > self.last_reward_block.read()` then blocks all subsequent calls in the same block with `REWARDS_ALREADY_UPDATED`. [2](#0-1) [3](#0-2) 

The spec explicitly restricts this function: [4](#0-3) 

### Impact Explanation
An attacker front-runs the sequencer's `update_rewards` call every block with `disable_rewards = true`. This:
1. Sets `last_reward_block = current_block` without distributing any rewards.
2. Causes the sequencer's legitimate call to revert with `REWARDS_ALREADY_UPDATED`.
3. All stakers lose their block rewards for that block.

Repeated every block, this permanently freezes all unclaimed yield for all stakers in the protocol. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The entry path requires no privilege: any externally owned address on Starknet can submit a transaction calling `update_rewards(any_valid_staker_address, disable_rewards: true)`. The attacker only needs to submit this transaction before the sequencer's own call in each block. Since the sequencer call is predictable and the attacker's call is valid (passes all checks), this is reliably exploitable at low cost.

### Recommendation
Add a caller authorization check at the top of `update_rewards`, analogous to `assert_caller_is_attestation_contract` used in `update_rewards_from_attestation_contract`:

```rust
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
}
``` [5](#0-4) 

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus()` returns false).
2. Attacker observes the mempool or simply submits at the start of each block.
3. Attacker calls `staking.update_rewards(staker_address: any_valid_staker, disable_rewards: true)`.
4. `last_reward_block` is set to the current block; no rewards are distributed.
5. Sequencer's legitimate `update_rewards(staker_address, disable_rewards: false)` call reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers receive zero rewards for that block.
7. Attacker repeats every block — all staker rewards are permanently frozen. [6](#0-5) [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
