### Title
`TurnBridge` Is a Permanent No-Op, Bypassing Admin Bridge-Disable Authority — (`x/cronos/keeper/msg_server.go`)

### Summary
The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is an unimplemented stub that returns `nil, nil` unconditionally. It performs no permission check and writes no state. As a result, the admin's ability to disable the bridge — the intended emergency circuit-breaker — is silently and permanently broken. Any unprivileged user can continue to execute bridge operations (token conversion, IBC transfers via EVM hooks) even after the admin has "successfully" called `TurnBridge(false)`.

### Finding Description

The `TurnBridge` handler is defined as:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Compare this with the correctly implemented `UpdateTokenMapping`, which checks `CanChangeTokenMapping` before acting:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [2](#0-1) 

The permission system explicitly defines `CanTurnBridge` as a distinct bit:

```go
CanChangeTokenMapping uint64 = 1 << iota // 1
CanTurnBridge                             // 2
All = CanChangeTokenMapping | CanTurnBridge // 3
``` [3](#0-2) 

The `CanTurnBridge` permission exists, the `MsgTurnBridge` message type exists, and the ADR explicitly documents `bridge_active` as the emergency control parameter. Yet the handler that is supposed to set that parameter does nothing.

None of the bridge execution paths — `ConvertVouchersToEvmCoins`, `IbcTransferCoins`, `SendToIbcHandler`, `SendToIbcV2Handler`, `SendCroToIbcHandler`, `SendToAccountHandler` — check any `bridge_active` state, because `TurnBridge` never writes one. [4](#0-3) [5](#0-4) [6](#0-5) 

This is structurally identical to the external report's root cause: an operation that should be blocked during a restricted state (`isUnderLiquidation`) proceeds because the guard is absent. Here, the restricted state (`bridge_active = false`) can never be written, so the guard can never fire.

### Impact Explanation

**High — Bypass of Cronos admin bridge authorization checks.**

An admin holding `CanTurnBridge` permission calls `MsgTurnBridge{Enable: false}` to halt the bridge during a security incident (e.g., oracle manipulation, contract exploit, IBC channel compromise). The transaction is accepted and returns success. No state is written. Every bridge path — `MsgConvertVouchers`, `MsgTransferTokens`, and all EVM-hook-triggered IBC transfers — continues to execute without restriction. Tokens continue to flow across the bridge against the admin's explicit intent, potentially draining escrow or minting unbacked vouchers during the window of the incident.

### Likelihood Explanation

The `TurnBridge` function is reachable by any permissioned address (or the admin) via a standard Cosmos SDK message. The bug is triggered the moment any operator attempts to use the intended emergency circuit-breaker. Given that the function silently succeeds, the operator may not immediately realize the bridge is still active, widening the damage window. Likelihood is **medium** (requires an emergency scenario) but the impact is severe.

### Recommendation

Implement `TurnBridge` to:
1. Check `CanTurnBridge` permission (mirroring `UpdateTokenMapping`'s pattern).
2. Read current params, set `params.BridgeActive = msg.Enable`, and persist via `k.SetParams`.
3. Add a `bridge_active` guard at the top of `ConvertVouchersToEvmCoins`, `IbcTransferCoins`, and each EVM log handler's `Handle` method, returning an error when `bridge_active == false`.

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    params := k.GetParams(ctx)
    params.BridgeActive = msg.Enable
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

### Proof of Concept

```
1. Grant address A the CanTurnBridge permission via MsgUpdatePermissions (admin).
2. Address A submits MsgTurnBridge{Enable: false}.
3. Transaction succeeds (code 0, no error).
4. Address B (unprivileged) submits MsgConvertVouchers or fires a __CronosSendToIbc EVM event.
5. The bridge operation completes successfully — no bridge_active check fires.
6. Repeat step 4 indefinitely; the bridge is permanently open regardless of admin intent.
```

func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
	}
	defer func() {
		for _, a := range coins {
			if a.Amount.IsInt64() {
				telemetry.SetGaugeWithLabels(
					[]string{"tx", "msg", "ConvertVouchersToEvmCoins"},
					float32(a.Amount.Int64()),
					[]metrics.Label{telemetry.NewLabel("denom", a.Denom)},
				)
			}
		}
	}()
	return nil
}

func (h SendToIbcHandler) handle(
	ctx sdk.Context,
	contract common.Address,
	senderAddress common.Address,
	recipient string,
	amountInt *big.Int,
	id *big.Int,
) error {
	denom, found := h.cronosKeeper.GetDenomByContract(ctx, contract)
	if !found {
		return fmt.Errorf("contract %s is not connected to native token", contract)
	}

	if !types.IsValidIBCDenom(denom) && !types.IsValidCronosDenom(denom) {
		return fmt.Errorf("the native token associated with the contract %s is neither an ibc voucher or a cronos token", contract)
	}

	contractAddr := sdk.AccAddress(contract.Bytes())
	sender := sdk.AccAddress(senderAddress.Bytes())
	amount := sdkmath.NewIntFromBigInt(amountInt)
	coins := sdk.NewCoins(sdk.NewCoin(denom, amount))

	var err error
	if types.IsSourceCoin(denom) {
		// it is a source token, we need to mint coins
		if err = h.bankKeeper.MintCoins(ctx, types.ModuleName, coins); err != nil {
			return err
		}
		// send the coin to the user
		if err = h.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, sender, coins); err != nil {
			return err
		}
	} else {
		// First, transfer IBC coin to user so that he will be the refunded address if transfer fails
		if err = h.bankKeeper.SendCoins(ctx, contractAddr, sender, coins); err != nil {
			return err
		}
	}

	channelId := ""
	if id != nil {
		channelId = "channel-" + id.String()
	}
	// Initiate IBC transfer from sender account
	if err = h.cronosKeeper.IbcTransferCoins(ctx, sender.String(), recipient, coins, channelId); err != nil {
		return err
	}
	return nil
}
