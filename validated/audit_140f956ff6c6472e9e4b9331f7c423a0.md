### Title
Gateway `skip_stateful_validations` Admits Invoke Transactions with Invalid Signatures via Overly Broad Mempool Presence Check - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function in the gateway's stateful transaction validator skips the `__validate__` entry point (signature verification) for any invoke transaction with `nonce == 1` when `account_tx_in_pool_or_recent_block` returns `true`. The check is too broad: it returns `true` if **any** transaction from the account is in the mempool pool, not exclusively a `deploy_account` transaction. An attacker can exploit this by observing a legitimate user's pending `deploy_account` in the mempool and immediately front-running the expected `invoke(nonce=1)` with an invalid-signature transaction that bypasses `__validate__` entirely.
