### Title
Missing Identity-Point Validation of `app_pk` in CKD Protocol Initialization - (File: `src/confidential_key_derivation/protocol.rs`)

### Summary
The `ckd()` function accepts `app_pk: PublicKey` (the application's ElGamal public key) without validating that it is not the group identity element. If `app_pk` is the identity, the ElGamal masking of the confidential derived key is completely bypassed, and the protocol output directly exposes the aggregate secret material.

### Finding Description
In `src/confidential_key_derivation/protocol.rs`, the public entry point `ckd()` accepts `app_pk: PublicKey` and passes it through to `run_ckd_protocol()` and ultimately to `compute_signature_share()` without any validation: [1](#0-0) 

Inside `compute_signature_share()`, the ElGamal masking term is computed as: [2](#0-1) 

The security of the CKD protocol depends entirely on the masking term `app_pk * y` being non-trivial. When `app_pk` is the group identity element:

- `app_pk * y.0 = identity`
- `big_c = big_s + identity = big_s = hash_point * private_share.to_scalar()`

Each participant's `C_i` is then just their unmasked secret share contribution. The coordinator aggregates these: [3](#0-2) 

The resulting `norm_big_c` accumulates to `msk * H(pk, app_id)` — the confidential derived key itself — with no masking whatsoever. The `CKDOutput` directly contains the secret. No other validation in `ckd()` catches this: [4](#0-3) 

The checks present (participant count, duplicates, membership) are all unrelated to the cryptographic validity of `app_pk`.

### Impact Explanation
**Critical — Extraction/disclosure of confidential derived secrets.**

The entire confidentiality guarantee of the CKD protocol rests on the ElGamal blinding: `C = S + y * A`. With `A = identity`, the blinding term vanishes. The coordinator's output `CKDOutput { big_y, big_c }` has `big_c = msk * H(pk, app_id)`, which is the confidential derived key in the clear. Any party that receives the coordinator's output (including the application that initiated the CKD call) obtains the secret without needing `app_sk`.

### Likelihood Explanation
The `ckd()` function is a public library API callable by any application integrating the library. A malicious or misconfigured caller can trivially pass `PublicKey::identity()` (the BLS12-381 G1 identity point). There is no type-level enforcement preventing this, and the function returns `Ok(...)` with a fully-formed protocol that runs to completion, producing a compromised output with no error signal to honest participants.

### Recommendation
Add an explicit identity-point check in `ckd()` before the protocol is initialized:

```rust
// Reject the identity element — masking would be completely bypassed
if app_pk == PublicKey::identity() {
    return Err(InitializationError::BadParameters(
        "app_pk must not be the identity element".to_string(),
    ));
}
```

This mirrors the pattern already used throughout the codebase for other critical parameter checks (e.g., `threshold < 2`, `participants.len() < 2`). [5](#0-4) 

### Proof of Concept

1. Caller invokes `ckd(participants, coordinator, me, key_pair, app_id, ElementG1::identity(), rng)`.
2. All validation checks in `ckd()` pass — no error is returned.
3. Each participant executes `compute_signature_share()`. With `app_pk = identity`, `big_c = big_s = hash_point * private_share`.
4. The coordinator aggregates: `C_total = Σ λᵢ · Cᵢ = msk · H(pk, app_id)` — the confidential derived key, fully unmasked.
5. The attacker reads `ckd_output.big_c` directly to obtain the confidential derived key without ever possessing `app_sk`. [6](#0-5)

pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    let comms = Comms::new();
    let chan = comms.shared_channel();

    let fut = run_ckd_protocol(
        chan,
        coordinator,
        me,
        participants,
        key_pair,
        app_id.into(),
        app_pk,
        rng,
    );
    Ok(make_protocol(comms, fut))

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
