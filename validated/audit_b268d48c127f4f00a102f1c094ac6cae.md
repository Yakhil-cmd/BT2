### Title
Reward Supplier Attempts Token Transfer Before L1 Funds Arrive, Temporarily Freezing Unclaimed Yield - (`src/reward_supplier/reward_supplier.cairo`)

### Summary

The `RewardSupplier` contract tracks `unclaimed_rewards` as an accounting value that can exceed the contract's actual token balance when L1 mint requests are in-flight. The `claim_rewards` function only validates `amount <= unclaimed_rewards` but then calls `checked_transfer`, which panics if the actual token balance is insufficient. Any staker or delegator who calls `claim_rewards` during the L1→L2 bridge latency window will have their transaction revert, temporarily freezing their unclaimed yield.

### Finding Description

The reward flow works as follows:

1. When rewards are computed, `update_unclaimed_rewards_from_staking_contract` increases `unclaimed_rewards` and calls `request_funds`.
2. `request_funds` computes `credit = balance + l1_pending_requested_amount` and sends L1 mint messages if `credit < unclaimed_rewards + threshold`, then increments `l1_pending_requested_amount`.
3. The L1 `RewardSupplier` mints STRK and bridges it via StarkGate. Only when StarkGate calls `on_receive` does the actual token balance increase. [1](#0-0) 

The invariant maintained by `request_funds` is:

```
balance + l1_pending_requested_amount >= unclaimed_rewards + threshold
```

This means the actual `balance` can be **less than** `unclaimed_rewards` whenever `l1_pending_requested_amount > threshold`. During this window, `claim_rewards` is called: [2](#0-1) 

The guard `assert!(amount <= unclaimed_rewards)` passes because the accounting is correct, but the subsequent `checked_transfer` panics because the contract's actual token balance is insufficient.

The staking contract reaches this path via `send_rewards_to_staker` → `claim_from_reward_supplier` → `reward_supplier.claim_rewards`: [3](#0-2) 

This is triggered by `staking.claim_rewards`, `staking.unstake_action`, and reward distribution calls. [4](#0-3) 

### Impact Explanation

Any staker or delegator who calls `claim_rewards` (or `unstake_action`, which also calls `send_rewards_to_staker`) while `l1_pending_requested_amount > 0` and `balance < unclaimed_rewards` will have their transaction revert. This is a **temporary freezing of unclaimed yield**: users cannot withdraw earned rewards until the L1→L2 bridge completes, which can take hours to days depending on L1 finality and bridge processing. This matches the **High: Temporary freezing of unclaimed yield** impact category.

### Likelihood Explanation

The L1→L2 bridge latency is a normal, expected part of the protocol's operation. Every time rewards accumulate beyond the contract's current balance, a mint request is sent and `l1_pending_requested_amount` becomes non-zero. During this window — which is the **normal operating state** of the protocol — any reward claim will fail. Since reward claims are a routine user action, this condition is highly likely to be triggered by ordinary stakers and delegators.

### Recommendation

In `claim_rewards`, add a check that the actual token balance is sufficient before attempting the transfer:

```cairo
fn claim_rewards(ref self: ContractState, amount: Amount) {
    ...
    let unclaimed_rewards = self.unclaimed_rewards.read();
    assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);
    
    // Add: verify actual balance is sufficient
    let balance = self.token_dispatcher.read().balance_of(account: get_contract_address());
    assert!(amount.into() <= balance, "{}", Error::INSUFFICIENT_BALANCE_FOR_CLAIM);
    
    self.unclaimed_rewards.write(unclaimed_rewards - amount);
    ...
}
```

Alternatively, the protocol should ensure that `claim_rewards` is never called until the L1 funds have arrived, by gating reward claims on `balance >= amount` rather than `unclaimed_rewards >= amount`.

### Proof of Concept

1. Staker earns rewards; `update_unclaimed_rewards_from_staking_contract(rewards: R)` is called.
2. `unclaimed_rewards` becomes `R + STRK_IN_FRIS`. `request_funds` finds `balance < unclaimed_rewards + threshold` and sends L1 mint messages; `l1_pending_requested_amount` is set to `N * base_mint_amount`.
3. L1 funds are in transit. At this point: `balance < unclaimed_rewards` (since `l1_pending_requested_amount > threshold`).
4. Staker calls `staking.claim_rewards(staker_address)`.
5. Staking contract calls `claim_from_reward_supplier(amount: R)`.
6. `reward_supplier.claim_rewards(R)`: `R <= unclaimed_rewards` passes.
7. `checked_transfer(recipient: staking_contract, amount: R)` panics with `INSUFFICIENT_BALANCE` because `balance < R`.
8. Transaction reverts. Staker cannot claim until `on_receive` is called by StarkGate. [5](#0-4) [6](#0-5)

        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }

        fn claim_rewards(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let caller_address = get_caller_address();
            let reward_address = staker_info.reward_address;
            assert!(
                caller_address == staker_address || caller_address == reward_address,
                "{}",
                Error::CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            // Transfer rewards to staker's reward address and write updated staker info to storage.
            // Note: `send_rewards_to_staker` alters `staker_info` thus commit to storage is
            // performed only after that.
            let amount = staker_info.unclaimed_rewards_own;
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            self.write_staker_info(:staker_address, :staker_info);
            amount
        }

        fn send_rewards_to_staker(
            ref self: ContractState,
            staker_address: ContractAddress,
            ref staker_info: InternalStakerInfoLatest,
            token_dispatcher: IERC20Dispatcher,
        ) {
            let reward_address = staker_info.reward_address;
            let amount = staker_info.unclaimed_rewards_own;
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();

            claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
            token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
            staker_info.unclaimed_rewards_own = Zero::zero();

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
