### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Trigger Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check whatsoever. Any unprivileged address can invoke it, triggering real reward accounting and token transfers from the `RewardSupplier` to delegation pools.

### Finding Description
The protocol specification explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation of `StakingRewardsManagerImpl::update_rewards` performs no `get_caller_address()` check. The function begins with `self.general_prerequisites()` (a pause check) and a `REWARDS_ALREADY_UPDATED` block guard, then proceeds directly to reward calculation and distribution: [2](#0-1) 

Compare this to other sensitive functions in the same contract — for example, `claim_rewards` explicitly asserts the caller is the staker or reward address: [3](#0-2) 

And `update_rewards_from_attestation_contract` asserts the caller is the attestation contract: [4](#0-3) 

No such guard exists in `update_rewards`. The interface definition also carries no access-control annotation: [5](#0-4) 

### Impact Explanation
When `disable_rewards: false` is passed and consensus rewards are active, `_update_rewards` is called. This function:
1. Calls `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract(rewards: staker_rewards + total_pools_rewards)` — inflating the `RewardSupplier`'s `unclaimed_rewards` counter and potentially triggering an L1 mint request.
2. Immediately calls `claim_from_reward_supplier` to pull `total_pools_rewards` tokens out of the `RewardSupplier` and transfer them to delegation pool contracts.
3. Increments `staker_info.unclaimed_rewards_own` for the targeted staker. [6](#0-5) 

An attacker can call `update_rewards` once per block (the `last_reward_block` guard prevents same-block repetition) for any active staker, causing:
- **Theft of unclaimed yield**: Pool rewards are immediately transferred out of the `RewardSupplier` to the pool contract on every attacker-triggered call. Delegators can then claim these prematurely distributed rewards. The attacker can front-run the legitimate sequencer call each block, ensuring the sequencer's own call reverts with `REWARDS_ALREADY_UPDATED`, permanently denying the sequencer the ability to call `update_rewards` with `disable_rewards: true` (which is used to record block participation without distributing rewards).
- **Permanent freezing of unclaimed yield**: By always calling with `disable_rewards: false` before the sequencer, the attacker prevents the sequencer from ever using `disable_rewards: true` for that block, corrupting the reward-distribution logic and potentially freezing yield for stakers who should not have received rewards in that block.

This maps to the allowed impacts: **Theft of unclaimed yield** and **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The function is publicly callable on-chain with no authentication barrier. Any address can call it at any block after consensus rewards are activated. The only cost is gas. The `last_reward_block` guard means only one call succeeds per block, but an attacker can reliably front-run the sequencer every block using standard mempool priority. Likelihood is **High**.

### Recommendation
Add a caller restriction matching the specification. Store the authorized sequencer address during initialization and assert it at the top of `update_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

Alternatively, if the sequencer address is not stored, restrict to the `app_governor` or another already-tracked privileged role until a dedicated sequencer address storage slot is added.

### Proof of Concept
1. Consensus rewards are activated (i.e., `consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. A staker with a delegation pool exists and has been active for K+ epochs.
3. Attacker (any EOA) calls `update_rewards(staker_address: victim_staker, disable_rewards: false)` at block N.
4. The call succeeds: `last_reward_block` is set to N, `staker_info.unclaimed_rewards_own` is incremented, and `total_pools_rewards` tokens are transferred from `RewardSupplier` to the pool contract.
5. When the legitimate sequencer attempts `update_rewards(staker_address: any_staker, disable_rewards: true)` at block N, it reverts with `REWARDS_ALREADY_UPDATED`.
6. The attacker repeats this every block, permanently preventing the sequencer from controlling reward distribution. [7](#0-6) [8](#0-7)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
