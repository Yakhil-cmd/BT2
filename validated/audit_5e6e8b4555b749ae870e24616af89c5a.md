### Title
Block-List Authorization Enforced Only at `CheckTx`, Bypassed at `DeliverTx` — (`app/block_address.go`)

### Summary
`BlockAddressesDecorator.AnteHandle` wraps all block-list enforcement inside `if ctx.IsCheckTx()`. During actual block execution (`DeliverTx`), `IsCheckTx()` returns `false`, so the entire guard is skipped and the decorator unconditionally calls `next(ctx, tx, simulate)`. Any transaction from a blocked address that reaches `DeliverTx` — whether via validator-direct inclusion, a `PrepareProposal` injection, or a mempool race after the address is blocked — executes without restriction.

### Finding Description
`BlockAddressesDecorator.AnteHandle` in `app/block_address.go` performs all three block-list checks (signer address, EVM destination address, EIP-7702 authorization list, and `MsgStoreBlockList` admin guard) exclusively inside the `if ctx.IsCheckTx()` branch:

```go
func (bad BlockAddressesDecorator) AnteHandle(...) (newCtx sdk.Context, err error) {
    if ctx.IsCheckTx() {
        // signer check
        // EVM destination check
        // EIP-7702 auth list check
        // MsgStoreBlockList admin check
    }
    return next(ctx, tx, simulate)   // ← always reached during DeliverTx
}
```

In Cosmos SDK / CometBFT, `CheckTx` is the mempool-admission gate; `DeliverTx` is the actual state-transition path. Because `IsCheckTx()` is `false` during `DeliverTx`, the decorator is a no-op at execution time. The block list therefore provides only a mempool-level hint, not a consensus-level enforcement. [1](#0-0) 

### Impact Explanation
The block list is a security-critical control (OFAC-style address sanctions, emergency circuit-breaker). Bypassing it allows a blocked address to:
- Send EVM transactions to or from blocked addresses
- Execute `MsgStoreBlockList` if the admin address is somehow blocked
- Interact with CRC20/CRC21 bridge functions, IBC transfers, and precompile calls that the block list is intended to prevent

This maps to **High: Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks**.

### Likelihood Explanation
Two unprivileged paths exist:

1. **Mempool race**: Attacker submits a transaction → it passes `CheckTx` (not yet blocked) → admin adds the address to the block list → the transaction is still in the mempool → a validator includes it in a block → `DeliverTx` executes it without checking the block list.

2. **Validator-direct inclusion**: In CometBFT, the block proposer can include transactions that were never admitted through the mempool (e.g., via `PrepareProposal`). A validator (or a validator colluding with the attacker) can include a blocked address's transaction directly in a block proposal, bypassing `CheckTx` entirely.

Path 1 requires no special privilege and is realistic whenever the block list is updated reactively (e.g., in response to an incident).

### Recommendation
Move the block-list enforcement outside the `if ctx.IsCheckTx()` guard so it runs on every ante-handler invocation, including `DeliverTx`:

```go
func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
    // Remove the `if ctx.IsCheckTx()` wrapper; enforce on every path
    if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
        // ... signer checks ...
    }
    for _, msg := range tx.GetMsgs() {
        // ... EVM destination, EIP-7702, MsgStoreBlockList checks ...
    }
    return next(ctx, tx, simulate)
}
```

Alternatively, keep the mempool-level check and add a separate `DeliverTx`-only check, but the simplest fix is to remove the `IsCheckTx()` guard entirely.

### Proof of Concept

1. Admin deploys Cronos with a block list containing `BLOCKED_ADDR`.
2. `BLOCKED_ADDR` submits `MsgEthereumTx` targeting a CRC20 bridge function. The transaction enters the mempool and passes `CheckTx` (assume it was submitted before being blocked, or the node's mempool is stale).
3. A validator proposes a block containing this transaction (either from the mempool or injected directly via `PrepareProposal`).
4. During `DeliverTx`, `BlockAddressesDecorator.AnteHandle` is called with `ctx.IsCheckTx() == false`. The entire block-list check is skipped.
5. `next(ctx, tx, simulate)` is called; the transaction executes successfully.
6. `BLOCKED_ADDR` has bypassed the block list and interacted with the bridge/CRC20/IBC stack. [2](#0-1) [3](#0-2)

```go
func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)
```
