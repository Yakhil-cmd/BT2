Based on my investigation, this is a real vulnerability.

**Key findings:**

1. The spec explicitly states `update_rewards` access control is **"Only starkware sequencer"** [1](#0-0) 

2. The actual implementation's `general_prerequisites()` only checks: contract is unpaused + caller is not zero address — **no sequencer/role check exists**: [2](#0-1) 

3. `last_reward_block` is a **global** (contract-wide) storage variable. The guard `current_block_number > self.last_reward_block.read()` means only **one** `update_rewards` call succeeds per block, for the entire contract across all stakers: [3](#0-2) 

4. After passing the block guard, `last_reward_block` is written **before** the `disable_rewards` branch, so even a `disable_rewards: true` call permanently consumes the block's reward slot: [4](#0-3)

### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress All Block Rewards — (`src/staking/staking.cairo`)

### Summary
`update_rewards` is documented as restricted to the Starkware sequencer but has no caller check in the implementation. Because `last_reward_block` is a global per-block gate, any unprivileged attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to permanently consume the block's reward slot without distributing any rewards, starving all stakers of consensus block rewards indefinitely.

### Finding Description
`StakingRewardsManagerImpl::update_rewards` in `src/staking/staking.cairo` calls only `self.general_prerequisites()`, which enforces:
- Contract is not paused
- Caller is not the zero address [5](#0-4) 

There is no role check, no `assert_caller_is_sequencer`, and no allowlist. The spec's "Only starkware sequencer" constraint is entirely absent from the code.

The global `last_reward_block` is written unconditionally at line 1485 before the `disable_rewards` early-return at line 1487. This means:

```
call update_rewards(valid_staker, disable_rewards=true)
  → passes general_prerequisites (not paused, caller != 0)
  → passes block guard (new block)
  → writes last_reward_block = current_block   ← slot consumed
  → disable_rewards == true → return            ← no rewards distributed
```

Any subsequent call in the same block (including the legitimate sequencer call) hits `REWARDS_ALREADY_UPDATED` and reverts. [4](#0-3) 

### Impact Explanation
An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` at the start of every block permanently prevents all consensus block rewards from being distributed to any staker. `unclaimed_rewards_own` for every staker remains zero. This matches **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The attack requires no capital, no privileged role, and no special setup — only a valid active staker address (publicly readable from chain events) and a transaction submitted each block. It is trivially automatable with a bot. The attacker has no profit motive but causes maximum damage to all stakers and delegators.

### Recommendation
Add a sequencer/role guard at the top of `update_rewards`, consistent with the spec:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert_caller_is_sequencer(@self);  // enforce spec: "Only starkware sequencer"
    ...
}
```

Alternatively, restrict via the existing roles framework (e.g., `SEQUENCER_ROLE`) so only the authorized sequencer address can invoke this function.

### Proof of Concept
1. Deploy two active stakers A and B (both past the K-epoch activation window, consensus rewards active).
2. Each block, before the sequencer acts, submit: `update_rewards(staker_A, disable_rewards: true)`.
3. The sequencer's legitimate call `update_rewards(staker_B, disable_rewards: false)` reverts with `REWARDS_ALREADY_UPDATED`.
4. After N blocks, observe `staker_A.unclaimed_rewards_own == 0` and `staker_B.unclaimed_rewards_own == 0` — no rewards ever distributed.
5. Compare against a baseline run without the hostile calls: both stakers accumulate expected block rewards each block.

The test infrastructure in `src/staking/tests/test.cairo` already demonstrates that `REWARDS_ALREADY_UPDATED` fires on a second same-block call regardless of caller identity, confirming the global gate is the mechanism. [6](#0-5) [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
