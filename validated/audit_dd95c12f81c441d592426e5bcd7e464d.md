### Title
Reentrancy in `claim_rewards` Allows Theft of Unclaimed Yield via Malicious `reward_address` - (File: src/staking/staking.cairo)

### Summary
The `claim_rewards` function in the Staking contract violates the checks-effects-interactions pattern. The staker's `unclaimed_rewards_own` field is not committed to storage before external calls are made to the reward supplier and the token contract. A staker who sets their `reward_address` to a malicious contract can re-enter `claim_rewards` repeatedly, draining the reward supplier's shared `unclaimed_rewards` pool.

### Finding Description
In `Staking::claim_rewards`, the function reads `staker_info` from storage, calls `send_rewards_to_staker`, and only then calls `write_staker_info` to persist the zeroed `unclaimed_rewards_own`: [1](#0-0) 

Inside `send_rewards_to_staker`, the execution order is:
1. Read `amount = staker_info.unclaimed_rewards_own` (from the in-memory struct, which mirrors storage at this point)
2. External call: `claim_from_reward_supplier(amount)` → reward supplier transfers STRK to the staking contract
3. External call: `token.checked_transfer(recipient: reward_address, amount)` → **reentrancy window opens here**
4. In-memory only: `staker_info.unclaimed_rewards_own = Zero::zero()` [2](#0-1) 

Storage is not updated until `write_staker_info` is called back in `claim_rewards`, **after** `send_rewards_to_staker` returns: [3](#0-2) 

During the token transfer at step 3, if `reward_address` is a malicious contract, it re-enters `claim_rewards(staker_address)`. The re-entrant call reads `staker_info` from storage, which still shows `unclaimed_rewards_own = R` (not yet zeroed). The authorization check passes because `caller == reward_address` is allowed: [4](#0-3) 

The re-entrant call then calls `reward_supplier.claim_rewards(R)` again. The reward supplier's `unclaimed_rewards` is a shared pool accumulating rewards for all stakers, so it typically holds far more than `R`: [5](#0-4) 

The same reentrancy window exists in `unstake_action`, which also calls `send_rewards_to_staker` before committing state, though the developers added a comment acknowledging partial reentrancy concern only for the subsequent calls: [6](#0-5) 

### Impact Explanation
Each re-entrant loop iteration claims an additional `R` STRK from the reward supplier's shared `unclaimed_rewards` pool. The attacker can repeat this until the reward supplier's balance is exhausted. This directly steals yield that belongs to other stakers and pool members — **Theft of unclaimed yield (High)**.

### Likelihood Explanation
Any staker can set their `reward_address` to an arbitrary contract address via `change_reward_address`. This is an unprivileged, permissionless action. The attacker only needs to: (1) stake the minimum amount, (2) wait for rewards to accumulate, (3) set `reward_address` to a malicious contract, (4) call `claim_rewards`. No privileged access, leaked keys, or external dependencies are required.

### Recommendation
Apply the checks-effects-interactions pattern: zero `staker_info.unclaimed_rewards_own` and commit it to storage **before** making any external calls. Concretely, in `send_rewards_to_staker` (or in `claim_rewards` before calling it), write the zeroed `unclaimed_rewards_own` to storage prior to calling `claim_from_reward_supplier` or `token.checked_transfer`. Alternatively, add a reentrancy guard (mutex flag in storage) to all external-facing state-mutating functions.

```
fn on_receive_strk(...) {
    // Called when staking contract transfers rewards
    if reentry_count < MAX_REENTRIES {
        reentry_count += 1;
        // Storage still shows unclaimed_rewards_own = R
        staking.claim_rewards(attacker_staker_address);
    }
}

// Attack sequence:
// 1. attacker.stake(min_stake, reward_address=MaliciousContract, ...)
// 2. Wait for rewards to accumulate (unclaimed_rewards_own = R)
// 3. staking.claim_rewards(attacker_staker_address)
//    → send_rewards_to_staker called
//    → reward_supplier.claim_rewards(R) succeeds (transfers R to staking)
//    → token.transfer(MaliciousContract, R) triggers on_receive_strk
//    → MaliciousContract re-enters claim_rewards
//    → storage still has unclaimed_rewards_own = R
//    → reward_supplier.claim_rewards(R) called again (succeeds if pool > R)
//    → token.transfer(MaliciousContract, R) again
//    → ... repeated N times until reward_supplier.unclaimed_rewards < R
// Net theft: N * R STRK from other stakers' yield pool
```

```text
        fn send_rewards_to_staker(
            ref self: ContractState,
            staker_address: ContractAddress,
            ref staker_info: InternalStakerInfoLatest,
            token_dispatcher: IERC20Dispatcher,
        ) {
            let reward_address = staker_info.reward_address;
            let amount = staker_info.unclaimed_rewards_own;
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();

            claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
            token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
            staker_info.unclaimed_rewards_own = Zero::zero();

            self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
        }
```

```text
        fn claim_rewards(ref self: ContractState, amount: Amount) {
            // Asserts.
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            let unclaimed_rewards = self.unclaimed_rewards.read();
            assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Update unclaimed_rewards and transfer the requested rewards to the staking contract.
            self.unclaimed_rewards.write(unclaimed_rewards - amount);
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
        }
```
