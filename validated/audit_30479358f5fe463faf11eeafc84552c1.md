### Title
Presignature Reuse Enabled by Non-Consuming `rerandomize_presign` API Allows Private Key Extraction — (`src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both the OT-based and Robust ECDSA implementations expose `rerandomize_presign` as a function that takes `&PresignOutput` (by shared reference) rather than consuming the value. Combined with `PresignOutput` deriving `Clone`, the library provides no API-level enforcement of the documented single-use invariant. A malicious coordinator can rerandomize the same `PresignOutput` for two different `(msg_hash, tweak)` pairs, run two signing sessions, and recover the aggregate private key from the resulting signatures via a standard related-nonce attack.

---

### Finding Description

Both ECDSA variants document an absolute constraint:

> **"Never reuse a presignature, even across failed, aborted, or partially completed signing sessions."**
> — `docs/ecdsa/robust_ecdsa/signing.md` lines 176–177

The documentation further explains the consequence:

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks."
> — `docs/ecdsa/robust_ecdsa/signing.md` lines 152–154

Despite this, the library's API makes reuse trivially possible:

Because `rerandomize_presign` borrows rather than moves `PresignOutput`, any caller can invoke it an arbitrary number of times on the same presignature with different `RerandomizationArguments` (different `msg_hash`, `tweak`, or `rho`). Each call produces a distinct `RerandomizedPresignOutput` with a different `big_r = delta * R`, but all share the same underlying nonce `k`. The relationship is:

```
k_rerandomized_1 = k * delta_1^{-1}
k_rerandomized_2 = k * delta_2^{-1}
=> k_rerandomized_1 / k_rerandomized_2 = delta_2 / delta_1  (fully known to attacker)
```

This is a related-nonce scenario. Given two ECDSA signatures `(R1, s1)` and `(R2, s2)` produced from the same `k` with different messages `h1`, `h2`, the private key `x` is recoverable by standard linear algebra over the scalar field.

The `sign()` function for the OT-based variant accepts any `RerandomizedPresignOutput` without any uniqueness check: [5](#0-4) 

The robust variant's `sign()` similarly accepts any `RerandomizedPresignOutput`: [6](#0-5) 

Neither signing function checks whether the presignature has been used before, nor does any other component in the library maintain such state.

---

### Impact Explanation

A malicious coordinator who participates in one presigning session obtains a `PresignOutput`. By calling `rerandomize_presign` twice with different `(msg_hash, tweak)` arguments, the coordinator produces two valid `RerandomizedPresignOutput` values from the same underlying nonce. Running two signing sessions (each with the minimum required honest participants) yields two ECDSA signatures with multiplicatively related nonces. Standard related-nonce key recovery then extracts the aggregate private signing key `x` in full.

**Impact class:** Critical — Extraction of the aggregate private signing key, enabling unauthorized creation of valid threshold signatures for any attacker-chosen message.

---

### Likelihood Explanation

The coordinator role is a normal, non-privileged participant in the signing protocol. No leaked keys, external compromise, or cryptographic break is required. The attack requires only:
1. Participation in one presigning session (standard protocol behavior).
2. Two calls to `rerandomize_presign` with different arguments (a single function call each, using the public API).
3. Two signing sessions with the minimum honest quorum.

The `PresignOutput` type is `Clone` and `Serialize`/`Deserialize`, making it trivial to store and reuse. The library provides no mechanism — not a type-system constraint, not a runtime check, not a session registry — to prevent this.

---

### Recommendation

1. **Make `rerandomize_presign` consume `PresignOutput`** by taking `presignature: PresignOutput` (by value) instead of `presignature: &PresignOutput`. This leverages Rust's ownership system to enforce single-use at compile time.

2. **Remove `Clone` from `PresignOutput`** in both variants. A presignature that cannot be cloned cannot be silently duplicated before being passed to `rerandomize_presign`.

3. **Apply the same treatment to `RerandomizedPresignOutput`**: remove `Clone` and ensure `sign()` consumes it by value (it already takes by value, but `Clone` allows callers to circumvent this).

---

### Proof of Concept

```rust
// Attacker is the coordinator. They participated in presigning and hold:
let presign_output: PresignOutput = /* result of presigning protocol */;

// Rerandomize the SAME presignature for two different messages.
let args1 = RerandomizationArguments { big_r: presign_output.big_r, tweak: tweak1, /* h1, rho */ };
let args2 = RerandomizationArguments { big_r: presign_output.big_r, tweak: tweak2, /* h2, rho */ };

// Both calls succeed because rerandomize_presign takes &PresignOutput (not consumed).
let rerand1 = RerandomizedPresignOutput::rerandomize_presign(&presign_output, &args1).unwrap();
let rerand2 = RerandomizedPresignOutput::rerandomize_presign(&presign_output, &args2).unwrap();

// Run two signing sessions with honest participants.
// Session 1 produces (R1, s1) for message h1.
// Session 2 produces (R2, s2) for message h2.
// Both R1 and R2 are derived from the same underlying nonce k.
//
// Key recovery:
//   s1 = h1 * k1 + Rx1 * sigma1   (k1 = k/delta1)
//   s2 = h2 * k2 + Rx2 * sigma2   (k2 = k/delta2)
//   ratio k1/k2 = delta2/delta1 is known to the attacker.
//   Solving the two-equation system yields x (the private key).
```

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    max_malicious: impl Into<MaxMalicious>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= 2 * max_malicious + 1
    let robust_ecdsa_threshold = max_malicious
        .into()
        .value()
        .checked_mul(2)
        .and_then(|v| v.checked_add(1))
        .ok_or_else(|| {
            InitializationError::BadParameters(
                "2*threshold+1 must be less than usize::MAX".to_string(),
            )
        })?;
    if robust_ecdsa_threshold > participants.len() {
        return Err(InitializationError::BadParameters(
            "2*max_malicious+1 must be less than or equals to participant count".to_string(),
        ));
    }

    // The next two conditions prevent split-view attacks
    // documented in docs/ecdsa/robust_ecdsa/signing.md
    if participants.len() != robust_ecdsa_threshold {
        return Err(InitializationError::BadParameters(
            "the number of participants during signing must be exactly 2*max_malicious+1 to avoid split view attacks".to_string(),
        ));
    }
    if bool::from(msg_hash.is_zero()) {
        return Err(InitializationError::BadParameters(
            "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
        ));
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
