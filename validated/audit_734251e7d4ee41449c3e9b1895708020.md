### Title
Missing Access Control on `update_rewards` Allows Any Caller to Consume Block Reward Slots Without Distributing Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public function with no caller restriction. Any unprivileged address can call it with `disable_rewards: true`, consuming the single per-block reward slot (`last_reward_block`) without distributing any consensus rewards. This is the direct analog of the M-06 finding: a function intended to be called only through a privileged entry point (the consensus mechanism) is instead callable by anyone, allowing the intended flow to be bypassed.

---

### Finding Description

`update_rewards` is implemented in `StakingRewardsManagerImpl` in `src/staking/staking.cairo`. Its only access guard is `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only: not-paused + caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
``` [2](#0-1) 

`general_prerequisites` contains no caller identity check beyond the zero-address guard.

The function then unconditionally writes the current block number to the global `last_reward_block` storage slot, and only distributes rewards if `disable_rewards == false` and consensus rewards are active: [3](#0-2) 

```cairo
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

Because `last_reward_block` is a **single global slot** (not per-staker), only one successful call to `update_rewards` is permitted per block. Any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

An attacker who calls `update_rewards(any_active_staker, true)` first in a block:
1. Advances `last_reward_block` to the current block.
2. Returns immediately without distributing rewards.
3. Causes every legitimate call to `update_rewards` in that block to revert.

The attacker needs only a valid, active staker address with non-zero balance (all public on-chain), and pays only gas.

---

### Impact Explanation

Every block in which the attacker front-runs the consensus mechanism's `update_rewards` call, **no staker receives block rewards**. Repeated across consecutive blocks, this permanently freezes unclaimed consensus yield for all stakers and their delegators. This maps to the allowed impact: **Permanent/temporary freezing of unclaimed yield (High)** and **Griefing with no profit motive but damage to users or protocol (Medium)**.

---

### Likelihood Explanation

- The function is fully public; no token balance, role, or privileged key is required.
- The attacker only needs to know any active staker address (trivially readable from on-chain events or `stakers` storage).
- The attack is repeatable every block at minimal cost.
- Front-running on Starknet is feasible via mempool observation or by simply submitting the transaction at the start of each block.

---

### Recommendation

Restrict `update_rewards` to the consensus mechanism's designated caller. The simplest fix is to add an `only_rewards_manager` (or equivalent) role check at the top of the function, analogous to how `update_rewards_from_attestation_contract` is restricted to the attestation contract: [5](#0-4) 

```cairo
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), ...);
    self.assert_caller_is_attestation_contract();   // ← access control
    ...
}
```

Apply the same pattern to `update_rewards`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_rewards_manager();   // add this
    ...
}
```

---

### Proof of Concept

1. Staker `S` is active with non-zero STRK balance. Consensus rewards are active (`is_pre_consensus() == false`).
2. Attacker `A` (any EOA) monitors the mempool for the consensus mechanism's `update_rewards(S, false)` transaction.
3. `A` submits `update_rewards(S, true)` with higher priority in the same block.
4. `A`'s transaction executes first: `last_reward_block` is set to the current block, no rewards distributed, function returns.
5. The consensus mechanism's `update_rewards(S, false)` reverts with `REWARDS_ALREADY_UPDATED`.
6. `S` and all delegators receive zero block rewards for that block.
7. `A` repeats every block. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
