### Title
Permanent Fund Freeze via Zero-Address `refund_address` in `ExitToNear` Precompile Error-Refund Path ŌĆö (File: `engine-precompiles/src/native.rs`)

---

### Summary

When the `error_refund` compile-time feature is enabled, the `ExitToNear` precompile parses a caller-supplied `refund_address` from the input bytes with no validation that it is non-zero. If the downstream NEAR `ft_transfer` promise fails and `refund_address` is `address(0)`, the engine calls `EvmErc20.mint(address(0), amount)`, which reverts unconditionally under OpenZeppelin's ERC-20 implementation. The callback then returns `ERR_REFUND_FAILURE`, the ERC-20 burn is already committed, and the user's tokens are permanently frozen.

### Finding Description

**Step 1 ŌĆö Attacker-controlled `refund_address` is parsed without zero-address validation.**

`parse_input` (compiled only when `error_refund` is enabled) reads bytes `[1..21]` of the raw precompile input as the `refund_address` with no check that the result is non-zero:

```rust
#[cfg(feature = "error_refund")]
fn parse_input(input: &[u8]) -> Result<(Address, &[u8]), ExitError> {
    validate_input_size(input, MIN_INPUT_SIZE, MAX_INPUT_SIZE)?;
    let mut buffer = [0; 20];
    buffer.copy_from_slice(&input[1..21]);
    let refund_address = Address::from_array(buffer);   // ŌåÉ no zero-address check
    Ok((refund_address, &input[21..]))
}
