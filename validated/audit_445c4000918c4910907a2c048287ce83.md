### Title
Global `last_reward_block` Blocks Multi-Staker Reward Distribution Per Block — (File: src/staking/staking.cairo)

---

### Summary

`last_reward_block` is a **single global** `BlockNumber` storage variable in the Staking contract, not a per-staker mapping. `update_rewards()` writes it to the current block number **before** distributing rewards, and the guard at the top of the function rejects any second call in the same block with `REWARDS_ALREADY_UPDATED`. Because the guard is global, the first call to `update_rewards` in any block — for **any** staker — permanently blocks every other staker from receiving rewards in that same block. Any unprivileged staker can exploit this to freeze unclaimed yield for all other stakers indefinitely.
