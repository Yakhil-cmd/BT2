### Title
BLS Public Key Aggregation Without Proof-of-Possession Enforcement Enables Rogue-Key Attack - (File: `fvm/crypto/crypto.go`)

### Summary

The FVM built-in function `AggregatePublicKeys` (exposed to Cadence as `BLS.aggregatePublicKeys()`) aggregates BLS public keys without verifying Proof of Possession (PoP) for any input key. Any unprivileged transaction sender or Cadence contract author participating in a BLS multi-signature scheme can supply a crafted rogue key — a valid BLS curve point for which no private key is known — to manipulate the aggregate key and gain unilateral signing authority.

### Finding Description

`AggregatePublicKeys` in `fvm/crypto/crypto.go` decodes each input key and immediately passes it to `crypto.AggregateBLSPublicKeys` with no PoP check:

```go
// AggregatePublicKeys aggregate multiple public keys into one; currently only works for BLS
func AggregatePublicKeys(keys []*runtime.PublicKey) (*runtime.PublicKey, error) {
    pks := make([]crypto.PublicKey, 0, len(keys))
    for _, key := range keys {
        // TODO: avoid validating the public keys again since Cadence makes sure runtime keys have been validated.
        pk, err := crypto.DecodePublicKey(crypto.BLSBLS12381, key.PublicKey)
        ...
        pks = append(pks, pk)
    }
    pk, err := crypto.AggregateBLSPublicKeys(pks)
    ...
}
``` [1](#0-0) 

The comment "Cadence makes sure runtime keys have been validated" refers only to structural curve-point validity (the key is a valid BLS12-381 point), **not** to proof of knowledge of the corresponding private key. PoP verification is a separate, optional call (`BLS.verifyPoP()`) that is never enforced by the aggregation path. [2](#0-1) 

This function is wired directly into the FVM environment and exposed to every Cadence script and transaction: [3](#0-2) 

The internal protocol aggregators (`SignatureAggregatorSameMessage`, `WeightedSignatureAggregator`, `TimeoutSignatureAggregator`) all explicitly document that they are only safe when PoPs have been verified externally. However, the Cadence-facing `AggregatePublicKeys` function carries no such enforcement and no such guarantee from its callers. [4](#0-3) 

### Impact Explanation

Any Cadence contract implementing a BLS multi-signature scheme (e.g., a multi-sig vault, DAO, or threshold wallet) that calls `BLS.aggregatePublicKeys()` without first calling `BLS.verifyPoP()` on every participant key is vulnerable to a rogue-key attack. A malicious participant can compute a rogue BLS public key `pk_rogue = pk_attacker - sum(pk_honest_i)` such that the aggregate equals `pk_attacker` alone. The attacker then holds the sole private key for the aggregate and can produce valid aggregate signatures unilaterally, bypassing all other participants. This enables unauthorized asset transfers or unilateral governance actions in any on-chain multi-sig scheme built on this API.

### Likelihood Explanation

The Cadence `BLS` built-in is a documented, publicly accessible API. Any contract author building a BLS multi-sig scheme is exposed to this risk unless they independently know to call `verifyPoP` before aggregation — a step the aggregation function itself does not require or document as mandatory. The `verifyPoP` function exists but is entirely optional from the API's perspective. [5](#0-4) 

### Recommendation

- Enforce PoP verification inside `AggregatePublicKeys` before passing keys to `crypto.AggregateBLSPublicKeys`. Each input key should be verified with `crypto.BLSVerifyPOP` before inclusion.
- Alternatively, document clearly at the Cadence language level that `BLS.aggregatePublicKeys()` is only safe when all input keys have been PoP-verified, and enforce this at the FVM layer by requiring a PoP-verified key type as input rather than a plain `PublicKey`.
- Remove or resolve the `TODO` comment that incorrectly equates curve-point validity with PoP validity.

### Proof of Concept

A malicious Cadence transaction sender participates in a multi-sig scheme:

```cadence
// Attacker computes pk_rogue = pk_attacker - pk_honest (valid BLS curve point, no PoP needed)
// Then calls:
let aggPk = BLS.aggregatePublicKeys([pk_honest, pk_rogue])!
// aggPk == pk_attacker; attacker signs alone and the verification passes
```

The FVM executes this via `BLSAggregatePublicKeys` → `crypto.AggregatePublicKeys` with no PoP check at any layer. [3](#0-2) [6](#0-5)

// VerifyPOP verifies a proof of possession (PoP) for the receiver public key; currently only works for BLS
func VerifyPOP(pk *runtime.PublicKey, s crypto.Signature) (bool, error) {

	key, err := crypto.DecodePublicKey(crypto.BLSBLS12381, pk.PublicKey)
	if err != nil {
		// at this stage, the runtime public key is valid and there are no possible user value errors
		panic(fmt.Errorf("verify PoP failed: runtime BLS public key should be valid %x", pk.PublicKey))
	}

	valid, err := crypto.BLSVerifyPOP(key, s)
	if err != nil {
		// no user errors possible at this stage
		panic(fmt.Errorf("verify PoP failed with unexpected error %w", err))
	}
	return valid, nil
}

// AggregatePublicKeys aggregate multiple public keys into one; currently only works for BLS
func AggregatePublicKeys(keys []*runtime.PublicKey) (*runtime.PublicKey, error) {
	pks := make([]crypto.PublicKey, 0, len(keys))
	for _, key := range keys {
		// TODO: avoid validating the public keys again since Cadence makes sure runtime keys have been validated.
		// This requires exporting an unsafe function in the crypto package.
		pk, err := crypto.DecodePublicKey(crypto.BLSBLS12381, key.PublicKey)
		if err != nil {
			// at this stage, the runtime public key is valid and there are no possible user value errors
			panic(fmt.Errorf("aggregate BLS public keys failed: runtime public key should be valid %x", key.PublicKey))
		}
		pks = append(pks, pk)
	}

	pk, err := crypto.AggregateBLSPublicKeys(pks)
	if err != nil {
		// check for a user error
		if crypto.IsBLSAggregateEmptyListError(err) || crypto.IsNotBLSKeyError(err) {
			return nil, err
		}
		panic(fmt.Errorf("aggregate BLS public keys failed with unexpected error %w", err))
	}

	return &runtime.PublicKey{
		PublicKey: pk.Encode(),
		SignAlgo:  CryptoToRuntimeSigningAlgorithm(crypto.BLSBLS12381),
	}, nil
}
