### Title
Missing Minimum Threshold Validation in Signing and Presigning Functions Allows Corrupted or Permanently Denied Signing Sessions - (File: src/frost/mod.rs)

### Summary
The `assert_sign_inputs` and `presign` functions in `src/frost/mod.rs`, and the `sign` function in `src/ecdsa/ot_based_ecdsa/sign.rs`, accept a caller-supplied `threshold` with only an upper-bound check (`threshold <= participants.len()`). No lower-bound check is enforced. A threshold of `0` or `1` passes all validation, while the DKG layer (`assert_key_invariants`) mandates `threshold >= 2`. Passing a sub-minimum threshold into a signing session causes Lagrange interpolation to run with the wrong degree, producing an unusable or cryptographically invalid signature and permanently denying that signing session for all honest participants.

### Finding Description

`assert_key_invariants` in `src/dkg.rs` correctly enforces a minimum threshold of 2:

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

However, the public signing-layer validation function `assert_sign_inputs` in `src/frost/mod.rs` only checks the upper bound:

```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [2](#0-1) 

There is no corresponding lower-bound guard. A caller supplying `threshold = 0` or `threshold = 1` passes every check in `assert_sign_inputs`. [3](#0-2) 

The same gap exists in the FROST `presign` function, which only checks the upper bound on `args.threshold`: [4](#0-3) 

And in the OT-based ECDSA `sign` function, which checks `participants.len() < threshold` (satisfied trivially when `threshold = 1` and `participants.len() >= 2`) but never checks `threshold >= 2`: [5](#0-4) 

`ReconstructionLowerBound` is a plain `usize` wrapper with no minimum value enforced at the type level: [6](#0-5) 

### Impact Explanation

FROST and OT-based ECDSA signing both rely on Lagrange interpolation over the participant set using the declared threshold as the polynomial degree. When `threshold = 1` is supplied but the key was generated with `threshold = 2`, the interpolation evaluates a degree-0 polynomial instead of degree-1, producing a signature share that is algebraically inconsistent with the actual secret sharing. The aggregated signature fails standard ECDSA/EdDSA verification. All honest participants in that session receive an unusable output. This matches the **High** allowed impact: *Corruption of sign outputs so honest parties accept inconsistent or unusable cryptographic outputs*, and also *Permanent denial of signing for honest parties under valid protocol inputs*.

### Likelihood Explanation

The threshold is a caller-supplied parameter at every signing invocation. A malicious coordinator, or any participant who controls the session setup, can supply `threshold = 1` (or `0`). No runtime guard prevents this. The gap is directly reachable through the public API without any special privilege beyond being a session initiator.

### Recommendation

Add a lower-bound check in `assert_sign_inputs`, `presign` (frost), and the OT-based ECDSA `sign` function, mirroring the existing guard in `assert_key_invariants`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold: threshold.value(), min: 2 });
}
```

This should be applied at:
- `src/frost/mod.rs` — `assert_sign_inputs` (after line 145)
- `src/frost/mod.rs` — `presign` (after line 72)
- `src/ecdsa/ot_based_ecdsa/sign.rs` — `sign` (alongside the existing participant-count check)

### Proof of Concept

```rust
// Key was generated with threshold = 2, participants = [p0, p1, p2]
// Malicious coordinator initiates signing with threshold = 1:

```

```rust
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
