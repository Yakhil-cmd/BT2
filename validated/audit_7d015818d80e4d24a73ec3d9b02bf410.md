### Title
Unrestricted `update_rewards` with Attacker-Controlled `disable_rewards` Enables Protocol-Wide Reward Griefing ‚Äî (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the `Staking` contract is publicly callable by any address and accepts an attacker-controlled `disable_rewards: bool` parameter. The function unconditionally writes the current block number to the global `last_reward_block` storage slot **before** checking `disable_rewards`. An unprivileged attacker can call this function every block with `disable_rewards: true`, permanently preventing all stakers from ever receiving consensus-era block rewards.

---

### Finding Description

`IStakingRewardsManager::update_rewards` is exposed as a public entry point via `#[abi(embed_v0)]`: [1](#0-0) 

The function accepts a caller-supplied `disable_rewards: bool` flag. Critically, the global `last_reward_block` is written **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

The guard that prevents double-execution in the same block checks this same slot: [3](#0-2) 

Because `last_reward_block` is a **single global slot** (not per-staker), one call per block with any valid `staker_address` and `disable_rewards: true` is sufficient to:

1. Advance `last_reward_block` to the current block.
2. Return early, skipping `_update_rewards` and all reward distribution.
3. Cause every subsequent `update_rewards` call in that block to revert with `REWARDS_ALREADY_UPDATED`.

The interface comment confirms the function is intended to be called by the consensus layer, but no caller restriction is enforced: [4](#0-3) 

---

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` once per block permanently suppresses consensus reward distribution for **all** stakers. No staker can ever receive block rewards because the slot is consumed each block with no actual payout. This constitutes **permanent freezing of unclaimed yield** for every participant in the protocol, matching the allowed impact tier.

---

### Likelihood Explanation

- Requires no funds, no privileged role, and no special knowledge beyond knowing any active staker address (which is public via `get_stakers`).
- The attack is fully automatable: a bot submits one cheap transaction per block.
- There is no economic cost to the attacker and no on-chain mechanism to stop it.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a designated consensus address), mirroring the pattern already used for `update_rewards_from_attestation_contract`: [5](#0-4) 

Alternatively, move the `last_reward_block.write` to **after** the `disable_rewards` guard so that a skipped-reward call does not consume the block's reward slot.

---

### Proof of Concept

```
// Attacker bot, runs once per block:
// 1. Read any active staker address S from get_stakers().
// 2. Call staking_contract.update_rewards(staker_address: S, disable_rewards: true).
//    - last_reward_block is set to current block N.
//    - No rewards are distributed.
// 3. Any legitimate call to update_rewards at block N now reverts:
//    assert!(current_block_number > self.last_reward_block.read(), ...)
//    => current_block_number (N) > last_reward_block (N) => FALSE => REVERT.
// 4. Repeat every block. All stakers receive zero consensus rewards indefinitely.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
