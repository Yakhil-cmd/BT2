### Title
Cairo0 Contract Bypasses `replace_class` Cairo1-Only Guard via Deprecated Hint Processor Path — (`crates/blockifier/src/execution/deprecated_syscalls/hint_processor.rs`)

### Summary
The deprecated hint processor's `replace_class` implementation omits the `is_cairo1` check that is present in `syscall_base::replace_class`. A Cairo0 contract can invoke `replace_class` via the deprecated syscall path and successfully replace its own class hash with another Cairo0 class hash, violating the protocol invariant that `replace_class` only accepts Cairo1 class hashes and corrupting the on-chain class hash state.

### Finding Description

`syscall_base::replace_class` enforces a Cairo1-only guard:

```rust
// syscall_base.rs:369-378
pub fn replace_class(&mut self, class_hash: ClassHash) -> SyscallResult<()> {
    let compiled_class = self.state.get_compiled_class(class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(SyscallExecutionError::ForbiddenClassReplacement { class_hash });
    }
    self.state.set_class_hash_at(self.call.storage_address, class_hash)?;
    Ok(())
}
``` [1](#0-0) 

The deprecated hint processor's implementation of the same syscall is:

```rust
// hint_processor.rs:795-807
fn replace_class(
    request: ReplaceClassRequest,
    _vm: &mut VirtualMachine,
    syscall_handler: &mut Self,
) -> DeprecatedSyscallResult<ReplaceClassResponse> {
    // Ensure the class is declared (by reading it).
    syscall_handler.state.get_compiled_class(request.class_hash)?;
    syscall_handler
        .state
        .set_class_hash_at(syscall_handler.storage_address, request.class_hash)?;
    Ok(ReplaceClassResponse {})
}
``` [2](#0-1) 

There is no `is_cairo1` check. The function only verifies the class is declared, then unconditionally writes the new class hash.

`DeprecatedSyscallSelector::ReplaceClass` is a fully supported selector in the deprecated executor dispatch table: [3](#0-2) 

It is also present in the `DeprecatedSyscallSelector` enum: [4](#0-3) 

And the selector string `"ReplaceClass"` is recognized and dispatched: [5](#0-4) 

### Impact Explanation

A Cairo0 contract (deployed by any unprivileged user) can call `replace_class` with a Cairo0 target class hash. The deprecated hint processor path will succeed, writing the Cairo0 class hash into the contract's class hash slot in state. This:

- Violates the invariant that `replace_class` only accepts Cairo1 class hashes.
- Corrupts the class hash storage value for the calling contract.
- Causes subsequent executions of that contract to use the wrong (Cairo0) class, producing wrong execution results — a **Critical** wrong-state-from-syscall-bug impact.

### Likelihood Explanation

Any user can deploy a Cairo0 contract and craft a transaction that invokes `replace_class` with a Cairo0 target class hash via the deprecated syscall path. No privileged access is required. The deprecated path is fully reachable in production execution.

### Recommendation

Add the `is_cairo1` check to the deprecated hint processor's `replace_class` implementation, mirroring `syscall_base::replace_class`:

```rust
fn replace_class(...) -> DeprecatedSyscallResult<ReplaceClassResponse> {
    let compiled_class = syscall_handler.state.get_compiled_class(request.class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(/* ForbiddenClassReplacement equivalent */);
    }
    syscall_handler.state.set_class_hash_at(syscall_handler.storage_address, request.class_hash)?;
    Ok(ReplaceClassResponse {})
}
```

### Proof of Concept

1. Declare a Cairo0 contract class `C0_target` on the network.
2. Deploy a Cairo0 contract `C0_caller` whose `__execute__` entry point issues a `replace_class` syscall with `class_hash = C0_target`.
3. Submit an `InvokeV1` transaction targeting `C0_caller`.
4. The deprecated hint processor dispatches `DeprecatedSyscallSelector::ReplaceClass` → `hint_processor::replace_class`, which calls `get_compiled_class(C0_target)` (succeeds, class is declared), then `set_class_hash_at(C0_caller_address, C0_target)` (succeeds, no Cairo1 check).
5. Assert the transaction succeeds (no `ForbiddenClassReplacement` error) and that the class hash of `C0_caller` in state is now `C0_target` (a Cairo0 class hash). [2](#0-1) [6](#0-5)
