### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards for All Stakers via `update_rewards` - (File: src/staking/staking.cairo)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. Because the function writes to the global `last_reward_block` storage slot **before** checking `disable_rewards`, any unprivileged caller can call it every block with `disable_rewards: true` to permanently prevent consensus reward distribution for every staker in the protocol.

---

### Finding Description

`update_rewards` is exposed via `#[abi(embed_v0)]` with no caller restriction beyond `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The critical sequence inside the function is:

1. Validates the staker exists and is active.
2. **Writes `current_block_number` to `last_reward_block`** (line 1485).
3. Checks `if disable_rewards || self.is_pre_consensus() { return; }` (line 1487). [2](#0-1) 

Because step 2 happens unconditionally before step 3, calling `update_rewards(any_valid_staker, disable_rewards: true)` in block N:

- Advances `last_reward_block` to N.
- Returns without distributing any rewards.
- Blocks every subsequent call to `update_rewards` in block N with `REWARDS_ALREADY_UPDATED`, because the guard at line 1455 requires `current_block_number > last_reward_block`. [3](#0-2) 

`general_prerequisites()` provides no protection — it only asserts the contract is unpaused and the caller address is non-zero. [4](#0-3) 

---

### Impact Explanation

An attacker calling `update_rewards(any_valid_staker, true)` once per block permanently prevents consensus reward accrual for **all** stakers. No staker can receive block rewards as long as the attacker continues. This constitutes **permanent freezing of unclaimed yield** for the entire protocol during the consensus rewards phase.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield or unclaimed royalties**.

---

### Likelihood Explanation

The attack requires only:
- A valid (migrated, active) staker address to pass the staker-existence checks.
- One transaction per block.
- No capital, no privileged role, no leaked key.

Staker addresses are publicly observable on-chain from `NewStaker` events. The cost is purely gas per block on Starknet L2, which is negligible. Any motivated griever (e.g., a competing validator, a protocol adversary) can sustain this indefinitely.

---

### Recommendation

Restrict `update_rewards` to a trusted caller. The most natural choice is the consensus layer's designated address (e.g., the attestation contract or a dedicated consensus-rewards caller role). Add a check analogous to the existing pattern used in `update_rewards_from_attestation_contract`: [5](#0-4) 

Concretely, either:
1. Add `self.assert_caller_is_attestation_contract()` (or a new `assert_caller_is_consensus_contract()`) at the top of `update_rewards`, or
2. Remove the `disable_rewards` parameter from the public interface and expose it only to a privileged internal path.

---

### Proof of Concept

```
// Attacker script — run once per block:
// 1. Pick any known active staker address S.
// 2. Call: staking_contract.update_rewards(staker_address: S, disable_rewards: true)
//    - Passes general_prerequisites() (contract unpaused, caller != 0).
//    - Passes staker-existence and staker-active checks.
//    - Writes last_reward_block = current_block_number.
//    - Hits `if disable_rewards { return; }` — no rewards distributed.
// 3. Any legitimate call to update_rewards in the same block now reverts with
//    REWARDS_ALREADY_UPDATED because current_block_number == last_reward_block.
// 4. Repeat every block → zero consensus rewards ever distributed.
```

Root cause: `last_reward_block` is updated at line 1485 before the `disable_rewards` guard at line 1487, and there is no access control on who may supply `disable_rewards: true`. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
