### Title
Missing Caller Access Control on `update_rewards` Allows Any Address to Permanently Suppress Block Rewards â€” (`src/staking/staking.cairo`)

### Summary
`update_rewards` is documented in the spec as restricted to "Only starkware sequencer," but the implementation contains no such check. Any unprivileged caller can invoke it once per block with `disable_rewards: true`, which atomically marks the block as processed and skips reward distribution â€” permanently discarding that block's rewards for all stakers.

### Finding Description

The spec explicitly states the access control for `update_rewards`:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the actual implementation contains no caller check. The only guards are a pause check (`general_prerequisites()`), a block-number gate, and staker validity checks: [2](#0-1) 

The word "sequencer" does not appear anywhere in `src/staking/` â€” only in `docs/spec.md`. The interface even contradicts the spec, annotating the function as callable by any address: [3](#0-2) 

The `last_reward_block` storage variable is **global** (not per-staker). When any caller calls `update_rewards(any_valid_staker, disable_rewards: true)` for block N:

1. Line 1485 writes `current_block_number` into `last_reward_block`.
2. Line 1487 returns early â€” no rewards are distributed.
3. Any subsequent call in the same block for **any** staker fails with `REWARDS_ALREADY_UPDATED` (line 1454â€“1458). [4](#0-3) 

The skipped block's rewards are not deferred or accumulated â€” they are permanently lost.

### Impact Explanation
An attacker calling `update_rewards(valid_staker, disable_rewards: true)` once per block suppresses all consensus block rewards for all stakers indefinitely. This constitutes **permanent freezing of unclaimed yield** â€” a High-severity impact under the allowed scope.

### Likelihood Explanation
The call is permissionless, costs only gas, and requires no capital or privileged access. The attacker only needs to know one valid active staker address (publicly observable on-chain). The attack is sustainable every block.

### Recommendation
Add a sequencer-only caller check at the top of `update_rewards`, consistent with the spec. On Starknet, the sequencer address is available via `starknet::get_execution_info().caller_address` or a stored sequencer address role. The check should assert `get_caller_address() == sequencer_address` before any other logic.

### Proof of Concept

1. Deploy the staking contract with consensus rewards active.
2. Stake as `staker_A` and wait K epochs for balance to become effective.
3. From an arbitrary EOA (attacker), call `update_rewards(staker_A, disable_rewards: true)` at the start of every block.
4. Observe that `last_reward_block` is set to the current block each time.
5. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. After N blocks, `staker_A.unclaimed_rewards_own` remains zero despite N rewardable blocks having elapsed.

This is directly confirmed by the existing test at lines 2882â€“2885, which shows `disable_rewards: true` with consensus active yields zero rewards and blocks further calls in the same block: [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
