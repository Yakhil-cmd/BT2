### Title
Unguarded `?` on Unknown Sender in Echo-Broadcast Receive Loop Enables Permanent DKG/Reshare/Refresh Denial — (`File: src/protocol/echo_broadcast.rs`)

---

### Summary

Inside `reliable_broadcast_receive_all`, the `MessageType::Send` branch calls `participants.index(from)?`, propagating a fatal error if the sender is not in the participant list. The `Echo` and `Ready` branches handle the same situation safely by using `ParticipantCounter::put`, which silently returns `false` for unknown senders. This inconsistency means any entity that can deliver a `MessageType::Send` message from a `Participant` identity outside the current protocol's list will abort the entire broadcast round — and with it, every DKG, reshare, or refresh session that depends on it.

---

### Finding Description

`reliable_broadcast_receive_all` is the core receive loop of the echo-broadcast primitive used throughout the library. When a message arrives, the loop first bounds-checks the session index `sid` with a safe `let Some(...) = state.get_mut(sid) else { continue }` guard. It then dispatches on the message type.

For `MessageType::Echo` and `MessageType::Ready`, unknown senders are handled safely:

```rust
// Echo branch — safe: returns false for unknown participant, loop continues
if !state_sid.seen_echo.put(from) || state_sid.finish_echo {
    continue;
}
```

`ParticipantCounter::put` returns `false` when `from` is absent from the list, so the loop simply skips the message.

For `MessageType::Send`, the same situation is handled with a fatal `?`:

```rust
// Send branch — UNSAFE: propagates Err if `from` is not in the list
if state_sid.finish_send || sid != participants.index(from)? {
    continue;
}
```

`participants.index(from)` returns `Err(ProtocolError::InvalidIndex)` for any `Participant` not in the list. The `?` operator immediately returns that error from `reliable_broadcast_receive_all`, unwinding the entire async future.

`do_broadcast` wraps this function and is called unconditionally in `do_keyshare` for:
- the session-ID exchange round (line 362),
- the commitment-and-proof broadcast round (line 435), and
- the final success-vote round via `broadcast_success` (line 531).

A single abort in any of these rounds terminates the DKG, reshare, or refresh for every honest participant in the session.

---

### Impact Explanation

**High — Permanent denial of key generation, reshare, and refresh for honest parties.**

When `reliable_broadcast_receive_all` returns an error, `do_broadcast` propagates it, `do_keyshare` propagates it, and the protocol future resolves to `Err`. There is no retry or recovery path inside the protocol itself. All honest participants must restart the entire multi-round protocol from scratch. If the attacker can repeat the injection on every attempt, the protocol is permanently blocked.

---

### Likelihood Explanation

The `recv_from_others` helper — used in every other receive site in the library — demonstrates that the underlying channel **does** deliver messages from participants outside the current list; it handles them by checking `seen.put(from)` and silently discarding them. This confirms the channel is not filtered to the protocol's participant set.

A malicious participant from a concurrent or previous protocol instance, or any participant that was removed from the new participant set during a reshare, can send a `MessageType::Send` message on the same waitpoint. Because `sid` is a plain `usize` serialized over the wire, the attacker can choose any value in `[0, n)` to pass the `state.get_mut(sid)` guard, then rely on their own `Participant` identity being absent from the victim session's list to trigger the error.

---

### Recommendation

Replace the fatal `?` with the same pattern used by the `Echo` and `Ready` branches:

```rust
// Before (aborts on unknown sender):
if state_sid.finish_send || sid != participants.index(from)? {
    continue;
}

// After (ignores unknown sender, consistent with Echo/Ready):
let Ok(from_idx) = participants.index(from) else { continue };
if state_sid.finish_send || sid != from_idx {
    continue;
}
```

This makes the `Send` branch consistent with the rest of the loop and eliminates the abort path for out-of-list senders.

---

### Proof of Concept

1. An honest set `{P1, P2, P3}` initiates DKG via `do_keyshare`. `do_broadcast` is called; `reliable_broadcast_receive_all` enters its loop.
2. A fourth party `P4` (not in the participant list) sends a message `(sid=0, MessageType::Send(data))` to the same waitpoint.
3. The loop receives the message. `state.get_mut(0)` succeeds (valid index). The match arm enters `MessageType::Send`.
4. `participants.index(P4)` returns `Err(ProtocolError::InvalidIndex)`.
5. The `?` propagates the error; `reliable_broadcast_receive_all` returns `Err`.
6. `do_broadcast` → `do_keyshare` → the protocol future resolves to `Err` for all honest parties.
7. No threshold signature key is ever produced.

    chan: &mut SharedChannel,
    participants: &'a ParticipantList,
    me: Participant,
    data: T,
) -> Result<ParticipantMap<'a, T>, ProtocolError>

    T: Serialize + Clone + DeserializeOwned + PartialEq,

    let wait_broadcast = chan.next_waitpoint();
    let send_vote = reliable_broadcast_send(chan, wait_broadcast, participants, me, data)?;
    let vote_list =

    Ok(vote_list)
}

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}
