Based on my analysis of the code, the attack path is concrete and fully traceable.

**Attack path:**

1. `Comms::push_message` (lines 286–296) accepts any message where `len >= 40` and the first 40 bytes parse as a valid `MessageHeader`. There is **no validation** that the sender is in the participant list, and no validation of the payload bytes. [1](#0-0) 

2. `Comms::recv` (lines 330–341) pops the **first** message from the queue for the given header, slices off the 40-byte header, and calls `rmp_serde::decode::from_slice` on the remainder. On failure it **immediately returns** `ProtocolError::DeserializationError` — there is no retry, no skip-bad-message loop. [2](#0-1) 

3. `recv_from_others` propagates the error immediately via `?` with no recovery. [3](#0-2) 

**The root cause** is the combination of: (a) no sender/payload validation in `push_message`, and (b) no retry-on-bad-deserialization in `recv`. A single injected message permanently terminates the honest participant's protocol.

### Title
Single malicious message with valid header and garbage msgpack body permanently terminates honest participant's protocol — (`src/protocol/internal.rs`)

### Summary
A malicious participant can inject one crafted message — 40 bytes of a valid `MessageHeader` (correct `root_shared` channel tag + target waitpoint) followed by arbitrary garbage — into a running protocol. `Comms::push_message` accepts and routes it to the correct `SubMessageQueue`. When the honest participant's `Comms::recv` dequeues it, `rmp_serde::decode::from_slice` fails and `ProtocolError::DeserializationError` is returned immediately with no retry, permanently aborting the protocol.

### Finding Description
`Comms::push_message` performs only two checks: message length ≥ 40 and that the first 40 bytes parse as a `MessageHeader`. [1](#0-0) 

It does not verify the sender is a registered participant, nor does it validate the payload. The message is unconditionally enqueued into the `SubMessageQueue` keyed by the parsed header.

`Comms::recv` dequeues the first available message for the requested header and immediately attempts deserialization:

```rust
let decoded: Result<T, _> = rmp_serde::decode::from_slice(message_data)
    .map_err(std::convert::Into::into);
Ok((from, decoded?))   // propagates error, no retry
``` [4](#0-3) 

There is no loop to skip messages that fail deserialization and wait for the next one. The error propagates through `SharedChannel::recv` → `recv_from_others` (via `?`) → the protocol future → `ProtocolExecutor::poke` → the caller. [5](#0-4) 

### Impact Explanation
Any malicious participant (or any caller who can invoke `Protocol::message`) can permanently abort a DKG, reshare, refresh, presign, or signing session for an honest participant by injecting a single message. The honest participant's protocol returns `ProtocolError::DeserializationError` and cannot be resumed. This matches **High: Permanent denial of signing, key generation, reshare, refresh, or CKD for honest parties under valid protocol inputs and documented trust assumptions**.

### Likelihood Explanation
The attacker only needs to know the `root_shared` channel tag (it is deterministic and publicly derivable — `SHA256(NEAR_CHANNEL_TAGS_DOMAIN || "root shared")`) and the target waitpoint (0 for the first round). Both are fixed constants. No secret material is required. Any participant in the session can execute this attack with a single crafted message.

### Recommendation
In `Comms::recv`, replace the single-shot dequeue-and-fail pattern with a retry loop that skips messages that fail deserialization (logging or counting them), rather than propagating the error immediately. Additionally, `push_message` should validate that the `from` participant is a member of the expected participant set before enqueuing.

### Proof of Concept
```rust
// Craft: valid root_shared header at waitpoint 0, garbage body
let channel_tag = ChannelTag::root_shared();
let header = MessageHeader { channel: channel_tag, waitpoint: 0 };
let mut msg = header.to_bytes().to_vec();
msg.extend_from_slice(b"\xff\xfe\xfd\xfc"); // invalid msgpack

// Inject before the honest participant processes round 1
protocol.message(malicious_participant, msg);

// Drive the protocol — it returns DeserializationError instead of completing
let result = run_protocol(protocol);
assert!(matches!(result, Err(ProtocolError::DeserializationError(_))));
```
