### Title
Unprivileged Caller Can Permanently Deny Block Rewards via `update_rewards(disable_rewards: true)` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract has no access control. Any unprivileged caller can invoke it with `disable_rewards: true` for any staker. Because `last_reward_block` is written to storage **before** the `disable_rewards` guard is evaluated, the block's reward slot is permanently consumed without distributing any rewards. Since `last_reward_block` is a single global variable, this blocks all stakers from receiving rewards for that block.

---

### Finding Description

`IStakingRewardsManager::update_rewards` is the V3 consensus-rewards entry point. Its implementation in `src/staking/staking.cairo` (lines 1449–1507) follows this sequence:

1. Calls `general_prerequisites()` — only checks the contract is unpaused and the caller is non-zero. No role or identity check.
2. Asserts `current_block_number > last_reward_block` (line 1455–1458).
3. Validates the staker exists and is active (lines 1466–1482).
4. **Writes `last_reward_block = current_block_number`** (line 1485).
5. Then checks `if disable_rewards || self.is_pre_consensus() { return; }` (line 1487–1489).

The write at step 4 occurs unconditionally before the `disable_rewards` branch. Once written, the assertion at step 2 will revert for any subsequent caller in the same block.

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← consumed here

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← no rewards distributed
}
``` [1](#0-0) 

There is no check that the caller is the staker, the attestation contract, or any privileged role. The `general_prerequisites` guard only enforces non-zero caller and unpaused state. [2](#0-1) 

`last_reward_block` is a single global storage slot shared across all stakers: [3](#0-2) 

---

### Impact Explanation

**High — Permanent freezing / theft of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` at the start of every block:

- Permanently consumes that block's reward slot (`last_reward_block` is set).
- Distributes zero rewards to any staker.
- Prevents the legitimate consensus mechanism from calling `update_rewards` for that block (it reverts with `REWARDS_ALREADY_UPDATED`).

Because block rewards are never retroactively recalculated, the lost rewards are gone permanently. Sustained over many blocks, this constitutes a complete denial of consensus block rewards for the entire protocol — matching the **High: Permanent freezing of unclaimed yield** impact category.

---

### Likelihood Explanation

**High.** The attack requires:
- No capital.
- No privileged role.
- Only gas to submit one transaction per block.

Any address that is non-zero and the contract is unpaused can execute this. The attacker can front-run the legitimate `update_rewards` call in every block.

---

### Recommendation

Add an access-control check to `update_rewards` so that only a trusted caller (e.g., the staker themselves, the attestation contract, or a designated consensus contract) can invoke it, particularly with `disable_rewards: true`. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Add: only the staker or a trusted consensus contract may call this.
    let caller = get_caller_address();
    assert!(
        caller == staker_address || caller == self.attestation_contract.read(),
        "{}",
        Error::UNAUTHORIZED_CALLER,
    );
    ...
}
```

Alternatively, move the `last_reward_block.write` to after the `disable_rewards` check, so that a call with `disable_rewards: true` does not consume the block slot.

---

### Proof of Concept

1. Attacker monitors the chain. At block N, they call:
   ```
   staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
   ```
2. Inside `update_rewards`:
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - Staker is validated as active with non-zero balance.
   - `last_reward_block.write(N)` executes.
   - `disable_rewards == true` → function returns with no rewards distributed.
3. The legitimate consensus mechanism attempts `update_rewards(staker, false)` for block N → reverts with `REWARDS_ALREADY_UPDATED`.
4. Staker receives zero rewards for block N. The opportunity is permanently lost.
5. Repeating this every block denies all stakers of all consensus block rewards indefinitely. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
