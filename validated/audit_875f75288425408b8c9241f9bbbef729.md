### Title
Unbounded Loop in `calculate_rewards` Enables Permanent Freezing of Pool Member Unclaimed Yield - (File: `src/pool/pool.cairo`)

### Summary
The `calculate_rewards` internal function in the `Pool` contract contains an explicitly acknowledged unbounded loop that iterates over every entry in a pool member's `pool_member_epoch_balance` trace. A pool member who accumulates a sufficiently large trace — by repeatedly entering and exiting the delegation pool — will cause every future `claim_rewards` call to revert with out-of-gas, permanently freezing their unclaimed yield.

### Finding Description
`calculate_rewards` in `src/pool/pool.cairo` iterates over the entire `pool_member_epoch_balance` trace for a given pool member: [1](#0-0) 

The code comment at line 857–858 explicitly acknowledges the risk:

> **Note**: The loop iterates over the balance changes in the pool member's balance trace. This loop is unbounded but unlikely to exceed gas limits.

Each time a pool member changes their delegated balance — via `enter_delegation_pool`, `add_to_delegation_pool`, `exit_delegation_pool_intent` / `exit_delegation_pool_action` — a new checkpoint is appended to `pool_member_epoch_balance`. The trace is never pruned or bounded. [2](#0-1) 

Inside each loop iteration, `find_sigma` is also called, which performs its own lookup into the global `cumulative_rewards_trace`, compounding per-iteration cost: [3](#0-2) 

`calculate_rewards` is called from `claim_rewards` (the public pool member entry point). Once the trace is large enough, every `claim_rewards` invocation will exceed the Starknet transaction gas limit and revert, with no recovery path since the trace cannot be pruned. [4](#0-3) 

### Impact Explanation
Once the `pool_member_epoch_balance` trace for a given address exceeds the gas threshold, `claim_rewards` permanently reverts for that address. All accrued rewards become permanently inaccessible — **permanent freezing of unclaimed yield** — matching the High impact category in the allowed scope.

### Likelihood Explanation
Any pool member is an unprivileged actor who can call `enter_delegation_pool` and `exit_delegation_pool_intent` / `exit_delegation_pool_action` in a loop. Each round-trip appends at least one new entry to the trace. On Starknet, transaction fees are low enough that an attacker (or an automated DeFi integration making frequent rebalances) can accumulate thousands of entries at modest cost. The protocol's own comment dismisses the risk as "unlikely," but provides no on-chain bound or guard.

### Recommendation
1. **Checkpoint-based claiming**: Store a `last_claimed_trace_idx` per pool member and only iterate from that index forward, so each `claim_rewards` call processes only the entries since the last claim rather than the full history. The current code already threads `entry_to_claim_from` through `calculate_rewards` but does not persist it between calls in a way that prevents re-scanning old entries.
2. **Bound balance-change frequency**: Enforce a minimum epoch gap between successive delegation changes per pool member to limit trace growth rate.
3. **Remove the dismissive comment** and treat the unbounded loop as a tracked risk with a concrete mitigation plan.

### Proof of Concept
1. Deploy the protocol locally with Starknet Foundry.
2. Register a staker and open a delegation pool.
3. As an unprivileged pool member, call `enter_delegation_pool` with the minimum amount, then `exit_delegation_pool_intent`, then `exit_delegation_pool_action` in a loop ~10,000 times. Each cycle appends entries to `pool_member_epoch_balance`.
4. Call `claim_rewards` for that pool member — the transaction reverts with out-of-gas because `calculate_rewards` must iterate all accumulated trace entries.
5. The pool member's rewards are permanently frozen; no alternative claim path exists. [5](#0-4) [6](#0-5)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
