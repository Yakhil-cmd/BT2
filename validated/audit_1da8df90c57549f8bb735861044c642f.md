Audit Report

## Title
SNS Swap `finalize_inner` ABORTED path: `restore_dapp_controllers` permanently blocked when `sweep_icp` has unresolvable failures - (File: `rs/sns/swap/src/swap.rs`)

## Summary
In `finalize_inner`, `sweep_icp` is unconditionally executed before `should_restore_dapp_control()` is checked, and any `failure` or `invalid` entry in the sweep result causes an early return via `has_error_message()`. This means that if a buyer's ICP amount is permanently below the transfer fee, or if the ICP ledger is persistently unavailable, `restore_dapp_controllers_for_finalize` is never reached in the ABORTED lifecycle path, leaving dapp canisters indefinitely under SNS Root control with no recovery path through the normal finalization flow.

## Finding Description
`finalize_inner` (`rs/sns/swap/src/swap.rs`, lines 1544–1584) executes steps sequentially with an early-return guard after each step:

1. `sweep_icp` is called (line 1557–1558).
2. `set_sweep_icp_result` (`rs/sns/swap/src/types.rs`, lines 895–901) calls `set_error_message` if `!sweep_icp_result.is_successful_sweep()` — i.e., if any `failure` or `invalid` count is nonzero.
3. The guard at line 1559–1561 checks `has_error_message()` and returns early if true.
4. `should_restore_dapp_control()` at line 1572 is never reached.
5. `restore_dapp_controllers_for_finalize` at line 1576 is never called.

The test `test_finalization_halts_when_sweep_icp_fails` (`rs/sns/swap/tests/swap.rs`, lines 2585–2654) explicitly documents and asserts this behavior: when a buyer has `amount_e8s < DEFAULT_TRANSFER_FEE`, `sweep_icp` returns `invalid: 1`, `set_dapp_controllers_call_result` is `None`, and the error message is set. The test comment acknowledges this buyer state "should never be possible in production, but sweep_icp must handle this case" — confirming the code path exists and is exercised.

`finalize_swap` is a public `#[update]` endpoint with no access control (`rs/sns/swap/canister/canister.rs`, lines 149–159), callable by any principal. The `finalize_swap_in_progress` lock is released after each failed call (line 1531), so the blocked state persists across all retries whenever the root cause is permanent.

## Impact Explanation
Dapp canisters remain under SNS Root control after a swap abortion with no path to restore them to the original developers via the normal finalization flow. This constitutes a significant SNS security impact with concrete harm: dapp developers permanently lose control of their canisters. This matches the allowed High impact category: "Significant SNS infrastructure security impact with concrete user or protocol harm."

## Likelihood Explanation
The most realistic permanent trigger is a buyer state with `amount_e8s < DEFAULT_TRANSFER_FEE`. While the code comments say this "should never be possible in production," the code explicitly handles it and the test confirms it permanently blocks finalization. Temporary ICP ledger unavailability produces `failure` entries and is self-resolving via retry, but a permanent invalid buyer state is not. No privileged access is required; any principal can call `finalize_swap` to observe and confirm the blocked state.

## Recommendation
Decouple `restore_dapp_controllers` from `sweep_icp` success in the ABORTED path. Specifically, `should_restore_dapp_control()` should be evaluated and acted upon before the `sweep_icp` error propagation check, or the early-return guard after `sweep_icp` should be skipped when the lifecycle is ABORTED. The ICP refund sweep and the dapp controller restoration are independent concerns; the latter must not be gated on the former.

## Proof of Concept
1. Arrange a swap in `LIFECYCLE_ABORTED` state with one buyer whose `amount_e8s < DEFAULT_TRANSFER_FEE`.
2. Call `finalize_swap` as any unprivileged principal.
3. `sweep_icp` returns `SweepResult { invalid: 1, ... }`.
4. `set_sweep_icp_result` sets `error_message` (confirmed by `rs/sns/swap/src/types.rs` lines 895–901).
5. `has_error_message()` returns `true`; `finalize_inner` returns at line 1560.
6. `should_restore_dapp_control()` (line 1572) and `restore_dapp_controllers_for_finalize` (line 1576) are never reached.
7. `set_dapp_controllers_call_result` remains `None` — confirmed by the existing test assertion at `rs/sns/swap/tests/swap.rs` line ~2648.
8. Dapp canisters remain under SNS Root control indefinitely; retrying `finalize_swap` produces the same result.
