### Title
`VersionedStateProxy::get_compiled_class_hash_v2` Bypasses Versioned Storage, Returning Stale/Missing Hash for Intra-Chunk Declared Classes — (`crates/blockifier/src/concurrency/versioned_state.rs`)

### Summary

`VersionedStateProxy::get_compiled_class_hash_v2` unconditionally delegates to `initial_state` without consulting the versioned `compiled_class_hashes` storage. Every other `get_*` method on `VersionedStateProxy` follows the correct pattern: check versioned storage first, fall back to `initial_state` only on a miss. This method is the sole exception. As a result, when tx_B (index j > i) calls `get_compiled_class_hash_v2` for a class first declared by tx_A (index i) within the same concurrent chunk, it reads from the pre-chunk state, which has no knowledge of tx_A's declaration, and returns either a `MissingCompiledClassHashV2` error or a stale zero hash instead of the value written by tx_A.

### Finding Description

**The correct pattern — every other `get_*` method:**

```rust

fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    let mut state_opt = self.state();
    let state = state_opt.inner_mut()?;
    match state.compiled_class_hashes.read(self.tx_index, class_hash) {
        Some(value) => Ok(value),
        None => {
            let initial_value = state.initial_state.get_compiled_class_hash(class_hash)?;
            state.compiled_class_hashes.set_initial_value(class_hash, initial_value);
            Ok(initial_value)
        }
    }
}
``` [2](#0-1) 

`get_storage_at`, `get_nonce_at`, `get_class_hash_at`, `get_compiled_class_hash`, and `get_compiled_class` all follow the versioned-read-then-fallback pattern. `get_compiled_class_hash_v2` is the only method that skips it entirely.

**Why the `compiled_class` parameter does not rescue the call:**

The `compiled_class` argument passed to `initial_state.get_compiled_class_hash_v2` is ultimately ignored. The default utility implementation re-fetches the class from the state reader:

```rust
// utils.rs lines 12-25
pub fn get_compiled_class_hash_v2(
    state_reader: &impl StateReader,
    class_hash: ClassHash,
    _compiled_class: &RunnableCompiledClass,   // <-- ignored
) -> StateResult<CompiledClassHash> {
    match state_reader.get_compiled_class(class_hash) {
        Ok(RunnableCompiledClass::V0(_)) | Err(StateError::UndeclaredClassHash(_)) =>
            Err(StateError::MissingCompiledClassHashV2(class_hash)),
        Ok(RunnableCompiledClass::V1(class)) => Ok(class.hash(&HashVersion::V2)),
        ...
    }
}
``` [3](#0-2) 

`state_reader` here is `initial_state`, not `VersionedStateProxy`. So `get_compiled_class` is called on the pre-chunk state, which returns `UndeclaredClassHash` for any class first declared within the chunk, causing `MissingCompiledClassHashV2` to propagate back to tx_B.

**The `StateReaderAndContractManager` cache does not reliably mitigate this:**

`StateReaderAndContractManager::get_compiled_class_hash_v2` checks a class manager cache first:

```rust
// state_reader_and_contract_manager.rs lines 162-189
fn get_compiled_class_hash_v2(...) -> StateResult<CompiledClassHash> {
    match self.contract_class_manager.get_compiled_class_hash_v2(&class_hash) {
        Some(compiled_class_hash) => Ok(compiled_class_hash),
        None => {
            let compiled_class_hash =
                self.state_reader.get_compiled_class_hash_v2(class_hash, compiled_class)?;
            self.contract_class_manager.set_compiled_class_hash_v2(class_hash, compiled_class_hash);
            Ok(compiled_class_hash)
        }
    }
}
``` [4](#0-3) 

`set_compiled_class_hash_v2` is only called inside this function, after a successful fetch from `state_reader`. For a class declared for the first time by tx_A within the chunk, `state_reader.get_compiled_class_hash_v2` will fail (class not in pre-chunk state), so `set_compiled_class_hash_v2` is never called, and the cache remains empty. The cache provides no mitigation for newly declared classes.

**`apply_writes` correctly writes to versioned storage — the data is there, just never read:**

```rust

    self.compiled_class_hashes.write(tx_index, key, value);

``` [5](#0-4) 

tx_A's `compiled_class_hash` is correctly written to `self.compiled_class_hashes` at index i. `get_compiled_class_hash` (the v1 method) would find it. `get_compiled_class_hash_v2` never looks there.

### Impact Explanation

When `traceTransaction` (or `simulateTransaction`) is called for tx_B, the execution engine calls `get_compiled_class_hash_v2` to populate the `compiled_class_hash` field in the trace for a class that was declared by tx_A earlier in the same concurrent chunk. The call bypasses versioned storage, hits `initial_state`, and returns either a `MissingCompiledClassHashV2` error (for a freshly declared class) or `CompiledClassHash(Felt::ZERO)` (the sentinel for undeclared classes set at line 420). The trace for tx_B carries a wrong or zero `compiled_class_hash_v2`, which is an authoritative-looking incorrect value returned by an RPC tracing endpoint. [6](#0-5) 

### Likelihood Explanation

The scenario requires two transactions in the same concurrent chunk where tx_A declares a Cairo 1 class and tx_B (or any later transaction in the chunk) triggers a code path that calls `get_compiled_class_hash_v2` for that class. Declaring a class is an ordinary, unprivileged Starknet operation. The concurrent execution path is the production sequencer path for block building. No special privileges are required.

### Recommendation

Apply the same versioned-read-then-fallback pattern used by every other `get_*` method:

```rust
fn get_compiled_class_hash_v2(
    &self,
    class_hash: ClassHash,
    compiled_class: &RunnableCompiledClass,
) -> StateResult<CompiledClassHash> {
    let mut state_opt = self.state();
    let state = state_opt.inner_mut()?;
    // Check versioned storage first (mirrors get_compiled_class_hash)
    match state.compiled_class_hashes.read(self.tx_index, class_hash) {
        Some(value) if value != CompiledClassHash::default() => Ok(value),
        _ => state.initial_state.get_compiled_class_hash_v2(class_hash, compiled_class),
    }
}
```

Alternatively, since `compiled_class` is already in scope at the call site, compute the hash directly from it (using `class.hash(&HashVersion::V2)`) rather than re-fetching from state, which would make the function independent of state entirely.
