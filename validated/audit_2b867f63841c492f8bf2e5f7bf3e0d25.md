### Title
`MsgTurnBridge` Handler is a No-Op Stub with No Authorization Check — (`x/cronos/keeper/msg_server.go`)

---

### Summary

The `TurnBridge` handler in `msgServer` unconditionally returns `(nil, nil)` with no permission check and no state mutation. Any unprivileged address can submit `MsgTurnBridge`, receive a success response, and the bridge state is never changed. The `CanTurnBridge` permission constant exists and is used elsewhere, but is never applied in this handler.

---

### Finding Description

`MsgTurnBridge` is fully registered as a routable `sdk.Msg`: [1](#0-0) 

The handler implementation is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [2](#0-1) 

Compare this to `UpdateTokenMapping`, which correctly gates on `HasPermission`: [3](#0-2) 

The `CanTurnBridge` permission bit is defined and the `HasPermission` helper is fully functional: [4](#0-3) [5](#0-4) 

ADR-009 explicitly mandates that `MsgTurnBridge` must be restricted to permissioned addresses, and that the bridge-disable path is a critical circuit breaker: [6](#0-5) 

---

### Impact Explanation

Two distinct impacts, both High:

1. **Authorization bypass**: Any unprivileged address submits `MsgTurnBridge` with any `sender` field. The handler returns `nil` error — the transaction succeeds on-chain. The `CanTurnBridge` permission guard is completely absent, violating the invariant that only admin/permissioned addresses may toggle the bridge.

2. **Bridge circuit breaker permanently inoperable**: Because the handler performs no state mutation, even a legitimately permissioned address calling `TurnBridge` cannot actually disable the bridge. The emergency circuit breaker — the primary defense against a compromised or exploited gravity bridge — is silently non-functional.

---

### Likelihood Explanation

The message is fully registered, routable, and reachable by any tx sender with gas. No special access is required. The integration test at `integration_tests/test_gravity.py` line 661–665 demonstrates the expected behavior (unprivileged call should fail with `code != 0`), confirming the intended design — but the production handler never enforces it. [7](#0-6) 

---

### Recommendation

Replace the stub with the correct implementation, mirroring `UpdateTokenMapping`:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // set bridge enabled/disabled state in keeper
    k.SetBridgeEnabled(ctx, msg.Enable)
    return &types.MsgTurnBridgeResponse{}, nil
}
```

---

### Proof of Concept

Unit test (no special setup required):

1. Create a random address with no permissions set.
2. Submit `MsgTurnBridge{From: randomAddr, Enable: false}` through the `msgServer`.
3. **Observed**: handler returns `(nil, nil)` — tx succeeds, no error.
4. **Expected**: handler returns `sdkerrors.ErrUnauthorized`.
5. Additionally confirm that bridge state is unchanged after the call (no `SetBridgeEnabled` or equivalent is ever called).

The integration test at `test_gravity_turn_bridge` already encodes this expectation at line 662 (`assert rsp["code"] != 0, "should not have the permission"`), but the production handler never satisfies it. [2](#0-1)

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
