### Title
`ft_transfer_call` Partial-Return Amount Not Validated in `exit_to_near_precompile_callback` — (File: `engine/src/contract_methods/connector.rs`)

### Summary

When a user exits ERC-20 tokens to NEAR using the Omni message path (`Message::Omni(msg)`), the engine burns the ERC-20 tokens and calls `ft_transfer_call` on the NEP-141 contract. Per the NEP-141 standard, the receiving NEAR contract may return a portion of the transferred tokens (unused amount). The `exit_to_near_precompile_callback` checks only whether the promise succeeded or failed — it discards the return value entirely (`PromiseResult::Successful(_)`) and never re-mints ERC-20 tokens for the difference. The user permanently loses the returned portion.

### Finding Description

The exit-to-NEAR precompile supports an Omni message path that calls `ft_transfer_call` on the NEP-141 contract, forwarding a user-supplied `msg` to the receiving NEAR contract: [1](#0-0) 

The callback args are constructed without any minimum-amount field: [2](#0-1) 

In `exit_to_near_precompile_callback`, the success branch discards the promise return value with `_`: [3](#0-2) 

The NEP-141 `ft_transfer_call` standard allows the receiving contract to return unused tokens. When it does, the NEP-141 contract's internal `ft_resolve_transfer` credits those tokens back to Aurora's NEP-141 balance. The promise still resolves as `Successful`, so the callback takes the success branch, does nothing, and returns. The ERC-20 tokens were already burned before the promise was dispatched; the returned NEP-141 tokens accumulate in Aurora's balance and are never re-minted for the user.

The `refund` field in `ExitToNearPrecompileCallbackArgs` only covers the case where the entire promise **fails**: [4](#0-3) 

There is no code path that handles a **successful** `ft_transfer_call` that returned fewer tokens than were sent.

### Impact Explanation

A user burns `X` ERC-20 tokens. The receiving NEAR contract (e.g., a DEX performing a swap) returns `Y` tokens unused. The NEP-141 contract credits `Y` tokens back to Aurora's NEP-141 balance. The user receives only `X − Y` NEP-141 tokens at the destination and cannot recover the `Y` tokens. Those `Y` tokens inflate Aurora's NEP-141 backing and will be silently absorbed by the next depositor. From the user's perspective this is a permanent, unrecoverable loss of `Y` tokens — **theft of user funds in motion**.

### Likelihood Explanation

Any user who calls the exit precompile with `Message::Omni(msg)` targeting a NEAR DeFi contract (AMM, aggregator, lending protocol) that may return unused tokens is affected. This is the intended use-case of the Omni path. Partial returns are a normal, spec-compliant NEP-141 behavior and require no attacker — only ordinary market conditions (slippage, partial fills, rejected deposits) trigger the loss.

### Recommendation

In `exit_to_near_precompile_callback`, when the promise result is `Successful`, decode the `U128` return value of `ft_transfer_call` (the amount actually consumed by the receiver). If it is less than the original exit amount, compute the difference and call `refund_on_error` (or an equivalent re-mint path) to restore the shortfall as ERC-20 tokens to the original sender. The `ExitToNearPrecompileCallbackArgs` struct should carry the original exit amount and the sender's EVM address so the callback can perform this check without additional storage reads.

### Proof of Concept

1. User holds 1000 units of a bridged ERC-20 token on Aurora.
2. User calls the exit precompile with `Message::Omni("swap:min_out=900")` and `amount = 1000`.
3. The engine burns 1000 ERC-20 tokens and schedules `ft_transfer_call(dex.near, 1000, "swap:min_out=900")`.
4. The DEX on NEAR executes a partial swap and returns 200 tokens unused via `ft_on_transfer`.
5. The NEP-141 contract's `ft_resolve_transfer` credits 200 tokens back to Aurora's NEP-141 balance.
6. `exit_to_near_precompile_callback` is invoked with `PromiseResult::Successful(800)`.
7. The callback matches the success branch at line 214, ignores the `800` return value, and returns `None`.
8. The user receives 800 NEP-141 tokens at the DEX destination and loses 200 tokens permanently. The 200 tokens sit in Aurora's NEP-141 balance with no mechanism to reclaim them. [5](#0-4) [6](#0-5)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
