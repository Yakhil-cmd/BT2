### Title
Pool Admin Can Permanently Block Protocol Fee Collection and Fee-Rate Updates via Malicious `adminFeeDestination` — (`metric-core/contracts/MetricOmmPoolFactory.sol`, `metric-core/contracts/MetricOmmPool.sol`)

---

### Summary

`MetricOmmPool.collectFees` transfers admin fees to `adminFeeDestination_` **before** transferring protocol fees to `FACTORY`. Because `setPoolAdminFeeDestination` accepts any non-zero address with no further validation, a pool admin can point the destination at a USDC-blacklisted address. Every subsequent call to `collectFees` — including the mandatory pre-flight call inside `setPoolProtocolFee` (factory-owner only) and `setPoolAdminFees` — reverts permanently. The factory owner has no override path to reset `adminFeeDestination`, so protocol fee collection and fee-rate management for that pool are permanently bricked.

---

### Finding Description

`setPoolAdminFeeDestination` enforces only a non-zero check:

```solidity
// MetricOmmPoolFactory.sol L438-446
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    ...
}
``` [1](#0-0) 

Inside `collectFees`, the admin-fee transfer executes **first**:

```solidity
// MetricOmmPool.sol L416-427
if (totalFee0ToAdmin > 0) {
    transferToken0(adminFeeDestination_, totalFee0ToAdmin);   // ← reverts here
}
if (totalFee1ToAdmin > 0) {
    transferToken1(adminFeeDestination_, totalFee1ToAdmin);
}
if (totalFee0ToProtocol > 0) {
    transferToken0(FACTORY, totalFee0ToProtocol);             // ← never reached
}
``` [2](#0-1) 

`collectFees` is called atomically (no try/catch) inside three factory functions:

| Caller | Access | Effect if reverts |
|---|---|---|
| `collectPoolFees` | anyone | fee collection DoS |
| `setPoolProtocolFee` | `onlyOwner` | factory owner cannot update protocol fee rate |
| `setPoolAdminFees` | `onlyPoolAdmin` | pool admin cannot update admin fee rate | [3](#0-2) [4](#0-3) [5](#0-4) 

For a USDC/USDT pool (explicitly in scope), `SafeERC20.safeTransfer` to a USDC-blacklisted address reverts. The factory owner has **no function** to override `poolAdminFeeDestination[pool]` — only `onlyPoolAdmin` can call `setPoolAdminFeeDestination`. [1](#0-0) 

---

### Impact Explanation

- **Protocol fees permanently locked**: `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` accumulate in the pool but `collectFees` always reverts before zeroing them or sending them to `FACTORY`. The protocol cannot extract its earned revenue from that pool.
- **Fee-rate management bricked**: `setPoolProtocolFee` (factory owner) always reverts for the affected pool, so the protocol cannot raise or lower its fee rate in response to market conditions.
- **Admin-boundary break**: A semi-trusted pool admin permanently blocks a factory-owner (protocol-level) action with no recovery path available to the protocol.

---

### Likelihood Explanation

- The pool admin role is semi-trusted and explicitly scoped to operate "only inside caps and timelocks." Blocking the factory owner's fee-management authority is outside that scope.
- For any USDC or USDT pool, a USDC-blacklisted address is trivially obtainable (e.g., a publicly known sanctioned address). The pool admin needs only one call to `setPoolAdminFeeDestination` to trigger the permanent DoS.
- The factory owner has zero on-chain recourse after the fact.

---

### Recommendation

**Option A (preferred):** Split `collectFees` into two independent transfers — admin fees and protocol fees — so a failure on the admin side does not block the protocol side. Use a try/catch or separate the calls:

```solidity
// Transfer admin fees; emit event on failure instead of reverting
try IERC20(TOKEN0).transfer(adminFeeDestination_, totalFee0ToAdmin) {} catch {
    emit AdminFeeTransferFailed(adminFeeDestination_, totalFee0ToAdmin);
}
// Protocol fees always proceed
transferToken0(FACTORY, totalFee0ToProtocol);
```

**Option B:** Allow the factory owner to override `poolAdminFeeDestination[pool]` via a dedicated `onlyOwner` function so the protocol can recover from a stuck destination.

**Option C:** Validate `adminFeeDestination` at set-time (e.g., require a successful zero-value transfer probe) to reject addresses that would cause `safeTransfer` to revert.

---

### Proof of Concept

```solidity
// Assume pool is a USDC/USDT pool
address blacklistedAddr = 0x...; // any USDC-sanctioned address

// Step 1: pool admin poisons the fee destination
factory.setPoolAdminFeeDestination(pool, blacklistedAddr);

// Step 2: factory owner attempts to update protocol fee — permanently reverts
// MetricOmmPoolFactory.setPoolProtocolFee calls pool.collectFees(... blacklistedAddr)
// pool.collectFees calls safeTransfer(USDC, blacklistedAddr, amount) → USDC reverts
vm.prank(factoryOwner);
vm.expectRevert();
factory.setPoolProtocolFee(pool, newSpreadFee, newNotionalFee); // always reverts

// Step 3: anyone calling collectPoolFees also permanently reverts
vm.expectRevert();
factory.collectPoolFees(pool);

// Protocol fees (notionalFeeToken0Scaled / notionalFeeToken1Scaled) remain
// permanently locked in the pool with no recovery path for the protocol.
```
