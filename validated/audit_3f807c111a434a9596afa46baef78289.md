### Title
Presignature Reuse Enabled by Non-Consuming `rerandomize_presign` API Allows Private Key Extraction - (File: `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Both ECDSA variants expose `RerandomizedPresignOutput::rerandomize_presign` taking `presignature: &PresignOutput` (a shared reference) rather than consuming the value. Because `PresignOutput` also derives `Clone`, there is no type-system or runtime enforcement of the library's own documented invariant that "each presignature is consumed exactly once." A malicious coordinator can call `rerandomize_presign` twice on the same `PresignOutput` with different entropy values, run two concurrent signing sessions, and recover the private key from the resulting pair of signatures.

`PresignOutput` derives `Clone` in both variants: [3](#0-2) [4](#0-3) 

Because the presignature is never moved or consumed, a caller can invoke `rerandomize_presign` an arbitrary number of times on the same `PresignOutput`, each time with a different `entropy` value. Each call deterministically produces a distinct `RerandomizedPresignOutput` (different `delta`), all derived from the same underlying nonce material `(R, k_i, sigma_i)`.

The library's own documentation acknowledges the invariant but relies entirely on the caller to enforce it:

> "it's crucial that a presignature is never reused"
> "Each presignature is consumed **exactly once** (one-time use)." [5](#0-4) [6](#0-5) 

The analog to the NEAR double-unlock is exact:

| NEAR double-unlock | This codebase |
|---|---|
| Check: pending transfer exists? → YES | Check: is presignature valid? → YES (reference, never invalidated) |
| Callback removes transfer (separate tx) | Presignature "consumed" conceptually but not in code |
| Second call passes check before callback | Second `rerandomize_presign` call succeeds silently |
| `remove_transfer` returns `None`, no panic | Returns `Ok(RerandomizedPresignOutput)`, no error |

---

### Impact Explanation

The security documentation explicitly states the consequence:

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks." [7](#0-6) 

Concretely: two rerandomizations of the same presignature yield `k₁ = k/δ₁` and `k₂ = k/δ₂`. Both `δ₁` and `δ₂` are public (derived from public inputs via HKDF). Given two ECDSA signatures `(R₁, s₁)` and `(R₂, s₂)` over messages `h₁` and `h₂`, the attacker solves for the private key `x` using the known linear relationship between `k₁` and `k₂`.

**Impact class**: Critical — extraction/reconstruction of the aggregate secret signing key.

---

### Likelihood Explanation

- A **malicious coordinator** controls the `entropy` field in `RerandomizationArguments` and can trivially call `rerandomize_presign` twice with different entropy values before initiating two signing sessions.
- An **honest but buggy coordinator** that retries a failed signing session without generating a fresh presignature hits the same path.
- No cryptographic capability is required; the attack requires only the ability to call the public API twice.
- The `RerandomizationArguments` struct is `Clone` and `entropy` is a plain `[u8; 32]` with no uniqueness enforcement. [8](#0-7) 

---

### Recommendation

Change `rerandomize_presign` to consume the presignature by value in both variants:

```rust
// Before (both variants)
pub fn rerandomize_presign(
    presignature: &PresignOutput,
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>

// After
pub fn rerandomize_presign(
    presignature: PresignOutput,           // ← move semantics enforce one-time use
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

This leverages Rust's ownership system to make presignature reuse a compile-time error. Additionally, remove the `Clone` derive from `PresignOutput` in both variants to prevent callers from cloning the value before passing it.

### Proof of Concept

```rust
// Attacker is a malicious coordinator with access to the presign output
let presign: PresignOutput = run_presign_protocol(…);

// Step 1: rerandomize twice with different entropy — both succeed silently
let args1 = RerandomizationArguments::new(pk, tweak, msg_hash1, presign.big_r, participants.clone(), entropy1);
let args2 = RerandomizationArguments::new(pk, tweak, msg_hash2, presign.big_r, participants.clone(), entropy2);

// Both calls succeed — presignature is never consumed
let rerand1 = RerandomizedPresignOutput::rerandomize_presign(&presign, &args1).unwrap();
let rerand2 = RerandomizedPresignOutput::rerandomize_presign(&presign, &args2).unwrap();

// Step 2: run two signing sessions concurrently
let sig1 = run_sign_session(rerand1, msg_hash1, …);
let sig2 = run_sign_session(rerand2, msg_hash2, …);

// Step 3: recover private key
// delta1, delta2 are public (HKDF outputs from public inputs)
// k1 = k/delta1, k2 = k/delta2  =>  k1*delta1 = k2*delta2
// Standard nonce-reuse recovery: x = (s1*k1 - h1) / r1

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }
