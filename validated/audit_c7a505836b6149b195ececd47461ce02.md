### Title
Unpermissioned `update_rewards` with `disable_rewards: true` Allows Anyone to Permanently Block Reward Distribution for All Stakers - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` has no caller access control. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` to the current block without distributing any rewards. Because the guard `current_block_number > self.last_reward_block.read()` is global (not per-staker), this single call blocks every legitimate reward update for that block. Repeated every block, an attacker can permanently freeze all stakers' unclaimed yield at zero cost beyond gas.

---

### Finding Description

`update_rewards` is part of the public `IStakingRewardsManager` interface and carries no `msg.sender` restriction: [1](#0-0) 

The function begins with `general_prerequisites()`, which only checks that the contract is not paused and the caller is non-zero: [2](#0-1) 

It then writes the current block number into the **global** `last_reward_block` storage slot unconditionally before the `disable_rewards` branch: [3](#0-2) 

If `disable_rewards` is `true`, the function returns immediately after writing `last_reward_block`, distributing nothing: [4](#0-3) 

Because `last_reward_block` is a single global value (not per-staker), any subsequent call to `update_rewards` in the same block — for any staker, with `disable_rewards: false` — will hit the guard and revert: [5](#0-4) 

The storage field declaration confirms it is a single scalar, not a per-staker map: [6](#0-5) 

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block:

1. Advances `last_reward_block` to the current block.
2. Causes every legitimate `update_rewards(..., disable_rewards: false)` call in that block to revert with `REWARDS_ALREADY_UPDATED`.
3. All stakers lose their block rewards for that block — permanently, because `last_reward_block` is never rolled back.

Repeated every block this constitutes **permanent freezing of unclaimed yield** for the entire protocol. Even sporadic calls constitute high-impact griefing because each missed block is an irrecoverable reward loss.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is publicly callable with no role check.
- The attacker needs only a valid (non-zero) staker address (readable from on-chain events) and enough gas to call once per block.
- There is no economic barrier; the attacker does not need to hold any stake.
- The attack is trivially scriptable and can be sustained indefinitely.

---

### Recommendation

Restrict `update_rewards` to a trusted caller — either the attestation contract, a dedicated consensus sequencer address, or a governance-controlled operator role. Add a check analogous to:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();
    // Add: only the authorized consensus/sequencer contract may call this
    assert!(
        get_caller_address() == self.authorized_rewards_updater.read(),
        "{}",
        Error::CALLER_NOT_AUTHORIZED,
    );
    // ... rest of function
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface entirely and handle the "no-reward" path internally based on attestation state, so the function can safely remain public.

---

### Proof of Concept

```python
# Attacker script — run once per block
# 1. Read any active staker address from NewStaker events
staker_address = get_any_active_staker()

# 2. Call update_rewards with disable_rewards=True
# This sets last_reward_block = current_block, distributes nothing
staking_contract.functions.update_rewards(staker_address, True).transact({'from': attacker})

# 3. Now, in the same block, the legitimate consensus call fails:
# staking_contract.functions.update_rewards(staker_address, False).transact(...)
# → panics: REWARDS_ALREADY_UPDATED

# Repeat every block to permanently freeze all staker rewards
```
