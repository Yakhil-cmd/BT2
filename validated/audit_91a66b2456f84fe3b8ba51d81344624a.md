### Title
Staking Contract Pause Blocks `exit_delegation_pool_action` Even When Delegator Funds Are Already in the Pool Contract — (`File: src/pool/pool.cairo`)

---

### Summary

When a staker has been fully removed via `unstake_action`, all delegated funds are transferred to the pool contract. However, `exit_delegation_pool_action` in the pool contract unconditionally calls `remove_from_delegation_pool_action` on the staking contract. That staking-contract function checks `general_prerequisites()` — which asserts the contract is unpaused — **before** it checks whether the intent amount is zero and returns early. If the staking contract is paused at the moment a pool member calls `exit_delegation_pool_action`, the transaction reverts and the pool member cannot retrieve funds that are already sitting in the pool contract.

---

### Finding Description

**Vulnerability class**: Token-freeze bug / state-transition bug

**Call chain**:

```
pool_member → Pool::exit_delegation_pool_action()
                └─→ Staking::remove_from_delegation_pool_action()
                        └─→ self.general_prerequisites()   ← REVERT if paused
                        └─→ if intent.amount.is_zero() { return; }  ← never reached
```

**Root cause in `src/pool/pool.cairo` lines 313–319**:

```cairo
// Note that if the intent was done after the staker was removed (unstake_action),
// the funds will already be in the pool contract, and the following call will do
// nothing.
let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
staking_pool_dispatcher
    .remove_from_delegation_pool_action(identifier: pool_member.into());
``` [1](#0-0) 

The code comment explicitly acknowledges the call "will do nothing" when the staker is removed. But it does not account for the staking contract being paused, which causes the call to **revert** instead of silently returning.

**Root cause in `src/staking/staking.cairo` lines 1113–1120**:

```cairo
fn remove_from_delegation_pool_action(ref self: ContractState, identifier: felt252) {
    self.general_prerequisites();   // ← pause check fires FIRST
    let pool_contract = get_caller_address();
    let undelegate_intent_key = UndelegateIntentKey { pool_contract, identifier };
    let undelegate_intent = self.get_pool_exit_intent(:undelegate_intent_key);
    if undelegate_intent.amount.is_zero() {
        return;   // ← would return here, but never reached when paused
    }
``` [2](#0-1) 

`general_prerequisites()` calls `assert_is_unpaused`:

```cairo
fn assert_is_unpaused(self: @ContractState) {
    assert!(!self.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
}
``` [3](#0-2) 

**Why the staker-removed case is the sharpest analog**: When a staker is removed, `undelegate_from_staking_contract_intent` in the pool contract returns `Time::now()` immediately without ever calling `remove_from_delegation_pool_intent` on the staking contract:

```cairo
if !self.is_staker_active() {
    ...
    return Time::now();   // no staking-contract call, no intent registered
}
``` [4](#0-3) 

Therefore the intent stored in the staking contract for `(pool_contract, pool_member)` is zero. The staking contract would immediately return early — but the pause check fires before that check, causing a revert. The pool member's funds are stranded in the pool contract with no way to retrieve them while the staking contract is paused.

---

### Impact Explanation

**Temporary freezing of funds** (High impact per allowed scope).

A pool member who has completed `exit_delegation_pool_intent` after their staker was removed holds funds that are physically present in the pool contract. The staking contract has no custody of those funds and its call would be a no-op. Nevertheless, a pause on the staking contract prevents `exit_delegation_pool_action` from completing, freezing the pool member's principal for the duration of the pause.

---

### Likelihood Explanation

**Low**. The scenario requires:
1. A staker to have completed `unstake_action` (staker removed, funds in pool contract).
2. A pool member to have called `exit_delegation_pool_intent` after staker removal (setting `unpool_time = Time::now()`).
3. The staking contract to be paused before the pool member calls `exit_delegation_pool_action`.

Pauses are emergency-only events, so the window is narrow. However, the impact when it occurs is concrete and user-facing.

---

### Recommendation

In `exit_delegation_pool_action` (pool.cairo), skip the staking-contract call when the staker has already been removed, since the funds are already in the pool contract and the intent in the staking contract is guaranteed to be zero:

```cairo
if self.is_staker_active() {
    let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
    staking_pool_dispatcher
        .remove_from_delegation_pool_action(identifier: pool_member.into());
}
```

Alternatively, in `remove_from_delegation_pool_action` (staking.cairo), move the zero-intent early-return check **before** the pause assertion, so that a no-op call never reverts due to pause.

---

### Proof of Concept

1. Staker stakes and opens a delegation pool.
2. Pool member calls `enter_delegation_pool` and `exit_delegation_pool_intent`.
3. Staker calls `unstake_intent` then `unstake_action` → staker removed, funds transferred to pool contract via `transfer_to_pools_when_unstake`. [5](#0-4) 
4. Pool member calls `exit_delegation_pool_intent(amount)` again (or for the first time after staker removal) → `undelegate_from_staking_contract_intent` returns `Time::now()` immediately; no intent is registered in the staking contract.
5. Security agent pauses the staking contract.
6. Pool member calls `exit_delegation_pool_action(pool_member)`.
7. Pool contract calls `staking_pool_dispatcher.remove_from_delegation_pool_action(identifier: pool_member.into())`.
8. Staking contract executes `self.general_prerequisites()` → `assert_is_unpaused` → **REVERT with "Contract is paused"**.
9. Pool member cannot retrieve their funds even though those funds are already in the pool contract and the staking contract call would have been a no-op.
