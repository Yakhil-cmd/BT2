### Title
Gateway Stateful Validator Checks Transaction L2 Gas Price Against Current Block's Price Instead of `next_l2_gas_price`, Causing Incorrect Admission Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price from the current (latest) block's `gas_prices.strk_gas_prices.l2_gas_price` to compute the admission threshold, but the transaction will be executed in the **next** block, which uses `next_l2_gas_price` (the EIP-1559-adjusted price stored separately in the block header). The developer acknowledged this with a TODO comment but left the wrong value in place. This causes the gateway to admit transactions that will fail at blockifier pre-validation (when the price is rising) and to reject transactions that would succeed (when the price is falling).

### Finding Description

In `validate_resource_bounds`, the gateway fetches block info and extracts the L2 gas price:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

This value is then used to compute the admission threshold:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold { ... }
``` [2](#0-1) 

The block header carries two distinct L2 gas price fields:

- `l2_gas_price: GasPricePerToken` — the price used for the **current** block's transactions.
- `next_l2_gas_price: GasPrice` — the EIP-1559-adjusted price that will be used for the **next** block. [3](#0-2) 

The gateway's `get_block_info()` returns a `BlockInfo` struct that exposes only `gas_prices` (current block prices) and has no field for `next_l2_gas_price`. [4](#0-3) 

The blockifier's definitive admission check in `check_fee_bounds` compares the transaction's `max_price_per_unit` against the **actual block's** gas price (which is `next_l2_gas_price` for the block being built): [5](#0-4) 

The two checks therefore use different base prices. The EIP-1559 mechanism (`calculate_next_base_gas_price`) can move the price up or down each block: [6](#0-5) 

### Impact Explanation

**When `next_l2_gas_price > l2_gas_price` (block was full, price rising):**

The gateway threshold is computed against the lower `l2_gas_price`. A transaction with `max_price_per_unit` satisfying `threshold_wrong ≤ tx_price < next_l2_gas_price` passes the gateway check and enters the mempool. When the batcher attempts to include it, `check_fee_bounds` in `perform_pre_validation_stage` rejects it with `MaxGasPriceTooLow`. The transaction pollutes the mempool and wastes batcher resources.

**When `next_l2_gas_price < l2_gas_price` (block was empty, price falling):**

The gateway threshold is computed against the higher `l2_gas_price`. A transaction with `max_price_per_unit` satisfying `next_l2_gas_price ≤ tx_price < threshold_wrong` is rejected by the gateway even though the blockifier would accept it. Valid user transactions are silently dropped at the admission layer — a user-facing DoS.

This matches: **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price every block based on gas usage relative to `gas_target`. Under sustained high or low load, the price drifts continuously, making the discrepancy between `l2_gas_price` and `next_l2_gas_price` persistent and predictable. Any user whose wallet or SDK sets `max_price_per_unit` based on the published `next_l2_gas_price` (the correct reference) will be affected when the price is falling. The trigger requires no privilege — any standard V3 transaction submission suffices.

### Recommendation

Replace the `get_block_info()` call with a direct read of `next_l2_gas_price` from the block header. The `GatewayFixedBlockStateReader` trait should expose a method (e.g., `get_next_l2_gas_price`) that reads `BlockHeaderWithoutHash::next_l2_gas_price` from storage, and `validate_resource_bounds` should use that value as the base for the threshold computation.

### Proof of Concept

Assume `min_gas_price_percentage = 90`, current block `l2_gas_price = 100 FRI`, and the block was full so `next_l2_gas_price = 112 FRI` (12% increase).

**Wrong threshold (current code):** `0.90 × 100 = 90 FRI`

**Correct threshold:** `0.90 × 112 = 100.8 FRI`

A transaction with `max_price_per_unit = 95 FRI`:
- Gateway check: `95 ≥ 90` → **admitted** (wrong)
- Blockifier check: `95 < 112` → **rejected** with `MaxGasPriceTooLow`

The transaction enters the mempool, is pulled by the batcher, and fails at `perform_pre_validation_stage` → `check_fee_bounds`. [7](#0-6)

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
