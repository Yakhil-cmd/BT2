Audit Report

## Title
Unprivileged Caller Can Dilute Any NNS Neuron's Age Bonus via Unauthenticated `ClaimOrRefresh` - (File: `rs/nns/governance/src/governance.rs`)

## Summary
The NNS governance `manage_neuron_internal` function dispatches `ClaimOrRefresh` commands before any neuron-ownership authorization checks. An unprivileged caller can transfer a small amount of ICP to any victim neuron's public subaccount and then call `manage_neuron` with `ClaimOrRefresh { by: NeuronIdOrSubaccount }` or `By::MemoAndController` to trigger `update_stake_adjust_age` on that neuron without the owner's consent. This permanently dilutes the victim neuron's `aging_since_timestamp_seconds`, reducing the age bonus (up to 1.25×) applied to voting power and staking rewards.

## Finding Description
In `rs/nns/governance/src/governance.rs`, `manage_neuron_internal` handles `ClaimOrRefresh` at L6106 before any normal neuron-ownership authorization checks run. The comment at L6104 explicitly states this is intentional for the claim path, but it also applies to the refresh path for existing neurons. Neither `refresh_neuron_by_id_or_subaccount` (L5875–5896) nor `refresh_neuron` (L5900–5962) contains any check that the caller is the neuron controller or a registered hotkey.

When `refresh_neuron` detects that the ledger balance exceeds the cached stake (`Ordering::Less` branch at L5950), it unconditionally calls `neuron.update_stake_adjust_age(balance.get_e8s(), now)`. This function (in `rs/nns/governance/src/neuron/types.rs` L999–1040) calls `combine_aged_stakes` with the added stake having `age_seconds = 0`, producing a weighted-average age that is strictly less than the original age. The result is permanently written back to `aging_since_timestamp_seconds`.

The `By::MemoAndController` path (L5857–5860) accepts an arbitrary `controller` field, allowing the attacker to target any neuron by specifying the victim's principal and memo. The `By::NeuronIdOrSubaccount` path allows direct targeting by neuron ID. Because every neuron's staking subaccount is a deterministically derived public ICP ledger account, any attacker can transfer ICP to it without the owner's consent, then call `ClaimOrRefresh` to trigger the age dilution.

The same unauthenticated `refresh_neuron` pattern exists in SNS governance at `rs/sns/governance/src/governance.rs` L4210–4227.

## Impact Explanation
A neuron's age bonus contributes up to 25% additional voting power (1.25× multiplier at 4 years, confirmed in `rs/nns/governance/src/neuron/voting_power.rs` L23–31) and proportionally higher staking rewards. By repeatedly sending small amounts of ICP to a victim's neuron subaccount and calling `ClaimOrRefresh`, an attacker can continuously dilute the victim's `aging_since_timestamp_seconds`. Each refresh permanently resets the age to a weighted average. A victim neuron aged 4 years (maximum age bonus) can have its age bonus materially reduced. The victim gains the attacker's ICP as additional stake, but the age bonus reduction permanently lowers their future reward rate and governance weight. This constitutes unauthorized modification of a governance asset (neuron age/voting power) with concrete, permanent harm to the neuron owner's reward rate and governance influence. This matches the **Medium** bounty impact: a meaningful security impact on NNS governance assets requiring real cost (ICP proportional to victim's stake) to execute.

## Likelihood Explanation
The attack requires no special permissions—only the ability to send ICP to a public ledger account and call `manage_neuron`. The victim's neuron subaccount is publicly derivable from their principal and memo. The `ClaimOrRefresh` path bypasses all normal neuron-ownership checks. Any ICP holder can execute this against any neuron at any time. Small amounts cause minor but real dilution at low cost; significant dilution requires ICP proportional to the victim's stake.

## Recommendation
Add an authorization check inside `refresh_neuron` (or at the `ClaimOrRefresh` dispatch site) requiring that the caller is the neuron controller or a registered hotkey before calling `update_stake_adjust_age` on an existing neuron. Alternatively, separate the "update cached stake" operation (which can remain permissionless, to allow the ledger balance to be reflected) from the "adjust age" operation (which should require owner authorization), so that third-party top-ups increase the cached stake without diluting the age.

## Proof of Concept
1. Victim has neuron N with stake S = 100 ICP and age A = 4 years (maximum age bonus 1.25×).
2. Attacker transfers δ = 100 ICP to neuron N's subaccount (a public ICP ledger account, derivable from victim's principal and memo).
3. Attacker calls `manage_neuron` with `ClaimOrRefresh { by: NeuronIdOrSubaccount(Empty {}) }` and `neuron_id_or_subaccount: NeuronId(N)`.
4. `refresh_neuron` reads `balance = 200 ICP > cached_stake = 100 ICP`, enters the `Ordering::Less` branch, and calls `update_stake_adjust_age(200 ICP, now)`.
5. `combine_aged_stakes(100_ICP, 4_years, 100_ICP, 0)` → new age = 2 years.
6. Victim's age bonus drops from 25% to ~12.5%; future reward rate is permanently reduced.
7. Attacker can repeat, each time halving the remaining age, at the cost of ICP that accrues to the victim's neuron.
8. A deterministic integration test using PocketIC can reproduce this by: creating a neuron, advancing time to accumulate age, transferring ICP from a second identity to the neuron's subaccount, calling `manage_neuron(ClaimOrRefresh { by: NeuronIdOrSubaccount })` from the second identity, and asserting that `aging_since_timestamp_seconds` has increased (age decreased).

    async fn refresh_neuron(
        &mut self,
        nid: NeuronId,
        subaccount: Subaccount,
        claim_or_refresh: &ClaimOrRefresh,
    ) -> Result<NeuronId, GovernanceError> {
        let account = neuron_subaccount(subaccount);
        // We need to lock the neuron to make sure it doesn't undergo
        // concurrent changes while we're checking the balance and
        // refreshing the stake.
        let now = self.env.now();
        let _neuron_lock = self.lock_neuron_for_command(
            nid.id,
            NeuronInFlightCommand {
                timestamp: now,
                command: Some(InFlightCommand::ClaimOrRefreshNeuron(
                    claim_or_refresh.clone(),
                )),
            },
        )?;

        // Get the balance of the neuron from the ledger canister.
        tla_log_locals! { neuron_id: nid.id };
        let balance = self.ledger.account_balance(account).await?;
        let min_stake = self.economics().neuron_minimum_stake_e8s;
        if balance.get_e8s() < min_stake {
            return Err(GovernanceError::new_with_message(
                ErrorType::InsufficientFunds,
                format!(
                    "Account does not have enough funds to refresh a neuron. \
                     Please make sure that account has at least {:?} e8s (was {:?} e8s)",
                    min_stake,
                    balance.get_e8s()
                ),
            ));
        }
        self.with_neuron_mut(&nid, |neuron| {
            match neuron.cached_neuron_stake_e8s.cmp(&balance.get_e8s()) {
                Ordering::Greater => {
                    println!(
                        "{}ERROR. Neuron cached stake was inconsistent.\
                     Neuron account: {} has less e8s: {} than the cached neuron stake: {}.\
                     Stake adjusted.",
                        LOG_PREFIX,
                        account,
                        balance.get_e8s(),
                        neuron.cached_neuron_stake_e8s
                    );
                    neuron.update_stake_adjust_age(balance.get_e8s(), now);
                }
                Ordering::Less => {
                    neuron.update_stake_adjust_age(balance.get_e8s(), now);
                }
                // If the stake is the same as the account balance,
                // just return the neuron id (this way this method
                // also serves the purpose of allowing to discover the
                // neuron id based on the memo and the controller).
                Ordering::Equal => (),
            };
        })?;

        Ok(nid)
    }

pub(crate) fn age_bonus_multiplier(age_seconds: u64) -> Decimal {
    let age_seconds = Decimal::from(age_seconds.clamp(0, MAX_NEURON_AGE_FOR_AGE_BONUS));

    // t is (clamped) age in units of max age, so its value is from 0.0 to 1.0
    let t = age_seconds / Decimal::from(MAX_NEURON_AGE_FOR_AGE_BONUS);

    // 0.25 * t + 1
    t / Decimal::from(4) + Decimal::from(1)
}
