### Title
No Deadline Parameter in `unstake_intent` and `exit_delegation_pool_intent` Allows Increased `exit_wait_window` to Extend User's Lock Period - (File: src/staking/staking.cairo)

---

### Summary
Both `unstake_intent` and `exit_delegation_pool_intent` compute the lock-release timestamp as `Time::now() + exit_wait_window` at execution time, reading the current global `exit_wait_window` from storage. Because `exit_wait_window` can be increased by the `TOKEN_ADMIN` at any time via `set_exit_wait_window`, a user's pending transaction can execute against a larger window than the one they observed when they signed and submitted it, temporarily freezing their funds for longer than expected.

---

### Finding Description

In `src/staking/staking.cairo`, `unstake_intent` reads the live `exit_wait_window` at execution time:

```rust
let unstake_time = Time::now().add(delta: self.exit_wait_window.read());
``` [1](#0-0) 

The internal helper `update_undelegate_intent_value`, called by `remove_from_delegation_pool_intent` (which backs `exit_delegation_pool_intent`), does the same through `compute_unpool_time`:

```rust
let unpool_time = staker_info
    .compute_unpool_time(exit_wait_window: self.exit_wait_window.read());
``` [2](#0-1) 

`compute_unpool_time` itself is:

```rust
fn compute_unpool_time(self: @InternalStakerInfoLatest, exit_wait_window: TimeDelta) -> Timestamp {
    if let Option::Some(unstake_time) = *self.unstake_time {
        return max(unstake_time, Time::now());
    }
    Time::now().add(delta: exit_wait_window)
}
``` [3](#0-2) 

The `exit_wait_window` storage variable is writable by `TOKEN_ADMIN` with no lower-bound guard on the new value relative to the old one:

```rust
fn set_exit_wait_window(ref self: ContractState, exit_wait_window: TimeDelta) {
    self.roles.only_token_admin();
    assert!(exit_wait_window <= MAX_EXIT_WAIT_WINDOW, "{}", Error::ILLEGAL_EXIT_DURATION);
    let old_exit_window = self.exit_wait_window.read();
    self.exit_wait_window.write(exit_wait_window);
    ...
}
``` [4](#0-3) 

Neither `unstake_intent` nor `exit_delegation_pool_intent` accepts a caller-supplied `max_exit_wait_window` (deadline) parameter. There is no check that the window at execution time matches what the user observed when they constructed the transaction.

---

### Impact Explanation

**Impact: High — Temporary freezing of funds.**

A staker or delegator who submits an exit-intent transaction while observing `exit_wait_window = W` will have their funds locked until `now_at_execution + W'`, where `W' > W` if the window was raised between submission and inclusion. The user cannot cancel the intent once it is recorded (re-calling `exit_delegation_pool_intent` with `amount = 0` cancels it, but the staker's `unstake_intent` has no cancellation path at all). The staker's own stake and all delegated balances are removed from `total_stake` at intent time, so the staker also loses attestation eligibility and rewards for the extended period.

---

### Likelihood Explanation

**Likelihood: Medium.**

`exit_wait_window` is a governance parameter expected to be adjusted over the protocol's lifetime (e.g., in response to security events or network upgrades). Starknet's sequencer model means a transaction can sit in the mempool for multiple blocks. A coincident governance update and a pending exit-intent transaction is a realistic scenario, especially during protocol upgrades or emergency parameter changes. No malicious intent by the `TOKEN_ADMIN` is required — a routine, well-intentioned increase is sufficient to trigger the issue.

---

### Recommendation

Add an optional `max_exit_wait_window: Option<TimeDelta>` parameter to both `unstake_intent` and `exit_delegation_pool_intent`. Before computing `unpool_time`, assert:

```rust
if let Option::Some(max_window) = max_exit_wait_window {
    assert!(self.exit_wait_window.read() <= max_window, Error::EXIT_WAIT_WINDOW_TOO_HIGH);
}
```

This gives callers a slippage-protection mechanism analogous to a deadline, ensuring the transaction reverts rather than silently locking funds for longer than the user accepted.

---

### Proof of Concept

1. `TOKEN_ADMIN` sets `exit_wait_window = 7 days` (current live value).
2. Staker observes `exit_wait_window = 7 days` and submits `unstake_intent()`.
3. Before the transaction is sequenced, `TOKEN_ADMIN` calls `set_exit_wait_window(21 days)` — a legitimate security measure.
4. Staker's `unstake_intent` transaction is included in the next block.
5. `unstake_time` is set to `now + 21 days` instead of the expected `now + 7 days`.
6. The staker's own stake and all delegated balances are already removed from `total_stake` (line 468), so the staker earns no rewards and cannot attest for the extra 14 days.
7. The staker has no way to cancel `unstake_intent` — funds are frozen for 21 days.

The same scenario applies to a delegator calling `exit_delegation_pool_intent`: the `unpool_time` stored in `pool_exit_intents` is computed from the live `exit_wait_window` at execution time. [5](#0-4) [6](#0-5)

        fn unstake_intent(ref self: ContractState) -> Timestamp {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            // Set the unstake time.
            let unstake_time = Time::now().add(delta: self.exit_wait_window.read());
            staker_info.unstake_time = Option::Some(unstake_time);
            self.write_staker_info(:staker_address, :staker_info);

            // Write the unstake intent epoch.
            self.staker_unstake_intent_epoch.write(staker_address, self.get_epoch_plus_k());

            // Write off the delegated stake from the total stake.
            for (pool_contract, token_address) in self
                .staker_pool_info
                .entry(staker_address)
                .pools {
                let amount = self.get_delegated_balance(:staker_address, :pool_contract);
                self.remove_from_total_stake(:token_address, :amount);
                let decimals = self.get_token_decimals(:token_address);
                self
                    .emit(
                        Events::StakeDelegatedBalanceChanged {
                            staker_address,
                            token_address,
                            old_delegated_stake: amount.to_native_amount(:decimals),
                            new_delegated_stake: Zero::zero(),
                        },
                    );
            }
            // Write off the self stake from the total stake.
            let old_self_stake = self.get_own_balance(:staker_address);
            self.remove_from_total_stake(token_address: STRK_TOKEN_ADDRESS, amount: old_self_stake);

            // Emit events.
            self.emit(Events::StakerExitIntent { staker_address, exit_timestamp: unstake_time });
            self
                .emit(
                    Events::StakeOwnBalanceChanged {
                        staker_address,
                        old_self_stake: old_self_stake.to_strk_native_amount(),
                        new_self_stake: Zero::zero(),
                    },
                );
            unstake_time
        }

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }
