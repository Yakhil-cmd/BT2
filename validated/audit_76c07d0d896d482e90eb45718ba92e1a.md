### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Block Reward Distribution - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the Staking contract is intended to be callable only by the Starkware sequencer (per the protocol spec), but the implementation contains no caller restriction. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block reward slot and permanently preventing the sequencer from distributing rewards for that block to all stakers.

### Finding Description
The spec at `docs/spec.md` line 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1449–1507 contains no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();  // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... NO caller check ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;  // returns without distributing rewards
    }
    // ... reward distribution ...
}
```

The `last_reward_block` is a single global storage variable. Once it is written to the current block number, any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`.

An attacker can:
1. At the start of every block, call `update_rewards(any_valid_staker, disable_rewards: true)`.
2. This writes `last_reward_block = current_block` and returns early with zero rewards distributed.
3. The sequencer's legitimate `update_rewards(..., disable_rewards: false)` call for that block reverts.
4. No staker accumulates rewards for that block.

Repeating this every block permanently freezes all unclaimed yield accumulation across the entire protocol.

### Impact Explanation
This is a **permanent freezing of unclaimed yield** for all stakers and delegators. Rewards are never credited to `unclaimed_rewards_own` for any staker, and pool rewards are never transferred to delegation pools. The attacker has no profit motive — this is pure griefing. The impact matches the allowed High impact: **"Permanent freezing of unclaimed yield or unclaimed royalties"**.

### Likelihood Explanation
The entry point is fully public with no authentication. The only cost to the attacker is gas per block. On Starknet, gas costs are low. The attack is trivially automatable and requires no special privileges, tokens, or setup. Any address can execute it indefinitely.

### Recommendation
Add a caller check at the top of `update_rewards` to enforce that only the authorized sequencer address (stored in contract configuration) can invoke it, consistent with the spec:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

### Proof of Concept

1. Staker stakes and K epochs pass so the staker becomes active.
2. Consensus rewards are enabled.
3. At the start of block N, attacker calls:
   ```
   staking.update_rewards(staker_address: any_valid_staker, disable_rewards: true)
   ```
4. `last_reward_block` is set to block N; function returns with no rewards distributed.
5. Sequencer calls `staking.update_rewards(staker_address: ..., disable_rewards: false)` — reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker's `unclaimed_rewards_own` remains zero; pool receives no funds.
7. Attacker repeats step 3 every block. No staker ever earns rewards.

            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake

            self.last_reward_block.write(current_block_number);

                    :staker_address,

                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,

                    :staker_pool_info,
                    :reward_supplier_dispatcher,
