### Title
Unprivileged caller can permanently freeze block rewards for all stakers by calling `update_rewards` with `disable_rewards = true` - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` accepts a caller-controlled `disable_rewards` boolean with no access-control check. Any unprivileged address can call it with `disable_rewards = true` to advance the global `last_reward_block` checkpoint without distributing rewards, permanently denying block rewards to every staker for that block.

### Finding Description
`update_rewards` is exposed as a public function in the `IStakingRewardsManager` interface. Its implementation in `StakingRewardsManagerImpl` performs only two generic guards — the contract must not be paused and the caller must not be the zero address — before writing the current block number into the global `last_reward_block` storage slot:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← written unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits before distributing
}
``` [1](#0-0) 

Because `last_reward_block` is a **single global slot** shared across all stakers, any subsequent call to `update_rewards` for the same block will revert with `REWARDS_ALREADY_UPDATED`:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

The `general_prerequisites()` guard that is the only access check called before the write enforces only "not paused" and "caller ≠ zero": [3](#0-2) 

There is no check that the caller is the attestation contract, the staker, or any other privileged role.

### Impact Explanation
An attacker calls `update_rewards(any_active_staker, true)` once per block. This:
1. Advances `last_reward_block` to the current block.
2. Returns without computing or distributing any STRK/BTC block rewards.
3. Permanently prevents any other call to `update_rewards` for that block (the assertion at line 1453–1458 will always fail for the same block number).

Because the slot is global, **every staker** loses their block reward for that block. Repeated across every block, the attacker can permanently freeze all consensus-era block rewards for the entire protocol at the cost of one transaction per block. The rewards are not deferred — they are simply never credited to `unclaimed_rewards_own` or forwarded to delegation pools.

This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The entry point is a public, permissionless function callable by any non-zero address. No stake, no role, no prior state is required. The only cost is the gas for one Starknet transaction per block. The attack is trivially automatable and economically rational for any party that benefits from suppressing staker rewards (e.g., a competing protocol or a griefing actor).

### Recommendation
Add an access-control guard to `update_rewards` so that only the attestation contract (or another designated trusted caller) may invoke it. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_attestation_contract(); // ← add this
    ...
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface entirely and derive the flag internally from whether the staker has attested in the current epoch.

### Proof of Concept

```
// Attacker (any address) calls once per block:
staking.update_rewards(
    staker_address: any_active_staker,
    disable_rewards: true,   // ← caller-controlled, no check
);
// Effect:
// - last_reward_block is now == current_block_number
// - no rewards were distributed
// - every subsequent update_rewards call for this block reverts with REWARDS_ALREADY_UPDATED
// - all stakers permanently lose block rewards for this block
``` [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
