### Title
Reward Supplier `claim_rewards` Transfers Without Checking Actual Token Balance, Causing Revert When L1 Funds Are Pending — (File: src/reward_supplier/reward_supplier.cairo)

---

### Summary

`RewardSupplier.claim_rewards` validates only the accounting ledger `unclaimed_rewards` before calling `checked_transfer`, but never checks whether the contract's actual STRK token balance is sufficient. Because L1-minted funds arrive asynchronously, the actual balance can be lower than `unclaimed_rewards` during the L1 bridge delay window. Any call that reaches `claim_rewards` during that window — including a staker's `claim_rewards` or `unstake_action` — will have `checked_transfer` panic and the entire transaction revert, temporarily freezing both unclaimed yield and staked principal.

---

### Finding Description

`RewardSupplier.claim_rewards` (lines 205–220) performs an accounting check:

```cairo
assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);
self.unclaimed_rewards.write(unclaimed_rewards - amount);
token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
``` [1](#0-0) 

The guard `amount <= unclaimed_rewards` is a ledger check, not a balance check. `unclaimed_rewards` is incremented immediately inside `update_unclaimed_rewards_from_staking_contract`: [2](#0-1) 

Actual STRK tokens only arrive later, via `on_receive` from the StarkGate bridge. The `request_funds` helper requests L1 minting when `balance + l1_pending_requested_amount < unclaimed_rewards + threshold`, but it counts *pending* L1 amounts as credit: [3](#0-2) 

During the window between reward accrual and L1 fund arrival, `actual_balance < unclaimed_rewards`. If `amount > actual_balance` at the moment `checked_transfer` executes, the ERC-20 transfer panics and the entire transaction reverts.

The staking contract reaches this code path in two unprivileged-user-accessible flows:

**Flow 1 — `Staking.claim_rewards`** (callable by staker or reward address): [4](#0-3) 

which calls `send_rewards_to_staker`: [5](#0-4) 

**Flow 2 — `Staking.unstake_action`** (callable by anyone for any staker that has completed the intent window): [6](#0-5) 

`unstake_action` calls `send_rewards_to_staker` before returning the principal, so a revert here also blocks the return of the staked principal.

---

### Impact Explanation

- **Temporary freezing of unclaimed yield (High):** Any staker whose `unclaimed_rewards_own > actual_balance_of_reward_supplier` cannot claim rewards; the transaction reverts.
- **Temporary freezing of funds (High):** `unstake_action` calls `send_rewards_to_staker` before transferring the principal back. If the reward supplier balance is insufficient, the staker cannot recover their staked STRK even after the exit window has elapsed.

---

### Likelihood Explanation

L1-to-L2 bridge latency is a normal, expected condition on Starknet. The `request_funds` mechanism explicitly acknowledges that funds are not immediately available (`l1_pending_requested_amount` tracks outstanding requests). Any epoch in which rewards are calculated but L1 funds have not yet been bridged creates the vulnerable window. Because `update_rewards` / `update_rewards_from_attestation_contract` is called every block, and `claim_rewards` / `unstake_action` can be called immediately after, the window is routinely open. An unprivileged staker with non-zero `unclaimed_rewards_own` can trigger the revert without any special access.

---

### Recommendation

Before executing `checked_transfer` in `claim_rewards`, verify the actual token balance:

```cairo
fn claim_rewards(ref self: ContractState, amount: Amount) {
    ...
    let unclaimed_rewards = self.unclaimed_rewards.read();
    assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

    // Add: guard against insufficient actual balance
    let balance: Amount = token_dispatcher
        .balance_of(account: get_contract_address())
        .try_into()
        .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);
    assert!(amount <= balance, "{}", Error::INSUFFICIENT_BALANCE);

    self.unclaimed_rewards.write(unclaimed_rewards - amount);
    token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
}
```

This mirrors the validation pattern already present in `request_funds` (which reads `balance_of`) and prevents a cryptic panic from the token contract propagating up to freeze staker operations.

---

### Proof of Concept

1. Epoch N ends; `update_rewards` is called for staker S, accruing 100 STRK in `unclaimed_rewards_own` and calling `update_unclaimed_rewards_from_staking_contract(100)`.
2. `request_funds` sees `balance=50, l1_pending=60, unclaimed=100, threshold=T`; since `50+60 >= 100+T` (assuming T≤10), no new L1 request is sent.
3. L1 bridge is congested; the 60 STRK pending from a prior request has not yet arrived. Actual balance remains 50 STRK.
4. Staker S calls `Staking.claim_rewards(S)` (or `unstake_action(S)` after the exit window).
5. Staking calls `reward_supplier.claim_rewards(100)`.
6. `assert!(100 <= 100)` passes; `unclaimed_rewards` is decremented to 0.
7. `token_dispatcher.checked_transfer(staking_contract, 100)` panics because actual balance is 50.
8. Entire transaction reverts; staker S cannot claim rewards or recover principal. [7](#0-6)

        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }
