### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `IStakingRewardsManager` is documented in the protocol spec as callable "Only starkware sequencer," but the implementation contains no such access control check. Any unprivileged caller can invoke it with `disable_rewards: true`, consuming the global per-block reward slot (`last_reward_block`) without distributing rewards. This permanently destroys the targeted staker's block rewards, since the sequencer's subsequent call in the same block is rejected with `REWARDS_ALREADY_UPDATED`.

### Finding Description
The `update_rewards` function at `src/staking/staking.cairo` lines 1449–1507 is the consensus-phase reward distribution entry point. The protocol specification explicitly states its access control is "Only starkware sequencer."

The implementation begins with `general_prerequisites()`:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

This only checks that the contract is not paused and the caller is non-zero. There is no check that `get_caller_address()` equals any authorized sequencer address. No sequencer address is stored in contract storage at all.

The function then unconditionally writes `current_block_number` to the global `last_reward_block` storage slot **before** evaluating `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

`last_reward_block` is a single global variable, not per-staker:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [3](#0-2) 

The guard at the top of the function enforces that only one call per block can succeed:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

**Attack path:** An unprivileged caller submits `update_rewards(staker_address: victim, disable_rewards: true)` in block N. The call passes all checks (contract not paused, caller non-zero, staker exists and active, staker has non-zero balance). `last_reward_block` is set to N and the function returns early without distributing rewards. The sequencer's own `update_rewards` call in block N then fails with `REWARDS_ALREADY_UPDATED`. The victim staker's block-N rewards are permanently lost — they are never credited to `unclaimed_rewards_own` and cannot be recovered.

The spec confirms the intended restriction: [5](#0-4) 

### Impact Explanation
Each successful attack permanently destroys one block's worth of unclaimed yield for the targeted staker. The `_update_rewards` internal function, which credits `staker_info.unclaimed_rewards_own` and forwards pool rewards, is never reached. Because `last_reward_block` is global, a single attacker call blocks reward distribution for the entire block. The loss is irreversible: there is no mechanism to retroactively credit missed block rewards. This constitutes **permanent freezing of unclaimed yield** (High severity per the allowed impact scope). [6](#0-5) 

### Likelihood Explanation
The attack requires only a valid staker address (publicly observable on-chain) and a non-zero caller. The transaction cost is a single Starknet fee. On Starknet the sequencer controls intra-block ordering, so the sequencer could place its own call first; however, the sequencer may not call `update_rewards` in every block (e.g., when a staker is not selected), and the absence of any on-chain enforcement means the invariant relies entirely on off-chain sequencer behavior. A motivated attacker can monitor the mempool and submit the griefing call whenever the sequencer's call has not yet appeared, or target blocks where the sequencer omits the call. The attack is cheap, repeatable every block, and requires no privileged access.

### Recommendation
Store an authorized sequencer address in contract storage and add an explicit caller check at the top of `update_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

This aligns the implementation with the specification's stated access control of "Only starkware sequencer."

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. Staker `S` has staked and waited K epochs so their balance is effective.
3. In block N, before the sequencer's `update_rewards` call, attacker `A` (any non-zero address) calls:
   ```
   update_rewards(staker_address: S, disable_rewards: true)
   ```
4. The call succeeds: staker exists, is active, has non-zero balance. `last_reward_block` is written to N. Function returns early — no rewards distributed.
5. Sequencer calls `update_rewards(staker_address: S, disable_rewards: false)` in block N → reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker `S`'s `unclaimed_rewards_own` is not incremented for block N. The block-N rewards are permanently lost.
7. Attacker repeats every block to continuously deny yield. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
