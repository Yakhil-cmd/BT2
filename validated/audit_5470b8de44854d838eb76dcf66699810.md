### Title
Unvalidated `app_pk` Identity Element in `ckd` Bypasses ElGamal Encryption, Disclosing Confidential Derived Key — (File: `src/confidential_key_derivation/protocol.rs`)

---

### Summary

The public `ckd` function accepts an `app_pk` (application public key) parameter without validating that it is not the identity element of the BLS12-381 G1 group. A malicious coordinator can pass the identity element as `app_pk`, causing the ElGamal encryption layer to be entirely bypassed and the confidential derived key `big_s = hash_point * msk` to be directly readable from the CKD output — without possessing the TEE application's private key `app_sk`.

---

### Finding Description

The CKD protocol's purpose is to deliver a deterministic secret `big_s = H(pk ‖ app_id) · msk` to a TEE application, encrypted under the application's public key `app_pk` so that only the TEE (holding `app_sk`) can decrypt it.

In `compute_signature_share` the encryption is performed as:

```
big_s  = hash_point * private_share          // confidential derived key share
big_c  = big_s + app_pk * y                  // ElGamal ciphertext
``` [1](#0-0) 

If `app_pk` is the identity element (point at infinity in G1), then `app_pk * y = identity`, so `big_c = big_s`. Each participant then sends `(λ_i · big_y_i, λ_i · big_s_i)` to the coordinator. The coordinator aggregates:

```
Σ λ_i · big_c_i  =  Σ λ_i · big_s_i  =  big_s   (Lagrange interpolation)
``` [2](#0-1) 

The final `CKDOutput.big_c` therefore equals `big_s` in the clear — no `app_sk` is needed to read it.

The public entry point `ckd` validates participant membership and coordinator presence, but performs **no check** that `app_pk` is not the identity element: [3](#0-2) 

---

### Impact Explanation

A malicious coordinator who distributes `app_pk = G1::identity()` to all participants receives a `CKDOutput` where `big_c = big_s = H(pk ‖ app_id) · msk`. This is a direct, unauthenticated disclosure of the confidential derived key — the exact secret the ElGamal layer is designed to protect. The coordinator learns the full derived key without ever possessing `app_sk`, violating the core confidentiality guarantee of the CKD protocol.

This matches the allowed Critical impact: **Extraction, reconstruction, or disclosure of confidential derived secrets**.

---

### Likelihood Explanation

The `ckd` function is a public library API. In typical deployments the coordinator is the party that distributes `app_pk` to all participants before the protocol run. A malicious coordinator can trivially supply `app_pk = ElementG1::identity()` — the point at infinity is a valid, serializable BLS12-381 G1 element that the `blstrs` library accepts without error. No cryptographic break, leaked key, or external oracle is required; the attack is a single parameter substitution at call time. [4](#0-3) 

---

### Recommendation

Reject the identity element for `app_pk` at the start of `ckd`, before the protocol is launched:

```rust
if app_pk == ElementG1::identity() {
    return Err(InitializationError::BadParameters(
        "app_pk must not be the identity element".to_string(),
    ));
}
```

This mirrors the existing pattern used elsewhere in the codebase (e.g., the `IdentityElement` error in `do_presign`). [5](#0-4) 

---

### Proof of Concept

1. Malicious coordinator calls `ckd(participants, coordinator, me, key_pair, app_id, ElementG1::identity(), rng)` for every participant.
2. Each honest participant executes `compute_signature_share` and computes `big_c_i = big_s_i + identity * y_i = big_s_i`.
3. Each participant sends `(λ_i · G · y_i, λ_i · big_s_i)` privately to the coordinator.
4. Coordinator aggregates: `Σ λ_i · big_s_i = H(pk ‖ app_id) · msk = big_s`.
5. The returned `CKDOutput` has `big_c = big_s`; the coordinator reads the confidential derived key directly, with no use of `app_sk`. [6](#0-5)

pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    let comms = Comms::new();
    let chan = comms.shared_channel();

    let fut = run_ckd_protocol(
        chan,
        coordinator,
        me,
        participants,
        key_pair,
        app_id.into(),
        app_pk,
        rng,
    );
    Ok(make_protocol(comms, fut))
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
