### Title
Pool Rewards Permanently Locked When Pool Balance Is Below `min_delegation_for_rewards` - (File: `src/pool/pool.cairo`)

### Summary
When a delegation pool's total balance is below `min_delegation_for_rewards`, the staking contract still transfers STRK reward tokens to the pool contract, but the pool's `compute_rewards_per_unit` returns zero, meaning those tokens are never credited to any pool member and are permanently locked in the pool contract with no recovery path.

### Finding Description
In `src/pool/pool.cairo`, the internal function `compute_rewards_per_unit` explicitly returns zero when `total_stake < self.min_delegation_for_rewards.read()`:

```cairo
fn compute_rewards_per_unit(
    self: @ContractState, staking_rewards: Amount, total_stake: Amount,
) -> Index {
    // Return zero if the total stake is too small, to avoid overflow below.
    if total_stake < self.min_delegation_for_rewards.read() {
        return Zero::zero();
    }
    ...
}
```

The code comment directly above this function acknowledges the consequence:

> **Note**: Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case.

The reward flow is:
1. `_update_rewards` in `src/staking/staking.cairo` calls `claim_from_reward_supplier` to pull `total_pools_rewards` tokens from the reward supplier into the staking contract.
2. It then calls `update_pool_rewards`, which calls `send_rewards_to_delegation_pool`, which executes `token_dispatcher.checked_transfer(recipient: pool_address, amount: amount.into())` — physically moving STRK tokens into the pool contract.
3. Simultaneously, `update_rewards_from_staking_contract` is called on the pool, which calls `compute_rewards_per_unit`. If `pool_balance < min_delegation_for_rewards`, this returns 0, so the `cumulative_rewards_trace` receives a zero increment for that epoch.
4. Pool members calculate their claimable rewards as the difference in the cumulative trace between their entry checkpoint and the current checkpoint. A zero increment means zero claimable rewards for that epoch.
5. The STRK tokens that were physically transferred to the pool contract in step 2 are now permanently stranded — no pool member can ever claim them, and there is no admin recovery function in the pool contract.

For STRK pools, `min_for_rewards = 10^18` (1 STRK). The pool's `enter_delegation_pool` only requires `amount.is_non_zero()`, so any delegator can enter with as little as 1 wei of STRK, placing the pool balance far below the minimum threshold.

### Impact Explanation
STRK reward tokens transferred to the pool contract when `pool_balance < min_delegation_for_rewards` are permanently frozen. There is no `recoverWrongToken`, no admin withdrawal, and no mechanism to redistribute these tokens. The only token outflows from the pool contract are `exit_delegation_pool_action` (returns principal) and `claim_rewards` (returns tracked rewards). Since the cumulative trace was not incremented, the locked tokens are invisible to the reward accounting system and can never be claimed. This constitutes **permanent freezing of unclaimed yield**.

### Likelihood Explanation
The minimum delegation amount enforced by the pool is `amount.is_non_zero()` — any non-zero amount is accepted. For STRK pools, `min_for_rewards = 10^18` (1 STRK). A delegator who enters with any amount from 1 wei to `10^18 - 1` STRK will trigger this condition. This is a realistic scenario for new pools or pools where most delegators have exited, leaving a residual balance below the threshold. The condition is reachable by any unprivileged delegator without any special access.

### Recommendation
When `compute_rewards_per_unit` returns zero due to insufficient pool balance, the staking contract should not transfer the corresponding reward tokens to the pool contract, or the pool contract should track and allow recovery of such "dust" rewards. One approach: in `update_rewards_from_staking_contract`, if `compute_rewards_per_unit` would return zero, return the reward tokens to the reward supplier rather than leaving them stranded in the pool.

### Proof of Concept

1. Deploy the system. Create a staker with a STRK delegation pool.
2. A delegator calls `enter_delegation_pool` with `amount = 10^18 - 1` (just below `min_for_rewards`).
3. The sequencer calls `update_rewards` (or the attestation contract calls `update_rewards_from_attestation_contract`).
4. Inside `_update_rewards` → `update_pool_rewards` → `send_rewards_to_delegation_pool`: STRK tokens are transferred to the pool contract.
5. Inside `update_rewards_from_staking_contract` → `compute_rewards_per_unit`: returns 0 because `pool_balance < min_delegation_for_rewards`. The `cumulative_rewards_trace` is incremented by 0.
6. The delegator calls `claim_rewards` → receives 0.
7. Inspect `token.balance_of(pool_contract)` — it holds the reward tokens with no way to extract them.

        ///
        /// `staking_rewards` is in `STRK_DECIMALS` decimals.
        /// `total_stake` is in the contract's token decimals.
        /// **Note**: Delegation rewards lost when pool balance is less than
        /// `min_delegation_for_rewards`. The staking contract continues to forward
        /// `pool_rewards` to the pool contract even in this case.
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }

fn delegate_min_strk_for_rewards_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    let stake_amount = system.staking.get_min_stake();
    let staker = system.new_staker(amount: stake_amount);
    system.stake(:staker, amount: stake_amount, pool_enabled: true, commission: Zero::zero());
    let pool = system.staking.get_pool(:staker);

    // Setup delegator.
    let delegate_amount = Pool::STRK_CONFIG.min_for_rewards;
    let delegator = system.new_delegator(amount: delegate_amount);

    // Enter pool with less than min STRK for rewards.
    system.delegate(:delegator, :pool, amount: delegate_amount - 1);

    // Attest.
    system.advance_k_epochs_and_attest(:staker);
    system.advance_epoch();

    // Check rewards.
    system.advance_epoch();
    let rewards = system.delegator_claim_rewards(:delegator, :pool);
    assert!(rewards == Zero::zero());

    // Add to delegation.
    system.add_to_delegation_pool(:delegator, :pool, amount: 1);

    // Attest.
    system.advance_k_epochs_and_attest(:staker);

    // Check rewards.
    system.advance_epoch();
    let rewards = system.delegator_claim_rewards(:delegator, :pool);
    assert!(rewards > Zero::zero());
}
