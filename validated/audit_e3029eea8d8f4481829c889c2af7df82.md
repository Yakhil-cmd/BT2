### Title
Unbounded Loop Over `stakers` Vec in `get_stakers()` Causes Unbounded Gas Consumption - (File: src/staking/staking.cairo)

### Summary
The `get_stakers()` function in `staking.cairo` iterates over the entire `stakers` storage Vec without any bound. Every call to `stake()` appends a new address to this Vec, and stakers are never removed from it even after unstaking. An unprivileged attacker can register arbitrarily many staker addresses (each with the minimum stake) to grow the Vec, eventually causing `get_stakers()` to exceed the Starknet transaction gas limit.

### Finding Description

**Root cause — unbounded Vec growth:**

Every successful `stake()` call unconditionally appends the new staker address to `self.stakers`: [1](#0-0) 

There is no corresponding removal. A grep for `stakers.pop` / `stakers.remove` returns zero results; only two `stakers.push` sites exist in the entire codebase. Stakers that have fully unstaked remain in the Vec forever.

**Root cause — unbounded iteration:**

`get_stakers()` (part of the public `IStakingConsensus` interface) iterates over every element of the Vec on every call: [2](#0-1) 

For each element it performs multiple storage reads (`is_staker_active`, `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, `get_peer_id_at_epoch`), making the per-iteration cost non-trivial.

**Attacker-controlled entry path:**

1. Attacker creates N distinct EOA/contract addresses.
2. Each address calls `stake()` with the protocol minimum stake, paying only the minimum STRK amount.
3. Each call appends one entry to `self.stakers` (line 348).
4. After N registrations, any call to `get_stakers()` must iterate all N entries.
5. At a sufficient N the transaction gas limit is exceeded and `get_stakers()` reverts.

The `stakers` Vec is also iterated in a second location inside `staking.cairo` (the other `into_iter_full_range` call), compounding the surface. [3](#0-2) 

### Impact Explanation

`get_stakers()` is the primary on-chain entry point for the consensus layer to enumerate active validators for a given epoch. Permanently blocking it prevents the protocol from producing the validator set, halting attestation and block-reward distribution. This matches the **Medium** impact category: *Griefing with no profit motive but damage to users or protocol* / *Unbounded gas consumption*.

### Likelihood Explanation

The minimum stake is a financial barrier but not a privileged-role requirement. A well-funded attacker (or a coordinated group of small stakers) can register enough addresses over time to push the Vec past the gas limit. Because stakers are never pruned, the Vec only grows monotonically; the attack is permanent once executed.

### Recommendation

1. **Lazy deletion / tombstone flag**: When a staker fully unstakes, mark their slot as inactive in a separate mapping rather than leaving the address in the Vec. Skip tombstoned entries during iteration (already partially done via `is_staker_active`), and periodically compact the Vec.
2. **Paginated iteration**: Add a `(offset, limit)` parameter to `get_stakers()` so callers can page through results across multiple transactions.
3. **Separate active-staker index**: Maintain a second, compacted Vec of *currently active* stakers that is updated on stake/unstake, and iterate only that Vec in `get_stakers()`.

### Proof of Concept

```
// Pseudocode — repeat N times until get_stakers() reverts
for i in 0..N:
    staker_i = new_address()
    strk_token.approve(staking_contract, MIN_STAKE)
    staking_contract.stake(
        reward_address   = staker_i,
        operational_address = staker_i,
        amount           = MIN_STAKE,
        pool_enabled     = false,
        commission       = 0,
    )
    // staking.cairo:348 → self.stakers.push(staker_i)

// After N registrations:
staking_contract.get_stakers(epoch_id = current_epoch)
// → iterates all N entries → exceeds gas limit → reverts
```

Each `stake()` call is a valid, unprivileged protocol action. No privileged key, bridge compromise, or external dependency is required.

```text
        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
```
