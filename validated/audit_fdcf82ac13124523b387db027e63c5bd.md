### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Wrong Admission Decisions ŌĆö (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **previous block's** `l2_gas_price`, but every transaction it admits will be executed in the **next block** whose L2 gas price is `next_l2_gas_price` ŌĆö a distinct EIP-1559-adjusted value stored in `BlockHeaderWithoutHash`. When the two values diverge (the normal case whenever a block's gas usage differs from the target), the gateway either admits transactions that will fail at execution or rejects transactions that would succeed.

---

### Finding Description

In `validate_resource_bounds`, the gateway reads the threshold price as:

```rust
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;          // ŌåÉ price of block N, not block N+1
``` [1](#0-0) 

The developer-acknowledged TODO immediately above this read states:

```
// TODO(Arni): getnext_l2_gas_price from the block header.
``` [2](#0-1) 

The correct value, `next_l2_gas_price`, is already computed and stored in `BlockHeaderWithoutHash`:

```rust
pub next_l2_gas_price: GasPrice,
``` [3](#0-2) 

It is this `next_l2_gas_price` that the consensus orchestrator propagates into the block being built:

```rust
self.l2_gas_price = self.calculate_next_l2_gas_price(height, l2_gas_used);
``` [4](#0-3) 

The same stale price is also carried into `run_validate_entry_point`, where the `BlockContext` for blockifier pre-validation is constructed from the previous block's `BlockInfo` (gas prices unchanged, only `block_number` incremented):

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
``` [5](#0-4) 

So both the explicit threshold check and the blockifier's `check_fee_bounds` (called via `perform_pre_validation_stage`) run against block N's prices, while the batcher will execute the transaction against block N+1's prices.

The blockifier's `check_fee_bounds` for `AllResources` transactions compares:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    // MaxGasPriceTooLow
}
``` [6](#0-5) 

At the gateway this `actual_gas_price` is block N's price; at the batcher it is block N+1's price.

---

### Impact Explanation

**Case A ŌĆö price rises (block N above gas target ŌåÆ `next_l2_gas_price > l2_gas_price`):**
A user submits a V3 `AllResources` transaction with `max_l2_gas_price = l2_gas_price_N`. The gateway threshold check passes (`l2_gas_price_N Ōēź 100% ├Ś l2_gas_price_N`). The blockifier pre-validation at the gateway also passes (same stale price). The transaction is admitted to the mempool. When the batcher executes it in block N+1 at `next_l2_gas_price > l2_gas_price_N`, `check_fee_bounds` fires `MaxGasPriceTooLow` and the transaction is rejected at execution. **Gateway admits an invalid transaction.**

**Case B ŌĆö price falls (block N below gas target ŌåÆ `next_l2_gas_price < l2_gas_price`):**
A user submits a transaction with `max_l2_gas_price = next_l2_gas_price_N`. The gateway threshold check fails (`next_l2_gas_price_N < 100% ├Ś l2_gas_price_N`). The transaction is rejected at the gateway. The batcher would have accepted it at block N+1's lower price. **Gateway rejects a valid transaction.**

Both outcomes match the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The EIP-1559 `calculate_next_base_gas_price` adjusts the L2 gas price every block whenever gas usage differs from the target:

```rust
let adjusted_price_u256 =
    if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };
``` [7](#0-6) 

Any block with non-target gas usage ŌĆö the common case under real load ŌĆö produces `next_l2_gas_price ŌēĀ l2_gas_price`. No special attacker capability is required; any user submitting a V3 `AllResources` transaction during a period of rising or falling gas prices triggers the discrepancy.

---

### Recommendation

Replace the read of `l2_gas_price` with `next_l2_gas_price` from the block header. The `GatewayFixedBlockStateReader` interface (or the `BlockInfo` struct returned by `get_block_info`) should be extended to expose `next_l2_gas_price`. The same corrected value should be injected into the `BlockContext` built inside `run_validate_entry_point` so that blockifier pre-validation also uses the price the transaction will actually face.

---

### Proof of Concept

1. Block N executes with gas usage above the target. The EIP-1559 formula computes `next_l2_gas_price_N = P_next > P_current`.
2. Alice submits a V3 `AllResources` invoke transaction with `l2_gas.max_price_per_unit = P_current`.
3. `validate_resource_bounds` computes `threshold = 100% ├Ś P_current = P_current`. Check: `P_current Ōēź P_current` ŌåÆ **passes**.
4. `run_validate_entry_point` builds a `BlockContext` with `gas_prices.strk_gas_prices.l2_gas_price = P_current` (block N's price, block number = N+1). Blockifier `check_fee_bounds` checks `P_current Ōēź P_current` ŌåÆ **passes**. Transaction is admitted to the mempool.
5. The batcher builds block N+1 with `l2_gas_price = P_next`. Blockifier `check_fee_bounds` checks `P_current Ōēź P_next` ŌåÆ **fails** with `MaxGasPriceTooLow` / `InsufficientResourceBounds`.
6. The transaction was admitted by the gateway but is rejected at execution ŌĆö an invalid transaction was accepted through the admission path. [8](#0-7) [9](#0-8)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
