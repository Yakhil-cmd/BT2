### Title
Gateway L2 Gas Price Threshold Floor-Truncates the Minimum, Admitting Transactions Below the Intended Threshold — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price by multiplying `previous_block_l2_gas_price` by `min_gas_price_percentage / 100` and then calling `.to_integer()`, which **floors** the result. When the product is non-integer, the enforced threshold is strictly lower than the intended minimum, so a transaction whose `max_price_per_unit` equals the floored value is admitted even though it is below the configured percentage of the previous block price.

---

### Finding Description

The threshold computation is:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← floor truncation
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
``` [1](#0-0) 

`Ratio::to_integer()` truncates towards zero (floor for positive values). When `min_gas_price_percentage` does not evenly divide 100 (e.g., 50, 80, 90), the product `(min_gas_price_percentage / 100) × previous_block_l2_gas_price` is non-integer, and the floor is strictly less than the true fractional threshold.

The admission check is `tx_l2_gas_price.0 < threshold` (strict less-than), so a transaction with `tx_l2_gas_price = floor(...)` passes the check even though it is below the intended minimum of `ceil(...)`.

The config field is documented as:

```rust
// Minimum gas price as percentage of threshold to accept transactions.
pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
``` [2](#0-1) 

The semantic contract is that every admitted transaction must offer at least `min_gas_price_percentage / 100` of the previous block's L2 gas price. Floor truncation breaks this contract.

---

### Impact Explanation

The gateway stateful admission path accepts transactions whose L2 gas price is below the operator-configured minimum percentage of the previous block's L2 gas price. This breaks the invariant:

> *All admitted transactions satisfy `tx_l2_gas_price ≥ (min_gas_price_percentage / 100) × previous_block_l2_gas_price`.*

This fits the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions before sequencing."*

---

### Likelihood Explanation

The bug is triggered whenever:
1. `min_gas_price_percentage` is set to a value that does not evenly divide 100 (e.g., 50, 80, 90), **and**
2. `previous_block_l2_gas_price` is not a multiple of `100 / gcd(min_gas_price_percentage, 100)`.

The default value is 100, so the issue does not manifest with the default configuration. However, the config is operator-tunable, and any user can submit a transaction with a gas price exactly at the floored threshold to exploit the gap. No privileged access is required once the operator has set a non-100 percentage.

---

### Recommendation

Use ceiling division instead of floor truncation:

```rust
// Replace:
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();

// With:
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

`Ratio::ceil()` from `num_rational` returns the smallest integer ≥ the ratio, ensuring the enforced threshold is never below the intended minimum.

---

### Proof of Concept

**Setup:** `min_gas_price_percentage = 50`, `previous_block_l2_gas_price = 101`.

**Current behaviour (floor):**
```
threshold = floor(50/100 × 101) = floor(50.5) = 50
```
A transaction with `tx_l2_gas_price = 50` satisfies `50 < 50 == false` → **admitted**.

**Intended behaviour (ceil):**
```
threshold = ceil(50/100 × 101) = ceil(50.5) = 51
```
The same transaction satisfies `50 < 51 == true` → **rejected**.

The transaction is admitted with a gas price 1 unit below the intended minimum, breaking the admission invariant. [3](#0-2) [4](#0-3)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
