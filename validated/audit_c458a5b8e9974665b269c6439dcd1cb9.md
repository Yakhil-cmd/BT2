### Title
Missing EVM Contract Existence Check in `CadenceOwnedAccount.call()` Allows Silent Asset Loss to Non-Contract Addresses - (File: fvm/evm/stdlib/contract.cdc)

### Summary
`CadenceOwnedAccount.call()` and the underlying `Account.Call()` handler perform no check that the target EVM address has deployed code before executing the call with a FLOW value. Per standard EVM semantics, a call to a non-existent or self-destructed address succeeds silently, returning `StatusSuccessful` with empty return data. Any FLOW value attached to such a call is permanently transferred to an address with no code and no way to retrieve it.

### Finding Description
`CadenceOwnedAccount.call()` in `fvm/evm/stdlib/contract.cdc` delegates directly to `InternalEVM.call()` with no pre-condition verifying that the `to` address has code:

```cadence
access(Owner | Call)
fun call(
    to: EVMAddress,
    data: [UInt8],
    gasLimit: UInt64,
    value: Balance
): Result {
    pre {
        !EVM.isPaused(): "EVM operations are temporarily paused"
    }
    return InternalEVM.call(
        from: self.addressBytes,
        to: to.bytes,
        data: data,
        gasLimit: gasLimit,
        value: value.attoflow
    ) as! Result
}
``` [1](#0-0) 

The Go-layer handler `Account.Call()` also performs no existence check and, unlike `Transfer()` and `Deposit()`, does not call `panicOnErrorOrInvalidOrFailedState` — it only calls `panicOnError(err)`, meaning a successful-but-empty call to a codeless address is silently returned to the Cadence caller as `StatusSuccessful`:

```go
func (a *Account) Call(to types.Address, data types.Data, gaslimit types.GasLimit, balance types.Balance) *types.ResultSummary {
    res, err := a.executeAndHandleAuthorizedCall(
        types.NewContractCall(a.address, to, data, uint64(gaslimit), balance, a.Nonce()),
        nil, false,
    )
    panicOnError(err)          // ← only errors, not failed/invalid state
    return res.ResultSummary()
}
``` [2](#0-1) 

Contrast with `Transfer()` and `Deposit()`, which both call `panicOnErrorOrInvalidOrFailedState(res, err)` and would panic on a failed EVM result: [3](#0-2) [4](#0-3) 

The EVM state layer confirms that for a non-existent address `GetCode` returns `nil` and `GetCodeSize` returns `0`, but neither is consulted before executing the call: [5](#0-4) 

The `DirectCall` dispatch in the emulator routes `ContractCallSubType` through `proc.runDirect(call)` with no code-existence guard: [6](#0-5) 

### Impact Explanation
A Cadence transaction author who owns a COA and calls `coa.call(to: destroyedContractAddr, data: calldata, gasLimit: X, value: someBalance)` will:
1. Receive `Result.status == EVM.Status.successful` — no error is surfaced.
2. Have the FLOW value permanently transferred to an address with no code and no recovery mechanism.
3. Receive empty `data` in the result, which may be misinterpreted as a zero-return rather than a no-op.

This is a **cross-VM asset loss**: FLOW tokens bridged into the EVM environment via a COA are irretrievably sent to a dead address. The `EVM.Status.successful` return gives false confidence that the intended contract logic executed.

### Likelihood Explanation
Any unprivileged Cadence transaction author who holds a `CadenceOwnedAccount` resource can trigger this. The scenario is realistic: a COA owner integrates with an EVM DeFi protocol (bridge, DEX, vault), the protocol contract is later self-destructed or the address is misconfigured, and subsequent calls with value silently succeed while burning the attached FLOW. No special privileges, leaked keys, or node compromise are required — only a standard Cadence transaction with `auth(Call)` access to a COA. [7](#0-6) 

### Recommendation
Add a pre-condition in `CadenceOwnedAccount.call()` that verifies the target address has non-zero code size before executing a call that carries value:

```cadence
pre {
    !EVM.isPaused(): "EVM operations are temporarily paused"
    value.attoflow == 0 || InternalEVM.codeSize(to.bytes) > 0:
        "EVM.CadenceOwnedAccount.call(): Cannot send value to a non-contract address"
}
```

Alternatively, at the Go layer in `Account.Call()` (`fvm/evm/handler/handler.go`), check `blk.CodeOf(to)` before constructing the `DirectCall` when `balance > 0`. At minimum, document this EVM behavior prominently in the `CadenceOwnedAccount.call()` NatSpec so integrators are aware that a successful result does not guarantee code execution. [8](#0-7) 

### Proof of Concept
```cadence
import EVM from <EVMContractAddress>
import FlowToken from <FlowTokenAddress>

// Attacker or victim scenario:
// 1. A contract was previously deployed at `victimContractAddr` and later self-destructed.
// 2. The COA owner believes the contract is still live and calls it with value.

transaction(victimContractAddrBytes: [UInt8; 20]) {
    prepare(account: auth(Storage) &Account) {
        let coa = account.storage.borrow<auth(EVM.Call, EVM.Owner) &EVM.CadenceOwnedAccount>(
            from: /storage/evmCOA
        ) ?? panic("no COA")

        // Fund the COA first (omitted for brevity)

        let target = EVM.EVMAddress(bytes: victimContractAddrBytes)
        let value = EVM.Balance(attoflow: 1_000_000_000_000_000_000) // 1 FLOW

        // Call to self-destructed contract with value
        let res = coa.call(
            to: target,
            data: [0xde, 0xad, 0xbe, 0xef], // any calldata
            gasLimit: 100_000,
            value: value
        )

        // res.status == EVM.Status.successful  ← no error!
        // res.data == []                        ← empty, no code ran
        // 1 FLOW is now permanently at `target` with no code to recover it
        assert(res.status == EVM.Status.successful, message: "expected silent success")
    }
}
```

The call succeeds because the underlying EVM treats a call to a codeless address as a plain value transfer. The FLOW is now at an address with no code and no owner, constituting permanent loss. [1](#0-0) [2](#0-1)

// GetCode returns the code of an address
//
// for non-existent accounts or accounts without a code (e.g. EOAs) it returns nil
func (v *BaseView) GetCode(addr gethCommon.Address) ([]byte, error) {
	return v.getCode(addr)
}

// GetCodeHash returns the code hash of an address
//
// for non-existent accounts it returns gethCommon.Hash{}
// and for accounts without a code (e.g. EOAs) it returns default empty
// hash value (gethTypes.EmptyCodeHash)
func (v *BaseView) GetCodeHash(addr gethCommon.Address) (gethCommon.Hash, error) {
	acc, err := v.getAccount(addr)
	codeHash := gethCommon.Hash{}
	if acc != nil {
		codeHash = acc.CodeHash
	}
	return codeHash, err
}

// GetCodeSize returns the code size of an address
//
// for non-existent accounts or accounts without a code (e.g. EOAs) it returns zero
func (v *BaseView) GetCodeSize(addr gethCommon.Address) (int, error) {
	code, err := v.GetCode(addr)
	return len(code), err
}
