### Title
Commission Increase via Commitment Mechanism Retroactively Reduces Delegator Rewards — (`src/staking/staking.cairo`)

---

### Summary

The `set_commission_commitment` mechanism allows a staker to commit to a `max_commission` that is **higher** than the current commission. Once the commitment is active, `set_commission` permits the staker to immediately raise commission up to `max_commission`. Because commission is read at reward-distribution time (not at delegation time), delegators who joined at a lower commission rate have their unclaimed yield silently reduced. Delegators cannot exit before the change takes effect because `exit_delegation_pool_intent` enforces a mandatory `exit_wait_window` (default 1 week).

---

### Finding Description

**Root cause — commission is read at distribution time, not at delegation time.**

In `calculate_staker_pools_rewards`, the commission applied to every pool reward calculation is the **current** commission stored in `staker_pool_info`: [1](#0-0) 

There is no per-delegator or per-epoch record of the commission rate that was in effect when a delegator joined.

**The commitment mechanism enables commission increases.**

Without a commitment, `set_commission` enforces a strict decrease-only rule: [2](#0-1) 

However, when an active `commission_commitment` exists, the staker may set commission to **any value ≤ `max_commission`**, including values higher than the current commission: [3](#0-2) 

A staker can set a commitment with `max_commission = 10000` (100%) expiring just one epoch in the future (the minimum allowed), then immediately call `set_commission(10000)` in the next transaction.

**Delegators are locked in during the change.**

`exit_delegation_pool_intent` enforces a mandatory wait window before funds can be withdrawn: [4](#0-3) 

Delegators who observe the commitment on-chain and wish to exit cannot do so before the commission increase takes effect.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

A staker who starts at 0% commission attracts delegators, then raises commission to 100% via the commitment mechanism. The next call to `update_rewards_from_attestation_contract` (V2/pre-consensus) or `update_rewards` (V3/consensus) distributes pool rewards using the new 100% commission: [5](#0-4) 

All pool rewards flow to the staker as commission; delegators receive zero. Because `exit_wait_window` is at least one week, delegators cannot escape before losing an entire epoch's (or multiple blocks') worth of yield.

---

### Likelihood Explanation

**Medium.** The attack requires two on-chain transactions by the staker (`set_commission_commitment` then `set_commission`). Both are public and observable, but:

- The commitment can be set with `expiration_epoch = current_epoch + 1` (minimum allowed), giving only a one-epoch window that is still sufficient to capture rewards.
- The commission increase takes effect immediately with no delay, while delegators face a mandatory `exit_wait_window` of at least one week.
- Delegators monitoring the chain may not react in time, especially in the pre-consensus model where rewards are distributed once per epoch at attestation time. [6](#0-5) 

---

### Recommendation

1. **Enforce a delay on commission increases.** Commission increases via commitment should not take effect until at least `exit_wait_window` after the `set_commission` call, giving delegators time to exit.
2. **Alternatively, record the commission at the time of each reward distribution epoch** in the pool's `cumulative_rewards_trace` so that delegators who joined before a commission increase are credited at the rate that was in effect when they delegated.
3. **Emit a prominent on-chain warning event** when a commitment is set with `max_commission > current_commission`, so off-chain monitoring tools can alert delegators.

---

### Proof of Concept

```
Epoch N, block 0:
  Staker has commission = 0.
  Delegator A delegates 1_000_000 STRK.

Epoch N, block 1 (staker):
  staking.set_commission_commitment(max_commission: 10000, expiration_epoch: N+1)
  staking.set_commission(commission: 10000)   // now 100%

Epoch N, block 2 (attestation / update_rewards):
  calculate_staker_pools_rewards reads commission = 10000
  split_rewards_with_commission → commission_rewards = pool_rewards_including_commission
                                 pool_rewards = 0
  update_rewards_from_staking_contract(rewards: 0, pool_balance: ...)
  Delegator A accrues 0 rewards for the entire epoch.

Delegator A calls exit_delegation_pool_intent → must wait exit_wait_window (≥ 1 week).
During that week, every reward update continues to use commission = 10000.
``` [7](#0-6) [8](#0-7)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
