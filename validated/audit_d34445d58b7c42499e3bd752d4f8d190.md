### Title
Missing Predecessor-Equals-Receiver Authorization on `action_deploy_global_contract` (`AccountId` Mode) — (`runtime/runtime/src/global_contracts.rs`)

### Summary

`DeployGlobalContract` with `GlobalContractDeployMode::AccountId` registers contract code globally under the **receiver account's** ID, yet the runtime never enforces that the action's predecessor equals the receiver. Any unprivileged contract can therefore create a cross-contract call that overwrites another account's global contract entry, replacing the code executed by every account that references it via `GlobalContractIdentifier::AccountId(victim)`.

### Finding Description

The production runtime spec explicitly enumerates the actions that require `predecessor_id == receiver_id`:

> "For the following actions, `predecessor_id` and `receiver_id` are required to be equal: `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, `DeleteAccount`." [1](#0-0) 

`DeployGlobalContract` is absent from this list. The `check_actor_permissions` call in `apply_action` enforces the above list but has no branch for `DeployGlobalContract`. [2](#0-1) 

Inside `action_deploy_global_contract`, the identifier written to the global trie is derived solely from `account_id` (the receipt receiver), with no check that the predecessor is the same account: [3](#0-2) 

`initiate_distribution` then stamps the receiver's account ID as the global contract identifier: [4](#0-3) 

The nonce-freshness guard (`check_and_update_nonce`) only prevents *stale* re-delivery of the same distribution receipt; it does not prevent a new, higher-nonce deployment from overwriting an existing entry: [5](#0-4) 

The host function `promise_batch_action_deploy_global_contract_by_account_id` is available to any executing contract and appends a `DeployGlobalContract(AccountId)` action to an arbitrary outgoing receipt: [6](#0-5) 

### Impact Explanation

An attacker who can deploy and call a contract can:

1. Call `promise_batch_create("victim.near")` to create an outgoing receipt targeting the victim.
2. Call `promise_batch_action_deploy_global_contract_by_account_id(promise_idx, malicious_code)` to attach a `DeployGlobalContract(AccountId)` action to that receipt.
3. The receipt executes on the victim's shard; `action_deploy_global_contract` charges storage cost from the **victim's** balance and writes the attacker's code to `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }`.
4. Every account that previously called `UseGlobalContract(AccountId("victim.near"))` now executes the attacker's code on every subsequent function call — a complete code-identity substitution across all dependent accounts.

The corrupted value is the trie entry at `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim) }`. [7](#0-6) 

### Likelihood Explanation

The attack requires only the ability to deploy and call a contract — standard unprivileged user capability. No validator, admin, or operator privilege is needed. The `AccountId` deploy mode is explicitly documented as the mechanism for "the owner to update the contract for all its users," making the absence of an ownership check a direct invariant violation reachable in production. [8](#0-7) 

### Recommendation

Add a `predecessor_id == receiver_id` guard inside `action_deploy_global_contract` (or extend `check_actor_permissions` to cover `Action::DeployGlobalContract` when `deploy_mode == AccountId`), mirroring the existing guard applied to `DeployContract`:

```rust
// In action_deploy_global_contract or check_actor_permissions:
if matches!(deploy_contract.deploy_mode, GlobalContractDeployMode::AccountId)
    && predecessor_id != account_id
{
    return Err(ActionErrorKind::ActorNoPermission {
        account_id: account_id.clone(),
        actor_id: predecessor_id.clone(),
    }.into());
}
```

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
}
