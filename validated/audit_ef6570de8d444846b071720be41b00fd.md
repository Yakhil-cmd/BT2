### Title
Unprivileged Caller Can Permanently Suppress Consensus Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function is publicly callable with no access control on the `disable_rewards` parameter. When called with `disable_rewards: true`, it advances the global `last_reward_block` checkpoint without distributing any rewards. Because `last_reward_block` is a single global slot, one such call per block permanently marks that block as "processed," preventing any legitimate reward distribution call from succeeding in the same block. An attacker can repeat this every block at the cost of gas alone, causing permanent freezing of all future consensus-phase block rewards for every staker.

### Finding Description
`update_rewards` is exposed as a public ABI entry point in `IStakingRewardsManager`. Its only caller-side gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. The function accepts an arbitrary `disable_rewards: bool` from any caller.

The critical path is:

```cairo
// src/staking/staking.cairo  lines 1452-1507
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← global slot written

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits before distributing
    }
    ...
}
```

`last_reward_block` is a single global storage slot:

```cairo
// src/staking/staking.cairo  line 187
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
```

Once written to `current_block_number`, every subsequent call in the same block fails at the `REWARDS_ALREADY_UPDATED` assertion. The attacker only needs to supply any currently-active staker address (the `stakers` Vec is public) and pass the non-zero balance check. No privileged role, no token ownership, and no special relationship to the staker is required.

### Impact Explanation
In the consensus-rewards phase, block rewards are the sole mechanism by which stakers and delegators accrue yield. If `last_reward_block` is advanced without distributing rewards, those block rewards are permanently lost — they are never credited to `unclaimed_rewards_own` or forwarded to pool contracts. Repeating the call every block causes **permanent freezing of all future unclaimed yield** for every staker and every pool member in the protocol. This matches the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

### Likelihood Explanation
The attack requires:
- No privileged role or token balance.
- Knowledge of any one active staker address (trivially obtained from the public `stakers` Vec or on-chain events).
- One transaction per block.

On Starknet, transaction fees are low. A sustained griefing campaign is economically feasible for any motivated adversary (e.g., a competing validator set, a short-seller of STRK, or a protocol antagonist). The entry path is entirely unprivileged and reachable from any EOA.

### Recommendation
Restrict who may pass `disable_rewards: true`. The simplest fix is to require the caller to be the staker themselves or their registered operational/reward address when `disable_rewards` is `false`, and to gate `disable_rewards: true` behind a privileged role (e.g., `security_agent` or `app_governor`). Alternatively, remove the `disable_rewards` parameter entirely and handle the pre-consensus early-return via the existing `is_pre_consensus()` check, which already covers that case.

### Proof of Concept

```
// Attacker script (pseudocode, one call per block)
let staker = staking.stakers[0];          // any active staker from public Vec
loop every block:
    staking.update_rewards(staker, disable_rewards: true);
    // last_reward_block = current_block, no rewards distributed
    // All legitimate update_rewards calls in this block now revert with REWARDS_ALREADY_UPDATED
```

Step-by-step:
1. Attacker reads any valid staker address from the public `stakers` storage vector.
2. At the start of each block, attacker calls `update_rewards(staker_address, disable_rewards: true)`.
3. `last_reward_block` is written to the current block number (line 1485).
4. The function returns early at line 1487 without entering `_update_rewards`.
5. Any subsequent call to `update_rewards` in the same block reverts at line 1455–1458 (`REWARDS_ALREADY_UPDATED`).
6. Block rewards for that block are never minted or credited to any staker or pool.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
