### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards by Griefing `update_rewards` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract writes to the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. Because the function has no caller-based access control, any unprivileged address can call it every block with `disable_rewards: true`, consuming the per-block reward slot without distributing any rewards. This permanently freezes unclaimed consensus yield for all stakers as long as the attacker keeps calling the function.

---

### Finding Description

`update_rewards` is a public function in `IStakingRewardsManager`. Its only gate is `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role check, no staker-identity check on the caller. [1](#0-0) 

The function enforces a one-call-per-block invariant via the global `last_reward_block`: [2](#0-1) 

Critically, `last_reward_block` is written **before** the `disable_rewards` branch: [3](#0-2) 

When `disable_rewards: true`, the function returns immediately after updating `last_reward_block`, distributing nothing. Because the slot is now consumed, every other staker's call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.

The `general_prerequisites` implementation confirms there is no role or identity restriction: [4](#0-3) 

The `last_reward_block` storage field is declared as a single global value shared across all stakers: [5](#0-4) 

---

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker_address, disable_rewards: true)` on every block:

1. Marks the block as "already rewarded" (`last_reward_block = current_block`).
2. Distributes zero rewards to anyone.
3. Causes every legitimate staker's `update_rewards` call in that block to revert.

Because consensus rewards are distributed exclusively through this function, sustained griefing permanently freezes all unclaimed consensus yield for every staker and delegator in the protocol. The attacker does not need to be a staker — they only need to supply a valid, active `staker_address` (publicly readable on-chain) and pay gas per block.

**Impact: High — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

- No stake required; only gas per block.
- Starknet L2 gas costs are low, making sustained per-block calls economically viable.
- The target staker address is trivially obtained from on-chain events (`NewStaker`).
- No privileged access, no external dependency, no phishing required.

**Likelihood: Medium.**

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the block producer's operational address, or the attestation contract), or move the `last_reward_block` write to **after** the `disable_rewards` guard so that a no-op call does not consume the block's reward slot:

```cairo
// Only consume the block slot when rewards are actually being processed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // moved here
// ... distribute rewards ...
```

Alternatively, add an access-control assertion (e.g., `only_app_governor` or a dedicated `REWARDS_UPDATER` role) before the `last_reward_block` write.

---

### Proof of Concept

1. Observe any active staker address `S` from on-chain `NewStaker` events.
2. Every block, attacker (any EOA) calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `last_reward_block` is set to the current block; no rewards are distributed.
4. Any legitimate staker calling `update_rewards` in the same block receives `REWARDS_ALREADY_UPDATED` and earns nothing.
5. Repeated indefinitely, all consensus-phase staking rewards are frozen. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
