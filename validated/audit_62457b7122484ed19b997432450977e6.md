### Title
Pool Rewards Permanently Frozen When Total Delegation Falls Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo)

### Summary
When a pool's total delegated balance is below `min_delegation_for_rewards`, the staking contract still calculates, claims from the reward supplier, and transfers STRK reward tokens to the pool contract. However, the pool's `compute_rewards_per_unit` silently returns zero, leaving the sigma (`cumulative_rewards_trace`) unchanged. The transferred STRK tokens are permanently locked in the pool contract with no recovery path.

### Finding Description
The reward distribution flow in `_update_rewards` (staking.cairo) unconditionally:
1. Calculates non-zero `pool_rewards` whenever `pool_balance_curr_epoch` is non-zero.
2. Calls `claim_from_reward_supplier` to pull those STRK tokens into the staking contract.
3. Calls `send_rewards_to_delegation_pool` to push them into the pool contract.
4. Calls `pool.update_rewards_from_staking_contract(rewards, pool_balance)`.

Inside `update_rewards_from_staking_contract` (pool.cairo lines 571–587), the pool appends to `cumulative_rewards_trace` using `compute_rewards_per_unit`. That function (pool.cairo lines 967–978) returns `Zero::zero()` whenever `total_stake < min_delegation_for_rewards`:

```rust
fn compute_rewards_per_unit(
    self: @ContractState, staking_rewards: Amount, total_stake: Amount,
) -> Index {
    // Return zero if the total stake is too small, to avoid overflow below.
    if total_stake < self.min_delegation_for_rewards.read() {
        return Zero::zero();
    }
    ...
}
```

The code's own comment at lines 964–966 acknowledges the consequence:

> **Note**: Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case.

The sigma is appended as `last + 0`, so no member can ever claim those rewards. There is no rescue function in the pool contract; `claim_rewards` is the only STRK egress path and it depends entirely on a non-zero sigma delta.

Thresholds:
- STRK pool: `min_for_rewards = 10^18` (1 STRK) — `STRK_CONFIG` at pool.cairo line 63.
- BTC pool (8 decimals): `min_for_rewards = 10^3` (0.001 BTC) — `get_token_rewards_config` at pool/utils.cairo line 135.

### Impact Explanation
STRK reward tokens transferred to the pool contract during epochs where `pool_balance < min_delegation_for_rewards` are permanently unclaimable. This is **permanent freezing of unclaimed yield**, matching the allowed High impact category. The frozen amount accumulates every epoch the pool remains below the threshold.

### Likelihood Explanation
Any pool that starts with a small total delegation (below 1 STRK for STRK pools, or below 0.001 BTC for BTC pools) will silently lose all rewards for every epoch until the threshold is crossed. New pools, pools with a single small delegator, or pools whose delegators partially exit below the threshold are all realistic scenarios. No privileged access is required; any unprivileged delegator can create or maintain this condition.

### Recommendation
Mirror the Tap bug fix: before transferring rewards to the pool contract, check whether `pool_balance >= min_delegation_for_rewards`. If not, either:
- Skip the transfer entirely for that epoch and do not update `unclaimed_rewards` in the reward supplier for that pool's share, or
- Accumulate the unclaimable rewards in a separate storage variable and allow them to be applied in a future epoch once the pool balance crosses the threshold.

The staking contract should not transfer tokens it knows the pool cannot distribute.

### Proof of Concept

1. Staker opens a STRK delegation pool.
2. Delegator enters with `amount = STRK_CONFIG.min_for_rewards - 1` (i.e., `10^18 - 1` FRI). Total pool balance is now below threshold.
3. Staker attests / `update_rewards` is called. `_update_rewards` calculates non-zero `pool_rewards`, pulls STRK from the reward supplier, and transfers it to the pool contract via `send_rewards_to_delegation_pool`.
4. `update_rewards_from_staking_contract(rewards, pool_balance)` is called on the pool. `compute_rewards_per_unit` returns 0 because `pool_balance < min_delegation_for_rewards`. The `cumulative_rewards_trace` entry is `last + 0`.
5. Delegator calls `claim_rewards`. `calculate_rewards` computes `amount * (sigma_to - sigma_from) / base_value = amount * 0 = 0`. Zero rewards returned.
6. The STRK tokens transferred in step 3 remain in the pool contract forever with no egress path.

The existing flow test `delegate_min_strk_for_rewards_flow_test` (src/flow_test/test.cairo lines 1713–1746) already demonstrates that rewards are zero below the threshold, but does not assert that the transferred STRK tokens are unrecoverable. pub(crate) fn get_token_rewards_config(token_address: ContractAddress) -> TokenRewardsConfig {
    if token_address == STRK_TOKEN_ADDRESS {
        STRK_CONFIG
    } else {
        // BTC token.
        let token_dispatcher = IERC20MetadataDispatcher { contract_address: token_address };
        let decimals = token_dispatcher.decimals();
        assert!(decimals >= 5 && decimals <= 18, "{}", GenericError::INVALID_TOKEN_DECIMALS);
        TokenRewardsConfig {
            decimals,
            min_for_rewards: 10_u128.pow(decimals.into() - 5),
            base_value: 10_u128.pow(decimals.into() + 5),
        }
    }

fn delegate_min_strk_for_rewards_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    let stake_amount = system.staking.get_min_stake();
    let staker = system.new_staker(amount: stake_amount);
    system.stake(:staker, amount: stake_amount, pool_enabled: true, commission: Zero::zero());
    let pool = system.staking.get_pool(:staker);

    // Setup delegator.
    let delegate_amount = Pool::STRK_CONFIG.min_for_rewards;
    let delegator = system.new_delegator(amount: delegate_amount);

    // Enter pool with less than min STRK for rewards.
    system.delegate(:delegator, :pool, amount: delegate_amount - 1);

    // Attest.
    system.advance_k_epochs_and_attest(:staker);
    system.advance_epoch();

    // Check rewards.
    system.advance_epoch();
    let rewards = system.delegator_claim_rewards(:delegator, :pool);
    assert!(rewards == Zero::zero());

    // Add to delegation.
    system.add_to_delegation_pool(:delegator, :pool, amount: 1);

    // Attest.
    system.advance_k_epochs_and_attest(:staker);

    // Check rewards.
    system.advance_epoch();
    let rewards = system.delegator_claim_rewards(:delegator, :pool);
    assert!(rewards > Zero::zero());
