### Title
Gateway L2 Gas Price Validation Uses Stale Previous-Block Price Instead of `next_l2_gas_price`, Admitting Transactions That Fail Blockifier Execution - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` checks a transaction's `max_price_per_unit` against the **current committed block's** `l2_gas_price`, but the transaction will actually execute in the **next** block whose L2 gas price is `previous_block.next_l2_gas_price` (the EIP-1559-adjusted value stored in the block header). When the EIP-1559 mechanism raises the price between blocks, the gateway admits transactions whose `max_price_per_unit` is below the next block's actual price. Those transactions pass gateway admission, enter the mempool, and are then rejected by the blockifier's `check_fee_bounds` during `perform_pre_validation_stage`, silently dropping them from the block.

---

### Finding Description

In `validate_resource_bounds` the gateway reads the reference price as:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

This is the L2 gas price **used inside** the latest committed block (block N). The block header also carries `next_l2_gas_price`, the EIP-1559-computed price for block N+1: [2](#0-1) 

`next_l2_gas_price` is computed by `calculate_next_base_gas_price` / `calculate_next_l2_gas_price_for_fin` and can be up to `price / gas_price_max_change_denominator` higher than the current price when the block is full: [3](#0-2) 

The same stale price is propagated into `run_validate_entry_point`, which builds the block context by copying the previous block's `BlockInfo` and only incrementing the block number, leaving gas prices unchanged: [4](#0-3) 

During actual execution the batcher builds a block context with `next_l2_gas_price` as the L2 gas price. The blockifier's `check_fee_bounds` inside `perform_pre_validation_stage` then compares the transaction's `max_price_per_unit` against that higher price: [5](#0-4) [6](#0-5) 

A transaction with `max_price_per_unit` in the range `[block_N.l2_gas_price × threshold%, block_N.next_l2_gas_price)` satisfies the gateway check but fails `check_fee_bounds` at execution time, causing the blockifier to return `TransactionPreValidationError::TransactionFeeError(InsufficientResourceBounds)` and the transaction to be silently excluded from the block.

The TODO comment in the source code explicitly acknowledges the correct fix: read `next_l2_gas_price` from the block header instead.

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

Any user who submits a V3 (`AllResources`) transaction with `l2_gas.max_price_per_unit` equal to the current block's L2 gas price (the natural value a wallet would use) will have their transaction accepted by the gateway and mempool, but silently dropped by the batcher whenever the EIP-1559 mechanism has raised the price for the next block. The user receives a successful gateway response but their transaction is never included in a block. This breaks the admission invariant: the gateway should only accept transactions that can pass blockifier pre-validation.

---

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price every block based on gas usage. When blocks are consistently above the gas target, `next_l2_gas_price` is strictly greater than `l2_gas_price` every block. Under normal load this condition is common. Any wallet or SDK that sets `max_price_per_unit = current_block.l2_gas_price` (the minimum accepted value per the gateway's own threshold) will trigger this path. No special privileges or timing are required; any unprivileged user submitting a V3 transaction during a period of above-target block utilization is affected.

---

### Recommendation

Replace the stale `l2_gas_price` read with `next_l2_gas_price` from the block header in `validate_resource_bounds`:

```rust
// Before (wrong):
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// After (correct):
let block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
let next_l2_gas_price = block_info.block_header_without_hash.next_l2_gas_price;
// use next_l2_gas_price as the threshold reference
```

Apply the same fix in `run_validate_entry_point`: when constructing the `BlockInfo` for the gateway's block context, replace `block_info.gas_prices.strk_gas_prices.l2_gas_price` with `next_l2_gas_price` so that `check_fee_bounds` inside the blockifier uses the same price the batcher will use. [7](#0-6) 

---

### Proof of Concept

1. Observe the current committed block N has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block N was above the gas target).
2. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = P` (exactly the current block price, which satisfies `validate_resource_bounds` with `min_gas_price_percentage = 100`).
3. Gateway calls `validate_resource_bounds`: `P >= P * 100% = P` → **passes**.
4. Gateway calls `run_validate_entry_point` with a block context whose `l2_gas_price = P` (stale): `check_fee_bounds` sees `max_price P >= actual_gas_price P` → **passes**.
5. Transaction is admitted to the mempool.
6. Batcher builds block N+1 with `l2_gas_price = P'`. Blockifier calls `perform_pre_validation_stage` → `check_fee_bounds` sees `max_price P < actual_gas_price P'` → **`InsufficientResourceBounds` error** → transaction is excluded from the block.
7. User's transaction is silently dropped despite receiving a successful gateway response. [8](#0-7) [9](#0-8)

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
