### Title
Pool Member's Remaining Delegated Balance Temporarily Frozen When Staker Unstakes With Active Partial Exit Intent - (File: `src/pool/pool.cairo`)

### Summary

When a pool member has an active partial exit intent and the staker subsequently calls `unstake_action`, the pool member's remaining delegated balance (transferred to the pool contract by `transfer_to_pools_when_unstake`) becomes temporarily inaccessible. The guard in `undelegate_from_staking_contract_intent` blocks all intent updates when the staker is inactive and an intent is already in progress — including the legitimate case where the pool member simply wants to exit their remaining balance. This forces the pool member to wait an additional full `exit_wait_window` period (up to 12 weeks) before they can access their remaining funds.

### Finding Description

The root cause is in `undelegate_from_staking_contract_intent` in `src/pool/pool.cairo`:

```cairo
fn undelegate_from_staking_contract_intent(
    self: @ContractState, pool_member: ContractAddress, amount: Amount,
) -> Timestamp {
    if !self.is_staker_active() {
        assert!(
            self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
            "{}",
            Error::UNDELEGATE_IN_PROGRESS,
        );
        return Time::now();
    }
    ...
}

This check is called from `exit_delegation_pool_intent`: [2](#0-1) 

**Attack path:**

1. Pool member has 100 tokens delegated (`delegated_stake = 100`, `unpool_amount = 0`).
2. Pool member calls `exit_delegation_pool_intent(amount=50)`:
   - `unpool_amount = 50`, `unpool_time = Some(T)`, `new_delegated_stake = 50`.
   - The staking contract records a `pool_exit_intents` entry for 50 tokens.
3. Staker calls `unstake_intent()` then `unstake_action()`:
   - `transfer_to_pools_when_unstake` transfers the remaining delegated balance (50 tokens) to the pool contract.
   - `set_staker_removed()` is called on the pool → `staker_removed = true`. [3](#0-2) 

4. Pool member now has:
   - 50 tokens in `pool_exit_intents` (staking contract) — retrievable after `T`.
   - 50 tokens sitting in the pool contract — **inaccessible**.
5. Pool member calls `exit_delegation_pool_intent(amount=50)` to exit the remaining balance:
   - Calls `undelegate_from_staking_contract_intent(pool_member, 50)`.
   - `!is_staker_active()` is `true` AND `unpool_time.is_some()` → **REVERTS with `UNDELEGATE_IN_PROGRESS`**.

The pool member cannot update or cancel their intent while the staker is inactive and an intent is in progress. The 50 tokens in the pool contract are frozen until the pool member:
1. Waits for `T` to pass.
2. Calls `exit_delegation_pool_action` to retrieve the first 50 tokens.
3. Calls `exit_delegation_pool_intent(50)` again (now succeeds since `unpool_time = None`).
4. Calls `exit_delegation_pool_action` immediately (staker inactive → `unpool_time = Time::now()`).

The maximum forced delay is the full `exit_wait_window`, which can be up to 12 weeks (`MAX_EXIT_WAIT_WINDOW`). [4](#0-3) 

### Impact Explanation

A pool member's remaining delegated balance — already transferred to the pool contract and therefore no longer earning rewards — is temporarily inaccessible for up to 12 weeks. The pool member is forced to undergo two full exit cycles instead of one, solely because the staker independently chose to unstake. This matches the **Temporary freezing of funds** impact in the allowed scope.

### Likelihood Explanation

This scenario is realistic and requires no privileged access:
- Pool members routinely create partial exit intents (e.g., to partially rebalance).
- Stakers independently decide when to unstake; they are not required to notify pool members.
- The two events (partial pool member intent + staker unstake) can occur in any order and are entirely independent.

No attacker is needed — this is triggered by normal user behavior.

### Recommendation

In `undelegate_from_staking_contract_intent`, when the staker is inactive and the pool member already has an active intent, allow the pool member to update their intent to include the remaining delegated balance (i.e., allow `amount >= pool_member_info.unpool_amount`). The problematic double-intent scenario described in the comment is only dangerous when the staking contract still holds the delegated balance; after `unstake_action`, the remaining balance has already been transferred to the pool contract, so there is no accounting inconsistency to prevent.

Alternatively, expose a dedicated function on the pool contract that allows a pool member to exit their full remaining balance in one step when the staker is already removed.

### Proof of Concept

```
1. Pool member delegates 100 tokens.
2. Pool member calls exit_delegation_pool_intent(50).
   → unpool_amount=50, unpool_time=Some(T), delegated_stake=50.
3. Staker calls unstake_intent() + unstake_action().
   → 50 tokens transferred to pool contract; staker_removed=true.
4. Pool member calls exit_delegation_pool_intent(50).
   → REVERTS: UNDELEGATE_IN_PROGRESS
   (staker inactive + unpool_time.is_some())
5. Pool member must wait until T, then:
   a. exit_delegation_pool_action() → receives 50 tokens.
   b. exit_delegation_pool_intent(50) → succeeds (unpool_time=None now).
   c. exit_delegation_pool_action() → receives remaining 50 tokens.
   Total extra delay: up to exit_wait_window (≤12 weeks).

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }

        fn transfer_to_pools_when_unstake(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
        ) {
            for (pool_contract, token_address) in staker_pool_info.pools {
                let pool_balance = self.get_delegated_balance(:staker_address, :pool_contract);
                let token_dispatcher = IERC20Dispatcher { contract_address: token_address };
                let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };

                self

                        :staker_address, :pool_contract, delegated_balance: Zero::zero(),
                    );
                let decimals = self.get_token_decimals(:token_address);
                token_dispatcher
                    .checked_transfer(
                        recipient: pool_contract,
                        amount: pool_balance.to_native_amount(:decimals).into(),
                    );
            }
