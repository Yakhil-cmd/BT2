### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

`update_rewards` is documented as "Only starkware sequencer" but the implementation enforces no such restriction. Any non-zero address can call it. Because `last_reward_block` is a **global** storage variable, a single call with `disable_rewards: true` permanently discards that block's rewards for the entire protocol.

---

### Finding Description

The spec at `docs/spec.md:1645` states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo:1449` enforces no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks: not paused, caller != zero
    ...
    self.last_reward_block.write(current_block_number);  // global write
    if disable_rewards || self.is_pre_consensus() {
        return;   // exits without distributing rewards
    }
    ...
}
``` [1](#0-0) 

`general_prerequisites` only checks pause state and non-zero caller — no role or sequencer check. [2](#0-1) 

`last_reward_block` is a single global `BlockNumber` in contract storage, not per-staker: [3](#0-2) 

The gate `current_block_number > self.last_reward_block.read()` is checked once per block globally. Once it is written, no further `update_rewards` call can succeed for that block number. [4](#0-3) 

---

### Impact Explanation

An attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)` in block N:

1. `last_reward_block` is written to N.
2. The function returns early — zero rewards are distributed.
3. Every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`.
4. Block N's rewards are **permanently and irrecoverably lost** for all stakers.

The attacker can repeat this every block, continuously suppressing all consensus block rewards across the entire protocol. There is no recovery path: `last_reward_block` is never decremented, and there is no "re-distribute missed block" mechanism.

Impact: **High — Permanent freezing of unclaimed yield** (matches allowed scope).

---

### Likelihood Explanation

- Requires only a non-zero EOA and gas. No stake, no role, no special setup.
- The call is cheap (no token transfers, no complex state).
- Griefing is economically rational for a competitor or malicious actor.
- Starknet is a public network; the function is callable by anyone.

---

### Recommendation

Add a sequencer/role check at the top of `update_rewards`, consistent with the spec. For example, using the existing roles component:

```cairo
fn update_rewards(...) {
    self.roles.only_sequencer(); // enforce spec: "Only starkware sequencer"
    self.general_prerequisites();
    ...
}
```

Alternatively, restrict via `assert_caller_is_attestation_contract()` or a dedicated sequencer role, matching the pattern already used in `update_rewards_from_attestation_contract`. [5](#0-4) 

---

### Proof of Concept

```
Block N:
  Attacker calls: staking.update_rewards(staker=<any_valid_staker>, disable_rewards=true)
    → general_prerequisites() passes (not paused, caller != 0)
    → current_block_number (N) > last_reward_block (N-1) → passes
    → last_reward_block written to N
    → disable_rewards == true → early return, no rewards distributed

  Legitimate sequencer calls: staking.update_rewards(staker=<intended_staker>, disable_rewards=false)
    → current_block_number (N) > last_reward_block (N) → FALSE → REWARDS_ALREADY_UPDATED revert

Block N rewards: permanently lost.
Repeat every block → all consensus rewards suppressed indefinitely.
``` [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
