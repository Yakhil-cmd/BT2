### Title
Unrestricted `update_rewards` with `disable_rewards: true` Allows Griefing of Block Reward Distribution — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control beyond the general prerequisites. It accepts a `disable_rewards` boolean parameter. When any caller invokes it with `disable_rewards: true`, the global `last_reward_block` is advanced to the current block **without distributing any rewards**. Because the function enforces a strict "once per block" invariant, any subsequent legitimate call for that block reverts with `REWARDS_ALREADY_UPDATED`. A malicious actor can front-run every legitimate `update_rewards` call to permanently suppress consensus block rewards for all stakers.

---

### Finding Description

`update_rewards` is exposed as a public entrypoint under `IStakingRewardsManager`: [1](#0-0) 

The only gate is `general_prerequisites()`, which checks the contract is unpaused and the caller is non-zero: [2](#0-1) 

Inside `update_rewards`, the global `last_reward_block` is written **before** the `disable_rewards` branch: [3](#0-2) 

The subsequent guard that prevents a second call in the same block reads this value: [4](#0-3) 

Because `last_reward_block` is a single global slot, a call with `disable_rewards: true` by **any** address consumes the per-block slot for **all** stakers. No rewards are computed or credited, yet the slot is permanently spent for that block.

---

### Impact Explanation

A malicious actor who front-runs every legitimate `update_rewards` call with `disable_rewards: true` causes:

- **Per-block**: The staker whose address was supplied receives zero block rewards for that block; the `_update_rewards` path is never reached.
- **Sustained attack**: If every block is front-run, all stakers are permanently denied consensus block rewards — their `unclaimed_rewards_own` is never incremented, constituting permanent freezing of unclaimed yield.
- **No funds at rest are moved**, but yield that should accrue is silently destroyed.

This maps to **High: Permanent freezing of unclaimed yield** (sustained) or **Medium: Griefing** (sporadic).

---

### Likelihood Explanation

- `update_rewards` must be called once per block by an off-chain keeper or the staker themselves; it is not automatic.
- On Starknet, transaction ordering within a block is deterministic and observable; a well-resourced attacker can submit a griefing transaction at the start of every block.
- The attacker bears only gas costs and gains nothing financially, making this a pure griefing vector. Likelihood is **Medium** for sustained attack, **High** for occasional disruption.

---

### Recommendation

1. **Restrict `disable_rewards: true` to privileged callers** (e.g., `security_agent` or `app_governor` role) so that only authorized actors can suppress reward distribution.
2. Alternatively, **remove the `disable_rewards` parameter** from the public interface and handle the disable case through a separate privileged function or an internal flag set by governance.
3. If `disable_rewards` must remain public, **move the `last_reward_block` write to after the `disable_rewards` check**, so that a suppressed call does not consume the per-block slot.

---

### Proof of Concept

```
Block N begins.

1. Legitimate keeper (or staker) prepares:
       update_rewards(staker_A, disable_rewards: false)

2. Attacker observes the pending transaction and submits first:
       update_rewards(staker_A, disable_rewards: true)
   → last_reward_block is written to N
   → function returns early; no rewards distributed

3. Keeper's transaction executes:
   → assert!(current_block_number > last_reward_block)
   → current_block_number == N == last_reward_block  → PANIC: REWARDS_ALREADY_UPDATED

4. staker_A.unclaimed_rewards_own is unchanged for block N.

5. Repeat for every block → staker_A (and any staker) receives zero consensus rewards indefinitely.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
