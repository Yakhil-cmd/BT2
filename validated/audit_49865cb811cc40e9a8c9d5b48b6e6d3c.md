### Title
Missing Caller Validation in `update_rewards` Allows Any Address to Permanently Block Reward Distribution - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `IStakingRewardsManager` is specified to be callable only by the Starkware sequencer, but the implementation contains no caller validation. Any unprivileged address can call this function with `disable_rewards: true` to consume the global per-block reward slot, permanently preventing the legitimate sequencer from distributing rewards to all stakers and delegators.

---

### Finding Description

The `update_rewards` function implements the `IStakingRewardsManager` interface. The protocol specification explicitly states its access control:

> **"Only starkware sequencer."** [1](#0-0) 

However, the implementation at `src/staking/staking.cairo` contains no such check. The function only verifies:
1. The contract is unpaused (`general_prerequisites()`)
2. The current block number is greater than `last_reward_block` (REWARDS_ALREADY_UPDATED)
3. The staker exists and is active
4. The staker has a non-zero balance [2](#0-1) 

The critical state mutation is at line 1485: `self.last_reward_block.write(current_block_number)`. This is a **single global storage variable** (not per-staker). Once written, no further `update_rewards` call can succeed in the same block for **any staker**, because the `REWARDS_ALREADY_UPDATED` assertion fires: [3](#0-2) 

If called with `disable_rewards: true`, the function returns early after writing `last_reward_block`, distributing zero rewards: [4](#0-3) 

The interface definition confirms this function is publicly exposed with no access restriction in the code: [5](#0-4) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker calling `update_rewards(any_valid_staker, disable_rewards: true)` once per block consumes the global reward slot. The legitimate sequencer's subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. Since this can be repeated every block at negligible cost, **all stakers and delegators are permanently denied their earned consensus-phase rewards**. The `unclaimed_rewards_own` field of every staker and the pool reward traces are never updated, freezing all yield in perpetuity.

---

### Likelihood Explanation

**High.** The attack requires:
- No special role or privileged key
- No capital at risk
- Only a valid (non-exiting, non-zero-balance) staker address to pass the `INVALID_STAKER` check — any publicly visible staker address from on-chain events suffices
- One transaction per block, automatable with a simple bot

The attacker has no profit motive but causes severe, irreversible damage to all protocol participants.

---

### Recommendation

Add a caller check at the top of `update_rewards` to enforce that only the authorized Starkware sequencer address can invoke it, consistent with the specification. For example, store the sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` before any other logic.

---

### Proof of Concept

1. Attacker identifies any valid, active staker address `S` (e.g., from a `NewStaker` event).
2. At the start of every new block, attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. This passes all checks (contract unpaused, block is new, staker is active and has balance) and writes `last_reward_block = current_block`.
4. When the legitimate Starkware sequencer attempts `update_rewards(staker_address: X, disable_rewards: false)` in the same block, the assertion `current_block_number > self.last_reward_block.read()` fails with `REWARDS_ALREADY_UPDATED`.
5. No rewards are distributed for that block for any staker.
6. Repeated every block, this permanently prevents all consensus-phase reward distribution across the entire protocol.

The root cause — the absence of a sequencer-only guard — is entirely within `src/staking/staking.cairo` in the `StakingRewardsManagerImpl::update_rewards` function. [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
