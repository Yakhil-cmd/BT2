### Title
Unprivileged Caller Can Freeze All Consensus Block Rewards via `update_rewards` with `disable_rewards: true` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract has no meaningful access control and accepts a `disable_rewards` boolean parameter. Any unprivileged caller can invoke `update_rewards(any_staker, disable_rewards: true)` to consume the global `last_reward_block` slot for the current block without distributing any rewards, permanently denying consensus block rewards to all stakers for that block. Repeated every block, this freezes all consensus-phase yield.

---

### Finding Description

`update_rewards` is exposed via `IStakingRewardsManager` and is gated only by `general_prerequisites()`: [1](#0-0) 

`general_prerequisites` only asserts the contract is not paused and the caller is non-zero — no role check, no staker-ownership check, no block-producer check. [2](#0-1) 

The critical sequence inside `update_rewards` is:

1. Assert `current_block_number > last_reward_block` (global, single slot).
2. **Write `last_reward_block = current_block_number`** — before the early-return guard.
3. If `disable_rewards == true` **or** `is_pre_consensus()` → return immediately, no rewards distributed. [3](#0-2) 

`last_reward_block` is a single global storage variable, not per-staker: [4](#0-3) 

Because the write to `last_reward_block` happens **before** the `disable_rewards` guard, any call — including one with `disable_rewards: true` — permanently consumes the block's reward slot. All subsequent calls to `update_rewards` in the same block revert with `REWARDS_ALREADY_UPDATED`.

---

### Impact Explanation

In the consensus phase (`is_pre_consensus()` returns `false`), stakers earn block rewards exclusively through `update_rewards`. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block:

- Consumes the single global reward slot for that block.
- Prevents any legitimate `update_rewards(victim, false)` call from succeeding.
- Denies consensus block rewards to **all** stakers for that block.

Repeated every block, this constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

- Callable by any non-zero address with no special role or stake.
- Requires only one cheap Starknet transaction per block.
- No front-running complexity: the attacker simply needs to be the first caller in each block.
- The attack is fully permissionless and economically rational for a competitor or griever.

---

### Recommendation

**Short term:** Add an authorized-caller check to `update_rewards` (e.g., restrict to a designated block-rewards distributor role, or to the staker themselves). At minimum, gate the `disable_rewards: true` path behind a privileged role.

**Long term:** Decouple the `last_reward_block` write from the `disable_rewards` early-return path, or make `last_reward_block` per-staker so one call cannot block all others.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` is set and current epoch ≥ it).
2. Attacker calls `update_rewards(any_active_staker, disable_rewards: true)` in block N.
3. `last_reward_block` is written to N; no rewards are distributed.
4. Legitimate block-producer call `update_rewards(victim_staker, disable_rewards: false)` in block N reverts: `current_block_number (N) > last_reward_block (N)` is false → `REWARDS_ALREADY_UPDATED`.
5. `victim_staker.unclaimed_rewards_own` is never incremented for block N.
6. Attacker repeats step 2 every block → all stakers permanently receive zero consensus rewards.

**Analog mapping to the external report:**
- Original: `executeTransaction` checks `transfer` and `approve` selectors but omits `increaseApproval` — an incomplete enumeration of equivalent token-movement operations allows bypassing the daily limit.
- Analog here: `update_rewards` checks that the caller is non-zero and the contract is unpaused, but omits any check that the caller is authorized to invoke `disable_rewards: true` — an incomplete authorization guard allows any caller to consume the reward slot without distributing rewards, bypassing the protocol's yield-distribution guarantee. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
