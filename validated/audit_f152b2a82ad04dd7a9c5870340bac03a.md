### Title
Malicious `reward_address` Can Permanently Block `unstake_action`, Temporarily Freezing Staker Funds - (File: `src/staking/staking.cairo`)

---

### Summary

`unstake_action` in `staking.cairo` is callable by **any address** (no access control). Its execution path unconditionally calls `send_rewards_to_staker`, which performs an ERC20 `checked_transfer` to `reward_address` â€” a value fully controlled by the staker. If `reward_address` is a contract that reverts on token receipt, every call to `unstake_action` will revert, permanently blocking the staker's exit and temporarily freezing their staked funds.

---

### Finding Description

`unstake_action` is the second step of the two-step staker exit pattern. Per the spec, **any address can execute it**:

```
#### access control
Any address can execute.
```

Its implementation unconditionally calls `send_rewards_to_staker` before returning stake:

```cairo

```

`send_rewards_to_staker` always calls `checked_transfer` to `reward_address`, regardless of whether the amount is zero:

```cairo
// src/staking/staking.cairo lines 1614â€“1629
fn send_rewards_to_staker(...) {
    let reward_address = staker_info.reward_address;
    let amount = staker_info.unclaimed_rewards_own;
    let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
    claim_from_reward_supplier(:reward_supplier_dispatcher, :amount, :token_dispatcher);
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
    staker_info.unclaimed_rewards_own = Zero::zero();
    self.emit(Events::StakerRewardClaimed { staker_address, reward_address, amount });
}
```

`reward_address` is fully user-controlled and can be changed at any time via `change_reward_address`:

```cairo

    ...
    staker_info.reward_address = reward_address;
    self.write_staker_info(:staker_address, :staker_info);
    ...

```

If a staker sets `reward_address` to a contract that reverts on ERC20 `transfer` (e.g., a contract with no token-receive logic, or a deliberate honeypot), then `checked_transfer` will revert, causing the entire `unstake_action` call to revert. Because `unstake_action` is the **only** way to complete the exit and recover staked funds, the staker's principal is frozen in the contract until `reward_address` is changed.

---

### Impact Explanation

**Temporary freezing of funds (High).**

The staker's entire staked principal is locked in the staking contract and cannot be recovered until `reward_address` is updated to a compliant address. In the worst case â€” where the staker is a smart contract that cannot invoke `change_reward_address` â€” the freeze becomes permanent. Even in the recoverable case, the staker suffers a forced delay on top of the already-mandatory `DEFAULT_EXIT_WAIT_WINDOW`, and any third-party caller of `unstake_action` loses gas with no compensation.

---

### Likelihood Explanation

**Medium.** The scenario arises in two realistic ways:

1. **Accidental**: A staker sets `reward_address` to a multisig, DAO treasury, or custom contract that does not implement ERC20 token receipt (e.g., lacks a fallback or `on_erc20_received` hook). This is a common integration mistake.
2. **Deliberate honeypot**: A staker deploys a `reward_address` contract that passes off-chain simulation (returns success) but reverts on-chain when called from `unstake_action` (e.g., by inspecting the call context). This griefs any third-party caller who attempts to trigger the exit.

No privileged access is required. The staker's own `change_reward_address` call is the only prerequisite.

---

### Recommendation

Guard the reward transfer in `send_rewards_to_staker` so that a failing transfer does not block the exit. Two options:

1. **Skip transfer on zero amount**: Only call `checked_transfer` when `amount > 0`. This eliminates the revert for the common case where no rewards have accrued.
2. **Decouple reward claim from exit**: Do not call `send_rewards_to_staker` inside `unstake_action`. Instead, leave `unclaimed_rewards_own` intact and allow the staker to claim rewards separately via `claim_rewards` after the exit. This fully decouples the two concerns and matches the pattern used in `exit_delegation_pool_action` (which does **not** force a reward transfer).

---

### Proof of Concept

1. Staker deploys `MaliciousRewardAddress` â€” a contract whose `transfer`/`transferFrom` hook always reverts.
2. Staker calls `change_reward_address(malicious_reward_address)`.
3. Staker calls `unstake_intent()`. The exit window timer starts.
4. After `DEFAULT_EXIT_WAIT_WINDOW` elapses, any address calls `unstake_action(staker_address)`.
5. Inside `unstake_action`, `send_rewards_to_staker` calls `checked_transfer(recipient: malicious_reward_address, amount: ...)`.
6. `MaliciousRewardAddress` reverts â†’ `checked_transfer` reverts â†’ `unstake_action` reverts.
7. The staker's principal remains locked in the staking contract. The caller loses gas with no compensation.
8. The staker must call `change_reward_address` to a compliant address before `unstake_action` can succeed.

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

            // Prerequisites and asserts.

            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let old_address = staker_info.reward_address;

            // Update reward_address and commit to storage.
            staker_info.reward_address = reward_address;
            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.
            self
                .emit(
                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,
                    },
                );
        }
