### Title
`setPoolProtocolFee` and `setPoolAdminFees` Permanently Blocked When `adminFeeDestination` Cannot Receive Tokens ÔÇö (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

Both `setPoolProtocolFee` (factory owner) and `setPoolAdminFees` (pool admin) unconditionally call `collectFees` on the pool before applying any fee-rate change. `collectFees` performs live ERC-20 transfers to `adminFeeDestination`. If that address is USDC/USDT-blacklisted or otherwise unable to receive tokens while fees have accrued, the transfer reverts, permanently blocking fee-rate updates and fee collection for the affected pool. Because only the pool admin can change `adminFeeDestination`, the factory owner has no independent path to recover.

### Finding Description

`setPoolProtocolFee` (factory owner) and `setPoolAdminFees` (pool admin) both follow the same pattern:

```
collectFees(old rates, adminFeeDestination)   // ÔćÉ external token transfers happen here
update poolFeeConfig
setPoolFees(new totals)
``` [1](#0-0) [2](#0-1) 

Inside `collectFees`, when accrued fees are non-zero, the pool calls `transferToken0` / `transferToken1` to `adminFeeDestination_`: [3](#0-2) 

If `adminFeeDestination` is USDC/USDT-blacklisted (both tokens are explicitly in scope), the ERC-20 `transfer` reverts. Because `collectFees` is called unconditionally before any state change, the entire transaction reverts and neither the fee-rate update nor the fee collection completes.

The permissionless `collectPoolFees` path is identically blocked: [4](#0-3) 

The only escape valve is `setPoolAdminFeeDestination`, which does **not** call `collectFees`: [5](#0-4) 

However, `setPoolAdminFeeDestination` is gated by `onlyPoolAdmin`. The factory owner has no function to override or replace `adminFeeDestination`. If the pool admin is unresponsive, compromised, or the same blacklisted entity, the factory owner is permanently locked out of:

1. Updating protocol fee rates for that pool (`setPoolProtocolFee`)
2. Collecting accrued protocol fees from that pool (all `collectFees` paths revert)

### Impact Explanation

- **Protocol fees are permanently inaccessible**: All accrued spread and notional fees remain locked inside the pool contract. The factory owner cannot sweep them.
- **Protocol fee-rate governance is broken**: The factory owner cannot adjust `protocolSpreadFeeE6` or `protocolNotionalFeeE8` for the affected pool, even in response to market conditions or security events.
- **Admin-boundary break**: A semi-trusted pool admin (who is only trusted within caps and timelocks) can unilaterally block a privileged factory-owner action with no cap or timelock on `setPoolAdminFeeDestination`.

### Likelihood Explanation

USDC and USDT both maintain on-chain blacklists used for regulatory/sanctions enforcement. A pool admin may legitimately set `adminFeeDestination` to a multisig or DAO treasury that is later sanctioned. The pool admin may then be unable or unwilling to rotate the destination. This is a realistic operational scenario, not a contrived attack.

### Recommendation

Decouple fee collection from fee-rate updates. Two options:

1. **Skip the pre-collection step**: Remove the `collectFees` call from `setPoolProtocolFee` and `setPoolAdminFees`. Callers who want to collect at old rates first can call `collectPoolFees` separately before updating.
2. **Try/catch around the admin transfer leg**: Wrap the `transferToken0/1(adminFeeDestination_, ÔÇŽ)` calls in a try/catch so that a broken destination does not block the protocol-leg transfer or the fee-rate update. Uncollected admin fees could be tracked for later retry.

### Proof of Concept

1. Deploy a pool with USDC as `token0`.
2. Pool admin calls `setPoolAdminFeeDestination(pool, victimAddress)` where `victimAddress` is a USDC-blacklisted address (or will be).
3. Traders execute swaps; spread and notional fees accrue (`surplus0Scaled > 0`, `notionalFeeToken0Scaled > 0`).
4. USDC blacklists `victimAddress`.
5. Factory owner calls `setPoolProtocolFee(pool, newRate, newRate)`:
   - Internally calls `collectFees(ÔÇŽ, victimAddress)`
   - `collectFees` reaches `transferToken0(victimAddress, totalFee0ToAdmin)` Ôćĺ USDC reverts
   - Entire `setPoolProtocolFee` reverts; fee rates unchanged.
6. Anyone calls `collectPoolFees(pool)` Ôćĺ same revert path.
7. Pool admin is unresponsive; factory owner has no function to change `adminFeeDestination`.
8. Protocol fees remain locked in the pool indefinitely; protocol fee governance for this pool is permanently broken. [6](#0-5) [7](#0-6)
