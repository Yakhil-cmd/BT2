### Title
Unauthenticated Invoke Admission via `skip_stateful_validations` — Any Attacker Can Bypass `__validate__` for Any Address with a Pending Deploy-Account - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`skip_stateful_validations` in the gateway's stateful path skips the `__validate__` entry-point call (i.e., signature verification) for any invoke transaction whose nonce is 1 and whose sender address has *any* transaction in the mempool. Because the check is purely address-based and not caller-restricted, an unprivileged attacker can submit an invoke with a garbage signature for a victim's address the moment the victim's `deploy_account` lands in the mempool, and the gateway will admit it without verifying the signature.

### Finding Description

`skip_stateful_validations` is a UX feature that allows a user to submit `deploy_account` + `invoke(nonce=1)` simultaneously, before the deploy is processed. The gateway skips `__validate__` for the invoke when three conditions hold:

1. `tx.nonce() == Nonce(Felt::ONE)`
2. `account_nonce == Nonce(Felt::ZERO)` (account not yet on-chain)
3. `mempool_client.account_tx_in_pool_or_recent_block(sender_address)` returns `true` [1](#0-0) 

Condition 3 is satisfied by the *victim's* `deploy_account` already being in the mempool. The function does not verify that the invoke submitter is the same party who submitted the `deploy_account`. Any third party can craft an invoke with `nonce=1`, any signature, and the victim's sender address, and all three conditions will be met.

When `skip_stateful_validations` returns `true`, `run_validate_entry_point` sets `execution_flags.validate = false`: [2](#0-1) 

Inside `StatefulValidator::perform_validations`, when `validate == false` the function returns `Ok(())` immediately after `perform_pre_validation_stage`, never calling `validate_tx` (which is where `__validate__` / ECDSA verification runs): [3](#0-2) 

`perform_pre_validation_stage` checks nonce and fee bounds but not the signature. The attacker's invoke with a garbage signature therefore passes all gateway checks and is admitted to the mempool.

The mempool's own `validate_tx` only checks nonce validity, not signatures: [4](#0-3) 

`account_tx_in_pool_or_recent_block` returns `true` for any address that has *any* transaction in the pool, not specifically a `deploy_account`: [5](#0-4) 

### Impact Explanation

An attacker can:

1. Monitor the mempool for new `deploy_account` transactions targeting address A.
2. Immediately submit `invoke(sender=A, nonce=1, signature=[garbage], high_tip)` to the gateway.
3. The gateway admits it without signature verification (skip path).
4. The attacker's invalid invoke occupies the `(A, nonce=1)` slot in the mempool with a high tip.
5. The victim's legitimate `invoke(nonce=1)` is either rejected as a duplicate nonce or must pay an even higher fee to replace it.
6. At execution time the attacker's invoke fails (since `__validate__` *is* called during block execution), but the victim's first post-deploy invoke is displaced or DoS'd.

This matches **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing**: an invalid (unauthenticated) transaction is admitted through the gateway into the mempool.

### Likelihood Explanation

The attack is fully unprivileged and requires only the ability to submit RPC transactions. The victim's `deploy_account` is visible in the mempool before it is included in a block, giving the attacker a window to front-run. The condition `nonce == 1 && account_nonce == 0` is deterministic and easy to detect. No special knowledge or resources are required beyond a standard RPC connection.

### Recommendation

Restrict the `skip_stateful_validations` path so that it only applies when the *same* address submitted the pending `deploy_account`. Concretely, the mempool should expose a query such as `has_pending_deploy_account(address) -> bool` that returns `true` only when a `DeployAccount` transaction (not just any transaction) is pending for that address. Replace the current `account_tx_in_pool_or_recent_block` call with this stricter check. [6](#0-5) 

### Proof of Concept

```
# 1. Victim submits deploy_account for address A (nonce=0).
#    Gateway admits it; mempool now contains deploy_account(A).

# 2. Attacker observes deploy_account(A) in the mempool.
#    Attacker crafts:
#      invoke(sender=A, nonce=1, signature=[0xdead, 0xbeef], high_tip)

# 3. Gateway stateful path evaluates skip_stateful_validations:
#    - tx.nonce() == 1                          ✓
#    - account_nonce(A) == 0                    ✓  (A not yet on-chain)
#    - account_tx_in_pool_or_recent_block(A)    ✓  (victim's deploy_account is there)
#    → skip_validate = true
#    → __validate__ is NOT called
#    → attacker's invoke admitted to mempool

# 4. Victim submits invoke(sender=A, nonce=1, valid_signature).
#    Mempool sees duplicate (A, nonce=1); victim must pay higher tip to replace,
#    or the attacker's entry wins the slot.

# 5. Batcher executes block:
#    deploy_account(A) succeeds.
#    attacker's invoke(nonce=1) executes __validate__ → ECDSA fails → tx reverts.
#    victim's legitimate invoke is never executed (displaced).
```

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
