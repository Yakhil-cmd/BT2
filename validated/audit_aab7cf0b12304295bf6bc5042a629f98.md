### Title
Staker Can Instantly Raise Commission to Any Level via `set_commission_commitment` + `set_commission`, Stealing Delegator Yield — (File: `src/staking/staking.cairo`)

---

### Summary

A staker can atomically set a `CommissionCommitment` with `max_commission = 10000` (100%) and immediately call `set_commission(10000)` in the next transaction, raising their commission from 0% to 100% with no time delay. Delegators have no opportunity to exit before their accrued and future rewards are entirely redirected to the staker. The codebase itself acknowledges this gap in a developer note.

---

### Finding Description

The Starknet Staking protocol restricts commission changes as follows:

- **Without** an active `CommissionCommitment`: commission may only be **decreased**.
- **With** an active `CommissionCommitment`: commission may be set to **any value ≤ `max_commission`**, including large increases.

`set_commission_commitment` takes effect **immediately** upon being written to storage. There is no activation delay, no epoch buffer, and no minimum time before the commitment can be exercised. [1](#0-0) 

The only constraint on `expiration_epoch` is that it must be strictly greater than the current epoch (`expiration_epoch > current_epoch`), meaning the minimum valid commitment is active from the current block until the end of the next epoch — but it is usable **immediately**. [2](#0-1) 

Once the commitment is set, `set_commission` calls `update_commission`, which — when an active commitment exists — only checks `commission <= commitment.max_commission` and `commission != old_commission`. No delay, no epoch boundary, no exit window is enforced. [3](#0-2) 

The protocol developers themselves acknowledge this gap with an explicit note directly above `set_commission_commitment`: [4](#0-3) 

> **Note**: Current commission increase safeguards still allow for sudden commission changes.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

A staker operating at 0% commission attracts delegators. At any moment the staker can:

1. Call `set_commission_commitment(max_commission=10000, expiration_epoch=current_epoch+1)`.
2. Immediately call `set_commission(commission=10000)`.

From that point forward, 100% of all pool rewards are routed to the staker. Delegators who have accrued but not yet claimed rewards will find those rewards consumed by the new commission rate when the next attestation epoch settles. Delegators who attempt to exit still face the `exit_wait_window` during which the elevated commission continues to apply to any rewards that accrue. [5](#0-4) 

---

### Likelihood Explanation

**High.** The entry path requires only the staker's own key — no privileged role, no governance action, no external dependency. The two-transaction sequence (`set_commission_commitment` → `set_commission`) is trivially executable by any staker at any time the contract is unpaused. The economic incentive (capturing 100% of delegator yield) is strong, and the attack is undetectable until after it has already taken effect. [5](#0-4) 

---

### Recommendation

Introduce a mandatory activation delay between setting a `CommissionCommitment` and being permitted to exercise it for a commission **increase**. Specifically:

- Record the epoch in which the commitment was set (`set_epoch`).
- In `update_commission`, when the new commission exceeds the old commission, require `current_epoch >= commitment.set_epoch + MIN_DELAY_EPOCHS` before allowing the increase.
- `MIN_DELAY_EPOCHS` should be at least as long as the `exit_wait_window` expressed in epochs, giving delegators a full exit window to react before the higher commission takes effect.

This mirrors the "time delay before transitioning from PENDING to INITIALIZED" recommendation in the reference report. [6](#0-5) 

---

### Proof of Concept

```
Epoch E, Block B:
  staker.set_commission_commitment(max_commission=10000, expiration_epoch=E+1)
  // CommissionCommitment { max_commission: 10000, expiration_epoch: E+1 } written to storage.
  // is_commission_commitment_active() → true (E+1 > E).

Epoch E, Block B+1 (or same block):
  staker.set_commission(commission=10000)
  // update_commission called:
  //   commission (10000) <= commitment.max_commission (10000) ✓
  //   commission (10000) != old_commission (0) ✓
  //   → commission written as 10000 immediately.

Next attestation in Epoch E:
  Rewards distributed: 100% to staker, 0% to delegators.
  Delegators cannot exit without waiting through exit_wait_window,
  during which the 100% commission continues to apply.
``` [3](#0-2) [7](#0-6)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
