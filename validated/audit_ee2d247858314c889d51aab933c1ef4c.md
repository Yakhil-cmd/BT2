### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Deny Block Rewards to Stakers — (`File: src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is documented in the spec as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged address can call it with `disable_rewards: true`, which writes `last_reward_block` to the current block and then returns early without distributing rewards. Because the contract enforces exactly one call per block, the legitimate sequencer call for that block is permanently blocked with `REWARDS_ALREADY_UPDATED`. The staker's block reward for that block is never created and is permanently lost.

---

### Finding Description

`update_rewards` is the sole mechanism by which block rewards are distributed to stakers in the V3 (consensus) reward model. Its spec entry explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation contains no caller check: [1](#0-0) 

The function:
1. Calls `general_prerequisites()` (only checks the pause flag).
2. Asserts `current_block_number > last_reward_block` — one call per block.
3. **Unconditionally writes `last_reward_block = current_block_number`** before checking `disable_rewards`.
4. Returns early if `disable_rewards == true`, distributing nothing. [2](#0-1) 

Because `last_reward_block` is updated at line 1485 regardless of `disable_rewards`, any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`.

The block rewards themselves are computed once per epoch via `calculate_block_rewards` → `update_current_epoch_block_rewards` → `yearly_mint`, which reads `get_current_total_staking_power()`: [3](#0-2) [4](#0-3) [5](#0-4) 

When `disable_rewards: true` is used to front-run the sequencer, the block reward for that block is never distributed to the staker and is permanently lost — it is not deferred or queued anywhere.

---

### Impact Explanation

Each block reward lost is permanent. An attacker who calls `update_rewards(staker_address, disable_rewards: true)` before the sequencer's call for every block can permanently zero out a targeted staker's (and their delegators') yield for as long as the attack continues. This maps directly to **Permanent freezing of unclaimed yield** (High).

---

### Likelihood Explanation

On Starknet, users submit transactions to the sequencer, which orders them. The sequencer's `update_rewards` call is a regular L2 transaction. An attacker can submit a competing transaction targeting the same staker in the same block. Because the code places no caller restriction, the sequencer has no protocol-level guarantee that its call arrives first. The attack requires only gas fees and can be automated to run every block against any staker.

---

### Recommendation

Add a caller check matching the spec's stated access control. The simplest fix is to restrict `update_rewards` to the sequencer address (stored in contract config) or to a dedicated operator role:

```cairo
assert!(
    get_caller_address() == self.sequencer_address.read(),
    "{}",
    Error::CALLER_IS_NOT_SEQUENCER,
);
```

Alternatively, move the `last_reward_block` write to **after** the `disable_rewards` guard so that a call with `disable_rewards: true` does not consume the block's reward slot.

---

### Proof of Concept

1. Staker `S` is active with non-zero balance in epoch `E` (consensus rewards active).
2. Attacker `A` monitors the block and submits `update_rewards(S, disable_rewards: true)`.
3. The call succeeds: `last_reward_block` is set to the current block number; no rewards are distributed.
4. The sequencer's legitimate `update_rewards(S, disable_rewards: false)` call reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker `S` receives zero rewards for this block. The block reward is permanently lost.
6. Repeated every block → staker `S` earns zero rewards indefinitely.

This is confirmed by the existing test infrastructure, which calls `update_rewards` directly from arbitrary addresses and observes the `REWARDS_ALREADY_UPDATED` revert on a second same-block call: [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

        fn update_current_epoch_block_rewards(ref self: ContractState) -> (Amount, Amount) {
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            self.set_avg_block_duration();
            // Calculate block rewards for the current epoch.
            let minting_curve_dispatcher = self.minting_curve_dispatcher.read();
            let yearly_mint = minting_curve_dispatcher.yearly_mint();
            let avg_block_duration = self.avg_block_duration.read();
            let total_rewards = mul_wide_and_div(
                lhs: yearly_mint,
                rhs: avg_block_duration.into(),
                div: BLOCK_DURATION_SCALE.into() * SECONDS_IN_YEAR.into(),
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW);
            let btc_rewards = calculate_btc_rewards(:total_rewards);
            let strk_rewards = total_rewards - btc_rewards;
            (strk_rewards, btc_rewards)
        }
