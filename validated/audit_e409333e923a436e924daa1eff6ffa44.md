### Title
Gateway Stateful Validator Uses Stale Previous-Block L2 Gas Price for Resource Bounds Admission, Allowing Transactions That Fail Batcher Execution to Enter the Mempool — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` reads the **previous committed block's** L2 gas price to check whether a transaction's `max_price_per_unit` meets the admission threshold. The batcher, however, builds the next block with a **freshly computed** L2 gas price derived from the EIP-1559 formula (`calculate_next_l2_gas_price_for_fin`). When the next block's price is higher than the previous block's price, a transaction whose `max_price_per_unit` falls between the two values passes gateway admission and enters the mempool, but is then rejected by the blockifier's `check_fee_bounds` during batcher execution — wasting mempool capacity and producing a silent admission-then-drop outcome for the submitter.

---

### Finding Description

**Gateway admission path — stale price used:**

In `validate_resource_bounds`, the gateway reads the last committed block's info and extracts its L2 gas price:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
```

The TODO comment itself acknowledges the correct value is the **next** block's price, not the previous block's price. [1](#0-0) 

`validate_tx_l2_gas_price_within_threshold` then computes a threshold as `previous_block_l2_gas_price * min_gas_price_percentage / 100` and rejects only if `tx.max_price_per_unit < threshold`. With `min_gas_price_percentage = 100` (the default), the threshold equals the previous block's price exactly. [2](#0-1) 

The blockifier validation inside `run_validate_entry_point` also uses the same previous block's `block_info` (with `block_number.unchecked_next()`), so `check_fee_bounds` during gateway validation also uses the previous block's gas prices — the transaction passes both checks. [3](#0-2) 

**Batcher execution path — fresh EIP-1559 price used:**

When the consensus orchestrator initiates a block proposal, it computes the next L2 gas price via `calculate_next_l2_gas_price_for_fin` (EIP-1559 formula) and passes it to the batcher as part of `ProposeBlockInput.block_info`: [4](#0-3) 

The batcher immediately calls `mempool_client.update_gas_price` with this new price, and the blockifier's `check_fee_bounds` during execution uses the same `block_info` gas prices: [5](#0-4) 

`check_fee_bounds` in `perform_pre_validation_stage` enforces `tx.max_price_per_unit >= actual_gas_price` from the block context. If the next block's price is higher than the previous block's price, this check fails with `MaxGasPriceTooLow`, a `TransactionPreValidationError` that causes the transaction to be dropped without inclusion. [6](#0-5) [7](#0-6) 

**The correct value already exists in storage but is not used:**

`StorageBlockHeader` carries a `next_l2_gas_price` field — the pre-computed EIP-1559 price for the next block — which is exactly what the gateway should use for admission validation: [8](#0-7) 

---

### Impact Explanation

Any transaction with `max_price_per_unit` in the range `[previous_block_l2_gas_price, next_block_l2_gas_price)` passes gateway admission and enters the mempool, but is silently dropped by the batcher at pre-validation time. This matches the allowed impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

Concretely:
- An attacker (or a misconfigured wallet) can flood the mempool with transactions priced just above the previous block's price. All pass gateway admission; all fail batcher execution. The mempool's gas-price threshold (`update_gas_price`) is updated to the new block price only at proposal time, so the window is open for every block where the EIP-1559 price increases.
- Legitimate users whose wallets price transactions at exactly the last-seen block price will have their transactions silently dropped rather than receiving an upfront rejection, forcing them to resubmit.

---

### Likelihood Explanation

The EIP-1559 formula (`calculate_next_base_gas_price`) increases the price whenever `gas_used > gas_target`. Under normal load (blocks above target), the price rises every block. The `gas_price_max_change_denominator` bounds the per-block increase, but even a 0.1% increase per block is enough to create the gap. The condition is therefore met continuously during any period of above-target block utilization — which is the normal operating state of a busy network. [9](#0-8) 

---

### Recommendation

Replace the `get_block_info().gas_prices.strk_gas_prices.l2_gas_price` read in `validate_resource_bounds` with the `next_l2_gas_price` field already stored in `StorageBlockHeader`. The `GatewayFixedBlockStateReader` trait should expose `get_next_l2_gas_price()` (reading `StorageBlockHeader::next_l2_gas_price`) and `validate_resource_bounds` should call it instead of reading the current block's price. This directly resolves the TODO comment at line 229 and aligns gateway admission with the price the batcher will actually enforce. [10](#0-9) 

---

### Proof of Concept

1. Observe the current committed block's L2 gas price: `P_prev`.
2. Compute the next block's EIP-1559 price: `P_next = calculate_next_base_gas_price(P_prev, gas_used, gas_target, min_price)`. When `gas_used > gas_target`, `P_next > P_prev`.
3. Submit an `InvokeV3` transaction with `AllResourceBounds` where `l2_gas.max_price_per_unit = P_prev` (exactly the previous block's price, which equals the gateway threshold when `min_gas_price_percentage = 100`).
4. The gateway's `validate_resource_bounds` computes `threshold = P_prev * 100 / 100 = P_prev`; the transaction satisfies `P_prev >= P_prev` → **admitted**.
5. The gateway's blockifier pre-validation also uses `P_prev` as the block gas price → **passes `check_fee_bounds`**.
6. Transaction enters the mempool.
7. The batcher proposes the next block with `block_info.gas_prices.strk_gas_prices.l2_gas_price = P_next > P_prev`.
8. Blockifier's `check_fee_bounds` evaluates `P_prev < P_next` → `MaxGasPriceTooLow` → `TransactionPreValidationError` → transaction **dropped without inclusion**.

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
```

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
```
