### Title
Unprivileged Caller Can Permanently Deny Consensus Block Rewards to All Stakers via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. Because `last_reward_block` is a **global** storage slot updated unconditionally before the rewards-distribution branch, any unprivileged caller can invoke `update_rewards(any_staker, disable_rewards: true)` once per block to consume the protocol's single per-block reward slot without distributing any rewards — permanently denying consensus block rewards to every staker in the protocol for as long as the attack continues.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` has no role guard; `general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero:

```cairo
// src/staking/staking.cairo  lines 1448-1507
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: unpaused + caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.  <-- GLOBAL write, happens BEFORE the guard
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits without distributing rewards
    }
    ...
}
```

`last_reward_block` is a single global `BlockNumber` in storage:

```cairo
// src/staking/staking.cairo  line 187
last_reward_block: BlockNumber,
```

The assertion `current_block_number > self.last_reward_block.read()` enforces that rewards can only be distributed **once per block across the entire protocol**. Once `last_reward_block` is written for block N, every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`.

Because `disable_rewards` is fully attacker-controlled and there is no access control, an attacker can:

1. Call `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block.
2. `last_reward_block` is updated to the current block number.
3. The function returns early — no rewards are distributed.
4. Every legitimate call to `update_rewards` in that block reverts.

The attacker needs only **one transaction per block** to deny rewards to the entire staker set.

---

### Impact Explanation

`last_reward_block` is global, not per-staker. A single call with `disable_rewards: true` blocks all stakers from receiving consensus block rewards for that block. Sustained over time this constitutes:

- **Permanent freezing of unclaimed yield** (High) — stakers and delegators accrue zero consensus rewards for every block the attack is active.
- **Temporary freezing of funds** (High) — staked principal is not at risk, but yield is frozen for the attack duration.

The attack is economically viable: one cheap L2 transaction per block is sufficient to zero out rewards for the entire protocol.

---

### Likelihood Explanation

- The function is publicly callable with no role restriction.
- The `disable_rewards` flag is a plain boolean parameter with no validation.
- The attacker requires no special privileges, no staked funds, and no prior relationship with any staker.
- The cost is one transaction per block on Starknet (low gas environment).
- The attack is trivially scriptable and can be run continuously.

---

### Recommendation

1. **Remove the `disable_rewards` parameter from the public interface**, or restrict it to a privileged role (e.g., `only_app_governor`).
2. Alternatively, make `last_reward_block` a **per-staker** mapping so that one call cannot block all other stakers.
3. If `disable_rewards` must remain for migration purposes, gate it behind `self.roles.only_app_governor()` or equivalent.

---

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
