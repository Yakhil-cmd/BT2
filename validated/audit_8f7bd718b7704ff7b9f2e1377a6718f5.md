### Title
`setPoolAdminFeeDestination` Overwrites Fee Destination Before Collecting Accrued Fees, Redirecting Owed Admin Fees to New Address - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

---

### Summary

`setPoolAdminFeeDestination` updates `poolAdminFeeDestination[pool]` to a new address without first flushing accrued fees to the old destination. Every other fee-state-mutating function in the factory (`setPoolAdminFees`, `setPoolProtocolFee`) explicitly calls `collectFees` with the **old** destination before writing new state. The omission in `setPoolAdminFeeDestination` means all fees that accrued while the old destination was active are silently redirected to the new destination on the next `collectPoolFees` call.

---

### Finding Description

`setPoolAdminFees` and `setPoolProtocolFee` both follow the same safe pattern: snapshot the current `poolFeeConfig` and `poolAdminFeeDestination`, call `pool.collectFees(…, poolAdminFeeDestination[pool])` to flush accrued fees to the **old** destination, and only then write the new state. [1](#0-0) 

`setPoolAdminFeeDestination` skips this flush entirely: [2](#0-1) 

After the write, `poolAdminFeeDestination[pool]` already points to the new address. The permissionless `collectPoolFees` then reads the updated mapping and passes the **new** address to `pool.collectFees`: [3](#0-2) 

Inside `collectFees`, both the spread surplus and the accumulated `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` are transferred to `adminFeeDestination_`: [4](#0-3) 

All fees that were earned while the old destination was the registered recipient are therefore delivered to the new destination instead.

---

### Impact Explanation

The old `adminFeeDestination` — which may be a separate revenue-sharing contract, a DAO treasury, or any address distinct from the pool admin — loses every token of admin fees that had accrued since the last collection. Those tokens are transferred to the new destination. This is a direct, irreversible loss of owed protocol-adjacent fees with no recovery path for the old recipient.

---

### Likelihood Explanation

The pool admin is a semi-trusted role that legitimately calls `setPoolAdminFeeDestination` during normal operations (treasury rotation, multisig migration, etc.). No timelock or cap gates this function. Any non-trivial pool with active swap volume will have accrued fees at the time of a destination change, making the loss practically certain whenever the function is used.

---

### Recommendation

Mirror the pattern used by `setPoolAdminFees` and `setPoolProtocolFee`: collect accrued fees to the **current** destination before overwriting it.

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();

    // Flush accrued fees to the OLD destination before rotating
    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6,
        c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8,
        c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]   // old destination
    );

    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

---

### Proof of Concept

1. Pool is deployed; `poolAdminFeeDestination[pool] = oldDest`.
2. Swaps occur; `notionalFeeToken0Scaled` and spread surplus accumulate on the pool.
3. Pool admin calls `setPoolAdminFeeDestination(pool, newDest)`.
   - `poolAdminFeeDestination[pool]` is immediately set to `newDest`.
   - No `collectFees` is called.
4. Anyone calls `collectPoolFees(pool)`.
   - Factory reads `poolAdminFeeDestination[pool]` → `newDest`.
   - `pool.collectFees(…, newDest)` transfers all accrued admin fees to `newDest`.
5. `oldDest` receives zero tokens despite having been the registered recipient throughout the entire accrual period.

The invariant broken: fees earned under `oldDest` must be settled to `oldDest` before the destination pointer is rotated — exactly the ordering guarantee that `setPoolAdminFees` and `setPoolProtocolFee` enforce but `setPoolAdminFeeDestination` omits. [5](#0-4) [2](#0-1)
