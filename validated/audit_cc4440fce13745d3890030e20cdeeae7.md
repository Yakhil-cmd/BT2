### Title
Pre-deployment of ERC-20 Address Permanently Blocks `mirror_erc20_token` on Silo - (File: engine/src/engine.rs)

### Summary

The `mirror_erc20_token` flow deploys an ERC-20 contract to a **fixed, publicly known address** on a Silo using `CreateScheme::Fixed`. Any unprivileged EVM user can pre-deploy a contract to that address on the Silo via `CREATE2` before the Silo owner calls `mirror_erc20_token`. When the callback executes, `transact_create_fixed` fails with `CreateCollision`, permanently preventing that NEP-141 token from ever being bridged to the Silo.

### Finding Description

`mirror_erc20_token` in `engine/src/engine.rs` is called from `mirror_erc20_token_callback` in `engine/src/contract_methods/connector.rs`. It receives `erc20_address` — the address of the ERC-20 on the **main** Aurora contract — and attempts to deploy the same ERC-20 bytecode to that **exact same address** on the Silo:

```rust
// engine/src/engine.rs ~line 1398
let address = match engine.deploy_code_with_input(input, Some(erc20_address), handler) {
```

`deploy_code_with_input` with `Some(erc20_address)` routes through `deploy_code` using `CreateScheme::Fixed`:

```rust
// engine/src/engine.rs ~line 544-562
let scheme = address.map_or_else(
    || CreateScheme::Legacy { caller: origin.raw() },
    |address| CreateScheme::Fixed(address.raw()),   // ← fixed address path
);
// ...
CreateScheme::Fixed(address) => executor.transact_create_fixed(
    origin.raw(), address, value.raw(), input, gas_limit, access_list,
),
```

`transact_create_fixed` (from `aurora-evm`) checks whether the target address already has code or a non-zero nonce. If it does, it returns `ExitError::CreateCollision`, which propagates as `TransactionStatus::CreateCollision` → `DeployErc20Error::Failed` → the callback returns an error and the entire `mirror_erc20_token_callback` NEAR receipt fails.

The target `erc20_address` is **publicly readable** from the main Aurora contract via `get_erc20_from_nep141`. Any EVM user on the Silo can deploy a contract to that address at any time before the Silo owner calls `mirror_erc20_token`, using a `CREATE2` factory:

```solidity
// Attacker deploys this on the Silo
contract Blocker {
    function block(bytes32 salt, address target) external {
        // craft salt so that keccak256(0xff ++ address(this) ++ salt ++ keccak256("")) == target
        new Dummy{salt: salt}();
    }
}
```

Since `CREATE2` addresses are deterministic and the attacker controls both the deployer address and the salt, they can always find a combination that produces the target address. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation

Once the attacker's contract occupies the target address on the Silo:

1. Every call to `mirror_erc20_token` for that NEP-141 token fails with `CreateCollision`.
2. The NEP-141 ↔ ERC-20 mapping is never registered on the Silo.
3. `ft_on_transfer` for that token will fail on the Silo, so users cannot bridge that token.
4. The only recovery path is deploying an entirely new Silo instance — a significant operational disruption.

This constitutes **permanent freezing of the bridging capability** for that token on the affected Silo, matching the **High — Temporary freezing of funds** impact tier (and potentially Critical — Permanent freezing of funds if the Silo cannot be replaced). [5](#0-4) [6](#0-5) 

### Likelihood Explanation

- The target address is **public state** on the main Aurora contract, readable by anyone via `get_erc20_from_nep141`.
- Any EVM account on the Silo can deploy a `CREATE2` factory and compute the right salt offline — no special privilege is required.
- The attack can be executed at any time before the Silo owner calls `mirror_erc20_token`; there is no race condition to win.
- Motivation exists for any party that wants to prevent a competitor's token from being available on a Silo. [7](#0-6) 

### Recommendation

Before calling `transact_create_fixed`, check whether the target address already has code in the Silo's EVM state. If it does, either:

1. **Reject early** with a clear error (e.g., `ERR_ERC20_ADDRESS_OCCUPIED`) so the owner is informed without wasting gas.
2. **Verify the existing code** matches the expected ERC-20 bytecode; if it does, skip deployment and only register the mapping — analogous to the resolution described in the referenced report.

```rust
// Suggested guard in mirror_erc20_token, before deploy_code_with_input:
let existing_code = engine::get_code(&io, &erc20_address);
if !existing_code.is_empty() {
    // Either verify it matches expected bytecode, or return an error
    return Err(DeployErc20Error::AddressAlreadyOccupied);
}
``` [8](#0-7) 

### Proof of Concept

1. Read the ERC-20 address for `token.near` from the main Aurora contract:
   ```
   aurora.get_erc20_from_nep141("token.near") → 0xABCD...1234
   ```
2. On the Silo, deploy a `CREATE2` factory and call it with a salt chosen so the deployed address equals `0xABCD...1234`. Any trivial bytecode (e.g., `STOP`) suffices.
3. Silo owner calls `mirror_erc20_token({contract_id: "aurora", nep141: "token.near"})`.
4. The callback `mirror_erc20_token_callback` runs, calls `engine::mirror_erc20_token`, which calls `transact_create_fixed` targeting `0xABCD...1234`.
5. `transact_create_fixed` returns `ExitError::CreateCollision` because the address already has code.
6. The callback fails; the NEP-141 mapping is never registered; `token.near` is permanently unbridgeable on the Silo. [9](#0-8) [10](#0-9)

/// Used to mirror deployed ERC-20 contract on main contract to silo.
pub fn mirror_erc20_token<I: IO + Copy, E: Env, P: PromiseHandler>(
    args: MirrorErc20TokenArgs,
    erc20_address: Address,
    erc20_metadata: Erc20Metadata,
    io: I,
    env: &E,
    handler: &mut P,
) -> Result<Address, DeployErc20Error> {
    let current_account_id = env.current_account_id();
    let input = setup_deploy_erc20_input(&current_account_id, Some(erc20_metadata));
    let mut engine: Engine<_, _> = Engine::new(
        aurora_engine_sdk::types::near_account_to_evm_address(
            env.predecessor_account_id().as_bytes(),
        ),
        current_account_id,
        io,
        env,
    )
    .map_err(DeployErc20Error::State)?;

    let address = match engine.deploy_code_with_input(input, Some(erc20_address), handler) {
        Ok(result) => match result.status {
            TransactionStatus::Succeed(ret) => {
                Address::new(H160(ret.as_slice().try_into().unwrap()))
            }
            other => return Err(DeployErc20Error::Failed(other)),
        },
        Err(e) => return Err(DeployErc20Error::Engine(e)),
    };

    assert_eq!(address, erc20_address);

    sdk::log!(
        "ERC-20 on: {} at address: {} has been mirrored",
        args.contract_id.as_ref(),
        address.encode()
    );
    engine
        .register_token(address, args.nep141)
        .map_err(DeployErc20Error::Register)?;

    Ok(address)
}

pub fn get_erc20_from_nep141<I: IO>(
    io: &I,
    nep141_account_id: &AccountId,
) -> Result<Address, GetErc20FromNep141Error> {
    let key = bytes_to_key(KeyPrefix::Nep141Erc20Map, nep141_account_id.as_bytes());
    io.read_storage(&key)
        .map(|v| {
            let mut buf = [0u8; 20];
            v.copy_to_slice(&mut buf);
            Address::from_array(buf)
        })
        .ok_or(GetErc20FromNep141Error::Nep141NotFound)
}

pub fn mirror_erc20_token<I: IO + Env + Copy, H: PromiseHandler>(
    io: I,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    // TODO: Add an admin access list of accounts allowed to do it.
    require_owner_only(&state, &io.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let args =
        MirrorErc20TokenArgs::try_from_slice(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    // We can't use a batch of actions here, since we need to get responses from both
    // view transactions in the `mirror_erc20_token_callback` callback.
    let promises = vec![
        PromiseCreateArgs {
            target_account_id: args.contract_id.clone(),
            method: "get_erc20_from_nep141".to_string(),
            args: borsh::to_vec(&args.nep141).map_err(|_| errors::ERR_SERIALIZE)?,
            attached_balance: Yocto::new(0),
            attached_gas: READ_PROMISE_ATTACHED_GAS,
        },
        PromiseCreateArgs {
            target_account_id: args.contract_id,
            method: "get_erc20_metadata".into(),
            args: serde_json::to_vec(&Erc20Identifier::from(args.nep141))
                .map_err(|_| errors::ERR_SERIALIZE)?,
            attached_balance: Yocto::new(0),
            attached_gas: READ_PROMISE_ATTACHED_GAS,
        },
    ];

    let callback = PromiseCreateArgs {
        target_account_id: io.current_account_id(),
        method: "mirror_erc20_token_callback".to_string(),
        args: input,
        attached_balance: Yocto::new(0),
        attached_gas: MIRROR_ERC20_TOKEN_CALLBACK_ATTACHED_GAS,
    };
    // Safe because these promises are read-only calls to the main engine contract,
    // and this transaction could be executed by the owner of the contract only.
    let base_id = handler.promise_create_and_combine(&promises);
    let callback_id = handler.promise_attach_callback(base_id, &callback);

    handler.promise_return(callback_id);

    Ok(())
```

#[named]
pub fn mirror_erc20_token_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;

        require_running(&state)?;
        env.assert_private_call()?;

        if handler.promise_results_count() != 2 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: MirrorErc20TokenArgs = io.read_input_borsh()?;
        let erc20_address =
            if let Some(PromiseResult::Successful(bytes)) = handler.promise_result(0) {
                Address::try_from_slice(&bytes)?
            } else {
                return Err(errors::ERR_GETTING_ERC20_FROM_NEP141.into());
            };

        let erc20_metadata =
            if let Some(PromiseResult::Successful(bytes)) = handler.promise_result(1) {
                serde_json::from_slice(&bytes).map_err(Into::<ParseArgsError>::into)?
            } else {
                return Err(errors::ERR_GETTING_ERC20_FROM_NEP141.into());
            };

        let address =
            engine::mirror_erc20_token(args, erc20_address, erc20_metadata, io, env, handler)?;

        io.return_output(&borsh::to_vec(address.as_bytes()).map_err(|_| errors::ERR_SERIALIZE)?);

        Ok(())
    })
}
