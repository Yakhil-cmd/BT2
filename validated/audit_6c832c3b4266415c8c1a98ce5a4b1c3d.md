### Title
`MsgTurnBridge` Handler Is a No-Op Stub With No Access Control — Any Unprivileged Sender Succeeds; Bridge Circuit Breaker Is Permanently Broken - (File: `x/cronos/keeper/msg_server.go`)

### Summary

The `TurnBridge` message handler, which is the designated emergency circuit breaker for the Gravity bridge, is implemented as a stub that unconditionally returns `nil, nil`. It performs no permission check and no state mutation. Any unprivileged address can submit `MsgTurnBridge` and receive a success response, while the `CanTurnBridge` permission system is silently bypassed. Simultaneously, no authorized party can ever actually disable the bridge, permanently breaking the circuit breaker.

### Finding Description

`MsgTurnBridge` is registered as a live gRPC transaction endpoint in the Cronos module. ADR-009 explicitly designates it as a restricted message requiring the `CanTurnBridge` permission. The permission constant is defined and the `HasPermission` guard is used correctly in `UpdateTokenMapping`. However, the `TurnBridge` handler itself was never implemented:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Compare with `UpdateTokenMapping`, which correctly gates on `CanChangeTokenMapping`: [2](#0-1) 

The `CanTurnBridge` permission bit is defined and the query endpoint reports it, but it is never checked in the handler: [3](#0-2) 

ADR-009 explicitly states both `MsgUpdateTokenMapping` and `MsgTurnBridge` must be restricted: [4](#0-3) 

### Impact Explanation

Two distinct impacts arise from the same root cause:

1. **Authorization bypass (High):** The `CanTurnBridge` permission check is completely absent. Any unprivileged address can broadcast `MsgTurnBridge` and receive a `code: 0` success response. The entire permission system for bridge control is silently bypassed.

2. **Permanent inability to disable the bridge (High):** Because the handler is a no-op, no party — including the admin or a permissioned address — can ever disable the Gravity bridge. The circuit breaker designed for emergency use is permanently non-functional. If a bridge exploit or critical bug occurs, the bridge cannot be halted via this message, leaving bridge/conversion flows permanently exposed.

### Likelihood Explanation

The endpoint is publicly reachable via the standard Cosmos SDK transaction path. No special access is required to submit `MsgTurnBridge`. The stub is in the production `msg_server.go`, not a test file. Any observer of the chain's protobuf service descriptor can discover and call it. [5](#0-4) 

### Recommendation

Implement `TurnBridge` with the same pattern used by `UpdateTokenMapping`:

1. Add a `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` guard at the top of the handler.
2. Implement the actual bridge enable/disable state mutation (write the `enable` flag to a keeper-managed store key).
3. Ensure the bridge-enabled flag is read by the Gravity bridge hooks before processing any bridge transactions.

### Proof of Concept

```
# Any unprivileged address can call MsgTurnBridge and receive code: 0
cronosd tx cronos turn-bridge false --from <any_unprivileged_key> -y

# Expected: ErrUnauthorized (code != 0)
# Actual:   code: 0, raw_log: "" — success, no state change, no permission check
```

The `ValidateBasic` for `MsgTurnBridge` only checks that the sender address is a valid bech32 string — no permission is enforced at any layer: [6](#0-5)

func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}
