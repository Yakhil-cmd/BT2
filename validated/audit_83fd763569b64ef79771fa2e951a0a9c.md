### Title
Gateway Stateful Validator Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` for Admission Threshold — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary
`StatefulTransactionValidator::validate_resource_bounds` reads the **current block's** `l2_gas_price` (P_N) from the latest committed block header to compute the admission threshold, but transactions will be executed in the **next block** at `next_l2_gas_price` (P_{N+1}). When P_{N+1} > P_N (the normal case after a high-gas block), the gateway admits transactions whose offered price satisfies `P_N ≤ tx_price < P_{N+1}`. Those transactions pass gateway admission, enter the mempool, and are then rejected by the batcher's `check_fee_bounds` with `MaxGasPriceTooLow` — a pre-validation failure that charges no fee and increments no nonce. The code itself acknowledges the defect with a TODO: `// TODO(Arni): getnext_l2_gas_price from the block header.`

### Finding Description
In `validate_resource_bounds`, the validator fetches block info from `gateway_fixed_block_state_reader.get_block_info()` and extracts:

```rust
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;          // ← P_N: price used in the *already-committed* block
``` [1](#0-0) 

The threshold is then `(min_gas_price_percentage / 100) × P_N`. A transaction passes if `tx_l2_gas_price ≥ threshold`.

Meanwhile, the batcher builds block N+1 using `next_l2_gas_price = P_{N+1}`, which is stored in block N's header as `BlockHeaderWithoutHash::next_l2_gas_price` and is set to `self.l2_gas_price` in the consensus orchestrator after each block is finalized:

```rust
let block_header_without_hash = BlockHeaderWithoutHash {
    l2_gas_price,                          // P_N
    next_l2_gas_price: self.l2_gas_price,  // P_{N+1}
    ...
};
``` [2](#0-1) 

`self.l2_gas_price` is updated via `update_l2_gas_price` after each committed block:

```rust
fn update_l2_gas_price(&mut self, height: BlockNumber, l2_gas_used: GasAmount) {
    self.l2_gas_price = self.calculate_next_l2_gas_price(height, l2_gas_used);
    ...
}
``` [3](#0-2) 

When the batcher executes a transaction in block N+1, `perform_pre_validation_stage` calls `check_fee_bounds`, which compares `tx_l2_gas_price` against the block context's `l2_gas_price = P_{N+1}`:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [4](#0-3) 

The gateway's `run_validate_entry_point` also builds its block context from the same stale `get_block_info()` (with only the block number bumped), so both the gateway's stateless pre-check and its blockifier validation use P_N, while the batcher uses P_{N+1}. The gap `[P_N, P_{N+1})` is the admission window that the gateway opens but the batcher closes.

### Impact Explanation
Any transaction with `P_N ≤ tx_l2_gas_price < P_{N+1}` is:
1. **Admitted** by the gateway (passes `validate_resource_bounds` and `run_validate_entry_point`)
2. **Admitted** to the mempool (passes `validate_by_mempool`)
3. **Rejected** by the batcher at pre-validation with `MaxGasPriceTooLow` — no fee charged, no nonce increment, no state change

An unprivileged user can flood the mempool with zero-cost invalid transactions whenever the L2 gas price is rising (the common case under load). This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation
The L2 gas price rises whenever a block's `l2_gas_used` exceeds the `gas_target`. Under any sustained load the price climbs monotonically block-by-block
