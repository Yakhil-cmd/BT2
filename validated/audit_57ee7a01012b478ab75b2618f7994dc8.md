### Title
Missing Maximum Commission Guard in `enter_delegation_pool` / `add_to_delegation_pool` Exposes Delegators to Unlimited Reward Dilution - (File: src/pool/pool.cairo)

---

### Summary
Delegators have no way to specify a maximum acceptable commission when entering or adding to a delegation pool. A staker who holds an active `CommissionCommitment` can raise commission to any value up to `max_commission` (up to 100 %) in a single transaction immediately after a delegator joins, stealing up to 100 % of the delegator's expected yield with no recourse.

---

### Finding Description

**Root cause — no slippage guard on pool entry**

`enter_delegation_pool` and `add_to_delegation_pool` in `src/pool/pool.cairo` accept only `reward_address` / `pool_member` and `amount`. Neither function accepts a `max_commission` parameter, so the delegator cannot express "revert if commission > X" atomically with the deposit. [1](#0-0) [2](#0-1) 

**Enabling mechanism — `CommissionCommitment` allows upward commission changes**

Without a commitment, `set_commission` enforces `commission < old_commission` (commission can only decrease). However, `set_commission_commitment` lets a staker publish a commitment with an arbitrary `max_commission` (up to `COMMISSION_DENOMINATOR = 10000`, i.e. 100 %). While that commitment is active, `update_commission` only requires `commission <= commitment.max_commission` and `commission != old_commission` — a single call can jump from 1 % to 100 %. [3](#0-2) [4](#0-3) 

**Commission is applied at attestation time, not at entry time**

Rewards are calculated and split in `_update_rewards` → `calculate_staker_pools_rewards` at the moment the staker attests. The pool contract receives only the net amount (after commission deduction) via `update_rewards_from_staking_contract`. A commission change that occurs between the delegator's deposit and the next attestation therefore silently reduces the delegator's share for every subsequent epoch. [5](#0-4) [6](#0-5) 

**Attack path (step-by-step)**

1. Staker calls `set_commission(commission: 500)` → 5 % commission, visible on-chain.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current + 200)` → commitment is now active; this is also visible on-chain but easy to overlook.
3. Delegator observes 5 % commission, calls `enter_delegation_pool(reward_address, amount: X)`. No `max_commission` guard exists; the call succeeds.
4. Staker immediately calls `set_commission(commission: 10000)` → 100 % commission. This satisfies `commission <= max_commission` and `commission != old_commission`, so it succeeds.
5. Staker attests; `calculate_staker_pools_rewards` applies 100 % commission, sending 0 net rewards to the pool contract.
6. Delegator calls `claim_rewards` and receives 0 STRK.

The delegator's principal is safe, but every epoch's yield is captured by the staker until the delegator exits (which itself requires waiting through the `exit_wait_window`). [7](#0-6) [8](#0-7) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**  
A malicious staker can redirect up to 100 % of delegated-pool rewards to themselves for every epoch the delegator remains in the pool. The delegator's staked principal is not lost, but all accrued yield is stolen. The attack is bounded only by the `max_commission` the staker chose to publish in their commitment, which can legally be 10 000 (100 %).

---

### Likelihood Explanation

**Medium.**  
The precondition is that the staker has an active `CommissionCommitment` with a `max_commission` above the current commission. This commitment is on-chain and technically inspectable, but:
- Most delegators rely on front-end UIs that display only the current commission, not the commitment ceiling.
- The staker can set the commitment long before attracting delegators, making the timing gap between "commitment set" and "commission raised" arbitrarily large.
- There is no protocol-level warning or delay between `set_commission_commitment` and the first upward `set_commission` call.
- The delegator has no atomic way to enter the pool and simultaneously assert `commission <= threshold`.

---

### Recommendation

Add an optional `max_commission: Option<Commission>` parameter to both `enter_delegation_pool` and `add_to_delegation_pool`. At the start of each function, read the current commission from the staking contract and assert it does not exceed the caller-supplied ceiling:

```rust
if let Option::Some(max_comm) = max_commission {
    let current_commission = self.get_commission_from_staking_contract();
    assert!(current_commission <= max_comm, "{}", Error::COMMISSION_TOO_HIGH);
}
```

This mirrors the standard AMM slippage-protection pattern (`minAmountOut`) and gives delegators an atomic, trustless guarantee about the fee they are accepting.

---

### Proof of Concept

```
// 1. Staker initialises with low commission
staking.set_commission(commission: 500);          // 5 %

// 2. Staker publishes a commitment allowing up to 100 %
staking.set_commission_commitment(
    max_commission: 10_000,
    expiration_epoch: current_epoch + 200,
);

// 3. Delegator enters pool — no max_commission guard
pool.enter_delegation_pool(
    reward_address: delegator_reward,
    amount: 1_000_000_000_000_000_000,   // 1 STRK
);

// 4. Staker immediately raises commission to 100 %
staking.set_commission(commission: 10_000);       // 100 %

// 5. Staker attests; pool receives 0 net rewards
attestation.attest(block_hash);

// 6. Delegator claims — receives 0
let rewards = pool.claim_rewards(pool_member: delegator);
assert!(rewards == 0);   // delegator's entire yield stolen
```

            ref self: ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Amount {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            assert!(
                caller_address == pool_member || caller_address == pool_member_info.reward_address,
                "{}",
                Error::CALLER_CANNOT_ADD_TO_POOL,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);

            // Transfer funds from the delegator to the staking contract.
            let token_dispatcher = self.token_dispatcher.read();
            let staker_address = self.staker_address.read();
            transfer_from_delegator(pool_member: caller_address, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            // Update the pool member's balance checkpoint.
            let old_delegated_stake = self.increase_member_balance(:pool_member, :amount);
            let new_delegated_stake = old_delegated_stake + amount;

            // Emit events.
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );

            new_delegated_stake
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
