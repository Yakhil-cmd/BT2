### Title
Unauthorized burn of native bank tokens from arbitrary address via `BankContract.burn` precompile — (`x/cronos/keeper/precompiles/bank.go`)

### Summary
The `BankContract.Run` handler for `BurnMethodName` takes the burn target address from ABI argument `args[0]` with no check that it equals `contract.Caller()`. Any EVM contract can burn `evm/{itself}` denomination tokens from any holder's account without that holder's consent.

### Finding Description
`BankContract.Run` handles `MintMethodName` and `BurnMethodName` in a shared branch. For burn, the address to debit is `args[0]` (caller-supplied), while the denom is `evm/{contract.Caller()}`. The missing invariant is:

```go
// MISSING guard:
if recipient != contract.Caller() {
    return nil, errors.New("burn: caller must be the token holder")
}
```

Without this, `AttackerContract` can call `burn(victimAddr, amount)` and the precompile will execute `SendCoinsFromAccountToModule(ctx, victimAddr, ...)` — draining `evm/{AttackerContract}` tokens from `victimAddr` irreversibly. [3](#0-2) 

### Impact Explanation
- **Critical**: Unauthorized, irreversible destruction of native Cosmos bank balances belonging to an arbitrary address.
- The attacker controls the denom (it is `evm/{AttackerContract}`) and can first mint tokens to a victim (via `mint(victimAddr, amount)`, which also has no consent check), then burn them at will.
- Any `evm/`-prefixed balance a victim holds for a given contract is fully at that contract's mercy.

### Likelihood Explanation
Exploitable by any deployed EVM contract with zero preconditions beyond the victim holding a balance of `evm/{attackerContract}`. The attacker can manufacture that precondition themselves by calling `mint(victimAddr, ...)` first (also unchecked). No admin keys, governance, or special permissions required.

### Recommendation
Add a caller-equality guard before the burn (and symmetrically for transfer's `sender`):

```go
if recipient != contract.Caller() {
    return nil, errors.New("burn: can only burn own tokens")
}
```

For `transfer`, enforce `sender == contract.Caller()` or implement an ERC20-style allowance mechanism at the precompile level.

### Proof of Concept
1. Deploy `AttackerContract` (address `0xATK`).
2. From `AttackerContract`, call `IBankModule(bankPrecompile).mint(victimAddr, 1000)` → victim now holds `evm/0xATK: 1000`.
3. From `AttackerContract`, call `IBankModule(bankPrecompile).burn(victimAddr, 1000)`.
4. Precompile executes `SendCoinsFromAccountToModule(ctx, victimAddr, evm_module, {evm/0xATK: 1000})` then `BurnCoins` — victim's balance is destroyed with no signature or approval from `victimAddr`. [4](#0-3)
