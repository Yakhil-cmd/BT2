### Title
Missing Validation in `assert_reshare_keys_invariants` Allows New Participant to Abort Reshare for All Honest Parties — (File: src/dkg.rs)

---

### Summary

`assert_reshare_keys_invariants` in `src/dkg.rs` contains a comment describing a required validation ("if me is not in the old participant set then ensure that `old_signing_key` is None") but the code only enforces the **opposite** direction. A malicious new participant (not in `old_participants`) can supply a non-`None` `old_signing_key`, bypass the public-API guard, and cause the reshare protocol to abort before any messages are sent — permanently stalling all honest participants who are waiting for that party's round messages.

---

### Finding Description

In `assert_reshare_keys_invariants` the comment and the guard are mismatched:

```rust
// if me is not in the old participant set then ensure that old_signing_key is None
if old_participants.contains(me) && old_signing_key.is_none() {   // ← wrong direction
    return Err(InitializationError::BadParameters(...));
}
``` [1](#0-0) 

The comment describes the intended invariant — a **new** participant must not supply a share — but the code only rejects an **old** participant that omits a share. The symmetric check is absent:

```rust
// MISSING:
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(...));
}
```

Because the public-API guard is silent, execution proceeds into `do_reshare`. There, the secret is computed as:

```rust
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(<C::Group as Group>::Field::zero);
``` [2](#0-1) 

`me` is not in `intersection`, but `lagrange` does not require the queried point to be a member of the list — it evaluates the Lagrange basis polynomial at any scalar. The result is a non-zero `secret`.

`do_keyshare` is then called with this non-zero secret and the `old_reshare_package`. The internal guard `assert_keyshare_inputs` catches the contradiction:

```rust
} else {
    //  return error if me is part of the old participants set
    if !old_participants.contains(me) {
        return Err(ProtocolError::AssertionFailed(
            format!("{me:?} is running Resharing with a non-zero share but does not belong to the old participant set")));
    }
}
``` [3](#0-2) 

This error fires **before** any network messages are sent. The malicious participant's coroutine terminates silently. Every honest participant that has already entered `do_keyshare` is now blocked waiting for round messages from the absent party, with no mechanism to distinguish a crash from a deliberate abort.

---

### Impact Explanation

**High — Permanent denial of reshare for honest parties.**

All honest participants who have committed to the reshare session are left waiting indefinitely for messages from the aborting party. Because the abort happens before the first broadcast round, no partial output is produced and the session cannot be salvaged without restarting from scratch. If the malicious participant is always included in the new participant set (a realistic assumption when the attacker controls one of the invited new members), they can repeat this attack on every reshare attempt, permanently blocking key rotation for the honest majority.

---

### Likelihood Explanation

**Medium.** The attacker only needs to be a legitimately invited new participant and to pass a non-`None` `SigningShare` to the public `reshare` API. No cryptographic material needs to be forged or leaked. The missing guard is a single-line omission that is invisible to callers reading the comment, making accidental triggering unlikely but deliberate exploitation straightforward.

---

### Recommendation

Add the missing symmetric check immediately after the existing guard in `assert_reshare_keys_invariants`:

```rust
// if me is not in the old participant set then ensure that old_signing_key is None
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
// ADD: converse check
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not present in the old participant list but provided a share"
    )));
}
``` [1](#0-0) 

Additionally, add a unit test that calls `reshare` with a new participant supplying a non-`None` `old_signing_key` and asserts that `InitializationError::BadParameters` is returned before the protocol starts.

---

### Proof of Concept

1. A reshare session is initiated with old participants `{Alice, Bob, Carol}` (threshold 2) and new participants `{Alice, Bob, Carol, Dave}`.
2. Dave is a new participant (not in `old_participants`).
3. Dave calls `reshare(...)` with `old_signing_key = Some(arbitrary_share)`.
4. `assert_reshare_keys_invariants` passes — the missing guard does not fire.
5. `do_reshare` computes `secret = lagrange(Dave, intersection) * arbitrary_share ≠ 0`.
6. `do_keyshare` calls `assert_keyshare_inputs`; the non-zero secret with Dave absent from `old_participants` triggers `ProtocolError::AssertionFailed`.
7. Dave's coroutine exits without sending any messages.
8. Alice, Bob, and Carol are blocked waiting for Dave's round-1 broadcast, stalling the entire reshare session. [4](#0-3) [5](#0-4)

fn assert_keyshare_inputs<C: Ciphersuite>(
    me: Participant,
    secret: &Scalar<C>,
    old_reshare_package: Option<(VerifyingKey<C>, ParticipantList)>,
) -> Result<(Option<VerifyingKey<C>>, Option<ParticipantList>), ProtocolError> {
    let is_zero_secret = *secret == <C::Group as Group>::Field::zero();

    if let Some((old_key, old_participants)) = old_reshare_package {
        if is_zero_secret {
            //  return error if me is not a purely new joiner to the participants set
            //  prevents accidentally calling keyshare with extremely old keyshares
            //  that have nothing to do with the current resharing
            if old_participants.contains(me) {
                return Err(ProtocolError::AssertionFailed(
                    format!("{me:?} is running Resharing with a zero share but does belong to the old participant set")));
            }
        } else {
            //  return error if me is part of the old participants set
            if !old_participants.contains(me) {
                return Err(ProtocolError::AssertionFailed(
                    format!("{me:?} is running Resharing with a non-zero share but does not belong to the old participant set")));
            }
        }
        Ok((Some(old_key), Some(old_participants)))
    } else {
        if is_zero_secret {
            return Err(ProtocolError::AssertionFailed(format!(
                "{me:?} is running DKG with a zero share"
            )));
        }
        Ok((None, None))
    }
}
