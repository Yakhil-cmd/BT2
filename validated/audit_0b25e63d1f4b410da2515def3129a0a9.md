### Title
Native ETH sent to `tick()` is permanently frozen when no minting is required - (File: `L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

`RewardSupplier.tick()` is `external payable` and accepts ETH to cover L1→L2 bridge and messaging fees. When `requiredMinting()` returns zero (no pending L2 mint requests), the entire fee-spending block is skipped and `msg.value` is neither used nor refunded, permanently locking the caller's ETH in the contract.

---

### Finding Description

`tick()` is declared `external payable` and callers are expected to supply ETH to fund two cross-chain operations:

1. `bridge().depositWithMessage{value: msgFee}(...)` — sends minted tokens to L2 via StarkGate.
2. `messagingContract().sendMessageToL2{value: msgFee}(...)` — sends a `totalSupply` update to the L2 minting curve.

Both calls are gated behind `if (amountToMint > 0)`:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ... only path that spends msg.value
        uint256 msgFee = msg.value / 2;
        bridge().depositWithMessage{value: msgFee}(...);
        msgFee = msg.value - msgFee;
        messagingContract().sendMessageToL2{value: msgFee}(...);
    }
    // ← no else branch; msg.value silently stays in the contract
}
``` [1](#0-0) 

`requiredMinting()` returns `(0, 0)` whenever `messagingContract().l2ToL1Messages(msgHash) == 0`, i.e., whenever there are no pending L2 mint-request messages. [2](#0-1) 

When that condition holds, `tick()` returns without spending or refunding `msg.value`. The contract has no `receive()`, `fallback()`, or ETH-withdrawal function anywhere in `RewardSupplier.sol` or `RewardSupplierStorage.sol`, so the ETH is irrecoverable. [3](#0-2) 

---

### Impact Explanation

Any ETH attached to a `tick()` call that finds `amountToMint == 0` is permanently frozen in the `RewardSupplier` contract. There is no withdrawal path. This constitutes permanent loss of the caller's funds — matching **High: Permanent freezing of funds / Temporary freezing of funds**.

---

### Likelihood Explanation

`tick()` has no access control; any public caller can invoke it. The condition `amountToMint == 0` is the normal state between reward epochs — the L2 reward supplier only sends mint requests when its balance falls below a threshold, so most `tick()` calls in practice will hit the zero-minting path. A caller who mistakenly (or routinely) attaches ETH to such a call loses it permanently.

---

### Recommendation

Add a refund branch when `amountToMint == 0`:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ... existing fee-spending logic ...
    } else if (msg.value > 0) {
        (bool ok, ) = msg.sender.call{value: msg.value}("");
        require(ok, "ETH_REFUND_FAILED");
    }
}
```

Alternatively, remove `payable` from `tick()` entirely and require callers to supply ETH only when `amountToMint > 0` (which they can check via the public `requiredMinting()` view first).

---

### Proof of Concept

1. Call `requiredMinting()` — it returns `(0, 0)` because no L2 mint requests are pending.
2. Call `tick{value: 1 ether}()`.
3. The `if (amountToMint > 0)` branch is skipped.
4. `tick()` returns. The 1 ETH is now held by `RewardSupplier` with no recovery path.
5. Confirm: `address(rewardSupplier).balance == 1 ether`; no function exists to withdraw it. [1](#0-0)

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
