### Title
Unprivileged Caller Can Permanently Grief Consensus Reward Distribution via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract enforces a **global** one-call-per-block limit via a single `last_reward_block` storage variable. Because the function is publicly callable with a caller-controlled `disable_rewards` parameter and no access control, any unprivileged actor can call it with `disable_rewards: true` each block to consume the global slot without distributing rewards, permanently blocking all stakers from receiving consensus block rewards.

---

### Finding Description

`update_rewards` is exposed on the public `IStakingRewardsManager` interface with no caller restriction beyond `general_prerequisites` (contract not paused, caller non-zero): [1](#0-0) 

The implementation enforces a **global** per-block gate using a single `last_reward_block` field: [2](#0-1) 

Specifically, the gate and the early-return path: [3](#0-2) 

The critical sequence is:

1. The function asserts `current_block_number > last_reward_block` — a **global** check, not per-staker.
2. It writes `current_block_number` into `last_reward_block` unconditionally.
3. If `disable_rewards` is `true`, it returns immediately without distributing any rewards.

Because `last_reward_block` is a single `BlockNumber` (not a `Map<ContractAddress, BlockNumber>`), the first successful call in any block exhausts the slot for **all** stakers. An attacker who calls `update_rewards(any_valid_active_staker, disable_rewards: true)` at the start of every block will:

- Pass all validity checks (requires only a valid, active staker with non-zero STRK balance — trivially found from on-chain events).
- Advance `last_reward_block` to the current block.
- Return without distributing rewards.
- Cause every subsequent legitimate call in that block to revert with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

---

### Impact Explanation

Every staker in the protocol is denied consensus block rewards for each block the attacker griefs. Sustained over time this constitutes **permanent freezing of unclaimed yield** for all stakers — matching the High-severity impact class. Even if performed intermittently it constitutes temporary freezing of unclaimed yield. The attacker gains nothing; the sole purpose is damage to stakers and the protocol.

---

### Likelihood Explanation

The attack costs one cheap Starknet transaction per block. The attacker needs only a valid active staker address, which is publicly observable from `NewStaker` events. There is no economic barrier to sustaining this indefinitely. No privileged role, leaked key, or external dependency is required.

---

### Recommendation

1. **Add access control**: restrict `update_rewards` so only the staker themselves or their operational address may call it.
2. **Remove the public `disable_rewards` flag**: if the parameter is needed internally (e.g., during K-epoch transition), make it an internal function rather than a public interface entry point.
3. **Alternatively**, make `last_reward_block` a per-staker mapping (`Map<ContractAddress, BlockNumber>`) so one staker's slot cannot block another's.

---

### Proof of Concept

```
1. Attacker observes any valid, active staker address `staker_A` with non-zero STRK balance
   (e.g., from a NewStaker event).

2. At the start of every block, attacker submits:
       staking_contract.update_rewards(staker_address: staker_A, disable_rewards: true)

3. Inside update_rewards:
   - current_block_number > last_reward_block  →  passes (new block)
   - staker_A is active with non-zero balance  →  passes
   - last_reward_block := current_block_number  (global slot consumed)
   - disable_rewards == true  →  returns, no rewards distributed

4. The legitimate staker (or any other staker) calls:
       staking_contract.update_rewards(staker_address: staker_X, disable_rewards: false)
   → reverts: REWARDS_ALREADY_UPDATED

5. No staker receives consensus block rewards for that block.

6. Repeated every block → permanent freezing of all consensus reward distribution.
``` [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
