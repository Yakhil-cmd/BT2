### Title
`KeygenOutput` and Presign Structs Derive `Debug`, Enabling Disclosure of Private Signing Shares and Nonce Secrets via Formatted Output — (File: `src/lib.rs`, `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

The primary secret-bearing structs in the library — `KeygenOutput<C>` (holding `private_share`), `PresignArguments` (embedding `KeygenOutput`), and both OT-based and Robust `PresignOutput` (holding secret nonce scalars) — all derive the `Debug` trait. Any code path that formats these structs with `{:?}` (error messages, logging, panic output, test assertions) will emit the raw private signing share or presign nonce secrets in plaintext. This is the direct analog to the external report's "sensitive information passed in command line arguments" vulnerability: the library's public API makes it trivially easy for a caller to accidentally disclose secret material through formatted output channels.

---

### Finding Description

At `src/lib.rs:48`, `KeygenOutput<C>` is defined as:

```rust
#[derive(Debug, Clone, Deserialize, Serialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct KeygenOutput<C: Ciphersuite> {
    pub private_share: SigningShare<C>,
    #[zeroize[skip]]
    pub public_key: VerifyingKey<C>,
}
```

The `Debug` derive means `{:?}` formatting on `KeygenOutput` will print `private_share`. The library's own snapshot tests confirm that `SigningShare` serializes to the raw scalar bytes as a hex string — e.g., `"private_share":"add8bea105e6085a3f9d4ee613834280..."` in `src/ecdsa/snapshots/threshold_signatures__ecdsa__test__keygen_determinism.snap` — so `Debug` output exposes the same raw scalar.

`PresignArguments` at `src/ecdsa/ot_based_ecdsa/mod.rs:23` also derives `Debug` and directly embeds `keygen_out: KeygenOutput`, compounding the exposure:

```rust
#[derive(Debug, Clone)]
pub struct PresignArguments {
    pub triple0: (TripleShare, TriplePub),
    pub triple1: (TripleShare, TriplePub),
    pub keygen_out: KeygenOutput,   // ← contains private_share
    pub threshold: ReconstructionLowerBound,
}
```

Both OT-based `PresignOutput` (`src/ecdsa/ot_based_ecdsa/mod.rs:40`) and Robust `PresignOutput` (`src/ecdsa/robust_ecdsa/mod.rs:26`) also derive `Debug` and contain secret nonce scalars (`k`, `sigma`, `c`, `e`, `alpha`, `beta`):

```rust
// OT-based
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput {
    pub big_r: AffinePoint,
    pub k: Scalar,      // secret nonce share
    pub sigma: Scalar,  // secret sigma share
}

// Robust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, ZeroizeOnDrop)]
pub struct PresignOutput {
    pub big_r: AffinePoint,
    pub c: Scalar,      // secret
    pub e: Scalar,      // secret
    pub alpha: Scalar,  // secret
    pub beta: Scalar,   // secret
}
```

The same issue applies to `RerandomizedPresignOutput` in both modules (`src/ecdsa/ot_based_ecdsa/mod.rs:54` and `src/ecdsa/robust_ecdsa/mod.rs:42`), which also derive `Debug` and carry the rerandomized secret nonce scalars.

---

### Impact Explanation

**Critical — Extraction or disclosure of private signing shares and presign nonce secrets.**

If any caller uses `{:?}` formatting on `KeygenOutput` or `PresignArguments` (e.g., in error handling, logging, or panic output), the raw private signing share is emitted in plaintext. Exposure of a threshold number of signing shares enables reconstruction of the aggregate private key. Exposure of presign nonce secrets (`k`, `sigma` for OT-based; `alpha`, `beta`, `e` for Robust) enables recovery of the private key from a single completed signature via standard ECDSA nonce-reuse techniques.

---

### Likelihood Explanation

**Medium.** The `Debug` derive is a public API choice that makes it trivially easy for callers to accidentally expose private shares. In practice, Rust error-handling idioms routinely use `{:?}` formatting — `unwrap()`, `expect("msg")`, `format!("{:?}", val)`, `dbg!(val)`, and `assert_eq!` all invoke `Debug`. The library's own snapshot tests demonstrate this pattern by printing `KeygenOutput` values including `private_share` in plaintext. Any integrating application that logs protocol state, surfaces errors to operators, or uses standard Rust debugging tools is at risk.

---

### Recommendation

Remove the `Debug` derive from `KeygenOutput`, `PresignArguments`, `PresignOutput`, and `RerandomizedPresignOutput`. Implement a custom `Debug` that redacts secret fields:

```rust
impl<C: Ciphersuite> fmt::Debug for KeygenOutput<C> {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        f.debug_struct("KeygenOutput")
            .field("private_share", &"[REDACTED]")
            .field("public_key", &self.public_key)
            .finish()
    }
}
```

Apply the same pattern to all presign output structs, redacting `k`, `sigma`, `c`, `e`, `alpha`, and `beta`.

### Proof of Concept

```rust
// Caller accidentally logs KeygenOutput during error handling:
let keygen_output: KeygenOutput<Secp256K1Sha256> = run_keygen(...);
// Any of the following leaks private_share in plaintext:
println!("{:?}", keygen_output);
// → KeygenOutput { private_share: SigningShare { scalar: Scalar(0xadd8bea1...) }, ... }

eprintln!("keygen result: {:?}", keygen_output);

// PresignArguments leaks private_share transitively:
let args = PresignArguments { keygen_out: keygen_output, ... };
log::debug!("{:?}", args);
// → PresignArguments { ..., keygen_out: KeygenOutput { private_share: ... }, ... }

// PresignOutput leaks nonce secrets:
let presign: ot_based_ecdsa::PresignOutput = run_presign(...);
println!("{:?}", presign);
// → PresignOutput { big_r: ..., k: Scalar(0x...), sigma: Scalar(0x...) }
