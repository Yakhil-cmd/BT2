### Title
Unverified Caller Can Invoke `update_rewards` With `disable_rewards: true` to Permanently Freeze Consensus Rewards for All Stakers — (`File: src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts an arbitrary `staker_address` and a caller-controlled `disable_rewards: bool` flag. Any unprivileged address can call it with `disable_rewards: true` every block, consuming the single per-block reward slot (enforced by the global `last_reward_block`) without distributing any rewards. This permanently freezes unclaimed consensus-epoch yield for all stakers.

---

### Finding Description

`update_rewards` is defined in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // exits without distributing rewards
    }
    ...
}
``` [1](#0-0) 

`general_prerequisites()` only asserts the contract is not paused and the caller is non-zero — there is no check that the caller is the staker, the staker's reward address, or any privileged role: [2](#0-1) 

`last_reward_block` is a **global** storage slot shared across all stakers: [3](#0-2) 

The assertion `current_block_number > self.last_reward_block.read()` means only **one** `update_rewards` call can succeed per block across the entire protocol: [4](#0-3) 

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` first in every block:
1. Passes all checks (staker exists, is active, has non-zero balance).
2. Writes `current_block_number` into `last_reward_block`.
3. Returns immediately without distributing rewards.
4. Causes every subsequent legitimate `update_rewards` call in that block to revert with `REWARDS_ALREADY_UPDATED`.

---

### Impact Explanation

This matches **High: Permanent freezing of unclaimed yield**. Consensus-era block rewards (`_update_rewards` path) are only distributed through `update_rewards`. If an attacker front-runs this call every block with `disable_rewards: true`, no staker ever accumulates consensus rewards. The yield is not merely delayed — it is permanently lost because block rewards are calculated per-block and not retroactively redistributed. [5](#0-4) 

---

### Likelihood Explanation

- Requires no special role, no staked funds, no privileged key — any EOA can execute the attack.
- The attacker only needs to submit one transaction per block, which is cheap relative to the value of staking rewards across the entire protocol.
- The attack is fully deterministic and requires no oracle or external dependency.
- The entry point is a public ABI function callable by any L2 address.

---

### Recommendation

1. **Remove the `disable_rewards` parameter from the public interface**, or restrict it to a privileged role (e.g., `only_security_agent`).
2. Alternatively, restrict `update_rewards` so that only the staker themselves, their reward address, or a designated operator role can call it for a given `staker_address`.
3. Consider making `last_reward_block` per-staker rather than global, so one attacker call cannot block all stakers simultaneously.

---

### Proof of Concept

```
// Attacker script (pseudocode, executed every block):
loop every block B:
    // Pick any valid staker address S
    staking_contract.update_rewards(staker_address=S, disable_rewards=true)
    // Effect: last_reward_block = B, no rewards distributed
    // Any legitimate update_rewards call in block B now reverts with REWARDS_ALREADY_UPDATED
```

Concrete call path:
1. Attacker (any address) → `Staking::update_rewards(valid_staker, disable_rewards=true)`
2. `general_prerequisites()` passes (not paused, caller ≠ 0). [6](#0-5) 
3. `current_block_number > last_reward_block` passes (first call this block). [4](#0-3) 
4. Staker existence and balance checks pass. [7](#0-6) 
5. `last_reward_block` written to current block. [8](#0-7) 
6. `disable_rewards == true` → early return, no rewards distributed. [9](#0-8) 
7. All subsequent `update_rewards` calls this block revert. Repeated every block → permanent yield freeze.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
