### Title
Stale Granted Permissions Persist After `CronosAdmin` Rotation, Allowing Unauthorized `MsgUpdateTokenMapping` Execution - (File: `x/cronos/keeper/permissions.go`, `x/cronos/keeper/msg_server.go`)

### Summary

When governance rotates `CronosAdmin` via `MsgUpdateParams`, all per-address permissions previously granted by the old admin remain active in the KV store. Any address that was granted `CanChangeTokenMapping` by the old admin can still call `MsgUpdateTokenMapping` successfully under the new admin, because `SetParams` never clears the permissions store. This is the direct Cronos analog of the external `__abdicate` / `pendingGov` bug: a privileged state variable is not reset when authority is transferred.

---

### Finding Description

Cronos implements a two-layer permission system:

1. **`CronosAdmin`** — a single address in module params, changeable only via governance through `MsgUpdateParams` → `SetParams`.
2. **Per-address permissions** — bitmask values stored persistently in the KV store under `AdminToPermissionsKey(address)`, writable only by the current `CronosAdmin` via `MsgUpdatePermissions`.

`HasPermission` grants access if **either** the caller is the current `CronosAdmin` **or** the caller's KV-stored permission bitmask satisfies the required bit:

```go
// x/cronos/keeper/permissions.go
func (k Keeper) HasPermission(...) bool {
    admin := k.GetParams(ctx).CronosAdmin
    for _, account := range accounts {
        if admin == account.String() {   // check 1: current admin
            return true
        }
        permission := k.GetPermissions(ctx, account)
        if permission&permissionsToCheck == permissionsToCheck { // check 2: KV store
            return true
        }
    }
    return false
}
```

`MsgUpdateTokenMapping` relies entirely on this check:

```go
// x/cronos/keeper/msg_server.go
func (k msgServer) UpdateTokenMapping(...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, ...)
    }
    if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil { ... }
}
```

`SetParams` (called by `MsgUpdateParams`) only overwrites the params blob — it never touches the permissions KV store:

```go
// x/cronos/keeper/params.go
func (k Keeper) SetParams(ctx sdk.Context, params types.Params) error {
    if err := params.Validate(); err != nil { return err }
    store := ctx.KVStore(k.storeKey)
    bz := k.cdc.MustMarshal(&params)
    store.Set(types.ParamsKey, bz)   // only params key written
    return nil
}
```

**Root cause:** There is no mechanism to clear or invalidate per-address permissions when `CronosAdmin` changes. Permissions granted by the old admin survive indefinitely.

---

### Impact Explanation

An address that was granted `CanChangeTokenMapping` by the old admin retains that permission after admin rotation. It can call `MsgUpdateTokenMapping` → `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom`, writing an arbitrary denom→contract mapping into the KV store. This corrupts the denom/contract binding that governs IBC voucher conversion and bridge flows — a direct High-severity impact under the allowed scope ("Corruption of token mappings, denom/contract binding, IBC channel/accounting state... with direct security impact").

Concretely, the attacker can:
- Remap a live IBC denom (e.g., `ibc/...`) to a malicious ERC20 contract they control
- All subsequent `OnRecvVouchers` calls for that denom will mint tokens into the attacker's contract instead of the legitimate one, enabling theft of bridged assets

---

### Likelihood Explanation

Admin rotation is an expected, documented governance operation (the `CronosAdmin` param "can be updated at runtime"). The old admin may have legitimately granted permissions to operational addresses (e.g., automated relayers, circuit-breaker bots). After rotation, those addresses retain full `CanChangeTokenMapping` authority. The new admin has no way to enumerate all previously-permissioned addresses (no iterator is exposed through the msg server), making silent revocation impossible. The scenario requires only that the old admin had previously called `MsgUpdatePermissions` for at least one address — a normal operational action.

---

### Recommendation

`SetParams` (or `MsgUpdateParams` handler) should detect a `CronosAdmin` change and clear all KV-stored permissions when the admin address changes:

```go
func (k Keeper) SetParams(ctx sdk.Context, params types.Params) error {
    if err := params.Validate(); err != nil {
        return err
    }
    // If admin is rotating, clear all stale per-address permissions
    old := k.GetParams(ctx)
    if old.CronosAdmin != params.CronosAdmin {
        k.ClearAllPermissions(ctx)
    }
    store := ctx.KVStore(k.storeKey)
    bz := k.cdc.MustMarshal(&params)
    store.Set(types.ParamsKey, bz)
    return nil
}
```

Alternatively, `HasPermission` should require that the granting admin still matches the current admin before honoring KV-stored permissions (e.g., by storing the granting admin alongside each permission entry).

---

### Proof of Concept

1. Governance sets `CronosAdmin = AdminA` via `MsgUpdateParams`.
2. `AdminA` calls `MsgUpdatePermissions{from: AdminA, address: AttackerAddr, permissions: 1}` — granting `CanChangeTokenMapping` to `AttackerAddr`. [1](#0-0) 
3. Governance passes a new proposal: `MsgUpdateParams` with `CronosAdmin = AdminB`. `SetParams` writes the new params but does not touch the permissions KV store. [2](#0-1) 
4. `AttackerAddr` submits `MsgUpdateTokenMapping{sender: AttackerAddr, denom: "ibc/...", contract: MaliciousContract}`.
5. `HasPermission` reads `CronosAdmin = AdminB` (no match), then reads `GetPermissions(AttackerAddr) = 1` (match) — returns `true`. [3](#0-2) 
6. `RegisterOrUpdateTokenMapping` writes the malicious denom→contract mapping. [4](#0-3) 
7. All subsequent IBC voucher conversions for that denom are routed to the attacker's contract. [5](#0-4)

func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil

// RegisterOrUpdateTokenMapping update the token mapping, register a coin metadata if needed
func (k Keeper) RegisterOrUpdateTokenMapping(ctx sdk.Context, msg *types.MsgUpdateTokenMapping) error {
	if types.IsSourceCoin(msg.Denom) {
		_, err := types.GetContractAddressFromDenom(msg.Denom)
		if err != nil {
			return err
		}

		if !common.IsHexAddress(msg.Contract) {
			return errors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid contract address (%s)", msg.Contract)
		}
		contract := common.HexToAddress(msg.Contract)
		if err := k.ensureContractCode(ctx, contract); err != nil {
			return err
		}
		if err := validateContractAddressForSourceDenom(msg.Denom, contract, true); err != nil {
			return err
		}

		if err := k.SetExternalContractForDenom(ctx, msg.Denom, contract); err != nil {
			return err
		}

		// check that the coin is registered, otherwise register it
		metadata, exist := k.bankKeeper.GetDenomMetaData(ctx, msg.Denom)
		if !exist {
			// create new metadata
			metadata = banktypes.Metadata{
				Base: msg.Denom,
				Name: msg.Denom,
			}
		}
		// update existing metadata
		metadata.Symbol = msg.Symbol
		metadata.Display = strings.ToLower(msg.Symbol)
		if msg.Decimal != 0 {
			metadata.DenomUnits = []*banktypes.DenomUnit{
				{
					Denom:    metadata.Base,
					Exponent: 0,
				},
				{
					Denom:    metadata.Display,
					Exponent: msg.Decimal,
				},
			}
		} else {
			metadata.DenomUnits = []*banktypes.DenomUnit{
				{
					Denom:    metadata.Base,
					Exponent: 0,
				},
			}
		}
		k.bankKeeper.SetDenomMetaData(ctx, metadata)
	} else {
		if len(msg.Contract) == 0 {
			// delete existing mapping
			k.DeleteExternalContractForDenom(ctx, msg.Denom)
		} else {
			if !common.IsHexAddress(msg.Contract) {
				return errors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid contract address (%s)", msg.Contract)
			}
			// update the mapping
			contract := common.HexToAddress(msg.Contract)
			if err := k.ensureContractCode(ctx, contract); err != nil {
				return err
			}
			if err := k.SetExternalContractForDenom(ctx, msg.Denom, contract); err != nil {
				return err
			}
		}
	}

	return nil
