### Title
Unrestricted `update_rewards` Allows Any Caller to Permanently Freeze All Staker Yield - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a `disable_rewards: bool` parameter and writes to the global `last_reward_block` storage slot on every call. Because there is no caller restriction, any unprivileged address can call `update_rewards(any_staker, disable_rewards: true)` once per block, consuming the single per-block reward slot without distributing any rewards. Repeating this every block permanently denies consensus-era block rewards to every staker in the protocol.

### Finding Description
`update_rewards` is exposed as part of the `IStakingRewardsManager` public interface with no access-control guard beyond `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function enforces a single global `last_reward_block` check: the current block number must be strictly greater than the stored value. On success it immediately writes the current block number back to storage. [2](#0-1) 

After updating `last_reward_block`, the function checks `disable_rewards || self.is_pre_consensus()` and returns early without distributing anything if either is true. [3](#0-2) 

Because `last_reward_block` is a single global slot shared across all stakers, one call per block with `disable_rewards: true` exhausts the per-block reward window for every staker simultaneously. [4](#0-3) 

The `general_prerequisites` helper imposes no role or identity check: [5](#0-4) 

The interface confirms the function is fully public: [6](#0-5) 

### Impact Explanation
During the consensus-rewards era (`is_pre_consensus() == false`), block rewards are the sole mechanism by which stakers and pool members accumulate yield. An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` at the start of every block:

1. Marks the block as "already processed" in `last_reward_block`.
2. Returns without crediting any staker's `unclaimed_rewards_own` or forwarding pool rewards.
3. Causes every subsequent legitimate `update_rewards` call in that block to revert with `REWARDS_ALREADY_UPDATED`.

Sustained over time this permanently freezes all unclaimed yield for all stakers and pool members across the entire protocol. This matches the **High: Permanent freezing of unclaimed yield** impact category.

### Likelihood Explanation
The entry path requires no tokens, no stake, no privileged role, and no special setup — only the ability to submit a transaction. The attacker must call the function once per block, which is a low-cost, repeatable operation on Starknet. Any motivated adversary (e.g., a competing validator, a griefing actor) can execute this continuously.

### Recommendation
Restrict `update_rewards` so that only an authorized caller — such as the Sequencer, a designated keeper role, or the staker themselves — can invoke it. The simplest fix is to add a role check (e.g., `only_app_governor` or a new `REWARDS_UPDATER_ROLE`) inside `update_rewards`, or to remove the `disable_rewards` parameter from the public interface and expose it only through an internal/privileged path.

### Proof of Concept

```
// Attacker (any EOA or contract) executes this every block:
//
// 1. Pick any currently-active staker address (readable from public `get_stakers`).
// 2. Call staking_contract.update_rewards(active_staker_address, disable_rewards: true).
//    - `general_prerequisites()` passes (contract unpaused, caller non-zero).
//    - `current_block_number > last_reward_block` passes (first call this block).
//    - `last_reward_block` is written to `current_block_number`.
//    - `disable_rewards == true` → early return, zero rewards distributed.
// 3. Any legitimate call to `update_rewards` later in the same block reverts:
//    assert!(current_block_number > last_reward_block) → REWARDS_ALREADY_UPDATED.
//
// Repeat step 2 every block.
// Result: `unclaimed_rewards_own` for every staker remains zero indefinitely;
//         pool `cumulative_rewards_trace` is never updated;
//         all staker and delegator yield is permanently frozen.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
