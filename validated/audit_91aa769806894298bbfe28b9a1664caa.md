### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Consensus Rewards - (File: `src/staking/staking.cairo`)

### Summary

The `update_rewards` function in the staking contract has no access control. Any unprivileged caller can invoke it with `disable_rewards: true`, consuming the global `last_reward_block` slot for the current block without distributing any rewards. This permanently blocks all stakers from receiving consensus (V3) block rewards for that block. Repeated every block, it freezes all unclaimed yield indefinitely.

---

### Finding Description

`update_rewards` is implemented in `StakingRewardsManagerImpl` at [1](#0-0)  and exposed via the `IStakingRewardsManager` interface at [2](#0-1) .

The function body:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // no rewards distributed
    }
    ...
``` [3](#0-2) 

Key observations:

1. **No access control** — `general_prerequisites()` only checks the pause flag; there is no `assert_caller_is_attestation_contract()` or equivalent. [4](#0-3) 

2. **Global `last_reward_block`** — The guard and the write both operate on a single contract-wide storage slot (no per-staker key). One call blocks every staker for that block. [5](#0-4) 

3. **`disable_rewards: true` skips distribution** — After writing `last_reward_block`, the function returns early with zero rewards distributed. [6](#0-5) 

**Attack path:**

1. Attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at block N.
2. `last_reward_block` is set to N; no rewards are minted or credited.
3. Any legitimate call to `update_rewards` at block N now reverts with `REWARDS_ALREADY_UPDATED`.
4. All stakers lose their consensus block rewards for block N.
5. Repeat every block → stakers never accumulate consensus rewards.

The staker must be valid (exists and active with non-zero balance) for the call to succeed past the staker-existence assertions, but any publicly known staker address satisfies this. [7](#0-6) 

---

### Impact Explanation

The attack permanently freezes all stakers' unclaimed consensus (V3) yield. `last_reward_block` is a global slot; a single call per block is sufficient to deny rewards to every staker in the system. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

Any unprivileged address can execute this. The only cost is one Starknet transaction per block. There is no economic barrier, no special role required, and no way for stakers to defend against it. The attack is trivially automatable.

---

### Recommendation

Add an access-control assertion at the top of `update_rewards` restricting callers to the attestation contract (or the designated consensus-rewards caller), analogous to the guard already present on `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.attestation_contract.read(),
        "{}",
        Error::CALLER_IS_NOT_ATTESTATION_CONTRACT,
    );
    ...
}
```

Alternatively, remove the `disable_rewards` parameter entirely and derive the disable condition internally from attestation state, so the caller cannot influence whether rewards are skipped.

---

### Proof of Concept

```
Block N:
  Attacker → staking.update_rewards(alice_staker, disable_rewards=true)
    → last_reward_block := N
    → returns (no rewards distributed)

  Attestation contract → staking.update_rewards(alice_staker, disable_rewards=false)
    → assert(N > N) FAILS → REWARDS_ALREADY_UPDATED

Block N+1: repeat
```

All stakers' `unclaimed_rewards_own` remain zero indefinitely. The `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` is never called, so no STRK is ever credited. [8](#0-7)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
