### Title
Gateway L2 Gas Price Threshold Uses Floor Division, Admitting Transactions Below the Intended Minimum — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price using `.to_integer()` on a `Ratio<u128>`, which performs **floor (truncating) division**. When `min_gas_price_percentage × previous_block_l2_gas_price` is not evenly divisible by 100, the computed threshold is 1 FRI lower than the intended value. Transactions whose L2 gas price equals the floor value — but is strictly below the mathematically correct ceiling threshold — pass gateway admission even though they violate the configured policy.

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← floor division
if tx_l2_gas_price.0 < threshold {
    return Err(...)
}
```

`Ratio::to_integer()` in `num_rational` is defined as `numer / denom`, i.e., integer truncation (floor for positive values). The intended invariant is "the transaction's L2 gas price must be at least `min_gas_price_percentage`% of the previous block's L2 gas price." Correct enforcement requires ceiling division:

```
threshold = ceil(min_gas_price_percentage × previous_block_l2_gas_price / 100)
```

With floor, the threshold is `⌊X% × P⌋` instead of `⌈X% × P⌉`. Whenever `X × P mod 100 ≠ 0`, the computed threshold is exactly 1 FRI below the true minimum, and the admission check `tx_l2_gas_price.0 < threshold` passes for a gas price that should be rejected.

**Concrete example** (`min_gas_price_percentage = 80`, `previous_block_l2_gas_price = 101`):
- Intended threshold: `⌈80 × 101 / 100⌉ = ⌈80.8⌉ = 81`
- Computed threshold: `⌊80 × 101 / 100⌋ = ⌊80.8⌋ = 80`
- A transaction with `tx_l2_gas_price = 80` passes the check (`80 < 80` is false) even though it is below 80% of the previous block price.

The analog to the external report is direct: the UniV3 bug used only one asset and rounded down, producing an offset too small to cover the actual cost; here, the threshold is rounded down, producing a value too small to enforce the configured minimum, so transactions that should be rejected are admitted.

### Impact Explanation

The gateway's stateful admission check is the last line of defence before a transaction enters the mempool. Transactions admitted with a gas price below the operator-configured minimum:
- Bypass the intended spam/underpricing filter
- Occupy mempool slots and propagate to peers
- May be sequenced into blocks at a price below the network's floor, undermining the fee-market invariant

This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The default `min_gas_price_percentage` is 100, which produces an exact ratio and no rounding error. The bug is triggered whenever an operator sets `min_gas_price_percentage` to any value that is not a divisor of 100 (e.g., 80, 50, 33, …) **and** the previous block's L2 gas price is not divisible by `100 / gcd(percentage, 100)`. Both conditions are routine in production. Any unprivileged user can then submit a transaction at the floor price to exploit the off-by-one admission.

### Recommendation

Replace `.to_integer()` with ceiling division:

```rust
// Before (floor):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();

// After (ceiling):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

`Ratio::ceil()` in `num_rational` returns `⌈numer/denom⌉`, which correctly enforces the "at least X%" invariant.
