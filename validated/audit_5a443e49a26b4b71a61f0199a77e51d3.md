### Title
Stale Delegated Permissions Not Cleared on `CronosAdmin` Rotation — (`x/cronos/keeper/msg_server.go`, `x/cronos/keeper/params.go`)

---

### Summary

When governance rotates `CronosAdmin` via `MsgUpdateParams`, all address-level permissions previously granted by the old admin via `MsgUpdatePermissions` remain live in the KV store. `HasPermission` checks the KV store independently of the current admin, so every previously-permissioned address retains `CanChangeTokenMapping` (and `CanTurnBridge`) indefinitely. A stale permissioned address can then call `MsgUpdateTokenMapping` to delete or corrupt denom-to-contract bindings, breaking IBC voucher conversion flows.

---

### Finding Description

Cronos implements a two-layer permission model:

1. **Admin-level**: `CronosAdmin` in module params has all permissions implicitly.
2. **Delegated-level**: The admin can grant explicit permissions to arbitrary addresses via `MsgUpdatePermissions`, which writes a `uint64` bitmask to the KV store under `AdminToPermissionsKey(address)`.

`HasPermission` checks both layers independently:

```go
// x/cronos/keeper/permissions.go
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
    admin := k.GetParams(ctx).CronosAdmin
    for _, account := range accounts {
        if admin == account.String() {
            return true
        }
        permission := k.GetPermissions(ctx, account)          // KV store lookup
        if permission&permissionsToCheck == permissionsToCheck {
            return true
        }
    }
    return false
}
```

When governance rotates the admin via `MsgUpdateParams`, `SetParams` only overwrites the params entry — it never touches the permissions KV store:

```go
// x/cronos/keeper/params.go
func (k Keeper) SetParams(ctx sdk.Context, params types.Params) error {
    if err := params.Validate(); err != nil {
        return err
    }
    store := ctx.KVStore(k.storeKey)
    bz := k.cdc.MustMarshal(&params)
    store.Set(types.ParamsKey, bz)   // only params written; permissions KV untouched
    return nil
}
```

`UpdateParams` calls only `SetParams`:

```go
// x/cronos/keeper/msg_server.go
func (k msgServer) UpdateParams(goCtx context.Context, msg *types.MsgUpdateParams) (*types.MsgUpdateParamsResponse, error) {
    ...
    if err := k.SetParams(ctx, msg.Params); err != nil {
        return nil, err
    }
    return &types.MsgUpdateParamsResponse{}, nil
}
```

There is no mechanism to enumerate all permissioned addresses (no public KV iterator is exposed), so the new admin cannot easily discover and revoke stale grants.

---

### Impact Explanation

A stale permissioned address X retaining `CanChangeTokenMapping` can call `MsgUpdateTokenMapping` with an empty `contract` field, which triggers the deletion path in `RegisterOrUpdateTokenMapping`:

```go
// x/cronos/keeper/keeper.go
if len(msg.Contract) == 0 {
    // delete existing mapping
    k.DeleteExternalContractForDenom(ctx, msg.Denom)
}
```

Deleting an external contract mapping for an active IBC denom breaks `ConvertVouchersToEvmCoins` for that denom — users can no longer convert IBC vouchers to EVM coins. X can also register new mappings for unmapped denoms, redirecting future conversions to attacker-controlled contracts.

This satisfies:
- **High: Bypass of Cronos admin/governance authority and permission checks** — the new admin's intent (that X should no longer be authorized) is silently violated.
- **High: Corruption of token mappings / denom-contract binding with direct security impact** — active IBC voucher conversion paths can be severed or redirected.

---

### Likelihood Explanation

Requires three conditions, analogous to the ODSafeManager finding:

1. Governance rotates `CronosAdmin` to a new address (a normal operational event).
2. An address previously permissioned by the old admin becomes adversarial or acts outside the new admin's intent.
3. The new admin does not proactively revoke the stale grant (which is difficult since there is no enumeration API for permissioned addresses).

All three conditions are plausible in a real governance transition, especially when the admin key is rotated due to a security incident or organizational change.

---

### Recommendation

In `SetParams` (or in `UpdateParams` before calling `SetParams`), detect when `CronosAdmin` is changing and clear all KV-stored permissions:

```go
func (k Keeper) SetParams(ctx sdk.Context, params types.Params) error {
    if err := params.Validate(); err != nil {
        return err
    }
    oldParams := k.GetParams(ctx)
    if oldParams.CronosAdmin != params.CronosAdmin {
        k.ClearAllPermissions(ctx)   // iterate AdminToPermissionsKey prefix and delete
    }
    store := ctx.KVStore(k.storeKey)
    bz := k.cdc.MustMarshal(&params)
    store.Set(types.ParamsKey, bz)
    return nil
}
```

Alternatively, expose a `GetAllPermissionedAddresses` query so the new admin can enumerate and revoke stale grants manually.

### Proof of Concept

**State before governance proposal:**

```
CronosAdmin = A
AdminToPermissionsKey(X) = CanChangeTokenMapping (1)
AdminToPermissionsKey(Y) = CanChangeTokenMapping | CanTurnBridge (3)
DenomToExternalContract["ibc/ATOM"] = 0xTokenContract
```

**Step 1 — Governance rotates admin:**
```
MsgUpdateParams { authority: gov_module, params: { cronos_admin: B, ... } }
```
`SetParams` writes new params. KV store for X and Y is untouched.

**Step 2 — X (stale permissioned address) calls:**
```
MsgUpdateTokenMapping { sender: X, denom: "ibc/ATOM", contract: "" }
```

`HasPermission` check:
- `admin == X.String()` → false (admin is now B)
- `GetPermissions(X)` → `1` (CanChangeTokenMapping) → **passes**

`RegisterOrUpdateTokenMapping` executes `DeleteExternalContractForDenom("ibc/ATOM")`.

**Result:** The `ibc/ATOM` → `0xTokenContract` mapping is deleted. All subsequent `MsgConvertVouchers` calls for `ibc/ATOM` fail (no contract found), permanently breaking IBC voucher conversion for that denom until the new admin B manually restores the mapping — assuming B even notices the deletion.

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {

func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
}
