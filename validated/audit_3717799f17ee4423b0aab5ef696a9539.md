### Title
Unauthenticated Invoke Admission via `skip_stateful_validations` Allows Any Caller to Occupy a Victim's Nonce Slot - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator skips the `__validate__` entry-point check for any invoke transaction whose nonce is `1` and whose sender address has *any* transaction already in the mempool. Because the check is purely address-based and not tied to the submitter's identity, an unprivileged attacker can submit an unsigned (or arbitrarily signed) invoke transaction targeting a victim's undeployed account address, have it admitted to the mempool without signature verification, and thereby block the victim's legitimate nonce-1 invoke from entering the mempool.

### Finding Description

`skip_stateful_validations` is called from `run_pre_validation_checks` inside `extract_state_nonce_and_run_validations`:

```
extract_state_nonce_and_run_validations
  └─ run_pre_validation_checks
       ├─ validate_state_preconditions   (nonce range, resource bounds)
       ├─ validate_by_mempool            (duplicate hash / nonce ordering)
       └─ skip_stateful_validations      ← returns true → __validate__ is skipped
```

The function's logic is:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  lines 429-461
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                ...
        }
    }
    Ok(false)
}
```

When it returns `true`, `run_validate_entry_point` sets `execution_flags.validate = false`:

```rust
// lines 311-312
let execution_flags =
    ExecutionFlags { only_query, charge_fee, validate: !skip_validate, strict_nonce_check };
```

Inside `StatefulValidator::perform_validations` the `__validate__` call is then unconditionally skipped:

```rust
// crates/blockifier/src/blockifier/stateful_validator.rs  lines 79-81
if !tx.execution_flags.validate {
    return Ok(());
}
```

The three conditions that trigger the bypass are:

| Condition | Attacker control |
|---|---|
| Transaction is `Invoke` | Yes – attacker chooses the type |
| `tx.nonce() == 1` | Yes – attacker sets the nonce field |
| `account_tx_in_pool_or_recent_block(sender)` returns `true` | Indirectly – true as soon as the victim's `deploy_account` lands in the mempool |

`account_tx_in_pool_or_recent_block` checks only whether *any* transaction for that address exists in the pool or was recently committed:

```rust
// crates/apollo_mempool/src/mempool.rs  lines 697-700
pub fn account_tx_in_pool_or_recent_block(&self, account_address: ContractAddress) -> bool {
    self.state.contains_account(account_address)
        || self.tx_pool.contains_account(account_address)
}
```

There is no check that the submitter of the invoke is the same entity that submitted the `deploy_account`. The gateway therefore admits the attacker's invoke into the mempool without ever calling the account's `__validate__` entry point.

### Impact Explanation

The admitted transaction occupies the victim's `(address, nonce=1)` slot in the mempool. The mempool enforces uniqueness per `(address, nonce)` and rejects the victim's legitimate nonce-1 invoke with `MempoolError::DuplicateNonce`. When the batcher later attempts to execute the attacker's transaction, the blockifier *does* call `__validate__` (because `execution_flags` are freshly constructed with `validate = true` for batcher execution), the call fails, and the transaction is rejected. The victim can then resubmit — but the attacker can repeat the attack every block at zero cost (failed-`__validate__` transactions are not charged fees in Starknet). This constitutes a sustained, free, targeted DoS against any account that is in the process of being deployed.

The corrupted admission decision is: **an invoke transaction with an arbitrary or absent valid signature is accepted by the gateway and placed in the mempool**, directly matching the "High – Mempool/gateway/RPC admission accepts invalid transactions" impact category.

### Likelihood Explanation

The attack requires only that the victim's `deploy_account` transaction be visible in the mempool (a public, observable event). The attacker needs no privileged access, no special knowledge of the victim's private key, and no on-chain funds beyond the minimum resource bounds required to pass stateless validation. The window is the entire time the `deploy_account` is pending (potentially many blocks). The attack is repeatable and free.

### Recommendation

The `skip_stateful_validations` bypass was designed for the UX pattern of submitting `deploy_account` + `invoke` atomically. The fix should tie the skip decision to the *same submitter*, not just the same address. Concrete options:

1. **Require the deploy_account and invoke to arrive in the same gateway request** (a paired-submission endpoint), so the gateway can verify both are signed by the same key before admitting either.
2. **Run `__validate__` against a speculative state** in which the pending `deploy_account` has already been applied, so the signature is verified even though the account is not yet on-chain.
3. **Store the deploy_account transaction hash alongside the mempool entry** and require the invoke to reference it, allowing the gateway to confirm the submitter controls the deploying key.

### Proof of Concept

```
1. Alice broadcasts deploy_account(class_hash=C, salt=S, constructor_calldata=[pk_alice])
   → gateway validates __validate_deploy__ (Alice's signature), admits to mempool.

2. Attacker observes alice_addr in the mempool.

3. Attacker sends Invoke { sender_address: alice_addr, nonce: 1, calldata: [...], signature: [garbage] }
   to the gateway.

4. Gateway stateful validation:
   - account_nonce = get_nonce_from_state(alice_addr) == 0   ✓
   - tx.nonce() == 1                                          ✓

   → mempool rejects: MempoolError::DuplicateNonce

6. Batcher executes alice's deploy_account → alice_addr deployed.
   Batcher executes attacker's invoke → __validate__ called → fails (bad sig) → rejected.
   Mempool removes attacker's tx.

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
