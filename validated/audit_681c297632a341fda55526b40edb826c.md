### Title
Missing `msg.value` Validation in `RewardSupplier::tick()` Permanently Locks ETH When No Minting Is Required - (File: `L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

`RewardSupplier.tick()` is a public `payable` function that accepts ETH to cover StarkGate bridge and L1→L2 messaging fees. However, the entire ETH-forwarding logic is gated behind `if (amountToMint > 0)`. When there are no pending L2 mint requests, the function silently returns without forwarding or refunding `msg.value`, permanently locking any ETH sent by the caller inside the contract. There is no withdrawal mechanism anywhere in the contract hierarchy.

---

### Finding Description

`RewardSupplier.tick()` is declared `external payable`: [1](#0-0) 

Inside, it first calls `requiredMinting()` to determine whether any L2 mint-request messages are pending: [2](#0-1) 

All ETH usage — splitting `msg.value` between the StarkGate bridge deposit and the `sendMessageToL2` call — is entirely inside the `if (amountToMint > 0)` branch: [3](#0-2) 

Specifically, the fee split: [4](#0-3) 

When `amountToMint == 0`, execution falls through the `if` block and the function returns. `msg.value` is never forwarded, never refunded, and is permanently trapped in the contract.

Neither `RewardSupplierStorage` nor `ProxySupportImpl` define a `receive()`, `fallback()`, or any ETH-rescue function: [5](#0-4) [6](#0-5) 

There is no on-chain path to recover ETH once it is deposited this way.

---

### Impact Explanation

Any ETH sent with a `tick()` call when `amountToMint == 0` is permanently frozen inside the `RewardSupplier` contract. This matches the allowed impact: **Medium — Griefing with no profit motive but damage to users or protocol**. The caller loses their ETH with no benefit to anyone.

---

### Likelihood Explanation

`tick()` is a public, permissionless function intended to be called by any account (the spec shows `anyAccount` as the caller): [7](#0-6) 

A caller who checks `requiredMinting()` off-chain and finds pending messages may still send slightly more ETH than needed to ensure the fees are covered. Alternatively, a caller may call `tick()` with ETH during a period when no mint requests are pending (e.g., between reward epochs), losing all sent ETH. Both scenarios are realistic for a public keeper-style function.

---

### Recommendation

Add a guard at the top of `tick()` to require that `msg.value == 0` when there is nothing to mint, or add a refund path for the no-op case:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ... existing logic ...
    } else {
        // Refund any ETH sent when there is nothing to process.
        require(msg.value == 0, "NO_MINT_REQUIRED_NO_FEE_NEEDED");
    }
}
```

Alternatively, refund excess ETH unconditionally after the branch:

```solidity
if (address(this).balance > 0) {
    payable(msg.sender).transfer(address(this).balance);
}
```

---

### Proof of Concept

1. Observe that `requiredMinting()` returns `(0, 0)` when no L2 mint-request messages are pending (i.e., `messagingContract().l2ToL1Messages(msgHash) == 0`).
2. Call `RewardSupplier.tick{value: 1 ether}()` at such a time.
3. The `if (amountToMint > 0)` branch is skipped entirely.
4. The function returns. `msg.value` (1 ETH) is now held by the contract.
5. There is no function in `RewardSupplier`, `RewardSupplierStorage`, or `ProxySupportImpl` that allows recovery of this ETH.
6. The 1 ETH is permanently locked. [8](#0-7)

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
