### Title
Cross-Address COA Ownership Proof Replay via Unbound `signedData` — (`fvm/evm/stdlib/contract.cdc`)

### Summary

`EVM.validateCOAOwnershipProof` does not bind the signed payload to the specific EVM address being proven. A signature produced to prove ownership of one COA (Cadence-Owned Account) can be replayed by an unprivileged attacker to pass ERC-1271 `isValidSignature` verification on a *different* COA owned by the same Cadence account, enabling unauthorized impersonation of that second COA in any EVM protocol that relies on ERC-1271.

### Finding Description

`EVM.validateCOAOwnershipProof` in `fvm/evm/stdlib/contract.cdc` accepts six caller-controlled arguments: `address`, `path`, `signedData`, `keyIndices`, `signatures`, and `evmAddress`. The function:

1. Verifies that the provided `signatures` are valid over `signedData` using the account's keys.
2. Borrows a `CadenceOwnedAccount` capability from the caller-supplied `path`.
3. Checks that the borrowed COA's EVM address matches the caller-supplied `evmAddress`.

Critically, **`signedData` is never required to commit to `evmAddress` or `path`**. The code itself documents this gap:

> *"Note: this function does not enforce that `signedData` includes `evmAddress`. In principle, a signature produced for one purpose could be replayed here against a different COA owned by the same Cadence account."* [1](#0-0) 

Because `path` is fully attacker-controlled and the signature check is decoupled from both `path` and `evmAddress`, an attacker who observes a valid proof for COA_A can construct a new proof that substitutes COA_B's `CapabilityPath` while reusing the original signatures verbatim. The three checks all pass independently:

- Signature check passes (same `signedData`, same keys).
- Capability borrow succeeds (COA_B exists at the new path).
- Address match passes (the attacker calls `COA_B.isValidSignature`, so `address(this)` == COA_B's address). [2](#0-1) 

The ERC-1271 entry point in the COA Solidity contract always passes `address(this)` as the EVM address argument, so the attacker only needs to call `isValidSignature` on the target COA with the replayed proof: [3](#0-2) 

The proof is decoded and forwarded to `validateCOAOwnershipProof` via the `coaOwnershipProofValidator` in the Cadence Arch precompile: [4](#0-3) 

### Impact Explanation

Any EVM protocol (DEX, lending, NFT marketplace, multisig) that uses ERC-1271 to authenticate a COA can be deceived into accepting a signature as valid for COA_B when the account holder only ever signed data for COA_A. Concrete consequences include:

- Unauthorized approval of token transfers from COA_B.
- Bypassing ERC-1271-gated access controls on DeFi contracts on behalf of COA_B.
- Forging NFT marketplace order signatures attributed to COA_B.

The attacker gains the ability to impersonate any COA owned by a victim who holds more than one COA, without ever possessing the victim's private key.

### Likelihood Explanation

- `EVM.validateCOAOwnershipProof` is `access(all)` and callable from any Cadence script or transaction without any privilege.
- The COA Solidity `isValidSignature` is a standard EVM call reachable by any EOA or contract.
- The attacker only needs to observe one valid proof from the victim (e.g., from a public on-chain transaction or off-chain authentication flow).
- The precondition—a user holding multiple COAs—is uncommon today but becomes more likely as COA usage grows and protocols encourage per-application COA isolation.

### Recommendation

Bind the signed payload to the specific EVM address. The simplest fix is to require that `signedData` encodes `evmAddress` (e.g., `signedData = keccak256(abi.encode(evmAddress, applicationData))`), and enforce this inside `validateCOAOwnershipProof` by hashing `signedData || evmAddress` before passing it to `keyList.verify`. Alternatively, the function should internally construct the message as `hash(signedData, evmAddress)` so that a proof for COA_A is cryptographically invalid for COA_B.

### Proof of Concept

```
Precondition: Alice owns a Cadence account with two COAs:
  - COA_A stored at /storage/coa_a, published at /public/coa_a
  - COA_B stored at /storage/coa_b, published at /public/coa_b

Step 1 – Alice legitimately proves ownership of COA_A:
  Alice signs `hash_H` and produces:
    proof_A = { Address: alice, CapabilityPath: "coa_a",
                KeyIndices: [0], Signatures: [sig_alice_over_H] }
  She calls COA_A.isValidSignature(hash_H, encode(proof_A)) → ValidERC1271Signature ✓

Step 2 – Attacker observes proof_A (e.g., from a public tx or off-chain auth flow).

Step 3 – Attacker constructs proof_B by replacing only CapabilityPath:
    proof_B = { Address: alice, CapabilityPath: "coa_b",
                KeyIndices: [0], Signatures: [sig_alice_over_H] }

Step 4 – Attacker calls COA_B.isValidSignature(hash_H, encode(proof_B)):
  → verifyCOAOwnershipProof(COA_B_evm_addr, hash_H, encode(proof_B))
  → validateCOAOwnershipProof(alice, /public/coa_b, hash_H, [0], [sig_alice_over_H], COA_B_evm_addr)
    (a) keyList.verify(sig_alice_over_H, hash_H) → true  (Alice signed hash_H)
    (b) borrow<&EVM.CadenceOwnedAccount>(/public/coa_b) → succeeds
    (c) coaRef.address() == COA_B_evm_addr → true
  → returns ValidationResult(isValid: true)
  → isValidSignature returns ValidERC1271Signature  ✗ (Alice never authorized this)
``` [5](#0-4) [6](#0-5)

	return func(proof *types.COAOwnershipProofInContext) (bool, error) {
		value, err := backend.Invoke(
			environment.ContractFunctionSpec{
				AddressFromChain: func(_ flow.Chain) flow.Address {
					return contractAddress
				},
				LocationName: "EVM",
				FunctionName: "validateCOAOwnershipProof",
				ArgumentTypes: []sema.Type{
					types.FlowAddressSemaType,
					types.PublicPathSemaType,
					types.SignedDataSemaType,
					types.KeyIndicesSemaType,
					types.SignaturesSemaType,
					types.AddressBytesSemaType,
				},
			},
			proof.ToCadenceValues(),
		)
		if err != nil {
			if types.IsAFatalError(err) {
				panic(err)
			}
			return false, err
		}
		data, ok := value.(cadence.Struct)
		if !ok {
			return false, fmt.Errorf("invalid output data received from validateCOAOwnershipProof")
		}

		isValidValue := cadence.SearchFieldByName(data, ValidationResultTypeIsValidFieldName)
		if isValidValue == nil {
			return false, fmt.Errorf("invalid output data received from validateCOAOwnershipProof")
		}

		return bool(isValidValue.(cadence.Bool)), nil
	}
