### Title
Unrestricted Mint via Bank Precompile Allows Any EVM Contract to Create Unlimited Native Cosmos Bank Tokens — (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The bank precompile deployed at `0x0000000000000000000000000000000000000064` exposes a `mint(address, uint256)` method that any EVM contract can call to create unlimited native Cosmos bank tokens with denom `evm/<callerAddress>` to any recipient, with no authorization check, no supply cap, and no requirement to burn corresponding ERC20 tokens.

### Finding Description
In `BankContract.Run()`, the `MintMethodName` case derives the minted token denom directly from `contract.Caller()` — the address of the calling EVM contract — and immediately mints native bank tokens to the specified recipient:

```go
denom := EVMDenom(contract.Caller())   // "evm/" + callerAddress.Hex()
amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
// ...
bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt))
bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt))
```

The only guards present are:
- `readonly` check (not a security guard)
- `amount.Sign() <= 0` (not a security guard)
- `checkBlockedAddr` (only prevents sending to module accounts)

There is **no check** that:
1. The caller is authorized to issue tokens for its denom
2. The caller has burned a corresponding amount of ERC20 tokens
3. Any supply cap or per-block limit exists

The `IBankModule` interface is publicly documented and callable by any EVM contract:

```solidity
interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
    ...
}
```

The design intent (illustrated in `TestBank.sol`) is that a contract burns its own ERC20 tokens before calling `bank.mint`, creating a 1:1 native representation. However, this burn step is never enforced by the precompile. A malicious contract can call `bank.mint` without any prior burn, creating native tokens from nothing.

### Impact Explanation
**Critical — Unauthorized mint of native bank tokens.**

Any unprivileged user can deploy an EVM contract and call `IBankModule(0x64).mint(recipient, amount)` to mint an unlimited supply of `evm/<contractAddress>` native Cosmos bank tokens to any address. These are real entries in the Cosmos `x/bank` module state. They can be:
- Transferred freely between Cosmos accounts
- Sent over IBC to other chains (if IBC transfer is enabled for arbitrary denoms)
- Used in any Cosmos module that accepts arbitrary native denoms

This is a direct analog to the DVGToken unlimited mint: the "owner" of the `evm/<contractAddress>` denom is effectively any EVM contract, and there is no restriction on how much it can mint.

### Likelihood Explanation
**High.** The attack requires only deploying an EVM contract (no special privilege, no admin key, no governance vote). The bank precompile is a standard precompile address accessible to every EVM contract on the network. The `IBankModule` interface is publicly documented. Any user with gas can execute this.

### Recommendation
Add an authorization check in the `mint` case of `BankContract.Run()` to enforce that the caller has burned a corresponding amount of its own ERC20 tokens in the same transaction before native tokens are issued, or restrict `bank.mint` to contracts that have been explicitly whitelisted by governance. At minimum, document clearly that the `evm/<contractAddress>` denom is fully controlled by the calling contract with no supply guarantee, so integrators are not misled.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
}

contract UnlimitedMintAttack {
    address constant BANK_PRECOMPILE = 0x0000000000000000000000000000000000000064;

    // Any unprivileged user deploys this contract and calls exploit().
    // No ERC20 burn, no authorization, no restriction.
    function exploit(address victim) external {
        IBankModule(BANK_PRECOMPILE).mint(victim, type(uint256).max);
        // Result: victim now holds type(uint256).max of
        // native denom "evm/0x<address of this contract>"
        // in the Cosmos x/bank module, minted from nothing.
    }
}
```

**Step-by-step:**
1. Attacker deploys `UnlimitedMintAttack` — no special privilege required.
2. Attacker calls `exploit(attackerAddress)`.
3. `BankContract.Run()` is invoked with `contract.Caller()` = `UnlimitedMintAttack` address.
4. Denom `evm/0x<UnlimitedMintAttack>` is minted in unlimited quantity to `attackerAddress` via `bankKeeper.MintCoins` + `SendCoinsFromModuleToAccount`.
5. Attacker holds unlimited native Cosmos bank tokens with no corresponding ERC20 burn or authorization.
