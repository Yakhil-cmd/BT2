### Title
Pool Admin Can Set Uncapped Per-Bin Additional Fees via `setPoolBinAdditionalFees`, Bypassing the Factory Fee Cap System - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary

`setPoolBinAdditionalFees` passes `addFeeBuyE6` and `addFeeSellE6` directly to the pool with no cap check, while the parallel `setPoolAdminFees` path is properly gated by `maxAdminSpreadFeeE6` / `maxAdminNotionalFeeE8`. A pool admin can therefore set per-bin additional fees to any value up to `uint16.max` (65 535 E6 ≈ 6.55 %), silently exceeding the factory-enforced fee ceiling and extracting more from traders than the protocol intends to allow.

### Finding Description

The factory enforces two separate fee-cap paths for the pool admin:

**Capped path – `setPoolAdminFees`:**
```solidity
// MetricOmmPoolFactory.sol L414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

**Uncapped path – `setPoolBinAdditionalFees`:**
```solidity
// MetricOmmPoolFactory.sol L450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` also performs no cap check:
```solidity
// MetricOmmPool.sol L464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    emit BinAdditionalFeesUpdated(bin, addFeeBuyE6, addFeeSellE6);
}
``` [3](#0-2) 

These per-bin fees are **additive** on top of the base spread fee during every swap:
```solidity
// MetricOmmPool.sol – swap math uses:
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)
``` [4](#0-3) 

The hard cap for the base spread is `HARD_MAX_SPREAD_FEE_E6 = 200 000` (20 %). The `uint16` ceiling for bin additional fees is `65 535` E6 ≈ 6.55 %, with no factory-level guard. A pool admin can call `setPoolBinAdditionalFees` for every active bin and push the effective per-swap fee to ≈ 26.55 % while `poolFeeConfig` still reports a compliant admin spread fee.

### Impact Explanation

Traders executing swaps through any bin where the pool admin has set `addFeeBuyE6` or `addFeeSellE6` to the maximum pay up to 6.55 % more than the protocol's intended ceiling. Because the fee is deducted from the swap output (or added to the required input) before the balance check, the excess is silently retained in the pool and later collectible by the admin via `collectPoolFees`. This constitutes a direct loss of user principal above Sherlock medium thresholds.

### Likelihood Explanation

Any pool admin — a role that is semi-trusted only within the factory's cap system — can trigger this with a single transaction at any time after pool deployment. No timelock, no owner co-signature, and no on-chain guard prevents it. The call is permissioned only to `poolAdmin[pool]`, which is a normal operational role.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool. The simplest approach is to bound the sum of the base spread fee and the bin additional fee:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce that `poolFeeConfig[pool].adminSpreadFeeE6 + addFeeBuyE6 <= maxAdminSpreadFeeE6` to cap the combined admin-controlled fee component.

### Proof of Concept

1. Factory owner deploys a pool with `maxAdminSpreadFeeE6 = 10 000` (1 %) and `adminSpreadFeeE6 = 10 000`.
2. Pool admin calls `setPoolAdminFees(pool, 10_000, 0)` — accepted, at cap.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` — **no revert**.
4. A trader swaps through bin 0: effective fee = base spread (1 %) + bin additional (≈ 6.55 %) = ≈ 7.55 %, far above the 1 % cap the factory owner intended.
5. The excess ≈ 6.55 % of the swap notional is retained in the pool and flows to the admin on the next `collectPoolFees` call. [2](#0-1) [3](#0-2) [1](#0-0)
