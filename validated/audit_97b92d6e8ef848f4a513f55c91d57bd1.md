### Title
Unauthenticated gRPC `Set` RPC Allows Any Network Peer to Overwrite Arbitrary Ledger Registers — (`cmd/ledger/main.go`, `ledger/remote/service.go`)

---

### Summary

The `cmd/ledger` production binary exposes a gRPC server via `--ledger-service-tcp` with no TLS, no authentication interceptor, and no authorization check in any handler. The `Set` RPC directly calls `s.ledger.Set(update)`, which writes to the WAL and trie forest. Any process that can reach the TCP port — including an unstaked Access or Observer node — can overwrite arbitrary register values in the execution node's persistent ledger.

---

### Finding Description

`cmd/ledger/main.go` creates the gRPC server as:

```go
grpcServer := grpc.NewServer(
    grpc.MaxRecvMsgSize(int(*maxRequestSize)),
    grpc.MaxSendMsgSize(int(*maxResponseSize)),
)
``` [1](#0-0) 

No `grpc.Creds(...)`, no `grpc.UnaryInterceptor(...)`, no `grpc.StreamInterceptor(...)` are registered. The server then binds to the operator-supplied TCP address (e.g., `0.0.0.0:9000`) when `--ledger-service-tcp` is set: [2](#0-1) 

The `Set` handler in `ledger/remote/service.go` performs only structural input validation (state hash length, key/value count parity) and then unconditionally calls `s.ledger.Set(update)`: [3](#0-2) 

There is no caller identity check, no token/credential verification, and no capability gate anywhere in the call path. The `ledger.Set` call writes a new trie node to the WAL and inserts it into the in-memory forest — the same path used by the execution node's own `CommitDelta` pipeline: [4](#0-3) 

---

### Impact Explanation

An attacker who can reach TCP port 9000 sends a `SetRequest` with:
- `state`: the current root hash (obtainable via the unauthenticated `InitialState` RPC on the same port)
- `keys`: any register key (e.g., a FlowToken balance register)
- `values`: arbitrary bytes

The call succeeds, durably writing the forged value to the WAL and producing a new trie root. Subsequent reads via `GetSingleValue` will return the attacker-controlled value. The execution node's `CommitView` pipeline will then build on top of this corrupted state, propagating the corruption into all future blocks. This directly violates the invariant that only the execution node's own `CommitDelta` pipeline may mutate the ledger WAL and trie forest.

---

### Likelihood Explanation

The `--ledger-service-tcp` flag is a documented, supported production option in the `cmd/ledger` binary. [5](#0-4) 

When an operator deploys the ledger service with TCP binding (e.g., in a containerized environment where the ledger runs as a sidecar reachable on a shared network), any peer on that network — including an unstaked Access or Observer node — can exploit this. No credentials, no staking, and no prior access are required beyond TCP reachability.

---

### Recommendation

1. **Remove TCP binding from production deployments** or gate it behind a firewall rule that permits only the co-located execution node's IP.
2. **Add mTLS** to the gRPC server: register `grpc.Creds(credentials.NewTLS(tlsConfig))` with a server certificate and require client certificates signed by a trusted CA known only to the execution node.
3. **Add a unary interceptor** that rejects any `Set` (or `Prove`) call from a client that has not presented a valid mutual-TLS certificate.
4. **Prefer Unix domain sockets** (`--ledger-service-socket`) for all production deployments; the socket's filesystem permissions (`0660`) provide OS-level access control that TCP cannot.

---

func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
	if req.State == nil || len(req.State.Hash) != len(ledger.State{}) {
		return nil, status.Error(codes.InvalidArgument, "invalid state")
	}

	if len(req.Keys) == 0 {
		return nil, status.Error(codes.InvalidArgument, "keys cannot be empty")
	}

	if len(req.Keys) != len(req.Values) {
		return nil, status.Error(codes.InvalidArgument, "keys and values length mismatch")
	}

	var state ledger.State
	copy(state[:], req.State.Hash)

	keys := make([]ledger.Key, len(req.Keys))
	for i, protoKey := range req.Keys {
		key, err := protoKeyToLedgerKey(protoKey)
		if err != nil {
			return nil, err // protoKeyToLedgerKey already returns status.Error
		}
		keys[i] = key
	}

	values := make([]ledger.Value, len(req.Values))
	for i, protoValue := range req.Values {
		var value ledger.Value
		// Reconstruct the original value type using is_nil flag
		// This preserves the distinction between nil and []byte{} that protobuf loses
		if len(protoValue.Data) == 0 {
			if protoValue.IsNil {
				// Original value was nil
				value = nil
			} else {
				// Original value was []byte{} (empty slice)
				value = ledger.Value([]byte{})
			}
		} else {
			// Non-empty value, use data as-is
			value = ledger.Value(protoValue.Data)
		}
		values[i] = value
	}

	update, err := ledger.NewUpdate(state, keys, values)
	if err != nil {
		return nil, status.Error(codes.InvalidArgument, err.Error())
	}

	newState, trieUpdate, err := s.ledger.Set(update)
	if err != nil {
		return nil, status.Error(codes.Internal, err.Error())
	}
```
