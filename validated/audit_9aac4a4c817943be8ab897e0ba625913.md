### Title
Any Caller Can Invoke `update_rewards` With `disable_rewards: true` to Permanently Freeze Staker Yield — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable by any address. When invoked with `disable_rewards: true`, it unconditionally writes the current block number to the global `last_reward_block` storage variable — marking the block as "processed" — and then returns early without distributing any rewards. Because `last_reward_block` is a global, per-block guard that prevents duplicate reward updates, an attacker can front-run every legitimate `update_rewards` call to permanently freeze all stakers' consensus (V3) yield.

---

### Finding Description

`update_rewards` is defined in `IStakingRewardsManager` and has no access-control guard beyond `general_prerequisites()` (which only checks the contract is unpaused and the caller is non-zero). [1](#0-0) 

The function first asserts the current block has not yet been processed: [2](#0-1) 

It then validates the staker is active and has non-zero balance, and **unconditionally** writes `last_reward_block` before checking `disable_rewards`: [3](#0-2) 

The critical state change — `self.last_reward_block.write(current_block_number)` — is performed at line 1485 **before** the early-return guard at line 1487. If `disable_rewards` is `true`, the function returns immediately without calling `_update_rewards`, so no rewards are ever credited to the staker or their pools. [4](#0-3) 

Because `last_reward_block` is a single global variable, once it is set to block `N`, no other caller can successfully call `update_rewards` for block `N` — the assertion at line 1455 will revert. An attacker who front-runs every block with `disable_rewards: true` permanently prevents any staker from accumulating consensus rewards.

The `staker_address` parameter is also caller-supplied with no ownership check, so the attacker can use any valid, active staker address (all of which are publicly enumerable from the `stakers` storage vector). [5](#0-4) 

---

### Impact Explanation

This is a **High** impact finding: **Permanent freezing of unclaimed yield**.

In the V3 (consensus rewards) regime, stakers earn STRK rewards per block via `update_rewards`. If an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at every block, `last_reward_block` is advanced each block but `_update_rewards` is never executed. Stakers' `unclaimed_rewards_own` fields are never incremented, and pool reward traces are never updated. The yield is permanently frozen — stakers can never claim rewards they are entitled to.

---

### Likelihood Explanation

**High.** The attack requires no privileged access, no leaked keys, and no external dependencies. Any EOA can call `update_rewards` with a valid staker address (publicly readable from `stakers` vec) and `disable_rewards: true`. The only cost is gas per block. On Starknet, the sequencer orders transactions, so a persistent attacker can reliably front-run every legitimate `update_rewards` call.

---

### Recommendation

Add an access-control check so that only the staker themselves (or an authorized operator) can call `update_rewards` with `disable_rewards: true`. Alternatively, move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` early-return guard, so that a no-op call does not consume the block's reward slot:

```cairo
// Update last block rewards ONLY when rewards will actually be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... rest of reward distribution
```

Or restrict `disable_rewards: true` to the staker address only:

```cairo
if disable_rewards {
    assert!(get_caller_address() == staker_address, "Unauthorized");
    return;
}
```

---

### Proof of Concept

1. Consensus rewards (V3) are active: `consensus_rewards_first_epoch` is set and the current epoch has passed it.
2. Attacker observes a new block `N` is produced.
3. Attacker calls `update_rewards(valid_staker_address, disable_rewards: true)`.
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - Staker is active with non-zero balance → assertions pass.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → early return, no rewards distributed.
4. Any legitimate staker or protocol bot that now calls `update_rewards(staker_X, disable_rewards: false)` for block `N` hits the assertion `current_block_number > last_reward_block` → **reverts** with `REWARDS_ALREADY_UPDATED`.
5. Repeating step 2–4 every block permanently freezes all staker yield. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
