Audit Report

## Title
Caller-Controlled `new_controller` in `disburse_to_neuron` Bypasses KYC for Child Neuron тАФ (File: `rs/nns/governance/src/governance.rs`)

## Summary
`disburse_to_neuron` enforces KYC only on the parent neuron but unconditionally propagates `kyc_verified = true` to a child neuron whose controller is an arbitrary, caller-supplied `PrincipalId` that has never undergone KYC. Because `disburse_neuron` gates liquid ICP disbursement on the neuron's `kyc_verified` flag rather than on the controller's identity, a non-KYC-verified principal who receives the child neuron can immediately call `disburse_neuron` and convert staked ICP to liquid tokens, fully circumventing the KYC requirement.

## Finding Description
In `disburse_to_neuron` (`rs/nns/governance/src/governance.rs`), the parent neuron's KYC status is checked at line 2949:

```rust
if !parent_neuron.kyc_verified {
    return Err(...);
}
``` [1](#0-0) 

After this check passes, the caller freely supplies `new_controller` тАФ any arbitrary `PrincipalId` тАФ as the controller of the child neuron, with no KYC check on that principal:

```rust
let child_controller = disburse_to_neuron.new_controller.ok_or_else(|| { ... })?;
``` [2](#0-1) 

The child neuron is then built with `child_controller` as its controller and the parent's `kyc_verified` flag (always `true` at this point):

```rust
let child_neuron = NeuronBuilder::new(child_nid, to_subaccount, child_controller, ...)
    .with_followees(...)
    .with_kyc_verified(parent_neuron.kyc_verified)
    .build();
``` [3](#0-2) 

The `DisburseToNeuron` command struct exposes a caller-supplied `kyc_verified: bool` field in the public Candid API: [4](#0-3) 

This field is accepted from the caller but is **never read** inside `disburse_to_neuron`; the child neuron's KYC status is set exclusively from `parent_neuron.kyc_verified`. The caller-supplied field is a dead input.

`disburse_neuron` gates liquid disbursement on the neuron's `kyc_verified` property, not on the controller's identity:

```rust
if !is_neuron_kyc_verified {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        format!("Neuron {} is not kyc verified.", id.id),
    ));
}
``` [5](#0-4) 

Because the child neuron carries `kyc_verified = true` inherited from the parent, the non-KYC-verified controller Bob passes this check and receives liquid ICP.

The `approve_genesis_kyc` function confirms that KYC is intended to be tied to the controller's principal identity тАФ it only sets `kyc_verified = true` on neurons whose `controller() == principal`: [6](#0-5) 

`disburse_to_neuron` breaks this invariant by allowing the KYC property to be transferred to a new, unverified controller.

## Impact Explanation
A KYC-verified genesis neuron owner (Alice) can call `disburse_to_neuron` specifying `new_controller = Bob`, where Bob is a principal that has never completed KYC. The resulting child neuron has `kyc_verified = true` and is controlled by Bob. Bob can then call `disburse_neuron` on the child neuron тАФ which requires `kyc_verified` to be true тАФ and convert the staked ICP to liquid tokens, fully bypassing the KYC requirement. This constitutes unauthorized access to governance-gated disbursement of ICP ledger assets by a non-KYC-verified party, matching the **High ($2,000тАУ$10,000)** impact class: "Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds where exploitation requires meaningful per-target work or other constraints."

## Likelihood Explanation
The attack requires a KYC-verified genesis neuron owner to willingly call `disburse_to_neuron` with a non-KYC-verified `new_controller`. This can be self-executed by Alice using a second identity (no third-party cooperation needed), or performed in collusion with Bob. The `manage_neuron` endpoint is a standard ingress call reachable by any unprivileged user who controls a KYC-verified neuron. No privileged access, admin keys, or subnet-majority corruption is required. The pool of genesis neurons with `kyc_verified = true` is finite but non-trivial, and the attack is repeatable as long as such neurons exist with sufficient stake.

## Recommendation
1. **Verify `new_controller` KYC status** inside `disburse_to_neuron` before creating the child neuron. Check that `new_controller` is also a controller of at least one KYC-verified neuron, or add an explicit registry lookup.
2. **Remove or enforce the `kyc_verified` field** from the `DisburseToNeuron` Candid interface. Since it is currently ignored, it is misleading. If it is intended to be caller-supplied, validate that it matches `parent_neuron.kyc_verified` and reject mismatches.
3. Alternatively, if the KYC system is considered fully legacy and no longer enforced in practice, remove the `kyc_verified` checks from both `disburse_neuron` and `disburse_to_neuron` entirely to avoid a false sense of compliance.

## Proof of Concept
1. Alice holds genesis neuron N with `kyc_verified = true` and sufficient stake (e.g., 2 ICP above minimum).
2. Bob is a principal with no KYC-verified neurons.
3. Alice sends an ingress `manage_neuron` call:
   ```rust
   Command::DisburseToNeuron(DisburseToNeuron {
       new_controller: Some(Bob),
       amount_e8s: min_stake + transaction_fee,
       dissolve_delay_seconds: 0,
       kyc_verified: true,  // ignored by governance
       nonce: 42,
   })
   ```
4. Governance checks `parent_neuron.kyc_verified == true` тЬУ (Alice's neuron passes at line 2949).
5. Child neuron C is created with `controller = Bob` and `kyc_verified = true` (inherited at line 3020).
6. Bob sends `manage_neuron` тЖТ `Command::Disburse` on neuron C.
7. Governance checks `neuron.kyc_verified == true` тЬУ (child neuron passes at line 1990).
8. Bob receives liquid ICP, having never undergone KYC.

A deterministic integration test can be written using the existing `fake::FakeDriver` test harness (as used in `rs/nns/governance/tests/governance.rs` `test_approve_kyc`) by setting up a KYC-verified parent neuron, calling `disburse_to_neuron` with a non-KYC-verified `new_controller`, and asserting that the child neuron's `kyc_verified == true` and that `disburse_neuron` succeeds for the non-KYC-verified controller.
