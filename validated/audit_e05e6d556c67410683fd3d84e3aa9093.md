### Title
Wrong Royalty Accumulation in Multi-NFT WalletConnect Offer Confirmation Display — (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary
`formatAmountWithRoyalties` divides the fungible amount by the number of NFTs before computing each royalty, producing a "Total Amount with Royalties" figure that is understated by a factor of N (number of NFTs) in the WalletConnect signing confirmation dialog. A user approving a `chia_wallet.take_offer` or `chia_wallet.create_offer_for_ids` WalletConnect request involving multiple royalty-bearing NFTs is shown a materially lower total than what the transaction actually deducts.

### Finding Description

`formatAmountWithRoyalties` in `parseCommandDisplay.ts` computes the displayed royalty total as:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← divides first
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [1](#0-0) 

For N NFTs with royalty percentages `r₁, r₂, …, rₙ`, this yields:

```
royaltyAmount = (amount / N) * (r₁ + r₂ + … + rₙ) / 10_000
```

The correct formula is:

```
royaltyAmount = amount * r₁/10_000 + amount * r₂/10_000 + … = amount * Σrᵢ / 10_000
```

The actual result is `1/N` of the correct royalty total. For a single NFT (N=1) the calculation is accidentally correct, which is why the existing test at line 409–458 passes — it only covers the single-NFT case. [2](#0-1) 

The `amountWithRoyalties` field produced by this function is rendered in the WalletConnect signing confirmation dialog (`Confirm.tsx`) as **"Total Amount with Royalties"** for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` commands. [3](#0-2) 

`parseCommandDisplay` is called for both WalletConnect commands: [4](#0-3) 

### Impact Explanation

**High.** The WalletConnect approval dialog is the sole user-facing checkpoint before a `take_offer` or `create_offer_for_ids` transaction is signed and submitted. When the offer involves two or more royalty-bearing NFTs, the displayed "Total Amount with Royalties" is understated by a factor of N. A user who relies on this figure to decide whether to approve will consent to a transaction that deducts significantly more XCH or CAT than shown. This is corruption of WalletConnect state that causes a user to approve the wrong amount — matching the allowed High impact category.

### Likelihood Explanation

**Medium-High.** Any dApp that constructs a multi-NFT offer (bundle trades, marketplace batch listings) and routes it through WalletConnect triggers the bug. No special attacker capability is required beyond sending a well-formed offer string. The single-NFT path is unaffected, so the bug is invisible in common single-NFT flows but activates reliably in multi-NFT ones.

### Recommendation

Remove the `splitAmount` division. Each royalty percentage should be applied to the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

Add a test case covering two NFTs with different royalty percentages to prevent regression.

### Proof of Concept

Offer: spend 1 XCH (1 000 000 000 000 mojos), receive two NFTs — NFT-A with 2.5 % royalty (250 basis points) and NFT-B with 3 % royalty (300 basis points).

**Expected display:** `1 000 000 000 000 + (1 000 000 000 000 × 250/10 000) + (1 000 000 000 000 × 300/10 000)` = `1 055 000 000 000` mojos → **1.055 XCH**

**Actual display (buggy):**
- `splitAmount = 1 000 000 000 000 / 2 = 500 000 000 000`
- `royaltyAmount = (500 000 000 000 × 250/10 000) + (500 000 000 000 × 300/10 000)` = `12 500 000 000 + 15 000 000 000` = `27 500 000 000`
- `totalAmount = 1 027 500 000 000` mojos → **1.0275 XCH**

The user sees **1.0275 XCH** but the transaction actually costs **1.055 XCH** — a ~2.6 % understatement that grows with the number of NFTs and royalty rates.

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }
```
