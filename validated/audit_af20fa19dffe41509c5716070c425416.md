### Title
Unprivileged Caller Can Freeze Consensus Rewards by Consuming `last_reward_block` with `disable_rewards=true` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable with no access control. It unconditionally writes `current_block_number` to `last_reward_block` **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(any_valid_staker, true)` every block to consume the global reward slot without distributing any rewards, permanently blocking all stakers from receiving consensus-mode block rewards.

---

### Finding Description

`update_rewards` is the entry point for distributing per-block consensus rewards to stakers. It enforces a global one-call-per-block invariant via `last_reward_block`: [1](#0-0) 

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

Critically, `last_reward_block` is updated **unconditionally** before the `disable_rewards` branch: [2](#0-1) 

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

The only gate on the function is `general_prerequisites()`, which only checks the pause flag and a non-zero caller: [3](#0-2) 

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

There is **no role check** and **no restriction on who may pass `disable_rewards=true`**. The `disable_rewards` parameter is caller-controlled.

**Attack path:**

1. Attacker monitors the chain. In every block N, they call `update_rewards(any_valid_staker, true)`.
2. `last_reward_block` is set to block N; no rewards are distributed.
3. The legitimate consensus-layer call `update_rewards(block_producer, false)` in the same block N fails with `REWARDS_ALREADY_UPDATED`.
4. Block producer receives zero rewards for block N.
5. Repeated every block → all stakers are permanently denied consensus rewards.

The attacker only needs a valid, active staker address with non-zero balance as the `staker_address` argument (any existing staker qualifies).

---

### Impact Explanation

This is a **permanent freezing of unclaimed yield**. Stakers in consensus-rewards mode (`is_pre_consensus() == false`) never accumulate `unclaimed_rewards_own` because `_update_rewards` is never reached. The rewards are not lost from the `RewardSupplier` but are simply never credited to stakers. The attack is sustained as long as the attacker continues spending gas (one transaction per block), which is economically feasible given Starknet's low fees relative to the rewards being denied.

---

### Likelihood Explanation

The attack requires no privileged access, no leaked keys, and no external dependencies. Any address can call `update_rewards` with `disable_rewards=true`. The only cost is gas per block. A competing staker or a protocol adversary has clear economic motivation (denying rivals' rewards). The attack is fully automatable.

---

### Recommendation

Add access control to `update_rewards` so that only a trusted consensus-layer address (or a designated operator role) may call it. Alternatively, only update `last_reward_block` when `disable_rewards == false`, so that a "no-op" call does not consume the block's reward slot:

```cairo
// Update last block rewards only when rewards are actually being distributed.
if !disable_rewards && !self.is_pre_consensus() {
    self.last_reward_block.write(current_block_number);
    // ... distribute rewards
}
```

The preferred fix is role-gating the function to the consensus layer, mirroring how `update_rewards_from_attestation_contract` is gated to the attestation contract: [4](#0-3) 

---

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(alice_staker, disable_rewards=true)
    → last_reward_block := N
    → returns early, no rewards distributed

  Consensus tx: update_rewards(block_producer, disable_rewards=false)
    → assert!(N > N)  ← FAILS with REWARDS_ALREADY_UPDATED
    → block_producer receives 0 rewards for block N

Repeat every block → all stakers receive 0 consensus rewards indefinitely.
```
