### Title
Simulation of BroadcastedDeclareV1Transaction always uses `class_hash = 0`, corrupting the returned `induced_state_diff` and transaction hash — (`crates/apollo_rpc/src/v0_8/api/mod.rs`)

### Summary
Any unprivileged client calling `starknet_simulateTransactions` or `starknet_estimateFee` with a `BroadcastedDeclareV1Transaction` receives a simulation result where `induced_state_diff.deprecated_declared_classes` contains `ClassHash(0)` instead of the actual class hash of the submitted contract class. The transaction hash used during simulation is also wrong for the same reason.

### Finding Description
In `TryFrom<BroadcastedDeclareTransaction> for ExecutableTransactionInput`, the V1 branch unconditionally sets `class_hash: ClassHash::default()` (i.e., `Felt::ZERO`) with the comment "The blockifier doesn't need the class hash." However, this field is consumed in two ways downstream:

1. **Transaction hash**: `get_declare_transaction_v1_hash` chains `transaction.class_hash.0` into the Pedersen hash. With `class_hash = 0`, the computed `tx_hash` is wrong — it does not match the hash the actual on-chain transaction would have. This wrong hash is passed to `BlockifierTransaction::from_api` and is what the account's `__validate_declare__` entry point sees in `tx_info.transaction_hash`.

2. **Induced state diff**: `execute_transactions` reads `class_hash` from the `DeclareTransactionV0V1` struct and passes it as `deprecated_declared_class_hash` to `induced_state_diff`, which places it verbatim into `ThinStateDiff::deprecated_declared_classes`. The simulation response therefore always reports `deprecated_declared_classes: [ClassHash(0)]` regardless of the actual submitted class.

### Impact Explanation
`starknet_simulateTransactions` returns an authoritative-looking `SimulatedTransaction` with a `state_diff` that contains `deprecated_declared_classes: [ClassHash(0)]`. Any client using this to predict what class hash will be declared (e.g., to pre-compute a contract address that depends on the declared class, or to verify the simulation matches expectations) receives a concretely wrong value. The wrong tx hash also means the simulated `__validate_declare__` call operates on a different hash than the real transaction would, making the simulation's validation result unreliable.

### Likelihood Explanation
Triggered by any unprivileged call to `starknet_simulateTransactions` or `starknet_estimateFee` containing a `BroadcastedDeclareV1Transaction`. No special privileges or preconditions required.

### Recommendation
Compute the actual class hash from the submitted `contract_class` before constructing `DeclareTransactionV0V1`. The `compute_deprecated_class_hash` function already exists in the codebase for this purpose. [6](#0-5) 

### Proof of Concept
1. Call `starknet_V0_8_simulateTransactions` with a `BroadcastedDeclareTransaction::V1` containing any non-trivial `contract_class`.
2. Observe the returned `SimulatedTransaction::transaction_trace` → `state_diff` → `deprecated_declared_classes`.
3. The list will contain `ClassHash(0x0)` regardless of the actual class hash of the submitted `contract_class`.
4. Separately compute the actual class hash of the submitted class using `compute_deprecated_class_hash`; it will differ from `0x0`.

The concrete corrupted value is `induced_state_diff.deprecated_declared_classes = [ClassHash(0)]` for every DeclareV1 simulation, and the tx hash used in simulation is `H(... || 0 || ...)` instead of `H(... || actual_class_hash || ...)`.

pub(crate) fn get_declare_transaction_v1_hash(
    transaction: &DeclareTransactionV0V1,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    Ok(TransactionHash(
        HashChain::new()
        .chain(&DECLARE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address.0.key())
        .chain(&Felt::ZERO) // No entry point selector in declare transaction.
        .chain(&HashChain::new().chain(&transaction.class_hash.0).get_pedersen_hash())
        .chain(&transaction.max_fee.0.into())
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce.0)
        .get_pedersen_hash(),
    ))
}

pub fn compute_deprecated_class_hash(
    contract_class: &ContractClass,
) -> Result<Felt, HintedClassHashError> {
    let hinted_class_hash = compute_cairo_hinted_class_hash(contract_class)?;
    let contract_definition_vec = serde_json::to_vec(contract_class)?;
    let contract_definition: CairoContractDefinition<'_> =
        serde_json::from_slice(&contract_definition_vec)?;

    let FlatEntryPointFelts { external, l1_handler, constructor } =
        get_flat_entry_point_felts(&contract_definition.entry_points_by_type);
    let builtins = ascii_strs_as_felts(&contract_definition.program.builtins);
    let bytecode = hex_strs_as_felts(&contract_definition.program.data);

    let mut hash_state = HashState::<Pedersen>::new();
    hash_state.update_single(&DEPRECATED_COMPILED_CLASS_VERSION);
    hash_state.update_with_hashchain(&external);
    hash_state.update_with_hashchain(&l1_handler);
    hash_state.update_with_hashchain(&constructor);
    hash_state.update_with_hashchain(&builtins);
    hash_state.update_single(&hinted_class_hash);
    hash_state.update_with_hashchain(&bytecode);
    Ok(hash_state.finalize())
