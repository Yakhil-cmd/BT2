### Title
`StatelessTransactionValidator::validate_resource_bounds` Unconditionally Applies L2 Gas `min_gas_price` Check, Incorrectly Rejecting Valid V3 Transactions With Only L1 Gas Bounds — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

The `validate_resource_bounds` function in `StatelessTransactionValidator` always checks `l2_gas.max_price_per_unit >= min_gas_price` for every V3 transaction, regardless of whether the transaction actually uses L2 gas. A valid V3 transaction that sets only L1 gas bounds (`l2_gas.max_price_per_unit = 0`) passes the zero-bounds guard but is then unconditionally rejected by the `min_gas_price` check in production (where `min_gas_price = 8,000,000,000`). The test suite masks this because `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING` sets `min_gas_price = 0`.

---

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs`, `validate_resource_bounds` runs two sequential checks:

**Check 1 — zero-bounds guard** (line 66):
```rust
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(ZeroResourceBounds { resource_bounds });
}
```
A transaction with only L1 gas set (`l1_gas = NON_EMPTY`, `l2_gas = Default::default()`) passes this check because the L1 gas contribution makes the max fee non-zero.

**Check 2 — unconditional L2 gas price floor** (line 71):
```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(MaxGasPriceTooLow { gas_price: ..., min_gas_price: ... });
}
```
`l2_gas.max_price_per_unit` is `GasPrice(0)` for a transaction that only uses L1 gas. In production `min_gas_price = 8_000_000_000` (the `Default` impl), so `0 < 8_000_000_000` is `true` and the transaction is rejected.

The test `valid_l1_gas` in `stateless_transaction_validator_test.rs` (line 70) explicitly asserts that a V3 transaction with only L1 gas bounds is valid:

```rust
#[case::valid_l1_gas(
    StatelessTransactionValidatorConfig {
        validate_resource_bounds: true,
        ..*DEFAULT_VALIDATOR_CONFIG_FOR_TESTING   // min_gas_price = 0 here
    },
    RpcTransactionArgs {
        resource_bounds: AllResourceBounds {
            l1_gas: NON_EMPTY_RESOURCE_BOUNDS,
            ..Default::default()          // l2_gas.max_price_per_unit = 0
        },
        ..Default::default()
    }
)]
```

This test passes only because `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING` sets `min_gas_price = 0`. The production default (`Default::default()`) sets `min_gas_price = 8_000_000_000`, so the same transaction is rejected in production.

The stateful validator's `validate_tx_l2_gas_price_within_threshold` (lines 364–388) correctly handles the analogous case for `ValidResourceBounds::L1Gas` legacy transactions by returning `Ok(())` with the comment "No validation required for legacy transactions." No equivalent guard exists in the stateless path for V3 transactions that only use L1 gas.

---

### Impact Explanation

**High — Mempool/gateway admission rejects valid transactions before sequencing.**

Any user who submits a V3 Invoke or DeployAccount transaction with only L1 gas bounds set (`l2_gas.max_price_per_unit = 0`) will receive a `MaxGasPriceTooLow` error from the gateway and the transaction will never reach the mempool. The `AllResourceBounds` type explicitly supports this configuration, and the test suite confirms it is intended to be valid. The rejection is silent from the user's perspective — the error message implies the L2 gas price is too low, not that L2 gas is unused.

---

### Likelihood Explanation

Any client that constructs a V3 transaction using only L1 gas bounds (a documented valid configuration of `AllResourceBounds`) will be affected. The `validate_resource_bounds: true` flag is the production default, so no special configuration is required to trigger the bug.

---

### Recommendation

Condition the `min_gas_price` check on whether the transaction actually specifies a non-zero L2 gas amount or price. For example:

```rust
// Only enforce the L2 gas price floor when the transaction actually uses L2 gas.
if resource_bounds.l2_gas.max_amount.0 > 0 || resource_bounds.l2_gas.max_price_per_unit.0 > 0 {
    if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
        return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
            gas_price: resource_bounds.l2_gas.max_price_per_unit,
            min_gas_price: self.config.min_gas_price,
        });
    }
}
```

This mirrors the pattern already used in the stateful validator, which explicitly skips the L2 gas price threshold check for transactions that do not use L2 gas.

---

### Proof of Concept

1. Construct a V3 Invoke transaction:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 1_000_000_000 },
       l2_gas:      ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
       l1_data_gas: ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
   }
   ```

2. Submit to the gateway (`POST /add_transaction`).

3. `StatelessTransactionValidator::validate` calls `validate_resource_bounds`:
   - **Check 1** (`ZeroResourceBounds`): `max_possible_fee(Tip::ZERO)` is non-zero because L1 gas contributes → **passes**.
   - **Check 2** (`min_gas_price`): `0 < 8_000_000_000` → **fails** with `MaxGasPriceTooLow`.

4. The gateway returns an error and the transaction is dropped, even though it is structurally valid and the test suite (`valid_l1_gas`) confirms it should be accepted.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
