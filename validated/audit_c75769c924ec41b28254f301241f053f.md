### Title
Unbounded Loop in `calculate_rewards` With No Limit on Pool Member Balance Changes Enables Permanent Freezing of Unclaimed Yield — (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` function in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop that iterates over every balance-change checkpoint stored in a pool member's `pool_member_epoch_balance` trace. Because there is no cap on how many checkpoints a pool member may accumulate, and because the loop's starting index (`entry_to_claim_from`) is only advanced when `claim_rewards` is successfully called, a pool member who defers claiming while continuing to make balance changes will eventually cause `claim_rewards` to exhaust the Starknet transaction gas limit, permanently freezing their unclaimed yield.
