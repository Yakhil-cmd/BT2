### Title
Gateway stateless validator unconditionally applies L2 gas price floor to all `AllResourceBounds` transactions, causing valid L1-gas-only V3 transactions to be permanently rejected — (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

`StatelessTransactionValidator::validate_resource_bounds` first confirms that at least one resource bound is non-zero (the fee-existence check), then unconditionally checks that `l2_gas.max_price_per_unit >= min_gas_price`. The production default for `min_gas_price` is `8_000_000_000`. Any V3 transaction that carries a valid non-zero L1 gas fee but leaves L2 gas price at zero will always pass the first check and always fail the second, causing permanent gateway rejection of a class of well-formed transactions.

---

### Finding Description

`validate_resource_bounds` in `StatelessTransactionValidator` runs two sequential checks:

**Check 1 — fee existence:**
```rust
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(ZeroResourceBounds { resource_bounds });
}
```
A transaction with `l1_gas = NON_EMPTY_RESOURCE_BOUNDS` and `l2_gas = Default::default()` passes this check because L1 gas contributes a non-zero `max_possible_fee`.

**Check 2 — L2 gas price floor:**
```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(MaxGasPriceTooLow { ... });
}
```
For the same transaction, `l2_gas.max_price_per_unit = 0`. With the production default `min_gas_price = 8_000_000_000`, the condition `0 < 8_000_000_000` is always true, so the transaction is always rejected.

The production config schema confirms the live value:

```json
"gateway_config.static_config.stateless_tx_validator_config.min_gas_price": {
    "value": 8000000000
}
```

The test suite masks this because `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING` sets `min_gas_price: 0`, making `0 < 0` false and the test pass. The test case `valid_l1_gas` is therefore only valid under the test config, not under the production config:

```rust
static DEFAULT_VALIDATOR_CONFIG_FOR_TESTING: LazyLock<...> =
    LazyLock::new(|| StatelessTransactionValidatorConfig {
        validate_resource_bounds: false,
        min_gas_price: 0,   // ← hides the production defect
        ...
    });
```

The struct comment documents the intent as "ensures that **at least one** resource bound is greater than zero", but the second check imposes an additional, undocumented requirement that L2 gas price must also meet a floor — even when L2 gas is not used at all (amount = 0, price = 0).

The analog to the external report is exact: `mekHoneyJarWithETH` reads `msg.value` but is missing `payable`, so every call with non-zero ETH reverts. Here, `validate_resource_bounds` reads L2 gas price but applies a floor that is never satisfiable for L1-gas-only transactions, so every such transaction is rejected.

---

### Impact Explanation

**High — Mempool/gateway admission rejects valid transactions before sequencing.**

Any unprivileged user who submits a well-formed V3 transaction with non-zero L1 gas bounds and zero L2 gas price will have their transaction permanently rejected at the stateless gateway layer. The transaction never reaches the mempool, the stateful validator, or the blockifier. The rejection is deterministic and unconditional given the production configuration.

---

### Likelihood Explanation

**High.** The condition is always true for L1-gas-only V3 transactions in production (`0 < 8_000_000_000`). No special attacker capability is required; any user or wallet that constructs a V3 transaction with only L1 gas bounds triggers the rejection. The test suite does not catch this because it uses `min_gas_price: 0`.

---

### Recommendation

Guard the L2 gas price floor check so it only applies when L2 gas is actually being used (non-zero amount):

```diff
-if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
+if resource_bounds.l2_gas.max_amount.0 > 0
+    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
     return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
         gas_price: resource_bounds.l2_gas.max_price_per_unit,
         min_gas_price: self.config.min_gas_price,
     });
 }
```

Additionally, the `valid_l1_gas` positive-flow test should be updated to use the production `min_gas_price` value to ensure the test actually validates production behavior.

---

### Proof of Concept

1. Construct a valid V3 invoke transaction with:
   - `l1_gas = ResourceBounds { max_amount: GasAmount(1000), max_price_per_unit: GasPrice(1_000_000_000) }`
   - `l2_gas = ResourceBounds::default()` (amount = 0, price = 0)
   - `l1_data_gas = ResourceBounds::default()`
2. Submit to the production gateway (`POST /add_transaction`).
3. The stateless validator runs `validate_resource_bounds`:
   - Check 1 passes: `max_possible_fee(Tip::ZERO) = 1000 * 1_000_000_000 > 0`.
   - Check 2 fails: `0 < 8_000_000_000` → `MaxGasPriceTooLow { gas_price: 0, min_gas_price: 8_000_000_000 }`.
4. The transaction is rejected before reaching the mempool, despite carrying a valid non-zero L1 gas fee.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
