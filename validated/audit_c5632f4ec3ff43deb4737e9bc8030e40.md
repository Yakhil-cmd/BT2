### Title
Missing Access Control on `update_rewards()` Allows Any Caller to Permanently Suppress Block Reward Distribution - (File: `src/staking/staking.cairo`)

### Summary

The `IStakingRewardsManager.update_rewards()` function is specified to be callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged address can call it with `disable_rewards: true` to advance `last_reward_block` to the current block without distributing any rewards, permanently preventing the legitimate sequencer from distributing rewards for that block.

### Finding Description

The specification at `docs/spec.md:1644-1645` explicitly states:

> **Access control:** Only starkware sequencer.

However, the implementation of `update_rewards` in `src/staking/staking.cairo` at lines 1449–1507 only calls `self.general_prerequisites()`, which at lines 1794–1797 merely asserts the contract is unpaused and the caller is non-zero:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

There is no check that `get_caller_address() == sequencer_address` or any equivalent role assertion. The function signature accepts two attacker-controlled parameters:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
)
``` [2](#0-1) 

Critically, `last_reward_block` is written to storage **before** the `disable_rewards` guard is evaluated:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [3](#0-2) 

The subsequent guard that prevents double-execution for the same block is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

Once an attacker calls `update_rewards(any_active_staker, disable_rewards: true)`, `last_reward_block` is set to the current block. Any subsequent call by the legitimate sequencer for that same block will revert with `REWARDS_ALREADY_UPDATED`. The rewards for that block are permanently unrecoverable.

The spec confirms the intended logic at `docs/spec.md:1647-1652`:

> 1. Calculate total block rewards.
> 2. Calculate staker rewards (include commission) and pool rewards.
> 3. Update `unclaimed_rewards_own` of the staker.
> 4. Update and transfer rewards to the pools, if exist.
> 5. Update Reward Supplier's `unclaimed_rewards`.
> 6. Update `last_reward_block` to the current block. [5](#0-4) 

Steps 1–5 are skipped when `disable_rewards: true`, but step 6 still executes, locking out the block permanently.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker calling `update_rewards(valid_staker, disable_rewards: true)` once per block permanently destroys all stakers' and pool members' yield for that block. Because `last_reward_block` is a single global value, one call per block is sufficient to zero out rewards for every staker in the protocol for that block. Repeated across many blocks, this constitutes a sustained denial of yield to all participants with no recovery path.

### Likelihood Explanation

**High.** The function is publicly callable by any non-zero address with no economic barrier. The attacker only needs to know any active staker address (trivially obtained from on-chain events such as `NewStaker`) and call the function once per block. Gas cost on Starknet L2 is negligible. There is no griefing cost beyond gas, and the attacker gains nothing financially, making this a pure griefing vector — but the damage to stakers and pool members is real and permanent per block.

### Recommendation

Add a caller check at the top of `update_rewards` to enforce that only the designated sequencer address (or an authorized role) may invoke it, consistent with the specification:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

Alternatively, if the sequencer address is not stored on-chain, restrict the call to the attestation contract or another trusted contract that already validates the caller.

### Proof of Concept

1. Alice is a registered staker (address known from `NewStaker` event).
2. At block N, before the sequencer calls `update_rewards`, attacker calls:
   ```
   staking.update_rewards(staker_address: alice, disable_rewards: true)
   ```
3. `general_prerequisites()` passes (contract unpaused, caller non-zero).
4. `current_block_number (N) > last_reward_block` passes.
5. `last_reward_block` is written to `N`.
6. `disable_rewards == true` → function returns early; no rewards are calculated or distributed.
7. Sequencer attempts `update_rewards(alice, false)` at block N → reverts with `REWARDS_ALREADY_UPDATED`.
8. All stakers and pool members receive zero rewards for block N.
9. Attacker repeats every block, permanently freezing all yield. [6](#0-5) [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
