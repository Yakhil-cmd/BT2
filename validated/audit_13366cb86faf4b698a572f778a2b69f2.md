### Title
Missing Correctness Proof on CKD Participant Shares Allows Malicious Participant to Corrupt CKD Output - (File: src/confidential_key_derivation/protocol.rs)

### Summary
The CKD coordinator in `do_ckd_coordinator` receives `(norm_big_y, norm_big_c)` shares from every participant and blindly aggregates them with no proof-of-correctness check. A single malicious participant can send arbitrary group elements, causing the coordinator to produce a silently wrong confidential key. This is the direct analog of the reported pattern: cryptographic material (here, ElGamal-style shares) is accepted and used without any validity check on how it was formed.

### Finding Description
In `src/confidential_key_derivation/protocol.rs`, `do_ckd_coordinator` collects one `CKDOutput` per participant and sums the two components:

```rust
for (_, participant_output) in
    recv_from_others::<CKDOutput>(&chan, waitpoint, &participants, me).await?
{
    norm_big_y += participant_output.big_y();   // no check
    norm_big_c += participant_output.big_c();   // no check
}
``` [1](#0-0) 

Each honest participant is supposed to compute:
- `norm_big_y = λ_i · y_i · G` (random blinding commitment)
- `norm_big_c = λ_i · (x_i · H(pk ‖ app_id) + y_i · app_pk)` (masked secret-share contribution) [2](#0-1) 

The coordinator has no way to verify either component. There is no discrete-log proof that `norm_big_y` was formed with a known `y_i`, and no proof that the same `y_i` appears in `norm_big_c`. The library already has exactly the right primitives for this — `dlog::prove_with_nonce` / `dlog::verify` and `dlogeq::prove_with_nonce` / `dlogeq::verify` — and uses them in the OT-based triple generation to enforce analogous consistency:

```rust
if !dlog::verify(
    &mut transcript.fork(b"dlog0", &from.bytes()),
    statement0,
    their_phi_proof0,
)? {
    return Err(ProtocolError::AssertionFailed(...));
}
``` [3](#0-2) 

No equivalent check exists anywhere in the CKD participant-share path. [4](#0-3) 

### Impact Explanation
**High — Corruption of CKD output.**

The final aggregated `(sum_Y, sum_C)` is the only output the TEE receives. The TEE unmasks it as `sum_C − app_sk · sum_Y = msk · H(pk ‖ app_id)`. If any single participant substitutes arbitrary G1 elements for its share, the unmasked value is wrong — silently, with no error returned to the caller. Honest parties accept a corrupted confidential derived key. This matches the allowed impact: *"High: Corruption of … CKD outputs so honest parties accept … unusable cryptographic outputs."* [5](#0-4) 

### Likelihood Explanation
Any participant in the CKD session is an unprivileged library caller who controls the bytes it sends. No special privilege, leaked key, or cryptographic break is required. The participant simply deviates from the protocol by sending `(rand_G1, rand_G1)` instead of its honest share. The coordinator has no mechanism to detect this. [6](#0-5) 

### Recommendation
Attach a proof-of-correctness to each participant's CKD share before sending, and verify it at the coordinator before aggregating:

1. **Dlog proof on `norm_big_y`**: The participant proves knowledge of `y_i` such that `norm_big_y = λ_i · y_i · G` using `dlog::prove_with_nonce`. The coordinator calls `dlog::verify` and rejects the share on failure.
2. **Dlogeq proof linking `norm_big_y` and the `y_i · app_pk` component of `norm_big_c`**: The participant proves the same `y_i` appears in both, using `dlogeq::prove_with_nonce` with `generator1 = app_pk`. The coordinator calls `dlogeq::verify`.

Both proof primitives already exist in `src/crypto/proofs/dlog.rs` and `src/crypto/proofs/dlogeq.rs` and are used in the OT-based triple generation for exactly this purpose. [7](#0-6) [8](#0-7) 

### Proof of Concept

**Attacker setup**: Participant `P_malicious` is one of `n` participants in a CKD session.

**Honest flow** (what `compute_signature_share` produces):
```
norm_big_y = λ_i · y_i · G
norm_big_c = λ_i · (x_i · H(pk‖app_id) + y_i · app_pk)
```

**Malicious deviation**: `P_malicious` ignores `compute_signature_share` entirely and sends:
```rust
let fake_y = ElementG1::generator() * Scalar::random(rng);  // arbitrary
let fake_c = ElementG1::generator() * Scalar::random(rng);  // arbitrary
chan.send_private(waitpoint, coordinator, &(fake_y, fake_c))?;
```

**Coordinator aggregation** (no check):
```rust
norm_big_y += fake_y;   // corrupted
norm_big_c += fake_c;   // corrupted
```

**TEE result**: `sum_C − app_sk · sum_Y ≠ msk · H(pk ‖ app_id)`. The TEE silently derives a wrong confidential key. No error is raised anywhere in the protocol. [5](#0-4)

/// Produce a proof for the given statement and witness, using a caller-provided nonce.
/// The nonce `(k, k * G)` must be sampled from a cryptographically secure RNG by the caller.
/// The challenge is derived via the Fiat-Shamir transform over the transcript.
pub fn prove_with_nonce<C: Ciphersuite>(
    transcript: &mut Transcript,
    statement: Statement<'_, C>,
    witness: Witness<C>,
    nonce: (Scalar<C>, Element<C>),
) -> Result<Proof<C>, ProtocolError> {
    transcript.message(NEAR_DLOG_STATEMENT_LABEL, &statement.encode()?);

    let (k, big_k) = nonce;

    // Create a serialization of big_k
    let ser = C::Group::serialize(&big_k).map_err(|_| ProtocolError::IdentityElement)?;
    transcript.message(NEAR_DLOG_COMMITMENT_LABEL, ser.as_ref());
    let mut rng = transcript.challenge_then_build_rng(NEAR_DLOG_CHALLENGE_LABEL);
    let e = frost_core::random_nonzero::<C, _>(&mut rng);

    let s = k + e * witness.x.0;
    Ok(Proof {
        e: SerializableScalar(e),
        s: SerializableScalar(s),
    })
}

/// Produce a proof for the given statement and witness, using a caller-provided nonce.
/// The nonce `k` must be sampled from a cryptographically secure RNG by the caller.
/// The challenge is derived via the Fiat-Shamir transform over the transcript.
pub fn prove_with_nonce<C: Ciphersuite>(
    transcript: &mut Transcript,
    statement: Statement<'_, C>,
    witness: Witness<C>,
    k: Scalar<C>,
) -> Result<Proof<C>, ProtocolError>
where
    Element<C>: ConstantTimeEq,
{
    if statement.generator1.ct_eq(&C::Group::identity()).into() {
        return Err(ProtocolError::IdentityElement);
    }

    transcript.message(NEAR_DLOGEQ_STATEMENT_LABEL, &statement.encode()?);

    let (big_k_0, big_k_1) = statement.phi(&k);

    // This will never raise error as k is not zero and generator1 is not the identity
    let enc = encode_two_points::<C>(&big_k_0, &big_k_1)?;

    transcript.message(NEAR_DLOGEQ_COMMITMENT_LABEL, &enc);
    let mut rng = transcript.challenge_then_build_rng(NEAR_DLOGEQ_CHALLENGE_LABEL);
    let e = frost_core::random_nonzero::<C, _>(&mut rng);

    let s = k + e * witness.x.0;
    Ok(Proof {
        e: SerializableScalar::<C>(e),
        s: SerializableScalar::<C>(s),
    })
}
