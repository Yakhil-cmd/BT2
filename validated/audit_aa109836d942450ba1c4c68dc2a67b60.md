### Title
Unpermissioned `update_rewards` Allows Anyone to Block Reward Distribution by Front-Running with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is callable by any address. It unconditionally writes `last_reward_block` to the current block number **before** checking the `disable_rewards` flag. An attacker can call it with `disable_rewards: true` in the same block as a legitimate block-proposer call, causing the legitimate call to revert and the staker to lose all block rewards for that block. Repeated across every block, this permanently freezes unclaimed yield with no cost to the attacker.

### Finding Description
`update_rewards` in `StakingRewardsManagerImpl` has the following structure:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // ← written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // ← exits WITHOUT distributing rewards
    }
    ...
    self._update_rewards(...);
}
```

`general_prerequisites()` only asserts the contract is not paused and the caller is non-zero — there is no role check, no staker-identity check, and no check that the caller is the block proposer or any privileged party. [1](#0-0) 

The `last_reward_block` guard is meant to ensure rewards are distributed at most once per block. However, because the write happens before the `disable_rewards` branch, any caller can "consume" the per-block slot without distributing any rewards. [2](#0-1) 

The interface exposes this as a fully public function with no access-control annotation: [3](#0-2) 

### Impact Explanation
An attacker who calls `update_rewards(valid_staker_address, disable_rewards: true)` in block N:

1. Sets `last_reward_block = N`.
2. Returns early — zero rewards are distributed.
3. Any subsequent call in block N (including the legitimate block-proposer call with `disable_rewards: false`) hits `assert!(current_block_number > last_reward_block)` → `N > N` → **reverts**.

The staker loses all block rewards for block N. If the attacker repeats this every block, stakers permanently receive zero consensus rewards. This constitutes **permanent freezing of unclaimed yield** with no profit to the attacker (pure griefing).

### Likelihood Explanation
- The function is part of the public ABI with no access control.
- The attacker only needs to submit a transaction with `disable_rewards: true` before the legitimate proposer call in each block.
- On Starknet L2 with a centralized sequencer, strict mempool front-running is harder than on Ethereum, but the attacker can still submit the transaction in the same block. Because the check is purely block-number-based (not transaction-order-based within a block), whichever call lands first wins.
- The attack costs only gas per block and requires no capital.

### Recommendation
Restrict `update_rewards` to a permissioned caller (e.g., the attestation contract, a designated consensus reporter role, or the staker's own operational address). Alternatively, separate the `last_reward_block` write so it only occurs when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);  // move write here
self._update_rewards(...);
```

Or add an access-control assertion before the `last_reward_block` write, analogous to how `update_rewards_from_attestation_contract` asserts `assert_caller_is_attestation_contract()`: [4](#0-3) 

### Proof of Concept

1. Staker `S` is active and eligible for consensus block rewards in block `N`.
2. The legitimate block proposer prepares `update_rewards(S, disable_rewards: false)`.
3. Attacker submits `update_rewards(S, disable_rewards: true)` in block `N` and it is sequenced first.
4. `last_reward_block` is written to `N`; no rewards are distributed.
5. The legitimate call in block `N` reverts: `N > N` is false → `REWARDS_ALREADY_UPDATED`.
6. Staker `S` receives zero rewards for block `N`.
7. Attacker repeats steps 3–6 for every subsequent block → staker's unclaimed yield is permanently frozen. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
