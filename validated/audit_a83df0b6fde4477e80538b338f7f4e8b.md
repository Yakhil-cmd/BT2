### Title
Pusher Consent Signature Replay in `allowPushers` Allows Creator to Re-Establish Revoked Delegation, Causing Stale Oracle Prices for Dependent Pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1.allowPushers` uses a deadline to bound consent signatures but does not track used signatures. After a pusher self-revokes via `revokePusher()`, the creator can replay the original consent signature (while the deadline is still valid) to silently re-establish the delegation. The pusher's subsequent `fallback()` pushes are then redirected to the creator's namespace instead of the pusher's own, leaving the pusher's namespace with stale data. Any Metric OMM pool whose price provider reads from `feedIdOf(pusher, …)` will receive a stale bid/ask quote, enabling bad-price execution.

### Finding Description

`allowPushers` signs consent over `keccak256(abi.encode(chainid, oracle, deadline, pusher, msg.sender))`. [1](#0-0) 

The only replay guard is `_ensureDeadline(deadline)`, which rejects calls after the deadline but accepts any number of calls before it. There is no per-signature nonce, no `usedSignatures` bitmap, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before writing.

When a pusher calls `revokePusher()`, the mapping is cleared: [2](#0-1) 

But the original signature remains valid until its deadline. The creator (who holds the signature from the original `allowPushers` call) can immediately call `allowPushers` again with the identical `(deadline, pusher, sig)` tuple, writing `namespaceRemapping[pusher] = creator` again.

The code comment itself acknowledges the structural risk: [3](#0-2) 

The documentation repeats it: [4](#0-3) 

The deadline is described as the fix, but it only prevents replay *after* expiry — not within the deadline window. This is the direct analog to the Optimism H-2 bug: the "already-processed" state (`namespaceRemapping[pusher] = address(0)`) is cleared by revocation, and the old credential is replayed to restore the forbidden state.

After re-establishment, the `fallback()` push path resolves the namespace as the creator's: [5](#0-4) 

The pusher, believing their revocation succeeded, continues pushing data. Every push lands in the creator's namespace; the pusher's own namespace (`feedIdOf(pusher, slotIndex, positionIndex)`) receives no new data and its timestamp freezes.

### Impact Explanation

Any Metric OMM pool whose price provider is bound to `feedIdOf(pusher, …)` will read a frozen timestamp. Once the provider's staleness threshold is crossed, the pool either halts swaps entirely (unusable liquidity flow) or — if the provider does not enforce staleness — executes swaps against a stale bid/ask, allowing traders to extract value from LPs at the wrong price. Both outcomes satisfy the "bad-price execution" and "broken core pool functionality" impact gates.

### Likelihood Explanation

The creator already holds the original signature (they submitted it in the first `allowPushers` call, which is on-chain). The replay requires only that the deadline has not yet expired — a window that can be days or weeks depending on the deadline chosen at delegation time. No additional privileges, off-chain data, or special conditions are needed beyond the creator calling `allowPushers` a second time.

### Recommendation

Track each consent signature hash in a `mapping(bytes32 => bool) private _usedConsentSignatures` and revert if the hash has already been consumed:

```solidity
bytes32 sigHash = keccak256(signatures[i]);
require(!_usedConsentSignatures[sigHash], "consent already used");
_usedConsentSignatures[sigHash] = true;
```

Alternatively, include a per-pusher nonce in the signed payload so that each consent is single-use by construction.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
