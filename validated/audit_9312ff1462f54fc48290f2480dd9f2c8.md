### Title
Staker Can Instantly Raise Commission to 100% via `CommissionCommitment`, Stealing Delegators' Unclaimed Yield - (File: src/staking/staking.cairo)

### Summary
A staker can exploit the `set_commission_commitment` + `set_commission` two-step mechanism to instantly raise their pool commission from any low value (e.g., 0%) to 100%, redirecting all future pool rewards away from delegators to themselves. The protocol's own code acknowledges this gap with an explicit `NOTE` comment but does not mitigate it.

### Finding Description
The `CommissionCommitment` mechanism in `Staking` is intended to let stakers signal a maximum commission ceiling to delegators. However, it contains no minimum commitment duration before the staker can act on it, and no delay between setting the commitment and raising the commission. The attack proceeds in two transactions (or one batched call):

**Step 1 — `set_commission_commitment`** (`src/staking/staking.cairo`, lines 748–785):

```cairo
fn set_commission_commitment(
    ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
) {
    ...
    assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
    ...
    assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
    assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
    assert!(
        expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(), ...
    );
    let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
    staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
```

A staker with commission = 0 can call this with `max_commission = 10000` (100%) and `expiration_epoch = current_epoch + 1`. All checks pass.

**Step 2 — `set_commission` → `update_commission`** (`src/staking/staking.cairo`, lines 725–743 and 1573–1609):

```cairo
fn update_commission(...) {
    if let Option::Some(commission_commitment) = staker_pool_info.commission_commitment.read() {
        if self.is_commission_commitment_active(:commission_commitment) {
            assert!(
                commission <= commission_commitment.max_commission, ...
            );
            assert!(commission != old_commission, ...);
        } else {
            assert!(commission < old_commission, ...);
        }
    } else {
        assert!(commission < old_commission, ...);
    }
    staker_pool_info.commission.write(Option::Some(commission));
```

With the active commitment (`max_commission = 10000`), the staker immediately calls `set_commission(10000)`. The only checks are `commission <= 10000` (passes) and `commission != old_commission` (passes since old = 0). The commission is written to storage with no delay.

The protocol's own developer comment at line 745–746 explicitly acknowledges this:

```cairo
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
```

The commission change takes effect immediately — it is applied at the next reward distribution call (`split_rewards_with_commission`), so delegators who joined expecting 0% commission receive 0 rewards from that point forward.

### Impact Explanation
This is **theft of unclaimed yield**. Delegators who joined a pool based on an advertised low commission rate (e.g., 0%) have their accrued and future rewards entirely redirected to the staker. The commission is applied at reward distribution time, so all rewards earned after the commission spike are captured by the staker. Delegators cannot exit instantly — they must call `exit_delegation_pool_intent` and wait out the full exit window (default 1 week), during which the 100% commission continues to drain their yield.

### Likelihood Explanation
The attack requires only two sequential transactions from the staker's own address — no special privileges, no external dependencies, no oracle manipulation. Any staker who has opened a delegation pool can execute this. The incentive is high: a staker with a large delegated pool can capture all pool rewards for at least one epoch before delegators can react and exit. The minimum commitment duration is 1 epoch, which is sufficient to steal at least one full epoch of delegator rewards.

### Recommendation
1. **Enforce a time-lock between setting a `CommissionCommitment` and acting on it**: require that `set_commission` with an upward change cannot be called until at least K epochs after `set_commission_commitment` was set, giving delegators time to observe and exit.
2. **Cap the maximum allowed commission increase per commitment**: e.g., limit `max_commission` to `current_commission + some_delta` rather than allowing a jump to 100%.
3. **Apply commission increases with a K-epoch delay** (analogous to how stake balance changes are deferred), so delegators can exit before the new rate takes effect.

### Proof of Concept

```
// Staker starts with commission = 0, pool has delegators.
// Epoch N:
staker.set_commission_commitment(max_commission: 10000, expiration_epoch: N + 1);
staker.set_commission(commission: 10000);
// Commission is now 100% immediately.
// Next reward distribution: delegators receive 0 STRK, staker captures all pool rewards.
// Delegators must wait exit_wait_window (~1 week) to recover principal.
// All yield earned during that window is stolen.
```

            if let Option::Some(commission_commitment) = staker_pool_info

            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);

            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
