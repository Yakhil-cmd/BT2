### Title
ETH Sent to `tick()` When No Minting Is Pending Is Permanently Locked — (`File: L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

`RewardSupplier.sol`'s `tick()` function is `payable` and accepts ETH to cover L1→L2 messaging fees. However, when there are no pending L2 mint requests (`amountToMint == 0`), the function returns immediately without spending any ETH. Because the contract has no `receive()`, no `fallback()`, and no ETH-withdrawal function anywhere in its inheritance chain, any ETH sent in that call is permanently locked.

---

### Finding Description

`tick()` is declared `payable` and splits `msg.value` between two fee-bearing calls:

```solidity
// L1/starkware/solidity/stake/RewardSupplier.sol  lines 107-143
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        ...
        uint256 msgFee = msg.value / 2;
        bridge().depositWithMessage{value: msgFee}(...);   // spends half
        msgFee = msg.value - msgFee;
        messagingContract().sendMessageToL2{value: msgFee}(...); // spends rest
    }
    // ← if amountToMint == 0, returns here; msg.value is never spent
}
```

When `amountToMint == 0` the entire `if` block is skipped. `msg.value` is silently retained by the contract. No path exists to recover it:

- `RewardSupplier` defines no `receive()` or `fallback()`.
- `ProxySupportImpl` / `ProxySupport` define no `receive()`, `fallback()`, or ETH-sweep function.
- The grep across all `L1/**/*.sol` files confirms `receive()` and `fallback()` appear **nowhere** in the L1 codebase. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

Any ETH sent to `tick()` during a no-op call (no pending L2 mint messages) is permanently frozen inside `RewardSupplier`. There is no owner-only sweep, no `receive()` to at least track it, and no upgrade path that would automatically recover it. The loss is irreversible for the caller.

**Matched allowed impact:** *Permanent freezing of funds* (High).

---

### Likelihood Explanation

`tick()` is an **unrestricted public function** — any address may call it. A legitimate operator (or an automated keeper) who sends ETH to pre-fund fees will lose those funds if the call races ahead of any L2 message being available. Because L2→L1 message availability is not atomically guaranteed at call time, this race is realistic in normal operation. A malicious actor can also grief the operator by front-running with a fee-less `tick()` call that consumes the pending messages first, causing the operator's subsequent ETH-bearing call to hit the `amountToMint == 0` branch. [4](#0-3) 

---

### Recommendation

1. **Add a `receive()` function** (even if it only emits an event) so accidental ETH deposits are at least trackable.
2. **Refund unused ETH** at the end of `tick()`:
   ```solidity
   if (msg.value > 0 && amountToMint == 0) {
       (bool ok, ) = msg.sender.call{value: msg.value}("");
       require(ok, "ETH_REFUND_FAILED");
   }
   ```
3. **Add a governance-gated ETH-sweep function** to recover any ETH already locked.

---

### Proof of Concept

1. L2 `RewardSupplier` has sent zero mint-request messages to L1 (or all have already been consumed).
2. Caller invokes `tick{value: 1 ether}()`.
3. `requiredMinting()` returns `(0, 0)` because `messagingContract().l2ToL1Messages(msgHash) == 0`.
4. The `if (amountToMint > 0)` block is skipped entirely.
5. Function returns; `address(rewardSupplier).balance` increases by 1 ETH.
6. No function in the contract or its parents can move that ETH out — it is permanently locked. [5](#0-4) [1](#0-0)

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }

abstract contract ProxySupport is MGovernance, BlockDirectCall, ContractInitializer {
    using Addresses for address;

    // The two function below (isFrozen & initialize) needed to bind to the Proxy.
    function isFrozen() external view virtual returns (bool) {
        return false;
    }

    /*
      The initialize() function serves as an alternative constructor for a proxied deployment.

      Flow and notes:
      1. This function cannot be called directly on the deployed contract, but only via
         delegate call.
      2. If an EIC is provided - init is passed onto EIC and the standard init flow is skipped.
         This true for both first intialization or a later one.
      3. The data passed to this function is as follows:
         [sub_contracts addresses, eic address, initData].

         When calling on an initialized contract (no EIC scenario), initData.length must be 0.
    */
    function initialize(bytes calldata data) external notCalledDirectly {
        uint256 eicOffset = 32 * numOfSubContracts();
        uint256 expectedBaseSize = eicOffset + 32;
        require(data.length >= expectedBaseSize, "INIT_DATA_TOO_SMALL");
        address eicAddress = abi.decode(data[eicOffset:expectedBaseSize], (address));

        bytes calldata subContractAddresses = data[:eicOffset];

        processSubContractAddresses(subContractAddresses);

        bytes calldata initData = data[expectedBaseSize:];

        // EIC Provided - Pass initData to EIC and the skip standard init flow.
        if (eicAddress != address(0x0)) {
            callExternalInitializer(eicAddress, initData);
            return;
        }

        if (isInitialized()) {
            require(initData.length == 0, "UNEXPECTED_INIT_DATA");
        } else {
            // Contract was not initialized yet.
            validateInitData(initData);
            initializeContractState(initData);
            initGovernance();
        }
    }

    function callExternalInitializer(address externalInitializerAddr, bytes calldata eicData)
        private
    {
        require(externalInitializerAddr.isContract(), "EIC_NOT_A_CONTRACT");

        // NOLINTNEXTLINE: low-level-calls, controlled-delegatecall.
        (bool success, bytes memory returndata) = externalInitializerAddr.delegatecall(
            abi.encodeWithSelector(this.initialize.selector, eicData)
        );
        require(success, string(returndata));
        require(returndata.length == 0, string(returndata));
    }
}
