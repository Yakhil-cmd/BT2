### Title
Missing Caller Validation on `update_rewards` Allows Any Address to Permanently Freeze Unclaimed Yield - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is callable by any address. The protocol specification explicitly requires this function to be restricted to the Starknet sequencer, but no such check exists in the implementation. Because `last_reward_block` is a global storage variable that gates the function to one call per block, an unprivileged attacker can call `update_rewards` with `disable_rewards: true` at every block, permanently preventing the sequencer from distributing consensus rewards to any staker.

---

### Finding Description

The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 enforces no such restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker existence checks ...
    self.last_reward_block.write(current_block_number);  // <-- global write
    if disable_rewards || self.is_pre_consensus() {
        return;  // <-- exits without distributing rewards
    }
    // ... reward distribution ...
}
```

The critical design detail is that `last_reward_block` is a **global** storage variable — not per-staker. The guard `current_block_number > self.last_reward_block.read()` means only **one** successful call to `update_rewards` is permitted per block across the entire contract. Once any caller writes `current_block_number` into `last_reward_block`, all subsequent calls in the same block revert with `REWARDS_ALREADY_UPDATED`.

An attacker exploits this by:
1. Monitoring the mempool/block production.
2. At each new block, calling `update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)`.
3. This writes `current_block_number` to `last_reward_block` and returns early without distributing any rewards.
4. The sequencer's legitimate call for that block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeating every block permanently freezes all consensus reward accrual. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

All stakers and delegators are denied consensus block rewards indefinitely. The `unclaimed_rewards_own` field of every staker never increases past its current value, and pool members never receive their share of block rewards. The attacker has no profit motive but causes direct, sustained damage to all protocol participants. This matches the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."* [3](#0-2) [4](#0-3) 

---

### Likelihood Explanation

**High.** The entry point is a public external function with no access control. Any Starknet account can call it. The only precondition is that a valid, active staker with non-zero balance exists — which is always true once the protocol has any stakers. The attack requires no special privileges, no tokens, and no capital. A bot calling `update_rewards(..., disable_rewards: true)` at every block is trivially implementable. [5](#0-4) 

---

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`. On Starknet, the sequencer address is accessible via `starknet::get_execution_info().unbox().tx_info.unbox().account_contract_address` or a stored `sequencer_address` config variable. The check should be:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

This mirrors the pattern already used for other privileged callers in the codebase (e.g., `assert_caller_is_staking_contract`). [6](#0-5) 

---

### Proof of Concept

```
1. Deploy the protocol with at least one active staker (staker_address = S).
2. Wait until consensus rewards are active (is_pre_consensus() == false).
3. At every new block N:
   a. Call staking_contract.update_rewards(staker_address: S, disable_rewards: true)
      from any EOA/account.
   b. This succeeds: last_reward_block is set to N, no rewards distributed.
4. The sequencer attempts staking_contract.update_rewards(staker_address: S, disable_rewards: false)
   in the same block N → reverts with REWARDS_ALREADY_UPDATED.
5. Repeat step 3 every block.
6. Observe: staker_info.unclaimed_rewards_own remains 0 indefinitely.
   All delegators' pool balances also remain frozen.
``` [7](#0-6) [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
