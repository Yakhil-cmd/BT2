### Title
Pool Admin Can Set Uncapped Per-Bin Additional Fees via `setPoolBinAdditionalFees`, Bypassing `maxAdminSpreadFeeE6` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6` / `addFeeSellE6` directly to `MetricOmmPool.setBinAdditionalFees` with **no upper-bound validation**. The pool admin can set either value to `type(uint16).max` (65 535, i.e. 6.5535% in E6 units), which is added on top of the oracle-derived base fee in every swap through that bin. This bypasses the `maxAdminSpreadFeeE6` / `HARD_MAX_SPREAD_FEE_E6` cap system that is supposed to constrain the pool admin, causing traders to pay fees that exceed the protocol-intended hard maximum.

---

### Finding Description

**Entry point — factory (pool-admin gated, no cap check):**

`MetricOmmPoolFactory.setPoolBinAdditionalFees` is callable by the pool admin via `onlyPoolAdmin(pool)`. It forwards the raw `addFeeBuyE6` / `addFeeSellE6` values to the pool with zero validation:

```solidity

function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [1](#0-0) 

**Pool-level write — no cap:**

`MetricOmmPool.setBinAdditionalFees` only validates the bin index; it writes the caller-supplied value directly into `BinState.addFeeBuyE6`:

```solidity
// MetricOmmPool.sol line 464-474

  BinState storage s = _binStates[bin];
  s.addFeeBuyE6 = addFeeBuyE6;   // ← no cap
  s.addFeeSellE6 = addFeeSellE6; // ← no cap

**Contrast with the capped path:**

`setPoolAdminFees` explicitly enforces `maxAdminSpreadFeeE6`:

```solidity

``` [3](#0-2) 

`HARD_MAX_SPREAD_FEE_E6 = 200_000` (20%). `type(uint16).max = 65_535` (6.5535%). The per-bin path has no equivalent guard. [4](#0-3) 

**Swap-time fee application:**

During every swap through the affected bin, `addFeeBuyE6` is added to the oracle-derived base fee before being passed to `SwapMath`:

```solidity
// MetricOmmPool.sol line 910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [5](#0-4) 

Inside `SwapMath.buyToken0InBinSpecifiedOut`, the total fee (including the uncapped per-bin component) determines the gross token1 input the trader must pay:

```solidity
// SwapMath.sol line 409-411
uint256 feeAmountScaled = Math.ceilDiv(amountInScaled * currBinBuyFeeX64, ONE_X64);
amountInScaled += feeAmountScaled;
uint256 protocolFeeAmountScaled = (feeAmountScaled * spreadFeeE6) / 1e6;
``` [6](#0-5) 

The LP fee (`feeAmountScaled - protocolFeeAmountScaled`) is retained in the bin's `token1BalanceScaled`, accruing to LPs. A pool admin who is also an LP (or who controls LP positions) directly captures the excess.

---

### Impact Explanation

Every swap through the manipulated bin charges the trader an additional up to 6.5535% on top of the base oracle spread fee. With a base spread fee already at the 20% hard cap, the effective fee reaches ~26.5535%, exceeding `HARD_MAX_SPREAD_FEE_E6`. The excess fee is retained in the bin as LP balance, so a pool admin who holds LP shares extracts it on `removeLiquidity`. Even without LP ownership, every trader through that bin suffers a direct, quantifiable loss of token1 principal on every swap.

---

### Likelihood Explanation

The pool admin role is semi-trusted and is explicitly in scope per the audit's "Admin path" pivot. The call requires only `msg.sender == poolAdmin[pool]` — no timelock, no protocol co-signature, no additional precondition. The pool admin can execute this at any time after pool creation, including immediately before a large swap.

---

### Recommendation

Add an upper-bound check in `MetricOmmPoolFactory.setPoolBinAdditionalFees` mirroring the guard in `setPoolAdminFees`:

```solidity
if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
```

Alternatively, enforce a dedicated per-bin cap constant (e.g. `HARD_MAX_BIN_ADDITIONAL_FEE_E6`) and validate against it in both `setPoolBinAdditionalFees` and `setBinAdditionalFees`.
