### Title
Stale Commission Applied to Accumulated Rewards When Commission Is Increased via Commitment - (File: src/staking/staking.cairo)

### Summary
`set_commission` updates the commission rate in storage immediately without first settling pending rewards. Because `calculate_staker_pools_rewards` reads the commission at the moment of reward distribution, any rewards accumulated since the last distribution are retroactively split using the new (potentially higher) commission. A staker who uses `set_commission_commitment` to raise their commission can drain an entire epoch's worth of delegator yield by increasing commission to the maximum just before triggering reward distribution.

### Finding Description
The reward-distribution path in `_update_rewards` reads the commission from storage at distribution time:

```
let commission = staker_pool_info.commission();   // line 1964
``` [1](#0-0) 

This commission is then used to split every unit of pool rewards accumulated since the last distribution:

```
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [2](#0-1) 

`set_commission`, however, writes the new rate directly to storage with no prior reward settlement:

```
fn set_commission(ref self: ContractState, commission: Commission) {
    ...
    self.update_commission(:staker_address, :staker_pool_info, :old_commission, :commission);
}
``` [3](#0-2) 

`update_commission` only validates the change and writes the value; it never calls `_update_rewards` first:

```
// Update commission in storage.
staker_pool_info.commission.write(Option::Some(commission));
``` [4](#0-3) 

The code itself acknowledges the gap with a comment immediately after `set_commission`:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [5](#0-4) 

Without a commitment, commission can only decrease (`assert!(commission < old_commission)`), so the staker is the only party harmed. The dangerous path is through `set_commission_commitment`, which allows the staker to raise commission up to `max_commission` during the commitment window:

```
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
}
``` [6](#0-5) 

The commitment requires only `expiration_epoch > current_epoch` (minimum 1 epoch ahead), so the window between commitment creation and commission increase can be as short as one epoch.

In the pre-consensus (attestation) model, reward distribution is triggered by the staker's own operational address calling `attest`, which calls `update_rewards_from_attestation_contract`:

```
staking_dispatcher.update_rewards_from_attestation_contract(
    staker_address: staking_attestation_info.staker_address(),
);
``` [7](#0-6) 

This gives the staker full control over the timing of reward distribution within the attestation window, allowing them to raise commission first and then attest.

### Impact Explanation
A staker can steal an entire epoch's worth of delegator yield. With commission raised to 100% (`COMMISSION_DENOMINATOR`), `pool_rewards` becomes zero and `commission_rewards` absorbs everything. Delegators receive nothing for the epoch despite having staked throughout it. This is a direct theft of unclaimed yield, matching the **High: Theft of unclaimed yield** impact category.

### Likelihood Explanation
The attack requires the staker to:
1. Call `set_commission_commitment` with a high `max_commission` (visible on-chain).
2. Wait at least one epoch (the minimum commitment duration).
3. Raise commission and attest in the same epoch.

Delegators who observe the commitment could attempt to exit, but `exit_delegation_pool_intent` initiates a time-based exit wait window (default 1 week). If the epoch length is shorter than 1 week, delegators cannot exit before the commission change takes effect. The staker controls the exact timing of attestation within the attestation window, so they can sequence the commission increase and attestation atomically or within the same block. Likelihood is **medium**: it requires deliberate setup but is fully executable by any staker without privileged access beyond their own staker role.

### Recommendation
Before writing the new commission to storage in `update_commission`, settle all pending rewards for the staker using the old commission. Concretely, call `_update_rewards` (or an equivalent internal settlement function) with the current commission before overwriting it. This mirrors the standard "checkpoint before parameter change" pattern used in most staking protocols.

### Proof of Concept
1. Staker stakes with `commission = 0`, enabling a STRK delegation pool.
2. Delegator delegates a large amount; K epochs pass so the balance becomes effective.
3. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
4. One epoch passes (rewards accumulate; no attestation yet).
5. Staker calls `set_commission(commission: 10000)` — commission is now 100%, no reward settlement occurs.
6. Staker's operational address calls `attest` within the attestation window.
7. `update_rewards_from_attestation_contract` → `_update_rewards` → `calculate_staker_pools_rewards` reads `commission = 10000`, so `pool_rewards = 0` and `commission_rewards = full_epoch_rewards`.
8. Delegator calls `claim_rewards` and receives 0 STRK for the epoch, while the staker has claimed the full epoch reward including the delegator's share. [8](#0-7) [9](#0-8)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );
                total_commission_rewards += commission_rewards;
                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)
