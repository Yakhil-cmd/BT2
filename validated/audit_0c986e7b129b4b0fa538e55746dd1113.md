### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Skip Block Reward Distribution — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is specified to be callable only by the Starkware sequencer, but no access control check is implemented. Any unprivileged caller can invoke it with `disable_rewards: true`, which writes `last_reward_block` to the current block number without distributing any rewards. Because `last_reward_block` is a single global storage slot, this permanently prevents the legitimate sequencer from distributing rewards for that block to all stakers.

---

### Finding Description

The protocol specification for `update_rewards` states:

> **Access control:** Only starkware sequencer. [1](#0-0) 

Every other privileged function in the codebase enforces its access control with an explicit assertion (e.g., `assert_caller_is_attestation_contract()`, `only_app_governor()`). The `update_rewards` implementation, however, only calls `general_prerequisites()` — which checks the pause flag — and then proceeds with no caller identity check: [2](#0-1) 

Critically, `last_reward_block` is written to the current block number **before** the `disable_rewards` early-return branch: [3](#0-2) 

`last_reward_block` is a single global storage variable (not per-staker): [4](#0-3) 

The guard at the top of `update_rewards` enforces that the function can only be called once per block: [5](#0-4) 

Therefore, if any address calls `update_rewards(valid_staker, disable_rewards: true)` before the sequencer does, the block is permanently marked as processed with zero rewards distributed. The sequencer's subsequent call reverts with `REWARDS_ALREADY_UPDATED`, and the rewards for that block are irrecoverably lost for all stakers.

The missing invocation is the access control guard — analogous to the external report's accidentally removed `processL2Tx` call — causing the critical reward-distribution step to be silently skipped.

---

### Impact Explanation

- **Permanent freezing of unclaimed yield** for all stakers for every targeted block.
- An attacker who front-runs the sequencer on every block can suppress all consensus-phase reward distribution indefinitely.
- Delegators in all pools also lose their share of rewards, since pool rewards flow through the same `_update_rewards` path: [6](#0-5) 

---

### Likelihood Explanation

- The entry point is fully public — no role, key, or privileged access is required.
- The attack requires only a single transaction per block: `update_rewards(any_valid_active_staker, disable_rewards: true)`.
- The attacker need not hold any stake or tokens; they only need to know a valid active staker address (which is public on-chain via `get_stakers`).
- The attack is repeatable every block with negligible cost.

---

### Recommendation

Add an explicit sequencer-identity check at the top of `update_rewards`, consistent with how other restricted functions are guarded:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    // ...
}
```

Alternatively, if Starknet does not expose a sequencer-identity syscall, restrict the caller to a configurable `sequencer_address` storage variable governed by the `APP_GOVERNOR` role, mirroring the pattern used for `attestation_contract`.

---

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`).
2. Block `N` begins. The sequencer prepares to call `update_rewards(staker_A, disable_rewards: false)`.
3. Attacker submits `update_rewards(staker_A, disable_rewards: true)` in the same block, front-running the sequencer.
4. Inside the call: `last_reward_block` is written to `N`; the function returns early — no rewards distributed.
5. Sequencer's call arrives: `assert!(current_block_number > self.last_reward_block.read())` fails → `REWARDS_ALREADY_UPDATED`.
6. All stakers and delegators receive zero rewards for block `N`. The loss is permanent.
7. Repeat for every subsequent block to suppress all reward distribution. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
