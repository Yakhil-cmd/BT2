### Title
Unprivileged Caller Can Permanently Freeze Block Rewards by Calling `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
In `update_rewards`, the global `last_reward_block` is advanced to the current block **before** the early-return guard that skips reward distribution when `disable_rewards` is `true`. Because `update_rewards` has no access-control beyond the pause check, any unprivileged caller can invoke it with `disable_rewards: true`, consuming the block's reward slot without distributing any rewards. Subsequent legitimate calls for the same block revert with `REWARDS_ALREADY_UPDATED`, permanently destroying that block's unclaimed yield for every staker.

---

### Finding Description

`update_rewards` in `src/staking/staking.cairo` is the consensus-phase entry point for per-block reward distribution. Its logic is:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);   // ← global state advanced HERE

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // ← early return, no rewards sent
    }
    // reward distribution only reached when disable_rewards == false
    ...
}
``` [1](#0-0) 

The critical ordering is:

1. `last_reward_block` is written to `current_block_number` unconditionally. [2](#0-1) 
2. Immediately after, if `disable_rewards` is `true`, the function returns without distributing any rewards. [3](#0-2) 
3. The uniqueness invariant `current_block_number > last_reward_block` means no second call can ever distribute rewards for that block. [4](#0-3) 

`last_reward_block` is a **single global** storage slot, not per-staker:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [5](#0-4) 

`general_prerequisites()` enforces only the pause flag — there is no role check on `update_rewards`. [6](#0-5) 

---

### Impact Explanation

This is a **permanent freezing of unclaimed yield** (High impact per scope).

- `last_reward_block` is global: one attacker call with `disable_rewards: true` at block N burns the reward slot for **all** stakers for block N.
- The rewards are not deferred — they are simply never computed or credited. The `_update_rewards` path that calls `update_unclaimed_rewards_from_staking_contract` and transfers tokens to stakers and pools is never reached. [7](#0-6) 
- The attack is repeatable every block at negligible cost.

---

### Likelihood Explanation

- The function is publicly callable (no role restriction).
- The attacker only needs to submit one transaction per block.
- The attack is cheap (a single contract call with a valid staker address and `disable_rewards: true`).
- The consensus rewards phase is the production target; the pre-consensus guard (`is_pre_consensus()`) does not protect against this because the attacker can wait until consensus rewards are active.

---

### Recommendation

Move the `last_reward_block` write to **after** the early-return guard, so that a skipped-reward call does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Only advance last_reward_block when rewards are actually distributed.
self.last_reward_block.write(current_block_number);

let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
...
```

Alternatively, restrict `disable_rewards: true` to a privileged caller (e.g., `only_app_governor`), or remove the parameter entirely and derive the skip condition from on-chain state only.

---

### Proof of Concept

1. Protocol transitions to consensus rewards phase (`is_pre_consensus()` returns `false`).
2. At block N, attacker calls:
   ```
   staking.update_rewards(any_valid_staker_address, disable_rewards: true)
   ```
3. `last_reward_block` is written to N; function returns immediately with no rewards distributed.
4. Any legitimate caller (sequencer, staker, keeper) attempts:
   ```
   staking.update_rewards(staker_address, disable_rewards: false)
   ```
   at block N → reverts with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. Block N's rewards are permanently lost for every staker and delegator in the protocol.

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self

                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
