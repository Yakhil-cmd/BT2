### Title
Gateway Stateful Validator Skips L1/L1DataGas Price Threshold Check, Allowing Zero-Price Transactions to Bypass Admission and Waste Block Space - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` only validates the `l2_gas.max_price_per_unit` field against the previous-block threshold. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields are never checked at the gateway admission layer. A transaction with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` (but a valid L2 gas price) passes every gateway check, enters the mempool, is included in a block, and then fails at blockifier pre-validation with `InsufficientResourceBounds` — without paying any fee and without incrementing the nonce. This allows an unprivileged attacker to repeatedly fill blocks with zero-cost failing transactions.

### Finding Description

**Stateless validator** (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, `validate_resource_bounds`):

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

Only `l2_gas.max_price_per_unit` is compared against `min_gas_price`. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are unchecked. [1](#0-0) 

The non-zero fee guard (`max_possible_fee(Tip::ZERO) == Fee(0)`) is satisfied as long as any single resource has a non-zero `max_amount × max_price_per_unit`, so a transaction with only `l2_gas` non-zero passes this check too. [2](#0-1) 

**Stateful validator** (`crates/apollo_gateway/src/stateful_transaction_validator.rs`, `validate_tx_l2_gas_price_within_threshold`):

```rust
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(
    &self,
    tx_resource_bounds: ValidResourceBounds,
    previous_block_l2_gas_price: NonzeroGasPrice,
) -> StatefulTransactionValidatorResult<()> {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // ... only l2_gas is checked
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
    Ok(())
}
```

The TODO comment explicitly acknowledges the gap. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared against any threshold. [3](#0-2) 

**Blockifier enforcement** (`crates/blockifier/src/transaction/account_transaction.rs`, `check_fee_bounds`):

The blockifier does check all three gas prices at execution time:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
```

But this check runs inside `perform_pre_validation_stage`, which is called during block execution — after the transaction has already been admitted to the mempool and included in a block. [4](#0-3) 

When `perform_pre_validation_stage` returns `InsufficientResourceBounds`, the transaction execution fails entirely. Because the blockifier uses a transactional state, the nonce increment from `handle_nonce` is rolled back. No fee is charged. [5](#0-4) 

**Mempool** (`crates/apollo_mempool/src/fee_transaction_queue.rs`, `insert`):

The mempool's `FeeTransactionQueue` only uses `max_l2_gas_price` for its priority/pending split. A transaction with `l1_gas.max_price_per_unit = 0` is placed in the priority queue as long as its `max_l2_gas_price` is above the threshold. [6](#0-5) 

### Impact Explanation

An unprivileged attacker can:

1. Submit `AllResources` transactions with `l2_gas.max_price_per_unit` ≥ `min_gas_price` (passes stateless check), `l2_gas.max_amount > 0` (passes non-zero fee check), and `l1_gas.max_price_per_unit = 0`, `l1_data_gas.max_price_per_unit = 0`.
2. All gateway checks pass. The transaction enters the mempool's priority queue.
3. The batcher includes the transaction in a block.
4. The blockifier rejects it at `check_fee_bounds` with `MaxGasPriceTooLow` for L1Gas.
5. No fee is charged. The nonce is not incremented. The attacker's account state is unchanged.
6. The attacker repeats indefinitely, filling blocks with zero-cost failing transactions.

The `verify_can_pay_committed_bounds` check only verifies the account can cover `max_possible_fee()`, which is computed from `l1_gas.max_amount × 0 + l2_gas.max_amount × l2_gas_price + l1_data_gas.max_amount × 0`. With zero L1/L1DataGas prices, the required balance is minimal. [7](#0-6) 

This matches: **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The attack requires only a funded account and the ability to submit RPC transactions. No privileged access is needed. The gap is explicitly acknowledged by a TODO comment in the production code. The default `min_gas_price` is `8_000_000_000` FRI, so the attacker must set `l2_gas.max_price_per_unit` above this value, but can set all other price fields to zero. [8](#0-7) 

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or the stateless `validate_resource_bounds`) to also enforce a minimum threshold on `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit`, analogous to the existing L2 gas price check. The TODO comment at line 358 of `stateful_transaction_validator.rs` already identifies this as a known gap.

### Proof of Concept

```
1. Craft an RpcInvokeTransaction V3 with:
     resource_bounds.l2_gas.max_price_per_unit = 8_000_000_001  // above min_gas_price
     resource_bounds.l2_gas.max_amount         = 1_000_000
     resource_bounds.l1_gas.max_price_per_unit = 0              // zero — not checked
     resource_bounds.l1_gas.max_amount         = 0
     resource_bounds.l1_data_gas.max_price_per_unit = 0         // zero — not checked
     resource_bounds.l1_data_gas.max_amount    = 0

2. Submit via starknet_addInvokeTransaction.

3. StatelessTransactionValidator::validate_resource_bounds passes:
   - max_possible_fee = l2_gas.max_amount × l2_gas.max_price_per_unit > 0  ✓
   - l2_gas.max_price_per_unit >= min_gas_price                             ✓

4. StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold passes:
   - only l2_gas.max_price_per_unit is checked                              ✓

5. verify_can_pay_committed_bounds passes:
   - committed_fee = l2_gas cost only (l1 prices are 0)                    ✓

6. Transaction enters mempool priority queue and is included in a block.

7. Blockifier check_fee_bounds fails:
   - l1_gas.max_price_per_unit (0) < actual_l1_gas_price (non-zero)
   → InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }

8. Transaction fails, no fee charged, nonce unchanged.
   Attacker repeats from step 1 with the same nonce.
```

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {

                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);

                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())

    pub fn perform_pre_validation_stage<S: State + StateReader>(
        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }

pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}
