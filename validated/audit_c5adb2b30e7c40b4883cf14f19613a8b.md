### Title
Missing Contract Existence Check in `CallModuleCRC21` Enables Silent Loss of Native Coins/IBC Vouchers During Token Conversion - (File: x/cronos/keeper/evm.go)

### Summary
`CallModuleCRC21` in `x/cronos/keeper/evm.go` calls `CallEVM` on a mapped contract address without verifying that the contract still has deployed code. If the mapped external contract is self-destructed after registration, the EVM call silently returns success (EVM design: calls to codeless addresses succeed with empty return data), causing `ConvertCoinFromNativeToCRC21` to permanently lose the user's native coins or IBC vouchers without minting any CRC21 tokens.

### Finding Description
`ensureContractCode` is correctly called at registration time in `RegisterOrUpdateTokenMapping`, but is **never called at invocation time** in `CallModuleCRC21` or in `onPacketResult`. [1](#0-0) 

`CallModuleCRC21` calls `CallEVM` directly on the stored contract address: [2](#0-1) 

The only failure check is `res.Failed()`, which tests `VmError != ""`. A call to a codeless (self-destructed) address produces no VM error and empty return data — `res.Failed()` is `false` — so the function returns `([]byte{}, nil)` as if the call succeeded.

`ConvertCoinFromNativeToCRC21` then proceeds as if the ERC20 operation completed:

For **non-source coins** (IBC vouchers): [3](#0-2) 

1. Native coins are transferred from the user to the contract's bank address (bank module, succeeds regardless of EVM code).
2. `mint_by_cronos_module` is called on the destroyed contract — silently succeeds.
3. No CRC21 tokens are minted. User's native coins are permanently stuck at the contract's bank address.

For **source coins** (`cronos0x...`): [4](#0-3) 

1. Native coins are burned from the user's account.
2. `transfer_from_cronos_module` is called on the destroyed contract — silently succeeds.
3. No CRC21 tokens are transferred. User's native coins are permanently burned with no compensation.

The same pattern exists in `onPacketResult`, which calls `CallEVM` on the ICA sender address without a code check: [5](#0-4) 

The existence check utility is present but only used at registration: [6](#0-5) 

### Impact Explanation
An external contract registered in the token mapping can be self-destructed by its owner after admin registration. Any subsequent call to `ConvertVouchersToEvmCoins` (via `MsgConvertVouchers` or IBC auto-conversion middleware) for that denom will silently burn or strand the user's IBC vouchers/native coins without minting CRC21 tokens. This is an unauthorized, irreversible destruction of user balances — matching the Critical impact class: *unauthorized burn/transfer of IBC vouchers or CRC20/CRC21 assets*. [7](#0-6) [8](#0-7) 

### Likelihood Explanation
- Admin registers an external CRC21 contract for an IBC denom (standard operational flow).
- The contract owner (distinct from admin) calls `selfdestruct` on the contract post-registration.
- Any user who subsequently converts vouchers for that denom via `MsgConvertVouchers` or IBC auto-conversion is affected.
- The `ensureContractCode` guard exists only at registration time, not at call time, so the stale mapping persists indefinitely.

### Recommendation
Add a contract existence check inside `CallModuleCRC21` before invoking `CallEVM`, reusing the existing `ensureContractCode` helper:

```go
func (k Keeper) CallModuleCRC21(ctx sdk.Context, contract common.Address, method string, args ...interface{}) ([]byte, error) {
    if err := k.ensureContractCode(ctx, contract); err != nil {
        return nil, fmt.Errorf("contract code missing at %s: %w", contract.Hex(), err)
    }
    // ... existing logic
}
```

Similarly, add the same guard at the top of `onPacketResult` before calling `CallEVM`.

### Proof of Concept
1. Admin calls `MsgUpdateTokenMapping` registering external contract `0xABCD...` for denom `ibc/XXXX`. `ensureContractCode` passes at registration time.
2. Contract owner calls `selfdestruct` on `0xABCD...`. EVM storage is cleared; `extcodesize(0xABCD...)` returns 0.
3. User calls `MsgConvertVouchers` with 1000 `ibc/XXXX` tokens.
4. `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` → `bankKeeper.SendCoins(user → 0xABCD...)` succeeds (1000 tokens now at contract bank address).
5. `CallModuleCRC21(ctx, 0xABCD, "mint_by_cronos_module", user, 1000)` → `CallEVM` → EVM executes empty code → `res.Failed() == false`, returns `([]byte{}, nil)`.
6. No CRC21 tokens minted. User's 1000 IBC vouchers are permanently stranded at `0xABCD`'s bank address with no recovery path.

func (k Keeper) CallModuleCRC21(ctx sdk.Context, contract common.Address, method string, args ...interface{}) ([]byte, error) {
	data, err := types.ModuleCRC21Contract.ABI.Pack(method, args...)
	if err != nil {
		return nil, err
	}
	_, res, err := k.CallEVM(ctx, &contract, data, big.NewInt(0), DefaultGasCap)
	if err != nil {
		return nil, err
	}
	if res.Failed() {
		return nil, fmt.Errorf("call contract failed: %s, %s, %s", contract.Hex(), method, res.Ret)
	}
	return res.Ret, nil
}

func (k Keeper) ensureContractCode(ctx sdk.Context, contract common.Address) error {
	if contract.Big().Cmp(big.NewInt(256)) < 0 {
		return errors.Wrapf(sdkerrors.ErrInvalidAddress,
			"crc21 contract must not be in precompile range: %s", contract.Hex())
	}
	resp, err := k.evmKeeper.Code(ctx, &evmtypes.QueryCodeRequest{
		Address: contract.Hex(),
	})
	if err != nil {
		return errors.Wrapf(sdkerrors.ErrInvalidAddress, "failed to query contract code (%s): %v", contract.Hex(), err)
	}
	if resp == nil || len(resp.Code) == 0 {
		return errors.Wrapf(sdkerrors.ErrInvalidRequest, "no contract code at address (%s)", contract.Hex())
	}
	return nil
}
