### Title
Commented-Out `nftSellerNetAmount` Deduction Causes NFT Offer Viewer to Display Inflated "Net Proceeds" to NFT Sellers тАФ (`packages/gui/src/components/offers/utils.ts`)

---

### Summary

In `calculateNFTRoyalties()`, the correct formula for `nftSellerNetAmount` (asking price minus royalties and maker fee) is commented out. As a result, `nftSellerNetAmount` is always set to the full asking price. This value is surfaced in two places in the NFT offer viewer: the prominent "Net Proceeds" label and the "You will receive" summary row shown to an NFT seller before they accept a `TokenForNFT` offer. The seller is shown an inflated amount and may accept an offer believing they will receive more than they actually will.

---

### Finding Description

In `packages/gui/src/components/offers/utils.ts`, `calculateNFTRoyalties` computes the amount the NFT seller will net after royalties and maker fees are deducted. The correct deduction is commented out:

```typescript
// packages/gui/src/components/offers/utils.ts, lines 314тАУ317
const nftSellerNetAmount: number = amount;
// : parseFloat(
//     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
//   );
``` [1](#0-0) 

Because the deduction is commented out, `nftSellerNetAmount` always equals `amount` (the full asking price), regardless of the NFT's `royaltyPercentage` or the maker fee.

This value propagates into two display paths in `NFTOfferViewer.tsx`:

**Path 1 тАФ "Net Proceeds" label (line 596):** When a `TokenForNFT` offer is viewed (someone offering tokens to buy an NFT), the viewer shows a "Net Proceeds" section with the tooltip *"The net proceeds include the asking price, minus any associated creator fees."* The displayed value is `nftSaleInfo?.nftSellerNetAmount`, which equals the full asking price instead of `amount - royaltyAmount - makerFee`. [2](#0-1) 

**Path 2 тАФ "You will receive" summary row (lines 397тАУ402):** `overrideNFTSellerAmount` is computed from `nftSaleInfo?.nftSellerNetAmount` and passed to `NFTOfferSummary` тЖТ `NFTOfferSummaryRow` тЖТ `OfferSummaryTokenRow`, where it overrides the displayed amount in the "You will receive" row. [3](#0-2) 

In `OfferSummaryTokenRow`, the override is applied directly:

```typescript
const amount = overrideNFTSellerAmount ?? originalAmount;
``` [4](#0-3) 

The same bug also affects the offer creation editor (`NFTOfferEditor.tsx`), where the "You will receive" line shown to the NFT seller during offer creation is equally inflated. [5](#0-4) 

---

### Impact Explanation

An NFT seller viewing a `TokenForNFT` offer (i.e., deciding whether to accept a buy offer for their NFT) is shown:

- **"Net Proceeds"** = full asking price (e.g., 10 XCH)
- **"You will receive"** = full asking price (e.g., 10 XCH)

But the actual on-chain settlement deducts royalties from the seller's proceeds. For an NFT with a 30% royalty and a 10 XCH asking price, the seller actually receives 7 XCH. The GUI displays 10 XCH in both the "Net Proceeds" and "You will receive" fields, directly contradicting the tooltip's own description. The seller accepts the offer under a false understanding of the amount they will receive.

This matches the **High** impact class: *"Corruption, spoofing, or unsafe trust ofтАж offerтАж state that causes a user toтАж display the wrong asset, identity, amount, destination, or status."*

---

### Likelihood Explanation

Any unprivileged user can create a `TokenForNFT` offer for an NFT that carries royalties. The NFT owner who views and accepts that offer will always see the inflated "Net Proceeds" figure. The bug is triggered by the normal offer-viewing flow with no special conditions required. NFTs with non-trivial royalty percentages (5тАУ30%) are common, making the discrepancy material.

---

### Recommendation

Uncomment and restore the correct `nftSellerNetAmount` formula in `calculateNFTRoyalties`:

```typescript
// packages/gui/src/components/offers/utils.ts
const nftSellerNetAmount: number = parseFloat(
  (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
);
``` [1](#0-0) 

This will cause the "Net Proceeds" and "You will receive" values in both `NFTOfferViewer` and `NFTOfferEditor` to correctly reflect the amount the NFT seller will actually receive after royalties and fees are deducted.

---

### Proof of Concept

1. Mint or obtain an NFT with a 20% royalty (`royaltyPercentage = 2000`).
2. Have a second wallet create a `TokenForNFT` offer: offer 10 XCH for the NFT.
3. Open the offer in the NFT owner's wallet via the offer viewer (`NFTOfferViewer`).
4. Observe:
   - **"Net Proceeds"** displays **10 XCH** (full asking price).
   - **"You will receive"** in the Purchase Summary also displays **10 XCH**.
   - The tooltip states *"The net proceeds include the asking price, minus any associated creator fees."*
5. Accept the offer. The on-chain settlement pays the royalty address 2 XCH and the NFT seller receives only **8 XCH** тАФ 2 XCH less than the GUI displayed.
