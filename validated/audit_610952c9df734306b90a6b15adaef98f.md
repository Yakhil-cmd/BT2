### Title
`StatelessTransactionValidator::validate_resource_bounds` unconditionally applies `min_gas_price` to `l2_gas.max_price_per_unit`, causing valid transactions with only `l1_gas`/`l1_data_gas` bounds to be rejected at gateway admission - (File: crates/apollo_gateway/src/stateless_transaction_validator.rs)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` checks `l2_gas.max_price_per_unit` against `min_gas_price` for every transaction regardless of whether the transaction carries any `l2_gas` bounds at all. In production the default `min_gas_price` is `8_000_000_000`. A valid V3 transaction that sets only `l1_gas` or `l1_data_gas` bounds (leaving `l2_gas` at its zero default) will always have `l2_gas.max_price_per_unit = 0`, which is less than `8_000_000_000`, so the gateway unconditionally rejects it with `MaxGasPriceTooLow`. The protocol explicitly supports such transactions (confirmed by the `valid_l1_gas` and `valid_l1_data_gas` positive test cases), but those tests only pass because the test fixture hard-codes `min_gas_price: 0`, masking the production breakage.

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs` the resource-bounds validation is:

```rust
// line 71-76
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
```

There is no guard that skips this check when `l2_gas.max_amount == 0`. The `ZeroResourceBounds` check immediately above only requires that *some* resource bound is non-zero; it does not require `l2_gas` to be non-zero. Therefore a transaction with:

```
AllResourceBounds {
    l1_gas:      ResourceBounds { max_amount: N, max_price_per_unit: P },
    l2_gas:      ResourceBounds::default(),   // max_amount=0, max_price_per_unit=0
    l1_data_gas: ResourceBounds::default(),
}
```

passes the `ZeroResourceBounds` check (because `l1_gas` fee > 0) but immediately fails the `MaxGasPriceTooLow` check because `0 < 8_000_000_000`.

The production default is confirmed in `crates/apollo_gateway_config/src/config.rs`:

```rust
// line 192
min_gas_price: 8_000_000_000,
```

and in the deployment config `crates/apollo_deployments/resources/app_configs/gateway_config.json`:

```json
"gateway_config.static_config.stateless_tx_validator_config.min_gas_price": 8000000000
```

The test fixture that hides this is in `crates/apollo_gateway/src/stateless_transaction_validator_test.rs`:

```rust
// line 54-67
static DEFAULT_VALIDATOR_CONFIG_FOR_TESTING: LazyLock<StatelessTransactionValidatorConfig> =
    LazyLock::new(|| StatelessTransactionValidatorConfig {
        validate_resource_bounds: false,
        min_gas_price: 0,          // ← masks the production value
        ...
    });
```

The positive test cases `valid_l1_gas` and `valid_l1_data_gas` spread this fixture and override only `validate_resource_bounds: true`, so they inherit `min_gas_price: 0` and pass. With the production value `8_000_000_000` they would fail.

The stateful validator's `validate_tx_l2_gas_price_within_threshold` correctly handles this asymmetry — it explicitly skips the check for `ValidResourceBounds::L1Gas(_)` with the comment "No validation required for legacy transactions" and carries a TODO: "Consider running this validation for all gas prices." The stateless validator has no equivalent guard.

### Impact Explanation

Any V3 transaction that legitimately carries only `l1_gas` or `l1_data_gas` resource bounds is permanently rejected at the gateway's stateless validation stage before it can reach the mempool or batcher. The rejection is deterministic and unconditional: no amount of fee escalation or retry can fix it because the `l2_gas.max_price_per_unit` field is structurally zero for such transactions. This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The condition is triggered by any caller that constructs a V3 transaction with non-zero `l1_gas` bounds and zero `l2_gas` bounds — a valid and protocol-supported combination. The production config has `validate_resource_bounds: true` and `min_gas_price: 8_000_000_000`, so the rejection path is always active. No special privilege or race condition is required; any unprivileged user can observe the rejection.

### Recommendation

Guard the `min_gas_price` check so it only fires when the transaction actually declares a non-zero `l2_gas` amount:

```rust
// In validate_resource_bounds, replace lines 71-76 with:
if resource_bounds.l2_gas.max_amount.0 > 0
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
```

Additionally, add a test case that runs `valid_l1_gas` and `valid_l1_data_gas` with the production `min_gas_price` value to prevent regression.

### Proof of Concept

1. Construct a V3 Invoke transaction with:
   ```
   AllResourceBounds {
       l1_gas:      ResourceBounds { max_amount: 1_000, max_price_per_unit: 100_000_000_000 },
       l2_gas:      ResourceBounds::default(),
       l1_data_gas: ResourceBounds::default(),
   }
   ```
2. Submit to the gateway running with the default production config (`min_gas_price = 8_000_000_000`, `validate_resource_bounds = true`).
3. `validate_resource_bounds` evaluates `0 < 8_000_000_000 → true` and returns `Err(MaxGasPriceTooLow { gas_price: GasPrice(0), min_gas_price: 8_000_000_000 })`.
4. The transaction is rejected before reaching the mempool.

The existing test `valid_l1_gas` (line 70–82 of `stateless_transaction_validator_test.rs`) is the direct positive-flow counterpart; it passes only because `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING.min_gas_price = 0`. Replacing that fixture with `StatelessTransactionValidatorConfig::default()` (production) causes the test to fail, confirming the bug. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
