### Title
NFT Offer "Net Proceeds" and "You Will Receive" Amounts Overstate Seller's True Payout Due to Omitted Royalty Deduction - (File: packages/gui/src/components/offers/utils.ts)

### Summary
In `calculateNFTRoyalties()`, the `nftSellerNetAmount` field is hardcoded to the full offered `amount` instead of `amount - royaltyAmount`. This incorrect value propagates into the NFT offer viewer, causing the "Net Proceeds" label and the "You will receive" token row to display the gross offered amount rather than the seller's actual post-royalty proceeds. An NFT seller viewing a `TokenForNFT` offer is shown an inflated receive amount and may accept an offer they would otherwise reject.

### Finding Description

In `packages/gui/src/components/offers/utils.ts`, `calculateNFTRoyalties()` computes the seller's net amount as:

```typescript
const nftSellerNetAmount: number = amount;
// : parseFloat(
//     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
//   );
``` [1](#0-0) 

The commented-out block was the correct formula. The live code sets `nftSellerNetAmount = amount` unconditionally, ignoring `royaltyAmount`.

This value flows into `NFTOfferViewer.tsx` in two ways:

**Path 1 — "Net Proceeds" label (TokenForNFT only):**

```tsx
<FormatLargeNumber value={new BigNumber(nftSaleInfo?.nftSellerNetAmount ?? 0)} /> {displayName}
``` [2](#0-1) 

The tooltip explicitly promises "The net proceeds include the asking price, **minus** any associated creator fees", but the displayed value never subtracts them. [3](#0-2) 

**Path 2 — "You will receive" token row override:**

```typescript
const overrideNFTSellerAmount =
  exchangeType === NFTOfferExchangeType.TokenForNFT
    ? assetType === OfferAsset.CHIA
      ? chiaToMojo(nftSaleInfo?.nftSellerNetAmount ?? 0)
      : catToMojo(nftSaleInfo?.nftSellerNetAmount ?? 0)
    : undefined;
``` [4](#0-3) 

This override is passed through `NFTOfferSummary` → `NFTOfferSummaryRow` → `OfferSummaryTokenRow`, where it replaces the displayed amount:

```typescript
const amount = overrideNFTSellerAmount ?? originalAmount;
``` [5](#0-4) 

Because `nftSellerNetAmount = amount` (gross), `overrideNFTSellerAmount ≈ originalAmount`, so the "You will receive" row shows the full gross amount — the same as if no royalty existed.

### Impact Explanation

For a `TokenForNFT` offer (buyer offers X tokens for an NFT with a Y% royalty):

| Field shown in GUI | Displayed value | Correct value |
|---|---|---|
| "You will receive" | X tokens | X − (X × Y%) tokens |
| "Net Proceeds" | X tokens | X − (X × Y%) tokens |

The NFT seller (taker) sees an overstated receive amount and may accept an offer they would reject if shown the true net proceeds. The on-chain settlement correctly deducts the royalty, so the seller receives less than the GUI promised. This is a direct accounting/balance impact on XCH or CAT assets. [6](#0-5) 

### Likelihood Explanation

Any `TokenForNFT` offer involving an NFT with a non-zero `royaltyPercentage` triggers this path. NFT royalties are a standard feature; many Chia NFTs carry royalties of 3–10%+. The seller need only open an imported offer file or a shared offer link — no special attacker action is required. The bug is always active.

### Recommendation

In `calculateNFTRoyalties()`, restore the correct net-amount formula:

```typescript
const nftSellerNetAmount: number = parseFloat(
  (amount - parseFloat(royaltyAmountString)).toFixed(12),
);
``` [7](#0-6) 

Also verify that `overrideNFTSellerAmount` is only set when `nftSaleInfo` is non-null (currently `nftSaleInfo?.nftSellerNetAmount ?? 0` silently passes `0` when NFT metadata is unavailable, which would zero out the displayed receive amount). [4](#0-3) 

### Proof of Concept

1. Mint or obtain an NFT with a 10% royalty (`royaltyPercentage = 1000` in Chia encoding).
2. A buyer creates a `TokenForNFT` offer: "I offer 100 XCH for NFT X."
3. The NFT seller opens the offer file in Chia GUI (`NFTOfferViewer`).
4. GUI displays:
   - "You will receive: **100 XCH**"
   - "Creator Fee (10%): 10 XCH"
   - "Net Proceeds: **100 XCH**"
5. Seller accepts, believing they will net 100 XCH.
6. On-chain settlement pays 10 XCH royalty to the creator; seller receives **90 XCH** — 10 XCH less than the GUI stated. [8](#0-7) [9](#0-8)

export function calculateNFTRoyalties(
  amount: number,
  makerFee: number,
  royaltyPercentage: number,
  exchangeType: NFTOfferExchangeType,
): CalculateNFTRoyaltiesResult {
  const royaltyAmount: number = royaltyPercentage ? (royaltyPercentage / 100) * amount : 0;
  const royaltyAmountString: string = formatAmount(royaltyAmount);
  const nftSellerNetAmount: number = amount;
  // : parseFloat(
  //     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
  //   );
  const totalAmount: number =
    exchangeType === NFTOfferExchangeType.NFTForToken ? amount + royaltyAmount : amount + makerFee + royaltyAmount;
  const totalAmountString: string = formatAmount(totalAmount);

  return {
    royaltyAmount,
    royaltyAmountString,
    nftSellerNetAmount,
    totalAmount,
    totalAmountString,
  };
}
