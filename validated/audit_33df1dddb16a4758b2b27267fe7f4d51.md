### Title
Malicious Equivocating Sender Triggers Early-Abort in `reliable_broadcast_receive_all`, Permanently Aborting All Honest Parties in DKG — (`src/protocol/echo_broadcast.rs`)

---

### Summary

The non-standard early-abort optimization added at lines 241–263 of `reliable_broadcast_receive_all` causes **all** honest parties to return `Err(AssertionFailed(...))` when a malicious sender equivocates (sends value A to some parties, value B to others) and then echoes the minority value. Because `do_keyshare` propagates this error directly, the entire DKG/reshare/refresh run is permanently aborted for every honest participant. The attack requires only one malicious party and works within the documented `MaxFaulty = (N-1)/3` tolerance.

---

### Finding Description

`echo_ready_thresholds` computes, for n=4:

```
broadcast_threshold = (4-1)/3 = 1
echo_threshold      = midpoint(4, 1) = 2
``` [1](#0-0) 

The early-abort block fires whenever, after counting all echoes received so far, no single value has enough remaining potential votes to strictly exceed `echo_t`:

```rust
let non_received_echo_cnt = n - received_echo_cnt;
for (_, cnt) in state_sid.data_echo.iter() {
    if cnt + non_received_echo_cnt > echo_t { is_enough = true; break; }
}
if !is_enough {
    return Err(ProtocolError::AssertionFailed(...));
}
``` [2](#0-1) 

**Concrete attack (n=4, threshold=2, 1 malicious M):**

| Step | Action |
|------|--------|
| Send phase | M sends `Send(true)` to H0, H1; `Send(false)` to H2 |
| Echo phase | H0 echoes `true`; H1 echoes `true`; H2 echoes `false`; M's simulated echo is `false` (version 1) |
| Each honest party's final tally for M's session | `true=2`, `false=2`, `total=4`, `non_received=0` |
| Early-abort check | `true: 2+0=2`, NOT `> 2`; `false: 2+0=2`, NOT `> 2` → `is_enough=false` → **ABORT** |

All three honest parties hit the same arithmetic dead-end simultaneously and all return `Err(AssertionFailed("The original sender in session 3 is malicious! ..."))`.

This is not a theoretical scenario — the existing test `test_three_honest_one_dihonest` **version 1** already asserts this exact error as the expected outcome: [3](#0-2) 

The error propagates unhandled through `do_broadcast` → `do_keyshare` → `do_keygen`/`do_reshare`: [4](#0-3) 

---

### Impact Explanation

Every honest party's DKG/reshare/refresh run terminates with a fatal `ProtocolError::AssertionFailed`. There is no recovery path inside the protocol; callers must restart from scratch. Because the malicious party can repeat the same equivocation on every restart, the denial is persistent. This matches **High: Permanent denial of key generation, reshare, or refresh for honest parties under valid protocol inputs and documented trust assumptions**.

---

### Likelihood Explanation

- Requires exactly **one** malicious participant — within the documented `MaxFaulty = (N-1)/3` bound (n=4 → MaxFaulty=1).
- The malicious party controls only its own `send_private` calls and its own simulated echo value; no cryptographic break is needed.
- The attack is already encoded as a test helper (`do_broadcast_dishonest_consume_version_1`) and confirmed to succeed.
- Any deployment with n ≥ 4 is affected. [5](#0-4) 

---

### Recommendation

Remove or gate the early-abort optimization. The standard Bracha reliable broadcast (Cachin–Guerraoui–Rodrigues) does **not** include this check; it simply waits. If early termination is desired for latency reasons, it must only fire when it is **impossible** for the protocol to deliver **any** value to **any** honest party — a strictly stronger condition that requires accounting for the possibility that the sender equivocated. The simplest safe fix is to delete lines 241–263 entirely and rely on the existing timeout/cancellation mechanism to handle a permanently stalled session.

---

### Proof of Concept

The existing test already proves the path. To make it explicit as a standalone PoC:

```rust
// n=4: 3 honest (H0,H1,H2) + 1 malicious (M=3)
// M sends true to H0,H1 and false to H2, then echoes false itself.
// Expected: ALL honest parties return Err(AssertionFailed("...malicious..."))
let result = broadcast_dishonest(
    &generate_participants(3),
    Participant::from(3u32),
    &[true, true, true],
    do_broadcast_dishonest_consume_version_1,  // split-send + minority echo
);
assert!(matches!(result, Err(ProtocolError::AssertionFailed(_))));
// Honest parties H0, H1, H2 all aborted — DKG permanently denied.
``` [6](#0-5)

fn echo_ready_thresholds(n: usize) -> (usize, usize) {
    // case where no malicious parties are assumed: when n <= 3/
    // In this case the echo and ready thresholds are both 0
    // later we compare if we have collected more votes than these thresholds
    if n <= 3 {
        return (0, 0);
    }
    // we should always have n >= 3*threshold + 1
    let broadcast_threshold = (n - 1) / 3;
    let echo_threshold = usize::midpoint(n, broadcast_threshold);
    (echo_threshold, broadcast_threshold)
}

        let vote_false = MessageType::Send(false);
