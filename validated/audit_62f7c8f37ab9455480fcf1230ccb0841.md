### Title
Any Unprivileged Caller Can Permanently Deny Consensus Rewards to All Stakers via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` in the `Staking` contract is a public, unauthenticated function that accepts a `disable_rewards: bool` parameter. The function unconditionally writes `last_reward_block` to the current block number **before** checking `disable_rewards`. Because `last_reward_block` is a global gate that prevents any second call in the same block, an attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to consume the per-block reward slot without distributing any rewards, permanently starving all stakers of consensus-mode block rewards.

---

### Finding Description

`update_rewards` is exposed as part of `IStakingRewardsManager` with no role guard ā€” only `general_prerequisites()` is called, which checks for pause state and a non-zero caller. [1](#0-0) 

The function first validates the staker, then writes `last_reward_block` to the current block number, and only **after** that checks `disable_rewards`: [2](#0-1) 

Because `last_reward_block` is written before the early-return branch, any subsequent call to `update_rewards` in the same block will fail with `REWARDS_ALREADY_UPDATED`: [3](#0-2) 

The `general_prerequisites` function imposes no role restriction ā€” it only asserts the contract is unpaused and the caller is non-zero: [4](#0-3) 

The attacker only needs to supply any currently-active staker address with non-zero balance (publicly readable from chain state) as the `staker_address` argument.

---

### Impact Explanation

In consensus-rewards mode (post `consensus_rewards_first_epoch`), block rewards are distributed exactly once per block via `update_rewards`. If an attacker calls `update_rewards(valid_staker, disable_rewards: true)` every block, `last_reward_block` is advanced without any rewards being calculated or transferred. All stakers and pool members are denied their per-block yield for as long as the attack continues. This constitutes **temporary (and practically permanent while sustained) freezing of unclaimed yield** for every participant in the protocol.

This maps to the allowed impact: **High ā€” Temporary freezing of funds / Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is fully public; no token, stake, or privileged role is required.
- The attacker's only cost is one transaction per block (gas fees on Starknet L2 are low).
- The attack is trivially automatable with a simple bot.
- The attacker needs only to know one valid active staker address, which is observable on-chain from `NewStaker` events or the `stakers` Vec. [5](#0-4) 

Likelihood is **High**.

---

### Recommendation

Add an authorized-caller check to `update_rewards`. Only the consensus layer (e.g., a designated sequencer address, the attestation contract, or a governance-controlled role) should be permitted to call this function. Alternatively, move the `self.last_reward_block.write(current_block_number)` statement to **after** the `disable_rewards` guard so that a no-op call does not consume the per-block slot.

---

### Proof of Concept

1. Consensus rewards are active (`get_current_epoch() >= consensus_rewards_first_epoch`).
2. Attacker identifies any valid active staker address `S` (e.g., from on-chain events).
3. Each block, attacker submits:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Inside `update_rewards`:
   - `general_prerequisites()` passes (contract not paused, caller non-zero). [6](#0-5) 
   - `current_block_number > last_reward_block` passes (first call this block). [3](#0-2) 
   - Staker validity checks pass. [7](#0-6) 
   - `last_reward_block` is written to current block. [8](#0-7) 
   - `disable_rewards == true` ā†’ early return, zero rewards distributed. [9](#0-8) 
5. Any legitimate call to `update_rewards` in the same block now reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeated every block ā†’ all stakers receive zero consensus rewards indefinitely.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );
