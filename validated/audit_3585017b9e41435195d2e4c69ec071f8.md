### Title
Unbounded Iteration Over Stale Staker Vector in `get_stakers` Causes Unbounded Gas Consumption - (File: `src/staking/staking.cairo`)

### Summary
The `get_stakers` function iterates over a `Vec<ContractAddress>` (`self.stakers`) that grows monotonically and is **never pruned** when stakers exit. As the protocol accumulates historical stakers, every call to `get_stakers` must traverse all past and present stakers, leading to unbounded gas consumption that grows permanently with protocol usage.

### Finding Description
In `src/staking/staking.cairo`, the `stakers` storage field is a `Vec<ContractAddress>` that is appended to on every `stake()` call but never has entries removed: [1](#0-0) 

The comment explicitly acknowledges this: *"Stakers are not removed from this vector when they unstake."*

When a staker calls `unstake_action`, `remove_staker` is invoked, which clears `staker_info` and other mappings, but **never removes the address from `self.stakers`**: [2](#0-1) 

The `get_stakers` function then iterates over the **entire** `self.stakers` vector unconditionally using `into_iter_full_range()`: [3](#0-2) 

For each entry, it calls `is_staker_active`, `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, and `get_peer_id_at_epoch` — multiple storage reads per staker address. The `is_staker_active` check for exited stakers hits the `_ => false` branch (since their `staker_info` is `VInternalStakerInfo::None`), but the storage read still occurs: [4](#0-3) 

### Impact Explanation
`get_stakers` is a consensus-critical view function called by the network to determine the validator set for a given epoch. As the number of historical stakers grows unboundedly, the gas cost of this call grows proportionally. On Starknet, view functions executed as `call` do not consume user gas but do consume sequencer/node resources. If `get_stakers` is called on-chain (e.g., by another contract or as part of a transaction), it will eventually exceed the block gas limit, permanently breaking the ability to retrieve the validator set. Even off-chain, the call becomes prohibitively expensive for nodes.

**Impact:** Unbounded gas consumption (Medium per allowed scope).

### Likelihood Explanation
This is a certainty over time: every new staker permanently increases the iteration cost. The protocol is designed for long-term operation with many participants. No privileged action is required — any user calling `stake()` contributes to the growth. The vector already contains all stakers since genesis and will never shrink.

### Recommendation
1. Remove staker addresses from `self.stakers` upon `unstake_action` (e.g., swap-and-pop pattern), or
2. Replace `Vec` with a structure that supports deletion, or
3. Add pagination parameters to `get_stakers` so callers can iterate in bounded chunks.

### Proof of Concept
1. N stakers call `stake()` over the protocol's lifetime. Each appends to `self.stakers`: [5](#0-4) 
2. Each of those stakers eventually calls `unstake_intent` + `unstake_action`. `remove_staker` is called but does **not** touch `self.stakers`: [6](#0-5) 
3. A call to `get_stakers(epoch_id)` now iterates all N addresses, performing at minimum one storage read per address (`staker_info` map lookup in `is_staker_active`): [7](#0-6) 
4. With N large enough, the transaction exceeds the Starknet block gas limit, making `get_stakers` permanently uncallable on-chain and extremely expensive off-chain.

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }

        fn is_staker_active(
            self: @ContractState, staker_address: ContractAddress, epoch_id: Epoch,
        ) -> bool {
            match self.staker_info.read(staker_address) {
                VInternalStakerInfo::V1(staker_info_v1) => {
                    // `intent_epoch` is zero if the intent exists from before V3.
                    staker_info_v1.unstake_time.is_none()
                        || epoch_id < self.staker_unstake_intent_epoch.read(staker_address)
                },
                _ => false,
            }
        }
