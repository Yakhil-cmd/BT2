### Title
Secret Key Share Directly Recoverable from `PresignOutput` via `beta / c` — (`src/ecdsa/robust_ecdsa/presign.rs`)

### Summary

`PresignOutput` in the robust ECDSA path stores `beta_me = c_me * x_me` and `c_me` as separate `pub` fields. Any caller who receives a `PresignOutput` can recover the participant's secret key share `x_me` by computing `beta * c^{-1}` — a single field division with no cryptographic hardness.

### Finding Description

In `do_presign` (`src/ecdsa/robust_ecdsa/presign.rs`), the final steps compute:

```
c_me   = w^{-1} * a_me          // Step 3.13
beta_me = c_me * x_me            // Step 3.15
```

and return both values in the output struct: [1](#0-0) 

The struct exposes every field as `pub`: [2](#0-1) 

Because `beta = c * x_me` holds exactly (no blinding, no additive noise, no commitment), the relationship is invertible:

```
x_me = beta * c^{-1}   (mod curve order)
```

`c` is guaranteed non-zero by the `invert().unwrap()` on the line above, so the division always succeeds.

### Impact Explanation

**Critical — Extraction of private signing shares.**

Any party that legitimately receives a `PresignOutput` (e.g., a signing coordinator, a participant who stores outputs, or any code path that serialises/deserialises the struct) can recover the full secret key share of the issuing participant with a single scalar inversion and multiplication. Collecting outputs from `t+1` participants allows full reconstruction of the aggregate secret key.

### Likelihood Explanation

The `PresignOutput` struct is `pub`, `Serialize`/`Deserialize`, and all fields are `pub`. It is the direct return value of the public `presign()` API. No special privilege is required beyond receiving the output of a completed presign round — which is the normal, intended usage.

### Recommendation

`beta` must not be stored alongside `c` in a form that allows division. Options:

1. **Remove `c` from `PresignOutput`** — keep only the values needed for the signing phase (`alpha`, `beta`, `e`, `big_r`). `c` is an intermediate value; if the signing step does not need it independently, it should not be persisted.
2. **Blind `beta` additively** — store `beta + r` for a random scalar `r` kept separately, so the stored value is not a scalar multiple of `c`.
3. **Make sensitive fields private** — at minimum, remove `pub` from `c`, `beta`, `alpha`, and `e` so they cannot be read outside the crate, and audit all internal uses.
