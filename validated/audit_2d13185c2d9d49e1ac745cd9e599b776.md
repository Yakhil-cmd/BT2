### Title
Staker Can Suddenly Increase Commission to 100% via Commitment Mechanism, Stealing Delegator Yield — (`File: src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

`enter_delegation_pool` and `add_to_delegation_pool` accept no `max_commission` parameter. A staker holding an active `CommissionCommitment` can atomically raise commission to `max_commission` (up to 100%) at any time, causing all existing and newly entering delegators to earn zero yield with no recourse until the exit wait window expires.

---

### Finding Description

The `update_commission` internal function in `src/staking/staking.cairo` enforces three distinct branches:

- **No commitment present**: commission can only decrease (`assert!(commission < old_commission)`).
- **Active commitment**: commission may be set to **any value ≤ `max_commission`**, including a large increase.
- **Expired commitment**: commission can only decrease.

The protocol itself documents this gap at lines 745–746:

> **Note**: Current commission increase safeguards still allow for sudden commission changes.

The attack path is:

1. Staker starts with `commission = 200` (2%) to attract delegators.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`. The only constraint is `current_commission <= max_commission`, which is satisfied.
3. Staker immediately calls `set_commission(commission: 10000)`, raising commission to 100%.
4. All delegators in the pool now earn **0% of rewards**; the staker captures 100%.
5. Delegators must wait the full `exit_wait_window` (default 1 week, up to 12 weeks) before recovering their principal.

Neither `enter_delegation_pool` nor `add_to_delegation_pool` in `src/pool/pool.cairo` accept a `max_commission` parameter, so delegators have no on-chain slippage guard analogous to the `minimumOutputTokens` recommended in the external report.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

During the exit wait window, delegators earn zero rewards while the staker captures 100% of the epoch rewards that would otherwise have been split. For a pool with significant delegated stake and a 1-week exit window, this represents a direct, quantifiable theft of yield from every delegator in the pool. The staker profits proportionally.

---

### Likelihood Explanation

**Medium-High.** The attack requires only two sequential transactions by the staker (no external dependencies, no oracle, no bridge). The `set_commission_commitment` + `set_commission` sequence is fully permissionless for any staker. Starknet's sequencer ordering means a staker can execute both calls in the same block before any delegator can react. The protocol's own inline comment acknowledges the gap, indicating it is a known, unmitigated risk.

---

### Recommendation

1. Add a `max_commission: Commission` parameter to `enter_delegation_pool` and `add_to_delegation_pool` in `src/pool/pool.cairo`, and revert if the pool's current commission exceeds it:
   ```rust
   assert!(
       self.contract_parameters_v1().commission <= max_commission,
       "Commission too high"
   );
   ```
2. Consider enforcing a minimum notice period (e.g., one epoch) before a commission increase takes effect, so delegators can exit before the new rate applies.

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);

            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);

            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);

            self
                .pool_member_info

            self
                .emit(

        fn add_to_delegation_pool(
            ref self: ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Amount {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            assert!(
                caller_address == pool_member || caller_address == pool_member_info.reward_address,
                "{}",
                Error::CALLER_CANNOT_ADD_TO_POOL,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);

            // Transfer funds from the delegator to the staking contract.
            let token_dispatcher = self.token_dispatcher.read();
            let staker_address = self.staker_address.read();
            transfer_from_delegator(pool_member: caller_address, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            // Update the pool member's balance checkpoint.
            let old_delegated_stake = self.increase_member_balance(:pool_member, :amount);
            let new_delegated_stake = old_delegated_stake + amount;

            // Emit events.
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );

            new_delegated_stake
        }
