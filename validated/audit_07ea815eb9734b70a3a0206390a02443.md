### Title
Missing Upper-Bound Validation for `l2_gas.max_amount` in Declare Transactions Allows Policy-Bypassing Admission — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces an upper bound (`config.max_l2_gas_amount`) on `l2_gas.max_amount` for Invoke and DeployAccount transactions, but explicitly skips this check for Declare transactions. A Declare transaction with an arbitrarily large `l2_gas.max_amount` passes stateless gateway admission when an equivalent Invoke or DeployAccount would be rejected.

### Finding Description

In `validate_resource_bounds`, the upper-bound check is guarded by a type branch that silently no-ops for Declare:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← no check at all
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

The production default for `max_l2_gas_amount` is `1_210_000_000` and `min_gas_price` is `8_000_000_000`. [2](#0-1) 

The test suite explicitly confirms this asymmetry: `test_invalid_max_l2_gas_amount` only covers `TransactionType::DeployAccount` and `TransactionType::Invoke`, and a separate test `valid_l2_gas_amount_on_declare` asserts that a Declare with `max_l2_gas_amount + 1` is **accepted**: [3](#0-2) 

The downstream stateful path (`verify_can_pay_committed_bounds`) only rejects if `max_possible_fee > account_balance`. For a Declare with `l2_gas.max_amount = 1_210_000_001` and `l2_gas.max_price_per_unit = 8_000_000_000`, `max_possible_fee ≈ 9.68 STRK` — a reachable balance for any real account. [4](#0-3) 

`initial_sierra_gas()` is set directly from `l2_gas.max_amount`: [5](#0-4) 

The OS Cairo code for Declare also reads `l2_gas.max_amount` as the initial gas budget before capping at `VALIDATE_MAX_SIERRA_GAS`: [6](#0-5) 

### Impact Explanation

A Declare transaction with `l2_gas.max_amount` exceeding `config.max_l2_gas_amount` (1,210,000,000) is admitted through the stateless gateway and, if the sender holds sufficient balance, through the stateful validator and into the mempool. This violates the gateway's own admission invariant: the same field is bounded for every other transaction type. The execution gas is capped by versioned constants (`validate_max_sierra_gas = 100,000,000`, `execute_max_sierra_gas = 1,110,000,000`), so the transaction executes identically to one within bounds — but the admission policy is bypassed, and the `max_possible_fee` committed by the sender is inflated relative to what the policy intends to allow.

**Impact: High — Mempool/gateway admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

Any unprivileged user can craft a Declare transaction with an oversized `l2_gas.max_amount`. The only prerequisite is holding enough STRK to cover the inflated `max_possible_fee`. With `l2_gas.max_amount = 1_210_000_001` and `l2_gas.max_price_per_unit = 8_000_000_000`, the required balance is ~9.68 STRK — trivially achievable. No special access, no privileged role, no dependency on external state beyond a funded account.

### Recommendation

Remove the Declare-specific no-op branch and apply the same `max_l2_gas_amount` upper-bound check uniformly across all transaction types:

```rust
// Remove the Declare exemption:
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If Declare transactions legitimately require a higher gas budget (e.g., for large class compilation), introduce a separate `max_l2_gas_amount_declare` config field with an explicit, documented upper bound rather than leaving it unbounded.

### Proof of Concept

1. Build a valid `RpcDeclareTransactionV3` with:
   - `l2_gas.max_amount = 1_210_000_001` (one above the configured limit)
   - `l2_gas.max_price_per_unit = 8_000_000_000` (at `min_gas_price`)
   - `l1_gas` and `l1_data_gas` set to zero
   - A valid Sierra contract class, valid `compiled_class_hash`, valid signature
2. Fund the sender account with ≥ 9.68 STRK.
3. Submit via `add_tx` to the HTTP gateway.
4. **Stateless check**: `validate_resource_bounds` hits the `if let RpcTransaction::Declare(_) = tx {}` branch and returns `Ok(())` without checking `max_amount`.
5. **Stateful check**: `verify_can_pay_committed_bounds` computes `max_possible_fee = 1_210_000_001 × 8_000_000_000 ≈ 9.68 STRK`; the funded account passes.
6. The transaction is forwarded to the mempool — admitted with a `l2_gas.max_amount` that would be rejected for any Invoke or DeployAccount transaction.

For comparison, submitting the identical transaction as `RpcInvokeTransactionV3` with the same `l2_gas.max_amount` returns `StatelessTransactionValidatorError::MaxGasAmountTooHigh` at step 4, confirming the asymmetry. [7](#0-6) [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
