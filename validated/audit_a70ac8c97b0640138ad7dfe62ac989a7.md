### Title
Missing EIP-7623 Floor Data Gas Validation in `EvmStatelessChecks` Allows Gas-Free Block Space Consumption — (`app/ante/evm_checktx.go`)

---

### Summary

`EvmStatelessChecks` validates intrinsic gas but not EIP-7623 floor data gas. A transaction where `gasLimit >= intrinsicGas` but `gasLimit < floorDataGas` passes CheckTx, passes `ProcessProposal`, and is included in a block. During `ProcessBlock`, go-ethereum's `StatelessChecks()` (which does enforce EIP-7623) is called in the ante handler — but it fires **before** `BuyGas()`, so when it fails, no gas fee is deducted. The attacker occupies block space at zero cost.

---

### Finding Description

Prague/Pectra is active from genesis on Sei (`PragueTime = 0`): [1](#0-0) 

EIP-7623 defines a floor data gas cost: `21000 + 10 × tokens` (vs. intrinsic gas `21000 + 4 × tokens`). A transaction with 1000 zero-byte calldata has intrinsicGas = 25000 and floorDataGas = 31000. Setting `gasLimit = 27500` satisfies the intrinsic check but violates the floor check.

**`EvmStatelessChecks`** (called in both CheckTx and DeliverTx paths) only checks intrinsic gas: [2](#0-1) 

No floor data gas check is present anywhere in `EvmStatelessChecks`. [3](#0-2) 

During **DeliverTx**, `EvmDeliverChargeFees` calls `EvmCheckAndChargeFees` with `statelessChecks=true`: [4](#0-3) 

Inside `EvmCheckAndChargeFees`, `st.StatelessChecks()` (which enforces EIP-7623) is called **before** `st.BuyGas()`: [5](#0-4) 

When `StatelessChecks()` returns an error, the function returns early — `BuyGas()` is never reached, so **no gas fee is deducted**. The `DecorateNonceCallback` registered earlier still bumps the nonce, but the attacker's balance is untouched. [6](#0-5) 

The codebase itself acknowledges this gap in a test comment: [7](#0-6) 

---

### Impact Explanation

An unprivileged attacker can craft transactions that:
1. Pass CheckTx (no floor data gas check → admitted to mempool)
2. Pass `ProcessProposal` (`checkTotalBlockGas` uses `etx.Gas()`, not floor data gas)
3. Are included in a proposed block
4. Fail `StatelessChecks()` in the ante handler during `ProcessBlock`
5. Consume block space with **zero gas cost**

The nonce is bumped per transaction, but the attacker can use multiple accounts. Block space is consumed without payment, degrading throughput for legitimate users.

The **Giga path** is not affected: gas is pre-charged via `SubBalance` before `Execute()` is called, so even when the floor data gas check fires inside `Execute()`, the fee is already committed. [8](#0-7) 

---

### Likelihood Explanation

- No special privileges required — any account with a valid nonce suffices
- The attacker does not need sufficient balance (since `BuyGas()` is never reached)
- Prague is active from genesis (`PragueTime = 0`), so the condition is always met
- The attack is trivially scriptable: craft a tx with large zero-byte calldata and `gasLimit` between intrinsicGas and floorDataGas

---

### Recommendation

Add a floor data gas check to `EvmStatelessChecks` (and mirror it in `BasicDecorator`), conditioned on Prague being active:

```go
// After the existing intrinsic gas check in EvmStatelessChecks:
if ethCfg.IsPrague(blockNum, blockTime) {
    floorGas := core.FloorDataGas(etx.Data())
    if etx.Gas() < floorGas {
        return core.ErrIntrinsicGas // or a dedicated ErrFloorDataGas
    }
}
```

This mirrors the fix applied in the referenced Monad PR and ensures the check fires before `BuyGas()` — and before the transaction is admitted to the mempool or a block.

---

### Proof of Concept

The existing test `TestGiga_FailedExecution_ProducesReceipt` already demonstrates the gap: [9](#0-8) 

The test confirms `gasLimit=27500` passes `EvmStatelessChecks` (intrinsic=25000) but fails inside `Execute()` with a floor-data-gas error. Adapting this to the V2 path (standard ante handler) would show the same transaction passing CheckTx and `ProcessProposal`, then failing the ante handler during `ProcessBlock` with no gas charged.

func EvmStatelessChecks(ctx sdk.Context, tx sdk.Tx, chainID *big.Int) error {
	if err := evmante.ValidateNoCosmosTxFields(tx); err != nil {
		return err
	}

	if len(tx.GetMsgs()) != 1 {
		return sdkerrors.Wrap(sdkerrors.ErrInvalidRequest, "EVM transaction must have exactly 1 message")
	}
	msg, ok := tx.GetMsgs()[0].(*evmtypes.MsgEVMTransaction)
	if !ok {
		return sdkerrors.Wrap(sdkerrors.ErrInvalidRequest, "not EVM message")
	}
	if err := msg.ValidateBasic(); err != nil {
		return err
	}
	if msg.Derived != nil && msg.Derived.PubKey == nil {
		// this means the message has `Derived` set from the outside, in which case we should reject
		return sdkerrors.ErrInvalidPubKey
	}
	txData, err := evmtypes.UnpackTxData(msg.Data)
	if err != nil {
		return err
	}
	if _, ok := txData.(*ethtx.AssociateTx); ok {
		return nil
	}
	etx, _ := msg.AsTransaction()
	if etx.To() == nil && len(etx.Data()) > params.MaxInitCodeSize {
		return fmt.Errorf("%w: code size %v, limit %v", core.ErrMaxInitCodeSizeExceeded, len(etx.Data()), params.MaxInitCodeSize)
	}

	if etx.Value().Sign() < 0 {
		return sdkerrors.ErrInvalidCoins
	}

	intrGas, err := core.IntrinsicGas(etx.Data(), etx.AccessList(), etx.SetCodeAuthorizations(), etx.To() == nil, true, true, true)
	if err != nil {
		return err
	}
	if etx.Gas() < intrGas {
		return core.ErrIntrinsicGas
	}

	if etx.Type() == ethtypes.BlobTxType {
		return sdkerrors.ErrUnsupportedTxType
	}

	// Check if gas exceed the limit
	if cp := ctx.ConsensusParams(); cp != nil && cp.Block != nil {
		// If there exists a maximum block gas limit, we must ensure that the tx
		// does not exceed it.
		if cp.Block.MaxGas > 0 && etx.Gas() > uint64(cp.Block.MaxGas) { //nolint:gosec
			return sdkerrors.Wrapf(sdkerrors.ErrOutOfGas, "tx gas limit %d exceeds block max gas %d", etx.Gas(), cp.Block.MaxGas)
		}
	}

	if txData.GetGasTipCap().Sign() < 0 {
		return sdkerrors.Wrapf(sdkerrors.ErrInvalidRequest, "gas fee cap cannot be negative")
	}

	// validate chain ID on the transaction
	txChainID := etx.ChainId()
	switch etx.Type() {
	case ethtypes.LegacyTxType:
		// legacy either can have a zero or correct chain ID
		if txChainID.Cmp(big.NewInt(0)) != 0 && txChainID.Cmp(chainID) != 0 {
			logger.Debug("chainID mismatch", "txChainID", txChainID, "chainID", chainID)
			return sdkerrors.ErrInvalidChainID
		}
	default:
		// after legacy, all transactions must have the correct chain ID
		if txChainID.Cmp(chainID) != 0 {
			logger.Debug("chainID mismatch", "txChainID", txChainID, "chainID", chainID)
			return sdkerrors.ErrInvalidChainID
		}
	}

	txGas := txData.GetGas()
	if txGas > math.MaxInt64 {
		return errors.New("tx gas exceeds max")
	}
	return nil

func DecorateNonceCallback(ctx sdk.Context, ek *evmkeeper.Keeper, evmAddr common.Address, txNonce uint64) sdk.Context {
	if ek.EthReplayConfig.Enabled || ek.EthBlockTestConfig.Enabled {
		return ctx
	}
	startingNonce := ek.GetNonce(ctx, evmAddr)
	if startingNonce != txNonce {
		return ctx
	}
	return ctx.WithDeliverTxCallback(func(callCtx sdk.Context) {
		// bump nonce if it is for some reason not incremented (e.g. ante failure)
		if ek.GetNonce(callCtx, evmAddr) == startingNonce {
			ek.SetNonce(callCtx, evmAddr, startingNonce+1)
			ek.SetNonceBumped(callCtx)
		}
	})
