### Title
Unprivileged Caller Can Freeze All Staker Consensus Rewards via Caller-Controlled `disable_rewards` Parameter - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `staking.cairo` is publicly callable by any address and accepts a caller-controlled `disable_rewards` boolean. The function unconditionally writes `current_block_number` to the global `last_reward_block` storage slot **before** checking `disable_rewards`. Because the contract enforces a strict one-call-per-block invariant on `last_reward_block`, any unprivileged caller can consume the per-block reward slot for every block without distributing rewards, permanently freezing all staker consensus-mode yield.

---

### Finding Description

`update_rewards` is the sole reward-distribution path in consensus mode (`is_pre_consensus() == false`). The pre-consensus path (`update_rewards_from_attestation_contract`) explicitly asserts `is_pre_consensus()` and is therefore unavailable once consensus rewards are active.

The vulnerable sequence inside `update_rewards`:

```cairo
// src/staking/staking.cairo  lines 1452-1507
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,                   // one call per block enforced globally
    );
    ...
    // ← last_reward_block is written HERE, before the disable_rewards branch
    self.last_reward_block.write(current_block_number);   // consumes the slot for this block

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits without distributing anything
    }
    ...
    self._update_rewards(...);                            // never reached when disable_rewards=true
}
``` [1](#0-0) 

The global `last_reward_block` field is a single slot shared across all stakers:

```cairo

``` [2](#0-1) 

There is no role check, no ownership check, and no validation that the caller is the staker or any privileged party. `general_prerequisites` only asserts the contract is unpaused and the caller is non-zero:

```cairo

    self.assert_is_unpaused();
    assert_caller_is_not_zero();

``` [3](#0-2) 

The pre-consensus reward path is gated and cannot substitute:

```cairo

    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);

    ...
}

---

### Impact Explanation

In consensus mode, `update_rewards` is the **only** mechanism to distribute block rewards to stakers and their pools. By calling `update_rewards(any_valid_staker, disable_rewards: true)` once per block, an attacker:

1. Marks the block as "already processed" via `last_reward_block`.
2. Distributes zero rewards.
3. Blocks every legitimate call for that block with `REWARDS_ALREADY_UPDATED`.

Repeated across every block, this permanently freezes all unclaimed consensus-mode yield for every staker in the protocol — matching the **High: Permanent freezing of unclaimed yield** impact category.

---

### Likelihood Explanation

- No privileged role is required; any EOA or contract with a non-zero address can call `update_rewards`.
- The attacker only needs to supply any currently-active staker address (readable from public events or `get_stakers`).
- On Starknet, transaction fees are low, making sustained per-block griefing economically feasible.
- The attack is deterministic and requires no special timing beyond being the first caller in a given block.

---

### Recommendation

Two complementary fixes:

1. **Move `last_reward_block` write after the `disable_rewards` guard**, so a call with `disable_rewards: true` does not consume the per-block slot:

```cairo
// Update last block rewards only when rewards are actually distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);  // moved here
self._update_rewards(...);
```

2. **Restrict who may pass `disable_rewards: true`** — for example, only the staker themselves or a designated operator role — so the parameter cannot be weaponised by arbitrary callers.

---

### Proof of Concept

```
// Attacker (any non-zero address) repeats this every block:
staking.update_rewards(
    staker_address = <any active staker>,
    disable_rewards = true
)

// Effect per block N:
// 1. last_reward_block is written to N.
// 2. No rewards are distributed (_update_rewards is never called).
// 3. All subsequent calls for block N revert with REWARDS_ALREADY_UPDATED.
// 4. All stakers and their pools receive 0 STRK for block N.
//
// Repeated indefinitely → all consensus-mode yield is permanently frozen.
```
