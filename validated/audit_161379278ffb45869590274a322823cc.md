### Title
Unchecked `Vec<Commitment>` length in `do_generation_many` causes index-out-of-bounds panic, permanently aborting triple generation for all honest parties — (`src/ecdsa/ot_based_ecdsa/triples/generation.rs`)

---

### Summary

In `do_generation_many`, the round-1 receive loop indexes into a deserialized `Vec<Commitment>` without first verifying its length equals `N`. A malicious participant can send a valid MessagePack-encoded empty (or short) `Vec<Commitment>`, causing an index-out-of-bounds **panic** — not a graceful `ProtocolError` — in every honest party that processes the message. Because `ProtocolExecutor::poke()` does not catch panics, the panic propagates to the caller, permanently aborting triple generation for all honest parties.

---

### Finding Description

In `do_generation_many<const N: usize>` the round-1 receive loop is:

```rust
// generation.rs lines 176-184
while all_commitments_vec
    .iter()
    .any(|all_commitments| !all_commitments.full())
{
    let (from, commitments): (_, Vec<_>) = chan.recv(wait0).await?;
    for i in 0..N {
        all_commitments_vec[i].put(from, commitments[i]); // ← panics if commitments.len() < N
    }
}
``` [1](#0-0) 

`chan.recv(wait0)` deserializes the incoming bytes with `rmp_serde::decode::from_slice` into `Vec<Commitment>`. [2](#0-1) 

MessagePack has no schema enforcement: a valid msgpack array of length 0 deserializes successfully as `Vec<Commitment>` with `.len() == 0`. The `?` operator on line 180 therefore returns `Ok((from, vec![]))`. On the very next iteration of the inner `for i in 0..N` loop, `commitments[0]` is evaluated — an index-out-of-bounds **panic**, not a `Result::Err`.

The executor that drives the protocol, `ProtocolExecutor::poke()`, polls the future with `fut.poll_unpin(&mut cx)` and stores the `Poll::Ready` result — but it does **not** wrap the poll in `std::panic::catch_unwind`. [3](#0-2) 

The panic therefore unwinds through `poke()` to the integrator. The honest party's protocol state is destroyed. Every other honest party is now waiting at `wait0` for a message from the crashed party that will never arrive, permanently stalling triple generation.

---

### Impact Explanation

**High — Permanent denial of triple generation for all honest parties.**

Triple generation is a prerequisite for OT-based ECDSA presigning and signing. Aborting it permanently prevents any subsequent threshold signature from being produced. The attack requires only one malicious participant in the protocol session.

---

### Likelihood Explanation

Any participant in the protocol can inject the malicious message. The `Protocol::message()` entry point accepts arbitrary `MessageData` with no prior authentication or length check. [4](#0-3) 

Constructing a valid msgpack empty array (`0x90` — a single byte) is trivial. The attack is deterministic and requires no cryptographic capability.

---

### Recommendation

Before indexing, validate that the received vector has exactly `N` elements and return a `ProtocolError` on mismatch:

```rust
let (from, commitments): (_, Vec<Commitment>) = chan.recv(wait0).await?;
if commitments.len() != N {
    return Err(ProtocolError::AssertionFailed(format!(
        "commitments from {from:?} has wrong length: expected {N}, got {}",
        commitments.len()
    )));
}
for i in 0..N {
    all_commitments_vec[i].put(from, commitments[i]);
}
```

The same length guard should be applied to every other round that receives a `Vec` and indexes it by `i in 0..N` (e.g., the `PolynomialCommitmentsMessageMany` fields at lines 335–342, the `(a_j_i_v, b_j_i_v)` receive at lines 406–411, and the `big_c_j_v` / `their_phi_proofs` receive at lines 470–475). [5](#0-4) [6](#0-5) [7](#0-6)

### Proof of Concept

```rust
// Pseudocode — inject a 0-length Vec<Commitment> as the round-1 message
// from a malicious participant into an honest party's protocol.

let malicious_payload: Vec<Commitment> = vec![];
// Serialize as msgpack: 0x90 (fixarray of length 0)
let mut raw = header_bytes_for_wait0.to_vec();
rmp_serde::encode::write(&mut raw, &malicious_payload).unwrap();

// Deliver to an honest party via Protocol::message()
honest_protocol.message(malicious_participant, raw);

// Next poke() call panics with:
// "index out of bounds: the len is 0 but the index is 0"
honest_protocol.poke(); // ← panics, not Err
```

The panic propagates through `ProtocolExecutor::poke()` because `fut.poll_unpin` is not wrapped in `catch_unwind`, permanently destroying the honest party's protocol state and stalling all remaining honest parties at `wait0`. [8](#0-7)

```rust
    fn poke(&mut self) -> Result<Action<Self::Output>, ProtocolError> {
        let mut polled_once_already = false;
        loop {
            // If there's outgoing messages, request to send them.
            if let Some(outgoing) = self.comms.outgoing() {
                return Ok(match outgoing {
                    Message::Many(m) => Action::SendMany(m),
                    Message::Private(to, m) => Action::SendPrivate(to, m),
                });
            }
            // If we already have a return result, return it.
            if let Some(result) = self.result.take() {
                return Ok(Action::Return(result?));
            }
            // If this is the second iteration, we already polled the future and there's no
            // progress that can be made.
            if polled_once_already {
                return Ok(Action::Wait);
            }
            // If we don't have a future, this is an extraneous poke() call, so return Wait.
            let Some(fut) = self.fut.as_mut() else {
                return Ok(Action::Wait);
            };
            // Now poll the future. It may generate some more messages to send or a return value,
            // so go back and check all of those again.
            polled_once_already = true;
            let waker = noop_waker();
            let mut cx = Context::from_waker(&waker);
            if let std::task::Poll::Ready(result) = fut.poll_unpin(&mut cx) {
                self.result = Some(result);
                self.fut = None;
            }
        }
    }
