### Title
`set_silo_params(None)` Silently Removes Fallback Address While Address Whitelist Remains Active, Allowing ERC-20 Tokens to Be Minted to Non-Whitelisted Addresses and Permanently Frozen — (`engine/src/engine.rs`)

---

### Summary

`set_silo_params(None)` atomically removes both `fixed_gas` and `erc20_fallback_address` from storage. The Address whitelist is not touched. After this call, `receive_erc20_tokens` skips its only whitelist-aware redirect (because the `if let Some(fallback_address)` guard short-circuits to false), minting tokens directly to any caller-supplied recipient address regardless of whitelist status. If that address is not whitelisted, it cannot submit EVM transactions, permanently freezing the bridged tokens.

---

### Finding Description

`set_silo_params` with `None` calls `set_erc20_fallback_address(io, None)`, which executes `io.remove_storage(&key)`: [1](#0-0) [2](#0-1) 

The sole whitelist enforcement point inside `receive_erc20_tokens` is a Rust let-chain:

```rust
if let Some(fallback_address) = silo::get_erc20_fallback_address(&self.io)
    && !silo::is_allow_receive_erc20_tokens(&self.io, &recipient)
{
    recipient = fallback_address;
}
``` [3](#0-2) 

Both sub-conditions must be true for the redirect to fire. When `get_erc20_fallback_address` returns `None` (after `set_silo_params(None)`), the entire block is skipped — `is_allow_receive_erc20_tokens` is **never evaluated**. The ERC-20 mint proceeds unconditionally to the caller-supplied recipient: [4](#0-3) 

Meanwhile, `is_allow_submit` still enforces the Address whitelist for any subsequent EVM transaction from that address: [5](#0-4) 

The result is a state where tokens exist at an address that is barred from transacting.

The `set_silo_params` entry point is owner-gated: [6](#0-5) 

However, the owner calling `set_silo_params(None)` is a **legitimate configuration operation** (e.g., disabling fixed-gas silo mode while retaining access-control whitelisting). The code provides no warning, no atomic whitelist teardown, and no guard preventing this inconsistent state.

---

### Impact Explanation

After `set_silo_params(None)` with the Address whitelist still enabled:

1. Any `ft_on_transfer` / `receive_erc20_tokens` call with a non-whitelisted recipient mints tokens to that address with no redirect.
2. The recipient address cannot call `submit` or any EVM method — `is_allow_submit` returns `false`.
3. The tokens are stranded. Recovery requires the owner to retroactively discover every affected address and add each to the whitelist — an operationally unreliable process that may never happen for addresses not under anyone's control.

Impact: **Critical — Permanent freezing of funds**.

---

### Likelihood Explanation

The trigger is a single owner call (`set_silo_params(None)`) made while the Address whitelist is active — a plausible operational sequence when an operator wants to disable fixed-gas billing but retain access control. The code offers no safeguard, no documentation warning, and no atomic coupling between silo-param teardown and whitelist teardown. Any bridge transfer arriving after that call and before the operator notices is affected.

---

### Recommendation

Decouple the whitelist check from fallback availability. The redirect guard should be restructured so that when the Address whitelist is enabled, a non-whitelisted recipient is **always** blocked (either redirected or rejected), regardless of whether a fallback address is configured:

```rust
if !silo::is_allow_receive_erc20_tokens(&self.io, &recipient) {
    match silo::get_erc20_fallback_address(&self.io) {
        Some(fallback) => recipient = fallback,
        None => return Err(/* non-whitelisted recipient, no fallback */),
    }
}
```

Alternatively, `set_silo_params(None)` should atomically disable the Address whitelist, or the contract should reject the call when the whitelist is active and no replacement fallback is provided.

pub fn set_erc20_fallback_address<I: IO>(io: &mut I, address: Option<Address>) {
    let key = erc20_fallback_address_key();

    if let Some(address) = address {
        io.write_storage(&key, address.as_bytes());
    } else {
        io.remove_storage(&key);
    }
}
