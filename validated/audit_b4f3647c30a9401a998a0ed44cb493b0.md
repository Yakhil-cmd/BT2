### Title
Staker Can Immediately Raise Commission to 100% via Uncapped `max_commission` in Commitment, Stealing All Delegator Yield — (File: src/staking/staking.cairo)

---

### Summary
The `set_commission_commitment` function places no practical upper bound on `max_commission` below `COMMISSION_DENOMINATOR` (10 000 = 100%). Once a commitment is active, `update_commission` permits the staker to raise commission to any value ≤ `max_commission` with no notice period. A staker can therefore bait delegators with a low commission, set a commitment with `max_commission = 10000`, and immediately raise commission to 100%, capturing the entire pool reward and leaving delegators with zero yield.

---

### Finding Description

`COMMISSION_DENOMINATOR` is defined as 10 000 (representing 100%): [1](#0-0) 

`set_commission_commitment` only rejects values strictly above `COMMISSION_DENOMINATOR`, so `max_commission = 10000` (100%) is accepted: [2](#0-1) 

When a commitment is active, `update_commission` enforces only two conditions — the new commission must not exceed `max_commission` and must differ from the old value. There is no requirement that the commission decrease, and no time-lock before the increase takes effect: [3](#0-2) 

The new commission is written to storage immediately: [4](#0-3) 

Reward splitting uses the live commission value at the moment rewards are calculated: [5](#0-4) 

When `commission = COMMISSION_DENOMINATOR`, `compute_commission_amount_rounded_down` returns the full reward amount, leaving `pool_rewards = 0`: [6](#0-5) 

---

### Impact Explanation
Delegators who joined a pool at a low commission rate (e.g., 1%) receive **zero yield** for any epoch in which the staker has raised commission to 100% via the commitment mechanism. The staker captures the entire pool reward as commission. This constitutes direct theft of unclaimed yield from delegators — matching the **High** impact tier.

---

### Likelihood Explanation
Any registered staker who has opened a delegation pool can execute this attack in two sequential transactions with no special privileges. The only prerequisite is that delegators have already deposited funds. Because the commitment takes effect immediately and the commission change requires no time-lock, delegators have no on-chain mechanism to react before the epoch's rewards are calculated.

---

### Recommendation
Add a protocol-level cap on `max_commission` in `set_commission_commitment` that is meaningfully below `COMMISSION_DENOMINATOR` (e.g., 3 000 = 30%), analogous to adding a maximum fee limit. Additionally, enforce a minimum delay (e.g., K epochs) between setting a commitment and the first permitted commission increase, giving delegators time to exit before the higher rate applies.

---

### Proof of Concept

```
Epoch N:
  1. Staker calls set_commission(commission: 100)   // 1% — delegators join
  2. Delegators call enter_delegation_pool(...)

Epoch N (same or next block):
  3. Staker calls set_commission_commitment(
         max_commission: 10000,          // 100% — no upper-bound check
         expiration_epoch: N + 1
     )
  4. Staker calls set_commission(commission: 10000) // passes: 10000 <= 10000, 10000 != 100

Reward calculation (still Epoch N):
  split_rewards_with_commission(rewards, commission=10000)
    => commission_rewards = rewards * 10000 / 10000 = rewards
    => pool_rewards = 0

  Delegators receive 0 STRK. Staker receives 100% of pool rewards as commission.
``` [7](#0-6) [8](#0-7)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}

pub(crate) fn compute_commission_amount_rounded_down(
    rewards_including_commission: Amount, commission: Commission,
) -> Amount {
    mul_wide_and_div(
        lhs: rewards_including_commission,
        rhs: commission.into(),
        div: COMMISSION_DENOMINATOR.into(),
    )
        .expect_with_err(err: InternalError::COMMISSION_ISNT_AMOUNT_TYPE)
}
