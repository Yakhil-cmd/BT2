### Title
Silo `fixed_gas` yield bypass via `max_fee_per_gas = 0` — (`engine/src/engine.rs`)

---

### Summary

When silo mode is active (`fixed_gas = Some(fg)`), a user can submit a transaction with `max_fee_per_gas = 0`. Because Aurora's `block_base_fee_per_gas` is always zero, `effective_gas_price` resolves to zero, `prepaid_amount` becomes zero, and `refund_unused_gas` early-returns without crediting the relayer. The silo operator collects no yield despite `fixed_gas` being configured.

---

### Finding Description

**`charge_gas` — `engine/src/engine.rs:468–515`**

The early-return guard at lines 476–481 is:

```rust
if transaction.max_fee_per_gas.is_zero()
    && fixed_gas.is_none()
    && block_base_fee_per_gas.is_zero()
{
    return Ok(GasPaymentResult::default());
}
```

When `fixed_gas = Some(fg)`, `fixed_gas.is_none()` is `false`, so the guard does **not** fire. Execution continues, but with `max_fee_per_gas = 0` and `block_base_fee_per_gas = 0`:

```
priority_fee_per_gas = min(0, 0 − 0) = 0
effective_gas_price  = 0 + 0          = 0
prepaid_amount       = fixed_gas * 0  = 0
``` [1](#0-0) 

The sender's balance is reduced by zero. The function returns `GasPaymentResult { prepaid_amount: 0, effective_gas_price: 0, priority_fee_per_gas: 0 }`.

**`refund_unused_gas` — `engine/src/engine.rs:1262–1303`**

```rust
if gas_result.effective_gas_price.is_zero() {
    return Ok(());
}
``` [2](#0-1) 

Because `effective_gas_price = 0`, the function returns immediately. The relayer receives no reward.

**No upstream guard prevents this.** The full pre-`charge_gas` validation path (`engine/src/engine.rs:1049–1106`) only checks:
- Whitelist access (`assert_access`)
- Chain ID
- Nonce
- `fixed_gas > gas_limit` overflow (line 1066)
- Intrinsic gas coverage (line 1079)
- `max_priority_fee_per_gas > max_fee_per_gas` (line 1083 — passes when both are 0) [3](#0-2) 

There is no check that `max_fee_per_gas > 0` when `fixed_gas` is `Some`.

---

### Impact Explanation

The silo operator configures `fixed_gas` specifically to guarantee that every transaction pays `fixed_gas × gas_price` to the relayer as yield. Any whitelisted sender can set `max_fee_per_gas = 0`, pay nothing for gas, and cause the relayer to receive zero reward — for every transaction they submit. This directly violates the silo yield invariant and constitutes **theft of unclaimed yield** (High).

---

### Likelihood Explanation

Any address on the silo whitelist can exploit this. The only prerequisite is that the silo operator has set `fixed_gas` (the normal silo configuration). The attacker simply constructs a legacy or EIP-1559 transaction with `gas_price = 0` / `max_fee_per_gas = 0`. No special privilege, key compromise, or external dependency is required.

---

### Recommendation

In `charge_gas`, add an explicit guard: when `fixed_gas` is `Some` and `effective_gas_price` resolves to zero, return `Err(GasPaymentError::MaxFeePerGasLessThanBaseFee)` (or a new dedicated error). Concretely, after computing `effective_gas_price` at line 493, insert:

```rust
if fixed_gas.is_some() && effective_gas_price.is_zero() {
    return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
}
```

This enforces that silo transactions always carry a non-zero gas price, preserving the yield invariant.

fn test_silo_fixed_gas_zero_price_bypass() {
    let origin = Address::zero();
    let current_account_id = AccountId::default();
    let env = Fixed::default(); // block_base_fee_per_gas = 0
    let storage = RefCell::new(Storage::default());
    let mut io = StoragePointer(&storage);
    // Give sender a non-zero balance
    add_balance(&mut io, &origin, Wei::new_u64(1_000_000_000)).unwrap();
    let mut engine: Engine<_, _> =
        Engine::new_with_state(EngineState::default(), origin, current_account_id, io, &env);

    let transaction = NormalizedEthTransaction {
        address: Address::default(),
        chain_id: None,
        nonce: U256::default(),
        gas_limit: 2_000_000.into(),
        max_priority_fee_per_gas: U256::zero(), // attacker sets 0
        max_fee_per_gas: U256::zero(),           // attacker sets 0
        to: None,
        value: Wei::default(),
        data: vec![],
        access_list: vec![],
        authorization_list: vec![],
    };

    let fixed_gas = Some(EthGas::new(1_000_000)); // silo operator configured this

    let result = engine
        .charge_gas(&origin, &transaction, None, fixed_gas)
        .unwrap();

    // Both assertions pass — sender paid nothing, relayer will get nothing
    assert_eq!(result.prepaid_amount, Wei::zero());
    assert_eq!(result.effective_gas_price, U256::zero());

    let relayer = Address::zero();
    let refund_result = refund_unused_gas(&mut io, &origin, 1_000_000, &result, &relayer, fixed_gas);
    assert!(refund_result.is_ok());
    // relayer balance remains zero — no yield collected
}
