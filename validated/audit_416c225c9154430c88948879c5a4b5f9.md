### Title
Missing Input Length Guard in `BankContract.RequiredGas` Causes Panic / Chain Halt — (File: x/cronos/keeper/precompiles/bank.go)

---

### Summary

`BankContract.RequiredGas` unconditionally slices `input[:4]` without first verifying `len(input) >= 4`. Every other precompile in the same package (`RelayerContract`, `IcaContract`) performs this guard. An unprivileged EVM caller can send a transaction to the bank precompile address (`0x0000…0064`) with fewer than 4 bytes of calldata, triggering a Go index-out-of-range panic that crashes the validator process and halts the chain.

---

### Finding Description

`BankContract.RequiredGas` reads the 4-byte method selector without a length guard:

```go
// x/cronos/keeper/precompiles/bank.go  lines 80-90
func (bc *BankContract) RequiredGas(input []byte) uint64 {
    baseCost := uint64(len(input)) * bc.kvGasConfig.WriteCostPerByte
    var methodID [4]byte
    copy(methodID[:], input[:4])   // ← panics when len(input) < 4
    ...
}
``` [1](#0-0) 

Both sibling precompiles guard against this exact case before touching `input[:4]`:

`RelayerContract.RequiredGas`:
```go
if inputLen < 4 {
    bc.logger.Error("invalid input length", "input", input)
    return getRequiredGas(DefaultGasRequired, baseCost, intrinsicGas)
}
``` [2](#0-1) 

`IcaContract.RequiredGas`:
```go
if len(input) < 4 {
    return baseCost
}
``` [3](#0-2) 

`BankContract` has no equivalent guard. [1](#0-0) 

The EVM framework calls `RequiredGas` before `Run` to meter gas. There is no minimum-calldata enforcement at the EVM layer before this call, so any calldata of length 0–3 reaches `RequiredGas` unfiltered. The `Run` function has the same unguarded slice at line 105 (`contract.Input[:4]`), but it is never reached because `RequiredGas` panics first. [4](#0-3) 

---

### Impact Explanation

A Go `runtime error: slice bounds out of range` panic that is not recovered crashes the validator process. Because every honest validator executing the same block will hit the same panic on the same transaction, the chain halts deterministically. This maps directly to the allowed Critical impact: **consensus divergence / chain halt triggered by an unprivileged transaction**.

---

### Likelihood Explanation

The bank precompile is deployed at a fixed, publicly known address (`common.BytesToAddress([]byte{100})` → `0x0000…0064`). [5](#0-4)  Any EVM user can craft a transaction to that address with 0–3 bytes of calldata. No special privilege, key, or prior state is required.

---

### Recommendation

Add the same length guard used by `RelayerContract` and `IcaContract` at the top of `BankContract.RequiredGas`:

```go
func (bc *BankContract) RequiredGas(input []byte) uint64 {
    baseCost := uint64(len(input)) * bc.kvGasConfig.WriteCostPerByte
    if len(input) < 4 {
        return baseCost
    }
    var methodID [4]byte
    copy(methodID[:], input[:4])
    ...
}
```

Apply the same fix to the `Run` function's `contract.Input[:4]` slice for defence-in-depth.

---

### Proof of Concept

1. Deploy or use any EOA on Cronos.
2. Send an EVM transaction to `0x0000000000000000000000000000000000000064` (bank precompile) with calldata `0xAB` (1 byte).
3. The EVM calls `BankContract.RequiredGas([]byte{0xAB})`.
4. `copy(methodID[:], input[:4])` evaluates `input[:4]` where `len(input) == 1` → Go runtime panic.
5. The panic propagates unrecovered through the block-processing stack, crashing every validator node that processes the block, halting the chain.
