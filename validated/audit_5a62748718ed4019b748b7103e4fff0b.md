### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is specified as callable only by the Starknet sequencer, but the implementation contains no access control enforcement. Any unprivileged caller can invoke it with `disable_rewards: true` in every block, consuming the global `last_reward_block` slot without distributing rewards, permanently denying yield to all stakers.

### Finding Description

The spec for `update_rewards` explicitly states:

> **Access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` line 1449 contains no role or caller check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no sequencer/role check
``` [1](#0-0) 

Compare this with `update_rewards_from_attestation_contract`, which correctly enforces its caller restriction with an explicit `assert_caller_is_attestation_contract()` call: [2](#0-1) 

The `last_reward_block` storage variable is **global** — it is shared across all stakers. Once `update_rewards` is called in block N, no further reward distribution can occur in block N for any staker: [3](#0-2) 

When `disable_rewards: true` is passed, the function returns immediately after writing `last_reward_block`, distributing nothing: [4](#0-3) 

The spec confirms this is the intended sequencer-only path for suppressing rewards in a given block, but without access control it is exploitable by anyone.

### Impact Explanation

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in every block:

1. Writes the current block number into `last_reward_block`.
2. Distributes zero rewards.
3. Causes the legitimate sequencer's subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

Stakers and pool members permanently lose all block-level reward accrual for every block the attacker front-runs. Because the attacker only needs to submit one transaction per block (a low, sustained cost on Starknet), this constitutes an unbounded griefing attack with direct financial damage to all protocol participants.

**Allowed impact matched**: *Medium — Griefing with no profit motive but damage to users or protocol; Unbounded gas consumption.*

### Likelihood Explanation

- Entry path is fully unprivileged: any externally-owned account can call `update_rewards`.
- The only prerequisite is supplying any currently-active staker address, which is public on-chain.
- No special knowledge, leaked keys, or bridge compromise is required.
- The attack is cheap to sustain (one tx per block) and requires no capital.

### Recommendation

Add a sequencer-role check at the top of `update_rewards`, mirroring the pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
```

Alternatively, restrict the function to a dedicated `REWARDS_MANAGER_ROLE` that is assigned only to the sequencer address.

### Proof of Concept

1. Staker A stakes and becomes active (K epochs pass so balance is effective).
2. In every new block, attacker calls:
   ```
   staking.update_rewards(staker_A_address, disable_rewards: true)
   ```
3. `last_reward_block` is set to the current block number; no rewards are minted or transferred.
4. The legitimate sequencer attempts `update_rewards(staker_A_address, disable_rewards: false)` in the same block → reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker A (and all other stakers, since `last_reward_block` is global) accrues zero rewards for every block the attacker sustains the call.
6. Pool members' `unclaimed_rewards` never increase; `claim_rewards` on the pool returns zero indefinitely. [5](#0-4) [2](#0-1)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
