### Title
Malformed Message Causes Unrecoverable Protocol Abort in `recv_from_others` — (File: `src/protocol/helpers.rs`)

### Summary

`recv_from_others` propagates deserialization errors directly to callers via `?`, unlike `reliable_broadcast_receive_all` which silently ignores them. A single malicious participant can send a syntactically malformed message at any waitpoint consumed by `recv_from_others` to abort DKG, presign, sign, or CKD for all honest parties. Because there is no retry or skip-and-continue mechanism, the abort is permanent for that session and can be repeated on every restart.

### Finding Description

`recv_from_others` in `src/protocol/helpers.rs` is the primary gather-all helper used across every protocol in the library:

```rust
// src/protocol/helpers.rs
while !seen.full() {
    let (from, msg) = chan.recv(waitpoint).await?;   // <-- propagates error
    if seen.put(from) {
        messages.push((from, msg));
    }
}
```

`chan.recv` calls `Comms::recv`, which deserializes the payload with `rmp_serde`:

```rust
// src/protocol/internal.rs
let decoded: Result<T, Box<dyn error::Error + Send + Sync>> =
    rmp_serde::decode::from_slice(message_data).map_err(std::convert::Into::into);
Ok((from, decoded?))   // <-- returns ProtocolError on bad bytes
```

Any `ProtocolError` returned here is immediately propagated by the `?` in `recv_from_others`, aborting the calling protocol function.

By contrast, `reliable_broadcast_receive_all` explicitly swallows receive errors and continues the loop:

```rust
// src/protocol/echo_broadcast.rs
match chan.recv(wait).await {
    Ok(value) => (from, (sid, vote)) = value,
    _ => continue,   // malformed messages are silently dropped
};
```

The DKG protocol (`src/dkg.rs`) uses `recv_from_others` for at least the hash-commitment exchange (Round 2.9) and the polynomial-commitment exchange (Round 4.1). A malicious participant who sends garbage bytes at either waitpoint causes every honest party's `do_keyshare` to return `ProtocolError::DeserializationError`, aborting the session. The attack is trivially repeatable on every restart.

### Impact Explanation

**High — Permanent denial of key generation, reshare, refresh, presign, sign, and CKD for honest parties.**

A single malicious participant (well within the `MaxFaulty = (N−1)/3` budget) can abort every protocol session indefinitely by sending one malformed message per round. Because `recv_from_others` has no skip-and-continue path, there is no way for honest parties to make progress without removing the malicious participant from the set entirely.

### Likelihood Explanation

**Medium.** Any registered participant can send arbitrary bytes over the authenticated channel. No cryptographic material or privileged access is required. The attacker only needs to know the current waitpoint value, which is deterministic and predictable from the protocol round sequence.

### Recommendation

Mirror the error-handling pattern already used in `reliable_broadcast_receive_all`: skip messages that fail to deserialize rather than propagating the error. Additionally, mark the offending sender as "seen" so the loop does not wait indefinitely for a replacement message from them:

```rust
while !seen.full() {
    match chan.recv::<T>(waitpoint).await {
        Ok((from, msg)) => {
            if seen.put(from) {
                messages.push((from, msg));
            }
        }
        Err(_) => {
            // Optionally log; do not abort the session.
            continue;
        }
    }
}
```

If the protocol requires that every participant contribute a valid message (i.e., a missing contribution is itself a fault), the caller should handle the resulting incomplete map rather than having the transport layer abort the session.

### Proof of Concept

1. Start a DKG session with `N = 4` participants (`MaxFaulty = 1`).
2. Malicious participant `P_m` waits for the hash-commitment waitpoint (Round 2.9 in `do_keyshare`).
3. `P_m` sends a message whose header bytes match the current waitpoint but whose payload is random garbage (e.g., `[0xFF; 64]`).
4. Each honest party's `recv_from_others` call receives the garbage message, `rmp_serde::decode::from_slice` fails, `Comms::recv` returns `ProtocolError::DeserializationError`, and the `?` in `recv_from_others` propagates it.
5. All honest parties' `do_keyshare` futures return an error; the DKG session is aborted.
6. `P_m` repeats step 3 on every restart, permanently preventing key generation.
