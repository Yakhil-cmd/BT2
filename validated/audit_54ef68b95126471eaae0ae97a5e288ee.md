### Title
Unauthenticated Admin HTTP Server Allows Any Network-Reachable Caller to Invoke Privileged Node Commands — (`admin/command_runner.go`)

### Summary
The Flow node admin HTTP server exposes a `POST /admin/run_command` endpoint that executes privileged node management commands with no caller authentication. Any party that can reach the node's admin port (default `0.0.0.0:9002`) can invoke commands including changing consensus sealing parameters, crashing the node, reading internal block/transaction data, and manipulating transaction rate limits — operations that are intended exclusively for authorized node operators.

### Finding Description

`CommandRunner.runAdminServer()` starts two servers:

1. A gRPC server on a **Unix socket** (local-only, safe): [1](#0-0) 

2. An HTTP gateway server bound to the caller-supplied `bindAddress`: [2](#0-1) 

The HTTP server has **no authentication middleware** of any kind. `tlsConfig` is `nil` by default, so the server starts as plain HTTP: [3](#0-2) 

The `runCommand` handler processes any command without inspecting the caller's identity: [4](#0-3) 

The reference deployment templates for **all node roles** (access, collection, consensus, execution, verification) explicitly bind the admin server to all interfaces: [5](#0-4) [6](#0-5) [7](#0-6) [8](#0-7) 

The registered default commands include highly privileged operations: [9](#0-8) 

Node-type-specific commands registered on execution nodes include `stop-at-height` (crash the node) and `set-config` (change live consensus parameters): [10](#0-9) 

The `stop-at-height` command can crash the execution node process: [11](#0-10) 

The `set-config` command can change live consensus parameters such as `consensus-required-approvals-for-sealing` with no authentication: [12](#0-11) 

### Impact Explanation

An unauthenticated attacker who can reach port 9002 on any Flow node can:

- **Consensus nodes**: Set `consensus-required-approvals-for-sealing` to 0, causing the node to seal blocks without waiting for verification approvals; read internal block/seal/result data via `read-blocks`, `read-results`, `read-seals`.
- **Execution nodes**: Crash the node via `stop-at-height` with `crash: true`; read internal transaction data via `get-transactions`; trigger WAL checkpoints at will.
- **Collection nodes**: Manipulate the `ingest-tx-rate-limit` allowlist, selectively blocking or unblocking specific payer addresses from submitting transactions to the mempool.
- **All nodes**: Change global log levels, alter any live config value, read internal protocol state.

The most security-relevant non-DoS impact is the ability to change `consensus-required-approvals-for-sealing` to 0 on a consensus node (bypassing the verification approval requirement for sealing) and the exposure of internal block/transaction data that should not be publicly readable.

### Likelihood Explanation

The reference deployment templates bind the admin server to `0.0.0.0:9002` for every node role. TLS is optional and disabled by default (`tlsConfig == nil`). No authentication middleware exists in the codebase. Any attacker with network access to port 9002 — including other nodes in the same data center, cloud co-tenants, or any peer that can route to the node — can exploit this without credentials.

### Recommendation

1. **Bind to localhost by default**: Change the default `bindAddress` to `127.0.0.1:9002` instead of `0.0.0.0:9002`.
2. **Require mTLS**: Make `WithTLS` with `tls.RequireAndVerifyClientCert` mandatory, not optional. The test suite already demonstrates this pattern: [13](#0-12) 
3. **Add an IP allowlist middleware** wrapping the HTTP mux before any command is dispatched.
4. **Add caller identity logging** at minimum so unauthorized access attempts are auditable.

### Proof of Concept

With a node running the reference deployment configuration (`--admin-addr=0.0.0.0:9002`, no TLS):

```bash
# Read internal sealed block data (information disclosure)
curl -s http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "read-blocks", "data": {"block": "sealed"}}'

# Change consensus sealing threshold to 0 on a consensus node
curl -s http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config", "data": {"consensus-required-approvals-for-sealing": 0}}'

# Crash an execution node
curl -s http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "stop-at-height", "data": {"height": 1, "crash": true}}'

# Manipulate collection node transaction rate limits
curl -s http://<node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "ingest-tx-rate-limit", "data": {"command": "add", "addresses": "a08d349e8037d6e5"}}'
```

All requests succeed with HTTP 200 and the command output, with no authentication required.

func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
	r.logger.Info().Str("command", command).Msg("received new command")

	req := &CommandRequest{Data: data}

	if validator := r.getValidator(command); validator != nil {
		if validationErr := validator(req); validationErr != nil {
			// for expected validation errors, return code InvalidArgument and the error text
			if IsInvalidAdminParameterError(validationErr) {
				return nil, status.Error(codes.InvalidArgument, validationErr.Error())
			}
			// for unexpected errors, return code Internal and log a warning
			r.logger.Err(validationErr).Msg("unexpected error validating admin request")
			return nil, status.Error(codes.Internal, validationErr.Error())
		}
	}

	var handleResult any
	var handleErr error

	if handler := r.getHandler(command); handler != nil {
		if handleResult, handleErr = handler(ctx, req); handleErr != nil {
			if errors.Is(handleErr, context.Canceled) {
				return nil, status.Error(codes.Canceled, "client canceled")
			} else if errors.Is(handleErr, context.DeadlineExceeded) {
				return nil, status.Error(codes.DeadlineExceeded, "request timed out")
			} else {
				r.logger.Err(handleErr).Msg("unexpected error handling admin request")
				s, _ := status.FromError(handleErr)
				return nil, s.Err()
			}
		}
	} else {
		return nil, status.Error(codes.Unimplemented, "invalid command")
	}

	return handleResult, nil
}

func (s *StopAtHeightCommand) Handler(_ context.Context, req *admin.CommandRequest) (any, error) {
	sah := req.ValidatorData.(StopAtHeightReq)

	oldParams := s.stopControl.GetStopParameters()
	newParams := stop.StopParameters{
		StopBeforeHeight: sah.height,
		ShouldCrash:      sah.crash,
	}

	err := s.stopControl.SetStopParameters(newParams)

	if err != nil {
		return nil, err
	}

	log.Info().
		Interface("newParams", newParams).
		Interface("oldParams", oldParams).
		Msgf("admintool: New En stop parameters set")

	return "ok", nil
