### Title
Wrong Multi-NFT Royalty Amount Calculation in WalletConnect Offer Confirmation Display - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary

`formatAmountWithRoyalties` in `parseCommandDisplay.ts` incorrectly divides the fungible amount by the number of NFTs before computing each royalty, causing the "Total Amount with Royalties" shown in the WalletConnect signing confirmation dialog to be systematically understated when an offer involves multiple NFTs. A user approves the offer seeing a lower total cost than what the protocol will actually deduct.

### Finding Description

In `formatAmountWithRoyalties`, when multiple NFTs carry royalties, the code first splits the base amount equally across all NFTs and then applies each royalty percentage to the split amount:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← wrong division
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

On the Chia protocol, each NFT's royalty is applied to the **full** fungible amount being paid, not a fractional share of it. The division by `royaltyPercentages.length` has no protocol basis and causes the displayed royalty total to be understated by a factor of `N` (the number of NFTs), where the error grows with both NFT count and royalty percentage.

**Concrete example** (from the existing test):
- Amount: 100,000,000 mojos (0.0001 XCH)
- Two NFTs: royalty percentages 500 (5%) and 10 (0.1%)
- **Displayed** `amountWithRoyalties`: 0.00010255 XCH (what the test asserts and the dialog shows)
- **Correct** total: `100,000,000 + (100,000,000 × 500/10,000) + (100,000,000 × 10/10,000)` = 105,100,000 mojos = **0.0001051 XCH** [2](#0-1) 

The result of `walletDeltaToDisplay` (which calls `withRoyaltyTotals` → `formatAmountWithRoyalties`) is surfaced directly in the WalletConnect confirmation dialog as the `amountWithRoyalties` field on each spending line: [3](#0-2) 

This dialog is the sole approval gate for `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` commands arriving via WalletConnect: [4](#0-3) 

The understated total is rendered to the user in `Confirm.tsx`: [5](#0-4) 

### Impact Explanation

A WalletConnect-connected dApp presents a `chia_wallet.take_offer` command for an offer that includes multiple NFTs with royalties. The user sees a "Total Amount with Royalties" that is lower than what the Chia protocol will actually deduct from their wallet. The user approves the transaction based on the understated figure. The actual on-chain spend is higher than what was shown in the confirmation dialog, causing direct, unrecoverable XCH/CAT loss relative to the user's informed consent.

The discrepancy scales with the number of NFTs and their royalty percentages. With 5 NFTs each carrying a 10% royalty and a 1 XCH base price, the dialog shows 1.1 XCH total but the protocol deducts 1.5 XCH — a 36% understatement.

### Likelihood Explanation

Any WalletConnect dApp (unprivileged, no key access required) can craft a `chia_wallet.take_offer` call for a legitimate multi-NFT offer. The user must only approve the dialog. The bug is triggered automatically whenever `royaltyPercentages.length > 1`, which is a normal condition for bundle NFT offers.

### Recommendation

Remove the `splitAmount` division. Each royalty percentage should be applied to the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [6](#0-5) 

### Proof of Concept

1. A dApp connects via WalletConnect and issues `chia_wallet.take_offer` with an offer string encoding:
   - `requested: { xch: "100000000" }` (0.0001 XCH)
   - `offered: { <NFT1>: "1", <NFT2>: "1" }` where NFT1 has 5% royalty and NFT2 has 0.1% royalty
2. `parseCommandDisplay` calls `offerSummaryRoyaltyPercentages` → `royaltyPercentagesForSide` → `formatAmountWithRoyalties` with `royaltyPercentages = [500, 10]`.
3. `splitAmount = 100_000_000n / 2n = 50_000_000n`; displayed total = 102,550,000 mojos = **0.00010255 XCH**.
4. Correct total = 105,100,000 mojos = **0.0001051 XCH**.
5. User sees 0.00010255 XCH in the "Total Amount with Royalties" field, approves, and the protocol deducts 0.0001051 XCH — 2.5% more than shown. The gap grows proportionally with NFT count and royalty rates. [7](#0-6) [8](#0-7)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

async function walletDeltaToDisplay(
  walletDelta: WalletDelta,
  walletInfos: Record<string, WalletInfo>,
  assetKinds: AssetDisplayKinds,
  royaltyPercentages: AssetRoyaltyPercentages,
  fee?: bigint,
): Promise<DisplayWalletDelta> {
  const { spending, receiving } = walletDelta;
  const spendingItems = await Promise.all(
    Object.entries(spending).map(async ([key, value]) => ({
      key,
      line: await parseWalletDeltaItem(
        key,
        value,
        walletInfos,
        assetKinds.spending[key],
        royaltyPercentages.spending[key],
      ),
    })),
  );
  const receivingItems = await Promise.all(
    Object.entries(receiving).map(async ([key, value]) => ({
      key,
      line: await parseWalletDeltaItem(
        key,
        value,
        walletInfos,
        assetKinds.receiving[key],
        royaltyPercentages.receiving[key],
      ),
    })),
  );
  const spendingLines = spendingItems.map(({ line }) => line);
  const receivingLines = receivingItems.map(({ line }) => line);

  return {
    spending: withRoyaltyTotals(spendingItems, spending, receivingLines),
    receiving: withRoyaltyTotals(receivingItems, receiving, spendingLines),
    fee: fee !== undefined ? mojoToChiaLocaleString(fee) : undefined,
  };
}
