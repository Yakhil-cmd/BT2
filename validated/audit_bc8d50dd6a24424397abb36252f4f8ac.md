### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the Staking contract is documented to be callable only by the Starkware sequencer, but the implementation contains no caller validation. Any unprivileged address can invoke it with `disable_rewards: true`, which marks the current block as processed (`last_reward_block` is updated) while skipping reward distribution entirely. Because the per-block guard prevents a second call in the same block, the rewards for that block are permanently unrecoverable.

### Finding Description
`IStakingRewardsManager::update_rewards` is the consensus-phase reward distribution entry point. The specification explicitly states its access control is **"Only starkware sequencer"**. However, the implementation at `src/staking/staking.cairo` lines 1449–1507 contains only a `general_prerequisites()` call (which checks the pause flag) and a `REWARDS_ALREADY_UPDATED` block-number guard. There is no `assert_caller_is_sequencer` or equivalent check.

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ← no caller check here
    self.last_reward_block.write(current_block_number);   // block is now "consumed"

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // rewards silently skipped
    }
    ...
}
```

When `disable_rewards = true` the function writes `last_reward_block = current_block_number` and returns immediately. Any subsequent call in the same block (including the legitimate sequencer call) reverts with `REWARDS_ALREADY_UPDATED`. The block's reward budget is permanently discarded.

### Impact Explanation
Each block that is "consumed" with `disable_rewards: true` loses its entire reward allocation for all stakers and pool members. The staker's `unclaimed_rewards_own` is never incremented, and pool contracts never receive their STRK transfer for that block. Because `last_reward_block` is a monotonically advancing cursor, there is no catch-up mechanism; the lost rewards cannot be retroactively credited. This constitutes **permanent freezing of unclaimed yield** for every affected block.

### Likelihood Explanation
On Starknet the sequencer orders transactions, so a racing attacker cannot reliably front-run the sequencer's own `update_rewards` call in every block. However:
- Any block in which the sequencer omits the call (e.g., during downtime, a bug, or a deliberate skip) is vulnerable.
- A single malicious transaction submitted before the sequencer's call in any such block permanently destroys that block's rewards.
- The attack is cheap (one transaction), requires no capital, and leaves no on-chain trace beyond the missing reward event.

### Recommendation
Add an explicit sequencer-only guard at the top of `update_rewards`, analogous to the `assert_caller_is_attestation_contract` pattern already used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

Alternatively, separate the "mark block as processed" write from the "distribute rewards" logic so that only the sequencer can advance `last_reward_block`.

### Proof of Concept
The existing test suite already demonstrates the missing guard without any caller spoofing:

```cairo
// src/staking/tests/test.cairo (test_update_rewards_without_distribute)
staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: true);
// ↑ called with no cheat_caller_address — any address succeeds
```

A concrete attack sequence:
1. Attacker monitors the mempool for blocks where the sequencer has not yet submitted `update_rewards`.
2. Attacker submits `update_rewards(staker_address, disable_rewards: true)`.
3. `last_reward_block` is set to the current block; the sequencer's subsequent call reverts with `REWARDS_ALREADY_UPDATED`.
4. All stakers and pool members lose that block's reward allocation permanently. [1](#0-0) [2](#0-1) [3](#0-2)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
