### Title
Missing Signer-vs-Address Authorization in `MsgRegisterEncryptionKey` Allows Arbitrary Overwrite of Any Account's Encryption Key — (`x/e2ee/keeper/keeper.go`, `x/e2ee/types/msg.go`)

### Summary

`Keeper.RegisterEncryptionKey` stores an attacker-supplied key under an arbitrary victim address with no check that the transaction signer matches `req.Address`. Any unprivileged account can permanently overwrite any other account's registered encryption key.

### Finding Description

`MsgRegisterEncryptionKey` carries two independent fields: `Address` (the target account) and `Key` (the age X25519 public key to register).

`ValidateBasic` only checks that `Address` is a valid bech32 string and `Key` is a parseable age X25519 recipient: [1](#0-0) 

There is no check that the signer of the transaction equals `m.Address`.

`Keeper.RegisterEncryptionKey` then blindly writes the supplied key under the supplied address's KV prefix: [2](#0-1) 

The internal `registerEncryptionKey` helper only validates that the address is decodable, then calls `KVStore.Set` unconditionally: [3](#0-2) 

At no point in the message handler, `ValidateBasic`, or the keeper is the transaction signer extracted from context and compared to `req.Address`. The `MsgRegisterEncryptionKey` struct does not even implement `GetSigners()` in a way that enforces the address field — the `Address` field is a free parameter independent of who signed the tx. [4](#0-3) 

### Impact Explanation

An attacker submits `MsgRegisterEncryptionKey{Address: victim_address, Key: attacker_age_pubkey}`. The tx passes `ValidateBasic` (valid bech32, valid age key), is included in a block, and `Keeper.RegisterEncryptionKey` overwrites the victim's KV entry with the attacker's public key. From that point:

- Any sender who queries the chain for the victim's encryption key and encrypts a message to it will encrypt to the attacker's key, not the victim's.
- The victim cannot decrypt those messages.
- The attacker, holding the corresponding private key, can decrypt all messages intended for the victim.
- The victim can re-register their own key, but the attacker can immediately overwrite it again in the next block — a persistent griefing/interception loop with no on-chain defense.

This is **corruption of e2ee key state with direct security impact**, matching the allowed High impact category.

### Likelihood Explanation

The attack requires only a funded account and a single transaction. No special privileges, admin access, or leaked keys are needed. The exploit is trivially reproducible.

### Recommendation

In `Keeper.RegisterEncryptionKey`, extract the signer from the SDK context and assert it equals `req.Address` before writing to the store:

```go
func (k Keeper) RegisterEncryptionKey(
    ctx context.Context,
    req *types.MsgRegisterEncryptionKey,
) (*types.MsgRegisterEncryptionKeyResponse, error) {
    sdkCtx := sdk.UnwrapSDKContext(ctx)
    signers := sdkCtx.TxSigner() // or retrieve via msg server signer field
    // assert signers[0] == req.Address
    ...
}
```

Alternatively, add a `GetSigners()` implementation on `MsgRegisterEncryptionKey` that returns `[]sdk.AccAddress{sdk.MustAccAddressFromBech32(m.Address)}` so the Cosmos SDK ante-handler enforces the invariant automatically, and add an explicit check in the keeper as defense-in-depth.

### Proof of Concept

```go
// Two accounts: A (attacker), B (victim)
// B registers their legitimate key
msgB := &types.MsgRegisterEncryptionKey{Address: addrB, Key: agePubKeyB}
keeper.RegisterEncryptionKey(ctx, msgB)

// A overwrites B's key with attacker's key — signed by A, targeting B
msgAttack := &types.MsgRegisterEncryptionKey{Address: addrB, Key: agePubKeyAttacker}
keeper.RegisterEncryptionKey(ctx, msgAttack) // no signer check, succeeds

// Assert: KVStore entry for B now contains agePubKeyAttacker
resp, _ := keeper.Key(ctx, &types.KeyRequest{Address: addrB})
assert(resp.Key == agePubKeyAttacker) // B's original key is gone

func (k Keeper) registerEncryptionKey(
	ctx context.Context,
	address string,
	key []byte,
) error {
	bz, err := k.addressCodec.StringToBytes(address)
	if err != nil {
		return err
	}
	sdk.UnwrapSDKContext(ctx).KVStore(k.storeKey).Set(types.KeyPrefix(bz), key)
	return nil
}
