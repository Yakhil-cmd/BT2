### Title
Attacker Can Grief Staker's Operational Address Change by Pre-Registering the Target Operational Address - (File: src/staking/staking.cairo)

### Summary
The two-step `declare_operational_address` → `change_operational_address` flow has a window in which any unprivileged attacker can stake with the victim staker's intended new operational address, permanently blocking the operational address rotation.

### Finding Description
Changing a staker's operational address requires two separate transactions:

1. The new operational address calls `declare_operational_address(staker_address)`, which writes `eligible_operational_addresses[new_op_addr] = staker_address`.
2. The staker calls `change_operational_address(new_op_addr)`.

`change_operational_address` enforces two independent checks before completing the rotation:

```cairo
// src/staking/staking.cairo lines 670-687
assert!(
    self.operational_address_to_staker_address.read(operational_address).is_zero(),
    "{}",
    Error::OPERATIONAL_EXISTS,
);
// ...
assert!(
    self.eligible_operational_addresses.read(operational_address) == staker_address,
    "{}",
    Error::OPERATIONAL_NOT_ELIGIBLE,
);
``` [1](#0-0) 

The `stake` function, callable by any unprivileged address, also registers an `operational_address` in `operational_address_to_staker_address` with no prerequisite `declare_operational_address` call:

```cairo
// src/staking/staking.cairo lines 298-302
assert!(
    self.operational_address_to_staker_address.read(operational_address).is_zero(),
    "{}",
    Error::OPERATIONAL_EXISTS,
);
// ...
self.operational_address_to_staker_address.write(operational_address, staker_address);
``` [2](#0-1) 

**Attack path:**

1. Victim staker coordinates with `new_op_addr`; `new_op_addr` calls `declare_operational_address(victim_staker)`. The `OperationalAddressDeclared` event is emitted on-chain and is publicly visible.
2. Attacker observes the event, immediately calls `stake(reward_address=X, operational_address=new_op_addr, amount=min_stake)` using their own staker address. This writes `operational_address_to_staker_address[new_op_addr] = attacker_staker`.
3. Victim staker calls `change_operational_address(new_op_addr)`. The first assert at line 671 finds `operational_address_to_staker_address[new_op_addr]` non-zero and reverts with `OPERATIONAL_EXISTS`. [3](#0-2) 

The attacker can repeat this for every new operational address the victim attempts to use, since each `declare_operational_address` call is observable on-chain before the staker acts. After the exit wait window the attacker can unstake and recycle the same funds for the next iteration. [4](#0-3) 

### Impact Explanation
A staker who needs to rotate their operational key (e.g., due to key compromise or hardware replacement) can be permanently prevented from doing so. The staker's current operational address remains in use, but if that key is compromised the staker cannot migrate to a safe key. This constitutes griefing with no profit motive but concrete damage to the staker's security posture and protocol integrity. Matches the allowed Medium impact: *Griefing with no profit motive but damage to users or protocol*.

### Likelihood Explanation
The `OperationalAddressDeclared` event is emitted on-chain and is trivially observable. The window between step 1 (`declare_operational_address`) and step 2 (`change_operational_address`) is unbounded — it can span many blocks. The attacker's only cost is the minimum stake amount temporarily locked, which is recoverable after the exit wait window. Any address with sufficient STRK can execute this attack. [5](#0-4) 

### Recommendation
Require that the `operational_address` supplied to `stake` has not been declared as eligible for a different staker, **or** require a prior `declare_operational_address` call in `stake` (binding the operational address to the staker before registration). Alternatively, allow `change_operational_address` to succeed even when `operational_address_to_staker_address[new_op_addr]` is non-zero, provided the existing registrant is the attacker's own staker and the `eligible_operational_addresses` mapping correctly points to the caller — i.e., give the `eligible_operational_addresses` declaration priority over the `operational_address_to_staker_address` mapping.

### Proof of Concept
```
1. Staker A (staker_address) and new_op_addr coordinate off-chain.
2. new_op_addr calls declare_operational_address(staker_A).
   → eligible_operational_addresses[new_op_addr] = staker_A
   → OperationalAddressDeclared event emitted on-chain.
3. Attacker observes the event and calls:
     stake(reward_address=attacker_reward, operational_address=new_op_addr, amount=min_stake)
   → operational_address_to_staker_address[new_op_addr] = attacker_staker
4. staker_A calls change_operational_address(new_op_addr).
   → assert!(operational_address_to_staker_address[new_op_addr].is_zero()) FAILS
   → Reverts with OPERATIONAL_EXISTS.
5. Attacker calls unstake_intent(), waits exit_wait_window, calls unstake_action().
   → Funds recovered; attacker repeats for any new operational address staker_A tries.
```

        fn change_operational_address(
            ref self: ContractState, operational_address: ContractAddress,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                self.operational_address_to_staker_address.read(operational_address).is_zero(),
                "{}",
                Error::OPERATIONAL_EXISTS,
            );
            assert!(
                !self.does_token_exist(token_address: operational_address),
                "{}",
                Error::OPERATIONAL_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            assert!(
                self.eligible_operational_addresses.read(operational_address) == staker_address,
                "{}",
                Error::OPERATIONAL_NOT_ELIGIBLE,
            );

            // Set operational address and write to storage.
            let old_address = staker_info.operational_address;
            self.operational_address_to_staker_address.write(old_address, Zero::zero());
            staker_info.operational_address = operational_address;
            self.write_staker_info(:staker_address, :staker_info);
            self.operational_address_to_staker_address.write(operational_address, staker_address);

            // Emit event.
            self
                .emit(
                    Events::OperationalAddressChanged {
                        staker_address, new_address: operational_address, old_address,
                    },
                );
        }

        fn declare_operational_address(ref self: ContractState, staker_address: ContractAddress) {
            self.general_prerequisites();
            let operational_address = get_caller_address();
            assert!(
                self.operational_address_to_staker_address.read(operational_address).is_zero(),
                "{}",
                Error::OPERATIONAL_IN_USE,
            );
            assert!(
                !self.does_token_exist(token_address: operational_address),
                "{}",
                Error::OPERATIONAL_IS_TOKEN,
            );
            if self.eligible_operational_addresses.read(operational_address) == staker_address {
                return;
            }
            self.eligible_operational_addresses.write(operational_address, staker_address);
            self.emit(Events::OperationalAddressDeclared { operational_address, staker_address });
        }
