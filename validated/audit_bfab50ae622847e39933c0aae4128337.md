### Title
Off-by-One in Identity-Point Flag Placement Causes G2 Add (0x0D) to Revert on Any Input Containing the Identity Point — (`engine-sdk/src/bls12_381/contract.rs`)

### Summary
`g2_add` in contract mode misplaces the NEAR host infinity flag by one byte when encoding the identity (all-zero) G2 point. The NEAR host `bls12381_p2_sum` receives a buffer with the flag byte clear and the first coordinate byte set to `0x40`, which is an out-of-range field element. The host returns a non-zero error code, causing the precompile to revert with `ERR_BLS12_ELEMENT_NOT_IN_G2` for every call that includes the identity point — a valid EIP-2537 input.

### Finding Description

`g2_add` builds a 386-byte buffer (`8 * FP_LENGTH + 2`) for `bls12381_p2_sum`. The NEAR host format is two consecutive 193-byte records, each structured as `[1-byte flag | 4*FP_LENGTH bytes coordinates]`. The flag byte for point 0 is at index 0; for point 1 at index 193.

For non-identity points the code correctly writes coordinates to bytes 1..193 and 194..386, leaving the flag bytes at 0 and 193 as `0x00`. [1](#0-0) 

For identity points the code writes:
- `g2_input[1] |= 0x40` — sets byte **1** (first coordinate byte of point 0), not byte **0** (flag byte)
- `g2_input[2 + 4 * FP_LENGTH] |= 0x40` = `g2_input[194]` — sets byte **194** (first coordinate byte of point 1), not byte **193** (flag byte) [2](#0-1) 

The NEAR host therefore sees:
- Flag byte = `0x00` → interprets the point as non-infinity
- First coordinate byte = `0x40` = 64 decimal, which exceeds the leading byte of the BLS12-381 field modulus (`0x1a` = 26), making it an invalid field element

The host returns a non-zero error code. [3](#0-2) 

This propagates as `Err(Bls12381Error::ElementNotInG2)` and surfaces to the EVM as `ERR_BLS12_ELEMENT_NOT_IN_G2`. [4](#0-3) 

**Confirming cross-reference:** `g2_msm` correctly sets the flag at `g2_input[offset]` (byte 0 of each block), and `pairing_check` correctly sets it at `g_input[data_offset + 2 * FP_LENGTH]` (byte 0 of the G2 sub-block). Only `g1_add` and `g2_add` carry this off-by-one. [5](#0-4) 

The same off-by-one exists in `g1_add` at `g1_input[1]` and `g1_input[2 + 2 * FP_LENGTH]`. [6](#0-5) 

### Impact Explanation
Any EVM contract that calls precompile `0x0D` (G2 Add) with the identity point as either operand will always revert. This is a standard pattern in BLS12-381 accumulation loops (e.g., initializing an aggregate signature to the identity and adding each partial signature). A contract that holds funds and whose only release path passes through such a G2 Add call is permanently bricked. The revert is deterministic and unconditional — no attacker action is required beyond submitting a valid EIP-2537 G2Add call with an identity operand.

### Likelihood Explanation
The trigger is a single EVM call to precompile `0x0D` with 256 zero bytes as either the first or second 256-byte operand. Any deployed contract implementing BLS12-381 signature aggregation, threshold schemes, or zero-knowledge verifiers that use the identity as an accumulator seed is affected. The call requires no privilege, no special gas, and no contract deployment — it is reachable by any EOA or contract on Aurora.

### Recommendation
Change the identity flag writes in `g2_add` from:
```rust
g2_input[1] |= 0x40;          // wrong: byte 1
g2_input[2 + 4 * FP_LENGTH] |= 0x40;  // wrong: byte 194
```
to:
```rust
g2_input[0] |= 0x40;          // correct: flag byte for point 0
g2_input[1 + 4 * FP_LENGTH] |= 0x40;  // correct: byte 193, flag byte for point 1
```
Apply the same fix to `g1_add` (`g2_input[0]` and `g2_input[1 + 2 * FP_LENGTH]`). Add contract-mode integration tests that call `g1_add`/`g2_add` with the identity as each operand and assert the output equals the other operand.

### Proof of Concept
In a NEAR sandbox with the `contract` feature enabled, call:

```rust
// 512 bytes: first 256 = valid G2 generator, last 256 = all zeros (identity)
let mut input = [0u8; 512];
input[..256].copy_from_slice(&G2_GENERATOR_EIP2537_ENCODING);
// input[256..] is already all zeros = identity

let result = bls12_381::g2_add(&input);
// Expected (EIP-2537): Ok(G2_GENERATOR_EIP2537_ENCODING)
// Actual (contract mode): Err(Bls12381Error::ElementNotInG2)
//   because g2_input[194] = 0x40 is passed to bls12381_p2_sum
//   with flag byte 193 = 0x00, making the host see an invalid field element
```

The standalone mode (blst) returns the correct result for the same input, confirming the bug is isolated to the NEAR host encoding path in `contract.rs`. [7](#0-6)
