### Title
`skip_stateful_validations` Bypasses `__validate__` Signature Check for Any Account with a Pending Mempool Transaction — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function is intended to skip the `__validate__` entry-point call only when a `deploy_account` transaction is pending in the mempool (UX improvement for simultaneous deploy+invoke). However, the predicate it uses — `account_tx_in_pool_or_recent_block` — returns `true` for **any** transaction from the sender, not exclusively a `deploy_account`. An attacker can therefore submit a forged Invoke V3 with `nonce=1` from any account whose state nonce is `0` and that already has **any** pending transaction in the mempool, and the gateway will admit it to the mempool without ever calling `__validate__` (i.e., without verifying the signature).

### Finding Description

**Trigger path** in `add_tx_inner` (`crates/apollo_gateway/src/gateway.rs`):

1. Stateless validation passes (format, size, resource bounds — no signature content check).
2. `convert_rpc_tx_to_internal_and_executable_txs` succeeds.
3. `extract_state_nonce_and_run_validations` is called:
   - `get_nonce_from_state` returns `Nonce(0)` for the victim account.
   - `run_pre_validation_checks` is called:
     - `validate_state_preconditions` → `validate_nonce` passes because `0 ≤ 1 ≤ max_allowed_nonce_gap`.
     - `validate_by_mempool` → mempool's `validate_tx` checks only duplicate hash and fee escalation — **no signature check**.
     - `skip_stateful_validations` is called:

### Impact Explanation

Every transaction admitted to the mempool is supposed to have passed the account's `__validate__` entry point, which is the sole on-chain mechanism for signature/authorization verification. Bypassing it at the gateway means:

- **Invalid (unsigned) transactions are admitted to the mempool** — matching the "High: Mempool/gateway/RPC admission accepts invalid transactions before sequencing" impact category.
- The forged transaction will be rejected by the blockifier at execution time (because `__validate__` is called there with `validate=true`), but it has already consumed mempool capacity and sequencer scheduling resources.
- An attacker can continuously inject forged `nonce=1` invokes for every account that has a pending `nonce=0` transaction, causing sustained mempool pollution and potential DoS of the sequencer's admission pipeline.

### Likelihood Explanation

The preconditions are easily met in normal operation:

1. Any deployed account with state nonce `0` that has submitted its first transaction (nonce=0) to the mempool satisfies the trigger.
2. All transaction data (sender address, nonce, mempool contents) is observable on-chain or via the RPC.
3. The attacker only needs to craft a syntactically valid Invoke V3 with `nonce=1`, any calldata, and any (even empty) signature — no cryptographic material is required.

### Recommendation

Replace the overly broad `account_tx_in_pool_or_recent_block` predicate with a check that is specific to `deploy_account` transactions. The mempool should expose a dedicated method such as `deploy_account_in_pool(address) -> bool` that returns `true` only when a `DeployAccount` transaction for the given address is present. The `skip_stateful_validations` function should call this narrower predicate instead of the current one.

```rust
// Proposed fix sketch
async fn skip_stateful_validations(...) -> ... {
    if let ExecutableTransaction::Invoke(...) = tx {
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            // Only skip if a deploy_account is specifically pending.
            return mempool_client
                .deploy_account_in_pool(tx.sender_address())  // new, narrower predicate
                .await ...;
        }
    }
    Ok(false)
}
```

### Proof of Concept

**Setup**

- Account `A` is deployed; its on-chain nonce is `0`.
- Account `A` submits a legitimate Invoke V3 with `nonce=0`; it passes `__validate__` and is accepted into the mempool.

**Attack**

1. Attacker constructs an Invoke V3 with `sender_address=A`, `nonce=1`, arbitrary `calldata`, and a random/empty `signature`.
2. Attacker submits it to the gateway.
3. Stateless validation passes (valid format, resource bounds, sizes).
4. `get_nonce_from_state(A)` → `Nonce(0)`.
5. `validate_nonce`: `0 ≤ 1 ≤ max_allowed_nonce_gap` → passes.
6. `validate_by_mempool`: no duplicate hash, no fee escalation issue → passes.
7. `skip_stateful_validations`: `nonce==1 && account_nonce==0 && account_tx_in_pool_or_recent_block(A)==true` → returns `true`.
8. `run_validate_entry_point(skip_validate=true)`: `execution_flags.validate=false`; `perform_validations` returns early; `__validate__` is **never called**.
9. `mempool_client.add_tx(...)` → forged transaction is now in the mempool.

**Result**: An unsigned, unauthorized Invoke transaction from account `A` is admitted to the mempool, violating the invariant that every mempool transaction has passed the account's authorization check. [6](#0-5) [1](#0-0) [2](#0-1)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
