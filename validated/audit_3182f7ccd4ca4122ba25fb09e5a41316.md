### Title
Stale KV-Store Permissions Survive Admin Clearance, Allowing Unauthorized Token-Mapping Remapping — (`x/cronos/keeper/permissions.go`, `x/cronos/keeper/msg_server.go`)

### Summary

When governance clears `CronosAdmin` (sets it to `""`), all previously-granted per-address permissions stored in the KV store remain active and irrevocable. Because `UpdatePermissions` requires `admin == msg.From` and an empty admin can never match any sender, there is no on-chain path to revoke those stale permissions. Any address that was granted `CanChangeTokenMapping` before the admin was cleared retains the ability to call `UpdateTokenMapping` indefinitely.

### Finding Description

**Step 1 — Admin grants permissions.**

`msgServer.UpdatePermissions` writes permissions directly to the KV store with no expiry or linkage to the current admin value: [1](#0-0) [2](#0-1) 

**Step 2 — Governance clears the admin.**

A governance proposal executes `MsgUpdateParams` with `CronosAdmin=""`. `UpdateParams` only calls `k.SetParams`, which updates the params store. It does not touch the permissions KV store. No existing per-address permission entries are cleared or invalidated. [3](#0-2) 

**Step 3 — `UpdatePermissions` is now permanently locked.**

With `admin == ""`, the guard `admin != msg.From` is always true for any non-empty sender, so `UpdatePermissions` always returns `ErrUnauthorized`. There is no alternative path (governance, module account, etc.) to revoke individual KV-store permissions. [4](#0-3) 

**Step 4 — Previously-permissioned address calls `UpdateTokenMapping`.**

`HasPermission` first checks `admin == account.String()` (false, since admin is `""`), then falls through to `GetPermissions`, which reads the stale KV entry and returns `CanChangeTokenMapping`. The check passes and `RegisterOrUpdateTokenMapping` executes. [5](#0-4) [6](#0-5) 

### Impact Explanation

The previously-permissioned address can remap any IBC/gravity denom to an arbitrary contract address. This corrupts the denom→contract binding used by `ConvertVouchers` and the EVM hooks, potentially redirecting token conversion to an attacker-controlled contract and enabling unauthorized asset movement. This falls squarely under **High: Bypass of Cronos admin/governance authority and permission authorization checks** and **High: Corruption of token mappings with direct security impact**.

### Likelihood Explanation

The scenario requires the admin to have previously delegated `CanChangeTokenMapping` to at least one address, and then governance to clear the admin. Both are normal, intended operations. The window of exploitation is permanent — it persists until a new admin is set and manually revokes the stale permissions (which itself requires knowing which addresses were previously permissioned).

### Recommendation

When `UpdateParams` sets `CronosAdmin` to a new value (especially to `""`), the module should either:
1. Iterate and delete all existing permission KV entries, or
2. Store permissions with a reference to the admin address that granted them and invalidate them when the admin changes, or
3. Allow governance (`MsgUpdateParams` authority) to also call a permission-revocation path directly.

Additionally, `UpdatePermissions` should allow the governance authority (not just the admin) to revoke permissions, providing a fallback when the admin is cleared.

### Proof of Concept

```

MsgUpdatePermissions{From: admin, Address: attacker, Permissions: 1}
// → SetPermissions stores attacker → 1 in KV store

// 2. Governance proposal: MsgUpdateParams with CronosAdmin=""
// → SetParams updates params; KV store unchanged

// 3. Verify UpdatePermissions is locked
MsgUpdatePermissions{From: attacker, Address: attacker, Permissions: 0}
// → returns ErrUnauthorized (admin="" != attacker)

// 4. Attacker remaps a gravity denom to a malicious contract
MsgUpdateTokenMapping{Sender: attacker, Denom: "gravity0x...", Contract: "0xMalicious"}
// → HasPermission: admin="" != attacker (skip), GetPermissions(attacker)=1, 1&1==1 → true
// → RegisterOrUpdateTokenMapping executes successfully
// → Token mapping is now corrupted

func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}

func (k msgServer) UpdateParams(goCtx context.Context, msg *types.MsgUpdateParams) (*types.MsgUpdateParamsResponse, error) {
	if msg.Authority != k.authority {
		return nil, errors.Wrapf(govtypes.ErrInvalidSigner, "invalid authority; expected %s, got %s", k.authority, msg.Authority)
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := k.SetParams(ctx, msg.Params); err != nil {
		return nil, err
	}

	return &types.MsgUpdateParamsResponse{}, nil
}

func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
}

func (k Keeper) SetPermissions(ctx sdk.Context, address sdk.AccAddress, permissions uint64) {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := sdk.Uint64ToBigEndian(permissions)
	store.Set(types.AdminToPermissionsKey(address), permissionsBytes)
}

func (k Keeper) GetPermissions(ctx sdk.Context, address sdk.AccAddress) uint64 {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := store.Get(types.AdminToPermissionsKey(address))
	return sdk.BigEndianToUint64(permissionsBytes)
}

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
