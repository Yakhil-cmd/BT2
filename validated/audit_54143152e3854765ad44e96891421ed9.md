### Title
Domain Allowlist Bypass via `endsWith` Without Dot Separator in `fetchOffer` — (`packages/gui/src/util/fetchOffer.ts`)

### Summary

The `fetchOffer` function validates offer URLs against an allowlist of trusted domains using `String.prototype.endsWith`. Because no dot separator is enforced, any domain whose string representation ends with an allowed domain name (e.g., `evildexie.space` ends with `dexie.space`) passes the check. An attacker who controls such a domain can send a blockchain notification containing a URL pointing to their server, causing the GUI to fetch and present attacker-controlled offer data to the victim.

---

### Finding Description

The domain check in `fetchOffer.ts` is:

```typescript
const domain = new URL(offerUrl).hostname;
const service = OfferServices.find(({ domains }) =>
  domains.find((localDomain) => domain.endsWith(localDomain))
);
``` [1](#0-0) 

The allowed domains are `dexie.space`, `mintgarden.io`, `spacescan.io`, and `offerpool.io`. [2](#0-1) 

`endsWith` is a pure string suffix check with no awareness of DNS label boundaries. Therefore:

| Attacker hostname | `endsWith('dexie.space')` |
|---|---|
| `evildexie.space` | **true** |
| `notmintgarden.io` | **true** |
| `notspacescan.io` | **true** |
| `notofferpool.io` | **true** |

All of these pass the guard and proceed to fetch offer content from the attacker's server.

---

### Attack Path

1. **Entry point:** An attacker sends a blockchain notification whose hex-encoded message decodes to `{"v":1,"t":1,"d":{"u":"https://evildexie.space/malicious.offer"}}`.
2. **Parsing:** `useBlockchainNotifications` decodes the message and stores `offerURL = "https://evildexie.space/malicious.offer"` in the notification object with no URL validation at this stage. [3](#0-2) 
3. **Rendering:** `NotificationOffer` calls `useOffer(offerURLOrData)`, which internally calls `fetchOffer`. [4](#0-3) 
4. **Domain check:** `new URL("https://evildexie.space/malicious.offer").hostname` → `"evildexie.space"`. `"evildexie.space".endsWith("dexie.space")` → `true`. The guard passes. [1](#0-0) 
5. **Fetch:** The GUI fetches content from `https://evildexie.space/malicious.offer`, parses it as offer data, calls `getOfferSummary` and `checkOfferValidity` on the local node, and presents the result to the user as a valid incoming offer. [5](#0-4) 

---

### Impact Explanation

The attacker can serve a crafted offer string (a valid Chia offer blob) that, when accepted by the victim, transfers assets to the attacker's wallet. The offer summary is displayed to the user, but a deceptive offer (e.g., offering a tiny amount of a worthless CAT in exchange for XCH) could trick a user into accepting it. This bypasses the intended trust boundary that restricts offer fetching to known, reputable services.

This maps to: **High — Unsafe handling of external URLs that produces direct asset loss or causes a user to approve the wrong asset/amount/destination.**

---

### Likelihood Explanation

- Registering a domain like `evildexie.space` costs ~$10/year and is trivially achievable.
- Sending a blockchain notification to a target address is a standard on-chain operation requiring only a small XCH fee.
- No special privileges, leaked keys, or local access are required.

---

### Recommendation

Replace the bare `endsWith` check with an exact match or a dot-anchored suffix check:

```typescript
// Before (vulnerable):
domain.endsWith(localDomain)

// After (correct):
domain === localDomain || domain.endsWith('.' + localDomain)
```

This ensures `evildexie.space` is rejected while `api.dexie.space` (a legitimate subdomain) is still accepted.

---

### Proof of Concept

1. Register `evildexie.space` and host a valid Chia offer blob at `https://evildexie.space/offer.bin`.
2. Send a blockchain notification to the victim's puzzle hash with message (hex-encoded): `{"v":1,"t":1,"d":{"u":"https://evildexie.space/offer.bin"}}`.
3. Open the Chia GUI. The notification appears and `fetchOffer` is called.
4. Add a `console.log` breakpoint at line 21 of `fetchOffer.ts` to confirm `domain = "evildexie.space"` and `service` is non-null.
5. Observe the attacker-controlled offer is fetched, summarized, and presented to the user for acceptance.
