### Title
`GetExecutionResultByID` Bypasses Finalized-Seal Guard Present in `GetExecutionResultForBlockID` - (File: `engine/access/rpc/backend/backend_execution_results.go`)

### Summary
`GetExecutionResultForBlockID` enforces that an execution result is only returned when a **finalized seal** exists for the queried block. `GetExecutionResultByID`, a sibling endpoint on the same `backendExecutionResults` struct, performs no such check — it returns any execution result stored locally, regardless of whether the corresponding block has ever been sealed or finalized. An unprivileged API caller can use the ID-based endpoint to retrieve execution results for blocks that are still pending, orphaned, or whose results are disputed, defeating the finalized-seal invariant that the block-ID endpoint enforces.

### Finding Description

`GetExecutionResultForBlockID` first calls `b.seals.FinalizedSealForBlock(blockID)` before returning any result. If no finalized seal exists, the call returns `codes.NotFound`. Only after a seal is confirmed does it fetch and return the execution result. [1](#0-0) 

`GetExecutionResultByID` skips this guard entirely. It calls `b.executionResults.ByID(id)` directly against the execution-results storage, which is populated as soon as an execution receipt is received — long before any seal is produced or finalized. [2](#0-1) 

Both endpoints are exposed on the same `backendExecutionResults` struct and registered on the same public gRPC/REST surface:

- gRPC: `GetExecutionResultByID` handler in `engine/access/rpc/handler.go` lines 1099–1121
- REST: `GetExecutionResultByID` route in `engine/access/rest/http/routes/execution_result.go` lines 38–61 [3](#0-2) [4](#0-3) 

**Attacker entry path:**

1. Attacker submits or observes a transaction in a block that has been executed but not yet sealed (or whose execution result is disputed by a competing result from a different execution node).
2. Attacker calls `GetExecutionReceiptsByBlockID` (no seal check) to obtain the execution result ID from a receipt stored on the Access node.
3. Attacker calls `GetExecutionResultByID(resultID)` to retrieve the full execution result — including all chunk data, service events, and state commitments — for a block that `GetExecutionResultForBlockID` would refuse to serve.

The execution-results storage is populated by the Access ingestion engine upon receipt of execution receipts, which arrive well before sealing. The `GetExecutionResultByID` path has no guard against this. [5](#0-4) 

### Impact Explanation

An unprivileged external caller can read execution results — including embedded service events (epoch setup/commit events, protocol state transitions) and state commitments — for blocks that have not been finalized or sealed. This is a **private data exposure** issue: the protocol's intent is that only the sealed, consensus-confirmed execution result is authoritative and publicly served. Serving pre-seal results exposes:

- Disputed or incorrect execution results (when two execution nodes disagree, both results are stored; the ID-based endpoint returns whichever the caller asks for, including the losing result that will never be sealed).
- Service events from blocks that may be orphaned, giving a false picture of protocol state transitions.

There is no loss of on-chain funds, but the inconsistency in access control defeats the finalized-seal invariant that `GetExecutionResultForBlockID` enforces, exactly mirroring the original report's pattern.

### Likelihood Explanation

The endpoint is publicly reachable by any unauthenticated gRPC or REST client. The only prerequisite is knowing a valid execution result ID, which is trivially obtainable via `GetExecutionReceiptsByBlockID` (also unauthenticated, also no seal check). No privileged access, staked node role, or key material is required.

### Recommendation

In `GetExecutionResultByID`, after fetching the result, verify that a finalized seal exists for `result.BlockID` before returning the result:

```go
func (b *backendExecutionResults) GetExecutionResultByID(ctx context.Context, id flow.Identifier) (*flow.ExecutionResult, error) {
    result, err := b.executionResults.ByID(id)
    if err != nil {
        return nil, rpc.ConvertStorageError(err)
    }

    // Enforce the same finalized-seal guard as GetExecutionResultForBlockID
    _, err = b.seals.FinalizedSealForBlock(result.BlockID)
    if err != nil {
        return nil, rpc.ConvertStorageError(err)
    }

    return result, nil
}
```

This aligns the two sibling endpoints and ensures that only sealed, consensus-confirmed execution results are served to external callers.

### Proof of Concept

```
# Step 1: obtain a receipt for a recently executed (but not yet sealed) block
grpcurl -plaintext <access-node>:9000 flow.access.AccessAPI/GetExecutionReceiptsByBlockID \
  -d '{"block_id": "<hex-block-id>"}'
# → returns receipt(s) including execution_result.id

# Step 2: fetch the full execution result directly, bypassing the seal check
grpcurl -plaintext <access-node>:9000 flow.access.AccessAPI/GetExecutionResultByID \
  -d '{"id": "<hex-result-id-from-step-1>"}'
# → returns full ExecutionResult including chunks, service_events, state commitments
# → GetExecutionResultForBlockID for the same block would return codes.NotFound at this point
``` [6](#0-5)

func (b *backendExecutionResults) GetExecutionResultForBlockID(ctx context.Context, blockID flow.Identifier) (*flow.ExecutionResult, error) {
	// Query seal by blockID
	seal, err := b.seals.FinalizedSealForBlock(blockID)
	if err != nil {
		return nil, rpc.ConvertStorageError(err)
	}

	// Query result by seal.ResultID
	result, err := b.executionResults.ByID(seal.ResultID)
	if err != nil {
		return nil, rpc.ConvertStorageError(err)
	}

	return result, nil
}

// GetExecutionResultByID gets execution result by the ID.
func GetExecutionResultByID(r *common.Request, backend access.API, link commonmodels.LinkGenerator) (any, error) {
	req, err := request.GetExecutionResultRequest(r)
	if err != nil {
		return nil, common.NewBadRequestError(err)
	}

	res, err := backend.GetExecutionResultByID(r.Context(), req.ID)
	if err != nil {
		return nil, err
	}

	if res == nil {
		err := fmt.Errorf("execution result with ID: %s not found", req.ID.String())
		return nil, common.NewNotFoundError(err.Error(), err)
	}

	var response commonmodels.ExecutionResult
	err = response.Build(res, link)
	if err != nil {
		return nil, err
	}

	return response, nil
