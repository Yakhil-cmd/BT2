### Title
Missing Access Control on `update_rewards` Allows Any Staker to Steal Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is intended to be callable only by the Starkware sequencer (per the specification), but no on-chain access control enforces this. Any valid staker can call it with their own `staker_address` and `disable_rewards: false`, claiming block rewards that should go to the legitimate block producer, or call it with `disable_rewards: true` to permanently burn the block producer's rewards for that block.

### Finding Description
The specification for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation only calls `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. There is no check that the caller is the sequencer. [1](#0-0) 

The `general_prerequisites` function: [2](#0-1) 

The only per-block guard is `current_block_number > self.last_reward_block.read()`, which prevents two calls in the same block but does not restrict who can make the call: [3](#0-2) 

In V3 consensus rewards mode (`is_pre_consensus()` returns false), the function calculates block rewards and distributes them entirely to the supplied `staker_address`: [4](#0-3) 

The `last_reward_block` is written before the early-return check, so even a call with `disable_rewards: true` consumes the block's reward slot: [5](#0-4) 

### Impact Explanation
**High — Theft of unclaimed yield / Permanent freezing of unclaimed yield.**

- **Theft path**: An attacker who is a registered staker with non-zero balance (effective after K epochs) calls `update_rewards(attacker_staker_address, false)` before the sequencer's call. The attacker's staker receives the full block rewards. The legitimate block producer's rewards for that block are permanently lost.
- **Freeze path**: The attacker calls `update_rewards(any_valid_staker, true)`. `last_reward_block` is updated, the block's reward slot is consumed, and no rewards are distributed. The legitimate block producer permanently loses their yield for that block with no recourse.

### Likelihood Explanation
**Low-Medium.** In Starknet, the sequencer controls transaction ordering and would normally include its own `update_rewards` call as the first transaction in a block. However:
1. The on-chain contract provides **zero enforcement** of this ordering guarantee.
2. If the sequencer ever fails to call `update_rewards` in a given block (bug, downtime, or the designated block producer is not a registered staker), any attacker can call it in the next block before the sequencer recovers.
3. The `disable_rewards: true` griefing path requires no profit motive and can be executed by any staker at any time the sequencer's call has not yet landed.

### Recommendation
Add an explicit caller check in `update_rewards` to restrict it to the sequencer address (stored in contract storage or as a constant), analogous to how `update_rewards_from_attestation_contract` restricts its caller: [6](#0-5) 

A similar pattern should be applied to `update_rewards`:
```cairo
fn update_rewards(...) {
    self.assert_caller_is_sequencer(); // new guard
    self.general_prerequisites();
    ...
}
```

### Proof of Concept

1. Attacker stakes the minimum amount and waits K epochs for their balance to become effective.
2. Consensus rewards are active (`is_pre_consensus()` returns false).
3. In block N, before the sequencer submits its `update_rewards` transaction, the attacker submits:
   ```
   staking.update_rewards(staker_address: attacker_address, disable_rewards: false)
   ```
4. The call passes all checks: contract not paused, caller not zero, `block_number > last_reward_block`, staker active, non-zero balance.
5. `last_reward_block` is set to block N. The attacker's staker receives the full block rewards via `_update_rewards`.
6. The sequencer's subsequent call for the legitimate block producer reverts with `REWARDS_ALREADY_UPDATED`.
7. The legitimate block producer's yield for block N is permanently lost. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
