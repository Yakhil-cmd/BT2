### Title
Missing Minimum Threshold Validation in FROST Signing and Presigning Initialization - (File: src/frost/mod.rs)

### Summary
`assert_sign_inputs` and `frost::presign` in `src/frost/mod.rs` do not enforce a minimum threshold of 2, unlike every other protocol entry point in the codebase. A caller supplying `threshold = 0` or `threshold = 1` passes all guards and launches a signing or presigning protocol that will produce a cryptographically unusable output, corrupting the sign transcript for all honest participants.

### Finding Description

`assert_sign_inputs` (the shared validation helper for all FROST EdDSA and RedJubjub signing entry points) performs the following checks: [1](#0-0) 

It validates:
- `participants.len() >= 2`
- `me` is in the participant list
- `threshold <= participants.len()`
- `coordinator` is in the participant list

It does **not** validate `threshold >= 2`.

`frost::presign` has the same gap: [2](#0-1) 

By contrast, every other protocol entry point in the codebase enforces the minimum:

`assert_key_invariants` (DKG): [3](#0-2) 

`validate_triple_inputs` (OT-based ECDSA triple generation): [4](#0-3) 

All three FROST signing entry points delegate to `assert_sign_inputs` without adding their own minimum-threshold guard:

- `sign_v1` / `sign_v2` in `src/frost/eddsa/sign.rs` [5](#0-4) 

- `sign` in `src/frost/redjubjub/sign.rs` [6](#0-5) 

`ReconstructionLowerBound` is a plain `usize` wrapper with no invariant enforcement: [7](#0-6) 

So `ReconstructionLowerBound::from(0usize)` or `ReconstructionLowerBound::from(1usize)` are accepted without error.

### Impact Explanation

When `threshold = 1` is accepted by `assert_sign_inputs`, the coordinator only waits for a single signature share before aggregating. The Lagrange coefficient is then computed over a singleton signing set, yielding coefficient 1 for that participant. The resulting aggregate scalar `s = r + c·x_i` does not satisfy `s·G = R + c·X` (where `X` is the multi-party public key), so the produced signature is cryptographically invalid. Honest parties that receive and attempt to use this output obtain an unusable signature — a corruption of the sign transcript.

When `threshold = 0` is accepted, the degree-of-polynomial arithmetic inside the signing protocol (Lagrange interpolation over an empty or degenerate set) produces undefined or erroneous scalar values, again yielding an unusable output.

**Matched impact:** High — Corruption of sign outputs so honest parties accept unusable cryptographic outputs.

### Likelihood Explanation

The `assert_sign_inputs` function is a public API surface. Any library caller constructing signing arguments — including a misconfigured integrator or a malicious coordinator who deliberately supplies `threshold = 1` to force a degenerate signing round — can trigger this path. No special privilege is required beyond being the party that calls the signing entry point.

### Recommendation

Add the same minimum-threshold guard that `assert_key_invariants` and `validate_triple_inputs` already enforce:

```rust
// In assert_sign_inputs, after converting threshold:
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Apply the same guard in `frost::presign` immediately after the `threshold > participants.len()` check.

### Proof of Concept

```rust
use threshold_signatures::frost::assert_sign_inputs;
use threshold_signatures::participants::Participant;

let p0 = Participant::from(1u32);
let p1 = Participant::from(2u32);
let participants = vec![p0, p1];

// threshold = 1 — should be rejected, but is accepted
let result = assert_sign_inputs(&participants, 1usize, p0, p1);
assert!(result.is_ok()); // passes today; should return ThresholdTooSmall

// The signing protocol now runs with threshold = 1.
// The coordinator aggregates a single share with Lagrange coefficient 1,
// producing s·G ≠ R + c·X — an invalid, unusable signature.
``` [8](#0-7)

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
