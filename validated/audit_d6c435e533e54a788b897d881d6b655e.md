### Title
Staker Can Instantly Increase Commission to 100% via `set_commission_commitment` + `set_commission`, Stealing Delegators' Yield — (File: `src/staking/staking.cairo`)

---

### Summary

A staker can exploit the `set_commission_commitment` + `set_commission` two-step mechanism to instantly raise their pool commission from any value (including 0%) to 100% in two back-to-back transactions, with no delay or notice period for delegators. All future pool rewards are then redirected to the staker, leaving delegators with zero yield on their locked tokens.

---

### Finding Description

The `set_commission_commitment` function allows a staker to declare a `max_commission` ceiling (up to `COMMISSION_DENOMINATOR = 10000`, i.e. 100%) and an `expiration_epoch` as close as `current_epoch + 1`. Once the commitment is active, `set_commission` permits the staker to set commission to **any value ≤ `max_commission`**, including values higher than the current commission.

The only guard against commission increases is the commitment mechanism itself — without a commitment, `update_commission` enforces `commission < old_commission`. But with an active commitment, the check is relaxed to `commission <= max_commission` (and `commission != old_commission`). [1](#0-0) 

There is no enforced delay between setting the commitment and exercising it. A staker can call both functions in the same block:

1. `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — passes all checks because `current_commission (e.g. 0) <= 10000`, `expiration_epoch > current_epoch`, and `expiration_epoch - current_epoch (= 1) <= epochs_in_year`.
2. `set_commission(10000)` — passes because the commitment is immediately active and `10000 <= max_commission`. [2](#0-1) [3](#0-2) 

The codebase itself acknowledges this gap with the comment:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [4](#0-3) 

After the commitment expires, `update_commission` requires `commission < old_commission` (the `COMMISSION_COMMITMENT_EXPIRED` branch). The staker must decrease commission by at least 1 unit, but can then immediately set a new commitment with `max_commission = 10000` and repeat the cycle indefinitely. [5](#0-4) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Pool members (delegators) who have locked tokens under a staker's pool earn yield each epoch. The commission at the time rewards are distributed (during attestation or `update_rewards`) determines the delegator's share. By raising commission to 100%, the staker captures the entire pool reward, leaving delegators with zero yield on their locked principal. Rewards that have accrued since the last distribution epoch but have not yet been claimed are effectively redirected to the staker. Delegators cannot exit instantly — they must wait through the `exit_wait_window` (default 1 week, up to 12 weeks) after calling `exit_delegation_pool_intent`, during which they continue to earn zero yield. [6](#0-5) 

---

### Likelihood Explanation

**High.** Any staker who has opened a delegation pool can execute this in two transactions at any time. There is no economic cost beyond gas. The attack is repeatable every epoch (set commitment → raise to 100% → wait for expiry → lower by 1 → set new commitment → raise again). Delegators have no on-chain mechanism to react before the commission change takes effect, and the exit window means they cannot immediately recover their principal.

---

### Recommendation

1. **Enforce a delay before a commitment-enabled commission increase takes effect.** Require that the new (higher) commission only applies starting from `current_epoch + K` (the same delay used for balance changes), giving delegators time to exit.
2. **Cap the rate of commission increase per epoch** (e.g., no more than X basis points per epoch).
3. **Emit a `CommissionIncreaseScheduled` event** when a commitment is set with `max_commission > current_commission`, so off-chain monitoring can alert delegators.
4. **Require a minimum notice period** between `set_commission_commitment` and the first `set_commission` call that increases commission (e.g., at least 1 epoch must pass).

---

### Proof of Concept

```
// Epoch N
// Staker has commission = 0%, pool has delegators

// Step 1: Set commitment with max_commission = 100%, expires next epoch
staker.set_commission_commitment(max_commission: 10000, expiration_epoch: N + 1)
// Passes: 0 <= 10000, N+1 > N, 1 <= epochs_in_year

// Step 2: Immediately raise commission to 100% (same block/tx)
staker.set_commission(commission: 10000)
// Passes: active commitment, 10000 <= 10000, 10000 != 0

// Result: all pool rewards from this point forward go to the staker.
// Delegators earn 0% yield while their tokens remain locked for up to exit_wait_window.

// Epoch N+1: commitment expires
// Step 3: Lower commission by 1 to satisfy COMMISSION_COMMITMENT_EXPIRED check
staker.set_commission(commission: 9999)

// Step 4: Set a new commitment and raise back to 100%
staker.set_commission_commitment(max_commission: 10000, expiration_epoch: N + 2)
staker.set_commission(commission: 10000)
// Cycle repeats indefinitely
``` [7](#0-6) [8](#0-7)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
