### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Block Rewards - (`File: src/staking/staking.cairo`)

### Summary

`IStakingRewardsManager::update_rewards` is intended to be callable only by the Starkware sequencer (per the protocol spec), but the implementation contains no access control check. Because the function writes to a single global `last_reward_block` storage slot on every successful call, any unprivileged caller can frontrun the sequencer in any block, permanently preventing legitimate reward distribution for that block.

### Finding Description

The spec explicitly states the access control for `update_rewards`:

> **Access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` performs no caller identity check. The only guard is the block-level deduplication check against `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);   // global lock for this block
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // ... distribute rewards to staker_address
}
``` [2](#0-1) 

`last_reward_block` is a single global `BlockNumber` field — not per-staker: [3](#0-2) 

**Attack path:**

1. At the start of any block, an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)`.
2. This passes all checks (staker exists, has non-zero balance), writes `last_reward_block = current_block`, then returns early without distributing any rewards.
3. When the legitimate sequencer attempts `update_rewards(intended_staker, disable_rewards: false)` in the same block, it hits `REWARDS_ALREADY_UPDATED` and reverts.
4. The intended staker's block reward is permanently lost — there is no mechanism to retroactively distribute missed block rewards.

Alternatively, if the attacker is themselves a staker, they call `update_rewards(attacker_staker, disable_rewards: false)` to collect their own rewards while simultaneously blocking the sequencer from distributing rewards to any other staker in that block.

The `calculate_block_rewards` function caches per-epoch block rewards in `block_rewards` storage, so the reward amount itself is not manipulable — but the choice of *which staker* receives it (or whether anyone does) is fully attacker-controlled: [4](#0-3) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Each block's consensus rewards can only be distributed once (enforced by the global `last_reward_block` lock). A staker who is denied their block reward by this attack has no recourse; the missed reward is gone permanently. An attacker who repeats this every block can continuously deny all stakers except one (or none) from ever receiving consensus rewards, causing unbounded cumulative yield loss across the protocol.

### Likelihood Explanation

**High.** The entry point is a public, permissionless function. No special role, token balance, or privileged access is required. The only cost is a Starknet transaction fee per block. The attack is trivially automatable: a bot monitors the mempool for sequencer `update_rewards` calls and frontrunning them with `disable_rewards: true` targeting any valid staker.

The absence of any access control is confirmed by the test suite, which calls `update_rewards` directly without spoofing any privileged caller address: [5](#0-4) [6](#0-5) 

### Recommendation

Add a caller check at the top of `update_rewards` to enforce the "Only starkware sequencer" access control documented in the spec. The sequencer address should be stored in contract storage (similar to how `attestation_contract` is stored) and validated with an assertion analogous to `assert_caller_is_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_sequencer();   // enforce spec: Only starkware sequencer
    let current_block_number = starknet::get_block_number();
    // ...
}
```

### Proof of Concept

```
1. Deploy the system in consensus-rewards mode (consensus_rewards_first_epoch reached).
2. Staker A stakes and becomes active.
3. Advance to a new block.
4. Attacker (any EOA) calls:
       staking.update_rewards(staker_A_address, disable_rewards: true)
   → Succeeds. last_reward_block = current_block. No rewards distributed.
5. Sequencer calls:
       staking.update_rewards(staker_A_address, disable_rewards: false)
   → Reverts with REWARDS_ALREADY_UPDATED.
6. Staker A's unclaimed_rewards_own is unchanged. The block reward is permanently lost.
7. Repeat step 3–6 every block to permanently deny all consensus rewards.
``` [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
