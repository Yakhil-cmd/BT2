### Title
Unauthenticated `MsgSubmitEvidence` Allows Fabricated `Equivocation` to Tombstone Any Active Validator — (`sei-cosmos/x/evidence/keeper/infraction.go`)

---

### Summary

Any unprivileged account can submit a `MsgSubmitEvidence` transaction containing a fabricated `Equivocation` struct targeting any active, bonded validator. Because `HandleEquivocationEvidence` performs no cryptographic verification of the alleged double-sign, the targeted validator is permanently slashed and tombstoned. Repeating this against validators holding ≥ 1/3 of voting power halts the chain.

---

### Finding Description

**Entrypoint — public `MsgSubmitEvidence` transaction**

`Equivocation` is explicitly registered as a valid evidence type for user submission: [1](#0-0) 

`MsgSubmitEvidence` is a standard, permissionless Cosmos SDK message routed through the evidence keeper's `SubmitEvidence`: [2](#0-1) 

`SubmitEvidence` checks only for an exact-hash duplicate, then dispatches to the registered handler: [3](#0-2) 

**Stateless validation is trivially satisfied**

`ValidateBasic` only checks that `Height ≥ 1`, `Time > 0`, `Power ≥ 1`, and `ConsensusAddress != ""`. No signature or proof-of-double-sign is required: [4](#0-3) 

**`GetConsensusAddress` silently swallows decode errors**

If the bech32 address is malformed, the error is discarded and an empty `ConsAddress` is returned. For a well-formed address pointing to a real validator this is a no-op, but it means no address-level rejection occurs: [5](#0-4) 

**`HandleEquivocationEvidence` performs no cryptographic verification**

The handler checks only liveness conditions (pubkey registered, evidence age, validator bonded, signing info present, not already tombstoned). There is no verification that the validator actually produced two conflicting votes: [6](#0-5) 

If all liveness checks pass, the validator is slashed with the attacker-supplied `Power` value, jailed, and permanently tombstoned: [7](#0-6) 

**BeginBlock path is separate and unaffected**

The BeginBlock path (`app/legacyabci/begin_block.go:62`) receives evidence from Tendermint consensus, which has already cryptographically verified it. The vulnerability is exclusive to the `MsgSubmitEvidence` user-submission path. [8](#0-7) 

---

### Impact Explanation

An attacker who pays only gas fees can:

1. Identify the consensus addresses of active validators (public on-chain data).
2. Submit one `MsgSubmitEvidence` per target with any recent `Height`/`Time` and `Power ≥ 1`.
3. Each submission permanently tombstones the targeted validator.
4. Tombstoning validators holding ≥ 1/3 of total voting power prevents new blocks from being finalized, halting the chain.

Because tombstoning is irreversible without a hard fork, recovery requires coordinated chain governance intervention.

---

### Likelihood Explanation

- No special privilege, key, or stake is required — only a funded account to pay gas.
- Validator consensus addresses are publicly visible.
- The attack is deterministic and requires no timing precision.
- A single attacker can execute it in a handful of transactions.

---

### Recommendation

1. **Remove or disable `MsgSubmitEvidence` for `Equivocation`**. Newer Cosmos SDK versions (≥ v0.47) deprecated this path precisely because equivocation evidence cannot be safely verified at the application layer without the full Tendermint vote data. Equivocation should only be processed via the BeginBlock ABCI path where Tendermint has already verified the cryptographic proof.
2. If user-submitted equivocation evidence must be retained, require the full `DuplicateVoteEvidence` protobuf (including both signed vote objects) and verify both signatures against the validator's registered public key before executing any slashing.
3. As a short-term mitigation, add a governance-gated allow-list or disable the `RouteEquivocation` handler in the `MsgSubmitEvidence` router while keeping it active in the BeginBlock path.

---

### Proof of Concept

```
// Attacker knows validator V has consensus address "seivalcons1..."
// and is currently bonded at block height H with block time T.

msg := MsgSubmitEvidence{
    Submitter: attackerAddress,
    Evidence: Any{
        TypeUrl: "/cosmos.evidence.v1beta1.Equivocation",
        Value: Equivocation{
            Height:           H,          // any recent height
            Time:             T,          // any recent timestamp
            Power:            1,          // any positive value
            ConsensusAddress: "seivalcons1...", // target validator
        },
    },
}

// Submit and sign with any funded account.
// Result: validator V is slashed and permanently tombstoned.
// Repeat for validators holding ≥ 1/3 voting power → chain halt.
```

}

func (k Keeper) SubmitEvidence(ctx sdk.Context, evidence exported.Evidence) error {
	if _, ok := k.GetEvidence(ctx, evidence.Hash()); ok {
		return sdkerrors.Wrap(types.ErrEvidenceExists, evidence.Hash().String())
	}
	if !k.router.HasRoute(evidence.Route()) {
		return sdkerrors.Wrap(types.ErrNoEvidenceHandlerExists, evidence.Route())
	}

	handler := k.router.GetRoute(evidence.Route())
	if err := handler(ctx, evidence); err != nil {
		return sdkerrors.Wrap(types.ErrInvalidEvidence, err.Error())
	}

	ctx.EventManager().EmitEvent(
		sdk.NewEvent(
			types.EventTypeSubmitEvidence,
			sdk.NewAttribute(types.AttributeKeyEvidenceHash, evidence.Hash().String()),
		),
	)

	k.SetEvidence(ctx, evidence)
	return nil
}

func (k Keeper) HandleEquivocationEvidence(ctx sdk.Context, evidence *types.Equivocation) {

	consAddr := evidence.GetConsensusAddress()

	if _, err := k.slashingKeeper.GetPubkey(ctx, consAddr.Bytes()); err != nil {
		// Ignore evidence that cannot be handled.
		//
		// NOTE: We used to panic with:
		// `panic(fmt.Sprintf("Validator consensus-address %v not found", consAddr))`,
		// but this couples the expectations of the app to both Tendermint and
		// the simulator.  Both are expected to provide the full range of
		// allowable but none of the disallowed evidence types.  Instead of
		// getting this coordination right, it is easier to relax the
		// constraints and ignore evidence that cannot be handled.
		return
	}

	// calculate the age of the evidence
	infractionHeight := evidence.GetHeight()
	infractionTime := evidence.GetTime()
	ageDuration := ctx.BlockHeader().Time.Sub(infractionTime)
	ageBlocks := ctx.BlockHeader().Height - infractionHeight

	// Reject evidence if the double-sign is too old. Evidence is considered stale
	// if the difference in time and number of blocks is greater than the allowed
	// parameters defined.
	cp := ctx.ConsensusParams()
	if cp != nil && cp.Evidence != nil {
		if ageDuration > cp.Evidence.MaxAgeDuration && ageBlocks > cp.Evidence.MaxAgeNumBlocks {
			logger.Info(
				"ignored equivocation; evidence too old",
				"validator", consAddr,
				"infraction_height", infractionHeight,
				"max_age_num_blocks", cp.Evidence.MaxAgeNumBlocks,
				"infraction_time", infractionTime,
				"max_age_duration", cp.Evidence.MaxAgeDuration,
			)
			return
		}
	}

	validator := k.stakingKeeper.ValidatorByConsAddr(ctx, consAddr)
	if validator == nil || validator.IsUnbonded() {
		// Defensive: Simulation doesn't take unbonding periods into account, and
		// Tendermint might break this assumption at some point.
		return
	}

	if ok := k.slashingKeeper.HasValidatorSigningInfo(ctx, consAddr); !ok {
		panic(fmt.Sprintf("expected signing info for validator %s but not found", consAddr))
	}

	// ignore if the validator is already tombstoned
	if k.slashingKeeper.IsTombstoned(ctx, consAddr) {
		logger.Info(
			"ignored equivocation; validator already tombstoned",
			"validator", consAddr,
			"infraction_height", infractionHeight,
			"infraction_time", infractionTime,
		)
		return
	}
