### Title
NFT Seller Shown Incorrect "Net Proceeds" Due to Commented-Out Royalty Deduction — (`packages/gui/src/components/offers/utils.ts`)

### Summary

In `calculateNFTRoyalties`, the royalty deduction from `nftSellerNetAmount` is commented out, causing it to equal the full offer amount. This value propagates through `NFTOfferDetails` → `overrideNFTSellerAmount` → `OfferSummaryTokenRow`, causing the "You will receive" and "Net Proceeds" displays to show the full offer price rather than the post-royalty net amount when an NFT seller reviews an imported `TokenForNFT` offer.

### Finding Description

In `packages/gui/src/components/offers/utils.ts`, `calculateNFTRoyalties` computes `nftSellerNetAmount` as simply `amount` — the royalty subtraction is commented out: [1](#0-0) 

```typescript
const royaltyAmount: number = royaltyPercentage ? (royaltyPercentage / 100) * amount : 0;
const royaltyAmountString: string = formatAmount(royaltyAmount);
const nftSellerNetAmount: number = amount;
// : parseFloat(
//     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
//   );
```

In `NFTOfferDetails`, `nftSaleInfo.nftSellerNetAmount` (= full `amount`) is used to compute `overrideNFTSellerAmount`: [2](#0-1) 

This value is passed to `NFTOfferSummary` → `NFTOfferSummaryRow` → `OfferSummaryTokenRow`, where it replaces the original amount: [3](#0-2) 

```typescript
const amount = overrideNFTSellerAmount ?? originalAmount;
```

The "Net Proceeds" section also directly renders `nftSaleInfo?.nftSellerNetAmount`, with a tooltip explicitly stating it is "the asking price, minus any associated creator fees" — but it shows the full amount: [4](#0-3) 

### Impact Explanation

An NFT seller importing a `TokenForNFT` offer for an NFT with a non-zero `royaltyPercentage` will see:

- **"You will receive": 100 XCH** (full offer amount)
- **"Net Proceeds": 100 XCH** (labeled as post-royalty, but equals full amount)
- **"Creator Fee (10%)": 10 XCH** (correctly shown separately)

The actual on-chain settlement pays 10 XCH to the creator, so the seller receives only 90 XCH. The "Net Proceeds" label explicitly promises the post-royalty figure, but delivers the pre-royalty figure. A seller relying on this labeled field accepts an offer expecting 100 XCH and receives 90 XCH — a direct financial loss proportional to the royalty rate.

This fits: *"High: Corruption... of offer... state that causes a user to approve... the wrong... amount."*

### Likelihood Explanation

- Triggered by any imported `TokenForNFT` offer where the NFT has `royaltyPercentage > 0` — an extremely common case for NFTs with creator royalties.
- No attacker action is required beyond crafting a standard offer for a royalty-bearing NFT. The bug is in the viewer's own calculation, not in attacker-supplied data.
- The NFT seller must only import and view the offer — a normal workflow.

### Recommendation

Restore the commented-out deduction in `calculateNFTRoyalties`:

```typescript
const nftSellerNetAmount: number = parseFloat(
  (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
);
```

Verify that `overrideNFTSellerAmount` and the "Net Proceeds" display both reflect this corrected value.

### Proof of Concept

Unit test `calculateNFTRoyalties` with a 10% royalty NFT and a 100 XCH offer:

```typescript
const result = calculateNFTRoyalties(
  100,          // amount
  0,            // makerFee
  10,           // royaltyPercentage (10%)
  NFTOfferExchangeType.TokenForNFT,
);

// Current (buggy) behavior:
assert(result.nftSellerNetAmount === 100); // WRONG

// Expected behavior:
assert(result.nftSellerNetAmount === 90);  // amount - royalty
```

With the bug present, `overrideNFTSellerAmount` in `NFTOfferDetails` becomes `chiaToMojo(100)` instead of `chiaToMojo(90)`, and the "Net Proceeds" row displays `100 XCH` instead of `90 XCH`. [5](#0-4) [6](#0-5) [7](#0-6)

export function calculateNFTRoyalties(
  amount: number,
  makerFee: number,
  royaltyPercentage: number,
  exchangeType: NFTOfferExchangeType,
): CalculateNFTRoyaltiesResult {
  const royaltyAmount: number = royaltyPercentage ? (royaltyPercentage / 100) * amount : 0;
  const royaltyAmountString: string = formatAmount(royaltyAmount);
  const nftSellerNetAmount: number = amount;
  // : parseFloat(
  //     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
  //   );
  const totalAmount: number =
    exchangeType === NFTOfferExchangeType.NFTForToken ? amount + royaltyAmount : amount + makerFee + royaltyAmount;
  const totalAmountString: string = formatAmount(totalAmount);

  return {
    royaltyAmount,
    royaltyAmountString,
    nftSellerNetAmount,
    totalAmount,
    totalAmountString,
  };
}
