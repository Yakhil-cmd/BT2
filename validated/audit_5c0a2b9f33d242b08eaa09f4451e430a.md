### Title
Presignature Nonce Material Reusable via Non-Consuming `rerandomize_presign` API — (File: `src/ecdsa/robust_ecdsa/mod.rs`)

### Summary
`PresignOutput` derives `Clone` and `RerandomizedPresignOutput::rerandomize_presign` accepts `&PresignOutput` (a shared reference, not consuming ownership). This means the same presignature — which encodes a secret nonce `k` — can be rerandomized an arbitrary number of times and used to produce signatures under different messages. Two signatures produced from the same underlying nonce are sufficient to solve for the private key algebraically, even when the rerandomization changes `R` to `delta*R`.

### Finding Description

In `src/ecdsa/robust_ecdsa/mod.rs`, `PresignOutput` is declared as:

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, ZeroizeOnDrop)]
pub struct PresignOutput { ... }
``` [1](#0-0) 

And `rerandomize_presign` is:

```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // shared reference — NOT consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError> { ... }
``` [2](#0-1) 

Because the function takes `&PresignOutput`, the caller can invoke it twice on the same object:

```rust
let rerandomized1 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args1)?;
let rerandomized2 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args2)?;
// Both are now valid signing contexts backed by the same secret nonce k.
```

The library's own comment in `src/ecdsa/robust_ecdsa/presign.rs` acknowledges the danger:

> "it's crucial that a presignature is never reused" [3](#0-2) 

Yet the API provides no enforcement — no ownership transfer, no usage flag, no presignature ID registry.

The `sign` function in `src/ecdsa/robust_ecdsa/sign.rs` also explicitly warns about the split-view variant of this class of attack: [4](#0-3) 

### Impact Explanation

**Critical — private key extraction.**

In the robust ECDSA scheme the presignature encodes a secret nonce `k` (via `c = 1/k`). After rerandomization with scalar `delta`, the effective nonce becomes `k·delta` and the effective public key becomes `x + tweak`. The resulting signature satisfies:

```
s · k · delta = h + (x + tweak) · Rx'
```

where `Rx' = x_coord(delta · R)`.

If the same `PresignOutput` is rerandomized with `delta1` and `delta2` and used to sign messages `h1` and `h2`:

```
s1 · k · delta1 = h1 + (x + tweak1) · r1'   ... (1)
s2 · k · delta2 = h2 + (x + tweak2) · r2'   ... (2)
```

This is a system of two linear equations in the two unknowns `k` and `x`. Solving:

```
k = (h2·r1' − h1·r2') / (s2·delta2·r1' − s1·delta1·r2')
x = (s1·k·delta1 − h1) / r1'
```

All quantities on the right-hand side (`s1`, `s2`, `h1`, `h2`, `delta1`, `delta2`, `r1'`, `r2'`) are public or known to the attacker. The master private key `x` is fully recovered.

This maps directly to the allowed impact: **"Critical: Extraction, reconstruction, or disclosure of private signing shares, aggregate secret material, presign secrets, nonce material, or confidential derived secrets."**

### Likelihood Explanation

**Medium-High.** The attack is reachable through two realistic paths:

1. **Malicious coordinator / orchestration layer**: A coordinator that controls which presignature is handed to each signing session can call `rerandomize_presign` twice on the same `PresignOutput` and run two signing sessions. Participants have no mechanism to detect or refuse this — the library provides no presignature ID or single-use token.

2. **Buggy application layer**: Because `PresignOutput` is `Clone` and `rerandomize_presign` takes a shared reference, an application developer who stores presignatures in a cache or retries a failed signing session will silently reuse the nonce. The API shape actively invites this mistake.

No cryptographic break, no leaked keys, and no trusted-host compromise are required. The attacker only needs to observe two valid signatures produced from the same presignature.

### Recommendation

1. **Consume `PresignOutput` on rerandomization** — change the signature to `fn rerandomize_presign(presignature: PresignOutput, ...) -> Result<Self, ProtocolError>`. Rust's ownership system then statically prevents double-use.

2. **Remove `Clone` from `PresignOutput`** — or at minimum document that cloning it is a critical security violation and gate it behind an `unsafe` or explicitly named method.

3. **Add a presignature-ID mechanism** — assign each `PresignOutput` a unique random ID at creation time and require callers to register IDs before use, rejecting duplicates.

4. **Mirror the fix in the OT-based ECDSA path** — `src/ecdsa/ot_based_ecdsa/presign.rs` has the same structural pattern and should be audited for the same issue. [5](#0-4) 

### Proof of Concept

```rust
// After a normal presign run, a malicious coordinator holds presign_out: PresignOutput.

// Rerandomize TWICE with different arguments — library does not prevent this.
let args1 = RerandomizationArguments { big_r: presign_out.big_r, tweak: tweak1 };
let args2 = RerandomizationArguments { big_r: presign_out.big_r, tweak: tweak2 };

let rerand1 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args1).unwrap();
let rerand2 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args2).unwrap();

// Run two signing sessions — participants comply because they have no reuse-detection.
let sig1 = run_sign_session(rerand1, h1, ...); // (R1, s1)
let sig2 = run_sign_session(rerand2, h2, ...); // (R2, s2)

// Recover private key x from (s1, delta1, r1', h1) and (s2, delta2, r2', h2):
// k = (h2*r1' - h1*r2') / (s2*delta2*r1' - s1*delta1*r2')
// x = (s1*k*delta1 - h1) / r1'
```

The `rerandomize_presign` call on line 55 of `src/ecdsa/robust_ecdsa/mod.rs` is the necessary vulnerable step: it accepts `&PresignOutput` and returns a fresh `RerandomizedPresignOutput` without invalidating or consuming the source, making the duplication invisible to the type system and to participants. impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta * R
        let rerandomized_big_r = presignature.big_r * delta;

        // alpha * delta^{-1}
        let rerandomized_alpha = presignature.alpha * inv_delta;

        // (beta + c*tweak) * delta^{-1}
        let rerandomized_beta =
            (presignature.beta + presignature.c * args.tweak.value()) * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            alpha: rerandomized_alpha,
            beta: rerandomized_beta,
            e: presignature.e,
        })
    }
```
