### Title
Delegator Rewards Permanently Frozen in Pool Contract When Staker Exits - (`src/pool/pool.cairo`)

---

### Summary

Rewards transferred to the `Pool` contract via `send_rewards_to_delegation_pool` can become permanently unclaimable by delegators when the staker calls `unstake_intent`. The pool's `claim_rewards` function returns 0 for delegators whose balance was not yet active (due to the 2-epoch activation delay) at the time rewards were calculated, leaving the STRK tokens permanently stuck in the pool contract with no withdrawal mechanism.

---

### Finding Description

The staking contract calculates pool rewards using `get_staker_delegated_balance_at_epoch`, which reflects the delegated balance immediately upon delegation. However, the pool's `calculate_rewards` function uses the pool member's balance trace (`pool_member_epoch_balance`), which only becomes active 2 epochs after delegation. This creates a structural discrepancy:

**Step-by-step root cause:**

1. Delegator delegates in epoch `E`. The staking contract records `delegated_balance > 0` immediately.
2. Staker attests in epoch `E+K` where `K < 2`. `_update_rewards` in `staking.cairo` calls `calculate_staker_pools_rewards`, which reads `get_staker_delegated_balance_at_epoch` — this returns the non-zero delegated balance.
3. Pool rewards are calculated as non-zero, `claim_from_reward_supplier` pulls tokens from the reward supplier, and `send_rewards_to_delegation_pool` transfers them to the pool contract. The pool's `cumulative_rewards_trace` is updated.
4. Staker calls `unstake_intent`. From this point, `update_rewards_from_attestation_contract` reverts with `UNSTAKE_IN_PROGRESS` — no further reward updates are possible.
5. Delegator calls `claim_rewards`. The pool's `calculate_rewards` function iterates over the pool member's balance trace. Because the delegator's balance was not yet active (2-epoch delay), the trace shows balance = 0 for the relevant epochs. The function returns 0.
6. Delegator exits. Staker exits. The STRK tokens sent in step 3 remain in the pool contract permanently — there is no `sweep`, `rescue`, or governance withdrawal function.

The test at `src/flow_test/test.cairo:342–391` explicitly documents this outcome with `// TODO: Change this after implement calculate_rewards.` comments, confirming the rewards are stuck and the delegator receives nothing.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Delegators lose their proportional share of rewards that were legitimately earned and already transferred to the pool contract. The pool contract has no function to recover or redistribute these orphaned tokens. The test confirms the pool retains `> 100` tokens (a non-trivial amount, not rounding dust) while the delegator's reward address holds exactly 0.

---

### Likelihood Explanation

**High.** This scenario arises in the ordinary protocol lifecycle:
- A delegator joins a pool.
- The staker attests within fewer than 2 epochs of the delegation (which is the normal cadence).
- The staker subsequently calls `unstake_intent`.

No adversarial action is required. Any delegator who joins a pool shortly before a staker exits will lose their rewards.

---

### Recommendation

1. **Implement a complete `calculate_rewards` path** that correctly attributes rewards to delegators even when their balance was not yet active at the time of the attestation (the TODO in the test confirms this is pending).
2. **Add a governance-controlled sweep function** to the pool contract that can recover orphaned reward tokens and redistribute them, analogous to `withdrawProfits()` in `RageDepository`.
3. **Alternatively**, modify `calculate_staker_pools_rewards` in the staking contract to only count pool balance that is already active in the pool's member trace, ensuring the staking contract and pool contract share a consistent view of the delegated balance before sending rewards.

```text
        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
```
