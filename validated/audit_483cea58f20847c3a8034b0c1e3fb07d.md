### Title
Staker Can Suddenly Increase Commission to 100% via Commitment Mechanism, Stealing Delegator Yield - (File: src/staking/staking.cairo)

### Summary
A staker can use the `set_commission_commitment` + `set_commission` two-step mechanism to instantly raise their pool commission from a low advertised rate to 100%, redirecting all delegator rewards to themselves as commission. The codebase itself acknowledges this in a code comment. This is a direct analog to the LiquidityPoolProxy owner-controlled implementation swap: the staker is the privileged role, and the commission rate is the critical parameter that can be changed to redirect user funds.

### Finding Description
The `set_commission_commitment` function allows a staker to register a commitment with an arbitrary `max_commission` up to `COMMISSION_DENOMINATOR` (10000 = 100%). [1](#0-0) 

The only constraint is that `max_commission >= current_commission` and `expiration_epoch > current_epoch`. Once the commitment is active, `set_commission` (via `update_commission`) permits setting commission to **any value up to `max_commission`**, including a sudden jump to 100%: [2](#0-1) 

The codebase explicitly acknowledges this gap: [3](#0-2) 

When rewards are subsequently distributed via `update_rewards_from_attestation_contract` or `update_rewards`, the commission is read live from storage at calculation time: [4](#0-3) 

With commission = 10000, `split_rewards_with_commission` assigns 100% to `commission_rewards` (credited to the staker) and 0 to `pool_rewards` (credited to delegators). The pool contract receives zero tokens for that reward cycle. [5](#0-4) 

### Impact Explanation
Delegators who joined the pool expecting a low commission rate (e.g., 2%) receive **zero yield** for any epoch in which the staker has raised commission to 100%. The staker captures the entire pool reward as commission. This constitutes **theft of unclaimed yield** (High impact per the allowed scope). Already-distributed rewards sitting in the pool contract are unaffected; only the next reward distribution cycle is stolen.

### Likelihood Explanation
The attack requires only two transactions executable in the same block:
1. `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`
2. `set_commission(10000)`

No external dependency, no governance approval, and no time-lock is required. Delegators cannot exit in time: `exit_delegation_pool_intent` starts a 1-week exit window, which is far longer than the single epoch needed to execute the attack. Any staker with an active delegation pool can perform this. Likelihood is **Medium** (requires deliberate malicious intent, but the mechanism is fully permissionless and atomic).

### Recommendation
- **Short term**: Enforce a minimum notice period before a commission increase takes effect — for example, require that the new (higher) commission only applies starting from `current_epoch + K` (where K is the epoch delay already used for stake changes), giving delegators time to exit.
- **Long term**: Cap the maximum commission increase per commitment period (e.g., no more than +5% per epoch), and document the current risk clearly so delegators understand that a staker can raise commission to `max_commission` at any time during an active commitment window.

### Proof of Concept
1. Alice deploys a pool with `commission = 200` (2%) and attracts 1,000,000 STRK in delegations.
2. At the start of epoch N, Alice calls:
   ```
   set_commission_commitment(max_commission: 10000, expiration_epoch: N+1)
   ```
3. In the same block, Alice calls:
   ```
   set_commission(10000)   // commission is now 100%
   ```
4. Alice attests for epoch N, triggering `update_rewards_from_attestation_contract`.
5. Inside `calculate_staker_pools_rewards`, `commission = 10000` is read from storage.
6. `split_rewards_with_commission` returns `(pool_rewards_including_commission, 0)` — all pool rewards go to Alice as commission; the pool contract receives 0 STRK.
7. All delegators claim 0 rewards for epoch N. Alice has stolen the entire delegator yield for that epoch. [6](#0-5) [7](#0-6)

            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();

            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
