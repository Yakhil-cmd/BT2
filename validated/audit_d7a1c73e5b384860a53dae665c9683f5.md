### Title
Delegation Pool Receives STRK Rewards When Pool Balance Is Below `min_delegation_for_rewards`, Permanently Freezing Delegator Yield - (File: `src/pool/pool.cairo`)

### Summary
When a delegation pool's total balance is below `min_delegation_for_rewards`, the staking contract still calculates and forwards STRK rewards to the pool contract. However, the pool contract's `compute_rewards_per_unit` returns zero for balances below this threshold, so the cumulative rewards trace never updates. The forwarded STRK tokens become permanently stuck in the pool contract with no recovery path for delegators.

### Finding Description
The `compute_rewards_per_unit` function in `src/pool/pool.cairo` returns zero when `total_stake < min_delegation_for_rewards` to prevent arithmetic overflow: [1](#0-0) 

The code comment directly above this function acknowledges the consequence: [2](#0-1) 

Despite this, the staking contract's `calculate_staker_pools_rewards` computes `pool_rewards` proportional to `pool_balance_curr_epoch` with no check against `min_delegation_for_rewards`: [3](#0-2) 

Then `update_pool_rewards` unconditionally transfers those STRK tokens to the pool contract and calls `update_rewards_from_staking_contract`: [4](#0-3) 

Inside `update_rewards_from_staking_contract`, `compute_rewards_per_unit` is called with the pool balance. When that balance is below `min_delegation_for_rewards`, it returns 0, so the cumulative rewards trace is incremented by zero: [5](#0-4) 

The STRK tokens are now in the pool contract's balance, but the trace that drives `calculate_rewards` never recorded them. Delegators calling `claim_rewards` compute rewards from the unchanged trace and receive nothing. There is no administrative withdrawal function in the pool contract to recover these tokens.

        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }

            for (pool_contract, token_address, pool_balance, pool_rewards) in pools_rewards_data {

                        amount: pool_rewards,
                        token_dispatcher: strk_token_dispatcher,
                    );
                let decimals = self.get_token_decimals(:token_address);
                pool_dispatcher
                    .update_rewards_from_staking_contract(
                        rewards: pool_rewards,
                        pool_balance: pool_balance.to_native_amount(:decimals),
                    );
                pool_rewards_list.append((pool_contract, pool_rewards));
            }
            pool_rewards_list
        }
