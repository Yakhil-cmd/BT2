### Title
Unbounded Loop in `calculate_rewards` Enables Permanent Freezing of Unclaimed Yield via Trace Bloat - (File: src/pool/pool.cairo)

### Summary

The `calculate_rewards` function in `pool.cairo` contains an unbounded loop that iterates over every entry in a pool member's `pool_member_epoch_balance` trace since their last claim. Because `add_to_delegation_pool` and `exit_delegation_pool_intent` each append a new trace entry per distinct epoch, a delegator (or their reward address) who makes repeated balance changes across many epochs without claiming rewards will accumulate an arbitrarily large trace. When `claim_rewards` is eventually called, the loop must process all accumulated entries, potentially exceeding Starknet's block gas limit and permanently freezing the delegator's unclaimed yield.

### Finding Description

`claim_rewards` in `pool.cairo` calls `calculate_rewards`, which contains the following loop:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    rewards += compute_rewards_rounded_down(...);
    entry_to_claim_from += 1;
}
```

The loop iterates from `entry_to_claim_from` (the index saved at the last `claim_rewards` call) up to the current trace length. Each call to `add_to_delegation_pool` or `exit_delegation_pool_intent` invokes `set_member_balance`, which calls `trace.insert(key: self.get_epoch_plus_k(), ...)`. The `insert` function appends a **new** checkpoint whenever the epoch key differs from the last entry's key — meaning one new entry is added per distinct epoch in which a balance change occurs.

`add_to_delegation_pool` is callable by both the pool member and their `reward_address`:

```cairo
assert!(
    caller_address == pool_member || caller_address == pool_member_info.reward_address,
    "{}",
    Error::CALLER_CANNOT_ADD_TO_POOL,
);
```

The minimum amount enforced is only `amount.is_non_zero()` (i.e., 1 wei), so the cost per trace entry is negligible.

### Impact Explanation

If a delegator (or their reward address) makes one balance change per epoch across N epochs without claiming rewards, the `pool_member_epoch_balance` trace grows to N entries. When `claim_rewards` is finally called, the loop must process all N entries. If N is large enough to exceed Starknet's per-transaction gas limit, `claim_rewards` reverts on every attempt, permanently freezing the delegator's unclaimed yield. The `entry_to_claim_from` cursor is only advanced inside `claim_rewards`, so there is no way to process the trace in batches across multiple calls.

**Allowed impact matched**: *Permanent freezing of unclaimed yield* (High).

### Likelihood Explanation

The trace grows at most one entry per epoch. If epochs are on the order of weeks, reaching a trace size that exceeds gas limits requires years of continuous balance changes without claiming. However:

- The minimum cost per entry is 1 wei, so the economic barrier is negligible.
- The reward address (which may be a contract) can call `add_to_delegation_pool` independently of the pool member.
- The developers themselves flag this in a code comment: *"This loop is unbounded but unlikely to exceed gas limits"* — acknowledging the risk exists.
- A delegator who legitimately adds to their delegation every epoch and defers claiming for a long period faces this organically, without any malicious actor.

Likelihood is **low** but the consequence when triggered is irreversible.

### Recommendation

1. **Checkpoint-based batching**: Allow `claim_rewards` to accept a `max_entries` parameter so rewards can be claimed incrementally across multiple transactions, advancing `entry_to_claim_from` each time.
2. **Enforce regular claiming**: Require or incentivize claiming before balance changes, so the trace never grows beyond a bounded window.
3. **Bound the trace at write time**: Reject or auto-claim when a balance change would push the unclaimed trace length beyond a safe threshold.

### Proof of Concept

1. Delegator enters pool via `enter_delegation_pool`.
2. Each epoch, delegator calls `add_to_delegation_pool(pool_member, 1)` — appending one entry to `pool_member_epoch_balance` per epoch.
3. Delegator never calls `claim_rewards`, so `entry_to_claim_from` stays at 0.
4. After N epochs, `pool_member_epoch_balance` has N entries.
5. Delegator calls `claim_rewards`. The loop in `calculate_rewards` iterates N times, each iteration reading a storage slot and calling `find_sigma`. For sufficiently large N, this exceeds the block gas limit and reverts.
6. Every subsequent `claim_rewards` call also reverts. The delegator's accumulated yield is permanently inaccessible.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
