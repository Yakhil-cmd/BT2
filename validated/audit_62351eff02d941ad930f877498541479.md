### Title
`PresignOutput` Not Consumed After Rerandomization Enables Nonce Reuse and Private Key Extraction — (`src/ecdsa/ot_based_ecdsa/mod.rs`)

---

### Summary

`RerandomizedPresignOutput::rerandomize_presign` accepts the `PresignOutput` by shared reference (`&PresignOutput`) rather than by value, so the underlying presignature is never consumed. A malicious coordinator can exploit this by issuing two distinct `RerandomizationArguments` (for two different messages) against the same presignature held by honest participants. Because the rerandomization scalar `delta` is derived entirely from public inputs, an observer who collects both resulting signatures can recover the shared nonce `k` and extract the private signing key.

---

### Finding Description

**Root cause — API does not consume the presignature:**

`rerandomize_presign` is defined as:

```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // ← shared reference, NOT consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
``` [1](#0-0) 

Because `presignature` is borrowed, the caller retains full ownership of the `PresignOutput` after the call returns. Nothing in the type system prevents calling `rerandomize_presign` a second time with the same `PresignOutput` and a different `RerandomizationArguments`.

By contrast, the downstream `sign` function correctly takes `presignature: RerandomizedPresignOutput` by value, consuming it:

```rust
pub fn sign(
    ...
    presignature: RerandomizedPresignOutput,   // ← owned, consumed
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError>
``` [2](#0-1) 

This asymmetry means the one-time-use invariant is enforced for `RerandomizedPresignOutput → sign`, but **not** for `PresignOutput → rerandomize_presign`.

**How rerandomization works (and why reuse breaks it):**

`rerandomize_presign` computes:

```
delta  = HKDF(entropy, pk, tweak, msg_hash, big_r, participants)
k_new  = k / delta
R_new  = delta · R
sigma_new = (sigma + tweak·k) / delta
``` [3](#0-2) 

`delta` is derived exclusively from **public** fields of `RerandomizationArguments` (entropy, pk, tweak, msg\_hash, big\_r, participants): [4](#0-3) 

If the same `PresignOutput` (same underlying `k`) is rerandomized twice — once for message `h1` producing `delta1`, and once for message `h2` producing `delta2` — then:

```
k_new1 = k / delta1
k_new2 = k / delta2
```

Both `delta1` and `delta2` are computable by any observer from the public `RerandomizationArguments`. Therefore:

```
k = k_new1 · delta1 = k_new2 · delta2
```

Standard ECDSA nonce-reuse recovery then yields the private key from the two signatures `(R1, s1)` and `(R2, s2)`.

**Attack path — malicious coordinator:**

1. Coordinator and honest participants complete the presign protocol; each participant holds a `PresignOutput` with secret shares `(k_i, sigma_i)`.
2. Coordinator constructs `RerandomizationArguments` for message `h1` and distributes them. Participants call `rerandomize_presign(&presign_out, &args1)` and then `sign(...)`. The `PresignOutput` is **not** destroyed.
3. Coordinator constructs a second `RerandomizationArguments` for message `h2` (different `msg_hash`, same `big_r`). Participants, unaware that the presignature has already been used, call `rerandomize_presign(&presign_out, &args2)` and `sign(...)` again.
4. Coordinator collects both signatures. Since `delta1` and `delta2` are fully public, the coordinator recovers `k` and then the private key.

The documentation explicitly warns that presignatures must never be reused: [5](#0-4) [6](#0-5) 

Yet the API provides no enforcement mechanism.

---

### Impact Explanation

**Critical — Extraction of private signing shares.**

Nonce reuse in ECDSA is a well-known total-break: given two signatures `(r, s1)` and `(r, s2)` over messages `h1` and `h2` sharing the same nonce `k`, the private key `x` is recovered as:

```
k  = (h1 - h2) / (s1 - s2)
x  = (s1·k - h1) / r
```

Because `delta` is public, the attacker can reduce the two rerandomized nonces back to the same `k`, making this a straightforward private-key extraction. All signing shares held by honest participants are compromised.

---

### Likelihood Explanation

The attack requires a coordinator role, which is a normal protocol participant — no privileged access is needed. The coordinator is the natural aggregator of signatures and already receives all partial signature shares. A single malicious coordinator can execute this attack against all honest participants in one presign session. The only precondition is that participants do not independently track presignature usage, which the library API does not help them do.

---

### Recommendation

Change `rerandomize_presign` to consume the `PresignOutput` by value:

```rust
pub fn rerandomize_presign(
    presignature: PresignOutput,   // owned — compiler enforces single use
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

Rust's ownership system will then statically prevent any caller from passing the same `PresignOutput` to `rerandomize_presign` more than once, mirroring the existing protection already applied to `RerandomizedPresignOutput` in `sign`.

---

### Proof of Concept

```
1. Run presign for participants [P1, P2, P3]; each holds PresignOutput { big_r: R, k: k_i, sigma: sigma_i }.

2. Malicious coordinator constructs:
     args1 = RerandomizationArguments { msg_hash: h1, entropy: e1, big_r: R, ... }
     args2 = RerandomizationArguments { msg_hash: h2, entropy: e2, big_r: R, ... }

3. Coordinator sends args1 to participants.
   Each participant calls:
     rp1 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args1)
     // presign_out is NOT consumed
   Participants run sign(rp1, h1) → signature (R1, s1).

4. Coordinator sends args2 to participants.
   Each participant calls:
     rp2 = RerandomizedPresignOutput::rerandomize_presign(&presign_out, &args2)
   Participants run sign(rp2, h2) → signature (R2, s2).

5. Coordinator computes:
     delta1 = args1.derive_randomness()   // fully public
     delta2 = args2.derive_randomness()   // fully public
     k_agg  = k_new1 * delta1             // = k_new2 * delta2
     x      = (s1*k_agg - h1) / r1       // private key recovered
```

    pub fn derive_randomness(&self) -> Result<Scalar, ProtocolError> {
        // create a string containing (pk, msg_hash, big_r, sorted(participants))
        let pk_encoded_point = self.pk.to_encoded_point(true);
        let encoded_pk: &[u8] = pk_encoded_point.as_bytes();
        let encoded_tweak: &[u8] = &<Secp256K1ScalarField as Field>::serialize(&self.tweak.value());
        let encoded_msg_hash: &[u8] = &self.msg_hash;
        let big_r_encoded_point = self.big_r.to_encoded_point(true);
        let encoded_big_r: &[u8] = big_r_encoded_point.as_bytes();

        // concatenate all the bytes
        let mut concatenation = Vec::new();
        // 1 byte counter, used in the unlikely case that the hash result is 0
        concatenation.extend_from_slice(&[0u8, 1]);
        concatenation.extend_from_slice(encoded_pk);
        concatenation.extend_from_slice(encoded_tweak);
        concatenation.extend_from_slice(encoded_msg_hash);
        concatenation.extend_from_slice(encoded_big_r);
        // Append each ParticipantId's
        for participant in self.participants.participants() {
            concatenation.extend_from_slice(&participant.bytes());
        }

        // initiate hkdf with the salt and with some `good' entropy
        let hk = Hkdf::<sha3::Sha3_256>::new(Some(&Self::SALT), &self.entropy);

        let mut delta = Scalar::ZERO;
        // If the randomness created is 0 then we want to generate a new randomness
        while bool::from(delta.is_zero()) {
            // Generate randomization out of HKDF(counter, entropy, pk, msg_hash, big_r, participants, )
            // where entropy is a public but unpredictable random string.
            // The counter depends on the number of times we enter into this loop
            let mut okm = [0u8; 32];

            hk.expand(&concatenation, &mut okm)
                .map_err(|_| ProtocolError::HashingError)?;

            // derive the randomness delta
            delta = Scalar::from_repr(okm.into()).unwrap_or(
                // if delta falls outside the field
                // probability is negligible: in the order of 1/2^224
                Scalar::ZERO,
            );
            // Increment the counter, the probability that this overflows is astronomically low
            let concatenation_0 = concatenation
                .first_mut()
                .ok_or(ProtocolError::InvalidIndex)?;
            *concatenation_0 += 1;
        }
        Ok(delta)
    }
