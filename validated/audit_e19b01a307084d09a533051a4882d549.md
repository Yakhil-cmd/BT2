### Title
Unauthorized Global Contract Code Replacement via Cross-Account `DeployGlobalContractAction` (AccountId mode) — (`File: runtime/runtime/src/global_contracts.rs`)

### Summary

`DeployGlobalContractAction` with `GlobalContractDeployMode::AccountId` stores the deployed Wasm binary under the **receiver account's** ID as the trie key. Unlike `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, and `DeleteAccount`, `DeployGlobalContract` is **not** restricted to receipts where `predecessor_id == receiver_id`. Any smart contract can therefore create a cross-account promise targeting victim account `B` with `DeployGlobalContract(AccountId mode)`, unconditionally overwriting the global contract code stored under `B`'s account ID. Every account that has opted in via `UseGlobalContract(AccountId(B))` will subsequently execute the attacker-supplied Wasm.

### Finding Description

**Invariant violated**: Only the account whose ID is used as the `AccountId`-mode global contract identifier should be able to update that contract. The protocol documentation explicitly states: *"This allows the **owner** to update the contract for all its users."*

**Root cause — `action_deploy_global_contract`** in `runtime/runtime/src/global_contracts.rs`:

```rust
pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,   // ← always the receipt receiver_id
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    // Only checks balance; NO predecessor_id == receiver_id guard
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else { ... };
    ...
    initiate_distribution(state_update, account_id.clone(), ...)?;
}
``` [1](#0-0) 

Inside `initiate_distribution`, when `deploy_mode == AccountId`, the identifier is derived from the **receiver's** `account_id`, not the sender's:

```rust
GlobalContractDeployMode::AccountId => {
    GlobalContractIdentifier::AccountId(account_id.clone())
}
``` [2](#0-1) 

**Missing guard**: The protocol spec lists exactly which actions require `predecessor_id == receiver_id`:

> `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, `DeleteAccount`

`DeployGlobalContract` is **absent** from this list. [3](#0-2) 

The action-level validator `validate_deploy_global_contract_action` only enforces a size limit — no ownership or sender-identity check:

```rust
fn validate_deploy_global_contract_action(
    limit_config: &LimitConfig,
    action: &DeployGlobalContractAction,
) -> Result<(), ActionsValidationError> {
    if action.code.len() as u64 > limit_config.max_contract_size {
        return Err(ActionsValidationError::ContractSizeExceeded { ... });
    }
    Ok(())
}
``` [4](#0-3) 

The VM host function `promise_batch_action_deploy_global_contract_by_account_id` imposes no restriction on which account the promise targets — `sir` (sender-is-receiver) only affects gas pricing: [5](#0-4) 

**Exact corrupted state**: The trie key `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim_id) }` is overwritten with attacker-controlled Wasm bytes, and `TrieKey::GlobalContractNonce { identifier: ... }` is incremented, causing the distribution receipt to propagate the malicious code to all shards. [6](#0-5) 

### Impact Explanation

All accounts that have called `UseGlobalContract(AccountId(victim))` now point to attacker-controlled Wasm. On the next function call to any such account, the attacker's code executes with full access to that account's storage and balance via the NEAR host functions (`storage_write`, `promise_batch_create`, etc.). This enables:

- **Theft of funds** from every account using the hijacked global contract.
- **State corruption** across all shards (the distribution receipt propagates the malicious code globally).
- **Secondary balance drain** of the victim deployer account (storage cost is deducted from the receiver's balance, not the attacker's). [7](#0-6) 

### Likelihood Explanation

The attack requires only:
1. A funded account with any deployed smart contract (standard user capability).
2. Knowledge of the victim's account ID (public information).
3. One cross-account function call that emits a `DeployGlobalContract(AccountId)` promise targeting the victim.

No validator, operator, or privileged role is needed. The `GlobalContractDeployMode::AccountId` feature is production-enabled and the host function is exposed to all Wasm contracts.

### Recommendation

Add a `predecessor_id == receiver_id` guard inside `action_deploy_global_contract` (or enforce it at the receipt-application layer alongside `DeployContract`), specifically when `deploy_mode == GlobalContractDeployMode::AccountId`. The `CodeHash` mode is inherently safe because the identifier is derived from the code content, not from a mutable account-controlled namespace. The fix should mirror the existing guard applied to `DeployContract`:

```rust
// In apply_action_receipt, alongside the existing check for DeployContract:
Action::DeployGlobalContract(deploy) if matches!(deploy.deploy_mode, GlobalContractDeployMode::AccountId) => {
    if predecessor_id != receiver_id {
        return Err(ActionErrorKind::ActorNoPermission { ... }.into());
    }
}
```

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());

        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

pub(crate) fn use_global_contract(
    state_update: &mut TrieUpdate,
    account_id: &AccountId,
    account: &mut Account,
    contract_identifier: &GlobalContractIdentifier,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let key = TrieKey::GlobalContractCode { identifier: contract_identifier.clone().into() };
    if !state_update.contains_key(&key, AccessOptions::DEFAULT)? {
        result.result = Err(ActionErrorKind::GlobalContractDoesNotExist {
            identifier: contract_identifier.clone(),
        }
        .into());
        return Ok(());
    }
    clear_account_contract_storage_usage(state_update, account_id, account)?;
    if account.contract().is_local() {
        state_update.remove(TrieKey::ContractCode { account_id: account_id.clone() });
    }
    let contract = match contract_identifier {
        GlobalContractIdentifier::CodeHash(code_hash) => AccountContract::Global(*code_hash),
        GlobalContractIdentifier::AccountId(id) => AccountContract::GlobalByAccount(id.clone()),
    };
    account.set_storage_usage(
        account.storage_usage().checked_add(contract_identifier.len() as u64).ok_or_else(|| {
            StorageError::StorageInconsistentState(format!(
                "Storage usage integer overflow for account {}",
                account_id
            ))
        })?,
    );
    account.set_contract(contract);
    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())

fn apply_distribution_current_shard(
    receipt: &Receipt,
    global_contract_data: &GlobalContractDistributionReceipt,
    apply_state: &ApplyState,
    state_update: &mut TrieUpdate,
) -> Result<Compute, RuntimeError> {
    let identifier = match &global_contract_data.id() {
        GlobalContractIdentifier::CodeHash(hash) => GlobalContractCodeIdentifier::CodeHash(*hash),
        GlobalContractIdentifier::AccountId(account_id) => {
            GlobalContractCodeIdentifier::AccountId(account_id.clone())
        }
    };

    let is_nonce_fresh = check_and_update_nonce(global_contract_data, &identifier, state_update)?;
    if !is_nonce_fresh {
        return Ok(0);
    }

    let config = apply_state.config.wasm_config.clone();
    let trie_key = TrieKey::GlobalContractCode { identifier };
    let code_len = global_contract_data.code().len() as u64;
    state_update.set(trie_key, global_contract_data.code().to_vec());
    state_update.commit(StateChangeCause::ReceiptProcessing { receipt_hash: receipt.get_hash() });
    let code_hash = match global_contract_data.id() {
        GlobalContractIdentifier::CodeHash(hash) => Some(*hash),
        GlobalContractIdentifier::AccountId(_) => None,
    };
    precompile_contract_with_warming(
        &ContractCode::new(global_contract_data.code().to_vec(), code_hash),
        config,
        apply_state.next_wasm_config.clone(),
        apply_state.cache.as_deref(),
    );
    near_vm_runner::report_metrics(apply_state.shard_id, "global_contract");
    let fees = &apply_state.config.fees;
    let per_byte_total = fees
        .deploy_global_contract_execution_per_byte
        .checked_mul(code_len)
        .ok_or(IntegerOverflowError)?;
    let compute = fees
        .deploy_global_contract_execution_base
        .checked_add(per_byte_total)
        .ok_or(IntegerOverflowError)?;
    Ok(compute)
}
