### Title
Hardcoded `DEFAULT_AVG_BLOCK_DURATION` with No Setter Causes First-Epoch Per-Block Reward Miscalculation — (File: src/reward_supplier/reward_supplier.cairo)

---

### Summary

`RewardSupplier` initializes `avg_block_duration` to a hardcoded constant (`DEFAULT_AVG_BLOCK_DURATION = 3 * BLOCK_DURATION_SCALE = 300`, i.e. 3 s) and has no setter for it (a `// TODO: Setter.` comment acknowledges the gap). Because the first call to `set_avg_block_duration` always returns early without updating the value, the entire first epoch's per-block reward calculation runs on this hardcoded figure. If the actual Starknet block time deviates from 3 s, every staker's reward for that epoch is proportionally wrong.

---

### Finding Description

**Hardcoded default with no setter** [1](#0-0) 

```cairo
pub(crate) const DEFAULT_AVG_BLOCK_DURATION: u64 = 3 * BLOCK_DURATION_SCALE;
```

The constructor writes this value unconditionally and there is no public function to override it: [2](#0-1) 

```cairo
self.avg_block_duration.write(DEFAULT_AVG_BLOCK_DURATION);
```

The storage field comment confirms the setter is missing: [3](#0-2) 

```cairo
/// Average block duration in units of 1 / BLOCK_DURATION_SCALE seconds.
// TODO: Setter.
// TODO: View?
avg_block_duration: u64,
```

**First call always skips the update**

`set_avg_block_duration` is the only path that can change `avg_block_duration`. On the very first invocation the snapshot is `(0, 0)`, so the function stores the snapshot and returns without writing a new duration: [4](#0-3) 

```cairo
if snapshot_block_number.is_zero() || snapshot_timestamp.is_zero() {
    return;
}
```

**Hardcoded value feeds the reward formula**

`update_current_epoch_block_rewards` reads `avg_block_duration` immediately after calling `set_avg_block_duration`, so the first epoch always uses `300`: [5](#0-4) 

```cairo
let avg_block_duration = self.avg_block_duration.read();
let total_rewards = mul_wide_and_div(
    lhs: yearly_mint,
    rhs: avg_block_duration.into(),
    div: BLOCK_DURATION_SCALE.into() * SECONDS_IN_YEAR.into(),
)
```

The formula is `yearly_mint × avg_block_duration / (BLOCK_DURATION_SCALE × SECONDS_IN_YEAR)`. A wrong `avg_block_duration` scales every staker's reward linearly.

---

### Impact Explanation

- **Over-rewarding** (actual block time < 3 s): per-block rewards are inflated. Stakers collectively drain the `RewardSupplier` balance faster than the minting curve intends, constituting theft of unclaimed yield from the pool that should cover future epochs.
- **Under-rewarding** (actual block time > 3 s): per-block rewards are deflated. Stakers permanently lose yield they are entitled to for the first epoch — yield that is never retroactively corrected.

Both cases map to the **High** impact tier (theft / permanent freezing of unclaimed yield).

---

### Likelihood Explanation

Starknet's current average block time is close to 3 s, so the default is not wildly wrong today. However:
1. Block times fluctuate with network load and sequencer upgrades.
2. There is no mechanism to correct the first epoch retroactively.
3. The missing setter (acknowledged by the TODO) means an operator cannot compensate even if they detect the discrepancy before the epoch ends.

Likelihood is **low-to-medium**: the miscalculation is bounded but non-zero, and the window is exactly one epoch.

---

### Recommendation

Either:
- Accept `avg_block_duration` as a constructor parameter (analogous to how `attestation_window` is passed to the `Attestation` constructor), or
- Implement the missing setter guarded by `only_app_governor` so the value can be corrected before the first epoch's reward call.

```cairo
// In constructor:
pub fn constructor(
    ...
    initial_avg_block_duration: u64,  // <-- add parameter
) {
    ...
    self.avg_block_duration.write(initial_avg_block_duration);
}
```

---

### Proof of Concept

1. Deploy `RewardSupplier`; `avg_block_duration` = 300 (3 s).
2. Staking contract calls `update_current_epoch_block_rewards` at the first epoch boundary.
3. `set_avg_block_duration` fires, finds `snapshot_block_number == 0`, stores the snapshot, and returns — `avg_block_duration` stays at 300.
4. Reward formula executes with `avg_block_duration = 300`.
5. If the actual Starknet block time at that moment is 2 s (200 in scaled units), rewards are over-paid by `300/200 = 1.5×` — a 50 % inflation of first-epoch yield drawn from the supplier's balance.
6. No setter exists to correct this before or after the fact.
