### Title
`TurnBridge` Is a No-Op — Bridge Cannot Be Disabled by Any Actor - (File: `x/cronos/keeper/msg_server.go`)

### Summary

The `MsgTurnBridge` message is defined in the proto with an `enable bool` field, clearly intending to allow authorized actors to toggle the IBC/bridge on and off. However, the server-side implementation is a complete no-op: it performs no permission check, writes no state, and always returns `(nil, nil)`. As a result, the bridge can never be disabled, and all bridge-transfer entry points (`SendToIbcHandler`, `SendToIbcV2Handler`, `SendCroToIbcHandler`, `IbcTransferCoins`, `ConvertVouchersToEvmCoins`) contain no bridge-enabled guard. Any emergency bridge shutdown is permanently impossible.

### Finding Description

`MsgTurnBridge` is declared in the protobuf service and wired into the message router, but its handler body is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

No bridge-enabled flag is ever written to the KV store. Consequently, none of the bridge execution paths check such a flag:

- `SendToIbcHandler.Handle` proceeds directly to `IbcTransferCoins` with no bridge-state check. [2](#0-1) 
- `SendCroToIbcHandler.Handle` only validates the contract address whitelist, not bridge-enabled state. [3](#0-2) 
- `SendToIbcV2Handler.Handle` delegates to the same `handle()` path with no bridge-state check. [4](#0-3) 
- `IbcTransferCoins` and `ConvertVouchersToEvmCoins` in the keeper contain no bridge-enabled guard. [5](#0-4) 

The proto definition confirms the intent to support toggling:

```protobuf
message MsgTurnBridge {
  option (cosmos.msg.v1.signer) = "sender";
  string sender = 1;
  bool   enable = 2;
}
``` [6](#0-5) 

### Impact Explanation

Because `TurnBridge` is a no-op and no bridge-enabled state is ever stored or checked, it is impossible for governance or the Cronos admin to halt bridge operations under any circumstances — including active exploits, critical bugs, or emergency security incidents. Any unprivileged user can continue to call `__CronosSendToIbc`, `__CronosSendToIbcV2`, or `__CronosSendCroToIbc` EVM events, or submit `MsgTransferTokens` / `MsgConvertVouchers` Cosmos transactions, and the bridge will process them unconditionally. This is a permanent bypass of the bridge authorization/control mechanism, matching the **High** impact category: *Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks*.

### Likelihood Explanation

The vulnerability is always present and requires no special conditions. Any user can submit bridge transactions at any time. The only prerequisite for harm is a scenario where the bridge *should* be disabled (e.g., a live exploit draining funds via the bridge), which is precisely when the missing control matters most.

### Recommendation

1. Add a `BridgeEnabled bool` field to the `Params` struct (or a dedicated KV store key).
2. Implement `TurnBridge` to write this flag after verifying the sender has the appropriate permission (e.g., `CanTurnBridge` permission or governance authority).
3. Add a guard at the top of `IbcTransferCoins` and `ConvertVouchersToEvmCoins` (and in each EVM log handler's `Handle` method) that returns an error if the bridge is disabled.

### Proof of Concept

1. Governance submits and passes a `MsgTurnBridge{sender: gov_module, enable: false}` transaction.
2. The transaction is accepted (returns `nil` error) but writes nothing to state.
3. An attacker immediately calls `__CronosSendToIbc` from any CRC20 contract mapped to an IBC denom.
4. `SendToIbcHandler.Handle` → `IbcTransferCoins` → `ibcSendTransfer` executes successfully, transferring tokens out of Cronos despite the governance-mandated bridge shutdown.
5. The bridge remains fully operational indefinitely; no subsequent `TurnBridge` call can change this.

func (h SendToIbcHandler) handle(
	ctx sdk.Context,
	contract common.Address,
	senderAddress common.Address,
	recipient string,
	amountInt *big.Int,
	id *big.Int,
) error {
	denom, found := h.cronosKeeper.GetDenomByContract(ctx, contract)
	if !found {
		return fmt.Errorf("contract %s is not connected to native token", contract)
	}

	if !types.IsValidIBCDenom(denom) && !types.IsValidCronosDenom(denom) {
		return fmt.Errorf("the native token associated with the contract %s is neither an ibc voucher or a cronos token", contract)
	}

	contractAddr := sdk.AccAddress(contract.Bytes())
	sender := sdk.AccAddress(senderAddress.Bytes())
	amount := sdkmath.NewIntFromBigInt(amountInt)
	coins := sdk.NewCoins(sdk.NewCoin(denom, amount))

	var err error
	if types.IsSourceCoin(denom) {
		// it is a source token, we need to mint coins
		if err = h.bankKeeper.MintCoins(ctx, types.ModuleName, coins); err != nil {
			return err
		}
		// send the coin to the user
		if err = h.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, sender, coins); err != nil {
			return err
		}
	} else {
		// First, transfer IBC coin to user so that he will be the refunded address if transfer fails
		if err = h.bankKeeper.SendCoins(ctx, contractAddr, sender, coins); err != nil {
			return err
		}
	}

	channelId := ""
	if id != nil {
		channelId = "channel-" + id.String()
	}
	// Initiate IBC transfer from sender account
	if err = h.cronosKeeper.IbcTransferCoins(ctx, sender.String(), recipient, coins, channelId); err != nil {
		return err
	}
	return nil
}

func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
	}
	defer func() {
		for _, a := range coins {
			if a.Amount.IsInt64() {
				telemetry.SetGaugeWithLabels(
					[]string{"tx", "msg", "ConvertVouchersToEvmCoins"},
					float32(a.Amount.Int64()),
					[]metrics.Label{telemetry.NewLabel("denom", a.Denom)},
				)
			}
		}
	}()
	return nil
}
