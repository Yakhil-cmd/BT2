### Title
Any Caller Can Consume the Global Block-Reward Slot with `disable_rewards: true`, Permanently Freezing Staker Yield - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `staking.cairo` is documented as "Only starkware sequencer" but has **no access-control enforcement** in code. Any unprivileged caller can invoke it with `disable_rewards: true`, which updates the global `last_reward_block` without distributing any rewards. Because the contract enforces a strict one-call-per-block invariant (`REWARDS_ALREADY_UPDATED`), the legitimate sequencer call is then rejected for that block. An attacker who front-runs every sequencer call can permanently freeze all staker block rewards.

---

### Finding Description

`update_rewards` is the consensus-era reward distribution entry point. Its only gate is `general_prerequisites()`, which checks:

1. Contract is not paused
2. Caller is not the zero address [1](#0-0) 

There is no check that the caller is the Starkware sequencer, despite the spec explicitly stating "Only starkware sequencer": [2](#0-1) 

The function signature accepts a caller-controlled `disable_rewards: bool` parameter: [3](#0-2) 

When `disable_rewards: true`, the function writes `last_reward_block` to the current block and returns immediately without distributing any rewards: [4](#0-3) 

`last_reward_block` is a **single global** storage variable (not per-staker): [5](#0-4) 

The one-call-per-block guard checks this global value: [6](#0-5) 

**Attack path:**

1. Attacker monitors the mempool/block for the sequencer's `update_rewards` transaction.
2. Attacker front-runs it by calling `update_rewards(staker_address: any_valid_active_staker, disable_rewards: true)`.
3. `last_reward_block` is set to the current block; no rewards are distributed.
4. The sequencer's legitimate call reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block → all stakers permanently receive zero block rewards.

The `staker_address` argument only needs to pass the "staker exists and is active" check; the attacker can use any known active staker's address. [7](#0-6) 

---

### Impact Explanation

All stakers and pool members lose 100% of their consensus-era block rewards for every block the attacker front-runs. Because `last_reward_block` is global, a single attacker call per block is sufficient to freeze yield for the entire protocol. This maps to **Permanent freezing of unclaimed yield** (High severity).

---

### Likelihood Explanation

- No privileged role or leaked key is required — any EOA or contract can call `update_rewards`.
- The attack requires only a standard front-run (higher gas or priority fee) on a predictable, recurring sequencer transaction.
- The cost to the attacker is only gas; the damage to stakers is total loss of block rewards.
- Likelihood: **High**.

---

### Recommendation

Add an explicit caller check inside `update_rewards` to restrict it to the authorized sequencer address (stored in contract storage or a role), consistent with the spec's stated access control:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
}
```

Alternatively, store the sequencer address in a dedicated storage slot and enforce it here, mirroring the pattern used for `assert_caller_is_attestation_contract`: [8](#0-7) 

---

### Proof of Concept

```
// Precondition: consensus rewards are active, staker_address is a valid active staker.
// Attacker calls (with higher gas priority than the sequencer):
IStakingRewardsManager(staking_contract)
    .update_rewards(staker_address: known_active_staker, disable_rewards: true);

// Result: last_reward_block == current_block_number, no rewards distributed.

// Sequencer's legitimate call now reverts:
IStakingRewardsManager(staking_contract)
    .update_rewards(staker_address: known_active_staker, disable_rewards: false);
// → panics with REWARDS_ALREADY_UPDATED

// Stakers receive 0 block rewards for this block.
// Repeat every block to permanently freeze all yield.
``` [9](#0-8) [10](#0-9)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
