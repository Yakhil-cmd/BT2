### Title
Staker Can Increase Commission to 100% via Commitment Mechanism, Stealing All Delegator Rewards — (`src/staking/staking.cairo`)

### Summary
The `set_commission_commitment` function places no upper cap on `max_commission` below `COMMISSION_DENOMINATOR` (10000 = 100%). A staker can exploit this to raise their commission to 100% after delegators have joined, causing `split_rewards_with_commission` to assign the entire pool reward to the staker as commission and zero to delegators — a direct theft of unclaimed yield.

### Finding Description

`set_commission_commitment` validates only that `max_commission <= COMMISSION_DENOMINATOR`: [1](#0-0) 

Once an active commitment exists, `update_commission` allows the staker to set commission to **any** value up to `max_commission`, including values **higher** than the current commission: [2](#0-1) 

Without a commitment, commission can only be decreased: [3](#0-2) 

When commission reaches `COMMISSION_DENOMINATOR`, `split_rewards_with_commission` computes:

```
commission_rewards = rewards_including_commission * 10000 / 10000 = rewards_including_commission
pool_rewards = rewards_including_commission - commission_rewards = 0
``` [4](#0-3) 

The zero `pool_rewards` value is then forwarded to the pool contract, meaning every delegator receives nothing for that epoch. [5](#0-4) 

### Impact Explanation

**High — Theft of unclaimed yield.**

All rewards that would have accrued to delegators are instead captured by the staker as commission. The delegators contributed real stake and are entitled to proportional rewards; those rewards are redirected to the staker with no recourse.

### Likelihood Explanation

**High.** The attack requires no special privilege beyond being a registered staker. The steps are:

1. Stake with a low commission (e.g., 100 = 1%) to attract delegators.
2. Call `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
3. Call `set_commission(commission: 10000)` — passes because `10000 <= max_commission` and `10000 != old_commission`.
4. Delegators receive zero rewards for the commitment period.
5. After expiry, set a new commitment and repeat.

The attack is repeatable and requires only on-chain transactions available to any staker.

### Recommendation

Enforce a protocol-level maximum commission cap (e.g., `MAX_COMMISSION = 5000` for 50%) in both `set_commission` (initial initialization path) and `set_commission_commitment`:

```cairo
const MAX_COMMISSION: Commission = 5000; // 50%

// In set_commission (initialization branch):
assert!(commission <= MAX_COMMISSION, "{}", Error::COMMISSION_OUT_OF_RANGE);

// In set_commission_commitment:
assert!(max_commission <= MAX_COMMISSION, "{}", Error::COMMISSION_OUT_OF_RANGE);
```

This mirrors the fix recommended in the external report: enforce a floor/ceiling on the percentage parameter so that the non-operator party (delegators) always retains a meaningful share of rewards.

### Proof of Concept

```
1. Staker calls stake() with commission = 100 (1%).
2. Delegators enter the pool, contributing stake.
3. Staker calls set_commission_commitment(max_commission: 10000, expiration_epoch: E+1).
   → Passes: 10000 <= COMMISSION_DENOMINATOR, current_commission (100) <= max_commission (10000).
4. Staker calls set_commission(commission: 10000).
   → update_commission: commitment is active, 10000 <= 10000 ✓, 10000 != 100 ✓ → succeeds.
5. Attestation triggers _update_rewards → calculate_staker_pools_rewards:
     pool_rewards_including_commission = (total_rewards * pool_balance) / total_stake  [non-zero]
     split_rewards_with_commission(pool_rewards_including_commission, commission=10000):
       commission_rewards = pool_rewards_including_commission * 10000 / 10000
                          = pool_rewards_including_commission
       pool_rewards = 0
6. Pool contract receives 0 tokens. All delegators claim 0 rewards.
   Staker's unclaimed_rewards includes the full pool_rewards_including_commission as commission.
```

pub(crate) fn split_rewards_with_commission(

    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
