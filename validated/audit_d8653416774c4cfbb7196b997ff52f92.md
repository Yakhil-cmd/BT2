### Title
Gateway L2 Gas Price Admission Check Uses Previous-Block Price Instead of Next-Block Price, Causing Valid Transaction Rejection and Invalid Transaction Admission — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` gates admission on the **previous block's** L2 gas price, but every admitted transaction is executed in the **next block** at the EIP-1559-computed `next_l2_gas_price`. The two prices diverge every block. When the price falls, valid transactions are rejected at the gateway; when the price rises, invalid transactions are admitted and then fail inside the batcher. A developer TODO in the code explicitly acknowledges the wrong value is being used.

---

### Finding Description

`validate_resource_bounds` reads the previous block's STRK L2 gas price and enforces that the transaction's `max_price_per_unit` for L2 gas meets a percentage threshold of that price:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold is then computed as:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(StarknetError { code: "GAS_PRICE_TOO_LOW", ... });
}
``` [2](#0-1) 

The block header stores a separate `next_l2_gas_price` field (the EIP-1559-computed price for the **next** block). The consensus orchestrator computes this via `calculate_next_l2_gas_price_for_fin` and writes it into the fin/block header: [3](#0-2) 

The batcher, when it starts building the next block, uses this EIP-1559-computed `next_l2_gas_price` as the actual block gas price against which `check_fee_bounds` is enforced: [4](#0-3) 

The gateway's own blockifier validation (`run_validate_entry_point`) also uses the previous block's gas prices (only the block number is incremented), so the gateway is internally consistent — but it is inconsistent with the batcher's actual execution context: [5](#0-4) 

The default `min_gas_price_percentage` is 100, meaning the threshold equals the previous block's price exactly: [6](#0-5) 

---

### Impact Explanation

**Case 1 — Valid transaction rejected (price decreasing):**  
When the previous block was underutilized, EIP-1559 computes `next_l2_gas_price < previous_block_l2_gas_price`. A user who correctly sets `max_price_per_unit = next_l2_gas_price` submits a transaction that would pass `check_fee_bounds` in the batcher. The gateway rejects it with `GAS_PRICE_TOO_LOW` because `next_l2_gas_price < 100% × previous_block_l2_gas_price`. The user's transaction is valid for the next block but is permanently blocked at the gateway.

**Case 2 — Invalid transaction admitted (price increasing):**  
When the previous block was overutilized, EIP-1559 computes `next_l2_gas_price > previous_block_l2_gas_price`. A transaction with `max_price_per_unit = previous_block_l2_gas_price` passes the gateway check (`previous_block_price >= 100% × previous_block_price`) but fails the batcher's `check_fee_bounds` with `MaxGasPriceTooLow` because the actual execution price is higher. The gateway has admitted a transaction that cannot be sequenced.

Both outcomes match the **High** impact scope: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price every block based on gas usage relative to the target. Any block that is not exactly at the gas target produces a `next_l2_gas_price` that differs from the current block's price. This is the normal operating condition of the fee market, so the mismatch occurs continuously. The developer TODO comment confirms the issue is known and unresolved.

---

### Recommendation

Replace the use of `previous_block_l2_gas_price` with the `next_l2_gas_price` stored in the block header, as the TODO comment already prescribes:

```rust
// Use next_l2_gas_price from the block header instead of the current block's price.
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .next_l2_gas_price; // read the EIP-1559-computed next price
```

This aligns the gateway admission check with the price that will actually be enforced during batcher execution, eliminating both the false-rejection and false-admission cases.

---

### Proof of Concept

**Scenario A — Valid transaction rejected:**

1. Block N has `l2_gas_price = 1000 fri` and was underutilized (gas used < gas target).
2. EIP-1559 computes `next_l2_gas_price = 990 fri` for block N+1.
3. User submits an `AllResources` V3 invoke with `l2_gas.max_price_per_unit = 990 fri`.
4. Gateway calls `validate_tx_l2_gas_price_within_threshold(990, 1000)`.
5. `threshold = (100/100) * 1000 = 1000`. Since `990 < 1000`, the gateway returns `GAS_PRICE_TOO_LOW`.
6. The transaction is rejected, even though the batcher would have accepted it at 990 fri.

**Scenario B — Invalid transaction admitted:**

1. Block N has `l2_gas_price = 1000 fri` and was overutilized (gas used > gas target).
2. EIP-1559 computes `next_l2_gas_price = 1010 fri` for block N+1.
3. User submits an `AllResources` V3 invoke with `l2_gas.max_price_per_unit = 1000 fri`.
4. Gateway calls `validate_tx_l2_gas_price_within_threshold(1000, 1000)`.
5. `threshold = 1000`. Since `1000 >= 1000`, the gateway admits the transaction.
6. The batcher's `check_fee_bounds` uses `next_l2_gas_price = 1010 fri`. Since `1000 < 1010`, the transaction fails with `MaxGasPriceTooLow` and is dropped.

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
