### Title
Unvalidated Caller-Controlled `disable_rewards` Parameter in `update_rewards` Allows Any Address to Suppress Block Reward Distribution - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable by any non-zero address and accepts a caller-controlled `disable_rewards: bool` parameter. When set to `true`, this parameter causes the function to consume the protocol's single per-block reward slot (by writing to `last_reward_block`) without distributing any rewards. An attacker can exploit this to suppress block reward distribution for all stakers by front-running legitimate calls in every block.

---

### Finding Description

The `update_rewards` function is exposed via the `IStakingRewardsManager` interface with no access control beyond `general_prerequisites()`, which only checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

The implementation accepts a `disable_rewards: bool` parameter. When `true`, the function returns early after writing `current_block_number` to `last_reward_block` — without distributing any rewards: [2](#0-1) 

The `last_reward_block` storage variable acts as a global per-block gate. The assertion `current_block_number > self.last_reward_block.read()` ensures only one successful call per block: [3](#0-2) 

Once an attacker calls `update_rewards(any_valid_staker, true)` in a block, the slot is consumed and no legitimate reward distribution can occur in that block. The `disable_rewards` parameter is analogous to the unsanitized token metadata in the original report: it is attacker-controlled input that passes through the function without validation, causing the protocol to silently skip its core operation. [4](#0-3) 

---

### Impact Explanation

An attacker calling `update_rewards(staker_address, true)` in every block permanently suppresses block reward distribution for all stakers. Each block's rewards are irrecoverably lost because the per-block gate is consumed without distributing rewards. This constitutes griefing with no profit motive but direct, measurable damage to all stakers and delegators in the protocol.

**Matched impact**: Griefing with no profit motive but damage to users or protocol (Medium).

---

### Likelihood Explanation

The attack is realistic and low-cost:
1. Any valid active staker with non-zero balance can be identified via public view functions (`get_stakers`, `staker_info_v3`).
2. The attacker submits `update_rewards(staker_address, true)` in every block before the legitimate caller.
3. Gas cost per call is low relative to the damage caused (all stakers lose block rewards).
4. The attack can be fully automated with a bot and sustained indefinitely.

No privileged role, leaked key, or external dependency is required.

---

### Recommendation

Add access control to `update_rewards` to restrict callers to a designated role (e.g., the attestation contract or a permissioned rewards distributor). Alternatively, remove the `disable_rewards` parameter entirely and handle the pre-consensus case internally by checking `self.is_pre_consensus()` without exposing it as a caller-controlled input: [5](#0-4) 

---

### Proof of Concept

1. Attacker identifies any valid active staker `S` with non-zero balance (publicly queryable).
2. In each block, attacker calls `update_rewards(S, disable_rewards: true)` before the legitimate caller.
3. The function writes `current_block_number` to `last_reward_block` and returns early without distributing rewards.
4. Any subsequent call to `update_rewards` in the same block fails with `REWARDS_ALREADY_UPDATED`.
5. Rewards for that block are permanently lost for all stakers.
6. Attacker repeats in every block to suppress all block rewards indefinitely. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
