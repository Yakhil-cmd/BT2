### Title
Unprivileged Caller Can Front-Run `update_rewards` with `disable_rewards: true` to Grief Staker Reward Collection - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is part of a public ABI (`IStakingRewardsManager`) and is callable by any address. It accepts an attacker-controlled `disable_rewards` boolean parameter. Because `last_reward_block` is written to storage **before** the `disable_rewards` guard is checked, an attacker can front-run a staker's legitimate `update_rewards(staker_A, false)` call with `update_rewards(staker_A, true)`, consuming the per-block slot without distributing any rewards.
