### Title
Permissionless `update_rewards` with `disable_rewards: true` Permanently Freezes All Staker Consensus Rewards — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is a permissionless external function that accepts a caller-controlled `disable_rewards: bool` parameter. The function unconditionally writes the current block number to the global `last_reward_block` storage variable **before** checking `disable_rewards`. Because `last_reward_block` is a single global gate that allows only one reward-distribution call per block, any unprivileged caller can invoke `update_rewards(valid_staker, disable_rewards: true)` every block to permanently prevent all stakers from ever receiving consensus-phase rewards.

---

### Finding Description

`update_rewards` is declared in `IStakingRewardsManager` and embedded with `#[abi(embed_v0)]`, making it a fully public external entry point with no role-based access control. [1](#0-0) 

The function enforces a single-call-per-block invariant via a global `last_reward_block` variable: [2](#0-1) 

Critically, `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

Because `last_reward_block` is global (not per-staker), once it is set to block `N` by any caller, every subsequent call to `update_rewards` in block `N` reverts with `REWARDS_ALREADY_UPDATED`. The attacker therefore needs only one transaction per block to consume the entire block's reward-distribution slot while distributing nothing.

The `disable_rewards` path is structurally identical to the pre-consensus early-return path and carries no access restriction: [4](#0-3) 

The only prerequisite the attacker must satisfy is supplying a valid, active staker address with non-zero balance — information that is publicly observable from the `stakers` vector and on-chain events. [5](#0-4) 

---

### Impact Explanation

This is a **permanent freezing of unclaimed yield** for all stakers in the consensus-rewards phase. By calling `update_rewards(any_valid_staker, disable_rewards: true)` on every block, an attacker ensures that `last_reward_block` is always equal to the current block, so no legitimate call can ever distribute rewards. Stakers and delegators accumulate zero `unclaimed_rewards_own` and zero pool rewards indefinitely. The attack is not self-limiting: it persists as long as the attacker continues to submit one transaction per block.

This matches the allowed High impact: **Permanent freezing of unclaimed yield or unclaimed royalties**.

---

### Likelihood Explanation

- The entry point is fully permissionless — no role, no signature, no stake required.
- The only cost is one Starknet transaction per block (~2–3 seconds), which is economically feasible for a motivated attacker.
- The required input (a valid active staker address) is trivially obtained from public chain state.
- The attack is analogous to the M-8 pattern: a permissionless function that updates a global "high-water mark" (`last_reward_block` here, `navPerShareHighMark` in M-8) before the reward-distribution logic, locking out all future distributions.

---

### Recommendation

1. **Remove `disable_rewards` from the public interface**, or gate it behind a privileged role (e.g., `only_app_governor`).
2. Alternatively, move the `self.last_reward_block.write(current_block_number)` call to **after** the `disable_rewards` guard, so that a no-op call does not consume the block's reward slot.
3. Consider making `last_reward_block` per-staker rather than global, so that one call cannot block all other stakers.

---

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
