### Title
Pool Members' Funds Temporarily Frozen When Associated Staker Has Not Migrated to V3 — (`src/staking/staking.cairo`)

---

### Summary

`remove_from_delegation_pool_intent` in `staking.cairo` calls `internal_staker_info`, which enforces a V3 migration check (`STAKER_NOT_MIGRATED`). Any pool member delegating to a staker that has not yet migrated to V3 is blocked from calling `exit_delegation_pool_intent`, temporarily freezing their delegated funds.

---

### Finding Description

The withdrawal path for a pool member is:

```
pool member → exit_delegation_pool_intent (pool.cairo:256)
           → undelegate_from_staking_contract_intent (pool.cairo:637–663)
           → staking_pool_dispatcher.remove_from_delegation_pool_intent (staking.cairo:1048)
           → self.internal_staker_info(:staker_address) (staking.cairo:1056)
           → PANICS: STAKER_NOT_MIGRATED
```

`internal_staker_info` (the public migration-aware version) enforces:

```cairo
// staking.cairo:942–953
fn internal_staker_info(
    self: @ContractState, staker_address: ContractAddress,
) -> InternalStakerInfoLatest {
    let internal_staker_info = self._internal_staker_info(:staker_address);
    // Assert staker already migrated to V3.
    assert!(
        self.staker_version.read(staker_address).is_latest(),
        "{}",
        Error::STAKER_NOT_MIGRATED,
    );
    internal_staker_info
}
``` [1](#0-0) 

`remove_from_delegation_pool_intent` calls this version:

```cairo
// staking.cairo:1054–1056
self.general_prerequisites();
let staker_info = self.internal_staker_info(:staker_address);
``` [2](#0-1) 

The `staker_info` value is only used downstream in `update_delegated_stake` to check `staker_info.unstake_time.is_none()` — a field present in all versions. The V3 migration check is therefore not required for the function's logic, yet it gates the entire pool-member exit path. [3](#0-2) 

The private `_internal_staker_info` (used only inside `staker_migration`) handles V1-format staker info without the migration assertion:

```cairo
// staking.cairo:1527–1540
fn _internal_staker_info(...) -> InternalStakerInfoLatest {
    match versioned_internal_staker_info {
        VInternalStakerInfo::V1(internal_staker_info_v1) => internal_staker_info_v1,
        ...
    }
}
``` [4](#0-3) 

`undelegate_from_staking_contract_intent` in the pool only bypasses the staking contract call when `staker_removed == true`. While the staker is still active (not yet removed), it always routes through `remove_from_delegation_pool_intent`:

```cairo
// pool.cairo:640–663
if !self.is_staker_active() {
    ...
    return Time::now();
}
...
staking_pool_dispatcher.remove_from_delegation_pool_intent(...)
``` [5](#0-4) 

---

### Impact Explanation

Pool members who delegated to a staker that has not yet called (or had called on their behalf) `staker_migration` cannot initiate a withdrawal via `exit_delegation_pool_intent`. Their delegated STRK (or BTC) tokens remain locked in the staking contract for the duration of the freeze. This matches **High: Temporary freezing of funds** in the allowed impact scope.

---

### Likelihood Explanation

During the V2→V3 upgrade window, every staker that has not yet been migrated blocks all of its pool members from exiting. `staker_migration` is permissionless — any address can call it for any staker — so pool members can self-unblock, but only if `pool_eic_class_hash` has been set by governance. Until that hash is configured, or until the staker migrates, the freeze is absolute. The scenario is realistic for any pool member who attempts to exit during the upgrade period before their staker's migration is complete. [6](#0-5) 

---

### Recommendation

Replace `internal_staker_info` with `_internal_staker_info` inside `remove_from_delegation_pool_intent`. The migration check is not needed there — only `unstake_time` is consumed from the returned struct, and that field is available in all staker info versions. This mirrors the fix in the external report: exclude the blocking check from the withdrawal path.

```cairo
// staking.cairo:1056 — change:
let staker_info = self.internal_staker_info(:staker_address);
// to:
let staker_info = self._internal_staker_info(:staker_address);
``` [7](#0-6) 

---

### Proof of Concept

1. Staker stakes before the V3 upgrade; `staker_version` is unset (not `LATEST_STAKER_VERSION`).
2. Pool member calls `enter_delegation_pool` and deposits STRK.
3. Pool member calls `exit_delegation_pool_intent(amount)`.
4. Pool contract calls `undelegate_from_staking_contract_intent`; staker is still active so it calls `remove_from_delegation_pool_intent` on the staking contract.
5. `remove_from_delegation_pool_intent` calls `internal_staker_info(staker_address)`.
6. `internal_staker_info` reads `staker_version` → not latest → panics with `STAKER_NOT_MIGRATED`.
7. Pool member's funds remain locked; they cannot exit until `staker_migration` is successfully executed for their staker. [8](#0-7) [9](#0-8)

        fn remove_from_delegation_pool_intent(
            ref self: ContractState,
            staker_address: ContractAddress,
            identifier: felt252,
            amount: Amount,
        ) -> Timestamp {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let staker_info = self.internal_staker_info(:staker_address);
            let pool_contract = get_caller_address();
            let token_address = self
                .staker_pool_info
                .entry(staker_address)
                .get_pool_token(:pool_contract)
                .expect_with_err(Error::CALLER_IS_NOT_POOL_CONTRACT);
            let decimals = self.get_token_decimals(:token_address);
            let normalized_amount = NormalizedAmountTrait::from_native_amount(:amount, :decimals);

            // Update the delegated stake according to the new intent.
            let undelegate_intent_key = UndelegateIntentKey { pool_contract, identifier };
            let old_intent_amount = self.get_pool_exit_intent(:undelegate_intent_key).amount;
            let new_intent_amount = normalized_amount;
            // After this call, the staker balance will be updated.
            let (old_delegated_stake, new_delegated_stake) = self
                .update_delegated_stake(
                    :staker_address,
                    :token_address,
                    :pool_contract,
                    :staker_info,
                    :old_intent_amount,
                    :new_intent_amount,
                );
            self
                .update_undelegate_intent_value(
                    :token_address, :staker_info, :undelegate_intent_key, :new_intent_amount,
                );

            self
                .emit(
                    Events::RemoveFromDelegationPoolIntent {
                        staker_address,
                        pool_contract,
                        token_address,
                        identifier,
                        old_intent_amount: old_intent_amount.to_native_amount(:decimals),
                        new_intent_amount: amount,
                    },
                );
            // If the staker is in the process of unstaking (intent called),
            // an event indicating the staked amount (own and delegated) to be zero
            // had already been emitted, thus unneeded now.
            if staker_info.unstake_time.is_none() {
                self
                    .emit(
                        Events::StakeDelegatedBalanceChanged {
                            staker_address,
                            token_address,
                            old_delegated_stake: old_delegated_stake.to_native_amount(:decimals),
                            new_delegated_stake: new_delegated_stake.to_native_amount(:decimals),
                        },
                    );
            }
            self.get_pool_exit_intent(:undelegate_intent_key).unpool_time
        }

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }
