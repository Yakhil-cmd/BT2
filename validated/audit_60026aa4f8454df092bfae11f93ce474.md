### Title
Follow-on transactions (nonce=N+1) are TTL-expired while their staged predecessor (nonce=N) is preserved, creating a sequencing gap — (`crates/apollo_mempool/src/transaction_pool.rs`)

### Summary
`TimedTransactionMap::remove_txs_older_than` uses the predicate `tx.nonce < staged_nonce` to protect staged transactions from TTL expiry. Because `staged_nonce = N+1` for a tx staged at nonce=N, only nonces strictly less than N+1 (i.e., ≤ N) are protected. A follow-on transaction at nonce=N+1 satisfies `N+1 < N+1 = false` and is therefore expired and removed from the pool, while the staged tx at nonce=N remains.

### Finding Description
When `get_txs` stages a transaction at nonce=N, `MempoolState::stage` records `staged[address] = N+1` (the next expected nonce). [1](#0-0) 

`remove_expired_txs` then calls `tx_pool.remove_txs_older_than(ttl, &self.state.staged)`, forwarding the staged map as the exclusion set. [2](#0-1) 

Inside `TimedTransactionMap::remove_txs_older_than`, the guard condition is:
```rust
exclude_txs.get(&tx.address).is_some_and(|nonce| tx.nonce < *nonce)
``` [3](#0-2) 

This protects only transactions whose nonce is **strictly less than** the staged nonce (N+1), i.e., nonce ≤ N. Any follow-on transaction at nonce=N+1 fails the guard (`N+1 < N+1` is false) and is unconditionally expired and removed from the pool. [4](#0-3) 

The staged tx at nonce=N is a soft-delete (returned by `get_txs` but not removed from the pool), so it remains in `txs_by_submission_time` and is protected. Its follow-on at nonce=N+1 is not. [5](#0-4) 

### Impact Explanation
After the staged tx at nonce=N commits, the account's on-chain nonce advances to N+1. The pool no longer contains the nonce=N+1 transaction (it was TTL-expired). The account's subsequent transactions are blocked from sequencing until the user resubmits. This is a valid transaction being rejected before sequencing, matching the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation
The scenario requires:
1. An account submits nonce=N and nonce=N+1 transactions.
2. The sequencer stages nonce=N via `get_txs`.
3. Block production stalls or is slow enough that the nonce=N+1 tx ages past TTL before nonce=N commits.

This is a realistic condition under network congestion, slow block production, or when TTL is short relative to block time. No privileged access is required; any unprivileged user submitting two sequential transactions is affected.

### Recommendation
Change the exclusion predicate to protect all follow-on transactions for an account that has any staged transaction. The simplest fix is to change the condition from `tx.nonce < *nonce` to `tx.nonce <= *nonce` — but this only protects nonce=N+1. To protect the full chain (nonce=N+1, N+2, …), the predicate should be `exclude_txs.contains_key(&tx.address)` (i.e., preserve **all** transactions for any account that has a staged tx), or alternatively extend the staged map to record the lowest staged nonce rather than the next expected nonce.

    pub fn remove_txs_older_than(
        &mut self,
        duration: Duration,
        exclude_txs: &HashMap<ContractAddress, Nonce>,
    ) -> Vec<TransactionReference> {
        let split_off_value = SubmissionID {
            submission_time: self.clock.now() - duration,
            tx_hash: Default::default(),
            batching_time: None,
        };
        let old_txs = self.txs_by_submission_time.split_off(&split_off_value);

        let mut removed_txs = Vec::new();
        for (submission_id, tx) in old_txs.into_iter() {
            if exclude_txs.get(&tx.address).is_some_and(|nonce| tx.nonce < *nonce) {
                // The transaction should be preserved. Add it back.
                self.txs_by_submission_time.insert(submission_id, tx);
            } else {
                self.hash_to_submission_id.remove(&tx.tx_hash).expect(
                    "Transaction should have a submission ID if it is in the timed transaction \
                     map.",
                );
                removed_txs.push(tx);
            }
        }

        removed_txs
    }
