### Title
Unrestricted `update_rewards` with `disable_rewards: true` Consumes the Per-Block Reward Slot Without Distributing Rewards, Enabling Griefing of All Stakers - (File: src/staking/staking.cairo)

### Summary
`update_rewards` is publicly callable by any address and accepts a caller-controlled `disable_rewards` flag. The function writes the global `last_reward_block` sentinel **before** checking `disable_rewards`, so calling it with `disable_rewards: true` permanently consumes the current block's reward slot without distributing any yield. Because the sentinel is global (one slot per block for the entire protocol), an attacker can call this once per block for any valid staker to ensure no staker ever receives consensus rewards.

### Finding Description
`StakingRewardsManagerImpl::update_rewards` enforces a single-reward-per-block invariant through the global `last_reward_block` storage variable:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [1](#0-0) 

After passing all staker-validity checks, the function unconditionally writes the current block number to `last_reward_block`, then immediately returns if `disable_rewards` is `true`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

The function has no role guard beyond `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero: [3](#0-2) 

`last_reward_block` is a single protocol-wide slot, not per-staker: [4](#0-3) 

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block:
1. Passes all validity checks (staker must exist, be active, and have non-zero STRK balance).
2. Writes `last_reward_block = current_block`.
3. Returns without distributing any rewards.
4. Causes every subsequent legitimate call to `update_rewards` in that block to revert with `REWARDS_ALREADY_UPDATED`.

The analog to the original report is exact: just as `carryVoteForward()` performed the voting action without setting `voted[_tokenId] = true`, `update_rewards` here performs the block-consumption action (writing `last_reward_block`) without completing the reward-distribution action, leaving the protocol in a state where the block is "used" but no yield was paid out.

### Impact Explanation
Every consensus-epoch block that the attacker front-runs results in zero rewards for all stakers. If sustained across multiple blocks or epochs, this constitutes a continuous, attacker-controlled suppression of unclaimed yield for every staker in the protocol. The attacker has no profit motive; the sole effect is damage to stakers and the protocol's reward mechanism. This matches **Medium: Griefing with no profit motive but damage to users or protocol**, and can escalate to **High: Permanent freezing of unclaimed yield** if the attacker sustains the attack across epochs.

### Likelihood Explanation
The entry path requires no privilege: any non-zero address can call `update_rewards` on any valid, active staker. The only cost is gas per block. The attack is trivially scriptable and requires no special knowledge beyond knowing a valid staker address (which is public on-chain).

### Recommendation
Move `last_reward_block.write` to **after** the `disable_rewards` guard so that the block slot is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Only consume the block slot when rewards are actually distributed.
self.last_reward_block.write(current_block_number);

// ... rest of reward distribution logic
```

Alternatively, restrict `update_rewards` to authorized callers (e.g., only the staker themselves or a designated consensus operator role).

### Proof of Concept
1. Attacker identifies any valid, active staker address `S` with non-zero STRK balance.
2. At the start of each new block `N`, attacker calls `update_rewards(S, disable_rewards: true)`.
3. `last_reward_block` is set to `N`; no rewards are distributed.
4. The legitimate staker (or any caller) attempts `update_rewards(S, disable_rewards: false)` in block `N` — it reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block, all stakers are permanently denied consensus rewards for as long as the attacker continues. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
