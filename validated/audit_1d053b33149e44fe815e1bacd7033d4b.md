### Title
Unbounded `stakers` Vec Growth Enables Permanent O(N) DoS of `get_stakers` Consensus Interface — (`src/staking/staking.cairo`)

---

### Summary

The `stakers` storage Vec never shrinks. Any caller can stake with N addresses (meeting minimum stake), unstake all of them to recover funds, and leave N permanent dead entries. Every subsequent call to `get_stakers` iterates the full Vec, performing at least one storage read per dead entry, with no cap or pagination.

---

### Finding Description

The `stakers` Vec is declared with an explicit note that entries are never removed: [1](#0-0) 

`get_stakers` iterates over the **entire** Vec unconditionally: [2](#0-1) 

For every entry, `is_staker_active` performs at minimum one storage read (`staker_info.read`), and a second read (`staker_unstake_intent_epoch.read`) when `unstake_time` is set: [3](#0-2) 

For entries that pass the active check, `get_staker_staking_power_at_epoch` is called (additional reads per pool). There is no `MAX_STAKERS` constant, no pagination, and no mechanism to prune dead entries.

The `get_stakers` interface is open to any caller: [4](#0-3) 

---

### Impact Explanation

After the attack, every call to `get_stakers` — including calls by consensus nodes to determine the validator set — performs O(N) storage reads. On Starknet, both on-chain calls and off-chain view calls have execution resource limits. A sufficiently large N causes `get_stakers` to exceed those limits, permanently breaking the consensus interface's ability to return the validator set.

---

### Likelihood Explanation

The attacker must hold `min_stake × N` STRK temporarily (recovered after the exit wait window, minus fees). The capital requirement is substantial, making this a griefing attack with a real economic cost. However:
- The damage is **permanent** — dead Vec entries cannot be removed.
- The attacker **recovers their principal** after unstaking.
- The only ongoing cost is transaction fees for N stake/unstake_intent/unstake_action calls.
- No privileged role is required; `stake()` and `unstake_intent()` are permissionless.

---

### Recommendation

1. **Pagination**: Add a `(offset, limit)` parameter to `get_stakers` so callers can paginate.
2. **Compaction**: When `unstake_action` finalizes, swap-and-pop the staker's entry from the `stakers` Vec (or maintain a separate active-staker count).
3. **Cap**: Enforce a `MAX_STAKERS` limit in `stake()` to bound the Vec size.

---

### Proof of Concept

```
1. For i in 0..N:
     stake(staker_i, min_stake)          // appends staker_i to stakers Vec
2. For i in 0..N:
     unstake_intent(staker_i)            // sets unstake_time + staker_unstake_intent_epoch
3. Advance exit_wait_window blocks.
4. For i in 0..N:
     unstake_action(staker_i)            // staker_i's funds returned; staker_info cleared
                                         // BUT staker_i remains in stakers Vec forever
5. get_stakers(current_epoch):
     // iterates all N entries
     // each calls is_staker_active → staker_info.read (1 storage read per entry)
     // total: N storage reads minimum, growing without bound
```

Assert: storage reads in `get_stakers` grow linearly with N. At sufficient N, the call exceeds Starknet execution limits and reverts, permanently DoSing the consensus interface.
