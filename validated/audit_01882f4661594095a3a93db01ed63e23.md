### Title
Gateway Stateful Validator Checks L2 Gas Price Against Current Block Instead of Next Block, Causing Wrong Admission Decisions - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` compares a transaction's `max_price_per_unit` against the **current committed block's** L2 gas price (`block_info.gas_prices.strk_gas_prices.l2_gas_price`) rather than the **next block's** L2 gas price (`next_l2_gas_price` stored in the block header). Because the EIP-1559 mechanism adjusts the L2 gas price every block, the threshold used for admission is systematically wrong, causing the gateway to accept transactions that will fail at blockifier time (when the price is rising) and to reject valid transactions (when the price is falling).

### Finding Description

`validate_resource_bounds` reads the previous block's L2 gas price through `GatewayFixedBlockStateReader::get_block_info()`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:223-243
async fn validate_resource_bounds(...) {
    if self.config.validate_resource_bounds {
        // TODO(Arni): getnext_l2_gas_price from the block header.
        let previous_block_l2_gas_price = self
            .gateway_fixed_block_state_reader
            .get_block_info()
            .await?
            .gas_prices
            .strk_gas_prices
            .l2_gas_price;          // ← current block's price, NOT next block's price
        self.validate_tx_l2_gas_price_within_threshold(
            executable_tx.resource_bounds(),
            previous_block_l2_gas_price,
        )?;
    }
}
```

The `GatewayFixedBlockSyncStateClient` constructs `BlockInfo` from the block header, mapping `l2_gas_price.price_in_fri` into `strk_gas_prices.l2_gas_price`:

```rust
// crates/apollo_gateway/src/gateway_fixed_block_state_reader.rs:36-57
let block_info = BlockInfo {
    gas_prices: GasPrices {
        strk_gas_prices: GasPriceVector {
            l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
            // ↑ This is the CURRENT block's price
            ...
        },
    },
    ...
};
```

However, `BlockHeaderWithoutHash` carries a distinct `next_l2_gas_price: GasPrice` field — the EIP-1559-adjusted price for the **next** block:

```rust
// crates/starknet_api/src/block.rs:231-248
pub struct BlockHeaderWithoutHash {
    pub l2_gas_price: GasPricePerToken,      // current block's price
    pub next_l2_gas_price: GasPrice,          // price for the NEXT block ← correct field
    ...
}
```

The `StorageBlockHeader` also stores both fields separately:

```rust
// crates/apollo_storage/src/header.rs:71-113
pub struct StorageBlockHeader {
    pub l2_gas_price: GasPricePerToken,
    pub next_l2_gas_price: GasPrice,
    ...
}
```

The threshold computation in `validate_tx_l2_gas_price_within_threshold` is arithmetically correct but operates on the wrong input value:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:359-390
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)   // ← wrong price
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...GAS_PRICE_TOO_LOW...);
}
```

The `GatewayFixedBlockStateReader` trait only exposes `get_block_info() -> BlockInfo`, and `BlockInfo` has no `next_l2_gas_price` field, so the correct value is structurally inaccessible through the current interface.

### Impact Explanation

A submitted transaction is destined for the **next** block. The blockifier's `check_fee_bounds` (called during `perform_pre_validation_stage`) compares `max_price_per_unit` against the **next block's actual gas prices**. The gateway's pre-filter should enforce the same price, but it enforces the **current block's** price instead.

**Case 1 — Price rising (next > current):** A transaction with `max_price_per_unit` satisfying `current_price * threshold% ≤ price < next_price * threshold%` passes the gateway check but fails at blockifier execution with `MaxGasPriceTooLow`. The gateway incorrectly admits an invalid transaction.

**Case 2 — Price falling (next < current):** A transaction with `max_price_per_unit` satisfying `next_price * threshold% ≤ price < current_price * threshold%` is rejected by the gateway with `GAS_PRICE_TOO_LOW` even though it would succeed in the next block. The gateway incorrectly rejects a valid transaction.

Both cases match the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The EIP-1559 mechanism (`calculate_next_base_gas_price`) adjusts the L2 gas price every block based on gas usage relative to the target. Any block with gas usage different from the target produces a price change. Given that the sequencer is designed for high throughput, price changes are routine. The bug is triggered by any user submitting a transaction during a period of price movement, which is the normal operating condition. The TODO comment in the source code explicitly acknowledges the wrong field is being used.

### Recommendation

1. Extend `GatewayFixedBlockStateReader` with a method to return `next_l2_gas_price`, or add `next_l2_gas_price: GasPrice` to `BlockInfo`.
2. In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, expose `block_header.next_l2_gas_price`.
3. In `validate_resource_bounds`, use `next_l2_gas_price` as the reference price instead of `strk_gas_prices.l2_gas_price`.

### Proof of Concept

1. Observe the current committed block has L2 gas price `P_current` and `next_l2_gas_price = P_next` where `P_next > P_current` (high-congestion block).
2. Submit an invoke transaction V3 with `AllResources` bounds where `l2_gas.max_price_per_unit = P_current` (exactly equal to the current block price, above the 100% threshold).
3. The gateway's `validate_resource_bounds` computes `threshold = 100% * P_current = P_current` and passes the check (`P_current >= P_current`). Transaction is admitted to the mempool.
4. When the batcher executes the transaction in the next block (which uses `P_next`), `check_fee_bounds` computes `actual_gas_price = P_next > P_current = max_price_per_unit` and returns `MaxGasPriceTooLow`. The transaction reverts.
5. The gateway has admitted a transaction that was guaranteed to fail — an invalid admission decision. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }
