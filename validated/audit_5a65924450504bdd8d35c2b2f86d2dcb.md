### Title
NFT Seller Net Proceeds Displayed Incorrectly in Offer Viewer, Causing Seller to Approve Wrong Amount - (File: packages/gui/src/components/offers/utils.ts)

### Summary

`calculateNFTRoyalties` in `packages/gui/src/components/offers/utils.ts` sets `nftSellerNetAmount = amount` (the full offered price) instead of `amount - royaltyAmount`. The commented-out code reveals the original intent was to subtract royalties. The "Net Proceeds" label in `NFTOfferViewer` and "You will receive" in `NFTOfferEditor` both render this wrong value, causing an NFT seller to see an inflated net proceeds figure when reviewing a buyer-initiated (`TokenForNFT`) offer and potentially accept it under false pretenses.

### Finding Description

In `calculateNFTRoyalties`:

```typescript
const nftSellerNetAmount: number = amount;
// : parseFloat(
//     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
//   );
```

The correct formula (now commented out) was `amount - royaltyAmountString - makerFee`. The current code unconditionally assigns `nftSellerNetAmount = amount`, ignoring royalties entirely. [1](#0-0) 

The tooltip in `NFTOfferViewer` explicitly states the correct semantics: *"The net proceeds include the asking price, minus any associated creator fees."* But the rendered value contradicts this: [2](#0-1) 

The same wrong value is rendered in `NFTOfferEditor` as "You will receive" / "They will receive": [3](#0-2) 

Additionally, `overrideNFTSellerAmount` is derived from the wrong `nftSellerNetAmount` and passed into the offer summary row, propagating the incorrect figure into the purchase summary panel the seller reviews before accepting: [4](#0-3) 

### Impact Explanation

When a buyer creates a `TokenForNFT` offer (e.g., "I offer 10 XCH for NFT X"), the GUI tooltip confirms royalties are paid *from* the offered amount. A 10% royalty on a 10 XCH offer means the seller actually receives 9 XCH. However, the GUI displays "Net Proceeds: 10 XCH" — the full offered amount — because `nftSellerNetAmount` is never reduced by `royaltyAmount`. The seller approves the offer believing they will receive 10 XCH but the on-chain settlement delivers 9 XCH. This is a direct, concrete asset loss caused by the GUI displaying the wrong amount in the offer approval flow.

### Likelihood Explanation

Any unprivileged actor can create a buyer-side (`TokenForNFT`) offer targeting an NFT with royalties. NFTs with royalties are common on Chia. The NFT owner will see the inflated "Net Proceeds" figure every time they open such an offer in the viewer. No special access, leaked keys, or social engineering is required beyond the normal offer-sharing mechanism.

### Recommendation

Restore the correct net-proceeds formula in `calculateNFTRoyalties`:

```typescript
// Before (wrong):
const nftSellerNetAmount: number = amount;

// After (correct):
const nftSellerNetAmount: number =
  exchangeType === NFTOfferExchangeType.TokenForNFT
    ? parseFloat((amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12))
    : amount; // For NFTForToken, seller receives full amount; royalties paid by buyer on top
``` [5](#0-4) 

### Proof of Concept

1. Mint or identify an NFT with a 20% royalty (`royaltyPercentage = 2000`).
2. As an attacker/buyer, create a `TokenForNFT` offer: offer 10 XCH for the NFT.
3. Share the offer file with the NFT owner.
4. NFT owner opens the offer in `NFTOfferViewer`. The GUI displays:
   - **Creator Fee (20%):** 2 XCH
   - **Net Proceeds:** 10 XCH ← **wrong; should be 8 XCH**
5. NFT owner accepts the offer expecting 10 XCH.
6. On-chain settlement: royalty recipient receives 2 XCH, NFT owner receives 8 XCH.
7. NFT owner has lost 2 XCH relative to what the GUI promised. [6](#0-5) [7](#0-6)

export function calculateNFTRoyalties(
  amount: number,
  makerFee: number,
  royaltyPercentage: number,
  exchangeType: NFTOfferExchangeType,
): CalculateNFTRoyaltiesResult {
  const royaltyAmount: number = royaltyPercentage ? (royaltyPercentage / 100) * amount : 0;
  const royaltyAmountString: string = formatAmount(royaltyAmount);
  const nftSellerNetAmount: number = amount;
  // : parseFloat(
  //     (amount - parseFloat(royaltyAmountString) - makerFee).toFixed(12),
  //   );
  const totalAmount: number =
    exchangeType === NFTOfferExchangeType.NFTForToken ? amount + royaltyAmount : amount + makerFee + royaltyAmount;
  const totalAmountString: string = formatAmount(totalAmount);

  return {
    royaltyAmount,
    royaltyAmountString,
    nftSellerNetAmount,
    totalAmount,
    totalAmountString,
  };
}
