### Title
`DepositAllowlistExtension` gates position owner instead of economic actor (payer), allowing non-allowlisted addresses to deposit tokens — (`metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`)

### Summary

`DepositAllowlistExtension.beforeAddLiquidity` silently discards the `sender` argument and checks only `allowedDepositor[pool][owner]`. Because `owner` is the LP-share recipient — not the address providing tokens — any non-allowlisted address can bypass the gate by naming an allowlisted address as `owner`.

### Finding Description

**Call chain (direct path, no periphery needed):**

1. Attacker calls `pool.addLiquidity(owner = allowlisted_addr, ...)` directly.
2. Pool invokes `_beforeAddLiquidity(msg.sender = attacker, owner = allowlisted_addr, ...)`. [1](#0-0) 
3. `ExtensionCalling` encodes and dispatches `beforeAddLiquidity(sender=attacker, owner=allowlisted_addr, ...)` to the extension. [2](#0-1) 
4. Inside the extension, `sender` is the **unnamed, discarded** first parameter. The guard is:
   ```solidity
   if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner])
   ```
   `msg.sender` here is the **pool**, so this resolves to `allowedDepositor[pool][allowlisted_addr]` → **true** → no revert. [3](#0-2) 
5. Attacker provides tokens in the `metricOmmModifyLiquidityCallback`; `allowlisted_addr` receives LP shares.

**Same bypass via `MetricOmmPoolLiquidityAdder`:**

`addLiquidityExactShares(pool, owner=allowlisted_addr, ...)` stores `msg.sender` (attacker) as the payer in transient context and calls `pool.addLiquidity(owner=allowlisted_addr, ...)`. The extension still only sees `owner`, passes, and tokens are pulled from the attacker. [4](#0-3) 

The contract's own NatDoc states *"Gates `addLiquidity` by depositor address, per pool"* — the depositor is the entity providing tokens, not the position recipient. The implementation contradicts this invariant. [5](#0-4) 

### Impact Explanation

The deposit curation policy configured by the pool admin is fully bypassed. Any non-allowlisted address can inject tokens into the pool and credit LP shares to an allowlisted address. Pools using this extension for regulatory compliance (KYC/AML gating) or economic curation receive liquidity from unintended, ungated sources. This is an admin-boundary break: an unprivileged path circumvents a pool-admin-configured access control.

### Likelihood Explanation

The bypass requires only a public call to `pool.addLiquidity` or `MetricOmmPoolLiquidityAdder.addLiquidityExactShares` with `owner` set to any allowlisted address. No special role, flash loan, or oracle manipulation is needed. Any attacker who knows one allowlisted address can execute it.

### Recommendation

The extension must check the actual economic actor. For direct pool calls, `sender` (the first parameter, currently unnamed and discarded) is the correct address to validate:

```solidity
function beforeAddLiquidity(address sender, address owner, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
    return IMetricOmmExtensions.beforeAddLiquidity.selector;
}
```

For periphery flows where `sender` is the `MetricOmmPoolLiquidityAdder` contract rather than the end user, the adder itself must be allowlisted and the per-user gate must be enforced inside the adder before calling the pool, or the actual payer must be forwarded through `extensionData` and decoded by the extension.

### Proof of Concept

```solidity
// Setup: pool admin allowlists `alice`, not `bob`
extension.setAllowedToDeposit(address(pool), alice, true);

// Bob (non-allowlisted) deposits tokens, alice receives LP shares
vm.prank(bob);

    alice,          // owner = allowlisted address
    salt,
    deltas,
    callbackData,   // bob provides tokens here

// Assert: alice has LP shares, bob's tokens were pulled
uint256 aliceShares = stateView.positionBinShares(address(pool), alice, salt, binIdx);
assertGt(aliceShares, 0);  // passes — allowlist bypassed
```
