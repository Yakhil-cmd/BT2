### Title
Staker Can Grind Stake Amount to Manipulate Target Attestation Block - (File: src/attestation/attestation.cairo)

### Summary
`_calculate_target_attestation_block` uses the staker's current-epoch STRK stake as a manipulable input to the Poseidon hash that determines the target attestation block. Because stake changes take effect K epochs later, a staker can grind their stake amount K epochs in advance to select a favorable attestation window, undermining the liveness proof mechanism and claiming rewards without providing the intended unpredictable liveness.

### Finding Description
In `_calculate_target_attestation_block` (`src/attestation/attestation.cairo`, lines 221–239), the target attestation block is computed as:

```
hash = Poseidon(stake, epoch_id, staker_address)
block_offset = hash % (epoch_len - attestation_window)
target_block = epoch_start + block_offset
``` [1](#0-0) 

The `stake` field is the staker's total STRK balance at the current epoch, fetched live in `get_attestation_info_by_operational_address` (`src/staking/staking.cairo`, lines 1436–1440): [2](#0-1) 

This value is populated into `AttestationInfo.stake` and passed directly into the hash: [3](#0-2) 

Because stake changes take effect K epochs later, a staker can call `increase_stake` at epoch `N-K` with varying `delta` values to iterate over possible Poseidon hash outputs and find a `stake` value that maps to a favorable `block_offset` at epoch `N`. The staker's address and epoch ID are both fixed and non-manipulable, making `stake` the sole grindable input. [4](#0-3) 

### Impact Explanation
By grinding to a `block_offset` near zero, the staker places their target block at the very start of the epoch. They can then attest at block `epoch_start + MIN_ATTESTATION_WINDOW` (the earliest allowed block in the window) and go offline for the rest of the epoch. They still receive full attestation rewards. This is **theft of unclaimed yield**: rewards are intended for validators who are genuinely live at unpredictable times; a grinding staker collects those rewards while providing near-zero liveness beyond a single chosen window.

### Likelihood Explanation
Any registered staker can execute this attack with no privileged access. The only cost is gas for multiple `increase_stake` calls K epochs in advance. The Poseidon hash output is uniformly distributed over `epoch_len - attestation_window` possible offsets, so on average `O(epoch_len / attestation_window)` attempts are needed to land a target block in the first `attestation_window` blocks of the epoch — computationally trivial off-chain. The attack is repeatable every epoch.

### Recommendation
Remove `stake` from the hash inputs in `_calculate_target_attestation_block`. Use only `epoch_id` and `staker_address` (both fixed and non-manipulable by the staker within the epoch), or introduce a protocol-level entropy source (e.g., a committed epoch seed or VRF output) that cannot be influenced by any individual staker.

### Proof of Concept
1. Staker S is registered with stake `X` at epoch `N-K`.
2. S computes off-chain: `target = Poseidon(X, N, S) % (epoch_len - window)`. Result is late in the epoch.
3. S calls `increase_stake(S, delta_1)` at epoch `N-K`. New stake = `X + delta_1`.
4. S recomputes: `target = Poseidon(X + delta_1, N, S) % (epoch_len - window)`. Still late.
5. S iterates `delta_2, delta_3, ...` until `Poseidon(X + delta_i, N, S) % (epoch_len - window)` falls within the first `window` blocks of the epoch.
6. At epoch `N`, S attests at block `epoch_start + target + MIN_ATTESTATION_WINDOW` (the earliest allowed block in the chosen window) and triggers `update_rewards_from_attestation_contract`, claiming full epoch rewards.
7. S goes offline for the remainder of the epoch. [5](#0-4)

        fn _calculate_target_attestation_block(
            self: @ContractState, staking_attestation_info: StakingAttestationInfo,
        ) -> BlockNumber {
            // Compute staker hash for the attestation.
            let hash = PoseidonTrait::new()
                .update(staking_attestation_info.stake().into())
                .update(staking_attestation_info.epoch_id().into())
                .update(staking_attestation_info.staker_address().into())
                .finalize();
            // Calculate staker's block number in this epoch.
            let attestation_window = self.attestation_window.read();
            let block_offset: u256 = hash
                .into() % (staking_attestation_info.epoch_len() - attestation_window.into())
                .into();
            // Calculate actual block number for attestation.
            let target_attestation_block = staking_attestation_info.current_epoch_starting_block()
                + block_offset.try_into().unwrap();
