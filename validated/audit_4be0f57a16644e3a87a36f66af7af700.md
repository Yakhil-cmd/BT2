### Title
Floor Truncation in `validate_tx_l2_gas_price_within_threshold` Allows Below-Minimum Gas Price Transactions Through Gateway Admission - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary
The gateway's stateful L2 gas price admission check computes its threshold using `Ratio::to_integer()`, which performs floor (truncating) integer division. When `min_gas_price_percentage × previous_block_l2_gas_price` is not evenly divisible by 100, the threshold is rounded **down**, allowing transactions whose gas price is strictly below the intended minimum percentage to pass admission.

### Finding Description
In `validate_tx_l2_gas_price_within_threshold`:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...); // reject
}
``` [1](#0-0) 

`Ratio::new(percentage, 100)` creates the rational `percentage/100`. Multiplying by `previous_block_l2_gas_price.get().0` (a `u128`) yields `(percentage × price) / 100` as a `Ratio<u128>`. Calling `.to_integer()` on a `num_rational::Ratio` returns the **floor** of the rational value — i.e., it truncates toward zero.

When `percentage × price` is not divisible by 100, the computed `threshold` is `⌊percentage × price / 100⌋`, which is strictly less than the true fractional threshold `percentage × price / 100`.

**Concrete example:**
- `previous_block_l2_gas_price = 101`
- `min_gas_price_percentage = 50`
- True threshold = `50 × 101 / 100 = 50.5`
- Computed threshold = `⌊50.5⌋ = 50`
- A transaction with `tx_l2_gas_price = 50` satisfies `50 < 50 → false`, so it **passes**
- But `50 < 50.5`, meaning the transaction's gas price is below 50% of the previous block price

The existing test suite only exercises cases where `percentage × price` is exactly divisible by 100 (e.g., `price=100, percentage=50 → threshold=50`), so this truncation is never exercised in tests. [2](#0-1) 

The `min_gas_price_percentage` field is a `u8` configured as a percentage: [3](#0-2) 

The default is 100 (no truncation since `100/100 = 1`), but any other value (50, 75, 80, etc.) combined with a gas price not divisible by 100 triggers the bug.

### Impact Explanation
The invariant broken is: *"transactions with an L2 gas price below `min_gas_price_percentage%` of the previous block's L2 gas price must be rejected at the gateway."* Due to floor truncation, transactions with a gas price up to 1 unit below the intended threshold pass the stateful admission check and enter the mempool. This is a **High** impact: **Mempool/gateway admission accepts invalid transactions before sequencing**.

### Likelihood Explanation
The default `min_gas_price_percentage = 100` is immune (ratio is exactly 1). However, any non-100 configured value combined with a gas price not divisible by 100 triggers truncation. Since L2 gas prices are large integers (e.g., `30_000_000_000` FRI) and the percentage is arbitrary, the product `percentage × price` is almost never divisible by 100 for non-trivial percentages. Any unprivileged user can craft a transaction with `tx_l2_gas_price = ⌊percentage × price / 100⌋` to exploit this.

### Recommendation
Replace `.to_integer()` with ceiling division. `num_rational::Ratio` provides a `.ceil()` method that rounds up:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This ensures the threshold is `⌈percentage × price / 100⌉`, making the admission check strictly enforce the intended minimum.

### Proof of Concept
1. Configure gateway with `min_gas_price_percentage = 50`.
2. Previous block has `l2_gas_price = 101`.
3. Gateway computes `threshold = ⌊50 × 101 / 100⌋ = ⌊50.5⌋ = 50`.
4. Submit a transaction with `tx_l2_gas_price = 50`.
5. Check: `50 < 50` → `false` → transaction **admitted**.
6. Actual ratio: `50 / 101 ≈ 49.5%` — below the configured 50% minimum.
7. The transaction enters the mempool despite violating the gas price floor policy.
