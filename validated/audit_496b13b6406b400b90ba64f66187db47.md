### Title
Permissionless `update_rewards` with `disable_rewards=true` Allows Any Caller to Permanently Deny Block Rewards to All Stakers — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is public and permissionless. It accepts a caller-controlled `disable_rewards: bool` parameter and updates a **global** `last_reward_block` storage slot regardless of whether rewards are actually distributed. An unprivileged attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to consume the single allowed reward-update slot for that block without distributing any rewards, permanently denying block rewards to every staker for that block.

---

### Finding Description

`update_rewards` is exposed via `IStakingRewardsManager` with no role-based access control. Its only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. [1](#0-0) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
``` [2](#0-1) 

The function then unconditionally writes the current block number to the **global** `last_reward_block` field before checking `disable_rewards`: [3](#0-2) 

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

`last_reward_block` is a single global slot, not per-staker: [4](#0-3) 

Because the guard `current_block_number > self.last_reward_block.read()` is checked first and the write happens before the `disable_rewards` branch, any subsequent call in the same block — including a legitimate call with `disable_rewards: false` — will revert with `REWARDS_ALREADY_UPDATED`. The rewards for that block are permanently lost; there is no catch-up mechanism.

---

### Impact Explanation

**High — Permanent freezing / theft of unclaimed yield.**

In the consensus rewards phase (after `consensus_rewards_first_epoch`), block rewards are the sole mechanism for distributing yield to stakers and their delegators. By calling `update_rewards(any_valid_staker, true)` once per block, an attacker permanently voids every block's reward distribution for all stakers. Each skipped block's rewards are irrecoverable — the `RewardSupplier` is never notified, `unclaimed_rewards_own` is never incremented, and pool contracts never receive their share. [5](#0-4) 

---

### Likelihood Explanation

**High.** The attack requires no special role, no capital, and no prior relationship with any staker. The attacker only needs to submit one transaction per block on Starknet (low fee). The attack is economically rational for a competing validator or any party wishing to harm the protocol. The only precondition is that consensus rewards are active (`is_pre_consensus()` returns `false`). [6](#0-5) 

---

### Recommendation

Restrict `update_rewards` to a trusted caller — either the consensus/sequencer infrastructure, a designated operator role, or the staker's own operational address. The `disable_rewards` flag should not be freely settable by an arbitrary caller. A minimal fix is to add a role check (e.g., `self.roles.only_operator()`) or to restrict the caller to the staker's registered operational address, mirroring the access control already applied to `update_rewards_from_attestation_contract`: [7](#0-6) 

---

### Proof of Concept

```
// Precondition: consensus rewards are active (consensus_rewards_first_epoch has passed).
// attacker = any non-zero address with enough fee tokens.

loop every block:
    staking.update_rewards(
        staker_address = <any currently active staker>,
        disable_rewards = true
    )
    // Effect:
    //   last_reward_block := current_block_number   (global slot consumed)
    //   rewards NOT distributed (early return)
    //
    // Any legitimate call to update_rewards in this block now reverts:
    //   assert!(current_block_number > last_reward_block)  => FAIL
```

All stakers and their delegators receive zero block rewards for every block the attacker front-runs. The lost yield accumulates permanently with each block.
