### Title
Unauthenticated Admin HTTP Server Allows Any Network-Reachable Attacker to Modify Critical Node Configuration — (`admin/command_runner.go`)

### Summary
The Flow node admin HTTP server exposes sensitive configuration-mutating commands with no mandatory authentication. When TLS mutual authentication is not configured (the default), any entity that can reach the admin port can invoke commands such as `set-config`, `stop-at-height`, and `set-uploader-enabled` without any credential check. This is the direct analog of the reported `setTolerancePercentageForRewards()` missing access modifier: a privileged operation is callable by anyone because no authorization gate is enforced.

### Finding Description
`admin/command_runner.go` starts two servers: a Unix-socket gRPC backend and an HTTP frontend bound to `r.httpAddress`. The HTTP server is started with `httpServer.ListenAndServe()` when `r.tlsConfig == nil`, and with `httpServer.ListenAndServeTLS("", "")` otherwise. [1](#0-0) 

TLS is only applied when the operator explicitly provides `--admin-cert`, `--admin-key`, and `--admin-client-certs`. The scaffold code makes this conditional: [2](#0-1) 

When those flags are absent (the default), the HTTP server starts with no authentication middleware of any kind. There is no IP allowlist, no token check, no session validation — nothing. Any HTTP POST to `/admin/run_command` is accepted and dispatched.

The registered default admin commands include: [3](#0-2) 

Execution nodes additionally register `stop-at-height` (crash the node) and `set-uploader-enabled`: [4](#0-3) 

The `set-config` command accepts any field registered with the `updatable_configs.Manager` and applies it immediately with no caller identity check: [5](#0-4) 

The `admin/README.md` documents `consensus-required-approvals-for-sealing` as a live-settable field: [6](#0-5) 

Production and benchnet deployment templates bind the admin server to all interfaces: [7](#0-6) 

### Impact Explanation
An attacker who can reach the admin port on a consensus node can:

1. **Set `consensus-required-approvals-for-sealing` to `0`** — the consensus node will seal execution results without waiting for any verification approvals, allowing unverified (potentially invalid) execution results to be sealed into the canonical chain.
2. **Set `network-id-provider-blocklist`** — silently drop all messages from legitimate staked nodes, isolating the target node from the network.
3. **On execution nodes, call `stop-at-height` with `crash: true`** — crash the execution node at an attacker-chosen height.

The most protocol-critical impact is (1): bypassing the verification approval requirement undermines the core security property that execution results are checked before finalization.

### Likelihood Explanation
- The admin server is opt-in (`--admin-addr` must be set), but it is enabled by default in all integration test harnesses and benchnet deployments, and operators routinely enable it for live nodes.
- When bound to `0.0.0.0` (as shown in benchnet templates and the `admin/README.md` examples using `localhost:9002`), the port is reachable by any process on the same host or network segment — including an unstaked Access or Observer node co-located in the same data center or VPC.
- No exploit tooling is required: a single `curl` POST suffices, as shown in the official README.
- TLS mutual auth is available but not enforced; operators who do not read the security implications of omitting `--admin-cert` leave the server completely open.

### Recommendation
1. **Require authentication by default.** Either enforce mutual TLS when `--admin-addr` is set, or add a shared-secret token check as a minimum baseline.
2. **Bind to loopback by default.** Change the default bind address from an arbitrary user-supplied value to `127.0.0.1` and require an explicit opt-in to bind to other interfaces.
3. **Add an IP allowlist middleware** so that even without TLS, only operator-controlled addresses can reach the endpoint.

### Proof of Concept
Assuming a consensus node is running with `--admin-addr=0.0.0.0:9002` and no TLS flags:

```bash
# From any host that can reach the consensus node's port 9002:
curl http://<consensus-node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config", "data": {"consensus-required-approvals-for-sealing": 0}}'
# Expected response: {"output":{"newValue":0,"oldValue":5}}
# The consensus node now seals blocks with zero verification approvals.
```

No credentials, no prior access, no staked identity required. The call succeeds because `runAdminServer` starts `httpServer.ListenAndServe()` with no authentication middleware when `tlsConfig` is nil. [8](#0-7)

```go
func (s *SetConfigCommand) Handler(_ context.Context, req *admin.CommandRequest) (any, error) {
	validatedReq := req.ValidatorData.(validatedSetConfigData)

	oldValue := validatedReq.field.Get()

	err := validatedReq.field.Set(validatedReq.value)
	if err != nil {
		if updatable_configs.IsValidationError(err) {
			return nil, fmt.Errorf("config update failed due to invalid input: %w", err)
		}
		return nil, fmt.Errorf("unexpected error setting config field %s: %w", validatedReq.field.Name, err)
	}

	res := map[string]any{
		"oldValue": oldValue,
		"newValue": validatedReq.value,
	}

	return res, nil
```
