### Title
Unvalidated `app_pk` Identity Point in CKD Eliminates Blinding, Directly Exposing Confidential Derived Key - (File: src/confidential_key_derivation/protocol.rs)

### Summary

The `ckd()` entry point accepts `app_pk: PublicKey` (alias for `blstrs::G1Projective`) from the caller with no check that it is a non-identity point. When `app_pk` is the G1 identity element, the per-participant blinding term `y · app_pk` collapses to the identity, causing `big_c` to equal the raw BLS share `x_i · H(pk ∥ app_id)`. The coordinator's aggregated `CKDOutput.big_c` then equals `msk · H(pk ∥ app_id)` — the confidential derived key itself — which any caller can recover by calling `unmask(Scalar::ZERO)`.

### Finding Description

`PublicKey` is defined as `ElementG1 = blstrs::G1Projective`. The `ckd()` function performs four input checks (participant count, duplicates, self-membership, coordinator-membership) but performs **no validation on `app_pk`**. [1](#0-0) 

The value flows unchanged into `compute_signature_share()`: [2](#0-1) 

At line 174, the blinding step is:

```rust
let big_c = big_s + app_pk * y.0;   // C ← S + y·A
```

When `app_pk = G1Projective::identity()`:

- `app_pk * y.0` → identity (G1 zero element)
- `big_c = big_s + identity = big_s = x_i · H(pk ∥ app_id)`

Each participant sends `(λ_i · Y_i, λ_i · big_c_i)` to the coordinator. The coordinator sums them: [3](#0-2) 

The aggregated `norm_big_c = Σ λ_i · x_i · H(pk ∥ app_id) = msk · H(pk ∥ app_id)`, which is the confidential key `s` defined in the protocol spec. [4](#0-3) 

`unmask(Scalar::ZERO)` returns `big_c − 0 · big_y = big_c = msk · H(pk ∥ app_id)`, directly yielding `s`.

The G2 group deserialization in the ciphersuite **does** guard against the identity: [5](#0-4) 

But this guard applies only to the MPC network's own G2 public key during DKG deserialization — it is never invoked for the caller-supplied G1 `app_pk`.

### Impact Explanation

A malicious caller submitting `app_pk = G1Projective::identity()` causes every MPC node to compute `big_c = x_i · H(pk ∥ app_id)` with no blinding. The coordinator's output `CKDOutput.big_c` is exactly `msk · H(pk ∥ app_id)` — the confidential derived secret `s` that the protocol is designed to keep hidden from all parties except the TEE app holding the matching private key. The attacker recovers `s` by calling `unmask(Scalar::ZERO)` on the returned `CKDOutput`. This is a **Critical** impact: extraction and disclosure of a confidential derived secret.

### Likelihood Explanation

`app_pk` is a direct caller-controlled parameter to the public `ckd()` API. In the documented deployment model, the TEE app submits `A` (its fresh public key) via the MPC contract. A malicious app — or any entity that can invoke the MPC contract — can trivially supply the G1 identity point. No cryptographic capability or insider access is required; `G1Projective::identity()` is a valid Rust value of the correct type. [6](#0-5) 

### Recommendation

Add an identity check on `app_pk` inside `ckd()` before the protocol starts:

```rust
if app_pk.is_identity().into() {
    return Err(InitializationError::BadParameters(
        "app_pk must not be the identity element".to_string(),
    ));
}
```

Optionally also verify that `app_pk` is a valid compressed-point encoding by round-tripping through `G1Affine::from_compressed` if the key arrives as bytes, mirroring the guard already present for G2 elements in `BLS12381G2Group::deserialize`.

    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,

    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;
