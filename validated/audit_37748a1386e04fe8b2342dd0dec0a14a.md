### Title
Missing Zero-Hash Validation in OT-Based ECDSA `sign()` Enables Presign Secret Disclosure to Coordinator — (File: `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

The OT-based ECDSA `sign()` function accepts `msg_hash = 0` without any rejection, unlike the robust ECDSA variant which explicitly enforces this check. When `msg_hash = 0` is passed, the signature-share formula collapses to a form that directly exposes each participant's rerandomized presign secret (`sigma_i`) to the coordinator. The library's own docstring acknowledges the risk but leaves enforcement entirely to the caller — a caller-controlled bypass of a mandatory security property, directly analogous to the `displayMessage` toggle in the external report.

---

### Finding Description

**Root cause — missing check in `src/ecdsa/ot_based_ecdsa/sign.rs`:**

The public entry point `sign()` performs several structural checks (participant count, coordinator membership, threshold) but contains **no check that `msg_hash != 0`**:

```rust
// src/ecdsa/ot_based_ecdsa/sign.rs  lines 22-76
pub fn sign(
    ...
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    // ... participant/threshold checks ...
    // ← NO zero-hash guard here
    Ok(make_protocol(ctx, fut))
}
```

The docstring at line 19–21 acknowledges the danger but does not enforce it:

> **WARNING** You must absolutely hash an actual message before passing it to this function. Allowing the signing of arbitrary scalars *is* a security risk, and this function only tolerates this risk to allow for genericity.

**Contrast with the robust variant** (`src/ecdsa/robust_ecdsa/sign.rs` lines 91–95), which enforces the check:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
    ));
}
```

**What happens when `msg_hash = 0` reaches the protocol:**

Each participant's signature share is computed in `compute_signature_share` (`src/ecdsa/ot_based_ecdsa/sign.rs` lines 139–159):

```rust
let k_i   = lambda * presignature.k;
let sigma_i = lambda * presignature.sigma;
let r = x_coordinate(&presignature.big_r);
Ok(msg_hash * k_i + r * sigma_i)
//  ^^^^^^^^ = 0 when msg_hash = 0
```

With `msg_hash = 0` this reduces to:

```
s_i = Rx · λ_i · σ_i
```

where `σ_i` is the participant's rerandomized presign secret (a share of `k·x`, nonce times secret key). Because `Rx` and `λ_i` are both public, the coordinator who receives `s_i` can immediately recover:

```
σ_i = s_i / (Rx · λ_i)
```

This is information that the coordinator is **not** supposed to learn in the normal protocol (where `h ≠ 0` keeps `k_i` and `σ_i` entangled in `s_i = h·k_i + Rx·σ_i`).

---

### Impact Explanation

**Disclosure of presign secrets.** Each honest participant's rerandomized `sigma_i` — a share of `k·x` (nonce × secret key) — is directly computable by the coordinator from the received signature share when `msg_hash = 0`. This is a disclosure of presign secret material that does not occur under any honest execution of the protocol.

Additionally, the resulting signature `(R, s)` will fail the coordinator's final verification check (`sig.verify(&public_key, &msg_hash)` at line 129), so the signing session is also aborted — constituting a denial of signing for honest parties.

---

### Likelihood Explanation

The `msg_hash` parameter is fully caller-controlled. Any application layer that invokes `sign()` — including a malicious coordinator orchestrating a signing session — can supply `msg_hash = 0`. The library provides only a documentation warning, not an enforcement barrier. The robust ECDSA variant demonstrates that the library authors know this check is necessary; its absence in the OT-based path is an inconsistency that a malicious caller can exploit without any cryptographic capability.

---

### Recommendation

Add the same zero-hash guard that the robust variant already enforces, immediately after the threshold check in `src/ecdsa/ot_based_ecdsa/sign.rs`:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0".to_string(),
    ));
}
```

This removes the caller-controlled bypass and makes the OT-based path consistent with the robust path.

---

### Proof of Concept

1. Run a normal DKG + presign to obtain `PresignOutput` for each participant.
2. Rerandomize each presignature via `RerandomizedPresignOutput::rerandomize_presign(...)`.
3. Call `sign(participants, coordinator, threshold, me, public_key, presignature, Scalar::ZERO)` for every participant — the library accepts this without error.
4. Each participant sends `s_i = Rx · λ_i · σ_i` to the coordinator.
5. The coordinator computes `σ_i = s_i / (Rx · λ_i)` for every `i`, recovering all rerandomized presign secrets.
6. The coordinator's final `sig.verify(...)` call fails (denial of signing), but the presign secrets have already been disclosed.

/// The signature protocol, allowing us to use a presignature to sign a message.
///
/// **WARNING** You must absolutely hash an actual message before passing it to
/// this function. Allowing the signing of arbitrary scalars *is* a security risk,
/// and this function only tolerates this risk to allow for genericity.
pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    presignature: &RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<Scalar, ProtocolError> {
    // Round 1
    // Linearize ki
    // Spec 1.1
    let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
    let k_i = lambda * presignature.k;

    // Linearize sigmai
    // Spec 1.2
    let sigma_i = lambda * presignature.sigma;

    // Compute si = h * ki + Rx * sigmai
    // Spec 1.3
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)
}
