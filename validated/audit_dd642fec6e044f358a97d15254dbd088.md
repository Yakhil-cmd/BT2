### Title
Permissionless `update_rewards` with global `last_reward_block` lets any caller permanently freeze consensus-phase reward distribution — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is callable by any non-zero address and writes to a single global `last_reward_block` storage slot. Passing `disable_rewards: true` advances that slot to the current block without distributing any rewards. Because the function enforces a strict "one call per block" invariant on that same global slot, a griefing attacker can front-run every legitimate call with `disable_rewards: true`, permanently preventing all stakers from accumulating consensus-phase rewards.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` has no role guard — `general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

The function then enforces a global, per-block gate:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

Immediately after the gate, `last_reward_block` is unconditionally written to the current block — **before** the `disable_rewards` branch is evaluated:

```cairo
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [3](#0-2) 

`last_reward_block` is a single global storage slot shared across all stakers:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [4](#0-3) 

**Attack path:**

1. In every new block, attacker calls `update_rewards(any_valid_staker, disable_rewards: true)`.
2. `last_reward_block` is set to the current block number; no rewards are distributed.
3. Any subsequent call to `update_rewards` in the same block — whether by the consensus mechanism, a staker, or anyone else — reverts with `REWARDS_ALREADY_UPDATED`.
4. Repeating this every block permanently prevents `_update_rewards` from ever executing in the consensus phase.

The `_update_rewards` path that actually credits `staker_info.unclaimed_rewards_own` and forwards pool rewards is never reached: [5](#0-4) 

---

### Impact Explanation

In the consensus-rewards phase (`!is_pre_consensus()`), `update_rewards` is the sole mechanism by which per-block STRK rewards are calculated and credited to stakers and their delegation pools. Blocking it every block means:

- `staker_info.unclaimed_rewards_own` never increases for any staker.
- Pool contracts never receive their share of block rewards via `update_rewards_from_staking_contract`.
- All accrued yield is permanently frozen — matching the **High** impact category: *"Permanent freezing of unclaimed yield."*

---

### Likelihood Explanation

- The function is fully permissionless; any EOA or contract can call it.
- The attacker needs only to submit one cheap transaction per block (no tokens, no stake required).
- Front-running is trivial on Starknet where transaction ordering is deterministic within a block.
- There is no economic cost to the attacker and no way for stakers to recover the missed block rewards.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus/sequencer role, or the attestation contract), mirroring the access control already applied to `update_rewards_from_attestation_contract`:

```cairo
fn assert_caller_is_attestation_contract(self: @ContractState) {
    assert!(
        get_caller_address() == self.attestation_contract.read(),
        "{}",
        Error::CALLER_IS_NOT_ATTESTATION_CONTRACT,
    );
}
``` [6](#0-5) 

Alternatively, make `last_reward_block` per-staker so that a call for one staker cannot block reward distribution for others.

---

### Proof of Concept

```
Block N opens.

Attacker tx (arrives first):
  staking.update_rewards(staker=<any_valid_staker>, disable_rewards=true)
  → last_reward_block := N
  → returns early, zero rewards distributed

Legitimate consensus tx (arrives second, same block):
  staking.update_rewards(staker=<target_staker>, disable_rewards=false)
  → assert!(N > last_reward_block)  // N > N is false → REVERT

Repeat every block → all stakers earn zero consensus rewards indefinitely.
```

The root cause is the unconditional write to the global `last_reward_block` at line 1485 before the `disable_rewards` guard at line 1487, combined with the absence of any caller restriction on the `update_rewards` entry point. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
