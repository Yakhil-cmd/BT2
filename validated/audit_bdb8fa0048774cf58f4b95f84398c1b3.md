### Title
Gateway Stateful Validator Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` for Admission Control — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`validate_resource_bounds` in the gateway's stateful validator reads the **current committed block's** `l2_gas_price` as the reference price for admission control. The transaction will actually be executed in the **next** block, whose price is `next_l2_gas_price` — a distinct field computed via EIP-1559 from the current block's gas usage. The two values diverge every time a block is not exactly at the gas target, causing the gateway to either reject valid transactions or admit transactions that will fail at execution.

---

### Finding Description

In `validate_resource_bounds`, the code reads:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`gas_prices.strk_gas_prices.l2_gas_price` is populated from `block_header.l2_gas_price.price_in_fri` — the price that was used **inside** the latest committed block: [2](#0-1) 

The block header also carries a separate `next_l2_gas_price` field — the EIP-1559-adjusted price for the **next** block, computed from the current block's gas consumption: [3](#0-2) 

The `next_l2_gas_price` is what the blockifier will actually enforce when the transaction is executed. `check_fee_bounds` compares `max_price_per_unit` against the block context's gas price, which is `next_l2_gas_price`: [4](#0-3) 

The gateway's threshold is `min_gas_price_percentage` (default **100%**) of the stale `l2_gas_price`: [5](#0-4) [6](#0-5) 

The TODO comment in the source code explicitly acknowledges the wrong field is being read.

---

### Impact Explanation

**Case A — False rejection (valid transaction rejected):**
When the previous block is underutilized, `next_l2_gas_price < l2_gas_price`. A transaction with `max_price_per_unit ∈ [next_l2_gas_price, l2_gas_price)` would succeed in the next block (blockifier's `check_fee_bounds` would pass), but the gateway rejects it with `GAS_PRICE_TOO_LOW` because it compares against the higher stale price. The user cannot submit a valid transaction.

**Case B — False admission (invalid transaction admitted):**
When the previous block is overutilized, `next_l2_gas_price > l2_gas_price`. A transaction with `max_price_per_unit ∈ [l2_gas_price, next_l2_gas_price)` passes the gateway check but fails `check_fee_bounds` at execution time. The transaction enters the mempool, is picked up by the batcher, and is reverted — with the user charged a fee for the failed execution.

Both cases match: **High — Mempool/gateway admission rejects valid transactions or admits invalid transactions before sequencing.**

---

### Likelihood Explanation

The EIP-1559 price adjustment fires every block. Any block that is not exactly at `gas_target` produces a `next_l2_gas_price` that differs from `l2_gas_price`. With a `gas_price_max_change_denominator` that allows up to ~12.5% change per block, the affected price band is non-trivial. Under normal network conditions (variable load), this mismatch is present in virtually every block. [7](#0-6) 

---

### Recommendation

Replace the read of `gas_prices.strk_gas_prices.l2_gas_price` with `next_l2_gas_price` from the block header. The `BlockHeaderWithoutHash` struct already carries this field (confirmed by its use in sync tests): [8](#0-7) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` should be extended to expose `next_l2_gas_price`, and `validate_resource_bounds` should use that value as the threshold reference.

---

### Proof of Concept

1. Observe the latest committed block has `l2_gas_price = P` and was underutilized (`gas_used < gas_target`).
2. Compute `next_l2_gas_price = P' < P` via `calculate_next_base_gas_price`.
3. Submit an `InvokeV3` transaction with `AllResources` bounds where `l2_gas.max_price_per_unit = P'`.
4. The gateway calls `validate_tx_l2_gas_price_within_threshold(P', P)` with `min_gas_price_percentage = 100`, computing `threshold = P`.
5. Since `P' < P`, the gateway returns `GAS_PRICE_TOO_LOW` and rejects the transaction.
6. The same transaction submitted directly to the blockifier with a block context using `next_l2_gas_price = P'` would pass `check_fee_bounds` and execute successfully — confirming the gateway incorrectly rejected a valid transaction. [5](#0-4) [9](#0-8)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
