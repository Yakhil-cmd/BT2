After thorough analysis of the threshold-signatures codebase, I investigated whether the M-07 vulnerability class — **threshold-state bug** (updating a configuration parameter without properly adjusting dependent state/indices, causing incorrect outputs) — has an analog here.

**Investigation scope:**

The primary candidate was `do_reshare` in `src/dkg.rs`, which updates the participant set and threshold. I examined:

1. **`do_reshare` Lagrange computation** (lines 611–620): The `intersection = old_participants ∩ participants` is computed, and the Lagrange coefficient `λ_i(intersection)` is used to linearize the old share. This matches the protocol spec in `docs/dkg.md` (step 2.3: `f_i(0) ← λ_i(I) · secret_i`). The implementation is consistent with the spec. [1](#0-0) 

2. **`assert_reshare_keys_invariants`** (lines 638–669): The `old_threshold` parameter is caller-supplied and not validated to be ≥ 2 (unlike the new threshold). However, even if a wrong `old_threshold` is passed, the post-hoc check in `do_keyshare` (lines 489–496) catches any mismatch between the newly derived public key and `old_public_key`, causing the protocol to abort rather than silently produce wrong output. [2](#0-1)

[3](#0-2)

pub(crate) struct ParticipantMap<'a, T> {
    #[serde(skip_serializing)]
    participants: &'a ParticipantList,
    data: Vec<Option<T>>,
    #[serde(skip_serializing)]
    count: usize,
}

impl<'a, T> ParticipantMap<'a, T> {
    /// Create a new map from a list of participants.
    ///
    /// This map only lives as long as that list of participants.
    pub fn new(participants: &'a ParticipantList) -> Self {
        // We could also require a T: Clone bound instead of doing this initialization manually.
        let size = participants.participants.len();
        let mut data = Vec::with_capacity(size);
        for _ in 0..size {
            data.push(None);
        }

        Self {
            participants,
            data,
            count: 0,
        }
    }

    /// Check if this map is full, i.e. if every participant has put something in.
    pub fn full(&self) -> bool {
        self.count == self.data.len()
    }

    /// Place the data for a participant in this map.
    ///
    /// This will do nothing if the participant is unknown, or already has a value
    pub fn put(&mut self, participant: Participant, data: T) {
        if let Some(&i) = self.participants.indices.get(&participant) {
            if let Some(data_i) = self.data.get_mut(i) {
                if data_i.is_none() {
                    *data_i = Some(data);
                    self.count += 1;
                }
            }
        }
    }
