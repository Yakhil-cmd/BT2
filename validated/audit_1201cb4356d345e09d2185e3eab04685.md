### Title
Hardcoded Flat `InclusionEffort` Allows Multi-Authorizer Fee Pooling — (`File: model/flow/transaction.go`)

---

### Summary

`TransactionBody.InclusionEffort()` returns a hardcoded flat value of `1.0 UFix64` regardless of transaction size, number of authorizers, or number of signatures. Because a single Flow transaction can carry up to 100 authorizers (each a distinct account), a group of accounts can pool their on-chain operations into one multi-authorizer transaction and collectively pay only one inclusion fee instead of N separate fees — a direct analog to the sheepDog per-user flat-fee pooling bug.

---

### Finding Description

`InclusionEffort()` is the sole input to the inclusion-fee component of every transaction fee deduction:

```go
// model/flow/transaction.go
func (tb TransactionBody) InclusionEffort() uint64 {
    // Hardcoded inclusion effort (of 1.0 UFix).
    // Eventually this will be dynamic and will depend on the transaction properties
    inclusionEffort := uint64(100_000_000)
    return inclusionEffort
}
``` [1](#0-0) 

This value is passed directly to `DeductTransactionFees` as the inclusion component:

```go
// fvm/transactionInvoker.go
_, err = executor.env.DeductTransactionFees(
    executor.proc.Transaction.Payer,
    executor.proc.Transaction.InclusionEffort(),   // always 1.0
    computationUsed)
``` [2](#0-1) 

Meanwhile, `ByteSize()` already correctly accounts for every authorizer and every signature:

```go
size += len(tb.Authorizers) * AddressLength
for _, s := range tb.PayloadSignatures { size += s.ByteSize() }
for _, s := range tb.EnvelopeSignatures { size += s.ByteSize() }
``` [3](#0-2) 

`ByteSize()` is never consulted by `InclusionEffort()`.

The REST-API parser enforces a ceiling of 100 authorizers per transaction:

```go
const maxAuthorizers = 100
if len(tx.Authorizers) > maxAuthorizers {
    return fmt.Errorf("too many authorizers...")
}
``` [4](#0-3) 

This limit is only applied in the REST parser; the gRPC and SDK submission paths do not enforce it at the same layer. Regardless, even 2–100 accounts pooling into one transaction exploits the flat fee.

---

### Impact Explanation

N accounts that each would otherwise submit an independent transaction pay only **one** inclusion fee instead of N. The payer account absorbs the single flat inclusion cost; the remaining N−1 authorizers pay nothing toward inclusion. At the default `inclusionEffortCost` of `0.00001 FLOW`, 100 accounts pooling into one transaction save `99 × 0.00001 = 0.00099 FLOW` per round. At scale or with elevated `inclusionEffortCost` parameters, the protocol systematically under-collects inclusion fees proportional to the degree of pooling. The fee model fails to price the actual network resource consumed (bandwidth ∝ `ByteSize()`). [5](#0-4) 

---

### Likelihood Explanation

**High.** Any unprivileged transaction sender can craft a multi-authorizer transaction. No special privilege, staked node, or compromised key is required — only the cooperation of N accounts willing to co-sign a single transaction body. The code comment in `InclusionEffort()` itself acknowledges the flat value is a known placeholder ("Eventually this will be dynamic"), confirming the gap is real and unaddressed. [6](#0-5) 

---

### Recommendation

Replace the hardcoded constant in `InclusionEffort()` with a value derived from `ByteSize()` (or at minimum from `len(tb.Authorizers)` and signature counts), so that the inclusion fee scales with the actual on-chain footprint of the transaction. The infrastructure for computing byte size already exists and is correct; it simply needs to be wired into the fee calculation.

---

### Proof of Concept

1. Account A, B, C … (up to 100) agree to batch their operations into one Cadence transaction:
   ```cadence
   transaction {
       prepare(a: &Account, b: &Account, c: &Account, ...) {
           // each account performs its intended operation
       }
   }
   ```
2. A is designated payer; B, C, … sign the payload; A signs the envelope.
3. `InclusionEffort()` returns `100_000_000` (1.0 UFix64) — identical to a single-authorizer transaction.
4. `DeductTransactionFees` charges A exactly `surgeFactor × inclusionEffortCost × 1.0` — the same amount a lone account would pay.
5. B, C, … pay **zero** inclusion fee despite consuming proportionally more block bandwidth (more addresses, more signatures, larger serialized transaction body as computed by `ByteSize()`). [7](#0-6) [8](#0-7)

func (tb TransactionBody) ByteSize() uint {
	size := 0
	size += len(tb.ReferenceBlockID)
	size += len(tb.Script)
	for _, arg := range tb.Arguments {
		size += len(arg)
	}
	size += 8 // gas size
	size += tb.ProposalKey.ByteSize()
	size += AddressLength                       // payer address
	size += len(tb.Authorizers) * AddressLength // Authorizers
	for _, s := range tb.PayloadSignatures {
		size += s.ByteSize()
	}
	for _, s := range tb.EnvelopeSignatures {
		size += s.ByteSize()
	}
	return uint(size)
}
```

func (executor *transactionExecutor) deductTransactionFees() (err error) {
	if !executor.env.TransactionFeesEnabled() {
		return nil
	}

	computationLimit := executor.txnState.TotalComputationLimit()

	computationUsed, err := executor.env.ComputationUsed()
	if err != nil {
		return errors.NewTransactionFeeDeductionFailedError(
			executor.proc.Transaction.Payer,
			computationLimit,
			err)
	}

	if computationUsed > computationLimit {
		computationUsed = computationLimit
	}

	_, err = executor.env.DeductTransactionFees(
		executor.proc.Transaction.Payer,
		executor.proc.Transaction.InclusionEffort(),
		computationUsed)

	if err != nil {
		return errors.NewTransactionFeeDeductionFailedError(
			executor.proc.Transaction.Payer,
			computationUsed,
			err)
	}
	return nil
```

// DeductTransactionFees executes the fee deduction function
// on the FlowFees account.
func (sys *SystemContracts) DeductTransactionFees(
	payer flow.Address,
	inclusionEffort uint64,
	executionEffort uint64,
) (cadence.Value, error) {
	return sys.Invoke(
		deductTransactionFeeSpec,
		[]cadence.Value{
			cadence.BytesToAddress(payer.Bytes()),
			cadence.UFix64(inclusionEffort),
			cadence.UFix64(executionEffort),
		},
	)
}
