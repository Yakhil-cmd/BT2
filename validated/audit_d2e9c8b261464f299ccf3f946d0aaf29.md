### Title
Unconstrained `disable_rewards` Parameter in `update_rewards` Allows Any Caller to Permanently Freeze Staker Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is callable by any non-zero address and accepts an attacker-controlled `disable_rewards: bool` parameter. When set to `true`, the function updates the global `last_reward_block` storage variable to the current block number **before** checking `disable_rewards`, then returns early without distributing any rewards. Because `last_reward_block` is a global gate that prevents `update_rewards` from being called more than once per block, a single attacker call with `disable_rewards: true` silently discards all staker rewards for that block and blocks any legitimate call for the same block.

### Finding Description
`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

Inside the function, `last_reward_block` is written to storage **before** the `disable_rewards` branch: [2](#0-1) 

The guard that prevents double-execution per block is: [3](#0-2) 

Because `last_reward_block` is a single global value (not per-staker), once an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N, every subsequent call to `update_rewards` in block N — including legitimate calls by stakers or their operators — will revert with `REWARDS_ALREADY_UPDATED`. The rewards for that block are permanently lost for all stakers.

The analog to the grace period bug is exact: just as a borrower could supply an unbounded `gracePeriod_` to prevent the admin from triggering a default, here any caller can supply `disable_rewards: true` to prevent the protocol from distributing block rewards, with no validation or access control preventing it.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` once per block causes every staker in the protocol to permanently lose their consensus block rewards for that block. Repeated across every block, this completely halts reward accrual for all stakers and delegators. The lost rewards are never recoverable because `last_reward_block` is already marked as processed.

### Likelihood Explanation
**Medium.** The attack requires no special privilege, no capital, and no prior setup — only the ability to submit a transaction before the legitimate `update_rewards` call each block. The attacker pays gas per block. On Starknet, gas costs are low, making sustained griefing economically feasible. The attacker has no direct profit motive, but a competitor, short-seller, or malicious actor could sustain this indefinitely.

### Recommendation
1. **Add access control**: Restrict `update_rewards` to a trusted caller (e.g., the attestation contract, a designated rewards manager role, or the staker/operator themselves). This is the most robust fix.
2. **Move `last_reward_block` write after the `disable_rewards` check**: If the function returns early due to `disable_rewards: true`, do not consume the block's reward slot. This prevents the griefing vector while preserving the existing call semantics.
3. **Make `last_reward_block` per-staker**: If the function is intended to be public, scoping the guard to the individual staker prevents one call from blocking all others.

### Proof of Concept
1. Staker Alice is active with non-zero STRK balance in the current epoch.
2. At block N, before Alice or any legitimate caller invokes `update_rewards`, attacker Bob calls:
   ```
   staking.update_rewards(staker_address: alice, disable_rewards: true)
   ```
3. The function passes `general_prerequisites()` (Bob is non-zero, contract is unpaused).
4. `last_reward_block` is written to `N`.
5. The `disable_rewards` branch fires; `_update_rewards` is never called; Alice's `unclaimed_rewards_own` is unchanged.
6. Alice (or anyone) now calls `update_rewards(alice, disable_rewards: false)` in block N → reverts with `REWARDS_ALREADY_UPDATED`.
7. Alice's block-N rewards are permanently lost.
8. Bob repeats this every block, freezing all staker rewards indefinitely. [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
