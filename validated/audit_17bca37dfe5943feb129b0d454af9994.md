### Title
Unprivileged Caller Can Invoke `update_rewards` With `disable_rewards: true`, Permanently Freezing Staker Yield â€” (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` in `staking.cairo` is publicly callable by any non-zero address. It accepts a `disable_rewards` parameter that, when `true`, writes the global `last_reward_block` to the current block number and returns immediately without distributing any rewards. Because `last_reward_block` is a single global variable, this blocks the legitimate sequencer from calling `update_rewards` for any staker in that same block. An attacker who front-runs the sequencer every block can permanently freeze yield accrual for all stakers.

---

### Finding Description

The spec at `docs/spec.md:1645` states that `update_rewards` is restricted to "Only starkware sequencer." However, the implementation at `src/staking/staking.cairo:1449` enforces only `general_prerequisites()`, which checks the pause flag and a non-zero caller â€” no sequencer-identity check exists. [1](#0-0) 

The critical ordering flaw is that `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

`last_reward_block` is a single global storage slot, not per-staker: [3](#0-2) 

So when any address calls `update_rewards(valid_staker, disable_rewards: true)` in block N:
1. `last_reward_block` is set to N.
2. The function returns without distributing rewards.
3. The sequencer's subsequent call in block N fails with `REWARDS_ALREADY_UPDATED`.
4. No staker earns rewards in block N.

The attacker only needs to supply any currently-active staker address (publicly readable from chain state) to pass the staker-existence and non-zero-balance assertions. [4](#0-3) 

The spec's access-control requirement is documented but not enforced: [5](#0-4) 

---

### Impact Explanation

**High â€” Permanent freezing of unclaimed yield.**

By front-running the sequencer every block with `update_rewards(any_active_staker, disable_rewards: true)`, an attacker prevents `_update_rewards` from ever executing, meaning `unclaimed_rewards_own` is never incremented for any staker and pool rewards are never transferred. All stakers and delegators are permanently denied yield. This matches the allowed impact: *"Permanent freezing of unclaimed yield."* [6](#0-5) 

---

### Likelihood Explanation

**Medium.** Starknet is a sequenced chain; front-running requires submitting a transaction with higher priority in the same block. The attacker must repeat this every block to sustain the freeze, which has a gas cost, but no special privilege or key is required â€” any EOA can execute the attack. The entry path is fully permissionless.

---

### Recommendation

Add an access-control check at the top of `update_rewards` that asserts the caller is the designated Starkware sequencer address (consistent with the spec). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the pre-consensus no-op path internally, so external callers cannot influence whether rewards are distributed.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
