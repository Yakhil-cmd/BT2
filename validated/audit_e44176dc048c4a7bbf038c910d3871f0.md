### Title
Unguarded `new` Initialization Allows Front-Running to Hijack Engine Ownership — (`File: engine/src/contract_methods/admin.rs`)

### Summary

The Aurora Engine's `new` initialization function contains no caller/predecessor check. Any unprivileged NEAR account can call it before the legitimate deployer does, setting themselves as `owner_id`. Because the deployment and initialization are performed in two separate transactions, a window exists for an attacker to front-run the `new` call and seize full administrative control of the engine.

### Finding Description

`admin::new` is the sole initialization path for the Aurora Engine's on-chain state. It is exposed as the public NEAR contract method `new` with no access modifier: [1](#0-0) 

The implementation only guards against double-initialization: [2](#0-1) 

There is no check on `env.predecessor_account_id()`. Any NEAR account that calls `new` before the legitimate deployer does will have its supplied `owner_id`, `chain_id`, and `upgrade_delay_blocks` written as the canonical engine state.

The workspace deployment helper confirms the two-transaction pattern — `deploy` and `new` are separate `await` calls with no atomicity guarantee between them: [3](#0-2) 

Contrast this with the XCC router, which is protected because `CreateAccount + DeployContract + FunctionCall("initialize")` are issued as a single atomic NEAR batch, so no external account can interpose: [4](#0-3) 

The Aurora Engine's `new` has no equivalent protection.

### Impact Explanation

`owner_id` is the gating identity for every privileged engine operation:

- `upgrade` — deploys arbitrary replacement WASM to the engine account [5](#0-4) 
- `attach_full_access_key` — adds a full-access key to the engine account, enabling complete account takeover [6](#0-5) 
- `pause_contract` — permanently freezes all user transactions [7](#0-6) 
- `factory_update` — replaces XCC router bytecode for all users [8](#0-7) 

An attacker who wins the race becomes the sole owner. They can immediately call `upgrade` to deploy a malicious engine that drains all bridged ETH and ERC-20 tokens, or call `attach_full_access_key` to take over the NEAR account entirely. This satisfies **Critical: direct theft of user funds** and **Critical: permanent freezing of funds**.

### Likelihood Explanation

NEAR is a public blockchain. Any observer monitoring the chain for newly deployed Aurora Engine contracts (e.g., watching for accounts whose code hash matches the engine WASM) can detect the uninitialized state and submit a `new` call in the same or next block. The workspace deployment code performs deploy and init as two separate transactions, making the race window concrete and reproducible. No special privilege is required — only a funded NEAR account.

### Recommendation

**Short term:** Add a predecessor check inside `new` so only the contract account itself (i.e., `env.predecessor_account_id() == env.current_account_id()`) can initialize it. This forces the deployer to use a NEAR batch transaction that atomically combines `DeployContract` and `FunctionCall("new")`, eliminating the race window — exactly the pattern already used for the XCC router.

**Long term:** Audit all other public entry points that write state before authentication is established. Consider a deployment script that always issues deploy + init as a single batch action.

### Proof of Concept

1. Alice deploys the Aurora Engine WASM to account `aurora.near` (transaction T1).
2. Eve observes T1 on-chain. The engine state is absent (`get_state` returns `Err`).
3. Eve submits transaction T2 calling `new` with `owner_id = "eve.near"`, `chain_id = <any>`, `upgrade_delay_blocks = 0`.
4. T2 executes before Alice's legitimate `new` call (T3). `state::set_state` writes Eve's parameters.
5. Alice's T3 hits the `ERR_ALREADY_INITIALIZED` guard and reverts.
6. Eve, now the recorded `owner_id`, calls `attach_full_access_key` to add her key to `aurora.near`, or calls `upgrade` to deploy a fund-draining contract. [2](#0-1) [9](#0-8)

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
