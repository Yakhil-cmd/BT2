### Title
Unprivileged Caller Can Permanently Suppress Per-Block Reward Distribution via `disable_rewards` Flag - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in `StakingRewardsManagerImpl` accepts a caller-controlled `disable_rewards: bool` parameter and has **no access control**, despite the specification mandating "Only starkware sequencer." Any unprivileged address can call `update_rewards(staker_address, disable_rewards: true)`, which atomically marks the current block as "already processed" (`last_reward_block` is written) while skipping all reward calculation and distribution. Because the `REWARDS_ALREADY_UPDATED` guard prevents a second call in the same block, the legitimate sequencer is permanently locked out of distributing rewards for that block.

### Finding Description

**Root cause — missing access control + caller-controlled suppression flag:**

`src/staking/staking.cairo` lines 1449–1507:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← rewards silently skipped
    }
    ...
``` [1](#0-0) 

The spec explicitly states:

> **access control**: Only starkware sequencer. [2](#0-1) 

But the implementation contains **no `get_caller_address()` check**. `general_prerequisites()` only enforces the pause flag. The `disable_rewards` boolean is accepted verbatim from the caller.

**Exploit path:**

1. Attacker monitors the mempool / block production.
2. For each new block (after consensus rewards are active), attacker calls:
   ```
   update_rewards(victim_staker_address, disable_rewards: true)
   ```
3. The contract writes `last_reward_block = current_block_number` and returns early — no rewards are calculated or transferred.
4. Any subsequent call (including the legitimate sequencer's) reverts with `REWARDS_ALREADY_UPDATED`.
5. Rewards for that block are permanently lost for the staker and all pool members.

The attacker can repeat this every block, continuously draining the staker's and delegators' yield.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

- Staker's `unclaimed_rewards_own` is never incremented for the attacked blocks.
- Pool members' rewards (transferred via `send_rewards_to_delegation_pool`) are never sent.
- The `last_reward_block` slot is consumed, making the loss irreversible for each targeted block.
- No funds are stolen by the attacker; the yield is simply destroyed, matching the "permanent freezing of unclaimed yield" impact category.

### Likelihood Explanation

High. The function is publicly callable with no authentication. An attacker needs only a funded Starknet account and knowledge of any active staker address (which is public on-chain). The attack is cheap (one transaction per block) and can be automated. There is no economic barrier.

### Recommendation

Enforce the access control mandated by the specification. Add a caller check at the top of `update_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
```

Alternatively, if `disable_rewards` is only ever set to `true` by the sequencer for a specific protocol reason, restrict the entire function to the sequencer address. The `disable_rewards` flag should never be caller-controlled by an unprivileged address.

### Proof of Concept

1. Deploy the staking system and advance to the consensus rewards epoch.
2. Stake as `STAKER_A` and advance `K` epochs so the stake becomes effective.
3. As an **arbitrary unprivileged address** (not the sequencer), call:
   ```
   IStakingRewardsManagerDispatcher { contract_address: staking_contract }
       .update_rewards(staker_address: STAKER_A, disable_rewards: true);
   ```
4. Observe: call succeeds, `last_reward_block` is updated to the current block, no `StakerRewardsUpdated` event is emitted.
5. Now call with `disable_rewards: false` (simulating the legitimate sequencer):
   ```
   .update_rewards(staker_address: STAKER_A, disable_rewards: false);
   ```
6. Observe: reverts with `REWARDS_ALREADY_UPDATED`.
7. Call `claim_rewards(STAKER_A)` — returns zero for that block.

This is directly confirmed by the existing test `test_update_rewards_without_distribute` in `src/staking/tests/test.cairo` lines 3985–4051, which demonstrates that calling with `disable_rewards: true` suppresses rewards and that a subsequent call in the same block is rejected — but the test never checks that the caller is the sequencer. fn test_update_rewards_without_distribute() {
    let mut cfg: StakingInitConfig = Default::default();
    general_contract_system_deployment(ref :cfg);
    let staking_contract = cfg.test_info.staking_contract;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    let staking_rewards_dispatcher = IStakingRewardsManagerDispatcher {
        contract_address: staking_contract,
    };
    let staking_config_dispatcher = IStakingConfigDispatcher { contract_address: staking_contract };
    stake_for_testing_using_dispatcher(:cfg);
    advance_k_epochs_global();
    let staker_address = cfg.test_info.staker_address;
    let staker_info_before = staking_dispatcher.staker_info_v1(:staker_address);
    // `disable_rewards = true`, and self.is_pre_consensus().
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: true);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);

    // `disable_rewards = false`, and self.is_pre_consensus().
    advance_block_number_global(blocks: 1);
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: false);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);

    // Set consensus rewards.
    let current_epoch = staking_dispatcher.get_current_epoch();
    cheat_caller_address_once(
        contract_address: staking_contract, caller_address: cfg.test_info.app_governor,
    );
    staking_config_dispatcher.set_consensus_rewards_first_epoch(epoch_id: current_epoch + 2);

    // Still self.is_pre_consensus(), test with `disable_rewards = true` and
    // `disable_rewards = false`.
    advance_block_number_global(blocks: 1);
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: true);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);
    advance_block_number_global(blocks: 1);
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: false);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);

    // Advance one epoch, still self.is_pre_consensus(), test with `disable_rewards =
    // true` and `disable_rewards = false`.
    advance_epoch_global();
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: true);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);
    advance_block_number_global(blocks: 1);
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: false);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);

    advance_epoch_global();
    // `disable_rewards = true`, and !self.is_pre_consensus().

    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_before);

    // rewards.
    advance_block_number_global(blocks: 1);
