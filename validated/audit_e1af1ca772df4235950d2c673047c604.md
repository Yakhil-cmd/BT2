### Title
Single-Step `setPoolAdminFeeDestination` Allows Irrecoverable Misdirection of Admin Fees - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary

`setPoolAdminFeeDestination` immediately overwrites `poolAdminFeeDestination[pool]` in a single transaction with no confirmation step. Every other critical address change in the protocol uses a two-step pattern, but this function does not, creating an asymmetric risk for admin fee loss.

### Finding Description

The factory implements careful two-step patterns for all other critical address changes:

- **Pool admin transfer**: `proposePoolAdminTransfer` → `acceptPoolAdmin` (new admin must confirm) [1](#0-0) 

- **Price provider change**: `proposePoolPriceProvider` → `executePoolPriceProviderUpdate` (timelock enforced) [2](#0-1) 

- **Factory owner transfer**: `Ownable2Step` from OpenZeppelin [3](#0-2) 

By contrast, `setPoolAdminFeeDestination` performs a single-step, immediate overwrite:

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;   // immediate, no confirmation
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
``` [4](#0-3) 

The stored `poolAdminFeeDestination[pool]` is then passed directly as `adminFeeDestination_` in every subsequent `collectFees` call: [5](#0-4) 

Inside `collectFees`, all admin-share tokens are transferred unconditionally to that address: [6](#0-5) 

### Impact Explanation

Two distinct loss paths exist:

**Path A – Wrong EOA/contract that accepts tokens**: All admin fees (spread + notional) accrued after the mistaken call are permanently transferred to the wrong address. The pool admin can call `setPoolAdminFeeDestination` again to stop further loss, but fees already collected to the wrong address are unrecoverable.

**Path B – Address that reverts on token receipt**: `collectFees` reverts on the `safeTransfer` to `adminFeeDestination_`. This DoS propagates to:
- `collectPoolFees` (anyone can call, now reverts)
- `setPoolAdminFees` (pool admin, calls `collectFees` first before updating fees)
- `setPoolProtocolFee` (factory owner, calls `collectFees` first before updating protocol fees) [7](#0-6) 

Path B breaks core fee-management functionality for the factory owner — a privileged actor who is otherwise fully trusted — until the pool admin corrects the destination.

### Likelihood Explanation

The pool admin is a semi-trusted role expected to manage operational parameters. A typo or clipboard error when entering a 42-character hex address is a realistic, non-malicious mistake. The zero-address guard is the only validation; no other sanity check (e.g., code-size check, token-receipt test) exists. The protocol's own design precedent — two-step patterns everywhere else — signals awareness of this risk class, making the omission here notable.

### Recommendation

Apply the same two-step pattern already used for pool admin transfers:

1. Add a `pendingAdminFeeDestination[pool]` mapping.
2. `proposePoolAdminFeeDestination` stores the candidate and emits an event.
3. A separate `acceptPoolAdminFeeDestination` (callable only by the proposed address, or after a short timelock) finalises the change.

Alternatively, at minimum, require the proposed address to be a non-reverting token recipient by performing a zero-value `safeTransfer` dry-run before committing the change.

### Proof of Concept

```
1. Pool is live; poolAdmin[pool] = Alice; poolAdminFeeDestination[pool] = Alice's multisig.
2. Alice calls setPoolAdminFeeDestination(pool, 0xDEAD...BEEF) — a typo (wrong address).
   → poolAdminFeeDestination[pool] = 0xDEAD...BEEF immediately.
3. Any subsequent collectPoolFees() call transfers all admin-share tokens to 0xDEAD...BEEF.
4. If 0xDEAD...BEEF is a contract that reverts on ERC-20 receipt:
   → collectPoolFees() reverts.
   → factory owner's setPoolProtocolFee() reverts (it calls collectFees first).
   → Alice's own setPoolAdminFees() reverts (it calls collectFees first).
   Core fee management is bricked until Alice calls setPoolAdminFeeDestination again.
5. Fees accrued between step 2 and Alice's corrective call are permanently lost.
```

  function proposePoolAdminTransfer(address pool, address newAdmin) external override nonReentrant onlyPoolAdmin(pool) {
    if (newAdmin == address(0)) revert InvalidAdmin();
    if (newAdmin == poolAdmin[pool]) revert InvalidAdmin();
    pendingPoolAdmin[pool] = newAdmin;
    emit PoolAdminTransferProposed(pool, poolAdmin[pool], newAdmin);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function acceptPoolAdmin(address pool) external override nonReentrant {
    address pending = pendingPoolAdmin[pool];
    if (pending == address(0)) revert NoPendingPoolAdminTransfer();
    if (msg.sender != pending) revert NotPendingPoolAdmin(pool, msg.sender, pending);
    address previousAdmin = poolAdmin[pool];
    poolAdmin[pool] = pending;
    delete pendingPoolAdmin[pool];
    emit PoolAdminTransferred(pool, previousAdmin, pending);
  }
