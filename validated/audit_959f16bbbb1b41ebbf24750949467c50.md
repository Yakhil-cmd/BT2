### Title
Staker Can Instantly Raise Commission to 100% via `set_commission_commitment` + `set_commission`, Stealing Delegator Yield — (`src/staking/staking.cairo`)

### Summary

A staker can atomically set a commission commitment with `max_commission = 10000` (100%) and immediately call `set_commission(10000)` in the same block. Because `is_commission_commitment_active` returns `true` the instant the commitment is written, there is no enforced delay before the increase takes effect. The staker then attests, capturing 100% of pool rewards as commission, leaving delegators with zero yield for that epoch. The code itself acknowledges this gap with an inline note.

### Finding Description

`set_commission_commitment` writes a `CommissionCommitment { max_commission, expiration_epoch }` to storage and immediately returns. The only timing constraint is `expiration_epoch > current_epoch`, so the minimum valid commitment is `expiration_epoch = current_epoch + 1`. [1](#0-0) 

`is_commission_commitment_active` checks `current_epoch < expiration_epoch`, which is trivially satisfied the moment the commitment is stored. [2](#0-1) 

`update_commission` then allows any `commission <= max_commission` while the commitment is active, with no further delay: [3](#0-2) 

The commission is written to storage immediately and is read at the next reward calculation: [4](#0-3) [5](#0-4) 

The protocol's own inline comment at the `set_commission_commitment` function signature explicitly acknowledges the gap: [6](#0-5) 

### Impact Explanation

When commission is 100%, `split_rewards_with_commission` routes the entire pool reward to the staker as commission and sends zero to the pool contract: [7](#0-6) 

Delegators receive no yield for every epoch in which the staker holds commission at 100%. After the commitment expires the staker lowers commission back to an attractive rate, repeating the cycle. This constitutes **theft of unclaimed yield** from delegators — a High-severity impact under the allowed scope.

### Likelihood Explanation

Any registered staker who has opened a delegation pool can execute this in two sequential transactions (or a multicall) with no special privileges. The staker controls both `set_commission_commitment` and `set_commission`; no external party can block the sequence. The attack is repeatable every epoch once the prior commitment expires.

### Recommendation

Enforce a minimum activation delay between when a commitment is set and when it may be used to *increase* commission. For example, require `expiration_epoch >= current_epoch + MIN_COMMITMENT_NOTICE_EPOCHS` where `MIN_COMMITMENT_NOTICE_EPOCHS` is at least 1–2 epochs, **and** additionally require that `set_commission` cannot increase commission until at least one full epoch has elapsed since the commitment was recorded. This gives delegators a window to observe the pending increase and exit before it takes effect.

### Proof of Concept

```
// Alice is a staker with commission = 100 (1%) and an active delegation pool.
// Step 1 – set a commitment allowing up to 100% commission, expiring next epoch.
staking.set_commission_commitment(
    max_commission: 10000,
    expiration_epoch: staking.get_current_epoch() + 1,
);
// Commitment is immediately active: get_current_epoch() < expiration_epoch.

// Step 2 – raise commission to 100% in the same block.
staking.set_commission(commission: 10000);
// update_commission passes: 10000 <= max_commission (10000), 10000 != 100.

// Step 3 – attest at end of epoch.
// calculate_staker_pools_rewards reads commission = 10000.
// split_rewards_with_commission returns (all_rewards, 0) for the pool.
// Delegators receive zero yield; Alice receives 100% as commission.

// Step 4 – after expiration_epoch, lower commission back to 100 (1%).
staking.set_commission(commission: 100);
// Repeat next epoch.
```

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
