### Title
Permanent Loss of User Tokens When `exit_to_near` Promise Fails Without `error_refund` Feature — (File: `engine/src/contract_methods/connector.rs`)

---

### Summary

When a user calls the `exit_to_near` precompile to bridge tokens from Aurora to NEAR, their ERC-20 tokens (or ETH) are burned inside the EVM and a NEAR `ft_transfer` promise is created. If that promise fails at runtime (e.g., the recipient is not registered with the NEP-141 contract) and the binary was compiled without the `error_refund` feature, the `exit_to_near_precompile_callback` silently returns `Ok(None)` without issuing any refund. The burned tokens are permanently lost.

---

### Finding Description

**Step 1 — Token burning happens before the NEAR promise.**

In `engine-precompiles/src/native.rs`, the `exit_to_near` precompile burns the user's ERC-20 tokens (or ETH) inside the EVM and then emits a log that the engine converts into a NEAR `ft_transfer` promise. The burn is committed to EVM state before the NEAR-side promise executes. [1](#0-0) 

**Step 2 — The refund args are only populated under a compile-time feature flag.**

`refund_call_args` is gated behind `#[cfg(feature = "error_refund")]`. Without that feature, the function does not exist, and `ExitToNearPrecompileCallbackArgs::refund` is always `None`. [1](#0-0) 

**Step 3 — The callback silently drops the failure when `args.refund` is `None`.**

In `exit_to_near_precompile_callback`, the failure branch is:

```rust
} else if let Some(args) = args.refund {
    // Exit call failed; need to refund tokens
    let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
    ...
} else {
    None   // ← reached when error_refund is absent; no refund, no error
};
```

When `args.refund` is `None` (no `error_refund` feature), the `else` arm is taken, `maybe_result` is `None`, and the function returns `Ok(None)`. The burned tokens are gone with no on-chain record of the failure. [2](#0-1) 

**Step 4 — `refund_on_error` exists but is never reached.**

The refund logic in `engine/src/engine.rs` correctly re-mints ERC-20 tokens or transfers ETH back from the precompile address, but it is only reachable when `args.refund` is `Some`. [3](#0-2) 

**Step 5 — Tests confirm the loss.**

The integration tests explicitly document the two behaviors:

```rust
#[cfg(feature = "error_refund")]
let balance = FT_TRANSFER_AMOUNT.into();          // tokens refunded
#[cfg(not(feature = "error_refund"))]
let balance = (FT_TRANSFER_AMOUNT - FT_EXIT_AMOUNT).into(); // tokens lost
``` [4](#0-3)

---

### Impact Explanation

**Critical — Permanent freezing / direct loss of user funds.**

Any user who calls `exit_to_near` (ERC-20 or ETH variant) and whose downstream `ft_transfer` promise fails will have their tokens burned on Aurora with no recovery path. The tokens are neither on Aurora (burned) nor on NEAR (transfer failed). They are destroyed. This matches the "permanent freezing of funds" and "direct theft of any user funds in motion" impact categories.

---

### Likelihood Explanation

**Medium.** The `ft_transfer` promise fails whenever the recipient NEAR account is not registered with the NEP-141 contract — a common user mistake. It also fails if the NEP-141 contract is paused, has insufficient storage, or rejects the transfer for any other reason. These are realistic, unprivileged, externally triggerable conditions. The only prerequisite is that the production binary was compiled without `error_refund`, which the codebase explicitly supports as a valid build configuration.

---

### Recommendation

1. Make `error_refund` a **default feature** in `engine-precompiles/Cargo.toml` so it is always active unless explicitly opted out.
2. Add a compile-time assertion or CI gate that rejects production builds lacking `error_refund`.
3. Alternatively, remove the feature flag entirely and unconditionally populate `ExitToNearPrecompileCallbackArgs::refund` so the refund path is always available.

---

### Proof of Concept

1. Deploy Aurora Engine compiled **without** the `error_refund` feature.
2. User holds 100 ERC-20 tokens (bridged NEP-141) on Aurora.
3. User calls the `exit_to_near` precompile targeting a NEAR account that is **not registered** with the NEP-141 contract.
4. EVM execution burns the 100 tokens from the user's EVM balance and emits the exit log.
5. The engine schedules a NEAR `ft_transfer` promise to the unregistered recipient.
6. The `ft_transfer` promise fails (recipient not registered).
7. `exit_to_near_precompile_callback` is invoked; `args.refund` is `None`; the `else` branch returns `Ok(None)`.
8. The user's 100 tokens are permanently destroyed — neither on Aurora nor on NEAR. [6](#0-5) [7](#0-6)

fn refund_call_args(
    params: &ExitToNearParams,
    event: &events::ExitToNear,
) -> Option<RefundCallArgs> {
    Some(RefundCallArgs {
        recipient_address: match params {
            ExitToNearParams::BaseToken(params) => params.refund_address,
            ExitToNearParams::Erc20TokenParams(params) => params.refund_address,
        },
        erc20_address: match params {
            ExitToNearParams::BaseToken(_) => None,
            ExitToNearParams::Erc20TokenParams(_) => {
                let erc20_address = match event {
                    events::ExitToNear::Legacy(legacy) => legacy.erc20_address,
                    events::ExitToNear::Omni(omni) => omni.erc20_address,
                };
                Some(erc20_address)
            }
        },
        amount: types::u256_to_arr(&match event {
            events::ExitToNear::Legacy(legacy) => legacy.amount,
            events::ExitToNear::Omni(omni) => omni.amount,
        }),
    })
}

pub fn refund_on_error<I: IO + Copy, E: Env, P: PromiseHandler>(
    io: I,
    env: &E,
    state: EngineState,
    args: &RefundCallArgs,
    handler: &mut P,
) -> EngineResult<SubmitResult> {
    let current_account_id = env.current_account_id();
    if let Some(erc20_address) = args.erc20_address {
        // ERC-20 exit; re-mint burned tokens
        let erc20_admin_address = current_address(&current_account_id);
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, erc20_admin_address, current_account_id, io, env);

        let refund_address = args.recipient_address;
        let amount = U256::from_big_endian(&args.amount);
        let input = setup_refund_on_error_input(amount, refund_address);

        engine.call(
            &erc20_admin_address,
            &erc20_address,
            Wei::zero(),
            input,
            u64::MAX,
            Vec::new(),
            Vec::new(),
            handler,
        )
    } else {
        // ETH exit; transfer ETH back from precompile address
        let exit_address = exit_to_near::ADDRESS;
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, exit_address, current_account_id, io, env);
        let refund_address = args.recipient_address;
        let amount = Wei::new(U256::from_big_endian(&args.amount));
        engine.call(
            &exit_address,
            &refund_address,
            amount,
            Vec::new(),
            u64::MAX,
            vec![
                (exit_address.raw(), Vec::new()),
                (refund_address.raw(), Vec::new()),
            ],
            Vec::new(),
            handler,
        )
    }
