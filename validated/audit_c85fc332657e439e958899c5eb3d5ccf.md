### Title
Presignature Not Consumed After Rerandomization Enables Reuse and Private Key Extraction - (File: `src/ecdsa/ot_based_ecdsa/mod.rs`)

---

### Summary

`RerandomizedPresignOutput::rerandomize_presign()` accepts `presignature: &PresignOutput` by shared reference rather than by value. Because the `PresignOutput` is never consumed, the same presignature nonce material can be rerandomized an arbitrary number of times with different `RerandomizationArguments`, producing multiple `RerandomizedPresignOutput` instances that all share the same underlying nonce `k`. Using any two of these in separate signing sessions allows a standard ECDSA nonce-reuse attack to recover the private key.

---

### Finding Description

The `PresignOutput` struct holds the per-session nonce share `k` and commitment `big_r` that are the cryptographic heart of the OT-based ECDSA presignature: [1](#0-0) 

The struct derives `Clone`, `Serialize`, and `Deserialize`, so it can be freely copied and persisted. The rerandomization entry point is: [2](#0-1) 

The critical signature is:

```rust
pub fn rerandomize_presign(
    presignature: &PresignOutput,   // <-- shared reference, NOT moved/consumed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

Because `presignature` is taken by `&` reference, the caller retains full ownership of the `PresignOutput` after the call returns. Nothing in the library marks the presignature as used, zeros it, or prevents a second call with different `args`. This is the direct analog of the reported `withdraw()` bug: just as `unlockBlock` was never reset to `0` after funds were withdrawn, `PresignOutput` is never consumed or invalidated after being used to produce a `RerandomizedPresignOutput`.

The same root cause exists in the robust ECDSA path, where `PresignOutput` is similarly a plain data struct passed into rerandomization without being consumed. [3](#0-2) 

---

### Impact Explanation

The library's own security documentation explicitly states the consequence:

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks." [4](#0-3) 

Two signatures `(R, s₁)` and `(R', s₂)` produced from the same `k` (even after different scalar rerandomizations `δ₁`, `δ₂`) satisfy a linear relation in `k` and `x` (the private key). Solving the resulting system of two equations in two unknowns yields the aggregate secret key. This is **Critical** impact: full extraction of the private signing key.

---

### Likelihood Explanation

A malicious coordinator is an explicitly modeled adversary in this library. The attack path is:

1. The coordinator initiates a first signing session; each honest participant calls `rerandomize_presign(&presign_out, &args1)` and then `sign(...)`, consuming the `RerandomizedPresignOutput` by value — but **not** the underlying `PresignOutput`.
2. Because `PresignOutput` is not consumed, participants who store it (e.g., in a database keyed by session ID) still hold valid presignature material.
3. The coordinator immediately initiates a second signing session referencing the same presignature ID but with a different message hash `h` or tweak `ε`.
4. Participants call `rerandomize_presign(&presign_out, &args2)` again — the library offers no resistance.
5. The coordinator collects both completed signatures and solves for the private key offline.

The `sign()` function itself takes `presignature: RerandomizedPresignOutput` by value, so the rerandomized output is consumed — but this only prevents reuse of the *rerandomized* form, not the *original* `PresignOutput` from which unlimited rerandomized copies can be derived. [5](#0-4) 

---

### Recommendation

Change `rerandomize_presign` to consume the `PresignOutput` by value:

```rust
pub fn rerandomize_presign(
    presignature: PresignOutput,   // moved, not borrowed
    args: &RerandomizationArguments,
) -> Result<Self, ProtocolError>
```

This mirrors the fix applied in the referenced report (resetting `unlockBlock` to `0` after withdrawal): the presignature state is "reset" by being moved into the function and dropped, making reuse a compile-time error. Additionally, remove the `Clone` derive from `PresignOutput` so that callers cannot circumvent the move semantics by cloning before passing. [1](#0-0) 

---

### Proof of Concept

```rust
// Attacker-controlled coordinator scenario
let presign_out: PresignOutput = run_presign_protocol(...);

// Session 1: sign message h1 with tweak eps1
let rerandomized1 = RerandomizedPresignOutput::rerandomize_presign(
    &presign_out,                          // presign_out NOT consumed
    &RerandomizationArguments { big_r: presign_out.big_r, tweak: eps1, msg_hash: h1, ... },
)?;
let sig1 = run_sign_protocol(rerandomized1, h1, ...);

// Session 2: sign DIFFERENT message h2 — presign_out still valid
let rerandomized2 = RerandomizedPresignOutput::rerandomize_presign(
    &presign_out,                          // reused — library does not prevent this
    &RerandomizationArguments { big_r: presign_out.big_r, tweak: eps2, msg_hash: h2, ... },
)?;
let sig2 = run_sign_protocol(rerandomized2, h2, ...);

// Both sig1 and sig2 share the same underlying nonce k.
// Solve the two-equation linear system to recover the private key x.
```

The library compiles and executes both sessions without error. The only protection is the documentation warning, which is not enforced at the API level — exactly the pattern described in the external report where off-chain logic was the only mitigation. [2](#0-1) [6](#0-5)

    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
