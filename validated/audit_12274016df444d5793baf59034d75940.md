### Title
Inaccessible `resume_precompiles` When Contract Is Paused — (`engine/src/contract_methods/admin.rs`)

### Summary

`resume_precompiles` enforces `require_running`, which means it is completely blocked whenever the contract-level pause flag (`is_paused == true`) is set. If the exit precompiles are paused and the contract is simultaneously paused, the owner cannot lift the precompile pause without first resuming the entire contract — a step that may itself be unsafe or undesirable during an active incident.

### Finding Description

`resume_precompiles` in `engine/src/contract_methods/admin.rs` begins with:

```rust
pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← blocks when is_paused == true
        ...
        pauser.resume_precompiles(flags);
        Ok(())
    })
}
``` [1](#0-0) 

`require_running` returns `ERR_PAUSED` whenever `state.is_paused` is `true`: [2](#0-1) 

`pause_contract` sets `is_paused = true`: [3](#0-2) 

The precompile pause flags (`EXIT_TO_NEAR`, `EXIT_TO_ETHEREUM`) are stored independently in NEAR storage under `PAUSE_FLAGS_KEY`: [4](#0-3) 

These two pause mechanisms are orthogonal — the contract-level `is_paused` flag and the precompile-level `PrecompileFlags` bitmask are stored separately. Yet `resume_precompiles` unconditionally requires the contract to be running before it will clear the precompile flags. There is no equivalent `require_running` guard on `resume_contract`, so the contract-level pause can be lifted while paused, but the precompile-level pause cannot.

### Impact Explanation

The `EXIT_TO_NEAR` and `EXIT_TO_ETHEREUM` precompiles are the sole on-chain bridge exit paths for Aurora users. When these precompiles are paused, users cannot withdraw ERC-20 tokens or ETH back to NEAR or Ethereum: [5](#0-4) [6](#0-5) 

The realistic emergency scenario is:

1. A bridge exploit is detected; the operator calls `pause_precompiles` to block the exit precompiles.
2. The operator also calls `pause_contract` for a full emergency shutdown.
3. The exploit is contained. The operator now wants to selectively re-enable bridge exits while keeping the contract paused (e.g., while a fix is being staged).
4. `resume_precompiles` fails with `ERR_PAUSED` because `is_paused == true`.
5. The operator is forced to call `resume_contract` first — re-exposing all other contract entry points — before being able to clear the precompile flags.

During the window between steps 2 and 5, user funds locked in Aurora ERC-20 tokens are frozen: they cannot be bridged out via the exit precompiles, and the operator has no way to lift only the precompile pause without lifting the full contract pause. This constitutes a **High: Temporary freezing of funds**.

### Likelihood Explanation

The scenario requires both `pause_precompiles` and `pause_contract` to be active simultaneously, which is a realistic emergency response pattern (pause precompiles first to stop a specific exploit, then pause the whole contract for a broader shutdown). The `pause_contract` path is owner-only and the `pause_precompiles` path is also owner-controlled via `get_authorizer`: [7](#0-6) 

Both actions are expected to be taken together in a serious incident. Likelihood is **Low-to-Medium** given that the dual-pause state is an intended operational mode.

### Recommendation

Remove the `require_running` guard from `resume_precompiles`. Resuming precompiles is a recovery action that should be available regardless of the contract-level pause state. The owner-only check already provides sufficient access control:

```rust
pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        // Remove: require_running(&state)?;
        let predecessor_account_id = env.predecessor_account_id();
        require_owner_only(&state, &predecessor_account_id)?;
        let args: PausePrecompilesCallArgs = io.read_input_borsh()?;
        let flags = PrecompileFlags::from_bits_truncate(args.paused_mask);
        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.resume_precompiles(flags);
        Ok(())
    })
}
```

Symmetrically, consider whether `pause_precompiles` should also be callable when the contract is paused, so that the precompile pause state can be managed independently of the contract-level pause.

### Proof of Concept

1. Deploy the Aurora Engine contract.
2. Call `pause_precompiles` with mask `0b11` (both `EXIT_TO_NEAR` and `EXIT_TO_ETHEREUM`) as the owner.
3. Call `pause_contract` as the owner. Contract is now fully paused.
4. Attempt to call `resume_precompiles` with mask `0b11` as the owner.
5. Observe the call fails with `ERR_PAUSED` — the precompile pause flags cannot be cleared.
6. Users attempting to bridge out via `EXIT_TO_NEAR` or `EXIT_TO_ETHEREUM` receive `ERR_PAUSED` from the precompile set: [8](#0-7) 

The only resolution path is to call `resume_contract` first (re-exposing all contract entry points), then call `resume_precompiles`, then optionally call `pause_contract` again — a three-step sequence that is not atomic and exposes the contract during the intermediate step.

    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;

        state.is_paused = true;
        state::set_state(&mut io, &state)?;

impl<I: IO> EnginePrecompilesPauser<I> {
    /// Key for storing [`PrecompileFlags`].
    const PAUSE_FLAGS_KEY: &'static [u8; 11] = b"PAUSE_FLAGS";

    /// Creates new [`EnginePrecompilesPauser`] instance that reads from and writes into storage accessed using `io`.
    pub const fn from_io(io: I) -> Self {
        Self { io }
    }

    fn read_flags_from_storage(&self) -> PrecompileFlags {
        self.io
            .read_storage(&Self::storage_key())
            .map_or_else(PrecompileFlags::empty, |bytes| {
                const U32_SIZE: usize = size_of::<u32>();
                assert_eq!(bytes.len(), U32_SIZE, "PrecompileFlags value is corrupted");
                let mut buffer = [0u8; U32_SIZE];
                bytes.copy_to_slice(&mut buffer);
                PrecompileFlags::from_bits_truncate(u32::from_le_bytes(buffer))
            })
    }

    fn write_flags_into_storage(&mut self, pause_flags: PrecompileFlags) {
        self.io
            .write_storage(&Self::storage_key(), &pause_flags.bits().to_le_bytes());
    }

    fn storage_key() -> Vec<u8> {
        bytes_to_key(KeyPrefix::Config, Self::PAUSE_FLAGS_KEY)
    }
