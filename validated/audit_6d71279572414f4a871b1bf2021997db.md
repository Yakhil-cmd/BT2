### Title
`skip_stateful_validations` Allows Signature-Bypass Front-Running to Censor Victim Invoke Transactions — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator skips the `__validate__` entry-point call (i.e., signature verification) for an invoke transaction with `nonce == 1` when the on-chain account nonce is `0`, provided `account_tx_in_pool_or_recent_block(sender_address)` returns `true`. That check only tests whether **any** transaction exists in the mempool for the address — it does not verify that the existing transaction is specifically a `deploy_account` for that address. An attacker who observes a victim's `deploy_account` transaction in the mempool can immediately submit a forged invoke transaction (same `sender_address`, `nonce = 1`, arbitrary invalid signature) that exploits the skip path, occupies the `(address, nonce=1)` slot in the mempool without signature verification, and causes the victim's legitimate invoke transaction to be rejected as `DuplicateNonce`.

---

### Finding Description

**Invariant broken:** Every transaction admitted to the mempool must either (a) have passed `__validate__` (signature verified) or (b) have been admitted via the skip path only when a legitimate `deploy_account` for that exact address is already in the mempool. The code enforces (b) only by checking address presence, not transaction type.

### Impact Explanation

**Matching impact category:** *High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.*

1. The attacker's forged invoke transaction (invalid signature, arbitrary `sender_address = victim`) is **accepted** by the gateway and admitted to the mempool without signature verification.
2. The victim's legitimate invoke transaction is **rejected** at the gateway's `validate_by_mempool` step with `DuplicateNonce`.
3. The attacker's tx will fail during batcher execution (when `__validate__` is called for real), but the victim's tx was already dropped and must be resubmitted.
4. Because the attacker's tx fails `__validate__` during execution, **no fee is charged to the attacker**, making the attack free and repeatable — the attacker can continuously censor the victim's first post-deploy invoke transaction.

---

### Likelihood Explanation

- **Unprivileged trigger:** Any network participant can submit an RPC transaction with an arbitrary `sender_address`.
- **Observable precondition:** The victim's `deploy_account` transaction is visible in the mempool (public mempool propagation).
- **Timing:** The attacker only needs to front-run the victim's invoke tx after the deploy_account is in the mempool — a standard mempool-watching attack.
- **Cost:** Zero — the attacker's tx fails `__validate__` during execution, so no fee is deducted.
- **Repeatability:** The attacker can repeat indefinitely, permanently preventing the victim from executing their first post-deploy invoke.

---

### Recommendation

Replace the address-only `account_tx_in_pool_or_recent_block` check with a type-specific check that verifies a `deploy_account` transaction (not just any transaction) exists in the mempool for the given address. Concretely:

1. Add a `deploy_account_in_pool(address: ContractAddress) -> bool` method to the mempool that checks `tx_pool` for a `DeployAccount` variant at that address.
2. In `skip_stateful_validations`, replace the call to `account_tx_in_pool_or_recent_block` with `deploy_account_in_pool`.
3. Alternatively, require that the `deploy_account` transaction hash be supplied by the caller (as done in `native_blockifier`'s `PyValidator::should_run_stateful_validations` which takes an explicit `deploy_account_tx_hash: Option<TransactionHash>`) and verify it is present in the mempool with the correct type. ### Proof of Concept

```
1. Victim generates a new account at address X (class_hash C, salt S, constructor_calldata D).
2. Victim submits deploy_account tx (nonce=0, address=X) → admitted to mempool.
3. Attacker observes X in the mempool via account_tx_in_pool_or_recent_block.
4. Attacker crafts an RpcInvokeTransactionV3:
     sender_address = X
     nonce          = 1
     calldata       = [] (arbitrary)
     signature      = [0x1, 0x2]  (invalid for X, but passes size check)
     resource_bounds = valid non-zero bounds
5. Attacker submits the forged invoke tx to the gateway.
   Gateway path:
     a. StatelessTransactionValidator::validate → passes (valid address range, valid bounds, valid sig size)
     b. convert_rpc_tx_to_internal → computes tx_hash_attacker (includes X, nonce=1, calldata)
     c. extract_state_nonce_and_run_validations:
          get_nonce_from_state(X) → 0
          validate_nonce: 0 ≤ 1 ≤ max_allowed_nonce_gap → OK
          validate_by_mempool: no (X, nonce=1) in pool → OK
          skip_stateful_validations: nonce==1, account_nonce==0,
            account_tx_in_pool_or_recent_block(X) == true (deploy_account is there) → skip=true
          run_validate_entry_point(skip=true) → __validate__ NOT called → OK
     d. mempool.add_tx(attacker_tx) → (X, nonce=1) slot occupied
6. Victim submits legitimate invoke tx (nonce=1, valid signature).
   Gateway path:
     validate_by_mempool → validate_fee_escalation →
       get_by_address_and_nonce(X, 1) found → DuplicateNonce → REJECTED
7. Victim's invoke tx is dropped. Attacker's tx will revert during batcher execution
   (no fee charged). Attacker repeats step 4-5 indefinitely.
```

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
