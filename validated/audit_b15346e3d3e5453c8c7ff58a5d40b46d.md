### Title
Pool Admin Can Set Per-Bin Additional Fees Without Any Cap, Bypassing the Factory Fee-Cap System — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees()` forwards `addFeeBuyE6` / `addFeeSellE6` directly to the pool with **no upper-bound check**, while the parallel `setPoolAdminFees()` path enforces `maxAdminSpreadFeeE6` / `maxAdminNotionalFeeE8`. A pool admin can therefore set per-bin additional fees to the full `uint16` maximum (65 535 = 6.5535 %) with no timelock and no factory-level cap, causing traders to pay materially more than the oracle price implies.

---

### Finding Description

`setPoolAdminFees` enforces caps before writing:

```solidity
// MetricOmmPoolFactory.sol lines 414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees` has **no equivalent check**:

```solidity
// MetricOmmPoolFactory.sol lines 450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` writes the values unconditionally:

```solidity
// MetricOmmPool.sol lines 464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
  if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
  BinState storage s = _binStates[bin];
  s.addFeeBuyE6 = addFeeBuyE6;
  s.addFeeSellE6 = addFeeSellE6;
  ...
}
``` [3](#0-2) 

During every swap the per-bin additional fee is added on top of the oracle-derived base spread fee:

```solidity
// MetricOmmPool.sol lines 910, 999
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [4](#0-3) 

The hard cap for the entire spread-fee system is `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20 %). [5](#0-4) 

A pool admin calling `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` sets an additional 6.5535 % fee on bin 0 with no factory resistance — no cap check, no timelock, no owner approval. This is structurally inconsistent with the cap system that governs every other admin fee path.

---

### Impact Explanation

Every swap routed through the affected bin pays `baseFee + 6.5535 %` instead of `baseFee`. At a typical 0.3 % base spread the effective fee becomes ~6.85 %, meaning a trader executing a $100 000 buy receives ~$6 550 less token0 than the oracle price implies. The excess accrues to LPs, not to the protocol, so it is not recoverable by the trader. This is a direct, quantifiable loss of swap output above any reasonable Sherlock threshold.

---

### Likelihood Explanation

The pool admin role is semi-trusted and is explicitly scoped as "inside caps." The call path is a single transaction with no timelock, no co-signer, and no owner approval. Any pool admin — including one that was legitimately assigned at pool creation but later acts adversarially — can execute this immediately. The `uint16` type provides no meaningful protection: 65 535 / 1e6 = 6.5535 % is a large, economically significant fee.

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` analogous to the one in `setPoolAdminFees`. The simplest approach is to introduce a `maxAdminBinAdditionalFeeE6` constant (or owner-settable variable) and revert if either `addFeeBuyE6` or `addFeeSellE6` exceeds it:

```solidity
if (addFeeBuyE6  > maxAdminBinAdditionalFeeE6) revert AdminFeeTooHigh();
if (addFeeSellE6 > maxAdminBinAdditionalFeeE6) revert AdminFeeTooHigh();
```

Alternatively, enforce that `baseFee + addFee` cannot exceed `maxAdminSpreadFeeE6` at the time of the call.

---

### Proof of Concept

```solidity
// Foundry test sketch
function test_binFeeBypassesCap() public {
    address pool = factory.createPool(defaultParams());

    // Pool admin sets per-bin additional fee to uint16 max — no revert
    vm.prank(poolAdmin);
    factory.setPoolBinAdditionalFees(pool, 0, 65535, 65535);

    // Trader executes a buy swap in bin 0
    uint256 amountIn = 1e18;
    vm.prank(trader);
    (uint256 amountOut,) = pool.swap(SwapParams({...}));

    // Effective fee ≈ baseFee + 6.5535% — far above any documented cap
    // amountOut is ~6.5% less than oracle price implies
    assertLt(amountOut, oracleImpliedOut * 94 / 100);
}
```
