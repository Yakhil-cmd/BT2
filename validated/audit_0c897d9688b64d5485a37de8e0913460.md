### Title
Staker Can Instantly Raise Commission to 100% via `set_commission_commitment` + `set_commission` in the Same Block — (`src/staking/staking.cairo`)

### Summary

A staker can atomically set a `CommissionCommitment` with `max_commission = 10000` (100%) and immediately call `set_commission(10000)` in the same block. Because the commitment is active the instant it is written (no activation delay), the commission jumps from any value to 100% with zero notice to delegators, stealing all future yield from the pool.

The code itself acknowledges this with an explicit developer note:

> **Note**: Current commission increase safeguards still allow for sudden commission changes.

### Finding Description

The `set_commission_commitment` function writes a `CommissionCommitment { max_commission, expiration_epoch }` to storage and immediately returns. There is no activation delay — the commitment is considered active as soon as `current_epoch < expiration_epoch`.

`is_commission_commitment_active` at `src/staking/staking.cairo:2178`:
```cairo
fn is_commission_commitment_active(
    self: @ContractState, commission_commitment: CommissionCommitment,
) -> bool {
    self.get_current_epoch() < commission_commitment.expiration_epoch
}
```

`set_commission_commitment` at `src/staking/staking.cairo:771`:
```cairo
assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
```

The minimum valid `expiration_epoch` is `current_epoch + 1`, which satisfies `current_epoch < expiration_epoch` immediately. So the commitment is live the moment it is stored.

`update_commission` at `src/staking/staking.cairo:1583`:

When the commitment is active, the only constraints are `commission <= max_commission` and `commission != old_commission`. There is **no lower-bound check** — commission can be raised to any value up to `max_commission`.

**Attack sequence (two calls, same block):**

1. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
   - Valid: `10000 >= current_commission`, `current_epoch + 1 > current_epoch`.
   - Commitment is immediately active.
2. Staker calls `set_commission(commission: 10000)`.
   - Commitment is active → only checks `10000 <= 10000` ✓ and `10000 != old_commission` ✓.
   - Commission is written as 100% instantly.

### Impact Explanation

All delegator rewards for every epoch the commission is at 100% are redirected entirely to the staker's reward address. Delegators receive zero yield. This is **theft of unclaimed yield** (High severity). Delegators must exit the pool to stop the bleeding, but rewards accrued during the 100%-commission period are permanently lost to them.

### Likelihood Explanation

- Any staker (unprivileged) can execute this.
- No leaked keys, no governance, no bridge compromise required.
- Both calls can be submitted in the same block (Starknet supports multicall natively via account abstraction).
- The staker can target a specific large delegation by watching the mempool/sequencer queue and front-running it, or simply raise commission after attracting delegators with a low advertised rate.
- The developer comment at `src/staking/staking.cairo:745` explicitly acknowledges the gap: *"Current commission increase safeguards still allow for sudden commission changes."*

### Recommendation

Introduce an activation delay for commission increases. The commitment should only become usable for **raising** commission starting from a future epoch (e.g., `current_epoch + K` where K is the same delay used for balance changes). Specifically:

- Store a `valid_from_epoch` alongside the commitment.
- In `update_commission`, when the commitment is active and `commission > old_commission`, assert `current_epoch >= commitment.valid_from_epoch`.

This ensures delegators have at least one full epoch of advance notice before a commission increase takes effect, matching the intent-action pattern used elsewhere in the protocol.

### Proof of Concept

```
// Staker has commission = 500 (5%), pool has delegators.
// Block N, epoch E:

// Tx 1:
staking.set_commission_commitment(
    max_commission: 10000,          // 100%
    expiration_epoch: E + 1         // minimum valid value
);
// Commitment is immediately active: E < E+1

// Tx 2 (same block):
staking.set_commission(commission: 10000);
// update_commission: commitment active → only checks 10000 <= 10000 ✓ and 10000 != 500 ✓
// Commission is now 100% — all delegator rewards redirected to staker.
```

```text
        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
```

```text
        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
```
