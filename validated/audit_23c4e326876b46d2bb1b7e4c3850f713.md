### Title
Gateway L2 Gas Price Threshold Uses Floor Division, Admitting Transactions Below True Minimum — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price by multiplying `previous_block_l2_gas_price` by `min_gas_price_percentage / 100` and calling `.to_integer()`, which performs **floor (truncating) integer division**. When the product is not exactly divisible by 100, the computed threshold is 1 unit lower than the true fractional threshold. A transaction whose `max_price_per_unit` equals this floor value passes the check even though it is strictly below the intended minimum, violating the gateway admission invariant.

---

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← floor division
if tx_l2_gas_price.0 < threshold {
    return Err(...)
}
```

`Ratio::new(p, 100) * price` produces `Ratio::new(p * price, 100)`. Calling `.to_integer()` on a `num_rational::Ratio<u128>` returns `(p * price) / 100` using integer (floor) division.

The semantically correct threshold is `⌈p * price / 100⌉` (ceiling), because any gas price strictly below that fractional value should be rejected. Using floor instead of ceiling means the enforced threshold is `⌊p * price / 100⌋`, which is one unit lower whenever `(p * price) % 100 ≠ 0`.

**Concrete example:**
- `min_gas_price_percentage = 99`, `previous_block_l2_gas_price = 101`
- True threshold: `⌈99 × 101 / 100⌉ = ⌈99.99⌉ = 100`
- Computed threshold: `⌊99 × 101 / 100⌋ = ⌊99.99⌋ = 99`
- A transaction with `max_price_per_unit = 99` satisfies `99 < 99` → **false** → **admitted**
- But `99 < 100` (true threshold) → should be **rejected**

The check `tx_l2_gas_price.0 < threshold` uses strict less-than, so the off-by-one from floor division directly opens a 1-unit gap below the intended floor. [1](#0-0) 

The `min_gas_price_percentage` field is a `u8` (0–255, representing 0–100%) and is operator-configurable: [2](#0-1) 

The default is 100, which produces no rounding error (`100 * price / 100 = price` exactly). Any non-100% value where `(percentage * prev_price) % 100 ≠ 0` triggers the off-by-one.

---

### Impact Explanation

This is a **gateway/mempool admission** invariant violation. The gateway's stateful validator is the last line of defense before a transaction enters the mempool. A transaction admitted here with `max_price_per_unit` one unit below the true threshold:

1. Bypasses the intended minimum gas price floor.
2. Is forwarded to the mempool and eventually sequenced.
3. Pays a gas price the protocol intended to reject.

This maps to: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

The economic impact per transaction is small (1 FRI unit), but the invariant is broken for every transaction submitted at exactly the floor value, and the condition is reliably reproducible whenever `(min_gas_price_percentage * prev_price) % 100 ≠ 0`.

---

### Likelihood Explanation

- Requires `min_gas_price_percentage` to be set to a value other than 0 or 100 (e.g., 80, 90, 99).
- Requires `(percentage * prev_price) % 100 ≠ 0`, which is true for the vast majority of real gas prices.
- Any unprivileged user can trigger this by submitting a transaction with `max_price_per_unit = ⌊percentage * prev_price / 100⌋`.
- No special account, contract, or state is required.

---

### Recommendation

Replace `.to_integer()` (floor) with ceiling division so the threshold is never lower than the true fractional minimum:

```rust
// Before (floor):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();

// After (ceiling):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

`Ratio::ceil()` returns the smallest integer ≥ the rational value, which correctly enforces that any gas price strictly below the fractional threshold is rejected.

---

### Proof of Concept

- `min_gas_price_percentage = 99`, `previous_block_l2_gas_price = 101`
- `gas_price_threshold_multiplier = Ratio::new(99, 100)`
- `product = Ratio::new(99 * 101, 100) = Ratio::new(9999, 100)`
- `threshold = 9999 / 100 = 99` (floor)
- Submit transaction with `max_price_per_unit = GasPrice(99)`
- Check: `99 < 99` → `false` → transaction **passes** gateway admission
- True threshold: `⌈9999/100⌉ = 100` → transaction should **fail**
- The transaction enters the mempool and is sequenced with a gas price the protocol intended to block. [3](#0-2) [4](#0-3)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
