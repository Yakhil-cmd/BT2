### Title
Missing Lower-Bound Validation on `threshold` in FROST Signing Allows Single-Participant Signature Forgery - (`File: src/frost/mod.rs`)

---

### Summary

`assert_sign_inputs` and `presign` in `src/frost/mod.rs` validate only the upper bound of the `threshold` parameter (`threshold <= participants.len()`) but omit the lower-bound check (`threshold >= 2`) that is consistently enforced in every DKG entry point. A malicious library caller can pass `threshold = 1` (or `0`) to the FROST signing functions, causing `KeyPackage::new` to be constructed with `min_signers = 1`, which allows a single participant to unilaterally produce a valid threshold signature without the required quorum.

---

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces both bounds on `threshold`:

```rust
// src/dkg.rs:573-582
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

`assert_sign_inputs` in `src/frost/mod.rs` enforces **only** the upper bound:

```rust
// src/frost/mod.rs:144-150
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no lower-bound check
``` [2](#0-1) 

The same omission exists in `presign` in the same file:

```rust
// src/frost/mod.rs:71-77
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← no lower-bound check
``` [3](#0-2) 

`ReconstructionLowerBound` is a plain `usize` newtype with no internal invariant, so any value including `0` or `1` is accepted: [4](#0-3) 

The unchecked `threshold` flows directly into `construct_key_package`, which passes it as `min_signers` to `KeyPackage::new`:

```rust
// src/frost/redjubjub/sign.rs:249-257
let key_package = KeyPackage::new(
    identifier,
    signing_share,
    verifying_share,
    verifying_key,
    u16::try_from(threshold.value())...,
);
``` [5](#0-4) 

The same `construct_key_package` helper is called in both the coordinator path (`do_sign_coordinator`) and the participant path (`do_sign_participant`): [6](#0-5) [7](#0-6) 

With `min_signers = 1`, the underlying FROST library requires only one signature share to aggregate a valid signature, so the coordinator alone can produce a fully valid output without any other participant.

The same pattern applies to the EdDSA path via `src/frost/eddsa/sign.rs`, which also calls `assert_sign_inputs`. [8](#0-7) 

---

### Impact Explanation

A caller who controls the `threshold` argument to `frost::redjubjub::sign::sign` or `frost::eddsa::sign::sign` can set it to `1`. The FROST aggregation step then accepts a single signature share as sufficient, allowing the coordinator (or any single participant acting as coordinator) to produce a cryptographically valid threshold signature for an arbitrary message without the required quorum of honest participants. This constitutes **unauthorized creation of a valid threshold signature for attacker-chosen inputs** — the highest severity class in the allowed impact scope.

---

### Likelihood Explanation

The `threshold` parameter is supplied at call time by the library caller and is not bound to the `KeygenOutput` struct (which carries only `private_share` and `public_key`, not the original threshold). Any participant who controls the invocation of the signing function — including a malicious coordinator — can freely choose this value. No cryptographic capability or key material beyond what a normal participant already holds is required.

---

### Recommendation

Add the same lower-bound guard that `assert_key_invariants` already applies, inside `assert_sign_inputs` and `presign`:

```rust
// src/frost/mod.rs — assert_sign_inputs
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}

// src/frost/mod.rs — presign
if args.threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: args.threshold.value(),
        min: 2,
    });
}
```

Alternatively, encode the invariant directly in `ReconstructionLowerBound` by making its constructor private and requiring `value >= 2` at construction time, so the type system prevents invalid values from ever reaching protocol logic.

---

### Proof of Concept

1. Run a legitimate DKG with `n = 3` participants and `threshold = 2`, producing valid `KeygenOutput` for each participant.
2. Call `frost::redjubjub::sign::sign(participants, /*threshold=*/1usize, me, coordinator, keygen_output, presignature, message, randomizer)`.
3. `assert_sign_inputs` accepts `threshold = 1` (passes the only check: `1 <= 3`).
4. `construct_key_package` builds a `KeyPackage` with `min_signers = 1`.
5. The coordinator runs `aggregate` with only its own signature share; FROST accepts it because `min_signers = 1`.
6. A valid, verifiable threshold signature is returned — produced by one participant, bypassing the `t = 2` quorum established at key generation.

/// Verifies that the sign inputs are valid
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
```
