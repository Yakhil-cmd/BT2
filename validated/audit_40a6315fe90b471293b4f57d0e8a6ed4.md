### Title
Attacker-Controlled Cadence Event Fields Allow Arbitrary Address Injection into Victim's Transaction History Index - (`module/state_synchronization/indexer/extended/account_transactions.go`)

### Summary

`extractAddresses` unconditionally extracts every `cadence.Address` and every non-nil `cadence.Optional<Address>` value from every field of every event emitted by a transaction, with no check that the extracted address is a genuine participant in that transaction. Because any Flow account can deploy a contract that emits events with arbitrary address values, an attacker can cause any victim address to be indexed as `TransactionRoleInteracted` for the attacker's own transaction, permanently polluting the victim's transaction history as returned by `GetAccountTransactions`.

### Finding Description

`ProcessBlockData` iterates over all events emitted by each transaction and calls `extractAddresses` on each one: [1](#0-0) 

`extractAddresses` handles `cadence.Optional` by type-asserting the inner value to `cadence.Address` and appending it unconditionally: [2](#0-1) 

The only guard before assigning `TransactionRoleInteracted` is `chain.IsValid(addr)`, which merely validates the address format for the current chain — it does not verify that the address is a signer, payer, proposer, or any genuine execution-layer participant of the transaction: [3](#0-2) 

The indexed entries are persisted and later served verbatim by `GetAccountTransactions` through the Access API backend: [4](#0-3) 

which is exposed over the REST experimental endpoint: [5](#0-4) 

### Impact Explanation

An attacker can permanently insert their own transaction ID into any victim account's transaction history with role `interacted`. Any client querying `GET /v1/experimental/accounts/{victim}/transactions` will receive the attacker's transaction in the result set. The victim's actual on-chain assets and capabilities are unaffected, but the integrity of the account-transaction index — the sole source of truth for the Access API's history view — is violated without any authorization from the victim.

### Likelihood Explanation

The attack requires only:
1. A funded Flow account (no special privileges).
2. Deploying a contract that declares an event with an `Optional<Address>` field.
3. Executing a transaction that emits that event with the victim's address as the field value.

All three steps are standard, permissionless Flow operations executable on any network including localnet/emulator. No staked node control, key compromise, or social engineering is required.

### Recommendation

In `ProcessBlockData`, restrict which events are eligible for address extraction to only those emitted by system contracts (e.g., `FungibleToken.Deposited`, `FungibleToken.Withdrawn`, core Flow events), or alternatively, cross-reference extracted addresses against the transaction's declared signers/payer/proposer and discard any address that does not appear there. A simpler short-term fix is to only process events whose type prefix matches a known, trusted contract address, rejecting user-deployed contract events entirely for the purpose of `TransactionRoleInteracted` indexing.

### Proof of Concept

```cadence
// AttackerContract.cdc — deployed by attacker account 0xATTACKER
pub contract AttackerContract {
    pub event Poison(recipient: Address?)

    pub fun fire(victim: Address) {
        emit Poison(recipient: victim)
    }
}
```

```cadence
// attacker_tx.cdc — submitted by 0xATTACKER
import AttackerContract from 0xATTACKER

transaction(victim: Address) {
    prepare(acct: AuthAccount) {
        AttackerContract.fire(victim: victim)
    }
}
```

After the block is sealed and indexed:

```
GET /v1/experimental/accounts/0xVICTIM/transactions
```

Response will include the attacker's `txID` with `"roles": ["interacted"]`, despite the victim having no involvement in the transaction whatsoever. The invariant that `TransactionRoleInteracted` is only assigned to genuine execution-layer participants is broken. [6](#0-5)

func (a *AccountTransactions) extractAddresses(event flow.Event) ([]flow.Address, error) {
	cadenceEvent, err := events.DecodePayload(event)
	if err != nil {
		return nil, fmt.Errorf("failed to decode event payload: %w", err)
	}

	addresses := make([]flow.Address, 0)

	fields := cadence.FieldsMappedByName(cadenceEvent)
	for _, field := range fields {
		switch v := field.(type) {
		case cadence.Address:
			addresses = append(addresses, flow.Address(v))
		case cadence.Optional:
			if v.Value == nil {
				continue
			}

			addr, ok := v.Value.(cadence.Address)
			if !ok {
				continue
			}

			addresses = append(addresses, flow.Address(addr))
		}
	}
	return addresses, nil

func (b *AccountTransactionsBackend) GetAccountTransactions(
	ctx context.Context,
	address flow.Address,
	limit uint32,
	cursor *accessmodel.AccountTransactionCursor,
	filter AccountTransactionFilter,
	expandOptions AccountTransactionExpandOptions,
	encodingVersion entities.EventEncodingVersion,
) (*accessmodel.AccountTransactionsPage, error) {
	limit, err := b.normalizeLimit(limit)
	if err != nil {
		return nil, status.Errorf(codes.InvalidArgument, "invalid limit: %v", err)
	}

	if !b.chain.IsValid(address) {
		return nil, status.Errorf(codes.NotFound, "account %s is not valid on chain %s", address, b.chain.ChainID())
	}
	// TODO: check if account exists for the chain

	iter, err := b.store.ByAddress(address, cursor)
	if err != nil {
		return nil, mapReadError(ctx, "account transactions", err)
	}

	collected, nextCursor, err := iterator.CollectResults(iter, limit, filter.Filter())
	if err != nil {
		err = fmt.Errorf("error collecting transactions: %w", err)
		irrecoverable.Throw(ctx, err)
		return nil, err
	}

	page := accessmodel.AccountTransactionsPage{
		Transactions: collected,
		NextCursor:   nextCursor,
	}

	for i := range page.Transactions {
		tx := &page.Transactions[i]
		if err := b.expand(ctx, tx, expandOptions, encodingVersion); err != nil {
			err = fmt.Errorf("unexpected error expanding transaction: %w", err)
			irrecoverable.Throw(ctx, err)
			return nil, err
		}
	}

	return &page, nil
}

func GetAccountTransactions(r *common.Request, backend extended.API, link models.LinkGenerator) (any, error) {
	req, err := request.NewGetAccountTransactions(r)
	if err != nil {
		return nil, common.NewBadRequestError(err)
	}

	expandOptions := extended.AccountTransactionExpandOptions{
		Transaction: r.Expands("transaction"),
		Result:      r.Expands("result"),
	}

	page, err := backend.GetAccountTransactions(r.Context(), req.Address, req.Limit, req.Cursor, req.Filter, expandOptions, entities.EventEncodingVersion_JSON_CDC_V0)
	if err != nil {
		return nil, err
	}

	resp := models.AccountTransactionsResponse{
		Transactions: make([]models.AccountTransaction, len(page.Transactions)),
	}
	for i := range page.Transactions {
		err := resp.Transactions[i].Build(&page.Transactions[i], link)
		if err != nil {
			return nil, common.NewRestError(http.StatusInternalServerError, "failed to build transaction", err)
		}
	}
	if page.NextCursor != nil {
		resp.NextCursor, err = request.EncodeAccountTransactionCursor(page.NextCursor)
		if err != nil {
			return nil, common.NewRestError(http.StatusInternalServerError, "failed to encode next cursor", err)
		}
	}

	return resp, nil
