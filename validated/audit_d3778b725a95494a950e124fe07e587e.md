### Title
Unprivileged caller can permanently freeze all stakers' block rewards by calling `update_rewards` with `disable_rewards = true` — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the `Staking` contract is publicly callable with no access control. It unconditionally writes the current block number to the global `last_reward_block` storage slot **before** checking `disable_rewards`, consuming the block's single reward slot without distributing any rewards. Any unprivileged caller can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to permanently destroy that block's rewards for every staker in the protocol.

---

### Finding Description

`update_rewards` serves two distinct purposes:

1. **Bookkeeping**: record the current block as processed by writing `last_reward_block`.
2. **Distribution**: compute and forward block rewards to the staker and their pools.

The one-call-per-block gate and the bookkeeping write happen unconditionally, **before** the early-return guard:

```cairo
// src/staking/staking.cairo  lines ~1453-1507
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
// ...
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← always written

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing
}
// ... reward calculation and distribution follow
```

`last_reward_block` is a **single global slot** (not per-staker):

```cairo
// src/staking/staking.cairo  line ~187
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
```

The only gate on the function is `general_prerequisites()`, which only checks pause state and non-zero caller — no role or authorization check:

```cairo
// src/staking/staking.cairo  lines ~1794-1797
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

**Attack path:**

1. Attacker calls `update_rewards(any_active_staker, disable_rewards: true)` in block N.
2. `last_reward_block` is set to N; no rewards are distributed.
3. Every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`.
4. Block N's rewards are permanently lost for **all** stakers and their pool members.

This is the direct analog to the Union Finance H-2 bug: a function that serves dual purposes (slot-consumption bookkeeping **and** reward distribution) exits early for one purpose while still completing the other, causing participants to permanently lose yield.

---

### Impact Explanation

`last_reward_block` is global. A single call per block is sufficient to prevent every staker in the protocol from receiving rewards for that block. Because the slot is consumed and cannot be reclaimed, the rewards are **permanently destroyed** — not merely delayed. Pool members' `cumulative_rewards_trace` is never updated for that block, so their `calculate_rewards` computation will silently skip it.

This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators.

---

### Likelihood Explanation

The attack requires no privilege, no capital, and no special setup — only the gas cost of one transaction per block. The attacker needs only to front-run (or simply call before) the legitimate consensus caller. On Starknet, where transaction ordering is deterministic and public, this is straightforward. The attack is sustainable indefinitely.

---

### Recommendation

Add an authorization check so that only the designated consensus caller (or a specific role) may invoke `update_rewards`. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_rewards_updater(); // new role check
    // ...
}
```

Alternatively, move the `last_reward_block.write` **after** the `disable_rewards` guard so that a disabled call does not consume the block's reward slot, allowing a legitimate caller to still distribute rewards in the same block.

---

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(alice, disable_rewards=true)
    → last_reward_block := N
    → returns early, no rewards distributed

  Consensus tx: update_rewards(alice, disable_rewards=false)
    → assert!(N > N)  ← PANICS: REWARDS_ALREADY_UPDATED

Block N rewards for all stakers: permanently lost.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
