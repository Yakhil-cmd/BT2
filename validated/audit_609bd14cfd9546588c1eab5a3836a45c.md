### Title
Arbitrary Cadence Contract Events Allow Unauthorized Injection of Victim Addresses into Account Transaction Index — (`module/state_synchronization/indexer/extended/account_transactions.go`)

---

### Summary

`extractAddresses` blindly extracts every `cadence.Address` and `cadence.Optional<cadence.Address>` field from **any** event emitted by a transaction, regardless of the event's source contract or type. Because there is no allowlist or source-contract check, an attacker who controls any deployed contract can emit a custom event containing a victim's address and cause the victim's account to appear in the extended account-transaction index for a transaction the victim never signed or authorized. The result is served verbatim by `GET /accounts/{victim}/transactions` with role `"interacted"`.

---

### Finding Description

`ProcessBlockData` iterates over every event associated with a transaction and calls `extractAddresses` on each one:

```go
// account_transactions.go lines 172–190
for _, event := range eventsByTxIndex[txIndex] {
    eventAddresses, err := a.extractAddresses(event)
    ...
    for _, addr := range eventAddresses {
        ...
        if chain.IsValid(addr) {
            addRole(addrRoles, addr, access.TransactionRoleInteracted)
        }
    }
}
```

`extractAddresses` performs no filtering on event type or emitting contract:

```go
// account_transactions.go lines 218–235
fields := cadence.FieldsMappedByName(cadenceEvent)
for _, field := range fields {
    switch v := field.(type) {
    case cadence.Address:
        addresses = append(addresses, flow.Address(v))
    case cadence.Optional:
        ...
        addresses = append(addresses, flow.Address(addr))
    }
}
```

The sole guard is `chain.IsValid(addr)`, which only verifies the address is well-formed for the current network — it does **not** verify that the address signed, authorized, or was otherwise a party to the transaction. [1](#0-0) [2](#0-1) 

The design intent is to capture legitimate FT/NFT transfer recipients (e.g., `FungibleToken.Deposited` with a `to: Address` field). However, the implementation applies the same logic to **every** event from **every** contract, including arbitrary user-deployed contracts.

The `TransactionRoleInteracted` role is defined as "Account is referenced by an event in the transaction" — the code faithfully implements that definition, but the definition itself creates the attack surface when applied to untrusted user events. [3](#0-2) 

The REST handler `GetAccountTransactions` reads directly from this index and returns the result to any caller: [4](#0-3) 

---

### Impact Explanation

Any attacker with a deployed contract (no special privileges required) can inject an arbitrary victim address into the victim's account transaction history in the extended index. The victim's `GET /accounts/{victim}/transactions` response will include the attacker's transaction with role `"interacted"`, violating the invariant that only transactions signed or authorized by an account appear in that account's index. This constitutes unauthorized mutation of the victim's account state in the extended index.

---

### Likelihood Explanation

The preconditions are minimal: the attacker needs only a funded account to deploy a contract and submit a transaction — both are normal, permissionless operations on Flow. No privileged access, leaked keys, or node compromise is required. The attack is repeatable at negligible cost per victim address injected.

---

### Recommendation

Restrict `extractAddresses` (or its callers) to a known allowlist of trusted event types — specifically the system/protocol events and well-known standard contract events (e.g., `FungibleToken.Deposited`, `NonFungibleToken.Deposited`) whose `Address` fields carry semantic meaning about account participation. User-deployed contract events should not contribute addresses to the `TransactionRoleInteracted` index unless the event type is explicitly trusted.

Alternatively, restrict indexing to events whose emitting contract address matches the transaction's authorizers, payer, or proposer, so that only events from contracts the transaction's signers control can influence the index.

---

func (a *AccountTransactions) extractAddresses(event flow.Event) ([]flow.Address, error) {
	cadenceEvent, err := events.DecodePayload(event)
	if err != nil {
		return nil, fmt.Errorf("failed to decode event payload: %w", err)
	}

	addresses := make([]flow.Address, 0)

	fields := cadence.FieldsMappedByName(cadenceEvent)
	for _, field := range fields {
		switch v := field.(type) {
		case cadence.Address:
			addresses = append(addresses, flow.Address(v))
		case cadence.Optional:
			if v.Value == nil {
				continue
			}

			addr, ok := v.Value.(cadence.Address)
			if !ok {
				continue
			}

			addresses = append(addresses, flow.Address(addr))
		}
	}
	return addresses, nil

func GetAccountTransactions(r *common.Request, backend extended.API, link commonmodels.LinkGenerator) (any, error) {
	address, err := parser.ParseAddress(r.GetVar("address"), r.Chain)
	if err != nil {
		return nil, common.NewBadRequestError(err)
	}

	var limit uint32
	if raw := r.GetQueryParam("limit"); raw != "" {
		parsed, err := strconv.ParseUint(raw, 10, 32)
		if err != nil {
			return nil, common.NewBadRequestError(fmt.Errorf("invalid limit: %w", err))
		}
		limit = uint32(parsed)
	}

	var cursor *accessmodel.AccountTransactionCursor
	if raw := r.GetQueryParam("cursor"); raw != "" {
		c, err := parseCursor(raw)
		if err != nil {
			return nil, common.NewBadRequestError(err)
		}
		if c.BlockHeight != 0 || c.TransactionIndex != 0 {
			cursor = c
		}
	}

	var filter extended.AccountTransactionFilter
	if raw := r.GetQueryParam("roles"); raw != "" {
		roles := strings.SplitSeq(raw, ",")
		for role := range roles {
			parsed, err := accessmodel.ParseTransactionRole(strings.TrimSpace(role))
			if err != nil {
				return nil, common.NewBadRequestError(fmt.Errorf("invalid role: %w", err))
			}
			filter.Roles = append(filter.Roles, parsed)
		}
	}

	expandOptions := extended.AccountTransactionExpandOptions{
		Transaction: r.Expands("transaction"),
		Result:      r.Expands("result"),
	}

	page, err := backend.GetAccountTransactions(r.Context(), address, limit, cursor, filter, expandOptions, entities.EventEncodingVersion_JSON_CDC_V0)
	if err != nil {
		return nil, err
	}
