Audit Report

## Title
Unrestricted Third-Party Buyer Registration Enables Fake Participant Inflation and Neurons' Fund Drain in SNS Swap — (File: `rs/sns/swap/src/swap.rs`)

## Summary
The SNS Swap canister's `refresh_buyer_tokens` endpoint accepts an arbitrary `buyer` principal from the caller without verifying `caller == buyer`. Because `min_participation_reached()` counts `self.buyers.len()` with no restriction on principal type or origin, a single attacker-controlled canister can pre-fund N distinct subaccounts and register N fake principals as buyers, artificially satisfying `min_participants`. When combined with meeting `min_direct_participation_icp_e8s`, this forces the swap to commit, triggering `settle_neurons_fund_participation` and causing the NNS to mint and transfer real ICP maturity to the SNS treasury under false pretenses.

## Finding Description

**Root cause — no caller-equals-buyer enforcement:**

In `rs/sns/swap/canister/canister.rs`, the `refresh_buyer_tokens` handler extracts the buyer principal from the request argument with no check that it matches the caller:

```rust
let p: PrincipalId = if arg.buyer.is_empty() {
    caller_principal_id()
} else {
    PrincipalId::from_str(&arg.buyer).unwrap()  // any principal accepted
};
``` [1](#0-0) 

The core function `refresh_buyer_token_e8s` then queries the ICP ledger balance of the subaccount derived from `buyer` (not from the caller) and credits that amount to `buyer` in `self.buyers`:

```rust
let account = Account {
    owner: this_canister.get().0,
    subaccount: Some(principal_to_subaccount(&buyer)),
};
``` [2](#0-1) 

**Commit condition — counts all principals without restriction:**

`sufficient_participation()` requires both `min_participation_reached()` AND `min_direct_participation_icp_e8s_reached()`: [3](#0-2) 

`min_participation_reached()` counts `self.buyers.len()` with no check on whether principals are genuine users or canister-controlled identities: [4](#0-3) 

`can_commit()` fires when `sufficient_participation()` is true and the swap is due or the ICP target is reached: [5](#0-4) 

**Only guard is anonymous-principal rejection — trivially bypassed:**

`new_sale_ticket` only rejects the anonymous principal; any canister principal passes: [6](#0-5) 

**Attack path:**
1. Attacker pre-funds N subaccounts on the ICP ledger: `Account { owner: swap_canister, subaccount: principal_to_subaccount(fake_principal_i) }` for N distinct fake principals, each with at least `min_participant_icp_e8s` ICP. Total ICP spent must also satisfy `min_direct_participation_icp_e8s`.
2. Attacker calls `refresh_buyer_tokens(buyer = fake_principal_i)` for each, inserting each into `self.buyers`.
3. `self.buyers.len() >= min_participants` and `current_direct_participation_e8s >= min_direct_participation_icp_e8s` → `sufficient_participation()` returns `true`.
4. At swap due time, `try_commit()` succeeds.
5. `finalize()` calls `settle_neurons_fund_participation()`, which sends a `Committed` request to NNS governance.
6. NNS governance mints ICP maturity and transfers it to the SNS treasury. [7](#0-6) 

**NNS governance minting path confirmed:** [8](#0-7) 

**SNS init validation does NOT enforce `min_direct_participation_icp_e8s >= min_participants * min_participant_icp_e8s`** — it only enforces `max_direct_participation_icp_e8s >= min_participants * min_participant_icp_e8s`: [9](#0-8) 

This means a malicious SNS can set `min_direct_participation_icp_e8s` to its minimum (> 0) while setting `min_participants` to a large number, making the participant-count threshold the binding constraint — and the attacker can satisfy it cheaply with fake principals.

## Impact Explanation

**Concrete impact: Neurons' Fund ICP maturity drain (High — Significant SNS/NF security impact with concrete protocol harm).**

When the swap commits with fake participants, `settle_neurons_fund_participation` causes the NNS governance canister to mint ICP from Neurons' Fund maturity and transfer it to the SNS treasury. This is a direct, irreversible ledger event — real ICP is minted and sent to a project that did not achieve genuine decentralization. The reserved NF maturity (computed at SNS creation time, potentially millions of ICP in value) is permanently converted and transferred. The attacker's cost is bounded by `min_direct_participation_icp_e8s` ICP, which can be set to a low value by the malicious SNS project. The attacker does not need to recover the ICP — the goal is to drain NF maturity into a treasury they control as the SNS project.

## Likelihood Explanation

- The entry path requires no privileged role beyond having an approved `CreateServiceNervousSystem` NNS proposal (a normal governance process, not a majority-corruption assumption).
- The attack is fully automatable from a single canister using a loop over derived subaccounts.
- The economic cost is `min_direct_participation_icp_e8s` ICP (set by the malicious SNS project itself, subject only to being > 0).
- No existing check in `refresh_buyer_token_e8s` prevents canister-based multi-account registration; the only guard (`is_anonymous()`) is trivially bypassed.
- The attack is repeatable across any SNS swap with `neurons_fund_participation` enabled.

## Recommendation

1. **Enforce `caller == buyer`** in `refresh_buyer_tokens`: when `arg.buyer` is non-empty and differs from `caller_principal_id()`, reject the call. This prevents a single canister from registering arbitrary principals as buyers.
2. **Restrict canister-principal participation**: in both `new_sale_ticket` and `refresh_buyer_token_e8s`, reject callers whose principal is a canister ID (opaque principal type), permitting only self-authenticating principals.
3. **Add a lower bound on `min_direct_participation_icp_e8s`** relative to `min_participants * min_participant_icp_e8s` in `validate_participation_constraints`, so the ICP cost of satisfying both thresholds is non-trivial.

## Proof of Concept

```rust
// Attacker canister pseudo-code
async fn inflate_and_drain(swap: CanisterId, icp_ledger: CanisterId,
                            n: u64, min_participant_icp_e8s: u64) {
    for i in 0..n {
        let fake_principal = PrincipalId::new_user_test_id(i); // distinct, uncontrolled
        let subaccount = principal_to_subaccount(&fake_principal);
        // Step 1: fund the subaccount
        icp_ledger.transfer(Account { owner: swap.into(), subaccount: Some(subaccount) },
                            min_participant_icp_e8s).await;
        // Step 2: register fake_principal as buyer — no caller==buyer check
        swap.refresh_buyer_tokens(RefreshBuyerTokensRequest {
            buyer: fake_principal.to_text(),
            confirmation_text: None,
        }).await;
    }
    // self.buyers.len() == n >= min_participants
    // current_direct_participation_e8s >= min_direct_participation_icp_e8s
    // sufficient_participation() == true → swap commits at due time
    // settle_neurons_fund_participation() mints NF ICP to SNS treasury
}
```

    pub fn min_participation_reached(&self) -> bool {
        if let (Some(params), Some(init)) = (&self.params, &self.init) {
            if init.neurons_fund_participation.is_some() {
                // Only count direct participants for determining swap's success.
                // Note that a valid Swap Init should either have `neurons_fund_participation` or
                // `cf_participants`, but not both at the same time; here, we defensively perform
                // the check again anyway.
                if !self.cf_participants.is_empty() {
                    log!(
                        ERROR,
                        "Inconsistent Swap Init: cf_participants has {} elements (starting with \
                        {:?}) while neurons_fund_participation is set.",
                        self.cf_participants.len(),
                        self.cf_participants[0],
                    );
                }
                (self.buyers.len() as u32) >= params.min_participants
            } else {
                (self.cf_participants.len().saturating_add(self.buyers.len()) as u32)
                    >= params.min_participants
            }
        } else {
            false
        }
    }
