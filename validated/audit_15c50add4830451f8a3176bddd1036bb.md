### Title
Silent Zero Return in `compute_rewards_per_unit` Permanently Freezes STRK Rewards in Pool Contract - (File: src/pool/pool.cairo)

### Summary
When a pool's total delegated balance is below `min_delegation_for_rewards`, the pool's internal `compute_rewards_per_unit` silently returns `Zero::zero()` instead of reverting. The staking contract has already transferred real STRK tokens to the pool contract before calling `update_rewards_from_staking_contract`. Because the cumulative sigma index is not incremented, those STRK tokens are permanently unclaimable by any delegator — they are frozen in the pool contract forever.

### Finding Description
The pool contract's internal `compute_rewards_per_unit` function (line 967–978 of `src/pool/pool.cairo`) guards against overflow by returning zero when `total_stake < self.min_delegation_for_rewards.read()`:

```cairo
fn compute_rewards_per_unit(
    self: @ContractState, staking_rewards: Amount, total_stake: Amount,
) -> Index {
    // Return zero if the total stake is too small, to avoid overflow below.
    if total_stake < self.min_delegation_for_rewards.read() {
        return Zero::zero();
    }
    ...
}
``` [1](#0-0) 

This function is called inside `update_rewards_from_staking_contract`, which appends `last + compute_rewards_per_unit(...)` to the cumulative rewards trace:

```cairo
fn update_rewards_from_staking_contract(
    ref self: ContractState, rewards: Amount, pool_balance: Amount,
) {
    self.assert_caller_is_staking_contract();
    let (_, last) = self.cumulative_rewards_trace.last().unwrap();
    self.cumulative_rewards_trace.insert(
        key: self.get_current_epoch(),
        value: last + self.compute_rewards_per_unit(
            staking_rewards: rewards, total_stake: pool_balance,
        ),
    );
}

The staking contract's `update_pool_rewards` **first** transfers STRK tokens to the pool, **then** calls `update_rewards_from_staking_contract`:

```cairo

...

    rewards: pool_rewards, pool_balance: pool_balance.to_native_amount(:decimals),

When `pool_balance < min_delegation_for_rewards` (e.g., `< 10^18` fractions for STRK), the sigma index is not incremented. The STRK tokens already transferred to the pool contract are permanently unclaimable because all reward claims are gated on the sigma delta. The code's own comment acknowledges this: *"Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case."* [4](#0-3) 

The `min_for_rewards` for STRK is `10^18` (1 STRK): [5](#0-4) 

### Impact Explanation
STRK reward tokens transferred to the pool contract when `pool_balance < min_delegation_for_rewards` are permanently frozen. No delegator can ever claim them — `claim_rewards` computes rewards as `amount * (sigma_until - sigma_from)`, and since sigma never increased, the delta is zero. There is no recovery function in the pool contract. This constitutes **permanent freezing of unclaimed yield** (High severity per the allowed impact scope).

### Likelihood Explanation
This is reachable by any unprivileged delegator. A delegator entering a pool with `min_for_rewards - 1` fractions of STRK (e.g., `10^18 - 1`) is sufficient to trigger the condition for the entire pool's epoch rewards. The flow test `delegate_min_strk_for_rewards_flow_test` confirms this is a reachable, tested path: [6](#0-5) 

Additionally, if a pool's total balance naturally drops below the threshold (e.g., most delegators exit), all remaining delegators silently lose their epoch rewards while the STRK tokens are frozen in the pool.

### Recommendation
The staking contract should not transfer STRK tokens to the pool when the pool balance is below `min_delegation_for_rewards`. The check should be moved upstream into `calculate_staker_pools_rewards` or `update_pool_rewards` so that `pool_rewards` is set to zero (and no transfer occurs) when `pool_balance < min_delegation_for_rewards`. Alternatively, `update_rewards_from_staking_contract` should revert when `compute_rewards_per_unit` returns zero but `rewards > 0`, preventing the silent token lock.

### Proof of Concept
1. Staker stakes and opens a STRK delegation pool.
2. Delegator enters the pool with `Pool::STRK_CONFIG.min_for_rewards - 1` = `10^18 - 1` fractions of STRK.
3. Staker attests; the staking contract calculates non-zero `pool_rewards` (proportional to the pool's share of total stake).
4. `update_pool_rewards` transfers `pool_rewards` STRK to the pool contract.
5. `update_rewards_from_staking_contract` is called; `compute_rewards_per_unit` returns `0` because `pool_balance < min_delegation_for_rewards`.
6. The cumulative sigma trace is updated with `last + 0 = last` — no change.
7. The delegator calls `claim_rewards`; receives `0` because `sigma_until == sigma_from`.
8. The STRK tokens transferred in step 4 remain permanently locked in the pool contract with no recovery path.

            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {

fn delegate_min_strk_for_rewards_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    let stake_amount = system.staking.get_min_stake();
    let staker = system.new_staker(amount: stake_amount);
    system.stake(:staker, amount: stake_amount, pool_enabled: true, commission: Zero::zero());
    let pool = system.staking.get_pool(:staker);

    // Setup delegator.
    let delegate_amount = Pool::STRK_CONFIG.min_for_rewards;
    let delegator = system.new_delegator(amount: delegate_amount);

    // Enter pool with less than min STRK for rewards.
    system.delegate(:delegator, :pool, amount: delegate_amount - 1);

    // Attest.
    system.advance_k_epochs_and_attest(:staker);
    system.advance_epoch();

    // Check rewards.
    system.advance_epoch();
    let rewards = system.delegator_claim_rewards(:delegator, :pool);
    assert!(rewards == Zero::zero());
