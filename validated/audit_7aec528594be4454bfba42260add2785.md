### Title
Global `last_reward_block` Allows Any Caller to Block Per-Block Reward Distribution for All Stakers - (File: `src/staking/staking.cairo`)

### Summary

`update_rewards` in `staking.cairo` uses a single **global** `last_reward_block` storage variable to enforce a one-call-per-block limit. Because this variable is shared across all stakers and the function lacks enforced access control in code (despite the spec stating "Only starkware sequencer"), any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` for any valid staker address, consuming the per-block slot without distributing rewards. Any subsequent legitimate call in the same block — including the sequencer's reward-distributing call — reverts with `REWARDS_ALREADY_UPDATED`, causing all stakers to lose that block's rewards.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` reads and writes a single global `last_reward_block`:

```cairo
// src/staking/staking.cairo:1453-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After passing the check, `last_reward_block` is unconditionally written **before** the `disable_rewards` branch:

```cairo
// src/staking/staking.cairo:1484-1488
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

This means a call with `disable_rewards: true` updates `last_reward_block` to the current block and returns immediately without distributing any rewards. Any subsequent call in the same block — including the sequencer's legitimate call with `disable_rewards: false` — hits the `REWARDS_ALREADY_UPDATED` assert and reverts.

The `last_reward_block` variable is global (not per-staker), so a single poisoning call blocks reward distribution for **every** staker in that block.

The function's only guard is `general_prerequisites()`, which checks the pause state. The flow tests confirm no sequencer-role check is enforced in code: `system.update_rewards(:staker, disable_rewards: true)` is called throughout without any `cheat_caller_address_once` setup, meaning any address can call it. ---

### Impact Explanation

An attacker calls `update_rewards(staker_address: any_valid_staker, disable_rewards: true)` once per block. This:

1. Passes the `REWARDS_ALREADY_UPDATED` guard (new block).
2. Writes `last_reward_block = current_block`.
3. Returns early — zero rewards distributed.
4. Blocks the sequencer's legitimate `update_rewards(disable_rewards: false)` call for **any** staker in the same block.

All stakers lose that block's STRK rewards. Repeated every block, this permanently freezes unclaimed yield for the entire staker set. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**. [3](#0-2) 

---

### Likelihood Explanation

- The attacker only needs to know any valid active staker address (publicly readable on-chain).
- The call requires no special role, no tokens, and no privileged access.
- The attacker submits one transaction per block; the sequencer includes it before its own system call.
- The flow tests confirm the function is reachable without any role restriction. [4](#0-3) [5](#0-4) 

---

### Recommendation

1. **Enforce sequencer-only access control** in code (not just in the spec). Add a `only_sequencer` check inside `update_rewards` so unprivileged callers cannot invoke it.
2. Alternatively, make `last_reward_block` **per-staker** rather than global, so a poisoning call for one staker does not block reward distribution for others.

---

### Proof of Concept

```
Block N:
  Tx 1 (attacker): update_rewards(staker_address: alice, disable_rewards: true)
    → last_reward_block = N, no rewards distributed, succeeds.

  Tx 2 (sequencer): update_rewards(staker_address: alice, disable_rewards: false)
    → assert!(N > N) fails → REWARDS_ALREADY_UPDATED → reverts.

  Tx 3 (sequencer): update_rewards(staker_address: bob, disable_rewards: false)
    → assert!(N > N) fails → REWARDS_ALREADY_UPDATED → reverts.

Result: All stakers lose Block N rewards. Attacker repeats every block.
```

The existing test suite already documents the same-block revert behavior:

```cairo

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
