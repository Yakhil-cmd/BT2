### Title
Lack of Access Control in `update_rewards` Allows Any Staker to Monopolize Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract has no caller access control. Any staker can invoke it with their own address every block, consuming the single per-block reward slot (`last_reward_block`) and directing all block rewards to themselves, while permanently starving every other staker of consensus-based rewards.

---

### Finding Description

The consensus-based reward path in `src/staking/staking.cairo` exposes `update_rewards` as a public, permissionless entry point:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // ← global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    ...
    self._update_rewards(
        :staker_address,
        strk_total_rewards: strk_block_rewards,
        btc_total_rewards:  btc_block_rewards,
        strk_total_stake:   staker_total_strk_balance,   // ← individual balance, not network total
        btc_total_stake:    staker_total_btc_balance,
        ...
    );
}
``` [1](#0-0) 

There is no `assert_caller_is_X` guard, no role check, and no verification that the caller is the consensus mechanism or that `staker_address` is the block-selected staker. The only guard is the per-block deduplication via `last_reward_block`.

**Contrast with the attestation-based path**, which correctly uses the full network stake as the denominator:

```cairo
let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
self._update_rewards(
    :staker_address,
    strk_total_rewards: strk_epoch_rewards,
    :strk_total_stake,          // ← total network stake
    ...
);
``` [2](#0-1) 

In the consensus path the denominator passed to `_update_rewards` is `staker_total_strk_balance` — the attacker's own balance — not the total network stake. If `_update_rewards` computes `staker_share = total_rewards × staker_balance / total_stake`, substituting the attacker's balance for both numerator and denominator yields 100 % of block rewards regardless of actual stake size.

**Secondary griefing vector**: any address (not even a staker) can call `update_rewards(any_valid_staker, disable_rewards: true)`. This writes `last_reward_block` and returns immediately without distributing rewards, silently voiding that block's reward for every participant. [3](#0-2) 

---

### Impact Explanation

| Scenario | Impact |
|---|---|
| Attacker (staker) calls `update_rewards(self, false)` every block | Receives all block rewards; every other staker receives zero — **theft of unclaimed yield** |
| Any address calls `update_rewards(valid_staker, true)` every block | No rewards distributed to anyone — **permanent freezing of unclaimed yield** |

Both impacts fall within the allowed scope:
- *High: Theft of unclaimed yield*
- *High: Permanent freezing of unclaimed yield*

---

### Likelihood Explanation

- The function is unconditionally public; no special role or key is required.
- Becoming a staker requires only the minimum stake (`min_stake`), a low economic barrier.
- The attack is fully automatable: a script calls `update_rewards(attacker_address, false)` on every new block.
- The griefing variant (`disable_rewards: true`) requires zero stake and zero cost beyond gas.
- No privileged access, bridge compromise, or external dependency is involved.

---

### Recommendation

Add an access-control guard so that only the designated consensus contract (or an authorized role) may call `update_rewards`. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.assert_caller_is_consensus_contract(); // add this guard
    ...
}
```

Alternatively, store the authorized consensus caller address in storage and assert `get_caller_address() == self.consensus_contract.read()` at the top of the function, mirroring the pattern already used in `update_rewards_from_attestation_contract`:

```cairo
self.assert_caller_is_attestation_contract();
``` [4](#0-3) 

Also fix the denominator discrepancy: the consensus path should pass `self.get_current_total_staking_power()` as `strk_total_stake`, not the individual staker's balance, to match the attestation path's proportional reward calculation.

---

### Proof of Concept

1. Attacker stakes the minimum amount via `stake(reward_addr, op_addr, min_stake)`.
2. Wait for the protocol to enter consensus-rewards mode (`consensus_rewards_first_epoch` reached).
3. Each block, attacker calls:
   ```
   staking_contract.update_rewards(attacker_staker_address, false)
   ```
4. Because `last_reward_block` is updated to the current block, no other caller can distribute rewards in the same block.
5. `_update_rewards` is invoked with `strk_total_stake = attacker_own_balance`, yielding a disproportionately large (potentially 100 %) share of `strk_block_rewards` credited to the attacker's `unclaimed_rewards_own`.
6. Attacker calls `claim_rewards(attacker_staker_address)` to withdraw the accumulated rewards.
7. All other stakers' `unclaimed_rewards_own` remain at zero for every block the attacker front-runs. [1](#0-0)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
