### Title
Unprivileged Caller Can Suppress All Consensus Rewards Per Block via `disable_rewards` Flag — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control. Any caller can pass `disable_rewards: true` to consume the global per-block reward slot (`last_reward_block`) without distributing rewards to any staker, permanently freezing unclaimed yield across the entire protocol.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` accepts a `staker_address` and a `disable_rewards: bool` parameter from any non-zero caller. The function enforces a global one-call-per-block invariant via `last_reward_block`: [1](#0-0) 

After validating the staker exists and has non-zero balance, it unconditionally writes `current_block_number` to `last_reward_block` **before** checking `disable_rewards`: [2](#0-1) 

Because `last_reward_block` is a single global storage slot shared across all stakers: [3](#0-2) 

once any caller invokes `update_rewards(any_valid_staker, disable_rewards: true)` in a given block, the slot is consumed. Every subsequent call in the same block fails the `current_block_number > last_reward_block` assertion, meaning **no staker can receive consensus rewards for that block**.

There is no access control on `update_rewards` beyond `general_prerequisites()`, which only checks the pause flag and a non-zero caller: [4](#0-3) [5](#0-4) 

The analog to the KintoWallet KYC bypass is direct: in KintoWallet, the validation is incomplete because only `owner[0]`'s KYC is checked while the other owners are unvalidated. Here, the validation is incomplete because the staker's existence is checked but the **caller's authorization to suppress reward distribution** is never checked. An unprivileged caller exploits the unvalidated `disable_rewards` path exactly as the KintoWallet attacker exploits the unvalidated owners.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_valid_staker, true)` every block prevents the entire protocol from distributing consensus rewards. Stakers and delegators accumulate zero `unclaimed_rewards_own` and zero pool rewards indefinitely. The yield is not stolen — it is simply never credited, matching the "permanent freezing of unclaimed yield" impact category.

---

### Likelihood Explanation

**High.** The function is permissionlessly callable by any EOA or contract. The attacker only needs to know one valid, active staker address (all staker addresses are emitted in `NewStaker` events and stored in the public `stakers` vector). The only cost is one transaction per block. No privileged access, leaked key, or external dependency is required.

---

### Recommendation

Restrict `update_rewards` to an authorized caller (e.g., the consensus layer contract or a designated operator role), or split the function into two: one permissionless path that always distributes rewards, and a privileged path that can disable them. At minimum, the `disable_rewards: true` branch must require an authorized caller check analogous to `assert_caller_is_attestation_contract`.

---

### Proof of Concept

1. Attacker identifies any valid, active staker address `S` (e.g., from on-chain `NewStaker` events).
2. Each block, attacker submits: `staking_contract.update_rewards(S, disable_rewards: true)`.
3. Inside `update_rewards`:
   - Staker `S` passes all validity checks (exists, active, non-zero balance).
   - `last_reward_block` is written to `current_block_number`.
   - Execution returns early due to `disable_rewards == true` — no rewards distributed.
4. Any legitimate caller (staker, delegator, consensus layer) who attempts `update_rewards` in the same block hits: `assert!(current_block_number > self.last_reward_block.read(), ...)` and reverts.
5. Repeated every block: `staker_info.unclaimed_rewards_own` for all stakers remains zero; pool `cumulative_rewards_trace` is never updated; delegators earn nothing. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
