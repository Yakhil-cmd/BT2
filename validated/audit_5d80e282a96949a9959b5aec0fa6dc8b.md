### Title
Staker Can Instantly Raise Commission to Maximum After Setting Commitment, Stealing Delegators' Unclaimed Yield During Exit Lock — (`src/staking/staking.cairo`)

---

### Summary

The `set_commission_commitment` + `set_commission` two-step mechanism in `src/staking/staking.cairo` allows a staker to increase their commission rate to any value up to `max_commission` with **zero mandatory delay** between setting the commitment and applying the increase. Because delegators are locked in for the full `exit_wait_window` (~1 week) before they can withdraw, they are forced to earn rewards at the new, higher commission rate they never agreed to. The code itself acknowledges this gap with the comment: *"Current commission increase safeguards still allow for sudden commission changes."*

---

### Finding Description

Without a commitment, `set_commission` only allows a staker to **decrease** their commission:

```
} else {
    assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
}
```

However, a staker can bypass this restriction by first calling `set_commission_commitment` to register a `CommissionCommitment { max_commission, expiration_epoch }`. Once the commitment is stored, `update_commission` permits any commission value `<= max_commission`, including a large increase:

```
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission, ...
    );
    assert!(commission != old_commission, ...);
}
```

The only temporal guard on `set_commission_commitment` is:

```
assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
```

This means `expiration_epoch = current_epoch + 1` is the minimum, and the commitment is **immediately active** the moment it is written to storage. There is no enforced waiting period before the staker can call `set_commission` to apply the increase. Both calls can be made in the same block.

The commission change is applied instantly with no epoch delay:

```
// Update commission in storage.
staker_pool_info.commission.write(Option::Some(commission));
```

And the new commission is used directly in the next reward calculation:

```
let commission = staker_pool_info.commission();
...
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
```

Delegators who wish to exit must first call `exit_intent` and then wait the full `exit_wait_window` (default ~1 week) before calling `exit_action`. During this entire window they continue to earn rewards at the new, higher commission rate.

---

### Impact Explanation

A staker who attracted delegators with a 0% commission can atomically:
1. Call `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`
2. Call `set_commission(commission: 10000)` in the same block

Commission is now 100%. Every attestation reward earned by the pool during the exit_wait_window is entirely captured by the staker as commission; delegators receive zero pool rewards. This constitutes **theft of unclaimed yield** for the duration of the lock window.

This matches the allowed High impact: *"Theft of unclaimed yield or unclaimed royalties."*

---

### Likelihood Explanation

Any registered staker who has opened a delegation pool can execute this attack. No privileged role, leaked key, or external dependency is required. The staker is an unprivileged protocol participant. The attack is two transactions, costs only gas, and is repeatable. Delegators have no on-chain mechanism to react before the commission is raised because both transactions can land in the same block.

---

### Recommendation

Enforce a minimum delay between setting a commitment and using it to increase commission. For example, require that `expiration_epoch >= current_epoch + MIN_COMMITMENT_NOTICE_EPOCHS` (e.g., `MIN_COMMITMENT_NOTICE_EPOCHS = K = 2`) **and** that the commission increase cannot be applied until at least one full epoch after the commitment was set. This gives delegators at least one epoch of notice to observe the pending increase and submit an `exit_intent` before the higher rate takes effect.

Alternatively, apply commission increases with the same K-epoch delay used for balance changes, so the new rate only takes effect at `current_epoch + K`.

---

### Proof of Concept

```
// Setup: staker opens pool at 0% commission, delegators deposit.
staker.set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1);
// Same block:
staker.set_commission(commission: 10000);  // Commission is now 100%, immediately active.

// Delegators react and call exit_intent — but must wait exit_wait_window (~1 week).
delegator.exit_intent(pool, amount);

// During exit_wait_window, staker attests each epoch:
//   calculate_staker_pools_rewards reads commission = 10000
//   split_rewards_with_commission → commission_rewards = 100%, pool_rewards = 0
// Delegators receive zero pool rewards for the entire lock period.

delegator.exit_action(pool);  // Funds returned, but all yield during lock was stolen.
```

        /// **Note**: Current commission increase safeguards still allow for sudden commission
        /// changes.
        /// **Note**: Updating epoch info can impact the commission commitment expiration date.
        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
