### Title
Missing Declared-Class Existence Check in OS `execute_replace_class` Causes Blockifier/OS State Divergence — (`crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The Starknet OS Cairo program's `execute_replace_class` function does not verify that the target class hash is declared before writing it into the contract state. The blockifier's `replace_class` implementation enforces this check (and additionally requires the replacement class to be Cairo1). When a contract calls `replace_class` with an undeclared class hash, the blockifier reverts the transaction while the OS silently succeeds and mutates state — producing divergent state diffs. The OS itself acknowledges this gap with an explicit TODO comment dated 1/1/2026.

### Finding Description

**Root cause — OS side (new syscalls):** [1](#0-0) 

`execute_replace_class` reduces gas, reads the current `StateEntry` via the `GetContractAddressStateEntry` hint, and unconditionally writes `new_state_entry` with the caller-supplied `class_hash`. The only failure path is insufficient gas. There is no lookup to confirm the class is declared:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(...);
```

**Root cause — OS side (deprecated syscalls):** [2](#0-1) 

The deprecated path has the same omission — no class-existence check before the `dict_update`.

**Blockifier enforces the check (new syscalls):** [3](#0-2) 

```rust
pub fn replace_class(&mut self, class_hash: ClassHash) -> SyscallResult<()> {
    // Ensure the class is declared (by reading it), and of type V1.
    let compiled_class = self.state.get_compiled_class(class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(SyscallExecutionError::ForbiddenClassReplacement { class_hash });
    }
    self.state.set_class_hash_at(self.call.storage_address, class_hash)?;
    Ok(())
}
```

`get_compiled_class` returns `StateError::UndeclaredClassHash` for any hash not in the declared-classes table, which propagates as a syscall failure back to the Cairo VM, causing the contract to panic and the transaction to revert.

**Blockifier enforces the check (deprecated syscalls):** [4](#0-3) 

```rust
fn replace_class(...) {
    // Ensure the class is declared (by reading it).
    syscall_handler.state.get_compiled_class(request.class_hash)?;
    syscall_handler.state.set_class_hash_at(...)?;
    Ok(ReplaceClassResponse {})
}
```

**Divergence summary:**

| Component | `replace_class(undeclared_hash)` result |
|---|---|
| Blockifier (new + deprecated) | Syscall error → Cairo panic → transaction **reverts** |
| OS Cairo program (new + deprecated) | No check → state updated → transaction **succeeds** |

The OS is the ground truth for the ZK prover. When the prover re-executes a block, it runs the OS Cairo program. For any transaction containing `replace_class(undeclared_hash)`, the OS produces a state diff where the contract's class hash is overwritten with the undeclared value, while the blockifier's committed state diff shows a reverted transaction. These two state diffs are irreconcilable.

### Impact Explanation

**Wrong class hash written into contract state / wrong revert result from syscall execution logic.**

The OS state diff produced for a block containing `replace_class(undeclared_hash)` will differ from the blockifier's committed state diff. Concretely:

- The OS writes `contract.class_hash = <undeclared_hash>` into `contract_state_changes`.
- The blockifier records the transaction as reverted, leaving `contract.class_hash` unchanged.

Depending on which component is authoritative for the on-chain state commitment:

1. **If the prover derives the committed state from the OS output** (the OS is the ground truth for the proof): the undeclared class hash is committed on L1 as the contract's live class hash. Any subsequent call to that contract will attempt to load a class that does not exist, permanently bricking the contract. This is an unauthorized, irreversible state mutation.

2. **If the prover attempts to prove the blockifier's state diff**: the OS produces a contradictory state, the proof is unprovable, and the block cannot be finalized on L1 — a liveness failure.

Either outcome is critical: unauthorized class-hash mutation or permanent block-finalization failure.

### Likelihood Explanation

- **No privileged access required.** Any deployed Cairo1 contract can call `replace_class` with an arbitrary felt value as the class hash.
- **Trivially triggerable.** A single invoke transaction calling a contract that executes `replace_class(0xdeadbeef)` (any undeclared hash) is sufficient.
- **Both new and deprecated syscall paths are affected**, covering all Cairo versions.
- The TODO comment (`// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`) confirms the gap is known and unresolved as of the codebase snapshot.

### Recommendation

In `execute_replace_class` (both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`), add a lookup into the declared-classes dictionary before writing the new `StateEntry`. The check should mirror the blockifier's logic:

1. Verify the class hash exists in the declared-classes segment (analogous to `get_compiled_class`).
2. For the new-syscalls path, additionally verify the class is a Cairo1 class (not a deprecated Cairo0 class), matching the `is_cairo1` guard in `syscall_base.rs`.

The fix should be applied to both the new-syscalls OS path (`syscall_impls.cairo:execute_replace_class`) and the deprecated-syscalls OS path (`deprecated_execute_syscalls.cairo:execute_replace_class`) to eliminate the divergence with the blockifier on all transaction versions.

### Proof of Concept

1. Deploy a Cairo1 contract `Attacker` with the following logic:
   ```cairo
   #[external(v0)]
   fn trigger(ref self: ContractState) {
       // 0xdeadbeef is not declared on-chain
       starknet::syscalls::replace_class_syscall(
           class_hash_const::<0xdeadbeef>()
       ).unwrap_syscall();
   }
   ```
2. Submit an invoke transaction calling `Attacker::trigger`.
3. **Blockifier path**: `replace_class` calls `get_compiled_class(0xdeadbeef)` → `StateError::UndeclaredClassHash` → syscall returns failure → Cairo panics → transaction reverts. `Attacker`'s class hash is unchanged in the blockifier's state diff.
4. **OS path**: `execute_replace_class` skips the existence check, writes `StateEntry { class_hash: 0xdeadbeef, ... }` into `contract_state_changes`, and returns success. The OS state diff records `Attacker.class_hash = 0xdeadbeef`.
5. The two state diffs diverge. The prover either commits an invalid class hash for `Attacker` on L1, or fails to produce a valid proof for the block. [5](#0-4) [3](#0-2) [2](#0-1) [4](#0-3)
