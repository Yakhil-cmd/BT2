### Title
STRK Reward Tokens Permanently Locked in Pool Contract When Pool Balance Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo)

### Summary

When a delegation pool's total balance is below `min_delegation_for_rewards`, the staking contract still calculates and physically transfers STRK reward tokens to the pool contract, but the pool contract's `compute_rewards_per_unit` returns zero, leaving those tokens permanently unclaimable by any pool member or admin.

### Finding Description

The staking contract's `_update_rewards` flow (via `calculate_staker_pools_rewards` → `update_pool_rewards`) calculates pool rewards proportional to `pool_balance / total_stake` and unconditionally transfers the resulting STRK tokens to the pool contract before calling `update_rewards_from_staking_contract`. [1](#0-0) 

Inside `update_rewards_from_staking_contract`, the pool contract calls `compute_rewards_per_unit`: [2](#0-1) 

`compute_rewards_per_unit` returns **zero** when `total_stake < min_delegation_for_rewards`, so the `cumulative_rewards_trace` receives a zero increment: [3](#0-2) 

The code comment at line 964 explicitly acknowledges this: *"Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case."* [4](#0-3) 

Because the `cumulative_rewards_trace` is not incremented, `claim_rewards` computes zero rewards for all pool members for that epoch. The STRK tokens already transferred to the pool contract have no recovery path — there is no sweep, rescue, or admin-withdrawal function in the pool contract. [5](#0-4) 

For STRK pools, `min_delegation_for_rewards = 10^18` (1 STRK): [6](#0-5) 

### Impact Explanation

STRK reward tokens are physically transferred into the pool contract but are permanently unclaimable. No pool member, staker, or governance role can recover them. This constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

### Likelihood Explanation

Any pool whose total delegated balance is below `min_delegation_for_rewards` (1 STRK for STRK pools; token-decimal-dependent for BTC pools) triggers this on every attestation/`update_rewards` call. This is reachable by any unprivileged delegator who enters a new or lightly-used pool with a small amount. No special role or privileged access is required.

### Recommendation

The staking contract should skip transferring rewards to a pool when the pool balance is below `min_delegation_for_rewards`, or the pool contract should track "unattributed" STRK and expose a governance-controlled recovery function. The simplest fix is to gate the `send_rewards_to_delegation_pool` + `update_rewards_from_staking_contract` call pair on `pool_balance >= min_delegation_for_rewards` inside `calculate_staker_pools_rewards` in `staking.cairo`.

### Proof of Concept

1. Staker stakes and opens a STRK delegation pool.
2. Delegator enters the pool with `amount = 0.5 STRK` (< `min_delegation_for_rewards = 1 STRK`).
3. After K epochs, staker attests or `update_rewards` is called.
4. `calculate_staker_pools_rewards` computes `pool_rewards = total_rewards * 0.5e18 / total_stake` — a non-zero STRK amount.
5. `send_rewards_to_delegation_pool` transfers that STRK to the pool contract.
6. `update_rewards_from_staking_contract(rewards, pool_balance=0.5e18)` is called; `compute_rewards_per_unit` returns 0 because `0.5e18 < 1e18`.
7. `cumulative_rewards_trace` is unchanged; `claim_rewards` returns 0 for the delegator.
8. The transferred STRK sits in the pool contract with no withdrawal path — permanently frozen.

        fn update_pool_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            pools_rewards_data: Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>,
        ) -> Array<(ContractAddress, Amount)> {
            let mut pool_rewards_list = array![];
            let strk_token_dispatcher = strk_token_dispatcher();
            for (pool_contract, token_address, pool_balance, pool_rewards) in pools_rewards_data {
                let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };
                // Rewards are always in STRK.
                self
                    .send_rewards_to_delegation_pool(
                        :staker_address,
                        pool_address: pool_contract,
                        amount: pool_rewards,
                        token_dispatcher: strk_token_dispatcher,
                    );
                let decimals = self.get_token_decimals(:token_address);
                pool_dispatcher
                    .update_rewards_from_staking_contract(
                        rewards: pool_rewards,
                        pool_balance: pool_balance.to_native_amount(:decimals),
                    );
                pool_rewards_list.append((pool_contract, pool_rewards));
            }
            pool_rewards_list
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn update_rewards_from_staking_contract(
            ref self: ContractState, rewards: Amount, pool_balance: Amount,
        ) {
            self.assert_caller_is_staking_contract();

            // `rewards_info` is initialized in the constructor or in the upgrade proccess,
            // so unwrapping should be safe.
            let (_, last) = self.cumulative_rewards_trace.last().unwrap();
            self
                .cumulative_rewards_trace
                .insert(
                    key: self.get_current_epoch(),
                    value: last
                        + self
                            .compute_rewards_per_unit(
                                staking_rewards: rewards, total_stake: pool_balance,
                            ),
                );
        }

        /// Compute the rewards for the pool trace.
        ///
        /// `staking_rewards` is in `STRK_DECIMALS` decimals.
        /// `total_stake` is in the contract's token decimals.
        /// **Note**: Delegation rewards lost when pool balance is less than
        /// `min_delegation_for_rewards`. The staking contract continues to forward
        /// `pool_rewards` to the pool contract even in this case.
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
