### Title
Unbounded `stakers` Vector Growth Enables Griefing of `get_stakers()` Consensus Query — (File: `src/staking/staking.cairo`)

---

### Summary

The `stakers` storage `Vec` in the `Staking` contract grows permanently with every new staker address and is **never pruned** when stakers exit. The `get_stakers()` consensus function iterates over the **entire** vector on every call. An unprivileged attacker can cheaply and repeatedly bloat this vector by staking with fresh addresses and then fully unstaking (recovering their principal), causing `get_stakers()` to consume unbounded gas/computation over time — directly analogous to the oracle DoS pattern where requests consume resources without financial penalty.

---

### Finding Description

**Root cause — permanent vector growth with no cleanup:**

`stake()` unconditionally appends the caller's address to `self.stakers`: [1](#0-0) 

The storage comment explicitly acknowledges that this vector is never pruned: [2](#0-1) 

`unstake_action()` deletes the staker's `staker_info` record and clears pool data, but **does not remove the address from `self.stakers`**: [3](#0-2) 

**Root cause — full-range iteration in `get_stakers()`:**

`get_stakers()` iterates over `self.stakers.into_iter_full_range()` — every slot ever written, including all exited stakers: [4](#0-3) 

For each slot it reads the address, checks `is_staker_active`, and (for active stakers) calls `get_staker_staking_power_at_epoch`, `get_public_key_at_epoch`, and `get_peer_id_at_epoch`. Even for exited stakers the slot read and `is_staker_active` check still execute, so the cost scales linearly with the total number of addresses ever staked.

**Attack path:**

1. Attacker generates N fresh addresses (each address can only stake once due to `STAKER_EXISTS` and `assert_staker_address_not_reused` guards, but fresh addresses are unlimited).
2. Each address calls `stake(min_stake)` → address is pushed to `self.stakers`.
3. Each address calls `unstake_intent()` → starts the exit window.
4. After the exit window, each address calls `unstake_action()` → principal is returned in full.
5. Repeat with new addresses.

The attacker **recovers their principal** on every cycle. The only cost is gas fees per stake/unstake round-trip — exactly the "no financial penalty" pattern from the oracle report.

Each completed cycle permanently adds one dead slot to `self.stakers`. After N cycles, every call to `get_stakers()` must iterate N extra slots.

The `DiverseStakerVecFlow` test confirms that exited stakers remain in the vector: [5](#0-4) 

---

### Impact Explanation

`get_stakers()` is the critical consensus interface used by validator nodes to determine the active staker set for a given epoch: [6](#0-5) 

As the `stakers` vector grows without bound, the gas/computation cost of `get_stakers()` grows linearly. At sufficient scale this function becomes too expensive to execute within block gas limits (if called on-chain) or causes unacceptable latency for consensus nodes (if called off-chain). Either outcome degrades or halts the protocol's ability to run consensus — constituting **unbounded gas consumption** and **griefing damage to the protocol**.

**Impact: Medium** — Unbounded gas consumption; griefing with no profit motive but damage to the protocol.

---

### Likelihood Explanation

The attack is economically viable because the attacker recovers their full principal after each stake/unstake cycle. The only recurring cost is Starknet gas fees per cycle. With a low `min_stake` setting, an attacker can bloat the vector at minimal net cost. The attack is permissionless, requires no privileged role, and can be executed from any number of addresses in parallel.

**Likelihood: Medium** — Requires sustained effort and gas spend, but the principal-recovery mechanism removes the primary financial deterrent.

---

### Recommendation

1. **Track and skip exited stakers efficiently**: Maintain a separate counter of active stakers and store a compact active-staker set (e.g., a swap-and-pop pattern or a separate active index) so `get_stakers()` only iterates live entries.
2. **Remove exited addresses from the vector**: In `unstake_action()`, swap the exiting staker's slot with the last element and decrement the vector length, keeping the vector compact.
3. **Impose a minimum stake cost that is not fully recoverable**: Introduce a small non-refundable registration fee or slash a small amount on exit to raise the per-cycle cost of the attack.
4. **Cap the stakers vector size**: Enforce a protocol-level maximum number of registered stakers to bound the worst-case iteration cost.

```
// Attacker script (pseudocode):
for i in 0..N:
    addr = fresh_address(i)
    strk.approve(staking, min_stake)
    staking.stake(reward=addr, operational=fresh_op(i), amount=min_stake)
    // addr is now in self.stakers[i] permanently
    staking.unstake_intent()           // called from addr
    advance_time(exit_wait_window)
    staking.unstake_action(addr)       // principal returned to addr
    // addr's staker_info is deleted, but self.stakers[i] == addr forever

// After N iterations:
// self.stakers.len() == N (all dead slots)
// get_stakers(epoch_id) iterates all N slots, costing O(N) gas
// Legitimate consensus nodes calling get_stakers() pay O(N) cost
// for a vector that is O(N) dead entries
```
