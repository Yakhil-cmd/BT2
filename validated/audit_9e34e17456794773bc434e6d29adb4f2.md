### Title
Missing Access Control on `update_rewards` Allows Any Caller to Steal or Permanently Freeze Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `Staking.cairo` is specified to be callable only by the Starkware sequencer, but no caller check is implemented. Any unprivileged address can call it with an arbitrary `staker_address`, causing block rewards to be distributed to a staker who did not produce the block, while permanently preventing the legitimate block producer from receiving their reward for that block.

### Finding Description
The `IStakingRewardsManager::update_rewards` function is the consensus-phase reward distribution entry point. According to the protocol specification, its access control is:

> **"Only starkware sequencer."** [1](#0-0) 

However, the implementation contains no such check. The function body only verifies:
1. The contract is not paused (`general_prerequisites()`).
2. The current block number is greater than `last_reward_block` (prevents double-calling within the same block).
3. The supplied `staker_address` exists and is active with non-zero balance. [2](#0-1) 

There is no `assert_caller_is_sequencer()` or equivalent guard. Compare this with the analogous pre-consensus function `update_rewards_from_attestation_contract`, which correctly enforces its caller restriction:

```cairo
self.assert_caller_is_attestation_contract();
``` [3](#0-2) 

The `update_rewards` function writes `last_reward_block` to the current block number unconditionally before the reward distribution logic, meaning the first caller per block wins and all subsequent calls revert with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

### Impact Explanation
**Attack vector 1 — Theft of unclaimed yield (High):**
An attacker calls `update_rewards(victim_staker, false)` in the same block that the legitimate sequencer would have called it for the actual block producer. The victim staker (who did not produce the block) receives the full block reward. The actual block producer's reward for that block is permanently lost because `last_reward_block` is already set.

**Attack vector 2 — Permanent freezing of unclaimed yield (High):**
An attacker calls `update_rewards(any_active_staker, disable_rewards: true)`. This updates `last_reward_block` to the current block but distributes zero rewards (the `disable_rewards` flag causes an early return). No staker receives rewards for that block, and the block producer's yield is permanently frozen for that block. [4](#0-3) 

### Likelihood Explanation
The function is publicly callable on-chain with no barriers. An attacker monitoring the mempool or block production can front-run the sequencer's legitimate call every block. The attack requires no special privileges, no capital, and no complex setup — only the ability to submit a transaction.

### Recommendation
Introduce a stored sequencer address and enforce it:

```cairo
// In storage:
sequencer_address: ContractAddress,

// At the top of update_rewards:
assert!(
    get_caller_address() == self.sequencer_address.read(),
    "{}",
    Error::CALLER_IS_NOT_SEQUENCER,
);
```

Alternatively, if Starknet's protocol guarantees that only the sequencer can include this call via a forced-transaction mechanism, document this explicitly and add a comment explaining why no on-chain check is needed. If no such protocol guarantee exists, the on-chain check is mandatory.

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. A new block `N` is produced by staker `B` (the legitimate block producer).
3. Attacker submits `update_rewards(staker_A, false)` in block `N`, where `staker_A` is any active staker other than `B`.
4. The call succeeds: `last_reward_block` is set to `N`, and `staker_A.unclaimed_rewards_own` is increased by the block reward.
5. The sequencer attempts `update_rewards(staker_B, false)` — this reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number == last_reward_block`.
6. Staker `B` permanently loses their block reward for block `N`. Staker `A` receives rewards they did not earn. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
