### Title
Production `create_signature_manager()` Hardcodes a Publicly-Known Testing Private Key, Binding the Wrong Signer to Every Consensus Precommit Vote — (File: `crates/apollo_signature_manager/src/lib.rs`)

### Summary
The production factory function `create_signature_manager()` unconditionally instantiates `LocalKeyStore::new_for_testing()`, which embeds a hardcoded STARK private key that is visible to anyone with access to the repository. Because the `SignatureManager` uses this key to sign consensus precommit votes and peer-identity challenges, every signature the sequencer emits is forgeable by any party who has read the source code.

### Finding Description

`create_signature_manager()` is the public, non-test production entry point for the `SignatureManager` component:

```rust
// crates/apollo_signature_manager/src/lib.rs
pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()          // → LocalKeyStore::new_for_testing()
}
``` [1](#0-0) 

`SignatureManager::new()` delegates to `LocalKeyStore::new_for_testing()`, which hard-codes a constant private key:

```rust
// crates/apollo_signature_manager/src/signature_manager.rs
pub(crate) const fn new_for_testing() -> Self {
    const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    ));
    ...
}
``` [2](#0-1) 

The `SignatureManager` uses this key for two operations:

* `sign_precommit_vote(block_hash)` — signs the block hash in the Tendermint consensus round.
* `sign_identification(peer_id, challenge)` — signs peer-identity challenges. [3](#0-2) 

The correct parameterised constructor `LocalKeyStore::_new(private_key: PrivateKey)` exists but is dead code (underscore-prefixed to suppress the compiler warning), confirming the production path never loads a real key: [4](#0-3) 

A developer TODO in `lib.rs` acknowledges the gap:

> `// TODO(Elin): understand how key store would look in production and better define the way the signature manager is created.` [5](#0-4) 

**Secondary exposure vector**: `PrivateKey` derives `Debug` and `Serialize`/`Deserialize`: [6](#0-5) 

`LocalKeyStore` also derives `Debug`, and `SignatureManager<KS>` derives `Debug` with `pub keystore: KS`. Rust's `#[derive(Debug)]` emits all fields, including private ones, so any `{:?}` log of the `SignatureManager` prints the raw private key scalar. [7](#0-6) 

### Impact Explanation

The private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` is publicly known from the repository. Any party can:

1. Derive the corresponding public key with `starknet_crypto::get_public_key`.
2. Call `ecdsa_sign(private_key, block_hash_digest)` to produce a valid precommit-vote signature for an arbitrary block hash.
3. Inject that forged signature into the consensus protocol.

Because every sequencer node deployed with the default configuration shares the same key, no node can be uniquely identified as the legitimate signer. The signature/hash logic binds the wrong signer to every consensus vote, matching the **High** impact: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

### Likelihood Explanation

The trigger requires no privilege: the private key is a compile-time constant in a public source file. Any operator who deploys the sequencer without explicitly wiring a real key store (which the codebase provides no mechanism to do — `_new` is unused) will run with the exposed key. The TODO comment confirms no production key-loading path exists yet.

### Recommendation

1. **Remove the hardcoded key from production code.** Gate `new_for_testing()` behind `#[cfg(test)]` or a `testing` feature flag so it cannot be reached from `create_signature_manager()`.
2. **Activate `LocalKeyStore::_new`** (rename, remove the underscore) and load the private key from a secrets manager, HSM, or environment variable at startup — analogous to the recommendation in the original report.
3. **Redact `PrivateKey` from `Debug` output.** Implement `Debug` manually to emit `"PrivateKey([REDACTED])"` instead of the raw scalar, preventing accidental log exposure.
4. **Add a startup assertion** that rejects the known testing key value in non-test builds.

### Proof of Concept

```rust
use starknet_crypto::{ecdsa_sign, get_public_key};
use starknet_types_core::felt::Felt;

fn main() {
    // Private key is hardcoded in production source:
    // crates/apollo_signature_manager/src/signature_manager.rs:103
    let private_key = Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    );

    // Forge a precommit vote for an arbitrary block hash.
    let target_block_hash = Felt::from_hex_unchecked("0xdeadbeef");
    let forged_sig = ecdsa_sign(&private_key, &target_block_hash).unwrap();

    // forged_sig is indistinguishable from a legitimate sequencer precommit vote.
    println!("Forged precommit: r={:#x} s={:#x}", forged_sig.r, forged_sig.s);

    // Verify it passes with the sequencer's public key.
    let public_key = get_public_key(&private_key);
    let valid = starknet_crypto::verify(&public_key, &target_block_hash,
                                        &forged_sig.r, &forged_sig.s).unwrap();
    assert!(valid); // passes
}
```
