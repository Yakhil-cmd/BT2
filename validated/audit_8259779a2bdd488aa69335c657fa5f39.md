### Title
Staker can raise commission from 0% to 100% in a single step via `CommissionCommitment`, stealing all delegator yield - (File: src/staking/staking.cairo)

### Summary
A staker can attract delegators by advertising 0% commission, then use `set_commission_commitment` with `max_commission = 10000` to immediately raise commission to 100% in the same block, redirecting all future pool rewards to themselves and leaving delegators with nothing.

### Finding Description

The `set_commission` function initializes commission to any value (0–10000) when it is `Option::None`: [1](#0-0) 

Once initialized, commission can only be **decreased** without a commitment: [2](#0-1) 

However, `set_commission_commitment` allows a staker to set `max_commission` to any value ≥ current commission. If current commission is 0%, `max_commission` can be set to 10000 (100%): [3](#0-2) 

With an active commitment, `update_commission` permits raising commission to **any** value ≤ `max_commission` (as long as it differs from the old value): [4](#0-3) 

A commitment is active immediately after being set, since `is_commission_commitment_active` only checks `current_epoch < expiration_epoch`, and the minimum `expiration_epoch` is `current_epoch + 1`: [5](#0-4) 

The commission is applied immediately to all subsequent reward distributions: [6](#0-5) 

The developers themselves acknowledge this gap in a code comment: [7](#0-6) 

**Attack sequence (all steps executable in the same block):**

1. Staker calls `set_commission(0)` — initializes commission at 0%.
2. Staker calls `set_open_for_delegation(STRK_TOKEN_ADDRESS)` — opens pool with 0% commission.
3. Delegators observe 0% commission on-chain and join the pool.
4. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — commitment is immediately active.
5. Staker calls `set_commission(10000)` — commission jumps from 0% to 100% in the same block. This passes because `10000 <= 10000` and `10000 != 0`.
6. All future pool rewards are routed entirely to the staker via `split_rewards_with_commission`; delegators receive zero yield. [8](#0-7) 

### Impact Explanation

Every STRK reward that would have accrued to delegators is instead captured by the staker as commission. Delegators cannot exit immediately — they must call `exit_delegation_pool_intent` and wait out the full `exit_wait_window` (up to 12 weeks). During that window, all rewards generated by their locked stake flow entirely to the staker. This constitutes **theft of unclaimed yield** from delegators, matching the High-severity impact category.

### Likelihood Explanation

Any registered staker with a pool can execute this attack with no special privileges, no external dependencies, and no capital beyond the minimum stake. Steps 4 and 5 can be submitted in consecutive transactions within the same epoch, giving delegators zero reaction time. Likelihood is **High**.

### Recommendation

Enforce a meaningful delay before a newly set commitment allows commission increases. For example, require that `expiration_epoch >= current_epoch + K` (where `K` is the existing epoch-delay constant), so that delegators have at least `K` epochs to observe the pending increase and exit before it takes effect. Alternatively, cap the maximum single-step commission increase (e.g., no more than 500 basis points per epoch) regardless of the commitment ceiling.

### Proof of Concept

```
// 1. Staker registers and initializes commission at 0%
staking.stake(reward_address, operational_address, min_stake);
staking.set_commission(commission: 0);

// 2. Open STRK delegation pool (commission = 0% is visible on-chain)
let pool = staking.set_open_for_delegation(STRK_TOKEN_ADDRESS);

// 3. Delegators join, trusting the 0% commission
pool.enter_delegation_pool(reward_address, large_amount);

// 4. Staker sets commitment with max_commission = 10000 (active immediately)
staking.set_commission_commitment(
    max_commission: 10000,
    expiration_epoch: current_epoch + 1   // minimum allowed
);

// 5. Staker raises commission to 100% in the same block
//    Passes: 10000 <= 10000 AND 10000 != 0
staking.set_commission(commission: 10000);

// 6. Next reward distribution: split_rewards_with_commission(rewards, 10000)
//    => commission_rewards = rewards * 10000 / 10000 = rewards
//    => pool_rewards = 0
//    Delegators receive zero yield for the entire exit_wait_window period.
```

pub(crate) fn split_rewards_with_commission(

    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,

    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
