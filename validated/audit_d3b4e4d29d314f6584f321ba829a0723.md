### Title
Missing declaration-at-state-number check for Cairo 0 (deprecated) classes in `ExecutionStateReader::get_compiled_class` causes RPC fee estimation and simulation to return wrong values for historical blocks - (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class` enforces a declaration-at-state-number guard for Cairo 1 (`ContractClass::V1`) classes when the class manager path is active, but **unconditionally skips that guard for Cairo 0 (`ContractClass::V0`) classes**. The code even carries an explicit TODO acknowledging the gap. As a result, any caller of `estimate_fee`, `simulate_transactions`, `trace_transaction`, or `call` can supply a historical `block_id` and a Cairo 0 class hash that was declared in a *later* block; the RPC will execute against that class and return an authoritative-looking fee estimate or trace instead of the correct `UndeclaredClassHash` error.

---

### Finding Description

In `ExecutionStateReader::get_compiled_class`, when `self.class_manager_handle` is `Some`, the function fetches the executable class from the class manager and then branches on its variant:

```rust
return match contract_class {
    ContractClass::V1(casm_contract_class) => {
        let is_declared = is_contract_class_declared(
            &self.storage_reader.begin_ro_txn()...,
            &class_hash,
            self.state_number,          // ← checked against queried block
        )...;
        if is_declared {
            Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
        } else {
            Err(StateError::UndeclaredClassHash(class_hash))
        }
    }
    // TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
    ContractClass::V0(deprecated_contract_class) => {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
        // ↑ NO declaration-at-state-number check
    }
};
``` [1](#0-0) 

The class manager is populated during sync for every declared class, regardless of the block at which it was declared. Therefore, a Cairo 0 class declared at block N is present in the class manager and will be returned by `get_executable` even when `self.state_number` corresponds to a block before N. The V1 branch correctly rejects this with `UndeclaredClassHash`; the V0 branch does not.

The production RPC server wires a live class manager client into every `estimate_fee` and `simulate_transactions` call:

```rust
let class_manager_client =
    create_class_manager_client(self.class_manager_client.clone()).await;
// … passed into exec_estimate_fee / exec_simulate_transactions
``` [2](#0-1) 

This means `class_manager_handle` is `Some` in every production RPC execution, activating the vulnerable branch.

---

### Impact Explanation

An attacker (or any user) can call `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_traceTransaction` with:
- `block_id` = any block number **before** the Cairo 0 class was declared, and
- a transaction whose execution path touches that Cairo 0 class.

The node will execute the transaction against the class and return a fee estimate or full execution trace as if the class were available at that historical block. The correct response is `StateError::UndeclaredClassHash`. The returned value is authoritative-looking and wrong, matching the allowed impact: **"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

---

### Likelihood Explanation

- The class manager is always configured in the Apollo sequencer node.
- Any Cairo 0 class ever declared on-chain is present in the class manager.
- The RPC methods accept arbitrary `block_id` values from unauthenticated callers.
- No special privilege is required; a standard JSON-RPC call suffices.

---

### Recommendation

Add the same `is_contract_class_declared` (or equivalent deprecated-class declaration) check for the `ContractClass::V0` arm. The TODO comment references `get_class_definition_block_number` being broken for deprecated classes; once that is fixed, the guard should be:

```rust
ContractClass::V0(deprecated_contract_class) => {
    let is_declared = is_deprecated_contract_class_declared(
        &self.storage_reader.begin_ro_txn()...,
        &class_hash,
        self.state_number,
    )?;
    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
```

Until the storage helper is fixed, a safe interim measure is to fall through to the storage-based path (lines 144–160) for V0 classes, which already performs the correct state-number-bounded lookup.

---

### Proof of Concept

1. Observe that Cairo 0 class `C` is declared at block `N` (present in the class manager).
2. Call `starknet_estimateFee` with `block_id = N-1` and an `INVOKE` transaction whose calldata targets a contract whose class hash is `C`.
3. **Expected**: the node returns `UndeclaredClassHash(C)` because `C` was not declared at block `N-1`.
4. **Actual**: the node fetches `C` from the class manager (no state-number check), executes the transaction, and returns a non-zero fee estimate — an authoritative-looking wrong value.

The asymmetry is directly visible in the source: the V1 arm calls `is_contract_class_declared` and gates on its result; the V0 arm returns unconditionally. [3](#0-2)
