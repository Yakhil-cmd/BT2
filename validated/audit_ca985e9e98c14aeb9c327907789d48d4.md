### Title
Unprivileged Caller Can Permanently Freeze Staker Rewards via `disable_rewards` Flag in `update_rewards` - (File: src/staking/staking.cairo)

### Summary
Any unprivileged caller can invoke `update_rewards` with `disable_rewards: true`, which advances the global `last_reward_block` state without distributing any rewards. Because the block-consumption check is a strict greater-than guard on a single global variable, the legitimate reward update for that block is permanently blocked for every staker in the system.

### Finding Description
`StakingRewardsManagerImpl::update_rewards` is a public ABI function (decorated `#[abi(embed_v0)]`) with no role-based access control — only `general_prerequisites()` is called, which checks the contract is unpaused and the caller is non-zero. [1](#0-0) 

Inside the function, `last_reward_block` is written to the current block number **before** the early-return branch that checks `disable_rewards`: [2](#0-1) 

The guard that prevents double-processing is: [3](#0-2) 

Because `last_reward_block` is a single global variable shared across all stakers, once an attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)` at block N, every subsequent call at block N reverts with `REWARDS_ALREADY_UPDATED`. No staker receives rewards for block N.

The `disable_rewards` flag and the `is_pre_consensus()` short-circuit path share the same early-return, but `is_pre_consensus()` is a protocol state while `disable_rewards` is an attacker-controlled parameter: [4](#0-3) 

The attacker only needs to supply any valid, active staker address with non-zero STRK balance — all of which are public on-chain data.

### Impact Explanation
For every block where the attacker front-runs the legitimate `update_rewards` call with `disable_rewards: true`, **all stakers in the protocol permanently lose the block rewards for that block**. The lost rewards are never recoverable because `last_reward_block` is monotonically consumed. Sustained over many blocks, this constitutes permanent freezing of unclaimed yield for the entire staker set.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The attack requires no special privilege, no capital, and no leaked key. The attacker only needs to:
1. Monitor the mempool or block production for a new block.
2. Call `update_rewards(any_valid_staker_address, true)` before any honest party does.

On Starknet, where block times are short and transactions are cheap, this is trivially repeatable every block. The attacker has no profit motive but causes direct, measurable damage to all stakers.

### Recommendation
Remove the `disable_rewards` parameter from the public interface entirely, or restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a privileged role). If `disable_rewards` must remain, it should only be settable by a privileged role, not passed as a caller-controlled argument. Additionally, `last_reward_block` should only be updated after confirming that rewards will actually be distributed.

### Proof of Concept
1. Staker Alice is active with non-zero STRK balance at block N.
2. Attacker calls `update_rewards(alice_address, disable_rewards: true)` at block N.
3. `last_reward_block` is written to N; function returns early — no rewards distributed.
4. Alice (or anyone else) calls `update_rewards(alice_address, false)` at block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Alice permanently loses block-N rewards. Repeat every block for continuous griefing. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
