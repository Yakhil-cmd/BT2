### Title
Unprivileged Caller Can Suppress Block Reward Distribution via `update_rewards` - (File: src/staking/staking.cairo)

### Summary
`update_rewards` in `StakingRewardsManagerImpl` carries no access control and accepts a caller-controlled `disable_rewards: bool` parameter. Any address can call it with `disable_rewards: true` to consume the per-block reward slot without distributing any rewards, permanently blocking legitimate reward updates for that block.

### Finding Description
`update_rewards` enforces a single-call-per-block invariant via `last_reward_block`: [1](#0-0) 

After validating the staker, it unconditionally writes the current block number to `last_reward_block` **before** checking `disable_rewards`: [2](#0-1) 

Because `last_reward_block` is already committed, any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. The function signature exposes `disable_rewards` as a plain public parameter with no role check: [3](#0-2) 

There is no `only_app_governor`, `only_token_admin`, or any other role guard anywhere in the function body.

### Impact Explanation
An attacker who front-runs the legitimate `update_rewards` call in every block with `disable_rewards: true` causes the staking contract to record the block as processed while skipping all reward accounting. Stakers and pool members accumulate zero rewards for every suppressed block. Sustained over time this constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

### Likelihood Explanation
The entry path is fully permissionless: any EOA or contract can call `update_rewards(any_active_staker_address, true)`. No special role, leaked key, or privileged access is required. The only cost is one Starknet transaction per block. Starknet transaction fees are low, making continuous griefing economically viable.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a designated consensus caller) by adding a role check at the top of the function:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_app_governor(); // or a dedicated consensus-caller role
    ...
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface entirely and derive the flag internally from on-chain state.

### Proof of Concept

1. Staker Alice is active with non-zero STRK balance.
2. In block N, attacker calls:
   ```
   staking_contract.update_rewards(alice_address, disable_rewards: true)
   ```
3. The function passes all staker-existence checks, writes `last_reward_block = N`, then returns early — no rewards are distributed.
4. The legitimate consensus caller attempts `update_rewards(alice_address, false)` in the same block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeating step 2 every block permanently prevents reward accrual for all stakers. [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
