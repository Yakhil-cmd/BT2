### Title
Staker Can Steal Delegator Unclaimed Yield via Immediate Commission Spike Using `set_commission_commitment` + `set_commission` Before Reward Distribution — (File: src/staking/staking.cairo)

### Summary
A staker can atomically set a `commission_commitment` with `max_commission = 10000` (100 %) and immediately call `set_commission(10000)` in the same block. Because commission is written directly to storage with no epoch-delay trace, the very next `update_rewards` or `update_rewards_from_attestation_contract` call reads the new 100 % commission and routes the entire pool-reward share to the staker as commission, leaving delegators with zero yield for that distribution.

### Finding Description
`set_commission` in `staking.cairo` writes the new commission value directly to storage:

```cairo
staker_pool_info.commission.write(Option::Some(commission));
``` [1](#0-0) 

`calculate_staker_pools_rewards` reads that value at reward-distribution time:

```cairo
let commission = staker_pool_info.commission();
``` [2](#0-1) 

and immediately applies it to split rewards:

```cairo
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [3](#0-2) 

There is no epoch-delay between when `set_commission` is called and when the new value is used.

`set_commission_commitment` allows a staker to set a commitment that permits commission to be raised up to `max_commission`:

```cairo
assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
``` [4](#0-3) 

Once the commitment is active, `set_commission` permits any value ≤ `max_commission`:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
``` [5](#0-4) 

Both calls can be submitted in the same block. The developers themselves note this gap:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [6](#0-5) 

`update_rewards` (consensus path) and `update_rewards_from_attestation_contract` (attestation path) both call `_update_rewards`, which calls `calculate_staker_pools_rewards` with the live commission value, then immediately transfers pool rewards to the pool contract: [7](#0-6) 

### Impact Explanation
A staker who raises commission to 10 000 (100 %) causes `split_rewards_with_commission` to return `commission_rewards = pool_rewards_including_commission` and `pool_rewards = 0`. All tokens that would have been transferred to the delegation pool contract are instead credited to the staker's `unclaimed_rewards_own`. Delegators receive zero yield for every distribution that occurs while commission is at 100 %. This constitutes **theft of unclaimed yield** from delegators — a High-severity impact under the allowed scope.

### Likelihood Explanation
The attack requires two sequential transactions in the same block (or across consecutive blocks):
1. `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — no prior commitment must be active.
2. `set_commission(10000)` — immediately valid because the commitment is already active.

No sequencer cooperation is needed; the staker submits both as ordinary user transactions. Delegators have no on-chain mechanism to react before the next `update_rewards` call (which fires every block in the consensus-rewards regime). The commitment expiry can be set up to one year out, giving the staker a prolonged window to drain yield. Likelihood is **Medium** (requires a malicious staker, but the execution path is fully permissionless and atomic).

### Recommendation
Apply an epoch-delay to commission changes by recording them in a checkpoint trace (analogous to how staker balances use `StakerBalanceTrace`). The new commission should only become effective at `current_epoch + K`, giving delegators the same advance-notice window they have for balance changes. Alternatively, cap the maximum single-step commission increase (e.g., no more than X basis points per epoch) regardless of the commitment ceiling.

### Proof of Concept
```
Block N:
  tx1: staker.set_commission_commitment(max_commission=10000, expiration_epoch=current_epoch+1)
  tx2: staker.set_commission(commission=10000)
  // commission is now 10000 in storage

Block N+1 (or same block):
  sequencer: update_rewards(staker_address)
    → calculate_staker_pools_rewards reads commission = 10000
    → split_rewards_with_commission(pool_rewards_including_commission, 10000)
        commission_rewards = pool_rewards_including_commission   // 100%
        pool_rewards = 0
    → staker.unclaimed_rewards_own += commission_rewards
    → pool contract receives 0 STRK
    → delegators' unclaimed_rewards remain 0
```

All delegator yield for that block is permanently redirected to the staker. The attack repeats every block for the duration of the commitment period (up to one year). [8](#0-7) [9](#0-8)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
