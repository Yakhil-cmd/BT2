### Title
Gateway Admits Transactions with Zero L1/L1-Data Gas Price That Fail Blockifier Pre-Validation — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateless and stateful validators only enforce a minimum threshold on the **L2 gas price**, leaving L1 gas price and L1 data gas price completely unchecked. The blockifier's `perform_pre_validation_stage` enforces all three gas prices. A transaction with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` (but a valid L2 gas price) passes every gateway check and enters the mempool, then fails with `InsufficientResourceBounds` when the batcher calls `check_fee_bounds`. The invariant "only transactions that can survive blockifier pre-validation are admitted" is broken.

---

### Finding Description

**Stateless validator** (`crates/apollo_gateway/src/stateless_transaction_validator.rs`) checks only:
- `l2_gas.max_price_per_unit >= config.min_gas_price`
- `l2_gas.max_amount <= config.max_l2_gas_amount`
- Total fee > 0 (at least one resource bound non-zero) [1](#0-0) 

L1 gas price and L1 data gas price are never inspected.

**Stateful validator** (`crates/apollo_gateway/src/stateful_transaction_validator.rs`) checks only the L2 gas price against the previous block's L2 gas price threshold. The code even carries an explicit TODO acknowledging the gap:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(...)
``` [2](#0-1) 

**Blockifier pre-validation** (`crates/blockifier/src/transaction/account_transaction.rs`) enforces all three gas prices:

```rust
vec![
    (L1Gas,     l1_gas_resource_bounds,     ..., *l1_gas_price),
    (L1DataGas, l1_data_gas_resource_bounds,..., *l1_data_gas_price),
    (L2Gas,     l2_gas_resource_bounds,     ..., *l2_gas_price),
]
// ...
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [3](#0-2) 

`perform_pre_validation_stage` calls `check_fee_bounds` unconditionally (when `charge_fee` is true): [4](#0-3) 

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An attacker submits an `AllResources` invoke transaction with:
- `l2_gas.max_price_per_unit = min_gas_price` (passes both gateway checks)
- `l1_gas.max_price_per_unit = 0` (below any real L1 gas price)
- `l1_data_gas.max_price_per_unit = 0` (below any real L1 data gas price)
- `l2_gas.max_amount = 1` (makes total fee > 0)

The transaction clears all gateway filters and occupies a mempool slot. When the batcher calls `perform_pre_validation_stage`, `check_fee_bounds` immediately rejects it with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`. The nonce is not consumed and the slot is freed, but the attacker can continuously re-submit, saturating the mempool with permanently-unexecutable transactions.

---

### Likelihood Explanation

**High.** The attack requires only crafting a standard V3 invoke transaction with zero L1/L1-data gas prices. No privileged access, no special account, no on-chain state required. The stateless validator explicitly accepts transactions where only L2 gas is non-zero: [5](#0-4) 

The test case `valid_l2_gas` confirms a transaction with only L2 gas set (L1 and L1-data gas at zero) passes stateless validation.

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also check `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding prices, mirroring the three-resource check already present in `check_fee_bounds`. The existing TODO comment in the code acknowledges this gap:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
``` [6](#0-5) 

Similarly, `StatelessTransactionValidator::validate_resource_bounds` should add a floor check on `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` analogous to the existing `min_gas_price` check for L2 gas.

---

### Proof of Concept

```
1. Craft RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l2_gas:      { max_amount: 1,    max_price_per_unit: <min_gas_price> },
         l1_gas:      { max_amount: 1000, max_price_per_unit: 0 },   // ← zero price
         l1_data_gas: { max_amount: 1000, max_price_per_unit: 0 },   // ← zero price
     }

2. POST to gateway /add_transaction.

3. StatelessTransactionValidator::validate_resource_bounds:
   - max_possible_fee(Tip::ZERO) = 1 * min_gas_price > 0  → PASS
   - l2_gas.max_price_per_unit >= min_gas_price            → PASS
   - l2_gas.max_amount <= max_l2_gas_amount                → PASS
   (L1 and L1-data gas prices never checked)               → PASS

4. StatefulTransactionValidator::validate_resource_bounds:
   - validate_tx_l2_gas_price_within_threshold(l2_gas_price) → PASS
   (L1 and L1-data gas prices never checked)                 → PASS

5. Transaction enters mempool.

6. Batcher calls perform_pre_validation_stage → check_fee_bounds:
   - l1_gas.max_price_per_unit (0) < actual_l1_gas_price (e.g. 1e9)
     → InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }
   Transaction rejected; nonce unchanged; mempool slot freed.

7. Repeat indefinitely to saturate mempool.
``` [1](#0-0) [7](#0-6) [4](#0-3)

        &self,

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
