### Title
Unbounded `stakers` Vec Growth Enables Griefing Attack Causing Unbounded Gas Consumption in `get_stakers()` - (File: src/staking/staking.cairo)

### Summary
The `stakers` storage Vec in the Staking contract accumulates every address that has ever called `stake()` and is never pruned, even after a staker fully exits via `unstake_action()`. The `get_stakers()` function, consumed by the consensus layer to determine per-epoch staking power, iterates over the entire Vec unconditionally. An unprivileged actor can systematically inflate this Vec with dead entries by cycling through many addresses, each staking the minimum amount and then unstaking, causing `get_stakers()` to consume ever-increasing gas until it becomes unusable.

### Finding Description

**Root cause — Vec never shrinks:**

The storage declaration explicitly documents the invariant:

```
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [1](#0-0) 

Every successful `stake()` call appends to this Vec unconditionally:

```cairo
// Add staker address to the stakers vector.
self.stakers.push(staker_address);
``` [2](#0-1) 

`unstake_action()` deletes the staker's `InternalStakerInfo` record and clears the operational-address mapping, but it never removes the address from `stakers`. [3](#0-2) 

**Consumption site — full linear scan in `get_stakers()`:**

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    ...
}
``` [4](#0-3) 

Every dead entry still costs a storage read (`staker_address_ptr.read()`) plus a call to `is_staker_active()`, which itself reads `staker_info` and `staker_unstake_intent_epoch` from storage. [5](#0-4) 

**One-time-use address constraint:** `assert_staker_address_not_reused` prevents recycling the same address, so each attack cycle requires a fresh address. [6](#0-5) 

### Impact Explanation

`get_stakers()` is the sole `IStakingConsensus` function that returns the full staker set with staking power, public keys, and peer IDs for a given epoch. It is the canonical data source for the consensus layer to build the validator committee. [7](#0-6) 

As the Vec grows, each call to `get_stakers()` performs O(n) storage reads where n is the cumulative count of all addresses that have ever staked. Once n is large enough, the function exceeds Starknet's per-transaction gas ceiling and reverts on every invocation. The consensus layer loses the ability to enumerate stakers, constituting **unbounded gas consumption** and **griefing damage to the protocol**.

### Likelihood Explanation

The attacker must stake at least `min_stake` per address and wait through `exit_wait_window` (default one week, max twelve weeks) before recovering funds. [8](#0-7) 

However, the attacker recovers the principal after each cycle; the only permanent cost is gas. By running many addresses in parallel (each at a different stage of the exit window), the attacker can sustain a high throughput of new dead entries with capital equal to `min_stake × parallelism`. The attack is economically feasible for a well-funded adversary motivated to disrupt the network, and the protocol has no on-chain defence against it.

### Recommendation

1. **Remove on exit**: When `unstake_action()` is called, swap-and-pop the staker's entry from `stakers` (or use an `IterableMap` that supports deletion, as already used for `btc_tokens`).
2. **Lazy compaction**: Track a "live count" and skip zero-address slots during iteration.
3. **Cap registration**: Enforce a maximum number of concurrent stakers, rejecting new `stake()` calls when the cap is reached.

### Proof of Concept

```
// Attacker controls addresses A_1 … A_N, each pre-funded with min_stake STRK.

for i in 1..=N:
    A_i.approve(staking_contract, min_stake)
    staking_contract.stake(reward_addr, op_addr_i, min_stake)   // stakers.push(A_i)

// Wait exit_wait_window (1 week default)
for i in 1..=N:
    staking_contract.unstake_intent()   // called from A_i
    // after window:
    staking_contract.unstake_action(A_i)  // staker_info deleted, A_i stays in Vec

// stakers Vec now contains N dead entries.
// Each subsequent call to get_stakers(epoch_id) reads all N entries.
// At sufficient N, get_stakers() exceeds the block gas limit and reverts.
```

The attacker recovers all `min_stake × N` principal; the only sunk cost is gas for `2N` transactions. The `stakers` Vec is permanently polluted with no protocol-level remedy.
