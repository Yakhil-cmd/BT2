### Title
Gas for WASM Bytecode Decompression Not Charged Before `Uncompress` in `MsgStoreCode` - (File: `sei-wasmd/x/wasm/keeper/keeper.go`)

---

### Summary

In Sei's CosmWasm integration, the `create` function charges compile gas **only after** successful gzip decompression of the submitted WASM bytecode. If decompression fails (e.g., due to a crafted corrupt payload), no compile gas is charged for the CPU-intensive decompression work. An unprivileged attacker can repeatedly submit small, highly-compressed gzip payloads that expand near `MaxWasmSize` and then fail, paying only the cheap tx-size ante cost while forcing validators to perform expensive decompression work for free.

---

### Finding Description

In `sei-wasmd/x/wasm/keeper/keeper.go`, the `create` function (called by `MsgStoreCode`) follows this order:

1. **Decompress first** — `ioutils.Uncompress` is called, which invokes `gzip.NewReader` + `io.ReadAll` with a `LimitReader`.
2. **Return error with zero compile gas charged** — if decompression fails, the function returns immediately.
3. **Charge gas only on success** — `CompileCosts(len(wasmCode))` is consumed only after successful decompression. [1](#0-0) 

The `Uncompress` function in `ioutils/ioutil.go` performs the full gzip decompression stream up to `MaxWasmSize` bytes before returning any error: [2](#0-1) 

The only ante-level cost that applies regardless of outcome is `TxSizeCostPerByte * len(ctx.TxBytes())`, charged against the **compressed** payload size: [3](#0-2) 

This is a generic tx-size cost, not priced for decompression CPU work. The `DefaultCompileCost` is 3 gas per byte of **decompressed** code: [4](#0-3) 

A gzip payload of ~100 KB that decompresses to ~800 KB (near `MaxWasmSize`) and then fails (corrupted checksum at the end) would:
- Pay tx-size cost for ~100 KB compressed bytes
- Avoid `800_000 * 3 = 2_400_000` compile gas
- Force validators to run the full decompression stream

---

### Impact Explanation

Validators perform unbounded (up to `MaxWasmSize`) gzip decompression work without being compensated by gas. An attacker can spam `MsgStoreCode` transactions with crafted payloads to consume validator CPU disproportionate to the fees paid, potentially causing block processing delays. This maps to **Low** (wrong fee calculation) or **Medium** (block delay >2.5s under sustained spam) in the Sei bounty scope.

---

### Likelihood Explanation

The attack requires no special privileges — `MsgStoreCode` is open to any account with sufficient funds to pay the (underpriced) tx fee. Crafting a valid gzip stream that decompresses to near-limit and then fails is straightforward. The attack is cheap to repeat.

---

### Recommendation

Charge gas proportional to the **compressed** input size before calling `Uncompress`, so that even failed decompression attempts are priced. For example, charge `CompileCosts(len(wasmCode))` (using the compressed length as a lower bound) before the `Uncompress` call, then charge the delta after successful decompression. This mirrors the fix recommended in the Arbitrum report: gas must be charged even if activation/decompression fails.

---

### Proof of Concept

1. Craft a gzip stream: compress ~800 KB of valid WASM bytes, then corrupt the final checksum byte.
2. Submit `MsgStoreCode` with this payload via the Cosmos RPC.
3. The ante handler charges only `TxSizeCostPerByte * compressed_size` (cheap).
4. `create` calls `ioutils.Uncompress`, which streams ~800 KB through the gzip decoder before hitting the checksum error.
5. `create` returns `ErrCreateFailed` — zero compile gas charged.
6. Repeat in a tight loop across multiple transactions to saturate validator CPU during block execution. [5](#0-4) [6](#0-5)

```go
func Uncompress(src []byte, limit uint64) ([]byte, error) {
	switch n := uint64(len(src)); {
	case n < 3:
		return src, nil
	case n > limit:
		return nil, types.ErrLimit
	}
	if !bytes.Equal(gzipIdent, src[0:3]) {
		return src, nil
	}
	zr, err := gzip.NewReader(bytes.NewReader(src))
	if err != nil {
		return nil, err
	}
	zr.Multistream(false)
	defer func() { _ = zr.Close() }()
	return io.ReadAll(LimitReader(zr, int64(limit))) // #nosec G115 -- checked above
}
```
