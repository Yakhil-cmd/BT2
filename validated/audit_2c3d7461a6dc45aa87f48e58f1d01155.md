### Title
Staker Permanently Loses Epoch Rewards If Starknet Experiences Downtime During Attestation Window - (File: src/attestation/attestation.cairo)

### Summary

The `Attestation` contract enforces a strict block-number-based window for stakers to submit their attestation. If Starknet itself experiences downtime or block production halts during a staker's assigned attestation window, the staker cannot submit their transaction, the window expires, and the epoch's rewards are permanently lost with no recovery mechanism.

### Finding Description

In `src/attestation/attestation.cairo`, the `_assert_attest_in_window` function enforces that the current block number falls within `[target_attestation_block + MIN_ATTESTATION_WINDOW, target_attestation_block + attestation_window]`:

```cairo
fn _assert_attest_in_window(self: @ContractState, target_attestation_block: BlockNumber) {
    let attestation_window = self.attestation_window.read();
    let current_block_number = get_block_number();
    let min_block = target_attestation_block + MIN_ATTESTATION_WINDOW.into();
    let max_block = target_attestation_block + attestation_window.into();
    assert!(
        min_block <= current_block_number && current_block_number <= max_block,
        "{}",
        Error::ATTEST_OUT_OF_WINDOW,
    );
}
```

The minimum configurable window size is `MIN_ATTESTATION_WINDOW = 11` blocks, and the default deployment uses exactly this minimum (confirmed by `assert!(old_attestation_window == MIN_ATTESTATION_WINDOW)` in tests), making the valid attestation range as narrow as a single block (`[target+11, target+11]`).

The `attest()` function is the **only** path to trigger `update_rewards_from_attestation_contract` during the pre-consensus phase. If a staker misses the window, there is no fallback, no grace period, and no way to retroactively claim the epoch's rewards. The epoch rewards are simply never credited.

Starknet is itself an L2 network and has experienced documented downtime events. If block production halts during a staker's narrow attestation window, the staker's transaction cannot be included, the window passes, and the epoch rewards are permanently lost.

### Impact Explanation

This is a **permanent freezing of unclaimed yield**. The staker's epoch rewards are calculated but never credited because `update_rewards_from_attestation_contract` is never called. There is no mechanism to recover rewards for a missed epoch. The staker's `unclaimed_rewards_own` is never incremented for that epoch, and the rewards are effectively burned.

### Likelihood Explanation

Starknet has experienced network-level downtime in the past. The attestation window can be as narrow as 11 blocks (the enforced minimum), which at typical Starknet block times represents a very short real-world window. Any network congestion, sequencer issue, or downtime event that overlaps with a staker's assigned window causes the loss. Since each staker's target block is deterministically assigned via a Poseidon hash, a staker cannot choose a "safer" window.

### Recommendation

1. Implement a grace period or recovery function that allows a staker to claim rewards for an epoch where they missed attestation due to documented network downtime (e.g., governed by a flag set by the app governor).
2. Alternatively, increase the minimum `attestation_window` significantly so the valid window spans a meaningful number of blocks, reducing the probability of a complete miss during downtime.
3. Consider allowing attestation in the next epoch for the previous epoch's rewards if the staker can prove (via block hash) that their target block fell within a downtime period.

### Proof of Concept

1. Staker is registered with a target attestation block `T` for epoch `E`. Valid window: `[T+11, T+attestation_window]`.
2. Starknet experiences a sequencer outage. Block production halts at block `T+5` and resumes at block `T+attestation_window+1`.
3. Staker attempts to call `attest(block_hash)` after resumption.
4. `_assert_attest_in_window` checks `current_block_number <= max_block` → `T+attestation_window+1 <= T+attestation_window` → **false** → reverts with `ATTEST_OUT_OF_WINDOW`.
5. The epoch advances. `update_rewards_from_attestation_contract` is never called for epoch `E`.
6. Staker's `unclaimed_rewards_own` is not incremented for epoch `E`. Epoch rewards are permanently lost.

        fn attest(ref self: ContractState, block_hash: felt252) {
            let operational_address = get_caller_address();
            let staking_dispatcher = IStakingAttestationDispatcher {
                contract_address: self.staking_contract.read(),
            };
            // Note: This function checks for a zero staker address and will panic if so.
            let staking_attestation_info = staking_dispatcher
                .get_attestation_info_by_operational_address(:operational_address);
            self._validate_attestation(:block_hash, :staking_attestation_info);
            // Work is one tx per epoch.
            self
                ._mark_attestation_is_done(
                    staker_address: staking_attestation_info.staker_address(),
                    current_epoch: staking_attestation_info.epoch_id(),
                );
            staking_dispatcher
                .update_rewards_from_attestation_contract(
                    staker_address: staking_attestation_info.staker_address(),
                );
        }

        fn update_rewards_from_attestation_contract(
            ref self: ContractState, staker_address: ContractAddress,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
            self.assert_caller_is_attestation_contract();
            let mut staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            // Get current epoch data.
            let (strk_epoch_rewards, btc_epoch_rewards) = reward_supplier_dispatcher
                .calculate_current_epoch_rewards();
            let (strk_total_stake, btc_total_stake) = self.get_current_total_staking_power();
            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let curr_epoch = self.get_current_epoch();
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_epoch_rewards,
                    btc_total_rewards: btc_epoch_rewards,
                    :strk_total_stake,
                    :btc_total_stake,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
