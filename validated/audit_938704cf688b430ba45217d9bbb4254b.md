### Title
Missing Zero-Check for `strk_total_stake` in `calculate_staker_total_staking_power` Causes Division-by-Zero Panic - (File: src/staking/utils.cairo)

---

### Summary

`calculate_staker_total_staking_power` in `src/staking/utils.cairo` guards the BTC division against a zero denominator but performs no equivalent guard for the STRK division. If `strk_total_stake` is zero while any active staker has a non-zero STRK balance, the function panics. This panic propagates through `get_staker_staking_power_at_epoch` → `get_stakers`, causing the public `get_stakers` entrypoint to revert and blocking the Attestation contract from distributing rewards.

---

### Finding Description

In `src/staking/utils.cairo` lines 130–135, the STRK staking-power term divides unconditionally by `strk_total_stake`:

```cairo
let strk_staking_power = mul_wide_and_div(
    lhs: staker_strk_total_amount.to_amount_18_decimals(),
    rhs: STRK_WEIGHT_FACTOR,
    div: strk_total_stake.to_amount_18_decimals(),   // ← no zero-check
)
    .unwrap();
```

The immediately following BTC path (lines 136–145) explicitly guards the identical operation:

```cairo
let btc_staking_power = if btc_total_stake.is_zero() {
    Zero::zero()
} else {
    mul_wide_and_div(
        lhs: staker_btc_total_amount.to_amount_18_decimals(),
        rhs: BTC_WEIGHT_FACTOR,
        div: btc_total_stake.to_amount_18_decimals(),
    )
        .unwrap()
};
```

The asymmetry is exact: BTC is protected, STRK is not.

The call chain is:

1. `get_stakers` (`staking.cairo:901`) — public, callable by anyone — reads `strk_total_stake` from the trace at `epoch_id` and passes it to…
2. `get_staker_staking_power_at_epoch` (`staking.cairo:2388`) — which only early-returns when `staker_strk_total_amount.is_zero()` (line 2400), then calls…
3. `calculate_staker_total_staking_power` (`utils.cairo:124`) — which divides by `strk_total_stake` without a zero-check.

`strk_total_stake` is read from `tokens_total_stake_trace` via `balance_at_epoch` at the caller-supplied `epoch_id` (`staking.cairo:2418–2419`). If the trace returns zero for that epoch (e.g., the protocol has just been initialised, all STRK stakers have completed their exit, or a future epoch has not yet received any stake entries) while a staker record still appears active at that epoch, the division by zero fires. ---

### Impact Explanation

A panic in `get_stakers` causes every call from the Attestation contract to revert. The Attestation contract relies on `get_stakers` to enumerate stakers for a given epoch before it can record attestations and trigger reward updates. A persistent revert here freezes unclaimed yield for all stakers and delegators for as long as the condition holds — matching the **High: Temporary freezing of unclaimed yield** impact class.

---

### Likelihood Explanation

The condition requires `strk_total_stake == 0` at `epoch_id` while at least one staker is still considered active with a non-zero STRK balance at that epoch. This can arise from:

- **Epoch-boundary race**: a staker's own balance trace is written for epoch `E+K` before the global `tokens_total_stake_trace` is updated for the same epoch, creating a window where the individual balance is non-zero but the aggregate is still zero.
- **Fresh deployment / empty-pool state**: the protocol is live but no STRK has been staked yet for the queried future epoch, yet a staker record exists.
- **Full-exit timing**: all stakers complete `unstake_action` in the same epoch, zeroing the total-stake trace, but a stale active-staker entry remains readable for a future epoch.

Likelihood is **low** in steady-state operation but non-negligible at protocol boundaries, and the missing guard is directly analogous to the confirmed original bug.

---

### Recommendation

Mirror the BTC guard for the STRK path in `calculate_staker_total_staking_power`:

```cairo
let strk_staking_power = if strk_total_stake.is_zero() {
    Zero::zero()
} else {
    mul_wide_and_div(
        lhs: staker_strk_total_amount.to_amount_18_decimals(),
        rhs: STRK_WEIGHT_FACTOR,
        div: strk_total_stake.to_amount_18_decimals(),
    )
        .unwrap()
};
```

Additionally, `get_staker_staking_power_at_epoch` should assert or early-return when `strk_total_stake` is zero to make the invariant explicit at the call site.

---

### Proof of Concept

1. Deploy the staking contract in a state where no STRK has been staked for epoch `E` (i.e., `tokens_total_stake_trace` for STRK returns `0` at epoch `E`).
2. Ensure at least one staker address is present in the `stakers` vec and `is_staker_active` returns `true` for epoch `E` (possible during the epoch-boundary window described above).
3. Call `get_stakers(epoch_id: E)`.
4. `get_total_staking_power_at_epoch` returns `strk_total_stake = 0`.
5. For the active staker, `get_staker_total_strk_btc_balance_at_epoch` returns a non-zero STRK amount, so the early-return at line 2400 is skipped.
6. `calculate_staker_total_staking_power` is called with `strk_total_stake = 0`; `mul_wide_and_div(..., div: 0)` panics.
7. The transaction reverts; the Attestation contract cannot proceed; rewards are frozen. [5](#0-4) [6](#0-5) [7](#0-6)

pub(crate) fn calculate_staker_total_staking_power(
    staker_strk_total_amount: NormalizedAmount,
    staker_btc_total_amount: NormalizedAmount,
    strk_total_stake: NormalizedAmount,
    btc_total_stake: NormalizedAmount,
) -> StakingPower {
    let strk_staking_power = mul_wide_and_div(
        lhs: staker_strk_total_amount.to_amount_18_decimals(),
        rhs: STRK_WEIGHT_FACTOR,
        div: strk_total_stake.to_amount_18_decimals(),
    )
        .unwrap();
    let btc_staking_power = if btc_total_stake.is_zero() {
        Zero::zero()
    } else {
        mul_wide_and_div(
            lhs: staker_btc_total_amount.to_amount_18_decimals(),
            rhs: BTC_WEIGHT_FACTOR,
            div: btc_total_stake.to_amount_18_decimals(),
        )
            .unwrap()
    };
    strk_staking_power + btc_staking_power
}

        fn get_total_staking_power_at_epoch(
            self: @ContractState, epoch_id: Epoch,
        ) -> (NormalizedAmount, NormalizedAmount) {
            let strk_total_stake_trace = self.tokens_total_stake_trace.entry(STRK_TOKEN_ADDRESS);
            let strk_curr_total_stake = balance_at_epoch(trace: strk_total_stake_trace, :epoch_id);
            let mut btc_curr_total_stake: NormalizedAmount = Zero::zero();
            for (token_address, active_status) in self.btc_tokens {

                    btc_curr_total_stake +=
                        balance_at_epoch(trace: btc_total_stake_trace, :epoch_id);
                }
