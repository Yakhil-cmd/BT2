### Title
Missing Actual Token Balance Check Before Reward Transfer Causes Temporary Freezing of Unclaimed Yield - (File: src/reward_supplier/reward_supplier.cairo)

### Summary
`RewardSupplier::claim_rewards` validates only the accounting variable `unclaimed_rewards` before attempting a token transfer, but never checks the contract's actual on-chain token balance. Because `unclaimed_rewards` can exceed the real balance during the L1→L2 mint-arrival window, any staker who calls `claim_rewards` during that window will have their transaction revert, temporarily freezing their unclaimed yield.

### Finding Description
In `claim_rewards` (lines 205–220 of `src/reward_supplier/reward_supplier.cairo`), the guard is:

```cairo
let unclaimed_rewards = self.unclaimed_rewards.read();
assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);
// ...
token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
``` [1](#0-0) 

`unclaimed_rewards` is a pure accounting counter: it is incremented by `update_unclaimed_rewards_from_staking_contract` whenever the staking contract registers new epoch rewards, and decremented here when rewards are paid out. [2](#0-1) 

When `unclaimed_rewards` grows, `request_funds` is called to bridge the gap by sending L1 mint messages. [3](#0-2) 

However, `request_funds` only dispatches L1 messages; the actual tokens arrive later via `on_receive` after the L1→L2 message delay (which can be hours to days). [4](#0-3) 

During this window, `unclaimed_rewards >= amount` (the accounting check passes) but the real ERC-20 balance of the contract is less than `amount`. The subsequent `checked_transfer` panics, reverting the entire call chain back to the staker's `claim_rewards` entry point in the staking contract. [5](#0-4) 

The missing guard is:
```cairo
let balance = token_dispatcher.balance_of(account: get_contract_address());
assert!(amount.into() <= balance, "Insufficient token balance");
```

Without it, the contract proceeds blindly into a transfer that is guaranteed to fail, wasting gas and providing no actionable error to the caller.

### Impact Explanation
**High – Temporary freezing of unclaimed yield.**

Any staker (or their reward address) who calls `staking::claim_rewards` while the reward supplier's real balance is below the owed amount will receive a revert. Their `unclaimed_rewards_own` remains non-zero and locked until the L1 mint arrives. During high-activity epochs (many attestations, large reward batches), the gap between accounting and real balance can persist for an extended period, blocking all reward claims across the protocol.

### Likelihood Explanation
**Moderate.** The L1→L2 message delay is a structural property of the system. Every epoch transition that triggers `update_unclaimed_rewards_from_staking_contract` with a large `rewards` value creates a window where the real balance lags the accounting balance. Stakers who attest and immediately try to claim rewards will reliably hit this window. The larger the staking pool and the longer the L1 congestion, the wider the window.

### Recommendation
Add an explicit balance check inside `claim_rewards` before the transfer, analogous to the `maxFlashLoan` guard recommended in the reference report:

```cairo
fn claim_rewards(ref self: ContractState, amount: Amount) {
    let staking_contract = self.staking_contract.read();
    assert!(
        get_caller_address() == staking_contract,
        "{}",
        GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
    );
    let unclaimed_rewards = self.unclaimed_rewards.read();
    assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

    // NEW: guard against real balance shortfall before attempting transfer
    let token_dispatcher = self.token_dispatcher.read();
    let balance: Amount = token_dispatcher
        .balance_of(account: get_contract_address())
        .try_into()
        .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);
    assert!(amount <= balance, "{}", GenericError::INSUFFICIENT_BALANCE);

    self.unclaimed_rewards.write(unclaimed_rewards - amount);
    token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
}
```

This fails early with a clear protocol error, saves gas, and makes the L1-pending state observable to callers.

### Proof of Concept
1. Epoch N ends; attestation is processed; `update_unclaimed_rewards_from_staking_contract(rewards: 1_000_000)` is called.
2. `unclaimed_rewards` becomes `1_000_000 + STRK_IN_FRIS`; `request_funds` dispatches L1 mint messages but the real balance is still `STRK_IN_FRIS` (≈ 1 STRK).
3. Staker A has `unclaimed_rewards_own = 500_000` and calls `staking::claim_rewards`.
4. Staking contract calls `reward_supplier::claim_rewards(500_000)`.
5. `assert!(500_000 <= 1_000_000 + STRK_IN_FRIS)` — passes.
6. `token_dispatcher.checked_transfer(staking_contract, 500_000)` — **panics** because real balance ≈ 1 STRK < 500_000.
7. Entire transaction reverts; staker A's yield is frozen until the L1 mint message is processed. [6](#0-5) [7](#0-6)

        fn request_funds(ref self: ContractState, unclaimed_rewards: Amount) {
            // Read current balance.
            let token_dispatcher = self.token_dispatcher.read();
            let balance: Amount = token_dispatcher
                .balance_of(account: get_contract_address())
                .try_into()
                .expect_with_err(InternalError::BALANCE_ISNT_AMOUNT_TYPE);

            // Calculate credit, which is the contract's balance plus the amount already requested
            // from L1, and the debit, which is the unclaimed rewards.
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            let credit = balance + l1_pending_requested_amount;
            let debit = unclaimed_rewards;

            // If there isn't enough credit to cover the debit + threshold, request funds.
            let base_mint_amount = self.base_mint_amount.read();
            let threshold = compute_threshold(base_mint_amount);
            if credit < debit + threshold {
                let diff = debit + threshold - credit;
                let num_msgs = ceil_of_division(dividend: diff, divisor: base_mint_amount);
                let total_amount = num_msgs * base_mint_amount;
                for _ in 0..num_msgs {
                    self.send_mint_request_to_l1_reward_supplier();
                }
                self.emit(Events::MintRequest { total_amount, num_msgs });
                l1_pending_requested_amount += total_amount;
            }

            // Commit to storage the requested amount, which is now part of the credit.
            self.l1_pending_requested_amount.write(l1_pending_requested_amount);
        }

pub(crate) fn claim_from_reward_supplier(
    reward_supplier_dispatcher: IRewardSupplierDispatcher,
    amount: Amount,

    let staking_contract = get_contract_address();
