### Title
`setPoolAdminFeeDestination` Does Not Flush Accrued Fees Before Updating Destination, Redirecting Earned Fees to New Address — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolAdminFeeDestination` updates `poolAdminFeeDestination[pool]` without first collecting the fees that have already accrued in the pool under the old destination. All accrued admin fees (both notional and spread) are subsequently paid to the new destination when `collectFees` is next triggered, permanently depriving the old destination of fees it earned.

---

### Finding Description

The factory stores two types of admin-owed fee balances on each pool:

1. **Notional fees** — accumulated in `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` (pool slot 2) on every swap.
2. **Spread fees** — the surplus of `balance - binTotals` that builds up between collections.

Both are realized only when `collectFees` is called, which reads `adminFeeDestination_` as a parameter supplied by the factory from `poolAdminFeeDestination[pool]`.

`setPoolAdminFees` and `setPoolProtocolFee` both flush accrued fees **before** mutating their respective config values:

```solidity
// MetricOmmPoolFactory.sol – setPoolAdminFees (lines 417-425)
IMetricOmmPoolCollectFees(pool).collectFees(
    c.protocolSpreadFeeE6,
    c.adminSpreadFeeE6,
    c.protocolNotionalFeeE8,
    c.adminNotionalFeeE8,
    poolAdminFeeDestination[pool]   // ← old destination used before update
);
c.adminSpreadFeeE6 = newAdminSpreadFeeE6;
```

`setPoolAdminFeeDestination` does **not**:

```solidity
// MetricOmmPoolFactory.sol – setPoolAdminFeeDestination (lines 438-447)
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;   // ← no flush
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

After this call, any subsequent invocation of the permissionless `collectPoolFees` (or any fee-changing call that triggers `collectFees`) will deliver all previously accrued admin fees to the **new** destination:

```solidity
// MetricOmmPoolFactory.sol – collectPoolFees (lines 379-389)
function collectPoolFees(address pool) external override nonReentrant {
    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6, c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8, c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]   // ← now points to new destination
    );
}
```

The pool's `collectFees` then transfers admin-share tokens directly to `adminFeeDestination_` and zeroes the notional accumulators:

```solidity
// MetricOmmPool.sol – collectFees (lines 416-430)
if (totalFee0ToAdmin > 0) transferToken0(adminFeeDestination_, totalFee0ToAdmin);
if (totalFee1ToAdmin > 0) transferToken1(adminFeeDestination_, totalFee1ToAdmin);
...
notionalFeeToken0Scaled = 0;
notionalFeeToken1Scaled = 0;
```

There is no mechanism to recover or re-route fees once the accumulators are zeroed.

---

### Impact Explanation

The old admin fee destination permanently loses all admin fees that accrued between the last collection and the destination change. The new destination receives tokens it did not earn. This is a direct, irreversible loss of owed protocol/admin assets. The magnitude scales with the volume traded and the notional fee rate since the last `collectFees` call.

---

### Likelihood Explanation

The pool admin is semi-trusted and can call `setPoolAdminFeeDestination` at any time without a timelock. The loss can occur unintentionally (admin forgets to collect first) or intentionally (admin front-runs a collection to redirect fees). Because `collectPoolFees` is permissionless, the old destination cannot atomically protect itself — the admin can front-run any collection attempt.

---

### Recommendation

Mirror the pattern used in `setPoolAdminFees` and `setPoolProtocolFee`: call `collectFees` with the **current** `poolAdminFeeDestination[pool]` before overwriting it.

```solidity
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    // Flush accrued fees to the OLD destination before switching.
    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool).collectFees(
        c.protocolSpreadFeeE6, c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8, c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]
    );
    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    emit PoolAdminFeeDestinationUpdated(pool, newAdminFeeDestination);
}
```

---

### Proof of Concept

1. Pool is live with `adminFeeDestination = OLD`. Swaps accumulate `notionalFeeToken0Scaled > 0` and spread surplus.
2. Pool admin calls `factory.setPoolAdminFeeDestination(pool, NEW)`.
   - `poolAdminFeeDestination[pool]` is now `NEW`.
   - Accrued notional and spread balances are untouched on the pool.
3. Anyone (or the admin) calls `factory.collectPoolFees(pool)`.
   - `collectFees` is invoked with `adminFeeDestination_ = NEW`.
   - All admin-share tokens (notional + spread) are transferred to `NEW`.
   - `notionalFeeToken0Scaled` and `notionalFeeToken1Scaled` are zeroed.
4. `OLD` receives nothing. Fees earned under `OLD` are permanently lost to it. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
