### Title
Stale `eligible_operational_addresses` Entry Enables Unauthorized Re-Association of Operational Address — (File: `src/staking/staking.cairo`)

---

### Summary

The `eligible_operational_addresses` mapping in the `Staking` contract is never cleared after `change_operational_address` executes. This leaves a permanent stale consent entry that allows a staker to re-associate with a previously used operational address without requiring fresh consent from that address, bypassing the two-step consent mechanism entirely.

---

### Finding Description

The Staking contract implements a two-step consent mechanism for changing operational addresses:

1. The new operational address calls `declare_operational_address(staker_address)`, which writes `eligible_operational_addresses[operational_address] = staker_address`.
2. The staker calls `change_operational_address(operational_address)`, which verifies the consent entry and updates `operational_address_to_staker_address`.

The root cause is in `change_operational_address`: [1](#0-0) 

At lines 690–694, the function clears the old operational address from `operational_address_to_staker_address` and sets the new one, but **never clears `eligible_operational_addresses[operational_address]`**: [2](#0-1) 

Similarly, `remove_staker` clears `operational_address_to_staker_address` but also never clears `eligible_operational_addresses`: [3](#0-2) 

The `eligible_operational_addresses` mapping has no write-to-zero path anywhere in the contract — it is only ever written in `declare_operational_address`: [4](#0-3) 

**Attack path:**

1. `Op1` calls `declare_operational_address(staker_A)` → `eligible_operational_addresses[Op1] = staker_A`
2. `staker_A` calls `change_operational_address(Op1)` → `operational_address_to_staker_address[Op1] = staker_A`; `eligible_operational_addresses[Op1]` is **not cleared**
3. `staker_A` calls `change_operational_address(Op2)` → `operational_address_to_staker_address[Op1] = 0` (freed); `eligible_operational_addresses[Op1] = staker_A` remains stale
4. `staker_A` calls `change_operational_address(Op1)` again:
   - `operational_address_to_staker_address[Op1].is_zero()` → ✓ passes
   - `eligible_operational_addresses[Op1] == staker_A` → ✓ passes (stale entry!)
   - Succeeds — `Op1` is re-locked to `staker_A` without `Op1`'s fresh consent

---

### Impact Explanation

`staker_A` can force `Op1` back into `operational_address_to_staker_address` without `Op1` re-declaring consent. While `Op1` is locked to `staker_A`, no other staker can use `Op1` as their operational address (the `OPERATIONAL_EXISTS` check at line 671 blocks it). `staker_A` can repeat this cycle indefinitely — each time `Op1` is freed, `staker_A` can immediately re-claim it using the stale entry — permanently preventing `Op1` from being used by any other staker. This constitutes **griefing with damage to users** (the operator of `Op1` and any staker wishing to use it).

Impact: **Medium — Griefing with no profit motive but damage to users or protocol.**

---

### Likelihood Explanation

The precondition is simply that a staker has previously used an operational address and then changed away from it — a normal operational lifecycle event (e.g., rotating validator keys). No privileged access, leaked keys, or external dependencies are required. Any staker can trigger this against any operational address they previously held. Likelihood: **Medium**.

---

### Recommendation

Clear the `eligible_operational_addresses` entry for the newly adopted operational address inside `change_operational_address`, immediately after the consent check passes:

```cairo
// After verifying consent and before updating operational_address_to_staker_address:
self.eligible_operational_addresses.write(operational_address, Zero::zero());
```

This ensures each consent declaration is single-use, matching the intended design of the two-step mechanism.

---

### Proof of Concept

```
1. Op1.declare_operational_address(staker_A)
   → eligible_operational_addresses[Op1] = staker_A

2. staker_A.change_operational_address(Op1)
   → operational_address_to_staker_address[Op1] = staker_A
   → eligible_operational_addresses[Op1] = staker_A  ← NOT cleared

3. staker_A.change_operational_address(Op2)
   → operational_address_to_staker_address[Op1] = 0  (freed)
   → eligible_operational_addresses[Op1] = staker_A  ← stale

4. staker_A.change_operational_address(Op1)  ← no re-declaration by Op1
   → operational_address_to_staker_address[Op1].is_zero() == true  ✓
   → eligible_operational_addresses[Op1] == staker_A  ✓  (stale passes check)
   → operational_address_to_staker_address[Op1] = staker_A  ← Op1 re-locked

5. Repeat steps 3–4 indefinitely to permanently block Op1 from any other staker.
```

        fn change_operational_address(
            ref self: ContractState, operational_address: ContractAddress,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                self.operational_address_to_staker_address.read(operational_address).is_zero(),
                "{}",
                Error::OPERATIONAL_EXISTS,
            );
            assert!(
                !self.does_token_exist(token_address: operational_address),
                "{}",
                Error::OPERATIONAL_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            assert!(
                self.eligible_operational_addresses.read(operational_address) == staker_address,
                "{}",
                Error::OPERATIONAL_NOT_ELIGIBLE,
            );

            // Set operational address and write to storage.
            let old_address = staker_info.operational_address;
            self.operational_address_to_staker_address.write(old_address, Zero::zero());
            staker_info.operational_address = operational_address;
            self.write_staker_info(:staker_address, :staker_info);
            self.operational_address_to_staker_address.write(operational_address, staker_address);

            // Emit event.
            self
                .emit(
                    Events::OperationalAddressChanged {
                        staker_address, new_address: operational_address, old_address,
                    },
                );
        }

        fn declare_operational_address(ref self: ContractState, staker_address: ContractAddress) {
            self.general_prerequisites();
            let operational_address = get_caller_address();
            assert!(
                self.operational_address_to_staker_address.read(operational_address).is_zero(),
                "{}",
                Error::OPERATIONAL_IN_USE,
            );
            assert!(
                !self.does_token_exist(token_address: operational_address),
                "{}",
                Error::OPERATIONAL_IS_TOKEN,
            );
            if self.eligible_operational_addresses.read(operational_address) == staker_address {
                return;
            }
            self.eligible_operational_addresses.write(operational_address, staker_address);
            self.emit(Events::OperationalAddressDeclared { operational_address, staker_address });
        }
