### Title
`EVM.createCadenceOwnedAccount()` Implicitly Deploys an EVM Smart Contract Without User Awareness - (File: `fvm/evm/stdlib/contract.cdc`)

---

### Summary

Calling `EVM.createCadenceOwnedAccount()` silently deploys a full EVM smart contract (the COA contract) consuming `723,000` EVM gas units as a mandatory side effect. The function name implies account creation, not contract deployment. A user who calls this function without knowing about the implicit deployment may have their transaction fail due to insufficient compute budget, or unknowingly consume far more computation than anticipated.

---

### Finding Description

`EVM.createCadenceOwnedAccount()` in `fvm/evm/stdlib/contract.cdc` creates a Cadence-Owned Account (COA) resource and immediately triggers an EVM contract deployment:

```cadence
fun createCadenceOwnedAccount(): @CadenceOwnedAccount {
    pre { !self.isPaused(): "EVM operations are temporarily paused" }
    let acc <-create CadenceOwnedAccount()
    let addr = InternalEVM.createCadenceOwnedAccount(uuid: acc.uuid)  // ← triggers EVM deploy
    acc.initAddress(addressBytes: addr)
    emit CadenceOwnedAccountCreated(...)
    return <-acc
}
``` [1](#0-0) 

`InternalEVM.createCadenceOwnedAccount` maps to `newInternalEVMTypeCreateCadenceOwnedAccountFunction` in `fvm/evm/impl/impl.go`, which calls `handler.DeployCOA(uint64(uuid))`: [2](#0-1) 

`DeployCOA` calls the internal `deployCOA` function, which constructs and executes a full EVM contract deployment call using `coa.ContractBytes` with a gas limit of `coa.ContractDeploymentRequiredGas`: [3](#0-2) 

The required gas for this implicit deployment is hardcoded at **723,000 EVM gas units**: [4](#0-3) 

The `checkGasLimit` function checks this against the remaining Flow computation budget before proceeding: [5](#0-4) 

If the transaction's compute limit is insufficient to cover the 723,000-gas COA contract deployment, the call panics with `ErrInsufficientComputation` and the entire transaction reverts — even though the user only intended to "create an account."

---

### Impact Explanation

Any unprivileged Cadence transaction author who calls `EVM.createCadenceOwnedAccount()` without budgeting for the implicit 723,000-gas EVM contract deployment will have their transaction fail. If they do budget for it, they pay significantly more computation than the function name implies. The function name `createCadenceOwnedAccount` gives no indication that it deploys an EVM smart contract. This is the direct analog to the original report: a user-facing action (account creation vs. liquidity minting) silently triggers an expensive contract deployment without the user's explicit awareness or consent.

---

### Likelihood Explanation

`EVM.createCadenceOwnedAccount()` is the standard, publicly documented entry point for any user who wants to interact with EVM from Cadence. It is called in every COA setup flow. Any Cadence transaction author — including those following official documentation or tutorials — will trigger this implicit deployment. The likelihood is high.

---

### Recommendation

1. Rename or document the function to make the EVM contract deployment explicit (e.g., `createAndDeployCadenceOwnedAccount()`), or emit a clear pre-condition error message indicating the required compute budget.
2. Consider separating the COA resource creation from the EVM contract deployment so users can explicitly opt into the deployment step, mirroring the mitigation suggested in the original report.
3. At minimum, add inline documentation to `createCadenceOwnedAccount()` in `contract.cdc` stating that it deploys an EVM contract consuming approximately 723,000 EVM gas units.

---

func (h *ContractHandler) deployCOA(uuid uint64) (*types.Result, error) {
	// step 1 - check enough computation is available
	gaslimit := types.GasLimit(coa.ContractDeploymentRequiredGas)
	err := h.checkGasLimit(gaslimit)
	if err != nil {
		return nil, err
	}

	// step 2 - allocate a new address for the COA
	target := h.addressAllocator.AllocateCOAAddress(uuid)

	// step 3 - create a COA deployment call
	factory := h.addressAllocator.COAFactoryAddress()
	factoryAccount := h.AccountByAddress(factory, false)
	factoryNonce := factoryAccount.Nonce()
	call := types.NewDeployCallWithTargetAddress(
		factory,
		target,
		coa.ContractBytes,
		uint64(gaslimit),
		new(big.Int),
		factoryNonce,
	)

	// step 4 - execute the call
	res, err := h.executeAndHandleCall(call, nil, false)
	if err != nil {
		return nil, err
	}

	// step 5 - if successful COA metrics
	h.backend.SetNumberOfDeployedCOAs(factoryNonce)
	return res, nil
}

// checkGasLimit checks if enough computation is left in the environment

func (h *ContractHandler) checkGasLimit(limit types.GasLimit) error {
	// check gas limit against what has been left on the transaction side
	usage := common.ComputationUsage{
		Kind:      environment.ComputationKindEVMGasUsage,
		Intensity: uint64(limit),
	}
	if !h.backend.ComputationAvailable(usage) {
		return types.ErrInsufficientComputation
	}
	return nil
}
