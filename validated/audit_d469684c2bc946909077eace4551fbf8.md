### Title
Missing Access Control on `update_rewards` Allows Any Caller to Grief Consensus Reward Distribution - (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` is documented as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged actor can call it with `disable_rewards: true`, consuming the global per-block reward slot without distributing rewards. The legitimate sequencer call for that block then reverts with `REWARDS_ALREADY_UPDATED`, permanently destroying the block's yield for all stakers and delegators.

---

### Finding Description

The specification at `docs/spec.md` line 1645 explicitly states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 enforces only `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no sequencer or role check whatsoever:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);  // <-- consumes the slot

    if disable_rewards || self.is_pre_consensus() {
        return;   // <-- exits without distributing any rewards
    }
    ...
}
```

`last_reward_block` is a **global** storage variable (line 187). The guard `current_block_number > last_reward_block` means only **one** call to `update_rewards` is accepted per block, across all stakers. Once the slot is consumed, every subsequent call in the same block reverts.

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` before the sequencer does:

1. Passes all validation (staker must exist, be active, have non-zero balance — all publicly readable on-chain).
2. Writes `last_reward_block = current_block_number`.
3. Returns immediately without distributing rewards.
4. The sequencer's legitimate call in the same block hits `REWARDS_ALREADY_UPDATED` and reverts.
5. The block's rewards are permanently lost — they are never credited to `unclaimed_rewards_own` or forwarded to pools.

This is structurally identical to the reference vulnerability: a two-step operation (consume the nonce/slot → distribute) where the first step can be replicated by a third party, causing the second step to fail.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

- Consensus-mode block rewards are computed once per block and credited only inside `_update_rewards`. If the slot is consumed with `disable_rewards: true`, those rewards are never minted/credited; there is no catch-up mechanism.
- The attack affects **all** stakers and delegators simultaneously because `last_reward_block` is global.
- Sustained griefing (one cheap transaction per block) can deny 100 % of consensus rewards indefinitely.

---

### Likelihood Explanation

**High.**

- The function is fully public; no special key or role is required.
- The only prerequisite is knowing a valid, active staker address with non-zero balance — trivially obtained from on-chain events (`NewStaker`, `StakeOwnBalanceChanged`).
- The cost to the attacker is one transaction per block (gas only); there is no economic barrier.
- Starknet's sequencer ordering means a malicious actor can submit the griefing transaction in the same block as the legitimate sequencer call and have it ordered first.

---

### Recommendation

Add an access-control check matching the specification. The simplest fix is to restrict `update_rewards` to a dedicated role (e.g., the existing `SEQUENCER_ROLE` or a new `REWARDS_MANAGER_ROLE`):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer();   // enforce spec: "Only starkware sequencer"
    self.general_prerequisites();
    ...
}
```

Alternatively, verify `get_caller_address() == sequencer_address` against a stored sequencer address, consistent with how `assert_caller_is_attestation_contract` is implemented at line 2219.

---

### Proof of Concept

```
Block N arrives.

1. Attacker reads on-chain state: finds active staker `S` with non-zero STRK balance.

2. Attacker submits (before sequencer):
       staking.update_rewards(staker_address: S, disable_rewards: true)
   → general_prerequisites() passes (not paused, caller != 0)
   → current_block_number (N) > last_reward_block (N-1) → passes
   → last_reward_block.write(N)          ← slot consumed
   → disable_rewards == true → return    ← no rewards distributed

3. Sequencer submits its legitimate call:
       staking.update_rewards(staker_address: S, disable_rewards: false)
   → current_block_number (N) > last_reward_block (N) → FALSE
   → REWARDS_ALREADY_UPDATED panic       ← reverts

Result: block N's consensus rewards are permanently lost for all stakers and delegators.
Repeat every block to deny all consensus rewards indefinitely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
