### Title
Pool Contract `claim_rewards` Bypasses Staking Contract Pause - (File: `src/pool/pool.cairo`)

### Summary

The Staking contract enforces a pause via `general_prerequisites()` on all its state-changing functions, including its own `claim_rewards`. However, the Pool contract's `claim_rewards` function performs no pause check whatsoever, allowing any pool member (or their reward address) to withdraw accumulated STRK rewards from the Pool contract even while the Staking contract is paused.

### Finding Description

The Staking contract stores a pause flag and exposes it through `IStakingPause`. Every state-mutating function in the Staking contract calls `self.general_prerequisites()` as its first action, which asserts the contract is not paused. [1](#0-0) 

The pause tests confirm that `claim_rewards` on the Staking contract is blocked when paused: [2](#0-1) 

The Staking contract's `claim_rewards` calls `general_prerequisites()`: [3](#0-2) 

By contrast, the Pool contract has **no pause mechanism at all** — no `is_paused` storage, no pause interface, and no call to the Staking contract to check its pause state. The Pool's `claim_rewards` function only checks caller authorization, then immediately computes and transfers STRK rewards: [4](#0-3) 

The STRK transfer at the end of Pool's `claim_rewards` moves tokens directly from the Pool contract's balance to the reward address, with no interaction with the Staking contract that could be intercepted by the pause: [5](#0-4) 

The Pool contract has no `is_paused` storage field and no pause-related interface: [6](#0-5) 

### Impact Explanation

When the Staking contract is paused (e.g., due to a discovered reward-accounting bug), the operator's intent is to freeze all fund movements. The Staking-level `claim_rewards` is correctly blocked. However, pool members can still call `Pool::claim_rewards` directly, draining their accumulated STRK rewards from the Pool contract's balance. If the pause was triggered precisely because rewards were miscalculated (e.g., inflated due to an accounting error), pool members can claim those inflated rewards before the issue is corrected — constituting **theft of unclaimed yield**. At minimum, the inconsistency undermines the operator's ability to halt the protocol in an emergency.

**Matched impact:** High — Theft of unclaimed yield.

### Likelihood Explanation

`Pool::claim_rewards` is a public function callable by any pool member or their designated reward address. There is no privileged role required. During a pause event, rational pool members have a direct financial incentive to call this function immediately to secure their (potentially inflated) rewards before the protocol is unpaused and corrected. The entry path is fully unprivileged and requires no special conditions beyond having a pool membership.

### Recommendation

Add a check at the start of `Pool::claim_rewards` (and `Pool::change_reward_address` for consistency) that queries the Staking contract's pause state and reverts if it is paused:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    // Add pause check:
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
    // ... rest of function
}
```

Alternatively, introduce a mirrored pause flag in the Pool contract that the Staking contract sets when it pauses/unpauses.

### Proof of Concept

1. A pool member delegates stake and accumulates STRK rewards over several epochs.
2. The security agent calls `IStakingPause::pause()` on the Staking contract (e.g., due to a discovered reward inflation bug).
3. The pool member calls `Pool::claim_rewards(pool_member)` directly on the Pool contract.
4. The Pool contract has no pause check — it computes rewards from its internal `cumulative_rewards_trace` and calls `reward_token.checked_transfer(recipient: reward_address, amount: rewards.into())`.
5. The STRK transfer succeeds. The pool member has claimed (potentially inflated) rewards despite the protocol being paused.
6. Calling `Staking::claim_rewards` in the same state would revert with `"Contract is paused"` — confirming the inconsistency. [7](#0-6) [3](#0-2)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
