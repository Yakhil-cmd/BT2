### Title
Relayer Precompile Returns Raw Protobuf Bytes Instead of ABI-Encoded `bytes`, Causing All Caller Contracts to Revert - (File: x/cronos/keeper/precompiles/relayer.go, x/cronos/keeper/precompiles/utils.go)

### Summary
The `IRelayerFunctions` Solidity interface declares every IBC relay function as `returns (bytes calldata)`. The Go precompile's generic `exec` helper, however, returns raw protobuf-marshalled bytes with no ABI encoding. Any Solidity contract that calls the relayer precompile through the published interface and attempts to use the return value will have its ABI decoder revert, permanently blocking IBC relay operations submitted via the precompile.

### Finding Description

The Solidity interface published for the relayer precompile declares all IBC relay functions with a `bytes calldata` return type:

```solidity
// x/cronos/events/bindings/src/RelayerFunctions.sol
function createClient(bytes calldata data) external payable returns (bytes calldata);
function recvPacket(bytes calldata data)   external payable returns (bytes calldata);
// … all 18 relay methods follow the same pattern
``` [1](#0-0) 

The Go precompile's `Run` method dispatches every IBC operation through the generic `exec` helper:

```go
case RecvPacket:
    res, err = exec(e, bc.ibcKeeper.RecvPacket)
// … same for all 18 methods
``` [2](#0-1) 

`exec` serialises the response with a raw protobuf marshal and returns those bytes directly:

```go
output, err := e.cdc.Marshal(res)   // raw protobuf, NOT ABI-encoded
return output, nil
``` [3](#0-2) 

ABI-encoding of a `bytes` return value requires a 32-byte offset pointer followed by a 32-byte length prefix before the payload. Raw protobuf output has neither. When a Solidity ≥0.8 contract calls the precompile through the typed interface and the compiler inserts the standard ABI decoder for the `bytes` return, the decoder reads the first 32 bytes of the protobuf blob as an offset, jumps to a position that is almost certainly outside the returned data, and reverts with an out-of-bounds read.

### Impact Explanation

Every Solidity contract that calls the relayer precompile via the `IRelayerFunctions` interface and consumes the return value will unconditionally revert. This permanently prevents:
- IBC packet relay (`recvPacket`, `acknowledgement`, `timeout`, `timeoutOnClose`) through the precompile
- IBC channel and connection lifecycle operations (`channelOpenInit`, `channelOpenTry`, etc.) through the precompile
- IBC client operations (`createClient`, `updateClient`, `upgradeClient`) through the precompile

This matches the **High** impact class: *permanent or long-lived inability for honest users to process valid transactions, bridge/conversion flows, precompile calls, or IBC transfers under normal network assumptions.*

### Likelihood Explanation

The `IRelayerFunctions.sol` interface is the canonical, published ABI for the relayer precompile at address `0x0000000000000000000000000000000000000065`. Any developer who follows the documented interface and writes a Solidity contract that uses the return value will hit this revert on every call. No special conditions or privileges are required.

### Recommendation

In `exec` (`utils.go`), ABI-encode the protobuf output before returning it, so the return data matches the `bytes` type declared in the Solidity interface:

```go
// After marshalling:
abiBytes, err := method.Outputs.Pack(output)   // wraps in ABI bytes encoding
return abiBytes, nil
```

Alternatively, change the Solidity interface to declare `returns (bytes memory)` and document that the returned bytes are raw protobuf, and update all callers accordingly.

### Proof of Concept

1. Deploy a Solidity contract on Cronos that calls the relayer precompile:
   ```solidity
   IRelayerFunctions rel = IRelayerFunctions(0x0000000000000000000000000000000000000065);
   bytes memory result = rel.createClient(encodedMsg); // ABI decoder fires here
   ```
2. Submit a valid `MsgCreateClient` protobuf payload as `encodedMsg`.
3. The precompile executes successfully and returns raw protobuf bytes.
4. The Solidity ABI decoder attempts to read a 32-byte offset from the first word of the protobuf blob, computes an out-of-range data pointer, and the transaction reverts with no state change.
5. The same revert occurs for every one of the 18 relay methods, making the entire relayer precompile unusable from Solidity contracts that follow the published interface. [4](#0-3) [5](#0-4) [6](#0-5)

		return nil, err

func exec[Req any, PReq interface {
	*Req
	proto.Message
}, Resp proto.Message](
	e *Executor,
	action func(context.Context, PReq) (Resp, error),
) ([]byte, error) {
	msg := PReq(new(Req))
	if err := e.cdc.Unmarshal(e.input, msg); err != nil {
		return nil, fmt.Errorf("fail to Unmarshal %T %w", msg, err)
	}

	signers, _, err := e.cdc.GetMsgV1Signers(msg)
	if err != nil {
		return nil, fmt.Errorf("fail to get signers %w", err)
	}

	if len(signers) != 1 {
		return nil, errors.New("don't support multi-signers message")
	}
	caller := common.BytesToAddress(signers[0])
	if caller != e.caller {
		return nil, fmt.Errorf("caller is not authenticated: expected %s, got %s", e.caller.Hex(), caller.Hex())
	}

	var res Resp
	if err := e.stateDB.ExecuteNativeAction(e.contract, e.converter, func(ctx sdk.Context) error {
		var err error
		res, err = action(ctx, msg)
		return err
	}); err != nil {
		return nil, err
	}

	output, err := e.cdc.Marshal(res)
	if err != nil {
		return nil, fmt.Errorf("fail to Marshal %T %w", res, err)
	}
	return output, nil
