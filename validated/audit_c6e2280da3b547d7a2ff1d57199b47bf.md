### Title
Pusher delegation cannot be effectively revoked before deadline — creator replays the same nonce-free signature to re-establish it — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` verifies an EIP-191 signature whose preimage contains no nonce. After a pusher calls `revokePusher()`, the creator can immediately call `allowPushers` again with the identical `(deadline, pushers[], signatures[])` arguments to re-establish the delegation. The pusher is locked into the creator's namespace until the deadline timestamp expires, regardless of how many times they revoke.

---

### Finding Description

**Root cause — no nonce in the signed preimage**

`allowPushers` constructs the hash as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The five fields are all static for the lifetime of the signature. Nothing in the preimage changes when `revokePusher` clears `namespaceRemapping[pusher]`:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

Because the contract never records that the signature was consumed, the creator can call `allowPushers` with the same calldata an unlimited number of times before `block.timestamp > deadline`.

**The code comment acknowledges the concern but misidentifies the fix**

```
// The deadline is likewise required: the signed consent carries no timestamp of
// its own, so an undated signature could re-establish a delegation AFTER the
// pusher revoked it.
``` [3](#0-2) 

The deadline prevents replay *after* it expires. It does not prevent replay *before* it expires, which is exactly the window in which revocation is supposed to take effect.

**Contrast with `updateBySignature`**, which correctly neutralizes replay via the slot's embedded 56-bit timestamp monotonicity check — no nonce needed there because the payload itself is stateful: [4](#0-3) 

**Contrast with `allowContractPushers`**, which uses a live `isPusher(creator)` staticcall and explicitly notes "there is nothing to replay": [5](#0-4) 

Only the EOA-signature path (`allowPushers`) is missing replay protection.

---

### Impact Explanation

`namespaceRemapping[pusher]` determines which creator's namespace receives every fallback push from that pusher address: [6](#0-5) 

If the creator's namespace is consumed by a `CompressedOracleV1`-backed price provider that feeds a live Metric OMM pool, the pusher's data continues to drive `getBidAndAskPrice()` for that pool even after the pusher revokes. A pusher who discovers their data is being misused (e.g., the creator is routing it to a pool with manipulated parameters) cannot stop the attribution until the deadline passes. Every swap executed against that pool during the replay window receives a price derived from the pusher's feed, satisfying the **bad-price execution** impact gate.

Additionally, the pusher's own namespace remains empty for the duration, so any pool or provider that legitimately reads from the pusher's own namespace receives stale/zero data — a **zero-state fail-open** risk for any downstream consumer of the pusher's direct feed.

---

### Likelihood Explanation

- The creator already holds the signed bytes (they submitted them in the first `allowPushers` call).
- Re-submitting is a single permissionless transaction with no additional cost.
- Deadlines are typically set days or weeks in the future (the test suite uses `block.timestamp + 1 days`).
- No privileged role is required; any namespace creator who received a pusher's consent signature can execute this. [7](#0-6) 

---

### Recommendation

Add a per-`(pusher, creator)` nonce to the signed preimage and invalidate it on first use:

```solidity
mapping(address => mapping(address => uint256)) public delegationNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid,
        address(this),
        deadline,
        pusher,
        msg.sender,
        delegationNonce[pusher][msg.sender]   // <-- add nonce
    ))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
delegationNonce[pusher][msg.sender]++;        // <-- consume nonce
namespaceRemapping[pusher] = msg.sender;
```

This ensures each signed consent can be used exactly once, so `revokePusher()` permanently terminates the current delegation epoch.

---

### Proof of Concept

```
1. Pusher signs consent: hash = keccak256(chainid, oracle, deadline=T+7days, pusher, creator)
2. Creator calls allowPushers(T+7days, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓
3. Pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓ (pusher believes they are free)
4. Creator calls allowPushers(T+7days, [pusher], [sig])  ← SAME calldata
   → _ensureDeadline passes (block.timestamp < T+7days)
   → ECDSA.recover returns pusher  (signature is still valid)
   → namespaceRemapping[pusher] = creator  ← delegation silently restored
5. Pusher's subsequent fallback pushes land in creator's namespace.
   Creator's pool price provider reads creator's namespace → pool swaps
   execute at prices derived from pusher's data.
6. Steps 3-5 repeat indefinitely until block.timestamp > T+7days.
```

    ///         monotonicity check below — no deadline is needed.
    function updateBySignature(address feedCreator, uint256 newSlotValue, bytes calldata signature)
        external
        override
        returns (bool)
    {
        require(feedCreator != address(0), InvalidNamespace());

        uint256 namespace;
        assembly ("memory-safe") {
            namespace := shl(96, feedCreator) // [creator:20][zeros:12]
        }

        uint8 slotId = uint8(newSlotValue); // LSB
        TimeMs timestampMs = toTimeMs(newSlotValue >> 8 & X56);
        timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
        bytes32 key = bytes32(namespace | uint256(slotId));
        uint256 old = uint256(_loadStorage(key));
        TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

        bool newer = timestampMs.isAfter(oldTimestampMs);
        if (!newer) {
            return false;
        }

        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(this), feedCreator, newSlotValue))
        );
        require(feedCreator == ECDSA.recover(hash, signature));

        _writeStorage(key, bytes32(newSlotValue & ~uint256(0xff)));

        return true;
