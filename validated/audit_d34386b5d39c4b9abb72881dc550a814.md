### Title
Unauthenticated Public Exposure of Complete Account Transaction History via Extended Access API - (File: `engine/access/rest/router/routes_experimental.go`)

### Summary

The Flow Access Node's experimental REST API exposes a `GET /accounts/{address}/transactions` endpoint that returns the full indexed transaction history for any Flow account address — including block height, timestamp, transaction ID, transaction index, account roles, and optionally the full transaction body (script, arguments, payer, authorizers, signatures) and result — to any unauthenticated caller over the network. No authentication, authorization, or ownership check is performed at any layer. Any unprivileged network peer can enumerate the complete on-chain activity of any account by supplying its address.

### Finding Description

The extended indexer pipeline indexes every sealed block and records, for each account address that participated in a transaction (as payer, proposer, authorizer, or event-referenced address), the block height, transaction ID, transaction index, and role. This data is stored in a Pebble database at `extendedIndexingDBPath` opened via `pstorage.SafeOpen` with no encryption.

The route is registered unconditionally in `ExperimentalRoutes`:

```
GET /accounts/{address}/transactions → routes.GetAccountTransactions
```

The handler `routes.GetAccountTransactions` calls `request.NewGetAccountTransactions`, which parses only the address, limit, cursor, and role filter from the HTTP request — no authentication token, session, or identity check is present anywhere in the call chain.

The backend `AccountTransactionsBackend.GetAccountTransactions` performs only a chain-validity check on the address (`b.chain.IsValid(address)`), then calls `b.store.ByAddress(address, cursor)` and returns all indexed transactions. With `expand=transaction`, the full `flow.TransactionBody` — including `Script`, `Arguments`, `Payer`, `Authorizers`, `PayloadSignatures`, and `EnvelopeSignatures` — is returned. With `expand=result`, the full execution result including events is returned.

The root cause is the complete absence of any access control check between the public HTTP listener and the account transaction index. The index itself is stored in plaintext Pebble (no encryption option is passed in `OpenExtendedIndexDB`), and the API layer adds no ownership or authentication gate.

**Attacker-controlled entry path:**
1. Attacker sends `GET /experimental/v1/accounts/<victim_address>/transactions?expand=transaction,result` to the Access Node's public REST port.
2. The request passes through logging middleware only, reaches `GetAccountTransactions`, address is validated as a valid chain address, and the full paginated transaction history is returned.
3. Attacker paginates using `next_cursor` to retrieve the complete history.

### Impact Explanation

Any unauthenticated network caller can retrieve the complete transaction history of any Flow account, including:
- Every transaction the account participated in (as payer, proposer, authorizer, or event-referenced party), with block height and timestamp
- With `expand=transaction`: the full Cadence script, all arguments (JSON-CDC encoded), payer address, all authorizer addresses, and all cryptographic signatures
- With `expand=result`: full execution result including emitted events

This constitutes **cross-user private data exposure**: an attacker can reconstruct the complete financial and operational history of any account, identify counterparties, infer private contract interactions, and correlate addresses — all without any credential. The data persists in the index indefinitely and is accessible even if the account owner has taken no action since the node was deployed.

**Likelihood: High.** The endpoint is reachable by any unprivileged HTTP client. No exploit tooling is required beyond a standard HTTP GET request. The only precondition is that the target Access Node has `extendedIndexingEnabled = true`, which is the intended production configuration for nodes serving this API.

### Likelihood Explanation

The endpoint is registered as a standard HTTP GET route on the public Access Node REST interface. Any client that can reach the node's REST port — including any Observer node, any dApp client, or any internet-connected host if the port is publicly exposed — can call it. The only validation is that the address is a valid Flow address, which is trivially satisfied by any known account address. The attack requires zero privileges, zero on-chain interaction, and zero cryptographic capability.

### Recommendation

1. **Short term:** Add an authentication/authorization middleware to the experimental route group that requires the caller to prove ownership of the queried address (e.g., via a signed challenge), or restrict the experimental API to authenticated operators only (e.g., via mTLS or an API key).
2. **Long term:** Evaluate whether the account transaction index and transfer history APIs should be gated behind account ownership proofs. Consider encrypting the extended Pebble index at rest (analogous to the existing `secretsdb-key` mechanism used for the Badger secrets database) so that physical access to the node's storage does not yield plaintext transaction history.

```
# Retrieve full transaction history for any account, unauthenticated:
curl -s "http://<access-node>:<rest-port>/experimental/v1/accounts/0x<victim_address>/transactions?expand=transaction,result&limit=100"

# Response includes:
# - transaction_id, block_height, timestamp, roles
# - full Cadence script (base64), arguments, payer, authorizers, signatures (with expand=transaction)
# - execution events (with expand=result)

# Paginate to retrieve complete history:
CURSOR=$(curl -s "..." | jq -r '.next_cursor')
curl -s "...?cursor=$CURSOR&expand=transaction,result&limit=100"
```
