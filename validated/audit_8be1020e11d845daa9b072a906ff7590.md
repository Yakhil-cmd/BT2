### Title
Gateway L2 Gas Price Threshold Uses Floor Instead of Ceiling, Admitting Below-Minimum Transactions - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price by multiplying `previous_block_l2_gas_price` by `min_gas_price_percentage / 100` and calling `.to_integer()`, which **truncates toward zero (floor)**. When the product is non-integer, the computed threshold is strictly lower than the intended minimum, allowing transactions whose L2 gas price is below the true threshold to pass gateway admission.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, the stateful gateway path checks whether an incoming transaction's L2 gas price meets the minimum threshold derived from the previous block's L2 gas price:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
```

`Ratio::to_integer()` from `num_rational` truncates toward zero — for a positive rational it is equivalent to `floor()`. When `min_gas_price_percentage * previous_block_l2_gas_price` is not evenly divisible by 100, the computed `threshold` is one unit lower than the true intended minimum.

**Concrete example:**
- `min_gas_price_percentage = 50`, `previous_block_l2_gas_price = 101`
- True minimum = ⌈50/100 × 101⌉ = ⌈50.5⌉ = **51**
- Computed threshold = ⌊50.5⌋ = **50**
- A transaction with `max_price_per_unit = 50` satisfies `50 < 50 == false` → **admitted**
- But 50 < 51 → it should be **rejected**

The rest of the codebase consistently uses ceiling for protocol-protective conversions: `convert_l1_to_l2_gas_price_round_up`, `l1_gas_to_sierra_gas_amount_round_up`, `sierra_gas_to_l1_gas_amount_round_up`, and `to_discounted_l1_gas` all use `.ceil()`. This function is the sole exception. [1](#0-0) [2](#0-1) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

A user can submit a transaction with an L2 gas price that is exactly `floor(min_gas_price_percentage/100 * prev_price)` — one unit below the intended minimum — and it will be accepted by the stateful gateway validator and forwarded to the mempool. This bypasses the economic spam-prevention invariant that the gateway is designed to enforce. The effect is systematic: for any `previous_block_l2_gas_price` that is not divisible by `100 / gcd(min_gas_price_percentage, 100)`, every block produces a window where one gas-price unit below the intended floor is accepted.

### Likelihood Explanation

The condition is triggered whenever `(min_gas_price_percentage * previous_block_l2_gas_price) % 100 != 0`. With a default `min_gas_price_percentage` of a typical value (e.g., 50) and a dynamically-adjusted L2 gas price that is an odd integer, this occurs on roughly half of all blocks. Any user who monitors the previous block's L2 gas price and submits a transaction at exactly `floor(threshold)` will reliably exploit this. [3](#0-2) 

### Recommendation

Replace `.to_integer()` with `.ceil().to_integer()` to match the ceiling convention used everywhere else in the fee/gas conversion code:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()          // ← add this
    .to_integer();
``` [4](#0-3) 

### Proof of Concept

Using the existing test harness in `crates/apollo_gateway/src/stateful_transaction_validator_test.rs`:

```rust
// prev_l2_gas_price = 101, min_gas_price_percentage = 50
// floor(50/100 * 101) = 50  → tx with price=50 is ACCEPTED (wrong)
// ceil(50/100 * 101)  = 51  → tx with price=50 should be REJECTED
#[case::floor_rounding_admits_below_threshold(
    101_u128.try_into().unwrap(),  // previous_block_l2_gas_price
    50,                            // min_gas_price_percentage
    50_u128.into(),                // tx_gas_price_per_unit  (= floor, below true ceil threshold)
    Err(StarknetError { ... })     // should fail, but currently Ok(())
)]
```

The existing passing test `tx_gas_price_meets_threshold_with_factor_pass` uses `prev=100, pct=50, tx=50` — a case where `50/100 * 100 = 50.0` exactly, so floor == ceil and the bug is invisible. The bug only manifests when the product is non-integer. [5](#0-4)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
