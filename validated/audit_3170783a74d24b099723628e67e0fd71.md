### Title
Missing Zero Check on `msg_hash` in OT-Based ECDSA `sign` Allows Degenerate Threshold Signature Production - (File: `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

The OT-based ECDSA `sign` function accepts `msg_hash = 0` without rejection. The robust ECDSA `sign` function in the same codebase explicitly rejects a zero message hash to prevent algebraic attacks. The OT-based variant is missing this guard, allowing any caller to produce a valid threshold ECDSA signature on the zero scalar, which encodes the product `k·x` in the signature `s` component and enables private key extraction when combined with presignature reuse.

---

### Finding Description

`src/ecdsa/ot_based_ecdsa/sign.rs` `sign()` accepts `msg_hash: Scalar` and performs several input validations (participant count, coordinator membership, threshold bounds), but never checks whether `msg_hash` is the zero scalar: [1](#0-0) 

The function proceeds directly to `fut_wrapper`, which calls `compute_signature_share`: [2](#0-1) 

With `msg_hash = 0`, the share formula reduces to:

```
s_i = 0 * k_i + r * sigma_i  =  r * sigma_i
```

Summing across all participants gives `s = r · sigma = r · k · x`, a valid ECDSA signature (the coordinator's final verification at line 129 passes). The signature `(R, s)` with `R = k⁻¹·G` and `s = r·k·x` satisfies the ECDSA verification equation because `s⁻¹·(0·G + r·X) = (r·k·x)⁻¹·r·x·G = k⁻¹·G = R`.

By contrast, the robust ECDSA `sign` function in the same repository explicitly rejects this input: [3](#0-2) 

The OT-based `sign` function's own docstring acknowledges the risk of signing arbitrary scalars but does not enforce the zero-hash guard: [4](#0-3) 

The `ReconstructionLowerBound` and `MaxMalicious` wrapper types carry no construction-time validation, so a zero threshold can also be passed without rejection at the type level: [5](#0-4) 

---

### Impact Explanation

When `msg_hash = 0` is accepted:

1. **Valid threshold signature on the zero hash is produced.** Each honest participant contributes `s_i = r · sigma_i` without objection. The coordinator aggregates a fully valid ECDSA signature. This constitutes creation of a valid threshold signature for an attacker-chosen input (the zero scalar).

2. **Private key extraction via presignature reuse.** If the same presignature is reused (a separate documented risk), an attacker holding both a zero-hash signature `s₁ = r·k·x` and a normal signature `s₂ = k·(h' + r·x)` on the same presignature can recover the private key:
   - `k = (s₂ − s₁) / h'`
   - `x = s₁ / (k · r)`

   The zero-hash signature is the enabling precondition for this two-signature key-extraction path.

**Impact: High** — Honest participants accept and contribute to a degenerate signing session that produces a valid threshold signature on a cryptographically weak input, and the resulting signature directly enables private key reconstruction when combined with presignature reuse.

---

### Likelihood Explanation

A malicious or buggy caller (application layer, coordinator, or any party constructing the `sign` call) can pass `msg_hash = 0` directly. No network-level coercion of other participants is required; each participant receives `msg_hash` as a local function argument. The check is trivially one line and its absence is an oversight evidenced by the identical guard present in the sibling robust ECDSA path.

**Likelihood: Low** — Requires a caller to deliberately or accidentally pass the zero scalar; presignature reuse is a further prerequisite for full key extraction.

---

### Recommendation

Add the same zero-hash guard that the robust ECDSA `sign` function already enforces, immediately after the threshold check in `src/ecdsa/ot_based_ecdsa/sign.rs`:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0: signing the zero scalar is cryptographically unsafe".to_string(),
    ));
}
```

This mirrors the existing guard at: [3](#0-2)

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    presignature: &RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<Scalar, ProtocolError> {
    // Round 1
    // Linearize ki
    // Spec 1.1
    let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
    let k_i = lambda * presignature.k;

    // Linearize sigmai
    // Spec 1.2
    let sigma_i = lambda * presignature.sigma;

    // Compute si = h * ki + Rx * sigmai
    // Spec 1.3
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)
}
