### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Wrong Admission Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` validates a transaction's `max_price_per_unit` against `block_info.strk_gas_prices.l2_gas_price` — the L2 gas price of the **already-committed** previous block. However, the actual price that will govern execution of the admitted transaction is `next_l2_gas_price`, a separately computed EIP-1559 field stored in the block header. The two values diverge whenever gas usage deviates from the target. The code itself acknowledges the bug with `// TODO(Arni): getnext_l2_gas_price from the block header.` The gateway therefore enforces the wrong price floor, either rejecting valid transactions (when the price is falling) or admitting transactions that will fail at execution (when the price is rising).

### Finding Description

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` constructs a `BlockInfo` by reading `block_header.l2_gas_price.price_in_fri` and placing it in `strk_gas_prices.l2_gas_price`: [1](#0-0) 

`validate_resource_bounds` then reads that field as `previous_block_l2_gas_price` and uses it as the reference for the admission threshold: [2](#0-1) 

The TODO comment on line 229 explicitly flags that `next_l2_gas_price` should be used instead.

The block header stores both values as distinct fields. `l2_gas_price` is the price that was in effect **during** the committed block. `next_l2_gas_price` is the EIP-1559-adjusted price that will be in effect for the **next** block — the one the admitted transaction will actually execute in: [3](#0-2) 

The consensus orchestrator computes `next_l2_gas_price` via `calculate_next_l2_gas_price_for_fin`, which applies the EIP-1559 formula and can move the price by up to `1/gas_price_max_change_denominator` per block: [4](#0-3) 

The batcher's own `get_block_info` also reads only `l2_gas_price`, not `next_l2_gas_price`, confirming that `next_l2_gas_price` is never surfaced to the gateway: [5](#0-4) 

### Impact Explanation

**Direction 1 — Gateway rejects valid transactions (price falling):** When consecutive blocks are under-full, `next_l2_gas_price < l2_gas_price`. The gateway computes a threshold from the higher stale price and rejects transactions whose `max_price_per_unit` satisfies the actual execution price but not the stale one. These transactions are valid and would succeed at execution, but are incorrectly denied admission.

**Direction 2 — Gateway admits transactions that will fail (price rising):** When blocks are over-full, `next_l2_gas_price > l2_gas_price`. The gateway computes a threshold from the lower stale price and admits transactions whose `max_price_per_unit` is above the stale threshold but below the actual execution price. These transactions pass gateway admission, enter the mempool, and then fail at execution with `MaxGasPriceTooLow`.

Both directions represent incorrect admission decisions. Direction 1 matches the "High. Mempool/gateway/RPC admission … rejects valid transactions before sequencing" impact category.

### Likelihood Explanation

The EIP-1559 mechanism is always active. Any sustained period of blocks that are consistently above or below the gas target will cause `next_l2_gas_price` to diverge from `l2_gas_price`. No special attacker action is required; normal network load variation is sufficient to trigger the mismatch. The `min_gas_price_percentage` config multiplier scales the threshold, but does not eliminate the divergence between the two price fields.

### Recommendation

Replace the read of `block_info.strk_gas_prices.l2_gas_price` in `validate_resource_bounds` with a read of `block_header_without_hash.next_l2_gas_price`. Extend `GatewayFixedBlockStateReader::get_block_info` (or add a dedicated method) to expose `next_l2_gas_price` from the stored header, and use that value as the reference price for the admission threshold. [6](#0-5) 

### Proof of Concept

1. The network runs at high load for N blocks; `next_l2_gas_price` rises to `P_next = 2 × P_prev` via EIP-1559.
2. A user submits an `AllResources` invoke transaction with `l2_gas.max_price_per_unit = 1.5 × P_prev`.
3. Gateway calls `validate_resource_bounds`. It reads `previous_block_l2_gas_price = P_prev` (the stale field). With `min_gas_price_percentage = 100`, threshold = `P_prev`. The transaction passes: `1.5 × P_prev ≥ P_prev`. ✓
4. The transaction enters the mempool and is included in the next block, which executes at `next_l2_gas_price = 2 × P_prev`.
5. Blockifier's `check_fee_bounds` checks `max_price_per_unit ≥ actual_gas_price`: `1.5 × P_prev < 2 × P_prev`. The transaction fails with `MaxGasPriceTooLow`.

Conversely, for the rejection direction: if `next_l2_gas_price = 0.5 × P_prev` and `min_gas_price_percentage = 100`, a transaction with `max_price_per_unit = 0.7 × P_prev` is rejected by the gateway (below `P_prev`) even though it would succeed at execution (above `0.5 × P_prev`).

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
```
