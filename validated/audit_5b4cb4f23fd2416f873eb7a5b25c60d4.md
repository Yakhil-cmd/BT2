### Title
RPC Simulation/Trace Fails for Valid Transactions Using `range_check96`, `add_mod`, or `mul_mod` Builtins — (`crates/apollo_rpc_execution/src/objects.rs`)

---

### Summary

`vm_resources_to_execution_resources` has an explicit `TODO` acknowledging three unhandled `BuiltinName` variants. Any transaction that blockifier successfully executes using a contract with `range_check96`, `add_mod`, or `mul_mod` builtins will cause `starknet_simulateTransactions` and `starknet_traceTransaction` to return `Err(ExecutionError::UnknownBuiltin)` instead of a valid trace.

---

### Finding Description

In `vm_resources_to_execution_resources`, the `match` over `builtin_name` handles 8 variants but leaves three commented out with a `TODO`: [1](#0-0) 

```rust
// TODO(DanB): what about the following?
// BuiltinName::range_check96 => todo!(),
// BuiltinName::add_mod => todo!(),
// BuiltinName::mul_mod => todo!(),
_ => {
    return Err(ExecutionError::UnknownBuiltin { builtin_name });
}
```

This function is called from `FunctionInvocation::try_from((CallInfo, GasVector))` via the `?` operator: [2](#0-1) 

The `FunctionInvocation` type is the building block for all `TransactionTrace` variants (`InvokeTransactionTrace`, `DeclareTransactionTrace`, etc.), which are the output of `starknet_simulateTransactions` and `starknet_traceTransaction`. [3](#0-2) 

These three builtins are **fully supported by blockifier** — they appear in every versioned constants file with defined gas costs: [4](#0-3) 

The `starknet_api` `Builtin` enum already has `AddMod`, `MulMod`, and `RangeCheck96` variants: [5](#0-4) 

And `cende_client_types.rs` already has a complete `From<BuiltinName>` conversion for all three: [6](#0-5) 

---

### Impact Explanation

A user deploys or calls any Sierra contract that uses `range_check96`, `add_mod`, or `mul_mod` builtins (standard Cairo 1 arithmetic). Blockifier executes the transaction successfully and records the builtin counts in `VmExecutionResources.builtin_instance_counter`. When the RPC layer converts the resulting `CallInfo` into a `FunctionInvocation` for the trace response, `vm_resources_to_execution_resources` hits the `_` arm and returns `Err(ExecutionError::UnknownBuiltin)`. The simulation or trace RPC call returns an error for a transaction that was validly executed, violating the invariant that any successfully executed transaction is traceable.

Impact category: **High — RPC simulation/tracing returns an error instead of a valid trace for a valid transaction.**

---

### Likelihood Explanation

`range_check96` is used by the Cairo 1 compiler for 96-bit range checks, which appear in many arithmetic-heavy contracts. `add_mod` and `mul_mod` are used by modular arithmetic operations. Any contract using these features triggers the bug. The trigger is fully unprivileged: any user can deploy such a contract and invoke it.

---

### Recommendation

Add the three missing arms to the `match` in `vm_resources_to_execution_resources`, mapping to the already-existing `Builtin` variants:

```rust
BuiltinName::range_check96 => {
    builtin_instance_counter.insert(Builtin::RangeCheck96, count)
}
BuiltinName::add_mod => {
    builtin_instance_counter.insert(Builtin::AddMod, count)
}
BuiltinName::mul_mod => {
    builtin_instance_counter.insert(Builtin::MulMod, count)
}
```

The target `Builtin` variants already exist in `starknet_api::execution_resources::Builtin` and the conversion pattern is already established in `cende_client_types.rs`. [7](#0-6) 

---

### Proof of Concept

```rust
#[test]
fn test_range_check96_causes_unknown_builtin_error() {
    use cairo_vm::vm::runners::cairo_runner::ExecutionResources as VmExecutionResources;
    use cairo_vm::types::builtin_name::BuiltinName;
    use std::collections::HashMap;
    use starknet_api::execution_resources::GasVector;

    let mut counter = HashMap::new();
    counter.insert(BuiltinName::range_check96, 5usize);

    let vm_resources = VmExecutionResources {
        n_steps: 100,
        n_memory_holes: 0,
        builtin_instance_counter: counter,
    };

    let result = vm_resources_to_execution_resources(
        vm_resources,
        GasVector::default(),
    );

    // This asserts the bug: a successfully-executed transaction
    // with range_check96 usage cannot be traced.
    assert!(matches!(
        result,
        Err(ExecutionError::UnknownBuiltin { .. })
    ));
}

```rust
fn vm_resources_to_execution_resources(
    vm_resources: VmExecutionResources,
    GasVector { l1_gas, l1_data_gas, l2_gas }: GasVector,
) -> ExecutionResult<ExecutionResources> {
    let mut builtin_instance_counter = HashMap::new();
    for (builtin_name, count) in vm_resources.builtin_instance_counter {
        if count == 0 {
            continue;
        }
        let count = u64_from_usize(count);
        match builtin_name {
            BuiltinName::output => continue,
            BuiltinName::pedersen => builtin_instance_counter.insert(Builtin::Pedersen, count),
            BuiltinName::range_check => builtin_instance_counter.insert(Builtin::RangeCheck, count),
            BuiltinName::ecdsa => builtin_instance_counter.insert(Builtin::Ecdsa, count),
            BuiltinName::bitwise => builtin_instance_counter.insert(Builtin::Bitwise, count),
            BuiltinName::ec_op => builtin_instance_counter.insert(Builtin::EcOp, count),
            BuiltinName::keccak => builtin_instance_counter.insert(Builtin::Keccak, count),
            BuiltinName::poseidon => builtin_instance_counter.insert(Builtin::Poseidon, count),
            BuiltinName::segment_arena => {
                builtin_instance_counter.insert(Builtin::SegmentArena, count)
            }
            // TODO(DanB): what about the following?
            // BuiltinName::range_check96 => todo!(),
            // BuiltinName::add_mod => todo!(),
            // BuiltinName::mul_mod => todo!(),
            _ => {
                return Err(ExecutionError::UnknownBuiltin { builtin_name });
            }
        };
    }
    Ok(ExecutionResources {
        steps: u64_from_usize(vm_resources.n_steps),
        builtin_instance_counter,
        memory_holes: u64_from_usize(vm_resources.n_memory_holes),
        da_gas_consumed: StarknetApiGasVector { l1_gas, l2_gas, l1_data_gas },
        gas_consumed: StarknetApiGasVector::default(),
    })
}
