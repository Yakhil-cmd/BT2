### Title
Missing Caller Validation in `exit_delegation_pool_action` Allows Anyone to Force a Pool Member's Exit - (File: src/pool/pool.cairo)

### Summary

`Pool.exit_delegation_pool_action()` does not validate that the caller is the pool member or their reward address, while sibling functions `claim_rewards()` and `add_to_delegation_pool()` both enforce this check. Any unprivileged caller can invoke `exit_delegation_pool_action()` on behalf of any pool member who has a pending exit intent, forcibly executing their exit and preventing them from canceling it.

### Finding Description

The `Pool` contract enforces a caller check in two of its member-facing functions:

`claim_rewards` checks the caller: [1](#0-0) 

`add_to_delegation_pool` checks the caller: [2](#0-1) 

However, the sibling function `exit_delegation_pool_action` takes `pool_member` as a parameter and performs **no caller check whatsoever**: [3](#0-2) 

The function only verifies that an exit intent exists and that the wait window has elapsed — it never asserts `get_caller_address() == pool_member || get_caller_address() == reward_address`. This means any external account can call `exit_delegation_pool_action(victim_pool_member)` and execute the exit on behalf of the victim.

Critically, a pool member **can cancel a pending intent** at any time — including after the wait window expires — by calling `exit_delegation_pool_intent(0)`: [4](#0-3) 

There is no deadline after which cancellation is blocked. This means the race condition is real: a pool member who changes their mind after the wait window can be front-run by an attacker calling `exit_delegation_pool_action` before the cancellation transaction lands.

### Impact Explanation

An attacker can force any pool member with an expired exit intent out of the pool. The pool member receives their funds (no direct theft), but:

- They permanently lose their current pool position and accumulated reward state.
- They are forced to re-enter the pool in a new transaction, potentially under different commission terms or staker conditions.
- A partial-intent victim (who intended to exit only a fraction of their stake) is forced to execute that partial exit even if they changed their mind.

This constitutes **griefing with no profit motive but concrete damage to users**, which maps to the **Medium** impact tier.

### Likelihood Explanation

The entry path is fully unprivileged — any account can call `exit_delegation_pool_action(target)` on the pool contract. The only precondition is that the target pool member has a non-zero `unpool_time` that has elapsed, which is a normal and common state for delegators managing their positions. Front-running a cancellation transaction is straightforward on Starknet.

### Recommendation

Add the same caller guard used in `claim_rewards` and `add_to_delegation_pool`:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
+   let caller_address = get_caller_address();
+   let pool_member_info_check = self.internal_pool_member_info(:pool_member);
+   assert!(
+       caller_address == pool_member || caller_address == pool_member_info_check.reward_address,
+       "{}",
+       Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
+   );
    // ... rest of function
```

### Proof of Concept

1. Alice (pool member) calls `exit_delegation_pool_intent(amount)` — sets `unpool_time = now + exit_wait_window`.
2. The wait window elapses.
3. Alice reconsiders and submits a transaction calling `exit_delegation_pool_intent(0)` to cancel.
4. Bob (attacker) observes Alice's pending cancellation in the mempool and front-runs it by calling `pool.exit_delegation_pool_action(alice_address)`.
5. `exit_delegation_pool_action` succeeds: it calls `staking_pool_dispatcher.remove_from_delegation_pool_action(identifier: pool_member.into())` and transfers `unpool_amount` tokens to Alice.
6. Alice's cancellation transaction now reverts with `MISSING_UNDELEGATE_INTENT` because the intent was already cleared.
7. Alice is ejected from the pool against her will. [3](#0-2) [5](#0-4) [4](#0-3)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
