### Title
Unprivileged Caller Can Permanently Deny Consensus Block Rewards to All Stakers via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control beyond a zero-address check. It accepts a caller-controlled `disable_rewards` boolean. Critically, the global `last_reward_block` checkpoint is written **before** the `disable_rewards` guard is evaluated. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to permanently consume that block's reward slot without distributing any rewards, denying consensus-phase block rewards to all stakers for that block.

### Finding Description

`IStakingRewardsManager::update_rewards` is a public entrypoint protected only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

Inside the function, the global `last_reward_block` is updated unconditionally before the `disable_rewards` branch: [2](#0-1) 

Because `last_reward_block` is a single contract-wide storage slot, once it is written to the current block number, the guard at the top of the function:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

will revert every subsequent legitimate call to `update_rewards` for that same block. [3](#0-2) 

The storage field confirms `last_reward_block` is global, not per-staker: [4](#0-3) 

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block will:
1. Advance `last_reward_block` to the current block.
2. Return immediately without distributing any STRK or BTC block rewards.
3. Block every other caller from distributing rewards for that block.

The only requirement is that the supplied `staker_address` is a currently active staker with non-zero STRK balance, which is trivially satisfiable by inspecting on-chain state.

### Impact Explanation

Every block in which the attacker fires this call, **all** stakers and their delegators permanently lose their consensus block rewards. Rewards are never retroactively recalculated; the missed block is gone. Continuous execution across all blocks would freeze the entire consensus reward stream for the protocol. This maps to **Permanent freezing of unclaimed yield** (High) and **Griefing with no profit motive but damage to users or protocol** (Medium) in the allowed impact scope.

### Likelihood Explanation

- No special role or privilege is required; any EOA or contract can call `update_rewards`.
- The attacker only needs to know one active staker address (publicly readable from events or `get_stakers`).
- The cost is one transaction per block. On Starknet, L2 gas costs are low, making sustained griefing economically feasible.
- A competing staker, a protocol adversary, or a front-running bot could all execute this attack.

### Recommendation

1. **Restrict `disable_rewards`**: Gate the `disable_rewards: true` path behind an authorized caller (e.g., `only_security_agent` or the attestation contract). Alternatively, remove the parameter entirely and derive the disable condition from on-chain state.
2. **Write `last_reward_block` only when rewards are actually processed**: Move `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` / `is_pre_consensus` guard so that a no-op call does not consume the block slot.

### Proof of Concept

```
// Attacker script (pseudocode, run once per block)
let active_staker = staking.get_stakers(current_epoch)[0].address;
staking.update_rewards(staker_address: active_staker, disable_rewards: true);
// Result: last_reward_block == current_block; no rewards distributed;
//         all subsequent update_rewards calls for this block revert with REWARDS_ALREADY_UPDATED.
``` [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
