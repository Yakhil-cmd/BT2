### Title
`Debug` Derive on Secret-Bearing Structs Exposes Private Signing Shares and Presign Secrets in Plaintext — (`File: src/lib.rs`, `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`, `src/frost/mod.rs`)

---

### Summary

Multiple structs holding secret cryptographic material unconditionally derive `Debug`, causing the raw secret scalars (private signing shares, nonce shares, presign secrets) to be rendered in plaintext whenever the struct is formatted with `{:?}`. Any caller that logs, propagates, or prints these structs — a common and expected integration pattern — silently discloses the secrets in plaintext, directly analogous to the ZecWallet `senttxstore.dat` plaintext-storage finding.

---

### Finding Description

The following production structs derive `Debug` while containing secret scalar fields:

**1. `KeygenOutput<C>` — `src/lib.rs:48–55`**
```rust
#[derive(Debug, Clone, Deserialize, Serialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct KeygenOutput<C: Ciphersuite> {
    pub private_share: SigningShare<C>,   // ← secret signing share
    #[zeroize[skip]]
    pub public_key: VerifyingKey<C>,
}
```
`private_share` is the participant's secret signing share — the core secret of the entire threshold scheme. `SigningShare<C>` from `frost_core` wraps a `SerializableScalar<C>` which itself wraps a raw scalar; `Debug` on `SerializableScalar` renders the scalar bytes in plaintext. Any call to `format!("{:?}", keygen_output)`, `tracing::debug!("{:?}", ...)`, `dbg!(keygen_output)`, or inclusion in a `panic!` / `unwrap()` message exposes the secret share verbatim.

**2. `PresignOutput` (OT-based ECDSA) — `src/ecdsa/ot_based_ecdsa/mod.rs:40–49`**
```rust
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct PresignOutput {
    #[zeroize[skip]]
    pub big_r: AffinePoint,
    pub k: Scalar,      // ← secret nonce share
    pub sigma: Scalar,  // ← secret sigma share
}
```
`k` and `sigma` are secret presign nonce shares. Disclosure of `k` combined with a signed message allows full private-key recovery.

**3. `PresignOutput` (Robust ECDSA) — `src/ecdsa/robust_ecdsa/mod.rs:26–37`**
```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, ZeroizeOnDrop)]
pub struct PresignOutput {
    #[zeroize(skip)]
    pub big_r: AffinePoint,
    pub c: Scalar,      // ← secret nonce share
    pub e: Scalar,      // ← secret nonce share
    pub alpha: Scalar,  // ← secret nonce share
    pub beta: Scalar,   // ← secret nonce share
}
```

**4. `RerandomizedPresignOutput` (both ECDSA variants) — `src/ecdsa/ot_based_ecdsa/mod.rs:54` and `src/ecdsa/robust_ecdsa/mod.rs:42`** — also derive `Debug` and expose the same rerandomized secret scalar fields.

**5. `PresignOutput<C>` (FROST/EdDSA) — `src/frost/mod.rs:36–41`**
```rust
#[derive(Debug, Clone, Serialize, Deserialize, Eq, PartialEq)]
pub struct PresignOutput<C: Ciphersuite + Send + 'static> {
    pub nonces: SigningNonces<C>,  // ← secret nonce pair (hiding + binding scalars)
    pub commitments_map: BTreeMap<Identifier<C>, SigningCommitments<C>>,
}
```
`SigningNonces<C>` from `frost_core` holds two secret nonce scalars. Reuse or disclosure of FROST nonces is catastrophic (full key recovery).

---

### Impact Explanation

**Critical — Disclosure of private signing shares and presign/nonce secrets.**

- Disclosure of `KeygenOutput.private_share` for any participant directly yields that participant's secret signing share. With `threshold` such shares an attacker reconstructs the aggregate private key and forges arbitrary signatures.
- Disclosure of `PresignOutput.k` (OT-ECDSA) or `PresignOutput.nonces` (FROST) for a single participant, combined with the corresponding signed message, enables full private-key extraction via standard ECDSA/EdDSA nonce-reuse attacks.
- The `Debug` output is plaintext, unencrypted, and unredacted — identical in character to the plaintext `senttxstore.dat` file in the ZecWallet report.

---

### Likelihood Explanation

**High.** This is a library, not an application. Every integrator is expected to wire up logging, error reporting, and observability. The idiomatic Rust patterns that trigger this are ubiquitous:

- `tracing::debug!("{:?}", keygen_output)` — standard structured logging
- `anyhow::Context` / `.context(format!("{:?}", ...))` — error wrapping
- `dbg!(presign_output)` — developer debugging left in code
- `assert_eq!(a, b)` in integration tests that print both sides on failure
- Sentry / crash-reporter SDKs that capture `Debug` representations of local variables in panic frames

None of these require attacker interaction; a single inadvertent log line by any integrator suffices. The structs are `pub`, their fields are `pub`, and the `Debug` impl is unconditional.

---

### Recommendation

1. **Remove `Debug` from all secret-bearing structs.** Replace with a manual `Debug` implementation that redacts secret fields:
   ```rust
   impl<C: Ciphersuite> std::fmt::Debug for KeygenOutput<C> {
       fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
           f.debug_struct("KeygenOutput")
            .field("private_share", &"[REDACTED]")
            .field("public_key", &self.public_key)
            .finish()
       }
   }
   ```
2. Apply the same pattern to `PresignOutput` (both ECDSA variants and FROST), `RerandomizedPresignOutput`, and any future struct holding secret scalars.
3. Consider adding a `#[cfg_attr(debug_assertions, ...)]` lint or a custom `clippy` rule to prevent `#[derive(Debug)]` on types that implement `ZeroizeOnDrop`, as a structural guard.
4. Document in the public API that `KeygenOutput` and `PresignOutput` must never be logged or included in error messages.

### Proof of Concept

```rust
use threshold_signatures::{keygen, KeygenOutput};
use threshold_signatures::participants::Participant;
use threshold_signatures::frost_core::Ciphersuite;

// After running DKG and obtaining keygen_output: KeygenOutput<C>
// Any of the following leaks the raw private signing share in plaintext:

// 1. Direct debug print
println!("{:?}", keygen_output);
// Output: KeygenOutput { private_share: SigningShare(SerializableScalar(0x<SECRET_SCALAR_HEX>)), public_key: ... }

// 2. Tracing/log integration (extremely common in production services)
tracing::debug!(output = ?keygen_output, "DKG complete");

// 3. Panic / unwrap on a Result containing KeygenOutput
let result: Result<KeygenOutput<_>, _> = Ok(keygen_output);
result.expect("should succeed");  // on Err, prints Debug of Ok variant in some wrappers

// 4. anyhow error context
use anyhow::Context;
some_fallible_op().with_context(|| format!("keygen_output={keygen_output:?}"))?;
```

The same applies to `PresignOutput` structs: `println!("{:?}", presign_output)` renders `k`, `sigma`, `c`, `e`, `alpha`, `beta`, or `nonces` as plaintext scalars, enabling nonce-reuse key recovery. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)
