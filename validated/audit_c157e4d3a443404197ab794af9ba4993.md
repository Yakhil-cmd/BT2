### Title
Missing Minimum Threshold Enforcement in FROST Signing Allows Single-Party Signature Creation - (`File: src/frost/mod.rs`)

### Summary

`assert_sign_inputs` in `src/frost/mod.rs` does not enforce a minimum threshold of 2, unlike every other protocol entry point in the library. A caller passing `threshold = 1` bypasses the quorum requirement entirely, allowing a single participant to produce a valid threshold signature without the required number of honest co-signers.

### Finding Description

The library enforces a minimum threshold of 2 in every protocol entry point **except** FROST signing.

`assert_key_invariants` (DKG) explicitly rejects threshold < 2: [1](#0-0) 

`validate_triple_inputs` (OT triple generation) also rejects threshold < 2: [2](#0-1) 

`assert_sign_inputs` (FROST signing) only checks the upper bound — it never checks the lower bound: [3](#0-2) 

The only threshold check present is:
```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
```

There is no corresponding `ThresholdTooSmall` guard. The `ReconstructionLowerBound` type itself carries no minimum invariant: [4](#0-3) 

With `threshold = 1`, FROST Lagrange interpolation requires only one signature share. The single contributing participant's Lagrange coefficient evaluates to 1 (trivially), so the aggregated signature is mathematically valid and will verify against the group public key.

### Impact Explanation

A caller supplying `threshold = 1` to the FROST signing entry point causes the protocol to accept a single participant's share as a complete threshold signature. This directly violates the core security property of threshold signatures: that at least `t` honest parties must cooperate. The resulting signature is indistinguishable from one produced by a legitimate quorum and will pass standard FROST verification.

**Impact class**: Critical — Unauthorized creation of a valid threshold signature for attacker-chosen inputs.

### Likelihood Explanation

The entry point is a public library API. Any integrator, malicious coordinator, or participant who controls the `threshold` argument passed to the signing function can trigger this. No privileged access, leaked keys, or cryptographic breaks are required. The inconsistency with the DKG path (which does enforce the minimum) makes accidental misconfiguration equally likely alongside deliberate abuse.

### Recommendation

Add the same lower-bound guard that exists in `assert_key_invariants` and `validate_triple_inputs` to `assert_sign_inputs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This should be inserted in `src/frost/mod.rs` inside `assert_sign_inputs`, immediately after the `participants.len() < 2` check, mirroring the pattern at: [5](#0-4) 

### Proof of Concept

```rust
// Attacker-controlled call — threshold = 1 passes assert_sign_inputs
let participants = vec![p0, p1, p2]; // 3 participants, key generated with threshold=3
let protocol = frost::sign(
    &participants,
    coordinator,
    ReconstructionLowerBound::from(1usize), // ← passes validation, should be rejected
    me,
    keygen_output,
    msg_hash,
).unwrap(); // succeeds — no ThresholdTooSmall error

// p0 alone drives the protocol to completion.
// The resulting signature verifies against the group public key
// despite only 1 of 3 required parties contributing.
```

The `assert_sign_inputs` call at the top of the signing entry point returns `Ok` for `threshold = 1` because the only guard present is `threshold > participants.len()`, which is `1 > 3 = false`. [6](#0-5)

```rust
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
```
