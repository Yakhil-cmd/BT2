### Title
Missing `max_commission` Slippage Guard in `enter_delegation_pool` and `switch_delegation_pool` — (`src/pool/pool.cairo`)

---

### Summary

`enter_delegation_pool` and `switch_delegation_pool` in `src/pool/pool.cairo` accept no `max_commission` parameter (analogous to `minOutputAmount` in the original report). A delegator who submits either transaction expecting a specific commission rate has no on-chain protection against the staker raising their commission before the transaction is included, causing the delegator to silently accept a higher commission — and therefore earn less yield — than intended.

---

### Finding Description

`enter_delegation_pool` takes only `reward_address` and `amount`: [1](#0-0) 

`switch_delegation_pool` takes only `to_staker`, `to_pool`, and `amount`: [2](#0-1) 

Neither function checks the pool's current commission against any caller-supplied bound. The commission is stored in the staking contract and queried dynamically by the pool contract via `get_commission_from_staking_contract`: [3](#0-2) 

A staker can legitimately raise their commission at any time provided they hold an active `commission_commitment` whose `max_commission` is above the current rate: [4](#0-3) 

The `set_commission_commitment` mechanism explicitly allows a staker to set a `max_commission` ceiling and then call `set_commission` up to that ceiling at will: [5](#0-4) 

---

### Impact Explanation

A delegator who submits `enter_delegation_pool` or `switch_delegation_pool` while observing commission rate `C` may have their transaction execute against commission rate `C' > C` if the staker raises commission (within their commitment) before the transaction is included. The delegator's entire future yield stream is then taxed at the higher rate with no recourse. This constitutes ongoing theft of unclaimed yield: the excess commission flows to the staker rather than the delegator for every reward epoch until the delegator manually exits.

**Allowed impact matched**: *High — Theft of unclaimed yield.*

---

### Likelihood Explanation

1. `commission_commitment` is a first-class protocol feature designed to be used by stakers who want to attract delegators. Any staker who has set one can raise commission up to `max_commission` at any time.
2. On Starknet L2, the sequencer controls transaction ordering. A staker who is also a sequencer operator, or who monitors the public mempool/pending transaction feed, can trivially front-run a delegator's delegation transaction by submitting `set_commission` first.
3. Even without active front-running, ordinary network congestion (e.g., during high-activity periods) can delay a delegator's transaction long enough for the staker to raise commission in the interim — exactly the scenario the original report describes for L2s.
4. The delegator has no way to express "only delegate if commission ≤ X" and no way to cancel a pending transaction on-chain.

---

### Recommendation

Add a `max_commission: Commission` parameter to both `enter_delegation_pool` and `switch_delegation_pool`. At the start of each function, assert that the pool's current commission does not exceed the caller-supplied bound:

```rust
fn enter_delegation_pool(
    ref self: ContractState,
    reward_address: ContractAddress,
    amount: Amount,
    max_commission: Commission,   // <-- add
) {
    let current_commission = self.get_commission_from_staking_contract();
    assert!(current_commission <= max_commission, "{}", Error::COMMISSION_TOO_HIGH);
    // ... rest of function unchanged
}
```

Apply the same guard to `switch_delegation_pool`. This is the direct analog of the `minOutputAmount` slippage guard recommended in the original report.

---

### Proof of Concept

1. Staker S has commission 2 % and holds a `commission_commitment` with `max_commission = 8 %`.
2. Delegator D observes 2 % commission and submits `enter_delegation_pool(reward_address, amount)` (no commission bound possible).
3. Before D's transaction is sequenced, S calls `set_commission(800)` (8 %). This is valid because `800 ≤ max_commission`.
4. D's transaction executes. `enter_delegation_pool` performs no commission check; D is now a pool member earning rewards at 8 % commission.
5. For every subsequent epoch, D receives `(1 - 0.08) × pool_rewards` instead of `(1 - 0.02) × pool_rewards`. The 6 % difference accrues to S indefinitely until D manually exits — a continuous drain of D's unclaimed yield with no on-chain protection.

The root cause is the absence of a caller-supplied commission bound in `enter_delegation_pool` [6](#0-5)  and `switch_delegation_pool` [7](#0-6) , combined with the protocol's legitimate ability to raise commission within a commitment window. [5](#0-4)

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }

        fn get_commission_from_staking_contract(self: @ContractState) -> Commission {
            if self.staker_removed.read() {
                return Zero::zero();
            }
            let staking_dispatcher = IStakingDispatcher {
                contract_address: self.staking_pool_dispatcher.contract_address.read(),
            };
            // The staker must have commission since it has a pool (this contract). So unwrap is
            // safe.
            staking_dispatcher
                .staker_pool_info(staker_address: self.staker_address.read())
                .commission
                .unwrap()
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment

                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
