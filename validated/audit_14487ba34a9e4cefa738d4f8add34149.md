### Title
Pool Contract `claim_rewards` and `change_reward_address` Bypass Staking Contract Pause - (File: `src/pool/pool.cairo`)

### Summary
The pool contract's `claim_rewards` and `change_reward_address` functions contain no check against the staking contract's pause state. While every state-changing function in the staking contract enforces the pause via `general_prerequisites()`, the pool contract functions that do not internally call the staking contract are entirely unguarded. This allows pool members to drain accumulated rewards and redirect reward addresses even during an active emergency pause.

### Finding Description
The staking contract enforces a pause via `general_prerequisites()`:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

`assert_is_unpaused` panics with `CONTRACT_IS_PAUSED` if `is_paused` is set: [2](#0-1) 

Every state-changing function in the staking contract calls `general_prerequisites()` first, including `claim_rewards`: [3](#0-2) 

Pool functions that internally call the staking contract (`exit_delegation_pool_intent`, `exit_delegation_pool_action`, `switch_delegation_pool`) inherit the pause check transitively. However, two pool functions operate entirely without calling the staking contract and contain **no pause check**:

The pool contract already uses `IStakingDispatcher` to query the staking contract (e.g., `get_current_epoch`), so querying `is_paused()` is architecturally feasible: [6](#0-5) 

The STRK rewards transferred in `claim_rewards` come from the pool contract's own balance, which was funded by the staking contract via `send_rewards_to_delegation_pool` during prior reward distributions: [7](#0-6) 

### Impact Explanation
When the staking contract is paused — typically in response to a discovered bug or security incident — the intent is to freeze all reward flows. If the pause was triggered because rewards were over-calculated or incorrectly distributed, pool members can still call `pool.claim_rewards` to drain the incorrectly computed STRK rewards already sitting in the pool contract, constituting theft of unclaimed yield. Additionally, `pool.change_reward_address` allows a pool member (or an attacker who has compromised a pool member's key) to redirect the reward destination to an attacker-controlled address before the pause is lifted, enabling reward misrouting that cannot be reversed once the pause ends.

### Likelihood Explanation
The staking contract has an explicit pause mechanism operated by a security agent. The scenario requires: (1) the security agent pauses the contract in response to an anomaly, and (2) a pool member (or attacker) calls `pool.claim_rewards` or `pool.change_reward_address` before the issue is resolved. Pool members are unprivileged actors with direct access to these functions. The entry path is fully reachable with no privileged access required beyond being an existing pool member.

### Recommendation
Add a pause check to both functions in `src/pool/pool.cairo`. The pool contract already holds a reference to the staking contract address via `staking_pool_dispatcher`. Both functions should query `IStakingDispatcher { contract_address: self.staking_pool_dispatcher.contract_address.read() }.is_paused()` and assert it returns `false` before proceeding, mirroring the pattern used in `get_current_epoch`.

### Proof of Concept
1. Deploy the system and have a pool member delegate funds.
2. Advance epochs so the staker attests and rewards accumulate in the pool contract (STRK is transferred to the pool via `send_rewards_to_delegation_pool`).
3. Security agent calls `staking.pause()` — all staking contract functions now revert with `CONTRACT_IS_PAUSED`.
4. Pool member calls `pool.claim_rewards(pool_member)` directly on the pool contract.
5. The call succeeds: rewards are calculated from `cumulative_rewards_trace` and STRK is transferred to `reward_address` with no pause check encountered.
6. Separately, pool member calls `pool.change_reward_address(attacker_address)` — this also succeeds, redirecting all future reward claims to the attacker's address, with no pause check.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            assert!(
                self.token_dispatcher.contract_address.read() != reward_address,
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_address = pool_member_info.reward_address;

            // Update reward_address and commit to storage.
            pool_member_info.reward_address = reward_address;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardAddressChanged {
                        pool_member, new_address: reward_address, old_address,
                    },
                );
        }
