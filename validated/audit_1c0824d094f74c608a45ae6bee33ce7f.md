### Title
Truncating Integer Division in `native_per_pubdata` Silently Rounds Pubdata Cost to Zero — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

The `native_per_pubdata` conversion factor is computed with floor (truncating) integer division. When `pubdata_price < native_price`, the result rounds to **zero**, making all pubdata completely free in terms of native resources. Any unprivileged transaction sender can then generate pubdata up to the block limit without paying native resource costs, causing the operator to bear DA costs without compensation.

---

### Finding Description

In `validate_and_compute_fee_for_transaction`, the per-byte pubdata cost in native resource units is computed as:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs:142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`wrapping_div` is floor (truncating) integer division. When `pubdata_price < native_price`, the result is **0**. [1](#0-0) 

The same truncating division appears in the public API helper used for off-chain pre-validation:

```rust
// api/src/helpers.rs:427
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [2](#0-1) 

This is asymmetric with `native_per_gas`, which correctly uses **ceiling** division to avoid undercharging:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs:135
u256_try_to_u64(&gas_price.div_ceil(native_price))
``` [3](#0-2) 

When `native_per_pubdata == 0`, two downstream charging sites both produce zero cost:

1. **Intrinsic pubdata overhead** in `create_resources_for_tx`:
   ```rust
   let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
   // → 0 * intrinsic_pubdata = 0
   ``` [4](#0-3) 

2. **Execution pubdata charge** in `get_resources_to_charge_for_pubdata`:
   ```rust
   let native = current_pubdata_spent.checked_mul(native_per_pubdata)...;
   // → pubdata_bytes * 0 = 0
   ``` [5](#0-4) 

The `check_enough_resources_for_pubdata` check therefore always passes (0 native required), and the transaction succeeds regardless of how much pubdata it generates. [6](#0-5) 

---

### Impact Explanation

When `pubdata_price < native_price`, the operator intended pubdata to cost a fractional amount of native resource per byte. Due to floor division, it costs **nothing**. A transaction sender can:

- Perform storage writes (SSTORE) up to the block's `pubdata_limit` without paying any native resource cost for the resulting pubdata.
- The operator publishes this pubdata to L1 (DA layer) and bears the cost without receiving native resource compensation.

This is a direct loss of fee revenue for the operator: the protocol charges 0 native for pubdata that has a real DA cost. [7](#0-6) 

---

### Likelihood Explanation

`pubdata_price` and `native_price` are independent operator-set parameters with different units (`native_price` = cost per RISC-V cycle; `pubdata_price` = cost per pubdata byte). It is entirely realistic for `pubdata_price < native_price` in production — for example, `native_price = 1000`, `pubdata_price = 500` gives `native_per_pubdata = 0`. The default test configuration already uses `pubdata_price = 0`, confirming this is a common operating condition. [8](#0-7) 

Any unprivileged L2 transaction sender can exploit this whenever the operator's price configuration satisfies `pubdata_price < native_price`.

---

### Recommendation

Replace the truncating `wrapping_div` with ceiling division (`div_ceil`) for `native_per_pubdata`, consistent with how `native_per_gas` is computed:

```rust
// Before (truncates to 0 when pubdata_price < native_price):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))...;

// After (rounds up, ensuring non-zero cost when pubdata_price > 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))...;
```

Apply the same fix in `api/src/helpers.rs` line 427. [9](#0-8) 

---

### Proof of Concept

**Setup:**
- `native_price = 1000`
- `pubdata_price = 999`
- `native_per_pubdata = 999 / 1000 = 0` (floor division)

**Attack:**
1. Submit an L2 transaction with `max_fee_per_gas` satisfying `gas_price / native_price >= 1` (passes `native_per_gas` check).
2. Execute a loop of SSTORE operations generating, e.g., 10,000 bytes of pubdata.
3. `get_resources_to_charge_for_pubdata` computes `10000 * 0 = 0` native charged.
4. `check_enough_resources_for_pubdata` passes (0 ≤ available native).
5. Transaction succeeds; operator publishes 10,000 bytes to L1 and receives 0 native resource compensation for it.

The operator intended to charge `10000 * 0.999 ≈ 9990` native units for this pubdata; they receive 0. [10](#0-9)

```rust
pub fn get_resources_to_charge_for_pubdata<S: EthereumLikeTypes>(
    system: &mut System<S>,
    native_per_pubdata: u64,
    base_pubdata: Option<u64>,
) -> Result<(u64, S::Resources), SystemError> {
    let current_pubdata_spent = system
        .net_pubdata_used()?
        .saturating_sub(base_pubdata.unwrap_or(0));
    let native = current_pubdata_spent
        .checked_mul(native_per_pubdata)
        .ok_or(out_of_native_resources!())?;
    let native = <S::Resources as zk_ee::system::Resources>::Native::from_computational(native);
    Ok((current_pubdata_spent, S::Resources::from_native(native)))
}
```

pub fn check_enough_resources_for_pubdata<S: EthereumLikeTypes>(
    system: &mut System<S>,
    native_per_pubdata: u64,
    resources: &S::Resources,
    base_pubdata: Option<u64>,
) -> Result<(bool, S::Resources, u64), SystemError> {
    let (pubdata_used, resources_for_pubdata) =
        get_resources_to_charge_for_pubdata(system, native_per_pubdata, base_pubdata)?;
    system_log!(system, "Checking gas for pubdata, resources_for_pubdata: {resources_for_pubdata:?}, resources: {resources:?}\n");
    let enough = resources.has_enough(&resources_for_pubdata);
    Ok((enough, resources_for_pubdata, pubdata_used))
