### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze All Stakers' Unclaimed Yield - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is specified to be callable only by the Starkware sequencer, but the implementation contains **no access control check**. Because `last_reward_block` is a single global storage variable that is written unconditionally before the `disable_rewards` guard, any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` every block to consume the per-block reward slot without distributing any rewards, permanently destroying unclaimed yield for every staker in the protocol.

---

### Finding Description

The protocol specification at `docs/spec.md` lines 1644–1645 states:

> **access control**: Only starkware sequencer.

However, the implementation of `update_rewards` in `src/staking/staking.cairo` at lines 1449–1507 only calls `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no sequencer or role check is present. [1](#0-0) 

The critical ordering flaw is:

```
1. last_reward_block.write(current_block_number);   // line 1485 — always executed
2. if disable_rewards || self.is_pre_consensus() {  // line 1487
       return;                                       // line 1488 — no rewards distributed
   }
```

`last_reward_block` is a **single global** storage slot (not per-staker): [2](#0-1) 

The guard at the top of `update_rewards` enforces that only one call per block can succeed: [3](#0-2) 

Because `last_reward_block` is written before the `disable_rewards` branch, a call with `disable_rewards: true` consumes the block's reward slot for **all stakers** while distributing nothing. Any subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

The interface definition confirms the function is public with no role guard: [4](#0-3) 

---

### Impact Explanation

**Permanent freezing of unclaimed yield** (High).

Consensus rewards are distributed exactly once per block via `update_rewards`. There is no mechanism to retroactively distribute rewards for a block that was already "consumed" by a no-op call. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block permanently destroys that block's rewards for every staker and every delegation pool in the protocol. The yield is irrecoverably lost.

---

### Likelihood Explanation

**High**. The attack requires:
- No capital, no privileged role, no leaked key.
- Only knowledge of one valid active staker address (all staker addresses are public on-chain via `NewStaker` events).
- One transaction per block on Starknet (low gas cost relative to the yield destroyed).

The spec-vs-implementation mismatch means the missing guard is a straightforward oversight, not a design trade-off, making exploitation trivially reachable by any unprivileged address.

---

### Recommendation

1. **Add the intended access control**: Restrict `update_rewards` to the Starkware sequencer address (or a designated role), matching the specification.
2. **Alternatively, reorder the write**: Move `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` / `is_pre_consensus` guard so that a no-op call does not consume the block's reward slot.
3. **Or make `last_reward_block` per-staker**: This would scope the block-lock to a single staker rather than the entire protocol, limiting blast radius even if access control is absent.

---

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`).
2. Attacker identifies any active staker address `S` with non-zero balance (K+ epochs after `stake`).
3. In every block N, attacker calls:
   ```
   update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Inside the call: staker/balance checks pass → `last_reward_block` is written to N → function returns early, zero rewards distributed.
5. Any legitimate sequencer call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`.
6. Block N's rewards are permanently lost for all stakers and all delegation pools.
7. Attacker repeats every block; total protocol yield approaches zero.

The existing test `update_rewards_disable_rewards_consensus_rewards_flow_test` (flow_test/test.cairo) already demonstrates that `update_rewards(..., disable_rewards: true)` succeeds from an arbitrary caller and blocks a subsequent call in the same block — confirming the attack path is live. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {

#[test]
#[feature("safe_dispatcher")]
fn update_rewards_disable_rewards_consensus_rewards_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    let stake_amount = system.staking.get_min_stake();
    let staker = system.new_staker(amount: stake_amount);
    let commission = 200;
    system.stake(:staker, amount: stake_amount, pool_enabled: false, :commission);
    system.advance_k_epochs();

    // Disable rewards = true with consensus off - no rewards
    system.update_rewards(:staker, disable_rewards: true);
    let rewards = system.staker_claim_rewards(:staker);
    assert!(rewards.is_zero());

    // Attempt again same block - panic
    let result = system
        .staking
        .rewards_manager_safe_dispatcher()
        .update_rewards(staker_address: staker.staker.address, disable_rewards: true);
    assert_panic_with_error(
        :result, expected_error: StakingError::REWARDS_ALREADY_UPDATED.describe(),
    );
    advance_block_number_global(blocks: 1);
