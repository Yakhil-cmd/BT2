### Title
Pool Rewards Permanently Locked When Pool Balance Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo, src/staking/staking.cairo)

### Summary
In the Starknet Staking system, STRK reward tokens are unconditionally transferred to a pool contract even when the pool's total delegated balance is below `min_delegation_for_rewards`. Because `compute_rewards_per_unit` returns zero in that case, the cumulative rewards trace is not updated, and pool members can never claim those tokens. The STRK is permanently locked in the pool contract with no recovery path.

### Finding Description

The reward distribution path for delegation pools is:

1. `_update_rewards` (staking.cairo) calls `calculate_staker_pools_rewards`, which computes `pool_rewards` for each pool with a non-zero balance.
2. `update_pool_rewards` (staking.cairo lines 1865–1891) then calls `send_rewards_to_delegation_pool`, which unconditionally transfers STRK to the pool contract: [1](#0-0) 

3. Immediately after the transfer, `update_rewards_from_staking_contract` is called on the pool: [2](#0-1) 

4. Inside the pool, `update_rewards_from_staking_contract` delegates to `compute_rewards_per_unit`: [3](#0-2) 

5. `compute_rewards_per_unit` returns **zero** when `total_stake < min_delegation_for_rewards`: [4](#0-3) 

The code comment at line 960–966 explicitly acknowledges this:

> "**Note**: Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case." [5](#0-4) 

Because the cumulative rewards trace receives a zero increment, `calculate_rewards` will compute zero rewards for all pool members for that epoch. The STRK tokens already transferred to the pool contract have no recovery path: `claim_rewards` only pays out what the trace records, and there is no admin sweep or fallback function. [6](#0-5) 

The STRK constant for `min_delegation_for_rewards` is `10^18` (1 STRK): [7](#0-6) 

### Impact Explanation

STRK reward tokens forwarded to a pool contract whose total delegated balance is below `min_delegation_for_rewards` are permanently frozen inside that contract. No pool member, staker, or admin can recover them. This constitutes **permanent freezing of unclaimed yield**, which is within the allowed High impact scope.

### Likelihood Explanation

- `enter_delegation_pool` enforces only `amount.is_non_zero()` — there is no minimum delegation floor for pool members.
- A pool whose cumulative delegated balance is between 1 wei and `10^18 - 1` wei of STRK (or the BTC equivalent) will trigger this condition every epoch it receives rewards.
- New pools naturally start with small balances. A single delegator contributing a dust amount (e.g., 1 wei) is sufficient.
- The staking contract's `calculate_staker_pools_rewards` will still compute a non-zero `pool_rewards` for such a pool as long as `total_rewards * pool_balance / total_stake > 0` (integer arithmetic), so the transfer will occur.

### Recommendation

In `update_pool_rewards`, skip the STRK transfer (or redirect it to the staker's reward address) when the pool balance is below `min_delegation_for_rewards`. Alternatively, move the `min_delegation_for_rewards` guard upstream into `calculate_staker_pools_rewards` so that pools below the threshold are excluded from `pool_rewards_array` entirely, preventing the transfer from occurring in the first place.

### Proof of Concept

1. Staker S stakes and opens a STRK delegation pool P.
2. Pool member M calls `enter_delegation_pool` with `amount = 1` (1 wei of STRK). Pool P now has total balance = 1 wei < `min_delegation_for_rewards` (10^18).
3. An epoch passes. `update_rewards` is called for S.
4. `calculate_staker_pools_rewards` computes `pool_rewards > 0` for P (because `total_rewards * 1 / total_stake` rounds to a small but non-zero value given large enough epoch rewards).
5. `update_pool_rewards` calls `send_rewards_to_delegation_pool` → STRK is transferred to pool P.
6. `pool_dispatcher.update_rewards_from_staking_contract(rewards, pool_balance=1)` is called.
7. Inside `compute_rewards_per_unit`: `1 < 10^18` → returns `0`. Cumulative rewards trace unchanged.
8. M calls `pool.claim_rewards(M)` → `calculate_rewards` returns 0 → no STRK transferred to M.
9. The STRK sent in step 5 is permanently locked in pool P.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

            let (_, last) = self.cumulative_rewards_trace.last().unwrap();
            self
                .cumulative_rewards_trace
                .insert(
                    key: self.get_current_epoch(),
                    value: last
                        + self

                                staking_rewards: rewards, total_stake: pool_balance,
                            ),
                );
        }

        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
