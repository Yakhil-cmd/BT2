### Title
`StatelessTransactionValidator` skips `max_l2_gas_amount` check for `Declare` transactions, admitting over-bound transactions to the mempool - (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces the `max_l2_gas_amount` ceiling on `l2_gas.max_amount` for `Invoke` and `DeployAccount` transactions but silently skips it for `Declare` transactions. An attacker with a realistic STRK balance can submit a `Declare` transaction whose declared `l2_gas.max_amount` far exceeds the configured limit, bypass the stateless gateway check, pass stateful balance validation, and have the transaction admitted to the mempool — exactly the class of invalid transaction the check was designed to block.

### Finding Description

In `validate_resource_bounds`, after the `min_gas_price` check that applies to all transaction types, the code branches on transaction type:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← empty arm: no check performed
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

The `if let RpcTransaction::Declare(_) = tx { }` arm is intentionally empty — the `max_l2_gas_amount` bound is never applied to `Declare` transactions. The inline TODO confirms the gap is known but unresolved.

The production configuration sets `max_l2_gas_amount = 1_210_000_000` and `min_gas_price = 8_000_000_000 fri`. [2](#0-1) 

A `Declare` transaction with `l2_gas.max_amount = 2_420_000_000` (2× the limit) and `l2_gas.max_price_per_unit = 8_000_000_000 fri` produces a `max_possible_fee` of:

```
2_420_000_000 × 8_000_000_000 = 19_360_000_000_000_000_000 fri ≈ 19.36 STRK
```

Because STRK has 18 decimal places, 19.36 STRK is a realistic balance. The stateful validator's `verify_can_pay_committed_bounds` check passes for any account holding that amount, so the transaction clears both gateway stages and enters the mempool. [3](#0-2) 

The stateful validator only checks the L2 gas *price* against the previous block's price; it has no independent `max_l2_gas_amount` ceiling: [4](#0-3) 

### Impact Explanation

The `max_l2_gas_amount` stateless check is the gateway's sole defence against `Declare` transactions that declare an unrealistically large gas budget. Bypassing it allows:

1. **Invalid transactions admitted to the mempool** — a `Declare` with `max_amount` many multiples above the configured ceiling passes both gateway stages and enters the mempool, violating the invariant the check was designed to enforce.
2. **Mempool resource exhaustion / priority distortion** — an attacker with a modest STRK balance can flood the mempool with `Declare` transactions carrying inflated gas declarations, distorting fee-priority ordering and consuming mempool capacity.

This matches the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The trigger requires only:
- Constructing a valid `Declare` V3 transaction with `l2_gas.max_amount > max_l2_gas_amount`.
- Holding enough STRK to satisfy `verify_can_pay_committed_bounds` (≈ 19 STRK for 2× the limit at minimum gas price).

No privileged access, no special contract, no race condition. Any unprivileged account can reproduce this deterministically.

### Recommendation

Remove the empty `Declare` arm and apply the same `max_l2_gas_amount` ceiling unconditionally to all transaction types:

```rust
// Apply to all transaction types, including Declare.
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

The existing test `valid_l2_gas_amount_on_declare` (which asserts that a `Declare` with `max_amount = 200 > limit 100` passes) should be updated to expect rejection after this fix. [5](#0-4) 

### Proof of Concept

1. Build a `RpcDeclareTransaction::V3` with:
   - `l2_gas.max_amount = GasAmount(2_420_000_000)` (2× `max_l2_gas_amount`)
   - `l2_gas.max_price_per_unit = GasPrice(8_000_000_000)` (exactly `min_gas_price`)
   - All other fields valid.

2. Call `StatelessTransactionValidator::validate(&tx)` — it returns `Ok(())` because the `Declare` arm is empty.

3. Fund the sender account with ≥ 19.36 STRK and call `extract_state_nonce_and_run_validations` — it also returns `Ok(nonce)` because `verify_can_pay_committed_bounds` passes.

4. The transaction is forwarded to the mempool with a declared gas amount 2× the enforced ceiling, which an `Invoke` or `DeployAccount` with identical fields would have been rejected for at step 2. [6](#0-5)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}
