### Title
Missing Caller Validation on `update_rewards` Allows Anyone to Permanently Deny Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is documented as restricted to "Only starkware sequencer" but has no caller check in its implementation. Any unprivileged caller can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` to the current block without distributing any rewards. Because the function enforces a one-call-per-block invariant, the legitimate sequencer call for that block is permanently blocked, and the block's rewards are irrecoverably lost for all stakers and delegators.

### Finding Description

The `IStakingRewardsManager::update_rewards` function is the consensus-phase reward distribution entry point. The protocol specification at `docs/spec.md` line 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation in `src/staking/staking.cairo` at lines 1449–1507 contains no such check. The only gate is `general_prerequisites()`, which merely asserts the contract is not paused and the caller is non-zero — both trivially satisfied by any EOA or contract.

The function writes `last_reward_block` to the current block number **unconditionally**, before branching on `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // line 1485

if disable_rewards || self.is_pre_consensus() {
    return;                                            // line 1487-1489
}
```

`last_reward_block` is a single global storage slot (not per-staker). The function's first assertion is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

Once any caller sets `last_reward_block` to block N, every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`, including the legitimate sequencer call.

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` at block N. The global `last_reward_block` is set to N, no rewards are computed or transferred, and the function returns. The sequencer's own call for block N then reverts. The STRK block rewards that would have been distributed to every staker and every delegation pool for block N are permanently lost — they are never credited to `unclaimed_rewards_own` and never forwarded to pool contracts via `update_rewards_from_staking_contract`. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators in the targeted block.

The attacker can repeat this every block, continuously suppressing all consensus-phase reward distribution at negligible cost (only gas for the call).

### Likelihood Explanation

The function is publicly callable by any non-zero address with no preconditions beyond the contract being unpaused and a valid active staker address being supplied (any existing staker address works). On Starknet, transaction ordering within a block is controlled by the sequencer, but the attacker only needs to submit a transaction that lands in the same block as the sequencer's reward update. Because `update_rewards` is a permissionless external call, a malicious actor — including a competing validator or any griefing party — can reliably race the sequencer call. The attack requires no privileged access, no leaked keys, and no external dependencies.

### Recommendation

Add a sequencer-only access check at the top of `update_rewards`, consistent with the documented specification. The roles component already present in the contract should be used:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer();   // enforce "Only starkware sequencer"
    self.general_prerequisites();
    ...
}
```

Alternatively, if a dedicated sequencer role is not yet defined, restrict the caller to a stored sequencer address and enforce it with an explicit `assert!` before any state mutation.

### Proof of Concept

1. Deploy the system and advance past `consensus_rewards_first_epoch` so consensus rewards are active.
2. Advance two epochs so a staker has non-zero balance at the current epoch (satisfying `INVALID_STAKER` checks).
3. From any unprivileged address, call:
   ```
   IStakingRewardsManagerDispatcher { contract_address: staking_contract }
       .update_rewards(staker_address: any_active_staker, disable_rewards: true);
   ```
4. Observe that `last_reward_block` is now set to the current block and no `StakerRewardsUpdated` event was emitted.
5. Attempt the legitimate sequencer call in the same block:
   ```
   IStakingRewardsManagerDispatcher { contract_address: staking_contract }
       .update_rewards(staker_address: any_active_staker, disable_rewards: false);
   ```
6. The call reverts with `REWARDS_ALREADY_UPDATED`. Block rewards are permanently lost.

The test suite at `src/staking/tests/test.cairo` lines 3887–3894 already demonstrates that calling `update_rewards` with `disable_rewards: true` sets `last_reward_block` and causes all subsequent calls in the same block to revert — confirming the mechanism. The missing piece is that the test never asserts a caller restriction, because none exists.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
