### Title
Incorrect Royalty Summation in `formatAmountWithRoyalties` Understates Total Spend Shown to User During WalletConnect Offer Approval - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
`formatAmountWithRoyalties` in `parseCommandDisplay.ts` incorrectly divides the fungible amount by the number of NFTs before computing each royalty, producing an `amountWithRoyalties` display value that is systematically lower than the actual amount the user will pay. When a WalletConnect dApp presents a `chia_wallet.take_offer` command containing multiple royalty-bearing NFTs, the user sees an understated total and approves a transaction that deducts more XCH/CAT than shown.

### Finding Description

The `formatAmountWithRoyalties` function computes the royalty-inclusive total to display to the user:

```typescript
// packages/gui/src/electron/commands/parseCommandDisplay.ts  lines 354-375
function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  ...
  const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← BUG
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;
  ...
}
``` [1](#0-0) 

`royaltyPercentages` is built by `royaltyPercentagesForSide`, which collects one entry per NFT on the opposite side of the trade: [2](#0-1) 

Each NFT's royalty is applied to the **full** fungible amount — not to a per-NFT slice. The correct formula is:

```
royaltyAmount = Σ (amount × royaltyPercentage_i / 10_000)
```

The code instead computes:

```
splitAmount   = amount / N
royaltyAmount = Σ (splitAmount × royaltyPercentage_i / 10_000)
              = (amount / N) × Σ (royaltyPercentage_i / 10_000)
```

This is exactly `1/N` of the correct value. For a single NFT (`N = 1`) the result is correct; for two or more NFTs the displayed total is understated by a factor of `N`.

**Concrete example** (matching the existing test at lines 251-335 of the test file):

| Parameter | Value |
|---|---|
| Fungible amount | 100 000 000 mojos (0.0001 XCH) |
| NFT 1 royalty | 500 (5 %) |
| NFT 2 royalty | 10 (0.1 %) |
| **Displayed** `amountWithRoyalties` | 0.00010255 XCH |
| **Correct** `amountWithRoyalties` | 0.0001051 XCH |
| Under-statement | ~0.0000025 XCH per 0.0001 XCH offered | [3](#0-2) 

The test itself encodes the wrong expected value, confirming the bug is present and untested-against-correct-behavior.

`withRoyaltyTotals` calls `formatAmountWithRoyalties` and attaches the result as `amountWithRoyalties` on every fungible spending line: [4](#0-3) 

This value is then surfaced in the WalletConnect approval dialog via `walletDeltaToDisplay` → `parseCommandDisplay` → the Electron `Confirm` dialog: [5](#0-4) 

### Impact Explanation

When a WalletConnect-connected dApp (or any party that can craft a valid offer string) presents a `chia_wallet.take_offer` command containing two or more royalty-bearing NFTs, the Chia GUI approval dialog shows the user an `amountWithRoyalties` figure that is lower than what the Chia daemon will actually deduct. The user reads the understated total, decides it is acceptable, clicks Approve, and the daemon executes the transaction — deducting the correct (higher) royalty-inclusive amount from their XCH or CAT balance. The discrepancy grows with the number of NFTs and their royalty rates.

This matches the allowed High impact: **"Corruption … of … offer … state that causes a user to approve … the wrong … amount."**

### Likelihood Explanation

- The trigger requires only a valid multi-NFT offer string passed via WalletConnect `chia_wallet.take_offer`; no privileged access is needed.
- Multi-NFT offers with royalties are a supported and documented feature of the Chia offer protocol.
- The bug is silent — no error is thrown, the display simply shows a lower number.
- The existing test suite validates the wrong value, so the bug has not been caught by CI.

### Recommendation

Remove the `splitAmount` division. Each NFT's royalty is independently applied to the full fungible amount:

```typescript
function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  // Each NFT royalty is applied to the full amount, not a per-NFT slice.
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }
  return mojoToCATLocaleString(totalAmount);
}
```

Update the corresponding test expectation for the two-NFT case from `'0.00010255'` to `'0.0001051'`.

### Proof of Concept

**Setup:** WalletConnect dApp calls `chia_wallet.take_offer` with an offer where:
- The taker (user) pays 1 000 000 000 000 mojos (1 XCH)
- The taker receives NFT-A (royalty 500 = 5 %) and NFT-B (royalty 200 = 2 %)

**Current (buggy) display calculation:**
```
splitAmount   = 1_000_000_000_000 / 2 = 500_000_000_000
royaltyAmount = (500_000_000_000 × 500 / 10_000)
              + (500_000_000_000 × 200 / 10_000)
              = 25_000_000_000 + 10_000_000_000
              = 35_000_000_000 mojos
amountWithRoyalties displayed = 1.035 XCH
```

**Correct calculation:**
```
royaltyAmount = (1_000_000_000_000 × 500 / 10_000)
              + (1_000_000_000_000 × 200 / 10_000)
              = 50_000_000_000 + 20_000_000_000
              = 70_000_000_000 mojos
amountWithRoyalties correct = 1.07 XCH
```

The user sees **1.035 XCH** in the approval dialog but the daemon deducts **1.07 XCH** — a 0.035 XCH discrepancy that grows linearly with the number of NFTs and their royalty rates. [6](#0-5)

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

    };
