### Title
`TurnBridge` msg_server handler is a no-op with no permission check, permanently disabling the bridge circuit breaker - (File: `x/cronos/keeper/msg_server.go`)

### Summary
The `TurnBridge` gRPC handler in `x/cronos/keeper/msg_server.go` is implemented as a stub that unconditionally returns `nil, nil`. It performs no permission check and modifies no state. Any unprivileged user can call it and receive a success response, while the bridge state is never actually changed. The bridge circuit breaker — the only mechanism to disable the gravity bridge in an emergency — is permanently non-functional.

### Finding Description
The Cronos permission system (ADR-009) defines two restricted operations: `MsgUpdateTokenMapping` (requires `CanChangeTokenMapping`) and `MsgTurnBridge` (requires `CanTurnBridge`). The `UpdateTokenMapping` handler correctly enforces its permission:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [1](#0-0) 

The `TurnBridge` handler, however, is a stub:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [2](#0-1) 

Two distinct defects are present simultaneously:
1. **No permission check**: The `CanTurnBridge` permission is never verified. Any unprivileged address can submit `MsgTurnBridge` and receive a success response.
2. **No state mutation**: The handler never reads or writes the bridge-enabled flag. The bridge state is immutable regardless of what any caller submits.

The `CanTurnBridge` permission constant is defined and the permission system is wired up: [3](#0-2) 

The integration test `test_gravity_turn_bridge` confirms the expected behavior: a validator with `CanTurnBridge` permission calls `turn_bridge("false")` and expects the bridge to stop processing batches. Because the handler is a no-op, this invariant is silently violated. [4](#0-3) 

### Impact Explanation
**High — Bypass of Cronos admin/permission authorization check and permanent inability to disable the bridge.**

- Any unprivileged address can submit `MsgTurnBridge` with `enable=false` or `enable=true` and receive a `200 OK` with no error. The `CanTurnBridge` permission gate is completely bypassed.
- The bridge circuit breaker — the sole emergency mechanism to halt gravity bridge operations — is permanently broken. No address, including the admin, can disable the bridge. If a security incident (e.g., a Gravity contract exploit or oracle manipulation) requires halting bridge outflows, the on-chain mechanism to do so does not function.
- This directly maps to the allowed High impact: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks"* and *"Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows."*

### Likelihood Explanation
The entry path is fully unprivileged: any address can submit a `MsgTurnBridge` Cosmos transaction. The proto service is registered and the CLI command `CmdTurnBridge` is exposed. [5](#0-4) 

The defect is reachable on every block. The missing permission check means the auth bypass is exploitable by any account with gas. The no-op nature means the bridge can never be disabled, which is a latent permanent impact.

### Recommendation
Implement `TurnBridge` with the same pattern as `UpdateTokenMapping`:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)

    senderAddr, err := sdk.AccAddressFromBech32(msg.Sender)
    if err != nil {
        return nil, err
    }
    if !k.HasPermission(ctx, []sdk.AccAddress{senderAddr}, CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }

    params := k.GetParams(ctx)
    params.EnableAutoDeployment = msg.Enable  // or the appropriate bridge-enabled field
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

The exact field to toggle should be confirmed against the Cronos params struct and the gravity bridge keeper. The permission check against `CanTurnBridge` is mandatory.

### Proof of Concept
1. Any address (no `CanTurnBridge` permission) submits:
   ```
   cronosd tx cronos turn-bridge false --from <any_account>
   ```
2. The transaction is accepted on-chain with code `0` (success).
3. The bridge continues processing batches — the state was never changed.
4. A permissioned admin also calls `turn-bridge false` — same result: success response, no effect.
5. The gravity bridge cannot be halted by any on-chain mechanism. [2](#0-1) [6](#0-5)

func CmdTurnBridge() *cobra.Command {
	cmd := &cobra.Command{
		Use:   "turn-bridge [true/false]",
		Short: "Turn Bridge",
		Args:  cobra.ExactArgs(1),
		RunE: func(cmd *cobra.Command, args []string) error {
			clientCtx, err := client.GetClientTxContext(cmd)
			if err != nil {
				return err
			}

			enable, err := strconv.ParseBool(args[0])
			if err != nil {
				return err
			}
			msg := types.NewMsgTurnBridge(clientCtx.GetFromAddress().String(), enable)
			if err := msg.ValidateBasic(); err != nil {
				return err
			}
			return tx.GenerateOrBroadcastTxCLI(clientCtx, cmd.Flags(), msg)
		},
	}

	flags.AddTxFlagsToCmd(cmd)
	return cmd
