### Title
Unbounded `calculate_rewards` Loop Over User-Inflatable Balance Trace Permanently Freezes Unclaimed Yield - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` function in the Pool contract iterates over a pool member's entire `pool_member_epoch_balance` trace without any bound. A pool member who repeatedly modifies their balance across many epochs without claiming rewards will accumulate an unbounded number of trace entries. Once the trace is large enough, every call to `claim_rewards` (and `pool_member_info_v1`) will exceed the Starknet block gas limit, permanently freezing the pool member's unclaimed yield with no recovery path.

### Finding Description

`calculate_rewards` in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch { break; }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

Each iteration performs multiple storage reads: one `pool_member_trace.at(entry_to_claim_from)` and one `find_sigma` call (which reads from `cumulative_rewards_trace`). Storage reads are the most expensive operations on Starknet.

The trace grows via `set_member_balance`, which inserts at key `current_epoch + K`:

```cairo
fn set_member_balance(ref self: ContractState, pool_member: ContractAddress, amount: Amount) {
    let trace = self.pool_member_epoch_balance.entry(pool_member);
    let pool_member_balance = PoolMemberBalanceTrait::new(
        balance: amount,
        cumulative_rewards_trace_idx: self.cumulative_rewards_trace_length() + 1,
    );
    trace.insert(key: self.get_epoch_plus_k(), value: pool_member_balance);
}
``` [2](#0-1) 

The `insert` implementation only updates the last entry if the key matches; otherwise it appends a new checkpoint:

```cairo
if last.key == key {
    last.value = value;
    checkpoints[len - 1].write(last);
} else {
    assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
    checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
}
``` [3](#0-2) 

Therefore, one new trace entry is created per epoch in which the pool member modifies their balance. `set_member_balance` is called from:

- `add_to_delegation_pool` (via `increase_member_balance`) — callable by the pool member or their reward address
- `exit_delegation_pool_intent` — callable by the pool member [4](#0-3) [5](#0-4) 

The `entry_to_claim_from` cursor is only advanced when `claim_rewards` is called successfully. If the pool member adds to their balance every epoch but defers claiming rewards, the loop must process all accumulated entries in a single transaction when they eventually call `claim_rewards`. [6](#0-5) 

There is no mechanism to prune or compact the trace. The `pool_member_info_v1` view function also calls `calculate_rewards` unconditionally, so even reading the member's state will fail once the trace is large enough. [7](#0-6) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Once the `pool_member_epoch_balance` trace grows beyond the gas-iteration threshold, every call to `claim_rewards` and `pool_member_info_v1` reverts with an out-of-gas error. The pool member's accumulated STRK rewards are permanently locked in the contract with no recovery path, since there is no partial-claim, trace-pruning, or emergency-withdrawal mechanism.

### Likelihood Explanation

**Medium.** Any pool member who:
1. Calls `add_to_delegation_pool` (or `exit_delegation_pool_intent`) at least once per epoch, AND
2. Does not call `claim_rewards` for an extended period

will accumulate one trace entry per epoch. This is a realistic pattern for delegators who regularly compound their stake or adjust their position. The developers themselves acknowledge the risk in the comment "This loop is unbounded but unlikely to exceed gas limits," indicating the assumption has not been formally bounded. Given Starknet's per-transaction gas limits and the cost of storage reads per iteration, a delegator active over hundreds of epochs without claiming is at material risk.

### Recommendation

1. **Enforce a maximum trace length** by requiring `claim_rewards` to be called before `add_to_delegation_pool` or `exit_delegation_pool_intent` if the trace exceeds a safe threshold (e.g., 100 entries).
2. **Alternatively, implement partial reward claiming** that processes a bounded number of trace entries per call and advances `entry_to_claim_from` incrementally, allowing the pool member to drain the backlog over multiple transactions.
3. **Add a protocol-level invariant** that the trace length for any pool member is bounded by a constant (e.g., `MAX_UNCLAIMED_EPOCHS`), enforced at every balance-modifying entry point.

### Proof of Concept

1. Pool member `A` enters the delegation pool with `min_amount`.
2. Each epoch for `N` epochs, `A` calls `add_to_delegation_pool(pool_member: A, amount: 1)` without ever calling `claim_rewards`. Each call appends one new entry to `pool_member_epoch_balance[A]` (different epoch key each time).
3. After `N` epochs, `pool_member_epoch_balance[A]` has `N` entries.
4. `A` calls `claim_rewards(pool_member: A)`. The `calculate_rewards` loop must iterate all `N` entries, each requiring 2–3 storage reads via `find_sigma`. At Starknet storage-read costs, for sufficiently large `N` (protocol-dependent, but achievable over months of active delegation), the transaction exceeds the block gas limit and reverts.
5. Every subsequent call to `claim_rewards` or `pool_member_info_v1` for `A` also reverts. `A`'s accumulated STRK rewards are permanently frozen. [8](#0-7) [9](#0-8)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
