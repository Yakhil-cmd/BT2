### Title
Gateway `validate_resource_bounds` Omits `l1_gas` and `l1_data_gas` Price Checks, Admitting Transactions That Fail Blockifier Pre-Validation Fee-Free - (File: crates/apollo_gateway/src/stateless_transaction_validator.rs)

---

### Summary

Both the stateless and stateful gateway validators check only `l2_gas.max_price_per_unit` against configured minimums. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields of `AllResourceBounds` are never validated at the gateway admission layer. A transaction with these fields set to zero passes every gateway check, is admitted to the mempool, and then fails inside the blockifier's `check_fee_bounds` pre-validation stage — without paying any fee — because the blockifier does validate all three gas prices against the actual block prices.

---

### Finding Description

**Stateless validator — only `l2_gas` price is checked:**

`StatelessTransactionValidator::validate_resource_bounds` (lines 56–88) performs two checks on `AllResourceBounds`:

1. Total fee > 0 (the `ZeroResourceBounds` guard, line 66).
2. `l2_gas.max_price_per_unit >= self.config.min_gas_price` (line 71).

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never read. [1](#0-0) 

**Stateful validator — same gap, with an explicit TODO:**

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` (lines 358–390) compares only `l2_gas.max_price_per_unit` against the previous block's L2 gas price. The function carries the comment:

```
// TODO(Arni): Consider running this validation for all gas prices.
``` [2](#0-1) 

**Blockifier — validates all three gas prices:**

`AccountTransaction::check_fee_bounds` (lines 374–475) iterates over all three resources (`L1Gas`, `L1DataGas`, `L2Gas`) and for each checks both `max_price_per_unit >= actual_gas_price` and `max_amount >= minimal_gas_amount`. If any check fails it returns `TransactionPreValidationError::TransactionFeeError(InsufficientResourceBounds)`. [3](#0-2) 

**The gap:** A transaction with `l1_gas.max_price_per_unit = 0` and `l2_gas.max_price_per_unit = valid_price` satisfies:

- `ZeroResourceBounds` check — total fee is non-zero because `l2_gas` contributes.
- `min_gas_price` check — only `l2_gas` price is compared.
- Stateful L2-price threshold — only `l2_gas` price is compared.

It therefore clears all gateway checks, enters the mempool, and is later handed to the blockifier. There, `check_fee_bounds` compares `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (non-zero)` and returns `InsufficientResourceBounds`. Because this failure occurs inside `perform_pre_validation_stage` — before any execution or fee transfer — **no fee is charged**. [4](#0-3) 

---

### Impact Explanation

An unprivileged attacker can flood the gateway with transactions that:

1. Pass all stateless and stateful gateway checks.
2. Are admitted to the mempool.
3. Fail during blockifier pre-validation (`check_fee_bounds`) with zero fee charged.
4. Are never included in a block.

The admission invariant is broken: the gateway accepts transactions it should reject. The attacker bears no economic cost per rejected transaction, enabling sustained mempool pollution and sequencer resource exhaustion. This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The trigger requires no privilege. Any user can craft a V3 `AllResourceBounds` transaction with `l1_gas.max_price_per_unit = 0` and a valid `l2_gas.max_price_per_unit`. The attack is stateless, requires no on-chain state, and can be repeated indefinitely at zero fee cost.

---

### Recommendation

Extend `validate_resource_bounds` in both validators to check `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit`:

- **Stateless validator**: compare each price against `self.config.min_gas_price` (the same static floor already applied to `l2_gas`).
- **Stateful validator**: compare each price against the corresponding previous-block gas price (L1 gas price and L1 data gas price are already available in `BlockInfo::gas_prices`), applying the same `min_gas_price_percentage` threshold.

The existing TODO comment in `validate_tx_l2_gas_price_within_threshold` already acknowledges this gap. [5](#0-4) 

---

### Proof of Concept

Construct an `RpcInvokeTransactionV3` with:

```
resource_bounds = AllResourceBounds {
    l1_gas:      ResourceBounds { max_amount: 1, max_price_per_unit: 0 },   // price = 0
    l1_data_gas: ResourceBounds { max_amount: 1, max_price_per_unit: 0 },   // price = 0
    l2_gas:      ResourceBounds { max_amount: 1_000_000,
                                  max_price_per_unit: 8_000_000_000 },       // valid
}
```

**Step 1 — Stateless validation passes:**
- `ZeroResourceBounds`: total fee = `1_000_000 * 8_000_000_000 > 0` → passes.
- `min_gas_price` check: `l2_gas.max_price_per_unit (8e9) >= min_gas_price (8e9)` → passes.
- `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never read. [6](#0-5) 

**Step 2 — Stateful validation passes:**
- `validate_tx_l2_gas_price_within_threshold`: only `l2_gas.max_price_per_unit` is compared against the previous block L2 price → passes. [7](#0-6) 

**Step 3 — Mempool admits the transaction.**

**Step 4 — Blockifier `check_fee_bounds` fails:**
- `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (non-zero)` → `InsufficientResourceBounds { resource: L1Gas, … }`.
- Error propagates as `TransactionPreValidationError`; no fee is charged; transaction is dropped. [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())

        tx_resource_bounds: ValidResourceBounds,
