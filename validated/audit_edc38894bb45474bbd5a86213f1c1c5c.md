### Title
Pool Contract `claim_rewards` Bypasses Staking Contract Pause Mechanism - (File: `src/pool/pool.cairo`)

### Summary
The `Pool` contract's `claim_rewards` function does not check whether the staking contract is paused. Because it transfers STRK rewards directly from the pool contract's own balance without invoking any staking contract function, it bypasses the `general_prerequisites()` / `assert_is_unpaused()` guard that protects every other state-changing path in the protocol. This is the direct analog of M-29: the pool contract is the "whitelisted intermediary" through which the pause restriction can be circumvented.

### Finding Description

The staking contract enforces a pause flag via `general_prerequisites()`:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
fn assert_is_unpaused(self: @ContractState) {
    assert!(!self.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
}
``` [1](#0-0) [2](#0-1) 

Every public staking function that moves funds or updates state calls `general_prerequisites()` first, including the staking contract's own `claim_rewards`:

```cairo
fn claim_rewards(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    self.general_prerequisites();   // ← pause enforced here
    ...
}
``` [3](#0-2) 

The pool contract's `claim_rewards`, however, makes **no call to the staking contract** and performs **no pause check**:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    ...
    let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
    reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
    ...
}
``` [4](#0-3) 

Rewards are pre-funded into the pool contract by `update_pool_rewards` (called from `_update_rewards`), which transfers STRK tokens and updates `cumulative_rewards_trace` before any pause could be applied:

```cairo
fn update_pool_rewards(...) {
    ...
    self.send_rewards_to_delegation_pool(..., amount: pool_rewards, ...);
    pool_dispatcher.update_rewards_from_staking_contract(rewards: pool_rewards, ...);
    ...
}
``` [5](#0-4) 

Once those tokens are in the pool contract and `cumulative_rewards_trace` is updated, `claim_rewards` can distribute them to any pool member regardless of the staking contract's pause state.

The pool contract has no independent pause storage field and no mechanism to query the staking contract's pause status before distributing rewards. [6](#0-5) 

### Impact Explanation

If the security agent pauses the staking contract in response to an incorrect or inflated reward calculation event, pool members can immediately call `Pool::claim_rewards` to drain those incorrectly calculated STRK rewards from the pool contract. The staking contract's `claim_rewards` is blocked, but the pool contract's equivalent is not, creating an asymmetry that defeats the purpose of the pause. This constitutes **theft of unclaimed yield** — the exact impact class listed as High in the allowed scope.

### Likelihood Explanation

The entry path is fully unprivileged: any pool member (or their registered reward address) can call `Pool::claim_rewards` at any time. No special role, key, or bridge interaction is required. The pool contract is publicly deployed and its address is known. The only precondition is that rewards have been distributed to the pool contract in a prior epoch, which is the normal operating state of any active pool.

### Recommendation

Add a pause check inside `Pool::claim_rewards` by querying the staking contract's pause status before distributing rewards:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "Contract is paused");
    ...
}
```

Alternatively, expose a dedicated `assert_is_unpaused` function on the staking contract and call it from the pool contract at the start of every state-changing pool function, consistent with how `enter_delegation_pool` and `add_to_delegation_pool` already propagate the pause check indirectly through `add_stake_from_pool`.

### Proof of Concept

1. Staker and pool are active; rewards accumulate over several epochs. `_update_rewards` transfers STRK into the pool contract and updates `cumulative_rewards_trace`.
2. A reward calculation bug inflates pool rewards. The security agent calls `Staking::pause()`.
3. Pool member calls `Pool::claim_rewards(pool_member)` directly on the pool contract.
4. The pool contract reads `cumulative_rewards_trace`, computes the inflated reward amount, and calls `STRK.transfer(reward_address, inflated_amount)` — no staking contract call is made, no pause check is hit.
5. The pool member receives the inflated STRK rewards. The staking contract's `claim_rewards` would have reverted with `CONTRACT_IS_PAUSED`, but the pool path does not.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
