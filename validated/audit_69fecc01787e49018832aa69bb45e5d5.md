### Title
Unprivileged Caller Can Permanently Suppress Block Reward Distribution via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards` flag. When called with `disable_rewards: true`, it advances the **global** `last_reward_block` checkpoint without distributing any rewards. Because the function enforces a per-block uniqueness guard on that same global slot, any attacker can front-run every legitimate reward call, permanently preventing all stakers from receiving their consensus block rewards.

---

### Finding Description

`update_rewards` is exposed as a public entry point with no caller authorization check. Its only gate is `general_prerequisites()`, which only verifies the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function then enforces a global, single-use-per-block guard: [2](#0-1) 

After passing staker validity checks, it unconditionally writes the current block number to the global `last_reward_block` slot: [3](#0-2) 

When `disable_rewards` is `true`, the function returns immediately after writing `last_reward_block`, distributing nothing: [4](#0-3) 

Because `last_reward_block` is a **single global storage field** (not per-staker), any subsequent call to `update_rewards` in the same block — regardless of which staker is targeted — will revert with `REWARDS_ALREADY_UPDATED`. The block's reward opportunity is permanently consumed. [5](#0-4) 

The analog to the external report is direct: just as unregistered players could interact with the game contract without meeting the age-check prerequisite, here any unregistered/unprivileged caller can invoke a reward-accounting function without meeting the prerequisite of being an authorized reward distributor, bypassing the intended access restriction entirely.

---

### Impact Explanation

After `consensus_rewards_first_epoch` is set, block rewards are the sole yield mechanism for stakers. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block:

1. Consumes the global `last_reward_block` slot for that block.
2. Causes every legitimate `update_rewards(staker, disable_rewards: false)` call in that block to revert.
3. Permanently destroys the block's reward allocation — the `reward_supplier` funds are never claimed and the stakers' `unclaimed_rewards_own` is never incremented.

Sustained over multiple blocks, this constitutes **permanent freezing of unclaimed yield** for all stakers — a **High** impact per the allowed scope. [6](#0-5) 

---

### Likelihood Explanation

- The function is fully public; no role, no registration, no stake required.
- The attacker only needs to pay gas to call `update_rewards` with a valid staker address and `disable_rewards: true`.
- Valid staker addresses are discoverable on-chain via the `NewStaker` event or the `stakers` vector.
- The attack can be automated to fire at the first transaction of every block, making it a persistent, low-cost griefing campaign. [7](#0-6) 

---

### Recommendation

Restrict `update_rewards` to authorized callers only. The simplest fix is to require the caller to be the staker themselves, their operational address, or a designated keeper role — mirroring the pattern used in `update_rewards_from_attestation_contract`, which enforces `assert_caller_is_attestation_contract()`: [8](#0-7) 

Alternatively, remove the public `disable_rewards` parameter entirely and handle the pre-consensus early-return internally, so external callers cannot suppress reward distribution.

---

### Proof of Concept

```
Block N begins.

1. Attacker calls:
     staking.update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)

2. Inside update_rewards:
   - general_prerequisites() passes (contract not paused, caller != 0).
   - current_block_number (N) > last_reward_block → passes.
   - Staker validity checks pass (staker exists, is active, has non-zero balance).
   - last_reward_block.write(N)          ← global slot consumed.
   - disable_rewards == true → return.   ← no rewards distributed.

3. Legitimate keeper calls:
     staking.update_rewards(staker_address=<staker>, disable_rewards=false)

4. Inside update_rewards:
   - current_block_number (N) > last_reward_block (N) → FALSE → REVERT: REWARDS_ALREADY_UPDATED.

5. Block N's rewards are permanently lost for all stakers.
   Repeat every block → all consensus yield is permanently frozen.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
