### Title
Excess ETH Permanently Locked in `RewardSupplier.tick()` When No Minting Is Required - (File: `L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

The `tick()` function in `RewardSupplier.sol` is `external payable` and accepts ETH for L1→L2 messaging fees. However, when `amountToMint == 0` (no pending mint requests), the entire `msg.value` is silently retained in the contract with no refund and no withdrawal mechanism. Any ETH sent by a caller in this scenario is permanently frozen.

---

### Finding Description

`tick()` is a public, permissionless, payable function. Its only use of `msg.value` is inside the `if (amountToMint > 0)` branch:

```solidity
// L1/starkware/solidity/stake/RewardSupplier.sol

function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ...
        uint256 msgFee = msg.value / 2;
        bridge().depositWithMessage{value: msgFee}(...);   // uses half
        // ...
        msgFee = msg.value - msgFee;
        messagingContract().sendMessageToL2{value: msgFee}(...); // uses other half
    }
    // If amountToMint == 0: msg.value is never used, never refunded
}
``` [1](#0-0) 

When `amountToMint == 0`, the `if` block is skipped entirely. There is no `else` branch, no `require(msg.value == 0)` guard, no `payable(msg.sender).transfer(msg.value)` refund, and no ETH withdrawal function anywhere in the contract or its storage contract. [2](#0-1) 

The contract inherits from `RewardSupplierStorage` and `ProxySupportImpl`, neither of which provides an ETH rescue or sweep function.

---

### Impact Explanation

Any ETH sent to `tick()` when `requiredMinting()` returns `amountToMint == 0` is permanently locked in the `RewardSupplier` contract. There is no recovery path. This constitutes **permanent freezing of the caller's funds**.

This matches the allowed impact: **Permanent freezing of funds**.

---

### Likelihood Explanation

`tick()` is a public, permissionless function intended to be called by keepers or bots. A caller must supply ETH to cover the two L1→L2 message fees when minting is needed. The condition `amountToMint == 0` is a normal, frequently occurring state (whenever the L2 reward supplier has sufficient balance and has not emitted new mint requests). Any caller who sends ETH without first calling the view function `requiredMinting()` — or who races with another `tick()` call that drains the pending messages — will have their ETH permanently locked. [3](#0-2) 

---

### Recommendation

Add a guard at the start of `tick()` to reject ETH when no minting will occur, or refund excess ETH unconditionally:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ... existing logic ...
    } else {
        // Refund any ETH sent when there is nothing to do
        require(msg.value == 0, "No minting required: refund ETH");
        // or: if (msg.value > 0) payable(msg.sender).transfer(msg.value);
    }
}
```

Alternatively, add a strict check at the top: `require(amountToMint > 0 || msg.value == 0, ...)`.

---

### Proof of Concept

1. The L2 reward supplier has sufficient balance; `requiredMinting()` returns `(0, 0)`.
2. A keeper bot calls `tick{value: 0.01 ether}()` (supplying ETH in anticipation of fees).
3. `amountToMint == 0`, so the `if` block is skipped.
4. The 0.01 ETH is now held by `RewardSupplier` with no function to retrieve it.
5. No event is emitted, no revert occurs — the ETH is silently and permanently frozen. [1](#0-0)

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
