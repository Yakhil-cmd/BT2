### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Freeze All Stakers' Consensus Rewards — (File: `src/staking/staking.cairo`)

### Summary

The `update_rewards` function in `staking.cairo` has no access control beyond a basic non-zero caller check. It accepts a public `disable_rewards: bool` parameter. Because `last_reward_block` is a **global** (not per-staker) state variable, any unprivileged caller can invoke `update_rewards(any_valid_staker, true)` once per block to mark that block as processed while skipping all reward distribution. Any subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`. An attacker who front-runs the consensus layer's call every block permanently freezes all stakers' unclaimed consensus-phase yield.

### Finding Description

`update_rewards` is the entry point for distributing per-block consensus rewards to the designated block proposer. Its only guard is `general_prerequisites()`:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

No role check, no allowlist, no restriction to the consensus layer. The function then unconditionally writes the current block number to the global `last_reward_block` storage slot before branching on `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

`last_reward_block` is declared as a single scalar (not a per-staker map):

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [3](#0-2) 

The guard at the top of the function enforces that only one call per block can succeed:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [4](#0-3) 

**Attack path:**

1. Attacker watches the mempool for any `update_rewards` call in block N.
2. Attacker front-runs with `update_rewards(any_valid_staker_address, true)`.
3. `last_reward_block` is set to N; `_update_rewards` is never called; no STRK is credited to any staker or pool.
4. The legitimate consensus-layer call arrives and reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeating this every block permanently prevents all stakers from accumulating consensus rewards.

The attacker does not need to be a staker. They only need to supply any currently-active staker address with non-zero balance (trivially readable from on-chain state) to pass the staker-existence assertions:

```cairo
let staker_info = self.internal_staker_info(:staker_address);
...
assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);
``` [5](#0-4) 

### Impact Explanation

When `disable_rewards` is `true`, `_update_rewards` is never reached, so `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` is never called and no STRK is transferred to any staker or pool for that block. The skipped block rewards are not deferred — they are permanently lost. An attacker who executes this once per block causes **permanent freezing of all stakers' unclaimed consensus-phase yield**, matching the allowed High impact: *"Permanent freezing of unclaimed yield."*

### Likelihood Explanation

Medium. Starknet transaction fees are low. The attacker needs to submit one transaction per block (every few seconds). The call is cheap: it reads a few storage slots, writes one, and returns early. No capital is required. The attacker has no profit motive but can inflict sustained, protocol-wide damage. Front-running is straightforward because the target function is public and the mempool is observable.

### Recommendation

Restrict `update_rewards` to an authorized caller — either the consensus layer contract address stored in a dedicated storage slot, or a dedicated role (e.g., `CONSENSUS_LAYER_ROLE`). The `disable_rewards` flag should only be settable by the entity that can authoritatively determine whether a block proposer participated in consensus.

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.roles.only_consensus_layer(); // <-- add role check
    ...
}
```

Alternatively, split the function: expose a permissionless `trigger_rewards(staker_address)` that always distributes rewards, and a privileged `slash_rewards(staker_address)` that skips them.

### Proof of Concept

```
Block N arrives.

Attacker tx (submitted first, higher gas):
  staking.update_rewards(alice_staker_address, disable_rewards=true)
  → last_reward_block := N
  → returns early, no rewards distributed

Consensus layer tx (arrives second):
  staking.update_rewards(alice_staker_address, disable_rewards=false)
  → assert!(N > N)  ← FAILS with REWARDS_ALREADY_UPDATED

Alice receives 0 STRK for block N.
Repeat every block → Alice's (and every other staker's) consensus rewards are permanently frozen.
```
