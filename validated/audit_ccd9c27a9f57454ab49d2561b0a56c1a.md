### Title
Missing `is_declared` Guard for V0 Classes in `get_compiled_class` Class-Manager Path ŌĆö (`crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class` enforces a declaration check (`is_declared`) for V1 (Sierra) classes when the class-manager path is active, but **unconditionally returns** a `RunnableCompiledClass::V0` for deprecated (Cairo 0) classes without any such check. Because the class manager is a global, block-number-agnostic store, it can hold V0 classes that were declared at a block number *later* than the `state_number` being queried. An unprivileged user can exploit this via the public RPC simulation endpoint to execute code from a class that is not declared at the simulated block, producing authoritative-looking wrong simulation results.

---

### Finding Description

In `get_compiled_class`, when `class_manager_handle` is `Some`, the function dispatches on the returned `ContractClass` variant:

```
ContractClass::V1  ŌåÆ  is_declared check against storage at self.state_number
                       ŌåÆ UndeclaredClassHash if not declared  Ō£ō

ContractClass::V0  ŌåÆ  unconditional Ok(RunnableCompiledClass::V0(...))
                       ŌåÆ no declaration check at all           Ō£Ś
``` [1](#0-0) 

The developer-acknowledged TODO comment on line 136 confirms this is a known gap:

> `// TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.` [2](#0-1) 

The class manager (`ClassManager`) stores deprecated classes via `add_deprecated_class`, which writes to `FsClassStorage` keyed only by `class_id` (hash), with **no block-number metadata**. Once a V0 class is stored, `get_executable` returns it for any query regardless of the `state_number` the caller is using. [3](#0-2) [4](#0-3) 

The V1 path correctly calls `is_contract_class_declared` with `self.state_number` and returns `UndeclaredClassHash` on failure: [5](#0-4) 

The `SyncStateReader` (gateway path) has a parallel test case `class_not_declared_but_in_class_manager` that explicitly expects `UndeclaredClassHash` for V1 ŌĆö but no equivalent test exists for V0, confirming the gap is untested and unguarded. [6](#0-5) 

---

### Impact Explanation

**High ŌĆö RPC simulation/execution returns an authoritative-looking wrong value.**

When a user calls `starknet_simulateTransactions` (or `starknet_estimateFee`, `starknet_traceTransaction`) at a historical block number M, the `ExecutionStateReader` is constructed with `state_number = right_after_block(M)`. If a V0 class C was declared at block N > M, the class manager already holds C. A transaction that issues a `library_call` syscall with class hash C will cause `get_compiled_class(C)` to return `RunnableCompiledClass::V0(C)` ŌĆö executing code from a class that does not exist at block M. The simulation result (trace, fee estimate, return data, events) is wrong and presented to the caller as authoritative.

---

### Likelihood Explanation

- The RPC simulation endpoint is public and unprivileged.
- The attacker only needs to know a V0 class hash that was declared at any block after the target simulation block. This is trivially discoverable from on-chain data (e.g., Starknet explorer).
- The attacker deploys a contract (using any declared class) whose `__execute__` or any entry point issues a `library_call` with the target V0 class hash.
- No special privileges, keys, or sequencer access are required.

---

### Recommendation

Apply the same `is_declared` guard to the `ContractClass::V0` arm. The storage already supports `get_deprecated_class_definition_block_number` which returns the block at which a deprecated class was first declared: [7](#0-6) 

The fix is to call `is_contract_class_declared` (or an equivalent that checks `deprecated_declared_classes_block_table`) before returning `RunnableCompiledClass::V0`, mirroring the V1 branch. The TODO comment should be resolved once `get_class_definition_block_number` is fixed for Cairo 0 classes.

---

### Proof of Concept

**Rust unit test sketch** (mirrors the existing `class_not_declared_but_in_class_manager` V1 test):

```rust
// 1. Build an ExecutionStateReader with:
//    - storage_reader: storage where V0 class C is NOT declared at state_number S
//      (e.g., declared at block 5, state_number = right_after_block(2))
//    - class_manager_handle: mock that returns ContractClass::V0(C) for class hash H

let mut mock_class_manager = MockClassManagerClient::new();
mock_class_manager
    .expect_get_executable()
    .with(eq(class_hash_H))
    .return_once(|_| Ok(Some(ContractClass::V0(deprecated_class_C))));

let reader = ExecutionStateReader {
    storage_reader: storage_with_class_declared_at_block_5,
    state_number: StateNumber::right_after_block(BlockNumber(2)), // before declaration
    maybe_pending_data: None,
    missing_compiled_class: Cell::new(None),
    class_manager_handle: Some((Arc::new(mock_class_manager), runtime_handle)),
};

// 2. Call get_compiled_class ŌĆö currently returns Ok(V0) instead of UndeclaredClassHash
let result = reader.get_compiled_class(class_hash_H);

// 3. Assert the BUG: currently passes (returns Ok), should return Err
assert!(
    matches!(result, Err(StateError::UndeclaredClassHash(_))),
    "Expected UndeclaredClassHash but got {:?}", result
);
```

The test will **fail** on the current code (returning `Ok(RunnableCompiledClass::V0(...))`) and **pass** only after the `is_declared` guard is added to the V0 arm.
