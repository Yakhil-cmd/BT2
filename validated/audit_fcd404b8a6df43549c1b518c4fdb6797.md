### Title
Unbounded Iteration Over Monotonically Growing `stakers` Vec in `get_stakers` Causes Unbounded Gas Consumption — (File: `src/staking/staking.cairo`)

---

### Summary

The `stakers` storage `Vec` in the Staking contract grows by one entry on every `stake()` call and is **never pruned** when a staker exits. The `get_stakers` consensus function iterates over the entire Vec unconditionally. Any unprivileged user can call `stake()` to permanently append their address to this Vec, even if they immediately unstake. Over time — or via deliberate spam — the Vec grows without bound, making `get_stakers` increasingly expensive until it exceeds the gas limit and becomes uncallable.

---

### Finding Description

**Root cause — monotonically growing Vec:**

Every call to `stake()` unconditionally appends the caller to `self.stakers`: [1](#0-0) 

The storage declaration carries an explicit developer note that entries are never removed: [2](#0-1) 

**Root cause — full-range iteration with no bound:**

`get_stakers` iterates over every element ever appended, including long-since-unstaked addresses: [3](#0-2) 

For each element the loop reads the address from storage and calls `is_staker_active`, which itself performs additional storage reads. There is no early-exit, no pagination, no gas guard, and no cap on the Vec length.

**Parallel in `pool.cairo`:**

The `calculate_rewards` inner loop carries the same structural problem and is explicitly flagged by the developers: [4](#0-3) 

---

### Impact Explanation

`get_stakers` is the consensus-layer entrypoint that returns the active validator set for a given epoch. As the `stakers` Vec grows, the per-call gas cost grows linearly. Once the Vec is large enough, every invocation of `get_stakers` exceeds the Starknet block gas limit and reverts. The consensus layer loses the ability to enumerate validators, which breaks liveness proofs and reward distribution. This maps to **Medium — Unbounded gas consumption / Griefing with damage to the protocol**.

---

### Likelihood Explanation

**Medium.** The attack is permissionless: `stake()` requires only meeting `min_stake` and token approval. An attacker registers N accounts, stakes the minimum with each, waits one `exit_wait_window` (~1 week), and calls `unstake_action` to recover principal. The addresses remain in `self.stakers` permanently. The financial cost is the opportunity cost of locking `N × min_stake` for one week, not a permanent loss. Natural protocol growth also inflates the Vec without any attacker involvement, making this a latent time-bomb that worsens every day.

---

### Recommendation

1. **Replace the append-only Vec with an active-staker count + skip-inactive logic** or maintain a separate `active_stakers` set that is updated on `unstake_action`.
2. **Alternatively**, when `unstake_action` is called, swap-and-pop the staker's entry from `self.stakers` to keep the Vec compact.
3. **Add pagination** to `get_stakers` (offset + limit parameters) so callers can retrieve the set in bounded chunks regardless of Vec size.
4. **Mirror the `calculate_rewards` fix**: add a `max_iterations` guard or require callers to claim in bounded windows, preventing the analogous unbounded loop in `pool.cairo` from permanently freezing a member's unclaimed yield.

---

            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
