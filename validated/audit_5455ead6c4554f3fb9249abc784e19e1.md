### Title
Missing Caller Authorization on `update_rewards` Allows Anyone to Suppress Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary
`update_rewards` in the `StakingRewardsManager` interface is specified to be callable only by the Starkware sequencer, but the implementation contains no such caller check. Any unprivileged address can invoke it with `disable_rewards: true`, consuming the per-block reward slot and permanently discarding that block's yield for all stakers.

### Finding Description
The protocol specification explicitly restricts `update_rewards` to a single caller:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. The only gate is `general_prerequisites()`, which only asserts the contract is unpaused and the caller is non-zero: [2](#0-1) [3](#0-2) 

The function then unconditionally writes `current_block_number` into `last_reward_block`: [4](#0-3) 

Because `last_reward_block` is a single global slot, any call that succeeds in a given block prevents every subsequent call in that same block from passing the `REWARDS_ALREADY_UPDATED` guard. When the caller passes `disable_rewards: true`, the function returns immediately after writing `last_reward_block`, distributing zero rewards: [5](#0-4) 

The rewards that would have been distributed in that block are permanently lost — they are never added to `unclaimed_rewards_own` or forwarded to delegation pools.

### Impact Explanation
An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` before the sequencer in a given block:

1. Marks that block as "already rewarded" via `last_reward_block`.
2. Distributes nothing to any staker or pool.
3. Prevents the sequencer from distributing rewards for that block (the sequencer's call reverts with `REWARDS_ALREADY_UPDATED`).

Rewards for that block are permanently unrecoverable. If repeated across many blocks, this constitutes permanent freezing of unclaimed yield for all stakers and delegators — matching the **High** impact category: *Permanent freezing of unclaimed yield*.

### Likelihood Explanation
On Starknet the sequencer controls transaction ordering, so in normal operation it can always place its own `update_rewards` call first. However:

- The sequencer is not required to call `update_rewards` in every block; any block it skips is a window for the attacker.
- A griefing actor with no profit motive only needs to monitor for blocks where the sequencer has not yet called the function and submit a cheap transaction.
- The missing check is a direct, unconditional code path — no special setup, no funds, no privileged role required.

Likelihood is **Medium**.

### Recommendation
Add an explicit caller check at the top of `update_rewards`, analogous to the checks already present in `update_rewards_from_attestation_contract` and `update_unclaimed_rewards_from_staking_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Add: assert caller is the designated sequencer address stored in storage,
    // or use a role-based check (e.g., self.roles.only_sequencer()).
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

The sequencer address should be stored in contract storage and settable by governance, mirroring the pattern used for `attestation_contract`. [6](#0-5) 

### Proof of Concept

1. Deploy the system normally; advance past the K-epoch delay so a staker has active balance.
2. At the start of a new block (before the sequencer's transaction), submit:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(staker_address, disable_rewards: true)
   ```
3. Observe that `last_reward_block` is now set to the current block number.
4. The sequencer's subsequent call to `update_rewards` reverts with `REWARDS_ALREADY_UPDATED`.
5. Confirm that `staker_info.unclaimed_rewards_own` and the pool balance are unchanged — the block's rewards are gone.
6. Repeat each block to permanently suppress all reward accrual. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
