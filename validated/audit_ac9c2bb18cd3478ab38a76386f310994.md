### Title
Unguarded `new()` Initialization Allows Arbitrary Caller to Seize Engine Ownership - (File: engine/src/contract_methods/admin.rs)

### Summary
The `new()` function, which initializes the Aurora Engine's critical state (including `owner_id`), performs no caller authentication. Any NEAR account that calls it before the legitimate deployer can set themselves as the engine owner, gaining full administrative control over the contract.

### Finding Description
The `new()` function in `engine/src/contract_methods/admin.rs` is exposed as a public NEAR entry point via `lib.rs`. Its only guard is a check that the state has not yet been written:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // No check on env.predecessor_account_id() whatsoever
    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input)...;
    let state: EngineState = args.into();   // owner_id comes from args
    state::set_state(&mut io, &state)?;
    Ok(())
}
``` [1](#0-0) 

The entry point in `lib.rs` exposes this with no additional guard:

```rust
pub extern "C" fn new() {

The `owner_id` stored in `EngineState` is the sole principal checked by every privileged operation: `set_owner`, `upgrade`, `stage_upgrade`, `pause_contract`, `resume_contract`, `set_key_manager`, `attach_full_access_key`, `factory_update`, `factory_set_wnear_address`, `set_upgrade_delay_blocks`, and others all gate on `require_owner_only`. [3](#0-2) 

### Impact Explanation
An attacker who wins the race to call `new()` with attacker-controlled `NewCallArgs` (setting `owner_id` to their own NEAR account) becomes the sole owner of the Aurora Engine. From that position they can:

- Call `upgrade()` or `stage_upgrade()` + `deploy_upgrade()` to replace the contract bytecode with arbitrary code, enabling direct theft of all ETH and ERC-20 balances held in the Aurora EVM.
- Call `pause_contract()` to permanently freeze all user funds.
- Call `attach_full_access_key()` to add a full-access key to the engine account, giving unrestricted on-chain control.

Impact: **Critical** — direct theft of all user funds at rest, or permanent fund freeze. [4](#0-3) [5](#0-4) 

### Likelihood Explanation
In NEAR, a contract deployment and a subsequent function call can be bundled into a single atomic batch transaction, which would close this window. However:

1. The code itself imposes **no enforcement** of this — nothing prevents `new()` from being called in a separate transaction after deployment.
2. Any deployment script, CI pipeline, or operator procedure that issues the `DeployContract` action and the `new()` call as **two separate transactions** opens a block-level race window.
3. A validator, a monitoring bot, or any NEAR account that observes the deployment transaction in the mempool or on-chain can immediately submit a `new()` call with attacker-controlled arguments before the legitimate initializer does.
4. Aurora Engine is intended to be deployable by third parties (Silo mode, custom instances), increasing the surface area for misconfigured deployments.

### Recommendation
Add a predecessor check inside `new()` so that only the contract's own account (i.e., a self-call from a deploy batch) or a pre-agreed deployer account can initialize the state:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // Enforce that only the contract itself (via a deploy batch) may initialize
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    // ... rest of initialization
}
```

Alternatively, document and enforce that deployment must always bundle the `DeployContract` and `new()` call in a single NEAR batch action, and add an integration test that verifies the function reverts when called by an external account.

### Proof of Concept

1. Attacker monitors the NEAR network for a `DeployContract` action targeting a new Aurora Engine account (e.g., `aurora.example.near`).
2. Before the deployer's `new()` transaction is included, attacker submits:
   ```
   near call aurora.example.near new \
     '{"owner_id":"attacker.near","chain_id":[...], ...}' \
     --accountId attacker.near
   ```
3. If attacker's transaction is ordered first, `state::get_state` returns `Err` (no state yet), the guard passes, and `EngineState { owner_id: "attacker.near", ... }` is written.
4. The legitimate deployer's `new()` call now returns `ERR_ALREADY_INITIALIZED` and fails.
5. Attacker calls `attach_full_access_key` or `upgrade` to take full control of the engine account and drain all bridged ETH and ERC-20 tokens. [1](#0-0) [2](#0-1)

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
}
