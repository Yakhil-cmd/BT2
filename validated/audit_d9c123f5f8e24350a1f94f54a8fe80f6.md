### Title
Gateway Stateful Validator Uses Stale Current-Block L2 Gas Price Instead of `next_l2_gas_price`, Causing Invalid Transaction Admission and Valid Transaction Rejection — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's `validate_resource_bounds` function compares a transaction's `max_price_per_unit` against the **current committed block's** L2 gas price, but the transaction will actually be executed in the **next block**, which uses a different `next_l2_gas_price` computed by the EIP-1559 fee market. The code itself acknowledges this with a `TODO` comment. This stale reference creates a systematic mismatch: when the network is heavily loaded (next block price rises), transactions are admitted to the mempool that will be rejected by the blockifier at execution time; when the network is lightly loaded (next block price falls), valid transactions are rejected at the gateway before they ever reach the batcher.

The threshold is then computed as:

```rust

The value read is `strk_gas_prices.l2_gas_price` — the **current block's** L2 gas price. The correct value to use is `next_l2_gas_price`, which is stored separately in the block header and computed by `calculate_next_l2_gas_price` / `calculate_next_base_gas_price` using EIP-1559 logic based on actual gas consumed:

```rust
pub struct StorageBlockHeader {
    pub l2_gas_price: GasPricePerToken,   // current block price
    pub next_l2_gas_price: GasPrice,      // price for the NEXT block
    ...
}
``` [3](#0-2) 

The EIP-1559 update is:

```rust
let price_change = (price_u256 * gas_delta) / denominator;
let adjusted_price_u256 =
    if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };
``` [4](#0-3) 

The blockifier's pre-validation at execution time checks against the **actual next-block price**:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [5](#0-4) 

This creates two disjoint failure modes depending on the direction of price movement between blocks.

---

### Impact Explanation

**False admission (invalid tx accepted):** When the current block is heavily loaded, `next_l2_gas_price > current_l2_gas_price`. A transaction with `max_price_per_unit = X` satisfying `threshold(current) ≤ X < next_l2_gas_price` passes the gateway check but is rejected by the blockifier's `check_fee_bounds` at execution time. The transaction enters the mempool, consumes mempool capacity, and is silently dropped without executing — the user's nonce is not bumped and no fee is charged, but the transaction is lost.

**False rejection (valid tx rejected):** When the current block is lightly loaded, `next_l2_gas_price < current_l2_gas_price`. A transaction with `max_price_per_unit = X` satisfying `threshold(next) ≤ X < threshold(current)` would succeed at execution time but is rejected at the gateway. A legitimate user is denied admission for a transaction that would have been valid.

Both outcomes match the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price by up to `gas_delta / (gas_target × gas_price_max_change_denominator)` per block. Under sustained high load (e.g., `gas_used = max_block_size`), the price rises each block. The gap between `current_l2_gas_price` and `next_l2_gas_price` grows proportionally to load. With `min_gas_price_percentage < 100` (the threshold is a fraction of the current price), the false-admission window widens further. The condition is reachable by any unprivileged user submitting a standard V3 `AllResources` transaction during a period of changing network load.

---

### Recommendation

Replace the stale `l2_gas_price` read with `next_l2_gas_price` from the block header, as the TODO comment already acknowledges:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
// Fix: read next_l2_gas_price instead of l2_gas_price
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .next_l2_gas_price;  // use the EIP-1559-computed next price
```

This ensures the gateway's admission decision is consistent with the price the blockifier will enforce at execution time.

---

### Proof of Concept

1. Observe current block's L2 gas price `P` and compute `next_l2_gas_price P'` (available from the block header or by simulating `calculate_next_base_gas_price` with current gas usage).
2. Assume `min_gas_price_percentage = 100`, so gateway threshold = `P`.
3. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = P` (exactly at threshold) during a block where `gas_used > gas_target`, so `P' > P`.
4. **Gateway admits the transaction** (passes `validate_tx_l2_gas_price_within_threshold` since `P >= P`).
5. Batcher picks up the transaction and passes it to the blockifier for the next block.
6. Blockifier's `check_fee_bounds` evaluates `P < P'` → emits `ResourceBoundsError::MaxGasPriceTooLow` → transaction fails pre-validation.
7. Transaction is dropped; user must resubmit. Repeat at scale to exhaust mempool capacity with transactions that will never execute. [6](#0-5) [7](#0-6)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
