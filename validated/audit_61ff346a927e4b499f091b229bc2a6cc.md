### Title
Pool Admin Bypasses Admin Fee Cap via Uncapped Bin Additional Fees — (`File: metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`setPoolBinAdditionalFees` imposes no cap on `addFeeBuyE6` / `addFeeSellE6`, while the parallel `setPoolAdminFees` path enforces `maxAdminSpreadFeeE6`. A semi-trusted pool admin can set per-bin additional fees up to `uint16.max = 65 535` (≈ 6.55 % in E6 units) on every bin, silently stacking them on top of the already-capped spread fee and causing traders to pay more than the protocol's hard-cap permits.

---

### Finding Description

The factory enforces a hard cap on admin-controlled spread fees through `maxAdminSpreadFeeE6` (itself bounded by `HARD_MAX_SPREAD_FEE_E6 = 200 000`, i.e. 20 %). `setPoolAdminFees` checks this cap before writing:

```solidity
// MetricOmmPoolFactory.sol lines 414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees`, callable by the same pool admin, performs no analogous check:

```solidity
// MetricOmmPoolFactory.sol lines 450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external
    override
    nonReentrant
    onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

`setBinAdditionalFees` in the pool also performs no cap check — it writes the raw values directly:

```solidity
// MetricOmmPool.sol lines 464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    emit BinAdditionalFeesUpdated(bin, addFeeBuyE6, addFeeSellE6);
}
``` [3](#0-2) 

During a swap, the bin additional fee is added directly to `baseFeeX64` when computing the effective buy/sell fee for that bin:

```solidity
// MetricOmmPool.sol lines 910, 999
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [4](#0-3) 

The `spreadFeeE6` stored in the pool already encodes `protocolSpreadFeeE6 + adminSpreadFeeE6` (both capped at 20 % each, so up to 40 % total). Adding `addFeeBuyE6 = uint16.max = 65 535` (6.55 %) on top of that produces an effective per-bin fee of up to **46.55 %**, far exceeding the 20 % hard cap the factory owner intended to enforce. [5](#0-4) 

---

### Impact Explanation

Every swap routed through an affected bin pays `addFeeBuyE6` or `addFeeSellE6` on top of the already-capped spread fee. The excess fee accrues as surplus inside the pool (it is not tracked in `notionalFeeToken{0,1}Scaled`) and is collected by the pool admin via `collectPoolFees` as spread-fee surplus. Traders receive less output than the protocol's hard cap permits — a direct loss of user principal on every swap through that bin.

---

### Likelihood Explanation

The pool admin is explicitly semi-trusted: the factory owner constrains them via `maxAdminSpreadFeeE6` and `maxAdminNotionalFeeE8`. A pool admin who wishes to extract more value than those caps allow can do so immediately and without any timelock by calling `setPoolBinAdditionalFees` on every bin. No privileged escalation beyond the pool-admin role is required.

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` mirroring the one in `setPoolAdminFees`. The simplest fix is to enforce that `addFeeBuyE6` and `addFeeSellE6` do not exceed `maxAdminSpreadFeeE6` (or a dedicated `maxAdminBinAdditionalFeeE6` if the protocol intends a separate limit):

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

---

### Proof of Concept

1. Factory owner deploys factory and sets `maxAdminSpreadFeeE6 = 10_000` (1 %).
2. Pool admin calls `setPoolAdminFees(pool, 10_001, 0)` → reverts with `AdminFeeTooHigh`. Cap is enforced.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` → succeeds. No revert.
4. A trader swaps through bin 0. The effective buy fee is `spreadFeeE6 + 65_535`, far exceeding the 1 % cap the factory owner set.
5. The excess fee surplus is collected by the pool admin via `collectPoolFees`, extracting value beyond the intended cap at the trader's expense. [6](#0-5) [2](#0-1)
