### Title
Gateway `validate_resource_bounds` checks only `l2_gas.max_price_per_unit` against `min_gas_price`, admitting transactions with zero L1/L1-data gas price - (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces the configured `min_gas_price` floor exclusively against `l2_gas.max_price_per_unit`. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields are never compared to this floor. The stateful validator's `validate_tx_l2_gas_price_within_threshold` has the same scope restriction, acknowledged by an explicit `// TODO(Arni): Consider running this validation for all gas prices.` comment. Any unprivileged user can craft a V3 `AllResources` transaction that satisfies both gateway validators while carrying `l1_gas.max_price_per_unit = 0`, bypassing the spam-admission filter for L1 and L1-data gas dimensions.

---

### Finding Description

**Stateless validator — incomplete price floor check**

`validate_resource_bounds` in `stateless_transaction_validator.rs` performs three checks when `validate_resource_bounds` is enabled:

1. `AllResources(resource_bounds).max_possible_fee(Tip::ZERO) != Fee(0)` — total fee non-zero.
2. `resource_bounds.l2_gas.max_price_per_unit >= config.min_gas_price` — L2 gas price floor.
3. `resource_bounds.l2_gas.max_amount <= config.max_l2_gas_amount` — L2 gas amount cap (non-Declare only). [1](#0-0) 

Neither `l1_gas.max_price_per_unit` nor `l1_data_gas.max_price_per_unit` is compared to `min_gas_price`. The config field is documented as **"Minimum gas price for transactions"** (not "Minimum L2 gas price"), implying it should govern all three resource dimensions. [2](#0-1) 

Production default: `min_gas_price = 8_000_000_000`. [3](#0-2) 

**Stateful validator — same L2-only scope**

`validate_tx_l2_gas_price_within_threshold` reads only `tx_resource_bounds.l2_gas.max_price_per_unit` and compares it to a percentage of the previous block's L2 gas price. The function carries an explicit developer acknowledgement of the gap: [4](#0-3) 

**Downstream blockifier guard does not substitute for gateway admission**

`AccountTransaction::perform_pre_validation_stage` → `check_fee_bounds` does check all three prices against the block's actual gas prices: [5](#0-4) 

However, this check fires only after the transaction has already been admitted to the mempool and included in a block. The gateway's role is to reject such transactions *before* they consume sequencer resources.

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

A transaction with `l1_gas.max_price_per_unit = 0` (below the 8 GWei floor) and `l2_gas.max_price_per_unit ≥ min_gas_price` passes both the stateless and stateful gateway validators and enters the mempool. The spam-admission invariant — "every admitted transaction must be willing to pay at least `min_gas_price` for every resource it declares" — is broken for the L1 and L1-data gas dimensions. The blockifier will later reject such transactions with `MaxGasPriceTooLow`, but only after they have occupied mempool slots and been included in a block proposal, wasting sequencer resources and polluting block space.

---

### Likelihood Explanation

High. The attack requires no privilege, no special account state, and no on-chain setup. Any caller of the HTTP gateway can submit a well-formed V3 `AllResources` invoke transaction with `l1_gas.max_price_per_unit = 0` and a valid L2 gas price. The stateless validator is purely local and synchronous; the stateful validator only fetches the previous block's L2 gas price. Neither path touches L1 gas price.

---

### Recommendation

1. In `validate_resource_bounds` (`stateless_transaction_validator.rs`), extend the price-floor check to all three resource dimensions:
   ```rust
   for (price, label) in [
       (resource_bounds.l1_gas.max_price_per_unit.0, "L1"),
       (resource_bounds.l1_data_gas.max_price_per_unit.0, "L1 data"),
       (resource_bounds.l2_gas.max_price_per_unit.0, "L2"),
   ] {
       if price < self.config.min_gas_price {
           return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
       }
   }
   ```

2. In `validate_tx_l2_gas_price_within_threshold` (`stateful_transaction_validator.rs`), resolve the TODO by also comparing `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against their respective previous-block prices scaled by `min_gas_price_percentage`.

---

### Proof of Concept

Submit the following V3 invoke transaction to the gateway (with `validate_resource_bounds: true`, `min_gas_price: 8_000_000_000`):

```json
{
  "version": "0x3",
  "resource_bounds": {
    "L1_GAS":      { "max_amount": "0x3E8", "max_price_per_unit": "0x0" },
    "L1_DATA_GAS": { "max_amount": "0x3E8", "max_price_per_unit": "0x0" },
    "L2_GAS":      { "max_amount": "0x1000", "max_price_per_unit": "0x1DCD65000" }
  },
  ...
}
```

**Step 1 — Stateless validator passes**: `l2_gas.max_price_per_unit = 8_000_000_000 >= min_gas_price`; total fee > 0 (from L2 gas). `l1_gas.max_price_per_unit = 0` is never compared to `min_gas_price`. [6](#0-5) 

**Step 2 — Stateful validator passes**: `validate_tx_l2_gas_price_within_threshold` reads only `l2_gas.max_price_per_unit`; L1 gas price is not evaluated. [7](#0-6) 

**Step 3 — Transaction admitted to mempool.**

**Step 4 — Blockifier pre-validation fails**: `check_fee_bounds` compares `l1_gas.max_price_per_unit = 0` against the block's actual L1 gas price (> 0) and returns `MaxGasPriceTooLow`. The transaction is included in the block but fails, wasting block space and sequencer execution resources. [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
