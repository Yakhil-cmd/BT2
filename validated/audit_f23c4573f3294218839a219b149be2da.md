### Title
Pool Contract `claim_rewards` and `change_reward_address` Bypass Staking Contract Pause Mechanism - (File: `src/pool/pool.cairo`)

### Summary
The staking contract has a pause mechanism that explicitly blocks all state-changing functions including `claim_rewards`. However, the pool contract (`src/pool/pool.cairo`) is a fully independent contract with **no pause check whatsoever**. Pool members can call `claim_rewards` and `change_reward_address` on the pool contract even when the staking contract is paused, draining unclaimed yield that the pause was intended to freeze.

### Finding Description
The staking contract implements a pause flag and enforces it on every state-mutating entrypoint. The pause test suite at `src/staking/tests/pause_test.cairo` confirms that `claim_rewards` on the staking contract panics with `"Contract is paused"` when the flag is set. [1](#0-0) 

The pool contract, however, contains zero pause-related logic. A `grep` over `src/pool/**` for `pause`, `is_paused`, or `CONTRACT_IS_PAUSED` returns no matches. The two pool functions that do **not** call any staking-contract entrypoint — and therefore receive no indirect pause protection — are:

**`claim_rewards`** (lines 335–377 of `src/pool/pool.cairo`):
- Reads the pool's internal `cumulative_rewards_trace` and `pool_member_epoch_balance`.
- Calls only `get_current_epoch` on the staking contract, which is a **read-only** view function not gated by the pause.
- Transfers STRK tokens directly via `reward_token.checked_transfer(...)`.
- Updates `pool_member_info` in pool storage. [2](#0-1) 

**`change_reward_address`** (lines 505–526 of `src/pool/pool.cairo`):
- Makes **no call** to the staking contract at all.
- Writes the new reward address directly into pool storage. [3](#0-2) 

The staking contract's `IStakingPause` interface documents the intent clearly:

> "Pausing the staking contract prevents any state changes (balance changes, staker settings, etc.)" [4](#0-3) 

Yet the pool contract — which manages pool member balances, reward addresses, and reward disbursements — is entirely outside this protection.

### Impact Explanation
When the security agent pauses the staking contract in response to a discovered vulnerability, pool members can still:

1. Call `pool.change_reward_address(attacker_address)` to redirect future reward payments.
2. Call `pool.claim_rewards(pool_member)` to immediately transfer all accrued STRK rewards to the (now attacker-controlled) reward address.

This drains unclaimed yield that the pause was specifically designed to freeze. The staking contract's own `claim_rewards` is blocked, but the pool contract's equivalent is not, creating an asymmetric and exploitable gap. **Impact: Theft of unclaimed yield.**

### Likelihood Explanation
High. Any pool member can execute this two-step sequence at any time after the staking contract is paused. No special privileges, no leaked keys, and no external dependencies are required — only a valid pool membership. The pool contract is publicly callable.

### Recommendation
Add a pause check to the pool contract's state-mutating functions. The pool contract already holds a reference to the staking contract via `staking_pool_dispatcher`. It should call `IStakingDispatcher { contract_address: staking_contract }.is_paused()` and assert `false` at the top of `claim_rewards`, `change_reward_address`, `enter_delegation_pool`, `add_to_delegation_pool`, `exit_delegation_pool_intent`, and `switch_delegation_pool`. [5](#0-4) 

Alternatively, introduce a dedicated `IPoolPause` interface on the pool contract (mirroring `IStakingPause`) and have the security agent pause both contracts atomically, or have the pool contract read the staking contract's pause state on every write path.

### Proof of Concept

```
1. Pool member delegates STRK to a staker via pool.enter_delegation_pool(...).
   → Rewards accrue over several epochs.

2. Security agent calls staking.pause().
   → staking.is_paused() == true.
   → staking.claim_rewards(staker_address) now panics: "Contract is paused".

3. Pool member calls pool.change_reward_address(attacker_address).
   → No pause check in pool contract.
   → Succeeds. Reward address is now attacker_address.

4. Pool member (or anyone) calls pool.claim_rewards(pool_member).
   → No pause check in pool contract.
   → get_current_epoch() is a read-only call, not blocked.
   → calculate_rewards() runs against internal pool state.
   → reward_token.checked_transfer(recipient: attacker_address, amount: rewards) executes.
   → All accrued STRK rewards are transferred to attacker_address.

5. The staking contract's pause has been fully bypassed for pool reward claims.
``` [6](#0-5) [7](#0-6)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

    #[abi(embed_v0)]
    impl StakingPauseImpl of IStakingPause<ContractState> {
        fn pause(ref self: ContractState) {
            self.roles.only_security_agent();
            if self.is_paused() {
                return;
            }
            self.is_paused.write(true);
            self.emit(PauseEvents::Paused { account: get_caller_address() });
        }

        fn unpause(ref self: ContractState) {
            self.roles.only_security_admin();
            if !self.is_paused() {
                return;
            }
            self.is_paused.write(false);
            self.emit(PauseEvents::Unpaused { account: get_caller_address() });
        }
    }
