### Title
Mnemonic Displayed in Plaintext in Paste Dialog Due to `multiline` + `type="password"` Incompatibility — (`packages/wallets/src/components/PasteMnemonic.tsx`)

---

### Summary

The `MnemonicPaste` dialog renders the mnemonic input field with both `multiline` and `type="password"`. MUI's `TextField` with `multiline` renders a `<textarea>` element, and the HTML specification does not define a `type` attribute for `<textarea>` — all browsers including Electron's Chromium silently ignore it. The mnemonic is therefore displayed in plaintext, fully visible on screen, despite the developer's intent to mask it.

---

### Finding Description

In `packages/wallets/src/components/PasteMnemonic.tsx`, the `mnemonicList` field is declared as:

```tsx
<TextField
  autoFocus
  multiline        // ← forces <textarea> rendering
  rows={5}
  name="mnemonicList"
  variant="filled"
  type="password"  // ← ignored on <textarea>
  fullWidth
/>
``` [1](#0-0) 

The `@chia-network/core` `TextField` wrapper passes all props through `{...rest}` directly to MUI's `MaterialTextField` with no interception or correction of the `multiline` + `type` conflict: [2](#0-1) 

When MUI receives `multiline={true}`, it renders a `<textarea>` instead of `<input>`. The HTML specification defines `type` only for `<input>` elements; `<textarea>` has no `type` attribute and browsers ignore it entirely. The result is a 5-row plaintext textarea containing the full mnemonic phrase.

---

### Impact Explanation

**Critical — Secret exposure.** The mnemonic is the BIP-39 master secret from which all wallet keys are derived. Displaying it in plaintext during the import workflow means any observer with line-of-sight to the screen (shoulder-surfer, security camera, screen-sharing session, video call) can read and record the complete mnemonic, enabling full wallet takeover with no further privileges required.

---

### Likelihood Explanation

The "Paste Mnemonic" button is a prominent, first-class UI action on the wallet import screen. Users who prefer to paste rather than type word-by-word are likely to use it in contexts where the screen may be visible to others (offices, cafes, video calls). The user has no indication the masking is non-functional — the field visually resembles a password input in MUI's filled variant.

---

### Recommendation

Remove `multiline` and use a single-line `<input type="password">` for the mnemonic paste field, or — if multi-line display is required — implement masking via CSS (`-webkit-text-security: disc`) applied through `inputProps.style` or `sx`, which works on `<textarea>` elements in Chromium/Electron. Do not rely on `type="password"` when `multiline` is set.

---

### Proof of Concept

1. Launch the Chia GUI in Electron.
2. Navigate to **Add Wallet → Import Existing**.
3. Click **Paste Mnemonic**.
4. Open DevTools (`Ctrl+Shift+I`) and inspect the dialog's input element.
5. Observe: the rendered element is `<textarea>` (not `<input type="password">`), and any text typed or pasted into it is visible in plaintext in both the DOM and on screen.
