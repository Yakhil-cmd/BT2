### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary
The factory enforces `maxAdminSpreadFeeE6` on the base admin spread fee through `setPoolAdminFees`, but `setPoolBinAdditionalFees` forwards `addFeeBuyE6`/`addFeeSellE6` to the pool with **no cap check**, allowing the pool admin to impose per-bin fees up to `uint16` max (65 535 = 6.5535%) on any bin regardless of the configured cap.

### Finding Description
`setPoolAdminFees` correctly enforces the cap:

```solidity
// MetricOmmPoolFactory.sol lines 414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

But `setPoolBinAdditionalFees` passes the values straight through with no validation:

```solidity
// MetricOmmPoolFactory.sol lines 450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` also performs no cap check — it only validates the bin index:

```solidity
// MetricOmmPool.sol lines 464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
  if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
  BinState storage s = _binStates[bin];
  s.addFeeBuyE6 = addFeeBuyE6;
  s.addFeeSellE6 = addFeeSellE6;
  emit BinAdditionalFeesUpdated(bin, addFeeBuyE6, addFeeSellE6);
}
``` [3](#0-2) 

These per-bin fees are then added on top of the base fee in every swap direction:

```solidity
// MetricOmmPool.sol line 910 (buy token0 specified-out)
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
// MetricOmmPool.sol line 999 (buy token0 specified-in)
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
// MetricOmmPool.sol line 1088 (buy token1 specified-out)
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)
// MetricOmmPool.sol line 1177 (buy token1 specified-in)
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)
``` [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) 

The hard cap constants are:

```solidity
uint24 internal constant HARD_MAX_SPREAD_FEE_E6 = 200_000;  // 20%
uint24 internal constant HARD_MAX_NOTIONAL_FEE_E8 = 1_000_000;
``` [8](#0-7) 

`maxAdminSpreadFeeE6` is owner-configurable down to 0 via `setFeeCaps`. A `uint16` bin additional fee can reach 65 535 (6.5535%), which is entirely unconstrained by whatever `maxAdminSpreadFeeE6` the owner has set.

### Impact Explanation
Traders pay fees that exceed the protocol's intended admin cap. Every swap routed through a bin with inflated `addFeeBuyE6`/`addFeeSellE6` overpays by the uncapped delta. This is a direct, per-swap loss of user principal with no recovery path. The pool admin can apply this to every bin simultaneously (one call per bin index in range `[LOWEST_BIN, HIGHEST_BIN]`) without any timelock.

### Likelihood Explanation
The pool admin is a semi-trusted role that is explicitly constrained to operate "inside caps." The `setPoolBinAdditionalFees` path is callable at any time with no timelock, no prior notice, and no protocol co-signature. The admin can silently raise per-bin fees to `uint16` max immediately after pool creation or at any later point.

### Recommendation
Add the same cap guard in `setPoolBinAdditionalFees` that `setPoolAdminFees` already enforces:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce a combined cap: `baseAdminSpreadFee + addFeeBuyE6 ≤ maxAdminSpreadFeeE6`.

### Proof of Concept

1. Protocol owner calls `setFeeCaps(5_000, 1_000, ...)` — setting `maxAdminSpreadFeeE6 = 1_000` (0.1%).
2. Pool admin attempts `setPoolAdminFees(pool, 1_001, 0)` → reverts `AdminFeeTooHigh`. Cap is enforced.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` → **succeeds**. No cap check.
4. `_binStates[0].addFeeBuyE6 = 65_535` and `addFeeSellE6 = 65_535` are now stored on the pool.
5. Any trader swapping through bin 0 pays `baseFeeX64 + mulDiv(65_535, ONE_X64, 1e6)` — an additional 6.5535% fee — far above the 0.1% cap the protocol intended.
6. The admin can repeat for every bin in `[LOWEST_BIN, HIGHEST_BIN]` with no timelock. [9](#0-8)
