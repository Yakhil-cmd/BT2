### Title
Gateway Stateful Validator Uses Current Block's L2 Gas Price Instead of `next_l2_gas_price` for Admission Threshold — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful `validate_resource_bounds` reads `block_info.gas_prices.strk_gas_prices.l2_gas_price` (the **current** committed block's L2 gas price) to compute the admission threshold, but the transaction will be executed in the **next** block whose L2 gas price is `block_header.next_l2_gas_price`. Because the EIP-1559 fee market adjusts the price every block, the two values diverge whenever gas usage is above or below the target. The developer acknowledged the wrong variable with an inline TODO comment. The result is a systematic mismatch between the price the gateway enforces and the price the batcher actually uses, causing the gateway to either accept transactions that will fail at execution or reject transactions that would succeed.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, `validate_resource_bounds` (lines 223–243) reads the latest committed block's info and extracts `l2_gas_price`:

```rust

``` [1](#0-0) 

`validate_tx_l2_gas_price_within_threshold` then computes:

```
threshold = (min_gas_price_percentage / 100) * previous_block_l2_gas_price
```

and rejects the transaction if `tx.max_price_per_unit < threshold`. [2](#0-1) 

The correct reference price is `block_header.next_l2_gas_price`, which is the EIP-1559-derived price that the consensus orchestrator will use for the next block. This field is computed by `calculate_next_l2_gas_price_for_fin` / `calculate_next_base_gas_price` and stored in the block header: [3](#0-2) 

The `run_validate_entry_point` path also uses the current block's gas prices (only the block number is incremented), so the blockifier pre-validation inside the gateway is equally misaligned: [4](#0-3) 

The batcher, however, receives the actual next-block gas prices from the consensus orchestrator and passes them to the blockifier's `check_fee_bounds`: [5](#0-4) 

### Impact Explanation

**When `next_l2_gas_price > current_l2_gas_price` (price rising, e.g., high gas usage):**
- The gateway threshold is too low.
- A transaction with `max_price_per_unit` in the range `[threshold(current), threshold(next))` passes gateway admission and enters the mempool.
- At execution time the batcher's `check_fee_bounds` uses the higher `next_l2_gas_price` and rejects the transaction with `MaxGasPriceTooLow`, causing a revert and charging the user a revert fee.
- **Gateway admits transactions that will deterministically fail at execution.**

**When `next_l2_gas_price < current_l2_gas_price` (price falling, e.g., low gas usage):**
- The gateway threshold is too high.
- A transaction with `max_price_per_unit` in the range `[threshold(next), threshold(current))` is rejected by the gateway even though it would succeed at execution.
- **Gateway rejects valid transactions before sequencing.**

Both branches match the "High" impact criterion: *Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.*

### Likelihood Explanation

The L2 gas price changes every block via the EIP-1559 mechanism. The maximum per-block change is `price / gas_price_max_change_denominator`. Under sustained high or low load the divergence between `l2_gas_price` and `next_l2_gas_price` grows monotonically. Any user submitting a transaction with `max_price_per_unit` near the threshold boundary is affected without any special attacker capability. The TODO comment in the source confirms the developers are aware the wrong field is being read.

### Recommendation

Replace the read of `block_info.gas_prices.strk_gas_prices.l2_gas_price` with a read of `block_header.next_l2_gas_price` from the latest committed block header. The `GatewayFixedBlockStateReader` trait (or the underlying storage reader) should expose `next_l2_gas_price` so that `validate_resource_bounds` can compute the threshold against the price that will actually govern the next block's execution.

### Proof of Concept

1. Observe the current committed block N has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block N was above gas target).
2. Submit an `InvokeV3` transaction with `resource_bounds.l2_gas.max_price_per_unit = P * min_gas_price_percentage / 100` (exactly at the current threshold).
3. Gateway `validate_resource_bounds` computes `threshold = P * min_gas_price_percentage / 100` and accepts the transaction.
4. Blockifier validation inside the gateway also uses `P` (same block info), so `check_fee_bounds` passes.
5. Transaction enters the mempool.
6. Batcher builds block N+1 with `l2_gas_price = P'`. `check_fee_bounds` computes `threshold = P' * min_gas_price_percentage / 100 > tx.max_price_per_unit` and reverts the transaction.
7. User is charged the revert fee for a transaction the gateway guaranteed would be admitted. [1](#0-0) [6](#0-5)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
