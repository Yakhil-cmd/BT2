### Title
Staker Can Redirect Accumulated Unclaimed Rewards Away From Old Reward Address Without Prior Claim - (File: src/staking/staking.cairo)

### Summary
`change_reward_address` in both the staking contract and the pool contract updates the reward destination without first claiming accumulated unclaimed rewards to the old address. Because `claim_rewards` always sends to the **current** `reward_address` stored in state, any rewards that accrued while the old address was registered are silently redirected to the new address. The old reward address simultaneously loses its authorization to call `claim_rewards`, making the loss permanent.

### Finding Description
`change_reward_address` in `staking.cairo` (lines 517–540) performs a bare storage write of the new reward address with no prior reward flush:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    ...
    staker_info.reward_address = reward_address;   // ← no claim of unclaimed_rewards_own first
    self.write_staker_info(:staker_address, :staker_info);
    ...
}
``` [1](#0-0) 

`send_rewards_to_staker`, called by every reward-claiming path, reads `reward_address` from the **current** staker info and transfers there:

```cairo
fn send_rewards_to_staker(...) {
    let reward_address = staker_info.reward_address;   // always the new address after the change
    ...
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
    staker_info.unclaimed_rewards_own = Zero::zero();
    ...
}
``` [2](#0-1) 

`claim_rewards` enforces that only the staker address **or** the current reward address may call it:

```cairo
assert!(
    caller_address == staker_address || caller_address == reward_address,
    "{}",
    Error::CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
);
``` [3](#0-2) 

Once `change_reward_address` executes, the old reward address is no longer `staker_info.reward_address`, so it is permanently locked out of `claim_rewards`. All previously accumulated `unclaimed_rewards_own` are then claimed to the new address.

The identical pattern exists in the pool contract: [4](#0-3) 

Pool `claim_rewards` likewise transfers to the current `reward_address`: [5](#0-4) 

### Impact Explanation
The protocol explicitly grants the reward address the right to call `claim_rewards` independently of the staker, establishing it as a legitimate, independent beneficiary of accrued yield. A staker who has designated a third-party reward address (e.g., a yield aggregator, DAO treasury, or separate custody wallet) can atomically:

1. Call `change_reward_address(attacker_address)` after rewards have accumulated.
2. Call `claim_rewards(staker_address)` — all accumulated yield is sent to `attacker_address`.

The old reward address is permanently barred from recovering those rewards. This constitutes **theft of unclaimed yield** (High impact).

### Likelihood Explanation
Any staker or pool member who has set a reward address different from their staker/pool-member address is exposed. This is a common and explicitly supported configuration (the protocol allows the reward address to independently trigger claims). The attack requires no special privilege — only the staker key, which the staker already holds. It can be executed in a single transaction with no preconditions beyond having accumulated rewards.

### Recommendation
Before updating `reward_address`, flush any accumulated rewards to the **old** address:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    ...
    let staker_address = get_caller_address();
    let mut staker_info = self.internal_staker_info(:staker_address);

    // Flush accumulated rewards to the OLD reward address first.
    if staker_info.unclaimed_rewards_own.is_non_zero() {
        let token_dispatcher = strk_token_dispatcher();
        self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
    }

    let old_address = staker_info.reward_address;
    staker_info.reward_address = reward_address;
    self.write_staker_info(:staker_address, :staker_info);
    ...
}
```

Apply the same fix to `pool.cairo`'s `change_reward_address`.

### Proof of Concept
1. Staker `S` stakes and sets `reward_address = A` (a third-party yield aggregator).
2. Attestations occur; `unclaimed_rewards_own` accumulates to `R` STRK.
3. `S` calls `change_reward_address(S_own_wallet)` — no rewards are claimed; `staker_info.reward_address` is now `S_own_wallet`.
4. Address `A` attempts `claim_rewards(S)` → reverts with `CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS` because `A` is no longer the registered reward address.
5. `S` calls `claim_rewards(S)` → `R` STRK is transferred to `S_own_wallet`.
6. `A` permanently loses `R` STRK of yield it was entitled to receive.

        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                !self.does_token_exist(token_address: reward_address),
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let old_address = staker_info.reward_address;

            // Update reward_address and commit to storage.
            staker_info.reward_address = reward_address;
            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.
            self
                .emit(
                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,
                    },
                );
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
