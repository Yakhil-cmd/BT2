Audit Report

## Title
Any Deployed SNS Swap Canister Can Settle Another Swap's Neurons' Fund Participation With a False Result - (File: rs/nns/governance/src/governance.rs)

## Summary
`settle_neurons_fund_participation` in NNS governance authorizes callers by checking only whether the caller is *any* deployed SNS swap canister, not whether it is the swap canister specifically associated with the target proposal. The code's own comment acknowledges this gap: *"Note that a Swap could settle each other's participation."* An attacker controlling a legitimately deployed SNS swap canister can call this function with a victim SNS's `nns_proposal_id` and `result = Aborted`, permanently poisoning the victim swap's Neurons' Fund settlement before the victim's own `finalize` call executes.

## Finding Description
In `settle_neurons_fund_participation`, the authorization check at lines 7018–7038 converts the caller to a `CanisterId` and passes it to `is_canister_id_valid_swap_canister_id`: [1](#0-0) 

`is_canister_id_valid_swap_canister_id` (lines 8191–8226) queries SNS-W's `list_deployed_snses` and checks whether the caller appears as *any* swap canister in the global list: [2](#0-1) 

There is no check binding the caller to the specific proposal being settled. `ProposalData` does not store the swap canister ID, and no such comparison is performed anywhere in the function. [3](#0-2) 

The state machine guard at lines 7112–7186 does **not** block the attack. On the first call for a proposal (before any prior settlement), `previously_computed_neurons_fund_refunds` and `previously_computed_final_neurons_fund_participation` are both `None` and `initial_neurons_fund_participation` is `Some`, which falls into "Ok case III" (line 7180) and proceeds unconditionally: [4](#0-3) 

After Swap A's malicious `Aborted` call completes, the lifecycle is set to `Aborted` (terminal) at line 7235, `final_neurons_fund_participation` is set to an empty snapshot (zero NF participation), and `neurons_fund_refunds` is set. When Swap B subsequently calls with `Committed`, it hits "Ok case I" (line 7166) and returns the previously computed empty snapshot without minting any ICP: [5](#0-4) 

`Lifecycle::is_terminal()` confirms that both `Committed` and `Aborted` are terminal, so the lifecycle lock is permanent once set: [6](#0-5) 

## Impact Explanation
The victim SNS-B loses its entire Neurons' Fund ICP contribution. The Neurons' Fund neurons have their reserved maturity refunded (no loss to them), but the SNS treasury never receives the matched ICP it was entitled to. Swap B's `finalize` call either fails or completes with an empty Neurons' Fund participation, producing an inconsistent state where SNS tokens may be distributed to Neurons' Fund participants but no corresponding ICP arrives in the SNS treasury. This constitutes a significant SNS-level security impact with concrete protocol harm, matching: **High ($2,000–$10,000) — Significant NNS/SNS security impact with concrete user or protocol harm.** [7](#0-6) 

## Likelihood Explanation
The attacker must first deploy a legitimate SNS through the NNS governance proposal process, which requires community approval — a high barrier. However, once any SNS is deployed, its swap canister is a persistent on-chain actor. The attack window is the period between a victim swap reaching a terminal lifecycle state (`Committed` or `Aborted`) and the victim swap canister executing `finalize`. Per the swap proto comments, `finalize` is not automatic and must be called explicitly, creating a non-trivial window: [8](#0-7) 

The attack is repeatable: a single deployed SNS can attack any number of future victim swaps.

## Recommendation
Bind the authorization to the specific proposal. The swap canister ID for a given SNS is known at deployment time (it is returned by SNS-W and stored in `list_deployed_snses`). The fix should either:

1. Store the swap canister ID in `ProposalData` at SNS deployment time and require `caller == proposal_data.swap_canister_id`, or
2. Extend `is_canister_id_valid_swap_canister_id` to also accept the `nns_proposal_id` and verify that the caller's swap canister ID matches the SNS instance associated with that specific proposal ID in SNS-W's registry.

The existing `is_canister_id_valid_swap_canister_id` already fetches the full `list_deployed_snses` response; it can be extended to cross-reference the proposal ID against the SNS instance's `nns_proposal_id` field if that is available in the SNS-W response. [9](#0-8) 

## Proof of Concept
1. Attacker submits and passes an NNS governance proposal to create SNS-A. SNS-A's swap canister (`swap_a`) is deployed and appears in `list_deployed_snses`.
2. Victim SNS-B has `nns_proposal_id = 42`, is in `Committed` state, and has Neurons' Fund participation reserved (`initial_neurons_fund_participation` is `Some`, both `previously_computed_*` fields are `None`).
3. Before SNS-B's swap canister calls `finalize`, attacker calls from `swap_a`:
   ```
   settle_neurons_fund_participation({
       nns_proposal_id: 42,
       result: Aborted {}
   })
   ```
4. NNS governance checks: is `swap_a` in `list_deployed_snses`? Yes → authorized. State machine: `(Some(_), None, None)` with non-terminal lifecycle → Ok case III → proceeds.
5. `direct_participation_icp_e8s` is set to 0 (Aborted path, line 7217). `final_neurons_fund_participation` is computed as empty. Lifecycle set to `Aborted`. All reserved Neurons' Fund maturity is refunded. `final_neurons_fund_participation` and `neurons_fund_refunds` are persisted.
6. SNS-B's swap canister calls `finalize` → `settle_neurons_fund_participation` with `Committed` → hits Ok case I → returns the previously computed empty snapshot.
7. SNS-B's finalization completes with zero Neurons' Fund ICP; the SNS treasury never receives the matched funding it was entitled to.

A deterministic integration test using PocketIC can reproduce this by deploying two SNS instances, advancing SNS-B to `Committed`, then calling `settle_neurons_fund_participation` from SNS-A's swap canister with SNS-B's proposal ID and `Aborted`, and asserting that SNS-B's subsequent `finalize` returns an empty Neurons' Fund participation. async fn is_canister_id_valid_swap_canister_id(
    target_canister_id: CanisterId,
    env: &dyn Environment,
) -> Result<(), String> {
    let list_deployed_snses_response = env
        .call_canister_method(
            SNS_WASM_CANISTER_ID,
            "list_deployed_snses",
            Encode!(&ListDeployedSnsesRequest {}).expect(""),
        )
        .await
        .map_err(|err| {
            format!(
                "Failed to call the list_deployed_snses method on sns_wasm ({SNS_WASM_CANISTER_ID}): {err:?}",
            )
        })?;

    let list_deployed_snses_response =
        Decode!(&list_deployed_snses_response, ListDeployedSnsesResponse).map_err(|err| {
            format!(
                "Unable to decode response as ListDeployedSnsesResponse: {err}. reply_bytes = {list_deployed_snses_response:#?}",
            )
        })?;

    let is_swap = list_deployed_snses_response
        .instances
        .iter()
        .any(|sns| sns.swap_canister_id == Some(target_canister_id.into()));
    if !is_swap {
        return Err(format!(
            "target_swap_canister_id is not the ID of any swap canister known to sns_wasm: {target_canister_id}"
        ));
    }

    Ok(())
}

    pub async fn settle_neurons_fund_participation(
        &mut self,
        nns_governance_client: &mut impl NnsGovernanceClient,
    ) -> SettleNeuronsFundParticipationResult {
        use settle_neurons_fund_participation_request::{Aborted, Committed};

        // Check if any work needs to be done.
        if !self.cf_participants.is_empty() {
            log!(
                INFO,
                "settle_neurons_fund_participation has already been called \
                successfully and cf_participants has been set. Returning successfully."
            );

            return SettleNeuronsFundParticipationResult::new_ok(
                self.current_neurons_fund_participation_e8s(),
                self.cf_participants.len() as u64,
            );
        }

        let init = match self.init_and_validate() {
            Ok(init) => init,
            Err(error_message) => {
                return SettleNeuronsFundParticipationResult::new_error(error_message);
            }
        };

        // Check if the Neurons' Fund participation is requested by the SNS.
        match init.neurons_fund_participation {
            // If Neurons' Fund participation is specifically requested, keep going.
            Some(neurons_fund_participation) if neurons_fund_participation => (),
            Some(_) | None => {
                log!(
                    INFO,
                    "settle_neurons_fund_participation has not been requested. \
                     Returning successfully.",
                );

                return SettleNeuronsFundParticipationResult::new_ok(
                    self.current_neurons_fund_participation_e8s(),
                    self.cf_participants.len() as u64,
                );
            }
        }

        // The following methods are safe to call since we validated Init in the above block
        let nns_proposal_id = init.nns_proposal_id();
        let sns_governance_canister_id = init.sns_governance_or_panic();

        // Build the NNS Governance request struct
        let swap_result = if self.lifecycle() == Lifecycle::Committed {
            settle_neurons_fund_participation_request::Result::Committed(Committed {
                sns_governance_canister_id: Some(sns_governance_canister_id.get()),
                total_direct_participation_icp_e8s: Some(self.current_direct_participation_e8s()),
                total_neurons_fund_participation_icp_e8s: Some(
                    self.current_neurons_fund_participation_e8s(),
                ),
            })
        } else {
            settle_neurons_fund_participation_request::Result::Aborted(Aborted {})
        };
        let request = SettleNeuronsFundParticipationRequest {
            nns_proposal_id: Some(nns_proposal_id),
            result: Some(swap_result),
        };

        // Issue the request to nns governance.
        let response: Result<SettleNeuronsFundParticipationResponse, CanisterCallError> =
            nns_governance_client
                .settle_neurons_fund_participation(request)
                .await;
