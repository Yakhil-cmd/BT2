### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Reward Distribution — (File: `src/staking/staking.cairo`)

### Summary
`IStakingRewardsManager::update_rewards` is specified to be callable only by the Starknet sequencer, but the implementation contains no such access control check. Any unprivileged caller can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` sentinel without distributing any rewards. Because the sentinel is global and checked before any reward logic, the sequencer is then blocked from distributing rewards for that block to any staker. Repeated calls at every block permanently suppress all yield accrual.

### Finding Description
The spec explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, only calls `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no sequencer role is enforced: [2](#0-1) 

The critical accounting state `last_reward_block` is written **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

This means an attacker who calls `update_rewards(valid_staker, disable_rewards: true)` at block N will:
1. Pass all precondition checks (staker exists, has non-zero balance, block is new).
2. Write `last_reward_block = N`.
3. Return immediately without distributing any rewards.

When the sequencer subsequently attempts `update_rewards(any_staker, disable_rewards: false)` at the same block N, the guard `current_block_number > last_reward_block` evaluates to `N > N = false` and the call reverts with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

The `last_reward_block` variable is a single global sentinel, not per-staker, so one attacker call per block silences reward distribution for **all** stakers and all delegation pools for that block. [5](#0-4) 

### Impact Explanation
Each suppressed block permanently destroys the yield that would have been minted and distributed for that block. Pool members' `cumulative_rewards_trace` is never updated, so the lost rewards are unrecoverable. If the attacker calls the function at every block, all staker and delegator yield is permanently frozen. This matches **High: Permanent freezing of unclaimed yield**. [6](#0-5) 

### Likelihood Explanation
The entry point is fully public — any non-zero address on Starknet can call `update_rewards`. The attacker only needs to know one valid, active staker address (trivially obtained from `NewStaker` events). The cost is one transaction per block. There is no profit motive required; a griefing attacker or a competitor staker has clear incentive to suppress rivals' rewards.

### Recommendation
Add a sequencer-only access control guard at the top of `update_rewards`, consistent with the spec. A dedicated `REWARDS_MANAGER` role (or reuse of an existing operator role) should be checked via the `RolesComponent` before any state is mutated:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_rewards_manager(); // <-- add this
    self.general_prerequisites();
    ...
```

Alternatively, move the `last_reward_block.write(current_block_number)` call to after the `disable_rewards` guard so that a call with `disable_rewards: true` does not consume the block slot.

### Proof of Concept

```
Block N:
  Attacker calls: Staking::update_rewards(staker=<any_valid_staker>, disable_rewards=true)
    → general_prerequisites() passes (contract unpaused, caller != 0)
    → N > last_reward_block passes (first call this block)
    → staker exists and is active: passes
    → last_reward_block.write(N)          ← sentinel consumed
    → disable_rewards == true → return    ← no rewards distributed

  Sequencer calls: Staking::update_rewards(staker=<intended_staker>, disable_rewards=false)
    → N > last_reward_block = N > N = false
    → PANICS: REWARDS_ALREADY_UPDATED     ← sequencer blocked

Result: All stakers and pool members receive zero rewards for block N.
        Pool's cumulative_rewards_trace is not updated.
        Yield for block N is permanently lost.

Repeat at every block → all yield permanently frozen.
``` [7](#0-6) [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
