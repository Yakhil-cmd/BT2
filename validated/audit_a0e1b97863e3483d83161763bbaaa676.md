### Title
Unbounded Loop in `calculate_rewards` Over Delegator's Balance Trace Enables Permanent Freezing of Unclaimed Yield - (File: `src/pool/pool.cairo`)

### Summary

The `calculate_rewards` function in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop that iterates over a delegator's `pool_member_epoch_balance` trace. A delegator who repeatedly changes their balance across many epochs without claiming rewards can grow this trace without bound, eventually causing `claim_rewards` to exceed the Starknet transaction gas limit and permanently revert, freezing their unclaimed yield.

### Finding Description

`calculate_rewards` (line 837) iterates over every entry in `pool_member_epoch_balance` for a given pool member since their last claim checkpoint:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The trace (`pool_member_epoch_balance`) is a `Vec`-backed structure that appends a new checkpoint whenever `set_member_balance` is called with a strictly larger epoch key than the last entry. The key used is always `current_epoch + K`:

```cairo
fn set_member_balance(ref self: ContractState, pool_member: ContractAddress, amount: Amount) {
    let trace = self.pool_member_epoch_balance.entry(pool_member);
    ...
    trace.insert(key: self.get_epoch_plus_k(), value: pool_member_balance);
}
``` [2](#0-1) 

`set_member_balance` is called by:
- `enter_delegation_pool` (line 201)
- `increase_member_balance` → called by `add_to_delegation_pool` (line 242) and `enter_delegation_pool_from_staking_contract` (line 456/464)
- `exit_delegation_pool_intent` (line 278) [3](#0-2) [4](#0-3) [5](#0-4) 

Because the epoch key advances by 1 per epoch, each epoch in which a delegator changes their balance appends exactly one new entry to the trace. The `entry_to_claim_from` cursor is only advanced when `claim_rewards` succeeds:

```cairo
pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
``` [6](#0-5) 

If a delegator makes one balance change per epoch and never calls `claim_rewards`, the trace grows by one entry per epoch indefinitely. When `claim_rewards` is eventually called, the loop must process all accumulated entries in a single transaction.

The `PoolMemberBalanceTrace` structure confirms entries are never removed — only appended: [7](#0-6) 

### Impact Explanation

Once the trace is large enough to exhaust the Starknet transaction gas limit, every call to `claim_rewards` for that pool member will revert. Since `entry_to_claim_from` is only updated on a successful `claim_rewards`, the cursor never advances, and the delegator's unclaimed yield is permanently frozen. The `pool_member_info_v1` view function also calls `calculate_rewards` and would similarly fail. [8](#0-7) 

**Impact**: Permanent freezing of unclaimed yield for the affected pool member.

### Likelihood Explanation

A delegator who makes one balance change per epoch (via `add_to_delegation_pool` or `exit_delegation_pool_intent`) and defers claiming rewards accumulates one trace entry per epoch. Given that Starknet epochs can be short (hours to days), hundreds to thousands of entries can accumulate over months. Each loop iteration performs multiple storage reads (`pool_member_trace.at(...)` and `find_sigma` reading `cumulative_rewards_trace`), making gas exhaustion reachable at a finite trace length. The code itself acknowledges the risk with the comment "This loop is unbounded but unlikely to exceed gas limits." [9](#0-8) 

The attack is self-inflicted (only the pool member or their reward address can call `add_to_delegation_pool`; only the pool member can call `exit_delegation_pool_intent`), but it is reachable through normal protocol flows with no privileged access required. [10](#0-9) 

### Recommendation

Track a separate `entry_to_claim_from` cursor that is advanced on every balance-changing call (not only on `claim_rewards`), so that the loop in `calculate_rewards` only ever processes entries since the last balance change, bounding its length to a small constant (at most K+1 entries between any two consecutive balance changes). Alternatively, enforce a maximum number of unclaimed balance-change epochs before a new balance change is allowed, requiring the delegator to claim first.

### Proof of Concept

1. Delegator calls `enter_delegation_pool` in epoch 0 → trace length = 1.
2. Each subsequent epoch N: delegator calls `add_to_delegation_pool(amount: 1)` → appends one entry at epoch N+K. Trace length grows by 1 per epoch.
3. Delegator never calls `claim_rewards`. After M epochs, trace length = M+1.
4. After sufficiently many epochs (e.g., M = 10,000), delegator calls `claim_rewards`. The loop in `calculate_rewards` must iterate over all M entries, each requiring multiple storage reads from `pool_member_epoch_balance` and `cumulative_rewards_trace`. The transaction exceeds the gas limit and reverts.
5. Because `entry_to_claim_from` is never updated (the transaction always reverts), the delegator's accumulated yield is permanently frozen. [11](#0-10) [12](#0-11)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

    /// Inserts a (`key`, `value`) pair into a Trace so that it is stored as the checkpoint.
    /// This is done by either inserting a new checkpoint, or updating the last one.
    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
