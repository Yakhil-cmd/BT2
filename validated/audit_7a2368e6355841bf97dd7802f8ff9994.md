### Title
Unrestricted `update_rewards` with `disable_rewards=true` Permanently Freezes All Consensus-Phase Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract has no access control and unconditionally advances the global `last_reward_block` sentinel even when called with `disable_rewards = true`. Because only one call per block is permitted, any unprivileged caller can invoke `update_rewards(any_active_staker, disable_rewards: true)` every block, consuming the single per-block reward slot without distributing any yield, permanently freezing all consensus-phase staker and delegator rewards.

### Finding Description
`IStakingRewardsManager::update_rewards` is the sole mechanism for distributing consensus-phase block rewards. Its access guard is only `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

Inside the function, `last_reward_block` is written to the current block number **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

The guard that enforces one call per block is: [3](#0-2) 

Because `last_reward_block` is a single global storage slot, once it is written in block N, every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`. The `disable_rewards = true` path returns immediately after writing the sentinel, distributing nothing: [4](#0-3) 

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in every block therefore occupies the single reward slot for that block without any yield being credited to `unclaimed_rewards` in the `RewardSupplier`, to `staker_info.unclaimed_rewards_own`, or to any pool's `cumulative_rewards_trace`. [5](#0-4) 

### Impact Explanation
All consensus-phase block rewards are permanently frozen. Stakers never accumulate `unclaimed_rewards_own`, and pool members never receive updates to `cumulative_rewards_trace`, so `claim_rewards` on both the staking and pool contracts returns zero indefinitely. This matches **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The attack requires only a non-zero Starknet address and the gas cost of one transaction per block. No capital, no privileged key, and no special state is needed. The attacker only needs to know any currently-active staker address (publicly readable from the `stakers` vector). The attack is sustainable as long as the attacker is willing to pay per-block gas fees, which on Starknet are low.

### Recommendation
Move the `last_reward_block` write to **after** the `disable_rewards` guard so that a no-reward call does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only advance the sentinel when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... rest of reward calculation
```

Alternatively, add an access-control check (e.g., `only_app_governor` or a dedicated consensus-contract role) to `update_rewards` so that only the authorized consensus contract can call it.

### Proof of Concept

1. Deploy the staking system in consensus-rewards mode (`consensus_rewards_first_epoch` is set and the current epoch has passed it).
2. Identify any active staker `S` with non-zero STRK balance (readable from `stakers` vec + `staker_info`).
3. In every new block, call:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Observe that `last_reward_block` advances to the current block each time.
5. Any legitimate call to `update_rewards` for any staker in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. After many blocks, call `staking_contract.claim_rewards(S)` — it returns 0. Call `pool.claim_rewards(pool_member)` — it also returns 0. No yield has accrued to any participant. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
