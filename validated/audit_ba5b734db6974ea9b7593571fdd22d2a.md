### Title
Malicious Participant Can Corrupt CKD Output by Supplying Mismatched `app_id` or `app_pk` — (File: `src/confidential_key_derivation/protocol.rs`)

---

### Summary

The CKD (Confidential Key Derivation) protocol does not cross-verify that all participants use the same `app_id` and `app_pk` before summing their shares. A malicious participant can call `ckd` with a different `app_id` (or `app_pk`) than the rest of the group, causing the coordinator to silently accept a cryptographically corrupted output that corresponds to no valid derived key for any application.

---

### Finding Description

In `do_ckd_coordinator`, the coordinator computes its own share and then receives `CKDOutput` from every other participant, summing them unconditionally: [1](#0-0) 

Each participant's share is computed in `compute_signature_share` using the `app_id` and `app_pk` that were passed **locally** to that participant's `ckd` call: [2](#0-1) 

The coordinator never broadcasts `app_id` or `app_pk` to the group, and participants never broadcast or verify each other's `app_id`/`app_pk`. The only message exchanged is the raw `(norm_big_y, norm_big_c)` pair: [3](#0-2) 

Because `compute_signature_share` hashes `app_id` together with the master public key to produce the base point `hash_point = H(pk ‖ app_id)`, a participant who substitutes `app_id'` contributes:

```
λ_j · (x_j · H(pk, app_id') + y_j · app_pk)
```

instead of the expected:

```
λ_j · (x_j · H(pk, app_id) + y_j · app_pk)
```

The coordinator sums this with the honest participants' shares and returns the result as a valid `CKDOutput`, with no error or indication of inconsistency.

---

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
