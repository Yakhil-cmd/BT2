### Title
Gateway Resource-Bounds Admission Checks Only L2 Gas Price, Admitting Transactions with Zero L1/L1-Data Gas Price That Will Fail Blockifier Pre-Validation - (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

Both the stateless and stateful gateway validators check only `l2_gas.max_price_per_unit` when validating resource bounds. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields are never checked at the admission stage. A transaction with `l1_gas.max_price_per_unit = 0` (or below the current block's L1 gas price) passes all gateway checks, enters the mempool, and is only rejected later during blockifier `perform_pre_validation_stage` — after it has already consumed mempool and batcher resources.

---

### Finding Description

**Stateless validator — only L2 gas price is checked:** [1](#0-0) 

```rust
let resource_bounds = *tx.resource_bounds();
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(...ZeroResourceBounds...);
}
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(...MaxGasPriceTooLow...);
}
```

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared against any minimum. A transaction with `l1_gas.max_price_per_unit = 0` and `l2_gas.max_price_per_unit >= min_gas_price` passes this check.

**Stateful validator — same single-resource pattern, with an explicit TODO acknowledging the gap:** [2](#0-1) 

```rust

            // Only l2_gas is checked against the previous block price.
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
}
```

``` [4](#0-3)

```

The gateway checks only L2 gas price; the blockifier checks L1 gas, L1 data gas, and L2 gas prices. The invariant "a transaction admitted by the gateway will not fail blockifier pre-validation due to insufficient resource bounds" is broken for the L1 and L1-data dimensions.

---

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An attacker submits invoke transactions with:
- `l2_gas.max_price_per_unit >= min_gas_price` (passes both gateway checks)
- `l1_gas.max_price_per_unit = 0` (never checked by gateway)

These transactions pass `StatelessTransactionValidator::validate`, pass `StatefulTransactionValidator::validate_resource_bounds`, enter the mempool, are pulled by the batcher, and are only rejected at `AccountTransaction::perform_pre_validation_stage` with `TransactionFeeError::InsufficientResourceBounds`. The mempool and batcher process transactions that are guaranteed to fail, enabling a low-cost DoS against the sequencer's admission pipeline.

---

### Likelihood Explanation

**Likelihood: High.**

The attack requires no privileged access. Any user can submit a well-formed V3 transaction with `l1_gas.max_price_per_unit = 0`. The gateway's `validate_resource_bounds` only rejects transactions where the *total* max fee is zero (all three resources at zero), so a transaction with non-zero L2 gas bounds and zero L1/L1-data gas price passes the zero-fee guard. The TODO comment in the stateful validator confirms this is a known, unaddressed gap.

---

### Recommendation

Extend `validate_resource_bounds` in both validators to check `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the current block's prices (or a configured minimum), mirroring the logic already applied to `l2_gas.max_price_per_unit`. The stateful validator already fetches `get_block_info()` and has access to `strk_gas_prices.l1_gas_price` and `strk_gas_prices.l1_data_gas_price`; those values should be used to gate admission.

---

### Proof of Concept

1. Construct a V3 invoke transaction with:
   - `resource_bounds.l2_gas.max_price_per_unit = min_gas_price` (e.g., 1)
   - `resource_bounds.l2_gas.max_amount = 1_000_000`
   - `resource_bounds.l1_gas.max_price_per_unit = 0`
   - `resource_bounds.l1_data_gas.max_price_per_unit = 0`
2. Submit via `POST /add_transaction`.
3. `StatelessTransactionValidator::validate_resource_bounds` passes: total fee > 0 (L2 gas contributes), L2 gas price ≥ `min_gas_price`. [5](#0-4) 
4. `StatefulTransactionValidator::validate_resource_bounds` passes: only L2 gas price is compared against the previous block threshold. [6](#0-5) 
5. Transaction enters the mempool and is returned by `get_txs`. [7](#0-6) 
6. Batcher executes it; `AccountTransaction::perform_pre_validation_stage` calls `check_fee_bounds`, which finds `l1_gas.max_price_per_unit (0) < actual_l1_gas_price` and returns `TransactionFeeError::InsufficientResourceBounds`. [8](#0-7) 
7. Transaction is rejected post-admission, having already consumed mempool and batcher capacity.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);

                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
    pub fn get_txs(&mut self, n_txs: usize) -> MempoolResult<Vec<InternalRpcTransaction>> {
        // All transactions are enqueued in FIFO mode.
        if !self.is_fifo() {
            self.add_ready_declares();
        }
        let mut eligible_tx_references: Vec<TransactionReference> = Vec::with_capacity(n_txs);
        let mut n_remaining_txs = n_txs;

        let mut account_nonce_updates = AddressToNonce::new();
        while n_remaining_txs > 0 && self.tx_queue.has_ready_txs() {
            let chunk = self.tx_queue.pop_ready_chunk(n_remaining_txs);

            let (valid_txs, expired_txs_updates) = self.prune_expired_nonqueued_txs(chunk);
            account_nonce_updates.extend(expired_txs_updates);

            // In FIFO mode, all transactions are already enqueued. In fee-priority mode,
            // we need to enqueue the next eligible transaction for each address.
            if !self.is_fifo() {
                self.enqueue_next_eligible_txs(&valid_txs)?;
            }

            n_remaining_txs -= valid_txs.len();
            eligible_tx_references.extend(valid_txs);
        }

        // Update the mempool state with the given transactions' nonces.
        for tx_reference in &eligible_tx_references {
            self.state.stage(tx_reference)?;
        }

        let n_returned_txs = eligible_tx_references.len();
        if n_returned_txs != 0 {
            info!("Returned {n_returned_txs} out of {n_txs} transactions, ready for sequencing.");
            debug!(
                "Returned mempool txs: {:?}",
                eligible_tx_references.iter().map(|tx| tx.tx_hash).collect::<Vec<_>>()
            );
            // Mark these txs as "taken for batching" (returned by `get_txs`). The
            // `TRANSACTION_TIME_SPENT_UNTIL_BATCHED` metric is recorded only once they actually
            // commit (to avoid counting txs that were returned by `get_txs` but ultimately not
            // included).
            let now = self.clock.now();
            for tx_ref in &eligible_tx_references {
                self.tx_pool.mark_taken_for_batching(tx_ref.tx_hash, now);
            }
        }

        metric_set_get_txs_size(n_returned_txs);
        self.update_accounts_with_gap(account_nonce_updates);
        self.update_state_metrics();

        Ok(eligible_tx_references
            .iter()
            .map(|tx_reference| {
                self.tx_pool
                    .get_by_tx_hash(tx_reference.tx_hash)
                    .expect("Transaction hash from queue must appear in pool.")
            })
            .cloned() // Soft-delete: return without deleting from mempool.
            .collect())
    }
