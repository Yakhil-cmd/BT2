### Title
Gateway Stateful Validator Checks Only L2 Gas Price, Admitting Transactions with Insufficient L1/L1DataGas Bounds - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary
`StatefulTransactionValidator::validate_resource_bounds` fetches only the previous block's **L2 gas price** and validates only that field. The L1 gas price and L1 data gas price are never compared against the previous block's prices. A transaction with a compliant L2 gas price but zero (or arbitrarily low) L1 gas price and L1 data gas price passes gateway admission, enters the mempool, and is only rejected later during blockifier pre-validation — after consuming sequencer resources. This is the direct analog of the Balancer bug: a multi-dimensional check uses only a strict subset of the required inputs, leaving the remaining dimensions unguarded.

### Finding Description

`validate_resource_bounds` in `StatefulTransactionValidator` reads only `strk_gas_prices.l2_gas_price` from the previous block and delegates to `validate_tx_l2_gas_price_within_threshold`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:229-240
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
```

`validate_tx_l2_gas_price_within_threshold` inspects only `l2_gas.max_price_per_unit`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:364-388
match tx_resource_bounds {
    ValidResourceBounds::AllResources(tx_resource_bounds) => {
        let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
        ...
        if tx_l2_gas_price.0 < threshold { return Err(...) }
    }
    ValidResourceBounds::L1Gas(_) => { /* No validation required */ }
}
```

The function carries an explicit acknowledgement of the gap:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:358
// TODO(Arni): Consider running this validation for all gas prices.
```

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared against the previous block's `l1_gas_price` or `l1_data_gas_price`. A transaction that sets both to zero passes every gateway check.

The blockifier's `check_fee_bounds` (called inside `perform_pre_validation_stage`) does validate all three gas prices for `AllResources` transactions:

```rust
// crates/blockifier/src/transaction/account_transaction.rs:398-424
ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_gas_resource_bounds,
    l2_gas: l2_gas_resource_bounds,
    l1_data_gas: l1_data_gas_resource_bounds,
}) => {
    vec![
        (L1Gas,     l1_gas_resource_bounds,     ..., *l1_gas_price),
        (L1DataGas, l1_data_gas_resource_bounds, ..., *l1_data_gas_price),
        (L2Gas,     l2_gas_resource_bounds,      ..., *l2_gas_price),
    ]
}
```

So the invariant "reject at the gateway any transaction whose resource bounds are below the current market price" is only partially enforced: the L2 dimension is guarded; the L1 and L1DataGas dimensions are not.

### Impact Explanation

An unprivileged sender can craft `AllResources` transactions with:
- `l2_gas.max_price_per_unit` ≥ threshold (passes both stateless and stateful gateway checks)
- `l1_gas.max_price_per_unit = 0`
- `l1_data_gas.max_price_per_unit = 0`

These transactions pass all gateway admission checks, occupy mempool slots, and are only discarded when the sequencer attempts to execute them and `check_fee_bounds` fires. The gateway's purpose — to prevent the mempool from filling with transactions that will never be sequenced — is defeated for the L1 and L1DataGas price dimensions. This matches the **High** impact tier: *Mempool/gateway/RPC admission accepts invalid transactions before sequencing*.

### Likelihood Explanation

The attack requires no privilege, no special account, and no on-chain state. Any client that can submit a V3 `invoke` or `declare` transaction can trigger it. The only cost is the L2 gas price field, which must meet the threshold; the attacker sets L1 and L1DataGas prices to zero at no cost.

### Recommendation

Extend `validate_resource_bounds` to fetch all three gas prices from the previous block and pass them to a generalised threshold check, mirroring the pattern already used for L2:

```rust
let previous_block_l1_gas_price = block_info.gas_prices.strk_gas_prices.l1_gas_price;
let previous_block_l1_data_gas_price = block_info.gas_prices.strk_gas_prices.l1_data_gas_price;
// validate l1_gas and l1_data_gas prices against their respective thresholds
```

Alternatively, factor the threshold multiplier into a single helper that iterates over all three `(resource, tx_price, block_price)` tuples, consistent with how `check_fee_bounds` already iterates over all resources.

### Proof of Concept

1. Read the current block's `strk_gas_prices.l2_gas_price` (e.g. `P`).
2. Construct a V3 invoke transaction with:
   - `resource_bounds.l2_gas.max_price_per_unit = P` (meets the `min_gas_price_percentage` threshold)
   - `resource_bounds.l1_gas.max_price_per_unit = 0`
   - `resource_bounds.l1_data_gas.max_price_per_unit = 0`
3. Submit to the gateway. `StatelessTransactionValidator::validate_resource_bounds` passes (L2 price ≥ `min_gas_price`; total fee > 0 if L2 amount > 0). `StatefulTransactionValidator::validate_resource_bounds` passes (L2 price ≥ threshold; L1/L1DataGas not checked).
4. Transaction enters the mempool.
5. When the sequencer calls `perform_pre_validation_stage`, `check_fee_bounds` compares `l1_gas.max_price_per_unit = 0` against the actual L1 gas price and returns `InsufficientResourceBounds`. The transaction is discarded — after consuming a mempool slot and sequencer execution time.

Repeating at high volume constitutes a low-cost mempool-exhaustion attack against the sequencer. [1](#0-0) [2](#0-1) [3](#0-2)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
