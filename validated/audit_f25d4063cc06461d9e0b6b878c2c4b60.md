### Title
Block Native Computational Limit Exhaustion via Zero-Byte Calldata Underpricing — (File: `basic_bootloader/src/bootloader/constants.rs`)

---

### Summary

ZKsync OS enforces two independent block-level resource limits: an EVM gas limit and a native computational limit (`MAX_NATIVE_COMPUTATIONAL = 2^35`). The intrinsic EVM gas cost for calldata is 4× cheaper for zero bytes than for non-zero bytes, but the intrinsic **native computational cost per calldata byte is identical** for both. EIP-7623 (which would impose a floor gas cost on calldata tokens) is gated behind a compile-time feature flag `eip_7623` that is **not enabled by default**. An unprivileged sender can therefore craft transactions with large zero-byte calldata that exhaust the block's native computational limit while consuming only one-quarter of the EVM gas that equivalent non-zero calldata would require, enabling a cheaper block-stuffing attack.
