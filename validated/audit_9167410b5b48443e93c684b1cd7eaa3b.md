### Title
`native_per_pubdata` Truncates to Zero via Floor Division, Bypassing Pubdata Native Resource Charging — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

When `pubdata_price < native_price`, the integer floor division `pubdata_price.wrapping_div(native_price)` produces `native_per_pubdata = 0`. This silently zeroes out all native resource charging for pubdata, allowing any unprivileged transaction sender to generate pubdata up to the block limit without paying the intended native resource cost. The root cause is an inconsistency: `native_per_gas` uses ceiling division (`div_ceil`) to guarantee a non-zero charge, while `native_per_pubdata` uses floor division (`wrapping_div`), which truncates to zero whenever `pubdata_price < native_price`.

---

### Finding Description

In `validate_and_compute_fee_for_transaction` (ZK path), `native_per_pubdata` is derived as:

```rust
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
``` [1](#0-0) 

By contrast, `native_per_gas` uses ceiling division:

```rust
u256_try_to_u64(&gas_price.div_ceil(native_price)).ok_or(...)
``` [2](#0-1) 

When `pubdata_price < native_price` (e.g., `pubdata_price = 50`, `native_price = 100`), `wrapping_div` yields `0`. This zero propagates through three downstream enforcement points:

**1. Intrinsic pubdata overhead in `create_resources_for_tx`:**
```rust
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// = 0 * intrinsic_pubdata = 0 → no native deducted for intrinsic pubdata
``` [3](#0-2) 

**2. Execution pubdata charge in `get_resources_to_charge_for_pubdata`:**
```rust
let native = current_pubdata_spent.checked_mul(native_per_pubdata)
// = current_pubdata_spent * 0 = 0 → no native charged for any pubdata generated
``` [4](#0-3) 

**3. Post-execution sufficiency check in `check_enough_resources_for_pubdata`:**
```rust
let enough = resources.has_enough(&resources_for_pubdata);
// resources_for_pubdata = 0 → always true, pubdata check never fails
``` [5](#0-4) 

The same zero propagates into the L1 transaction path via `prepare_and_check_resources`, where `native_per_pubdata = (gas_per_pubdata as u64).checked_mul(native_per_gas)` can also be zero when `native_per_gas = 0` (free L1 tx), but the L2 path is the primary concern since it is reachable by any unprivileged sender. [6](#0-5) 

The `api/src/helpers.rs` off-chain validation helper mirrors this same floor-division bug:
```rust
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [7](#0-6) 

---

### Impact Explanation

When `pubdata_price < native_price`, every L2 transaction can generate pubdata up to the block pubdata limit at zero native resource cost. The intended economic invariant — that pubdata generation is bounded by the sender's native resource budget — is completely bypassed. The operator loses all native resource revenue for pubdata, and the native resource accounting diverges from the actual proving cost incurred. The block pubdata limit (`pubdata_limit` in `BlockMetadataFromOracle`) still caps total pubdata per block, bounding the per-block impact, but within that cap any sender can fill pubdata slots for free. [8](#0-7) 

---

### Likelihood Explanation

The condition `pubdata_price < native_price` is realistic. The operator sets both values independently in `BlockMetadataFromOracle`. The default test configuration already uses `pubdata_price = 0`, and any deployment where `pubdata_price` is set to a value smaller than `native_price` (e.g., `pubdata_price = 50`, `native_price = 100`) silently triggers the zero truncation. The operator may not realize that integer floor division causes `native_per_pubdata = 0` rather than the intended fractional charge. The inconsistency with `native_per_gas` (which uses `div_ceil`) makes this a latent misconfiguration trap. [9](#0-8) 

---

### Recommendation

Replace `wrapping_div` with `div_ceil` for `native_per_pubdata`, consistent with how `native_per_gas` is computed:

```rust
// Before (floor division — truncates to 0 when pubdata_price < native_price):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (ceiling division — guarantees ≥ 1 when pubdata_price > 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix to the off-chain helper in `api/src/helpers.rs` line 427. Additionally, consider adding a validation that rejects block metadata where `pubdata_price > 0` but `native_per_pubdata` would compute to 0, to surface the misconfiguration explicitly rather than silently zeroing the charge.

---

### Proof of Concept

**Setup:**
- `native_price = 100`
- `pubdata_price = 50` (operator intends to charge 0.5 native per pubdata byte)
- `native_per_pubdata = 50 / 100 = 0` (floor division truncates)

**Attack path:**
1. Attacker submits a transaction that performs many storage writes (each write generates ~32 bytes of pubdata).
2. During validation, `native_per_pubdata = 0`, so `intrinsic_pubdata_overhead = 0` — no native is deducted from the native limit for intrinsic pubdata.
3. During execution, the transaction generates N bytes of pubdata.
4. Post-execution, `get_resources_to_charge_for_pubdata` computes `N * 0 = 0` native to charge.
5. `check_enough_resources_for_pubdata` returns `enough = true` unconditionally.
6. The transaction succeeds. The attacker paid zero native resource cost for pubdata, regardless of how much pubdata was generated.
7. Repeat across multiple transactions to fill the block's pubdata quota at zero native cost.

The analogous structure to M-17: just as `quorumVotes = totalSupply * bps / 10_000` rounds to 0 when `totalSupply` is small (allowing a proposal to pass with zero votes), `native_per_pubdata = pubdata_price / native_price` rounds to 0 when `pubdata_price < native_price` (allowing pubdata generation with zero native cost). In both cases, a computed threshold that was intended to enforce a minimum cost silently becomes 0 due to integer division truncation, and the enforcement check that depends on it is trivially satisfied.
