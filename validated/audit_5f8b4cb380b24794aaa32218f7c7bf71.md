### Title
Unprivileged Caller Can Permanently Freeze All Consensus Block Rewards via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function is publicly callable by any address and accepts a caller-controlled `disable_rewards` boolean. When called with `disable_rewards: true`, it writes the current block number into the **global** `last_reward_block` storage variable without distributing any rewards. Because the global guard is checked before the `disable_rewards` branch, any subsequent legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`. An attacker can call this once per block with any valid staker address and `disable_rewards: true`, permanently starving every staker of consensus block rewards.

---

### Finding Description

`update_rewards` is the sole entry point for distributing consensus block rewards to stakers:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits before distributing
    }
    ...
}
``` [1](#0-0) 

The critical ordering is:

1. `last_reward_block` is written to `current_block_number` **before** the `disable_rewards` branch is evaluated.
2. `disable_rewards` is a free parameter — there is no access control beyond `general_prerequisites`, which only checks the contract is unpaused and the caller is non-zero. [2](#0-1) 

`last_reward_block` is a **single global** storage slot, not per-staker: [3](#0-2) 

Therefore, one call with `disable_rewards: true` in block N exhausts the entire block's reward slot for **all** stakers. Any subsequent call in block N — regardless of staker or caller — hits the `REWARDS_ALREADY_UPDATED` assertion and reverts.

The analog to the external report is exact: in the zksync case, `l2SystemContractsUpgradeBlockNumber` was **not set** when it should have been, blocking future upgrades. Here, `last_reward_block` **is set** (by an attacker) when it should not be, blocking all reward distribution for that block. Both are state-transition bugs where a single storage variable's incorrect value makes a critical code path unreachable.

---

### Impact Explanation

Every staker in the protocol loses their consensus block rewards for every block the attacker targets. Because the attacker needs only one call per block (not per staker), the cost is minimal while the damage scales with the total staking reward pool. Sustained over time this constitutes **permanent freezing of unclaimed yield** for all stakers — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

The attack requires:
- Any valid, active staker address (trivially readable from on-chain `NewStaker` events).
- One transaction per block with `disable_rewards: true`.
- Gas costs only (low on Starknet L2).

The attack is fully automatable, requires no privileged access, no leaked keys, and no external dependency. Likelihood is **High**.

---

### Recommendation

Restrict who may supply `disable_rewards: true`. Options in increasing strictness:

1. **Access-control the parameter**: require the caller to be a designated consensus contract when `disable_rewards` is `true`.
2. **Access-control the whole function**: only allow a trusted consensus contract to call `update_rewards` at all.
3. **Make `last_reward_block` per-staker**: so one staker's update cannot block others (though this changes the intended single-update-per-block semantics).

The simplest fix consistent with the existing design is option 1 or 2 — add an `assert_caller_is_consensus_contract` guard analogous to the existing `assert_caller_is_attestation_contract`. [4](#0-3) 

---

### Proof of Concept

1. Attacker reads any `NewStaker` event to obtain a valid `staker_A` address.
2. Each block N, attacker submits: `update_rewards(staker_A, disable_rewards: true)`.
3. Inside the call: `last_reward_block` is written to N; the function returns early — zero rewards distributed.
4. Any legitimate call `update_rewards(any_staker, disable_rewards: false)` in block N hits:
   ```
   assert!(current_block_number > self.last_reward_block.read(), …REWARDS_ALREADY_UPDATED);
   ```
   and reverts.
5. Repeated every block → all stakers receive zero consensus rewards indefinitely. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
