### Title
Unprivileged Caller Can Permanently Destroy Per-Block Consensus Rewards via `disable_rewards` Flag — (File: `src/staking/staking.cairo`)

---

### Summary

In V3 consensus-rewards mode, `update_rewards` is a public, permissionless function that accepts a `disable_rewards` flag. When called with `disable_rewards: true`, it writes the current block number into `last_reward_block` but distributes nothing. Because the contract enforces a strict one-call-per-block invariant, any unprivileged caller can front-run the legitimate block-proposer call, permanently destroying that block's rewards for every staker and pool member.

---

### Finding Description

`update_rewards` is exposed through `IStakingRewardsManager` with no role guard beyond `general_prerequisites()`, which only checks the pause flag and a non-zero caller. [1](#0-0) 

The one-call-per-block gate is enforced by a **global** `last_reward_block` field: [2](#0-1) 

Immediately after the gate check passes, `last_reward_block` is written to the current block number. Only then does the code branch on `disable_rewards`: [3](#0-2) 

Because `last_reward_block` is updated **before** the `disable_rewards` branch, any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. The rewards that would have been minted for that block are permanently unrecoverable.

`general_prerequisites` imposes no caller restriction: [4](#0-3) 

---

### Impact Explanation

Each block in consensus-rewards mode produces a fixed reward amount calculated from `yearly_mint * avg_block_duration / SECONDS_IN_YEAR`: [5](#0-4) 

When a block's reward slot is consumed with `disable_rewards: true`, those rewards are never credited to `unclaimed_rewards` in the `RewardSupplier`, never transferred to stakers or pools, and cannot be reclaimed. Repeated across many blocks, this constitutes **permanent freezing of unclaimed yield** for all stakers and delegators — matching the High-severity impact tier.

---

### Likelihood Explanation

- The function is callable by any non-zero address with no privilege requirement.
- The attacker need only submit a transaction with `disable_rewards: true` and a valid (but arbitrary) staker address before the legitimate block-proposer call lands.
- On Starknet, transaction ordering within a block is deterministic and visible to searchers, making front-running straightforward.
- The attacker spends only gas; there is no minimum stake or role requirement.

---

### Recommendation

Restrict `update_rewards` so that only the operational address of the staker being rewarded (i.e., the block proposer) may call it, or gate the `disable_rewards: true` path behind a privileged role (e.g., `only_security_agent`). At minimum, remove the ability for an unprivileged caller to pass `disable_rewards: true`.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Block `N` begins. The legitimate block proposer prepares a call to `update_rewards(proposer_address, disable_rewards: false)`.
3. Attacker submits `update_rewards(any_valid_staker, disable_rewards: true)` with higher priority.
4. Attacker's transaction executes first:
   - `current_block_number (N) > last_reward_block` → gate passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → function returns immediately; no rewards distributed.
5. Block proposer's transaction executes next:
   - `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
6. Block `N`'s rewards are permanently lost. No staker or pool member receives yield for this block.
7. Repeating this every block eliminates all consensus-mode rewards at the cost of one transaction per block. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
