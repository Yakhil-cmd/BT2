### Title
Unrestricted `disable_rewards` Flag in `update_rewards` Allows Any Caller to Permanently Freeze All Staker Consensus Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The public `update_rewards` function accepts a `disable_rewards: bool` parameter that, when `true`, writes the current block number to the **global** `last_reward_block` storage slot without distributing any rewards. Because `last_reward_block` is a single contract-wide value, any non-zero address can call `update_rewards(any_active_staker, true)` as the first transaction in every block and permanently prevent all stakers from receiving consensus rewards.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks two things: the contract is not paused, and the caller is not the zero address. [1](#0-0) 

No check restricts who may supply `disable_rewards: true`. When that flag is set, the function:

1. Validates the staker exists, is active, and has non-zero STRK balance.
2. **Writes `current_block_number` to the global `last_reward_block`** (line 1485).
3. Returns immediately without calling `_update_rewards` (line 1487–1489). [2](#0-1) 

`last_reward_block` is a single global storage field, not a per-staker map: [3](#0-2) 

The guard at line 1455 enforces that only **one** `update_rewards` call can succeed per block: [4](#0-3) 

An attacker who calls `update_rewards(any_active_staker, true)` first in a block consumes the entire block's reward slot. No other staker can receive consensus rewards in that block. Repeating this every block permanently freezes all consensus reward accrual.

The `_update_rewards` path that is bypassed is responsible for:
- Incrementing `staker_info.unclaimed_rewards_own`
- Calling `update_unclaimed_rewards_from_staking_contract` on the reward supplier
- Transferring pool rewards to delegation pools [5](#0-4) 

`general_prerequisites` confirms there is no role check: [6](#0-5) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

All stakers' `unclaimed_rewards_own` fields are never incremented, and delegation pool rewards are never transferred. The attacker does not profit; the damage is entirely to stakers and delegators. The attack is sustainable indefinitely: the attacker only needs to be the first caller in each block.

---

### Likelihood Explanation

**High.** Any non-zero address can execute this attack. Active staker addresses are publicly readable on-chain. The only cost is gas per block. No privileged key, bridge access, or governance capture is required.

---

### Recommendation

Restrict the `disable_rewards: true` path to the staker themselves, their registered operational address, or a privileged protocol role. For example:

```cairo
if disable_rewards {
    assert!(
        get_caller_address() == staker_address
            || get_caller_address() == staker_info.operational_address,
        "{}",
        Error::UNAUTHORIZED_DISABLE_REWARDS,
    );
    return;
}
```

Alternatively, if `disable_rewards` is only needed for internal protocol transitions, remove it from the public ABI and expose it only through a role-gated internal path.

---

### Proof of Concept

```
Block N:
  Attacker calls: update_rewards(staker_A, disable_rewards=true)
    → last_reward_block := N
    → no rewards distributed
    → staker_A.unclaimed_rewards_own unchanged
    → pool rewards not sent

  Any legitimate call: update_rewards(staker_B, disable_rewards=false)
    → PANICS: REWARDS_ALREADY_UPDATED (current_block_number == last_reward_block)

Block N+1:
  Attacker repeats → same outcome

Result after K blocks: zero consensus rewards accrued by any staker or pool.
```

The attacker supplies any valid active staker address (e.g., read from the public `stakers` vector) and calls the function once per block. This is the direct analog of the external report's `stewardSpend` classification bypass: just as `stewardSpend` bypasses the 5%-of-treasury Extended threshold that `distribute` must satisfy, `update_rewards(_, true)` bypasses the reward-distribution logic that every legitimate consensus-reward block must execute — and in both cases the bypass is reachable by an unprivileged caller with no authorization check.

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
