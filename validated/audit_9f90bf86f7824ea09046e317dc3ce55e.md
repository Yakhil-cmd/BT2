### Title
`RewardSupplier.sol#tick` silently locks ETH when no minting is required — (`File: L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

`RewardSupplier.tick()` is `payable` and expects the caller to supply ETH as L1→L2 message fees. However, the entire fee-forwarding logic is gated inside an `if (amountToMint > 0)` block. When no pending mint requests exist, `msg.value` is accepted but never forwarded or refunded, permanently locking the caller's ETH in the contract.

---

### Finding Description

`tick()` is declared `external payable` and splits `msg.value` across two downstream payable calls:

1. `bridge().depositWithMessage{value: msgFee}(...)` — bridges minted tokens to L2
2. `messagingContract().sendMessageToL2{value: msgFee}(...)` — sends a total-supply update to the L2 minting curve

Both calls are inside the `if (amountToMint > 0)` branch: [1](#0-0) 

When `requiredMinting()` returns `(0, 0)` — i.e., no L2 mint-request messages are pending — the branch is skipped entirely. Any ETH sent by the caller is accepted by the `payable` function but never used or returned. [2](#0-1) 

The contract has no `receive()` fallback, no `withdraw()` function, and no ETH-recovery mechanism visible in `RewardSupplierStorage` or `ProxySupportImpl`. [3](#0-2) 

---

### Impact Explanation

Any ETH sent to `tick()` when `amountToMint == 0` is permanently frozen inside the `RewardSupplier` contract. There is no path to recover it. This matches the allowed impact: **Temporary/Permanent freezing of funds**.

---

### Likelihood Explanation

`tick()` is an unrestricted `external` function — any address can call it. The condition `amountToMint == 0` is the normal steady-state: it is true whenever the L2 reward supplier has not yet emitted new mint-request messages (e.g., between epochs, or when the contract balance is already sufficient). A caller who pre-pays ETH fees to trigger a tick during such a window loses their ETH with no recourse. [4](#0-3) 

---

### Recommendation

Refund `msg.value` to the caller when no minting work is performed, or revert if ETH is sent unnecessarily:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ... existing logic ...
    } else {
        // Refund unused ETH
        if (msg.value > 0) {
            (bool ok, ) = msg.sender.call{value: msg.value}("");
            require(ok, "ETH_REFUND_FAILED");
        }
    }
}
```

Alternatively, remove `payable` from `tick()` and require callers to pre-fund the contract separately, or revert with a clear error when `msg.value > 0` and `amountToMint == 0`.

---

### Proof of Concept

1. No L2 mint-request messages are pending (normal steady-state between epochs).
2. Caller invokes `tick{value: 1 ether}()`.
3. `requiredMinting()` returns `(0, 0)`.
4. The `if (amountToMint > 0)` block is skipped.
5. The function returns normally; `1 ether` is now locked in `RewardSupplier` with no recovery path. [1](#0-0)

    function requiredMinting() public view returns (uint256, uint256) {
        uint256[] memory messagePayload = new uint256[](1);
        messagePayload[0] = TOKENS_PER_MINT_REQUEST;

        bytes32 msgHash = l2ToL1MsgHash(mintRequestSource(), address(this), messagePayload);
        // Limit the number of msgs to consume to limit.
        uint256 numMsgsToConsume = Math.min(
            messagingContract().l2ToL1Messages(msgHash),
            MAX_MESSAGES_TO_PROCESS_PER_TICK
        );

        return (TOKENS_PER_MINT_REQUEST * numMsgsToConsume, numMsgsToConsume);
    }

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
