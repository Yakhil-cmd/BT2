### Title
Missing Caller Access Control on `update_rewards` Allows Any Address to Suppress Block Rewards — (`File: src/staking/staking.cairo`)

### Summary

The spec mandates "Only starkware sequencer" access for `update_rewards`, but the implementation contains no caller check. Because `last_reward_block` is a **global** (single-slot) variable, any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards=true)` to consume the block's single reward slot without distributing rewards, permanently discarding that block's yield for all stakers.

---

### Finding Description

The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 is:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks is_paused
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}", Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker validity checks ...
    self.last_reward_block.write(current_block_number);   // global slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                            // exits without distributing rewards
    }
    // ... reward distribution ...
}
```

There is no `assert!(get_caller_address() == sequencer_address, ...)` or equivalent. `general_prerequisites()` is shared with `stake`, `claim_rewards`, and other public functions, so it cannot be checking for a privileged caller.

`last_reward_block` is declared as a single `BlockNumber` storage slot (line 187), not a per-staker map:

```
last_reward_block: BlockNumber,
```

This means exactly **one** `update_rewards` call can succeed per block. Once it succeeds, `last_reward_block` equals the current block and every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

**Attack path:**

1. Attacker monitors the mempool / block production.
2. In any block where the sequencer has not yet called `update_rewards`, the attacker calls `update_rewards(any_valid_active_staker, disable_rewards=true)`.
3. The function passes all staker-validity checks, writes `last_reward_block = current_block`, then hits `if disable_rewards … return` and exits without distributing any rewards.
4. The sequencer's legitimate `update_rewards` call for that block reverts with `REWARDS_ALREADY_UPDATED`.
5. That block's STRK/BTC rewards are permanently lost — they are never credited to any staker and never tracked for future recovery.

The attacker can repeat this every block, continuously suppressing all consensus-era block rewards.

---

### Impact Explanation

Each suppressed block permanently discards the STRK (and BTC) block rewards that would have been credited to a staker's `unclaimed_rewards_own` and forwarded to delegation pools. There is no catch-up mechanism; `last_reward_block` is only ever written forward, never replayed. This constitutes **permanent freezing of unclaimed yield** for every staker and delegator in the protocol.

---

### Likelihood Explanation

The function is publicly callable on-chain with no economic barrier. The attacker only needs to know any valid active staker address (trivially obtained from `NewStaker` events or `get_stakers`). The attack costs only gas. It can be automated to run every block. On Starknet the sequencer controls ordering, but the sequencer is not guaranteed to call `update_rewards` in every block; any block it skips is permanently exploitable.

---

### Recommendation

Add an explicit caller check at the top of `update_rewards`, consistent with the spec:

```cairo
assert!(
    get_caller_address() == self.sequencer_address.read(),
    "{}", Error::CALLER_IS_NOT_SEQUENCER,
);
```

Store the authorized sequencer address in contract storage (set at construction or via governance), and enforce it as the first check inside `StakingRewardsManagerImpl::update_rewards`.

### Proof of Concept

```
// Block N: sequencer has not yet called update_rewards
// Attacker calls:
staking.update_rewards(
    staker_address: any_valid_active_staker,
    disable_rewards: true,

// Sequencer then calls:

// → REVERTS: REWARDS_ALREADY_UPDATED

// Block N rewards are permanently lost.
```
