### Title
Unrestricted `update_rewards(disable_rewards: true)` Permanently Denies Staker Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and unconditionally writes `last_reward_block` **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(victim_staker, disable_rewards: true)` to consume the block's single reward slot without distributing any rewards, permanently denying the legitimate staker their block rewards for that block. Repeated across every block, this constitutes a persistent griefing attack that freezes all consensus-mode yield for the targeted staker.

---

### Finding Description

`update_rewards` is a public, permissionless function callable by any address:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
)
``` [1](#0-0) 

The function enforces a **global, single-slot-per-block** invariant via `last_reward_block`:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

Critically, `last_reward_block` is written **before** the `disable_rewards` branch:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← slot consumed here

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← rewards never distributed
}
```
