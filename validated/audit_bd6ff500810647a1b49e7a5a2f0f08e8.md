### Title
Unvalidated `app_pk` in CKD Protocol Allows Confidential Derived Key Extraction via Identity-Point Bypass - (File: `src/confidential_key_derivation/protocol.rs`)

---

### Summary

The `ckd()` function in `src/confidential_key_derivation/protocol.rs` accepts a caller-controlled `app_pk: PublicKey` (an `ElementG1` point) with no validation that it is a non-identity group element. When a caller supplies the G1 identity point as `app_pk`, the ElGamal encryption term `y_i * A` collapses to the identity, causing each participant's `big_c` share to equal their raw secret contribution `x_i * H(pk, app_id)`. The coordinator then aggregates these into `C = msk * H(pk, app_id)` — the unencrypted confidential derived key — which is returned in plaintext inside `CKDOutput`. The attacker recovers it by calling `unmask(0)`.

---

### Finding Description

The CKD protocol is designed to deliver the confidential derived key `msk * H(pk, app_id)` **encrypted** under the app's ElGamal public key `A = a * G1`, so that only the holder of the private scalar `a` can decrypt it. Each participant computes:

```
S_i  = x_i * H(pk, app_id)
C_i  = S_i + y_i * A
Y_i  = y_i * G1
```

and sends `(λ_i * Y_i, λ_i * C_i)` to the coordinator, which aggregates to `(Y, C)` where `C = msk * H(pk, app_id) + a * Y`. The app decrypts with `C - a * Y`.

The `ckd()` entry point performs only structural checks (participant count, duplicates, membership): [1](#0-0) 

It passes `app_pk` directly into `compute_signature_share` without any validation: [2](#0-1) 

Specifically, line 174 computes:

```rust
let big_c = big_s + app_pk * y.0;
``` [3](#0-2) 

If `app_pk` is `G1Projective::identity()` (the additive identity), then `app_pk * y.0 = identity`, so `big_c = big_s = x_i * H(pk, app_id)`. After Lagrange normalization and coordinator aggregation:

```
C = Σ λ_i * C_i = Σ λ_i * x_i * H(pk, app_id) = msk * H(pk, app_id)
```

The `CKDOutput` type exposes this directly: [4](#0-3) 

The attacker calls `ckd_output.unmask(0)` which computes `C - Y * 0 = C = msk * H(pk, app_id)` — the confidential derived key in plaintext.

The `PublicKey` type alias is `ElementG1 = blstrs::G1Projective`, which freely represents the identity point. No downstream check in `compute_signature_share`, `do_ckd_participant`, or `do_ckd_coordinator` rejects it: [5](#0-4) 

---

### Impact Explanation

**Critical — Extraction/disclosure of confidential derived secrets.**

The entire security guarantee of the CKD protocol is that `msk * H(pk, app_id)` is only accessible to the legitimate app holding the private key `a`. Supplying `app_pk = identity` completely removes the ElGamal encryption layer, causing the MPC network to output the secret in cleartext. An attacker who can invoke `ckd()` on any node with an arbitrary `app_pk` can extract the confidential derived key for any `app_id` of their choosing, without possessing any legitimate credential.

---

### Likelihood Explanation

The `ckd()` function is a public API entry point. Any caller — including an unprivileged library user or a malicious participant acting as coordinator — can supply `app_pk`. The identity point is a valid `G1Projective` value that requires no special privilege to construct (e.g., `G1Projective::identity()` or `G1Projective::default()`). No on-chain or off-chain gating is enforced inside the library itself. The attack requires a single malformed `ckd()` invocation.

---

### Recommendation

Add an explicit check in `ckd()` (or at the top of `compute_signature_share`) that rejects the identity point:

```rust
if bool::from(app_pk.is_identity()) {
    return Err(InitializationError::BadParameters(
        "app_pk must not be the identity point".to_string(),
    ));
}
```

This mirrors the existing `msg_hash == 0` guard in the robust ECDSA signing path: [6](#0-5) 

Additionally, consider validating that `app_pk` is a point on the correct subgroup (prime-order subgroup check) to prevent small-subgroup attacks.

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
