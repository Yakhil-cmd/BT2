### Title
Pool Contract's `exit_delegation_pool_intent` Bypasses Staking Contract Pause When Staker Is Removed - (File: src/pool/pool.cairo)

### Summary
The Pool contract's `exit_delegation_pool_intent` function can be invoked while the Staking contract is paused, specifically when the associated staker has been removed. The spec mandates "Staking contract is unpaused" as a precondition for this function, but the implementation silently bypasses the cross-contract call — and therefore the pause enforcement — when `!is_staker_active()`. This allows a pool member to start the exit-window countdown during a pause period, potentially enabling them to exit immediately once the pause is lifted.

### Finding Description

The Staking contract exposes a pause mechanism: [1](#0-0) 

Every state-changing function in the Staking contract checks `is_paused` before proceeding, including `remove_from_delegation_pool_intent` and `remove_from_delegation_pool_action`.

The Pool contract's `exit_delegation_pool_intent` is supposed to enforce the same pause invariant by delegating to the Staking contract. The spec is explicit: [2](#0-1) 

The implementation routes through `undelegate_from_staking_contract_intent`: [3](#0-2) 

When `!is_staker_active()` (i.e., the staker has been removed via `unstake_action`), the function **skips the cross-contract call entirely** and returns `Time::now()` directly. Because the pause check lives inside the Staking contract's `remove_from_delegation_pool_intent`, it is never reached. The pool member's `unpool_time` is set to the current timestamp with no pause guard.

The exit action path contains a developer comment acknowledging this bypass: [4](#0-3) 

The comment confirms that when the intent is set after staker removal, `remove_from_delegation_pool_action` is expected to be a no-op (funds already reside in the pool contract). This means `exit_delegation_pool_action` can also complete after the pause is lifted, transferring funds to the pool member. [5](#0-4) 

### Impact Explanation

A pool member whose staker has been removed can call `exit_delegation_pool_intent` while the Staking contract is paused. The exit-window countdown (`unpool_time = Time::now()`) begins immediately. If the pause persists for longer than the configured `exit_wait_window`, the pool member can call `exit_delegation_pool_action` the moment the pause is lifted and withdraw funds instantly — bypassing the intended exit wait window entirely. The exit wait window is a core network-stability mechanism; circumventing it allows pool members to exit faster than the protocol intends, undermining the security guarantees of the staking system. This maps to **Medium: Griefing with no profit motive but damage to users or protocol**.

### Likelihood Explanation

The preconditions are:
1. A staker must have completed `unstake_action` (staker removed, pool funds already in pool contract).
2. The Staking contract must be paused (e.g., triggered by a security agent during an incident).
3. The pause must last at least as long as `exit_wait_window`.

Condition 1 is a normal protocol lifecycle event. Condition 2 is the exact scenario the pause mechanism is designed for. Condition 3 is plausible during a prolonged security incident. The combination is realistic and requires no privileged access beyond being a pool member.

### Recommendation

Add an explicit pause check inside `exit_delegation_pool_intent` (or inside `undelegate_from_staking_contract_intent`) that queries the Staking contract's pause state regardless of whether the staker is active:

```cairo
fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
    // Query staking contract pause state directly
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "{}", GenericError::CONTRACT_IS_PAUSED);
    // ... rest of function
}
```

This ensures the pause invariant is enforced on all code paths, consistent with the spec's precondition.

### Proof of Concept

1. Staker stakes and opens a delegation pool; pool member delegates funds.
2. Staker calls `unstake_intent()` then `unstake_action()` → staker is removed, pool funds transferred to pool contract, `staker_removed = true`.
3. Security agent calls `staking_pause_dispatcher.pause()` → Staking contract is paused.
4. Pool member calls `pool.exit_delegation_pool_intent(amount)`:
   - `undelegate_from_staking_contract_intent` is called.
   - `!is_staker_active()` is `true` → staking contract call is skipped.
   - `unpool_time = Time::now()` is set locally. **No pause check is performed.**
   - Call succeeds.
5. Exit window elapses while the contract remains paused.
6. Security admin calls `staking_pause_dispatcher.unpause()`.
7. Pool member immediately calls `pool.exit_delegation_pool_action(pool_member)`:
   - `remove_from_delegation_pool_action` is called on the (now unpaused) Staking contract.
   - No intent exists in the Staking contract → call does nothing (funds already in pool contract per developer comment).
   - `token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount)` executes.
   - Pool member receives funds immediately, having bypassed the exit wait window. [6](#0-5) [7](#0-6)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }
