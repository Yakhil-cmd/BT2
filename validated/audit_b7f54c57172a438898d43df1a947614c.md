### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Consensus Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the staking contract has no caller restriction. Any non-zero address can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` checkpoint without distributing any rewards. Because `last_reward_block` is a single shared slot, one such call per block permanently prevents all stakers from receiving consensus rewards for that block. Repeated across every block, this freezes all unclaimed yield indefinitely.

---

### Finding Description

`update_rewards` is the V3 consensus-rewards entry point. Each block, exactly one call is allowed (enforced by the global `last_reward_block` guard). The function accepts a `disable_rewards: bool` parameter; when `true`, it writes the current block number to `last_reward_block` and returns immediately without computing or distributing any rewards. [1](#0-0) 

The only gate applied is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero: [2](#0-1) 

There is no check that the caller is the attestation contract or any other privileged role. Compare this with the V2 equivalent `update_rewards_from_attestation_contract`, which explicitly enforces `assert_caller_is_attestation_contract()`: [3](#0-2) 

The V3 function omits this guard entirely.

The global reward-slot variable: [4](#0-3) 

Once `last_reward_block` is set to the current block number, the assertion `current_block_number > self.last_reward_block.read()` fails for every subsequent caller in the same block, including the legitimate attestation contract.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, true)` once per block:

1. Advances `last_reward_block` to the current block.
2. Distributes zero rewards.
3. Blocks every other caller (including the attestation contract) from distributing rewards for that block.

Sustained across every block, no staker ever accumulates consensus rewards. All unclaimed yield is frozen indefinitely. The attacker needs only one valid active staker address (public on-chain) and enough gas to submit one transaction per block (~3 s on Starknet).

---

### Likelihood Explanation

**Medium.**

- Entry path is fully permissionless; no privileged key or external dependency is required.
- The attacker has no profit motive (pure griefing), which limits rational adversaries.
- However, the cost per block is low on Starknet, and a motivated attacker (e.g., a competing protocol or a staker who wants to suppress rivals) could sustain the attack indefinitely.
- Active staker addresses are trivially discoverable from on-chain events.

---

### Recommendation

Add the same caller restriction that exists in the V2 path:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_attestation_contract(); // <-- add this
    ...
}
```

Alternatively, split the function: keep a public, rewards-only path (no `disable_rewards` parameter) and restrict the `disable_rewards: true` path to the attestation contract.

---

### Proof of Concept

1. Staker `S` is active and eligible for consensus rewards at block `B`.
2. Attacker `A` (any non-zero EOA) calls `update_rewards(S, true)` at block `B`.
3. `last_reward_block` is written to `B`; no rewards are computed or transferred.
4. The attestation contract attempts `update_rewards(S, false)` at block `B` — reverts with `REWARDS_ALREADY_UPDATED` because `B > B` is false.
5. Staker `S` receives zero rewards for block `B`.
6. Repeating step 2 every block freezes all consensus rewards permanently. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
