### Title
Unrestricted `update_rewards` with `disable_rewards: true` Allows Griefing Freeze of Consensus Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `IStakingRewardsManager::update_rewards` function in `src/staking/staking.cairo` is callable by any unprivileged address and accepts a caller-controlled `disable_rewards: bool` parameter. The function unconditionally writes `current_block_number` to `last_reward_block` **before** checking `disable_rewards`, consuming the single per-block reward slot. When called with `disable_rewards: true`, no rewards are distributed to any staker for that block. An attacker can call this every block to permanently suppress all consensus block rewards across the entire protocol.

---

### Finding Description

`update_rewards` enforces a one-call-per-block invariant via `last_reward_block`:

```
assert!(current_block_number > self.last_reward_block.read(), …);
…
self.last_reward_block.write(current_block_number);   // ← committed unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits before distributing rewards
}
``` [1](#0-0) 

There is no access-control guard on this function beyond `general_prerequisites()` (which only checks the pause flag). Any external caller may supply `disable_rewards: true`. [2](#0-1) 

Because `last_reward_block` is written before the `disable_rewards` branch, a single call per block with `disable_rewards: true` is sufficient to exhaust the slot. Legitimate stakers or the consensus layer that attempt to call `update_rewards` in the same block will revert with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

Block rewards are computed from `avg_block_duration` and the yearly mint; they are not carried forward to the next block. Any block whose slot is consumed without distributing rewards results in permanently lost yield. [4](#0-3) 

---

### Impact Explanation

Every block for which the attacker fires `update_rewards(any_valid_staker, disable_rewards: true)` causes the entire block's STRK and BTC consensus rewards to be silently discarded. Stakers and delegators never receive those rewards; they cannot be recovered in a later block. Sustained over an epoch, this constitutes a **permanent freezing of unclaimed yield** for all protocol participants.

Allowed impact matched: **High — Permanent freezing of unclaimed yield or unclaimed royalties; Temporary freezing of funds.**

---

### Likelihood Explanation

- The entry point is fully public; no stake, role, or special state is required.
- The attacker only needs to submit one transaction per block with a valid (active, non-zero-balance) staker address as the argument.
- On Starknet, L2 transaction fees are low, making sustained per-block griefing economically viable.
- The attacker may have an indirect economic incentive: if they hold stake outside the protocol (e.g., in a competing venue), suppressing on-chain staking rewards reduces competition for their own yield.

---

### Recommendation

1. **Remove the `disable_rewards` parameter from the public interface.** If the flag is needed internally (e.g., for the first epoch block), gate it behind an internal function or a privileged caller check (e.g., `only_app_governor` or restrict to the attestation/consensus contract).
2. **Restrict `update_rewards` to a trusted caller** (e.g., the attestation contract or a designated consensus address), analogous to how `update_rewards_from_attestation_contract` already enforces `assert_caller_is_attestation_contract`. [5](#0-4) 

---

### Proof of Concept

```
// Attacker script — run once per block indefinitely
let any_valid_staker = <any active staker address>;
staking_contract.update_rewards(
    staker_address: any_valid_staker,
    disable_rewards: true,   // ← caller-controlled
);
// Effect: last_reward_block = current_block; no rewards distributed.
// All subsequent calls in this block revert with REWARDS_ALREADY_UPDATED.
// Block rewards for this block are permanently lost.
```

The attacker requires no capital, no flash loan, and no privileged role. The only cost is the gas for one Starknet transaction per block. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
