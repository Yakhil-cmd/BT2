### Title
Missing Sender Authorization in Bank Precompile `transfer()` Enables Unauthorized Token Theft — (File: `x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The bank precompile's `transfer(sender, recipient, amount)` method accepts the `sender` address as a calldata argument but derives the token denom from `contract.Caller()`. There is no check that `contract.Caller() == sender` or that the caller holds any allowance from `sender`. Any unprivileged contract can therefore call `bankPrecompile.transfer(victim, attacker, amount)` and drain the victim's `evm/<caller_contract>` native-denom balance without the victim's consent.

---

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` case of `BankContract.Run()` is:

```go
sender    := args[0].(common.Address)   // taken from calldata — not msg.sender
recipient := args[1].(common.Address)
amount    := args[2].(*big.Int)
...
from  := sdk.AccAddress(sender.Bytes())
to    := sdk.AccAddress(recipient.Bytes())
if err := bc.checkBlockedAddr(to); err != nil { return nil, err }
denom := EVMDenom(contract.Caller())    // "evm/<calling_contract>"
amt   := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
    ...
    return bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
})
```

The only guards present are:
- `readonly` check
- `amount > 0`
- `checkBlockedAddr(to)` — only blocks module accounts as recipients

There is **no check** that `contract.Caller() == sender`, and no allowance/approval mechanism. `bc.bankKeeper.SendCoins` is called with an arbitrary `from` address supplied entirely by the attacker.

The denom is `EVMDenom(contract.Caller())` = `"evm/" + callerContractHex`. Any user who holds tokens of that denom is vulnerable to having them transferred away by the issuing contract at any time.

The `mint` method (same switch block, lines 113–156) also has no authorization check on the target address, meaning a contract can freely mint its own denom to any address, setting up victims before executing the theft.

The integration test file (`integration_tests/test_bank_precompiles.py`) is entirely skipped at module level (`pytest.skip(...)`), so this code path has no live test coverage.

---

### Impact Explanation

**Critical — Unauthorized transfer of `evm/<contract>` denominated assets.**

A malicious contract can:
1. Call `bankPrecompile.mint(victim, N)` to credit victims with `evm/<attacker_contract>` tokens (e.g., as fake DeFi rewards or liquidity receipts).
2. At any later time call `bankPrecompile.transfer(victim, attacker, N)` to drain those tokens from the victim's Cosmos SDK bank balance to the attacker's address — with no approval from the victim and no on-chain signal that the theft is occurring.

Because `evm/<contract>` denoms are first-class Cosmos SDK bank coins, they can be used in IBC transfers, staking, governance, or any other bank-module-aware flow. Theft of these balances is a direct, irreversible accounting change.

---

### Likelihood Explanation

Any unprivileged EOA can deploy a contract and immediately exploit this. The only precondition is that victims hold `evm/<attacker_contract>` tokens, which the attacker can arrange by calling `mint` first. No admin keys, governance votes, or validator compromise are required. The attack is fully self-contained within a single EVM transaction.

---

### Recommendation

1. **Enforce caller == sender**: Add a guard at the top of the `TransferMethodName` case:
   ```go
   if contract.Caller() != sender {
       return nil, errors.New("transfer: caller is not the sender")
   }
   ```
2. **Alternatively, implement an allowance model**: Allow contracts to call `transfer` on behalf of a third-party address only if that address has previously approved the calling contract via an `approve(spender, amount)` call stored in the precompile's state.
3. **Add live integration tests** for the bank precompile (the test file is currently fully skipped).

```solidity
interface IBankModule {
    function mint(address, uint256) external payable returns (bool);
    function transfer(address, address, uint256) external payable returns (bool);
}

contract BankPrecompileTheft {
    IBankModule constant bank = IBankModule(address(0x64)); // precompile at 0x64

    // Step 1: attacker mints evm/<this> tokens to victim
    function seedVictim(address victim, uint256 amount) external {
        bank.mint(victim, amount);
    }

    // Step 2: attacker drains victim's evm/<this> balance — no victim approval needed
    function steal(address victim, address attacker, uint256 amount) external {
        bank.transfer(victim, attacker, amount);  // succeeds with no auth check
    }
}
```

After `seedVictim(victim, 1000)`, the victim holds 1000 `evm/<BankPrecompileTheft>` tokens in the Cosmos bank. After `steal(victim, attacker, 1000)`, those tokens are transferred to the attacker. The victim has no recourse.
