### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Consume Per-Block Reward Slot - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the `StakingRewardsManagerImpl` is documented to be callable **only by the starkware sequencer**, but the implementation contains **no caller check**. Because a single global `last_reward_block` lock allows only one successful call per block, any unprivileged address can consume the reward slot for a given block, permanently denying all other stakers their block rewards for that block.

### Finding Description
The specification for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1449–1507 performs no `get_caller_address()` check whatsoever:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no caller identity check ...
``` [1](#0-0) 

The `last_reward_block` field is a **global** (not per-staker) storage slot. Once any call to `update_rewards` succeeds in a given block, `last_reward_block` is updated to that block number: [2](#0-1) 

All subsequent calls in the same block revert with `REWARDS_ALREADY_UPDATED`. This is confirmed by the spec:

> **pre-condition**: Rewards did not distributed for the current block yet. [3](#0-2) 

The analog to the external report is direct: just as `foreclose` and `claimProtection` were supposed to be callable by `RemoteProtocolProxy` but the check was missing, `update_rewards` is supposed to be callable only by the sequencer but the check is entirely absent.

### Impact Explanation
An attacker who calls `update_rewards(staker_address: victim_staker, disable_rewards: false)` at the start of any block:

1. Consumes the single per-block reward slot (`last_reward_block` is set to the current block).
2. Prevents the legitimate sequencer from calling `update_rewards` for **any other staker** in that block — those stakers' block rewards are never minted or credited.
3. Block rewards that are not distributed in their block are **permanently lost** — they do not accumulate for future distribution.

If the attacker is themselves a staker, they can call `update_rewards` for their own `staker_address` every block, ensuring they receive their proportional block rewards while all other stakers receive nothing. This constitutes **permanent freezing of unclaimed yield** for all non-favored stakers.

### Likelihood Explanation
The entry path requires no privilege, no special setup, and no capital. Any EOA or contract can call `update_rewards` with an arbitrary `staker_address` (as long as that staker exists and is active). The attack can be sustained indefinitely at the cost of a single transaction per block.

### Recommendation
Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured starkware sequencer address (stored in contract storage), consistent with the access control stated in the specification. This mirrors the pattern already used for `update_rewards_from_attestation_contract`: [4](#0-3) 

### Proof of Concept
The existing test `test_update_rewards_only_staker` (line 3488) already demonstrates the absence of any caller restriction — it calls `update_rewards` from the default (unprivileged) test address with no `cheat_caller_address` setup and succeeds: [5](#0-4) 

An attacker PoC would be:
```cairo
// Attacker calls at the start of every block, consuming the reward slot
// for their own staker_address, denying all other stakers block rewards.
staking_rewards_dispatcher.update_rewards(
    staker_address: attacker_staker_address,
    disable_rewards: false
);
// Sequencer's subsequent call for any other staker now reverts with REWARDS_ALREADY_UPDATED.
```

fn test_update_rewards_only_staker() {
    let mut cfg: StakingInitConfig = Default::default();
    general_contract_system_deployment(ref :cfg);
    let staking_contract = cfg.test_info.staking_contract;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    let staking_rewards_dispatcher = IStakingRewardsManagerDispatcher {
        contract_address: staking_contract,
    };
    let staking_config_dispatcher = IStakingConfigDispatcher { contract_address: staking_contract };
    let minting_curve_contract = cfg.reward_supplier.minting_curve_contract;
    advance_epoch_global();
    let current_epoch = staking_dispatcher.get_current_epoch();
    cheat_caller_address_once(
        contract_address: staking_contract, caller_address: cfg.test_info.app_governor,
    );
    staking_config_dispatcher.set_consensus_rewards_first_epoch(epoch_id: current_epoch + 2);
    // Advance `K` epochs to start consensus rewards.
    advance_k_epochs_global();
    stake_for_testing_using_dispatcher(:cfg);
    advance_k_epochs_global();
    let staker_address = cfg.test_info.staker_address;
    let staker_info_before = staking_dispatcher.staker_info_v1(:staker_address);
    let (strk_block_rewards, _) = calculate_current_block_rewards_v3(:minting_curve_contract);
    let staker_info_expected = StakerInfoV1 {
        unclaimed_rewards_own: strk_block_rewards, ..staker_info_before,
    };
    let mut spy = snforge_std::spy_events();
    staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: false);
    let staker_info_after = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info_after == staker_info_expected);
    // Validate StakerRewardsUpdated event.
    let events = spy.get_events().emitted_by(contract_address: staking_contract).events;
    assert_number_of_events(actual: events.len(), expected: 1, message: "update_rewards");
    assert_staker_rewards_updated_event(
        spied_event: events[0],
        :staker_address,
        staker_rewards: strk_block_rewards,
        pool_rewards: [].span(),
    );
}
