### Title
Unimplemented `TurnBridge` Handler Bypasses Authorization and Silently Discards Bridge-Disable Intent — (`x/cronos/keeper/msg_server.go`)

### Summary
`msgServer.TurnBridge` is a stub that returns `(nil, nil)` unconditionally. Any sender — privileged or not — receives a success response. No permission check against `CanTurnBridge` or `CronosAdmin` is performed, and no bridge state is written. The gravity bridge circuit breaker is permanently broken.

### Finding Description
The `MsgTurnBridge` message is fully registered in the gRPC router and reachable via a signed transaction. Its handler:

```go
// x/cronos/keeper/msg_server.go:85-87
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

Compare with `UpdateTokenMapping`, which correctly gates on `HasPermission`: [5](#0-4) 

`HasPermission` checks both the `CronosAdmin` param and the per-address `CanTurnBridge` bit: [6](#0-5) 

Neither check is present in `TurnBridge`. There is no ante handler intercepting `MsgTurnBridge` before it reaches the msg server. [1](#0-0) 

### Impact Explanation
Two simultaneous High impacts:

1. **Authorization bypass**: Any unprivileged address can submit `MsgTurnBridge` and receive a `code: 0` success response. The `CanTurnBridge` permission system and `CronosAdmin` guard are entirely bypassed for this message.

2. **Broken circuit breaker**: Even the legitimate admin or a permissioned address cannot disable the gravity bridge through this message — the state write is missing. In an emergency (e.g., a bridge exploit), the intended kill-switch silently no-ops, leaving the bridge permanently active.

Both map to the High scope: *"Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks"* and *"Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows."*

### Likelihood Explanation
The message is publicly routable via CLI (`CmdTurnBridge`) and gRPC. No special access is required. The integration test `test_gravity_turn_bridge` explicitly expects `turn_bridge("false", from_="community")` to fail with a non-zero code — this test would pass on-chain only if the permission check were present, confirming the stub is a regression. [7](#0-6) 

### Recommendation
Implement `TurnBridge` following the same pattern as `UpdateTokenMapping`:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // write bridge-enabled param to state
    params := k.GetParams(ctx)
    params.EnableGravity = msg.Enable   // or equivalent field
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

```go
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
