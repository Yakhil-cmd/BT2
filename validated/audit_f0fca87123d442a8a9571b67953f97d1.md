### Title
Unprivileged Caller Can Permanently Suppress Consensus Rewards for All Stakers via `update_rewards(disable_rewards=true)` - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is publicly callable with no access control beyond a non-zero-caller check. Any address can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` sentinel to the current block while skipping all reward computation. Because `last_reward_block` is a single protocol-wide variable, one such call per block permanently destroys that block's consensus rewards for every staker in the protocol.

### Finding Description
`update_rewards` is exposed as a public ABI entry point via `#[abi(embed_v0)]` on `StakingRewardsManagerImpl`. Its only gate is `general_prerequisites`, which checks that the contract is not paused and that the caller is non-zero. [1](#0-0) 

The function accepts a caller-controlled `disable_rewards: bool` parameter. When `true`, execution returns immediately after writing `last_reward_block`: [2](#0-1) 

`last_reward_block` is a single global storage slot: [3](#0-2) 

The guard at the top of the function asserts `current_block_number > last_reward_block`: [4](#0-3) 

Once an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block `N`, `last_reward_block` is set to `N`. Every subsequent legitimate call in block `N` reverts with `REWARDS_ALREADY_UPDATED`. The block's rewards are computed per-block and are not accumulated for future blocks: [5](#0-4) 

The block rewards are fetched from `calculate_block_rewards` which returns the rewards for the current epoch's block. There is no carry-forward mechanism — rewards not distributed in block `N` are permanently lost. [6](#0-5) 

### Impact Explanation
An attacker calling `update_rewards(any_staker, true)` once per block permanently destroys that block's STRK and BTC consensus rewards for **all** stakers and all delegation pools. Sustained over time this constitutes permanent freezing (and effective theft) of unclaimed yield across the entire protocol. This matches the allowed High impact: *Permanent freezing of unclaimed yield*.

### Likelihood Explanation
The attack requires no special role, no staked capital, and no leaked key — only the ability to submit one transaction per block. Gas cost is minimal. The attacker needs only a valid (non-zero, non-paused) staker address to pass the `internal_staker_info` lookup, which is trivially obtained from on-chain events. Likelihood is **High**.

### Recommendation
Add an access-control check to `update_rewards` so that only the attestation contract, a designated rewards-manager role, or the staker themselves can invoke it. Alternatively, remove the `disable_rewards` parameter from the public ABI and handle the pre-consensus short-circuit internally, or gate the `disable_rewards=true` path behind `only_security_agent` / `only_app_governor`.

### Proof of Concept

1. Attacker observes that staker `S` is registered and active.
2. At every new block `N`, attacker calls:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `general_prerequisites` passes (contract not paused, caller non-zero).
4. `current_block_number (N) > last_reward_block` passes on the first call of the block.
5. `last_reward_block` is written to `N`; execution returns before any reward math.
6. Any legitimate call to `update_rewards` for any staker in block `N` now reverts with `REWARDS_ALREADY_UPDATED`.
7. Block `N`'s STRK and BTC rewards are permanently lost for all stakers and pools.
8. Repeat every block — zero cost beyond gas, zero privileges required. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

        fn calculate_block_rewards(
            ref self: ContractState,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) -> (Amount, Amount) {
            if curr_epoch > self.last_calculated_epoch.read() {
                self.last_calculated_epoch.write(curr_epoch);
                let block_rewards = reward_supplier_dispatcher.update_current_epoch_block_rewards();
                self.block_rewards.write(block_rewards);
                block_rewards
            } else {
                self.block_rewards.read()
            }
        }
