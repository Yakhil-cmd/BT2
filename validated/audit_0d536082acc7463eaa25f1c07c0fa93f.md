### Title
Missing Participant-Status Validation in Reshare Allows New Participant to Permanently Deny Reshare for Honest Parties - (File: src/dkg.rs)

### Summary
`assert_reshare_keys_invariants` in `src/dkg.rs` validates that old participants supply a signing key, but contains no symmetric check that **new** participants (absent from `old_participants`) must supply `None`. A malicious new joiner can pass any non-`None` `old_signing_key`, causing `do_reshare` to abort before transmitting a single protocol message. Every honest participant then blocks indefinitely inside `recv_from_others`, permanently stalling the reshare round.

### Finding Description

`assert_reshare_keys_invariants` enforces only one direction of the old/new participant invariant:

```rust
// src/dkg.rs:662-667
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
```

The complementary guard — rejecting a new participant who supplies a non-`None` key — is absent. [1](#0-0) 

When `reshare` (the public entry point) calls `do_reshare` with this unchecked input, the async body immediately computes:

```rust
// src/dkg.rs:611-620
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(<C::Group as Group>::Field::zero);
``` [2](#0-1) 

Because the new participant `me` is not in `intersection`, `intersection.lagrange::<C>(me)` calls `compute_lagrange_coefficient`, which explicitly returns `Err(ProtocolError::InvalidInterpolationArguments)` when the queried point is absent from the identifier set:

```rust
// src/crypto/polynomials.rs:437-440
if !contains_i {
    return Err(ProtocolError::InvalidInterpolationArguments);
}
``` [3](#0-2) 

This error propagates through `.transpose()?`, causing `do_reshare` to return before `do_keyshare` is ever reached — and therefore before any network messages are sent. [4](#0-3) 

The inner guard inside `assert_keyshare_inputs` (which would also catch a non-zero secret for a non-member) is never reached because the failure occurs upstream in `do_reshare`. [5](#0-4) 

### Impact Explanation

All honest participants enter `do_keyshare` and block in `recv_from_others` waiting for the round-1 broadcast from every peer, including the malicious new joiner. Because the malicious participant's async task terminates with an error before emitting any message, honest participants never receive the expected message and stall indefinitely. The reshare round cannot complete; no new key shares are produced. [6](#0-5) 

This matches **High: Permanent denial of reshare for honest parties** — the current reshare attempt is irrecoverably aborted, and honest parties must restart the entire ceremony after identifying and excluding the malicious joiner.

### Likelihood Explanation

The `reshare` public API accepts `old_signing_key: Option<SigningShare<C>>` directly from the caller. Any new participant (one not present in `old_participants`) can trivially pass `Some(arbitrary_share)`. No cryptographic capability, leaked material, or privileged access is required — only the ability to call the library's own public function with a non-`None` argument. [7](#0-6) 

### Recommendation

Add the symmetric check to `assert_reshare_keys_invariants`:

```rust
// After the existing check at line 663
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not present in the old participant list but provided a share"
    )));
}
```

This mirrors the existing guard and ensures the invalid state is rejected at the public API boundary with a clear error, before any async protocol work begins. [8](#0-7)

pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],
) -> Result<(ParticipantList, ParticipantList), InitializationError> {
    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

    let participants = assert_key_invariants(participants, me, threshold)?;

    let old_participants =
        ParticipantList::new(old_participants).ok_or(InitializationError::DuplicateParticipants)?;

    // Step 1.1
    if old_participants.intersection(&participants).len() < old_threshold {
        return Err(InitializationError::NotEnoughParticipantsForNewThreshold {
            threshold: old_threshold,
            participants: old_participants.intersection(&participants).len(),
        });
    }
    // Step 1.1
    // if me is not in the old participant set then ensure that old_signing_key is None
    if old_participants.contains(me) && old_signing_key.is_none() {
        return Err(InitializationError::BadParameters(format!(
            "party {me:?} is present in the old participant list but provided no share"
        )));
    }
    Ok((participants, old_participants))
}

pub fn reshare<C: Ciphersuite>(
    old_participants: &[Participant],
    old_threshold: impl Into<ReconstructionLowerBound> + Send + 'static,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    new_participants: &[Participant],
    new_threshold: impl Into<ReconstructionLowerBound> + Copy + Send + 'static,
    me: Participant,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = KeygenOutput<C>>, InitializationError>
where
    Element<C>: Send,
    Scalar<C>: Send,
{
    let comms = Comms::new();
    let threshold = new_threshold;
    let (participants, old_participants) = assert_reshare_keys_invariants::<C>(
        new_participants,
        me,
        threshold,
        old_signing_key,
        old_threshold,
        old_participants,
    )?;
    let fut = do_reshare(
        comms.shared_channel(),
        participants,
        me,
        threshold,
        old_signing_key,
        old_public_key,
        old_participants,
        rng,
    );
    Ok(make_protocol(comms, fut))
}
