### Title
Unprivileged Caller Can Permanently Suppress Staker Block Rewards via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` has no caller restriction and accepts a caller-controlled `disable_rewards: bool` parameter. Any unprivileged address can call it with `disable_rewards: true` for any staker, permanently consuming the per-block reward slot without distributing rewards, because the function unconditionally advances `last_reward_block` before checking the flag.

---

### Finding Description

`update_rewards` is the consensus-phase per-block reward distribution entry point. Its implementation in `StakingRewardsManagerImpl` follows this sequence:

1. **No caller check** — `general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero; it does not restrict who may call the function.
2. **One-call-per-block gate** — asserts `current_block_number > self.last_reward_block.read()`.
3. **Unconditionally advances the gate** — `self.last_reward_block.write(current_block_number)` executes before the `disable_rewards` branch.
4. **Early return suppresses rewards** — `if disable_rewards || self.is_pre_consensus() { return; }` exits without distributing anything. [1](#0-0) 

Because `last_reward_block` is written at step 3, any subsequent legitimate call in the same block fails the gate at step 2 with `REWARDS_ALREADY_UPDATED`. The staker's block rewards for that block are permanently lost. [2](#0-1) [3](#0-2) 

The interface confirms the function is public with no documented caller restriction: [4](#0-3) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who front-runs the legitimate `update_rewards` call every block for a target staker causes that staker to accrue zero consensus block rewards indefinitely. The rewards are not deferred; they are simply never computed or credited. Because `last_reward_block` is a single global slot shared across all stakers (not per-staker), a single attacker transaction per block is sufficient to suppress rewards for every staker simultaneously if called once per block. [5](#0-4) 

---

### Likelihood Explanation

**High.** The function is permissionlessly callable by any EOA or contract. The only cost to the attacker is gas per block. No privileged access, leaked key, or external dependency is required. The attacker has no profit motive but can cause sustained, irreversible yield loss to all stakers.

---

### Recommendation

Restrict who may supply `disable_rewards: true`. Options:
- Gate the entire function to a trusted caller (e.g., the attestation contract or a designated consensus reporter).
- Split into two functions: a permissionless `update_rewards(staker_address)` that always distributes, and a privileged `update_rewards_no_distribution(staker_address)` callable only by the protocol.
- Move `last_reward_block.write` to after the `disable_rewards` branch so that a suppressed call does not consume the block slot.

---

### Proof of Concept

```
Block N arrives.

Attacker calls:
  staking.update_rewards(victim_staker, disable_rewards: true)

Execution:
  1. general_prerequisites() passes (attacker is non-zero, contract unpaused).
  2. current_block_number (N) > last_reward_block (N-1) → passes.
  3. last_reward_block.write(N)  ← slot consumed.
  4. disable_rewards == true → return (no rewards distributed).

Legitimate caller (validator / anyone) calls:
  staking.update_rewards(victim_staker, disable_rewards: false)

Execution:
  2. current_block_number (N) > last_reward_block (N) → FAILS with REWARDS_ALREADY_UPDATED.

Result: victim_staker receives zero block rewards for block N.
Repeated every block → permanent freezing of all consensus block rewards.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
