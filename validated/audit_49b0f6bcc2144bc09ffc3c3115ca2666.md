### Title
Unrestricted `update_rewards` with `disable_rewards=true` Permanently Freezes Per-Block Yield for All Stakers - (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` in the Staking contract is callable by any unprivileged address and accepts a caller-controlled `disable_rewards` boolean. When `disable_rewards=true`, the function updates the global `last_reward_block` counter to the current block number but returns early without distributing any rewards. Because `last_reward_block` is a global gate that allows only one successful `update_rewards` call per block, an attacker who calls this function first in every block permanently prevents all stakers from receiving consensus block rewards for those blocks.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

There is no role check or restriction on who may call it. The function accepts a `staker_address` (which must be a valid, active staker — public information from emitted events) and a `disable_rewards: bool` parameter fully controlled by the caller.

Inside the function, `last_reward_block` is written unconditionally before the `disable_rewards` branch: [2](#0-1) 

When `disable_rewards=true`, the function returns immediately after writing `last_reward_block`, skipping the entire `_update_rewards` call: [3](#0-2) 

The block-uniqueness guard at the top of the function means only one call per block can succeed: [4](#0-3) 

Because `last_reward_block` is a single global storage slot (not per-staker), a single attacker transaction per block is sufficient to consume the slot for the entire protocol, blocking every staker's legitimate reward update for that block.

---

### Impact Explanation

When the attacker calls `update_rewards(any_valid_staker, disable_rewards=true)` at the start of every block:

1. `last_reward_block` is set to the current block number.
2. No rewards are calculated or credited to any staker.
3. All subsequent legitimate calls to `update_rewards` in the same block revert with `REWARDS_ALREADY_UPDATED`.
4. The missed block rewards are **permanently lost** — there is no catch-up mechanism; `last_reward_block` is never rolled back.

This constitutes **permanent freezing of unclaimed yield** for all stakers in the consensus rewards phase, matching the High impact tier.

---

### Likelihood Explanation

- The entry point is fully public; no privileged role or leaked key is required.
- The attacker only needs to know one valid, active staker address, which is trivially obtained from `NewStaker` events or the public `stakers` vector.
- On Starknet, transaction fees are low, making sustained per-block griefing economically viable.
- The attack is silent — it produces no revert, only a missing reward event.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the attestation contract, a designated sequencer address, or the staker/operational address themselves), or remove the `disable_rewards` parameter from the public interface and handle that logic internally. At minimum, `last_reward_block` must not be written before the caller's authorization and the `disable_rewards` flag are both validated.

---

### Proof of Concept

```
// Attacker script (pseudocode, one call per block)
loop every block:
    staking.update_rewards(
        staker_address = <any known active staker>,
        disable_rewards = true
    )
    // last_reward_block is now set to current block
    // no rewards distributed
    // all legitimate update_rewards calls this block revert
```

The staker's `unclaimed_rewards_own` is never incremented for any block where the attacker front-runs with `disable_rewards=true`, and the loss is irrecoverable because `last_reward_block` is never decremented or replayed. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
