### Title
Missing `msg_hash != 0` Validation in OT-Based ECDSA `sign()` Enables Presign Secret Share Disclosure to Malicious Coordinator - (File: src/ecdsa/ot_based_ecdsa/sign.rs)

### Summary
The OT-based ECDSA `sign()` function accepts a zero `msg_hash` without rejection. When `msg_hash = 0`, each participant's signature share degenerates to `s_i = r * sigma_i`, directly exposing their secret presign share `sigma_i` (a share of `sigma = k * x`) to the coordinator. The robust ECDSA `sign()` function explicitly rejects `msg_hash = 0`, but the OT-based variant has no equivalent guard.

### Finding Description

The OT-based ECDSA `sign()` function in `src/ecdsa/ot_based_ecdsa/sign.rs` performs several input validations (participant count, duplicates, coordinator membership, threshold) but contains no check that `msg_hash` is non-zero: [1](#0-0) 

Each participant computes their signature share via `compute_signature_share()`:

```rust
// Spec 1.3
let r = x_coordinate(&presignature.big_r);
Ok(msg_hash * k_i + r * sigma_i)
``` [2](#0-1) 

When `msg_hash = 0`, this collapses to:

```
s_i = 0 * k_i + r * sigma_i  =  r * sigma_i
```

Since `r = x_coordinate(big_r)` is a public value present in the `RerandomizedPresignOutput`, the coordinator can trivially recover each participant's secret presign share:

```
sigma_i = s_i / r
```

The participant sends this value unconditionally to the coordinator via `chan.send_private()`: [3](#0-2) 

By contrast, the robust ECDSA `sign()` function explicitly rejects `msg_hash = 0`: [4](#0-3) 

The `sigma_i` values are secret shares of `sigma = k * x` (nonce times private key), classified as presign secrets. They are never supposed to leave the participant's local state in recoverable form.

### Impact Explanation

A malicious coordinator calls `ot_based_ecdsa::sign()` with `msg_hash = Scalar::ZERO`. All honest participants compute `s_i = r * sigma_i` and send it privately to the coordinator. The coordinator divides each received value by the public `r` to obtain every participant's `sigma_i`.

With all `sigma_i` shares the coordinator reconstructs `sigma = k * x`. This is a direct disclosure of presign secret material. Additionally, if the coordinator can induce presignature reuse (a separately documented risk), they can combine `sigma = k * x` with a second signing session `s' = k*(h' + r*x)` to solve for both `k` and `x`, achieving full private key extraction.

**Impact category**: Critical — disclosure of presign secrets / nonce material (`sigma_i` shares).

### Likelihood Explanation

The coordinator is a participant in the protocol and controls which `msg_hash` value is passed to `sign()`. No special privilege beyond being the designated coordinator is required. The coordinator simply passes `Scalar::ZERO` as `msg_hash`. All honest participants will comply without error, as there is no guard at the participant level either. The attack is single-round and requires no cryptographic capability.

### Recommendation

Add a zero-check on `msg_hash` in `ot_based_ecdsa::sign()`, mirroring the existing guard in `robust_ecdsa::sign()`:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0: it leaks presign secret shares to the coordinator".to_string(),
    ));
}
``` [5](#0-4) 

This guard should be placed alongside the existing threshold check, before the protocol future is constructed.

### Proof of Concept

1. Run a normal DKG and presign to obtain a `RerandomizedPresignOutput` for each participant. Let `r = x_coordinate(presignature.big_r)` (public).
2. Coordinator calls `ot_based_ecdsa::sign(participants, coordinator, threshold, me, public_key, presignature, Scalar::ZERO)`.
3. Each participant executes `compute_signature_share()` and sends `s_i = r * sigma_i` to the coordinator.
4. Coordinator computes `sigma_i = s_i * r.invert()` for each received share.
5. Coordinator interpolates `sigma = sum(lambda_i * sigma_i)` — the full secret `k * x` — without any cryptographic hardness assumption.

The `verify()` call at the end of `do_sign_coordinator` passes because the equation `s^{-1} * r * X = R` holds with `h = 0`, so no error is raised and the leakage is silent: [6](#0-5) [7](#0-6)

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

    pub fn verify(&self, public_key: &AffinePoint, msg_hash: &Scalar) -> bool {
        let r: Scalar = x_coordinate(&self.big_r);
        if r.is_zero().into() || self.s.is_zero().into() {
            return false;
        }
        // Check if s has been normalized
        if self.s.is_high().into() {
            return false;
        }
        // tested earlier is not zero, so inversion will not raise an error and unwrap cannot panic
        let s_inv = self.s.invert_vartime().unwrap();
        let reproduced = (ProjectivePoint::GENERATOR * (*msg_hash * s_inv))
            + (ProjectivePoint::from(*public_key) * (r * s_inv));
        x_coordinate(&reproduced.into()) == r
    }
