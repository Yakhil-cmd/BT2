### Title
Unprivileged Caller Can Frontrun `update_rewards` With `disable_rewards: true` to Suppress Consensus Rewards for All Stakers — (File: `src/staking/staking.cairo`)

---

### Summary

The public `update_rewards` function accepts a caller-controlled `disable_rewards` boolean flag with no access control. When any unprivileged address calls it with `disable_rewards: true`, the global `last_reward_block` checkpoint is advanced to the current block number **without distributing any rewards**. Because `last_reward_block` is a single shared slot, this consumes the one-per-block reward slot for every staker simultaneously. An attacker can frontrun every legitimate `update_rewards` call — or simply call it first each block — to permanently suppress consensus reward accrual across the entire protocol.

---

### Finding Description

`update_rewards` is part of the `IStakingRewardsManager` interface and is gated only by `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero. No role or identity check restricts who may pass `disable_rewards: true`. [1](#0-0) 

Inside the function, `last_reward_block` is written unconditionally **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

The early return when `disable_rewards` is true means no reward calculation or distribution occurs, yet the block slot is permanently consumed: [3](#0-2) 

`last_reward_block` is a single global storage variable — not per-staker — so one call with any `staker_address` and `disable_rewards: true` blocks all stakers from receiving rewards for that block: [4](#0-3) 

The constant `K = 2` means balance changes take effect two epochs in the future, so there is no way for stakers to retroactively recover the skipped block's rewards. [5](#0-4) 

---

### Impact Explanation

Every block in which the attacker calls `update_rewards(any_active_staker, true)` first produces zero consensus rewards for the entire staker set. Sustained over many blocks this constitutes a **permanent freezing of unclaimed yield** for all stakers and their delegators. Even a partial campaign (e.g., targeting high-value epochs) constitutes griefing with direct financial damage to users and the protocol.

Matches allowed impact: **Griefing with no profit motive but damage to users or protocol (Medium)** / borderline **Permanent freezing of unclaimed yield (High)** depending on attacker persistence.

---

### Likelihood Explanation

- The entry point is fully public; no special role or key is required.
- On Starknet, transaction fees are low, making sustained per-block calls economically feasible.
- The attacker only needs to submit one transaction per block ahead of any legitimate `update_rewards` call — a straightforward mempool-watching strategy.
- The attack is amplified because a single call suppresses rewards for **all** stakers simultaneously, maximising damage per unit of attacker cost.

**Likelihood: Medium** (requires sustained effort but is technically trivial and cheap on Starknet).

---

### Recommendation

1. **Restrict `disable_rewards`**: Gate the `disable_rewards: true` path behind a privileged role (e.g., `only_security_agent`) or remove the parameter from the public ABI entirely, exposing it only as an internal function.
2. **Separate the checkpoint write**: Only advance `last_reward_block` when rewards are actually computed and distributed, so a no-op call cannot consume the block slot.
3. **Per-staker reward tracking**: Consider tracking `last_reward_block` per staker rather than globally, so a call for one staker cannot block others.

---

### Proof of Concept

```
Block N:
  Attacker observes mempool: staker S is about to call update_rewards(S, false)
  Attacker submits: update_rewards(S, true)   ← higher gas tip, lands first

  Result inside update_rewards:
    last_reward_block.write(N)   ← slot consumed
    if disable_rewards { return; }  ← no rewards distributed

  Staker S's legitimate call in same block:
    assert!(N > last_reward_block.read())  ← FAILS: N == N
    → reverts with REWARDS_ALREADY_UPDATED

Block N+1, N+2, … : attacker repeats.
All stakers accrue zero consensus rewards indefinitely.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
