### Title
STRK Rewards Permanently Locked in Pool Contract When Pool Balance Falls Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo)

### Summary
When a delegation pool's total balance is below `min_delegation_for_rewards`, the staking contract still physically transfers STRK reward tokens to the pool contract, but the pool's `compute_rewards_per_unit` returns zero, leaving those tokens permanently unclaimable by any delegator. There is no sweep or recovery function in the pool contract.

### Finding Description

The `Pool` contract's `compute_rewards_per_unit` function explicitly returns zero when `total_stake < min_delegation_for_rewards`: [1](#0-0) 

The inline comment on that function acknowledges the problem directly:

> "Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case."

The staking contract's `_update_rewards` flow:

1. Calls `claim_from_reward_supplier` to pull `total_pools_rewards` STRK tokens into the staking contract.
2. Calls `update_pool_rewards`, which calls `send_rewards_to_delegation_pool` to physically transfer those tokens to the pool contract address.
3. Then calls `pool_dispatcher.update_rewards_from_staking_contract(rewards, pool_balance)`. [2](#0-1) 

Inside `update_rewards_from_staking_contract`, the pool appends `compute_rewards_per_unit(staking_rewards, pool_balance)` to its `cumulative_rewards_trace`. When `pool_balance < min_delegation_for_rewards`, this appends **zero**, so the trace does not advance: [3](#0-2) 

Because `claim_rewards` distributes only what the cumulative trace records, the STRK tokens that were physically deposited into the pool contract are never attributed to any delegator and can never be withdrawn. The pool contract has no sweep, admin-recovery, or fallback function. [4](#0-3) 

The `min_delegation_for_rewards` threshold for STRK pools is `10^18` (1 STRK): [5](#0-4) 

The same mechanism applies to BTC pools, confirmed by the `PoolWithMinBtcFlow` test which shows `pool_rewards.is_zero()` while `staker_rewards.is_non_zero()` when pool balance is one unit below the threshold. [6](#0-5) 

### Impact Explanation

STRK reward tokens are physically transferred to the pool contract but are permanently unclaimable. No delegator can retrieve them via `claim_rewards`, `exit_delegation_pool_action`, or `switch_delegation_pool` (all of which operate on the cumulative trace or the principal balance, not the raw token balance). There is no governance or admin recovery path. This constitutes **permanent freezing of unclaimed yield**.

### Likelihood Explanation

Any pool whose total delegated balance is non-zero but below 1 STRK (e.g., a single delegator who deposited 0.5 STRK, or a pool where most delegators exited leaving a dust balance) will silently lock every future reward distribution. No special privilege is required: `enter_delegation_pool` only checks `amount.is_non_zero()`, so any unprivileged delegator can create this condition. The staker's normal attestation or `update_rewards` call then triggers the irreversible lock. [7](#0-6) 

### Recommendation

Two complementary fixes should be applied:

1. **Do not transfer rewards to the pool when `pool_balance < min_delegation_for_rewards`**: In `update_pool_rewards` (staking contract), skip the `send_rewards_to_delegation_pool` call and redirect those rewards back to the reward supplier or to the staker's own unclaimed balance when the pool balance is below the threshold.

2. **Add a recovery path in the pool contract**: If the design intentionally allows small-balance pools to receive tokens, add an admin or governance-callable function to recover the surplus STRK balance (i.e., `token.balance_of(this) - tracked_rewards`) so it is not permanently frozen.

### Proof of Concept

1. Deploy the full system (staking, pool, reward supplier, minting curve).
2. Staker stakes and enables a STRK delegation pool.
3. Delegator calls `enter_delegation_pool` with `amount = 1` (well below `min_delegation_for_rewards = 10^18`).
4. Advance K epochs and trigger attestation (or call `update_rewards`).
5. Observe: STRK tokens equal to the pool's share of block rewards are transferred to the pool contract (`token.balance_of(pool) > 0`).
6. Delegator calls `claim_rewards` → returns 0.
7. Delegator calls `exit_delegation_pool_intent` + `exit_delegation_pool_action` → receives only the 1-token principal.
8. `token.balance_of(pool)` remains non-zero with no callable function able to drain it.

The STRK reward tokens are permanently locked in the pool contract.

            ref self: ContractState, reward_address: ContractAddress, amount: Amount,

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        /// Compute the rewards for the pool trace.
        ///
        /// `staking_rewards` is in `STRK_DECIMALS` decimals.
        /// `total_stake` is in the contract's token decimals.
        /// **Note**: Delegation rewards lost when pool balance is less than
        /// `min_delegation_for_rewards`. The staking contract continues to forward
        /// `pool_rewards` to the pool contract even in this case.
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }

        fn update_pool_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            pools_rewards_data: Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>,
        ) -> Array<(ContractAddress, Amount)> {
            let mut pool_rewards_list = array![];
            let strk_token_dispatcher = strk_token_dispatcher();
            for (pool_contract, token_address, pool_balance, pool_rewards) in pools_rewards_data {
                let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };
                // Rewards are always in STRK.
                self
                    .send_rewards_to_delegation_pool(
                        :staker_address,
                        pool_address: pool_contract,
                        amount: pool_rewards,
                        token_dispatcher: strk_token_dispatcher,
                    );
                let decimals = self.get_token_decimals(:token_address);
                pool_dispatcher
                    .update_rewards_from_staking_contract(
                        rewards: pool_rewards,
                        pool_balance: pool_balance.to_native_amount(:decimals),
                    );
                pool_rewards_list.append((pool_contract, pool_rewards));
            }
            pool_rewards_list
        }
