### Title
Unprivileged Caller Can DOS Staker Reward Distribution via `update_rewards` with `disable_rewards: true` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `staking.cairo` is publicly callable by any address with no access control. An attacker can call it with `disable_rewards: true` for any valid staker, which advances the global `last_reward_block` counter without distributing any rewards. Because `last_reward_block` is a single global slot, this blocks every subsequent `update_rewards` call in the same block, causing the intended staker to lose their block reward. Repeated every block, this permanently freezes unclaimed yield.

---

### Finding Description

`update_rewards` is exposed as a public ABI entry point under `IStakingRewardsManager`:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // only checks: not paused, caller != 0
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // Update last block rewards.
        self.last_reward_block.write(current_block_number);   // ← global write, no per-staker scope

        if disable_rewards || self.is_pre_consensus() {
            return;                                           // ← exits WITHOUT distributing rewards
        }
        ...
        self._update_rewards(...);
    }
}
``` [1](#0-0) 

The critical properties are:

1. **No caller restriction.** `general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero. [2](#0-1) 

2. **`last_reward_block` is a single global slot.** It is not scoped per staker. Once written in a block, the assertion `current_block_number > last_reward_block` fails for every subsequent call in that block. [3](#0-2) 

3. **`disable_rewards: true` writes the block counter but skips reward distribution.** The write to `last_reward_block` happens unconditionally before the early-return branch. [4](#0-3) 

**Attack path:**

```
Block N:
  Attacker tx:  update_rewards(any_valid_active_staker, disable_rewards=true)
                → last_reward_block := N
                → returns early, no rewards distributed

  Victim tx:    update_rewards(victim_staker, disable_rewards=false)
                → assert(N > N) FAILS → REWARDS_ALREADY_UPDATED
                → victim earns nothing for block N
```

---

### Impact Explanation

Each successful attacker call silently discards one block's worth of STRK rewards for the targeted staker and their delegators. Executed every block, this constitutes **permanent freezing of unclaimed yield** for the victim. The attacker gains nothing; the damage is pure griefing. This falls under the **High** allowed impact: *Permanent freezing of unclaimed yield*.

---

### Likelihood Explanation

- The function is unconditionally public; no role or ownership check exists.
- Valid staker addresses are enumerable from the `stakers` vector in storage and from on-chain events.
- The attack costs only the gas of a single contract call per block.
- On Starknet today the sequencer is centralised, but the protocol is designed for decentralisation; once permissionless sequencing is live, reliable transaction ordering by an attacker becomes trivial.
- Even under a centralised sequencer, a malicious or colluding party can execute this without front-running.

---

### Recommendation

1. **Restrict the caller.** Only allow the staker themselves, their operational address, or a designated consensus contract to call `update_rewards`. Add an assertion such as:
   ```cairo
   assert!(
       get_caller_address() == staker_address
           || get_caller_address() == staker_info.operational_address,
       "{}",
       Error::UNAUTHORIZED_CALLER,
   );
   ```
2. **Scope `last_reward_block` per staker** rather than as a single global slot, so one staker's update cannot block another's.
3. **Remove or gate `disable_rewards`.** If the parameter is only meaningful for the consensus layer, restrict it to a privileged caller or derive it internally from attestation state rather than accepting it from an arbitrary caller.

---

### Proof of Concept

```
// Setup: staker_A is a valid, active staker with non-zero STRK balance.

// Block N — attacker submits first:
staking_contract.update_rewards(staker_A, disable_rewards=true);
// Effect: last_reward_block = N, no rewards distributed.

// Block N — victim (or consensus layer) submits second:
staking_contract.update_rewards(staker_A, disable_rewards=false);
// Reverts: "Rewards already updated for this block"

// Repeat every block → staker_A and all delegators earn zero rewards indefinitely.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
