### Title
`exit_delegation_pool_action` Callable by Any Address Allows Griefing Pool Members Attempting `switch_delegation_pool` - (File: src/pool/pool.cairo)

### Summary
`Pool::exit_delegation_pool_action` accepts an arbitrary `pool_member` address and carries no caller restriction. Any unprivileged address can invoke it for any pool member whose exit window has elapsed. Because the function is stateful — it clears the pool member's `unpool_time`/`unpool_amount` and consumes the staking-contract intent — a subsequent call to `switch_delegation_pool` by the legitimate pool member will revert with `MISSING_UNDELEGATE_INTENT`, permanently blocking the pool switch for that intent cycle.

### Finding Description

`Pool::exit_delegation_pool_action` (pool.cairo lines 295–333) takes `pool_member: ContractAddress` as a parameter and transfers the exit funds directly to that address, regardless of who the caller is:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    // ...
    staking_pool_dispatcher
        .remove_from_delegation_pool_action(identifier: pool_member.into()); // consumes intent
    pool_member_info.unpool_amount = Zero::zero();
    pool_member_info.unpool_time = Option::None;          // clears state
    self.write_pool_member_info(:pool_member, :pool_member_info);
    token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
    unpool_amount
}
```

The spec confirms: **"Any address can execute."**

The stateful side-effects are:
1. The staking contract's `pool_exit_intents` entry keyed `{pool_contract, pool_member}` is cleared via `remove_from_delegation_pool_action`.
2. The pool contract's `pool_member_info.unpool_time` and `unpool_amount` are zeroed.

A pool member who has called `exit_delegation_pool_intent` and then decides to redirect funds via `switch_delegation_pool` relies on both of those state entries being intact. `Pool::switch_delegation_pool` (pool.cairo) calls `staking.switch_staking_delegation_pool`, which asserts:

```cairo
assert!(
    undelegate_intent_value.is_non_zero(),
    "{}",
    GenericError::MISSING_UNDELEGATE_INTENT,
);
```

If an attacker has already called `exit_delegation_pool_action(pool_member)`, the intent is gone and the pool member's `switch_delegation_pool` reverts.

**Attack path:**
1. Pool member calls `exit_delegation_pool_intent(amount)` — intent created, exit window starts.
2. Exit window elapses.
3. Pool member decides to switch pools and submits `switch_delegation_pool(to_staker, to_pool, amount)`.
4. Attacker (any address) calls `pool.exit_delegation_pool_action(pool_member)` — intent consumed, funds sent to pool_member, state cleared.
5. Pool member's `switch_delegation_pool` reverts: `MISSING_UNDELEGATE_INTENT`.

### Impact Explanation

The pool member's funds are returned to them (no direct theft), but the intended pool switch is permanently blocked for that intent cycle. The pool member must:
- Accept the forced exit and re-stake from scratch (new `stake` call, new delegation, new exit window if they want to switch again).
- Absorb gas costs for the failed transaction and the re-staking sequence.
- Forfeit staking rewards during the forced idle period between exit and re-entry.

This is **griefing with no profit motive but concrete damage to users**, matching the allowed Medium impact: *"Griefing with no profit motive but damage to users or protocol."*

### Likelihood Explanation

The attack requires no privileged access. Any address can call `exit_delegation_pool_action` for any pool member once the exit window has elapsed. On Starknet the sequencer orders transactions, but an attacker watching on-chain state (the `PoolMemberExitIntent` event) can submit the griefing call in the same block or any subsequent block before the victim's switch. The attack is cheap (one function call), repeatable, and requires no capital.

### Recommendation

Restrict `exit_delegation_pool_action` so that only the pool member themselves (or their registered reward address) can trigger the exit action. The "permissionless" design was likely intended to allow automation/bots to help users exit, but it creates a griefing surface. A simple caller check suffices:

```cairo
assert!(
    get_caller_address() == pool_member
        || get_caller_address() == pool_member_info.reward_address,
    "{}",
    Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
);
```

Alternatively, if permissionless execution is desired, add a guard that prevents the action when the pool member has a pending `switch_delegation_pool` in flight (e.g., a flag set during the switch flow).

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }

        fn remove_from_delegation_pool_action(ref self: ContractState, identifier: felt252) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let pool_contract = get_caller_address();
            let undelegate_intent_key = UndelegateIntentKey { pool_contract, identifier };
            let undelegate_intent = self.get_pool_exit_intent(:undelegate_intent_key);
            if undelegate_intent.amount.is_zero() {
                return;
            }
            assert!(
                Time::now() >= undelegate_intent.unpool_time,
                "{}",
                GenericError::INTENT_WINDOW_NOT_FINISHED,
            );

            // Clear the intent.
            self.clear_undelegate_intent(:undelegate_intent_key);
            // Extract the token address of the pool contract.
            let token_address = get_undelegate_intent_token(:undelegate_intent);
            let decimals = self.get_token_decimals(:token_address);
            // Transfer the intent amount to the pool contract.
            let native_amount = undelegate_intent.amount.to_native_amount(:decimals);
            let token_dispatcher = IERC20Dispatcher { contract_address: token_address };
            token_dispatcher
                .checked_transfer(recipient: pool_contract, amount: native_amount.into());

            // Emit event.
            self
                .emit(
                    Events::RemoveFromDelegationPoolAction {
                        pool_contract, token_address, identifier, amount: native_amount,
                    },
                );
        }
