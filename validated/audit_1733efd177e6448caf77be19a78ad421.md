### Title
Staker Can Bait-and-Switch Commission via `set_commission_commitment` to Steal Delegator Yield — (File: src/staking/staking.cairo)

---

### Summary

A staker can attract delegators by advertising 0% commission (a vampire-attack-style lure), then exploit the `set_commission_commitment` mechanism to immediately raise commission to 100%, stealing all future pool rewards from delegators who are locked in by the exit wait window.

---

### Finding Description

The `update_commission` internal function enforces a strict asymmetry:

- **Without an active commitment**: commission can only *decrease*.
- **With an active commitment**: commission can be set to *any value* ≤ `max_commission`, including values higher than the current commission. [1](#0-0) 

The `set_commission_commitment` function requires only that `max_commission >= current_commission` and `expiration_epoch > current_epoch`. There is no minimum delay between setting the commitment and calling `set_commission` to raise the rate. [2](#0-1) 

This creates a two-step bait-and-switch:

1. Staker initialises with 0% commission and opens a delegation pool.
2. Delegators join, attracted by the 0% rate.
3. Staker calls `set_commission_commitment(max_commission=10000, expiration_epoch=current_epoch+2)`.
4. In the same block (or the next), staker calls `set_commission(commission=10000)`.
5. All future pool rewards are now routed entirely to the staker as commission.

Delegators who react and call `exit_delegation_pool_intent` are still locked in for the full `exit_wait_window` (default: one week). [3](#0-2) 

During that window, every reward calculation in `calculate_staker_pools_rewards` reads the *current* commission from storage, so the 100% rate applies immediately to all subsequent reward updates. [4](#0-3) 

The developers acknowledge the gap with the comment: *"Current commission increase safeguards still allow for sudden commission changes"*, but the full economic attack path (attract → lock → drain) is not mitigated. [5](#0-4) 

---

### Impact Explanation

Every delegator in the pool loses 100% of their yield for at least one full exit-wait-window period (≥ 1 week). Because rewards are calculated at the commission rate in effect at the time `_update_rewards` / `update_rewards_from_staking_contract` is called, no past-accrued-but-unclaimed rewards are safe once the commission flips. This maps directly to **theft of unclaimed yield (High)**. [6](#0-5) 

---

### Likelihood Explanation

The attack requires no privileged role — any registered staker with an open pool can execute it. The steps are fully on-chain, atomic, and require no external coordination. The economic incentive is clear: the staker captures the entire pool reward stream for at least one week. Likelihood is **High**. [7](#0-6) 

---

### Recommendation

1. **Enforce a time-lock on commission increases**: when `set_commission_commitment` is called with a `max_commission` higher than the current commission, require that the new rate cannot take effect until at least `exit_wait_window` seconds after the commitment is set, giving delegators time to exit.
2. **Alternatively**, prohibit commission increases entirely (even under an active commitment) and restrict the commitment mechanism to only lowering the ceiling below the current rate.
3. **At minimum**, emit a distinct on-chain event when commission is *raised* (not just changed) so monitoring tools can alert delegators immediately.

---

### Proof of Concept

```
Epoch N:
  staker.stake(reward_address, operational_address, min_stake)
  staker.set_commission(commission=0)          // 0% — attracts delegators
  staker.set_open_for_delegation(STRK)

  delegator_A.enter_delegation_pool(reward_addr, large_amount)
  delegator_B.enter_delegation_pool(reward_addr, large_amount)

  // --- attack ---
  staker.set_commission_commitment(
      max_commission = 10_000,          // 100%
      expiration_epoch = N + 2          // minimum valid value
  )
  staker.set_commission(commission = 10_000)   // 100% — succeeds because commitment is active

Epoch N+1 … N+1+exit_wait_window:
  // Every call to update_rewards / update_rewards_from_attestation_contract
  // routes 100% of pool rewards to the staker as commission.
  // delegator_A and delegator_B earn zero yield.

  delegator_A.exit_delegation_pool_intent(amount)
  // Must wait exit_wait_window (~1 week) before funds are released.
  // All rewards during the wait period are stolen.
``` [8](#0-7)

        /// **Note**: Current commission increase safeguards still allow for sudden commission
        /// changes.
        /// **Note**: Updating epoch info can impact the commission commitment expiration date.
        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
