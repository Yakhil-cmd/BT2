### Title
Staking Contract Pause Check Bypass in Pool's `exit_delegation_pool_intent` When Staker Is Removed - (File: src/pool/pool.cairo)

### Summary

The `DelegationPool` contract's `exit_delegation_pool_intent` function enforces the staking contract's pause check only by delegating to `remove_from_delegation_pool_intent` on the staking contract. However, when the associated staker has been removed (`staker_removed == true`), the internal helper `undelegate_from_staking_contract_intent` returns early with `Time::now()` **without ever calling the staking contract**. This means the `CONTRACT_IS_PAUSED` guard — which the spec explicitly requires — is silently skipped, allowing a pool member to signal an exit intent (and subsequently execute `exit_delegation_pool_action`) while the staking contract is paused.

### Finding Description

The pool contract's `exit_delegation_pool_intent` calls the internal helper `undelegate_from_staking_contract_intent`: [1](#0-0) 

When `!self.is_staker_active()` (i.e., `staker_removed == true`), the function takes the early-return branch: [2](#0-1) 

It returns `Time::now()` immediately, **never invoking** `staking_pool_dispatcher.remove_from_delegation_pool_intent(...)`. The staking contract's pause guard is therefore never reached.

The spec (`docs/spec.md`) explicitly lists `CONTRACT_IS_PAUSED` as a required error for `exit_delegation_pool_intent` and states as a pre-condition: *"Staking contract is unpaused."* [3](#0-2) 

The same bypass applies to `exit_delegation_pool_action`: the comment in the implementation acknowledges that when the staker is removed, the call to `remove_from_delegation_pool_action` on the staking contract "will do nothing," meaning the staking contract's pause check may not be reached in that path either. [4](#0-3) 

The spec also requires `CONTRACT_IS_PAUSED` for `exit_delegation_pool_action`: [5](#0-4) 

By contrast, all staking contract functions that are tested for pause correctly enforce it: [6](#0-5) 

### Impact Explanation

The pause mechanism is the protocol's emergency brake — it is activated when a vulnerability is discovered. If the staking contract is paused while a staker has already been removed (a common lifecycle event), any pool member of that staker can:

1. Call `exit_delegation_pool_intent` — succeeds because the pause check is bypassed.
2. Wait for `unpool_time` (which is set to `Time::now()`, so it is immediately satisfiable).
3. Call `exit_delegation_pool_action` — if `remove_from_delegation_pool_action` also skips the pause check when the staker is removed (as the "will do nothing" comment implies), the pool member receives their principal tokens while the protocol is in an emergency-paused state.

This matches the **Temporary freezing of funds** impact: funds that the protocol intends to freeze during an emergency can be withdrawn, defeating the security invariant the pause is designed to enforce.

### Likelihood Explanation

The precondition — staker removed — is a normal, expected lifecycle event (stakers exit regularly). An attacker does not need to manufacture this state; they only need to be a pool member of any staker who has already completed `unstake_action`. Given that the staking contract is paused precisely during high-risk windows (active exploits), the window where both conditions hold simultaneously is realistic and meaningful.

### Recommendation

Add an explicit pause check at the top of `exit_delegation_pool_intent` and `exit_delegation_pool_action` in the pool contract, independent of whether the staker is active:

```cairo
fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
    // Check staking contract pause status directly.
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "{}", GenericError::CONTRACT_IS_PAUSED);
    // ... rest of function
}
```

Apply the same guard to `exit_delegation_pool_action`. This ensures the pause invariant is enforced regardless of the staker's lifecycle state, matching the spec's stated pre-conditions.

### Proof of Concept

1. Deploy the full system; stake and open a delegation pool.
2. A pool member enters the delegation pool.
3. The staker calls `unstake_intent` then `unstake_action` — funds are transferred to the pool contract; `staker_removed` is set to `true`.
4. The security agent calls `staking_contract.pause()`.
5. The pool member calls `pool_contract.exit_delegation_pool_intent(amount)`.
   - `undelegate_from_staking_contract_intent` sees `!is_staker_active()` → returns `Time::now()` without calling the staking contract → **no pause check fires**.
6. Since `unpool_time == Time::now()`, the pool member immediately calls `exit_delegation_pool_action`.
   - `remove_from_delegation_pool_action` on the staking contract "does nothing" (funds already in pool) → pool member receives tokens while the contract is paused.

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }
