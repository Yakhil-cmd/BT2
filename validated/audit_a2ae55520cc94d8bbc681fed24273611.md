### Title
Gateway `skip_stateful_validations` Bypasses Signature Verification via Stale Pool Membership — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function skips the `__validate__` entry-point call for invoke transactions with nonce=1 from undeployed accounts when `account_tx_in_pool_or_recent_block` returns `true`. That helper returns `true` if the account has **any** transaction in the pool — not specifically a `deploy_account` transaction. Once an invoke tx (nonce=1) is admitted with skipped validation (while the deploy_account tx was present), it can itself satisfy the check for all future invoke txs for the same account, even after the deploy_account tx has been removed from the pool via TTL expiry.

### Finding Description

**Root cause — `skip_stateful_validations`:** [1](#0-0) 

The function returns `true` (skip validation) when:
1. The tx is an `Invoke` with `nonce == 1`
2. The on-chain `account_nonce == 0`
3. `account_tx_in_pool_or_recent_block(sender_address)` returns `true`

**Root cause — `account_tx_in_pool_or_recent_block`:** [2](#0-1) 

This returns `true` if `self.tx_pool.contains_account(account_address)` — i.e., if the account has **any** transaction in the pool, including a previously-admitted invoke tx that itself skipped validation.

**The broken invariant:** The code comment at line 440–443 states:

> "It is sufficient to check if the account exists in the mempool since it means that either it has a deploy_account transaction or transactions with future nonces that passed validations." [3](#0-2) 

The second clause is false for undeployed accounts (nonce=0): no invoke tx for an undeployed account can pass `__validate__` (the contract does not exist), so any invoke tx in the pool for such an account was itself admitted with skipped validation. The check is self-referential.

**Attack path:**

1. Attacker submits `deploy_account` tx for address `A` → admitted, `tx_pool.contains_account(A) = true`
2. Attacker submits invoke tx (nonce=1, sender=A, signature=`[0xdead, 0xbeef]`) → `account_tx_in_pool_or_recent_block(A) = true`, `__validate__` skipped, admitted to pool
3. The `deploy_account` tx expires via TTL (default 60 s per test config) while the invoke tx (submitted 1 s later) has not yet expired [4](#0-3) 

4. Attacker submits a new invoke tx (nonce=1, sender=A, signature=`[0xcafe, 0xbabe]`, higher tip) → `account_tx_in_pool_or_recent_block(A) = true` (step-2 invoke tx still in pool), `__validate__` skipped, admitted via fee escalation

At step 4 there is no `deploy_account` tx in the pool. The gateway has accepted an invoke tx with an arbitrary, unverified signature for a non-existent account.

**Why eviction does not prevent this:** Eviction only removes accounts with gaps when the pool is full. [5](#0-4) 

If the pool is not at capacity, the invoke tx from step 2 survives after the deploy_account tx expires, keeping `tx_pool.contains_account(A) = true`.

**Why the `state.contains_account` path is not the issue:** The `committed` and `staged` maps only contain accounts whose transactions were committed to a block. If a deploy_account tx was committed, the on-chain nonce becomes 1, so `account_nonce == Nonce(Felt::ZERO)` is false and the skip path is never entered. [6](#0-5) 

### Impact Explanation

The gateway's stateful admission control accepts invoke transactions with **arbitrary, unverified signatures** for undeployed accounts after the corresponding deploy_account tx has been removed from the pool. The `__validate__` entry point — the account's own signature-verification logic — is never called. These transactions will fail during blockifier execution (contract does not exist), but the gateway has violated its admission invariant: it accepted a transaction it should have rejected.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

The attack requires only:
- Submitting a `deploy_account` tx (no special privilege)
- Submitting an invoke tx with nonce=1 (no special privilege; validation is intentionally skipped)
- Waiting for the TTL window (configurable; default 60 s in tests)

No privileged access, no special network position, and no cryptographic break is required. Any unprivileged user can execute this.

### Recommendation

Replace the broad `account_tx_in_pool_or_recent_block` check with a check that specifically verifies a `deploy_account` transaction (nonce=0) exists in the pool for the sender address. For example, add a dedicated mempool query `has_deploy_account_tx_in_pool(address: ContractAddress) -> bool` that checks `tx_pool.get_by_address_and_nonce(address, Nonce(Felt::ZERO)).is_some()`. This closes the self-referential loop where a previously-skipped invoke tx can justify skipping validation for future invoke txs.

### Proof of Concept

```
T+0s:   POST /add_transaction  deploy_account{class=C, salt=S, nonce=0}
        → admitted; tx_pool has deploy_account for A

T+1s:   POST /add_transaction  invoke{sender=A, nonce=1, sig=[0xdead,0xbeef], tip=10}
        → account_tx_in_pool_or_recent_block(A) = true (deploy_account in pool)
        → __validate__ SKIPPED; admitted

T+61s:  (deploy_account TTL expires, removed by remove_expired_txs on next add_tx)
        invoke tx (submitted T+1s) still in pool (expires at T+61s)

T+61s:  POST /add_transaction  invoke{sender=A, nonce=1, sig=[0xcafe,0xbabe], tip=20}
        → remove_expired_txs removes deploy_account (expired), invoke tx survives
        → account_tx_in_pool_or_recent_block(A) = true (invoke tx from T+1s in pool)
        → __validate__ SKIPPED; admitted with arbitrary signature, no deploy_account in pool

Result: Gateway accepted invoke tx with unverified signature for non-existent account A.
```

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

#[rstest]
fn add_tx_old_transactions_cleanup() {
    // Create a mempool with a fake clock.
    let fake_clock = Arc::new(FakeClock::default());
    let mut mempool = Mempool::new(
        MempoolConfig {
            dynamic_config: MempoolDynamicConfig { transaction_ttl: Duration::from_secs(60) },
            ..Default::default()
        },
        fake_clock.clone(),
    );

    // Add a new transaction to the mempool.
    let first_tx =
        add_tx_input!(tx_hash: 1, address: "0x0", tx_nonce: 0, account_nonce: 0, tip: 100);
    add_tx(&mut mempool, &first_tx);

    // Advance the clock and add another transaction.
    fake_clock.advance(mempool.config.dynamic_config.transaction_ttl / 2);
    let second_tx =
        add_tx_input!(tx_hash: 2, address: "0x1", tx_nonce: 0, account_nonce: 0, tip: 50);
    add_tx(&mut mempool, &second_tx);

    // Verify that both transactions are in the mempool.
    let expected_txs = [&first_tx.tx, &second_tx.tx];
    let expected_mempool_content = MempoolTestContentBuilder::new()
        .with_pool(expected_txs.map(|tx| tx.clone()))
        .with_priority_queue(expected_txs.map(TransactionReference::new))
        .build();
    expected_mempool_content.assert_eq(&mempool.content());

    // Advance the clock and add a new transaction.
    fake_clock.advance(mempool.config.dynamic_config.transaction_ttl / 2 + Duration::from_secs(5));
    let third_tx =
        add_tx_input!(tx_hash: 3, address: "0x2", tx_nonce: 0, account_nonce: 0, tip: 10);
    add_tx(&mut mempool, &third_tx);

    // The first transaction should be removed from the mempool.
    let expected_txs = [&second_tx.tx, &third_tx.tx];
    let expected_mempool_content = MempoolTestContentBuilder::new()
        .with_pool(expected_txs.map(|tx| tx.clone()))
        .with_priority_queue(expected_txs.map(TransactionReference::new))
        .build();
    expected_mempool_content.assert_eq(&mempool.content());
