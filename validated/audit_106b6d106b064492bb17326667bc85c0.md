### Title
Missing Validation of `old_signing_key` for New Reshare Participants Enables Denial of Resharing - (File: src/dkg.rs)

---

### Summary

`assert_reshare_keys_invariants` in `src/dkg.rs` enforces that an **old** participant must supply its signing key, but omits the symmetric check: that a **new** participant (absent from `old_participants`) must **not** supply one. A malicious new participant can pass a non-`None` `old_signing_key`, causing `do_reshare` to compute a non-zero secret for it, which corrupts the aggregated public key and aborts the resharing for every honest party.

---

### Finding Description

`assert_reshare_keys_invariants` performs the following guard:

```rust
// if me is not in the old participant set then ensure that old_signing_key is None
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
``` [1](#0-0) 

The comment describes the **intended** invariant ("if me is not in the old participant set then ensure that old_signing_key is None"), but the code only implements the **opposite** half of that invariant. The case `!old_participants.contains(me) && old_signing_key.is_some()` is never rejected.

When a new participant (not in `old_participants`) passes `old_signing_key = Some(x_i)`, `do_reshare` proceeds to compute:

```rust
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(<C::Group as Group>::Field::zero);
``` [2](#0-1) 

Because `me` is absent from `intersection`, `intersection.lagrange::<C>(me)` evaluates the Lagrange basis polynomial at `me`'s scalar identifier — a point outside the set — yielding a **non-zero** field element. The product `lambda * x_i.to_scalar()` is therefore non-zero, and this value becomes the constant term of `me`'s secret polynomial inside `do_keyshare`.

Inside `do_keyshare`, the new participant correctly sends **no** proof of knowledge (since `generate_proof = old_participants.as_ref().is_none_or(|old| old.contains(me))` evaluates to `false`): [3](#0-2) 

Other participants verify the absent proof and accept it — because `me` is genuinely new, the `verify_proof_of_knowledge` branch for `None` proof passes without checking the committed constant term: [4](#0-3) 

The non-zero constant term shifts the aggregated verifying key away from the old public key. The final consistency check then fails for every participant:

```rust
if old_vk != verifying_key {
    return Err(ProtocolError::AssertionFailed(
        "new public key does not match old public key".to_string(),
    ));
}
``` [5](#0-4) 

The resharing aborts for **all** honest parties, not just the attacker.

---

### Impact Explanation

**High — Permanent denial of resharing for honest parties.**

Every honest participant in the resharing session reaches the public-key consistency check and aborts. As long as the malicious new participant is included in the new participant set (a legitimate configuration), it can repeat this attack on every resharing attempt, permanently blocking key rotation, threshold changes, and participant set updates for the group.

---

### Likelihood Explanation

**Medium.** The attacker must be a legitimate new participant invited into the resharing. No cryptographic material from the old epoch is required — any arbitrary scalar suffices as `old_signing_key`. The public API `reshare` accepts the call without error, so no special privilege or insider access beyond being a new participant is needed. [6](#0-5) 

---

### Recommendation

Add the missing symmetric guard immediately after the existing check in `assert_reshare_keys_invariants`:

```rust
// Existing check: old participant must supply a share
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
// Missing check: new participant must NOT supply a share
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not in the old participant list but provided a share"
    )));
}
``` [7](#0-6) 

---

### Proof of Concept

1. Run `keygen` with `participants = [P1, P2, P3]`, `threshold = 2`. Each party obtains a `KeygenOutput`.
2. Initiate reshare: `old_participants = [P1, P2, P3]`, `new_participants = [P1, P2, P3, P4]`, `new_threshold = 3`.
3. **P4** (new participant, absent from `old_participants`) calls `reshare(...)` with `old_signing_key = Some(arbitrary_signing_share)`.
4. `assert_reshare_keys_invariants` returns `Ok(...)` — the missing check does not fire.
5. `do_reshare` computes `secret = intersection.lagrange::<C>(P4) * arbitrary_signing_share.to_scalar()` — a non-zero value.
6. P4 broadcasts a commitment whose constant term encodes this non-zero secret; no proof of knowledge is required or sent.
7. All participants compute the aggregated verifying key; it differs from `old_public_key`.
8. Every participant hits `ProtocolError::AssertionFailed("new public key does not match old public key")` and aborts.
9. P4 can repeat this on every subsequent resharing attempt, permanently blocking key rotation.

pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],
) -> Result<(ParticipantList, ParticipantList), InitializationError> {
    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

    let participants = assert_key_invariants(participants, me, threshold)?;

    let old_participants =
        ParticipantList::new(old_participants).ok_or(InitializationError::DuplicateParticipants)?;

    // Step 1.1
    if old_participants.intersection(&participants).len() < old_threshold {
        return Err(InitializationError::NotEnoughParticipantsForNewThreshold {
            threshold: old_threshold,
            participants: old_participants.intersection(&participants).len(),
        });
    }
    // Step 1.1
    // if me is not in the old participant set then ensure that old_signing_key is None
    if old_participants.contains(me) && old_signing_key.is_none() {
        return Err(InitializationError::BadParameters(format!(
            "party {me:?} is present in the old participant list but provided no share"
        )));
    }
    Ok((participants, old_participants))
}

pub fn reshare<C: Ciphersuite>(
    old_participants: &[Participant],
    old_threshold: impl Into<ReconstructionLowerBound> + Send + 'static,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    new_participants: &[Participant],
    new_threshold: impl Into<ReconstructionLowerBound> + Copy + Send + 'static,
    me: Participant,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = KeygenOutput<C>>, InitializationError>
where
    Element<C>: Send,
    Scalar<C>: Send,
{
    let comms = Comms::new();
    let threshold = new_threshold;
    let (participants, old_participants) = assert_reshare_keys_invariants::<C>(
        new_participants,
        me,
        threshold,
        old_signing_key,
        old_threshold,
        old_participants,
    )?;
    let fut = do_reshare(
        comms.shared_channel(),
        participants,
        me,
        threshold,
        old_signing_key,
        old_public_key,
        old_participants,
        rng,
    );
    Ok(make_protocol(comms, fut))
}
