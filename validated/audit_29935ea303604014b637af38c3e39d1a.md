### Title
Unprivileged Caller Can Permanently Freeze All Consensus Rewards via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `Staking` contract has no access control beyond a zero-address check. It accepts a caller-controlled `disable_rewards` flag and updates a **global** `last_reward_block` storage variable before checking that flag. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` every block, consuming the per-block reward slot without distributing any rewards, permanently freezing all consensus-phase yield for all stakers.

### Finding Description
`update_rewards` is exposed as a public entrypoint under `IStakingRewardsManager`. Its only gate is `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

Inside the function, `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

`last_reward_block` is a **global** (not per-staker) variable. The guard at the top of the function asserts `current_block_number > last_reward_block`, meaning once any call succeeds in a given block, every subsequent call in that block reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

When `disable_rewards: true`, the function returns immediately after writing `last_reward_block`, skipping the entire reward calculation and distribution: [4](#0-3) 

The result: the block's reward slot is consumed, no rewards are minted or credited to any staker, and no other caller can reclaim that slot in the same block.

### Impact Explanation
An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` on every block permanently prevents all consensus-phase block rewards from being distributed to any staker or delegation pool. The `unclaimed_rewards` in `RewardSupplier` is never incremented, and stakers' `unclaimed_rewards_own` fields are never updated. This constitutes **permanent freezing of unclaimed yield** for the entire protocol. [5](#0-4) 

### Likelihood Explanation
- The function is fully public with no role restriction.
- The attacker only needs to submit one cheap transaction per block.
- The attacker needs only a valid (active, non-zero-balance) staker address as the argument — this is publicly observable on-chain via `NewStaker` events.
- No capital, no leaked key, no privileged access is required.
- The attack is sustainable indefinitely at negligible cost on Starknet.

### Recommendation
Restrict `update_rewards` to a trusted caller — either the block-producing operational address of the named staker, or a dedicated consensus-layer role. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Only the staker's registered operational address may call this.
    let caller = get_caller_address();
    let staker_info = self.internal_staker_info(:staker_address);
    assert!(caller == staker_info.operational_address, "{}", Error::UNAUTHORIZED_CALLER);
    ...
}
```

Alternatively, introduce a dedicated `REWARDS_UPDATER_ROLE` and gate the function on it, consistent with the existing `RolesComponent` pattern used throughout the contract. [6](#0-5) 

### Proof of Concept

```
Pre-condition:
  - Consensus rewards are active (consensus_rewards_first_epoch is set and current_epoch >= it).
  - At least one valid staker S exists with non-zero balance.

Attack loop (repeat every block):
  1. Attacker (any EOA) calls:
       staking.update_rewards(staker_address: S, disable_rewards: true)
  2. Inside the call:
       a. general_prerequisites() passes (contract not paused, caller != 0).
       b. current_block_number > last_reward_block  →  assertion passes.
       c. Staker S is active with non-zero balance  →  assertions pass.
       d. last_reward_block.write(current_block_number)  ← slot consumed.
       e. disable_rewards == true  →  early return, zero rewards distributed.
  3. Any legitimate call to update_rewards in the same block now reverts
     with REWARDS_ALREADY_UPDATED.
  4. Staker S (and every other staker) receives zero block rewards for
     this block.

Result after N blocks:
  - last_reward_block == current block every block.
  - unclaimed_rewards in RewardSupplier is never incremented.
  - All stakers' unclaimed_rewards_own remain at zero.
  - All consensus-phase yield is permanently frozen.
``` [7](#0-6)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
