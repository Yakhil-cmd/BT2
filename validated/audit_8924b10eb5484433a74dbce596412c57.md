### Title
Single Malicious Participant Can Permanently Abort DKG/Reshare/Refresh by Broadcasting False Success - (`src/dkg.rs`)

### Summary
The `broadcast_success` function in `src/dkg.rs` requires **all** N participants to broadcast `true` to confirm successful DKG completion. A single malicious participant can broadcast `false` at the final round to force all honest parties to abort the protocol, even after all cryptographic work has been completed successfully. This is a direct analog to the push-based ETH transfer DoS: just as a malicious borrower's `receive` revert blocks `auctionBuyNft`, a malicious participant's false broadcast blocks DKG completion for every honest party.

### Finding Description

At the end of `do_keyshare`, after all polynomial commitments, proofs of knowledge, and secret shares have been exchanged and verified, `broadcast_success` is called: [1](#0-0) 

```rust
broadcast_success(&mut chan, &participants, me, session_id).await?;
```

Inside `broadcast_success`, every participant broadcasts `(true, session_id)` via the echo-broadcast protocol, and then the result is checked with a strict all-or-nothing condition: [2](#0-1) 

```rust
async fn broadcast_success(...) -> Result<(), ProtocolError> {
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    let vote_list = vote_list.into_vec_or_none()...;

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed("...wrong session id..."));
    }
    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed("...failed its checks..."));
    }
    Ok(())
}
```

The echo-broadcast protocol (`do_broadcast`) guarantees **consistency**: all honest parties will agree on the same value for each participant's broadcast. It does **not** filter out or override a malicious participant's `false` vote. The `vote_list` returned by `do_broadcast` contains every participant's delivered value, including the malicious `false`. The subsequent `.all(|&(boolean, _)| boolean)` check then fails unconditionally, causing every honest party to return `Err` and abort. [3](#0-2)

These thresholds govern Byzantine fault tolerance for **consistency** of the broadcast value, but they do not prevent a single participant from broadcasting a legitimately-delivered `false` that then poisons the all-or-nothing check in `broadcast_success`.

The same abort path applies to `do_reshare` and the refresh flow, since both call `do_keyshare` internally: [5](#0-4) 

### Impact Explanation

**High — Permanent denial of key generation, reshare, and refresh for all honest parties.**

A single malicious participant can:
1. Participate honestly through every cryptographic round (commitment hashes, proofs of knowledge, secret shares).
2. In the final `broadcast_success` round, broadcast `(false, session_id)`.
3. The echo-broadcast protocol faithfully delivers this `false` to all honest parties.
4. Every honest party hits the `.all(|&(boolean, _)| boolean)` check, gets `false`, and aborts.

The attack is repeatable at zero marginal cost: the adversary can trigger it on every DKG/reshare/refresh attempt, permanently preventing honest parties from ever obtaining a usable key share. There is no recovery path within the protocol itself.

### Likelihood Explanation

**High.** Any single participant in the protocol can execute this attack with no cryptographic capability beyond normal protocol participation. No leaked keys, no external compromise, and no threshold violation is required. The attacker simply sends a single bit (`false`) in the last round. Because the protocol provides no mechanism to exclude or override a single participant's final broadcast, the attack succeeds deterministically every time it is attempted.

### Recommendation

Replace the strict all-or-nothing check in `broadcast_success` with a threshold-based check. Abort only if more than `f = (n-1)/3` participants report failure (consistent with the Byzantine fault tolerance already assumed by the echo-broadcast layer). Honest parties that successfully completed all checks should be able to finalize their key shares even if up to `f` participants broadcast `false`. Alternatively, identify and exclude the specific participant(s) who broadcast `false` and surface them as `MaliciousParticipant` errors rather than aborting the entire protocol.

### Proof of Concept

Setup: Alice (P0), Bob (P1), Charlie (P2) run DKG with threshold 2. Alice is malicious.

1. **Rounds 1–5**: Alice participates honestly — she sends valid commitment hashes, a valid proof of knowledge, and valid secret shares to Bob and Charlie. Bob and Charlie verify all of Alice's contributions successfully.
2. **Final round** (`broadcast_success`): Bob and Charlie broadcast `(true, session_id)`. Alice broadcasts `(false, session_id)`.
3. The echo-broadcast protocol delivers Alice's `(false, session_id)` consistently to Bob and Charlie.
4. Bob's `vote_list` = `[(true, sid), (false, sid), (true, sid)]`. The `.all(|&(boolean, _)| boolean)` check fails.
5. Bob returns `Err(AssertionFailed("A participant seems to have failed its checks. Aborting Protocol!"))`.
6. Charlie experiences the identical abort.
7. DKG fails. Alice can repeat this on every subsequent attempt, permanently denying Bob and Charlie a usable key. [2](#0-1) [6](#0-5)

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)

fn echo_ready_thresholds(n: usize) -> (usize, usize) {
    // case where no malicious parties are assumed: when n <= 3/
    // In this case the echo and ready thresholds are both 0
    // later we compare if we have collected more votes than these thresholds
    if n <= 3 {
        return (0, 0);
    }
    // we should always have n >= 3*threshold + 1
    let broadcast_threshold = (n - 1) / 3;
    let echo_threshold = usize::midpoint(n, broadcast_threshold);
    (echo_threshold, broadcast_threshold)
}
