### Title
Anyone Can Call `update_rewards` with `disable_rewards: true` to Permanently Block All Staker Reward Distribution — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable with no access control. It accepts a `disable_rewards: bool` parameter. Any unprivileged caller can invoke it with `disable_rewards: true` for any valid staker, consuming the protocol-wide per-block reward slot without distributing any rewards. Because `last_reward_block` is a single global storage variable, this prevents every legitimate reward update in that block. An attacker can front-run every legitimate call, permanently freezing unclaimed yield for all stakers and delegators.

---

### Finding Description

`update_rewards` is exposed as a public function under `IStakingRewardsManager`: [1](#0-0) 

The only guard is `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero — no role or identity check: [2](#0-1) 

Inside `update_rewards`, the function first asserts the current block is strictly greater than `last_reward_block`, then **unconditionally writes the current block number to `last_reward_block`**, and only then checks `disable_rewards`: [3](#0-2) 

`last_reward_block` is a single global storage slot, not per-staker: [4](#0-3) 

The consequence is:

1. Attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N.
2. `last_reward_block` is set to N; the function returns early — zero rewards distributed.
3. Any subsequent call to `update_rewards` in block N fails with `REWARDS_ALREADY_UPDATED`.
4. All stakers and all delegators receive no block rewards for block N.
5. Repeated every block, this permanently freezes all reward accrual.

The analog to the flash loan report is exact: just as anyone could call `flashLoan` specifying an arbitrary `_receiver` and force that receiver to pay fees, here anyone can call `update_rewards` specifying an arbitrary `staker_address` with `disable_rewards: true` and force the entire protocol to forfeit that block's rewards.

---

### Impact Explanation

`last_reward_block` is global. A single attacker call per block is sufficient to deny rewards to **every** staker and **every** pool member for that block. Sustained over time this constitutes permanent freezing of unclaimed yield — a **High** impact under the allowed scope.

---

### Likelihood Explanation

The attack requires no special role, no tokens, and no prior state. The only cost is gas. The attacker simply needs to submit a transaction in every block before any legitimate `update_rewards` call. On Starknet, where transaction ordering is observable, this is straightforward to automate. Likelihood is **High**.

---

### Recommendation

Restrict `update_rewards` to a trusted caller. The most natural choice is the attestation contract (already used for `update_rewards_from_attestation_contract`) or a dedicated consensus rewards manager role. Add a check analogous to `assert_caller_is_attestation_contract`: [5](#0-4) 

Alternatively, remove `disable_rewards` from the public interface entirely and handle the "no-reward" case internally based on protocol state (e.g., `is_pre_consensus()`), so the parameter cannot be weaponised by an external caller.

---

### Proof of Concept

```
Block N arrives.

1. Attacker submits:
     staking.update_rewards(staker_address=<any active staker>, disable_rewards=true)

2. Execution:
     - general_prerequisites() passes (contract not paused, caller != 0).
     - current_block_number (N) > last_reward_block (N-1) → assertion passes.
     - last_reward_block.write(N)          ← slot consumed.
     - disable_rewards == true → return.   ← no rewards distributed.

3. Legitimate node operator submits:
     staking.update_rewards(staker_address=<their staker>, disable_rewards=false)

4. Execution:
     - current_block_number (N) > last_reward_block (N) → FALSE → panic REWARDS_ALREADY_UPDATED.

5. All stakers and delegators receive 0 block rewards for block N.

6. Attacker repeats every block → permanent freeze of all unclaimed yield.
``` [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
