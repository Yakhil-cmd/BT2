### Title
Unprivileged `MsgConvertVouchers` Permanently Blocks Admin External Contract Registration via Auto-Deployment Race - (File: `x/cronos/keeper/evm.go`, `x/cronos/keeper/keeper.go`)

### Summary

An unprivileged user holding any valid IBC or gravity denom tokens can call `MsgConvertVouchers` to trigger auto-deployment of a `ModuleCRC21` contract for that denom. Because `SetExternalContractForDenom` enforces a create-only guardrail (`ensureDenomNotMapped`) that rejects any denom already mapped — including auto-deployed contracts — the admin's subsequent `MsgUpdateTokenMapping` for the same denom permanently fails with `ErrDenomAlreadyMapped`. There is no admin-accessible path to delete an auto-deployed contract mapping, making the block irrecoverable.

### Finding Description

**Two independent code paths write to the same denom→contract mapping key-space:**

**Path 1 — Unprivileged auto-deployment** (`MsgConvertVouchers` → `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21`):

`ConvertVouchers` in `msg_server.go` carries no authorization check:

```go
func (k msgServer) ConvertVouchers(goCtx context.Context, msg *types.MsgConvertVouchers) (*types.MsgConvertVouchersResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    err := k.ConvertVouchersToEvmCoins(ctx, msg.Address, msg.Coins)
``` [1](#0-0) 

When `EnableAutoDeployment` is `true` and no contract exists for the denom, `ConvertCoinFromNativeToCRC21` deploys a new `ModuleCRC21` contract and registers it via `SetAutoContractForDenom`:

```go
contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
...
if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
    return err
}
``` [2](#0-1) 

**Path 2 — Admin external contract registration** (`MsgUpdateTokenMapping` → `RegisterOrUpdateTokenMapping` → `SetExternalContractForDenom`):

`UpdateTokenMapping` requires `CanChangeTokenMapping` permission:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [3](#0-2) 

`SetExternalContractForDenom` calls `ensureDenomNotMapped`, which rejects the call if **any** mapping (external or auto) already exists for the denom:

```go
func (k Keeper) ensureDenomNotMapped(ctx sdk.Context, denom string) error {
    if contract, found := k.GetContractByDenom(ctx, denom); found {
        return errors.Wrapf(
            types.ErrDenomAlreadyMapped, ...
        )
    }
    return nil
}
``` [4](#0-3) 

`GetContractByDenom` checks both external and auto contracts:

```go
func (k Keeper) GetContractByDenom(ctx sdk.Context, denom string) (contract common.Address, found bool) {
    contract, found = k.getExternalContractByDenom(ctx, denom)
    if !found {
        contract, found = k.getAutoContractByDenom(ctx, denom)
    }
    return contract, found
}
``` [5](#0-4) 

**There is no admin path to delete an auto-deployed contract mapping.** `RegisterOrUpdateTokenMapping` with an empty contract only calls `DeleteExternalContractForDenom`, which does not touch the `DenomToAutoContract` store key:

```go
if len(msg.Contract) == 0 {
    // delete existing mapping
    k.DeleteExternalContractForDenom(ctx, msg.Denom)
``` [6](#0-5) 

**The IBC middleware path does NOT trigger auto-deployment for new denoms.** `canBeConverted` only returns `true` if a contract already exists:

```go
func (im IBCConversionModule) canBeConverted(ctx sdk.Context, denom string) bool {
    params := im.cronoskeeper.GetParams(ctx)
    if denom == params.IbcCroDenom {
        return true
    }
    _, found := im.cronoskeeper.GetContractByDenom(ctx, denom)
    return found
}
``` [7](#0-6) 

So the only path that triggers auto-deployment for a **new** denom is `MsgConvertVouchers`, which is unprivileged.

### Impact Explanation

Once an attacker triggers auto-deployment for denom X, `SetExternalContractForDenom` permanently returns `ErrDenomAlreadyMapped` for that denom. The admin's `MsgUpdateTokenMapping` — whether sent directly or via governance `TokenMappingChangeProposal` — is irrecoverably blocked. The admin cannot substitute the auto-deployed minimal `ModuleCRC21` contract with a preferred external contract (e.g., a proxy contract with specific bridge integrations, access controls, or token economics). This constitutes:

- **Bypass of Cronos admin authority** over token-mapping registration
- **Permanent corruption of denom/contract binding** — the admin-intended external contract can never be registered for the targeted denom [8](#0-7) 

### Likelihood Explanation

**Conditions required:**
1. `EnableAutoDeployment` is `true` (the default for Cronos mainnet per ADR-004)
2. The attacker holds any nonzero amount of the target IBC or gravity denom

**Attacker acquisition of tokens:** Any user can obtain IBC tokens by sending assets from another Cosmos chain to Cronos via a standard IBC transfer. The IBC transfer itself does not trigger auto-deployment (because `canBeConverted` returns `false` for unmapped denoms). The attacker then calls `MsgConvertVouchers` with those tokens to trigger auto-deployment.

The attack can be executed **proactively** (preemptively blocking any denom the admin might later want to register) or **reactively** (watching the mempool for a pending `MsgUpdateTokenMapping` and front-running it). No special privilege is required beyond holding a small amount of the target denom.

### Recommendation

1. **Expose an admin-only `DeleteAutoContractForDenom` operation** in `RegisterOrUpdateTokenMapping` (or a dedicated message), so the admin can clear a stale or attacker-injected auto-mapping before registering an external contract.

2. **Alternatively**, allow `SetExternalContractForDenom` to atomically replace an existing auto-deployed contract mapping (not an external one), since the auto-deployed contract is module-controlled and the admin's external contract registration is an intentional upgrade of it.

3. **Or**, gate `MsgConvertVouchers`-triggered auto-deployment behind an additional check that the denom has no pending admin registration intent, or require a minimum time delay before auto-deployment is committed.

### Proof of Concept

```
// Setup: EnableAutoDeployment = true, denom X has no mapping yet.

// Step 1: Attacker obtains IBC tokens of denom X by sending from another chain.
// (Standard IBC transfer — no auto-deployment occurs because canBeConverted returns false)

// Step 2: Attacker calls MsgConvertVouchers with denom X.
// ConvertVouchersToEvmCoins → ConvertCoinFromNativeToCRC21(autoDeploy=true)
// → DeployModuleCRC21(X) → SetAutoContractForDenom(X, contractB)
// State: DenomToAutoContract[X] = contractB

// Step 3: Admin broadcasts MsgUpdateTokenMapping(denom=X, contract=contractA)
// → RegisterOrUpdateTokenMapping → SetExternalContractForDenom(X, contractA)
// → ensureDenomNotMapped(X) → GetContractByDenom(X) finds contractB
// → returns ErrDenomAlreadyMapped  ← PERMANENT FAILURE

// Step 4: Admin attempts to clear via MsgUpdateTokenMapping(denom=X, contract="")
// → DeleteExternalContractForDenom(X) — no external contract exists, no-op
// DenomToAutoContract[X] = contractB still present
// Admin's preferred contractA can never be registered.

func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}

func (k Keeper) SetAutoContractForDenom(ctx sdk.Context, denom string, address common.Address) error {

	if err := validateContractAddressForSourceDenom(denom, address, isSource); err != nil {

		return err

	store.Set(types.DenomToAutoContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}
