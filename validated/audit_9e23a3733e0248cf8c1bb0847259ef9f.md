### Title
Unbounded `stakers` Vec Growth Enables Permanent DoS of `get_stakers()` via Sybil Staking - (File: `src/staking/staking.cairo`)

---

### Summary

The `stakers` storage `Vec` grows without bound because every `stake()` call appends a new address but `unstake_action()` never removes it. The consensus-critical `get_stakers()` function iterates the **entire** Vec on every call. An attacker can inflate this Vec indefinitely by cycling the minimum stake amount through a series of fresh addresses, eventually causing `get_stakers()` to exceed the Starknet gas limit and become permanently uncallable.

---

### Finding Description

**Root cause — append-only `stakers` Vec with no size cap:**

In `stake()`, every new staker is unconditionally pushed to the `stakers` Vec:

```cairo
// Add staker address to the stakers vector.
self.stakers.push(staker_address);
``` [1](#0-0) 

The storage declaration carries an explicit design note confirming entries are never cleaned up:

```cairo
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [2](#0-1) 

**Root cause — full-range iteration in `get_stakers()`:**

`get_stakers()` iterates every element of `stakers` on every call, with no pagination or size guard:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    ...
}
``` [3](#0-2) 

**Root cause — address reuse is permanently blocked:**

`assert_staker_address_not_reused` prevents any address from staking a second time, even after a full exit. This means each attack cycle must consume a fresh address, but the STRK tokens themselves are fully recovered after `unstake_action()`:

```cairo
fn assert_staker_address_not_reused(self: @ContractState, staker_address: ContractAddress) {
    assert!(
        self.staker_own_balance_trace.entry(key: staker_address).is_empty(),
        "{}",
        Error::STAKER_ADDRESS_ALREADY_USED,
    );
}
``` [4](#0-3) 

**Attack flow:**

1. Attacker holds `min_stake` STRK.
2. Calls `stake()` from address A₁ → A₁ is appended to `stakers`.
3. Calls `unstake_intent()` → waits `exit_wait_window` (default 1 week).
4. Calls `unstake_action()` → recovers `min_stake` STRK; A₁ remains in `stakers` forever.
5. Repeats from step 2 with address A₂, A₃, … Aₙ using the same recovered funds.

After N cycles the `stakers` Vec has N entries. `get_stakers()` must read and process all N entries. Because Starknet enforces per-transaction gas limits on view calls, once N is large enough the function reverts on every invocation.

---

### Impact Explanation

`get_stakers()` is the consensus-layer entry point for determining the active validator set each epoch:

```cairo
fn get_stakers(
    self: @ContractState, epoch_id: Epoch,
) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)>
``` [5](#0-4) 

If this function becomes uncallable due to gas exhaustion, the consensus layer cannot retrieve the validator set, which halts block production. This is **unbounded gas consumption** leading to a protocol-level denial of service — matching the "Unbounded gas consumption" (Medium) impact in the allowed scope.

---

### Likelihood Explanation

- The attacker needs only `min_stake` STRK at any one time; funds are fully recovered after each cycle.
- The only ongoing cost is gas fees per cycle and the opportunity cost of locking funds for one `exit_wait_window` (~1 week by default).
- Cycles can be parallelised across many addresses simultaneously to accelerate Vec growth.
- No privileged access, no oracle manipulation, and no external dependency is required — only the public `stake()` / `unstake_intent()` / `unstake_action()` interface.
- The attack is permanent: once an address is in `stakers` it can never be removed.

---

### Recommendation

1. **Remove stakers on exit**: In `remove_staker()` (called from `unstake_action()`), mark the slot in the Vec as a sentinel zero-address and skip zero entries in `get_stakers()`, or use a swap-and-pop pattern.
2. **Enforce a maximum staker count**: Add a configurable `max_stakers` limit checked inside `stake()`, analogous to the mempool's `txmp.config.Size` check.
3. **Paginate `get_stakers()`**: Accept an offset/limit parameter so the call can never exceed the gas limit regardless of Vec size.

---

### Proof of Concept

// Pseudocode — repeat N times with fresh address each iteration
for i in 0..N:
    addr_i = fresh_address()
    fund(addr_i, min_stake)
    staking.stake(addr_i, reward_addr, operational_addr, min_stake)
    staking.unstake_intent()                    // from addr_i
    advance_time(exit_wait_window + 1)
    staking.unstake_action(addr_i)              // recovers min_stake
    // addr_i permanently remains in stakers Vec

// After N iterations:
staking.get_stakers(current_epoch)              // OOG revert when N is large enough
```

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
