### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped Per-Bin Additional Fees - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees()` allows a pool admin to set per-bin additional spread fees (`addFeeBuyE6`, `addFeeSellE6`) with no cap check, while the parallel admin fee setter `setPoolAdminFees()` strictly enforces `maxAdminSpreadFeeE6`. The per-bin fees are added directly to the effective swap fee in all four swap paths, so a pool admin can charge traders more than the protocol's fee cap system permits.

### Finding Description

The factory enforces a two-layer fee cap system. The factory owner sets `maxAdminSpreadFeeE6` (bounded by `HARD_MAX_SPREAD_FEE_E6 = 200_000`, i.e. 20%). The pool admin's global spread fee is checked against this cap in `setPoolAdminFees`:

```solidity
// MetricOmmPoolFactory.sol lines 414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
```

However, the per-bin fee setter has no analogous check:

```solidity
// MetricOmmPoolFactory.sol lines 450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

The pool-level `setBinAdditionalFees` only validates the bin index, not the fee values:

```solidity
// MetricOmmPool.sol lines 464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    ...
}
```

These per-bin fees are then added on top of `baseFeeX64` (the oracle-derived spread) in every swap path:

```solidity
// MetricOmmPool.sol line 910 (buyToken0SpecifiedOut)
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)

// MetricOmmPool.sol line 999 (buyToken0SpecifiedIn)
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)

// MetricOmmPool.sol line 1088 (buyToken1SpecifiedOut)
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)
```

Since `addFeeBuyE6` and `addFeeSellE6` are `uint16`, the maximum value is `65535`, which in E6 units equals **6.5535%** additional fee per bin. This is a completely separate fee channel from `adminSpreadFeeE6` and is not subject to any cap.

### Impact Explanation

A pool admin can set `addFeeBuyE6 = addFeeSellE6 = 65535` (6.5535%) on every active bin. This additional fee is applied to every swap through those bins, making the effective bid/ask spread worse than the protocol's fee cap system intends to allow. If the factory owner has set `maxAdminSpreadFeeE6 = 0` to restrict admin fees entirely, the pool admin can still impose up to 6.5535% additional spread per bin via this uncapped path. Traders receive worse execution prices than the cap system guarantees, constituting bad-price execution and an admin-boundary break.

### Likelihood Explanation

The pool admin is a semi-trusted actor with direct access to `setPoolBinAdditionalFees`. No timelock, no protocol approval, and no cap check stands between the admin and setting `addFeeBuyE6 = type(uint16).max` on any bin. The call is a single transaction. Any pool admin who wishes to extract value or harm traders can trigger this immediately.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` analogous to the check in `setPoolAdminFees`. The per-bin additional fees should be bounded by `maxAdminSpreadFeeE6` (or a dedicated per-bin cap):

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

### Proof of Concept

1. Factory owner sets `maxAdminSpreadFeeE6 = 0` via `setFeeCaps(0, 0, ...)` to prevent admin from charging any spread fee.
2. Pool admin calls `setPoolAdminFees(pool, 1, 0)` → reverts `AdminFeeTooHigh`. Cap is enforced.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` → **succeeds**. No cap check.
4. A trader calls `swap(...)` routing through bin 0. The effective fee applied is `baseFeeX64 + 65535/1e6` (6.5535% additional), far exceeding the `maxAdminSpreadFeeE6 = 0` cap the factory owner intended to enforce.
