### Title
Premature `lastBlockList` Update Enables Permanent Blocklist Bypass via Key-Rotation Race — (`app/proposal.go`)

---

### Summary

A bonded validator can rotate their e2ee encryption key between the moment the admin queries validator keys and the moment `MsgStoreBlockList` is included in a block. When `SetBlockList` subsequently fails to decrypt the blob (key mismatch), a premature write to `h.lastBlockList` permanently prevents the validator from ever retrying decryption of that blob, leaving the validator's in-memory blocklist stale or empty for all blocks it proposes.

---

### Finding Description

**Step 1 — No atomic link between key snapshot and blob storage.**

`encrypt-to-validators` queries the `Keys` RPC at an arbitrary block height, then the admin separately submits `MsgStoreBlockList`. There is no on-chain mechanism that ties the key snapshot to the stored blob. [1](#0-0) 

**Step 2 — `MsgRegisterEncryptionKey` is permissionless.**

Any account (including a bonded validator) can overwrite their registered key at any time with no rate-limit or cooldown. [2](#0-1) 

**Step 3 — `lastBlockList` is written before decryption succeeds.**

In `SetBlockList`, `h.lastBlockList` is updated to the new blob *before* `age.Decrypt` is called. If decryption fails (because the validator rotated its key after the admin's snapshot), the function returns an error but `h.lastBlockList` already holds the new blob. [3](#0-2) 

**Step 4 — The early-exit check permanently blocks recovery.**

On any subsequent call with the same blob (e.g., admin re-submits the identical `MsgStoreBlockList`), the equality check at line 215 fires and returns `nil` immediately — decryption is never retried. The validator is permanently stuck with its old (or empty) blocklist until the admin produces a *different* ciphertext blob encrypted to the new key. [4](#0-3) 

**Step 5 — `h.blocklist` is not updated on failure.**

Because `h.blocklist = m` is only reached after successful decryption and JSON parsing, a failed decryption leaves `h.blocklist` at its previous value (empty on first-time setup, stale otherwise). [5](#0-4) 

**Step 6 — Blocklist bypass in `PrepareProposal` / `ProcessProposal`.**

Both `ValidateTransaction` and `ProcessProposalHandler` read directly from `h.blocklist`. An empty or stale map means blocked addresses pass the filter for every block proposed or processed by the affected validator. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

- The attacker's validator node silently stops enforcing the current blocklist for all blocks it proposes and processes.
- If this is the first-ever blocklist deployment, the validator's blocklist remains permanently empty until the admin re-encrypts with a fresh blob.
- The `lastBlockList` premature-write bug means the admin cannot fix the affected validator simply by re-submitting the same blob; a new encryption round (producing different ciphertext bytes) is required.
- This is a direct bypass of the block-list authorization check (High impact per scope rules).

**Correction to the question's claim:** `h.blocklist` is *not* cleared to empty on decryption failure — it retains its previous value. The real harm is that it becomes permanently stale/unupdatable for that blob, not that it is zeroed.

---

### Likelihood Explanation

- `MsgRegisterEncryptionKey` is permissionless and cheap; key rotation requires only a standard Cosmos SDK transaction.
- The window between the admin's `Keys` query and the `MsgStoreBlockList` tx landing on-chain is at minimum one block (~5 s), giving the attacker ample time to front-run with a key rotation.
- The attacker only needs to be a bonded validator; no admin or governance compromise is required.
- The bug is self-sustaining: once triggered, the same blob can never fix the affected validator.

---

### Recommendation

1. **Fix the premature write**: Move `h.lastBlockList` update to *after* successful decryption and parsing, so a failed attempt can be retried.

```go
// proposed fix in SetBlockList
reader, err := age.Decrypt(bytes.NewBuffer(blob), h.Identity)
if err != nil {
    return err
}
// ... parse blocklist ...
h.blocklist = m
// Only cache after full success:
h.lastBlockList = make([]byte, len(blob))
copy(h.lastBlockList, blob)
return nil
```

2. **Bind the key snapshot to the blob**: Include the block height of the `Keys` query in the `MsgStoreBlockList` payload and reject the message on-chain if any validator's registered key has changed since that height.

3. **Alternatively**: Require validators to sign a commitment to their current key before the admin encrypts, creating an atomic link.

### Proof of Concept

```
1. Attacker (bonded validator V) submits MsgRegisterEncryptionKey{key: K1}.
2. Admin runs encrypt-to-validators at block H → queries Keys → gets K1 for V.
3. Admin encrypts blocklist to all validators (including K1) → produces blob B.
4. Attacker submits MsgRegisterEncryptionKey{key: K2} (different key).
   This tx lands in block H+1 before the admin's MsgStoreBlockList.
5. Admin's MsgStoreBlockList{blob: B} lands in block H+2.
   On-chain state: KeyPrefix[V] = K2, BlockList = B (encrypted to K1).
6. Validator node V calls SetBlockList(B):
   - h.lastBlockList is set to B  ← premature write
   - age.Decrypt(B, K2) → ErrIncorrectIdentity
   - returns error; h.blocklist unchanged (empty or stale)
7. Admin re-submits MsgStoreBlockList{blob: B} (same bytes).
   SetBlockList(B): bytes.Equal(lastBlockList, B) → return nil immediately.
   h.blocklist is never updated. Bypass is permanent for blob B.
8. V proposes a block; ValidateTransaction checks h.blocklist (empty/stale).
   Blocked address A's transactions pass the filter and are included.

func (h *ProposalHandler) SetBlockList(blob []byte) error {
	if h.Identity == nil {
		return nil
	}

	if bytes.Equal(h.lastBlockList, blob) {
		return nil
	}
	h.lastBlockList = make([]byte, len(blob))
	copy(h.lastBlockList, blob)

	if len(blob) == 0 {
		h.blocklist = make(map[string]struct{})
		return nil
	}

	reader, err := age.Decrypt(bytes.NewBuffer(blob), h.Identity)
	if err != nil {
		return err
	}

	data, err := io.ReadAll(reader)
	if err != nil {
		return err
	}

	var blocklist BlockList
	if err := json.Unmarshal(data, &blocklist); err != nil {
		return err
	}

	// convert to map
	m := make(map[string]struct{}, len(blocklist.Addresses))
	for _, s := range blocklist.Addresses {
		addr, err := h.addressCodec.StringToBytes(s)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", s, err)
		}
		if IsUnblockable(addr) {
			continue
		}
		encoded, err := h.addressCodec.BytesToString(addr)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", s, err)
		}
		m[encoded] = struct{}{}
	}

	h.blocklist = m
	return nil
}
