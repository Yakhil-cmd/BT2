### Title
ETH Sent to `tick()` When No Minting Is Required Is Permanently Locked — (`L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary
`RewardSupplier.tick()` is declared `external payable` and relies on `msg.value` to pay StarkGate bridge fees. However, when `requiredMinting()` returns zero (no pending L2 mint requests), the entire `if (amountToMint > 0)` block is skipped and any ETH forwarded by the caller is silently trapped in the contract forever. The contract has no `receive()` function, no `fallback()` function, and no ETH-withdrawal mechanism anywhere in its inheritance chain (`RewardSupplierStorage`, `Identity`, `ProxySupportImpl`, `ProxySupport`, `GovernanceStub`), making recovery impossible.

---

### Finding Description

`tick()` is the sole entry point for triggering L1 reward minting and bridging to L2. It is marked `payable` because it must forward ETH as fees to two external calls:

```solidity
uint256 msgFee = msg.value / 2;
bridge().depositWithMessage{value: msgFee}(...);   // line 126
msgFee = msg.value - msgFee;
messagingContract().sendMessageToL2{value: msgFee}(...); // line 137
```

Both calls are inside the `if (amountToMint > 0)` guard. When `amountToMint == 0` the guard is not entered, the two fee-forwarding calls never execute, and `msg.value` is never spent. Because the contract defines no `receive()`, no `fallback()`, and no withdrawal function, the ETH accumulates in the contract with no path to recovery. [1](#0-0) 

The full inheritance chain confirms no ETH-recovery mechanism exists anywhere: [2](#0-1) [3](#0-2) 

---

### Impact Explanation

Any ETH sent to `tick()` during a call where `requiredMinting()` returns `(0, 0)` is permanently frozen in the `RewardSupplier` contract. There is no owner-rescue function, no `receive()` to at least acknowledge the deposit, and no `fallback()`. The loss is irreversible.

This maps to the allowed impact: **High — Permanent freezing of funds**.

---

### Likelihood Explanation

`tick()` is callable by any account (the spec diagram labels the caller "anyAccount"). Callers are expected to supply ETH to cover bridge fees. A caller who does not first query `requiredMinting()` — or who races with another caller that drains the pending queue — will send ETH into a no-op execution path and lose it permanently. Because the function is public and permissionless, this scenario is realistic and requires no privileged access. [4](#0-3) 

---

### Recommendation

Add a `receive()` function so the contract can at least accept ETH intentionally pre-funded for fees, and refund any unused `msg.value` at the end of `tick()`:

```solidity
receive() external payable {}

function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();
    if (amountToMint > 0) {
        // ... existing logic ...
    }
    // Refund unused ETH to caller
    if (address(this).balance > 0) {
        Addresses.performEthTransfer(msg.sender, address(this).balance);
    }
}
```

Alternatively, revert when `msg.value > 0` and `amountToMint == 0` to prevent accidental loss.

---

### Proof of Concept

```solidity
function testEthLockedWhenNoMinting() public {
    // Ensure no pending L2 mint requests exist
    (uint256 amountToMint,) = rewardSupplier.requiredMinting();
    assertEq(amountToMint, 0);

    uint256 ethBefore = address(caller).balance;

    vm.prank(caller);
    // tick() is payable; ETH is accepted but never forwarded
    rewardSupplier.tick{value: 1 ether}();

    // ETH is now permanently locked in the contract
    assertEq(address(rewardSupplier).balance, 1 ether);
    assertEq(address(caller).balance, ethBefore - 1 ether);

    // No receive/fallback/withdraw — ETH cannot be recovered
    vm.prank(caller);
    (bool success,) = address(rewardSupplier).call{value: 0}("");
    // Direct ETH send also fails (no receive/fallback)
    assertFalse(success);
}

        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.

                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload

        }

abstract contract ProxySupport is MGovernance, BlockDirectCall, ContractInitializer {
    using Addresses for address;

    // The two function below (isFrozen & initialize) needed to bind to the Proxy.
    function isFrozen() external view virtual returns (bool) {
        return false;
    }

    /*
      The initialize() function serves as an alternative constructor for a proxied deployment.

      Flow and notes:
      1. This function cannot be called directly on the deployed contract, but only via
         delegate call.
      2. If an EIC is provided - init is passed onto EIC and the standard init flow is skipped.
         This true for both first intialization or a later one.
      3. The data passed to this function is as follows:
         [sub_contracts addresses, eic address, initData].

         When calling on an initialized contract (no EIC scenario), initData.length must be 0.
    */
    function initialize(bytes calldata data) external notCalledDirectly {
        uint256 eicOffset = 32 * numOfSubContracts();
        uint256 expectedBaseSize = eicOffset + 32;
        require(data.length >= expectedBaseSize, "INIT_DATA_TOO_SMALL");
        address eicAddress = abi.decode(data[eicOffset:expectedBaseSize], (address));

        bytes calldata subContractAddresses = data[:eicOffset];

        processSubContractAddresses(subContractAddresses);

        bytes calldata initData = data[expectedBaseSize:];

        // EIC Provided - Pass initData to EIC and the skip standard init flow.
        if (eicAddress != address(0x0)) {
            callExternalInitializer(eicAddress, initData);
            return;
        }

        if (isInitialized()) {
            require(initData.length == 0, "UNEXPECTED_INIT_DATA");
        } else {
            // Contract was not initialized yet.
            validateInitData(initData);
            initializeContractState(initData);
            initGovernance();
        }
    }
