### Title
Incorrect Multi-NFT Royalty Estimation in WalletConnect Offer Confirmation Causes User to Approve Wrong Spend Amount - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary

`formatAmountWithRoyalties` in `parseCommandDisplay.ts` incorrectly divides the fungible amount equally among all NFTs before applying each NFT's royalty percentage. This produces a systematically underestimated "Total Amount with Royalties" displayed in the WalletConnect confirmation dialog (`Confirm.tsx`). A user approving a `take_offer` WalletConnect command involving multiple NFTs with royalties will see a lower total spend than what the blockchain will actually deduct, causing them to approve a transaction under false financial pretenses.

### Finding Description

`formatAmountWithRoyalties` is called during WalletConnect command parsing to compute the `amountWithRoyalties` display field shown in the "You Spend" section of the confirmation dialog:

```typescript
// packages/gui/src/electron/commands/parseCommandDisplay.ts
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← wrong assumption
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

The function assumes the fungible amount is split equally among all NFTs, then applies each NFT's royalty percentage to its share. In Chia's protocol, each NFT's royalty is applied to the **full** fungible amount, not a fractional share. The correct formula is:

```
royaltyAmount = Σ (amount × royaltyPercentage_i / 10_000)
```

**Concrete example** (from the existing test):
- 2 NFTs: royalty percentages 500 (5%) and 10 (0.1%)
- Fungible amount: 100,000,000 mojos (0.0001 XCH)

Current (wrong) display:
- splitAmount = 50,000,000
- royaltyAmount = (50,000,000 × 500)/10,000 + (50,000,000 × 10)/10,000 = 2,500,000 + 50,000 = **2,550,000 mojos**
- Shown total: **0.00010255 XCH**

Correct calculation:
- royaltyAmount = (100,000,000 × 500)/10,000 + (100,000,000 × 10)/10,000 = 5,000,000 + 100,000 = **5,100,000 mojos**
- Actual total: **0.0001051 XCH**

The underestimation grows with the number of NFTs and the magnitude of royalty percentages. With N NFTs, the displayed royalty is 1/N of the correct value.

The `amountWithRoyalties` field is rendered directly in the WalletConnect approval dialog: [2](#0-1) 

The `parseCommandDisplay` function is the sole source of this value for WalletConnect `take_offer` commands: [3](#0-2) 

The regular offer UI (`NFTOfferViewer`, `NFTOfferEditor`) uses the separate `calculateNFTRoyalties` function which correctly applies the royalty to the full amount without splitting: [4](#0-3) 

Only the WalletConnect path is affected.

### Impact Explanation

A user approving a WalletConnect `take_offer` command sees an underestimated "Total Amount with Royalties" in the confirmation dialog. The actual blockchain transaction deducts the correct (higher) royalty amount. The user consents to a spend amount lower than what is actually taken from their wallet. This is a direct financial discrepancy caused by incorrect state displayed in the WalletConnect approval flow, matching the High impact category: *"WalletConnect state that causes a user to approve... the wrong... amount."*

### Likelihood Explanation

Any WalletConnect-connected dApp can craft a `take_offer` command containing multiple NFTs with royalties. No special privileges are required. The dApp need not be malicious — even a legitimate dApp presenting a valid multi-NFT offer will trigger the underestimation. The discrepancy scales with the number of NFTs and royalty percentages, making it significant for collections with high royalties.

### Recommendation

Replace the split-amount approach with the correct per-NFT full-amount calculation:

```typescript
function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  // Each NFT's royalty is applied to the full fungible amount, not a split share
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;
  ...
}

### Proof of Concept

1. A WalletConnect dApp sends a `chia_wallet.take_offer` command for an offer containing 2 NFTs: NFT-A (5% royalty) and NFT-B (5% royalty) in exchange for 1 XCH (1,000,000,000,000 mojos).
2. The confirmation dialog calls `formatAmountWithRoyalties` with `royaltyPercentages = [500, 500]` and `amount = 1,000,000,000,000n`.
3. Current code: `splitAmount = 500,000,000,000`; `royaltyAmount = 25,000,000,000 + 25,000,000,000 = 50,000,000,000`; displayed total = **1.05 XCH**.
4. Correct value: `royaltyAmount = 50,000,000,000 + 50,000,000,000 = 100,000,000,000`; actual total = **1.10 XCH**.
5. User approves believing they spend 1.05 XCH; blockchain deducts 1.10 XCH — a 0.05 XCH discrepancy per additional NFT with royalties. [6](#0-5)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
