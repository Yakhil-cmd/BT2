### Title
Two-Step Admin Bypass of Version-Beacon Stop Height Constraint - (`File: engine/execution/ingestion/stop/stop_control.go`)

### Summary

`StopControl.validateStopChange` enforces that when a version beacon and a manual admin command set conflicting stop heights, the **earlier** height wins. However, this cross-source constraint can be bypassed in two sequential admin calls, allowing an execution node operator to push the stop height arbitrarily later than the version beacon mandated boundary.

### Finding Description

`StopControl` tracks two sources for a stop height: `stopBoundarySourceManual` (admin command) and `stopBoundarySourceVersionBeacon` (on-chain version beacon service event). The documented invariant is:

> "stop is already set by the version beacon and is set manually. The resulting stop height is the lower one."

`validateStopChange` enforces this with the following logic: [1](#0-0) 

The critical flaw is at line 365–368: **when the new stop comes from the same source as the existing stop, the update is unconditionally allowed**, regardless of direction. This creates a two-step bypass:

1. **Step 1** — Version beacon has set stop at height `H` (source = `versionBeacon`). Admin calls `SetStopParameters({StopBeforeHeight: H-1})`. This passes validation because `H-1 < H` (different source, new stop is earlier). The stop is now `H-1` with source = `manual`.

2. **Step 2** — Admin calls `SetStopParameters({StopBeforeHeight: H+N})` for any large `N`. This passes validation because **source is now `manual → manual`** (same source), so the check at line 365 returns `nil` unconditionally. The stop is now `H+N`, far past the version beacon boundary.

The public entry point is the `stop-at-height` admin command: [2](#0-1) 

which calls `setStopParameters` → `validateStopChange`. [3](#0-2) 

### Impact Explanation

The version beacon is a protocol-level safety mechanism that mandates execution nodes stop before a specific block height when the node software is incompatible with the next protocol version. Bypassing it causes the execution node to produce execution receipts for blocks it is not qualified to execute. These receipts, generated by an outdated node version, may contain incorrect state commitments or events, corrupting the execution layer and potentially causing verification failures or incorrect seals to be included in the chain.

### Likelihood Explanation

The attack requires access to the admin HTTP API of an execution node, which is available to the node operator. The operator is a semi-trusted party (analogous to a "role provider" in the original report). Two sequential HTTP calls to the `stop-at-height` admin endpoint are sufficient. No special timing, front-running, or cryptographic material is needed. [4](#0-3) 

### Recommendation

The `validateStopChange` function must track the **original** stop height set by the version beacon independently of the manual stop, and enforce that a manual stop can never exceed the version beacon's boundary regardless of intermediate state. One approach: store the version beacon stop height in a separate field and always enforce `manualStop.StopBeforeHeight <= versionBeaconStop.StopBeforeHeight` when a version beacon boundary is active.

### Proof of Concept

```
// Version beacon sets stop at height 1000 (source = versionBeacon)
// s.stopBoundary = {StopBeforeHeight: 1000, source: versionBeacon}

// Step 1: Admin sets stop at 999 (allowed: 999 < 1000, different source)
sc.SetStopParameters(StopParameters{StopBeforeHeight: 999})
// s.stopBoundary = {StopBeforeHeight: 999, source: manual}

// Step 2: Admin sets stop at 9999 (allowed: same source manual→manual, no height check)
sc.SetStopParameters(StopParameters{StopBeforeHeight: 9999})
// s.stopBoundary = {StopBeforeHeight: 9999, source: manual}
// Version beacon boundary at 1000 is now completely bypassed.
``` [5](#0-4)

func (s *StopControl) SetStopParameters(
	stop StopParameters,
) error {
	s.Lock()
	defer s.Unlock()

	boundary := stopBoundary{
		StopParameters: stop,
		source:         stopBoundarySourceManual,
	}

	return s.setStopParameters(boundary)
}

func (s *StopControl) setStopParameters(
	stopBoundary stopBoundary,
) error {
	log := s.log.With().
		Stringer("old_stop", s.stopBoundary).
		Stringer("new_stop", stopBoundary).
		Logger()

	err := s.validateStopChange(stopBoundary)
	if err != nil {
		log.Info().Err(err).Msg("cannot set stopHeight")
		return err
	}

	log.Info().Msg("new stop set")
	s.stopBoundary = stopBoundary

	return nil
}
