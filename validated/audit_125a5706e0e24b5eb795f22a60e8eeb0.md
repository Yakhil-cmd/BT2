### Title
Unauthorized Native Token Transfer via Missing Caller Validation in Bank Precompile `transfer` Method - (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The `BankContract.Run()` function in the bank precompile accepts an arbitrary `sender` address as the first ABI argument for the `transfer` method and uses it directly as the `from` address in `bankKeeper.SendCoins()`, without verifying that `sender == contract.Caller()`. Any EVM contract can pass a victim's address as `sender` and drain that victim's `evm/<callerContractAddress>` native tokens without authorization.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` case unpacks three ABI arguments: `sender`, `recipient`, and `amount`. The `sender` argument is taken directly from the call data and used as the `from` address for the native bank transfer:

```go
sender := args[0].(common.Address)
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())
...
if err := bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt)); err != nil {
```

There is no check that `sender == contract.Caller()`. The `denom` is correctly scoped to `EVMDenom(contract.Caller())` (i.e., `evm/<callerContractAddress>`), but the `from` address is fully attacker-controlled. Any contract can call `bank.transfer(victimAddress, attackerAddress, amount)` and the precompile will execute `bankKeeper.SendCoins(victim, attacker, coins)` without any authorization from the victim.

This is the direct analog of the external report's pattern: the wrong entity is validated (the denom is scoped to the caller, but the spending authority is not). [1](#0-0) 

Compare with the `mint`/`burn` case, which correctly uses `recipient` (the argument) as the target but does not need a caller check because minting/burning is a privileged operation of the token-issuing contract. The `transfer` case is different: it moves tokens *from* a user, which requires that user's authorization — but no such check exists. [2](#0-1) 

### Impact Explanation
**Critical**: An unprivileged attacker who controls any EVM contract can drain `evm/<contractAddress>` native tokens from any victim address that holds them. The `bankKeeper.SendCoins()` call executes an unconditional Cosmos-side native token transfer, bypassing all EVM-level approval mechanisms. This constitutes an unauthorized transfer of precompile-controlled assets. [3](#0-2) 

### Likelihood Explanation
The `evm/` denom native tokens are minted when users call `bank.mint()` from a contract (as shown in `TestBank.sol`'s `moveToNative`). Any contract that issues such tokens and has users holding them is exploitable. The attacker only needs to deploy a contract (or control an existing one) and call the bank precompile's `transfer` method with a victim's address as the first argument. [4](#0-3) 

### Recommendation
Add a check that the `sender` argument equals `contract.Caller()` before executing the transfer:

```go
case TransferMethodName:
    ...
    sender := args[0].(common.Address)
    // Enforce that only the calling contract can authorize transfers from an address
    if sender != contract.Caller() {
        return nil, errors.New("sender must be the calling contract")
    }
    ...
```

This mirrors the authorization model used in `mint`/`burn`, where the denom is scoped to `contract.Caller()` and only that contract can trigger the operation. [5](#0-4) 

### Proof of Concept

1. Attacker deploys `MaliciousContract` which calls the bank precompile at `0x0000000000000000000000000000000000000064`.
2. Victim calls `bank.mint(victimAddress, 1000)` from `MaliciousContract`, receiving 1000 `evm/<MaliciousContractAddress>` native tokens.
3. Attacker calls `MaliciousContract.steal(victimAddress, attackerAddress, 1000)`, which internally calls:
   ```solidity
   bank.transfer(victimAddress, attackerAddress, 1000)
   ```
4. The bank precompile executes `bankKeeper.SendCoins(victim, attacker, 1000 evm/<MaliciousContractAddress>)` with no authorization check.
5. Victim's 1000 native tokens are transferred to the attacker. [6](#0-5) [7](#0-6)

	// parse input
	methodID := contract.Input[:4]

	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}

		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
