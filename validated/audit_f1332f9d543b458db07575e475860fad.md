### Title
Revoking key-0 does not revoke logical aliases (duplicate keys) sharing the same key material — (`fvm/environment/accounts.go`, `fvm/environment/account-key-metadata/metadata.go`)

---

### Summary

The revoked-status for key-0 is stored exclusively in the `apk_0` register, while the revoked-status for every key at index ≥ 1 is stored independently in the `keyMetadataBytes` section of the AccountStatus register, indexed by logical key index. When a duplicate key is added (deduplication maps key-1's `storedKeyIndex` to 0, sharing key-0's raw key material), revoking key-0 writes only to `apk_0`. The `keyMetadataBytes` entry for key-1 is never touched. The transaction verifier reads `GetRuntimeAccountPublicKey(addr, 1)` which returns `Revoked=false`, and the transaction signed with the shared private key material under key-index=1 is accepted.

---

### Finding Description

**Storage split for revoked status:**

- Key-0: revoked bit lives inside the `apk_0` register, encoded with the full `AccountPublicKey` struct. [1](#0-0) 

- Keys ≥ 1: revoked bit lives in `keyMetadataBytes` (part of AccountStatus), at position `keyIndex-1` in the run-length-encoded weight/revoked list. [2](#0-1) [3](#0-2) 

**Revocation of key-0 only touches `apk_0`:**

`RevokeAccountPublicKey(addr, 0)` dispatches to `revokeAccountPublicKey0()`, which reads the `apk_0` register, flips `Revoked=true`, and writes it back. It never reads or modifies `keyMetadataBytes`. [4](#0-3) 

**Reading key-1 returns its own independent revoked bit:**

`GetRuntimeAccountPublicKey(addr, 1)` calls `status.AccountPublicKeyMetadata(1)`, which calls `accountkeymetadata.GetKeyMetadata(keyMetadataBytes, 1, deduplicated)`. This returns `revoked` from `keyMetadataBytes` at position `keyIndex-1 = 0` — the revoked status of key-1, not key-0. The `storedKeyIndex` returned is 0 (pointing to the same raw key bytes), but the `revoked` field is sourced from `keyMetadataBytes`, not from `apk_0`. [5](#0-4) [6](#0-5) 

**Transaction verifier checks only the per-logical-index revoked bit:**

`getAccountKeysAndAggregateWeights` calls `GetRuntimeAccountPublicKey(addr, keyIndex)` for each signature's key index and rejects only if `accountKey.Revoked == true`. For key-index=1 after only key-0 has been revoked, `Revoked` is `false`, so the check passes. [7](#0-6) 

**No test covers the partial-revocation case:**

The existing test `"account with 2 duplicate key"` revokes both key-0 and key-1 in a loop. It never tests the scenario where only key-0 is revoked and key-1 is subsequently queried or used to sign. [8](#0-7) 

---

### Impact Explanation

An account owner who revokes key-0 (e.g., after discovering a key compromise) believes they have fully invalidated that key material. If a duplicate key-1 was previously appended (by the attacker during a period of temporary access, or by the owner themselves for multi-sig purposes), the underlying private key remains usable for signing via key-index=1. The attacker retains full signing capability — including proposer, payer, and authorizer roles — on an account the owner believes is secured.

---

### Likelihood Explanation

The attack requires the attacker to have had prior signing access to the account (to append key-1). This is a realistic post-compromise scenario: an attacker who briefly controls an account plants a duplicate key, then the owner revokes key-0 thinking they have cleaned up. The deduplication feature is new and the partial-revocation invariant is not documented or enforced anywhere in the codebase.

---

### Recommendation

When revoking a key at any logical index, the runtime must also revoke all other logical key indices whose `storedKeyIndex` maps to the same stored key. Concretely:

1. After revoking key-0, scan `keyMetadataBytes` for all entries whose `storedKeyIndex == 0` and set their revoked bit.
2. After revoking key-N (N ≥ 1), resolve its `storedKeyIndex`, then revoke key-0 if `storedKeyIndex == 0`, and revoke all other key-M entries in `keyMetadataBytes` with the same `storedKeyIndex`.

Alternatively, store the revoked bit per `storedKeyIndex` rather than per logical key index, so a single revocation atomically covers all aliases.

func revokeAccountPublicKey0(
	a Accounts,
	address flow.Address,
) error {
	accountPublicKey0RegisterID := flow.AccountPublicKey0RegisterID(address)

	publicKey, err := a.GetValue(accountPublicKey0RegisterID)
	if err != nil {
		return err
	}

	const keyIndex = uint32(0)

	if len(publicKey) == 0 {
		return errors.NewAccountPublicKeyNotFoundError(
			address,
			keyIndex)
	}

	decodedPublicKey, err := flow.DecodeAccountPublicKey(publicKey, keyIndex)
	if err != nil {
		return fmt.Errorf(
			"failed to decode account public key 0: %w",
			err)
	}

	decodedPublicKey.Revoked = true

	encodedPublicKey, err := flow.EncodeAccountPublicKey(decodedPublicKey)
	if err != nil {
		encoded, _ := decodedPublicKey.MarshalJSON()
		return errors.NewValueErrorf(
			string(encoded),
			"failed to encode revoked account public key 0: %w",
			err)
	}

	return a.SetValue(accountPublicKey0RegisterID, encodedPublicKey)

// GetKeyMetadata returns weight, revoked status, and stored key index for given account public key index.
func GetKeyMetadata(b []byte, keyIndex uint32, deduplicated bool) (
	weight uint16,
	revoked bool,
	storedKeyIndex uint32,
	err error,
) {
	// Key metadata only stores weight and revoked status for keys at index > 0.

	if keyIndex == 0 {
		err = errors.NewKeyMetadataUnexpectedKeyIndexError("failed to query key metadata", 0)
		return
	}

	// Get raw weight and revoked status bytes
	weightAndRevokedStatusBytes, rest, err := parseWeightAndRevokedStatusFromKeyMetadataBytes(b)
	if err != nil {
		return 0, false, 0, err
	}

	// Get weight and revoked status for given account key index
	revoked, weight, err = getWeightAndRevokedStatus(weightAndRevokedStatusBytes, keyIndex-1)
	if err != nil {
		return 0, false, 0, err
	}

	// If keys are not deduplicated, storedKeyIndex is the same as the given keyIndex.
	if !deduplicated {
		return weight, revoked, keyIndex, nil
	}

	// Get raw key index mapping bytes
	startIndexForMapping, mappingBytes, _, err := parseStoredKeyMappingFromKeyMetadataBytes(rest)
	if err != nil {
		return 0, false, 0, err
	}

	// StoredKeyIndex is the same as the given keyIndex if deduplication happens afterwards.
	if keyIndex < startIndexForMapping {
		return weight, revoked, keyIndex, nil
	}

	// Get stored key index from mapping
	storedKeyIndex, err = getStoredKeyIndexFromMappings(mappingBytes, keyIndex-startIndexForMapping)
	if err != nil {
		return 0, false, 0, err
	}

	return weight, revoked, storedKeyIndex, nil
}

// SetRevokedStatus revokes key and returns encoded key metadata.
// NOTE: b may be modified.
func SetRevokedStatus(b []byte, keyIndex uint32) ([]byte, error) {
	// Key metadata only stores weight and revoked status for keys at index > 0.

	if keyIndex == 0 {
		return nil, errors.NewKeyMetadataUnexpectedKeyIndexError("failed to set revoked status", 0)
	}

	weightAndRevokedStatusBytes, rest, err := parseWeightAndRevokedStatusFromKeyMetadataBytes(b)
	if err != nil {
		return nil, err
	}

	newWeightAndRevokedStatusBytes, err := setRevokedStatus(slices.Clone(weightAndRevokedStatusBytes), keyIndex-1)
	if err != nil {
		return nil, err
	}

	newB := make([]byte, lengthPrefixSize+len(newWeightAndRevokedStatusBytes)+len(rest))
	off := 0

	binary.BigEndian.PutUint32(newB, uint32(len(newWeightAndRevokedStatusBytes)))
	off += 4

	n := copy(newB[off:], newWeightAndRevokedStatusBytes)
	off += n

	copy(newB[off:], rest)
	return newB, nil
}

func (a *StatefulAccounts) GetRuntimeAccountPublicKey(
	address flow.Address,
	keyIndex uint32,
) (
	flow.RuntimeAccountPublicKey,
	error,
) {
	err := a.accountPublicKeyIndexInRange(address, keyIndex)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, err
	}

	if keyIndex == 0 {
		key, err := getAccountPublicKey0(a, address)
		if err != nil {
			return flow.RuntimeAccountPublicKey{}, fmt.Errorf("failed to get account public key at index %d for %s: %w", keyIndex, address, err)
		}
		return flow.RuntimeAccountPublicKey{
			Index:     keyIndex,
			PublicKey: key.PublicKey,
			SignAlgo:  key.SignAlgo,
			HashAlgo:  key.HashAlgo,
			Weight:    key.Weight,
			Revoked:   key.Revoked,
		}, nil
	}

	status, err := a.getAccountStatus(address)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, err
	}

	// Get account public key metadata.
	weight, revoked, storedKeyIndex, err := status.AccountPublicKeyMetadata(keyIndex)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, fmt.Errorf("failed to get account public key at index %d for %s: %w", keyIndex, address, err)
	}

	// Get stored public key.
	storedKey, err := getStoredPublicKey(a, address, storedKeyIndex)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, fmt.Errorf("failed to get account public key at index %d for %s: %w", keyIndex, address, err)
	}

	return flow.RuntimeAccountPublicKey{
		Index:     keyIndex,
		PublicKey: storedKey.PublicKey,
		SignAlgo:  storedKey.SignAlgo,
		HashAlgo:  storedKey.HashAlgo,
		Weight:    int(weight),
		Revoked:   revoked,
	}, nil
}

func (a *StatefulAccounts) RevokeAccountPublicKey(
	address flow.Address,
	keyIndex uint32,
) error {
	err := a.accountPublicKeyIndexInRange(address, keyIndex)
	if err != nil {
		return err
	}

	if keyIndex == 0 {
		return revokeAccountPublicKey0(a, address)
	}

	status, err := a.getAccountStatus(address)
	if err != nil {
		return err
	}

	err = status.RevokeAccountPublicKey(keyIndex)
	if err != nil {
		return fmt.Errorf("failed to revoke public key at index %d for %s: %w", keyIndex, address, err)
	}

	return a.setAccountStatusAfterAccountStatusSizeChange(address, status)
}
