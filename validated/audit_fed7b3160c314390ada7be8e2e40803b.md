### Title
Paused/Blocked External CRC21 Contract Causes Permanent IBC Receive Failure, Denying Funds to Receiver - (File: x/cronos/middleware/conversion_middleware.go)

### Summary
When an IBC denom is mapped to an **external** CRC21 contract, `IBCConversionModule.OnRecvPacket` calls `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` → `CallModuleCRC21(contract, "mint_by_cronos_module", ...)`. If the external contract's `mint_by_cronos_module` reverts for any reason (pause mechanism, blocklist, ownership transfer, etc.), the entire IBC receive fails with an error acknowledgement. Because `commit()` is never called on the shared `cacheCtx`, the underlying IBC voucher credit is also rolled back — the receiver gets nothing, not even the raw IBC vouchers.

### Finding Description

**Root cause — `OnRecvPacket` couples the underlying transfer and the conversion in a single `cacheCtx`:** [1](#0-0) 

Both `im.app.OnRecvPacket` (which credits the receiver with IBC vouchers) and `ConvertVouchersToEvmCoins` (which mints CRC21 tokens) execute against the same `cacheCtx`. If conversion fails, `commit()` is never reached, so the voucher credit is also discarded and an error ack is returned.

**Propagation path — `ConvertVouchersToEvmCoins` calls into the external contract:** [2](#0-1) 

For any non-CRO IBC denom, `ConvertCoinFromNativeToCRC21` is called. [3](#0-2) 

At line 137, `CallModuleCRC21(ctx, contract, "mint_by_cronos_module", sender, coin.Amount.BigInt())` is called on the mapped contract. If the contract is an **external** one (registered via `SetExternalContractForDenom`), its `mint_by_cronos_module` can contain arbitrary logic.

**`CallModuleCRC21` treats any EVM revert as a hard error:** [4](#0-3) 

`res.Failed()` → error returned → propagates all the way back to `OnRecvPacket` → error ack emitted → `commit()` skipped → receiver gets nothing.

**`canBeConverted` triggers the path for any denom with a registered contract:** [5](#0-4) 

Any denom with an external contract mapping satisfies `canBeConverted`, so the conversion is always attempted.

**Registration allows arbitrary external contracts with no behavioral validation:** [6](#0-5) 

`ensureContractCode` only verifies that bytecode exists at the address; it does not validate that `mint_by_cronos_module` will always succeed.

### Impact Explanation

**High — Permanent or long-lived inability for honest users to process IBC bridge/conversion flows.**

Once an external contract's `mint_by_cronos_module` begins reverting (pause, blocklist, self-destruct, ownership change), every IBC receive packet for that denom returns an error ack. The receiver is permanently denied their funds until the CronosAdmin manually deletes the mapping. During that window, all users bridging that token to Cronos are silently refunded on the source chain with no recourse on the Cronos side.

### Likelihood Explanation

The external contract owner is an unprivileged third party from the Cronos protocol's perspective — they hold no `CronosAdmin` role. After the CronosAdmin maps the denom to their contract, the external contract owner can unilaterally:
- Add and activate a pause mechanism
- Add a blocklist that includes the module address (`0x89A7EF2F08B1c018D5Cc88836249b84Dd5392905`)
- Upgrade the proxy to a version that always reverts `mint_by_cronos_module`

None of these actions require any Cronos admin key. The attack is reachable by any relayer submitting a standard IBC `MsgRecvPacket`.

### Recommendation

1. **Decouple the underlying transfer from the conversion.** Commit the underlying IBC voucher credit unconditionally; only attempt conversion in a nested `cacheCtx`. If conversion fails, log and continue (matching the pattern already used in `OnAcknowledgementPacket` and `OnTimeoutPacket`):

```go
ack := im.app.OnRecvPacket(ctx, channelVersion, packet, relayer)
if ack.Success() && im.canBeConverted(ctx, denom) {
    convCtx, convCommit := ctx.CacheContext()
    if err := im.cronoskeeper.ConvertVouchersToEvmCoins(convCtx, ...); err != nil {
        im.cronoskeeper.Logger(ctx).Error("conversion failed, receiver keeps vouchers", ...)
    } else {
        convCommit()
    }
}
return ack
```

2. **Add behavioral validation at mapping registration time** — call `mint_by_cronos_module` with a zero amount in a simulated context and reject the mapping if it reverts.

3. **Emit a Cosmos event** when conversion is skipped so monitoring can detect stuck denoms quickly.

### Proof of Concept

1. CronosAdmin calls `MsgUpdateTokenMapping` mapping `ibc/XXXX` → `ExternalContract` (a contract with a `pause()` function).
2. External contract owner calls `pause()` on `ExternalContract`.
3. User on chain A sends `ibc/XXXX` tokens to a Cronos address via IBC.
4. Relayer submits `MsgRecvPacket` on Cronos.
5. `IBCConversionModule.OnRecvPacket` is called:
   - `im.app.OnRecvPacket(cacheCtx, ...)` succeeds — vouchers credited in `cacheCtx`.
   - `canBeConverted` returns `true`.
   - `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` → `CallModuleCRC21(ExternalContract, "mint_by_cronos_module", ...)`.
   - `ExternalContract.mint_by_cronos_module` reverts (paused).
   - `CallModuleCRC21` returns error.
   - `OnRecvPacket` returns `NewErrorAcknowledgement` — `commit()` never called.
6. Receiver's Cronos address has zero balance. Source chain refunds sender.
7. Every subsequent IBC packet for `ibc/XXXX` fails identically until CronosAdmin deletes the mapping.

// CallModuleCRC21 call a method of ModuleCRC21 contract
func (k Keeper) CallModuleCRC21(ctx sdk.Context, contract common.Address, method string, args ...interface{}) ([]byte, error) {
	data, err := types.ModuleCRC21Contract.ABI.Pack(method, args...)
	if err != nil {
		return nil, err
	}
	_, res, err := k.CallEVM(ctx, &contract, data, big.NewInt(0), DefaultGasCap)
	if err != nil {
		return nil, err
	}
	if res.Failed() {
		return nil, fmt.Errorf("call contract failed: %s, %s, %s", contract.Hex(), method, res.Ret)
	}
	return res.Ret, nil
}
