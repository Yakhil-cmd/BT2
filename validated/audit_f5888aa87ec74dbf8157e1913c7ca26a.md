### Title
Unprivileged Caller Can Permanently Freeze All Staker Consensus Rewards via `update_rewards` with `disable_rewards=true` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and accepts a `disable_rewards` boolean parameter. Any unprivileged caller can invoke it with `disable_rewards=true` to advance the global `last_reward_block` counter without distributing any rewards. Because only one call per block is permitted (enforced by the global `last_reward_block` guard), an attacker who front-runs every block with this call permanently prevents all stakers from receiving consensus rewards.

---

### Finding Description

`update_rewards` is exposed as a public ABI function under `StakingRewardsManagerImpl`. Its only gate is `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no role or identity check is performed. [1](#0-0) 

The function enforces a single-call-per-block invariant using the **global** storage variable `last_reward_block`: [2](#0-1) 

Regardless of the `disable_rewards` flag, `last_reward_block` is always written before the early-return branch is evaluated: [3](#0-2) 

When `disable_rewards=true`, the function returns immediately after writing `last_reward_block`, distributing nothing: [4](#0-3) 

Because `last_reward_block` is a single global slot (not per-staker), consuming it with `disable_rewards=true` blocks reward distribution for **every** staker in that block. [5](#0-4) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield for all stakers.**

An attacker who calls `update_rewards(any_active_staker, true)` in every block prevents the consensus reward pipeline from ever executing `_update_rewards`. Stakers accumulate zero `unclaimed_rewards_own` and pools receive zero rewards. The attack is continuous and self-sustaining: each block the attacker spends one cheap transaction; every staker and delegator in the protocol loses that block's rewards forever.

---

### Likelihood Explanation

**High.** The attack requires:
- No privileged role or special key.
- No capital at risk.
- Only the gas cost of one transaction per block.

On Starknet, blocks are produced every few seconds. The attacker simply monitors the chain and submits `update_rewards(any_active_staker_address, true)` at the start of each block. There is no economic barrier and no on-chain mechanism to prevent it.

---

### Recommendation

Restrict `update_rewards` to an authorized caller (e.g., the attestation contract, a dedicated rewards-manager role, or the staker's own operational address). The simplest fix is to add a role check analogous to the one already present in `update_rewards_from_attestation_contract`: [6](#0-5) 

Alternatively, remove the `disable_rewards` parameter entirely and handle the zero-balance / pre-consensus cases through the existing `is_pre_consensus()` guard and the balance assertion already in the function body.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker identifies any currently active staker address `S` (readable from the public `stakers` Vec or events).
3. At the start of block `N`, attacker submits:
   ```
   staking_contract.update_rewards(S, disable_rewards=true)
   ```
4. The function passes all checks (contract unpaused, caller non-zero, staker active, balance non-zero), writes `last_reward_block = N`, then returns early — **no rewards distributed**.
5. Any subsequent call to `update_rewards` in block `N` (by the legitimate consensus layer or any staker) reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number > last_reward_block` is now `false`.
6. Attacker repeats step 3 in block `N+1`, `N+2`, … indefinitely.
7. All stakers' `unclaimed_rewards_own` remain at zero; all pool `cumulative_rewards_trace` entries stop growing. Rewards are permanently frozen. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
