### Title
Unprivileged Caller Can Suppress All Consensus Block Rewards by Calling `update_rewards` with `disable_rewards: true` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is callable by any unprivileged address. It accepts a `disable_rewards: bool` parameter. When set to `true`, the function still writes the current block number to the global `last_reward_block` storage slot — consuming the per-block reward slot — but returns without distributing any rewards. An attacker can call this every block to permanently deny all stakers their consensus block rewards.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero. No role or identity check is performed. [1](#0-0) 

The function enforces a single-call-per-block invariant via a global `last_reward_block` variable: [2](#0-1) 

Critically, `last_reward_block` is written **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

Because `last_reward_block` is a single global slot shared across all stakers, any caller who invokes `update_rewards(any_active_staker, true)` in block N:

1. Passes all precondition checks (staker exists, is active, has non-zero balance).
2. Writes `N` to `last_reward_block`.
3. Returns immediately — no rewards are distributed.

Every subsequent call to `update_rewards` in block N will revert with `REWARDS_ALREADY_UPDATED`, including legitimate calls from actual stakers or their infrastructure. The attacker repeats this every block.

---

### Impact Explanation

**High — Temporary (indefinite) freezing of unclaimed yield.**

All stakers are denied consensus block rewards for every block the attacker sustains the attack. Rewards that should have accrued are simply never computed or stored. Because the attack is cheap (one transaction per block on Starknet) and requires no special privilege, it can be sustained indefinitely, making the yield freeze effectively permanent in practice.

---

### Likelihood Explanation

**High.** Any address can call `update_rewards`. Active staker addresses are publicly visible on-chain (emitted in `NewStaker` events and stored in the public `stakers` vector). The attacker only needs one valid active staker address and enough gas to submit one transaction per block. [4](#0-3) 

---

### Recommendation

Restrict who may call `update_rewards` with `disable_rewards: true`. Options include:

1. **Remove `disable_rewards` from the public ABI** — handle the "skip rewards" logic internally (e.g., keyed on `is_pre_consensus()` only, which is already checked inside the function).
2. **Add a role check** — require the caller to hold a specific role (e.g., `SECURITY_AGENT` or a dedicated rewards-manager role) when `disable_rewards` is `true`.
3. **Write `last_reward_block` only after the `disable_rewards` guard** — so that a call with `disable_rewards: true` does not consume the per-block slot.

---

### Proof of Concept

```
// Attacker script — run once per block
let staker_address = any_active_staker_from_chain();   // public info
staking_dispatcher.update_rewards(staker_address, disable_rewards: true);
// last_reward_block is now set to current block.
// All legitimate update_rewards calls in this block revert with REWARDS_ALREADY_UPDATED.
// No rewards are distributed.
// Repeat next block.
```

The attack path is:

1. Attacker reads any active staker address from on-chain events or the `stakers` vector.
2. Each block, attacker calls `update_rewards(staker_address, true)`.
3. `last_reward_block` is updated; the early-return fires; zero rewards are distributed.
4. Any honest party attempting `update_rewards(staker_address, false)` in the same block hits the `REWARDS_ALREADY_UPDATED` assertion and reverts.
5. Sustained indefinitely, all consensus block rewards are frozen. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
