### Title
Lax Byte-Length Validation in gRPC `convert.Address()` Silently Pads Short Addresses, Resolving to a Different Account - (File: `engine/common/rpc/convert/validate.go`)

### Summary
The gRPC Access API's `convert.Address()` function accepts raw address byte slices of any non-zero length without enforcing the canonical 8-byte (`flow.AddressLength`) size. The underlying `flow.BytesToAddress()` silently left-pads short inputs with zero bytes, producing a structurally different address. If the padded address passes `chain.IsValid()`, all account-query RPCs proceed against the wrong account and return data for it without any error to the caller.

### Finding Description
`convert.Address()` is the central address validator for all gRPC account-query handlers in `engine/access/rpc/handler.go`:

```go
// engine/common/rpc/convert/validate.go
func Address(rawAddress []byte, chain flow.Chain) (flow.Address, error) {
    if len(rawAddress) == 0 {                          // only rejects empty
        return flow.EmptyAddress, status.Error(...)
    }
    address := flow.BytesToAddress(rawAddress)         // silently pads short input
    if !chain.IsValid(address) {
        return flow.EmptyAddress, status.Errorf(...)
    }
    return address, nil
}
``` [1](#0-0) 

`flow.BytesToAddress()` is explicitly documented to left-pad with zeroes when the input is shorter than 8 bytes:

```go
// model/flow/address.go
// If b is smaller than 8, b will be appended by zeroes at the front.
func BytesToAddress(b []byte) Address {
    var a Address
    if len(b) > AddressLength {
        b = b[len(b)-AddressLength:]
    }
    copy(a[AddressLength-len(b):], b)   // zero-fills leading bytes
    return a
}
``` [2](#0-1) 

A caller sending 4 bytes `[0xAB, 0xCD, 0xEF, 0x01]` receives the address `0x00000000abcdef01` — a completely different account — with no error. The same file already contains `IsNonEmptyAddress()` which correctly enforces `len(address) == flow.AddressLength`, but this check is never applied inside `Address()`: [3](#0-2) 

By contrast, the REST API parser correctly enforces exactly 16 hex characters (8 bytes) before proceeding: [4](#0-3) 

The gRPC handler uses `convert.Address()` for every account-related RPC: [5](#0-4) [6](#0-5) [7](#0-6) [8](#0-7) 

### Impact Explanation
Any unprivileged gRPC caller (dApp, wallet, SDK) that sends a short address byte slice to `GetAccount`, `GetAccountAtLatestBlock`, `GetAccountAtBlockHeight`, `GetAccountBalanceAtLatestBlock`, `GetAccountBalanceAtBlockHeight`, `GetAccountKeyAtLatestBlock`, `GetAccountKeyAtBlockHeight`, `GetAccountKeysAtLatestBlock`, or `GetAccountKeysAtBlockHeight` will silently receive account data (balance, public keys, contracts) for a different account. No error is returned. A dApp or wallet relying on this data will display or act on information belonging to the wrong account, without any indication that the address was malformed.

### Likelihood Explanation
The gRPC Access API is the primary programmatic interface for Flow SDKs and dApps. Address bytes are passed directly as protobuf `bytes` fields. A truncated address (e.g., from a copy-paste error, encoding bug, or off-by-one in a client SDK) will be silently accepted. Flow's linear-code address space contains 2^45 valid addresses out of 2^64 possible 8-byte values; a short input padded with leading zeros has a non-negligible chance of landing on a valid address for low-index accounts (e.g., service address `0x0000000000000001` is reachable from a 1-byte input `[0x01]`). The discrepancy between the REST API (strict 16-char enforcement) and the gRPC API (no length check) makes this an inconsistency that is likely to be hit in practice.

### Recommendation
Add an explicit length check in `convert.Address()` before calling `flow.BytesToAddress()`:

```go
func Address(rawAddress []byte, chain flow.Chain) (flow.Address, error) {
    if len(rawAddress) != flow.AddressLength {
        return flow.EmptyAddress, status.Errorf(codes.InvalidArgument,
            "address must be %d bytes, got %d", flow.AddressLength, len(rawAddress))
    }
    address := flow.BytesToAddress(rawAddress)
    if !chain.IsValid(address) {
        return flow.EmptyAddress, status.Errorf(codes.InvalidArgument,
            "address %s is invalid for chain %s", address, chain)
    }
    return address, nil
}
```

This mirrors the existing `IsNonEmptyAddress()` helper and the REST API's strict regex enforcement.

### Proof of Concept
1. Connect to a Flow Access node gRPC endpoint.
2. Send a `GetAccountAtLatestBlock` request with `address = []byte{0x01}` (1 byte).
3. `convert.Address()` passes the non-empty check.
4. `flow.BytesToAddress([]byte{0x01})` returns `0x0000000000000001` (the Flow service address).
5. `chain.IsValid(0x0000000000000001)` returns `true`.
6. The handler returns the service account's data — balance, keys, contracts — with no error, even though the caller supplied a 1-byte input that bears no resemblance to the service address. [1](#0-0) [9](#0-8)
