### Title
Stale Granted Permissions Not Revoked on `CronosAdmin` Change — (`x/cronos/keeper/permissions.go`)

### Summary
When the `CronosAdmin` parameter is updated via governance (`MsgUpdateParams`), permissions previously granted to addresses via `MsgUpdatePermissions` are **not automatically cleared**. Those addresses retain `CanChangeTokenMapping` (and `CanTurnBridge`) indefinitely, even though the new admin never authorized them. This is a direct analog to the external report's "previous operator power is not revoked" class: an authority is stored at grant-time and persists after the principal that issued it is replaced.

### Finding Description

`UpdatePermissions` writes permissions into the KV store: [1](#0-0) 

`HasPermission` checks the **current** admin first, then falls through to stored permissions for any other account: [2](#0-1) 

`UpdateParams` (the governance handler that changes `CronosAdmin`) only calls `SetParams`; it performs **no sweep of the permissions store**: [3](#0-2) 

`UpdateTokenMapping` enforces only a live `HasPermission` check, which passes for any address that still has stored bits: [4](#0-3) 

Concrete stale-authority path:

1. Admin A grants `CanChangeTokenMapping` to address X via `MsgUpdatePermissions`. The bit is written to `AdminToPermissionsKey(X)`.
2. Governance passes `MsgUpdateParams` changing `CronosAdmin` to B. Only the param is updated; `AdminToPermissionsKey(X)` is untouched.
3. X is now unauthorized from B's perspective, but `HasPermission(ctx, [X], CanChangeTokenMapping)` still returns `true` because the stored bit survives.
4. X submits `MsgUpdateTokenMapping` and `UpdateTokenMapping` accepts it.

The old admin themselves can also pre-emptively grant themselves stored permissions before the governance vote finalizes, ensuring they retain `CanChangeTokenMapping` after being replaced — exactly the "operator knows they are exchanged in the future" scenario described in the external report.

### Impact Explanation

With `CanChangeTokenMapping`, X can:

- Call `RegisterOrUpdateTokenMapping` with an empty `Contract` field to **delete** the active mapping for any denom (e.g., `gravity0x…` → legitimate CRC21 contract C1).
- Immediately re-register that denom pointing to a malicious contract C2 X controls.

After remapping, `ConvertCoinFromNativeToCRC21` resolves the denom to C2: [5](#0-4) 

For non-source denoms the keeper sends native tokens to `sdk.AccAddress(C2.Bytes())` and then calls `mint_by_cronos_module` on C2. A malicious C2 that accepts the call but does not mint (or mints to X) causes the user's native tokens to be permanently transferred to C2 with no CRC21 tokens returned — a direct theft of bridged assets.

The same remapping corrupts the EVM-hook path: `SendToIbcHandler` and `SendToAccountHandler` both call `GetDenomByContract` at execution time: [6](#0-5) [7](#0-6) 

After remapping, only C2's events are processed; C1's events fail with "contract not connected to native token," permanently breaking bridge operations for all existing C1 holders.

**Impact category**: High — Corruption of token mappings / denom-contract binding with direct security impact; Bypass of Cronos admin / token-mapping authorization checks.

### Likelihood Explanation

- The old admin (or any address they granted permissions to) is the attacker; no external key compromise is required if the old admin is itself malicious or foresaw replacement.
- Admin rotation via governance is a documented, expected operation (`MsgUpdateParams`).
- The permissions store is never inspected or cleared during a governance upgrade, so the window is permanent until the new admin explicitly revokes each stale entry — which requires knowing they exist.

### Recommendation

In `UpdateParams`, after `SetParams` succeeds, iterate over all entries under `KeyPrefixAdminToPermissions` and delete them (or set them to `0`). This mirrors the external report's fix: re-validate authority against the **current** principal at the point of use, rather than relying on a stored grant that may have outlived the principal that issued it.

Alternatively, add an explicit `MsgRevokeAllPermissions` that governance must include in the same proposal as the admin change, and document this as a required step.

### Proof of Concept

```
// State before admin change
SetPermissions(ctx, X, CanChangeTokenMapping)   // stored in KV
SetParams(ctx, {CronosAdmin: A, ...})

// Governance proposal executes
SetParams(ctx, {CronosAdmin: B, ...})           // permissions store untouched

// X submits MsgUpdateTokenMapping — accepted
HasPermission(ctx, [X], CanChangeTokenMapping)
  → admin = B  (X != B, skip)
  → GetPermissions(ctx, X) = CanChangeTokenMapping  ← still set
  → returns true

// X deletes legitimate mapping and installs malicious contract
RegisterOrUpdateTokenMapping(ctx, {Denom: "gravity0x…", Contract: ""})   // delete C1
RegisterOrUpdateTokenMapping(ctx, {Denom: "gravity0x…", Contract: C2})   // install C2

// Victim calls MsgConvertVouchers for "gravity0x…"
ConvertCoinFromNativeToCRC21 → GetContractByDenom → C2
bankKeeper.SendCoins(victim, C2, coins)          // native tokens sent to C2
CallModuleCRC21(C2, "mint_by_cronos_module", …)  // C2 does not mint → victim loses funds
```

func (k Keeper) SetPermissions(ctx sdk.Context, address sdk.AccAddress, permissions uint64) {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := sdk.Uint64ToBigEndian(permissions)
	store.Set(types.AdminToPermissionsKey(address), permissionsBytes)
}

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

// UpdateTokenMapping implements the grpc method
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}
```

func (k msgServer) UpdateParams(goCtx context.Context, msg *types.MsgUpdateParams) (*types.MsgUpdateParamsResponse, error) {
	if msg.Authority != k.authority {
		return nil, errors.Wrapf(govtypes.ErrInvalidSigner, "invalid authority; expected %s, got %s", k.authority, msg.Authority)
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := k.SetParams(ctx, msg.Params); err != nil {
		return nil, err
	}

	return &types.MsgUpdateParamsResponse{}, nil
}
