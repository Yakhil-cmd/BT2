### Title
Signature-Bypass via `skip_stateful_validations` Allows Unauthenticated Invoke Admission for Any Undeployed Account - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator skips the `__validate__` entry point (signature verification) for any invoke transaction with `nonce == 1` when the account has any transaction in the mempool. An attacker who observes a victim's `deploy_account` transaction in the mempool can immediately submit a malicious invoke transaction for the victim's address with an arbitrary/invalid signature, which is admitted to the mempool without any signature check. This blocks the victim's legitimate invoke from being admitted (DuplicateNonce), or forces the victim to pay a fee-escalation premium to displace the attacker's transaction.

### Finding Description

The gateway stateful validator's `skip_stateful_validations` function returns `true` (skip `__validate__`) when three conditions hold simultaneously:

1. The incoming transaction is an `Invoke` with `nonce == 1`
2. The on-chain account nonce is `0` (account not yet deployed)
3. `account_tx_in_pool_or_recent_block(sender_address)` returns `true` [1](#0-0) 

When `skip_validate == true`, `run_validate_entry_point` sets `execution_flags.validate = false`, so the blockifier's `StatefulValidator::validate` call returns immediately without ever calling the account's `__validate__` entry point: [2](#0-1) [3](#0-2) 

The `account_tx_in_pool_or_recent_block` check is satisfied by the **victim's own** `deploy_account` transaction already sitting in the mempool: [4](#0-3) 

The stateless validator performs no cryptographic signature verification — it only checks signature **length**: [5](#0-4) 

**Attack path:**

1. Victim broadcasts `deploy_account` (nonce=0) → accepted into mempool.
2. Attacker observes the `deploy_account`, computes the deterministic victim address from its `class_hash`, `salt`, and `constructor_calldata`.
3. Attacker submits `Invoke { sender_address: victim_addr, nonce: 1, calldata: <arbitrary>, signature: <garbage> }` with any valid resource bounds.
4. Gateway stateful validator: `account_nonce = 0`, `tx.nonce = 1`, `account_tx_in_pool_or_recent_block = true` → `skip_validate = true` → `__validate__` is never called → transaction admitted.
5. Victim submits their legitimate invoke (nonce=1). The mempool's `validate_fee_escalation` finds an existing transaction at `(victim_addr, nonce=1)` and either rejects with `DuplicateNonce` or forces the victim to pay ≥10% higher fees to displace the attacker's transaction. [6](#0-5) 

The mempool's `validate_tx` path (called from `validate_by_mempool`) only checks for duplicate tx hash and nonce-too-old, not for duplicate `(address, nonce)` pairs — that check is deferred to `validate_fee_escalation`: [7](#0-6) 

### Impact Explanation

An attacker can admit an invoke transaction with an **invalid/arbitrary signature** into the production mempool for any undeployed account whose `deploy_account` is visible in the mempool. This:

- **Admits an invalid transaction**: the attacker's invoke carries no valid signature yet passes all gateway admission checks.
- **Rejects a valid transaction**: the victim's correctly-signed invoke is denied with `DuplicateNonce`, or the victim is forced to pay a fee premium (≥10% by default) to displace the attacker.

This matches the **High** impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

The attacker's invalid invoke will eventually fail at blockifier execution time (since the batcher creates `AccountTransaction` with `validate: true` by default), but by then the victim's invoke has already been denied admission. The victim must resubmit after the attacker's transaction is evicted. [8](#0-7) 

### Likelihood Explanation

- **No privileged access required**: any unprivileged user can submit an RPC transaction.
- **Trigger is observable**: the victim's `deploy_account` is visible in the public mempool.
- **Victim address is computable**: Starknet deploy addresses are deterministic from public parameters in the `deploy_account` transaction.
- **Cost to attacker**: only the gas fees for submitting the malicious invoke (which will be rejected at execution, so the fee is charged but the attacker controls the amount).
- **Repeatability**: the attacker can re-submit after each eviction cycle, continuously blocking the victim's invoke until the `deploy_account` is included in a block and the account nonce advances past 0.

### Recommendation

Replace the mempool-presence heuristic with an explicit, caller-supplied `deploy_account_tx_hash` binding, mirroring the approach already used in the legacy `PyValidator`: [9](#0-8) 

In the new gateway path, require the client to supply the hash of the `deploy_account` transaction alongside the invoke. In `skip_stateful_validations`, verify that the specific `deploy_account` transaction hash exists in the mempool (not just any transaction from the address). This ensures only the legitimate account owner — who knows the hash of their own `deploy_account` — can trigger the skip-validate path.

Alternatively, if the `deploy_account_tx_hash` binding is not feasible, the gateway should accept the invoke into the mempool but mark it as **pending validation** and defer `__validate__` to the moment the `deploy_account` is committed, rather than permanently waiving signature verification.

### Proof of Concept

```
// Setup: victim submits deploy_account
let victim_deploy = deploy_account_tx(class_hash, salt, constructor_calldata);
gateway.add_transaction(victim_deploy).await; // accepted, nonce=0 in mempool

// Attacker: compute victim address (deterministic)
let victim_addr = compute_contract_address(class_hash, salt, constructor_calldata, deployer=0);

// Attacker: craft malicious invoke with garbage signature
let malicious_invoke = InvokeTransactionV3 {
    sender_address: victim_addr,
    nonce: 1,
    calldata: vec![/* drain funds or arbitrary */],
    signature: vec![Felt::from(0xdeadbeef_u64)], // invalid
    resource_bounds: valid_bounds(),
    ..Default::default()
};

// Gateway accepts it: account_nonce=0, tx.nonce=1,
// account_tx_in_pool_or_recent_block=true (victim's deploy_account is in pool)
// => skip_validate=true => __validate__ never called
gateway.add_transaction(malicious_invoke).await; // ACCEPTED without signature check

// Victim: submit legitimate invoke
let victim_invoke = InvokeTransactionV3 {
    sender_address: victim_addr,
    nonce: 1,
    calldata: vec![/* legitimate call */],
    signature: valid_signature,
    resource_bounds: same_bounds(),
    ..Default::default()
};
// REJECTED: DuplicateNonce (or must pay >=10% fee premium to replace)
gateway.add_transaction(victim_invoke).await; // ERROR
```

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)

    pub fn should_run_stateful_validations(
        &mut self,
        account_tx: &AccountTransaction,
        deploy_account_tx_hash: Option<TransactionHash>,
    ) -> StatefulValidatorResult<bool> {
        if account_tx.tx_type() != TransactionType::InvokeFunction {
            return Ok(true);
        }
        let tx_info = account_tx.create_tx_info();
        let nonce = self.stateful_validator.get_nonce(tx_info.sender_address())?;

        let deploy_account_not_processed =
            deploy_account_tx_hash.is_some() && nonce == Nonce(Felt::ZERO);
        let tx_nonce = tx_info.nonce();
        let is_post_deploy_nonce = Nonce(Felt::ONE) <= tx_nonce;
        let nonce_small_enough_to_qualify_for_validation_skip =
            tx_nonce <= self.max_nonce_for_validation_skip;

        let skip_validate = deploy_account_not_processed
            && is_post_deploy_nonce
            && nonce_small_enough_to_qualify_for_validation_skip;

        Ok(!skip_validate)
    }
