### Title
COA Ownership Proof Signature Replay Across EVM Contracts Due to Missing Context Binding in `validateCOAOwnershipProof` - (File: fvm/evm/stdlib/contract.cdc)

### Summary

`EVM.validateCOAOwnershipProof` does not enforce that `signedData` encodes the `evmAddress` or any context-specific nonce. A valid COA ownership proof observed on-chain (e.g., from a prior ERC-1271 `isValidSignature` call) can be replayed by any unprivileged actor to impersonate the COA owner in a different EVM contract context, as long as the same `_hash` value is reused. The protocol itself documents this gap in a code comment but does not enforce a fix.

### Finding Description

`EVM.validateCOAOwnershipProof` in `fvm/evm/stdlib/contract.cdc` verifies a set of Cadence account signatures over an arbitrary `signedData` blob, then checks that the COA at the given `path` matches `evmAddress`. The function is `access(all)` — callable by any Cadence script or transaction — and accepts fully attacker-controlled inputs.

The root cause is that the function verifies signatures over `signedData` without requiring `signedData` to encode `evmAddress`, a nonce, a contract address, or any other context-binding field:

```cadence
let isValid = keyList.verify(
    signatureSet: signatureSet,
    signedData: signedData,          // ← no enforcement that evmAddress is inside
    domainSeparationTag: "FLOW-V0.0-user"
)
```

The code comment at lines 1003–1009 explicitly acknowledges this:

> "Note: this function does not enforce that `signedData` includes `evmAddress`. In principle, a signature produced for one purpose could be replayed here against a different COA owned by the same Cadence account."

The ERC-1271 entry point in `coa.sol` (`isValidSignature`) calls the Cadence Arch precompile `verifyCOAOwnershipProof(address(this), _hash, _sig)`, passing the caller-supplied `_hash` directly as `signedData`. Any EVM contract that calls `isValidSignature` on a COA will pass its own `_hash`, and the proof verification succeeds for any proof whose `signedData` equals that `_hash` — regardless of which EVM contract originally requested the signature.

**Attack path:**

1. Alice signs `signedData = H` to authorize action A on EVM contract X (e.g., an NFT marketplace order). The proof `{address, path, keyIndices, signatures}` is submitted on-chain and is publicly observable.
2. EVM contract Y (e.g., a governance contract or a token-approval contract) also uses ERC-1271 and happens to present the same hash `H` for a different action B.
3. An attacker calls `Y.someFunction(...)` which internally calls `coaAddress.isValidSignature(H, proof)`.
4. `isValidSignature` calls `verifyCOAOwnershipProof(coaAddress, H, proof)` via the Cadence Arch precompile.
5. `validateCOAOwnershipProof` verifies Alice's signatures over `H` — which are valid — and returns `isValid: true`.
6. Contract Y accepts this as Alice's authorization for action B, which Alice never intended to authorize. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation

An unprivileged attacker who observes a valid COA ownership proof on-chain can replay it against any EVM contract that uses ERC-1271 (`isValidSignature`) for authorization, provided the same `_hash` value is used. This enables unauthorized authorization of EVM contract actions on behalf of a COA owner — e.g., approving token transfers, signing governance votes, or authorizing marketplace orders — without the owner's consent for that specific action. The COA is the native EVM smart-contract wallet type for Flow; unauthorized ERC-1271 authorization directly affects asset control and on-chain governance. [5](#0-4) 

### Likelihood Explanation

The attacker requires no privileged access. The only precondition is observing a valid proof on-chain (all Flow transactions are public). The attack is viable whenever two EVM contracts independently use ERC-1271 with the same hash value, which is a realistic scenario for standardized message formats (e.g., EIP-712 typed data, ERC-20 permit hashes). The `validateCOAOwnershipProof` function is `access(all)` and reachable from any Cadence script or EVM transaction. The protocol comment explicitly acknowledges the risk, confirming the developers are aware it is a real, reachable condition. [6](#0-5) [7](#0-6) 

### Recommendation

Enforce that `signedData` encodes `evmAddress` inside `validateCOAOwnershipProof` before accepting the proof as valid. Concretely, add a pre-condition check that the first 20 bytes of `signedData` match `evmAddress`, or require callers to include a domain separator and nonce. Alternatively, the ERC-1271 `isValidSignature` implementation in `coa.sol` should construct `signedData` as `keccak256(abi.encode(evmAddress, _hash))` before passing it to the precompile, binding the signature to the specific COA address and preventing cross-contract replay. [8](#0-7) [9](#0-8) 

### Proof of Concept

1. Alice creates a COA at `/public/coa` on her Cadence account `0xAlice`. Her COA's EVM address is `0xCOA_A`.
2. EVM contract X (an NFT marketplace) calls `0xCOA_A.isValidSignature(H, proof)` to verify Alice authorized a listing. Alice submits a valid proof: `{address: 0xAlice, path: /public/coa, keyIndices: [0], signatures: [sig_H]}` where `sig_H` is Alice's signature over `H`.
3. The proof is recorded in the transaction on-chain. Attacker observes `{H, sig_H, address: 0xAlice, path: /public/coa, keyIndices: [0]}`.
4. EVM contract Y (a governance contract) presents the same hash `H` for a governance vote. Attacker calls `Y.vote(...)` which internally calls `0xCOA_A.isValidSignature(H, proof)` with the replayed proof.
5. `validateCOAOwnershipProof` is invoked with `signedData = H`, verifies `sig_H` over `H` using Alice's key — valid — then checks that the COA at `/public/coa` matches `0xCOA_A` — also valid. Returns `isValid: true`.
6. Contract Y records a governance vote from Alice's COA that Alice never cast. [10](#0-9) [11](#0-10) [12](#0-11)

func (f *proofVerifier) ComputeGas(input []byte) uint64 {
	// we compute the gas using a fixed base fee and extra fees
	// per signatures. Note that the input data is already trimmed from the function selector
	// and the remaining is ABI encoded of the inputs

	// skip to the encoded signature part of args (skip address and bytes32 data part)
	index := EncodedAddressSize + Bytes32DataReadSize
	// Reading the encoded signature bytes
	encodedSignature, err := ReadBytes(input, index)
	if err != nil {
		// if any error run would anyway fail, so returning any non-zero value here is fine
		return ProofVerifierBaseGas
	}
	// this method would return the number of signatures from the encoded signature data
	// this saves the extra time needed for full decoding
	// given ComputeGas function is called before charging the gas, we need to keep
	// this function as light as possible
	count, err := types.COAOwnershipProofSignatureCountFromEncoded(encodedSignature)
	if err != nil {
		// if any error run would anyway fail, so returning any non-zero value here is fine
		return ProofVerifierBaseGas
	}
	return ProofVerifierBaseGas + uint64(count)*ProofVerifierGasMultiplerPerSignature
}

func (f *proofVerifier) Run(input []byte) ([]byte, error) {
	proof, err := DecodeABIEncodedProof(input)
	if err != nil {
		return nil, err
	}
	verified, err := f.proofVerifier(proof)
	if err != nil {
		return nil, err
	}

	buffer := make([]byte, EncodedBoolSize)
	return buffer, EncodeBool(verified, buffer, 0)
}

// COAOwnershipProofInContext contains all the data
// needed to verify a COAOwnership proof.
// The proof is verified by checking the signatures over the
// input signed data (SignedData), then loading the resource
// capability from the provided path in the proof, and
// at last checking if the EVMAddress of the resource matches
// the provided one.
type COAOwnershipProofInContext struct {
	COAOwnershipProof
	SignedData SignedData
	EVMAddress Address
}

func NewCOAOwnershipProofInContext(sd []byte, addr Address, encodedProof []byte) (*COAOwnershipProofInContext, error) {
	proof, err := COAOwnershipProofFromEncoded(encodedProof)
	if err != nil {
		return nil, err
	}
	return &COAOwnershipProofInContext{
		COAOwnershipProof: *proof,
		SignedData:        sd,
		EVMAddress:        addr,
	}, nil
}
