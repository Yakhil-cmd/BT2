### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Deny Block Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the `StakingRewardsManagerImpl` implementation is callable by any address, despite the protocol specification explicitly requiring it to be restricted to "Only starkware sequencer." Because the function writes to a single global `last_reward_block` storage slot, an unprivileged attacker can front-run the sequencer each block with `disable_rewards: true`, permanently preventing reward distribution for all stakers for that block.

---

### Finding Description

The specification at `docs/spec.md` lines 1644–1645 states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1449–1507 contains no caller check whatsoever:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no assert on get_caller_address() ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distribute rewards ...
}
``` [1](#0-0) 

The `last_reward_block` field is a single global storage variable (not a per-staker map). The guard `current_block_number > self.last_reward_block.read()` enforces that `update_rewards` can only succeed once per block across the entire contract. [2](#0-1) 

When `disable_rewards: true` is passed, the function writes `last_reward_block` and returns immediately without distributing any rewards. [3](#0-2) 

The specification confirms this is the intended privileged path: [4](#0-3) 

---

### Impact Explanation

An attacker who calls `update_rewards(staker_address: <any_valid_active_staker>, disable_rewards: true)` before the sequencer in any block will:

1. Advance `last_reward_block` to the current block number.
2. Cause the function to return early with zero rewards distributed.
3. Make every subsequent call in the same block revert with `REWARDS_ALREADY_UPDATED`.

Because block rewards are computed per-block and the window is exactly one block wide, the missed rewards for that block are **permanently lost** — they are never retroactively credited. This constitutes **permanent freezing of unclaimed yield** for all stakers for every block the attacker front-runs.

The attacker can repeat this every block at minimal cost (one cheap transaction per block), continuously denying all stakers their consensus rewards indefinitely.

This matches the allowed impact: **Permanent freezing of unclaimed yield or unclaimed royalties (High)**.

---

### Likelihood Explanation

- No special role, token balance, or privileged key is required.
- Any externally-owned account can call `update_rewards` with any valid staker address (all staker addresses are publicly visible on-chain).
- The attack is fully automatable: a bot submits the griefing transaction at the start of every block.
- The only precondition is that at least one active staker with non-zero balance exists, which is always true in a live deployment.

Likelihood is **High**.

---

### Recommendation

Add a caller check at the top of `update_rewards` to enforce that only the Starkware sequencer (or a designated privileged address stored in contract configuration) may invoke it, consistent with the specification:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    self.general_prerequisites();
    // ...
}
```

Alternatively, if the sequencer address is not stored on-chain, use Starknet's `get_execution_info` syscall to verify the transaction originates from the expected sequencer.

---

### Proof of Concept

```cairo
// Attacker contract (Cairo pseudocode)
fn grief_block_rewards(
    staking_contract: ContractAddress,
    any_valid_staker: ContractAddress,
) {
    // Called by attacker at the start of every block, before the sequencer.
    // No special role needed.
    let staking = IStakingRewardsManagerDispatcher {
        contract_address: staking_contract,
    };
    // disable_rewards: true → last_reward_block is updated, zero rewards distributed.
    staking.update_rewards(staker_address: any_valid_staker, disable_rewards: true);
    // Sequencer's subsequent update_rewards call now reverts with REWARDS_ALREADY_UPDATED.
    // All stakers permanently lose their block rewards for this block.
}
```

The attacker only needs to know one active staker address (trivially obtained from on-chain events) and submit this transaction before the sequencer each block. The result is that `last_reward_block` is consumed without any rewards being credited, and the sequencer's legitimate call is blocked for the remainder of that block. [5](#0-4) [6](#0-5)

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),

            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;

            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();

                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
