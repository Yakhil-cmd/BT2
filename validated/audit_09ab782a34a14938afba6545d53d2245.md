### Title
Unbounded Iteration Over Ever-Growing `stakers` Vec in `get_stakers` - (File: src/staking/staking.cairo)

### Summary
The `stakers` storage Vec in `staking.cairo` is append-only: stakers are added when they call `stake()` but are **never removed** when they unstake. The `get_stakers` function in `IStakingConsensus` iterates over the full range of this Vec on every call. As the protocol accumulates stakers over time, the per-call gas cost grows without bound, eventually making the function prohibitively expensive or unreachable within Starknet's transaction gas limits when invoked from another contract.

### Finding Description
`staking.cairo` declares the staker registry as a plain `Vec<ContractAddress>`:

```
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
``` [1](#0-0) 

`get_stakers` (part of the publicly accessible `IStakingConsensus` interface) iterates over **every element** of this Vec unconditionally:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;   // skips inactive stakers but still reads every slot
    }
    ...
}
``` [2](#0-1) 

The interface documents that `get_stakers` is callable by **any address**: [3](#0-2) 

Because unstaked validators remain in the Vec permanently, the iteration length is the **total number of stakers ever registered**, not the number of currently active ones. Each inactive staker still costs a storage read (`staker_address_ptr.read()`) plus the `is_staker_active` check inside the loop body.

### Impact Explanation
**Medium — Unbounded gas consumption.**

Every call to `get_stakers` (whether from an on-chain contract or the consensus layer) must traverse the entire historical staker list. As the protocol matures and staker churn accumulates, the cost grows linearly with the total number of ever-registered stakers. When called from another contract in a Starknet transaction, this will eventually exceed the transaction gas limit, causing the call to revert. Even in off-chain (node-level) execution, sufficiently large lists will exceed node execution resource caps, breaking the consensus layer's ability to query the validator set.

### Likelihood Explanation
The growth is organic and inevitable: every staker who joins and later exits permanently inflates the Vec. No special attacker action is required — normal protocol usage over months/years is sufficient. An adversary wishing to accelerate the issue can register many stakers at minimum stake, each permanently occupying a Vec slot after unstaking.

### Recommendation
Remove the staker's address from the `stakers` Vec when `unstake_action` is executed (or use a swap-and-pop pattern to avoid shifting costs). Alternatively, maintain a separate active-staker count and use a data structure that supports O(1) removal (analogous to OpenZeppelin's `EnumerableSet` recommended in the reference report), so `get_stakers` only iterates over currently active stakers.

### Proof of Concept

1. Alice calls `stake()` → her address is appended to `stakers` (Vec length = 1).
2. Alice calls `unstake_intent()` then `unstake_action()` → her address **remains** in `stakers` (Vec length still = 1, but she is inactive).
3. Repeat steps 1–2 with N distinct addresses (each funded with the minimum stake).
4. Call `get_stakers(epoch_id: current_epoch + 1)` — the loop executes N iterations, reading N storage slots and performing N `is_staker_active` checks, even though zero active stakers may remain.
5. As N grows, the gas cost of `get_stakers` grows linearly. At sufficiently large N, any on-chain caller of `get_stakers` will run out of gas. [4](#0-3) [1](#0-0)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
