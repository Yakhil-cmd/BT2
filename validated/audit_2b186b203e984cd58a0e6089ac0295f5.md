### Title
Unbounded Loop in `calculate_rewards` Enables Permanent Freezing of Delegator Unclaimed Yield — (File: `src/pool/pool.cairo`)

---

### Summary

The `calculate_rewards` function in the Pool contract contains an explicitly acknowledged unbounded loop that iterates over the entire `pool_member_epoch_balance` trace since the last claim. Because the trace grows by one entry per epoch per balance-change operation, a delegator (or their reward address) who makes balance changes every epoch without claiming rewards will grow this trace without bound. Once the trace is large enough, `claim_rewards` will exceed Starknet's per-transaction gas limit and always revert, permanently freezing the delegator's unclaimed yield.

---

### Finding Description

`calculate_rewards` in `src/pool/pool.cairo` iterates over every entry in `pool_member_epoch_balance` from `entry_to_claim_from` to `pool_member_trace_length`:

```
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch { break; }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    ...
    entry_to_claim_from += 1;
}
```

The trace is stored as a `Vec<PoolMemberBalanceCheckpoint>` and grows by one entry per distinct epoch in which a balance change occurs. The `insert` function in `PoolMemberBalanceTrace` appends a new checkpoint only when the key (epoch) differs from the last entry's key; multiple calls within the same epoch overwrite the last entry. Therefore the trace grows at most one entry per epoch.

The trace is grown by:
- `enter_delegation_pool` → calls `set_member_balance` (key = `current_epoch + K`)
- `add_to_delegation_pool` → calls `increase_member_balance` → `set_member_balance`
- `exit_delegation_pool_intent` → calls `set_member_balance`

`add_to_delegation_pool` is callable by either the pool member or their reward address:

```rust
assert!(
    caller_address == pool_member || caller_address == pool_member_info.reward_address,
    "{}",
    Error::CALLER_CANNOT_ADD_TO_POOL,
);
```

The minimum `amount` is 1 unit (enforced only as non-zero). The `transfer_from_delegator` call uses `caller_address`, so the reward address pays from their own balance.

`entry_to_claim_from` is only advanced on a *successful* `claim_rewards` call. If the delegator never claims, `entry_to_claim_from` stays at 0 and every future `claim_rewards` must iterate the entire trace. Each iteration performs:
1. One storage read: `pool_member_trace.at(entry_to_claim_from)`
2. One or more storage reads inside `find_sigma` (reading `cumulative_rewards_trace`)

On Starknet, every storage read consumes a fixed amount of gas. With a trace of sufficient length, the cumulative gas cost exceeds the per-transaction limit, causing the call to always revert.

---

### Impact Explanation

Once `claim_rewards` consistently reverts due to gas exhaustion, the delegator's accumulated unclaimed yield is permanently inaccessible. There is no alternative code path to claim rewards in smaller batches; the entire trace must be processed in a single call. This constitutes **permanent freezing of unclaimed yield** (High severity).

---

### Likelihood Explanation

The attack requires only that a delegator (or their reward address) calls `add_to_delegation_pool` with 1 unit of token once per epoch, without ever calling `claim_rewards`. The cost per epoch is 1 token unit (essentially negligible). Starknet epochs are short (on the order of hours), so over months of operation a legitimate active delegator who defers claiming can accumulate hundreds to thousands of trace entries organically. The code itself acknowledges the loop is unbounded. Likelihood is **medium**.

---

### Recommendation

1. **Paginated claiming**: Introduce a `claim_rewards_partial(max_entries: u64)` entry point that processes at most `max_entries` trace entries per call, advancing `entry_to_claim_from` and accumulating partial rewards into `_unclaimed_rewards_from_v0` on each call.
2. **Cap trace growth**: Enforce a maximum number of unclaimed balance-change entries (e.g., 500) before requiring the delegator to claim rewards, reverting `add_to_delegation_pool` if the cap is exceeded.
3. **Bound `add_to_delegation_pool` frequency**: Require that the delegator has claimed rewards within the last N epochs before allowing a new balance change.

---

### Proof of Concept

```
Epoch 1:  delegator calls enter_delegation_pool(amount=MIN)
          → trace length = 1, entry_to_claim_from = 0

Epoch 2:  delegator calls add_to_delegation_pool(amount=1)
          → trace length = 2, entry_to_claim_from = 0

Epoch 3:  delegator calls add_to_delegation_pool(amount=1)
          → trace length = 3, entry_to_claim_from = 0

...

Epoch N:  delegator calls add_to_delegation_pool(amount=1)
          → trace length = N, entry_to_claim_from = 0

Epoch N+1: delegator calls claim_rewards()

               pool_member_trace.at(i)   [storage read]
               find_sigma(...)           [1+ storage reads]

           → if N is large enough, transaction exceeds gas limit → REVERT

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
