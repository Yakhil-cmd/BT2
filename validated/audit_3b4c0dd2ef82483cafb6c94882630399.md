### Title
Unauthorized Global Contract Namespace Overwrite via Cross-Contract Promise — (`File: runtime/runtime/src/global_contracts.rs`)

### Summary

Any unprivileged contract can overwrite another account's `AccountId`-mode global contract by creating a cross-contract promise targeting the victim account and appending a `DeployGlobalContractAction` with `GlobalContractDeployMode::AccountId`. The runtime stores the code under `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim_id) }` using the receipt *receiver's* account ID, with no check that the receipt predecessor equals the receiver. Every account that previously called `UseGlobalContract(AccountId(victim))` will subsequently execute the attacker's injected code.

### Finding Description

`action_deploy_global_contract` in `runtime/runtime/src/global_contracts.rs` is called with `account_id` equal to the receipt receiver. When `deploy_mode == AccountId`, `initiate_distribution` constructs the global-contract identifier as `GlobalContractIdentifier::AccountId(account_id.clone())` — the receiver's ID — and writes the attacker-supplied bytecode to `TrieKey::GlobalContractCode { identifier: AccountId(receiver) }`. [1](#0-0) 

The host function `promise_batch_action_deploy_global_contract_by_account_id` (and its wasmtime counterpart) appends this action to any promise index, including promises targeting arbitrary third-party accounts. The only guards present are:

1. `is_view()` — rejects view calls.
2. `code_len <= max_contract_size` — size limit.
3. Gas payment. [2](#0-1) 

There is no check that the promise's receiver equals `current_account_id`, and no check in `validate_deploy_global_contract_action` that the deployer is the account whose ID will be used as the namespace key. [3](#0-2) 

The action execution path in `lib.rs` dispatches `DeployGlobalContract` without any predecessor-vs-receiver authorization check: [4](#0-3) 

The nonce mechanism (`increment_nonce` / `check_and_update_nonce`) only prevents *stale* re-delivery of the same distribution receipt; it does not prevent a fresh overwrite by a different sender with a higher nonce. [5](#0-4) 

The `GlobalContractCode` trie key is a shard-global namespace keyed solely by the identifier, not by any per-account prefix: [6](#0-5) 

### Impact Explanation

Every account that has called `UseGlobalContract(AccountId(victim))` stores `AccountContract::GlobalByAccount(victim_id)` in its account record. At execution time the runtime resolves this to `TrieKey::GlobalContractCode { identifier: AccountId(victim_id) }`. After the overwrite, all such accounts execute the attacker's bytecode. The attacker can drain funds, steal keys, or corrupt state of every user of the victim's global contract. Storage cost is deducted from the *victim's* balance, not the attacker's.

### Likelihood Explanation

`global_contract_host_fns` is enabled in the current stable protocol version (confirmed in the runtime config snapshot). Any deployed contract can invoke `promise_batch_action_deploy_global_contract_by_account_id` targeting any account. The attacker only needs to pay gas; the victim pays storage. The attack requires no special privilege — only the ability to deploy and call a contract. [7](#0-6) 

### Recommendation

In `action_deploy_global_contract`, when `deploy_mode == AccountId`, assert that `receipt.predecessor_id() == account_id` (i.e., the action is self-directed). Alternatively, add a validation rule in `validate_deploy_global_contract_action` or `validate_action_receipt` that rejects `DeployGlobalContractAction(AccountId)` when the receipt predecessor differs from the receiver. The `CodeHash` mode is unaffected and requires no change.

### Proof of Concept

1. Attacker deploys contract `attacker.near` containing:
   ```rust
   // inside a #[no_mangle] pub extern "C" fn attack()
   let promise = env::promise_batch_create("victim.near");
   env::promise_batch_action_deploy_global_contract_by_account_id(
       promise, MALICIOUS_WASM.len() as u64, MALICIOUS_WASM.as_ptr() as u64
   );
   ```
2. Attacker calls `attacker.near::attack()` via a signed transaction.
3. The runtime creates a receipt with `receiver_id = victim.near`, `predecessor_id = attacker.near`, action = `DeployGlobalContractAction { code: MALICIOUS_WASM, deploy_mode: AccountId }`.
4. `action_deploy_global_contract` executes with `account_id = victim.near`, writes `MALICIOUS_WASM` to `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }`.
5. All accounts holding `AccountContract::GlobalByAccount("victim.near")` now execute `MALICIOUS_WASM` on their next function call.

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())

fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
}

    fn promise_batch_action_deploy_global_contract_impl(

        self.pay_action_per_byte(ActionCosts::deploy_global_contract_byte, code_len, sir)?;

        self.ext.append_action_deploy_global_contract(receipt_idx, code, mode);
        Ok(())

/// Validates `DeployGlobalContractAction`. Checks that the given contract size doesn't exceed the limit.
fn validate_deploy_global_contract_action(
    limit_config: &LimitConfig,

        return Err(ActionsValidationError::ContractSizeExceeded {
            size: action.code.len() as u64,
            limit: limit_config.max_contract_size,

    }

    Ok(())
