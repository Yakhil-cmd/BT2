### Title
Rerandomization Participant Set Not Bound to `RerandomizedPresignOutput`, Enabling Split-View Secret Key Extraction — (`src/ecdsa/mod.rs`, `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

The `participants` list supplied in `RerandomizationArguments` is hashed into the HKDF-derived scalar `delta` that rerandomizes a presignature, but this binding is never stored in `RerandomizedPresignOutput`. When `sign()` is later called it accepts a fresh `participants` argument with no cross-validation against the rerandomization participants. A malicious coordinator can therefore present different `participants` values to different signers during rerandomization, causing each signer to compute a distinct `delta`, while the signing phase proceeds with a single consistent participant list. The resulting partial shares from two such sessions expose multiplicatively related nonces, enabling full secret key extraction via standard ECDSA nonce-reuse analysis.

---

### Finding Description

**Phase 1 – Rerandomization (offline, coordinator-driven)**

`RerandomizationArguments` bundles `pk`, `tweak`, `msg_hash`, `big_r`, `participants`, and `entropy`. Inside `derive_randomness()` the participant identities are serialised and fed into HKDF:

```rust
for participant in self.participants.participants() {
    concatenation.extend_from_slice(&participant.bytes());
}
let hk = Hkdf::<sha3::Sha3_256>::new(Some(&Self::SALT), &self.entropy);
hk.expand(&concatenation, &mut okm)...
```

The resulting `delta` is then used to transform the presignature shares:

```rust
// OT-based
let rerandomized_k     = presignature.k     * inv_delta;
let rerandomized_sigma = (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

// Robust
let rerandomized_alpha = presignature.alpha * inv_delta;
let rerandomized_beta  = (presignature.beta + presignature.c * args.tweak.value()) * inv_delta;
```

The output type `RerandomizedPresignOutput` stores only the transformed scalars and `big_r`; the `participants` that determined `delta` are **discarded**.

**Phase 2 – Signing (online)**

`sign()` (both variants) accepts a fresh `participants: &[Participant]` slice and uses it to compute Lagrange coefficients:

```rust
// OT-based sign.rs
let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
let k_i    = lambda * presignature.k;
let sigma_i = lambda * presignature.sigma;
```

There is **no check** that this `participants` slice matches the one embedded in the `RerandomizationArguments` that produced the `presignature`. The robust variant enforces `participants.len() == 2*max_malicious + 1` (count only), but not participant identity.

**Root cause (direct analog to the external report)**

The external report's bug: a fee is computed at announcement time from the current position size and stored; the position size can change before execution, making the stored fee stale. Fix: recompute at execution time.

Here: `delta` is computed at rerandomization time from `participants_rerand` and baked into the shares; the signing phase uses `participants_sign` which can differ. Fix: bind `participants_rerand` to `RerandomizedPresignOutput` and validate equality in `sign()`.

---

### Impact Explanation

The security note in `docs/ecdsa/robust_ecdsa/signing.md` explicitly states:

> "If different subsets of size at least 2t+1 sign different (h, ε) values using shares derived from the same presignature, the resulting signatures use multiplicatively related nonces and the secret key can be recovered using standard ECDSA nonce-reuse attacks."

The same attack surface exists when the **participant list** differs between rerandomization calls, because `participants` is part of the HKDF input. Two sessions that use the same `(h, ε, entropy)` but different `participants` in `RerandomizationArguments` produce `delta_1 ≠ delta_2`, hence nonces `k/delta_1` and `k/delta_2` that are multiplicatively related. The coordinator, who receives all partial shares, can recover the secret key.

Impact: **Critical – extraction of the aggregate signing secret**.

---

### Likelihood Explanation

The coordinator is the natural trust boundary in both signing protocols. A malicious coordinator constructs `RerandomizationArguments` for each signer and is the only party that sees all partial signature shares. No honest signer can detect the inconsistency because `RerandomizedPresignOutput` carries no record of which `participants` were used. The attack requires only two signing sessions and is fully within the coordinator's unilateral control.

---

### Recommendation

1. **Store rerandomization participants in `RerandomizedPresignOutput`** (both OT-based and Robust variants):

```rust
pub struct RerandomizedPresignOutput {
    pub big_r: AffinePoint,
    pub k: Scalar,
    pub sigma: Scalar,
    pub participants: ParticipantList,   // ← add
}
```

2. **Validate in `sign()`** that the supplied `participants` slice equals the stored one:

```rust
if ParticipantList::new(participants) != Some(presignature.participants.clone()) {
    return Err(InitializationError::BadParameters(
        "signing participants must match rerandomization participants".to_string(),
    ));
}
```

This mirrors the external report's recommended fix: compute (or validate) the critical value at execution time rather than trusting a value set at an earlier phase.

### Proof of Concept

```
Setup: N = 2t+1 = 3 participants {P1, P2, P3}, presignature (R, k_i, sigma_i).

Session 1 (malicious coordinator):
  - Tells P1: RerandomizationArguments { participants: {P1,P2,P3}, h, ε, ρ }
              → delta_1 = HKDF(ρ, pk, ε, h, R, P1||P2||P3)
  - Tells P2: RerandomizationArguments { participants: {P1,P2,P3}, h, ε, ρ }
              → same delta_1
  - Calls sign() with participants = {P1,P2,P3}
  - Observes s1 = h*(k/delta_1) + r*(sigma/delta_1)

Session 2 (same presignature, different rerandomization participants):
  - Tells P1: RerandomizationArguments { participants: {P1,P2,P4}, h, ε, ρ }
              → delta_2 = HKDF(ρ, pk, ε, h, R, P1||P2||P4) ≠ delta_1
  - Tells P2: same args → delta_2
  - Calls sign() with participants = {P1,P2,P3}   ← same signing set, different rerand set
  - Observes s2 = h*(k/delta_2) + r*(sigma/delta_2)

From s1 and s2:
  s1/s2 = delta_2/delta_1  (known to coordinator)
  s1 = (h*k + r*sigma) / delta_1
  s2 = (h*k + r*sigma) / delta_2
  → h*k + r*sigma = s1*delta_1 = s2*delta_2
  → nonce k = (s1*delta_1 - r*sigma) / h  (sigma recoverable similarly)
  → private key x = sigma / k

    pub fn derive_randomness(&self) -> Result<Scalar, ProtocolError> {
        // create a string containing (pk, msg_hash, big_r, sorted(participants))
        let pk_encoded_point = self.pk.to_encoded_point(true);
        let encoded_pk: &[u8] = pk_encoded_point.as_bytes();
        let encoded_tweak: &[u8] = &<Secp256K1ScalarField as Field>::serialize(&self.tweak.value());
        let encoded_msg_hash: &[u8] = &self.msg_hash;
        let big_r_encoded_point = self.big_r.to_encoded_point(true);
        let encoded_big_r: &[u8] = big_r_encoded_point.as_bytes();

        // concatenate all the bytes
        let mut concatenation = Vec::new();
        // 1 byte counter, used in the unlikely case that the hash result is 0
        concatenation.extend_from_slice(&[0u8, 1]);
        concatenation.extend_from_slice(encoded_pk);
        concatenation.extend_from_slice(encoded_tweak);
        concatenation.extend_from_slice(encoded_msg_hash);
        concatenation.extend_from_slice(encoded_big_r);
        // Append each ParticipantId's
        for participant in self.participants.participants() {
            concatenation.extend_from_slice(&participant.bytes());
        }

        // initiate hkdf with the salt and with some `good' entropy
        let hk = Hkdf::<sha3::Sha3_256>::new(Some(&Self::SALT), &self.entropy);

#[derive(Debug, Clone, Serialize, Deserialize, ZeroizeOnDrop)]
pub struct RerandomizedPresignOutput {
    /// The rerandomized public nonce commitment.
    #[zeroize[skip]]
    pub big_r: AffinePoint,
    /// Our rerandomized share of the nonce value.
    pub k: Scalar,
    /// Our rerandomized share of the sigma value.
    pub sigma: Scalar,
}

impl RerandomizedPresignOutput {
    pub fn rerandomize_presign(
        presignature: &PresignOutput,
        args: &RerandomizationArguments,
    ) -> Result<Self, ProtocolError> {
        if presignature.big_r != args.big_r {
            return Err(ProtocolError::IncompatibleRerandomizationInputs);
        }
        let delta = args.derive_randomness()?;
        if delta.is_zero().into() {
            return Err(ProtocolError::ZeroScalar);
        }

        // cannot be zero due to the previous check
        let inv_delta = delta.invert().unwrap();

        // delta . R
        let rerandomized_big_r = presignature.big_r * delta;

        //  (sigma + tweak * k) * delta^{-1}
        let rerandomized_sigma =
            (presignature.sigma + args.tweak.value() * presignature.k) * inv_delta;

        // k * delta^{-1}
        let rerandomized_k = presignature.k * inv_delta;

        Ok(Self {
            big_r: rerandomized_big_r.into(),
            k: rerandomized_k,
            sigma: rerandomized_sigma,
        })
    }

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    presignature: &RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<Scalar, ProtocolError> {
    // Round 1
    // Linearize ki
    // Spec 1.1
    let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
    let k_i = lambda * presignature.k;

    // Linearize sigmai
    // Spec 1.2
    let sigma_i = lambda * presignature.sigma;

    // Compute si = h * ki + Rx * sigmai
    // Spec 1.3
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)
}

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    max_malicious: impl Into<MaxMalicious>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= 2 * max_malicious + 1
    let robust_ecdsa_threshold = max_malicious
        .into()
        .value()
        .checked_mul(2)
        .and_then(|v| v.checked_add(1))
        .ok_or_else(|| {
            InitializationError::BadParameters(
                "2*threshold+1 must be less than usize::MAX".to_string(),
            )
        })?;
    if robust_ecdsa_threshold > participants.len() {
        return Err(InitializationError::BadParameters(
            "2*max_malicious+1 must be less than or equals to participant count".to_string(),
        ));
    }

    // The next two conditions prevent split-view attacks
    // documented in docs/ecdsa/robust_ecdsa/signing.md
    if participants.len() != robust_ecdsa_threshold {
        return Err(InitializationError::BadParameters(
            "the number of participants during signing must be exactly 2*max_malicious+1 to avoid split view attacks".to_string(),
        ));
    }
    if bool::from(msg_hash.is_zero()) {
        return Err(InitializationError::BadParameters(
            "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
        ));
