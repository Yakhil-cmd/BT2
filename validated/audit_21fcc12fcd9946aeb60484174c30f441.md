### Title
Missing Maximum Commission Slippage Check in Delegation Functions Allows Staker to Frontrun Delegators and Steal Yield - (File: src/pool/pool.cairo)

### Summary

The `enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool` functions in `src/pool/pool.cairo` accept no `max_commission` parameter. A staker who holds an active `CommissionCommitment` can atomically increase their commission to `max_commission` immediately before a delegator's delegation transaction is included, causing the delegator to enter (or switch into) a pool at a far higher commission rate than they observed and intended. The delegator's yield is silently redirected to the staker with no on-chain protection.

### Finding Description

When a delegator calls `enter_delegation_pool` or `switch_delegation_pool`, they observe the staker's current commission off-chain and decide to delegate based on that rate. The functions accept only `reward_address` and `amount` (or `to_staker`, `to_pool`, `amount`), with no parameter allowing the delegator to assert a maximum acceptable commission:

### Impact Explanation

A staker can:
1. Set commission to 1% (attractive to delegators).
2. Call `set_commission_commitment(max_commission: 9900, expiration_epoch: current + 1)` — this is valid since `current_commission (100) <= max_commission (9900)`.
3. Observe a delegator's pending `enter_delegation_pool` or `switch_delegation_pool` transaction.
4. Submit `set_commission(9900)` with higher priority to execute first.
5. The delegator's transaction executes and they are now locked into a 99% commission pool.

During the mandatory exit wait window (at least one week by default), the delegator earns almost no yield. All yield is redirected to the staker as commission. This constitutes **theft of unclaimed yield** — the staker profits directly at the delegator's expense.

### Likelihood Explanation

- The `CommissionCommitment` mechanism is a documented, intended feature that explicitly allows commission increases within the committed range.
- Starknet's sequencer controls transaction ordering; a staker who is also a sequencer, or who cooperates with one, can trivially frontrun delegators.
- Even without sequencer cooperation, a staker can simply call `set_commission` in the same block before the delegator's transaction is included.
- The attack requires no privileged role beyond being a registered staker with an active commitment — any staker can execute it.

### Recommendation

Add a `max_commission` parameter to `enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool`. Before completing the delegation, assert that the staker's current commission does not exceed the caller's stated maximum:

```cairo
fn enter_delegation_pool(
    ref self: ContractState,
    reward_address: ContractAddress,
    amount: Amount,
    max_commission: Commission,  // <-- add this
) {
    // ... existing checks ...
    let current_commission = self.get_commission_from_staking_contract();
    assert!(current_commission <= max_commission, "COMMISSION_EXCEEDS_MAX");
    // ... rest of function ...
}
```

Apply the same pattern to `add_to_delegation_pool` and `switch_delegation_pool`.

### Proof of Concept

1. Staker registers with commission = 100 (1%).
2. Staker calls `set_commission_commitment(max_commission: 9900, expiration_epoch: current_epoch + 1)`.
3. Delegator observes commission = 1% and submits `enter_delegation_pool(reward_address, amount)`.
4. Staker submits `set_commission(9900)` — this succeeds because an active commitment allows it.
5. Staker's transaction is sequenced before the delegator's.
6. Delegator's `enter_delegation_pool` executes; `get_commission_from_staking_contract()` now returns 9900.
7. For the entire exit wait window, the delegator earns ~1% of the yield they expected; the remaining 98% flows to the staker as commission.

The `get_commission_from_staking_contract` call inside `pool_member_info_v1` confirms the commission is read live from the staking contract at claim time, not locked at entry time:

```cairo
fn get_commission_from_staking_contract(self: @ContractState) -> Commission {
    ...
    staking_dispatcher
        .staker_pool_info(staker_address: self.staker_address.read())
        .commission
        .unwrap()
}
