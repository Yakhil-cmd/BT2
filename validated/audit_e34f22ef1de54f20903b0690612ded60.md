### Title
Unpermissioned `update_rewards` Allows Any Caller to Freeze Staker Block Rewards via `disable_rewards: true` - (File: src/staking/staking.cairo)

---

### Summary

The `IStakingRewardsManager::update_rewards` function in `src/staking/staking.cairo` carries **no access-control guard** despite the protocol specification requiring "Only starkware sequencer." Any unprivileged address can call it with `disable_rewards: true`, consuming the single per-block reward slot (`last_reward_block`) without distributing any rewards. Because the slot is global and can only be consumed once per block, the legitimate sequencer is then blocked from distributing rewards for that block. Repeated across every block, this permanently freezes all staker consensus-based block rewards.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` (lines 1449–1507) enforces only two preconditions:

1. `general_prerequisites()` — checks the contract is not paused.
2. `current_block_number > self.last_reward_block.read()` — ensures the slot has not been consumed yet for this block. [1](#0-0) 

There is no check that `get_caller_address()` equals the sequencer or any privileged role. After passing these checks, the function immediately writes the current block number to the global `last_reward_block`: [2](#0-1) 

If `disable_rewards == true`, the function returns immediately — no rewards are calculated or distributed. The slot is consumed, and the legitimate sequencer cannot call `update_rewards` again for the same block (it would revert with `REWARDS_ALREADY_UPDATED`).

The `last_reward_block` storage variable is **global** (not per-staker), so a single call with any valid active staker address and `disable_rewards: true` blocks reward distribution for **all** stakers in that block.

The protocol specification explicitly states the intended access control:

> **access control**: Only starkware sequencer. [3](#0-2) 

But the implementation contains no such enforcement.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` once per block, every block, prevents every staker from ever accumulating consensus block rewards. Since `last_reward_block` is a single global slot, one call per block is sufficient to deny rewards to the entire protocol. The attacker needs no capital, no special role, and gains nothing — pure griefing. If sustained, stakers and pool members receive zero block rewards indefinitely, constituting permanent freezing of unclaimed yield.

---

### Likelihood Explanation

**High.** The function is publicly callable with no economic barrier. The attacker only needs to submit one transaction per block (or even just front-run the sequencer's legitimate call). The attack is cheap, deterministic, and requires no privileged access.

---

### Recommendation

Add a sequencer-only access control guard at the top of `update_rewards`, consistent with the specification. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer(); // <-- add this
    self.general_prerequisites();
    ...
}
```

Alternatively, restrict the `disable_rewards` parameter so that only a privileged caller can pass `true`, while allowing any caller to trigger reward distribution with `false`.

---

### Proof of Concept

1. Consensus rewards are active (post `consensus_rewards_first_epoch`).
2. A valid active staker `S` exists.
3. Attacker `Bob` (any address) calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
   at block `N`.
4. `last_reward_block` is written to `N`. No rewards are distributed.
5. The legitimate sequencer attempts:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: false)
   ```
   at block `N` — this reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker `S` (and all other stakers, since the slot is global) receives zero block rewards for block `N`.
7. Bob repeats this every block. All stakers are permanently denied consensus block rewards. [4](#0-3) [5](#0-4)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
