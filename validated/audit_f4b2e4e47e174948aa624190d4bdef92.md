### Title
Missing Validation of ElGamal Public Key (`app_pk`) Allows Disclosure of Confidential Derived Key - (File: `src/confidential_key_derivation/protocol.rs`)

---

### Summary

The `ckd()` function in `src/confidential_key_derivation/protocol.rs` accepts an `app_pk: PublicKey` (a BLS12-381 G1 point) from an external caller without validating that it is a non-identity, valid curve point. If a malicious caller submits the identity element as `app_pk`, the ElGamal blinding term collapses to zero, causing the coordinator to return the raw BLS signature `msk · H(pk, app_id)` in plaintext on-chain, fully disclosing the confidential derived key.

---

### Finding Description

The `ckd()` entry point performs only participant-list sanity checks before launching the protocol: [1](#0-0) 

No check is made that `app_pk` is a valid, non-identity G1 point. The value flows directly into `compute_signature_share()`: [2](#0-1) 

At line 174, each node computes `C_i = S_i + y_i * app_pk`. If `app_pk` is the identity element `O ∈ G1`, then `y_i * O = O`, so `C_i = S_i = x_i · H(pk, app_id)`. After Lagrange aggregation by the coordinator:

```
C = Σ λ_i · C_i = msk · H(pk, app_id)
```

The coordinator returns `(Y, C)` as `CKDOutput`: [3](#0-2) 

`C` is now the raw BLS signature `msk · H(pk, app_id)` with no blinding, transmitted on-chain in the clear.

The `PublicKey` type is simply `ElementG1 = blstrs::G1Projective`: [4](#0-3) 

`blstrs::G1Projective` accepts the identity element as a valid value; there is no automatic rejection at the type level.

By contrast, the `verify_signature` function in the same module correctly validates that neither the signature nor the verifying key is the identity: [5](#0-4) 

No equivalent guard exists at the `ckd()` entry point for `app_pk`.

---

### Impact Explanation

**Critical — Extraction/disclosure of confidential derived key.**

The entire security guarantee of CKD is that the BLS signature `msk · H(pk, app_id)` is never revealed to any party other than the TEE app holding the corresponding secret scalar `a`. When `app_pk = O`, the blinding `a · Y` vanishes from `C`, and the signature is broadcast on-chain in plaintext. Any observer of the blockchain response obtains `C = msk · H(pk, app_id)` directly and can compute `s = HKDF(C)` — the confidential derived key — without possessing the TEE app's secret.

This matches the allowed Critical impact: *"Extraction, reconstruction, or disclosure of … confidential derived secrets."*

---

### Likelihood Explanation

The `app_pk` parameter is supplied by an external caller (the TEE app or the MPC contract relaying the on-chain request). A malicious caller — or any party who can submit a CKD request with a crafted `A` — can trivially set `A = G1::identity()`. The identity element is a canonical, easily constructable value. No cryptographic capability is required; the attack is a single malformed input.

---

### Recommendation

Add a validity check for `app_pk` at the start of `ckd()` (or at the top of `compute_signature_share()`), rejecting the identity element and any point not on the curve or not in the prime-order subgroup:

```rust
// In ckd() after participant checks, before launching the protocol:
use elliptic_curve::group::prime::PrimeCurveAffine;
let app_pk_affine: blstrs::G1Affine = app_pk.into();
if (!app_pk_affine.is_on_curve()
    | !app_pk_affine.is_torsion_free()
    | app_pk_affine.is_identity())
    .into()
{
    return Err(InitializationError::InvalidInput("app_pk is not a valid G1 point".into()));
}
```

This mirrors the validation already present in `verify_signature` for the signature and verifying key elements.

pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    let comms = Comms::new();
    let chan = comms.shared_channel();

    let fut = run_ckd_protocol(
        chan,
        coordinator,
        me,
        participants,
        key_pair,
        app_id.into(),
        app_pk,
        rng,
    );
    Ok(make_protocol(comms, fut))

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
