### Title
Gateway L2 Gas Price Admission Uses Previous-Block Price While Blockifier Execution Enforces Current-Block Price — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful admission check (`validate_resource_bounds`) validates a transaction's `l2_gas.max_price_per_unit` against the **previous block's** L2 gas price, while the blockifier's `check_fee_bounds` — called during actual batcher execution — enforces the **current block's** L2 gas price. Because the EIP-1559 fee market can raise the L2 gas price by up to 12.5% per block, a transaction that clears the gateway threshold can fail pre-validation at execution time, meaning the gateway admits transactions that the sequencer will silently drop.

### Finding Description

**Gateway admission path** — `validate_resource_bounds` reads the previous block's L2 gas price and computes a threshold:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:230-240
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
```

`validate_tx_l2_gas_price_within_threshold` then accepts the transaction if:

```
tx.l2_gas.max_price_per_unit  >=  (min_gas_price_percentage / 100)  *  previous_block_l2_gas_price
```

With the production default of `min_gas_price_percentage = 100`, the threshold equals the previous block's price exactly.

**Blockifier execution path** — `check_fee_bounds`, called inside `perform_pre_validation_stage` during batcher execution, enforces the **current block's** gas price:

```rust
// crates/blockifier/src/transaction/account_transaction.rs:441-448
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow {
            resource: *resource,
            max_gas_price: resource_bounds.max_price_per_unit,
            actual_gas_price: (*actual_gas_price).into(),
        },
    );
}
```

`actual_gas_price` here comes from `block_context.block_info.gas_prices`, which is the **current block's** L2 gas price set by the consensus orchestrator via EIP-1559 (`calculate_next_base_gas_price`).

The two price sources are structurally independent:

| Stage | Price source |
|---|---|
| Gateway `validate_resource_bounds` | `get_block_info().gas_prices.strk_gas_prices.l2_gas_price` (previous block) |
| Blockifier `check_fee_bounds` | `block_context.block_info.gas_prices` (current block, EIP-1559 adjusted) |

Note also that `run_validate_entry_point` (the gateway's blockifier call) only runs the `__validate__` entry point — it does **not** call `perform_pre_validation_stage`, so `check_fee_bounds` is never exercised at gateway time.

### Impact Explanation

When consecutive blocks are full, EIP-1559 raises the L2 gas price by up to 1/8 per block. A transaction submitted with `max_price_per_unit = P_prev` passes the gateway check (`P_prev >= 100% × P_prev`) but fails the batcher's `check_fee_bounds` (`P_prev < P_prev × 1.125`). The transaction is admitted to the mempool, occupies a slot, and is then silently dropped at pre-validation without being included in any block and without the user paying fees. The user must resubmit.

This matches the **High** impact category: *Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.*

### Likelihood Explanation

The condition is triggered whenever the current block's L2 gas price exceeds the previous block's price — a routine outcome under sustained load with EIP-1559. No special privileges or adversarial setup are required; any ordinary user submitting a transaction priced at exactly the previous block's threshold will hit this.

### Recommendation

Replace the previous-block price reference in `validate_resource_bounds` with the **next block's** projected L2 gas price (i.e., apply `calculate_next_base_gas_price` to the previous block's price and usage before computing the threshold), or lower `min_gas_price_percentage` to leave headroom for the maximum EIP-1559 increase (e.g., set it to `≤ 88` to tolerate a full 12.5% jump). The TODO comment at line 229 (`// TODO(Arni): getnext_l2_gas_price from the block header`) already acknowledges this gap.
