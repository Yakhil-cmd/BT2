### Title
`old_threshold` Not Validated Against Minimum in `assert_reshare_keys_invariants`, Allowing Reshare With Insufficient Old Participants - (File: src/dkg.rs)

### Summary

`assert_key_invariants` enforces a minimum threshold of 2 for keygen and for the *new* threshold in reshare. However, `assert_reshare_keys_invariants` never validates that `old_threshold >= 2`. A caller can supply `old_threshold = 1`, causing the intersection-size guard to pass with only one old participant. The reshare then proceeds with a single-point Lagrange coefficient of 1, so the "reconstructed" secret equals one participant's raw share rather than the actual group secret, corrupting every new share derived from it.

### Finding Description

`assert_key_invariants` (called for the new participant set) enforces:

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

`assert_reshare_keys_invariants` calls `assert_key_invariants` only for the **new** threshold/participants, then uses the caller-supplied `old_threshold` directly as a lower bound for the intersection size:

```rust
let threshold = usize::from(threshold.into());
let old_threshold = usize::from(old_threshold.into());

let participants = assert_key_invariants(participants, me, threshold)?;  // new threshold validated

let old_participants =
    ParticipantList::new(old_participants).ok_or(InitializationError::DuplicateParticipants)?;

if old_participants.intersection(&participants).len() < old_threshold {
    return Err(InitializationError::NotEnoughParticipantsForNewThreshold { ... });
}
``` [2](#0-1) 

`old_threshold` is never checked to be `>= 2`. Supplying `old_threshold = 1` makes the intersection guard pass with a single old participant.

In `do_reshare`, the intersection is used to compute Lagrange coefficients:

```rust
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(<C::Group as Group>::Field::zero);
``` [3](#0-2) 

When `intersection` contains exactly one participant, `lagrange(me)` evaluates to the scalar `1` (single-point Lagrange basis polynomial evaluated at zero). Therefore `secret = 1 · x_i = x_i` — the participant's raw share — rather than the correctly reconstructed group secret `Σ λ_j · x_j`.

This incorrect `secret` becomes the constant term of the new polynomial in `do_keyshare`:

```rust
let secret_coefficients = Polynomial::<C>::generate_polynomial(Some(secret), degree, rng)?;
``` [4](#0-3) 

All new shares are derived from this corrupted constant term. The resulting aggregate public key will not match `old_public_key`, triggering the consistency check:

```rust
if old_vk != verifying_key {
    return Err(ProtocolError::AssertionFailed(
        "new public key does not match old public key".to_string(),
    ));
}
``` [5](#0-4) 

The reshare aborts for every participant who ran with the manipulated parameters, permanently denying the reshare operation.

### Impact Explanation

A malicious caller (e.g., a compromised coordinator or a participant who controls the reshare invocation parameters) passes `old_threshold = 1` together with a reduced `old_participants` list (e.g., `[me]`). The pre-check in `assert_reshare_keys_invariants` passes. Inside `do_keyshare`, the corrupted constant term propagates into every participant's new signing share. The aggregate verifying key diverges from the original, the protocol aborts, and honest parties are left with no valid new shares — a permanent denial of the reshare operation.

This matches the allowed impact: **High — Corruption of reshare outputs so honest parties accept inconsistent public keys or unusable cryptographic outputs / Permanent denial of reshare for honest parties.**

### Likelihood Explanation

**Low.** The attacker must control the parameters passed to `reshare()` — either by acting as a coordinator who distributes reshare parameters to all participants, or by being a participant who independently calls `reshare()` with manipulated inputs. In a well-administered deployment the correct `old_threshold` is known to all parties, but the library itself provides no enforcement, leaving the door open to misconfiguration or a malicious coordinator.

### Recommendation

Add a minimum-threshold validation for `old_threshold` inside `assert_reshare_keys_invariants`, mirroring the check already applied to the new threshold:

```rust
pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    ...
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],
) -> Result<(ParticipantList, ParticipantList), InitializationError> {
    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

+   // Mirror the same lower-bound check applied to the new threshold
+   if old_threshold < 2 {
+       return Err(InitializationError::ThresholdTooSmall { threshold: old_threshold, min: 2 });
+   }
+   if old_participants.len() < 2 {
+       return Err(InitializationError::NotEnoughParticipants { participants: old_participants.len() });
+   }

    let participants = assert_key_invariants(participants, me, threshold)?;
    ...
}
``` [6](#0-5) 

### Proof of Concept

1. Run a 3-of-5 keygen to obtain `(old_public_key, shares[0..5])`.
2. Call `reshare()` on behalf of participant 0 with:
   - `old_threshold = 1` (bypasses the `>= 2` check that was never applied)
   - `old_participants = [participant_0]` (intersection with new set = 1)
   - `old_signing_key = Some(shares[0])`
3. `assert_reshare_keys_invariants` passes: `intersection.len() = 1 >= 1 = old_threshold`.
4. Inside `do_reshare`, `intersection.lagrange(participant_0) = 1`, so `secret = shares[0].scalar()`.
5. `do_keyshare` generates a polynomial with constant term `shares[0]` (not the group secret).
6. The aggregate verifying key ≠ `old_public_key`; the protocol returns `AssertionFailed("new public key does not match old public key")`.
7. All participants who accepted these parameters are left without valid new shares — reshare is permanently denied.

```rust
pub fn assert_reshare_keys_invariants<C: Ciphersuite>(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_threshold: impl Into<ReconstructionLowerBound>,
    old_participants: &[Participant],
) -> Result<(ParticipantList, ParticipantList), InitializationError> {
    let threshold = usize::from(threshold.into());
    let old_threshold = usize::from(old_threshold.into());

    let participants = assert_key_invariants(participants, me, threshold)?;

    let old_participants =
        ParticipantList::new(old_participants).ok_or(InitializationError::DuplicateParticipants)?;

    // Step 1.1
    if old_participants.intersection(&participants).len() < old_threshold {
        return Err(InitializationError::NotEnoughParticipantsForNewThreshold {
            threshold: old_threshold,
            participants: old_participants.intersection(&participants).len(),
        });
    }
    // Step 1.1
    // if me is not in the old participant set then ensure that old_signing_key is None
    if old_participants.contains(me) && old_signing_key.is_none() {
        return Err(InitializationError::BadParameters(format!(
            "party {me:?} is present in the old participant list but provided no share"
        )));
    }
    Ok((participants, old_participants))
}
