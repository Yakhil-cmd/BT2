### Title
Unpermissioned `update_rewards` with `disable_rewards: true` Permanently Freezes All Consensus Rewards - (File: `src/staking/staking.cairo`)

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is callable by any non-zero address and accepts a caller-controlled `disable_rewards: bool` parameter. When called with `disable_rewards: true`, it consumes the global `last_reward_block` slot for the current block and returns early without distributing any rewards. Because `last_reward_block` is a single global variable, any subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`. An attacker can call this once per block, permanently freezing all consensus-phase rewards for every staker in the protocol.

### Finding Description

Two separate code paths exist for distributing consensus rewards:

**Path A — `update_rewards_from_attestation_contract`** (lines 1394–1423):
```cairo
fn update_rewards_from_attestation_contract(
    ref self: ContractState, staker_address: ContractAddress,
) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
    self.assert_caller_is_attestation_contract();   // <-- access-controlled
    ...
}
```

**Path B — `update_rewards`** (lines 1449–1507):
```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                   // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);  // <-- slot consumed HERE

    if disable_rewards || self.is_pre_consensus() {
        return;                                          // <-- early exit, no rewards
    }
    ...
}
```

The check inconsistency is direct:

| | Access control | `disable_rewards` bypass |
|---|---|---|
| `update_rewards_from_attestation_contract` | `assert_caller_is_attestation_contract()` | N/A |
| `update_rewards` | **None** (only `assert_caller_is_not_zero`) | **Caller-controlled** |

`general_prerequisites` only enforces: [1](#0-0) 

The global `last_reward_block` is written **before** the `disable_rewards` early-return branch: [2](#0-1) 

This means the slot is consumed even when no rewards are distributed.

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block. Each call:
1. Passes all validation (staker exists, is active, has non-zero balance).
2. Writes `last_reward_block = current_block_number`.
3. Returns early — zero rewards distributed.

Any legitimate call to `update_rewards` in the same block then fails:
```
assert!(current_block_number > self.last_reward_block.read(), "REWARDS_ALREADY_UPDATED");
```

Repeated every block, this permanently freezes all consensus-phase STRK and BTC rewards for every staker and every delegation pool in the protocol. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The function is fully public — no role, no whitelist, no fee.
- The attacker only needs a valid staker address, which is trivially readable from on-chain events (`NewStaker`).
- Cost is one transaction per block (gas only). On Starknet L2, this is negligible.
- No profit motive is required; the attack is pure griefing.

### Recommendation

Add an access-control guard to `update_rewards` so that only an authorized caller (e.g., a designated rewards-updater role, or the attestation contract) can invoke it. Alternatively, remove the `disable_rewards` parameter from the public interface and handle that logic internally or through a privileged role.

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.roles.only_rewards_updater(); // or equivalent role check
    ...
}
```

### Proof of Concept

**Setup:** Consensus rewards are active (`consensus_rewards_first_epoch` is set and current epoch ≥ that value). At least one valid staker `S` exists with non-zero STRK balance.

**Attack sequence (repeated every block):**

1. Attacker (any EOA) calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
2. Inside `update_rewards`:
   - `current_block_number > last_reward_block` → passes (new block).
   - Staker `S` is active with non-zero balance → passes.
   - `last_reward_block` is written to `current_block_number`.
   - `disable_rewards == true` → early return, **no rewards distributed**.
3. Any legitimate call to `update_rewards` for the same block reverts with `REWARDS_ALREADY_UPDATED`.
4. Attacker repeats at the next block.

**Result:** All stakers and delegation pool members accumulate zero consensus rewards indefinitely. Their `unclaimed_rewards_own` fields are never incremented. The yield is permanently frozen. [3](#0-2) [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
