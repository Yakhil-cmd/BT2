### Title
Staker and Pool Member Can Set Reward Address to Zero, Permanently Burning Unclaimed Yield - (`src/staking/staking.cairo`, `src/pool/pool.cairo`)

### Summary
Both `change_reward_address` functions — for stakers in `staking.cairo` and for pool members in `pool.cairo` — accept `ContractAddress::zero()` as a valid new reward address. When rewards are subsequently claimed or auto-disbursed, the STRK tokens are transferred to address zero and are permanently unrecoverable.

### Finding Description
`change_reward_address` in `staking.cairo` validates only that the new address is not a registered token address: [1](#0-0) 

The only guard is `!self.does_token_exist(token_address: reward_address)`. There is no check that `reward_address != Zero::zero()`. The same pattern exists in `pool.cairo`: [2](#0-1) 

The only guard there is `self.token_dispatcher.contract_address.read() != reward_address` — again, no zero-address check.

When rewards are disbursed, `send_rewards_to_staker` unconditionally transfers to `staker_info.reward_address`: [3](#0-2) 

And `claim_rewards` in `pool.cairo` does the same: [4](#0-3) 

Both paths transfer to whatever address is stored, including zero.

### Impact Explanation
Any staker or pool member who sets their reward address to zero will have all future (and currently accumulated) unclaimed STRK yield transferred to address `0x0` on the next `claim_rewards` or `unstake_action` call. On Starknet, address zero is not a recoverable account; the tokens are permanently burned. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The call is permissionless — any staker or pool member can invoke `change_reward_address(Zero::zero())` directly. A user could do this accidentally (e.g., passing an uninitialized variable), or a malicious front-end could craft such a call. The `unstake_action` path is particularly dangerous because it auto-calls `send_rewards_to_staker` before returning the principal, meaning a staker who set their reward address to zero and then unstakes loses all accumulated rewards with no warning. [5](#0-4) 

### Recommendation
Add a zero-address guard in both `change_reward_address` implementations:

```cairo
// In staking.cairo and pool.cairo change_reward_address:
assert!(reward_address.is_non_zero(), "Reward address cannot be zero");
```

This mirrors the fix recommended in the external report (`require(newAdmin != 0, ...)`).

### Proof of Concept

1. Staker calls `change_reward_address(reward_address: Zero::zero())` on the staking contract.
   - The only check `!self.does_token_exist(token_address: Zero::zero())` passes (zero is not a registered token).
   - `staker_info.reward_address` is written as `0x0`.
2. Staker accumulates rewards over subsequent epochs via attestation.
3. Anyone calls `claim_rewards(staker_address)` (or the staker calls `unstake_intent` + `unstake_action`).
4. `send_rewards_to_staker` executes `token_dispatcher.checked_transfer(recipient: 0x0, amount: rewards)`.
5. All accumulated STRK yield is transferred to address zero and is permanently unrecoverable.

The same sequence applies to a pool member calling `change_reward_address` on the pool contract, with `claim_rewards` in `pool.cairo` transferring to `reward_address = 0x0`. [6](#0-5) [7](#0-6)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
