### Title
Pause Mechanism Blocks `unstake_action` and `exit_delegation_pool_action`, Temporarily Freezing User Funds - (File: `src/staking/staking.cairo`)

---

### Summary
The `SECURITY_AGENT` role can pause the Starknet Staking contract, which blocks `unstake_action` (and the pool's `exit_delegation_pool_action` path through `remove_from_delegation_pool_action`) via the `general_prerequisites()` guard. Stakers and delegators who have already waited the full `exit_wait_window` after signaling intent are unable to retrieve their funds until the `SECURITY_ADMIN` unpauses the contract.

---

### Finding Description

The staking contract stores a boolean `is_paused` flag in storage and exposes `pause()` / `unpause()` through `IStakingPause`. The `SECURITY_AGENT` role can call `pause()` at any time.

Every state-changing user function begins with `self.general_prerequisites()`, which asserts the contract is not paused. This includes `unstake_action`:

```cairo
fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    // Prerequisites and asserts.
    self.general_prerequisites();   // <-- reverts with CONTRACT_IS_PAUSED when paused
    ...
```

The intent-action withdrawal pattern is two-step:
1. `unstake_intent()` — records `unstake_time = now + exit_wait_window` and removes the staker from total stake.
2. `unstake_action()` — callable only after `unstake_time` has elapsed; transfers STRK back to the staker.

Both steps are gated by `general_prerequisites()`. A staker who has already completed step 1 and waited the full `exit_wait_window` (default: 1 week, max: 12 weeks) is still blocked from executing step 2 while the contract is paused.

The same applies to pool delegators: `exit_delegation_pool_action` in `pool.cairo` calls `staking_pool_dispatcher.remove_from_delegation_pool_action(...)`, which routes into the staking contract and is similarly blocked.

---

### Impact Explanation

**Temporary freezing of funds (High).**

Stakers and delegators who have already signaled exit intent and waited the mandatory `exit_wait_window` cannot execute the action step to recover their principal. Their STRK (or BTC-equivalent token) remains locked in the staking contract for the entire duration of the pause. The pause has no time limit — only `SECURITY_ADMIN` can lift it. This constitutes a temporary freeze of user funds.

---

### Likelihood Explanation

**Low.**

Triggering this requires the `SECURITY_AGENT` to pause the contract. This is a privileged role and pausing is expected to be rare (emergencies, upgrades). However, the EIC upgrade flow explicitly requires the contract to be paused, meaning every protocol upgrade creates a window during which users with a matured exit intent are frozen. The pause duration during upgrades is not bounded by the protocol.

---

### Recommendation

Remove the `general_prerequisites()` call (or at minimum the `is_paused` check) from `unstake_action` and from `remove_from_delegation_pool_action` in the staking contract. Users who have already waited the full `exit_wait_window` should always be able to retrieve their principal regardless of pause state. New deposits (`stake`, `increase_stake`, `enter_delegation_pool`) and reward-accruing operations can remain paused.

---

### Proof of Concept

1. Staker calls `unstake_intent()`. The contract records `unstake_time = now + exit_wait_window` and removes the staker's balance from `tokens_total_stake_trace`.
2. `SECURITY_AGENT` calls `pause()`, setting `is_paused = true`.
3. After `exit_wait_window` elapses (e.g., 1 week), the staker calls `unstake_action(staker_address)`.
4. `general_prerequisites()` reads `is_paused == true` and reverts with `CONTRACT_IS_PAUSED`.
5. The staker's STRK remains locked in the staking contract. The staker has no recourse until `SECURITY_ADMIN` calls `unpause()`.

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
