### Title
Missing Lower-Bound Threshold Validation in FROST Signing Allows Corrupted Signing Output — (File: `src/frost/mod.rs`)

---

### Summary

`assert_sign_inputs` in `src/frost/mod.rs` validates that the threshold is not too large, but omits the lower-bound check (`threshold >= 2`) that is explicitly enforced in `assert_key_invariants` for DKG. This allows a caller to pass `threshold = 1` to all FROST signing entry points (`sign_v1`, `sign_v2` in EdDSA and `sign` in RedJubjub), causing the FROST library to construct a `KeyPackage` with `min_signers = 1`. A malicious coordinator can exploit this to present a signing package containing only their own nonce commitment; honest participants will accept and sign it (because `min_signers = 1` passes the FROST library's internal check), producing a cryptographically invalid signature and permanently denying honest parties a usable signing output.

---

### Finding Description

**Root cause — missing lower-bound check in `assert_sign_inputs`:**

`assert_key_invariants` (DKG initialization) enforces:

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

`assert_sign_inputs` (FROST signing initialization) only enforces the upper bound:

```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [2](#0-1) 

There is **no** `threshold < 2` guard. The same omission exists in the generic `presign` function: [3](#0-2) 

**Propagation into `KeyPackage`:**

Every signing participant calls `construct_key_package`, which passes the caller-supplied threshold directly to the FROST library as `min_signers`:

```rust
Ok(KeyPackage::new(
    identifier,
    signing_share,
    verifying_share,
    *verifying_key,
    u16::try_from(threshold.value())...,
))
``` [4](#0-3) 

With `threshold = 1`, `min_signers = 1`. The FROST library's `round2::sign` then accepts any signing package that contains at least one commitment — including a package crafted by the coordinator to contain only their own nonce commitment.

**All three FROST signing entry points are affected:**

- `sign_v1` / `sign_v2` in `src/frost/eddsa/sign.rs` — both call `assert_sign_inputs` then forward `threshold` to `construct_key_package`. [5](#0-4) 

- `sign` in `src/frost/redjubjub/sign.rs` — same pattern. [6](#0-5) 

---

### Impact Explanation

In FROST, the nonce commitment `R` included in the signing package determines the challenge `c = H(R, Y, m)`. If the coordinator crafts a signing package containing only their own commitment `R_coord`, honest participants compute signature shares `z_i = r_i + s_i · c · λ_i` using a challenge derived from `R_coord` alone. The aggregated signature `(R_coord, z)` will not verify against the master public key because the nonce used in the challenge does not match the sum of all participants' nonces. Every honest participant's signing session terminates with an invalid, unusable signature — a permanent denial of signing for valid protocol inputs.

This matches the allowed impact: **High — Corruption of sign outputs so honest parties accept inconsistent or unusable cryptographic outputs.**

---

### Likelihood Explanation

The threshold parameter is fully caller-controlled at the API boundary. Any participant who acts as coordinator can pass `threshold = 1` to their own `sign_v1` call. Because `assert_sign_inputs` does not reject it, the protocol proceeds. The coordinator then has the ability to send a signing package with only one commitment. No cryptographic capability or key material beyond normal participation is required.

---

### Recommendation

Add the same lower-bound guard that exists in `assert_key_invariants` to `assert_sign_inputs` (and to the generic `presign` function):

```rust
// In assert_sign_inputs (src/frost/mod.rs)
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Apply the same guard inside `presign` in `src/frost/mod.rs` after the upper-bound check.

---

### Proof of Concept

1. Run DKG with 3 participants and `threshold = 2` (valid, enforced by `assert_key_invariants`).
2. The malicious coordinator calls `sign_v1` with `threshold = 1` — `assert_sign_inputs` accepts it.
3. The coordinator's `KeyPackage` is built with `min_signers = 1`.
4. The coordinator crafts a `SigningPackage` containing only their own `SigningCommitments`; the FROST library accepts it because `min_signers = 1`.
5. The coordinator broadcasts this package; honest participants call `round2::sign` — it succeeds because the package satisfies `min_signers = 1`.
6. The coordinator aggregates all shares into a signature `(R_coord, z)`.
7. Verification of `(R_coord, z)` against the master public key fails: the challenge was computed from `R_coord` alone, not from the sum of all participants' nonces.
8. Honest parties receive an unusable, invalid signature; signing is permanently denied for this session. [7](#0-6) [8](#0-7) [9](#0-8)

/// Verifies that the sign inputs are valid
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}

/// A function that takes a signing share and a keygenOutput
/// and construct a public key package used for frost signing
fn construct_key_package(
    threshold: ReconstructionLowerBound,
    me: Participant,
    signing_share: SigningShare,
    verifying_key: &VerifyingKey,
) -> Result<KeyPackage, ProtocolError> {
    let identifier = me.to_identifier()?;
    let verifying_share = signing_share.into();

    Ok(KeyPackage::new(
        identifier,
        signing_share,
        verifying_share,
        *verifying_key,
        u16::try_from(threshold.value()).map_err(|_| {
            ProtocolError::Other("threshold cannot be converted to u16".to_string())
        })?,
    ))
}
