### Title
WalletConnect Multi-NFT Offer Confirmation Displays Understated "Total Amount with Royalties" Due to Integer Division Error - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts` incorrectly divides the fungible amount by the number of NFTs before computing each royalty, causing the "Total Amount with Royalties" shown in the WalletConnect confirmation dialog to be understated by a factor of N (number of NFTs) when N > 1. A malicious WalletConnect dApp can craft a multi-NFT offer where the displayed total is materially lower than what the user will actually spend, inducing approval of a transaction for more XCH/CAT than the user believes they are authorizing.

### Finding Description
In `formatAmountWithRoyalties`:

```typescript
// packages/gui/src/electron/commands/parseCommandDisplay.ts
const splitAmount = amount / BigInt(royaltyPercentages.length);   // line 363
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

The function first divides `amount` by the count of NFTs (`royaltyPercentages.length`), then multiplies each royalty percentage against that reduced `splitAmount`. The effective formula is:

```
displayed_royalty = (amount / N) × Σ(royaltyPercentage_i) / 10_000
```

The correct formula — matching how the Chia protocol charges royalties on-chain, where each NFT creator receives a royalty on the **full** fungible amount — is:

```
correct_royalty = Σ(amount × royaltyPercentage_i / 10_000)
               = amount × Σ(royaltyPercentage_i) / 10_000
```

For N = 1 the two are identical. For N > 1 the displayed royalty is understated by exactly a factor of N (ignoring BigInt truncation rounding).

**Concrete proof from the existing test** (lines 251–335 of `parseCommandDisplay.test.ts`):

| Parameter | Value |
|---|---|
| NFTs | 2 (royalty_percentage 500 and 10) |
| amount | 100,000,000 mojos (0.0001 XCH) |
| Displayed `amountWithRoyalties` | 0.00010255 XCH (royalty = 2,550,000 mojos) |
| Correct total | 0.0001051 XCH (royalty = 5,100,000 mojos) |

The display understates the royalty by 50% (factor of N = 2). With 3 NFTs the understatement is 67%, and so on.

The computed `amountWithRoyalties` string is surfaced directly to the user in the WalletConnect confirmation dialog (`Confirm.tsx` lines 109–114, 130–135, 148–153) under the label **"Total Amount with Royalties"**, which is the sole summary figure the user sees before clicking Confirm. ### Impact Explanation
A user who accepts a WalletConnect `chia_wallet.take_offer` or `chia_wallet.create_offer_for_ids` command containing multiple NFTs with royalties will see a "Total Amount with Royalties" that is N times lower than the amount actually deducted from their wallet. Because the confirmation dialog presents this figure as the authoritative total cost, the user approves a spend they would not have approved had the correct amount been shown. This constitutes a direct, quantifiable asset loss caused by a corrupted WalletConnect state display — matching the High impact criterion: *"Corruption … of WalletConnect state that causes a user to approve … the wrong … amount."*

### Likelihood Explanation
Any WalletConnect-connected dApp (an unprivileged external actor that has received one-time user approval for the connection) can trigger this path by issuing a `take_offer` or `create_offer_for_ids` command referencing an offer that bundles two or more NFTs with non-zero royalties. No special permissions beyond the initial WalletConnect pairing are required. The `take_offer` command always requires user confirmation, so the user will see — and be misled by — the understated figure on every such request.

### Recommendation
Remove the erroneous division by `royaltyPercentages.length`. Each NFT's royalty should be computed against the full `amount`:

```typescript
// Correct: each NFT royalty is a fraction of the full amount
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
```

Update the corresponding test expectations in `parseCommandDisplay.test.ts` to reflect the corrected values (e.g., for the two-NFT case: royalty = 5,100,000 mojos → total = 0.0001051 XCH).

### Proof of Concept
1. Attacker operates a dApp and obtains a WalletConnect pairing with the victim's wallet.
2. Attacker constructs an offer containing 3 NFTs, each with `royalty_percentage = 1000` (10%), requesting 1 XCH.
3. Attacker sends `chia_wallet.take_offer` via WalletConnect.
4. `parseCommandDisplay` calls `formatAmountWithRoyalties(line, 1_000_000_000_000n, [1000, 1000, 1000])`.
   - `splitAmount = 1_000_000_000_000n / 3n = 333_333_333_333n`
   - `royaltyAmount = 3 × (333_333_333_333n × 1000n / 10_000n) = 3 × 33_333_333_333n = 99_999_999_999n`
   - Displayed total ≈ **1.1 XCH**
5. Correct total: `3 × (1_000_000_000_000n × 1000n / 10_000n) = 300_000_000_000n` → total = **1.3 XCH**
6. User sees "Total Amount with Royalties: 1.1 XCH", approves, and 1.3 XCH is deducted — a 0.2 XCH shortfall versus what was displayed. [4](#0-3) [5](#0-4) [6](#0-5)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}
