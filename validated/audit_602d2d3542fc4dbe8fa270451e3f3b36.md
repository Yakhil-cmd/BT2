### Title
Missing Caller Access Control on `update_rewards` Allows Any Address to Permanently Deny Block Rewards to All Stakers - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` is documented to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` checkpoint without distributing rewards. Because the function reverts with `REWARDS_ALREADY_UPDATED` if called again in the same block, the sequencer is permanently locked out from distributing rewards for that block to every staker.

### Finding Description

The spec explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.  ← written unconditionally
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                    // ← exits without distributing anything
    }
    ...
``` [2](#0-1) 

`last_reward_block` is a single **global** storage variable shared across all stakers. Once it is written to the current block number, the `REWARDS_ALREADY_UPDATED` assertion prevents any further call to `update_rewards` in that block, for any staker. [3](#0-2) [4](#0-3) 

The interface definition confirms the function is public and takes an arbitrary `staker_address` and a caller-controlled `disable_rewards` flag: [5](#0-4) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` before the sequencer does. This:

1. Passes all guards (contract unpaused, staker active, block not yet recorded).
2. Writes `current_block_number` into `last_reward_block`.
3. Returns immediately without distributing any rewards.

The sequencer's subsequent call for the same block reverts with `REWARDS_ALREADY_UPDATED`. Block rewards for **every** staker are permanently lost for that block — they can never be retroactively credited because the accounting is strictly per-block.

This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- No special role, funds, or permissions are required.
- The attacker only needs to submit a transaction in the same block as the sequencer's reward distribution call.
- On Starknet, the sequencer processes transactions in order; a malicious user can submit this call at any time, and the sequencer itself (if compromised or front-run at the L2 level) could exploit this.
- The attack is repeatable every block at negligible cost.

### Recommendation

Add a caller check at the top of `update_rewards` that restricts execution to the authorized sequencer address (stored in contract configuration), mirroring the pattern already used in `update_rewards_from_attestation_contract`: [6](#0-5) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_sequencer(); // enforce "Only starkware sequencer"
    ...
```

The sequencer address should be stored in contract storage and set during initialization, analogous to how `attestation_contract_address` is stored and checked.

### Proof of Concept

1. Deploy the staking system and advance past `K` epochs so consensus rewards are active.
2. As an unprivileged address (not the sequencer), call:
   ```
   IStakingRewardsManagerDispatcher { contract_address: staking }.update_rewards(
       staker_address: any_active_staker,
       disable_rewards: true,
   );
   ```
3. Observe that `last_reward_block` is now set to the current block and `unclaimed_rewards_own` of the staker is unchanged.
4. The sequencer now attempts `update_rewards(staker_address, disable_rewards: false)` for the same block — it reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers permanently lose their block rewards for that block.

The existing test `test_update_rewards_without_distribute` already demonstrates that calling with `disable_rewards: true` advances `last_reward_block` without crediting rewards, and `test_update_rewards_assertions_already_consensus` confirms `REWARDS_ALREADY_UPDATED` is enforced per-block — both called from an unprivileged address with no access-control failure.         fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
