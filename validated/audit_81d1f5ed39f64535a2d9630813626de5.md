### Title
Signature Bypass via `skip_stateful_validations` Allows Unsigned Invoke Transactions into Mempool — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `skip_stateful_validations` function skips the `__validate__` entry point (signature verification) for any invoke transaction with nonce 1 when the account's on-chain nonce is 0 and `account_tx_in_pool_or_recent_block` returns `true`. The check does not verify that the transaction in the mempool is specifically a `deploy_account` transaction. An unprivileged attacker who observes a `deploy_account` in the mempool can immediately submit an invoke with nonce 1 carrying an arbitrary or empty signature for the same address, and the gateway will admit it without ever calling `__validate__`.
