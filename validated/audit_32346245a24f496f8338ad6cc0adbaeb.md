### Title
Gateway Stateful Validator Compares Transaction L2 Gas Price Against Current Block's Price Instead of Next Block's Price — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `validate_resource_bounds` function in `StatefulTransactionValidator` reads the L2 gas price from the **latest committed (previous) block** and uses it as the admission threshold for incoming transactions. However, those transactions will be executed in the **next** block, whose L2 gas price is computed by an EIP-1559-like mechanism and is not equal to the previous block's price. The code itself acknowledges this with a `TODO` comment. This is the direct analog of H-1: one quantity (current block's L2 gas price) is used where a different, non-equal quantity (next block's L2 gas price) is required.

The threshold is then computed as:

    * previous_block_l2_gas_price.get().0)

``` [2](#0-1) 

The next block's L2 gas price is computed separately by `calculate_next_l2_gas_price_for_fin` in the consensus orchestrator, which applies an EIP-1559-like adjustment based on actual gas usage: [3](#0-2) 

These two values — `previous_block_l2_gas_price` and `next_block_l2_gas_price` — are **not equal** in general. The next block's price rises when gas usage exceeds the target and falls when usage is below it.

The blockifier's own `check_fee_bounds` in `perform_pre_validation_stage` independently validates the transaction's `max_price_per_unit` against the **actual execution block's** gas price: [4](#0-3) 

This creates a split: the gateway enforces one price, the blockifier enforces another.

### Impact Explanation

**Scenario A — False rejection (primary impact):** When the next block's L2 gas price is lower than the current block's (network underutilized, EIP-1559 price decreasing), a transaction whose `max_price_per_unit` satisfies `next_block_price ≤ max_price_per_unit < min_gas_price_percentage% × previous_block_price` is **rejected at the gateway** even though the blockifier would accept it. Valid transactions are permanently denied admission without recourse other than resubmitting with a higher gas price.

**Scenario B — False admission (secondary impact):** When the next block's L2 gas price is higher than the current block's (network congested, price rising), a transaction whose `max_price_per_unit` satisfies `min_gas_price_percentage% × previous_block_price ≤ max_price_per_unit < next_block_price` passes gateway validation but fails blockifier pre-validation. The blockifier catches this, but the transaction pollutes the mempool.

Scenario A has no existing guard — the transaction is simply dropped. Scenario B is caught by the blockifier.

### Likelihood Explanation

The L2 gas price follows an EIP-1559-like mechanism. During periods of low network activity (below the gas target), the price decreases each block. This is the normal operating condition for a network that is not at capacity. In such conditions, `next_block_price < previous_block_price`, and Scenario A is continuously active. The magnitude of the mismatch grows with the number of consecutive low-usage blocks. With `min_gas_price_percentage = 100` (the maximum tested value), any transaction priced between the two values is rejected. The TODO comment in the production code confirms the developers are aware the wrong value is being used.

### Recommendation

Replace the read of `strk_gas_prices.l2_gas_price` from the latest committed block with the **next** block's L2 gas price, as the TODO comment already prescribes. The next L2 gas price is available from the block header (stored as `next_l2_gas_price` in `BlockHeaderWithoutHash`) or can be computed via `calculate_next_l2_gas_price_for_fin` using the previous block's gas usage. This mirrors the fix recommended for H-1: use the correct quantity (balance difference / next block price) rather than the proxy quantity (shares returned / current block price).

### Proof of Concept

1. Network is below gas target for several blocks; `next_block_l2_gas_price = 0.90 × previous_block_l2_gas_price`.
2. Gateway is configured with `min_gas_price_percentage = 100`, so `threshold = previous_block_l2_gas_price`.
3. User submits a V3 invoke transaction with `l2_gas.max_price_per_unit = 0.95 × previous_block_l2_gas_price`.
4. `validate_tx_l2_gas_price_within_threshold` computes `threshold = previous_block_l2_gas_price` and rejects the transaction with `GAS_PRICE_TOO_LOW` because `0.95 × P < 1.00 × P`.
5. The blockifier would have accepted this transaction because `0.95 × P > 0.90 × P = next_block_price`.
6. The user's valid transaction is permanently denied at the gateway. [5](#0-4)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
