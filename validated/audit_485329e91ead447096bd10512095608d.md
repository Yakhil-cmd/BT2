### Title
`update_rewards` Advances `last_reward_block` Before Distributing Rewards, Allowing Anyone to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `StakingRewardsManagerImpl::update_rewards` function writes `last_reward_block` to storage **before** checking the `disable_rewards` flag. Because `disable_rewards` is an unguarded public parameter, any unprivileged caller can invoke `update_rewards(victim_staker, disable_rewards: true)` to advance `last_reward_block` for the current block without distributing any rewards. The per-block guard then permanently prevents a legitimate reward call for that block, causing the staker to lose unclaimed yield.

### Finding Description
In `src/staking/staking.cairo`, `update_rewards` executes in this order:

1. **Guard check** – asserts `current_block_number > last_reward_block` (lines 1454–1458). Only one call per block is allowed.
2. **Staker validity checks** – asserts staker is active and has non-zero balance (lines 1466–1482).
3. **`last_reward_block` is written** – `self.last_reward_block.write(current_block_number)` at line 1485. This happens unconditionally.
4. **Early return if `disable_rewards || is_pre_consensus()`** – lines 1487–1489. No rewards are distributed.
5. **Reward distribution** – only reached if step 4 is not triggered (lines 1492–1506). [1](#0-0) 

Because step 3 occurs before step 4, any caller who passes `disable_rewards: true` consumes the block's reward slot without distributing anything. The guard in step 1 then makes it impossible for anyone else to distribute rewards for that block.

`general_prerequisites()` only checks that the contract is unpaused and the caller is non-zero — there is no role or ownership check on `disable_rewards`. [2](#0-1) 

### Impact Explanation
**High – Permanent freezing of unclaimed yield.**

An attacker can call `update_rewards(victim_staker, disable_rewards: true)` once per block for every block in an epoch. Each call advances `last_reward_block` without distributing rewards. Because the per-block guard is a strict greater-than check, the missed rewards for each block are irrecoverable. Repeating this every block permanently denies the targeted staker (and their delegators) all consensus-era block rewards.

### Likelihood Explanation
**High.** The function is public, requires no privileged role, and costs only gas. The attacker gains nothing financially but can continuously grief any staker. The attack is trivially automatable.

### Recommendation
Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` / `is_pre_consensus()` guard, so the block slot is only consumed when rewards are actually distributed:

```cairo
// Update last block rewards ONLY when rewards will be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}

self.last_reward_block.write(current_block_number); // moved here

let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
...
```

Alternatively, restrict who may pass `disable_rewards: true` to a privileged role (e.g., `only_security_agent`).

### Proof of Concept
1. Staker A is active with non-zero STRK balance; consensus rewards are live (`is_pre_consensus()` is false).
2. At block N, attacker calls `update_rewards(staker_A_address, disable_rewards: true)`.
3. The function passes all validity checks, writes `last_reward_block = N`, then returns early — no rewards distributed.
4. Any subsequent call to `update_rewards` for block N reverts with `REWARDS_ALREADY_UPDATED` (line 1457).
5. Staker A permanently loses block-N rewards.
6. Attacker repeats at block N+1, N+2, … effectively zeroing all future consensus rewards for staker A. [3](#0-2)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
