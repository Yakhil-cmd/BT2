### Title
`update_rewards` Lacks Caller Access Control, Allowing Any Staker to Block Consensus Reward Distribution - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in the `IStakingRewardsManager` interface is documented as callable only by the Starkware sequencer, but the implementation contains no caller validation. Any valid staker can invoke it with `disable_rewards: true`, consuming the per-block `last_reward_block` slot without distributing rewards, permanently blocking the sequencer from distributing consensus rewards to all stakers for that block.

### Finding Description

The `StakingRewardsManagerImpl::update_rewards` function at `src/staking/staking.cairo` is the consensus-era reward distribution entry point. The specification explicitly restricts it to "Only starkware sequencer": [1](#0-0) 

The implementation, however, performs no such check. The only guards are:

1. Contract is unpaused (`general_prerequisites`)
2. `current_block_number > last_reward_block` (one call per block, globally)
3. Staker exists and is active
4. Staker has non-zero balance [2](#0-1) 

There is no `assert_caller_is_sequencer()` or equivalent. Compare this with the analogous attestation path, which does enforce its caller restriction: [3](#0-2) 

The critical state variable `last_reward_block` is a **global** (not per-staker) slot. Writing it once per block with `disable_rewards: true` consumes the slot without distributing any rewards: [4](#0-3) 

The `staker_address` parameter is also unsanitized beyond existence/activity checks — any registered staker's address can be passed, meaning the attacker does not need to target their own account.

### Impact Explanation

An attacker who is a registered staker with non-zero balance can call `update_rewards(any_valid_staker_address, disable_rewards: true)` in every block. Because `last_reward_block` is global, this:

- Prevents the sequencer from calling `update_rewards` for any staker in that block (the `REWARDS_ALREADY_UPDATED` assertion fires for all subsequent calls in the same block).
- Distributes zero rewards (because `disable_rewards: true` causes an early return before reward calculation).

Sustained over multiple blocks, this permanently freezes all unclaimed consensus-era yield for every staker and every delegation pool in the protocol. The attacker gains nothing financially but inflicts total reward denial on all participants.

**Impact category matched:** High — Permanent freezing of unclaimed yield.

### Likelihood Explanation

- The attacker only needs to be a registered staker (minimum stake threshold met), which is an unprivileged role open to anyone.
- The call is cheap (no token transfer, no complex computation).
- The attack is repeatable every block with a single transaction per block.
- No privileged key, bridge compromise, or external dependency is required.

### Recommendation

Add an explicit sequencer-address check at the top of `update_rewards`, analogous to `assert_caller_is_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
}
```

Store the sequencer address in contract storage (set at initialization or by governance), and enforce it here. Alternatively, restrict the function to a dedicated role (e.g., `SEQUENCER_ROLE`) using the existing `RolesComponent`.

### Proof of Concept

1. Deploy the staking system and register `attacker` as a staker with minimum stake.
2. Advance K epochs so `attacker` has non-zero balance.
3. Enable consensus rewards (`set_consensus_rewards_first_epoch`).
4. In every block, `attacker` calls:
   ```cairo
   IStakingRewardsManagerDispatcher { contract_address: staking_contract }
       .update_rewards(staker_address: attacker_address, disable_rewards: true);
   ```
5. The sequencer's subsequent call to `update_rewards` for any legitimate staker reverts with `REWARDS_ALREADY_UPDATED`.
6. No staker accumulates `unclaimed_rewards_own` for any block the attacker front-runs.

The existing test suite confirms no caller restriction is tested or expected: [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
