### Title
Unrestricted `exit_delegation_pool_action` Allows Any Caller to Front-Run and Block a Pool Member's `switch_delegation_pool` - (File: src/pool/pool.cairo)

### Summary

`Pool::exit_delegation_pool_action` accepts a `pool_member` address parameter and has no access control. Any external caller can trigger the exit action on behalf of any pool member once the wait window has elapsed. Because `switch_delegation_pool` requires `pool_member_info.unpool_time.is_some()`, an attacker who calls `exit_delegation_pool_action` first clears `unpool_time` and `unpool_amount`, permanently blocking the victim's ability to switch pools in that exit window.

### Finding Description

`exit_delegation_pool_action` in `src/pool/pool.cairo` takes `pool_member: ContractAddress` as a parameter and performs no caller check:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    ...
    pool_member_info.unpool_amount = Zero::zero();
    pool_member_info.unpool_time = Option::None;
    self.write_pool_member_info(:pool_member, :pool_member_info);
    token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
``` [1](#0-0) 

The spec explicitly documents this as open: "Any address can execute." [2](#0-1) 

Meanwhile, `switch_delegation_pool` — the competing action a pool member may want to take instead of exiting — requires `unpool_time.is_some()`:

```cairo
fn switch_delegation_pool(...) -> Amount {
    let pool_member = get_caller_address();
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(
        pool_member_info.unpool_time.is_some(),
        "{}",
        GenericError::MISSING_UNDELEGATE_INTENT,
    );
``` [3](#0-2) 

Once `exit_delegation_pool_action` runs, `unpool_time` is set to `None` and `unpool_amount` to zero, so any subsequent call to `switch_delegation_pool` by the pool member reverts with `MISSING_UNDELEGATE_INTENT`. [4](#0-3) 

### Impact Explanation

The pool member receives their funds back (no direct token loss), but they lose the ability to switch delegation pools in that exit window. To switch pools they must re-enter the delegation pool from scratch, which means waiting K epochs before their stake becomes effective again and earning no staking rewards during that period. This constitutes griefing with no profit motive but measurable damage to the user (lost yield for K epochs, extra gas costs).

This falls under: **Medium — Griefing with no profit motive but damage to users or protocol.**

### Likelihood Explanation

The attack requires no special privilege. Any address can call `exit_delegation_pool_action(victim)` once the exit wait window has elapsed. An attacker monitoring on-chain state (or the mempool) for `PoolMemberExitIntent` events can trivially time the call to race the victim's `switch_delegation_pool` transaction. The cost to the attacker is only gas.

### Recommendation

Add a caller check to `exit_delegation_pool_action` so that only the pool member (or their designated reward address) can trigger the exit action:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let pool_member_info = self.internal_pool_member_info(:pool_member);
    let caller = get_caller_address();
    assert!(
        caller == pool_member || caller == pool_member_info.reward_address,
        "{}",
        Error::CALLER_CANNOT_EXIT_POOL,
    );
    ...
```

This mirrors the access control already applied to `claim_rewards` and `add_to_delegation_pool`. [5](#0-4) [6](#0-5) 

### Proof of Concept

1. Pool member calls `exit_delegation_pool_intent(amount)`. This sets `unpool_time = Some(T)` and `unpool_amount = amount`.
2. Pool member waits for the exit window to elapse, intending to call `switch_delegation_pool(to_staker, to_pool, amount)` to move their stake rather than fully exit.
3. Attacker observes the `PoolMemberExitIntent` event on-chain and, once `Time::now() >= T`, calls `exit_delegation_pool_action(pool_member)` before the pool member's `switch_delegation_pool` transaction is included.
4. `exit_delegation_pool_action` succeeds: it clears `unpool_time` to `None` and `unpool_amount` to `0`, and transfers `amount` tokens back to `pool_member`.
5. Pool member's `switch_delegation_pool` transaction now reverts at the assertion `pool_member_info.unpool_time.is_some()` with `MISSING_UNDELEGATE_INTENT`.
6. To switch pools, the pool member must call `enter_delegation_pool` again and wait K epochs before their stake is effective, losing staking rewards for that period. [7](#0-6) [1](#0-0)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
