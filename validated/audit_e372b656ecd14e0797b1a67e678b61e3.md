### Title
Excess STRK Deposited via StarkGate to `RewardSupplier` is Permanently Locked — (`src/reward_supplier/reward_supplier.cairo`)

### Summary

The `RewardSupplier` contract's `on_receive` function accepts STRK deposits from any depositor via StarkGate, but only updates `l1_pending_requested_amount` — never `unclaimed_rewards`. Because `claim_rewards` only transfers STRK up to `unclaimed_rewards`, any STRK received beyond that tracked amount is permanently locked with no recovery path.

### Finding Description

`on_receive` is the StarkGate callback that fires whenever STRK is bridged to the `RewardSupplier`. The code explicitly notes that the depositor is not validated: [1](#0-0) 

The function only adjusts `l1_pending_requested_amount` and returns `true`. It never touches `unclaimed_rewards`. The only outbound path for STRK is `claim_rewards`, which enforces a hard ceiling of `unclaimed_rewards`: [2](#0-1) 

`unclaimed_rewards` is only ever increased by `update_unclaimed_rewards_from_staking_contract`, which is restricted to the staking contract: [3](#0-2) 

There is no sweep, rescue, or governance-withdrawal function anywhere in the contract. Any STRK balance that exceeds `unclaimed_rewards` is irrecoverable.

### Impact Explanation

An unprivileged actor can deposit an arbitrary amount of STRK to the `RewardSupplier` via StarkGate. Those tokens accumulate in the contract's ERC-20 balance but are never credited to `unclaimed_rewards`, so `claim_rewards` can never reach them. The funds are permanently frozen inside the contract. This matches the **Medium** impact category: griefing with no profit motive but concrete, irreversible damage to the protocol (permanently inaccessible STRK).

### Likelihood Explanation

The entry path requires no privilege. StarkGate's deposit flow is public; any L1 address can call `depositWithMessage` targeting the `RewardSupplier`. The contract's own comment confirms this is an expected, unguarded scenario. Likelihood is medium — it requires deliberate action but zero access control.

### Recommendation

1. **Restrict the depositor**: check that `depositor` equals the registered `l1_reward_supplier` address inside `on_receive`, rejecting unsolicited deposits.
2. **Or add a recovery function**: expose a governance-gated function that transfers any STRK balance in excess of `unclaimed_rewards` to a designated address.

### Proof of Concept

1. Attacker calls StarkGate on L1 with `depositWithMessage`, targeting the `RewardSupplier` as recipient, for amount `X` STRK.
2. StarkGate calls `on_receive(l2_token=STRK, amount=X, depositor=attacker, message=[])` on the `RewardSupplier`.
3. `on_receive` passes both assertions (caller is StarkGate, token is STRK), decrements `l1_pending_requested_amount` (possibly to zero), and returns `true`.
4. The contract's STRK ERC-20 balance increases by `X`, but `unclaimed_rewards` is unchanged.
5. Any subsequent call to `claim_rewards(amount)` is bounded by `unclaimed_rewards`; the `X` STRK deposited by the attacker can never be transferred out.
6. No function in the contract can recover these tokens. [4](#0-3)

        fn claim_rewards(ref self: ContractState, amount: Amount) {
            // Asserts.
            let staking_contract = self.staking_contract.read();
            assert!(
                get_caller_address() == staking_contract,
                "{}",
                GenericError::CALLER_IS_NOT_STAKING_CONTRACT,
            );
            let unclaimed_rewards = self.unclaimed_rewards.read();
            assert!(amount <= unclaimed_rewards, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Update unclaimed_rewards and transfer the requested rewards to the staking contract.
            self.unclaimed_rewards.write(unclaimed_rewards - amount);
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: staking_contract, amount: amount.into());
        }

        fn on_receive(
            ref self: ContractState,
            l2_token: ContractAddress,
            amount: u256,
            depositor: EthAddress,
            message: Span<felt252>,
        ) -> bool {
            // Note that the deposit can be done by anyone (not just the L1 reward supplier), so
            // depositor is not checked.

            // These messages accepted only from the token bridge.
            assert!(
                get_caller_address() == self.starkgate_address.read(),
                "{}",
                Error::ON_RECEIVE_NOT_FROM_STARKGATE,
            );
            // The bridge may serve multiple tokens, only the correct token may be received.
            assert!(
                l2_token == self.token_dispatcher.contract_address.read(),
                "{}",
                Error::UNEXPECTED_TOKEN,
            );
            let amount_u128: Amount = amount
                .try_into()
                .expect_with_err(GenericError::AMOUNT_TOO_HIGH);
            let mut l1_pending_requested_amount = self.l1_pending_requested_amount.read();
            if amount_u128 > l1_pending_requested_amount {
                self.l1_pending_requested_amount.write(Zero::zero());
            } else {
                l1_pending_requested_amount -= amount_u128;
                self.l1_pending_requested_amount.write(l1_pending_requested_amount);
            }
            true
        }
