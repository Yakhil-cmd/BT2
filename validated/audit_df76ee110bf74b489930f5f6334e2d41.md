### Title
`exit_to_near_precompile_callback` Ignores Unused Amount from `ft_transfer_call`, Permanently Burning User ERC-20 Tokens - (File: engine/src/contract_methods/connector.rs)

### Summary

When a user exits ERC-20 tokens to NEAR using the Omni message path (`Message::Omni`), the exit precompile burns the full requested amount of ERC-20 tokens and issues a `ft_transfer_call` promise to the NEP-141 contract. The callback `exit_to_near_precompile_callback` only distinguishes between a failed promise (triggering a refund) and a successful promise (doing nothing). It never inspects the unused-amount value returned by `ft_transfer_call`. If the NEP-141 receiver's `ft_on_transfer` returns a non-zero unused amount, the NEP-141 contract refunds those tokens to Aurora's own balance, but the corresponding ERC-20 tokens remain permanently burned and are never re-minted for the user.

### Finding Description

**Entry path – the Omni exit flow**

In `engine-precompiles/src/native.rs`, `exit_erc20_token_to_near` handles the `Message::Omni` variant by selecting `"ft_transfer_call"` as the NEP-141 method and setting `transfer_near_args = None`: [1](#0-0) 

A callback `exit_to_near_precompile_callback` is attached only when `callback_args != default()`. With the `error_refund` feature enabled, `refund` is `Some(...)`, so the callback is always attached for the Omni path: [2](#0-1) 

**The accounting gap in the callback**

`exit_to_near_precompile_callback` branches on whether the promise succeeded or failed. On success it does nothing (returns `None`); on failure it calls `refund_on_error`: [3](#0-2) 

The `_` wildcard at line 214 silently discards the bytes inside `PromiseResult::Successful(bytes)`. Per the NEP-141 standard, those bytes are the JSON-encoded **unused amount** returned by the receiver's `ft_on_transfer`. The NEP-141 contract has already refunded that unused amount to Aurora's own NEP-141 balance (the `ft_transfer_call` sender), but the callback never reads it and never re-mints the corresponding ERC-20 tokens for the user.

**Contrast with the failure path**

When the promise fails, `refund_on_error` correctly re-mints the full burned amount: [4](#0-3) 

No equivalent recovery exists for the partial-consumption success case.

**ERC-20 burn is unconditional**

The ERC-20 tokens are burned inside `withdrawToNear` before any promise is created. With `error_refund` enabled, the `EvmErc20V2` contract burns the full `amount` from the caller: [5](#0-4) 

There is no mechanism to un-burn them after a partial success.

### Impact Explanation

Let `X` = amount the user requested to exit, `Y` = unused amount returned by the receiver (0 < Y ≤ X).

| Party | ERC-20 balance change | NEP-141 balance change |
|---|---|---|
| User | −X (burned, never restored) | 0 |
| Receiver | 0 | +(X−Y) |
| Aurora | 0 | +Y (orphaned surplus) |

The user permanently loses Y ERC-20 tokens. The Y NEP-141 tokens sit in Aurora's balance with no corresponding ERC-20 tokens, creating a supply imbalance. This constitutes **direct theft of user funds** (Critical).

### Likelihood Explanation

The Omni exit path is a production feature reachable by any EVM user who calls `withdrawToNear` with an Omni-formatted message. Any NEP-141 receiver whose `ft_on_transfer` returns a non-zero unused amount (e.g., a contract that partially fills an order, enforces a cap, or rejects the transfer entirely) triggers the loss. No special privilege is required; a user can self-trigger the loss by targeting their own contract.

### Recommendation

In `exit_to_near_precompile_callback`, when the promise result is `Successful(bytes)`, parse the unused amount from `bytes` (the JSON-encoded `U128` returned by `ft_transfer_call`). If the unused amount is non-zero, call `refund_on_error` (or an equivalent re-mint path) to restore the corresponding ERC-20 tokens to the user's address stored in `refund.recipient_address`.

```rust
if let Some(PromiseResult::Successful(bytes)) = handler.promise_result(0) {
    // Parse unused amount returned by ft_transfer_call
    let unused: u128 = serde_json::from_slice::<U128>(&bytes)
        .map(|v| v.0)
        .unwrap_or(0);
    if unused > 0 {
        if let Some(mut refund_args) = args.refund {
            // Adjust refund amount to only the unused portion
            let unused_u256 = U256::from(unused);
            refund_args.amount = types::u256_to_arr(&unused_u256);
            let refund_result = engine::refund_on_error(io, env, state, &refund_args, handler)?;
            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }
        }
    }
    // ... existing transfer_near logic
}
```

### Proof of Concept

1. Deploy a NEP-141 token and bridge it to Aurora (creating an ERC-20 mirror).
2. Deploy a NEAR contract `ReceiverContract` whose `ft_on_transfer` always returns the full `amount` (i.e., refuses all tokens).
3. As an EVM user holding 1000 ERC-20 tokens, call `withdrawToNear` with an Omni message targeting `ReceiverContract` for 1000 tokens.
4. Observe: ERC-20 balance drops to 0. `ft_transfer_call` succeeds (promise result = `"1000"` unused). `exit_to_near_precompile_callback` fires, sees `Successful`, does nothing. Aurora's NEP-141 balance increases by 1000. User's ERC-20 balance remains 0 with no refund. [6](#0-5) [1](#0-0)

#[named]
pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
