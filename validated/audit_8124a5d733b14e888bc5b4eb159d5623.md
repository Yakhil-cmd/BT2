### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Deny Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `StakingRewardsManagerImpl` is documented to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, which advances the global `last_reward_block` checkpoint without distributing rewards, permanently preventing the sequencer from distributing rewards for that block to all stakers and delegators.

### Finding Description
The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1447–1507 contains no `get_caller_address()` check whatsoever:

```rust
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // only checks paused
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... no caller identity check ...
        self.last_reward_block.write(current_block_number);   // global write
        if disable_rewards || self.is_pre_consensus() {
            return;                                           // exits without distributing
        }
        // ... reward distribution only reached if disable_rewards == false
    }
}
```

`last_reward_block` is a single **global** storage variable shared across all stakers. Once it is written to `current_block_number`, the guard at line 1454–1458 (`current_block_number > self.last_reward_block.read()`) will revert with `REWARDS_ALREADY_UPDATED` for any subsequent call in the same block — including the legitimate sequencer call.

An attacker therefore only needs to:
1. Call `update_rewards(any_valid_active_staker, disable_rewards: true)` before the sequencer in any block.
2. `last_reward_block` is advanced to the current block.
3. No rewards are computed or distributed.
4. The sequencer's own call for that block reverts with `REWARDS_ALREADY_UPDATED`.
5. The rewards that would have accrued for that block are **permanently lost** — there is no catch-up mechanism for skipped blocks in the consensus-rewards path.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

Every block the attacker front-runs, all stakers and all delegation-pool members lose their pro-rata share of block rewards forever. The attacker can repeat this every block at negligible cost (a single transaction per block), effectively zeroing out all consensus-phase staking rewards for the entire protocol.

### Likelihood Explanation
**Medium.** The attack requires no special role, no funds, and no privileged access — only the ability to submit a transaction before the sequencer's reward-update transaction in each block. On Starknet, where the sequencer ordering is observable, a determined griever can reliably front-run this call. The cost is one transaction per block; the damage is permanent per-block reward loss for every participant.

### Recommendation
Add an explicit caller check at the top of `update_rewards` that restricts execution to the designated sequencer address (stored in contract storage or derived from `get_execution_info`), mirroring the pattern already used for `update_rewards_from_attestation_contract` which correctly asserts `CALLER_IS_NOT_ATTESTATION_CONTRACT`. For example:

```rust
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
}
```

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker (any EOA) observes the mempool / block production and submits:
   ```
   staking_rewards_dispatcher.update_rewards(
       staker_address: <any_valid_active_staker>,
       disable_rewards: true,
   )
   ```
   before the sequencer's own `update_rewards` call in block `N`.
3. Inside `update_rewards`:
   - `general_prerequisites()` passes (contract is unpaused). [1](#0-0) 
   - `current_block_number > last_reward_block` passes (first call this block). [2](#0-1) 
   - `last_reward_block` is written to block `N`. [3](#0-2) 
   - `disable_rewards == true` → early return, zero rewards distributed. [4](#0-3) 
4. The sequencer's `update_rewards` call for block `N` now reverts: `current_block_number > last_reward_block` is `false`. [5](#0-4) 
5. Block `N` rewards are permanently lost for all stakers and delegators.
6. Repeat every block.

The spec explicitly states this function is "Only starkware sequencer" [6](#0-5)  but the implementation enforces no such restriction. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
