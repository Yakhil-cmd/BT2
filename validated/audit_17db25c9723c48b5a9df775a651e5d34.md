### Title
Anyone Can Call `update_rewards` With `disable_rewards: true` to Permanently Freeze Staker Yield — (`File: src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the Staking contract is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. When set to `true`, the function still writes the current block number to the global `last_reward_block` storage slot, consuming the block's reward slot, but returns before distributing any rewards. Because `last_reward_block` is a global gate that prevents re-entry in the same block, an attacker who calls this every block can permanently freeze all staker yield with no profit motive.

---

### Finding Description

`update_rewards` is declared under `StakingRewardsManagerImpl` with no role check — only `general_prerequisites()`, which verifies the contract is not paused and the caller is non-zero. [1](#0-0) 

The function accepts an arbitrary `staker_address` and a caller-controlled `disable_rewards: bool`: [2](#0-1) 

The critical sequence is:

1. The block-level gate is checked: [3](#0-2) 

2. `last_reward_block` is **unconditionally** written to the current block number: [4](#0-3) 

3. Then, if `disable_rewards` is `true`, the function returns early with no rewards distributed: [5](#0-4) 

Because `last_reward_block` is a **global** (not per-staker) storage variable, once an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N, any legitimate call to `update_rewards` in the same block N will revert with `REWARDS_ALREADY_UPDATED`. The rewards for block N are permanently lost — they are never credited to `unclaimed_rewards` in the reward supplier and never transferred to any staker.

An attacker repeating this call every block (trivially achievable by front-running or simply submitting first) permanently prevents all consensus-era reward distribution across the entire protocol.

---

### Impact Explanation

Each block's rewards are computed from the minting curve and block duration, then credited to stakers via `_update_rewards`. When `disable_rewards: true` is used and `last_reward_block` is consumed, those block rewards are silently discarded — they are never added to `unclaimed_rewards` in the reward supplier and never reach any staker's `unclaimed_rewards_own`. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators in the protocol. [6](#0-5) 

---

### Likelihood Explanation

The entry path requires no privilege: any non-zero address can call `update_rewards` on any valid, active staker. The attacker only needs to supply a valid `staker_address` (publicly enumerable from the `stakers` Vec in storage) and set `disable_rewards: true`. Gas cost is the only barrier. The attack is sustainable indefinitely and can be automated. [7](#0-6) 

---

### Recommendation

Restrict who may call `update_rewards` or who may set `disable_rewards: true`. Options:

1. **Restrict the caller**: Gate `update_rewards` to only the attestation contract, a designated rewards-manager role, or the staker themselves.
2. **Restrict the `disable_rewards` flag**: Only allow `disable_rewards: true` from a privileged role (e.g., `only_security_agent()`), while permitting unprivileged callers only with `disable_rewards: false`.
3. **Decouple the gate update**: Only write `last_reward_block` after rewards have been successfully distributed (or after the `disable_rewards` branch is confirmed to be privileged).

---

### Proof of Concept

```
// Attacker (any address) calls every block:
staking.update_rewards(
    staker_address: any_valid_active_staker,
    disable_rewards: true,
);
// Result:
// - last_reward_block is set to current block
// - No rewards are distributed
// - Legitimate update_rewards calls in the same block revert with REWARDS_ALREADY_UPDATED
// - Repeating this every block permanently freezes all staker yield
```

The attacker needs only a valid `staker_address`, obtainable by reading the public `stakers` Vec. No tokens, no role, no special access required. [8](#0-7)
