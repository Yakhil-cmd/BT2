### Title
Missing Caller Access Control on `update_rewards` Allows Any Address to Permanently Suppress Reward Distribution - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` is documented as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged address can call it with `disable_rewards: true` every block, consuming the single per-block reward slot and permanently preventing legitimate reward distribution to all stakers and delegators.

### Finding Description

The specification for `update_rewards` explicitly states:

> **Access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo:1449-1507` only calls `self.general_prerequisites()`, which checks two things:

1. The contract is not paused
2. The caller is not the zero address [1](#0-0) 

There is no assertion that `get_caller_address()` equals the sequencer or any privileged role. The function is callable by any non-zero address.

The function uses a single global `last_reward_block` storage variable to enforce a "one call per block" invariant: [2](#0-1) 

When `disable_rewards: true` is passed (or when `is_pre_consensus()` is true), the function writes `current_block_number` to `last_reward_block` and returns early **without distributing any rewards**: [3](#0-2) 

Because `last_reward_block` is a single global value (not per-staker), any call to `update_rewards` for **any** valid staker address in a given block exhausts the slot for that block. A subsequent call in the same block — even with `disable_rewards: false` — will revert with `REWARDS_ALREADY_UPDATED`.

The spec confirms this is the intended design: [4](#0-3) 

### Impact Explanation

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` once per block. This:

1. Sets `last_reward_block` to the current block number.
2. Returns without distributing any STRK or BTC rewards.
3. Causes every subsequent call to `update_rewards` in that block to revert with `REWARDS_ALREADY_UPDATED`.

The sequencer's legitimate call with `disable_rewards: false` is permanently blocked for that block. Repeating this every block freezes all unclaimed yield for all stakers and delegators indefinitely.

This matches the allowed impact: **High: Permanent freezing of unclaimed yield or unclaimed royalties**.

### Likelihood Explanation

- The function is publicly callable with no access control beyond "not paused" and "not zero address."
- The attacker only needs to know one valid, active staker address (all staker addresses are public on-chain).
- The cost is one transaction per block — trivially affordable.
- No special privileges, leaked keys, or external dependencies are required.
- The attack is confirmed by the test suite, which calls `update_rewards` without any `cheat_caller_address_once` setup, demonstrating it is callable by any address. [5](#0-4) 

### Recommendation

Add a caller check at the top of `update_rewards` to enforce that only the authorized sequencer address (or a designated rewards manager role) can call the function, consistent with the specification:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

Alternatively, introduce a dedicated `REWARDS_MANAGER` role and gate the function on that role, mirroring how `update_rewards_from_attestation_contract` is gated on the attestation contract address. [6](#0-5) 

### Proof of Concept

1. Deploy the system (any valid configuration).
2. Advance K epochs so a staker has non-zero balance and consensus rewards are active.
3. In every block, call `update_rewards(staker_address, disable_rewards: true)` from any EOA before the sequencer's transaction.
4. Observe that `last_reward_block` is updated to the current block.
5. The sequencer's call to `update_rewards(staker_address, disable_rewards: false)` reverts with `REWARDS_ALREADY_UPDATED`.
6. After N blocks, `staker_info.unclaimed_rewards_own` remains zero despite N blocks of elapsed time.
7. All delegators' pool balances also remain zero — no STRK is ever transferred to pool contracts.

The existing test `test_update_rewards_assertions` at `src/staking/tests/test.cairo:3488` already demonstrates that `update_rewards` is callable by any address without a caller check, and that `REWARDS_ALREADY_UPDATED` is triggered on a second call in the same block regardless of who calls first. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
