### Title
`set_commission` Applies Commission Increase Retroactively to Current Epoch Rewards Without Settling Pending Delegator Yield — (File: src/staking/staking.cairo)

---

### Summary

The `set_commission` function in the staking contract updates the commission rate immediately in storage without first distributing pending rewards to the pool. Because rewards are calculated at attestation time using the commission that is current at that moment, a staker who uses `set_commission_commitment` to unlock a commission increase and then calls `set_commission` just before `attest()` can apply the new higher commission retroactively to the entire epoch's rewards, stealing delegators' yield for that epoch.

---

### Finding Description

**Root cause — `set_commission` does not settle pending rewards:**

In `src/staking/staking.cairo`, `set_commission` (line 725) writes the new commission directly to storage with no prior reward settlement: [1](#0-0) 

The internal helper `update_commission` (line 1573) likewise only validates the new value and writes it; it never triggers a reward distribution: [2](#0-1) 

**Commission is read at reward-distribution time, not at accrual time:**

In `calculate_staker_pools_rewards` (line 1964), the commission used to split rewards is the value that is current in storage at the moment `attest()` fires: [3](#0-2) 

In the pre-consensus (attestation-based) model, `update_rewards_from_attestation_contract` calculates and distributes the **entire epoch's** rewards in a single call. Whatever commission is stored at that instant is applied to all of the epoch's pool rewards.

**`set_commission_commitment` enables commission increases:**

Without a commitment, `set_commission` can only decrease commission. With an active commitment, a staker may increase commission up to `max_commission`: [4](#0-3) 

The developers themselves note the residual risk at line 745–746: [5](#0-4) 

**Attack sequence:**

1. Staker sets commission = 0 to attract delegators.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
3. Delegators join the pool.
4. Within the attestation window, staker calls `set_commission(10000)` (100%).
5. Staker's operational address immediately calls `attest()`.
6. `_update_rewards` → `calculate_staker_pools_rewards` reads `commission = 10000`.
7. `split_rewards_with_commission(pool_rewards_including_commission, 10000)` → `pool_rewards = 0`.
8. Pool receives zero rewards; staker captures all pool rewards as commission.
9. Delegators' `unclaimed_rewards` remain zero for the entire epoch. [6](#0-5) 

---

### Impact Explanation

**High — Theft of unclaimed yield.** Delegators who staked for an entire epoch under a 0% commission can have their complete epoch yield stolen. The staker captures all pool rewards as commission by timing a single `set_commission` call just before `attest()`. The loss is bounded by `max_commission` in the commitment, but `max_commission` can be set to 10 000 (100%), making the loss total for that epoch.

---

### Likelihood Explanation

**Medium.** Prerequisites are: (a) the staker must have previously called `set_commission_commitment` (a public, on-chain action), and (b) the staker must time `set_commission` before `attest()`. Because the staker controls the operational address that calls `attest()`, they can trivially sequence both calls in the same block or consecutive blocks within the attestation window. The `set_commission_commitment` step is the only non-trivial prerequisite.

---

### Recommendation

Before writing the new commission value in `set_commission`, distribute any pending rewards to the pool using the old commission rate (analogous to the external report's recommendation to "take the fee and reset the timestamp"). Alternatively, commission changes should be epoch-delayed — the new rate should only take effect starting from the next epoch, not the current one, so that rewards accrued under the old rate are always settled at the old rate.

---

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );
                total_commission_rewards += commission_rewards;
                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)
