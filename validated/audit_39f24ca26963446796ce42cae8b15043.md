### Title
Missing Minimum Threshold Enforcement in FROST `presign` Allows Single-Party Signature Forgery — (`File: src/frost/mod.rs`)

---

### Summary

The FROST `presign` function validates that the caller-supplied threshold does not exceed the participant count, but omits the symmetric lower-bound check (`threshold >= 2`) that every other protocol entry-point in the library enforces. A malicious coordinator can therefore invoke `presign` with `threshold = 1`, reducing the effective signing quorum to a single party and enabling unauthorized threshold signature creation.

---

### Finding Description

Every other threshold-bearing entry-point in the library enforces a minimum threshold of 2:

- `assert_key_invariants` (`src/dkg.rs:580–582`) rejects `threshold < 2` with `ThresholdTooSmall { threshold, min: 2 }`.
- `validate_triple_inputs` (`src/ecdsa/ot_based_ecdsa/triples/generation.rs:699–703`) performs the identical check for OT-based ECDSA triple generation.

The FROST `presign` function (`src/frost/mod.rs:44–88`) performs only the upper-bound check:

```rust
// src/frost/mod.rs:71-77
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge {
        threshold: args.threshold.into(),
        max: participants.len(),
    });
}
```

The lower-bound guard — `if args.threshold.value() < 2 { return Err(ThresholdTooSmall) }` — is entirely absent. [1](#0-0) 

Compare with the enforced minimum in DKG: [2](#0-1) 

And in OT-based triple generation: [3](#0-2) 

The `presign` function then passes the unchecked threshold directly into `do_presign`, which uses it to parameterize the nonce-commitment aggregation and the subsequent signing phase. [4](#0-3) 

---

### Impact Explanation

**Critical — Unauthorized creation of a valid threshold signature.**

With `threshold = 1`, Shamir secret sharing degenerates: the secret polynomial has degree 0, meaning the secret equals every participant's share. During the FROST signing phase, Lagrange interpolation with a single share reconstructs the full signing key. A single malicious participant holding one share can therefore:

1. Call `presign` with `threshold = 1` (accepted without error).
2. Proceed to the signing phase where only their own partial signature is required.
3. Produce a cryptographically valid aggregate FROST signature over an attacker-chosen message without cooperation from any other participant.

This completely breaks the t-of-n security guarantee for EdDSA and RedJubjub schemes.

---

### Likelihood Explanation

The `presign` function is a public API entry-point callable by any library user acting as coordinator. No special privilege, leaked key, or cryptographic break is required — only passing an integer `1` as the threshold argument. The keygen phase correctly rejects `threshold = 1`, so the key shares were generated under a valid threshold; the bypass occurs exclusively at the presign call site, which is a separate, independently-invocable function. Any coordinator (trusted or malicious) who controls the `PresignArguments` struct can trigger this.

---

### Recommendation

Add the minimum threshold guard to `presign` in `src/frost/mod.rs`, immediately after the upper-bound check, mirroring the pattern used in `assert_key_invariants` and `validate_triple_inputs`:

```rust
// src/frost/mod.rs — after the ThresholdTooLarge check
if args.threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: args.threshold.into(),
        min: 2,
    });
}
```

This aligns FROST presign with the invariant enforced uniformly across DKG, reshare, refresh, and OT-based triple generation. [5](#0-4) 

---

### Proof of Concept

```rust
use threshold_signatures::frost::presign;
use threshold_signatures::participants::Participant;

// Key was generated honestly with threshold=2
let participants = vec![Participant::from(1u32), Participant::from(2u32)];
let me = participants[0];

// Malicious coordinator supplies threshold=1 — accepted without error
let args = PresignArguments {
    threshold: ReconstructionLowerBound::from(1usize), // bypasses minimum check
    keygen_out: honest_keygen_output,                  // share from legitimate keygen
};

// presign succeeds; threshold=1 is never rejected
let protocol = presign(&participants, me, &args, rng).unwrap();

// In the subsequent sign phase, Lagrange interpolation with threshold=1
// requires only participant[0]'s partial signature to produce a valid
// aggregate signature — no cooperation from participant[1] needed.
```

The root cause is the absent lower-bound check at `src/frost/mod.rs:71–77`, which allows `threshold = 1` to flow into `do_presign` and ultimately into the FROST signing aggregation, reducing the effective quorum to one. [6](#0-5)

pub fn presign<C>(
    participants: &[Participant],
    me: Participant,
    args: &PresignArguments<C>,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = PresignOutput<C>>, InitializationError>
where
    C: Ciphersuite + Send,
    <<<C as frost_core::Ciphersuite>::Group as Group>::Field as Field>::Scalar: Send,
    <<C as frost_core::Ciphersuite>::Group as frost_core::Group>::Element: std::marker::Send,
{
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if args.threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: args.threshold.into(),
            max: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = do_presign(
        ctx.shared_channel(),
        participants,
        me,
        args.keygen_out.private_share,
        rng,
    );
    Ok(make_protocol(ctx, fut))
}
