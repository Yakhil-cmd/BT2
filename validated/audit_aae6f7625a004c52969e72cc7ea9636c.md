### Title
Missing Caller Restriction on `update_rewards` Allows Anyone to Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `StakingRewardsManagerImpl` is callable by any address, despite the specification explicitly requiring "Only starkware sequencer." An unprivileged attacker can call it with `disable_rewards: true` before the sequencer does, consuming the block's reward slot without distributing rewards, thereby permanently freezing unclaimed yield for stakers.

### Finding Description
The `IStakingRewardsManager::update_rewards` function is the consensus-era reward distribution entry point. The specification at `docs/spec.md` line 1645 states its access control is **"Only starkware sequencer."** However, the implementation contains no such check.

The function writes `last_reward_block` to the current block number unconditionally ŌĆö before the `disable_rewards` branch ŌĆö meaning any caller can consume the block's reward slot without distributing rewards:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker validity checks ...

    // last_reward_block written BEFORE disable_rewards check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;  // returns without distributing rewards
    }
    // ... actual reward distribution ...

The spec mandates the restriction: [2](#0-1) 

The tests confirm no caller restriction is enforced ŌĆö `update_rewards` is called without any `cheat_caller_address` setup: [3](#0-2) 

### Impact Explanation
An attacker calls `update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)` in every block before the sequencer. This:
1. Sets `last_reward_block` to the current block number.
2. Returns immediately without distributing rewards.
3. Causes the sequencer's subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

Sustained across blocks, this permanently freezes unclaimed yield for all stakers whose reward distribution depends on the sequencer calling this function. This matches **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation
Any unprivileged address can call `update_rewards` with a valid staker address (all staker addresses are public on-chain). The attacker needs no funds, no special role, and no privileged access. The only cost is gas per block. Likelihood is **Medium** ŌĆö it requires sustained effort but is trivially executable.

### Recommendation
Add a caller restriction to `update_rewards` matching the specification. Introduce an `assert_caller_is_sequencer` check analogous to the existing `assert_caller_is_attestation_contract` pattern used in `update_rewards_from_attestation_contract`: [4](#0-3) 

The sequencer address should be stored in contract state during initialization and validated at the top of `update_rewards`.

### Proof of Concept
1. Deploy the staking system with a staker who has active stake and consensus rewards enabled.
2. At the start of any block, before the sequencer acts, call:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(
       staker_address: <valid_staker>,
       disable_rewards: true
   )
   ```
3. Observe that `last_reward_block` is now set to the current block.
4. The sequencer's call to `update_rewards` for any staker in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat every block ŌĆö no rewards are ever distributed; `unclaimed_rewards_own` for all stakers remains frozen at zero. [5](#0-4) [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
