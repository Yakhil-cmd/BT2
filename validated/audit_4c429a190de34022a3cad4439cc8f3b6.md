### Title
`pause_contract()` Called While `ExitToNear` Callback Is Pending Permanently Freezes User ERC-20/ETH Funds - (`engine/src/contract_methods/connector.rs`)

### Summary

The `pause_contract()` admin function can be called at any time with no guard against in-flight async promise flows. When called between the EVM execution of the `ExitToNear` precompile (which burns user tokens) and the execution of the `exit_to_near_precompile_callback` receipt, the callback unconditionally fails the `require_running()` check before it can reach the refund path. Because NEAR callbacks are one-shot receipts, the refund opportunity is permanently lost and the user's already-burned ERC-20 tokens (or ETH) are frozen forever.

### Finding Description

The `ExitToNear` precompile executes a multi-step async flow across NEAR receipts:

**Step 1 — EVM execution (Receipt A):** The user calls an EVM contract that triggers the `ExitToNear` precompile. Inside the precompile's `run()` method, ERC-20 tokens are burned (or ETH is debited from the user's EVM balance). A NEAR promise is scheduled to call `ft_transfer` / `near_withdraw` on the NEP-141 contract, with `exit_to_near_precompile_callback` attached as a callback. [1](#0-0) 

**Step 2 — NEP-141 transfer (Receipt B):** The `ft_transfer` or `near_withdraw` call executes on the NEP-141 contract. This can fail (e.g., recipient account not registered).

**Step 3 — Callback (Receipt C):** `exit_to_near_precompile_callback` is called. Its first substantive action is `require_running(&state)?` at line 203, which returns `Err(ERR_PAUSED)` if the contract is paused. This error propagates through `sdk_unwrap()` as a panic, reverting all state changes of Receipt C. The refund branch at line 231–239 is never reached. [2](#0-1) 

The `pause_contract()` function has no guard preventing it from being called while such callbacks are pending: [3](#0-2) 

`require_running()` is a hard gate that returns an error for any paused state: [4](#0-3) 

### Impact Explanation

**Critical — Permanent freezing of funds.**

When the callback panics due to `ERR_PAUSED`:
- The ERC-20 token burn (or ETH debit) from Receipt A is **not** reverted — it occurred in a separate, already-finalized receipt.
- The refund logic (`engine::refund_on_error`) that would re-mint the burned ERC-20 tokens or restore ETH is never executed.
- NEAR receipts are one-shot. Once Receipt C is processed and panics, it is gone. Calling `resume_contract()` afterward does not replay the callback.
- The user's funds are permanently destroyed with no recovery path. [5](#0-4) 

### Likelihood Explanation

The owner has legitimate reasons to call `pause_contract()` at any time (e.g., emergency response to a separate security incident). The window between Receipt A and Receipt C is typically one to two NEAR blocks. If the owner issues a pause during this window — even in good faith — and the NEP-141 transfer in Receipt B fails (e.g., recipient not registered, NEP-141 contract paused, insufficient storage deposit), the user's funds are permanently frozen. The `test_exit_to_near_refund` test confirms that the NEP-141 transfer failure path is a realistic, tested scenario. [6](#0-5) 

### Recommendation

Move the `require_running()` check **after** the refund-necessity check, or exempt the refund branch from the pause guard entirely. The refund path restores user funds and does not advance any protocol state that the pause is meant to protect:

```rust
pub fn exit_to_near_precompile_callback<...>(...) -> Result<...> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        env.assert_private_call()?;
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }
        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            // Only block the success path (NEAR transfer) when paused
            require_running(&state)?;
            // ... existing transfer logic ...
            None
        } else if let Some(args) = args.refund {
            // Refund path: allow even when paused to prevent permanent fund loss
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }
            Some(refund_result)
        } else {
            None
        };
        Ok(maybe_result)
    })
}
```

### Proof of Concept

1. User holds ERC-20 tokens (bridged NEP-141) on Aurora.
2. User calls `withdrawToNear("unregistered.near", amount)` on the ERC-20 contract, triggering the `ExitToNear` precompile. ERC-20 tokens are burned in the EVM. Receipt A finalizes.
3. Owner calls `pause_contract()` (legitimate emergency action). Receipt P finalizes.
4. The `ft_transfer` call to the NEP-141 contract executes and fails because `"unregistered.near"` has no storage deposit. Receipt B finalizes with failure.
5. `exit_to_near_precompile_callback` executes as Receipt C. `require_running()` reads `is_paused = true` and returns `Err(ERR_PAUSED)`. The function propagates the error through `sdk_unwrap()`, panicking. Receipt C is reverted.
6. The refund (`engine::refund_on_error` → ERC-20 re-mint) never executes.
7. Owner calls `resume_contract()`. Receipt C is already gone — it will never be retried.
8. User's ERC-20 tokens are permanently burned with no on-chain recovery path. [7](#0-6) [3](#0-2) [8](#0-7)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

#[named]
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}

pub fn refund_on_error<I: IO + Copy, E: Env, P: PromiseHandler>(
    io: I,
    env: &E,
    state: EngineState,
    args: &RefundCallArgs,
    handler: &mut P,
) -> EngineResult<SubmitResult> {
    let current_account_id = env.current_account_id();
    if let Some(erc20_address) = args.erc20_address {
        // ERC-20 exit; re-mint burned tokens
        let erc20_admin_address = current_address(&current_account_id);
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, erc20_admin_address, current_account_id, io, env);

        let refund_address = args.recipient_address;
        let amount = U256::from_big_endian(&args.amount);
        let input = setup_refund_on_error_input(amount, refund_address);

        engine.call(
            &erc20_admin_address,
            &erc20_address,
            Wei::zero(),
            input,
            u64::MAX,
            Vec::new(),
            Vec::new(),
            handler,
        )
    } else {
        // ETH exit; transfer ETH back from precompile address
        let exit_address = exit_to_near::ADDRESS;
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, exit_address, current_account_id, io, env);
        let refund_address = args.recipient_address;
        let amount = Wei::new(U256::from_big_endian(&args.amount));
        engine.call(
            &exit_address,
            &refund_address,
            amount,
            Vec::new(),
            u64::MAX,
            vec![
                (exit_address.raw(), Vec::new()),
                (refund_address.raw(), Vec::new()),
            ],
            Vec::new(),
            handler,
        )
    }
