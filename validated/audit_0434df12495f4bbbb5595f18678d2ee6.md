### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Suppress Per-Block Yield — (`File: src/staking/staking.cairo`)

### Summary
`IStakingRewardsManager::update_rewards` is callable by any address despite the specification mandating "Only starkware sequencer." An unprivileged caller can invoke it with `disable_rewards: true` before the sequencer does, consuming the one-call-per-block slot and permanently discarding that block's yield for the targeted staker and all their delegators.

### Finding Description
The spec at `docs/spec.md` lines 1644–1645 states:

```
#### access control
Only starkware sequencer.
```

The implementation at `src/staking/staking.cairo` lines 1449–1507 contains no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                            // exits without distributing rewards
    }
    ...
```

The guard `current_block_number > self.last_reward_block.read()` enforces that the function executes **at most once per block**. When an attacker calls it with `disable_rewards: true`, `last_reward_block` is updated to the current block and the function returns immediately without distributing rewards. The sequencer's subsequent call for the same block reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

Because block rewards are computed on-the-fly from the minting curve and are never accumulated anywhere for later distribution, the yield for that block is **permanently lost** — it is never credited to `unclaimed_rewards_own` or transferred to pools. [4](#0-3) 

### Impact Explanation
Every block in which the attacker front-runs the sequencer, the targeted staker and all their pool delegators receive zero rewards. The lost yield is irrecoverable because `_update_rewards` is the only code path that increments `staker_info.unclaimed_rewards_own` and transfers pool rewards. [5](#0-4) 

This matches **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The entry point is a public, permissionless function. The attacker needs only to submit a transaction with `disable_rewards: true` targeting any active staker's address at the start of each block. No special role, key, or capital is required. The cost is only L2 gas. A motivated griefer can sustain this indefinitely.

### Recommendation
Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the configured sequencer address (or a dedicated rewards-manager role), mirroring the pattern already used in `update_rewards_from_attestation_contract`:

```cairo
self.assert_caller_is_attestation_contract(); // analogous pattern
```

A dedicated `assert_caller_is_sequencer()` guard should be introduced and called as the first check inside `update_rewards`. [6](#0-5) 

### Proof of Concept

1. Staker `S` is active and consensus rewards are live.
2. Block `N` begins. The sequencer prepares to call `update_rewards(S, false)`.
3. Attacker `A` (any address) calls `update_rewards(S, true)` first.
   - `current_block_number (N) > last_reward_block` → passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → function returns; no rewards distributed.
4. Sequencer calls `update_rewards(S, false)`.
   - `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker `S` and all pool members receive **zero** rewards for block `N`.
6. Repeat every block to continuously suppress all yield. [7](#0-6) [8](#0-7)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
