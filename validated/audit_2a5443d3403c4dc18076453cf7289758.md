### Title
Bank Precompile `transfer` Missing Sender Block-List Check Allows Draining Blocked Address Native Tokens — (`x/cronos/keeper/precompiles/bank.go`)

### Summary

The `TransferMethodName` handler in the bank precompile checks the block-list only for the **recipient** (`to`), never for the **sender** (`from`). Because the `sender` argument is fully attacker-controlled ABI input and the Cronos block-list is enforced only at the mempool/proposal layer (tx signers and EVM tx destinations), a malicious contract can call `transfer(blocked_addr, attacker, amount)` and drain a blocked address's `evm/<contract>` native tokens with no obstruction.

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` case:

```go
// lines 175-184
sender := args[0].(common.Address)   // fully attacker-controlled
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
if err := bc.checkBlockedAddr(to); err != nil {   // ← only recipient checked
    return nil, err
}
// no check on `from`
``` [1](#0-0) 

`checkBlockedAddr` itself only consults `bc.bankKeeper.BlockedAddr(addr)`, which is the Cosmos SDK's **built-in module-account block-list**, not the Cronos operator block-list populated by `MsgStoreBlockList`:

```go
func (bc *BankContract) checkBlockedAddr(addr sdk.AccAddress) error {
    ...
    if bc.bankKeeper.BlockedAddr(to) {   // SDK module accounts only
        return errorsmod.Wrapf(...)
    }
    return nil
}
``` [2](#0-1) 

The Cronos block-list is enforced exclusively at two points:

1. `BlockAddressesDecorator.AnteHandle` — checks **tx signers** and **EVM tx `To` field** at CheckTx.
2. `ProposalHandler.ValidateTransaction` — same checks at proposal time. [3](#0-2) [4](#0-3) 

Neither layer inspects ABI arguments passed to a precompile call. A transaction signed by the attacker (not the blocked address) whose EVM `To` field is the malicious contract (not the blocked address) passes all block-list gates cleanly, then executes `SendCoins(ctx, blocked_addr, attacker, evm/<contract> tokens)` inside the precompile with no further check. [5](#0-4) 

### Impact Explanation

A blocked address's `evm/<contract>` native token balance is fully drained by a third-party contract without the blocked address signing anything. This directly violates the block-list invariant that blocked addresses must not be able to move funds out of their account. Impact category: **High — bypass of Cronos block-list authorization**.

### Likelihood Explanation

The precondition (blocked address holds `evm/<contract>` tokens) is realistic: tokens can be minted to the address before it is blocked, or the contract owner can mint them after the fact (the `mint` path only checks the recipient is not a module account). The attacker is the contract deployer — an unprivileged role. No leaked keys or admin compromise is required.

### Recommendation

Add a sender-side block-list check inside the `TransferMethodName` handler. Because `checkBlockedAddr` only covers SDK module accounts, a separate check against the Cronos operator block-list (the same map used by `BlockAddressesDecorator`) must be threaded into the precompile, or the precompile must be given access to the proposal handler's block-list and reject calls where `from` appears in it.

### Proof of Concept

1. Deploy contract `M` (any contract that calls the bank precompile).
2. Contract `M` calls `mint(blocked_addr, 1000)` → blocked address now holds 1000 `evm/M` tokens.
3. Admin submits `MsgStoreBlockList` adding `blocked_addr`.
4. Attacker (tx signer = attacker, EVM `To` = contract `M`) calls `M.transfer(blocked_addr, attacker, 1000)`.
5. The ante handler and proposal handler pass (signer = attacker ∉ block-list; EVM `To` = `M` ∉ block-list).
6. Inside the precompile: `checkBlockedAddr(attacker)` passes; `SendCoins(ctx, blocked_addr, attacker, 1000 evm/M)` executes.
7. Assert: `blocked_addr` balance of `evm/M` = 0; attacker balance = 1000. Block-list bypassed.

func (bad BlockAddressesDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	if ctx.IsCheckTx() {
		if sigTx, ok := tx.(signing.SigVerifiableTx); ok {
			signers, err := sigTx.GetSigners()
			if err != nil {
				return ctx, err
			}
			for _, signer := range signers {
				if _, ok := bad.blockedMap[sdk.AccAddress(signer).String()]; ok {
					return ctx, fmt.Errorf("signer is blocked: %s", sdk.AccAddress(signer).String())
				}
			}
		}

		for _, msg := range tx.GetMsgs() {
			msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
			if ok {
				ethTx := msgEthTx.AsTransaction()
				// check the destination address
				if ethTx.To() != nil {
					if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
						return ctx, fmt.Errorf("destination address is blocked: %s", sdk.AccAddress(ethTx.To().Bytes()).String())
					}
				}
				// check EIP-7702 authorisation list
				if ethTx.SetCodeAuthorizations() != nil {
					for _, auth := range ethTx.SetCodeAuthorizations() {
						addr, err := auth.Authority()
						if err == nil {
							if _, ok := bad.blockedMap[sdk.AccAddress(addr.Bytes()).String()]; ok {
								return ctx, fmt.Errorf("signer is blocked: %s", addr.String())
							}
						}
						// check the target address
						if _, ok := bad.blockedMap[sdk.AccAddress(auth.Address.Bytes()).String()]; ok {
							return ctx, fmt.Errorf("authorisation address is blocked: %s", sdk.AccAddress(auth.Address.Bytes()).String())
						}
					}
				}
			}
		}

		admin := bad.getParams(ctx).CronosAdmin
		for _, msg := range tx.GetMsgs() {
			if blocklistMsg, ok := msg.(*types.MsgStoreBlockList); ok {
				if admin != blocklistMsg.From {
					return ctx, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
				}
			}
		}
	}
	return next(ctx, tx, simulate)

func (h *ProposalHandler) ValidateTransaction(tx sdk.Tx, txBz []byte) error {
	if len(h.blocklist) == 0 {
		// fast path, accept all txs
		return nil
	}

	var err error
	if tx == nil {
		tx, err = h.TxDecoder(txBz)
		if err != nil {
			return err
		}
	}

	sigTx, ok := tx.(signing.SigVerifiableTx)
	if !ok {
		return fmt.Errorf("tx of type %T does not implement SigVerifiableTx", tx)
	}

	signers, err := sigTx.GetSigners()
	if err != nil {
		return err
	}
	for _, signer := range signers {
		encoded, err := h.addressCodec.BytesToString(signer)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", signer, err)
		}
		if _, ok := h.blocklist[encoded]; ok {
			return fmt.Errorf("signer is blocked: %s", encoded)
		}
	}

	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if ok {
			ethTx := msgEthTx.AsTransaction()
			// check the destination address
			if ethTx.To() != nil {
				encoded, err := h.addressCodec.BytesToString(ethTx.To().Bytes())
				if err != nil {
					return fmt.Errorf("invalid bech32 address: %s, err: %w", ethTx.To(), err)
				}
				if _, ok := h.blocklist[encoded]; ok {
					return fmt.Errorf("destination address is blocked: %s", encoded)
				}
			}
			// check EIP-7702 authorisation list
			if ethTx.SetCodeAuthorizations() != nil {
				for _, auth := range ethTx.SetCodeAuthorizations() {
					addr, err := auth.Authority()

							return fmt.Errorf("signer is blocked: %s", encoded)
						}
					}
					// check the target address

						return fmt.Errorf("invalid bech32 address: %s, err: %w", auth.Address, err)

					if _, ok := h.blocklist[encoded]; ok {
						return fmt.Errorf("authorisation address is blocked: %s", encoded)
					}
				}
