### Title
Unpermissioned `update_rewards` with Attacker-Controlled `disable_rewards` Enables Permanent Griefing of All Consensus Rewards — (`src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts an attacker-controlled `disable_rewards: bool` flag. Because the global `last_reward_block` is written unconditionally before the flag is checked, any unprivileged caller can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to consume the per-block reward slot without distributing any rewards, permanently blocking all stakers from receiving consensus block rewards for as long as the attack is sustained.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` in `src/staking/staking.cairo` has no access-control guard beyond `general_prerequisites()`, which only checks that the caller is non-zero and the contract is unpaused. [1](#0-0) 

The function writes `last_reward_block` to the current block number **before** branching on `disable_rewards`: [2](#0-1) 

The guard that prevents a second call in the same block is: [3](#0-2) 

`last_reward_block` is a single global storage slot, not per-staker: [4](#0-3) 

Because the slot is consumed before the `disable_rewards` branch is evaluated, any caller who passes `disable_rewards: true` silently discards the block's rewards for **every** staker and makes the legitimate block-proposer call revert with `REWARDS_ALREADY_UPDATED` for that block.

The interface definition confirms the function is fully public: [5](#0-4) 

The only prerequisite the attacker must satisfy is supplying a valid, active staker address with non-zero balance. All staker addresses are publicly enumerable from the `stakers` vector and from `NewStaker` events: [6](#0-5) 

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block:

1. Writes `last_reward_block = current_block`.
2. Returns early — zero rewards are credited to any staker or pool.
3. Causes every legitimate `update_rewards` call in the same block to revert.

Sustained over time this constitutes **permanent freezing of unclaimed yield** for all stakers and their delegators. No funds are stolen, but all consensus-era block rewards are permanently withheld as long as the attacker continues the attack. This matches the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties; Temporary freezing of funds."*

---

### Likelihood Explanation

- Entry point is a public, permissionless external function — no role, no signature, no privileged key required.
- The only input constraint is a valid staker address, which is trivially obtained from on-chain events.
- Gas cost per block on Starknet L2 is low; a motivated attacker (e.g., a competing validator or someone shorting the protocol) can sustain the attack indefinitely.
- No front-running skill is required; the attacker simply calls the function at the start of each block.

---

### Recommendation

Add an access-control check so that only the designated consensus/block-proposer role (or the staker's own operational address) may call `update_rewards`. Alternatively, move the `last_reward_block.write` to **after** the `disable_rewards` branch so that a call with `disable_rewards: true` does not consume the per-block slot.

```cairo
// Short-term fix: gate the function
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.roles.only_app_governor(); // or a dedicated REWARDS_MANAGER role
    ...
}
```

### Proof of Concept

```cairo
// Attacker contract — called once per block
fn grief_rewards(staking: IStakingRewardsManagerDispatcher, valid_staker: ContractAddress) {
    // disable_rewards = true → last_reward_block is written, no rewards distributed
    staking.update_rewards(staker_address: valid_staker, disable_rewards: true);
    // Legitimate block-proposer call now reverts with REWARDS_ALREADY_UPDATED
}
```

Step-by-step:
1. Attacker identifies any active staker address from `NewStaker` events.
2. Each block, attacker calls `Staking::update_rewards(valid_staker, true)`.
3. `last_reward_block` is set to the current block; no rewards are minted or credited.
4. The consensus layer's legitimate `update_rewards(staker, false)` call reverts because `current_block_number > last_reward_block` is now false.
5. All stakers and pool delegators receive zero block rewards for that block.
6. Repeated every block, this permanently freezes all consensus reward accrual.
