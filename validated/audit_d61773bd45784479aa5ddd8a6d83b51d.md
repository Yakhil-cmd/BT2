### Title
Unbounded Loop in `calculate_rewards` Enables Permanent Freezing of Unclaimed Yield via Trace Bloat — (`src/pool/pool.cairo`)

---

### Summary

The `calculate_rewards` internal function in the Pool contract iterates over a pool member's entire `pool_member_epoch_balance` trace in an unbounded loop. A delegator who repeatedly changes their balance across many epochs without claiming rewards will grow this trace without bound. Once the trace is large enough, every call to `claim_rewards` will exceed the Starknet transaction gas limit and revert, permanently freezing the delegator's unclaimed yield with no recovery path.

---

### Finding Description

`calculate_rewards` in `src/pool/pool.cairo` contains the following loop, which the code itself acknowledges is unbounded:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    rewards += compute_rewards_rounded_down(...);
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The loop iterates from the stored `entry_to_claim_from` index up to the full length of the `pool_member_epoch_balance` trace. Each iteration performs a storage read via `pool_member_trace.at(entry_to_claim_from)` and a call to `find_sigma`, which itself performs multiple storage reads from `cumulative_rewards_trace`. [2](#0-1) 

The trace grows by one checkpoint per epoch in which the pool member changes their balance. The `insert` function in `PoolMemberBalanceTrace` deduplicates same-epoch writes (updating the last entry rather than appending), so the maximum growth rate is one new checkpoint per epoch. [3](#0-2) 

The `entry_to_claim_from` pointer is only advanced on a **successful** `claim_rewards` call:

```cairo
pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
pool_member_info.reward_checkpoint = until_checkpoint;
self.write_pool_member_info(:pool_member, :pool_member_info);
``` [4](#0-3) 

If `claim_rewards` reverts (due to gas exhaustion), `entry_to_claim_from` is never updated. The next call will attempt to iterate the same (or larger) trace from the same starting index, and will also revert. There is no partial-claim mechanism and no trace pruning function in `PoolMemberBalanceTrace`. [5](#0-4) 

---

### Impact Explanation

Once the `pool_member_epoch_balance` trace grows large enough to cause `calculate_rewards` to exceed the Starknet per-transaction gas limit, the delegator's `claim_rewards` call will **always revert**. Because:

1. `entry_to_claim_from` is only updated on success, it never advances past the bloated region.
2. There is no function to prune or compact the trace.
3. There is no partial-claim interface that accepts an epoch range.

The delegator's accumulated unclaimed STRK yield is permanently frozen in the contract with no recovery path. This matches the allowed impact: **Permanent freezing of unclaimed yield (High)**.

---

### Likelihood Explanation

The trace grows at most one entry per epoch. The attacker (the delegator themselves, or their reward address) must call `add_to_delegation_pool` or `exit_delegation_pool_intent` in distinct epochs without ever calling `claim_rewards`. The exact number of iterations required to exceed the gas limit depends on Starknet's current gas parameters, but given that each iteration performs multiple storage reads (including `find_sigma` which reads from `cumulative_rewards_trace`), the threshold is realistically reachable over months of active delegation without claiming.

The entry path requires no privileged role: `add_to_delegation_pool` is callable by the pool member or their reward address. [6](#0-5) 

Likelihood: **Medium** — requires sustained activity over many epochs, but is fully within the control of an unprivileged delegator and is a realistic pattern for active participants who defer reward claims.

---

### Recommendation

1. **Add a loop iteration cap** in `calculate_rewards`: process at most `MAX_ITERATIONS` entries per call and update `entry_to_claim_from` to the stopping point, allowing subsequent calls to continue from where the previous one left off.
2. **Alternatively**, expose a `claim_rewards_until_epoch(until_epoch: Epoch)` variant that allows partial claiming up to a caller-specified epoch, so the delegator can drain the trace incrementally.
3. **Document the risk** more prominently and consider adding an on-chain guard that warns or prevents balance changes when the trace length exceeds a safety threshold.

---

### Proof of Concept

```
1. Delegator calls enter_delegation_pool(reward_address, amount) to join a pool.
2. For N consecutive epochs (without calling claim_rewards):
   a. Advance to the next epoch.
   b. Call add_to_delegation_pool(pool_member, small_amount).
      → set_member_balance → trace.insert(key: current_epoch + K, ...)
      → Because the key differs from the previous entry, a new checkpoint is appended.
      → pool_member_epoch_balance trace length increases by 1.
3. After N epochs, call claim_rewards(pool_member).
   → calculate_rewards iterates from entry_to_claim_from=0 to pool_member_trace_length=N.
   → Each iteration calls find_sigma (multiple storage reads) + compute_rewards_rounded_down.
   → When N is large enough, the transaction exceeds the Starknet gas limit and reverts.
4. entry_to_claim_from remains 0. Every subsequent claim_rewards call also reverts.
   → Unclaimed yield is permanently frozen.
```

pub impl PoolMemberBalanceTraceImpl of PoolMemberBalanceTraceTrait {
    /// Retrieves the most recent checkpoint from the trace structure.
    ///
    /// # Returns
    /// A tuple containing:
    /// - `Epoch`: Timestamp/key of the last checkpoint
    /// - `PoolMemberBalance`: Value stored in the last checkpoint
    ///
    /// # Panics
    /// If the trace structure is empty (no checkpoints exist)
    ///
    /// # Note
    /// This will return the last inserted checkpoint that maintains the structure's
    /// invariant of non-decreasing keys.
    fn last(self: StoragePath<PoolMemberBalanceTrace>) -> (Epoch, PoolMemberBalance) {
        self._nth_back(0)
    }

    /// Retrieves the second last checkpoint from the trace structure.
    fn second_last(self: StoragePath<PoolMemberBalanceTrace>) -> (Epoch, PoolMemberBalance) {
        self._nth_back(1)
    }

    /// Retrieves the third last checkpoint from the trace structure.
    fn third_last(self: StoragePath<PoolMemberBalanceTrace>) -> (Epoch, PoolMemberBalance) {
        self._nth_back(2)
    }

    /// Returns the total number of checkpoints.
    fn length(self: StoragePath<PoolMemberBalanceTrace>) -> u64 {
        self.checkpoints.len()
    }

    /// Returns whether the trace is non empty.
    fn is_non_empty(self: StoragePath<PoolMemberBalanceTrace>) -> bool {
        self.checkpoints.len().is_non_zero()
    }

    /// Returns the checkpoint at the given position.
    ///
    /// # Panics
    /// If the position is out of bounds.
    fn at(self: StoragePath<PoolMemberBalanceTrace>, pos: VecIndex) -> PoolMemberCheckpoint {
        let checkpoints = self.checkpoints;
        let len = checkpoints.len();
        assert!(pos < len, "{}", TraceErrors::INDEX_OUT_OF_BOUNDS);
        let checkpoint = checkpoints[pos].read();

            epoch: checkpoint.key,
            balance: checkpoint.value.balance,
            cumulative_rewards_trace_idx: checkpoint.value.cumulative_rewards_trace_idx,
        )

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
