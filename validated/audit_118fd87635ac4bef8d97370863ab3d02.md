### Title
Unprivileged caller can permanently suppress per-block consensus rewards by calling `update_rewards` with `disable_rewards = true` — (File: `src/staking/staking.cairo`)

---

### Summary

`Staking::update_rewards` has no caller-authorization check. Any address can invoke it with `disable_rewards = true`, which writes the current block number into the global `last_reward_block` slot **before** the early-return guard is evaluated. Because the function enforces strict `current_block_number > last_reward_block`, the legitimate reward-distribution call for that block is permanently blocked. An attacker can repeat this once per block, silently zeroing out all consensus-era staker rewards indefinitely.

---

### Finding Description

`update_rewards` is the V3 consensus-rewards entry point. Its only gate is `general_prerequisites()`, which checks the contract is not paused and the caller is not the zero address — no role restriction exists.

```cairo
// src/staking/staking.cairo  line 1449-1507
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // ← only pause + non-zero check
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← slot consumed HERE

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits with no rewards paid
    }
    ...
    self._update_rewards(...);                             // ← never reached
}
```

The critical ordering flaw: `last_reward_block` is written **before** the `disable_rewards` branch is evaluated. Once written, the assertion `current_block_number > last_reward_block` will fail for every subsequent call in the same block, regardless of who calls it or what parameters they use.

Contrast this with `update_rewards_from_attestation_contract` (the V2 path), which correctly restricts its caller:

```cairo
// src/staking/staking.cairo  line 1394-1401
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), "{}", Error::CONSENSUS_REWARDS_IS_ACTIVE);
    self.assert_caller_is_attestation_contract();   // ← access control present
    ...
}
```

The V3 path (`update_rewards`) has no equivalent guard.

The analog to the external report is direct:

| External report | This codebase |
|---|---|
| Checks "is PT mature" (`mscale > 0`) | Checks "is this a new block" (`block > last_reward_block`) |
| Does **not** check "is redeem restricted" | Does **not** check "is caller authorized to disable rewards" |
| Always tries to redeem → fails when restricted | Always consumes the block slot → rewards lost when `disable_rewards = true` |

---

### Impact Explanation

`last_reward_block` is a **single global slot** shared across all stakers. One call per block is all that is permitted. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at every block:

- Consumes the per-block reward slot with zero payout.
- Permanently destroys the STRK rewards that would have accrued to every staker and every delegation pool for that block.
- Causes **permanent loss of unclaimed yield** for all protocol participants.

This maps to the allowed impact: **High — Theft of unclaimed yield / Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is `#[abi(embed_v0)]` — publicly callable by any EOA or contract.
- The attack costs one transaction per block (cheap on Starknet).
- No special knowledge, role, or capital is required.
- The attacker gains nothing financially, making this a pure griefing attack, but the damage to stakers is real and permanent.

---

### Recommendation

Two complementary fixes:

1. **Add caller authorization** — restrict `update_rewards` to the attestation contract (or a designated rewards-manager role), mirroring the guard already present in `update_rewards_from_attestation_contract`:

```cairo
self.assert_caller_is_attestation_contract(); // or a dedicated role
```

2. **Move the `last_reward_block` write after the `disable_rewards` check** — so that a call with `disable_rewards = true` does not consume the block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // moved here
self._update_rewards(...);
```

Both fixes together eliminate the attack surface entirely.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. At block N, attacker calls `staking.update_rewards(any_valid_staker, disable_rewards: true)`.
3. `last_reward_block` is written to N; function returns with no rewards distributed.
4. The legitimate caller (e.g., attestation contract) calls `update_rewards(real_staker, false)` at block N → reverts with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. All stakers lose their block-N rewards permanently.
6. Attacker repeats step 2 at every subsequent block, zeroing all consensus-era rewards indefinitely.

```text
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
```
