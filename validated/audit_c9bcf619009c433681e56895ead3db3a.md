Audit Report

## Title
Missing Per-Proposal Swap Canister Initiator Validation in `settle_neurons_fund_participation` - (File: rs/nns/governance/src/governance.rs)

## Summary

`settle_neurons_fund_participation` in NNS Governance validates only that the caller is *some* valid SNS Swap canister in the global SNS-W registry, not that it is the *specific* Swap canister associated with the `nns_proposal_id` supplied in the request. The code itself acknowledges this gap with the comment "Note that a Swap could settle each other's participation." Any legitimately deployed SNS Swap canister can therefore trigger Neurons' Fund settlement — including ICP minting — for a completely different SNS's proposal, directing minted ICP to an attacker-controlled address.

## Finding Description

The authorization check in `settle_neurons_fund_participation` calls `is_canister_id_valid_swap_canister_id`, which queries SNS-W's `list_deployed_snses` and checks only whether the caller appears anywhere in the global list of deployed SNS Swap canisters:

```rust
let is_swap = list_deployed_snses_response
    .instances
    .iter()
    .any(|sns| sns.swap_canister_id == Some(target_canister_id.into()));
``` [1](#0-0) 

There is no subsequent check that the caller's canister ID matches the Swap canister recorded for the specific `nns_proposal_id` in the request. SNS-W already exposes a `get_deployed_sns_by_proposal_id` method that maps a proposal ID to its specific `DeployedSns` record (including `swap_canister_id` and `governance_canister_id`), but this method is never called from `settle_neurons_fund_participation`. [2](#0-1) 

The `Committed` variant of the request carries a caller-supplied `sns_governance_canister_id` field that is passed directly to `mint_to_sns_governance` as the ICP mint destination:

```rust
let destination =
    AccountIdentifier::new(sns_governance_canister_id, /* subaccount = */ None);
let _ = self.ledger.transfer_funds(amount_icp_e8s, 0, None, destination, 0).await...
``` [3](#0-2) [4](#0-3) 

No cross-check against the proposal's own SNS governance canister ID is performed before minting. The `execute_create_service_nervous_system_proposal` function does not store the deployed swap canister ID in `ProposalData`, so the only way to perform the cross-check is via SNS-W's `get_deployed_sns_by_proposal_id`. [5](#0-4) 

The full exploit path:

1. Attacker controls SNS-A's Swap canister (legitimately deployed, passes `is_canister_id_valid_swap_canister_id`).
2. SNS-B has a live `CreateServiceNervousSystem` proposal (ID = `P_B`) with Neurons' Fund participation reserved; its swap is in `Committed` state but `finalize` has not been called yet.
3. Attacker's Swap canister calls `settle_neurons_fund_participation` with `nns_proposal_id = P_B`, `result = Committed { sns_governance_canister_id = <attacker_principal>, total_direct_participation_icp_e8s = <max_value>, ... }`.
4. `is_canister_id_valid_swap_canister_id` passes because SNS-A's Swap canister is in the global list.
5. No check verifies SNS-A's Swap canister is the swap canister for proposal `P_B`.
6. NNS Governance computes Neurons' Fund participation for `P_B`, mints ICP, and transfers it to `<attacker_principal>`.
7. SNS-B's proposal lifecycle is set to `Committed`; the idempotency check (lines 7166–7174) then causes SNS-B's own Swap canister's subsequent `finalize` call to return the cached attacker-supplied result, permanently blocking correct settlement. [6](#0-5) [7](#0-6) 

## Impact Explanation

The financial impact is bounded by the total Neurons' Fund maturity reserved for SNS-B's swap, which can be in the millions of ICP range for large SNS launches. The attack results in: (1) illegal minting of ICP to an attacker-controlled address, (2) permanent loss of Neurons' Fund maturity that should have been refunded to NNS neurons, and (3) permanent corruption of SNS-B's proposal lifecycle, preventing SNS-B's legitimate Swap canister from ever settling correctly. This matches the Critical impact category: **Theft, permanent loss, illegal minting, or protocol insolvency involving exorbitant ICP/Cycles or in-scope chain-key/ledger assets, especially over $1M.**

## Likelihood Explanation

The attacker must control a legitimately deployed SNS Swap canister. The IC ecosystem already has dozens of deployed SNS instances on mainnet, each with a Swap canister that satisfies the `is_canister_id_valid_swap_canister_id` check. Any operator of an existing SNS Swap canister can execute this attack against any other SNS whose Neurons' Fund settlement has not yet been finalized. The attack window is the period between SNS-B's swap reaching a terminal lifecycle state and SNS-B's own Swap canister calling `finalize`. The attack is repeatable against any unsettled SNS-B proposal and requires no victim mistakes or external dependencies.

## Recommendation

After confirming the caller is a valid SNS Swap canister, additionally call SNS-W's `get_deployed_sns_by_proposal_id` with `request.nns_proposal_id` and verify that the caller's canister ID matches the `swap_canister_id` of the returned `DeployedSns`. Additionally, validate the `sns_governance_canister_id` supplied in the `Committed` payload against the `governance_canister_id` returned by `get_deployed_sns_by_proposal_id`, rather than using it as-is as the ICP mint destination.

## Proof of Concept

A deterministic PocketIC integration test can prove this:

1. Deploy two SNS instances (SNS-A and SNS-B) via two `CreateServiceNervousSystem` proposals, both with Neurons' Fund participation enabled.
2. Advance SNS-B's swap to `Committed` state without calling `finalize`.
3. From SNS-A's Swap canister identity, call `settle_neurons_fund_participation` on NNS Governance with `nns_proposal_id = P_B` and `result = Committed { sns_governance_canister_id = <attacker_account>, total_direct_participation_icp_e8s = <max>, total_neurons_fund_participation_icp_e8s = <max> }`.
4. Assert that the ICP ledger shows a mint to `<attacker_account>` rather than SNS-B's governance canister.
5. Assert that SNS-B's proposal lifecycle is now `Committed`.
6. Assert that SNS-B's own Swap canister's subsequent `settle_neurons_fund_participation` call returns the cached (attacker-supplied) result without minting additional ICP to the correct treasury.
