### Title
Missing Access Control on `update_rewards` Allows Griefing via `disable_rewards` Flag — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no caller access control. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` to consume the global `last_reward_block` slot for the current block without distributing any rewards. Repeated every block, this permanently freezes all consensus-era yield for every staker and delegator.

---

### Finding Description

`update_rewards` is the consensus-rewards entry point (V3). Its logic is:

1. Assert `current_block_number > last_reward_block` — ensures one reward event per block.
2. **Write `last_reward_block = current_block_number`** — consumes the slot.
3. If `disable_rewards || is_pre_consensus()` → **return early, no rewards distributed**.
4. Otherwise calculate and distribute block rewards. [1](#0-0) 

The only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero: [2](#0-1) 

There is **no check** that the caller is the staker, the attestation contract, or any privileged role. The `disable_rewards` flag is therefore freely settable by any external account.

The critical ordering flaw — directly analogous to the reference report — is that `last_reward_block` is written **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

Once `last_reward_block` equals the current block, every subsequent call in that block fails with `REWARDS_ALREADY_UPDATED`. The slot is consumed but no yield is minted.

This mirrors the reference bug exactly: a state mutation (`last_reward_block` write) occurs before the intended computation (reward distribution), and an unprivileged actor can trigger that mutation with a flag that suppresses the computation, corrupting the expected outcome for the entire block.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, true)` in every block:

- Consumes the per-block reward slot for all stakers globally (the variable is shared).
- Distributes zero STRK to every staker and every delegation pool for that block.
- Blocks all subsequent legitimate `update_rewards` calls in the same block.

Because the attack requires only a public call with no special privilege, it can be automated and sustained indefinitely, permanently preventing consensus rewards from accruing to any staker or delegator.

---

### Likelihood Explanation

**High.** The attacker needs only:

1. A non-zero Starknet address (no stake, no role).
2. Enough gas to submit one transaction per block.
3. Transaction ordering priority over legitimate callers (achievable by submitting early in the block or via fee priority).

Starknet gas costs are low. The attack is fully automatable with a simple bot.

---

### Recommendation

Restrict `update_rewards` to callers that are authorized to trigger reward events (e.g., the attestation contract, or the staker themselves). A minimal fix:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    // Add: only the attestation contract (or staker) may call this.
    self.assert_caller_is_attestation_contract();
    ...
}
```

Alternatively, move the `last_reward_block.write(...)` call to **after** the `disable_rewards` guard, so that a suppressed call does not consume the block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;  // do NOT write last_reward_block
}
self.last_reward_block.write(current_block_number);
// ... distribute rewards
```

---

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(alice_staker, disable_rewards=true)
    → last_reward_block := N
    → returns early, zero rewards distributed

  Legitimate tx (same block, processed after):
    update_rewards(alice_staker, disable_rewards=false)
    → assert(N > N) FAILS → REWARDS_ALREADY_UPDATED

Block N+1: attacker repeats.
```

All stakers and their delegators receive zero consensus rewards for every block the attacker front-runs. The `unclaimed_rewards_own` field on every `InternalStakerInfoV1` and the `cumulative_rewards_trace` in every Pool contract remain frozen indefinitely. [4](#0-3) [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
