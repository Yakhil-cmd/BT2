### Title
Staker Can Front-Run Delegator's `enter_delegation_pool` / `add_to_delegation_pool` to Steal All Yield via Commission Hike — (`File: src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary

A staker holding an active `commission_commitment` with `max_commission = COMMISSION_DENOMINATOR` (100%) can atomically raise their commission to 100% in the same block as a delegator's delegation transaction, capturing all future yield from that delegator. Neither `enter_delegation_pool` nor `add_to_delegation_pool` accepts a maximum-commission guard, so the delegator has no on-chain protection against this race.

---

### Finding Description

The `set_commission_commitment` function allows a staker to publish a commitment with any `max_commission` up to `COMMISSION_DENOMINATOR` (10 000 = 100%): [1](#0-0) 

While the commitment is active, `update_commission` permits the staker to **increase** commission to any value ≤ `max_commission` — the only constraint is `commission != old_commission`: [2](#0-1) 

The developers themselves flagged this gap in a code comment immediately above `set_commission_commitment`: [3](#0-2) 

Commission changes take effect immediately (written to storage in the same transaction) and are read at reward-calculation time: [4](#0-3) [5](#0-4) 

Neither delegation entry-point accepts a `max_commission` parameter or performs any commission check: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

With commission at 100%, `split_rewards_with_commission` routes the entire pool reward to the staker as commission, leaving the delegator with zero yield: [8](#0-7) 

The delegator's principal is recoverable only after the mandatory `exit_wait_window` (~1 week). All yield accrued during that window at 100% commission is permanently transferred to the staker. This is **theft of unclaimed yield** — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

The attack requires only that the staker:
1. Call `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — publicly visible on-chain.
2. Monitor the mempool for any `enter_delegation_pool` or `add_to_delegation_pool` transaction targeting their pool.
3. Submit `set_commission(10000)` with higher gas to land before the delegator's transaction.

Starknet's sequencer ordering makes step 3 straightforward. No privileged access, leaked key, or external dependency is required. Any staker is a permissionless protocol participant who can execute this.

---

### Recommendation

Add a `max_commission` (slippage) parameter to both `enter_delegation_pool` and `add_to_delegation_pool`. Before transferring funds, assert that the pool's current commission does not exceed the caller-supplied ceiling, reverting if it does. This mirrors the `DepositStakeWithSlippage` / `WithdrawStakeWithSlippage` fix applied in the referenced report.

---

### Proof of Concept

```
// 1. Staker starts with commission = 500 (5%).
//    Staker calls:
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1);

// 2. Delegator submits (pending in mempool):
pool.enter_delegation_pool(reward_address: delegator_reward, amount: 1_000_000);

// 3. Staker front-runs with higher gas:
staking.set_commission(commission: 10000);  // raises to 100%

// 4. Delegator's tx lands — funds are locked, commission is now 100%.
//    All rewards for the next ~1 week accrue entirely to the staker.
//    Delegator receives 0 yield.

// 5. After exit_wait_window, delegator recovers principal but loses all yield.
//    Staker then lowers commission back:
staking.set_commission(commission: 499);  // must decrease without new commitment
```

The flow test at `src/flow_test/flows.cairo:895–930` already demonstrates that staking with `commission = 10000` (100%) results in zero tokens reaching the pool contract, confirming the yield-theft outcome. [9](#0-8)

        fn enter_delegation_pool(
            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.
            self
                .pool_member_info
                .write(pool_member, VInternalPoolMemberInfoTrait::new_latest(:reward_address));

            // Emit events.
            self
                .emit(
                    Events::NewPoolMember { pool_member, staker_address, reward_address, amount },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake: Zero::zero(), new_delegated_stake: amount,
                    },
                );
        }

        fn add_to_delegation_pool(
            ref self: ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Amount {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            assert!(
                caller_address == pool_member || caller_address == pool_member_info.reward_address,
                "{}",
                Error::CALLER_CANNOT_ADD_TO_POOL,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);

            // Transfer funds from the delegator to the staking contract.
            let token_dispatcher = self.token_dispatcher.read();
            let staker_address = self.staker_address.read();
            transfer_from_delegator(pool_member: caller_address, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            // Update the pool member's balance checkpoint.
            let old_delegated_stake = self.increase_member_balance(:pool_member, :amount);
            let new_delegated_stake = old_delegated_stake + amount;

            // Emit events.
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );

            new_delegated_stake
        }
