### Title
Missing `msg_hash != 0` Validation in OT-Based ECDSA `sign()` Allows Valid Threshold Signature for Zero Message Hash — (File: `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

The OT-based ECDSA `sign()` function accepts `msg_hash = 0` without rejection, while the Robust ECDSA `sign()` function explicitly rejects it. When `msg_hash = 0`, the signing equation degenerates to `s = r · k · x`, producing a mathematically valid ECDSA signature that passes the coordinator's own verification check. This allows any caller to obtain a valid threshold signature for an attacker-chosen zero-hash input, consuming a one-time presignature and partially exposing private key material.

---

### Finding Description

In `src/ecdsa/ot_based_ecdsa/sign.rs`, the public entry point `sign()` performs several input checks (participant count, self-presence, coordinator-presence, threshold size) but **never checks whether `msg_hash` is zero**:

```rust
// src/ecdsa/ot_based_ecdsa/sign.rs  lines 22–76
pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,          // ← no is_zero() guard anywhere below
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    ...
    // no msg_hash validation
    Ok(make_protocol(ctx, fut))
}
``` [1](#0-0) 

The per-participant share computation is:

```rust
// src/ecdsa/ot_based_ecdsa/sign.rs  lines 139–159
fn compute_signature_share(..., msg_hash: Scalar) -> Result<Scalar, ProtocolError> {
    let k_i    = lambda * presignature.k;
    let sigma_i = lambda * presignature.sigma;
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)   // when msg_hash=0 → r·σᵢ
}
``` [2](#0-1) 

When `msg_hash = 0` every participant's share collapses to `sᵢ = r · σᵢ`. After Lagrange-weighted summation the coordinator holds `s = r · σ = r · k · x`. The coordinator's own post-signing verification:

```rust
// src/ecdsa/ot_based_ecdsa/sign.rs  lines 128–133
if !sig.verify(&public_key, &msg_hash) {
    return Err(ProtocolError::AssertionFailed(...));
}
``` [3](#0-2) 

**passes**, because `(R, r·k·x)` is a valid ECDSA signature for `msg_hash = 0` (verification: `(r·k·x)⁻¹ · r · X = k⁻¹ · G = R` ✓).

By contrast, the Robust ECDSA variant explicitly guards against this:

```rust
// src/ecdsa/robust_ecdsa/sign.rs  lines 91–95
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
    ));
}
``` [4](#0-3) 

The OT-based ECDSA variant is missing this identical guard.

---

### Impact Explanation

**Impact: Critical — Unauthorized creation of a valid threshold signature for attacker-chosen inputs.**

1. **Valid signature for attacker-chosen input.** Any caller who passes `msg_hash = 0` receives a fully valid, verifiable ECDSA threshold signature `(R, s)` with `s = r · k · x`. The signature is not for any real message; it is a signature on the zero scalar, an attacker-controlled value.

2. **Partial private-key material disclosure.** The output signature satisfies `s / r = k · x`. Because `r` is public (it is the x-coordinate of the public nonce point `R`), the product `k · x` is directly readable from the signature. This is a disclosure of a function of the aggregate secret `x` combined with the presign nonce `k`. If `k` is ever recovered (e.g., via a separate nonce-reuse vulnerability or side-channel), the master private key `x` is immediately computable as `x = (s/r) / k`.

3. **Presignature consumed.** The one-time presignature is irrevocably spent producing this degenerate signature, denying its use for a legitimate message.

---

### Likelihood Explanation

The `sign()` function is a public library API. Any caller — including a malicious coordinator who controls which `msg_hash` value is broadcast to co-signers — can supply `msg_hash = 0`. No out-of-band mechanism prevents this; the library is the only enforcement point. The Robust ECDSA path already demonstrates the developers identified this risk and added the guard there; the OT-based path was simply left unguarded. Exploitation requires only calling the public API with a zero scalar, which is trivially reachable.

---

### Recommendation

Add the same zero-hash guard to `sign()` in `src/ecdsa/ot_based_ecdsa/sign.rs` that already exists in `src/ecdsa/robust_ecdsa/sign.rs`:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0".to_string(),
    ));
}
```

Place it immediately after the participant-count check, before the protocol future is constructed.

let protocol = ot_based_ecdsa::sign::sign(
    &participants,
    coordinator,
    threshold,
    me,
    public_key,

// result is Ok(Some(sig)) where sig = (R, r·k·x)

```

The function returns `Ok(...)` without error, the protocol runs to completion, and the coordinator emits a valid `Signature { big_r: R, s: r·k·x }` that passes standard ECDSA verification for `msg_hash = 0`.

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    presignature: &RerandomizedPresignOutput,

    // Compute si = h * ki + Rx * sigmai
    // Spec 1.3
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)
}
