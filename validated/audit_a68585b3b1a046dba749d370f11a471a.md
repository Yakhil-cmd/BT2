### Title
Missing Commission Slippage Protection in `switch_delegation_pool` and `enter_delegation_pool` Allows Staker to Front-Run Delegators - (File: src/pool/pool.cairo)

---

### Summary

`switch_delegation_pool` and `enter_delegation_pool` accept no `max_commission` parameter. A staker holding an active `commission_commitment` can legally increase their commission (up to `max_commission`) at any time with immediate effect. A delegator who submits a pool-entry or pool-switch transaction has no on-chain mechanism to reject execution if the commission has risen above their acceptable threshold by the time the transaction is included.

---

### Finding Description

`set_commission` in `src/staking/staking.cairo` writes the new commission value directly to storage with no delay: [1](#0-0) 

When a `commission_commitment` is active, the commission may be set to **any value up to `max_commission`**, including values higher than the current commission. The code itself acknowledges this: [2](#0-1) 

The comment at the function definition reinforces this: [3](#0-2) 

Neither `enter_delegation_pool` nor `switch_delegation_pool` expose a `max_commission` guard: [4](#0-3) [5](#0-4) 

The commission read by the pool contract is fetched live from the staking contract at query time, not locked at intent time: [6](#0-5) 

---

### Impact Explanation

A delegator who observes commission = 5 % and submits `switch_delegation_pool` (or `enter_delegation_pool`) can have that transaction execute against a pool whose commission has just been raised to `max_commission` (e.g. 50 %). The delegator is now bound to a pool with a commission rate they never accepted, and all future yield accrual is reduced accordingly. There is no on-chain mechanism to abort the operation if the commission exceeds the delegator's tolerance.

**Impact: Medium** — damage to delegators (reduced yield) with no ability to protect themselves at the protocol level.

---

### Likelihood Explanation

**Likelihood: Medium.**

Requirements:
1. The target staker must have called `set_commission_commitment` with `max_commission > current_commission`.
2. The staker submits a `set_commission` transaction raising the rate, ordering it before the delegator's switch/entry in the same block (or in a preceding block).
3. The delegator's transaction executes against the new, higher commission.

Step 1 is a normal protocol action. Steps 2–3 require deliberate ordering but no privileged access — any staker can do this to any delegator who is switching to or entering their pool.

---

### Recommendation

Add a `max_commission` parameter to both `enter_delegation_pool` and `switch_delegation_pool`, and assert the live commission does not exceed it at execution time:

```diff
- fn enter_delegation_pool(
-     ref self: ContractState, reward_address: ContractAddress, amount: Amount,
+ fn enter_delegation_pool(
+     ref self: ContractState, reward_address: ContractAddress, amount: Amount,
+     max_commission: Commission,
  ) {
+     let live_commission = self.get_commission_from_staking_contract();
+     assert!(live_commission <= max_commission, "commission exceeds max");
      ...
  }

- fn switch_delegation_pool(
-     ref self: ContractState,
-     to_staker: ContractAddress,
-     to_pool: ContractAddress,
-     amount: Amount,
+ fn switch_delegation_pool(
+     ref self: ContractState,
+     to_staker: ContractAddress,
+     to_pool: ContractAddress,
+     amount: Amount,
+     max_commission: Commission,
  ) -> Amount {
+     let to_pool_dispatcher = IPoolDispatcher { contract_address: to_pool };
+     let live_commission = to_pool_dispatcher.get_commission_from_staking_contract();
+     assert!(live_commission <= max_commission, "commission exceeds max");
      ...
  }
```

### Proof of Concept

1. Staker calls `set_commission_commitment(max_commission: 5000, expiration_epoch: E+10)` — commits to allow up to 50 % commission.
2. Staker currently has `commission = 200` (2 %).
3. Delegator observes 2 % commission and submits `switch_delegation_pool(to_staker, to_pool, amount)`.
4. Staker front-runs by submitting `set_commission(commission: 5000)` in the same block, ordered first.
   - `update_commission` passes: `5000 <= max_commission (5000)` and `5000 != 200`. [7](#0-6) 
   - Commission is written immediately. [1](#0-0) 
5. Delegator's `switch_delegation_pool` executes. No commission check exists. [8](#0-7) 
6. Delegator is now a member of a 50 % commission pool, receiving half the yield they expected, with no recourse at the protocol level.

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }

        fn get_commission_from_staking_contract(self: @ContractState) -> Commission {
            if self.staker_removed.read() {
                return Zero::zero();
            }
            let staking_dispatcher = IStakingDispatcher {
                contract_address: self.staking_pool_dispatcher.contract_address.read(),
            };
            // The staker must have commission since it has a pool (this contract). So unwrap is
            // safe.
            staking_dispatcher
                .staker_pool_info(staker_address: self.staker_address.read())
                .commission
                .unwrap()
        }
