### Title
Silo Whitelist Bypass via `call` and `deploy_code` Entry Points — (File: `engine/src/contract_methods/evm_transactions.rs`)

### Summary
In Aurora Engine's Silo mode, the whitelist access guard (`assert_access`) is only enforced inside `submit_with_alt_modexp`, which backs the `submit` and `submit_with_args` entry points. The `call` and `deploy_code` entry points skip this guard entirely, allowing any unprivileged NEAR account to bypass the Silo whitelist and interact with the EVM without authorization.

### Finding Description

Aurora Engine's Silo mode provides four whitelists (`Admin`, `EvmAdmin`, `Account`, `Address`) to restrict who may submit transactions or deploy code to the EVM. The enforcement functions are: [1](#0-0) 

The access check is invoked inside `submit_with_alt_modexp`: [2](#0-1) 

However, the `call` entry point only checks `require_running` and never calls `assert_access`: [3](#0-2) 

Likewise, `deploy_code` only checks `require_running`: [4](#0-3) 

This is structurally identical to the Rubicon M-34 pattern: the guard is placed in one sub-path (`submit_with_alt_modexp`) but absent from the alternative entry points (`call`, `deploy_code`). The `call` and `deploy_code` NEAR contract methods are publicly callable by any NEAR account, so the whitelist is silently bypassed whenever these paths are used instead of `submit`.

The `call_with_args` method on `Engine` is also invoked by the admin-only `set_erc20_metadata` without any whitelist check, confirming that `call_with_args` itself does not internally enforce the silo whitelist. [5](#0-4) 

The `require_running` guard that *is* present in both paths only checks the global `is_paused` flag, not the silo whitelist: [6](#0-5) 

### Impact Explanation

When the Silo whitelist is enabled (its intended production purpose), any unauthorized NEAR account can:

1. **Call arbitrary EVM contracts** via the `call` entry point — bypassing the `Account`/`Address` whitelist — and drain funds held in EVM contracts.
2. **Deploy arbitrary EVM bytecode** via the `deploy_code` entry point — bypassing the `Admin`/`EvmAdmin` whitelist.

This constitutes **direct theft of user funds (Critical)** and unauthorized code deployment.

### Likelihood Explanation

The Silo whitelist is an explicitly supported production feature. [7](#0-6) 

Any NEAR account can call public methods on the Aurora Engine contract without restriction at the NEAR protocol level. An attacker who observes that the whitelist is active can trivially substitute a `call` invocation for a `submit` invocation. No special privileges, leaked keys, or governance capture are required.

### Recommendation

Move the `assert_access` check (or equivalent silo whitelist enforcement using `silo::is_allow_submit` / `silo::is_allow_deploy`) into the `call` and `deploy_code` functions in `engine/src/contract_methods/evm_transactions.rs`, immediately after `require_running`, mirroring the enforcement already present in `submit_with_alt_modexp`. [3](#0-2) 

### Proof of Concept

1. Deploy Aurora Engine in Silo mode; enable the `Account` and `Address` whitelists via `set_whitelist_status`.
2. Do **not** add the attacker's NEAR account or derived EVM address to any whitelist.
3. Attempt `submit` with a signed Ethereum transaction from the attacker → rejected by `assert_access` inside `submit_with_alt_modexp`.
4. Call the `call` NEAR entry point directly from the attacker's NEAR account, targeting the same EVM contract with the same calldata → **succeeds**, whitelist check never executed, funds accessible. [8](#0-7)

            .try_parse_bytes(args.tx_data.as_slice(), block_height)
            .map_err(EngineErrorKind::FailedTransactionParse)?;
        tx.try_into()
            .map_err(|_e| EngineErrorKind::InvalidSignature)?
    };
    // Retrieve the signer of the transaction:
    let sender = transaction.address;

    let fixed_gas = silo::get_fixed_gas(&io);

pub fn deploy_code<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let input = io.read_input().to_vec();
        let current_account_id = env.current_account_id();
        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&env.predecessor_account_id()),
            current_account_id,
            io,
            env,
        );
        let result = engine.deploy_code_with_input(input, None, handler)?;
        let result_bytes = borsh::to_vec(&result).map_err(|_| errors::ERR_SERIALIZE)?;
        io.return_output(&result_bytes);
        Ok(result)
    })
}

pub fn call<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let bytes = io.read_input().to_vec();
        let args = CallArgs::deserialize(&bytes).ok_or(errors::ERR_BORSH_DESERIALIZE)?;
        let current_account_id = env.current_account_id();
        let predecessor_account_id = env.predecessor_account_id();

        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&predecessor_account_id),
            current_account_id,
            io,
            env,
        );
        let result = engine.call_with_args(args, handler)?;
        let result_bytes = borsh::to_vec(&result).map_err(|_| errors::ERR_SERIALIZE)?;
        io.return_output(&result_bytes);
        Ok(result)
    })
}

    /// Enable a whitelist. (A whitelist is disabled after creation).
    pub fn enable(&mut self) {
        let key = self.key(STATUS);

    }

    pub fn disable(&mut self) {
        let key = self.key(STATUS);

    }

    /// Check if the whitelist is enabled.
    pub fn is_enabled(&self) -> bool {
        // White list is disabled by default. So return `false` if the key doesn't exist.
        let key = self.key(STATUS);
        self.io
            .read_storage(&key)
            .is_some_and(|value| value.to_vec() == [1])
