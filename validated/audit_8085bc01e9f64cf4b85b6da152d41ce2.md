Audit Report

## Title
`RequestContext` Hash/PartialEq Omits `request_type`, Enabling QueryV2-to-QueryV3 Cache Collision - (`rs/boundary_node/ic_boundary/src/routes.rs`)

## Summary
The hand-written `Hash` and `PartialEq` implementations for `RequestContext` exclude the `request_type` field, causing `QueryV2` and `QueryV3` requests with identical `canister_id`, `sender`, `method_name`, `ingress_expiry`, and `arg` to map to the same boundary-node cache entry. An attacker who populates the cache with a `QueryV2` response can cause subsequent `QueryV3` requests sharing those parameters to receive the cached uncertified response, stripping the node-level BLS signature that `QueryV3` is designed to deliver.

## Finding Description
**Root cause — `routes.rs` lines 94–125:**
`preprocess_request` (`process.rs` line 149) correctly sets `ctx.request_type` from the `Extension<RequestType>` injected by `validate_canister_request` (`validate.rs` lines 36–47), which distinguishes `QueryV2` from `QueryV3` based on the matched URL path. However, the `Hash` impl (lines 94–108) hashes only `canister_id`, `sender`, `method_name`, `ingress_expiry`, and `arg`/`http_request`; `request_type` is never fed to the hasher. The `PartialEq` impl (lines 111–125) has the identical omission.

**Cache path — `cache.rs` lines 33–86:**
`KeyExtractorContext` extracts `Arc<RequestContext>` directly as the cache key. `BypasserIC::bypass` allows any request where `ctx.request_type.is_query()` returns `true` and no nonce is present; `is_query()` (`http/mod.rs` lines 62–64) returns `true` for both `QueryV2` and `QueryV3`. There is no per-version bypass.

**Attack flow:**
1. Attacker sends `POST /api/v2/canister/{id}/query` (anonymous, no nonce) with parameters `(canister_id=C, sender=anon, method_name=M, ingress_expiry=T, arg=A)`. Cache miss → replica returns uncertified QueryV2 response → stored under key `hash(C, anon, M, T, A)`.
2. Victim sends `POST /api/v3/canister/{id}/query` with the same `(C, anon, M, T, A)`. Cache key is identical → **cache hit** → victim receives the QueryV2 response with no node signature.

**Existing guards are insufficient:** The nonce bypass only fires when `ctx.nonce.is_some()`; neither the attacker's nor the typical victim's request carries a nonce. The non-anonymous bypass is irrelevant for anonymous queries. No other guard differentiates `QueryV2` from `QueryV3` in the cache path.

**Constraint:** The `ingress_expiry` field (nanosecond timestamp) is included in the cache key. For a cache hit to occur, the attacker's QueryV2 request and the victim's QueryV3 request must share the exact same `ingress_expiry` value. This is a meaningful constraint that limits opportunistic exploitation but does not eliminate it: client libraries that round to second-level precision, or scenarios where the attacker can observe or predict the victim's `ingress_expiry`, make the collision achievable.

## Impact Explanation
A QueryV3 client that receives a cached QueryV2 response obtains a CBOR body with no node-level BLS signature. If the client enforces certificate verification, the check fails; if it falls back to the uncertified body, it silently accepts a response whose authenticity cannot be verified. This constitutes a **forged or stale certified response accepted under constrained conditions**, matching the Medium bounty impact tier ($200–$2,000): "forged or stale certified response accepted only under constrained conditions."

## Likelihood Explanation
The attack requires no privileges and no special network position. The primary constraint is matching `ingress_expiry`. Client libraries that use second-level (or coarser) timestamp granularity reduce the search space to ~300 values over a 5-minute validity window, making targeted poisoning feasible with a modest number of requests. The attacker must also match `canister_id`, `sender` (anonymous), `method_name`, and `arg`, all of which are public or predictable for well-known canisters and methods. The attack is repeatable for the duration of the cache TTL.

## Recommendation
Add `request_type` to both `Hash` and `PartialEq` in `RequestContext` (`rs/boundary_node/ic_boundary/src/routes.rs`):

```rust
impl Hash for RequestContext {
    fn hash<H: Hasher>(&self, state: &mut H) {
        self.request_type.hash(state);   // add
        self.canister_id.hash(state);
        self.sender.hash(state);
        self.method_name.hash(state);
        self.ingress_expiry.hash(state);
        if self.http_request.is_some() {
            self.http_request.hash(state);
        } else {
            self.arg.hash(state);
        }
    }
}

impl PartialEq for RequestContext {
    fn eq(&self, other: &Self) -> bool {
        let r = self.request_type == other.request_type   // add
            && self.canister_id == other.canister_id
            && self.sender == other.sender
            && self.method_name == other.method_name
            && self.ingress_expiry == other.ingress_expiry;
        if self.http_request.is_some() {
            r && self.http_request == other.http_request
        } else {
            r && self.arg == other.arg
        }
    }
}
```

`RequestType` already derives `Hash`, `PartialEq`, and `Eq`, so no further changes are needed to that type.

## Proof of Concept
A deterministic unit test can be added to `rs/boundary_node/ic_boundary/src/http/middleware/cache.rs` alongside the existing `test_cache`:

```rust
#[tokio::test]
async fn test_no_cross_version_cache_hit() -> Result<(), Error> {
    // build cache with cache_non_anonymous=true so anonymous requests are cached
    let cli = cli::Cache { cache_size: Some(MAX_MEM_SIZE), cache_max_item_size: MAX_RESP_SIZE,
                           cache_ttl: Duration::from_secs(3600), cache_non_anonymous: false };
    let cache_state = Arc::new(CacheState::new(&cli, &Registry::new()).unwrap());
    let mut app = Router::new().route("/", post(handler))
        .layer(middleware::from_fn_with_state(cache_state, cache_middleware));

    // Step 1: QueryV2 request → cache miss, response stored
    let req_v2 = gen_request_with_params(CANISTER_1, RequestType::QueryV2,
                                         false, DEFAULT_SIZE, 42, true, StatusCode::OK);
    let res = app.call(req_v2).await.unwrap();
    assert!(matches!(res.extensions().get::<CacheStatus<BypassReasonIC>>().unwrap(),
                     CacheStatus::Miss(_)));

    // Step 2: QueryV3 request with identical parameters → MUST be a miss, not a hit
    let req_v3 = gen_request_with_params(CANISTER_1, RequestType::QueryV3,
                                         false, DEFAULT_SIZE, 42, true, StatusCode::OK);
    let res = app.call(req_v3).await.unwrap();
    // Before fix: CacheStatus::Hit — proves cross-version collision
    // After fix:  CacheStatus::Miss — correct behaviour
    assert!(matches!(res.extensions().get::<CacheStatus<BypassReasonIC>>().unwrap(),
                     CacheStatus::Miss(_)),
            "QueryV3 must not receive a cached QueryV2 response");
    Ok(())
}
```

Running this test against the unfixed code will fail at the final assertion, confirming the collision. After adding `request_type` to `Hash`/`PartialEq`, the test passes.
