### Title
Unprivileged User Can Permanently Lock Admin Out of Token Mapping Updates via Auto-Deployment Pre-emption — (`x/cronos/keeper/keeper.go`, `x/cronos/keeper/evm.go`)

### Summary

When `EnableAutoDeployment` is `true`, any unprivileged user holding IBC vouchers can call `MsgConvertVouchers` to trigger auto-deployment of a CRC21 contract for a denom. The 2026-04 guardrail in `SetExternalContractForDenom` (`ensureDenomNotMapped`) then permanently blocks the admin or governance from ever registering an external contract for that denom, because auto-mapped denoms are treated identically to externally-mapped ones. Since there is no `DeleteAutoContractForDenom` primitive, the lock is irreversible without a chain upgrade.

### Finding Description

**Step 1 — Auto-deployment path (unprivileged)**

`ConvertVouchersToEvmCoins` (called from `MsgConvertVouchers`, no permission check) delegates to `ConvertCoinFromNativeToCRC21` with `autoDeploy = params.EnableAutoDeployment`: [1](#0-0) 

When no contract is found for the denom and `autoDeploy=true`, a new CRC21 is deployed and `SetAutoContractForDenom` writes both the forward key (`DenomToAutoContractKey`) and the reverse key (`ContractToDenomKey`): [2](#0-1) 

**Step 2 — The blocking guardrail**

`SetExternalContractForDenom` (the only write path for admin/governance token-mapping updates) calls `ensureDenomNotMapped` before writing: [3](#0-2) 

`ensureDenomNotMapped` calls `GetContractByDenom`, which checks **both** external and auto mappings: [4](#0-3) [5](#0-4) 

If an auto mapping already exists, `GetContractByDenom` returns `found=true`, and `ensureDenomNotMapped` returns `ErrDenomAlreadyMapped`, aborting the admin's `SetExternalContractForDenom` call.

**Step 3 — No escape hatch**

`DeleteExternalContractForDenom` only removes the `DenomToExternalContractKey` entry: [6](#0-5) 

There is no `DeleteAutoContractForDenom`. The spec confirms: *"There's no way to delete a token mapping currently."* [7](#0-6) 

`RegisterOrUpdateTokenMapping` (the admin/governance entry point) calls `SetExternalContractForDenom` for non-source denoms, so both the `MsgUpdateTokenMapping` message path and the `TokenMappingChangeProposal` governance path are blocked: [8](#0-7) 

### Impact Explanation

Once an attacker triggers auto-deployment for a denom, the admin permanently loses the ability to:
- Register an external (proxy/upgradeable) contract for that denom
- Fix bugs in the auto-deployed minimal CRC21 contract
- Implement gravity-bridge proxy contracts (as exercised in `test_gravity_proxy_contract`)
- Respond to security incidents requiring contract replacement

This is a **permanent corruption of the token-mapping state** and a **permanent bypass of admin token-mapping authority** for any denom the attacker targets. The only remediation path is a chain upgrade.

This matches the allowed High impact: *"Bypass of Cronos admin, governance authority, permission, token-mapping… authorization checks"* and *"Corruption of token mappings, denom/contract binding… with direct security impact."*

### Likelihood Explanation

- `EnableAutoDeployment` is a chain parameter that can be `true` (it is `true` in the `genesis_token_mapping.jsonnet` test config and is the intended production mode for IBC token auto-wrapping).
- The attacker only needs to hold any amount of an IBC voucher for the target denom — a trivially low bar.
- `MsgConvertVouchers` has no permission check; any account can submit it.
- The attack is a single transaction and is irreversible.

### Recommendation

`SetExternalContractForDenom` should distinguish between an existing **external** mapping (which should block re-registration) and an existing **auto** mapping (which the admin should be allowed to supersede). Concretely:

1. In `ensureDenomNotMapped`, check only `getExternalContractByDenom` rather than the combined `GetContractByDenom`.
2. When `SetExternalContractForDenom` succeeds and an auto mapping exists for the same denom, atomically retire the auto mapping (delete `DenomToAutoContractKey` and update the reverse entry) so the 1-to-1 invariant is preserved.

Alternatively, expose a privileged `DeleteAutoContractForDenom` callable only by admin/governance so the lock can be broken without a chain upgrade.

### Proof of Concept

```
Precondition: EnableAutoDeployment = true, IBC token denom = "ibc/ABCD..."

1. Attacker receives 1 unit of "ibc/ABCD..." via IBC transfer.

2. Attacker submits MsgConvertVouchers{Address: attacker, Coins: [{denom:"ibc/ABCD...", amount:1}]}
   → ConvertVouchersToEvmCoins → ConvertCoinFromNativeToCRC21(autoDeploy=true)
   → GetContractByDenom returns (_, false)          // no mapping yet
   → DeployModuleCRC21 deploys CRC21 at address 0xAUTO
   → SetAutoContractForDenom writes:
       DenomToAutoContractKey["ibc/ABCD..."] = 0xAUTO
       ContractToDenomKey[0xAUTO]            = "ibc/ABCD..."

3. Admin submits MsgUpdateTokenMapping{Denom:"ibc/ABCD...", Contract:"0xEXTERNAL"}
   → RegisterOrUpdateTokenMapping → SetExternalContractForDenom
   → ensureDenomNotMapped → GetContractByDenom
       → getExternalContractByDenom: not found
       → getAutoContractByDenom:     found (0xAUTO)  ← returns (0xAUTO, true)
   → ensureDenomNotMapped returns ErrDenomAlreadyMapped
   → MsgUpdateTokenMapping FAILS permanently.

4. Admin submits MsgUpdateTokenMapping{Denom:"ibc/ABCD...", Contract:""}
   → DeleteExternalContractForDenom: external key not found → returns false, no-op.
   Auto mapping remains. Step 3 still fails.
```

func (k Keeper) ConvertCoinFromNativeToCRC21(ctx sdk.Context, sender common.Address, coin sdk.Coin, autoDeploy bool) error {
	if !types.IsValidCoinDenom(coin.Denom) {
		return fmt.Errorf("coin %s is not supported for conversion", coin.Denom)
	}
	var err error
	// external contract is returned in preference to auto-deployed ones
	contract, found := k.GetContractByDenom(ctx, coin.Denom)
	if !found {
		if !autoDeploy {
			return fmt.Errorf("no contract found for the denom %s", coin.Denom)
		}
		contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
		if err != nil {
			return err
		}
		if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
			return err
		}

		k.Logger(ctx).Info(fmt.Sprintf("contract address %s created for coin denom %s", contract.String(), coin.Denom))
	}

// SetExternalContractForDenom sets denom→external CRC21 mapping for an unmapped denom.
// Caller is responsible for source-denom specific validation before calling this method.
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}

// DeleteExternalContractForDenom delete the external contract mapping for native denom,
// returns false if mapping not exists.
func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
	store := ctx.KVStore(k.storeKey)
	contract, found := k.getExternalContractByDenom(ctx, denom)
	if !found {
		return false
	}
	store.Delete(types.DenomToExternalContractKey(denom))
	deleteReverseIfOwned(store, contract, denom)
	if auto, found := k.getAutoContractByDenom(ctx, denom); found {
		bz := store.Get(types.ContractToDenomKey(auto.Bytes()))
		if len(bz) == 0 {
			store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
		} else if existingDenom := string(bz); existingDenom != denom {
			if k.contractOwnedByDenom(ctx, existingDenom, auto) {
				// auto address is already owned by another denom; drop local auto mapping
				store.Delete(types.DenomToAutoContractKey(denom))
			} else {
				// stale reverse entry
				store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
			}
		}
	}
	return true
}
