### Title
Unbounded Balance Trace Loop in `calculate_rewards` Enables Permanent Freezing of Delegator Unclaimed Yield - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` function in the Pool contract iterates over a pool member's entire `pool_member_epoch_balance` trace without any bound. A delegator who makes repeated deposits across many epochs without claiming rewards will grow this trace unboundedly. Once the trace is large enough, the gas cost of iterating it exceeds Starknet's block gas limit, permanently bricking the delegator's ability to ever call `claim_rewards`.

### Finding Description
In `src/pool/pool.cairo`, the internal `calculate_rewards` function iterates over every entry in the `pool_member_epoch_balance` trace that falls between the delegator's last-claimed checkpoint and the current epoch:

```
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The trace grows by one entry each time `set_member_balance` is called with a new epoch key. This happens inside `increase_member_balance`, which is called by `add_to_delegation_pool`: [2](#0-1) 

The `insert` function in the trace only appends a new checkpoint when the epoch key differs from the last entry's key; same-epoch calls overwrite the last entry: [3](#0-2) 

Therefore, one call to `add_to_delegation_pool` per epoch appends exactly one new entry to the trace. The `entry_to_claim_from` cursor in `pool_member_info` advances only when `claim_rewards` is called: [4](#0-3) 

If a delegator never calls `claim_rewards`, the cursor stays at 0 and every future `claim_rewards` call must iterate the entire accumulated trace from the beginning.

### Impact Explanation
Once the trace length is large enough that iterating it exceeds the Starknet block gas limit, `claim_rewards` will always revert. The delegator's accrued yield is permanently frozen in the contract with no recovery path — there is no administrative function to reset `entry_to_claim_from` or to partially claim rewards up to a caller-specified index. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
A delegator who compounds their stake by calling `add_to_delegation_pool` once per epoch over a long period (e.g., daily top-ups over months or years) without regularly claiming rewards will organically grow their trace to a dangerous size. The protocol provides no warning, no cap, and no partial-claim mechanism. The developers themselves acknowledge the risk in the comment "This loop is unbounded but unlikely to exceed gas limits," indicating awareness but no mitigation. [5](#0-4) 

### Recommendation
1. **Short term:** Add a `max_entries` parameter to `claim_rewards` (or a separate `partial_claim_rewards(up_to_entry: VecIndex)`) so a delegator can drain the trace in multiple bounded transactions.
2. **Long term:** Enforce a maximum trace length per pool member, or automatically checkpoint (claim) rewards whenever `add_to_delegation_pool` is called, so the trace never grows beyond a small constant size.

### Proof of Concept
1. Delegator calls `enter_delegation_pool` with the minimum amount in epoch E.
2. Delegator calls `add_to_delegation_pool` with 1 unit in each of epochs E+1, E+2, …, E+N (one call per epoch, never calling `claim_rewards`).
3. After N epochs, `pool_member_epoch_balance` trace for this delegator has N+1 entries and `entry_to_claim_from == 0`.
4. Delegator calls `claim_rewards`. The loop at line 859 iterates all N+1 entries.
5. For sufficiently large N (determined by Starknet's per-transaction gas cap), the transaction runs out of gas and reverts.
6. Every subsequent `claim_rewards` call also reverts — the delegator's accumulated yield is permanently frozen. [6](#0-5) [7](#0-6)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
