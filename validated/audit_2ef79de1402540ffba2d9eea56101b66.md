### Title
Missing Sequencer-Only Access Control on `update_rewards` Enables Permanent Freezing of Consensus Rewards — (File: `src/staking/staking.cairo`)

---

### Summary
The `update_rewards` function in the `StakingRewardsManagerImpl` is specified as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged caller can invoke it with `disable_rewards: true`, consuming the per-block reward slot (via `last_reward_block`) without distributing any rewards. Repeated across every block, this permanently freezes all consensus-era unclaimed yield for all stakers.

---

### Finding Description

The spec for `update_rewards` states:

> **Access control:** Only starkware sequencer. [1](#0-0) 

The implementation, however, only calls `general_prerequisites()` (which checks the paused flag and that the caller is non-zero) and a per-block guard: [2](#0-1) 

There is no check that `get_caller_address() == sequencer_address`. After the block guard passes, `last_reward_block` is unconditionally written to the current block number: [3](#0-2) 

If `disable_rewards` is `true`, the function returns immediately after that write, distributing nothing: [4](#0-3) 

Because `last_reward_block` is a single global slot, once it is set to the current block no further call to `update_rewards` can succeed for that block (the `REWARDS_ALREADY_UPDATED` assertion fires). The rewards for that block are permanently lost.

The analog to the original report is direct: the spec declares a protection ("only sequencer"), but the code stores no enforcement of that protection alongside the guarded logic — the "lock" is described in the spec but the "key check" is absent from the code, making the protection illusory.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_valid_active_staker, disable_rewards: true)` once per block:

1. Consumes the block's single reward slot.
2. Distributes zero rewards to any staker.
3. Prevents the legitimate sequencer from ever calling `update_rewards` for that block.

Repeated every block, this permanently prevents all consensus-era staking rewards from accruing to any staker or pool member. Accumulated `unclaimed_rewards_own` balances are never incremented; pool reward traces are never updated. The yield is not redirected — it is simply never created.

---

### Likelihood Explanation

**High.** The entry point is a public, permissionless function on a deployed contract. No privileged role, leaked key, or external dependency is required. The only cost to the attacker is gas per block. On Starknet, transaction fees are low, making sustained griefing economically viable. The attacker gains nothing financially, but the damage to all stakers and delegators is total and irreversible for every affected block.

---

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`, consistent with the spec. In Cairo/Starknet this is done by reading the sequencer address from the block info and asserting the caller matches:

```cairo
let exec_info = starknet::get_execution_info().unbox();
assert!(
    get_caller_address() == exec_info.block_info.unbox().sequencer_address,
    "{}",
    Error::CALLER_IS_NOT_SEQUENCER,
);
```

This mirrors the pattern already used for the attestation contract guard (`CALLER_IS_NOT_ATTESTATION_CONTRACT`). [5](#0-4) 

---

### Proof of Concept

1. Deploy the system normally and advance past the consensus rewards start epoch.
2. As any EOA (not the sequencer), call:
   ```
   staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
   ```
3. Observe that `last_reward_block` is now equal to the current block number.
4. The legitimate sequencer attempts `update_rewards` for the same block → reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat step 2 every block. After N blocks, all stakers have `unclaimed_rewards_own` unchanged from before consensus rewards started; pool reward traces are never updated. No yield has been distributed. [6](#0-5) [7](#0-6)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
