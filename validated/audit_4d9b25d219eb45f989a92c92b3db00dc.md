### Title
Unprivileged caller can permanently freeze staker consensus block rewards via `update_rewards` with `disable_rewards: true` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `Staking.cairo` is a public, access-control-free function that writes to the global `last_reward_block` storage slot **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` in a given block to consume the `last_reward_block` slot without distributing any rewards, causing every subsequent legitimate `update_rewards` call in that same block to revert with `REWARDS_ALREADY_UPDATED`. Repeated across blocks, this permanently freezes all staker consensus block rewards.

---

### Finding Description

`update_rewards` is the sole mechanism for distributing per-block consensus rewards to stakers. Its implementation is:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // ← last_reward_block is written HERE, before the disable_rewards check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← returns with NO rewards distributed
    }
    // actual reward distribution follows
    ...
}
``` [1](#0-0) 

The critical ordering flaw: `last_reward_block` is updated at line 1485 unconditionally, **before** the `disable_rewards` guard at line 1487. Because `general_prerequisites` only asserts the contract is unpaused and the caller is non-zero, any address can call this function. [2](#0-1) 

The `last_reward_block` is a single global slot shared across all stakers: [3](#0-2) 

Once it is set to block `N`, the assertion `current_block_number > last_reward_block` fails for every other call in block `N`, regardless of which staker is targeted or whether rewards should be distributed.

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block:

1. Consumes the `last_reward_block` slot with no reward distribution.
2. Forces every legitimate `update_rewards(block_producer, disable_rewards: false)` call in that block to revert.
3. Block producers never accumulate `unclaimed_rewards_own`; the STRK that would have been minted and transferred from the `RewardSupplier` is never claimed.

This constitutes **permanent freezing of unclaimed yield** (consensus block rewards) for all stakers, matching the allowed High impact category. The rewards are not merely delayed — they are skipped for each attacked block and cannot be retroactively recovered, because `calculate_block_rewards` computes rewards for the *current* epoch/block and there is no catch-up mechanism. [4](#0-3) 

---

### Likelihood Explanation

- `update_rewards` has **no access control** — any non-zero address can call it.
- The attacker only needs to supply any currently active staker address with non-zero balance (trivially observable on-chain).
- On Starknet the sequencer orders transactions; an attacker submitting a `disable_rewards: true` call with a competitive fee in every block can reliably front-run the legitimate consensus-layer call.
- The cost to the attacker is one transaction per block; the damage is total denial of consensus rewards to all stakers.

---

### Recommendation

**Short term**: Move the `last_reward_block` write to after the `disable_rewards` guard, so that a no-op call does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);  // ← moved here
// reward distribution ...
```

**Long term**: Add access control to `update_rewards` so that only the designated consensus layer address (or a trusted operator role) can invoke it, eliminating the unprivileged entry path entirely.

---

### Proof of Concept

1. Staker `S` is a valid, active staker with non-zero STRK balance. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. In block `N`, the consensus layer prepares to call `update_rewards(S, disable_rewards: false)` to distribute block rewards to `S`.
3. Attacker submits `update_rewards(S, disable_rewards: true)` with a higher fee, getting it sequenced first in block `N`.
4. Execution: `last_reward_block` is written to `N`; function returns early — **no rewards distributed**.
5. Consensus layer's call arrives: `assert!(N > N)` → **reverts with `REWARDS_ALREADY_UPDATED`**.
6. Staker `S` receives zero rewards for block `N`. The STRK that would have been minted remains unclaimed in the `RewardSupplier`.
7. Attacker repeats in block `N+1`, `N+2`, … — all stakers are permanently denied consensus block rewards. [5](#0-4) [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

            if curr_epoch > self.last_calculated_epoch.read() {

                self.block_rewards.write(block_rewards);
                block_rewards
            } else {
                self.block_rewards.read()
            }
