### Title
Unbounded Loop in `calculate_rewards` Enables Permanent Freezing of Pool Member Unclaimed Yield - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` internal function in `src/pool/pool.cairo` contains a `while` loop that iterates over every balance-change entry in a pool member's `pool_member_epoch_balance` trace without any cap on iterations. The code itself acknowledges this with the comment "This loop is unbounded but unlikely to exceed gas limits." A pool member who accumulates a sufficiently large trace — by repeatedly delegating and undelegating across many epochs without claiming — will eventually cause every `claim_rewards` call to revert due to gas exhaustion, permanently freezing their unclaimed yield.

### Finding Description

The root cause is in `calculate_rewards` at lines 857–877 of `src/pool/pool.cairo`:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The loop bound is `pool_member_trace_length`, which is the total number of entries ever written to the pool member's `pool_member_epoch_balance` storage trace. Each time a pool member changes their delegated balance (via `add_to_delegation_pool`, `pool_member_exit_intent`, `switch_delegation_pool`, etc.), a new checkpoint is appended to this trace. There is no cap on how many checkpoints can accumulate, and there is no pagination or partial-claim mechanism.

The `calculate_rewards` function is called from the public `claim_rewards` entry point. When the trace grows large enough that iterating over it exceeds the Starknet block gas limit, every `claim_rewards` invocation reverts, and the pool member's accumulated yield becomes permanently inaccessible. [2](#0-1) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Once the trace length crosses the gas-exhaustion threshold, no transaction can successfully execute `claim_rewards` for that pool member. The yield is locked in the contract forever with no recovery path, because there is no partial-claim or checkpoint-reset mechanism. This matches the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

### Likelihood Explanation

A pool member who is active over many epochs (e.g., adjusting delegation amounts each epoch) will grow their trace linearly with the number of balance changes. Starknet epochs are short; a member active for hundreds of epochs without claiming could accumulate hundreds of trace entries. The gas cost per iteration is non-trivial (storage reads inside `find_sigma` → `find_sigma_standard_case`), so the threshold is reachable in practice. This can also be self-inflicted accidentally by a legitimate long-term delegator who simply never claimed.

### Recommendation

1. **Paginate `claim_rewards`**: Accept a `max_entries` parameter and return the updated `entry_to_claim_from` so callers can claim in multiple transactions.
2. **Cap trace growth**: Consolidate or prune old trace entries when a claim is made, so the trace length is bounded by the number of unclaimed epochs rather than the lifetime of the account.
3. **Enforce a maximum trace length**: Reject balance-change operations that would push the trace beyond a safe bound, forcing the member to claim first.

### Proof of Concept

1. Deploy the staking system and create a pool.
2. As an unprivileged pool member, call `add_to_delegation_pool` once per epoch for N epochs without ever calling `claim_rewards`. Each call appends one entry to `pool_member_epoch_balance`.
3. After N epochs, call `claim_rewards`. The `calculate_rewards` loop must iterate N times, each iteration performing storage reads via `find_sigma` → `find_sigma_standard_case`.
4. As N grows large enough (exact threshold depends on per-iteration gas cost), the transaction runs out of gas and reverts.
5. Every subsequent `claim_rewards` call also reverts — the yield is permanently frozen. [3](#0-2)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
