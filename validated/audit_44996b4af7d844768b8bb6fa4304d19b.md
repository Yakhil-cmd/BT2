### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Suppress Staker Yield — (`File: src/staking/staking.cairo`)

---

### Summary

`update_rewards` is specified to be callable only by the Starknet sequencer, but the implementation contains no `get_caller_address()` check. Any unprivileged address can call it with `disable_rewards: true`, which writes `last_reward_block` to the current block and returns early without distributing rewards. Because the guard `current_block_number > last_reward_block` is checked first, the legitimate sequencer call in the same block will always revert with `REWARDS_ALREADY_UPDATED`, permanently suppressing per-block yield for every staker.

---

### Finding Description

`IStakingRewardsManager::update_rewards` in `src/staking/staking.cairo` is the sole mechanism for distributing consensus (V3) block rewards to stakers and their delegation pools.

The specification at `docs/spec.md:1645` states:

> **access control**: Only starkware sequencer.

The implementation, however, performs no such check:

```
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(current_block_number > self.last_reward_block.read(), Error::REWARDS_ALREADY_UPDATED);
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← written unconditionally
    if disable_rewards || self.is_pre_consensus() {
        return;                                           // ← exits before any reward math
    }
    ...
}
```

`last_reward_block` is committed to storage **before** the `disable_rewards` branch is evaluated. An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in block N:

1. Passes the `REWARDS_ALREADY_UPDATED` guard (first call in block N).
2. Writes `last_reward_block = N`.
3. Returns immediately — zero rewards are calculated or transferred.

Any subsequent call in block N (including the legitimate sequencer call) hits the guard and reverts. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who front-runs the sequencer's `update_rewards` call every block with `disable_rewards: true` causes every staker to earn zero consensus rewards indefinitely. The staker's `unclaimed_rewards_own` is never incremented, and pool contracts never receive their reward transfers. Because the attack costs only a single L2 transaction per block and requires no capital, it is economically viable as a griefing vector against the entire protocol.

Even sporadic execution (e.g., targeting specific high-value stakers or specific blocks) constitutes temporary freezing of unclaimed yield.

---

### Likelihood Explanation

**High.** The function is publicly callable with no access control. The attacker needs only to know any active staker address (trivially discoverable from on-chain events) and submit a transaction before the sequencer in each block. No privileged key, capital, or external dependency is required.

---

### Recommendation

Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the expected sequencer address (stored in contract state), mirroring the pattern used for `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer();   // add this guard
    ...
}
```

Alternatively, move the `self.last_reward_block.write(current_block_number)` call to **after** the `disable_rewards` / `is_pre_consensus` early-return so that a no-op call does not consume the block's reward slot. [3](#0-2) 

---

### Proof of Concept

```
// Block N begins.

// Step 1 — Attacker front-runs the sequencer.
// Caller: any_address (no restriction enforced)
staking.update_rewards(staker_address: alice, disable_rewards: true);
// → last_reward_block is now N
// → function returns early; alice earns 0 rewards

// Step 2 — Legitimate sequencer attempts its call.
staking.update_rewards(staker_address: alice, disable_rewards: false);
// → assert!(current_block_number > last_reward_block) FAILS
// → reverts with REWARDS_ALREADY_UPDATED

// alice.unclaimed_rewards_own unchanged; pool receives no tokens.
// Repeat every block → permanent yield freeze.
``` [4](#0-3) [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
