### Title
Threshold Value Above `u16::MAX` Accepted by DKG but Permanently Rejected During Signing — (`src/frost/eddsa/sign.rs`, `src/frost/redjubjub/sign.rs`)

---

### Summary

`assert_key_invariants` imposes no upper bound on the threshold beyond `participants.len()`. However, `construct_key_package` in both FROST signing modules silently converts the `usize` threshold to `u16` via a checked cast. Any threshold value above `u16::MAX` (65 535) will pass DKG successfully, produce valid-looking key shares, and then cause every subsequent signing attempt to permanently fail with a non-recoverable error — without any way to use the already-distributed key material.

---

### Finding Description

`ReconstructionLowerBound` wraps a `usize`, and the only validation performed before DKG is in `assert_key_invariants`:

```rust
// src/dkg.rs  lines 572-582
if threshold > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { … });
}
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { … });
}
``` [1](#0-0) 

No check enforces `threshold <= u16::MAX`. Because `Participant` is a `u32`, the participant count can reach `u32::MAX`, so a threshold of, say, 70 000 passes all documented guards.

DKG then runs to completion and distributes key shares. Later, when any party calls `construct_key_package` inside the signing protocol, the threshold is narrowed to `u16`:

```rust
// src/frost/eddsa/sign.rs  lines 365-368
u16::try_from(threshold.value()).map_err(|_| {
    ProtocolError::Other("threshold cannot be converted to u16".to_string())
})?
``` [2](#0-1) 

The identical pattern appears in the RedJubjub path:

```rust
// src/frost/redjubjub/sign.rs  lines 254-256
u16::try_from(threshold.value()).map_err(|_| {
    ProtocolError::Other("threshold cannot be converted to u16".to_string())
})?
``` [3](#0-2) 

`assert_sign_inputs` (called before the signing future is spawned) also performs no `u16` check, so the error surfaces only inside the running protocol, after all parties have already committed to the signing round. [4](#0-3) 

The result is a **threshold-state inconsistency**: the threshold accepted and committed to during DKG is structurally incompatible with the threshold accepted during signing, yet no validation bridges the two phases.

---

### Impact Explanation

**High — Permanent denial of signing for honest parties under valid protocol inputs.**

Once DKG completes with threshold > 65 535, the resulting key shares are permanently unusable for signing. Every call to `sign_v1`, `sign_v2`, or the RedJubjub `sign` will fail inside `construct_key_package` before any signature share is produced. Recovery requires re-running DKG with a new threshold, which changes the public key and invalidates all previously distributed shares. The original key material is irrecoverably locked out of the signing protocol.

---

### Likelihood Explanation

**Low.** A threshold exceeding 65 535 requires a participant set of at least 65 536 members. While the `Participant(u32)` type and `usize`-based threshold permit this, it is an unusual deployment scale. The risk is elevated in automated or programmatic deployments where threshold values are computed rather than manually chosen, and where the library's lack of an explicit upper-bound error could mislead integrators into believing any value satisfying `2 ≤ threshold ≤ n` is fully supported end-to-end.

---

### Recommendation

Add an explicit `u16` upper-bound check inside `assert_key_invariants` (and/or `assert_reshare_keys_invariants`) so that the error is surfaced at initialization time, before any DKG round begins:

```rust
if threshold > usize::from(u16::MAX) {
    return Err(InitializationError::ThresholdTooLarge {
        threshold,
        max: usize::from(u16::MAX),
    });
}
```

This ensures the threshold constraint is enforced uniformly across DKG and signing, eliminating the phase-split inconsistency.

---

### Proof of Concept

1. Call `keygen` (or `reshare`) with `threshold = 70_000` and `participants.len() >= 70_000`. `assert_key_invariants` passes; DKG completes and returns `KeygenOutput` for every party.
2. Call `sign_v1` (EdDSA) or `sign` (RedJubjub) with the same threshold and the produced key shares.
3. `assert_sign_inputs` passes.
4. Inside the spawned future, `construct_key_package` executes `u16::try_from(70_000)` → `Err(_)` → `ProtocolError::Other("threshold cannot be converted to u16")`.
5. Every party's signing future returns this error. No signature is ever produced. The distributed key shares cannot be used for signing at any future point without re-running DKG. [5](#0-4) [6](#0-5) [7](#0-6)

pub fn assert_key_invariants(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
) -> Result<ParticipantList, InitializationError> {
    let threshold = usize::from(threshold.into());
    // need enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // Step 1.1
    // validate threshold
    if threshold > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold,
            max: participants.len(),
        });
    }
    // Step 1.1
    if threshold < 2 {
        return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
    }

    // ensure uniqueness of participants in the participant list
    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }
    Ok(participants)
}

fn construct_key_package(
    threshold: ReconstructionLowerBound,
    me: Participant,
    keygen_output: &KeygenOutput,
) -> Result<KeyPackage, ProtocolError> {
    let identifier = me.to_identifier()?;
    let signing_share = keygen_output.private_share;
    let verifying_share = signing_share.into();
    let verifying_key = keygen_output.public_key;
    let key_package = KeyPackage::new(
        identifier,
        signing_share,
        verifying_share,
        verifying_key,
        u16::try_from(threshold.value()).map_err(|_| {
            ProtocolError::Other("threshold cannot be converted to u16".to_string())
        })?,
    );

    // Ensures the values are zeroized on drop
    Ok(key_package)
}
