### Title
Unprivileged IBC Transfer Permanently Locks Token Mapping, Blocking Admin and Governance from Registering External Contracts - (File: x/cronos/keeper/keeper.go)

### Summary
When `EnableAutoDeployment` is true, any unprivileged user can send an IBC transfer of a new token to permanently prevent the Cronos admin and governance from ever registering an external CRC21 contract for that denom. The 2026-04 `ErrDenomAlreadyMapped` enforcement in `SetExternalContractForDenom` blocks re-registration of any already-mapped denom, and there is no admin-accessible path to delete or replace auto-deployed contracts.

### Finding Description

**Root cause — `SetExternalContractForDenom` is create-only:**

`SetExternalContractForDenom` calls `ensureDenomNotMapped` before writing:

```go
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
    if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
        return err   // ErrDenomAlreadyMapped if ANY mapping exists
    }
    ...
}
``` [1](#0-0) 

`ensureDenomNotMapped` checks both external **and** auto-deployed contracts via `GetContractByDenom`:

```go
func (k Keeper) ensureDenomNotMapped(ctx sdk.Context, denom string) error {
    if contract, found := k.GetContractByDenom(ctx, denom); found {
        return errors.Wrapf(types.ErrDenomAlreadyMapped, ...)
    }
    return nil
}
``` [2](#0-1) 

**Trigger path — unprivileged IBC transfer deploys auto contract:**

When `EnableAutoDeployment` is true, any incoming IBC packet for a new denom flows through `IBCConversionModule.OnRecvPacket` → `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21`:

```go
contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
if err != nil { return err }
if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
    return err
}
``` [3](#0-2) 

After this, `GetContractByDenom` returns the auto contract, so `ensureDenomNotMapped` returns `ErrDenomAlreadyMapped` for any subsequent `SetExternalContractForDenom` call. [4](#0-3) 

**No admin escape hatch — `DeleteExternalContractForDenom` is external-only:**

The only deletion primitive available to the admin is `DeleteExternalContractForDenom`, which returns immediately if no external contract exists:

```go
func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
    contract, found := k.getExternalContractByDenom(ctx, denom)
    if !found {
        return false   // no-op when only auto contract exists
    }
    ...
}
``` [5](#0-4) 

There is no `DeleteAutoContractForDenom` function exposed to the admin or governance.

**Both admin message and governance proposal are blocked:**

`MsgUpdateTokenMapping` (admin path) calls `RegisterOrUpdateTokenMapping`, which for non-source tokens either calls `DeleteExternalContractForDenom` (no-op) or `SetExternalContractForDenom` (fails with `ErrDenomAlreadyMapped`): [6](#0-5) 

`TokenMappingChangeProposal` (governance path) also calls `RegisterOrUpdateTokenMapping` with the same restriction: [7](#0-6) 

The test suite explicitly confirms this behavior is enforced: [8](#0-7) 

### Impact Explanation

**High — Bypass of Cronos admin and governance authority over token-mapping.**

Once an auto contract is deployed for a denom, neither the `CronosAdmin` account nor a passed `TokenMappingChangeProposal` can ever register an external contract for that denom. This permanently strips admin and governance of their intended authority to manage token mappings — a core security control of the Cronos bridge. If the auto-deployed contract contains a bug, is exploited, or needs to be replaced with a more capable external contract, there is no on-chain remedy short of a chain upgrade.

### Likelihood Explanation

Requires `EnableAutoDeployment` to be `true` (a governance-controlled parameter). When enabled, any unprivileged user can trigger the lock by simply sending a single IBC transfer of any new token denom to Cronos. The attacker needs no special keys, no funds beyond IBC transfer fees, and no coordination. The window is the entire period during which `EnableAutoDeployment` is active. [9](#0-8) 

### Recommendation

1. Expose a `DeleteAutoContractForDenom` keeper method and wire it to `RegisterOrUpdateTokenMapping` so that passing an empty `Contract` field removes both the external **and** auto-deployed mapping for a denom.
2. Alternatively, modify `RegisterOrUpdateTokenMapping` for non-source tokens to atomically retire the existing auto contract when a new external contract is being registered (mirroring the pre-2026-04 behavior that PR #1998 partially addressed).
3. Add an explicit integration test covering the case where an admin attempts to register an external contract after auto-deployment has occurred.

### Proof of Concept

```
1. Governance passes MsgUpdateParams enabling EnableAutoDeployment = true.

2. Unprivileged user sends IBC transfer of new token "ibc/AAAA...":
     → IBCConversionModule.OnRecvPacket
     → ConvertVouchersToEvmCoins
     → ConvertCoinFromNativeToCRC21 (autoDeploy=true, no contract found)
     → DeployModuleCRC21("ibc/AAAA...")
     → SetAutoContractForDenom("ibc/AAAA...", <auto_addr>)
   State: DenomToAutoContract["ibc/AAAA..."] = <auto_addr>

3. Admin sends MsgUpdateTokenMapping{Denom:"ibc/AAAA...", Contract:"0xExternal..."}:
     → RegisterOrUpdateTokenMapping (non-source branch, contract non-empty)
     → SetExternalContractForDenom
     → ensureDenomNotMapped → GetContractByDenom finds <auto_addr>
     → returns ErrDenomAlreadyMapped  ← BLOCKED

4. Admin sends MsgUpdateTokenMapping{Denom:"ibc/AAAA...", Contract:""}:
     → RegisterOrUpdateTokenMapping (non-source branch, contract empty)
     → DeleteExternalContractForDenom
     → getExternalContractByDenom → not found → returns false  ← NO-OP

5. Governance submits TokenMappingChangeProposal{Denom:"ibc/AAAA...", Contract:"0xExternal..."}:
     → NewTokenMappingChangeProposalHandler → RegisterOrUpdateTokenMapping
     → same ErrDenomAlreadyMapped failure  ← BLOCKED

Result: Token mapping for "ibc/AAAA..." is permanently locked to <auto_addr>.
        Admin and governance authority over this mapping is permanently bypassed.
```

// SetExternalContractForDenom sets denom→external CRC21 mapping for an unmapped denom.
// Caller is responsible for source-denom specific validation before calling this method.
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}

func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
	store := ctx.KVStore(k.storeKey)
	contract, found := k.getExternalContractByDenom(ctx, denom)
	if !found {
		return false
	}
	store.Delete(types.DenomToExternalContractKey(denom))
	deleteReverseIfOwned(store, contract, denom)
	if auto, found := k.getAutoContractByDenom(ctx, denom); found {
		bz := store.Get(types.ContractToDenomKey(auto.Bytes()))
		if len(bz) == 0 {
			store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
		} else if existingDenom := string(bz); existingDenom != denom {
			if k.contractOwnedByDenom(ctx, existingDenom, auto) {
				// auto address is already owned by another denom; drop local auto mapping
				store.Delete(types.DenomToAutoContractKey(denom))
			} else {
				// stale reverse entry
				store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
			}
		}
	}
	return true
}

// OnRecvPacket implements the IBCModule interface.
func (im IBCConversionModule) OnRecvPacket(
	ctx sdk.Context,
	channelVersion string,
	packet channeltypes.Packet,
	relayer sdk.AccAddress,
) exported.Acknowledgement {
	cacheCtx, commit := ctx.CacheContext()
	ack := im.app.OnRecvPacket(cacheCtx, channelVersion, packet, relayer)
	if !ack.Success() {
		// Underlying transfer failed: discard cacheCtx writes and return the
		// failure ack. Committing would persist a half-applied transfer.
		return ack
	}
	data, err := transferTypes.UnmarshalPacketData(packet.GetData(), channelVersion, "")
	if err != nil {
		return channeltypes.NewErrorAcknowledgement(errors.Wrap(sdkerrors.ErrUnknownRequest,
			"cannot unmarshal ICS-20 transfer packet data in middleware"))
	}
	denom := im.getIbcDenomFromPacketAndData(packet, data.Token)
	if im.canBeConverted(cacheCtx, denom) {
		transferAmount, ok := sdkmath.NewIntFromString(data.Token.Amount)
		if !ok {
			return channeltypes.NewErrorAcknowledgement(errors.Wrapf(
				transferTypes.ErrInvalidAmount,
				"unable to parse transfer amount (%s) into sdk.Int in middleware",
				data.Token.Amount,
			))
		}
		token := sdk.NewCoin(denom, transferAmount)
		if err := im.cronoskeeper.ConvertVouchersToEvmCoins(cacheCtx, data.Receiver, sdk.NewCoins(token)); err != nil {
			im.cronoskeeper.Logger(ctx).Error(
				"failed to convert vouchers on recv",
				"denom", denom,
				"receiver", data.Receiver,
				"error", err,
			)
			return channeltypes.NewErrorAcknowledgement(err)
		}
	}
	commit()
	return ack
