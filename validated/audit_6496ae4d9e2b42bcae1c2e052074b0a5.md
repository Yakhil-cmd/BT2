### Title
Missing Zero-Amount Guard in `exit_delegation_pool_intent` Enables Griefing via Misleading Event Spam — (File: `src/pool/pool.cairo`)

---

### Summary

`exit_delegation_pool_intent` in `src/pool/pool.cairo` intentionally accepts `amount = 0` to cancel an existing undelegate intent. However, it performs no check that an intent actually exists before proceeding. When called with `amount = 0` and no prior intent, the function emits four misleading on-chain events, makes an unnecessary cross-contract call into the staking contract, and writes unchanged values back to storage — all without reverting. Any pool member can repeat this call indefinitely.

---

### Finding Description

`exit_delegation_pool_intent` computes `total_amount = old_delegated_stake + pool_member_info.unpool_amount` and asserts only `amount <= total_amount`. When `amount = 0` this assertion is trivially satisfied regardless of whether an intent exists. [1](#0-0) 

The function then unconditionally:

1. Calls `undelegate_from_staking_contract_intent(pool_member, 0)`, which forwards to `remove_from_delegation_pool_intent` in the staking contract with `amount = 0`. [2](#0-1) 

2. Inside `remove_from_delegation_pool_intent`, with `old_intent_amount = 0` and `new_intent_amount = 0`, `update_delegated_stake` is a no-op, yet the function still emits `RemoveFromDelegationPoolIntent` (old=0, new=0) and `StakeDelegatedBalanceChanged` (old=new, no change). [3](#0-2) 

3. Back in the pool, `set_member_balance` is called with the unchanged balance, writing a new checkpoint to the `pool_member_epoch_balance` trace.

4. `PoolMemberExitIntent { amount: 0, exit_timestamp: 0 }` and `PoolMemberBalanceChanged { old == new }` are emitted. [4](#0-3) 

The net result per call: four misleading events, two storage writes, and one cross-contract call — with zero actual state change.

The spec lists `AMOUNT_IS_ZERO` as an error for `enter_delegation_pool`, `add_to_delegation_pool`, and `switch_delegation_pool`, but not for `exit_delegation_pool_intent`, leaving this path unguarded. [5](#0-4) 

---

### Impact Explanation

**Medium — Griefing with no profit motive but damage to users or protocol.**

- Off-chain indexers and frontends that consume `PoolMemberExitIntent` events will observe spurious "exit intent registered" signals with `amount = 0` and `exit_timestamp = 0`, potentially displaying false pending-withdrawal states to users.
- `StakeDelegatedBalanceChanged` events with identical old/new values pollute monitoring dashboards and reward-tracking systems.
- Each call inserts a new entry into the `pool_member_epoch_balance` trace (one per epoch). Over many epochs this trace grows, increasing the gas cost of `calculate_rewards` for that pool member — a slow but compounding degradation.
- No funds are at risk; the attack is pure griefing.

---

### Likelihood Explanation

**High.** Any existing pool member (an unprivileged role) can call `exit_delegation_pool_intent(0)` at any time, at the cost of gas only. No token transfer, no approval, and no special state is required. The call succeeds silently.

---

### Recommendation

Add an explicit guard at the top of `exit_delegation_pool_intent` that rejects a zero-amount call when no undelegate intent is currently active:

```cairo
fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
    let pool_member = get_caller_address();
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    // Reject no-op: amount=0 is only meaningful to cancel an existing intent.
    assert!(
        amount.is_non_zero() || pool_member_info.unpool_time.is_some(),
        "{}",
        GenericError::AMOUNT_IS_ZERO,
    );
    ...
}
```

This preserves the legitimate "cancel intent" use-case (amount=0 with an active intent) while preventing the griefing path.

---

### Proof of Concept

1. Alice is a pool member with `delegated_amount = X`, no active undelegate intent.
2. Alice calls `pool.exit_delegation_pool_intent(amount: 0)`.
3. The `amount <= total_amount` assertion passes (`0 <= X`).
4. `undelegate_from_staking_contract_intent(alice, 0)` is called → `remove_from_delegation_pool_intent(staker, alice.into(), 0)` executes in the staking contract.
5. Staking contract emits `RemoveFromDelegationPoolIntent { old_intent_amount: 0, new_intent_amount: 0 }` and `StakeDelegatedBalanceChanged { old == new }`.
6. Pool contract emits `PoolMemberExitIntent { amount: 0, exit_timestamp: 0 }` and `PoolMemberBalanceChanged { old == new }`.
7. Alice's state is unchanged. Alice repeats step 2 every epoch indefinitely. [1](#0-0) [6](#0-5)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }

        fn remove_from_delegation_pool_intent(
            ref self: ContractState,
            staker_address: ContractAddress,
            identifier: felt252,
            amount: Amount,
        ) -> Timestamp {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let staker_info = self.internal_staker_info(:staker_address);
            let pool_contract = get_caller_address();
            let token_address = self
                .staker_pool_info
                .entry(staker_address)
                .get_pool_token(:pool_contract)
                .expect_with_err(Error::CALLER_IS_NOT_POOL_CONTRACT);
            let decimals = self.get_token_decimals(:token_address);
            let normalized_amount = NormalizedAmountTrait::from_native_amount(:amount, :decimals);

            // Update the delegated stake according to the new intent.
            let undelegate_intent_key = UndelegateIntentKey { pool_contract, identifier };
            let old_intent_amount = self.get_pool_exit_intent(:undelegate_intent_key).amount;
            let new_intent_amount = normalized_amount;
            // After this call, the staker balance will be updated.
            let (old_delegated_stake, new_delegated_stake) = self
                .update_delegated_stake(
                    :staker_address,
                    :token_address,
                    :pool_contract,
                    :staker_info,
                    :old_intent_amount,
                    :new_intent_amount,
                );
            self
                .update_undelegate_intent_value(
                    :token_address, :staker_info, :undelegate_intent_key, :new_intent_amount,
                );

            self
                .emit(
                    Events::RemoveFromDelegationPoolIntent {
                        staker_address,
                        pool_contract,
                        token_address,
                        identifier,
                        old_intent_amount: old_intent_amount.to_native_amount(:decimals),
                        new_intent_amount: amount,
                    },
                );
            // If the staker is in the process of unstaking (intent called),
            // an event indicating the staked amount (own and delegated) to be zero
            // had already been emitted, thus unneeded now.
            if staker_info.unstake_time.is_none() {
                self
                    .emit(
                        Events::StakeDelegatedBalanceChanged {
                            staker_address,
                            token_address,
                            old_delegated_stake: old_delegated_stake.to_native_amount(:decimals),
                            new_delegated_stake: new_delegated_stake.to_native_amount(:decimals),
                        },
                    );
            }
            self.get_pool_exit_intent(:undelegate_intent_key).unpool_time
        }
