### Title
Griefing Attack via Unauthorized `exit_delegation_pool_action` Execution Blocking Pool Switch - (File: src/pool/pool.cairo)

### Summary
`exit_delegation_pool_action` in `src/pool/pool.cairo` accepts an arbitrary `pool_member` address and performs **no caller authorization check**. Any unprivileged address can call it on behalf of any pool member once the exit window has elapsed. This allows an attacker to front-run a pool member's `switch_delegation_pool` transaction, forcibly completing the exit and destroying the `unpool_time` state that `switch_delegation_pool` requires, permanently blocking the switch for that intent cycle.

### Finding Description

`exit_delegation_pool_action` is callable by any address with any `pool_member` argument: [1](#0-0) 

No `get_caller_address()` check is performed. The spec explicitly documents this as intentional: [2](#0-1) 

When called, the function clears `unpool_time` and `unpool_amount` from the pool member's record and transfers the tokens to `pool_member`: [3](#0-2) 

`switch_delegation_pool`, however, requires `unpool_time.is_some()` as a hard precondition: [4](#0-3) 

Once an attacker calls `exit_delegation_pool_action(victim)`, `unpool_time` is `None`, and any pending `switch_delegation_pool` transaction from the victim will revert with `MISSING_UNDELEGATE_INTENT`.

### Impact Explanation

A pool member who has declared an exit intent and then decides to redirect their stake to a different staker's pool via `switch_delegation_pool` can be griefed:

1. Their switch is permanently blocked for that intent cycle.
2. They are forced to receive their tokens back and must re-enter the pool from scratch.
3. Re-entry requires a new `enter_delegation_pool` call plus a K-epoch activation delay before the stake earns rewards again, causing a direct loss of unclaimed yield for those epochs.
4. The attacker gains nothing — this is pure griefing.

Impact category: **Griefing with no profit motive but damage to users or protocol (Medium)**, with a secondary **Temporary freezing / loss of unclaimed yield (High)** component for the K-epoch reward gap forced on the victim.

### Likelihood Explanation

- The attack requires only a public RPC call with the victim's address — no capital, no privilege, no leaked key.
- The window of opportunity is any time after `unpool_time` has elapsed while the victim's `switch_delegation_pool` transaction is pending in the mempool.
- On Starknet, transaction ordering is observable, making front-running straightforward for a motivated attacker.
- Any pool member who uses `switch_delegation_pool` (a documented, first-class protocol feature) is a potential victim.

### Recommendation

Restrict `exit_delegation_pool_action` so that only the pool member themselves (or their registered `reward_address`) can trigger it:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let caller = get_caller_address();
    let pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(
        caller == pool_member || caller == pool_member_info.reward_address,
        "{}",
        Error::CALLER_CANNOT_EXIT_POOL,
    );
    // ... rest of function
}
```

This mirrors the access control pattern already applied to `claim_rewards` and `add_to_delegation_pool`. [5](#0-4) 

### Proof of Concept

**Setup:**
- `STAKER_A` has a delegation pool `POOL_A`.
- `STAKER_B` has a delegation pool `POOL_B` (lower commission).
- `VICTIM` is a pool member of `POOL_A` with `amount = 1000`.

**Attack steps:**

1. `VICTIM` calls `POOL_A::exit_delegation_pool_intent(amount: 1000)`.
   - `pool_member_info.unpool_time = Some(T + exit_wait_window)`
   - `pool_member_info.unpool_amount = 1000`

2. Time advances past `T + exit_wait_window`.

3. `VICTIM` submits `POOL_A::switch_delegation_pool(to_staker: STAKER_B, to_pool: POOL_B, amount: 1000)` to the mempool.

4. `ATTACKER` observes the pending transaction and front-runs it by calling:
   `POOL_A::exit_delegation_pool_action(pool_member: VICTIM)`
   - No caller check — succeeds immediately.
   - `pool_member_info.unpool_time = None`, `unpool_amount = 0`.
   - 1000 tokens transferred back to `VICTIM`.

5. `VICTIM`'s `switch_delegation_pool` transaction executes and reverts:
   ```
   assert!(pool_member_info.unpool_time.is_some(), ...) // FAILS: unpool_time is None
   ``` [4](#0-3) 

6. `VICTIM` must now re-approve tokens, call `enter_delegation_pool` on `POOL_B`, and wait K epochs before earning rewards — losing yield for the entire re-entry delay period. [6](#0-5)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
```
