### Title
Unbounded Loop in `calculate_rewards` Enables Permanent Freezing of Delegator Funds - (File: src/pool/pool.cairo)

---

### Summary

The `calculate_rewards` internal function in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop that iterates over every entry in a pool member's `pool_member_epoch_balance` trace. A delegator who repeatedly changes their delegated balance across many epochs without claiming rewards accumulates an unbounded number of trace entries. Once the trace is large enough, any call to `claim_rewards` (and by extension `exit_action`, which must settle rewards before returning principal) will exceed the Starknet block gas limit, permanently freezing the delegator's unclaimed yield and staked principal.

---

### Finding Description

In `src/pool/pool.cairo`, the internal function `calculate_rewards` iterates over the entire `pool_member_epoch_balance` trace from `entry_to_claim_from` to the current epoch:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The trace grows by one entry per epoch in which the pool member changes their balance (via `add_to_delegation_pool` or `exit_intent`). The `entry_to_claim_from` cursor is only advanced when `claim_rewards` succeeds and writes back `pool_member_info.entry_to_claim_from`. If a delegator never calls `claim_rewards`, the cursor stays at zero and every subsequent call must iterate the entire accumulated trace. [2](#0-1) 

The trace insert logic in `src/pool/pool_member_balance_trace/trace.cairo` appends a new checkpoint whenever the epoch key differs from the last entry, so one balance change per epoch = one new entry per epoch. [3](#0-2) 

`claim_rewards` in pool.cairo is the public entry point that calls `calculate_rewards` and then transfers rewards to the reward address: [4](#0-3) 

---

### Impact Explanation

Once the trace length exceeds the per-transaction gas budget, every call to `claim_rewards` reverts. Because `exit_action` must also settle rewards before returning the delegator's principal, it too will revert. The delegator's staked tokens and all accumulated unclaimed yield become permanently inaccessible — matching the allowed impact **"Permanent freezing of unclaimed yield or unclaimed royalties"** and **"Temporary/permanent freezing of funds"**.

---

### Likelihood Explanation

- A delegator who makes one balance change per epoch and never calls `claim_rewards` accumulates one trace entry per epoch. Starknet epochs are short (hours), so over months of active delegation the trace can grow into the thousands.
- The developers themselves flag this in a code comment: *"This loop is unbounded but unlikely to exceed gas limits"* — acknowledging the risk while betting on practical limits.
- No privileged access is required. Any unprivileged delegator can reach this state through normal protocol usage (`add_to_delegation_pool` / `exit_intent` calls).
- The condition is irreversible once reached: there is no administrative function to truncate the trace or split the claim into batches.

---

### Recommendation

1. **Checkpoint on every balance change**: Update `pool_member_info.reward_checkpoint` and `entry_to_claim_from` atomically whenever the balance changes (inside `add_to_delegation_pool` and `exit_intent`), so the loop only ever needs to cover the entries since the last balance change (at most K entries, where K is the epoch delay constant).
2. **Alternatively, cap the loop**: Introduce a maximum iteration count per call and allow partial reward claims, storing the updated `entry_to_claim_from` so subsequent calls continue from where the previous one stopped.
3. **Enforce periodic claiming**: Require `claim_rewards` to be called before any balance change, keeping the trace length bounded to at most K entries at any time.

---

### Proof of Concept

1. Delegator Alice calls `add_to_delegation_pool` once per epoch for N epochs (N = enough to exceed gas limit, e.g., several thousand), never calling `claim_rewards`.
2. Each call appends one entry to `pool_member_epoch_balance` via `set_member_balance` → `trace.insert`.
3. Alice calls `claim_rewards`. `calculate_rewards` enters the `while` loop with `entry_to_claim_from = 0` and `pool_member_trace_length = N`.
4. The loop executes N iterations, each reading a storage slot (`pool_member_trace.at(entry_to_claim_from)`) and calling `find_sigma`. The transaction runs out of gas and reverts.
5. Alice calls `exit_intent` and waits for the exit window, then calls `exit_action`. `exit_action` internally settles rewards via the same `calculate_rewards` path and also reverts.
6. Alice's staked principal and all unclaimed yield are permanently frozen in the pool contract. [5](#0-4)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
