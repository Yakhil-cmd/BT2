### Title
Unprivileged Caller Can Permanently Freeze Any Staker's Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable by any non-zero address. It accepts a caller-controlled `disable_rewards: bool` parameter. Critically, the `last_reward_block` storage variable is written **before** the `disable_rewards` guard is evaluated. An attacker can call `update_rewards(victim_staker, disable_rewards: true)` once per block to permanently deny that staker's consensus-era block rewards for that block, with no cost beyond gas.

---

### Finding Description

`update_rewards` is exposed as a public ABI function with no caller restriction beyond the generic `general_prerequisites()` check (contract not paused, caller not zero). [1](#0-0) 

`general_prerequisites` only asserts the contract is unpaused and the caller is non-zero: [2](#0-1) 

Inside `update_rewards`, the per-block guard is enforced and then `last_reward_block` is unconditionally written to the current block number **before** the `disable_rewards` branch is evaluated: [3](#0-2) 

The sequence is:

1. Assert `current_block_number > last_reward_block` (line 1454–1458) — ensures only one call per block succeeds.
2. **Write** `last_reward_block = current_block_number` (line 1485) — consumes the block's reward slot.
3. `if disable_rewards { return; }` (line 1487) — exits without distributing any rewards.

Because step 2 happens before step 3, any subsequent call to `update_rewards` for the same block will fail with `REWARDS_ALREADY_UPDATED`. The staker's rewards for that block are permanently lost — they are never accrued to `unclaimed_rewards_own` and never forwarded to the reward supplier.

---

### Impact Explanation

**Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(victim_staker, disable_rewards: true)` once per block, for every block in an epoch, causes the victim staker to receive **zero** consensus block rewards for the entire epoch. The lost rewards are not deferred or recoverable; the `last_reward_block` slot is consumed and the reward calculation path is skipped entirely. Pool members delegating to the victim staker are equally affected, since pool rewards flow through the same `_update_rewards` path.

---

### Likelihood Explanation

**High.** The function is permissionlessly callable by any non-zero address. The attacker needs only to submit a transaction once per block targeting the victim staker. There is no economic barrier — the attacker spends only gas. The attack can be automated and sustained indefinitely. A competitor staker, a griefing actor, or anyone with a motive to harm a specific staker can execute this continuously.

---

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a call with `disable_rewards: true` does not consume the block's reward slot:

```cairo
// Update last block rewards ONLY when rewards are actually processed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... proceed with reward calculation
```

Alternatively, restrict who may pass `disable_rewards: true` (e.g., only the staker itself or a designated operator), or remove the `disable_rewards` parameter from the public interface entirely and handle the disabling logic internally.

---

### Proof of Concept

1. Staker `S` is registered and active in the consensus-rewards era.
2. At block `B`, attacker `A` (any non-zero address) calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. Inside `update_rewards`:
   - `current_block_number (B) > last_reward_block` → assertion passes.
   - `last_reward_block` is written to `B`.
   - `disable_rewards == true` → function returns early; no rewards are computed or accrued.
4. Any subsequent call to `update_rewards` for block `B` (by `S` or anyone else) fails: `current_block_number (B) > last_reward_block (B)` is false → `REWARDS_ALREADY_UPDATED`.
5. Staker `S` permanently loses block rewards for block `B`.
6. Attacker repeats step 2 at every block, causing `S` to earn zero rewards indefinitely. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
