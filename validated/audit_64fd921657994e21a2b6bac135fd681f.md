### Title
Permissionless `update_rewards` with caller-controlled `disable_rewards` flag allows griefing of staker reward distribution — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is callable by any address and accepts a caller-controlled `disable_rewards: bool` parameter. A malicious actor can call this function with `disable_rewards: true` to consume the global `last_reward_block` slot for the current block without distributing any rewards, permanently blocking legitimate reward distribution for that block.

---

### Finding Description

`update_rewards` is exposed as a public ABI function via `StakingRewardsManagerImpl` with no caller access control beyond the general pause check. It accepts a `staker_address` and a `disable_rewards: bool` from the caller.

The critical ordering flaw is that `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← committed unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing rewards
}
``` [1](#0-0) 

The function enforces a strict one-call-per-block invariant:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

Because `last_reward_block` is a **single global value** (not per-staker), any call that writes it — even one that distributes nothing — exhausts the slot for the entire block. A malicious actor who calls `update_rewards(any_valid_staker, disable_rewards: true)` first in a block:

1. Passes all staker-existence and balance checks (staker address is public from emitted events).
2. Writes `last_reward_block = current_block`.
3. Returns early, distributing zero rewards.
4. Causes every subsequent legitimate call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

The attacker can repeat this every block at the cost of one cheap Starknet transaction per block.

---

### Impact Explanation

This matches **"Permanent freezing of unclaimed yield or unclaimed royalties" (High)** and **"Griefing with no profit motive but damage to users or protocol" (Medium)**.

- **Per-block**: one attacker transaction silently voids all stakers' block rewards for that block.
- **Sustained**: an attacker running this every block permanently prevents the entire staker set from accumulating consensus-phase block rewards. The `unclaimed_rewards` in `RewardSupplier` never grows from block rewards, and stakers' `unclaimed_rewards_own` fields are never incremented. [3](#0-2) 

---

### Likelihood Explanation

- `update_rewards` is `#[abi(embed_v0)]` — fully permissionless, no role check.
- Valid staker addresses are emitted in `NewStaker` events and are trivially discoverable on-chain.
- Starknet transaction fees are low; sustaining one tx/block is economically viable for a motivated griefer or competitor. [4](#0-3) 

---

### Recommendation

Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a no-op call does not consume the block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only commit last_reward_block when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... distribute rewards ...
```

Alternatively, restrict `update_rewards` to a trusted caller (e.g., the attestation contract or a designated operator role) so that the `disable_rewards` flag cannot be weaponized by an arbitrary address.

---

### Proof of Concept

1. Attacker reads any active `staker_address` `S` from a past `NewStaker` event.
2. At block `N`, attacker front-runs the legitimate reward-distribution call by submitting `update_rewards(S, disable_rewards: true)`.
3. `last_reward_block` is set to `N`; no rewards are distributed.
4. Any honest call to `update_rewards` in block `N` reverts with `REWARDS_ALREADY_UPDATED`.
5. Attacker repeats step 2–4 every block, permanently suppressing all block-reward accrual for all stakers in the consensus phase.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
