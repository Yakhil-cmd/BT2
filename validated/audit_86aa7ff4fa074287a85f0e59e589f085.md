### Title
STRK Reward Tokens Permanently Frozen in Pool Contract When Pool Balance Falls Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo)

### Summary
The `Pool` contract receives STRK reward tokens from the staking contract even when the total pool balance is below `min_delegation_for_rewards`. Because `compute_rewards_per_unit` returns zero in that case, the cumulative rewards trace is never updated and no pool member can ever claim those tokens. There is no admin withdrawal or recovery function in the pool contract, so the transferred STRK is permanently frozen.

### Finding Description
The reward distribution flow in `_update_rewards` (staking.cairo) always transfers the calculated pool rewards to the pool contract via `send_rewards_to_delegation_pool`, regardless of whether the pool balance meets the minimum threshold: [1](#0-0) 

Inside `update_pool_rewards`, `send_rewards_to_delegation_pool` unconditionally calls `checked_transfer` to move STRK tokens into the pool contract: [2](#0-1) 

Immediately after, `update_rewards_from_staking_contract` is called on the pool, which invokes `compute_rewards_per_unit`. The pool contract's own code explicitly documents the problem: [3](#0-2) 

When `total_stake < min_delegation_for_rewards` (1 STRK = `10^18` fractions for STRK pools), `compute_rewards_per_unit` returns zero, so the `cumulative_rewards_trace` is updated with a zero increment: [4](#0-3) 

Because the trace never advances, `calculate_rewards` will compute zero rewards for every pool member for those epochs, and the STRK tokens already sitting in the pool contract balance are unclaimable. The `Pool` contract exposes no admin withdrawal, sweep, or recovery function anywhere in its interface. [5](#0-4) 

### Impact Explanation
STRK reward tokens are transferred into the pool contract but can never be claimed by any pool member. They accumulate silently with each reward epoch during which the pool balance remains below the threshold. This constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

### Likelihood Explanation
The minimum delegation check in `enter_delegation_pool` only requires `amount > 0`: [6](#0-5) 

Any delegator can enter with 1 fri (1 unit of STRK), far below `min_delegation_for_rewards = 10^18`. The existing flow test `delegate_min_strk_for_rewards_flow_test` confirms that a pool with balance just below the threshold receives rewards but distributes zero to members: [7](#0-6) 

This is a realistic, low-effort condition: a single delegator entering with a dust amount is sufficient to trigger it for every subsequent reward epoch until the pool balance crosses the threshold.

### Recommendation
Either:
1. **Skip the token transfer** in `update_pool_rewards` when `pool_rewards` is zero (i.e., when the pool balance is below `min_delegation_for_rewards`), so no tokens are sent to the pool contract in the first place; or
2. **Add a recovery path** in the pool contract (e.g., an admin-callable sweep of the surplus STRK balance that exceeds the sum of all claimable rewards).

Option 1 is simpler and eliminates the root cause. The staking contract already has the pool balance available at the call site and can gate the transfer.

### Proof of Concept
1. Staker stakes and opens a STRK delegation pool.
2. Delegator enters the pool with `amount = 10^18 - 1` (just below `min_delegation_for_rewards`).
3. Staker attests; `_update_rewards` runs, calculates non-zero `pool_rewards`, and calls `send_rewards_to_delegation_pool` — STRK tokens land in the pool contract.
4. `update_rewards_from_staking_contract` is called; `compute_rewards_per_unit` returns 0 because `pool_balance < min_delegation_for_rewards`.
5. `cumulative_rewards_trace` is updated with `last + 0`.
6. Delegator calls `claim_rewards` → receives 0 STRK.
7. The STRK tokens transferred in step 3 remain in the pool contract with no mechanism to recover them.

This matches the existing test assertion at line 1734–1735 of `src/flow_test/test.cairo`, which confirms `rewards == Zero::zero()` while the pool contract holds a non-zero STRK balance. [8](#0-7)

        fn update_rewards_from_staking_contract(
            ref self: ContractState, rewards: Amount, pool_balance: Amount,
        ) {
            self.assert_caller_is_staking_contract();

            // `rewards_info` is initialized in the constructor or in the upgrade proccess,
            // so unwrapping should be safe.
            let (_, last) = self.cumulative_rewards_trace.last().unwrap();
            self
                .cumulative_rewards_trace
                .insert(
                    key: self.get_current_epoch(),
                    value: last
                        + self
                            .compute_rewards_per_unit(
                                staking_rewards: rewards, total_stake: pool_balance,
                            ),
                );
        }

        /// Compute the rewards for the pool trace.
        ///
        /// `staking_rewards` is in `STRK_DECIMALS` decimals.
        /// `total_stake` is in the contract's token decimals.
        /// **Note**: Delegation rewards lost when pool balance is less than
        /// `min_delegation_for_rewards`. The staking contract continues to forward
        /// `pool_rewards` to the pool contract even in this case.
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }

fn delegate_min_strk_for_rewards_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    let stake_amount = system.staking.get_min_stake();
    let staker = system.new_staker(amount: stake_amount);
    system.stake(:staker, amount: stake_amount, pool_enabled: true, commission: Zero::zero());
    let pool = system.staking.get_pool(:staker);

    // Setup delegator.
    let delegate_amount = Pool::STRK_CONFIG.min_for_rewards;
    let delegator = system.new_delegator(amount: delegate_amount);

    // Enter pool with less than min STRK for rewards.
    system.delegate(:delegator, :pool, amount: delegate_amount - 1);

    // Attest.
    system.advance_k_epochs_and_attest(:staker);
    system.advance_epoch();

    // Check rewards.
    system.advance_epoch();
    let rewards = system.delegator_claim_rewards(:delegator, :pool);
    assert!(rewards == Zero::zero());

    // Add to delegation.
    system.add_to_delegation_pool(:delegator, :pool, amount: 1);

    // Attest.
    system.advance_k_epochs_and_attest(:staker);

    // Check rewards.
    system.advance_epoch();
    let rewards = system.delegator_claim_rewards(:delegator, :pool);
    assert!(rewards > Zero::zero());
