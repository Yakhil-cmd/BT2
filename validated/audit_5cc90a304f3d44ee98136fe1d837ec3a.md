### Title
Unbounded Loop in `calculate_rewards` Over Attacker-Inflatable `pool_member_epoch_balance` Trace Can Permanently Freeze Delegator's Unclaimed Yield — (File: `src/pool/pool.cairo`)

---

### Summary

The `calculate_rewards` function in the Pool contract iterates over every entry in a delegator's `pool_member_epoch_balance` trace without any cap. A delegator can inflate this trace by repeatedly calling `add_to_delegation_pool` (or `exit_delegation_pool_intent`) across many epochs. Once the trace is large enough, any call to `claim_rewards` will exceed Starknet's gas limit and revert, permanently freezing the delegator's unclaimed yield.

---

### Finding Description

`calculate_rewards` in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch { break; }
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The loop iterates from `entry_to_claim_from` (the last claimed index, stored in `pool_member_info`) up to `pool_member_trace_length`. Each iteration reads a storage slot and calls `find_sigma`, making it storage-heavy.

The trace grows via `set_member_balance`, which calls `trace.insert(key: self.get_epoch_plus_k(), ...)`: [2](#0-1) 

The `insert` function in the trace appends a **new checkpoint** whenever the epoch key differs from the last entry's key: [3](#0-2) 

This means every call to `add_to_delegation_pool` or `exit_delegation_pool_intent` in a **different epoch** appends one new entry to the trace. There is no maximum trace length enforced anywhere.

`claim_rewards` calls `calculate_rewards` with the stored `entry_to_claim_from`, which only advances after a successful claim: [4](#0-3) 

If a delegator never claims (or claims infrequently), `entry_to_claim_from` stays low while the trace grows, causing the loop to iterate over an ever-increasing number of entries on the next claim.

---

### Impact Explanation

Once the trace is large enough that iterating from `entry_to_claim_from` to `pool_member_trace_length` exceeds the Starknet transaction gas limit, `claim_rewards` will **always revert**. Because `entry_to_claim_from` is only updated inside a successful `claim_rewards` call, there is no way to advance it in smaller steps. The delegator's accumulated unclaimed yield is **permanently frozen** with no recovery path.

This matches the allowed High impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- Any delegator can trigger this on themselves (or be a victim of their own inaction).
- The minimum cost per trace entry is 1 wei of the pool token plus gas for `add_to_delegation_pool`.
- Starknet epochs are on the order of days/weeks; a long-term delegator who adjusts their delegation frequently (e.g., dollar-cost averaging) and defers claiming will naturally accumulate hundreds of entries over months/years.
- The developers themselves acknowledge the loop is unbounded (the comment at line 858 says "unlikely to exceed gas limits" — an assumption, not a guarantee).
- No privileged role is required; any pool member can reach this state. [5](#0-4) 

---

### Recommendation

1. **Enforce a maximum trace length**: Reject `add_to_delegation_pool` / `exit_delegation_pool_intent` if the trace already has more than a safe maximum number of unclaimed entries (e.g., 500).
2. **Alternatively, allow partial reward claims**: Expose a function that processes at most `N` trace entries per call, advancing `entry_to_claim_from` incrementally, so a delegator with a large trace can drain it over multiple transactions.
3. **Remove the "unlikely" assumption**: The comment at line 858 should be replaced with an enforced invariant.

---

### Proof of Concept

```
1. Delegator D enters pool P with 1000 STRK.
2. Each epoch for E epochs, D calls add_to_delegation_pool(pool_member: D, amount: 1).
   - Each call in a new epoch appends one entry to pool_member_epoch_balance[D].
   - After E epochs, trace length = E + 1 (initial entry + E additions).
3. D never calls claim_rewards during this period.
   - entry_to_claim_from remains at 0.
4. After E is large enough (empirically determinable from Starknet's gas limit
   and the per-iteration cost of reading storage + find_sigma), D calls claim_rewards.
5. calculate_rewards loops E times, each iteration reading storage and calling find_sigma.
6. Transaction exceeds gas limit → reverts.
7. D's accumulated rewards are permanently inaccessible.
``` [6](#0-5) [7](#0-6)

                );

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
