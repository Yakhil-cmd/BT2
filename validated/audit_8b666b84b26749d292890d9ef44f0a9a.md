### Title
Staker Can Instantly Increase Commission to `max_commission` During Active Commitment, Stealing Delegator Yield — (File: `src/staking/staking.cairo`)

---

### Summary

The `CommissionCommitment` mechanism in `staking.cairo` is intended to give delegators a guarantee about the maximum commission a staker will charge. However, when a commitment is active, the staker is permitted to **increase** commission to any value up to `max_commission` in a single transaction with no time delay. A staker can attract delegators with a low commission, set a commitment with `max_commission = 10000` (100%), and immediately raise commission to 100%, capturing all pool rewards and leaving delegators with zero yield.

The developers themselves acknowledge this in a code comment directly above `set_commission_commitment`:

> `/// **Note**: Current commission increase safeguards still allow for sudden commission changes.`

---

### Finding Description

Without an active commitment, `update_commission` enforces a strict monotone-decrease invariant:

```cairo
} else {
    assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
}
``` [1](#0-0) 

When a commitment **is** active, this invariant is replaced by a ceiling check only:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
}
``` [2](#0-1) 

`set_commission_commitment` only requires `max_commission >= current_commission`:

```cairo
assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
``` [3](#0-2) 

There is no upper bound on `max_commission` relative to `current_commission`, no minimum time between commission changes, and no gradual-increase limit. A staker with `commission = 100` (1%) can set `max_commission = 10000` (100%) and then call `set_commission(10000)` in the very next transaction.

`is_commission_commitment_active` is a simple epoch comparison with no additional guards:

```cairo
fn is_commission_commitment_active(
    self: @ContractState, commission_commitment: CommissionCommitment,
) -> bool {
    self.get_current_epoch() < commission_commitment.expiration_epoch
}
``` [4](#0-3) 

The commission is applied inside `split_rewards_with_commission` at reward distribution time, so any increase takes effect on the very next reward cycle.

---

### Impact Explanation

**High — Theft of unclaimed yield.**

When commission is raised to 100%, the pool receives zero rewards. All staking yield that would have accrued to delegators is instead captured by the staker. Delegators who entered the pool based on a low advertised commission rate have their future yield stolen. The staker profits directly and proportionally to the delegated stake.

---

### Likelihood Explanation

Any staker who has opened a delegation pool can execute this attack permissionlessly:

1. The staker controls both `set_commission` and `set_commission_commitment`.
2. No governance approval, no time-lock, and no co-signer is required.
3. The attack is atomic: commitment set in one transaction, commission raised in the next.
4. Delegators have no on-chain mechanism to exit before the commission change takes effect within the same epoch.

The attack is economically rational whenever the delegated pool balance is large enough to make the captured yield exceed the reputational cost.

---

### Recommendation

1. **Prohibit commission increases during a commitment period.** The commitment should only allow decreases (or no change), not increases. Remove the ability to set `commission > old_commission` while a commitment is active.
2. **Cap the gap between `current_commission` and `max_commission`.** For example, `max_commission <= current_commission + delta` for some protocol-defined `delta`.
3. **Enforce a minimum notice period before a commission increase takes effect** (e.g., at least K epochs after the change is announced), giving delegators time to exit.
4. **Require `max_commission` to equal `current_commission`** if the intent is purely to allow future decreases, which is the semantically correct interpretation of a "commitment."

---

### Proof of Concept

```
Epoch N:
  staker calls set_commission(commission: 100)          // 1% commission, attracts delegators
  staker calls set_commission_commitment(
      max_commission: 10000,                            // 100% ceiling — passes: 100 <= 10000
      expiration_epoch: N + 52                         // ~1 year commitment
  )

Epoch N (same or next block):
  delegators observe 1% commission and join the pool

Epoch N (immediately after):
  staker calls set_commission(commission: 10000)        // passes: 10000 <= 10000, 10000 != 100
  // Commission is now 100%

Epoch N+1 onward:
  All pool rewards flow entirely to the staker.
  Delegators receive zero yield on their staked tokens.
```

The entry path is fully permissionless: `set_commission` → `set_commission_commitment` → `set_commission` (increase). No privileged role, no external dependency, no bridge interaction required. [5](#0-4) [6](#0-5) [7](#0-6)

        /// **Note**: Current commission increase safeguards still allow for sudden commission
        /// changes.
        /// **Note**: Updating epoch info can impact the commission commitment expiration date.
        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
