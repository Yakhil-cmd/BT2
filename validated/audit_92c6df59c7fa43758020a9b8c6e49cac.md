### Title
Gateway Stateful Validator Checks Only L2 Gas Price Against Threshold, Admitting Transactions With Zero L1/L1-Data Gas Prices That Will Always Fail Execution - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`validate_tx_l2_gas_price_within_threshold` validates only the `l2_gas.max_price_per_unit` field against the previous block's L2 gas price threshold. It explicitly skips L1 gas and L1 data gas price validation (a `TODO` in the code acknowledges this). A transaction with `l1_gas.max_price_per_unit = 0` and/or `l1_data_gas.max_price_per_unit = 0` passes every gateway check and is admitted to the mempool, but will deterministically fail `check_fee_bounds` inside `perform_pre_validation_stage` at execution time. This is the direct sequencer analog of the `canHedge` bug: the check validates one dimension of the resource-bounds vector (L2 gas price direction) but ignores the magnitude of the other two dimensions, breaking the admission invariant that only transactions viable for execution should enter the mempool.

### Finding Description

**Stateless path** (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, lines 56–88):

```rust

```

Only `l2_gas.max_price_per_unit` is compared against the static `min_gas_price`. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never checked.

`check_fee_bounds` iterates over all three resources (L1Gas, L1DataGas, L2Gas) and rejects the transaction if any price is below the block's actual gas price. A transaction with `l1_gas.max_price_per_unit = 0` will always fail here.

### Impact Explanation

The gateway's admission invariant is: *only transactions that can plausibly be executed in the next block should enter the mempool*. By checking only the L2 gas price dimension, the gateway admits transactions that are structurally guaranteed to fail `check_fee_bounds` at execution time. This maps directly to the High impact scope item: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

An attacker can flood the mempool with transactions that pass all gateway checks but will never execute, consuming mempool capacity, batcher CPU, and state-reader resources without paying any fee (since the transactions are rejected before fee transfer).

### Likelihood Explanation

Exploitation requires only crafting a valid V3 `InvokeTransaction` or `DeclareTransaction` with `AllResourceBounds` where `l1_gas.max_price_per_unit = 0` and `l2_gas.max_price_per_unit >= min_gas_price_percentage * previous_block_l2_gas_price / 100`. This is trivially constructable by any unprivileged user via the public RPC endpoint. No special account, key, or privileged access is required.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (and the stateless `validate_resource_bounds`) to check all three gas prices against their respective current-block prices, mirroring the logic already present in `check_fee_bounds`:

```rust
// For AllResources, check all three prices:
let GasPriceVector { l1_gas_price, l1_data_gas_price, l2_gas_price } =
    block_info.gas_prices.gas_price_vector(fee_type);
for (price_field, block_price) in [
    (tx_resource_bounds.l1_gas.max_price_per_unit, l1_gas_price),
    (tx_resource_bounds.l1_data_gas.max_price_per_unit, l1_data_gas_price),
    (tx_resource_bounds.l2_gas.max_price_per_unit, l2_gas_price),
] {
    let threshold = (percentage * block_price.get().0).to_integer();
    if price_field.0 < threshold {
        return Err(...);
    }
}
```

### Proof of Concept

1. Obtain the previous block's L2 gas price `P_l2` from the RPC.
2. Construct a V3 `InvokeTransaction` with `AllResourceBounds`:
   - `l2_gas.max_price_per_unit = P_l2` (passes stateful threshold check)
   - `l2_gas.max_amount = 1`
   - `l1_gas.max_price_per_unit = 0` ← not validated at gateway
   - `l1_data_gas.max_price_per_unit = 0` ← not validated at gateway
   - `l1_gas.max_amount = 1`, `l1_data_gas.max_amount = 1`
3. Submit via `add_transaction` RPC.
4. **Stateless check passes**: `max_possible_fee(Tip::ZERO) = P_l2 * 1 > 0`; `l2_gas.max_price_per_unit >= min_gas_price`.
5. **Stateful check passes**: `l2_gas.max_price_per_unit >= threshold`.
6. **Mempool `validate_tx` passes**: nonce and fee-escalation checks are unrelated to gas prices.
7. Transaction is admitted to the mempool.
8. At execution time, `check_fee_bounds` computes `actual_l1_gas_price > 0 > l1_gas.max_price_per_unit = 0` and returns `ResourceBoundsError::MaxGasPriceTooLow`, rejecting the transaction without executing it or charging a fee.
9. Repeat at high rate to exhaust mempool capacity with permanently-invalid transactions. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
