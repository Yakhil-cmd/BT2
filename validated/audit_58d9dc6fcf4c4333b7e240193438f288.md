### Title
Missing Zero-Address Validation for `reward_address` Allows Temporary Freezing of Funds - (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary
Both `staking.cairo` and `pool.cairo` accept a zero address as `reward_address` in `stake`, `enter_delegation_pool`, and `change_reward_address` without any zero-address guard. Because reward disbursement (`send_rewards_to_staker`, `claim_rewards`) unconditionally calls `checked_transfer(recipient: reward_address, ...)`, a zero `reward_address` causes every reward-claiming path — including `unstake_action` — to revert, temporarily freezing both unclaimed yield and the staker's principal until the address is corrected.

---

### Finding Description

Neither `stake` nor `change_reward_address` in `staking.cairo` validates that `reward_address` is non-zero: [1](#0-0) 

The only guard present is `REWARD_ADDRESS_IS_TOKEN`, which checks the address is not a registered token contract — it does not reject `address(0)`. [2](#0-1) 

The same omission exists in `pool.cairo`'s `enter_delegation_pool` and `change_reward_address`: [3](#0-2) [4](#0-3) 

When rewards are disbursed, `send_rewards_to_staker` calls `checked_transfer` directly to the stored `reward_address`: [5](#0-4) 

And `claim_rewards` in the pool does the same: [6](#0-5) 

The STRK token is a standard OpenZeppelin ERC20 on Starknet, which reverts on `transfer` to `address(0)`. Therefore, any call that routes through `send_rewards_to_staker` or `pool.claim_rewards` will revert when `reward_address` is zero.

Critically, `unstake_action` calls `send_rewards_to_staker` **before** returning the principal: [7](#0-6) 

This means a staker with `reward_address = 0` cannot execute `unstake_action` — their principal stake is frozen until they call `change_reward_address` with a valid address.

---

### Impact Explanation

**High — Temporary freezing of funds.**

A staker or pool member who sets (or initially registers with) `reward_address = 0` will find that:
1. `claim_rewards` always reverts → unclaimed yield is inaccessible.
2. `unstake_action` always reverts → the staker's principal STRK is inaccessible.

The freeze is temporary because `change_reward_address` has no restrictions and can be called at any time to restore a valid address. However, until that correction is made, both yield and principal are locked.

---

### Likelihood Explanation

**Medium.** The entry path is a single public call — `stake(reward_address: 0, ...)` or `change_reward_address(0)` — with no prerequisite other than being a registered staker or pool member. A user error (e.g., passing an uninitialized variable) or a front-end bug could trivially produce this state. The protocol provides no on-chain protection.

---

### Recommendation

Add a non-zero guard in all four entry points:

```cairo
// staking.cairo: stake() and change_reward_address()
assert!(reward_address.is_non_zero(), "Reward address cannot be zero");

// pool.cairo: enter_delegation_pool() and change_reward_address()
assert!(reward_address.is_non_zero(), "Reward address cannot be zero");
``` [8](#0-7) [9](#0-8) 

---

### Proof of Concept

1. Staker calls `staking.stake(reward_address: 0, operational_address: X, amount: MIN_STAKE)`. No revert — zero address passes all existing checks.
2. Staker accumulates rewards over several epochs.
3. Staker calls `staking.unstake_intent()`. Succeeds.
4. After the exit window, anyone calls `staking.unstake_action(staker_address)`.
5. Execution reaches `send_rewards_to_staker` → `checked_transfer(recipient: 0, amount: rewards)` → **reverts** (OZ ERC20 rejects zero recipient).
6. The staker's principal STRK remains locked in the staking contract. The staker must call `change_reward_address(valid_address)` and retry `unstake_action` to recover. [10](#0-9) [11](#0-10)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());

        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            assert!(
                self.token_dispatcher.contract_address.read() != reward_address,
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_address = pool_member_info.reward_address;

            // Update reward_address and commit to storage.
            pool_member_info.reward_address = reward_address;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardAddressChanged {
                        pool_member, new_address: reward_address, old_address,
                    },
                );
        }
