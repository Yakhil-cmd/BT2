### Title
Permissioned Address Can Override Admin's Token Mapping via Delete-Then-Set, Corrupting IBC Voucher Conversion - (File: x/cronos/keeper/msg_server.go)

### Summary
`MsgUpdateTokenMapping` uses `HasPermission`, which grants equal write access to both the `CronosAdmin` and any address holding `CanChangeTokenMapping`. Because `RegisterOrUpdateTokenMapping` exposes a delete path (empty contract) followed by a set path, a permissioned address can atomically erase the admin's legitimate denom→contract binding and substitute a malicious contract. All subsequent IBC voucher conversions for that denom then route through the attacker-controlled contract, permanently trapping users' IBC vouchers.

### Finding Description

`UpdateTokenMapping` in `x/cronos/keeper/msg_server.go` gates access with a single `HasPermission` call:

```go
// x/cronos/keeper/msg_server.go:69-82
func (k msgServer) UpdateTokenMapping(...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, ...)
    }
    if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil { ... }
}
```

`HasPermission` returns `true` for **either** the `CronosAdmin` **or** any address whose stored permission bits include `CanChangeTokenMapping`:

```go
// x/cronos/keeper/permissions.go:32-48
func (k Keeper) HasPermission(...) bool {
    admin := k.GetParams(ctx).CronosAdmin
    for _, account := range accounts {
        if admin == account.String() { return true }
        permission := k.GetPermissions(ctx, account)
        if permission&permissionsToCheck == permissionsToCheck { return true }
    }
    return false
}
```

`RegisterOrUpdateTokenMapping` exposes two distinct sub-paths for non-source denoms:

```go
// x/cronos/keeper/keeper.go:384-401
if len(msg.Contract) == 0 {
    k.DeleteExternalContractForDenom(ctx, msg.Denom)   // ← delete path
} else {
    ...
    k.SetExternalContractForDenom(ctx, msg.Denom, contract) // ← set path
}
```

`SetExternalContractForDenom` calls `ensureDenomNotMapped`, which rejects overwrites of an existing mapping. However, the delete path removes that guard entirely. A permissioned address therefore executes a two-transaction sequence:

1. **Tx 1**: `MsgUpdateTokenMapping{Denom: ibcDenom, Contract: ""}` → `DeleteExternalContractForDenom` removes the admin's legitimate binding.
2. **Tx 2**: `MsgUpdateTokenMapping{Denom: ibcDenom, Contract: maliciousAddr}` → `SetExternalContractForDenom` succeeds because the denom is now unmapped.

After Tx 2, `GetContractByDenom` returns the attacker's contract for `ibcDenom`.

### Impact Explanation

Every subsequent IBC voucher receipt for `ibcDenom` flows through:

```
OnRecvVouchers → ConvertVouchersToEvmCoins → ConvertCoinFromNativeToCRC21
```

Inside `ConvertCoinFromNativeToCRC21` (non-source path):

```go
// x/cronos/keeper/evm.go:131-138
err = k.bankKeeper.SendCoins(ctx, sdk.AccAddress(sender.Bytes()),
    sdk.AccAddress(contract.Bytes()), coins)   // IBC vouchers sent to malicious contract
...
_, err = k.CallModuleCRC21(ctx, contract, "mint_by_cronos_module", sender, ...)
```

The IBC vouchers are transferred to the malicious contract **before** `mint_by_cronos_module` is called. If the attacker deploys a contract that implements `mint_by_cronos_module` as a no-op (does not revert), the `CacheContext` in `OnRecvVouchers` commits successfully, the IBC vouchers are permanently locked in the malicious contract, and the user receives nothing. This constitutes an unauthorized transfer of IBC vouchers — a **Critical** accounting impact under the allowed scope.

Additionally, between Tx 1 and Tx 2, the denom is unmapped, causing all in-flight `OnRecvVouchers` calls to fail — a secondary **High** impact (long-lived inability to process valid IBC conversion flows).

### Likelihood Explanation

The attacker must hold `CanChangeTokenMapping` permission, which the `CronosAdmin` grants via `MsgUpdatePermissions`. Once granted, the two-transaction attack can be executed in consecutive blocks before the admin detects and revokes the permission. The admin has no on-chain mechanism to atomically revoke permission and restore the mapping in a single transaction, leaving a race window.

### Recommendation

1. **Separate delete from set authority**: Restrict `DeleteExternalContractForDenom` to `CronosAdmin` only; permissioned addresses should only be allowed to register mappings for previously unmapped denoms.
2. **Prevent override via delete+set**: Track the address that originally set a mapping and require the same authority (or admin) to delete it.
3. **Timelock for mapping changes**: Introduce a delay for mapping changes by permissioned addresses, giving the admin time to veto.

### Proof of Concept

func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}
