### Title
`MsgTurnBridge` Handler Is a No-Op Stub — Any Sender Gets Success, Bridge State Never Changes — (`x/cronos/keeper/msg_server.go`)

### Summary
`msgServer.TurnBridge` is implemented as a two-line stub that returns `(nil, nil)` unconditionally. Any address, privileged or not, can submit `MsgTurnBridge` and receive a success response. The `CanTurnBridge` permission constant exists and is used nowhere in this handler. The bridge circuit-breaker is completely non-functional.

### Finding Description
`UpdateTokenMapping` (the sibling handler) correctly gates execution behind `HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping)`: [1](#0-0) 

`TurnBridge`, by contrast, is: [2](#0-1) 

The permission infrastructure is fully in place — `CanTurnBridge` is defined, `HasPermission` supports it, and `SetPermissions`/`GetPermissions` work correctly: [3](#0-2) 

But none of it is called from `TurnBridge`. The handler never reads `msg.Enable`, never calls any gravity/bridge keeper method, and never emits an event.

### Impact Explanation
- **Authorization bypass (High):** Any unprivileged sender can call `MsgTurnBridge` and receive a success response, bypassing the `CanTurnBridge` permission check entirely.
- **Emergency circuit-breaker is dead:** The entire purpose of `MsgTurnBridge` is to allow authorized parties to disable the gravity bridge in an emergency. Because the handler is a no-op, the bridge can never be disabled via this message path, regardless of who sends it. This permanently removes the emergency shutdown capability.

### Likelihood Explanation
The message is registered, routable, and accepted by the chain. Any account with enough gas can submit it. The stub is not gated by any build flag or feature flag.

### Recommendation
Implement `TurnBridge` analogously to `UpdateTokenMapping`:
1. Check `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `ErrUnauthorized` if it fails.
2. Call the gravity bridge keeper to enable/disable the bridge based on `msg.Enable`.
3. Emit an appropriate event.

### Proof of Concept
```go
// Unit test (no privileged setup needed)
func TestTurnBridgeUnprivileged(t *testing.T) {
    ctx, k := setupKeeper(t)
    srv := keeper.NewMsgServerImpl(k)

    unprivileged := sdk.AccAddress([]byte("unprivileged_____"))

    resp, err := srv.TurnBridge(ctx, &types.MsgTurnBridge{
        Sender: unprivileged.String(),
        Enable: false,
    })
    // Both are nil — success returned with zero permission check
    require.NoError(t, err)   // passes
    require.Nil(t, resp)      // passes (nil response, not MsgTurnBridgeResponse{})

    // Bridge state is unchanged — no keeper call was made
    // No event was emitted
}
```

The stub at [4](#0-3)  returns `nil, nil` for every caller without reading any field of `msg`, making the bridge circuit-breaker completely inoperative and the authorization model for `MsgTurnBridge` entirely bypassed.
