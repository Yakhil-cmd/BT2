### Title
Unpermissioned `update_rewards` Allows Any Caller to Permanently Suppress All Staker Block Rewards via `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
`update_rewards` in the Staking contract is callable by any address with no access control, despite the specification requiring "Only starkware sequencer." A caller-controlled `disable_rewards: bool` parameter, when set to `true`, causes the function to advance the global `last_reward_block` checkpoint without distributing any rewards. Because `last_reward_block` is a single contract-wide slot, one such call per block permanently consumes the reward slot for every staker in that block.

### Finding Description
The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1448–1507 contains no such guard:

```rust
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();          // only checks pause flag
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... staker validity checks ...

        // ← global slot consumed here, before the disable_rewards branch
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;   // ← rewards silently skipped
        }
        // actual reward distribution only reached when disable_rewards == false
    }
}

`last_reward_block` is a single global `BlockNumber` in contract storage:

```rust
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The spec's intended access control is documented but never enforced in code: [3](#0-2) 

### Impact Explanation
`last_reward_block` is a **single global slot** shared across all stakers. One call to `update_rewards(any_valid_staker, disable_rewards: true)` per block:

1. Writes `current_block_number` into `last_reward_block`.
2. Returns immediately without distributing any STRK or BTC block rewards.
3. Causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

An attacker repeating this every block permanently prevents all stakers and their delegators from accumulating consensus block rewards. This constitutes **permanent freezing of unclaimed yield** for the entire protocol.

**Allowed impact matched**: *Permanent freezing of unclaimed yield or unclaimed royalties* (High).

### Likelihood Explanation
- No privileged role, no token, no bridge access required.
- Any EOA or contract can call `update_rewards` with `disable_rewards: true` and a valid active staker address.
- The cost is one transaction per block; on Starknet L2 this is negligible.
- The attack is fully deterministic and requires no race condition.

### Recommendation
Add a caller restriction matching the specification. The simplest fix is to add a sequencer-only guard at the top of `update_rewards`, analogous to the existing role checks used elsewhere in the contract (e.g., `self.roles.only_security_agent()`). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the pre-consensus no-op path internally, so external callers cannot influence whether rewards are distributed.

### Proof of Concept
1. Deploy the system and advance past the consensus rewards start epoch.
2. At the start of any block, call:
   ```
   IStakingRewardsManagerDispatcher { contract_address: staking }
       .update_rewards(staker_address: any_active_staker, disable_rewards: true);
   ```
3. Observe that `last_reward_block` is now equal to the current block and no `StakerRewardsUpdated` event was emitted.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat every block. Stakers and delegators accumulate zero consensus rewards indefinitely.

The flow test at `src/flow_test/test.cairo` lines 2785–2788 already demonstrates that `disable_rewards: true` produces zero rewards even when consensus is active:

```rust
// Call update_rewards with disable rewards = true - no rewards
system.update_rewards(:staker, disable_rewards: true);
let rewards = system.staker_claim_rewards(:staker);
assert!(rewards == Zero::zero());
``` [4](#0-3) 

The unit test at `src/staking/tests/test.cairo` lines 3887–3894 further confirms that after a `disable_rewards: true` call, a `disable_rewards: false` call in the same block is blocked:

```rust
staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: true);
// Catch REWARDS_ALREADY_UPDATE - with distribute = false.
let result = staking_rewards_safe_dispatcher
    .update_rewards(:staker_address, disable_rewards: false);
assert_panic_with_error(:result, expected_error: Error::REWARDS_ALREADY_UPDATED.describe());
``` [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
