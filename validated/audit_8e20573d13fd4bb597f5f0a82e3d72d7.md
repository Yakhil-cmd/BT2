### Title
Missing Commission Validation in `switch_delegation_pool` Allows Staker to Steal Delegator Yield - (File: src/pool/pool.cairo)

### Summary
When a pool member switches delegation pools, neither `switch_delegation_pool` in `pool.cairo` nor `switch_staking_delegation_pool` in `staking.cairo` validates the destination pool's commission rate at execution time. A malicious staker can advertise a low commission to attract switchers, then raise it to 100% after the pool member's exit intent is visible on-chain but before the switch executes, causing the pool member to lose all future staking rewards.

### Finding Description

The `switch_delegation_pool` function in `pool.cairo` accepts `to_staker`, `to_pool`, and `amount` but provides no mechanism for the pool member to specify a maximum acceptable commission: [1](#0-0) 

The function immediately delegates to `switch_staking_delegation_pool` in the staking contract, which performs the following checks: [2](#0-1) 

The checks verify: `to_staker` is not in exit window, `to_pool` belongs to `to_staker`, and token addresses match. **No commission check is performed.** The commission of `to_staker` at execution time is never validated.

Meanwhile, `set_commission` allows a staker to change their commission at any time with no delay, as explicitly acknowledged in the codebase: [3](#0-2) 

The `CommissionCommitment` feature exists precisely to optionally cap commission changes, but it is entirely voluntary — a malicious staker would simply not set one: [4](#0-3) 

**Attack scenario:**

1. Attacker deploys a staker with 0% commission to attract delegators.
2. Victim (pool member) sees the attractive commission and calls `exit_delegation_pool_intent(amount)` on their current pool.
3. Attacker observes the on-chain intent and calls `set_commission(10000)` (100% — `COMMISSION_DENOMINATOR = 10000`).
4. Victim calls `switch_delegation_pool(to_staker=Attacker, to_pool=AttackerPool, amount)`.
5. No commission check is performed — the switch succeeds.
6. Victim is now in a pool with 100% commission. All future rewards flow entirely to the attacker.

The attacker does not need to front-run in the traditional sense. Since the exit intent is recorded on-chain and the pool member may call `switch_delegation_pool` in a separate transaction (possibly much later), the attacker has a wide window to change commission after observing the intent.

### Impact Explanation

**High — Theft of unclaimed yield.** Once the pool member is in the 100% commission pool, every epoch's rewards are fully captured by the staker. The pool member's principal is safe (they can exit), but all yield earned while in the pool is stolen. The pool member may not notice the commission change if they do not re-query it before switching.

### Likelihood Explanation

**Medium.** The attack requires no special privilege — any staker can set commission freely. The attacker only needs to monitor on-chain exit intents (public state) and call `set_commission` before the victim's `switch_delegation_pool` executes. The exit intent and the switch are two separate transactions, giving the attacker a clear window. The `CommissionCommitment` mechanism is opt-in and a malicious staker would never use it.

### Recommendation

Add a `max_commission` parameter to `switch_delegation_pool` that the pool member specifies at call time. In `switch_staking_delegation_pool` (or `enter_delegation_pool_from_staking_contract`), assert that the destination pool's current commission does not exceed this value:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
    max_commission: Commission,  // <-- add this
) -> Amount {
    ...
    // After switch_staking_delegation_pool resolves to_staker's commission:
    let actual_commission = get_commission_for(to_staker);
    assert!(actual_commission <= max_commission, Error::COMMISSION_TOO_HIGH);
    ...
}
```

### Proof of Concept

```
Setup:
  - Attacker stakes and opens a STRK pool with commission = 0
  - Victim delegates 1000 STRK to an honest pool

Step 1: Victim calls exit_delegation_pool_intent(amount=1000) on honest pool
  → Staking contract records UndelegateIntentKey{pool=honest_pool, identifier=victim}

Step 2: Attacker observes the intent on-chain, calls set_commission(10000) on staking contract
  → Attacker's pool commission is now 100%

Step 3: Victim calls switch_delegation_pool(to_staker=Attacker, to_pool=AttackerPool, amount=1000)
  → switch_staking_delegation_pool checks: Attacker not in exit window ✓, to_pool valid ✓, token match ✓
  → NO commission check
  → Victim's 1000 STRK is now delegated to AttackerPool

Result:
  - Every epoch, staking contract calls update_rewards_from_staking_contract(rewards, pool_balance)
    with rewards already reduced by 100% commission → rewards = 0 sent to pool
  - Victim earns zero yield; all yield captured by Attacker
  - Victim must call exit_delegation_pool_intent again and wait the full exit window to escape

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn switch_staking_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            switched_amount: Amount,
            data: Span<felt252>,
            identifier: felt252,
        ) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            if switched_amount.is_zero() {
                return;
            }
            let from_pool = get_caller_address();
            let undelegate_intent_key = UndelegateIntentKey {
                pool_contract: from_pool, identifier,
            };
            let mut undelegate_intent_value = self.get_pool_exit_intent(:undelegate_intent_key);
            assert!(
                undelegate_intent_value.is_non_zero(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            // Extract the token address of the `from_pool` contract.
            let token_address = get_undelegate_intent_token(
                undelegate_intent: undelegate_intent_value,
            );
            let old_intent_amount = undelegate_intent_value.amount;
            assert!(to_pool != from_pool, "{}", Error::SELF_SWITCH_NOT_ALLOWED);
            let decimals = self.get_token_decimals(:token_address);
            let normalized_switched_amount = NormalizedAmountTrait::from_native_amount(
                amount: switched_amount, :decimals,
            );
            assert!(
                normalized_switched_amount <= old_intent_amount,
                "{}",
                GenericError::AMOUNT_TOO_HIGH,
            );

            let to_staker_info = self.internal_staker_info(staker_address: to_staker);

            // More asserts.
            assert!(to_staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let to_token_address = self
                .staker_pool_info
                .entry(to_staker)
                .get_pool_token(pool_contract: to_pool)
                .expect_with_err(Error::DELEGATION_POOL_MISMATCH);
            assert!(token_address == to_token_address, "{}", Error::TOKEN_MISMATCH);
