### Title
Gateway Stateful Admission Validates L2 Gas Price Against Stale Previous-Block Price Instead of Next-Block Price - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The gateway's stateful admission path checks a transaction's `max_price_per_unit` against the **previous** committed block's L2 gas price, while actual execution in the batcher uses the **next** block's L2 gas price (computed via EIP-1559). Transactions whose `max_price_per_unit` falls between the previous block's price and the (higher) next block's price pass gateway admission but are guaranteed to fail `check_fee_bounds` during execution, meaning the gateway admits transactions that are invalid for the block they will be sequenced into.

### Finding Description

In `validate_resource_bounds`, the gateway fetches the latest committed block's info and reads its `strk_gas_prices.l2_gas_price` as the reference price:

```rust

let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold check admits any transaction where `tx.max_price_per_unit >= min_gas_price_percentage% * previous_block_l2_gas_price` (default: 100% of previous block price). [2](#0-1) 

In `run_validate_entry_point`, the same stale block info is reused for the blockifier validation — only the block number is incremented, while gas prices remain from the previous block:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
``` [3](#0-2) 

When the batcher actually sequences the transaction, it builds a `BlockContext` with the **real** next-block gas price (computed by the EIP-1559 fee market from actual gas usage). The blockifier's `check_fee_bounds` in `perform_pre_validation_stage` then compares `resource_bounds.max_price_per_unit` against this real price: [4](#0-3) 

Any transaction with `previous_block_price <= max_price_per_unit < next_block_price` passes gateway admission but fails `check_fee_bounds` during execution.

The TODO comment in the source code explicitly acknowledges the correct fix: `// TODO(Arni): getnext_l2_gas_price from the block header.` [5](#0-4) 

The `StatefulTransactionValidatorConfig` default confirms `validate_resource_bounds: true` and `min_gas_price_percentage: 100`, so this path is active in production: [6](#0-5) 

### Impact Explanation

The gateway's admission invariant — that admitted transactions are executable in the next block — is broken. Transactions with `max_price_per_unit` in the range `[previous_block_price, next_block_price)` are admitted to the mempool, consume a nonce slot, and are included in a block, but revert during execution with `MaxGasPriceTooLow`. The user is charged a fee for the failed execution and their nonce is consumed. This constitutes the gateway accepting transactions that are invalid for the block they will be sequenced into, matching the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The EIP-1559 mechanism (`calculate_next_base_gas_price`) adjusts the L2 gas price upward whenever block gas usage exceeds the target. Under sustained load — the normal operating condition for a busy sequencer — the next block's price is routinely higher than the previous block's price. Any user who sets `max_price_per_unit` exactly at the previous block's price (a natural choice given the gateway's own threshold) will have their transaction admitted but fail during execution. No special privileges are required; any unprivileged user submitting a `ValidResourceBounds::AllResources` transaction can trigger this. [7](#0-6) 

### Recommendation

Replace the stale `previous_block_l2_gas_price` with the **next** block's L2 gas price, which is already computed and stored (e.g., in the block header as `next_l2_gas_price`). The TODO comment in the code already identifies this fix:

```rust
// Replace:
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// With:
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .next_l2_gas_price; // read the EIP-1559-computed next price from the header
```

Similarly, `run_validate_entry_point` should populate `block_info.gas_prices` with the projected next-block prices rather than reusing the previous block's prices verbatim.

### Proof of Concept

1. Observe the current committed block has L2 gas price `P` (e.g., 100 fri).
2. The block is at or above the EIP-1559 gas target, so the next block's price will be `P' = P + P/denominator` (e.g., 112 fri with denominator=8).
3. Submit an invoke transaction (V3, `AllResources`) with `l2_gas.max_price_per_unit = P` (100 fri).
4. Gateway `validate_resource_bounds` computes `threshold = 100% * P = 100` and admits the transaction (`100 >= 100`). ✓
5. Gateway `run_validate_entry_point` runs blockifier validation with `block_info.gas_prices.strk_gas_prices.l2_gas_price = P` (stale) — passes. ✓
6. Transaction enters the mempool and is included in the next block.
7. Batcher builds the block with the real next-block price `P' = 112`.
8. Blockifier `check_fee_bounds` checks `max_price_per_unit (100) < actual_gas_price (112)` → `MaxGasPriceTooLow` → transaction reverts.
9. User is charged a fee for the failed transaction; nonce is consumed. [8](#0-7) [9](#0-8)

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
