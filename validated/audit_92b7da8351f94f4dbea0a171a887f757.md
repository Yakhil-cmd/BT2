### Title
Missing Sequencer Uptime Check in L2 Price Providers Allows Stale-Price Swaps After Sequencer Downtime - (File: `smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

### Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are deployed on L2 networks (Arbitrum, Base, Avalanche, BSC, Berachain) but contain no sequencer uptime check. Their only L2 accommodation is `FUTURE_TOLERANCE`, which handles oracle clock skew, not sequencer downtime. After a sequencer outage, oracle data that was last pushed before the outage can still pass the `MAX_TIME_DELTA` staleness check, allowing swaps to execute at pre-downtime prices that no longer reflect real-world market conditions.

### Finding Description

Both L2 provider contracts implement `_isStale` with a `futureTol` parameter to tolerate oracle `refTime` slightly ahead of `block.timestamp` (sequencer clock skew):

```solidity
function _isStale(
    uint256 refTime, uint256 nowTs,
    uint256 maxDelta, uint256 futureTol
) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) {
        return (refTime - nowTs) > futureTol;
    }
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

This is the **only** L2-specific guard. Neither contract stores a `sequencerUptimeFeed`, queries Chainlink's Sequencer Uptime Feed, nor enforces a grace period after sequencer recovery. The constructor accepts no such parameter:

The `_getBidAndAskPrice` path in both contracts reads oracle data and applies only the staleness check before producing the bid/ask quote consumed by the pool swap: [5](#0-4) [6](#0-5) 

**Attack scenario:**

1. Oracle data is pushed at `T=0` with price `P0` and `refTime=T0`. `MAX_TIME_DELTA` is set to 1 hour (a typical value within the 7-day cap).
2. The L2 sequencer goes offline at `T=1`. No new oracle data can be pushed on-chain.
3. Real-world price moves from `P0` to `P1` (e.g., a 5–10% crash) during downtime.
4. Sequencer resumes at `T=2`, where `T2 - T0 < MAX_TIME_DELTA` (e.g., 30-minute outage).
5. The staleness check passes: `(T2 - T0) <= MAX_TIME_DELTA`. The oracle still returns `P0`.
6. An attacker (who pre-positioned or acts immediately on resumption) calls `pool.swap` and executes at the stale `P0` price, buying underpriced assets from LPs.

On Arbitrum specifically, the delayed inbox allows forced transaction inclusion during sequencer downtime, enabling an attacker to guarantee their swap is the first transaction processed on resumption.

### Impact Explanation

LPs suffer direct loss of principal. The pool's bid/ask is derived from the stale pre-downtime oracle price. A trader buying token0 at a stale high bid (or selling token1 at a stale low ask) extracts value from LP reserves at a price that does not reflect real-world market movement during the outage. This is a bad-price execution impact: the stale quote reaches the pool swap math and settles a trade at an incorrect price.

### Likelihood Explanation

L2 sequencer downtime is a documented, recurring event on Arbitrum, Base, and Optimism. The protocol is explicitly deployed on these networks (registry confirms Arbitrum, Base, Avalanche, BSC, Berachain deployments of `PriceProviderFactoryL2`). Any trader can trigger the exploit immediately after sequencer resumption with no special role or setup. The window of vulnerability equals `MAX_TIME_DELTA` minus the actual downtime duration — for a 30-minute outage with a 1-hour `MAX_TIME_DELTA`, the window is 30 minutes.

### Recommendation

1. Add a `sequencerUptimeFeed` (Chainlink Sequencer Uptime Feed) immutable to both `PriceProviderL2` and `ProtectedPriceProviderL2`.
2. In `_getBidAndAskPrice`, before the staleness check, query the uptime feed: if the sequencer is down or the grace period (e.g., 1 hour) since recovery has not elapsed, return `(0, type(uint128).max)` (fail closed).
3. Update `PriceProviderFactoryL2.createPriceProvider` to accept and forward the `_sequencerUptimeFeed` parameter.
4. Enforce that `MAX_TIME_DELTA` is no longer than the sequencer uptime grace period to prevent the staleness window from exceeding the recovery buffer.

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }

    /// @dev Downstream pricing: staleness, price guard, confidence spread, marginStep.
    function _computeBidAsk(uint256 price, uint256 spread, uint256 refTime)
        internal view returns (uint128, uint128)
    {
        // 1. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 2. Basic validity — price must be positive, spread must not be stalled marker
        if (price == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 3. Price guard check
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (price < guardMin || price > guardMax) {
            return (0, type(uint128).max);

        // 4. Compute bid/ask from mid + confidence-adjusted spread
