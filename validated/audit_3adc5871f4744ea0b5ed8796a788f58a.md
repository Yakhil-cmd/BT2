### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

### Summary

`setPoolBinAdditionalFees` imposes no upper-bound check on `addFeeBuyE6` / `addFeeSellE6`, while the parallel path `setPoolAdminFees` enforces `maxAdminSpreadFeeE6`. A pool admin can set per-bin additional fees to `type(uint16).max` (65 535 E6 ≈ 6.55 %) on every bin, regardless of what the factory owner has configured as the admin spread cap, extracting fees from traders above the intended ceiling.

### Finding Description

`MetricOmmPoolFactory.setPoolAdminFees` correctly gates the global admin spread component:

```solidity
// MetricOmmPoolFactory.sol L414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
```

The factory owner can tighten this cap at any time via `setFeeCaps` (L284-315). However, `setPoolBinAdditionalFees` passes the values straight through with no analogous check:

```solidity
// MetricOmmPoolFactory.sol L450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

These values are then added directly to `baseFeeX64` (itself derived from `spreadFeeE6`) in every swap path:

```solidity
// MetricOmmPool.sol L999
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
// MetricOmmPool.sol L1177
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)
```

and in the quoted bid/ask prices returned to traders:

```solidity
// MetricOmmPool.sol L540-541
uint256 buyFeeX64  = baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6,  ONE_X64, 1e6);
uint256 sellFeeX64 = baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6);
```

The surplus that accumulates from these inflated fees is swept by `collectFees` and split between admin and protocol in proportion to their stored `poolFeeConfig` components, so the pool admin directly profits from the excess.

### Impact Explanation

- **Admin-boundary break:** The factory owner's `maxAdminSpreadFeeE6` cap (enforced on the global path) is completely bypassed on the per-bin path. A pool admin can set `addFeeBuyE6 = addFeeSellE6 = 65535` on every active bin, imposing an effective additional spread of ≈ 6.55 % per bin on top of the already-capped global spread fee.
- **Direct trader loss:** Every swap through an affected bin pays the inflated fee. The excess is real token value transferred out of the trader's position into the pool surplus, then swept to the admin fee destination.
- **Broken invariant:** The protocol's fee-cap system is the primary protection against pool-admin overreach. The per-bin path renders that system ineffective for the spread dimension.

### Likelihood Explanation

The pool admin is a valid, reachable actor (not a factory owner / trusted-admin exclusion). A single call to `setPoolBinAdditionalFees` with `addFeeBuyE6 = type(uint16).max` on each bin is sufficient. No special preconditions, no timelock, no co-signer required. The factory owner lowering `maxAdminSpreadFeeE6` (a normal operational action) is the trigger that makes the bypass meaningful.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` mirroring the one in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(
    address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6
) external override nonReentrant onlyPoolAdmin(pool) {
    if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce a combined cap so that `spreadFeeE6 + addFeeBuyE6` and `spreadFeeE6 + addFeeSellE6` cannot exceed `maxAdminSpreadFeeE6 + maxProtocolSpreadFeeE6`.

### Proof of Concept

1. Factory owner calls `setFeeCaps(…, newMaxAdminSpreadFeeE6 = 1_000, …)` — capping admin spread at 0.1 %.
2. Pool admin calls `setPoolAdminFees(pool, 1_000, 0)` — sets global admin spread to the cap (0.1 %).
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` — sets per-bin additional fees to 6.5535 % with no revert.
4. A trader swaps through bin 0. The effective fee applied is `baseFeeX64 + 65535/1e6 ≈ 6.65 %`, far above the 0.1 % cap the factory owner intended.
5. The excess accumulates as pool surplus; `collectPoolFees` sweeps it to the admin fee destination.
