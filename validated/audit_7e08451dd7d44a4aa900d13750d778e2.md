### Title
Unprivileged Caller Can Permanently Deny Block Rewards via Unguarded `disable_rewards` Parameter in `update_rewards` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable by any address and accepts a caller-controlled `disable_rewards: bool` parameter. Because there is no access control on this function, any attacker can call `update_rewards(staker_address, disable_rewards: true)` before the legitimate consensus caller does. This consumes the single per-block reward-update slot while skipping actual reward distribution, permanently destroying that block's rewards for the targeted staker.

---

### Finding Description

`IStakingRewardsManager::update_rewards` is exposed as a public interface with no role check: [1](#0-0) 

Its implementation in `staking.cairo` enforces a strict one-call-per-block invariant by writing `last_reward_block` unconditionally **before** the `disable_rewards` branch: [2](#0-1) 

The critical sequence is:

1. **No access control** — `general_prerequisites()` only checks the pause flag; there is no `only_*` role assertion.
2. **Per-block gate is consumed unconditionally** — `self.last_reward_block.write(current_block_number)` executes regardless of `disable_rewards`.
3. **Reward distribution is then skipped** — `if disable_rewards || self.is_pre_consensus() { return; }` exits without distributing rewards. [3](#0-2) 

Any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

The `disable_rewards` flag is analogous to `effective_token_from_amount` in the external report: it is a caller-supplied parameter that controls whether the core economic action (reward distribution) actually executes, yet it is not authenticated or bounded in any way.

---

### Impact Explanation

**High — Theft of unclaimed yield / Permanent freezing of unclaimed yield.**

Block rewards that are skipped are not deferred; they are permanently lost. The `last_reward_block` slot for that block is consumed, and the block can never be revisited. An attacker who calls `update_rewards(victim_staker, disable_rewards: true)` once per block can drain 100 % of a staker's consensus-phase block rewards indefinitely. Because the staker's `unclaimed_rewards_own` is never incremented, the yield is destroyed rather than merely delayed.

---

### Likelihood Explanation

The attack requires no special privilege, no token balance, and no cryptographic material. The attacker only needs:

- The target staker's address (publicly readable from `stakers` vector or events).
- The ability to submit a transaction — any Starknet account suffices.

The per-block gas cost of the attack is a single cheap state-write transaction. The economic damage scales with the staker's stake and the reward rate. For a large validator the ratio of damage-to-cost is highly favorable to the attacker, making sustained griefing economically rational.

---

### Recommendation

Restrict `update_rewards` to an authorized caller. The most natural fix is to add a role check (e.g., `only_app_governor` or a dedicated `BLOCK_PROPOSER_ROLE`) so that only the consensus infrastructure can invoke it. Alternatively, remove `disable_rewards` from the public interface entirely and handle the "disable" path through a separate privileged function or an internal flag set by the staker's own exit flow.

---

### Proof of Concept

```
1. Staker S is an active validator with a large STRK stake.
2. Consensus rewards are active (post `consensus_rewards_first_epoch`).
3. Attacker A (any EOA) submits, for every new block B:
       staking.update_rewards(staker_address = S, disable_rewards = true)
4. The call passes all checks (contract not paused, S is active, S has non-zero balance).
5. `last_reward_block` is set to B.
6. The `if disable_rewards` branch fires; no rewards are computed or transferred.
7. The legitimate consensus call for block B reverts with REWARDS_ALREADY_UPDATED.
8. S's `unclaimed_rewards_own` is never incremented for block B.
9. Repeating steps 3–8 every block permanently zeroes S's yield.
```

The attack path is entirely on-chain, requires no mempool visibility, and is reachable by any unprivileged Starknet account.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
