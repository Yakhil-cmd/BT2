### Title
Hardcoded `STRK_TOKEN_ADDRESS` in `Pool::claim_rewards` Causes Permanent Freezing of BTC Pool Delegator Rewards - (File: src/pool/pool.cairo)

### Summary

The `claim_rewards` function in the Pool contract hardcodes `STRK_TOKEN_ADDRESS` when transferring rewards to the delegator's reward address. For BTC delegation pools (introduced in V3 multi-token support), the pool receives BTC rewards from the staking contract, but `claim_rewards` attempts to transfer STRK instead of BTC. This permanently freezes BTC delegator rewards inside the pool contract.

### Finding Description

In `src/pool/pool.cairo`, the `claim_rewards` function constructs a token dispatcher using the hardcoded constant `STRK_TOKEN_ADDRESS` rather than the pool's own `token_dispatcher` (which holds the correct token address for the pool):

```cairo
// src/pool/pool.cairo, lines 364-366
let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
``` [1](#0-0) 

The pool contract is initialized with a `token_address` parameter that is stored as `token_dispatcher`:

```cairo
// src/pool/pool.cairo, lines 154-161
pub fn constructor(
    ...
    token_address: ContractAddress,
    ...
) {
    ...
    self.token_dispatcher.contract_address.write(token_address);
``` [2](#0-1) 

This `token_address` can be BTC (as introduced in V3). The pool's `exit_action` function correctly uses `self.token_dispatcher.read()` for principal transfers:

```cairo
// src/pool/pool.cairo, lines 329-330
let token_dispatcher = self.token_dispatcher.read();
token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
``` [3](#0-2) 

The staking contract's `send_rewards_to_delegation_pool` also correctly accepts a `token_dispatcher` parameter and uses it for transfers to the pool:

```cairo
// src/staking/staking.cairo, lines 1635-1642
fn send_rewards_to_delegation_pool(
    ref self: ContractState,
    staker_address: ContractAddress,
    pool_address: ContractAddress,
    amount: Amount,
    token_dispatcher: IERC20Dispatcher,
) {
    token_dispatcher.checked_transfer(recipient: pool_address, amount: amount.into());
``` [4](#0-3) 

So BTC rewards are correctly routed from the staking contract into the BTC pool contract in BTC. However, when a delegator calls `claim_rewards`, the pool attempts to transfer STRK (not BTC) to the reward address. Since the BTC pool holds BTC (not STRK), this call will revert (insufficient STRK balance), permanently locking the BTC rewards inside the pool.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

BTC pool delegators can never successfully call `claim_rewards`. Their earned BTC rewards accumulate inside the pool contract but are permanently inaccessible. There is no alternative code path to retrieve them. The staked principal is unaffected (returned via `exit_action` using the correct `token_dispatcher`), but all accrued yield is frozen.

### Likelihood Explanation

**High.** Any delegator who joins a BTC delegation pool and attempts to claim rewards will trigger this bug. It requires no special privileges — any unprivileged delegator calling the public `claim_rewards` entrypoint on a BTC pool is affected. The V3 upgrade explicitly introduced BTC pool support, making this a live code path. [5](#0-4) 

### Recommendation

Replace the hardcoded `STRK_TOKEN_ADDRESS` in `claim_rewards` with the pool's stored `token_dispatcher`, consistent with how `exit_action` handles token transfers:

```cairo
// Before (incorrect):
let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

// After (correct):
let reward_token = self.token_dispatcher.read();
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
``` [1](#0-0) 

### Proof of Concept

1. A BTC delegation pool is deployed with `token_address = BTC_TOKEN_ADDRESS`.
2. A delegator calls `enter_delegation_pool` on the BTC pool, depositing BTC.
3. The associated staker attests; the staking contract calls `send_rewards_to_delegation_pool` with a BTC `token_dispatcher`, transferring BTC rewards into the pool contract.
4. The delegator calls `claim_rewards(pool_member)`.
5. Inside `claim_rewards`, the pool computes `rewards > 0` correctly, then executes:
   ```cairo
   let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
   reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
   ```
6. The BTC pool contract holds no STRK, so `checked_transfer` reverts.
7. The delegator's BTC rewards remain permanently locked in the pool contract with no recovery path. [6](#0-5) [7](#0-6)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
