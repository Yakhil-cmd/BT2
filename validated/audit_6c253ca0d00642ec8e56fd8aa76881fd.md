Audit Report

## Title
SNS Swap Canister Permanently Locks SNS Tokens on Abort — (`rs/sns/swap/src/swap.rs`)

## Summary

When an SNS decentralization swap transitions to `ABORTED`, `finalize_inner` returns early after restoring dapp controllers without ever transferring the pre-loaded SNS tokens back to the SNS governance treasury. No alternative code path exists to recover these tokens. Once the swap canister is deleted per the documented lifecycle, the tokens become permanently inaccessible.

## Finding Description

In `finalize_inner` (lines 1544–1584), the ABORTED execution path is:

1. `sweep_icp` — refunds ICP to buyers (line 1558)
2. `settle_neurons_fund_participation` — notifies NNS governance (line 1564)
3. `should_restore_dapp_control()` returns `true` → restores dapp controllers → **returns early at line 1583**

The `sweep_sns` call at lines 1593–1598 is only reached after this early return and is therefore never executed in the ABORTED lifecycle. Additionally, `sweep_sns` itself explicitly rejects any call when `lifecycle() != Lifecycle::Committed` (lines 2170–2178), providing a second guard that would prevent recovery even if called directly.

A search across the entire swap canister source confirms there is no `sweep_sns_to_treasury`, no treasury transfer, and no other function that moves SNS tokens out of the swap canister's ledger account in the ABORTED state. The SNS tokens pre-loaded into the swap canister's account `(swap_canister_id, None)` on the SNS ledger have no exit path on the abort branch.

The lifecycle diagram in the proto documentation explicitly shows `ABORTED → <DELETED>`, meaning the swap canister is eventually deleted. After deletion, its principal can no longer authorize transfers from its SNS ledger account, making the tokens permanently unrecoverable.

## Impact Explanation

This is a permanent, irreversible loss of SNS tokens for any SNS project whose decentralization swap fails. The tokens are minted/transferred into the swap canister before the swap opens and have no return path on abort. The impact maps to the High bounty category: **"Significant SNS security impact with concrete user or protocol harm"** — specifically, permanent loss of governance tokens from the SNS treasury with no recovery mechanism. Depending on the SNS token valuation and the quantity pre-loaded, this could reach Critical thresholds (permanent loss exceeding $1M is plausible for high-value SNS projects).

## Likelihood Explanation

No adversarial action is required. Any SNS swap that fails to reach `min_participants` or `min_icp_e8s` before the deadline will naturally transition to `ABORTED` via `try_abort`. This is a documented, expected failure mode. The abort path is exercised in existing tests (`test_try_commit_or_abort_insufficient_participation_with_no_time_remaining`, `test_finalize_swap_abort_matched_funding`). Low community interest in a new SNS project is sufficient to trigger this.

## Recommendation

In `finalize_inner`, before the early return at line 1583, add a call to transfer the full SNS token balance from the swap canister's ledger account back to `init.sns_governance_canister_id` (or a designated treasury subaccount). Alternatively, add a dedicated `sweep_sns_to_treasury` function callable only in the `ABORTED` state. The fix must be applied before the swap canister is deleted, so it should be part of the standard `finalize` flow.

## Proof of Concept

1. Deploy an SNS swap with SNS tokens pre-loaded (e.g., `with_sns_tokens(10 * E8)`).
2. Allow the swap to expire without reaching `min_participants` or `min_icp_e8s`.
3. Call `try_abort` → lifecycle transitions to `ABORTED`.
4. Call `finalize` → `finalize_inner` runs `sweep_icp`, `settle_neurons_fund_participation`, hits `should_restore_dapp_control() == true`, and returns early at line 1583.
5. `sweep_sns` is never called; `sweep_sns_result` is never populated in `FinalizeSwapResponse`.
6. The SNS ledger account of the swap canister retains the full SNS token balance.
7. The swap canister is deleted per the lifecycle diagram.
8. The SNS tokens are permanently inaccessible.

The existing test `test_finalize_swap_abort_matched_funding` (lines 1557–1600 of `rs/sns/swap/tests/swap.rs`) confirms the abort path completes without any `sweep_sns_result` being set, directly demonstrating the missing token recovery step. [1](#0-0) [2](#0-1) [3](#0-2)
