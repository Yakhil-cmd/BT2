### Title
`Modelv3` Omitted from `VersionedDecode` Switch — Protocol State Corruption on Upgrade to v3 - (File: `state/protocol/protocol_state/kvstore/encoding.go`)

### Summary

`VersionedDecode` in `encoding.go` handles version cases `0`, `1`, and `2`, but has no `case 3` for `Modelv3`. After the network upgrades the Protocol State KV store to version 3 (via a `ProtocolStateVersionUpgrade` service event), any node that encodes a v3 snapshot and later attempts to decode it will receive `ErrUnsupportedVersion`. This causes an irrecoverable exception in the consensus hot-path, halting the node. The root cause is structurally identical to the reported bug: a new version adds fields (`ExecutionMeteringParameters`, `ExecutionComponentVersion`, `CadenceComponentVersion`) that the decode path does not know how to reconstruct, so the live state becomes unreadable after the upgrade activates.

---

### Finding Description

`Modelv3` is a fully defined, reachable model version in the codebase. [1](#0-0) 

`Modelv2.Replicate(3)` successfully produces a `Modelv3` instance and stores it via `ProtocolKVStore.BatchStore`, which calls `Modelv3.VersionedEncode()` and persists the blob with `Version = 3`. [2](#0-1) 

However, `VersionedDecode` only handles versions `0`, `1`, and `2`. Version `3` falls through to the `default` branch and returns `ErrUnsupportedVersion`: [3](#0-2) 

Both `ByBlockID` and `ByID` in `ProtocolKVStore` call `VersionedDecode` and propagate the error upward: [4](#0-3) 

`initializeOrthogonalStateMachines` calls `ByBlockID` on every candidate block and treats `ErrIncompatibleVersionChange` (which wraps `ErrUnsupportedVersion`) as an irrecoverable exception, halting the node: [5](#0-4) 

Additionally, `NewKVStore` (used during bootstrapping) has no `case 3`, so a node cannot even bootstrap with a v3 store: [6](#0-5) 

The `TestEncodeDecode` test suite covers only v0, v1, and v2 — there is no round-trip test for v3: [7](#0-6) 

---

### Impact Explanation

Once the network emits a valid `ProtocolStateVersionUpgrade` service event targeting version 3, all nodes will:

1. Replicate the parent `Modelv2` state into a `Modelv3` snapshot via `Modelv2.Replicate(3)`.
2. Encode and persist the v3 snapshot to the database.
3. On the next block, attempt to decode the v3 snapshot via `VersionedDecode(3, ...)`, receive `ErrUnsupportedVersion`, and crash with an irrecoverable exception.

This halts consensus across all nodes simultaneously at the activation view, producing a network-wide outage. Any state written under v3 (execution metering parameters, Cadence/Execution component versions) is permanently unreadable by the current software.

The analog to the external report is exact: a new version adds fields that the decode path does not handle, so the live state becomes unreadable after the upgrade activates — mirroring the V2→V1 rollback storage-slot corruption.

---

### Likelihood Explanation

The `ProtocolStateVersionUpgrade` service event is emitted by the `NodeVersionBeacon` system smart contract and processed by `PSVersionUpgradeStateMachine`. The upgrade to v3 is enforced to be exactly `currentVersion + 1`, so it is a normal, planned protocol operation. The missing `case 3` in `VersionedDecode` is a deterministic code defect that will trigger the first time any node processes a block at or past the v3 activation view. [8](#0-7) 

---

### Recommendation

1. Add `case 3: target = new(Modelv3)` to the `VersionedDecode` switch in `encoding.go`.
2. Add `case 3: return &Modelv3{...}` to `NewKVStore` in `models.go`.
3. Add a round-trip encode/decode test for `Modelv3` in `TestEncodeDecode`.
4. Enforce a compile-time or test-time invariant that every model version reachable via `Replicate` is also present in `VersionedDecode`.

---

### Proof of Concept

**Step 1 — Trigger upgrade to v3 via service event:**
A governance transaction calls `NodeVersionBeacon` to emit `ProtocolStateVersionUpgrade{NewProtocolStateVersion: 3, ActiveView: V}`. This is accepted by `PSVersionUpgradeStateMachine.processSingleEvent` because `currentVersion(2) + 1 == 3`. [9](#0-8) 

**Step 2 — Activation view reached; `Modelv2.Replicate(3)` called:**
`initializeOrthogonalStateMachines` detects `candidateView >= versionUpgrade.ActivationView` and calls `parentState.Replicate(3)`, producing a `Modelv3` instance with the new fields. [10](#0-9) 

**Step 3 — v3 snapshot encoded and stored:**
`ProtocolKVStore.BatchStore` calls `Modelv3.VersionedEncode()`, which returns `version=3` and a msgpack blob. This is written to the database. [11](#0-10) 

**Step 4 — Next block: decode fails:**
`ByBlockID` calls `VersionedDecode(3, blob)`. The switch has no `case 3`, so it returns `ErrUnsupportedVersion`. `initializeOrthogonalStateMachines` wraps this as an irrecoverable exception and the node halts. [12](#0-11)

func (model *Modelv2) Replicate(protocolVersion uint64) (protocol_state.KVStoreMutator, error) {
	currentVersion := model.GetProtocolStateVersion()
	if currentVersion == protocolVersion {
		// no need for migration, return a complete copy
		return clone.Clone(model), nil
	}
	nextVersion := currentVersion + 1
	if protocolVersion != nextVersion {
		// can only Replicate into model with numerically consecutive version
		return nil, fmt.Errorf("unsupported replication version %d, expect %d: %w",
			protocolVersion, nextVersion, ErrIncompatibleVersionChange)
	}

	// perform actual replication to the next version
	v3 := &Modelv3{
		Modelv2: clone.Clone(*model),
		// Execution component versions and metering parameters are set to default values when upgrading to v3
		CadenceComponentVersion: protocol.UpdatableField[protocol.MagnitudeVersion]{
			CurrentValue: protocol.MagnitudeVersion{Major: 0, Minor: 0},
		},
		ExecutionComponentVersion: protocol.UpdatableField[protocol.MagnitudeVersion]{
			CurrentValue: protocol.MagnitudeVersion{Major: 0, Minor: 0},
		},
		ExecutionMeteringParameters: protocol.UpdatableField[protocol.ExecutionMeteringParameters]{
			CurrentValue: protocol.DefaultExecutionMeteringParameters(),
		},
	}
	if v3.GetProtocolStateVersion() != protocolVersion {
		return nil, fmt.Errorf("sanity check: replicate resulted in unexpected version (%d != %d)", v3.GetProtocolStateVersion(), protocolVersion)
	}
	return v3, nil
}

func VersionedDecode(version uint64, bz []byte) (protocol_state.KVStoreAPI, error) {
	var target protocol_state.KVStoreAPI
	switch version {
	case 0:
		target = new(Modelv0)
	case 1:
		target = new(Modelv1)
	case 2:
		target = new(Modelv2)
	default:
		return nil, ErrUnsupportedVersion
	}
	err := msgpack.NewDecoder(bytes.NewBuffer(bz)).Decode(&target)
	if err != nil {
		return nil, irrecoverable.NewExceptionf("could not decode kvstore (version=%d): %w", version, err)
	}
	return target, nil
}

func (p *ProtocolKVStore) ByBlockID(blockID flow.Identifier) (protocol_state.KVStoreAPI, error) {
	versionedData, err := p.ProtocolKVStore.ByBlockID(blockID)
	if err != nil {
		return nil, fmt.Errorf("could not query KV store at block (%x): %w", blockID, err)
	}
	kvStore, err := VersionedDecode(versionedData.Version, versionedData.Data)
	if err != nil {
		return nil, fmt.Errorf("could not decode protocol state (version=%d) at block (%x): %w", versionedData.Version, blockID, err)
	}
	return kvStore, err
}
