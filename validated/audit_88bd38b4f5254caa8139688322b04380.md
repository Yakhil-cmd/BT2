### Title
Unbounded Per-Epoch Reward Accumulation via Global `last_reward_block` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` uses a single **global** `last_reward_block` variable to gate reward distributions, with no per-staker epoch-level tracking. Because `update_rewards` enforces no caller access control (the spec states "Only starkware sequencer" but the code does not enforce this), any staker can call it for themselves in every block within an epoch. Since `block_rewards` is cached as a constant value for the entire epoch, each call distributes the same reward amount, allowing a staker to accumulate far more rewards than intended and drain the reward supplier.

---

### Finding Description

In V3 consensus rewards mode, `update_rewards` distributes `block_rewards` to a specific staker. The only guard against duplicate distributions is:

```cairo
// src/staking/staking.cairo lines 1453-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

followed by:

```cairo
// src/staking/staking.cairo line 1485
self.last_reward_block.write(current_block_number);
```

`last_reward_block` is a **single global storage variable** (not per-staker). [1](#0-0) 

`block_rewards` is cached per epoch — the same value is returned for every block within the same epoch:

```cairo
// src/staking/staking.cairo lines 1563-1570
if curr_epoch > self.last_calculated_epoch.read() {
    self.last_calculated_epoch.write(curr_epoch);
    let block_rewards = reward_supplier_dispatcher.update_current_epoch_block_rewards();
    self.block_rewards.write(block_rewards);
    block_rewards
} else {
    self.block_rewards.read()
}
``` [2](#0-1) 

The access control on `update_rewards` is only `general_prerequisites`, which checks:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [3](#0-2) 

There is **no sequencer address check**, despite the spec declaring "Only starkware sequencer" as the access control. [4](#0-3) 

Each successful call to `_update_rewards` immediately:
1. Increases the reward supplier's `unclaimed_rewards` counter
2. Transfers pool rewards directly to the pool contract
3. Accumulates staker rewards in `unclaimed_rewards_own` [5](#0-4) 

**Root cause analog**: Just as the `GaugeController` in the external report lacks a `userTotalGaugeVotes` cumulative cap, `staking.cairo` lacks a per-staker epoch-level reward distribution counter. The global `last_reward_block` only prevents two calls in the *same block*, not multiple calls across different blocks within the same epoch.

---

### Impact Explanation

- **Theft of unclaimed yield (High)**: A staker calling `update_rewards` for themselves in every block of an epoch (E blocks) receives `E × block_rewards` instead of the intended `~block_rewards`. Pool rewards are immediately transferred to the pool contract on each call, making them unrecoverable.
- **Permanent freezing of unclaimed yield for other stakers (High)**: Because `last_reward_block` is global, the attacker's calls in each block prevent the sequencer from distributing rewards to other stakers in those same blocks. Other stakers receive zero rewards for the epoch.
- **Protocol insolvency (Critical)**: The reward supplier's `unclaimed_rewards` counter is inflated by `E × block_rewards` per epoch per attacker. If multiple stakers race to call `update_rewards` in every block, the reward supplier is drained far faster than the minting curve intends.

---

### Likelihood Explanation

- Any staker with non-zero balance (after the K-epoch delay) can execute this attack.
- No privileged role, leaked key, or external dependency is required.
- The call is a standard public function with no on-chain access restriction.
- The attacker only needs to submit one transaction per block, which is trivially achievable.

---

### Recommendation

1. **Add per-staker epoch tracking**: Record the last epoch for which each staker received rewards and revert if `update_rewards` is called again in the same epoch for the same staker.
2. **Enforce access control**: Add an explicit check that `get_caller_address() == sequencer_address` (or an authorized rewards manager role) inside `update_rewards`, consistent with the spec's stated intent.

---

### Proof of Concept

1. Staker A stakes 100 STRK own + 100 STRK delegated = 200 STRK total. Consensus rewards are active. Epoch has E = 100 blocks. `block_rewards = R`.
2. Block 1: Staker A calls `update_rewards(staker_A, false)`.
   - `last_reward_block` = 1. `staker_own_rewards = R × 100/200 = R/2`. Pool receives `R/2`.
3. Block 2: Staker A calls `update_rewards(staker_A, false)` again.
   - `last_reward_block` = 2. Same `block_rewards = R` (cached). Staker A receives another `R/2`. Pool receives another `R/2`.
4. After 100 blocks: Staker A has accumulated `50R` in `unclaimed_rewards_own`; pool has received `50R` in direct transfers. Total extracted = `100R`.
5. Intended behavior: Staker A should receive `~R` total (one block's worth per epoch).
6. Other stakers received **zero** rewards because `last_reward_block` was updated by staker A in every block, blocking the sequencer from distributing to them. [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
