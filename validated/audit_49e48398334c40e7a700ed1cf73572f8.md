### Title
Instantaneous Commission Increase via Minimum-Duration Commitment Allows Theft of Delegator Yield - (File: `src/staking/staking.cairo`)

### Summary

A staker can set a `CommissionCommitment` with `expiration_epoch = current_epoch + 1` (the minimum allowed) and immediately invoke `set_commission` in the same epoch to raise commission to `max_commission` (up to 100%). Because `is_commission_commitment_active` returns `true` the instant the commitment is created, there is zero enforced delay before the increase takes effect. Delegators are then locked in the pool for the full `exit_wait_window` earning zero (or near-zero) yield while the staker captures all rewards.

The codebase itself acknowledges this at the call-site:

> `/// **Note**: Current commission increase safeguards still allow for sudden commission changes.` [1](#0-0) 

---

### Finding Description

**Root cause — `is_commission_commitment_active` uses a strict less-than check with no minimum duration:**

```cairo
fn is_commission_commitment_active(
    self: @ContractState, commission_commitment: CommissionCommitment,
) -> bool {
    self.get_current_epoch() < commission_commitment.expiration_epoch
}
``` [2](#0-1) 

`set_commission_commitment` enforces only `expiration_epoch > current_epoch`, so the minimum valid value is `current_epoch + 1`:

```cairo
assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
``` [3](#0-2) 

A commitment with `expiration_epoch = current_epoch + 1` is therefore **immediately active** (`current_epoch < current_epoch + 1 == true`) the moment it is written. There is no waiting period before the staker can use it.

**`update_commission` allows raising commission to `max_commission` while the commitment is active:**

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
}
``` [4](#0-3) 

The new commission is written to storage immediately with no epoch delay:

```cairo
staker_pool_info.commission.write(Option::Some(commission));
``` [5](#0-4) 

**Attack sequence (two transactions, same epoch N):**

1. `set_commission_commitment(max_commission=10000, expiration_epoch=N+1)` — valid because `N+1 > N`.
2. `set_commission(commission=10000)` — valid because `N < N+1` (commitment is active), and `10000 <= 10000`.

Commission is now 100%, effective immediately. The `CommissionCommitment` struct and the `expiration_epoch` field are defined as:

```cairo
pub struct CommissionCommitment {
    pub max_commission: Commission,
    pub expiration_epoch: Epoch,
}
``` [6](#0-5) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

Delegators who are already in the pool cannot exit atomically. They must call `exit_delegation_pool_intent` and then wait for the full `exit_wait_window` before `exit_delegation_pool_action` is callable. During that entire window, the staker's commission is 100%, so every reward epoch transfers the delegators' share entirely to the staker. The delegators receive zero yield for the duration of the exit window, which constitutes direct theft of unclaimed yield.

The spec confirms that rewards stop accruing to the delegator's balance once the intent is filed, but the exit window is a time-based lock enforced by `INTENT_WINDOW_NOT_FINISHED`: [7](#0-6) 

---

### Likelihood Explanation

**High.** The attack requires only two sequential transactions from the staker's own address — no privileged protocol role, no external dependency, no leaked key. A staker who has attracted significant delegated capital has a direct financial incentive to execute this. The minimum commitment duration of one epoch is short (configurable, but typically on the order of days), and the two transactions can be submitted back-to-back in the same block.

---

### Recommendation

Enforce a minimum delay between when a `CommissionCommitment` is set and when it may be used to raise commission. Specifically, require that `expiration_epoch >= current_epoch + 2` (analogous to the `set_consensus_rewards_first_epoch` guard already present in the codebase):

```cairo
assert!(epoch_id >= curr_epoch + 2, "{}", Error::INVALID_EPOCH);
``` [8](#0-7) 

This ensures at least one full epoch elapses between the public announcement of the commitment and the earliest moment the staker can act on it, giving delegators time to observe the pending increase and exit if desired.

Additionally, consider requiring that `set_commission` with an upward change can only take effect at the start of the epoch following the call (i.e., store a "pending commission" and apply it lazily), mirroring the pattern used in the referenced Solana SPL stake-pool patch.

---

### Proof of Concept

```
Epoch N, block B:
  tx1: staker.set_commission_commitment(max_commission=10000, expiration_epoch=N+1)
       → CommissionCommitment { max_commission: 10000, expiration_epoch: N+1 } stored
       → is_commission_commitment_active: N < N+1 == true  ✓

  tx2 (same block B): staker.set_commission(commission=10000)
       → update_commission called
       → is_commission_commitment_active: N < N+1 == true  ✓
       → 10000 <= 10000  ✓
       → commission written to storage as 10000 immediately

Epoch N, block B+1 onward:
  All delegator rewards flow 100% to staker.
  Delegators call exit_delegation_pool_intent → must wait exit_wait_window.
  During entire wait window, delegators earn 0 yield.
  Staker captures all delegator yield for the duration of the exit window.
``` [2](#0-1) [9](#0-8) [10](#0-9)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
