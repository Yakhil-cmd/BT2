### Title
Staker Can Instantly Increase Commission via `set_commission_commitment` + `set_commission` to Steal Delegator Yield - (`src/staking/staking.cairo`)

---

### Summary

A staker can atomically call `set_commission_commitment` followed by `set_commission` within the same block to raise their commission from near 0% to up to 100%, stealing all future yield from existing and incoming delegators. The protocol's own code acknowledges this gap with an inline warning comment.

---

### Finding Description

Without a commitment, `set_commission` enforces a strict decrease-only rule:

```
assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
```

However, once a commitment is active, `update_commission` only checks that the new value does not exceed `max_commission` and is not equal to the old value:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
}
``` [1](#0-0) 

There is **no delay** between setting the commitment and using it. A staker can call `set_commission_commitment(max_commission=10000, expiration_epoch=current_epoch+1)` and then immediately call `set_commission(commission=10000)` in the same block. The protocol developers explicitly acknowledge this gap:

```cairo
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
``` [2](#0-1) 

`set_commission_commitment` requires only that `max_commission >= current_commission` and `expiration_epoch > current_epoch`: [3](#0-2) 

`is_commission_commitment_active` returns `true` as long as `current_epoch < expiration_epoch`, so a commitment set with `expiration_epoch = current_epoch + 1` is immediately active: [4](#0-3) 

The commission is applied at reward distribution time (during attestation), not at delegation time, so **all existing delegators** in the pool are affected retroactively from the next reward epoch.

---

### Impact Explanation

Commission is deducted from the delegation pool's share of rewards before they reach delegators. A staker who raises commission to 100% (`COMMISSION_DENOMINATOR = 10000`) causes the pool to receive zero rewards, routing everything to the staker's own reward address. This is a direct theft of unclaimed yield from all delegators in the pool. [5](#0-4) 

**Impact category**: High — Theft of unclaimed yield.

---

### Likelihood Explanation

The attack requires only that the attacker be a registered staker with an open delegation pool. No privileged role, leaked key, or external dependency is needed. The two-step sequence (`set_commission_commitment` → `set_commission`) can be submitted in consecutive transactions within the same block. A rational attacker profits by accumulating delegators at a low commission rate, then spiking to 100% just before a reward epoch closes.

---

### Recommendation

Enforce a mandatory delay (e.g., at least one full epoch) between the block in which `set_commission_commitment` is set and the earliest block in which `set_commission` may use that commitment to **increase** the commission. Alternatively, cap the maximum single-epoch commission increase (e.g., no more than 5% per epoch). This mirrors the intent behind the commitment mechanism but closes the same-block loophole.

---

### Proof of Concept

1. Staker registers with commission = 200 (2%) and opens a delegation pool.
2. Many delegators call `enter_delegation_pool`, attracted by the low commission.
3. Just before the next attestation epoch, the staker submits two transactions in the same block:
   - `set_commission_commitment(max_commission=10000, expiration_epoch=current_epoch+1)`
   - `set_commission(commission=10000)`
4. `is_commission_commitment_active` returns `true` immediately because `current_epoch < current_epoch+1`.
5. Commission is now 100%. At the next attestation, `split_rewards_with_commission` routes 100% of pool rewards to the staker; delegators receive zero.
6. Delegators who call `claim_rewards` receive nothing for that epoch. [6](#0-5) [7](#0-6)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
