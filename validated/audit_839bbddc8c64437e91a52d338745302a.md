### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Deny Staker Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is documented as callable by "Only starkware sequencer," but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` to consume the per-block reward slot without distributing rewards, permanently blocking the legitimate sequencer from distributing consensus rewards to a targeted staker.

### Finding Description
The specification explicitly restricts `update_rewards` to the Starkware sequencer:

> **access control:** Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. The function body only validates:
1. Contract is not paused (`general_prerequisites`)
2. Current block is newer than `last_reward_block`
3. The target staker exists and is active with non-zero balance [2](#0-1) 

There is no `assert!(get_caller_address() == sequencer_address, ...)` or equivalent guard anywhere in `StakingRewardsManagerImpl::update_rewards`.

The critical state mutation is on line 1485: `self.last_reward_block.write(current_block_number)` — this write happens unconditionally before the `disable_rewards` branch check. When an attacker calls with `disable_rewards: true`, the slot is consumed and the function returns early without distributing any rewards. [3](#0-2) 

The guard that prevents double-calling is `current_block_number > self.last_reward_block.read()`. Once the attacker's call sets `last_reward_block` to the current block, the sequencer's legitimate call in the same block will revert with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

### Impact Explanation
An attacker who front-runs the sequencer every block with `update_rewards(victim_staker, disable_rewards: true)` permanently prevents the victim staker (and their delegation pools) from accumulating any consensus-era block rewards. The staker's `unclaimed_rewards_own` is never incremented, and pool members never receive their share. This constitutes **permanent freezing of unclaimed yield** for the targeted staker and all their delegators.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield or unclaimed royalties.**

### Likelihood Explanation
- The function is publicly callable with no authentication barrier.
- The only cost to the attacker is gas per block; there is no economic stake required.
- A single attacker can target any staker address by passing it as the `staker_address` argument.
- The attack is fully permissionless and requires no privileged access, leaked keys, or external dependencies.
- Starknet transaction fees are low, making sustained per-block griefing economically viable.

### Recommendation
Add a caller check at the top of `update_rewards` that asserts the caller is the authorized sequencer address (stored in contract storage, analogous to how `update_rewards_from_attestation_contract` asserts `assert_caller_is_attestation_contract`):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

The sequencer address should be stored at initialization and be updatable only by a privileged role (e.g., `token_admin`), consistent with how `reward_supplier` is managed. [5](#0-4) 

### Proof of Concept
1. Consensus rewards are activated (`set_consensus_rewards_first_epoch` has been called and K epochs have passed).
2. Staker Alice stakes and her balance becomes effective (K epochs after `stake`).
3. Each block, attacker Bob calls `staking.update_rewards(alice_address, disable_rewards: true)` before the sequencer.
4. `last_reward_block` is set to the current block number; no rewards are distributed.
5. The sequencer's subsequent call to `update_rewards(alice_address, disable_rewards: false)` reverts with `REWARDS_ALREADY_UPDATED`.
6. Alice's `unclaimed_rewards_own` remains zero indefinitely; her pool members also receive nothing.
7. Alice can call `claim_rewards` but receives zero tokens — her yield is permanently frozen as long as Bob continues the attack. [2](#0-1)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
