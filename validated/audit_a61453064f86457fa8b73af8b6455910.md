### Title
`precompiledCalls` Field in `EVM.TransactionExecuted` Events Lacks Cryptographic Integrity Binding — Malicious Access/Observer Node Can Silently Corrupt Off-Chain EVM State Reconstruction - (File: `fvm/evm/offchain/sync/replay.go`)

---

### Summary

The `precompiledCalls` field emitted in every `EVM.TransactionExecuted` event is consumed verbatim by the off-chain EVM replay engine to inject Cadence Arch precompile outputs (e.g., `getFlowBlockHeight`, `getFlowBlockHash`). This field is not bound to any cryptographic commitment that a client can independently verify — it is absent from the EVM transaction hash, the EVM block hash, `TransactionHashRoot`, and `ReceiptRoot`. The only guard, `stateUpdateChecksum`, is a 4-byte value that lives in the same mutable event payload and is trivially co-tampered by any node serving the event. A malicious unstaked Access or Observer node can therefore serve a modified `precompiledCalls` (with a matching recomputed `stateUpdateChecksum`) and silently drive off-chain replayers into a divergent EVM state.

---

### Finding Description

**How `precompiledCalls` is produced and consumed**

During on-chain EVM execution the `CallTracker` records every call made to Cadence Arch precompiles. The encoded blob is stored in `Result.PrecompiledCalls` and written verbatim into the `EVM.TransactionExecuted` Cadence event: [1](#0-0) 

The off-chain replayer reads this blob back and injects it as a `ReplayerPrecompiledContract` so that the EVM re-execution sees the same precompile outputs without needing the live Cadence state: [2](#0-1) 

**What is NOT committed to**

The EVM transaction hash covers only the standard Ethereum transaction fields (nonce, gas, to, value, data, v/r/s). `precompiledCalls` is not an Ethereum field. The EVM block hash covers `ParentBlockHash`, `Height`, `Timestamp`, `TotalSupply`, `ReceiptRoot`, `TransactionHashRoot`, `TotalGasUsed`, `PrevRandao`: [3](#0-2) 

`TransactionHashRoot` is a Merkle root of EVM tx hashes: [4](#0-3) 

None of these structures include `precompiledCalls`. The field is therefore invisible to any commitment a client can derive from the block header alone.

**The `stateUpdateChecksum` guard is defeatable**

`ValidateResult` checks a 4-byte `stateUpdateChecksum` against the state-change delta produced by replay: [5](#0-4) 

This guard has two weaknesses:

1. **It is skipped entirely for legacy events** (empty checksum `{0,0,0,0}`): [6](#0-5) 

2. **It lives in the same mutable event payload.** A malicious Access/Observer node that tampers with `precompiledCalls` can re-execute the transaction locally with the injected precompile outputs, compute the resulting state-change delta, derive the matching 4-byte checksum, and write it back into `stateUpdateChecksum` in the same event. The client receives a self-consistent but fabricated event and the `ValidateResult` check passes.

`ValidateResult` does **not** independently verify `precompiledCalls` against any external commitment: [7](#0-6) 

**The Cadence event definition confirms the field is unbound**

The `EVM.TransactionExecuted` event definition in the EVM contract lists `precompiledCalls` as a plain `[UInt8]` field with no hash or commitment counterpart in the block or receipt structures: [8](#0-7) 

---

### Impact Explanation

Off-chain replayers (`ReplayBlockExecution`, `ReplayEVMEventsToStore`, `OffchainReplayBackwardCompatibilityTest`) reconstruct the full EVM state from events. Any application that relies on this reconstructed state — indexers, bridges, cross-VM asset accounting tools — will silently operate on a fabricated EVM state. Specifically:

- EVM contracts that call Cadence Arch for Flow block height/hash (e.g., time-locked vaults, randomness consumers) will have their precompile return values replaced, producing incorrect storage slots and log events in the replayed state.
- A bridge that reads replayed EVM state to authorize asset releases can be fed a state where a lock has "expired" or a balance has "changed," enabling unauthorized withdrawals.
- The `stateUpdateChecksum` bypass for legacy events means the attack requires zero computational effort for any historical block. [9](#0-8) 

---

### Likelihood Explanation

An unstaked Access or Observer node is explicitly in scope as an attacker. Such a node:

- Serves `GetEventsForHeightRange` / `GetTransactionResult` responses directly from its local index with no additional signature or proof.
- Can modify the CCF-encoded event payload before returning it to any client that does not independently verify the `EventCollection` Merkle root from the sealed execution result.
- Needs only to re-execute the target transaction locally (the EVM emulator is open-source) to produce a consistent `stateUpdateChecksum` for the tampered `precompiledCalls`.

No staked node compromise, no key leak, and no brute force is required.

---

### Recommendation

1. **Bind `precompiledCalls` to a verifiable commitment.** Include a cryptographic hash of `precompiledCalls` (e.g., SHA3-256) in the EVM block structure or in a per-transaction receipt field that is covered by `ReceiptRoot` or `TransactionHashRoot`. Clients can then verify the hash independently from the block header.

2. **Replace the 4-byte `stateUpdateChecksum` with a full-length hash** (32 bytes) that covers both the state-change delta **and** the `precompiledCalls` input, making co-tampering computationally infeasible.

3. **Require clients to verify events against the `EventCollection` Merkle root** before passing them to `ReplayBlockExecution`. The `EventCollection` is already committed to in the `ChunkBody` and verified by verification nodes; exposing Merkle proofs via the Access API would close the gap. [10](#0-9) 

---

### Proof of Concept

```
Attacker = malicious Observer node

1. Observer intercepts client request for EVM.TransactionExecuted events at block B.
2. For target tx T that called getFlowBlockHeight() via Cadence Arch:
   a. Original event: precompiledCalls = encode([{addr: CadenceArch, runCalls: [{output: height=1000}]}])
                      stateUpdateChecksum = 0xAABBCCDD
   b. Attacker re-executes T locally with injected output height=9999999.
   c. Attacker computes new stateUpdateChecksum = 0x11223344 from the tampered delta.
   d. Attacker serves modified event:
        precompiledCalls = encode([{addr: CadenceArch, runCalls: [{output: height=9999999}]}])
        stateUpdateChecksum = 0x11223344
3. Client calls ReplayBlockExecution with the tampered events.
4. replayTransactionExecution injects height=9999999 via ReplayerPrecompiledContract.
5. ValidateResult passes: gas, error code, logs, deployed address, and stateUpdateChecksum all match.
6. Client's reconstructed EVM state reflects height=9999999 instead of 1000.
7. A time-lock contract that checks "blockHeight > 5000000" now appears unlocked in the
   client's view, enabling a bridge to release assets prematurely.
``` [11](#0-10) [12](#0-11)

func ValidateResult(
	res *types.Result,
	txEvent *events.TransactionEventPayload,
) error {

	// we should never produce invalid transaction, since if the transaction was emitted from the evm core
	// it must have either been successful or failed, invalid transactions are not emitted
	if res.Invalid() {
		return fmt.Errorf("invalid transaction: %w", res.ValidationError)
	}

	// check gas consumed
	if res.GasConsumed != txEvent.GasConsumed {
		return fmt.Errorf("gas consumption mismatch %d != %d", res.GasConsumed, txEvent.GasConsumed)
	}

	// check error code
	txEventErrorCode := types.ErrorCode(txEvent.ErrorCode)
	if errorCode := res.ResultSummary().ErrorCode; errorCode != txEventErrorCode {
		return fmt.Errorf("error code mismatch %d != %d", errorCode, txEventErrorCode)
	}

	// check encoded logs
	encodedLogs, err := res.RLPEncodedLogs()
	if err != nil {
		return fmt.Errorf("failed to RLP-encode logs: %w", err)
	}
	if !bytes.Equal(encodedLogs, txEvent.Logs) {
		return fmt.Errorf("encoded logs mismatch %x != %x", encodedLogs, txEvent.Logs)
	}

	// check deployed address
	if deployedAddress := res.DeployedContractAddressString(); deployedAddress != txEvent.ContractAddress {
		return fmt.Errorf("deployed address mismatch %s != %s", deployedAddress, txEvent.ContractAddress)
	}

	// check the state change checksum
	// if empty checksum skip (supporting blocks before checksum integration)
	if checksum := res.StateChangeChecksum(); txEvent.StateUpdateChecksum != emptyChecksum &&
		checksum != txEvent.StateUpdateChecksum {
		return fmt.Errorf("state change checksum mismatch %x != %x", checksum, txEvent.StateUpdateChecksum)
	}

	return nil

func ReplayEVMEventsToStore(
	log zerolog.Logger,
	store environment.ValueStore,
	chainID flow.ChainID,
	rootAddr flow.Address,
	evmBlockEvent *events.BlockEventPayload, // EVM block event
	evmTxEvents []events.TransactionEventPayload, // EVM transaction event
) (
	map[flow.RegisterID]flow.RegisterValue, // EVM state transition updates
	map[flow.RegisterID]flow.RegisterValue, // block provider updates
	error,
) {

	bpStorage := evmStorage.NewEphemeralStorage(store)
	bp, err := blocks.NewBasicProvider(chainID, bpStorage, rootAddr)
	if err != nil {
		return nil, nil, err
	}

	err = bp.OnBlockReceived(evmBlockEvent)
	if err != nil {
		return nil, nil, err
	}

	sp := testutils.NewTestStorageProvider(store, evmBlockEvent.Height)
	cr := sync.NewReplayer(chainID, rootAddr, sp, bp, log, nil, true)
	res, results, err := cr.ReplayBlockEvents(evmTxEvents, evmBlockEvent)
	if err != nil {
		return nil, nil, err
	}

	// commit all register changes from the EVM state transition
	for k, v := range res.StorageRegisterUpdates() {
		err = store.SetValue([]byte(k.Owner), []byte(k.Key), v)
		if err != nil {
			return nil, nil, err
		}
	}

	blockProposal := blocks.ReconstructProposal(evmBlockEvent, results)

	err = bp.OnBlockExecuted(evmBlockEvent.Height, res, blockProposal)
	if err != nil {
		return nil, nil, err
	}

	// commit all register changes from non-EVM state transition, such
	// as block hash list changes
	for k, v := range bpStorage.StorageRegisterUpdates() {
		// verify the block hash list changes are included in the trie update

		err = store.SetValue([]byte(k.Owner), []byte(k.Key), v)
		if err != nil {
			return nil, nil, err
		}
	}

	return res.StorageRegisterUpdates(), bpStorage.StorageRegisterUpdates(), nil
