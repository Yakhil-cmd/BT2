### Title
Missing Minimum Threshold Validation in FROST Presign and Sign Initialization — (`src/frost/mod.rs`)

### Summary
The FROST presign and sign validation functions in `src/frost/mod.rs` omit the lower-bound check `threshold < 2` that every other protocol entry point in the codebase enforces. A malicious library caller or coordinator can supply `threshold = 1`, reducing the effective signing quorum to a single participant and enabling unauthorized signature creation without the required multi-party cooperation.

### Finding Description

Every other threshold-bearing entry point in the codebase enforces both an upper bound and a lower bound on the threshold value:

`src/dkg.rs` `assert_key_invariants` (lines 572–582): [1](#0-0) 

```rust
if threshold > participants.len() { ... ThresholdTooLarge ... }
if threshold < 2 { ... ThresholdTooSmall ... }   // ← lower-bound guard
```

`src/ecdsa/ot_based_ecdsa/triples/generation.rs` `validate_triple_inputs` (lines 693–704): [2](#0-1) 

```rust
if threshold_value > participants.len() { ... }
if threshold_value < 2 { ... ThresholdTooSmall ... }   // ← lower-bound guard
```

The FROST entry points perform **only** the upper-bound check and silently accept any value ≥ 1:

`src/frost/mod.rs` `presign` (lines 71–77): [3](#0-2) 

```rust
// validate threshold
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← NO check for threshold < 2
```

`src/frost/mod.rs` `assert_sign_inputs` (lines 144–150): [4](#0-3) 

```rust
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// ← NO check for threshold < 2
```

`ReconstructionLowerBound` is a plain `usize` newtype with no internal invariant enforcing a minimum of 2: [5](#0-4) 

So `ReconstructionLowerBound(1)` (or even `ReconstructionLowerBound(0)`) passes both FROST guards without error.

### Impact Explanation

In FROST, the coordinator aggregates exactly `threshold` signature shares via Lagrange interpolation to reconstruct the group signature. With `threshold = 1`, the Lagrange coefficient for the single contributing participant equals 1 (trivial interpolation), meaning the coordinator's own share alone is sufficient to produce a cryptographically valid signature over any attacker-chosen message. No other participant's nonce or share is required. This directly enables **unauthorized creation of a valid threshold signature for attacker-chosen inputs** — the Critical impact tier.

### Likelihood Explanation

The entry points `presign` and `assert_sign_inputs` are public library API. Any caller constructing a `PresignArguments` or invoking `assert_sign_inputs` controls the `threshold` field directly. A malicious coordinator who is also a library caller (a documented threat actor in threshold-signature systems) can trivially pass `threshold = 1` without triggering any error. The participants list still satisfies `len() >= 2`, so all other guards pass. No cryptographic capability or key leakage is required.

### Recommendation

Add the same lower-bound guard present in `assert_key_invariants` and `validate_triple_inputs` to both FROST entry points:

```rust
// In frost/mod.rs presign(), after the upper-bound check:
if args.threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: args.threshold.value(),
        min: 2,
    });
}

// In frost/mod.rs assert_sign_inputs(), after the upper-bound check:
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Optionally, enforce the invariant inside `ReconstructionLowerBound::from` / a constructor so it is impossible to construct a value below 2 at the type level. [5](#0-4) 

### Proof of Concept

```rust
use threshold_signatures::frost::{presign, assert_sign_inputs, PresignArguments};
use threshold_signatures::{ReconstructionLowerBound, KeygenOutput};
use threshold_signatures::participants::Participant;

let p1 = Participant::from(1u32);
let p2 = Participant::from(2u32);
let participants = vec![p1, p2];

// threshold = 1 — accepted without error by both FROST guards
let threshold = ReconstructionLowerBound::from(1usize);

// assert_sign_inputs returns Ok even though threshold < 2
let result = assert_sign_inputs(&participants, threshold, p1, p1);
assert!(result.is_ok()); // passes — missing ThresholdTooSmall guard

// presign also accepts threshold = 1
let args = PresignArguments {
    keygen_out: /* any valid KeygenOutput */,
    threshold,
};
let proto = presign::<SomeCiphersuite>(&participants, p1, &args, rng);
assert!(proto.is_ok()); // passes — missing ThresholdTooSmall guard

// With threshold=1, the coordinator (p1) can complete signing
// using only their own Lagrange share, producing a valid group
// signature over any chosen message without p2's participation.
```
