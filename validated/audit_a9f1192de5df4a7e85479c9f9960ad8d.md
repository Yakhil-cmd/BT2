### Title
Missing Permission Check and No-Op Bridge State in `msgServer.TurnBridge` — (`x/cronos/keeper/msg_server.go`)

### Summary
`msgServer.TurnBridge` is an unimplemented stub. It performs no `CanTurnBridge` permission check and writes no state. This produces two simultaneous failures: (1) any unprivileged address can submit `MsgTurnBridge` and receive a success response, bypassing the intended authorization gate; (2) even a legitimately permissioned address (including the admin) cannot actually disable the gravity bridge, rendering the emergency circuit breaker completely non-functional.

### Finding Description
The Cronos module defines a permission-gated message `MsgTurnBridge` intended to allow authorized addresses to enable or disable the gravity bridge as an emergency circuit breaker (see ADR-009). The permission bit `CanTurnBridge` is defined, the `HasPermission` helper is implemented, and the sibling handler `UpdateTokenMapping` correctly enforces its analogous permission. However, the `TurnBridge` handler body is:

```go
// x/cronos/keeper/msg_server.go:84-87
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

No permission check is performed. No bridge state (e.g., a `BridgeEnabled` flag in params or a dedicated store key) is written. The function is a dead stub. [1](#0-0) 

### Impact Explanation

**Authorization bypass (High):** Because `TurnBridge` returns `nil, nil` for every caller, any address — including one with zero permissions — can submit `MsgTurnBridge` and receive a transaction success. The `CanTurnBridge` permission check is entirely absent. The integration test at `test_gravity.py:661-662` explicitly requires `code != 0` for an unpermissioned caller; the current code returns `code == 0`. [6](#0-5) 

**Broken circuit breaker (High):** Because no state is written, even the admin cannot disable the bridge. In an emergency (e.g., a gravity bridge exploit), the intended mitigation path is completely non-functional. The integration test at `test_gravity.py:664-674` expects that after a successful `TurnBridge(false)`, subsequent bridge operations fail — this invariant is also violated. [7](#0-6) 

### Likelihood Explanation
The message is registered in the router and reachable by any funded account via a standard Cosmos SDK transaction. No special access or leaked keys are required. The bypass is unconditional.

### Recommendation
Implement `TurnBridge` analogously to `UpdateTokenMapping`:

1. Check `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `sdkerrors.ErrUnauthorized` on failure.
2. Write the bridge-enabled state (e.g., update a `BridgeEnabled` param or a dedicated store key).
3. Ensure the gravity bridge end-blocker and IBC hooks read this state and gate their logic accordingly.

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // write bridge enabled/disabled state
    // ...
    return &types.MsgTurnBridgeResponse{}, nil
}
``` [8](#0-7)

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
