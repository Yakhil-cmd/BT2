### Title
Missing `msg_hash == 0` Validation in OT-Based ECDSA `sign()` Enables Presign Secret Disclosure — (File: `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary
The OT-based ECDSA `sign()` function accepts a zero `msg_hash` without rejection. When `msg_hash = 0`, each participant's signature share collapses to `s_i = r * sigma_i`, directly exposing the presign secret `sigma_i` (a share of `k * x`) to the coordinator. The robust ECDSA `sign()` function has an explicit guard against this; the OT-based path does not.

---

### Finding Description

`src/ecdsa/robust_ecdsa/sign.rs` lines 91–95 explicitly rejects a zero message hash:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
    ));
}
```

The `grep_search` confirms `msg_hash.is_zero()` appears **only** in `src/ecdsa/robust_ecdsa/sign.rs` — it is entirely absent from the OT-based ECDSA code path.

`src/ecdsa/ot_based_ecdsa/sign.rs` lines 22–76 validate participant lists, coordinator membership, and threshold, but perform **no check on `msg_hash`**.

The signature share computation at lines 139–159 is:

```rust
fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    presignature: &RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<Scalar, ProtocolError> {
    let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
    let k_i = lambda * presignature.k;
    let sigma_i = lambda * presignature.sigma;
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)   // ← when msg_hash=0: s_i = r * sigma_i
}
```

`RerandomizedPresignOutput` (`src/ecdsa/ot_based_ecdsa/mod.rs` lines 55–63) carries:
- `k`: "Our rerandomized share of the nonce value"
- `sigma`: "Our rerandomized share of the sigma value" (i.e., a share of `k_nonce * x`)

When `msg_hash = 0`, the `k_i` term vanishes and `s_i = r * sigma_i`. Because `r = x_coordinate(big_r)` is a public value, the coordinator immediately recovers `sigma_i = s_i / r` for every participant, then reconstructs the aggregate presign secret `sigma = k_nonce * x`.

---

### Impact Explanation

**Presign secret disclosure (Critical).** `sigma_i = lambda_i * k_nonce * x_i` is a secret computed during presigning and is never supposed to be individually visible to the coordinator. Forcing `msg_hash = 0` makes every participant's share trivially recoverable from the message they send.

**Full private-key extraction (Critical, with presignature reuse).** Once the coordinator holds `sigma_i` for all participants, a second signing session over the same presignature with any `h' ≠ 0` yields `s'_i = h' * k_i + r * sigma_i`. Subtracting the known `r * sigma_i` gives `k_i = (s'_i − r * sigma_i) / h'`. Reconstructing `k_nonce` from the shares and dividing `sigma / k_nonce` recovers the private key `x`. Presignature reuse is not enforced at the protocol level; the library only documents it as a design principle.

---

### Likelihood Explanation

In typical MPC deployments the coordinator is the entity that proposes the message hash to all participants. A malicious coordinator can supply `msg_hash = 0` to every participant without any protocol-level rejection in the OT-based path. No special privilege beyond being the coordinator is required. The entry point is the public `sign()` API.

---

### Recommendation

Add the same guard that exists in the robust ECDSA path to `src/ecdsa/ot_based_ecdsa/sign.rs`, immediately after the threshold check:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0".to_string(),
    ));
}
```

### Proof of Concept

1. Malicious coordinator initiates a signing session and supplies `msg_hash = Scalar::ZERO` to all participants.
2. Each honest participant calls `sign(…, msg_hash = 0)` — no check rejects it (`src/ecdsa/ot_based_ecdsa/sign.rs` lines 22–76).
3. `compute_signature_share` (lines 139–159) evaluates `0 * k_i + r * sigma_i = r * sigma_i` and returns `s_i`.
4. Each participant sends `s_i` to the coordinator via `chan.send_private`.
5. Coordinator computes `sigma_i = s_i * r^{-1}` for each participant — presign secret `sigma_i` is now disclosed.
6. Coordinator reconstructs `sigma = k_nonce * x` from the shares.
7. *(Key extraction)* Coordinator reuses the same `RerandomizedPresignOutput` with `h' ≠ 0`; receives `s'_i = h' * k_i + r * sigma_i`; solves `k_i = (s'_i − r * sigma_i) / h'`; reconstructs `k_nonce`; computes `x = sigma / k_nonce`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,

    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
