### Title
`native_per_pubdata` Truncates to Zero via Floor Division, Making Pubdata Completely Free — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

`native_per_pubdata` is computed with floor (truncating) integer division. When `pubdata_price < native_price`, the result is zero. Because no guard rejects a zero `native_per_pubdata`, every subsequent pubdata charge in the transaction lifecycle multiplies by zero, making all pubdata consumption completely free. The pubdata counter still advances normally; only the cost is silently zeroed. This is the direct ZKsync OS analog of the Yield ERC20Rewards accumulator bug: the "rate" truncates to zero while the "counter" keeps ticking.

---

### Finding Description

**Root cause — floor division instead of ceiling division**

In `validation_impl.rs` line 142:

```rust
// We checked native_price != 0 above
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`wrapping_div` is Rust's standard truncating (floor) integer division. Whenever `pubdata_price < native_price`, the quotient is 0. No subsequent check rejects `native_per_pubdata == 0` as invalid; the transaction proceeds normally.

The identical pattern appears in the API helper at `api/src/helpers.rs` line 427:

```rust
// native_per_pubdata = pubdata_price / native_price
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
```

**Contrast with `native_per_gas`**

Immediately above, `native_per_gas` is computed with *ceiling* division (`div_ceil`) at line 135:

```rust
u256_try_to_u64(&gas_price.div_ceil(native_price))
```

The asymmetry is intentional for gas (ceiling ensures the user always pays at least one native unit per gas unit), but the same conservative rounding was not applied to pubdata.

**Downstream propagation of zero**

Once `native_per_pubdata = 0`, every pubdata cost calculation in the transaction lifecycle collapses:

1. **Intrinsic pubdata overhead** (`gas_helpers.rs` line 352):
   ```rust
   let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
   // → 0 * intrinsic_pubdata = 0
   ```
   No native is deducted from the prepaid budget for intrinsic pubdata.

2. **Post-execution pubdata charge** (`gas_helpers.rs` lines 430–432):
   ```rust
   let native = current_pubdata_spent
       .checked_mul(native_per_pubdata)   // → 0
       .ok_or(out_of_native_resources!())?;
   ```
   No matter how many storage slots the transaction writes, the native cost is zero.

3. **Pubdata sufficiency check** (`gas_helpers.rs` line 454):
   ```rust
   let enough = resources.has_enough(&resources_for_pubdata);
   // resources_for_pubdata is zero → always passes
   ```
   The post-execution revert path for "not enough gas for pubdata" is never triggered.

The pubdata counter (`net_pubdata_used`) still increments correctly for every storage write; only the *cost* is zeroed. This is structurally identical to the Yield bug: `lastUpdated` advances while `accumulated` stays constant.

---

### Impact Explanation

When `pubdata_price < native_price` (a realistic block-context configuration), any unprivileged user can:

- Write to an arbitrary number of storage slots within the block's pubdata limit and pay **zero** native cost for those state diffs.
- Effectively spam the chain with state changes at no marginal cost beyond EVM gas, exhausting the block's pubdata budget and crowding out legitimate transactions.
- Cause the operator to bear the full L1 data-availability cost for pubdata that was never charged to the sender.

The financial loss falls on the operator/protocol: pubdata fees that should have been collected are silently waived. The block pubdata limit is the only bound on how much free pubdata a single transaction can consume.

---

### Likelihood Explanation

The condition `pubdata_price < native_price` is realistic and does not require any privileged action by the attacker:

- The operator sets both prices in the block context. During periods of low L1 gas prices (low `pubdata_price`) relative to the native token price (high `native_price`), the condition holds naturally.
- The default test configuration in `BlockMetadataFromOracle::new_for_test()` sets `pubdata_price = 0` and `native_price = 10`, demonstrating the condition is considered normal.
- An attacker only needs to observe the public block context and submit a storage-heavy transaction when `pubdata_price < native_price`.
- No signature forgery, privileged key, or governance manipulation is required.

---

### Recommendation

Replace the floor division with ceiling division for `native_per_pubdata`, mirroring the treatment of `native_per_gas`:

```rust
// Before (floor division — can produce 0 when pubdata_price < native_price)
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (ceiling division — always charges at least 1 native unit when pubdata_price > 0)
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix to `api/src/helpers.rs` line 427. Additionally, consider adding an explicit guard that rejects (or warns on) `native_per_pubdata == 0` when `pubdata_price > 0`, to make the invariant explicit.

---

### Proof of Concept

**Setup**: Deploy a contract that writes to 100 distinct storage slots. Set the block context with `pubdata_price = 500` and `native_price = 1000` (both non-zero, but `pubdata_price < native_price`).

**Expected (correct) behavior**: `native_per_pubdata = ceil(500 / 1000) = 1`. Each pubdata byte costs 1 native unit. Writing 100 × 32-byte keys + value diffs ≈ 6400 bytes of pubdata costs ≈ 6400 native units.

**Actual (buggy) behavior**:
```
native_per_pubdata = 500 / 1000 = 0   (floor division)
intrinsic_pubdata_overhead = 0 * L2_TX_INTRINSIC_PUBDATA = 0
post_execution_pubdata_cost = 6400 * 0 = 0
```
The transaction succeeds, 6400 bytes of pubdata are written to the state diff, and the sender pays **zero** native cost for pubdata. The operator absorbs the full L1 data-availability cost.
