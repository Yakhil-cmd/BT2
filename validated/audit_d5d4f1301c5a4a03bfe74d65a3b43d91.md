### Title
WalletConnect `chia_sendTransaction` `puzzle_decorator` Clawback Timelock Displayed as Opaque JSON Without Humanization — (`packages/gui/src/electron/commands/Commands.ts`)

### Summary
A WalletConnect dApp with `chia_sendTransaction` permission can inject an arbitrary `puzzle_decorator` value (e.g., `[{decorator:'CLAWBACK',clawbackTimelock:1}]`). The approval dialog renders the entire field as raw JSON. The `clawbackTimelock` integer (in seconds) is never converted to a human-readable duration, so the user cannot determine whether the clawback window is 1 second, 1 hour, or 1 year before clicking "Send."

### Finding Description
`puzzle_decorator` is declared in the `chia_wallet.send_transaction` command schema with `type: 'json'` and no `humanize` property. [1](#0-0) 

`humanizeParamValue` serializes `json`-typed params with `JSONBig.stringify`, producing a raw JSON blob in the confirmation dialog. [3](#0-2) 

The `clawbackTimelock` field is a raw integer count of seconds. The normal send UI (`WalletSend.tsx`) collects days/hours/minutes from the user and converts them; the WalletConnect path has no equivalent conversion or display.

### Impact Explanation
- A dApp sets `clawbackTimelock: 1` (1 second — effectively no clawback protection). The user sees `"clawbackTimelock": 1` in the dialog and has no way to know this is 1 second, not 1 day.
- A dApp sets `clawbackTimelock: 31536000` (1 year). The recipient's funds are locked for a year; the user approves without understanding the duration.
- In both cases the user approves a send with the wrong clawback status because the security-critical parameter is displayed as opaque JSON.

This fits the High impact category: *"Corruption, spoofing, or unsafe trust of WalletConnect state that causes a user to approve... the wrong... status."*

### Likelihood Explanation
Any dApp that has been granted `chia_sendTransaction` permission (a common, expected permission for DeFi dApps) can trigger this. No additional privileges are required. The precondition is the normal pairing flow.

### Recommendation
Add a `humanize` handler for `puzzle_decorator` in `humanizeParamValue.ts` that detects `CLAWBACK` decorator entries and converts `clawbackTimelock` (seconds) to a human-readable string (e.g., "Clawback: 2 days, 3 hours"). Alternatively, add a dedicated `type: 'clawback'` or structured display for this param in the command schema, similar to how `mojo-to-xch` is handled for amounts.

### Proof of Concept
1. Pair a WalletConnect dApp and grant it `chia_sendTransaction`.
2. Send a `chia_sendTransaction` request with:
   ```json
   {
     "amount": "1000000000",
     "fee": "0",
     "address": "txch1...",
     "puzzleDecorator": [{"decorator": "CLAWBACK", "clawbackTimelock": 1}]
   }
   ```
3. Observe the confirmation dialog: "Puzzle Decorator: `[{"decorator":"CLAWBACK","clawbackTimelock":1}]`"
4. Assert that no human-readable duration ("1 second") is shown anywhere in the dialog.
5. Click "Send" — the transaction is submitted with a 1-second clawback window.
