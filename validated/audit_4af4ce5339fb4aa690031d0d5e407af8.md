### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Burn a Staker's Block Rewards via `disable_rewards` Flag - (File: src/staking/staking.cairo)

---

### Summary

`IStakingRewardsManager::update_rewards` imposes no caller restriction. Any address can invoke it for any staker with `disable_rewards: true`, which advances `last_reward_block` to the current block while skipping reward distribution. Because the per-block guard fires on the block number, the legitimate call by the staker in the same block is permanently rejected, and that block's rewards are irrecoverably lost.

---

### Finding Description

`update_rewards` is the consensus-era entry point through which a block proposer claims per-block STRK/BTC rewards:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: unpaused + caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← rewards skipped, block consumed
    }
    ...
``` [1](#0-0) 

`general_prerequisites` only asserts the contract is unpaused and the caller is non-zero — there is no check that the caller is the staker, the staker's operational address, or any privileged role: [2](#0-1) 

The `IStakingRewardsManager` interface exposes this function with no documented access restriction: [3](#0-2) 

**Attack path:**

1. In block N, the legitimate block proposer (staker's operational address) submits `update_rewards(staker_A, disable_rewards: false)`.
2. An attacker submits `update_rewards(staker_A, disable_rewards: true)` in the same block, ordered before the legitimate call.
3. The attacker's call writes `last_reward_block = N` and returns without distributing rewards.
4. The staker's call hits `assert!(current_block_number > self.last_reward_block.read())` → panics with `REWARDS_ALREADY_UPDATED`.
5. Block N's rewards for staker A are permanently lost — `last_reward_block` is already at N, so no future call can reclaim them.

The attacker repeats this every block to deny the staker all consensus rewards indefinitely.

---

### Impact Explanation

Each suppressed call permanently destroys that block's STRK (and BTC) rewards for the targeted staker. Rewards are not deferred — `last_reward_block` is advanced, making the slot unclaimable. Sustained execution across many blocks constitutes **permanent freezing of unclaimed yield** (High) and, because the attacker can selectively target specific stakers, amounts to **theft of unclaimed yield** (High) relative to the honest distribution.

---

### Likelihood Explanation

On Starknet the sequencer currently controls transaction ordering, so the attacker must either be the sequencer or collude with it. However:

- The vulnerability is a protocol-level missing enforcement, not a sequencer-specific assumption.
- As Starknet moves toward decentralized sequencing, this becomes exploitable by any participant who can influence intra-block ordering.
- The cost to the attacker is only gas per block; there is no capital requirement.
- The victim has no on-chain recourse once the block slot is consumed.

Likelihood: **Medium** (currently gated by sequencer trust; becomes High under decentralized sequencing).

---

### Recommendation

Restrict `update_rewards` so only the staker's registered operational address (or the staker address itself) may call it:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let caller = get_caller_address();
    let staker_info = self.internal_staker_info(:staker_address);
    assert!(
        caller == staker_address || caller == staker_info.operational_address,
        "{}",
        Error::UNAUTHORIZED_CALLER,
    );
    ...
```

Alternatively, remove the `disable_rewards` parameter from the public interface and handle the migration/pre-consensus skip internally, so no external caller can suppress reward distribution.

---

### Proof of Concept

```
Block N:
  tx1 (attacker): update_rewards(staker=ALICE, disable_rewards=true)
      → last_reward_block := N
      → rewards skipped (returns early)
      → ALICE.unclaimed_rewards_own unchanged

  tx2 (ALICE's node): update_rewards(staker=ALICE, disable_rewards=false)
      → assert!(N > N)  ← FAILS: REWARDS_ALREADY_UPDATED
      → ALICE earns zero rewards for block N

Block N+1:
  tx1 (attacker): update_rewards(staker=ALICE, disable_rewards=true)
      → last_reward_block := N+1
      → rewards skipped again
  ...
```

Repeated every block, ALICE's `unclaimed_rewards_own` never increases despite being an active block proposer. The attacker spends only gas; ALICE loses all consensus-era yield. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
