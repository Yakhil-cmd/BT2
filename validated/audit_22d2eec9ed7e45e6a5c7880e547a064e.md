Audit Report

## Title
Single Failing ICP Transfer in `sweep_icp` Blocks Entire SNS Swap Finalization Pipeline - (File: rs/sns/swap/src/swap.rs)

## Summary
In `finalize_inner`, after calling `sweep_icp`, the pipeline immediately checks `has_error_message()` and returns early if any single buyer's ICP transfer fails or is classified as invalid. This halts all subsequent finalization steps — SNS token distribution, neuron creation, and governance mode transition — for all participants, not just the failing one. The `invalid` case (`AmountTooSmall`) is explicitly unretryable and permanently blocks finalization on every future call.

## Finding Description
`finalize_inner` in `rs/sns/swap/src/swap.rs` calls `sweep_icp` and immediately gates the rest of the pipeline on the result: [1](#0-0) 

`set_sweep_icp_result` in `rs/sns/swap/src/types.rs` sets an error message whenever `is_successful_sweep()` returns false, which occurs when `failure > 0` OR `invalid > 0`: [2](#0-1) 

Inside `sweep_icp`, the loop does continue past individual failures — it increments counters and uses `continue` — so per-buyer iteration is not blocked. However, the pipeline-level check after `sweep_icp` returns treats any partial failure as a hard stop: [3](#0-2) 

The `AmountTooSmall` / `invalid` path is explicitly noted in the code as requiring "manual intervention via an upgrade to correct": [4](#0-3) 

This means a buyer whose `amount_e8s < DEFAULT_TRANSFER_FEE` will always produce `invalid += 1` on every retry, permanently preventing `is_successful_sweep()` from returning true, and permanently blocking all downstream steps: `settle_neurons_fund_participation`, `create_sns_neuron_recipes`, `sweep_sns`, `claim_swap_neurons`, and `set_sns_governance_to_normal_mode`.

## Impact Explanation
This is a **High** severity finding matching the allowed impact: *"Significant SNS security impact with concrete user or protocol harm."* When a committed SNS swap has even one buyer whose ICP transfer is classified `invalid`, SNS tokens are never distributed to any participant, SNS neurons are never created, and SNS Governance remains permanently in `PreInitializationSwap` mode — the SNS is operationally bricked with no path to recovery short of a canister upgrade. For the `failure` (transient) case, all participants are blocked from receiving tokens and neurons during every failed finalization attempt until the failing transfer resolves.

## Likelihood Explanation
The `failure` path (transient ledger error) is realistic and can occur in normal operation; it is self-resolving on retry but still blocks all other participants during each failed attempt. The `invalid` path requires `BuyerState.amount_e8s < DEFAULT_TRANSFER_FEE`, which the code acknowledges as a defensive case that "should never happen in production" but is reachable if ICP transfer fees change after participation (a governance action) or via a bug in `refresh_buyer_tokens`. Once reached, it is permanently unretryable without a canister upgrade. An unprivileged participant near the fee boundary combined with a fee parameter change can trigger the permanent case.

## Recommendation
1. Decouple pipeline halting from individual `failure` and `invalid` counts. Only halt on `global_failures > 0` (corrupted global state). Individual per-buyer failures should be logged and skipped, not used to block the entire pipeline.
2. Treat `invalid` entries as permanently skipped rather than as a hard stop — they are already acknowledged as unretryable and requiring manual intervention.
3. Specifically, change the `has_error_message()` gate after `sweep_icp` to only trigger on `global_failures > 0`, allowing the pipeline to proceed for all other participants even when some individual transfers fail or are invalid.

## Proof of Concept
The existing test `test_finalization_halts_when_sweep_icp_fails` in `rs/sns/swap/tests/swap.rs` directly and reproducibly confirms the full pipeline halt: [5](#0-4) 

The test sets up one `invalid` buyer (`amount_e8s < DEFAULT_TRANSFER_FEE`) and one buyer with a mocked ledger failure, calls `finalize`, and asserts that `settle_neurons_fund_participation_result`, `create_sns_neuron_recipes_result`, `sweep_sns_result`, `set_mode_call_result`, and `claim_neuron_result` are all `None` — confirming the entire pipeline is halted by a single partial sweep failure.

/// Tests that if transferring does not complete fully, finalize will halt finalization
#[tokio::test]
async fn test_finalization_halts_when_sweep_icp_fails() {
    // Step 1: Prepare the world

    // Setup the necessary buyers for the test
    let mut swap = Swap {
        lifecycle: Committed as i32,
        init: Some(init()),
        params: Some(params()),
        buyers: btreemap! {
            // This Buyer is `Invalid` because the amount committed is less than the
            // DEFAULT_TRANSFER_FEE of the ICP Ledger. This should never be possible
            // in production, but sweep_icp must handle this case.
            i2principal_id_string(1000) => BuyerState {
                icp: Some(TransferableAmount {
                    amount_e8s: DEFAULT_TRANSFER_FEE.get_e8s() - 1,
                    ..Default::default()
                }),
                has_created_neuron_recipes: Some(false),
            },
            // This buyer's state is valid, but a mock call to the ledger will fail the transfer,
            // which should result in a failure field increment.
            i2principal_id_string(1003) => BuyerState {
                icp: Some(TransferableAmount {
                    amount_e8s: 10 * E8,
                    ..Default::default()
                }),
                has_created_neuron_recipes: Some(false),
            },
        },
        ..Default::default()
    };

    let mut clients = CanisterClients {
        icp_ledger: SpyLedger::new(vec![
            // This mocked reply should produce a successful transfer in SweepResult
            LedgerReply::TransferFunds(Err(NervousSystemError::new_with_message(
                "Error when transferring funds",
            ))),
        ]),
        ..spy_clients()
    };

    // Step 2: Call sweep_icp
    let result = swap.finalize(now_fn, &mut clients).await;

    assert_eq!(
        result.sweep_icp_result,
        Some(SweepResult {
            success: 0,
            skipped: 0,
            failure: 1, // Single failed transfer
            invalid: 1, // Single invalid buyer
            global_failures: 0,
        })
    );

    assert_eq!(
        result.error_message,
        Some(String::from(
            "Transferring ICP did not complete fully, some transfers were invalid or failed. Halting swap finalization"
        ))
    );

    // Assert that all other fields are set to None because finalization was halted.
    assert!(result.settle_neurons_fund_participation_result.is_none());
    assert!(result.set_dapp_controllers_call_result.is_none());
    assert!(result.create_sns_neuron_recipes_result.is_none());
    assert!(result.sweep_sns_result.is_none());
    assert!(result.set_mode_call_result.is_none());
    assert!(result.claim_neuron_result.is_none());
}
