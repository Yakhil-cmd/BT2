### Title
Gateway L2 Gas Price Admission Check Uses Wrong Block Price Variable (`l2_gas_price` Instead of `next_l2_gas_price`) - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` compares a transaction's `max_price_per_unit` against a threshold derived from the **current block's** `strk_gas_prices.l2_gas_price`. However, the transaction will be executed in the **next block**, whose gas price is `next_l2_gas_price` — a distinct, EIP-1559-computed value stored in the same block header. The developer acknowledged this with an inline TODO. The mismatch causes the gateway to admit transactions that will fail during batcher execution (when prices rise) and to reject transactions that would succeed (when prices fall).

### Finding Description

In `validate_resource_bounds`, the gateway reads the latest committed block's info and extracts `strk_gas_prices.l2_gas_price`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`BlockHeaderWithoutHash` carries two distinct L2 gas price fields:

- `l2_gas_price` — the price **used in the already-committed block** (past).
- `next_l2_gas_price` — the EIP-1559-derived price that **will be used in the next block** (future). [2](#0-1) 

The batcher builds the next block using `next_l2_gas_price` from the latest committed header as the execution-time gas price. This is confirmed by the consensus orchestrator, which writes `next_l2_gas_price: self.l2_gas_price` into the block header and uses `self.l2_gas_price` (updated via `update_l2_gas_price`) as the price for the block being built: [3](#0-2) 

Meanwhile, `run_validate_entry_point` — the blockifier-level validation that runs after `validate_resource_bounds` — also uses the previous block's gas prices (it only increments the block number, not the price): [4](#0-3) 

So the gateway's admission threshold is computed against `l2_gas_price` (the committed block's price), while actual execution in the batcher uses `next_l2_gas_price`. These two values diverge every block via the fee market formula in `calculate_next_base_gas_price`. [5](#0-4) 

### Impact Explanation

**When `next_l2_gas_price > l2_gas_price` (rising prices):**
The gateway threshold is computed against the lower `l2_gas_price`. A transaction with `max_price_per_unit` in the range `[threshold(l2_gas_price), next_l2_gas_price)` passes gateway admission and enters the mempool, but fails `check_fee_bounds` during batcher execution because the actual execution price is `next_l2_gas_price`. The gateway admits transactions that will be rejected by the sequencer.

**When `next_l2_gas_price < l2_gas_price` (falling prices):**
The gateway threshold is computed against the higher `l2_gas_price`. A transaction with `max_price_per_unit` in the range `[threshold(next_l2_gas_price), threshold(l2_gas_price))` is rejected by the gateway even though it would succeed during batcher execution. Valid transactions are denied admission.

Both cases match the allowed impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The fee market adjusts `next_l2_gas_price` every block based on gas consumption relative to the target. Any block that is not exactly at the gas target produces a `next_l2_gas_price ≠ l2_gas_price`. With the default `min_gas_price_percentage = 100`, the threshold equals the previous block price exactly, so any price movement creates a window of incorrectly handled transactions. This is a normal, continuous condition in production. [6](#0-5) 

### Recommendation

Replace `strk_gas_prices.l2_gas_price` with `next_l2_gas_price` from the block header in `validate_resource_bounds`. The `GatewayFixedBlockStateReader::get_block_info` already returns the full `BlockInfo`; the fix is to read `block_header_without_hash.next_l2_gas_price` instead of `gas_prices.strk_gas_prices.l2_gas_price`. This is exactly what the existing TODO comment requests.

### Proof of Concept

1. Observe the latest committed block has `l2_gas_price = P` and `next_l2_gas_price = P' > P` (e.g., block was above gas target).
2. Submit an invoke transaction with `l2_gas.max_price_per_unit = P` (meets the gateway threshold `100% * P`).
3. Gateway calls `validate_tx_l2_gas_price_within_threshold` with `previous_block_l2_gas_price = P`; check passes (`P >= P`).
4. Transaction enters the mempool.
5. Batcher builds the next block using `next_l2_gas_price = P'` as the execution gas price.
6. Blockifier's `check_fee_bounds` finds `max_price_per_unit (P) < actual_gas_price (P')` and rejects the transaction with `MaxGasPriceTooLow`.
7. The gateway admitted a transaction that the sequencer cannot include. [7](#0-6) [8](#0-7)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
