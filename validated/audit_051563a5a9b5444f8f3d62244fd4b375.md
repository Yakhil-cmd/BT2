### Title
Missing Caller Validation on `update_rewards` Allows Any Address to Permanently Freeze Staker Yield - (File: `src/staking/staking.cairo`)

### Summary

`IStakingRewardsManager::update_rewards` is a public, permissionless entry point that the protocol specification explicitly restricts to "Only starkware sequencer." The implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` once per block, consuming the global `last_reward_block` slot and permanently preventing the legitimate sequencer from distributing rewards for that block.

### Finding Description

The specification at `docs/spec.md` line 1645 states:

```
#### access control
Only starkware sequencer.
```

The implementation in `src/staking/staking.cairo` at `StakingRewardsManagerImpl::update_rewards` (lines 1447–1488) performs no such check:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();          // only checks pause flag
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... no assert_caller_is_sequencer() or equivalent
``` [1](#0-0) 

The function then unconditionally writes the current block number to the **global** `last_reward_block` storage variable before checking `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

Because `last_reward_block` is a single global slot (not per-staker), one call per block with any valid `staker_address` and `disable_rewards: true` is sufficient to mark the block as "already processed" and cause every subsequent call in that block — including the legitimate sequencer call — to revert with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The interface confirms the function is embedded as a public ABI endpoint:

```cairo
#[starknet::interface]
pub trait IStakingRewardsManager<TContractState> {
    fn update_rewards(
        ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
    );
}
``` [4](#0-3) 

### Impact Explanation

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` once per block:

1. Passes all existing guards (`general_prerequisites`, `REWARDS_ALREADY_UPDATED`, `INVALID_STAKER`).
2. Writes `current_block_number` to `last_reward_block`.
3. Returns early — no rewards are distributed.
4. Every subsequent call in the same block (including the sequencer's) reverts with `REWARDS_ALREADY_UPDATED`.

Sustained across blocks, this permanently freezes all unclaimed yield for every staker and every delegation pool. This matches the allowed impact: **Permanent freezing of unclaimed yield / unclaimed royalties (High)**.

### Likelihood Explanation

- The function is a public, permissionless ABI endpoint on a deployed contract.
- Starknet transaction fees are low; calling once per block is economically trivial.
- No special role, key, or bridge access is required — any EOA or contract can execute this.
- The attacker needs only one valid `staker_address` (publicly discoverable from `NewStaker` events).

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`, consistent with the specification. The pattern already used for `update_rewards_from_attestation_contract` (which calls `self.assert_caller_is_attestation_contract()`) should be replicated:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
}
``` [5](#0-4) 

### Proof of Concept

1. Deploy or use the existing staking contract after consensus rewards are active.
2. Obtain any valid `staker_address` from emitted `NewStaker` events.
3. From an arbitrary EOA, call `update_rewards(staker_address, disable_rewards: true)` at the start of every block.
4. Observe that `last_reward_block` is updated to the current block and no rewards are distributed.
5. Observe that the sequencer's legitimate call to `update_rewards(staker_address, disable_rewards: false)` reverts with `REWARDS_ALREADY_UPDATED` for the remainder of that block.
6. Repeat indefinitely — staker `unclaimed_rewards_own` never increases; pool balances remain zero. [6](#0-5)
