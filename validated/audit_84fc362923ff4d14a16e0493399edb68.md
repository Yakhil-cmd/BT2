### Title
Missing Zero Address Validation in `change_reward_address` Allows Permanent Freezing of Unclaimed Yield - (`src/pool/pool.cairo`, `src/staking/staking.cairo`)

---

### Summary

Both `Pool.change_reward_address` and `Staking.change_reward_address` accept a zero address as the new `reward_address` without reverting. Once set, all subsequent `claim_rewards` calls will attempt to transfer accumulated yield to the zero address, permanently destroying or freezing unclaimed rewards. The same missing check exists in `enter_delegation_pool`, allowing a pool member to register with a zero reward address from the start.

---

### Finding Description

`Pool.change_reward_address` (pool.cairo line 505) performs only one validation — that the new address is not the pool's token contract address — and then unconditionally writes the caller-supplied value:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    assert!(
        self.token_dispatcher.contract_address.read() != reward_address,
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    let pool_member = get_caller_address();
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let old_address = pool_member_info.reward_address;
    pool_member_info.reward_address = reward_address;   // zero accepted here
    self.write_pool_member_info(:pool_member, :pool_member_info);
    ...
}
```

There is no `assert!(reward_address.is_non_zero(), ...)` guard. [1](#0-0) 

The identical omission exists in `Staking.change_reward_address`: the only guard is `does_token_exist(reward_address)`, which returns `false` for the zero address, so the zero address passes silently. [2](#0-1) 

`enter_delegation_pool` in pool.cairo also passes `reward_address` directly to `VInternalPoolMemberInfoTrait::new_latest` with no zero-address check, so a member can register with a zero reward address from inception. [3](#0-2) 

Once `reward_address` is zero, `Pool.claim_rewards` computes the owed yield and then calls:

```cairo
let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
``` [4](#0-3) 

The transfer destination is the zero address. Depending on the STRK ERC-20 implementation:
- If the token **reverts** on transfer to zero address, `claim_rewards` permanently reverts for this pool member — unclaimed yield is frozen forever.
- If the token **allows** transfer to zero address, the rewards are burned — unclaimed yield is permanently destroyed.

The same path exists for stakers via `Staking.claim_rewards` → `send_rewards_to_staker`. [5](#0-4) 

Additionally, `Staking.unstake_action` calls `send_rewards_to_staker` before returning the principal. If that call reverts (zero-address transfer), the staker is permanently unable to exit, freezing their staked principal as well. [6](#0-5) 

---

### Impact Explanation

**Permanent freezing of unclaimed yield** (and potentially staked principal). A pool member or staker who sets `reward_address` to zero — deliberately or by mistake — will have all future reward claims either revert permanently or silently burn the tokens. If the revert path is triggered inside `unstake_action`, the staked principal is also frozen.

---

### Likelihood Explanation

Any pool member or staker can call `change_reward_address` at any time with no privileged role required. A user could do this accidentally (e.g., passing an uninitialized variable) or deliberately as a griefing action against themselves. The entry point is fully unprivileged and reachable in normal protocol operation.

---

### Recommendation

Add a zero-address guard at the top of both `change_reward_address` implementations and inside `enter_delegation_pool`:

```cairo
// In Pool.change_reward_address and Staking.change_reward_address:
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

The error constant `GenericError::ZERO_ADDRESS` ("Zero address") already exists in the codebase. [7](#0-6) 

---

### Proof of Concept

1. Pool member Alice calls `pool.change_reward_address(reward_address: 0)`.
   - The only check (`token_address != reward_address`) passes because `0 != token_address`.
   - `pool_member_info.reward_address` is written as `0`.
2. After K epochs, Alice calls `pool.claim_rewards(pool_member: alice)`.
   - Rewards are computed correctly.
   - `reward_token.checked_transfer(recipient: 0, amount: rewards)` is executed.
   - If the STRK token reverts on zero-address transfer: `claim_rewards` reverts permanently; Alice's accumulated yield is frozen.
   - If the STRK token allows zero-address transfer: rewards are burned; Alice's yield is permanently destroyed.
3. For the staker variant: staker Bob calls `staking.change_reward_address(0)`, then calls `staking.unstake_action(bob)`. `send_rewards_to_staker` attempts `checked_transfer(recipient: 0, ...)`, which reverts, permanently blocking Bob from recovering his staked principal.

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }
