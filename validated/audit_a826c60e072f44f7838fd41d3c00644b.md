### Title
Anyone Can Force a Pool Member's Exit via `exit_delegation_pool_action`, Blocking Pool Switching or Intent Cancellation - (File: src/pool/pool.cairo)

### Summary
`exit_delegation_pool_action` in the delegation pool contract accepts a `pool_member` address as a parameter and performs no caller check. Any unprivileged address can call it for any pool member once the exit window has elapsed, forcibly completing the exit and transferring funds to `pool_member`. This prevents the pool member from exercising their remaining options — switching to another pool via `switch_delegation_pool` or canceling the intent via `exit_delegation_pool_intent(0)` — and causes them to lose staking rewards for the K-epoch re-entry delay.

### Finding Description
After a pool member calls `exit_delegation_pool_intent(amount)`, they enter an exit window. During and after this window they retain two additional options:

1. **Switch pools**: call `switch_delegation_pool(to_staker, to_pool, amount)` to redirect the `unpool_amount` to a different validator's pool without withdrawing.
2. **Cancel intent**: call `exit_delegation_pool_intent(0)` to zero out `unpool_amount` and clear `unpool_time`, returning to normal delegation.

`exit_delegation_pool_action` has no access-control guard:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    // Asserts.
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    let unpool_time = pool_member_info
        .unpool_time
        .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
    assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);
    // ... no get_caller_address() check ...
    token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
``` [1](#0-0) 

Once `unpool_time` has passed, any external address can call this function for any `pool_member`, irrevocably completing the exit. The pool member's `unpool_amount` and `unpool_time` are cleared, destroying both the switch-pool and cancel-intent paths. [2](#0-1) 

The spec explicitly documents this as intentional ("Any address can execute"), but the design fails to account for the harm it causes to pool members who have not yet acted on their remaining options. [3](#0-2) 

### Impact Explanation
**Medium — Griefing with no profit motive but damage to users.**

The pool member receives their `unpool_amount` tokens back (no principal loss), but:

- They are **blocked from switching pools**: `switch_delegation_pool` requires `pool_member_info.unpool_time.is_some()`, which is cleared by the forced action. [4](#0-3) 

- They are **blocked from canceling**: `exit_delegation_pool_intent(0)` can no longer be called because `unpool_time` is already `None`. [5](#0-4) 

- To resume earning rewards they must call `enter_delegation_pool` again and wait K epochs before their stake takes effect, losing staking yield for that entire delay period. [6](#0-5) 

### Likelihood Explanation
High. The call requires no tokens, no permissions, and no special setup — only knowledge of a pool member address that has a pending exit intent with an elapsed `unpool_time`. Pool member exit intents are publicly observable on-chain via emitted `PoolMemberExitIntent` events. A griefing bot can monitor these events and call `exit_delegation_pool_action` the moment `unpool_time` passes, before the pool member can switch or cancel. [7](#0-6) 

### Recommendation
Restrict `exit_delegation_pool_action` so that only the pool member (or optionally their registered `reward_address`) can trigger it, mirroring the access control already applied to `exit_delegation_pool_intent` and `switch_delegation_pool`:

```cairo
fn exit_delegation_pool_action(
    ref self: ContractState, pool_member: ContractAddress,
) -> Amount {
    let caller = get_caller_address();
    let pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(
        caller == pool_member || caller == pool_member_info.reward_address,
        "{}",
        Error::UNAUTHORIZED_CALLER,
    );
    // ... rest of function
``` [1](#0-0) 

### Proof of Concept
```
1. pool_member calls exit_delegation_pool_intent(full_amount)
   → unpool_time = now + exit_wait_window, unpool_amount = full_amount

2. pool_member decides to switch to a better validator and prepares
   to call switch_delegation_pool(to_staker, to_pool, full_amount)

3. Attacker observes the PoolMemberExitIntent event on-chain.
   After unpool_time elapses, attacker calls:
     pool_contract.exit_delegation_pool_action(pool_member)

4. Result:
   - unpool_amount transferred to pool_member (principal returned)
   - pool_member_info.unpool_time = None
   - pool_member_info.unpool_amount = 0

5. pool_member's call to switch_delegation_pool now reverts with
   MISSING_UNDELEGATE_INTENT (requires unpool_time.is_some())

6. pool_member must re-enter via enter_delegation_pool and wait K
   epochs before stake is active, losing rewards for that period.
``` [8](#0-7) [9](#0-8)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
