### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Block Rewards - (File: `src/staking/staking.cairo`)

### Summary
`update_rewards` in `StakingRewardsManagerImpl` has no caller restriction despite the specification explicitly requiring "Only starkware sequencer." Because `last_reward_block` is a single global slot updated unconditionally before the `disable_rewards` guard, any unprivileged address can front-run the sequencer's call with `disable_rewards: true`, consuming the per-block slot and permanently discarding all stakers' block rewards for that block.

### Finding Description
The spec at `docs/spec.md:1645` states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo:1449-1507` contains no such check:

```cairo
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();          // only checks is_paused
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... staker validity checks ...
        self.last_reward_block.write(current_block_number);   // ← global slot consumed here

        if disable_rewards || self.is_pre_consensus() {
            return;                                            // ← early exit, no rewards
        }
        // ... distribute block rewards ...
    }
}
```

`last_reward_block` is a single global `BlockNumber` field: [1](#0-0) 

It is written unconditionally before the `disable_rewards` branch: [2](#0-1) 

The function signature accepts any caller — `general_prerequisites` only asserts the contract is not paused: [3](#0-2) 

The spec access-control requirement is documented but not enforced: [4](#0-3) 

### Impact Explanation
Because `last_reward_block` is global (one slot for the entire contract, not per-staker), a single successful call to `update_rewards` in block N — regardless of which `staker_address` is passed — prevents every other call in block N from succeeding (`REWARDS_ALREADY_UPDATED`). The sequencer is expected to call this exactly once per block for the block proposer. An attacker who calls first with `disable_rewards: true` causes the block's rewards to be silently skipped and permanently unrecoverable; there is no catch-up mechanism for missed blocks.

**Impact category matched**: *Permanent freezing of unclaimed yield or unclaimed royalties* (High).

### Likelihood Explanation
- The function is publicly callable with no role requirement.
- The attacker needs only to submit a transaction before the sequencer in the same block — a standard front-run on any L2 with a public mempool or on L1 if the call originates there.
- The cost is a single cheap transaction per block; the attacker has no financial risk.
- The attack can be repeated every block to permanently suppress all consensus-phase block rewards across all stakers.

### Recommendation
Add an access-control guard matching the specification. For example, introduce a `SEQUENCER_ROLE` or reuse an existing privileged role and assert it at the top of `update_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer();   // enforce "Only starkware sequencer"
    self.general_prerequisites();
    // ...
}
```

Alternatively, derive the rewarded staker from the block proposer identity rather than accepting it as a caller-controlled parameter, eliminating the front-run surface entirely.

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. In block N, before the sequencer submits `update_rewards(proposer, false)`, attacker submits:
   ```
   update_rewards(any_valid_active_staker, disable_rewards: true)
   ```
3. Attacker's tx executes first:
   - `last_reward_block` is written to N.
   - Function returns early; no rewards distributed.
4. Sequencer's tx executes:
   - `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
5. Block N's rewards are permanently lost for all stakers. The attacker repeats this every block. [5](#0-4) [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
