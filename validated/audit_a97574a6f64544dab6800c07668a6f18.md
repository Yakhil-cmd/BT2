### Title
Unprivileged Caller Can Permanently Freeze Consensus Block Rewards via `update_rewards` with `disable_rewards=true` — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` has no caller restriction. Any address can call it with `disable_rewards=true`, which advances the global `last_reward_block` checkpoint without distributing any rewards. Because the checkpoint is global and the function reverts if called twice in the same block, an attacker who front-runs every block permanently prevents all stakers from receiving consensus block rewards.

---

### Finding Description

`update_rewards` is part of the public `IStakingRewardsManager` interface and carries no `assert_caller_is_*` guard: [1](#0-0) 

The function begins with `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero — no role or address restriction: [2](#0-1) 

After validating the staker, it unconditionally writes the current block number to the global `last_reward_block`: [3](#0-2) 

It then checks `disable_rewards` and returns early — skipping all reward computation and distribution — if the flag is `true`: [4](#0-3) 

The guard that prevents double-processing is a strict `>` comparison against this same global field: [5](#0-4) 

Because `last_reward_block` is a single global slot (not per-staker), one call with `disable_rewards=true` for *any* staker exhausts the entire block's reward slot for *every* staker.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield (consensus block rewards).**

An attacker who calls `update_rewards(<any_valid_staker>, disable_rewards=true)` at the start of every block:

1. Advances `last_reward_block` to the current block number.
2. Returns without distributing STRK or BTC block rewards to any staker or pool.
3. Blocks every subsequent legitimate call in that block with `REWARDS_ALREADY_UPDATED`.

Repeated every block, this permanently denies all stakers their consensus-era block rewards. The stakers' `unclaimed_rewards_own` fields are never incremented; pool `cumulative_rewards_trace` is never updated. Rewards are not merely delayed — they are permanently lost for each skipped block.

---

### Likelihood Explanation

The attack requires only:
- A funded Starknet account (gas costs only).
- A script that submits one transaction per block.

No privileged access, no leaked keys, no external dependencies. The attacker has no profit motive but causes direct, measurable harm to every staker and delegator in the protocol. The attack is fully automatable and cheap relative to the value locked.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the block proposer address registered in the contract, or a dedicated `REWARDS_MANAGER` role). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "skip rewards" logic internally based on on-chain conditions that cannot be spoofed by an unprivileged caller.

---

### Proof of Concept

```
// Attacker script (pseudocode, runs once per block)
loop every new block N:
    staking_contract.update_rewards(
        staker_address = <any active staker>,
        disable_rewards = true
    )
    // Effects:
    //   last_reward_block := N
    //   no rewards distributed
    //   all subsequent update_rewards calls in block N revert with REWARDS_ALREADY_UPDATED
```

Concrete call path:
1. Attacker calls `Staking::update_rewards(staker_address, disable_rewards=true)`.
2. `general_prerequisites()` passes (contract unpaused, caller non-zero). [6](#0-5) 
3. `current_block_number > last_reward_block` passes (first call this block). [5](#0-4) 
4. Staker existence and activity checks pass (any valid active staker). [7](#0-6) 
5. `last_reward_block` is written to `N`. [8](#0-7) 
6. `disable_rewards == true` → early return, zero rewards distributed. [4](#0-3) 
7. Any legitimate call to `update_rewards` in block `N` now reverts. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
