Looking at the `update_rewards` function in `src/staking/staking.cairo` and its spec in `docs/spec.md`, I found a direct analog.

The spec explicitly states `update_rewards` access control is **"Only starkware sequencer"**, but the implementation has **no caller restriction** — only `general_prerequisites()` (pause check) and a block-number guard. Any unprivileged address can call it with `disable_rewards: true`, front-running the sequencer and permanently suppressing reward distribution.

### Title
Unprivileged caller can front-run `update_rewards` with `disable_rewards: true`, permanently suppressing staker block rewards - (`src/staking/staking.cairo`)

### Summary
`update_rewards` is documented as callable only by the Starkware sequencer, but the implementation contains no caller restriction. Any address can call it with `disable_rewards: true` before the sequencer, consuming the single global `last_reward_block` slot for that block and causing the sequencer's legitimate call to revert with `REWARDS_ALREADY_UPDATED`. Repeated across every block, this permanently freezes all staker and delegator yield.

### Finding Description

The spec at `docs/spec.md:1644-1645` states:
> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo:1448-1507` enforces no such restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: is_paused
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);  // global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;   // exits without distributing any rewards
    }
    // ... reward distribution only reached if disable_rewards == false
``` [1](#0-0) 

The critical state variable `last_reward_block` is a **single global** (not per-staker):

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The spec confirms the intended restriction: [3](#0-2) 

**Exploit path:**

1. Attacker monitors the network for the sequencer's pending `update_rewards` call.
2. Attacker submits `update_rewards(any_valid_staker_address, disable_rewards: true)` first (or simply calls it at the start of every block).
3. `last_reward_block` is written to the current block number.
4. The sequencer's call reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number > last_reward_block` is now false.
5. No rewards are distributed for that block for **any** staker (the guard is global).
6. Repeated every block → permanent yield freeze.

The attacker needs only a valid `staker_address` (any active staker, publicly readable from the `stakers` Vec) and enough gas to call the function once per block.

### Impact Explanation

Every block where the attacker front-runs, all stakers and delegators lose their block rewards permanently — the missed block cannot be retroactively compensated. Sustained across many blocks this constitutes **permanent freezing of unclaimed yield** for all protocol participants. This matches the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"*.

### Likelihood Explanation

The function is publicly callable with no access control. The attacker needs no privileged keys, no tokens staked, and no special setup — only the address of any active staker (publicly available on-chain) and gas. The cost to the attacker is one cheap transaction per block; the cost to victims is all block rewards for that block. The asymmetry makes sustained griefing economically rational.

### Recommendation

Add a caller restriction matching the spec. The simplest fix is to add a sequencer-only role check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract()`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_operator();  // or a dedicated sequencer role
    self.general_prerequisites();
    ...
``` [4](#0-3)
