### Title
Gateway Stateful Validator Admits Transactions Using Previous Block's Gas Prices While Batcher Executes Against Next Block's Gas Prices — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful validator validates resource bounds against the **previous block's** gas prices. The batcher executes those same transactions against the **next block's** dynamically-calculated gas prices. When gas prices rise between blocks, transactions that passed gateway admission will fail `check_fee_bounds` inside the batcher, wasting sequencer resources and allowing an attacker to flood the mempool with transactions that are guaranteed to fail at execution time.

### Finding Description

**Step 1 — Gateway reads previous block gas prices for the L2 threshold check.**

`validate_resource_bounds` fetches `previous_block_l2_gas_price` from `gateway_fixed_block_state_reader.get_block_info()` and passes it to `validate_tx_l2_gas_price_within_threshold`. The function only validates the L2 gas price; L1 and L1-data gas prices are explicitly skipped with the comment `// TODO(Arni): Consider running this validation for all gas prices.` [1](#0-0) [2](#0-1) 

**Step 2 — `run_validate_entry_point` also uses the previous block's gas prices.**

The blockifier validation inside the gateway constructs a `BlockContext` from `gateway_fixed_block_state_reader.get_block_info()`, increments only the block number, and keeps the previous block's gas prices unchanged. The TODO comment explicitly acknowledges the intent to use the *next* block's L2 gas price instead:

```
// TODO(Arni): getnext_l2_gas_price from the block header.
``` [3](#0-2) 

**Step 3 — Blockifier `check_fee_bounds` enforces the *current* block's gas prices at execution time.**

When the batcher executes the transaction, `perform_pre_validation_stage` calls `check_fee_bounds`, which compares `resource_bounds.max_price_per_unit` against `block_info.gas_prices.gas_price_vector(fee_type)` — the **current** block's prices, not the previous block's prices used during gateway admission. [4](#0-3) [5](#0-4) 

**The gap:** The gateway admits a transaction if `tx.l2_gas_price >= prev_block_price × threshold%`. The batcher rejects it if `tx.l2_gas_price < next_block_price`. Because the next block's L2 gas price is computed dynamically (EIP-1559-style) and can rise significantly when the previous block was congested, the two checks can diverge. [6](#0-5) 

### Impact Explanation

This matches the allowed High impact: **"Mempool/gateway/RPC admission accepts invalid transactions before sequencing."**

Transactions that pass all gateway checks — stateless, stateful pre-validation, and blockifier `__validate__` — are placed in the mempool. When the batcher later attempts to execute them, `check_fee_bounds` fails because the next block's gas price exceeds the transaction's `max_price_per_unit`. The sequencer expends CPU and state-read resources on transactions that cannot succeed, and the mempool fills with permanently-invalid entries that consume queue capacity.

### Likelihood Explanation

The L2 gas price is adjusted every block based on actual gas consumption. A single heavily-loaded block causes the next block's price to rise. An attacker who observes a congested block can immediately submit a batch of transactions priced just above the gateway's `min_gas_price_percentage × prev_price` threshold but below the predictable next-block price. All of these pass gateway admission and fail at batcher execution. No privileged access is required; any unprivileged user can submit transactions via the public RPC endpoint.

### Recommendation

1. **Use the next block's L2 gas price** (already stored in the block header as `next_l2_gas_price`) for the threshold check in `validate_resource_bounds`, as the TODO comment already anticipates.
2. **Extend the threshold check to L1 and L1-data gas prices** as the second TODO comment suggests, so that the gateway's early-rejection check is consistent with the blockifier's `check_fee_bounds`.
3. Consider adding a small upward buffer (e.g., the maximum single-block price increase) to the threshold to absorb one block of price movement between gateway admission and batcher execution.

### Proof of Concept

1. Observe that the previous block was congested; its L2 gas price is `P_prev = 1_000_000`.
2. The gateway's `min_gas_price_percentage` is 50, so the admission threshold is `500_000`.
3. Submit 10,000 invoke transactions with `l2_gas.max_price_per_unit = 600_000` (above threshold, passes `validate_resource_bounds`) and `l1_gas.max_price_per_unit = 0` (not checked by `validate_resource_bounds`).
4. Each transaction also passes `run_validate_entry_point` because the blockifier validation uses the previous block's L1 gas price (which may be low or zero in a bootstrap scenario, or the attacker sets L1 gas price just above the previous block's L1 price).
5. All 10,000 transactions are admitted to the mempool.
6. The next block's L2 gas price is calculated as `P_next = 1_003_003` (a ~0.3% increase from congestion).
7. The batcher calls `check_fee_bounds` with `P_next`; `600_000 < 1_003_003` → `MaxGasPriceTooLow` for every transaction.
8. All 10,000 transactions fail at pre-validation in the batcher; the sequencer has wasted state reads, blockifier CPU, and mempool queue slots on permanently-invalid transactions. [7](#0-6) [2](#0-1) [8](#0-7)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
