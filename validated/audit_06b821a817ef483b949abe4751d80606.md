### Title
Overly Broad `last_reward_block` Update in `update_rewards` Allows Griefing to Permanently Freeze Consensus Rewards - (File: src/staking/staking.cairo)

### Summary
`update_rewards` is a public, permissionless function that updates a global `last_reward_block` sentinel **before** checking the `disable_rewards` flag. Because any caller can pass `disable_rewards: true`, an attacker can consume the per-block reward slot every block without distributing any rewards, permanently preventing all stakers from receiving consensus-era yield.

### Finding Description
`update_rewards` in `StakingRewardsManagerImpl` enforces a one-call-per-block invariant via a global `last_reward_block` storage variable:

```
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [1](#0-0) 

Immediately after the assertion passes, `last_reward_block` is written unconditionally:

```
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

The function has no access control beyond `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero: [3](#0-2) 

This means any unprivileged address can call `update_rewards(valid_staker, true)` in every block. The call:
1. Passes the `last_reward_block` guard (first call in the block).
2. Writes `last_reward_block = current_block`.
3. Returns early due to `disable_rewards == true` — **no rewards are distributed**.
4. Blocks every subsequent legitimate `update_rewards` call in the same block with `REWARDS_ALREADY_UPDATED`.

This is structurally identical to the soulbound token bug: the `last_reward_block` write is analogous to the `require(!isSoulbound)` check — it is applied unconditionally to **all** invocations (including the "no-op" `disable_rewards: true` path) when it should only apply to the path that actually distributes rewards.

### Impact Explanation
Every staker in the consensus-rewards phase is permanently denied their per-block STRK yield. The attacker does not need to hold any stake; they only need to identify one currently active staker (trivially obtained from the public `get_stakers` view) and submit one cheap transaction per block. This constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

### Likelihood Explanation
The entry path is fully permissionless. No privileged role, no token balance, and no prior relationship with the protocol is required. The only cost is Starknet gas per block. Because the attack is cheap, continuous, and requires no coordination, likelihood is **High**.

### Recommendation
Move the `last_reward_block` write to after the early-return guard, so the block slot is only consumed when rewards are actually distributed:

```cairo
fn update_rewards(...) {
    ...
    assert!(current_block_number > self.last_reward_block.read(), ...);
    ...
    if disable_rewards || self.is_pre_consensus() {
        return;   // do NOT write last_reward_block here
    }

    // Only consume the block slot when rewards are actually distributed.
    self.last_reward_block.write(current_block_number);
    ...
}
```

Alternatively, restrict `disable_rewards: true` calls to a privileged role (e.g., `only_app_governor`).

### Proof of Concept

1. Staking contract is live in consensus-rewards mode (`is_pre_consensus()` returns `false`).
2. Attacker identifies any active staker `S` with non-zero balance (via `get_stakers`).
3. In every new block `N`, attacker calls `update_rewards(S, true)`:
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → function returns early, zero rewards distributed.
4. Any legitimate sequencer/validator call to `update_rewards(any_staker, false)` in block `N` now fails with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. Repeated every block: no staker ever accumulates consensus rewards. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
