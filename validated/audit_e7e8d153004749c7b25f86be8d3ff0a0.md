### Title
Missing Consensus Vote Signature Verification Allows Quorum Forgery from Any Network Peer - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary
The `handle_vote` function in `SingleHeightConsensus` accepts prevotes and precommits without verifying their cryptographic signatures. An explicit `// TODO(Asmaa): verify the signature` comment confirms this is an unimplemented guard. Any peer reachable on the P2P consensus network can forge votes attributed to any committee member and drive the state machine to a quorum decision without legitimate validator participation.

### Finding Description

**Root cause — no signature check in `handle_vote`:**

`handle_vote` performs only two admission checks before forwarding a vote to the state machine:
1. `vote.height == current_height`
2. `vote.voter` is a member of the committee

The `signature` field on the `Vote` struct is never inspected. [1](#0-0) 

The comment at line 242 reads `// TODO(Asmaa): verify the signature`, confirming the check is intentionally deferred but not yet implemented.

**Votes are broadcast with empty signatures:**

In `make_self_vote` inside `state_machine.rs`, the sequencer itself emits votes with `signature: RawSignature::default()`: [2](#0-1) 

The comment `// TODO(Asmaa): sign the vote` confirms signing is also unimplemented. The signature field therefore carries no cryptographic commitment in either direction.

**Verification functions exist but are never called in the consensus path:**

`verify_precommit_vote_signature` and `verify_identity` are defined in `apollo_signature_manager` and return `Result<bool, SignatureVerificationError>`: [3](#0-2) 

A `grep` across all `.rs` files shows these functions are called only inside `signature_manager_test.rs` — never from any consensus code path.

**The `bool`-returning pattern (direct analog to `ecrecover()` returning 0):**

`verify_message_hash_signature` and `verify_block_signature` both return `Result<bool, …>` rather than `Result<(), …>`, meaning a caller that unwraps the `Ok` without asserting the inner `bool` silently accepts an invalid signature: [4](#0-3) [5](#0-4) 

This is the structural analog to the Solidity `ecrecover()` returning `address(0)` — the error is encoded as a value, not a revert, and callers must remember to check it.

### Impact Explanation

An attacker with access to the P2P consensus broadcast channel can:

1. Observe the committee for height H (public information from the committee provider).
2. Wait for a proposal with commitment C to be broadcast.
3. Craft `Vote { vote_type: Precommit, height: H, round: R, proposal_commitment: Some(C), voter: V_i, signature: RawSignature::default() }` for enough committee members V_i to exceed the 2/3 quorum threshold.
4. Broadcast these forged votes.

`handle_vote` will accept each one (height matches, voter is in committee), insert them into the state machine, and `upon_decision` will fire — committing a block without the real validators ever signing it.

This satisfies **"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload"**: every accepted vote is bound to a signer who never produced it.

### Likelihood Explanation

The P2P vote broadcast channel is open to any connected peer. No privileged position is required — only network reachability and knowledge of the current committee (derivable from public chain state). The attack is fully unprivileged.

### Recommendation

1. **Implement signing in `make_self_vote`**: call `SignatureManager::sign_precommit_vote` (or an equivalent prevote variant) before broadcasting.
2. **Implement verification in `handle_vote`**: after confirming `vote.voter` is in the committee, retrieve that member's public key and call `verify_precommit_vote_signature` (or a prevote equivalent). Reject and report the peer on failure.
3. **Change `verify_message_hash_signature` / `verify_block_signature` return types** from `Result<bool, E>` to `Result<(), E>` so that a successful return unambiguously means the signature is valid, eliminating the `ecrecover()`-style silent-false footgun.

    /// Handle vote messages from peer nodes.
    #[instrument(skip_all)]
    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }

pub fn verify_message_hash_signature(
    message_hash: &Felt,
    signature: &Signature,
    public_key: &PublicKey,
) -> Result<bool, CryptoError> {
    starknet_crypto::verify(&public_key.0, message_hash, &signature.r, &signature.s).map_err(
        |err| match err {
            starknet_crypto::VerifyError::InvalidPublicKey => {
                CryptoError::InvalidPublicKey(*public_key)
            }
            starknet_crypto::VerifyError::InvalidMessageHash => {
                CryptoError::InvalidMessageHash(*message_hash)
            }
            starknet_crypto::VerifyError::InvalidR => CryptoError::InvalidR(signature.r),
            starknet_crypto::VerifyError::InvalidS => CryptoError::InvalidS(signature.s),
        },
    )
}

pub fn verify_block_signature(

    signature: &BlockSignature,
    state_diff_commitment: &GlobalRoot,
    block_hash: &BlockHash,

    let message_hash = Poseidon::hash_array(&[block_hash.0, state_diff_commitment.0]);

            block_hash: *block_hash,
            error: err,
        },
    )
}
