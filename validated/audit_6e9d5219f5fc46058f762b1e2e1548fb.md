### Title
Gateway Admission Accepts Transactions with Zero L1/L1DataGas `max_price_per_unit` Due to Incomplete Resource Bounds Price Validation — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateless and stateful validators only enforce a minimum price floor on the **L2 gas** resource bound. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields are never checked against the current block's actual gas prices at the admission layer. A transaction carrying zero prices for L1 gas and L1 data gas passes all gateway checks and enters the mempool, but will be rejected by the blockifier's `check_fee_bounds` at execution time (when `charge_fee = true`). This breaks the admission invariant: the gateway should only admit transactions that can be executed.

### Finding Description

**Stateless validator — only L2 gas price is checked:** [1](#0-0) 

Lines 66–68 reject transactions whose total `max_possible_fee` is zero. Lines 71–76 then enforce a `min_gas_price` floor, but **only on `l2_gas.max_price_per_unit`**. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared against any floor.

**Stateful validator — same gap, explicitly acknowledged with a TODO:** [2](#0-1) 

The function is named `validate_tx_l2_gas_price_within_threshold` and the comment at line 358 reads:

```
// TODO(Arni): Consider running this validation for all gas prices.
```

Only `tx_resource_bounds.l2_gas.max_price_per_unit` is compared against `previous_block_l2_gas_price`. The `L1Gas(_)` arm explicitly does nothing.

**Blockifier execution — all three prices are checked:** [3](#0-2) 

When `charge_fee = true` (the production default during block execution), `check_fee_bounds` iterates over all three resources and rejects any transaction where `resource_bounds.max_price_per_unit < actual_gas_price.get()`. A transaction with `l1_gas.max_price_per_unit = 0` will always fail this check because the actual L1 gas price is always non-zero.

**Concrete admission path for an invalid transaction:**

1. Attacker submits an `InvokeV3` transaction with:
   - `l1_gas = { max_amount: 0, max_price_per_unit: 0 }`
   - `l2_gas = { max_amount: N, max_price_per_unit: min_gas_price }` (passes both gateway checks)
   - `l1_data_gas = { max_amount: 0, max_price_per_unit: 0 }`
2. `max_possible_fee(Tip::ZERO) = N * min_gas_price > 0` → passes `ZeroResourceBounds` check.
3. `l2_gas.max_price_per_unit >= min_gas_price` → passes stateless price check.
4. `l2_gas.max_price_per_unit >= threshold` → passes stateful price check.
5. Transaction enters the mempool.
6. Batcher picks it up; blockifier calls `check_fee_bounds` with `charge_fee = true`.
7. `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (non-zero)` → `InsufficientResourceBounds` error; transaction is rejected at execution.

The `ZeroResourceBounds` guard does not close this gap because it only requires the **product** `max_amount * max_price_per_unit` to be non-zero across all resources combined — a non-zero L2 gas contribution is sufficient to pass it even when L1 gas price is zero. [4](#0-3) 

### Impact Explanation

Every such transaction admitted to the mempool consumes a mempool slot, is propagated to peer nodes (P2P broadcast), and is later fetched by the batcher, only to be discarded at execution. An attacker with a valid account (nonce ≥ 0) can continuously submit such transactions, flooding the mempool and degrading throughput for legitimate users. This matches the **High** impact criterion: *"Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."*

### Likelihood Explanation

The attack requires no special privilege. Any account that can submit a V3 transaction can set `l1_gas.max_price_per_unit = 0`. The gateway's own TODO comment confirms the gap is known but unmitigated. The stateless validator config `min_gas_price` applies only to L2 gas, so no configuration change closes the hole.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel function) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK prices, mirroring the check already performed by `check_fee_bounds` in the blockifier. The TODO comment at line 358 already identifies this as the intended fix. [5](#0-4) [6](#0-5)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub struct StatefulTransactionValidatorConfig {
    // If true, ensures the max L2 gas price exceeds (a configurable percentage of) the base gas
    // price of the previous block.
    pub validate_resource_bounds: bool,
    pub max_allowed_nonce_gap: u32,
    pub reject_future_declare_txs: bool,
    pub max_nonce_for_validation_skip: Nonce,
    pub versioned_constants_overrides: Option<VersionedConstantsOverrides>,
    // Minimum gas price as percentage of threshold to accept transactions.
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}

impl Default for StatefulTransactionValidatorConfig {
    fn default() -> Self {
        StatefulTransactionValidatorConfig {
            validate_resource_bounds: true,
            max_allowed_nonce_gap: 200,
            reject_future_declare_txs: true,
            max_nonce_for_validation_skip: Nonce(Felt::ONE),
            min_gas_price_percentage: 100,
            versioned_constants_overrides: None,
        }
    }
}
