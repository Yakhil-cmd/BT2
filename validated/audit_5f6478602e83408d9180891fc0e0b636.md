### Title
Unprivileged Caller Can Suppress Consensus Block Rewards by Calling `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. Because `last_reward_block` is written unconditionally before the `disable_rewards` guard, any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` once per block to consume the single allowed update slot without distributing rewards, permanently denying all stakers their consensus block rewards for that block.

### Finding Description
`update_rewards` in `src/staking/staking.cairo` is exposed via `IStakingRewardsManager` with no caller restriction. The function enforces a one-call-per-block invariant using the global `last_reward_block` storage slot:

```cairo
// line 1453-1458
let current_block_number = starknet::get_block_number();
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After validating the staker, the function writes `last_reward_block` **unconditionally** at line 1485, before checking `disable_rewards`:

```cairo
// line 1485
self.last_reward_block.write(current_block_number);

// line 1487
if disable_rewards || self.is_pre_consensus() {
    return;   // ← returns without distributing any rewards
}
```

Because `last_reward_block` is a single global slot (not per-staker), one call with `disable_rewards: true` exhausts the entire block's reward slot. Any subsequent call in the same block — including the legitimate one with `disable_rewards: false` — reverts with `REWARDS_ALREADY_UPDATED`.

There is no access control on `update_rewards`. The `general_prerequisites()` call only checks the pause flag. The `disable_rewards` parameter is fully attacker-controlled.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

For every block in which the attacker front-runs with `update_rewards(valid_staker, disable_rewards: true)`, all stakers permanently lose their consensus block rewards for that block. The rewards are not deferred; they are simply never computed or credited. Repeated across every block, this completely halts consensus reward distribution for the entire protocol.

### Likelihood Explanation
**High.** The attack requires no stake, no privileged role, and no special setup. Any address can call the function. The only cost is gas per block. On Starknet, gas costs are low. A competing staker or any griefing actor has clear economic or adversarial motivation. The attack is deterministic and requires no mempool front-running on Starknet (sequencer ordering can be exploited or the attacker simply calls first in each block).

### Recommendation
Remove the `disable_rewards` parameter from the public interface, or restrict who may pass `disable_rewards: true` (e.g., only the staking contract itself or a privileged role). Alternatively, move the `last_reward_block.write` to after the `disable_rewards` guard so that a suppressed call does not consume the block's update slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only write last_reward_block when rewards are actually distributed
self.last_reward_block.write(current_block_number);
// ... rest of reward distribution
```

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. At block N, attacker calls:
   ```
   staking.update_rewards(any_valid_active_staker, disable_rewards: true)
   ```
3. Inside the function:
   - Line 1455: `current_block_number (N) > last_reward_block` → passes.
   - Line 1485: `last_reward_block` is written to N.
   - Line 1487: `disable_rewards == true` → early return, zero rewards distributed.
4. Any legitimate call to `update_rewards(staker, disable_rewards: false)` in block N now hits:
   ```
   assert!(N > N, ...) // REWARDS_ALREADY_UPDATED — reverts
   ```
5. All stakers lose their block rewards for block N.
6. Attacker repeats at block N+1, N+2, … indefinitely. [1](#0-0) [2](#0-1)
