### Title
EIP-7623 Floor Gas Not Enforced as Minimum `gas_used` in Refund — Sender Recovers Floor-Gas Overhead That Should Be Burned (`engine/src/engine.rs`)

---

### Summary

`floor_gas` (EIP-7623) is computed and used only to gate the `gas_limit` validation check. It is never applied as a minimum on `gas_used` before `refund_unused_gas` is called. As a result, when `floor_gas > intrinsic_gas`, the sender is refunded the entire `(gas_limit − gas_used) × P` delta — including the floor-gas overhead that EIP-7623 mandates must be consumed (burned). The sender pays less than `floor_gas × P` net, defeating the anti-spam floor entirely.

---

### Finding Description

**Step 1 — floor_gas is computed and used only for the gas_limit gate.**

`floor_gas` is calculated at: [1](#0-0) 

The only effect of `floor_gas` is to reject transactions whose `gas_limit < max(intrinsic_gas, floor_gas)`. Once that check passes, `floor_gas` is never referenced again.

**Step 2 — charge_gas deducts `gas_limit × effective_gas_price`.** [2](#0-1) 

The full `gas_limit` (which equals `floor_gas` in the minimal case) is deducted from the sender.

**Step 3 — EVM executes with `gas_limit` as its budget; raw `gas_used` is returned.** [3](#0-2) 

`gas_used` is the raw EVM-reported value. There is no `gas_used = max(gas_used, floor_gas)` adjustment anywhere in the call path.

**Step 4 — `refund_unused_gas` uses raw `gas_used`, refunding the floor-gas overhead to the sender.** [4](#0-3) 

`spent_amount = gas_used × effective_gas_price`  
`refund = prepaid_amount − spent_amount = (gas_limit − gas_used) × P`

When `gas_limit = floor_gas` and `gas_used = intrinsic_gas < floor_gas`, the delta `(floor_gas − intrinsic_gas) × P` is returned to the sender. EIP-7623 requires that delta to be burned.

**Concrete numbers (1000 zero-byte calldata, Prague config):**

| Quantity | Value |
|---|---|
| `floor_gas` | `21000 + 1000×1×10 = 31000` |
| `intrinsic_gas` | `21000 + 1000×4 = 25000` |
| `gas_limit` (minimum valid) | `31000` |
| `gas_used` (simple transfer) | `25000` |
| `prepaid` | `31000 × P` |
| Sender refund | `6000 × P` ← should be burned |
| Net sender cost | `25000 × P` ← should be `31000 × P` |

---

### Impact Explanation

The sender recovers `(floor_gas − intrinsic_gas) × effective_gas_price` on every calldata-heavy transaction. This is yield that EIP-7623 designates as burned (consumed by the protocol). The relayer also receives only `gas_used × priority_fee` instead of `floor_gas × priority_fee`, under-compensating it for calldata processing. The floor-gas anti-spam mechanism is entirely neutralised: a sender can submit arbitrarily large zero-byte calldata payloads and pay only the intrinsic-gas cost, not the floor-gas cost.

**Impact: High — Theft of unclaimed yield** (sender recovers protocol-designated burned gas; relayer under-compensated for calldata yield).

---

### Likelihood Explanation

- Prague config is already the hardcoded `CONFIG` constant in the engine: [6](#0-5) 
- No special privileges are required; any EOA can submit a transaction with zero-byte calldata.
- The scenario is deterministic and reproducible with a single crafted transaction.

Likelihood: **High**.

---

### Recommendation

After EVM execution, clamp `gas_used` to the floor before computing the refund:

```rust
// engine/src/engine.rs — after line 1146
let gas_used = gas_used.max(floor_gas);
```

`floor_gas` is already in scope at that point (computed at line 1073–1075). This ensures `refund_unused_gas` never returns the floor-gas overhead to the sender, matching EIP-7623 semantics.

### Proof of Concept

```rust
// Deterministic local test (no mainnet required)
let config = aurora_evm::Config::prague(); // has_floor_gas = true

// 1000 zero bytes → floor_gas=31000, intrinsic_gas=25000
let calldata = vec![0u8; 1000];
let tx = NormalizedEthTransaction {
    gas_limit: U256::from(31_000u64), // = floor_gas (minimum valid)
    max_fee_per_gas: U256::from(1u64),
    max_priority_fee_per_gas: U256::from(1u64),
    data: calldata,
    to: Some(some_eoa), // simple transfer → gas_used = intrinsic_gas = 25000
    ..
};

// charge_gas deducts 31000 * 1 = 31000 wei
// EVM uses 25000 gas
// refund_unused_gas returns (31000 - 25000) * 1 = 6000 wei to sender
// Invariant violated: net_sender_cost (25000) < floor_gas_cost (31000)
assert_eq!(net_sender_cost, floor_gas * effective_gas_price); // FAILS
```

The invariant `net_sender_cost >= floor_gas × effective_gas_price` is broken for all transactions where `floor_gas > intrinsic_gas` and `gas_used < floor_gas`.

    pub fn floor_gas(&self, config: &aurora_evm::Config) -> Result<u64, Error> {
        if config.has_floor_gas {
            let num_zero_bytes = u64::try_from(self.data.iter().filter(|b| **b == 0).count())
                .map_err(|_e| Error::IntegerConversion)?;
            let data_len = u64::try_from(self.data.len()).map_err(|_e| Error::IntegerConversion)?;
            let num_non_zero_bytes = data_len
                .checked_sub(num_zero_bytes)
                .ok_or(Error::GasOverflow)?;

            let base_gas = config.gas_transaction_call;
            let tokens_in_calldata = num_non_zero_bytes
                .checked_mul(4)
                .and_then(|gas| gas.checked_add(num_zero_bytes))
                .ok_or(Error::GasOverflow)?;

            tokens_in_calldata
                .checked_mul(config.total_cost_floor_per_token)
                .and_then(|gas| gas.checked_add(base_gas))
                .ok_or(Error::GasOverflow)
        } else {
            Ok(0)
        }
    }
