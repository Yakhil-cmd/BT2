Audit Report

## Title
CMC `remove_subnet_from_authorized_subnet_list` Fails to Clean `default_subnets`, Leaving Stale Routing Entries After Subnet Deletion ‚Äî (File: rs/nns/cmc/src/main.rs)

## Summary
The Cycles Minting Canister maintains two independent subnet lists: `authorized_subnets` (per-principal) and `default_subnets` (global fallback). The only removal hook callable by the registry canister, `remove_subnet_from_authorized_subnet_list`, only cleans `authorized_subnets` and never touches `default_subnets`. Furthermore, `do_delete_subnet` in the registry canister does not call this hook at all. After a subnet deletion, stale entries persist in `default_subnets`, causing canister creation requests to be routed to a non-existent subnet and `get_default_subnets()` to return incorrect data.

## Finding Description
In `rs/nns/cmc/src/main.rs` lines 1117‚Äì1122, `remove_subnet_from_authorized_subnet_list` exclusively iterates over `authorized_subnets`:

```rust
with_state_mut(|state| {
    state
        .authorized_subnets
        .values_mut()
        .for_each(|subnet_list| subnet_list.retain(|subnet| *subnet != subnet_to_remove))
});
```

`default_subnets` is never modified. In `rs/registry/canister/src/mutations/do_delete_subnet.rs` lines 32‚Äì86, `do_delete_subnet` removes the subnet from `subnet_list`, the subnet record, the catch-up package, the threshold signing pubkey, and routing table shards ‚Äî but never calls `remove_subnet_from_authorized_subnet_list` on the CMC. Even if it did, `default_subnets` would still retain the stale entry.

The stale entry is then consumed in `do_create_canister` (lines 2207‚Äì2230). For `SubnetSelection::Subnet { subnet }`, the guard is:

```rust
if state.default_subnets.contains(&subnet) || ...
```

A deleted subnet still present in `default_subnets` passes this check unconditionally, and the call proceeds to `ic00::create_canister` on the non-existent subnet. For `subnet_selection = None`, `get_subnets_for` returns `default_subnets` and the deleted subnet may be randomly selected. The query endpoint `get_default_subnets()` (lines 792‚Äì803) also returns the stale list.

## Impact Explanation
This matches **High** ‚Äî "Significant NNS/infrastructure security impact with concrete user or protocol harm." The CMC is a core NNS canister. After a legitimate subnet deletion, any unprivileged user calling `create_canister` with `SubnetSelection::Subnet { subnet: deleted_subnet }` is guaranteed to fail because the stale `default_subnets` entry bypasses the authorization guard. Users relying on `get_default_subnets()` receive incorrect routing information, leading to repeated failed creation attempts. Cycles paid via `call_with_payment128` are returned by the IC protocol on inter-canister call failure, but the original caller's refund depends on the CMC's error-handling path, which is not fully guaranteed to be lossless in all edge cases.

## Likelihood Explanation
The preconditions are two legitimate governance proposals: (1) `set_authorized_subnetwork_list(who = None, subnets = [subnet_X])` to add a subnet to `default_subnets`, and (2) `do_delete_subnet(subnet_id = subnet_X)` to delete it. Both are normal administrative operations. `do_delete_subnet` is currently restricted to `SubnetType::CloudEngine` subnets, which are an active deployment target. Once these two proposals execute, any unprivileged ingress caller triggers the bug with no special access. The governance proposals are not adversarial ‚Äî they are the expected lifecycle of a CloudEngine subnet.

## Recommendation
1. Update `remove_subnet_from_authorized_subnet_list` to also clean `default_subnets`:
   ```rust
   state.default_subnets.retain(|s| *s != subnet_to_remove);
   state.authorized_subnets.values_mut()
       .for_each(|list| list.retain(|s| *s != subnet_to_remove));
   ```
2. Update `do_delete_subnet` in `rs/registry/canister/src/mutations/do_delete_subnet.rs` to call `remove_subnet_from_authorized_subnet_list` on the CMC as part of the deletion flow, analogous to how it already calls `remove_subnet_from_routing_table`.
3. Consider renaming the function to `remove_subnet_from_all_subnet_lists` to reflect its intended scope.

## Proof of Concept
1. Submit governance proposal A: call `set_authorized_subnetwork_list(who = None, subnets = [subnet_X])` on the CMC ‚Äî `subnet_X` is added to `default_subnets`.
2. Submit governance proposal B: call `do_delete_subnet(subnet_id = subnet_X)` on the registry canister ‚Äî `subnet_X` is removed from the registry. `remove_subnet_from_authorized_subnet_list` is never called; `default_subnets` still contains `subnet_X`.
3. Any user calls `create_canister` with `subnet_selection = SubnetSelection::Subnet { subnet: subnet_X }`. The CMC evaluates `state.default_subnets.contains(&subnet_X)` ‚Üí `true`, proceeds, and the downstream `ic00::create_canister` call fails unconditionally.
4. Any user calls `create_canister` with no `subnet_selection`. If `subnet_X` is randomly selected from `default_subnets`, the call fails.
5. `get_default_subnets()` still returns `subnet_X` as a valid deployment target.
A deterministic integration test using PocketIC can reproduce this by executing both proposals and then asserting that `create_canister` with the deleted subnet ID returns an error while `get_default_subnets()` still lists it.
