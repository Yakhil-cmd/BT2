### Title
Staker Can Instantly Increase Commission to 100% via `set_commission_commitment`, Stealing All Delegator Yield - (File: src/staking/staking.cairo)

---

### Summary

A staker can exploit the `set_commission_commitment` mechanism to set `max_commission = 10000` (100%) and then immediately call `set_commission` to raise their commission to 100% in the same block. This drains all future pool rewards away from delegators with no delay or warning. The codebase itself acknowledges this gap with an inline note.

---

### Finding Description

Without a `CommissionCommitment`, `set_commission` enforces a strict monotone-decrease invariant: commission can only go down. [1](#0-0) 

However, when an active `CommissionCommitment` exists, the invariant is replaced by a ceiling check: commission may be set to **any** value up to `max_commission`, including values higher than the current commission. [2](#0-1) 

`set_commission_commitment` permits `max_commission` up to `COMMISSION_DENOMINATOR` (10 000 = 100%), requires only that `max_commission >= current_commission`, and that `expiration_epoch > current_epoch` (minimum: one epoch in the future). [3](#0-2) 

There is **no enforced delay** between setting the commitment and using it to raise commission. A staker can therefore:

1. Call `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
2. Immediately call `set_commission(commission: 10000)` in the same block.

The developers acknowledge this gap with an inline note directly above `set_commission_commitment`: [4](#0-3) 

The commission is applied at reward-distribution time inside `_update_rewards` via `split_rewards_with_commission`. At 100% commission, `pool_rewards = 0` for every pool. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

All rewards that delegators have earned (or will earn) for the current and subsequent epochs are redirected entirely to the staker. Rewards already accumulated in the pool's `cumulative_rewards_trace` before the commission change are safe, but every reward distribution that occurs after the change yields zero to the pool. Delegators must wait through the full `exit_wait_window` (default 1 week, up to 12 weeks) before they can recover their principal, during which they earn nothing.

---

### Likelihood Explanation

**High.** Any staker who has opened a delegation pool can execute this attack permissionlessly in two consecutive transactions. The staker has a direct financial incentive: capturing 100% of the pool's share of epoch rewards. Delegators have no on-chain mechanism to react before the commission change takes effect, because the commitment and the commission update can land in the same block.

---

### Recommendation

Enforce a minimum delay between setting a `CommissionCommitment` and the first commission increase that uses it. For example, require that `expiration_epoch - current_epoch >= MIN_COMMITMENT_DELAY_EPOCHS` **and** that any commission increase via a commitment cannot take effect until at least `MIN_COMMITMENT_DELAY_EPOCHS` epochs after the commitment was set. This gives delegators time to observe the pending change and exit before it applies.

Alternatively, cap the maximum single-step commission increase (e.g., no more than N basis points per epoch) regardless of the commitment ceiling.

---

### Proof of Concept

```
// Precondition: staker has commission = 100 (1%), pool has delegators.

// Step 1 – set commitment with max_commission = 10000, expiring next epoch.
staking.set_commission_commitment(
    max_commission: 10000,
    expiration_epoch: staking.get_current_epoch() + 1,
);
// is_commission_commitment_active → true (current_epoch < expiration_epoch)

// Step 2 – immediately raise commission to 100% in the same block.
staking.set_commission(commission: 10000);
// update_commission: commitment active → assert(10000 <= 10000) ✓
//                                        assert(10000 != 100)   ✓
// Commission is now 10000.

// Step 3 – next reward distribution:
//   split_rewards_with_commission(pool_rewards_including_commission, commission=10000)
//   → commission_rewards = pool_rewards_including_commission * 10000 / 10000
//   → pool_rewards = 0
// Delegators receive zero rewards for every epoch going forward.
```

The test suite already demonstrates that `max_commission = COMMISSION_DENOMINATOR` is accepted by `set_commission_commitment` and that commission can be raised to `max_commission` immediately after: [7](#0-6) [8](#0-7)

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}

fn test_set_commission_with_commitment() {
    let mut cfg: StakingInitConfig = Default::default();
    let staking_contract = deploy_staking_contract(:cfg);
    cfg.test_info.staking_contract = staking_contract;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    stake_with_strk_pool_enabled(:cfg);

    // Set commitment.
    let staker_address = cfg.test_info.staker_address;
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    let max_commission = staker_info.get_pool_info().commission + 2;
    let expiration_epoch = staking_dispatcher.get_current_epoch() + 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission_commitment(:max_commission, :expiration_epoch);

    // Update commission.
    let mut commission = max_commission;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission(:commission);

    // Assert commission is updated.
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info.get_pool_info().commission == commission);

    // Advance to the expiration epoch.
    advance_epoch_global();

    // Lower commission.
    commission = commission - 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission(:commission);

    // Assert commission is updated.
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    assert!(staker_info.get_pool_info().commission == commission);
}

fn test_set_commission_commitment() {
    let mut cfg: StakingInitConfig = Default::default();
    general_contract_system_deployment(ref :cfg);
    let staking_contract = cfg.test_info.staking_contract;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    stake_with_strk_pool_enabled(:cfg);
    let staker_address = cfg.test_info.staker_address;
    let max_commission = COMMISSION_DENOMINATOR;
    let mut spy = snforge_std::spy_events();
    let expiration_epoch = staking_dispatcher.get_current_epoch() + 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission_commitment(:max_commission, :expiration_epoch);
    let commission_commitment = staking_dispatcher
        .get_staker_commission_commitment(:staker_address);
    let expected_commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
    assert!(commission_commitment == expected_commission_commitment);
    // Validate the CommissionCommitmentSet event.
    let events = spy.get_events().emitted_by(contract_address: staking_contract).events;
    assert_number_of_events(
        actual: events.len(), expected: 1, message: "set_commission_commitment",
    );
    assert_commission_commitment_set_event(
        spied_event: events[0], :staker_address, :max_commission, :expiration_epoch,
    );
}
