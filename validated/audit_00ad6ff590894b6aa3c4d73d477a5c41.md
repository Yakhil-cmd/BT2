### Title
Gateway Stateful Admission Validates Only L2 Gas Price, Allowing Transactions with Zero L1/L1-Data Gas Price to Enter the Mempool — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check validates only the L2 gas price against the previous block's threshold. L1 gas price and L1 data gas price are never checked at admission time. A V3 (`AllResources`) transaction with `l1_gas.max_price_per_unit = 0` passes both stateless and stateful gateway validation, is admitted to the mempool, and is guaranteed to be rejected by the blockifier at execution time with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` (lines 223–243) calls `validate_tx_l2_gas_price_within_threshold`, which only inspects `l2_gas.max_price_per_unit`: [1](#0-0) 

The function itself carries an explicit acknowledgement of the gap:

```
// TODO(Arni): Consider running this validation for all gas prices.
```

For `ValidResourceBounds::AllResources`, only `l2_gas.max_price_per_unit` is compared against the previous block's L2 gas price threshold. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared against the current L1 gas prices. [2](#0-1) 

The stateless validator (`StatelessTransactionValidator::validate_resource_bounds`) also only checks `l2_gas.max_price_per_unit` against a static floor; it does not inspect L1 gas prices at all: [3](#0-2) 

The `max_possible_fee` zero-check passes as long as any single resource bound is non-zero, so a transaction with `l2_gas` bounds set and `l1_gas.max_price_per_unit = 0` clears it. [4](#0-3) 

Downstream, the blockifier's `check_fee_bounds` inside `perform_pre_validation_stage` checks **all three** resource prices against the actual block prices: [5](#0-4) 

A transaction with `l1_gas.max_price_per_unit = 0` will fail here with `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`, but only after it has already been admitted to the mempool.

### Impact Explanation

The gateway/mempool admission accepts transactions that are structurally invalid with respect to the current L1 gas price. These transactions will never be included in a block. An unprivileged attacker can flood the mempool with such transactions at negligible cost (no execution fee is ever charged because the transaction is rejected before execution), consuming mempool capacity and degrading sequencer throughput.

This matches the stated High impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

Exploitation requires only crafting a standard V3 invoke transaction with `l1_gas.max_price_per_unit = 0` and a valid L2 gas price. No privileged access, special account, or contract deployment is needed. The attack is trivially repeatable.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or introduce a parallel function) to also validate `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding STRK gas prices, applying the same `min_gas_price_percentage` threshold. The previous block's L1 gas prices are already fetched from `gateway_fixed_block_state_reader.get_block_info()` in `validate_resource_bounds`: [6](#0-5) 

The fix should mirror the existing L2 check for `l1_gas` and `l1_data_gas` fields of `AllResourceBounds`, using `block_info.gas_prices.strk_gas_prices.l1_gas_price` and `l1_data_gas_price` as the reference values.

### Proof of Concept

1. Craft a V3 invoke transaction with `ValidResourceBounds::AllResources`:
   - `l2_gas.max_price_per_unit` = current L2 gas price (passes threshold)
   - `l2_gas.max_amount` = any non-zero value
   - `l1_gas.max_price_per_unit` = `0`
   - `l1_data_gas.max_price_per_unit` = `0`
2. Submit to the gateway via `POST /gateway/add_transaction`.
3. **Stateless check passes**: `max_possible_fee > 0` (L2 contribution is non-zero); `l2_gas.max_price_per_unit >= min_gas_price`.
4. **Stateful check passes**: `validate_tx_l2_gas_price_within_threshold` only checks `l2_gas.max_price_per_unit`; L1 prices are ignored.
5. Transaction is admitted to the mempool.
6. When the batcher pulls the transaction and the blockifier calls `check_fee_bounds`, it evaluates `l1_gas.max_price_per_unit (0) < actual_l1_gas_price` and returns `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`.
7. The transaction is discarded without ever being included in a block; no fee is charged.

Repeating step 1–5 at high frequency fills the mempool with permanently-invalid transactions.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
