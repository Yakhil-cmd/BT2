### Title
Unprivileged Caller Can Permanently Freeze All Stakers' Consensus Rewards via `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in the Staking contract has no access control. Any non-zero address can call it with `disable_rewards: true` for any valid staker address. Because `last_reward_block` is a **global** slot that allows only one call per block, an attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block permanently prevents all stakers from receiving consensus block rewards.

### Finding Description

`update_rewards` is exposed as a public function in `IStakingRewardsManager`: [1](#0-0) 

The only caller restriction is `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address: [2](#0-1) 

The function enforces a global per-block gate via `last_reward_block`: [3](#0-2) 

After passing that gate, it writes the current block number unconditionally: [4](#0-3) 

When `disable_rewards` is `true`, the function returns immediately after updating `last_reward_block`, distributing nothing: [5](#0-4) 

Because `last_reward_block` is a single global value shared across all stakers, the first call in any block "consumes" the slot for that block. Any subsequent call in the same block fails with `REWARDS_ALREADY_UPDATED`. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block:

1. Passes all checks (non-zero caller, valid staker address, block > last_reward_block).
2. Writes `last_reward_block = current_block`.
3. Returns early — zero rewards distributed.
4. Blocks every legitimate `update_rewards` call for that block.

The attacker does not need to be a staker. They only need to supply any currently-active staker address, which is publicly observable from `NewStaker` events. [6](#0-5) 

### Impact Explanation

Continuous execution of this attack permanently freezes consensus block rewards for every staker and every delegation pool in the protocol. No staker can accumulate `unclaimed_rewards_own`, and no pool receives `update_rewards_from_staking_contract` calls. This matches the allowed impact: **Permanent freezing of unclaimed yield**. [7](#0-6) 

### Likelihood Explanation

- The entry point is a public function with no role gate.
- The attacker needs only a non-zero address and knowledge of one active staker (trivially obtained from events).
- On Starknet, transaction fees are low, making per-block griefing economically feasible.
- No special privilege, leaked key, or external dependency is required.

### Recommendation

Restrict `update_rewards` to a trusted caller — either the attestation contract, a designated consensus contract, or a specific role (e.g., `only_app_governor` / `only_operator`). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "no-reward" path internally based on attestation state, so an unprivileged caller cannot suppress reward distribution.

### Proof of Concept

```
// Attacker script (pseudocode, run once per block)
loop every new block N:
    staking_contract.update_rewards(
        staker_address = <any known active staker>,
        disable_rewards = true
    )
    // Effects:
    //   last_reward_block := N
    //   No rewards distributed
    //   All other update_rewards calls in block N revert with REWARDS_ALREADY_UPDATED
```

Concrete call path:
1. Attacker (any non-zero EOA) calls `update_rewards(staker_A, true)` in block N.
2. `general_prerequisites()` passes — contract not paused, caller ≠ 0.
3. `current_block_number (N) > last_reward_block` — passes (first call this block).
4. `internal_staker_info(staker_A)` — passes (staker_A is valid).
5. `is_staker_active(staker_A, curr_epoch)` — passes.
6. `last_reward_block.write(N)` — slot consumed.
7. `disable_rewards == true` → early return, zero rewards.
8. Staker B calls `update_rewards(staker_B, false)` in block N → reverts: `REWARDS_ALREADY_UPDATED`.
9. Attacker repeats in block N+1, N+2, … → all stakers permanently frozen. [8](#0-7)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
