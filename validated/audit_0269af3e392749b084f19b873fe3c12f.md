### Title
Missing Authorization Check and No-Op Implementation in `TurnBridge` Bypasses Bridge Circuit Breaker - (File: `x/cronos/keeper/msg_server.go`)

---

### Summary

`MsgTurnBridge` is the designated emergency circuit breaker for the Gravity bridge. Its handler `TurnBridge` in `x/cronos/keeper/msg_server.go` contains **no authorization check** and is implemented as a complete no-op (`return nil, nil`). Any unprivileged user can submit `MsgTurnBridge` and receive a success response, while the bridge state is never actually changed. The `CanTurnBridge` permission system is entirely bypassed, and the circuit breaker is permanently non-functional.

---

### Finding Description

`UpdateTokenMapping` — the sibling restricted message — correctly enforces its permission gate:

```go
// x/cronos/keeper/msg_server.go:69-81
func (k msgServer) UpdateTokenMapping(...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    ...
}
```

`TurnBridge`, which is explicitly listed alongside `UpdateTokenMapping` in ADR-009 as requiring `CanTurnBridge` permission, has no such check and performs no state mutation:

```go
// x/cronos/keeper/msg_server.go:84-87
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
```

The permission constants are defined and the `CanTurnBridge` bit exists in the permission system:

```go
// x/cronos/keeper/permissions.go:13-17
const (
    CanChangeTokenMapping uint64 = 1 << iota // 1
    CanTurnBridge                            // 2
    All = CanChangeTokenMapping | CanTurnBridge // 3
)
```

The proto definition confirms `MsgTurnBridge` is a live, routable message with a `sender` signer field:

```proto
// proto/cronos/tx.proto:82-89
message MsgTurnBridge {
  option (cosmos.msg.v1.signer) = "sender";
  string sender = 1;
  bool   enable = 2;
}
```

The integration test `test_gravity_turn_bridge` demonstrates the intended behavior: a non-permissioned caller should be rejected (`code != 0`), and a permissioned caller should succeed and actually disable bridge operations. Neither outcome is achievable with the current no-op implementation.

---

### Impact Explanation

Two distinct impacts compound each other:

1. **Auth bypass (High):** The `CanTurnBridge` permission gate — the only mechanism restricting who may invoke the bridge circuit breaker — is entirely absent. Any unprivileged address can submit `MsgTurnBridge` and receive a success response, bypassing the Cronos admin/permission authorization model.

2. **Permanent circuit-breaker failure (High):** Because the handler performs no state mutation, the Gravity bridge can never be disabled regardless of who calls it. In a bridge exploit scenario, operators have no on-chain mechanism to halt outflows. This is a permanent, protocol-level inability to execute a critical bridge-control flow under normal network assumptions.

---

### Likelihood Explanation

- `MsgTurnBridge` is a publicly routable gRPC message reachable by any address with gas.
- No preconditions, no keys, no validator cooperation required.
- The attacker path is a single signed transaction.
- The missing implementation is unconditional — it cannot be worked around by any operational procedure.

---

### Recommendation

Apply the same permission guard used by `UpdateTokenMapping`, and implement the actual bridge-state toggle:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)

    signer, err := sdk.AccAddressFromBech32(msg.Sender)
    if err != nil {
        return nil, err
    }
    if !k.HasPermission(ctx, []sdk.AccAddress{signer}, CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }

    // persist bridge-enabled flag and act on it in EndBlocker / gravity hooks
    k.SetBridgeEnabled(ctx, msg.Enable)
    return &types.MsgTurnBridgeResponse{}, nil
}
```

---

### Proof of Concept

1. Any address submits `MsgTurnBridge{sender: <attacker>, enable: false}`.
2. The handler executes, performs no permission check, performs no state change, and returns `nil, nil` (success).
3. The Gravity bridge remains fully operational regardless of the `enable` field value.
4. A legitimate admin with `CanTurnBridge` permission also cannot disable the bridge, because the implementation is a no-op for all callers.
5. In a live bridge exploit, there is no on-chain path to halt bridge outflows.
