### Title
Unchecked Deployer Identity in `GlobalContractDeployMode::AccountId` Allows Any Account to Overwrite Another Account's Global Contract - (File: `runtime/runtime/src/global_contracts.rs`)

---

### Summary

`action_deploy_global_contract` in `runtime/runtime/src/global_contracts.rs` does not verify that the receipt's `predecessor_id` (the transaction signer) matches the `account_id` (the receipt receiver) when deploying a global contract in `GlobalContractDeployMode::AccountId` mode. Any unprivileged account A can overwrite account B's global contract by sending a standard transaction with `receiver_id = B` and `DeployGlobalContract { deploy_mode: AccountId, code: malicious_code }`. Because all accounts that previously called `UseGlobalContract(AccountId(B))` hold a live reference `AccountContract::GlobalByAccount(B)` that is resolved at call time, they immediately execute A's malicious code on their next function call.

---

### Finding Description

**Protocol model.** In NEAR, a transaction with `signer_id = A` and `receiver_id = B` produces a receipt with `predecessor_id = A` and `receiver_id = B`. The runtime applies the receipt's actions to B's account. The `account_id` variable inside `apply_action_receipt` is set to `receipt.receiver_id()`: [1](#0-0) 

**The missing ownership check.** When `DeployGlobalContract` is dispatched, `account_id` (B, the receiver) is passed directly to `action_deploy_global_contract`: [2](#0-1) 

Inside `action_deploy_global_contract`, the only guard is a balance check on B's account. There is no assertion that `predecessor_id == account_id`: [3](#0-2) 

`initiate_distribution` then derives the global contract identifier directly from `account_id` (B), not from the signer: [4](#0-3) 

The nonce is incremented from whatever is currently stored for `GlobalContractIdentifier::AccountId(B)`, so A's distribution receipt carries a strictly higher nonce than B's last deployment and passes the freshness check: [5](#0-4) 

**No existing guard.** `check_account_existence` only verifies that B's account exists, not that the predecessor is B: [6](#0-5) 

`validate_deploy_global_contract_action` only checks code size: [7](#0-6) 

Transaction-level access-key validation enforces that A holds a `FullAccess` key on A's own account, but imposes no constraint on `receiver_id` for non-`FunctionCall` actions: [8](#0-7) 

**Attacker-controlled field.** The `receiver_id` field of the signed transaction is fully attacker-controlled. It flows into `account_id` inside `apply_action_receipt` and then into `initiate_distribution` as the `AccountId` key under which the global contract code is stored.

**Corrupted value.** After the attack, `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(B) }` contains A's malicious WASM binary instead of B's legitimate code.

**Live-reference propagation.** `use_global_contract` stores `AccountContract::GlobalByAccount(B)` on every account that opted in: [9](#0-8) 

This reference is resolved at function-call time, so all opted-in accounts immediately execute A's code without any further action on their part.

---

### Impact Explanation

- **Scope:** global contract/code selection.
- **Corrupted value:** `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(B) }` — the WASM binary executed by every account holding `AccountContract::GlobalByAccount(B)`.
- **Effect:** A can deploy arbitrary WASM under B's namespace. Every account that previously called `UseGlobalContract(AccountId(B))` will execute A's code on their next function call. A's code runs with the full host-function surface of the victim account, enabling theft of funds, state corruption, or key deletion across an unbounded set of victim accounts.
- **Storage drain:** The storage cost is deducted from B's balance, not A's, so A also drains B's NEAR balance proportional to the malicious contract size.
- **Severity:** Critical — one transaction affects all users of B's global contract simultaneously.

---

### Likelihood Explanation

- A needs only a standard `FullAccess` key on their own account (the default for any NEAR account).
- B must have enough balance to cover the storage cost of A's payload; if B is a popular global-contract provider this is virtually guaranteed.
- The attack is a single signed transaction; no special timing, validator access, or privileged role is required.
- `GlobalContractDeployMode::AccountId` is explicitly documented as the mutable-upgrade path ("This allows the owner to update the contract for all its users"), making it a high-value target. [10](#0-9) 

---

### Recommendation

Add an authorization guard at the top of `action_deploy_global_contract` (or equivalently in `apply_action` before dispatch) that rejects the action when the receipt's `predecessor_id` differs from `account_id`:

```rust
// In action_deploy_global_contract, add before the balance check:
if receipt.predecessor_id() != account_id {
    result.result = Err(ActionErrorKind::ActorNoPermission {
        account_id: account_id.clone(),
        actor_id: receipt.predecessor_id().clone(),
    }.into());
    return Ok(());
}
```

This mirrors the existing pattern used in `apply_delegate_action`, which explicitly checks `delegate_action.sender_id() == sender_id`: [11](#0-10) 

Alternatively, enforce the constraint at the action-validation layer so that `DeployGlobalContract { deploy_mode: AccountId }` is only accepted when `signer_id == receiver_id` at the transaction level.

---

### Proof of Concept

```
// Step 1: B deploys a legitimate global contract under B's account ID.
SignedTransaction {
    signer_id:   "b.near",
    receiver_id: "b.near",
    actions: [DeployGlobalContract { code: legitimate_wasm, deploy_mode: AccountId }],
    // signed by B's FullAccess key
}

// Step 2: Many accounts opt in.
SignedTransaction {
    signer_id:   "victim.near",
    receiver_id: "victim.near",
    actions: [UseGlobalContract { contract_identifier: AccountId("b.near") }],
}
// ... repeated for every victim account

// Step 3: Attacker A overwrites B's global contract.
SignedTransaction {
    signer_id:   "a.near",      // attacker's own account
    receiver_id: "b.near",      // victim deployer — no ownership check performed
    actions: [DeployGlobalContract { code: malicious_wasm, deploy_mode: AccountId }],
    // signed by A's own FullAccess key — passes all access-key validation
}

// Result: TrieKey::GlobalContractCode { identifier: AccountId("b.near") }
//         now contains malicious_wasm.
//         Every call to any victim account executes malicious_wasm.
```

The distribution receipt produced in step 3 carries nonce = (B's current nonce + 1), which passes `check_and_update_nonce`, propagates to all shards, and atomically replaces B's code network-wide. [12](#0-11)

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn check_and_update_nonce(
    global_contract_data: &GlobalContractDistributionReceipt,
    identifier: &GlobalContractCodeIdentifier,
    state_update: &mut TrieUpdate,
) -> Result<bool, RuntimeError> {
    let nonce_key = TrieKey::GlobalContractNonce { identifier: identifier.clone() };
    let stored_nonce = get_nonce(state_update, &nonce_key)?;
    let incoming_nonce = global_contract_data.nonce();

    // Allow the same nonce since the nonce is updated immediately when
    // initiating distribution to prevent multiple distributions with the same
    // nonce from being initiated.
    if incoming_nonce < stored_nonce {
        return Ok(false);
    }

    set_nonce(state_update, nonce_key, incoming_nonce);
    Ok(true)
