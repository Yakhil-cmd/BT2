### Title
Pool Rewards Permanently Locked When Total Delegation Falls Below `min_delegation_for_rewards` — (File: `src/pool/pool.cairo`)

---

### Summary

When the total delegated balance in a `Pool` contract falls below `min_delegation_for_rewards` (1 STRK = 10^18 fractions), the staking contract still transfers STRK reward tokens to the pool contract, but the pool's internal accounting (`cumulative_rewards_trace`) is not updated. Pool members can never claim these rewards, and no emergency withdrawal mechanism exists in the pool contract. The STRK tokens are permanently locked.

---

### Finding Description

The `Pool` contract tracks per-epoch reward accrual via a `cumulative_rewards_trace`. When the staking contract distributes rewards, it calls two functions in sequence:

**Step 1 — Transfer tokens to pool** (`staking.cairo`, `send_rewards_to_delegation_pool`): [1](#0-0) 

**Step 2 — Update pool's internal accounting** (`pool.cairo`, `update_rewards_from_staking_contract`): [2](#0-1) 

Inside step 2, `compute_rewards_per_unit` is called. When `total_stake < min_delegation_for_rewards`, it returns **zero**: [3](#0-2) 

The developer comment directly above this function acknowledges the problem: [4](#0-3) 

The result: STRK tokens are physically transferred to the pool contract in step 1, but the `cumulative_rewards_trace` receives a zero increment in step 2. Pool members compute their claimable rewards exclusively from `cumulative_rewards_trace`: [5](#0-4) 

Since the trace shows no increase, `claim_rewards` transfers nothing for those epochs. The STRK tokens sit in the pool contract with no function capable of recovering them — there is no `withdraw`, `sweep`, or emergency-exit function anywhere in `Pool`.

The staking contract only adds a pool to the reward distribution array when `pool_rewards.is_non_zero()`: [6](#0-5) 

`pool_rewards` can be non-zero even when `pool_balance < min_delegation_for_rewards` because the reward calculation uses normalized amounts divided by total stake, which can yield a non-zero result for small-but-nonzero pool balances when total epoch rewards are large. The `min_delegation_for_rewards` guard fires only later, inside `compute_rewards_per_unit`, after the tokens have already been sent.

---

### Impact Explanation

STRK reward tokens transferred to a `Pool` contract whose total delegation is below 1 STRK are permanently unclaimable. The pool contract exposes no mechanism to recover arbitrary token balances. `exit_delegation_pool_action` returns only the delegated principal; `claim_rewards` returns only what `cumulative_rewards_trace` records. The locked tokens match the allowed impact: **Permanent freezing of unclaimed yield** (High).

---

### Likelihood Explanation

This condition is reachable without any privileged access:

1. A staker opens a pool. Early in the pool's life, or after most delegators exit via `exit_delegation_pool_intent` / `exit_delegation_pool_action`, the remaining total delegation can drop below 1 STRK.
2. `enter_delegation_pool` enforces only `amount.is_non_zero()` — there is no minimum delegation floor.
3. As long as the staker continues to attest (triggering `update_rewards` / `_update_rewards`), rewards are computed and forwarded to the pool each epoch.
4. Every epoch where `pool_balance < 10^18` fractions of STRK, the forwarded reward tokens are locked.

The scenario is realistic during pool wind-down or when a pool is bootstrapped with a trivially small delegation.

---

### Recommendation

The staking contract should guard against forwarding rewards to a pool whose balance is below `min_delegation_for_rewards`. Concretely, in `calculate_staker_pools_rewards` (or `update_pool_rewards`), skip the `send_rewards_to_delegation_pool` + `update_rewards_from_staking_contract` call pair when `pool_balance.to_native_amount(:decimals) < pool_dispatcher.min_delegation_for_rewards()`. Alternatively, expose a governance-controlled `sweep_unclaimable_rewards` function on the `Pool` contract that can recover tokens not reflected in `cumulative_rewards_trace`.

---

### Proof of Concept

1. Staker S stakes and opens a STRK pool P.
2. Delegator D calls `enter_delegation_pool` with `amount = 5 * 10^17` (0.5 STRK). Pool total balance = 0.5 STRK < `min_delegation_for_rewards` (1 STRK).
3. Staker S attests in epoch E, triggering `update_rewards` → `_update_rewards` → `calculate_staker_pools_rewards`.
4. Suppose `total_rewards = 10^22` (10,000 STRK) and `total_stake = 10^22`. Then `pool_rewards_including_commission = 10^22 * 5*10^17 / 10^22 = 5*10^17`. After commission split, `pool_rewards > 0`.
5. `update_pool_rewards` calls `send_rewards_to_delegation_pool` → STRK transferred to pool P.
6. `update_rewards_from_staking_contract(rewards = pool_rewards, pool_balance = 5*10^17)` is called. Inside, `compute_rewards_per_unit` checks `5*10^17 < 10^18` → returns 0. `cumulative_rewards_trace` is updated with `last + 0`.
7. D calls `claim_rewards(D)`. `calculate_rewards` computes `rewards = 0` because the trace shows no increase.
8. The STRK tokens from step 5 remain in pool P forever with no recovery path.

```text
            let (_, last) = self.cumulative_rewards_trace.last().unwrap();
            self
                .cumulative_rewards_trace
                .insert(
                    key: self.get_current_epoch(),
                    value: last
                        + self

                                staking_rewards: rewards, total_stake: pool_balance,
                            ),
                );
```

```text
        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
```

```text
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
```
