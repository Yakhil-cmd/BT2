### Title
Commission Rate Change Applied Retroactively to Entire Epoch's Rewards - (File: src/staking/staking.cairo)

### Summary
The `calculate_staker_pools_rewards` function reads the current commission at reward calculation time and applies it to the **entire epoch's** pool rewards. A staker who holds an active `CommissionCommitment` can increase their commission mid-epoch, causing the new higher rate to be applied retroactively to all rewards accrued since the epoch began, stealing unclaimed yield from delegators.

### Finding Description
In `calculate_staker_pools_rewards`, the commission is fetched from live storage with no historical segmentation:

```cairo
let commission = staker_pool_info.commission();   // line 1964 – always current value
```

This single value is then passed directly to `split_rewards_with_commission` and applied to the full epoch's pool rewards. [1](#0-0) 

The commission is stored as a plain scalar with no epoch-keyed trace: [2](#0-1) 

`update_rewards_from_attestation_contract` (the pre-consensus reward path) calls `_update_rewards` exactly once per epoch, at attestation time. Whatever commission is stored at that moment is applied to the entire epoch's rewards: [3](#0-2) 

The `set_commission_commitment` mechanism explicitly allows a staker to raise their commission above the current value, up to `max_commission`, at any time while the commitment is active: [4](#0-3) 

The developers themselves note this gap: *"Current commission increase safeguards still allow for sudden commission changes."* [5](#0-4) 

### Impact Explanation
A staker can steal unclaimed yield from every delegator in their pool. The stolen amount equals:

```
(new_commission − old_commission) × epoch_rewards × (fraction_of_epoch_before_change)
```

For example, if a staker raises commission from 5 % to 100 % one block before attestation, they capture nearly the entire epoch's pool rewards that delegators had been earning all epoch. This is a direct, quantifiable theft of delegator yield with no principal loss required.

**Impact: High — Theft of unclaimed yield.**

### Likelihood Explanation
- Any staker can call `set_commission_commitment` (no privileged role required) to unlock upward commission changes.
- The staker can then call `set_commission` at any point in the epoch to raise the rate.
- The attack requires no external dependency, no oracle, and no bridge interaction.
- The only cost is the staker's own commission revenue for the portion of the epoch before the change (which they sacrifice to steal a larger share from delegators).
- Likelihood: **Medium** — requires deliberate timing but is fully permissionless.

### Recommendation
Track commission changes with an epoch-keyed trace (analogous to how `staker_own_balance_trace` and `staker_delegated_balance_trace` work). When computing pool rewards, segment the epoch at each commission-change boundary and apply the rate that was in effect for each segment. Alternatively, enforce that commission changes only take effect at the start of the next epoch (i.e., delay by K epochs, consistent with balance-change semantics already used elsewhere in the protocol).

### Proof of Concept

**Attack steps (pre-consensus / V2 attestation path):**

1. Staker stakes and opens a pool with `commission = 500` (5 %).
2. Delegators enter the pool and begin accruing rewards for the epoch.
3. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
4. Immediately, staker calls `set_commission(commission: 10000)` — allowed because `10000 ≤ max_commission`.
5. One block later, the staker's operational address submits an attestation, triggering `update_rewards_from_attestation_contract`.
6. `calculate_staker_pools_rewards` reads `commission = 10000` and applies it to the **full epoch's** pool rewards.
7. Delegators receive 0 STRK for an epoch they spent earning rewards at 5 % commission; the staker receives 100 % of pool rewards.

**Relevant code path:**

```
set_commission_commitment  →  set_commission (raises rate)
  ↓
attestation  →  update_rewards_from_attestation_contract
  ↓
_update_rewards  →  calculate_staker_pools_rewards
  let commission = staker_pool_info.commission();   // ← reads new 100% rate
  split_rewards_with_commission(pool_rewards, commission)  // ← applied to full epoch
``` [6](#0-5) [7](#0-6)

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
