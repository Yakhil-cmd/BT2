### Title
`exit_to_near_precompile_callback` Does Not Validate the Amount Returned by `ft_transfer_call`, Causing Permanent ERC-20 Token Loss on Partial NEP-141 Refund — (`engine/src/contract_methods/connector.rs`)

---

### Summary

When a user exits ERC-20 tokens to NEAR via the Omni path (using `ft_transfer_call`), the `exit_to_near_precompile_callback` only checks whether the promise *succeeded*, but discards the actual returned value. Under NEP-141, `ft_transfer_call` returns the amount actually kept by the receiver (i.e., `exit_amount − refund_amount`). If the receiver's `ft_on_transfer` returns a non-zero partial refund, the ERC-20 tokens are permanently burned while the refunded NEP-141 tokens are stranded in Aurora's balance with no corresponding ERC-20 backing and no recovery path.

---

### Finding Description

**Step 1 — Omni exit path selects `ft_transfer_call`.**

In `engine-precompiles/src/native.rs`, `exit_erc20_token_to_near` selects the method based on the message suffix:

```rust
Some(Message::Omni(msg)) => (
    nep141_account_id,
    ft_transfer_call_args(&exit_params.receiver_account_id, exit_params.amount, msg)?,
    "ft_transfer_call",   // <-- uses ft_transfer_call, not ft_transfer
    None,
    ...
),
``` [1](#0-0) 

**Step 2 — ERC-20 tokens are burned before the promise is dispatched.**

The precompile is called from the ERC-20 burn function. By the time the promise is scheduled, the ERC-20 tokens are already destroyed.

**Step 3 — The callback discards the returned amount.**

In `engine/src/contract_methods/connector.rs`, `exit_to_near_precompile_callback` pattern-matches on `PromiseResult::Successful(_)` with a wildcard `_`, silently discarding the bytes that encode how many tokens the receiver actually kept:

```rust
let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
    // success branch — no check of the returned amount
    if let Some(args) = args.transfer_near { ... }
    None
} else if let Some(args) = args.refund {
    // only reached on full promise failure
    let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
    ...
}
``` [2](#0-1) 

Under NEP-141, `ft_transfer_call` returns `amount − refund_amount` as a JSON-encoded string. If the receiver's `ft_on_transfer` returns `R > 0`, the NEP-141 contract refunds `R` tokens back to Aurora's balance and the promise still resolves as `Successful`. The callback sees `Successful` and takes no action, so the `R` refunded NEP-141 tokens are stranded in Aurora's balance with no corresponding ERC-20 tokens.

**Step 4 — The `error_refund` guard only covers full promise failure.**

The `refund` branch (line 231) is only entered when `promise_result(0)` is *not* `Successful`. A partial refund still produces a `Successful` result, so the refund branch is never reached. [3](#0-2) 

---

### Impact Explanation

A user who exits `X` ERC-20 tokens via the Omni path loses `R` tokens permanently if the receiver partially refunds:

- `X` ERC-20 tokens are burned (irreversible).
- `X − R` NEP-141 tokens arrive at the receiver (correct).
- `R` NEP-141 tokens are refunded to Aurora's NEP-141 balance.
- No ERC-20 tokens are reminted for the `R` refunded NEP-141 tokens.
- The `R` NEP-141 tokens are permanently stranded in Aurora's balance with no ERC-20 backing and no withdrawal path.

**Impact: Permanent freezing/loss of user funds (Critical).**

---

### Likelihood Explanation

The Omni exit path is a production feature designed for cross-chain interactions (e.g., Omni bridge). Any NEAR receiver contract that legitimately processes only part of a transfer — such as a DEX that partially fills an order, a lending protocol that caps deposits, or any contract that conditionally accepts tokens — will return a non-zero refund. This is standard NEP-141 behavior and requires no malicious actor: the user themselves triggers the loss by exiting to a partially-accepting receiver.

**Likelihood: Medium** — the Omni path is reachable by any EVM user, and partial refunds are a normal, specified behavior of the NEP-141 standard.

---

### Recommendation

In `exit_to_near_precompile_callback`, parse the bytes returned by the successful `ft_transfer_call` promise and compare the kept amount to the original exit amount. If `kept_amount < exit_amount`, remint `exit_amount − kept_amount` ERC-20 tokens to the user's address (analogous to the existing `refund_on_error` path). The `ExitToNearPrecompileCallbackArgs` should be extended to carry the original exit amount and the ERC-20 address so the callback has enough context to perform the remint.

---

### Proof of Concept

1. User holds 100 ERC-20 tokens (backed by 100 NEP-141 tokens in Aurora's balance).
2. User calls the `ExitToNear` precompile with flag `0x1` and input `receiver.near:{"min_amount":"60"}` (Omni message).
3. Precompile burns 100 ERC-20 tokens from the user's EVM balance.
4. Precompile schedules `ft_transfer_call(receiver.near, 100, '{"min_amount":"60"}')` on the NEP-141 contract.
5. NEP-141 calls `receiver.near.ft_on_transfer(aurora, 100, '{"min_amount":"60"}')`.
6. Receiver accepts only 60 tokens and returns `40` (refund).
7. NEP-141 refunds 40 tokens to Aurora's NEP-141 balance and resolves the promise with `"60"`.
8. `exit_to_near_precompile_callback` receives `PromiseResult::Successful(b"\"60\"")`, matches `Successful(_)`, and returns `None` — no remint.
9. **Final state:** User has 0 ERC-20 tokens, receiver has 60 NEP-141 tokens, Aurora's NEP-141 balance has 40 orphaned tokens. The user has permanently lost 40 tokens. [4](#0-3) [5](#0-4)

```rust
pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
