## Title
Use-After-Revocation via Duplicate Key Index — Revoking Key 0 Does Not Revoke Duplicate Key Material at Index ≥ 1 - (`fvm/environment/account_public_key_util.go`, `fvm/environment/accounts.go`, `fvm/transactionVerifier.go`)

## Summary

Revoking account key 0 only sets `Revoked=true` in the dedicated `apk_0` register. The revoked-status bit for any duplicate of key 0 added at index ≥ 1 is stored independently in `keyMetadataBytes` inside the `AccountStatus` register and is never touched by `revokeAccountPublicKey0`. After key 0 is revoked, a transaction signed with the same private key material using `ProposalKey.KeyIndex=1` passes every revocation guard in both `TransactionSequenceNumberChecker` and `TransactionVerifier`.

## Finding Description

Flow uses a split storage model for account public keys:

- **Key 0** — stored in a dedicated `apk_0` register; its `Revoked` flag is embedded in the encoded `flow.AccountPublicKey` struct.
- **Keys ≥ 1** — the public key bytes are stored in batch registers; weight and revoked status are stored independently per-index in `keyMetadataBytes` inside the `AccountStatus` register.

When a duplicate of key 0 is added at index 1, `appendAccountPublicKeyMetadata` detects the duplicate via `FindDuplicateKey` and calls `AppendDuplicateKeyMetadata`. This stores the new key's own independent `revoked=false` bit in `keyMetadataBytes` and records a `storedKeyIndex` mapping back to stored key 0 (so the raw public key bytes are shared), but the revoked status is **not** shared. [1](#0-0) 

When key 0 is revoked, `RevokeAccountPublicKey(addr, 0)` dispatches to `revokeAccountPublicKey0`, which reads the `apk_0` register, sets `Revoked=true`, and writes it back. It does not scan for duplicate keys at other indices. [2](#0-1) [3](#0-2) 

When key 1 is revoked, `RevokeAccountPublicKey(addr, 1)` dispatches to `status.RevokeAccountPublicKey(1)`, which calls `accountkeymetadata.SetRevokedStatus(keyMetadataBytes, 1)` — a completely separate write path. [4](#0-3) 

The two revoked-status stores are therefore fully independent. Revoking key 0 leaves key 1's `revoked` bit in `keyMetadataBytes` as `false`.

## Impact Explanation

Both revocation guards read the revoked status through the same split dispatch:

**`TransactionSequenceNumberChecker`** (`fvm/transactionSequenceNum.go` line 56):
```go
revoked, err := accounts.GetAccountPublicKeyRevokedStatus(proposalKey.Address, proposalKey.KeyIndex)
```
For `keyIndex=1` this reads from `keyMetadataBytes` → returns `false` → passes. [5](#0-4) 

**`TransactionVerifier.getAccountKeysAndAggregateWeights`** (`fvm/transactionVerifier.go` line 298):
```go
accountKey, err := accounts.GetRuntimeAccountPublicKey(signature.Address, signature.KeyIndex)
if accountKey.Revoked { ... }
```
For `keyIndex=1`, `GetRuntimeAccountPublicKey` reads `revoked` from `keyMetadataBytes` → returns `false` → passes. [6](#0-5) [7](#0-6) 

The cryptographic signature verification then succeeds because the same private key signs for both index 0 and index 1 (the stored public key bytes are shared via `storedKeyIndex` mapping). The transaction is accepted.

## Likelihood Explanation

The precondition — holding `Keys.Add` entitlement — is a standard account capability in Flow. The realistic attack path is:

1. An account owner grants `Keys.Add` to a third party (e.g., a smart contract or co-signer).
2. The third party adds a duplicate of key 0 at index 1 before key 0 is revoked.
3. The account owner revokes key 0 (believing the key material is fully invalidated).
4. The third party continues to authorize transactions via index 1 with the same private key.

A second path requires no third party: an account owner who adds a backup duplicate key at index 1 and later revokes key 0 will unknowingly leave index 1 active.

## Recommendation

`revokeAccountPublicKey0` (and `RevokeAccountPublicKey` for any index) must scan all other key indices for duplicates (via the `storedKeyIndex` mapping in `keyMetadataBytes`) and revoke them atomically. Alternatively, revocation should be keyed on the stored key index rather than the logical key index, so that revoking any alias of a stored key revokes all aliases simultaneously.

Tx 3 passes both the sequence-number revocation check and the verifier revocation check, and the cryptographic signature is valid because the stored public key bytes at `storedKeyIndex=0` are shared. func revokeAccountPublicKey0(
	a Accounts,
	address flow.Address,
) error {
	accountPublicKey0RegisterID := flow.AccountPublicKey0RegisterID(address)

	publicKey, err := a.GetValue(accountPublicKey0RegisterID)
	if err != nil {
		return err
	}

	const keyIndex = uint32(0)

	if len(publicKey) == 0 {
		return errors.NewAccountPublicKeyNotFoundError(
			address,
			keyIndex)
	}

	decodedPublicKey, err := flow.DecodeAccountPublicKey(publicKey, keyIndex)
	if err != nil {
		return fmt.Errorf(
			"failed to decode account public key 0: %w",
			err)
	}

	decodedPublicKey.Revoked = true

	encodedPublicKey, err := flow.EncodeAccountPublicKey(decodedPublicKey)
	if err != nil {
		encoded, _ := decodedPublicKey.MarshalJSON()
		return errors.NewValueErrorf(
			string(encoded),
			"failed to encode revoked account public key 0: %w",
			err)
	}

	return a.SetValue(accountPublicKey0RegisterID, encodedPublicKey)
}

// AccountPublicKeyRevokedStatus returns revoked status of account public key at the given key index stored in key metadata.
// NOTE: To avoid checking keyIndex range repeatedly at different levels, caller must ensure keyIndex > 0 and < AccountPublicKeyCount().
func (a *AccountStatus) AccountPublicKeyRevokedStatus(keyIndex uint32) (bool, error) {
	return accountkeymetadata.GetRevokedStatus(a.keyMetadataBytes, keyIndex)

// AccountPublicKeyMetadata returns weight, revoked status, and stored key index of account public key at the given key index stored in key metadata.
// NOTE: To avoid checking keyIndex range repeatedly at different levels, caller must ensure keyIndex > 0 and < AccountPublicKeyCount().
func (a *AccountStatus) AccountPublicKeyMetadata(keyIndex uint32) (
	weight uint16,
	revoked bool,
	storedKeyIndex uint32,
	err error,
) {
	return accountkeymetadata.GetKeyMetadata(a.keyMetadataBytes, keyIndex, a.IsAccountKeyDeduplicated())

// RevokeAccountPublicKey revokes account public key at the given key index stored in key metadata.
// NOTE: To avoid checking keyIndex range repeatedly at different levels, caller must ensure keyIndex > 0 and < AccountPublicKeyCount().
func (a *AccountStatus) RevokeAccountPublicKey(keyIndex uint32) error {
	var err error
	a.keyMetadataBytes, err = accountkeymetadata.SetRevokedStatus(a.keyMetadataBytes, keyIndex)

	}

	return nil
