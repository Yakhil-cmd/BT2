### Title
Stale `target_shard_index` from pre-resharding epoch used to index post-resharding `outcome_roots` in `GetExecutionOutcome` — (`chain/client/src/view_client_actor.rs`)

### Summary

`ViewClientActor::handle(GetExecutionOutcome)` computes `target_shard_index` from the **outcome block's epoch layout**, then calls `get_next_block_hash_with_new_chunk` which may cross a resharding boundary and return a block `h` in a **new epoch with a different shard layout**. The `outcome_roots` collected from block `h` are indexed by the new layout, but `target_shard_index` is never recomputed. The Merkle path returned as `outcome_root_proof` is therefore for the wrong shard's outcome root.

### Finding Description

The handler computes `target_shard_index` once, from the epoch of the outcome block:

```rust
let epoch_id = *self.chain.get_block(&outcome_proof.block_hash)?.header().epoch_id();
let shard_layout = self.epoch_manager.get_shard_layout(&epoch_id)...;
let target_shard_id = account_id_to_shard_id(..., &account_id, &epoch_id)...;
let target_shard_index = shard_layout.get_shard_index(target_shard_id)...;
``` [1](#0-0) 

It then calls `get_next_block_hash_with_new_chunk`, which explicitly handles resharding by replacing the original `shard_id` with a child shard's ID when the epoch boundary is crossed:

```rust
shard_ids = shard_ids.into_iter().flat_map(|id| {
    next_shard_layout.get_children_shards_ids(id)...
}).collect();
``` [2](#0-1) 

The function returns `(h, child_shard_id)` where `h` is in the **new epoch** and `child_shard_id` is a child of the original shard. The outer code shadows `target_shard_id` with the child shard ID but **never recomputes `target_shard_index`**:

```rust
if let Some((h, target_shard_id)) = res {
    outcome_proof.block_hash = h;
    let outcome_roots = self.chain.get_block(&h)?.chunks().iter()
        .map(|header| *header.prev_outcome_root()).collect::<Vec<_>>();
    ...
    outcome_root_proof: merklize(&outcome_roots).1[target_shard_index].clone(),
``` [3](#0-2) 

`outcome_roots` is indexed by the **new epoch's shard layout** (more shards, different ordering), but `target_shard_index` is from the **old epoch's shard layout**. The Merkle path returned is for the wrong shard's outcome root.

### Impact Explanation

The `outcome_root_proof` invariant requires that `merklize(&outcome_roots).1[i]` is the Merkle path for the shard that actually executed the transaction. When resharding occurs between the outcome block and block `h`:

- Old layout (epoch E): parent shard has index `P`
- New layout (epoch E+1): child shards have indices `C1`, `C2` (different from `P`)
- `outcome_roots[P]` is the outcome root of an **unrelated shard** in the new layout
- `outcome_roots[C1]` or `outcome_roots[C2]` is the correct child shard's outcome root (containing the parent's last execution outcome root)

The returned `outcome_root_proof = merklize(&outcome_roots).1[P]` is for the wrong shard. `verify_path(block_outcome_root, outcome_root_proof, chunk_outcome_root)` returns `false`. Light clients cannot verify execution outcomes for transactions executed in the last block(s) of an epoch before resharding.

### Likelihood Explanation

Requires a resharding event (epoch boundary with shard layout change) and a transaction executed in the last block of the pre-resharding epoch. With dynamic resharding enabled, this is a realistic production scenario. The user only needs to submit a transaction and query `GetExecutionOutcome` via the public RPC after resharding — no special privileges required.

### Recommendation

After `get_next_block_hash_with_new_chunk` returns `(h, target_shard_id)`, recompute `target_shard_index` using the epoch of block `h`:

```rust
if let Some((h, target_shard_id)) = res {
    let h_epoch_id = *self.chain.get_block(&h)?.header().epoch_id();
    let h_shard_layout = self.epoch_manager.get_shard_layout(&h_epoch_id)...;
    let target_shard_index = h_shard_layout.get_shard_index(target_shard_id)...;
    ...
}
```

### Proof of Concept

1. Configure a test-loop with 2 shards and a resharding event at epoch boundary (e.g., shard 0 splits into shards 2 and 3).
2. Submit a transaction from an account on shard 0 in the **last block** of the pre-resharding epoch.
3. After resharding, query `GetExecutionOutcome` with the transaction hash and sender ID.
4. Collect the returned `outcome_root_proof` and the block's `outcome_root`.
5. Assert `verify_path(outcome_root, outcome_root_proof, chunk_outcome_root)` — it returns `false`, confirming the proof is for the wrong shard index.

The existing test `check_outcome_proofs` in `test-loop-tests/src/tests/light_client.rs` does not exercise a resharding boundary and would not catch this bug. [4](#0-3)

fn check_outcome_proofs(
    env: &mut TestLoopEnv,
    account_id: &AccountId,
    seed_hash: CryptoHash,
    tx: SignedTransaction,
) {
    let outcome = env.rpc_runner().execute_tx(tx, Duration::seconds(5)).unwrap();
    // Advance so the outcome's block is final and included in later blocks' merkle roots.
    env.rpc_runner().run_for_number_of_blocks(4);

    let mut ids = vec![TransactionOrReceiptId::Transaction {
        transaction_hash: outcome.transaction_outcome.id,
        sender_id: account_id.clone(),
    }];
    for receipt_outcome in &outcome.receipts_outcome {
        ids.push(TransactionOrReceiptId::Receipt {
            receipt_id: receipt_outcome.id,
            receiver_id: account_id.clone(),
        });
    }

    let mut rpc = env.rpc_node_mut();
    let view_client = rpc.view_client_actor();

    // The light client head and the block merkle root all proofs are checked against.
    let light_client_block = view_client
        .handle(GetNextLightClientBlock { last_block_hash: seed_hash })
        .unwrap()
        .unwrap();
    let light_client_head = light_client_block_hash(&light_client_block);
    let block_merkle_root = light_client_block.inner_lite.block_merkle_root;

    for id in ids {
        let execution_outcome = view_client.handle(GetExecutionOutcome { id }).unwrap();
        let outcome_proof = execution_outcome.outcome_proof;
        let block_proof = view_client
            .handle(GetBlockProof {
                block_hash: outcome_proof.block_hash,
                head_block_hash: light_client_head,
            })
            .unwrap();

        // outcome -> chunk outcome root -> block outcome root.
        let chunk_outcome_root =
            compute_root_from_path_and_item(&outcome_proof.proof, &outcome_proof.to_hashes());
        assert!(verify_path(
            block_proof.block_header_lite.inner_lite.outcome_root,
            &execution_outcome.outcome_root_proof,
            &chunk_outcome_root,
        ));

        // The light block header recomputes to the proof's block hash.
        assert_eq!(block_proof.block_header_lite.hash(), outcome_proof.block_hash);

        // block hash -> light client head's block merkle root.
        assert!(verify_hash(block_merkle_root, &block_proof.proof, outcome_proof.block_hash));
    }
