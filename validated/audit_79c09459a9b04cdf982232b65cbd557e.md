### Title
Unbounded loop in `calculate_rewards` over pool member balance trace can permanently freeze unclaimed yield - (File: src/pool/pool.cairo)

---

### Summary

The `calculate_rewards` internal function in the delegation pool contract iterates over every balance-change checkpoint in a pool member's `pool_member_epoch_balance` trace that has not yet been processed. The loop has no upper bound and no partial-processing mechanism. A pool member who makes many balance changes across many epochs without claiming rewards will accumulate a large trace; a subsequent `claim_rewards` call will exhaust gas and revert, permanently freezing the pool member's unclaimed yield with no recovery path.

---

### Finding Description

In `src/pool/pool.cairo`, the `calculate_rewards` internal function explicitly acknowledges the risk:

```
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
``` [1](#0-0) 

The loop processes every entry in `pool_member_epoch_balance` from the stored `entry_to_claim_from` index up to the current trace length. Each iteration also calls `find_sigma`, adding further per-iteration cost. [2](#0-1) 

Trace entries are appended by `set_member_balance` (called from both `increase_member_balance` and directly), which invokes `PoolMemberBalanceTrace.insert`. The `insert` function appends a **new** checkpoint whenever the epoch key differs from the last entry's key — i.e., every balance change in a new epoch creates a distinct, permanent entry. [3](#0-2) 

Balance changes are triggered by unprivileged pool-member actions:

- `add_to_delegation_pool` → `increase_member_balance` → `set_member_balance`
- `exit_delegation_pool_intent` → `set_member_balance` [4](#0-3) [5](#0-4) 

The `claim_rewards` function passes the stored `entry_to_claim_from` to `calculate_rewards` and only updates it on a **successful** return:

```cairo
pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
``` [6](#0-5) 

There is no partial-processing path: the entire loop must complete in a single transaction. If the transaction reverts due to gas exhaustion, `entry_to_claim_from` is never advanced, so every future `claim_rewards` attempt faces the same (or larger) loop and will also revert.

---

### Impact Explanation

A pool member whose `pool_member_epoch_balance` trace has grown large enough to exhaust gas in `calculate_rewards` will have their unclaimed STRK rewards permanently frozen. The funds are held in the pool contract and cannot be transferred to the reward address. Because there is no partial-claim or trace-compaction mechanism, the state is unrecoverable without a contract upgrade.

**Impact class**: Permanent freezing of unclaimed yield — **High**.

---

### Likelihood Explanation

Any pool member who actively adjusts their delegation (increases or partially exits) across many epochs without claiming rewards will grow their trace. The number of loop iterations equals the number of distinct-epoch balance changes since the last successful claim. On Starknet, the per-transaction gas ceiling is finite; the codebase itself flags this with the comment "This loop is unbounded but unlikely to exceed gas limits," confirming the developers are aware the bound is not enforced. A pool member who makes one balance change per epoch and claims rewards infrequently (e.g., once a year at ~365 epochs) will accumulate hundreds of entries, each requiring a storage read and arithmetic in `find_sigma`.

**Likelihood**: Medium — normal active-delegation behaviour over a long unclaimed period is sufficient.

---

### Recommendation

1. **Partial-claim mechanism**: Allow `claim_rewards` to accept a `max_entries` parameter, advance `entry_to_claim_from` by that many steps, accumulate partial rewards into `pool_member_info._unclaimed_rewards_from_v0` (already exists for legacy rewards), and return without requiring full trace traversal.
2. **Claim-on-balance-change**: Force a reward settlement (updating `entry_to_claim_from` and accumulating rewards) every time `set_member_balance` is called, so the loop in `calculate_rewards` never needs to process more than a small constant number of entries.
3. **Trace compaction**: After a successful `claim_rewards`, truncate processed entries from the front of the trace.

---

### Proof of Concept

1. Pool member enters a delegation pool (creates first trace entry at epoch `E+K`).
2. Pool member calls `add_to_delegation_pool` once per epoch for `N` epochs, each in a distinct epoch, creating `N` additional checkpoints in `pool_member_epoch_balance`.
3. Pool member never calls `claim_rewards` during this period.
4. After `N` epochs, pool member calls `claim_rewards`.
5. `calculate_rewards` enters the `while` loop with `pool_member_trace_length = N+1` and `entry_to_claim_from = 0`.
6. Each iteration reads a storage slot (`pool_member_trace.at(entry_to_claim_from)`) and calls `find_sigma` (another storage read + arithmetic).
7. For sufficiently large `N`, the cumulative gas cost exceeds the Starknet transaction gas limit; the call reverts.
8. Because the revert rolls back `pool_member_info.entry_to_claim_from`, every subsequent `claim_rewards` call faces the same loop and also reverts — the pool member's accumulated yield is permanently frozen. [7](#0-6) [8](#0-7)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
