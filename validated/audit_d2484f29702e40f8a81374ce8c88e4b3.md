### Title
Unprivileged Caller Can Permanently Suppress Block Reward Distribution via `disable_rewards` Parameter - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable with no access control. Its `disable_rewards: bool` parameter is fully attacker-controlled. When called with `disable_rewards: true`, the function advances the global `last_reward_block` checkpoint without distributing any rewards, permanently destroying the block's yield for all stakers. An attacker can front-run this call every block to permanently freeze all consensus-phase reward distribution.

### Finding Description
`IStakingRewardsManager::update_rewards` is exposed as a public entry point with no role check beyond `general_prerequisites` (which only asserts the contract is unpaused and the caller is non-zero). [1](#0-0) 

The function unconditionally writes the current block number to the global `last_reward_block` storage slot before inspecting `disable_rewards`: [2](#0-1) 

Because `last_reward_block` is a single global value shared across all stakers, one call per block is sufficient to mark the entire block as "already processed": [3](#0-2) 

When `disable_rewards` is `true` the function returns immediately after writing `last_reward_block`, skipping the entire `_update_rewards` path that credits stakers and pools: [4](#0-3) 

The rewards for that block are never minted or credited; they are permanently lost. The `general_prerequisites` guard provides no protection: [5](#0-4) 

The interface definition confirms no caller restriction is documented or enforced: [6](#0-5) 

### Impact Explanation
In the consensus rewards phase (after `consensus_rewards_first_epoch`), `update_rewards` is the sole mechanism for distributing per-block STRK rewards to stakers and their delegation pools. An attacker who calls `update_rewards(any_staker, true)` once per block permanently destroys that block's yield for every staker in the protocol. Repeated across all blocks, this constitutes **permanent freezing of unclaimed yield** for the entire protocol. The rewards are never minted to the `RewardSupplier` and never credited to `unclaimed_rewards_own` or pool balances — the loss is irreversible.

This matches the allowed High impact: *Permanent freezing of unclaimed yield or unclaimed royalties*.

### Likelihood Explanation
The attack requires no stake, no special role, and no capital — only gas. The attacker simply calls `update_rewards(any_valid_staker_address, true)` once per block. On Starknet, front-running is straightforward because the mempool is observable. The attacker can automate this with a simple bot. There is no economic barrier and no on-chain defense.

### Recommendation
Restrict who may supply `disable_rewards: true`. Either:
1. Add an access-control check (e.g., `only_app_governor` or a dedicated consensus-layer role) before honouring `disable_rewards: true`, or
2. Remove the `disable_rewards` parameter from the public interface entirely and handle the "skip rewards" logic through a separate privileged function.

The simplest safe fix is to assert that `disable_rewards` can only be `true` when the caller holds a trusted role:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();
    if disable_rewards {
        self.roles.only_app_governor(); // or equivalent trusted role
    }
    // ... rest of function
}
```

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. Attacker (any EOA) monitors the chain and, at the start of each new block, submits:
   ```
   staking_contract.update_rewards(any_active_staker_address, disable_rewards=true)
   ```
3. `last_reward_block` is set to the current block number.
4. Any subsequent legitimate call to `update_rewards(..., false)` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. No rewards are distributed for that block; the yield is permanently destroyed.
6. Repeating this every block causes total, permanent suppression of all consensus-phase reward distribution across the entire protocol. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
