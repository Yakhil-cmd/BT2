### Title
Missing Authorization Check and No-Op Implementation in `MsgTurnBridge` Handler - (File: `x/cronos/keeper/msg_server.go`)

### Summary
The `TurnBridge` gRPC message handler is a stub that returns `nil, nil` unconditionally. It performs no authorization check and makes no state change. Any unprivileged address can submit `MsgTurnBridge` and receive a success response, completely bypassing the `CanTurnBridge` permission system. Simultaneously, the bridge can never actually be disabled via this message, permanently disabling the emergency circuit-breaker.

### Finding Description

The Cronos module defines a permission system with two bits: `CanChangeTokenMapping` (1) and `CanTurnBridge` (2), documented in ADR-009 as the mechanism for "disabling the bridge module in case of emergency." [1](#0-0) 

`UpdateTokenMapping` correctly enforces this system: [2](#0-1) 

However, the `TurnBridge` handler — the other message that ADR-009 explicitly states requires `CanTurnBridge` — is a complete stub:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [3](#0-2) 

The proto definition, CLI command, and permission infrastructure are all fully wired: [4](#0-3) [5](#0-4) 

The integration test explicitly asserts that an unauthorized sender must be rejected (`rsp["code"] != 0`), and that after a successful `turn_bridge("false")` the bridge stops processing — neither of which can hold with the current stub: [6](#0-5) 

### Impact Explanation

Two simultaneous impacts:

1. **Auth bypass (High):** The `CanTurnBridge` permission check is entirely absent. Any unprivileged address can submit `MsgTurnBridge` and receive a success response (code 0). The entire permission infrastructure for this message is dead code.

2. **Permanent inability to disable the bridge (High):** Because the handler makes no state change, the Gravity bridge emergency circuit-breaker is permanently non-functional. If the bridge is exploited and operators attempt an emergency shutdown via `MsgTurnBridge`, the call silently succeeds but the bridge continues operating, allowing continued asset loss.

### Likelihood Explanation

The entry path is fully public: any address with enough gas can submit `MsgTurnBridge` via the standard Cosmos SDK transaction flow. No special privileges, leaked keys, or cryptographic breaks are required. The bypass is unconditional — the stub returns success for every caller.

### Recommendation

Implement `TurnBridge` with the same pattern as `UpdateTokenMapping`:

1. Check `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `sdkerrors.ErrUnauthorized` on failure.
2. Persist the bridge enabled/disabled state (e.g., a `BridgeEnabled` field in `Params` or a dedicated KV key).
3. Enforce the stored state in the Gravity bridge EVM hook that processes `__CronosSendToEvmChain` events.

### Proof of Concept

```
# Any unprivileged address submits MsgTurnBridge
cronosd tx cronos turn-bridge false --from attacker --gas-prices ...

# Result: code=0 (success), no permission check performed
# Bridge state: unchanged (still enabled)
# CanTurnBridge permission system: completely bypassed
```

The handler at `x/cronos/keeper/msg_server.go:85-87` returns `nil, nil` before any authorization or state logic executes. The `CanTurnBridge` bit defined in `x/cronos/keeper/permissions.go:15` is never consulted for this message. [3](#0-2) [7](#0-6)

// UpdateTokenMapping implements the grpc method
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)

	// check permission
	if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}

	// msg is already validated
	if err := k.RegisterOrUpdateTokenMapping(ctx, msg); err != nil {
		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}

			if err != nil {
				return err
			}

				return err
			}
			return tx.GenerateOrBroadcastTxCLI(clientCtx, cmd.Flags(), msg)
		},
	}

	return cmd
}
