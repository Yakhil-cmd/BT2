### Title
Pool Member Record Not Deleted After Full Exit, Permanently Blocking Re-Entry - (File: src/pool/pool.cairo)

### Summary
`exit_delegation_pool_action` in `pool.cairo` zeroes out the exiting pool member's `unpool_amount` and `unpool_time` but never deletes the `pool_member_info` storage record (never writes `VInternalPoolMemberInfo::None`). Because `enter_delegation_pool` gates entry with a strict `is_none()` check, any pool member who fully exits is permanently blocked from re-entering the same pool directly.

### Finding Description
When a pool member calls `exit_delegation_pool_intent(amount: total_balance)` followed by `exit_delegation_pool_action`, the function transfers the funds back and updates the record:

```cairo
// src/pool/pool.cairo lines 321-326
let unpool_amount = pool_member_info.unpool_amount;
pool_member_info.unpool_amount = Zero::zero();
pool_member_info.unpool_time = Option::None;
// Write the updated pool member info to storage.
self.write_pool_member_info(:pool_member, :pool_member_info);
```

`write_pool_member_info` always writes `VInternalPoolMemberInfo::V1(...)` — never `VInternalPoolMemberInfo::None`. [1](#0-0) 

After a full exit the record therefore persists with `amount = 0`, `unpool_amount = 0`, `unpool_time = None`. When the same address later calls `enter_delegation_pool`, the guard:

```cairo
// src/pool/pool.cairo lines 188-190
assert!(
    self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
);
```

reads `VInternalPoolMemberInfo::V1(...)` (not `None`) and reverts with `POOL_MEMBER_EXISTS`. [2](#0-1) 

By contrast, `enter_delegation_pool_from_staking_contract` (the pool-switch path) handles the `Option::Some` case and can update an existing record, so re-entry via a cross-pool switch is still possible — but direct re-entry is permanently blocked. [3](#0-2) 

The staking contract's `remove_staker` correctly writes `VInternalStakerInfo::None` on full staker exit, showing the intended pattern is to null out the record. [4](#0-3) 

### Impact Explanation
Any pool member who fully exits a delegation pool is permanently unable to re-enter that same pool through the normal `enter_delegation_pool` path. Their address is effectively tombstoned in the pool contract. This constitutes permanent damage to the user's ability to participate in the protocol — a medium-severity griefing impact with no profit motive.

### Likelihood Explanation
The scenario is straightforward and requires no special privileges: any unprivileged delegator who calls `exit_delegation_pool_intent` with their full balance and then `exit_delegation_pool_action` triggers the bug. Re-entry to the same pool is a natural user action (e.g., after a staker improves their commission or after temporarily withdrawing funds).

### Recommendation
In `exit_delegation_pool_action`, after transferring funds, check whether the pool member's remaining balance and unclaimed rewards are both zero. If so, write `VInternalPoolMemberInfo::None` to fully delete the record:

```cairo
if pool_member_info._unclaimed_rewards_from_v0.is_zero()
    && self.get_last_member_balance(:pool_member).is_zero() {
    self.pool_member_info.write(pool_member, VInternalPoolMemberInfo::None);
} else {
    self.write_pool_member_info(:pool_member, :pool_member_info);
}
```

Alternatively, relax the `enter_delegation_pool` guard to allow re-entry when the existing record has zero balance and no pending exit intent.

### Proof of Concept
1. Pool member calls `enter_delegation_pool(reward_address, amount: X)` — record created.
2. Pool member calls `exit_delegation_pool_intent(amount: X)` — full balance queued for exit.
3. Wait for `exit_wait_window`.
4. Pool member calls `exit_delegation_pool_action(pool_member)` — X tokens returned; record updated to `{amount:0, unpool_amount:0, unpool_time:None}` but **not** deleted. [5](#0-4) 
5. Pool member calls `enter_delegation_pool(reward_address, amount: Y)` — **reverts** with `POOL_MEMBER_EXISTS` because `pool_member_info.read(pool_member)` is `V1(...)`, not `None`. [6](#0-5)

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
