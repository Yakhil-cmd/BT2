### Title
Unprivileged Caller Can Permanently Freeze All Staker Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control beyond a non-zero caller check. It accepts a `disable_rewards: bool` parameter that, when `true`, causes the function to update the global `last_reward_block` sentinel **without distributing any rewards**. Because `last_reward_block` is written before the `disable_rewards` branch is evaluated, any unprivileged caller can consume the single per-block reward slot every block, permanently preventing rewards from reaching stakers and pool members.

---

### Finding Description

`update_rewards` is exposed as a public ABI function under `IStakingRewardsManager`:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();          // only: not paused, caller != 0
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // ← last_reward_block is written HERE, before the disable check
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;                            // ← exits with no rewards distributed
        }
        ...
    }
``` [1](#0-0) 

`general_prerequisites` enforces only two conditions: the contract is not paused and the caller is not the zero address. [2](#0-1) 

`last_reward_block` is a single global storage slot. Once it is written for a given block number, the guard `current_block_number > self.last_reward_block.read()` causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The attacker's path is:

1. Pick any currently-active staker address (readable from the public `stakers` vector or events).
2. Call `update_rewards(valid_staker, disable_rewards: true)` once per block.
3. `last_reward_block` advances to the current block; no rewards are minted or distributed.
4. Any legitimate call to `update_rewards` for that block reverts.
5. Block rewards for that block are permanently lost — they are never credited to `unclaimed_rewards_own` nor forwarded to pool contracts.

The analog to the external report is direct: just as a hat admin could use a phantom intermediate hat-ID to bypass the per-level child cap (a missing validation on a structural parameter), here a caller can supply `disable_rewards: true` — an unauthorized parameter value — to bypass the reward-distribution step that is supposed to accompany every `last_reward_block` advancement. In both cases the protocol advances a global counter (child count / last reward block) without performing the work that counter is meant to gate.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

If the attacker submits this call once per block (trivially automatable), no staker ever accumulates `unclaimed_rewards_own` and no pool contract ever receives a `update_rewards_from_staking_contract` call. All yield for the entire protocol is frozen indefinitely. The attacker bears only gas cost; there is no profit motive required. [4](#0-3) 

---

### Likelihood Explanation

**High.** The entry point is a public ABI function with no role check. The only preconditions are: (a) the contract is not paused, (b) the caller address is non-zero, and (c) a valid active staker address is supplied. All three are trivially satisfiable by any external account. The attack requires no privileged access, no token balance, and no prior relationship with the protocol.

---

### Recommendation

1. **Restrict `disable_rewards: true` to a privileged caller.** If the flag is intended for use by the attestation contract or a governance role, add an explicit check:
   ```cairo
   if disable_rewards {
       self.assert_caller_is_attestation_contract(); // or a dedicated role
   }
   ```
2. **Alternatively**, move `last_reward_block.write` to after the `disable_rewards` branch so that a no-op call does not consume the block's reward slot.
3. **Or**, remove the `disable_rewards` parameter from the public interface entirely and handle the "skip rewards" logic internally based on on-chain state (e.g., `is_pre_consensus()`).

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
