### Title
Unprivileged Caller Can Permanently Freeze All Staker Block Rewards via `disable_rewards` Parameter - (File: src/staking/staking.cairo)

### Summary
`update_rewards` is documented as "Only starkware sequencer" but has no on-chain access control. Any unprivileged caller can invoke it with a caller-controlled `disable_rewards: true` flag, which consumes the global `last_reward_block` slot for the current block without distributing any rewards. Because `last_reward_block` is a single contract-wide value, one such call per block permanently prevents all stakers from receiving consensus block rewards.

### Finding Description
The spec explicitly states the access control for `update_rewards` is "Only starkware sequencer": [1](#0-0) 

However, the implementation only calls `general_prerequisites()`, which checks for pause state and a non-zero caller — no sequencer identity check: [2](#0-1) 

The function accepts two caller-controlled parameters: `staker_address` and `disable_rewards`. The critical path is: [3](#0-2) 

`last_reward_block` is written unconditionally **before** the `disable_rewards` guard. If `disable_rewards` is `true`, the function returns immediately after writing the block number, distributing nothing. Because `last_reward_block` is a single global storage slot (not per-staker), any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

The global nature of `last_reward_block` is confirmed by its storage declaration: [5](#0-4) 

### Impact Explanation
An attacker who calls `update_rewards(any_valid_active_staker, disable_rewards: true)` once per block:
- Consumes the single per-block reward slot for the entire contract.
- Prevents the sequencer from distributing STRK block rewards to **all** stakers and their delegation pools for that block.
- Repeated every block, this permanently freezes all unclaimed yield accumulation for every staker in the protocol.

This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The entry path requires only a non-zero, non-paused caller and a valid (active, non-zero-balance) staker address — both trivially satisfiable by any on-chain observer. Staker addresses are public via the `stakers` vector and `NewStaker` events. The attack costs only gas and can be automated to fire at the start of every block.

### Recommendation
Add an explicit sequencer-only guard to `update_rewards`, consistent with the spec. For example, introduce a stored `sequencer_address` and assert `get_caller_address() == self.sequencer_address.read()` at the top of the function, analogous to how `update_rewards_from_attestation_contract` enforces `assert_caller_is_attestation_contract()`: [6](#0-5) 

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. A valid staker `S` exists with non-zero balance at the current epoch.
3. Attacker (any address) calls `update_rewards(staker_address: S, disable_rewards: true)` at block `B`.
4. `last_reward_block` is set to `B`; no rewards are distributed.
5. The legitimate sequencer attempts `update_rewards(staker_address: S, disable_rewards: false)` at block `B` — it reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker `S` (and all other stakers, since `last_reward_block` is global) receives zero block rewards for block `B`.
7. Repeating step 3 at every block permanently freezes all staker yield. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
