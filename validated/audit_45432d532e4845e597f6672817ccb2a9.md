### Title
Missing Caller Authorization in `update_rewards` Allows Any Address to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract has no caller authorization check. The specification explicitly states "Only starkware sequencer" may call it, but the implementation is callable by any address. An attacker can call it with `disable_rewards: true` every block, consuming the global `last_reward_block` slot without distributing rewards, permanently blocking all stakers from receiving block rewards.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` at `src/staking/staking.cairo` lines 1449–1507 is the V3 consensus-rewards distribution entry point. The spec at `docs/spec.md` line 1645 states its access control is **"Only starkware sequencer."**

The implementation, however, contains **no `get_caller_address()` check** against any stored sequencer address:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ← NO caller check here
    ...
    self.last_reward_block.write(current_block_number);   // global slot consumed
    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits without distributing
    }
    ...
}
``` [1](#0-0) 

The critical detail is that `last_reward_block` is a **single global storage variable** (not per-staker). Once it is written to the current block number, no further call to `update_rewards` for **any** staker in that block can succeed — all will revert with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

The spec confirms the intended restriction: [3](#0-2) 

---

### Impact Explanation

An attacker calls `update_rewards(any_active_staker, disable_rewards: true)` at the start of every block (before the sequencer). This:

1. Passes all validation (staker exists, is active, has non-zero balance after K epochs).
2. Writes `last_reward_block = current_block_number`.
3. Returns immediately due to `disable_rewards == true` — **zero rewards distributed**.
4. The sequencer's subsequent call for any staker in that block reverts with `REWARDS_ALREADY_UPDATED`.

All stakers and their delegators permanently lose block rewards for every attacked block. This maps directly to **"Permanent freezing of unclaimed yield"** (High severity).

---

### Likelihood Explanation

- **Any unprivileged address** can call `update_rewards` — no token balance, no staker registration, no role required.
- The attacker only needs to submit a transaction before the sequencer's `update_rewards` transaction in each block.
- On Starknet, the sequencer itself orders transactions, but the function is a public entry point callable by any L2 account.
- The attack is cheap (one transaction per block, no funds at risk for the attacker) and repeatable indefinitely.

---

### Recommendation

Add a caller authorization check at the top of `update_rewards`, verifying the caller is the registered sequencer address:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
}
```

A `sequencer_address` storage slot and corresponding setter (restricted to governance) should be added to the contract.

---

### Proof of Concept

```
1. Deploy staking contract; staker stakes STRK.
2. Advance K epochs so staker has effective balance.
3. Enable consensus rewards (set_consensus_rewards_first_epoch).
4. Advance K more epochs.
5. Attacker (any address) calls:
       update_rewards(staker_address, disable_rewards: true)
   → succeeds; last_reward_block = current_block.
6. Sequencer calls:
       update_rewards(staker_address, disable_rewards: false)
   → reverts: "Rewards already updated for this block."
7. Staker's unclaimed_rewards_own is unchanged — block rewards permanently lost.
8. Repeat step 5 every block to permanently freeze all yield.
```

The test `test_update_rewards_assertions_before_consensus` (and `_already_consensus`) in `src/staking/tests/test.cairo` calls `update_rewards` without any caller restriction, confirming the function is publicly accessible. [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
