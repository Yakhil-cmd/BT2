### Title
ETH Permanently Locked in `RewardSupplier.tick()` When No Minting Is Required - (File: `L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

`RewardSupplier.tick()` is declared `external payable` and uses `msg.value` to pay bridge and messaging fees — but only inside the `if (amountToMint > 0)` branch. When called with ETH while no minting is pending, the function returns without consuming `msg.value`. Because the contract has no `receive()`, `fallback()`, or ETH-withdrawal function, any ETH sent in that path is permanently locked.

---

### Finding Description

`tick()` is the public entry point that triggers L1→L2 reward minting:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

    if (amountToMint > 0) {
        // ...
        uint256 msgFee = msg.value / 2;
        bridge().depositWithMessage{value: msgFee}(...);
        msgFee = msg.value - msgFee;
        messagingContract().sendMessageToL2{value: msgFee}(...);
    }
    // ← if amountToMint == 0, returns here; msg.value is never used
}
``` [1](#0-0) 

`msg.value` is only forwarded to the bridge and messaging contract when `amountToMint > 0`. When `requiredMinting()` returns `(0, 0)` — i.e., there are no pending L2 mint-request messages — the function exits immediately, leaving any attached ETH stranded.

The contract inherits from `RewardSupplierStorage`, `Identity`, and `ProxySupportImpl` and defines no `receive()`, `fallback()`, or ETH-recovery function. [2](#0-1) 

There is no access control on `tick()` — it is callable by any address.

---

### Impact Explanation

Any ETH sent to `tick()` when `amountToMint == 0` is permanently irrecoverable. This constitutes **permanent freezing of funds** (High impact per the allowed scope). The locked ETH is the caller's own funds; however, because `tick()` is the protocol's reward-distribution trigger and is expected to be called with ETH to cover bridge fees, repeated accidental or adversarial calls with ETH during idle periods drain the caller's balance and permanently reduce the ETH available to the protocol's operational flow.

---

### Likelihood Explanation

`tick()` has no access control and is `external payable`. Any public caller — including automated bots, MEV searchers, or a griefing actor — can invoke it with ETH at any time. The condition `amountToMint == 0` is the normal state between reward epochs when no L2 mint requests are pending, making this a frequently reachable path.

---

### Recommendation

Add a guard at the top of `tick()` to revert if ETH is attached but will not be used:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();
    if (amountToMint == 0) {
        require(msg.value == 0, "NO_MINT_REQUIRED_ETH_REJECTED");
        return;
    }
    // ... existing logic
}
```

Alternatively, refund unused ETH at the end of the function, or restrict `tick()` to a trusted caller role.

---

### Proof of Concept

1. Observe that no L2 mint-request messages are pending (`requiredMinting()` returns `(0, 0)`).
2. Call `tick{value: 1 ether}()` from any EOA.
3. The `if (amountToMint > 0)` branch is skipped; `msg.value` is never forwarded.
4. The contract holds 1 ETH with no mechanism to recover it.
5. Repeat to accumulate permanently locked ETH. [1](#0-0)

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
