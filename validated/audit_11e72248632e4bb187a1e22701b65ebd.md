### Title
Commission Increase Before Attestation Retroactively Steals Delegator Epoch Rewards - (File: src/staking/staking.cairo)

---

### Summary

`set_commission` updates the staker's commission rate immediately in storage without first settling pending rewards for the current epoch. Because `update_rewards_from_attestation_contract` (V2 / pre-consensus) calculates the **entire epoch's** pool rewards using the commission value at the moment of attestation, a staker who holds an active `CommissionCommitment` can increase their commission to `max_commission` immediately before attesting, causing all of that epoch's delegator rewards to be split at the new (higher) rate — retroactively stealing yield that delegators earned under the old rate.

---

### Finding Description

`set_commission` writes the new commission directly to storage with no prerequisite reward settlement:

```
fn set_commission(ref self: ContractState, commission: Commission) {
    ...
    let staker_pool_info = self.staker_pool_info.entry(staker_address);
    if let Option::Some(old_commission) = staker_pool_info.commission.read() {
        self.update_commission(:staker_address, :staker_pool_info, :old_commission, :commission);
    } else { ... }
}
``` [1](#0-0) 

`update_commission` (the internal helper) only validates the new value against the commitment and writes it; it never calls any reward-settlement path:

```
fn update_commission(...) {
    ...
    // Update commission in storage.
    staker_pool_info.commission.write(Option::Some(commission));
    ...
}
``` [2](#0-1) 

When the staker later attests, `update_rewards_from_attestation_contract` calls `_update_rewards`, which calls `calculate_staker_pools_rewards`. That function reads the **current** commission from storage and applies it to the **entire epoch's** pool rewards:

```
let commission = staker_pool_info.commission();
for (pool_contract, token_address) in staker_pool_info.pools {
    ...
    let (commission_rewards, pool_rewards) = split_rewards_with_commission(
        rewards_including_commission: pool_rewards_including_commission, :commission,
    );
``` [3](#0-2) 

There is no per-epoch commission snapshot; the single stored value is used for the whole epoch's calculation.

`set_commission_commitment` allows a staker to set a `max_commission` up to `COMMISSION_DENOMINATOR` (10 000 = 100 %). While the commitment is active, `update_commission` permits raising the commission to any value ≤ `max_commission`:

```
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
}
``` [4](#0-3) 

---

### Impact Explanation

A staker with an active commitment (e.g., `max_commission = 10 000`) can:

1. Let delegators accumulate stake for an entire epoch at commission 10 %.
2. Immediately before the attestation window, call `set_commission(10000)`.
3. Attest — `update_rewards_from_attestation_contract` calculates the epoch's pool rewards at 100 % commission.
4. Delegators receive **zero** pool rewards for the epoch; the staker captures all of it as commission.

This is **theft of unclaimed yield** (High severity per the allowed impact scope). Delegators who delegated expecting 10 % commission lose all rewards for that epoch with no recourse.

---

### Likelihood Explanation

- Any staker can call `set_commission_commitment` to obtain a commitment with `max_commission` up to 100 %. [5](#0-4) 
- The staker controls the timing of their attestation within the attestation window, so they can sequence the commission increase immediately before attesting.
- The attack requires no privileged access, no external dependency, and no leaked keys — only a staker address with an active pool and a commitment.
- Delegators cannot prevent this once they have delegated; they have no mechanism to exit within the same epoch.

---

### Recommendation

Before updating the commission value, settle any pending rewards for the current epoch. Concretely, `set_commission` (and its internal `update_commission` helper) should call the reward-update path (equivalent to `update_rewards_from_attestation_contract` / `update_rewards`) for the staker's pools before writing the new commission to storage. This ensures rewards accrued under the old commission rate are distributed correctly before the new rate takes effect — directly analogous to calling `keeperManagementFeeClaim()` before updating fee params in the referenced report.

---

### Proof of Concept

**Pre-conditions (V2 / pre-consensus mode):**

1. Staker stakes with commission = 1 000 (10 %).
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current + 50)`.
3. Delegator delegates a large amount; K epochs pass so the balance becomes effective.
4. One full epoch elapses — delegator has earned ~90 % of pool rewards at 10 % commission.

**Attack (within the attestation window of epoch E):**

```
// Step 1 — raise commission to 100% (allowed by active commitment)
staking.set_commission(commission: 10000);

// Step 2 — attest; _update_rewards now uses commission = 10000
attestation_contract.attest(staker_operational_address);
// → calculate_staker_pools_rewards reads commission = 10000
// → split_rewards_with_commission gives 100% to staker, 0% to pool
// → pool.update_rewards_from_staking_contract(rewards: 0, ...)
```

**Result:** Delegator's `claim_rewards` returns 0 for epoch E. The staker's `unclaimed_rewards_own` contains the full epoch reward including what should have been the delegator's share. All delegator yield for epoch E is permanently redirected to the staker.

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }
