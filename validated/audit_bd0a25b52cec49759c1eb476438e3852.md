Audit Report

## Title
`is_funded()` Omits `staked_maturity_e8s_equivalent`, Causing Dissolved Neurons With Only Staked Maturity to Be Permanently Skipped by the Unstake-Maturity Timer Task — (`rs/nns/governance/src/neuron/types.rs`)

## Summary

`Neuron::is_funded()` does not include `staked_maturity_e8s_equivalent` in its funding calculation. A neuron that holds only staked maturity (zero ICP stake, zero regular maturity) is therefore classified as "not funded," and once dissolved for more than two weeks it satisfies every condition of `is_inactive()`. The periodic timer task that converts staked maturity to spendable maturity iterates exclusively over active neurons, so such a neuron is permanently skipped and the user's staked maturity becomes inaccessible. This directly contradicts the invariant encoded in the existing proptest.

## Finding Description

`Neuron::is_funded()` computes only:

```rust
let amount_e8s = self.stake_e8s().saturating_add(self.maturity_e8s_equivalent);
amount_e8s > 0
``` [1](#0-0) 

`staked_maturity_e8s_equivalent` is absent. `is_inactive()` returns `true` when `is_funded()` is `false` and the neuron dissolved more than two weeks ago: [2](#0-1) 

`with_active_neurons_iter_sections()` filters the full neuron set to those where `!n.is_inactive(now)`: [3](#0-2) 

`list_neurons_ready_to_unstake_maturity()` — the only discovery path for the timer task — calls that filtered iterator: [4](#0-3) 

`ready_to_unstake_maturity()` correctly identifies dissolved neurons with staked maturity > 0, but is never reached for the affected neurons: [5](#0-4) 

The existing proptest explicitly encodes the intended invariant — `staked_maturity_e8s_equivalent` must count toward funding — and would fail for the input `staked_maturity_e8s_equivalent=1, cached_neuron_stake_e8s=0, maturity_e8s_equivalent=0`: [6](#0-5) 

**Exploit path:**
1. User creates a neuron, stakes maturity via `StakeMaturity`.
2. User starts dissolving; dissolve delay > 2 weeks.
3. Neuron reaches `Dissolved` state.
4. User calls `Disburse` to withdraw ICP stake; `cached_neuron_stake_e8s` becomes 0.
5. More than two weeks elapse after dissolution.
6. State: `cached_neuron_stake_e8s = 0`, `maturity_e8s_equivalent = 0`, `staked_maturity_e8s_equivalent > 0`, dissolved > 2 weeks ago.
7. `is_funded()` → `false`; `is_inactive()` → `true`.
8. Every subsequent run of `UnstakeMaturityOfDissolvedNeuronsTask` skips this neuron.
9. `staked_maturity_e8s_equivalent` is never moved to `maturity_e8s_equivalent`; the user cannot spawn or disburse it.

## Impact Explanation

The user's staked maturity is permanently inaccessible: it cannot be spawned, disbursed, or otherwise converted to spendable ICP. This constitutes a permanent loss of governance-controlled funds for affected users. The impact maps to the **High** bounty tier: significant NNS governance security impact with concrete user fund harm. Recovery requires non-obvious workarounds (re-staking ICP or re-locking the neuron) that are undocumented and impose additional costs on the user.

## Likelihood Explanation

The scenario is reachable by any unprivileged governance participant following a natural usage pattern: stake maturity for voting power, dissolve, disburse ICP, expect maturity to be available. No special privileges, social engineering, or external dependencies are required. The proptest mismatch confirms the implementation diverges from the intended invariant, and the condition is a stable trap — once entered, no automatic recovery occurs.

## Recommendation

Include `staked_maturity_e8s_equivalent` in `is_funded()`:

```rust
pub fn is_funded(&self) -> bool {
    let amount_e8s = self
        .stake_e8s()
        .saturating_add(self.staked_maturity_e8s_equivalent.unwrap_or(0))
        .saturating_add(self.maturity_e8s_equivalent);
    amount_e8s > 0
}
```

This aligns the implementation with the proptest invariant at [7](#0-6) 
and ensures any neuron holding staked maturity is never classified as inactive.

## Proof of Concept

The existing proptest `test_neuron_is_inactive_based_on_funding` already encodes the invariant and will fail deterministically for the input `(cached_neuron_stake_e8s=0, staked_maturity_e8s_equivalent=1, neuron_fees_e8s=0, maturity_e8s_equivalent=0)`. Running `cargo test -p ic-governance test_neuron_is_inactive_based_on_funding` with proptest seeded to that input reproduces the failure. The integration-level PoC in the submission (`test_staked_maturity_not_locked_after_stake_disburse`) directly exercises `unstake_maturity_of_dissolved_neurons` and asserts the staked maturity is moved — it will fail against the current implementation.

    pub fn is_inactive(&self, now: u64) -> bool {
        // Require condition 1.
        if self.is_seed_neuron() || self.is_ect_neuron() {
            return false;
        }

        // Require condition 2.
        if self.is_funded() {
            return false;
        }

        // Require condition 3.

        // 3.1: Interpret dissolve_state field.
        let dissolved_at_timestamp_seconds = match self.dissolved_at_timestamp_seconds() {
            // None -> not dissolving -> will be dissolved in the future -> not dissolved now ->
            // certainly was not dissolved sufficiently "long" ago!
            None => {
                return false;
            }
            Some(ok) => ok,
        };

        // 3.2: Now, we know when self is "dissolved" (could be in the past, present, or future).
        // Thus, we can evaluate whether that happened sufficiently long ago.
        let max_dissolved_at_timestamp_seconds_to_be_inactive =
            now.saturating_sub(2 * 7 * ONE_DAY_SECONDS);
        if dissolved_at_timestamp_seconds > max_dissolved_at_timestamp_seconds_to_be_inactive {
            return false;
        }

        // Finally, require condition 4: Member of the Neuron's Fund.
        if self.is_a_neurons_fund_member() {
            return false;
        }

        // All requirements have been met.
        true
    }
