### Title
Synthetic Ratio Truncates to Zero for Low-Value Base Tokens, Permanently Bricking Pool Swaps — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider._getBidAndAskPrice()` computes a synthetic cross-rate as `mid = Math.mulDiv(mid1, ORACLE_DECIMALS, mid2)` (i.e., `mid1 * 1e8 / mid2`). When the base-feed price is small relative to the quote-feed price, integer division truncates `mid` to zero. The downstream `_computeBidAsk(0, …)` then returns the fail-closed sentinel `(0, type(uint128).max)`, causing every call to `getBidAndAskPrice()` to revert with `FeedStalled()`. Because `quoteFeedId` is immutable, the pool is permanently unable to execute swaps.

---

### Finding Description

In `_getBidAndAskPrice()`, the two-feed synthetic path is:

```solidity
// AnchoredPriceProvider.sol lines 263-268
if (_quote != bytes32(0)) {
    (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
    if (!ok2 || mid2 == 0) return (0, type(uint128).max);
    // Synthetic ratio (8-decimal): mid1 / mid2.
    mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // ← truncates to 0
    spreadBps += spreadBps2;
}
return _computeBidAsk(mid, spreadBps);
``` [1](#0-0) 

`ORACLE_DECIMALS = 1e8`. The result is `mid1 * 1e8 / mid2`. When `mid1 * 1e8 < mid2` (i.e., `base_price_USD < quote_price_USD / 1e8`), the division truncates to zero. No post-division zero-check exists before `_computeBidAsk` is called.

Inside `_computeBidAsk`, a zero `mid` propagates through `_bandEdge`:

```solidity
// lines 309-312
uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
    return (0, type(uint128).max);
}
``` [2](#0-1) 

`_bandEdge(0, …)` calls `Math.mulDiv(0, …) = 0`, so `refBid == 0` and the sentinel `(0, type(uint128).max)` is returned. The public entry point then reverts:

```solidity
// line 216
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [3](#0-2) 

Because `quoteFeedId` is an immutable set at construction, the provider cannot be patched in place:

```solidity
// line 141
quoteFeedId = _quoteFeedId;
``` [4](#0-3) 

---

### Impact Explanation

Every pool swap calls `getBidAndAskPrice()` on its price provider. With the sentinel returned on every call, all swaps revert permanently. LPs are stranded in a pool that cannot trade; they can only remove liquidity proportionally, but the pool's core function (price-discovery and swap settlement) is completely broken. This matches the allowed impact: **broken core pool functionality causing unusable swap flows**.

---

### Likelihood Explanation

The condition `base_price_USD < quote_price_USD / 1e8` is realistic for any low-value token paired synthetically against a high-value token:

| Base feed | Quote feed | base_price | quote_price / 1e8 | Truncates? |
|---|---|---|---|---|
| SHIB/USD | BTC/USD | $0.00001 | $0.0005 | **Yes** |
| PEPE/USD | ETH/USD | $0.000001 | $0.00002 | **Yes** |
| ETH/USD | BTC/USD | $2000 | $0.0005 | No |

The `AnchoredProviderFactory` deploys `AnchoredPriceProvider` with arbitrary `baseFeedId`/`quoteFeedId` pairs. Any pool configured with a micro-cap base token and a high-value quote token will hit this silently at deployment time or whenever the base token's price drops below the threshold.

---

### Recommendation

Add a zero-check on the synthetic ratio immediately after the `Math.mulDiv` call:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
if (mid == 0) return (0, type(uint128).max);   // ← add this guard
```

Alternatively, scale the numerator further before division (e.g., use `mid1 * ORACLE_DECIMALS * ORACLE_DECIMALS / mid2` and carry the extra factor through `_computeBidAsk`) to preserve sub-cent precision for low-value tokens.

---

### Proof of Concept

1. Deploy `AnchoredPriceProvider` with:
   - `baseFeedId` = SHIB/USD feed (price = `1000`, i.e., $0.00001 with 8 decimals)
   - `quoteFeedId` = BTC/USD feed (price = `5_000_000_000_000`, i.e., $50,000 with 8 decimals)
2. Call `getBidAndAskPrice()`:
   - `mid1 = 1000`, `mid2 = 5_000_000_000_000`
   - `mid = Math.mulDiv(1000, 1e8, 5_000_000_000_000) = 100_000_000_000 / 5_000_000_000_000 = 0`
   - `_computeBidAsk(0, spreadBps)` → `refBid = _bandEdge(0, …) = 0` → returns `(0, type(uint128).max)`
   - `getBidAndAskPrice()` reverts with `FeedStalled()`
3. Every subsequent swap on the pool reverts. The pool is permanently bricked for trading.
