### Title
Staker Can Instantly Increase Commission to 100% via Commitment, Stealing Delegators' Unclaimed Yield - (File: src/staking/staking.cairo)

### Summary
A staker can exploit the `set_commission_commitment` + `set_commission` flow to instantly raise their commission from 0% to 100% in a single epoch, with no delay for delegators to react. All delegator rewards for that epoch are then routed to the staker upon attestation. The code itself acknowledges this gap with an inline note.

### Finding Description
The `set_commission` function in `src/staking/staking.cairo` has two paths for updating an existing commission:

1. **Without an active commitment** (line 1596): commission can only *decrease* (`commission < old_commission`). This protects delegators.
2. **With an active commitment** (lines 1583–1589): commission can be set to *any* value up to `commission_commitment.max_commission`, including `COMMISSION_DENOMINATOR` (10 000 = 100%).

The `set_commission_commitment` function (lines 748–785) allows a staker to create a commitment with `max_commission` up to `COMMISSION_DENOMINATOR` as long as `max_commission >= current_commission`. There is no minimum notice period, no epoch delay, and no cap on how large `max_commission` may be.

The attack sequence:
1. Staker stakes with commission = 0 to attract delegators.
2. Delegators join the pool and accumulate unclaimed rewards over multiple epochs.
3. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)` — valid because `10000 >= 0`.
4. In the same transaction or the next block, staker calls `set_commission(commission: 10000)` — valid because `10000 <= max_commission` and `commission != old_commission`.
5. Commission is now 100%, effective immediately (written directly to storage at line 1600, no K-epoch delay).
6. Staker calls `attest`. Reward calculation uses the current 100% commission, routing all delegator rewards to the staker. Delegators receive 0.

The code itself acknowledges this at lines 745–746:
```
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
```

### Impact Explanation
**High — Theft of unclaimed yield.**

Delegators who have been accumulating rewards over one or more epochs lose their entire epoch reward in a single atomic operation. The staker captures 100% of the pool's reward share. This maps directly to the BribeVault analog: a privileged party (here, the staker) can change the fee/commission to 100% and retroactively apply it to already-delegated funds before the affected parties can exit.

### Likelihood Explanation
Any staker can execute this attack without any special privilege, leaked key, or external dependency. The only prerequisite is having an active delegation pool with delegators. The attack is fully on-chain and atomic. A rational attacker would wait until delegators have accumulated significant unclaimed rewards before executing.

### Recommendation
1. **Cap `max_commission` in `set_commission_commitment`**: Enforce a protocol-level maximum (e.g., 30–50%) so that even with a commitment, commission cannot reach 100%.
2. **Enforce a minimum notice period**: Require that a commission *increase* only takes effect after at least one full epoch has elapsed since the commitment was set, giving delegators time to exit.
3. **Epoch-delay commission increases**: Apply the same K-epoch delay used for balance changes to commission increases, so delegators can observe the pending change and react before it affects their rewards.

### Proof of Concept

**Entry path** (no privileged access required — any staker):

```
// Step 1: Staker stakes with 0% commission, attracts delegators
staking.stake(reward_address, operational_address, amount)
// commission = 0, delegators join over multiple epochs

// Step 2: Set a commitment allowing increase to 100%
staking.set_commission_commitment(
    max_commission: 10000,          // COMMISSION_DENOMINATOR = 100%
    expiration_epoch: current + 1   // valid: > current_epoch, <= current + epochs_in_year
)
// Precondition satisfied: max_commission (10000) >= current_commission (0)

// Step 3: Immediately raise commission to 100%
staking.set_commission(commission: 10000)
// Passes: 10000 <= max_commission (10000), 10000 != old_commission (0)
// Written to storage immediately — no epoch delay

// Step 4: Attest — all delegator rewards for this epoch go to staker
attestation.attest(staker_address, block_number, block_hash)
// Delegators receive 0 rewards
```

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }
