### Title
Mandatory Transfer to Hardcoded Recipient in `unstake_action` and `exit_delegation_pool_action` Permanently Freezes Principal When Recipient Is Blacklisted - (File: src/staking/staking.cairo, src/pool/pool.cairo)

### Summary
Both `unstake_action` (staking contract) and `exit_delegation_pool_action` (pool contract) unconditionally transfer the staking principal back to a fixed, non-changeable address (`staker_address` and `pool_member` respectively). If the staking token (STRK or a BTC-wrapped token) implements a blacklist and the recipient address is blacklisted, the transfer reverts and the entire exit flow is permanently blocked. Because neither `staker_address` nor `pool_member` can be changed by the user, there is no recovery path.

### Finding Description

**Case 1 — `unstake_action` in `staking.cairo`:**

`unstake_action` first calls `send_rewards_to_staker`, which transfers STRK rewards to `reward_address`. The staker *can* change `reward_address` via `change_reward_address`, so a blacklisted `reward_address` is a temporary freeze. However, the function then unconditionally transfers the staked principal to the immutable `staker_address`:

```cairo
// src/staking/staking.cairo line 504-506
// Return stake to staker.
token_dispatcher
    .checked_transfer(recipient: staker_address, amount: staker_amount.into());
```

`staker_address` is the address that originally called `stake()` and is permanently recorded. There is no `change_staker_address` function. If STRK blacklists `staker_address`, every call to `unstake_action` reverts at this line. The staker's entire principal is permanently locked in the staking contract with no recovery path.

**Case 2 — `exit_delegation_pool_action` in `pool.cairo`:**

After the exit window elapses, the pool contract transfers the delegated principal directly to `pool_member`:

```cairo
// src/pool/pool.cairo line 328-330
// Transfer delegated amount to the pool member.
let token_dispatcher = self.token_dispatcher.read();
token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());
```

`pool_member` is the address that called `enter_delegation_pool()` and is stored immutably. There is no mechanism to redirect this transfer. For BTC-pool delegators, the staking token is a BTC-wrapped ERC20 (which is more likely than STRK to carry a blacklist). If `pool_member` is blacklisted by that token, `exit_delegation_pool_action` always reverts and the delegated BTC principal is permanently frozen in the pool contract.

### Impact Explanation

- **`unstake_action`**: The staker's entire self-staked STRK principal is permanently locked in the staking contract. The staker record cannot be deleted, and no other address can redirect the principal. This constitutes **permanent freezing of funds**.
- **`exit_delegation_pool_action`**: The pool member's delegated BTC (or STRK) principal is permanently locked in the pool contract. This constitutes **permanent freezing of funds**.

Both match the allowed impact: *High — Permanent freezing of unclaimed yield or unclaimed royalties; Temporary freezing of funds*.

### Likelihood Explanation

- STRK on Starknet is a standard ERC20 and may gain a blacklist in a future upgrade (as has happened with other major tokens). BTC-wrapped tokens used in the multi-token pool are third-party assets that commonly carry blacklist functionality.
- The scenario requires: (1) a staker or pool member opens a position, (2) the token admin later blacklists their address (e.g., due to sanctions or legal action), (3) the exit flow is called. This is the same low-but-non-zero likelihood acknowledged in the original H-01 report.
- Critically, unlike the `reward_address` case (which is changeable), neither `staker_address` nor `pool_member` can be updated, making the freeze permanent once it occurs.

### Recommendation

1. **Decouple reward transfer from principal transfer in `unstake_action`**: Do not revert the entire unstake if the reward transfer fails. Record unclaimed rewards separately so the principal can still be returned.
2. **Add a `claims` mapping or pull-payment pattern**: Instead of pushing principal to `staker_address` / `pool_member` in the exit action, record the owed amount in a `claimable[address]` mapping and let the user pull it. This is the exact mitigation suggested in the original H-01 report.
3. **For `exit_delegation_pool_action`**: Allow an authorized address (e.g., the pool member themselves, set at entry time) to redirect the principal transfer destination, similar to how `reward_address` is a separate, changeable field.

### Proof of Concept

**`unstake_action` permanent freeze:**

1. Staker calls `stake()` with `staker_address = A`, `reward_address = B`.
2. Staker calls `unstake_intent()`.
3. Exit window elapses.
4. STRK token admin blacklists address `A`.
5. Anyone calls `unstake_action(staker_address: A)`.
6. `send_rewards_to_staker` succeeds (transfers rewards to `B`).
7. `checked_transfer(recipient: A, amount: staker_amount)` reverts — `A` is blacklisted.
8. Entire transaction reverts. Staker record is restored. Principal remains in staking contract.
9. Steps 5–8 repeat forever. Principal is permanently locked. [1](#0-0) [2](#0-1) 

**`exit_delegation_pool_action` permanent freeze (BTC pool):**

1. Pool member calls `enter_delegation_pool()` with `pool_member = C` in a BTC pool.
2. Pool member calls `exit_delegation_pool_intent(amount)`.
3. Exit window elapses.
4. BTC token admin blacklists address `C`.
5. Anyone calls `exit_delegation_pool_action(pool_member: C)`.
6. `remove_from_delegation_pool_action` succeeds (BTC transferred from staking contract to pool contract).
7. `checked_transfer(recipient: C, amount: unpool_amount)` reverts — `C` is blacklisted by BTC token.
8. Entire transaction reverts. BTC remains in pool contract.
9. Steps 5–8 repeat forever. BTC principal is permanently locked in the pool contract. [3](#0-2)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount

        fn exit_delegation_pool_action(
            ref self: ContractState, pool_member: ContractAddress,
        ) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let unpool_time = pool_member_info
                .unpool_time
                .expect_with_err(GenericError::MISSING_UNDELEGATE_INTENT);
            assert!(Time::now() >= unpool_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberExitAction {
                        pool_member, unpool_amount: pool_member_info.unpool_amount,
                    },
                );

            // Perform removal action in the staking contract, receiving funds if needed.
            // Note that if the intent was done after the staker was removed (unstake_action),
            // the funds will already be in the pool contract, and the following call will do
            // nothing.
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_action(identifier: pool_member.into());

            let unpool_amount = pool_member_info.unpool_amount;
            pool_member_info.unpool_amount = Zero::zero();
            pool_member_info.unpool_time = Option::None;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer delegated amount to the pool member.
            let token_dispatcher = self.token_dispatcher.read();
            token_dispatcher.checked_transfer(recipient: pool_member, amount: unpool_amount.into());

            unpool_amount
        }
