### Title
`ExitToEthereum` Precompile Has No Refund Callback When Downstream `withdraw` Promise Fails — (`engine-precompiles/src/native.rs`)

---

### Summary

The `ExitToEthereum` precompile burns a user's ERC-20 (or base ETH) tokens inside the EVM, then schedules a NEAR promise to call `withdraw` on the eth-connector / NEP-141 contract. If that promise fails (e.g., because the eth-connector's `engine_withdraw` feature is paused by an admin), the burn is already committed and there is no callback or refund path. The user permanently loses their tokens.

---

### Finding Description

**Two-step operation with asymmetric state checks**

The `ExitToEthereum` precompile (`ExitToEthereum::run`) handles two token types:

- **Flag `0x0` (ETH base token):** burns ETH from the EVM balance, then schedules a promise to call `withdraw` on the eth-connector account.
- **Flag `0x1` (ERC-20):** the ERC-20 contract burns tokens before calling the precompile, then the precompile schedules a promise to call `withdraw` on the corresponding NEP-141 contract.

In both cases the promise is created as a bare `PromiseArgs::Create` with **no callback**:

```rust
// engine-precompiles/src/native.rs  line 985
let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
``` [1](#0-0) 

The EVM state change (burn) is committed in the first NEAR receipt. The `withdraw` promise executes in a subsequent, independent receipt. If that receipt fails, NEAR reverts only the promise's own state changes — the EVM burn is **not** rolled back.

**Contrast with `ExitToNear`**

`ExitToNear` has a conditional refund path via `exit_to_near_precompile_callback`:

```rust
// engine-precompiles/src/native.rs  lines 449-483
let callback_args = ExitToNearPrecompileCallbackArgs {
    #[cfg(feature = "error_refund")]
    refund: refund_call_args(&exit_to_near_params, &exit_event),
    #[cfg(not(feature = "error_refund"))]
    refund: None,
    transfer_near: transfer_near_args,
};
``` [2](#0-1) 

`ExitToEthereum` has no equivalent callback at all.

**The eth-connector `engine_withdraw` can be independently paused**

The engine exposes its own precompile-level pause flags (`EXIT_TO_ETHEREUM`, `EXIT_TO_NEAR`): [3](#0-2) 

These are separate from the eth-connector's own feature-level pause (`pa_pause_feature` / `"engine_withdraw"`). An admin can pause the eth-connector's `engine_withdraw` while leaving the engine-level `EXIT_TO_ETHEREUM` precompile active. The engine-level pause prevents the EVM call from reaching the precompile (so the burn never happens). But the eth-connector-level pause only blocks the downstream NEAR promise — after the burn has already been committed.

---

### Impact Explanation

When the eth-connector's `engine_withdraw` is paused (or the NEP-141 `withdraw` fails for any reason) while the engine-level `EXIT_TO_ETHEREUM` precompile is still active:

1. The EVM transaction succeeds: ERC-20 tokens are burned / ETH is deducted.
2. The NEAR promise to `withdraw` fails.
3. No refund callback exists in `ExitToEthereum`.
4. The user's tokens are **permanently destroyed** with no recovery path.

**Impact: Critical — Permanent freezing of user funds.**

---

### Likelihood Explanation

The eth-connector is a separate deployed contract with its own admin-controlled pause mechanism, as demonstrated by the `pa_pause_feature` / `"engine_withdraw"` key used in integration tests: [4](#0-3) 

A legitimate security incident (e.g., a bridge exploit) would cause the eth-connector admin to pause `engine_withdraw` while leaving the Aurora engine running. Any user who submits an `ExitToEthereum` transaction during that window — or whose transaction was already in-flight — permanently loses their tokens. This is a realistic operational scenario, not a theoretical one.

---

### Recommendation

Add a callback to `ExitToEthereum` analogous to the `exit_to_near_precompile_callback` already present for `ExitToNear`. On promise failure, the callback should re-credit the burned amount to the original sender's EVM address (for ETH) or re-mint the ERC-20 tokens (for ERC-20 exits). The `error_refund` feature pattern already used in `ExitToNear` should be extended to cover `ExitToEthereum`.

---

### Proof of Concept

1. Admin pauses `engine_withdraw` on the eth-connector contract (legitimate security action).
2. User calls the ERC-20 `withdraw` function, which burns tokens and invokes the `ExitToEthereum` precompile (engine-level precompile is **not** paused, so the EVM call succeeds).
3. Engine schedules `PromiseArgs::Create(withdraw_promise)` targeting the eth-connector's `withdraw` method. [1](#0-0) 
4. The NEAR promise executes and fails because `engine_withdraw` is paused on the eth-connector.
5. NEAR reverts only the promise receipt; the EVM burn from step 2 is **not** reverted.
6. No `exit_to_ethereum_precompile_callback` exists to refund the user. [5](#0-4) 
7. User's ERC-20 tokens are permanently destroyed; no ETH is released on Ethereum.

impl<I: IO> Precompile for ExitToEthereum<I> {
    fn required_gas(_input: &[u8]) -> Result<EthGas, ExitError> {
        Ok(costs::EXIT_TO_ETHEREUM_GAS)
    }

    #[allow(clippy::too_many_lines)]
    fn run(
        &self,
        input: &[u8],
        target_gas: Option<EthGas>,
        context: &Context,
        is_static: bool,
    ) -> EvmPrecompileResult {
        // ETH (Base token) transfer input format (min size 21 bytes)
        //  - flag (1 byte)
        //  - eth_recipient (20 bytes)
        // ERC-20 transfer input format: max 53 bytes
        //  - flag (1 byte)
        //  - amount (32 bytes)
        //  - eth_recipient (20 bytes)
        validate_input_size(input, 21, 53)?;

        let required_gas = Self::required_gas(input)?;

        if let Some(target_gas) = target_gas
            && required_gas > target_gas
        {
            return Err(ExitError::OutOfGas);
        }

        // It's not allowed to call exit precompiles in static mode
        if is_static {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_STATIC")));
        } else if context.address != exit_to_ethereum::ADDRESS.raw() {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_DELEGATE")));
        }

        // The first byte of the input is a flag, selecting the behavior to be triggered:
        //  0x00 -> ETH (Base token) token transfer
        //  0x01 -> ERC-20 transfer
        let mut input = input;
        let flag = input[0];
        input = &input[1..];

        let (nep141_address, serialized_args, exit_event) = match flag {
            0x0 => {
                // ETH (base) transfer
                //
                // Input slice format:
                //  eth_recipient (20 bytes) - the address of recipient which will receive ETH on Ethereum
                let recipient_address: Address = input
                    .try_into()
                    .map_err(|_| ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS")))?;
                let serialize_fn = match get_withdraw_serialize_type(&self.io)? {
                    WithdrawSerializeType::Json => json_args,
                    WithdrawSerializeType::Borsh => borsh_args,
                };
                let eth_connector_account_id = self.get_eth_connector_contract_account()?;

                (
                    eth_connector_account_id,
                    // There is no way to inject json, given the encoding of both arguments
                    // as decimal and hexadecimal respectively.
                    serialize_fn(recipient_address, context.apparent_value)?,
                    events::ExitToEth {
                        sender: Address::new(context.caller),
                        erc20_address: events::ETH_ADDRESS,
                        dest: recipient_address,
                        amount: context.apparent_value,
                    },
                )
            }
            0x1 => {
                // ERC-20 transfer
                //
                // This precompile branch is expected to be called from the ERC20 withdraw function
                // (or burn function with some flag provided that this is expected to be withdrawn)
                //
                // Input slice format:
                //  amount (U256 big-endian bytes) - the amount that was burned
                //  eth_recipient (20 bytes) - the address of recipient which will receive ETH on Ethereum

                if context.apparent_value != U256::from(0) {
                    return Err(ExitError::Other(Cow::from(
                        "ERR_ETH_ATTACHED_FOR_ERC20_EXIT",
                    )));
                }

                let erc20_address = context.caller;
                let nep141_address = get_nep141_from_erc20(erc20_address.as_bytes(), &self.io)?;
                let amount = parse_amount(&input[..32])?;

                input = &input[32..];

                if input.len() == 20 {
                    // Parse ethereum address in hex
                    let mut buffer = [0; 40];
                    hex::encode_to_slice(input, &mut buffer).unwrap();
                    let recipient_in_hex = str::from_utf8(&buffer).map_err(|_| {
                        ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS"))
                    })?;
                    // unwrap cannot fail since we checked the length already
                    let recipient_address = Address::try_from_slice(input)
                        .map_err(|_| ExitError::Other(Cow::from("ERR_WRONG_ADDRESS")))?;

                    (
                        nep141_address,
                        // There is no way to inject json, given the encoding of both arguments
                        // as decimal and hexadecimal respectively.
                        format!(
                            r#"{{"amount": "{}", "recipient": "{}"}}"#,
                            amount.as_u128(),
                            recipient_in_hex
                        )
                        .into_bytes(),
                        events::ExitToEth {
                            sender: Address::new(erc20_address),
                            erc20_address: Address::new(erc20_address),
                            dest: recipient_address,
                            amount,
                        },
                    )
                } else {
                    return Err(ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS")));
                }
            }
            _ => {
                return Err(ExitError::Other(Cow::from(
                    "ERR_INVALID_RECEIVER_ACCOUNT_ID",
                )));
            }
        };

        let withdraw_promise = PromiseCreateArgs {
            target_account_id: nep141_address,
            method: "withdraw".to_string(),
            args: serialized_args,
            attached_balance: Yocto::new(1),
            attached_gas: costs::WITHDRAWAL_GAS,
        };

        let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
        let promise_log = Log {
            address: exit_to_ethereum::ADDRESS.raw(),
            topics: Vec::new(),
            data: promise,
        };
        let ethabi::RawLog { topics, data } = exit_event.encode();
        let exit_event_log = Log {
            address: exit_to_ethereum::ADDRESS.raw(),
            topics: topics.into_iter().map(|h| H256::from(h.0)).collect(),
            data,
        };

        Ok(PrecompileOutput {
            logs: vec![promise_log, exit_event_log],
            cost: required_gas,
            output: Vec::new(),
        })
    }
