### Title
`require_running` Guard in `exit_to_near_precompile_callback` Causes Permanent Fund Loss When Engine Is Paused During In-Flight Exit - (File: `engine/src/contract_methods/connector.rs`)

---

### Summary

When a user initiates an `ExitToNear` precompile call (e.g., to unwrap wNEAR), the EVM execution burns the user's ERC-20 tokens and schedules a NEAR async callback `exit_to_near_precompile_callback`. If the engine owner pauses the contract between the initial EVM execution and the callback execution, the callback is blocked by `require_running` and panics. Because NEAR callbacks are not retried on failure, the NEAR tokens already released by `near_withdraw` are permanently stranded in Aurora's account and the user's ERC-20 tokens are already burned — a permanent, irrecoverable fund loss.

---

### Finding Description

The `exit_to_near_precompile_callback` function is the NEAR-level callback that completes the two-step `ExitToNear` precompile flow. Its first action is to call `require_running`:

```rust
pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I, env: &E, handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;   // ← blocks if is_paused == true
        ...
        if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                // Transfer NEAR to user (wNEAR-unwrap path)
                handler.promise_create_batch(&promise);
            }
        } else if let Some(args) = args.refund {
            // Refund burned ERC-20 tokens (error_refund path)
            engine::refund_on_error(io, env, state, &args, handler)?;
        }
    })
}

`require_running` returns `Err(ERR_PAUSED)` whenever `state.is_paused == true`: [2](#0-1) 

The error propagates through `sdk_unwrap()` in `lib.rs`, causing the NEAR receipt to panic and fail: [3](#0-2) 

The `ExitToNear` precompile, when processing a wNEAR unwrap, creates a two-step promise: first `near_withdraw` on the wNEAR contract (which releases NEAR from Aurora's account), then `exit_to_near_precompile_callback` to forward those NEAR tokens to the user: [4](#0-3) 

The callback is the only place where the NEAR transfer to the user is issued. If it fails, the NEAR tokens remain in Aurora's account with no recovery path for the user.

The engine can be paused by the owner at any time via `pause_contract`: [5](#0-4) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Two concrete loss paths exist:

1. **wNEAR unwrap path (no feature flag required):** User burns wNEAR ERC-20 tokens → `near_withdraw` succeeds and NEAR tokens are released into Aurora's account → engine is paused → callback runs → `require_running` panics → the NEAR transfer to the user is never issued → NEAR tokens are permanently stranded in Aurora's account. The user's wNEAR ERC-20 tokens are already burned and the NEAR is unrecoverable through any user-accessible method.

2. **ERC-20 exit with `error_refund` feature:** User burns ERC-20 tokens → NEP-141 `ft_transfer` fails → engine is paused → callback runs → `require_running` panics → `refund_on_error` is never called → ERC-20 tokens are permanently lost.

In both cases, NEAR's async execution model means the callback receipt is final once it fails; there is no retry mechanism.

---

### Likelihood Explanation

**Medium.** The engine owner can pause the contract for legitimate maintenance. The window between the initial `submit` call (which burns tokens and schedules the promise) and the callback execution spans at least one NEAR block. Any pause issued in that window — even briefly — triggers the loss. The owner has no on-chain visibility into in-flight `ExitToNear` callbacks before pausing. Users with pending exits have no way to cancel or protect themselves once the EVM execution has already burned their tokens.

---

### Recommendation

Remove the `require_running` guard from `exit_to_near_precompile_callback`. This callback is a private self-call (`env.assert_private_call()` is enforced immediately after) that was already authorized by a prior `submit` call that passed the `require_running` check at that time. Blocking the callback does not prevent any new user action; it only prevents the completion of an already-committed exit, causing irreversible fund loss.

### Proof of Concept

1. User calls `submit` with a transaction that invokes the `ExitToNear` precompile to unwrap wNEAR. The EVM execution burns the user's wNEAR ERC-20 tokens and schedules:
   - Promise A: `near_withdraw` on the wNEAR NEP-141 contract
   - Promise B (callback): `exit_to_near_precompile_callback` on Aurora

2. Promise A executes successfully. NEAR tokens are now held in Aurora's account.

3. The engine owner calls `pause_contract` (e.g., for a routine upgrade). `state.is_paused` is set to `true`.

4. Promise B (the callback) executes. `exit_to_near_precompile_callback` is entered. `require_running` reads `state.is_paused == true` and returns `Err(ERR_PAUSED)`. The function propagates the error; `sdk_unwrap()` panics; the NEAR receipt fails.

5. The NEAR tokens released in step 2 remain in Aurora's account. The user's wNEAR ERC-20 tokens are already burned. No user-callable method can recover the NEAR tokens while the engine is paused (all write methods check `require_running`), and after unpausing, the callback receipt is already finalized — NEAR has no replay mechanism for failed receipts. pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })

pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
