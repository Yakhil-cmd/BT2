### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Freeze Staker Yield - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `IStakingRewardsManager` is callable by any address despite the specification explicitly restricting it to "Only starkware sequencer." The function writes to the global `last_reward_block` storage variable, which gates all subsequent calls in the same block. An unprivileged attacker can call `update_rewards` with `disable_rewards: true` at the start of every block, consuming the per-block slot and causing the legitimate sequencer call to revert with `REWARDS_ALREADY_UPDATED`, permanently preventing reward distribution for all stakers.

---

### Finding Description

`update_rewards` is defined in `IStakingRewardsManager` and implemented in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distribute rewards
}
``` [1](#0-0) 

`last_reward_block` is a **global** (not per-staker) storage variable: [2](#0-1) 

The specification for this function explicitly states:

> **access control**: Only starkware sequencer. [3](#0-2) 

However, the implementation contains **no caller check** — `general_prerequisites()` only verifies the contract is not paused. Any address can invoke `update_rewards`.

Because `last_reward_block` is global, a single call by any address in a given block sets the flag for that entire block. The check `current_block_number > self.last_reward_block.read()` then causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

---

### Impact Explanation

An attacker who calls `update_rewards(valid_staker_address, disable_rewards: true)` at the start of every block:

1. Sets `last_reward_block` to the current block number.
2. Causes the legitimate sequencer call to revert with `REWARDS_ALREADY_UPDATED`.
3. Because `disable_rewards: true` was passed, no rewards are distributed.
4. Repeated every block → **all stakers permanently receive zero consensus-phase block rewards**.

This matches the allowed impact: **Permanent freezing of unclaimed yield**. [5](#0-4) 

---

### Likelihood Explanation

**High.** The only precondition is knowing any valid, active staker address with non-zero balance — all of which are public on-chain. No privileged role, no capital at risk, no rare timing condition. The attacker simply front-runs the sequencer's `update_rewards` call every block with `disable_rewards: true`. [6](#0-5) 

---

### Recommendation

Add a caller check at the top of `update_rewards` that restricts execution to the authorized sequencer address (analogous to `assert_caller_is_attestation_contract` used in `update_rewards_from_attestation_contract`):

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
``` [7](#0-6) 

Alternatively, store the authorized sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation.

---

### Proof of Concept

1. Deploy the staking system and advance past the consensus rewards first epoch so `is_pre_consensus()` returns `false`.
2. Stake as a legitimate staker and advance K epochs so the staker has non-zero balance.
3. At the start of block N, attacker calls:
   ```
   update_rewards(staker_address, disable_rewards: true)
   ```
   This succeeds: `last_reward_block` is written to N, no rewards distributed.
4. The legitimate sequencer then calls:
   ```
   update_rewards(staker_address, disable_rewards: false)
   ```
   This reverts with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. Repeat step 3–4 every block. The staker's `unclaimed_rewards_own` never increases.
6. The existing test `test_update_rewards_assertions_already_consensus` already demonstrates the `REWARDS_ALREADY_UPDATED` revert path — the only missing piece is that the first call in step 3 is made by an unprivileged attacker rather than the sequencer. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
