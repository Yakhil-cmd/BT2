### Title
Missing Access Control on `update_rewards` Allows Any Staker to Permanently Freeze Other Stakers' Unclaimed Yield — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is documented in the spec as callable "Only starkware sequencer," but the implementation contains no enforcement of this restriction. Because `last_reward_block` is a **global** variable updated on every successful call, any registered staker can call `update_rewards` before the sequencer in each block, consuming the single allowed slot and permanently preventing all other stakers from receiving their consensus block rewards.

---

### Finding Description

`update_rewards` is a public function in the `IStakingRewardsManager` interface. Its only gate is `general_prerequisites()`, which checks:

1. Contract is not paused
2. Caller is not the zero address [1](#0-0) 

No check verifies that the caller is the authorized Starknet sequencer, despite the spec explicitly requiring it. [2](#0-1) 

The function enforces at most one reward distribution per block using a **single global** `last_reward_block` variable: [3](#0-2) 

Because `last_reward_block` is not per-staker, the first successful call in any block exhausts the slot for that block. Any registered V3-migrated staker can call `update_rewards(attacker_staker, false)` first in every block, setting `last_reward_block` to the current block number. Every subsequent call by the sequencer for any other staker will revert with `REWARDS_ALREADY_UPDATED`.

---

### Impact Explanation

An attacker who is a registered staker calls `update_rewards(attacker_staker, false)` in every block:

1. The attacker's staker receives its legitimate block rewards.
2. `last_reward_block` is set to the current block number.
3. The sequencer's `update_rewards` calls for all other stakers fail with `REWARDS_ALREADY_UPDATED`.
4. All other stakers permanently miss their consensus block rewards.

This constitutes **permanent freezing of unclaimed yield** for every staker except the attacker — a **High** impact per the allowed impact scope. [4](#0-3) 

---

### Likelihood Explanation

Any address that has completed `stake()` and `staker_migration()` (i.e., is a V3-migrated staker) can trigger this. The attacker needs only to submit `update_rewards` transactions with sufficient priority to be included before the sequencer's own reward-distribution transactions. Because the function is fully public and the only per-block guard is the global `last_reward_block`, the attack is repeatable every block with no additional cost beyond gas. [5](#0-4) 

---

### Recommendation

Add an access control check at the top of `update_rewards` that asserts the caller is the authorized sequencer address (stored in contract storage, analogous to how `assert_caller_is_attestation_contract` is implemented):

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // enforce spec requirement
    ...
``` [6](#0-5) 

---

### Proof of Concept

1. Attacker registers as a staker via `stake()` with the minimum required amount.
2. Attacker calls `staker_migration()` to reach V3.
3. In every block (once consensus rewards are active), attacker calls `update_rewards(attacker_address, false)`.
4. `last_reward_block` is set to the current block number.
5. The sequencer's `update_rewards` calls for all other stakers revert with `REWARDS_ALREADY_UPDATED`.
6. All other stakers receive zero block rewards indefinitely.
7. Attacker collects their own block rewards every block while the rest of the protocol's stakers are frozen out. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
