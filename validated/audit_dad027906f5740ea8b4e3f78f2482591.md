### Title
`assert_reshare_keys_invariants` Missing Validation That a New Participant Does Not Supply an Old Signing Key — (File: `src/dkg.rs`)

---

### Summary

`assert_reshare_keys_invariants` enforces only one direction of the role/key consistency invariant for reshare participants. It checks that an *old* participant must supply a signing key, but it never checks that a *new* participant (one absent from `old_participants`) must **not** supply a signing key. A malicious new participant can therefore pass initialization with a fabricated `old_signing_key`, causing the reshare protocol to abort mid-execution and permanently denying the reshare for all honest parties.

---

### Finding Description

`assert_reshare_keys_invariants` is the public-API guard called by both `reshare()` and `refresh()` before the async protocol future is spawned. [1](#0-0) 

The function's own comment at line 662 states the intended invariant:

```
// if me is not in the old participant set then ensure that old_signing_key is None
```

But the code that follows checks only the **opposite** direction:

```rust
if old_participants.contains(me) && old_signing_key.is_none() {   // line 663
    return Err(...)
}
``` [2](#0-1) 

The symmetric guard — `!old_participants.contains(me) && old_signing_key.is_some()` — is entirely absent. Because `assert_reshare_keys_invariants` returns `Ok(...)` in this case, `reshare()` returns `Ok(protocol)` with no `InitializationError`. [3](#0-2) 

When the returned protocol is executed, control enters `do_reshare`, which computes:

```rust
let intersection = old_participants.intersection(&participants);
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)          // me ∉ intersection → error
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?                       // propagates ProtocolError
    .unwrap_or_else(...);
``` [4](#0-3) 

Because `me` is not in `old_participants`, it is not in `intersection`, so `lagrange` returns an error that is propagated as a `ProtocolError` from inside the async future. The participant's protocol task terminates without ever sending the Round-1 session-ID broadcast or any subsequent messages. Every other honest participant that has already started the reshare is left waiting indefinitely for messages that will never arrive.

The inner guard `assert_keyshare_inputs` inside `do_keyshare` would also catch this, but it is never reached because `do_reshare` fails before calling `do_keyshare`. [5](#0-4) 

---

### Impact Explanation

A malicious participant who is legitimately invited into the **new** participant set but was **not** part of the old set can call `reshare()` with an arbitrary `old_signing_key = Some(fake_share)`. The initialization guard passes, the protocol object is returned, and when the protocol is driven, the participant's task aborts with a `ProtocolError` before emitting any network messages. All honest participants that have already entered the reshare loop are permanently blocked waiting for that participant's Round-1 broadcast. The reshare cannot complete, and the key set cannot be updated.

**Impact class**: High — Permanent denial of reshare for honest parties.

---

### Likelihood Explanation

Any participant that is newly added to the resharing set (i.e., `me ∉ old_participants`) and that deliberately or accidentally supplies a non-`None` `old_signing_key` triggers this path. The attacker only needs to be included in the new participant list — a condition that is under the control of the protocol orchestrator, but which is a normal operational step when onboarding new nodes. The fabricated share value is never cryptographically verified at initialization time, so any `SigningShare` value suffices.

---

### Recommendation

Add the symmetric guard immediately after the existing check in `assert_reshare_keys_invariants`:

```rust
// Existing check (line 663)
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}

// Missing symmetric check — add this:
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not present in the old participant list but provided a share"
    )));
}
``` [6](#0-5) 

This mirrors the logic already present in the internal `assert_keyshare_inputs` function and ensures the invariant is enforced at the public API boundary before any protocol state is created.

---

### Proof of Concept

1. Run a normal DKG with participants `[P1, P2, P3]` and threshold 2.
2. Introduce a new participant `P4` (not in the old set).
3. `P4` calls `reshare(old_participants=[P1,P2,P3], old_threshold=2, old_signing_key=Some(fake_share), old_public_key=pk, new_participants=[P1,P2,P3,P4], new_threshold=2, me=P4, rng)`.
4. `assert_reshare_keys_invariants` checks `old_participants.contains(P4) && old_signing_key.is_none()` → `false && false` → no error. Returns `Ok(protocol)`.
5. `P1`, `P2`, `P3` call `reshare()` correctly and begin executing the protocol, waiting for `P4`'s Round-1 session-ID broadcast.
6. `P4`'s protocol future enters `do_reshare`, computes `intersection = {P1,P2,P3} ∩ {P1,P2,P3,P4} = {P1,P2,P3}`, then calls `intersection.lagrange::<C>(P4)` → error (P4 ∉ intersection) → `ProtocolError` propagated via `?`.
7. `P4`'s task terminates. `P1`, `P2`, `P3` never receive `P4`'s broadcast and the reshare stalls permanently. [7](#0-6)

/// This function prevents calling keyshare function with inproper inputs
fn assert_keyshare_inputs<C: Ciphersuite>(
    me: Participant,
    secret: &Scalar<C>,
    old_reshare_package: Option<(VerifyingKey<C>, ParticipantList)>,
) -> Result<(Option<VerifyingKey<C>>, Option<ParticipantList>), ProtocolError> {
    let is_zero_secret = *secret == <C::Group as Group>::Field::zero();

    if let Some((old_key, old_participants)) = old_reshare_package {
        if is_zero_secret {
            //  return error if me is not a purely new joiner to the participants set
            //  prevents accidentally calling keyshare with extremely old keyshares
            //  that have nothing to do with the current resharing
            if old_participants.contains(me) {
                return Err(ProtocolError::AssertionFailed(
                    format!("{me:?} is running Resharing with a zero share but does belong to the old participant set")));
            }
        } else {
            //  return error if me is part of the old participants set
            if !old_participants.contains(me) {
                return Err(ProtocolError::AssertionFailed(
                    format!("{me:?} is running Resharing with a non-zero share but does not belong to the old participant set")));
            }
        }
        Ok((Some(old_key), Some(old_participants)))
    } else {
        if is_zero_secret {
            return Err(ProtocolError::AssertionFailed(format!(
                "{me:?} is running DKG with a zero share"
            )));
        }
        Ok((None, None))
    }
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
