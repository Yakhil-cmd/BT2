### Title
WalletConnect `take_offer` Confirmation Displays Underestimated Royalty Total When Multiple NFTs Are Involved - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

---

### Summary

The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts`, used to compute the `amountWithRoyalties` field shown in the WalletConnect offer-approval dialog, incorrectly divides the fungible amount equally among all NFTs before computing each royalty. When a `chia_wallet.take_offer` request involves multiple NFTs with royalties, the displayed total spend is underestimated by a factor of n (number of NFTs), causing the user to approve a transaction believing they will spend less than they actually will.

---

### Finding Description

`formatAmountWithRoyalties` is called from `withRoyaltyTotals` → `walletDeltaToDisplay` → `parseCommandDisplay` for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` WalletConnect commands. It receives the full fungible `amount` and an array of `royaltyPercentages` (one per NFT on the opposite side of the trade). [1](#0-0) 

The function first computes `splitAmount = amount / BigInt(royaltyPercentages.length)`, then applies each NFT's royalty percentage to that split amount:

```
royaltyAmount = (amount/n)*r1/10000 + (amount/n)*r2/10000 + ...
              = amount * (r1+r2+...+rn) / (n * 10000)
```

The Chia protocol calculates royalties per NFT on the **full** fungible amount (as confirmed by the `calculate_royalties` RPC used in `OfferBuilderProvider.tsx`):

```
royaltyAmount_correct = amount*r1/10000 + amount*r2/10000 + ...
                      = amount * (r1+r2+...+rn) / 10000
```

The split approach underestimates total royalties by exactly a factor of n. [2](#0-1) 

The incorrect `amountWithRoyalties` string is then surfaced in the WalletConnect confirmation dialog as the total the user will spend. [3](#0-2) 

---

### Impact Explanation

A user approving a WalletConnect `chia_wallet.take_offer` for multiple NFTs sees a materially lower total XCH/CAT spend than what the protocol will actually deduct. The actual transaction is constructed and submitted by the backend with correct royalties; only the approval dialog is wrong. The user consents to a spend amount that is smaller than reality.

**Concrete example** (matching the existing test at line 251–335):

| | Displayed (buggy) | Actual (protocol) |
|---|---|---|
| amount | 0.0001 XCH | 0.0001 XCH |
| NFT1 royalty (5%) | 0.0000025 XCH | 0.000005 XCH |
| NFT2 royalty (0.1%) | 0.0000000050 XCH | 0.000000010 XCH |
| **Total shown** | **0.00010255 XCH** | **0.0001051 XCH** |

With high-royalty NFTs (e.g., two NFTs each at 10%), the displayed total is half the actual spend. [4](#0-3) 

---

### Likelihood Explanation

Any dApp connected via WalletConnect can send a `take_offer` request containing multiple NFTs. No special privileges, leaked keys, or host compromise are required. The attacker only needs to craft an offer with multiple high-royalty NFTs and present it through a WalletConnect session. The user sees a plausible but incorrect total and approves.

---

### Recommendation

Remove the `splitAmount` division. Each NFT's royalty should be computed on the full `amount`:

```typescript
// Correct: each NFT royalty applies to the full amount
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [5](#0-4) 

---

### Proof of Concept

Using the existing test scenario (two NFTs, royalties 500 and 10, amount 100,000,000 mojos):

**Current (buggy) code path:**
```
splitAmount = 100_000_000n / 2n = 50_000_000n
royalty1    = 50_000_000n * 500n / 10_000n = 2_500_000n
royalty2    = 50_000_000n * 10n  / 10_000n =    50_000n
total shown = 102_550_000n mojos = 0.00010255 XCH   ← dialog shows this
```

**Correct protocol behavior:**
```
royalty1    = 100_000_000n * 500n / 10_000n = 5_000_000n
royalty2    = 100_000_000n * 10n  / 10_000n =   100_000n
total actual= 105_100_000n mojos = 0.0001051 XCH    ← wallet actually spends this
```

The WalletConnect approval dialog shows **0.00010255 XCH** while the user's wallet is debited **0.0001051 XCH** — a ~2.5% understatement that scales with both the number of NFTs and their royalty percentages. With two NFTs each at 10% royalty, the dialog would show 10% extra cost while the actual cost is 20% extra. [1](#0-0) [6](#0-5)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

async function walletDeltaToDisplay(
  walletDelta: WalletDelta,
  walletInfos: Record<string, WalletInfo>,
  assetKinds: AssetDisplayKinds,
  royaltyPercentages: AssetRoyaltyPercentages,
  fee?: bigint,
): Promise<DisplayWalletDelta> {
  const { spending, receiving } = walletDelta;
  const spendingItems = await Promise.all(
    Object.entries(spending).map(async ([key, value]) => ({
      key,
      line: await parseWalletDeltaItem(
        key,
        value,
        walletInfos,
        assetKinds.spending[key],
        royaltyPercentages.spending[key],
      ),
    })),
  );
  const receivingItems = await Promise.all(
    Object.entries(receiving).map(async ([key, value]) => ({
      key,
      line: await parseWalletDeltaItem(
        key,
        value,
        walletInfos,
        assetKinds.receiving[key],
        royaltyPercentages.receiving[key],
      ),
    })),
  );
  const spendingLines = spendingItems.map(({ line }) => line);
  const receivingLines = receivingItems.map(({ line }) => line);

  return {
    spending: withRoyaltyTotals(spendingItems, spending, receivingLines),
    receiving: withRoyaltyTotals(receivingItems, receiving, spendingLines),
    fee: fee !== undefined ? mojoToChiaLocaleString(fee) : undefined,
  };
}
