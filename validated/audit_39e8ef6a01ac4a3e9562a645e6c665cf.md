### Title
Pause blocks `unstake_action` and `remove_from_delegation_pool_action` after exit window expires, causing temporary freezing of funds - (File: src/staking/staking.cairo)

---

### Summary

Both `unstake_action` and `remove_from_delegation_pool_action` in the Staking contract enforce the `general_prerequisites()` guard, which calls `assert_is_unpaused()`. When the Security Agent pauses the contract — even for a legitimate reason — stakers and delegators whose exit wait window has already elapsed are unable to withdraw their principal. The exit-window clock continues to run during the pause, but the withdrawal action is blocked, causing temporary freezing of funds.

---

### Finding Description

`general_prerequisites()` is the first call inside every public state-changing function in the Staking contract:

```cairo
// src/staking/staking.cairo:1793-1797
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();   // panics with "Contract is paused"
    assert_caller_is_not_zero();
}
```

`assert_is_unpaused` is:

```cairo
// src/staking/staking.cairo:1657-1659
fn assert_is_unpaused(self: @ContractState) {
    assert!(!self.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
}
```

Both settlement functions call this guard unconditionally:

**`unstake_action`** (staking.cairo:483-485):
```cairo
fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    self.general_prerequisites();   // ← blocked when paused
    ...
    assert!(Time::now() >= unstake_time, ...);
```

**`remove_from_delegation_pool_action`** (staking.cairo:1113-1115):
```cairo
fn remove_from_delegation_pool_action(ref self: ContractState, identifier: felt252) {
    self.general_prerequisites();   // ← blocked when paused
    ...
    assert!(Time::now() >= undelegate_intent.unpool_time, ...);
```

The Pool contract's `exit_delegation_pool_action` (pool.cairo:295-319) does not have its own pause check, but it calls `remove_from_delegation_pool_action` on the Staking contract:

```cairo
// src/pool/pool.cairo:317-319
let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
staking_pool_dispatcher
    .remove_from_delegation_pool_action(identifier: pool_member.into());
```

This means the pause propagates through the pool contract as well, blocking `exit_delegation_pool_action` for all pool members.

The exit wait window is a time-based mechanism: `unstake_time = now + exit_wait_window` (default 1 week, max 12 weeks). Once the window expires, the staker/delegator has a legitimate right to withdraw. However, if the contract is paused at that moment, the withdrawal is blocked indefinitely until the Security Admin (a separate, higher-privilege role) calls `unpause()`. The Security Agent can pause but cannot unpause — only the Security Admin can.

---

### Impact Explanation

**Temporary freezing of funds (High).**

Stakers and delegators who have completed their mandatory exit wait window cannot withdraw their principal while the contract is paused. Their funds are locked in the Staking contract with no recourse until the Security Admin unpauses. This directly violates the protocol's guarantee that funds are returnable after the exit window. In a worst-case scenario (e.g., Security Admin key unavailable, or a malicious Security Agent repeatedly pausing), the freeze could be extended significantly.

---

### Likelihood Explanation

**Medium.**

The Security Agent role is explicitly designed for rapid response to off-chain events (e.g., oracle failures, bridge anomalies). The protocol documentation acknowledges that false positives are acceptable. Any legitimate pause that coincides with — or outlasts — a staker's exit window will trigger this freeze. Given that exit windows are 1–12 weeks and pauses can be triggered at any time by the Security Agent, the overlap is a realistic operational scenario, not a theoretical edge case.

---

### Recommendation

Remove the `general_prerequisites()` call (or at minimum the `assert_is_unpaused()` sub-call) from `unstake_action` and `remove_from_delegation_pool_action`. These are "settlement" functions that finalize a previously committed exit intent; they do not introduce new state changes that a pause is meant to prevent. Allowing them to execute while paused mirrors the fix applied in Reserve Protocol's M-01: settlement of an already-initiated exit should not be gated by the pause mechanism.

Concretely:

```cairo
fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
    // Remove self.general_prerequisites(); here
    assert_caller_is_not_zero();  // keep the zero-address check
    ...
}

fn remove_from_delegation_pool_action(ref self: ContractState, identifier: felt252) {
    // Remove self.general_prerequisites(); here
    assert_caller_is_not_zero();
    ...
}
```

---

### Proof of Concept

**Staker scenario:**

1. Staker calls `unstake_intent()` → `unstake_time = now + exit_wait_window` (e.g., 1 week).
2. One week passes; the exit window expires. Staker is entitled to withdraw.
3. Security Agent calls `pause()` (for a legitimate reason, e.g., oracle anomaly).
4. Staker calls `unstake_action(staker_address)` → reverts: `"Contract is paused"` via `general_prerequisites()` → `assert_is_unpaused()`.
5. Staker's principal is frozen in the Staking contract. No withdrawal is possible until the Security Admin calls `unpause()`.

**Pool member scenario:**

1. Pool member calls `exit_delegation_pool_intent(amount)` on the Pool contract → Pool calls `remove_from_delegation_pool_intent` on Staking → `unpool_time` is set.
2. Exit window expires.
3. Security Agent calls `pause()`.
4. Pool member calls `exit_delegation_pool_action(pool_member)` on the Pool contract → Pool calls `staking_pool_dispatcher.remove_from_delegation_pool_action(...)` → Staking contract reverts: `"Contract is paused"`.
5. Pool member's funds are frozen.

    impl StakingPauseImpl of IStakingPause<ContractState> {
        fn pause(ref self: ContractState) {
            self.roles.only_security_agent();
            if self.is_paused() {
                return;
            }
            self.is_paused.write(true);
            self.emit(PauseEvents::Paused { account: get_caller_address() });
        }

        fn unpause(ref self: ContractState) {
            self.roles.only_security_admin();
            if !self.is_paused() {
                return;
            }
            self.is_paused.write(false);
            self.emit(PauseEvents::Unpaused { account: get_caller_address() });
        }
