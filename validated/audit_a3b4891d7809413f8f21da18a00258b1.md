### Title
Staker with active `CommissionCommitment` can front-run delegator's pool entry to impose a higher commission - (`src/staking/staking.cairo`)

---

### Summary

A staker who holds an active `CommissionCommitment` can call `set_commission` to **increase** their commission rate at any time (up to `max_commission`). A delegator who observes a low commission and submits `enter_delegation_pool` or `switch_delegation_pool` can be front-run by the staker, who raises the commission before the delegator's transaction executes. The delegator ends up locked into a pool at a commission rate they never agreed to, losing a material portion of their yield.

---

### Finding Description

`set_commission` in `src/staking/staking.cairo` delegates to `update_commission` when a commission is already set. Inside `update_commission`, the logic branches on whether a `CommissionCommitment` is active:

```cairo
// src/staking/staking.cairo:1580-1597
if let Option::Some(commission_commitment) = staker_pool_info
    .commission_commitment
    .read() {
    if self.is_commission_commitment_active(:commission_commitment) {
        assert!(
            commission <= commission_commitment.max_commission, ...
        );
        assert!(commission != old_commission, ...);   // only "not same"
    } else {
        assert!(commission < old_commission, ...);    // must decrease
    }
} else {
    assert!(commission < old_commission, ...);        // must decrease
}
```

Without a commitment the commission can only decrease. **With** an active commitment the only constraints are `commission <= max_commission` and `commission != old_commission`. There is no lower-bound check, so the staker is free to **raise** the commission from, say, 5 % to 50 % in a single call, as long as `max_commission ≥ 50 %`.

The protocol's own inline comment acknowledges this: *"**Note**: Current commission increase safeguards still allow for sudden commission changes."* [1](#0-0) 

The commission change takes effect **immediately** in storage. [2](#0-1) 

The pool contract reads the commission dynamically on every query via `get_commission_from_staking_contract`, which calls back into the staking contract at call-time: [3](#0-2) 

Neither `enter_delegation_pool` nor `switch_delegation_pool` snapshot or validate the commission at the moment of entry: [4](#0-3) [5](#0-4) 

---

### Impact Explanation

A delegator who joins (or switches to) a pool expecting commission = 5 % but receives commission = 50 % loses 45 percentage-points of every reward epoch for as long as they remain in the pool. Because the commission is applied to every reward distribution forwarded from the staking contract to the pool, this constitutes **ongoing theft of unclaimed yield**. The staker profits directly from the excess commission at the delegator's expense.

The `switch_delegation_pool` path is especially harmful: the delegator has already exited their previous pool via `exit_delegation_pool_intent` and cannot trivially reverse the switch without waiting another full exit-window period, during which the inflated commission continues to apply.

---

### Likelihood Explanation

The attack requires:
1. The staker to have previously called `set_commission_commitment` with a `max_commission` materially higher than the current commission. This is a public, on-chain action that a delegator *could* inspect, but many delegators rely on the current displayed commission rather than reading the commitment storage slot.
2. The staker to submit a `set_commission` transaction that is ordered before the delegator's `enter_delegation_pool` / `switch_delegation_pool` transaction. On Starknet the sequencer currently orders transactions, so a staker who monitors the pending transaction queue (or who is the sequencer) can reliably achieve this ordering.

The combination is realistic for any staker who wants to attract delegators with a low advertised commission and then extract yield once funds are committed.

---

### Recommendation

Add a **time-delay** (e.g., one epoch or K epochs) before a commission *increase* takes effect, mirroring the delay already applied to balance changes (`epoch + K`). Alternatively, allow delegators to pass a `max_commission` parameter to `enter_delegation_pool` / `switch_delegation_pool` and revert if the current commission exceeds it at execution time — analogous to the fix proposed in the reference report.

```diff
// In enter_delegation_pool / switch_delegation_pool:
+ fn enter_delegation_pool(
+     ref self: ContractState,
+     reward_address: ContractAddress,
+     amount: Amount,
+     max_commission: Commission,   // caller-supplied slippage guard
+ ) {
+     let current_commission = self.get_commission_from_staking_contract();
+     assert!(current_commission <= max_commission, "COMMISSION_EXCEEDS_MAX");
      ...
```

---

### Proof of Concept

1. Staker calls `set_commission(500)` (5 %) — commission is now 500.
2. Staker calls `set_commission_commitment(max_commission: 5000, expiration_epoch: current + 10)` — commitment allows raising to 50 %. [6](#0-5) 
3. Delegator observes commission = 500 (5 %) and submits `enter_delegation_pool(reward_address, amount)`.
4. Staker observes the pending transaction and submits `set_commission(5000)` (50 %) — valid because `5000 <= max_commission` and `5000 != 500`. [7](#0-6) 
5. Sequencer orders the staker's `set_commission` before the delegator's `enter_delegation_pool`.
6. Delegator's transaction executes; they are now a pool member with commission = 50 %. [4](#0-3) 
7. Every subsequent reward epoch, 50 % of the delegator's share is taken by the staker instead of the expected 5 %, constituting continuous theft of unclaimed yield.

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);

                "{}",

            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }

        fn get_commission_from_staking_contract(self: @ContractState) -> Commission {
            if self.staker_removed.read() {
