### Title
Extension Code Mutability via Upgradeable Proxy Allows Post-Deployment Pool Freeze â€” (`metric-core/contracts/libraries/ValidateExtensionsConfig.sol`)

---

### Summary

`ValidateExtensionsConfig.validateExtensionsConfig` is declared `internal pure` and therefore cannot read EVM state. It only checks that extension addresses are non-zero and non-duplicate. It performs no `extcodesize` or `extcodehash` check. An extension supplied as an upgradeable proxy (or, on HyperEVM which may not enforce EIP-6780, a CREATE2-deployed contract with a `SELFDESTRUCT` path) passes all factory validation at pool creation time, but its implementation can be silently replaced after the pool is live. Because extension addresses are baked into the pool as immutables and there is no per-call code-hash guard anywhere in `metric-core/contracts/`, the pool will call whatever code now lives at those addresses.

---

### Finding Description

**Root cause â€” `validateExtensionsConfig` is `pure` and cannot inspect code:** [1](#0-0) 

The only per-address check is `extensions[i] == address(0)`. No `extcodesize` or `extcodehash` is read. The function cannot read them because it is `pure`.

**One-time `initialize` call is the only runtime check â€” and it happens before the pool is used:** [2](#0-1) 

`CallExtension.callExtension` verifies the extension responds with the correct selector during `initialize`. After that, no further code-integrity check is ever performed.

**Extension addresses are frozen as immutables:** [3](#0-2) 

There is no mechanism to update or remove an extension after deployment. The pool is permanently bound to whatever code lives at those addresses.

**Every core operation calls the extension without any code-hash guard:** [4](#0-3) 

`_callExtensionsInOrder` calls `CallExtension.callExtension(extension, data)` unconditionally. A grep across all of `metric-core/contracts/**/*.sol` returns zero matches for `extcodesize`, `extcodehash`, `codesize`, or `codehash`.

**A reverting extension permanently blocks `removeLiquidity`:** [5](#0-4) 

`_beforeRemoveLiquidity` and `_afterRemoveLiquidity` are called inside `removeLiquidity`. If either reverts, the entire transaction reverts and LP funds cannot be withdrawn. The test suite explicitly confirms this behaviour for `beforeSwap`; the same propagation applies to all six hook points.

---

### Impact Explanation

After the extension implementation is replaced:

- **`beforeRemoveLiquidity` or `afterRemoveLiquidity` reverts** â†’ all LP positions are permanently frozen; no principal can be recovered.
- **`beforeSwap` or `afterSwap` reverts** â†’ the pool is completely non-functional for trading.
- **`beforeAddLiquidity` or `afterAddLiquidity` reverts** â†’ new liquidity cannot be added, compounding the freeze.

This satisfies the "Broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows" impact gate.

---

### Likelihood Explanation

`createPool` is permissionless. Any caller can supply any non-zero, non-duplicate address as an extension. A pool creator who uses a third-party extension that happens to be an upgradeable proxy (a common pattern) is exposed. The extension deployer â€” a party entirely outside the Metric OMM trust model â€” can upgrade the implementation at any time after pool creation and after LPs have deposited funds. No timelock, no factory consent, and no on-chain signal is required.

---

### Recommendation

1. **Snapshot `extcodehash` at creation time.** Change `validateExtensionsConfig` to `internal view`, record `extcodehash(extensions[i])` for each extension in factory storage during `createPool`, and verify the hash has not changed inside `_callExtensionsInOrder` before each call.
2. **Alternatively, require extensions to be deployed by a factory-controlled deployer** (analogous to how `MetricOmmPoolDeployer` is the only path to deploy pools), guaranteeing no upgradeable proxy can be registered.
3. **Minimum viable mitigation:** add an `extcodesize(extensions[i]) > 0` check inside `validateExtensionsConfig` (requires changing it to `view`) to at least reject EOA addresses, and document that upgradeable-proxy extensions are unsupported.

---

### Proof of Concept

```solidity
// 1. Attacker deploys a benign upgradeable proxy extension.
//    The proxy's initialize() returns the correct selector â€” passes factory validation.
address benignImpl = address(new BenignExtension());
TransparentUpgradeableProxy ext = new TransparentUpgradeableProxy(benignImpl, attacker, "");

// 2. Legitimate pool creator creates a pool using this extension.
//    validateExtensionsConfig passes (non-zero, no duplicates).
//    CallExtension.callExtension(ext, initialize(...)) succeeds.
address pool = factory.createPool(paramsWithExtension(address(ext)));

// 3. LPs deposit funds.
pool.addLiquidity(...);

// 4. Attacker upgrades the proxy to a malicious implementation.
//    No factory interaction required.
address maliciousImpl = address(new MaliciousExtension()); // beforeRemoveLiquidity always reverts
proxyAdmin.upgrade(ext, maliciousImpl);

// 5. LPs attempt to withdraw â€” permanently reverts.
pool.removeLiquidity(...); // reverts: MaliciousExtension.beforeRemoveLiquidity() reverts
// LP principal is permanently frozen in the pool.
```

  function validateExtensionsConfig(
    address[] calldata extensions,
    ExtensionOrders calldata orders,
    bytes[] calldata extensionInitData
  ) internal pure {
    if (extensions.length > MAX_EXTENSIONS) revert InvalidExtensionsConfig();
    if (extensionInitData.length != extensions.length) revert InvalidExtensionsConfig();

    if (extensions.length == 0) {
      if (!_ordersAreZero(orders)) revert InvalidExtensionsConfig();
      return;
    }

    for (uint256 i = 0; i < extensions.length; i++) {
      if (extensions[i] == address(0)) revert InvalidExtensionsConfig();
      for (uint256 j = 0; j < i; j++) {
        if (extensions[i] == extensions[j]) revert InvalidExtensionsConfig();
      }
    }

  function _callExtensionsInOrder(uint256 order, bytes memory data) private {
    if (order == 0) return;

    while (true) {
      uint256 extensionIndex = order & 0x7;
      if (extensionIndex == 0) break;
      address extension = _extensionAddress(extensionIndex);
      if (extension == address(0)) revert PanicEmptyExtension();
      CallExtension.callExtension(extension, data);
      order >>= 3;
    }
  }
