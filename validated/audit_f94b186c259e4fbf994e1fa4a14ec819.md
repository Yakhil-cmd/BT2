### Title
Pool Admin Bypasses Fee Cap System via Uncapped `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6` and `addFeeSellE6` directly to the pool with **no upper-bound check**, while the parallel admin fee setter `setPoolAdminFees` enforces `maxAdminSpreadFeeE6`. A pool admin can set per-bin fees up to `uint16` max (65 535 ≈ 6.55 % in E6 units) on any bin, bypassing the factory's intended fee-cap invariant and causing traders to pay fees above the protocol-configured maximum.

### Finding Description

`setPoolAdminFees` enforces the cap:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees` has **no such check** — it passes the caller-supplied values straight through:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` also performs no cap check — only a bin-index range check:

```solidity
// MetricOmmPool.sol:464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    ...
}
``` [3](#0-2) 

These per-bin fees are **directly additive** to the oracle-derived base fee in every swap path:

```solidity
// MetricOmmPool.sol:910 (buyToken0, specifiedOut)
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
// MetricOmmPool.sol:1088 (buyToken1, specifiedOut)
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)
``` [4](#0-3) [5](#0-4) 

The same pattern applies to the specified-input paths at lines 999 and 1177. [6](#0-5) 

The hard cap for global spread fees is `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20 %), and `maxAdminSpreadFeeE6` is bounded by that. [7](#0-6) 

A pool admin can set `addFeeBuyE6 = 65535` (uint16 max ≈ 6.55 %) on every bin, adding up to 6.55 % on top of whatever the global spread fee already is — with no factory-level enforcement.

### Impact Explanation

Every swap through an affected bin pays a higher effective fee than the protocol's cap permits. The excess fee accrues as pool surplus (split between admin and protocol via `collectFees`), meaning the pool admin directly profits from the overcharge. Traders lose principal above the documented maximum. This is a direct, quantifiable loss of user funds caused by an admin-boundary break.

### Likelihood Explanation

The pool admin role is semi-trusted and is explicitly scoped as "semi-trusted only inside caps and timelocks." The call path is a single permissioned transaction with no timelock, no proposal step, and no on-chain guard. Any pool admin can execute this immediately after pool deployment.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` mirroring the one in `setPoolAdminFees`. For example, introduce a `maxAdminBinFeeE6` storage variable (bounded by `HARD_MAX_SPREAD_FEE_E6`) and revert if either `addFeeBuyE6` or `addFeeSellE6` exceeds it:

```solidity
if (addFeeBuyE6 > maxAdminBinFeeE6) revert AdminFeeTooHigh();
if (addFeeSellE6 > maxAdminBinFeeE6) revert AdminFeeTooHigh();
```

Alternatively, enforce that `addFeeBuyE6` and `addFeeSellE6` cannot exceed `maxAdminSpreadFeeE6` (reusing the existing cap).

### Proof of Concept

```solidity
// Foundry test sketch
function test_binFeeBypassesCap() public {
    address pool = _createPool(); // admin = address(this)

    // Set bin 0 buy fee to uint16 max (~6.55%)
    factory.setPoolBinAdditionalFees(pool, 0, 65535, 65535);
    // No revert — factory accepts uncapped value

    (,,, uint16 buyFee, uint16 sellFee) = PoolStateLibrary._binState(pool, 0);
    assertEq(buyFee, 65535); // 6.55% additional fee, above any admin cap

    // A subsequent swap through bin 0 pays baseFee + 6.55%,
    // exceeding maxAdminSpreadFeeE6 without any revert.
}
```

The existing test `test_setPoolBinAdditionalFees_updatesStorage_emitsEvent` already confirms the factory accepts arbitrary `uint16` values (e.g., 500, 700) with no cap enforcement. [8](#0-7)
