### Title
Unprivileged caller can permanently suppress all consensus block rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` has no access control beyond a non-zero caller check. It accepts a caller-controlled `disable_rewards: bool` parameter. Critically, the global `last_reward_block` storage variable is written **before** the `disable_rewards` guard, so any unprivileged caller can consume the per-block reward slot without distributing any rewards, permanently destroying that block's yield for all stakers.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero address — no role or identity check. [1](#0-0) 

The function enforces a one-call-per-block invariant using the global `last_reward_block`: [2](#0-1) 

After validating the staker is active, it unconditionally writes the current block number to `last_reward_block`: [3](#0-2) 

Only **after** that write does it check `disable_rewards`: [4](#0-3) 

Because `last_reward_block` is a single global slot shared across all stakers: [5](#0-4) 

once an attacker calls `update_rewards(any_active_staker, disable_rewards: true)` in a block, every subsequent call in that same block reverts with `REWARDS_ALREADY_UPDATED`. The block's reward opportunity is permanently consumed and no yield is ever accrued.

---

### Impact Explanation

This matches **High: Permanent freezing of unclaimed yield**.

Consensus block rewards are calculated and distributed exactly once per block via `update_rewards`. If the slot is consumed with `disable_rewards: true`, the reward supplier's funds for that block are never claimed and never credited to any staker or pool. The loss is irreversible — there is no retroactive reward mechanism. An attacker who calls this every block eliminates 100% of consensus-phase yield for all stakers and delegators indefinitely.

---

### Likelihood Explanation

The attack requires only:
1. Knowledge of any one active staker address (publicly observable from `NewStaker` events or `get_stakers()`).
2. Sending one transaction per block with `disable_rewards: true`.

There is no financial cost beyond gas fees (Starknet L2 fees are low). There is no profit motive required — a competing protocol, a censoring party, or a malicious actor with a grievance can sustain this indefinitely. The attack is fully permissionless and requires no privileged access.

---

### Recommendation

1. **Restrict who may call `update_rewards`**: Only the designated block proposer (identified via the consensus layer or a registered operator role) should be permitted to call this function.
2. **Move the `last_reward_block` write after the `disable_rewards` guard**: If `disable_rewards` is true, do not consume the per-block slot — return without updating `last_reward_block` so a legitimate caller can still distribute rewards in the same block.
3. **Remove or gate the `disable_rewards` parameter**: If the parameter is needed for operational reasons, restrict it to a privileged role (e.g., `security_agent` or `app_governor`).

---

### Proof of Concept

```
// Attacker script (pseudocode, one call per block):
loop every block:
    staking_contract.update_rewards(
        staker_address = <any_active_staker>,  // publicly known
        disable_rewards = true
    )
    // Effect:
    //   last_reward_block = current_block  (slot consumed)
    //   no rewards distributed
    //   all subsequent update_rewards calls in this block revert
```

Concrete call path:
1. Attacker calls `update_rewards(active_staker, true)` — passes `general_prerequisites()`, passes staker-active check, writes `last_reward_block = N`, hits `if disable_rewards { return; }`.
2. Block proposer calls `update_rewards(staker, false)` in the same block — hits `assert!(current_block_number > last_reward_block)` → **reverts**.
3. No `_update_rewards` is ever reached; `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` is never called; staker `unclaimed_rewards_own` is never incremented.
4. Repeated every block → all consensus-phase yield is permanently destroyed. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
