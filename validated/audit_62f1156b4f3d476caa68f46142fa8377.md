### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped Per-Bin Additional Fees — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolBinAdditionalFees` passes `addFeeBuyE6` / `addFeeSellE6` directly to the pool with no factory-level cap check, while the parallel function `setPoolAdminFees` enforces `maxAdminSpreadFeeE6`. A pool admin can set per-bin fees to `uint16` max (65535), adding 6.5535% extra fee per bin on top of the already-capped global spread, causing traders to pay fees far above the documented factory ceiling.

---

### Finding Description

`setPoolAdminFees` enforces the factory cap:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

But `setPoolBinAdditionalFees` has no such check — it passes the raw `uint16` values straight through:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` only validates the bin index, not the fee values:

```solidity
// MetricOmmPool.sol:464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6) external onlyFactory ... {
  if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
  BinState storage s = _binStates[bin];
  s.addFeeBuyE6 = addFeeBuyE6;
  s.addFeeSellE6 = addFeeSellE6;
``` [3](#0-2) 

These stored values are then added directly to `baseFeeX64` in every swap path through that bin:

```solidity
// MetricOmmPool.sol:540-541 (getBidAndAskPrice)
uint256 buyFeeX64  = baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6,  ONE_X64, 1e6);
uint256 sellFeeX64 = baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6);
``` [4](#0-3) 

And identically in the live swap loops:

```solidity
// MetricOmmPool.sol:910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6),
``` [5](#0-4) 

```solidity
// MetricOmmPool.sol:1088
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6),
``` [6](#0-5) 

Setting `addFeeBuyE6 = addFeeSellE6 = 65535` injects `65535 / 1e6 ≈ 6.5535%` additional fee per bin, completely outside the `maxAdminSpreadFeeE6` cap that is supposed to bound the pool admin's fee authority.

---

### Impact Explanation

Traders swapping through any bin where the pool admin has set `addFeeBuyE6` / `addFeeSellE6` to the uint16 maximum pay up to **6.5535% extra fee** above the global spread cap on every swap through that bin. This is a direct, quantifiable loss of trader principal. The fee is silently extracted — the trader receives less output than the oracle-permitted price would allow, with no on-chain enforcement preventing it.

---

### Likelihood Explanation

Any `poolAdmin[pool]` can execute this in a single transaction with no timelock, no co-signer, and no protocol-owner involvement. The pool admin role is assigned at pool creation and is transferable. A malicious or compromised admin can activate this immediately on any active pool they control.

---

### Recommendation

Add a factory-level cap check in `setPoolBinAdditionalFees`, mirroring the pattern used in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6  > maxAdminBinFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminBinFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

The cap value (`maxAdminBinFeeE6`) should be set by the factory owner and bounded by the same hard ceiling as `maxAdminSpreadFeeE6`, or alternatively reuse `maxAdminSpreadFeeE6` directly as the per-bin ceiling.

---

### Proof of Concept

```solidity
// Pool admin sets bin 0 additional fees to uint16 max
vm.prank(poolAdmin);
factory.setPoolBinAdditionalFees(pool, 0, 65535, 65535);

// Trader swaps through bin 0
uint256 amountIn = 1e18;
uint256 amountOut = pool.swap(...); // swap through bin 0

// amountOut is reduced by ~6.5535% beyond the global spread cap
// assert amountOut < oracle_fair_output * (1 - maxAdminSpreadFeeE6/1e6)
```

The pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)`. Any subsequent swap routed through bin 0 computes `buyFeeX64 = baseFeeX64 + (65535 * ONE_X64 / 1e6)`, charging the trader 6.5535% above the base spread. The factory's `maxAdminSpreadFeeE6` cap is never consulted for this path.
