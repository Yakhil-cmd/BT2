### Title
Missing Permission Check and No-Op `TurnBridge` Handler Permanently Disables Bridge Circuit Breaker - (File: x/cronos/keeper/msg_server.go)

### Summary
The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is a stub that unconditionally returns `nil, nil` â€” it performs no permission check and makes no state change. Any unprivileged user can call `MsgTurnBridge` and receive a success response, bypassing the `CanTurnBridge` permission entirely. More critically, because the implementation is empty, the bridge circuit breaker is permanently non-functional: even the admin cannot disable the Gravity bridge in an emergency.

### Finding Description

The `x/cronos` module defines a permission system with two privileged operations: `CanChangeTokenMapping` and `CanTurnBridge`. The ADR-009 design explicitly requires that `MsgTurnBridge` be restricted to addresses holding the `CanTurnBridge` permission.

`UpdateTokenMapping` correctly enforces this:

```go
// x/cronos/keeper/msg_server.go:69-82
func (k msgServer) UpdateTokenMapping(...) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    ...
}
```

But `TurnBridge` does not:

```go
// x/cronos/keeper/msg_server.go:84-87
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

The handler is a live, registered gRPC endpoint: [2](#0-1) 

The permission constant `CanTurnBridge` is defined but never checked in the handler: [3](#0-2) 

The ADR-009 design intent is explicit â€” `MsgTurnBridge` must be permission-gated: [4](#0-3) 

### Impact Explanation

Two distinct impacts arise:

1. **Auth bypass (High)**: Any unprivileged user can submit `MsgTurnBridge` with any `sender` address and receive a success response. The `CanTurnBridge` permission â€” the only authorization guard for bridge control â€” is completely bypassed. This directly matches: *"High: Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks."*

2. **Permanent bridge circuit-breaker failure (High)**: Because the handler is a no-op, the Gravity bridge state is never modified. Even the admin or a permissioned address cannot disable the bridge in an emergency (e.g., a bridge exploit draining funds). This matches: *"High: Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows..."*

### Likelihood Explanation

The `MsgTurnBridge` message is a registered, publicly reachable Cosmos SDK transaction endpoint. Any account on the network can submit it. No special conditions, tokens, or privileges are required. The missing implementation is unconditional â€” it affects every invocation.

### Recommendation

Implement `TurnBridge` with the same permission pattern used by `UpdateTokenMapping`:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    sender, err := sdk.AccAddressFromBech32(msg.Sender)
    if err != nil {
        return nil, err
    }
    if !k.HasPermission(ctx, []sdk.AccAddress{sender}, CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // implement actual bridge enable/disable state change
    ...
    return &types.MsgTurnBridgeResponse{}, nil
}
```

The `HasPermission` helper already handles both the admin shortcut and the per-address permission bitmask: [5](#0-4) 

### Proof of Concept

1. Any unprivileged account `attacker` submits:
   ```
   cronosd tx cronos turn-bridge true --from attacker
   ```
2. The transaction is accepted and returns code `0` (success), despite `attacker` holding no `CanTurnBridge` permission.
3. The bridge state is unchanged â€” the call is a no-op.
4. The admin subsequently submits the same message to disable the bridge in an emergency â€” it also succeeds with code `0` but has no effect. The bridge cannot be disabled.

The `CanTurnBridge` permission bit is defined and the `HasPermission` infrastructure is fully functional: [6](#0-5) 

but is never consulted by the `TurnBridge` handler: [1](#0-0)

func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
