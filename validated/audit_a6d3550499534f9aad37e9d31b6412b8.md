Audit Report

## Title
SNS Swap Canister Permanently Locks Pre-Loaded SNS Tokens on Abort - (`rs/sns/swap/src/swap.rs`)

## Summary
When an SNS decentralization swap reaches `LIFECYCLE_ABORTED`, `finalize_inner` returns early after sweeping ICP back to buyers and restoring dapp controllers, but never transfers the pre-loaded SNS tokens back to the SNS governance canister. Those tokens remain permanently locked in the swap canister's SNS ledger account. Because the lifecycle diagram shows `ABORTED → <DELETED>`, the swap canister is expected to be deleted after abort, making the locked tokens permanently inaccessible.

## Finding Description
The swap canister is pre-funded with SNS tokens in the `PENDING` state before the swap opens, as documented in the proto file. [1](#0-0) 

`finalize_inner` handles both the `COMMITTED` and `ABORTED` paths. On the `ABORTED` path, after `sweep_icp` and `settle_neurons_fund_participation`, `should_restore_dapp_control()` returns `true`: [2](#0-1) 

This causes `finalize_inner` to call `restore_dapp_controllers_for_finalize` and then **return early**, skipping `sweep_sns` entirely: [3](#0-2) 

`sweep_sns` itself has a hard guard that rejects any call when lifecycle is not `Committed`: [4](#0-3) 

The `sweep_sns` call that distributes SNS tokens only appears in the `COMMITTED` branch, which is never reached on abort: [5](#0-4) 

There is no other function in the swap canister API that transfers SNS tokens back to governance. The proto documentation for the `ABORTED` state only describes ICP being returned: [6](#0-5) 

The proto also states "The `swap` canister can be deleted when all tokens registered with the `swap` canister have been disbursed to their rightful owners," but no code path disburses SNS tokens on abort. ICRC-1 has no admin-transfer capability, so SNS governance cannot forcibly recover tokens from the swap canister's ledger account.

## Impact Explanation
Every SNS swap that reaches `LIFECYCLE_ABORTED` permanently destroys the entire pre-loaded SNS token allocation. For any SNS project with a meaningful token valuation, this allocation can easily exceed $1M. This constitutes permanent loss of in-scope SNS ledger assets — a protocol-level accounting invariant violation. The lifecycle diagram confirms the swap canister is expected to be deleted after abort, making recovery impossible once deletion occurs.

**Severity: High** (with potential Critical escalation for large SNS projects where the token allocation exceeds $1M).

## Likelihood Explanation
Swap abortion is a normal, expected protocol outcome triggered whenever `min_participants` or `min_icp_e8s` is not reached before the swap deadline. No special privileges are required — any unprivileged user can cause a swap to abort simply by not participating (or participating below the minimum threshold). This is fully reachable on mainnet for any SNS project whose decentralization swap fails to attract sufficient participation.

## Recommendation
In `finalize_inner`, before returning early on the `ABORTED` path, add a step to transfer the remaining SNS token balance from the swap canister's SNS ledger account back to the SNS governance canister's treasury account. This mirrors how `sweep_icp` returns ICP to buyers on abort:

```rust
if self.should_restore_dapp_control() {
    // Return pre-loaded SNS tokens to SNS governance treasury
    self.sweep_sns_to_governance(now_fn, environment.sns_ledger()).await;

    finalize_swap_response.set_set_dapp_controllers_result(
        self.restore_dapp_controllers_for_finalize(environment.sns_root_mut()).await,
    );
    return finalize_swap_response;
}
```

A new `sweep_sns_to_governance` function should transfer the full SNS token balance held by the swap canister's default account back to the SNS governance canister principal, analogous to how `sweep_icp` handles ICP refunds.

## Proof of Concept
1. An SNS is created and its swap canister is pre-loaded with `N` SNS tokens in `PENDING` state.
2. The swap opens (`OPEN` state) but fails to reach `min_icp_e8s` or `min_participants` before the deadline.
3. `try_abort` is called, transitioning the swap to `ABORTED`.
4. `finalize` is called. `finalize_inner` executes:
   - `sweep_icp` returns all ICP to buyers.
   - `should_restore_dapp_control()` returns `true` (lifecycle is `Aborted`).
   - `restore_dapp_controllers_for_finalize` is called.
   - Function returns early — `sweep_sns` is never invoked.
5. Query the SNS ledger for the swap canister's default account balance: it still holds `N` SNS tokens minus any fees.
6. No callable function exists to recover these tokens.
7. Delete the swap canister (as the lifecycle diagram prescribes for `ABORTED`): the tokens are permanently inaccessible.

A deterministic integration test using PocketIC can reproduce this by: initializing a swap, pre-funding it with SNS tokens, advancing time past the deadline without meeting minimum participation, calling `finalize`, and asserting that the swap canister's SNS ledger balance is non-zero after finalization.
