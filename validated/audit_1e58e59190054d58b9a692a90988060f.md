### Title
Unrestricted `disable_rewards` Parameter in `update_rewards` Allows Any Caller to Grief All Stakers Out of Block Rewards - (File: src/staking/staking.cairo)

### Summary
The public `update_rewards` function in `staking.cairo` accepts a caller-controlled `disable_rewards: bool` parameter. When set to `true`, the function still writes the current block number to the global `last_reward_block` storage variable but returns before performing any reward calculation. Because `last_reward_block` is a single global gate that enforces "at most one reward update per block", any unprivileged caller can permanently consume that gate for a given block without distributing rewards, silently denying all stakers their block rewards for that block.

### Finding Description
`update_rewards` is exposed as a fully public ABI entry-point via `#[abi(embed_v0)]` on `StakingRewardsManagerImpl`. Its only access guard is `general_prerequisites()`, which checks only that the contract is not paused and the caller is non-zero.

The critical ordering inside the function is:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← gate consumed here

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← reward skipped here
}
```

`last_reward_block` is written **before** the `disable_rewards` branch is evaluated. Once written, the guard at the top of the function:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

prevents any subsequent call in the same block from succeeding. Therefore a single call with `disable_rewards = true` irrevocably consumes the reward slot for that block for every staker in the protocol.

The attacker only needs to supply any currently-active staker address (publicly readable from the `stakers` Vec) to pass the staker-existence checks.

### Impact Explanation
Every block in the consensus-rewards phase for which the attacker fires this transaction, **all stakers and all delegation-pool members** receive zero block rewards. The attacker pays only the gas cost of one transaction per block; the aggregate reward loss across all stakers is proportional to the total staked value and the block reward rate. This constitutes griefing with no profit motive but measurable, ongoing damage to users and the protocol.

This matches the allowed impact: **Griefing with no profit motive but damage to users or protocol (Medium)**.

### Likelihood Explanation
The function is callable by any non-zero address with no role requirement. The only prerequisite is knowing one valid, active staker address, which is trivially obtained from on-chain state. The attack is cheap (one L2 transaction per block) and requires no capital. It is executable from the first block of the consensus-rewards phase onward.

### Recommendation
Move the `last_reward_block` write to **after** the early-return guard, so that a `disable_rewards = true` call does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only consume the gate when rewards are actually being processed.
self.last_reward_block.write(current_block_number);
// ... reward calculation ...
```

Alternatively, restrict who may pass `disable_rewards = true` (e.g., only the staker itself or a privileged role), or remove the parameter entirely and derive the disable condition from on-chain state.

### Proof of Concept
1. Consensus-rewards phase is active (`is_pre_consensus()` returns `false`).
2. Attacker reads any active staker address `S` from the public `stakers` storage vector.
3. At block `N`, attacker calls `update_rewards(staker_address: S, disable_rewards: true)`.
4. Inside the function: `last_reward_block` is written to `N`; execution returns early — no rewards are calculated or distributed.
5. Any legitimate call to `update_rewards` for block `N` now reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers receive zero block rewards for block `N`.
7. Attacker repeats step 3 every block.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
