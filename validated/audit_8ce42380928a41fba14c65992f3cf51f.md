### Title
Missing Caller Access Control on `update_rewards` Allows Any Address to Block Reward Distribution - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the staking contract is documented as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` every block, updating `last_reward_block` without distributing rewards, permanently preventing the legitimate sequencer from distributing block rewards to stakers.

### Finding Description
The spec explicitly states the access control for `update_rewards`:

> **access control**: Only starkware sequencer. [1](#0-0) 

However, the implementation at `StakingRewardsManagerImpl::update_rewards` performs no caller identity check whatsoever: [2](#0-1) 

The function writes `last_reward_block` to the current block number unconditionally before the `disable_rewards` branch: [3](#0-2) 

Because `last_reward_block` is a single contract-wide storage slot, once it is written for a given block, the guard at line 1454–1458 causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` first in every block permanently occupies the slot without distributing any rewards, and the sequencer's legitimate call is rejected for that block.

### Impact Explanation
An unprivileged attacker can grief the entire protocol by front-running the sequencer's `update_rewards` call in every block with `disable_rewards: true`. This prevents all stakers and their delegation pools from accumulating block rewards indefinitely. The impact is **permanent freezing of unclaimed yield** for all stakers — a High-severity impact under the allowed scope.

### Likelihood Explanation
The entry path requires no special role, no funds, and no privileged access. Any Starknet account can submit the transaction. The attack is cheap (one call per block), fully permissionless, and can be automated. Likelihood is **High**.

### Recommendation
Add an explicit sequencer-only guard at the top of `update_rewards`, analogous to the pattern used in `update_unclaimed_rewards_from_staking_contract` and `claim_rewards` in the reward supplier: [4](#0-3) 

Concretely, store the authorized sequencer address in contract storage during initialization and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation in `update_rewards`.

### Proof of Concept
1. Deploy the system normally; wait until consensus rewards are active (`!is_pre_consensus()`).
2. At the start of every new block, submit a transaction calling:
   ```
   IStakingRewardsManager(staking_contract).update_rewards(
       staker_address: <any_valid_staker>,
       disable_rewards: true
   )
   ```
3. The call succeeds (no caller check), writes `last_reward_block = current_block`, and returns without distributing rewards.
4. The sequencer's legitimate `update_rewards(..., disable_rewards: false)` call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Stakers and pool members accumulate zero `unclaimed_rewards_own` indefinitely.

The missing check is confirmed by the absence of any `assert!(get_caller_address() == ...)` in the function body at: [5](#0-4)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
