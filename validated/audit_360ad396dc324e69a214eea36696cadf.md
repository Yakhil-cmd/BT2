### Title
Staker Can Instantly Raise Commission to 100% via `set_commission_commitment` + `set_commission`, Stealing Delegator Yield — (`src/staking/staking.cairo`)

---

### Summary

A staker can atomically raise their delegation pool commission from 0% to 100% with no advance notice to delegators, by combining `set_commission_commitment` (with `expiration_epoch = current_epoch + 1`) and `set_commission` in the same block. All future rewards earned by delegators are then redirected to the staker. Delegators cannot exit immediately — they must wait through the `exit_wait_window` — during which they earn zero yield. The codebase itself acknowledges this gap with an inline comment.

---

### Finding Description

The `set_commission_commitment` function enforces only that `expiration_epoch > current_epoch`, meaning the minimum valid commitment window is a single epoch boundary. A staker can set `max_commission = 10000` (100%) and `expiration_epoch = current_epoch + 1`, then immediately call `set_commission(commission: 10000)` in the same block. The `update_commission` internal function permits any commission value `<= max_commission` while the commitment is active, with no lower bound or delay requirement.

The code itself carries an explicit acknowledgment of this gap:

> **Note**: Current commission increase safeguards still allow for sudden commission changes. [1](#0-0) 

**Step 1 — `set_commission_commitment` accepts `expiration_epoch = current_epoch + 1`:** [2](#0-1) 

**Step 2 — `is_commission_commitment_active` returns `true` immediately after the call:** [3](#0-2) 

**Step 3 — `update_commission` allows any value up to `max_commission` while active, with no floor:** [4](#0-3) 

**Step 4 — `set_commission` calls `update_commission` with no additional delay guard:** [5](#0-4) 

---

### Impact Explanation

Once commission is raised to 100%, every reward epoch distributes 0% to delegators and 100% to the staker's reward address. Delegators cannot exit without first calling `exit_delegation_pool_intent` and then waiting through the full `exit_wait_window` before `exit_delegation_pool_action` is available. All yield accrued during that window is stolen.

This matches the allowed impact: **High — Theft of unclaimed yield**. [6](#0-5) 

---

### Likelihood Explanation

- Any registered staker can execute this attack — no governance role, leaked key, or external dependency is required.
- The attack is two sequential transactions (or two calls in the same block) with no preconditions beyond having an active pool.
- The incentive is clear: a staker who attracted large delegations at 0% commission can extract all future yield in a single epoch.
- The codebase comment at line 745–746 confirms the protocol team is aware the safeguard is incomplete. [1](#0-0) 

---

### Recommendation

Enforce a minimum delay between `set_commission_commitment` and the first `set_commission` call that raises commission. Specifically:

1. Require `expiration_epoch >= current_epoch + MIN_NOTICE_EPOCHS` (e.g., at least 2–4 epochs), giving delegators time to observe the pending increase and exit before it takes effect.
2. Alternatively, make the commission increase itself only take effect at `expiration_epoch`, not immediately after `set_commission_commitment` is set.

This mirrors the recommended fix in the external report: a mandatory delay window before a fee increase becomes enforceable. [7](#0-6) 

---

### Proof of Concept

```
// Epoch N, staker has commission = 0%, pool has many delegators.

// Tx 1 (or same block as Tx 2):
staking.set_commission_commitment(
    max_commission: 10000,          // 100%
    expiration_epoch: current_epoch + 1   // minimum allowed
);
// Commitment is immediately active: is_commission_commitment_active returns true
// because current_epoch (N) < expiration_epoch (N+1).

// Tx 2 (same block):
staking.set_commission(commission: 10000);
// update_commission: commitment is active, 10000 <= max_commission (10000), 10000 != 0 → passes.
// Commission is now 100%.

// All subsequent reward epochs: delegators receive 0 yield.
// Delegators must call exit_delegation_pool_intent and wait exit_wait_window
// before they can recover principal — all yield during that window is lost.
``` [8](#0-7) [2](#0-1)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }
