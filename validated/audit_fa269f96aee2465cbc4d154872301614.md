### Title
Missing Cairo1-Only Guard in `DeprecatedSyscallHintProcessor::replace_class` Allows Cairo0 Contract to Write Cairo0 Class Hash via `replace_class` Syscall â€” (`crates/blockifier/src/execution/deprecated_syscalls/hint_processor.rs`)

### Summary

`SyscallBase::replace_class` (the Cairo1 path) enforces that the replacement class must be Cairo1. The `DeprecatedSyscallHintProcessor::replace_class` (the Cairo0 path) is missing this check entirely. A Cairo0 contract can call `replace_class` with any declared Cairo0 class hash and succeed, writing a Cairo0 class hash into the contract's address slot â€” a state mutation the protocol explicitly forbids.

### Finding Description

**Cairo1 path** (`SyscallBase::replace_class`, `crates/blockifier/src/execution/syscalls/syscall_base.rs`):

```rust
pub fn replace_class(&mut self, class_hash: ClassHash) -> SyscallResult<()> {
    // Ensure the class is declared (by reading it), and of type V1.
    let compiled_class = self.state.get_compiled_class(class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(SyscallExecutionError::ForbiddenClassReplacement { class_hash });
    }
    self.state.set_class_hash_at(self.call.storage_address, class_hash)?;
    Ok(())
}

### Impact Explanation

An unprivileged user who controls (or deploys) a Cairo0 contract can:

1. Declare any Cairo0 class on-chain (standard `declare` transaction, no privilege required).
2. Invoke their Cairo0 contract's entry point that calls `replace_class` with the declared Cairo0 class hash.
3. `DeprecatedSyscallHintProcessor::replace_class` calls `get_compiled_class` (succeeds â€” class is declared), then calls `set_class_hash_at` with the Cairo0 class hash (succeeds â€” no type check).
4. The contract's class hash slot now holds a Cairo0 class hash chosen by the attacker.

Concrete corrupted values:
- `state.get_class_hash_at(contract_address)` returns the attacker-chosen Cairo0 class hash.
- Any subsequent RPC `starknet_getClassHashAt`, simulation, fee estimation, or trace that reads the contract's class hash returns the wrong value.
- Any subsequent call to the contract dispatches to the attacker-chosen Cairo0 class, not the original class â€” wrong contract code selected for execution.

This directly matches two allowed impact categories:
- **Critical**: Wrong compiled class / contract code selected for execution.
- **High**: RPC simulation/view returns an authoritative-looking wrong value.

### Likelihood Explanation

- No privilege required: any user can deploy a Cairo0 contract and declare a Cairo0 class.
- The `replace_class` syscall selector is explicitly handled in `execute_deprecated_syscall_from_selector` and routed to the vulnerable handler. [6](#0-5) 
- Cairo0 contracts remain deployable on Starknet mainnet.
- The missing check is a single `if !is_cairo1` guard â€” there is no compensating control elsewhere in the deprecated execution path.

### Recommendation

Add the same `is_cairo1` guard to `DeprecatedSyscallHintProcessor::replace_class`:

```rust
fn replace_class(
    request: ReplaceClassRequest,
    _vm: &mut VirtualMachine,
    syscall_handler: &mut Self,
) -> DeprecatedSyscallResult<ReplaceClassResponse> {
    // Ensure the class is declared (by reading it), and of type V1.
    let compiled_class = syscall_handler.state.get_compiled_class(request.class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(/* DeprecatedSyscallExecutionError equivalent of ForbiddenClassReplacement */);
    }
    syscall_handler
        .state
        .set_class_hash_at(syscall_handler.storage_address, request.class_hash)?;
    Ok(ReplaceClassResponse {})
}
```

### Proof of Concept

Build a Rust unit test in `crates/blockifier`:

1. Declare a Cairo0 class (e.g., `FeatureContract::Empty(CairoVersion::Cairo0)`) and a Cairo0 "attacker" contract (e.g., one whose entry point calls `replace_class` with the target class hash).
2. Deploy the attacker Cairo0 contract.
3. Execute the entry point that calls `replace_class` with the declared Cairo0 class hash via `DeprecatedSyscallHintProcessor`.
4. Assert the call returns `Ok(())` (no error).
5. Assert `state.get_class_hash_at(attacker_contract_address)` returns the Cairo0 class hash â€” confirming the forbidden state mutation succeeded.

The existing `cairo0_class_hash` test at line 33 of `replace_class.rs` already demonstrates the Cairo1 path rejects this. The same test structure applied to a Cairo0 caller contract will demonstrate the deprecated path accepts it. [5](#0-4)
