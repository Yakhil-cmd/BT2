### Title
`last_reward_block` Updated Before Condition Check Allows Any Caller to Permanently Deny Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in `src/staking/staking.cairo` writes `last_reward_block` to the current block number **before** checking whether rewards should actually be distributed (`disable_rewards || is_pre_consensus()`). Because the function has no access control (only a non-zero caller check), any unprivileged address can call it with `disable_rewards=true` to consume the per-block slot without distributing any rewards, permanently preventing the sequencer from distributing rewards for that block.

---

### Finding Description

In `StakingRewardsManagerImpl::update_rewards` the ordering is:

```
// line 1485 — state written unconditionally
self.last_reward_block.write(current_block_number);

// line 1487 — condition checked AFTER the write
if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [1](#0-0) 

The deduplication guard at the top of the function rejects any second call in the same block:

```
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

`general_prerequisites()` — the only gate — only checks pause state and a non-zero caller; it does **not** restrict the caller to the Starkware sequencer:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [3](#0-2) 

The specification explicitly states access should be "Only starkware sequencer": [4](#0-3) 

The combination of (a) missing access control and (b) the state write preceding the condition check means any non-zero address can call `update_rewards(valid_staker_address, disable_rewards=true)`, mark the current block as "already processed," and return without distributing any rewards. The sequencer's subsequent call in the same block fails with `REWARDS_ALREADY_UPDATED`, and that block's rewards are permanently lost.

---

### Impact Explanation

Each Starknet block for which the attacker front-runs the sequencer results in **permanently lost block rewards** for all stakers and delegators. The rewards that would have been distributed via `_update_rewards` — updating `unclaimed_rewards_own` and transferring STRK to delegation pools — are never computed or transferred. [5](#0-4) 

This matches the allowed impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is publicly callable by any non-zero address with no token cost beyond gas.
- The attacker only needs to supply any currently-active staker address with non-zero balance (all publicly observable on-chain).
- The attack is repeatable every block; a single attacker transaction per block is sufficient to deny all stakers their consensus rewards indefinitely.
- The required staker validity checks (exists, active, non-zero balance at current epoch) are trivially satisfied using any live staker: [6](#0-5) 

---

### Recommendation

1. **Enforce access control**: Add a sequencer-only check at the top of `update_rewards`, consistent with the specification.
2. **Move the state write after the condition check** (analogous to the external report's fix): only write `last_reward_block` when rewards are actually distributed (i.e., after the `disable_rewards || is_pre_consensus()` guard), so a no-op call does not consume the block slot.

---

### Proof of Concept

1. Consensus rewards are active (`!is_pre_consensus()`). A valid staker `S` has been staked for `K` epochs and has non-zero balance.
2. Block `N` begins. Attacker calls:
   ```
   update_rewards(staker_address: S, disable_rewards: true)
   ```
   from any non-zero address.
3. Inside the function: all staker validity checks pass → `last_reward_block` is written to `N` → the `disable_rewards=true` branch returns early with no reward distribution.
4. The Starkware sequencer calls `update_rewards(S, disable_rewards: false)` in the same block `N`. The check `current_block_number (N) > last_reward_block (N)` is `false` → reverts with `REWARDS_ALREADY_UPDATED`.
5. Block `N`'s rewards for all stakers are permanently lost. The attacker repeats this every block to continuously deny all consensus rewards. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
