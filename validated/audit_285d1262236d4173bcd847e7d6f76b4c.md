### Title
Permanent Bloat of `stakers` Vector Enables Unbounded Gas Consumption in `get_stakers` - (File: src/staking/staking.cairo)

### Summary

The `stakers` append-only vector in the `Staking` contract is never pruned when a staker exits. Any unprivileged address can stake with the minimum amount, exit (recovering all funds after the exit window), and permanently leave a dead entry in the vector. Because `get_stakers` iterates over the full vector on every call, an attacker can inflate the iteration cost without bound, causing unbounded gas consumption in a function critical to the consensus layer.

### Finding Description

When a new staker registers, their address is appended to `self.stakers`: [1](#0-0) 

When a staker exits via `unstake_action`, `remove_staker` is called, which deletes the staker's info from storage but **never removes the address from `self.stakers`**. The flow test `DiverseStakerVecFlow` confirms this directly — after `staker_in_action` completes `unstake_action`, the vector still reports length 4 and the exited staker remains at index 3: [2](#0-1) 

The `get_stakers` function in `StakingConsensusImpl` iterates over the **full range** of this vector on every invocation: [3](#0-2) 

Inactive/exited stakers are skipped via `is_staker_active` and a zero-staking-power check, but the iteration itself is still performed for every entry ever pushed. There is no cap on the number of stakers and no mechanism to compact or prune the vector.

### Impact Explanation

`get_stakers` is the function the consensus layer uses to build the validator set for a given epoch. Its access control is "Any address" and it is expected to be called by the sequencer each epoch. As the `stakers` vector grows without bound, the gas cost of `get_stakers` grows linearly with the total number of stakers ever registered (including all exited ones). A sufficiently large vector can make the function prohibitively expensive or cause it to exceed block resource limits, disrupting the consensus mechanism.

This matches the allowed impact: **Medium — Griefing with no profit motive but damage to users or protocol; Unbounded gas consumption**.

### Likelihood Explanation

The `stake` function is callable by any address with sufficient STRK to meet `min_stake`. After the `exit_wait_window`, the attacker recovers all staked funds via `unstake_action`. The only cost to the attacker is gas for the stake/unstake cycle. Each cycle permanently adds one dead entry to the `stakers` vector. The attack is therefore repeatable at low net cost and requires no privileged access. [4](#0-3) [5](#0-4) 

### Recommendation

Remove the staker's address from `self.stakers` inside `remove_staker` (called by `unstake_action`), or replace the append-only `Vec` with a data structure that supports O(1) deletion (e.g., a swap-and-pop pattern). Alternatively, maintain a separate active-staker count and enforce a hard cap on the total number of ever-registered stakers to bound the iteration.

### Proof of Concept

1. Attacker controls N addresses, each funded with `min_stake` STRK.
2. For each address: call `stake(...)` → `self.stakers.push(address)` executes, vector length increases by 1.
3. For each address: call `unstake_intent()` then, after `exit_wait_window`, call `unstake_action(staker_address)` → funds returned, but address remains in `self.stakers`.
4. Net cost to attacker: gas only (stake capital fully recovered).
5. After N cycles, `self.stakers` has N dead entries.
6. Every subsequent call to `get_stakers(epoch_id)` iterates all N entries via `self.stakers.into_iter_full_range()`, performing storage reads and `is_staker_active` checks for each dead entry.
7. Gas cost of `get_stakers` scales as O(N), with N unbounded. [6](#0-5) [7](#0-6)

        fn get_stakers(
            self: @ContractState, epoch_id: Epoch,
        ) -> Span<(ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>)> {
            let curr_epoch = self.get_current_epoch();
            assert!(
                curr_epoch <= epoch_id && epoch_id < curr_epoch + K.into(),
                "{}",
                Error::INVALID_EPOCH,
            );

            let (strk_total_stake, btc_total_stake) = self
                .get_total_staking_power_at_epoch(:epoch_id);

            let mut stakers: Array<
                (ContractAddress, StakingPower, Option<PublicKey>, Option<PeerId>),
            > =
                array![];
            for staker_address_ptr in self.stakers.into_iter_full_range() {
                let staker_address = staker_address_ptr.read();
                if !self.is_staker_active(:staker_address, :epoch_id) {
                    continue;
                }

                let staking_power = self
                    .get_staker_staking_power_at_epoch(
                        :staker_address, :epoch_id, :strk_total_stake, :btc_total_stake,
                    );
                if staking_power.is_zero() {
                    continue;
                }

                let public_key = self.get_public_key_at_epoch(:staker_address, :epoch_id);
                let peer_id = self.get_peer_id_at_epoch(:staker_address, :epoch_id);
                stakers.append((staker_address, staking_power, public_key, peer_id));
            }
            stakers.span()
        }
