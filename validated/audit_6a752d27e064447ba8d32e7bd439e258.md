### Title
Missing Access Control on `update_rewards` Allows Any Caller to Monopolize Per-Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is specified to be callable only by the Starkware sequencer, but the implementation enforces no such restriction. Because `last_reward_block` is a single global storage slot, only **one** staker can receive block rewards per block. An unprivileged staker can call `update_rewards` for themselves in every block, monopolizing all consensus-phase block rewards and permanently denying other stakers their yield.

---

### Finding Description

The protocol specification explicitly states:

> **access control**: Only starkware sequencer.

for `update_rewards`.

The implementation, however, only calls `general_prerequisites()`, which checks:
1. Contract is not paused.
2. Caller is not the zero address.

There is no sequencer-role check anywhere in the function body. [1](#0-0) 

`general_prerequisites` itself: [2](#0-1) 

The global `last_reward_block` storage variable is written once per call and gates all subsequent calls in the same block: [3](#0-2) 

Because `last_reward_block` is a **single global slot** (not per-staker), exactly one staker can receive rewards per block. The sequencer is intended to rotate through stakers, calling `update_rewards` for each in successive blocks. An attacker who calls `update_rewards` for their own staker first in every block:

1. Receives the full `block_rewards` for that block.
2. Sets `last_reward_block` to the current block number.
3. Causes every subsequent call in the same block (including the sequencer's calls for other stakers) to revert with `REWARDS_ALREADY_UPDATED`.

The block rewards in V3 (consensus phase) are calculated as:

```
block_rewards = yearly_mint * avg_block_time / SECONDS_IN_YEAR
``` [4](#0-3) 

These rewards are then distributed to the attacker's staker and their pools via `_update_rewards`, which updates `unclaimed_rewards_own` and transfers STRK to pool contracts: [5](#0-4) 

The spec confirms the intended access control: [6](#0-5) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

In the consensus-rewards phase, block rewards are the sole source of staker yield. Because only one staker receives rewards per block, an attacker who calls `update_rewards` for themselves in every block receives 100% of all block rewards. All other stakers receive zero yield for as long as the attack continues. The stolen yield is immediately credited to `unclaimed_rewards_own` and is claimable by the attacker via `claim_rewards`. [7](#0-6) 

---

### Likelihood Explanation

**Low-Medium.** On the current centralized Starknet sequencer, the sequencer controls transaction ordering and can place its own `update_rewards` calls ahead of any user transaction. However:

- The missing access control is a direct violation of the spec and leaves the invariant entirely unenforced at the contract level.
- If the sequencer is ever decentralized, compromised, or if a staker submits transactions that the sequencer includes before its own system calls (e.g., during sequencer downtime or reordering), the attack becomes fully executable.
- Any staker can already call `update_rewards` for themselves today without restriction, which is contrary to the stated design.

---

### Recommendation

Introduce a sequencer role (analogous to the existing `security_agent`, `token_admin`, etc.) and add a role check at the top of `update_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer(); // <-- add this
    self.general_prerequisites();
    ...
}
```

Alternatively, store the authorized sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()`.

---

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. Attacker is a registered staker with non-zero balance (K epochs after `stake`).
3. In every new block, attacker submits a transaction calling `update_rewards(attacker_address, false)`.
4. The call passes all checks: contract not paused, caller non-zero, `current_block > last_reward_block`, staker active with non-zero balance.
5. `last_reward_block` is set to the current block number.
6. Attacker receives `block_rewards` credited to `unclaimed_rewards_own`.
7. The sequencer's `update_rewards` calls for all other stakers in the same block revert with `REWARDS_ALREADY_UPDATED`.
8. Attacker calls `claim_rewards` to withdraw the accumulated yield.
9. All other stakers accumulate zero yield indefinitely. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
