### Title
Missing Pause Check in `exit_delegation_pool_intent` When Staker Is Removed - (File: src/pool/pool.cairo)

### Summary
The `exit_delegation_pool_intent` function in the Pool contract bypasses the staking contract's pause check when the associated staker has been removed. The spec explicitly requires the staking contract to be unpaused as a pre-condition for this function, but the implementation skips the staking contract call entirely in the staker-removed path, meaning no pause check is ever performed.

### Finding Description
The Pool contract's `exit_delegation_pool_intent` enforces the pause pre-condition only transitively — by calling `staking_pool_dispatcher.remove_from_delegation_pool_intent(...)` on the staking contract, which internally calls `assert_is_unpaused`. However, the internal helper `undelegate_from_staking_contract_intent` has a branch for when the staker is inactive (removed):

```cairo
fn undelegate_from_staking_contract_intent(
    self: @ContractState, pool_member: ContractAddress, amount: Amount,
) -> Timestamp {
    if !self.is_staker_active() {
        assert!(
            self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
            "{}",
            Error::UNDELEGATE_IN_PROGRESS,
        );
        return Time::now();   // ← returns immediately, no staking contract call, no pause check
    }
    let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
    let staker_address = self.staker_address.read();
    staking_pool_dispatcher
        .remove_from_delegation_pool_intent(
            :staker_address, identifier: pool_member.into(), :amount,
        )
}
```

When `staker_removed == true`, the function returns `Time::now()` without ever calling the staking contract. Because the Pool contract has no independent pause mechanism, this path has zero pause enforcement.

The spec (`docs/spec.md` lines 1966–1970) is unambiguous:

> **pre-condition**
> 4. Staking contract is unpaused.

The staking contract exposes `assert_is_unpaused` (staking.cairo line 1657–1659) and the pause tests confirm that `remove_from_delegation_pool_intent` panics with `"Contract is paused"` when the staking contract is paused. The staker-removed branch in the pool completely sidesteps this.

### Impact Explanation
When the staking contract is paused (e.g., after a security incident), a pool member whose staker has already been removed can still successfully call `exit_delegation_pool_intent`. This:

1. Writes `unpool_time = Time::now()` and `unpool_amount = amount` into pool member storage.
2. Reduces the pool member's tracked delegated balance immediately.
3. Because `unpool_time` is set to the current timestamp (exit window = 0 for removed stakers), the pool member can call `exit_delegation_pool_action` the instant the contract is unpaused and withdraw funds without any waiting period.

The pause is a security freeze intended to halt all state-changing operations. This bypass allows pool members to pre-position their exit intent during the pause window, circumventing the protocol's emergency-stop invariant. This maps to **Medium: griefing with damage to the protocol** — the security agent's pause does not fully freeze the pool's exit lifecycle for the staker-removed case.

### Likelihood Explanation
Any pool member whose staker has completed `unstake_action` (staker removed) can trigger this at any time the staking contract is paused. No privileged access is required. The caller is an unprivileged pool member. The condition (`staker_removed == true`) is a normal, reachable protocol state.

### Recommendation
Add an explicit pause check inside the staker-inactive branch of `undelegate_from_staking_contract_intent`, or add it directly at the top of `exit_delegation_pool_intent`. The pool contract already holds a reference to the staking contract via `staking_pool_dispatcher`; it can call `IStakingDispatcher { contract_address: ... }.is_paused()` and assert it is false before proceeding.

```cairo
fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
    // Add this check:
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
    // ... rest of function
}
```

### Proof of Concept

1. Deploy the system normally; staker stakes and opens a pool.
2. Pool member calls `enter_delegation_pool` with some amount.
3. Staker calls `unstake_intent` then `unstake_action` — staker is removed, `staker_removed = true` in the pool.
4. Security agent calls `staking_contract.pause()` — staking contract is now paused.
5. Pool member calls `pool.exit_delegation_pool_intent(amount)`.
   - `is_staker_active()` returns `false` → staker-removed branch executes.
   - No call to staking contract → no pause check → **call succeeds**.
   - `unpool_time = Time::now()`, `unpool_amount = amount` written to storage.
6. Pool member waits for unpause, then immediately calls `pool.exit_delegation_pool_action(pool_member)`.
   - `Time::now() >= unpool_time` is satisfied immediately.
   - Funds are transferred to pool member.

The pool member successfully initiated their exit during the pause window, violating the spec's pre-condition that the staking contract must be unpaused for `exit_delegation_pool_intent`.

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(

                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }
