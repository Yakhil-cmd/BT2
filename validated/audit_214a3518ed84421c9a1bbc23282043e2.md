### Title
Unprivileged Caller Can Permanently Block Consensus Reward Distribution by Calling `update_rewards` with `disable_rewards=true` - (File: `src/staking/staking.cairo`)

### Summary
`update_rewards` in `staking.cairo` writes `last_reward_block` unconditionally before checking `disable_rewards`. Because the function has no access control (contrary to the spec's "Only starkware sequencer" requirement), any unprivileged caller can invoke it with `disable_rewards=true` every block, consuming the global block-reward slot without distributing any rewards. This permanently denies consensus-phase rewards to all stakers and their delegators.

### Finding Description
The `IStakingRewardsManager::update_rewards` function is the sole mechanism for distributing per-block consensus rewards. Its implementation in `StakingRewardsManagerImpl` performs the following sequence:

1. Calls `general_prerequisites()` — checks only that the contract is unpaused and the caller is non-zero. No role check.
2. Asserts `current_block_number > self.last_reward_block.read()` — enforces one call per block globally.
3. **Writes `self.last_reward_block.write(current_block_number)` at line 1485** — consuming the block slot.
4. Checks `if disable_rewards || self.is_pre_consensus() { return; }` at line 1487 — returns early without distributing rewards.

The critical flaw is that step 3 (the write) occurs **before** step 4 (the early return). When `disable_rewards=true` is passed, `last_reward_block` is updated to the current block number, but no rewards are calculated or distributed. Any subsequent call in the same block — including a legitimate call from the sequencer with `disable_rewards=false` — will revert with `REWARDS_ALREADY_UPDATED`.

The spec at `docs/spec.md` line 1645 states access control is "Only starkware sequencer," but the implementation enforces no such restriction. `general_prerequisites()` only checks `assert_is_unpaused()` and `assert_caller_is_not_zero()`. [1](#0-0) [2](#0-1) [3](#0-2) 

The analog to the external report's bug is direct: in `CommunityRound.sol`, a storage slot (`balances[sender_]`) is **assigned** instead of **accumulated**, discarding prior state. Here, `last_reward_block` is **assigned** the current block number before the guard that would prevent reward distribution, discarding the block's reward opportunity for all stakers.

### Impact Explanation
`last_reward_block` is a **global** storage variable shared across all stakers. A single call to `update_rewards(any_valid_staker, disable_rewards=true)` per block blocks reward distribution for **every** staker and every delegator in that block. An attacker who calls this function once per block (trivially automatable) causes all consensus-phase staking rewards to be permanently lost — stakers and delegators earn zero rewards indefinitely.

This matches the allowed impact: **Medium — Griefing with no profit motive but damage to users or protocol** (complete denial of consensus reward accrual for all protocol participants).

### Likelihood Explanation
- The function is publicly callable by any non-zero address with no role restriction.
- The attack requires one transaction per block — low cost on Starknet L2.
- No special knowledge, tokens, or privileges are required.
- The attack is fully automatable and can be sustained indefinitely.
- The attacker gains nothing financially, but the damage to stakers and delegators is total during the attack window.

### Recommendation
Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` / `is_pre_consensus()` guard, so the block slot is only consumed when rewards are actually distributed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Update last block rewards only when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... rest of reward distribution logic
```

Additionally, enforce the access control specified in the spec by adding a sequencer-only role check at the top of `update_rewards`, consistent with the documented "Only starkware sequencer" requirement.

### Proof of Concept

1. Consensus rewards are activated (`consensus_rewards_first_epoch` is set and passed).
2. A staker has been active for K+ epochs and has non-zero balance.
3. Attacker (any address) calls `update_rewards(staker_address, disable_rewards=true)` at block N.
   - `last_reward_block` is written to N.
   - Function returns early; no rewards distributed.
4. Sequencer (or staker) calls `update_rewards(staker_address, disable_rewards=false)` at block N.
   - Reverts with `REWARDS_ALREADY_UPDATED` because `N > N` is false.
5. Attacker repeats step 3 at every subsequent block.
6. Result: `staker_info.unclaimed_rewards_own` remains zero indefinitely; pool `cumulative_rewards_trace` is never updated; all stakers and delegators earn zero consensus rewards.

The test at `src/staking/tests/test.cairo` lines 3886–3894 inadvertently documents this exact behavior — calling `update_rewards` with `disable_rewards=true` then immediately blocking a `disable_rewards=false` call with `REWARDS_ALREADY_UPDATED` — but treats it as expected rather than a vulnerability.     impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
