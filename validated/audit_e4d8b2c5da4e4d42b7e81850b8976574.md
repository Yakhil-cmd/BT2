### Title
Missing Caller Authorization in `update_rewards` Allows Any Caller to Permanently Freeze Consensus Rewards — (`src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` accepts a `disable_rewards: bool` parameter from any caller with no access-control check. Because `last_reward_block` is written to storage **before** the early-return guard on `disable_rewards`, an unprivileged attacker can call `update_rewards(any_valid_staker, true)` once per block to advance the global reward-block cursor without distributing any rewards, permanently blocking all consensus-phase reward accrual for every staker and pool member.

---

### Finding Description

`update_rewards` is the entry point through which the consensus mechanism distributes per-block STRK/BTC rewards to stakers and their delegation pools. The function signature is:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
)
```

The only gate is `general_prerequisites()`, which checks that the contract is not paused and that the caller is non-zero. [1](#0-0) 

After validating the staker, the function unconditionally writes the current block number to the global `last_reward_block` storage slot, then checks `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

The guard that prevents a second call in the same block is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

Because `last_reward_block` is already updated before the `disable_rewards` branch, any subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`. The attacker only needs to supply any currently-active staker address with a non-zero STRK balance to pass the staker-validity assertions. [4](#0-3) 

---

### Impact Explanation

By calling `update_rewards(any_active_staker, disable_rewards: true)` once per block, an attacker:

1. Advances `last_reward_block` to the current block without distributing any STRK or BTC rewards.
2. Causes every legitimate call to `update_rewards` in that block to revert.
3. Permanently destroys the per-block reward allocation for that block — rewards are never retroactively recalculated.

Sustained over multiple blocks this constitutes **permanent freezing of unclaimed yield** for all stakers and all delegation-pool members, matching the High-severity impact tier in the allowed scope. Even sporadic execution causes **temporary freezing of unclaimed yield**.

---

### Likelihood Explanation

The attack requires no privileged role, no tokens, and no prior relationship with any staker. Any externally-owned account on Starknet can call `update_rewards` with a valid staker address (which is public on-chain) and `disable_rewards: true`. Gas costs on Starknet are low, making sustained per-block griefing economically feasible. Likelihood is **High**.

---

### Recommendation

Restrict `update_rewards` to a trusted caller — either the attestation contract, a dedicated consensus-rewards caller role, or the staker's own operational address. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_attestation_contract(); // or a dedicated rewards-manager role
    ...
}
```

Alternatively, remove the `disable_rewards` parameter from the public interface and derive the flag internally from on-chain attestation state, so no external caller can suppress reward distribution.

---

### Proof of Concept

```
// Precondition: consensus rewards are active (consensus_rewards_first_epoch has passed).
// Alice is a valid, active staker with non-zero STRK balance.

// Attacker (any address) calls once per block:
staking.update_rewards(alice_address, disable_rewards: true);
// → last_reward_block is set to current_block_number
// → no rewards are distributed (early return)

// Alice (or the consensus mechanism) tries to distribute rewards for the same block:
staking.update_rewards(alice_address, disable_rewards: false);
// → REVERTS with REWARDS_ALREADY_UPDATED

// Repeat every block → all consensus rewards are permanently frozen.
``` [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
