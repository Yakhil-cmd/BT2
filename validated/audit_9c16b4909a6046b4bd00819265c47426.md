### Title
Gateway Stateful Validator Uses Previous Block's L2 Gas Price Instead of `next_l2_gas_price` for Admission Threshold — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **previous block's** `l2_gas_price` field, while every transaction admitted through the gateway will actually be executed in the **next** block, whose L2 gas price is computed by the EIP-1559 fee market and stored separately as `BlockHeaderWithoutHash::next_l2_gas_price`. The code itself acknowledges the bug with an inline TODO. The result is a systematic mismatch between the price used for gateway admission and the price used for blockifier execution: transactions can be admitted that will fail at execution time, and valid transactions can be rejected.

---

### Finding Description

In `validate_resource_bounds` the gateway reads the latest committed block's info and extracts `strk_gas_prices.l2_gas_price`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`validate_tx_l2_gas_price_within_threshold` then enforces:

```
tx.l2_gas.max_price_per_unit  >=  (min_gas_price_percentage / 100)  *  previous_block_l2_gas_price
``` [2](#0-1) 

The default `min_gas_price_percentage` is **100**, meaning the threshold equals the previous block's price exactly. [3](#0-2) 

However, the Starknet protocol computes a **separate** `next_l2_gas_price` for each block via the EIP-1559 fee market (`calculate_next_base_gas_price`) and stores it in `BlockHeaderWithoutHash::next_l2_gas_price`:

```rust
pub struct BlockHeaderWithoutHash {
    pub l2_gas_price: GasPricePerToken,   // price used in THIS block
    pub next_l2_gas_price: GasPrice,      // price to use in the NEXT block
    ...
}
``` [4](#0-3) 

The fee market formula adjusts the price proportionally to how far actual gas consumption deviates from the target:

```rust
let price_change = (price_u256 * gas_delta) / denominator;
let adjusted_price_u256 =
    if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };
``` [5](#0-4) 

The consensus context writes `next_l2_gas_price` into the block header when committing a block:

```rust
let block_header_without_hash = BlockHeaderWithoutHash {
    l2_gas_price,                              // current block's price
    next_l2_gas_price: self.l2_gas_price,      // next block's price
    ...
};
``` [6](#0-5) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` converts the block header into a `BlockInfo` but only copies `l2_gas_price` (the current block's price) into `BlockInfo::gas_prices`; `next_l2_gas_price` is silently discarded:

```rust
strk_gas_prices: GasPriceVector {
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
    ...
},
``` [7](#0-6) 

`run_validate_entry_point` also builds its `BlockContext` from the same stale `get_block_info()` call, only bumping the block number but leaving gas prices unchanged:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
``` [8](#0-7) 

This means the blockifier's `check_fee_bounds` during gateway validation also uses the previous block's gas prices, not the next block's.

---

### Impact Explanation

**False admission (next block price rises):** When the previous block was heavily utilized, `next_l2_gas_price > previous_block.l2_gas_price`. A transaction with `tx_l2_gas_price` in the range `[previous_block.l2_gas_price, next_l2_gas_price)` passes the gateway threshold check and is admitted to the mempool. When the batcher later executes it, `check_fee_bounds` compares against the actual next block price and rejects it with `MaxGasPriceTooLow`. The transaction is wasted and the user receives an unexpected execution failure after a successful gateway response.

**False rejection (next block price falls):** When the previous block was lightly utilized, `next_l2_gas_price < previous_block.l2_gas_price`. A transaction with `tx_l2_gas_price` in the range `[next_l2_gas_price, previous_block.l2_gas_price)` would be perfectly valid for the next block but is rejected at the gateway with `GAS_PRICE_TOO_LOW`. Valid transactions are denied admission.

The magnitude of the discrepancy is bounded by the EIP-1559 formula. For a fully saturated block the price change per block is `price * (max_block_size - gas_target) / (gas_target * gas_price_max_change_denominator)`, which can be several percent per block. Over a sustained period of high or low utilization the cumulative gap between `l2_gas_price` and `next_l2_gas_price` grows further.

This matches the **High** impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The condition is triggered whenever the L2 gas price is changing — i.e., whenever the network is not exactly at the gas target. This is the normal operating state of any EIP-1559 fee market. Any user submitting a transaction priced at or near the current block's gas price during a period of sustained high or low utilization will be affected. No special privileges are required; the trigger is a standard `add_transaction` RPC call.

---

### Recommendation

1. Extend `GatewayFixedBlockStateReader` (and its `GatewayFixedBlockSyncStateClient` implementation) to expose `next_l2_gas_price` from `BlockHeaderWithoutHash::next_l2_gas_price`.
2. In `validate_resource_bounds`, replace `previous_block_l2_gas_price` with the retrieved `next_l2_gas_price`.
3. In `run_validate_entry_point`, populate `block_info.gas_prices.strk_gas_prices.l2_gas_price` with `next_l2_gas_price` so that the blockifier's `check_fee_bounds` during gateway validation uses the same price the batcher will use.

---

### Proof of Concept

1. Observe a sequence of blocks with high utilization so that `next_l2_gas_price` is consistently above `l2_gas_price` (e.g., by 5%).
2. Submit an `InvokeV3` transaction with `l2_gas.max_price_per_unit = previous_block.l2_gas_price` (exactly at the threshold when `min_gas_price_percentage = 100`).
3. The gateway's `validate_tx_l2_gas_price_within_threshold` computes `threshold = 100% * previous_block.l2_gas_price = tx_l2_gas_price` → check passes; transaction is admitted to the mempool.
4. The batcher builds the next block using `next_l2_gas_price > tx_l2_gas_price`; `check_fee_bounds` fires `ResourceBoundsError::MaxGasPriceTooLow { resource: L2Gas, max_gas_price: tx_l2_gas_price, actual_gas_price: next_l2_gas_price }`.
5. The transaction is reverted or dropped despite having passed gateway admission — a direct analog to the external report's use of a stale value (`balanceOf` / `previous_block_l2_gas_price`) instead of the time-current value (`getVotingPower` / `next_l2_gas_price`).

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
