### Title
Unrestricted `update_rewards` with `disable_rewards=true` Permanently Freezes All Stakers' Consensus Yield — (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the `StakingRewardsManagerImpl` is publicly callable by any address. It accepts a `disable_rewards: bool` parameter. When `true`, the function writes the current block number to the global `last_reward_block` storage slot and returns early — without distributing any rewards. Because `last_reward_block` is a single global gate that prevents re-entry for the same block, any attacker can call `update_rewards(any_active_staker, disable_rewards=true)` once per block to permanently suppress all consensus reward distributions across the entire protocol.

---

### Finding Description

`update_rewards` is exposed in the public ABI under `IStakingRewardsManager` with no caller restriction beyond `general_prerequisites()`, which only checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

The function's first substantive action after the staker validity checks is to unconditionally write `current_block_number` to `last_reward_block`: [2](#0-1) 

Only *after* this write does the function branch on `disable_rewards`: [3](#0-2) 

Because `last_reward_block` is a single global value (not per-staker), the guard at the top of the function:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

means that once any caller invokes `update_rewards(..., disable_rewards=true)` for a given block, **no one** — including the legitimate staker — can call `update_rewards` again for that block to actually receive rewards. [4](#0-3) 

The attacker only needs to supply any currently-active staker address with non-zero balance (public on-chain information) to pass the validity checks.

---

### Impact Explanation

In consensus rewards mode (after `consensus_rewards_first_epoch`), `update_rewards` is the sole mechanism for distributing per-block STRK and BTC rewards to stakers and their delegation pools. An attacker who calls `update_rewards(any_active_staker, disable_rewards=true)` once per block permanently discards that block's rewards for every staker in the protocol. Sustained over time, this constitutes **permanent freezing of unclaimed yield** for all stakers and delegators.

This matches the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"* (High).

---

### Likelihood Explanation

- The function is in the public ABI with no privileged-role guard.
- The attacker needs only a valid active staker address (trivially obtained from on-chain events such as `NewStaker`).
- The cost is one Starknet transaction per block (~seconds apart). Gas costs on Starknet are low.
- There is no profit requirement; the attacker's only motivation is griefing.
- No special setup, no leaked keys, no bridge compromise required.

---

### Recommendation

Add an access-control check so that `update_rewards` can only be called by a designated trusted caller (e.g., the consensus layer contract or a specific operator role). For example:

```cairo
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();
    self.assert_caller_is_consensus_contract(); // add this guard
    ...
}
```

Alternatively, if `disable_rewards=true` is only ever a protocol-internal signal, remove the parameter from the public interface entirely and handle it through a separate privileged entry point.

---

### Proof of Concept

1. Protocol is in consensus rewards mode (`consensus_rewards_first_epoch` has been set and passed).
2. Attacker observes any active staker address `S` from on-chain `NewStaker` events.
3. Each block, attacker calls:
   ```
   staking_contract.update_rewards(staker_address=S, disable_rewards=true)
   ```
4. `last_reward_block` is set to the current block number with no rewards distributed.
5. Any subsequent call to `update_rewards` for the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers and delegators receive zero consensus rewards for every block the attacker covers.
7. Repeated indefinitely → permanent freezing of all consensus yield across the protocol. [5](#0-4)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
