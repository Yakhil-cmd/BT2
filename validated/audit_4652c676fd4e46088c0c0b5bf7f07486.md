### Title
Missing Access Control on `update_rewards` Allows Any Caller to Block Consensus Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `Staking` contract is documented in the spec as callable only by the Starknet sequencer, but no access control is enforced in the implementation. Any unprivileged caller can invoke it with `disable_rewards=true` to advance the global `last_reward_block` without distributing rewards, permanently blocking the sequencer's legitimate reward calls for that block across all stakers.

### Finding Description
The protocol spec at `docs/spec.md:1645` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `update_rewards` in `src/staking/staking.cairo` contains only a pause check via `general_prerequisites()` and a per-block deduplication guard — no sequencer identity check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.  <-- written BEFORE the disable_rewards branch
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                    // exits without distributing any rewards
    }
    ...
``` [1](#0-0) 

The critical ordering is: `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated. This means a call with `disable_rewards=true` consumes the block's reward slot without distributing anything.

`last_reward_block` is a **global** (not per-staker) storage variable. [2](#0-1) 

Once it is set to the current block number, every subsequent call in the same block — including the sequencer's legitimate calls for all other stakers — reverts with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

The spec confirms this is the intended single-call-per-block design: [4](#0-3) 

### Impact Explanation
An attacker who calls `update_rewards(any_active_staker, disable_rewards=true)` once per block:

1. Advances `last_reward_block` to the current block without distributing rewards.
2. Causes every sequencer call for that block (for every staker) to revert with `REWARDS_ALREADY_UPDATED`.
3. All stakers and their delegation pool members receive zero consensus rewards for that block.

Sustained over time this constitutes **permanent freezing of unclaimed yield** for all stakers and pool members in the consensus-rewards epoch. Even a sporadic attack causes **temporary freezing of unclaimed yield**. Both are in the allowed impact scope.

### Likelihood Explanation
- The function is publicly callable with no authentication.
- The attacker only needs to know any one active staker address (all are emitted as `NewStaker` events and stored in the public `stakers` vector). [5](#0-4) 
- The cost is one transaction per block (gas only); no capital is required.
- The attack is trivially scriptable and requires no privileged access.

### Recommendation
Add a sequencer-only access control guard at the top of `update_rewards`, consistent with the spec. For example, store the authorized sequencer address in contract storage and assert `get_caller_address() == sequencer_address` before any state mutation. Alternatively, move the `last_reward_block.write` to after the `disable_rewards` branch so that a call with `disable_rewards=true` does not consume the block's reward slot.

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. Attacker identifies any active staker address `S` (from on-chain events).
3. At the start of each block, attacker submits: `staking.update_rewards(S, disable_rewards=true)`.
4. `last_reward_block` is set to the current block; no rewards are distributed.
5. The sequencer's call `staking.update_rewards(S', disable_rewards=false)` for any staker `S'` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeat every block → all stakers accumulate zero consensus rewards indefinitely. [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
