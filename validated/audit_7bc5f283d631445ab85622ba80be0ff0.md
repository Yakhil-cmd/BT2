### Title
Missing Caller Access Control in `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (`File: src/staking/staking.cairo`)

### Summary
`update_rewards` is callable by any unprivileged address. By calling it with `disable_rewards: true`, an attacker writes the current block number into the global `last_reward_block` storage slot and returns early, permanently discarding that block's consensus rewards for every staker. Because `last_reward_block` is a single global gate (not per-staker), one hostile call per block is sufficient to suppress all consensus rewards indefinitely.

### Finding Description

The spec mandates "Only starkware sequencer" access control for `update_rewards`: [1](#0-0) 

The production implementation enforces no such check. The function body begins with only a pause check (`general_prerequisites`) and a block-number gate: [2](#0-1) 

After validating the staker exists and is active, it unconditionally writes `current_block_number` to `last_reward_block` before checking `disable_rewards`: [3](#0-2) 

`last_reward_block` is a single global storage variable. Once written for block N, every subsequent call to `update_rewards` in block N — for any staker — reverts with `REWARDS_ALREADY_UPDATED`. The attacker therefore needs only one call per block to permanently discard that block's rewards for the entire protocol.

### Impact Explanation

An unprivileged attacker calling `update_rewards(any_active_staker_address, disable_rewards: true)` once per block:

1. Advances `last_reward_block` to the current block.
2. Causes the early return at line 1487, skipping all reward accounting.
3. Blocks every legitimate sequencer call for that block with `REWARDS_ALREADY_UPDATED`.

Repeated across every block, this permanently freezes all unclaimed consensus-reward yield for all stakers and delegators. No funds already staked are moved, but all future yield accrual is zeroed out.

**Impact category:** High — Permanent freezing of unclaimed yield.

### Likelihood Explanation

The function is part of the public ABI (`#[abi(embed_v0)]`): [4](#0-3) 

Any EOA or contract on Starknet can call it at zero cost beyond gas. The attacker needs no stake, no role, and no special knowledge beyond knowing a single active staker address (which is public on-chain). The attack is repeatable every block with no capital at risk.

### Recommendation

Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the authorized Starkware sequencer address (stored in contract configuration), consistent with the spec's stated access control. The check should be placed before the `last_reward_block` write so a rejected call cannot consume the block's reward slot.

### Proof of Concept

```
// Block N, attacker (any address) calls:
staking.update_rewards(
    staker_address: any_valid_active_staker,
    disable_rewards: true   // attacker-controlled
);
// Effect: last_reward_block = N, no rewards distributed.

// Legitimate sequencer now tries in the same block:
staking.update_rewards(staker_address: real_staker, disable_rewards: false);
// Reverts: REWARDS_ALREADY_UPDATED

// Repeat every block → all consensus rewards permanently suppressed.
```
