### Title
Malicious Participant Can Permanently Stall DKG/Reshare by Completing Early Rounds Then Withholding Private Shares - (File: `src/dkg.rs`, `src/protocol/internal.rs`)

---

### Summary

A malicious participant in the DKG, Reshare, or Refresh protocol can cause a permanent denial-of-service for all honest parties by completing the commitment-broadcast rounds (establishing a "committed" state visible to all peers) and then refusing to send their private polynomial evaluations in the final round. Because the protocol's message-receive primitive blocks indefinitely with no timeout or abort path, honest parties are permanently stuck waiting. There is no mechanism to identify the culprit or recover the session.

---

### Finding Description

`do_keyshare` in `src/dkg.rs` runs five sequential rounds:

1. **Round 1** – each party broadcasts a commitment hash via `chan.send_many(wait_round_1, &commitment_hash)`.
2. **Round 2** – each party broadcasts its full polynomial commitment and proof-of-knowledge via `do_broadcast(...)`.
3. **Round 3** – each party sends a private polynomial evaluation to every other participant via `chan.send_private(wait_round_3, p, &signing_share_to_p)`.
4. **Round 4** – each party collects those private shares via `recv_from_others(&chan, wait_round_3, &participants, me)`.
5. **Round 5** – `broadcast_success` confirms completion. [1](#0-0) [2](#0-1) 

The `recv_from_others` call at Round 4 waits for **every** other participant to deliver a private share before it returns. The underlying primitive that drives this wait is `MessageBuffer::pop()`:

```rust
async fn pop(&self, header: MessageHeader) -> (Participant, MessageData) {
    ...
    receiver_lock.next().await.expect("Reference to sender held")
}
``` [3](#0-2) 

`next().await` on an `UnboundedReceiver` blocks forever if no message arrives. There is no `tokio::time::timeout`, no cancellation token, and no fallback path anywhere in the protocol executor: [4](#0-3) 

The executor's `poke()` loop simply returns `Action::Wait` when the future is pending; it never times out or aborts.

**Attack sequence:**

1. Malicious participant `P_m` joins a DKG session with `n` honest parties.
2. `P_m` completes Round 1 (sends commitment hash to all) and Round 2 (participates in echo-broadcast, sending commitment + proof). All honest parties now hold `P_m`'s commitment and consider the session in progress.
3. `P_m` deliberately sends **no** private share in Round 3.
4. Every honest party reaches `recv_from_others` in Round 4 and blocks indefinitely waiting for `P_m`'s share.
5. The DKG session never completes. No key is generated. There is no timeout, no blame mechanism, and no way to restart without external intervention.

The same pattern applies identically to `do_reshare` (which calls `do_keyshare`) and to the presign protocol's `recv_from_others` calls: [5](#0-4) [6](#0-5) 

---

### Impact Explanation

**High – Permanent denial of key generation, reshare, and presigning for honest parties.**

A single malicious participant who completes the commitment phase but withholds private shares permanently blocks all honest parties in an unbounded async wait. No key material is produced. The session cannot be salvaged without an external timeout and restart, and even then the malicious party cannot be identified from within the protocol, so they can repeat the attack on every subsequent attempt.

---

### Likelihood Explanation

Any participant admitted to a DKG or reshare session can execute this attack. The attacker needs only to send valid messages for Rounds 1–2 (trivially cheap) and then go silent. No cryptographic capability is required. In a permissioned MPC deployment where one node is compromised or adversarial, this is a straightforward griefing vector that permanently prevents key generation.

---

### Recommendation

Introduce a per-round deadline enforced at the protocol-executor level. Concretely:

- Wrap each `recv_from_others` / `MessageBuffer::pop` call with a configurable timeout (e.g., `tokio::time::timeout`).
- On timeout, record which participants have not yet responded and surface them as `ProtocolError::UnresponsiveParticipant(id)`.
- Allow the caller to abort the session and exclude the identified participant from the next attempt.

This mirrors the "blame" mechanism recommended in the original report: rather than waiting indefinitely, the protocol should detect non-participation after a bounded time and attribute it to the responsible party.

---

Round 3: P_m sends NO private share (simply drops the send_private call).

  → never returns
  → DKG permanently stalled, no KeygenOutput produced.

P_m cost: two cheap broadcast messages.

    fn poke(&mut self) -> Result<Action<Self::Output>, ProtocolError> {
        let mut polled_once_already = false;
        loop {
            // If there's outgoing messages, request to send them.
            if let Some(outgoing) = self.comms.outgoing() {
                return Ok(match outgoing {
                    Message::Many(m) => Action::SendMany(m),
                    Message::Private(to, m) => Action::SendPrivate(to, m),
                });
            }
            // If we already have a return result, return it.
            if let Some(result) = self.result.take() {
                return Ok(Action::Return(result?));
            }
            // If this is the second iteration, we already polled the future and there's no
            // progress that can be made.
            if polled_once_already {
                return Ok(Action::Wait);
            }
            // If we don't have a future, this is an extraneous poke() call, so return Wait.
            let Some(fut) = self.fut.as_mut() else {
                return Ok(Action::Wait);
            };
            // Now poll the future. It may generate some more messages to send or a return value,
            // so go back and check all of those again.
            polled_once_already = true;
            let waker = noop_waker();
            let mut cx = Context::from_waker(&waker);
            if let std::task::Poll::Ready(result) = fut.poll_unpin(&mut cx) {
                self.result = Some(result);
                self.fut = None;
            }
        }
    }
