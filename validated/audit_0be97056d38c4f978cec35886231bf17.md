### Title
Gateway L2 Gas Price Admission Uses Single-Block Spot Price Instead of Next Block's Actual Execution Price - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` gates mempool admission using the **previous block's** L2 gas price as a single-block snapshot, while the blockifier executes the same transaction against the **next block's** L2 gas price set by the proposer. The two values diverge by up to `1/gas_price_max_change_denominator` (~2%) per block, creating a systematic window where the gateway admits transactions that the batcher will silently drop, and rejects transactions the batcher would have accepted.

### Finding Description

In `validate_resource_bounds`, the gateway reads the L2 gas price from the latest committed block via `GatewayFixedBlockStateReader::get_block_info()`:

```rust

self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,

``` [1](#0-0) 

The threshold computed is `min_gas_price_percentage% × previous_block_l2_gas_price`. The in-code TODO explicitly acknowledges the wrong value is being used. [2](#0-1) 

`GatewayFixedBlockSyncStateClient` is constructed once per transaction at `instantiate_validator` time, pinned to `latest_block_number` at that moment: [3](#0-2) 

The batcher, however, executes the transaction inside a block context whose gas prices are taken from the proposer's `ProposalInit` — the **next** block's prices, computed by the EIP-1559 mechanism from the previous block's gas usage. `check_fee_bounds` in `perform_pre_validation_stage` enforces:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [4](#0-3) 

The next block's L2 gas price is derived from the EIP-1559 formula with `gas_price_max_change_denominator = 48`, meaning it can rise or fall by up to ~2% relative to the previous block's price per block. [5](#0-4) 

### Impact Explanation

**Admission of invalid transactions (price-increase case):** When block N has high gas utilization, the next block's L2 gas price rises above block N's price. The gateway admits any transaction whose `l2_gas.max_price_per_unit` satisfies `>= min_gas_price_percentage% × P_N`. The batcher then rejects those transactions whose `max_price_per_unit < P_{N+1}` via `check_fee_bounds`, dropping them without inclusion. These transactions consumed mempool slots and triggered full stateful validation (including blockifier `__validate__` execution) for nothing.

**Rejection of valid transactions (price-decrease case):** When block N has low gas utilization, the next block's L2 gas price falls below block N's price. The gateway rejects transactions whose `max_price_per_unit >= P_{N+1}` but `< min_gas_price_percentage% × P_N`, even though the batcher would have accepted them. Legitimate users are incorrectly told their transaction is too cheap.

Both directions match the **High** impact: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."

### Likelihood Explanation

The EIP-1559 mechanism guarantees the next block's L2 gas price diverges from the previous block's price whenever gas utilization deviates from the target. With `gas_target = 1,040,000,000` and `gas_price_max_change_denominator = 48`, any block that is even slightly over- or under-utilized produces a price mismatch. This is a normal operating condition, not an edge case. The window is bounded (~2% per block) but is continuously present and grows if utilization is consistently above or below target across multiple blocks.

### Recommendation

Replace the single-block snapshot with the **next block's** L2 gas price, which is already computed and available. The consensus orchestrator calculates `next_l2_gas_price` via `calculate_next_l2_gas_price` and embeds it in `ProposalFin`/`ProposalInit`. The gateway should either:

1. Read the next L2 gas price from the pending block header once it is available (as the TODO suggests), or
2. Apply the same EIP-1559 step locally: `next_price = eip1559_step(previous_block_l2_gas_price, previous_block_gas_used, gas_target, denominator)` to compute the threshold that the batcher will actually enforce. [6](#0-5) 

### Proof of Concept

1. Observe block N with gas usage significantly above `gas_target` (1.04 × 10⁹). The EIP-1559 formula increases the next block's L2 gas price by `P_N / 48 ≈ 2%`.

2. Submit an invoke transaction with `l2_gas.max_price_per_unit = P_N` (exactly the previous block's price). With `min_gas_price_percentage = 100` (default), the gateway threshold is `1.0 × P_N`, so the transaction passes `validate_tx_l2_gas_price_within_threshold`. [7](#0-6) 

3. The transaction is forwarded to the mempool and eventually picked up by the batcher for block N+1.

4. The batcher builds block N+1 with `l2_gas_price = P_{N+1} ≈ 1.02 × P_N` (set by the proposer in `ProposalInit`).

5. `perform_pre_validation_stage` → `check_fee_bounds` compares `max_price_per_unit = P_N` against `actual_gas_price = P_{N+1}`. Since `P_N < P_{N+1}`, `ResourceBoundsError::MaxGasPriceTooLow` is raised and the transaction is dropped from the block. [8](#0-7) 

The transaction was admitted by the gateway (passing all stateful checks including blockifier `__validate__`) but is silently excluded from the block, violating the invariant that gateway admission implies eventual inclusion for a correctly-formed transaction.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
