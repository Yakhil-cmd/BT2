### Title
Pool Admin Bypasses Factory Fee Caps via Uncapped `setBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary
The factory enforces `maxAdminSpreadFeeE6` to cap the pool admin's spread fee, but `setPoolBinAdditionalFees` forwards per-bin additional fees to the pool with **no cap check**, letting the pool admin charge fees that exceed the factory-owner-enforced limit.

### Finding Description

`setPoolAdminFees` correctly validates against the factory cap: [1](#0-0) 

```solidity
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
```

But `setPoolBinAdditionalFees` performs **no such check**: [2](#0-1) 

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

The pool stores and applies these values directly in swap math: [3](#0-2) 

```solidity
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
```

These per-bin fees are then **added on top of** the pool-level spread fee during every swap: [4](#0-3) 

```solidity
(curPosInBinCache, delta0Scaled, delta1Scaled, binLpFeeAmountScaled) = SwapMath.buyToken0InBinSpecifiedOut(
    binState,
    curPosInBinCache,
    state,
    params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6),
    ...
```

### Impact Explanation

`uint16` max = 65 535, which in E6 units equals **6.5535%** additional fee per bin. This is additive to the pool-level spread fee. If the factory owner sets `maxAdminSpreadFeeE6 = 0` to prohibit any admin spread fee, the pool admin can still impose up to 6.5535% per-bin fee on every swap through every bin, directly extracting value from traders beyond the factory-enforced boundary. The fee is charged on every swap that crosses the affected bin, causing direct, repeatable loss of trader principal.

### Likelihood Explanation

Any pool admin (a role granted at pool creation and transferable) can call `setPoolBinAdditionalFees` at any time with no timelock. The factory owner has no mechanism to prevent or override it after the fact, short of protocol-pausing the pool. The trigger requires only a valid pool admin key — no special conditions.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce a combined cap: `poolFeeConfig[pool].adminSpreadFeeE6 + addFeeBuyE6 <= maxAdminSpreadFeeE6`.

### Proof of Concept

1. Factory owner deploys factory and sets `maxAdminSpreadFeeE6 = 0` (no admin spread fee permitted).
2. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)`.
3. No revert occurs — the factory forwards the call directly.
4. Every swap through bin 0 now pays an additional 6.5535% buy/sell fee to the pool, which accrues as surplus and is collected by the pool admin via `collectPoolFees`, bypassing the factory owner's zero-cap intent. [2](#0-1)
