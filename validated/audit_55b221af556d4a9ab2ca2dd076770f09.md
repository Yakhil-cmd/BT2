### Title
Missing Validation for New-Joiner Signing Key in Reshare Initialization Causes Permanent Denial of Reshare — (`File: src/dkg.rs`)

---

### Summary

`assert_reshare_keys_invariants` contains a comment that explicitly states it should reject a new joiner (a participant not in the old participant set) who supplies a non-`None` `old_signing_key`. The code, however, only enforces the opposite direction: it rejects an *old* participant who supplies `None`. The missing guard allows a malicious new joiner to pass initialization, after which the protocol fails silently inside `do_keyshare` before any messages are sent, leaving every honest participant waiting indefinitely for the new joiner's Round-1 broadcast.

---

### Finding Description

In `src/dkg.rs`, `assert_reshare_keys_invariants` is the public initialization gate for both `reshare` and `refresh`. Its comment at line 662 reads:

> `// if me is not in the old participant set then ensure that old_signing_key is None`

The code that follows, however, only checks the *opposite* invariant:

```rust
// src/dkg.rs  L662-667
// if me is not in the old participant set then ensure that old_signing_key is None
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
```

The guard that *should* exist — `!old_participants.contains(me) && old_signing_key.is_some()` — is absent. A new joiner who passes `old_signing_key = Some(arbitrary_share)` therefore clears `assert_reshare_keys_invariants` without error.

`reshare` in `src/lib.rs` then forwards the same `old_signing_key` to `do_reshare`:

```rust
// src/lib.rs  L130-139
let fut = do_reshare(
    comms.shared_channel(),
    participants,
    me,
    threshold,
    old_signing_key,   // ← non-None for the malicious new joiner
    old_public_key,
    old_participants,
    rng,
);
```

Inside `do_reshare`, the secret is computed as:

```rust
// src/dkg.rs  L613-620
let secret = old_signing_key
    .map(|x_i| {
        intersection
            .lagrange::<C>(me)
            .map(|lambda| lambda * x_i.to_scalar())
    })
    .transpose()?
    .unwrap_or_else(<C::Group as Group>::Field::zero);
```

Because `me` is *not* in `old_participants`, it is also not in `intersection`. `intersection.lagrange::<C>(me)` evaluates the Lagrange basis polynomial for `me`'s scalar at zero relative to the intersection set; since `me`'s scalar is never zero (it is `u64::from(self.0) + 1`), the result is a non-zero scalar. The product `lambda * x_i.to_scalar()` is therefore non-zero.

`do_keyshare` is then called with this non-zero `secret` and `old_reshare_package = Some(...)`. The inner guard `assert_keyshare_inputs` catches it:

```rust
// src/dkg.rs  L39-44
} else {
    //  return error if me is part of the old participants set
    if !old_participants.contains(me) {
        return Err(ProtocolError::AssertionFailed(
            format!("{me:?} is running Resharing with a non-zero share but does not belong to the old participant set")));
    }
}
```

Critically, this check fires **before** any network I/O (the first `do_broadcast` call is at line 362, after `assert_keyshare_inputs` at lines 354-355). The malicious new joiner's protocol instance therefore aborts without ever sending its Round-1 session-ID broadcast. Every honest participant has already entered `do_broadcast` waiting for that message and will wait indefinitely, because the library is documented to wait forever for expected messages.

---

### Impact Explanation

**High — Permanent denial of reshare for honest parties.**

Honest participants invoke `do_broadcast` at line 362 of `do_keyshare` and block waiting for the new joiner's session-ID. Because the new joiner's instance has already returned an error and will never send that message, the honest participants are permanently stalled. The reshare (and any subsequent signing that depends on the new key shares) cannot complete. The library explicitly documents that callers must implement their own timeouts; absent that, the stall is indefinite.

---

### Likelihood Explanation

Any participant designated as a new joiner in a reshare session controls the `old_signing_key` argument it passes to `reshare`. No special privilege is required — the new joiner is by definition an unprivileged caller who does not hold a prior share. Supplying `Some(arbitrary_share)` instead of `None` is a single-line deviation from correct usage that passes the only initialization check meant to catch it.

---

### Recommendation

Add the symmetric guard that the comment already describes:

```rust
// src/dkg.rs  assert_reshare_keys_invariants
if old_participants.contains(me) && old_signing_key.is_none() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is present in the old participant list but provided no share"
    )));
}
// ADD: mirror check
if !old_participants.contains(me) && old_signing_key.is_some() {
    return Err(InitializationError::BadParameters(format!(
        "party {me:?} is not in the old participant list but provided a share"
    )));
}
```

This surfaces the error at `InitializationError` level before the protocol future is even created, preventing the stall.

---

### Proof of Concept

1. Run a normal keygen with participants `[P1, P2, P3]`, threshold 2.
2. Initiate a reshare to `[P1, P2, P3, P4]` (P4 is a new joiner).
3. P4 calls `reshare(old_participants=[P1,P2,P3], old_threshold=2, old_signing_key=Some(arbitrary_share), ...)`.
4. `assert_reshare_keys_invariants` returns `Ok(...)` — no error.
5. `do_reshare` computes a non-zero `secret` for P4 using `intersection.lagrange(P4) * arbitrary_share`.
6. `do_keyshare` calls `assert_keyshare_inputs`, which returns `Err(ProtocolError::AssertionFailed(...))` because P4 is not in `old_participants` but has a non-zero secret.
7. P4's protocol instance exits before sending any message.
8. P1, P2, P3 are blocked in `do_broadcast` at line 362 of `do_keyshare`, waiting for P4's session-ID broadcast that will never arrive.
9. The reshare is permanently stalled.

fn assert_keyshare_inputs<C: Ciphersuite>(
    me: Participant,
    secret: &Scalar<C>,
    old_reshare_package: Option<(VerifyingKey<C>, ParticipantList)>,
) -> Result<(Option<VerifyingKey<C>>, Option<ParticipantList>), ProtocolError> {
    let is_zero_secret = *secret == <C::Group as Group>::Field::zero();

    if let Some((old_key, old_participants)) = old_reshare_package {
        if is_zero_secret {
            //  return error if me is not a purely new joiner to the participants set
            //  prevents accidentally calling keyshare with extremely old keyshares
            //  that have nothing to do with the current resharing
            if old_participants.contains(me) {
                return Err(ProtocolError::AssertionFailed(
                    format!("{me:?} is running Resharing with a zero share but does belong to the old participant set")));
            }
        } else {
            //  return error if me is part of the old participants set
            if !old_participants.contains(me) {
                return Err(ProtocolError::AssertionFailed(
                    format!("{me:?} is running Resharing with a non-zero share but does not belong to the old participant set")));
            }
        }
        Ok((Some(old_key), Some(old_participants)))
    } else {
        if is_zero_secret {
            return Err(ProtocolError::AssertionFailed(format!(
                "{me:?} is running DKG with a zero share"
            )));
        }
        Ok((None, None))
    }
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
}

pub fn reshare<C: Ciphersuite>(
    old_participants: &[Participant],
    old_threshold: impl Into<ReconstructionLowerBound> + Send + 'static,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    new_participants: &[Participant],
    new_threshold: impl Into<ReconstructionLowerBound> + Copy + Send + 'static,
    me: Participant,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = KeygenOutput<C>>, InitializationError>
where
    Element<C>: Send,
    Scalar<C>: Send,
{
    let comms = Comms::new();
    let threshold = new_threshold;
    let (participants, old_participants) = assert_reshare_keys_invariants::<C>(
        new_participants,
        me,
        threshold,
        old_signing_key,
        old_threshold,
        old_participants,
    )?;
    let fut = do_reshare(
        comms.shared_channel(),
        participants,
        me,
        threshold,
        old_signing_key,
        old_public_key,
        old_participants,
        rng,
    );
    Ok(make_protocol(comms, fut))
}
