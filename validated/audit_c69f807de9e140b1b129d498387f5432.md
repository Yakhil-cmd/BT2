### Title
Pool Admin Bypasses Admin Fee Cap via Uncapped `setPoolBinAdditionalFees` — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

The factory enforces `maxAdminSpreadFeeE6` on `setPoolAdminFees`, but `setPoolBinAdditionalFees` passes through to the pool with **no cap check**. A pool admin can set per-bin additional fees to any `uint16` value (max 65 535 E6 ≈ 6.55%), increasing the total fee charged to traders and proportionally increasing the admin's absolute fee extraction beyond the intended cap.

---

### Finding Description

The factory defines two separate pool-admin fee paths:

**Path 1 — `setPoolAdminFees` (capped):** [1](#0-0) 

```solidity
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
```

**Path 2 — `setPoolBinAdditionalFees` (no cap):** [2](#0-1) 

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

The pool's `setBinAdditionalFees` also performs no value cap — it only validates the bin index: [3](#0-2) 

The per-bin values are stored as `uint16`, so the maximum settable value is **65 535 E6 ≈ 6.55%** per direction per bin.

In swap math, the effective fee applied to every trade in a bin is:

```solidity
uint256 buyFeeX64 = baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6);
``` [4](#0-3) 

The total fee charged to the trader (`feeAmountScaled`) is computed from this combined `buyFeeX64`. The protocol/admin share is then:

```solidity
uint256 protocolFeeAmountScaled = (feeAmountScaled * spreadFeeE6) / 1e6;
``` [5](#0-4) 

Because `feeAmountScaled` grows with `addFeeBuyE6`, the admin's absolute fee extraction (computed in `collectFees` as `feeAmountScaled * adminSpreadFeeE6 / spreadFeeE6`) also grows proportionally — **without any cap check on the bin additional fee input**.

The hard cap `HARD_MAX_SPREAD_FEE_E6 = 200 000` (20%) governs only the `spreadFeeE6` fraction: [6](#0-5) 

There is no corresponding hard cap on `addFeeBuyE6` / `addFeeSellE6`.

---

### Impact Explanation

- **Traders** pay up to 6.55% more per swap per bin than the oracle spread alone would imply, with no protocol-enforced ceiling.
- **Admin absolute fee extraction** scales with the inflated `feeAmountScaled`, so the admin collects more than `maxAdminSpreadFeeE6` was designed to allow in absolute terms.
- The pool admin can apply `addFeeBuyE6 = 65535` to every active bin, making the pool systematically more expensive than the capped fee schedule permits — a direct, repeatable loss of trader principal above Sherlock Medium thresholds for any non-trivial swap size.

---

### Likelihood Explanation

The pool admin is a **semi-trusted** role, trusted only within the caps the factory enforces. `setPoolBinAdditionalFees` is callable at any time by the pool admin with no timelock and no cap guard. The bypass requires a single transaction and is immediately effective on the next swap.

---

### Recommendation

Add a cap check inside `setPoolBinAdditionalFees` in `MetricOmmPoolFactory.sol`, analogous to the check in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > MAX_BIN_ADDITIONAL_FEE_E6) revert BinAdditionalFeeTooHigh();
    if (addFeeSellE6 > MAX_BIN_ADDITIONAL_FEE_E6) revert BinAdditionalFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Define `MAX_BIN_ADDITIONAL_FEE_E6` as a hard constant (e.g. equal to `HARD_MAX_SPREAD_FEE_E6` or a separate governance-settable cap), ensuring the pool admin cannot set per-bin fees beyond the protocol-intended ceiling.

---

### Proof of Concept

1. Factory is deployed with `maxAdminSpreadFeeE6 = 200_000` (20%).
2. Pool admin calls `setPoolAdminFees(pool, 200_000, 0)` — accepted, at the cap.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` — **no revert**, sets `addFeeBuyE6 = addFeeSellE6 = 65535` (≈6.55%) on bin 0.
4. A trader swaps through bin 0. The effective buy fee is `baseFeeX64 + 65535/1e6`, charging the trader ≈6.55% more than the oracle spread alone.
5. `protocolFeeAmountScaled = feeAmountScaled * spreadFeeE6 / 1e6` is proportionally larger; the admin's share of that (collected via `collectFees`) exceeds what `maxAdminSpreadFeeE6` was intended to bound in absolute terms.
6. The pool admin repeats step 3 on every bin — the entire pool now charges traders 6.55% above the oracle spread with no factory-level guard. [2](#0-1) [3](#0-2) [5](#0-4)
