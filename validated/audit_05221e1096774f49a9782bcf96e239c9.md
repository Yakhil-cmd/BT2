### Title
Pool `claim_rewards` Bypasses Staking Contract Pause Mechanism - (File: `src/pool/pool.cairo`)

### Summary
The `Pool` contract's `claim_rewards` function contains no pause check and does not query the Staking contract's `is_paused` state. When the Staking contract is paused in an emergency, pool members (delegators) can still drain rewards from the Pool contract, bypassing the protocol's emergency stop entirely.

### Finding Description
The Staking contract implements a global pause mechanism via `general_prerequisites()`, which asserts `!self.is_paused.read()` before any state-changing operation. Every user-facing function in `Staking` that moves funds calls this guard:

- `stake` [1](#0-0) 
- `claim_rewards` (staker) [2](#0-1) 
- `unstake_intent` [3](#0-2) 
- `add_stake_from_pool` [4](#0-3) 
- `remove_from_delegation_pool_action` [5](#0-4) 

The Pool contract, however, has **no pause storage and no pause interface**. Its `claim_rewards` function neither calls `general_prerequisites()` nor queries `IStaking::is_paused()`:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    // Asserts.
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    // ... no pause check ...
    let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
    reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());
    ...
}
``` [6](#0-5) 

Unlike `exit_delegation_pool_intent` or `exit_delegation_pool_action`, which call back into the Staking contract (and thus hit `general_prerequisites()` indirectly), `claim_rewards` in Pool transfers STRK directly from the Pool contract's own balance without any cross-contract call to Staking. The pause on the Staking contract has zero effect on this code path.

The asymmetry is explicit: staker-level `claim_rewards` in `Staking` is paused [7](#0-6) , but delegator-level `claim_rewards` in `Pool` is not. [8](#0-7) 

### Impact Explanation
**High — Theft of unclaimed yield.**

The pause mechanism exists precisely to halt all fund movements during emergencies such as a reward calculation bug. If rewards were over-distributed to a Pool contract (e.g., due to a minting curve miscalculation or an incorrect `update_rewards_from_staking_contract` call), the Security Agent can pause the Staking contract to stop further damage. However, pool members can immediately call `Pool::claim_rewards` to extract the already-distributed (potentially inflated) STRK rewards before any remediation is possible. The Staking contract's pause provides no protection for STRK already held in Pool contracts.

### Likelihood Explanation
**Medium.** The Staking contract's pause is an emergency tool. Any scenario that triggers a pause (reward miscalculation, accounting bug, exploit in progress) also creates the window where pool members race to claim rewards through the unguarded Pool path. The entry point is fully permissionless — any pool member or their registered reward address can call `Pool::claim_rewards` at any time.

### Recommendation
Add a pause check to `Pool::claim_rewards`. The simplest approach is to query the Staking contract's `is_paused()` view function at the start of the call:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "{}", Error::CONTRACT_IS_PAUSED);
    // ... rest of function
}
```

Alternatively, implement a dedicated pause flag in the Pool contract that the Staking contract sets when it pauses, mirroring the pattern used in `set_staker_removed`. [9](#0-8) 

### Proof of Concept
1. Staking contract accumulates a reward calculation bug; inflated rewards are forwarded to a Pool contract via `update_rewards_from_staking_contract`. [10](#0-9) 
2. Security Agent calls `Staking::pause()`, setting `is_paused = true`. [11](#0-10) 
3. All Staking functions now revert via `general_prerequisites()`. Staker `claim_rewards` is blocked.
4. Pool member calls `Pool::claim_rewards(pool_member)` directly on the Pool contract.
5. The function executes without any pause check, calculates inflated rewards from the cumulative trace, and transfers STRK to `reward_address`. [12](#0-11) 
6. Inflated yield is extracted before the protocol team can remediate the accounting error.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
