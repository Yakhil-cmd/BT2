### Title
Gateway L2 Gas Price Threshold Rounds Down, Admitting Transactions Below Intended Minimum - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `validate_tx_l2_gas_price_within_threshold` function in the gateway stateful path computes the minimum acceptable L2 gas price using floor division (`.to_integer()`), when it should use ceiling division. This mirrors the ERC4626 rounding-direction bug: a guard that should favor the protocol (reject borderline-low prices) instead favors the submitter, admitting transactions whose `max_price_per_unit` falls below the intended threshold.

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`, the threshold is computed as:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...)
}
``` [1](#0-0) 

`Ratio::to_integer()` from the `num_rational` crate returns the **floor** (truncation toward zero) of the rational number. For any `min_gas_price_percentage` value that does not divide evenly into 100, the computed threshold is strictly less than the mathematically exact value.

**Concrete example:**
- `min_gas_price_percentage = 90`, `previous_block_l2_gas_price = 11 FRI`
- Exact threshold: `(90/100) × 11 = 9.9 FRI`
- Computed threshold: `floor(9.9) = 9 FRI`
- A transaction with `max_price_per_unit = 9 FRI` satisfies `9 >= 9` and is **admitted**
- The intended invariant (`9 < 9.9`) would **reject** it

The analogous invariant in the external report is: "when calculating the amount a user must provide, round up." Here the invariant is: "when computing the minimum price a submitter must offer, round up." Both are broken by the same floor-vs-ceiling mistake.

The `increased_enough` function in the mempool correctly uses `div_ceil(100)` for fee escalation:

```rust
let Some(escalation_qualified_value) = existing_value
    .checked_mul(percentage)
    .map(|scaled| scaled.div_ceil(100))
    ...
``` [2](#0-1) 

The gateway threshold calculation is inconsistent with this correct pattern.

### Impact Explanation

This is a **High** impact finding matching: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

The gateway stateful path is the admission gate for all V3 transactions. A transaction admitted with a gas price 1 FRI below the intended floor will:
1. Pass `validate_tx_l2_gas_price_within_threshold` (broken invariant)
2. Enter the mempool
3. Proceed to blockifier execution at the next block's gas price

If the next block's actual L2 gas price equals or is below the previous block's price (the common case), the transaction also passes `check_fee_bounds` in the blockifier and executes successfully. The sequencer thus processes a transaction that the operator's configuration was designed to exclude.

### Likelihood Explanation

Triggered whenever:
1. `validate_resource_bounds = true` (the production default)
2. `min_gas_price_percentage` is set to any value whose product with the current L2 gas price is not an integer multiple of 100 (e.g., `min_gas_price_percentage = 90` and any gas price not divisible by 10)
3. A submitter crafts a transaction with `max_price_per_unit` equal to the floor-rounded threshold

Condition 3 requires no special privilege — any user can read the previous block's gas price and compute the floor threshold.

### Recommendation

Replace `.to_integer()` with `.ceil().to_integer()` so the threshold is rounded **up**, consistent with the "favor the protocol" invariant and with the `div_ceil` pattern already used in `increased_enough`:

```rust
// Before (rounds down):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();

// After (rounds up):
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
``` [3](#0-2) 

### Proof of Concept

```
min_gas_price_percentage = 90
previous_block_l2_gas_price = 11 FRI

Exact threshold  = 90/100 × 11 = 9.9 FRI
Floor threshold  = 9 FRI   ← what the code computes
Ceil  threshold  = 10 FRI  ← what the invariant requires

Attacker submits tx with max_price_per_unit = 9 FRI:
  tx_l2_gas_price.0 (9) < threshold (9)  →  false  →  tx ADMITTED
  (should be: 9 < 10  →  true  →  tx REJECTED)
```

The test `test_fee_escalation_rounds_up_required_bump` in the mempool explicitly documents that the analogous escalation check must round up: [4](#0-3) 

No equivalent ceiling-rounding test exists for `validate_tx_l2_gas_price_within_threshold`, confirming the gap.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
