### Title
Incorrect Multi-NFT Royalty Total Displayed in WalletConnect Offer Confirmation Causes User to Approve Transactions at Understated Cost - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
`formatAmountWithRoyalties` in `parseCommandDisplay.ts` divides the fungible payment amount by the number of NFTs before computing each royalty, producing a "Total Amount with Royalties" figure that is `1/n` of the correct value when `n > 1` NFTs are involved. This understated total is rendered in the WalletConnect confirmation dialog ("You Spend" / "Total Amount with Royalties"), causing a user to approve a `take_offer` or `create_offer_for_ids` command believing they will spend less than the blockchain will actually deduct.

### Finding Description

In `packages/gui/src/electron/commands/parseCommandDisplay.ts`, `formatAmountWithRoyalties` computes the royalty-inclusive total as follows:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← divides by n
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

When `royaltyPercentages.length === n`, the effective royalty computed is:

```
displayed_royalty = (amount / n) × Σ(royaltyPercentage_i) / 10_000
```

The correct formula is:

```
correct_royalty = amount × Σ(royaltyPercentage_i) / 10_000
```

The displayed royalty is exactly `1/n` of the correct royalty. The `amountWithRoyalties` field is then attached to the spending line and rendered in the WalletConnect confirmation dialog under the label **"Total Amount with Royalties"**: [2](#0-1) 

The `withRoyaltyTotals` function feeds this field into every fungible spending line when the opposite side of the offer contains NFTs: [3](#0-2) 

This path is reached for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` WalletConnect commands: [4](#0-3) 

The existing test for the multi-NFT case encodes the buggy value as the expected output, confirming the flaw is present and untested-against-correct-behavior: [5](#0-4) 

### Impact Explanation

A user receiving a WalletConnect request to take an offer containing two NFTs each with a 50% royalty and a 1 XCH payment would see:

- **Displayed**: "Total Amount with Royalties: 1.5 XCH" (`splitAmount = 0.5`, royalty = 0.5 XCH)
- **Actual on-chain deduction**: 2 XCH (royalty = 1 XCH, 50% per NFT on the full 1 XCH)

The user approves based on the understated figure. The blockchain enforces the correct royalty, so the user's wallet is debited 2 XCH despite the dialog showing 1.5 XCH. This constitutes displaying the wrong amount in a WalletConnect approval context, directly causing the user to authorize a spend larger than what was disclosed.

### Likelihood Explanation

The trigger condition — a WalletConnect `take_offer` or `create_offer_for_ids` request containing two or more NFTs with non-zero royalties — is a standard, supported offer type. Any dApp connected via WalletConnect can present such an offer. No special privileges, leaked keys, or social engineering beyond the normal WalletConnect approval flow are required. The discrepancy grows linearly with the number of NFTs and the royalty percentages involved.

### Recommendation

Remove the `splitAmount` division. Each NFT's royalty should be computed on the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

### Proof of Concept

Using the values from the existing test (`parseCommandDisplay.test.ts`, lines 251–335):

- 2 NFTs: royalty percentages 500 (5%) and 10 (0.1%)
- Payment: 100,000,000 mojos (0.0001 XCH)

**Current (buggy) calculation:**
- `splitAmount = 100,000,000 / 2 = 50,000,000`
- `royaltyAmount = (50,000,000 × 500)/10,000 + (50,000,000 × 10)/10,000 = 2,500,000 + 50,000 = 2,550,000`
- `totalAmount = 102,550,000 mojos → 0.00010255 XCH` ← shown to user

**Correct calculation:**
- `royaltyAmount = (100,000,000 × 500)/10,000 + (100,000,000 × 10)/10,000 = 5,000,000 + 100,000 = 5,100,000`
- `totalAmount = 105,100,000 mojos → 0.0001051 XCH` ← actual on-chain cost

The user is shown 0.00010255 XCH but pays 0.0001051 XCH — a 2.4% understatement in this mild case, scaling to 33% understatement with 3 NFTs, 50% with 4, etc.

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }
