### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` is specified to be callable only by the Starkware sequencer, but the implementation contains no such access control check. Any unprivileged caller can invoke `update_rewards(staker_address, disable_rewards: true)` to consume the global per-block reward slot without distributing rewards, permanently denying all stakers their yield for that block.

### Finding Description

The `update_rewards` function in `StakingRewardsManagerImpl` enforces a global one-call-per-block invariant via the `last_reward_block` storage field. The function writes `last_reward_block` to the current block number **before** the `disable_rewards` branch check:

```
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;   // ← returns with NO rewards distributed
}
```

The spec at `docs/spec.md` line 1644–1645 explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1447–1507 contains only:

1. `self.general_prerequisites()` — a pause check, not a sequencer check.
2. `assert!(current_block_number > self.last_reward_block.read(), ...)` — a replay guard.

There is no role check, no `assert_caller_is_sequencer`, and no equivalent guard. Any externally-owned address can call this function freely.

An attacker exploits this by calling `update_rewards(any_valid_staker_address, disable_rewards: true)` once per block. This:

1. Passes all assertions (contract not paused, staker exists and is active, block is new).
2. Writes `last_reward_block = current_block_number`.
3. Returns immediately without distributing any rewards.
4. Causes every subsequent call in the same block — including the legitimate sequencer call — to revert with `REWARDS_ALREADY_UPDATED`.

Because `last_reward_block` is a **single global field** (not per-staker), one attacker transaction per block is sufficient to deny rewards to **all** stakers for that block. Missed block rewards are never retroactively recovered; they are permanently lost.

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Each block the attacker frontruns the sequencer, all stakers permanently lose the block rewards that would have accrued. Over time this constitutes a complete denial of yield for the entire protocol. The attacker has no profit motive; this is pure griefing.

### Likelihood Explanation

**High.** The entry path requires no privileges, no tokens, and no special setup. Any address can call `update_rewards` on any valid staker. The attacker only needs to submit a transaction before the sequencer's reward-distribution transaction in each block. On Starknet, where the sequencer ordering is observable, this is straightforward to automate.

### Recommendation

Add a sequencer-only access control guard at the top of `update_rewards`, consistent with the spec. The simplest approach is to introduce a dedicated role (e.g., `REWARDS_MANAGER`) and check it:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_rewards_manager(); // ← add this
    self.general_prerequisites();
    ...
}
```

Alternatively, if the sequencer address is known at deployment, store it and assert `get_caller_address() == self.sequencer_address.read()`.

### Proof of Concept

1. Deploy the staking system with a valid staker `S`.
2. Advance `K` epochs so `S` has non-zero staking power.
3. Enable consensus rewards (`set_consensus_rewards_first_epoch`).
4. At the start of block `B`, attacker calls:
   ```
   IStakingRewardsManagerDispatcher { contract_address: staking_contract }
       .update_rewards(staker_address: S, disable_rewards: true);
   ```
5. `last_reward_block` is now set to `B`; no rewards are distributed.
6. The sequencer's legitimate call `update_rewards(S, disable_rewards: false)` reverts with `REWARDS_ALREADY_UPDATED`.
7. Staker `S` (and all other stakers) receive zero rewards for block `B`.
8. Repeat every block to permanently freeze all staker yield.
