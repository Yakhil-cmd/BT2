### Title
Missing Lower-Bound Validation on `threshold` in FROST Signing Allows Corrupted Protocol Outputs — (`src/frost/mod.rs`)

---

### Summary

The `assert_sign_inputs` and `presign` functions in `src/frost/mod.rs` validate that the `threshold` does not exceed the participant count but **never enforce a minimum value of 2**. This is directly inconsistent with the DKG layer (`src/dkg.rs`) which explicitly rejects `threshold < 2`. A malicious coordinator or misconfigured caller can invoke FROST signing with `threshold = 1` (or `0`), bypassing the intended security invariant and causing corrupted or unusable cryptographic outputs for all honest participants.

---

### Finding Description

The `assert_key_invariants` function in `src/dkg.rs` correctly enforces a lower bound: [1](#0-0) 

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
```

Similarly, `validate_triple_inputs` in the OT-based ECDSA triple generation enforces the same lower bound: [2](#0-1) 

However, the FROST signing layer omits this check entirely. `assert_sign_inputs` only validates the upper bound: [3](#0-2) 

```rust
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
```

There is no corresponding `threshold < 2` guard. The same omission exists in the FROST `presign` function: [4](#0-3) 

```rust
// validate threshold
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
```

The `ReconstructionLowerBound` type itself imposes no minimum — it is a plain `usize` newtype with no invariant: [5](#0-4) 

This means `ReconstructionLowerBound(1)` or `ReconstructionLowerBound(0)` are accepted without error by both `assert_sign_inputs` and `presign`.

---

### Impact Explanation

**High — Corruption of signing outputs so honest parties accept unusable cryptographic outputs.**

In FROST, the `threshold` value governs Lagrange interpolation during partial signature aggregation. If a signing session is initiated with `threshold = 1` while the key was generated with `threshold = 2` (a degree-1 polynomial), the Lagrange coefficients computed during signing are wrong. The aggregated signature will be cryptographically invalid and unverifiable. All honest participants who contributed valid partial signatures receive a corrupted, unusable output. With `threshold = 0`, Lagrange interpolation over an empty set causes a panic or arithmetic failure, permanently aborting the signing session.

---

### Likelihood Explanation

**Medium.** The entry point is any caller of the public `assert_sign_inputs` or `presign` functions in `src/frost/mod.rs`. These are library-level APIs. A malicious coordinator controlling the session setup, or a misconfigured participant, can supply `threshold = 1` without any library-level rejection. No special privilege is required beyond being a protocol participant who initiates or configures a signing round.

---

### Recommendation

Add the same lower-bound guard present in `assert_key_invariants` to both `assert_sign_inputs` and `presign` in `src/frost/mod.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This should be placed immediately after the upper-bound check in both functions, mirroring the pattern already established in `src/dkg.rs` at line 580 and `src/ecdsa/ot_based_ecdsa/triples/generation.rs` at line 699.

Alternatively, encode the invariant directly into `ReconstructionLowerBound` by making its constructor fallible and rejecting values below 2 at the type level.

---

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
