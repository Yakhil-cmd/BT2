### Title
Unauthorized Overwrite of Account-Owned Global Contract via Cross-Contract Call — (`File: runtime/runtime/src/global_contracts.rs`)

### Summary

`action_deploy_global_contract` in `runtime/runtime/src/global_contracts.rs` does not verify that the receipt's `predecessor_id` matches the `account_id` (receiver) when `deploy_mode == GlobalContractDeployMode::AccountId`. Any unprivileged contract can create a cross-contract receipt targeting a victim account and overwrite the victim's account-keyed global contract with arbitrary Wasm code. Every account that has opted into `GlobalContractIdentifier::AccountId(victim)` will subsequently execute the attacker-supplied code.

### Finding Description

`GlobalContractDeployMode::AccountId` is documented as an owner-controlled slot: the contract is stored under the deploying account's ID and "allows the owner to update the contract for all its users." [1](#0-0) 

The runtime enforces no such ownership. `action_deploy_global_contract` receives only the receipt receiver (`account_id`) and the action payload; the receipt's `predecessor_id` is never consulted: [2](#0-1) 

`initiate_distribution` then unconditionally uses `account_id` as the `GlobalContractIdentifier::AccountId` key: [3](#0-2) 

The static validation path (`validate_deploy_global_contract_action`) only checks contract size: [4](#0-3) 

No predecessor-vs-receiver check exists anywhere in the action application pipeline: [5](#0-4) 

The NEAR VM host function `promise_batch_action_deploy_global_contract_by_account_id` lets any executing contract append this action to a receipt targeting an arbitrary account: [6](#0-5) 

### Impact Explanation

**Corrupted value**: `TrieKey::GlobalContractCode { identifier: GlobalContractCodeIdentifier::AccountId(victim) }` is overwritten with attacker-supplied Wasm on every shard via the distribution receipt chain. [7](#0-6) 

Every account whose `AccountContract` is `GlobalByAccount(victim)` — set via a prior `UseGlobalContract` action — will execute the attacker's code on the next function call. This enables arbitrary code execution in the context of all victim-contract users: balance theft, storage corruption, and denial of service across an unbounded set of accounts.

### Likelihood Explanation

The attack requires only:
1. Deploying a malicious contract (standard NEAR account operation).
2. Calling a function that issues `promise_batch_create(victim)` followed by `promise_batch_action_deploy_global_contract_by_account_id`.
3. The victim account must exist and hold enough balance to cover the storage cost (`global_contract_storage_amount_per_byte × code_len`), which is charged to the victim — not the attacker. [8](#0-7) 

No validator privilege, no special key, and no admin access is required. The attacker pays only gas. Any account with sufficient balance is a valid target.

### Recommendation

In `action_deploy_global_contract`, when `deploy_mode == GlobalContractDeployMode::AccountId`, assert that the receipt's `predecessor_id` equals `account_id` (the receiver). If they differ, return an `ActionErrorKind` (e.g., `ActorNoPermission` or a new `GlobalContractOwnerMismatch` variant). The `predecessor_id` is available on the `Receipt` passed to the action application loop in `lib.rs` and can be threaded into `action_deploy_global_contract` or checked at the call site before dispatching.

### Proof of Concept

```rust
// Attacker's contract (deployed on attacker.near)
#[near_bindgen]
impl AttackerContract {
    pub fn overwrite_victim_global_contract(&self) {
        // Creates a receipt targeting victim.near
        let promise = env::promise_batch_create("victim.near");
        // Appends DeployGlobalContractByAccountId with malicious Wasm
        env::promise_batch_action_deploy_global_contract_by_account_id(
            promise,
            &MALICIOUS_WASM_BYTES,
        );
    }
}
```

When `overwrite_victim_global_contract` executes:
1. A `GlobalContractDistributionReceipt` is created with `id = GlobalContractIdentifier::AccountId("victim.near")` and a fresh nonce (higher than any prior nonce for that key).
2. The distribution propagates to all shards, writing the malicious Wasm to `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }`.
3. All accounts with `AccountContract::GlobalByAccount("victim.near")` now execute the attacker's Wasm on every subsequent function call. [9](#0-8)

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())
