### Title
Pausing the Engine Contract Permanently Freezes Funds in In-Flight `exit_to_near` wNEAR Unwrap Operations - (`engine/src/contract_methods/connector.rs`)

### Summary

`exit_to_near_precompile_callback` enforces `require_running` (i.e., the contract must not be paused) even though it is a NEAR promise callback that finalizes an already-committed EVM-side token burn. If the owner calls `pause_contract` between the EVM execution that burns the user's wNEAR ERC-20 tokens and the execution of the callback, the callback panics with `ERR_PAUSED`, the unwrapped NEAR tokens are permanently stranded in Aurora's account, and the user's ERC-20 tokens are already gone.

### Finding Description

The `exit_to_near` precompile supports a wNEAR-unwrap flow. When a user calls it with the `UnwrapWnear` message:

1. The EVM transaction burns the user's wNEAR ERC-20 tokens on Aurora.
2. A NEAR promise is scheduled to call `near_withdraw` on the wNEAR contract, moving NEAR into Aurora's account.
3. A callback `exit_to_near_precompile_callback` is scheduled to transfer those NEAR tokens to the user's target account. [1](#0-0) 

The callback is always scheduled when `transfer_near` is `Some`: [2](#0-1) 

Inside the callback, the very first substantive check is `require_running`: [3](#0-2) 

`require_running` returns `ERR_PAUSED` and aborts the callback if `state.is_paused == true`: [4](#0-3) 

`pause_contract` is callable by the owner at any time the contract is running: [5](#0-4) 

NEAR receipts (including promise callbacks) are processed asynchronously across blocks. There is a real window between the block that executes the EVM burn + `near_withdraw` and the block that executes `exit_to_near_precompile_callback`. If `pause_contract` lands in that window, the callback aborts. The NEAR runtime does not retry failed callbacks; the unwrapped NEAR is permanently stranded in Aurora's account with no recovery path, because the ERC-20 tokens were already burned in the EVM.

The same issue applies when the `error_refund` feature is compiled in: if the base `ft_transfer` promise fails and the contract is paused before the refund callback runs, the refund is also blocked and the burned ETH/ERC-20 tokens are permanently lost. [6](#0-5) 

### Impact Explanation

**Critical — Permanent freezing of funds.**

For the wNEAR unwrap path: the user's wNEAR ERC-20 tokens are burned in the EVM (irreversible), `near_withdraw` succeeds and moves NEAR into Aurora's account, but the callback that transfers those NEAR to the user is blocked. The NEAR is permanently locked in Aurora's account with no on-chain mechanism to recover it.

For the `error_refund` path: the user's ETH or ERC-20 tokens are burned, the outbound transfer fails, and the refund callback is blocked, resulting in permanent loss of those tokens.

### Likelihood Explanation

The window is narrow but real: NEAR processes receipts across blocks, and any `pause_contract` call that lands between the `near_withdraw` receipt and the `exit_to_near_precompile_callback` receipt triggers the bug. The owner does not need to act maliciously — a routine emergency pause during high-activity periods is sufficient. The wNEAR unwrap path is a documented, production-supported flow.

### Recommendation

Remove `require_running` from `exit_to_near_precompile_callback`. This callback is a system-internal receipt that finalizes an already-committed EVM state change; it must be allowed to complete regardless of the contract's pause state. The analogous fix in the referenced report was to remove `whenNotPaused` from `addBid` so that in-flight operations could complete even when the contract is paused. [7](#0-6) 

### Proof of Concept

1. User calls `submit` with an EVM transaction that calls the `exit_to_near` precompile with the wNEAR unwrap flag. The EVM burns the user's wNEAR ERC-20 tokens. A `PromiseArgs::Callback` is emitted with base=`near_withdraw` and callback=`exit_to_near_precompile_callback`. [8](#0-7) 

2. In the next NEAR block, `near_withdraw` executes successfully; NEAR tokens are now held by the Aurora contract.

3. Owner calls `pause_contract` in the same or a subsequent block before the callback receipt is processed. [9](#0-8) 

4. The NEAR runtime processes `exit_to_near_precompile_callback`. It reads state, hits `require_running`, and panics with `ERR_PAUSED`. [10](#0-9) 

5. The callback receipt fails. The NEAR tokens remain in Aurora's account. The user's wNEAR ERC-20 tokens are already burned. There is no on-chain recovery path.

```rust
pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
```

```rust
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
```
