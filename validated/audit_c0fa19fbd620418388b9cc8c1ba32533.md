### Title
Integer Division Truncation in WalletConnect Royalty Display Understates Actual Spend Amount - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts` uses BigInt integer division at two points when computing the royalty-inclusive total shown in the WalletConnect confirmation dialog. When multiple NFTs with royalties are involved, the displayed "You Spend" total is systematically lower than the amount that will actually be deducted from the user's wallet, causing the user to approve a transaction under false pretenses.

### Finding Description
In `packages/gui/src/electron/commands/parseCommandDisplay.ts`, the function `formatAmountWithRoyalties` computes the `amountWithRoyalties` string that is rendered in the WalletConnect confirmation dialog's "You Spend" section:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // truncating division #1
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,  // truncating division #2
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

Two cascading BigInt integer divisions truncate toward zero:

1. **`amount / BigInt(royaltyPercentages.length)`** — the total fungible amount is divided by the number of NFTs in the offer. Any remainder is silently discarded.
2. **`(splitAmount * BigInt(royaltyPercentage)) / 10_000n`** — the per-NFT royalty is computed on the already-truncated `splitAmount`, and the result is again truncated.

The resulting `amountWithRoyalties` string is passed through `withRoyaltyTotals` → `walletDeltaToDisplay` and ultimately rendered in `Confirm.tsx`'s `WalletDeltaSection` as the authoritative "You Spend" figure the user sees before clicking Approve. [2](#0-1) [3](#0-2) 

### Impact Explanation
The Chia blockchain's on-chain royalty transfer program calculates each NFT's royalty against the **full trade price**, not a split fraction. The GUI, however, divides the total amount by the number of NFTs before computing each royalty. This means the displayed total is lower than the actual on-chain deduction whenever more than one NFT with a royalty is present in the offer.

**Concrete example** — user buys 2 NFTs, each with 5% royalty (500 basis points), for 10,000 mojos total:

| Step | GUI display calculation | On-chain calculation |
|---|---|---|
| Per-NFT base | `10000 / 2 = 5000` (truncated) | `10000` (full price per NFT) |
| Royalty per NFT | `(5000 * 500) / 10000 = 250` | `(10000 * 500) / 10000 = 500` |
| Total royalty | `250 + 250 = 500` | `500 + 500 = 1000` |
| **Displayed total** | **10,500 mojos** | **11,000 mojos** |

The user approves believing they spend 10,500 mojos; the blockchain deducts 11,000 mojos. The discrepancy grows with the number of NFTs and the royalty percentage.

Additionally, for very small amounts (e.g., `splitAmount * royaltyPercentage < 10_000`), the second division truncates the royalty to `0n`, making the display show no royalty at all even when a non-zero royalty will be enforced on-chain.

### Likelihood Explanation
Any WalletConnect-connected dApp or malicious peer can craft a `take_offer` or `create_offer_for_ids` request containing multiple NFTs with royalties. The victim only needs to have a WalletConnect session open — no leaked keys or host compromise is required. The attacker controls the offer parameters (number of NFTs, royalty percentages, trade price) and can tune them to maximize the gap between the displayed and actual spend amount.

### Recommendation
Replace the two-step truncating division with a calculation that preserves precision:

```typescript
// Compute total royalty without splitting first
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) =>
    total + (amount * BigInt(royaltyPercentage)) / (10_000n * BigInt(royaltyPercentages.length)),
  0n,
);
```

Or, if the intent is to split the trade price equally among NFTs, apply a single rounding-aware split and document that this matches the on-chain formula. Also add a guard that refuses to display `amountWithRoyalties` when the computed royalty is `0n` but `royaltyPercentages` is non-empty, to avoid silently hiding a non-zero royalty obligation.

### Proof of Concept
1. Attacker establishes a WalletConnect session with the victim's Chia GUI.
2. Attacker sends a `chia_wallet.take_offer` command whose embedded offer summary contains:
   - `requested.xch = "10000"` (10,000 mojos)
   - Two NFT launcher IDs in `offered`, each with `royalty_percentage: 500` in `infos`
3. `parseCommandDisplay` calls `formatAmountWithRoyalties` with `amount = 10000n` and `royaltyPercentages = [500, 500]`.
4. `splitAmount = 10000n / 2n = 5000n`; each royalty = `(5000n * 500n) / 10000n = 250n`; total royalty = `500n`.
5. Confirmation dialog renders **"You Spend: 10,500 mojos"**.
6. Actual on-chain deduction: **11,000 mojos** (500 royalty per NFT × 2).
7. Victim approves; wallet is debited 500 mojos more than displayed. [4](#0-3) [5](#0-4)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }
