### Title
Unprivileged Caller Can Grief All Stakers' Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary
`IStakingRewardsManager::update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards: bool` parameter. Any unprivileged address can call `update_rewards(any_active_staker, disable_rewards: true)` to advance the global `last_reward_block` checkpoint without distributing any rewards. Because `last_reward_block` is a single global slot shared across all stakers, one such call per block permanently consumes that block's reward opportunity for every staker in the protocol.

---

### Finding Description

`IStakingRewardsManager` exposes `update_rewards` with no access-control guard beyond `general_prerequisites()` (which only checks the contract is unpaused and the caller is non-zero):

```
// src/staking/interface.cairo lines 304-311
pub trait IStakingRewardsManager<TContractState> {
    fn update_rewards(
        ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
    );
}
```

The implementation writes `last_reward_block` unconditionally before branching on `disable_rewards`:

```
// src/staking/staking.cairo lines 1453-1507
fn update_rewards(...) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // <-- global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // <-- no rewards distributed
    }
    // actual reward distribution follows only if disable_rewards == false
    ...
}
```

`last_reward_block` is a single storage slot (not per-staker). Once it is set to block N, every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`, regardless of which staker is targeted. An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in block N therefore silently discards the entire protocol's block-reward budget for block N.

The analog to the external report is direct: just as `cancelOrder()` exposes order data that lets a watcher fill the order before cancellation takes effect, `update_rewards` exposes a global checkpoint that lets any watcher (or proactive attacker) consume the block-reward slot before the legitimate sequencer/staker call can distribute rewards.

---

### Impact Explanation

**High — Temporary (but indefinitely sustainable) freezing of unclaimed yield for all stakers.**

- Every block in which the attacker submits `update_rewards(active_staker, true)` before the legitimate call, zero STRK/BTC block rewards are distributed to any staker.
- Because the attack requires only one cheap transaction per block and targets a global slot, a single attacker can suppress rewards for the entire validator set continuously.
- Stakers and delegators accumulate no `unclaimed_rewards_own` / pool rewards for griefed blocks; those rewards are permanently lost (the minting curve does not retroactively compensate for missed blocks).

---

### Likelihood Explanation

**Medium.** On Starknet the sequencer controls ordering, so classic mempool front-running is limited. However, the attacker does not need to front-run a specific transaction: they can simply submit `update_rewards(any_active_staker, true)` in every block independently. The only cost is one transaction per block. Given that Starknet gas fees are low and the protocol's total reward stream is large, the attack is economically rational for a motivated griever (e.g., a competing validator set or a short-seller of STRK).

---

### Recommendation

1. **Restrict `disable_rewards: true` to privileged callers.** Add a role check (e.g., `only_security_agent` or `only_app_governor`) when `disable_rewards` is `true`, so only trusted parties can advance `last_reward_block` without distributing rewards.
2. **Alternatively, remove the `disable_rewards` parameter from the public interface** and handle the "no-reward" path internally (e.g., when `is_pre_consensus()` is true or when the staker is being removed), so external callers can never suppress reward distribution.
3. **Consider making `last_reward_block` per-staker** rather than global, so a griefing call against one staker cannot affect others.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Active staker `S` exists with non-zero STRK balance.
3. Attacker `A` (any EOA) submits in block N:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Inside `update_rewards`:
   - `current_block_number (N) > last_reward_block` → assertion passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → early return; `_update_rewards` is never called.
5. Any subsequent call to `update_rewards` in block N (by the sequencer or any staker) reverts with `REWARDS_ALREADY_UPDATED`.
6. All stakers receive zero block rewards for block N.
7. Attacker repeats step 3 in block N+1, N+2, … indefinitely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
