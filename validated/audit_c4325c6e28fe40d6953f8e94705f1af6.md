### Title
Gateway Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Incorrect Transaction Admission/Rejection - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` gates admission on `previous_block.l2_gas_price` (the price that was used for the already-committed block), but the block the transaction will actually execute in uses `previous_block.next_l2_gas_price` as its `l2_gas_price`. The two values diverge every block via the fee-market algorithm. The code itself acknowledges the bug with a TODO comment. The result is that the gateway's admission threshold is computed against the wrong price: when the price is rising, transactions are admitted that will fail at blockifier execution; when the price is falling, valid transactions are rejected.

---

### Finding Description

**Root cause — wrong field read in `validate_resource_bounds`**

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price for the threshold check from `GatewayFixedBlockStateReader::get_block_info()`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `BlockInfo.gas_prices.strk_gas_prices.l2_gas_price` from `block_header.l2_gas_price.price_in_fri`:

```rust
strk_gas_prices: GasPriceVector {
    ...
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
},
``` [2](#0-1) 

`block_header.next_l2_gas_price` — a distinct field in `BlockHeaderWithoutHash` — is never read here and is not exposed through the `BlockInfo` struct at all. [3](#0-2) 

**What the batcher actually uses**

The consensus orchestrator computes `next_l2_gas_price` via `calculate_next_l2_gas_price_for_fin` and places it in `ProposalInit.l2_gas_price_fri`. The batcher builds the block context from `ProposalInit`, so every transaction executed in the new block is checked against `previous_block.next_l2_gas_price`, not `previous_block.l2_gas_price`. [4](#0-3) 

The blockifier's `check_fee_bounds` then enforces:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [5](#0-4) 

where `actual_gas_price` is the block context's `l2_gas_price` — i.e., `previous_block.next_l2_gas_price`.

**The invariant broken**

The gateway's admission threshold and the blockifier's execution threshold must be computed from the same price. They are not:

| Stage | Price used |
|---|---|
| Gateway `validate_resource_bounds` | `previous_block.l2_gas_price` |
| Blockifier `check_fee_bounds` at execution | `previous_block.next_l2_gas_price` |

These two values diverge every block. The fee market can move the price by up to ~0.3% per block (denominator 333), and by much more when the minimum price schedule (`min_l2_gas_price_per_height`) applies. [6](#0-5) 

---

### Impact Explanation

**Scenario A — price rising (busy network):**
`next_l2_gas_price > l2_gas_price`. A transaction with `max_price_per_unit` in the range `[l2_gas_price, next_l2_gas_price)` passes the gateway threshold check and is admitted to the mempool, but `check_fee_bounds` at execution time rejects it with `MaxGasPriceTooLow`. The gateway has admitted a transaction that will never execute.

**Scenario B — price falling (quiet network):**
`next_l2_gas_price < l2_gas_price`. A transaction with `max_price_per_unit` in the range `[next_l2_gas_price, threshold)` is rejected by the gateway even though it would satisfy `check_fee_bounds` at execution. A valid transaction is incorrectly rejected.

Both scenarios are reachable by any unprivileged user submitting a standard V3 (`AllResources`) transaction. The gap between the two prices grows with network load and with the `min_l2_gas_price_per_height` schedule, which can jump the price by an arbitrary configured amount at a given block height. [7](#0-6) 

---

### Likelihood Explanation

The fee market adjusts `next_l2_gas_price` every block. The two prices are equal only when the block is exactly at the gas target. In practice, on a live network with variable load, they differ on every block. The discrepancy is small per block (~0.3%) but is amplified when `min_l2_gas_price_per_height` enforces a step-change minimum. The bug is therefore continuously active in production.

---

### Recommendation

`validate_resource_bounds` must read `next_l2_gas_price` from the block header, not `l2_gas_price`. The `GatewayFixedBlockStateReader` trait (or a new helper) should expose `next_l2_gas_price` from `BlockHeaderWithoutHash`, and `validate_resource_bounds` should use that value as the threshold base. The existing TODO comment at line 229 already identifies the fix needed. [8](#0-7) 

---

### Proof of Concept

1. Observe the previous committed block header: `l2_gas_price = P`, `next_l2_gas_price = P'` where `P' > P` (network is busy, price is rising).
2. Submit a V3 `InvokeTransaction` with `resource_bounds.l2_gas.max_price_per_unit = P` (exactly equal to the old price, above the 100% threshold).
3. Gateway `validate_resource_bounds` computes threshold = `100% × P = P`. The transaction satisfies `P >= P` → **admitted**.
4. The batcher builds the next block with `l2_gas_price = P'` (from `next_l2_gas_price`).
5. Blockifier `check_fee_bounds` checks `P >= P'` → **false** → `MaxGasPriceTooLow` → transaction rejected at execution.

The gateway has accepted a transaction that the blockifier will always reject, violating the admission invariant. With `min_l2_gas_price_per_height` configured to jump the price at a known block height, an attacker can predict the exact block where the gap is largest and flood the mempool with transactions that pass admission but fail execution. [9](#0-8) [10](#0-9)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

                            block_info.gas_prices.gas_price_vector(fee_type);

                                minimal_gas_amount_vector.l1_data_gas,

                                L2Gas,

                                minimal_gas_amount_vector.l2_gas,
                                *l2_gas_price,

                        ]

                let insufficiencies = resources_amount_tuple
                    .iter()
                    .flat_map(
                        |(resource, resource_bounds, minimal_gas_amount, actual_gas_price)| {
                            let mut insufficiencies_resource = vec![];
                            if minimal_gas_amount > &resource_bounds.max_amount {
                                insufficiencies_resource.push(
                                    ResourceBoundsError::MaxGasAmountTooLow {
                                        resource: *resource,
                                        max_gas_amount: resource_bounds.max_amount,
                                        minimal_gas_amount: *minimal_gas_amount,
                                    },
                                );
                            }
                            if resource_bounds.max_price_per_unit < actual_gas_price.get() {
                                insufficiencies_resource.push(
                                    ResourceBoundsError::MaxGasPriceTooLow {
                                        resource: *resource,
                                        max_gas_price: resource_bounds.max_price_per_unit,
                                        actual_gas_price: (*actual_gas_price).into(),
                                    },
                                );
                            }
                            insufficiencies_resource
                        },
                    )
                    .collect::<Vec<_>>();
                if !insufficiencies.is_empty() {
                    return Err(Box::new(TransactionFeeError::InsufficientResourceBounds {

                    }))?;
                }
            }

pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
