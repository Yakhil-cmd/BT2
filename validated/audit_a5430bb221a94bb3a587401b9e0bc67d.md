### Title
Gateway L2 Gas Price Admission Check Uses Wrong Block Price (`l2_gas_price` Instead of `next_l2_gas_price`) — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful admission check for L2 gas price compares a transaction's `max_price_per_unit` against the **current block's** `l2_gas_price`, but the correct reference is the **next block's** `next_l2_gas_price` (the EIP-1559-derived price that will actually govern the block the transaction enters). The developers acknowledge this in a `TODO` comment at the exact site of the bug. The mismatch causes the gateway to reject valid transactions when gas prices are falling, and to admit transactions that will fail the blockifier's fee check when gas prices are rising.

---

### Finding Description

In `validate_resource_bounds`, the gateway reads `BlockInfo` from `GatewayFixedBlockStateReader` and extracts `strk_gas_prices.l2_gas_price`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`l2_gas_price` is the price **of** the latest committed block — the price that was used for transactions already included in that block. The price that will govern the **next** block is `next_l2_gas_price`, which is stored separately in `BlockHeaderWithoutHash` and computed via EIP-1559 fee market logic:

```rust
pub struct StorageBlockHeader {
    pub l2_gas_price: GasPricePerToken,   // price of this block
    pub next_l2_gas_price: GasPrice,       // price for the next block
    ...
}
``` [2](#0-1) 

The `GatewayFixedBlockSyncStateClient` converts `BlockHeaderWithoutHash` into `BlockInfo`, but `BlockInfo` has no `next_l2_gas_price` field, so `next_l2_gas_price` is silently dropped:

```rust
let block_info = BlockInfo {
    ...
    strk_gas_prices: GasPriceVector {
        l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
        // next_l2_gas_price is not forwarded
    },
    ...
};
``` [3](#0-2) 

The threshold check itself is:

```rust
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0).to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(StarknetError { code: GAS_PRICE_TOO_LOW, ... });
}
``` [4](#0-3) 

With the default `min_gas_price_percentage = 100`, the threshold equals `l2_gas_price` exactly — the price of the **previous** block, not the price of the **next** block. [5](#0-4) 

Meanwhile, the blockifier's `check_fee_bounds` (called during `perform_pre_validation_stage`) compares the transaction's `max_price_per_unit` against the **actual block gas prices** — which are set to `next_l2_gas_price` when the block is built:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [6](#0-5) 

The two checks use different reference prices, creating a split-horizon admission invariant.

---

### Impact Explanation

**Scenario A — Rising gas prices (`next_l2_gas_price > l2_gas_price`, i.e., previous block was heavily loaded):**

A user submits a transaction with `max_price_per_unit = l2_gas_price`. The gateway admits it (threshold = `l2_gas_price`, check passes). The batcher builds the next block at `next_l2_gas_price`. The blockifier's `check_fee_bounds` rejects the transaction because `l2_gas_price < next_l2_gas_price`. The transaction was admitted through the gateway and mempool but fails at execution, wasting sequencer resources and causing unexpected user-facing failures.

**Scenario B — Falling gas prices (`next_l2_gas_price < l2_gas_price`, i.e., previous block was lightly loaded):**

A user submits a transaction with `max_price_per_unit = next_l2_gas_price` (the correct price for the next block). The gateway rejects it with `GAS_PRICE_TOO_LOW` because `next_l2_gas_price < l2_gas_price = threshold`. The transaction is valid and would succeed in the blockifier, but is denied at the gateway. This is a direct, unprivileged, deterministic denial of valid transactions.

Both scenarios are triggered by normal EIP-1559 fee market dynamics — no special privileges required.

---

### Likelihood Explanation

The EIP-1559 fee market (`calculate_next_base_gas_price`) adjusts `next_l2_gas_price` every block based on gas consumption relative to `gas_target`. Any block that is not exactly at target utilization produces a `next_l2_gas_price ≠ l2_gas_price`. This is the common case in production. The divergence grows with block utilization variance, making Scenario B (valid-tx rejection) a routine occurrence during periods of decreasing load. [7](#0-6) 

---

### Recommendation

1. Extend `GatewayFixedBlockStateReader::get_block_info` (or add a new method) to return `next_l2_gas_price` from `BlockHeaderWithoutHash`.
2. In `validate_resource_bounds`, replace `block_info.gas_prices.strk_gas_prices.l2_gas_price` with the `next_l2_gas_price` value from the block header.
3. Update `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` to propagate `block_header.next_l2_gas_price` through the interface. [8](#0-7) 

---

### Proof of Concept

1. Observe the latest committed block has `l2_gas_price = P` and `next_l2_gas_price = P * 0.9` (lightly loaded block, price decreasing by ~10%).
2. Submit an invoke transaction with `resource_bounds.l2_gas.max_price_per_unit = P * 0.95` (above `next_l2_gas_price`, so valid for the next block).
3. The gateway calls `validate_resource_bounds`, computes `threshold = 100% * P = P`, and rejects the transaction with `GAS_PRICE_TOO_LOW` because `P * 0.95 < P`.
4. The transaction would have passed `check_fee_bounds` in the blockifier (since `P * 0.95 > P * 0.9 = next_l2_gas_price`).
5. The user must overpay (`max_price_per_unit >= P`) to be admitted, even though the next block only requires `P * 0.9`.

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
