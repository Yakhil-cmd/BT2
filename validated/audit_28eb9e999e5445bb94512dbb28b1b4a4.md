### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `Staking.cairo` is publicly callable with no access control and accepts a caller-controlled `disable_rewards: bool` parameter. When `disable_rewards: true`, the function consumes the single per-block reward slot (by writing `last_reward_block`) without distributing any rewards. An unprivileged attacker can call this in every block to permanently freeze all consensus reward accrual for every staker in the protocol.

---

### Finding Description

`update_rewards` is defined in `StakingRewardsManagerImpl` with no role check: [1](#0-0) 

The function enforces a global, per-block mutex via `last_reward_block`: [2](#0-1) 

After validating the staker, it unconditionally writes the current block number to `last_reward_block`: [3](#0-2) 

Then, if `disable_rewards` is `true`, it returns early with no reward distribution: [4](#0-3) 

Because `last_reward_block` is a **global** storage field (not per-staker), only one `update_rewards` call is permitted per block across the entire protocol: [5](#0-4) 

An attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block. Each call:
1. Passes all staker-validity checks (staker exists, is active, has non-zero balance).
2. Writes `last_reward_block = current_block`.
3. Returns early — zero rewards distributed.

Any legitimate keeper or staker attempting `update_rewards` in the same block hits `REWARDS_ALREADY_UPDATED` and reverts. The attacker repeats this every block, permanently starving the entire protocol of consensus rewards.

---

### Impact Explanation

This is **Permanent freezing of unclaimed yield** (High severity per the allowed impact scope). Once the attacker begins the campaign, no staker in the protocol can ever accumulate consensus block rewards. The `unclaimed_rewards_own` field in every staker's `InternalStakerInfoLatest` struct never increases. Delegators' `cumulative_rewards_trace` in `Pool.cairo` is never updated. The yield is not merely delayed — it is destroyed for every block the attack runs. [6](#0-5) 

---

### Likelihood Explanation

**High.** The entry point is a public function with no role restriction. The attacker needs only a valid staker address (readable from the public `stakers` vector) and enough gas to submit one transaction per block. On Starknet, per-block transaction costs are low. There is no economic barrier; the attacker gains nothing but inflicts maximum damage on all protocol participants. The attack is trivially automatable with a simple keeper script.

---

### Recommendation

1. **Remove the `disable_rewards` parameter from the public interface.** If reward disabling is needed (e.g., for penalizing non-attesting stakers), gate it behind a privileged role (`security_agent` or `app_governor`) or derive it deterministically from on-chain attestation state rather than accepting it as caller input.
2. **Alternatively**, make `last_reward_block` per-staker rather than global, so one attacker call cannot block all other stakers.
3. **At minimum**, add an access-control check (e.g., `only_app_governor`) to `update_rewards` to prevent unprivileged callers from invoking it with arbitrary parameters.

---

### Proof of Concept

```
// Attacker script (pseudocode, runs every block)
loop every new Starknet block:
    valid_staker = staking.stakers[0]   // any active staker from public vector
    staking.update_rewards(
        staker_address = valid_staker,
        disable_rewards = true          // caller-controlled, no check
    )
    // Effect: last_reward_block = current_block, zero rewards distributed
    // Any legitimate update_rewards call this block → REWARDS_ALREADY_UPDATED
```

**Step-by-step:**

1. Consensus rewards become active (`consensus_rewards_first_epoch` is set and `current_epoch >= consensus_rewards_first_epoch`).
2. Attacker reads any valid staker address from the public `stakers` storage vector.
3. Each block, attacker calls `update_rewards(valid_staker, disable_rewards: true)`.
4. `last_reward_block` is set to the current block number at line 1485.
5. The `if disable_rewards || self.is_pre_consensus() { return; }` branch at line 1487 fires — `_update_rewards` is never reached.
6. Any honest call to `update_rewards` in the same block fails at the `REWARDS_ALREADY_UPDATED` assertion (line 1455–1458).
7. Staker `unclaimed_rewards_own` is never incremented; pool `cumulative_rewards_trace` is never updated.
8. All protocol yield is permanently frozen for the duration of the attack. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
