### Title
Unbounded `pool_member_epoch_balance` Trace Causes Linear-Time `calculate_rewards` Loop, Enabling Permanent Freeze of Delegator Unclaimed Yield - (File: `src/pool/pool.cairo`)

---

### Summary

The `calculate_rewards` internal function in the Pool contract iterates over every entry in a per-delegator balance-change trace with no upper bound and no pagination. The trace grows by one checkpoint per epoch in which the delegator changes their balance. A delegator who actively manages their delegation across many epochs without claiming rewards will eventually cause `claim_rewards` to exceed the Starknet transaction gas limit, permanently freezing their unclaimed yield.

---

### Finding Description

`calculate_rewards` in `src/pool/pool.cairo` iterates over the entire `pool_member_epoch_balance` trace from `entry_to_claim_from` to `pool_member_trace_length`:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The codebase itself acknowledges the loop is unbounded. The `pool_member_epoch_balance` trace is a `Vec`-backed checkpoint structure with no enforced cap: [2](#0-1) 

A new checkpoint is appended whenever the delegator's balance changes in a new epoch (via `delegate`, `add_to_delegation_pool`, or `exit_intent`). The `insert` function only deduplicates within the same epoch; a change in each distinct epoch always appends: [3](#0-2) 

There is no maximum trace length enforced anywhere in the pool contract. Inside the loop, `find_sigma` is called once per checkpoint, adding further per-iteration cost: [4](#0-3) 

---

### Impact Explanation

A delegator who changes their balance once per epoch (the minimum required to grow the trace) and defers claiming rewards accumulates one checkpoint per epoch indefinitely. When `claim_rewards` is eventually called, `calculate_rewards` must iterate over every accumulated checkpoint. Once the trace is long enough, the transaction will exceed the Starknet per-transaction gas limit and revert. Because the entire reward computation must succeed atomically and there is no pagination mechanism, the delegator's unclaimed yield becomes permanently inaccessible — matching the **"Permanent freezing of unclaimed yield"** (High) impact category.

---

### Likelihood Explanation

Any pool member (unprivileged delegator) can grow their own trace by calling `add_to_delegation_pool` or `exit_intent` once per epoch. The growth rate is one checkpoint per epoch, so the attack is slow but entirely self-executable without any privileged access. Long-lived delegators who actively rebalance their stake across many epochs are the natural victims; they need not act maliciously — normal protocol usage is sufficient to trigger the condition over time.

---

### Recommendation

1. **Enforce a maximum trace length per pool member** (e.g., `MAX_BALANCE_CHECKPOINTS = 512`) and reject balance-changing operations that would exceed it, or require the member to claim rewards first (resetting `entry_to_claim_from`).
2. **Add pagination to `claim_rewards`**: store `entry_to_claim_from` in persistent pool-member state and allow partial reward claims across multiple transactions, so no single call must traverse the entire trace.
3. **Prune stale checkpoints** during reward claims: once a checkpoint has been fully accounted for, it can be removed or its index advanced, bounding the work for future calls.

---

### Proof of Concept

1. Delegator calls `enter_delegation_pool` to join a pool.
2. Each epoch, delegator calls `add_to_delegation_pool` with a minimal amount — this triggers `pool_member_epoch_balance.insert` with a new epoch key, appending one checkpoint per epoch. [5](#0-4) 
3. After `N` epochs of balance changes without claiming, `pool_member_trace_length == N`.
4. Delegator calls `claim_rewards`. The pool contract calls `calculate_rewards`, which enters the `while` loop and executes `N` iterations, each calling `find_sigma`. [6](#0-5) 
5. For sufficiently large `N`, the transaction exceeds the Starknet gas limit and reverts. All subsequent `claim_rewards` calls also revert. The delegator's accumulated yield is permanently frozen.

    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
