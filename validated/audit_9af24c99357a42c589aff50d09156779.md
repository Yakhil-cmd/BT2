### Title
Source Coin (`cronos0x`) Token Mapping Is Permanently Irremovable and Irremappable Once Registered - (`x/cronos/keeper/keeper.go`)

### Summary

`RegisterOrUpdateTokenMapping` in `x/cronos/keeper/keeper.go` has an asymmetric deletion path: non-source coin mappings (`ibc/`, `gravity0x`) can be deleted by passing an empty contract string, but source coin mappings (`cronos0x...`) have no deletion path at all. Compounding this, `SetExternalContractForDenom` now enforces `ensureDenomNotMapped`, which permanently blocks re-registration of any already-mapped denom. Once a `cronos0x...` source denom is mapped to a CRC21 contract, neither the admin (`MsgUpdateTokenMapping`) nor governance (`TokenMappingChangeProposal`) can remove or replace it through any on-chain message path.

### Finding Description

`RegisterOrUpdateTokenMapping` branches on `types.IsSourceCoin(msg.Denom)`:

```
// x/cronos/keeper/keeper.go:330-404
if types.IsSourceCoin(msg.Denom) {
    // ... validates and calls SetExternalContractForDenom
    // NO deletion path exists here
} else {
    if len(msg.Contract) == 0 {
        // delete existing mapping  ← only reachable for non-source coins
        k.DeleteExternalContractForDenom(ctx, msg.Denom)
    } else { ... }
}
``` [1](#0-0) 

The non-source branch explicitly handles `len(msg.Contract) == 0` as a deletion signal. The source coin branch has no equivalent. [2](#0-1) 

`SetExternalContractForDenom` enforces `ensureDenomNotMapped`, which returns `ErrDenomAlreadyMapped` if the denom already resolves to any contract: [3](#0-2) 

`ensureDenomNotMapped` calls `GetContractByDenom` and rejects any denom that already has an active mapping: [4](#0-3) 

The governance path (`TokenMappingChangeProposal`) routes through the same `RegisterOrUpdateTokenMapping` function and is equally blocked: [5](#0-4) 

The spec itself acknowledges this: "There's no way to delete a token mapping currently." [6](#0-5) 

`DeleteExternalContractForDenom` exists as a low-level keeper primitive but is only invoked from the non-source coin branch of `RegisterOrUpdateTokenMapping` — it is never reachable for source coin denoms through any admin-accessible message. [7](#0-6) 

### Impact Explanation

**High — Corruption of token mappings / permanent inability to update source coin denom-contract binding with direct security impact.**

A `cronos0x...` source denom encodes a CRC21 contract address in the denom string itself. Once the admin registers this mapping, the EVM hooks (`SendToIbcHandler`, `ConvertVouchersToEvmCoins`, `BurnVouchersToEvmCoins`) permanently route all bridge and conversion flows through that contract. [8](#0-7) 

If the mapped CRC21 contract is later found to be exploitable (e.g., an unauthorized `mint_by_cronos_module` path, a reentrancy in `burn_by_cronos_module`, or a backdoor), an unprivileged attacker can exploit it continuously. The admin has no on-chain mechanism to remove or replace the mapping to stop the attack. Every IBC receive packet that triggers `ConvertVouchersToEvmCoins` or every EVM hook event continues to route through the vulnerable contract. This constitutes a permanent inability to halt a live bridge/conversion exploit, matching the **High** impact category: "Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows... under normal network assumptions."

Additionally, if the admin accidentally registers the wrong contract for a source denom, there is no correction path — the mapping is permanently locked.

### Likelihood Explanation

The admin must have previously registered a source coin mapping (a normal, expected operation for `cronos0x...` tokens). The vulnerability activates the moment any such mapping is registered. Given that source coin mappings are a core feature of the Cronos bridge architecture and are expected to be used in production, the precondition is routinely satisfied. The exploiting attacker is fully unprivileged — they only need to interact with the vulnerable CRC21 contract after the mapping is locked in.

### Recommendation

Add a deletion path for source coin mappings in `RegisterOrUpdateTokenMapping`. The simplest fix mirrors the non-source coin branch:

```go
if types.IsSourceCoin(msg.Denom) {
    if len(msg.Contract) == 0 {
        // Allow admin/governance to remove a source coin mapping
        k.DeleteExternalContractForDenom(ctx, msg.Denom)
        return nil
    }
    // ... existing registration logic
}
```

Additionally, consider whether `SetExternalContractForDenom` should permit updates (not just creates) for source coin denoms when called from the privileged admin path, so that a vulnerable contract can be replaced with a patched one without requiring a full delete-then-re-register cycle.

### Proof of Concept

1. Admin calls `MsgUpdateTokenMapping` with `denom = "cronos0x<ContractA>"`, `contract = "<ContractA>"`. Mapping is registered successfully.
2. `ContractA` is discovered to have a vulnerability allowing unauthorized minting.
3. Admin attempts to remove the mapping by calling `MsgUpdateTokenMapping` with `denom = "cronos0x<ContractA>"`, `contract = ""`. The source coin branch is entered; there is no `len(msg.Contract) == 0` check, so the call proceeds to `common.IsHexAddress("")` which returns false, and the call returns `ErrInvalidAddress`. The mapping is not removed. [9](#0-8) 
4. Admin attempts to remap by calling `MsgUpdateTokenMapping` with `denom = "cronos0x<ContractA>"`, `contract = "<ContractB>"`. `validateContractAddressForSourceDenom` rejects this because the contract encoded in the denom (`ContractA`) does not match `ContractB`. [10](#0-9) 
5. Admin attempts to register the same denom again with the correct contract. `SetExternalContractForDenom` calls `ensureDenomNotMapped`, which finds the existing mapping and returns `ErrDenomAlreadyMapped`. [11](#0-10) 
6. Governance submits a `TokenMappingChangeProposal` — it routes through the same `RegisterOrUpdateTokenMapping` and hits the same blocks. [5](#0-4) 
7. The attacker continues exploiting `ContractA` indefinitely through the permanently locked bridge mapping.

// SetExternalContractForDenom sets denom→external CRC21 mapping for an unmapped denom.
// Caller is responsible for source-denom specific validation before calling this method.
func (k Keeper) SetExternalContractForDenom(ctx sdk.Context, denom string, address common.Address) error {
	if err := k.ensureDenomNotMapped(ctx, denom); err != nil {
		return err
	}
	if err := k.ensureContractNotMapped(ctx, denom, address); err != nil {
		return err
	}

	store := ctx.KVStore(k.storeKey)
	store.Set(types.DenomToExternalContractKey(denom), address.Bytes())
	store.Set(types.ContractToDenomKey(address.Bytes()), []byte(denom))
	return nil
}

// DeleteExternalContractForDenom delete the external contract mapping for native denom,
// returns false if mapping not exists.
func (k Keeper) DeleteExternalContractForDenom(ctx sdk.Context, denom string) bool {
	store := ctx.KVStore(k.storeKey)
	contract, found := k.getExternalContractByDenom(ctx, denom)
	if !found {
		return false
	}
	store.Delete(types.DenomToExternalContractKey(denom))
	deleteReverseIfOwned(store, contract, denom)
	if auto, found := k.getAutoContractByDenom(ctx, denom); found {
		bz := store.Get(types.ContractToDenomKey(auto.Bytes()))
		if len(bz) == 0 {
			store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
		} else if existingDenom := string(bz); existingDenom != denom {
			if k.contractOwnedByDenom(ctx, existingDenom, auto) {
				// auto address is already owned by another denom; drop local auto mapping
				store.Delete(types.DenomToAutoContractKey(denom))
			} else {
				// stale reverse entry
				store.Set(types.ContractToDenomKey(auto.Bytes()), []byte(denom))
			}
		}
	}
	return true

func (k Keeper) RegisterOrUpdateTokenMapping(ctx sdk.Context, msg *types.MsgUpdateTokenMapping) error {
	if types.IsSourceCoin(msg.Denom) {
		_, err := types.GetContractAddressFromDenom(msg.Denom)
		if err != nil {
			return err
		}

		if !common.IsHexAddress(msg.Contract) {
			return errors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid contract address (%s)", msg.Contract)
		}
		contract := common.HexToAddress(msg.Contract)
		if err := k.ensureContractCode(ctx, contract); err != nil {
			return err
		}
		if err := validateContractAddressForSourceDenom(msg.Denom, contract, true); err != nil {
			return err
		}

		if err := k.SetExternalContractForDenom(ctx, msg.Denom, contract); err != nil {
			return err
		}

		// check that the coin is registered, otherwise register it
		metadata, exist := k.bankKeeper.GetDenomMetaData(ctx, msg.Denom)
		if !exist {
			// create new metadata
			metadata = banktypes.Metadata{
				Base: msg.Denom,
				Name: msg.Denom,
			}
		}
		// update existing metadata
		metadata.Symbol = msg.Symbol
		metadata.Display = strings.ToLower(msg.Symbol)
		if msg.Decimal != 0 {
			metadata.DenomUnits = []*banktypes.DenomUnit{
				{
					Denom:    metadata.Base,
					Exponent: 0,
				},
				{
					Denom:    metadata.Display,
					Exponent: msg.Decimal,
				},
			}
		} else {
			metadata.DenomUnits = []*banktypes.DenomUnit{
				{
					Denom:    metadata.Base,
					Exponent: 0,
				},
			}
		}
		k.bankKeeper.SetDenomMetaData(ctx, metadata)
	} else {
		if len(msg.Contract) == 0 {
			// delete existing mapping
			k.DeleteExternalContractForDenom(ctx, msg.Denom)
		} else {
			if !common.IsHexAddress(msg.Contract) {
				return errors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid contract address (%s)", msg.Contract)
			}
			// update the mapping
			contract := common.HexToAddress(msg.Contract)
			if err := k.ensureContractCode(ctx, contract); err != nil {
				return err
			}
			if err := k.SetExternalContractForDenom(ctx, msg.Denom, contract); err != nil {
				return err
			}
		}
	}

func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
	}
	defer func() {
		for _, a := range coins {
			if a.Amount.IsInt64() {
				telemetry.SetGaugeWithLabels(
					[]string{"tx", "msg", "ConvertVouchersToEvmCoins"},
					float32(a.Amount.Int64()),
					[]metrics.Label{telemetry.NewLabel("denom", a.Denom)},
				)
			}
		}
	}()
	return nil
