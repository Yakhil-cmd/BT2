### Title
Unrestricted `update_rewards` Allows Any Caller to Permanently Deny Block Rewards to All Stakers - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the Staking contract has no access control beyond basic prerequisites (contract not paused, caller not zero). A global state variable `last_reward_block` is written unconditionally before the `disable_rewards` flag is checked. Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` to consume the block's reward slot without distributing any rewards, permanently denying block rewards for that block to every staker in the protocol.

### Finding Description
In V3 consensus rewards mode (active after `consensus_rewards_first_epoch` is set), `update_rewards` is the sole mechanism for distributing per-block rewards. The function enforces a one-call-per-block invariant via the global `last_reward_block` storage variable.

The critical sequence in `update_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // line 1485 — written unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // line 1487-1489 — exits with no rewards
}
``` [1](#0-0) 

`last_reward_block` is updated **before** the `disable_rewards` guard. The only access control is `general_prerequisites()`:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [2](#0-1) 

There is no check that the caller is the staker, the staker's operational address, or any privileged role. The `disable_rewards: bool` parameter is fully attacker-controlled.

After the attacker's call sets `last_reward_block = N`, any subsequent call in block N fails:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

Block N's rewards are permanently lost — `update_unclaimed_rewards_from_staking_contract` is never called, so the `RewardSupplier`'s `unclaimed_rewards` is never incremented for that block. [4](#0-3) 

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

For every block where the attack is executed, the per-block STRK (and BTC) rewards are permanently destroyed: they are never minted, never credited to `unclaimed_rewards`, and never transferred to any staker or pool. An attacker who calls `update_rewards(any_valid_staker, true)` in every block can reduce all staker and delegator yield to zero indefinitely, at a cost of only gas per block.

A secondary variant — calling `update_rewards(attacker_staker, false)` — redirects the block reward to the attacker's own staker account, constituting direct theft of yield from the legitimate block-reward recipient.

### Likelihood Explanation
Any unprivileged address can call `update_rewards`. The only precondition is that a valid, active staker address with non-zero balance is supplied. The attacker does not need any stake, delegation, or privileged role. The attack is repeatable every block. On Starknet, where transaction ordering within a block is controlled by the sequencer, a well-resourced attacker (or the sequencer itself) can reliably front-run every legitimate `update_rewards` call.

### Recommendation
Add caller authorization to `update_rewards`. Only the staker themselves, their registered operational address, or a designated protocol role should be permitted to call this function. The `disable_rewards` flag should similarly be restricted to an authorized caller (e.g., the consensus layer or a governance-controlled address). Example guard:

```cairo
let caller = get_caller_address();
let staker_info = self.internal_staker_info(:staker_address);
assert!(
    caller == staker_address || caller == staker_info.operational_address,
    "{}",
    Error::UNAUTHORIZED_CALLER,
);
```

### Proof of Concept
1. V3 consensus rewards are active (`consensus_rewards_first_epoch` has been set by governance).
2. In block N, attacker calls `update_rewards(staker_A, true)` where `staker_A` is any active staker with non-zero balance.
3. `general_prerequisites()` passes (contract not paused, attacker address non-zero). [2](#0-1) 
4. `staker_A` is confirmed active with non-zero balance. [5](#0-4) 
5. `last_reward_block` is written to N. [6](#0-5) 
6. `disable_rewards = true` causes early return — no rewards distributed, `update_unclaimed_rewards_from_staking_contract` never called. [7](#0-6) 
7. Any legitimate staker attempting `update_rewards` in block N receives `REWARDS_ALREADY_UPDATED`. Block N's rewards are permanently lost.
8. Attacker repeats in block N+1, N+2, … at gas cost only, zeroing all consensus rewards indefinitely.
