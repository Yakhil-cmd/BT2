Audit Report

## Title
Missing Guard Against Empty Permission List Allows Permanent Neuron Fund Lock — (`rs/sns/governance/src/governance.rs`)

## Summary

The `remove_neuron_permissions` function in the SNS Governance canister performs no post-removal invariant check to ensure at least one principal retains `ManagePrincipals`. A principal already holding `ManagePrincipals` on a multi-principal neuron can strip every permission from every principal — including themselves — across sequential ingress calls, leaving `neuron.permissions.len() == 0`. Once that state is reached, no on-chain path exists to call `Disburse`, `AddNeuronPermissions`, or any other neuron operation, permanently locking all staked governance tokens in the neuron's ledger subaccount.

## Finding Description

`remove_neuron_permissions` in `rs/sns/governance/src/governance.rs` enforces only one precondition before mutating state: the caller must hold `ManagePrincipals` (or `ManageVotingPermission` for voting-only changes). [1](#0-0) 

The authorization helper `check_principal_authorized_to_change_permissions` in `rs/sns/governance/src/neuron.rs` checks only that the caller holds a sufficient permission type. It places no restriction on *which* principal's permissions are being removed and does not prevent the caller from removing their own `ManagePrincipals`. [2](#0-1) 

The underlying mutation `remove_permissions_for_principal` explicitly allows the result to be an empty permission list, removing the `NeuronPermission` entry entirely when no types remain. [3](#0-2) 

The proto definition and the Rust doc-comment both acknowledge this is dangerous but provide no guard. [4](#0-3) [5](#0-4) 

**Exploit path for a multi-principal neuron:**

1. Neuron N has principals A and B, both holding `ManagePrincipals`.
2. A calls `manage_neuron` → `RemoveNeuronPermissions` targeting B with all of B's permissions. Succeeds because A holds `ManagePrincipals`.
3. A calls `manage_neuron` → `RemoveNeuronPermissions` targeting themselves, removing all remaining permissions.
4. `neuron.permissions.len() == 0`. No principal holds any permission.
5. Every subsequent operation — `Disburse`, `DisburseMaturity`, `ConfigureDissolveState`, `Split`, `AddNeuronPermissions` — requires a permission check that will always fail.

`Disburse` enforces `NeuronPermissionType::Disburse` on the caller; with an empty list this check always returns `NotAuthorized`. [6](#0-5) 

`ManagePrincipals` is granted by default to every neuron claimer, making the precondition trivially satisfied for any neuron owner. [7](#0-6) 

## Impact Explanation

Staked governance tokens held in the neuron's ledger subaccount become permanently inaccessible. No principal can disburse, split, or otherwise recover the tokens; the neuron cannot be dissolved or reconfigured. This constitutes permanent, irreversible loss of user funds held in an in-scope SNS canister. This matches the **High** impact tier: "Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds where exploitation requires meaningful per-target work or other constraints." The constraint is that the attacker must already hold `ManagePrincipals` on the target neuron, which limits the attack to shared/multi-principal neurons but does not require any external compromise.

## Likelihood Explanation

`ManagePrincipals` is a default permission granted to every neuron claimer. Any SNS deployment that allows multiple principals to share a neuron (team-controlled neurons, DAO treasury neurons, or any neuron where `AddNeuronPermissions` has been used) is exposed. The attack requires only two sequential standard ingress calls to the SNS Governance canister — no special keys, no governance majority, no subnet compromise. A single compromised or malicious co-manager is sufficient.

## Recommendation

Add a post-removal invariant check inside `remove_neuron_permissions` (or inside `remove_permissions_for_principal`) that rejects the operation if it would leave the neuron with zero principals holding `ManagePrincipals`. Concretely: after computing the post-removal permission state, verify that at least one principal would still hold `ManagePrincipals`; if not, return an `InvalidCommand` or `NotAuthorized` error. This mirrors the standard access-control invariant that an ACL must never be left in an irrecoverable state.

## Proof of Concept

The existing integration test `test_neuron_remove_all_permissions_of_self` already demonstrates the full attack path end-to-end and asserts `neuron.permissions.len() == 0` after the call. [8](#0-7) 

Extending this test to then attempt a `Disburse` call on the same neuron would confirm the permanent fund lock: the call would return `NotAuthorized` for every caller, with no on-chain remediation path available.

    pub(crate) fn check_principal_authorized_to_change_permissions(
        &self,
        caller: &PrincipalId,
        permissions_to_change: NeuronPermissionList,
    ) -> Result<(), GovernanceError> {
        // If the permissions to change are exclusively voting related,
        // ManagePrincipals or ManageVotingPermission is sufficient.
        // Otherwise, only ManagePrincipals is sufficient.
        let sufficient_permissions = if permissions_to_change.is_exclusively_voting_related() {
            vec![
                NeuronPermissionType::ManagePrincipals,
                NeuronPermissionType::ManageVotingPermission,
            ]
        } else {
            vec![NeuronPermissionType::ManagePrincipals]
        };

        // The caller is authorized if they have any of the sufficient permissions
        let caller_authorized = sufficient_permissions
            .iter()
            .any(|sufficient_permission| self.is_authorized(caller, *sufficient_permission));

        if caller_authorized {
            Ok(())
        } else {
            let caller_permissions = self.permissions_for_principal(caller);
            Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                format!(
                    "Caller '{caller:?}' is not authorized to modify permissions {permissions_to_change} for neuron '{}' as it does not have any of {sufficient_permissions:?}. (Caller's permissions are {caller_permissions})",
                    self.id.as_ref().expect("Neuron must have a NeuronId"),
                ),
            ))
        }
    }

#[test]
fn test_neuron_remove_all_permissions_of_self() {
    local_test_on_sns_subnet(|runtime| async move {
        let user = Sender::from_keypair(&TEST_USER1_KEYPAIR);
        let account_identifier = Account {
            owner: user.get_principal_id().0,
            subaccount: None,
        };
        let alloc = Tokens::from_tokens(1000).unwrap();

        let system_params = NervousSystemParameters {
            neuron_claimer_permissions: Some(NeuronPermissionList {
                permissions: NeuronPermissionType::all(),
            }),
            ..NervousSystemParameters::with_default_values()
        };

        let sns_init_payload = SnsTestsInitPayloadBuilder::new()
            .with_ledger_account(account_identifier, alloc)
            .with_nervous_system_parameters(system_params)
            .build();

        let sns_canisters = SnsCanisters::set_up(&runtime, sns_init_payload).await;

        let neuron_id = sns_canisters.stake_and_claim_neuron(&user, None).await;
        let neuron = sns_canisters.get_neuron(&neuron_id).await;
        let subaccount = neuron.subaccount().expect("Error creating the subaccount");

        // Assert that the Claimer has been granted all permissions
        assert_eq!(neuron.permissions.len(), 1);
        let mut neuron_permission =
            get_neuron_permission_from_neuron(&neuron, &user.get_principal_id());
        // .sort() emits () and needs to be called outside of the assert!
        neuron_permission.permission_type.sort_unstable();
        assert_eq!(
            neuron_permission.permission_type,
            NeuronPermissionType::all(),
        );

        sns_canisters
            .remove_neuron_permissions_or_panic(
                &user,
                &subaccount,
                &user.get_principal_id(),
                NeuronPermissionType::all(),
            )
            .await;

        let neuron = sns_canisters.get_neuron(&neuron_id).await;
        assert_eq!(neuron.permissions.len(), 0);

        Ok(())
    });
