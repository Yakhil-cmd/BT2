### Title
Zero ETH Gas Cost for Bridge Withdrawal Precompiles Causes Uncompensated NEAR Gas Consumption - (File: engine-precompiles/src/native.rs)

### Summary
The `EXIT_TO_NEAR_GAS` and `EXIT_TO_ETHEREUM_GAS` constants in `engine-precompiles/src/native.rs` are both set to `EthGas::new(0)`, meaning the `ExitToNear` and `ExitToEthereum` bridge withdrawal precompiles charge zero ETH gas to callers. These precompiles internally schedule NEAR cross-contract calls that consume tens to hundreds of teragas of NEAR gas. Because the ETH gas cost is zero, the relayer who submits the NEAR transaction and pays the NEAR gas is never compensated through ETH gas fees, constituting a systematic theft of relayer yield.

### Finding Description
In `engine-precompiles/src/native.rs`, the `costs` module defines:

```rust
// TODO(#483): Determine the correct amount of gas
pub(super) const EXIT_TO_NEAR_GAS: EthGas = EthGas::new(0);

// TODO(#483): Determine the correct amount of gas
pub(super) const EXIT_TO_ETHEREUM_GAS: EthGas = EthGas::new(0);
``` [1](#0-0) 

These constants are used as the `required_gas` return value for the `ExitToNear` and `ExitToEthereum` precompiles. Despite costing 0 ETH gas, these precompiles schedule real NEAR cross-contract calls with significant NEAR gas budgets:

```rust
pub(super) const FT_TRANSFER_GAS: NearGas = NearGas::new(10_000_000_000_000);      // 10 TGas
pub(super) const FT_TRANSFER_CALL_GAS: NearGas = NearGas::new(70_000_000_000_000); // 70 TGas
pub(super) const EXIT_TO_NEAR_CALLBACK_GAS: NearGas = NearGas::new(10_000_000_000_000); // 10 TGas
pub(super) const WITHDRAWAL_GAS: NearGas = NearGas::new(100_000_000_000_000);      // 100 TGas
``` [2](#0-1) 

In Aurora's fee model, the relayer submits the NEAR transaction and pays NEAR gas upfront. The relayer is compensated through the ETH gas fee (`gas_used * effective_gas_price`) collected from the EVM transaction sender. Because `EXIT_TO_NEAR_GAS = 0` and `EXIT_TO_ETHEREUM_GAS = 0`, the precompile contributes zero to `gas_used`, so the relayer receives zero ETH compensation for the NEAR gas it spends executing the cross-contract calls. [3](#0-2) 

The same pattern exists in `prepaid_gas.rs`, `random.rs`, and `account_ids.rs`, all carrying the same `TODO(#483)` marker: [4](#0-3) [5](#0-4) [6](#0-5) 

The bridge withdrawal precompiles are the most severe because they trigger expensive NEAR cross-contract calls.

### Impact Explanation
**High — Theft of unclaimed yield (relayer fees).**

Every call to `ExitToNear` or `ExitToEthereum` causes the relayer to spend 10–100 TGas of NEAR gas without receiving any ETH gas fee compensation. An attacker holding Aurora tokens can repeatedly call these precompiles, paying only the EVM intrinsic gas (21,000 gas) for the outer transaction while triggering up to 100 TGas of NEAR gas consumption per call. The relayer's NEAR balance is drained without compensation, constituting systematic theft of the yield the relayer is entitled to earn.

### Likelihood Explanation
**High.** The entry path requires only a standard EVM transaction calling the `ExitToNear` or `ExitToEthereum` precompile address. Any token holder on Aurora can trigger this. The zero-cost precompile is live in production code with no guard or minimum-gas enforcement. The TODO comment confirms the developers are aware the value is a placeholder, not an intentional design choice.

### Recommendation
- **Short term**: Set `EXIT_TO_NEAR_GAS` and `EXIT_TO_ETHEREUM_GAS` to values that reflect the actual NEAR gas cost of the cross-contract calls they schedule (e.g., proportional to the TGas budgets already defined in the same `costs` module). Apply the same fix to `PREPAID_GAS_COST`, `RANDOM_BYTES_GAS`, and `PREDECESSOR_ACCOUNT_GAS` / `CURRENT_ACCOUNT_GAS`.
- **Long term**: Resolve issue `#483` with a systematic methodology for mapping NEAR gas costs to EVM gas costs, and add regression tests that assert non-zero gas costs for all precompiles that schedule NEAR cross-contract calls.

### Proof of Concept
1. Alice holds 1,000 Aurora ERC-20 tokens.
2. Alice deploys a contract on Aurora that calls `ExitToNear` (precompile at the bridge withdrawal address) in a loop, each call withdrawing a dust amount.
3. Each call costs Alice only the EVM intrinsic gas (21,000 gas × gas_price in ETH), but triggers a NEAR cross-contract call consuming 10–100 TGas.
4. The relayer pays NEAR gas for each cross-contract call but receives 0 ETH gas fee for the precompile portion.
5. After N iterations, the relayer has lost N × (NEAR gas cost of FT_TRANSFER_GAS) in NEAR without compensation, while Alice has paid only minimal ETH gas. [7](#0-6)
