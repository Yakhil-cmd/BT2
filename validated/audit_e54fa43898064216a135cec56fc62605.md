### Title
`allowPushers` Signature Replay Nullifies `revokePusher` Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1.allowPushers` uses only a deadline to bound the pusher's consent signature. No nonce or consumed-signature flag is stored. After a pusher calls `revokePusher()`, the creator can immediately replay the original consent signature (while the deadline is still valid) to re-establish the delegation, permanently nullifying the pusher's revocation right until the deadline expires.

### Finding Description

`allowPushers` verifies the pusher's EIP-191 consent signature and maps `namespaceRemapping[pusher] = msg.sender` (the creator): [1](#0-0) 

The only replay guard is `_ensureDeadline(deadline)`: [2](#0-1) 

which only rejects calls **after** the deadline. Within the deadline window, the exact same `(deadline, pusher, signature)` tuple is accepted an unlimited number of times.

`revokePusher` clears the mapping: [3](#0-2) 

But because no consumed-signature record exists, the creator can immediately call `allowPushers` again with the identical signature to restore `namespaceRemapping[pusher] = creator`, undoing the revocation in the same block.

The code comment itself acknowledges the intent of the deadline: [4](#0-3) 

The comment claims the deadline prevents re-establishment "after the pusher revoked it," but this is incorrect — the deadline only prevents replay after it **expires**, not after revocation occurs within the window.

### Impact Explanation

A pusher whose key is compromised calls `revokePusher()` to stop the attacker from writing bad oracle data into the creator's namespace. The creator (or an automated system holding the original signature) calls `allowPushers` again with the same signature, re-establishing the delegation. The compromised pusher key continues to push manipulated prices into the creator's feed namespace. Any `MetricOmmPool` backed by a `PriceProvider` reading that feed receives bad bid/ask quotes, causing bad-price execution and direct LP fund loss.

### Likelihood Explanation

The pusher's consent signature is typically broadcast on-chain in the original `allowPushers` calldata, making it publicly observable. Any creator whose system automatically re-delegates (e.g., a keeper bot) will replay it without malicious intent. A deadline of days or weeks — common in practice — leaves a large window. The pusher has no on-chain recourse other than waiting for expiry.

### Recommendation

Record each consumed signature hash and reject reuse:

```solidity
mapping(bytes32 => bool) private _usedConsentHashes;

function allowPushers(...) external {
    _ensureDeadline(deadline);
    for (uint256 i; i < l; i++) {
        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
        );
        require(!_usedConsentHashes[hash], "signature already used");
        require(pusher == ECDSA.recover(hash, signatures[i]));
        _usedConsentHashes[hash] = true;
        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

Alternatively, introduce a per-pusher nonce incremented on every successful `allowPushers` or `revokePusher` call and include it in the signed payload.

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline 7 days out
uint256 deadline = block.timestamp + 7 days;
bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
);
(uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
bytes memory sig = abi.encodePacked(r, s, v);

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0));

// 4. Creator replays the SAME signature — succeeds, revocation nullified
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig)); // no revert
assertEq(oracle.namespaceRemapping(pusher), creator);   // delegation restored
```

The compromised pusher key can now continue writing to the creator's feed namespace, delivering manipulated prices to any pool consuming that feed.

```text
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```

```text
    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }
```
