### Title
Gateway `skip_stateful_validations` Admits Unsigned Invoke Transactions for Undeployed Accounts — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator skips the `__validate__` entry-point call (the account's cryptographic signature check) for any Invoke transaction with `nonce == 1` sent to an address that already has *any* transaction in the mempool. An unprivileged attacker who observes a legitimate `deploy_account` transaction in the mempool can immediately submit a forged Invoke with `nonce = 1` for the same address and an arbitrary fake signature. The gateway admits the forged transaction into the mempool without ever verifying the signature, occupying the nonce-1 slot and blocking the legitimate user's first post-deploy invoke.

### Finding Description

**Invariant broken**: Every transaction admitted to the mempool must either (a) have passed the account's `__validate__` entry point, or (b) be a `deploy_account` whose constructor and `__validate_deploy__` were fully executed.

**Root cause — `skip_stateful_validations`** [1](#0-0) 

The function returns `true` (skip validation) when all three conditions hold:

1. The transaction is an `Invoke` with `tx.nonce() == Nonce(Felt::ONE)`.
2. The on-chain account nonce is `Nonce(Felt::ZERO)` (account not yet deployed).
3. `account_tx_in_pool_or_recent_block(sender_address)` returns `true`. [2](#0-1) 

Condition 3 is satisfied as soon as *any* transaction for the target address exists in the mempool pool or committed-block state: [3](#0-2) 

**How the skip propagates to admission**

When `skip_stateful_validations` returns `true`, `run_validate_entry_point` sets `validate: false`: [4](#0-3) 

Inside `StatefulValidator::perform_validations`, the `__validate__` call is gated on this flag: [5](#0-4) 

So the account's signature is never verified at the gateway level.

**Batcher uses `new_for_sequencing` — always validates**

When the batcher later executes the transaction it uses `AccountTransaction::new_for_sequencing`, which hard-codes `validate: true` and `strict_nonce_check: true`: [6](#0-5) [7](#0-6) 

The batcher will therefore call `__validate__` and the forged transaction will revert — but only *after* it has already been admitted to the mempool and has occupied the nonce-1 slot.

### Impact Explanation

An attacker can inject a forged Invoke transaction (arbitrary calldata, fake signature) into the mempool for any account whose `deploy_account` is currently pending. The forged transaction:

- Occupies the nonce-1 slot for the victim's address.
- Causes the legitimate user's first post-deploy Invoke to be rejected as a duplicate nonce (or forced into a fee-escalation race).
- Wastes batcher execution resources when the forged transaction is eventually executed and reverts.

This matches the **High** impact scope: *"Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."*

### Likelihood Explanation

The mempool is public; any observer can see a pending `deploy_account`. The attack requires only a single HTTP POST to the gateway with a well-formed Invoke transaction (correct sender address, nonce = 1, any signature). No privileged access, no special tooling, and no hash-preimage computation is needed. The race window is the time between the `deploy_account` entering the mempool and the batcher committing it — typically multiple seconds to minutes.

### Recommendation

Replace the mempool-presence heuristic with a check that is specific to `deploy_account` transactions. The skip should only be granted when a `deploy_account` transaction (not just *any* transaction) for the sender address is confirmed to be in the mempool. Additionally, consider bounding the skip to a short time-to-live so that stale entries cannot be exploited indefinitely.

Concretely, `skip_stateful_validations` should query the mempool for a pending `deploy_account` at nonce 0 for the sender address, rather than the generic `account_tx_in_pool_or_recent_block` check.

```
  "type": "INVOKE",
  "sender_address": "0xVICTIM",
  "nonce": "0x1",
  "signature": ["0xDEAD", "0xBEEF"],   # ← completely fake
  "calldata": ["<arbitrary>"],
  ...
}
# Gateway path:
#   validate_nonce: account_nonce=0, tx_nonce=1 → OK (within [0, 200])
#   validate_by_mempool: no duplicate hash → OK
#   skip_stateful_validations:
#     tx.nonce()==1 ✓, account_nonce==0 ✓,
#     account_tx_in_pool_or_recent_block(0xVICTIM)==true ✓
#     → returns true (SKIP __validate__)
#   run_validate_entry_point: validate=false → __validate__ NOT called
# → Forged Invoke admitted to mempool at (0xVICTIM, nonce=1)

# Step 3 – Legitimate user tries to submit their real Invoke
POST /gateway/add_transaction
{
  "type": "INVOKE",
  "sender_address": "0xVICTIM",
  "nonce": "0x1",
  "signature": ["<real_r>", "<real_s>"],
  ...
}
# → Rejected: nonce-1 slot already occupied by attacker's forged transaction

# Step 4 – Batcher executes deploy_account (succeeds), then forged Invoke
#   __validate__ is called with fake signature → REVERT
#   Legitimate user's transaction was never executed
```

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
```
