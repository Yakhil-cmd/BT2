### Title
Unrestricted `update_rewards` with `disable_rewards=true` Allows Any Caller to Permanently Freeze Consensus Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `IStakingRewardsManager` is publicly callable with no access control. A global `last_reward_block` guard allows only one call per block. Any unprivileged caller can invoke `update_rewards(valid_staker, disable_rewards: true)` on every block, consuming the per-block slot without distributing rewards, permanently preventing all stakers from receiving consensus block rewards.

---

### Finding Description

`update_rewards` is exposed as a public interface with no role restriction: [1](#0-0) 

The implementation enforces a single global `last_reward_block` guard: [2](#0-1) 

The only caller check is `general_prerequisites()`, which only verifies the contract is unpaused and the caller is non-zero: [3](#0-2) 

When `disable_rewards: true` is passed, the function writes `current_block_number` to `last_reward_block` and returns immediately without distributing any rewards: [4](#0-3) 

Because `last_reward_block` is a **global** storage slot (not per-staker), once it is written for a given block, every subsequent call in that block reverts with `REWARDS_ALREADY_UPDATED`. An attacker who calls `update_rewards(any_valid_staker, true)` as the first transaction of every block permanently occupies the slot and starves all stakers of consensus rewards.

The attacker only needs any valid, active staker address with non-zero balance — all of which are publicly observable on-chain from `NewStaker` events.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Consensus block rewards (`_update_rewards` path) are the primary reward mechanism in V3. If the per-block slot is consumed with `disable_rewards: true` on every block, no staker ever accumulates consensus rewards. The `unclaimed_rewards_own` field in every `InternalStakerInfoV1` struct remains at zero indefinitely. [5](#0-4) 

---

### Likelihood Explanation

**Medium.** The attacker must submit one transaction per block. On Starknet, block times are short (~3 s) and transaction fees are low, making sustained griefing economically feasible for a motivated actor (e.g., a competing validator, a short-seller, or a protocol adversary). No privileged access, leaked key, or external dependency is required — only a funded wallet.

---

### Recommendation

1. **Remove `disable_rewards` from the public interface.** If the parameter is needed internally (e.g., during migration), make the function `internal` or restrict it to a specific role (e.g., `only_app_governor`).
2. **Alternatively**, make `last_reward_block` per-staker so one call cannot block all others.
3. **At minimum**, add an access-control guard (e.g., `only_operator` or `only_staker_or_reward_address`) so an unprivileged third party cannot call the function with arbitrary parameters.

---

### Proof of Concept

```
// Attacker script (pseudocode, one tx per block):
loop every block:
    staking_contract.update_rewards(
        staker_address = <any known active staker>,
        disable_rewards = true
    )
// Effect:
//   last_reward_block := current_block
//   rewards NOT distributed
//   All subsequent legitimate calls in this block revert with REWARDS_ALREADY_UPDATED
//   All stakers accumulate zero consensus rewards indefinitely
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
