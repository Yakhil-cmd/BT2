### Title
Delegation Rewards Permanently Frozen When Pool Balance Is Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo)

### Summary

When a pool's delegated balance is non-zero but below `min_delegation_for_rewards` (1 STRK = 10^18 for STRK pools), the staking contract still calculates and forwards non-zero pool rewards to the pool contract. However, the pool contract's `compute_rewards_per_unit` silently returns zero, causing the `cumulative_rewards_trace` to not advance while the reward tokens are already transferred and permanently locked in the pool contract with no recovery path.

### Finding Description

The pool contract's `compute_rewards_per_unit` function contains a guard that returns zero when `total_stake < min_delegation_for_rewards`: [1](#0-0) 

The code comment at line 964 explicitly acknowledges the consequence: [2](#0-1) 

This function is called from `update_rewards_from_staking_contract`, which always inserts a new entry into `cumulative_rewards_trace` regardless of whether `compute_rewards_per_unit` returned zero: [3](#0-2) 

The staking contract's `calculate_staker_pools_rewards` only skips a pool when `pool_rewards == 0`.
