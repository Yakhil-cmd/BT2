### Title
Malformed message body from any protocol participant causes permanent DKG/protocol abort via unhandled `DeserializationError` in `recv_from_others` — (`src/protocol/helpers.rs`, `src/protocol/internal.rs`)

### Summary

A participant in the protocol (operating within the documented authenticated-channel trust model) can send a message whose header bytes are valid but whose body is not a valid msgpack encoding of the expected type. When `recv_from_others` pops this message and `Comms::recv` attempts deserialization, it returns `Err(ProtocolError::DeserializationError(...))`, which is immediately propagated via `?` and permanently aborts the honest participant's protocol execution.

### Finding Description

**Entry point — `Comms::push_message`:**

`push_message` performs only two checks before buffering an incoming message: minimum length and valid header parse. It does **not** inspect or validate the message body. [1](#0-0) 

A message with a correctly formed `MessageHeader` (matching `root_shared` channel tag and the target waitpoint) but arbitrary/garbage bytes in the body passes both checks and is buffered.

**Deserialization failure — `Comms::recv`:**

When the protocol later pops that message, `rmp_serde::decode::from_slice` is called on the body. On failure it returns `Err(ProtocolError::DeserializationError(...))` via the `?` operator. [2](#0-1) 

**Propagation — `recv_from_others`:**

`recv_from_others` calls `chan.recv(waitpoint).await?`. The `?` propagates the deserialization error **before** the `seen.put(from)` guard is ever evaluated, so there is no opportunity to skip the bad message and continue waiting for valid ones. [3](#0-2) 

**Concrete DKG trigger — `do_keyshare` at `wait_round_3`:**

In the DKG protocol, `recv_from_others` is called at `wait_round_3` to collect `SigningShare` values from every other participant. A single malformed message from any one participant aborts the entire `do_keyshare` future. [4](#0-3) 

The same pattern is present in every other protocol that uses `recv_from_others` (presign, sign, CKD, FROST sign, triple generation).

### Impact Explanation

A single malicious participant (one who is legitimately in the participant list and whose identity is authenticated by the transport layer per the documented assumption) can permanently terminate the DKG execution of every honest participant by sending one message with a valid header and an invalid body at the correct waitpoint. Key generation fails for all honest parties; no key share is produced.

This maps to: **High — Permanent denial of key generation for honest parties under valid protocol inputs and documented trust assumptions.**

### Likelihood Explanation

The attack requires only that the adversary be a legitimate participant (within the `t`-malicious threat model) and know the correct `ChannelTag` and waitpoint value. Both are deterministic and computable from public parameters. No cryptographic break is needed. The attack is trivially reproducible.

### Recommendation

In `recv_from_others`, catch `ProtocolError::DeserializationError` from `chan.recv(...)` and skip the offending message (optionally logging the sender as malicious) rather than propagating the error. Alternatively, move deserialization errors into a per-sender error path so that one bad sender does not abort the loop for all others.

### Proof of Concept

1. Compute the `MessageHeader` bytes for `ChannelTag::root_shared()` with `waitpoint = wait_round_3` (the 5th waitpoint allocated in `do_keyshare`, deterministic from the protocol flow).
2. Append random non-msgpack bytes as the body.
3. Call `protocol.message(malicious_participant, crafted_bytes)` on the honest participant's protocol instance.
4. Call `protocol.poke()` repeatedly.
5. Observe that `poke()` returns `Err(ProtocolError::DeserializationError(...))`, permanently aborting the DKG — no key share is ever produced. [5](#0-4) [6](#0-5)

        &self,
        header: MessageHeader,
    ) -> Result<(Participant, T), ProtocolError> {
        let (from, data) = self.incoming.pop(header).await;
        let message_data = data.get(MessageHeader::LEN..).ok_or_else(|| {
            ProtocolError::DeserializationError("Failed to deserialize message data".to_string())
        })?;
        let decoded: Result<T, Box<dyn error::Error + Send + Sync>> =
            rmp_serde::decode::from_slice(message_data).map_err(std::convert::Into::into);
        Ok((from, decoded?))
    }
