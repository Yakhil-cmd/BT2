### Title
Unrestricted `update_rewards` Caller with Attacker-Controlled `disable_rewards` Enables Griefing of All Stakers' Consensus Block Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `src/staking/staking.cairo` has no caller restriction and accepts a fully caller-controlled `disable_rewards` parameter. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards: true)` to unconditionally advance the global `last_reward_block` to the current block without distributing any rewards. Because the function enforces `current_block_number > last_reward_block` as a one-shot-per-block gate, this permanently blocks any legitimate reward update for that block, griefing all stakers out of their consensus block rewards.

---

### Finding Description

In `src/staking/staking.cairo`, `StakingRewardsManagerImpl::update_rewards` (lines 1449–1500+) is part of the `IStakingRewardsManager` interface and is callable by **any address** with no role or identity check. The execution path is:

```
update_rewards(staker_address, disable_rewards)
  → general_prerequisites()          // only checks is_paused
  → assert current_block > last_reward_block
  → validate staker exists and is active
  → validate staker has non-zero STRK balance
  → last_reward_block.write(current_block_number)   ← UNCONDITIONAL
  → if disable_rewards || is_pre_consensus() { return; }
  → distribute rewards
```

The critical flaw: `last_reward_block` is written **before** the `disable_rewards` branch is evaluated. `last_reward_block` is a **single global storage variable** (not per-staker), so updating it blocks reward updates for every staker in that block.

An attacker's exploit path:
1. Observe any active staker address `S` on-chain (public information).
2. At block `N`, call `update_rewards(S, disable_rewards: true)` from any EOA.
3. `last_reward_block` is set to `N`; no rewards are distributed.
4. Any subsequent call to `update_rewards` at block `N` reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeat at block `N+1`, `N+2`, … to permanently suppress all consensus reward accrual.

The analog to M-02 is direct: just as the UStb `_beforeTokenTransfer` check omits `hasRole(WHITELISTED_ROLE, from)` when `transferState == WHITELIST_ENABLED`, `update_rewards` omits any check on **who** is allowed to supply the `disable_rewards=true` flag, allowing an unprivileged caller to bypass the intended reward-distribution guard.

---

### Impact Explanation

`last_reward_block` is global. A single call per block with `disable_rewards: true` silently poisons the reward slot for **all** stakers simultaneously. Sustained over multiple blocks, this constitutes permanent freezing of all consensus-phase unclaimed yield for all stakers — matching the **High: Permanent freezing of unclaimed yield** impact category. Even sporadic attacks cause measurable, unrecoverable yield loss per targeted block.

---

### Likelihood Explanation

- Requires zero privileged access; any address can call `update_rewards`.
- Active staker addresses are publicly readable from on-chain events and storage.
- The attacker must pay one transaction per block; on Starknet this is low-cost.
- No complex setup, no flash loans, no oracle manipulation — a single call suffices.
- The attack is fully deterministic and reproducible.

---

### Recommendation

Restrict who may supply `disable_rewards: true`. Options:

1. **Add a caller check**: Restrict `update_rewards` to an authorized role (e.g., `OPERATOR` or a dedicated consensus contract address), mirroring how `update_rewards_from_attestation_contract` is restricted to `assert_caller_is_attestation_contract`.
2. **Decouple the write**: Only write `last_reward_block` when rewards are actually distributed (move the write after the `disable_rewards` branch).
3. **Remove the parameter**: Derive `disable_rewards` internally from protocol state rather than accepting it from an untrusted caller.

---

### Proof of Concept

```
// Any unprivileged address executes this at block N:
IStakingRewardsManagerDispatcher { contract_address: staking_contract }
    .update_rewards(staker_address: any_active_staker, disable_rewards: true);

// last_reward_block is now N; no rewards distributed.

// Legitimate consensus call at block N now reverts:
IStakingRewardsManagerDispatcher { contract_address: staking_contract }
    .update_rewards(staker_address: any_active_staker, disable_rewards: false);
// → panics: "Rewards already updated"

// Repeat every block → all stakers permanently frozen out of consensus rewards.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
