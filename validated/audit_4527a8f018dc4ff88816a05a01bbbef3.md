### Title
Stale Pending Block Synthesizes `DeprecatedPendingBlock` With Zero `l1_data_gas_price`, Causing `estimate_fee`/`simulate_transactions` to Return `NonzeroGasPrice::MIN` (= 1) as `l1_data_gas_price` — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

When the in-memory pending block is stale (its `parent_block_hash` does not match the latest committed block hash), `read_pending_data` synthesizes a fallback `DeprecatedPendingBlock`. That struct has no `l1_data_gas_price` field, so `l1_data_gas_price()` returns `GasPricePerToken::default()` (zero). `create_block_context` then substitutes `NonzeroGasPrice::MIN` (= 1) for the zero value. Any call to `estimate_fee`, `simulate_transactions`, or `estimate_message_fee` with `block_id = Tag(Pending)` during this window returns an authoritative-looking `l1_data_gas_price` of 1 instead of the real committed-block value.

### Finding Description

**Step 1 — `read_pending_data` synthesizes a `DeprecatedPendingBlock` on hash mismatch.** [1](#0-0) 

When `pending_data.block.parent_block_hash() != latest_header.block_hash`, the fallback block is constructed with `..Default::default()`. The synthesized block copies only `eth_l1_gas_price` and `strk_l1_gas_price` from the latest header's `l1_gas_price`; it does **not** copy `l1_data_gas_price`.

**Step 2 — `DeprecatedPendingBlock` has no `l1_data_gas_price` field.** [2](#0-1) 

**Step 3 — `l1_data_gas_price()` on a `Deprecated` variant always returns zero.** [3](#0-2) 

The comment "In older versions, data gas price was 0" is correct for historical blocks, but the synthesized fallback block is supposed to represent the *current* pending context, not a historical one.

**Step 4 — `client_pending_data_to_execution_pending_data` forwards the zero price.** [4](#0-3) 

**Step 5 — `create_block_context` substitutes `NonzeroGasPrice::MIN` = 1 for the zero.** [5](#0-4) 

**Step 6 — `tx_execution_output_to_fee_estimation` reads `l1_data_gas_price` directly from the `BlockContext`.** [6](#0-5) 

The returned `FeeEstimation.l1_data_gas_price` is 1 (wei or fri) instead of the actual committed-block value.

**Step 7 — `estimate_fee` with `block_id = Tag(Pending)` is the public entry point.** [7](#0-6) 

The same path is taken by `simulate_transactions` and `estimate_message_fee`. [8](#0-7) [9](#0-8) 

### Impact Explanation

Any caller of `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_estimateMessageFee` with `block_id = Tag(Pending)` during the stale-pending window receives a fee estimation where `l1_data_gas_price` is 1 (the `NonzeroGasPrice::MIN` sentinel) instead of the real value (which on mainnet is on the order of tens of gwei). For transactions with nonzero L1 data gas usage (blob-DA transactions), the `overall_fee` and `data_gas_consumed` cost component are computed against a price that is orders of magnitude too low. Wallets and dApps that rely on this endpoint to set `max_fee` or `resource_bounds.l1_data_gas` will systematically under-price transactions, causing them to be rejected by the sequencer or to fail fee checks.

### Likelihood Explanation

The stale-pending condition is a **normal, recurring operational state**: it occurs every time a new block is committed to storage but the pending-data writer has not yet updated the shared `RwLock<PendingData>`. This is a race window that exists on every block boundary and requires no attacker action. Any user who happens to call `estimate_fee(block_id=Pending)` during that window — which can last for multiple seconds — receives the corrupted value. The condition is not exceptional; it is the designed fallback path.

### Recommendation

In the stale-path branch of `read_pending_data`, also copy `l1_data_gas_price` (both `price_in_wei` and `price_in_fri`) from `latest_header.block_header_without_hash.l1_data_gas_price`:

```rust
// crates/apollo_rpc/src/v0_8/api/api_impl.rs  (read_pending_data, stale branch)
Ok(PendingData {
    block: PendingBlockOrDeprecated::Deprecated(DeprecatedPendingBlock {
        parent_block_hash: latest_header.block_hash,
        eth_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_wei,
        strk_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_fri,
        // ADD: carry l1_data_gas_price from the latest committed header
        // (requires adding eth_l1_data_gas_price / strk_l1_data_gas_price fields
        //  to DeprecatedPendingBlock, or switching to PendingBlockOrDeprecated::Current)
        timestamp: latest_header.block_header_without_hash.timestamp,
        sequencer_address: latest_header.block_header_without_hash.sequencer,
        ...
    }),
    ...
})
```

The cleanest fix is to synthesize a `PendingBlockOrDeprecated::Current` (`PendingBlock`) instead of `Deprecated`, since `PendingBlock` carries an explicit `l1_data_gas_price: GasPricePerToken` field and can be populated directly from the latest header. [10](#0-9)
