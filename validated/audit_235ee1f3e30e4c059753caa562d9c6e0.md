### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Invalid Transaction Admission or Valid Transaction Rejection — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful admission check for L2 gas price compares a transaction's `max_price_per_unit` against the **previous block's execution price** (`l2_gas_price`) rather than the **EIP-1559-computed price for the next block** (`next_l2_gas_price`). The code even carries a developer TODO acknowledging the wrong field is used. Because the two prices diverge whenever the network is congested or uncongested, the gateway systematically admits transactions that will fail at blockifier pre-validation (consuming mempool and batcher resources for free) or rejects transactions that would succeed at execution.

---

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` is called for every incoming `AllResources` V3 transaction during the stateful gateway path:

```rust

    -> StatefulTransactionValidatorResult<()>
{
    if self.config.validate_resource_bounds {
        // TODO(Arni): getnext_l2_gas_price from the block header.   ← developer-acknowledged bug
        let previous_block_l2_gas_price = self
            .gateway_fixed_block_state_reader
            .get_block_info()
            .await?
            .gas_prices
            .strk_gas_prices
            .l2_gas_price;                                            // ← WRONG field
        self.validate_tx_l2_gas_price_within_threshold(
            executable_tx.resource_bounds(),
            previous_block_l2_gas_price,
        )?;
    }
    Ok(())
}

The `GatewayFixedBlockStateReader` trait only exposes `get_block_info() -> BlockInfo`, and `BlockInfo` has no `next_l2_gas_price` field, so the correct value is structurally inaccessible through the current interface. [4](#0-3) 

The correct reference is `block_header_without_hash.next_l2_gas_price`, which is the EIP-1559-adjusted price that the consensus orchestrator will actually use when building the next block:

```rust

This is the price stored in the block header and used by the consensus orchestrator as `self.l2_gas_price` when building the next block proposal:

### Impact Explanation

**Congested network (`next_l2_gas_price > l2_gas_price`):**

The EIP-1559 formula raises the price when `gas_used > gas_target`. The gateway threshold is computed against the lower stale price, so it is too permissive. Any transaction with:

```
min_gas_price_percentage% × l2_gas_price  ≤  tx.max_price_per_unit  <  next_l2_gas_price
```

passes gateway admission but will be rejected by `check_fee_bounds` inside `perform_pre_validation_stage` at blockifier execution time with `MaxGasPriceTooLow`. Because pre-validation failure means no fee is charged, an unprivileged attacker can flood the mempool and batcher with zero-cost failing transactions during any period of sustained congestion. [7](#0-6) 

**Uncongested network (`next_l2_gas_price < l2_gas_price`):**

The threshold is too strict. Transactions with:

```
min_gas_price_percentage% × next_l2_gas_price  ≤  tx.max_price_per_unit  <  min_gas_price_percentage% × l2_gas_price
```

are rejected at the gateway even though they would pass blockifier execution. This is a systematic denial of valid transactions.

---

### Likelihood Explanation

The condition is triggered automatically by normal EIP-1559 price dynamics. Any block with `gas_used ≠ gas_target` produces a `next_l2_gas_price` that differs from `l2_gas_price`. The divergence is bounded per block by `price / gas_price_max_change_denominator` (approximately 1–2% per block based on the test snapshots), but it is persistent and cumulative during sustained congestion or low-usage periods. No privileged access, special contract, or external oracle manipulation is required; any user submitting a V3 (`AllResources`) transaction with a carefully chosen `max_price_per_unit` triggers the incorrect admission decision.

---

### Recommendation

1. **Expose `next_l2_gas_price` through the reader interface.** Add a `get_next_l2_gas_price()` method to `GatewayFixedBlockStateReader` (or add the field to `BlockInfo`) and populate it from `block_header_without_hash.next_l2_gas_price` in `GatewayFixedBlockSyncStateClient`.

2. **Fix `validate_resource_bounds`.** Replace the read of `gas_prices.strk_gas_prices.l2_gas_price` with the newly exposed `next_l2_gas_price` so the threshold reflects the price that will actually govern execution.

3. **Align the blockifier gateway context.** `run_validate_entry_point` also builds a `BlockContext` from the same stale `get_block_info()` result; it should similarly use `next_l2_gas_price` as the L2 gas price so that gateway blockifier validation and batcher execution use the same price. [8](#0-7) 

---

### Proof of Concept

1. Read the latest committed block header. Observe `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (any congested block).
2. Compute the gateway threshold: `T = min_gas_price_percentage% × P`.
3. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = T` (satisfies gateway check) where `T < P'`.
4. The gateway calls `validate_tx_l2_gas_price_within_threshold` with `previous_block_l2_gas_price = P`; the transaction passes because `T ≥ T`.
5. The transaction enters the mempool and is picked up by the batcher.
6. The batcher builds the next block with `l2_gas_price = P'`. `check_fee_bounds` fires `MaxGasPriceTooLow` because `T < P'`.
7. The transaction is rejected at pre-validation; no fee is charged.
8. Repeat from step 3 to sustain a zero-cost mempool flood during any congested period. [9](#0-8) [10](#0-9)

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn check_fee_bounds(
        &self,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let minimal_gas_amount_vector = estimate_minimal_gas_vector(
            &tx_context.block_context,
            self,
            &tx_context.get_gas_vector_computation_mode(),
        );
        let TransactionContext { block_context, tx_info } = tx_context;
        let block_info = &block_context.block_info;
        let fee_type = &tx_info.fee_type();
        match tx_info {
            TransactionInfo::Current(context) => {
                let resources_amount_tuple = match &context.resource_bounds {
                    ValidResourceBounds::L1Gas(l1_gas_resource_bounds) => vec![(
                        L1Gas,
                        l1_gas_resource_bounds,
                        minimal_gas_amount_vector.to_l1_gas_for_fee(
                            tx_context.get_gas_prices(),
                            &tx_context.block_context.versioned_constants,
                        ),
                        block_info.gas_prices.l1_gas_price(fee_type),
                    )],
                    ValidResourceBounds::AllResources(AllResourceBounds {
                        l1_gas: l1_gas_resource_bounds,
                        l2_gas: l2_gas_resource_bounds,
                        l1_data_gas: l1_data_gas_resource_bounds,
                    }) => {
                        let GasPriceVector { l1_gas_price, l1_data_gas_price, l2_gas_price } =
                            block_info.gas_prices.gas_price_vector(fee_type);
                        vec![
                            (
                                L1Gas,
                                l1_gas_resource_bounds,
                                minimal_gas_amount_vector.l1_gas,
                                *l1_gas_price,
                            ),
                            (
                                L1DataGas,
                                l1_data_gas_resource_bounds,
                                minimal_gas_amount_vector.l1_data_gas,
                                *l1_data_gas_price,
                            ),
                            (
                                L2Gas,
                                l2_gas_resource_bounds,
                                minimal_gas_amount_vector.l2_gas,
                                *l2_gas_price,
                            ),
                        ]
                    }
                };
                let insufficiencies = resources_amount_tuple
                    .iter()
                    .flat_map(
                        |(resource, resource_bounds, minimal_gas_amount, actual_gas_price)| {
                            let mut insufficiencies_resource = vec![];
                            if minimal_gas_amount > &resource_bounds.max_amount {
                                insufficiencies_resource.push(
                                    ResourceBoundsError::MaxGasAmountTooLow {
                                        resource: *resource,
                                        max_gas_amount: resource_bounds.max_amount,
                                        minimal_gas_amount: *minimal_gas_amount,
                                    },
                                );
                            }
                            if resource_bounds.max_price_per_unit < actual_gas_price.get() {
                                insufficiencies_resource.push(
                                    ResourceBoundsError::MaxGasPriceTooLow {
                                        resource: *resource,
                                        max_gas_price: resource_bounds.max_price_per_unit,
                                        actual_gas_price: (*actual_gas_price).into(),
                                    },
                                );
                            }
                            insufficiencies_resource
                        },
                    )
                    .collect::<Vec<_>>();
                if !insufficiencies.is_empty() {
                    return Err(Box::new(TransactionFeeError::InsufficientResourceBounds {
                        errors: insufficiencies,
                    }))?;
                }
            }
            TransactionInfo::Deprecated(context) => {
                let max_fee = context.max_fee;
                let min_fee = get_fee_by_gas_vector(
                    block_info,
                    minimal_gas_amount_vector,
                    fee_type,
                    tx_context.effective_tip(),
                );
                if max_fee < min_fee {
                    return Err(TransactionPreValidationError::TransactionFeeError(Box::new(
                        TransactionFeeError::MaxFeeTooLow { min_fee, max_fee },
                    )));
                }
            }
        };
        Ok(())
    }
