### Title
Gateway stateful validator checks `l2_gas_price` of the previous block instead of `next_l2_gas_price` for L2 resource-bounds admission — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary
`StatefulTransactionValidator::validate_resource_bounds` reads `gas_prices.strk_gas_prices.l2_gas_price` from `get_block_info()`, which is the L2 gas price **of the already-committed block**. The next block will execute at `next_l2_gas_price` (the EIP-1559-derived price stored in the same block header). When the network is congested and the price is rising, any V3 transaction whose `max_price_per_unit` sits between `l2_gas_price` and `next_l2_gas_price` passes every gateway check and enters the mempool, but is then rejected by the blockifier during batcher execution with `MaxGasPriceTooLow`.

### Finding Description

**Root cause — wrong field read in `validate_resource_bounds`** [1](#0-0) 

The code reads `block_header.l2_gas_price.price_in_fri` (the price *used in* the previous block). The TODO comment on line 229 explicitly acknowledges the mistake: `// TODO(Arni): getnext_l2_gas_price from the block header.`

**What `get_block_info()` actually returns**

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `BlockInfo.gas_prices` from `block_header.l2_gas_price`, not from `block_header.next_l2_gas_price`: [2](#0-1) 

The `BlockHeaderWithoutHash` struct carries both fields: [3](#0-2) 

`next_l2_gas_price` is the price the *next* block will charge, computed by the EIP-1559 formula in `calculate_next_base_gas_price`: [4](#0-3) 

Under full congestion the price can rise by up to `price / gas_price_max_change_denominator` per block (≈ 12.5 % for the default denominator of 8).

**The threshold check that uses the stale price** [5](#0-4) 

With the production default `min_gas_price_percentage = 100`, the threshold equals `previous_block_l2_gas_price` exactly.

**Why the blockifier validation inside the gateway also passes**

`run_validate_entry_point` builds its `BlockContext` from the same stale `get_block_info()` result (with only the block number incremented): [6](#0-5) 

So `check_fee_bounds` inside the gateway also compares against `l2_gas_price`, not `next_l2_gas_price`. The transaction passes both gateway checks.

**Where the batcher uses the correct price**

The consensus orchestrator stores `l2_gas_price` as the price for the *next* block (i.e. `next_l2_gas_price` from the previous header): [7](#0-6) 

When the batcher executes the transaction, `check_fee_bounds` compares `max_price_per_unit` against this higher price and rejects the transaction: [8](#0-7) 

### Impact Explanation
Any unprivileged user can submit a V3 `invoke` transaction whose `l2_gas.max_price_per_unit` equals the previous block's `l2_gas_price`. The gateway admits it; the batcher rejects it at pre-validation with `MaxGasPriceTooLow`. The mempool accumulates transactions that will never be sequenced, and users receive unexpected failures. This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

The symmetric case (price falling) causes the gateway to reject transactions that the batcher would have accepted, silently blocking valid users.

### Likelihood Explanation
The EIP-1559 price adjusts every block. Under sustained congestion (a normal operating condition) the price rises by up to 12.5 % per block. Any transaction priced at exactly the previous block's rate — a natural choice for a user or wallet that reads the last committed block — triggers the discrepancy. No special privileges or unusual conditions are required.

### Recommendation
Replace the `l2_gas_price` read with `next_l2_gas_price` from the block header. Concretely:

1. Extend `GatewayFixedBlockStateReader::get_block_info` (or add a companion method) to expose `next_l2_gas_price` from `BlockHeaderWithoutHash.next_l2_gas_price`.
2. In `validate_resource_bounds`, use that value as `previous_block_l2_gas_price`.
3. In `run_validate_entry_point`, set `block_info.gas_prices.strk_gas_prices.l2_gas_price` to `next_l2_gas_price` so the blockifier pre-validation inside the gateway mirrors what the batcher will enforce.

### Proof of Concept

```
State:
  previous block l2_gas_price (fri) = 100_000_000_000   (100 gwei)
  previous block l2_gas_consumed    = max_block_size     (fully congested)
  next_l2_gas_price (EIP-1559)      = 112_500_000_000   (+12.5 %)

Attack:
  1. User submits V3 invoke with
       l2_gas.max_price_per_unit = 100_000_000_000

  2. Gateway stateful validator:
       threshold = 100_000_000_000 * 100% = 100_000_000_000
       tx_l2_gas_price (100g) >= threshold (100g)  → PASS

  3. Gateway blockifier validation (run_validate_entry_point):
       block_context.l2_gas_price = 100_000_000_000   (stale)
       check_fee_bounds: 100g >= 100g                 → PASS
       Transaction admitted to mempool.

  4. Batcher executes the transaction:
       block_context.l2_gas_price = 112_500_000_000   (next_l2_gas_price)
       check_fee_bounds: 100g < 112.5g
       → TransactionFeeError::InsufficientResourceBounds / MaxGasPriceTooLow
       Transaction reverted; user charged revert fee.
```

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
