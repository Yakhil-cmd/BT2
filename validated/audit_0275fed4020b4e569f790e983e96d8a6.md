### Title
Single-Step Ownership Transfer in `set_owner` Allows Irrecoverable Loss of Engine Control - (File: `engine/src/contract_methods/admin.rs`)

---

### Summary

The Aurora Engine's `set_owner` function performs an immediate, single-step ownership transfer with no confirmation required from the new owner. If an incorrect NEAR `AccountId` is supplied (e.g., a typo), ownership is irrecoverably transferred to an inaccessible account, permanently blocking all `require_owner_only`-gated functions ã including `resume_precompiles` and `resume_contract` ã which can lead to a permanent freeze of user funds.

---

### Finding Description

`set_owner` in `engine/src/contract_methods/admin.rs` reads the new owner from calldata and immediately overwrites `state.owner_id`:

```rust
pub fn set_owner<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;

        let args: SetOwnerArgs = io.read_input_borsh()?;
        if state.owner_id == args.new_owner {
            return Err(errors::ERR_SAME_OWNER.into());
        }

        state.owner_id = args.new_owner;   // ã immediate, irrecoverable
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
``` [1](#0-0) 

The only guard is `ERR_SAME_OWNER` ã there is no pending-owner pattern, no confirmation step, and no zero/existence check on the new account ID. The `SetOwnerArgs` struct carries only a bare `AccountId`: [2](#0-1) 

`require_owner_only` enforces a strict equality check against `state.owner_id`: [3](#0-2) 

Every critical administrative function gates on this check, including:

- `resume_precompiles` ã restores paused bridge precompiles
- `resume_contract` ã unpauses the entire engine
- `upgrade` / `stage_upgrade` ã deploys contract fixes
- `set_eth_connector_contract_account` ã controls which contract handles deposits/withdrawals
- `set_key_manager`, `attach_full_access_key` [4](#0-3) [5](#0-4) 

Critically, `pause_precompiles` does **not** require owner ã it uses a broader `EngineAuthorizer`: [6](#0-5) 

But `resume_precompiles` does require owner: [4](#0-3) 

---

### Impact Explanation

**Impact: High ã Permanent Freezing of Funds**

If ownership is transferred to an inaccessible account:

1. An authorized (non-owner) account can still call `pause_precompiles` to pause bridge precompiles (e.g., `exitToNear`, `exitToEthereum`).
2. `resume_precompiles` requires `require_owner_only` and can never be called again.
3. All bridge withdrawal operations are permanently blocked ã ETH and ERC-20 tokens bridged to Aurora cannot be exited back to Ethereum or NEAR.
4. Additionally, if the contract is already paused at the time of the mistake, `resume_contract` can never be called, freezing all engine operations permanently.
5. The contract cannot be upgraded to recover, since `upgrade` also requires owner.

The `EngineState` stores `owner_id` as the sole authority for all recovery paths: [7](#0-6) 

---

### Likelihood Explanation

**Likelihood: Medium**

- NEAR account IDs are arbitrary strings (e.g., `aurora.near`, `aurora-dao.near`). A single-character typo produces a syntactically valid but uncontrolled account ID that passes all current validation.
- Ownership transfers happen during governance transitions ã exactly the high-pressure, error-prone moments when typos occur.
- The mistake may not be noticed immediately, since owner functions are not called on every block. By the time it is discovered, precompiles may already have been paused for unrelated maintenance, making the freeze permanent.
- There is no on-chain event or confirmation delay that would allow detection before the state is committed.

---

### Recommendation

Replace the single-step transfer with a two-step pending-owner pattern:

1. **Step 1 (`propose_owner`)**: The current owner sets a `pending_owner_id` in state. The current owner retains full control.
2. **Step 2 (`accept_owner`)**: Only the account matching `pending_owner_id` can call `accept_owner`, which atomically moves `pending_owner_id` into `owner_id`.

If an incorrect account is proposed in step 1, the current owner simply calls `propose_owner` again with the correct account. The ownership change only completes when the new owner proves control by signing step 2.

Apply the same pattern to `set_key_manager` and `set_eth_connector_contract_account` for defense in depth. [8](#0-7) [9](#0-8) 

---

### Proof of Concept

```
1. Current owner_id = "aurora.near"
2. Owner calls set_owner({ new_owner: "auroa.near" })   // typo
3. state.owner_id is now "auroa.near" ã an account whose key is unknown
4. An authorized relayer calls pause_precompiles({ paused_mask: 0xFF })
   ã succeeds (no owner check on pause_precompiles)
5. Anyone calls resume_precompiles(...)
   ã fails: require_owner_only(&state, "aurora.near") != "auroa.near"
6. Bridge withdrawals (exitToNear, exitToEthereum precompiles) are permanently paused
7. Users cannot exit funds from Aurora ã permanent freeze
``` [10](#0-9) [11](#0-10) [6](#0-5)

pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;

        let predecessor_account_id = env.predecessor_account_id();

        let flags = PrecompileFlags::from_bits_truncate(args.paused_mask);
        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.resume_precompiles(flags);
        Ok(())
    })
}

pub fn pause_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        require_running(&state::get_state(&io)?)?;

        Ok(())

pub fn set_key_manager<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;

        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;

        let key_manager =
            serde_json::from_slice::<RelayerKeyManagerArgs>(&io.read_input().to_vec())
                .map(|args| args.key_manager)
                .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;

        if state.key_manager == key_manager {
            return Err(errors::ERR_SAME_KEY_MANAGER.into());
        }

        state.key_manager = key_manager;
        state::set_state(&mut io, &state)?;

        Ok(())
    })
}
