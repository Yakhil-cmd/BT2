Based on my thorough analysis of the codebase, I have identified a valid analog to the external report's "insufficient input validation" pattern.

### Title
Missing Lower-Bound Validation on L1/L1-Data Gas `max_price_per_unit` in `validate_resource_bounds` Allows Invalid Transactions to Pass Gateway Admission - (File: crates/apollo_gateway/src/stateless_transaction_validator.rs)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a minimum gas price floor only on `l2_gas.max_price_per_unit`, leaving `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` completely unchecked. An attacker can submit an `AllResources`-bounded transaction with zero L1/L1-data gas price that passes both stateless and stateful gateway admission, is inserted into the mempool, and then fails at blockifier `check_fee_bounds` pre-validation — the exact "invalid transaction admitted before sequencing" impact class.

### Finding Description

In `StatelessTransactionValidator::validate_resource_bounds`:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
``` [1](#0-0) 

Only `l2_gas.max_price_per_unit` is compared against `min_gas_price` (default `8_000_000_000`). The fields `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` receive no lower-bound check. The only guard that could catch a zero L1 price is the aggregate zero-fee check:

```rust
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(StatelessTransactionValidatorError::ZeroResourceBounds { ... });
}
``` [2](#0-1) 

But this check passes as long as **any** resource contributes a non-zero fee. A transaction with `l1_gas.max_price_per_unit = 0`, `l1_data_gas.max_price_per_unit = 0`, and `l2_gas.max_price_per_unit = 8_000_000_000` (meeting the L2 floor) produces a non-zero aggregate fee and clears both guards.

The stateful validator repeats the same asymmetry — it only validates L2 gas price against the previous block's price:

```rust
fn validate_tx_l2_gas_price_within_threshold(
    &self,
    tx_resource_bounds: ValidResourceBounds,
    previous_block_l2_gas_price: NonzeroGasPrice,
) -> StatefulTransactionValidatorResult<()> {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            ...
        }
        ValidResourceBounds::L1Gas(_) => {
            // No validation required for legacy transactions.
        }
    }
    Ok(())
}
``` [3](#0-2) 

A developer TODO in the same file acknowledges this gap: `// TODO(Arni): Consider running this validation for all gas prices.` [4](#0-3) 

The transaction therefore passes the full gateway pipeline (`stateless_tx_validator.validate` → `extract_state_nonce_and_run_validations`) and is forwarded to the mempool. [5](#0-4) 

At sequencing time, blockifier's `check_fee_bounds` inside `perform_pre_validation_stage` evaluates **all three** resource prices against the actual block gas prices:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [6](#0-5) 

With `l1_gas.max_price_per_unit = 0` and a non-zero actual L1 gas price, this check fails unconditionally, causing the transaction to be rejected at pre-validation — after it has already occupied a mempool slot.

### Impact Explanation

This matches the **High** impact class: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."* Any transaction whose `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` is below the actual block gas price will be admitted by the gateway but rejected by the blockifier. The gateway's own config (`StatelessTransactionValidatorConfig::min_gas_price`, default `8_000_000_000`) is applied only to L2 gas, creating a systematic gap for L1 and L1-data resources. [7](#0-6) 

### Likelihood Explanation

Exploitation requires only crafting a standard V3 `AllResources` transaction with `l1_gas.max_price_per_unit = 0` and a valid L2 gas price. No privileged access, special account state, or race condition is needed. Any unprivileged user can submit such a transaction through the public RPC endpoint.

### Recommendation

Extend `validate_resource_bounds` in `StatelessTransactionValidator` to apply the `min_gas_price` floor to all three resource price fields, mirroring what blockifier already enforces at execution time:

```rust
for (price, label) in [
    (resource_bounds.l1_gas.max_price_per_unit.0, "l1_gas"),
    (resource_bounds.l1_data_gas.max_price_per_unit.0, "l1_data_gas"),
    (resource_bounds.l2_gas.max_price_per_unit.0, "l2_gas"),
] {
    if price < self.config.min_gas_price {
        return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
            gas_price: price.into(),
            min_gas_price: self.config.min_gas_price,
        });
    }
}
```

Apply the same extension to `validate_tx_l2_gas_price_within_threshold` in `StatefulTransactionValidator` to cover the dynamic threshold check as well.

### Proof of Concept

1. Construct a V3 Invoke transaction with `AllResources` bounds:
   - `l1_gas = { max_amount: 1, max_price_per_unit: 0 }` ← zero price, bypasses stateless check
   - `l1_data_gas = { max_amount: 1, max_price_per_unit: 0 }` ← zero price
   - `l2_gas = { max_amount: 1_000_000, max_price_per_unit: 8_000_000_000 }` ← meets `min_gas_price`
   - Valid nonce, valid signature, valid sender address

2. Submit via `starknet_addInvokeTransaction`. The gateway runs:
   - `max_possible_fee(Tip::ZERO) = 8_000_000_000 * 1_000_000 > 0` → zero-bounds check passes
   - `l2_gas.max_price_per_unit (8e9) >= min_gas_price (8e9)` → L2 price check passes
   - Stateful L2 price threshold check passes
   - `__validate__` entry point runs successfully (valid signature)
   - Transaction is inserted into the mempool ✓

3. When the sequencer calls `perform_pre_validation_stage` with `charge_fee = true`:
   - `l1_gas.max_price_per_unit (0) < actual_l1_gas_price (> 0)` → `ResourceBoundsError::MaxGasPriceTooLow` for L1Gas
   - Transaction is rejected at pre-validation, never executed ✗

The transaction occupied a mempool slot despite being guaranteed to fail at sequencing, confirming the admission invariant is broken.
