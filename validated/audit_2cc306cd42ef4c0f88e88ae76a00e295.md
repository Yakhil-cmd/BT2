The bug is real and concrete. Here is the analysis:

### Title
Missing `state_number` Temporal Guard in `get_compiled_class_hash` Returns Future-Declared Class Hash — (`crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary

`ExecutionStateReader::get_compiled_class_hash` does not check whether the block at which a Sierra class was declared is before or after `self.state_number`. As a result, when a class is declared at block N and the reader's `state_number` is right-after block M < N, the function returns the actual `CompiledClassHash` from block N instead of `CompiledClassHash::default()`. Every sibling method that performs the same storage lookup correctly guards with a temporal check; this one does not.

### Finding Description

`get_compiled_class_hash` retrieves the block number at which a class was declared via `get_class_definition_block_number`, then unconditionally fetches the state diff at that block and returns the compiled class hash: [1](#0-0) 

After obtaining `block_number` at line 185, there is no check of the form `if self.state_number.is_before(block_number) { return Ok(CompiledClassHash::default()); }`. The function proceeds directly to fetch and return the compiled class hash regardless of whether `block_number > state_number`.

Contrast this with `get_contract_class` in `execution_utils.rs`, which has the explicit guard: [2](#0-1) 

And `is_contract_class_declared`, which also enforces the temporal invariant: [3](#0-2) 

The storage layer itself also has a correct time-bounded lookup (`get_compiled_class_hash_at` uses `state_number.block_after()` as the upper bound), but `get_compiled_class_hash` in the state reader bypasses this by calling `get_class_definition_block_number` and then `get_state_diff(block_number)` directly, without any temporal filter. [4](#0-3) 

### Impact Explanation

`get_compiled_class_hash` is called by `CachedState::get_compiled_class_hash` in blockifier when the cache is cold for a given class hash: [5](#0-4) 

During RPC simulation or fee estimation at a historical state M, if a class was declared at block N > M, the reader returns the real `CompiledClassHash` from block N. Blockifier then caches and uses this value for:

- **State diff computation**: `class_hash_to_compiled_class_hash` in the simulated state diff will contain an entry for a class that does not exist at state M, producing a wrong simulation result.
- **Declare transaction handling**: A declare transaction for that class hash will appear to already have a compiled class hash in the base state, causing blockifier to compute a wrong state change delta and potentially wrong fee/resource accounting for the simulation.
- **Bouncer weight calculation**: `try_update` uses `state_changes_keys` derived from the state diff; a spurious compiled class hash entry inflates or deflates the marginal state changes, corrupting the simulated bouncer weight. [6](#0-5) 

The impact is confined to RPC simulation/estimation paths (not live sequencer execution, which uses the `class_manager_handle` path with `is_contract_class_declared` guard). The result is an authoritative-looking wrong simulation/estimation value returned to the caller.

### Likelihood Explanation

Any unprivileged user can trigger this by submitting an `starknet_simulateTransactions` or `starknet_estimateFee` RPC call at a historical block ID M, referencing a class hash that was declared at any block N > M. Both the class hash and the block at which it was declared are publicly visible on-chain. No special privileges are required.

### Recommendation

Add the missing temporal guard immediately after obtaining `block_number` in `get_compiled_class_hash`, mirroring the pattern used in `get_contract_class`:

```rust
let Some(block_number) = maybe_block_number else {
    return Ok(CompiledClassHash::default());
};

// *** ADD THIS GUARD ***
if self.state_number.is_before(block_number) {
    return Ok(CompiledClassHash::default());
}
```

Alternatively, replace the manual `get_class_definition_block_number` + `get_state_diff` chain with the existing storage-layer `get_compiled_class_hash_at(state_number, &class_hash)`, which already enforces the temporal bound correctly.

### Proof of Concept


The existing test suite covers `get_compiled_class` with the temporal guard but has no equivalent test for `get_compiled_class_hash`, which is why this gap went undetected. [7](#0-6)

```rust
#[test]
fn get_compiled_class() {
    let casm_contract_class = CasmContractClass {
        compiler_version: "0.0.0".to_string(),
        prime: Default::default(),
        bytecode: Default::default(),
        bytecode_segment_lengths: Default::default(),
        hints: Default::default(),
        pythonic_hints: Default::default(),
        entry_points_by_type: Default::default(),
    };
    let expected_result = casm_contract_class.clone();

    let mut mock_class_manager_client = MockClassManagerClient::new();
    mock_class_manager_client.expect_get_executable().returning(move |_| {
        Ok(Some(ContractClass::V1((casm_contract_class.clone(), SierraVersion::default()))))
    });
    let shared_mock_class_manager_client = Arc::new(mock_class_manager_client);

    // Set execution state reader with state_number to be right after block 0.
    let (mut exec_state_reader, mut storage_writer) =
        set_execution_state_reader(shared_mock_class_manager_client.clone(), 0);

    // Prepare storage so class with hash 0x2 is declared in block 0 and class with hash 0x3 is
    // declared in block 1.
    let class_hash_0x2 = class_hash!("0x2");
    let class_hash_0x3 = class_hash!("0x3");

    storage_writer
        .begin_rw_txn()
        .unwrap()
        .append_header(BlockNumber(0), &BlockHeader::default())
        .unwrap()
        .append_state_diff(
            BlockNumber(0),
            ThinStateDiff {
                class_hash_to_compiled_class_hash: indexmap!(class_hash_0x2 => compiled_class_hash!(1_u8)),
                ..Default::default()
            },
        )
        .unwrap()
        .append_header(
            BlockNumber(1),
            &BlockHeader { block_hash: BlockHash(felt!(1_u128)), ..Default::default() },
        )
        .unwrap()
        .append_state_diff(
            BlockNumber(1),
            ThinStateDiff {
                class_hash_to_compiled_class_hash: indexmap!(class_hash_0x3 => compiled_class_hash!(2_u8)),
                ..Default::default()
            },
        )
        .unwrap()
        .commit()
        .unwrap();

    // At state_number right after block 0, class with hash 0x2 is already declared.
    let result = exec_state_reader.get_compiled_class(class_hash_0x2).unwrap();
    assert_eq!(

        RunnableCompiledClass::V1(
            (expected_result.clone(), SierraVersion::default()).try_into().unwrap()
        )
    );

    // Class with hash 0x3 is not yet declared.
    let result = exec_state_reader.get_compiled_class(class_hash_0x3);
    assert_matches!(result, Err(StateError::UndeclaredClassHash(hash)) if hash == class_hash_0x3);

    // Adjust execution state reader state_number to be right after block 1.
    exec_state_reader.state_number = StateNumber::unchecked_right_after_block(BlockNumber(1));

    // Class with hash 0x2 is still declared.
    let result = exec_state_reader.get_compiled_class(class_hash_0x2).unwrap();
    assert_eq!(
        result,
        RunnableCompiledClass::V1(
            (expected_result.clone(), SierraVersion::default()).try_into().unwrap()
        )
    );

    // And class with hash 0x3 is declared too.
    let result = exec_state_reader.get_compiled_class(class_hash_0x3).unwrap();
    assert_eq!(
        result,
        RunnableCompiledClass::V1((expected_result, SierraVersion::default()).try_into().unwrap())
    );
```
