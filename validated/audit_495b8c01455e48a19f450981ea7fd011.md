### Title
`Debug` Trait on Secret-Bearing Types Exposes Private Signing Shares and Presign Secrets to Callers — (`File: src/lib.rs`, `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/ot_based_ecdsa/triples/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

Multiple production types holding secret cryptographic material unconditionally derive or implement `std::fmt::Debug`, causing their full secret contents to be emitted whenever a caller formats them with `{:?}`. Any logging framework, error-reporting service, or tracing backend that receives one of these values will capture and potentially persist the raw private signing share, Beaver-triple scalars, or presign nonce shares.

---

### Finding Description

The following production types derive `Debug` while containing secret scalar material:

**1. `KeygenOutput<C>` — `src/lib.rs:48–55`**
```rust
#[derive(Debug, Clone, Deserialize, Serialize, Eq, PartialEq, ZeroizeOnDrop)]
pub struct KeygenOutput<C: Ciphersuite> {
    pub private_share: SigningShare<C>,   // ← secret signing share
    #[zeroize[skip]]
    pub public_key: VerifyingKey<C>,
}
```
`private_share` is the participant's share of the distributed secret key. Formatting `KeygenOutput` with `{:?}` emits the raw scalar.

**2. `TripleShare` — `src/ecdsa/ot_based_ecdsa/triples/mod.rs:72–77`**
```rust
#[derive(Clone, Debug, Serialize, Deserialize, ZeroizeOnDrop)]
pub struct TripleShare {
    pub a: Scalar,   // ← secret
```

The module's own documentation states: *"It's important that the value of the underlying scalars in the triple is kept secret, otherwise the private key used to create a signature with that triple could be recovered."* Despite this, `Debug` is derived unconditionally.

### Impact Explanation

**Critical — Extraction, reconstruction, or disclosure of private signing shares, presign secrets, and nonce material.**

- Leaking `KeygenOutput.private_share` gives an adversary one participant's signing share. Collecting threshold-many such shares (e.g., from a shared log sink, a crash-reporting service, or a tracing backend) allows full private key reconstruction.
- Leaking `TripleShare.(a, b, c)` for a triple used in a presignature allows the attacker to recover the private signing key from the resulting signature, as documented in the module's own comments.
- Leaking `PresignOutput.(k, sigma)` or the robust equivalents `(c, e, alpha, beta)` allows nonce recovery and therefore private key extraction from any signature produced with that presignature.

---

### Likelihood Explanation

**High.** These are public library types returned directly to callers. Rust's `Debug` trait is routinely invoked by:
- `tracing`/`log` macros (`tracing::debug!("{:?}", keygen_out)`)
- Error-handling crates that format context values (`anyhow`, `thiserror`)
- Crash/panic reporters and third-party observability services (the direct analog to ShakeBugs)
- Test harnesses and `assert!` macros that print `{:?}` on failure
- `serde_json::to_string` alternatives that fall back to `Debug`

No attacker action is required beyond the caller using standard Rust idioms. The `ZeroizeOnDrop` on these types shows the authors are aware of secret-handling requirements, making the unguarded `Debug` derive an oversight rather than an intentional design choice.

---

### Recommendation

- **Short term:** Replace `#[derive(Debug)]` on all secret-bearing types with a manual `Debug` implementation that redacts secret fields, e.g.:
  ```rust
  impl<C: Ciphersuite> fmt::Debug for KeygenOutput<C> {
      fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
          f.debug_struct("KeygenOutput")
              .field("private_share", &"[REDACTED]")
              .field("public_key", &self.public_key)
              .finish()
      }
  }
  ```
  Apply the same pattern to `TripleShare`, `PresignArguments`, `PresignOutput` (both variants), and `RerandomizedPresignOutput` (both variants).
- **Long term:** Audit all types that implement or derive `Debug`, `Display`, `Serialize`, or `Clone` for inadvertent secret exposure. Consider a `#[sensitive]`-style proc-macro or a newtype wrapper (e.g., `Secret<T>`) that enforces redacted formatting across the codebase.

### Proof of Concept

```rust
use threshold_signatures::{KeygenOutput, ecdsa::ot_based_ecdsa::triples::TripleShare};

// After DKG completes, a caller logs the output — standard Rust idiom:
let keygen_out: KeygenOutput<_> = run_keygen(...);
tracing::debug!("DKG complete: {:?}", keygen_out);
// → emits: KeygenOutput { private_share: Scalar(0xdeadbeef...), public_key: ... }
// The raw private signing share scalar is now in the log.

// Similarly for PresignArguments, which bundles everything:
let args = PresignArguments { triple0, triple1, keygen_out, threshold };
tracing::error!("Presign failed, args: {:?}", args);
// → emits all three secret scalars of triple0, triple1, AND the private signing share.
```
