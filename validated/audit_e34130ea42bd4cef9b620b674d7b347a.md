### Title
IBC Packet Permanently DOS'd for Blocklisted Receiver via `IBCConversionModule.OnRecvPacket` — (`x/cronos/middleware/conversion_middleware.go`)

---

### Summary

When an IBC denom is mapped to an external CRC20 contract that implements a blocklist, and the IBC packet receiver is blocklisted by that contract, `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` → `CallModuleCRC21("mint_by_cronos_module", ...)` reverts. Because `OnRecvPacket` propagates this error as `channeltypes.NewErrorAcknowledgement(err)` and never calls `commit()`, the underlying IBC transfer is also rolled back. The receiver is permanently unable to receive IBC transfers of that denom.

---

### Finding Description

`IBCConversionModule.OnRecvPacket` wraps the underlying transfer module in a `cacheCtx`:

```go
// x/cronos/middleware/conversion_middleware.go:112-146
cacheCtx, commit := ctx.CacheContext()
ack := im.app.OnRecvPacket(cacheCtx, channelVersion, packet, relayer)
if !ack.Success() {
    return ack
}
// ...
if err := im.cronoskeeper.ConvertVouchersToEvmCoins(cacheCtx, data.Receiver, sdk.NewCoins(token)); err != nil {
    im.cronoskeeper.Logger(ctx).Error(...)
    return channeltypes.NewErrorAcknowledgement(err)   // commit() is never called
}
commit()
return ack
``` [1](#0-0) 

`ConvertVouchersToEvmCoins` for non-IBC-CRO denoms calls `ConvertCoinFromNativeToCRC21`, which calls `CallModuleCRC21(ctx, contract, "mint_by_cronos_module", sender, amount)`:

```go
// x/cronos/keeper/evm.go:137
_, err = k.CallModuleCRC21(ctx, contract, "mint_by_cronos_module", sender, coin.Amount.BigInt())
``` [2](#0-1) 

`CallModuleCRC21` checks `res.Failed()` and returns an error if the EVM call reverts:

```go
// x/cronos/keeper/evm.go:64-66
if res.Failed() {
    return nil, fmt.Errorf("call contract failed: %s, %s, %s", contract.Hex(), method, res.Ret)
}
``` [3](#0-2) 

If the external CRC20 contract's `mint_by_cronos_module` reverts for a blocklisted receiver (e.g., a USDC-style contract that checks a blocklist before minting), the error propagates all the way back to `OnRecvPacket`, which returns an error acknowledgement. Because `commit()` is never reached, the underlying IBC transfer state is also discarded. The receiver gets neither CRC20 tokens nor IBC vouchers.

This is in direct contrast to the `OnAcknowledgementPacket` and `OnTimeoutPacket` refund paths, which explicitly log and continue on conversion failure so the IBC refund is never blocked:

```go
// x/cronos/middleware/conversion_middleware.go:179-188
if err := im.convertVouchers(...); err != nil {
    // Intentional: log and continue so the IBC refund is not blocked.
    im.cronoskeeper.Logger(ctx).Error(...)
}
``` [4](#0-3) 

The `OnRecvPacket` path has no equivalent safety valve.

---

### Impact Explanation

Every IBC packet sent to a blocklisted receiver for a denom mapped to a blocklist-capable external CRC20 contract will permanently fail with an error acknowledgement. The receiver is completely unable to receive IBC transfers of that denom — not even as raw IBC vouchers — because the `cacheCtx` rollback discards the underlying transfer module's state writes. This is a **permanent, long-lived inability for an honest user to receive valid IBC transfers**, qualifying as **High** impact under the allowed scope.

---

### Likelihood Explanation

The preconditions are realistic:
1. An external CRC20 contract with a blocklist (e.g., a USDC-equivalent) is registered via `UpdateTokenMapping` for an IBC denom — a standard Cronos admin operation.
2. The token issuer (contract owner) exercises their legitimate authority to blocklist a user.

The token issuer is not a Cronos admin or governance actor; they are an external party operating their own contract. This mirrors the original report's scenario (USDC issuer blacklisting a user). The `TestBlackListERC20` contract already present in the integration test suite confirms the team has considered this token class. [5](#0-4) 

---

### Recommendation

Apply the same "log and continue" pattern used in `OnAcknowledgementPacket` and `OnTimeoutPacket` to the `OnRecvPacket` conversion step. If `ConvertVouchersToEvmCoins` fails, log the error, call `commit()` so the underlying IBC transfer is preserved, and return the success acknowledgement. The receiver retains the raw IBC vouchers and can retry conversion manually — identical to the refund-path design already in place. [6](#0-5) 

---

### Proof of Concept

1. Deploy an external CRC20 contract that implements `mint_by_cronos_module` but reverts when the target address is on its internal blocklist.
2. Register the contract for an IBC denom via `MsgUpdateTokenMapping` (CronosAdmin action).
3. Add victim address to the contract's blocklist.
4. Relay an IBC fungible-token packet whose `receiver` is the victim address and whose denom resolves to the mapped contract.
5. `OnRecvPacket` calls `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` → `CallModuleCRC21("mint_by_cronos_module", victim, amount)`.
6. The contract reverts; `res.Failed()` is `true`; `CallModuleCRC21` returns an error.
7. `OnRecvPacket` returns `channeltypes.NewErrorAcknowledgement(err)` without calling `commit()`.
8. The source chain refunds the sender; the victim receives nothing and cannot receive this denom via IBC as long as they remain blocklisted. [7](#0-6) [8](#0-7)

// OnRecvPacket implements the IBCModule interface.
func (im IBCConversionModule) OnRecvPacket(
	ctx sdk.Context,
	channelVersion string,
	packet channeltypes.Packet,
	relayer sdk.AccAddress,
) exported.Acknowledgement {
	cacheCtx, commit := ctx.CacheContext()
	ack := im.app.OnRecvPacket(cacheCtx, channelVersion, packet, relayer)
	if !ack.Success() {
		// Underlying transfer failed: discard cacheCtx writes and return the
		// failure ack. Committing would persist a half-applied transfer.
		return ack
	}
	data, err := transferTypes.UnmarshalPacketData(packet.GetData(), channelVersion, "")
	if err != nil {
		return channeltypes.NewErrorAcknowledgement(errors.Wrap(sdkerrors.ErrUnknownRequest,
			"cannot unmarshal ICS-20 transfer packet data in middleware"))
	}
	denom := im.getIbcDenomFromPacketAndData(packet, data.Token)
	if im.canBeConverted(cacheCtx, denom) {
		transferAmount, ok := sdkmath.NewIntFromString(data.Token.Amount)
		if !ok {
			return channeltypes.NewErrorAcknowledgement(errors.Wrapf(
				transferTypes.ErrInvalidAmount,
				"unable to parse transfer amount (%s) into sdk.Int in middleware",
				data.Token.Amount,
			))
		}
		token := sdk.NewCoin(denom, transferAmount)
		if err := im.cronoskeeper.ConvertVouchersToEvmCoins(cacheCtx, data.Receiver, sdk.NewCoins(token)); err != nil {
			im.cronoskeeper.Logger(ctx).Error(
				"failed to convert vouchers on recv",
				"denom", denom,
				"receiver", data.Receiver,
				"error", err,
			)
			return channeltypes.NewErrorAcknowledgement(err)
		}
	}
	commit()
	return ack
}

func (k Keeper) ConvertCoinFromNativeToCRC21(ctx sdk.Context, sender common.Address, coin sdk.Coin, autoDeploy bool) error {
	if !types.IsValidCoinDenom(coin.Denom) {
		return fmt.Errorf("coin %s is not supported for conversion", coin.Denom)
	}
	var err error
	// external contract is returned in preference to auto-deployed ones
	contract, found := k.GetContractByDenom(ctx, coin.Denom)
	if !found {
		if !autoDeploy {
			return fmt.Errorf("no contract found for the denom %s", coin.Denom)
		}
		contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
		if err != nil {
			return err
		}
		if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
			return err
		}

		k.Logger(ctx).Info(fmt.Sprintf("contract address %s created for coin denom %s", contract.String(), coin.Denom))
	}

	isSource := types.IsSourceCoin(coin.Denom)
	coins := sdk.NewCoins(coin)
	if isSource {
		// burn coins
		err = k.bankKeeper.SendCoinsFromAccountToModule(ctx, sdk.AccAddress(sender.Bytes()), types.ModuleName, sdk.NewCoins(coin))
		if err != nil {
			return err
		}
		err = k.bankKeeper.BurnCoins(ctx, types.ModuleName, coins)
		if err != nil {
			return err
		}
		// unlock crc tokens
		_, err = k.CallModuleCRC21(ctx, contract, "transfer_from_cronos_module", sender, coin.Amount.BigInt())
		if err != nil {
			return err
		}
	} else {
		// send coins to contract address
		err = k.bankKeeper.SendCoins(ctx, sdk.AccAddress(sender.Bytes()), sdk.AccAddress(contract.Bytes()), coins)
		if err != nil {
			return err
		}
		// mint crc tokens
		_, err = k.CallModuleCRC21(ctx, contract, "mint_by_cronos_module", sender, coin.Amount.BigInt())
		if err != nil {
			return err
		}
	}

	return nil
}

func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
	}
	defer func() {
		for _, a := range coins {
			if a.Amount.IsInt64() {
				telemetry.SetGaugeWithLabels(
					[]string{"tx", "msg", "ConvertVouchersToEvmCoins"},
					float32(a.Amount.Int64()),
					[]metrics.Label{telemetry.NewLabel("denom", a.Denom)},
				)
			}
		}
	}()
	return nil
