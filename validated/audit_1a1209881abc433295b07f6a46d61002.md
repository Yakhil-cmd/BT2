### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Deny or Redirect Block Rewards — (`File: src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is documented as callable only by the Starkware sequencer, but the implementation contains no caller verification. Any unprivileged address can invoke it, either to permanently deny block rewards to all stakers (by passing `disable_rewards: true`) or to redirect block rewards to an attacker-controlled staker (by passing `disable_rewards: false` with their own staker address).

---

### Finding Description

`update_rewards` is the consensus-era reward distribution entry point. Its spec explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation only calls `general_prerequisites()` (which checks the pause flag) and a block-number guard. There is no `get_caller_address()` check against a stored sequencer address. [1](#0-0) 

The critical ordering flaw is that `last_reward_block` is written to storage **before** rewards are distributed and **before** any early-return guard: [2](#0-1) 

Once `last_reward_block` equals the current block number, every subsequent call in that block reverts with `REWARDS_ALREADY_UPDATED`: [3](#0-2) 

`last_reward_block` is a single global storage slot shared across all stakers: [4](#0-3) 

The interface definition confirms no access-control annotation exists at the ABI level either: [5](#0-4) 

**Attack path A — freeze all block rewards (griefing):**
1. Attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of block N.
2. `last_reward_block` is set to N; the function returns early with no rewards distributed.
3. The sequencer's subsequent call for block N reverts with `REWARDS_ALREADY_UPDATED`.
4. All stakers permanently lose their block-N rewards.

**Attack path B — steal block rewards (theft):**
1. Attacker calls `update_rewards(attacker_staker, disable_rewards: false)` before the sequencer.
2. The attacker's staker receives the full block rewards.
3. The sequencer cannot distribute rewards to the intended staker for that block.
4. Repeated every block, the attacker captures all consensus block rewards.

The `_update_rewards` internal function, which performs the actual reward accounting and pool transfers, is only reached after `last_reward_block` is already committed: [6](#0-5) 

---

### Impact Explanation

- **Attack path A**: Permanent freezing of unclaimed yield for all stakers. Every block's rewards are irreversibly lost because `last_reward_block` cannot be rolled back.
- **Attack path B**: Theft of unclaimed yield. The attacker's staker accumulates `unclaimed_rewards_own` and pool rewards that rightfully belong to the sequencer-selected staker.

Both impacts are within the allowed scope: *"Theft of unclaimed yield"* and *"Permanent freezing of unclaimed yield"*.

---

### Likelihood Explanation

The Starknet sequencer controls transaction ordering and could mitigate this by always placing its own `update_rewards` call first. However:

1. The contract itself provides **zero on-chain enforcement** — the vulnerability is unconditionally present in the deployed code.
2. Any liveness gap (sequencer downtime, missed block, upgrade window) opens the attack.
3. The attacker needs only a valid staker address (publicly available from `NewStaker` events) and the ability to submit a transaction.
4. No capital, no privilege, and no complex setup is required.

Likelihood: **Medium** (sequencer ordering is an off-chain mitigation, not an on-chain guarantee).

---

### Recommendation

Add an explicit sequencer-address check inside `update_rewards`, analogous to how `update_rewards_from_attestation_contract` enforces its caller: [7](#0-6) 

Store the authorized sequencer address in contract storage and assert it at the top of `update_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   assert!(
+       get_caller_address() == self.sequencer_address.read(),
+       "{}",
+       Error::CALLER_IS_NOT_SEQUENCER,
+   );
    // ... rest of function
}
```

Alternatively, use an existing role (e.g., a dedicated `REWARDS_MANAGER_ROLE`) from the `RolesComponent` already present in the contract.

---

### Proof of Concept

```
// Precondition: consensus rewards are active, staker_A is a valid active staker.

// Step 1: Attacker (any address) calls update_rewards with disable_rewards=true.
//         No special role or funds required.
staking.update_rewards(staker_address: staker_A, disable_rewards: true);
// last_reward_block is now set to current_block. No rewards distributed.

// Step 2: Sequencer attempts to distribute rewards for the same block.
staking.update_rewards(staker_address: staker_B, disable_rewards: false);
// → PANICS with Error::REWARDS_ALREADY_UPDATED

// Result: staker_B (and all other stakers) permanently lose block rewards for this block.
// Repeating step 1 every block denies all consensus block rewards indefinitely.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
