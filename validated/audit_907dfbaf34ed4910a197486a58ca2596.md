### Title
Missing Caller Authorization in `update_rewards` Allows Anyone to Permanently Freeze Consensus Rewards - (File: `src/staking/staking.cairo`)

### Summary
The `update_rewards` function in the staking contract is publicly callable with no access control. It accepts a `disable_rewards: bool` parameter and updates the global `last_reward_block` storage variable regardless of whether rewards are actually distributed. An unprivileged attacker can call `update_rewards(any_valid_staker, disable_rewards: true)` every block to consume the per-block reward slot without distributing any rewards, permanently freezing consensus yield for all stakers.

### Finding Description

`update_rewards` is exposed via the `IStakingRewardsManager` public interface with no caller restriction: [1](#0-0) 

The function enforces a global one-call-per-block invariant via `last_reward_block`: [2](#0-1) 

It then unconditionally writes the current block number to `last_reward_block` before checking `disable_rewards`: [3](#0-2) 

`last_reward_block` is a single global storage slot, not per-staker: [4](#0-3) 

Because the write to `last_reward_block` happens **before** the `disable_rewards` early-return, any caller who passes `disable_rewards: true` burns the reward slot for the entire block. Any subsequent legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`.

The analog to the external report is exact: just as `placeMagnet` accepts `_empire` as a caller-supplied parameter without verifying the planet belongs to that empire, `update_rewards` accepts `staker_address` and `disable_rewards` without verifying the caller is authorized to suppress reward distribution. In both cases an unauthorized caller can affect the protocol's accounting for another entity.

The interface declaration confirms no access restriction is documented: [5](#0-4) 

By contrast, the attestation-triggered reward path correctly gates itself: [6](#0-5) 

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` once per block:
- Writes `last_reward_block = current_block` with zero rewards distributed.
- Blocks every legitimate call in that block with `REWARDS_ALREADY_UPDATED`.
- Repeating this every block permanently prevents all stakers from accumulating consensus block rewards.

Because `last_reward_block` is global, a single attacker transaction per block is sufficient to freeze yield for the entire staker set. The lost rewards are not recoverable — there is no catch-up mechanism.

### Likelihood Explanation

**Medium.** The attack requires no special role, no funds at risk for the attacker (only gas), and no coordination. Any public caller can execute it. The only cost is gas per block, which on Starknet is low. A motivated griever (e.g., a competing validator) has clear incentive to suppress rivals' rewards.

### Recommendation

Add a caller restriction to `update_rewards`. The function should only be callable by the attestation contract (or another trusted, protocol-controlled address), mirroring the pattern already used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   self.assert_caller_is_attestation_contract(); // or a dedicated rewards manager role
    ...
}
```

Alternatively, derive `disable_rewards` internally from protocol state rather than accepting it as a caller-supplied parameter.

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker (any EOA) monitors the mempool for each new block.
3. At each block, attacker calls:
   ```
   staking.update_rewards(staker_address=<any_active_staker>, disable_rewards=true)
   ```
4. The call passes all checks (staker is valid, `current_block > last_reward_block`), writes `last_reward_block = current_block`, then returns early — zero rewards distributed.
5. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeating step 3 every block permanently freezes all consensus block rewards for all stakers.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
