### Title
Unauthorized Overwrite of Any User's E2EE Encryption Key via Missing Signer Authorization Check — (`File: x/e2ee/keeper/keeper.go`)

### Summary
The `RegisterEncryptionKey` message handler in `x/e2ee/keeper/keeper.go` stores an encryption key for the address specified in `req.Address` without verifying that the transaction signer is the same address. Any unprivileged user can overwrite any other user's (or validator's) registered encryption key with an attacker-controlled key, corrupting e2ee key state and enabling interception of encrypted communications.

### Finding Description
The `RegisterEncryptionKey` handler unconditionally writes `req.Key` to the KV store under `req.Address`:

```go
// x/e2ee/keeper/keeper.go
func (k Keeper) RegisterEncryptionKey(
    ctx context.Context,
    req *types.MsgRegisterEncryptionKey,
) (*types.MsgRegisterEncryptionKeyResponse, error) {
    if err := k.registerEncryptionKey(ctx, req.Address, []byte(req.Key)); err != nil {
        return nil, err
    }
    return &types.MsgRegisterEncryptionKeyResponse{}, nil
}
``` [1](#0-0) 

The internal helper simply converts the address and writes to the store with no authorization check:

```go
func (k Keeper) registerEncryptionKey(
    ctx context.Context,
    address string,
    key []byte,
) error {
    bz, err := k.addressCodec.StringToBytes(address)
    if err != nil {
        return err
    }
    sdk.UnwrapSDKContext(ctx).KVStore(k.storeKey).Set(types.KeyPrefix(bz), key)
    return nil
}
``` [2](#0-1) 

There is no guard of the form `if req.Signer != req.Address { return ErrUnauthorized }`. The `req.Address` field is a free parameter in the message body, entirely independent of who signed the transaction.

### Impact Explanation
An attacker submits a `MsgRegisterEncryptionKey` with `Address = victim` and `Key = attacker_pubkey`. The victim's entry in the KV store is silently overwritten. Any party that subsequently queries the victim's key (e.g., to encrypt a message for that validator) will encrypt to the attacker's key instead. This directly corrupts e2ee key/message state with security impact — the attacker can decrypt all future messages intended for the victim. This falls squarely within the allowed **High** impact category: *"Corruption of … e2ee key/message state with direct security impact."*

### Likelihood Explanation
The attack requires only a funded account to pay gas. No special permissions, leaked keys, or admin access are needed. The message is a standard Cosmos SDK `Msg` reachable by any unprivileged user on-chain. The attacker can target any address whose key is registered, including validators.

### Recommendation
Add a signer-equals-address check at the top of `RegisterEncryptionKey`:

```go
func (k Keeper) RegisterEncryptionKey(
    ctx context.Context,
    req *types.MsgRegisterEncryptionKey,
) (*types.MsgRegisterEncryptionKeyResponse, error) {
    // Ensure the transaction signer is the address whose key is being registered.
    signers := req.GetSigners() // or extract from ctx
    if len(signers) == 0 || signers[0].String() != req.Address {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized,
            "signer must match the address in MsgRegisterEncryptionKey")
    }
    if err := k.registerEncryptionKey(ctx, req.Address, []byte(req.Key)); err != nil {
        return nil, err
    }
    return &types.MsgRegisterEncryptionKeyResponse{}, nil
}
```

### Proof of Concept

```
// Attacker (addr_attacker) overwrites validator's (addr_validator) encryption key:

MsgRegisterEncryptionKey {
    Address: "crc1<addr_validator>",   // victim's address
    Key:     "<attacker_pubkey>",       // attacker-controlled key
    // Signed by: addr_attacker
}

// Result: KVStore[KeyPrefix(addr_validator)] = attacker_pubkey
// Any subsequent query:
//   KeyRequest{Address: "crc1<addr_validator>"} → attacker_pubkey
// All messages encrypted to addr_validator are now readable by attacker.

	if err != nil {
		return nil, err
	}

	value := sdkCtx.KVStore(k.storeKey).Get(types.KeyPrefix(bz))
	return &types.KeyResponse{Key: string(value)}, nil
