### Title
Permissionless `collectPoolFees` unconditionally resets notional fee accumulators before floor-rounding transfer, permanently destroying accrued fees for low-decimal tokens — (`File: metric-core/contracts/MetricOmmPool.sol`, `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.collectPoolFees` is explicitly permissionless. When called while `notionalFeeToken0Scaled` or `notionalFeeToken1Scaled` holds a value smaller than the token's scale multiplier, `deltasScaledToExternal(..., Math.Rounding.Floor)` returns 0 external units, no tokens are transferred, but the accumulators are unconditionally zeroed. The accrued scaled fee is permanently destroyed. An attacker can call `collectPoolFees` after every small swap to drain all notional fees for USDC/USDT/WBTC pools.

---

### Finding Description

`collectPoolFees` in the factory is documented and coded as callable by any address: [1](#0-0) 

It forwards to `MetricOmmPool.collectFees`, which converts scaled fee amounts to external token units using floor rounding: [2](#0-1) 

After the (possibly zero) transfers, the notional accumulators are unconditionally zeroed: [3](#0-2) 

The scale multiplier for a token with `d` decimals is `10^(18 - d)`. For USDC/USDT (`d = 6`), `TOKEN_0_SCALE_MULTIPLIER = 10^12`. For WBTC (`d = 8`), it is `10^10`. Any `notionalFeeToken*Scaled` value below the multiplier threshold produces a floor-division result of 0 external units. The transfer is skipped (guarded by `> 0`), but the accumulator is still set to 0: [4](#0-3) 

The notional fee accumulates during every swap: [5](#0-4) 

An attacker calls `collectPoolFees` immediately after each swap that generates fewer than `TOKEN_*_SCALE_MULTIPLIER` scaled notional fee units. Each call zeros the accumulator with no transfer. The fees are permanently lost.

---

### Impact Explanation

Every unit of notional fee that accumulates below the scale-multiplier threshold is silently destroyed. For a USDC pool (`TOKEN_0_SCALE_MULTIPLIER = 10^12`), any swap generating less than `1e12` scaled notional fee units (i.e., less than 1 USDC of notional fee) is fully erasable. An attacker who calls `collectPoolFees` after each such swap causes total loss of all notional fees for both protocol and admin. This is a direct loss of protocol and admin revenue above Sherlock thresholds, matching the "public fee-collection leakage" invariant: public fee collection must extract only already-accrued fees and must never destroy LP- or protocol-owned value.

---

### Likelihood Explanation

`collectPoolFees` requires no privilege, no capital, and no special timing beyond "call after a small swap." On any USDC, USDT, or WBTC pool with a non-zero `notionalFeeE8`, an attacker can sustain the attack indefinitely at negligible gas cost. The condition `notionalFeeToken*Scaled < TOKEN_*_SCALE_MULTIPLIER` is trivially reachable on every pool with low-decimal tokens and any realistic swap size.

---

### Recommendation

Do not unconditionally zero `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled`. Instead, subtract only the scaled amount that was actually converted and transferred:

```solidity
// compute scaled amounts that were actually sent (reverse the floor conversion)
uint256 sent0Scaled = totalFee0ToAdmin * TOKEN_0_SCALE_MULTIPLIER
                    + totalFee0ToProtocol * TOKEN_0_SCALE_MULTIPLIER;
uint256 sent1Scaled = totalFee1ToAdmin * TOKEN_1_SCALE_MULTIPLIER
                    + totalFee1ToProtocol * TOKEN_1_SCALE_MULTIPLIER;

notionalFeeToken0Scaled = uint128(notionalFee0AmountScaled - sent0Scaled);
notionalFeeToken1Scaled = uint128(notionalFee1AmountScaled - sent1Scaled);
```

Alternatively, accumulate the remainder and only zero the accumulator when the full scaled amount has been transferred.

---

### Proof of Concept

```
Setup:
  token0 = USDC (6 decimals), TOKEN_0_SCALE_MULTIPLIER = 1e12
  notionalFeeE8 = 1e6 (1%)
  Pool has liquidity; oracle price ≈ 1.

Step 1: Swap 99 USDC worth of token1 → token0 (exact-in).
  notional fee on output ≈ 0.99 USDC = 990_000 raw USDC units
  notionalFeeToken0Scaled += 990_000 * 1e12 = 9.9e17  (< 1e12? No, 9.9e17 > 1e12)

Adjust: use a tiny swap of 0.000001 USDC worth.
  output ≈ 1 raw USDC unit (1e-6 USDC)
  notional fee = 1 * 1e6 / 1e8 = 0.01 raw units → rounds to 0 in external
  notionalFeeToken0Scaled += 1 * 1e12 * 1e6 / 1e8 = 1e10

Step 2: Attacker calls collectPoolFees immediately.
  notionalFee0AmountScaled = 1e10
  totalFee0ToAdmin + totalFee0ToProtocol via deltasScaledToExternal(1e10, Floor)
    = floor(1e10 / 1e12) = 0 USDC transferred
  notionalFeeToken0Scaled = 0  ← accumulator destroyed

Step 3: Repeat after every small swap.
  All notional fees for USDC are permanently lost.
  Protocol and admin receive 0 USDC despite non-zero notionalFeeE8.
```
