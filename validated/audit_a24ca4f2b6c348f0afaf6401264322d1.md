### Title
Gateway Stateful Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Incorrect Admission Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check compares a transaction's `max_price_per_unit` against the **current block's** `l2_gas_price` (a spot value from the last committed block). The actual execution block will use `next_l2_gas_price`, which is computed by the EIP-1559 fee market and stored in the block header. When the fee market drives `next_l2_gas_price` above `current_block.l2_gas_price`, the gateway admits transactions that will deterministically fail at execution. The code itself acknowledges this with an explicit TODO comment.

### Finding Description

In `validate_resource_bounds`, the gateway reads the **current** block's `l2_gas_price` from `gateway_fixed_block_state_reader.get_block_info()` and uses it as the reference price for admission:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold check then admits any transaction whose `l2_gas.max_price_per_unit >= min_gas_price_percentage% × current_block.l2_gas_price`: [2](#0-1) 

The same stale `block_info` is reused in `run_validate_entry_point` to build the `BlockContext` for blockifier pre-validation (`check_fee_bounds` / `verify_can_pay_committed_bounds`):

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
``` [3](#0-2) 

So both the soft admission check and the blockifier pre-validation at the gateway use `current_block.l2_gas_price`. However, the actual execution block uses `next_l2_gas_price`, which is computed by the EIP-1559 fee market (`calculate_next_l2_gas_price_for_fin`) and stored in the block header: [4](#0-3) [5](#0-4) 

When gas usage exceeds the target, `next_l2_gas_price > current_block.l2_gas_price`. A transaction admitted with `max_price_per_unit = current_block.l2_gas_price` will pass both the gateway soft check and the blockifier pre-validation at the gateway, enter the mempool, and then fail at execution in the batcher with `MaxGasPriceTooLow` because the actual block price is higher.

The inverse also holds: when gas usage is below target, `next_l2_gas_price < current_block.l2_gas_price`. Transactions with `max_price_per_unit` between the two values are incorrectly rejected at the gateway even though they would succeed at execution.

### Impact Explanation

This matches **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

- **Invalid transactions admitted**: When `next_l2_gas_price > current_block.l2_gas_price` (high-load blocks), transactions with `max_price_per_unit` in the range `[current_block.l2_gas_price, next_l2_gas_price)` pass gateway admission and enter the mempool, but will fail at execution in the batcher. These transactions consume mempool slots and batcher execution resources.
- **Valid transactions rejected**: When `next_l2_gas_price < current_block.l2_gas_price` (low-load blocks), transactions with `max_price_per_unit` in the range `[next_l2_gas_price, threshold)` are incorrectly rejected at the gateway even though they would succeed at execution.

### Likelihood Explanation

The EIP-1559 fee market adjusts `next_l2_gas_price` every block based on gas usage relative to `gas_target`. Any block with gas usage above or below target produces a `next_l2_gas_price` that differs from `current_block.l2_gas_price`. This is the normal operating condition of the fee market, not an edge case. The TODO comment in the production code confirms the developers are aware the wrong value is being used. [6](#0-5) 

### Recommendation

Replace the read of `block_info.gas_prices.strk_gas_prices.l2_gas_price` with a read of `next_l2_gas_price` from the block header. The `StorageBlockHeader` already stores this field. The `GatewayFixedBlockStateReader` (or its underlying sync-state reader) should expose `next_l2_gas_price` so that `validate_resource_bounds` can compare the transaction's `max_price_per_unit` against the price that will actually govern the next block. Similarly, `run_validate_entry_point` should build its `BlockContext` using `next_l2_gas_price` as the L2 gas price so that blockifier pre-validation at the gateway is consistent with actual execution.

### Proof of Concept

1. Observe the current block's `l2_gas_price` = P and `next_l2_gas_price` = P' > P (any block with gas usage above target).
2. Submit an invoke transaction V3 with `l2_gas.max_price_per_unit = P` and `min_gas_price_percentage = 100`.
3. Gateway `validate_resource_bounds` computes threshold = `100% × P = P`. Since `P >= P`, the transaction is admitted.
4. Blockifier pre-validation at the gateway (`check_fee_bounds`) uses the same `block_info` with `l2_gas_price = P`. Since `P >= P`, it passes.
5. Transaction enters the mempool.
6. Batcher builds the next block with `l2_gas_price = P'`. Blockifier `check_fee_bounds` now checks `P >= P'`, which fails with `MaxGasPriceTooLow`.
7. The transaction is reverted; the fee for the validation phase is charged, but the execution fails. [7](#0-6) [8](#0-7) [9](#0-8)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }
