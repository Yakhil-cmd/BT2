### Title
Unbounded Loop in `move_to_block` Exhausts NEAR Gas on First Transaction After Idle Period - (File: engine-hashchain/src/hashchain.rs)

### Summary
The `Hashchain::move_to_block` function contains an unbounded `while` loop that iterates once per skipped NEAR block height, calling the `keccak` NEAR host function on every iteration. If the Aurora engine is idle for a sufficiently large number of NEAR blocks, the first subsequent transaction will exhaust the 300 Tgas NEAR gas limit, permanently bricking the engine and freezing all user funds.

### Finding Description

`move_to_block` in `engine-hashchain/src/hashchain.rs` iterates from the stored `current_block_height` to the current NEAR block height, computing a keccak hash on every step: [1](#0-0) 

Each loop body calls `compute_block_hashchain`, which invokes `keccak` — a NEAR host function — on a concatenated byte slice: [2](#0-1) 

This is triggered unconditionally on every transaction via `load_hashchain`: [3](#0-2) 

Which is called by both `with_hashchain` and `with_logs_hashchain`, the wrappers used for all contract method dispatch: [4](#0-3) [5](#0-4) 

There is no cap, checkpoint, or pagination on the number of iterations. The NEAR block time is approximately 1 second, and NEAR enforces a hard 300 Tgas gas limit per transaction. Each `keccak` host function call consumes a fixed amount of NEAR gas. Once the idle gap grows large enough that a single `move_to_block` call exceeds 300 Tgas, **every subsequent transaction fails**, because the gap only ever grows — it is never reset by a failed call (the hashchain state is only written on success via `save_hashchain`). [6](#0-5) 

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once the NEAR block gap between the last successful Aurora transaction and the current block height exceeds the threshold at which `move_to_block` exhausts 300 Tgas, no transaction can ever succeed again. The hashchain state is never advanced (because `save_hashchain` is only reached on success), so the gap monotonically increases. All ETH and ERC-20 tokens held in the Aurora engine become permanently inaccessible.

### Likelihood Explanation

NEAR produces approximately one block per second. The NEAR `keccak256` host function costs on the order of ~1–2 Tgas per call for a short input. At 300 Tgas per transaction, the threshold is on the order of a few hundred seconds of engine inactivity — a realistic scenario during low-traffic periods, planned maintenance, or a deliberate griefing campaign where an attacker simply waits for the engine to go quiet before submitting the first transaction to confirm the freeze. No special privileges are required; any unprivileged EVM user can submit the triggering transaction.

### Recommendation

1. **Cap the loop**: Limit `move_to_block` to a maximum number of iterations per call (e.g., 100), and persist the intermediate state so subsequent calls can continue from where the previous one left off.
2. **Lazy/amortized advancement**: Instead of catching up all skipped blocks in one call, advance the hashchain by at most N blocks per transaction and store the intermediate `current_block_height` after each partial advance.
3. **Separate catch-up method**: Provide a dedicated admin method to advance the hashchain in bounded increments, decoupled from the user transaction path.

### Proof of Concept

1. Deploy Aurora engine with hashchain enabled. Record `current_block_height = H`.
2. Allow the NEAR chain to produce `K` blocks without any Aurora transactions, where `K` is large enough that `K` keccak calls exceed 300 Tgas.
3. Submit any valid Aurora transaction (e.g., a simple ETH transfer).
4. `load_hashchain` reads the stored hashchain at height `H`, then calls `hashchain.move_to_block(H + K)`.
5. The `while` loop executes `K` iterations, each calling `keccak` via `compute_block_hashchain`.
6. The NEAR runtime aborts the transaction with an out-of-gas error before `save_hashchain` is reached.
7. The stored hashchain remains at height `H`. Every future transaction repeats steps 4–6 with an ever-growing `K`, permanently freezing the engine.

    pub fn move_to_block(
        &mut self,
        next_block_height: u64,
    ) -> Result<(), BlockchainHashchainError> {
        if next_block_height <= self.current_block_height {
            return Err(BlockchainHashchainError::BlockHeightIncorrect);
        }

        while self.current_block_height < next_block_height {
            self.previous_block_hashchain = self.block_hashchain_computer.compute_block_hashchain(
                &self.chain_id,
                self.contract_account_id.as_bytes(),
                self.current_block_height,
                self.previous_block_hashchain,
            );

            self.block_hashchain_computer.clear_txs();
            self.current_block_height += 1;
        }

        Ok(())
    }

pub fn with_hashchain<I, E, T, F>(
    mut io: I,
    env: &E,
    function_name: &str,
    f: F,
) -> Result<T, ContractError>
where
    I: IO + Copy,
    E: Env,
    F: for<'a> FnOnce(CachedIO<'a, I>) -> Result<T, ContractError>,
{
    let block_height = env.block_height();
    let maybe_hashchain = load_hashchain(&io, block_height)?;

    let cache = RefCell::new(IOCache::default());
    let hashchain_io = CachedIO::new(io, &cache);
    let result = f(hashchain_io)?;

    if let Some(mut hashchain) = maybe_hashchain {
        let cache_ref = cache.borrow();
        hashchain.add_block_tx(
            block_height,
            function_name,
            &cache_ref.input,
            &cache_ref.output,
            &Bloom::default(),
        )?;
        save_hashchain(&mut io, &hashchain)?;
    }

    Ok(result)
}

fn load_hashchain<I: IO>(io: &I, block_height: u64) -> Result<Option<Hashchain>, ContractError> {
    let mut maybe_hashchain = read_current_hashchain(io)?;
    if let Some(hashchain) = maybe_hashchain.as_mut()
        && block_height > hashchain.get_current_block_height()
    {
        hashchain.move_to_block(block_height)?;
    }
    Ok(maybe_hashchain)
}

pub fn save_hashchain<I: IO>(io: &mut I, hashchain: &Hashchain) -> Result<(), ContractError> {
    let key = storage::bytes_to_key(KeyPrefix::Hashchain, HASHCHAIN_STATE);
    let bytes = hashchain
        .try_serialize()
        .map_err(|_| BlockchainHashchainError::SerializationFailed)?;
    io.write_storage(&key, &bytes);
    Ok(())
}
