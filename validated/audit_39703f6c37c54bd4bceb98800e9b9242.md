### Title
Wrong `walletId` Used for All NFTs in Bulk Transfer and DID Assignment Operations — (File: `packages/api/src/wallets/NFT.ts`, `packages/gui/src/components/nfts/NFTTransferAction.tsx`, `packages/gui/src/components/nfts/NFTMoveToProfileDialog.tsx`)

---

### Summary

When a user selects multiple NFTs from different DID profiles (different `walletId` values) via the gallery's multi-select mode and initiates a bulk transfer or bulk DID assignment, the GUI always uses `nfts[0].walletId` for every NFT in the operation. This is a direct structural analog to H06: a function that accepts a target collection (each NFT with its own wallet identity) silently substitutes a fixed identity (the first NFT's wallet) for all targets. The backend receives an incorrect `wallet_id` for every NFT not belonging to the first selected NFT's wallet, causing those NFTs to be submitted under the wrong wallet context.

**Reachability:** The NFT gallery exposes a multi-select mode that allows selecting NFTs from any visible profile simultaneously. NFTs assigned to different DID profiles live in different NFT wallets (different `walletId` values), so a cross-profile selection is a normal user action. [5](#0-4) 

---

### Impact Explanation

For every NFT in the selection that does **not** belong to `nfts[0]`'s wallet, the backend `nft_transfer_bulk` / `nft_set_did_bulk` RPC receives a `wallet_id` that does not match the actual owning wallet of that NFT coin. The backend uses `wallet_id` to look up the signing key and authorize the spend. Sending the wrong `wallet_id` causes the backend to attempt to sign/spend those NFT coins under the wrong wallet context. Depending on backend validation strictness, this results in either:

- **Silent partial failure**: NFTs belonging to the wrong wallet are silently dropped or fail individually while the first wallet's NFTs succeed, leaving the user with a false impression that all NFTs were transferred/moved.
- **Wrong DID assignment**: In the `nft_set_did_bulk` path, the backend may associate the wrong DID profile with NFTs it cannot correctly authorize, corrupting the NFT's on-chain ownership record.

This matches the allowed High impact: *"Corruption… of RPC… state that causes a user to… send… the wrong asset, identity… or status."*

---

### Likelihood Explanation

The multi-select feature is prominently exposed in the NFT gallery UI. Any user who has NFTs spread across multiple DID profiles (a common configuration) and uses multi-select to bulk-transfer or bulk-move them will trigger this path. No attacker involvement is required — the user themselves initiates the action in good faith.

---

### Recommendation

In both `NFTTransferAction.tsx` and `NFTMoveToProfileDialog.tsx`, replace the single `walletId: nfts[0].walletId` argument with per-NFT wallet IDs. Since `NFT.ts`'s bulk methods accept a flat `walletId`, the callers should either:

1. Group the selected NFTs by `walletId` and issue one bulk call per group, or
2. Pass each NFT's own `walletId` into the `nftCoinList` entries directly (requires updating the `transferNft`/`setNftDid` API signatures to accept `Array<{ nftCoinId: string; walletId: number }>`).

The single-NFT paths (`nft_transfer_nft`, `nft_set_nft_did`) are not affected because they operate on exactly one NFT whose `walletId` is always correct.

---

### Proof of Concept

1. Create two DID profiles (Profile A → wallet ID 5, Profile B → wallet ID 7).
2. Mint at least one NFT under each profile.
3. Open the NFT gallery, enable multi-select, and select one NFT from Profile A and one from Profile B.
4. Click "Transfer NFT" and enter a destination address.
5. Observe the RPC call sent to the daemon: both entries in `nft_coin_list` carry `wallet_id: 5` (Profile A's wallet), even though the Profile B NFT belongs to wallet 7.
6. The backend receives `{ nft_coin_id: <B's coin>, wallet_id: 5 }` — an incorrect wallet context for Profile B's NFT — causing that NFT's spend to be attempted under the wrong wallet, resulting in a failed or incorrectly authorized spend for that coin. [6](#0-5) [7](#0-6)

  async transferNft(
    args: { walletId: number; nftCoinIds: string[]; targetAddress: string; fee: string } & AllowUnsyncedArg,
  ) {
    const { walletId, nftCoinIds, targetAddress, fee, allowUnsynced } = args;
    const extra = allowUnsynced != null ? { allowUnsynced } : {};
    if (nftCoinIds.length === 1) {
      return this.command<{
        walletId: number;
        spendBundle: SpendBundle;
      }>('nft_transfer_nft', {
        walletId,
        nftCoinId: nftCoinIds[0],
        targetAddress,
        fee,
        ...extra,
      });
    }
    return this.command<{
      walletId: number[];
      spendBundle: SpendBundle;
      txNum: number;
    }>('nft_transfer_bulk', {
      nftCoinList: nftCoinIds.map((nftId: string) => ({ nft_coin_id: nftId, wallet_id: walletId })),
      targetAddress,
      fee,
      ...extra,
    });
