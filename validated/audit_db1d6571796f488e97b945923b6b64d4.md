### Title
`unstake_action` Permanently Blocked by Zero-Value Token Transfer in Empty BTC Pool — (`src/staking/staking.cairo`)

---

### Summary

`transfer_to_pools_when_unstake` iterates over every pool registered to a staker and unconditionally calls `checked_transfer` for each, including pools whose delegated balance is zero. If the BTC token reverts on zero-value transfers (a documented "weird ERC20" behaviour), `unstake_action` panics on every call, permanently trapping the staker's STRK principal in the contract.

---

### Finding Description

`unstake_action` calls `transfer_to_pools_when_unstake` after erasing the staker's on-chain record:

```
unstake_action
  └─ remove_staker(...)          // staker_info set to None, own balance zeroed
  └─ checked_transfer(staker)    // STRK principal returned to staker address
  └─ transfer_to_pools_when_unstake(...)   // ← iterates ALL pools
       └─ for each (pool_contract, token_address):
            pool_dispatcher.set_staker_removed()
            insert_staker_delegated_balance(..., Zero)
            token_dispatcher.checked_transfer(pool_contract, pool_balance)  // ← panics if amount == 0 and token reverts
``` [1](#0-0) 

The inner loop does **not** guard against a zero `pool_balance` before calling `checked_transfer`. [2](#0-1) 

A zero balance arises in two realistic ways:

1. **Staker opens a BTC pool but no delegator ever joins.** `set_open_for_delegation` deploys a pool contract with no minimum delegation requirement. The pool is registered in `staker_pool_info.pools` and will be iterated forever.
2. **All delegators exit before the staker calls `unstake_action`.** After `unstake_intent` is called, delegators may still call `exit_delegation_pool_action`, which reduces the staking contract's recorded delegated balance to zero. By the time the exit-wait window expires and the staker calls `unstake_action`, the BTC pool balance is zero.

Because the entire `unstake_action` is one atomic transaction, a panic inside `transfer_to_pools_when_unstake` rolls back everything — the staker's record is restored, but the call reverts. The staker is now in an irrecoverable state: `unstake_intent` cannot be called again (`UNSTAKE_IN_PROGRESS` guard), and there is no `cancel_unstake_intent` function. Every subsequent call to `unstake_action` hits the same zero-value transfer and panics again. [3](#0-2) 

---

### Impact Explanation

The staker's STRK principal is permanently frozen inside the staking contract. They cannot complete `unstake_action`, cannot re-enter the protocol, and cannot recover their stake. This matches **High: Permanent freezing of unclaimed yield or unclaimed royalties; Temporary freezing of funds** (and arguably Critical for principal loss).

---

### Likelihood Explanation

- A staker opening a BTC pool with no delegators is a normal, expected operation (`set_open_for_delegation` is a standard staker action).
- Delegators exiting before a staker's exit window closes is also a normal operation.
- The BTC token on Starknet is an externally-bridged asset. Several bridged tokens (and the ERC-20 "weird token" catalogue) revert on zero-value transfers. The code-level root cause — the missing zero-balance guard — is in the Starknet Staking contract itself; the external token behaviour is a necessary co-condition, not the sole cause.

---

### Recommendation

Skip the transfer when the pool balance is zero:

```cairo
fn transfer_to_pools_when_unstake(...) {
    for (pool_contract, token_address) in staker_pool_info.pools {
        let pool_balance = self.get_delegated_balance(:staker_address, :pool_contract);
        let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };
        pool_dispatcher.set_staker_removed();
        self.insert_staker_delegated_balance(
            :staker_address, :pool_contract, delegated_balance: Zero::zero(),
        );
        if pool_balance.is_non_zero() {   // ← guard added
            let decimals = self.get_token_decimals(:token_address);
            let token_dispatcher = IERC20Dispatcher { contract_address: token_address };
            token_dispatcher.checked_transfer(
                recipient: pool_contract,
                amount: pool_balance.to_native_amount(:decimals).into(),
            );
        }
    }
}
```

---

### Proof of Concept

1. Staker stakes STRK and calls `set_open_for_delegation(btc_token_address)` — a BTC pool is created with zero delegated balance.
2. Staker calls `unstake_intent()` — exit timestamp is set.
3. Exit-wait window elapses.
4. Staker calls `unstake_action(staker_address)`.
5. Execution reaches `transfer_to_pools_when_unstake`; the BTC pool has `pool_balance == 0`.
6. `token_dispatcher.checked_transfer(pool_contract, 0)` is called on the BTC token.
7. If the BTC token reverts on zero-value transfers, the entire transaction panics.
8. The staker retries — same result every time. `unstake_intent` cannot be called again. Staker's STRK is permanently locked. [4](#0-3) [5](#0-4)

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }
