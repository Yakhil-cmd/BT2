### Title
Delegation Rewards Permanently Frozen in Pool Contract When Total Pool Balance Falls Below `min_delegation_for_rewards` — (File: src/pool/pool.cairo)

---

### Summary

When the total delegated balance of a pool falls below `min_delegation_for_rewards` (1 STRK for STRK pools), the pool contract's `compute_rewards_per_unit` silently returns zero, leaving the `cumulative_rewards_trace` unchanged. However, the staking contract **still calculates and transfers STRK reward tokens to the pool contract**. Those tokens accumulate in the pool contract with no accounting entry, making them permanently unclaimable by any pool member.

---

### Finding Description

**Root cause — `compute_rewards_per_unit` silently drops rewards:** [1](#0-0) 

When `total_stake < min_delegation_for_rewards` (10¹⁸ for STRK), the function returns `Zero::zero()`. The `cumulative_rewards_trace` is then updated with `last + 0 = last` — no new rewards are recorded.

**The staking contract still forwards STRK tokens to the pool contract regardless:** [2](#0-1) 

The developer comment on line 964–966 explicitly acknowledges this:

> *"Delegation rewards lost when pool balance is less than `min_delegation_for_rewards`. The staking contract continues to forward `pool_rewards` to the pool contract even in this case."* [3](#0-2) 

**The staking contract calculates non-zero pool rewards even for dust pools:**

In `calculate_staker_pools_rewards`, pool rewards are proportional to `pool_balance_curr_epoch / total_stake`. As long as `pool_balance_curr_epoch > 0` and `total_stake > 0`, `pool_rewards` is non-zero and appended to `pool_rewards_array`. [4](#0-3) 

These rewards are then claimed from the reward supplier and transferred to the pool contract: [5](#0-4) 

**No recovery path exists.** The pool contract has no function to sweep or return stuck STRK tokens. `claim_rewards` reads only from `cumulative_rewards_trace` (which was never updated), and `exit_delegation_pool_action` returns only the delegated principal. [6](#0-5) 

**No minimum delegation is enforced at entry:**

`enter_delegation_pool` only checks `amount.is_non_zero()`, so any amount from 1 wei upward is accepted, making it trivial to create or leave a dust pool. [7](#0-6) 

---

### Impact Explanation

STRK reward tokens forwarded to a pool contract whose total balance is below `min_delegation_for_rewards` are **permanently frozen** — they sit in the pool contract's ERC-20 balance but are never credited to any pool member's claimable rewards. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

The amount frozen per epoch is proportional to `pool_balance / total_stake × epoch_rewards`. Over many epochs with a dust pool, the cumulative frozen amount grows unboundedly.

---

### Likelihood Explanation

The scenario is reachable by any unprivileged delegator:

1. **Direct dust entry**: A delegator calls `enter_delegation_pool` with any amount below `min_delegation_for_rewards` (< 1 STRK). No minimum is enforced.
2. **Partial exit leaving dust**: A delegator calls `exit_delegation_pool_intent` with an amount that reduces the total pool balance below `min_delegation_for_rewards`. The remaining member(s) permanently lose future rewards.
3. **Griefing**: An attacker deliberately creates a dust pool to freeze rewards for an existing pool member.

The `min_delegation_for_rewards` threshold of 1 STRK (`10^18`) is low enough that legitimate partial exits can cross it accidentally.

---

### Recommendation

1. **Preferred fix**: In `_update_rewards` / `update_pool_rewards`, skip the `send_rewards_to_delegation_pool` transfer (and the `claim_from_reward_supplier` debit) when `pool_balance < min_delegation_for_rewards`. Return those tokens to the reward supplier or simply do not claim them.
2. **Alternative**: Enforce a minimum delegation amount in `enter_delegation_pool` and `add_to_delegation_pool`, and prevent `exit_delegation_pool_intent` from leaving a residual balance below `min_delegation_for_rewards` (analogous to the `MIN_NET_COLL` check recommended in the reference report).

---

### Proof of Concept

```
1. Staker stakes and opens a STRK delegation pool.
2. Delegator A calls enter_delegation_pool(amount = 5e17)  // 0.5 STRK < min_delegation_for_rewards (1e18)
3. Staker attests; _update_rewards is triggered.
4. calculate_staker_pools_rewards computes pool_rewards = 5e17 / total_stake * epoch_rewards  (non-zero).
5. claim_from_reward_supplier pulls pool_rewards STRK from the reward supplier into the staking contract.
6. update_pool_rewards calls:
     a. pool.update_rewards_from_staking_contract(rewards=pool_rewards, pool_balance=5e17)
        → compute_rewards_per_unit returns 0 (5e17 < 1e18)
        → cumulative_rewards_trace unchanged
     b. send_rewards_to_delegation_pool transfers pool_rewards STRK to the pool contract address.
7. Delegator A calls claim_rewards → receives 0 STRK.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        /// Compute the rewards for the pool trace.
        ///
        /// `staking_rewards` is in `STRK_DECIMALS` decimals.
        /// `total_stake` is in the contract's token decimals.
        /// **Note**: Delegation rewards lost when pool balance is less than
        /// `min_delegation_for_rewards`. The staking contract continues to forward
        /// `pool_rewards` to the pool contract even in this case.
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
