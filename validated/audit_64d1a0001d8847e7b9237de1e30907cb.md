### Title
`native_per_pubdata` Truncates to Zero via Floor Division, Allowing Pubdata to Be Published Without Native Resource Charge — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

`native_per_pubdata` is computed with truncating integer division (`wrapping_div`), while the analogous `native_per_gas` uses ceiling division (`div_ceil`). When `pubdata_price < native_price`, `native_per_pubdata` silently rounds to zero. Every downstream pubdata-charging call then multiplies by zero, so any amount of pubdata written during a transaction costs zero native resources. The user is undercharged, the operator/protocol absorbs the proving cost, and the `delta_gas` correction in the refund path is also suppressed.

---

### Finding Description

In `validation_impl.rs` the two rates are computed back-to-back:

```rust
// line 135 — ceiling division
u256_try_to_u64(&gas_price.div_ceil(native_price))

// line 142 — truncating division
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
``` [1](#0-0) 

The same asymmetry is present in the API helper:

```rust
// native_per_gas = ceil(gas_price / native_price)
let native_per_gas = u256_try_to_u64(&gas_price.div_ceil(native_price)).ok_or(())?;

// native_per_pubdata = pubdata_price / native_price   ← floor, no ceil
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [2](#0-1) 

When `0 < pubdata_price < native_price`, `wrapping_div` returns `0`. This zero propagates through every pubdata-charging site:

**1. Intrinsic pubdata overhead in `create_resources_for_tx`:**
```rust
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// = 0 * intrinsic_pubdata = 0  → no native deducted at tx start
``` [3](#0-2) 

**2. Post-execution pubdata charge in `get_resources_to_charge_for_pubdata`:**
```rust
let native = current_pubdata_spent.checked_mul(native_per_pubdata)
// = pubdata_bytes * 0 = 0  → no native charged for any storage writes
``` [4](#0-3) 

**3. `delta_gas` correction in `compute_gas_refund`:**
```rust
let delta_gas = if native_per_gas == 0 { 0 }
else { (native_used / native_per_gas) as i64 - (gas_used as i64) };
```
Because `native_used` is artificially low (pubdata was never charged), `delta_gas` is also suppressed, so the user receives a larger gas refund than they should. [5](#0-4) 

---

### Impact Explanation

The native resource models the off-chain proving cost, dominated by publishing data.

> "The native resource models the offchain cost of processing a transaction. Currently, this is dominated by proving and publishing data." [6](#0-5) 

When `native_per_pubdata = 0`, every byte of pubdata written to storage is free in native resources. A user can write the maximum number of storage slots allowed by the block's pubdata limit without paying the corresponding native resource cost. The operator/protocol must absorb the proving cost for that pubdata. Additionally, the `delta_gas` mechanism that would otherwise force the user to pay extra gas to cover native consumption is also zeroed out, so the user receives a full gas refund on the pubdata-related native work.

---

### Likelihood Explanation

`pubdata_price` and `native_price` are independent operator-set oracle values. The condition `pubdata_price < native_price` is a realistic production configuration — for example, `pubdata_price = 5 wei/byte` and `native_price = 10 wei/cycle` gives `native_per_pubdata = 0` instead of the correct `1`. Any unprivileged user submitting a storage-writing transaction during such a block triggers the bug with no special setup. The user does not need to know the prices in advance; the undercharge happens automatically.

---

### Recommendation

Replace `wrapping_div` with `div_ceil` for `native_per_pubdata`, consistent with how `native_per_gas` is computed:

```rust
// Before (truncates to 0 when pubdata_price < native_price):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (rounds up, matching native_per_gas):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix in `api/src/helpers.rs` line 427. [7](#0-6) [8](#0-7)

### Proof of Concept

**Setup:** Operator sets `pubdata_price = 5`, `native_price = 10` (both valid, non-zero, `pubdata_price < native_price`).

**Computation:**
```
native_per_pubdata = 5.wrapping_div(10) = 0   ← truncates, should be 1 with div_ceil
```

**Transaction:** User submits a ZK transaction that writes to 20 storage slots (generating ~640 bytes of pubdata).

**Expected charge:**
```
native_for_pubdata = 640 * 1 = 640 native units
```

**Actual charge:**
```
native_for_pubdata = 640 * 0 = 0 native units
```

**Result:** The user pays zero native resources for 640 bytes of pubdata. The `delta_gas` correction is also suppressed because `native_used` is lower than it should be, so the user receives a larger gas refund. The operator bears the full proving cost of the pubdata without compensation. [9](#0-8) [4](#0-3) [10](#0-9)
