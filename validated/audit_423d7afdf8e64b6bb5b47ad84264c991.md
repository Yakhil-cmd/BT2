### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Valid Transactions to Be Rejected or Invalid Transactions to Be Admitted — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **current committed block's** `strk_gas_prices.l2_gas_price`, but every admitted transaction executes in the **next** block whose L2 gas price is the EIP-1559-adjusted `next_l2_gas_price`. The correct value is stored in `BlockHeaderWithoutHash::next_l2_gas_price` but is never propagated to the admission check. A developer TODO in the source code explicitly acknowledges the error.

### Finding Description

In `validate_resource_bounds`, the gateway reads the threshold price as:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

This value is the **current** block's L2 gas price. The transaction will be sequenced into the **next** block, whose price is computed by `calculate_next_base_gas_price` (EIP-1559) and stored as `BlockHeaderWithoutHash::next_l2_gas_price`: [2](#0-1) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` reads from `block.block_header_without_hash` but constructs a `BlockInfo` that contains only `l2_gas_price`, silently discarding `next_l2_gas_price`: [3](#0-2) 

The EIP-1559 adjustment is computed by `calculate_next_base_gas_price`, which can raise or lower the price each block based on gas utilization: [4](#0-3) 

The threshold check then compares the transaction's `max_price_per_unit` against the stale price: [5](#0-4) 

The blockifier's `check_fee_bounds` at execution time uses the **actual** next-block price, creating a permanent mismatch between the gateway admission price and the execution price: [6](#0-5) 

### Impact Explanation

Two distinct failure modes arise:

**Mode 1 — Valid transactions rejected (High):** When the L2 gas price is falling (low block utilization), `next_l2_gas_price < l2_gas_price`. A transaction with `max_price_per_unit` in the range `[next_l2_gas_price, l2_gas_price)` satisfies the execution constraint but fails the gateway threshold. The transaction is permanently rejected at the gateway even though it would succeed at execution. No guard exists to recover these transactions.

**Mode 2 — Invalid transactions admitted:** When the L2 gas price is rising (high block utilization), `next_l2_gas_price > l2_gas_price`. A transaction with `max_price_per_unit` in the range `[l2_gas_price, next_l2_gas_price)` passes the gateway threshold but fails `check_fee_bounds` at execution with `MaxGasPriceTooLow`. The blockifier rejects it, but the transaction has already consumed mempool resources and caused user confusion.

Both modes match the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The EIP-1559 adjustment is bounded per block by `gas_price_max_change_denominator`, so the per-block gap between `l2_gas_price` and `next_l2_gas_price` is small. However:

- During sustained high utilization the price rises monotonically, and the cumulative gap between the gateway's stale reference and the actual execution price grows with each block.
- Any user who calibrates `max_price_per_unit` to the current block's price (the natural thing to do) will be affected when the price is falling.
- The developer TODO comment confirms this is a known, unresolved production defect.

### Recommendation

1. Expose `next_l2_gas_price` through `GatewayFixedBlockStateReader` (either by adding it to `BlockInfo` or as a separate method).
2. In `validate_resource_bounds`, replace `previous_block_l2_gas_price` with the `next_l2_gas_price` read from `BlockHeaderWithoutHash`.
3. Update `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` to propagate `block_header.next_l2_gas_price`.

### Proof of Concept

**Scenario (valid transaction rejected):**

1. Current committed block N has `l2_gas_price = 1000 fri`, low utilization.
2. EIP-1559 computes `next_l2_gas_price = 900 fri` for block N+1.
3. User submits a V3 invoke with `l2_gas.max_price_per_unit = 950 fri`.
4. Gateway threshold (with `min_gas_price_percentage = 100`) = `1000 fri`.
5. `950 < 1000` → gateway rejects with `GAS_PRICE_TOO_LOW`.
6. At execution in block N+1, `950 >= 900` → transaction would have succeeded.
7. The user's valid transaction is permanently lost.

**Scenario (invalid transaction admitted):**

1. Current committed block N has `l2_gas_price = 1000 fri`, high utilization.
2. EIP-1559 computes `next_l2_gas_price = 1100 fri` for block N+1.
3. User submits a V3 invoke with `l2_gas.max_price_per_unit = 1050 fri`.
4. Gateway threshold = `1000 fri`; `1050 >= 1000` → gateway admits.
5. At execution in block N+1, `check_fee_bounds` finds `1050 < 1100` → `MaxGasPriceTooLow`, transaction reverts/fails.
6. Mempool slot consumed; user pays for a failed transaction.

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
