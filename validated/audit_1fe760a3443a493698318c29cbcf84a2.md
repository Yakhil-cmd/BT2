### Title
Commission Rate Updated Without Settling Pending Epoch Rewards, Allowing Staker to Retroactively Increase Commission on Delegators - (File: `src/staking/staking.cairo`)

---

### Summary

A staker with an active commission commitment can increase their commission rate at any point within an epoch. Because the commission is read from storage at reward-distribution time (attestation / `update_rewards`), not at reward-earning time, the new higher rate is applied retroactively to the entire epoch's rewards, stealing yield from delegators who joined under the lower rate.

---

### Finding Description

**Root cause — commission overwrite with no pending-reward settlement**

`set_commission` in `staking.cairo` calls `update_commission`, which directly overwrites the stored commission with no requirement to first distribute rewards that accrued under the old rate: [1](#0-0) 

When an active `CommissionCommitment` exists, the only constraints are `commission <= max_commission` and `commission != old_commission`. There is no lower-bound check, so the staker is free to **increase** commission up to `max_commission`: [2](#0-1) 

**Root cause — commission consumed at distribution time**

`calculate_staker_pools_rewards` reads the commission from storage at the moment rewards are distributed, not at the moment they were earned: [3](#0-2) 

The freshly-read `commission` is immediately passed to `split_rewards_with_commission`, which determines how much of the epoch's pool rewards the staker keeps versus what is forwarded to the pool contract. There is no snapshot of the commission that was in effect during the epoch.

**Entry path**

In the pre-consensus (attestation) phase the staker controls the exact moment they call `attest`, which triggers `update_rewards_from_attestation_contract` → `_update_rewards` → `calculate_staker_pools_rewards`. The staker can therefore:

1. Call `set_commission_commitment(max_commission: 5000, …)` — commits to ≤ 50 %.
2. Call `set_commission(commission: 100)` — advertises 1 % to attract delegators.
3. Wait for delegators to stake and for the epoch to progress.
4. Call `set_commission(commission: 5000)` — raises to 50 % (allowed by active commitment).
5. Immediately call `attest` — rewards for the entire epoch are split using 50 %.

Delegators who staked expecting 1 % commission receive rewards calculated at 50 % commission for that epoch.

In the consensus phase `update_rewards` is called by the sequencer, but the staker can still front-run it with a commission increase in the same block or earlier in the same epoch. [4](#0-3) 

---

### Impact Explanation

Delegators receive materially fewer STRK rewards than they earned under the commission rate that was in effect when they staked. The difference flows entirely to the staker as extra commission. This is a direct theft of unclaimed yield from delegators — **High impact**.

---

### Likelihood Explanation

The staker must have previously called `set_commission_commitment` with a `max_commission` above the current rate. This is a one-time setup step that any staker can perform. After that, the attack is repeatable every epoch with no further privileged access. The staker is the only party who needs to act, and the action (calling `set_commission` then `attest`) is entirely within their normal operational rights. **Medium-High likelihood.**

---

### Recommendation

Apply commission changes only from the **next epoch** onward. Record the epoch at which each commission change takes effect alongside the rate, and use the historically correct rate when computing rewards for a given epoch. Alternatively, require that any pending epoch rewards be settled (i.e., `update_rewards_from_attestation_contract` / `update_rewards` called) before `set_commission` is allowed to increase the rate.

---

### Proof of Concept

```
Epoch N begins.
  commission = 100  (1%)
  Delegator stakes 1 000 000 STRK.

Mid-epoch:
  Staker calls set_commission(commission: 5000)  // 50%, allowed by active commitment

End of epoch N — staker attests:
  calculate_staker_pools_rewards reads commission = 5000
  pool_rewards_including_commission = 10 000 STRK (example)
  commission_rewards = 10 000 * 5000 / 10000 = 5 000 STRK  → staker
  pool_rewards       = 5 000 STRK                           → delegators

Expected (at 1%):
  commission_rewards = 100 STRK  → staker
  pool_rewards       = 9 900 STRK → delegators

Delegator loss: 4 900 STRK per epoch, stolen by the staker.
``` [5](#0-4) [6](#0-5)

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }

                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()

                    rewards_including_commission: pool_rewards_including_commission, :commission,

                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)
