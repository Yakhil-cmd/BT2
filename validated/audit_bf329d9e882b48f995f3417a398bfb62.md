### Title
Missing Zero Address Validation in `change_reward_address` Allows Permanent Freezing of Unclaimed Yield - (File: `src/staking/staking.cairo`, `src/pool/pool.cairo`)

---

### Summary
Both `staking.cairo` and `pool.cairo` expose a `change_reward_address` function that accepts any `ContractAddress` without validating it is non-zero. A staker or pool member can set their reward address to `0x0`, causing all subsequent `claim_rewards` calls to silently transfer accrued yield to the zero address, permanently destroying it.

---

### Finding Description
`staking.cairo::change_reward_address` (line 517) validates only that the new address is not a registered token address, but performs no zero-address check:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    self.general_prerequisites();
    assert!(
        !self.does_token_exist(token_address: reward_address),
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    // No assert!(!reward_address.is_zero(), ...) here
    staker_info.reward_address = reward_address;
    ...
}
``` [1](#0-0) 

`pool.cairo::change_reward_address` (line 505) has the same gap — it only checks the address is not the pool token:

```cairo
fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
    assert!(
        self.token_dispatcher.contract_address.read() != reward_address,
        "{}",
        GenericError::REWARD_ADDRESS_IS_TOKEN,
    );
    // No zero-address check
    pool_member_info.reward_address = reward_address;
    ...
}
``` [2](#0-1) 

When `claim_rewards` is subsequently called, the internal `send_rewards_to_staker` helper reads the stored `reward_address` and calls `checked_transfer` directly to it:

```cairo
token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
``` [3](#0-2) 

On Starknet, address `0x0` is not a deployed contract; an ERC-20 transfer to it succeeds at the token level (balance is credited to a slot no one controls) but the tokens are permanently inaccessible.

The same missing check exists in the `stake` constructor path for stakers: [4](#0-3) 

---

### Impact Explanation
Any staker or pool member who calls `change_reward_address(0)` — accidentally or through a front-end bug — will have all future and pending unclaimed STRK rewards silently sent to address `0x0` on every subsequent `claim_rewards` invocation. The tokens are permanently unrecoverable. This matches the allowed impact: **Permanent freezing of unclaimed yield (High)**.

---

### Likelihood Explanation
`change_reward_address` is a routine self-service function callable by any active staker or pool member with no privileged role required. A fat-finger error, a buggy front-end that passes an uninitialized address, or a scripted migration that omits the address field are all realistic triggers. The function is also callable at any time after staking, so the window of exposure is the entire staker lifetime.

---

### Recommendation
Add an explicit non-zero assertion at the top of both `change_reward_address` implementations, and in the `stake` / `enter_delegation_pool` entry points:

```cairo
assert!(!reward_address.is_zero(), "Reward address cannot be zero");
```

Apply the same guard to `stake` in `staking.cairo` and `enter_delegation_pool` in `pool.cairo` where `reward_address` is first stored.

---

### Proof of Concept
1. Alice is an active staker with accrued rewards.
2. Alice calls `staking.change_reward_address(reward_address: 0)`. The call passes all existing checks (zero is not a registered token address).
3. Alice's `staker_info.reward_address` is now `0x0`.
4. The protocol distributes rewards; `staker_info.unclaimed_rewards_own` grows.
5. Anyone calls `staking.claim_rewards(staker_address: alice)`. The access-control check passes because `caller == staker_address`.
6. `send_rewards_to_staker` executes `checked_transfer(recipient: 0x0, amount: rewards)`. The ERC-20 transfer succeeds; tokens are credited to address `0x0` and are permanently lost.
7. Alice's `unclaimed_rewards_own` is zeroed; the yield is gone with no recovery path.

        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                !self.does_token_exist(token_address: reward_address),
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let old_address = staker_info.reward_address;

            // Update reward_address and commit to storage.
            staker_info.reward_address = reward_address;
            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.
            self
                .emit(
                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,
                    },
                );
        }
