### Title
Zero-Gas Aurora-Native Precompiles Enable Cheap Bridge-Spam and Temporary Fund Freeze - (File: `engine-precompiles/src/native.rs`)

---

### Summary

Multiple Aurora-native precompiles have their EVM gas cost hardcoded to `EthGas::new(0)` with an unresolved `TODO(#483)`. The most impactful are `ExitToNear` and `ExitToEthereum`, which trigger real NEAR cross-contract calls (bridge operations). Because the EVM charges zero gas for each invocation, an attacker can call these precompiles hundreds of thousands of times within a single EVM transaction at negligible cost, spamming the NEAR bridge and causing temporary freezing of legitimate bridge operations.

---

The same zero-gas pattern affects three additional Aurora-native precompiles:

- `PrepaidGas`: `PREPAID_GAS_COST = EthGas::new(0)` [4](#0-3) 
- `RandomSeed`: `RANDOM_BYTES_GAS = EthGas::new(0)` [5](#0-4) 
- `PredecessorAccount` / `CurrentAccount`: `PREDECESSOR_ACCOUNT_GAS = EthGas::new(0)` / `CURRENT_ACCOUNT_GAS = EthGas::new(0)` [6](#0-5) 

The critical distinction is that `ExitToNear` and `ExitToEthereum` are not read-only queries — each successful invocation emits a NEAR promise log that the Aurora engine processes as a real NEAR cross-contract call, consuming NEAR gas from the Aurora contract's prepaid gas budget and enqueuing bridge operations. [7](#0-6) 

---

### Impact Explanation

**High — Temporary freezing of funds.**

An attacker deploys a loop contract on Aurora that calls `ExitToNear` (address `0xe9217bc70b7ed1f598ddd3199e80b093fa71124f`) repeatedly, each time bridging a dust amount (e.g., 1 wei of ETH). Because the precompile costs 0 EVM gas, the only per-iteration cost is the EVM `CALL` opcode overhead (~100 gas warm, ~2600 gas cold). With a 30 M gas transaction limit, the attacker can trigger approximately 11,500–300,000 bridge promise emissions in a single transaction.

Each emitted promise is processed by `filter_promises_from_logs` and dispatched as a real NEAR cross-contract call. [8](#0-7) 

This saturates the NEAR cross-contract call queue for the Aurora contract, exhausts the NEAR gas budget allocated to the transaction, and delays or fails legitimate bridge withdrawals queued behind the spam. Users whose funds are in transit via the bridge experience a temporary freeze until the spam backlog clears.

---

### Likelihood Explanation

**High.** The attack requires only:
1. A small amount of ETH on Aurora (dust amounts per call).
2. A deployed loop contract — trivial to write in Solidity.
3. No special permissions, no admin access, no oracle dependency.

The zero-gas constants are present in the production codebase with an open TODO, meaning the misconfiguration is live and reachable by any EVM user today.

---

### Recommendation

Benchmark each Aurora-native precompile and assign a non-zero `required_gas` value proportional to the actual NEAR-side computational cost. For `ExitToNear` and `ExitToEthereum`, the gas cost should reflect the NEAR gas consumed by the resulting cross-contract call (analogous to how `CrossContractCall` already does this with `CROSS_CONTRACT_CALL_BASE = EthGas::new(343_650)` and a per-byte component). [9](#0-8)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IExitToNear {
    // flag=0x00 (ETH exit), recipient = "attacker.near"
}

contract BridgeSpammer {
    address constant EXIT_TO_NEAR = 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f;

    // Call with msg.value = N * 1 wei
    function spam(uint256 iterations) external payable {
        bytes memory input = abi.encodePacked(
            uint8(0x00),          // flag: ETH base token exit
            bytes("attacker.near") // recipient account id
        );
        for (uint256 i = 0; i < iterations; i++) {
            // Each call costs 0 EVM gas (EXIT_TO_NEAR_GAS = 0)
            // but emits a real NEAR cross-contract call promise
            (bool ok,) = EXIT_TO_NEAR.call{value: 1}(input);
            require(ok);
        }
    }
}
```

With `iterations = 10_000` and `msg.value = 10_000 wei`, the attacker pays only ~1 M EVM gas (for CALL overhead) and triggers 10,000 NEAR bridge cross-contract calls, saturating the bridge queue at negligible cost. The root cause is `EXIT_TO_NEAR_GAS = EthGas::new(0)` at `engine-precompiles/src/native.rs:46`. [1](#0-0)
