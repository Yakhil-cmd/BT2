### Title
Gateway Minimum Gas Price Check Covers Only `l2_gas.max_price_per_unit`, Allowing Transactions With Zero L1/L1-Data Gas Prices to Bypass Admission Filters - (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The stateless and stateful gateway validators enforce a minimum gas price exclusively on `l2_gas.max_price_per_unit`. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields of `AllResourceBounds` are never checked against any floor. A transaction carrying zero prices for L1 gas and L1 data gas passes every gateway admission check, enters the mempool, and then unconditionally fails at blockifier pre-validation because the block's L1 gas prices are always non-zero (`NonzeroGasPrice`). The developers have already noted the gap with an explicit TODO comment in the stateful validator.

### Finding Description

**Stateless validator** (`validate_resource_bounds`, lines 56–88):

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { … });
}
```

Only `l2_gas.max_price_per_unit` is compared against `config.min_gas_price` (default 8 Gwei). `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never tested. [1](#0-0) 

The preceding `ZeroResourceBounds` guard only rejects a transaction when the **total** `max_possible_fee` is zero. A transaction with `l2_gas.max_price_per_unit = 8 Gwei`, `l2_gas.max_amount = 1`, and both L1 prices at zero produces a non-zero total fee and passes that guard. [2](#0-1) 

**Stateful validator** (`validate_resource_bounds`, lines 223–243):

```rust
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
```

Again, only the L2 gas price is compared against the previous block's L2 gas price threshold. The function is named `validate_tx_l2_gas_price_within_threshold` and carries an explicit developer TODO:

**Blockifier pre-validation** (`check_fee_bounds`, lines 374–475):

When the transaction is later executed, `perform_pre_validation_stage` calls `check_fee_bounds`, which iterates over all three resources and enforces:

```rust

```

Because the block's L1 gas price is always a `NonzeroGasPrice`, a transaction with `l1_gas.max_price_per_unit = 0` **always** fails here. [4](#0-3)

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

Any unprivileged user can craft a V3 (`AllResourceBounds`) transaction with:
- `l2_gas.max_price_per_unit ≥ min_gas_price` (passes stateless check)
- `l2_gas.max_amount ≥ 1` (makes `enforce_fee = true`, so the blockifier will actually run the fee check)
- `l1_gas.max_price_per_unit = 0`
- `l1_data_gas.max_price_per_unit = 0`

Such a transaction clears all gateway admission filters and occupies a mempool slot. When the batcher dequeues it, `check_fee_bounds` immediately rejects it with `MaxGasPriceTooLow` for L1 gas before any execution occurs. No fee is charged and the nonce is not advanced (the transactional state is discarded on pre-validation failure). The attacker can repeat this at scale to exhaust mempool capacity and batcher processing budget with zero cost.

### Likelihood Explanation

Trivially reachable by any external caller. No privileged role, special account state, or race condition is required. The crafted transaction is a well-formed, correctly signed V3 transaction; the only unusual property is `l1_gas.max_price_per_unit = 0`, which no gateway check currently rejects.

### Recommendation

Extend the minimum gas price check in `StatelessTransactionValidator::validate_resource_bounds` to cover all three resource price fields:

```rust
for (price, label) in [
    (resource_bounds.l2_gas.max_price_per_unit.0, "l2_gas"),
    (resource_bounds.l1_gas.max_price_per_unit.0, "l1_gas"),
    (resource_bounds.l1_data_gas.max_price_per_unit.0, "l1_data_gas"),
] {
    if price < self.config.min_gas_price {
        return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
            gas_price: GasPrice(price),
            min_gas_price: self.config.min_gas_price,
        });
    }
}
```

Similarly, resolve the TODO in `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` to also validate `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's corresponding prices. [6](#0-5) 

### Proof of Concept

1. Construct a valid V3 invoke transaction with `AllResourceBounds`:
   - `l2_gas = { max_amount: 1_000_000, max_price_per_unit: 8_000_000_000 }` — passes `min_gas_price` check
   - `l1_gas = { max_amount: 0, max_price_per_unit: 0 }` — **not checked by gateway**
   - `l1_data_gas = { max_amount: 0, max_price_per_unit: 0 }` — **not checked by gateway**

2. Submit via the HTTP gateway. The stateless validator computes `max_possible_fee = 1_000_000 * 8_000_000_000 > 0`, passes the `ZeroResourceBounds` guard, and passes the `min_gas_price` guard (only L2 is checked). The stateful validator's `validate_tx_l2_gas_price_within_threshold` also passes. The transaction enters the mempool.

3. When the batcher executes the transaction, `perform_pre_validation_stage` → `check_fee_bounds` evaluates `l1_gas.max_price_per_unit (0) < block.l1_gas_price (NonzeroGasPrice > 0)` and returns `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }`. The transaction is discarded with no fee charged and no nonce increment.

4. Repeat step 1–2 with fresh nonces to flood the mempool indefinitely at zero cost. [7](#0-6) [8](#0-7)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
