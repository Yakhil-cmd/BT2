### Title
`binTotalShares` uint256 Overflow in `addLiquidity` Unchecked Block Enables Theft of New LP Funds â€” (`metric-core/contracts/libraries/LiquidityLib.sol`)

---

### Summary

`addLiquidity` runs entirely inside an `unchecked` block. The line `binTotalShares[binIdx] = binTotalSharesVal + sharesToAdd` (line 120) can silently wrap to zero when a bin has been fully consumed by swaps (zero token balance). An attacker can exploit this to corrupt `binTotalShares` to zero at no cost, then steal all tokens deposited by the next LP who adds liquidity to that bin.

---

### Finding Description

`_binTotalShares` is a `mapping(int256 => uint256)` â€” a full 256-bit counter with no cap. [1](#0-0) 

The entire `addLiquidity` body is wrapped in `unchecked`: [2](#0-1) 

Inside that block, the token-amount calculation for a non-empty bin is:

```solidity
amount0Scaled = Math.ceilDiv(_checkedMul(binState.token0BalanceScaled, sharesToAdd), binTotalSharesVal);
amount1Scaled = Math.ceilDiv(_checkedMul(binState.token1BalanceScaled, sharesToAdd), binTotalSharesVal);
``` [3](#0-2) 

`_checkedMul` is a plain function body **outside** any `unchecked` block, so Solidity 0.8 overflow protection applies there: [4](#0-3) 

**The bypass:** when a bin has been fully consumed by swaps, `token0BalanceScaled == 0` and `token1BalanceScaled == 0`. Then `_checkedMul(0, sharesToAdd) = 0` regardless of how large `sharesToAdd` is â€” no overflow, no revert, zero tokens required. The callback is skipped entirely: [5](#0-4) 

But the unchecked accumulation still executes:

```solidity
binTotalShares[binIdx] = binTotalSharesVal + sharesToAdd;   // line 120 â€” overflows silently
positionBinShares[posKey] = newUserShares;                  // line 121 â€” attacker's huge share count
``` [6](#0-5) 

With `sharesToAdd = type(uint256).max - binTotalSharesVal + 1`, the sum wraps to exactly `0`. `binTotalShares[bin]` is now `0` while the attacker holds `positionBinShares â‰ˆ type(uint256).max`.

**Downstream theft:** The next LP who adds liquidity to this bin sees `binTotalSharesVal == 0` and takes the "empty bin" initialisation path, paying real tokens: [7](#0-6) 

After the new LP deposits, `binTotalShares = newLP_shares`. The attacker calls `removeLiquidity` with `sharesToRemove = newLP_shares`. The only guard is `userShares >= sharesToRemove` (line 196), which passes because the attacker's `positionBinShares â‰ˆ type(uint256).max >> newLP_shares`: [8](#0-7) 

The attacker receives `token0BalanceScaled * newLP_shares / newLP_shares = token0BalanceScaled` â€” the entire bin balance. `binTotalShares` drops to `0` again. The new LP's subsequent `removeLiquidity` hits division by zero at line 205 and reverts permanently: [9](#0-8) 

---

### Impact Explanation

Direct, complete theft of any LP's deposited principal in a bin that was previously fully consumed by swaps. The new LP's funds are unrecoverable (division-by-zero DoS on their `removeLiquidity`). This satisfies the Critical/High direct-loss-of-user-principal threshold.

---

### Likelihood Explanation

Bins being fully consumed by swaps is routine AMM operation in a bin-based model. The attack requires no privileged role, no special token, and no front-running â€” the attacker can set up the corrupted state in advance and wait for any LP to deposit. The only precondition is a bin with zero token balance and non-zero `binTotalShares`, which is a normal post-swap state.

---

### Recommendation

Remove the outer `unchecked` wrapper from `addLiquidity`, or at minimum protect the two accumulations that must not overflow:

```solidity
// Replace unchecked additions:
binTotalShares[binIdx] = binTotalSharesVal + sharesToAdd;   // must be checked
positionBinShares[posKey] = newUserShares;                  // newUserShares already computed above
```

A `SafeCast` or explicit `require(binTotalSharesVal + sharesToAdd >= binTotalSharesVal)` guard before line 120 would close the overflow. Similarly, `newUserShares = userShares + sharesToAdd` at line 76 should be checked (it is currently inside `unchecked`). [10](#0-9)
