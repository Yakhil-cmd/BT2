The implementation has a concrete, exploitable vulnerability. Here is the analysis:

---

### Title
Missing Caller Access Control in `update_rewards` Allows Any Address to Permanently Suppress Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

The spec for `update_rewards` declares access control as **"Only starkware sequencer"**, but the production implementation contains **no caller check**. Any unprivileged address can call `update_rewards(any_valid_staker, disable_rewards=true)` first in a block, which atomically marks the block as processed and skips reward distribution, permanently discarding that block's rewards for all stakers.

---

### Finding Description

`update_rewards` is implemented as:

```
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks is_paused
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}", Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // global, not per-staker

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits without distributing
    }
    ...
}
``` [1](#0-0) 

Two properties combine to create the vulnerability:

1. **`last_reward_block` is a global storage slot** (not keyed by `staker_address`). Once any call succeeds for block N, every subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

2. **`disable_rewards=true` writes `last_reward_block` and returns immediately** without distributing any rewards. [3](#0-2) 

The spec explicitly states access should be restricted:

> **access control**: Only starkware sequencer. [4](#0-3) 

But `general_prerequisites()` only enforces the pause flag — there is no `get_caller_address()` check, no role assertion, and no sequencer validation anywhere in the function body. [5](#0-4) 

The `staker_total_strk_balance.is_non_zero()` check at line 1482 is the only meaningful precondition — the attacker simply needs to supply any valid, active staker address with non-zero STRK balance as the `staker_address` argument. [6](#0-5) 

---

### Impact Explanation

An attacker who front-runs every block's `update_rewards` call with `disable_rewards=true` permanently discards that block's consensus rewards. Because `last_reward_block` is global, no legitimate call can succeed in the same block. Rewards are not deferred — they are simply never computed or credited. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators.

---

### Likelihood Explanation

The call requires no funds, no privileged role, and no special setup beyond knowing any active staker address (which is public on-chain). The attacker only needs to submit a transaction before the sequencer's own `update_rewards` call in each block. On Starknet, transaction ordering within a block is controlled by the sequencer, which limits practical exploitability — however, the vulnerability is architecturally present and exploitable if the sequencer is not the exclusive caller, which the code currently does not enforce.

---

### Recommendation

Add a caller check at the top of `update_rewards` that asserts `get_caller_address()` equals the authorized sequencer address (stored in contract storage), consistent with the spec's stated access control.

---

### Proof of Concept

1. Deploy the staking contract with an active staker (non-zero STRK balance, past the K-epoch activation window).
2. Enable consensus rewards via `set_consensus_rewards_first_epoch`.
3. From any arbitrary address, call `update_rewards(staker_address, disable_rewards=true)`.
4. Observe: `last_reward_block` is written to the current block; no rewards are distributed.
5. Attempt a second call (from the legitimate sequencer) in the same block: it reverts with `REWARDS_ALREADY_UPDATED`.
6. Advance one block and repeat from step 3.
7. After N blocks, verify `staker_info.unclaimed_rewards_own` remains zero despite N rewardable blocks having elapsed.

This is directly confirmed by the existing test at line 2882–2885 of `src/flow_test/test.cairo`, which demonstrates that calling with `disable_rewards=true` yields zero rewards and blocks any retry in the same block. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
