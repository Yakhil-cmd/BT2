### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Grief Staker Block Rewards — (`src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no caller restriction. It accepts a `disable_rewards: bool` parameter and unconditionally writes to the global `last_reward_block` storage slot regardless of that flag. Any unprivileged address can call `update_rewards(any_staker, disable_rewards: true)` to consume the per-block update slot without distributing rewards, permanently blocking the legitimate consensus caller from distributing rewards for that block.

---

### Finding Description

`update_rewards` is exposed on the public `IStakingRewardsManager` interface with no `assert_caller_is_*` guard: [1](#0-0) 

The very first state-mutating action after the basic sanity checks is: [2](#0-1) 

`last_reward_block` is a **single global** storage slot shared across all stakers: [3](#0-2) 

Once it is written in block N, the guard at line 1454–1458 causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

When `disable_rewards` is `true`, the function returns immediately after writing `last_reward_block`, distributing nothing: [5](#0-4) 

The analog to the NFT report is direct: just as `_isApprovedOrOwner` was overridden so that only `NftHandler` could trigger a transfer (breaking the standard approval path), here the absence of a caller guard means an **unauthorized** path (any EOA or contract) can consume the one-per-block update slot with `disable_rewards=true`, permanently blocking the **authorized** consensus path for that block.

---

### Impact Explanation

**Medium — Griefing with no profit motive but damage to users or protocol.**

For every block in which an attacker front-runs the legitimate consensus caller with `update_rewards(any_active_staker, disable_rewards: true)`:

- `last_reward_block` is set to the current block number.
- The legitimate consensus call reverts with `REWARDS_ALREADY_UPDATED`.
- **No staker receives block rewards for that block** — the reward epoch accumulation is silently skipped.

Sustained griefing (one cheap transaction per block) can suppress consensus reward distribution indefinitely, causing permanent loss of unclaimed yield for all stakers and delegators.

---

### Likelihood Explanation

High. The entry point is a public, permissionless function. The attacker only needs to submit a transaction with a valid (active, migrated) `staker_address` and `disable_rewards: true`. No special role, key, or capital is required. The cost is a single L2 transaction per block.

---

### Recommendation

Add a caller restriction to `update_rewards` so that only the designated consensus contract (analogous to how `update_rewards_from_attestation_contract` restricts to the attestation contract) may invoke it:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.assert_caller_is_consensus_contract(); // <-- add this
    self.general_prerequisites();
    ...
}
```

Store the consensus contract address in storage (similar to `attestation_contract`) and expose a governance-gated setter. Alternatively, remove the `disable_rewards` parameter entirely and let the consensus contract simply not call `update_rewards` for blocks where rewards should be withheld. [6](#0-5) 

---

### Proof of Concept

1. Staker Alice is active and migrated to V3. Consensus rewards have started (`is_pre_consensus()` returns `false`).
2. In block N, the legitimate consensus contract is about to call `update_rewards(alice, disable_rewards: false)` to credit Alice's epoch rewards.
3. Attacker Eve submits `update_rewards(alice, disable_rewards: true)` with a higher gas price, front-running the consensus call.
4. Eve's transaction succeeds: `last_reward_block` is written to N, but no rewards are distributed (early return at line 1487–1489).
5. The consensus contract's call reverts: `current_block_number (N) > last_reward_block (N)` is false → `REWARDS_ALREADY_UPDATED`.
6. Alice receives zero block rewards for block N. Eve repeats this every block at negligible cost. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
