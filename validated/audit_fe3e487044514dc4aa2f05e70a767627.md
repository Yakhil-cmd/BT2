### Title
Gateway L2 Gas Price Threshold Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Admitting Transactions That Fail at Execution - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The gateway's stateful resource-bounds check and its embedded blockifier validation both read `block_header.l2_gas_price` (the price that was used in the **previous** block) as the reference for the admission threshold. The batcher, however, builds the next block using `block_header.next_l2_gas_price` (the EIP-1559-adjusted price for the **upcoming** block). When the network is above the gas target, `next_l2_gas_price > l2_gas_price`, so the gateway admits transactions whose `max_price_per_unit` satisfies the stale threshold but is below the actual execution-block price. Those transactions pass every gateway check, enter the mempool, and then fail `check_fee_bounds` inside the blockifier at batcher execution time.

### Finding Description

**Gateway admission path** — `validate_resource_bounds` in `StatefulTransactionValidator`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;          // ← stale: the price of the already-committed block
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The `GatewayFixedBlockSyncStateClient` constructs `BlockInfo` by mapping `block_header.l2_gas_price` into `gas_prices.strk_gas_prices.l2_gas_price` and **never reads `next_l2_gas_price`**:

```rust
strk_gas_prices: GasPriceVector {
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
    // next_l2_gas_price is present in the header but is silently dropped
},
``` [2](#0-1) 

The same stale `BlockInfo` is forwarded into `run_validate_entry_point`, so the embedded blockifier `check_fee_bounds` also uses the wrong price:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
``` [3](#0-2) 

**Batcher/consensus execution path** — the consensus context stores `self.l2_gas_price` which is set from `next_l2_gas_price` of the synced block:

```rust
self.l2_gas_price = max(
    sync_block.block_header_without_hash.next_l2_gas_price,
    VersionedConstants::latest_constants().min_gas_price,
);
``` [4](#0-3) 

This `self.l2_gas_price` becomes the `l2_gas_price` of the block being built. The blockifier's `check_fee_bounds` then enforces:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    // MaxGasPriceTooLow error
}
``` [5](#0-4) 

where `actual_gas_price` is `next_l2_gas_price` from the previous block — a value the gateway never checked against.

The `StorageBlockHeader` stores both fields and `next_l2_gas_price` is already available:

```rust
pub l2_gas_price: GasPricePerToken,
pub next_l2_gas_price: GasPrice,
``` [6](#0-5) 

The TODO comment in the gateway code is a developer acknowledgment of the exact bug: `// TODO(Arni): getnext_l2_gas_price from the block header.` [7](#0-6) 

### Impact Explanation

The gateway makes an incorrect admission decision: it accepts transactions that will deterministically fail at batcher execution with `MaxGasPriceTooLow`. This matches the **High** impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

Concretely:
- Transactions that should be rejected at the gateway door are admitted into the mempool.
- They consume mempool slots and batcher processing cycles.
- They are silently dropped at execution, causing user-visible failures with no gateway-level error.
- The discrepancy is systematic whenever the network is above the gas target (rising price regime), which is the normal operating condition under load.

### Likelihood Explanation

The EIP-1559 fee market (`calculate_next_base_gas_price`) adjusts the price every block based on gas usage vs. the target. Under any sustained load above the gas target, `next_l2_gas_price > l2_gas_price` every block. The default `min_gas_price_percentage` is 100, so the gateway threshold equals exactly `l2_gas_price`. Any user who sets `max_price_per_unit` to the current block's price (a natural choice when following the gateway's own threshold) will have their transaction admitted but fail at execution. No privileged access is required; any unprivileged user submitting a V3 (`AllResources`) transaction triggers this path. [8](#0-7) 

### Recommendation

In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, replace the `l2_gas_price` field in the returned `BlockInfo` with `block_header.next_l2_gas_price`:

```rust
strk_gas_prices: GasPriceVector {
    l2_gas_price: block_header.next_l2_gas_price.try_into()?,
    // ...
},
```

This ensures both `validate_tx_l2_gas_price_within_threshold` and the embedded blockifier `check_fee_bounds` in `run_validate_entry_point` use the same price that the batcher will use when executing the transaction. [9](#0-8) 

### Proof of Concept

1. Previous committed block N has `l2_gas_price = P` (in FRI) and `next_l2_gas_price = P' = P * 1.001` (block was above gas target by a small margin — normal under load).
2. User submits an `InvokeV3` transaction with `l2_gas.max_price_per_unit = P` (exactly at the gateway's 100% threshold).
3. **Gateway stateless check** (`validate_resource_bounds`): `P >= min_gas_price` (static config floor) → passes.
4. **Gateway stateful check** (`validate_tx_l2_gas_price_within_threshold`): threshold = `100% * P = P`; `P >= P` → passes.
5. **Gateway blockifier check** (`run_validate_entry_point` → `check_fee_bounds`): block context uses `l2_gas_price = P`; `P >= P` → passes.
6. Transaction is admitted to the mempool.
7. Batcher builds block N+1 with `l2_gas_price = P'` (from `next_l2_gas_price` of block N).
8. **Batcher blockifier check** (`perform_pre_validation_stage` → `check_fee_bounds`): `P >= P'` → **FAILS** with `MaxGasPriceTooLow`.
9. Transaction is rejected at execution; user receives no fee refund and no useful error from the gateway. [10](#0-9) [11](#0-10)

```rust
    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
