### Title
Gateway Admits Transactions With Insufficient L1/L1-Data Gas Prices That Blockifier Will Reject — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`, `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

Both the stateless and stateful gateway validators only enforce a minimum price threshold for **L2 gas**. They never check `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` against any threshold. A transaction carrying `AllResourceBounds` with L1 gas price = 0 and L1 data gas price = 0 passes every gateway check and enters the mempool, but is deterministically rejected by `AccountTransaction::check_fee_bounds` at blockifier pre-validation time because that function checks all three resources against the actual block gas prices.

---

### Finding Description

**Stateless path** — `StatelessTransactionValidator::validate_resource_bounds` checks only `l2_gas.max_price_per_unit` against the static `min_gas_price` config value:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { … });
}
```

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never tested here. [1](#0-0) 

**Stateful path** — `validate_tx_l2_gas_price_within_threshold` (the only price check in the stateful validator) reads the previous block's L2 gas price and compares it against the transaction's L2 gas price. The function carries an explicit TODO acknowledging the gap:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(…) {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            …
            if tx_l2_gas_price.0 < threshold { return Err(…); }
        }
        ValidResourceBounds::L1Gas(_) => { /* No validation required */ }
    }
    Ok(())
}
```

L1 gas price and L1 data gas price are never fetched from the block header and never compared. [2](#0-1) 

**Blockifier enforcement** — `AccountTransaction::check_fee_bounds` iterates over **all three** resources for `AllResourceBounds` transactions and rejects any whose `max_price_per_unit` is below the block's actual gas price:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) => {
    let GasPriceVector { l1_gas_price, l1_data_gas_price, l2_gas_price } = …;
    vec![
        (L1Gas,     l1_gas,     …, *l1_gas_price),
        (L1DataGas, l1_data_gas, …, *l1_data_gas_price),
        (L2Gas,     l2_gas,     …, *l2_gas_price),
    ]
}
…
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { … });
}
``` [3](#0-2) 

The result is a split invariant: the gateway enforces L2 gas price but not L1/L1-data gas prices; the blockifier enforces all three. Any transaction that satisfies the gateway's L2-only check but sets L1 or L1-data gas price below the block's actual price is admitted to the mempool and then deterministically fails at `perform_pre_validation_stage`. [4](#0-3) 

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An unprivileged user can submit an `AllResourceBounds` V3 transaction with:
- `l2_gas.max_price_per_unit` ≥ threshold (passes both gateway checks)
- `l1_gas.max_price_per_unit = 0`
- `l1_data_gas.max_price_per_unit = 0`

The transaction passes `StatelessTransactionValidator::validate`, passes `StatefulTransactionValidator::validate_resource_bounds`, enters the mempool, and is only rejected when the batcher hands it to the blockifier. The sequencer wastes stateful validation resources (nonce lookup, blockifier `__validate__` entry-point execution) on transactions that can never be included in a block. Because the check is purely stateless from the attacker's perspective (no on-chain cost), this can be repeated at high volume to degrade sequencer throughput.

---

### Likelihood Explanation

Any user who can submit a V3 transaction to the gateway can trigger this. No privileged access, no special account state, and no dependency on external contracts is required. The attacker only needs to set `l1_gas.max_price_per_unit = 0` in an otherwise valid transaction.

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or replace it with a new `validate_all_gas_prices_within_threshold`) to fetch `l1_gas_price` and `l1_data_gas_price` from the previous block header alongside `l2_gas_price`, and apply the same percentage-threshold check to all three resources for `AllResourceBounds` transactions. Mirror the same fix in the stateless validator's `validate_resource_bounds` by checking `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against `min_gas_price` in addition to the existing L2 check.

---

### Proof of Concept

1. Obtain the current block's L2 gas price `P_l2` from the gateway.
2. Craft a V3 Invoke transaction with:
   ```
   resource_bounds = AllResourceBounds {
       l2_gas:     { max_price_per_unit: P_l2,  max_amount: 1_000_000 },
       l1_gas:     { max_price_per_unit: 0,     max_amount: 1_000_000 },
       l1_data_gas:{ max_price_per_unit: 0,     max_amount: 1_000_000 },
   }
   ```
3. Submit to the gateway. `StatelessTransactionValidator::validate_resource_bounds` passes (L2 price ≥ `min_gas_price`). `StatefulTransactionValidator::validate_resource_bounds` passes (L2 price ≥ threshold; L1/L1-data prices are never read).
4. Transaction enters the mempool.
5. When the batcher picks it up, `AccountTransaction::check_fee_bounds` fires `ResourceBoundsError::MaxGasPriceTooLow` for `L1Gas` and `L1DataGas` and the transaction is rejected at `perform_pre_validation_stage`, consuming sequencer CPU without producing a block inclusion.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });

                min_gas_price: self.config.min_gas_price,
