### Title
Missing Access Control on `update_rewards` Allows Any Caller to Suppress Block Rewards for All Stakers — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is documented in the spec as restricted to "Only starkware sequencer," but the implementation enforces no such check. Any unprivileged address can call it with `disable_rewards: true`, consuming the single per-block reward slot (`last_reward_block`) without distributing any rewards. Because `last_reward_block` is a **global** storage variable, one such call per block silently denies rewards to every staker and every delegation pool for that block.

---

### Finding Description

The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 is:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker validity checks ...

    // ← last_reward_block is written unconditionally, before the disable_rewards branch
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;   // ← exits without distributing anything
    }
    // ... actual reward distribution ...
}
``` [1](#0-0) 

`general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero — there is no sequencer-only guard. [2](#0-1) 

`last_reward_block` is a single global slot shared across all stakers: [3](#0-2) 

The spec explicitly requires sequencer-only access: [4](#0-3) 

---

### Impact Explanation

**Medium — Griefing with no profit motive but damage to users or protocol.**

An attacker who calls `update_rewards(staker_address: <any_valid_staker>, disable_rewards: true)` in block N:

1. Writes `last_reward_block = N`.
2. Returns immediately without updating any staker's `unclaimed_rewards_own` or transferring pool rewards.
3. Any subsequent call to `update_rewards` in block N — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`.

Because the slot is global, **all stakers and all delegation pools** lose their block reward for block N. Repeated across every block, this permanently suppresses all consensus-era yield generation. The attacker gains nothing; the damage is entirely borne by stakers and delegators.

---

### Likelihood Explanation

- The function is publicly callable by any non-zero address with no role check.
- The only cost to the attacker is gas per block.
- On Starknet the sequencer controls ordering, but the attacker's transaction can be submitted in any block where the sequencer has not yet included its own `update_rewards` call; the attacker does not need to front-run in the traditional sense — they only need to land their call in the same block before the sequencer's call, or in any block the sequencer skips.
- The `disable_rewards: true` path is a legitimate code path (used during pre-consensus or for stakers with zero balance), so the call will not revert on any valid active staker.

---

### Recommendation

Add a sequencer-only access control guard at the top of `update_rewards`, consistent with the spec. For example, store the authorized sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()`. Alternatively, use the existing `RolesComponent` to define a `SEQUENCER_ROLE` and gate the function with `self.roles.only_sequencer()`.

---

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
    → last_reward_block written to N
    → returns early, zero rewards distributed

  Sequencer tx: update_rewards(staker_address=<staker>, disable_rewards=false)
    → assert!(current_block_number > last_reward_block)  // N > N  → FALSE
    → PANICS with REWARDS_ALREADY_UPDATED

Block N+1: repeat.
```

All stakers' `unclaimed_rewards_own` fields remain unchanged; all pool contracts receive zero STRK; the reward supplier's `unclaimed_rewards` is never incremented. The entire consensus-rewards distribution mechanism is silently neutralised at the cost of one gas-paying transaction per block. [5](#0-4) [6](#0-5)

            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.

            // becomes effective.
