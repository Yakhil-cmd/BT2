### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary
The factory owner can set `maxAdminSpreadFeeE6` to limit pool-admin-controlled spread fees. `setPoolAdminFees` enforces this cap, but `setPoolBinAdditionalFees` applies no cap at all, allowing the pool admin to charge per-bin additional fees up to `uint16.max = 65 535` (≈ 6.55 % in E6 units) regardless of the factory-owner-imposed ceiling.

### Finding Description
`setFeeCaps` lets the factory owner lower `maxAdminSpreadFeeE6` below the hard maximum of `200 000` (20 %) to protect traders from excessive admin fees. [4](#0-3) 

`setPoolAdminFees` correctly enforces this cap: [5](#0-4) 

`setPoolBinAdditionalFees` does **not**: [2](#0-1) 

The pool-level `setBinAdditionalFees` simply writes the caller-supplied values with no validation beyond the bin-index range check: [6](#0-5) 

During every swap, `addFeeBuyE6` / `addFeeSellE6` are added directly to `baseFeeX64`, making them economically equivalent to an additional spread fee: [7](#0-6) 

### Impact Explanation
A pool admin can set `addFeeBuyE6 = addFeeSellE6 = 65 535` on every bin, imposing a ≈ 6.55 % per-swap surcharge on traders regardless of what `maxAdminSpreadFeeE6` the factory owner has configured. For a $1 M swap this is a $65 535 overcharge per trade — a direct, repeatable loss of trader principal that bypasses the factory owner's intended protection ceiling.

### Likelihood Explanation
The pool admin is semi-trusted (not fully trusted). Any pool whose admin key is compromised, or any pool created by a malicious deployer who sets themselves as admin, can immediately exploit this path with a single transaction and no timelock. The trigger is a valid, permissioned call — no external oracle manipulation or non-standard token is required.

### Recommendation
Add a cap check inside `setPoolBinAdditionalFees` analogous to the one in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
+   if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
+   if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce that the sum of the base admin spread fee and the bin additional fee does not exceed `maxAdminSpreadFeeE6`.

### Proof of Concept
1. Factory owner calls `setFeeCaps(…, newMaxAdminSpreadFeeE6 = 500, …)` — intending to cap admin fees at 0.05 %.
2. Pool admin calls `setPoolAdminFees(pool, 500, 0)` — correctly capped at 0.05 %.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65_535, 65_535)` — no revert; bin 0 now carries a ≈ 6.55 % additional fee.
4. Every swap routed through bin 0 pays ≈ 6.55 % + 0.05 % = ≈ 6.60 % total spread fee, 131× the factory-owner-intended ceiling.
