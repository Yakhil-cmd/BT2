### Title
Gateway Stateful Validator Uses Stale Previous-Block L2 Gas Price for Admission Threshold, Admitting Transactions That Will Fail Execution - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary
`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **previous block's** L2 gas price, but every admitted transaction executes in the **next block**, whose L2 gas price is computed by EIP-1559 mechanics and can be materially higher. Transactions whose `l2_gas.max_price_per_unit` falls between the two prices pass the gateway filter yet fail blockifier pre-validation at execution time, polluting the mempool with transactions the sequencer will never successfully include.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs`, `validate_resource_bounds` reads the L2 gas price from the **latest committed block**:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`validate_tx_l2_gas_price_within_threshold` then computes the admission threshold as `min_gas_price_percentage% × previous_block_l2_gas_price`:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
``` [2](#0-1) 

The TODO comment explicitly acknowledges the correct value is the **next** block's L2 gas price, which is stored in `BlockHeaderWithoutHash::next_l2_gas_price` and computed by `calculate_next_base_gas_price` via EIP-1559:

```rust
let adjusted_price_u256 =
    if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };
``` [3](#0-2) 

When the network is congested (`gas_used > gas_target`), the next block's price `P'` exceeds the previous block's price `P`. A transaction with `l2_gas.max_price_per_unit = P` satisfies the gateway threshold (`P ≥ min_gas_price_percentage% × P`) but fails blockifier pre-validation at execution time because `check_fee_bounds` compares against the actual block's gas price `P'`:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { ... },
    );
}
``` [4](#0-3) 

Note that `run_validate_entry_point` in the gateway also constructs its `BlockContext` from the previous block's `block_info` (only incrementing `block_number`), so the gateway's own blockifier simulation also uses the stale price and does not catch the mismatch:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
let block_context = BlockContext::new(block_info, ...);
``` [5](#0-4) 

The batcher, however, builds the actual block with the EIP-1559-derived next price, so the discrepancy surfaces only at real execution time.

### Impact Explanation

The gateway's admission invariant — "only admit transactions whose L2 gas price is sufficient for the block they will be included in" — is broken. Any transaction with `l2_gas.max_price_per_unit` in the range `[min_gas_price_percentage% × P, P')` passes the gateway filter and enters the mempool, but is rejected by `check_fee_bounds` during blockifier pre-validation when the batcher attempts to include it. The sequencer wastes resources processing these transactions, and the mempool accumulates stale entries that can never be sequenced. This matches **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing**.

### Likelihood Explanation

The condition triggers whenever the network is congested enough that `gas_used > gas_target` for the current block, causing EIP-1559 to raise the next block's price. With the default `min_gas_price_percentage = 100`, the threshold equals exactly `previous_block_l2_gas_price`, so any transaction priced at exactly the previous block's price is admitted but will fail if the price rises even by 1 unit. During sustained congestion the price can increase by up to `price / gas_price_max_change_denominator` per block, widening the vulnerable band proportionally. [6](#0-5) 

### Recommendation

Replace the read of `gas_prices.strk_gas_prices.l2_gas_price` (the current block's price) with the `next_l2_gas_price` field stored in `BlockHeaderWithoutHash`, which is already populated by the sync path:

```rust
// Before (stale):
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// After (correct):
let next_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_header()   // expose next_l2_gas_price from BlockHeaderWithoutHash
    .await?
    .next_l2_gas_price;
```

Expose `next_l2_gas_price` through `GatewayFixedBlockStateReader` and use it as the reference price in `validate_tx_l2_gas_price_within_threshold`. The same correction should be applied to the `BlockContext` constructed inside `run_validate_entry_point` so that the gateway's own blockifier simulation also uses the correct price.

### Proof of Concept

1. Current block: `gas_used > gas_target` → EIP-1559 computes `next_l2_gas_price = P' = P + P/D` where `D = gas_price_max_change_denominator`.
2. User submits an invoke transaction with `l2_gas.max_price_per_unit = P` (exactly the previous block's price).
3. **Stateless validation**: passes (no price check here).
4. **Stateful `validate_resource_bounds`**: threshold = `100% × P = P`; `P ≥ P` → **PASS** (stale price used).
5. **Stateful blockifier simulation** (`run_validate_entry_point`): block context built from previous block's `block_info` with `block_number + 1` but same gas prices → `check_fee_bounds` sees `P ≥ P` → **PASS**.
6. Transaction is forwarded to the mempool and accepted.
7. **Batcher execution**: block context uses `next_l2_gas_price = P'`; `check_fee_bounds` sees `P < P'` → `MaxGasPriceTooLow` → transaction fails pre-validation and is never included. [7](#0-6) [8](#0-7)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
