### Title
Pool Contract Lacks Pause Mechanism While Staking Contract Is Fully Pauseable - (File: src/pool/pool.cairo)

### Summary

The `Pool` contract (`src/pool/pool.cairo`) has no pause mechanism at all, while the `Staking` contract (`src/staking/staking.cairo`) implements a comprehensive `is_paused` guard on every state-changing function. When the security agent triggers an emergency pause on the staking contract, the pool contract's user-facing functions — most critically `claim_rewards` — remain fully callable and can drain STRK tokens from the pool contract's balance without any recourse.

### Finding Description

The `Staking` contract stores an `is_paused` flag and enforces it across all state-mutating functions. The pause tests in `src/staking/tests/pause_test.cairo` confirm that every staking function reverts with `"Contract is paused"` when the flag is set. [1](#0-0) 

The `Pool` contract, by contrast, contains no `is_paused` storage variable, no `pause`/`unpause` entry points, and no pause check in any of its functions. A search across all files under `src/pool/` returns zero matches for `is_paused`, `pause`, or any equivalent guard. [2](#0-1) 

The critical function is `claim_rewards`. It calculates accrued rewards from the pool's internal `cumulative_rewards_trace` and then transfers STRK tokens **directly from the pool contract's own balance** to the reward address, with no call into the staking contract: [3](#0-2) 

Because this transfer path bypasses the staking contract entirely, pausing the staking contract does **not** block `claim_rewards` on the pool. The same applies to `exit_delegation_pool_intent` when the staker has already been removed (`staker_removed == true`), which also skips the staking contract call: [4](#0-3) 

By contrast, `enter_delegation_pool`, `add_to_delegation_pool`, `exit_delegation_pool_action`, and `switch_delegation_pool` all call back into the staking contract and would therefore be indirectly blocked by the staking pause. Only `claim_rewards` and the inactive-staker path of `exit_delegation_pool_intent` are fully independent. [5](#0-4) [6](#0-5) 

### Impact Explanation

**High — Theft of unclaimed yield / Temporary freezing of funds.**

During an emergency (e.g., a reward-calculation exploit or a bridge compromise), the security agent can pause the staking contract to halt all staking-side fund flows. However, any STRK tokens already sitting in pool contracts as accrued rewards remain fully withdrawable via `claim_rewards`. If the emergency involves inflated or incorrectly computed rewards, pool members can drain those tokens before the team can react, because there is no mechanism to pause the pool contract independently.

### Likelihood Explanation

The staking contract's pause mechanism exists precisely for emergencies. The pool contract is a first-class user-facing contract that holds real STRK balances. The asymmetry is reachable by any pool member calling `claim_rewards` at any time, with no preconditions beyond having accrued rewards. The likelihood is **medium**: it requires an emergency scenario, but the entry path is trivially accessible to any unprivileged pool member.

### Recommendation

Add an `IPoolPause` interface and a corresponding `is_paused` storage variable to the `Pool` contract, mirroring the pattern in `StakingPauseImpl`. Add a pause check at the top of `claim_rewards`, `exit_delegation_pool_intent`, `enter_delegation_pool`, and `add_to_delegation_pool`. The pause/unpause functions should be callable by the same `security_agent` / `security_admin` roles used in the staking contract. Alternatively, the pool contract could query the staking contract's `is_paused()` view function at the start of each state-changing call. [7](#0-6) 

### Proof of Concept

1. Security agent detects an exploit and calls `staking.pause()` — all staking functions now revert with `"Contract is paused"`.
2. Pool contract holds 10,000 STRK in accrued rewards for its members.
3. Any pool member calls `pool.claim_rewards(pool_member)` — the function reads the internal `cumulative_rewards_trace`, computes the reward amount, and calls `STRK_TOKEN.transfer(reward_address, rewards)` directly.
4. The call succeeds; STRK tokens leave the pool contract with no way for the protocol to prevent it.
5. The staking contract's pause is ineffective for the pool's reward balance. [8](#0-7) [9](#0-8)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }
