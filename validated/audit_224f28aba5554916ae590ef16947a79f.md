### Title
Malicious Participant Can Corrupt CKD Output via Inconsistent Public Key in Signature Share Computation - (File: src/confidential_key_derivation/protocol.rs)

### Summary
In the Confidential Key Derivation (CKD) protocol, every participant independently computes a signature share using `hash_app_id_with_pk(&key_pair.public_key, app_id)`. The correctness of the aggregated CKD output depends on all participants using the **same** aggregate public key (`key_pair.public_key`) so that the coordinator's summation yields `msk · H(pk ‖ app_id)`. However, no participant ever broadcasts or cross-validates their `public_key`, and the coordinator blindly sums received shares without checking consistency. A malicious participant can supply a `KeygenOutput` carrying a different `public_key`, silently corrupting the CKD output accepted by honest parties.

---

### Finding Description

**Root cause — `compute_signature_share` (lines 168–174):** [1](#0-0) 

```rust
// H(pk || app_id) when H is a random oracle
let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

// S <- x . H(app_id)
let big_s = hash_point * private_share.to_scalar();
```

`key_pair.public_key` is the aggregate verifying key produced by DKG — the same value for every honest participant. The hash `H(pk ‖ app_id)` is therefore supposed to be identical across all parties. The function accepts this value from the caller without any cross-participant validation.

**Aggregation without consistency check — `do_ckd_coordinator` (lines 50–55):** [2](#0-1) 

```rust
for (_, participant_output) in
    recv_from_others::<CKDOutput>(&chan, waitpoint, &participants, me).await?
{
    norm_big_y += participant_output.big_y();
    norm_big_c += participant_output.big_c();
}
```

The coordinator sums `big_y` and `big_c` from every participant with no check that each share was computed against the correct aggregate public key. There is no mechanism — no broadcast, no commitment, no proof — that binds a participant's share to a specific `public_key`.

**The `KeygenOutput` struct carries both fields independently:** [3](#0-2) 

```rust
Ok(KeygenOutput {
    private_share: SigningShare::new(my_signing_share),
    public_key: verifying_key,
})
```

After DKG, every participant holds the same `verifying_key` (the aggregate). Nothing in the CKD entry point `ckd()` or in `compute_signature_share` enforces that the `public_key` field of the caller-supplied `KeygenOutput` matches what other participants are using.

**Entry point — `ckd()` (lines 66–117):** [4](#0-3) 

The function validates participant membership and coordinator presence but performs no check on `key_pair.public_key`.

### Impact Explanation

The expected CKD output satisfies:

```
big_c = Σ λᵢ · (xᵢ · H(pk_correct ‖ app_id) + yᵢ · A)
      = msk · H(pk_correct ‖ app_id) + (Σ λᵢ · yᵢ) · A
```

If participant `i` uses `pk_wrong`, their contribution becomes `λᵢ · (xᵢ · H(pk_wrong ‖ app_id) + yᵢ · A)`. The coordinator's sum is then:

```
(msk − λᵢ·xᵢ) · H(pk_correct ‖ app_id) + λᵢ·xᵢ · H(pk_wrong ‖ app_id) + (Σ λⱼ·yⱼ) · A
```

The `unmask` call (`big_c − a · big_y`) no longer yields `msk · H(pk_correct ‖ app_id)`. The derived confidential key is silently wrong. The coordinator returns this corrupted `CKDOutput` as `Some(ckd_output)` with no error, so honest parties accept an unusable cryptographic output.

**Mapped impact:** *High — Corruption of CKD outputs so honest parties accept inconsistent or unusable cryptographic outputs.*

---

### Likelihood Explanation

Any participant in the CKD session is a reachable attacker. They control the `KeygenOutput` they pass to `ckd()`. Choosing `pk_wrong` requires no cryptographic capability — it is a single field substitution. The attack is completely silent: the coordinator emits no error, and the application receives a plausible-looking `CKDOutput` struct. The malicious participant's identity is not exposed because no participant ever proves knowledge of the correct public key during the CKD round.

---

### Recommendation

1. **Broadcast and cross-validate `public_key` during the CKD round.** Each participant should include their `key_pair.public_key` in their message to the coordinator. The coordinator should assert that all received public keys are identical before summing shares.

2. **Alternatively**, the coordinator can broadcast the expected aggregate public key at the start of the protocol, and each participant must abort if it does not match their local `key_pair.public_key`.

3. **Bind the share to the public key via a commitment or proof.** Each participant could send a commitment `H(pk ‖ norm_big_c)` in a prior round, forcing them to commit to a specific `pk` before revealing the share.

---

### Proof of Concept

```
Setup (honest DKG):
  pk_correct = msk · G₂   (aggregate public key, known to all)
  xᵢ         = participant i's private share
  app_id     = b"target_app"

Attack:
  Malicious participant i calls ckd(..., key_pair = KeygenOutput {
      public_key:   pk_wrong,   // ≠ pk_correct
      private_share: xᵢ,        // legitimate share
  }, ...)

  compute_signature_share computes:
      hash_point = H(pk_wrong ‖ app_id)   // wrong hash
      big_s      = hash_point * xᵢ
      big_c_i    = big_s + yᵢ · A

  Coordinator receives big_c_i from malicious i and big_c_j (correct) from all j≠i.
  Sums them unconditionally → corrupted CKDOutput.

  Application calls ckd_output.unmask(app_sk):
      result ≠ msk · H(pk_correct ‖ app_id)
      → derived confidential key is wrong, no error raised.
``` [5](#0-4) [6](#0-5)

pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    let comms = Comms::new();
    let chan = comms.shared_channel();

    let fut = run_ckd_protocol(
        chan,
        coordinator,
        me,
        participants,
        key_pair,
        app_id.into(),
        app_pk,
        rng,
    );
    Ok(make_protocol(comms, fut))
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}

pub fn hash_app_id_with_pk(pk: &VerifyingKey, app_id: &[u8]) -> ElementG1 {
    let compressed_pk = pk.to_element().to_compressed();
    let input = [compressed_pk.as_slice(), app_id].concat();
    ciphersuite::hash_to_curve(&input)
}
