### Title
Any Caller Can Permanently Suppress Consensus Block Rewards for All Stakers via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function is callable by any address and accepts a `disable_rewards` boolean. The function writes the global `last_reward_block` state **before** checking `disable_rewards`, meaning an attacker can consume the per-block reward slot for the entire protocol without distributing any rewards. Because `last_reward_block` is a single global value, one such call per block permanently denies all stakers their consensus rewards for that block.

### Finding Description
In `update_rewards` (lines 1449–1507 of `src/staking/staking.cairo`), the execution order is:

1. Check `current_block_number > self.last_reward_block.read()` — prevents duplicate calls per block.
2. Validate staker exists and is active.
3. **Write `self.last_reward_block.write(current_block_number)`** — consumes the block slot globally.
4. `if disable_rewards || self.is_pre_consensus() { return; }` — exits without distributing rewards. [1](#0-0) 

The critical ordering flaw: `last_reward_block` is committed to storage at step 3, **before** the `disable_rewards` guard at step 4. Since the function has no access control beyond `general_prerequisites()` (which only checks pause state and non-zero caller), any address can call:

```
update_rewards(any_valid_staker_address, disable_rewards: true)
```

This atomically:
- Marks the current block as "already rewarded" in global state.
- Returns without distributing any rewards.

Any subsequent legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

The `last_reward_block` storage field is a single global value, not per-staker: [3](#0-2) 

The `general_prerequisites` function imposes no caller restriction: [4](#0-3) 

### Impact Explanation
**High — Permanent freezing of unclaimed yield.** Rewards for a griefed block are permanently lost; there is no mechanism to retroactively distribute them. An attacker who calls `update_rewards(valid_staker, true)` once per block denies **all** stakers their consensus block rewards for every griefed block. The lost yield is irrecoverable.

### Likelihood Explanation
The attack requires no privilege, no stake, and no capital — only the gas cost of one transaction per block. A motivated adversary (e.g., a competing validator set, a protocol saboteur) can sustain this indefinitely. The entry path is fully permissionless and reachable from any EOA or contract.

### Recommendation
Move the `last_reward_block` write to **after** the `disable_rewards` guard, so that a no-op call does not consume the block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only consume the block slot when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... proceed with reward distribution
```

Alternatively, restrict the `disable_rewards: true` path to a privileged role (e.g., `only_app_governor`), or remove the parameter from the public interface entirely if it is only needed internally.

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. At block N, attacker calls `update_rewards(any_valid_staker, disable_rewards: true)`.
3. `last_reward_block` is written to N; function returns without distributing rewards.
4. Legitimate staker (or keeper) calls `update_rewards(staker, disable_rewards: false)` at block N.
5. Transaction reverts: `REWARDS_ALREADY_UPDATED` (`current_block_number > self.last_reward_block.read()` is false).
6. All stakers permanently lose consensus rewards for block N.
7. Attacker repeats every block at negligible cost. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
