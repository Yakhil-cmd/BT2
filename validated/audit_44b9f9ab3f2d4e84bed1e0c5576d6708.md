Audit Report

## Title
Byzantine Block Proposer Can Permanently Freeze a Flexible HTTP Callback via Non-Candid Response Body — (`rs/https_outcalls/consensus/src/payload_builder.rs`)

## Summary
`validate_canister_http_payload_impl` never Candid-decodes response bodies in `FlexibleCanisterHttpResponses`, so a Byzantine block proposer can include a cryptographically valid but non-Candid-encoded response body that passes all validation checks. When the block is finalized, `into_messages` silently drops the `ConsensusResponse` for that callback, yet `parse_past_payload_ids` records the `callback_id` as delivered, permanently suppressing any future timeout. The canister's callback is frozen indefinitely with no response, no timeout, and no cycles refund.

## Finding Description

**Validation gap.** `validate_flexible_response_with_proof` (utils.rs L147–198) checks only callback-ID consistency, committee membership, registry version, `crypto_hash` match, content size, and `is_reject` flag. [1](#0-0) 
It never attempts `Decode!(&data, CanisterHttpResponsePayload)`. A Byzantine node can place arbitrary bytes in `CanisterHttpResponseContent::Success(data)`, compute the correct `crypto_hash` over that response, sign the metadata, and the proof passes every check in `validate_canister_http_payload_impl`. [2](#0-1) 

**Silent drop in `into_messages`.** `flexible_ok_responses_into_consensus_response` Candid-decodes each body with `.collect::<Option<_>>()?`; a single decode failure short-circuits the entire group and returns `None`. [3](#0-2) 
The caller chains `.flatten()` on the `Option`, so the `ConsensusResponse` for that `callback_id` is silently omitted from the returned vec. [4](#0-3) 
The existing test `flexible_ok_responses_into_messages_decode_failure_is_skipped` explicitly asserts `responses.len() == 0` when one entry carries invalid Candid. [5](#0-4) 

**Timeout suppression.** The code comment at L992–996 claims "eventually a timeout will gracefully end the outstanding callback." [6](#0-5) 
This is incorrect. `parse_past_payload_ids` extracts the `callback_id` from every finalized `FlexibleResponses` message — including the malicious one — and adds it to `delivered_ids`. [7](#0-6) 
In `get_canister_http_payload_impl`, the very first check per open context is `if delivered_ids.contains(callback_id) { continue; }`, which is reached before the timeout check at L234. [8](#0-7) 
No timeout is ever proposed or finalized for this `callback_id`.

**Execution-layer leak.** `canister_http_request_contexts.remove(&callback_id)` is called only when a `ConsensusResponse` is processed. [9](#0-8) 
Because no `ConsensusResponse` was delivered, the context remains in `canister_http_request_contexts` indefinitely.

## Impact Explanation

The targeted canister's open callback is permanently frozen: no response is delivered, no timeout is generated, the `CanisterHttpRequestContext` leaks forever, cycles attached to the request are never refunded, and if the canister exhausts its callback slots it cannot make further HTTP outcalls. This is a concrete, targeted, application/platform-level DoS matching the **High** impact class: "Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS."

## Likelihood Explanation

The attack requires a single Byzantine subnet node (well within the `f < n/3` fault-tolerance threshold) that satisfies two conditions simultaneously: (1) it is a member of the flexible committee for the targeted request, and (2) it is the block proposer for a round while that request is pending. Both conditions occur in normal rotation. With `min_responses = 1` — a valid canister-controlled parameter — no collusion with other Byzantine nodes is needed. The attack is local-testable: the existing unit test already proves steps 2–3 without any mocking of cryptography.

## Recommendation

1. **Add a Candid-decode check in `validate_canister_http_payload_impl`**: for each `CanisterHttpResponseContent::Success(data)` in `payload.flexible_responses`, attempt `Decode!(&data, CanisterHttpResponsePayload)` and return `InvalidArtifact` on failure. This closes the gap at the validation boundary.
2. **Alternatively**, change `flexible_ok_responses_into_consensus_response` to emit a `SysTransient` reject `ConsensusResponse` on decode failure instead of returning `None`, so the callback is always resolved even in the degenerate case.
3. **Remove or correct the misleading comment** at L992–996 — the timeout fallback does not work because `delivered_ids` suppresses it.

## Proof of Concept

The existing test `flexible_ok_responses_into_messages_decode_failure_is_skipped` (tests.rs L3002–3026) already proves steps 2–3 without any additional setup. To prove the full end-to-end path, extend `setup_test_with_flexible_context` with `min_responses = 1`, construct a `FlexibleCanisterHttpResponses` carrying one entry whose `response.content` is `Success(b"not candid".to_vec())`, compute `crypto_hash(&response)` and set it as the proof's `content_hash`, sign with a committee node key, call `validate_payload` (expected: `Ok(())`), then call `into_messages` (expected: `responses.len() == 0`, `flexible_ok_responses_candid_failures == 1`), and finally call `get_canister_http_payload_impl` with the finalized block in `past_payloads` (expected: the callback is skipped, no timeout is proposed).

pub(crate) fn validate_flexible_response_with_proof(
    response_with_proof: &FlexibleCanisterHttpResponseWithProof,
    callback_id: CallbackId,
    flex_committee: &BTreeSet<NodeId>,
    seen_signers: &mut HashSet<NodeId>,
    consensus_registry_version: RegistryVersion,
    per_replica_allowance: Cycles,
) -> Result<(), InvalidCanisterHttpPayloadReason> {
    if response_with_proof.response.id != callback_id {
        return Err(
            InvalidCanisterHttpPayloadReason::FlexibleCallbackIdMismatch {
                callback_id,
                mismatched_id: response_with_proof.response.id,
            },
        );
    }

    validate_response_share(
        &response_with_proof.proof,
        callback_id,
        flex_committee,
        seen_signers,
        consensus_registry_version,
        per_replica_allowance,
    )?;

    let calculated_hash = crypto_hash(&response_with_proof.response);
    if &calculated_hash != response_with_proof.proof.content.content_hash() {
        return Err(InvalidCanisterHttpPayloadReason::ContentHashMismatch {
            metadata_hash: response_with_proof.proof.content.content_hash().clone(),
            calculated_hash,
        });
    }

    let calculated_size = response_with_proof.response.content.count_bytes() as u32;
    if calculated_size != response_with_proof.proof.content.content_size() {
        return Err(InvalidCanisterHttpPayloadReason::ContentSizeMismatch {
            metadata_size: response_with_proof.proof.content.content_size(),
            calculated_size,
        });
    }

    let calculated_is_reject = response_with_proof.response.content.is_reject();
    if calculated_is_reject != response_with_proof.proof.content.is_reject() {
        return Err(InvalidCanisterHttpPayloadReason::IsRejectMismatch {
            metadata_is_reject: response_with_proof.proof.content.is_reject(),
            calculated_is_reject,
        });
    }

    Ok(())
}

fn flexible_ok_responses_into_messages_decode_failure_is_skipped() {
    let callback_id = CallbackId::from(42);

    let valid_data = Encode!(&CanisterHttpResponsePayload {
        status: 200,
        headers: vec![],

    let valid_entry = flexible_response(42, 0, &valid_data);
    let invalid_entry = flexible_response(42, 1, b"this is invalid candid");

        callback_id,
        responses: vec![valid_entry, invalid_entry],

    let bytes = payload_to_bytes_max_4mb(payload);

    let (responses, stats) = CanisterHttpPayloadBuilderImpl::into_messages(&bytes);

    assert_eq!(responses.len(), 0);
    assert_eq!(stats.flexible_ok_responses, 0);
    assert_eq!(stats.flexible_ok_responses_candid_failures, 1);
}
