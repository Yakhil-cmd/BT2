The `AddVote` function in `sei-cosmos/x/gov/keeper/vote.go` only checks `proposal.Status != types.StatusVotingPeriod` but does **not** check whether `ctx.BlockHeader().Time >= proposal.VotingEndTime`. The proposal status is only updated to `StatusPassed`/`StatusRejected` inside `EndBlocker` (`sei-cosmos/x/gov/abci.go`), which runs **after** all transactions in a block are processed. This means in the exact block where `VotingEndTime` is reached, a user can still cast a vote — it will be included in the tally before `EndBlocker` finalizes the proposal.

### Title
Late Vote Accepted in Finalization Block Due to Missing Time Check in `AddVote` - (`File: sei-cosmos/x/gov/keeper/vote.go`)

### Summary
`AddVote` guards against voting on inactive proposals by checking `proposal.Status`, but this status is only updated by `EndBlocker` after all transactions execute. A voter can submit a `MsgVote` in the same block where the voting period expires, and the vote will be counted in the final tally.

### Finding Description
In `AddVote`, the only guard is a status check:

```go
if proposal.Status != types.StatusVotingPeriod {
    return sdkerrors.Wrapf(types.ErrInactiveProposal, "%d", proposalID)
}
``` [1](#0-0) 

There is no check against `proposal.VotingEndTime`. The `EndBlocker` finalizes proposals whose `VotingEndTime <= ctx.BlockHeader().Time`:

```go
keeper.IterateActiveProposalsQueue(ctx, ctx.BlockHeader().Time, func(proposal types.Proposal) bool {
    ...
    passes, burnDeposits, tallyResults := keeper.Tally(ctx, proposal)
``` [2](#0-1) 

Because Cosmos SDK processes all transactions before running `EndBlocker`, a `MsgVote` submitted in the finalization block is processed first (status still `StatusVotingPeriod`), then `EndBlocker` tallies including that vote.

The `msgServer.Vote` path confirms no additional time check exists: [3](#0-2) 

The same path is reachable via the EVM gov precompile's `vote()` function, which calls `p.govMsgServer.Vote(goCtx, msg)` directly: [4](#0-3) 

### Impact Explanation
A governance proposal's outcome can be manipulated by a late voter who monitors the chain and submits a vote in the finalization block. Governance proposals in Sei can execute arbitrary state changes (parameter updates, software upgrades, community pool spends). A manipulated outcome could constitute **unintended contract/handler execution** triggered by a network-level ordering artifact. This maps to **Medium** in the Sei bounty scope.

### Likelihood Explanation
The attacker must:
1. Hold or control enough voting power to swing the outcome
2. Monitor the chain to identify the exact finalization block
3. Submit the transaction in that specific block before `EndBlocker` runs

The window is a single block, but on Sei (with ~400ms block times) this is a predictable, automatable attack. The attacker can observe the current tally via query and decide whether to vote strategically.

### Recommendation
Add a time-based guard in `AddVote` before recording the vote:

```go
if !ctx.BlockHeader().Time.Before(proposal.VotingEndTime) {
    return sdkerrors.Wrapf(types.ErrInactiveProposal, "voting period has ended for proposal %d", proposalID)
}
```

This should be placed immediately after the status check at line 17 of `sei-cosmos/x/gov/keeper/vote.go`. [5](#0-4) 

### Proof of Concept
1. A proposal is active with `VotingEndTime = T`.
2. A block is produced with `BlockHeader.Time = T` (or `T + ε`).
3. Attacker queries current tally, determines the winning side.
4. Attacker submits `MsgVote` (or calls the EVM gov precompile) in that block.
5. Transaction executes: `proposal.Status == StatusVotingPeriod` → vote is stored.
6. `EndBlocker` runs: `IterateActiveProposalsQueue` picks up the proposal (since `VotingEndTime <= BlockTime`), calls `Tally` which includes the attacker's vote, and finalizes the proposal with the manipulated outcome.

func (p PrecompileExecutor) vote(ctx sdk.Context, method *abi.Method, caller common.Address, args []interface{}, value *big.Int) ([]byte, error) {
	if err := pcommon.ValidateNonPayable(value); err != nil {
		return nil, err
	}

	if err := pcommon.ValidateArgsLength(args, 2); err != nil {
		return nil, err
	}
	voter, found := p.evmKeeper.GetSeiAddress(ctx, caller)
	if !found {
		return nil, types.NewAssociationMissingErr(caller.Hex())
	}
	proposalID := args[0].(uint64)
	voteOption := args[1].(int32)

	msg := govtypes.NewMsgVote(voter, proposalID, govtypes.VoteOption(voteOption))
	err := msg.ValidateBasic()
	if err != nil {
		return nil, err
	}

	goCtx := sdk.WrapSDKContext(ctx)
	_, err = p.govMsgServer.Vote(goCtx, msg)
	if err != nil {
		return nil, err
	}
	return method.Outputs.Pack(true)
