### Title
Gateway Stateful Validator Checks L2 Gas Price Against Previous Block's Price Instead of Next Block's Price — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds check compares a transaction's `max_price_per_unit` against `l2_gas_price` of the **previous** (already-committed) block, but the transaction will be executed in the **next** block whose price is `next_l2_gas_price`. When EIP-1559 raises the price between blocks, the gateway admits transactions that will fail at execution, breaking the admission invariant.

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price from `get_block_info()`:

```rust

    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` populates `strk_gas_prices.l2_gas_price` from `block_header.l2_gas_price.price_in_fri`:

```rust
strk_gas_prices: GasPriceVector {
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
    ...
},
``` [2](#0-1) 

However, `BlockHeaderWithoutHash` carries a distinct field `next_l2_gas_price` — the EIP-1559-adjusted price that will govern the **next** block:

```rust
pub struct BlockHeaderWithoutHash {
    pub l2_gas_price: GasPricePerToken,   // price used in THIS block
    pub l2_gas_consumed: GasAmount,
    pub next_l2_gas_price: GasPrice,      // price for the NEXT block
    ...
}
``` [3](#0-2) 

`next_l2_gas_price` is computed via `calculate_next_l2_gas_price_for_fin` using the EIP-1559 formula and stored in the block header: [4](#0-3) 

The `GatewayFixedBlockStateReader` trait and `BlockInfo` struct never expose `next_l2_gas_price`, so `validate_resource_bounds` has no way to reach it and silently falls back to the stale `l2_gas_price`. The developer-written TODO comment at line 229 explicitly acknowledges this gap.

The threshold check itself is correct in structure:

```rust
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0).to_integer();
if tx_l2_gas_price.0 < threshold { return Err(...); }
``` [5](#0-4) 

But it is applied to the wrong reference price.

### Impact Explanation

When the network is busy (`l2_gas_consumed > gas_target`), `next_l2_gas_price > l2_gas_price`. A transaction with `max_price_per_unit` in the range `[l2_gas_price, next_l2_gas_price)` passes the gateway's stateful check and enters the mempool. When the batcher executes it, blockifier's `check_fee_bounds` compares against the actual block price (`next_l2_gas_price`) and rejects the transaction with `MaxGasPriceTooLow`. The gateway has admitted an invalid transaction — matching the **High** impact: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."

Consequences:
- Mempool is polluted with transactions that will always revert at execution.
- Users receive unexpected execution failures after passing gateway validation.
- An attacker can cheaply flood the mempool with such transactions during high-load periods, degrading throughput.

### Likelihood Explanation

This triggers whenever `l2_gas_consumed > gas_target` for the latest block — a routine condition during any period of elevated network activity. No special privileges are required; any user submitting a v3 `AllResources` transaction with `max_price_per_unit` just above the previous block's price triggers the path. The `min_gas_price_percentage` default of 100 makes the window `[l2_gas_price, next_l2_gas_price)`, which grows proportionally with network load.

### Recommendation

1. Extend `GatewayFixedBlockStateReader::get_block_info` (or add a new method) to return `next_l2_gas_price` from `BlockHeaderWithoutHash`.
2. In `validate_resource_bounds`, use `next_l2_gas_price` as the reference price instead of `l2_gas_price`.
3. Update `GatewayFixedBlockSyncStateClient` to populate this value from `block_header.next_l2_gas_price`.
4. Add a test that sets `next_l2_gas_price > l2_gas_price` and verifies that a transaction with `max_price_per_unit` between the two values is rejected.

### Proof of Concept

1. Observe a block N where `l2_gas_consumed > gas_target`, so `next_l2_gas_price = P_next > P_prev = l2_gas_price`.
2. Submit an `InvokeV3` transaction with `resource_bounds.l2_gas.max_price_per_unit = P_prev` (exactly the previous block price).
3. Gateway stateful check: `P_prev >= min_gas_price_percentage% * P_prev` → **passes**.
4. Transaction enters mempool and is picked up by the batcher for block N+1.
5. Blockifier `check_fee_bounds` in `perform_pre_validation_stage`: `P_prev < P_next` → **fails** with `MaxGasPriceTooLow`.
6. Transaction is reverted; gateway has admitted an invalid transaction. [6](#0-5) [7](#0-6) [8](#0-7)
