### Title
Invoke Transaction with Invalid Signature Bypasses `__validate__` via `skip_stateful_validations` and Is Admitted to the Mempool — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `skip_stateful_validations` UX feature unconditionally skips the `__validate__` entry-point call for an Invoke transaction with nonce = 1 when the account has a pending deploy-account transaction in the mempool. Because neither the stateless validator nor the mempool's `validate_tx` path inspects the cryptographic signature, an attacker who controls a fresh account can inject an Invoke transaction carrying an empty or forged signature into the mempool without any signature verification.

### Finding Description

**Stateless validator — no minimum signature length check**

`StatelessTransactionValidator::validate_tx_signature_size` only rejects signatures that exceed `max_signature_length`; it never enforces a minimum length. An empty signature (`[]`) passes the check silently. [1](#0-0) 

**`skip_stateful_validations` — unconditional `__validate__` bypass**

`skip_stateful_validations` returns `true` (skip) when all three conditions hold:

1. The transaction is an `Invoke` with `nonce == 1`.
2. The on-chain account nonce is `0` (account not yet deployed).
3. `mempool_client.account_tx_in_pool_or_recent_block(sender)` returns `true`. [2](#0-1) 

When `skip_validate = true`, `run_validate_entry_point` sets `execution_flags.validate = false`: [3](#0-2) 

The blockifier's `StatefulValidator::perform_validations` then returns `Ok(())` immediately for Invoke transactions without calling `validate_tx` (the `__validate__` entry point): [4](#0-3) 

**`validate_by_mempool` does not inspect the signature**

`ValidationArgs`, the struct passed to the mempool's `validate_tx`, contains only address, nonce, tx hash, tip, and gas price — no signature field. The mempool therefore cannot detect an invalid signature. [5](#0-4) 

**End-to-end admission path with invalid signature**

```
extract_state_nonce_and_run_validations
  └─ run_pre_validation_checks
       ├─ validate_state_preconditions   (nonce + resource bounds — no sig)
       ├─ validate_by_mempool            (nonce/fee/capacity — no sig)
       └─ skip_stateful_validations ──► true
  └─ run_validate_entry_point(skip_validate=true)
       └─ StatefulValidator::perform_validations
            └─ if !execution_flags.validate { return Ok(()); }  ← __validate__ never called
``` [6](#0-5) 

### Impact Explanation

An Invoke transaction carrying an empty or otherwise invalid signature is accepted by the gateway and inserted into the mempool. The broken invariant is: **every transaction admitted to the mempool must have passed account-level signature verification**. The invalid transaction will fail when the batcher later executes it (the batcher does run `__validate__`), but it occupies mempool capacity and forces the batcher to waste execution resources on a transaction that is guaranteed to fail. This matches the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."*

### Likelihood Explanation

The preconditions are reachable by any unprivileged user:

1. Generate a fresh key pair and compute the corresponding contract address.
2. Submit a valid `DeployAccount` transaction (nonce = 0, valid signature) — it enters the mempool.
3. Immediately submit an `Invoke` transaction (nonce = 1, **empty signature**) for the same address.
4. The gateway reads on-chain nonce = 0, finds the deploy-account in the mempool, sets `skip_validate = true`, and admits the Invoke without calling `__validate__`.

No privileged access, no special network position, and no prior on-chain state are required beyond the cost of one valid deploy-account transaction.

### Recommendation

Add a guard inside `skip_stateful_validations` (or in `run_validate_entry_point`) that verifies the signature is structurally non-empty before allowing the skip. Alternatively, perform a lightweight ECDSA format check (e.g., require exactly two non-zero Felt elements for a standard STARK signature) in the stateless validator so that structurally invalid signatures are rejected before reaching the stateful path. The skip should only suppress the on-chain `__validate__` call, not the gateway's own signature-presence check.

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
