### Title
Unguarded `new()` Initializer Allows Any Caller to Seize Engine Ownership — (`File: engine/src/contract_methods/admin.rs`)

### Summary
The Aurora Engine's `new()` initializer performs no caller authentication. Any NEAR account that calls `new()` before the legitimate deployer does will permanently install itself as `owner_id` in the engine state. Because every privileged admin function gates on `state.owner_id`, the attacker gains full administrative control of the engine, including the ability to upgrade the contract to arbitrary code and redirect the ETH connector, enabling theft of all bridged funds.

### Finding Description

`new()` in `engine/src/contract_methods/admin.rs` contains exactly one guard:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // ... no check on env.predecessor_account_id()
    let state: EngineState = args.into();   // owner_id comes from attacker-supplied args
    state::set_state(&mut io, &state)?;
    Ok(())
}
``` [1](#0-0) 

There is no check that `env.predecessor_account_id() == env.current_account_id()` or any other privileged identity. The `EngineState` written to storage contains `owner_id`, which is taken directly from the caller-supplied `NewCallArgs`: [2](#0-1) 

In NEAR Protocol, contract deployment (uploading WASM) and initialization (calling `new()`) are separate transactions. Between those two steps, any NEAR account can call `new()` on the freshly deployed contract and supply an arbitrary `owner_id`.

Once the attacker's `new()` call lands first, the legitimate deployer's subsequent `new()` call is rejected with `ERR_ALREADY_INITIALIZED`. The engine state now records the attacker as `owner_id`. [3](#0-2) 

Recovery is impossible through the engine's own API: `set_owner` requires `require_owner_only`, which compares `state.owner_id` against `predecessor_account_id`: [4](#0-3) 

So the legitimate deployer, even holding full NEAR access keys to the account, cannot call `set_owner` because their account ID does not match the attacker-controlled `state.owner_id`.

### Impact Explanation

Every destructive admin action is gated on `require_owner_only`:

- `stage_upgrade` + `upgrade` — deploys arbitrary new WASM to the engine contract: [5](#0-4) [6](#0-5) 

- `set_eth_connector_contract_account` — redirects all ETH/ERC-20 bridge flows to an attacker-controlled contract: [7](#0-6) 

With ownership, the attacker can: (1) call `stage_upgrade` to stage malicious WASM, (2) call `upgrade` to deploy it, and (3) drain all bridged ETH and ERC-20 tokens held by the engine and its connector. This is **direct theft of all user funds at rest** — Critical severity.

### Likelihood Explanation

NEAR Protocol validators order transactions within a block. An attacker monitoring the NEAR mempool (or simply watching for new contract deployments) can observe the WASM deployment action and immediately submit a `new()` call targeting the freshly deployed contract before the legitimate deployer's initialization transaction is included. The attack window is one block (~1 second on NEAR mainnet). The attack requires no special privileges, no capital, and is a single transaction. It is a one-time, permanent, and irreversible takeover.

### Recommendation

Restrict `new()` so that only the contract account itself can call it. In NEAR, the contract account's own ID is `env.current_account_id()`. Add the following guard at the top of `new()`:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // ...
}
```

This mirrors the pattern already used by `factory_update_address_version` (`env.assert_private_call()`) and ensures only the account itself — controlled by the deployer's full-access key — can initialize the engine state. [8](#0-7) 

### Proof of Concept

1. Legitimate deployer uploads Aurora Engine WASM to account `aurora` via a `DeployContract` action.
2. Attacker observes the deployment in the NEAR mempool and immediately submits:
   ```
   FunctionCall {
     contract_id: "aurora",
     method_name: "new",
     args: NewCallArgsV4 {
       chain_id: <any>,
       owner_id: "attacker.near",
       upgrade_delay_blocks: 0,
       key_manager: "attacker.near",
       initial_hashchain: None,
     }
   }
   ```
3. Attacker's `new()` is included in the same or next block before the deployer's `new()`.
4. `state::set_state` writes `owner_id = "attacker.near"` to engine storage.
5. Legitimate deployer's `new()` call returns `ERR_ALREADY_INITIALIZED`.
6. Attacker calls `stage_upgrade` with malicious WASM bytes (succeeds because `state.owner_id == "attacker.near"`).
7. Attacker calls `upgrade`, deploying malicious code that transfers all bridged ETH and ERC-20 balances to the attacker. [9](#0-8) [5](#0-4) [6](#0-5)

#[named]
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}
