### Title
Notional Fee Truncation to Zero on Low-Decimal Tokens Permanently Loses Protocol/Admin Fees on Every `collectFees` Call — (`metric-core/contracts/MetricOmmPool.sol`)

---

### Summary

`MetricOmmPool.collectFees` unconditionally resets `notionalFeeToken0Scaled` and `notionalFeeToken1Scaled` to zero after converting accumulated scaled fees to external token units using floor division. For tokens with fewer than 18 decimals (e.g., USDC/USDT with 6 decimals, `TOKEN_X_SCALE_MULTIPLIER = 10^12`), any accumulated notional fee below one external token unit rounds to zero and is never transferred, yet the storage slot is still cleared. The sub-threshold amount is permanently removed from notional fee accounting. Because `collectPoolFees` on the factory is permissionless, any caller can trigger this loss repeatedly after every swap.

---

### Finding Description

**Scale multiplier derivation** (`_getScaleMultipliers` in the factory, mirrored in `PoolInitPreprocessor`):

```
token0ScaleMultiplier = 10 ^ (max(18, decimals) - token0.decimals())
```

For USDC/USDT (6 decimals): `TOKEN_0_SCALE_MULTIPLIER = 10^12`.

**Notional fee accumulation** during swaps (e.g., exact-in zeroForOne path):

```solidity
uint256 notionalFeeScaled = uint256(-amount1DeltaScaled) * notionalFeeE8 / 1e8;
notionalFeeToken1Scaled = (uint256(notionalFeeToken1Scaled) + notionalFeeScaled).toUint128();
``` [1](#0-0) 

**Fee collection** in `collectFees`:

```solidity
(uint256 totalFee0ToAdmin, uint256 totalFee1ToAdmin) =
    deltasScaledToExternal(totalFee0ToAdminScaled, totalFee1ToAdminScaled, Math.Rounding.Floor);
(uint256 totalFee0ToProtocol, uint256 totalFee1ToProtocol) =
    deltasScaledToExternal(totalFee0ToProtocolScaled, totalFee1ToProtocolScaled, Math.Rounding.Floor);
// ... conditional transfers ...
notionalFeeToken0Scaled = 0;   // ← unconditional reset
notionalFeeToken1Scaled = 0;
``` [2](#0-1) 

`deltasScaledToExternal` with `Math.Rounding.Floor` performs:

```solidity
deltaAmount0 = scaledDeltaAmount0 / TOKEN_0_SCALE_MULTIPLIER;
``` [3](#0-2) 

If `totalFee0ToAdminScaled + totalFee0ToProtocolScaled < 10^12` (for USDC), both external amounts are `0`. No transfer occurs, but `notionalFeeToken0Scaled` is still zeroed. The sub-threshold scaled amount is removed from notional fee tracking.

**What happens to the "lost" amount?** The pool's `balance0()` is unchanged. On the next `collectFees` call, the surplus calculation is:

```solidity
surplus0Scaled = balance0() * TOKEN_0_SCALE_MULTIPLIER - binTotals.scaledToken0 - notionalFee0AmountScaled;
``` [4](#0-3) 

With `notionalFeeToken0Scaled = 0`, the previously lost amount now appears as spread-fee surplus. If `spreadSumE6 == 0` (pool has only notional fees, no spread fees), the surplus is never distributed — the amount is permanently stranded in the pool, benefiting LPs instead of admin/protocol. If `spreadSumE6 > 0`, the amount is eventually collected but split according to spread-fee rates rather than notional-fee rates, corrupting the intended admin/protocol split.

**Permissionless trigger:** `collectPoolFees` on the factory has no access control:

```solidity
function collectPoolFees(address pool) external override nonReentrant {
``` [5](#0-4) 

Any address can call it at any time, including after every individual swap.

---

### Impact Explanation

For a USDC/USDT pool (6 decimals, `TOKEN_0_SCALE_MULTIPLIER = 10^12`):

- Maximum loss per `collectFees` call: up to `10^12 − 1` scaled units ≈ **1 USDC** in notional fees.
- An attacker (or a bot) calling `collectPoolFees` after every swap ensures `notionalFeeToken0Scaled` never accumulates past the threshold, draining all notional fee revenue for that token.
- On a pool with zero spread fee, the drained notional fees are permanently stranded as LP surplus — a direct loss of protocol and admin fee revenue.
- On a pool with nonzero spread fee, the admin/protocol split is silently corrupted (notional-fee amounts are redistributed at spread-fee rates).

---

### Likelihood Explanation

- USDC and USDT (6 decimals) are explicitly in scope and are the most common stablecoin pair for AMM pools.
- `collectPoolFees` is permissionless — no special role is required.
- The loss is deterministic and repeatable: every `collectFees` call where the accumulated notional fee is below one external token unit triggers the bug.
- High-frequency swap pools accumulate notional fees in small increments, making sub-threshold accumulation the common case between collections.

---

### Recommendation

Do not reset `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` to zero unconditionally. Instead, only deduct the portion that was actually converted to external units:

```solidity
// Compute how many scaled units were actually transferred
uint256 actualFee0Transferred = (totalFee0ToAdmin + totalFee0ToProtocol) * TOKEN_0_SCALE_MULTIPLIER;
uint256 actualFee1Transferred = (totalFee1ToAdmin + totalFee1ToProtocol) * TOKEN_1_SCALE_MULTIPLIER;

// Retain the remainder for the next collection
notionalFeeToken0Scaled = uint128(notionalFee0AmountScaled - actualFee0Transferred);
notionalFeeToken1Scaled = uint128(notionalFee1AmountScaled - actualFee1Transferred);
```

This preserves sub-threshold notional fees across collection calls so they accumulate until they can be transferred as at least one external token unit.

### Proof of Concept

**Setup:**
- Pool: USDC (6 dec, token0) / WETH (18 dec, token1)
- `TOKEN_0_SCALE_MULTIPLIER = 10^12`
- `notionalFeeE8 = 10_000` (0.01%)
- Only notional fee configured; `spreadSumE6 = 0`

**Step 1 — Swap generating sub-threshold notional fee:**

A swap produces `amount1DeltaScaled = 5 × 10^11` (0.5 USDC worth of scaled token0 output). Notional fee:

```
notionalFeeScaled = 5e11 * 10_000 / 1e8 = 50_000 scaled units
```

`notionalFeeToken0Scaled = 50_000` (far below `10^12`).

**Step 2 — Attacker calls `collectPoolFees`:**

```
totalFee0ToAdminScaled + totalFee0ToProtocolScaled = 50_000
totalFee0ToAdmin = 50_000 / 10^12 = 0  (floor)
totalFee0ToProtocol = 0
```

No transfer. `notionalFeeToken0Scaled = 0`.

**Step 3 — Next collection:**

`surplus0Scaled` now includes the 50,000 scaled units. Since `spreadSumE6 = 0`, the surplus is not distributed. The 50,000 scaled units (≈ 0.00000005 USDC per event, but compounding across thousands of swaps) are permanently stranded.

**Scaled over time:** With 10,000 swaps/day each generating 50,000 scaled units of notional fee, and `collectPoolFees` called after each swap, the daily loss is `10,000 × 50,000 = 5 × 10^8` scaled units = **0.0005 USDC/day** — small per pool but systematic and griefable across all low-decimal pools in the protocol.
