### Title
Unrestricted `update_rewards` with `disable_rewards=true` Permanently Freezes Consensus Reward Distribution ‚Äî (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control beyond a non-zero caller check. A global `last_reward_block` variable is written unconditionally before the early-return path triggered by `disable_rewards=true`. Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards=true)` in every block, consuming the single per-block reward slot without distributing any rewards, permanently freezing all stakers' unclaimed consensus yield.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. No role or identity check restricts who may call it. [1](#0-0) 

Inside the function, `last_reward_block` is written to the current block number **unconditionally**, before the guard that short-circuits reward distribution: [2](#0-1) 

Because `last_reward_block` is a **single global value** shared across all stakers, and the function asserts `current_block_number > self.last_reward_block.read()` at entry, only one successful call is possible per block for the entire protocol. [3](#0-2) 

An attacker who calls `update_rewards(any_valid_staker, disable_rewards=true)` first in a block:
1. Passes all validation checks (staker must exist, be active, have non-zero balance).
2. Writes `last_reward_block = current_block`.
3. Returns early without distributing any rewards.

Any subsequent legitimate call in the same block (e.g., from the block proposer) reverts with `REWARDS_ALREADY_UPDATED`.

The `last_reward_block` storage field is declared as a single global: [4](#0-3) 

---

### Impact Explanation

By repeating this call in every block, an attacker permanently prevents consensus rewards from being distributed to all stakers and their delegators. Stakers' `unclaimed_rewards_own` and pool `cumulative_rewards_trace` are never updated. This constitutes **permanent freezing of unclaimed yield** for the entire protocol ‚Äî matching the allowed High impact category. [5](#0-4) 

---

### Likelihood Explanation

The attack requires no special privileges, no funds, and no collusion. Any non-zero address can call `update_rewards` with any valid active staker address and `disable_rewards=true`. On Starknet, the attacker simply needs to submit this transaction in each block before the legitimate block proposer's reward update. Gas cost is the only barrier, and the attacker has no profit motive requirement ‚Äî a competing staker wishing to suppress rivals' rewards has direct incentive. [6](#0-5) 

---

### Recommendation

Add access control to `update_rewards` so that only the designated consensus mechanism (e.g., a trusted operator role or the block proposer) can call it. Alternatively, decouple the `last_reward_block` write from the `disable_rewards` early-return path so that a no-op call does not consume the per-block slot.

---

### Proof of Concept

1. Staker Alice stakes and becomes active with non-zero STRK balance.
2. Attacker (any address) calls `staking.update_rewards(alice_address, disable_rewards=true)` in block N.
3. `last_reward_block` is set to N; no rewards are distributed; function returns early.
4. Block proposer calls `staking.update_rewards(alice_address, disable_rewards=false)` in block N ‚Üí reverts with `REWARDS_ALREADY_UPDATED`.
5. Attacker repeats step 2 in every subsequent block.
6. All stakers' consensus rewards are permanently frozen; `unclaimed_rewards_own` is never incremented; pool `cumulative_rewards_trace` is never updated. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
