### Title
USDC-Blacklisted `adminFeeDestination` Permanently Blocks Fee Collection and Fee-Rate Updates - (File: `metric-core/contracts/MetricOmmPoolFactory.sol` / `metric-core/contracts/MetricOmmPool.sol`)

---

### Summary

`collectFees()` in `MetricOmmPool` performs forced ERC-20 transfers to `adminFeeDestination_` before returning. Three factory entry-points ŌĆö the permissionless `collectPoolFees()`, the pool-admin-gated `setPoolAdminFees()`, and the owner-gated `setPoolProtocolFee()` ŌĆö all call `collectFees()` unconditionally. If either pool token is USDC and `adminFeeDestination` is USDC-blacklisted, every one of those calls reverts, permanently locking accrued protocol and admin fees inside the pool and preventing any fee-rate change until the admin separately updates the destination.

---

### Finding Description

`MetricOmmPool.collectFees()` computes the admin and protocol shares of accrued spread and notional fees, then transfers them in sequence:

```solidity
// MetricOmmPool.sol lines 416-427
if (totalFee0ToAdmin > 0) {
    transferToken0(adminFeeDestination_, totalFee0ToAdmin);   // ŌåÉ reverts if blacklisted
}
if (totalFee1ToAdmin > 0) {
    transferToken1(adminFeeDestination_, totalFee1ToAdmin);   // ŌåÉ reverts if blacklisted
}
if (totalFee0ToProtocol > 0) {
    transferToken0(FACTORY, totalFee0ToProtocol);
}
if (totalFee1ToProtocol > 0) {
    transferToken1(FACTORY, totalFee1ToProtocol);
}
notionalFeeToken0Scaled = 0;   // ŌåÉ never reached on revert
notionalFeeToken1Scaled = 0;
```

The state-clearing lines (`notionalFeeToken0Scaled = 0`) are placed **after** the transfers, so a revert on any transfer leaves the pool's fee accounting unchanged and the tokens stranded.

### Impact Explanation

When `adminFeeDestination` is USDC-blacklisted:

1. **`collectPoolFees()` permanently reverts** ŌĆö the permissionless fee-collection path is DoS'd; accrued spread surplus and `notionalFeeToken{0,1}Scaled` are never cleared and grow with every subsequent swap.
2. **`setPoolAdminFees()` permanently reverts** ŌĆö the pool admin cannot adjust fee rates in response to market conditions.
3. **`setPoolProtocolFee()` permanently reverts** ŌĆö the protocol owner cannot change the protocol fee component for this pool.
4. **Fees are locked in the pool** ŌĆö all accrued protocol and admin revenue is stranded until the admin separately calls `setPoolAdminFeeDestination()` to rotate to a non-blacklisted address.

Core swap and liquidity operations (`swap`, `addLiquidity`, `removeLiquidity`) are unaffected. The impact is loss of protocol/admin fee revenue and broken fee-management governance for the affected pool.

---

### Likelihood Explanation

USDC is an explicitly in-scope pool token. USDC's blacklist is exercised by Circle in response to regulatory/sanctions actions. A pool's `adminFeeDestination` could be a multisig or treasury address that later becomes sanctioned. The likelihood is **low** (same as the reference report), but the consequence ŌĆö locked fees and frozen fee governance ŌĆö is **medium** severity because it affects protocol revenue and admin control without any self-healing mechanism until the admin acts.

---

### Recommendation

Adopt a pull-payment (claim) pattern for fee collection: instead of pushing tokens inside `collectFees()`, credit internal balance mappings and expose a separate `claimFees(address recipient)` function. This decouples fee accounting from token delivery, so a blacklisted destination cannot block fee-rate updates or the accounting reset.

Alternatively, split the fee-rate-update functions (`setPoolAdminFees`, `setPoolProtocolFee`) from the fee-collection step: update rates first, then attempt collection in a separate, non-reverting call. At minimum, `setPoolAdminFees` and `setPoolProtocolFee` should not gate the rate update on a successful token transfer.

---

### Proof of Concept

1. Deploy a pool with USDC as `token0` and set `adminFeeDestination = 0xSanctioned`.
2. Execute several swaps so that `notionalFeeToken0Scaled > 0` and a spread surplus accumulates.
3. USDC Circle blacklists `0xSanctioned`.
4. Call `collectPoolFees(pool)` ŌåÆ reverts at `transferToken0(0xSanctioned, ...)`.
5. Pool admin calls `setPoolAdminFees(pool, newFee, newFee)` ŌåÆ same revert; fee rates cannot be changed.
6. Protocol owner calls `setPoolProtocolFee(pool, newFee, newFee)` ŌåÆ same revert.
7. `notionalFeeToken0Scaled` is never zeroed; fees accumulate indefinitely inside the pool.
8. Recovery: pool admin calls `setPoolAdminFeeDestination(pool, 0xClean)` (this function does **not** call `collectFees()`), then `collectPoolFees(pool)` succeeds. [5](#0-4) [1](#0-0) [6](#0-5)

  function collectFees(
    uint256 protocolSpreadFeeE6_,
    uint256 adminSpreadFeeE6_,
    uint256 protocolNotionalFeeE8_,
    uint256 adminNotionalFeeE8_,
    address adminFeeDestination_
  ) external onlyFactory nonReentrant(PoolActions.COLLECT_FEES) {
    uint256 spreadSumE6;
    uint256 notionalSumE8;
    unchecked {
      spreadSumE6 = protocolSpreadFeeE6_ + adminSpreadFeeE6_;
      notionalSumE8 = protocolNotionalFeeE8_ + adminNotionalFeeE8_;
      if (spreadSumE6 == 0 && notionalSumE8 == 0) {
        return;
      }
    }

    uint256 notionalFee0AmountScaled = notionalFeeToken0Scaled;
    uint256 notionalFee1AmountScaled = notionalFeeToken1Scaled;

    uint256 surplus0Scaled =
      balance0() * TOKEN_0_SCALE_MULTIPLIER - uint256(binTotals.scaledToken0) - notionalFee0AmountScaled;
    uint256 surplus1Scaled =
      balance1() * TOKEN_1_SCALE_MULTIPLIER - uint256(binTotals.scaledToken1) - notionalFee1AmountScaled;

    unchecked {
      uint256 spreadFee0ToAdminScaled = spreadSumE6 == 0 ? 0 : (surplus0Scaled * adminSpreadFeeE6_) / spreadSumE6;
      uint256 spreadFee1ToAdminScaled = spreadSumE6 == 0 ? 0 : (surplus1Scaled * adminSpreadFeeE6_) / spreadSumE6;

      uint256 spreadFee0ToProtocolScaled = spreadSumE6 == 0 ? 0 : (surplus0Scaled * protocolSpreadFeeE6_) / spreadSumE6;
      uint256 spreadFee1ToProtocolScaled = spreadSumE6 == 0 ? 0 : (surplus1Scaled * protocolSpreadFeeE6_) / spreadSumE6;

      uint256 notionalFee0ToAdminScaled =
        notionalSumE8 == 0 ? 0 : (notionalFee0AmountScaled * adminNotionalFeeE8_) / notionalSumE8;
      uint256 notionalFee1ToAdminScaled =
        notionalSumE8 == 0 ? 0 : (notionalFee1AmountScaled * adminNotionalFeeE8_) / notionalSumE8;

      uint256 notionalFee0ToProtocolScaled = notionalFee0AmountScaled - notionalFee0ToAdminScaled;
      uint256 notionalFee1ToProtocolScaled = notionalFee1AmountScaled - notionalFee1ToAdminScaled;

      uint256 totalFee0ToAdminScaled = spreadFee0ToAdminScaled + notionalFee0ToAdminScaled;
      uint256 totalFee1ToAdminScaled = spreadFee1ToAdminScaled + notionalFee1ToAdminScaled;

      uint256 totalFee0ToProtocolScaled = spreadFee0ToProtocolScaled + notionalFee0ToProtocolScaled;
      uint256 totalFee1ToProtocolScaled = spreadFee1ToProtocolScaled + notionalFee1ToProtocolScaled;

      (uint256 totalFee0ToAdmin, uint256 totalFee1ToAdmin) =
        deltasScaledToExternal(totalFee0ToAdminScaled, totalFee1ToAdminScaled, Math.Rounding.Floor);
      (uint256 totalFee0ToProtocol, uint256 totalFee1ToProtocol) =
        deltasScaledToExternal(totalFee0ToProtocolScaled, totalFee1ToProtocolScaled, Math.Rounding.Floor);

      if (totalFee0ToAdmin > 0) {
        transferToken0(adminFeeDestination_, totalFee0ToAdmin);
      }
      if (totalFee1ToAdmin > 0) {
        transferToken1(adminFeeDestination_, totalFee1ToAdmin);
      }
      if (totalFee0ToProtocol > 0) {
        transferToken0(FACTORY, totalFee0ToProtocol);
      }
      if (totalFee1ToProtocol > 0) {
        transferToken1(FACTORY, totalFee1ToProtocol);
      }

      notionalFeeToken0Scaled = 0;
      notionalFeeToken1Scaled = 0;

      emit ProtocolFeesCollected(totalFee0ToProtocol, totalFee1ToProtocol, totalFee0ToAdmin, totalFee1ToAdmin);
    }
  }
