### Title
Missing Access Control on `update_rewards` Enables Any Caller to Permanently Freeze Protocol-Wide Reward Distribution — (`src/staking/staking.cairo`)

---

### Summary

`update_rewards` in the Staking contract is documented as callable by "Only starkware sequencer" but has **no on-chain caller restriction**. Because it writes to the global `last_reward_block` storage slot, any unprivileged address can call it every block with `disable_rewards: true` to permanently prevent the sequencer from distributing rewards to any staker.

---

### Finding Description

The spec explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation only calls `general_prerequisites()`, which checks:

1. Contract is not paused
2. Caller is not the zero address

```cairo
// src/staking/staking.cairo:1449-1452
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only: unpaused + caller != 0
``` [1](#0-0) 

There is no check that `get_caller_address() == sequencer_address`. The function then unconditionally writes the current block number into the **global** `last_reward_block` slot:

```cairo
// src/staking/staking.cairo:1484-1485
// Update last block rewards.
self.last_reward_block.write(current_block_number);
``` [2](#0-1) 

The guard at the top of the function prevents any second call in the same block:

```cairo
// src/staking/staking.cairo:1453-1458
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

`last_reward_block` is a single global field shared across all stakers:

```cairo
// src/staking/staking.cairo:186-187
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [4](#0-3) 

---

### Impact Explanation

An attacker who calls `update_rewards(staker_address: any_valid_staker, disable_rewards: true)` in every block:

1. Passes all checks (staker exists, is active, has non-zero balance).
2. Writes `last_reward_block = current_block`.
3. Returns early — **no rewards are distributed** (`disable_rewards: true` short-circuits before any reward calculation).
4. The sequencer's own call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

Because the attacker can repeat this every block at low cost, **all stakers' unclaimed yield is permanently frozen** for as long as the attack continues. The attacker does not need to be a staker themselves — they only need to supply any valid, active staker address (public information from on-chain events).

**Impact category matched**: *Permanent freezing of unclaimed yield* (High).

---

### Likelihood Explanation

- No privileged role is required; any non-zero address suffices.
- Valid staker addresses are publicly observable from `NewStaker` events.
- Calling once per block is sufficient to maintain the DoS; gas costs on Starknet are low.
- The attack is immediately executable after the consensus rewards epoch begins (`is_pre_consensus()` returns `false`).

---

### Recommendation

Add an explicit on-chain sequencer check inside `update_rewards`, consistent with the spec. On Starknet this can be done via:

```cairo
let exec_info = starknet::get_execution_info().unbox();
assert!(
    get_caller_address() == exec_info.block_info.unbox().sequencer_address,
    "{}",
    Error::CALLER_IS_NOT_SEQUENCER,
);
```

Alternatively, store the sequencer address in contract storage (set by governance) and assert against it.

---

### Proof of Concept

1. Deploy the protocol and advance past the consensus rewards first epoch.
2. Identify any valid, active staker address `S` (from `NewStaker` events).
3. From any EOA `A` (not a staker, not the sequencer), call:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Observe: call succeeds; `last_reward_block` is set to the current block; no rewards distributed.
5. Advance one block. Repeat step 3.
6. The sequencer's `update_rewards` call in every block reverts with `REWARDS_ALREADY_UPDATED`.
7. All stakers accumulate zero `unclaimed_rewards_own` indefinitely.

The existing test suite confirms the absence of any caller restriction — tests invoke `update_rewards` directly without impersonating a sequencer address:

```cairo
// src/staking/tests/test.cairo:3515
staking_rewards_dispatcher.update_rewards(:staker_address, disable_rewards: false);
``` [5](#0-4)
