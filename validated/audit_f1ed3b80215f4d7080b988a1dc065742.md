### Title
Stale Cached BTC Token Decimals Enable Inflated Staking Power and Theft of Unclaimed Yield After Token Upgrade - (File: `src/staking/staking.cairo`)

---

### Summary

The staking contract caches a BTC token's decimals once at `add_token` time into the `token_decimals` storage map. Every subsequent normalization and denormalization of delegated amounts reads from this stale cache rather than querying the live token. If the BTC token is later upgraded to change its decimal count, all accounting diverges from reality, allowing an unprivileged delegator to obtain inflated staking power and drain unclaimed yield from the protocol.

---

### Finding Description

In `add_token`, the contract reads the token's decimals once and writes them to storage: [1](#0-0) 

```cairo
let token_dispatcher = IERC20MetadataDispatcher { contract_address: token_address };
let decimals = token_dispatcher.decimals();
assert!(decimals >= 5 && decimals <= 18, "{}", GenericError::INVALID_TOKEN_DECIMALS);
self.btc_tokens.write(token_address, (STARTING_EPOCH, false));
self.token_decimals.write(token_address, decimals);
```

This cached value is then used in every normalization call throughout the contract's lifetime. For example, in `add_stake_from_pool`: [2](#0-1) 

```cairo
let decimals = self.get_token_decimals(:token_address);
let normalized_amount = NormalizedAmountTrait::from_native_amount(:amount, :decimals);
```

The normalization formula is: [3](#0-2) 

```cairo
fn from_native_amount(amount: Amount, decimals: u8) -> NormalizedAmount {
    assert!(decimals >= 5 && decimals <= 18, "{}", GenericError::INVALID_TOKEN_DECIMALS);
    NormalizedAmount { amount_18_decimals: amount * 10_u128.pow(18 - decimals.into()) }
}
```

The same stale decimals are used in `remove_from_delegation_pool_intent`, `remove_from_delegation_pool_action`, `switch_staking_delegation_pool`, `unstake_intent`, and `transfer_to_pools_when_unstake`: [4](#0-3) [5](#0-4) 

There is **no function** in `IStakingTokenManager` to update the cached decimals after the fact — only `add_token`, `enable_token`, and `disable_token` exist: [6](#0-5) 

---

### Impact Explanation

If the BTC token's decimals increase from `d_old` to `d_new` (e.g., 8 → 18):

- A delegator sends `X` native units, which now represent `X / 10^18` BTC (nearly zero real value).
- The staking contract normalizes using stale `d_old = 8`: `X * 10^(18-8) = X * 10^10`.
- The correct normalization with `d_new = 18` would be: `X * 10^(18-18) = X`.
- The delegator's `NormalizedAmount` is inflated by `10^10`.
- Staking power and reward share are computed from `NormalizedAmount`: [7](#0-6) 

- The attacker earns `10^10× ` more rewards than their real stake warrants — **theft of unclaimed yield (High)**.

---

### Likelihood Explanation

BTC tokens on Starknet (e.g., wrapped BTC) are upgradeable proxy contracts. A governance-approved upgrade that adjusts decimals (a known pattern in EVM token history) is a realistic trigger. The staking contract explicitly supports multiple BTC tokens via `add_token` and has no guard against this scenario. Any unprivileged delegator can exploit the window between the token upgrade and any manual remediation, since there is no on-chain synchronization mechanism.

---

### Recommendation

1. Add a `update_token_decimals(token_address)` function (callable by `token_admin`) that re-reads the live decimals and adjusts all existing normalized balances proportionally.
2. Alternatively, always read decimals live from the token contract instead of caching, accepting the extra call overhead.
3. At minimum, add a consistency check in `add_stake_from_pool` and `remove_from_delegation_pool_action` that asserts the live decimals match the cached value, reverting if they diverge.

---

### Proof of Concept

1. Token admin calls `add_token(wbtc)` when wBTC has 8 decimals. `token_decimals[wbtc] = 8` is stored.
2. wBTC governance upgrades the token to 18 decimals.
3. Eve calls `enter_delegation_pool(reward_addr, 1_000_000_00)` (= 1 wBTC at 8 decimals = `10^-10` wBTC at 18 decimals).
4. Pool calls `add_stake_from_pool(staker, 100_000_000)`.
5. Staking contract reads `decimals = 8` (stale), normalizes: `10^8 * 10^10 = 10^18` normalized units.
6. Correct normalization would be `10^8 * 10^0 = 10^8` normalized units.
7. Eve's staking power is `10^10×` inflated.
8. Eve's share of block rewards is `10^10×` larger than her real contribution.
9. Eve calls `claim_rewards`, draining unclaimed yield from honest stakers.

    #[abi(embed_v0)]
    impl StakingTokenManagerImpl of IStakingTokenManager<ContractState> {
        fn add_token(ref self: ContractState, token_address: ContractAddress) {
            self.roles.only_token_admin();
            assert!(token_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
            assert!(self.staker_info.read(token_address).is_none(), "{}", Error::TOKEN_IS_STAKER);
            assert!(token_address != STRK_TOKEN_ADDRESS, "{}", Error::INVALID_TOKEN_ADDRESS);
            assert!(
                self.btc_tokens.read(token_address).is_none(), "{}", Error::TOKEN_ALREADY_EXISTS,
            );
            let token_dispatcher = IERC20MetadataDispatcher { contract_address: token_address };
            let decimals = token_dispatcher.decimals();
            assert!(decimals >= 5 && decimals <= 18, "{}", GenericError::INVALID_TOKEN_DECIMALS);
            self.btc_tokens.write(token_address, (STARTING_EPOCH, false));
            self.token_decimals.write(token_address, decimals);
            // Initialize the token total stake trace.
            self
                .tokens_total_stake_trace
                .entry(token_address)
                .insert(key: STARTING_EPOCH, value: Zero::zero());
            self.emit(TokenManagerEvents::TokenAdded { token_address });
        }

        fn enable_token(ref self: ContractState, token_address: ContractAddress) {
            self.roles.only_token_admin();
            let is_active_opt: Option<(Epoch, bool)> = self.btc_tokens.read(token_address);
            assert!(is_active_opt.is_some(), "{}", Error::TOKEN_NOT_EXISTS);
            let (is_active_first_epoch, is_active) = is_active_opt.unwrap();
            let curr_epoch = self.get_current_epoch();
            assert!(curr_epoch >= is_active_first_epoch, "{}", Error::INVALID_EPOCH);
            assert!(!is_active, "{}", Error::TOKEN_ALREADY_ENABLED);
            let next_is_active_first_epoch = self.get_epoch_plus_k();
            self.btc_tokens.write(token_address, (next_is_active_first_epoch, true));
            self.emit(TokenManagerEvents::TokenEnabled { token_address });
        }

        fn disable_token(ref self: ContractState, token_address: ContractAddress) {
            self.roles.only_security_agent();
            let is_active_opt: Option<(Epoch, bool)> = self.btc_tokens.read(token_address);
            assert!(is_active_opt.is_some(), "{}", Error::TOKEN_NOT_EXISTS);
            let (is_active_first_epoch, is_active) = is_active_opt.unwrap();
            let curr_epoch = self.get_current_epoch();
            assert!(curr_epoch >= is_active_first_epoch, "{}", Error::INVALID_EPOCH);
            assert!(is_active, "{}", Error::TOKEN_ALREADY_DISABLED);
            let next_is_active_first_epoch = self.get_epoch_plus_k();
            self.btc_tokens.write(token_address, (next_is_active_first_epoch, false));
            self.emit(TokenManagerEvents::TokenDisabled { token_address });
        }
    }

pub(crate) fn calculate_staker_total_staking_power(
    staker_strk_total_amount: NormalizedAmount,
    staker_btc_total_amount: NormalizedAmount,
    strk_total_stake: NormalizedAmount,
    btc_total_stake: NormalizedAmount,
) -> StakingPower {
    let strk_staking_power = mul_wide_and_div(
        lhs: staker_strk_total_amount.to_amount_18_decimals(),
        rhs: STRK_WEIGHT_FACTOR,
        div: strk_total_stake.to_amount_18_decimals(),
    )
        .unwrap();
    let btc_staking_power = if btc_total_stake.is_zero() {
        Zero::zero()
    } else {
        mul_wide_and_div(
            lhs: staker_btc_total_amount.to_amount_18_decimals(),
            rhs: BTC_WEIGHT_FACTOR,
            div: btc_total_stake.to_amount_18_decimals(),
        )
            .unwrap()
    };
    strk_staking_power + btc_staking_power
}
