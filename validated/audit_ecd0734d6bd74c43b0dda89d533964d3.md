### Title
Missing Caller Validation in `update_rewards` Allows Any Address to Permanently Deny Block Rewards to Stakers - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the staking contract is documented to be callable only by the Starknet sequencer, but the implementation contains no caller validation. Any unprivileged address can call it with `disable_rewards: true`, permanently consuming the per-block reward slot and preventing the legitimate sequencer call from distributing rewards for that block.

### Finding Description
The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` lines 1449–1507 contains no `get_caller_address()` check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ← no caller identity check here
    ...
    self.last_reward_block.write(current_block_number);
    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distribute block rewards
}
```

The guard `current_block_number > self.last_reward_block.read()` is a one-shot per-block gate. Once any caller writes `last_reward_block` to the current block, every subsequent call in that block reverts with `REWARDS_ALREADY_UPDATED`. An attacker calls `update_rewards(staker_address, disable_rewards: true)` before the sequencer, consuming the slot without distributing rewards. The sequencer's legitimate call with `disable_rewards: false` then fails, and the staker's block rewards for that block are permanently lost.

### Impact Explanation
When `disable_rewards: true` is passed and the function returns early, no rewards are minted or credited. The staker's `unclaimed_rewards_own` is never incremented for that block. Because `last_reward_block` is already set to the current block, the sequencer cannot retry. The rewards for that block are permanently unrecoverable — matching **High: Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The entry point is fully public. No token balance, staked position, or privileged role is required. The attacker only needs to submit a transaction calling `update_rewards` with `disable_rewards: true` before the sequencer does. On Starknet L2, transaction ordering within a block is controlled by the sequencer, but the function is exposed as a public ABI endpoint and can be invoked by any externally-owned account. Repeated execution across blocks can deny rewards to any targeted staker indefinitely.

### Recommendation
Add an explicit caller check at the top of `update_rewards` to enforce the documented access control:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "Caller is not the sequencer",
    );
    ...
}
```

Alternatively, restrict the function to a stored, governance-configurable sequencer address, consistent with the pattern used for `attestation_contract` validation at line 2219–2224.

### Proof of Concept

1. Staker stakes and K epochs pass so their balance is effective.
2. Consensus rewards are active (`is_pre_consensus()` returns false).
3. Attacker (any address) calls `update_rewards(staker_address, disable_rewards: true)` at block N.
4. `last_reward_block` is written to N; function returns early — no rewards distributed.
5. Sequencer calls `update_rewards(staker_address, disable_rewards: false)` at block N.
6. Assert `current_block_number > self.last_reward_block.read()` fails → `REWARDS_ALREADY_UPDATED`.
7. Staker's `unclaimed_rewards_own` remains unchanged; block N rewards are permanently lost.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
