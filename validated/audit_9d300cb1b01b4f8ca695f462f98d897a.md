### Title
`UseGlobalContract` Lacks `predecessor_id == receiver_id` Guard, Allowing Any Contract to Overwrite a Victim Account's Code — (`runtime/runtime/src/global_contracts.rs`)

---

### Summary

The NEAR runtime enforces that `predecessor_id == receiver_id` for `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, and `DeleteAccount`. `UseGlobalContract` — which has the identical effect of replacing an account's executable code — is absent from that list. Any deployed contract can therefore attach a `UseGlobalContract` action to a cross-contract promise targeting an arbitrary victim account and permanently replace the victim's contract code with an attacker-chosen global contract, at the cost of gas only.

---

### Finding Description

**Protocol invariant (documented).**
`docs/RuntimeSpec/Actions.md` lines 26-33 enumerate the actions that require `predecessor_id == receiver_id`:

> `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, `DeleteAccount`

`UseGlobalContract` is not listed. [1](#0-0) 

**Action execution — no predecessor check.**
`action_use_global_contract` in `runtime/runtime/src/global_contracts.rs` delegates immediately to `use_global_contract`, which (1) verifies the global contract exists on the shard, (2) removes the account's existing local code, and (3) writes the new `AccountContract::Global` reference — all without inspecting `predecessor_id`. [2](#0-1) 

**Action validation — no predecessor check.**
`validate_action_with_mode` in `runtime/runtime/src/action_validation.rs` dispatches `UseGlobalContract` to `validate_use_global_contract_action`, which only validates the identifier format. No `predecessor_id == receiver_id` constraint is imposed. [3](#0-2) 

**Host function — unrestricted promise target.**
`promise_batch_action_use_global_contract_impl` in `runtime/near-vm-runner/src/logic/logic.rs` computes the `sir` (self-is-receiver) flag solely for gas-pricing purposes and then unconditionally appends the action to the receipt. There is no guard that rejects a non-self promise target. [4](#0-3) 

**Dispatch in `apply_action` — no predecessor check.**
The `apply_action` dispatcher in `runtime/runtime/src/lib.rs` calls `check_actor_permissions` (which enforces `predecessor_id == receiver_id` for the five listed actions) and then dispatches `UseGlobalContract` directly to `action_use_global_contract` with no additional guard. [5](#0-4) 

**Contrast with `DeployContract`.**
`DeployContract` is in the protected list and is blocked by `check_actor_permissions` when `predecessor_id != receiver_id`. `UseGlobalContract` achieves the same state mutation (replacing the account's code pointer) but bypasses that gate entirely. [1](#0-0) 

---

### Impact Explanation

An attacker who controls any deployed contract can:

1. Deploy an arbitrary global contract (e.g., one that unconditionally transfers all funds to the attacker on every call).
2. Create a cross-contract promise targeting any victim account.
3. Attach `UseGlobalContract(<malicious_hash>)` to that promise.

Upon execution the victim's existing local code is deleted (`state_update.remove(TrieKey::ContractCode { account_id: … })`) and the account's `contract` field is set to `AccountContract::Global(malicious_hash)`. Every subsequent call to the victim's account executes the attacker's code. The victim's funds, storage, and access keys remain on the account but are now under the attacker's logic.

**Corrupted value:** `Account::contract` field of the victim — changed from the legitimate code hash to an attacker-controlled global contract identifier. [6](#0-5) 

---

### Likelihood Explanation

- Requires only a deployed contract and gas — no privileged role, no validator access, no special key.
- The `promise_batch_action_use_global_contract` host function is available to every Wasm contract with no restriction on the promise receiver.
- The global contract to be injected need only exist on the victim's shard (propagation is automatic after `DeployGlobalContract`).
- Cost to the attacker: one `DeployGlobalContract` transaction + one cross-contract call. Cost to the victim: permanent code replacement.

---

### Recommendation

Add `UseGlobalContract` (and `DeployGlobalContract` if it can similarly target foreign accounts) to the set of actions that `check_actor_permissions` restricts to `actor_id == account_id`, mirroring the existing guard for `DeployContract`. Alternatively, add an explicit check inside `action_use_global_contract` / `use_global_contract` that returns `ActionErrorKind::ActorNoPermission` when `predecessor_id != receiver_id`.

---

### Proof of Concept

```
// Attacker's contract (pseudo-code using NEAR host functions)
fn attack(victim: AccountId, malicious_hash: CryptoHash) {
    // 1. Create a promise targeting the victim
    let p = promise_batch_create(victim);
    // 2. Attach UseGlobalContract — no predecessor check prevents this
    promise_batch_action_use_global_contract(p, malicious_hash);
    // 3. Schedule the promise — victim's code is now replaced on execution
    promise_return(p);
}
```

Step-by-step:

1. Attacker calls `DeployGlobalContract` with malicious Wasm (e.g., drains `account.amount` to attacker on every call). The global contract is identified by `malicious_hash`.
2. Attacker submits a transaction to their own contract calling `attack(victim_account, malicious_hash)`.
3. The runtime executes the attacker's contract, which creates a receipt `{ predecessor: attacker_contract, receiver: victim_account, actions: [UseGlobalContract(malicious_hash)] }`.
4. The runtime applies that receipt on the victim's shard: `action_use_global_contract` removes the victim's local code and sets `account.contract = Global(malicious_hash)`.
5. All future calls to `victim_account` execute the malicious Wasm.

pub(crate) fn use_global_contract(
    state_update: &mut TrieUpdate,
    account_id: &AccountId,
    account: &mut Account,
    contract_identifier: &GlobalContractIdentifier,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let key = TrieKey::GlobalContractCode { identifier: contract_identifier.clone().into() };
    if !state_update.contains_key(&key, AccessOptions::DEFAULT)? {
        result.result = Err(ActionErrorKind::GlobalContractDoesNotExist {
            identifier: contract_identifier.clone(),
        }
        .into());
        return Ok(());
    }
    clear_account_contract_storage_usage(state_update, account_id, account)?;
    if account.contract().is_local() {
        state_update.remove(TrieKey::ContractCode { account_id: account_id.clone() });
    }
    let contract = match contract_identifier {
        GlobalContractIdentifier::CodeHash(code_hash) => AccountContract::Global(*code_hash),
        GlobalContractIdentifier::AccountId(id) => AccountContract::GlobalByAccount(id.clone()),
    };
    account.set_storage_usage(
        account.storage_usage().checked_add(contract_identifier.len() as u64).ok_or_else(|| {
            StorageError::StorageInconsistentState(format!(
                "Storage usage integer overflow for account {}",
                account_id
            ))
        })?,
    );
    account.set_contract(contract);
    Ok(())

    fn promise_batch_action_use_global_contract_impl(
        &mut self,
        promise_idx: u64,
        contract_id_ptr: GlobalContractIdentifierPtrData,
        method_name: &str,
    ) -> Result<()> {
        self.result_state.gas_counter.pay_base(base)?;
        if self.context.is_view() {
            return Err(HostError::ProhibitedInView { method_name: method_name.to_owned() }.into());
        }
        let contract_id = self.read_contract_id(contract_id_ptr)?;

        let (receipt_idx, sir) = self.promise_idx_to_receipt_idx_with_sir(promise_idx)?;

        self.pay_action_base(ActionCosts::use_global_contract_base, sir)?;
        let len = contract_id.len() as u64;
        self.pay_action_per_byte(ActionCosts::use_global_contract_byte, len, sir)?;

        self.ext.append_action_use_global_contract(receipt_idx, contract_id);
        Ok(())
    }
