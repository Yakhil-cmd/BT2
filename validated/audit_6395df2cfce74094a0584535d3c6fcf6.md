### Title
Pool Members Can Claim Rewards While Staking Contract Is Paused - (`File: src/pool/pool.cairo`)

### Summary
The `Pool` contract's `claim_rewards` function contains no check against the `Staking` contract's pause state. When a Security Agent pauses the `Staking` contract to halt all protocol activity during a security incident, pool members (delegators) can still call `Pool::claim_rewards` directly to extract their accumulated STRK rewards, bypassing the emergency stop entirely.

### Finding Description
The `Staking` contract enforces a global pause via `general_prerequisites()`, which is called at the top of every state-changing function. When `is_paused` is `true`, all staking operations revert with `"Contract is paused"`. This is confirmed by the exhaustive pause tests in `src/staking/tests/pause_test.cairo`. [1](#0-0) 

The `Pool` contract, however, has no pause awareness whatsoever. A `grep` for `is_paused` in `src/pool/pool.cairo` returns zero matches. The `Pool::claim_rewards` function calculates accumulated STRK rewards from the `cumulative_rewards_trace` and transfers them directly from the pool contract's STRK balance to the pool member's `reward_address`, with no call to the staking contract and no pause guard: [2](#0-1) 

Specifically, the transfer at the end of `claim_rewards` is a direct ERC-20 transfer from the pool contract's own balance: [3](#0-2) 

By contrast, the staking-side `claim_rewards` for stakers is properly guarded: [4](#0-3) 

The pause test suite confirms `Staking::claim_rewards` is blocked when paused, but there is no corresponding test for `Pool::claim_rewards`: [5](#0-4) 

### Impact Explanation
**High — Theft of unclaimed yield.**

When the Security Agent pauses the staking contract in response to a security incident (e.g., a reward calculation bug, an accounting exploit, or a compromised reward supplier), the intent is to freeze all protocol activity. However, pool members can still call `Pool::claim_rewards` to drain STRK rewards that were accumulated in the pool contract's balance before or during the incident. If the pause was triggered precisely because rewards were miscalculated or over-minted, pool members can extract those inflated rewards before the issue is corrected, constituting theft of unclaimed yield.

### Likelihood Explanation
**High.** Any pool member (fully unprivileged) can call `Pool::claim_rewards` at any time. The pool contract is a separate deployed contract with no dependency on the staking contract's pause flag. The attacker needs only to be a registered pool member with accrued rewards — a normal operational state for any active delegator.

### Recommendation
Add a pause check to `Pool::claim_rewards` (and other fund-moving pool functions such as `exit_delegation_pool_action`) by querying the staking contract's `is_paused()` view function before proceeding:

```cairo
fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
    // Add pause guard
    let staking_dispatcher = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking_dispatcher.is_paused(), "Contract is paused");
    // ... rest of function
}
```

Alternatively, introduce a dedicated pause component in the `Pool` contract that mirrors the staking contract's pause state, or expose a shared pause-check helper callable from pool contracts.

### Proof of Concept

1. Security Agent calls `Staking::pause()` due to a security incident.
2. `Staking::is_paused()` returns `true`.
3. Any call to `Staking::claim_rewards` reverts with `"Contract is paused"`. [5](#0-4) 
4. Attacker (a pool member with accrued rewards) calls `Pool::claim_rewards(pool_member: attacker_address)` directly on the pool contract.
5. `Pool::claim_rewards` executes without any pause check, calculates rewards from `cumulative_rewards_trace`, and transfers STRK to `reward_address`. [2](#0-1) 
6. STRK rewards are successfully extracted from the pool contract despite the staking contract being paused, bypassing the emergency stop.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
