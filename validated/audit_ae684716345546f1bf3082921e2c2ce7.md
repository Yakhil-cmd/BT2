### Title
`AnchoredPriceProvider._readSource` Uses `staticcall` for a Non-View `getBidAndAskPrice()`, Silently Failing and Permanently Bricking Swaps When Source Is a State-Writing Provider — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

### Summary

`AnchoredPriceProvider._readSource` calls the source's `getBidAndAskPrice()` via a raw `staticcall`. The `IPriceProvider` interface declares `getBidAndAskPrice()` as `external returns` (non-view). Both `ProtectedPriceProvider` and `PriceProvider` implement it as non-view because they call `IPricedOracle.price(feedId, pool)`, which emits events and writes state. When either is set as the `source`, the `staticcall` fails at the EVM level, `_readSource` returns `(false, 0, 0)`, `_computeBidAsk` returns the `(0, type(uint128).max)` sentinel, and `getBidAndAskPrice()` reverts with `FeedStalled()` — permanently breaking every swap on any pool backed by this `AnchoredPriceProvider`.

### Finding Description

`_readSource` is declared `internal view` and issues a `staticcall` to the source:

```solidity
// AnchoredPriceProvider.sol L385-L411
function _readSource(address _source)
    internal view returns (bool ok, uint256 srcBid, uint256 srcAsk)
{
    bytes4 sel = IAnchorSource.getBidAndAskPrice.selector;
    ...
    success := staticcall(SOURCE_GAS_LIMIT, _source, ptr, 0x04, ptr, 0x40)
    ...
    if (!success || retSize != 64) return (false, 0, 0);
    ...
}
```

The `IPriceProvider` interface — the canonical interface for all price providers in the system — declares `getBidAndAskPrice()` as non-view:

```solidity
// metric-core/contracts/interfaces/IPriceProvider/IPriceProvider.sol L15
function getBidAndAskPrice() external returns (uint128 bidPrice, uint128 askPrice);
```

`ProtectedPriceProvider.getBidAndAskPrice()` and `PriceProvider.getBidAndAskPrice()` both call `IPricedOracle(address(offchainOracle)).price(feedId, msg.sender)`, which emits a `PriceRead` event and writes state. The EVM's `staticcall` opcode reverts any state-modifying sub-call, so `success` is `false`. `_readSource` returns `(false, 0, 0)`. `_computeBidAsk` then returns `(0, type(uint128).max)`. `getBidAndAskPrice()` reverts with `FeedStalled()`. Every swap on the pool reverts.

The failure is silent at the factory level: `setSource` performs no view-compatibility check on the supplied address.

### Impact Explanation

Every swap on any pool whose `AnchoredPriceProvider` has a non-view source set reverts unconditionally with `FeedStalled()`. Liquidity is locked in the pool: LPs cannot exit via swaps, and the pool's core function is permanently broken until the source is replaced. This satisfies the "broken core pool functionality causing unusable swap flows" impact gate.

### Likelihood Explanation

`ProtectedPriceProvider` and `PriceProvider` both implement `IPriceProvider` — the same interface `AnchoredPriceProvider` itself implements. Chaining one provider as the source of another is the natural composition pattern. `setSource` accepts any address without a view-compatibility probe. A pool admin or APP factory operator setting a `ProtectedPriceProvider` as the source is a realistic, non-adversarial action that silently bricks the pool.

### Recommendation

Replace the `staticcall` in `_readSource` with a regular `call` (matching the non-view declaration of `getBidAndAskPrice()`), or add a `staticcall` probe during `setSource` to reject sources whose `getBidAndAskPrice()` reverts under `staticcall` before they are stored. The simplest fix is to change `_readSource` to use `call` instead of `staticcall`, since `_getBidAndAskPrice` (its only caller) is already non-view.

### Proof of Concept

1. Deploy a `ProtectedPriceProvider` for a valid feed.
2. Deploy an `AnchoredPriceProvider` with `MUTABLE_PARAMS = true` and `source = address(0)`.
3. Call `setSource(address(protectedPriceProvider))` from the APP factory.
4. Call `getBidAndAskPrice()` on the `AnchoredPriceProvider`.
5. Observe: `_readSource` issues `staticcall` → `ProtectedPriceProvider.getBidAndAskPrice()` calls `oracle.price()` which emits an event → EVM reverts the staticcall → `success = false` → `_readSource` returns `(false, 0, 0)` → `_computeBidAsk` returns `(0, type(uint128).max)` → `getBidAndAskPrice()` reverts with `FeedStalled()`.

    function _readSource(address _source)
        internal view returns (bool ok, uint256 srcBid, uint256 srcAsk)
    {
        bytes4 sel = IAnchorSource.getBidAndAskPrice.selector;
        bool success;
        uint256 retSize;
        uint256 b;
        uint256 a;
        assembly ("memory-safe") {
            // Scratch beyond the free-memory pointer; never updated, so this is memory-safe.
            let ptr := mload(0x40)
            mstore(ptr, sel) // 4-byte selector, left-aligned
            // Input is consumed before output is written, so in/out may share ptr. Output is capped
            // at 0x40 bytes: a larger returndata is NOT copied (only returndatasize() reports it).
            success := staticcall(SOURCE_GAS_LIMIT, _source, ptr, 0x04, ptr, 0x40)
            retSize := returndatasize()
            b := mload(ptr)
            a := mload(add(ptr, 0x20))
        }
        if (!success || retSize != 64) return (false, 0, 0);

        srcBid = b;
        srcAsk = a;
        if (srcBid == 0 || srcBid >= srcAsk || srcAsk > type(uint128).max) return (false, 0, 0);

        return (true, srcBid, srcAsk);
    }
