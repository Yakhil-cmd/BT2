### Title
Missing Caller Authorization in `update_rewards` Allows Any Address to Permanently Suppress Staker Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is specified to be callable only by the Starkware sequencer, but the implementation contains no such access control check. Any unprivileged address can call `update_rewards` with `disable_rewards: true` on every block, permanently consuming each block's reward slot without distributing any rewards, causing permanent loss of unclaimed yield for all stakers and delegators.

---

### Finding Description

The protocol specification for `update_rewards` explicitly states:

> **access control:** Only starkware sequencer. [1](#0-0) 

However, the implementation in `StakingRewardsManagerImpl` only calls `self.general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — it does **not** verify the caller is the authorized sequencer: [2](#0-1) 

The `general_prerequisites` function: [3](#0-2) 

The critical state mutation is at line 1485: `last_reward_block` is written to the current block number **before** the `disable_rewards` early-return guard: [4](#0-3) 

The guard that prevents double-processing a block checks `current_block_number > self.last_reward_block.read()`: [5](#0-4) 

Because `last_reward_block` is a **single global value** (not per-staker), once any caller consumes a block with `disable_rewards: true`, that block's rewards are permanently unrecoverable — no subsequent call can process it.

The `UndelegateIntentKey` struct shows the analogous ownership-scoping pattern used correctly elsewhere (keyed by `pool_contract`), but `update_rewards` has no equivalent caller binding: [6](#0-5) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` on every block permanently prevents reward distribution for all stakers and all delegation pools for those blocks. The rewards are never minted/transferred and cannot be recovered. This affects every staker and every pool member in the protocol simultaneously.

---

### Likelihood Explanation

**High.** The function is publicly callable by any non-zero address with no economic barrier. The attack requires only a single transaction per block and can be fully automated. There is no cost to the attacker beyond gas fees, and the damage is irreversible.

---

### Recommendation

Add an access control check at the top of `update_rewards` to enforce that only the authorized sequencer address (or a designated role) can call it, consistent with the specification. For example:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
}
```

Alternatively, store the sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation.

---

### Proof of Concept

1. Attacker monitors the Starknet mempool/block production.
2. On every new block, attacker calls:
   ```
   staking.update_rewards(any_active_staker_address, disable_rewards: true)
   ```
3. Inside `update_rewards`, `general_prerequisites()` passes (attacker is non-zero, contract is unpaused).
4. `last_reward_block` is written to `current_block_number` at line 1485.
5. The `disable_rewards || self.is_pre_consensus()` branch at line 1487 triggers an early return — no rewards are calculated or transferred.
6. Any legitimate sequencer call for the same block now reverts with `REWARDS_ALREADY_UPDATED` (line 1454–1458).
7. Repeating this every block permanently zeroes out all staker and pool rewards. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
