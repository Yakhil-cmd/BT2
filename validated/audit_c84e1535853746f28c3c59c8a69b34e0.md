Audit Report

## Title
Duplicate `StrippedIngressMessage` entries in crafted `StrippedBlockProposal` cause unbounded JoinSet spawning and forced `AssembleResult::Unwanted`, blocking block assembly — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/assembler.rs`)

## Summary
`TryFrom<pb::StrippedBlockProposal> for StrippedBlockProposal` collects `ingress_messages` into a `Vec` with no uniqueness check. A Byzantine block maker can serve a crafted `StrippedBlockProposal` containing N duplicate `SignedIngressId` entries. `assemble_message` spawns one unbounded `get_or_fetch` Tokio task per entry; when the second task for a duplicate ID completes, `try_insert` returns `InsertionError::AlreadyInserted`, which `assemble_message` treats as fatal and immediately returns `AssembleResult::Unwanted`, permanently aborting block assembly for that height on the affected replica.

## Finding Description
**Root cause — no deduplication on deserialization:**
`TryFrom<pb::StrippedBlockProposal>` at `stripped.rs` lines 100–108 maps `value.ingress_messages` directly into a `Vec<SignedIngressId>` with no uniqueness check. A crafted proto with N identical entries deserializes into N identical `SignedIngressId` values. [1](#0-0) 

**Duplicate entries preserved through `BlockProposalAssembler::new`:**
`BlockProposalAssembler::new` maps each `SignedIngressId` to `(id, None)` without deduplication, so all N duplicate entries appear in `self.ingress_messages`. [2](#0-1) 

**All N entries returned as missing:**
`missing_stripped_messages()` returns every entry where the slot is `None`; with N duplicates all slots start as `None`, so N tasks are queued. [3](#0-2) 

**Unbounded JoinSet spawning:**
`assemble_message` spawns one `get_or_fetch` task per entry with no size cap. [4](#0-3) 

**`AlreadyInserted` on second duplicate task:**
`try_insert` uses `find` (first-match linear scan). After the first task fills the first matching slot, the second task for the same ID finds that slot already `Some` and returns `InsertionError::AlreadyInserted`. The second duplicate slot (also `None`) is never reached because `find` always returns the first match. [5](#0-4) 

**Fatal treatment of `AlreadyInserted`:**
`assemble_message` treats any `InsertionError` as fatal and returns `AssembleResult::Unwanted`, bypassing `try_assemble()` and its ID-mismatch check entirely. [6](#0-5) 

The developer comment "We can have at most 1000 elements in the vector" confirms the expected bound, but it is never enforced. [7](#0-6) 

## Impact Explanation
A Byzantine block maker below the consensus fault threshold can cause every receiving replica to return `AssembleResult::Unwanted` for their block at a given height, preventing that block from ever being assembled. Consensus must wait for the next block maker rotation. If the Byzantine node holds the block maker role repeatedly (deterministic rotation), it can stall subnet progress for multiple consecutive heights. This matches the allowed High impact: **Application/platform-level DoS / consensus blocking / subnet availability impact not based on raw volumetric DDoS** ($2,000–$10,000).

## Likelihood Explanation
The block maker role rotates deterministically. A single Byzantine node below the fault threshold that holds the block maker role for any round can execute this attack with no threshold corruption, key compromise, or external infrastructure required. The crafted `StrippedBlockProposal` passes all current deserialization checks. The attack is repeatable every time the Byzantine node holds the block maker role.

## Recommendation
1. **Deduplicate on deserialization**: In `TryFrom<pb::StrippedBlockProposal>`, reject the message (return `ProxyDecodeError`) if any duplicate `SignedIngressId` or `(dealer_index, IDkgArtifactId)` entries are present.
2. **Cap JoinSet size**: Before spawning tasks in `assemble_message`, assert or enforce that `stripped_message_ids.len()` does not exceed the protocol-level ingress payload limit (~1000).
3. **Fix `AlreadyInserted` handling**: Treat `InsertionError::AlreadyInserted` as a no-op (the slot is already filled by a concurrent task for the same ID) rather than as a fatal error, so duplicate tasks do not abort a legitimately assembled block.

## Proof of Concept
```rust
// Craft a StrippedBlockProposal with 10_000 duplicate ingress IDs
let ingress_id = fake_ingress_message("dup").id(); // StrippedMessageId::Ingress
let ids = vec![ingress_id; 10_000];
let stripped = fake_stripped_block_proposal_with_messages(ids);

// Serialize and deserialize — passes all current checks
let proto = pb::StrippedBlockProposal::from(stripped.clone());
let deserialized = StrippedBlockProposal::try_from(proto).unwrap();

// BlockProposalAssembler::new creates 10_000 (id, None) entries
let assembler = BlockProposalAssembler::new(deserialized);
// missing_stripped_messages() returns 10_000 entries
assert_eq!(assembler.missing_stripped_messages().len(), 10_000);

// assemble_message spawns 10_000 tasks with no cap
// When any two tasks complete for the same ID:
//   first → Ok(()), second → Err(InsertionError::AlreadyInserted)
//   assemble_message returns AssembleResult::Unwanted immediately
// try_assemble() ID-mismatch check is never reached
```
This can be exercised as a unit test using the existing `fake_stripped_block_proposal_with_messages` test utility already present in the codebase.

    pub(crate) fn missing_stripped_messages(&self) -> Vec<StrippedMessageId> {
        let ingress_messages = PayloadAssembler::<SignedIngress>::missing_artifacts(self)

        let idkg_dealings = PayloadAssembler::<SignedIDkgDealing>::missing_artifacts(self)
