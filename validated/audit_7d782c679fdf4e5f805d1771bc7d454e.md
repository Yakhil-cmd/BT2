### Title
Unbounded `move_to_block` Loop in `load_hashchain` Causes Permanent Contract Freeze After Extended Inactivity — (`engine/src/hashchain.rs` / `engine-hashchain/src/hashchain.rs`)

---

### Summary

The `load_hashchain` function unconditionally calls `hashchain.move_to_block(current_block_height)` to catch up the stored hashchain to the current NEAR block height. `move_to_block` iterates **once per block** in the gap, executing a `keccak` hash per iteration. After a sufficiently long period of inactivity, this loop exceeds NEAR's per-transaction gas limit, causing every wrapped contract method — including `pause_contract` — to fail permanently. Because `start_hashchain` (the only reset path) requires the contract to be paused first, and `pause_contract` itself fails, the contract enters an irrecoverable frozen state.

---

**The deadlock:**

| Step | Action | Result |
|---|---|---|
| 1 | Gap grows large (N blocks of inactivity) | Hashchain is N blocks behind |
| 2 | Any user calls `submit`, `pause_contract`, etc. | `with_hashchain` → `load_hashchain` → `move_to_block(N)` → out-of-gas |
| 3 | Admin calls `pause_contract` to recover | Same path, same out-of-gas failure |
| 4 | Admin calls `start_hashchain` | Fails: `require_paused` — contract is still running |
| 5 | All paths exhausted | Contract permanently frozen |

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

All ETH and ERC-20 token balances held inside Aurora become permanently inaccessible. No EVM transaction (`submit`), no bridge withdrawal (`withdraw`), no administrative recovery (`pause_contract`) can succeed once the gas threshold is crossed. A contract upgrade is the only theoretical escape, but that path also goes through `with_hashchain`-wrapped methods for staging.

---

### Likelihood Explanation

NEAR produces approximately one block per second. At that rate:

- If each `keccak` + loop iteration costs ~1 MGas, and NEAR's per-transaction gas cap is 300 TGas, the threshold is roughly **300,000 blocks ≈ 3.5 days** of zero Aurora activity.
- Aurora is a live mainnet chain with continuous activity, so organic inactivity of this duration is unlikely but not impossible (e.g., during a prolonged incident, a competing migration, or a targeted griefing campaign that reverts every submitted transaction for several days).
- The vulnerability requires no special privilege — any period of inactivity, however caused, triggers it.

---

### Recommendation

Cap the number of blocks `move_to_block` advances in a single call, or switch to a lazy/checkpoint design that does not require iterating through every skipped block. For example:

```rust
// In load_hashchain: skip intermediate blocks, jump directly
hashchain.jump_to_block(block_height); // O(1) instead of O(N)
```

Alternatively, `pause_contract` should bypass `with_hashchain` (or use a separate, non-hashchain-dependent path) so that the admin can always pause and then call `start_hashchain` to reset the hashchain to the current block height.

---

### Proof of Concept

1. Deploy Aurora Engine with hashchain enabled (`NewCallArgsV4` with `initial_hashchain = Some([0u8; 32])`).
2. Record the current NEAR block height `H0`.
3. Submit no Aurora transactions for `N` NEAR blocks (where `N` exceeds the gas threshold, e.g., 400,000 blocks).
4. At block `H0 + N`, any caller submits any Aurora transaction (e.g., a simple ETH transfer via `submit`).
5. Execution path: `with_hashchain` → `load_hashchain` → `move_to_block(H0 + N)` → loop runs N times → NEAR out-of-gas error.
6. Admin attempts `pause_contract` — same failure.
7. Admin attempts `start_hashchain` — fails with `require_paused`.
8. All user funds are permanently frozen. [1](#0-0) [6](#0-5) [4](#0-3) [7](#0-6)

    pub fn move_to_block(
        &mut self,
        next_block_height: u64,
    ) -> Result<(), BlockchainHashchainError> {
        if next_block_height <= self.current_block_height {
            return Err(BlockchainHashchainError::BlockHeightIncorrect);
        }

        while self.current_block_height < next_block_height {

            );

            self.block_hashchain_computer.clear_txs();
            self.current_block_height += 1;
        }

        Ok(())

pub fn start_hashchain<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    let mut state = state::get_state(&io)?;
    require_paused(&state)?;
    require_key_manager_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let args = StartHashchainArgs::try_from_slice(&input).map_err(|_| errors::ERR_SERIALIZE)?;
    let block_height = env.block_height();

    // Starting hashchain must be for an earlier block
    if block_height < args.block_height {
        return Err(errors::ERR_ARGS.into());
    }

    let mut hashchain = Hashchain::new(
        state.chain_id,
        env.current_account_id(),
        args.block_height + 1,
        args.block_hashchain,
    );

    if hashchain.get_current_block_height() < block_height {
        hashchain.move_to_block(block_height)?;
    }
