### Title
Recreated EVM Account State Silently Dropped After Self-Destruct in Same Transaction — (`File: fvm/evm/emulator/state/stateDB.go`)

---

### Summary

In `StateDB.Commit()`, when an EVM account is both self-destructed and recreated (via `CREATE2`) within the same transaction, the commit logic deletes the old account but never writes the new account's state to `BaseView`. The recreated contract's code, storage slots, and ETH balance are permanently lost.

---

### Finding Description

`StateDB.Commit()` processes dirty addresses in two phases: first it checks `HasSelfDestructed`, and if true, calls `baseView.DeleteAccount` and sets `deleted = true`, then immediately `continue`s — skipping the `IsCreated` check entirely.

```go
// fvm/evm/emulator/state/stateDB.go
for _, addr := range sortedAddresses {
    deleted := false
    if db.HasSelfDestructed(addr) {          // true for recreated accounts
        err = db.baseView.DeleteAccount(addr) // old account deleted
        deleted = true
    }
    if deleted {
        continue  // <-- IsCreated check is NEVER reached
    }
    ...
    if db.IsCreated(addr) {                  // never executed for recreated accounts
        err = db.baseView.CreateAccount(...)
        continue
    }
```

The `recreated` path is set up in `DeltaView.CreateAccount()`: when `CreateAccount` is called on an address that already exists, it calls `SelfDestruct` (adding the address to `toBeDestructed`) and then adds it to `created`. After this, both `HasSelfDestructed` and `IsCreated` return `true` for that address.

```go
// fvm/evm/emulator/state/delta.go
func (d *DeltaView) CreateAccount(addr gethCommon.Address) error {
    ...
    if exist {
        destructed, balance := d.HasSelfDestructed(addr)
        if !destructed {
            err = d.SelfDestruct(addr)  // sets toBeDestructed[addr]
        }
        d.recreated[addr] = struct{}{}
        ...
    }
    d.created[addr] = struct{}{}  // also sets created[addr]
    return nil
}
```

The slot update loop in `Commit()` also skips recreated accounts:

```go
for _, sk := range sortedSlots {
    if db.HasSelfDestructed(sk.Address) {
        continue  // new slots for recreated account are also dropped
    }
```

---

### Impact Explanation

When a transaction:
1. Creates a contract at address A (via `CREATE2`)
2. Self-destructs it (EIP-6780 `SelfDestruct6780` applies since it was created in the same tx)
3. Recreates it at address A (same `CREATE2` salt)

The recreated contract's new code, storage, and any ETH balance accumulated after recreation are permanently lost. `BaseView` ends up with the address deleted rather than recreated. Any ETH sent to the recreated contract address within the same transaction is unrecoverable — a direct cross-VM asset loss.

---

### Likelihood Explanation

This is reachable by any unprivileged EVM transaction sender. The pattern of self-destruct + `CREATE2` redeploy at the same address is a known EVM pattern used in upgradeable contract designs. A malicious or buggy factory contract can trigger this path. No privileged access, staked node control, or key compromise is required — only the ability to submit an EVM transaction.

---

### Recommendation

In `StateDB.Commit()`, after calling `baseView.DeleteAccount` for a self-destructed address, check whether the address is also marked as `IsCreated` (i.e., it was recreated in the same transaction). If so, proceed to call `baseView.CreateAccount` with the new state instead of `continue`ing:

```go
if db.HasSelfDestructed(addr) {
    err = db.baseView.DeleteAccount(addr)
    deleted = true
}
if deleted {
    if db.IsCreated(addr) {
        // handle recreated account — fall through to CreateAccount below
    } else {
        continue
    }
}
```

The slot update loop must similarly be updated to allow new slots for recreated (not merely self-destructed) accounts to be written.

---

### Proof of Concept

1. Deploy a factory contract on Flow EVM.
2. The factory calls `CREATE2` to deploy a child contract at a deterministic address A. The child contract has a `destroy()` function that calls `SELFDESTRUCT` (EIP-6780 applies since it was created in this tx).
3. In the same transaction, the factory calls `destroy()` on A, then sends 1 ETH to A, then calls `CREATE2` again with the same salt to redeploy at A.
4. Within the transaction, the EVM correctly sees A as a new contract with 1 ETH balance.
5. After `StateDB.Commit()` is called, `HasSelfDestructed(A)` returns `true` → `DeleteAccount(A)` is called → `continue` skips `IsCreated` → the new contract is never written to `BaseView`.
6. After the transaction, address A does not exist in `BaseView`. The 1 ETH is permanently lost.

func (d *DeltaView) CreateAccount(addr gethCommon.Address) error {
	// if is already created return
	if d.IsCreated(addr) {
		return nil
	}
	exist, err := d.Exist(addr)
	if err != nil {
		return err
	}
	if exist {
		// check if already destructed
		destructed, balance := d.HasSelfDestructed(addr)
		if !destructed {
			balance, err = d.GetBalance(addr)
			if err != nil {
				return err
			}
			err = d.SelfDestruct(addr)
			if err != nil {
				return err
			}
		}

		d.nonces[addr] = 0
		d.codes[addr] = nil
		d.codeHashes[addr] = gethTypes.EmptyCodeHash
		// carrying over the balance. (legacy behavior of the Geth stateDB)
		d.balances[addr] = balance

		// flag addr as recreated, this flag helps with postponing deletion of slabs
		// otherwise we have to iterate over all slabs of this account and set the to nil
		d.recreated[addr] = struct{}{}

		// remove slabs from cache related to this account
		for k := range d.slots {
			if k.Address == addr {
				delete(d.slots, k)
			}
		}
	}
	d.dirtyAddresses[addr] = struct{}{}
	d.created[addr] = struct{}{}
	return nil
}

// DeleteAccount deletes an account's meta data, code, and
// storage slots associated with that address
func (v *BaseView) DeleteAccount(addr gethCommon.Address) error {
	// 1. check account exists
	acc, err := v.getAccount(addr)
	if err != nil {
		return err
	}
	if acc == nil { // if account doesn't exist return
		return nil
	}

	// 2. remove the code
	if acc.HasCode() {
		err = v.updateAccountCode(addr, nil, gethTypes.EmptyCodeHash)
		if err != nil {
			return err
		}
	}

	// 3. update the cache
	delete(v.cachedAccounts, addr)

	// 4. collections
	err = v.accounts.Remove(addr.Bytes())
	if err != nil {
		return err
	}

	// 5. remove storage slots
	if len(acc.CollectionID) > 0 {
		col, found := v.slots[addr]
		if !found {
			col, err = v.collectionProvider.CollectionByID(acc.CollectionID)
			if err != nil {
				return err
			}
		}
		// delete all slots related to this account (eip-6780)
		keys, err := col.Destroy()
		if err != nil {
			return err
		}

		delete(v.slots, addr)

		for _, key := range keys {
			delete(v.cachedSlots, types.SlotAddress{
				Address: addr,
				Key:     gethCommon.BytesToHash(key),
			})
		}
	}
	return nil
}
