### Title
Missing Caller Restriction in `update_rewards` Allows Griefing of Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function is specified in the protocol spec to be callable **only by the Starkware sequencer**, but the implementation enforces no such restriction. Any non-zero address can call it with `disable_rewards: true`, which advances the global `last_reward_block` checkpoint without distributing rewards, causing the sequencer's legitimate call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

### Finding Description
The vulnerability class is a **missing caller-restriction check in a privileged reward-distribution function** — a direct analog to the external report's missing blacklist check in `custodianMint`/`excessStakeMint`/`redeem`.

**The check exists for the analogous function but is absent here:**

`update_rewards_from_attestation_contract` correctly enforces its caller restriction: [1](#0-0) 

```cairo
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), ...);
    self.assert_caller_is_attestation_contract();   // ← caller check present
    ...
}
```

`update_rewards` calls only `general_prerequisites()` (pause + non-zero caller), with **no sequencer-identity check**: [2](#0-1) 

The spec explicitly documents the intended restriction: [3](#0-2) 

> **access control**: Only starkware sequencer.

The critical ordering flaw: `last_reward_block` is written to storage **before** the `disable_rewards` branch:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← written unconditionally

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing
}
``` [4](#0-3) 

`last_reward_block` is a **global** (not per-staker) checkpoint: [5](#0-4) 

### Impact Explanation
An attacker calls `update_rewards(any_active_staker, disable_rewards: true)` once per block. This:
1. Advances `last_reward_block` to the current block without distributing any rewards.
2. Causes the sequencer's subsequent call (`disable_rewards: false`) to revert on `current_block_number > self.last_reward_block.read()` — because `N > N` is false.
3. All stakers and pool members receive zero block rewards for that block.

Sustained across blocks, this constitutes **permanent freezing of unclaimed yield** for all stakers and their delegators. The attacker bears only the gas cost of one transaction per block; on Starknet this is negligible relative to the aggregate rewards denied.

**Allowed impact matched**: High — Permanent/Temporary freezing of unclaimed yield.

### Likelihood Explanation
The entry path is fully unprivileged: `general_prerequisites()` only checks `is_paused` and `caller != zero`. Any externally-owned account or contract can call `update_rewards`. The only precondition is supplying any currently-active staker address, which is publicly readable from the `stakers` vector. No special knowledge, role, or capital is required.

### Recommendation
Add a sequencer-identity assertion inside `update_rewards`, mirroring the pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer();   // add this check
    ...
}
```

The sequencer address should be stored in contract storage (similar to `attestation_contract`) and set during construction or via a governance-controlled setter.

### Proof of Concept
1. Consensus rewards are active (`!is_pre_consensus()`).
2. Attacker identifies any active staker `S` from the public `stakers` vector.
3. In block `N`, attacker submits: `update_rewards(S, disable_rewards: true)`.
4. `last_reward_block` is written to `N`; no rewards are distributed.
5. Sequencer submits `update_rewards(block_proposer, disable_rewards: false)` in block `N`.
6. Check `N > N` fails → reverts with `REWARDS_ALREADY_UPDATED`.
7. Block proposer and all pool members receive zero rewards for block `N`.
8. Repeat each block to permanently suppress all consensus reward distribution.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
