### Title
Missing Access Control on `update_rewards` Allows Anyone to Permanently Deny Staker Rewards - (File: `src/staking/staking.cairo`)

### Summary

`StakingRewardsManagerImpl::update_rewards` in `staking.cairo` is specified as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block reward slot (by writing `last_reward_block`) without distributing any rewards. Because `last_reward_block` is a global variable, a single such call per block permanently denies rewards to all stakers for that block.

### Finding Description

The spec at `docs/spec.md` line 1645 states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo` lines 1449–1507 contains no such check:

```cairo
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // only checks pause
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... staker existence / activity checks ...
        self.last_reward_block.write(current_block_number);   // consumes the slot
        if disable_rewards || self.is_pre_consensus() {
            return;                                            // exits without distributing
        }
        // ... reward distribution ...
    }
}
```

`last_reward_block` is a single global storage variable (not per-staker). The guard `current_block_number > self.last_reward_block.read()` means only **one** successful call to `update_rewards` is possible per block, for any staker. Once the slot is consumed, every subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

**Attack path:**

1. Attacker picks any valid, active staker address (publicly readable from chain state).
2. In each block, attacker calls `update_rewards(any_staker, disable_rewards: true)`.
3. `last_reward_block` is written to the current block; the early-return at line 1487 fires before any reward calculation or transfer.
4. The sequencer's legitimate `update_rewards` call for the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers lose rewards for that block. Repeated every block → permanent, total denial of consensus-era rewards.

### Impact Explanation

**High — Permanent freezing / theft of unclaimed yield.**

Once consensus rewards are active (`!self.is_pre_consensus()`), every block that the attacker front-fills yields zero rewards for every staker and every delegation pool. Rewards are never minted for those blocks; they are permanently lost, not merely delayed. The attacker needs no stake, no privileged key, and spends only gas.

### Likelihood Explanation

**High.** The entry point is a public external function with no caller restriction. The attacker needs only a funded address and the address of any active staker (trivially obtained from on-chain events). The attack is repeatable every block at negligible cost.

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`, consistent with the spec. For example:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
}
```

Alternatively, if the sequencer address is not stored, use Starknet's `get_sequencer_address()` syscall.

### Proof of Concept

```
// Attacker calls once per block, for any valid staker:
IStakingRewardsManagerDispatcher { contract_address: staking }
    .update_rewards(staker_address: any_active_staker, disable_rewards: true);

// Sequencer's legitimate call in the same block now reverts:
// panics with Error::REWARDS_ALREADY_UPDATED
IStakingRewardsManagerDispatcher { contract_address: staking }
    .update_rewards(staker_address: any_staker, disable_rewards: false);
```

The existing test suite at `src/staking/tests/test.cairo` lines 3878–3884 already demonstrates that a second call in the same block reverts with `REWARDS_ALREADY_UPDATED`, confirming the slot-consumption mechanic. The missing piece — that the first call can be made by anyone, not just the sequencer — is the root cause. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
