### Title
No deadline or max-commission guard in `switch_delegation_pool()` allows stale execution at adversarially-raised commission rates - (File: src/pool/pool.cairo)

### Summary

`switch_delegation_pool()` in `src/pool/pool.cairo` accepts a target staker and pool but carries no deadline parameter and no `max_commission` guard. Because a staker can legitimately raise their commission at any time while an active `commission_commitment` is in force (via `set_commission()` in `src/staking/staking.cairo`), a delegator's pending switch transaction can be executed—or front-run—after the target pool's commission has been raised to an arbitrarily high value, silently redirecting the delegator's future yield to the staker.

---

### Finding Description

`switch_delegation_pool()` is the pool-level entry point that moves a delegator's funds from one staker's pool to another:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
) -> Amount {
    assert!(amount.is_non_zero(), ...);
    let pool_member = get_caller_address();
    let mut pool_member_info = self.internal_pool_member_info(:pool_member);
    assert!(pool_member_info.unpool_time.is_some(), ...);
    assert!(amount <= pool_member_info.unpool_amount, ...);
    // ← no deadline check, no max_commission check
    ...
``` [1](#0-0) 

The function verifies only that an undelegate intent exists and that the requested amount does not exceed the intent amount. It never inspects the commission of `to_pool` at execution time, and it accepts no caller-supplied deadline or `max_commission` bound.

The commission increase path is fully reachable. `update_commission()` in the staking contract permits a staker to **raise** their commission to any value up to `commission_commitment.max_commission` while a commitment is active:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(commission <= commission_commitment.max_commission, ...);
    assert!(commission != old_commission, ...);
``` [2](#0-1) 

A staker can set `max_commission = 10000` (100 %) via `set_commission_commitment()`, then call `set_commission(10000)` at any time during the commitment window. [3](#0-2) 

---

### Impact Explanation

Once `switch_delegation_pool()` executes, the delegator is registered in the target pool at whatever commission the staker currently charges. All future rewards accrued by the delegator in that pool are split according to the new, higher commission. Because the switch is irreversible (the undelegate intent is consumed), the delegator has no recourse. If the staker raised commission to 100 %, the delegator receives zero yield going forward—their entire unclaimed yield stream is redirected to the staker.

This matches the **High: Theft of unclaimed yield** impact category.

---

### Likelihood Explanation

The preconditions are realistic and attacker-controlled:

1. A staker calls `set_commission_commitment(max_commission=10000, expiration_epoch=current+N)` — a permissionless, unprivileged call available to any staker with a pool.
2. A delegator, observing the staker's current low commission, calls `exit_delegation_pool_intent()` and then submits `switch_delegation_pool()`.
3. Before the switch transaction is included, the staker calls `set_commission(10000)`.
4. The switch executes at 100 % commission.

On Starknet L2 the sequencer is not strictly FIFO and network congestion events (analogous to the Arbitrum airdrop scenario cited by the original judge) can delay user transactions while the staker's commission-raise lands first. The staker also has an economic incentive to time this precisely.

---

### Recommendation

Add a `max_commission: Commission` parameter to `switch_delegation_pool()` (and symmetrically to `enter_delegation_pool()` and `add_to_delegation_pool()`). Before completing the switch, assert that the target pool's current commission does not exceed the caller-supplied bound:

```cairo
fn switch_delegation_pool(
    ref self: ContractState,
    to_staker: ContractAddress,
    to_pool: ContractAddress,
    amount: Amount,
    max_commission: Commission,   // ← new
) -> Amount {
    ...
    let actual_commission = self.get_commission_from_staking_contract();
    assert!(actual_commission <= max_commission, Error::COMMISSION_TOO_HIGH);
    ...
}
```

Optionally, also accept a `deadline: Timestamp` parameter and assert `Time::now() <= deadline` to guard against stale execution independent of commission changes.

---

### Proof of Concept

**Setup:**
- Staker B deploys with commission 5 % (500).
- Staker B calls `set_commission_commitment(max_commission=10000, expiration_epoch=current+100)`.

**Step 1 – Delegator Alice initiates switch:**
- Alice is currently in Pool A and calls `exit_delegation_pool_intent(amount=1000)`.
- Alice submits `switch_delegation_pool(to_staker=B, to_pool=B_pool, amount=1000)`.
- The transaction enters the mempool / sequencer queue.

**Step 2 – Staker B front-runs:**
- Staker B observes Alice's pending transaction.
- Staker B calls `set_commission(commission=10000)` — valid because the commitment allows it.
- This transaction is included before Alice's.

**Step 3 – Alice's switch executes:**
- `switch_delegation_pool()` runs. No commission check exists.
- Alice is registered in Pool B at 100 % commission.

**Step 4 – Yield theft:**
- All future rewards attributed to Alice's 1000-unit stake flow entirely to Staker B as commission.
- Alice's `claim_rewards()` returns zero for every subsequent epoch.

The root cause is the absence of any commission or deadline guard in `switch_delegation_pool()` at `src/pool/pool.cairo` lines 379–429, combined with the legitimate commission-raise path in `update_commission()` at `src/staking/staking.cairo` lines 1573–1609. [4](#0-3) [5](#0-4)

        fn switch_delegation_pool(
            ref self: ContractState,
            to_staker: ContractAddress,
            to_pool: ContractAddress,
            amount: Amount,
        ) -> Amount {
            // Asserts.
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            assert!(
                pool_member_info.unpool_time.is_some(),
                "{}",
                GenericError::MISSING_UNDELEGATE_INTENT,
            );
            assert!(amount <= pool_member_info.unpool_amount, "{}", GenericError::AMOUNT_TOO_HIGH);
            let reward_address = pool_member_info.reward_address;

            // Update pool_member_info and write to storage.
            pool_member_info.unpool_amount -= amount;
            if pool_member_info.unpool_amount.is_zero() {
                // unpool_amount is zero, clear unpool_time.
                pool_member_info.unpool_time = Option::None;
            }
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Serialize the switch pool data and invoke the staking contract to switch pool.
            let switch_pool_data = SwitchPoolData { pool_member, reward_address };
            let mut serialized_data = array![];
            switch_pool_data.serialize(ref output: serialized_data);
            self
                .staking_pool_dispatcher
                .read()
                .switch_staking_delegation_pool(
                    :to_staker,
                    :to_pool,
                    switched_amount: amount,
                    data: serialized_data.span(),
                    identifier: pool_member.into(),
                );

            // Emit event.
            self
                .emit(
                    Events::SwitchDelegationPool {
                        pool_member, new_delegation_pool: to_pool, amount,
                    },
                );

            pool_member_info.unpool_amount
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,

                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);

            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
