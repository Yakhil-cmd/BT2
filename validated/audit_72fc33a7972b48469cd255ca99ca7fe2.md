### Title
Unprivileged Caller Can Permanently Freeze Block Rewards via `update_rewards` with `disable_rewards: true` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and unconditionally writes `last_reward_block` before checking the `disable_rewards` flag. Any unprivileged caller can invoke `update_rewards(any_valid_staker, disable_rewards: true)` once per block, consuming the block's reward slot without distributing any rewards. Because `last_reward_block` is a global singleton, this permanently blocks all legitimate reward distribution for that block.

---

### Finding Description

`update_rewards` is exposed as a public ABI function under `StakingRewardsManagerImpl`:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // only checks: not paused, caller != 0
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // Update last block rewards.  ← written BEFORE the disable_rewards check
        self.last_reward_block.write(current_block_number);

        if disable_rewards || self.is_pre_consensus() {
            return;                    // ← early exit, no rewards distributed
        }
        ...
    }
}
``` [1](#0-0) 

`general_prerequisites` enforces only two conditions: the contract is not paused and the caller is not the zero address. [2](#0-1) 

`last_reward_block` is a **global** storage variable — not per-staker — so a single call per block exhausts the reward slot for every staker. [3](#0-2) 

The attack path is:

1. Attacker calls `update_rewards(staker_address: <any active staker>, disable_rewards: true)` as the first transaction of each block.
2. `last_reward_block` is written to `current_block_number` (line 1485).
3. The function hits the `disable_rewards` branch and returns without distributing any rewards.
4. Every subsequent call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number > last_reward_block` is now false.
5. All stakers receive zero block rewards for that block.
6. The attacker repeats this every block at the cost of a single cheap transaction.

The only precondition is that the attacker supplies a valid, active staker address with non-zero balance — trivially satisfied by reading any existing staker from on-chain state. [4](#0-3) 

---

### Impact Explanation

This matches **"Permanent freezing of unclaimed yield"** from the allowed impact scope. Block rewards are the primary income stream for stakers in the consensus-rewards phase. An attacker who front-runs the legitimate `update_rewards` call every block causes all stakers to accrue zero rewards indefinitely. The funds are not stolen — they simply are never credited — which is a permanent freeze of yield.

---

### Likelihood Explanation

- **No privilege required**: `general_prerequisites` imposes no role check.
- **No capital required**: the attacker spends only gas (one Starknet transaction per block).
- **Fully automatable**: a bot can submit the griefing transaction at the start of every block.
- **No external conditions**: the attack works as long as at least one active staker exists, which is always true in a live network.

---

### Recommendation

Two independent fixes, either of which closes the vulnerability:

1. **Add access control**: restrict `update_rewards` to a trusted caller (e.g., the consensus/sequencer role) so that arbitrary accounts cannot invoke it.

2. **Move the `last_reward_block` write after the early-exit guard**:

```cairo
// Only consume the block slot if rewards are actually going to be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);  // moved here
...
```

This ensures that a `disable_rewards: true` call does not consume the block's reward slot.

---

### Proof of Concept

```
Block N begins.

Attacker tx (first in block):
  staking.update_rewards(
      staker_address = <any active staker>,
      disable_rewards = true
  )
  → last_reward_block := N
  → returns early, zero rewards distributed

Legitimate consensus tx (later in same block):
  staking.update_rewards(
      staker_address = <legitimate staker>,
      disable_rewards = false
  )
  → assert!(N > N)  ← FAILS with REWARDS_ALREADY_UPDATED

Result: all stakers receive 0 block rewards for block N.
Repeat every block → permanent yield freeze at cost of ~1 tx/block.
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
