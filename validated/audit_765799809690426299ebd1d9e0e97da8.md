### Title
Single Malicious Participant Can Permanently Abort DKG, Reshare, and Refresh via `broadcast_success` - (File: src/dkg.rs)

---

### Summary

The `broadcast_success` function in `src/dkg.rs` is the final gate of every `do_keyshare` execution (DKG, reshare, and refresh). It requires **every** participant to echo-broadcast `(true, session_id)`. A single malicious participant who instead broadcasts `(false, session_id)` causes an unconditional, unrecoverable abort for all honest parties, with no fallback path inside the protocol.

---

### Finding Description

`do_keyshare` is the shared core of `do_keygen`, `do_reshare`, and the refresh path. Its very last step is:

```rust
// src/dkg.rs line 531
broadcast_success(&mut chan, &participants, me, session_id).await?;
```

`broadcast_success` (lines 307–338) works as follows:

1. Each participant broadcasts `(true, session_id)` via the echo-broadcast primitive `do_broadcast`.
2. After collecting the full `vote_list`, it asserts that **all** entries are `true` and carry the correct `session_id`:

```rust
// dkg.rs lines 329–335
if !vote_list.iter().all(|&(boolean, _)| boolean) {
    return Err(ProtocolError::AssertionFailed(
        "A participant
            seems to have failed its checks. Aborting Protocol!"
            .to_string(),
    ));
}
```

The echo-broadcast layer (`do_broadcast` → `reliable_broadcast_receive_all`) faithfully delivers whatever value a participant sends. If a malicious participant sends `(false, session_id)`, the echo-broadcast protocol guarantees that every honest party receives exactly that `false` value. The subsequent `all(…)` check then fails for every honest party, and `do_keyshare` returns `Err(…)`.

There is no catch block, no retry, no threshold-based tolerance, and no way to identify and exclude the offending participant. The entire key-management operation is permanently aborted from the perspective of the honest parties.

The same abort path is triggered if the malicious participant sends `(true, wrong_session_id)`, because the session-id equality check (lines 321–327) fires first.

---

### Impact Explanation

A single malicious participant can unconditionally abort:

- **DKG** (`do_keygen`, lines 540–554) — honest parties never obtain key shares.
- **Reshare** (`do_reshare`, lines 600–635) — the key cannot be migrated to a new participant set.
- **Refresh** — shares cannot be refreshed, leaving long-lived shares permanently exposed.

All three operations funnel through `do_keyshare` and terminate at the same `broadcast_success` call (line 531). The honest parties have already completed all expensive cryptographic rounds (polynomial generation, commitment broadcast, share distribution, share verification) before the abort fires, so the attack wastes their work and leaves them without a usable key.

This matches the allowed impact: **High — Permanent denial of key generation, reshare, or refresh for honest parties under valid protocol inputs and documented trust assumptions.**

---

### Likelihood Explanation

The attack requires no cryptographic capability. The malicious participant only needs to deviate in the final broadcast round by sending `(false, session_id)` instead of `(true, session_id)`. Any participant who is willing to disrupt the protocol can do this trivially. Because the check is `all(…)` with no threshold tolerance, even a 1-of-N adversary suffices regardless of the configured signing threshold `t`.

---

### Recommendation

Replace the unanimous `all(…)` check with a threshold-tolerant check. Specifically:

- Accept the DKG output if at least `threshold` (or `n − f`) participants broadcast `true`.
- Identify participants who broadcast `false` and surface them as `MaliciousParticipant` errors so callers can exclude them and retry without the offender.
- Alternatively, remove `broadcast_success` entirely and rely on the existing per-share verification (`validate_received_share`) and public-key consistency check (`old_vk != verifying_key`) as the sole abort conditions, since those already catch cryptographic misbehavior without requiring unanimous agreement.

---

### Proof of Concept

1. Alice, Bob, and Charlie run DKG with threshold `t = 2`.
2. All three complete rounds 1–5 of `do_keyshare` successfully: session IDs exchanged, commitments broadcast and verified, secret shares distributed and validated, public key derived.
3. In the `broadcast_success` round, Charlie (malicious) sends `(false, session_id)` instead of `(true, session_id)`.
4. The echo-broadcast protocol faithfully delivers Charlie's `false` to Alice and Bob.
5. Alice and Bob each execute `vote_list.iter().all(|&(boolean, _)| boolean)` → `false`, and return `Err(ProtocolError::AssertionFailed("A participant seems to have failed its checks. Aborting Protocol!"))`.
6. Neither Alice nor Bob obtains a `KeygenOutput`. Key generation is permanently denied for this session. The only recovery is to restart DKG with a different participant set, requiring out-of-band coordination.

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
}
