### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Grief Stakers by Blocking Reward Distribution - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in `staking.cairo` is documented in the spec as callable by "Only starkware sequencer," but the implementation contains no caller check. Any public address can call it with `disable_rewards: true`, which marks the current block as already processed (`last_reward_block` is updated) without distributing any rewards. Because the function enforces a one-call-per-block invariant (`REWARDS_ALREADY_UPDATED`), the legitimate sequencer is then blocked from distributing rewards for that block, causing the staker to permanently lose that block's yield.

### Finding Description
`IStakingRewardsManager::update_rewards` is the sole mechanism by which per-block consensus rewards are credited to stakers and their delegation pools. Its implementation in `StakingRewardsManagerImpl` (staking.cairo, line 1449) begins with `self.general_prerequisites()` (pause check only) and a `REWARDS_ALREADY_UPDATED` guard, but contains no check that `get_caller_address()` equals the sequencer or any privileged role.

The critical path is:

1. The function unconditionally writes `self.last_reward_block.write(current_block_number)` at line 1485, regardless of `disable_rewards`.
2. Only after that write does it branch: `if disable_rewards || self.is_pre_consensus() { return; }` (line 1487).
3. Any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED` (line 1454–1458).

An attacker who calls `update_rewards(staker_address, disable_rewards: true)` before the sequencer does will:
- Consume the one allowed call for that block.
- Prevent any reward from being credited to `staker_info.unclaimed_rewards_own` or transferred to delegation pools.
- Leave the staker with zero additional yield for that block, with no recovery path.

Evidence that no caller restriction exists: every unit test that exercises `update_rewards` (e.g., `test_update_rewards_assertions_before_consensus`, `test_update_rewards_only_staker`, `test_update_rewards_disable_rewards`) calls the function directly via `IStakingRewardsManagerDispatcher` without any `cheat_caller_address` setup, confirming the function accepts calls from any address.

### Impact Explanation
**High — Permanent freezing / theft of unclaimed yield.**

Each block for which the attacker front-runs the sequencer is a block of rewards permanently lost to the staker and its delegators. The staker's `unclaimed_rewards_own` is never incremented for that block, and delegation pools never receive their share. Because `last_reward_block` is already set, the sequencer cannot retroactively credit the missed block. Repeated across many blocks, this constitutes a sustained drain of yield owed to stakers and delegators.

### Likelihood Explanation
**High.** The function is public with no access control. A griefing bot needs only to monitor the mempool (or simply call `update_rewards` with `disable_rewards: true` every block) to reliably block reward distribution. No privileged key, bridge access, or external dependency is required — only the ability to submit a Starknet transaction.

### Recommendation
Add a sequencer-only guard at the top of `update_rewards`, analogous to the existing role checks used elsewhere in the contract (e.g., `self.roles.only_app_governor()`). Introduce a stored `sequencer_address` (or reuse an existing operator role) and assert `get_caller_address() == self.sequencer_address.read()` before any state mutation.

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
