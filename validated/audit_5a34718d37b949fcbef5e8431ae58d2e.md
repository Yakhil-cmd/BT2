### Title
Pool Contract `claim_rewards` Executes While Staking Contract Is Paused, Bypassing Incident Containment - (File: `src/pool/pool.cairo`)

### Summary
The `Pool` contract has no pause mechanism and does not query the staking contract's `is_paused` state. As a result, `claim_rewards` and `change_reward_address` remain fully callable by any pool member even when the staking contract is paused by the security agent, undermining the protocol's ability to contain damage during a security incident.

### Finding Description
The staking contract (`src/staking/staking.cairo`) implements a pause mechanism via the `is_paused` storage variable and `IStakingPause`. Every state-mutating function in the staking contract is guarded by a pause check, as confirmed by the exhaustive pause test suite in `src/staking/tests/pause_test.cairo` (lines 126–361). The pause is the primary emergency lever for the security agent.

The `Pool` contract (`src/pool/pool.cairo`) is a separate contract. Most of its state-mutating functions call back into the staking contract (e.g., `enter_delegation_pool` → `add_stake_from_pool`, `exit_delegation_pool_intent` → `remove_from_delegation_pool_intent`), so they are indirectly blocked when the staking contract is paused.

However, two functions make **no call to the staking contract** and therefore execute freely regardless of the staking contract's pause state:

**`claim_rewards` (lines 335–377):** [1](#0-0) 

This function reads the pool member's accumulated reward state, computes the owed amount, updates internal storage, and transfers STRK tokens directly from the pool contract to the reward address. There is no call to the staking contract and no check of `is_paused`.

**`change_reward_address` (lines 505–526):** [2](#0-1) 

This function updates the reward address for a pool member in pool-local storage. Again, no staking contract call and no pause check.

### Impact Explanation
When the security agent pauses the staking contract in response to a security incident (e.g., a reward accounting bug that inflated pool rewards), the intent is to freeze all fund flows and state changes. Because `claim_rewards` in the pool contract is not gated on the staking contract's pause state, pool members — including an attacker who triggered the incident — can immediately drain accumulated (potentially inflated) STRK rewards from the pool contract before the team can investigate and remediate. This constitutes **theft of unclaimed yield**: rewards that should be frozen during the incident window are instead claimable and transferred out.

Additionally, `change_reward_address` allows a pool member to redirect future reward flows to an attacker-controlled address during the pause window, compounding the damage.

**Impact: High — Theft of unclaimed yield.**

### Likelihood Explanation
- The entry point (`claim_rewards`) is callable by any pool member or their reward address with no preconditions beyond membership.
- No privileged role is required.
- The staking contract's `is_paused` state is publicly readable, so an attacker can monitor for a pause event and immediately call `claim_rewards` on the pool contract.
- The pool contract holds real STRK balances (deposited via `update_rewards_from_staking_contract`), making the drain immediately profitable.

**Likelihood: High.**

### Recommendation
The pool contract should query the staking contract's pause state before executing any state-mutating function. Add a helper such as:

```cairo
fn assert_staking_not_paused(self: @ContractState) {
    let staking = IStakingDispatcher {
        contract_address: self.staking_pool_dispatcher.contract_address.read(),
    };
    assert!(!staking.is_paused(), "Contract is paused");
}
```

Call this at the top of `claim_rewards`, `change_reward_address`, and any other pool function that does not already call into the staking contract (which would revert on its own pause check).

### Proof of Concept
1. Deploy the full system (staking + pool contracts).
2. Pool member delegates tokens; rewards accumulate over several epochs via `update_rewards_from_staking_contract`.
3. A security incident is detected; the security agent calls `staking_pause_dispatcher.pause()` on the staking contract.
4. Verify `staking_dispatcher.is_paused()` returns `true`.
5. Pool member calls `pool_dispatcher.claim_rewards(pool_member)` directly on the pool contract.
6. The call **succeeds**: STRK tokens are transferred to the reward address despite the staking contract being paused.
7. Contrast with `pool_dispatcher.exit_delegation_pool_intent(amount)`, which reverts with `"Contract is paused"` because it calls `remove_from_delegation_pool_intent` on the paused staking contract.

The asymmetry between functions that call the staking contract (blocked) and those that do not (`claim_rewards`, `change_reward_address` — not blocked) confirms the missing pause propagation in the pool contract. [5](#0-4) [6](#0-5)

            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();

                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,

            // Calculate rewards and update entry_to_claim_from.

                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.

                        pool_member, reward_address, amount: rewards,
                    },

    #[abi(embed_v0)]
    impl StakingPauseImpl of IStakingPause<ContractState> {
        fn pause(ref self: ContractState) {
            self.roles.only_security_agent();
            if self.is_paused() {
                return;
            }
            self.is_paused.write(true);
            self.emit(PauseEvents::Paused { account: get_caller_address() });
        }

        fn unpause(ref self: ContractState) {
            self.roles.only_security_admin();
            if !self.is_paused() {
                return;
            }
            self.is_paused.write(false);
            self.emit(PauseEvents::Unpaused { account: get_caller_address() });
        }
    }
