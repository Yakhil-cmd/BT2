### Title
`MsgTurnBridge` Handler Is a No-Op — Missing Permission Check and Missing State Change - (File: x/cronos/keeper/msg_server.go)

### Summary
The `TurnBridge` message handler in `x/cronos/keeper/msg_server.go` is implemented as a complete no-op (`return nil, nil`). It performs no permission check and makes no state change. Any unprivileged user can call it and receive a success response, bypassing the `CanTurnBridge` permission system. Simultaneously, the bridge can never actually be disabled, permanently removing the emergency shutdown capability.

### Finding Description
The `TurnBridge` gRPC handler is:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

The rest of the codebase defines a full permission system for this operation. `CanTurnBridge` is a dedicated permission bit, `HasPermission` enforces it, and `UpdatePermissions` / `UpdateTokenMapping` both correctly gate their handlers behind permission checks: [2](#0-1) [3](#0-2) [4](#0-3) 

The integration test `test_gravity_turn_bridge` explicitly expects that an unprivileged caller (`community`) is rejected (`rsp["code"] != 0`) and that a privileged caller (`validator`) succeeds and the bridge is actually disabled: [5](#0-4) 

Neither property holds: the handler accepts any caller and changes nothing.

### Impact Explanation
Two distinct High-severity impacts:

1. **Bypass of bridge authorization**: Any unprivileged user can submit `MsgTurnBridge` and receive a success response (`code == 0`), bypassing the `CanTurnBridge` permission check entirely. This directly satisfies the "Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks" impact class.

2. **Permanent inability to disable the bridge**: Because no state is written, the bridge can never be turned off. In an incident where the gravity bridge or CRC20/CRC21 conversion is being exploited, the authorized admin has no functional emergency stop. This satisfies the "Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows" impact class (from the admin's perspective of being unable to halt a compromised bridge).

### Likelihood Explanation
The entry path is a standard Cosmos SDK transaction (`MsgTurnBridge`) reachable by any account with gas. No special privileges, leaked keys, or cryptographic breaks are required. The missing guard is a single missing implementation body — the proto definition, CLI command, and permission infrastructure all exist and are wired up, but the handler body was never completed.

### Recommendation
Implement `TurnBridge` to:
1. Check `HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `sdkerrors.ErrUnauthorized` on failure.
2. Read current params, set the bridge-enabled flag to `msg.Enable`, and persist via `k.SetParams`.

### Proof of Concept
1. Any account (no `CanTurnBridge` permission) submits:
   ```
   cronosd tx cronos turn-bridge false --from attacker
   ```
2. The handler at `msg_server.go:85-87` returns `nil, nil` — the transaction succeeds with `code == 0`.
3. The bridge state is unchanged; `send_to_evm_chain` continues to succeed.
4. A legitimate admin calling `turn-bridge false` also gets `code == 0` but the bridge is still live — emergency shutdown is impossible. [1](#0-0) [3](#0-2) [6](#0-5)

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```

// CmdTurnBridge returns a CLI command handler for enable or disable the bridge
func CmdTurnBridge() *cobra.Command {
	cmd := &cobra.Command{
		Use:   "turn-bridge [true/false]",
		Short: "Turn Bridge",
		Args:  cobra.ExactArgs(1),
		RunE: func(cmd *cobra.Command, args []string) error {
			clientCtx, err := client.GetClientTxContext(cmd)
			if err != nil {
				return err
			}

			enable, err := strconv.ParseBool(args[0])
			if err != nil {
				return err
			}
			msg := types.NewMsgTurnBridge(clientCtx.GetFromAddress().String(), enable)
			if err := msg.ValidateBasic(); err != nil {
				return err
			}
			return tx.GenerateOrBroadcastTxCLI(clientCtx, cmd.Flags(), msg)
		},
	}

	flags.AddTxFlagsToCmd(cmd)
	return cmd
}
