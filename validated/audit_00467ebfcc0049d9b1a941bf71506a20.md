### Title
Unprivileged Front-Run of `initialize()` on RewardSupplier Proxy Grants Infinite Token Approval to Attacker-Controlled Bridge — (`L1/starkware/solidity/stake/RewardSupplier.sol`)

---

### Summary

The `initialize()` function in `ProxySupport.sol` is callable by any external account on the proxy. The `notCalledDirectly` modifier only blocks direct calls to the **implementation** contract; it does not block calls routed through the proxy via `delegatecall`. On a freshly deployed proxy where `mintDestination == 0`, `isInitialized()` returns `false`, allowing an attacker to supply arbitrary initialization data — including an attacker-controlled `bridge` address — causing `IERC20(token).approve(bridge, type(uint256).max)` to grant infinite allowance to the attacker.

---

### Finding Description

**The `notCalledDirectly` modifier does not protect `initialize()` from external callers on the proxy.**

`BlockDirectCall` stores the implementation's own address at construction time as an immutable `this_`: [1](#0-0) 

The modifier passes when `this_ != address(this)`. When a user calls `proxy.initialize(data)`, the proxy `delegatecall`s the implementation, so `address(this)` becomes the **proxy** address while `this_` remains the **implementation** address — they differ, so the modifier passes for any external caller.

`initialize()` then checks `isInitialized()`: [2](#0-1) 

`RewardSupplier.isInitialized()` returns `mintDestination() != 0`: [3](#0-2) 

On a freshly deployed proxy, `mintDestination` is `0`, so `isInitialized()` is `false`. The code falls into the `else` branch and calls `initializeContractState(initData)` with the attacker's data — no governance or role check exists before this point.

Inside `initializeContractState`, the critical line is: [4](#0-3) 

The attacker supplies the legitimate STRK token address as `token` (passes `isContract()`) and their own malicious contract as `bridge` (also passes `isContract()`). The `approve` call grants `type(uint256).max` allowance on the real STRK token to the attacker's bridge.

All storage setters use `setAddressValueOnce` / `setUintValueOnce`: [5](#0-4) 

After the attacker's initialization succeeds, every subsequent legitimate initialization attempt reverts with `"ALREADY_SET"`, permanently locking the proxy in the attacker's configuration.

---

### Impact Explanation

- **Infinite STRK approval** is granted to the attacker's bridge. Any STRK tokens subsequently minted into the proxy (via `tick()` → `mintManager().mintRequest(...)`) can be drained by the attacker's bridge via `transferFrom`.
- **Legitimate initialization is permanently blocked** — `setAddressValueOnce` reverts on any re-attempt.
- **Protocol insolvency**: the RewardSupplier can never function correctly; all future reward minting is either stolen or permanently frozen.

Impact: **Critical** — direct theft of all tokens ever minted by the RewardSupplier + protocol insolvency.

---

### Likelihood Explanation

- Requires only mempool monitoring and front-running a single deployment transaction on Ethereum L1 — a well-understood, commonly executed attack.
- Attacker must pre-deploy three contracts (bridge, mintManager, messagingContract) to pass `isContract()` checks in `validateInitData`. This is trivial.
- No privileged access, leaked keys, or external dependency compromise required.

---

### Recommendation

Add an access control check at the top of `initialize()` in `ProxySupport.sol` gating the call to the proxy's governance/admin role, so only the authorized deployer can call it. Alternatively, enforce a commit-reveal or constructor-based initialization that atomically sets `mintDestination` during proxy deployment, ensuring `isInitialized()` is never `false` in a publicly observable state.

---

### Proof of Concept

1. Deploy three minimal contracts: `AttackerBridge`, `FakeMintManager`, `FakeMessaging` (each just needs to be a contract to pass `isContract()`).
2. Monitor the mempool for the legitimate `proxy.initialize(...)` transaction.
3. Front-run it by calling `proxy.initialize(abi.encode(AttackerBridge, STRK_TOKEN, FakeMintManager, FakeMessaging, nonzero_src, nonzero_dst, nonzero_curve))`.
4. `notCalledDirectly` passes (proxy addr ≠ impl addr). `isInitialized()` returns `false`. `validateInitData` passes (all addresses are contracts, all uints nonzero). `initializeContractState` runs, calling `IERC20(STRK).approve(AttackerBridge, type(uint256).max)`.
5. Legitimate `initialize()` call reverts with `"ALREADY_SET"`.
6. When any STRK is minted to the proxy, call `STRK.transferFrom(proxy, attacker, balance)` from `AttackerBridge` — succeeds due to infinite approval.
