### Title
Unauthorized Global Contract Namespace Hijacking via Cross-Contract `DeployGlobalContractAction` with `AccountId` Mode — (File: `runtime/runtime/src/global_contracts.rs`)

### Summary
`action_deploy_global_contract` in `runtime/runtime/src/global_contracts.rs` stores a global contract under `GlobalContractIdentifier::AccountId(account_id)` where `account_id` is the receipt's **receiver**, not the predecessor (caller). Unlike `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, and `DeleteAccount` — which all require `predecessor_id == receiver_id` — `DeployGlobalContractAction` with `AccountId` mode has no such authorization guard. Any deployed contract can call `promise_batch_create("victim.near")` and append `promise_batch_action_deploy_global_contract_by_account_id`, creating a receipt that executes on victim.near's shard and overwrites the global contract stored under victim.near's name with arbitrary attacker-controlled code.

### Finding Description

In `initiate_distribution` the namespace key is derived solely from the receipt receiver:

```rust
GlobalContractDeployMode::AccountId => {
    GlobalContractIdentifier::AccountId(account_id.clone())
}
``` [1](#0-0) 

`account_id` is the receipt's `receiver_id`, set at line 789 of `apply_action_receipt`:

```rust
let account_id = receipt.receiver_id();
``` [2](#0-1) 

The protocol documentation explicitly lists the actions that require `predecessor_id == receiver_id` — `DeployContract`, `Stake`, `AddKey`, `DeleteKey`, `DeleteAccount` — and `DeployGlobalContract` is absent from that list: [3](#0-2) 

The `action_deploy_global_contract` function signature accepts no `predecessor_id` parameter, confirming no caller-identity check is performed inside it: [4](#0-3) 

Any Wasm contract can invoke the host function `promise_batch_action_deploy_global_contract_by_account_id` on a receipt created with `promise_batch_create("victim.near")`: [5](#0-4) 

The resulting receipt is routed to victim.near's shard and executed there, with `account_id = victim.near`, causing the attacker's code to be written to `TrieKey::GlobalContractCode { identifier: AccountId("victim.near") }` and propagated to every shard via the distribution receipt chain. [6](#0-5) 

### Impact Explanation

- **Global contract code identity corrupted**: every account that has called `UseGlobalContract(AccountId("victim.near"))` now executes the attacker's Wasm on every function call, with full access to that account's state, balance, and cross-contract call authority.
- **Balance drained from victim**: the storage cost `global_contract_storage_amount_per_byte × code_len` is subtracted from victim.near's account balance. [7](#0-6) 

- **Namespace permanently hijacked** until victim.near re-deploys, which itself costs balance.

### Likelihood Explanation

- Reachable by any deployed Wasm contract via the standard `promise_batch_create` + `promise_batch_action_deploy_global_contract_by_account_id` host-function sequence — no privileged role required.
- The attacker pays only gas; the victim pays the storage cost.
- The victim account need only exist and hold enough balance to cover storage; no interaction or consent from the victim is needed.

### Recommendation

Add a `predecessor_id == receiver_id` guard for `DeployGlobalContractAction` with `AccountId` mode, mirroring the existing guard for `DeployContract`. This can be done either:

1. In `validate_actions_with_mode` (in `runtime/runtime/src/verifier.rs`), by adding `DeployGlobalContract` (when `deploy_mode == AccountId`) to the set of actions that require `predecessor_id == receiver_id`; or
2. Inside `action_deploy_global_contract` itself, by accepting `predecessor_id` as a parameter and returning `ActionErrorKind::ActorNoPermission` when `predecessor_id != account_id`.

The `CodeHash` deploy mode is unaffected and requires no change, as its key is content-addressed.

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())
