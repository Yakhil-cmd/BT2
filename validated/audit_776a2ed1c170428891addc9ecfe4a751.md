### Title
Unbounded `pool_member_epoch_balance` Trace Causes Permanent Freezing of Unclaimed Yield in `calculate_rewards` - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` internal function in `Pool.cairo` iterates over every entry in a pool member's `pool_member_epoch_balance` trace in a single transaction, with no upper bound on the number of iterations. A pool member who makes balance changes across many epochs without claiming rewards accumulates an unbounded number of trace entries. When `claim_rewards` is eventually called, the loop must process all accumulated entries at once, potentially exhausting the Starknet transaction gas limit and permanently freezing the pool member's unclaimed yield.

### Finding Description
`calculate_rewards` in `src/pool/pool.cairo` contains an explicitly acknowledged unbounded loop:

```
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The trace backing this loop is `pool_member_epoch_balance: Map<ContractAddress, PoolMemberBalanceTrace>`, a `Vec`-backed checkpoint store. [2](#0-1) 

The `insert` function in the trace implementation appends a **new** checkpoint whenever the epoch key differs from the last stored key, and only overwrites in-place when the key matches:

```
if last.key == key {
    last.value = value;
    checkpoints[len - 1].write(last);
} else {
    checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
}
``` [3](#0-2) 

Every call to `set_member_balance` inserts with key `current_epoch + K`: [4](#0-3) 

This means each of the following public functions, when called in a different epoch, appends one new entry to the trace:
- `enter_delegation_pool` (line 201)
- `add_to_delegation_pool` (line 242)
- `exit_delegation_pool_intent` (line 278)
- `enter_delegation_pool_from_staking_contract` (lines 456, 464) [5](#0-4) 

The `entry_to_claim_from` pointer stored in `pool_member_info` advances only after a **successful** `claim_rewards` call: [6](#0-5) 

If the transaction reverts (out of gas), the pointer is never advanced, so every subsequent `claim_rewards` call starts from the same position and faces the same oversized loop — permanently locking the rewards.

### Impact Explanation
Once the `pool_member_epoch_balance` trace grows large enough to exhaust the Starknet per-transaction gas limit inside `calculate_rewards`, every call to `claim_rewards` for that pool member will revert. Because there is no mechanism to claim rewards for a partial epoch range, and because `entry_to_claim_from` is only persisted on success, the pool member's accumulated unclaimed yield is **permanently frozen**. This matches the allowed High impact: *Permanent freezing of unclaimed yield*.

### Likelihood Explanation
Any pool member (unprivileged delegator) can trigger this by making one balance-changing call per epoch (e.g., `add_to_delegation_pool` with a minimal amount) across many epochs without ever calling `claim_rewards`. Each such call costs only a transaction fee. Over hundreds or thousands of epochs the trace grows proportionally. The developers themselves acknowledge the loop is unbounded in a code comment, indicating awareness of the risk. The attack is entirely self-directed and requires no privileged access, leaked keys, or external dependencies.

### Recommendation
Enforce a maximum number of unprocessed balance trace entries before allowing further balance changes, analogous to the BNB/BSC fix referenced in the external report. Alternatively, implement a partial-claim mechanism that allows `claim_rewards` to process a caller-specified number of trace entries per call and persist the updated `entry_to_claim_from` pointer incrementally, so rewards can always be claimed in bounded-size batches regardless of trace length.

### Proof of Concept
1. Pool member `A` enters a delegation pool in epoch `E₀` via `enter_delegation_pool`.
2. In each subsequent epoch `E₁, E₂, …, Eₙ`, `A` calls `add_to_delegation_pool` with a minimal amount (e.g., 1 unit). Each call appends one new checkpoint to `pool_member_epoch_balance` because `get_epoch_plus_k()` returns a strictly increasing key each epoch.
3. `A` never calls `claim_rewards`, so `entry_to_claim_from` remains at 0.
4. After `N` epochs the trace has `N` entries. `A` (or anyone) calls `claim_rewards(pool_member: A)`.
5. `calculate_rewards` enters the loop with `entry_to_claim_from = 0` and `pool_member_trace_length = N`. Each iteration reads a storage slot (`pool_member_trace.at(i)`) and calls `find_sigma` (additional storage reads). For sufficiently large `N`, the transaction exhausts the Starknet gas limit and reverts.
6. Because the revert rolls back the `entry_to_claim_from` update, every future `claim_rewards` call faces the same `N`-iteration loop and also reverts. `A`'s accumulated yield is permanently frozen. [7](#0-6) [8](#0-7)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }

```text
    fn insert(
        self: StoragePath<Mutable<PoolMemberBalanceTrace>>, key: Epoch, value: PoolMemberBalance,
    ) -> (PoolMemberBalance, PoolMemberBalance) {
        let checkpoints = self.checkpoints;

        let len = checkpoints.len();
        if len == Zero::zero() {
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
            return (Zero::zero(), value);
        }

        // Update or append new checkpoint.
        let mut last = checkpoints[len - 1].read();
        let prev = last.value;
        if last.key == key {
            last.value = value;
            checkpoints[len - 1].write(last);
        } else {
            // Checkpoint keys must be non-decreasing.
            assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
            checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
        }
        (prev, value)
    }
