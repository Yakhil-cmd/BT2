### Title
Unrestricted `update_rewards` with `disable_rewards: true` Enables Permanent Griefing of Consensus Reward Distribution — (File: `src/staking/staking.cairo`)

---

### Summary

Any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` for any active staker, consuming the global `last_reward_block` slot for the current block without distributing any rewards. Repeated every block, this permanently freezes all consensus reward accrual for every staker and delegator in the protocol.

---

### Finding Description

`update_rewards` is a public function gated only by `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no role check exists. [1](#0-0) 

The critical ordering flaw is that `last_reward_block` is written **unconditionally**, before the `disable_rewards` branch that exits without distributing rewards: [2](#0-1) 

`last_reward_block` is a **global** storage variable: [3](#0-2) 

The function asserts `current_block_number > self.last_reward_block.read()` at entry, meaning only **one** call per block is permitted across the entire protocol. An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in a block:

1. Passes all validation (staker is active, has non-zero balance at current epoch)
2. Writes `last_reward_block = current_block`
3. Returns early — `staker_info.unclaimed_rewards_own` is never incremented, pool `cumulative_rewards_trace` is never updated, and `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` is never called [4](#0-3) 

Any subsequent legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.

The attacker needs only any active staker address as the `staker_address` argument — these are publicly emitted via `NewStaker` events. [5](#0-4) 

---

### Impact Explanation

If the attacker repeats this every block, no staker or delegator ever accrues consensus rewards. The rewards for each block are silently discarded. `staker_info.unclaimed_rewards_own` remains zero for all stakers, and pool `cumulative_rewards_trace` is never advanced, so `claim_rewards` and pool `claim_rewards` always return zero. This constitutes **permanent freezing of unclaimed yield** for all protocol participants — a HIGH-severity impact under the allowed scope.

---

### Likelihood Explanation

- The attacker requires no special role, no privileged key, and no insider knowledge beyond any active staker address (publicly available on-chain).
- The only cost is gas per block. On Starknet, transaction fees are low, making sustained griefing economically viable.
- The attack is trivially automatable with a simple script that fires one transaction per block.

---

### Recommendation

Either:

1. Move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so a `disable_rewards: true` call does not consume the block's reward slot; or
2. Add access control so that only the staker themselves (or a designated operator address) can call `update_rewards` for a given `staker_address`.

---

### Proof of Concept

```
// Attacker script — runs once per block:
for each new block N:
    staking_contract.update_rewards(
        staker_address: <any active staker from NewStaker events>,
        disable_rewards: true
    )
    // Effects:
    //   last_reward_block := N
    //   no rewards distributed
    //   any legitimate update_rewards call in block N reverts with REWARDS_ALREADY_UPDATED
```

After this runs for every block:
- `staker_info.unclaimed_rewards_own` for all stakers stays at zero
- Pool `cumulative_rewards_trace` is never updated
- `claim_rewards` (staker) and pool `claim_rewards` always transfer zero STRK
- All consensus reward accrual is permanently frozen [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
