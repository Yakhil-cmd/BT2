`new_settlement_layer_chain_id_storage` Excluded from IO Frame Snapshot â€” Reverted Transaction Permanently Poisons Block-Level Settlement-Layer Chain-ID State - (File: `basic_system/src/system_implementation/system/io_subsystem.rs`)

### Summary
`FullIO::new_settlement_layer_chain_id_storage` is omitted from both the IO frame snapshot struct (`FullIOStateSnapshot`) and the per-transaction reset (`begin_next_tx`). Any write to this field that occurs inside a subsequently-reverted call frame is never undone, permanently marking the block as having received a settlement-layer chain-ID update even though the triggering transaction reverted. This is the direct analog of the reported pattern: a shared, context-undifferentiated state that leaks across consumer boundaries.

### Finding Description
`FullIO` holds six sub-storages that must be snapshotted and rolled back together when a call frame reverts. Five of them are handled correctly; `new_settlement_layer_chain_id_storage` is silently omitted.

`FullIOStateSnapshot` captures only five fields:

```rust

    events: usize,

```

`start_io_frame` therefore never snapshots it:


`NewSettlementLayerChainIdStorage::update` enforces a write-once-per-block invariant:


Because the storage is never rolled back, a reverted transaction that triggers `update_settlement_layer_chain_id` (via the `SettlementLayerChainIdUpdated` event hook in `system_context_event_hook`) permanently sets `new_settlement_layer_chain_id_storage` for the rest of the block, even though all other IO effects (events, logs, storage writes) are correctly undone.

The event hook itself does not validate the emitting address â€” it fires for any event whose first topic matches `SL_CHAIN_ID_UPDATED_EVENT_SIG`:

```rust

    topics: &arrayvec::ArrayVec<...>,
    data: &[u8],
    caller_ee: u8,          // not checked
    system: &mut System<S>,
    resources: &mut S::Resources,

    if topics[0].as_u8_array() == SL_CHAIN_ID_UPDATED_EVENT_SIG {
        new_sl_chain_id_event_hook(topics, data, caller_ee, system, resources)
    } else { Ok(()) }
}
```
