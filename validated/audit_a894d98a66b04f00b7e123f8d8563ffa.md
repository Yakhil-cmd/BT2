### Title
WalletConnect Multi-NFT Offer Royalty Amount Understated in Confirmation Dialog - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

---

### Summary

`formatAmountWithRoyalties` in `parseCommandDisplay.ts` incorrectly divides the fungible amount by the number of NFTs before computing each royalty contribution. When a WalletConnect `take_offer` or `create_offer_for_ids` command involves two or more NFTs with royalties, the `amountWithRoyalties` shown to the user in the confirmation dialog is understated by a factor of N (the number of NFTs), causing the user to approve a transaction believing they will pay less than they actually will.

---

### Finding Description

`formatAmountWithRoyalties` computes the royalty-inclusive total as:

```js
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← divides by N
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
``` [1](#0-0) 

The correct formula is `royaltyAmount = Σ (amount × royaltyPercentage_i / 10_000)`. The code instead computes `royaltyAmount = (amount / N) × Σ (royaltyPercentage_i / 10_000)`, which is exactly `1/N` of the correct value.

The test suite at line 251–334 encodes this wrong behavior as the expected output: with 2 NFTs (royalties 500 and 10 basis points) and 100,000,000 mojos, the test asserts `amountWithRoyalties: '0.00010255'` (royalty = 2,550,000 mojos), while the correct value is 0.0001051 XCH (royalty = 5,100,000 mojos). [2](#0-1) 

`parseCommandDisplay` is the function that populates the WalletConnect confirmation dialog for `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids`. [3](#0-2) 

The `amountWithRoyalties` field is attached to each fungible spending line and rendered to the user before they click Approve. [4](#0-3) 

---

### Impact Explanation

A WalletConnect-connected dApp sends a `take_offer` command containing an offer where the user must spend XCH (or CATs) to receive two or more NFTs that each carry royalties. The confirmation dialog displays `amountWithRoyalties` as the total the user will pay. Because the displayed value is `1/N` of the correct royalty-inclusive total, the user sees a materially lower number than the blockchain will actually deduct. The user approves the transaction under a false understanding of the cost. The actual on-chain royalty deduction is computed correctly by the backend and is higher than what was shown.

This satisfies the High impact criterion: *"Corruption, spoofing, or unsafe trust of … WalletConnect state that causes a user to approve … the wrong … amount."*

---

### Likelihood Explanation

Any dApp that has an established WalletConnect session (already approved by the user) can send a `take_offer` command. Multi-NFT offers with royalties are a normal, supported use case. No special privileges, leaked keys, or cryptographic breaks are required. The discrepancy grows linearly with the number of NFTs: with 3 NFTs the displayed royalty is 1/3 of correct, with 4 NFTs it is 1/4, etc.

---

### Recommendation

Remove the division by `royaltyPercentages.length`. Each royalty percentage should be applied to the full `amount`, not to a split share:

```js
// Correct
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [5](#0-4) 

Update the corresponding test assertion at line 320 to reflect the corrected value. [2](#0-1) 

---

### Proof of Concept

**Scenario**: A dApp sends `chia_wallet.take_offer` via WalletConnect. The offer requests 100,000,000 mojos (0.0001 XCH) from the user in exchange for two NFTs: NFT-A (royalty 500 bps = 5%) and NFT-B (royalty 10 bps = 0.1%).

**Correct total the user will pay on-chain:**
- Royalty for NFT-A: 100,000,000 × 500 / 10,000 = 5,000,000 mojos
- Royalty for NFT-B: 100,000,000 × 10 / 10,000 = 100,000 mojos
- Total: 105,100,000 mojos = **0.0001051 XCH**

**What the confirmation dialog shows (current buggy code):**
- splitAmount = 100,000,000 / 2 = 50,000,000
- Royalty for NFT-A: 50,000,000 × 500 / 10,000 = 2,500,000 mojos
- Royalty for NFT-B: 50,000,000 × 10 / 10,000 = 50,000 mojos
- Displayed total: 102,550,000 mojos = **0.00010255 XCH**

The user sees 0.00010255 XCH and approves. The blockchain deducts 0.0001051 XCH — approximately 2.5% more than displayed. The gap scales with the number of NFTs and the royalty percentages involved. [5](#0-4)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }
