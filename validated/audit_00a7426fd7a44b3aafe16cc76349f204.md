### Title
`DelegateAction` Signed Hash Lacks Chain-ID Domain Separator, Enabling Cross-Network Signature Replay - (File: `core/primitives/src/action/delegate.rs`)

### Summary

`DelegateAction` (NEP-366 meta-transactions) is signed over a hash that contains no chain identifier. A `SignedDelegateAction` produced for testnet is cryptographically valid on mainnet (and vice versa) whenever the same account and key exist on both networks, the nonce is still acceptable, and `max_block_height` has not elapsed. Any unprivileged user who obtains the off-chain `SignedDelegateAction` blob can wrap it in a fresh mainnet outer transaction and cause the sender's actions to execute on the wrong network without the sender's consent.

### Finding Description

`DelegateAction.get_nep461_hash()` constructs the signed digest as:

```
SHA-256( borsh(MessageDiscriminant(2^30 + 366)) || borsh(DelegateAction) )
```

`DelegateAction` contains: `sender_id`, `receiver_id`, `actions`, `nonce`, `max_block_height`, `public_key`. [1](#0-0) 

There is no `chain_id`, `network_id`, or `block_hash` field. [2](#0-1) 

The `SignableMessage` wrapper adds only a 4-byte NEP discriminant — it carries no network context. [3](#0-2) 

`apply_delegate_action` verifies the signature, checks `max_block_height`, validates the nonce, and dispatches the inner actions. None of these steps bind the signature to the current chain. [4](#0-3) 

Regular `TransactionV0`/`TransactionV1` are immune because they embed a `block_hash` that is verified as an ancestor of the current chain head via `check_transaction_validity_period`; a testnet block hash is simply not present in mainnet's store. [5](#0-4) 

`DelegateAction` has no equivalent anchor — `max_block_height` is a plain integer shared across all NEAR networks. [6](#0-5) 

### Impact Explanation

An attacker who obtains a `SignedDelegateAction` that Alice created for testnet can:

1. Construct a new mainnet outer transaction (signed by the attacker as relayer) addressed to Alice's mainnet account.
2. Embed Alice's testnet `SignedDelegateAction` verbatim.
3. Submit to mainnet.

`apply_delegate_action` will accept the signature (the hash is identical on both networks), advance Alice's mainnet access-key nonce, and dispatch the inner actions — token transfers, function calls, `AddKey`, `DeleteKey`, etc. — as if Alice had authorized them on mainnet.

The corrupted values are: Alice's mainnet access-key nonce (permanently advanced), her mainnet token balance (if a `Transfer` or `FunctionCall` with deposit is included), and any mainnet contract state mutated by the inner actions.

### Likelihood Explanation

- Many developers and power users share the same account name and key pair across testnet and mainnet.
- `max_block_height` values chosen for testnet (e.g., current height + 100) are numerically valid on mainnet, which runs at a similar block height.
- The signed `DelegateAction` blob is transmitted off-chain to a relayer; a malicious relayer, a compromised relay endpoint, or a network observer can capture it.
- No special protocol privilege is required: submitting a transaction to mainnet is open to any user.

### Recommendation

Include the chain's network identifier in the signed digest. The simplest approach is to add a `chain_id: String` field to `DelegateAction` (and `DelegateActionV2`) and populate it from `ApplyState` or the genesis config, then verify it matches during `apply_delegate_action`. Alternatively, incorporate the chain ID into the `SignableMessage` wrapper so all future message types inherit the binding automatically.

The `chain_id` host function already exposes the genesis chain ID to contracts; the same value should be committed into the `DelegateAction` hash. [7](#0-6) 

### Proof of Concept

**Setup**: Alice has account `alice.near` on both testnet and mainnet, same ed25519 key, mainnet access-key nonce = 0.

**Step 1 – Alice creates a testnet meta-transaction** (e.g., via a dApp):

```rust
let delegate_action = DelegateAction {
    sender_id:       "alice.near".parse().unwrap(),
    receiver_id:     "token.near".parse().unwrap(),
    actions:         vec![ft_transfer_to_bob_1000_tokens],
    nonce:           1,                // valid on mainnet (ak_nonce == 0)
    max_block_height: 200_000_000,     // far future, valid on mainnet too
    public_key:      alice_pubkey,
};
// Hash = SHA256(discriminant_366 || borsh(delegate_action))
// No chain_id in the hash.
let signature = alice_sk.sign(delegate_action.get_nep461_hash().as_bytes());
let signed_da = SignedDelegateAction { delegate_action, signature };
```

Alice sends `signed_da` to a testnet relayer. [8](#0-7) 

**Step 2 – Attacker intercepts `signed_da` and submits it to mainnet**:

```rust
// Attacker wraps the SAME signed_da in a mainnet outer transaction.
let outer_tx = Transaction::V0(TransactionV0 {
    signer_id:   attacker_account,
    public_key:  attacker_pubkey,
    nonce:       attacker_nonce,
    receiver_id: "alice.near".parse().unwrap(),  // outer receiver = DA sender
    block_hash:  mainnet_recent_block_hash,       // valid mainnet block hash
    actions:     vec![Action::Delegate(Box::new(signed_da))],
});
let signed_outer = outer_tx.sign(&attacker_sk);
// Submit to mainnet RPC.
```

**Step 3 – Mainnet runtime executes**:

`apply_delegate_action` is called. [9](#0-8) 

- `signed_delegate_action.verify()` → **passes** (hash is identical; no chain_id).
- `block_height > max_block_height` → **passes** (200 000 000 is far future).
- Nonce check: mainnet ak_nonce = 0 < delegate nonce 1 → **passes**.
- Inner `ft_transfer` receipt is dispatched on mainnet.

Alice's mainnet token balance is drained without her consent. Her mainnet access-key nonce is advanced to 1, preventing her from immediately reusing that nonce.

impl SignedDelegateAction {
    pub fn verify(&self) -> bool {
        let delegate_action = &self.delegate_action;
        let hash = delegate_action.get_nep461_hash();
        let public_key = &delegate_action.public_key;

        self.signature.verify(hash.as_ref(), public_key)
    }

    pub fn sign(singer: &Signer, delegate_action: DelegateAction) -> Self {
        let signature = singer.sign(delegate_action.get_nep461_hash().as_bytes());
        Self { delegate_action, signature }
    }

        let signable = SignableMessage::new(&self, SignableMessageType::DelegateAction);

pub(crate) fn apply_delegate_action(
    state_update: &mut TrieUpdate,
    apply_state: &ApplyState,
    action_receipt: &VersionedActionReceipt,
    sender_id: &AccountId,
    signed_delegate_action: VersionedSignedDelegateActionRef<'_>,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    if !signed_delegate_action.verify() {
        result.result = Err(ActionErrorKind::DelegateActionInvalidSignature.into());
        return Ok(());
    }
    let delegate_action = signed_delegate_action.delegate_action();
    if apply_state.block_height > delegate_action.max_block_height() {
        result.result = Err(ActionErrorKind::DelegateActionExpired.into());
        return Ok(());
    }
    if delegate_action.sender_id().as_str() != sender_id.as_str() {
        result.result = Err(ActionErrorKind::DelegateActionSenderDoesNotMatchTxReceiver {
            sender_id: delegate_action.sender_id().clone(),
            receiver_id: sender_id.clone(),
        }
        .into());
        return Ok(());
    }

    validate_delegate_action_key(state_update, apply_state, delegate_action, result)?;
