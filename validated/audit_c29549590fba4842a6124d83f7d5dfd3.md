### Title
Staker Can Instantly Raise Commission to 100% via `set_commission_commitment` + `set_commission`, Stealing Delegator Yield - (File: src/staking/staking.cairo)

### Summary

A staker can atomically set a `CommissionCommitment` with `max_commission = 10000` (100%) and immediately call `set_commission(10000)` in the same block. Because the commission change takes effect instantly with no epoch delay, all subsequent pool rewards are routed entirely to the staker as commission. Delegators already in the pool cannot exit immediately due to the mandatory exit wait window, causing theft of their unclaimed yield during that period.

### Finding Description

Without a `CommissionCommitment`, `set_commission` enforces that commission can only **decrease**: [1](#0-0) 

With an active `CommissionCommitment`, the constraint changes: commission may be set to **any value ≤ `max_commission`**, including values higher than the current commission: [2](#0-1) 

The `set_commission_commitment` function enforces only that `max_commission >= current_commission` and `expiration_epoch > current_epoch` (minimum 1 epoch window): [3](#0-2) 

The commission change writes directly to storage with no epoch-based delay: [4](#0-3) 

At reward calculation time, the **current** commission is read from storage, meaning the raised commission applies immediately to the next reward distribution: [5](#0-4) 

The developers themselves acknowledge this gap in a code comment: [6](#0-5) 

### Impact Explanation

**High — Theft of unclaimed yield.**

A malicious staker executes two transactions in the same block:
1. `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`
2. `set_commission(10000)`

After step 2, `split_rewards_with_commission` routes 100% of pool rewards to the staker as commission and 0% to the pool: [7](#0-6) 

Delegators who are already in the pool cannot exit immediately. The exit wait window (default 1 week) forces them to remain exposed for the entire window, during which all their yield is stolen. [8](#0-7) 

### Likelihood Explanation

**Medium.** Any staker who has opened a delegation pool can execute this attack with no special privileges. The two calls can be submitted in the same block, giving delegators zero reaction time. The attack is economically rational for a staker who has accumulated a large delegated pool and wishes to extract maximum value before delegators can exit.

### Recommendation

Introduce an epoch delay between setting a `CommissionCommitment` and being allowed to raise commission under it. Specifically, require that a commission increase via commitment only takes effect `K` epochs after the commitment is set, mirroring the delay already used for stake balance changes. This gives delegators time to observe the pending increase and exit before it applies.

Alternatively, cap `max_commission` at a small increment above the current commission per commitment period, preventing a single-step jump to 100%.

### Proof of Concept

```
// Epoch N: Staker has commission = 1000 (10%), delegator has delegated.
// Staker executes atomically:
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: N + 1);
staking.set_commission(commission: 10000);

// Epoch N reward distribution:
// split_rewards_with_commission(pool_rewards, commission=10000)
//   => commission_rewards = pool_rewards * 10000 / 10000 = pool_rewards
//   => pool_rewards = 0
// Delegator receives 0 yield. Staker receives 100% of pool rewards.
// Delegator cannot exit for exit_wait_window (default 1 week).

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
