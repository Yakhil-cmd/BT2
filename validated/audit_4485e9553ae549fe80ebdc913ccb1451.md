### Title
Unprivileged Caller Can Permanently Freeze Staker Block Rewards via `update_rewards(disable_rewards: true)` — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is publicly callable with no access control. It accepts a `disable_rewards: bool` parameter that, when `true`, advances the global `last_reward_block` checkpoint **without distributing any rewards**. Because the contract enforces a single call per block via this global checkpoint, any unprivileged caller can race ahead of legitimate stakers every block, permanently destroying their unclaimed block rewards.

---

### Finding Description

`update_rewards` is exposed as a public ABI function under `StakingRewardsManagerImpl`:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // only: not-paused + non-zero caller
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        ...
        // Update last block rewards — happens unconditionally.
        self.last_reward_block.write(current_block_number);   // <-- global checkpoint advanced

        if disable_rewards || self.is_pre_consensus() {
            return;                                           // <-- rewards silently skipped
        }
        ...
        self._update_rewards(...);   // staker_info.unclaimed_rewards_own += staker_rewards
    }
}
``` [1](#0-0) 

`general_prerequisites` imposes no caller identity check:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [2](#0-1) 

The global `last_reward_block` is the sole gate preventing duplicate calls in the same block:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

When `_update_rewards` is **not** called, the reward supplier's unclaimed balance is never incremented and the staker's `unclaimed_rewards_own` is never updated:

```cairo
reward_supplier_dispatcher
    .update_unclaimed_rewards_from_staking_contract(
        rewards: staker_rewards + total_pools_rewards,
    );
...
staker_info.unclaimed_rewards_own += staker_rewards;
``` [4](#0-3) 

Because the reward supplier is never notified and `unclaimed_rewards_own` is never incremented, the block's rewards are **permanently unrecoverable** for that block — there is no catch-up mechanism.

**Attack path (no privileged access required):**

1. Attacker monitors the mempool / block production.
2. In every block N, attacker submits `update_rewards(any_active_staker, disable_rewards: true)`.
3. `last_reward_block` is written to N; no rewards are distributed.
4. Any legitimate call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers lose their consensus block rewards for block N, permanently.

The attacker needs only one transaction per block and any valid `staker_address` (the staker whose rewards are targeted, or any active staker to consume the global slot).

---

### Impact Explanation

This matches **High: Permanent freezing of unclaimed yield**. For every block in which the attacker front-runs with `disable_rewards: true`, the corresponding block rewards are irrecoverably lost: the reward supplier is never told to reserve them, and `unclaimed_rewards_own` is never incremented. Sustained execution across many blocks causes material, permanent loss of yield for all stakers and their delegators.

---

### Likelihood Explanation

Any unprivileged address can call `update_rewards`. The only cost is one transaction per block. A competitor, a griefing actor, or an automated bot can sustain this indefinitely. No leaked key, no governance role, no bridge compromise is required.

---

### Recommendation

Restrict who may call `update_rewards` with `disable_rewards: true`. The simplest fix is to require the caller to be the staker's registered operational address (or the staker themselves) before accepting `disable_rewards: true`. Alternatively, remove the `disable_rewards` parameter entirely and handle the pre-consensus no-op path internally without exposing it as a caller-controlled flag.

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
+   if disable_rewards {
+       // Only the staker or their operational address may suppress reward distribution.
+       let caller = get_caller_address();
+       let staker_info = self.internal_staker_info(:staker_address);
+       assert!(
+           caller == staker_address || caller == staker_info.operational_address,
+           "{}",
+           Error::UNAUTHORIZED_DISABLE_REWARDS,
+       );
+   }
    ...
}
```

### Proof of Concept

```
Block N:
  Attacker tx:  update_rewards(alice_staker, disable_rewards=true)
    → last_reward_block := N
    → returns early, no rewards distributed

  Alice's tx:   update_rewards(alice_staker, disable_rewards=false)
    → assert!(N > N)  ← FAILS: REWARDS_ALREADY_UPDATED
    → alice.unclaimed_rewards_own unchanged
    → reward_supplier never notified

Block N+1 (if attacker repeats): same outcome.

Net effect: alice (and every other staker) permanently loses block rewards
for every block the attacker front-runs.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
