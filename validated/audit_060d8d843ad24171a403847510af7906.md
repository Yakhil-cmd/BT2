### Title
Unpermissioned `update_rewards` Caller with `disable_rewards` Flag Permanently Freezes Consensus Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is a public function with no caller access control. It accepts a `disable_rewards: bool` parameter. When called with `disable_rewards: true`, it writes the current block number to the global `last_reward_block` storage slot and returns early without distributing any rewards. Because the function asserts `current_block_number > last_reward_block` before proceeding, any subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`. An unprivileged attacker can call this once per block to permanently suppress consensus block rewards for every staker.

---

### Finding Description

`update_rewards` is declared in the public `IStakingRewardsManager` interface and implemented in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // global slot updated unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits before _update_rewards
    }
    ...
    self._update_rewards(...);
}
``` [1](#0-0) 

`general_prerequisites` enforces only two things: the contract is not paused, and the caller is not the zero address. [2](#0-1) 

There is no `assert_caller_is_attestation_contract()` or any equivalent guard analogous to the checks present on `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), ...);
    self.assert_caller_is_attestation_contract();   // <-- access control present here
    ...
}
``` [3](#0-2) 

The `last_reward_block` slot is a single global value shared across all stakers: [4](#0-3) 

Because `last_reward_block` is written before the `disable_rewards` branch is evaluated, calling `update_rewards(any_active_staker, true)` consumes the per-block reward slot for every staker in the system.

---

### Impact Explanation

An attacker who calls `update_rewards(valid_staker, disable_rewards: true)` once per Starknet block:

1. Advances `last_reward_block` to the current block number.
2. Returns without invoking `_update_rewards`, so no staker's `unclaimed_rewards_own` is incremented and no pool receives rewards.
3. Causes every subsequent legitimate call to `update_rewards` in that block to revert with `REWARDS_ALREADY_UPDATED`.

Consensus block rewards for that block are permanently lost — they cannot be retroactively recovered because the accounting is strictly per-block. Repeating this every block results in **permanent, total suppression of consensus rewards** for all stakers and all delegation pools.

**Impact category: High — Permanent freezing of unclaimed yield.**

---

### Likelihood Explanation

- The function is part of a public `#[starknet::interface]` and requires no token balance, no staking position, and no privileged role — only a non-zero caller address and knowledge of any currently active staker address (all staker addresses are emitted as `NewStaker` events and stored in the public `stakers` vector).
- The cost per block is a single transaction. On Starknet, transaction fees are low.
- The attack is fully permissionless and repeatable indefinitely.

**Likelihood: High.**

---

### Recommendation

Add a caller restriction to `update_rewards` identical to the pattern used in `update_rewards_from_attestation_contract`. Introduce and call a dedicated `assert_caller_is_rewards_manager` (or reuse the attestation contract guard if the same contract is the intended caller), so that only the authorized contract can invoke this function. The `disable_rewards` flag should also be re-evaluated: if it is only needed for internal migration paths, it should be moved to a separate, access-controlled entrypoint rather than exposed on the public interface.

---

### Proof of Concept

1. Observe any active staker address `S` (e.g., from `NewStaker` events or the public `stakers` vector).
2. Each Starknet block, call:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
3. `last_reward_block` is set to the current block; no rewards are distributed.
4. Any legitimate call to `update_rewards` for the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. All stakers and delegation pools receive zero consensus block rewards for every block the attacker front-runs, permanently destroying that yield.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
