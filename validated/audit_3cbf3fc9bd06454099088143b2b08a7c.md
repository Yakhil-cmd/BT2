### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Deny Staker Rewards - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the Staking contract is documented as restricted to "Only starkware sequencer" but has no caller check in the implementation. Any unprivileged address can call it with `disable_rewards: true` to consume the per-block reward slot, permanently preventing the sequencer from distributing rewards to stakers and delegators.

### Finding Description

The `IStakingRewardsManager::update_rewards` function is the consensus-era reward distribution entry point. The spec explicitly states its access control is **"Only starkware sequencer"**: [1](#0-0) 

However, the implementation contains no such check. The function body begins with `general_prerequisites()` (a pause check) and a block-number guard, but no caller identity assertion: [2](#0-1) 

The block-number guard uses a **single global** `last_reward_block` storage variable. Once any caller writes to it for the current block, the sequencer's call for that same block reverts with `REWARDS_ALREADY_UPDATED`: [3](#0-2) 

When `disable_rewards = true`, the function updates `last_reward_block` but skips all reward accounting and returns immediately — no rewards are credited to the staker or their pool.

The function signature accepts any `staker_address` and any `disable_rewards` flag: [4](#0-3) 

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at every new block:

1. Writes `last_reward_block = current_block` with no rewards distributed.
2. The sequencer's subsequent call for the same block reverts (`REWARDS_ALREADY_UPDATED`).
3. Stakers and all delegators in their pools receive zero rewards for that block.
4. Repeated every block → **permanent freezing of all unclaimed yield** for all stakers and delegators.

Because `last_reward_block` is global (not per-staker), a single call per block is sufficient to deny rewards to the entire protocol.

**Impact: High — Permanent freezing of unclaimed yield.**

### Likelihood Explanation

- No special role, key, or privilege is required.
- The call costs only gas.
- The attacker needs only a valid `staker_address` (any active staker, publicly readable from events).
- The attack is trivially automatable: watch for new blocks, front-run the sequencer each block.

**Likelihood: High.**

### Recommendation

Add a caller check matching the spec's stated access control. In Cairo, this is typically done by asserting `get_caller_address() == sequencer_address` or by using a stored `sequencer_address` configuration value, analogous to how `update_rewards_from_attestation_contract` enforces its caller: [5](#0-4) 

Apply the same pattern to `update_rewards`:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
```

Store the sequencer address at initialization and expose a governance-controlled setter, similar to how `attestation_contract` is managed.

### Proof of Concept

1. Staker stakes and K epochs pass — staker is active and eligible for consensus rewards.
2. Attacker (any address) calls `update_rewards(staker_address, disable_rewards: true)` at block N.
3. `last_reward_block` is set to N; no rewards are distributed.
4. Sequencer calls `update_rewards(staker_address, disable_rewards: false)` at block N → reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker's `unclaimed_rewards_own` is unchanged; delegators receive nothing.
6. Attacker repeats at block N+1, N+2, … → staker and all delegators accumulate zero rewards indefinitely. [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
