### Title
Missing Access Control on `update_rewards` Allows Any Caller to Redirect Block Proposer Rewards to Attacker-Controlled Staker — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` is documented as callable "Only starkware sequencer" but has **no on-chain access control enforcement**. Any registered staker can call it, front-run the sequencer, and redirect per-block consensus rewards to their own staker address, causing the legitimate block proposer to lose their unclaimed yield for that block.

---

### Finding Description

The protocol spec explicitly restricts `update_rewards` to the Starkware sequencer:

> **access control:** Only starkware sequencer. [1](#0-0) 

However, the implementation contains no such check. The function body only asserts:

1. Contract is not paused (`general_prerequisites()`)
2. `current_block_number > last_reward_block` (one call per block)
3. Staker exists and is active
4. Staker has non-zero STRK balance at the current epoch [2](#0-1) 

There is no `only_sequencer()` guard, no caller identity check, and no error variant for an unauthorized caller. Compare this with `update_rewards_from_attestation_contract`, which **does** enforce its access control with an explicit `CALLER_IS_NOT_ATTESTATION_CONTRACT` error: [3](#0-2) 

The global `last_reward_block` storage variable is the critical shared state: it is written unconditionally before the rewards branch, so whichever caller wins the race for a given block number locks out all subsequent callers for that block. [4](#0-3) 

---

### Impact Explanation

**High — Theft of unclaimed yield.**

In the V3 consensus-rewards phase, the sequencer is expected to call `update_rewards(block_proposer_staker, false)` once per block to credit the block proposer with their share of inflation rewards. Because `last_reward_block` is a single global slot, only one successful call is possible per block number.

An attacker who is a registered staker with non-zero effective balance can:

1. Call `update_rewards(attacker_staker_address, disable_rewards: false)` in the same block before the sequencer's transaction is processed.
2. `last_reward_block` is updated to the current block; the attacker's staker accrues `strk_block_rewards` proportional to their stake.
3. The sequencer's call for the legitimate block proposer reverts with `REWARDS_ALREADY_UPDATED`.
4. The block proposer receives **zero** rewards for that block.
5. The attacker later calls `claim_rewards(attacker_staker_address)` to extract the stolen yield.

---

### Likelihood Explanation

**Medium.** The attacker must:
- Be a registered, active staker with non-zero effective STRK balance (K-epoch delay must have passed).
- Submit a transaction that is ordered before the sequencer's own `update_rewards` system call in the same block.

On Starknet today the sequencer controls ordering, but this is not enforced on-chain. The absence of any on-chain guard means the vulnerability is latent and becomes immediately exploitable if sequencer ordering guarantees weaken (e.g., decentralized sequencing, forced-inclusion mechanisms, or sequencer bugs). The attack requires no special privilege beyond being a staker.

---

### Recommendation

Add an access control guard to `update_rewards` that restricts the caller to the authorized sequencer address, mirroring the pattern already used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_sequencer(); // <-- add this
    self.general_prerequisites();
    ...
}
```

Alternatively, store the expected sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` at the top of the function.

---

### Proof of Concept

```
Setup:
  - Consensus rewards are active (post-consensus_rewards_first_epoch + K epochs).
  - Attacker has staked min_stake and waited K epochs (effective balance > 0).
  - Block N is about to be produced by legitimate block proposer P.

Attack (block N):
  1. Attacker submits tx: staking.update_rewards(attacker_staker, disable_rewards=false)
     → last_reward_block := N
     → attacker_staker.unclaimed_rewards_own += strk_block_rewards (proportional to attacker stake)
  2. Sequencer submits tx: staking.update_rewards(P_staker, disable_rewards=false)
     → PANICS: current_block_number (N) > last_reward_block (N) is FALSE
     → P_staker receives 0 rewards for block N.

Extraction:
  3. Attacker calls staking.claim_rewards(attacker_staker)
     → STRK transferred to attacker's reward_address.
```

The entry path is fully unprivileged: `update_rewards` is a public ABI function with no caller restriction. [6](#0-5)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

        fn _update_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            strk_total_rewards: Amount,
            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.
            let staker_own_rewards = self
                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
            // Emit event.
            self
                .emit(
                    Events::StakerRewardsUpdated {
                        staker_address, staker_rewards, pool_rewards: pool_rewards_list.span(),
                    },
                );

            // Write staker rewards to storage.
            self.write_staker_info(:staker_address, :staker_info);
        }
