### Title
Missing `is_contract_class_declared` Guard for Cairo V0 Classes in `get_compiled_class` When `class_manager_handle` Is `Some` — (`crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class` applies a declaration-at-`state_number` check only for `ContractClass::V1` when the class manager path is taken. The `ContractClass::V0` arm unconditionally returns `Ok(RunnableCompiledClass::V0(...))`. A TODO comment in the code explicitly acknowledges this omission. An attacker can trigger RPC simulation or fee estimation to execute a Cairo V0 class that was not declared at the requested `state_number`, producing an authoritative-looking wrong result.

---

### Finding Description

In `get_compiled_class`, when `self.class_manager_handle` is `Some`, the function fetches the compiled class from the class manager and then branches on the variant: [1](#0-0) 

For `ContractClass::V1`, `is_contract_class_declared` is called and the function returns `Err(StateError::UndeclaredClassHash(class_hash))` if the class is not declared at `self.state_number`: [2](#0-1) 

For `ContractClass::V0`, no such check exists. The TODO comment at line 136 explicitly acknowledges this: [3](#0-2) 

`is_contract_class_declared` checks whether the class's `get_class_definition_block_number` is before `state_number`: [4](#0-3) 

The fallback path (when `class_manager_handle` is `None`) correctly enforces the declaration boundary for V0 classes via `get_deprecated_class_definition_at(state_number, class_hash)`: [5](#0-4) 

The `class_manager_handle` is passed through all public RPC execution entry points — `execute_call`, `estimate_fee`, and `simulate_transactions` — and is forwarded directly into `ExecutionStateReader`: [6](#0-5) 

---

### Impact Explanation

When a user submits an RPC request (`starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`) at a historical `state_number` (e.g., block N−k), and the transaction triggers a library call to a Cairo V0 `class_hash` that:

- **exists** in the class manager (declared at block N), but
- **was not declared** at `state_number` (block N−k),

the function returns `Ok(RunnableCompiledClass::V0(...))` instead of `Err(StateError::UndeclaredClassHash(class_hash))`. The RPC layer then returns a successful execution trace or fee estimate for a transaction that should have failed with an undeclared class error. This is a concrete wrong value from an authoritative RPC endpoint — matching the "High: RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value" impact category.

---

### Likelihood Explanation

The class manager path is active whenever the node is configured with a class manager client (the `class_manager_handle: Some(...)` branch). Historical simulations at a `state_number` before a V0 class was declared are a normal use case. The attacker only needs to:

1. Know a V0 `class_hash` that is in the class manager but was declared after the target `state_number`.
2. Craft or find a contract that performs a library call with that `class_hash` (library call class hashes can be passed as parameters in Cairo).
3. Submit an RPC simulation request at the appropriate historical `state_number`.

No privileged access is required. All inputs are available to any unprivileged RPC caller.

---

### Recommendation

Apply the same `is_contract_class_declared` guard to the `ContractClass::V0` arm, mirroring the `V1` logic:

```rust
ContractClass::V0(deprecated_contract_class) => {
    let is_declared = is_contract_class_declared(
        &self.storage_reader.begin_ro_txn().map_err(storage_err_to_state_err)?,
        &class_hash,
        self.state_number,
    )
    .map_err(|e| StateError::StateReadError(e.to_string()))?;

    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
```

Remove the TODO comment once the fix is applied. The `get_class_definition_block_number` function used inside `is_contract_class_declared` already handles V0 classes correctly via the storage reader, so no additional fix to that function is needed.

### Proof of Concept

1. Declare a Cairo V0 class at block N. The class manager now holds it as `ContractClass::V0`.
2. Issue `starknet_simulateTransactions` at `state_number = N−1` (before the declaration).
3. The simulated transaction calls a contract that performs a `library_call` with the V0 `class_hash`.
4. `get_compiled_class` is invoked with that `class_hash`.
5. The class manager returns `ContractClass::V0(...)`.
6. The `V0` arm executes without calling `is_contract_class_declared`.
7. `Ok(RunnableCompiledClass::V0(...))` is returned and the library call proceeds.
8. The simulation returns a successful trace — the undeclared class was executed.

Expected: `StateError::UndeclaredClassHash(class_hash)` propagated as an execution error.
Actual: Successful execution using the undeclared V0 class.

pub(crate) fn is_contract_class_declared(
    txn: &StorageTxn<'_, RO>,
    class_hash: &ClassHash,
    state_number: StateNumber,
) -> Result<bool, ExecutionUtilsError> {
    Ok(txn
        .get_state_reader()?
        .get_class_definition_block_number(class_hash)?
        .is_some_and(|block_number| state_number.is_after(block_number)))
}
