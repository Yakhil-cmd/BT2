### Title
Missing Validation of `DKGParticipantKeys` When `DKGIndexMap` Is `nil` in `NewEpochCommit` Constructor - (File: `model/flow/epoch.go`)

### Summary

`NewEpochCommit` in `model/flow/epoch.go` intentionally skips all `DKGParticipantKeys` / `DKGIndexMap` consistency checks when `DKGIndexMap == nil`. This is a deliberate but temporary backward-compatibility bypass (marked `TODO(mainnet27)`). The result is that a well-formed `EpochCommit` with a non-nil `DKGGroupKey`, a valid `ClusterQCs` list, but a completely empty/nil `DKGParticipantKeys` and a nil `DKGIndexMap` passes the constructor without error and is stored in the Protocol State. The downstream `IsValidEpochCommit` validator in `state/protocol/validity.go` does **not** share this bypass — it always enforces `len(DKGParticipantKeys) == len(DKGIndexMap)` — but the constructor is the gate that produces the trusted `EpochCommit` object that gets committed to the epoch state machine.

### Finding Description

In `NewEpochCommit` (`model/flow/epoch.go`, lines 393–428), the only two hard checks are:

1. `DKGGroupKey != nil`
2. `len(ClusterQCs) > 0`

All `DKGParticipantKeys` and `DKGIndexMap` invariant checks are wrapped in:

```go
// TODO(mainnet27): remove this conditional: https://github.com/onflow/flow-go/issues/6772
if untrusted.DKGIndexMap != nil {
    // ... all key-count and index-range checks ...
}
``` [1](#0-0) 

When `DKGIndexMap` is `nil`, the constructor accepts any value of `DKGParticipantKeys` — including `nil` or an empty slice — and returns a valid `*EpochCommit`. The struct invariant documented on the type itself (`len(DKGParticipantKeys) == len(DKGIndexMap)`) is therefore not enforced at construction time for this case. [2](#0-1) 

The downstream `IsValidEpochCommit` in `state/protocol/validity.go` (lines 157–220) does **not** have this bypass: it always checks `len(commit.DKGParticipantKeys) != len(commit.DKGIndexMap)` (where a nil map has length 0), so a commit with `DKGIndexMap=nil` and `DKGParticipantKeys=[]key{...}` would be caught there. [3](#0-2) 

However, the constructor is the trust boundary. The `EpochCommit` type is annotated `//structwrite:immutable - mutations allowed only within the constructor`, meaning the constructor is the sole place where the invariant should be enforced. The `TODO(mainnet27)` comment acknowledges this is a known gap, but the gap is currently open. [4](#0-3) 

### Impact Explanation

An `EpochCommit` with `DKGIndexMap=nil` and non-empty `DKGParticipantKeys` (or vice versa) can be constructed and stored. When the `DKG` adapter (`state/protocol/inmem/dkg.go`) is used to serve Random Beacon key lookups, `KeyShare` indexes directly into `DKGParticipantKeys[index]` using a value from `DKGIndexMap`. If `DKGIndexMap` is nil, all `Index()` and `KeyShare()` calls return `IdentityNotFoundError`, effectively making the Random Beacon committee appear empty to any node querying it. [5](#0-4) 

This means a malformed `EpochCommit` that passes the constructor could, if it somehow reached the Protocol State (bypassing `IsValidEpochCommit`), cause consensus nodes to be unable to produce or verify Random Beacon signatures for the committed epoch, triggering Epoch Fallback Mode or halting consensus.

### Likelihood Explanation

The `IsValidEpochCommit` validator is called during both bootstrapping (`state/protocol/badger/state.go`) and live epoch transitions (`state/protocol/protocol_state/epochs/happy_path_statemachine.go`), so the malformed commit would be rejected before being persisted in the normal flow. However, the constructor itself is the stated trust boundary for the immutable type, and the missing check means any internal code path that calls `NewEpochCommit` directly (e.g., deserialization, test fixtures, or future code) and passes `DKGIndexMap=nil` with non-empty `DKGParticipantKeys` will silently produce an invariant-violating object. The `EpochCommitFixture` in `utils/unittest/fixtures.go` already does exactly this — it creates an `EpochCommit` with `DKGParticipantKeys` set but `DKGIndexMap=nil`. [6](#0-5) 

### Recommendation

Remove the `if untrusted.DKGIndexMap != nil` conditional guard in `NewEpochCommit` and always enforce the invariant `len(DKGParticipantKeys) == len(DKGIndexMap)`. If backward compatibility with pre-`mainnet27` `EpochCommit` objects (which have `DKGIndexMap=nil`) is required during a migration window, the check should be: if `DKGIndexMap` is nil then `DKGParticipantKeys` must also be nil or empty, and vice versa. The `TODO(mainnet27)` issue (#6772) should be resolved to remove the conditional entirely once the migration is complete.

### Proof of Concept

The following input to `NewEpochCommit` passes without error today:

```go
commit, err := flow.NewEpochCommit(flow.UntrustedEpochCommit{
    Counter:            1,
    ClusterQCs:         validClusterQCs,
    DKGGroupKey:        validDKGGroupKey,
    DKGParticipantKeys: []crypto.PublicKey{key1, key2, key3}, // 3 keys
    DKGIndexMap:        nil,                                   // nil — all checks skipped
})
// err == nil, commit != nil
// commit.DKGParticipantKeys has 3 entries
// commit.DKGIndexMap is nil
// Invariant len(DKGParticipantKeys)==len(DKGIndexMap) is VIOLATED (3 != 0)
``` [7](#0-6)

func NewEpochCommit(untrusted UntrustedEpochCommit) (*EpochCommit, error) {
	if untrusted.DKGGroupKey == nil {
		return nil, fmt.Errorf("DKG group key must not be nil")
	}
	if len(untrusted.ClusterQCs) == 0 {
		return nil, fmt.Errorf("cluster QCs list must not be empty")
	}
	// TODO(mainnet27): remove this conditional: https://github.com/onflow/flow-go/issues/6772
	if untrusted.DKGIndexMap != nil {
		// enforce invariant: len(DKGParticipantKeys) == len(DKGIndexMap)
		n := len(untrusted.DKGIndexMap) // size of the DKG committee
		if len(untrusted.DKGParticipantKeys) != n {
			return nil, fmt.Errorf("number of %d Random Beacon key shares is inconsistent with number of DKG participants (len=%d)", len(untrusted.DKGParticipantKeys), len(untrusted.DKGIndexMap))
		}

		// enforce invariant: DKGIndexMap values form the set {0, 1, ..., n-1} where n=len(DKGParticipantKeys)
		encounteredIndex := make([]bool, n)
		for _, index := range untrusted.DKGIndexMap {
			if index < 0 || index >= n {
				return nil, fmt.Errorf("index %d is outside allowed range [0,n-1] for a DKG committee of size n=%d", index, n)
			}
			if encounteredIndex[index] {
				return nil, fmt.Errorf("duplicated DKG index %d", index)
			}
			encounteredIndex[index] = true
		}
	}

	return &EpochCommit{
		Counter:            untrusted.Counter,
		ClusterQCs:         untrusted.ClusterQCs,
		DKGGroupKey:        untrusted.DKGGroupKey,
		DKGParticipantKeys: untrusted.DKGParticipantKeys,
		DKGIndexMap:        untrusted.DKGIndexMap,
	}, nil
}

	index, exists := d.DKGIndexMap[nodeID]
