### Title
Missing `me`-in-participants Validation in `generate_triple` / `generate_triple_many` Allows Corrupt Triple Share to Propagate into Presign - (File: `src/ecdsa/ot_based_ecdsa/triples/generation.rs`)

### Summary
`generate_triple` and `generate_triple_many` are the only public protocol entry-points in the library that do **not** validate that the caller's own identity (`me`) is a member of the supplied `participants` list. Every other protocol function performs this check. If `me` is absent from `participants`, the internal `do_generation_many` routine runs with a structurally invalid identity, producing a `TripleShare` whose scalar values are evaluated at an unrecognized point and a `TriplePub` whose `participants` field does not contain `me`. Because `presign` explicitly omits the reciprocal check (that the presign participant set was present for triple generation), this corrupted triple can silently flow into a presign session, producing an inconsistent presignature.

### Finding Description

Every other public protocol initializer in the codebase guards against a caller supplying a `me` that is not in the participant list:

- `assert_key_invariants` (DKG) [1](#0-0) 
- `ot_based_ecdsa::presign` [2](#0-1) 
- `ot_based_ecdsa::sign` [3](#0-2) 
- `robust_ecdsa::presign` [4](#0-3) 
- `robust_ecdsa::sign` [5](#0-4) 
- `frost::presign` [6](#0-5) 
- `ckd` [7](#0-6) 

`generate_triple` and `generate_triple_many` delegate all validation to `validate_triple_inputs`, which checks participant count, threshold bounds, and uniqueness, but **never checks that `me` is present**: [8](#0-7) 

`generate_triple` then passes the unvalidated `me` directly into `do_generation`: [9](#0-8) 

Inside `do_generation_many`, `ParticipantList::others(me)` is used to enumerate peers for private-share distribution:

```rust
for p in participants.others(me) {
    ...
    chan.send_private(wait6, p, &c_i_j_v)?;
}
``` [10](#0-9) 

`others` is defined as:
```rust
pub fn others(&self, me: Participant) -> impl Iterator<Item = Participant> + '_ {
    self.participants.iter().filter(move |x| **x != me).copied()
}
``` [11](#0-10) 

When `me` is absent from `participants`, `others(me)` returns **all** participants. The invalid `me` instance therefore:
1. Sends its private polynomial evaluations to every participant (including the slot that should be its own).
2. Waits for private messages from every participant via `recv_from_others` (since `me` is not filtered out).
3. Evaluates the aggregated polynomials at `me`'s scalar identifier — a point that is not a recognized Shamir evaluation point for the session.

The resulting `TripleShare` has `(a, b, c)` values at an unrecognized point, and the `TriplePub` records `participants` that do not include `me`: [12](#0-11)

```rust
fn validate_triple_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
) -> Result<(ParticipantList, ReconstructionLowerBound), InitializationError> {
    let threshold = threshold.into();
    let threshold_value = threshold.value();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    // Spec 1.1
    if threshold_value > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold_value,
            max: participants.len(),
        });
    }
    if threshold_value < 2 {
        return Err(InitializationError::ThresholdTooSmall {
            threshold: threshold_value,
            min: 2,
        });
    }
    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;
    Ok((participants, threshold))
}
