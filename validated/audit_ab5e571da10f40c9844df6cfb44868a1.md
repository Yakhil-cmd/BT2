### Title
Missing Caller Validation on `update_rewards` Allows Any User to Permanently Deny Staker Yield - (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` is documented to be callable only by the Starknet sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` to consume the per-block reward slot before the sequencer does, permanently denying yield to any targeted staker for that block. The attack is repeatable every block.

---

### Finding Description

The spec explicitly restricts `update_rewards` to the Starknet sequencer:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, performs no such check. After `general_prerequisites()` (a pause guard), the only gate is a per-block deduplication check on `last_reward_block`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no assert on get_caller_address() ...
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    // distribute rewards ...
``` [2](#0-1) 

The `last_reward_block` is written unconditionally before the `disable_rewards` branch, so whichever caller wins the race for a given block number permanently consumes the reward slot for that block. [3](#0-2) 

---

### Impact Explanation

An attacker who calls `update_rewards(victim_staker, disable_rewards: true)` before the sequencer does in any block:

1. Sets `last_reward_block` to the current block number.
2. Returns early without distributing rewards (`disable_rewards=true` short-circuits).
3. When the sequencer subsequently calls `update_rewards` for the same block, it reverts with `REWARDS_ALREADY_UPDATED`.

The staker and their delegation pool receive zero rewards for that block. Because the attack costs only a transaction fee and can be repeated every block, an attacker can continuously deny yield to any targeted staker. This maps directly to **High: Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is public with no role guard.
- The attacker needs only to know a valid, active staker address (all stakers are enumerable via the `stakers` vector in storage and `NewStaker` events).
- The attack requires no capital, no privileged key, and no external dependency — only a transaction submitted before the sequencer's.
- On Starknet, transaction ordering within a block is controlled by the sequencer, but the sequencer itself is the intended caller; a malicious user submitting a transaction in a prior block or at the start of the same block can race the sequencer's own inclusion of `update_rewards`.

---

### Recommendation

Add a caller check that restricts `update_rewards` to the Starknet sequencer address (or a configured operator address stored in contract state), analogous to how `update_rewards_from_attestation_contract` restricts its caller:

```cairo
self.assert_caller_is_attestation_contract(); // pattern to follow
``` [4](#0-3) 

Concretely, store the expected sequencer/operator address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` at the top of `update_rewards`, before the `last_reward_block` write.

---

### Proof of Concept

1. Staker Alice stakes the minimum amount and waits K epochs for her balance to become effective.
2. Consensus rewards are active (`is_pre_consensus()` returns false).
3. Attacker Bob monitors the chain. At the start of each new block, Bob submits:
   ```
   staking.update_rewards(alice_staker_address, disable_rewards: true)
   ```
4. Bob's transaction is included before the sequencer's own `update_rewards` call.
5. `last_reward_block` is set to the current block; the function returns without distributing rewards.
6. The sequencer's `update_rewards` call reverts with `REWARDS_ALREADY_UPDATED`.
7. Alice's `unclaimed_rewards_own` is never incremented for that block.
8. Bob repeats step 3 every block. Alice accrues zero yield indefinitely despite having an active stake.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
