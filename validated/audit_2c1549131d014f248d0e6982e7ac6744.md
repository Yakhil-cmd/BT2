### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Freeze All Staker Yield - (`File: src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is documented to be callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged address can call it with `disable_rewards: true`, consuming the single per-block reward slot (`last_reward_block`) without distributing any rewards. Because `last_reward_block` is a **global** variable shared across all stakers, this permanently freezes yield for every staker in the protocol for that block. Repeated every block, this constitutes a complete, permanent freeze of all consensus-era staker and delegator yield.

---

### Finding Description

The spec at `docs/spec.md:1644-1645` states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo:1447-1507` enforces no such check:

```rust
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only checks: unpaused + caller != zero
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ... staker validation ...
        self.last_reward_block.write(current_block_number);  // global write

        if disable_rewards || self.is_pre_consensus() {
            return;   // exits without distributing rewards
        }
        // ... reward distribution ...
    }
}
```

`general_prerequisites` only asserts the contract is unpaused and the caller is non-zero:

```rust
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
```

`last_reward_block` is a single global storage slot:

```rust
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
```

Because the check `current_block_number > self.last_reward_block.read()` is global (not per-staker), the first call to `update_rewards` in any block — regardless of caller or `disable_rewards` value — consumes the slot for **all** stakers. Any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

---

### Impact Explanation

An attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)` as the first transaction in every block. This:

1. Passes all checks (contract unpaused, caller non-zero, staker active, block is new).
2. Writes `last_reward_block = current_block`.
3. Returns immediately at `if disable_rewards || self.is_pre_consensus() { return; }` — **zero rewards distributed**.
4. Blocks every legitimate sequencer call for that block with `REWARDS_ALREADY_UPDATED`.

Repeated every block, this permanently freezes all consensus-era staker and delegator yield. No staker or delegator can recover their rewards for any griefed block; the loss is irreversible. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- The function is public with no access control.
- The attacker needs only a valid (active, post-K-epoch) staker address to pass the staker validation check — any staker address visible on-chain suffices.
- The cost is one transaction per block. On Starknet, transaction fees are low.
- No special privilege, leaked key, or external dependency is required.
- The attack is reachable immediately after `consensus_rewards_first_epoch` is reached.

---

### Recommendation

Add a sequencer-only guard to `update_rewards`, consistent with the spec. The simplest approach is to assert `get_caller_address() == sequencer_address` at the top of the function, where `sequencer_address` is stored in contract storage and set during initialization (or via a privileged setter). Alternatively, restrict it to the existing `only_app_governor` or a dedicated role.

---

        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            self.last_reward_block.write(current_block_number);

            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
