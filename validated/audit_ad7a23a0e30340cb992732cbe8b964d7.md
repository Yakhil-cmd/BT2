### Title
Missing Commission Slippage Protection in `enter_delegation_pool` / `add_to_delegation_pool` — (File: src/pool/pool.cairo)

---

### Summary

`enter_delegation_pool()` and `add_to_delegation_pool()` in the Pool contract accept no `max_commission` parameter. When a staker holds an active `commission_commitment`, they can increase commission up to `max_commission` at any time. A staker can front-run a delegator's entry transaction by spiking commission, causing the delegator to earn far less yield than expected — with no on-chain protection available to the delegator.

---

### Finding Description

`enter_delegation_pool()` and `add_to_delegation_pool()` in `src/pool/pool.cairo` accept only `reward_address` and `amount` parameters. There is no `max_commission` parameter that would let a delegator bound the commission rate they are willing to accept at entry time. [1](#0-0) [2](#0-1) 

Commission is stored in the staking contract and applied at reward-distribution time. Normally, commission can only decrease. However, when a staker has an active `commission_commitment` (set via `set_commission_commitment()`), the internal `update_commission()` function allows commission to be set to **any value ≤ `max_commission`**, including values **higher** than the current commission: [3](#0-2) 

The codebase itself acknowledges this risk with the comment directly above `set_commission_commitment`: [4](#0-3) 

> **Note**: Current commission increase safeguards still allow for sudden commission changes.

A staker can exploit this by:
1. Setting a `commission_commitment` with a high `max_commission` (e.g., `10000` = 100%) via `set_commission_commitment()`.
2. Advertising a low commission (e.g., 1%) to attract delegators.
3. Observing a delegator's pending `enter_delegation_pool` transaction in the mempool.
4. Front-running it by calling `set_commission(10000)` — valid because the commitment is active and `10000 <= max_commission`.
5. The delegator's transaction executes, entering the pool now bound to 100% commission.
6. All future reward distributions apply 100% commission, leaving the delegator with zero yield.

The `enter_delegation_pool` implementation performs no commission check whatsoever: [5](#0-4) 

The `set_commission_commitment` function allows `max_commission` to be set as high as `COMMISSION_DENOMINATOR` (10000 = 100%): [6](#0-5) 

The constraint only requires `max_commission >= current_commission`, so a staker with 1% commission can immediately commit to a ceiling of 100%. [7](#0-6) 

---

### Impact Explanation

The delegator's future rewards are entirely captured by the staker. In the extreme case (commission = 100%), the delegator receives zero rewards on all future epochs for as long as they remain in the pool. This constitutes **theft of unclaimed yield** (High impact). The delegator's principal is not immediately at risk, but their entire yield stream is redirected to the staker.

---

### Likelihood Explanation

Any staker can call `set_commission_commitment()` at any time to establish a high ceiling, then call `set_commission()` to spike commission immediately before a delegator's entry transaction lands. No privileged access, leaked key, or external dependency is required — only the staker's own address. Starknet's sequencer ordering makes transaction ordering manipulation realistic. The attack is economically rational for a staker seeking to maximize commission revenue from incoming delegators.

---

### Recommendation

Add a `max_commission: Commission` parameter to `enter_delegation_pool()` and `add_to_delegation_pool()`. Before transferring funds, assert that the pool's current commission does not exceed the caller's stated maximum:

```rust
fn enter_delegation_pool(
    ref self: ContractState,
    reward_address: ContractAddress,
    amount: Amount,
    max_commission: Commission,  // <-- new
) {
    // ... existing asserts ...
    let current_commission = self.get_current_commission(); // read from staking contract
    assert!(current_commission <= max_commission, "Commission exceeds max acceptable");
    // ... rest of logic ...
}
```

Apply the same guard to `add_to_delegation_pool()` and `switch_delegation_pool()`.

---

### Proof of Concept

```
1. Staker stakes and calls set_commission(commission: 100)  // 1%
2. Staker calls set_commission_commitment(max_commission: 10000, expiration_epoch: E+1)
   // Now staker can raise commission to 100% at any time before epoch E+1
3. Delegator sees commission=1%, approves tokens, submits enter_delegation_pool(reward_address, amount)
4. Staker observes pending tx, front-runs with set_commission(commission: 10000)  // 100%
   // Allowed: 10000 <= max_commission AND 10000 != 100 (old_commission)
5. Delegator's enter_delegation_pool executes — no commission check, succeeds
6. Next reward epoch: update_rewards() calculates pool rewards with commission=10000
   => commission_rewards = 100% of pool rewards
   => pool_rewards sent to pool contract = 0
7. Delegator claims rewards: receives 0 STRK
``` [8](#0-7) [9](#0-8)

        /// changes.
        /// **Note**: Updating epoch info can impact the commission commitment expiration date.
        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );

        fn enter_delegation_pool(
            ref self: ContractState, reward_address: ContractAddress, amount: Amount,
        ) {
            // Asserts.
            self.assert_staker_is_active();
            let pool_member = get_caller_address();
            assert!(
                self.pool_member_info.read(pool_member).is_none(), "{}", Error::POOL_MEMBER_EXISTS,
            );
            assert!(amount.is_non_zero(), "{}", GenericError::AMOUNT_IS_ZERO);
            let token_dispatcher = self.token_dispatcher.read();
            let token_address = token_dispatcher.contract_address;
            assert!(token_address != pool_member, "{}", Error::POOL_MEMBER_IS_TOKEN);
            assert!(token_address != reward_address, "{}", GenericError::REWARD_ADDRESS_IS_TOKEN);
            // Transfer funds from the delegator to the staking contract.
            let staker_address = self.staker_address.read();
            transfer_from_delegator(:pool_member, :amount, :token_dispatcher);
            self.transfer_to_staking_contract(:amount, :token_dispatcher, :staker_address);

            self.set_member_balance(:pool_member, :amount);

            // Create the pool member record.
            self
                .pool_member_info
                .write(pool_member, VInternalPoolMemberInfoTrait::new_latest(:reward_address));

            // Emit events.
            self
                .emit(
                    Events::NewPoolMember { pool_member, staker_address, reward_address, amount },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake: Zero::zero(), new_delegated_stake: amount,
                    },
                );
        }
