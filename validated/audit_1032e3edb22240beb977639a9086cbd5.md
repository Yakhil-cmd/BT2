### Title
Silent Conversion Success on Self-Destructed External CRC21 Contract Causes Permanent IBC Voucher / Source-Coin Loss — (File: x/cronos/keeper/evm.go)

---

### Summary

`ConvertCoinFromNativeToCRC21` performs no contract-existence check at conversion time. `ensureContractCode` is called only at token-mapping registration (`RegisterOrUpdateTokenMapping`), not when the conversion is actually executed. If an externally-registered CRC21 contract is self-destructed after registration, the EVM call to `mint_by_cronos_module` (or `transfer_from_cronos_module`) returns success with empty data — the documented EVM behavior for calls to non-existent addresses — while the user's IBC vouchers are permanently locked at the dead contract's Cosmos address and no CRC21 tokens are ever minted.

---

### Finding Description

**Registration path — check exists:**

`RegisterOrUpdateTokenMapping` calls `ensureContractCode` before writing the mapping for both source and non-source denoms. [1](#0-0) [2](#0-1) 

`ensureContractCode` queries the EVM for bytecode and returns an error if the address has no code: [3](#0-2) 

**Conversion path — check absent:**

`ConvertCoinFromNativeToCRC21` retrieves the stored contract address via `GetContractByDenom` and immediately proceeds to bank operations and the EVM call, with no `ensureContractCode` guard: [4](#0-3) 

For **non-source coins** (IBC / gravity denoms):
1. `bankKeeper.SendCoins` transfers the user's IBC vouchers to `sdk.AccAddress(contract.Bytes())` — this bank-level send succeeds regardless of whether EVM code exists at that address.
2. `CallModuleCRC21(ctx, contract, "mint_by_cronos_module", ...)` is called on the dead address. [5](#0-4) 

`CallModuleCRC21` calls `CallEVM` → `evmKeeper.ApplyMessage`. Per EVM specification, a `CALL` to an address with no code returns `(success=true, data=[])`; `res.Failed()` is therefore `false`, and the function returns `nil` — no error, no minted tokens. [6](#0-5) 

For **source coins** (`cronos0x...`):
1. `bankKeeper.BurnCoins` destroys the user's native tokens.
2. `CallModuleCRC21(ctx, contract, "transfer_from_cronos_module", ...)` silently succeeds on the dead contract — no CRC21 tokens are released. [7](#0-6) 

The IBC hook `ConvertVouchersToEvmCoins` is the primary unprivileged entry point that reaches this code: [8](#0-7) 

---

### Impact Explanation

**Critical** — Permanent, unrecoverable loss of IBC vouchers or source-coin assets for honest users:

- Non-source path: IBC vouchers are sent to the dead contract's Cosmos address. No module account controls that address; the coins are locked forever. No CRC21 tokens are minted.
- Source path: native `cronos0x…` coins are burned. No CRC21 tokens are released. The user's balance is destroyed with no compensation.

This is an unauthorized, irreversible balance/accounting change for IBC vouchers and CRC21 assets, matching the Critical impact tier.

---

### Likelihood Explanation

1. The Cronos admin registers an external CRC21 contract via `MsgUpdateTokenMapping`. `ensureContractCode` passes at that moment.
2. The contract owner (a separate, unprivileged Cronos actor) calls `selfdestruct` on the contract — either intentionally (malicious contract deployer) or as part of a contract upgrade.
3. Any user who subsequently calls `MsgConvertVouchers` (or triggers the IBC auto-conversion hook) for that denom loses their funds silently.

The contract owner is not a Cronos admin or validator; they are unprivileged within the Cronos protocol. The admin's prior trust of the contract does not prevent the contract owner from later destroying it.

---

### Recommendation

**Short term:** Add an `ensureContractCode` call inside `ConvertCoinFromNativeToCRC21` immediately after `GetContractByDenom` returns a contract address, before any bank or EVM operations are performed. This mirrors the guard already present in `RegisterOrUpdateTokenMapping`.

```go
contract, found := k.GetContractByDenom(ctx, coin.Denom)
if !found { ... }
// ADD: guard against self-destructed contracts
if err := k.ensureContractCode(ctx, contract); err != nil {
    return fmt.Errorf("contract for denom %s is invalid: %w", coin.Denom, err)
}
```

**Long term:** Consider adding a periodic or on-demand mechanism to validate that all registered external contracts still have live bytecode, and emit a governance alert or automatically remove the mapping if a contract is found to be dead.

---

### Proof of Concept

1. Attacker deploys a Solidity contract implementing the CRC21 interface (`mint_by_cronos_module`, `burn_by_cronos_module`, `transfer_from_cronos_module`) with an owner-callable `selfdestruct`.
2. Cronos admin calls `MsgUpdateTokenMapping` for denom `ibc/AAAA…` pointing to the attacker's contract. `ensureContractCode` passes — contract has code.
3. Attacker calls `selfdestruct` on their contract. EVM state for that address is now empty.
4. Victim user holds `ibc/AAAA…` vouchers and calls `MsgConvertVouchers`.
5. `ConvertVouchersToEvmCoins` → `ConvertCoinFromNativeToCRC21` is invoked.
6. `GetContractByDenom` returns the attacker's (now dead) address — no error, mapping still exists.
7. `bankKeeper.SendCoins(ctx, userAddr, deadContractCosmosAddr, ibcCoins)` — succeeds; IBC vouchers transferred to dead address.
8. `CallModuleCRC21(ctx, deadContract, "mint_by_cronos_module", userEVMAddr, amount)` — EVM call to address with no code returns `(success, [])`. `res.Failed()` is `false`. Function returns `nil`.
9. `ConvertCoinFromNativeToCRC21` returns `nil` — no error propagated.
10. User's IBC vouchers are permanently locked at the dead contract's Cosmos address. Zero CRC21 tokens minted. Funds are unrecoverable.

func (k Keeper) ensureContractCode(ctx sdk.Context, contract common.Address) error {
	if contract.Big().Cmp(big.NewInt(256)) < 0 {
		return errors.Wrapf(sdkerrors.ErrInvalidAddress,
			"crc21 contract must not be in precompile range: %s", contract.Hex())
	}
	resp, err := k.evmKeeper.Code(ctx, &evmtypes.QueryCodeRequest{
		Address: contract.Hex(),
	})
	if err != nil {
		return errors.Wrapf(sdkerrors.ErrInvalidAddress, "failed to query contract code (%s): %v", contract.Hex(), err)
	}
	if resp == nil || len(resp.Code) == 0 {
		return errors.Wrapf(sdkerrors.ErrInvalidRequest, "no contract code at address (%s)", contract.Hex())
	}
	return nil
}

func (k Keeper) ConvertCoinFromNativeToCRC21(ctx sdk.Context, sender common.Address, coin sdk.Coin, autoDeploy bool) error {
	if !types.IsValidCoinDenom(coin.Denom) {
		return fmt.Errorf("coin %s is not supported for conversion", coin.Denom)
	}
	var err error
	// external contract is returned in preference to auto-deployed ones
	contract, found := k.GetContractByDenom(ctx, coin.Denom)
	if !found {
		if !autoDeploy {
			return fmt.Errorf("no contract found for the denom %s", coin.Denom)
		}
		contract, err = k.DeployModuleCRC21(ctx, coin.Denom)
		if err != nil {
			return err
		}
		if err = k.SetAutoContractForDenom(ctx, coin.Denom, contract); err != nil {
			return err
		}

		k.Logger(ctx).Info(fmt.Sprintf("contract address %s created for coin denom %s", contract.String(), coin.Denom))
	}

	isSource := types.IsSourceCoin(coin.Denom)
	coins := sdk.NewCoins(coin)
	if isSource {
		// burn coins
		err = k.bankKeeper.SendCoinsFromAccountToModule(ctx, sdk.AccAddress(sender.Bytes()), types.ModuleName, sdk.NewCoins(coin))
		if err != nil {
			return err
		}
		err = k.bankKeeper.BurnCoins(ctx, types.ModuleName, coins)
		if err != nil {
			return err
		}
		// unlock crc tokens
		_, err = k.CallModuleCRC21(ctx, contract, "transfer_from_cronos_module", sender, coin.Amount.BigInt())
		if err != nil {
			return err
		}
	} else {
		// send coins to contract address
		err = k.bankKeeper.SendCoins(ctx, sdk.AccAddress(sender.Bytes()), sdk.AccAddress(contract.Bytes()), coins)
		if err != nil {
			return err
		}
		// mint crc tokens
		_, err = k.CallModuleCRC21(ctx, contract, "mint_by_cronos_module", sender, coin.Amount.BigInt())
		if err != nil {
			return err
		}
	}

	return nil
}
