### Title
Unbounded `NearPromise::And` Vector in `recursive_promise_create` Causes Permanent Fund Freeze via NEAR Gas Exhaustion in `execute_scheduled` - (File: `etc/xcc-router/src/lib.rs`)

---

### Summary

The XCC router's `recursive_promise_create` function iterates over an unbounded `NearPromise::And(Vec<Self>)` vector without any size limit. A user can schedule a `Delayed` XCC promise containing an arbitrarily large `And` vector while paying only the cheap scheduling EVM gas cost (fixed 5 Tgas equivalent). When `execute_scheduled` is later called, the unbounded iteration exhausts NEAR gas (max 300 Tgas per transaction), making the promise permanently unexecutable. Any wNEAR tokens transferred to the engine's implicit address as part of the promise's `attached_balance` are permanently frozen with no recovery path.

---

### Finding Description

`NearPromise` is a recursive enum defined in `engine-types/src/parameters/promise.rs`:

```rust
pub enum NearPromise {
    Simple(SimpleNearPromise),
    Then { base: Box<Self>, callback: SimpleNearPromise },
    And(Vec<Self>),   // ← unbounded vector, no size cap
}
``` [1](#0-0) 

The XCC router's `recursive_promise_create` iterates over this vector without any bound check:

```rust
NearPromise::And(promises) => {
    let indices: Vec<PromiseIndex> = promises
        .iter()
        .map(Self::recursive_promise_create)  // ← unbounded iteration
        .collect();
    env::promise_and(&indices)
}
``` [2](#0-1) 

`execute_scheduled` removes the promise from storage and then calls `promise_create`, which calls `recursive_promise_create`:

```rust
pub fn execute_scheduled(&mut self, nonce: U64) {
    let Some(promise) = self.scheduled_promises.remove(&nonce.0) else {
        env::panic_str("ERR_PROMISE_NOT_FOUND")
    };
    let promise_id = Self::promise_create(promise);
    env::promise_return(promise_id);
}
``` [3](#0-2) 

The critical asymmetry is in the XCC precompile. For `Eager` calls, the EVM gas cost includes the full execution cost (`ROUTER_EXEC_BASE + callback_count * ROUTER_EXEC_PER_CALLBACK`). For `Delayed` calls, only the fixed scheduling cost is charged:

```rust
CrossContractCallArgs::Delayed(call) => {
    let attached_near = call.total_near();
    let promise = PromiseCreateArgs {
        ...
        attached_gas: costs::ROUTER_SCHEDULE,  // ← only 5 Tgas, no execution cost
    };
    (promise, attached_near)
}
``` [4](#0-3) 

The EVM gas cost for a `Delayed` call is therefore:

```
cost = CROSS_CONTRACT_CALL_BASE + CROSS_CONTRACT_CALL_BYTE * input_len
     + ROUTER_SCHEDULE / CROSS_CONTRACT_CALL_NEAR_GAS
``` [5](#0-4) 

For 1,000 simple promises (~100 bytes each), `input_len ≈ 100,000 bytes`, giving an EVM gas cost of roughly `343,650 + 400,000 + 28,571 ≈ 772,221 EVM gas` — well within the 30M EVM gas limit. Yet executing those 1,000 promises in `recursive_promise_create` requires ~1,000 × `env::promise_create` host calls, each costing ~0.3 Tgas, totalling ~300 Tgas — at or beyond NEAR's 300 Tgas per-transaction ceiling.

When `execute_scheduled` runs out of NEAR gas, NEAR reverts all state changes in that receipt (the promise is restored to storage). However, the wNEAR transfer that occurred in the prior EVM transaction is already committed and **cannot be reverted**:

```rust
if required_near != ZERO_YOCTO {
    // transferFrom(user → engine_implicit, required_near) — committed in EVM state
    let (exit_reason, return_value) =
        handle.call(wnear_address.raw(), None, tx_data, None, false, &context);
    ...
}
``` [6](#0-5) 

There is no refund callback for a failed `execute_scheduled`, and no mechanism to recover wNEAR from the engine's implicit address. The funds are permanently frozen.

---

### Impact Explanation

**Permanent freezing of funds.** Any wNEAR transferred to the engine's implicit address as `attached_balance` for a `Delayed` promise whose `NearPromise::And` vector is too large to iterate within 300 Tgas is irrecoverable. The promise can never be executed (NEAR's hard gas cap is 300 Tgas per transaction), and no refund path exists. This matches the **Critical — Permanent freezing of funds** impact tier.

---

### Likelihood Explanation

**Medium.** The attack requires a user to submit a `Delayed` XCC call with a large `NearPromise::And` vector and non-zero `attached_balance`. This can happen:
- **Accidentally**: a developer building an on-chain aggregator that fans out to many NEAR contracts via `And` may not realize the execution will exceed 300 Tgas.
- **Intentionally (self-grief)**: a user could deliberately freeze their own funds; no admin compromise or external dependency is required.

The entry path is fully unprivileged — any EVM address can call the XCC precompile.

---

### Recommendation

1. **Enforce a maximum promise count** in the XCC precompile before accepting a `Delayed` call. Reject inputs where `call.promise_count()` exceeds a safe threshold (e.g., 20–25, consistent with the `ROUTER_EXEC_PER_CALLBACK` budget).
2. **Charge execution-proportional EVM gas for `Delayed` calls**, mirroring the `Eager` path:
   ```rust
   let router_exec_cost = costs::ROUTER_EXEC_BASE
       + NearGas::new(callback_count * costs::ROUTER_EXEC_PER_CALLBACK.as_u64());
   ```
   so that the EVM gas cost reflects the eventual NEAR execution cost.
3. **Add a refund callback** on the `schedule` cross-contract call so that if scheduling or later execution fails, the wNEAR is returned to the user.

---

### Proof of Concept

**Attacker steps:**

1. Deploy a contract on Aurora that calls the XCC precompile at `0x516cded1d16af10cad47d6d49128e2eb7d27b372` with:
   ```
   CrossContractCallArgs::Delayed(
       PromiseArgs::Recursive(
           NearPromise::And(vec![simple_promise_with_1_yocto; 1000])
       )
   )
   ```
   Borsh-serialized as the calldata. Each `simple_promise` has `attached_balance = 1 yoctoNEAR`.

2. The precompile charges `~772,221 EVM gas` (affordable) and transfers `1000 yoctoNEAR` worth of wNEAR from the caller to the engine's implicit address. The promise is stored in the router at nonce `N`.

3. Anyone calls `execute_scheduled(N)` on the router with 300 Tgas. `recursive_promise_create` iterates over 1,000 promises, exhausting NEAR gas. The call fails; NEAR state is reverted (promise back in storage).

4. Step 3 can be repeated indefinitely — it always fails. The 1,000 yoctoNEAR of wNEAR in the engine's implicit address is permanently inaccessible.

    #[allow(clippy::too_many_lines)]
    fn run_with_handle(
        &self,
        handle: &mut impl PrecompileHandle,
    ) -> Result<PrecompileOutput, PrecompileFailure> {
        let input = handle.input();
        let target_gas = handle.gas_limit().map(EthGas::new);
        let context = handle.context();
        utils::validate_no_value_attached_to_precompile(context.apparent_value)?;
        let is_static = handle.is_static();

        // This only includes the cost we can easily derive without parsing the input.
        // This allows failing fast without wasting computation on parsing.
        let input_len = u64::try_from(input.len()).map_err(utils::err_usize_conv)?;
        let mut cost =
            costs::CROSS_CONTRACT_CALL_BASE + costs::CROSS_CONTRACT_CALL_BYTE * input_len;
        let check_cost = |cost: EthGas| -> Result<(), PrecompileFailure> {
            if let Some(target_gas) = target_gas
                && cost > target_gas
            {
                return Err(PrecompileFailure::Error {
                    exit_status: ExitError::OutOfGas,
                });
            }
            Ok(())
        };
        check_cost(cost)?;

        // It's not allowed to call cross contract call precompile in static or delegate mode
        if is_static {
            return Err(revert_with_message(consts::ERR_STATIC));
        } else if context.address != cross_contract_call::ADDRESS.raw() {
            return Err(revert_with_message(consts::ERR_DELEGATE));
        }

        let sender = context.caller;
        let target_account_id = create_target_account_id(sender, self.engine_account_id.as_ref())?;
        let args = CrossContractCallArgs::try_from_slice(input)
            .map_err(|_| ExitError::Other(Cow::from(consts::ERR_INVALID_INPUT)))?;
        let (promise, attached_near) = match args {
            CrossContractCallArgs::Eager(call) => {
                let call_gas = call.total_gas();
                let attached_near = call.total_near();
                let callback_count = call
                    .promise_count()
                    .checked_sub(1)
                    .ok_or_else(|| ExitError::Other(Cow::from(consts::ERR_INVALID_INPUT)))?;
                let router_exec_cost = costs::ROUTER_EXEC_BASE
                    + NearGas::new(callback_count * costs::ROUTER_EXEC_PER_CALLBACK.as_u64());
                let promise = PromiseCreateArgs {
                    target_account_id,
                    method: consts::ROUTER_EXEC_NAME.into(),
                    args: borsh::to_vec(&call)
                        .map_err(|_| ExitError::Other(Cow::from(consts::ERR_SERIALIZE)))?,
                    attached_balance: ZERO_YOCTO,
                    attached_gas: router_exec_cost.saturating_add(call_gas),
                };
                (promise, attached_near)
            }
            CrossContractCallArgs::Delayed(call) => {
                let attached_near = call.total_near();
                let promise = PromiseCreateArgs {
                    target_account_id,
                    method: consts::ROUTER_SCHEDULE_NAME.into(),
                    args: borsh::to_vec(&call)
                        .map_err(|_| ExitError::Other(Cow::from(consts::ERR_SERIALIZE)))?,
                    attached_balance: ZERO_YOCTO,
                    // We don't need to add any gas to the amount need for the schedule call
                    // since the promise is not executed right away.
                    attached_gas: costs::ROUTER_SCHEDULE,
                };
                (promise, attached_near)
            }
        };
