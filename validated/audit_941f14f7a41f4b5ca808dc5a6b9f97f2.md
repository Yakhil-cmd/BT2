### Summary

When a `__CronosSendCroToIbc` event is emitted with an `amount` below `10^10` (e.g., `9999999999`), `Handle` unconditionally transfers evmDenom from the contract to the sender via `bankKeeper.SendCoins`, then calls `IbcTransferCoins`, which silently `continue`s (returns `nil`) without initiating any IBC transfer. The net result is that the sender receives evmDenom from the bridge contract with no corresponding IBC packet, permanently draining the contract's bank balance.

`amount` is taken directly from the unpacked event data with no minimum-amount guard. `bankKeeper.SendCoins(contractAddr, sender, coins)` executes unconditionally.

**Step 2 â€” `IbcTransferCoins` silently skips amounts below `10^10`:** [2](#0-1) 

For `evmDenom`, the function computes `amountToBurn = amount - (amount % 10^10)`. If `amount < 10^10`, `amountToBurn == 0` and the branch hits `continue` â€” returning `nil` to the caller with no burn, no IBC voucher release, and no IBC packet sent.

**Step 3 â€” `Handle` sees no error and returns `nil`:** [3](#0-2) 

Because `IbcTransferCoins` returned `nil`, `Handle` also returns `nil`. The EVM transaction succeeds. The sender's bank balance has increased by `amount` evmDenom; the contract's balance has decreased by the same amount; no IBC transfer was created.

### Impact Explanation

This is a **Critical** unauthorized balance change for evmDenom (CRO 18-decimal) assets. Any user who can cause the authorized CroBridge contract to emit `__CronosSendCroToIbc` with `0 < amount < 10^10` will receive evmDenom from the contract's bank account with no IBC transfer and no revert. Repeated calls drain the contract's entire evmDenom reserve in increments of up to `9999999999` aphoton per transaction.

### Recommendation

Return an error in `IbcTransferCoins` instead of silently continuing when `amountToBurn.IsZero()`:

```go

    return fmt.Errorf("amount %s is too small to bridge (minimum 10^10 evmDenom)", c.Amount)
}
```

This causes `Handle` to propagate the error, reverting the entire EVM transaction (including the preceding `SendCoins`), so no coins leave the contract without a matching IBC transfer.
