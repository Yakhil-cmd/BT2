### Title
Missing Minimum Threshold Lower-Bound Validation in FROST Signing Allows Sub-Threshold Signature Production - (File: `src/frost/mod.rs`)

---

### Summary

The `assert_sign_inputs` function in `src/frost/mod.rs` validates that `threshold <= participants.len()` but does **not** enforce a minimum threshold of 2. The DKG entry point (`assert_key_invariants` in `src/dkg.rs`) explicitly rejects `threshold < 2`, but the FROST signing entry point has no equivalent lower-bound guard. A caller can pass `threshold = 1` (or `threshold = 0`) to the FROST signing function; the validation passes, and the protocol proceeds with an insufficient threshold, allowing a single participant to produce a valid threshold signature without the required quorum.

---

### Finding Description

`assert_sign_inputs` is the shared validation gate for FROST (Ed25519 and RedJubjub) signing:

```rust
// src/frost/mod.rs  lines 120-159
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants { ... });
    }
    // ...
    // validate threshold
    if threshold.value() > participants.len() {          // ← only upper-bound check
        return Err(InitializationError::ThresholdTooLarge { ... });
    }
    // ...
    Ok(participants)
}
``` [1](#0-0) 

The DKG path enforces a minimum of 2:

```rust
// src/dkg.rs  lines 579-581
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [2](#0-1) 

No equivalent guard exists in `assert_sign_inputs`. `ReconstructionLowerBound` is a plain `usize` newtype with no built-in minimum:

```rust
// src/thresholds.rs  lines 9-13
pub struct ReconstructionLowerBound(usize);
impl ReconstructionLowerBound {
    pub fn value(self) -> usize { self.0 }
}
``` [3](#0-2) 

With `threshold = 1` the upper-bound check `1 > participants.len()` is false for any valid participant list (len ≥ 2), so `assert_sign_inputs` returns `Ok`. The FROST protocol then proceeds requiring only one participant's nonce commitment and signature share. Because each participant's signing share is a valid evaluation of the secret polynomial generated during DKG, a single participant can supply the sole required share and the coordinator assembles a cryptographically valid threshold signature — without the quorum the DKG threshold was designed to enforce.

---

### Impact Explanation

**Critical — Unauthorized creation of a valid threshold signature.**

A caller who invokes FROST signing with `threshold = 1` causes the protocol to accept a single participant's signature share as sufficient. The resulting signature is indistinguishable from one produced by a legitimate quorum, because the share is a genuine evaluation of the secret polynomial. This directly violates the threshold security guarantee: the private key should be unrecoverable and unusable by any coalition smaller than the agreed threshold.

---

### Likelihood Explanation

**Medium.** The threshold value is supplied by the library caller at signing time and is not cryptographically bound to the value used during DKG. Any participant or coordinator that controls the signing invocation (e.g., a malicious MPC node, a compromised orchestration layer, or a buggy integration) can pass `threshold = 1` without triggering any error. No special cryptographic capability is required — only the ability to call the public signing API with an undersized threshold argument.

---

### Recommendation

Add the same lower-bound guard that `assert_key_invariants` already enforces, directly inside `assert_sign_inputs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

Alternatively, encode the invariant in `ReconstructionLowerBound` itself (e.g., via a checked constructor) so that no call site can ever construct a value below 2.

---

### Proof of Concept

1. Complete a normal FROST DKG with 3 participants and `threshold = 2`.
2. Invoke FROST signing, passing the same 3 participants but `threshold = 1`.
3. `assert_sign_inputs` performs only the upper-bound check (`1 > 3` → false) and returns `Ok`.
4. The protocol runs with one required share; the coordinator collects only its own share and produces a valid, verifiable threshold signature.
5. The signature verifies correctly against the public key established during DKG, despite only one participant contributing — a quorum of one instead of the required two. [4](#0-3) [5](#0-4)

    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };
