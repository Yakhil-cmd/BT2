### Title
Unauthenticated `LedgerService.Set` gRPC Endpoint Allows Arbitrary Execution Ledger State Writes - (`ledger/remote/service.go`)

### Summary

The `cmd/ledger` standalone ledger service exposes a gRPC `LedgerService` with a `Set` RPC that writes arbitrary key-value pairs directly to the execution state trie. The gRPC server is created with no authentication interceptors, no TLS requirement, and no caller identity check. When the service is bound to a TCP address (e.g., `0.0.0.0:9000` as shown in the official README), any network-reachable client — including a malicious unstaked Access/Observer node — can call `Set` and overwrite arbitrary register values in the execution ledger, corrupting account balances, contract code, and all other on-chain state.

### Finding Description

**Root cause — `ledger/remote/service.go`, `Set` function (lines 132–198):**

The `Service.Set` method performs only structural validation (non-nil state hash, non-empty keys, matching key/value lengths). There is no caller authentication, no role check, and no capability proof:

```go
func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
    // only structural checks — no auth
    ...
    newState, trieUpdate, err := s.ledger.Set(update)
    ...
}

### Impact Explanation

The execution ledger is the authoritative store for all on-chain state: FLOW token balances, fungible/non-fungible token vaults, contract bytecode, and account keys. An unauthenticated caller who can reach the TCP port can:

1. **Overwrite account balances** — set any account's FLOW balance register to an arbitrary value, effectively stealing or minting tokens.
2. **Corrupt or replace contract code** — overwrite the register holding a deployed Cadence contract, injecting malicious logic.
3. **Destroy account keys** — zero out key registers, permanently locking accounts.
4. **Forge arbitrary state** — produce a new root hash accepted by the execution node as the canonical post-block state, breaking consensus integrity.

This is unauthorized mutation of on-chain assets, not merely a denial-of-service.

### Likelihood Explanation

The service is a production component used by execution nodes via `--ledger-service-addr`. The README explicitly documents binding to `0.0.0.0:9000`. Any network peer — including a malicious unstaked Access or Observer node — that can reach the TCP port can exploit this with a single unauthenticated gRPC call. No credentials, no staked identity, and no prior access are required. The attack requires only network reachability to the port.

### Recommendation

1. **Add mutual TLS (mTLS)** to the gRPC server in `cmd/ledger/main.go`, requiring clients to present a certificate signed by a trusted CA (analogous to the optional mTLS already implemented in the main admin server in `cmd/scaffold.go`).
2. **Add a gRPC authentication interceptor** that validates a shared secret or client certificate before dispatching any RPC, especially `Set`.
3. **Restrict the `Set` RPC** to Unix domain socket transport only (already supported via `--ledger-service-socket`), preventing TCP exposure entirely when the execution node and ledger service are co-located.
4. **Document the security requirement** that the TCP port must never be exposed to untrusted networks.

// Set updates keys with new values at a specific state and returns the new state
func (s *Service) Set(ctx context.Context, req *ledgerpb.SetRequest) (*ledgerpb.SetResponse, error) {
	if req.State == nil || len(req.State.Hash) != len(ledger.State{}) {
		return nil, status.Error(codes.InvalidArgument, "invalid state")
	}

	if len(req.Keys) == 0 {
		return nil, status.Error(codes.InvalidArgument, "keys cannot be empty")
	}

	if len(req.Keys) != len(req.Values) {
		return nil, status.Error(codes.InvalidArgument, "keys and values length mismatch")
	}

	var state ledger.State
	copy(state[:], req.State.Hash)

	keys := make([]ledger.Key, len(req.Keys))
	for i, protoKey := range req.Keys {
		key, err := protoKeyToLedgerKey(protoKey)
		if err != nil {
			return nil, err // protoKeyToLedgerKey already returns status.Error
		}
		keys[i] = key
	}

	values := make([]ledger.Value, len(req.Values))
	for i, protoValue := range req.Values {
		var value ledger.Value
		// Reconstruct the original value type using is_nil flag
		// This preserves the distinction between nil and []byte{} that protobuf loses
		if len(protoValue.Data) == 0 {
			if protoValue.IsNil {
				// Original value was nil
				value = nil
			} else {
				// Original value was []byte{} (empty slice)
				value = ledger.Value([]byte{})
			}
		} else {
			// Non-empty value, use data as-is
			value = ledger.Value(protoValue.Data)
		}
		values[i] = value
	}

	update, err := ledger.NewUpdate(state, keys, values)
	if err != nil {
		return nil, status.Error(codes.InvalidArgument, err.Error())
	}

	newState, trieUpdate, err := s.ledger.Set(update)
	if err != nil {
		return nil, status.Error(codes.Internal, err.Error())
	}

	// Encode trie update using centralized encoding function to ensure
	// client and server use the same encoding method
	trieUpdateBytes := encodeTrieUpdateForTransport(trieUpdate)

	return &ledgerpb.SetResponse{
		NewState: &ledgerpb.State{
			Hash: newState[:],
		},
		TrieUpdate: trieUpdateBytes,
	}, nil
}
