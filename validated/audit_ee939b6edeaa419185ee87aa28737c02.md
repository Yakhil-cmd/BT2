Let me look more carefully at the stateful validator's full flow and the `validate_resource_bounds` function, and also check the `extract_state_nonce_and_run_validations` function.

### Title
Gateway Stateful Validator Uses Stale Previous-Block L2 Gas Price for Admission Threshold — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **previous** block's L2 gas price instead of the **next** block's L2 gas price. The same stale price is also used inside `run_validate_entry_point` when building the `BlockContext` for blockifier pre-validation. Because the batcher executes transactions against the **actual** next-block gas price (computed by the EIP-1559 fee market), the gateway's admission decision and the blockifier's execution decision are made against different prices. This is the direct sequencer analog of the BondToken `globalPool.sharesPerToken` stale-value bug: a function that should record/validate against the *new* value silently uses the *old* value instead.

**Stale price in `run_validate_entry_point`** (lines 302–330): the function increments only `block_number` but leaves all gas prices at the previous block's values, so `check_fee_bounds` inside `perform_pre_validation_stage` also validates against the previous block's price:

``` [2](#0-1) 
Both checks therefore admit a transaction when `tx.max_price_per_unit ≥ previous_block_l2_gas_price`.

**The actual execution path** in the batcher uses the next block's L2 gas price, computed by `calculate_next_l2_gas_price` / `calculate_next_base_gas_price` via EIP-1559 adjustment. [3](#0-2)  The consensus context updates `self.l2_gas_price` before each block and passes it into `ProposalInit`, which becomes the `BlockInfo` used by the batcher. [4](#0-3)
```

### Impact Explanation

**When the next-block gas price rises** (congested block, EIP-1559 pushes price up):

- A transaction with `max_price_per_unit = P_prev` passes both gateway checks (threshold ≤ P_prev, blockifier check P_prev ≥ P_prev).
- At execution time the batcher uses `P_next > P_prev`; `check_fee_bounds` fires `MaxGasPriceTooLow` and the transaction is reverted or dropped.
- **Gateway admits transactions that are invalid for the block they will be sequenced in.**

**When the next-block gas price falls** (under-utilized block):

- A transaction with `max_price_per_unit = P_next` (where `P_next < threshold`) is rejected by the gateway even though it would succeed at execution.
- **Gateway rejects transactions that are valid for the block they would be sequenced in.**

Both directions match the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The EIP-1559 adjustment is bounded per block by `gas_price_max_change_denominator`, so the window of misclassified transactions is proportional to the per-block price swing. Under sustained congestion (block consistently above target) the price rises every block, making the accept-then-fail path continuously reachable by any unprivileged user who submits a transaction priced at exactly the previous block's price. The TODO comment in the source code confirms the developers are aware the wrong value is being used. [7](#0-6) 

### Recommendation

Replace `previous_block_l2_gas_price` with the projected next-block L2 gas price. The fee-market module already exposes `calculate_next_l2_gas_price_for_fin` / `calculate_next_base_gas_price`; the gateway can call this with the latest block's `l2_gas_used` and `l2_gas_price` to obtain the correct reference price before running either check. The same projected price should be injected into the `BlockContext` built inside `run_validate_entry_point` so that the blockifier's `check_fee_bounds` is also consistent with actual execution.

### Proof of Concept

1. Observe the current (previous) block's L2 gas price `P_prev` and its gas usage `G_used > gas_target` (congested block).
2. Compute `P_next = calculate_next_base_gas_price(P_prev, G_used, gas_target, min_price)` — this will be `> P_prev`.
3. Submit an `InvokeV3` transaction with `l2_gas.max_price_per_unit = P_prev` (exactly the previous block's price).
4. The gateway's `validate_resource_bounds` computes `threshold = (min_gas_price_percentage/100) × P_prev ≤ P_prev` → passes. [8](#0-7) 
5. The gateway's blockifier validation uses `P_prev` as the block gas price → `check_fee_bounds` passes (`P_prev ≥ P_prev`). [2](#0-1) 
6. Transaction enters the mempool and is pulled by the batcher.
7. The batcher builds the block with `P_next > P_prev`; `check_fee_bounds` fires `MaxGasPriceTooLow { max_gas_price: P_prev, actual_gas_price: P_next }` → transaction is reverted/dropped after consuming sequencer resources.
8. Repeat with many such transactions to amplify the DoS effect on the sequencer's execution pipeline.

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
```

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
```
