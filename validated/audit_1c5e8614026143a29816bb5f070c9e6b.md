### Title
Missing Proof of Correctness for CKD Participant Contributions Allows Derived Key Corruption — (File: src/confidential_key_derivation/protocol.rs)

---

### Summary
The CKD coordinator blindly accumulates participant-supplied `(big_y, big_c)` group elements without any proof of correctness. A single malicious participant can substitute arbitrary curve points, causing the coordinator to output a corrupted ElGamal ciphertext that decrypts to an attacker-influenced derived key instead of the legitimate `msk · H(pk, app_id)`.

---

### Finding Description

In `do_ckd_coordinator`, the coordinator receives a `CKDOutput` from every other participant and adds the two group elements together with no validation whatsoever:

```rust
for (_, participant_output) in
    recv_from_others::<CKDOutput>(&chan, waitpoint, &participants, me).await?
{
    norm_big_y += participant_output.big_y();
    norm_big_c += participant_output.big_c();
}
let ckd_output = CKDOutput::new(norm_big_y, norm_big_c);
Ok(Some(ckd_output))
``` [1](#0-0) 

Each honest participant is supposed to compute:
- `Y_i = λ_i · y_i · G₁`
- `C_i = λ_i · (x_i · H(pk, app_id) + y_i · app_pk)` [2](#0-1) 

The coordinator sums these to produce `(Y, C)`, an ElGamal encryption of `msk · H(pk, app_id)` under `app_pk`. However, there is **no proof of correctness** — no Sigma proof, no identity-element check, no consistency check — that the received `(big_y, big_c)` values are consistent with the participant's actual key share. The only gate is successful deserialization.

This is the direct analog of the external report's pattern: the output is computed from an externally supplied value that can be freely manipulated, and the result is accepted without checking it against any expected bound or commitment.

By contrast, the OT-based ECDSA presign protocol validates every received scalar against its public commitment before proceeding (`E =?= e·G`, `α·G =?= K + A`, `β·G =?= X + B`): [3](#0-2) [4](#0-3) 

The CKD protocol has no equivalent guard.

---

### Impact Explanation

A malicious participant sends `(big_y_m, big_c_m + Δ)` for any attacker-chosen non-identity point `Δ ∈ G₁`. The coordinator computes `C_final = C_correct + Δ`. The TEE application that calls `ckd_output.unmask(app_sk)` decrypts:

```
C_final − y_total · app_pk  =  msk · H(pk, app_id) + Δ
```

The TEE receives a derived key that differs from the intended `msk · H(pk, app_id)` by the fully attacker-controlled offset `Δ`. This matches the allowed impact: **Corruption of CKD outputs so honest parties accept incorrect derived secrets**. [5](#0-4) 

---

### Likelihood Explanation

Any single participant in the CKD session can trigger this. The attacker only needs to be one of the N participants (e.g., a compromised TEE node or a malicious library caller). No special privilege beyond participation is required. The public `ckd()` entry point is callable by any participant, and the malicious contribution is sent privately to the coordinator, making it undetectable by other honest participants. [6](#0-5) 

---

### Recommendation

Add a Sigma proof (proof of correct ElGamal encryption) alongside each participant's `(big_y, big_c)` contribution. Specifically, each participant should prove in zero-knowledge that:

1. They know `y_i` such that `big_y = λ_i · y_i · G₁`.
2. `big_c = λ_i · (x_i · H(pk, app_id) + y_i · app_pk)` for their committed public key share `x_i · G₂`.

The coordinator must verify all proofs before summing contributions, analogous to how the OT-based ECDSA presign verifies `E =?= e·G` before proceeding.

---

### Proof of Concept

1. N−1 honest participants and 1 malicious participant `P_m` run the CKD protocol targeting the coordinator.
2. `P_m` computes the correct `(norm_big_y_m, norm_big_c_m)` per `compute_signature_share` but sends `(norm_big_y_m, norm_big_c_m + Δ)` to the coordinator, where `Δ` is any non-identity point on G₁.
3. The coordinator sums all contributions without validation: `C_final = C_correct + Δ`.
4. The TEE application calls `unmask(app_sk)`: `C_final − y_total · app_pk = msk · H(pk, app_id) + Δ`.
5. The TEE receives a derived key offset by `Δ`, which is fully attacker-controlled, instead of the legitimate `msk · H(pk, app_id)`. [7](#0-6)

pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    let comms = Comms::new();
    let chan = comms.shared_channel();

    let fut = run_ckd_protocol(
        chan,
        coordinator,
        me,
        participants,
        key_pair,
        app_id.into(),
        app_pk,
        rng,
    );
    Ok(make_protocol(comms, fut))
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
