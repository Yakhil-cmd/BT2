### Title
Unrestricted `update_rewards` with `disable_rewards: true` Allows Any Caller to Permanently Freeze All Stakers' Consensus Block Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function is publicly callable by any address. It accepts a `disable_rewards` parameter that, when set to `true`, advances the global `last_reward_block` to the current block without distributing any rewards. Because `last_reward_block` is a single contract-wide variable (not per-staker), a single call per block is sufficient to permanently block every staker from receiving consensus-phase block rewards.

---

### Finding Description

`update_rewards` is exposed through the `IStakingRewardsManager` interface with no caller restriction beyond `general_prerequisites` (contract not paused, caller non-zero):

```
fn update_rewards(ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool) {
    self.general_prerequisites();                                    // no caller check
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),       // global gate
        "{}", Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);             // global write

    if disable_rewards || self.is_pre_consensus() {
        return;                                                      // no rewards distributed
    }
    ...
}
``` [1](#0-0) 

The storage field `last_reward_block` is a single `BlockNumber`, not a `Map<ContractAddress, BlockNumber>`:

```
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [2](#0-1) 

The attack flow is:

1. Attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N.
2. `last_reward_block` is written to N; no rewards are distributed.
3. Any legitimate call to `update_rewards(victim, false)` in block N reverts with `REWARDS_ALREADY_UPDATED`.
4. Attacker repeats every block.

The only precondition is that the attacker supplies a valid, active staker address with non-zero balance — all of which are publicly readable from the `stakers` vector and on-chain traces.

This is the direct analog to the reported oracle-manipulation pattern: just as an attacker manipulates `getAmountsOut()` (a view of on-chain state) to make slippage protection return near-zero, here an attacker manipulates `last_reward_block` (a piece of on-chain state) to make the reward-distribution gate permanently closed for every block.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Once the consensus-rewards epoch begins (`is_pre_consensus()` returns `false`), every block reward for every staker is gated by `last_reward_block`. An attacker who calls `update_rewards(..., true)` once per block prevents the entire staker set from ever accumulating consensus block rewards. The yield is not stolen — it is simply never credited, constituting a permanent freeze of unclaimed yield for all participants.

---

### Likelihood Explanation

**Medium.**

The attacker needs to submit one cheap transaction per block (every ~3 seconds on Starknet). There is no profit motive, but the cost is low and the damage is protocol-wide. The attack requires no privileged role, no key compromise, and no external dependency — only a funded address and a script that fires one transaction per block.

---

### Recommendation

1. **Make `last_reward_block` per-staker**: change `last_reward_block: BlockNumber` to `last_reward_block: Map<ContractAddress, BlockNumber>` so that one call cannot block all others.
2. **Restrict the caller**: require `get_caller_address() == staker_address` or that the caller is the staker's operational address before allowing `disable_rewards: true`.
3. **Remove the `disable_rewards` parameter from the public interface**: handle the pre-consensus no-op internally (the `is_pre_consensus()` branch already covers it) so external callers cannot suppress reward distribution.

---

### Proof of Concept

```
// Attacker script (pseudocode, runs every block)
loop {
    staking_contract.update_rewards(
        staker_address: any_active_staker,   // publicly known
        disable_rewards: true,
    );
    // last_reward_block is now current_block for the entire contract.
    // Every legitimate update_rewards call this block reverts.
    wait_for_next_block();
}

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
