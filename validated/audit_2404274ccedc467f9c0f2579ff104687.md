### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards by Front-Running `update_rewards` with `disable_rewards: true` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable with no access control. It accepts a caller-controlled `disable_rewards: bool` parameter and writes to the global `last_reward_block` storage slot regardless of whether rewards were actually distributed. An unprivileged actor can call `update_rewards(valid_staker, disable_rewards: true)` every block, consuming the one-call-per-block budget and permanently preventing any staker from receiving consensus rewards.

---

### Finding Description

`IStakingRewardsManager::update_rewards` is exposed as a public ABI entry point with no role check. Its only gate is:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

After passing this check, the function unconditionally writes the current block number to `last_reward_block` **before** branching on `disable_rewards`:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← always written

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits with no rewards
}
```

Because `last_reward_block` is a **global** (not per-staker) slot, a single call with `disable_rewards: true` exhausts the entire block's reward-distribution budget for every staker. Any subsequent call in the same block — even with `disable_rewards: false` — reverts with `REWARDS_ALREADY_UPDATED`.

The only precondition the attacker must satisfy is supplying a valid, active staker address with non-zero balance, which is trivially obtained from on-chain events (`NewStaker`, `StakeOwnBalanceChanged`).

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

If an attacker calls `update_rewards(any_active_staker, true)` in every block, no consensus rewards are ever distributed to any staker or delegation pool. Accumulated unclaimed yield is frozen indefinitely. The attack is protocol-wide: a single attacker blocks rewards for all stakers simultaneously because `last_reward_block` is shared state.

---

### Likelihood Explanation

**High.** The attack requires:
- Knowledge of one active staker address (public on-chain).
- Submitting one cheap transaction per block.
- No privileged role, no capital at risk, no dependency on external systems.

On Starknet, transaction ordering is controlled by the sequencer, but `update_rewards` is not restricted to the sequencer. Any externally-owned account can submit this call. The attacker's cost is bounded by Starknet gas fees per block, which is negligible compared to the yield frozen across all stakers.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the sequencer's operational address, the attestation contract, or a dedicated role). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "no rewards" case internally based on on-chain state (e.g., whether the caller is the expected block proposer). At minimum, the `last_reward_block` write should be gated behind the same condition that gates reward distribution, so a call with `disable_rewards: true` does not consume the block's reward slot.

---

### Proof of Concept

1. Staker Alice stakes STRK and is active in epoch `E` (consensus rewards active).
2. Attacker Bob (no stake, no role) reads Alice's address from the `NewStaker` event.
3. Each block, Bob submits:
   ```
   staking.update_rewards(alice_address, disable_rewards: true)
   ```
4. `last_reward_block` is set to the current block number; the function returns early.
5. The legitimate block-rewards call (e.g., from the sequencer or a keeper) in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. Alice's `unclaimed_rewards_own` is never incremented. Pool members' `cumulative_rewards_trace` is never updated.
7. Repeated every block → all consensus rewards are permanently frozen.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
