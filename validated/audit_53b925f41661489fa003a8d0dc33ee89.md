### Title
Gateway L2 Gas Price Threshold Computed with Floor Division Admits Transactions Below the Intended Minimum - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price using `.to_integer()` (floor/truncation) on a rational number. This rounds the threshold **down**, allowing transactions whose `max_price_per_unit` is strictly below the intended fractional threshold to pass gateway admission.

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`, the admission threshold is computed as:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();   // ← floor, not ceil
if tx_l2_gas_price.0 < threshold {
    return Err(...)
}
```

`Ratio::to_integer()` truncates toward zero (floor). When `min_gas_price_percentage * previous_block_l2_gas_price` is not an exact multiple of 100, the threshold is silently lowered by up to 99 units.

**Concrete example:**
- `min_gas_price_percentage = 90`, `previous_block_l2_gas_price = 11 FRI`
- Exact threshold: `90/100 × 11 = 9.9 FRI`
- Computed threshold: `floor(9.9) = 9 FRI`
- A transaction with `max_price_per_unit = 9 FRI` satisfies `9 >= 9` and is **admitted**, even though `9 < 9.9` (the intended minimum).

The fix is to use `.ceil().to_integer()` so the threshold is rounded **up**, preserving the invariant that admitted transactions always meet or exceed the configured percentage of the previous block price. [1](#0-0) 

### Impact Explanation

The gateway stateful path is supposed to reject transactions whose L2 gas price is below `min_gas_price_percentage`% of the previous block's L2 gas price. Due to floor rounding, transactions with a gas price up to `(100 - 1) / 100` FRI units below the exact threshold are incorrectly admitted. This breaks the admission invariant and matches the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions."*

### Likelihood Explanation

The condition is triggered whenever `min_gas_price_percentage × previous_block_l2_gas_price` is not divisible by 100, which is the common case for arbitrary gas prices. Any user who observes the previous block's L2 gas price can trivially craft a transaction at exactly `floor(threshold)` to exploit this. No special privileges are required.

### Recommendation

Replace `.to_integer()` with `.ceil().to_integer()` (or the equivalent `num_rational::Ratio::ceil`) so the threshold is rounded up:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This ensures that any transaction admitted by the gateway genuinely meets the configured minimum gas price percentage.

### Proof of Concept

```
min_gas_price_percentage = 90
previous_block_l2_gas_price = 11 FRI

Intended threshold = ceil(90/100 * 11) = ceil(9.9) = 10 FRI
Actual threshold   = floor(90/100 * 11) = floor(9.9) = 9 FRI

Transaction with max_price_per_unit = 9 FRI:
  - Should be REJECTED  (9 < 10)
  - Is ADMITTED         (9 >= 9)  ← invariant broken
``` [2](#0-1)

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
