### Title
Unprivileged Caller Can Permanently Suppress Block Reward Distribution via `disable_rewards` Parameter - (File: `src/staking/staking.cairo`)

### Summary
`update_rewards` in the Staking contract is a public function with no caller restriction. It accepts a caller-controlled `disable_rewards: bool` parameter. When set to `true`, the function advances the global `last_reward_block` to the current block number and returns early — permanently preventing reward distribution for that block for all stakers. Any unprivileged address can exploit this to continuously suppress staker yield.

### Finding Description
`update_rewards` is defined in `IStakingRewardsManager` and implemented without any access control beyond `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function first asserts that the current block has not yet been processed: [2](#0-1) 

It then unconditionally writes `current_block_number` to the global `last_reward_block` storage variable: [3](#0-2) 

If `disable_rewards` is `true`, the function returns immediately after writing `last_reward_block`, skipping the entire reward calculation and distribution path (`_update_rewards`). Because `last_reward_block` is now equal to the current block, any subsequent legitimate call to `update_rewards` in the same block will revert with `REWARDS_ALREADY_UPDATED`. The rewards for that block are permanently lost — they are never claimed from the `RewardSupplier` and never distributed to stakers or pools.

The `last_reward_block` storage field is global (not per-staker): [4](#0-3) 

The `general_prerequisites` guard provides no caller identity check: [5](#0-4) 

### Impact Explanation
An attacker calling `update_rewards(any_valid_staker, true)` once per block permanently forfeits all stakers' block rewards for that block. The `unclaimed_rewards` in `RewardSupplier` is never incremented for the suppressed block, and the STRK that would have been distributed is never claimed. This constitutes **theft of unclaimed yield** (High) — stakers are permanently deprived of yield they would otherwise have earned. Repeated calls across consecutive blocks can drain the entire consensus reward stream. [6](#0-5) 

### Likelihood Explanation
The entry path requires no privilege: any non-zero Starknet address can call `update_rewards` with `disable_rewards: true`. The attacker only needs to front-run or race the legitimate per-block call. On Starknet, transaction ordering within a block is sequencer-controlled, but the attack is viable across blocks since the attacker can submit the suppression call at the start of each block. The cost is only gas per block.

### Recommendation
Restrict who may call `update_rewards` with `disable_rewards: true`, or remove the `disable_rewards` parameter from the public interface entirely. If the parameter is needed for protocol-internal use (e.g., during the pre-consensus migration phase), gate it behind a privileged role check (e.g., `only_app_governor`) or derive the value internally from `self.is_pre_consensus()` rather than accepting it from the caller.

```cairo
// Suggested fix: derive disable_rewards internally
fn update_rewards(ref self: ContractState, staker_address: ContractAddress) {
    ...
    let disable_rewards = self.is_pre_consensus();
    ...
}
```

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has been set and passed).
2. At block `N`, attacker calls `update_rewards(any_active_staker, disable_rewards: true)`.
3. The function passes `general_prerequisites()` (contract unpaused, caller non-zero).
4. `current_block_number (N) > last_reward_block` — assertion passes.
5. `last_reward_block` is written to `N`.
6. `disable_rewards == true` → early return; `_update_rewards` is never called.
7. Legitimate staker or relayer calls `update_rewards(staker, false)` at block `N` → reverts with `REWARDS_ALREADY_UPDATED`.
8. Block `N` rewards are permanently lost for all stakers.
9. Attacker repeats at block `N+1`, `N+2`, … to continuously suppress all yield. [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
