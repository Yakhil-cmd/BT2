### Title
WalletConnect Offer Confirmation Displays Understated "Total Amount with Royalties" Due to Incorrect Royalty Accounting in `formatAmountWithRoyalties` - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary

The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts` divides the purchase amount by the number of NFTs before computing each royalty, producing a displayed total that is `1/N` of the correct value when `N > 1` NFTs are involved. The WalletConnect confirmation dialog renders this understated figure as "Total Amount with Royalties" under "You Spend", causing a user to approve a `chia_wallet.take_offer` or `chia_wallet.create_offer_for_ids` command believing they will spend less XCH/CAT than the backend will actually deduct.

### Finding Description

`formatAmountWithRoyalties` is called from `withRoyaltyTotals` to compute the displayed spending total for the WalletConnect confirmation dialog: [1](#0-0) 

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // ← divides by N
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
const totalAmount = amount + royaltyAmount;
```

The correct formula for total royalties when buying N NFTs is:

```
royaltyTotal = Σ (amount × royaltyPercentage_i / 10_000)
             = amount × Σ(royaltyPercentage_i) / 10_000
```

The implemented formula computes:

```
royaltyTotal = Σ ((amount / N) × royaltyPercentage_i / 10_000)
             = (amount / N) × Σ(royaltyPercentage_i) / 10_000
```

This is exactly `1/N` of the correct value. The result is attached to the `amountWithRoyalties` field of each spending line: [2](#0-1) 

and rendered in the confirmation dialog as "Total Amount with Royalties": [3](#0-2) 

The existing test for the two-NFT case encodes the wrong value as the expected output, confirming the bug is present and untested-against-correct-behavior: [4](#0-3) 

The `parseCommandDisplay` function is invoked for both `chia_wallet.take_offer` and `chia_wallet.create_offer_for_ids` WalletConnect commands: [5](#0-4) 

### Impact Explanation

A malicious WalletConnect dApp crafts an offer containing multiple high-royalty NFTs. The confirmation dialog shows the user a "Total Amount with Royalties" that is `1/N` of the true cost. The user approves, believing they are spending less than they actually are. The backend (Chia wallet daemon) computes royalties correctly and deducts the full amount. With two NFTs the displayed royalty is halved; with ten NFTs it is one-tenth of the real cost. For high-royalty NFTs (e.g., 5% each) and large purchase amounts, the gap between displayed and actual spend is material.

**Impact category**: High — corruption of WalletConnect state that causes a user to approve the wrong amount, resulting in direct asset loss.

### Likelihood Explanation

Any unprivileged WalletConnect dApp can send `chia_wallet.take_offer` or `chia_wallet.create_offer_for_ids` with a multi-NFT payload. No privileged access, leaked keys, or cryptographic break is required. The attacker only needs to construct a valid offer blob containing two or more NFTs with non-zero royalties, which is a normal Chia protocol operation.

### Recommendation

Remove the `splitAmount` division. Each NFT's royalty must be computed on the full `amount`:

```typescript
// Before (wrong):
const splitAmount = amount / BigInt(royaltyPercentages.length);
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);

// After (correct):
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

Update the corresponding test assertion in `parseCommandDisplay.test.ts` for the two-NFT case:

- Displayed: `0.00010255` XCH (wrong — `1/2` of correct royalties)
- Correct: `0.0001051` XCH (`100000000 + 5000000 + 100000 = 105100000` mojos)

### Proof of Concept

**Setup**: Malicious dApp sends via WalletConnect:
```json
{
  "command": "chia_wallet.take_offer",
  "params": { "offer": "<offer with 2 NFTs: NFT-A at 5% royalty, NFT-B at 0.1% royalty, price 1 XCH>" }
}
```

**Displayed in confirmation dialog**:
- `amount = 1_000_000_000_000n` mojos (1 XCH)
- `splitAmount = 1_000_000_000_000n / 2n = 500_000_000_000n`
- `royaltyAmount = (500_000_000_000 × 500)/10000 + (500_000_000_000 × 10)/10000`
  `= 25_000_000_000 + 500_000_000 = 25_500_000_000`
- **Shown**: `1.0255 XCH`

**Actual backend deduction**:
- NFT-A royalty: `1_000_000_000_000 × 500 / 10_000 = 50_000_000_000`
- NFT-B royalty: `1_000_000_000_000 × 10 / 10_000 = 1_000_000_000`
- **Actual spend**: `1.051 XCH`

The user sees `1.0255 XCH` and approves; the wallet deducts `1.051 XCH` — a `0.0255 XCH` discrepancy per transaction, scaling linearly with the number of NFTs and royalty percentages.

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }
