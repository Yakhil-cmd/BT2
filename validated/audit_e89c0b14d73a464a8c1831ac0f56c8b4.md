### Title
Any Caller Can Permanently Freeze Block Rewards via `update_rewards` with `disable_rewards: true` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in `staking.cairo` writes to the global `last_reward_block` storage variable **before** checking the `disable_rewards` flag. Because there is no access control on the `disable_rewards` parameter, any unprivileged caller can invoke `update_rewards(active_staker, disable_rewards: true)` each block, consuming the block's reward slot without distributing any rewards. This permanently destroys the unclaimed yield for every block in which the attack is executed.

---

### Finding Description

`update_rewards` is a public function in `StakingRewardsManagerImpl`. Its guard against double-claiming is the global `last_reward_block` field: the function asserts `current_block_number > last_reward_block`, then immediately writes `last_reward_block = current_block_number`. Only **after** that write does it check `disable_rewards`: [1](#0-0) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← global slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← rewards never distributed
    }
    ...
}
```

`general_prerequisites()` only checks that the contract is not paused and the caller is non-zero — there is no role check and no restriction on the value of `disable_rewards`: [2](#0-1) 

Because `last_reward_block` is a **single global slot** shared by all stakers, the first call in any block wins. An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in block N:

1. Passes the `current_block_number > last_reward_block` guard.
2. Writes `last_reward_block = N`, consuming the slot.
3. Returns early — no rewards are calculated or transferred.
4. Every subsequent legitimate call in block N reverts with `REWARDS_ALREADY_UPDATED`.

The block rewards that were skipped are permanently unrecoverable: `last_reward_block` is already set to N, so no future call can retroactively claim them. [3](#0-2) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

Each block in which the attack is executed results in the permanent loss of that block's STRK (and BTC) rewards for every staker and every delegation pool. The attacker can repeat the call every block, causing total and continuous suppression of all consensus-phase reward distribution. The rewards remain in the `RewardSupplier` contract but can never be claimed for the skipped blocks because the global `last_reward_block` slot has already been consumed.

---

### Likelihood Explanation

**High.** The entry point is fully public — no token ownership, no privileged role, no staking requirement. The only precondition is knowing the address of any currently active staker with non-zero balance, which is trivially readable from on-chain events (`NewStaker`, `StakeOwnBalanceChanged`). The gas cost on Starknet is low, making sustained per-block griefing economically viable.

---

### Recommendation

Add access control to `update_rewards` so that only the designated attestation contract (or the staker themselves) may call it, and/or restrict the `disable_rewards: true` path to a privileged role. At minimum, the `last_reward_block` write should be moved **after** the `disable_rewards` guard so that a no-op call does not consume the block's reward slot:

```cairo
// Move the write AFTER the early-return guard
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number); // only consume slot when rewards are actually distributed
```

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
