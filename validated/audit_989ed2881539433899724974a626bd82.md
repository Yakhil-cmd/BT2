### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards for All Stakers via `update_rewards` with `disable_rewards: true` - (File: `src/staking/staking.cairo`)

---

### Summary

`update_rewards` is a publicly callable function that accepts a `disable_rewards: bool` parameter. When called with `disable_rewards: true`, it consumes the protocol's single per-block reward slot by writing to `last_reward_block`, but distributes zero rewards. Because only one `update_rewards` call is permitted per block, any unprivileged caller can grief all stakers by front-running every block with `disable_rewards: true`, permanently suppressing consensus reward accrual.

---

### Finding Description

`StakingRewardsManagerImpl::update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. No role, ownership, or staker-identity check is performed. [1](#0-0) 

The function enforces a single-call-per-block invariant via a global `last_reward_block` storage slot: [2](#0-1) 

After validating the staker, it unconditionally writes the current block number to `last_reward_block` **before** checking `disable_rewards`: [3](#0-2) 

When `disable_rewards` is `true`, the function returns immediately after updating `last_reward_block`, distributing nothing. Any subsequent legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

The `last_reward_block` is a single global slot shared across all stakers: [5](#0-4) 

---

### Impact Explanation

In consensus rewards mode (`is_pre_consensus()` returns `false`), every block is supposed to deliver STRK and BTC block rewards to one staker via `_update_rewards`. By calling `update_rewards(any_valid_staker, disable_rewards: true)` on every block, an attacker silently burns each block's reward slot. No staker ever accrues `unclaimed_rewards_own`, and no pool ever receives `update_rewards_from_staking_contract`. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators.

This matches the allowed impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"*.

---

### Likelihood Explanation

- The function is fully public; no token deposit, stake, or privileged role is required.
- On Starknet, transaction fees are extremely low, making sustained per-block griefing economically viable.
- The attacker needs only to know any one valid, active staker address (trivially obtained from on-chain `NewStaker` events).
- There is no slashing or penalty for the attacker.

Likelihood: **Medium** — requires sustained effort but zero capital and negligible cost.

---

### Recommendation

1. **Remove the `disable_rewards` parameter from the public interface.** If a "no-op" update is needed for migration purposes, gate it behind a privileged role (e.g., `only_app_governor`) or handle it in a separate internal function.
2. **Move the `last_reward_block.write` after the `disable_rewards` guard**, so that a call with `disable_rewards: true` does not consume the block's reward slot.
3. **Restrict callers**: Only the staker whose address is passed (or a designated consensus contract) should be permitted to call `update_rewards` for a given staker.

---

### Proof of Concept

```
// Attacker script (pseudocode, executed every block):
// 1. Read any active staker address S from on-chain events.
// 2. Call: staking_contract.update_rewards(staker_address: S, disable_rewards: true)
//    - general_prerequisites() passes (contract not paused, caller != 0)
//    - current_block_number > last_reward_block  → passes (new block)
//    - staker S is active with non-zero balance   → passes
//    - last_reward_block.write(current_block_number)  ← slot consumed
//    - disable_rewards == true → return (no rewards distributed)
// 3. Any legitimate call to update_rewards in the same block now fails:
//    assert!(current_block_number > last_reward_block) → PANIC: REWARDS_ALREADY_UPDATED
// 4. Repeat every block → all stakers accrue zero consensus rewards indefinitely.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
