### Title
Revoking Key 0 Does Not Revoke Duplicate Keys in `keyMetadataBytes`, Allowing Post-Revocation Transaction Signing — (`fvm/environment/accounts.go`, `fvm/environment/accounts_status.go`, `fvm/environment/account_public_key_util.go`)

### Summary

The Flow key storage system uses two separate storage locations for revocation state: the `apk_0` register (for key index 0) and `keyMetadataBytes` inside the `AccountStatus` register (for keys at index > 0). When key 0 is revoked, only the `apk_0` register is updated. Any duplicate key at index N > 0 that maps to the same physical key as key 0 retains `revoked=false` in `keyMetadataBytes`. The transaction verifier and sequence number checker both read the revoked status for key N from `keyMetadataBytes`, so they see `Revoked=false` and accept signatures from key N even after key 0 has been revoked.

### Finding Description

**Storage architecture:**

Key 0 is stored in a dedicated register `apk_0` (`AccountPublicKey0RegisterID`), which encodes the full key including its `Revoked` flag. Keys at index > 0 store their `revoked` status and `weight` in `keyMetadataBytes` inside the `AccountStatus` register, with an optional mapping to a `storedKeyIndex` for deduplication. [1](#0-0) 

**Revocation path for key 0:**

`RevokeAccountPublicKey(addr, 0)` dispatches to `revokeAccountPublicKey0`, which reads the `apk_0` register, sets `Revoked=true`, and writes it back. It does not touch `keyMetadataBytes`. [2](#0-1) [3](#0-2) 

**Revocation path for key N > 0:**

`RevokeAccountPublicKey(addr, N)` dispatches to `status.RevokeAccountPublicKey(keyIndex)`, which calls `accountkeymetadata.SetRevokedStatus(a.keyMetadataBytes, keyIndex)`. This only updates the `revoked` bit for key N in `keyMetadataBytes`. It does not touch `apk_0`. [4](#0-3) [5](#0-4) 

**Deduplication: key 1 maps to stored key 0, but has its own `revoked` bit:**

When key 1 is added as a duplicate of key 0 (same encoded public key bytes), `AppendDuplicateKeyMetadata` is called. It stores `revoked=false` and `weight` for key 1 in `keyMetadataBytes`, and stores a mapping `key 1 → storedKeyIndex 0`. The `revoked` bit for key 1 in `keyMetadataBytes` is independent of the `Revoked` flag in `apk_0`. [6](#0-5) [7](#0-6) 

**`GetRuntimeAccountPublicKey` for key N > 0 reads `revoked` from `keyMetadataBytes`, not from `apk_0`:**

```go
// accounts.go lines 318-336
weight, revoked, storedKeyIndex, err := status.AccountPublicKeyMetadata(keyIndex)
// ...
storedKey, err := getStoredPublicKey(a, address, storedKeyIndex)
// ...
return flow.RuntimeAccountPublicKey{
    Revoked: revoked,  // <-- from keyMetadataBytes, NOT from apk_0
}, nil
``` [8](#0-7) 

The `storedKey` (`StoredPublicKey`) has no `Revoked` field — it only carries `PublicKey`, `SignAlgo`, `HashAlgo`. The revoked status is read exclusively from `keyMetadataBytes`.

**Transaction verifier checks `accountKey.Revoked` from `GetRuntimeAccountPublicKey`:** [9](#0-8) 

**Sequence number checker also reads from `keyMetadataBytes` for key N > 0:** [10](#0-9) 

Both checks read `revoked` from `keyMetadataBytes` for key N > 0. Since revoking key 0 does not update `keyMetadataBytes`, key 1 passes both checks with `Revoked=false`.

### Impact Explanation

An attacker who has temporary access to an account (e.g., via a compromised private key) can add a duplicate of key 0 as key 1 (same public key bytes, different weight). When the victim discovers the compromise and revokes key 0 — believing this removes the attacker's access — key 1 remains active with `Revoked=false` in `keyMetadataBytes`. The attacker retains the private key corresponding to the shared public key P and can continue signing and submitting authorized transactions on behalf of the account indefinitely. The victim has no indication that revocation was incomplete.

### Likelihood Explanation

The precondition requires the attacker to have had temporary account access sufficient to call `addPublicKey` (e.g., via a compromised device, phishing, or a malicious co-signer in a multi-key setup). This is a realistic scenario. The attack is silent — the victim's revocation transaction succeeds without error, giving false assurance. The exploit is deterministic and locally reproducible.

### Recommendation

When `RevokeAccountPublicKey(addr, 0)` is called, the implementation must also scan `keyMetadataBytes` for any key N > 0 whose `storedKeyIndex` mapping resolves to stored key index 0, and set `revoked=true` for each such key N. Alternatively, the revocation check in `GetRuntimeAccountPublicKey` for key N > 0 should, when `storedKeyIndex == 0`, also read the `Revoked` flag from `apk_0` and OR it with the `revoked` bit from `keyMetadataBytes`. The same fix must be applied to `GetAccountPublicKeyRevokedStatus`.

### Proof of Concept

```
1. Create account A with key 0 = public key P (weight 1000).
2. Using key 0, add key 1 = same public key P (weight 500) → deduplication stores
   keyMetadataBytes[key1].revoked = false, mapping key1 → storedKeyIndex 0.
3. Call RevokeAccountPublicKey(A, 0):
   - apk_0.Revoked = true  ✓
   - keyMetadataBytes[key1].revoked = false  (unchanged)
4. Call GetRuntimeAccountPublicKey(A, 1):
   - reads revoked from keyMetadataBytes → false
   - returns RuntimeAccountPublicKey{Revoked: false}
5. Submit transaction signed with private key of P, using key index 1 as proposer/authorizer.
6. TransactionVerifier: accountKey.Revoked == false → passes.
   TransactionSequenceNumberChecker: revoked == false → passes.
7. Transaction executes successfully despite key 0 being revoked.
``` [11](#0-10) [3](#0-2) [12](#0-11) [10](#0-9)

func (a *StatefulAccounts) GetRuntimeAccountPublicKey(
	address flow.Address,
	keyIndex uint32,
) (
	flow.RuntimeAccountPublicKey,
	error,
) {
	err := a.accountPublicKeyIndexInRange(address, keyIndex)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, err
	}

	if keyIndex == 0 {
		key, err := getAccountPublicKey0(a, address)
		if err != nil {
			return flow.RuntimeAccountPublicKey{}, fmt.Errorf("failed to get account public key at index %d for %s: %w", keyIndex, address, err)
		}
		return flow.RuntimeAccountPublicKey{
			Index:     keyIndex,
			PublicKey: key.PublicKey,
			SignAlgo:  key.SignAlgo,
			HashAlgo:  key.HashAlgo,
			Weight:    key.Weight,
			Revoked:   key.Revoked,
		}, nil
	}

	status, err := a.getAccountStatus(address)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, err
	}

	// Get account public key metadata.
	weight, revoked, storedKeyIndex, err := status.AccountPublicKeyMetadata(keyIndex)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, fmt.Errorf("failed to get account public key at index %d for %s: %w", keyIndex, address, err)
	}

	// Get stored public key.
	storedKey, err := getStoredPublicKey(a, address, storedKeyIndex)
	if err != nil {
		return flow.RuntimeAccountPublicKey{}, fmt.Errorf("failed to get account public key at index %d for %s: %w", keyIndex, address, err)
	}

	return flow.RuntimeAccountPublicKey{
		Index:     keyIndex,
		PublicKey: storedKey.PublicKey,
		SignAlgo:  storedKey.SignAlgo,
		HashAlgo:  storedKey.HashAlgo,
		Weight:    int(weight),

	}, nil

func revokeAccountPublicKey0(
	a Accounts,
	address flow.Address,
) error {
	accountPublicKey0RegisterID := flow.AccountPublicKey0RegisterID(address)

	publicKey, err := a.GetValue(accountPublicKey0RegisterID)
	if err != nil {
		return err
	}

	const keyIndex = uint32(0)

	if len(publicKey) == 0 {
		return errors.NewAccountPublicKeyNotFoundError(
			address,
			keyIndex)
	}

	decodedPublicKey, err := flow.DecodeAccountPublicKey(publicKey, keyIndex)
	if err != nil {
		return fmt.Errorf(
			"failed to decode account public key 0: %w",
			err)
	}

	decodedPublicKey.Revoked = true

	encodedPublicKey, err := flow.EncodeAccountPublicKey(decodedPublicKey)
	if err != nil {
		encoded, _ := decodedPublicKey.MarshalJSON()
		return errors.NewValueErrorf(
			string(encoded),
			"failed to encode revoked account public key 0: %w",
			err)
	}

	return a.SetValue(accountPublicKey0RegisterID, encodedPublicKey)
```

// AppendDuplicateKeyMetadata appends duplicate key metadata.
func (m *KeyMetadataAppender) AppendDuplicateKeyMetadata(
	keyIndex uint32,
	duplicateStoredKeyIndex uint32,
	revoked bool,
	weight uint16,
) (err error) {

	// Append revoked status and weight
	m.weightAndRevokedStatusBytes, err = appendWeightAndRevokedStatus(m.weightAndRevokedStatusBytes, revoked, weight)
	if err != nil {
		return err
	}

	if !m.deduplicated {
		// Set deduplication flag
		m.deduplicated = true

		// Save mapping start key index.
		m.startIndexForMapping = keyIndex
	}

	// Append duplicate stored key index to mapping
	m.mappingBytes, err = appendStoredKeyIndexToMappings(m.mappingBytes, duplicateStoredKeyIndex)
	if err != nil {
		return err
	}

	return nil
}
