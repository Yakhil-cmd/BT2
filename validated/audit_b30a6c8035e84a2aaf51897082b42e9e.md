### Title
Missing Lower-Bound Validation on `threshold` in FROST Signing Allows Inconsistent Protocol Parameters - (`src/frost/mod.rs`)

### Summary

`assert_sign_inputs` in `src/frost/mod.rs` validates the `threshold` parameter only from above (`threshold > participants.len()`), but never from below. A caller — including a malicious coordinator — can supply `threshold = 1` (or `threshold = 0`) to `sign_v1`, `sign_v2` (EdDSA), or `redjubjub::sign`. Because DKG enforces `threshold >= 2`, the signing protocol then runs with a threshold that is inconsistent with the key material, causing the FROST aggregation to produce an invalid signature or abort, permanently denying signing to honest parties.

### Finding Description

`assert_key_invariants` in `src/dkg.rs` enforces a minimum threshold of 2:

```rust
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

The same lower-bound guard is present in `validate_triple_inputs` for OT-based triple generation: [2](#0-1) 

However, `assert_sign_inputs` — the shared validation function called by every FROST signing entry point — only checks the upper bound:

```rust
// validate threshold
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
``` [3](#0-2) 

There is **no** `threshold < 2` guard. This function is the sole validation gate for:

- `frost::eddsa::sign::sign_v1` and `sign_v2`
- `frost::redjubjub::sign::sign` [4](#0-3) [5](#0-4) 

When `threshold = 1` is passed, `construct_key_package` creates a `KeyPackage` with `min_signers = 1`: [6](#0-5) 

The FROST `round2::sign` then accepts a signing package with only 1 commitment. The resulting signature share is computed against key material that was generated under `threshold >= 2`. The `aggregate` call verifies the signature internally; because the threshold used during signing is inconsistent with the threshold used during key generation, the aggregated signature is cryptographically invalid and `aggregate` returns an error, causing the entire signing session to abort with `ProtocolError::AssertionFailed`. [7](#0-6) 

### Impact Explanation

A malicious coordinator who calls any FROST signing entry point with `threshold = 1` causes every honest participant to complete their round-2 work and then have the session fail at aggregation. The signing output is permanently unusable. This maps to: **High — Permanent denial of signing for honest parties under valid protocol inputs and documented trust assumptions.**

### Likelihood Explanation

The `threshold` parameter is caller-supplied and fully attacker-controlled. No out-of-band mechanism prevents a coordinator from supplying `threshold = 1`. The missing check is a single missing guard that is already present in every other protocol entry point in the codebase, making accidental omission the most likely cause and exploitation trivial.

### Recommendation

Add the same lower-bound guard that exists in `assert_key_invariants` to `assert_sign_inputs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
``` [8](#0-7) 

### Proof of Concept

```rust
// Keys generated with threshold = 3 (valid, >= 2)
let keygen_outputs = run_keygen(&participants, 3, &mut rng);

// Malicious coordinator calls sign with threshold = 1 (no lower-bound check fires)
let result = sign_v1(
    &participants,
    1usize,          // threshold = 1, below the minimum of 2
    me,
    coordinator,
    keygen_outputs[0].1.clone(),
    message,
    rng,
);
// assert_sign_inputs passes (only checks threshold <= participants.len())
// construct_key_package creates KeyPackage { min_signers: 1 }
// aggregate fails: signature invalid against public key generated at threshold=3
// => ProtocolError::AssertionFailed propagated to all honest participants
```

/// Verifies that the sign inputs are valid
pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}

fn construct_key_package(
    threshold: ReconstructionLowerBound,
    me: Participant,
    signing_share: SigningShare,
    verifying_key: &VerifyingKey,
) -> Result<KeyPackage, ProtocolError> {
    let identifier = me.to_identifier()?;
    let verifying_share = signing_share.into();

    Ok(KeyPackage::new(
        identifier,
        signing_share,
        verifying_share,
        *verifying_key,
        u16::try_from(threshold.value()).map_err(|_| {
            ProtocolError::Other("threshold cannot be converted to u16".to_string())
        })?,
    ))
}
