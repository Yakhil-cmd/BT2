### Title
Gateway Stateless Validator Unconditionally Rejects Valid Transactions With Zero L2 Gas Price — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

`StatelessTransactionValidator::validate_resource_bounds` applies a `min_gas_price` floor check against `l2_gas.max_price_per_unit` for every `AllResources` transaction, regardless of whether the transaction allocates any L2 gas at all. In production the floor is `8_000_000_000` (8 Gwei). A structurally valid transaction that carries only L1 gas bounds (zero L2 gas amount and therefore zero L2 gas price) has `l2_gas.max_price_per_unit = 0`, which is unconditionally less than the floor, so the gateway rejects it with `MaxGasPriceTooLow` before the transaction ever reaches the mempool.

---

### Finding Description

`validate_resource_bounds` in `StatelessTransactionValidator` performs three sequential checks:

1. **Zero-fee guard** — rejects if `max_possible_fee(Tip::ZERO) == 0` (at least one resource must be non-zero).
2. **L2 gas price floor** — rejects if `l2_gas.max_price_per_unit.0 < min_gas_price`.
3. **L2 gas amount ceiling** — rejects if `l2_gas.max_amount > max_l2_gas_amount`. [1](#0-0) 

Check (1) is correctly conditioned: a transaction with only L1 gas set passes it because `max_possible_fee` is non-zero. Check (2) is **not** conditioned on whether the transaction actually allocates L2 gas. When `l2_gas.max_amount = 0` the price field is also `0`, and `0 < 8_000_000_000` is always true, so the transaction is rejected.

The production default for `min_gas_price` is `8_000_000_000`: [2](#0-1) 

The test suite masks this because `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING` overrides `min_gas_price` to `0`, so `0 < 0` is false and the `valid_l1_gas` / `valid_l1_data_gas` positive-flow cases pass: [3](#0-2) 

The positive-flow test cases that exercise only-L1-gas and only-L1-data-gas transactions are therefore only valid under the zeroed test config, not under the production config: [4](#0-3) 

---

### Impact Explanation

This is a **High** impact finding matching the scope item: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

Any user who submits a structurally valid `AllResources` transaction that allocates only L1 gas (or only L1 data gas) will have their transaction unconditionally rejected at the stateless gateway layer with `MaxGasPriceTooLow`. The transaction never reaches the mempool or blockifier. The admission decision is wrong: a valid transaction is treated as invalid.

---

### Likelihood Explanation

- The production `StatelessTransactionValidatorConfig` default has `validate_resource_bounds: true` and `min_gas_price: 8_000_000_000`, so the faulty path is active in every deployed gateway instance.
- Any unprivileged user can trigger it by submitting an `AllResources` invoke, declare, or deploy-account transaction with `l2_gas = ResourceBounds::default()` (zero amount, zero price) and a non-zero L1 gas bound.
- No special account, key, or privilege is required.

---

### Recommendation

Guard the L2 gas price floor check with a condition that the transaction actually allocates L2 gas:

```rust
// Only enforce the L2 gas price floor when the transaction allocates L2 gas.
if resource_bounds.l2_gas.max_amount.0 > 0
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
```

Additionally, add a positive-flow test case that uses the **production** `min_gas_price` value with a transaction that has only L1 gas bounds, to prevent regression.

---

### Proof of Concept

```
1. Configure a gateway with the production-default StatelessTransactionValidatorConfig
   (validate_resource_bounds: true, min_gas_price: 8_000_000_000).

2. Submit an RpcInvokeTransaction::V3 with:
     resource_bounds = AllResourceBounds {
         l1_gas: ResourceBounds {
             max_amount: GasAmount(1_000),
             max_price_per_unit: GasPrice(10_000_000_000),  // 10 Gwei — above min
         },
         l2_gas: ResourceBounds::default(),   // amount=0, price=0
         l1_data_gas: ResourceBounds::default(),
     }

3. StatelessTransactionValidator::validate_resource_bounds is called.
   - max_possible_fee(Tip::ZERO) = 1_000 * 10_000_000_000 > 0  → passes check (1).
   - l2_gas.max_price_per_unit.0 = 0 < 8_000_000_000           → FAILS check (2).
   - Returns Err(MaxGasPriceTooLow { gas_price: 0, min_gas_price: 8_000_000_000 }).

4. The transaction is rejected at the gateway before reaching the mempool,
   despite being structurally valid and carrying sufficient L1 gas to cover its fee.
``` [5](#0-4) [6](#0-5)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
