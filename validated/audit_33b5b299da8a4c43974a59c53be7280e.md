### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` to advance `last_reward_block` without distributing rewards, permanently destroying that block's yield for all stakers.

### Finding Description
The protocol specification for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation in `StakingRewardsManagerImpl` performs no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();   // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... no assert_caller_is_sequencer or equivalent
    self.last_reward_block.write(current_block_number);   // always written
    if disable_rewards || self.is_pre_consensus() {
        return;   // exits without distributing rewards
    }
    ...
``` [1](#0-0) 

The `last_reward_block` storage variable is a single global slot. Once it is written to block `N`, the `REWARDS_ALREADY_UPDATED` guard prevents any further call in the same block:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

The spec confirms this is intended to be sequencer-only: [3](#0-2) 

The tests confirm no caller restriction exists — `update_rewards` is called in tests with no `cheat_caller_address_once` setup: [4](#0-3) 

### Impact Explanation
**High — Permanent freezing of unclaimed yield.**

Attack path per block `N`:
1. Attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)` in block `N`.
2. `last_reward_block` is written to `N`; no rewards are distributed.
3. The legitimate sequencer's call in block `N` reverts with `REWARDS_ALREADY_UPDATED`.
4. When the sequencer calls in block `N+1`, reward calculation uses `(N+1 − N) = 1 block` of elapsed time, so block `N`'s rewards are permanently lost — they are never minted or credited to any staker.

Because `last_reward_block` is global, a single attacker call per block griefs every staker simultaneously. The lost yield is irrecoverable; there is no catch-up mechanism.

### Likelihood Explanation
**High.** The entry point is fully public, requires no stake, no role, and no special state. The only cost to the attacker is gas. The attacker can automate the call to front-run the sequencer in every block during the consensus-rewards phase. No privileged access, leaked key, or external dependency is required.

### Recommendation
Add an explicit sequencer-only guard at the top of `update_rewards`, analogous to the pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(...) {
    self.assert_caller_is_sequencer(); // add this
    self.general_prerequisites();
    ...
}
```

Alternatively, restrict the `disable_rewards` parameter so that only the sequencer can pass `true`, or remove the parameter entirely and derive the value internally.

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. A valid staker `S` exists with non-zero balance.
3. Attacker (any address) calls `update_rewards(S, disable_rewards: true)` in block `N`.
   - `last_reward_block` is set to `N`.
   - No rewards are distributed.
4. Sequencer calls `update_rewards(S, disable_rewards: false)` in block `N`.
   - Reverts: `REWARDS_ALREADY_UPDATED`.
5. Sequencer calls in block `N+1`.
   - Rewards calculated for 1 block only; block `N`'s rewards are permanently gone.
6. Repeat every block → continuous, complete denial of staker yield. [5](#0-4) [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
