### Title
Unprivileged Attacker Can Overwrite Any Address's E2EE Encryption Key — (`x/e2ee/keeper/keeper.go`, `x/e2ee/types/msg.go`)

### Summary

`MsgRegisterEncryptionKey` accepts an arbitrary `Address` field and an attacker-controlled `Key`, but neither `ValidateBasic` nor the `MsgServer` handler (`Keeper.RegisterEncryptionKey`) verifies that the transaction signer matches `req.Address`. Any funded account can permanently overwrite any other address's on-chain age X25519 encryption key.

### Finding Description

`MsgRegisterEncryptionKey` is defined with two fields — `Address` and `Key` — and no signer field: [1](#0-0) 

`ValidateBasic` only checks that `Address` is a valid bech32 string and that `Key` parses as a valid age X25519 recipient. There is no check that the signer equals `Address`: [2](#0-1) 

The `MsgServer` is implemented directly by `Keeper`, and `RegisterEncryptionKey` unconditionally calls `registerEncryptionKey(ctx, req.Address, []byte(req.Key))` with no signer extraction or authorization check: [3](#0-2) 

The inner `registerEncryptionKey` simply writes the key to the KV store under the given address: [4](#0-3) 

### Impact Explanation

An attacker can overwrite the canonical on-chain e2ee key for any address with an attacker-controlled age X25519 public key. Any sender who looks up the victim's key from the chain and encrypts a message to it will encrypt to the attacker's key instead. The victim cannot decrypt those messages. The attacker can also continuously re-overwrite the victim's key after any re-registration attempt, making the denial permanent. This falls squarely within the allowed High impact: **"Corruption of e2ee key/message state with direct security impact"** and **"Permanent or long-lived inability for honest users to process valid transactions."**

### Likelihood Explanation

The attack requires only a funded account and the ability to broadcast a standard signed transaction. No privileged access, leaked keys, or external assumptions are needed. The path is fully on-chain and requires no special conditions.

### Recommendation

In `Keeper.RegisterEncryptionKey`, extract the signer from the SDK context and verify it matches `req.Address` before writing to the store. For example:

```go
func (k Keeper) RegisterEncryptionKey(ctx context.Context, req *types.MsgRegisterEncryptionKey) (*types.MsgRegisterEncryptionKeyResponse, error) {
    sdkCtx := sdk.UnwrapSDKContext(ctx)
    signer := sdkCtx.EventManager()... // or use msg.GetSigners()
    // Verify signer == req.Address
    if signer.String() != req.Address {
        return nil, sdkerrors.ErrUnauthorized.Wrapf("signer %s cannot register key for address %s", signer, req.Address)
    }
    ...
}
```

Alternatively, add a `GetSigners() []sdk.AccAddress` method to `MsgRegisterEncryptionKey` that returns the parsed `Address`, so the SDK's ante handler enforces the signer constraint automatically. Also add a corresponding check in `ValidateBasic` or the msg server.

### Proof of Concept

```go
// Unit test (no chain needed):
func TestUnauthorizedKeyRegistration(t *testing.T) {
    ctx := ... // minimal SDK context with KV store
    k := keeper.NewKeeper(storeKey, addressCodec)

    victim := "cosmos1victim..."
    attacker := "cosmos1attacker..."
    attackerKey := "age1..." // valid age X25519 pubkey generated by attacker

    // Attacker submits msg with victim's address but attacker's key
    // ctx signer is attacker, req.Address is victim
    _, err := k.RegisterEncryptionKey(ctx, &types.MsgRegisterEncryptionKey{
        Address: victim,
        Key:     attackerKey,
    })
    require.NoError(t, err) // succeeds with no authorization error

    // Victim's key is now the attacker's key
    resp, _ := k.Key(ctx, &types.KeyRequest{Address: victim})
    require.Equal(t, attackerKey, resp.Key) // attacker controls victim's e2ee key
}
```

func (k Keeper) registerEncryptionKey(
	ctx context.Context,
	address string,
	key []byte,
) error {
	bz, err := k.addressCodec.StringToBytes(address)
	if err != nil {
		return err
	}
	sdk.UnwrapSDKContext(ctx).KVStore(k.storeKey).Set(types.KeyPrefix(bz), key)
	return nil
}
