### Title
`max_l2_gas_amount` Admission Check Skipped for Declare Transactions in Stateless Validator - (File: crates/apollo_gateway/src/stateless_transaction_validator.rs)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a `max_l2_gas_amount` ceiling on every transaction type **except** `Declare`. The guard is written as an empty `if`-branch that silently skips the check for Declare, so any Declare transaction with an arbitrarily large `l2_gas.max_amount` passes stateless gateway admission unchanged.

### Finding Description

Inside `validate_resource_bounds`, the `max_l2_gas_amount` check is structured as:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← empty: check is intentionally skipped for Declare
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

The default configured limit is `1_210_000_000` gas units. [2](#0-1) 

The test suite explicitly confirms the exemption: `test_invalid_max_l2_gas_amount` only parameterises over `TransactionType::DeployAccount` and `TransactionType::Invoke`, and a separate test `valid_l2_gas_amount_on_declare` asserts that a Declare transaction with `max_amount = max_l2_gas_amount + 1` **passes** validation. [3](#0-2) [4](#0-3) 

The stateful validator's `validate_resource_bounds` only checks the L2 gas **price** against the previous block price, not the gas **amount**, so the oversized amount is never caught there either. [5](#0-4) 

### Impact Explanation

A Declare transaction with `l2_gas.max_amount = u64::MAX` (or any value above `max_l2_gas_amount`) passes both the stateless and stateful gateway validators and is admitted to the mempool. The gateway's own admission invariant — that every accepted transaction must have `l2_gas.max_amount ≤ max_l2_gas_amount` — is broken for the Declare transaction type. This matches the **High** impact category: *Mempool/gateway/RPC admission accepts invalid transactions before sequencing*.

### Likelihood Explanation

Any unprivileged user can craft and submit a `RpcDeclareTransaction::V3` with an arbitrarily large `l2_gas.max_amount`. No special account state, balance, or privilege is required to reach the vulnerable code path. The gateway's `validate` entry point is the first line of defence and it is publicly reachable. [6](#0-5) 

### Recommendation

Remove the Declare-specific exemption and apply the same `max_l2_gas_amount` ceiling uniformly to all transaction types. The TODO comment already acknowledges this gap. The fix is to replace the empty `if`-branch with a direct check:

```rust
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If Declare transactions legitimately require a higher ceiling (e.g., because class compilation consumes more L2 gas), a separate, explicitly-configured `max_l2_gas_amount_declare` parameter should be introduced rather than leaving the check absent entirely.

### Proof of Concept

1. Construct an `RpcDeclareTransaction::V3` with `resource_bounds.l2_gas.max_amount = GasAmount(u64::MAX)` and any valid Sierra class.
2. Submit it to the gateway's `add_declare_transaction` endpoint.
3. `StatelessTransactionValidator::validate` calls `validate_resource_bounds`; the `if let RpcTransaction::Declare(_) = tx { }` branch is taken and the `MaxGasAmountTooHigh` error is never returned.
4. The stateful validator's `validate_resource_bounds` only checks the L2 gas price, not the amount — the transaction passes.
5. `validate_by_mempool` is called and the transaction is admitted to the mempool with `l2_gas.max_amount = u64::MAX`, violating the gateway's own admission invariant. [7](#0-6) [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
