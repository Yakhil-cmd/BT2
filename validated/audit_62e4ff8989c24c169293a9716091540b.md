Audit Report

## Title
Unprivileged Caller Can Force-Claim a Neuron for Any Principal via `MemoAndController` - (`rs/nns/governance/src/governance.rs` and `rs/sns/governance/src/governance.rs`)

## Summary

Both NNS and SNS Governance allow any unprivileged ingress caller to force-claim a neuron on behalf of an arbitrary principal by supplying that principal's ID in the `controller` field of `MemoAndController`. No check is performed to verify that the caller equals the specified controller. In NNS, the resulting neuron is created in `NotDissolving` state with `INITIAL_NEURON_DISSOLVE_DELAY`, locking the victim's tokens without their consent. This can break canister-based multi-step staking workflows and force premature token lockup.

## Finding Description

In `manage_neuron_internal`, when a `ClaimOrRefresh { by: MemoAndController(...) }` command is received, it is dispatched directly to `claim_or_refresh_neuron_by_memo_and_controller` with no authorization check on the `controller` field:

```rust
// rs/nns/governance/src/governance.rs L6123-6130
Some(By::MemoAndController(memo_and_controller)) => self
    .claim_or_refresh_neuron_by_memo_and_controller(
        caller,
        memo_and_controller.clone(),
        claim_or_refresh,
    )
    .await
    .map(ManageNeuronResponse::claim_or_refresh_neuron_response),
```

Inside `claim_or_refresh_neuron_by_memo_and_controller` (NNS, L5852–5871), the controller is resolved as:

```rust
let controller = memo_and_controller.controller.unwrap_or(*caller);
```

There is no guard of the form `if controller != *caller { return Err(...) }`. The function proceeds to compute `subaccount = hash(controller, memo)`, checks the ledger balance, and if sufficient, calls `claim_neuron(subaccount, controller, ...)`, creating a neuron owned by the specified `controller` regardless of who the actual caller is.

The identical pattern exists in SNS Governance at L4210–4227.

The NNS `claim_neuron` (L5986–6079) creates the neuron in `NotDissolving` state with `INITIAL_NEURON_DISSOLVE_DELAY`, locking the tokens. The SNS `claim_neuron` (L4332–4357) creates the neuron with `DissolveDelaySeconds(0)`.

This behavior is explicitly tested and confirmed as passing in `test_claim_neuron_memo_and_controller_by_proxy` (L4818–4826 of `rs/nns/governance/tests/governance.rs`), where `caller != owner` and the claim succeeds, assigning the neuron to `owner`.

The exploit path requires only:
1. Observing a pending staking transfer on the public ledger (memo and principal are both public).
2. Sending a single `manage_neuron` ingress call with `MemoAndController { memo: <victim_memo>, controller: Some(<victim_principal>) }`.

No privileged access, key material, or threshold corruption is required.

## Impact Explanation

**NNS (High):** An unprivileged attacker can force-claim a neuron for any principal P whose tokens are sitting in `governance_subaccount(P, memo)`. The neuron is created in `NotDissolving` state with `INITIAL_NEURON_DISSOLVE_DELAY`, locking P's tokens without P's consent. For canister-based staking protocols (DeFi, DAO treasuries, liquid staking), this breaks multi-step staking workflows: the canister sends ICP to the governance subaccount as step 1, the attacker force-claims in step 2, and the canister's subsequent claim attempt finds the neuron already exists, potentially causing a panic, incorrect internal state, or permanent loss of the staking operation. This constitutes a significant NNS security impact with concrete user and protocol harm, matching the High bounty tier.

**SNS (lower):** Same forced-claim behavior, but `DissolveDelaySeconds(0)` means the victim can immediately dissolve and recover tokens. The disruption to canister-based SNS staking workflows remains real but recovery is immediate.

## Likelihood Explanation

High. The attack is trivially automatable: the attacker monitors the NNS ledger for transfers to governance subaccounts, computes the target principal and memo from the transfer, and races to submit the `manage_neuron` call. Canister-based staking systems are particularly vulnerable because their staking subaccounts and memos are deterministic and publicly computable. No special privileges, key material, or coordination is required.

## Recommendation

Add a caller-equals-controller check before resolving the controller in both NNS and SNS `claim_or_refresh_neuron_by_memo_and_controller`:

```rust
if let Some(controller) = memo_and_controller.controller {
    if controller != *caller {
        return Err(GovernanceError::new_with_message(
            ErrorType::NotAuthorized,
            "Caller must equal the specified controller.",
        ));
    }
}
let controller = memo_and_controller.controller.unwrap_or(*caller);
```

If the proxy-claim feature is intentionally required for hardware wallet flows, it should be restricted to a whitelist of trusted intermediaries or require an explicit on-chain authorization from the controller principal, and its interaction with any future warmup periods or time-sensitive governance mechanisms must be explicitly analyzed.

## Proof of Concept

The existing test `test_claim_neuron_memo_and_controller_by_proxy` in `rs/nns/governance/tests/governance.rs` (L4818–4826) is a direct proof of concept: it calls `manage_neuron` with `caller = TEST_NEURON_2_OWNER_PRINCIPAL` and `controller = Some(TEST_NEURON_1_OWNER_PRINCIPAL)`, and asserts the claim succeeds with the neuron assigned to `TEST_NEURON_1_OWNER_PRINCIPAL`. No modification is needed — the test already demonstrates the vulnerability.

For a canister workflow disruption PoC:
1. Deploy a test canister C that sends ICP to `governance_subaccount(C.principal, nonce=42)` and then calls `manage_neuron` to claim.
2. Before C's claim call, submit `manage_neuron` from any other principal with `MemoAndController { memo: 42, controller: Some(C.principal) }`.

```rust
fn do_test_claim_neuron_by_memo_and_controller(owner: PrincipalId, caller: PrincipalId) {
    let memo = 1234_u64;
    let stake = Tokens::from_tokens(10_u64).unwrap();
    let (_, mut gov, _) =
        governance_with_staked_unclaimed_neuron(&owner, memo, Tokens::from_tokens(10_u64).unwrap());

    let manage_neuron_response = gov
        .manage_neuron(
            &caller,
            &ManageNeuron {
                neuron_id_or_subaccount: None,
                id: None,
                command: Some(Command::ClaimOrRefresh(ClaimOrRefresh {
                    by: Some(By::MemoAndController(MemoAndController {
                        memo,
                        controller: Some(owner),
                    })),
                })),
            },
        )
        .now_or_never()
        .unwrap();

    let nid = match manage_neuron_response.command.unwrap() {
        CommandResponse::ClaimOrRefresh(response) => response.refreshed_neuron_id,
        CommandResponse::Error(error) => panic!("Error claiming neuron: {error:?}"),
        _ => panic!("Invalid response."),
    };

    assert!(nid.is_some());
    let nid = nid.unwrap();
    let neuron = gov.neuron_store.with_neuron(&nid, |n| n.clone()).unwrap();
    assert_eq!(neuron.controller(), owner);
    assert_eq!(neuron.cached_neuron_stake_e8s, stake.get_e8s());
```
