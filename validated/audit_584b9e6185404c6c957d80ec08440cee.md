### Title
`MsgTurnBridge` Handler Is a No-Op, Making Bridge Pause Permanently Ineffective - (File: x/cronos/keeper/msg_server.go)

### Summary
The `TurnBridge` message server handler in `x/cronos/keeper/msg_server.go` is implemented as a stub that returns `nil, nil` without performing any state change. Any authorized admin calling `MsgTurnBridge{Enable: false}` receives a success response, but the bridge is never actually disabled. All bridge and conversion operations continue to execute as if the bridge were active, permanently bypassing the intended emergency-pause authority.

### Finding Description
The `TurnBridge` RPC is the designated mechanism for an authorized operator (holding `CanTurnBridge` permission) to halt the Gravity bridge in an emergency. The proto definition, the permission system, and the integration tests all treat it as a real control surface. However, the server-side implementation is:

```go
// x/cronos/keeper/msg_server.go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

It writes nothing to the KV store, sets no parameter, and emits no event. The `MsgTurnBridge` message is accepted by the chain, fees are charged, and the caller observes a successful transaction — but the bridge state is unchanged.

Every downstream path that should respect the bridge-active flag is therefore permanently unguarded:

- `IBCConversionModule.OnRecvPacket` calls `ConvertVouchersToEvmCoins` unconditionally; `canBeConverted` only checks denom mapping, never bridge status. [2](#0-1) [3](#0-2) 

- `ConvertVouchersToEvmCoins` mints EVM coins (including the native CRO equivalent) with no bridge-active guard. [4](#0-3) 

- `MsgConvertVouchers` (the direct user-facing message) delegates straight to `ConvertVouchersToEvmCoins` with no bridge-active check. [5](#0-4) 

- The EVM log handlers (`SendToIbcHandler`, `SendToIbcV2Handler`, `SendCroToIbcHandler`) call `IbcTransferCoins` without consulting bridge status. [6](#0-5) 

### Impact Explanation
**High — Bypass of Cronos admin/bridge authority check.**

When an operator with `CanTurnBridge` permission issues `TurnBridge(false)` to halt the bridge during an incident, the transaction succeeds silently and the bridge remains fully operational. Every unprivileged user can continue to:

- Receive IBC→CRC20/CRC21 conversions via `OnRecvPacket`
- Mint CRO EVM equivalents via `MsgConvertVouchers`
- Initiate IBC outbound transfers via EVM log hooks

The emergency-pause authority is completely nullified. This directly matches the "Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks" impact class.

### Likelihood Explanation
The `TurnBridge` message is a documented, permissioned, user-facing RPC. Any operator who believes they have halted the bridge (because the tx returned success) will not take further protective action, leaving the bridge open indefinitely. The integration test `test_gravity_turn_bridge` explicitly asserts that bridge operations fail after `turn_bridge("false")`, confirming the intended behavior is the opposite of what the code delivers. [7](#0-6) 

### Recommendation
Implement `TurnBridge` to actually persist the bridge-active state. The handler should update the module parameter (e.g., a `BridgeActive bool` field in `Params`) and all bridge entry points — `ConvertVouchersToEvmCoins`, `IBCConversionModule.OnRecvPacket`, and the EVM log handlers — should gate execution on that parameter:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    params := k.GetParams(ctx)
    params.BridgeActive = msg.Enable
    if err := k.SetParams(ctx, params); err != nil {
        return nil, err
    }
    return &types.MsgTurnBridgeResponse{}, nil
}
```

Then add a guard at the top of `ConvertVouchersToEvmCoins` and in `IBCConversionModule.OnRecvPacket`:

```go
if !params.BridgeActive {
    return errorsmod.Wrap(types.ErrBridgeDisabled, "bridge is currently disabled")
}
```

### Proof of Concept
1. Grant address `A` the `CanTurnBridge` permission via `MsgUpdatePermissions`.
2. Address `A` broadcasts `MsgTurnBridge{Sender: A, Enable: false}`.
3. Transaction is accepted on-chain with code 0 and no error.
4. Unprivileged address `B` sends an IBC packet to Cronos carrying a mapped denom.
5. `IBCConversionModule.OnRecvPacket` fires, `canBeConverted` returns `true` (denom mapping exists), `ConvertVouchersToEvmCoins` mints CRC20 tokens to `B` — bridge is still fully active despite the admin's halt attempt. [1](#0-0) [8](#0-7)

// OnRecvPacket implements the IBCModule interface.
func (im IBCConversionModule) OnRecvPacket(
	ctx sdk.Context,
	channelVersion string,
	packet channeltypes.Packet,
	relayer sdk.AccAddress,
) exported.Acknowledgement {
	cacheCtx, commit := ctx.CacheContext()
	ack := im.app.OnRecvPacket(cacheCtx, channelVersion, packet, relayer)
	if !ack.Success() {
		// Underlying transfer failed: discard cacheCtx writes and return the
		// failure ack. Committing would persist a half-applied transfer.
		return ack
	}
	data, err := transferTypes.UnmarshalPacketData(packet.GetData(), channelVersion, "")
	if err != nil {
		return channeltypes.NewErrorAcknowledgement(errors.Wrap(sdkerrors.ErrUnknownRequest,
			"cannot unmarshal ICS-20 transfer packet data in middleware"))
	}
	denom := im.getIbcDenomFromPacketAndData(packet, data.Token)
	if im.canBeConverted(cacheCtx, denom) {
		transferAmount, ok := sdkmath.NewIntFromString(data.Token.Amount)
		if !ok {
			return channeltypes.NewErrorAcknowledgement(errors.Wrapf(
				transferTypes.ErrInvalidAmount,
				"unable to parse transfer amount (%s) into sdk.Int in middleware",
				data.Token.Amount,
			))
		}
		token := sdk.NewCoin(denom, transferAmount)
		if err := im.cronoskeeper.ConvertVouchersToEvmCoins(cacheCtx, data.Receiver, sdk.NewCoins(token)); err != nil {
			im.cronoskeeper.Logger(ctx).Error(
				"failed to convert vouchers on recv",
				"denom", denom,
				"receiver", data.Receiver,
				"error", err,
			)
			return channeltypes.NewErrorAcknowledgement(err)
		}
	}
	commit()
	return ack
```

func (k Keeper) ConvertVouchersToEvmCoins(ctx sdk.Context, from string, coins sdk.Coins) error {
	acc, err := sdk.AccAddressFromBech32(from)
	if err != nil {
		return err
	}

	params := k.GetParams(ctx)
	evmParams := k.GetEvmParams(ctx)
	for _, c := range coins {
		switch c.Denom {
		case params.IbcCroDenom:
			if params.IbcCroDenom == "" {
				return errorsmod.Wrap(types.ErrIbcCroDenomEmpty, "ibc is disabled")
			}

			// Send ibc tokens to escrow address
			err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, acc, types.ModuleName, sdk.NewCoins(c))
			if err != nil {
				return err
			}
			// Compute new amount, because basecro is a 8 decimals token, we need to multiply by 10^10 to make it
			// a 18 decimals token
			amount18dec := sdk.NewCoin(evmParams.EvmDenom, c.Amount.Mul(sdkmath.NewIntFromBigInt(types.TenPowTen)))

			// Mint new evm tokens
			if err := k.bankKeeper.MintCoins(
				ctx, types.ModuleName, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

			// Send evm tokens to receiver
			if err := k.bankKeeper.SendCoinsFromModuleToAccount(
				ctx, types.ModuleName, acc, sdk.NewCoins(amount18dec),
			); err != nil {
				return err
			}

		default:
			err := k.ConvertCoinFromNativeToCRC21(ctx, common.BytesToAddress(acc.Bytes()), c, params.EnableAutoDeployment)
			if err != nil {
				return err
			}
		}
	}
	defer func() {
		for _, a := range coins {
			if a.Amount.IsInt64() {
				telemetry.SetGaugeWithLabels(
					[]string{"tx", "msg", "ConvertVouchersToEvmCoins"},
					float32(a.Amount.Int64()),
					[]metrics.Label{telemetry.NewLabel("denom", a.Denom)},
				)
			}
		}
	}()
	return nil
}
