### Title
Unrestricted `disable_rewards` Parameter in `update_rewards` Enables Permanent Yield Freezing — (`src/staking/staking.cairo`)

### Summary

The `update_rewards` function is publicly callable with no access control on its `disable_rewards` parameter. Any unprivileged caller can invoke `update_rewards(any_active_staker, disable_rewards: true)` once per block, consuming the global `last_reward_block` slot without distributing any rewards. Chaining this call across consecutive blocks permanently freezes unclaimed yield for every staker in the protocol.

### Finding Description

`update_rewards` is exposed as a public ABI function in `StakingRewardsManagerImpl`. Its only gate is `general_prerequisites`, which checks the pause flag and a non-zero caller — no role or identity check is performed. [1](#0-0) 

The function enforces a **global** one-call-per-block invariant via `last_reward_block`: [2](#0-1) 

After writing `last_reward_block = current_block_number`, the function checks `disable_rewards` and returns early — **without distributing any rewards** — if the caller passed `true`: [3](#0-2) 

Because `last_reward_block` is a single global storage slot shared across all stakers, one call per block is sufficient to block every staker's reward update for that block: [4](#0-3) 

The attacker chains these calls across blocks:

- **Block N**: `update_rewards(active_staker, disable_rewards: true)` → `last_reward_block = N`, zero rewards distributed.
- **Block N+1**: same call → `last_reward_block = N+1`, zero rewards distributed.
- Repeated indefinitely at the cost of one transaction per block.

Any legitimate call to `update_rewards` in the same block fails with `REWARDS_ALREADY_UPDATED` because `current_block_number > last_reward_block` is false.

The analogy to the external report is direct: just as a liquidator chains partial liquidations by strategically choosing which collateral to take first — each call individually valid, the sequence bypassing the 50 % cap — here an attacker chains `update_rewards(disable_rewards: true)` calls across blocks, each individually valid, the sequence permanently bypassing the protocol's intended reward distribution.

### Impact Explanation

All stakers and pool members accumulate zero unclaimed rewards for as long as the attack is sustained. Because the cost per block is only gas, a motivated attacker can maintain this indefinitely, constituting **permanent freezing of unclaimed yield** for the entire protocol.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield or unclaimed royalties**.

### Likelihood Explanation

- The function is `#[abi(embed_v0)]` with no role guard.
- Any EOA or contract with a non-zero address can call it.
- The only requirement is one transaction per block, which is cheap on Starknet.
- No special knowledge, capital, or privileged access is needed.

Likelihood is **High**.

### Recommendation

1. **Restrict who may pass `disable_rewards: true`**: add a role check (e.g., `only_operator` or `only_block_proposer`) before honouring `disable_rewards: true`.
2. **Alternatively**, split the function: keep a permissionless `update_rewards(staker_address)` that always distributes rewards, and add a separate privileged `skip_rewards(staker_address)` callable only by an authorised role.
3. **At minimum**, ensure that `last_reward_block` is only advanced when rewards are actually distributed or when the caller is authorised to suppress them.

### Proof of Concept

```
// Attacker script (pseudocode, one tx per block)
loop {
    staking.update_rewards(
        staker_address = any_active_staker,   // any valid staker
        disable_rewards = true,
    );
    wait_for_next_block();
}
```

Step-by-step:

1. Attacker calls `update_rewards(active_staker, true)` in block N.
2. `last_reward_block` is written to N; the early-return fires; no rewards are minted or transferred.
3. Any legitimate call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`.
4. Attacker repeats in block N+1, N+2, … indefinitely.
5. All stakers' `unclaimed_rewards_own` fields remain at their current values; pool `cumulative_rewards_trace` is never updated; pool members earn nothing. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
