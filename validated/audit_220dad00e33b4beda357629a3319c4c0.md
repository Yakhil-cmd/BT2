### Title
Unauthorized Native Token Transfer via Unchecked Arbitrary `sender` in Bank Precompile `transfer` Method - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The bank precompile's `transfer` method accepts a caller-supplied `sender` address as `args[0]` and uses it directly as the `from` address in `bankKeeper.SendCoins`, with no check that `sender == contract.Caller()` or that the sender has authorized the transfer. Any EVM contract can call `bankPrecompile.transfer(victimAddress, attackerAddress, amount)` to drain `evm/<callerContract>` native bank tokens from any address without the victim's consent.

### Finding Description

In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` case unpacks `sender` from `args[0]` (a fully attacker-controlled ABI parameter) and passes it directly as the `from` address to `bankKeeper.SendCoins`:

```go
// x/cronos/keeper/precompiles/bank.go lines 175-192
sender := args[0].(common.Address)   // caller-supplied, no validation
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
...
from := sdk.AccAddress(sender.Bytes())
to := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())  // denom is tied to calling contract
...
if err := bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt)); err != nil {
``` [1](#0-0) 

There is no guard of the form `if sender != contract.Caller() { return error }`. The only access control present is `checkBlockedAddr(to)`, which only prevents sending to module-blocked addresses — it does not protect the `from` address at all. [2](#0-1) 

The denom is correctly scoped to `contract.Caller()` (so only `evm/<callerContract>` tokens are affected), but the `from` address is entirely attacker-controlled. Any contract can therefore move `evm/<itsOwnAddress>` tokens out of any account that holds them, without that account's consent.

The intended usage shown in `TestBank.sol` passes `msg.sender` as the first argument:

```solidity
// integration_tests/contracts/contracts/TestBank.sol line 37
return bank.transfer(msg.sender, recipient, amount);
``` [3](#0-2) 

But nothing in the precompile enforces this pattern. A malicious contract can pass any address as `sender`.

### Impact Explanation

**Critical — Unauthorized transfer of native bank tokens.**

A malicious EVM contract can drain `evm/<contractAddress>` native bank module tokens from any user's account without their consent. The bank precompile's `SendCoins` call executes at the Cosmos SDK level, bypassing all EVM-layer approval mechanisms (no `approve`/`transferFrom` required). Any user who holds `evm/<maliciousContract>` tokens — whether received via `mint`, IBC, or a prior transfer — can have those tokens silently moved to an attacker-controlled address.

### Likelihood Explanation

The entry path is fully unprivileged: any deployed EVM contract can call the bank precompile at address `0x0000000000000000000000000000000000000064` with method selector `0xbeabacc8` (`transfer(address,address,uint256)`), supplying an arbitrary victim address as `args[0]`. [4](#0-3) 

A realistic attack scenario:

1. Attacker deploys `MaliciousContract` and advertises it as a DeFi protocol.
2. `MaliciousContract` calls `bank.mint(victimAddress, amount)` to issue `evm/MaliciousContract` tokens to users.
3. At any later time, `MaliciousContract` calls `bank.transfer(victimAddress, attackerAddress, amount)` — the precompile executes `bankKeeper.SendCoins(ctx, victim, attacker, coins)` with no authorization check on `victim`.
4. All `evm/MaliciousContract` tokens are drained from every victim account.

### Recommendation

Enforce that the `sender` argument equals `contract.Caller()`. The simplest fix is to replace the caller-supplied `sender` with `contract.Caller()` directly (removing the parameter), mirroring how `mint`/`burn` use `contract.Caller()` for the denom authority:

```go
case TransferMethodName:
    args, err := method.Inputs.Unpack(contract.Input[4:])
    ...
    // sender must be the calling contract itself
    from := sdk.AccAddress(contract.Caller().Bytes())
    recipient := args[0].(common.Address)
    amount := args[1].(*big.Int)
    ...
    to := sdk.AccAddress(recipient.Bytes())
    denom := EVMDenom(contract.Caller())
    ...
    bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

Alternatively, add an explicit guard before `SendCoins`:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be the calling contract")
}
```

The ABI interface in `x/cronos/events/bindings/src/Bank.sol` should be updated accordingly. [5](#0-4)

    function transfer(address, address, uint256) external payable returns (bool);
}

contract MaliciousContract {
    address constant bankPrecompile = 0x0000000000000000000000000000000000000064;
    IBankModule bank = IBankModule(bankPrecompile);

    // Step 1 (optional): mint evm/MaliciousContract tokens to victims
    function mintToVictim(address victim, uint256 amount) external {
        bank.mint(victim, amount);
    }

    // Step 2: drain victim's evm/MaliciousContract tokens with no approval
    function drain(address victim, address attacker, uint256 amount) external {
        // bank precompile executes SendCoins(victim → attacker) with no auth check
        bank.transfer(victim, attacker, amount);
    }
}
```

Calling `drain(victimAddress, attackerAddress, victimBalance)` transfers all `evm/MaliciousContract` native bank tokens from `victimAddress` to `attackerAddress` in a single unprivileged transaction, with no signature or approval from `victimAddress`.
