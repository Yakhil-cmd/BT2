### Title
Unprivileged Caller Can Permanently Freeze Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable by any non-zero address. By passing `disable_rewards: true`, an attacker can advance the global `last_reward_block` pointer without distributing any rewards, permanently preventing all stakers from receiving rewards for that block. This is an analog to the MIMOProxy vulnerability class: an unprivileged caller manipulates critical contract state (here, the reward-tracking pointer rather than a permissions mapping) in a way the protocol cannot reverse.

### Finding Description
`update_rewards` is exposed as part of the public `IStakingRewardsManager` interface with no role-based access control — only `general_prerequisites()` is enforced, which checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

The function writes `last_reward_block` to the current block number **before** evaluating the `disable_rewards` flag: [2](#0-1) 

If `disable_rewards` is `true`, the function returns immediately after that write, distributing nothing: [3](#0-2) 

`last_reward_block` is a single global storage slot shared across all stakers: [4](#0-3) 

The guard at the top of the function enforces strict monotonicity — once the pointer is advanced to block N, no call for block N can ever succeed again: [5](#0-4) 

Because the write happens unconditionally before the early return, an attacker who calls `update_rewards(any_active_staker, true)` for block N permanently consumes that block's reward slot for the entire protocol.

### Impact Explanation
Every block's rewards are a one-time opportunity gated by `last_reward_block`. Once an attacker advances the pointer for block N with `disable_rewards: true`, the STRK rewards that would have been minted and distributed to all stakers and delegators for that block are permanently unclaimable. Repeating this call once per block freezes all consensus-era yield indefinitely. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

### Likelihood Explanation
The attack requires one Starknet transaction per block. The attacker must supply any currently-active staker address with non-zero STRK balance (trivially discoverable on-chain). There is no financial cost beyond gas, and no privileged role or key is needed. The attack is therefore reachable by any unprivileged public caller, making likelihood **Medium** (continuous griefing, low per-block cost, no profit motive required).

### Recommendation
Two complementary fixes:

1. **Move the `last_reward_block` write to after the `disable_rewards` guard.** If rewards are disabled, the block slot should not be consumed:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only advance the pointer when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... rest of reward distribution
```

2. **Restrict who may pass `disable_rewards: true`.** If disabling rewards is a legitimate protocol operation (e.g., during staker removal), gate it behind a trusted role (e.g., `only_app_governor` or `only_security_agent`) rather than leaving it open to any caller.

### Proof of Concept

```
// Attacker (any non-zero address) executes once per block:
staking.update_rewards(
    staker_address: <any active staker with non-zero balance>,
    disable_rewards: true,
);
// Effect:
//   last_reward_block = current_block_number   (written unconditionally)
//   rewards distributed = 0                    (early return)
//
// All subsequent calls for this block revert with REWARDS_ALREADY_UPDATED.
// The STRK rewards for this block are permanently unclaimable by every staker
// and delegator in the protocol.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
