### Title
Missing CREATE2 Storage-Collision Check Allows Redeployment with Attacker-Controlled Storage State — (File: `evm_interpreter/src/ee_trait_impl.rs`)

---

### Summary

ZKsync OS's EVM implementation explicitly skips the EVM-spec-required check that a CREATE2 target address must have **empty storage** before deployment proceeds. This is the direct analog of the reported "missing pool existence check" pattern: a required pre-condition check is absent, allowing an attacker to pre-populate the resource (storage slots) that the victim's contract will inherit, leading to fund loss.

---

### Finding Description

EIP-684 / the Paris hard fork requires that a CREATE or CREATE2 deployment **fail** if the target address already has non-empty storage. ZKsync OS explicitly omits this check with an acknowledged comment:

```rust
// Check there's no contract already deployed at this address.
// NB: EVM also specifies that the address should have empty storage,
// but we cannot perform such a check for now.
// We need to check this here (not when we actually deploy the code)
// because if this check fails the constructor shouldn't be executed.
if deployee_code_len != 0 || deployee_nonce != 0 {
    ...
    return Ok(false);
}
``` [1](#0-0) 

Only `code_len` and `nonce` are checked; **storage emptiness is never verified**. The test suite confirms this is a known, active deviation:

```yaml
create2collisionStorageParis.json:
  enabled: false
  comment: We do not check storage for collision
``` [2](#0-1) [3](#0-2) 

---

### Impact Explanation

An attacker who can interact with a public CREATE2 factory (deployer address is a known contract, salt is user-controlled or predictable, initcode is public) can:

1. Call the factory to deploy a child contract at address **X** (CREATE2: same deployer + salt + initcode → deterministic address).
2. From within that contract, write attacker-controlled values to storage slots the victim's contract will use for security-critical state (e.g., `initialized = true`, `owner = attacker`, balance mappings).
3. Selfdestruct the contract — code and nonce are cleared, **storage persists**.
4. Trigger a redeployment at address **X** (same CREATE2 parameters).
5. On Ethereum mainnet this redeployment **reverts** (non-empty storage collision). On ZKsync OS it **succeeds**, and the constructor runs against the attacker's pre-set storage.
6. The redeployed contract inherits the malicious storage state, giving the attacker unauthorized control or enabling direct fund theft.

Impact: **direct loss of funds** for any user of the redeployed contract that relies on storage for initialization guards, ownership, or balance accounting.

---

### Likelihood Explanation

- Requires a publicly callable CREATE2 factory with a predictable or user-supplied salt — a common pattern (Uniswap V3 pool factory, ERC-4337 account factories, proxy deployers).
- Requires the victim contract to be redeployable (selfdestruct + redeploy pattern, or an upgradeable proxy that can be reset).
- No privileged access, leaked keys, or oracle manipulation needed — a single unprivileged EOA can execute all steps.
- The deviation is explicitly documented and test-disabled, confirming it is a persistent, unmitigated gap.

Likelihood: **Medium** — the preconditions (public factory + redeployable contract) are common in DeFi but not universal.

---

### Recommendation

Implement the storage-emptiness check before executing a CREATE/CREATE2 constructor frame. Concretely, in `before_executing_frame` in `evm_interpreter/src/ee_trait_impl.rs`, after the existing `code_len`/`nonce` checks, add a check that queries whether the target address has any non-zero storage slots. If any exist, treat it as a `CreateCollision` and burn all gas, consistent with the existing collision handling. Re-enable the `create2collisionStorageParis.json` test suite entries to validate correctness.

### Proof of Concept

```
Attacker EOA  →  PublicFactory.deploy(salt=S, initcode=I)
                   └─ CREATE2(deployer=Factory, salt=S, initcode=I) → address X
                        └─ constructor: SSTORE(slot 0, attacker_address)  // set owner
                                        SELFDESTRUCT(attacker)             // clear code, keep storage

// Storage at X: slot[0] = attacker_address  (persists after selfdestruct)
// Code at X: empty, nonce at X: 0

Victim EOA    →  PublicFactory.deploy(salt=S, initcode=I)   // same params → same address X
                   └─ CREATE2(deployer=Factory, salt=S, initcode=I) → address X
                        ZKsync OS check: code_len==0 ✓, nonce==0 ✓, storage? NOT CHECKED
                        └─ constructor runs; contract deployed at X
                           slot[0] already = attacker_address  ← inherited
                           contract's `owner` is now the attacker
                           attacker calls withdraw() → drains all funds
```
