### Title
Gateway Stateful Admission Uses Previous Block L2 Gas Price Instead of Next Block Price, Admitting Transactions That Will Fail Execution — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from the **previous block's** L2 gas price, but every admitted transaction is executed in the **next block**, whose L2 gas price is computed by the EIP-1559 mechanism and can be strictly higher. The gateway therefore admits V3 (`AllResources`) transactions whose `l2_gas.max_price_per_unit` lies in the half-open interval `[prev_price, next_price)`. Those transactions pass every gateway check, enter the mempool, and are then rejected by the batcher's `check_fee_bounds` with `MaxGasPriceTooLow`. The code itself carries an explicit acknowledgement of the wrong reference: `// TODO(Arni): getnext_l2_gas_price from the block header.`

---

### Finding Description

**Wrong price reference in admission check.**

`validate_resource_bounds` reads the threshold from the latest committed block:

```rust
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;                          // ← previous block price
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,            // ← used as threshold
)?;
``` [1](#0-0) 

The threshold function then enforces `tx_l2_gas_price ≥ prev_price × min_gas_price_percentage / 100`. With the production default of `min_gas_price_percentage = 100`, the threshold equals the previous block price exactly. [2](#0-1) [3](#0-2) 

**The blockifier validation inside the gateway also uses the wrong price.**

`run_validate_entry_point` increments the block number but keeps the same gas prices from the previous block:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
// gas_prices are NOT updated — still the previous block's prices
``` [4](#0-3) 

So `perform_pre_validation_stage → check_fee_bounds` inside the gateway also passes for a transaction priced at `prev_price`, because it compares against the same stale gas prices.

**The batcher uses the correct next-block price.**

The consensus orchestrator computes the next block's L2 gas price via EIP-1559 (`calculate_next_base_gas_price`). When a block is above the gas target, the next price is strictly higher:

```rust
let adjusted_price_u256 =
    if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };
``` [5](#0-4) 

The batcher builds the block with this higher price. `check_fee_bounds` in `perform_pre_validation_stage` then compares the transaction's `max_price_per_unit` against the actual next-block price and rejects the transaction with `MaxGasPriceTooLow`. [6](#0-5) 

**The gap is bounded but reliably non-zero during high utilization.** The EIP-1559 denominator (`gas_price_max_change_denominator`) limits the per-block increase, but during sustained high utilization the next-block price is always above the previous-block price, making the admission window permanently open.

---

### Impact Explanation

Every V3 (`AllResources`) transaction with `l2_gas.max_price_per_unit` equal to the previous block's L2 gas price passes the gateway's stateful admission check and enters the mempool, but is then rejected by the batcher's `check_fee_bounds` during block building. This matches the allowed impact:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

The batcher must process and discard these transactions, wasting execution resources. An attacker can continuously submit such transactions during any period of above-target block utilization to degrade batcher throughput without paying fees (the transactions are rejected before fee deduction).

---

### Likelihood Explanation

The condition is triggered whenever a block's gas usage exceeds the EIP-1559 gas target, which is the normal operating condition during any period of moderate-to-high network activity. No privileged access is required; any user can submit a V3 transaction priced at exactly the previous block's L2 gas price.

---

### Recommendation

Replace the previous-block L2 gas price with the **next block's** computed L2 gas price in `validate_resource_bounds`. The next-block price is already computed by `calculate_next_l2_gas_price_for_fin` / `calculate_next_base_gas_price` in the consensus orchestrator and should be exposed via the block header or a dedicated provider so the gateway can use it as the admission threshold. The existing TODO comment confirms this is the intended fix:

```
// TODO(Arni): getnext_l2_gas_price from the block header.
``` [7](#0-6) 

Similarly, `run_validate_entry_point` should construct its `BlockContext` with the next block's gas prices, not the previous block's, so that the blockifier pre-validation inside the gateway is consistent with what the batcher will enforce.

---

### Proof of Concept

1. Observe the current (previous) block's L2 gas price: `P_prev`.
2. The block is above the gas target, so the next block's price will be `P_next = P_prev + delta` (delta > 0, bounded by `gas_price_max_change_denominator`).
3. Submit a V3 invoke transaction with `l2_gas.max_price_per_unit = P_prev`.
4. **Gateway admission** (`validate_tx_l2_gas_price_within_threshold`): `P_prev ≥ P_prev × 100% / 100` → **passes**.
5. **Gateway blockifier validation** (`run_validate_entry_point`): uses previous block gas prices → `check_fee_bounds` compares `P_prev` against `P_prev` → **passes**.
6. Transaction enters the mempool.
7. **Batcher execution**: block context has `l2_gas_price = P_next > P_prev`; `check_fee_bounds` compares `P_prev` against `P_next` → **fails** with `MaxGasPriceTooLow`.
8. Transaction is discarded after consuming batcher resources; no fee is charged. [2](#0-1) [8](#0-7) [9](#0-8)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
