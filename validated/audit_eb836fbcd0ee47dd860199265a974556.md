### Title
Unprivileged Caller Can Permanently Freeze All Stakers' Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

Any unprivileged caller can invoke `update_rewards` with `disable_rewards: true` for any valid staker, consuming the protocol's single per-block reward slot without distributing any rewards. Because `last_reward_block` is a **global** counter updated on every call, an attacker can front-run legitimate reward updates every block, permanently preventing all stakers from accumulating consensus rewards.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. There is no role check or staker-identity check. [1](#0-0) 

The function accepts a caller-controlled `disable_rewards: bool` parameter. When `true`, the function:

1. Validates the staker exists and has non-zero balance (requires only a publicly visible staker address).
2. **Writes the current block number into the global `last_reward_block`.**
3. Returns early — no rewards are distributed. [2](#0-1) 

The global `last_reward_block` enforces a strict one-call-per-block invariant: [3](#0-2) 

`last_reward_block` is a single global storage slot, not per-staker: [4](#0-3) 

Once an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N, `last_reward_block` is set to N. Any subsequent legitimate call in block N fails with `REWARDS_ALREADY_UPDATED`. Because the slot is global, **all stakers** lose their rewards for that block.

The analog to the PearVault report is exact:

| PearVault | Starknet Staking |
|---|---|
| `requestWithdrawal(1 share)` inflates `totalPendingShares` | `update_rewards(staker, true)` updates `last_reward_block` |
| `_canDirectWithdraw()` returns `false` for all users | `current_block_number > last_reward_block` fails for all callers |
| All users forced into withdrawal queue | All stakers lose consensus rewards for that block |
| Persists until attacker's request is executed | Persists as long as attacker repeats each block |

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block prevents the entire protocol from distributing consensus rewards indefinitely. Stakers accumulate zero `unclaimed_rewards_own` and pools receive zero rewards. The yield is not stolen — it is simply never credited, constituting permanent freezing of unclaimed yield for all protocol participants.

---

### Likelihood Explanation

**High.** The attack requires:
- A non-zero caller address (trivially satisfied).
- Any valid staker address with non-zero balance (all staker addresses are publicly visible on-chain via `NewStaker` events and the `stakers` vector).
- Gas cost per block — the only economic barrier.

No privileged role, leaked key, or external dependency is needed. The attack is fully automatable.

---

### Recommendation

Add an access control check so that `update_rewards` can only be called by the staker themselves or their registered operational address. Alternatively, do not write `last_reward_block` when `disable_rewards: true` — only commit the block number after rewards have actually been distributed.

---

### Proof of Concept

1. Attacker observes any valid staker address `S` with non-zero STRK balance (e.g., from `NewStaker` events).
2. In every block, attacker submits `update_rewards(S, disable_rewards: true)` before any legitimate call.
3. `last_reward_block` is updated to the current block number; no rewards are distributed.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeated every block: all stakers permanently accumulate zero consensus rewards. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
