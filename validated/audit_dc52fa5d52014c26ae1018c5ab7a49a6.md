### Title
Missing Identity-Point Validation on ElGamal Public Key in CKD Protocol Discloses Confidential Derived Secret - (`File: src/confidential_key_derivation/protocol.rs`)

---

### Summary

The `ckd()` function in `src/confidential_key_derivation/protocol.rs` accepts an externally supplied ElGamal public key `app_pk: PublicKey` without validating that it is not the identity element (point at infinity) of BLS12-381 G1. A malicious caller who submits `app_pk = G1::identity()` causes every MPC node's ciphertext contribution `C_i` to collapse to the raw BLS signature share `S_i = x_i · H(pk, app_id)`. After Lagrange aggregation by the coordinator, the final `CKDOutput.big_c` equals `msk · H(pk, app_id)` — the confidential derived secret — which is then broadcast on-chain in plaintext.

---

### Finding Description

The CKD protocol is an ElGamal-based scheme. Each node computes:

```
C_i = S_i + y_i · A      where A = app_pk (the caller-supplied public key)
```

The security of the scheme depends entirely on `A` being a valid non-identity point with an unknown discrete logarithm `a`. The coordinator aggregates contributions and publishes `(Y, C)` on-chain, where `C = msk · H(pk, app_id) + a · Y`. The app recovers the secret as `sig = C − a · Y`.

In `compute_signature_share()`:

```rust
// C <- S + y . A
let big_c = big_s + app_pk * y.0;
```

If `app_pk = G1Projective::identity()`, then `app_pk * y.0 = identity`, so `big_c = big_s = x_i · H(pk, app_id)`. After Lagrange aggregation:

```
C = Σ λ_i · C_i = Σ λ_i · S_i = msk · H(pk, app_id)
```

The `CKDOutput { big_y: Y, big_c: C }` is returned and published on-chain. `big_c` is now the raw BLS signature — the confidential derived secret — exposed in plaintext.

The `ckd()` entry point performs several structural checks (participant count, duplicates, coordinator membership) but performs **no validation whatsoever on `app_pk`**:

```rust
pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,          // ← accepted without any check
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    if participants.len() < 2 { ... }
    // kick out duplicates
    // ensure my presence
    // ensure coordinator is participant
    // NO check on app_pk
    ...
}
```

This is the direct analog of the Axis Finance H-8 finding: just as `ECIES.isValid()` checked that a point was on the curve but omitted the field-modulus bound check, `ckd()` checks structural protocol parameters but omits the required point validity check on the externally supplied cryptographic input.

---

### Impact Explanation

**Critical — Extraction or disclosure of confidential derived secrets.**

When `app_pk = identity` is submitted:

- `big_c` in the published `CKDOutput` equals `msk · H(pk, app_id)` — the BLS signature that is the sole secret output of the CKD protocol.
- This value is broadcast on-chain as part of the protocol response and is visible to all observers.
- Any party can then compute `s = HKDF(big_c)` to obtain the derived application secret, which the protocol guarantees should be known only to the requesting TEE app.
- The security requirement stated in the spec is violated: *"s must be deterministic as a function of app_id and only known by app"*.

---

### Likelihood Explanation

The `app_pk` is a caller-supplied parameter. The `ckd()` function is a public API. The identity point `G1Projective::identity()` is a valid Rust value trivially constructable by any caller. No special privileges, leaked keys, or cryptographic breaks are required. Any entity that can submit a CKD request (i.e., any TEE app or anyone who can call the MPC contract) can trigger this.

---

### Recommendation

Add an identity-point check on `app_pk` at the entry point of `ckd()` in `src/confidential_key_derivation/protocol.rs`, before the protocol is launched:

```rust
pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError> {
    // Reject the identity point: using it as ElGamal public key
    // collapses the ciphertext to the raw BLS signature share.
    if bool::from(app_pk.is_identity()) {
        return Err(InitializationError::BadParameters(
            "app_pk must not be the identity element".to_string(),
        ));
    }
    // ... existing checks ...
}
```

    // H(pk || app_id) when H is a random oracle
