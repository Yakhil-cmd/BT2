### Title
`TurnBridge` Handler Is a No-Op Stub with No Permission Check or State Write — (`x/cronos/keeper/msg_server.go`)

### Summary

`msgServer.TurnBridge` is an unimplemented stub that unconditionally returns `(nil, nil)`. It performs no authorization check and writes no state. Any address can submit `MsgTurnBridge` and receive a success response, while the gravity bridge `bridge_active` parameter is never modified. The circuit-breaker for the gravity bridge is permanently non-functional.

### Finding Description

`UpdateTokenMapping` — the sibling handler — correctly gates execution behind `k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping)`: [1](#0-0) 

`TurnBridge`, by contrast, is a bare stub: [2](#0-1) 

`ValidateBasic` only checks that `Sender` is a valid bech32 address — no permission gate exists anywhere in the call path: [3](#0-2) 

The permission infrastructure (`CanTurnBridge`, `HasPermission`) is fully implemented and used elsewhere: [4](#0-3) 

ADR-009 explicitly states that `MsgTurnBridge` must be restricted to addresses with `CanTurnBridge` permission or the admin, and that the handler must alter bridge state: [5](#0-4) 

The gravity bridge `bridge_active` parameter that `TurnBridge` is supposed to toggle is a real, queryable chain parameter referenced in integration tests: [6](#0-5) 

### Impact Explanation

Two distinct impacts:

1. **Authorization bypass (High)**: Any unprivileged address can submit `MsgTurnBridge` with any `Sender` value and receive a `code: 0` success response. The `CanTurnBridge` permission check is entirely absent.

2. **Permanent circuit-breaker failure (High)**: Because the handler writes no state, the gravity bridge `bridge_active` flag can never be toggled off via this message. In an emergency (e.g., a bridge exploit), the intended circuit-breaker is permanently non-functional regardless of who calls it — including the admin.

Both map directly to the allowed High impact categories: bypass of bridge authorization checks, and permanent inability for honest users/validators to execute bridge-control flows.

### Likelihood Explanation

The message is registered, routed, and reachable via a standard signed transaction. No special preconditions are required beyond having a valid bech32 address and gas. The stub is not guarded by any feature flag or ante-handler check.

### Recommendation

Implement `TurnBridge` to mirror `UpdateTokenMapping`:

1. Check `k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge)` and return `sdkerrors.ErrUnauthorized` on failure.
2. Call the gravity keeper to set `bridge_active` to `msg.Enable` (equivalent to what `SetParams` does for the gravity module's `BridgeActive` field).
3. Emit an appropriate event.

```go
// Unit test (no network required)
func TestTurnBridgeStub(t *testing.T) {
    // Setup keeper with unprivileged sender (no CanTurnBridge permission, not admin)
    ctx, k := setupKeeper(t)
    srv := keeper.NewMsgServerImpl(k)

    unprivileged := sdk.AccAddress([]byte("random_address_____"))
    msg := types.NewMsgTurnBridge(unprivileged.String(), false)

    // 1. ValidateBasic passes for any valid bech32 address
    require.NoError(t, msg.ValidateBasic())

    // 2. Handler returns (nil, nil) — no error, no state written
    resp, err := srv.TurnBridge(sdk.WrapSDKContext(ctx), msg)
    require.NoError(t, err)   // PASSES — authorization bypass confirmed
    require.Nil(t, resp)      // stub returns nil response

    // 3. bridge_active is unchanged — circuit-breaker is non-functional
    // (query gravity params and assert bridge_active == true still)
}
```

The differential test: `UpdateTokenMapping` with the same unprivileged sender returns `ErrUnauthorized`; `TurnBridge` returns `nil`. The asymmetry proves the missing guard.

```go
func (k msgServer) UpdateTokenMapping(goCtx context.Context, msg *types.MsgUpdateTokenMapping) (*types.MsgUpdateTokenMappingResponse, error) {

		return nil, err
	}
	return &types.MsgUpdateTokenMappingResponse{}, nil
}
```

```go
const (
	CanChangeTokenMapping uint64                                  = 1 << iota // 1
	CanTurnBridge                                                             // 2
	All                   = CanChangeTokenMapping | CanTurnBridge             // 3
)

func (k Keeper) SetPermissions(ctx sdk.Context, address sdk.AccAddress, permissions uint64) {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := sdk.Uint64ToBigEndian(permissions)
	store.Set(types.AdminToPermissionsKey(address), permissionsBytes)
}

func (k Keeper) GetPermissions(ctx sdk.Context, address sdk.AccAddress) uint64 {
	store := ctx.KVStore(k.storeKey)
	permissionsBytes := store.Get(types.AdminToPermissionsKey(address))
	return sdk.BigEndianToUint64(permissionsBytes)
}

// HasPermission check if an account has a specific permission. by default cronos admin has all permissions
func (k Keeper) HasPermission(ctx sdk.Context, accounts []sdk.AccAddress, permissionsToCheck uint64) bool {
	// case when no permission is needed
	if permissionsToCheck == 0 {
		return true
	}
	admin := k.GetParams(ctx).CronosAdmin
	for _, account := range accounts {
		if admin == account.String() {
			return true
		}
		permission := k.GetPermissions(ctx, account)
		if permission&permissionsToCheck == permissionsToCheck {
			return true
		}
	}

	return false
```
