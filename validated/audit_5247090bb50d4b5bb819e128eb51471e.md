### Title
Rewards Permanently Frozen in Pool Contract When Pool Balance Falls Below `min_delegation_for_rewards` - (File: src/pool/pool.cairo)

### Summary

When a delegation pool's total balance is below `min_delegation_for_rewards` (1 STRK for STRK pools), the staking contract still calculates and transfers STRK rewards to the pool contract, but the pool contract's `compute_rewards_per_unit` silently returns zero. The cumulative rewards trace is never updated, so pool members can never claim those rewards. The STRK tokens are permanently locked in the pool contract with no recovery path.

### Finding Description

In `pool.cairo`, `compute_rewards_per_unit` guards against overflow by returning zero when `total_stake < min_delegation_for_rewards`: [1](#0-0) 

The code comment at lines 964–966 explicitly acknowledges the consequence: [2](#0-1) 

However, the staking contract's `update_pool_rewards` unconditionally transfers `pool_rewards` STRK tokens to the pool contract **before** calling `update_rewards_from_staking_contract`: [3](#0-2) 

The transfer happens at line 1876–1881 (via `send_rewards_to_delegation_pool`), and only then is `update_rewards_from_staking_contract` called. Inside that function, the cumulative rewards trace is updated with `last + 0`: [4](#0-3) 

Since `claim_rewards` for pool members is computed entirely from the cumulative rewards trace: [5](#0-4) 

...the STRK tokens transferred to the pool contract are never reflected in any claimable balance. There is no admin sweep, no recovery function, and no fallback path. The tokens are permanently frozen.

The entry point is permissionless: `enter_delegation_pool` enforces only `amount.is_non_zero()` with no minimum delegation floor: [6](#0-5) 

A pool balance below `min_delegation_for_rewards` can also arise naturally after partial exits via `exit_delegation_pool_intent`.

### Impact Explanation

Every epoch in which the pool's total balance is below `min_delegation_for_rewards`, the staking contract mints and forwards a proportional share of STRK rewards to the pool contract. Those tokens accumulate in the pool contract's ERC-20 balance but are never credited to any pool member. The loss is permanent: no function in the pool contract allows recovery of tokens that were never entered into the cumulative rewards trace. This constitutes **permanent freezing of unclaimed yield**.

### Likelihood Explanation

The `min_delegation_for_rewards` for STRK pools is `10^18` (1 STRK): [7](#0-6) 

Any pool member who delegates less than 1 STRK triggers the condition immediately. Additionally, a pool that previously had a larger balance can fall below the threshold after partial undelegation. Because there is no minimum delegation floor enforced at entry, this is reachable by any unprivileged delegator.

### Recommendation

The fix should ensure that rewards are only transferred to the pool contract when they will actually be distributed. Two options:

1. **Guard the transfer**: In `update_pool_rewards` (staking.cairo), skip the `send_rewards_to_delegation_pool` call when `pool_balance < min_delegation_for_rewards`. The rewards should remain in the reward supplier or be redistributed.
2. **Track stranded rewards**: In `update_rewards_from_staking_contract` (pool.cairo), when `compute_rewards_per_unit` returns zero, accumulate the received `rewards` in a separate recoverable storage slot so they can be redistributed or returned.

### Proof of Concept

1. Pool member calls `enter_delegation_pool` with `amount = 0.5 STRK` (< 1 STRK threshold).
2. Staker attests; `update_rewards_from_attestation_contract` → `_update_rewards` is called.
3. `calculate_staker_pools_rewards` computes `pool_rewards = total_rewards × 0.5 / total_stake × (1 − commission)` — a non-zero value.
4. `update_pool_rewards` calls `send_rewards_to_delegation_pool`, transferring `pool_rewards` STRK to the pool contract.
5. `pool_dispatcher.update_rewards_from_staking_contract(rewards: pool_rewards, pool_balance: 0.5e18)` is called.
6. Inside the pool, `compute_rewards_per_unit` checks `0.5e18 < 1e18 = min_delegation_for_rewards` → returns `0`.
7. `cumulative_rewards_trace` is updated with `last + 0`; no change.
8. Pool member calls `claim_rewards` → `calculate_rewards` returns `0` for all affected epochs.
9. The transferred STRK tokens remain in the pool contract's balance indefinitely with no recovery path.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }

        fn update_pool_rewards(
            ref self: ContractState,
            staker_address: ContractAddress,
            pools_rewards_data: Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>,
        ) -> Array<(ContractAddress, Amount)> {
            let mut pool_rewards_list = array![];
            let strk_token_dispatcher = strk_token_dispatcher();
            for (pool_contract, token_address, pool_balance, pool_rewards) in pools_rewards_data {
                let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };
                // Rewards are always in STRK.
                self
                    .send_rewards_to_delegation_pool(
                        :staker_address,
                        pool_address: pool_contract,
                        amount: pool_rewards,
                        token_dispatcher: strk_token_dispatcher,
                    );
                let decimals = self.get_token_decimals(:token_address);
                pool_dispatcher
                    .update_rewards_from_staking_contract(
                        rewards: pool_rewards,
                        pool_balance: pool_balance.to_native_amount(:decimals),
                    );
                pool_rewards_list.append((pool_contract, pool_rewards));
            }
            pool_rewards_list
        }
