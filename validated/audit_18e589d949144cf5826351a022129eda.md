### Title
Threshold Minimum Bypass via Direct `do_keygen` Call Produces Degree-0 Polynomial Giving Every Participant the Full Master Secret - (File: src/dkg.rs)

---

### Summary

`do_keygen` and `do_reshare` are `pub` functions in `src/dkg.rs` that can be called directly by a library caller, bypassing the validation enforced by `assert_key_invariants` and `assert_reshare_keys_invariants`. The most critical bypass is invoking `do_keygen` with `threshold = 1`, which generates a degree-0 (constant) polynomial. In this configuration every participant's final signing share equals the master secret, so any single participant can produce a valid threshold signature alone — completely defeating the threshold security guarantee.

---

### Finding Description

The library exposes two layers of the DKG API in `src/dkg.rs`:

**Validation layer (intended gate):** [1](#0-0) 

`assert_key_invariants` enforces `threshold >= 2`: [2](#0-1) 

**Protocol layer (bypasses the gate):** [3](#0-2) 

`do_keygen` is `pub async fn` and calls `do_keyshare` directly. It never calls `assert_key_invariants`, so the `threshold >= 2` invariant is never enforced on this path.

Inside `do_keyshare`, the polynomial degree is computed as: [4](#0-3) 

With `threshold = 1`, `degree = 0`. A degree-0 polynomial is a constant equal to the participant's secret `s_i`. Every participant i sends `f_i(j) = s_i` to every other participant j. Each participant j accumulates:

```
share_j = f_j(j) + Σ_{i≠j} f_i(j) = s_j + Σ_{i≠j} s_i = Σ_i s_i = S  (master secret)
``` [5](#0-4) 

Every participant ends up holding `S` as their signing share. The internal `assert_keyshare_inputs` only rejects a zero secret; it does not check the threshold value: [6](#0-5) 

The same bypass applies to `do_reshare`, which is also `pub` and does not call `assert_reshare_keys_invariants`: [7](#0-6) 

---

### Impact Explanation

With `threshold = 1` and a degree-0 polynomial, every participant's signing share equals the master secret `S`. During the OT-based ECDSA signing phase, the Lagrange coefficient for a single-participant signing set is `λ = 1`, so the participant's share is used directly as the full secret. The participant can produce a valid, verifiable ECDSA signature alone without cooperation from any other party: [8](#0-7) 

This maps to the **Critical** allowed impact: *Unauthorized creation of a valid threshold signature for attacker-chosen inputs* — any single participant can sign arbitrary messages unilaterally, breaking the core threshold security property.

---

### Likelihood Explanation

`do_keygen` is a `pub async fn` in `src/dkg.rs`. Any library caller who constructs a `ParticipantList` and a `SharedChannel` can invoke it directly with `threshold = 1`, bypassing `assert_key_invariants`. No special privileges, leaked keys, or cryptographic breaks are required. The caller only needs to be a participant in the protocol setup.

---

### Recommendation

Either:
1. Make `do_keygen` and `do_reshare` `pub(crate)` so they are not part of the public API surface, forcing all callers through the validated `keygen`/`reshare` wrappers in `lib.rs`; or
2. Move the threshold and participant invariant checks from `assert_key_invariants` into `do_keyshare` itself, so they are enforced regardless of which entry point is used.

The validation in `assert_key_invariants` should not be a separate opt-in step that callers can skip.

---

pub fn assert_key_invariants(
    participants: &[Participant],
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
) -> Result<ParticipantList, InitializationError> {
    let threshold = usize::from(threshold.into());
    // need enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // Step 1.1
    // validate threshold
    if threshold > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold,
            max: participants.len(),
        });
    }
    // Step 1.1
    if threshold < 2 {
        return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
    }

    // ensure uniqueness of participants in the participant list
    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }
    Ok(participants)
}

pub async fn do_reshare<C: Ciphersuite>(
    chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    old_signing_key: Option<SigningShare<C>>,
    old_public_key: VerifyingKey<C>,
    old_participants: ParticipantList,
    mut rng: impl CryptoRngCore,
) -> Result<KeygenOutput<C>, ProtocolError> {
    let threshold = threshold.into();
    let intersection = old_participants.intersection(&participants);
    // either extract the share and linearize it or set it to zero
    let secret = old_signing_key
        .map(|x_i| {
            intersection
                .lagrange::<C>(me)
                .map(|lambda| lambda * x_i.to_scalar())
        })
        .transpose()?
        .unwrap_or_else(<C::Group as Group>::Field::zero);

    let old_reshare_package = Some((old_public_key, old_participants));
    let keygen_output = do_keyshare::<C>(
        chan,
        participants,
        me,
        threshold,
        secret,
        old_reshare_package,
        &mut rng,
    )
    .await?;

    Ok(keygen_output)
}

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    presignature: &RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<Scalar, ProtocolError> {
    // Round 1
    // Linearize ki
    // Spec 1.1
    let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
    let k_i = lambda * presignature.k;

    // Linearize sigmai
    // Spec 1.2
    let sigma_i = lambda * presignature.sigma;

    // Compute si = h * ki + Rx * sigmai
    // Spec 1.3
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)
}
