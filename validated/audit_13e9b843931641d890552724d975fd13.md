### Title
Unrestricted `update_rewards` Allows Any Caller to Suppress Block Reward Distribution via `disable_rewards=true` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Starknet Staking contract is intended to be callable only by the Starkware sequencer (per spec), but the implementation contains no caller restriction. Any unprivileged address can call `update_rewards(staker_address, disable_rewards=true)`, which advances the global `last_reward_block` checkpoint without distributing rewards. Because `last_reward_block` is a single global variable, this blocks the sequencer's legitimate reward-distribution call for that entire block, causing all stakers to lose their block rewards.

### Finding Description
The spec for `update_rewards` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation at `src/staking/staking.cairo` contains no `get_caller_address()` check and no sequencer-address assertion:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks pause state
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    self.last_reward_block.write(current_block_number);   // global checkpoint advanced

    if disable_rewards || self.is_pre_consensus() {
        return;                            // exits without distributing rewards
    }
    // ... reward distribution logic
``` [1](#0-0) 

The `last_reward_block` storage variable is global (not per-staker). Writing it to the current block number with `disable_rewards=true` causes any subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`, including the sequencer's legitimate call. [2](#0-1) 

The spec confirms the intended access restriction that is absent from the code: [3](#0-2) 

### Impact Explanation
An attacker calling `update_rewards(any_valid_staker, disable_rewards=true)` in block N:
1. Advances `last_reward_block` to N without distributing any rewards.
2. The sequencer's subsequent call in block N reverts with `REWARDS_ALREADY_UPDATED`.
3. All stakers and all delegation pools receive zero block rewards for block N.

Repeated every block, this permanently suppresses all consensus-phase reward accrual for every staker and delegator in the protocol. This matches **High: Permanent freezing of unclaimed yield** (rewards that would have accrued are never created) or at minimum **Medium: Griefing with no profit motive but damage to users or protocol**.

### Likelihood Explanation
- The function is publicly callable with no access control.
- The attacker needs only to submit one transaction per block with a valid (existing) `staker_address` and `disable_rewards=true`.
- On Starknet, gas costs are low, making sustained griefing economically feasible.
- No privileged key, bridge compromise, or external dependency is required.
- The attack is confirmed by test code that calls `update_rewards` without any `cheat_caller_address_once` setup, demonstrating any address succeeds. [4](#0-3) 

### Recommendation
Add an explicit sequencer-address check at the top of `update_rewards`, analogous to the pattern used in `update_rewards_from_attestation_contract` and `update_current_epoch_block_rewards`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
```

Alternatively, store and validate the sequencer address at construction time and enforce it here, mirroring the `CALLER_IS_NOT_ATTESTATION_CONTRACT` guard used in `update_rewards_from_attestation_contract`. [5](#0-4) 

### Proof of Concept
1. Deploy the system with consensus rewards active.
2. Advance to a block where a staker has non-zero balance (past K-delay).
3. From any unprivileged address, call:
   ```
   staking.update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
   ```
4. Observe that `last_reward_block` is now set to the current block and no rewards were distributed.
5. The sequencer's legitimate call `update_rewards(staker_address, disable_rewards=false)` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeat every block: all stakers receive zero block rewards indefinitely. [6](#0-5) [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
