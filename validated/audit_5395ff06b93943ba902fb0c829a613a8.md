### Title
Signature Bypass via `skip_stateful_validations` Allows Invalid Invoke Transactions into Mempool - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful validator unconditionally skips the `__validate__` entry-point call (which performs signature verification) for any Invoke transaction with `nonce == 1` when the sender account has **any** transaction present in the mempool or a recent block. An attacker who observes a victim's `deploy_account` transaction in the public mempool can submit a crafted Invoke with `nonce == 1` for the victim's address carrying an **invalid signature** and a higher fee. The gateway accepts it without ever calling `__validate__`, replacing the victim's legitimate Invoke via fee-escalation. The victim's first post-deploy Invoke is permanently displaced from the mempool.

### Finding Description

The gateway stateful validation path is:

```
extract_state_nonce_and_run_validations
  └─ run_pre_validation_checks
       ├─ validate_state_preconditions   (nonce range, resource bounds)
       ├─ validate_by_mempool            (duplicate hash / fee-escalation check)
       └─ skip_stateful_validations      ← decides whether __validate__ is called
```

`skip_stateful_validations` returns `true` (skip) when all three conditions hold:

The mempool's `validate_tx` (called inside `validate_by_mempool`) permits fee-escalation replacement of an existing same-nonce transaction:

```rust
// crates/apollo_mempool/src/mempool.rs  lines 402-408
pub fn validate_tx(&mut self, args: ValidationArgs) -> MempoolResult<()> {
    let tx_reference = (&args).into();
    self.validate_incoming_tx(tx_reference, args.account_nonce)?;
    self.validate_fee_escalation(tx_reference)?;
    Ok(())
}
``` [5](#0-4) 

Because `validate_by_mempool` runs **before** `skip_stateful_validations`, a transaction that passes fee-escalation is then handed to the skip-check, which returns `true` (skip `__validate__`) because the account already has a `deploy_account` in the pool. The invalid-signature Invoke is admitted.

### Impact Explanation

An attacker can inject an Invoke transaction carrying an **arbitrary (invalid) signature** for any not-yet-deployed account whose `deploy_account` is visible in the mempool. The gateway never calls `__validate__` on it. Via fee-escalation the attacker's transaction displaces the victim's legitimate Invoke. When the batcher later executes the block, `__validate__` is called and fails; the victim's first post-deploy Invoke is lost and must be resubmitted. This matches the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."*

### Likelihood Explanation

The mempool is public. Any observer can detect a `deploy_account` for a fresh address and race to submit a higher-fee Invoke with `nonce == 1` before the block is sealed. No privileged access is required. The window is the time between the victim's `deploy_account` entering the mempool and the next block commit.

### Recommendation

Replace the coarse `account_tx_in_pool_or_recent_block` check with a precise check that the mempool contains a **`deploy_account` transaction specifically** for the sender address. Alternatively, do not skip `__validate__` entirely; instead run it against a synthetic "empty" account state so that the signature is still verified even before the account is deployed (as the legacy `PyValidator.should_run_stateful_validations` does with `max_nonce_for_validation_skip`). [6](#0-5) 

### Proof of Concept

1. **Victim** submits `deploy_account` (nonce 0) and `invoke` (nonce 1, valid signature) for address `A` to the gateway. Both are accepted; `A` appears in `tx_pool`.

2. **Attacker** observes `A` in the mempool. Attacker crafts `invoke_bad` for address `A`, `nonce = 1`, **invalid signature**, `tip` higher than victim's Invoke.

3. Attacker submits `invoke_bad` to the gateway:
   - `validate_state_preconditions`: `nonce(1) >= account_nonce(0)` → passes.
   - `validate_by_mempool` → `validate_fee_escalation`: attacker's tip is higher → replacement allowed, victim's Invoke is marked for removal.
   - `skip_stateful_validations`: `tx.nonce()==1 && account_nonce==0 && account_tx_in_pool_or_recent_block(A)==true` → returns `true`.
   - `run_validate_entry_point`: `validate=false` → `__validate__` **never called**.
   - `invoke_bad` is inserted into the mempool, victim's Invoke is evicted.

4. Batcher seals a block: `deploy_account` executes (account `A` is deployed). `invoke_bad` executes: `__validate__` is called, fails (invalid signature). Victim's legitimate Invoke is gone.

5. Victim must detect the failure and resubmit, while the attacker can repeat the attack for every new `deploy_account` they observe.

/// Check if validation of an invoke transaction should be skipped due to deploy_account not being
/// processed yet. This feature is used to improve UX for users sending deploy_account + invoke at
/// once.
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
