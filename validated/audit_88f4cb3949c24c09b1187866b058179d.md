### Title
Gateway L2 Gas Price Threshold Rounded Down (Floor) Instead of Up (Ceiling), Admitting Transactions Below the Intended Minimum — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum acceptable L2 gas price by multiplying the previous block's L2 gas price by `min_gas_price_percentage / 100` using `num_rational::Ratio::to_integer()`, which **truncates toward zero (floor)**. When the product is non-integer, the enforced threshold is strictly lower than the intended one, so transactions whose `max_price_per_unit` falls between the floor and the true fractional threshold are incorrectly admitted to the mempool.

### Finding Description

In `crates/apollo_gateway/src/stateful_transaction_validator.rs` lines 367–371:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ← truncates (floor), not ceiling
if tx_l2_gas_price.0 < threshold {
    return Err(...);
}
```

`Ratio::to_integer()` in `num_rational` returns the integer part of the rational number, i.e., `⌊numerator / denominator⌋`. When `min_gas_price_percentage × previous_block_l2_gas_price` is not divisible by 100, the threshold is silently rounded **down**, making the admission guard weaker than configured.

**Concrete example:**
- `previous_block_l2_gas_price = 10_000_000_001` FRI, `min_gas_price_percentage = 80`
- True threshold = `8_000_000_000.8` FRI
- Computed threshold = `8_000_000_000` FRI (floor)
- A transaction with `max_price_per_unit = 8_000_000_000` passes the check, even though it is below the intended 80 % floor.

The analogous correct direction — as in the EIP-4626 pattern — is to round the threshold **up** (ceiling), so the protocol is never shortchanged: any transaction that cannot cover the full fractional threshold should be rejected.

### Impact Explanation

The gateway stateful admission check is the first line of defence against under-priced transactions entering the mempool. With floor rounding, transactions whose L2 gas price is up to 1 FRI below the intended minimum are admitted. These transactions consume mempool capacity and batcher processing cycles. Under the allowed impact scope this is a **High** finding: "Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."

### Likelihood Explanation

The rounding error is triggered whenever `min_gas_price_percentage × previous_block_l2_gas_price` is not divisible by 100. Given that `previous_block_l2_gas_price` is a continuously varying EIP-1559 value (e.g., `30_312_500_000` FRI in the test suite) and `min_gas_price_percentage` is typically 80–100, non-divisibility is the common case. Any user who inspects the previous block's gas price and submits a transaction at exactly the floor value can reliably bypass the intended threshold.

### Recommendation

Replace `.to_integer()` with ceiling division. Using `num_rational::Ratio`:

```rust
// Before (floor):
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0)
    .to_integer();

// After (ceiling):
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

`Ratio::ceil()` returns the smallest integer ≥ the rational value, ensuring the enforced threshold is never lower than the configured percentage of the previous block price.

### Proof of Concept

```
previous_block_l2_gas_price = 101 FRI
min_gas_price_percentage    = 10

Ratio = 10/100 * 101 = 1010/100 = 10.1

to_integer()  → 10   (floor  — current code)
ceil()        → 11   (ceiling — correct)

Transaction with max_price_per_unit = 10:
  Current code:  10 < 10  → false → ADMITTED  ← wrong
  Fixed code:    10 < 11  → true  → REJECTED   ← correct
