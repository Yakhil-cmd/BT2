### Title
Unprivileged Caller Can Permanently Freeze Staker Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
Any unprivileged address can call `Staking::update_rewards()` with `disable_rewards: true`, consuming the single per-block global reward slot (`last_reward_block`) without distributing any rewards. Because only one `update_rewards` call can succeed per block, an attacker calling this every block prevents all stakers from ever accumulating block rewards — at zero cost beyond gas fees.

### Finding Description
`update_rewards` is a public function in `IStakingRewardsManager` with no access control beyond `general_prerequisites()`, which only checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

The function enforces a global, single-slot per-block guard: [2](#0-1) 

`last_reward_block` is a **single global storage variable** shared across all stakers: [3](#0-2) 

After the guard passes, `last_reward_block` is immediately written to the current block: [4](#0-3) 

If `disable_rewards` is `true`, the function returns immediately after writing `last_reward_block`, distributing zero rewards: [5](#0-4) 

Because `last_reward_block` is now set to the current block, every subsequent call to `update_rewards` in the same block will fail the guard check. No staker can receive block rewards for that block.

The only prerequisite for the attacker is to supply a valid, active staker address — trivially obtained from on-chain events (e.g., `NewStaker`). [6](#0-5) 

### Impact Explanation
If an attacker calls `update_rewards(any_valid_staker, disable_rewards: true)` once per block, `last_reward_block` is consumed every block with zero reward distribution. All stakers permanently lose their block rewards. This constitutes **permanent freezing of unclaimed yield** for all stakers in the protocol.

The `_update_rewards` path — which calls `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` and distributes STRK/BTC rewards — is never reached: [7](#0-6) 

### Likelihood Explanation
The attack requires:
1. Any non-zero address (no special role)
2. Any valid active staker address (publicly observable on-chain)
3. One transaction per block (minimal gas cost on Starknet)

There is no economic barrier. The attacker gains nothing but inflicts total loss of block rewards on all stakers. The attack is sustainable indefinitely.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the attestation contract, a designated sequencer address, or a role-gated caller). Alternatively, remove the `disable_rewards` parameter from the public interface and handle that logic internally via a privileged path. At minimum, add a role check such as `self.roles.only_operator()` before allowing `disable_rewards: true`.

### Proof of Concept
1. Observe any active staker address `S` from on-chain `NewStaker` events.
2. Each block, submit a transaction calling:
   ```
   Staking::update_rewards(staker_address: S, disable_rewards: true)
   ```
3. The call passes `general_prerequisites()` (contract not paused, caller non-zero).
4. `last_reward_block` is written to the current block number.
5. The function returns early — zero rewards distributed.
6. Any legitimate `update_rewards` call in the same block fails with `REWARDS_ALREADY_UPDATED`.
7. Repeat every block. All stakers accumulate zero block rewards indefinitely. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
