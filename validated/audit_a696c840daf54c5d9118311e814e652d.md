### Title
Missing Caller Authorization in `update_rewards` Allows Any Address to Permanently Block Block Rewards - (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` is documented in the protocol specification as callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` to unconditionally advance the global `last_reward_block` sentinel without distributing rewards, permanently blocking the legitimate sequencer from issuing rewards for that block.

---

### Finding Description

The `IStakingRewardsManager` interface exposes a single write function:

```rust
fn update_rewards(
    ref self: TContractState, staker_address: ContractAddress, disable_rewards: bool,
);
``` [1](#0-0) 

The protocol specification explicitly states the access control for this function is **"Only starkware sequencer"**: [2](#0-1) 

However, the implementation in `StakingRewardsManagerImpl` performs no such check. After `general_prerequisites()` (which only checks the pause flag), the function unconditionally writes the current block number to the global `last_reward_block` storage variable **before** evaluating `disable_rewards`:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);  // ← written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;  // ← exits without distributing any rewards
    }
    // ... reward distribution follows
``` [3](#0-2) 

Because `last_reward_block` is a **global** (not per-staker) sentinel, a single call with `disable_rewards: true` for any valid `staker_address` marks the entire block as "already processed." Any subsequent call by the legitimate sequencer for the same block reverts with `REWARDS_ALREADY_UPDATED`. [4](#0-3) 

There is no `assert_caller_is_sequencer` or equivalent guard anywhere in the staking contract — a grep for `sequencer` across all production Cairo sources returns zero matches.

---

### Impact Explanation

An attacker who front-runs the sequencer's `update_rewards` call every block with `disable_rewards: true` causes every block's rewards to be permanently lost for **all** stakers and their delegation pool members. The `last_reward_block` write is irreversible for that block height; there is no recovery path. This constitutes **permanent freezing of unclaimed yield** for the entire protocol.

Impact classification: **High — Permanent freezing of unclaimed yield / unclaimed royalties**.

---

### Likelihood Explanation

The function is publicly callable with no authentication barrier. On Starknet, transaction ordering within a block is deterministic and observable in the mempool. An attacker needs only to submit a call to `update_rewards(any_active_staker, disable_rewards: true)` at the start of each block before the sequencer's own call. The cost is a single cheap transaction per block. The attack is fully automatable and requires no privileged access, no capital, and no external dependencies.

---

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`, consistent with the specification's stated access control. For example:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == starknet::get_sequencer_address(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ...
}
```

Alternatively, restrict the interface so that only a designated, governance-controlled address (the sequencer operator) may invoke it, mirroring the pattern already used for `update_rewards_from_attestation_contract`, which correctly enforces `assert_caller_is_attestation_contract`. [5](#0-4) 

---

### Proof of Concept

```
// Attacker (any address) front-runs the sequencer each block:
//
// 1. Consensus rewards are active (post-consensus epoch).
// 2. Attacker observes a new block N in the mempool.
// 3. Attacker submits (with higher gas / earlier in block):
//      staking_rewards_manager.update_rewards(
//          staker_address: <any_valid_staker>,
//          disable_rewards: true
//      )
//    → last_reward_block is written to N; no rewards distributed.
//
// 4. Sequencer submits its legitimate update_rewards call for block N.
//    → Reverts with REWARDS_ALREADY_UPDATED because
//      current_block_number (N) == last_reward_block (N).
//
// 5. Block N's rewards are permanently lost for all stakers and
//    all delegation pool members.
//
// Repeat every block for a sustained, protocol-wide reward freeze.
```

The existing test suite confirms the absence of any caller restriction — `update_rewards` is called throughout the tests with no `cheat_caller_address_once` spoofing, demonstrating that any address succeeds: [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
