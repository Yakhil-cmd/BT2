### Title
Missing Validation of `app_pk` Allows ElGamal Encryption Bypass, Exposing Confidential Derived Secret in Plaintext - (File: src/confidential_key_derivation/protocol.rs)

### Summary
The CKD protocol's `ckd()` entry point accepts an `app_pk` (the TEE app's ElGamal public key) without validating that it is a non-identity G1 point. When `app_pk` is the identity element (`G1Projective::identity()`), the ElGamal blinding term `y_i * A` collapses to zero for every participant, causing `CKDOutput.big_c` to equal the unblinded BLS signature `msk * H(pk || app_id)` — the confidential derived secret `s` — in plaintext. This value is returned as part of the protocol output and, in the intended deployment, is published on-chain, making `s` visible to any observer.

### Finding Description
In `compute_signature_share` (`src/confidential_key_derivation/protocol.rs`, lines 148–182), each participant computes:

```
big_s  = x_i * H(pk || app_id)          // BLS signature share
big_c  = big_s + y_i * app_pk           // ElGamal ciphertext
``` [1](#0-0) 

When `app_pk = G1Projective::identity()`, the term `y_i * app_pk` evaluates to the identity for every participant, so `big_c = big_s`. After Lagrange normalization and coordinator aggregation:

```
C = Σ λ_i * x_i * H(pk || app_id) = msk * H(pk || app_id) = s
```

The `CKDOutput { big_y, big_c }` returned by the coordinator therefore contains `s` in `big_c` with no blinding. The `unmask(0)` call by the app recovers `s` correctly, but `s` is now readable by any party that observes the output.

The `ckd()` constructor validates participant list membership and coordinator presence, but performs **no check** that `app_pk` is a valid non-identity curve point: [2](#0-1) 

The `CKDOutput` type exposes `big_c` via a public accessor and is `Serialize`/`Deserialize`, making it trivially readable once published: [3](#0-2) 

The protocol specification states that `s` must be "only known by app" and that "the secret itself is never revealed to any entity other than the application itself, not even to individual MPC nodes": [4](#0-3) 

### Impact Explanation
When `app_pk` is the identity element, the confidential derived secret `s = msk * H(pk || app_id)` is disclosed in plaintext inside `CKDOutput.big_c`. In the documented deployment model, this output is published on-chain and relayed back to the app through the MPC contract. Any on-chain observer — including the operator, who is explicitly untrusted — can read `s` directly from the transaction data. This violates the core confidentiality requirement of CKD and maps to the **Critical** impact: *disclosure of confidential derived secrets*.

### Likelihood Explanation
The `app_pk` parameter is a caller-supplied `ElementG1` (`blstrs::G1Projective`). A developer who zero-initializes the key pair, uses a default value, or makes an error in key generation before calling `ckd()` will silently produce an unprotected output. The library provides no guard, no error, and no warning. The analog in the external report — users clicking "skip" on encryption without understanding the consequences — maps directly: the library silently accepts a degenerate key and produces output that appears structurally valid but is cryptographically unprotected.

### Recommendation
- **Short term**: Add a validation check in `ckd()` that rejects `app_pk` equal to `G1Projective::identity()` (and optionally checks that the point is on the curve), returning an `InitializationError::BadParameters` before the protocol begins.
- **Long term**: Consider requiring a proof of knowledge of the discrete logarithm of `app_pk` (i.e., that the caller knows `a` such that `A = a * G1`) to ensure the app can actually unmask the output and that the key was generated honestly.

use threshold_signatures::confidential_key_derivation::{protocol::ckd, AppId};

// app_pk is the identity element — ElGamal blinding is zero

// ckd() accepts this without error

//

    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,
    rng: impl CryptoRngCore + Send + 'static,

    // not enough participants
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    // kick out duplicates
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}

pub struct CKDOutput {
    big_y: ElementG1,
    big_c: ElementG1,
}

impl CKDOutput {
    pub fn new(big_y: ElementG1, big_c: ElementG1) -> Self {
        Self { big_y, big_c }
    }

    /// Outputs `big_y`
    pub fn big_y(&self) -> ElementG1 {
        self.big_y
    }

    /// Outputs `big_c`
    pub fn big_c(&self) -> ElementG1 {
        self.big_c
    }

    /// Takes a secret scalar and returns
    /// s <- C − a ⋅ Y = msk ⋅ H ( `app_id` )
    pub fn unmask(&self, secret_scalar: Scalar) -> Signature {
        self.big_c - self.big_y * secret_scalar
    }
