### Title
Any Contract Can Overwrite Another Account's `AccountId`-Mode Global Contract and Drain Its Balance — (`File: runtime/runtime/src/global_contracts.rs`)

### Summary

`action_deploy_global_contract` contains no check that the receipt's `predecessor_id` equals the receipt's `receiver_id` (i.e., no self-only guard). Because any NEAR contract can create a cross-contract promise to an arbitrary existing account and append `DeployGlobalContractAction` with `GlobalContractDeployMode::AccountId`, an unprivileged attacker can: (1) drain the victim account's balance by the storage cost, and (2) overwrite the global-contract code stored under the victim's account ID — corrupting the code identity for every account that has opted into `GlobalContractIdentifier::AccountId(victim)`.

### Finding Description

`GlobalContractDeployMode::AccountId` is documented as storing the contract "under the owner account id" so that "the owner" can update it for all users. The runtime enforces this ownership nowhere.

**Call path:**

1. Attacker deploys a malicious WASM contract to their own account.
2. Attacker calls a function on that contract. Inside the function the contract calls the host function `promise_batch_create(victim_account_id)` to create a receipt targeting the victim, then calls `promise_batch_action_deploy_global_contract_by_account_id(promise_idx, code_len, code_ptr)` to append a `DeployGlobalContractAction { deploy_mode: AccountId }` carrying attacker-chosen WASM.
3. The resulting `ActionReceipt` is routed to the victim's shard and processed by `apply_action_receipt`.
4. `action_deploy_global_contract` executes with `account_id = victim`, `account = victim's Account`:

```rust
// runtime/runtime/src/global_contracts.rs  lines 39-58
let Some(updated_balance) = account.amount().checked_sub(storage_cost) else { … };
account.set_amount(updated_balance);          // victim's balance drained

initiate_distribution(
    state_update,
    account_id.clone(),                       // victim's AccountId used as identifier
    deploy_contract.code.clone(),             // attacker's code
    &deploy_contract.deploy_mode,
    apply_state.shard_id,
    result,
)?;
```

5. `initiate_distribution` builds `GlobalContractIdentifier::AccountId(victim)` and calls `increment_nonce`, then emits a `GlobalContractDistributionReceipt` that propagates attacker's WASM to every shard under the key `TrieKey::GlobalContractCode { identifier: AccountId(victim) }`.

**Existing guards and why they fail:**

- `validate_deploy_global_contract_action` checks only code size.
- `check_and_update_nonce` accepts any nonce `>= stored_nonce`; the attacker's freshly incremented nonce always satisfies this.
- There is no `predecessor_id == receiver_id` (self-only) check anywhere in the action handler or the receipt validator.

### Impact Explanation

- **Balance**: `account.amount()` of the victim is permanently reduced by `global_contract_storage_amount_per_byte × code.len()` without the victim's consent.
- **Code identity**: `TrieKey::GlobalContractCode { identifier: AccountId(victim) }` is overwritten on every shard with attacker-controlled WASM. Every account that previously called `UseGlobalContract(AccountId(victim))` now executes the attacker's code on every subsequent function call — enabling theft of attached deposits, storage manipulation, or any other malicious action the attacker encodes.

### Likelihood Explanation

Any unprivileged NEAR account can deploy a contract and trigger this attack with a single transaction. No special key, validator role, or governance privilege is required. The only precondition is that the victim account has sufficient balance to cover the storage cost (otherwise the action fails with `LackBalanceForState`, causing no harm). Accounts that have published a widely-used global contract are high-value targets.

### Recommendation

Add a self-only guard in `action_deploy_global_contract` (or in `validate_deploy_global_contract_action` for the `AccountId` mode) that rejects the action when `predecessor_id != receiver_id`:

```rust
if deploy_contract.deploy_mode == GlobalContractDeployMode::AccountId
    && receipt.predecessor_id() != account_id
{
    result.result = Err(ActionErrorKind::ActorNoPermission {
        account_id: account_id.clone(),
        actor_id: receipt.predecessor_id().clone(),
    }.into());
    return Ok(());
}
```

Alternatively, restrict `GlobalContractDeployMode::AccountId` at the transaction-validation layer so that a `DeployGlobalContractAction` with `AccountId` mode is only accepted when `signer_id == receiver_id`.

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

fn initiate_distribution(
    state_update: &mut TrieUpdate,
    account_id: AccountId,
    contract_code: Arc<[u8]>,
    deploy_mode: &GlobalContractDeployMode,
    current_shard_id: ShardId,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let id = match deploy_mode {
        GlobalContractDeployMode::CodeHash => {
            GlobalContractIdentifier::CodeHash(hash(&contract_code))
        }
        GlobalContractDeployMode::AccountId => {
            GlobalContractIdentifier::AccountId(account_id.clone())
        }
    };
    // Increment the nonce and write it to state immediately to prevent multiple
    // distributions with the same nonce from being initiated. This requires
    // allowing the same nonce in the freshness check when applying the
    // distribution receipt.
    let nonce = increment_nonce(state_update, &id)?;
    let distribution_receipt =
        GlobalContractDistributionReceipt::new(id, current_shard_id, vec![], contract_code, nonce);
    let distribution_receipts =
        Receipt::new_global_contract_distribution(account_id, distribution_receipt);
    // No need to set receipt_id here, it will be generated as part of apply_action_receipt
    result.new_receipts.push(distribution_receipts);
    Ok(())
}

pub enum GlobalContractDeployMode {
    /// Contract is deployed under its code hash.
    /// Users will be able reference it by that hash.
    /// This effectively makes the contract immutable.
    CodeHash,
    /// Contract is deployed under the owner account id.
    /// Users will be able reference it by that account id.
    /// This allows the owner to update the contract for all its users.
    AccountId,
}
