### Title
Unprivileged Caller Can Permanently Freeze All Stakers' Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

---

### Summary

The public `update_rewards` function in `IStakingRewardsManager` accepts a caller-controlled `disable_rewards` parameter. When called with `disable_rewards: true`, the function writes the current block number to the **global** `last_reward_block` storage variable without distributing any rewards. Because the function enforces a one-call-per-block invariant using this same global variable, any unprivileged caller can permanently destroy all stakers' block rewards for any given block at the cost of gas alone.

---

### Finding Description

`update_rewards` is exposed as a public, permissionless function:

```cairo
#[abi(embed_v0)]
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();   // only checks: not paused, caller != zero
``` [1](#0-0) 

`general_prerequisites()` imposes no role-based access control — any non-zero address may call this function with any value of `disable_rewards`.

The function enforces a global one-call-per-block invariant using a single `BlockNumber` storage slot:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [2](#0-1) 

`last_reward_block` is a **global** (not per-staker) field:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
``` [3](#0-2) 

Critically, `last_reward_block` is written **before** the `disable_rewards` guard:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [4](#0-3) 

When `disable_rewards: true` is passed:
1. `last_reward_block` is stamped with the current block number.
2. The function returns immediately — no rewards are calculated or distributed.
3. Every subsequent call to `update_rewards` at the same block number fails with `REWARDS_ALREADY_UPDATED`.

Because the slot is global, a single attacker call at block N locks out **all** stakers from receiving block rewards for block N. The lost rewards are irrecoverable.

---

### Impact Explanation

**Permanent freezing of unclaimed yield** for all stakers.

- Block rewards calculated by `calculate_block_rewards` / `_update_rewards` are never credited to `staker_info.unclaimed_rewards_own` or forwarded to delegation pools.
- The attacker can repeat this every block, continuously suppressing the entire consensus reward stream.
- Delegators also lose their share of pool rewards, since `update_pool_rewards` is never reached. [5](#0-4) 

---

### Likelihood Explanation

- **Entry path**: Any non-zero address, no tokens, no role, no prior state required.
- **Cost**: Gas only.
- **Automation**: Trivially scriptable — watch for new blocks, front-run the legitimate `update_rewards` call each block.
- **Detection difficulty**: The transaction looks like a normal `update_rewards` call; the only observable difference is `disable_rewards: true` and the absence of a `StakerRewardsUpdated` event.

---

### Recommendation

Add access control to `update_rewards` so that only a trusted caller (e.g., the consensus layer or a designated rewards-manager role) may invoke it, or split the function into two: a public zero-`disable_rewards` path and a privileged `disable_rewards` path. At minimum, the `last_reward_block` write must be moved **after** the `disable_rewards` guard so that a disabled call does not consume the block's reward slot.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. At block N, attacker calls:
   ```
   staking.update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
   ```
3. Inside the function:
   - `last_reward_block` is written to N. [6](#0-5) 
   - `disable_rewards` is `true` → early return, no rewards distributed. [7](#0-6) 
4. Any legitimate call to `update_rewards` at block N (by the staker, a relayer, or the consensus layer) hits:
   ```
   assert!(current_block_number > self.last_reward_block.read(), ...)
   ```
   and reverts with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 
5. All stakers and their delegators permanently lose block rewards for block N.
6. Attacker repeats at block N+1, N+2, … — total consensus reward stream is suppressed indefinitely.
