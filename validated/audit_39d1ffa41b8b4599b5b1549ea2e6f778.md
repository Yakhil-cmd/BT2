### Title
Stale Pending Fallback Omits `l1_data_gas_price`/`l2_gas_price` from Committed Block, Causing `estimate_message_fee` to Return Wrong Fee Estimates - (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

When the in-memory pending data is stale (its `parent_block_hash` does not match the latest committed block hash), `read_pending_data` synthesizes a `DeprecatedPendingBlock` fallback that copies only `l1_gas_price` from the committed block header. It leaves `l1_data_gas_price` and `l2_gas_price` at their `Default` value of 0. `client_pending_data_to_execution_pending_data` forwards those zeros into `ExecutionPendingData`, and `create_block_context` silently clamps them to `NonzeroGasPrice::MIN = 1`. Any call to `estimate_message_fee` (or `estimate_fee`, `simulate_transactions`) with `block_id = Tag(Pending)` during this window returns a `FeeEstimation` whose `l1_data_gas_price`, `l2_gas_price`, and `overall_fee` are computed against price = 1 instead of the real committed-block prices.

### Finding Description

**Step 1 – Stale fallback construction.**
`read_pending_data` compares `pending_data.block.parent_block_hash()` with `latest_header.block_hash`. On mismatch it builds:

```rust
PendingBlockOrDeprecated::Deprecated(DeprecatedPendingBlock {
    parent_block_hash: latest_header.block_hash,
    eth_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_wei,
    strk_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_fri,
    timestamp: ...,
    sequencer_address: ...,
    starknet_version: ...,
    ..Default::default()   // ← l1_data_gas_price and l2_gas_price are absent
})
``` [1](#0-0) 

**Step 2 – `DeprecatedPendingBlock` always returns 0 for the missing prices.**

```rust

    match self {
        // In older versions, data gas price was 0.
        PendingBlockOrDeprecated::Deprecated(_) => GasPricePerToken::default(),
        ...
    }
}
pub fn l2_gas_price(&self) -> GasPricePerToken {
    match self {
        // In older versions, L2 gas price was 0.
        PendingBlockOrDeprecated::Deprecated(_) => GasPricePerToken::default(),
        ...
    }
}
``` [2](#0-1) 

**Step 3 – `client_pending_data_to_execution_pending_data` forwards the zeros.**

```rust
l1_data_gas_price: client_pending_data.block.l1_data_gas_price(), // → {0, 0}
l2_gas_price:      client_pending_data.block.l2_gas_price(),      // → {0, 0}
``` [3](#0-2) 

**Step 4 – `create_block_context` silently clamps zeros to `NonzeroGasPrice::MIN = 1`.**

```rust
l1_data_gas_price: NonzeroGasPrice::new(l1_data_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),   // 0 → 1
l2_gas_price: NonzeroGasPrice::new(l2_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),   // 0 → 1
``` [4](#0-3) 

**Step 5 – `estimate_message_fee` uses this `BlockContext` for the L1Handler fee.**

The `FeeEstimation` is assembled directly from `block_context.block_info().gas_prices`, so the returned `l1_data_gas_price`, `l2_gas_price`, and `overall_fee` all reflect price = 1. [5](#0-4) 

The committed block header **does** carry real `l1_data_gas_price` and `l2_gas_price` values; the fallback simply never reads them.

### Impact Explanation

Any unprivileged caller of `starknet_estimateMessageFee` (or `starknet_estimateFee`, `starknet_simulateTransactions`) with `block_id = "pending"` during the natural stale window (the interval between a new block being committed to storage and the pending-sync loop writing fresh pending data) receives a `FeeEstimation` where:

- `l1_data_gas_price` = 1 wei/fri (instead of the real committed-block value, e.g. ~10⁶ wei)
- `l2_gas_price` = 1 wei/fri (instead of the real value)
- `overall_fee` = computed against those wrong prices

The response is indistinguishable from a correct estimate. Clients that use it to set resource bounds or to decide whether to submit an L1→L2 message will systematically underprice the transaction.

### Likelihood Explanation

The stale window is a normal, recurring operational condition: it occurs on every block (~every few seconds on mainnet). No attacker action is required; the condition arises automatically. Any client polling `estimate_message_fee` with `block_id=pending` has a non-trivial probability of hitting the window.

### Recommendation

In the stale fallback inside `read_pending_data`, copy `l1_data_gas_price` and `l2_gas_price` from the committed block header, or use a `PendingBlock` (not `DeprecatedPendingBlock`) so that the accessor methods return real values:

```rust
// In the stale-fallback branch, also copy:
// l1_data_gas_price: latest_header.block_header_without_hash.l1_data_gas_price,
// l2_gas_price:      latest_header.block_header_without_hash.l2_gas_price,
```

Alternatively, add a dedicated `create_block_context_from_committed_header` path that reads all six gas-price fields directly from the stored header when no real pending data is available.

### Proof of Concept

```rust
// Pseudocode – set pending data stale, call estimate_message_fee, observe wrong prices
let mut pending_data = real_pending_data.clone();
// Force stale: parent hash no longer matches latest committed block
*pending_data.block.parent_block_hash_mutable() = BlockHash(felt!("0xdeadbeef"));
*server.pending_data.write().await = pending_data;

let fee = rpc.estimate_message_fee(message, BlockId::Tag(Tag::Pending)).await.unwrap();

// l1_gas_price is correct (copied from committed block header)
assert_eq!(fee.l1_gas_price, committed_block_header.l1_gas_price.price_in_wei);
// l1_data_gas_price and l2_gas_price are wrong: NonzeroGasPrice::MIN = 1
assert_eq!(fee.l1_data_gas_price, GasPrice(1));
assert_eq!(fee.l2_gas_price,      GasPrice(1));
// overall_fee is computed with price=1, not the real data/L2 gas price
``` [6](#0-5)

```rust
pub(crate) fn tx_execution_output_to_fee_estimation(
    tx_execution_output: &TransactionExecutionOutput,
    block_context: &BlockContext,
) -> ExecutionResult<FeeEstimation> {
    let gas_prices = &block_context.block_info().gas_prices;
    let (l1_gas_price, l1_data_gas_price, l2_gas_price) = (
        gas_prices.l1_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l1_data_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l2_gas_price(&tx_execution_output.price_unit.into()).get(),
    );

    let gas_vector = tx_execution_output.execution_info.receipt.gas;

    Ok(FeeEstimation {
        gas_consumed: gas_vector.l1_gas.0.into(),
        l1_gas_price,
        data_gas_consumed: gas_vector.l1_data_gas.0.into(),
        l1_data_gas_price,
        l2_gas_price,
        overall_fee: tx_execution_output.execution_info.receipt.fee,
        unit: tx_execution_output.price_unit,
    })
}
