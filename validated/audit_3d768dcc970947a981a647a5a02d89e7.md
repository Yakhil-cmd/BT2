### Title
Signature Validation Bypass for Nonce-1 Invoke Transactions via `skip_stateful_validations` — (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The `skip_stateful_validations` function in the gateway's stateful transaction validator unconditionally skips the `__validate__` entry-point call for any invoke transaction with nonce 1 when the sender address has any transaction in the mempool or a recently committed block. An unprivileged attacker who observes a pending `deploy_account` transaction in the mempool can submit an invoke with nonce 1 and an **invalid signature** for the same address, and the gateway will admit it to the mempool without ever verifying the signature.

---

### Finding Description

The gateway stateful validation path in `extract_state_nonce_and_run_validations` calls `run_pre_validation_checks`, which in turn calls `skip_stateful_validations` to decide whether to skip the blockifier `__validate__` entry point: [1](#0-0) 

`skip_stateful_validations` returns `true` (skip validation) when all of the following hold:

1. The transaction is an `Invoke`.
2. `tx.nonce() == Nonce(Felt::ONE)` (exactly nonce 1).
3. `account_nonce == Nonce(Felt::ZERO)` (account not yet deployed on-chain).
4. `mempool_client.account_tx_in_pool_or_recent_block(sender)` returns `true`. [2](#0-1) 

When `skip_stateful_validations` returns `true`, `run_validate_entry_point` is called with `skip_validate = true`, which sets `execution_flags.validate = false`, so the `__validate__` entry point is **never called** at gateway admission time: [3](#0-2) 

The `validate_by_mempool` call that precedes `skip_stateful_validations` only checks nonce range and duplicate-hash rules via the mempool's `validate_tx`; it does **not** verify the transaction signature: [4](#0-3) 

The mempool's `account_tx_in_pool_or_recent_block` returns `true` as soon as any transaction for the address is present in the pool or the mempool's committed-block state: [5](#0-4) 

**Attack path:**

1. Alice submits a `deploy_account` (nonce 0) for address `A`. It is admitted to the mempool after full execution-based validation.
2. An attacker observes the pending `deploy_account` in the mempool. `account_tx_in_pool_or_recent_block(A)` now returns `true`.
3. The attacker submits an `Invoke` for address `A`, nonce 1, with an **arbitrary/invalid signature** and a high `max_fee`.
4. The gateway calls `validate_nonce`: nonce 1 is within `[0, 0 + max_allowed_nonce_gap(200)]`, so it passes.
5. The gateway calls `validate_by_mempool`: nonce 1 is valid for account nonce 0, no duplicate hash — passes.
6. The gateway calls `skip_stateful_validations`: nonce == 1, account_nonce == 0, account in mempool → returns `true`.
7. `run_validate_entry_point` is called with `skip_validate = true` → `__validate__` is **not called**. The transaction is admitted to the mempool.
8. If Alice also submitted a legitimate nonce-1 invoke, the attacker's transaction can replace it via fee escalation (same `(address, nonce)` slot).
9. The batcher later executes the attacker's transaction. `__validate__` is called at execution time, fails (invalid signature), and the transaction reverts. A fee is charged to Alice's account. Alice's legitimate invoke is gone from the mempool.

The `max_nonce_for_validation_skip` field in `StatefulTransactionValidatorConfig` (default `Nonce(Felt::ONE)`) is defined but is **only used** in the legacy `PyValidator` path, not in the production `skip_stateful_validations` function: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An unprivileged attacker can inject invoke transactions with invalid signatures into the mempool for any address that has a pending `deploy_account`. The invalid transaction occupies the `(address, nonce=1)` slot. Via fee escalation it can displace a legitimate user's valid invoke. The reverted execution charges a fee to the victim's account. The victim must resubmit their transaction.

---

### Likelihood Explanation

The mempool is observable. Any address with a pending `deploy_account` is a valid target. The attacker needs only to craft an invoke with nonce 1 and any bytes as the signature. No privileged access is required.

---

### Recommendation

The skip logic should be tightened so that signature verification is not bypassed for arbitrary callers:

- Require that the invoke transaction's sender address matches the deployer address of the pending `deploy_account` in the mempool (i.e., verify the deploy_account tx hash is for the same address and is genuinely pending).
- Alternatively, perform a lightweight signature-format check (e.g., verify the signature is well-formed and non-empty) even when skipping the full `__validate__` entry point.
- Consider restricting the skip to only the case where the `deploy_account` and the `invoke` arrive in the **same gateway request** (atomic bundle), rather than relying on a mempool state query.

---

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
