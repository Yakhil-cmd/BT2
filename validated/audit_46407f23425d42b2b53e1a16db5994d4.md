### Title
Unbounded `self.stakers` Vec causes O(N) gas growth in `get_stakers`, permanently DoS-able by cycling stake/unstake across unique addresses — (`src/staking/staking.cairo`)

---

### Summary

`get_stakers` iterates over every address ever appended to `self.stakers` (a `Vec<ContractAddress>` that only grows). Stakers are appended on every `stake()` call and are **never removed** from the Vec, even after `unstake_action` completes. An unprivileged attacker can inflate the Vec to an arbitrary size by cycling a single `min_stake` STRK balance through fresh addresses, making `get_stakers` permanently exceed block gas limits.

---

### Finding Description

**Root cause — append-only Vec, never pruned on removal:**

On every `stake()` call, the staker address is pushed unconditionally: [1](#0-0) 

`remove_staker()`, called from `unstake_action`, clears `staker_info`, the operational-address mapping, commission, etc., but **never touches `self.stakers`**: [2](#0-1) 

**Root cause — full-range iteration in `get_stakers`:**

`get_stakers` iterates `self.stakers.into_iter_full_range()`, which walks every slot ever written, including all historical (now-removed) stakers: [3](#0-2) 

For each slot it performs at minimum two storage reads: one to load the address and one inside `is_staker_active` to read `staker_info`. Removed stakers return `false` from `is_staker_active` and are skipped via `continue`, but the storage reads still occur and consume gas. [4](#0-3) 

**Why address reuse does not limit the attack:**

`assert_staker_address_not_reused` prevents the *same* address from staking again after `unstake_action`: [5](#0-4) 

However, the attacker can use a *different* address each cycle. The flow is:

1. Stake with address `A1` (deposit `min_stake` STRK).
2. `unstake_intent()` → wait exit window → `unstake_action()` → recover STRK.
3. Transfer STRK to address `A2`, stake again.
4. Repeat N times.

Each cycle appends one permanent entry to `self.stakers`. The attacker only needs to hold `min_stake` STRK (20,000 STRK in tests) at any one time; the total capital cost is `min_stake` plus cumulative gas fees, not `N × min_stake`. [6](#0-5) 

---

### Impact Explanation

`get_stakers` is part of `IStakingConsensus` and is called by the consensus layer to determine the active validator set for each epoch. Once the Vec grows large enough that iterating it exceeds Starknet's block gas limit, every call to `get_stakers` reverts. The consensus layer loses the ability to query the validator set, permanently breaking the consensus interface for any `epoch_id`. There is no on-chain mechanism to prune `self.stakers` or cap its size.

---

### Likelihood Explanation

The attack is cheap: only `min_stake` STRK is needed at any time. The attacker cycles the same tokens through fresh addresses. The exit wait window (default 1 week) slows the rate of inflation but does not prevent it — over months of protocol operation, even organic usage accumulates a large Vec. A motivated attacker accelerates this deliberately. The function is public with no access control. [7](#0-6) 

---

### Recommendation

1. **Remove stakers from the Vec on `unstake_action`**: Replace the `Vec` with a data structure that supports O(1) removal (e.g., a swap-and-pop pattern using a `Map<index, address>` plus a length counter), and clear the entry in `remove_staker`.
2. **Alternatively, maintain a separate "active stakers" set**: Only track currently-active stakers in the iterable structure; add on `stake()`, remove on `unstake_intent` (when the staker's balance leaves the active set).
3. **Short-term mitigation**: Add an off-chain cap or an on-chain guard that reverts `get_stakers` if `self.stakers.len()` exceeds a safe threshold, to prevent silent gas exhaustion.

---

### Proof of Concept

```
// Pseudocode — repeat for i in 0..N:
//   address_i = fresh_address(i)
//   transfer(min_stake, to: address_i)
//   stake(address_i, reward=address_i, operational=op_i, amount=min_stake)
//   advance_time(exit_wait_window + 1)
//   unstake_intent(address_i)
//   advance_time(exit_wait_window + 1)
//   unstake_action(address_i)          // recovers min_stake STRK
//   // self.stakers.len() == i+1, entry i is permanently dead
//
// After N iterations:
//   get_stakers(current_epoch)
//   // iterates N slots, each costing ≥2 storage reads
//   // gas ∝ N; at some threshold N, tx exceeds block gas limit and reverts
```

The attacker's net STRK cost is zero (tokens are recovered); the only cost is gas for N stake/unstake cycles. The damage — a permanently broken `get_stakers` — is irreversible without a contract upgrade.
