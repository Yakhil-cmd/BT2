### Title
WalletConnect `chia_createOfferForIds` Displays Understated Royalty While Executing Attacker-Inflated Royalty from `driver_dict` - (File: packages/gui/src/electron/commands/parseCommandDisplay.ts)

### Summary
In the WalletConnect command-display pipeline for `chia_wallet.create_offer_for_ids`, the confirmation dialog computes `amountWithRoyalties` using the on-chain royalty fetched via `nftGetInfo`, while the actual offer submitted to the backend uses the royalty percentage from the dapp-supplied `driver_dict`. A malicious dapp can supply an arbitrarily inflated `royalty_percentage` in `driver_dict`, causing the user to approve a transaction whose displayed cost is lower than what is actually executed.

### Finding Description

`parseCommandDisplay` handles two WalletConnect offer commands. For `chia_wallet.create_offer_for_ids`, royalty percentages are first extracted from the dapp-controlled `driver_dict` via `createOfferRoyaltyPercentages`: [1](#0-0) 

These percentages are passed as the initial `royaltyPercentage` argument into `parseWalletDeltaItem` for each NFT asset. Inside that function, however, the value is **silently overridden** by a live `nftGetInfo` call that fetches the actual on-chain royalty: [2](#0-1) 

The NFT item's (now on-chain-sourced) `royaltyPercentage` is then used by `withRoyaltyTotals` → `royaltyPercentagesForSide` to compute `amountWithRoyalties` for the fungible spending side: [3](#0-2) 

The confirmation dialog therefore shows `amountWithRoyalties` derived from the **on-chain** royalty (e.g. 2.5 %), while the actual command dispatched to the backend carries the original, unmodified `driver_dict` with the **attacker-supplied** royalty (e.g. 99.99 %): [4](#0-3) 

The backend receives the raw params (including the inflated `driver_dict`) via `window.permissionsAPI.dispatchAsPair` and uses them to construct the offer, including the royalty payment. [5](#0-4) 

### Impact Explanation

A user who approves the WalletConnect request based on the displayed `amountWithRoyalties` (reflecting the correct on-chain royalty) will have an offer created whose royalty payment is computed from the dapp-supplied `driver_dict` royalty. If that royalty is inflated, the user's offer commits to paying substantially more than the displayed total. The excess royalty is paid to the NFT creator's `royalty_address` encoded in the driver, not to the attacker directly, but the user suffers a direct, unannounced financial loss on every offer acceptance.

This fits the allowed High impact: *"WalletConnect state that causes a user to approve … the wrong … amount."*

### Likelihood Explanation

Any WalletConnect dapp that has been granted the `chia_createOfferForIds` permission can trigger this silently. The user has no way to detect the discrepancy from the confirmation UI alone, because the displayed `amountWithRoyalties` looks correct (it is derived from the real on-chain royalty). The attack requires only a paired dapp — no key compromise, no phishing beyond the initial WalletConnect pairing.

**Uncertainty**: Whether the Chia wallet backend independently validates the `driver_dict` royalty against the on-chain NFT puzzle before constructing the offer is not verifiable from the GUI code alone. If the backend enforces the on-chain royalty and rejects inflated values, the financial-loss vector is blocked at execution time, though the display discrepancy remains a spoofing surface.

### Recommendation

In `parseWalletDeltaItem`, when `nftGetInfo` overrides the driver-dict royalty for display, the same corrected value should be propagated back so that the actual command params sent to the backend also use the on-chain royalty — or, alternatively, the confirmation dialog should cross-validate the `driver_dict` royalty against `nftGetInfo` and warn (or block) when they diverge. At minimum, the raw `driver_dict` royalty percentage should be shown alongside `amountWithRoyalties` so the user can detect the mismatch.

### Proof of Concept

1. Pair a malicious WalletConnect dapp with the Chia GUI wallet.
2. Dapp sends `chia_createOfferForIds` with:
   ```json
   {
     "offer": { "1": "-1000000000000", "<nft_launcher_id>": "1" },
     "driver_dict": {
       "<nft_launcher_id>": {
         "type": "singleton",
         "launcher_id": "0x<nft_launcher_id>",
         "also": {
           "type": "metadata",
           "also": {
             "type": "ownership",
             "transfer_program": {
               "type": "royalty transfer program",
               "royalty_percentage": "9999"
             }
           }
         }
       }
     }
   }
   ```
3. `parseCommandDisplay` calls `nftGetInfo("<nft_launcher_id>")`, which returns the real on-chain `royalty_percentage` (e.g. `250`). The confirmation dialog shows `amountWithRoyalties` = 1.025 XCH.
4. User approves.
5. The backend receives the original `driver_dict` with `royalty_percentage: "9999"` and constructs an offer whose royalty payment is ~100 XCH on top of the 1 XCH base — far exceeding what was displayed. [6](#0-5) [7](#0-6)

function royaltyPercentageForDriverInfo(driverInfo: unknown): number | undefined {
  if (!isPlainObject(driverInfo)) {
    return undefined;
  }

  const { also } = driverInfo;
  if (!isPlainObject(also)) {
    return undefined;
  }

  const ownershipLayer = also.also;
  if (!isPlainObject(ownershipLayer)) {
    return undefined;
  }

  const transferProgram = ownershipLayer.transfer_program;
  if (!isPlainObject(transferProgram)) {
    return undefined;
  }

  return parseRoyaltyPercentage(transferProgram.royalty_percentage);
}

function createOfferRoyaltyPercentages(
  walletDelta: WalletDelta,
  driverDict: Record<string, unknown>,
): AssetRoyaltyPercentages {
  const royaltyPercentages: AssetRoyaltyPercentages = {
    spending: {},
    receiving: {},
  };

  for (const assetId of Object.keys(walletDelta.spending)) {
    royaltyPercentages.spending[assetId] = royaltyPercentageForDriverInfo(
      driverDict[assetId] ?? driverDict[`0x${assetId}`],
    );
  }

  for (const assetId of Object.keys(walletDelta.receiving)) {
    royaltyPercentages.receiving[assetId] = royaltyPercentageForDriverInfo(
      driverDict[assetId] ?? driverDict[`0x${assetId}`],
    );
  }

  return royaltyPercentages;
}

function royaltyPercentagesForSide(lines: DisplayWalletDeltaItem[]): number[] {
  return lines
    .filter((line): line is Extract<DisplayWalletDeltaItem, { kind: 'nft' }> => line.kind === 'nft')
    .map((line) => line.royaltyPercentage)
    .filter(
      (royaltyPercentage): royaltyPercentage is number => royaltyPercentage !== undefined && royaltyPercentage > 0,
    );
}

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

export async function parseCommandDisplay(command: string, params: Record<string, unknown>) {
  if (command === 'chia_wallet.take_offer') {
    if (!params.offer || typeof params.offer !== 'string') {
      throw new Error('Offer is not valid');
    }

    const offerSummary = await getOfferSummary(params.offer);
    if (!offerSummary || !offerSummary.summary || !offerSummary.success) {
      throw new Error('Offer is not valid');
    }

    const { summary } = offerSummary;

    const walletDelta = offerSummaryToWalletDelta(summary);
    const walletInfos = await getWalletInfos();
    const assetKinds = offerSummaryAssetKinds(summary);
    const royaltyPercentages = offerSummaryRoyaltyPercentages(summary);
    const fees = parseMojos(summary.fees);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
    };
  }

  if (command === 'chia_wallet.create_offer_for_ids') {
    if (!params.offer || !isPlainObject(params.offer)) {
      throw new Error('Offer is not valid');
    }

    if (params.driver_dict !== undefined && !isPlainObject(params.driver_dict)) {
      throw new Error('Driver Dict is not valid');
    }

    const walletDelta = createOfferToWalletDelta(params.offer);
    const walletInfos = await getWalletInfos();
    const driverDict = params.driver_dict ?? {};
    const assetKinds = createOfferAssetKinds(walletDelta, walletInfos, driverDict);
    const royaltyPercentages = createOfferRoyaltyPercentages(walletDelta, driverDict);

    return {
      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, undefined),
    };
  }
