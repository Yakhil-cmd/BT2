### Title
Unbounded Loop in `calculate_rewards` Allows Delegator to Permanently Freeze Their Own Unclaimed Yield via Trace Inflation — (`src/pool/pool.cairo`)

---

### Summary

The `calculate_rewards` function in `Pool` contains an explicitly acknowledged unbounded loop that iterates over a pool member's `pool_member_epoch_balance` trace. A delegator can inflate this trace by repeatedly calling balance-changing operations across many epochs. Once the trace is large enough, every subsequent call to `claim_rewards` for that pool member will exhaust the Starknet gas limit and revert, permanently freezing the delegator's unclaimed yield.

---

### Finding Description

In `src/pool/pool.cairo`, the internal `calculate_rewards` function iterates over every entry in the pool member's balance trace to compute accumulated rewards:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The trace grows by one entry every time the pool member changes their delegated balance — via `add_to_delegation_pool`, `exit_intent` (partial or full), or `enter_delegation_pool`. Each such call at a new epoch appends a checkpoint to `pool_member_epoch_balance`: [2](#0-1) 

There is no cap on how many entries this trace can accumulate. The codebase itself acknowledges the loop is unbounded, relying only on the informal assumption that it is "unlikely to exceed gas limits."

**Attack path:**

1. Attacker delegates the minimum required amount to a pool.
2. Across N epochs, the attacker alternates between `add_to_delegation_pool` (tiny amount) and `exit_intent` (tiny amount), each call at a new epoch appending one entry to their trace.
3. After accumulating enough entries (trace length → large N), every call to `claim_rewards` for that pool member will loop through all N entries, exhausting the block gas limit and reverting.
4. The pool member's unclaimed yield is permanently inaccessible. [3](#0-2) 

---

### Impact Explanation

Once the trace is sufficiently large, `claim_rewards` will always revert. The delegator's accumulated STRK rewards are permanently locked in the pool contract with no recovery path — there is no administrative function to reset or truncate the trace, and no alternative claim path that bypasses `calculate_rewards`.

This matches the allowed impact: **Permanent freezing of unclaimed yield** (High). [4](#0-3) 

---

### Likelihood Explanation

The attack requires only that the attacker hold the minimum delegation amount and submit one transaction per epoch over many epochs. The cost is proportional to the number of epochs × gas per transaction. On Starknet, where epochs are short (governed by `K` blocks), this is achievable at modest cost. No privileged access is required — any unprivileged delegator can execute this against their own position. [5](#0-4) 

---

### Recommendation

Add a maximum number of balance-trace entries that `calculate_rewards` will process in a single call (analogous to the external report's recommendation to add a match limit). Introduce a checkpoint-based pagination mechanism: store the last processed `entry_to_claim_from` index in the pool member's state so that rewards can be claimed incrementally across multiple transactions rather than in a single unbounded loop.

---

// Pseudocode — attacker inflates their own pool_member_epoch_balance trace

let attacker: ContractAddress = ...;
let pool: ContractAddress = ...;
let min_amount: Amount = MIN_DELEGATION;

// Step 1: Enter pool
pool.enter_delegation_pool(reward_address: attacker, amount: min_amount);

// Step 2: Across N epochs, alternate tiny add/exit_intent to append trace entries
for i in 0..N {
    advance_epoch();                                          // new epoch → new trace entry
    pool.add_to_delegation_pool(pool_member: attacker, amount: 1);
    advance_epoch();
    pool.exit_intent(amount: 1);                             // another trace entry
}

// Step 3: After N*2 trace entries, claim_rewards loops N*2 times → gas exhaustion
pool.claim_rewards(pool_member: attacker);  // REVERTS: out of gas
// Attacker's accumulated yield is permanently frozen.
``` [6](#0-5)

        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
