### Title
Atomic Multi-Token Transfer Loop in `unstake_action` Permanently Blocks Staker Exit — (`File: src/staking/staking.cairo`)

### Summary
`unstake_action` calls `transfer_to_pools_when_unstake`, which iterates over every pool a staker has opened and issues a `checked_transfer` for each pool's token in a single atomic transaction. If any one of those token transfers reverts — because the BTC-pegged token is paused, the pool contract address is blacklisted, or the token contract behaves non-standardly — the entire `unstake_action` reverts. The staker's own STRK principal and all pool delegations remain locked in the staking contract with no alternative exit path.

### Finding Description

`unstake_action` executes the following sequence atomically:

1. Sends STRK rewards to the staker's reward address (`send_rewards_to_staker`, line 495).
2. Erases the staker record (`remove_staker`, line 502).
3. Transfers the staker's own STRK principal back to the staker (line 505–506).
4. Calls `transfer_to_pools_when_unstake` (line 508–511), which loops over every pool and calls `checked_transfer` for each pool's token.
5. Clears the pool list (line 513). [1](#0-0) 

Inside `transfer_to_pools_when_unstake`, the loop is:

```cairo
for (pool_contract, token_address) in staker_pool_info.pools {
    ...
    token_dispatcher
        .checked_transfer(
            recipient: pool_contract,
            amount: pool_balance.to_native_amount(:decimals).into(),
        );
}
``` [2](#0-1) 

`checked_transfer` panics on failure (it is not a try-pattern): [3](#0-2) 

Because all steps are in one transaction, a revert at step 4 rolls back steps 1–3 as well. The staker's record is restored, but `unstake_action` cannot be retried successfully as long as the problematic token transfer keeps failing.

A staker can legitimately hold multiple pools — one STRK pool and one or more BTC-pegged token pools — by calling `set_open_for_delegation` once per token: [4](#0-3) 

The protocol explicitly supports this multi-pool pattern and tests it: [5](#0-4) 

### Impact Explanation

If any BTC-pegged token registered in the protocol acquires a transfer restriction (pause, blacklist of the pool contract address, or any other non-standard ERC-20 behavior), every staker who opened a pool for that token is unable to execute `unstake_action`. Their STRK principal — which is held by the staking contract — is frozen for as long as the restriction persists. Because there is no partial-exit or skip-pool mechanism, the staker has no recourse within the protocol.

**Impact class:** Temporary freezing of funds (staker's STRK principal and all pool delegations).

### Likelihood Explanation

Real-world BTC-pegged tokens (e.g., WBTC) include blacklist and pause functionality. The protocol's `add_token` / `enable_token` flow does not restrict which ERC-20 contracts can be registered; it only validates decimals: [6](#0-5) 

Once a token is added and a staker opens a pool for it, the staker is permanently coupled to that token's transfer behavior during `unstake_action`. A token pause (even a routine maintenance pause by the token's own admin), a regulatory blacklisting of the pool contract address, or an upgrade to the token contract that introduces transfer restrictions would all trigger the freeze. The more BTC-pegged tokens the protocol supports, the larger the attack surface.

### Recommendation

Decouple the token transfers from the staker-removal accounting. Two approaches:

1. **Internal accounting first, pull-based withdrawal later**: Record each pool's owed balance in storage during `unstake_action` without transferring. Allow pool contracts (or delegators) to pull their funds in a separate call, so a single failing token cannot block the others.

2. **Per-pool try-transfer with residual tracking**: Wrap each `checked_transfer` in a safe-dispatcher call; on failure, record the owed amount in a separate "pending transfer" map that the pool can claim later. This mirrors the recommendation in the referenced external report.

### Proof of Concept

1. Staker calls `set_open_for_delegation` for STRK and for a BTC-pegged token that has a pause function.
2. Delegators enter both pools.
3. Staker calls `unstake_intent`; the exit window passes.
4. The BTC token admin pauses the token (or the pool contract address is blacklisted by the token).
5. Staker calls `unstake_action`.
6. `transfer_to_pools_when_unstake` reaches the BTC pool's `checked_transfer`, which reverts with `TRANSFER_FAILED`.
7. The entire transaction reverts; the staker's STRK principal remains locked in the staking contract.
8. The staker has no alternative exit path — `unstake_action` is the only function that returns principal, and it will keep reverting until the BTC token restriction is lifted. [7](#0-6)

            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

                .checked_transfer(recipient: staker_address, amount: staker_amount.into());

                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount

        fn set_open_for_delegation(

        fn transfer_to_pools_when_unstake(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
        ) {
            for (pool_contract, token_address) in staker_pool_info.pools {
                let pool_balance = self.get_delegated_balance(:staker_address, :pool_contract);
                let token_dispatcher = IERC20Dispatcher { contract_address: token_address };
                let pool_dispatcher = IPoolDispatcher { contract_address: pool_contract };
                pool_dispatcher.set_staker_removed();
                self
                    .insert_staker_delegated_balance(
                        :staker_address, :pool_contract, delegated_balance: Zero::zero(),
                    );
                let decimals = self.get_token_decimals(:token_address);
                token_dispatcher
                    .checked_transfer(
                        recipient: pool_contract,
                        amount: pool_balance.to_native_amount(:decimals).into(),
                    );
            }
        }

    let strk_pool_contract = stake_with_strk_pool_enabled(:cfg);
    // Open BTC pool.
    let staker_address = cfg.test_info.staker_address;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    let btc_token = cfg.test_info.btc_token;
    let btc_token_address = btc_token.contract_address();
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    let btc_pool_contract = staking_dispatcher
        .set_open_for_delegation(token_address: btc_token_address);
    // Enter STRK pool.
    enter_delegation_pool_for_testing_using_dispatcher(
        pool_contract: strk_pool_contract, :cfg, :token,
    );
    // Enter BTC pool.
    enter_delegation_pool_for_testing_using_dispatcher(
        pool_contract: btc_pool_contract, :cfg, token: btc_token,
    );
    // Set commission commitment.
    let staker_address = cfg.test_info.staker_address;
    let staking_dispatcher = IStakingDispatcher { contract_address: staking_contract };
    let staker_info = staking_dispatcher.staker_info_v1(:staker_address);
    let max_commission = staker_info.get_pool_info().commission;
    let expiration_epoch = staking_dispatcher.get_current_epoch() + 1;
    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);
    staking_dispatcher.set_commission_commitment(:max_commission, :expiration_epoch);

    cheat_caller_address_once(contract_address: staking_contract, caller_address: staker_address);

        .staker_info_v1(:staker_address)
        .unclaimed_rewards_own;
