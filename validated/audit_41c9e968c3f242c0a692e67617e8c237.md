### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Freeze Staker Yield - (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any public address can call it with `disable_rewards: true` to consume the global `last_reward_block` slot for the current block without distributing rewards, permanently blocking the legitimate sequencer from issuing rewards for that block.

---

### Finding Description

The specification at `docs/spec.md:1644-1645` explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `StakingRewardsManagerImpl::update_rewards` in `src/staking/staking.cairo` performs no such check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... staker validity checks ...

    // Update last block rewards.
    self.last_reward_block.write(current_block_number);  // <-- global slot consumed

    if disable_rewards || self.is_pre_consensus() {
        return;  // <-- exits without distributing any rewards
    }
    // ... actual reward distribution ...
}
```

The only guard is `current_block_number > self.last_reward_block.read()` — a **global, per-block** slot. Once any caller writes `last_reward_block` to the current block, no further call to `update_rewards` can succeed in that block (it reverts with `REWARDS_ALREADY_UPDATED`).

An attacker calls `update_rewards(any_valid_staker_address, disable_rewards: true)` at the start of every block. This:
1. Passes all staker-validity checks (requires only a valid, active staker address — public information).
2. Writes `last_reward_block = current_block_number` (line 1485).
3. Returns immediately at line 1487–1489 without distributing any rewards.
4. Causes every subsequent legitimate sequencer call in the same block to revert with `REWARDS_ALREADY_UPDATED`.

Repeated every block, this permanently freezes all consensus-phase staker and delegator yield.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

In the consensus rewards phase (V3), `update_rewards` is the sole mechanism by which stakers and delegators accumulate `unclaimed_rewards_own`. If the global `last_reward_block` is consumed with `disable_rewards: true` every block, no rewards are ever credited to any staker or pool. All stakers and delegators are permanently denied their yield with no recovery path short of a contract upgrade.

---

### Likelihood Explanation

**High.** The entry point is fully public — no tokens, no stake, no privileged role required. The attacker only needs to know one valid active staker address (trivially obtained from on-chain events). The attack is cheap (one transaction per block) and can be automated. There is no economic disincentive: the attacker spends only gas.

---

### Recommendation

Add a caller check at the top of `update_rewards` matching the spec's stated access control:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    // ... rest of function
}
```

Alternatively, if the sequencer address is not stored, restrict to a designated `REWARDS_MANAGER` role consistent with the existing `RolesComponent` pattern used elsewhere in the contract.

---

### Proof of Concept

1. Consensus rewards are active (`!self.is_pre_consensus()`).
2. Attacker (any EOA) observes a valid active staker address `S` from on-chain `Stake` events.
3. At the start of each new block, attacker calls:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. `last_reward_block` is set to the current block number; function returns without distributing rewards.
5. The legitimate sequencer calls `staking.update_rewards(staker_address: S, disable_rewards: false)` — reverts with `REWARDS_ALREADY_UPDATED`.
6. Staker's `unclaimed_rewards_own` is never incremented. Pool balances are never topped up.
7. Repeated every block → all stakers and delegators receive zero yield indefinitely.
