### Title
Blocklist Destination Check Bypassed via Intermediary Contract Call — (`app/block_address.go`, `app/proposal.go`)

### Summary
Cronos enforces a blocklist that prevents transactions whose direct destination (`ethTx.To()`) is a blocked address. However, this check only inspects the outermost EVM call target. An unprivileged attacker can route funds to a blocked address through an intermediary contract, bypassing the destination restriction entirely.

### Finding Description
Cronos implements a blocklist system with two enforcement layers:

1. **`BlockAddressesDecorator.AnteHandle`** (`app/block_address.go`) — runs only during `ctx.IsCheckTx()` (mempool admission). It checks the tx signer and `ethTx.To()`.
2. **`ProposalHandler.ValidateTransaction`** (`app/proposal.go`) — runs during `PrepareProposal` / `ProcessProposal`. It checks the tx signer and `ethTx.To()`.

Both layers check only the **direct** EVM call target:

```go
if ethTx.To() != nil {
    if _, ok := bad.blockedMap[sdk.AccAddress(ethTx.To().Bytes()).String()]; ok {
        return ctx, fmt.Errorf("destination address is blocked: ...")
    }
}
```

Neither layer inspects internal EVM sub-calls generated during contract execution. Critically, `BlockAddressesDecorator.AnteHandle` is entirely gated behind `ctx.IsCheckTx()`, so **no blocklist check runs at all during block execution (DeliverTx/FinalizeBlock)**. The proposal-level check also only sees `ethTx.To()`.

An attacker deploys a contract `C` (not on the blocklist) with a function that internally transfers CRO or CRC20/ERC20 tokens to blocked address `A`. The attacker submits a transaction with `to = C`. All blocklist checks pass because `C` is not blocked. During execution, `C` transfers to `A`, and `A` receives the funds.

### Impact Explanation
This is a **High** impact: bypass of Cronos block-list authorization checks. The destination check is an intentional security control (evidenced by `test_block_list_contract` in `integration_tests/test_e2ee.py` and CHANGELOG entries [#1922](https://github.com/crypto-org-chain/cronos/pull/1922), [#1924](https://github.com/crypto-org-chain/cronos/pull/1924)). The bypass allows any unprivileged user to deliver CRO, IBC vouchers, or CRC20/ERC20 tokens to a blocked address, defeating the purpose of the destination restriction.

### Likelihood Explanation
Exploitation requires only deploying a simple forwarding contract and calling it — no special privileges, leaked keys, or cryptographic assumptions. Any EVM-capable user can execute this.

### Recommendation
The destination blocklist check must be enforced at **execution time**, not only at mempool/proposal time. Options include:
- Removing `if ctx.IsCheckTx()` from `BlockAddressesDecorator.AnteHandle` so the check also runs during `DeliverTx`/`FinalizeBlock`.
- Adding an EVM state transition hook that inspects all internal call targets and log recipients against the blocklist during execution.
- Alternatively, accepting that the destination check cannot be enforced against internal calls and documenting this limitation explicitly.

### Proof of Concept

1. Blocked address `A` is added to the Cronos blocklist.
2. Attacker deploys contract `C`:
   ```solidity
   contract Forwarder {
       function forward(address payable target) external payable {
           target.transfer(msg.value);
       }
   }
   ```
3. Attacker submits EVM tx: `to = C`, `value = X`, calling `forward(A)`.
4. **CheckTx**: `ethTx.To() = C` — not blocked → admitted to mempool. [1](#0-0) 
5. **PrepareProposal/ProcessProposal**: same check, `C` not blocked → included in block. [2](#0-1) 
6. **Block execution**: `BlockAddressesDecorator` check is skipped entirely (guarded by `ctx.IsCheckTx()`). [3](#0-2) 
7. `C.forward(A)` executes; `A` receives `X` CRO — blocklist destination restriction fully bypassed.

The same technique works for CRC20/ERC20 token transfers: a contract holding an allowance or balance calls `transfer(A, amount)` internally, and neither the ante handler nor the proposal handler intercepts it. [4](#0-3)

```go
func (h *ProposalHandler) ValidateTransaction(tx sdk.Tx, txBz []byte) error {
	if len(h.blocklist) == 0 {
		// fast path, accept all txs
		return nil
	}

	var err error
	if tx == nil {
		tx, err = h.TxDecoder(txBz)
		if err != nil {
			return err
		}
	}

	sigTx, ok := tx.(signing.SigVerifiableTx)
	if !ok {
		return fmt.Errorf("tx of type %T does not implement SigVerifiableTx", tx)
	}

	signers, err := sigTx.GetSigners()
	if err != nil {
		return err
	}
	for _, signer := range signers {
		encoded, err := h.addressCodec.BytesToString(signer)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", signer, err)
		}
		if _, ok := h.blocklist[encoded]; ok {
			return fmt.Errorf("signer is blocked: %s", encoded)
		}
	}

	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if ok {
			ethTx := msgEthTx.AsTransaction()
			// check the destination address
			if ethTx.To() != nil {
				encoded, err := h.addressCodec.BytesToString(ethTx.To().Bytes())
				if err != nil {
					return fmt.Errorf("invalid bech32 address: %s, err: %w", ethTx.To(), err)
				}
				if _, ok := h.blocklist[encoded]; ok {
					return fmt.Errorf("destination address is blocked: %s", encoded)
				}
			}
			// check EIP-7702 authorisation list
			if ethTx.SetCodeAuthorizations() != nil {
				for _, auth := range ethTx.SetCodeAuthorizations() {
					addr, err := auth.Authority()
					if err == nil {
						encoded, err := h.addressCodec.BytesToString(addr.Bytes())
						if err != nil {
							return fmt.Errorf("invalid bech32 address: %s, err: %w", addr, err)
						}
						if _, ok := h.blocklist[encoded]; ok {
							return fmt.Errorf("signer is blocked: %s", encoded)
						}
					}
					// check the target address
					encoded, err := h.addressCodec.BytesToString(auth.Address.Bytes())
					if err != nil {
						return fmt.Errorf("invalid bech32 address: %s, err: %w", auth.Address, err)
					}
					if _, ok := h.blocklist[encoded]; ok {
						return fmt.Errorf("authorisation address is blocked: %s", encoded)
					}
				}
			}
		}
	}

	return nil
}
```
