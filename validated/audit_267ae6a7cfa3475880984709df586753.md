### Title
ETH Permanently Locked in `RewardSupplier` When `tick()` Called With `msg.value > 0` and No Pending L2 Messages — (`L1/starkware/solidity/stake/RewardSupplier.sol`)

### Summary
`tick()` is `external payable` with no access control and no refund path. When called with `msg.value > 0` while `amountToMint == 0`, the ETH is silently accepted and permanently locked in the contract. No `receive()`, `fallback()`, `withdraw()`, or rescue function exists anywhere in the contract or its inheritance chain.

### Finding Description
`tick()` is declared `external payable` and accepts ETH from any caller. [1](#0-0) 

The ETH is only consumed inside the `if (amountToMint > 0)` branch: [2](#0-1) 

When `requiredMinting()` returns `(0, 0)` — which happens whenever `messagingContract().l2ToL1Messages(msgHash) == 0` — the entire branch is skipped and `msg.value` is never forwarded or refunded: [3](#0-2) 

A search across the entire production codebase confirms there is no `receive()`, `fallback()`, `withdraw()`, or rescue function in `RewardSupplier`, `RewardSupplierStorage`, or `ProxySupportImpl`: [4](#0-3) [5](#0-4) 

### Impact Explanation
Any ETH sent to `tick()` when no L2 mint-request messages are pending is permanently locked in the `RewardSupplier` contract with no recovery path. The caller's ETH is irrecoverably lost. This matches **High: Permanent freezing of funds** (the question's own scope label of "temporary" understates it — there is no mechanism to ever unfreeze the ETH).

### Likelihood Explanation
- `tick()` has no access control; any EOA or contract can call it.
- The condition `amountToMint == 0` is the normal state between reward epochs — most calls to `tick()` will hit this branch.
- A caller who mistakenly (or experimentally) attaches ETH loses it permanently.
- A griefing actor can also deliberately trigger this to waste their own ETH and demonstrate the flaw, but the primary risk is accidental loss by legitimate callers who follow the pattern of always attaching a fee.

### Recommendation
Add an explicit refund at the end of `tick()` for any unused ETH:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();
    if (amountToMint > 0) {
        // ... existing logic ...
    } else if (msg.value > 0) {
        // Refund unused ETH
        (bool ok, ) = msg.sender.call{value: msg.value}("");
        require(ok, "ETH_REFUND_FAILED");
    }
}
```

Alternatively, remove `payable` from `tick()` entirely and require callers to pass fee amounts as a parameter, or revert if `msg.value > 0` and `amountToMint == 0`.

### Proof of Concept
```solidity
// 1. Deploy RewardSupplier with no pending L2 messages.
// 2. Call tick() with msg.value = 1 ether from any EOA.
// 3. requiredMinting() returns (0, 0) because l2ToL1Messages(msgHash) == 0.
// 4. The if-block is skipped; msg.value is never forwarded or refunded.
// 5. assert(address(rewardSupplier).balance == 1 ether);  // ETH is stuck
// 6. No withdraw(), rescue(), or receive() exists to recover it.
``` [6](#0-5)

```text
    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
