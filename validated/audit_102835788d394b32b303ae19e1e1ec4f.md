### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Per-Block Yield - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the staking contract carries no caller-identity check, yet it unconditionally advances the global `last_reward_block` cursor **before** it evaluates the `disable_rewards` flag. Any unprivileged address can therefore call `update_rewards(valid_staker, disable_rewards=true)` once per block, consuming the block's single reward-distribution slot while distributing nothing, permanently destroying that block's yield for every staker and delegator.

---

### Finding Description

`update_rewards` is part of the `IStakingRewardsManager` interface and is intended, per the specification, to be callable only by the Starkware sequencer:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation, however, enforces no such restriction. The only gate is `general_prerequisites()`, which exclusively checks the contract-paused flag: [2](#0-1) 

After passing that gate the function immediately writes the current block number into the global `last_reward_block` storage slot: [3](#0-2) 

Only **after** that write does it check `disable_rewards`:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;          // ← exits with no rewards distributed
}
``` [4](#0-3) 

Because `last_reward_block` is a single global value, any subsequent call in the same block—including the legitimate sequencer call—will revert with `REWARDS_ALREADY_UPDATED`:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [5](#0-4) 

The interface definition confirms there is no access-control annotation: [6](#0-5) 

**Analog to the external report**: In the Franklin Templeton bug, `request_full_liquidation` stored `amount = 0` in the pending transaction; the settlement function then checked `if (tx.amount > 0)` and silently skipped the burn. Here, an attacker stores `disable_rewards = true` as the effective "parameter" by being the first caller; the function then checks that flag and silently skips reward distribution—while the side-effect (`last_reward_block` advance) has already been committed, locking out the legitimate path for that block.

---

### Impact Explanation

Every block in which the attacker front-runs the sequencer call produces **zero yield** for all stakers and delegators. Because `last_reward_block` is global, a single attacker transaction per block is sufficient to suppress rewards for the entire protocol. The lost yield for each attacked block is irrecoverable—there is no mechanism to retroactively credit missed block rewards. This constitutes **permanent freezing of unclaimed yield** (High) and, at scale, approaches **protocol insolvency** (Critical) if sustained.

---

### Likelihood Explanation

- The function is publicly callable with no role check.
- The attacker only needs to know any valid, active staker address (all staker addresses are public on-chain).
- The cost is a single L2 transaction per block.
- Front-running the sequencer on Starknet is feasible because block production is not permissionless and transaction ordering is observable.

---

### Recommendation

**Short term**: Add a sequencer-only guard at the top of `update_rewards`, consistent with the specification:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // enforce spec access control
    ...
}
```

**Medium term**: Move the `last_reward_block.write(current_block_number)` call to **after** the `disable_rewards` / `is_pre_consensus()` early-return, so that a no-op invocation does not consume the block's reward slot.

**Long term**: Add integration tests that assert `update_rewards` reverts when called by an address that is not the sequencer, mirroring the post-condition testing gap identified in the external report.

---

### Proof of Concept

```
1. Staker Alice is active and has been staking for K+ epochs (rewards are live).
2. Attacker Eve monitors the mempool / block production.
3. In block N, before the sequencer submits its `update_rewards(alice, false)` tx,
   Eve submits: update_rewards(alice, disable_rewards=true)
4. Eve's tx executes first:
   - last_reward_block is set to N
   - disable_rewards=true → early return, zero rewards distributed
5. Sequencer's tx executes next:
   - current_block_number (N) == last_reward_block (N) → REWARDS_ALREADY_UPDATED → revert
6. Block N's rewards are permanently lost for Alice and all her delegators.
7. Eve repeats every block at negligible cost.
``` [7](#0-6)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
