### Title
Pool Rewards Permanently Locked When Delegation Balance Is Below `min_delegation_for_rewards` - (`src/pool/pool.cairo`)

---

### Summary

When a delegation pool's total balance is non-zero but below `min_delegation_for_rewards`, the staking contract transfers STRK reward tokens to the pool contract but `compute_rewards_per_unit` returns zero, leaving the cumulative rewards trace unchanged. Pool members can never claim these transferred tokens, permanently locking them in the pool contract.

---

### Finding Description

In `update_pool_rewards` (`src/staking/staking.cairo`), for each pool with non-zero rewards, two sequential operations occur:

1. `send_rewards_to_delegation_pool` unconditionally transfers `pool_rewards` STRK to the pool contract.
2. `pool_dispatcher.update_rewards_from_staking_contract(rewards: pool_rewards, pool_balance: pool_balance.to_native_amount(:decimals))` is called to record the distribution in the pool's `cumulative_rewards_trace`. [1](#0-0) 

Inside `update_rewards_from_staking_contract`, the cumulative trace is updated by adding `compute_rewards_per_unit(staking_rewards: rewards, total_stake: pool_balance)`: [2](#0-1) 

`compute_rewards_per_unit` contains an overflow-guard that returns **zero** when `total_stake < min_delegation_for_rewards`: [3](#0-2) 

For STRK pools, `min_delegation_for_rewards = 10^18` (1 STRK). For BTC-8d pools it is `10^3`. These thresholds are set at pool construction from `get_token_rewards_config`: [4](#0-3) [5](#0-4) 

When `pool_balance < min_delegation_for_rewards`, `compute_rewards_per_unit` returns 0, so the cumulative trace is updated as `last + 0 = last` — no increase. The STRK tokens already transferred to the pool contract are therefore unclaimable by any pool member, because `claim_rewards` computes payouts exclusively from the cumulative trace delta: [6](#0-5) 

There is no admin withdrawal function in the pool contract, so these tokens are permanently frozen.

The upstream calculation in `calculate_staker_pools_rewards` only gates on `pool_rewards.is_non_zero()` before appending to `pools_rewards_data` — it does **not** check whether `pool_balance < min_delegation_for_rewards`: [7](#0-6) 

This means a pool with, e.g., 0.5 STRK delegated (below the 1 STRK threshold) will receive a proportional share of block/epoch rewards transferred to it, but those rewards will never be distributable.

The existing flow test `PoolWithMinBtcFlow` acknowledges that `pool_rewards.is_zero()` for a delegator below the threshold, but does **not** assert that the pool contract's STRK balance is also zero — the locked tokens go undetected: [8](#0-7) 

---

### Impact Explanation

STRK reward tokens transferred to a delegation pool contract whose total balance is below `min_delegation_for_rewards` are permanently frozen. No pool member can ever claim them, and no admin function exists to recover them. This matches the allowed impact: **Permanent freezing of unclaimed yield**.

---

### Likelihood Explanation

- `enter_delegation_pool` enforces only `amount.is_non_zero()` — there is no minimum delegation floor.
- A pool's total balance can be below `min_delegation_for_rewards` (1 STRK for STRK pools, `10^3` satoshis for BTC-8d pools) at any point: at pool inception before sufficient delegation accumulates, or after partial exits reduce the balance below the threshold.
- Both the attestation-based (`update_rewards_from_attestation_contract`) and consensus-based (`update_rewards`) reward paths share the same `_update_rewards` → `update_pool_rewards` code path, so the bug is reachable in both protocol phases.
- Any staker can open a pool and any delegator can enter with a sub-threshold amount, making this reachable by unprivileged actors without any privileged role.

---

### Recommendation

Before transferring rewards to the pool contract, check whether `pool_balance.to_native_amount(:decimals) >= pool.min_delegation_for_rewards`. If not, skip both the transfer and the `update_rewards_from_staking_contract` call (or redirect the undistributable rewards back to the staker / reward supplier). Alternatively, move the `min_delegation_for_rewards` guard to `calculate_staker_pools_rewards` so that pools below the threshold are excluded from `pools_rewards_data` entirely, preventing the transfer from occurring in the first place.

---

### Proof of Concept

1. Staker stakes and opens a STRK delegation pool.
2. Delegator calls `enter_delegation_pool` with `amount = 5 * 10^17` (0.5 STRK, below `min_delegation_for_rewards = 10^18`).
3. After K epochs, `update_rewards` (or `update_rewards_from_attestation_contract`) is called.
4. `calculate_staker_pools_rewards` computes a non-zero `pool_rewards` (proportional to 0.5 STRK share).
5. `send_rewards_to_delegation_pool` transfers `pool_rewards` STRK to the pool contract. Pool contract STRK balance is now `pool_rewards > 0`.
6. `update_rewards_from_staking_contract(rewards: pool_rewards, pool_balance: 5*10^17)` is called. Since `5*10^17 < 10^18 = min_delegation_for_rewards`, `compute_rewards_per_unit` returns 0. Cumulative trace is unchanged.
7. Delegator calls `claim_rewards`. `calculate_rewards` computes `to_sigma - from_sigma = 0`. Delegator receives 0 STRK.
8. Pool contract retains `pool_rewards` STRK with no mechanism to recover it — permanently locked.

```text
        fn calculate_rewards(
            self: @ContractState,
            pool_member: ContractAddress,
            from_checkpoint: PoolMemberCheckpoint,
            until_checkpoint: PoolMemberCheckpoint,
            mut entry_to_claim_from: VecIndex,
        ) -> (Amount, VecIndex) {
            let pool_member_trace = self.pool_member_epoch_balance.entry(pool_member);
            // Note: `until_epoch` is the current epoch.
            let until_epoch = until_checkpoint.epoch();

            let mut rewards = 0;

            let pool_member_trace_length = pool_member_trace.length();

            let mut from_sigma = self.find_sigma(from_checkpoint, curr_epoch: until_epoch);
            let mut from_balance = from_checkpoint.balance();

            let base_value = self.staking_rewards_base_value.read();

            // **Note**: The loop iterates over the balance changes in the pool member's balance
            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
```

```text
        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
```

```text
/// Get token rewards configuration based on address and decimals.
pub(crate) fn get_token_rewards_config(token_address: ContractAddress) -> TokenRewardsConfig {
    if token_address == STRK_TOKEN_ADDRESS {
        STRK_CONFIG
    } else {
        // BTC token.
        let token_dispatcher = IERC20MetadataDispatcher { contract_address: token_address };
        let decimals = token_dispatcher.decimals();
        assert!(decimals >= 5 && decimals <= 18, "{}", GenericError::INVALID_TOKEN_DECIMALS);
        TokenRewardsConfig {
            decimals,
            min_for_rewards: 10_u128.pow(decimals.into() - 5),
            base_value: 10_u128.pow(decimals.into() + 5),
        }
    }
}
```
