### Title
`validate_resource_bounds` Unconditionally Applies `min_gas_price` to `l2_gas.max_price_per_unit`, Rejecting Valid L1-Only Transactions at the Gateway — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a hardcoded `min_gas_price` floor (production default: 8,000,000,000 fri / 8 Gwei) against `l2_gas.max_price_per_unit` for every incoming V3 transaction, regardless of whether the transaction actually uses L2 gas. A valid V3 transaction that specifies only L1 gas or only L1 data gas bounds will have `l2_gas.max_price_per_unit = 0`, which is unconditionally less than the production minimum, causing the gateway to reject it with `MaxGasPriceTooLow`. The test suite masks this by setting `min_gas_price: 0` in `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING`.

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs`, `validate_resource_bounds` reads the `AllResourceBounds` from the incoming RPC transaction and applies two checks:

```rust
// Line 71-76
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
```

The production default for `min_gas_price` is `8_000_000_000`:

```rust
// crates/apollo_gateway_config/src/config.rs, line 192
min_gas_price: 8_000_000_000,
```

Confirmed in `config_schema.json` (line 3205): `"value": 8000000000`.

The `AllResourceBounds` struct has three independent fields: `l1_gas`, `l2_gas`, `l1_data_gas`. The preceding `ZeroResourceBounds` check only requires that at least one of the three resources is non-zero. Therefore, a transaction with:

```
AllResourceBounds {
    l1_gas: ResourceBounds { max_amount: X, max_price_per_unit: Y },  // non-zero
    l2_gas: ResourceBounds::default(),   // 0 amount, 0 price
    l1_data_gas: ResourceBounds::default(),
}
```

passes the `ZeroResourceBounds` check but then hits `0 < 8_000_000_000 = true` and is rejected.

The test suite explicitly acknowledges these as valid transaction shapes — `valid_l1_gas` and `valid_l1_data_gas` are listed as positive-flow cases — but they only pass because `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING` sets `min_gas_price: 0`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator_test.rs, lines 54-57
static DEFAULT_VALIDATOR_CONFIG_FOR_TESTING: LazyLock<StatelessTransactionValidatorConfig> =
    LazyLock::new(|| StatelessTransactionValidatorConfig {
        validate_resource_bounds: false,
        min_gas_price: 0,
        ...
    });
```

The `valid_l1_gas` test overrides `validate_resource_bounds: true` but inherits `min_gas_price: 0`, so it passes. With the production value of `8_000_000_000`, the same transaction would be rejected.

### Impact Explanation

Any user submitting a structurally valid V3 transaction that specifies only L1 gas or only L1 data gas bounds (with L2 gas price = 0) will receive a `MaxGasPriceTooLow` rejection at the gateway stateless validation stage. The transaction never reaches the mempool or blockifier. This is a gateway admission error that rejects valid transactions before sequencing.

**Impact category: High. Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

### Likelihood Explanation

The trigger is any V3 transaction with `l2_gas.max_price_per_unit = 0` submitted to a production node. This is a valid and reachable transaction format per the `AllResourceBounds` schema. No privileged access is required; any unprivileged user can construct such a transaction. The condition is deterministic and reproducible.

### Recommendation

The `min_gas_price` check should be conditioned on whether L2 gas is actually being used. For example:

```rust
// Only enforce min_gas_price when L2 gas is non-zero
if resource_bounds.l2_gas.max_price_per_unit.0 > 0
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

Alternatively, if the intent is to require L2 gas on all transactions, that invariant should be made explicit and documented, and the `valid_l1_gas` / `valid_l1_data_gas` test cases should be updated to reflect that they are no longer valid.

### Proof of Concept

1. Construct a V3 invoke transaction with:
   - `l1_gas.max_amount = 1`, `l1_gas.max_price_per_unit = 1`
   - `l2_gas.max_amount = 0`, `l2_gas.max_price_per_unit = 0`
   - `l1_data_gas.max_amount = 0`, `l1_data_gas.max_price_per_unit = 0`
2. Submit to a production gateway with `min_gas_price = 8_000_000_000`.
3. `validate_resource_bounds` evaluates: `0 < 8_000_000_000` → `true`.
4. Gateway returns `MaxGasPriceTooLow { gas_price: GasPrice(0), min_gas_price: 8_000_000_000 }`.
5. The transaction is rejected despite being structurally valid per the `AllResourceBounds` schema and the `ZeroResourceBounds` check.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
