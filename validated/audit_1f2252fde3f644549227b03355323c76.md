### Title
Missing Access Control in `update_rewards` Allows Anyone to Suppress Block Rewards for All Stakers - (File: src/staking/staking.cairo)

### Summary

`IStakingRewardsManager::update_rewards` in `src/staking/staking.cairo` has no caller authorization check. Any unprivileged address can call it with `disable_rewards: true` for any valid staker. Because the function unconditionally writes the current block number to the **global** `last_reward_block` storage slot before inspecting `disable_rewards`, a single such call per block permanently marks that block as "already processed" and prevents any legitimate reward distribution for that block.

### Finding Description

`update_rewards` is a public function with no role guard:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.          ← written BEFORE the disable_rewards check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                            ← exits without distributing anything
    }
    ...
```

`last_reward_block` is a **single global** `BlockNumber` field (not per-staker):

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
```

Attack path:
1. Attacker calls `update_rewards(<any_valid_active_staker>, disable_rewards: true)` in block N.
2. `last_reward_block` is set to N; the function returns early — no rewards distributed.
3. Any subsequent legitimate call to `update_rewards` in block N reverts with `REWARDS_ALREADY_UPDATED`.
4. Repeating this every block permanently denies consensus-era block rewards to every staker.

The only prerequisite is that the attacker supplies a valid, active staker address. All staker addresses are publicly enumerable from the `stakers` vector in storage.

### Impact Explanation

An attacker can continuously call `update_rewards(..., disable_rewards: true)` once per block, causing **permanent freezing of unclaimed yield** for all stakers. No staker can ever receive consensus block rewards as long as the attack continues. This matches the allowed High-severity impact: *"Permanent freezing of unclaimed yield."*

### Likelihood Explanation

The entry point is fully public — no role, no allowance, no special state required. The attacker only needs to pay Starknet gas per block. Staker addresses are on-chain and trivially discoverable. The attack is mechanically simple and repeatable.

### Recommendation

Add an access-control guard to `update_rewards` so only the authorized consensus/rewards-manager role (or the staking contract itself) may call it. For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_rewards_manager();   // add appropriate role check
    ...
```

Alternatively, restrict the `disable_rewards: true` path to a privileged caller while allowing permissionless calls only when `disable_rewards` is `false` and the caller is the staker or their reward address.

### Proof of Concept

1. Deploy the staking contract in consensus-rewards mode (`consensus_rewards_first_epoch` set).
2. Register a staker (address `S`) with a non-zero STRK balance.
3. From an arbitrary address `A` (no role, no stake), call:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Observe that `last_reward_block` is now set to the current block and no rewards were distributed.
5. In the same block, attempt the legitimate call:
   ```
   staking.update_rewards(staker_address: S, disable_rewards: false)
   ```
6. This reverts with `REWARDS_ALREADY_UPDATED`.
7. Repeating step 3 every block means stakers accumulate zero consensus rewards indefinitely. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
