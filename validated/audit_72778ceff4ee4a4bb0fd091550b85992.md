Based on my investigation of the actual implementation, this is a **real vulnerability**.

## Key Findings

**1. No access control on `update_rewards`**

The spec at line 1645 states access control is "Only starkware sequencer," but the implementation contains zero caller validation: [1](#0-0) 

The only guards are `general_prerequisites()` (pause check) and the `last_reward_block` block-number gate. There is no `assert_caller_is_sequencer` or equivalent.

**2. `last_reward_block` is a single global storage slot**

The gate is not per-staker — it is a single contract-wide value: [2](#0-1) [3](#0-2) 

One successful call per block, for any staker, consumes the entire block's reward slot for every staker.

**3. `disable_rewards: true` silently discards the block reward**

After writing `last_reward_block`, the function returns immediately with no reward distribution: [4](#0-3) 

---

### Title
Unprivileged caller can permanently suppress all consensus block rewards via `update_rewards(disable_rewards: true)` — (`src/staking/staking.cairo`)

### Summary
`update_rewards` is documented as sequencer-only but has no on-chain caller check. Because `last_reward_block` is a single global slot, any attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` first in a block permanently discards that block's reward for all stakers. Repeating this every block freezes all consensus yield indefinitely.

### Finding Description
`StakingRewardsManagerImpl::update_rewards` enforces only two preconditions before writing `last_reward_block`:
- The contract is not paused (`general_prerequisites`)
- `current_block_number > last_reward_block` (one call per block)

After writing `last_reward_block = current_block_number`, if `disable_rewards == true` the function returns without minting or crediting any rewards. Because `last_reward_block` is global (not per-staker), any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`. [5](#0-4) 

The spec explicitly restricts this function to the Starkware sequencer: [6](#0-5) 

But the implementation never reads or checks `get_caller_address()`.

### Impact Explanation
An attacker calling `update_rewards(valid_staker, disable_rewards: true)` once per block permanently discards that block's consensus reward for every staker in the protocol. Sustained over time this constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

### Likelihood Explanation
The call is cheap (no token transfer, no approval needed), requires only a valid active staker address (publicly readable from events), and can be automated. The attacker has no profit motive but causes maximum damage to all stakers and delegators.

### Recommendation
Add a sequencer-only caller check at the top of `update_rewards`, consistent with the spec:
```cairo
assert!(
    get_caller_address() == self.sequencer_address.read(),
    "{}",
    Error::CALLER_IS_NOT_SEQUENCER,
);
```
Alternatively, restrict the `IStakingRewardsManager` interface so it is only callable from the attestation contract or a known privileged address.

### Proof of Concept
1. Deploy with two active stakers (both past the K-epoch activation window, consensus rewards active).
2. At block N, attacker calls `update_rewards(staker_A, disable_rewards: true)`.
   - `last_reward_block` is written to N; function returns with no rewards distributed.
3. Sequencer attempts `update_rewards(staker_A, disable_rewards: false)` — reverts `REWARDS_ALREADY_UPDATED`.
4. Advance to block N+1; repeat step 2.
5. After many blocks, `staker_A.unclaimed_rewards_own` remains zero despite being eligible every block.
6. Compare against a baseline run where the attacker does not intervene — baseline accumulates expected rewards each block.

The test infrastructure already demonstrates the `REWARDS_ALREADY_UPDATED` revert path: [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
