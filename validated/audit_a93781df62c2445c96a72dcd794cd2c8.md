### Title
`protocolUnpausePool` Always Restores to Admin-Paused State (Level 1) Instead of Pre-Pause State, Permanently Blocking Swaps After Protocol Pause Cycle — (`File: metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`protocolUnpausePool` unconditionally sets `pauseLevel` to 1 (admin-paused) regardless of what the pool's pause level was before `protocolPausePool` was called. When the protocol pauses an active pool (level 0 → 2) and then unpauses it (2 → 1), the pool is left in admin-paused state even though the admin never paused it. Swaps remain blocked until the pool admin separately calls `unpausePool`.

### Finding Description

The factory defines three pause levels:

- `0` — active (no pause)
- `1` — paused by pool admin
- `2` — paused by protocol owner

The four pause transitions are:

```solidity
// pausePool (admin): 0 → 1
if (cur != 0) revert InvalidPauseTransition(cur, 1);
IMetricOmmPoolFactoryActions(pool).setPause(1);

// unpausePool (admin): 1 → 0
if (cur != 1) revert InvalidPauseTransition(cur, 0);
IMetricOmmPoolFactoryActions(pool).setPause(0);

// protocolPausePool (owner): 0 or 1 → 2
if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
IMetricOmmPoolFactoryActions(pool).setPause(2);

// protocolUnpausePool (owner): 2 → 1  ← always 1, never 0
if (cur != 2) revert InvalidPauseTransition(cur, 1);
IMetricOmmPoolFactoryActions(pool).setPause(1);
``` [1](#0-0) 

`protocolPausePool` accepts pools at level 0 **or** 1, meaning it can override an active pool. However, the pre-pause level is never stored. `protocolUnpausePool` hard-codes the restoration target as `1`, so a pool that was at level 0 before the protocol pause is left at level 1 after the protocol unpause — a state the admin never set.

The `swap` function enforces `whenNotPaused`, which reverts on any non-zero `pauseLevel`:

```solidity
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
``` [2](#0-1) 

After the protocol pause+unpause cycle, `pauseLevel == 1 != 0`, so every `swap` call reverts.

### Impact Explanation

All swaps are blocked on the affected pool after a protocol pause+unpause cycle on an active pool. `addLiquidity` and `removeLiquidity` do not carry `whenNotPaused`, so LP funds are not locked, but the pool's core trading function is completely unusable. The pool admin must discover the unexpected state and call `unpausePool` to restore level 0. Until then, the pool is a non-functional liquidity sink.

**Severity: Medium** — broken core pool functionality (swap DoS), no direct principal loss, but requires an out-of-band admin action to recover from a state the admin never created.

### Likelihood Explanation

Any time the protocol owner exercises `protocolPausePool` on a pool that is currently active (level 0) — the common case, since most pools run at level 0 — and subsequently calls `protocolUnpausePool`, the pool is silently left at level 1. No special attacker is needed; the protocol owner's own routine pause/unpause operation triggers it.

### Recommendation

Store the pre-pause level before the protocol pause and restore it on unpause, or define a dedicated "protocol-only" restoration path that returns to level 0 when the pool was active before the protocol pause:

```diff
+   mapping(address => uint8) public prePausePauseLevel;

    function protocolPausePool(address pool) external override nonReentrant onlyOwner {
        (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
        if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
+       prePausePauseLevel[pool] = cur;
        IMetricOmmPoolFactoryActions(pool).setPause(2);
    }

    function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
        (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
        if (cur != 2) revert InvalidPauseTransition(cur, 1);
-       IMetricOmmPoolFactoryActions(pool).setPause(1);
+       uint8 restore = prePausePauseLevel[pool];
+       delete prePausePauseLevel[pool];
+       IMetricOmmPoolFactoryActions(pool).setPause(restore);
    }
```

### Proof of Concept

1. Pool is deployed; `pauseLevel = 0` (active).
2. Protocol owner calls `protocolPausePool(pool)`.
   - `cur = 0`; check `cur != 0 && cur != 1` → false → passes.
   - Pool set to `pauseLevel = 2`.
3. Protocol owner calls `protocolUnpausePool(pool)`.
   - `cur = 2`; check `cur != 2` → false → passes.
   - Pool set to `pauseLevel = 1` (hard-coded).
4. Any user calls `swap(...)`.
   - `_checkNotPaused()`: `pauseLevel = 1 != 0` → `revert PoolPaused()`.
5. Pool admin must call `unpausePool(pool)` to restore level 0, even though they never paused it. [3](#0-2) [4](#0-3)
