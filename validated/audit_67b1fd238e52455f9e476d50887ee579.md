### Title
Unrestricted `update_rewards` with `disable_rewards=true` allows any caller to permanently block all staker block-reward accrual — (File: src/staking/staking.cairo)

---

### Summary
The `update_rewards` function in the Staking contract is publicly callable with no role-based access control. Because it writes `last_reward_block` before checking the `disable_rewards` flag, any attacker can call it once per block with `disable_rewards: true` to consume the block's reward slot without distributing any rewards. As long as the attack continues, no staker accrues block rewards.

---

### Finding Description
`update_rewards` is exposed through `IStakingRewardsManager` and is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

The function enforces a single call per block via `last_reward_block`: [2](#0-1) 

Critically, `last_reward_block` is written **before** the `disable_rewards` guard is evaluated: [3](#0-2) 

When `disable_rewards` is `true`, the function returns immediately after updating `last_reward_block`, skipping the entire `_update_rewards` call that accrues block rewards to the staker and its pools: [4](#0-3) 

Because `last_reward_block` is a **global** slot (not per-staker), consuming it with `disable_rewards: true` blocks every staker from receiving rewards for that block. No subsequent call can succeed for the same block number.

In the consensus-rewards phase, `update_rewards_from_attestation_contract` is explicitly forbidden: [5](#0-4) 

So `update_rewards` is the **only** mechanism for distributing block rewards in the consensus phase, making this the sole path an attacker needs to block.

---

### Impact Explanation
Every block the attacker calls `update_rewards(any_valid_staker, disable_rewards: true)`, all stakers and their delegation pools lose that block's STRK and BTC rewards. The reward supplier never transfers those tokens to the staking contract, and `unclaimed_rewards_own` is never incremented for any staker. Sustained over time this constitutes permanent freezing of unclaimed yield for the entire protocol.

This matches the allowed impact: **"Permanent freezing of unclaimed yield or unclaimed royalties"** (High) / **"Griefing with no profit motive but damage to users or protocol"** (Medium).

---

### Likelihood Explanation
The entry point is fully public — any non-zero EOA or contract can call it. The attacker only needs to supply any currently-active staker address (trivially obtained from on-chain events or `get_stakers`). On Starknet, gas costs are low, making sustained per-block calls economically feasible. No privileged key, bridge access, or external dependency is required.

---

### Recommendation
1. **Move `last_reward_block` write after the `disable_rewards` check.** If `disable_rewards` is `true`, do not consume the block slot; let a legitimate caller still distribute rewards for that block.
2. **Add access control.** Restrict `update_rewards` to a trusted caller (e.g., the attestation contract, a designated rewards-manager role, or the block proposer's operational address) so that the `disable_rewards` flag cannot be abused by arbitrary actors.

---

### Proof of Concept
```
// Attacker script — run once per block
loop {
    staking.update_rewards(
        staker_address = <any active staker>,
        disable_rewards = true,
    );
    // last_reward_block is now set to current block
    // _update_rewards is never called
    // All stakers earn zero block rewards for this block
    // Any legitimate call reverts with REWARDS_ALREADY_UPDATED
    wait_for_next_block();
}
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
