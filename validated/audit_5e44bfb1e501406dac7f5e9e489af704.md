### Title
Unprivileged Caller Can Freeze Consensus Rewards for All Stakers via `update_rewards(disable_rewards=true)` - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is callable by any non-zero address. It unconditionally writes the current block number to the global `last_reward_block` storage slot **before** checking the `disable_rewards` flag. An unprivileged attacker can call `update_rewards(any_valid_staker, disable_rewards=true)` every block to consume the single per-block reward slot without distributing any rewards, permanently starving all stakers of consensus-era yield.

### Finding Description

`update_rewards` enforces only `general_prerequisites` — contract-not-paused and caller-not-zero — before proceeding:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only: unpaused + non-zero caller
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.          ← written BEFORE the disable check
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;                            ← exits with no rewards distributed
    }
    ...
}
``` [1](#0-0) 

The global `last_reward_block` field is a single contract-wide slot: [2](#0-1) 

Because only one call per block can pass the `REWARDS_ALREADY_UPDATED` guard, whichever call arrives first in a block wins the slot. An attacker who calls `update_rewards(valid_staker, true)` first in every block:

1. Passes all guards (contract unpaused, caller non-zero, staker active, staker has non-zero balance).
2. Writes `last_reward_block = current_block`.
3. Returns immediately — no rewards are calculated or distributed.
4. Every subsequent legitimate call in that block reverts with `REWARDS_ALREADY_UPDATED`.

`general_prerequisites` imposes no role check: [3](#0-2) 

### Impact Explanation

All stakers and pool members in the consensus-rewards era (`is_pre_consensus() == false`) accumulate zero `unclaimed_rewards_own` for every block the attacker front-runs. Because `last_reward_block` is global, a single attacker transaction per block denies rewards to **every** staker simultaneously. If sustained, this constitutes permanent freezing of unclaimed yield — matching the **High** impact tier: *Permanent freezing of unclaimed yield or unclaimed royalties; Temporary freezing of funds*.

### Likelihood Explanation

Starknet blocks are produced at a predictable cadence. The attacker needs only one cheap transaction per block (no special tokens, no privileged key, no ETH collateral). The attack is economically rational for any party that benefits from suppressing staker rewards (e.g., a competing protocol, a short-seller of STRK, or a griever). The entry path is fully permissionless.

### Recommendation

1. **Restrict `disable_rewards=true` to a privileged role** (e.g., `security_agent` or `app_governor`). Unprivileged callers should only be able to call with `disable_rewards=false`.
2. **Move the `last_reward_block` write after the `disable_rewards` guard**, so a call that does not distribute rewards does not consume the block slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only update last_reward_block when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
```

3. Alternatively, track `last_reward_block` per-staker rather than globally, so one staker's slot cannot block another's.

### Proof of Concept

```
// Attacker script — run once per Starknet block:
staking_contract.update_rewards(
    staker_address = <any_valid_active_staker>,
    disable_rewards = true
)
// Result: last_reward_block = current_block, no rewards distributed.
// All legitimate update_rewards calls in this block revert with REWARDS_ALREADY_UPDATED.
// Repeat every block → all stakers earn zero consensus rewards indefinitely.
```

The attacker requires no stake, no privileged role, and no flash loan — only the ability to submit one transaction per block before any honest caller.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
