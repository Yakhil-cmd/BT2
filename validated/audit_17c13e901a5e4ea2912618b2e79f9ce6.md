### Title
Unprivileged Caller Can Permanently Freeze All Staker Block Rewards via `disable_rewards` Flag - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is callable by any unprivileged address and accepts a `disable_rewards: bool` parameter. An attacker can call this function with `disable_rewards: true` once per block to prevent all stakers from receiving block rewards for that block. Because `last_reward_block` is a single global storage variable, one such call per block is sufficient to block all reward distributions protocol-wide.

### Finding Description
`update_rewards` at line 1449 of `src/staking/staking.cairo` has no access control beyond `general_prerequisites()`, which only checks that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function accepts a `disable_rewards: bool` parameter. When called with `disable_rewards: true`, the execution path is:

1. The `REWARDS_ALREADY_UPDATED` guard passes (new block, `current_block_number > last_reward_block`).
2. The global `last_reward_block` is written to `current_block_number`.
3. The function returns early without distributing any rewards. [2](#0-1) 

`last_reward_block` is a single global variable, not per-staker: [3](#0-2) 

Because only one call to `update_rewards` is permitted per block (enforced by the `last_reward_block` check), an attacker who calls `update_rewards(any_valid_staker, true)` first in a block:

- Prevents the targeted staker from receiving block rewards.
- Prevents **every other staker** from receiving block rewards for that block, because `last_reward_block` is now equal to the current block and all subsequent calls revert with `REWARDS_ALREADY_UPDATED`.

This is the direct analog to the "malicious signer" pattern: just as the compromised signer in the reference report could unilaterally create orders and spend approvals without the maker's consent, here any unprivileged caller can unilaterally consume the protocol's single per-block reward-update slot and suppress reward distribution for all participants — no key compromise or privileged role required.

### Impact Explanation
**High — Permanent freezing of unclaimed yield.** If the attacker calls `update_rewards(any_staker, true)` at the start of every block, no staker in the protocol ever receives consensus block rewards. The rewards are never credited and are not recoverable. This affects all stakers once the protocol enters the consensus rewards phase (`is_pre_consensus()` returns `false`). [4](#0-3) 

### Likelihood Explanation
**High.** No special privileges are required — any externally owned address can call `update_rewards`. The attacker's only cost is gas per block. On Starknet, per-transaction fees are low enough to make this economically feasible for a motivated attacker (e.g., a competing validator, a protocol adversary, or anyone shorting STRK). The attack requires no coordination, no leaked keys, and no bridge compromise.

### Recommendation
Remove the `disable_rewards` parameter from the public interface entirely, or gate it behind a privileged role (e.g., `token_admin` or `security_agent`). If the parameter is needed for migration, it should be callable only by the staker themselves or their operational address, not by arbitrary callers. Additionally, consider making `last_reward_block` per-staker rather than global to prevent a single call from blocking all stakers simultaneously.

### Proof of Concept
1. The protocol is in consensus rewards mode (`consensus_rewards_first_epoch` is set and the current epoch has passed it, so `is_pre_consensus()` returns `false`).
2. Attacker submits `update_rewards(any_valid_staker_address, disable_rewards: true)` as the first transaction of each block.
3. `last_reward_block` is updated to the current block number; the function returns early with no rewards distributed.
4. Any legitimate call to `update_rewards(staker, false)` in the same block reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number == last_reward_block`.
5. No staker receives block rewards for that block.
6. Attacker repeats every block, permanently freezing all consensus block reward distributions across the entire protocol. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
