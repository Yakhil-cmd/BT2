### Title
Unprivileged Caller Can Freeze All Consensus Rewards via `update_rewards(disable_rewards: true)` — (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the Staking contract has no access control and writes to the global `last_reward_block` state variable **before** checking the `disable_rewards` flag. Any non-zero address can call `update_rewards(any_valid_staker, disable_rewards: true)` in any block, consuming the contract's single per-block reward-distribution slot without distributing any rewards. Because `last_reward_block` is a global singleton, every subsequent legitimate call in the same block reverts with `REWARDS_ALREADY_UPDATED`. An attacker repeating this every block permanently freezes all consensus-phase yield for all stakers.

---

### Finding Description

`IStakingRewardsManager::update_rewards` is a public, permissionless entry point:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();          // only checks: not paused, caller != zero
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // ← last_reward_block is written HERE, before the disable_rewards branch
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;   // ← exits without distributing anything
    }
    // actual reward distribution follows
``` [1](#0-0) 

`general_prerequisites` only enforces "not paused" and "caller is not zero": [2](#0-1) 

`last_reward_block` is a single global field, not per-staker: [3](#0-2) 

The interface documents the parameter but imposes no caller restriction: [4](#0-3) 

The ordering flaw is the root cause: `last_reward_block` is stamped with the current block number unconditionally, then the function checks `disable_rewards`. Because only one call per block is permitted globally, the attacker's no-op call exhausts the slot.

---

### Impact Explanation

An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block prevents every staker from ever accumulating consensus-phase rewards. This constitutes **permanent freezing of unclaimed yield** for all protocol participants. No funds already held by stakers are moved, but all future yield accrual is blocked indefinitely.

---

### Likelihood Explanation

The attack requires:
- No privileged role or special key.
- No capital at risk.
- Only gas to submit one transaction per block.

On Starknet, per-transaction fees are low enough that this attack is economically sustainable. The attacker needs only to ensure their transaction is included before any legitimate `update_rewards` call in each block. Because the attacker can submit proactively (without needing to observe a victim transaction), front-running is not even required — the attacker simply calls the function first in every block.

---

### Recommendation

1. **Restrict `disable_rewards: true` to a privileged role** (e.g., `only_security_agent` or `only_app_governor`), so unprivileged callers can only invoke the function with `disable_rewards: false`.
2. **Alternatively**, move `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` guard, so a no-op call does not consume the block's reward slot.
3. **Consider making `update_rewards` callable only by the block producer or a designated rewards-manager role**, consistent with the intent implied by the `IStakingRewardsManager` interface name.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker picks any valid, active staker address `S`.
3. In block `N`, attacker calls `update_rewards(S, disable_rewards: true)`.
   - `last_reward_block` is written to `N`.
   - Function returns early; no rewards distributed.
4. The legitimate block producer (or any honest caller) calls `update_rewards(S, disable_rewards: false)` in block `N`.
   - Assertion `current_block_number > self.last_reward_block.read()` fails (`N > N` is false).
   - Transaction reverts with `REWARDS_ALREADY_UPDATED`.
5. No staker receives rewards for block `N`.
6. Attacker repeats steps 3–5 for every subsequent block, permanently freezing all consensus yield.

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
