### Title
No `cancelPoolPriceProviderProposal` and No Emergency Bypass Leaves Pool Stuck for Full Timelock When Active Price Provider Becomes Permanently Broken - (`metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

The `MetricOmmPoolFactory` price-provider rotation flow (`proposePoolPriceProvider` → `executePoolPriceProviderUpdate`) has no cancel function and no emergency-bypass path. If the pool's currently-active price provider becomes permanently broken (oracle feed discontinued, Pyth/Chainlink feed delisted, provider contract bricked), every swap reverts immediately and the pool admin has no way to expedite switching to a working provider — they must wait through the full `priceProviderTimelock` before the new provider can be activated. During this window, swaps are completely blocked and, if `removeLiquidity` also reads the oracle (as is typical in bin-based oracle AMMs), LP principal is locked with no exit path.

### Finding Description

The two-step price-provider rotation is implemented in `MetricOmmPoolFactory`:

```
proposePoolPriceProvider  →  [timelock wait]  →  executePoolPriceProviderUpdate
```

`proposePoolPriceProvider` records `pendingPriceProvider[pool]` and `pendingPriceProviderExecuteAfter[pool] = block.timestamp + timelock`. [1](#0-0) 

`executePoolPriceProviderUpdate` enforces the timelock strictly and then calls `setPriceProvider` on the pool. [2](#0-1) 

The `IMetricOmmPoolFactoryPoolAdmin` interface exposes only `proposePoolPriceProvider` and `executePoolPriceProviderUpdate` — there is **no `cancelPoolPriceProviderProposal`** function, unlike the admin-transfer flow which has `cancelPoolAdminTransfer`. [3](#0-2) 

Every swap calls `_getBidAndAskPriceX64()`, which calls `IPriceProvider(activePriceProvider).getBidAndAskPrice()` inside a `try/catch` that re-reverts on any failure: [4](#0-3) 

The active provider is resolved from the immutable slot or the mutable `priceProvider` storage slot: [5](#0-4) 

`_validatePriceProvider` (called at both propose and execute time) only checks that `token0()`/`token1()` match — it does **not** verify that `getBidAndAskPrice()` returns a valid price: [6](#0-5) 

**Stuck scenario:**

1. Pool is live with provider `P_old`. Admin proposes `P_new` with a 7-day timelock.
2. Before the timelock elapses, `P_old`'s underlying oracle feed is permanently discontinued (e.g., Pyth removes the feed, Chainlink deprecates the aggregator). `P_old.getBidAndAskPrice()` now always reverts with `FeedStalled`.
3. Every `swap()` call reverts at `_getBidAndAskPriceX64()`. The pool is completely unusable.
4. The admin cannot cancel the pending proposal and immediately switch — calling `proposePoolPriceProvider` again with a working provider only **resets** the timelock to `now + timelock`, extending the stuck period.
5. There is no emergency bypass. The protocol owner can pause the pool (level 2), but pausing does not help LPs exit — it only prevents swaps.
6. The pool remains unusable for the full timelock duration. If `removeLiquidity` also reads the oracle price (standard in bin-based oracle AMMs where bin positions are oracle-relative), LP principal is locked with no exit.

### Impact Explanation

- **Broken core pool functionality**: All swaps revert for the entire timelock duration (potentially days or weeks).
- **Potential LP fund lock**: If `removeLiquidity` requires an oracle price read (consistent with the bin-based, oracle-anchored architecture), LP principal cannot be withdrawn during the stuck period.
- **No recovery path within the timelock**: The admin's only option is to re-propose (resetting the clock) or wait. Neither provides immediate relief.

### Likelihood Explanation

Oracle feeds are discontinued in practice (Chainlink deprecates aggregators, Pyth removes low-liquidity feeds). A pool with a long timelock (e.g., `type(uint256).max` for immutable mode is blocked entirely, but even a 7–30 day mutable timelock) creates a meaningful window of exposure. The trigger is an external event (feed discontinuation) combined with a missing cancel/emergency function — no privileged malicious action is required.

### Recommendation

1. **Add `cancelPoolPriceProviderProposal(address pool)`** (pool-admin only) that clears `pendingPriceProvider[pool]` and `pendingPriceProviderExecuteAfter[pool]`, mirroring `cancelPoolAdminTransfer`.
2. **Add a protocol-owner emergency override** that can execute a provider switch immediately (bypassing the timelock) when the pool is in a broken state, analogous to the Perennial fix that allows finalizing the previous provider after a grace period.
3. Ensure `removeLiquidity` does not require a live oracle price, so LPs can always exit even when the price provider is broken.

### Proof of Concept

```
// Setup: mutable pool with 7-day timelock, active provider P_old
// Step 1: Admin proposes P_new
factory.proposePoolPriceProvider(pool, P_new);
// pendingPriceProviderExecuteAfter[pool] = block.timestamp + 7 days

// Step 2: P_old's oracle feed is discontinued (external event)
// P_old.getBidAndAskPrice() now always reverts with FeedStalled

// Step 3: Any swap attempt reverts
pool.swap(...);  // reverts: PriceProviderFailed(FeedStalled)

// Step 4: Admin tries to cancel — NO SUCH FUNCTION
// factory.cancelPoolPriceProviderProposal(pool);  // does not exist

// Step 5: Admin re-proposes with P_working — resets timelock to now + 7 days
factory.proposePoolPriceProvider(pool, P_working);
// Pool remains unusable for another full 7 days

// Step 6: No emergency bypass exists; protocol owner can only pause (level 2),
// which does not allow LP withdrawal

  function proposePoolPriceProvider(address pool, address newPriceProvider)
    external
    override
    nonReentrant
    onlyPoolAdmin(pool)
  {
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    uint256 timelock = priceProviderTimelock[pool];
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, newPriceProvider);

    address mutableProvider = PoolStateLibrary._slot3(pool);
    address current = mutableProvider != address(0) ? mutableProvider : p.immutablePriceProvider;
    uint256 executeAfter = block.timestamp + timelock;
    pendingPriceProvider[pool] = newPriceProvider;
    pendingPriceProviderExecuteAfter[pool] = executeAfter;
    emit PoolPriceProviderChangeProposed(pool, current, newPriceProvider, executeAfter);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function executePoolPriceProviderUpdate(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPriceProvider[pool];
    if (pending == address(0)) revert NoPriceProviderChangeProposed();
    uint256 execAfter = pendingPriceProviderExecuteAfter[pool];
    // forge-lint: disable-next-line(block-timestamp) -- timelock enforcement legitimately relies on `block.timestamp`.
    if (block.timestamp < execAfter) revert PriceProviderTimelockNotElapsed(execAfter, block.timestamp);
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, pending);
    IMetricOmmPoolFactoryActions(pool).setPriceProvider(pending);
    delete pendingPriceProvider[pool];
    delete pendingPriceProviderExecuteAfter[pool];
    emit PoolPriceProviderUpdated(pool, pending);
  }

  function _validatePriceProvider(address token0, address token1, address priceProvider) internal view {
    if (priceProvider == address(0)) revert InvalidPriceProvider();
    if (IPriceProvider(priceProvider).token0() != token0 || IPriceProvider(priceProvider).token1() != token1) {
      revert PriceProviderTokenMismatch();
    }

  function _getBidAndAskPriceX64() internal returns (uint128 bidPriceX64, uint128 askPriceX64) {
    address activePriceProvider = _resolvedPriceProvider();
    try IPriceProvider(activePriceProvider).getBidAndAskPrice() returns (uint128 bid, uint128 ask) {
      if (bid >= ask) revert BidGreaterThanAsk();
      if (bid == 0) revert BidIsZero();
      return (bid, ask);
    } catch (bytes memory reason) {
      revert PriceProviderFailed(reason);
    }
  }
