### Title
Missing `msg_hash == 0` Validation in OT-Based ECDSA Signing Enables Private Key Extraction by Malicious Coordinator - (File: `src/ecdsa/ot_based_ecdsa/sign.rs`)

---

### Summary

The OT-based ECDSA `sign()` function accepts a zero `msg_hash` without rejection. The robust ECDSA counterpart explicitly rejects `msg_hash == 0` to prevent a split-view attack. The OT-based variant has no such guard. A malicious coordinator who reuses a presignature — once with `msg_hash = 0` and once with `msg_hash = h ≠ 0` — can algebraically isolate the nonce `k` and recover the aggregate private key `x`.

---

### Finding Description

`src/ecdsa/robust_ecdsa/sign.rs` explicitly rejects a zero message hash:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0 to avoid potential split view attacks".to_string(),
    ));
}
```

`src/ecdsa/ot_based_ecdsa/sign.rs` performs no equivalent check. The function accepts any `msg_hash: Scalar`, including zero, and proceeds to launch the protocol:

```rust
pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,          // ← zero is accepted without error
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    // ... participant/threshold checks only, no msg_hash == 0 guard
    Ok(make_protocol(ctx, fut))
}
```

The per-participant signature share is computed as:

```rust
fn compute_signature_share(..., msg_hash: Scalar) -> Result<Scalar, ProtocolError> {
    let lambda = participants.lagrange::<Secp256K1Sha256>(me)?;
    let k_i = lambda * presignature.k;
    let sigma_i = lambda * presignature.sigma;
    let r = x_coordinate(&presignature.big_r);
    Ok(msg_hash * k_i + r * sigma_i)   // ← when msg_hash = 0, k_i term vanishes
}
```

When `msg_hash = 0`, every participant's share collapses to `s_i = r · σ_i`, eliminating the nonce contribution `h · k_i`. The aggregate signature becomes `s₀ = r · σ = r · k · x`.

A malicious coordinator who reuses the same presignature in two sessions obtains:

| Session | msg_hash | Aggregate s |
|---------|----------|-------------|
| 1 | 0 | `s₀ = r · k · x` |
| 2 | h ≠ 0 | `sₕ = k⁻¹(h + r·x)` |

From session 1: `k · x = s₀ / r` (call this `u`).  
From session 2: `sₕ · k = h + r · x = h + r · u / k`, giving the quadratic `sₕ · k² − h · k − r · u = 0`.  
Solving for `k` and substituting back yields `x = u / k` — the aggregate private key.

Presignature reuse is not enforced in code; the README states it as a caller obligation only. The coordinator controls which presignature is passed to each participant and can trivially reuse one.

---

### Impact Explanation

**Critical.** A malicious coordinator can extract the aggregate private signing key `x` by:
1. Initiating a signing session with `msg_hash = 0` (accepted without error by the OT-based `sign()`) using a given presignature.
2. Initiating a second signing session with `msg_hash = h ≠ 0` using the same presignature.
3. Solving the resulting two-equation system to recover `k` and then `x`.

This maps directly to the allowed Critical impact: *"Extraction, reconstruction, or disclosure of private signing shares, aggregate secret material, presign secrets, nonce material, or confidential derived secrets."*

---

### Likelihood Explanation

**Medium.** The attack requires a malicious coordinator, which is a documented trust assumption in the library (the robust ECDSA scheme explicitly documents coordinator-level split-view attacks). The coordinator is a legitimate participant who controls message routing. Presignature reuse is not prevented in code. The only missing barrier is the `msg_hash == 0` guard that already exists in the robust ECDSA path but is absent from the OT-based path.

---

### Recommendation

Add the same zero-hash guard present in `src/ecdsa/robust_ecdsa/sign.rs` to `src/ecdsa/ot_based_ecdsa/sign.rs`:

```rust
if bool::from(msg_hash.is_zero()) {
    return Err(InitializationError::BadParameters(
        "msg_hash cannot be 0".to_string(),
    ));
}
```

This should be placed in the `sign()` initialization block alongside the existing participant and threshold checks, before `make_protocol` is called.

---

### Proof of Concept

**Setup**: 3 participants, threshold = 2, one honest presignature `(R, k_i, σ_i)`.

**Step 1 — Zero-hash session**: Coordinator calls `sign(participants, coordinator, threshold, me, pk, presignature.clone(), Scalar::ZERO)`. This succeeds (no error). Each participant sends `s_i^{(0)} = r · σ_i`. Coordinator aggregates: `s₀ = r · k · x`.

**Step 2 — Normal session**: Coordinator calls `sign(participants, coordinator, threshold, me, pk, presignature.clone(), h)` with the **same** presignature and `h ≠ 0`. Each participant sends `s_i^{(h)} = h · k_i + r · σ_i`. Coordinator aggregates: `sₕ = k⁻¹(h + r·x)`.

**Step 3 — Key recovery**:
- `u = s₀ / r = k · x`
- Solve `sₕ · k² − h · k − r · u = 0` for `k`
- `x = u / k`

The coordinator now holds the aggregate private key and can forge arbitrary signatures unilaterally.

pub fn sign(
    participants: &[Participant],
    coordinator: Participant,
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    public_key: AffinePoint,
    presignature: RerandomizedPresignOutput,
    msg_hash: Scalar,
) -> Result<impl Protocol<Output = SignatureOption>, InitializationError> {
    let threshold = usize::from(threshold.into());
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }

    let participants =
        ParticipantList::new(participants).ok_or(InitializationError::DuplicateParticipants)?;

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }

    // ensure number of participants during the signing phase is >= threshold
    if participants.len() < threshold {
        return Err(InitializationError::NotEnoughParticipantsForThreshold {
            threshold,
            participants: participants.len(),
        });
    }

    let ctx = Comms::new();
    let fut = fut_wrapper(
        ctx.shared_channel(),
        participants,
        coordinator,
        me,
        public_key,
        presignature,
        msg_hash,
    );
    Ok(make_protocol(ctx, fut))
}
