### Title
ERC-20 Token Burn Without Guaranteed NEP-141 Transfer in `ExitToEthereum` Precompile — (`engine-precompiles/src/native.rs`)

---

### Summary

The `ExitToEthereum` precompile burns ERC-20 tokens atomically within EVM execution, then schedules an asynchronous NEAR promise to call `withdraw` on the NEP-141 connector contract. No callback or refund mechanism exists. If the NEAR `withdraw` promise fails, the ERC-20 tokens are permanently destroyed while the user receives nothing on Ethereum — an exact structural analog to the reported "accounting updated, actual transfer skipped" insolvency pattern.

---

### Finding Description

When a user calls `withdrawToEthereum` on a mirrored ERC-20 contract (`EvmErc20.sol`), the following sequence occurs:

**Step 1 — ERC-20 burn (EVM, atomic):**
The Solidity contract calls `_burn(_msgSender(), amount)` before invoking the precompile. The EVM state is committed; the user's ERC-20 balance is permanently reduced.

**Step 2 — Promise creation (no callback):**
Inside `ExitToEthereum::run`, the precompile constructs a NEAR promise to call `withdraw` on the NEP-141 contract and serialises it as `PromiseArgs::Create`:

```rust
let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
``` [1](#0-0) 

There is no `PromiseArgs::Callback` wrapper, no `exit_to_ethereum_precompile_callback`, and no `refund` field — unlike the `ExitToNear` path which conditionally adds a callback under the `error_refund` feature:

```rust
#[cfg(feature = "error_refund")]
refund: refund_call_args(&exit_to_near_params, &exit_event),
#[cfg(not(feature = "error_refund"))]
refund: None,
``` [2](#0-1) 

**Step 3 — Silent failure:**
If the NEAR `withdraw` promise fails (NEP-141 connector paused, Aurora's NEP-141 balance insufficient, or any other runtime error), the NEAR runtime simply marks the promise as failed. No EVM state is reverted — the burn already committed. No callback fires to re-mint the tokens. The user's funds are gone.

The `ExitToNear` path acknowledges this risk and partially addresses it with the `error_refund` feature and `exit_to_near_precompile_callback`. The `ExitToEthereum` path has no equivalent protection at all. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

The user's ERC-20 tokens are irreversibly burned inside the EVM. If the corresponding NEAR `withdraw` promise fails, the user receives nothing on Ethereum and cannot recover the burned tokens. The NEP-141 balance held by the Aurora Engine contract remains unchanged (the `withdraw` reverted on the connector side), creating a permanent accounting divergence: ERC-20 supply decreases while NEP-141 holdings do not. The affected user's funds are permanently frozen with no recovery path.

---

### Likelihood Explanation

**Medium.**

The NEAR `withdraw` promise on the ETH connector can fail under realistic conditions:

1. **Connector paused** — The ETH connector contract supports administrative pausing for security incidents or upgrades. Any pause during an in-flight `withdrawToEthereum` call causes the promise to fail.
2. **Accounting discrepancy** — If the ERC-20 supply ever exceeds the NEP-141 balance held by Aurora (e.g., due to an admin mint without a corresponding deposit, or a prior bug), the connector's `withdraw` will fail for the last withdrawers.
3. **NEAR runtime gas exhaustion** — If `WITHDRAWAL_GAS` (100 TGas) is insufficient for the connector's logic in a future upgrade, the promise fails silently.

Any unprivileged EVM user who calls `withdrawToEthereum` is exposed to this risk whenever any of the above conditions hold. [4](#0-3) 

---

### Recommendation

Mirror the `error_refund` pattern from `ExitToNear` for the `ExitToEthereum` path:

1. Add a `exit_to_ethereum_precompile_callback` NEAR method that checks the promise result.
2. If the `withdraw` promise failed, re-mint the burned ERC-20 tokens to the original sender's address using `refund_on_error`.
3. Wrap the `withdraw_promise` in `PromiseArgs::Callback` pointing to this new callback, analogous to:

```rust
PromiseArgs::Callback(PromiseWithCallbackArgs {
    base: withdraw_promise,
    callback: PromiseCreateArgs {
        target_account_id: self.current_account_id.clone(),
        method: "exit_to_ethereum_precompile_callback".to_string(),
        args: borsh::to_vec(&refund_args).unwrap(),
        attached_balance: Yocto::new(0),
        attached_gas: costs::EXIT_TO_ETHEREUM_CALLBACK_GAS,
    },
})
``` [5](#0-4) [6](#0-5) 

---

### Proof of Concept

1. User holds 1000 units of a mirrored ERC-20 token on Aurora (backed by 1000 NEP-141 units held by the Aurora Engine contract).
2. User calls `withdrawToEthereum(recipient_eth_address, 1000)` on the ERC-20 contract.
3. `_burn(msg.sender, 1000)` executes — ERC-20 balance drops to 0. EVM state committed.
4. `ExitToEthereum` precompile fires, creating `PromiseArgs::Create(withdraw_promise)` targeting the NEP-141 connector's `withdraw` method.
5. The ETH connector is paused (or Aurora's NEP-141 balance is 0 due to a prior discrepancy).
6. The NEAR `withdraw` promise fails. No callback exists. No re-mint occurs.
7. User has 0 ERC-20 tokens, 0 ETH on Ethereum, and no recovery path. Funds are permanently frozen. [7](#0-6) [8](#0-7)

        //  - eth_recipient (20 bytes)

        //  - amount (32 bytes)

        let required_gas = Self::required_gas(input)?;

        }

        //  0x00 -> ETH (Base token) token transfer

        input = &input[1..];

            0x0 => {
                // ETH (base) transfer
                //

                //  eth_recipient (20 bytes) - the address of recipient which will receive ETH on Ethereum

pub fn refund_on_error<I: IO + Copy, E: Env, P: PromiseHandler>(
    io: I,
    env: &E,
    state: EngineState,
    args: &RefundCallArgs,
    handler: &mut P,
) -> EngineResult<SubmitResult> {
    let current_account_id = env.current_account_id();
    if let Some(erc20_address) = args.erc20_address {
        // ERC-20 exit; re-mint burned tokens
        let erc20_admin_address = current_address(&current_account_id);
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, erc20_admin_address, current_account_id, io, env);

        let refund_address = args.recipient_address;
        let amount = U256::from_big_endian(&args.amount);
        let input = setup_refund_on_error_input(amount, refund_address);

        engine.call(
            &erc20_admin_address,
            &erc20_address,
            Wei::zero(),
            input,
            u64::MAX,
            Vec::new(),
            Vec::new(),
            handler,
        )
    } else {
        // ETH exit; transfer ETH back from precompile address
        let exit_address = exit_to_near::ADDRESS;
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, exit_address, current_account_id, io, env);
        let refund_address = args.recipient_address;
        let amount = Wei::new(U256::from_big_endian(&args.amount));
        engine.call(
            &exit_address,
            &refund_address,
            amount,
            Vec::new(),
            u64::MAX,
            vec![
                (exit_address.raw(), Vec::new()),
                (refund_address.raw(), Vec::new()),
            ],
            Vec::new(),
            handler,
        )
    }

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
