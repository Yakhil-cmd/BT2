### Title
Participant-Set Mismatch Between `RerandomizationArguments` and `sign()` Enables Malicious Coordinator to Extract Private Key via Multiplicatively Related Nonces — (`src/ecdsa/ot_based_ecdsa/sign.rs`, `src/ecdsa/mod.rs`)

---

### Summary

The `sign()` function for both OT-based and Robust ECDSA accepts a `RerandomizedPresignOutput` without verifying that the `participants` list embedded in the `RerandomizationArguments` (used to derive the rerandomization scalar `delta`) matches the `participants` list passed to `sign()` (used to compute Lagrange coefficients). A malicious coordinator can present different participant sets to different signers across two signing sessions that share the same presignature, causing each session to produce a valid signature whose nonce is a known scalar multiple of the other. This multiplicative nonce relationship is sufficient to recover the aggregate private key using standard ECDSA nonce-reuse algebra.
