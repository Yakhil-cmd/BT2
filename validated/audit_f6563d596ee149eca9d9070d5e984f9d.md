### Title
Unprotected `new` Initializer Allows Any Caller to Seize Engine Ownership Before Legitimate Deployment - (File: `engine/src/contract_methods/admin.rs`)

### Summary
The Aurora Engine's `new` function, which sets the contract owner and chain configuration, performs no check on the caller's identity. Any unprivileged NEAR account can call `new` on a freshly deployed but uninitialized engine contract and install themselves as `owner_id`. Once owner, the attacker can invoke `upgrade` to redeploy arbitrary contract code or `attach_full_access_key` to gain a full-access key to the engine account, enabling complete theft of all bridged funds.

### Finding Description
`engine/src/contract_methods/admin.rs` exposes the `new` initializer:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input)...;
    // ...
    state::set_state(&mut io, &state)?;   // owner_id comes entirely from attacker-supplied args
    Ok(())
}
``` [1](#0-0) 

The only guard is the "already initialized" check on line 57. There is no assertion that `env.predecessor_account_id() == env.current_account_id()` (the standard NEAR pattern for restricting initialization to the deploying account). The `env` parameter is passed in but never queried for the caller's identity. [2](#0-1) 

The public WASM export in `engine/src/lib.rs` exposes this function with no additional guard:

```rust
#[unsafe(no_mangle)]
pub extern "C" fn new() {
    let io = Runtime;
    let env = Runtime;
    contract_methods::admin::new(io, &env)
        .map_err(ContractError::msg)
        .sdk_unwrap();
}
``` [3](#0-2) 

The deployment tooling in `engine-workspace/src/lib.rs` issues the deploy and the `new` call as **two separate transactions**, creating an observable window between them:

```rust
let contract = account.deploy(&self.code...).await?;
// ...
engine.new(self.chain_id, self.owner_id, self.upgrade_delay_blocks)
    .transact().await
``` [4](#0-3) 

Once the attacker owns the engine, two critical owner-only paths are available:

1. **`upgrade`** — deploys arbitrary WASM to the engine account, replacing all contract logic: [5](#0-4) 

2. **`attach_full_access_key`** — adds a full-access key to the engine account, giving the attacker unrestricted NEAR account control: [6](#0-5) 

Both are gated solely by `require_owner_only`: [7](#0-6) 

### Impact Explanation
**Critical — Direct theft of all user funds.**

The Aurora Engine account holds bridged ETH and NEP-141 tokens on behalf of all users. An attacker who seizes ownership can:
- Call `upgrade` to deploy a malicious contract that drains all token balances.
- Call `attach_full_access_key` to obtain a full-access key, then sign arbitrary transactions from the engine account (transfer NEAR, delete the account, etc.).

This satisfies the "Direct theft of any user funds" critical impact criterion.

### Likelihood Explanation
The deployment tooling (`engine-workspace/src/lib.rs`) issues deploy and `new` as separate async transactions. Any NEAR account monitoring the chain for a newly deployed Aurora Engine contract (e.g., by watching for the `DeployContract` action on the target account) can immediately submit a `new` call with attacker-controlled `owner_id` before the legitimate deployer's `new` transaction is included. NEAR transaction ordering within a block is deterministic but not guaranteed to favor the deployer when two transactions arrive in the same block from different accounts. The attack requires no privileged access, no leaked keys, and no social engineering — only the ability to submit a NEAR transaction.

### Recommendation
Add a caller-identity check at the top of `new` to ensure only the contract account itself can initialize:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // ...
}
```

Alternatively, bundle the `DeployContract` and `FunctionCall("new", ...)` actions into a single NEAR batch transaction in all deployment paths so there is no window between deployment and initialization.

### Proof of Concept

1. Attacker monitors the NEAR chain for a `DeployContract` action on the target Aurora Engine account (e.g., `aurora`).
2. In the same block (or the next), attacker submits:
   ```
   FunctionCall {
     receiver_id: "aurora",
     method_name: "new",
     args: NewCallArgs { chain_id: ..., owner_id: "attacker.near", upgrade_delay_blocks: 0 },
     ...
   }
   ```
3. Because `new` has no caller check and the state is still uninitialized, the call succeeds and `state.owner_id = "attacker.near"` is written to storage.
4. Attacker calls `attach_full_access_key` with their own public key:
   ```
   FunctionCall {
     receiver_id: "aurora",
     method_name: "attach_full_access_key",
     args: FullAccessKeyArgs { public_key: <attacker_key> },
     ...
   }
   ```
5. Attacker now holds a full-access key to the `aurora` account and can drain all bridged funds.

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
