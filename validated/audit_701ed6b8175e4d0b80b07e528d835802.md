### Title
Unbounded `native_price` in `BlockMetadataFromOracle` Bypasses Native Resource Accounting for All L2 Transactions - (File: `basic_bootloader/src/bootloader/block_flow/zk/metadata_op.rs`)

---

### Summary

`BlockMetadataFromOracle` is deserialized from oracle input and stored as block-level metadata with only `gas_limit` validated. The `native_price` field has no upper-bound check. When an oracle-data-influencing caller (sequencer) sets `native_price` to `U256::MAX` (or any value larger than any realistic `gas_price`), the derived `native_per_gas` collapses to `0`, which unconditionally triggers the `free_native = true` path in `create_resources_for_tx`, granting every L2 transaction in the block `u64::MAX - 1` native resources. Simultaneously, `native_per_pubdata` also collapses to `0`, making pubdata free. The native resource pricing mechanism ŌĆö the primary mechanism preventing transactions from consuming unbounded proving work ŌĆö is entirely bypassed for the block.

---

### Finding Description

**Step 1 ŌĆō Oracle value stored without upper-bound validation.**

In `metadata_op.rs`, `BlockMetadataFromOracle` is fetched from the oracle and stored directly. The only guard is on `gas_limit`:

```rust
if metadata.block_gas_limit() > MAX_BLOCK_GAS_LIMIT
    || metadata.individual_tx_gas_limit() > MAX_TX_GAS_LIMIT
{
    return Err(internal_error!("block or tx gas limit is too high"));
}
```

`native_price`, `pubdata_price`, `eip1559_basefee`, `pubdata_limit`, and `timestamp` are accepted without any bounds check. [1](#0-0) 

**Step 2 ŌĆō `native_price` is deserialized verbatim.**

`BlockMetadataFromOracle::from_iter` reads all fields from the oracle iterator and constructs the struct with no validation: [2](#0-1) 

**Step 3 ŌĆō `native_per_gas` collapses to zero.**

In `validate_and_compute_fee_for_transaction`, `native_per_gas` is computed as:

```rust
u256_try_to_u64(&gas_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::NativeResourcesAreTooExpensive))?
```

When `native_price = U256::MAX`, `gas_price.div_ceil(U256::MAX) = 0` for any realistic `gas_price`. `u256_try_to_u64(0)` succeeds and returns `Some(0)`, so `native_per_gas = 0` ŌĆö no error is raised. [3](#0-2) 

**Step 4 ŌĆō `native_per_pubdata` also collapses to zero.**

```rust
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`pubdata_price.wrapping_div(U256::MAX) = 0`, so `native_per_pubdata = 0`. All pubdata charges become `0 * pubdata_used = 0`. [4](#0-3) 

**Step 5 ŌĆō `free_native = true` grants unlimited native resources.**

`create_resources_for_tx` is called with `free_native = (native_per_gas == 0)`: [5](#0-4) 

Inside `create_resources_for_tx`:

```rust
let native_limit = if cfg!(feature = "unlimited_native") || free_native {
    u64::MAX - 1
} else {
    native_prepaid_from_gas
};
```

With `free_native = true`, `native_limit = u64::MAX - 1`. After subtracting intrinsic pubdata overhead (which is `0` because `native_per_pubdata_byte = 0`), the limit is capped at `MAX_NATIVE_COMPUTATIONAL` for computation, with the remainder (`u64::MAX - 1 - MAX_NATIVE_COMPUTATIONAL`) withheld for pubdata. After execution, withheld resources are reclaimed, giving the transaction access to the full `u64::MAX - 1` native budget. [6](#0-5) 

---

### Impact Explanation

Every L2 transaction in a block where `native_price = U256::MAX` receives:
- **Unlimited computational native resources** (capped only by `MAX_NATIVE_COMPUTATIONAL` per transaction, but with no payment required)
- **Free pubdata** (`native_per_pubdata = 0`, so `pubdata_used * 0 = 0` native charged)

The native resource pricing mechanism ŌĆö which is the primary control preventing transactions from consuming unbounded ZK-proving work ŌĆö is completely bypassed. A sequencer can fill a block with transactions that consume the maximum allowed proving resources per transaction, with no economic cost. This is a **resource accounting bug** with a secondary **valid-execution unprovability** risk: a block that executes correctly in forward mode may be prohibitively expensive or impossible to prove in proving mode if the prover enforces a native resource budget that the block now silently exceeds.

---

### Likelihood Explanation

The attack requires an oracle-data-influencing caller ŌĆö specifically the sequencer/operator who provides `BlockMetadataFromOracle` to the bootloader. This is an explicitly listed valid attacker class in the scope. The manipulation is a single-field change (`native_price = U256::MAX`) in the oracle response, requires no special cryptographic capability, and is not detectable by the bootloader's current validation logic.

---

### Recommendation

Add an upper-bound check on `native_price` (and analogously `pubdata_price`) in `metadata_op.rs`, immediately after the existing `gas_limit` check:

```rust
if metadata.block_gas_limit() > MAX_BLOCK_GAS_LIMIT
    || metadata.individual_tx_gas_limit() > MAX_TX_GAS_LIMIT
{
    return Err(internal_error!("block or tx gas limit is too high"));
}
// Add:
if metadata.native_price() > U256::from(MAX_NATIVE_PRICE) {
    return Err(internal_error!("native price is too high"));
}
```

where `MAX_NATIVE_PRICE` is a constant chosen such that `gas_price.div_ceil(MAX_NATIVE_PRICE) >= 1` for any valid `gas_price`. This mirrors the pattern already applied to `gas_limit` and prevents the `native_per_gas = 0` collapse. [7](#0-6) 

---

### Proof of Concept

1. Sequencer constructs oracle response with `native_price = U256::MAX` (all other fields normal).
2. `metadata_op.rs` deserializes `BlockMetadataFromOracle` ŌĆö no error, only `gas_limit` is checked.
3. For any L2 transaction with `max_fee_per_gas = 1000` and `eip1559_basefee = 1000`:
   - `gas_price = 1000`
   - `native_per_gas = 1000.div_ceil(U256::MAX) = 0` Ō£ō no error
   - `native_per_pubdata = pubdata_price.wrapping_div(U256::MAX) = 0`
   - `native_prepaid_from_gas = 0 * gas_limit = 0`
4. `create_resources_for_tx(free_native = true, ...)` ŌåÆ `native_limit = u64::MAX - 1`.
5. Transaction executes with `MAX_NATIVE_COMPUTATIONAL` computational native + unlimited withheld pubdata native, paying nothing for either.
6. Block-level pubdata limit check: `pubdata_used > system.get_pubdata_limit()` ŌĆö if `pubdata_limit` is also left at its default `u64::MAX`, this check never fires either. [8](#0-7) [9](#0-8)
