### Title
Unrestricted `update_rewards` Allows Any Caller to Consume the Global Block-Reward Slot Without Distributing Rewards - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager.update_rewards` has no access control and accepts a caller-controlled `disable_rewards` flag. Because the global `last_reward_block` is written **before** the `disable_rewards` guard, any address can call `update_rewards(any_active_staker, disable_rewards: true)` once per block to permanently consume the block's reward slot without distributing any rewards, blocking all stakers from earning consensus-phase yield indefinitely.

### Finding Description
In `StakingRewardsManagerImpl.update_rewards` the function:

1. Checks only `general_prerequisites()` — pause state and non-zero caller. No role or identity check exists.
2. Asserts `current_block_number > last_reward_block`, then **immediately** writes `last_reward_block = current_block_number`.
3. Only **after** that write does it check `if disable_rewards || self.is_pre_consensus() { return; }`. [1](#0-0) 

`last_reward_block` is a single global storage slot, not per-staker: [2](#0-1) 

Any caller who supplies `disable_rewards: true` with any valid active staker address will:
- Advance `last_reward_block` to the current block.
- Return without distributing any rewards.
- Cause every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [3](#0-2) 

There is no cap on how many blocks this can be repeated. An attacker can call this function in every block, indefinitely, at the cost of one transaction per block. The `general_prerequisites` guard provides no protection: [4](#0-3) 

The analog to the external report is direct: just as `voter_admin` could call `mint()` an unlimited number of times to inflate `VOTES.totalSupply()` and manipulate governance thresholds, here any caller can invoke `update_rewards(..., disable_rewards: true)` an unlimited number of times to inflate `last_reward_block` and block the reward-distribution threshold check (`current_block_number > last_reward_block`) from ever being satisfied by a legitimate caller.

### Impact Explanation
All stakers in the consensus-rewards phase are denied their per-block yield for as long as the attacker continues. This constitutes **griefing with no profit motive but direct, measurable damage to all protocol participants** (Medium impact). The attack is not bounded — there is no supply cap, rate limit, or cooldown on calling `update_rewards`.

### Likelihood Explanation
The function is publicly callable with no access control. The only cost to the attacker is one transaction per block. No special privilege, leaked key, bridge compromise, or external dependency is required. Any address with sufficient gas can execute this attack continuously.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the staker's registered operational address, or a dedicated consensus contract address stored in the staking contract). Alternatively, move `self.last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard so that a call with `disable_rewards: true` does not consume the block's reward slot. [5](#0-4) 

### Proof of Concept
1. Consensus rewards are active (`is_pre_consensus() == false`).
2. Attacker identifies any active staker `S` with non-zero STRK balance (trivially discoverable from on-chain events).
3. In every block `B`, attacker calls `update_rewards(staker_address: S, disable_rewards: true)`.
4. `last_reward_block` is set to `B`; no rewards are distributed.
5. Any legitimate call to `update_rewards` in block `B` reverts with `REWARDS_ALREADY_UPDATED`.
6. Stakers earn zero consensus-phase rewards for every block the attacker maintains the attack.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
