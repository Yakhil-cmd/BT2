### Title
Missing Access Control on `update_rewards` Allows Any Caller to Freeze or Selectively Steal Staker Yield - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `staking.cairo` is documented in the spec as callable only by the Starkware sequencer, but the implementation contains **no caller access control**. Any unprivileged address can call it. Because `last_reward_block` is a single global variable (not per-staker), only one `update_rewards` call succeeds per block. An attacker can exploit this to either (a) permanently freeze all staker yield by calling with `disable_rewards: true` every block, or (b) selectively drain yield toward their own staker while starving all others.

### Finding Description

`update_rewards` is the sole mechanism for distributing block rewards to stakers and their delegation pools in the consensus-rewards phase. Its only guard is a global block-number check:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
// ...
self.last_reward_block.write(current_block_number);
``` [1](#0-0) [2](#0-1) 

After updating `last_reward_block`, the function checks `disable_rewards` and returns early without distributing anything if it is `true`:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [3](#0-2) 

The spec explicitly states the access control requirement:

> **access control**: Only starkware sequencer. [4](#0-3) 

But the implementation has **no `assert_caller_is_sequencer` or equivalent check** anywhere in the `StakingRewardsManagerImpl` block. Compare this with `update_rewards_from_attestation_contract`, which correctly enforces `assert_caller_is_attestation_contract()`: [5](#0-4) 

Because `last_reward_block` is a single global storage slot shared across all stakers, a successful call for any staker at block N blocks every other staker from receiving rewards at block N. [6](#0-5) 

### Impact Explanation

**Attack vector A – Permanent freezing of all unclaimed yield (High):**
An attacker calls `update_rewards(any_active_staker, disable_rewards: true)` in every block. Each call:
1. Passes the block-number guard and writes `last_reward_block = current_block`.
2. Returns immediately without distributing any rewards.
3. Prevents the legitimate sequencer from calling `update_rewards` for the same block (it reverts with `REWARDS_ALREADY_UPDATED`).

Repeated every block, this permanently prevents any staker or delegator from ever accumulating yield in the consensus-rewards phase.

**Attack vector B – Theft of other stakers' unclaimed yield (High):**
An attacker who is also a staker calls `update_rewards(attacker_staker, disable_rewards: false)` in every block. Each call distributes that block's rewards exclusively to the attacker's staker and its pools. All other stakers receive zero rewards for those blocks. Their proportional share is permanently lost (never requested from L1, never credited).

Both impacts are reachable with zero capital beyond gas and a valid staker registration. [7](#0-6) 

### Likelihood Explanation

- `update_rewards` is a public ABI function with no role gate.
- The attack requires only a valid `staker_address` argument (any active staker suffices for vector A; the attacker's own staker for vector B).
- It can be automated trivially: submit one transaction per block.
- No privileged key, bridge compromise, or external dependency is needed.
- The consensus-rewards phase is already live (controlled by `consensus_rewards_first_epoch`), so the precondition `!is_pre_consensus()` is satisfied in production. [8](#0-7) 

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, analogous to the existing `assert_caller_is_attestation_contract` pattern used in `update_rewards_from_attestation_contract`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // <-- add this
    ...
}
```

The sequencer address should be stored in contract storage and set during initialization or via a governance-controlled setter, consistent with the existing roles architecture. [9](#0-8) 

### Proof of Concept

```cairo
// Any unprivileged address can execute this every block:
fn grief_all_rewards(staking: IStakingRewardsManagerDispatcher, any_staker: ContractAddress) {
    // Vector A: freeze all yield
    // Call once per block; no special role needed.
    staking.update_rewards(staker_address: any_staker, disable_rewards: true);
    // last_reward_block is now set to current block.
    // The sequencer's subsequent call for the same block reverts with REWARDS_ALREADY_UPDATED.
    // No staker receives any yield for this block.
}

fn steal_other_stakers_yield(
    staking: IStakingRewardsManagerDispatcher, attacker_staker: ContractAddress
) {
    // Vector B: attacker calls every block for their own staker
    staking.update_rewards(staker_address: attacker_staker, disable_rewards: false);
    // Attacker's staker receives full block rewards proportional to their stake.
    // All other stakers receive zero for this block.
}
```

The `test_update_rewards_same_epoch_flow_test` integration test already demonstrates that `update_rewards` is callable by an arbitrary address without any `cheat_caller_address` setup, confirming the absence of access control. [10](#0-9)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

#[test]
fn test_update_rewards_same_epoch_flow_test() {
    let cfg: StakingInitConfig = Default::default();
    let mut system = SystemConfigTrait::basic_stake_flow_cfg(:cfg).deploy();
    system.advance_epoch();
    let min_stake = system.staking.get_min_stake();
    let stake_amount = min_stake * 2;
    let staker = system.new_staker(amount: stake_amount);
    let commission = 200;
    system.stake(:staker, amount: stake_amount, pool_enabled: true, :commission);
    system.start_consensus_rewards();
    let pool = system.staking.get_pool(:staker);
    let delegator = system.new_delegator(amount: stake_amount);
    system.delegate(:delegator, :pool, amount: stake_amount);
    let btc_pool = system
        .set_open_for_delegation(:staker, token_address: system.btc_token.contract_address());
    let btc_amount = TEST_MIN_BTC_FOR_REWARDS;
    let delegator = system.new_btc_delegator(amount: btc_amount, token: system.btc_token);
    system.delegate_btc(:delegator, pool: btc_pool, amount: btc_amount, token: system.btc_token);
    let epoch_length = system.staking.get_epoch_info().epoch_len_in_blocks();
    let mut blocks_till_next_epoch = epoch_length.into();
    system.update_rewards(:staker, disable_rewards: false);
    let mut staker_rewards = system.staker_claim_rewards(:staker);
    // Same epoch - same rewards.
    advance_blocks(blocks: 1, block_duration: AVG_BLOCK_DURATION);
    blocks_till_next_epoch -= 1;
    system.update_rewards(:staker, disable_rewards: false);
    assert!(system.staker_claim_rewards(:staker) == staker_rewards);
    advance_blocks(blocks: 100, block_duration: AVG_BLOCK_DURATION);
    blocks_till_next_epoch -= 100;
    system.update_rewards(:staker, disable_rewards: false);
    assert!(system.staker_claim_rewards(:staker) == staker_rewards);
