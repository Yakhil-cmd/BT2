### Title
`protocolUnpausePool` always transitions to admin-paused state (1) regardless of pre-pause state, leaving originally-active pools swap-blocked after protocol unpause — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`protocolUnpausePool` unconditionally sets the pool's pause level to `1` (admin-paused) when removing a protocol pause, even when the pool was in state `0` (fully active) before the protocol pause was applied. After a protocol pause-unpause cycle on an active pool, swaps remain blocked until the pool admin separately calls `unpausePool`.

---

### Finding Description

The protocol defines three pause levels:

| Level | Meaning |
|---|---|
| `0` | Active (swaps allowed) |
| `1` | Admin-paused (swaps blocked) |
| `2` | Protocol-paused (swaps blocked) |

`protocolPausePool` accepts pools in **either** state `0` or state `1` and transitions them to state `2`:

```solidity
// MetricOmmPoolFactory.sol L393-L396
function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}
```

`protocolUnpausePool` **always** transitions to state `1`, never to state `0`:

```solidity
// MetricOmmPoolFactory.sol L399-L403
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1);
}
```

The function handles only one pre-pause variant correctly:

- **Case A (handled correctly):** Pool was at state `1` (admin-paused) → protocol pauses → state `2` → protocol unpauses → state `1`. Correct: pool returns to its prior state.
- **Case B (not handled):** Pool was at state `0` (active) → protocol pauses → state `2` → protocol unpauses → state `1`. **Wrong:** pool is now admin-paused even though the admin never paused it.

The `swap` function carries `whenNotPaused`, which reverts on any non-zero pause level:

```solidity
// MetricOmmPool.sol L224
function swap(...) external whenNotPaused nonReentrant(PoolActions.SWAP) ...
```

```solidity
// MetricOmmPool.sol L643-L645
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
```

After the protocol unpause, the pool is at level `1`, so `pauseLevel != 0` is true and every swap reverts.

---

### Impact Explanation

After a protocol pause-unpause cycle on a pool that was originally active (state `0`), the pool is left in admin-paused state (state `1`). All swap calls revert with `PoolPaused`. Liquidity providers cannot trade, and the protocol and admin accrue no spread or notional fees from swaps. The pool admin must issue a separate `unpausePool` transaction to restore swap functionality — an action that was never required before the protocol intervened. If the admin key is a multisig with delayed execution, unavailable signers, or a contract without the `unpausePool` call path, the pool can remain swap-blocked indefinitely after the protocol owner believed it was restored.

---

### Likelihood Explanation

The trigger is a routine, privileged-but-valid operation: the protocol owner pauses an active pool (e.g., during an incident) and then unpauses it. No malicious setup is required. The protocol owner is semi-trusted and this is an expected operational flow. Every such pause-unpause cycle on an active pool reproduces the issue deterministically.

---

### Recommendation

Track the pre-pause state before applying the protocol pause, and restore it on unpause. One approach:

```solidity
mapping(address => uint8) public prePauseLevel; // new state variable

function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    prePauseLevel[pool] = cur;  // save pre-pause state
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}

function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    uint8 restoreLevel = prePauseLevel[pool]; // 0 or 1
    delete prePauseLevel[pool];
    IMetricOmmPoolFactoryActions(pool).setPause(restoreLevel);
}
```

---

### Proof of Concept

1. Pool is deployed and active: `pauseLevel == 0`.
2. Protocol owner calls `protocolPausePool(pool)`. Check passes (`cur == 0`). Pool transitions to `pauseLevel == 2`.
3. Protocol owner calls `protocolUnpausePool(pool)`. Check passes (`cur == 2`). Pool transitions to `pauseLevel == 1`.
4. Any user calls `pool.swap(...)`. `whenNotPaused` fires: `pauseLevel != 0` → `revert PoolPaused()`.
5. Pool admin must call `factory.unpausePool(pool)` to restore state `0`. Until then, all swaps revert. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
