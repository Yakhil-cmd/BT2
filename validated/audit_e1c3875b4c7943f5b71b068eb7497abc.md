### Title
WalletConnect Offer Royalty Amount Understated for Multi-NFT Offers - (`packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary

The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts` incorrectly divides the fungible amount by the number of NFTs before applying each NFT's royalty percentage. This causes the `amountWithRoyalties` field shown in the WalletConnect confirmation dialog to be systematically understated when a user accepts an offer containing multiple NFTs with royalties.

### Finding Description

In `formatAmountWithRoyalties` (lines 354–375), the royalty calculation is:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length); // ← divides by NFT count
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
``` [1](#0-0) 

This computes: `royaltyAmount = Σ((amount / N) × royaltyPct_i / 10_000)`, where N is the number of NFTs.

The correct formula is: `royaltyAmount = Σ(amount × royaltyPct_i / 10_000)`.

The existing test at line 320 confirms the buggy output is treated as expected: [2](#0-1) 

**Concrete proof:**
- amount = 100,000,000 mojos (0.0001 XCH)
- NFT1 royalty: 500 (5%), NFT2 royalty: 10 (0.1%)

| | Buggy | Correct |
|---|---|---|
| royaltyAmount | 2,550,000 mojos | 5,100,000 mojos |
| `amountWithRoyalties` shown | 0.00010255 XCH | 0.0001051 XCH |

With higher royalties (e.g., two NFTs each at 50%), a user buying 1 XCH worth of NFTs would see 1.5 XCH total but actually spend 2 XCH — a 33% understatement.

The `amountWithRoyalties` value flows directly into the WalletConnect confirmation dialog via `walletDeltaToDisplay` → `withRoyaltyTotals` → `Confirm.tsx`: [3](#0-2) [4](#0-3) 

### Impact Explanation

A user accepting a WalletConnect `chia_wallet.take_offer` request for an offer containing multiple NFTs with royalties is shown an understated total cost. The actual XCH/CAT deducted from their wallet is higher than what the confirmation dialog displays. This causes the user to approve a transaction they may not have approved had they seen the correct amount — a direct wallet accounting impact.

### Likelihood Explanation

Any dApp using WalletConnect to present a multi-NFT offer triggers this path. The bug is proportional to the number of NFTs and their royalty percentages. NFT royalties up to 100% are technically possible in Chia. An attacker controlling NFT royalty addresses can craft offers with multiple high-royalty NFTs to maximize the discrepancy between displayed and actual cost.

### Recommendation

Remove the `splitAmount` division. Apply each royalty percentage to the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

### Proof of Concept

1. Attacker creates two NFTs with royalty percentages of 5000 (50%) each, with royalty addresses they control.
2. Attacker creates an offer: sell both NFTs for 1 XCH.
3. Attacker sends a WalletConnect `chia_wallet.take_offer` request to a victim's wallet.
4. Victim's WalletConnect confirmation dialog shows `amountWithRoyalties: 1.5 XCH` (buggy: `(1 XCH / 2) × 50% × 2 + 1 XCH`).
5. Correct total is 2 XCH (`1 XCH × 50% × 2 + 1 XCH`).
6. Victim approves, expecting to spend 1.5 XCH, but 2 XCH is deducted — 0.5 XCH more than displayed. [5](#0-4)

function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}

function withRoyaltyTotals(
  items: DisplayWalletDeltaItemWithKey[],
  amounts: Record<string, bigint>,
  oppositeSideLines: DisplayWalletDeltaItem[],
): DisplayWalletDeltaItem[] {
  const royaltyPercentages = royaltyPercentagesForSide(oppositeSideLines);

  return items.map(({ key, line }) => {
    if (line.kind === 'nft') {
      return line;
    }

    const amount = amounts[key];
    const amountWithRoyalties = amount ? formatAmountWithRoyalties(line, amount, royaltyPercentages) : undefined;

    return amountWithRoyalties ? { ...line, amountWithRoyalties } : line;
  });
}

      walletDelta: await walletDeltaToDisplay(walletDelta, walletInfos, assetKinds, royaltyPercentages, fees),
