### Title
Slot-Level Timestamp in `CompressedOracleV1` Causes Per-Feed Staleness Check to Be Bypassed — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1` packs up to 4 feeds into a single 256-bit storage slot with **one shared timestamp** for all lanes. When any feed in the slot is updated, the slot timestamp is refreshed for all four feeds simultaneously. The `_price()` function returns this slot-level timestamp as `refTime`, which is then used by `PriceProvider` (and its variants) for the staleness check. A feed whose price data has not changed in hours will appear fresh to the staleness guard the moment any co-slotted feed is pushed with a new timestamp, allowing a stale bid/ask quote to reach pool swaps.

### Finding Description

**Root cause — shared slot timestamp:**

`CompressedOracleV1` stores one 56-bit timestamp at bits `[63:8]` of the 256-bit slot, shared across all four 48-bit feed lanes:

```
bits 255…208 : oracle[0] (48 bits)
bits 207…160 : oracle[1] (48 bits)
bits 159…112 : oracle[2] (48 bits)
bits 111… 64 : oracle[3] (48 bits)
bits  63…  8 : timestamp (uint56, unix milliseconds)   ← ONE timestamp for ALL four feeds
bits   7…  0 : reserved
```

`_loadSlotLayout` reads this single timestamp and assigns it to the entire layout:

```solidity
_layout.timestampMs = toTimeMs(slotValue >> 8 & X56);
```

`getOracleData` then copies the slot-level timestamp into the per-feed `OracleData` struct:

```solidity
data.timestampMs = _layout.timestampMs;   // slot-level, NOT per-feed
```

`_price()` returns it as `refTime`:

```solidity
return (uint256(data.price), uint256(data.spread0), data.spread1, data.timestampMs.toSeconds());
```

**Staleness check in `PriceProvider._getBidAndAskPrice()`:**

```solidity
(uint256 mid, uint256 spread, , uint256 refTime) =
    IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
    return (0, type(uint128).max);
}
```

**The push path overwrites the entire slot atomically.** The docs explicitly state: *"The entire slot is overwritten on update: if you update a single lane, you must still supply correct values for the other lanes."* This means a pusher who has fresh data for feed A but stale data for feed B must still supply feed B's old price bytes alongside a new, current timestamp. After the write, `refTime` for feed B equals the new slot timestamp — the staleness guard passes even though feed B's price bytes are hours old.

**Concrete scenario:**

1. Slot contains feed A (position 0, BTC/USD) and feed B (position 1, ETH/USD).
2. Feed B's data source goes offline; its price bytes have not changed for `MAX_TIME_DELTA + 1` seconds.
3. Feed A's data source is live; the pusher pushes a new slot word with fresh BTC/USD data and `timestamp = now`.
4. The slot is overwritten: feed B's price bytes are unchanged, but the slot timestamp is now `now`.
5. `CompressedOracleV1.price(feedB, pool)` returns `refTime = now`.
6. `_isStale(now, block.timestamp, MAX_TIME_DELTA)` → `false` — staleness check passes.
7. The pool executes a swap at feed B's stale price.

### Impact Explanation

A stale bid/ask quote reaches the pool swap path. Depending on how far the real market has moved from the stale price, traders can extract value from LPs (receiving more output than the current oracle price permits) or the pool can settle trades at incorrect prices, causing LP principal loss. This directly satisfies the "bad-price execution: stale bid/ask quote reaches a pool swap" impact gate.

### Likelihood Explanation

The trigger requires no privileged or malicious action. Any honest pusher who has live data for one feed in a slot but stale data for another will inadvertently refresh the slot timestamp for the stale feed on every normal push. This is a structural property of the slot-level timestamp design and will occur in any production deployment where co-slotted feeds have different update cadences or where one data source temporarily goes offline.

### Recommendation

Replace the single slot-level timestamp with per-lane timestamps. Each 48-bit lane currently uses `32 + 8 + 8 = 48` bits (`p`, `s0`, `s1`). One option is to reduce price precision slightly and embed a per-lane timestamp delta, or to use a second storage slot for per-feed timestamps. Alternatively, enforce at the push layer that a lane whose price bytes are unchanged must carry the stall marker (`s0 = 0xFF, s1 = 0xFF`) rather than the old price, so the consumer's spread check (`spread >= ORACLE_BPS`) halts the quote independently of the timestamp.

### Proof of Concept

```solidity
// Scenario: two feeds share slot 0; feed B goes stale while feed A stays live.
// After a normal push for feed A, feed B's staleness check incorrectly passes.

bytes32 feedA = oracle.feedIdOf(creator, 0, 0); // position 0
bytes32 feedB = oracle.feedIdOf(creator, 0, 1); // position 1 — SAME slot

// T=0: both feeds are fresh
pushSlot(creator, slotId=0, priceA=100e8, priceB=50e8, timestamp=T0);

// T = MAX_TIME_DELTA + 1: feed B's source goes offline; feed A is still live.
// Pusher pushes fresh data for feed A, keeping feed B's bytes unchanged.
// The slot timestamp is updated to T0 + MAX_TIME_DELTA + 1.
pushSlot(creator, slotId=0, priceA=101e8, priceB=50e8, timestamp=T0+MAX_TIME_DELTA+1);

// PriceProvider backed by CompressedOracleV1 reads feed B:
// refTime = T0 + MAX_TIME_DELTA + 1  (the slot timestamp, NOT feed B's last update time)
// _isStale(refTime, block.timestamp, MAX_TIME_DELTA) → false  ← BUG: should be true
// Pool executes swap at feed B's stale price of 50e8
```

The exact corrupted value is `refTime` returned by `CompressedOracleV1._price(feedB)`: it equals the slot's last-write timestamp rather than the timestamp of feed B's last actual price change, causing `_isStale` in `PriceProvider._getBidAndAskPrice()` to return `false` for a price that is `MAX_TIME_DELTA + 1` seconds old.     function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)
        }

        _layout.timestampMs = toTimeMs(slotValue >> 8 & X56);

        _layout.oracle0 = _decodeCompressedOracleData(uint48((slotValue >> 208) & X48));
        _layout.oracle1 = _decodeCompressedOracleData(uint48((slotValue >> 160) & X48));
        _layout.oracle2 = _decodeCompressedOracleData(uint48((slotValue >> 112) & X48));
        _layout.oracle3 = _decodeCompressedOracleData(uint48((slotValue >> 64) & X48));
    }
