### Title
`Declare` Transactions Bypass `max_l2_gas_amount` Admission Control in `StatelessTransactionValidator` — (File: crates/apollo_gateway/src/stateless_transaction_validator.rs)

### Summary
`StatelessTransactionValidator::validate_resource_bounds` enforces an upper bound (`max_l2_gas_amount`) on the L2 gas amount for `Invoke` and `DeployAccount` transactions but explicitly skips this check for `Declare` transactions. Any user can submit a `Declare` transaction with an arbitrarily large `l2_gas.max_amount`, bypassing the gateway's admission control for this resource bound. The stateful path has no compensating amount check, so a `Declare` with a moderately oversized gas claim (backed by sufficient account balance) is admitted to the mempool.

### Finding Description
In `validate_resource_bounds`, lines 78–85 of `crates/apollo_gateway/src/stateless_transaction_validator.rs`:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← no check at all
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

The default `max_l2_gas_amount` is `1_210_000_000` (production config, `crates/apollo_gateway_config/src/config.rs` line 193). A `Declare` transaction can set `l2_gas.max_amount` to any `u64` value without triggering this guard.

The stateful validator's `validate_resource_bounds` (`crates/apollo_gateway/src/stateful_transaction_validator.rs` lines 223–243) calls only `validate_tx_l2_gas_price_within_threshold`, which checks the L2 gas *price*, not the *amount*. The only downstream check that could catch an oversized amount is `verify_can_pay_committed_bounds` (`crates/blockifier/src/fee/fee_utils.rs` lines 173–203), which rejects the transaction only when `max_amount × max_price_per_unit` exceeds the account balance. For a moderate overage — e.g., `max_amount = 2_000_000_000` with `max_price_per_unit = 8_000_000_000` (the minimum allowed by `min_gas_price`) — the committed fee is `16 × 10^18` FRI. An account holding that balance passes the check and has the transaction admitted to the mempool.

The existing test `valid_l2_gas_amount_on_declare` (`crates/apollo_gateway/src/stateless_transaction_validator_test.rs` lines 173–201) explicitly confirms the bypass: a `Declare` with `max_amount: GasAmount(200)` when `max_l2_gas_amount: 100` is asserted to pass with `Ok(())`.

The analog to the external report is exact:
| External report | Sequencer |
|---|---|
| `veRAACToken::lock` has `MAX_TOTAL_SUPPLY` check | `Invoke`/`DeployAccount` have `max_l2_gas_amount` check |
| `veRAACToken::increase` skips the check | `Declare` skips the check (explicit `if let … {}` no-op) |
| Any user can call `increase` | Any user can submit a `Declare` |
| Total supply exceeds cap | `l2_gas.max_amount` exceeds the gateway cap |

### Impact Explanation
A `Declare` transaction with `l2_gas.max_amount` above `max_l2_gas_amount` is admitted through stateless validation and, if the account balance covers the committed fee, through stateful validation and into the mempool. The gateway's per-transaction L2 gas amount admission control is broken for the `Declare` transaction type. This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation
The trigger is fully unprivileged: any account that can afford the committed fee (`max_amount × min_gas_price`) can submit a `Declare` with an oversized gas claim. No special role or privileged access is required. The TODO comment in the source confirms the developers are aware the check is absent and have not yet resolved it.

### Recommendation
Remove the `Declare`-specific exemption and apply the `max_l2_gas_amount` check uniformly:

```diff
-        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
-        if let RpcTransaction::Declare(_) = tx {
-        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
+        if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
             return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                 gas_amount: resource_bounds.l2_gas.max_amount,
                 max_gas_amount: self.config.max_l2_gas_amount,
             });
         }
```

If `Declare` transactions legitimately require a higher L2 gas budget (e.g., for compilation overhead), introduce a separate `max_l2_gas_amount_declare` config parameter rather than removing the bound entirely.

### Proof of Concept
The existing test already demonstrates the bypass:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator_test.rs lines 173-201
fn valid_l2_gas_amount_on_declare(config, rpc_tx_args) {
    // config: max_l2_gas_amount = 100
    // rpc_tx_args: l2_gas.max_amount = GasAmount(200)  ← 2× the limit
    let tx_type = TransactionType::Declare;
    let tx_validator = StatelessTransactionValidator { config };
    let tx = rpc_tx_for_testing(tx_type, rpc_tx_args);
    assert_matches!(tx_validator.validate(&tx), Ok(()));  // passes — no error
}
```

To confirm end-to-end admission: submit a `Declare` RPC transaction with `l2_gas.max_amount = 2_000_000_000` (default limit is `1_210_000_000`) from an account holding at least `2_000_000_000 × 8_000_000_000 = 16 × 10^18` FRI. pub fn verify_can_pay_committed_bounds(
    state: &mut dyn StateReader,
    tx_context: &TransactionContext,
) -> TransactionFeeResult<()> {
    let tx_info = &tx_context.tx_info;
    let committed_fee = tx_context.max_possible_fee();
    let (balance_low, balance_high, can_pay) =
        get_balance_and_if_covers_fee(state, tx_context, committed_fee)?;
    if can_pay {
        Ok(())
    } else {
        Err(match tx_info {
            TransactionInfo::Current(context) => match &context.resource_bounds {
                L1Gas(l1_gas) => TransactionFeeError::GasBoundsExceedBalance {
                    resource: Resource::L1Gas,
                    max_amount: l1_gas.max_amount,
                    max_price: l1_gas.max_price_per_unit,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
                AllResources(bounds) => TransactionFeeError::ResourcesBoundsExceedBalance {
                    bounds: *bounds,
                    balance: balance_to_big_uint(&balance_low, &balance_high),
                },
            },
            TransactionInfo::Deprecated(context) => TransactionFeeError::MaxFeeExceedsBalance {
                max_fee: context.max_fee,
                balance: balance_to_big_uint(&balance_low, &balance_high),
            },
        })
    }
}
