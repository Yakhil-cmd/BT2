### Title
`EVM.mustRun()` Does Not Revert on `Status.failed`, Allowing Silent EVM Token Transfer Failures - (`fvm/evm/stdlib/contract.cdc`)

---

### Summary

`EVM.run()` and `EVM.mustRun()` both return an `EVM.Result` struct whose `status` field must be manually checked by the caller. `EVM.mustRun()` explicitly permits `Status.failed` outcomes to pass through without reverting the enclosing Cadence transaction. Any Cadence transaction or contract that uses these functions for EVM token transfers without asserting `res.status == EVM.Status.successful` will silently accept a failed transfer, producing a cross-VM asset accounting discrepancy.

---

### Finding Description

`EVM.run()` in `fvm/evm/stdlib/contract.cdc` is an `access(all)` function callable by any unprivileged Cadence transaction author. It executes an RLP-encoded EVM transaction and returns an `EVM.Result` value. The `EVM.Result` struct exposes a `status` field that can be one of four enum cases: `unknown`, `invalid`, `failed`, or `successful`.

`EVM.mustRun()`, also `access(all)`, is documented as a "safer" wrapper. Its implementation is:

```cadence
fun mustRun(tx: [UInt8], coinbase: EVMAddress): Result {
    let runResult = self.run(tx: tx, coinbase: coinbase)
    assert(
        runResult.status == Status.failed || runResult.status == Status.successful,
        message: "EVM.mustRun(): The provided transaction is not valid for execution"
    )
    return runResult
}
```

The assertion passes when `status == Status.failed`. The function's own NatSpec comment acknowledges this: *"Note that this method does not rollback if transaction is executed but an vm error is reported as the outcome of the execution (status: failed)."*

`Status.failed` is reached when the EVM executes the transaction but the VM reports an error — for example, the recipient contract reverts, the call runs out of gas, or any other EVM-level failure occurs. In all such cases the Cadence transaction continues without reverting, and the caller receives a `Result` with `status == failed` that it may or may not inspect.

Because `EVM.run()` and `EVM.mustRun()` are `access(all)`, any Cadence transaction author — including an unprivileged one — can invoke them. If the author omits a check on `res.status`, a failed EVM token transfer is indistinguishable from a successful one at the Cadence level.

---

### Impact Explanation

**Cross-VM asset loss.** A Cadence contract that orchestrates EVM token transfers via `EVM.run()` or `EVM.mustRun()` without asserting `res.status == EVM.Status.successful` will record the transfer as complete even when the EVM-side transfer reverted. The Cadence-side accounting (e.g., marking a withdrawal as fulfilled, releasing escrowed Cadence tokens, or crediting a recipient) proceeds while the EVM-side balance is unchanged. This creates an irrecoverable discrepancy between Cadence state and EVM state.

---

### Likelihood Explanation

`EVM.run()` and `EVM.mustRun()` are the primary public entry points for executing arbitrary EVM transactions from Cadence. The name `mustRun` implies a stronger guarantee than it provides, making it likely that contract authors will rely on it without adding an additional `Status.failed` check. The pattern is directly analogous to the Solidity `call` return-value omission that is endemic across the ecosystem. Any Cadence contract that bridges tokens, distributes rewards, or performs conditional logic based on EVM transfer success is susceptible.

---

### Recommendation

1. **In `EVM.mustRun()`**: extend the assertion to also reject `Status.failed`:

```cadence
fun mustRun(tx: [UInt8], coinbase: EVMAddress): Result {
    let runResult = self.run(tx: tx, coinbase: coinbase)
    assert(
        runResult.status == Status.successful,
        message: "EVM.mustRun(): transaction execution was not successful"
    )
    return runResult
}
```

2. **For callers of `EVM.run()`**: always assert the result status before proceeding with any Cadence-side state change:

```cadence
let res = EVM.run(tx: tx, coinbase: coinbase)
assert(res.status == EVM.Status.successful, message: "EVM transfer failed")
```
