The critical finding is in `general_prerequisites`. Let me confirm the exact implementation:### Title
Missing Sequencer-Only Access Control on `update_rewards` Allows Any Caller to Permanently Suppress Block Rewards — (`File: src/staking/staking.cairo`)

---

### Summary

`update_rewards` is specified as callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any unprivileged address can call it with `disable_rewards: true`, consuming the global `last_reward_block` slot for the current block and permanently discarding that block's consensus rewards for all stakers.

---

### Finding Description

The spec explicitly states:

> **access control**: Only starkware sequencer. [1](#0-0) 

The implementation's gate function `general_prerequisites` only checks that the contract is unpaused and the caller is non-zero:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [2](#0-1) 

`update_rewards` calls only `general_prerequisites()` before proceeding — there is no `only_sequencer`, role check, or caller validation of any kind:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    ...
}
``` [3](#0-2) 

`last_reward_block` is a **single global** storage slot, not per-staker: [4](#0-3) 

The `REWARDS_ALREADY_UPDATED` guard checks `current_block_number > last_reward_block`. Once any call writes `last_reward_block = current_block_number`, every subsequent call in the same block reverts. There is exactly one reward-distribution opportunity per block, and it is consumed on a first-come-first-served basis with no caller restriction.

---

### Impact Explanation

An attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)` at the start of every block. The call:

1. Passes `general_prerequisites()` (contract unpaused, caller non-zero).
2. Passes the `REWARDS_ALREADY_UPDATED` check (new block).
3. Writes `last_reward_block = current_block_number`.
4. Hits `if disable_rewards || self.is_pre_consensus() { return; }` and exits without distributing any rewards.

The legitimate sequencer call for the same block then reverts with `REWARDS_ALREADY_UPDATED`. No staker receives rewards for that block. Repeated every block, this permanently freezes all consensus yield across the entire protocol.

Impact: **High — Permanent freezing of unclaimed yield** (all stakers, all blocks, zero cost to attacker beyond gas).

---

### Likelihood Explanation

The function is publicly callable on-chain with no economic barrier. A single EOA or contract can front-run the sequencer every block. The attack is deterministic, requires no special knowledge, and is sustainable indefinitely.

---

### Recommendation

Add a sequencer-only access control check at the top of `update_rewards`, consistent with the spec. On Starknet this is typically enforced by comparing `get_caller_address()` against a stored sequencer/operator address, or by using a dedicated role (e.g., `only_rewards_manager_role`). Example:

```cairo
fn update_rewards(...) {
    self.roles.only_rewards_manager(); // or assert caller == sequencer_address
    self.general_prerequisites();
    ...
}
```

### Proof of Concept

```
Block N opens.

Attacker tx (arrives before sequencer):
  staking.update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
  → general_prerequisites() passes (unpaused, caller != 0)
  → current_block_number (N) > last_reward_block (N-1) → passes
  → last_reward_block := N
  → disable_rewards == true → return (no rewards distributed)

Sequencer tx (same block N):
  staking.update_rewards(staker_address=<intended_staker>, disable_rewards=false)
  → current_block_number (N) > last_reward_block (N) → FALSE → REWARDS_ALREADY_UPDATED panic

Result: Block N rewards are permanently lost for all stakers.
Repeat every block → total consensus yield frozen indefinitely.
``` [5](#0-4)
