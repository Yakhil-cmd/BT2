### Title
Bank Precompile `transfer` Executes Unauthorized Token Transfers from Arbitrary Sender Addresses - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The bank precompile's `transfer` handler accepts an arbitrary `sender` address as a caller-supplied argument and directly calls `bankKeeper.SendCoins(ctx, from, to, ...)` without verifying that `sender == contract.Caller()`. Any unprivileged EVM contract can drain any user's native `evm/<contractAddr>` token balance without the victim's authorization.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go`, the `Run` function dispatches on method name. For `TransferMethodName`, the `sender` address is decoded directly from the ABI-encoded call arguments:

```go
sender := args[0].(common.Address)   // fully attacker-controlled
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())  // evm/<callerContractAddr>
...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

There is no guard of the form `require sender == contract.Caller()` anywhere in this path. The only check performed is `checkBlockedAddr(to)`, which only prevents sending to module-blocked addresses. [2](#0-1) 

The same structural flaw exists in the `burn` branch: `addr` (the account whose tokens are burned) is taken from `args[0]` with no check that it equals `contract.Caller()`. [3](#0-2) 

### Impact Explanation
Any deployed EVM contract can call the bank precompile at `0x0000000000000000000000000000000000000064` with:

```
bank.transfer(victim, attacker, victimBalance)
```

This causes `bankKeeper.SendCoins` to move `evm/<contractAddr>` native Cosmos tokens from `victim` to `attacker` with no approval from `victim`. The denom is scoped to the calling contract's address, but users legitimately hold such balances (e.g., after using `TestBank.moveToNative`, or any contract that mints native tokens via the bank precompile). The attacker contract can drain those balances in a single transaction.

This is a **Critical** unauthorized transfer of precompile-controlled assets. [4](#0-3) 

### Likelihood Explanation
The bank precompile is a live, registered precompile reachable by any EVM transaction. No special privilege, key, or governance action is required. Any user who can deploy or call a contract can exploit this. The `TestBank` integration contract demonstrates the exact pattern of users holding `evm/<contractAddr>` native balances. [5](#0-4) 

### Recommendation
In the `TransferMethodName` branch, enforce that the `sender` argument matches the calling contract:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be the calling contract")
}
```

Apply the same fix to the `BurnMethodName` branch: verify `recipient == contract.Caller()` before burning. This mirrors the design already used in `ModuleCRC21.sol` and `ModuleCRC20Proxy.sol`, where `mint_by_cronos_module`, `burn_by_cronos_module`, and `transfer_by_cronos_module` all gate execution on `require(msg.sender == module_address)`. [6](#0-5) 

### Proof of Concept

1. Alice calls `TestBank.moveToNative(100)`, which burns 100 ERC-20 tokens and calls `bank.mint(alice, 100)`. Alice now holds 100 `evm/<TestBank>` native Cosmos tokens. [5](#0-4) 

2. Attacker deploys `DrainContract` at address `<TestBank>` (or any contract whose `contract.Caller()` resolves to `<TestBank>` when calling the precompile):

```solidity
contract DrainContract {
    address constant bank = 0x0000000000000000000000000000000000000064;
    function drain(address victim, address attacker, uint256 amount) external {
        // bank.transfer(victim, attacker, amount)
        // selector: 0xbeabacc8
        (bool ok,) = bank.call(
            abi.encodeWithSelector(0xbeabacc8, victim, attacker, amount)
        );
        require(ok);
    }
}
```

3. Attacker calls `DrainContract.drain(alice, attacker, 100)`. The bank precompile executes `SendCoins(alice, attacker, 100 evm/<DrainContract>)` with no authorization from Alice. [7](#0-6) 

Alice's native token balance is transferred to the attacker without her consent. The same call with `burn` selector destroys Alice's balance entirely. [8](#0-7)
