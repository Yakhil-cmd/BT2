Looking at the exact code path in `get_compiled_from_class_manager` and the surrounding system to determine whether the V0 cache-skip is exploitable.

### Title
Stale Cairo0 (V0) Class Executed After Block Revert via `library_call` — (`crates/blockifier/src/state/state_reader_and_contract_manager.rs`)

---

### Summary

`get_compiled_from_class_manager` intentionally skips the `is_declared` state check for `RunnableCompiledClass::V0` cache hits. After a block revert (reorg), a V0 class that was cached during the reverted block's execution remains in the shared `ContractClassManager` cache. An attacker who knows the class hash can trigger execution of that stale, undeclared V0 class in the new canonical chain via the `library_call` syscall, which accepts an attacker-controlled class hash and resolves it through the same unchecked cache path.

---

### Finding Description

In `get_compiled_from_class_manager`, when the `ContractClassManager` cache returns a hit, the code branches on the variant:

```rust
match &runnable_class {
    RunnableCompiledClass::V0(_) => {}   // ← no is_declared check
    _ => {
        // The Cairo1 class is cached; verify it is declared,
        // since existence in the cache does not guarantee that
        // (it might contain a declared class from a reverted block, for example).
        if !self.state_reader.is_declared(class_hash)? {
            return Err(StateError::UndeclaredClassHash(class_hash));
        }
    }
}
``` [1](#0-0) 

The comment on the `_` arm explicitly names the revert scenario as the motivation for the Cairo1 check, but the `V0` arm takes no action. The `ContractClassManager` is a shared, global LRU cache that is **not cleared on block revert**. A V0 class cached during execution of a reverted block therefore persists in the cache indefinitely.

The `library_call` syscall handler in the blockifier constructs a `CallEntryPoint` directly from the attacker-supplied `class_hash` field and calls `execute_inner_call` with no prior `is_declared` check:

```rust
fn library_call(request: LibraryCallRequest, ...) -> Result<LibraryCallResponse, Self::Error> {
    let entry_point = CallEntryPoint {
        class_hash: Some(request.class_hash),
        ...
    };
    let retdata_segment = execute_inner_call(entry_point, vm, syscall_handler, remaining_gas)?;
    Ok(LibraryCallResponse { segment: retdata_segment })
}
``` [2](#0-1) 

`execute_inner_call` resolves the class via `get_compiled_class`, which delegates to `get_compiled_from_class_manager`. If the class hash maps to a cached V0 entry, the stale class is returned and executed without any state validation.

The `is_declared` mechanism is by design inapplicable to V0 classes (both `DictStateReader` and `ApolloReader` return `false` for V0 hashes), so there is no fallback guard:

```rust
fn is_declared(&self, class_hash: ClassHash) -> StateResult<bool> {
    Ok(match self.class_hash_to_class.get(&class_hash) {
        Some(class) => !matches!(class, RunnableCompiledClass::V0(_)),
        None => false,
    })
}
``` [3](#0-2) 

A separate TODO in the RPC execution state reader confirms the developers are aware that Cairo0 declaration verification is unimplemented:

```rust

ContractClass::V0(deprecated_contract_class) => {
    Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))

---

### Impact Explanation

An attacker can execute arbitrary Cairo0 bytecode that was declared in a reverted block. The executed code is not part of the canonical chain state. This constitutes **wrong compiled class / contract code selected for execution** — a Critical impact per the allowed scope. Concrete consequences include: executing logic from a class that was never finalized, producing wrong storage writes, events, L2→L1 messages, or return values that are committed to the canonical block.

---

### Likelihood Explanation

The preconditions are:
1. A V0 class is declared and its execution causes a cache population during a block that is subsequently reverted (reorg or sequencer-level revert). The attacker can be the declarer.
2. The attacker knows the class hash (trivially true if they submitted the declare transaction).
3. The attacker deploys a Cairo1 contract with a `library_call` entry point (standard, unprivileged operation) and submits an invoke transaction in the new canonical chain.

Reorgs are an operational reality in any sequencer. The attacker controls both the reverted declare and the subsequent invoke, making this a self-contained, unprivileged exploit chain.

---

### Recommendation

Apply the same `is_declared` guard to `V0` cache hits. Since the existing `is_declared` trait method is defined to return `false` for V0 classes (by design), a separate mechanism is needed — for example, querying `get_deprecated_class_definition_at` (or equivalent) against the current state number to confirm the class is declared in the canonical chain before returning the cached V0 class. Alternatively, clear or invalidate the `ContractClassManager` cache entries for all classes declared in a reverted block when a revert is processed.

---

### Proof of Concept

**Step 1 — Populate cache with a V0 class in a block that will be reverted:**

```rust
// In the reverted block's execution context:
let v0_class = CompiledClasses::V0(/* some deprecated class */);
contract_class_manager.set_and_compile(class_hash, v0_class);
// Cache now contains: class_hash → RunnableCompiledClass::V0(...)
```

**Step 2 — Revert the block (remove the declaration from canonical state):**

```rust
// state_reader.get_compiled_classes(class_hash) now returns UndeclaredClassHash
// but ContractClassManager cache is NOT cleared
```

**Step 3 — In the new canonical chain, submit an invoke that calls `library_call`:**

```rust
// Attacker's Cairo1 contract calls:
// starknet::library_call_syscall(reverted_class_hash, selector, calldata)
```

**Step 4 — Trace through `get_compiled_from_class_manager`:**

```rust
// get_runnable returns Some(RunnableCompiledClass::V0(...))  ← cache hit
match &runnable_class {
    RunnableCompiledClass::V0(_) => {}  // ← NO is_declared check, falls through
    _ => { /* Cairo1 check — not reached */ }
}
return Ok(runnable_class);  // ← stale V0 class returned and executed
```

The reverted V0 class executes in the canonical block, producing state changes, events, and messages that are committed as if the class were legitimately declared.

    fn library_call(
        request: LibraryCallRequest,
        vm: &mut VirtualMachine,
        syscall_handler: &mut Self,
        remaining_gas: &mut u64,
    ) -> Result<LibraryCallResponse, Self::Error> {
        let entry_point = CallEntryPoint {
            class_hash: Some(request.class_hash),
            code_address: None,
            entry_point_type: EntryPointType::External,
            entry_point_selector: request.function_selector,
            calldata: request.calldata,
            // The call context remains the same in a library call.
            storage_address: syscall_handler.storage_address(),
            caller_address: syscall_handler.caller_address(),
            call_type: CallType::Delegate,
            // NOTE: this value might be overridden later on.
            initial_gas: *remaining_gas,
        };

        let retdata_segment = execute_inner_call(entry_point, vm, syscall_handler, remaining_gas)
            .map_err(|error| match error {
            SyscallExecutionError::Revert { .. } => error,
            _ => error.as_lib_call_execution_error(
                request.class_hash,
                syscall_handler.storage_address(),
                request.function_selector,
            ),
        })?;

        Ok(LibraryCallResponse { segment: retdata_segment })
