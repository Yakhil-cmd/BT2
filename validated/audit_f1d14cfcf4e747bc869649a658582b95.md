### Title
Unprivileged Caller Can Call `update_rewards` With `disable_rewards: true` to Permanently Consume Block Reward Slots and Freeze Staker Yield - (File: `src/staking/staking.cairo`)

---

### Summary

`StakingRewardsManagerImpl::update_rewards` has no access control beyond a pause check and non-zero caller check. Any unprivileged address can call it with `disable_rewards: true`, which updates the global `last_reward_block` to the current block number but skips reward distribution entirely. Because the guard `current_block_number > last_reward_block` is global and checked before any reward logic, this permanently consumes the block's reward slot, blocking all subsequent legitimate calls in the same block and causing stakers to permanently lose their consensus block rewards.

---

### Finding Description

`update_rewards` is declared in `IStakingRewardsManager` and implemented without any role restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← global slot consumed here

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← exits without distributing rewards
    }
    ...
``` [1](#0-0) 

`general_prerequisites` only asserts the contract is unpaused and the caller is non-zero: [2](#0-1) 

The global `last_reward_block` is written unconditionally before the `disable_rewards` branch is evaluated: [3](#0-2) 

**Attack path:**

1. At block N, attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)`.
2. `last_reward_block` is set to N; no rewards are distributed.
3. The legitimate consensus mechanism (or any honest caller) attempts `update_rewards(staker, disable_rewards: false)` at block N.
4. The call reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number == last_reward_block`.
5. Block N's rewards are permanently lost for all stakers.
6. Attacker repeats at every block.

The `disable_rewards` flag is intended to allow the slot to be consumed without distributing rewards (e.g., during migration edge cases), but because the function is fully public, any caller can weaponize it.

---

### Impact Explanation

Each block where the attacker front-runs the legitimate call with `disable_rewards: true` results in permanently lost consensus block rewards for all stakers and their pool members. The rewards are never minted/distributed for that block — they are not deferred, they are gone. This constitutes **permanent freezing of unclaimed yield** (and effectively theft, since the inflation that would have been minted is skipped). A sustained attacker calling this at every block would drain all staker yield indefinitely.

---

### Likelihood Explanation

The entry point is fully public — any non-zero address can call it. The attacker only needs to know one valid, active staker address (trivially obtained from on-chain events such as `NewStaker`). The attack requires no capital, no privileged key, and no external dependency. The only cost is gas per block. This is a realistic griefing scenario with no profit motive but severe damage to all stakers and delegators.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus layer contract or an operator role). Alternatively, move `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` check so that a no-op call does not consume the block's reward slot:

```cairo
if disable_rewards || self.is_pre_consensus() {
    return;
}
// Only consume the slot when rewards are actually distributed.
self.last_reward_block.write(current_block_number);
// ... distribute rewards ...
```

Or add an access-control assertion before the slot is consumed:

```cairo
self.roles.only_consensus_operator(); // or equivalent
```

---

### Proof of Concept

```cairo
// Any unprivileged address executes this at block N:
staking.update_rewards(
    staker_address: any_valid_staker,
    disable_rewards: true,   // consumes slot, skips rewards
);

// Legitimate consensus call at the same block N now reverts:
// → panics with Error::REWARDS_ALREADY_UPDATED
staking.update_rewards(
    staker_address: any_valid_staker,
    disable_rewards: false,
);
// Block N rewards are permanently lost for all stakers.
```
