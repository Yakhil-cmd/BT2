### Title
Unbounded Loop in `calculate_rewards` Allows Delegator to Permanently Freeze Their Own Unclaimed Yield - (File: src/pool/pool.cairo)

### Summary
The `calculate_rewards` internal function in `src/pool/pool.cairo` contains an unbounded `while` loop that iterates over a delegator's entire `pool_member_epoch_balance` trace. A delegator can grow this trace without limit by repeatedly calling `add_to_delegation_pool` and `exit_delegation_pool_intent`. Once the trace is large enough, any subsequent call to `claim_rewards` will exceed the Starknet block gas limit and revert, permanently locking the delegator's unclaimed yield in the pool contract.

---

### Finding Description

The `calculate_rewards` function iterates over every entry in the `pool_member_epoch_balance` trace for a given pool member:

```
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    ...
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    rewards += compute_rewards_rounded_down(...);
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The codebase itself acknowledges the issue with the comment "This loop is unbounded but unlikely to exceed gas limits." Each iteration calls `find_sigma`, which performs additional storage reads from `cumulative_rewards_trace_vec`. [2](#0-1) 

The trace grows by one entry each time a delegator changes their balance. Both `add_to_delegation_pool` (via `increase_member_balance`) and `exit_delegation_pool_intent` (via `set_member_balance`) insert a new checkpoint into `pool_member_epoch_balance`: [3](#0-2) [4](#0-3) 

`calculate_rewards` is called directly from the public `claim_rewards` function: [5](#0-4) 

There is no pagination, offset, or maximum-batch mechanism. If the loop exceeds the block gas limit, `claim_rewards` reverts and the delegator's accumulated rewards remain permanently locked in the pool contract.

---

### Impact Explanation

**Permanent freezing of unclaimed yield.** Once the `pool_member_epoch_balance` trace is large enough to cause `claim_rewards` to exceed the block gas limit, the delegator can never retrieve their accrued rewards. The rewards sit in the pool contract with no alternative withdrawal path. This matches the allowed impact: *Permanent freezing of unclaimed yield or unclaimed royalties*.

---

### Likelihood Explanation

Any unprivileged delegator can trigger this against themselves (griefing their own future claim) or, more relevantly, a malicious actor could deliberately inflate their own trace to demonstrate the attack vector. The cost is repeated `add_to_delegation_pool` + `exit_delegation_pool_intent` cycles, each of which requires waiting out the `exit_wait_window` (default ~1 week). This makes the attack slow but entirely permissionless and deterministic. Over a long staking horizon, a delegator who frequently adjusts their position in small increments can reach the limit organically without malicious intent.

---

### Recommendation

1. **Paginate `calculate_rewards`**: Accept an `entry_limit` or `max_entries` parameter so the caller can process the trace in bounded batches across multiple transactions, accumulating `_unclaimed_rewards_from_v0` between calls.
2. **Checkpoint on claim**: After each successful partial claim, persist the updated `entry_to_claim_from` and `reward_checkpoint` so subsequent calls resume from where the last left off (the current code already tracks `entry_to_claim_from` but does not allow partial execution).
3. **Cap balance-change frequency**: Enforce a minimum epoch gap between successive balance changes for the same pool member, limiting trace growth rate.

---

### Proof of Concept

1. Delegator calls `add_to_delegation_pool` with a minimal amount — inserts entry N into `pool_member_epoch_balance`.
2. Delegator calls `exit_delegation_pool_intent` for that amount — inserts entry N+1.
3. After `exit_wait_window` elapses, delegator calls `exit_delegation_pool_action` to recover funds.
4. Repeat steps 1–3 thousands of times across many epochs.
5. Delegator calls `claim_rewards`. The `calculate_rewards` loop must iterate over all N entries, each calling `find_sigma` (storage reads into `cumulative_rewards_trace`). The transaction exceeds the block gas limit and reverts.
6. All accumulated rewards remain locked in the pool contract with no recovery path. [6](#0-5)

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
