### Title
Unbounded `calculate_rewards` Loop Over `pool_member_epoch_balance` Trace Enables Permanent Freezing of Unclaimed Yield — (`File: src/pool/pool.cairo`)

### Summary

The `calculate_rewards` internal function in the Pool contract iterates over every entry in a pool member's `pool_member_epoch_balance` trace without any bound. A delegator who repeatedly calls `add_to_delegation_pool` across many epochs without ever claiming rewards will accumulate an unbounded number of trace entries. When `claim_rewards` is eventually called, the loop must process all accumulated entries in a single transaction. If the trace is large enough, the transaction exceeds the Starknet gas limit and reverts, permanently freezing the delegator's unclaimed yield with no recovery path.

---

### Finding Description

`Pool::claim_rewards` calls the internal `calculate_rewards` function, which contains the following loop:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
    if pool_member_checkpoint.epoch() >= until_epoch {
        break;
    }
    let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
    rewards += compute_rewards_rounded_down(...);
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The loop iterates from the stored `entry_to_claim_from` index up to the full length of the `pool_member_epoch_balance` trace. Each iteration performs at least one call to `find_sigma`, which itself reads up to three entries from the `cumulative_rewards_trace` storage vector. [2](#0-1) 

The trace grows by one entry each time `add_to_delegation_pool` is called in a new epoch, via `increase_member_balance` → `set_member_balance` → `trace.insert(key: self.get_epoch_plus_k(), ...)`. [3](#0-2) 

The `entry_to_claim_from` cursor is only advanced on a **successful** `claim_rewards` call:

```cairo
pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
pool_member_info.reward_checkpoint = until_checkpoint;
``` [4](#0-3) 

If the transaction reverts (due to gas exhaustion), the cursor is never updated, and the delegator is permanently unable to claim rewards.

There is no partial-claim mechanism and no way to reset or prune the trace. The `exit_delegation_pool_action` path does not call `claim_rewards`, so the delegator can recover their principal but the accumulated yield is irrecoverable. [5](#0-4) 

---

### Impact Explanation

A delegator who accumulates a sufficiently large `pool_member_epoch_balance` trace will find that every `claim_rewards` transaction reverts due to gas exhaustion. Because the cursor `entry_to_claim_from` is only written on success, the state never advances, and the delegator is permanently locked out of their unclaimed yield. This matches the allowed High impact: **Permanent freezing of unclaimed yield**.

The developers themselves acknowledge the risk in the code comment: *"This loop is unbounded but unlikely to exceed gas limits."* [6](#0-5) 

---

### Likelihood Explanation

The entry point is `add_to_delegation_pool`, which is callable by any pool member or their reward address with no privileged role required. [7](#0-6) 

Each call in a distinct epoch appends one entry to the trace. The number of epochs required to exhaust gas depends on Starknet's per-transaction gas ceiling and the per-iteration cost (multiple storage reads via `find_sigma`). Over a long protocol lifetime — or if epochs are short — a delegator who never claims can naturally reach the threshold. The scenario is also reachable by a delegator who deliberately avoids claiming in order to later grief themselves (e.g., to demonstrate the bug), or who simply forgets to claim for an extended period.

---

### Recommendation

1. **Introduce a claim-window cap**: Allow `claim_rewards` to process at most `MAX_ENTRIES` trace entries per call and advance `entry_to_claim_from` accordingly, so multiple calls can drain the trace incrementally.
2. **Enforce a maximum trace depth**: Reject `add_to_delegation_pool` calls that would push the unclaimed trace window beyond a safe bound, forcing the delegator to claim first.
3. **Prune consumed entries**: After `entry_to_claim_from` advances past old entries, allow them to be removed from storage to bound the trace length.

---

### Proof of Concept

1. Deploy the staking system and create a staker with a STRK pool.
2. Create a delegator and call `enter_delegation_pool`.
3. For each of N epochs (e.g., N = 10,000), advance the epoch and call `add_to_delegation_pool` with a minimal amount — without ever calling `claim_rewards`. Each call appends one entry to `pool_member_epoch_balance` via `set_member_balance`.
4. After N epochs, call `claim_rewards`. The `calculate_rewards` loop must iterate over all N entries, each invoking `find_sigma` (up to 3 storage reads). The transaction runs out of gas and reverts.
5. Observe that `entry_to_claim_from` remains at 0 and every subsequent `claim_rewards` call also reverts. The delegator can call `exit_delegation_pool_action` to recover their principal, but the accumulated yield is permanently frozen. [8](#0-7) [9](#0-8)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn find_sigma(
            self: @ContractState, pool_member_checkpoint: PoolMemberCheckpoint, curr_epoch: Epoch,
        ) -> Amount {
            let pool_member_checkpoint_epoch = pool_member_checkpoint.epoch();
            assert!(
                pool_member_checkpoint_epoch <= curr_epoch,
                "{}",
                InternalError::INVALID_EPOCH_IN_TRACE,
            );
            let cumulative_rewards_trace_vec = self.cumulative_rewards_trace;
            let cumulative_rewards_trace_idx = pool_member_checkpoint
                .cumulative_rewards_trace_idx();

            // **Reminder**:
            // Let `len` be the length of `cumulative_rewards_trace_vec` at the time the checkpoint
            // is written.
            // In old version: `cumulative_rewards_trace_idx` = `len`.
            // In this version: `cumulative_rewards_trace_idx` = `len + 1`.
            // For current checkpoint in both versions: `cumulative_rewards_trace_idx` = `len - 1`.
            // **Invariant**:
            // 1. `cumulative_rewards_trace_vec.length() >= 1`.
            // 2. `cumulative_rewards_trace_vec.length()` is only increased, never decreased.
            if let Some(sigma) =
                find_sigma_edge_cases(
                    :cumulative_rewards_trace_vec,
                    :cumulative_rewards_trace_idx,
                    target_epoch: pool_member_checkpoint_epoch,
                ) {
                return sigma;
            }

            find_sigma_standard_case(
                :cumulative_rewards_trace_vec,
                :cumulative_rewards_trace_idx,
                target_epoch: pool_member_checkpoint_epoch,
            )
        }
