### Title
Permanently Growing `stakers` Vec Causes Unbounded Gas Consumption in `get_stakers()` - (File: src/staking/staking.cairo)

### Summary
The `stakers: Vec<ContractAddress>` storage variable grows permanently with every `stake()` call, but staker addresses are **never removed** from this Vec even after a full `unstake_intent()` + `unstake_action()` cycle. The `get_stakers()` function, which is critical for consensus validator-set queries, iterates the **entire** Vec on every call. An unprivileged attacker can repeatedly stake with the minimum amount, unstake (recovering their capital), and use fresh addresses to inflate the Vec indefinitely. Eventually `get_stakers()` will exceed the Starknet gas limit, permanently breaking the consensus validator-set query.

### Finding Description
In `src/staking/staking.cairo`, the `stake()` function appends the caller's address to `self.stakers` (a `Vec<ContractAddress>`) on every new stake:

```cairo
// Add staker address to the stakers vector.
self.stakers.push(staker_address);
```

The inline comment at the storage declaration explicitly acknowledges the problem:

```cairo
/// Vector of staker addresses.
/// **Note**: Stakers are not removed from this vector when they unstake.
stakers: Vec<ContractAddress>,
```

After `unstake_action()` completes, `staker_info` is set to `VInternalStakerInfo::None` and the staker's funds are returned, but the address remains in `stakers` forever.

`get_stakers()` iterates the **full** Vec on every call:

```cairo
for staker_address_ptr in self.stakers.into_iter_full_range() {
    let staker_address = staker_address_ptr.read();
    if !self.is_staker_active(:staker_address, :epoch_id) {
        continue;
    }
    ...
}
```

For each entry — including every exited staker — the loop performs at least one storage read (`staker_info`). For active stakers it performs several more (balance traces, public key, peer ID). As the Vec grows, the gas cost of `get_stakers()` grows linearly and without bound.

The `assert_staker_address_not_reused` guard prevents reusing the same address, but an attacker can generate arbitrarily many fresh Starknet account addresses at negligible cost.

### Impact Explanation
`get_stakers()` is the consensus-layer entry point for determining the active validator set for a given epoch. If it reverts due to gas exhaustion, the Starknet sequencer/consensus layer cannot obtain the validator set, disrupting liveness of the network. This constitutes **unbounded gas consumption** and **griefing with no profit motive but damage to users or protocol**, both of which are in the allowed impact scope.

### Likelihood Explanation
The attacker's recurring cost per entry is only gas (the `min_stake` deposit is fully recovered after `unstake_action()`). On Starknet, deploying new account contracts is cheap. A sustained campaign over days or weeks can accumulate thousands of dead entries. The attack requires no privileged access, no leaked keys, and no external dependency — only the public `stake()` / `unstake_intent()` / `unstake_action()` flow.

### Recommendation
1. **Remove exited stakers from the Vec** — replace `Vec` with a data structure that supports O(1) removal (e.g., a swap-and-pop pattern or an `IterableMap`), and call the removal in `remove_staker()`.
2. **Alternatively, paginate `get_stakers()`** — accept a range/offset parameter so callers can retrieve the validator set in bounded batches.
3. **Impose a minimum re-stake cooldown or a per-address stake-count limit** to raise the cost of flooding the Vec.

### Proof of Concept
```
// Attacker repeats the following loop with fresh addresses:
// 1. Deploy new account at address A_i
// 2. Approve min_stake STRK to staking contract
// 3. staking.stake(reward_addr, operational_addr, min_stake)
//    → self.stakers.push(A_i)  [Vec length += 1]
// 4. staking.unstake_intent()
// 5. Wait exit_wait_window (1 week default)
// 6. staking.unstake_action(A_i)
//    → staker_info[A_i] = None  (funds returned)
//    → stakers Vec still contains A_i permanently
//
// After N iterations, stakers.len() == N.
// get_stakers() must iterate all N entries, reading storage for each.
// At sufficient N, get_stakers() exceeds the Starknet gas limit and reverts.
```
