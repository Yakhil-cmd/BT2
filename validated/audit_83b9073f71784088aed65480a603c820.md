### Title
Underestimated Exec-Fee for `TransferToGasKey`/`WithdrawFromGasKey` on `GasKeyFunctionCall` Keys — (`File: core/parameters/src/cost.rs`)

### Summary

`gas_key_transfer_exec_fee()` always uses `AccessKey::min_gas_key_borsh_len()` — the borsh size of the smallest possible gas key (`GasKeyFullAccess` with zero nonces) — as the trie-value-length component of the per-byte exec fee for `TransferToGasKey` and `WithdrawFromGasKey`. When the target key is a `GasKeyFunctionCall` key (which stores a variable-length `FunctionCallPermission` containing `receiver_id` and `method_names`), the actual trie value written/read is materially larger than the estimate. The protocol therefore charges less gas than the real trie I/O cost, breaking the accounting invariant that exec fees must cover execution work.

### Finding Description

`gas_key_transfer_exec_fee()` computes the exec-side per-byte fee as:

```rust
let trie_key_len = access_key_key_len(account_id_len, public_key_len);
let estimated_value_len = AccessKey::min_gas_key_borsh_len();   // ← always minimum
let per_byte = cfg
    .fee(ActionCosts::gas_key_byte)
    .exec_fee()
    .checked_mul((trie_key_len + estimated_value_len) as u64)
    .unwrap();
``` [1](#0-0) 

`AccessKey::min_gas_key_borsh_len()` returns the borsh length of `AccessKey::gas_key_full_access(0)` — the smallest possible gas-key variant: [2](#0-1) 

That value is approximately 27 bytes (`nonce:u64` + discriminant + `GasKeyInfo{balance:u128, num_nonces:u16}`).

However, `action_transfer_to_gas_key` and `action_withdraw_from_gas_key` both call `get_access_key` (trie read) and `set_access_key` (trie write) on the **full** stored `AccessKey` value: [3](#0-2) 

For a `GasKeyFunctionCall` key the stored value additionally contains a `FunctionCallPermission` struct with a variable-length `receiver_id: String` (up to 64 bytes) and `method_names: Vec<String>` (each up to the protocol's method-name limit). A realistic key with `receiver_id` of 64 bytes and two 32-byte method names produces a borsh value of ~160 bytes — roughly 6× the minimum estimate. The per-byte exec fee is therefore undercharged by `(actual_len − min_len) × gas_key_byte_exec_fee` on every `TransferToGasKey` or `WithdrawFromGasKey` execution.

The fee is applied in `exec_fee()` in `config.rs` using the action's public key length and the receiver account-ID length, but the value-length component is always the minimum: [4](#0-3) 

### Impact Explanation

The exec fee for `TransferToGasKey`/`WithdrawFromGasKey` is systematically underestimated for any `GasKeyFunctionCall` key. An unprivileged user can:

1. Create a `GasKeyFunctionCall` key with a maximum-length `receiver_id` and many long `method_names` (all costs paid once at `AddKey` time).
2. Repeatedly call `TransferToGasKey` (or `WithdrawFromGasKey`) on that key, paying only the minimum-value-length exec fee while the runtime performs full-value trie reads and writes.

This breaks the accounting invariant that gas fees must cover actual execution work, allowing chunk space to be consumed at below-cost rates — a sustained economic DoS vector against validators.

### Likelihood Explanation

The `GasKeyFunctionCall` permission type is a first-class, user-accessible variant reachable via `AddKey` with `AccessKeyPermission::GasKeyFunctionCall` or the `promise_batch_action_add_gas_key_with_function_call` host function. No privileged role is required. The underestimation is present on every execution of the affected actions against such a key, making it trivially and repeatedly exploitable once the feature is live.

### Recommendation

Replace the fixed `AccessKey::min_gas_key_borsh_len()` estimate with the actual borsh length of the key being operated on. Since `action_transfer_to_gas_key` and `action_withdraw_from_gas_key` already retrieve the full `AccessKey` from the trie, the actual serialized length is available. Pass it into the fee calculation, or compute the exec fee inside the action handler after the key is fetched, rather than computing it statically in `exec_fee()` before the key is known.

### Proof of Concept

1. Create a `GasKeyFunctionCall` key with `receiver_id = "a" * 64` and `method_names = ["m" * 32; 10]`.
2. Fund it with `TransferToGasKey`.
3. Observe that the exec gas charged equals `gas_key_transfer_base.exec + gas_key_byte.exec * (trie_key_len + min_gas_key_borsh_len)` — independent of the actual 200+ byte value stored in the trie.
4. Compare against the exec gas charged for the same action on a `GasKeyFullAccess` key: the fee is identical despite the trie I/O being materially larger for the `GasKeyFunctionCall` case.

The discrepancy is `(actual_borsh_len − min_gas_key_borsh_len) × gas_key_byte_exec_fee` per invocation, which for a maximally-sized `GasKeyFunctionCall` key can be several hundred gas units per byte multiplied by ~150 extra bytes — a non-trivial per-operation undercharge that compounds across repeated calls.

/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

pub(crate) fn action_transfer_to_gas_key(
    state_update: &mut TrieUpdate,
    result: &mut ActionResult,
    account_id: &AccountId,
    action: &TransferToGasKeyAction,
) -> Result<(), RuntimeError> {
    let Some(mut access_key) = get_access_key(state_update, account_id, &action.public_key)? else {
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };
    let Some(gas_key_info) = access_key.gas_key_info_mut() else {
        // Key exists but is not a gas key
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };

    gas_key_info.balance = gas_key_info.balance.checked_add(action.deposit).ok_or_else(|| {
        RuntimeError::StorageError(StorageError::StorageInconsistentState(
            "gas key balance integer overflow".to_string(),
        ))
    })?;
    set_access_key(state_update, account_id.clone(), action.public_key.clone(), &access_key);
    Ok(())
}
