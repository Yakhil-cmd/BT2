### Title
ICA Precompile Unconditionally Registers `onPacketResultCallback` Without Enforcing Implementation, Permanently Blocking IBC Ack/Timeout Processing — (File: `x/cronos/keeper/precompiles/ica.go`)

---

### Summary

The ICA precompile's `submitMsgs` path unconditionally embeds a `src_callback` address in every ICA packet memo. When the IBC ack or timeout returns, the Cronos keeper calls `onPacketResultCallback` on the originating contract via `CallEVM`. If the contract does not implement this function, the EVM call reverts, the keeper propagates the error, and the `MsgAcknowledgement`/`MsgTimeout` transaction permanently fails — permanently blocking IBC packet settlement for that packet.

---

### Finding Description

In `x/cronos/keeper/precompiles/ica.go`, the `SubmitMsgsMethodName` case unconditionally embeds the caller's address as the IBC callback recipient:

```go
Memo: fmt.Sprintf(`{"src_callback": {"address": "%s"}}`, caller.String()),
``` [1](#0-0) 

This registers the calling contract with the IBC callbacks middleware for every `submitMsgs` call, with no opt-out and no check that the contract implements the required interface.

When the IBC ack or timeout arrives, the IBC callbacks middleware invokes `IBCOnAcknowledgementPacketCallback` or `IBCOnTimeoutPacketCallback` on the Cronos keeper, both of which delegate to `onPacketResult`:

```go
func (k Keeper) onPacketResult(...) error {
    ...
    _, res, err := k.CallEVM(ctx, &senderAddr, data, big.NewInt(0), gasLimit)
    if err != nil {
        return err
    }
    if res.Failed() {
        return fmt.Errorf("IBC callback EVM execution reverted: %s", res.VmError)
    }
    return nil
}
``` [2](#0-1) 

`data` is ABI-encoded as `onPacketResultCallback(string,uint64,bool)` (selector `0xd2712162`): [3](#0-2) 

If the calling contract does not implement `onPacketResultCallback`, the EVM dispatches to an unknown selector, the call reverts, `res.Failed()` is `true`, and `onPacketResult` returns a non-nil error. Both `IBCOnAcknowledgementPacketCallback` and `IBCOnTimeoutPacketCallback` propagate this error directly: [4](#0-3) 

The IBC callbacks middleware treats a non-nil error from these hooks as a fatal failure, causing the entire `MsgAcknowledgement`/`MsgTimeout` transaction to revert. Because the contract never implements the callback, every relay attempt fails identically — the packet commitment is never cleared and the packet is permanently stuck.

The required interface is defined as:

```solidity
interface IICACallback {
    function onPacketResultCallback(string calldata packetSrcChannel, uint64 seq, bool ack) external payable returns (bool);
}
``` [5](#0-4) 

There is no check anywhere in the precompile or keeper that the calling contract implements this interface before the callback memo is set.

---

### Impact Explanation

**High** — Permanent inability for honest users to process valid IBC ack/timeout packets. Any contract that calls `submitMsgs` via the ICA precompile without implementing `onPacketResultCallback` will have its IBC packet permanently stuck: the ack/timeout can never be relayed, the packet commitment is never cleared, and any escrowed funds or ICA state associated with the packet are permanently frozen. This matches the allowed High impact: *"Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows, precompile calls, IBC transfers."*

---

### Likelihood Explanation

**High** — The ICA precompile unconditionally sets the callback memo on every `submitMsgs` call with no opt-out mechanism and no documentation or enforcement requiring `onPacketResultCallback`. Any contract author who uses the ICA precompile without knowing about this implicit requirement will trigger the issue. The `TestICA.sol` integration test contract implements `onPacketResultCallback` explicitly, confirming the requirement exists but is not enforced at the protocol level. [6](#0-5) 

---

### Recommendation

1. **Opt-in callbacks**: Remove the unconditional `src_callback` memo injection. Only set the callback memo if the calling contract explicitly signals support (e.g., via an ERC-165 check for `IICACallback`, or an explicit parameter to `submitMsgs`).
2. **Graceful degradation**: In `onPacketResult`, if `res.Failed()` due to an unknown selector (i.e., the contract does not implement `onPacketResultCallback`), log and return `nil` rather than propagating the error, so the IBC ack/timeout is not permanently blocked.
3. **Interface enforcement**: Before setting the callback memo in `submitMsgs`, call `supportsInterface(0xd2712162)` on the caller and revert with a clear error if the contract does not implement the callback.

---

### Proof of Concept

1. Deploy a contract `VictimICA` that calls `ica.submitMsgs(connectionID, data, timeout)` but does **not** implement `onPacketResultCallback`.
2. `VictimICA` calls `submitMsgs`. The ICA precompile sets memo to `{"src_callback": {"address": "<VictimICA>"}}`. [1](#0-0) 
3. The ICA packet is relayed to the counterparty chain and executed.
4. The counterparty relayer submits `MsgAcknowledgement` back to Cronos.
5. The IBC callbacks middleware calls `IBCOnAcknowledgementPacketCallback(ctx, packet, ack, relayer, "<VictimICA>", "<VictimICA>", version)`.
6. `onPacketResult` calls `CallEVM(ctx, &VictimICA, onPacketResultCallbackData, 0, gasLimit)`. [7](#0-6) 
7. The EVM dispatches selector `0xd2712162` to `VictimICA`. No matching function → EVM reverts.
8. `res.Failed() == true` → `onPacketResult` returns `"IBC callback EVM execution reverted"`.
9. `IBCOnAcknowledgementPacketCallback` returns this error → `MsgAcknowledgement` fails.
10. Every subsequent relay attempt fails identically. The packet commitment is never cleared. The IBC packet is permanently stuck.

func (k Keeper) onPacketResult(
	ctx sdk.Context,
	packet channeltypes.Packet,
	acknowledgement bool,
	relayer sdk.AccAddress,
	contractAddress,
	packetSenderAddress string,
) error {
	sender, err := sdk.AccAddressFromBech32(packetSenderAddress)
	if err != nil {
		return fmt.Errorf("invalid bech32 address: %s, err: %w", packetSenderAddress, err)
	}
	senderAddr := common.BytesToAddress(sender)
	contractAddr := common.HexToAddress(contractAddress)
	if senderAddr != contractAddr {
		return fmt.Errorf("sender is not authenticated: expected %s, got %s", senderAddr, contractAddr)
	}
	data, err := cronosprecompiles.OnPacketResultCallback(packet.SourceChannel, packet.Sequence, acknowledgement)
	if err != nil {
		return err
	}
	gasLimit := k.GetParams(ctx).MaxCallbackGas
	_, res, err := k.CallEVM(ctx, &senderAddr, data, big.NewInt(0), gasLimit)
	if err != nil {
		return err
	}
	if res.Failed() {
		return fmt.Errorf("IBC callback EVM execution reverted: %s", res.VmError)
	}
	return nil
}

    function onPacketResultCallback(string calldata packetSrcChannel, uint64 seq, bool ack) external payable returns (bool) {
        // To prevent called by arbitrary user
        require(msg.sender == module_address);
        Status currentStatus = statusMap[packetSrcChannel][seq];
        if (currentStatus != Status.PENDING) {
            return true;
        }
        delete statusMap[packetSrcChannel][seq];
        Status status = Status.FAIL;
        if (ack) {
            status = Status.SUCCESS;
        }
        statusMap[packetSrcChannel][seq] = status;
        emit OnPacketResult(packetSrcChannel, seq, status);
        return true;
    }
