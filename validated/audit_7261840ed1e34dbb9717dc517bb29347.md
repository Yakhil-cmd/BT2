### Title
Attestation Target Block Computed From Mutable Current Stake Allows Griefing to Permanently Freeze Staker Epoch Rewards - (File: src/attestation/attestation.cairo)

### Summary

The `_calculate_target_attestation_block` function in `src/attestation/attestation.cairo` computes the target block using the staker's **current total stake** fetched live at attestation time. Because the stake is mutable during an epoch (via `increase_stake` or `add_stake_from_pool`), any state change to the staker's total stake between when the staker pre-computes their target block and when they submit the attestation will produce a different target block, causing the attestation to fail and permanently denying the staker their epoch rewards.

### Finding Description

The `attest` function in `src/attestation/attestation.cairo` calls `get_attestation_info_by_operational_address` on the staking contract to fetch live attestation data, then passes it to `_calculate_target_attestation_block`: [1](#0-0) 

Inside `_calculate_target_attestation_block`, the target block is derived from a Poseidon hash of `(stake, epoch_id, staker_address)`: [2](#0-1) 

The `stake` field in `AttestationInfo` is populated by `get_attestation_info_by_operational_address` using `get_staker_total_strk_balance_at_epoch` at the **current** epoch, i.e., the live balance at call time: [3](#0-2) 

The `AttestationInfo` struct confirms `stake` is the total STRK balance at the current epoch: [4](#0-3) 

Because `stake` is fetched live, any change to the staker's total STRK balance during the epoch (own stake via `increase_stake`, or delegated stake via `add_stake_from_pool`) will produce a different Poseidon hash and therefore a different `target_attestation_block`. The staker pre-computed the target block with the old stake value, obtained the block hash for that block, but the contract now computes a different target block and rejects the submitted hash.

The `_assert_attest_in_window` check then either rejects the attestation as out-of-window (if the new target block's window has not yet opened or has already closed), or the `ATTEST_WRONG_BLOCK_HASH` assertion fires because the staker submitted the hash for the old target block: [5](#0-4) 

### Impact Explanation

When the attestation fails, `update_rewards_from_attestation_contract` is never called for that epoch. The staker's `unclaimed_rewards_own` is never incremented for that epoch, and pool rewards are never distributed. These rewards are permanently lost — they cannot be retroactively claimed for a missed epoch. This constitutes **permanent freezing of unclaimed yield** for the staker and all pool members.

### Likelihood Explanation

Any delegator can call `enter_delegation_pool` or `add_to_delegation_pool` on the staker's pool contract with a minimal amount (e.g., 1 wei of STRK), which triggers `add_stake_from_pool` on the staking contract and updates the staker's delegated balance trace for the current epoch. This changes the `stake` value returned by `get_attestation_info_by_operational_address`, shifting the target block. The attacker's cost is negligible (gas + 1 wei locked in pool), while the staker loses an entire epoch's rewards. The attack is repeatable every epoch.

### Recommendation

Fix the target block computation to use the staker's stake at the **start** of the epoch (i.e., the balance checkpoint from the previous epoch boundary), not the live current balance. This ensures the target block is stable for the entire epoch regardless of intra-epoch stake changes. Concretely, `get_attestation_info_by_operational_address` should query `get_staker_total_strk_balance_at_epoch` with `epoch_id - 1` (or the epoch-start snapshot), so that the target block is fixed at epoch start and cannot be shifted by mid-epoch stake changes.

### Proof of Concept

1. Epoch E begins. Staker has total stake `S`. Target block = `Poseidon(S, E, staker) % (epoch_len - window) + epoch_start`.
2. Staker queries `get_current_epoch_target_attestation_block`, obtains `target_block = B`.
3. Staker waits until block `B + MIN_ATTESTATION_WINDOW` (window opens), retrieves `block_hash(B)`.
4. Malicious delegator calls `enter_delegation_pool(amount=1)` on the staker's pool. This calls `add_stake_from_pool`, updating the staker's delegated balance trace for epoch E to `S + 1`.
5. Staker calls `attest(block_hash(B))`. The contract calls `get_attestation_info_by_operational_address`, which now returns `stake = S + 1`.
6. Contract computes new target block = `Poseidon(S+1, E, staker) % (epoch_len - window) + epoch_start = B'` where `B' ≠ B`.
7. `get_block_hash_syscall(B')` returns a hash different from `block_hash(B)`. The assertion `target_block_hash == block_hash` fails with `ATTEST_WRONG_BLOCK_HASH`.
8. Staker cannot attest for epoch E. Epoch rewards are permanently lost. [2](#0-1) [6](#0-5)

        fn attest(ref self: ContractState, block_hash: felt252) {
            let operational_address = get_caller_address();
            let staking_dispatcher = IStakingAttestationDispatcher {
                contract_address: self.staking_contract.read(),
            };
            // Note: This function checks for a zero staker address and will panic if so.

                .get_attestation_info_by_operational_address(:operational_address);
            self._validate_attestation(:block_hash, :staking_attestation_info);
            // Work is one tx per epoch.
            self
                ._mark_attestation_is_done(
                    staker_address: staking_attestation_info.staker_address(),
                    current_epoch: staking_attestation_info.epoch_id(),
                );
            staking_dispatcher
                .update_rewards_from_attestation_contract(
                    staker_address: staking_attestation_info.staker_address(),
                );

        fn _calculate_target_attestation_block(
            self: @ContractState, staking_attestation_info: StakingAttestationInfo,
        ) -> BlockNumber {
            // Compute staker hash for the attestation.
            let hash = PoseidonTrait::new()
                .update(staking_attestation_info.stake().into())
                .update(staking_attestation_info.epoch_id().into())
                .update(staking_attestation_info.staker_address().into())
                .finalize();
            // Calculate staker's block number in this epoch.
            let attestation_window = self.attestation_window.read();
            let block_offset: u256 = hash
                .into() % (staking_attestation_info.epoch_len() - attestation_window.into())
                .into();
            // Calculate actual block number for attestation.
            let target_attestation_block = staking_attestation_info.current_epoch_starting_block()
                + block_offset.try_into().unwrap();
            target_attestation_block
        }

        fn get_attestation_info_by_operational_address(
            self: @ContractState, operational_address: ContractAddress,
        ) -> AttestationInfo {
            let staker_address = self.get_staker_address_by_operational(:operational_address);

            // Return the attestation info.
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            let epoch_info = self.get_epoch_info();
            let epoch_len = epoch_info.epoch_len_in_blocks();
            let epoch_id = epoch_info.current_epoch();
            let current_epoch_starting_block = epoch_info.current_epoch_starting_block();
            let stake = self
                .get_staker_total_strk_balance_at_epoch(
                    :staker_address, :staker_pool_info, :epoch_id,
                )
                .to_strk_native_amount();
            AttestationInfoTrait::new(
                :staker_address, :stake, :epoch_len, :epoch_id, :current_epoch_starting_block,
            )
        }
