### Title
Secret Nonce and Private Share Material Exposed via `Debug` Derive on Presign and Keygen Output Structs - (File: `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`, `src/lib.rs`)

---

### Summary

Multiple structs holding secret cryptographic material — including private signing shares and presignature nonce shares — unconditionally derive `Debug`, making their full scalar contents printable via `{:?}`. Any caller that logs, formats, or panic-reports these structs exposes secret material that is sufficient to reconstruct the private key.

---

---

### Impact Explanation

**Critical — Extraction, reconstruction, or disclosure of private signing shares, presign secrets, and nonce material.**

- Exposure of `KeygenOutput.private_share` via `{:?}` directly reveals the participant's secret signing share. Collecting threshold-many such shares reconstructs the aggregate private key.
- Exposure of `PresignOutput.k` (OT-based) or `PresignOutput.c`/`e` (robust) via `{:?}` reveals secret nonce shares. As documented in the codebase, knowledge of the underlying triple scalars is sufficient to recover the private key from any signature produced with that presignature.
- Exposure of `TripleShare.a`/`b`/`c` via `{:?}` (reachable through `PresignArguments`) similarly enables private key recovery.

---

### Likelihood Explanation

**Medium-High.** This is a library. Callers routinely use `{:?}` in:
- Error propagation chains (`format!("presign failed: {:?}", presign_output)`)
- Panic messages and `unwrap()` / `expect()` calls that include context
- Crash reporters and structured loggers that serialize `Debug` output
- Test harnesses that print intermediate state on failure

The `Debug` derive is unconditional and public. No special attacker capability is required beyond the ability to read application logs or crash dumps — exactly the exploit scenario described in the reference report.

---

### Recommendation

1. **Remove `Debug` from all structs containing secret scalars**, or implement a redacting `Debug` that prints `"[REDACTED]"` for secret fields:
   - `KeygenOutput` (`src/lib.rs`)
   - `PresignOutput` and `RerandomizedPresignOutput` in both OT-based (`src/ecdsa/ot_based_ecdsa/mod.rs`) and robust (`src/ecdsa/robust_ecdsa/mod.rs`) ECDSA
   - `PresignArguments` (`src/ecdsa/ot_based_ecdsa/mod.rs`)
   - `TripleShare` (`src/ecdsa/ot_based_ecdsa/triples/mod.rs`)
   - FROST `PresignOutput` (`src/frost/mod.rs`)

2. **Add a `#[cfg(test)]`-gated `Debug` impl** if debug printing is needed only in tests, so it never ships in production builds.

3. **Audit all `Clone` derives** on secret-bearing structs to ensure copies do not proliferate beyond their required lifetime.

### Proof of Concept

A caller using the library in a standard Rust error-handling pattern:

```rust
// Caller code — no malicious intent, just standard error propagation
let presign_out = run_presign_protocol(...)?;
// Later, in a logging or panic path:
tracing::debug!("presign state: {:?}", presign_out);
// Output includes raw hex of k and sigma scalars:
// PresignOutput { big_r: ..., k: Scalar(0xdeadbeef...), sigma: Scalar(0xcafebabe...) }
```

An attacker with read access to the application log or crash dump recovers `k_i` for participant `i`. With threshold-many `k_i` values and the corresponding signature `(R, s)`, they apply standard ECDSA nonce-reuse recovery to extract the aggregate private key.
