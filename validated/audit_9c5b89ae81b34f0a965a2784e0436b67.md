### Title
`ExitToEthereum` Precompile Schedules `withdraw` Promise With No Error Callback, Permanently Burning User Tokens on Promise Failure - (File: `engine-precompiles/src/native.rs`)

---

### Summary

The `ExitToEthereum` precompile burns ERC-20 tokens (or locks ETH at the precompile address) during EVM execution, then schedules a NEAR-side `withdraw` promise to the eth connector. Unlike `ExitToNear`, which has an `error_refund` callback path to re-mint tokens on failure, `ExitToEthereum` always emits a bare `PromiseArgs::Create` with no error callback. If the NEAR-side `withdraw` promise fails for any reason, the EVM-side burn is already committed and irrecoverable, permanently destroying the user's funds.

---

### Finding Description

In `ExitToEthereum::run()`, after the ERC-20 tokens are burned by the calling ERC-20 contract (or ETH is transferred to the precompile address), the precompile constructs a `withdraw` promise and wraps it unconditionally as `PromiseArgs::Create`:

```rust
let withdraw_promise = PromiseCreateArgs {
    target_account_id: nep141_address,
    method: "withdraw".to_string(),
    args: serialized_args,
    attached_balance: Yocto::new(1),
    attached_gas: costs::WITHDRAWAL_GAS,
};

let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
``` [1](#0-0) 

There is no `PromiseArgs::Callback` wrapping, no `exit_to_ethereum_precompile_callback` method, and no error-handling path. The promise result is never inspected.

Contrast this with `ExitToNear`, which (when the `error_refund` feature is enabled) attaches a callback:

```rust
let promise = if callback_args == ExitToNearPrecompileCallbackArgs::default() {
    PromiseArgs::Create(transfer_promise)
} else {
    PromiseArgs::Callback(PromiseWithCallbackArgs {
        base: transfer_promise,
        callback: PromiseCreateArgs {
            method: "exit_to_near_precompile_callback".to_string(),
            ...
        },
    })
};
``` [2](#0-1) 

The callback in `exit_to_near_precompile_callback` handles the failure branch by calling `engine::refund_on_error` to re-mint burned tokens:

```rust
} else if let Some(args) = args.refund {
    // Exit call failed; need to refund tokens
    let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
``` [3](#0-2) 

No equivalent mechanism exists for `ExitToEthereum`.

The NEAR-side `withdraw` promise can fail when:
- The eth connector's `withdraw` is paused via the `PAUSE_WITHDRAW` flag (explicitly supported, as shown in `engine-tests-connector/src/utils.rs` line 27 and the pausability test at lines 478–570)
- The eth connector panics or returns an error for any internal reason [4](#0-3) 

When the promise fails, NEAR does not roll back the prior EVM execution. The ERC-20 burn (committed in the EVM state during the originating NEAR transaction) is permanent. The user's tokens are gone.

---

### Impact Explanation

**Critical — Permanent freezing/destruction of funds.**

For the ERC-20 exit path (flag `0x1`): the ERC-20 contract burns the user's tokens before calling the precompile. If the `withdraw` promise fails, those tokens are permanently destroyed in the EVM with no recovery path.

For the ETH (base token) exit path (flag `0x0`): the user's ETH is transferred to the precompile address (`0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab`) during EVM execution. If the `withdraw` promise fails, the ETH is permanently locked at that address with no user-accessible recovery.

In both cases the user loses their entire exited amount with no recourse.

---

### Likelihood Explanation

**Medium.** The eth connector supports a `PAUSE_WITHDRAW` administrative flag. If an admin pauses withdrawals as a security response while users have in-flight `ExitToEthereum` calls, those users' tokens are permanently destroyed. This is a realistic operational scenario (e.g., pausing during an incident). Additionally, any future bug or panic in the eth connector's `withdraw` function would trigger the same loss for all concurrent exit calls. No attacker action is required — the loss is triggered by normal user behavior combined with any `withdraw` promise failure.

---

### Recommendation

Add an error callback to `ExitToEthereum` analogous to `ExitToNear`'s `error_refund` path:

1. Add a new `exit_to_ethereum_precompile_callback` contract method that checks the promise result.
2. On failure, call `engine::refund_on_error` to re-mint the burned ERC-20 tokens to the user's address, or transfer ETH back from the precompile address to the user.
3. Change the promise construction in `ExitToEthereum::run()` from `PromiseArgs::Create` to `PromiseArgs::Callback`, attaching the new callback.

---

### Proof of Concept

1. User holds 100 bridged-ERC-20 tokens on Aurora.
2. User calls the ERC-20 contract's `withdraw(100, eth_recipient)` function.
3. ERC-20 contract burns 100 tokens from the user's EVM balance.
4. ERC-20 contract calls `ExitToEthereum` precompile.
5. Precompile emits `PromiseArgs::Create(withdraw_promise)` targeting the eth connector — **no callback**.
6. Admin pauses the eth connector's `withdraw` (or any other failure occurs).
7. The `withdraw` promise executes and fails/panics.
8. No callback fires; no refund is issued.
9. User's 100 ERC-20 tokens are permanently burned. User receives nothing on Ethereum.

pub fn refund_on_error<I: IO + Copy, E: Env, P: PromiseHandler>(
    io: I,
    env: &E,
    state: EngineState,
    args: &RefundCallArgs,
    handler: &mut P,
) -> EngineResult<SubmitResult> {
    let current_account_id = env.current_account_id();
    if let Some(erc20_address) = args.erc20_address {
        // ERC-20 exit; re-mint burned tokens
        let erc20_admin_address = current_address(&current_account_id);
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, erc20_admin_address, current_account_id, io, env);

        let refund_address = args.recipient_address;
        let amount = U256::from_big_endian(&args.amount);
        let input = setup_refund_on_error_input(amount, refund_address);

        engine.call(
            &erc20_admin_address,
            &erc20_address,
            Wei::zero(),
            input,
            u64::MAX,
            Vec::new(),
            Vec::new(),
            handler,
        )
    } else {
        // ETH exit; transfer ETH back from precompile address
        let exit_address = exit_to_near::ADDRESS;
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, exit_address, current_account_id, io, env);
        let refund_address = args.recipient_address;
        let amount = Wei::new(U256::from_big_endian(&args.amount));
        engine.call(
            &exit_address,
            &refund_address,
            amount,
            Vec::new(),
            u64::MAX,
            vec![
                (exit_address.raw(), Vec::new()),
                (refund_address.raw(), Vec::new()),
            ],
            Vec::new(),
            handler,
        )
    }
