### Title
Silo Owner Can Permanently Freeze User EVM Funds by Removing Address from Whitelist - (`engine/src/engine.rs`)

### Summary

In Aurora Engine's Silo mode, the contract owner can call `remove_entry_from_whitelist` to remove a user's EVM address from the `Address` whitelist. Because `assert_access` gates **all** EVM transaction submission — including exit/withdrawal transactions — a de-whitelisted user can never submit the EVM transaction required to call the `ExitToNear` or `ExitToEthereum` precompile. Their ETH balance and ERC-20 token balances inside the Aurora EVM are permanently frozen with no alternative recovery path.

### Finding Description

Aurora Engine's Silo mode enforces two whitelists for EVM transaction submission: `WhitelistKind::Account` (NEAR account ID of the relayer) and `WhitelistKind::Address` (EVM address of the transaction sender). Both must pass for a transaction to be accepted.

Which calls `is_allow_submit`, requiring the sender's EVM address to be present in the `Address` whitelist: [5](#0-4) [6](#0-5) 

If the address is absent, the call returns `EngineErrorKind::NotAllowed` and the transaction is rejected. There is no separate "emergency withdrawal" path that bypasses this check. The NEAR-level `withdraw` method only handles the NEP-141 ETH-on-NEAR side (not EVM-internal balances), so EVM-side ETH and ERC-20 balances have no alternative exit route.

### Impact Explanation

**Permanent freezing of funds.** Any ETH balance or ERC-20 token balance held at a de-whitelisted EVM address inside the Aurora Silo EVM becomes permanently inaccessible. The user cannot submit any EVM transaction (including exit precompile calls) to recover their assets. In the worst case, the Silo owner could remove all addresses from the whitelist, freezing all user funds inside the EVM state with no recovery mechanism.

### Likelihood Explanation

The Silo owner is a single NEAR account with unilateral power to call `remove_entry_from_whitelist`. This is a legitimate administrative function (e.g., for compliance or access control), but it carries the unintended consequence of permanently freezing the removed user's EVM-side funds. No timelock, governance delay, or user-notification mechanism is required before the removal takes effect. The existing test `test_submit_with_removing_entries` confirms that removal immediately and completely blocks all EVM transactions from the removed address. [7](#0-6) 

### Recommendation

Before removing an address from the `Address` whitelist, the contract should either:

1. **Allow a grace-period exit**: Permit de-whitelisted addresses to submit a restricted set of exit-only transactions (calls to `ExitToNear`/`ExitToEthereum` precompile addresses) for a configurable window after removal.
2. **Separate exit access from submit access**: Introduce a dedicated `is_allow_exit` check that is independent of the `Address` whitelist, so that fund withdrawal is never gated by whitelist membership.
3. **Drain on removal**: When removing an address, automatically schedule a promise to transfer the address's EVM balance to a pre-registered recipient, analogous to the Connext mitigation of sending funds to the recipient regardless of caller when approval is revoked.

### Proof of Concept

1. Silo mode is active; `Address` and `Account` whitelists are enabled.
2. Alice's EVM address `alice_addr` and NEAR account `alice.near` are whitelisted. Alice deposits ETH into the Aurora EVM via the bridge, accumulating a balance at `alice_addr`.
3. The Silo owner calls `remove_entry_from_whitelist` with `WhitelistKind::Address` and `alice_addr`. The storage key for `alice_addr` is deleted.
4. Alice attempts to submit an EVM transaction calling the `ExitToNear` precompile to withdraw her ETH.
5. `assert_access` calls `is_allow_submit` → `is_address_allowed` → `Whitelist::is_exist(alice_addr)` → `false` (key no longer exists).
6. The transaction is rejected with `EngineErrorKind::NotAllowed`. Alice's ETH balance remains in the EVM state with no recovery path. [8](#0-7) [9](#0-8)

fn assert_access<I: IO + Copy, E: Env>(
    io: &I,
    env: &E,
    transaction: &NormalizedEthTransaction,
) -> Result<(), EngineError> {
    let allowed = if transaction.to.is_some() {
        silo::is_allow_submit(io, &env.predecessor_account_id(), &transaction.address)
    } else {
        silo::is_allow_deploy(io, &env.predecessor_account_id(), &transaction.address)
    };

    if !allowed {
        return Err(EngineError {
            kind: EngineErrorKind::NotAllowed,
            gas_used: 0,
        });
    }

    Ok(())
}

fn test_submit_with_removing_entries() {
    let (mut runner, signer, receiver) = initialize_transfer();
    let sender = utils::address_from_secret_key(&signer.secret_key);
    let caller: AccountId = CALLER_ACCOUNT_ID.parse().unwrap();
    let transaction = utils::transfer_with_price(
        receiver,
        TRANSFER_AMOUNT,
        INITIAL_NONCE.into(),
        ONE_GAS_PRICE.raw(),
    );

    set_silo_params(&mut runner, Some(SILO_PARAMS_ARGS));
    enable_all_whitelists(&mut runner);

    // Allow submitting transactions.
    add_account_to_whitelist(&mut runner, caller.clone());
    add_address_to_whitelist(&mut runner, sender);

    validate_address_balance_and_nonce(&runner, sender, INITIAL_BALANCE, INITIAL_NONCE.into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, ZERO_BALANCE, INITIAL_NONCE.into())
        .unwrap();

    // perform transfer
    let result = runner
        .submit_transaction(&signer.secret_key, transaction.clone())
        .unwrap();
    assert!(matches!(result.status, TransactionStatus::Succeed(_)));

    // validate post-state
    validate_address_balance_and_nonce(
        &runner,
        sender,
        INITIAL_BALANCE - TRANSFER_AMOUNT - FIXED_GAS * ONE_GAS_PRICE,
        (INITIAL_NONCE + 1).into(),
    )
    .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, TRANSFER_AMOUNT, INITIAL_NONCE.into())
        .unwrap();

    // Remove account id and address from whitelists.
    remove_account_from_whitelist(&mut runner, caller);
    remove_address_from_whitelist(&mut runner, sender);

    // perform transfer
    let err = runner
        .submit_transaction(&signer.secret_key, transaction)
        .unwrap_err();
    assert_eq!(err.kind, EngineErrorKind::NotAllowed);

    // validate post-state
    validate_address_balance_and_nonce(
        &runner,
        sender,
        INITIAL_BALANCE - TRANSFER_AMOUNT - FIXED_GAS * ONE_GAS_PRICE,
        (INITIAL_NONCE + 1).into(),
    )
    .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, TRANSFER_AMOUNT, INITIAL_NONCE.into())
        .unwrap();
