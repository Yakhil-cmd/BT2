### Title
Unbounded Loop in `calculate_rewards` Allows Permanent Freezing of Pool Member Unclaimed Yield - (File: `src/pool/pool.cairo`)

### Summary
The `calculate_rewards` function in the Pool contract iterates over every entry in a pool member's `pool_member_epoch_balance` trace without any bound. Each call to `add_to_delegation_pool` or `exit_delegation_pool_intent` in a new epoch appends a new checkpoint to this trace. A pool member who makes balance changes across many epochs without claiming rewards will accumulate an unbounded trace, eventually causing every `claim_rewards` call to run out of gas (OOG) on Starknet, permanently freezing their unclaimed yield.

### Finding Description

**Root cause — unbounded loop in `calculate_rewards`:**

The code itself acknowledges the risk with an explicit comment:

```cairo
// **Note**: The loop iterates over the balance changes in the pool member's balance
// trace. This loop is unbounded but unlikely to exceed gas limits.
while entry_to_claim_from < pool_member_trace_length {
    ...
    entry_to_claim_from += 1;
}
``` [1](#0-0) 

The loop iterates from `entry_to_claim_from` (the last claimed index) up to `pool_member_trace_length`. Each iteration performs at least one storage read (`pool_member_trace.at(entry_to_claim_from)`) and a call to `find_sigma`, which itself performs additional storage reads into `cumulative_rewards_trace`.

**How trace entries are created:**

Every call to `set_member_balance` inserts a checkpoint keyed by `current_epoch + K`:

```cairo
fn set_member_balance(ref self: ContractState, pool_member: ContractAddress, amount: Amount) {
    let trace = self.pool_member_epoch_balance.entry(pool_member);
    let pool_member_balance = PoolMemberBalanceTrait::new(
        balance: amount,
        cumulative_rewards_trace_idx: self.cumulative_rewards_trace_length() + 1,
    );
    trace.insert(key: self.get_epoch_plus_k(), value: pool_member_balance);
}
``` [2](#0-1) 

The `insert` function only **updates** the last entry if the key matches; otherwise it **appends** a new entry:

```cairo
if last.key == key {
    last.value = value;
    checkpoints[len - 1].write(last);
} else {
    assert!(last.key < key, "{}", TraceErrors::UNORDERED_INSERTION);
    checkpoints.push(PoolMemberBalanceCheckpoint { key, value });
}
``` [3](#0-2) 

Therefore, **one new trace entry is appended per epoch in which a balance change occurs**. The three operations that trigger `set_member_balance` are:

1. `enter_delegation_pool` → `set_member_balance`
2. `add_to_delegation_pool` → `increase_member_balance` → `set_member_balance`
3. `exit_delegation_pool_intent` → `set_member_balance` [4](#0-3) [5](#0-4) [6](#0-5) 

**The `entry_to_claim_from` optimization does not prevent the attack:**

`entry_to_claim_from` is stored in `InternalPoolMemberInfoV1` and advances after each `claim_rewards` call, so the loop only processes entries since the last claim. However, if the pool member makes balance changes each epoch **without claiming rewards**, the gap between `entry_to_claim_from` and `pool_member_trace_length` grows linearly with the number of epochs elapsed. [7](#0-6) 

**Exploit path:**

1. Pool member calls `enter_delegation_pool` (1 trace entry).
2. Each epoch, pool member calls `add_to_delegation_pool` with the minimum non-zero amount (1 unit), appending one new trace entry per epoch.
3. Pool member never calls `claim_rewards`.
4. After `N` epochs, the trace has `N` entries. The next `claim_rewards` call must iterate all `N` entries, each requiring multiple Starknet storage reads.
5. Once `N` is large enough to exceed the Starknet transaction gas limit, every `claim_rewards` call reverts with OOG, permanently freezing the pool member's accumulated unclaimed yield.

The `claim_rewards` function is the **only** path to withdraw accrued rewards: [8](#0-7) 

### Impact Explanation

Once the trace is large enough to cause OOG on every `claim_rewards` invocation, the pool member's unclaimed STRK rewards are permanently locked in the pool contract with no recovery path. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The pool member only needs to call `add_to_delegation_pool` (minimum 1 unit) once per epoch to grow the trace.
- Starknet epochs are short (measured in blocks/time), so the trace can grow to a dangerous size within a realistic operational timeframe.
- A pool member who is actively managing their delegation (adding small amounts regularly) and deferring reward claims will hit this naturally, without any adversarial intent.
- The code comment explicitly acknowledges the loop is unbounded and relies on the assumption it is "unlikely to exceed gas limits" — an assumption that breaks down over time.

### Recommendation

1. **Enforce a claim before balance changes**: Require `claim_rewards` to be called (or auto-call it internally) before any operation that appends a new trace entry. This resets `entry_to_claim_from` to the current trace length, keeping the loop bounded to at most a few entries per call.
2. **Cap trace growth**: Limit the number of unprocessed trace entries (e.g., assert `pool_member_trace_length - entry_to_claim_from < MAX_UNCLAIMED_ENTRIES`) before appending a new checkpoint.
3. **Batch-claim with a limit parameter**: Allow `claim_rewards` to accept a `max_entries` parameter so rewards can be claimed incrementally even when the trace is large.

### Proof of Concept

```
Epoch 1:  pool_member calls enter_delegation_pool(amount=MIN_STAKE)
          → trace = [entry@epoch(1+K)]  (length=1)

Epoch 2:  pool_member calls add_to_delegation_pool(pool_member, amount=1)
          → trace = [entry@epoch(1+K), entry@epoch(2+K)]  (length=2)

Epoch 3:  pool_member calls add_to_delegation_pool(pool_member, amount=1)
          → trace = [..., entry@epoch(3+K)]  (length=3)

...

Epoch N:  trace length = N
          pool_member calls claim_rewards(pool_member)
          → calculate_rewards loops N times, each reading storage
          → if N > GAS_LIMIT / cost_per_iteration → OOG revert
          → pool member's unclaimed rewards permanently frozen
```

The `calculate_rewards` loop at `src/pool/pool.cairo:859` iterates `N` times with no bound, and `N` grows by 1 for every epoch in which the pool member modifies their balance without claiming rewards. [9](#0-8)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

            // trace. This loop is unbounded but unlikely to exceed gas limits.
            while entry_to_claim_from < pool_member_trace_length {
                let pool_member_checkpoint = pool_member_trace.at(entry_to_claim_from);
                // If the balance change is after `until_epoch` (and therefore does not affect
                // the current reward computation), exit the loop.
                if pool_member_checkpoint.epoch() >= until_epoch {
                    break;
                }

                // Compute rewards from (inclusive) the previous balance change (or from
                // `from_checkpoint`) to (exclusive) the current entry.
                let to_sigma = self.find_sigma(pool_member_checkpoint, curr_epoch: until_epoch);
                rewards +=
                    compute_rewards_rounded_down(
                        amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                    );
                from_sigma = to_sigma;
                from_balance = pool_member_checkpoint.balance();
                entry_to_claim_from += 1;
            }

            // Compute the remaining rewards from (inclusive) the last visited balance change in
            // `pool_member_trace` (or from `from_checkpoint`) to (exclusive) `until_checkpoint`.
            let to_sigma = self.find_sigma(until_checkpoint, curr_epoch: until_epoch);
            rewards +=
                compute_rewards_rounded_down(
                    amount: from_balance, interest: to_sigma - from_sigma, :base_value,
                );

            (rewards, entry_to_claim_from)
        }
