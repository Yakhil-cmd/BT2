### Title
Declare Transactions Bypass `max_l2_gas_amount` Admission Check at Gateway - (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces the `max_l2_gas_amount` ceiling for Invoke and DeployAccount transactions but explicitly skips it for Declare transactions via an empty `if let` branch. Any unprivileged user can submit a Declare transaction with `l2_gas.max_amount = u64::MAX`, bypassing the gateway's DoS admission guard. Because `l2_gas.max_amount` is later used verbatim as the VM's initial Sierra gas limit in `TransactionContext::initial_sierra_gas`, the sequencer node accepts and begins executing a Declare with an unbounded gas envelope, violating the invariant that the gateway rejects transactions whose claimed gas exceeds the configured ceiling.

### Finding Description

In `validate_resource_bounds` the code reads:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
    // ← empty: no max_l2_gas_amount check
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

For Invoke and DeployAccount the check fires; for Declare the branch body is empty and execution falls through unconditionally. The production ceiling is `max_l2_gas_amount = 1_210_000_000`. [2](#0-1) 

The admitted value flows directly into the VM gas budget. `TransactionContext::initial_sierra_gas` returns `l2_gas.max_amount` for any `AllResources` transaction:

```rust
TransactionInfo::Current(CurrentTransactionInfo {
    resource_bounds: ValidResourceBounds::AllResources(AllResourceBounds { l2_gas, .. }),
    ..
}) => { l2_gas.max_amount }
``` [3](#0-2) 

A Declare with `l2_gas.max_amount = u64::MAX` therefore enters the blockifier with an initial Sierra gas counter of `u64::MAX`, removing the L2 gas ceiling that the gateway was supposed to enforce before sequencing.

The gap is acknowledged by the codebase itself with a `TODO` comment, and a dedicated test (`valid_l2_gas_amount_on_declare`) explicitly asserts that a Declare with `max_amount = 200` passes even when the config limit is `100`: [4](#0-3) 

### Impact Explanation

Matches **High – Mempool/gateway/RPC admission accepts invalid transactions before sequencing**. The gateway's stateless admission guard is the first and cheapest line of defence. A Declare transaction with `l2_gas.max_amount` far above the configured ceiling is admitted to the mempool and forwarded to the blockifier with an unbounded gas envelope. The blockifier's `verify_can_pay_committed_bounds` will reject it only if the account balance is insufficient to cover `u64::MAX × max_price_per_unit`; an attacker who controls a funded account (or who sets `max_price_per_unit = 0`, which passes the stateless `min_gas_price` check when `min_gas_price = 0` in some configurations) can force the sequencer to begin executing a Declare with no effective L2 gas cap, consuming sequencer CPU and blocking the batcher slot for that transaction.

### Likelihood Explanation

Any user with a funded Starknet account can craft and submit a valid Declare transaction (valid Sierra class, correct `compiled_class_hash`, correct nonce) with `l2_gas.max_amount = u64::MAX`. No privileged access is required. The stateless validator passes it; the stateful validator's `validate_resource_bounds` only checks the L2 gas *price* against the previous block, not the gas *amount*: [5](#0-4) 

### Recommendation

Remove the empty `if let` branch and apply the same `max_l2_gas_amount` ceiling to Declare transactions. The existing `TODO` comment already identifies this gap. The fix is a one-line change: replace the empty `if let RpcTransaction::Declare(_) = tx {}` guard with a uniform check across all transaction types, or add an explicit `max_l2_gas_amount` check inside the Declare arm.

### Proof of Concept

1. Compile any valid Sierra contract and compute its `compiled_class_hash`.
2. Construct an `RpcDeclareTransactionV3` with:
   - `resource_bounds.l2_gas.max_amount = GasAmount(u64::MAX)`
   - `resource_bounds.l2_gas.max_price_per_unit = GasPrice(min_gas_price)` (passes the price floor)
   - All other fields valid (correct nonce, valid Sierra class, matching `compiled_class_hash`).
3. Submit via `starknet_addDeclareTransaction`.
4. Observe: the gateway's `StatelessTransactionValidator::validate_resource_bounds` returns `Ok(())` for the Declare variant, the transaction is forwarded to the mempool, and the blockifier receives it with `initial_sierra_gas = u64::MAX`.
5. For comparison, submit an identical Invoke transaction with the same `max_amount`; the gateway rejects it with `MaxGasAmountTooHigh`. [6](#0-5)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
