### Title
Pool Member Remaining Funds Temporarily Locked After Staker Removal Due to `UNDELEGATE_IN_PROGRESS` Guard - (File: src/pool/pool.cairo)

### Summary
When a staker completes `unstake_action` and is removed, pool members who previously submitted a **partial** exit intent cannot call `exit_delegation_pool_intent` for their remaining balance until the original intent window expires. The remaining funds sit idle in the pool contract but are inaccessible for the full exit-wait-window duration (up to 12 weeks).

### Finding Description

In `pool.cairo`, the internal helper `undelegate_from_staking_contract_intent` handles the case where the staker has already been removed (`staker_removed == true`): [1](#0-0) 

```cairo
fn undelegate_from_staking_contract_intent(
    self: @ContractState, pool_member: ContractAddress, amount: Amount,
) -> Timestamp {
    if !self.is_staker_active() {
        assert!(
            self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
            "{}",
            Error::UNDELEGATE_IN_PROGRESS,
        );
        return Time::now();
    }
    ...
}
```

`is_staker_active()` returns `!self.staker_removed.read()`. [2](#0-1) 

`staker_removed` is set to `true` only after `unstake_action` completes via `set_staker_removed`. [3](#0-2) 

**Vulnerable scenario:**

1. Pool member calls `exit_delegation_pool_intent(X)` — partial exit. `unpool_amount = X`, `unpool_time = T1`, remaining balance `Y` stays in the staker's delegated pool.
2. Staker calls `unstake_intent` then `unstake_action`. During `unstake_action`, the remaining `pool_amount = Y` is transferred to the pool contract, and `set_staker_removed` is called.
3. Pool member now wants to exit `Y` (which is already sitting in the pool contract). They call `exit_delegation_pool_intent(Y)`.
4. This calls `undelegate_from_staking_contract_intent(pool_member, Y)`. Since `staker_removed == true` and `unpool_time` is already set (from step 1), the assertion `unpool_time.is_none()` **fails with `UNDELEGATE_IN_PROGRESS`**.

The pool member is blocked from exiting `Y` until they:
- Wait for `T1` to elapse
- Call `exit_delegation_pool_action` to retrieve `X` (clearing `unpool_time`)
- Only then call `exit_delegation_pool_intent(Y)` (which now succeeds with `unpool_time = Time::now()`)
- Call `exit_delegation_pool_action` again to retrieve `Y`

The `exit_delegation_pool_intent` function itself does not check for pause, but the guard is inside `undelegate_from_staking_contract_intent`: [4](#0-3) 

The exit wait window is configurable between `DEFAULT_EXIT_WAIT_WINDOW` (1 week) and `MAX_EXIT_WAIT_WINDOW` (12 weeks): [5](#0-4) 

### Impact Explanation

A pool member's remaining delegated funds `Y` are **temporarily frozen** in the pool contract for up to the full exit-wait-window (up to 12 weeks) after the staker is removed. The funds are not permanently lost, but the pool member cannot initiate withdrawal of `Y` until their prior partial intent is fully processed. This matches the allowed impact: **Temporary freezing of funds (High)**.

### Likelihood Explanation

This is triggered by any unprivileged pool member who:
1. Submitted a **partial** exit intent (not full balance) at any point before the staker's removal, AND
2. Wants to exit their remaining balance after the staker calls `unstake_action`.

This is a realistic and common pattern — partial exits are a supported use case per the spec. [6](#0-5) 

No privileged access is required. The staker's decision to exit is independent of the pool member's intent.

### Recommendation

When `staker_removed == true` and the pool member already has an existing intent, the pool contract already holds the remaining funds `Y` (transferred during `unstake_action`). There is no need to interact with the staking contract for a second intent. The guard should be relaxed to allow the pool member to **replace or extend** their intent for the remaining balance directly within the pool contract, bypassing the staking contract call entirely:

```cairo
if !self.is_staker_active() {
    // Allow a new intent even if one is in progress, since funds are already
    // in the pool contract and no staking contract interaction is needed.
    return Time::now();
}
```

Alternatively, allow `exit_delegation_pool_intent` to accumulate the new amount into the existing intent when the staker is removed.

### Proof of Concept

```
1. Pool member delegates 200 STRK to a staker's pool.
2. Pool member calls exit_delegation_pool_intent(100).
   → unpool_amount = 100, unpool_time = now + exit_wait_window, remaining balance = 100.
3. Staker calls unstake_intent().
4. Staker calls unstake_action() after exit_wait_window.
   → Pool contract receives 100 STRK (the remaining pool_amount).
   → staker_removed = true.
5. Pool member calls exit_delegation_pool_intent(100) to exit remaining 100 STRK.
   → undelegate_from_staking_contract_intent is called.
   → !is_staker_active() == true.
   → unpool_time.is_none() == false  ← FAILS with UNDELEGATE_IN_PROGRESS.
6. Pool member's 100 STRK is locked in the pool contract until step 4's
   intent window expires (up to 12 weeks from step 2).
```

        fn exit_delegation_pool_intent(ref self: ContractState, amount: Amount) {
            // Asserts.
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_delegated_stake = self.get_last_member_balance(:pool_member);
            let total_amount = old_delegated_stake + pool_member_info.unpool_amount;
            assert!(amount <= total_amount, "{}", GenericError::AMOUNT_TOO_HIGH);

            // Notify the staking contract of the removal intent.
            let unpool_time = self.undelegate_from_staking_contract_intent(:pool_member, :amount);

            // Edit the pool member to reflect the removal intent, and write to storage.
            if amount.is_zero() {
                pool_member_info.unpool_time = Option::None;
            } else {
                pool_member_info.unpool_time = Option::Some(unpool_time);
            }
            pool_member_info.unpool_amount = amount;
            let new_delegated_stake = total_amount - amount;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Update the pool member's balance checkpoint.
            self.set_member_balance(:pool_member, amount: new_delegated_stake);

            // Emit events.
            self
                .emit(
                    Events::PoolMemberExitIntent {
                        pool_member, exit_timestamp: unpool_time, amount,
                    },
                );
            self
                .emit(
                    Events::PoolMemberBalanceChanged {
                        pool_member, old_delegated_stake, new_delegated_stake,
                    },
                );
        }

        fn undelegate_from_staking_contract_intent(
            self: @ContractState, pool_member: ContractAddress, amount: Amount,
        ) -> Timestamp {
            if !self.is_staker_active() {
                // Don't allow intent if an intent is already in progress and the staker is erased.
                // Avoid the following flow:
                // 1. Member intent - moves member balance from the staker's `pool_amount` to
                // `UndelegateIntentKey`.
                // 2. Staker intent.
                // 3. Staker action - transfer the `pool_amount` from the staker to the pool
                // contract.
                // 4. Member intent - here it is no longer possible to move balance between the
                // `pool_amount` and the `UndelegateIntentKey`.
                assert!(
                    self.internal_pool_member_info(:pool_member).unpool_time.is_none(),
                    "{}",
                    Error::UNDELEGATE_IN_PROGRESS,
                );
                return Time::now();
            }
            let staking_pool_dispatcher = self.staking_pool_dispatcher.read();
            let staker_address = self.staker_address.read();
            staking_pool_dispatcher
                .remove_from_delegation_pool_intent(
                    :staker_address, identifier: pool_member.into(), :amount,
                )
        }
