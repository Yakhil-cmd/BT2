### Title
Malicious Participant Can Abort Triple Generation Without Attribution via Crafted Identity-Producing dlog Proof — (`src/crypto/proofs/dlog.rs`)

### Summary

A malicious participant who knows their own witness `x` can craft a `dlog::Proof` with `s = e * x` for any `e`. This makes the reconstructed commitment `big_k = s*G - e*X` equal to the group identity, causing `dlog::verify` to return `Err(ProtocolError::IdentityElement)` instead of `Ok(false)`. Because all three call sites in `do_generation_many` use the `?` operator on the result, this error propagates immediately and aborts the entire triple generation session. Critically, the `IdentityElement` error carries no participant attribution, so honest parties cannot identify or exclude the cheater.

### Finding Description

In `dlog::verify`, the reconstructed commitment is computed and then serialized:

```rust
let big_k = C::Group::generator() * proof.s.0 - *statement.public * proof.e.0;
let ser = C::Group::serialize(&big_k).map_err(|_| ProtocolError::IdentityElement)?;
``` [1](#0-0) 

If a malicious prover sets `s = e * x` (where `x` is their own witness and `X = x*G` is their public statement), then:

```
big_k = s*G - e*X = (e*x)*G - e*(x*G) = e*x*G - e*x*G = identity
```

Serialization of the identity element fails, returning `Err(ProtocolError::IdentityElement)`.

All three `dlog::verify` call sites in `do_generation_many` use the `?` operator:

```rust
if !dlog::verify(
    &mut transcript.fork(b"dlog0", &from.bytes()),
    statement0,
    their_phi_proof0,
)? {   // <-- ? propagates IdentityElement directly
    return Err(ProtocolError::AssertionFailed(format!(
        "dlog proof from {from:?} failed to verify"
    )));
}
``` [2](#0-1) [3](#0-2) [4](#0-3) 

The `?` bypasses the `AssertionFailed(format!("dlog proof from {from:?} failed to verify"))` branch — the only branch that names the offending participant. The raw `IdentityElement` error that propagates contains no participant identity. [5](#0-4) 

### Impact Explanation

**High: Permanent denial of triple generation for honest parties.**

Every honest party's `do_generation_many` future returns `Err(ProtocolError::IdentityElement)` with no indication of which participant sent the malformed proof. Since the cheater is unidentifiable from the error, honest parties cannot exclude them and retry. A single malicious participant can indefinitely block all triple generation sessions, which blocks all OT-based ECDSA presigning and signing.

The question's claim of "nonce material partially exposed" is not supported by the code — honest parties' nonces are generated locally and never transmitted before the abort point.

### Likelihood Explanation

The attack requires only that the attacker know their own witness `x`, which they always do (it is their own secret polynomial evaluation). The crafted proof `Proof { e, s = e*x }` is trivially constructable. No cryptographic assumption needs to be broken. The attack is deterministic and repeatable.

### Recommendation

`dlog::verify` must never return `Err` for a proof that is merely invalid — only for inputs that are structurally malformed independent of the proof content. The fix is to treat an identity `big_k` as a verification failure (`Ok(false)`) rather than a protocol error:

```rust
// Instead of:
let ser = C::Group::serialize(&big_k).map_err(|_| ProtocolError::IdentityElement)?;

// Use:
let Ok(ser) = C::Group::serialize(&big_k) else {
    return Ok(false);
};
```

This ensures the `if !dlog::verify(...)?` pattern in `generation.rs` reaches the `AssertionFailed(format!("dlog proof from {from:?} failed to verify"))` branch, which correctly attributes the failure to the cheating participant.

The same pattern should be audited in `dlogeq::verify`. [6](#0-5) 

### Proof of Concept

```rust
// In src/crypto/proofs/dlog.rs tests:
#[test]
fn test_identity_proof_returns_err_not_false() {
    use k256::Scalar;
    use elliptic_curve::bigint::Uint;
    use elliptic_curve::scalar::FromUintUnchecked;
    use frost_secp256k1::Secp256K1Sha256;
    use k256::ProjectivePoint;

    // Attacker's witness
    let x = Scalar::from_uint_unchecked(Uint::from_be_hex(
        "FC9A011DF3753BD79D841C11F6521F25AD2AB1DECEB96B7E8C28D87EA3303A06",
    ));
    let big_x = ProjectivePoint::GENERATOR * x;

    // Pick any e
    let e = Scalar::from(42u64);
    // Craft s = e * x  =>  s*G - e*X = identity
    let s = e * x;

    let statement = Statement::<Secp256K1Sha256> { public: &big_x };
    let proof = Proof {
        e: SerializableScalar(e),
        s: SerializableScalar(s),
    };

    let transcript = Transcript::new(b"protocol");
    let result = verify(&mut transcript.fork(b"party", &[1]), statement, &proof);

    // BUG: returns Err(IdentityElement) instead of Ok(false)
    assert!(matches!(result, Err(ProtocolError::IdentityElement)));
    // Desired behavior: assert!(matches!(result, Ok(false)));
}
```
