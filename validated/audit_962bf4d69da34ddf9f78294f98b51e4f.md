### Title
`TurnBridge` Message Handler Is a No-Op, Permanently Disabling Bridge State Control - (File: `x/cronos/keeper/msg_server.go`)

### Summary

The `MsgTurnBridge` message handler in Cronos is fully unimplemented — it returns `nil, nil` without performing any state change or authorization check. This is a direct structural analog to the external report's pattern: a function that should enforce a constraint (bridge on/off state) exists in the codebase but is entirely inert, leaving the bridge state permanently unconstrained and uncontrollable.

### Finding Description

In `x/cronos/keeper/msg_server.go`, the `TurnBridge` handler is defined as:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Compare this to every other sensitive handler in the same file, which all enforce authorization before acting:

- `UpdateTokenMapping` checks `HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping)` [2](#0-1) 
- `UpdateParams` checks `msg.Authority != k.authority` [3](#0-2) 
- `UpdatePermissions` checks `admin != msg.From` [4](#0-3) 
- `StoreBlockList` checks `admin != msg.From` [5](#0-4) 

`TurnBridge` has **neither** an authorization check **nor** any state mutation. It silently succeeds for any caller. [6](#0-5) 

The permission system defines `CanTurnBridge` as a distinct permission bit, confirming this was intended to be a guarded, functional operation:

```go
CanChangeTokenMapping uint64 = 1 << iota // 1
CanTurnBridge                             // 2
All = CanChangeTokenMapping | CanTurnBridge // 3
``` [7](#0-6) 

The `CanTurnBridge` permission bit is allocated and named, but the handler that should consume it is a stub. The constraint (bridge state enforcement) is structurally present but never applied — exactly the pattern from the external report.

### Impact Explanation

The bridge on/off state can never be changed regardless of who calls `TurnBridge`. If the bridge is currently enabled, it **cannot be disabled** — not by the admin, not by governance. In an emergency (e.g., a critical exploit draining bridge funds), the intended circuit-breaker mechanism is permanently non-functional. This constitutes a **bypass of the bridge authorization and control mechanism**, and a permanent inability for authorized parties to halt bridge/conversion flows.

This maps to:
- **High: Bypass of Cronos admin, governance authority, permission, token-mapping, bridge, block-list, or module-account authorization checks**
- **High: Permanent or long-lived inability for honest users or validators to process valid transactions, bridge/conversion flows under normal network assumptions** (specifically: inability to halt them when required)

### Likelihood Explanation

The vulnerability is always present and requires no special conditions. Any unprivileged user can call `MsgTurnBridge` and receive a success response. The bridge state is never written, so the disable path is permanently broken from genesis.

### Recommendation

Implement `TurnBridge` with the same authorization pattern used by other sensitive handlers. It should:
1. Check that the caller holds `CanTurnBridge` permission (via `HasPermission`) or is the `CronosAdmin`.
2. Read and write the bridge-enabled state to the KV store.
3. Return an error on unauthorized calls.

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // persist bridge enabled/disabled state
    k.SetBridgeEnabled(ctx, msg.Enable)
    return &types.MsgTurnBridgeResponse{}, nil
}
```

### Proof of Concept

1. Deploy Cronos with bridge enabled (default state).
2. Submit a `MsgTurnBridge{Enable: false}` transaction from any unprivileged address.
3. Observe: transaction succeeds (returns `nil, nil`), no error, no state change.
4. Query bridge state: bridge remains enabled.
5. Repeat with `Enable: true` — same result. Bridge state is immutable regardless of calls.
6. In an emergency, the admin submits `MsgTurnBridge{Enable: false}` — it silently succeeds but the bridge continues processing transfers, draining funds.

func (k msgServer) UpdatePermissions(goCtx context.Context, msg *types.MsgUpdatePermissions) (*types.MsgUpdatePermissionsResponse, error) {
	ctx := sdk.UnwrapSDKContext(goCtx)
	admin := k.Keeper.GetParams(ctx).CronosAdmin
	// if admin is empty, no sender could be equal to it
	if admin != msg.From {
		return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
	}
	acc, err := sdk.AccAddressFromBech32(msg.Address)
	if err != nil {
		return nil, err
	}
	k.SetPermissions(ctx, acc, msg.Permissions)

	return &types.MsgUpdatePermissionsResponse{}, nil
}
