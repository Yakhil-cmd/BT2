### Title
Unchecked `sender` Argument in Bank Precompile `transfer` Allows Any Calling Contract to Drain Arbitrary Addresses' Native Token Balances - (File: `x/cronos/keeper/precompiles/bank.go`)

---

### Summary

The bank precompile's `transfer` method accepts a `sender` address as a calldata argument and executes `bankKeeper.SendCoins(ctx, from, to, ...)` without verifying that the calling contract (`contract.Caller()`) is authorized to move funds on behalf of `sender`. Any deployed contract can pass an arbitrary victim address as `sender` and drain that address's `evm/<callerContractAddress>` native token balance to any recipient, with no approval or consent from the victim.

---

### Finding Description

In `BankContract.Run()`, the `transfer` case unpacks `sender` and `recipient` directly from calldata:

```go
sender := args[0].(common.Address)
recipient := args[1].(common.Address)
// ...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
// ...
denom := EVMDenom(contract.Caller())          // evm/<callerContract>
// ...
bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

The denom is scoped to `contract.Caller()`, so only the calling contract can move its own denomination. However, **there is no check that `sender == contract.Caller()` or that `sender` has granted an allowance to the calling contract**. The calling contract supplies `sender` freely from calldata. [2](#0-1) 

The only guard present is `checkBlockedAddr(to)`, which only prevents sending to module-blocked addresses — it does not verify the `sender`'s consent. [3](#0-2) 

The design intent (visible in `TestBank.sol`) is that a contract passes `msg.sender` as the sender, making the contract the sole authority over its denomination. But this authority is entirely unchecked — any contract can pass any address as `sender`. [4](#0-3) 

---

### Impact Explanation

**Critical — Unauthorized transfer of precompile-controlled assets.**

A malicious contract can call `bank.transfer(victimAddress, attackerAddress, amount)` to move `evm/<maliciousContract>` native tokens from any victim to any attacker address without the victim's authorization. This is a direct unauthorized balance change for precompile-controlled assets (`evm/` denomination tokens managed by the bank precompile). [5](#0-4) 

---

### Likelihood Explanation

Any unprivileged user can deploy a contract on Cronos EVM. Once deployed, the contract can immediately call the bank precompile's `transfer` method with an arbitrary `sender`. Users who hold `evm/<contractAddress>` balances (obtained by interacting with the contract via `bank.mint`) are exposed to having those balances drained at any time by the contract deployer, with no on-chain mechanism to prevent it. [6](#0-5) 

---

### Recommendation

Add a caller-authorization check in the `transfer` case: require that `contract.Caller()` equals `sender`, or implement an allowance/approval mechanism so that a contract may only transfer tokens on behalf of addresses that have explicitly approved it. For example:

```go
if contract.Caller() != sender {
    return nil, errors.New("caller not authorized to transfer on behalf of sender")
}
``` [1](#0-0) 

---

### Proof of Concept

1. Attacker deploys `DrainContract` at address `0xATK`.
2. Victim calls `DrainContract.deposit()`, which internally calls `bank.mint(victim, 1000)` — victim now holds 1000 `evm/0xATK` native tokens.
3. Attacker calls `DrainContract.drain(victimAddress, attackerAddress, 1000)`, which calls `bank.transfer(victimAddress, attackerAddress, 1000)`.
4. The bank precompile executes `bankKeeper.SendCoins(ctx, victim, attacker, 1000 evm/0xATK)` with no authorization check.
5. Victim's entire `evm/0xATK` balance is transferred to the attacker without consent.

The entry path is fully unprivileged (contract deployment + standard EVM call). No key compromise, admin access, or governance action is required. [7](#0-6) [8](#0-7)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
```
