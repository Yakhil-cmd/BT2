### Title
Secret Presign Material and Private Signing Share Disclosed via `Debug` Trait Derivation — (File: `src/ecdsa/ot_based_ecdsa/mod.rs`, `src/ecdsa/robust_ecdsa/mod.rs`)

---

### Summary

`PresignArguments` (which embeds the private signing share via `keygen_out`) and both `PresignOutput` / `RerandomizedPresignOutput` structs (which carry secret nonce scalars) unconditionally derive Rust's `Debug` trait. Any caller that formats these types with `{:?}` — in error messages, log statements, panic handlers, or test output — will emit the raw secret scalar bytes in plaintext. This is the direct structural analog of the reported `console.log(address, password)` pattern.

---

### Finding Description

**`PresignArguments` — private signing share reachable through `Debug`** [1](#0-0) 

`PresignArguments` is annotated `#[derive(Debug, Clone)]` and contains `keygen_out: KeygenOutput`, which holds `private_share: SigningShare<C>` — the node's long-term secret key share. Any `{:?}` formatting of this struct (e.g., `format!("{:?}", args)`, `dbg!(args)`, or an error handler that logs its input) will print the raw private share scalar.

**`PresignOutput` (OT-based) — nonce shares `k` and `sigma` reachable through `Debug`** [2](#0-1) 

`PresignOutput` derives `Debug` and contains `k: Scalar` (nonce share) and `sigma: Scalar` (sigma share). Both are secret presign values; exposure of `k` across two signing sessions enables full private-key recovery via standard ECDSA nonce-reuse.

**`RerandomizedPresignOutput` (OT-based) — same fields, same derive** [3](#0-2) 

**`PresignOutput` (robust ECDSA) — secret scalars `c`, `e`, `alpha`, `beta` reachable through `Debug`** [4](#0-3) 

**`RerandomizedPresignOutput` (robust ECDSA) — same fields, same derive** [5](#0-4) 

All four presign output types carry `ZeroizeOnDrop` to protect memory at drop time, yet simultaneously expose the same secrets unconditionally through `Debug`. The zeroize protection is therefore bypassed the moment any caller formats the value for display.

---

### Impact Explanation

- **Private signing share disclosure** (`PresignArguments`): a single `{:?}` log line exposes `private_share`, which is the node's long-term secret. Possession of `threshold` such shares reconstructs the aggregate secret key, enabling unauthorized signature creation for any message.
- **Presign / nonce secret disclosure** (`PresignOutput`): exposure of `k` (nonce share) across two signing sessions is sufficient for full private-key extraction via ECDSA nonce-reuse. Exposure of `alpha`, `beta`, `c`, `e` in the robust variant similarly breaks the security of the presignature.

Both map directly to the **Critical** allowed impact: *"Extraction, reconstruction, or disclosure of private signing shares, aggregate secret material, presign secrets, nonce material."*

---

### Likelihood Explanation

The library is consumed by higher-level MPC node implementations. It is routine for integrators to add structured logging, error-context wrapping (e.g., `anyhow::Context`, `thiserror`), or `dbg!` calls during development and incident investigation. Because `Debug` is derived rather than manually implemented with redaction, there is no compile-time or runtime barrier preventing accidental exposure. The pattern is identical to the `console.log(address, password)` root cause in the reference report.

---

### Recommendation

1. **Remove `#[derive(Debug)]`** from all structs that contain secret scalars: `PresignArguments`, `PresignOutput`, and `RerandomizedPresignOutput` in both `ot_based_ecdsa` and `robust_ecdsa`.
2. **Implement `Debug` manually** for these types, printing only non-secret fields (e.g., `big_r`) and replacing secret fields with a fixed redaction string such as `"<redacted>"`.
3. Consider implementing the `secrecy` crate's `Secret<T>` wrapper or a custom `Zeroizing<T>` newtype that also suppresses `Debug` output for all secret scalars used in these structs.

---

### Proof of Concept

```rust
// Caller code — no unsafe, no special privileges required
use threshold_signatures::ecdsa::ot_based_ecdsa::{PresignArguments, PresignOutput};

// After a presign round completes:
let output: PresignOutput = /* ... protocol result ... */;

// The following line prints k and sigma in plaintext to any log sink:
log::debug!("presign result: {:?}", output);
// or equivalently:
eprintln!("{:?}", output);
// or in an error context:
return Err(anyhow::anyhow!("unexpected state: {:?}", output));
```

Because `Debug` is unconditionally derived, the secret scalars `k` and `sigma` (or `c`, `e`, `alpha`, `beta` in the robust variant) appear verbatim in the formatted output. No attacker capability beyond reading log output is required.
