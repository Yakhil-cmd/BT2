### Title
Unenforced Authorization on `update_rewards` Allows Any Caller to Permanently Freeze Consensus Block Rewards - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable by any non-zero address. It accepts a `disable_rewards: bool` parameter that, when set to `true`, causes the function to update the global `last_reward_block` storage variable to the current block number **without distributing any rewards**. Because `last_reward_block` is a single global gate that prevents more than one reward update per block, any unprivileged caller can permanently consume each block's reward slot, denying all stakers their consensus block rewards indefinitely.

### Finding Description
`update_rewards` is exposed via the `IStakingRewardsManager` ABI with no access control beyond `general_prerequisites()`, which only checks that the contract is not paused and the caller is not the zero address. [1](#0-0) 

Inside the function, `last_reward_block` is written to storage **before** the early-return guard that checks `disable_rewards`: [2](#0-1) 

The consequence is:

1. Attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)` in block `N`.
2. The assertion `current_block_number > self.last_reward_block.read()` passes (first call in block `N`).
3. `last_reward_block` is set to `N`.
4. The function returns early — no rewards are calculated or distributed.
5. Any subsequent call to `update_rewards` in block `N` (by the legitimate staker) reverts with `REWARDS_ALREADY_UPDATED`.
6. The block's reward is permanently lost; it cannot be retroactively claimed.

The attacker repeats this every block at negligible cost (only gas), permanently starving all stakers and delegators of consensus rewards.

### Impact Explanation
Every block in the consensus rewards phase carries a fixed STRK (and BTC) reward computed by `calculate_block_rewards`. Once `last_reward_block` is set for a block, that block's reward is irrecoverably skipped. An attacker who front-runs the legitimate `update_rewards` call every block causes **permanent freezing of all unclaimed consensus yield** for every staker and every delegation pool member in the protocol.

This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- The function is part of the public ABI with no role check.
- The attacker needs only a valid (active, non-zero-balance) staker address to pass the precondition checks — this information is publicly observable on-chain via events (`NewStaker`) and view functions.
- The cost is one transaction per block; on Starknet this is cheap.
- No privileged access, leaked key, or external dependency is required.

### Recommendation
Restrict who may call `update_rewards` with `disable_rewards: true`. Options include:

1. **Remove the `disable_rewards` parameter entirely** from the public interface and handle the pre-consensus no-op path internally via `is_pre_consensus()` alone.
2. **Require the caller to be the staker's operational address** (or the staker itself) when `disable_rewards = true`, mirroring the access pattern used in `update_rewards_from_attestation_contract`.
3. **Move the `last_reward_block` write after the `disable_rewards` guard**, so that a disabled call does not consume the block slot.

### Proof of Concept

```
// Attacker script (pseudocode, one call per block)
loop every block B:
    // Pick any staker that is active and has non-zero balance at current epoch
    staker = any_known_active_staker()
    staking_contract.update_rewards(staker_address: staker, disable_rewards: true)
    // last_reward_block is now B; no rewards distributed
    // Legitimate staker's call in block B will revert with REWARDS_ALREADY_UPDATED
```

Concrete call path:
1. `update_rewards(staker_address, disable_rewards=true)` — public, no role check. [3](#0-2) 
2. Passes `general_prerequisites()` (not paused, caller ≠ 0). [4](#0-3) 
3. Passes block-number gate. [5](#0-4) 
4. Passes staker-active and non-zero-balance checks. [6](#0-5) 
5. Writes `last_reward_block = current_block`. [7](#0-6) 
6. Returns early — zero rewards distributed. [8](#0-7)
