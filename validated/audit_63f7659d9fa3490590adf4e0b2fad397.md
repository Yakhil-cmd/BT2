The vulnerability is real. Here is the complete analysis:

---

### Title
Unprivileged caller can permanently suppress per-block consensus rewards for all stakers via global `last_reward_block` gate — (`src/staking/staking.cairo`)

### Summary

`update_rewards` has no caller access control. Any address can call it with `disable_rewards: true`, which consumes the global `last_reward_block` slot for the current block without distributing any rewards, permanently blocking the legitimate sequencer from issuing rewards for that block.

### Finding Description

The spec for `update_rewards` states access control is "Only starkware sequencer." The implementation enforces no such restriction.

`general_prerequisites()` only checks two things:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

`update_rewards` itself adds no caller check beyond that:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);  // <-- written unconditionally

    if disable_rewards || self.is_pre_consensus() {
        return;  // <-- exits without distributing rewards
    }
    ...
}
``` [2](#0-1) 

`last_reward_block` is a **global** (not per-staker) storage variable. Once it is written to the current block number, the `REWARDS_ALREADY_UPDATED` assertion prevents any further `update_rewards` call in the same block — for **any** staker.

The attack path:
1. Attacker monitors the mempool or simply calls `update_rewards(any_valid_active_staker, disable_rewards: true)` at the start of any block.
2. The call passes all checks (staker exists, is active, has non-zero STRK balance).
3. `last_reward_block` is written to the current block number.
4. The function returns early — zero rewards distributed.
5. The legitimate sequencer's subsequent `update_rewards` call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. That block's consensus rewards are permanently lost for all stakers and delegators.

This is repeatable every block at negligible cost (only gas).

### Impact Explanation

Every block where the attacker fires first, the entire block's consensus reward allocation is permanently discarded — it is never credited to any staker's `unclaimed_rewards_own` or transferred to any pool. This constitutes **permanent freezing of unclaimed yield** at a rate of one block's worth per attack, executable continuously.

### Likelihood Explanation

The function is public, requires no token balance, no stake, and no privileged role. The only cost is gas. An attacker can automate this to run every block. The spec-vs-code discrepancy (spec says sequencer-only, code has no such check) confirms this is an unintentional omission.

### Recommendation

Add a sequencer-only guard at the top of `update_rewards`, consistent with the spec. On Starknet this can be enforced by checking `starknet::get_execution_info().tx_info.account_contract_address` against a stored sequencer address, or by using a role-based check (e.g., `self.roles.only_sequencer()`).

### Proof of Concept

1. Deploy the staking contract with consensus rewards active.
2. Stand up two active stakers A and B (both past the K-epoch activation window).
3. From an unprivileged EOA, call `update_rewards(staker_A, disable_rewards: true)` at the first transaction of block N.
4. Observe `last_reward_block == N`.
5. Attempt `update_rewards(staker_A, disable_rewards: false)` — reverts `REWARDS_ALREADY_UPDATED`.
6. Attempt `update_rewards(staker_B, disable_rewards: false)` — also reverts `REWARDS_ALREADY_UPDATED`.
7. Advance to block N+1; both stakers have received zero rewards for block N.
8. Repeat for every block. Cumulative rewards for all stakers remain zero indefinitely.

The existing test `update_rewards_disable_rewards_consensus_rewards_flow_test` already demonstrates that `disable_rewards: true` with consensus active yields zero rewards and sets `last_reward_block`, confirming the mechanism — it just does not test an unprivileged caller. [3](#0-2)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
