### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the `Staking` contract lacks the access control specified in the protocol spec ("Only starkware sequencer"). Any unprivileged caller can invoke it with `disable_rewards: true`, which updates the global `last_reward_block` checkpoint but skips reward distribution. Because `last_reward_block` is written unconditionally before the `disable_rewards` guard, the sequencer's legitimate call for the same block is permanently blocked, causing all stakers to lose rewards for that block.

### Finding Description
The protocol specification for `update_rewards` states:

> **access control**: Only starkware sequencer.

However, the implementation only calls `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero. There is no check that the caller is the designated sequencer address. [1](#0-0) 

The critical ordering flaw is that `last_reward_block` is written to storage **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

`last_reward_block` is a single global variable shared across all stakers: [3](#0-2) 

Once it is written to the current block number, any subsequent call in the same block—including the sequencer's legitimate call—reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

### Impact Explanation
An attacker who calls `update_rewards(any_active_staker, disable_rewards: true)` in a block before the sequencer does will:

1. Pass all staker-validity checks (staker exists, is active, has non-zero balance).
2. Cause `last_reward_block` to be set to the current block number.
3. Trigger the early return, distributing zero rewards.
4. Permanently prevent the sequencer from distributing rewards for that block to **all** stakers, because the global `last_reward_block` guard will reject any further call in the same block.

Rewards for skipped blocks are never retroactively distributed; they are permanently lost. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation
- The function is publicly callable by any non-zero address with no role check.
- The attacker only needs to know any valid, active staker address, which is publicly readable from the `stakers` vector in storage.
- The cost is a single transaction per block.
- On Starknet, while the sequencer controls ordering, users can submit transactions that the sequencer may include before its own system call, especially if the sequencer does not always place its `update_rewards` call first in every block.

### Recommendation
Add an access-control check at the top of `update_rewards` that restricts the caller to the designated sequencer address (or a stored `sequencer_address` role), consistent with the protocol specification. Alternatively, move the `self.last_reward_block.write(current_block_number)` statement to **after** the `disable_rewards` guard so that a call with `disable_rewards: true` does not consume the block's reward slot.

### Proof of Concept

1. Staker `S` is active with non-zero balance.
2. Attacker `A` (any address) calls `update_rewards(S, disable_rewards: true)` at block `N`.
3. The function passes staker-validity checks, writes `last_reward_block = N`, then returns early — no rewards distributed.
4. The sequencer attempts `update_rewards(S, disable_rewards: false)` at block `N`.
5. The assertion `current_block_number > self.last_reward_block.read()` fails (`N > N` is false), reverting with `REWARDS_ALREADY_UPDATED`.
6. Staker `S` (and all other stakers) receive zero rewards for block `N`. The loss is permanent.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
