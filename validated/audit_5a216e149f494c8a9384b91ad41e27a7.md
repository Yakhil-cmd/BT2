### Title
Missing Access Control on `update_rewards` Allows Griefing of Staker Reward Distribution - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the `Staking` contract is documented to be callable only by the Starkware sequencer, but the implementation enforces no such restriction. Any non-zero address can call `update_rewards` with `disable_rewards: true` for any valid staker, which advances the global `last_reward_block` to the current block without distributing any rewards. This permanently blocks the sequencer from distributing rewards for that block, causing stakers to lose yield.

### Finding Description

The `IStakingRewardsManager` interface exposes `update_rewards` as a public function. The specification at `docs/spec.md` line 1645 states its access control is **"Only starkware sequencer"**, but the implementation only calls `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no sequencer identity check exists. [1](#0-0) 

`general_prerequisites()` enforces only: [2](#0-1) 

The function writes `last_reward_block` to the current block number unconditionally, before the `disable_rewards` early-return guard: [3](#0-2) 

Because `last_reward_block` is a **single global storage slot** (not per-staker), one call in block N locks out all further `update_rewards` calls in block N: [4](#0-3) 

An attacker therefore calls `update_rewards(any_valid_staker, disable_rewards: true)` in block N. `last_reward_block` is set to N with zero rewards distributed. The sequencer's intended `update_rewards(target_staker, disable_rewards: false)` in the same block reverts with `REWARDS_ALREADY_UPDATED`. The staker permanently loses the rewards for block N.

The spec confirms the intended design: [5](#0-4) 

### Impact Explanation

Each block where the attack succeeds causes the targeted staker to permanently lose that block's share of consensus rewards. If executed continuously across every block, the staker's `unclaimed_rewards_own` never increases, constituting **permanent freezing of unclaimed yield**. The pool's `cumulative_rewards_trace` also receives no new entries, so delegators in the staker's pool likewise lose all rewards. No funds already at rest are stolen, but all future yield accrual is frozen.

This matches the allowed impact: **Permanent freezing of unclaimed yield / Theft of unclaimed yield**.

### Likelihood Explanation

- `update_rewards` is a public, permissionless entry point — no special role or key is required.
- The attacker only needs to supply any currently-active staker address (trivially obtained from on-chain `NewStaker` events).
- The call with `disable_rewards: true` is cheap (no token transfers, no reward supplier interaction).
- On Starknet the sequencer controls intra-block ordering, but the attacker can submit the transaction in the same block before the sequencer's system-level call, or in any block where the sequencer has not yet called `update_rewards`.
- The attack is repeatable every block at low cost with no profit requirement, making sustained griefing economically viable.

Likelihood: **Medium**.

### Recommendation

Restrict `update_rewards` to the Starkware sequencer address, consistent with the specification. Add a role check analogous to the existing `assert_caller_is_attestation_contract` pattern:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // enforce spec-stated access control
    ...
}
```

Alternatively, store the authorized sequencer address in contract storage and assert `get_caller_address() == self.sequencer_address.read()` at the top of `update_rewards`.

### Proof of Concept

1. Deploy the staking system and start consensus rewards.
2. Staker stakes and waits K epochs for balance to become effective.
3. In block N, attacker (any non-zero address) calls:
   ```
   update_rewards(staker_address, disable_rewards: true)
   ```
   - `last_reward_block` is written to N.
   - No rewards are distributed (early return at `disable_rewards || is_pre_consensus()`).
4. Sequencer attempts to call `update_rewards(staker_address, disable_rewards: false)` in block N.
   - Assertion `current_block_number > last_reward_block` fails (`N > N` is false).
   - Transaction reverts with `REWARDS_ALREADY_UPDATED`.
5. Staker's `unclaimed_rewards_own` remains unchanged; rewards for block N are permanently lost.
6. Repeating step 3 every block causes the staker to receive zero rewards indefinitely. [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
