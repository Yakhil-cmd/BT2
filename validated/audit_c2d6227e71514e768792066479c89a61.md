### Title
Single Malicious Participant Can Permanently Abort DKG/Reshare/Refresh via False Success Broadcast — (File: `src/dkg.rs`)

---

### Summary

The `broadcast_success` function in `src/dkg.rs` requires **unanimous** success votes from every participant before the DKG/reshare/refresh protocol is considered complete. A single malicious participant can participate honestly through all cryptographic rounds and then broadcast a `false` vote in the final round, causing every honest participant to abort. Because the attacker can repeat this at will, it constitutes a permanent, low-cost denial of service against key generation, resharing, and refresh.

---

### Finding Description

`do_keyshare` — the shared core of `do_keygen`, `do_reshare`, and `do_refresh` — concludes with a call to `broadcast_success`:

```
// Step 5.4 and Step 5.5
broadcast_success(&mut chan, &participants, me, session_id).await?;
``` [1](#0-0) 

`broadcast_success` always broadcasts `(true, session_id)` on behalf of the honest caller, then collects every participant's vote via `do_broadcast` (the echo-broadcast primitive) and enforces that **all** votes are `true`:

```rust
let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
...
if !vote_list.iter().all(|&(boolean, _)| boolean) {
    return Err(ProtocolError::AssertionFailed(
        "A participant seems to have failed its checks. Aborting Protocol!"
    ));
}
``` [2](#0-1) 

A malicious participant controls their own node implementation. They can substitute `(false, session_id)` for `(true, session_id)` in the final broadcast. Because `do_broadcast` is the authenticated double-echo broadcast, the malicious `false` vote is **reliably delivered to every honest party**:

> "Agreement: If a message m is delivered by some correct process, then m is eventually delivered by every correct process." [3](#0-2) 

Every honest participant therefore sees a `false` entry in `vote_list` and returns an error, aborting the protocol. The attacker can immediately restart and repeat the attack on the next DKG/reshare/refresh attempt.

The attack path through `do_keyshare` is:

1. **Round 1** — attacker sends a valid session-ID via `do_broadcast`. [4](#0-3) 
2. **Round 2** — attacker sends a valid commitment hash via `chan.send_many`. [5](#0-4) 
3. **Round 3** — attacker broadcasts a valid commitment + proof of knowledge via `do_broadcast`. [6](#0-5) 
4. **Round 4** — attacker sends valid private shares to each participant via `chan.send_private`. [7](#0-6) 
5. **Final round** — attacker sends `(false, session_id)` instead of `(true, session_id)` in `broadcast_success`, aborting all honest parties. [1](#0-0) 

---

### Impact Explanation

This is a **High** impact finding matching the allowed scope:

> "High: Permanent denial of signing, key generation, reshare, refresh, or CKD for honest parties under valid protocol inputs and documented trust assumptions."

The README documents that the DKG tolerates up to ⌊(N−1)/3⌋ malicious parties:

> "our DKG/Resharing protocol can only tolerate n/3 malicious parties" [8](#0-7) 

However, `broadcast_success` requires **unanimous** success — a single malicious party (well within the documented fault budget for any N ≥ 4) can abort the protocol. The documented fault tolerance is therefore not upheld in the final round. Honest parties cannot complete key generation, resharing, or refresh as long as the attacker participates.

---

### Likelihood Explanation

- **Any** participant in the protocol can execute this attack — no special privilege is required.
- The attacker incurs no cryptographic cost: all prior rounds are executed honestly, so no proof-of-knowledge or share verification fails.
- The attack is repeatable at zero marginal cost after the first attempt.
- The `recv_from_others` helper used in earlier rounds already documents indefinite waiting as intentional behavior, so operators have no built-in signal to distinguish a stalled protocol from a completed one until the final abort error surfaces. [9](#0-8) 

---

### Recommendation

1. **Replace the unanimity check with a threshold check.** `broadcast_success` should accept the protocol output if at least `threshold` (or `n − MaxFaulty`) participants broadcast success, consistent with the stated fault tolerance.
2. **Expose the identity of the false-voter.** The `ParticipantMap` returned by `do_broadcast` already contains per-participant votes; surface which participant voted `false` so operators can exclude the identified malicious party from the next attempt.
3. **Consider removing the final success broadcast entirely.** Because all cryptographic material (commitments, proofs of knowledge, shares) is verified before `broadcast_success` is called, an honest participant already has local evidence of protocol success. The final broadcast adds no cryptographic guarantee and only introduces a new griefing surface.

---

### Proof of Concept

```
Setup: N=4 participants (P1, P2, P3 honest; P4 malicious). Threshold t=2.
       Echo-broadcast fault tolerance: ⌊(4-1)/3⌋ = 1 — P4 is within budget.

Round 1:  P4 sends valid my_session_id via do_broadcast.       ✓
Round 2:  P4 sends valid commitment_hash via send_many.        ✓
Round 3:  P4 sends valid (commitment, proof) via do_broadcast. ✓
Round 4:  P4 sends valid signing shares to P1, P2, P3.        ✓
Final:    P4 sends (false, session_id) instead of (true, ...).

Result:   do_broadcast delivers (false, session_id) to P1, P2, P3.
          broadcast_success check: vote_list contains P4→false.
          All of P1, P2, P3 return ProtocolError::AssertionFailed.
          DKG aborted. No key material produced.

P4 repeats on every subsequent DKG/reshare/refresh attempt.
```

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

pub async fn recv_from_others<T>(
    chan: &SharedChannel,
    waitpoint: u64,
    participants: &ParticipantList,
    me: Participant,
) -> Result<Vec<(Participant, T)>, ProtocolError>
where
    T: serde::de::DeserializeOwned,
{
    let mut seen = ParticipantCounter::new(participants);
    seen.put(me);
    let mut messages = Vec::with_capacity(participants.others(me).count());

    while !seen.full() {
        let (from, msg) = chan.recv(waitpoint).await?;
        if seen.put(from) {
            messages.push((from, msg));

    }
