### Title
Missing Minimum Threshold Enforcement in FROST Signing Allows Single-Party Signature — (`File: src/frost/mod.rs`)

---

### Summary

`assert_sign_inputs` in `src/frost/mod.rs` omits the lower-bound threshold check (`threshold >= 2`) that is explicitly enforced in `assert_key_invariants` in `src/dkg.rs`. A caller can pass `threshold = 1` to the FROST signing protocol, allowing a single participant to unilaterally produce a valid threshold signature without any other participants' involvement.

---

### Finding Description

`assert_key_invariants` enforces a minimum threshold of 2 during DKG:

```rust
// src/dkg.rs:580-582
if threshold < 2 {
    return Err(InitializationError::ThresholdTooSmall { threshold, min: 2 });
}
``` [1](#0-0) 

`assert_sign_inputs`, the analogous validation function for FROST signing, only checks the upper bound:

```rust
// src/frost/mod.rs:144-150
if threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge {
        threshold: threshold.value(),
        max: participants.len(),
    });
}
``` [2](#0-1) 

There is no corresponding lower-bound check. The function accepts `threshold = 1` without error, even though the participant count check (`participants.len() < 2`) still passes with two or more participants present. [3](#0-2) 

The same omission exists in `presign` in `src/frost/mod.rs`:

```rust
// src/frost/mod.rs:72-77
if args.threshold.value() > participants.len() {
    return Err(InitializationError::ThresholdTooLarge { ... });
}
// No lower-bound check here either
``` [4](#0-3) 

---

### Impact Explanation

With `threshold = 1`, the FROST signing protocol degenerates: only one participant's share is needed to reconstruct the signing key and produce a valid signature. This directly enables **unauthorized creation of a valid threshold signature for attacker-chosen inputs** — a Critical impact under the allowed scope. The entire security model of threshold signing (requiring `t` of `n` parties to cooperate) collapses to a single-party scheme, defeating the purpose of distributed key generation.

---

### Likelihood Explanation

The `ReconstructionLowerBound` type wraps a plain `usize` with no enforced minimum:

```rust
// src/thresholds.rs:9-12
pub struct ReconstructionLowerBound(usize);
impl ReconstructionLowerBound {
    pub fn value(self) -> usize { self.0 }
}
``` [5](#0-4) 

Any library caller constructing `PresignArguments` or calling `assert_sign_inputs` directly can supply `threshold = 1` without any type-level or runtime barrier. The DKG phase enforces `threshold >= 2`, but the signing phase does not re-validate this invariant, so a mismatch between keygen and signing thresholds is silently accepted.

---

### Recommendation

Add the same lower-bound check present in `assert_key_invariants` to both `assert_sign_inputs` and `presign` in `src/frost/mod.rs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This mirrors the existing guard in `src/dkg.rs` at line 580 and closes the gap between keygen and signing validation. [6](#0-5) 

---

### Proof of Concept

1. Run DKG with `threshold = 2`, `participants = [P0, P1, P2]` — succeeds normally.
2. Call `assert_sign_inputs([P0, P1, P2], threshold=1, me=P0, coordinator=P0)` — **returns `Ok`** because only the upper-bound check fires.
3. Call `presign([P0, P1, P2], me=P0, args=PresignArguments { threshold: 1, ... })` — **returns `Ok`**.
4. P0 alone proceeds through the FROST signing round using `threshold = 1`, producing a valid aggregate signature without any other participant's nonce or share contribution.
5. The resulting signature verifies against the group public key, constituting an unauthorized single-party threshold signature. [7](#0-6)

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}
