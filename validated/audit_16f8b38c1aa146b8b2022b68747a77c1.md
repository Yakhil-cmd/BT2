### Title
Missing Access Control on `update_rewards()` Allows Anyone to Suppress Consensus Block Rewards - (File: src/staking/staking.cairo)

### Summary

`StakingRewardsManagerImpl::update_rewards` in `src/staking/staking.cairo` has no caller access control. Any unprivileged address can invoke it with `disable_rewards: true`, causing the global `last_reward_block` checkpoint to advance without distributing rewards. Because the checkpoint is global, a single call per block is sufficient to permanently suppress consensus rewards for every staker in the protocol.

### Finding Description

`update_rewards` is exposed as a public entry point through the `IStakingRewardsManager` interface:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause flag
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ← always written

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ← skips reward math
    }
    ...
```

`last_reward_block` is a **single global** storage slot:

```cairo
/// Last block number for which rewards were distributed.
last_reward_block: BlockNumber,
```

There is no `only_operator`, `only_app_governor`, or any other role check before the function body. `general_prerequisites` only asserts the contract is not paused.

**Attack path:**
1. Attacker observes any active `staker_address` from on-chain events.
2. At each new block, attacker calls `update_rewards(staker_address, disable_rewards: true)`.
3. `last_reward_block` is written to the current block number; the reward-distribution branch is skipped.
4. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Repeating this every block permanently prevents consensus block rewards from accruing to any staker or pool member.

### Impact Explanation

Consensus block rewards (`strk_block_rewards`, `btc_block_rewards`) are calculated and credited inside `_update_rewards`, which is only reached when `disable_rewards == false`. By front-running or simply calling first each block with `disable_rewards: true`, an attacker causes every staker's `unclaimed_rewards_own` to remain at zero indefinitely. This constitutes **permanent freezing of unclaimed yield** for all stakers and their delegators, matching the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties"*.

### Likelihood Explanation

The function is publicly callable with no privileged role requirement. The only cost to the attacker is Starknet gas per block. Because Starknet gas fees are low and the attacker gains a competitive advantage (e.g., a rival validator suppressing competitors' rewards), the attack is economically rational and technically trivial to automate.

### Recommendation

Restrict `update_rewards` to a trusted caller. The most natural fix is to add an `only_operator` (or equivalent sequencer/block-proposer role) guard at the top of the function, consistent with how other privileged state-changing functions in the contract are protected:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.roles.only_operator();   // ← add this
    self.general_prerequisites();
    ...
```

Alternatively, restrict `disable_rewards: true` to a privileged role while allowing any caller to invoke the function with `disable_rewards: false`.

### Proof of Concept

```
// Pseudocode – runs once per block
loop {
    staking.update_rewards(
        staker_address = any_active_staker,   // obtained from NewStaker events
        disable_rewards = true,
    );
    wait_for_next_block();
}
// Result: last_reward_block advances every block, _update_rewards is never reached,
//         all stakers accumulate zero consensus rewards indefinitely.
