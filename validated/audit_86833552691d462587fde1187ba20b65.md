### Title
Caller-Supplied Identity `app_pk` Exposes Confidential Derived Key in Plaintext - (File: src/confidential_key_derivation/protocol.rs)

### Summary
The CKD protocol's `ckd()` entry point accepts a caller-supplied ElGamal public key `app_pk` (`A`) without validating that it is a non-identity point. When `app_pk = G1::identity()`, the ElGamal blinding term `y · A` collapses to zero, causing each node's ciphertext component `C_i` to equal its raw BLS share `S_i = x_i · H(pk, app_id)`. The coordinator's aggregated output `C` then equals `msk · H(pk, app_id)` — the confidential derived key `s` — in the clear, which is subsequently published on-chain.

### Finding Description

The `ckd()` function in `src/confidential_key_derivation/protocol.rs` accepts `app_pk: PublicKey` (typed as `blstrs::G1Projective`) and passes it directly into `compute_signature_share` without any identity-point check. [1](#0-0) 

Inside `compute_signature_share`, the blinding computation is:

```rust
let big_c = big_s + app_pk * y.0;
``` [2](#0-1) 

When `app_pk = G1Projective::identity()`, the scalar multiplication `app_pk * y.0` returns the identity element, so `big_c = big_s = hash_point * private_share.to_scalar()`. After Lagrange-weighted aggregation by the coordinator:

```
C = Σ λ_i · C_i = Σ λ_i · (x_i · H(pk, app_id)) = msk · H(pk, app_id)
``` [3](#0-2) 

This is exactly `s`, the confidential derived key, returned in `CKDOutput.big_c` with no masking. The protocol specification requires that `s` be "only known by app": [4](#0-3) 

The `AppId` type validates length but `app_pk` has no analogous guard: [5](#0-4) 

The `PublicKey` type alias is a raw projective point with no built-in non-identity enforcement: [6](#0-5) 

### Impact Explanation

The `CKDOutput` `(Y, C)` is sent on-chain and is publicly observable. When `app_pk = O`, `C = msk · H(pk, app_id)` is the BLS signature that constitutes the confidential derived secret `s`. Any on-chain observer — not just the intended app — can read `s` directly from the transaction output. This is a full disclosure of a confidential derived secret, matching the Critical impact class: *"Extraction, reconstruction, or disclosure of … confidential derived secrets."*

### Likelihood Explanation

The `ckd()` public API accepts `app_pk` as a raw `G1Projective` value with no validation. Any caller of the library — including an MPC node operator constructing the protocol invocation from on-chain data — can supply the identity point. In the deployment model, the MPC contract forwards `(app_id, A)` from the Developer contract to MPC nodes; if the Developer contract does not enforce `A ≠ O` (which the MPC library explicitly does not require), the attack is trivially reachable. The `blstrs` library represents the identity point as a valid `G1Projective` value, so no special capability is needed to construct it.

### Recommendation

1. **Reject identity `app_pk` at the library boundary.** Add a check in `ckd()` before protocol execution:
   ```rust
   if app_pk.is_identity().into() {
       return Err(InitializationError::BadParameters(
           "app_pk must not be the identity point".to_string(),
       ));
   }
   ```
2. **Require a proof of knowledge of `a`.** Require the caller to supply a Schnorr proof that they know the discrete log of `app_pk`, preventing substitution of arbitrary points (including the identity) without knowledge of the corresponding secret.

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
