### Title
Unauthorized Transfer of Native Bank Tokens via Missing Caller Verification in Bank Precompile `transfer` Method - (File: `x/cronos/keeper/precompiles/bank.go`)

### Summary
The `BankContract.Run()` function in the bank precompile handles a `transfer(address sender, address recipient, uint256 amount)` call by accepting the `sender` address directly from calldata without verifying it equals the actual EVM caller (`contract.Caller()`). Any EVM contract can therefore transfer `evm/<callerContract>` native bank tokens from any victim address to any destination without the victim's authorization.

### Finding Description
In `x/cronos/keeper/precompiles/bank.go`, the `TransferMethodName` branch unpacks `sender` from the ABI-encoded calldata and passes it directly to `bankKeeper.SendCoins` as the `from` address:

```go
case TransferMethodName:
    args, err := method.Inputs.Unpack(contract.Input[4:])
    ...
    sender := args[0].(common.Address)   // ← taken from calldata, never verified
    recipient := args[1].(common.Address)
    amount := args[2].(*big.Int)
    ...
    from := sdk.AccAddress(sender.Bytes())
    to := sdk.AccAddress(recipient.Bytes())
    ...
    denom := EVMDenom(contract.Caller())  // denom is evm/<callerContract>
    ...
    bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
```

There is no check that `sender == contract.Caller()`. The `mint` and `burn` branches in the same switch also accept an arbitrary address from calldata, but those are arguably by design (the contract as token issuer controls supply). `transfer`, however, moves existing tokens between user accounts and must require the sender's authorization. The Cosmos SDK `bankKeeper.SendCoins` performs no such check — it simply moves funds if the balance exists. [1](#0-0) 

### Impact Explanation
Any EVM contract at address `C` can call `bank.transfer(victimAddress, attackerAddress, amount)` and drain the victim's `evm/C` native bank tokens without the victim's consent. This is a **Critical** unauthorized transfer of precompile-controlled assets. Users who hold `evm/<contract>` tokens (obtained by calling `moveToNative` or equivalent on a contract that uses the bank precompile) are fully exposed to theft by that contract at any time. [2](#0-1) 

### Likelihood Explanation
The entry path is fully unprivileged: any deployed EVM contract can call the bank precompile at address `0x0000000000000000000000000000000000000064`. No special role, key, or governance action is required. The only precondition is that the victim holds a non-zero balance of `evm/<callerContract>` tokens, which is the normal state for any user who has converted CRC20 tokens to their native representation via the bank precompile. [3](#0-2) 

### Recommendation
Add a caller-authorization check in the `TransferMethodName` branch before executing `SendCoins`. The `sender` argument supplied in calldata must equal `contract.Caller()`:

```go
case TransferMethodName:
    ...
    sender := args[0].(common.Address)
    if sender != contract.Caller() {
        return nil, errors.New("sender must be the caller")
    }
    ...
```

This mirrors the pattern already used in `x/cronos/keeper/precompiles/utils.go` where `exec` verifies `caller != e.caller` before dispatching any native action. [4](#0-3) 

### Proof of Concept

1. Deploy a contract `MaliciousToken` that implements `moveToNative` (burns its ERC20 balance and calls `bank.mint(msg.sender, amount)`) — identical to `TestBank.sol`.
2. Victim calls `moveToNative(100)`, receiving 100 `evm/<MaliciousToken>` native bank tokens.
3. Attacker calls a function on `MaliciousToken` that executes:
   ```solidity
   bank.transfer(victimAddress, attackerAddress, 100);
   ```
4. The bank precompile unpacks `victimAddress` as `sender`, constructs denom `evm/<MaliciousToken>`, and calls `bankKeeper.SendCoins(victimAddress, attackerAddress, 100 evm/<MaliciousToken>)`.
5. The victim's 100 native tokens are transferred to the attacker with no signature or approval from the victim. [1](#0-0) [5](#0-4)

```go
func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
```
