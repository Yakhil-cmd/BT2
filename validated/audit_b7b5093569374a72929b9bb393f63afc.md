### Title
Ejected Cluster Node Passes Vote Validation via `IdentityByEpoch` Missing Ejection Check - (File: `consensus/hotstuff/committees/cluster_committee.go`)

### Summary
`Cluster.IdentityByEpoch` checks only whether a node exists in the initial cluster member list, but does not check whether the node has been ejected mid-epoch. `Validator.ValidateVote` calls `IdentityByEpoch` as its signer-authorization gate. As a result, a cluster node ejected mid-epoch can submit votes that pass the primary vote-validation step, analogous to a killed gauge still passing the `isGauge` existence check in VoterV3.

### Finding Description

`Cluster.IdentityByEpoch` in `consensus/hotstuff/committees/cluster_committee.go` performs only a membership lookup against `clusterMembers`, the static `IdentitySkeletonList` populated at epoch-setup time:

```go
func (c *Cluster) IdentityByEpoch(view uint64, participantID flow.Identifier) (*flow.IdentitySkeleton, error) {
    identity, ok := c.clusterMembers.ByNodeID(participantID)
    if !ok {
        return nil, model.NewInvalidSignerErrorf("node %v is not an authorized hotstuff participant", participantID)
    }
    return identity, nil   // ← no ejection check
}
``` [1](#0-0) 

`IdentityByBlock`, the sibling method on the same struct, applies `clusterMemberFilter` which is composed with `filter.IsValidCurrentEpochParticipant` and therefore correctly rejects ejected nodes:

```go
clusterMemberFilter: filter.And[flow.Identity](
    initialClusterMembersSelector,
    filter.IsValidCurrentEpochParticipant,   // ← rejects ejected
),
``` [2](#0-1) 

`filter.IsValidCurrentEpochParticipant` is defined as `HasParticipationStatus(flow.EpochParticipationStatusActive)`, so any node with `EpochParticipationStatusEjected` is excluded. [3](#0-2) 

The existing test suite explicitly documents and asserts this asymmetry:

```go
suite.Run("should return ErrInvalidSigner for existent but ejected cluster member", func() {
    suite.Run("non-root block", func() {
        _, err := suite.com.IdentityByBlock(nonRootBlockID, realEjectedClusterMember.NodeID)
        suite.Assert().True(model.IsInvalidSignerError(err))   // ← correctly rejected
    })
    suite.Run("by epoch", func() {
        actual, err := suite.com.IdentityByEpoch(rand.Uint64(), realEjectedClusterMember.NodeID)
        suite.Assert().NoError(err)                            // ← ejected node accepted
        suite.Assert().Equal(realEjectedClusterMember.IdentitySkeleton, *actual)
    })
})
``` [4](#0-3) 

`Validator.ValidateVote` — the primary vote-authorization gate — calls `IdentityByEpoch` to decide whether a signer is authorized:

```go
func (v *Validator) ValidateVote(vote *model.Vote) (*flow.IdentitySkeleton, error) {
    voter, err := v.committee.IdentityByEpoch(vote.View, vote.SignerID)
    if model.IsInvalidSignerError(err) {
        return nil, newInvalidVoteError(vote, err)
    }
    ...
    err = v.verifier.VerifyVote(voter, vote.SigData, vote.View, vote.BlockID)
    ...
    return voter, nil
}
``` [5](#0-4) 

Because `IdentityByEpoch` returns the ejected node's `IdentitySkeleton` without error, `ValidateVote` proceeds to cryptographic verification and, if the signature is valid, returns the ejected node as an authorized voter. The vote is then forwarded into the `VoteCollector` pipeline.

A secondary defense exists: `StakingVoteProcessor` builds its `WeightedSignatureAggregator` from `IdentitiesByBlock`, which does apply the ejection filter, so the ejected node's vote is ultimately rejected there. [6](#0-5) 

However, the primary authorization gate (`ValidateVote` / `IdentityByEpoch`) is bypassed, creating the same structural gap as the VoterV3 `isGauge`-without-`isAlive` pattern.

### Impact Explanation

An ejected (malicious) cluster collection node can craft cryptographically valid votes and submit them to honest cluster nodes. Those votes pass `ValidateVote` — the function that the `VoteAggregator` uses to decide whether to forward a vote into the `VoteCollector`. The votes are cached in the `VoteCollector` (during the `CachingVotes` phase) and only rejected later by `StakingVoteProcessor.Process`. During the window between `ValidateVote` acceptance and `StakingVoteProcessor` rejection, the ejected node's votes occupy cache slots and consume processing resources. More critically, any future refactor that removes or bypasses the `StakingVoteProcessor` secondary check — or any code path that relies solely on `ValidateVote` for authorization — would allow the ejected node's weight to be counted toward QC construction in cluster consensus, potentially enabling an ejected malicious node to influence which collections are included in blocks. [7](#0-6) 

### Likelihood Explanation

A cluster node ejected mid-epoch retains its private keys and network connectivity until the epoch ends. The ejection service event is delivered asynchronously and only takes effect once sealed on the main chain. During the window between the ejection service event being sealed and the epoch boundary, the ejected node can continuously submit votes. Because `IdentityByEpoch` uses the static initial cluster member list (populated at epoch setup, before any mid-epoch ejection), the ejected node will always pass the `IdentityByEpoch` check for the remainder of the epoch. This is a deterministic, repeatable condition requiring no special privileges beyond the ejected node's own keys. [8](#0-7) 

### Recommendation

Add an ejection-status check to `Cluster.IdentityByEpoch`, mirroring the guard already present in `IdentityByBlock`. The simplest fix is to maintain a dynamic ejection lookup (analogous to `clusterMemberFilter`) and apply it in `IdentityByEpoch`:

```go
func (c *Cluster) IdentityByEpoch(view uint64, participantID flow.Identifier) (*flow.IdentitySkeleton, error) {
    identity, ok := c.clusterMembers.ByNodeID(participantID)
    if !ok {
        return nil, model.NewInvalidSignerErrorf("node %v is not an authorized hotstuff participant", participantID)
    }
    // Add: reject ejected nodes, consistent with IdentityByBlock
    if identity.EpochParticipationStatus == flow.EpochParticipationStatusEjected {
        return nil, model.NewInvalidSignerErrorf("node %v is ejected", participantID)
    }
    return identity, nil
}
```

Because `clusterMembers` is an `IdentitySkeletonList` (static, no dynamic fields), the ejection state must be tracked separately — for example by consulting the protocol state snapshot at the current view, or by maintaining a separate ejected-node set updated via `EjectIdentity` service events, consistent with how `identity_ejector.go` tracks ejections in the protocol state machine. [9](#0-8) 

### Proof of Concept

The existing test in `cluster_committee_test.go` already demonstrates the root cause without any modification:

1. A cluster member is marked ejected: `realEjectedClusterMember.EpochParticipationStatus = flow.EpochParticipationStatusEjected`
2. `IdentityByBlock` correctly returns `InvalidSignerError` for this node.
3. `IdentityByEpoch` returns `NoError` and the node's full `IdentitySkeleton`.
4. `Validator.ValidateVote` calls `IdentityByEpoch` (line 302 of `validator.go`), so a vote signed by the ejected node with a valid staking signature passes `ValidateVote` and is forwarded into the `VoteCollector`. [10](#0-9) [11](#0-10)

// While ejection status is not part of the EpochSetup event, we can supplement this information as follows:
//   - Per convention, service events are delivered (asynchronously) in an *order-preserving* manner. Furthermore,
//     node ejection is also mediated by system smart contracts and delivered via service events.
//   - Therefore, the EpochSetup event contains the up-to-date snapshot of the cluster members. Any node ejection
//     that happened before should be reflected in the EpochSetup event. Specifically, ejected nodes
//     should be no longer listed in the EpochSetup event. Hence, when the EpochSetup event is emitted / processed,
//     the participation status of all cluster members equals flow.EpochParticipationStatusActive.
func votingClusterParticipants(clusterMembers flow.IdentitySkeletonList) flow.IdentityList {
	initialClusterIdentities := make(flow.IdentityList, 0, len(clusterMembers))
	for _, skeleton := range clusterMembers {
		if skeleton.InitialWeight == 0 {
			continue
		}
		initialClusterIdentities = append(initialClusterIdentities, &flow.Identity{
			IdentitySkeleton: *skeleton,
			DynamicIdentity: flow.DynamicIdentity{
				EpochParticipationStatus: flow.EpochParticipationStatusActive,
			},
		})
	}
	return initialClusterIdentities

func (v *Validator) ValidateVote(vote *model.Vote) (*flow.IdentitySkeleton, error) {
	voter, err := v.committee.IdentityByEpoch(vote.View, vote.SignerID)
	if model.IsInvalidSignerError(err) {
		return nil, newInvalidVoteError(vote, err)
	}
	if err != nil {
		return nil, fmt.Errorf("error retrieving voter Identity at view %d: %w", vote.View, err)
	}

	// check whether the signature data is valid for the vote in the hotstuff context
	err = v.verifier.VerifyVote(voter, vote.SigData, vote.View, vote.BlockID)
	if err != nil {
		// Theoretically, `VerifyVote` could also return a `model.InvalidSignerError`. However,
		// for the time being, we assume that _every_ HotStuff participant is also a member of
		// the random beacon committee. Consequently, `InvalidSignerError` should not occur atm.
		// TODO: if the random beacon committee is a strict subset of the HotStuff committee,
		//       we expect `model.InvalidSignerError` here during normal operations.
		if model.IsInvalidFormatError(err) || errors.Is(err, model.ErrInvalidSignature) {
			return nil, newInvalidVoteError(vote, err)
		}
		if errors.Is(err, model.ErrViewForUnknownEpoch) {
			return nil, fmt.Errorf("no Epoch information available for vote; symptom of internal bug or invalid bootstrapping information: %s", err.Error())
		}
		return nil, fmt.Errorf("cannot verify signature for vote (%x): %w", vote.ID(), err)
	}

	return voter, nil

// EpochParticipationStatus represents the status of a node's participation. Depending on what
// changes were applied to the protocol state, a node may be in one of four states:
// /   - joining - the node is not active in the current epoch and will be active in the next epoch.
// /   - active - the node was included in the EpochSetup event for the current epoch and is actively participating in the current epoch.
// /   - leaving - the node was active in the previous epoch but is not active in the current epoch.
// /   - ejected - the node has been permanently removed from the network.
//
// /            EpochSetup
// /	      ┌────────────⬤ unregistered ◯◄───────────┐
// /	┌─────▼─────┐        ┌───────────┐        ┌─────┴─────┐
// /	│  JOINING  ├───────►│  ACTIVE   ├───────►│  LEAVING  │
// /	└─────┬─────┘        └─────┬─────┘        └─────┬─────┘
// /	      │              ┌─────▼─────┐              │
// /          └─────────────►│  EJECTED  │◄─────────────┘
// /                         └───────────┘
//
// Only active nodes are allowed to perform certain tasks relative to other nodes.
// Nodes which are registered to join at the next epoch will appear in the
// identity table but aren't considered active until their first
// epoch begins. Likewise, nodes which were registered in the previous epoch
// but have left at the most recent epoch boundary will appear in the identity
// table with leaving participation status.
// A node may be ejected by either:
//   - requesting self-ejection to protect its stake in case the node operator suspects
//     the node's keys to be compromised
//   - committing a serious protocol violation or multiple smaller misdemeanours.
type EpochParticipationStatus int

const (
	EpochParticipationStatusJoining EpochParticipationStatus = iota
	EpochParticipationStatusActive
	EpochParticipationStatusLeaving
	EpochParticipationStatusEjected

func (e *ejector) Eject(nodeID flow.Identifier) bool {
	l := len(e.identityLists)
	if len(e.ejected) == 0 { // if this is the first ejection sealed in this block, we have to populate the lookup first
		for i := range l {
			e.identityLists[i].identityLookup = e.identityLists[i].dynamicIdentities.Lookup()
		}
	}
	e.ejected = append(e.ejected, nodeID)

	var nodeFound bool
	for i := range l {
		dynamicIdentity, found := e.identityLists[i].identityLookup[nodeID]
		if found {
			nodeFound = true
			dynamicIdentity.Ejected = true
		}
	}
	return nodeFound
}
