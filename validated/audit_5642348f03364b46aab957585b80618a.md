### Title
`msgServer.TurnBridge` Is a No-Op Stub: Permission Bypass and Inoperable Bridge Circuit-Breaker — (`x/cronos/keeper/msg_server.go`)

---

### Summary

`msgServer.TurnBridge` unconditionally returns `nil, nil` with no authorization check and no state mutation. Any unprivileged address can submit `MsgTurnBridge` and receive a success response (code 0), bypassing the `CanTurnBridge` permission system. Simultaneously, the bridge-disable emergency mechanism is permanently non-functional because no bridge-enabled state is ever written.

---

### Finding Description

The implementation at `x/cronos/keeper/msg_server.go` lines 84–87 is a bare stub:

```go
// TurnBridge implements the grpc method
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [1](#0-0) 

Compare this to the analogous `UpdateTokenMapping` handler immediately above it, which correctly gates on `CanChangeTokenMapping`:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [2](#0-1) 

The permission system defines `CanTurnBridge` as a distinct bit and the query layer correctly exposes it, but the message handler never consults it: [3](#0-2) 

`ValidateBasic` for `MsgTurnBridge` only validates the bech32 sender address, so any valid address passes pre-execution checks: [4](#0-3) 

A search across the entire `x/cronos` tree finds **zero occurrences** of any bridge-enabled state key (`BridgeEnabled`, `bridge_enabled`, `EnableBridge`, `IsBridgeEnabled`). The state that `TurnBridge` was supposed to write does not exist anywhere in the keeper, params, or store.

---

### Impact Explanation

Two distinct High-severity impacts are present simultaneously:

1. **Permission bypass**: Any unprivileged address can submit `MsgTurnBridge{Enable: false}` and receive code 0, bypassing the `CanTurnBridge` authorization gate. This directly satisfies the "Bypass of Cronos admin, governance authority, permission … authorization checks" High impact category.

2. **Inoperable emergency circuit-breaker**: Because no bridge-enabled state is ever written or read, the bridge can never be disabled by anyone — admin or not. This satisfies "Permanent or long-lived inability for honest users or validators to process valid … bridge/conversion flows."

The integration test at `integration_tests/test_gravity.py` line 661–662 documents the *intended* behavior: an unprivileged `community` sender should receive `code != 0`. The current implementation violates this invariant. [5](#0-4) 

---

### Likelihood Explanation

Exploitability is trivial. The attacker needs only a valid bech32 address and the ability to submit a transaction. No special state, no leaked keys, no validator access. The call path is: craft `MsgTurnBridge` → `ValidateBasic` passes (bech32 check only) → `TurnBridge` returns `nil, nil` → tx succeeds with code 0.

---

### Recommendation

1. Add the `CanTurnBridge` permission check at the top of `TurnBridge`, mirroring `UpdateTokenMapping`:
   ```go
   if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
       return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
   }
   ```
2. Implement the actual bridge-enabled state: add a store key, write `msg.Enable` to it in `TurnBridge`, and read it in the gravity/bridge hooks to gate outbound transfers.
3. Add a unit test asserting that an unprivileged sender receives `ErrUnauthorized` and that a privileged sender causes the bridge-enabled flag to change.

---

### Proof of Concept

```go
// Unit test — no chain needed
func TestTurnBridgeUnprivilegedSucceeds(t *testing.T) {
    // setup keeper with no admin configured
    ctx, k := setupKeeper(t)
    srv := NewMsgServerImpl(k)

    attacker := sdk.AccAddress([]byte("attacker_____________"))
    msg := types.NewMsgTurnBridge(attacker.String(), false)

    // ValidateBasic passes — only bech32 check
    require.NoError(t, msg.ValidateBasic())

    // TurnBridge returns nil,nil — code 0 — no permission check
    resp, err := srv.TurnBridge(sdk.WrapSDKContext(ctx), msg)
    // FAILS the invariant: should be ErrUnauthorized, is nil
    require.NoError(t, err)   // currently passes — demonstrates the bug
    require.NotNil(t, resp)
}
```

The existing integration test at `integration_tests/test_gravity.py:661` already encodes the correct invariant (`code != 0` for unprivileged sender) and would catch this if run against the current implementation. [6](#0-5)
