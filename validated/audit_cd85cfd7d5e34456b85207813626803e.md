### Title
Unprivileged Caller Can Suppress All Consensus Block Rewards by Calling `update_rewards` with `disable_rewards: true` - (File: src/staking/staking.cairo)

### Summary
`update_rewards` is a public, permissionless function that accepts a caller-controlled `disable_rewards` parameter. When called with `disable_rewards: true`, it advances the global `last_reward_block` state without distributing any rewards. Because the `REWARDS_ALREADY_UPDATED` guard prevents any second call in the same block, an unprivileged attacker can call this every block to permanently suppress all consensus block rewards for every staker and delegator in the protocol.

### Finding Description
`update_rewards` is declared in the public `IStakingRewardsManager` interface and embedded via `#[abi(embed_v0)]`, making it callable by any non-zero address. Its implementation in `StakingRewardsManagerImpl` follows this sequence:

1. Calls `general_prerequisites()`, which only checks the contract is unpaused and the caller is non-zero — no role check.
2. Asserts `current_block_number > self.last_reward_block.read()` — the global singleton guard.
3. Validates the supplied `staker_address` is active and has non-zero balance (publicly queryable via `get_stakers`).
4. **Unconditionally** writes `self.last_reward_block.write(current_block_number)`.
5. If `disable_rewards || self.is_pre_consensus()` is true, returns immediately — no rewards distributed.

The critical flaw is step 4: `last_reward_block` is always advanced, even when `disable_rewards: true` causes an early return with zero reward distribution. Once `last_reward_block` equals the current block, every subsequent call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`. The attacker has consumed the block's single reward slot without paying out anything. [1](#0-0) [2](#0-1) 

### Impact Explanation
Every staker's `unclaimed_rewards_own` field and every delegator's cumulative reward index are frozen. The `RewardSupplier.unclaimed_rewards` counter is never incremented (because `update_unclaimed_rewards_from_staking_contract` is never called), so no STRK rewards accrue to anyone. This constitutes permanent freezing of unclaimed yield for all protocol participants — matching the **High** impact tier: *Permanent freezing of unclaimed yield*. [3](#0-2) [4](#0-3) 

### Likelihood Explanation
The attack requires no privileges, no token holdings, and no prior setup. The attacker only needs to:
- Know any valid, active staker address (publicly readable via `get_stakers`).
- Submit one transaction per block with `disable_rewards: true`.

On Starknet, transaction fees are low. A motivated attacker (competing protocol, griefing actor) can sustain this indefinitely. The entry path is fully reachable by any unprivileged public caller. [5](#0-4) 

### Recommendation
Do not advance `last_reward_block` when `disable_rewards` is `true`. Move the write to after the early-return guard:

```cairo
// Only mark the block as processed when rewards are actually distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... distribute rewards ...
```

Alternatively, restrict `update_rewards` to a privileged caller (e.g., the attestation contract or a designated consensus driver), or remove the public `disable_rewards` parameter entirely and handle the pre-consensus no-op internally without advancing `last_reward_block`. [6](#0-5) 

### Proof of Concept
1. Consensus rewards are active: `consensus_rewards_first_epoch` is set and `current_epoch >= consensus_rewards_first_epoch`.
2. Attacker queries `get_stakers(current_epoch)` to obtain any valid `staker_address` with non-zero staking power.
3. In block N, attacker calls `update_rewards(valid_staker, disable_rewards: true)`.
   - `last_reward_block` is written to N.
   - Function returns early; no rewards distributed; `unclaimed_rewards_own` unchanged.
4. Any legitimate call to `update_rewards` in block N reverts: `current_block_number > last_reward_block` is false.
5. Attacker repeats in block N+1, N+2, … indefinitely.
6. Result: all stakers' `unclaimed_rewards_own` remain at zero; pool `cumulative_rewards_trace` is never updated; delegators earn nothing. The `RewardSupplier` never receives a `update_unclaimed_rewards_from_staking_contract` call, so no L1 mint requests are triggered for staker/delegator rewards. [7](#0-6) [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
