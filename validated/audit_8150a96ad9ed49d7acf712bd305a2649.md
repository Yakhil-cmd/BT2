### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards via `update_rewards` with `disable_rewards: true` — (File: src/staking/staking.cairo)

---

### Summary

`update_rewards` is a public, permissionless function. It writes `last_reward_block` to the current block number **before** checking the `disable_rewards` flag. Any unprivileged caller can invoke it with `disable_rewards: true` to advance the global `last_reward_block` state without distributing any rewards. Every subsequent legitimate call in the same block then reverts with `REWARDS_ALREADY_UPDATED`, permanently freezing consensus-epoch yield for all stakers.

---

### Finding Description

`update_rewards` in `StakingRewardsManagerImpl` has no role guard — only `general_prerequisites()`, which checks pause state and non-zero caller:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

Inside `update_rewards`, the critical ordering flaw is:

```cairo
// Update last block rewards.
self.last_reward_block.write(current_block_number);   // ← state advanced here

if disable_rewards || self.is_pre_consensus() {
    return;                                            // ← exits without distributing
}
``` [2](#0-1) 

The guard that prevents double-execution runs at the top of the function:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

Because `last_reward_block` is a **single global slot** (not per-staker), one call with `disable_rewards: true` consumes the entire block's reward slot for every staker.

---

### Impact Explanation

An attacker who calls `update_rewards(valid_staker, disable_rewards: true)` at the start of every block:

1. Passes all validation checks (not paused, non-zero caller, staker active, non-zero balance).
2. Writes `current_block_number` into `last_reward_block`.
3. Returns immediately — zero rewards distributed.
4. Every legitimate call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`.

Sustained across all blocks, this permanently freezes all consensus-epoch unclaimed yield for every staker and delegator in the protocol.

**Matched impact**: *High — Permanent freezing of unclaimed yield.*

---

### Likelihood Explanation

- `update_rewards` is a fully public entrypoint with no role restriction.
- The attacker only needs any one valid, active, V3-migrated staker address (publicly readable from chain state).
- The cost is one cheap transaction per block; no capital at risk for the attacker.
- The attack is deterministic and requires no oracle manipulation, bridge access, or privileged key.

---

### Recommendation

**Short term**: Add an access-control guard so only an authorized caller (e.g., the sequencer reward-manager role) may invoke `update_rewards`. Alternatively, move `last_reward_block.write` to **after** the `disable_rewards` branch so that a no-op call does not consume the block slot.

**Long term**: Identify all state-advancing operations that can be triggered by unprivileged callers and add invariant tests asserting that legitimate reward distribution always succeeds when called in the same block as a `disable_rewards: true` call.

---

### Proof of Concept

1. Deploy the staking system; advance past `consensus_rewards_first_epoch`.
2. At the start of block `N`, attacker calls:
   ```
   staking.update_rewards(valid_staker_address, disable_rewards: true)
   ```
3. `last_reward_block` is now `N`; no rewards distributed.
4. Legitimate sequencer/rewards-manager calls:
   ```
   staking.update_rewards(staker_address, disable_rewards: false)
   ```
   → reverts: `REWARDS_ALREADY_UPDATED` (`current_block_number == last_reward_block`).
5. Repeat step 2 every block → all consensus rewards are permanently frozen. [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
