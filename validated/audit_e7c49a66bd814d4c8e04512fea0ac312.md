### Title
Single Malicious Participant Can Permanently Abort DKG/Reshare/Refresh via Wrong Session ID in Final Broadcast — (`src/dkg.rs`, `src/protocol/echo_broadcast.rs`)

### Summary

In `broadcast_success` (`src/dkg.rs`), the final round of `do_keyshare` requires **unanimous** agreement on `session_id` across all participants. A single malicious participant can broadcast `(true, wrong_session_id)` in this round. Because `echo_ready_thresholds` returns `(0, 0)` for `n <= 3`, the echo-broadcast protocol faithfully delivers the malicious value to all honest parties with no resistance. The subsequent check `vote_list.iter().all(|(_, ref sid)| sid == &session_id)` then fails for every honest party, causing all of them to return `Err(AssertionFailed)` — permanently aborting DKG, reshare, and refresh under the documented threshold of `n=3, t=2, f=1`.

---

### Finding Description

**Step 1 — Final round call site.**

`do_keyshare` calls `broadcast_success` as its very last step before returning the key output: [1](#0-0) 

**Step 2 — `broadcast_success` broadcasts and then checks session IDs.**

`broadcast_success` runs `do_broadcast` with `(true, session_id)` and then asserts that every entry in the returned `vote_list` carries the same `session_id`: [2](#0-1) 

The check at line 321 is a strict **all-or-nothing** predicate — one wrong entry aborts every honest party.

**Step 3 — Echo-broadcast thresholds are zero for `n ≤ 3`.**

`echo_ready_thresholds` explicitly returns `(0, 0)` when `n ≤ 3`, with the comment "no malicious parties are assumed": [3](#0-2) 

With `echo_t = 0` and `ready_t = 0`, the conditions `count > echo_t` and `count > 2 * ready_t` reduce to `count > 0`. A single echo or ready message is sufficient to advance every phase and deliver the value.

**Step 4 — Malicious value is faithfully delivered.**

When the malicious party sends `MessageType::Send((true, wrong_session_id))`:
- Each honest party receives it, immediately echoes it (line 202–208).
- With `echo_t = 0`, receiving even one echo triggers `MessageType::Ready(...)` (line 223–230).
- With `ready_t = 0`, receiving even one ready triggers delivery (line 293–307).

All honest parties therefore receive `(true, wrong_session_id)` in the malicious party's slot of `vote_list`. [4](#0-3) 

**Step 5 — The session-ID check aborts all honest parties.**

Back in `broadcast_success`, `wrong_session_id != session_id`, so the predicate at line 321 is `false` for every honest party, and all return `Err(AssertionFailed)`: [5](#0-4) 

---

### Impact Explanation

Every honest party aborts `do_keyshare` before producing any output. DKG, reshare, and refresh all call `do_keyshare` as their core routine: [6](#0-5) [7](#0-6) 

The abort is permanent: re-running the protocol with the same malicious participant produces the same outcome. This matches the **High** impact category: *Permanent denial of key generation, reshare, or refresh for honest parties under valid protocol inputs and documented trust assumptions.*

---

### Likelihood Explanation

The scenario is `n = 3, threshold = 2, f = 1`. The DKG threshold explicitly tolerates one malicious party. The attack requires only that the malicious party substitute a random 32-byte value for `session_id` in the final broadcast — a trivial deviation from the honest protocol. No cryptographic assumption needs to be broken. The malicious party does not even need to deviate in earlier rounds; all prior checks pass normally.

---

### Recommendation

The `broadcast_success` check must not require unanimous session-ID agreement. Two concrete fixes:

1. **Remove the session-ID check entirely.** The `session_id` was already established by consensus in round 1 via `do_broadcast`. Re-checking it in the final round adds no security and creates this abort vector.

2. **Tolerate up to `f` wrong entries.** Replace the `all(...)` predicate with a threshold check: abort only if more than `f = n - threshold` entries carry a wrong session ID.

Additionally, `echo_ready_thresholds` should be re-examined: returning `(0, 0)` for `n ≤ 3` means the echo-broadcast protocol provides zero Byzantine fault tolerance for the smallest supported participant sets, which contradicts the DKG threshold guarantee.

---

### Proof of Concept

```
Setup: participants = [P0, P1, P2], threshold = 2
P2 is malicious.

Round 1–4 of do_keyshare: all parties behave honestly.

Round 5 (broadcast_success):
  P0 calls do_broadcast(chan, participants, P0, (true, session_id))
  P1 calls do_broadcast(chan, participants, P1, (true, session_id))
  P2 calls do_broadcast(chan, participants, P2, (true, RANDOM_BYTES))  ← malicious

echo_ready_thresholds(3) = (0, 0)

For P2's broadcast instance:
  P0 receives Send((true, RANDOM_BYTES)) from P2 → echoes it
  P1 receives Send((true, RANDOM_BYTES)) from P2 → echoes it
  echo count for (true, RANDOM_BYTES) = 2 > 0 → both send Ready
  ready count = 2 > 0 → both deliver (true, RANDOM_BYTES) for P2's slot

vote_list at P0: [(true, session_id), (true, session_id), (true, RANDOM_BYTES)]
vote_list at P1: [(true, session_id), (true, session_id), (true, RANDOM_BYTES)]

Check: vote_list.iter().all(|(_, sid)| sid == &session_id)
  → false (RANDOM_BYTES ≠ session_id)
  → Err(AssertionFailed) for P0 and P1

Result: DKG permanently aborted for all honest parties.
```

async fn broadcast_success(
    chan: &mut SharedChannel,
    participants: &ParticipantList,
    me: Participant,
    session_id: HashOutput,
) -> Result<(), ProtocolError> {
    // broadcast node me succeded
    let vote_list = do_broadcast(chan, participants, me, (true, session_id)).await?;
    // unwrap here would never fail as the broadcast protocol ends only when the map is full
    let vote_list = vote_list
        .into_vec_or_none()
        .ok_or_else(|| ProtocolError::AssertionFailed("vote_list is empty".to_string()))?;
    // go through all the list of votes and check if any is fail or some does not contain the session id

    if !vote_list.iter().all(|(_, ref sid)| sid == &session_id) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                broadcast the wrong session id. Aborting Protocol!"
                .to_string(),
        ));
    }

    if !vote_list.iter().all(|&(boolean, _)| boolean) {
        return Err(ProtocolError::AssertionFailed(
            "A participant
                seems to have failed its checks. Aborting Protocol!"
                .to_string(),
        ));
    }
    // Wait for all the tasks to complete
    Ok(())
}

fn echo_ready_thresholds(n: usize) -> (usize, usize) {
    // case where no malicious parties are assumed: when n <= 3/
    // In this case the echo and ready thresholds are both 0
    // later we compare if we have collected more votes than these thresholds
    if n <= 3 {
        return (0, 0);
    }
    // we should always have n >= 3*threshold + 1
    let broadcast_threshold = (n - 1) / 3;
    let echo_threshold = usize::midpoint(n, broadcast_threshold);
    (echo_threshold, broadcast_threshold)
}
