### Title
Unrestricted `update_rewards` with `disable_rewards: true` Permanently Denies Block Rewards for All Stakers — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `staking.cairo` is callable by any non-zero address with no meaningful access control. It accepts a `disable_rewards` boolean that, when `true`, advances the global `last_reward_block` cursor without distributing any rewards. Because only one `update_rewards` call can succeed per block (enforced by the `current_block_number > last_reward_block` guard), an attacker can front-run every legitimate reward update, permanently destroying that block's rewards for all stakers.

---

### Finding Description

`update_rewards` is gated only by `general_prerequisites()`, which checks that the contract is not paused and the caller is non-zero. [1](#0-0) 

The function immediately writes the current block number to the global `last_reward_block` storage slot **before** checking `disable_rewards`: [2](#0-1) 

If `disable_rewards` is `true`, execution returns early with no rewards distributed: [3](#0-2) 

`last_reward_block` is a single global field shared across all stakers: [4](#0-3) 

The per-block uniqueness guard is: [5](#0-4) 

Because `last_reward_block` is global and is written unconditionally before the `disable_rewards` branch, a single call with `disable_rewards: true` for **any** valid staker consumes the entire block's reward slot. No subsequent call in the same block can pass the guard, so the block's rewards are permanently unrecoverable.

---

### Impact Explanation

Block rewards in V3 (consensus rewards) are calculated per-block and credited to `staker_info.unclaimed_rewards_own` only inside `_update_rewards`: [6](#0-5) 

If `update_rewards` is called with `disable_rewards: true`, `_update_rewards` is never reached. The `reward_supplier` is never notified, `unclaimed_rewards_own` is never incremented, and pool rewards are never forwarded. There is no mechanism to retroactively credit a skipped block. The yield for that block is permanently frozen/destroyed for every staker and every delegator.

**Allowed impact matched**: *High — Permanent freezing of unclaimed yield.*

---

### Likelihood Explanation

- The function is fully public; any EOA or contract can call it.
- The attacker only needs to pay gas — there is no stake, bond, or permission required.
- The attack is repeatable every block with a simple script or bot.
- Front-running is straightforward on Starknet because the attacker can observe the mempool and submit a transaction with higher priority.
- There is no penalty or cost imposed on the attacker for doing this.

This is the direct analog to the reported issue: just as malicious TSS nodes faced no slashing for disrupting consensus, here any caller faces no penalty for disrupting the reward distribution process.

---

### Recommendation

1. **Restrict `disable_rewards`**: Add an access-control check (e.g., `only_app_governor` or `only_security_agent`) so that only a privileged role can pass `disable_rewards: true`.
2. **Write `last_reward_block` after the `disable_rewards` branch**: Move `self.last_reward_block.write(current_block_number)` to after the early-return guard so that a disabled call does not consume the block slot.
3. **Consider removing `disable_rewards` entirely** and handling migration/special cases through a separate privileged function.

---

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Block `N` begins. Staker Alice is active with non-zero STRK balance.
3. Attacker calls `staking.update_rewards(alice_address, disable_rewards: true)`.
4. Inside the function: `current_block_number (N) > last_reward_block` → passes. `last_reward_block` is written to `N`. `disable_rewards == true` → early return. No rewards distributed.
5. Alice (or anyone else) calls `staking.update_rewards(alice_address, disable_rewards: false)` in the same block. `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
6. Block `N`'s rewards are permanently lost for Alice and all her delegators.
7. Attacker repeats step 3 every block at negligible cost, continuously denying all stakers their yield. [7](#0-6)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
