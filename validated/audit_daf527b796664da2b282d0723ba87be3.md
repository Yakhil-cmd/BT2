### Title
Unvalidated `disable_rewards` Parameter in Public `update_rewards` Allows Anyone to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable by any address and accepts an attacker-controlled `disable_rewards: bool` parameter. When called with `disable_rewards: true`, the function unconditionally advances the global `last_reward_block` checkpoint to the current block — consuming the block's one-time reward slot — but then returns early without distributing any rewards. Because the slot can only be consumed once per block, the rewards for that block are permanently lost. Any unprivileged caller can repeat this every block to freeze all consensus-era staker yield indefinitely.

### Finding Description

`update_rewards` is part of the `IStakingRewardsManager` ABI and carries no access-control guard beyond `general_prerequisites()` (which only checks the contract is unpaused and the caller is non-zero). [1](#0-0) 

The critical sequence inside the function is:

1. Assert `current_block_number > last_reward_block` — enforcing the one-call-per-block invariant.
2. **Write `last_reward_block = current_block_number`** — consuming the slot.
3. Check `if disable_rewards || self.is_pre_consensus() { return; }` — skipping all reward math. [2](#0-1) 

Because step 2 happens **before** step 3, calling the function with `disable_rewards: true` permanently marks the current block as "processed" while distributing nothing. Any subsequent legitimate call in the same block fails with `REWARDS_ALREADY_UPDATED`.

The `disable_rewards` parameter is never validated against any role, flag, or protocol state that would restrict who may set it to `true`. The analog to the external report is exact: just as `sigMeta.newPrivilegeValue` was unvalidated and could be set to `0` by anyone to cancel a recovery, `disable_rewards` is unvalidated and can be set to `true` by anyone to cancel a block's reward distribution.

### Impact Explanation

**Impact: High — Permanent freezing of unclaimed yield.**

Each block's reward is a fixed amount computed by `calculate_block_rewards` from the minting curve. Once `last_reward_block` is advanced for a block without distributing rewards, those rewards are gone — they are not rolled forward to the next block and are never credited to any staker or pool member. An attacker who calls `update_rewards(<any_active_staker>, true)` at every block permanently prevents all stakers and delegators from accumulating consensus-era rewards. [3](#0-2) 

### Likelihood Explanation

**Likelihood: High.**

The function is permissionlessly callable. The only preconditions are that the contract is unpaused and the supplied `staker_address` is an active staker with non-zero balance — both trivially satisfiable by observing on-chain state. The attacker needs no tokens, no privileged role, and no special setup. The attack costs only gas and can be automated to run every block.

### Recommendation

Two complementary fixes:

1. **Remove `disable_rewards` from the public interface.** The pre-consensus early-return is already handled by `self.is_pre_consensus()` inside the function; the external `disable_rewards` flag is redundant and dangerous. Delete the parameter and the `|| disable_rewards` branch.

2. **If `disable_rewards` must remain for internal use**, gate the function so that only a trusted role (e.g., `SECURITY_AGENT` or the attestation contract) may pass `true`, and revert on any other caller attempting to set it.

### Proof of Concept

```
// Attacker (any EOA) calls at every block:
staking.update_rewards(
    staker_address: <any_active_staker>,
    disable_rewards: true   // unvalidated — accepted by the contract
);
// Effect:
//   last_reward_block = current_block   (slot consumed)
//   _update_rewards() never called      (rewards skipped)
//
// Legitimate keeper call in the same block:
staking.update_rewards(staker_address: <staker>, disable_rewards: false);
// → panics: "REWARDS_ALREADY_UPDATED"
//
// Rewards for this block are permanently lost.
// Attacker repeats every block → all consensus yield frozen indefinitely.
```

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }

        /// Calculates the rewards for a block in the current epoch (for STRK and BTC).
        ///
        /// Precondition: `curr_epoch` must be `get_current_epoch()`, it's passed as a param to save
        /// storage reads. This precondition is only violated during the migration to consensus
        /// rewards.
        // TODO: Consider view?
        fn calculate_block_rewards(
            ref self: ContractState,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) -> (Amount, Amount) {
            if curr_epoch > self.last_calculated_epoch.read() {
                self.last_calculated_epoch.write(curr_epoch);
                let block_rewards = reward_supplier_dispatcher.update_current_epoch_block_rewards();
                self.block_rewards.write(block_rewards);
                block_rewards
            } else {
                self.block_rewards.read()
            }
        }
