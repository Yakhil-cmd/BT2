2. `use_global_contract()` stores a **live reference** `AccountContract::GlobalByAccount(id.clone())` — not a snapshot of the code hash

### Title
Global contract `AccountId` mode allows any deployer to silently swap code and drain all opted-in user accounts — (`runtime/runtime/src/global_contracts.rs`)

### Summary
`GlobalContractDeployMode::AccountId` lets any unprivileged NEAR account deploy a global contract keyed by their own account ID, then later redeploy an arbitrary replacement. Every account that previously called `UseGlobalContract` with `GlobalContractIdentifier::AccountId(deployer)` holds a live pointer to the deployer's account ID — not a snapshot of the code hash. When the deployer pushes a malicious replacement, all opted-in accounts silently execute the new code on their next function call, enabling the deployer to drain their NEAR balances via `promise_batch_action_transfer`.

### Finding Description

**Step 1 — No guard on re-deployment.**
`action_deploy_global_contract()` checks only that the deployer has enough balance to pay storage cost. There is no check for whether a global contract already exists under the same account ID: [1](#0-0) 

The only rejection path is `LackBalanceForState`. Any account with sufficient NEAR can overwrite an existing global contract at any time.

**Step 2 — Unconditional overwrite during distribution.**
`apply_distribution_current_shard()` writes the new code bytes directly to the trie key without checking whether a previous version exists: [2](#0-1) 

**Step 3 — Victim accounts hold a live pointer, not a hash snapshot.**
`use_global_contract()` stores `AccountContract::GlobalByAccount(id.clone())` in the victim's account record: [3](#0-2) 

This is a mutable indirection. The victim's account does not record which code hash was current at opt-in time. Every subsequent call resolves the code by looking up `TrieKey::GlobalContractCode { identifier: AccountId(deployer) }` at execution time — whatever bytes are stored there at that moment.

**Step 4 — Documented mutability, no user protection.**
The protocol explicitly documents this as a feature: [4](#0-3) 

There is no timelock, no consent mechanism, no event that lets opted-in accounts detect or react to a code swap before their next call is executed.

**Step 5 — Confirmed by production test.**
`test_global_contract_update` proves the end-to-end behavior: accounts using a trivial global contract (no methods) immediately execute the new `rs_contract` after the deployer redeploys — without any action by the victim accounts: [5](#0-4) 

### Impact Explanation
A malicious deployer can drain the full NEAR balance of every account that opted into their global contract via `GlobalContractIdentifier::AccountId`. The malicious WASM calls `promise_batch_action_transfer` targeting the deployer's own account, transferring the victim's `account_balance`. Because the victim's account holds `AccountContract::GlobalByAccount`, the runtime resolves the deployer's current code at call time — the victim has no way to detect or prevent the swap before their next transaction is processed.

Corrupted value: `account.amount` of every opted-in victim account, reduced to zero (or near-zero after gas).

### Likelihood Explanation
The attacker is any unprivileged NEAR account. The steps require only:
1. Paying storage cost to deploy a legitimate global contract with `AccountId` mode.
2. Waiting for users to call `UseGlobalContract` with the deployer's account ID.
3. Paying storage cost again to redeploy a malicious contract.

No validator, admin, or special key is needed. The attack is fully on-chain and permissionless.

### Recommendation
Two mitigations, either sufficient:

1. **Make `AccountId` mode immutable after first deployment.** In `action_deploy_global_contract()`, check whether `TrieKey::GlobalContractCode { identifier: AccountId(account_id) }` already exists in the trie and reject the action if so. This aligns `AccountId` mode with `CodeHash` mode's immutability guarantee.

2. **Enforce a timelock on re-deployment.** Record the block height of the last deployment in a new trie key (e.g., `GlobalContractLastUpdated`). Reject re-deployment attempts within a configurable delay, giving opted-in accounts time to switch to `GlobalContractIdentifier::CodeHash` or a local contract.

### Proof of Concept

```
1. Attacker calls DeployGlobalContractAction { code: <legitimate_wasm>, deploy_mode: AccountId }
   → action_deploy_global_contract() passes (balance check only)
   → apply_distribution_current_shard() writes legitimate code to
     TrieKey::GlobalContractCode { identifier: AccountId("attacker.near") }

2. Victim calls UseGlobalContractAction { contract_identifier: AccountId("attacker.near") }
   → use_global_contract() sets victim account.contract =
     AccountContract::GlobalByAccount("attacker.near")
   → Victim's account now holds a live pointer, not a hash snapshot.

3. Attacker calls DeployGlobalContractAction { code: <malicious_wasm>, deploy_mode: AccountId }
   → No guard prevents this.
   → apply_distribution_current_shard() overwrites:
     TrieKey::GlobalContractCode { identifier: AccountId("attacker.near") }
     with malicious_wasm bytes (line 210).

4. Any caller invokes a function on the victim's account.
   → Runtime resolves AccountContract::GlobalByAccount("attacker.near")
   → Loads malicious_wasm from the trie.
   → Malicious code executes promise_batch_action_transfer(victim_balance → attacker).

5. Victim's account.amount = 0. Attacker receives victim's full balance.
```

pub(crate) fn action_deploy_global_contract(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    apply_state: &ApplyState,
    deploy_contract: &DeployGlobalContractAction,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    let _span = tracing::debug_span!(target: "runtime", "action_deploy_global_contract").entered();

    let storage_cost = apply_state
        .config
        .fees
        .storage_usage_config
        .global_contract_storage_amount_per_byte
        .saturating_mul(deploy_contract.code.len() as u128);
    let Some(updated_balance) = account.amount().checked_sub(storage_cost) else {
        result.result = Err(ActionErrorKind::LackBalanceForState {
            account_id: account_id.clone(),
            amount: storage_cost,
        }
        .into());
        return Ok(());
    };
    result.tokens_burnt =
        result.tokens_burnt.checked_add(storage_cost).ok_or(IntegerOverflowError)?;
    account.set_amount(updated_balance);

    initiate_distribution(
        state_update,
        account_id.clone(),
        deploy_contract.code.clone(),
        &deploy_contract.deploy_mode,
        apply_state.shard_id,
        result,
    )?;

    Ok(())
}

#[test]
fn test_global_contract_update() {
    let mut env = GlobalContractsTestEnv::setup(Balance::from_near(1000));
    let use_accounts = [env.account_shard_0.clone(), env.account_shard_1.clone()];

    env.deploy_trivial_global_contract(GlobalContractDeployMode::AccountId);

    for account in &use_accounts {
        env.use_global_contract(
            account,
            GlobalContractIdentifier::AccountId(env.deploy_account.clone()),
        );

        // Currently deployed trivial contract doesn't have any methods,
        // so we expect any function call to fail with MethodNotFound error
        let call_tx = env.call_global_contract_tx(account.clone(), account.clone());
        let call_outcome = env.execute_tx(call_tx);
        assert_matches!(
            call_outcome.status,
            FinalExecutionStatus::Failure(TxExecutionError::ActionError(ActionError {
                kind: ActionErrorKind::FunctionCallError(FunctionCallError::MethodResolveError(
                    MethodResolveError::MethodNotFound
                )),
                index: _
            }))
        );
    }

    env.deploy_global_contract(GlobalContractDeployMode::AccountId);

    for account in &use_accounts {
        // Function call should be successful after deploying rs contract
        // containing the function we call here
        env.assert_call_global_contract_success(account.clone(), account.clone());
    }
}
