### Title
Unprivileged Caller Can Permanently Freeze All Stakers' Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable with no access control. It accepts a `disable_rewards: bool` parameter. When called with `disable_rewards: true`, the function updates the **global** `last_reward_block` storage variable before returning early without distributing any rewards. Because `last_reward_block` is a single global variable, any subsequent call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`. An unprivileged attacker can call this once per block to permanently prevent all stakers from ever receiving consensus rewards.

### Finding Description

`StakingRewardsManagerImpl::update_rewards` is exposed as a public ABI function with no role-based access control — only `general_prerequisites()` is checked, which only asserts the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function accepts a caller-controlled `disable_rewards: bool` parameter. Critically, the global `last_reward_block` is written **before** the `disable_rewards` branch: [2](#0-1) 

`last_reward_block` is a single global storage slot, not a per-staker mapping: [3](#0-2) 

The guard at the top of the function enforces that only one call per block is allowed across **all** stakers: [4](#0-3) 

Consequence: an attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` at the start of every block:
1. Advances `last_reward_block` to the current block.
2. Returns without invoking `_update_rewards`, so no rewards are calculated or distributed.
3. Blocks every legitimate call to `update_rewards` for the remainder of that block.

The full reward distribution path that is bypassed: [5](#0-4) 

### Impact Explanation

All stakers and delegators permanently lose their consensus-period block rewards. This constitutes **permanent freezing of unclaimed yield** for every participant in the protocol. The `unclaimed_rewards_own` field of every staker and the cumulative rewards trace of every pool contract are never updated, so no rewards can ever be claimed.

This matches the allowed High impact: *"Permanent freezing of unclaimed yield or unclaimed royalties."*

### Likelihood Explanation

- The function is fully public; any EOA or contract can call it.
- The attacker only needs to supply any currently-active staker address (publicly readable from the `stakers` vector).
- The only cost is one Starknet transaction per block. Starknet transaction fees are low, making sustained griefing economically viable.
- No privileged key, bridge access, or external dependency is required.

### Recommendation

1. **Remove the `disable_rewards` parameter from the public interface.** Handle the pre-consensus early-return internally based on `is_pre_consensus()` alone.
2. If `disable_rewards` must remain for protocol-internal use (e.g., migration), gate it behind a privileged role check such as `self.roles.only_app_governor()` or `self.roles.only_operator()`.
3. Alternatively, move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard so that a no-op call does not consume the block slot.

### Proof of Concept

```
// Attacker script — run once per block:
// 1. Read any active staker address from staking.stakers[0].
// 2. Call:
staking.update_rewards(
    staker_address: <any_active_staker>,
    disable_rewards: true,
);
// Result:
//   - last_reward_block = current_block_number  (global, blocks all others)
//   - No rewards distributed to any staker or pool
//   - Any legitimate update_rewards call in this block reverts: REWARDS_ALREADY_UPDATED
// 3. Repeat next block.
// All stakers' unclaimed_rewards_own remains zero indefinitely.
```

The root cause is directly analogous to the Sound Protocol report: fees/rewards accrue through a mechanism (here, `update_rewards`) that any unprivileged caller can invoke with parameters that suppress the actual accounting, causing the protocol's reward distribution to be silently bypassed.

            btc_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_stake: NormalizedAmount,
            mut staker_info: InternalStakerInfoLatest,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            reward_supplier_dispatcher: IRewardSupplierDispatcher,
            curr_epoch: Epoch,
        ) {
            // Calculate self rewards.

                .calculate_staker_own_rewards(
                    :staker_address, :strk_total_rewards, :strk_total_stake, :curr_epoch,
                );

            // Calculate pools rewards.
            let (commission_rewards, total_pools_rewards, pools_rewards_data) = if staker_pool_info
                .has_pool() {
                self
                    .calculate_staker_pools_rewards(
                        :staker_address,
                        :staker_pool_info,
                        :strk_total_rewards,
                        :strk_total_stake,
                        :btc_total_rewards,
                        :btc_total_stake,
                        :curr_epoch,
                    )
            } else {
                (Zero::zero(), Zero::zero(), array![])
            };

            // Update reward supplier.
            let staker_rewards = staker_own_rewards + commission_rewards;
            // Update total rewards.
            reward_supplier_dispatcher
                .update_unclaimed_rewards_from_staking_contract(
                    rewards: staker_rewards + total_pools_rewards,
                );
            // Claim pools rewards.
            claim_from_reward_supplier(
                :reward_supplier_dispatcher,
                amount: total_pools_rewards,
                token_dispatcher: strk_token_dispatcher(),
            );
            // Update staker rewards.
            staker_info.unclaimed_rewards_own += staker_rewards;

            // Update pools rewards.
            let pool_rewards_list = self.update_pool_rewards(:staker_address, :pools_rewards_data);
