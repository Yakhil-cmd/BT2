### Title
Bank Precompile `transfer` Allows Unauthorized Native Token Transfer From Arbitrary User Accounts - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The bank precompile's `transfer` method accepts a `sender` address as a caller-supplied argument and uses it directly as the source of a `bankKeeper.SendCoins` call, without verifying that `sender` equals the EVM contract invoking the precompile (`contract.Caller()`). Any EVM contract can therefore drain native `evm/<contract>` tokens from any user account that holds them, with no approval or consent from the account owner.

### Finding Description
`BankContract.Run` in `x/cronos/keeper/precompiles/bank.go` handles the `transfer` method as follows:

```go
case TransferMethodName:
    args, err := method.Inputs.Unpack(contract.Input[4:])
    sender    := args[0].(common.Address)   // caller-supplied, never validated
    recipient := args[1].(common.Address)
    amount    := args[2].(*big.Int)
    from := sdk.AccAddress(sender.Bytes())
    to   := sdk.AccAddress(recipient.Bytes())
    denom := EVMDenom(contract.Caller())    // "evm/<calling_contract>"
    amt   := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
    ...
    bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt))
``` [1](#0-0) 

The `denom` is correctly scoped to the calling contract (`evm/<contract.Caller()>`), but `from` is taken verbatim from the ABI argument `sender`. There is no guard of the form `require sender == contract.Caller()`. The only validation present is a blocked-address check on the *recipient*, not the source. [2](#0-1) 

The `burn` path has the same structural issue — it burns from an arbitrary `addr` argument without checking that `addr == contract.Caller()` — but `transfer` is the direct analog to the external report's unauthorized-deposit pattern. [3](#0-2) 

### Impact Explanation
An attacker-controlled contract can call `transfer(victimAddress, attackerAddress, balance)` against the bank precompile and move all native `evm/<attacker_contract>` tokens out of the victim's Cosmos-side account in a single transaction. Because `bankKeeper.SendCoins` operates on the Cosmos native bank module, the transfer is final and irreversible. This constitutes an **unauthorized transfer of precompile-controlled assets**, which is Critical under the allowed impact scope.

### Likelihood Explanation
The prerequisite is that victim accounts hold native tokens of the denom `evm/<attacker_contract>`. This is realistic whenever:
- A DeFi protocol or reward system built on the bank precompile distributes `evm/<contract>` tokens to users (e.g., as liquidity rewards, staking receipts, or collateral receipts).
- A user receives such tokens through an IBC transfer or cross-chain bridge that maps to a bank-precompile-managed denom.

Once users hold the tokens, the attacker needs only to call `transfer(victim, attacker, amount)` from the issuing contract — a single unprivileged EVM transaction. No admin key, governance vote, or validator compromise is required.

### Recommendation
Enforce that the `sender` argument equals the EVM caller inside the `TransferMethodName` case:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be the calling contract")
}
```

Alternatively, remove the explicit `sender` parameter entirely and derive the source address from `contract.Caller()`, consistent with how `denom` is already derived. Apply the same fix to the `burn` path, which similarly accepts an arbitrary target address. [1](#0-0) 

### Proof of Concept

1. Attacker deploys `MaliciousToken` contract on Cronos EVM.
2. `MaliciousToken` calls `bankPrecompile.mint(victimAddress, 1_000_000)` — this mints `1_000_000 evm/<MaliciousToken>` native tokens to the victim (e.g., disguised as an airdrop or LP reward).
3. Victim's Cosmos-side bank balance now shows `1_000_000 evm/<MaliciousToken>`.
4. Attacker calls `bankPrecompile.transfer(victimAddress, attackerAddress, 1_000_000)` from `MaliciousToken`.
5. `BankContract.Run` unpacks `sender = victimAddress`, computes `denom = "evm/<MaliciousToken>"`, and executes `bankKeeper.SendCoins(ctx, victim, attacker, 1_000_000 evm/<MaliciousToken>)` — no approval, no signature from the victim.
6. Victim's balance is zero; attacker holds all tokens.

The same path applies to any legitimate protocol that uses the bank precompile to issue `evm/<contract>` tokens to users: the issuing contract can drain all user balances at will because `sender` is never validated against `contract.Caller()`. [4](#0-3)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
	case BalanceOfMethodName:
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		token := args[0].(common.Address)
		addr := args[1].(common.Address)
		// query from storage
		balance := bc.bankKeeper.GetBalance(stateDB.Context(), sdk.AccAddress(addr.Bytes()), EVMDenom(token)).Amount.BigInt()
		return method.Outputs.Pack(balance)
	case TransferMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		sender := args[0].(common.Address)
		recipient := args[1].(common.Address)
		amount := args[2].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		from := sdk.AccAddress(sender.Bytes())
		to := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(to); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if err := bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt)); err != nil {
				return errorsmod.Wrap(err, "fail to send coins in precompiled contract")
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
