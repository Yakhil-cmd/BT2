### Title
Unvalidated `disable_rewards` Parameter in `update_rewards` Allows Any Caller to Permanently Skip Block Reward Distribution — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `staking.cairo` is callable by any unprivileged address and accepts a `disable_rewards: bool` parameter with no access control. Any attacker can call `update_rewards(valid_staker, disable_rewards: true)` to advance the global `last_reward_block` cursor while suppressing reward distribution, permanently denying all stakers their block rewards for that slot.

---

### Finding Description

`update_rewards` is exposed as a public ABI function under `StakingRewardsManagerImpl`. Its only gate is `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no role restriction whatsoever. [1](#0-0) 

The function writes the current block number into the global `last_reward_block` storage slot unconditionally, before the `disable_rewards` branch: [2](#0-1) 

When `disable_rewards` is `true`, execution returns immediately after that write, skipping `_update_rewards` entirely: [3](#0-2) 

Because `last_reward_block` is a single global slot shared across all stakers, any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

An attacker who calls `update_rewards(any_valid_active_staker, disable_rewards: true)` once per block permanently consumes that block's reward slot for the entire protocol. The block rewards calculated by `calculate_block_rewards` / `reward_supplier.update_current_epoch_block_rewards` are never credited to any staker or pool for that block; they are irrecoverably skipped.

The `staker_address` argument only needs to satisfy "staker exists, is active, and has non-zero STRK balance" — all of which are publicly observable on-chain from `NewStaker` events. [5](#0-4) 

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

For every block in which the attacker fires the call first, the block reward (STRK and BTC) is never distributed to any staker or delegation pool. Because `last_reward_block` is advanced, no legitimate caller can retroactively claim that block's rewards. Sustained over many blocks, this drains the entire yield stream for all protocol participants. The reward supplier contract retains the tokens, but the staking contract never requests or distributes them for the skipped blocks.

---

### Likelihood Explanation

**High.** The entry point is fully permissionless — any EOA or contract can call it. The only cost is Starknet gas per block. No privileged access, no leaked key, no oracle manipulation, and no third-party dependency is required. The attacker only needs to know one currently-active staker address, which is trivially obtained from on-chain events.

---

### Recommendation

Restrict `update_rewards` to a trusted caller (e.g., the consensus/attestation contract, or a dedicated `REWARDS_MANAGER` role). Alternatively, remove the `disable_rewards` parameter from the public interface and handle the "skip" logic internally based on on-chain conditions that cannot be spoofed by an unprivileged caller.

---

### Proof of Concept

```
// Attacker script — run once per block
loop {
    staking.update_rewards(
        staker_address = <any valid active staker>,
        disable_rewards = true,
    );
    // last_reward_block is now set to current block.
    // No rewards distributed. Legitimate callers revert with REWARDS_ALREADY_UPDATED.
    wait_for_next_block();
}
```

Step-by-step:
1. Attacker observes a valid, active staker address from `NewStaker` events.
2. At each new block, attacker calls `update_rewards(staker, disable_rewards: true)`.
3. `general_prerequisites()` passes (contract unpaused, caller non-zero).
4. Staker existence and activity checks pass.
5. `last_reward_block` is written to the current block number.
6. Function returns early — `_update_rewards` is never called.
7. Any honest caller attempting `update_rewards` in the same block receives `REWARDS_ALREADY_UPDATED`.
8. Block rewards for that slot are permanently unaccounted for.

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
