### Title
Unchecked Precompile Call Return Value in `withdrawToNear`/`withdrawToEthereum` Causes Permanent Token Loss - (File: `etc/eth-contracts/contracts/EvmErc20.sol`)

---

### Summary

Both `EvmErc20.sol` and `EvmErc20V2.sol` burn the caller's ERC-20 tokens **before** calling the exit precompile, and then invoke the precompile via inline assembly without checking the return value. If the precompile call fails for any reason, the burn is not reverted, the cross-domain message is never sent, and the user's tokens are permanently destroyed with no corresponding release of NEP-141 tokens on NEAR (or ETH on Ethereum).

---

### Finding Description

In `EvmErc20.sol`, both `withdrawToNear` and `withdrawToEthereum` follow this pattern:

```solidity
function withdrawToNear(bytes memory recipient, uint256 amount) external override {
    _burn(_msgSender(), amount);          // tokens destroyed here

    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", amount_b, recipient);
    uint input_size = 1 + 32 + recipient.length;

    assembly {
        let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
        // res is NEVER checked
    }
}
``` [1](#0-0) 

The same pattern appears in `withdrawToEthereum` in `EvmErc20.sol`: [2](#0-1) 

And identically in both functions of `EvmErc20V2.sol`: [3](#0-2) [4](#0-3) 

The EVM `call` opcode returns `0` on failure and `1` on success. The variable `res` is assigned but never read after the assembly block. There is no `require(res == 1)` or equivalent guard. Because `_burn` executes unconditionally before the assembly block, a failed precompile call leaves the state with tokens burned and no cross-domain release message dispatched.

The `ExitToNear` precompile (`engine-precompiles/src/native.rs`) can return `ExitError` (causing `call` to return 0) in several reachable conditions: [5](#0-4) 

- `ExitError::OutOfGas` — if the gas forwarded to the precompile is less than `costs::EXIT_TO_NEAR_GAS`
- `ExitError::Other("ERR_TARGET_TOKEN_NOT_FOUND")` — if the ERC-20→NEP-141 mapping is absent
- `ExitError::Other("ERR_INVALID_IN_DELEGATE")` — if called via `delegatecall`

The same failure modes apply to `ExitToEthereum`: [6](#0-5) 

---

### Impact Explanation

**Critical — Permanent freezing/burning of funds.**

When the precompile call fails silently:
- For `withdrawToNear`: ERC-20 tokens are burned on Aurora, but the corresponding NEP-141 tokens held by the Aurora engine contract on NEAR are never released to the recipient. The NEP-141 tokens remain locked in the Aurora engine account forever with no recovery path.
- For `withdrawToEthereum`: ERC-20 tokens are burned on Aurora, but the ETH/ERC-20 custodian on Ethereum never receives the withdrawal instruction. Funds are permanently lost.

There is no admin recovery function that can re-issue the cross-domain message after the fact, because the burn has already been finalized on-chain.

---

### Likelihood Explanation

**Medium-High.** Every deployed `EvmErc20` / `EvmErc20V2` instance (one per bridged NEP-141 token) exposes these two functions to any token holder. The most reliable trigger is a transaction submitted with a gas limit that is sufficient for the `_burn` call but insufficient for the precompile's required gas. A user who miscalculates gas, or a contract that calls `withdrawToNear` with a fixed gas stipend below the precompile threshold, will silently lose funds. This requires no special privilege — any ERC-20 token holder can trigger it.

---

### Recommendation

Check the return value of the assembly `call` and revert on failure, so that the `_burn` is rolled back if the precompile does not succeed:

```solidity
assembly {
    let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
    if iszero(res) { revert(0, 0) }
}
```

Apply this fix to all four assembly blocks across `EvmErc20.sol` and `EvmErc20V2.sol`.

---

### Proof of Concept

1. Deploy an `EvmErc20` instance mapped to a NEP-141 token (standard bridge setup).
2. Mint tokens to address `A`.
3. From address `A`, call `withdrawToNear(recipient, amount)` with a gas limit set just above the cost of `_burn` but below `costs::EXIT_TO_NEAR_GAS` (the precompile's required gas).
4. The `_burn` executes successfully — `A`'s balance decreases by `amount`.
5. The `call` to the `ExitToNear` precompile returns `0` (out of gas within the precompile).
6. `res` is `0` but is never checked; the function returns without reverting.
7. Observe: `A`'s ERC-20 balance is reduced by `amount`, the NEP-141 balance of the Aurora engine account on NEAR is unchanged, and the recipient receives nothing. The tokens are permanently lost. [1](#0-0) [7](#0-6)

impl<I: IO> Precompile for ExitToNear<I> {
    fn required_gas(_input: &[u8]) -> Result<EthGas, ExitError> {
        Ok(costs::EXIT_TO_NEAR_GAS)
    }

    #[allow(clippy::too_many_lines)]
    fn run(
        &self,
        input: &[u8],
        target_gas: Option<EthGas>,
        context: &Context,
        is_static: bool,
    ) -> EvmPrecompileResult {
        // ETH (base) transfer input format: (85 bytes)
        //  - flag (1 byte)
        //  - refund_address (20 bytes), present if the feature "error_refund" is enabled
        //  - recipient_account_id (max MAX_INPUT_SIZE - 20 - 1 bytes)
        // ERC-20 transfer input format: (124 bytes)
        //  - flag (1 byte)
        //  - refund_address (20 bytes), present if the feature "error_refund" is enabled.
        //  - amount (32 bytes)
        //  - recipient_account_id (max MAX_INPUT_SIZE - 1 - (20) - 32 bytes)
        //  - `:unwrap` suffix in a case of wNEAR (7 bytes)
        let required_gas = Self::required_gas(input)?;

        if let Some(target_gas) = target_gas
            && required_gas > target_gas
        {
            return Err(ExitError::OutOfGas);
        }

        // It's not allowed to call exit precompiles in static mode
        if is_static {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_STATIC")));
        } else if context.address != exit_to_near::ADDRESS.raw() {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_DELEGATE")));
        }
