### Title
Gateway `skip_stateful_validations` Bypasses `__validate__` for Nonce-1 Invokes via Unguarded `account_tx_in_pool_or_recent_block` Proxy Check - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function uses `account_tx_in_pool_or_recent_block` as a proxy for "a deploy_account is pending for this account." The check is not specific to deploy_account transactions — it returns `true` for **any** transaction from the address in the mempool or recent block. An unprivileged attacker who observes a pending deploy_account for account A can submit an invoke with nonce=1 and an arbitrary/invalid signature for A, causing the gateway to skip the `__validate__` entry-point call and admit the invalid transaction to the mempool.
