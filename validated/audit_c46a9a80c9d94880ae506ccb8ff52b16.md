### Title
Missing Two-Step Procedure for Contract Deployment/Removal Authorizer Updates - (File: fvm/blueprints/scripts/setContractOperationAuthorizersTransactionTemplate.cdc)

### Summary
The Cadence transaction `setContractOperationAuthorizersTransactionTemplate.cdc`, invoked via `SetContractDeploymentAuthorizersTransaction` / `SetContractRemovalAuthorizersTransaction`, atomically replaces the entire list of addresses authorized to deploy or remove contracts in a single step. There is no pending/acceptance phase. If the service account operator supplies an incorrect address, that address immediately and irrevocably gains contract-deployment or contract-removal authority over the Flow network, while the previously authorized addresses are simultaneously stripped of their privileges.

### Finding Description
The transaction template is:

```cadence
transaction(addresses: [Address], path: StoragePath) {
    prepare(signer: auth(Storage) &Account) {
        signer.storage.load<[Address]>(from: path)
        signer.storage.save(addresses, to: path)
    }
}
``` [1](#0-0) 

This template is embedded and used by two Go-level helpers:

- `SetContractDeploymentAuthorizersTransaction` — writes to `ContractDeploymentAuthorizedAddressesPath`
- `SetContractRemovalAuthorizersTransaction` — writes to `ContractRemovalAuthorizedAddressesPath` [2](#0-1) 

The FVM reads these paths at transaction execution time via `GetAuthorizedAccounts`:

```go
value, err := runtime.ReadStored(
    common.Address(service),
    path)
``` [3](#0-2) 

And enforces them in `isAuthorizedForDeployment` / `isAuthorizedForRemoval`: [4](#0-3) 

The entire authorized list is replaced atomically in one transaction. There is no intermediate "pending" state, no acceptance step by the newly nominated addresses, and no time-lock. The old list is discarded the moment the new one is saved.

### Impact Explanation
If the service account operator mistakenly includes a wrong address (e.g., a typo, a zero address, or an address belonging to an unintended party), that address **immediately** acquires the ability to deploy or remove arbitrary Cadence contracts on the network. Simultaneously, the previously authorized addresses lose their privileges. Because the operation is atomic and irreversible within the same transaction, there is no window to detect and abort the mistake before it takes effect. A malicious or mistaken address with contract-deployment authority can deploy contracts that alter token behavior, drain vaults, or corrupt protocol-level system contracts.

### Likelihood Explanation
The service account is operated by a small set of privileged operators. Human error during address list management (copy-paste mistakes, stale addresses, wrong network addresses) is a realistic operational risk, especially during epoch transitions or protocol upgrades when the authorized list is most likely to change. The single-step replacement makes any such error immediately and fully effective.

### Recommendation
Adopt a two-step procedure analogous to `approve`/`accept`:

1. **Propose**: The service account stores a *pending* authorized-address list at a separate storage path, along with a minimum activation block height.
2. **Confirm**: A second transaction (signed by the service account, executable only after the activation height) promotes the pending list to the active list.

This gives operators a window to detect and cancel an erroneous proposal before it takes effect. Additionally, emit an on-chain event at proposal time so that monitoring infrastructure can alert operators to unexpected changes.

### Proof of Concept

1. Service account operator intends to authorize `0xAAAAAAAAAAAAAAAA` but accidentally passes `0xBBBBBBBBBBBBBBBB` (a typo or wrong-network address).
2. The operator submits a transaction using `SetContractDeploymentAuthorizersTransaction(serviceAddress, []flow.Address{0xBBBBBBBBBBBBBBBB})`.
3. `setContractOperationAuthorizersTransactionTemplate.cdc` executes: it loads the old list and immediately saves `[0xBBBBBBBBBBBBBBBB]` to `ContractDeploymentAuthorizedAddressesPath` in the service account's storage.
4. From the very next transaction onward, `GetAuthorizedAccounts` returns `[0xBBBBBBBBBBBBBBBB]`.
5. `isAuthorizedForDeployment` now grants contract-deployment rights to `0xBBBBBBBBBBBBBBBB` and denies them to all previously authorized addresses — including the service account itself if it was not re-included.
6. The operator has no on-chain mechanism to revert this without executing another privileged transaction, and if `0xBBBBBBBBBBBBBBBB` is controlled by a third party, that party can immediately deploy arbitrary contracts before the operator can correct the mistake. [5](#0-4) [6](#0-5)

func setContractAuthorizersTransaction(
	path cadence.Path,
	serviceAccount flow.Address,
	authorized []flow.Address,
) (*flow.TransactionBodyBuilder, error) {
	addressValues := make([]cadence.Value, 0, len(authorized))
	for _, address := range authorized {
		addressValues = append(
			addressValues,
			cadence.BytesToAddress(address.Bytes()))
	}

	addressesArg, err := jsoncdc.Encode(cadence.NewArray(addressValues))
	if err != nil {
		return nil, err
	}

	pathArg, err := jsoncdc.Encode(path)
	if err != nil {
		return nil, err
	}

	return flow.NewTransactionBodyBuilder().
		SetScript([]byte(setContractOperationAuthorizersTransactionTemplate)).
		SetPayer(serviceAccount).
		AddAuthorizer(serviceAccount).
		AddArgument(addressesArg).
		AddArgument(pathArg), nil
}
```

func (updater *ContractUpdaterImpl) isAuthorizedForDeployment(
	signingAccounts []flow.Address,
) bool {
	if updater.RestrictedDeploymentEnabled() {
		return updater.isAuthorized(
			signingAccounts,
			blueprints.ContractDeploymentAuthorizedAddressesPath)
	}
	return true
}

func (updater *ContractUpdaterImpl) isAuthorizedForRemoval(
	signingAccounts []flow.Address,
) bool {
	if updater.RestrictedRemovalEnabled() {
		return updater.isAuthorized(
			signingAccounts,
			blueprints.ContractRemovalAuthorizedAddressesPath)
	}
	return true
}

func (updater *ContractUpdaterImpl) isAuthorized(
	signingAccounts []flow.Address,
	path cadence.Path,
) bool {
	accts := updater.GetAuthorizedAccounts(path)
	for _, authorized := range accts {
		if slices.Contains(signingAccounts, authorized) {
			// a single authorized singer is enough
			return true
		}
	}
	return false
}
