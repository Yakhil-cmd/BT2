### Title
Cairo0 `replace_class` Syscall Bypasses Cairo1-Only Guard via Deprecated Hint Processor — (`crates/blockifier/src/execution/deprecated_syscalls/hint_processor.rs`)

---

### Summary

The deprecated syscall hint processor's `replace_class` implementation omits the `is_cairo1` check that is enforced in `SyscallHandlerBase::replace_class`. A Cairo0 contract can call `replace_class` with a Cairo0 target class hash, successfully mutating its own class hash in state to a Cairo0 class — violating the protocol invariant that `replace_class` only accepts Cairo1 class hashes.

---

### Finding Description

There are two independent `replace_class` implementations in the blockifier:

**1. `SyscallHandlerBase::replace_class` (Cairo1 path)** — enforces the invariant:

```rust
pub fn replace_class(&mut self, class_hash: ClassHash) -> SyscallResult<()> {
    let compiled_class = self.state.get_compiled_class(class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(SyscallExecutionError::ForbiddenClassReplacement { class_hash });
    }
    self.state.set_class_hash_at(self.call.storage_address, class_hash)?;
    Ok(())
}
``` [1](#0-0) 

**2. `DeprecatedSyscallHintProcessor::replace_class` (Cairo0 path)** — missing the `is_cairo1` check entirely:

```rust
fn replace_class(
    request: ReplaceClassRequest,
    _vm: &mut VirtualMachine,
    syscall_handler: &mut Self,
) -> DeprecatedSyscallResult<ReplaceClassResponse> {
    // Ensure the class is declared (by reading it).
    syscall_handler.state.get_compiled_class(request.class_hash)?;
    syscall_handler
        .state
        .set_class_hash_at(syscall_handler.storage_address, request.class_hash)?;
    Ok(ReplaceClassResponse {})
}
``` [2](#0-1) 

The `ReplaceClass` selector is explicitly supported in the deprecated syscall dispatch table: [3](#0-2) 

There is no higher-level guard that prevents Cairo0 contracts from invoking `replace_class`. The deprecated hint processor is the sole execution path for Cairo0 contracts, and it routes `ReplaceClass` directly to the above implementation.

---

### Impact Explanation

A Cairo0 contract can call `replace_class` with a Cairo0 target class hash. The call succeeds: `get_compiled_class` returns successfully (the class exists), and `set_class_hash_at` writes the Cairo0 class hash to the contract's class hash slot in state. The resulting on-chain state has a contract whose class hash points to a Cairo0 class — a state that the protocol explicitly forbids via the `ForbiddenClassReplacement` guard in the Cairo1 path. This is a **wrong state value from a syscall bug**, which falls squarely in the Critical impact category.

---

### Likelihood Explanation

Any deployed Cairo0 contract can trigger this. No privileged access is required. The attacker only needs to deploy a Cairo0 contract that calls `replace_class` with any declared Cairo0 class hash as the argument. The deprecated syscall path is the standard execution path for all Cairo0 contracts.

---

### Recommendation

Add the `is_cairo1` check to the deprecated hint processor's `replace_class` implementation, mirroring the guard in `SyscallHandlerBase::replace_class`:

```rust
fn replace_class(
    request: ReplaceClassRequest,
    _vm: &mut VirtualMachine,
    syscall_handler: &mut Self,
) -> DeprecatedSyscallResult<ReplaceClassResponse> {
    let compiled_class = syscall_handler.state.get_compiled_class(request.class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(/* ForbiddenClassReplacement equivalent */);
    }
    syscall_handler
        .state
        .set_class_hash_at(syscall_handler.storage_address, request.class_hash)?;
    Ok(ReplaceClassResponse {})
}
```

The `is_cairo1` function is already imported in `syscall_base.rs` from `crate::transaction::account_transaction::is_cairo1`. [4](#0-3) 

---

### Proof of Concept

1. Declare two Cairo0 classes: `class_A` (the contract to deploy) and `class_B` (the replacement target).
2. Deploy a contract at address `addr` using `class_A`.
3. From `class_A`'s execution, invoke the `replace_class` deprecated syscall with `class_hash = class_B`.
4. The deprecated hint processor calls `get_compiled_class(class_B)` — succeeds because `class_B` is declared.
5. The deprecated hint processor calls `set_class_hash_at(addr, class_B)` — succeeds with no Cairo1 check.
6. Assert `state.get_class_hash_at(addr) == class_B` — a Cairo0 class hash, which should have been rejected with `ForbiddenClassReplacement`.

The call returns `Ok(ReplaceClassResponse {})` instead of the expected error, and the state is durably corrupted with a Cairo0 class hash written via `replace_class`.
