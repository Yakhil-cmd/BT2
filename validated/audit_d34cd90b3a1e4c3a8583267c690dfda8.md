### Title
Missing Access Control on `update_rewards` Allows Any Caller to Steal Yield from Other Stakers - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in the `Staking` contract is documented in the spec as callable by "Only starkware sequencer," but the implementation enforces no such restriction. Any non-zero address can call it. Because the function uses a global `last_reward_block` guard (one call per block), an attacker who calls it for their own staker in every block can monopolize block reward distribution, starving other stakers of their yield.

### Finding Description

`update_rewards` is the consensus-era (V3) reward distribution entry point. The spec explicitly states:

> **access control**: Only starkware sequencer.

The implementation, however, only calls `general_prerequisites()`: [1](#0-0) 

`general_prerequisites()` only asserts the contract is not paused and the caller is non-zero: [2](#0-1) 

There is no check that the caller is the designated sequencer address. The spec's access control requirement is entirely absent from the code.

The function then writes the current block number to the global `last_reward_block`: [3](#0-2) 

This single global guard means only **one** `update_rewards` call succeeds per block across the entire contract. Whoever calls it first in a block "consumes" that block's reward slot for the staker they name.

The spec confirms this one-per-block, one-staker-per-call design: [4](#0-3) 

### Impact Explanation

In the intended design, the sequencer rotates `update_rewards` calls across all active stakers, so each staker receives block rewards proportional to their stake over time. With no access control:

1. An attacker calls `update_rewards(attacker_staker, false)` in every block.
2. Each call succeeds (new block number > `last_reward_block`) and distributes that block's STRK (and BTC) rewards to the attacker's staker.
3. The sequencer's subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.
4. All other stakers receive zero rewards for those blocks.

The attacker's staker accumulates rewards for every block; honest stakers accumulate rewards for no blocks. The "missed" rewards are not returned to a pool — they are simply never distributed to the stakers who earned them, constituting **permanent freezing of unclaimed yield** for all non-attacker stakers.

If the attacker's staker holds a minority share of total stake, the attacker also receives a disproportionately large fraction of total distributed rewards relative to what the sequencer would have allocated to them (since the sequencer would have rotated to other stakers in those blocks).

**Allowed impact matched**: High — Permanent freezing of unclaimed yield / Theft of unclaimed yield.

### Likelihood Explanation

- `update_rewards` is a public, permissionless entry point (beyond non-zero caller).
- The only cost to the attacker is one transaction per block.
- On Starknet, the sequencer controls transaction ordering within a block, which limits the attacker's ability to front-run the sequencer's own call. However, the sequencer may not call `update_rewards` in every block (e.g., during low-activity periods or if the sequencer's automation lags), leaving windows for exploitation.
- The missing access control is a straightforward code-level omission with no workaround for honest stakers.

### Recommendation

Add an explicit sequencer-only access control check at the top of `update_rewards`, analogous to how `update_rewards_from_attestation_contract` restricts callers to the attestation contract: [5](#0-4) 

Store the authorized sequencer address in contract storage and assert `get_caller_address() == sequencer_address` inside `update_rewards` before any other logic.

### Proof of Concept

1. Deploy the staking system with staker A (attacker, 10% of total stake) and staker B (honest, 90% of total stake). Start consensus rewards.
2. In every new block, attacker calls `staking.update_rewards(staker_A_address, false)` before the sequencer.
3. Each call succeeds: staker A's `unclaimed_rewards_own` increases by 10% of block rewards; pool rewards for staker A's pool are transferred.
4. The sequencer's call for staker B in the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. After N blocks: staker A has accumulated N × (10% of block rewards); staker B has accumulated 0. Staker B's yield is permanently frozen. [6](#0-5) [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
