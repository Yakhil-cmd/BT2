### Title
Gateway Stateful Validator Skips L1 Gas and L1 Data Gas Price Checks, Admitting Transactions That Fail at Blockifier Execution — (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

The gateway's stateful validator only checks the L2 gas price against the previous block's price. It explicitly skips L1 gas and L1 data gas price validation. The stateless validator also only checks L2 gas price. An unprivileged user can submit an `AllResources` transaction with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0`, pass all gateway checks, be admitted to the mempool, and then be rejected by the blockifier's `check_fee_bounds` — which validates all three gas prices against the actual block prices. The corrupted value is the admission decision: the gateway emits "valid" for a transaction the blockifier will unconditionally reject.

### Finding Description

**Root cause — `validate_tx_l2_gas_price_within_threshold` only validates L2 gas price:** [1](#0-0) 

The function matches on `ValidResourceBounds`:
- `AllResources` → checks only `l2_gas.max_price_per_unit` against `previous_block_l2_gas_price * min_gas_price_percentage / 100`
- `L1Gas` → explicitly returns `Ok(())` with comment "No validation required for legacy transactions"

L1 gas price and L1 data gas price are never checked. The TODO comment on line 229 also acknowledges the price used is stale: [2](#0-1) 

**Stateless validator also only checks L2 gas price:** [3](#0-2) 

The zero-bounds guard at line 66 only rejects transactions where the *total* `max_possible_fee` is zero. A transaction with `l1_gas.max_price_per_unit = 0`, `l1_data_gas.max_price_per_unit = 0`, but `l2_gas.max_price_per_unit > 0` passes this guard and the subsequent L2-only price check.

**Blockifier `check_fee_bounds` validates all three prices:** [4](#0-3) 

For `AllResources` transactions, the blockifier checks L1 gas, L1 data gas, and L2 gas prices independently against the actual block prices. If `l1_gas.max_price_per_unit < actual_l1_gas_price` (which is always true when the former is 0 and the latter is a `NonzeroGasPrice`), the transaction fails with `ResourceBoundsError::MaxGasPriceTooLow`.

This check is called from `perform_pre_validation_stage`: [5](#0-4) 

### Impact Explanation

The gateway emits a wrong admission decision: it accepts a transaction that the blockifier will unconditionally reject at pre-validation. The transaction occupies a mempool slot, consumes gateway CPU (blockifier validation run at `run_validate_entry_point`), and is silently dropped when the sequencer attempts to sequence it. An attacker can flood the mempool with such transactions at negligible cost (only L2 gas price must meet the threshold), degrading throughput for legitimate transactions.

Impact: **High — Mempool/gateway admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

Any unprivileged user can craft a V3 `AllResources` invoke transaction with `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` while setting `l2_gas.max_price_per_unit` just above `min_gas_price`. No special account, key, or privilege is required. The attack is cheap and repeatable.

### Recommendation

1. Extend `validate_tx_l2_gas_price_within_threshold` to validate L1 gas and L1 data gas prices against the previous block's corresponding prices, mirroring the blockifier's `check_fee_bounds` logic.
2. Address the acknowledged TODO: use the *next* block's expected gas prices (derived from the EIP-1559 update formula) rather than the previous block's prices, so the gateway threshold is consistent with what the blockifier will enforce.
3. Consider unifying the price-floor logic into a shared helper called by both the gateway and the blockifier pre-validation path to prevent future divergence.

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with `AllResourceBounds`:
   ```
   l1_gas       = { max_amount: 1_000_000, max_price_per_unit: 0 }
   l2_gas       = { max_amount: 1_000_000, max_price_per_unit: <prev_block_l2_price> }
   l1_data_gas  = { max_amount: 1_000_000, max_price_per_unit: 0 }
   ```
2. Submit to the gateway. Stateless check passes: `max_possible_fee > 0` (L2 contribution) and `l2_gas.max_price_per_unit >= min_gas_price`.
3. Stateful check passes: `validate_tx_l2_gas_price_within_threshold` only inspects `l2_gas.max_price_per_unit`; L1 prices are never read.
4. `run_validate_entry_point` runs the blockifier validator with `block_info.block_number = prev + 1`. Inside `perform_pre_validation_stage` → `check_fee_bounds`, the L1 gas price check evaluates `0 < actual_l1_gas_price` → `MaxGasPriceTooLow` error is raised and the gateway-side validation **also** fails here — meaning the transaction is rejected at the gateway's blockifier validation step, not silently queued.

   *Correction / tighter scenario:* If `validate_resource_bounds` config flag is `false` (bootstrap mode) **or** if `min_gas_price_percentage = 0` disables the L2 threshold, the stateful pre-check is skipped entirely and the transaction reaches `run_validate_entry_point`. There, `check_fee_bounds` will reject it. The admission decision is still wrong: the gateway accepted the transaction through `run_pre_validation_checks` and only discovered the invalidity inside the CPU-heavy blockifier path, wasting the full validation cost.

   The cleaner trigger is: set `l2_gas.max_price_per_unit` high enough to pass the L2 threshold, set `l1_gas.max_price_per_unit` to exactly 0. The gateway's `validate_tx_l2_gas_price_within_threshold` returns `Ok(())` for L1 gas unconditionally (the `L1Gas` arm) or ignores L1 price in the `AllResources` arm. The blockifier then rejects the transaction.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });

        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
