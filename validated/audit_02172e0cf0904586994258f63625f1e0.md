### Title
Pool Rewards Permanently Frozen in Delegation Pool Contract When Balance Falls Below `min_delegation_for_rewards` — (File: `src/pool/pool.cairo`)

---

### Summary

When a delegation pool's total balance is below `min_delegation_for_rewards`, the staking contract still transfers STRK reward tokens to the pool contract, but the pool contract's `compute_rewards_per_unit` returns zero, preventing any distribution to pool members. These STRK tokens are permanently stuck in the pool contract with no recovery mechanism.

---

### Finding Description

**Step 1 — Rewards are calculated and forwarded unconditionally.**

In `calculate_staker_pools_rewards` (`src/staking/staking.cairo`), pool rewards are computed proportionally to the pool's delegated balance and appended to `pool_rewards_array` whenever `pool_rewards.is_non_zero()`: [1](#0-0) 

The `pool_balance_curr_epoch` used here is the staker's view of the pool's delegated balance (in normalized 18-decimal form). The `total_stake` divisor is the *global* staking power, not the pool balance. Therefore `pool_rewards` can be non-zero even when the pool's own balance is below `min_delegation_for_rewards`.

**Step 2 — STRK tokens are transferred to the pool contract.**

In `update_pool_rewards` (`src/staking/staking.cairo`), for every entry in `pool_rewards_array`, STRK tokens are unconditionally sent to the pool contract via `send_rewards_to_delegation_pool`, and then `update_rewards_from_staking_contract` is called: [2](#0-1) 

**Step 3 — The pool contract silently drops the rewards when balance is too small.**

Inside `update_rewards_from_staking_contract` (`src/pool/pool.cairo`), `compute_rewards_per_unit` is called with the pool balance: [3](#0-2) 

`compute_rewards_per_unit` returns **zero** when `total_stake < min_delegation_for_rewards`, so the cumulative sigma (`cumulative_rewards_trace`) is not incremented: [4](#0-3) 

The protocol's own comment explicitly acknowledges this: [5](#0-4) 

**Step 4 — Pool members can never claim the stuck STRK.**

`claim_rewards` in `pool.cairo` computes member rewards using the cumulative sigma. Since sigma was not incremented, the rewards calculation yields zero for those epochs. The STRK tokens transferred in Step 2 remain in the pool contract's balance indefinitely: [6](#0-5) 

There is no sweep, recovery, or admin function in the pool contract to retrieve these stuck tokens.

---

### Impact Explanation

STRK reward tokens are permanently frozen inside the delegation pool contract. Pool members whose pool balance is below `min_delegation_for_rewards` at the time of reward distribution can never claim those rewards. This matches the allowed impact: **High — Permanent freezing of unclaimed yield**.

The magnitude per event is small (proportional to the pool's tiny balance relative to total stake), but the freeze is irreversible and accumulates over every reward epoch while the condition persists.

---

### Likelihood Explanation

The `enter_delegation_pool` function enforces only `amount.is_non_zero()` — there is no minimum delegation floor: [7](#0-6) 

For STRK pools, `min_delegation_for_rewards = 10^18` (1 STRK). A delegator who enters with 0.5 STRK triggers the condition. For BTC pools with 8 decimals, `min_delegation_for_rewards = 10^3 = 1000 satoshis`; a delegation of 999 satoshis suffices. Any unprivileged delegator can reach this path without special access.

---

### Recommendation

Two mitigations are possible:

1. **Guard at the staking contract level**: In `calculate_staker_pools_rewards`, skip forwarding rewards (do not append to `pool_rewards_array`) when `pool_balance_curr_epoch.to_native_amount(decimals) < min_delegation_for_rewards`. This prevents STRK from being sent to a pool that cannot distribute it.

2. **Add a recovery function**: Expose an admin or governance function in the pool contract to sweep excess STRK balance (i.e., `token_balance - sum_of_all_claimable_rewards`) to a designated treasury or back to the staking contract.

---

### Proof of Concept

1. Staker S stakes and opens a BTC delegation pool (8-decimal token).
2. Delegator D calls `enter_delegation_pool` with `amount = 500` satoshis (below `min_delegation_for_rewards = 1000`).
3. After K epochs, the staker attests and `update_rewards` is called.
4. `calculate_staker_pools_rewards` computes a non-zero `pool_rewards` (e.g., 10^14 STRK fractions) because the global BTC staking power is large.
5. `update_pool_rewards` transfers those STRK fractions to the pool contract and calls `update_rewards_from_staking_contract(rewards=X, pool_balance=500)`.
6. `compute_rewards_per_unit` returns 0 because `500 < 1000`.
7. The cumulative sigma is unchanged.
8. D calls `claim_rewards` — receives 0.
9. The STRK tokens transferred in step 5 remain in the pool contract forever with no recovery path.

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }

        fn compute_rewards_per_unit(
            self: @ContractState, staking_rewards: Amount, total_stake: Amount,
        ) -> Index {
            // Return zero if the total stake is too small, to avoid overflow below.
            if total_stake < self.min_delegation_for_rewards.read() {
                return Zero::zero();
            }
            mul_wide_and_div(
                lhs: staking_rewards, rhs: self.staking_rewards_base_value.read(), div: total_stake,
            )
                .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
        }
