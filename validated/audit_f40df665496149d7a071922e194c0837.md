### Title
`TurnBridge` Missing Permission Check and No-Op Implementation Permanently Disables Bridge Emergency Circuit Breaker - (File: x/cronos/keeper/msg_server.go)

### Summary
`MsgTurnBridge` is the designated emergency circuit breaker for the Gravity bridge. Its handler `TurnBridge` in `msg_server.go` performs no permission check and executes no state change — it unconditionally returns `nil, nil`. Any unprivileged sender can submit the message without error, and no authorized party can ever actually disable the bridge, because the implementation is a stub.

### Finding Description
The `x/cronos` module defines a permission system with two guarded operations: `CanChangeTokenMapping` and `CanTurnBridge`. [1](#0-0) 

`UpdateTokenMapping` correctly enforces its guard:

```go
if !k.HasPermission(ctx, msg.GetSigners(), CanChangeTokenMapping) {
    return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
}
``` [2](#0-1) 

`TurnBridge`, the analogous handler for the second permission, does neither a permission check nor any state mutation:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    return nil, nil
}
``` [3](#0-2) 

The proto definition confirms the message carries a `sender` and an `enable` boolean that are never consumed: [4](#0-3) 

This is the direct Cronos analog of the external report: a function that should enforce a permission flag before acting on it does not check the flag at all, and additionally passes the wrong effective value (always a no-op instead of the requested `enable` state).

### Impact Explanation
Two distinct impacts compound:

1. **Auth bypass (High):** Any unprivileged address can submit `MsgTurnBridge` and receive a success response. The `CanTurnBridge` permission — the only mechanism by which the admin delegates bridge-control authority — is completely bypassed. This matches: *"High: Bypass of Cronos admin, governance authority, permission … bridge … authorization checks."*

2. **Permanent bridge-control failure (High):** Because the handler is a no-op, no party — including the `CronosAdmin` and any address granted `CanTurnBridge` — can ever disable the Gravity bridge. The emergency circuit breaker documented in ADR-009 is permanently inoperative. [5](#0-4)  This matches: *"High: Permanent or long-lived inability for honest users or validators to process valid … bridge/conversion flows … under normal network assumptions."*

### Likelihood Explanation
The entry path is a standard Cosmos SDK message (`MsgTurnBridge`) reachable by any funded account with no special prerequisites. The missing guard and missing implementation are both present in the production handler today.

### Recommendation
Mirror the pattern used by `UpdateTokenMapping`:

```go
func (k msgServer) TurnBridge(goCtx context.Context, msg *types.MsgTurnBridge) (*types.MsgTurnBridgeResponse, error) {
    ctx := sdk.UnwrapSDKContext(goCtx)
    if !k.HasPermission(ctx, msg.GetSigners(), CanTurnBridge) {
        return nil, errors.Wrap(sdkerrors.ErrUnauthorized, "msg sender is not authorized")
    }
    // set bridge enabled/disabled state using msg.Enable
    // ...
    return &types.MsgTurnBridgeResponse{}, nil
}
```

Additionally, add a negative unit test that asserts an unprivileged sender receives `ErrUnauthorized`, analogous to the existing `TestHasPermissions` suite. [6](#0-5) 

### Proof of Concept
1. Fund any address `Eve` with enough gas.
2. Broadcast `MsgTurnBridge{sender: Eve, enable: false}`.
3. Observe the transaction succeeds (code 0) despite `Eve` holding zero permissions.
4. Query bridge state — it is unchanged, confirming the circuit breaker is permanently inoperative.
5. Repeat as the legitimate `CronosAdmin`; the bridge state remains unchanged, confirming no party can disable it.

func (suite *KeeperTestSuite) TestHasPermissions() {
	suite.SetupTest()
	keeper := suite.app.CronosKeeper

	// generate test address
	priv, err := ethsecp256k1.GenerateKey()
	suite.Require().NoError(err)
	address := common.BytesToAddress(priv.PubKey().Address().Bytes())
	cosmosAddress := []sdk.AccAddress{sdk.AccAddress(address.Bytes())}

	suite.Require().Equal(true, keeper.HasPermission(suite.ctx, cosmosAddress, 0))
	suite.Require().Equal(true, keeper.HasPermission(suite.ctx, cosmosAddress, 0))

	suite.Require().Equal(false, keeper.HasPermission(suite.ctx, cosmosAddress, CanChangeTokenMapping))
	suite.Require().Equal(false, keeper.HasPermission(suite.ctx, cosmosAddress, CanTurnBridge))

	keeper.SetPermissions(suite.ctx, cosmosAddress[0], CanChangeTokenMapping)
	suite.Require().Equal(true, keeper.HasPermission(suite.ctx, cosmosAddress, CanChangeTokenMapping))
	suite.Require().Equal(false, keeper.HasPermission(suite.ctx, cosmosAddress, CanTurnBridge))

	keeper.SetPermissions(suite.ctx, cosmosAddress[0], CanTurnBridge)
	suite.Require().Equal(false, keeper.HasPermission(suite.ctx, cosmosAddress, CanChangeTokenMapping))
	suite.Require().Equal(true, keeper.HasPermission(suite.ctx, cosmosAddress, CanTurnBridge))

	keeper.SetPermissions(suite.ctx, cosmosAddress[0], All)
	suite.Require().Equal(true, keeper.HasPermission(suite.ctx, cosmosAddress, CanChangeTokenMapping))
	suite.Require().Equal(true, keeper.HasPermission(suite.ctx, cosmosAddress, CanTurnBridge))
}
