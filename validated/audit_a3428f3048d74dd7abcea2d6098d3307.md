### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` - (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`setPoolAdminFees` enforces `maxAdminSpreadFeeE6` and `maxAdminNotionalFeeE8` caps on the pool admin's base spread fee. However, `setPoolBinAdditionalFees` passes `addFeeBuyE6` / `addFeeSellE6` directly to the pool with no cap check. Because per-bin additional fees are added on top of the base spread fee in every swap, a pool admin can exceed the factory-owner-intended fee ceiling by calling `setPoolBinAdditionalFees` instead of `setPoolAdminFees`.

### Finding Description

`setPoolAdminFees` explicitly enforces the factory-owner-set caps:

```solidity
// MetricOmmPoolFactory.sol L414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees`, by contrast, forwards the caller-supplied values directly with no cap check:

```solidity
// MetricOmmPoolFactory.sol L450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's swap math adds the per-bin additional fee on top of the oracle-derived base spread fee for every swap through that bin:

```solidity
// MetricOmmPool.sol L540-541
uint256 buyFeeX64  = baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6,  ONE_X64, 1e6);
uint256 sellFeeX64 = baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6);
``` [3](#0-2) 

The same uncapped additional fee is applied in all four swap variants (`buyToken0InBinSpecifiedIn/Out`, `buyToken1InBinSpecifiedIn/Out`): [4](#0-3) [5](#0-4) 

`addFeeBuyE6` and `addFeeSellE6` are `uint16`, so the maximum value a pool admin can set is `65535`, which equals **6.5535 %** in E6 units. The factory hard cap `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20%) applies only to the base spread fee path; there is no corresponding hard cap for bin additional fees. [6](#0-5) 

### Impact Explanation

Traders swapping through a bin whose `addFeeBuyE6` / `addFeeSellE6` has been set to `type(uint16).max` pay up to 6.5535 % extra spread on top of whatever the base spread fee already is. If the factory owner has set `maxAdminSpreadFeeE6 = 0` (zero admin spread) to protect users, the pool admin can still silently impose a 6.5535 % per-bin surcharge via `setPoolBinAdditionalFees`. This is a direct loss of user principal on every swap through the affected bin: the pool receives more input than the oracle-anchored price permits, violating the swap-conservation invariant from the user's perspective.

### Likelihood Explanation

The pool admin is a semi-trusted role that is explicitly bounded by factory-owner caps. The bypass requires only a single call to `setPoolBinAdditionalFees` with `addFeeBuyE6 = addFeeSellE6 = 65535`. No special conditions, timelocks, or external state are required. Any pool admin can execute this at any time on any bin within the pool's range.

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` analogous to the one in `setPoolAdminFees`. Either:

1. Introduce a `maxAdminBinAdditionalFeeE6` cap (settable by the factory owner via `setFeeCaps`) and enforce it in `setPoolBinAdditionalFees`, or
2. Reuse `maxAdminSpreadFeeE6` as the ceiling for per-bin additional fees, since they are additive to the base spread and represent the same economic lever.

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

### Proof of Concept

1. Factory owner sets `maxAdminSpreadFeeE6 = 0` to enforce zero admin spread for a pool.
2. Pool admin calls `setPoolAdminFees(pool, 1, 0)` → reverts with `AdminFeeTooHigh`.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` → succeeds, writing `addFeeBuyE6 = addFeeSellE6 = 65535` to bin 0.
4. Next swap through bin 0: `buyFeeX64 = baseFeeX64 + mulDiv(65535, ONE_X64, 1e6)` — the trader pays an extra 6.5535 % spread that the factory owner explicitly prohibited via the cap mechanism. [7](#0-6) [2](#0-1) [8](#0-7) [3](#0-2)
