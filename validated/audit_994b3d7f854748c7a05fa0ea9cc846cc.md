### Title
Unprivileged Caller Can Permanently Suppress Per-Block Rewards for All Stakers via Global `last_reward_block` Gate — (`src/staking/staking.cairo::update_rewards`)

---

### Summary

`update_rewards` has no caller access control. Any non-zero address can call it. The function writes to a single **global** `last_reward_block` storage slot *before* checking `disable_rewards`, so a hostile first-caller in any block can permanently discard that block's rewards for every staker.

---

### Finding Description

The spec for `update_rewards` states access control is **"Only starkware sequencer"**, but the implementation enforces no such restriction.

`general_prerequisites()` only checks two things:

```cairo
// src/staking/staking.cairo:1794-1797
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

There is no sequencer role check, no allowlist, and no `get_caller_address()` comparison anywhere in `update_rewards`.

Inside the function, the global `last_reward_block` is written **unconditionally** before the `disable_rewards` branch:

```cairo
// src/staking/staking.cairo:1484-1489
// Update last block rewards.
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
``` [2](#0-1) 

`last_reward_block` is a single contract-wide slot (not per-staker). The only guard against double-calling is:

```cairo
// src/staking/staking.cairo:1454-1458
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [3](#0-2) 

**Attack sequence per block:**

1. Attacker (any EOA) calls `update_rewards(any_active_staker, disable_rewards: true)` as the first transaction in a block.
2. `general_prerequisites()` passes (non-zero caller, contract not paused).
3. `current_block_number > last_reward_block` passes (first call this block).
4. Staker existence/active checks pass (attacker picks any valid staker address).
5. `last_reward_block` is written to `current_block_number`.
6. Function returns early — **zero rewards distributed**.
7. Every subsequent call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.

That block's rewards are **permanently lost** — there is no recovery path, no retry window, and no way to re-open a block number that has already been recorded in `last_reward_block`.

---

### Impact Explanation

**High — Permanent freezing of unclaimed yield.**

An attacker can suppress rewards for every staker in every block they choose to target, at negligible cost (only gas). Sustained over many blocks, this drains all future yield from the protocol. The lost rewards are not deferred; they are simply never minted/attributed.

---

### Likelihood Explanation

High. The call is permissionless, requires no stake, no special role, and no capital. The only cost is gas per block. A griefing attacker or a competing validator has a direct economic incentive to suppress rivals' rewards.

---

### Recommendation

Add a sequencer/operator role check at the top of `update_rewards`, consistent with the spec:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert_only_sequencer(@self); // enforce "Only starkware sequencer"
    ...
}
```

Alternatively, move the `last_reward_block.write(...)` to **after** the `disable_rewards` branch so that a disabled call does not consume the block slot — but this alone does not fix the missing access control.

---

### Proof of Concept

```
Block N arrives.

Attacker tx (first in block):
  call staking.update_rewards(staker=Alice, disable_rewards=true)
  → general_prerequisites() passes
  → block_number (N) > last_reward_block (N-1) → passes
  → staker Alice is active → passes
  → last_reward_block := N          ← slot consumed
  → disable_rewards == true → return (no rewards)

Sequencer tx (second in block):
  call staking.update_rewards(staker=Alice, disable_rewards=false)
  → block_number (N) > last_reward_block (N) → FALSE
  → REWARDS_ALREADY_UPDATED revert

Alice's rewards for block N: 0 (permanently lost).
``` [4](#0-3) [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;

            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
