### Title
Missing Zero Address Check for `reward_address` Causes Permanent Loss of Unclaimed Yield — (File: `src/pool/pool.cairo`, `src/staking/staking.cairo`)

---

### Summary

The `enter_delegation_pool()` and `change_reward_address()` functions in `Pool`, and the `stake()` and `change_reward_address()` functions in `Staking`, accept a `reward_address` parameter with no zero-address validation. If a staker or delegator supplies the zero address, all future reward claims silently transfer tokens to address `0x0`, permanently destroying the unclaimed yield with no recovery path.

---

### Finding Description

**In `src/pool/pool.cairo`:**

`enter_delegation_pool()` accepts `reward_address` and stores it directly: [1](#0-0) 

`change_reward_address()` updates the stored address with only a token-address collision check, no zero check: [2](#0-1) 

`claim_rewards()` then unconditionally transfers STRK to whatever `reward_address` is stored: [3](#0-2) 

**In `src/staking/staking.cairo`:**

`stake()` stores `reward_address` without a zero check: [4](#0-3) 

`change_reward_address()` updates it with only a token-address collision check: [5](#0-4) 

A `grep` across the entire repository for any zero-address assertion on `reward_address` returns no matches, confirming the check is absent everywhere.

---

### Impact Explanation

When `reward_address` is zero, `claim_rewards()` in the Pool contract executes:

```
reward_token.checked_transfer(recipient: 0x0, amount: rewards)
``` [6](#0-5) 

Tokens transferred to address `0x0` on Starknet are unrecoverable. There is no admin function to redirect a pool member's `reward_address` on their behalf, and the staker/delegator themselves would need to call `change_reward_address` to fix it — but by then any rewards already claimed are gone. The same applies to the staking contract's `claim_rewards` path via `send_rewards_to_staker`.

**Impact: High — Permanent freezing (destruction) of unclaimed yield.**

---

### Likelihood Explanation

The entry path requires only an unprivileged staker or delegator to supply `reward_address = 0` — either by accident (e.g., a front-end bug, a scripting error, or a default-value mistake) or by a griefing actor who controls the account. No privileged role, key leak, or external dependency is needed. The functions are publicly callable by any registered staker or pool member.

---

### Recommendation

Add an explicit non-zero assertion for `reward_address` at every entry point where it is accepted:

```cairo
assert!(reward_address.is_non_zero(), "{}", Error::REWARD_ADDRESS_IS_ZERO);
```

Apply this in:
- `Staking::stake()` — before writing `staker_info`
- `Staking::change_reward_address()` — before updating `staker_info.reward_address`
- `Pool::enter_delegation_pool()` — before writing `pool_member_info`
- `Pool::change_reward_address()` — before updating `pool_member_info.reward_address`

---

### Proof of Concept

**Scenario — delegator permanently loses yield:**

1. Delegator calls `Pool::enter_delegation_pool(reward_address: 0x0, amount: X)`.
   - No zero check; `pool_member_info.reward_address` is stored as `0x0`. [7](#0-6) 

2. Epochs pass; the pool accumulates rewards via `update_rewards_from_staking_contract`. [8](#0-7) 

3. Anyone calls `Pool::claim_rewards(pool_member)`.
   - `reward_address` is read as `0x0`.
   - `checked_transfer(recipient: 0x0, amount: rewards)` executes successfully.
   - Rewards are permanently destroyed. [3](#0-2) 

4. The delegator's principal is still intact (they can exit), but all accrued yield is irrecoverably lost.

        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(
                !self.does_token_exist(token_address: reward_address),
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let staker_address = get_caller_address();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let old_address = staker_info.reward_address;

            // Update reward_address and commit to storage.
            staker_info.reward_address = reward_address;
            self.write_staker_info(:staker_address, :staker_info);

            // Emit event.
            self
                .emit(
                    Events::StakerRewardAddressChanged {
                        staker_address, new_address: reward_address, old_address,
                    },
                );
        }
