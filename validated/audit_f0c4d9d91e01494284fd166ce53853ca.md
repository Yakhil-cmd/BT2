### Title
Unrestricted `update_rewards` with `disable_rewards: true` Allows Any Caller to Permanently Freeze Unclaimed Yield for All Stakers - (File: src/staking/staking.cairo)

### Summary
`IStakingRewardsManager::update_rewards` in `staking.cairo` has no caller restriction and accepts a `disable_rewards: bool` parameter. Any unprivileged address can call it with `disable_rewards: true` every block, consuming the single global `last_reward_block` slot without distributing any rewards, permanently freezing unclaimed yield for all stakers in the consensus rewards phase.

### Finding Description
`update_rewards` is the consensus-phase reward distribution entry point. It enforces a single-call-per-block invariant via a global `last_reward_block` storage variable:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;  // <-- returns without distributing rewards
    }
    ...
``` [1](#0-0) 

There is no `get_caller_address()` check anywhere in this function. The `last_reward_block` is a single global slot (not per-staker), so one call per block with any valid staker address and `disable_rewards: true` consumes the slot for the entire block, preventing any legitimate reward distribution call from succeeding in that block. [2](#0-1) 

By contrast, the analogous pre-consensus reward function `update_rewards_from_attestation_contract` is properly restricted:

```cairo
self.assert_caller_is_attestation_contract();
``` [3](#0-2) 

The spec confirms `update_rewards_from_attestation_contract` is "Only attestation contract," but `update_rewards` carries no equivalent access control in the spec or implementation.

### Impact Explanation
**High: Permanent freezing of unclaimed yield.**

In the consensus rewards phase (`!is_pre_consensus()`), block rewards are distributed once per block via `update_rewards`. An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` every block:

1. Sets `last_reward_block` to the current block number.
2. Returns immediately without distributing any rewards.
3. Any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`.

All stakers' `unclaimed_rewards_own` balances stop accumulating. Since the attack must be repeated each block, it constitutes a permanent freeze as long as the attacker continues (one cheap transaction per block on Starknet). Stakers and delegators can never claim rewards they are owed.

### Likelihood Explanation
**High.** The entry point is fully public — no role, no signature, no stake required. The attacker only needs to submit one transaction per block with a valid staker address (any existing staker). The cost is minimal gas per block. The attack is trivially scriptable and requires no privileged access, leaked keys, or external dependencies.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., a designated operator role, or the sequencer/relayer address), mirroring the restriction applied to `update_rewards_from_attestation_contract`. Alternatively, remove the `disable_rewards` parameter from the public interface and expose it only to a privileged role, so that an unprivileged caller cannot suppress reward distribution.

### Proof of Concept

1. Deploy the system and advance to the consensus rewards epoch (`set_consensus_rewards_first_epoch` + advance K epochs).
2. Stake as a legitimate staker and advance K epochs so the staker has an effective balance.
3. From an attacker address (no special role), call every block:
   ```cairo
   IStakingRewardsManagerDispatcher { contract_address: staking_contract }
       .update_rewards(staker_address: any_valid_staker, disable_rewards: true);
   ```
4. Observe that `last_reward_block` is updated each block but `staker_info.unclaimed_rewards_own` never increases.
5. Any legitimate call to `update_rewards` in the same block reverts with `REWARDS_ALREADY_UPDATED`.
6. After N blocks of this attack, the staker's claimable rewards remain zero despite being entitled to N × block_rewards. [4](#0-3)

            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
