### Title
`tick()` is `payable` but silently discards `msg.value` when no minting is required — (`File: L1/starkware/solidity/stake/RewardSupplier.sol`)

### Summary
`RewardSupplier.tick()` is declared `payable` and expects ETH to cover L1→L2 message fees. However, the entire fee-spending logic is gated inside `if (amountToMint > 0)`. When there are no pending L2→L1 mint requests, the function returns silently without spending or refunding `msg.value`, permanently trapping the caller's ETH in the contract.

### Finding Description
`tick()` is a public, permissionless function callable by any account. It accepts ETH via `payable` to fund two cross-chain messages:

1. `bridge().depositWithMessage{value: msgFee}(...)` — StarkGate bridge fee
2. `messagingContract().sendMessageToL2{value: msgFee}(...)` — L1→L2 messaging fee

Both calls are inside the `if (amountToMint > 0)` branch: [1](#0-0) 

When `requiredMinting()` returns `(0, 0)` — which happens whenever there are no pending L2→L1 mint-request messages — the branch is skipped entirely. Any ETH sent with the call is neither forwarded nor refunded. The contract has no `receive()`, `fallback()`, or admin-withdrawal function, so the ETH is permanently locked. [2](#0-1) 

### Impact Explanation
Any caller who sends ETH to `tick()` when `amountToMint == 0` permanently loses that ETH. The contract provides no recovery path. This is a direct, permanent loss of the caller's funds with no compensating benefit. Impact: **Medium — Griefing with no profit motive but damage to users.**

### Likelihood Explanation
`tick()` is `external` with no access control — any account can call it. The condition `amountToMint == 0` is the normal steady-state between mint cycles (mint requests are batched and infrequent). A caller who monitors the chain, sees a pending request, prepares a transaction with ETH, but whose transaction lands after another caller already consumed the messages will hit this path. Additionally, any caller who simply calls `tick()` with ETH "just in case" will lose funds silently. Likelihood: **Medium.**

### Recommendation
Add a guard at the top of `tick()` to revert if ETH is sent but will not be used, or unconditionally refund any unspent ETH at the end of the function:

```solidity
function tick() external payable {
    (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();
    require(amountToMint > 0 || msg.value == 0, "NO_MINT_REQUIRED");
    // ... rest of logic
}
```

Alternatively, refund leftover ETH after the branch:

```solidity
    } // end if (amountToMint > 0)
    if (address(this).balance > 0) {
        payable(msg.sender).transfer(address(this).balance);
    }
```

### Proof of Concept

1. L2 `RewardSupplier` has no pending mint-request messages (normal steady-state).
2. Attacker (or honest keeper) calls `RewardSupplier.tick{value: 1 ether}()`.
3. `requiredMinting()` returns `(0, 0)` because `messagingContract().l2ToL1Messages(msgHash) == 0`.
4. The `if (amountToMint > 0)` branch is skipped.
5. `tick()` returns. `msg.value` (1 ETH) is now held by the contract with no withdrawal path.
6. The caller's 1 ETH is permanently lost. [3](#0-2)

    function tick() external payable {
        // Check if minting is required, and how much.
        (uint256 amountToMint, uint256 numMsgsToConsume) = requiredMinting();

        if (amountToMint > 0) {
            // Prepare the L2->L1 mintRequest message for consumption.
            uint256[] memory messagePayload = new uint256[](1);
            messagePayload[0] = TOKENS_PER_MINT_REQUEST;

            // Consume the mintRequest messages.
            for (uint256 i = 0; i < numMsgsToConsume; i++) {
                messagingContract().consumeMessageFromL2(mintRequestSource(), messagePayload);
            }

            // Reuest minting of the requested amount from the mint manager.
            mintManager().mintRequest(token(), amountToMint);

            // Deposit the minted amount onto the bridge to the credit of `mintDestination`.
            uint256 msgFee = msg.value / 2;
            bridge().depositWithMessage{value: msgFee}(
                token(),
                amountToMint,
                mintDestination(),
                new uint256[](0)
            );
            emit ConsumedL2MintRequests(numMsgsToConsume, amountToMint);

            // Send a totalSupply update to L2MintCurve.
            msgFee = msg.value - msgFee;
            messagePayload[0] = IERC20(token()).totalSupply();
            messagingContract().sendMessageToL2{value: msgFee}(
                mintingCurve(),
                UPDATE_TOTAL_SUPPLY_SELECTOR,
                messagePayload
            );
        }
    }
