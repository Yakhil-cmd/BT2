### Title
Missing Access Control on `update_rewards` Allows Any Caller to Permanently Freeze Staker Yield - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `StakingRewardsManagerImpl` is specified to be callable **only by the Starkware sequencer**, but the implementation contains **no caller check whatsoever**. Any unprivileged address can call it with `disable_rewards: true`, consuming the per-block reward slot and permanently preventing the sequencer from distributing rewards for that block.

### Finding Description

The protocol specification explicitly states:

> **access control**: Only starkware sequencer.

(`docs/spec.md`, lines 1644–1645)

The implementation, however, performs no such check:

```rust
// src/staking/staking.cairo  ~line 1448
impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
    fn update_rewards(
        ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
    ) {
        self.general_prerequisites();                          // only checks pause
        let current_block_number = starknet::get_block_number();
        assert!(
            current_block_number > self.last_reward_block.read(),
            "{}",
            Error::REWARDS_ALREADY_UPDATED,
        );
        // ← NO assert_caller_is_sequencer() or equivalent
        ...
        self.last_reward_block.write(current_block_number);   // slot consumed
        if disable_rewards || self.is_pre_consensus() {
            return;                                           // rewards skipped
        }
        ...
    }
}
``` [1](#0-0) 

The function writes `current_block_number` to `last_reward_block` unconditionally before the early-return guard. Once that slot is consumed, any subsequent call in the same block reverts with `REWARDS_ALREADY_UPDATED`. [2](#0-1) 

Compare with `update_rewards_from_attestation_contract`, which correctly enforces its caller restriction:

```rust
self.assert_caller_is_attestation_contract();
``` [3](#0-2) 

No analogous guard exists in `update_rewards`.

The spec entry for the function: [4](#0-3) 

### Impact Explanation

An attacker calls `update_rewards(any_active_staker, disable_rewards: true)` in the same block the sequencer would call it. Because `last_reward_block` is written before the early-return, the sequencer's call reverts with `REWARDS_ALREADY_UPDATED`. The consensus-reward tokens that would have been distributed for that block are permanently unclaimable — **permanent freezing of unclaimed yield** for every staker and pool member in that block.

The attacker can repeat this every block, continuously suppressing all consensus reward distribution at negligible cost.

### Likelihood Explanation

- The function is publicly callable with no role or address restriction.
- The attack requires only a single transaction per block, cheaper than the sequencer's own call.
- On Starknet, a determined attacker can reliably front-run the sequencer's `update_rewards` transaction by submitting their own first.
- No special knowledge, funds, or privileged access is required beyond being an active network participant.

### Recommendation

Add a sequencer-only caller check at the top of `update_rewards`, mirroring the pattern already used for the attestation contract:

```rust
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // add this
    ...
}
```

The `assert_caller_is_sequencer` helper should read the stored sequencer address and assert `get_caller_address() == sequencer_address`, consistent with how `assert_caller_is_attestation_contract` is implemented. [5](#0-4) 

### Proof of Concept

1. Consensus rewards are active (`is_pre_consensus()` returns `false`).
2. Attacker observes the mempool and sees the sequencer is about to call `update_rewards(staker_X, false)`.
3. Attacker submits `update_rewards(staker_X, true)` with higher priority.
4. Attacker's transaction executes first: `last_reward_block` is set to the current block, no rewards distributed.
5. Sequencer's transaction reverts: `REWARDS_ALREADY_UPDATED`.
6. All stakers and pool members lose the rewards for that block permanently.
7. Repeat every block to suppress all consensus reward distribution indefinitely. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
