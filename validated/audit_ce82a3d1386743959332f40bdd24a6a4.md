### Title
Malicious Participant Equivocates FROST Presign Commitments, Rendering Presignature Unusable for Signing — (File: `src/frost/mod.rs`)

### Summary
The FROST presign protocol (`do_presign`) distributes nonce commitments via point-to-point `send_many` without any echo-broadcast consistency check. A malicious participant can send a different `SigningCommitments` value to each peer, causing every honest participant to build a distinct `commitments_map`. Because `sign_v2` derives the binding factor, group nonce, and Schnorr challenge directly from each party's locally-stored `commitments_map`, the resulting signature shares are computed over incompatible challenges and cannot be aggregated. The presignature is permanently unusable.

### Finding Description

`do_presign` in `src/frost/mod.rs` is the shared presigning routine for both EdDSA and RedJubjub. Each participant generates a nonce pair, broadcasts its commitment, and collects commitments from all others: [1](#0-0) 

The broadcast is performed with `chan.send_many`, which sends the caller's serialized value to every peer. There is **no echo-broadcast** and **no cross-participant consistency check** on the received commitments. Compare this with the DKG protocol, which explicitly uses `do_broadcast` (an echo-broadcast primitive) for every value that must be globally consistent: [2](#0-1) [3](#0-2) 

A malicious participant `P_m` controlling its own network layer can send commitment `C_m1` to the coordinator and `C_m2 ≠ C_m1` to every other signer. Each honest party inserts the commitment it received into its own `commitments_map`: [4](#0-3) 

The resulting `PresignOutput` stored by each party contains a different `commitments_map`: [5](#0-4) 

During `sign_v2`, every participant—coordinator and non-coordinator alike—derives the `SigningPackage` directly from its own stored `commitments_map` without receiving or verifying a canonical version from any trusted source: [6](#0-5) [7](#0-6) 

Because the FROST binding factor `ρ_j = H1(X, h, commits, j)` is computed over the full `commitments_map`, different maps produce different `ρ_j`, different group nonce `R`, and different Schnorr challenge `c`. Each party's signature share `s_i = a_i + c·λ_i·x_i` is therefore computed over a distinct `c`. The coordinator's `aggregate()` call receives shares that are mutually inconsistent and the aggregated signature fails internal verification, permanently denying signing for that presignature.

The same path exists in the RedJubjub variant: [8](#0-7) 

### Impact Explanation

Every honest participant accepts a `PresignOutput` whose `commitments_map` is silently inconsistent with every other participant's copy. The presignature is cryptographically unusable: no subset of honest parties can produce a valid aggregated signature from it. This matches the allowed High impact: **corruption of presign outputs so honest parties accept unusable cryptographic outputs**, and **permanent denial of signing for honest parties under valid protocol inputs**.

### Likelihood Explanation

Any single malicious participant in a FROST presign session can trigger this. The attacker needs only to send two different `SigningCommitments` serializations to different peers—a capability available to any participant who controls their own network stack. No cryptographic break, no key leakage, and no coordinator compromise is required. The attack is silent: honest parties complete `do_presign` without error and only discover the failure when `aggregate()` rejects the incompatible shares during signing.

### Recommendation

Replace the `send_many` + `recv_from_others` pattern in `do_presign` with the same `do_broadcast` echo-broadcast primitive already used in `do_keyshare`. After the broadcast round, each participant must verify that every peer's commitment was received identically by all parties before storing it in `commitments_map`. This is the exact fix applied to the DKG commitment exchange: [9](#0-8) 

Alternatively, have the coordinator collect and re-broadcast a canonical `commitments_map` during `sign_v2` (as is done in `sign_v1`), and require each participant to verify that the coordinator's map contains their own commitment unchanged before computing their signature share.

### Proof of Concept

1. Run FROST presign with participants `{P_c (coordinator), P_1, P_2}` and a malicious `P_m`.
2. `P_m` sends `C_m` (valid commitment) to `P_c` and `C_m'` (different commitment, same nonce structure but different hiding/binding points) to `P_1` and `P_2`.
3. All parties complete `do_presign` without error. `P_c.commitments_map[P_m] = C_m`; `P_1.commitments_map[P_m] = C_m'`.
4. Call `sign_v2` on all parties with the same message.
5. `P_c` computes `ρ_m = H1(X, h, {…, P_m→C_m, …}, m)` and derives challenge `c_c`.
6. `P_1` computes `ρ_m = H1(X, h, {…, P_m→C_m', …}, m)` and derives challenge `c_1 ≠ c_c`.
7. `P_1`'s share `s_1 = a_1 + c_1·λ_1·x_1` is incompatible with `P_c`'s expected `s_1 = a_1 + c_c·λ_1·x_1`.
8. `aggregate()` fails; no valid signature is produced. The presignature is permanently consumed and unusable. [10](#0-9) [11](#0-10) [12](#0-11)

async fn do_presign<C: Ciphersuite + Send>(
    mut chan: SharedChannel,
    participants: ParticipantList,
    me: Participant,
    signing_share: SigningShare<C>,
    mut rng: impl CryptoRngCore,
) -> Result<PresignOutput<C>, ProtocolError> {
    // Round 1
    let mut commitments_map: BTreeMap<Identifier<C>, SigningCommitments<C>> = BTreeMap::new();

    // Creating two commitments and corresponding nonces
    let (nonces, commitments) = commit(&signing_share, &mut rng);
    commitments_map.insert(me.to_identifier()?, commitments);

    let commit_waitpoint = chan.next_waitpoint();
    // Sending the commitments to all
    chan.send_many(commit_waitpoint, &commitments)?;

    // Collecting the commitments
    for (from, commitment) in recv_from_others(&chan, commit_waitpoint, &participants, me).await? {
        commitments_map.insert(from.to_identifier()?, commitment);
    }

    Ok(PresignOutput {
        nonces,
        commitments_map,
    })
}

async fn do_sign_coordinator_v2(
    mut chan: SharedChannel,
    participants: ParticipantList,
    threshold: ReconstructionLowerBound,
    me: Participant,
    keygen_output: KeygenOutput,
    presignature: PresignOutput,
    message: Vec<u8>,
) -> Result<SignatureOption, ProtocolError> {
    // --- Round 1
    let signing_package =
        frost_ed25519::SigningPackage::new(presignature.commitments_map, message.as_slice());

    let mut signature_shares: BTreeMap<frost_ed25519::Identifier, round2::SignatureShare> =
        BTreeMap::new();

    let vk_package = keygen_output.public_key;

    let key_package =
        construct_key_package(threshold, me, keygen_output.private_share, &vk_package)?;

    let key_package = Zeroizing::new(key_package);
    let signature_share = round2::sign(&signing_package, &presignature.nonces, &key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;
    signature_shares.insert(me.to_identifier()?, signature_share);

    let sign_waitpoint = chan.next_waitpoint();
    for (from, signature_share) in
        recv_from_others(&chan, sign_waitpoint, &participants, me).await?
    {
        signature_shares.insert(from.to_identifier()?, signature_share);
    }

    // --- Signature aggregation.
    // * Converted collected signature shares into the signature.
    // * Signature is verified internally during `aggregate()` call.
    // We supply empty map as `verifying_shares` because we have disabled "cheater-detection" feature flag.
    // Feature "cheater-detection" only points to a malicious participant, if there's such.
    // It doesn't bring any additional guarantees.
    let public_key_package = PublicKeyPackage::new(BTreeMap::new(), vk_package);
    let signature = aggregate(&signing_package, &signature_shares, &public_key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;

    Ok(Some(signature))

fn do_sign_participant_v2(
    mut chan: SharedChannel,
    threshold: ReconstructionLowerBound,
    me: Participant,
    coordinator: Participant,
    keygen_output: &KeygenOutput,
    presignature: PresignOutput,
    message: &[u8],
) -> Result<SignatureOption, ProtocolError> {
    // --- Round 1.
    // * Send our signature share.
    if coordinator == me {
        return Err(ProtocolError::AssertionFailed(
            "the do_sign_participant function cannot be called
            for a coordinator"
                .to_string(),
        ));
    }

    let vk_package = keygen_output.public_key;

    let key_package =
        construct_key_package(threshold, me, keygen_output.private_share, &vk_package)?;
    // Ensures the values are zeroized on drop
    let key_package = Zeroizing::new(key_package);

    let signing_package = SigningPackage::new(presignature.commitments_map, message);
    let signature_share = round2::sign(&signing_package, &presignature.nonces, &key_package)
        .map_err(|e| ProtocolError::AssertionFailed(e.to_string()))?;

    let sign_waitpoint = chan.next_waitpoint();
    chan.send_private(sign_waitpoint, coordinator, &signature_share)?;

    Ok(None)
