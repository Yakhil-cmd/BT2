### Title
Swap Allowlist Bypass via Router Intermediary — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router becomes `msg.sender` to the pool. If the router is allowlisted, any non-allowlisted user can bypass the swap gate entirely.

### Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap` (and subsequently to every registered extension). `SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]` — where `msg.sender` is the pool (the extension's caller) and `sender` is the argument received from the pool. When a swap is routed through `MetricOmmSimpleRouter`, the router calls `pool.swap()` directly, so `sender` arriving at the extension is the router's address, not the original economic actor. If `allowedSwapper[pool][router] == true`, the check passes unconditionally for any caller of the router.

### Impact Explanation
A pool admin deploys a pool with `SwapAllowlistExtension` in the `beforeSwap` order to restrict trading to a curated set of addresses (e.g., KYC'd counterparties). The admin allowlists the canonical `MetricOmmSimpleRouter` as a convenience for allowlisted users. Any non-allowlisted address can then call `router.exactInputSingle()` or `router.exactInput()` and execute swaps against LP balances without restriction. The intended access control is completely defeated, and LP funds are exposed to unauthorized traders.

### Likelihood Explanation
The `MetricOmmSimpleRouter` is the standard public swap entry point. Pool admins who want to allowlist legitimate users will naturally also allowlist the router. The bypass requires no special privileges, no malicious pool setup, and no non-standard token behavior — only a public call to a public function.

### Recommendation
`SwapAllowlistExtension.beforeSwap` should check the original economic actor, not the proxied `sender`. Two options:

1. **Pass the original caller through the router**: `MetricOmmSimpleRouter` should forward `msg.sender` in the `extensionData` payload, and the extension should decode and verify it. This requires a protocol-level convention.
2. **Check `tx.origin` as a fallback** (not recommended for general use, but acceptable in restricted-pool contexts where the admin controls who can call).
3. **Preferred**: The pool's `swap` interface should accept an explicit `swapper` parameter (separate from `recipient`) that the pool passes to extensions, and the router should populate it with `msg.sender` before calling the pool.

### Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, router allowlisted, attacker not allowlisted

// attacker is NOT allowlisted

    priceLimitX64: 0,

// Swap succeeds — allowlist bypassed
// SwapAllowlistExtension checked allowedSwapper[pool][router] == true, not allowedSwapper[pool][attacker]
```
