### Title
Unconditional `last_reward_block` Write Before `disable_rewards` Guard Enables Griefing of Staker Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

In `update_rewards`, the storage write `self.last_reward_block.write(current_block_number)` executes unconditionally before the `disable_rewards` guard. Because `update_rewards` is a fully public, permissionless function, any unprivileged caller can invoke it with `disable_rewards: true` for any staker, consuming the per-block reward slot without distributing rewards. The legitimate call in the same block then fails the guard `current_block_number > self.last_reward_block.read()`, permanently discarding that block's rewards for the victim staker.

---

### Finding Description

`update_rewards` in `IStakingRewardsManager` is callable by anyone for any `staker_address`. Its logic is:

```
fn update_rewards(ref self, staker_address, disable_rewards: bool) {
    ...
    assert!(current_block_number > self.last_reward_block.read(), ...);  // guard
    ...
    self.last_reward_block.write(current_block_number);   // ← UNCONDITIONAL write (line 1485)

    if disable_rewards || self.is_pre_consensus() {       // ← early return AFTER write (line 1487)
        return;
    }
    // actual reward distribution follows
    ...
}
``` [1](#0-0) 

The state mutation (`last_reward_block` update) and the conditional guard (`disable_rewards`) are in separate logical blocks — the mutation always fires, the guard only conditionally prevents reward distribution. This is structurally identical to FTN-3: the check and the action are decoupled, so the action executes even when it should not.

The entry point is fully open: [2](#0-1) 

No role check, no caller restriction. `general_prerequisites()` only asserts the contract is unpaused and the caller is non-zero. [3](#0-2) 

---

### Impact Explanation

An attacker front-runs every legitimate `update_rewards` call for a target staker with `update_rewards(staker_address, disable_rewards: true)`. Each such call:

1. Passes all validity checks (staker exists, is active, has non-zero balance).
2. Writes `last_reward_block = current_block_number` — consuming the slot.
3. Returns early — zero rewards distributed.

The staker's own call in the same block then hits:

```
assert!(current_block_number > self.last_reward_block.read(), Error::REWARDS_ALREADY_UPDATED);
```

and reverts. The block's rewards are permanently lost (consensus rewards are per-block, not accumulated for later). Repeated across every block, this constitutes **permanent freezing of unclaimed yield** — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

- `update_rewards` is a public, permissionless function with no caller restriction.
- The attacker's cost is one cheap transaction per block (no token transfer, no approval).
- The attack is mechanically simple: watch the mempool or simply submit first in every block.
- The victim has no on-chain defense: they cannot prevent a third party from calling `update_rewards` on their behalf with `disable_rewards: true`.

---

### Recommendation

Move `self.last_reward_block.write(current_block_number)` to after the `disable_rewards` guard, so the slot is only consumed when rewards are actually distributed:

```cairo
// After all validity checks...
if disable_rewards || self.is_pre_consensus() {
    return;
}

// Only mark the block as rewarded when rewards are actually distributed.
self.last_reward_block.write(current_block_number);

let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
...
``` [4](#0-3) 

---

### Proof of Concept

1. Staker `S` is active with non-zero balance in consensus-rewards mode.
2. In block `N`, attacker `A` calls `update_rewards(S, disable_rewards: true)`.
   - All asserts pass (staker valid, `N > last_reward_block`).
   - `last_reward_block` is written to `N`.
   - Function returns early — no rewards sent.
3. Staker `S` (or anyone) calls `update_rewards(S, disable_rewards: false)` in block `N`.
   - Assert `N > last_reward_block` → `N > N` → **false → PANIC**.
   - Block `N` rewards are permanently lost.
4. Attacker repeats step 2 for every subsequent block, zeroing all of `S`'s consensus rewards indefinitely. [5](#0-4)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
