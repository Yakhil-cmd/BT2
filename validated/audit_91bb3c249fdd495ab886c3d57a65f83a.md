### Title
Unprivileged Caller Can Permanently Deny Block Rewards for Any Block via `disable_rewards` Parameter - (File: src/staking/staking.cairo)

### Summary

The `update_rewards` function in `src/staking/staking.cairo` accepts a user-controlled `disable_rewards: bool` parameter with no access control. Any unprivileged caller can invoke it with `disable_rewards: true`, which advances the global `last_reward_block` checkpoint without distributing any rewards. Because `last_reward_block` is a single global value, this permanently forfeits all stakers' block rewards for that block at zero token cost to the attacker.

### Finding Description

`update_rewards` is a publicly callable function in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    self.last_reward_block.write(current_block_number);   // ← global checkpoint advanced

    if disable_rewards || self.is_pre_consensus() {
        return;                                           // ← exits before any reward math
    }
    ...
``` [1](#0-0) 

`general_prerequisites()` only asserts the contract is not paused and the caller is non-zero: [2](#0-1) 

The global `last_reward_block` is written unconditionally before the `disable_rewards` branch: [3](#0-2) 

Once `last_reward_block` equals the current block number, the guard at the top of the function causes every subsequent call for that block to revert with `REWARDS_ALREADY_UPDATED`: [4](#0-3) 

The attacker only needs to supply any valid, active staker address with non-zero balance (all public on-chain information) to pass the two staker-validity assertions: [5](#0-4) 

### Impact Explanation

In the consensus-rewards phase, `update_rewards` is the sole mechanism by which per-block STRK and BTC rewards are distributed to stakers and their delegation pools. Block rewards are computed as:

```
total_rewards = yearly_mint × avg_block_duration / (BLOCK_DURATION_SCALE × SECONDS_IN_YEAR)
``` [6](#0-5) 

By calling `update_rewards(valid_staker, disable_rewards: true)` once per block, an attacker permanently forfeits the entire block's reward allocation for every staker in the protocol. The rewards are not deferred — they are simply never credited. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators.

The attack can be sustained indefinitely at gas cost only, with no token expenditure required.

### Likelihood Explanation

- The function is `#[abi(embed_v0)]` — fully public, no role check.
- The only prerequisite is a non-zero caller and a valid staker address, both trivially satisfied.
- The attacker pays only L2 gas per block (~seconds cadence on Starknet).
- No special knowledge, privileged key, or front-running is required.
- The attack is immediately effective from the first block of the consensus-rewards phase.

### Recommendation

Restrict `update_rewards` so that `disable_rewards: true` can only be passed by a trusted role (e.g., `only_app_governor` or `only_security_agent`), or remove the parameter entirely and derive the disable condition from on-chain state. Alternatively, decouple the `last_reward_block` write from the reward-distribution path so that a skipped distribution does not consume the block's reward slot.

### Proof of Concept

1. Consensus rewards are active (`consensus_rewards_first_epoch` has been set and the current epoch has passed it).
2. Attacker reads any active staker address `S` with non-zero STRK balance from on-chain events.
3. At block `N`, attacker calls:
   ```
   staking_contract.update_rewards(staker_address: S, disable_rewards: true)
   ```
4. Inside the call:
   - `last_reward_block` is written to `N` (line 1485).
   - The `if disable_rewards` branch returns immediately (line 1487–1489).
   - No rewards are calculated or distributed.
5. Any legitimate call to `update_rewards` at block `N` (by any party, for any staker) now reverts with `REWARDS_ALREADY_UPDATED`.
6. Block `N`'s reward allocation is permanently lost for all stakers and delegators.
7. Attacker repeats at block `N+1`, `N+2`, … to continuously deny all consensus rewards at gas cost only.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
