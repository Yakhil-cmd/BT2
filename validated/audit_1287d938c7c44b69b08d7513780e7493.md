### Title
Malicious Pool Admin Can Permanently DoS `collectFees` by Setting `adminFeeDestination` to a USDC/USDT-Blacklisted Address, Locking Protocol Fees and Blocking `setPoolProtocolFee` — (File: `metric-core/contracts/MetricOmmPool.sol`, `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`collectFees` in `MetricOmmPool` uses a **push pattern** to transfer accrued fees to `adminFeeDestination_` via `safeTransfer`. A semi-trusted pool admin can call `setPoolAdminFeeDestination` to point that destination at a USDC/USDT-blacklisted address. Every subsequent call to `collectFees` will revert on the blacklisted transfer, permanently blocking `collectPoolFees`, `setPoolProtocolFee` (factory owner), and `setPoolAdminFees` (pool admin) for that pool.

---

### Finding Description

`MetricOmmPool.collectFees` pushes both the admin share and the protocol share of accrued fees in a single atomic transaction:

```solidity
// MetricOmmPool.sol lines 416–427
if (totalFee0ToAdmin > 0) {
    transferToken0(adminFeeDestination_, totalFee0ToAdmin);   // safeTransfer
}
if (totalFee1ToAdmin > 0) {
    transferToken1(adminFeeDestination_, totalFee1ToAdmin);   // safeTransfer
}
if (totalFee0ToProtocol > 0) {
    transferToken0(FACTORY, totalFee0ToProtocol);
}
if (totalFee1ToProtocol > 0) {
    transferToken1(FACTORY, totalFee1ToProtocol);
}
notionalFeeToken0Scaled = 0;
notionalFeeToken1Scaled = 0;
``` [1](#0-0) 

`transferToken0`/`transferToken1` call OpenZeppelin `safeTransfer`, which propagates any revert from the token contract: [2](#0-1) 

The pool admin controls `adminFeeDestination` via `setPoolAdminFeeDestination`. The only guard is a non-zero check:

```solidity
// MetricOmmPoolFactory.sol lines 438–447
function setPoolAdminFeeDestination(address pool, address newAdminFeeDestination)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminFeeDestination == address(0)) revert InvalidAdminFeeDestination();
    poolAdminFeeDestination[pool] = newAdminFeeDestination;
    ...
}
``` [3](#0-2) 

For pools whose token0 or token1 is **USDC or USDT** (explicitly in scope), USDC/USDT's `transfer` reverts when the recipient is on the blacklist. If the pool admin sets `adminFeeDestination` to a blacklisted address, every call to `collectFees` that computes `totalFee0ToAdmin > 0` or `totalFee1ToAdmin > 0` will revert before the protocol share is transferred and before `notionalFeeToken0Scaled`/`notionalFeeToken1Scaled` are cleared.

All three factory entry-points that invoke `collectFees` are therefore permanently bricked for that pool:

1. **`collectPoolFees`** (callable by anyone) — calls `collectFees` with `poolAdminFeeDestination[pool]`: [4](#0-3) 

2. **`setPoolProtocolFee`** (factory owner / protocol) — calls `collectFees` before updating the fee config: [5](#0-4) 

3. **`setPoolAdminFees`** (pool admin) — calls `collectFees` before updating admin fee rates: [6](#0-5) 

There is no escape hatch: the factory always reads `poolAdminFeeDestination[pool]` and passes it verbatim to `collectFees`; the factory owner has no alternative path to collect fees or update protocol fees without going through this function.

---

### Impact Explanation

- **Protocol fee revenue permanently locked**: `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` are never cleared, and the spread surplus never leaves the pool. All accrued protocol fees are stranded in the pool contract with no recovery path.
- **Admin-boundary break**: The factory owner's `setPoolProtocolFee` — a privileged protocol-level operation — is permanently blocked by a semi-trusted pool admin action that is entirely within the pool admin's technical capability. The factory owner cannot enforce fee policy changes on the affected pool.
- **`setPoolAdminFees` also blocked**: The pool admin loses the ability to update their own fee rates, but this is a self-inflicted consequence.

---

### Likelihood Explanation

- Requires a pool whose token0 or token1 is USDC or USDT (common deployment scenario).
- Requires the pool admin to deliberately set `adminFeeDestination` to a USDC/USDT-blacklisted address (e.g., a known OFAC-sanctioned address). The pool admin is semi-trusted and this call is entirely within their on-chain authority.
- Once set, the DoS is permanent unless the pool admin changes the destination — but `setPoolAdminFeeDestination` does **not** call `collectFees` first, so the admin can always rotate to a valid address. However, a griefing admin can repeatedly reset to a blacklisted address whenever the protocol tries to collect.
- Likelihood: **Low-Medium** (deliberate malicious pool admin on a USDC/USDT pool).

---

### Recommendation

Decouple fee collection from fee-rate updates. Specifically:

1. **Do not call `collectFees` inside `setPoolProtocolFee` and `setPoolAdminFees`**. Instead, snapshot the current rates and update them atomically without requiring a successful token transfer.
2. **Separate admin and protocol transfers in `collectFees`**: if the admin transfer fails, continue and still transfer the protocol share (pull pattern or try/catch per recipient).
3. Alternatively, adopt a **pull pattern**: credit `adminFeeDestination` and `FACTORY` in an internal mapping and let each party withdraw independently, so one reverting recipient cannot block the other.

---

### Proof of Concept

```
Setup:
  - Pool token0 = USDC, token1 = WETH
  - Pool admin = Attacker (semi-trusted)
  - adminSpreadFeeE6 > 0 (admin has a non-zero fee share)

Step 1: Attacker calls:
  factory.setPoolAdminFeeDestination(pool, USDC_BLACKLISTED_ADDRESS)
  // Only check: newAdminFeeDestination != address(0) → passes

Step 2: Swaps occur on the pool, accruing spread fees.
  surplus0Scaled > 0 → totalFee0ToAdmin > 0

Step 3: Factory owner calls:
  factory.setPoolProtocolFee(pool, newProtocolSpreadFeeE6, newProtocolNotionalFeeE8)
  // Internally calls pool.collectFees(...)
  // pool.collectFees calls: USDC.transfer(USDC_BLACKLISTED_ADDRESS, totalFee0ToAdmin)
  // USDC reverts: "Blacklistable: account is blacklisted"
  // setPoolProtocolFee reverts entirely

Step 4: Anyone calls:
  factory.collectPoolFees(pool)
  // Same revert path → protocol fees permanently stuck

Result:
  - notionalFeeToken0Scaled never reset to 0
  - Protocol fee revenue stranded in pool
  - Factory owner cannot update protocol fee on this pool
```
