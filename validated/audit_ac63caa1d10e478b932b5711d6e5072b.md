### Title
Unauthorized Transfer of Native `evm/` Tokens via Missing Caller Validation in Bank Precompile `transfer` Method - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The `transfer` method of the `BankContract` precompile accepts an arbitrary `sender` address argument without verifying that it equals the calling contract (`contract.Caller()`). Any EVM contract can invoke `bankPrecompile.transfer(victimAddr, attackerAddr, amount)` to drain `evm/<callerContract>` native Cosmos bank tokens from any holder without their authorization.

### Finding Description
The bank precompile exposes a `transfer(address sender, address recipient, uint256 amount)` method. The denom of the token being transferred is derived from the calling contract's address: `"evm/" + contract.Caller().Hex()`. The method validates `amount > 0` and checks that `recipient` is not a blocked module address, but it performs **no validation that `sender` equals `contract.Caller()`** — i.e., there is no check that the calling contract is authorized to move tokens on behalf of `sender`.

```go
// x/cronos/keeper/precompiles/bank.go
sender := args[0].(common.Address)
recipient := args[1].(common.Address)
amount := args[2].(*big.Int)
if amount.Sign() <= 0 {
    return nil, errors.New("invalid amount")
}
from := sdk.AccAddress(sender.Bytes())
to := sdk.AccAddress(recipient.Bytes())
if err := bc.checkBlockedAddr(to); err != nil {   // only `to` is checked
    return nil, err
}
denom := EVMDenom(contract.Caller())              // denom = "evm/<callerAddr>"
...
if err := bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt)); err != nil {
```

The `from` address is passed directly to `bankKeeper.SendCoins` with no authorization check. Any contract at address `0xMAL` can call `bankPrecompile.transfer(victimAddr, attackerAddr, N)` and move `evm/0xMAL` tokens from `victimAddr` to `attackerAddr`.

Additionally, `sender` is not checked against `address(0)`. If `address(0)` holds a balance of `evm/<callerAddr>` tokens (reachable because `mint(address(0), amount)` is not blocked — `checkBlockedAddr` only rejects Cosmos module accounts, not the zero EVM address), an attacker can drain those tokens via `transfer(address(0), attackerAddr, amount)`.

### Impact Explanation
**Critical** — Unauthorized transfer of precompile-controlled assets. Any deployed EVM contract can steal `evm/<contractAddress>` native bank tokens from any holder of that denomination without the holder's consent. This directly satisfies the Critical impact criterion: *"Unauthorized mint, burn, transfer, bridge, conversion, escrow release, or balance/accounting change for … precompile-controlled assets."*

### Likelihood Explanation
**Medium** — The bank precompile is callable by any EVM contract with no privilege requirement. The attack requires that victims hold `evm/<attackerContract>` tokens, which is a realistic scenario in any DeFi protocol built on top of the bank precompile (e.g., a contract that mints `evm/` tokens to users as part of a liquidity or staking mechanism). The attacker controls the calling contract and can execute the theft at any time.

### Recommendation
Add a guard in the `TransferMethodName` case that asserts `sender == contract.Caller()`:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be the calling contract")
}
```

Additionally, add a zero-address check for both `sender` and `recipient`:

```go
if sender == (common.Address{}) || recipient == (common.Address{}) {
    return nil, errors.New("sender and recipient must not be the zero address")
}
```

### Proof of Concept

1. Attacker deploys `MaliciousContract` at address `0xMAL`.
2. `MaliciousContract` calls `bankPrecompile.mint(victimAddr, 1000)` — minting 1000 `evm/0xMAL` tokens to `victimAddr`. (Victim may have legitimately received these tokens through a DeFi interaction.)
3. Attacker calls `bankPrecompile.transfer(victimAddr, attackerAddr, 1000)` from `MaliciousContract`.
4. The precompile computes `denom = "evm/0xMAL"`, constructs `from = sdk.AccAddress(victimAddr.Bytes())`, and calls `bankKeeper.SendCoins(ctx, from, attackerAddr, coins)`.
5. No authorization check fires. The 1000 `evm/0xMAL` tokens are transferred from `victimAddr` to `attackerAddr` without `victimAddr`'s consent.

**Zero-address variant:**
1. Attacker calls `bankPrecompile.mint(address(0), 1000)` from `MaliciousContract` — `checkBlockedAddr` does not reject `address(0)`.
2. Attacker calls `bankPrecompile.transfer(address(0), attackerAddr, 1000)`.
3. `bankKeeper.SendCoins` drains the zero address's `evm/0xMAL` balance to `attackerAddr`. [1](#0-0) [2](#0-1) [3](#0-2)
