### Title
Reward Address Set to Pool/Staking Contract Causes Permanent Freeze of Unclaimed Yield - (File: src/pool/pool.cairo, src/staking/staking.cairo)

### Summary
Both the `Pool` and `Staking` contracts allow users to set their `reward_address` to the contract's own address (or the staking contract address) without any validation. When rewards are subsequently claimed, STRK tokens are transferred to the contract itself with no recovery mechanism, permanently freezing unclaimed yield.

### Finding Description

**Pool contract — `change_reward_address` and `enter_delegation_pool`:**

`enter_delegation_pool` accepts a `reward_address` parameter and only validates that it is not the token contract address: [1](#0-0) 

`change_reward_address` performs the same single check: [2](#0-1) 

Neither function checks that `reward_address != get_contract_address()` (the pool contract itself) or `reward_address != staking_contract`. When `claim_rewards` is later called, rewards are unconditionally transferred to whatever `reward_address` is stored: [3](#0-2) 

The pool contract has no sweep or recovery function for arbitrary ERC20 tokens sent to it, so any STRK transferred to `get_contract_address()` is permanently frozen.

**Staking contract — `stake` and `change_reward_address`:**

`stake` only checks that `reward_address` is not a registered token: [4](#0-3) 

`change_reward_address` performs the same single check: [5](#0-4) 

Neither validates `reward_address != get_contract_address()`. When `send_rewards_to_staker` executes, it transfers to whatever `reward_address` is stored: [6](#0-5) 

STRK sent to the staking contract address in this way is indistinguishable from staked principal and is unrecoverable.

### Impact Explanation

A pool member or staker who sets `reward_address` to the pool contract address or staking contract address will have all future reward claims permanently transferred to that contract. The pool contract has no recovery path for ERC20 tokens sent to itself; the staking contract similarly has no mechanism to distinguish or recover tokens sent to it as a reward address. This constitutes **permanent freezing of unclaimed yield** (High severity per the allowed impact scope).

### Likelihood Explanation

Any unprivileged pool member can call `enter_delegation_pool` with `reward_address = pool_contract_address`, or call `change_reward_address` at any time with the pool or staking contract address. This requires no special access, no leaked keys, and no external dependency. An accidental mis-paste of the pool address (a common UX error) or a deliberate self-grief is sufficient to trigger the freeze. The entry path is fully reachable by any delegator.

### Recommendation

Add explicit guards in both `enter_delegation_pool` and `change_reward_address` in `pool.cairo`, and in `stake` and `change_reward_address` in `staking.cairo`:

```cairo
assert!(
    reward_address != get_contract_address(),
    "REWARD_ADDRESS_IS_POOL"  // or REWARD_ADDRESS_IS_STAKING_CONTRACT
);
```

For the pool contract, also guard against `reward_address == staking_contract`:

```cairo
assert!(
    reward_address != self.staking_pool_dispatcher.contract_address.read(),
    "REWARD_ADDRESS_IS_STAKING_CONTRACT"
);
```

This mirrors the existing `REWARD_ADDRESS_IS_TOKEN` guard pattern already present in both contracts.

### Proof of Concept

1. Staker deploys a pool via `set_open_for_delegation`.
2. Delegator calls `pool.enter_delegation_pool(reward_address: pool_contract_address, amount: X)`.
   - The only check at line 195 passes because `token_address != pool_contract_address`.
   - `reward_address = pool_contract_address` is stored in `pool_member_info`.
3. Epochs advance; rewards accrue.
4. Anyone calls `pool.claim_rewards(pool_member: delegator_address)`.
   - Line 366 executes: `reward_token.checked_transfer(recipient: pool_contract_address, amount: rewards)`.
   - STRK is transferred to the pool contract itself.
5. No function in `pool.cairo` allows recovery of these tokens. Yield is permanently frozen. [7](#0-6) [8](#0-7) [9](#0-8)

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

        fn change_reward_address(ref self: ContractState, reward_address: ContractAddress) {
            assert!(
                self.token_dispatcher.contract_address.read() != reward_address,
                "{}",
                GenericError::REWARD_ADDRESS_IS_TOKEN,
            );
            let pool_member = get_caller_address();
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let old_address = pool_member_info.reward_address;

            // Update reward_address and commit to storage.
            pool_member_info.reward_address = reward_address;
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardAddressChanged {
                        pool_member, new_address: reward_address, old_address,
                    },
                );
        }
