### Title
Unvalidated `disable_rewards` Parameter in `update_rewards` Allows Any Caller to Freeze All Consensus-Phase Staker Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in `src/staking/staking.cairo` is publicly callable by any non-zero address and accepts an unvalidated `disable_rewards: bool` parameter. When `true`, the function writes the current block number to the global `last_reward_block` storage slot — consuming the per-block reward slot — and then returns immediately without distributing any rewards. Because `last_reward_block` is a global singleton that gates all reward updates, an attacker who calls `update_rewards(any_valid_staker, true)` in every block permanently prevents all stakers from accumulating consensus-phase rewards.

---

### Finding Description

`update_rewards` is exposed via the `IStakingRewardsManager` ABI with no caller restriction beyond `general_prerequisites()`, which only checks the contract is unpaused and the caller is non-zero.

The critical execution path is:

```
update_rewards(staker_address, disable_rewards)
  → assert current_block > last_reward_block          // one slot per block, globally
  → validate staker exists and has non-zero balance
  → last_reward_block.write(current_block_number)     // ← slot consumed HERE
  → if disable_rewards { return; }                    // ← no rewards distributed
  → _update_rewards(...)                              // ← never reached
``` [1](#0-0) 

The `last_reward_block` write at line 1485 occurs **before** the `disable_rewards` guard at line 1487. Once the slot is consumed for block N, the assertion `current_block_number > last_reward_block` at line 1454 will reject every subsequent call in block N. [2](#0-1) 

Because `disable_rewards` is never validated against the caller's identity or role, any attacker can:

1. Identify any valid staker address (readable from `NewStaker` events or `staker_info` storage).
2. Call `update_rewards(valid_staker, disable_rewards: true)` in every block.
3. Each call writes `last_reward_block = current_block`, consuming the slot.
4. `_update_rewards` is never reached; `update_unclaimed_rewards_from_staking_contract` is never called; no staker accumulates rewards. [3](#0-2) 

The `general_prerequisites` function imposes no caller restriction: [4](#0-3) 

---

### Impact Explanation

Every staker in the consensus rewards phase is denied block rewards for every block in which the attacker fires the call. Because the attack is automatable and repeatable at negligible cost, this constitutes **permanent freezing of unclaimed yield** for all protocol participants — a High-severity impact under the allowed scope. The attacker receives no financial gain; the damage is purely to stakers and the protocol.

---

### Likelihood Explanation

- **No privilege required**: any non-zero EOA or contract can call `update_rewards`.
- **Staker address is public**: emitted in `NewStaker` events; any active staker suffices.
- **Automatable**: a simple bot monitoring new blocks and submitting one transaction per block is sufficient.
- **Low cost**: Starknet gas fees are low enough to sustain this indefinitely.
- **No detection barrier**: the call is indistinguishable on-chain from a legitimate "no-attest" block marker.

---

### Recommendation

**Short term**: Add an access-control check to `update_rewards` so that only the designated consensus/attestation contract (or a privileged operator role) may supply `disable_rewards: true`. Unprivileged callers should either be rejected outright or forced to use `disable_rewards: false`.

**Long term**: Separate the "mark block as processed" side-effect (writing `last_reward_block`) from the "distribute rewards" logic, and gate the former behind an explicit role check — consistent with the principle of validating all security-sensitive inputs before they affect critical protocol state.

---

### Proof of Concept

```
Block N:
  attacker calls: update_rewards(valid_staker_address, disable_rewards=true)
    → assert get_block_number() (N) > last_reward_block (N-1)  ✓
    → staker validation passes (staker exists, balance > 0)     ✓
    → last_reward_block.write(N)                                ← slot consumed
    → disable_rewards == true → return                          ← no rewards

  legitimate node calls: update_rewards(any_staker, disable_rewards=false)
    → assert get_block_number() (N) > last_reward_block (N)    ✗ REVERTS

Block N+1: attacker repeats. All stakers earn zero rewards indefinitely.
```

The root cause is the same vulnerability class as the referenced report: a critical input (`disable_rewards`) is consumed by the protocol without validating whether the caller is authorised to supply that value, allowing an unprivileged actor to corrupt the protocol's reward-accounting state.

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
