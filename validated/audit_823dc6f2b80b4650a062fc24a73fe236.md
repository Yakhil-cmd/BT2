### Title
Staker Can Instantly Increase Commission via Commission Commitment to Steal Delegator Yield — (File: `src/staking/staking.cairo`)

---

### Summary

A staker holding an active `CommissionCommitment` can instantly raise their commission rate up to `max_commission` (which may be set to 100 %). Because the commission change takes effect immediately while delegators require K epochs plus the full `exit_wait_window` to exit, delegators have no practical opportunity to leave before their yield is redirected to the staker as commission.

---

### Finding Description

**Root cause — `update_commission` allows increases when a commitment is active**

Without a commitment, `update_commission` enforces a strict decrease:

```cairo
} else {
    assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
}
``` [1](#0-0) 

When a `CommissionCommitment` is active, the guard is relaxed to allow any value up to `max_commission`, including values *above* the current commission:

```cairo
if self.is_commission_commitment_active(:commission_commitment) {
    assert!(
        commission <= commission_commitment.max_commission,
        "{}",
        Error::INVALID_COMMISSION_WITH_COMMITMENT,
    );
    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
}
``` [2](#0-1) 

The developers themselves note this gap:

```cairo
/// **Note**: Current commission increase safeguards still allow for sudden commission
/// changes.
``` [3](#0-2) 

**Setting a commitment with `max_commission = 10000` is always valid** because the only constraint is `current_commission <= max_commission`:

```cairo
let current_commission = staker_pool_info.commission();
assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
``` [4](#0-3) 

**Commission is consumed immediately at reward-calculation time** — there is no epoch delay on the write:

```cairo
staker_pool_info.commission.write(Option::Some(commission));
``` [5](#0-4) 

And it is read directly when splitting pool rewards:

```cairo
let commission = staker_pool_info.commission();
...
let (commission_rewards, pool_rewards) = split_rewards_with_commission(
    rewards_including_commission: pool_rewards_including_commission, :commission,
);
``` [6](#0-5) 

---

### Impact Explanation

All pool rewards for the affected epoch(s) are redirected to the staker as commission. Delegators receive near-zero yield on their delegated stake. This is **theft of unclaimed yield** — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

- The staker is an unprivileged, permissionless role; any staker can execute this.
- The `CommissionCommitmentSet` event is emitted on-chain, but delegators cannot act on it in time: exiting requires `exit_delegation_pool_intent` followed by the full `exit_wait_window` (default one week), while the commitment can be set for a single epoch (`expiration_epoch = current_epoch + 1`).
- The staker can call `set_commission_commitment` and `set_commission` in consecutive transactions (or the same block), giving delegators zero reaction time.
- No special infrastructure (e.g., Flashbots) is needed on Starknet; the two calls can be sequenced atomically.

---

### Recommendation

Apply the same K-epoch delay to commission *increases* that already governs balance changes. Specifically, when `commission > old_commission`, the new rate should only become effective at `current_epoch + K`, giving delegators the same window they have to react to balance-affecting events. Alternatively, prohibit commission increases within a commitment period entirely and restrict `set_commission` to decreases only (the current behaviour outside a commitment).

---

### Proof of Concept

1. Staker stakes with `commission = 200` (2 %) and opens a STRK pool.
2. Delegators enter the pool; their yield accrues each epoch.
3. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
   - Valid because `200 <= 10000` and `expiration_epoch > current_epoch`.
4. In the next transaction, staker calls `set_commission(commission: 9999)` (99.99 %).
   - Valid because `9999 <= 10000` and `9999 != 200`.
5. The attestation contract triggers `update_rewards_from_attestation_contract` (or `update_rewards` in V3). `calculate_staker_pools_rewards` reads `commission = 9999`, so `split_rewards_with_commission` routes 99.99 % of pool rewards to the staker and 0.01 % to the pool.
6. Delegators call `claim_rewards` and receive near-zero yield.
7. After `expiration_epoch`, the commitment expires; the staker must decrease commission — but the yield for the affected epoch has already been stolen.

Delegators cannot exit before step 5 because `exit_delegation_pool_intent` starts the `exit_wait_window` clock (≥ 1 week by default), which far exceeds the single-epoch commitment window. [7](#0-6) [8](#0-7) [9](#0-8)

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );
        }

        fn update_commission(
            ref self: ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<Mutable<InternalStakerPoolInfoV2>>,
            old_commission: Commission,
            commission: Commission,
        ) {
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                if self.is_commission_commitment_active(:commission_commitment) {
                    assert!(
                        commission <= commission_commitment.max_commission,
                        "{}",
                        Error::INVALID_COMMISSION_WITH_COMMITMENT,
                    );
                    assert!(commission != old_commission, "{}", Error::INVALID_SAME_COMMISSION);
                } else {
                    assert!(
                        commission < old_commission, "{}", Error::COMMISSION_COMMITMENT_EXPIRED,
                    );
                }
            } else {
                assert!(commission < old_commission, "{}", Error::INVALID_COMMISSION);
            }

            // Update commission in storage.
            staker_pool_info.commission.write(Option::Some(commission));

            // Emit event.
            self
                .emit(
                    Events::CommissionChanged {
                        staker_address, old_commission, new_commission: commission,
                    },
                );
        }

        fn calculate_staker_pools_rewards(
            self: @ContractState,
            staker_address: ContractAddress,
            staker_pool_info: StoragePath<InternalStakerPoolInfoV2>,
            strk_total_rewards: Amount,
            strk_total_stake: NormalizedAmount,
            btc_total_rewards: Amount,
            btc_total_stake: NormalizedAmount,
            curr_epoch: Epoch,
        ) -> (Amount, Amount, Array<(ContractAddress, ContractAddress, NormalizedAmount, Amount)>) {
            // Array for rewards data needed to update pools.
            // Contains tuples of (pool_contract, token_address, pool_balance, pool_rewards).
            let mut pool_rewards_array = array![];
            let mut total_commission_rewards: Amount = Zero::zero();
            let mut total_pools_rewards: Amount = Zero::zero();
            let commission = staker_pool_info.commission();
            for (pool_contract, token_address) in staker_pool_info.pools {
                if !self.is_active_token(:token_address, epoch_id: curr_epoch) {
                    continue;
                }
                let pool_balance_curr_epoch = self
                    .get_staker_delegated_balance_at_epoch(
                        :staker_address, :pool_contract, epoch_id: curr_epoch,
                    );
                let (total_rewards, total_stake) = if token_address == STRK_TOKEN_ADDRESS {
                    (strk_total_rewards, strk_total_stake)
                } else {
                    (btc_total_rewards, btc_total_stake)
                };
                // Calculate rewards for this pool.
                let pool_rewards_including_commission = if total_stake.is_non_zero() {
                    mul_wide_and_div(
                        lhs: total_rewards,
                        rhs: pool_balance_curr_epoch.to_amount_18_decimals(),
                        div: total_stake.to_amount_18_decimals(),
                    )
                        .expect_with_err(err: InternalError::REWARDS_COMPUTATION_OVERFLOW)
                } else {
                    Zero::zero()
                };
                let (commission_rewards, pool_rewards) = split_rewards_with_commission(
                    rewards_including_commission: pool_rewards_including_commission, :commission,
                );
                total_commission_rewards += commission_rewards;
                total_pools_rewards += pool_rewards;
                if pool_rewards.is_non_zero() {
                    pool_rewards_array
                        .append(
                            (pool_contract, token_address, pool_balance_curr_epoch, pool_rewards),
                        );
                }
            }
            (total_commission_rewards, total_pools_rewards, pool_rewards_array)
